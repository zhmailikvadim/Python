Symptom

When the amount of disposition routings on a site is more than maxRowCount option value calling updateNcCode PAPI can lead to "Routing "...", type "...", version "..." does not exist (Message 14977)" error message in response. This happens by the reason that only maxRowCount rows are retrieved during disposition routings validation process.

Other Terms

NcCodeConfigurationService, maxRowCount

Reason and Prerequisites

Step to reproduce:

1. Create the amount of Special routings more than maxRowCount option value. (nwa-> Java System Properties -> Additional VM Parameters -> -DmaxRowCount parameter)

2. Create an NC Code.

3. Call updateNcCode PAPI with created NC Code and some disposition routers.

Expected result: The NC Code is updated with disposition routers from the request.

Actual result: Message "Routing "...", type "...", version "..." does not exist (Message 14977)" is received in response.

Solution

The solution is provided in a form of a code correction. Disposition routings validation logic for updateNcCode PAPI method was rewritten to retrieve and check only routers from PAPI request, whereas previously all disposition routers from a site were extracted for the validation process.

Impact

The code change implemented to address the symptom above is isolated to NcCodeConfigurationService service and updateNcCode PAPI method.

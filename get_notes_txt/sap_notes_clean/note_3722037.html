Symptom

Upon replicating employee masterdata, the following error appears in SLG1 logs:

    • "The object could not be created: The class does not exist. (CLASS=CX_SY_CREATE_OBJECT_ERROR)."
    • Error message number: HRSFEC_PTP102

Environment

  • SuccessFactors Employee Central Payroll

Cause

  • The error is caused by an entry in the customer table "T77SFEC_ADR_IMPC" for a country specific class, which does not specify the country-specific class required to process IT0006 employee data.
  • If the country is a supported country, and a default entry exists in the SAP standard table "T77SFEC_ADR_IMPS". The incorrect entry in "T77SFEC_ADR_IMPC" overrides the default configuration, leading to the replication error.

Resolution

  1. Access the customer table "T77SFEC_ADR_IMPC".
  2. Locate the entry for affected country.
  3. Delete the entry from "T77SFEC_ADR_IMPC".
  4. Verify that the system now uses the default value maintained in the SAP standard table "T77SFEC_ADR_IMPS".
  5. Test the replication process for IT0006 employee data to confirm the issue is resolved.

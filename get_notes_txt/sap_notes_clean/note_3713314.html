Symptom

  • TU-Based Integration with Extended Warehouse Management
  • Multiple SO, return SO and STO (from different plants)... are planned within a single FO -> creating delivery from FO -> "Send Load to Finalized" in Fiori or Run Batch job -> EWM activities
  • After Send Loading Instructions in Fiori or running batch job(/SCMTMS/TOR_FO_PROC_BATCH), the LDAP message does not contain all deliveries. 

Environment

SAP Transportation Management SAP S/4HANA 2021 FPS01 and subsequent releases 

Reproducing the Issue

  1. Plan orders for different customers to single FO
  2. Create delivery for the freight order
  3. Send Loading Instructions in Fiori or run loading instruction batch job
  4. EWM side activities
  5. Check LDAP message

Cause

Standard behavior.

Resolution

The "Load Plan Finalized" status for EWM use case is configured at the STOP level.  

If only a single stop status is set at the top level in the FO, the loading instruction message will contain deliveries applicable to the current stage. As you move to the next stage, the corresponding deliveries for that stage will be loaded when new loading instructions are triggered for the current stop.

Furthermore, if marking one or several main stops and set the status on the Overview tab, the system will set the status for the all selected stops (no matter which stop is next in sequence). This allows the warehouse workers to already start with picking and packing in any of the warehouses.

Symptom

You run the SLD: fichero de bases (RPC_PAYES_CRETA_BASES) program with the Reportar diferencias (P_DIFFER) checkbox selected for an employee in following conditions:

  • The employee has a retrocalculation trigged.
  • The Porcentaje de FOGASA pluriempleados (PLPOR) field of the Seguridad Social (IT0061) infotype was added in the retrocalculation period.
  • The Porcentaje de Desempleo pluriempleados (PLPDE) field of the IT0061 infotype was added in the retrocalculation period.

However, the system generates negative values for the BASE DE FOGASA (702) and for the BASE DE DESEMPLEO (701) concepts.

Example:

You reproduce the symptom by following these steps:

Description

Retrocalculation with multiemployment addition.

Preparation

  1. You have an employee with a salary of EUR 5.000.
  2. Run the payroll and the RPC_PAYES_CRETA_BASES for this employee in a period.
  3. Change the IT0061 infotype by adding values to the PLPOR and the PLPDE fields.
  4. Run the payroll again.

Procedure

Run the RPC_PAYES_CRETA_BASES program for this employee with the P_DIFFER checkbox selected.

Result

The system generates negative values for the 701 and 702 concepts.

Other Terms

&CALC - PC00_M04_CALC

Pluriempleo, desempleo, fondo de garantía salarial.

For more information, see SAP Note 1071282 - CONSULT: List of tags for Spain notes.

Reason and Prerequisites

Reason

This issue is a side effect caused by the solution delivered by the SAP Note 2742560 - SLD: The concepts 701 and 702 are not being generated in case of multiemployment.

The solution of the SAP Note 2742560 created the 701 and 702 concepts in the retrocalculation through the entry of the IT0061 infotype.

Prerequisites

The following Support Package level is the minimum prerequisite:

Release Support Package
S4HCM 102 HR SP 00
S4HCM 101 HR SP 06
S4HCM 100 HR SP 19
SAP_HR 6.08 HR SP D0
SAP_HR 6.04 (EhP6 and EhP7)  HR SP K2

For more information, access SAP ERP Human Capital Management - Support Package Schedules.

Note:

When you implement an SAP Note, some prerequisites might be required. The prerequisites are SAP Notes that have already been released and that change the same context as the current one. Then, even if these prerequisites are not necessarily related to the symptom of this SAP Note, you must implement them for technical reasons.

Solution

From now on, the system uses the percentages informed at the Seguridad Social (SV) payroll table.

Example:

Following the steps from the example of the Symptom section.

Result

The system generates positive values for the 701 and 702 concepts.

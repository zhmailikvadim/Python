Symptom

An Offline OData client is sending a request (in its default JSON format) and it expects to receive the response in pages (server side paging/ server driven paging), however the SAP Gateway doesn't send back any delta token. The same scenario is working with XML format.

Environment

SAP Mobile Platform (SMP) SDK 3.1 - Native SDK - Offline OData libraries

Cause

By default SMP SDK is sending the request in JSON format, despite the SAP Gateway is able to send back the token in JSON there might be an issue on its side as well.

Resolution

  1. Add the below lines to the Offline Store properties:
var properties = {
...
"customHeaders":
{
"Content-Type" : "application/xml",
"accept": "application/xml"
}
},
var store = sap.OData.createOfflineStore(properties);
...
  1. Define the issue on the SAP Gateway side

SAP Gateway is able to return response in JSON format, however there can be an issue with it. Checking the response writing methods on the Gateway side might help narrowing down the issue. SAP Gateway Support can help with the root cause determining if such an issue raises.

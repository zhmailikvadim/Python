Symptom

Limitation on Instance Refresh for SmartRecruiters integrated tenant using Instance Refresh tool.

Performing an instance refresh on a source or target SAP SuccessFactors instance with an active SmartRecruiters integration is currently not recommended, as it can cause the integration to fail. Potential Impacts of an Instance Refresh includes loss of SmartRecruiters access and disruption in Foundation data synchronization. 

Key Impacts of Running a Refresh

  • Loss of SmartRecruiters Access: If the SmartRecruiters switch is enabled in provisioning, users may lose access to the SmartRecruiters system after the refresh.
  • Disrupted Foundation Data Sync: Any new records created in SuccessFactors after the refresh will not be automatically synced to SmartRecruiters, potentially disrupting the Foundation data sync.

Environment

SmartRecruiters for SAP SuccessFactors

Resolution

We are aware of the current limitation affecting Instance Refresh for tenants integrated with SmartRecruiters when using the Instance Refresh tool. Our team is currently developing a permanent solution, which will be delivered in an upcoming patch.

Please monitor this KBA for updates regarding the fix and the expected patch release timeline Click on star to bookmark this article in order to receive updates about this issue. For more information about updates from subscribed KBAs, you can refer to the KBA 2171560 - How to be notified of new or updated SAP Notes or KBAs - SAP ONE Support Launchpad. 

If you still want to perform an Instance Refresh:

  • Ensure that you complete the SAP SuccessFactors HCM suite refresh before integrating with the SmartRecruiters tenant.

  • If your SAP SuccessFactors HCM suite is integrated with an active SmartRecruiters tenant, create a support ticket with Technical Support using the component MNA-SR-SER.

Symptom

In transaction RZ10, when importing profile parameters from profile files, commented lines may be incorrectly parsed as valid (but inactive) profile parameters.

This creates a risk of user accidentally activating invalid parameters that were intented to be comments only.

Other Terms

RZ10, profile import, instance profile, parameter parsing, commented parameters, inactive parameters, active profile parameters, parameter activation

Reason and Prerequisites

The parsing logic in RZ10 has a flaw where it may incorrectly identify certain commented lines as valid parameters when:

  • The profile file contains a commented line.

  • The commented line contains an equal sign (=).

  • The text before the equal sign starts with an underscore (_) followed by 2 characters.

Solution

 Apply this note or the required Support Package.

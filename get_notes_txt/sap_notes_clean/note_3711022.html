Symptom

A procedure call fails with error:

SAP DBTech JDBC: [257]: sql syntax error: search table error: sql syntax error: incorrect syntax near "$".

Note:The error above is a general error when a SQL fails. Please review the prerequisites below to determine if your issue matches the one described in this SAP Note.

Other Terms

Procedure, Syntax Error

Reason and Prerequisites

Issue number 319250

Reason:

Due to a programming error in SAP HANA, conflicting naming conventions is not properly handled during Calculation Engine code generation.

Affected Releases:

  • SAP HANA 2:
    • Revisions of SPS00 - SPS06
    • Revisions <= 079.08 (SPS07)
    • Revisions <= 089.01 (SPS08)

Prerequisites:

  • Table variables return Calculation Engine(CE) related values which produces CE code generation internally.
  • And the table variables are referenced in SQL operation such as SELECT.
  • And the table variables are declared as an output parameter.
  • The issue can be reproduce with the following example:

CREATE TABLE MYTAB(A INT);

CREATE OR REPLACE PROCEDURE PR_TABLE_OUT(OUT O1 TABLE(A INT), OUT O2 TABLE(A INT))
AS BEGIN
TABLEVAR = CE_COLUMN_TABLE(MYTAB);
O1 = SELECT * FROM :TABLEVAR;
O2 = SELECT * FROM :O1; --conflict naming convention occurs.
END;

CALL PR_TABLE_OUT(?, ?) WITH HINT(IGNORE_PLAN_CACHE);

Solution

Apply one of the following SAP HANA Revisions:

  • SAP HANA 2:
    • Revisions >= 079.09 (SPS07)
    • Revisions of SPS08: SAP is currently working on a permanent solution to this issue. This SAP Note will be updated as soon as a revision containing the solution is available
    • or higher

  • or higher

    Workaround:

    • Rewrite the procedure body to remove read dependency of parameter value for other statements.

    For example, the procedure in the prerequisites could be rewritten like following:

    CREATE OR REPLACE PROCEDURE PR_TABLE_OUT(OUT O1 TABLE(A INT), OUT O2 TABLE(A INT))
    AS BEGIN
    TABLEVAR = CE_COLUMN_TABLE(MYTAB);
    O1_1 = SELECT * FROM :TABLEVAR;
    O1 = SELECT * FROM :O1_1;
    O2 = SELECT * FROM :O1_1;
    END;

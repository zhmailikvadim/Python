Symptom

  • A business rule with Job Information Model as base object which is checking for Pay Component Group Sum field value in Compensation Information returns a null value.
  • When hiring an employee for the first time, the value is retrieved correctly correctly. However, when rehiring an employee who was previously external, the value is not retrieved correctly.

Environment

SAP SuccessFactors Employee Central 

Reproducing the Issue

  1. Navigate to the "Add New Employee" page.
  2. Fill in the details in Identity Information
  3. Accept match with new employment
  4. Enter required details in Personal Information, Job Information, and Compensation Information.
  5. Submit the hire.
  6. Check the generated workflow details and observe that the fields that are set based on Pay Component Group Sum are not populated

Cause

The issue occurs because the onSave rule on the employmentInfo element is configured as the first onSave rule in BCUI. When this rule is executed, the Pay Component Group sum fields in Compensation Information are not yet updated, resulting in a null value.

Resolution

  • Engineering team has confirmed this will be fixed in 1H 2026 release, which is scheduled for deployment in preview on April 13, 2026, and in production on May 15-16, 2026.
  • Workaround : Move the rule to the second or later position in the list of onSave rules for the employmentInfo element in BCUI.

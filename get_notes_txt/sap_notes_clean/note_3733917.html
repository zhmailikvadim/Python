Symptom

In the process "Setup / Replacement of Devices (MOS View)" (DE_B_IM_DEV_CHANGE), posting meter reading results via method UPDATE_ADD_MR_RESULTS fails in step 0390 when the material number exceeds 18 characters. The system displays the error message "Material & does not exist" from BAPI_MTRREADDOC_GETLIST and BAPI_MTRREADDOC_UPLOAD.

This issue is relevant for the market role: MOS.

Other Terms

IM4G, Setup / Replacement of Devices, DE_B_IM_DEV_CHANGE, UPDATE_ADD_MR_RESULTS, /US4G/CL_PST_IM_DEV_REPL, /US4G/CL_PST_IM_DEV_REMOVAL, BAPI_MTRREADDOC_GETLIST, BAPI_MTRREADDOC_UPLOAD, material number

Reason and Prerequisites

The error occurred because the parameter lv_matnr18 was limited to 18 characters. When the material number exceeded this length, the system truncated it, causing BAPI_MTRREADDOC_GETLIST and BAPI_MTRREADDOC_UPLOAD to raise the error "Material & does not exist".

Solution

Implement the corrections of this SAP Note or import the relevant support package.

After implementing this SAP Note, the system successfully posts meter reading results via method UPDATE_ADD_MR_RESULTS, even when the material number exceeds 18 characters. You need to create the related master data and trigger a new process to validate the fix. You can verify the fix by creating the related master data and triggering a new process, then checking that meter reading results are successfully posted via method UPDATE_ADD_MR_RESULTS, even when the material number exceeds 18 characters.

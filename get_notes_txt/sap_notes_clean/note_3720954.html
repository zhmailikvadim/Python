Symptom

I need to set-up my Ariba Network Buyer account for Poland inbound invoicing flow

Resolution

  1. Set up SAP DRC (Document and Reporting Compliance Cloud Edition)
  2. Business Network Buyer account->Default transaction rules→Attachment File Extension rules→One of the below three has to be chosen:
    Allow all attachment file extensions OR
    Allow these file extensions only (should include XML) OR
    Do not allow these file extensions (should not include XML)
  3. Tax ID set up under Buyer entitlement page. Once this is done, it will reflect under Business Network Buyer account->Settings→Account Administration→Tax IDs

  4. Business Network Buyer account-Settings->Transactions→External Tax Invoice Rules. 
    Note:  Invoices that are received from KSeF are classified as external XML invoices. These invoices are not evaluated according to the standard country based invoice rules but based on External Tax Invoice Rules. If this section is not visible, then either GLO-337 has to be enabled or the entitlement 'Enable external tax invoice rule configuration' has to be enabled.

  5. Upload vendors via VUF to avoid duplicate private records.
  6. Set up Failed External Invoice Reporting. Business Network Buyer account->Reports→Create a new report for 'Failed External Invoice'
  7. Business Network Buyer account->Transactions→External Document Service Configuration→Create a record with the country as Poland.
    Note: Ensure that you append ?grant_type=client_credentials at the end of the Token URL that you copy from the Poland Service Binding. For example, if the Token URL in the SAP BTP cockpit is https://**************.com/oauth/token, you need to configure in the Buyer account as: https://**************.com//oauth/token?grant_type=client_credentials When the External Document Service Configuration has been completed the buyer’s Tax ID will automatically be registered on their SAP Business Network account and visible under Configuration >Tax ID’s.

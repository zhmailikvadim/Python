Symptom

When try to replicate documents to CFIN target system, in transaction /AIF/ERR it shows F5152 error,

However, when reprocessing these messages, the GUIDs get to post successfully.

Error code: F5152

Error text: Document number **** was already assigned.

Environment

  • Financial Accounting (FI)
  • SAP S/4HANA 
  • SAP S/4HANA Finance
  • SAP S/4HANA Cloud Private Edition

Reproducing the Issue

Replicate the documents from CFIN source system to CFIN target system, check the replication result in transaction /AIF/ERR.

Cause

The issue with number range reuse typically arises when the current status of the number range is incorrectly modified or set to an inappropriate value.

Resolution

  1. Verify the current status of the number range in transaction FBN1 for the related company code.
  2. Ensure the number range current status is correctly set in CFIN target system, if required, make changes to the related number range accordingly.
  3. Manually reprocess the AIF GUIDs, check if the postings get proceed successfully.
  4. If there are lots of messages stuck in AIF, to automate reprocessing for such large numbers of records, configure the AIF framework to set a rule for automatic reprocessing based on the error message. Kindly contact the AIF expert for setup assistance.

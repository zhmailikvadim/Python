Symptom

When processing fiscal documents for EFD-REINF, users are unable to generate a single, aggregated event grouped by CNO (Cadastro Nacional de Obras) directly within the standard SAP Tax Declaration Framework (TDF) solution. The system attempts to generate separate events for different branches (filiais) even if they share the same CNO, leading to processing issues or rejections by the tax authority (RFB).

Environment

  • SAP ERP
  • SAP S/4HANA
  • SAP S/4HANA Cloud Private Edition
  • SAP Tax Declaration Framework for Brazil
  • TDF REINF app
  • TDF Messaging Framework

Cause

An analysis by the SAP Product team confirmed that the standard architecture of the TDF solution does not support the functionality to generate aggregated REINF events by CNO. This capability is not part of the standard product design.

Resolution

The correct procedure for this scenario requires a two-step approach:

  1. External XML Generation: The aggregated REINF event XML file must be generated outside of the standard TDF functionality. This can be achieved using a separate, third-party system or through a custom development (e.g., a custom ABAP program) that creates the file in the final format required by the government.
  2. TDF Messaging Framework for Transmission: After the aggregated XML file has been successfully generated, use the TDF messaging/monitoring functionality solely to upload and transmit this externally-created file to the government. In this case, TDF acts only as the communication and submission tool.

For detailed instructions on how to load an external or legacy event file into the TDF Messaging Framework, please refer to the following SAP Community guide:

  • TDF: Como carregar eventos legados no relatório EF… - SAP Community

Symptom

It's observed that CKMLCP has identified incorrect output material as upper level material, in detail: valuation type is not expected.

For example, according to customer's expectation, price difference should be rolled up to material A with valuation type 1. However, price difference is rolled up to material A with valuation type 2.

Environment

  • SAP S/4HANA 
  • SAP ERP

Reproducing the Issue

  1. run t-code CKM3
  2. check Consumption folder

Cause

As SAP Note 1051224 mentions. there are additional conditions to be fulfilled in case of Process category 'production'

  • At the moment a new production order is created the valuation type of the material to be produced must already be known.
  • In the creation transaction the valuation type must already be filled in the master data of the order at the moment the production process is derived program-internally. Therefore, if you enter the valuation type manually when you create the order, it may already be too late.
  • If you change the valuation type later in the order (for example, at the time of the goods receipt), the change is no longer taken into account in the production process. The goods movements are then updated in the actual quantity structure in a production process of the incorrect valuation type.

Resolution

Valuation type was changed/entered after production order creation.

Please check table CKMLMV001 and CKMLMV013.  As for table CKMLMV013, please enter parameter AUFNR as selection condition. The corresponding item has field KALNR_PROC as 123456.

Please check table CKMLMV001. As for table CKMLMV001, please enter parameter BWKEY and Material as selection condition. In result screen, please look for the item with KALNR_PROC as 123456. It is found that corresponding valuation type is 2.

Keywords

valuation type, CKM3, CO03, discrepancy, procurement process

Symptom

During retroactive processing of Donation Tax Credits and Child Support, the following issues occur:

  • The existing child support deduction maintained in Recurring Payments/Deductions (IT0014) infotype is not considered during retroactive payroll processing.
  • When a multi-decimal currency is maintained in Basic Pay (IT0008) infotype and retroactive processing of Donation Tax Credits and Child Support is active, payroll execution terminates with a runtime error.

Other Terms

  • Include: PCMASNZ0
  • Report:HNZCALC0.
  • Transaction:PC00_M43_CALC.
  • Wage Types: /RTC, /RCS, /RTF, /RTG

Type of Object Change

Workbench Objects - No
Code - Yes
Schema/Rules - No
Wage Types - No
Customizing Tables – No

Reason and Prerequisites

  • The current retroactive processing logic does not correctly evaluate the existing child support deduction records from IT0014.
  • During retroactive processing, calculated amounts are transferred to technical wage types /RTC, /RCS, /RTF and /RTG with the currency maintained in IT0008. If a multi-decimal currency is defined in IT0008, the system does not properly adjust the currency precision before populating these technical wage types. This results in inconsistent currency precision and leads to a payroll runtime error.

Solution

Enhance the retroactive processing logic in include PCMASNZ0 as follows:

  • Ensure that existing child support deduction records from IT0014 are correctly evaluated and considered during retroactive payroll processing.

  • Adjust the currency handling logic when transferring calculated amounts to technical wage types /RTC, /RCS, /RTF, and /RTG.

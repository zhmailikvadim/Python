Symptom

1. The user successfully saves a locked status in the loss dialog even though the loss has a reserve balance that is not equal to zero.

2. The user successfully saves a locked status in the loss dialog even though an account exists for the loss that has a non-final status.

Reason and Prerequisites

Prerequisites: 

  • The user changes the loss status at a specific loss level in the classic loss dialog.
  • Without saving this status change, the user then navigates to a different loss level and saves the change there.

Reason:  When you navigate to a different loss level, the check for a locked status is missing.

Solution

History:

March 3, 2026: New

09 March 2026: Clarification of the solution

Troubleshooting Description:

In the described scenario, the system issues the following message:

"Save the status change to be able to continue working".

As a result, it is no longer possible to navigate to another loss level. The system prevents you from navigating to other loss levels until the user has saved the status change or has reset a status that has not been locked.

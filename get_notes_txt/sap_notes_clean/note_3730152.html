Symptom

Change output (e.g., ORDER_CONFIRMATION_CHANGE) is triggered too often and should occur only when specified changes are made.

Environment

  • SAP S/4HANA
  • SAP S/4HANA Cloud Private Edition

Reproducing the Issue

  1. Run report SDV03V02
  2. Output ORDER_CONFIRMATION_CHANGE is triggered for some orders selected by the report but there was no relevant change.

Cause

In SPRO → Sales and Distribution → Basic Functions → SAP S/4HANA Output Control → Output Control for Sales Documents → Output for Changed Documents → Assign Output Texts to Document Data, there are entries for tables such as VBAP, VBEP, VBKD, and VBAK.

If the UPDKZ flags are set to 'U' in XVBAP, XVBEP, or XVBKD in program SAPMV45A, the output is triggered for those tables when there is a corresponding entry in Assign Output Texts to Document Data.

Resolution

Restrict the exact fields that should trigger the change output in SPRO-> Sales and Distribution → Basic Functions → SAP S/4HANA Output Control → Output Control for Sales Documents → Output for Changed Documents → Assign Output Texts to Document Data

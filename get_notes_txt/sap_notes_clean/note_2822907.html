Symptom

During the shutdown of the NameServer a crash occurs. The crash stack can vary depending on the cause, but usually contains TRexAPI::TREXIndexServer::shutdown() or TrexThreads::ThreadManager::shutdown() at a lower position on the stack:

There may also be an entry in the NameServer trace file making reference to this note:

[000000]{-1}[-1/-1] 0000-00-00 00:00:00.000000 e NameServer TREXNameServer.cpp(03526) : LocalWatchDog thread not finished in time. Deleting it anyway to force termination -> Nameserver might crash (for more details see SAP Note 2822907)

Reason and Prerequisites

Reason:

During the shutdown of the nameserver, different tasks need to be performed like closing pending network connections or canceling running threads. If one of these tasks cannot be completed, the nameserver crashes intentionally in order to complete the shutdown.

For example, if a nameserver thread is currently establishing a network connection and the network is very slow, cancelling the thread might fail since it currently is in a wait state. Therefore, the nameserver performs an intended crash, in order to avoid a hanging situation.

Prerequisites:

  1. The nameserver is in the process of shutdown 
  2. Something is blocking the normal shutdown process

Solution

This crash works as designed - it is required in order to complete the shutdown procedure in cases where tasks, that are required for shutdown, are hanging.

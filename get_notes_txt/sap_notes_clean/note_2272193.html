Symptom

In Extractor Framework you can see that EEM extractors are failing with the error:

 "Overflow when converting from -<number>

 

 

You can as well see coresponding dump:
ABAP Program Error
CONVT_OVERFLOW
CX_SY_CONVERSION_OVERFLOW
CL_E2E_TA


Environment

SAP Solution Manager 7.10

Reproducing the Issue

  1. Open http://<Solman host>:<Solman port>/sap/bc/webdynpro/sap/wd_e2e_wc_extractor_fwk page in browser
  2. Check for failing EEM extractors

Cause

The issue happens when the data collected from Managed System has exceptionally large response times for RFC calls.

The times are too large to fit into the corresponding fields.

Resolution

To resolve this issue please apply SAP Note 2071875 in the Managed System.

See Also

2071875 - Incorrect Average RFC Interface Time ("CPIC/RFC") in ST03N Workload Overview ("Average time per step in ms")

Keywords

CPIC, RFC, EEM, Business Transaction XML,

Symptom

  • In the My Inbox Fiori application, users are unable to see a list of available users when attempting to create a substitute.

  • The "Manage My Substitute" -> "Add New Substitute" dialog displays an empty list or no search results.

  • The same functionality works correctly in the SAP GUI via transaction SBWP (SAP Business Workplace).

  • The issue also impacts the ability to forward tasks to other users.

Environment

SAP S/4HANA Cloud Private Edition 2023

Reproducing the Issue

  1. Log in to the Fiori Launchpad and open the My Inbox app.

  2. Click on the user profile icon and select Manage My Substitute.

  3. Click on Add New Substitute.

  4. Observe that the user list remains empty or the search returns no results.

Cause

The discrepancy between the SAP GUI and the Fiori app exists because:

  1. SBWP uses traditional SAP Business Workplace functionality with direct backend access.

  2. My Inbox relies on the Task Gateway Service. If the "Enable User List" flag is not activated in the Gateway configuration, the service will not fetch or provide user search capabilities to the Fiori frontend.

Resolution

Step 1: Enable User List in Task Gateway Service

  1. Log in to the Frontend/Gateway system.

  2. Execute transaction SPRO.

  3. Navigate to: SAP NetWeaver → Gateway Service Enablement → Content → Task Gateway → Task Gateway Service → Task Gateway Service Settings.

  4. Check the flag Enable User List. (Note: Ensure other flags are left blank unless specifically required for other processes).

  5. If no entry exists for your service, click New Entries to create the configuration.

  6. Save the changes.

Step 2: Verify System Alias Configuration

  1. Check the Task Gateway system alias configuration in the Gateway maintenance.

  2. Ensure that the flags Local SAP GW and For Local App are set correctly according to your specific landscape architecture (Hub vs. Embedded).

Step 3: Check for BAdI Interference

  1. Verify if the BAdI /IWPGW/BADI_TGW_USER_DETAIL is implemented.

  2. If an implementation exists, ensure it is not filtering out users incorrectly. Consider deactivating it temporarily to test if it is the source of the empty list.

Step 4: Backend Configuration

  1. In the Backend system, ensure that the task classes and substitution profiles are maintained according to the SAP Help Portal guidelines for Flexible Workflow.

  2. Confirm that users exist and are valid in both the frontend and backend systems.

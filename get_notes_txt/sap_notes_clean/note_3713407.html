Symptom

You run the report /FOM/REP02 (Evaluation of Asset Retirement Obligations on Key Date) to evaluate provisions for an Asset Retirement Obligation (ARO) on a specific date.

You notice that if the Provision Key Date you enter in the report's selection screen is the exact same date as an ARO transaction (for example, an ARO Adjustment) that modified the provisions, the report displays incorrect values. The report shows the provision amount from before the transaction was applied ("Provision Old") instead of the correct, updated amount valid after the transaction ("Provision New").

Other Terms

/FOM/REP02, ARO, Asset Retirement Obligation, Provision, Key Date, Adjustment, Provision Old, Provision New, Wrong Amount, Incorrect Value

Reason and Prerequisites

The issue is caused by a logic error in the data selection of report /FOM/REP02.

When the report's Provision Key Date is identical to the ARO Processing Key Date of the latest transaction for an ARO, the report's logic incorrectly retrieved the provision values that were valid before this transaction was processed. The logic did not correctly handle this specific boundary condition.

Solution

As a general rule, SAP recommends that you implement the Support Package specified for your release. However, if you need to install a solution earlier, use the SAP Note Assistant to implement the correction instruction. You can find more information about the SAP Note Assistant in SAP Support Portal, under SAP Note Assistant.

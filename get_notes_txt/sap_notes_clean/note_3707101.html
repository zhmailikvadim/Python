Symptom

When maintain BOM containing co-product BOM items, following behavior observed: 

  • Indicator BOM is recursive (STPO-REKRI) is automatically set for specific BOM item. 
  • Error 29110 "BOM is recursive" when maintaining BOM, despite BOM seems not to be recursive.

Reproducing the Issue

Scenario 1: 

  1. Material A and B are used in BOM maintenance. Material B is as a co-product. 
  2. Create BOM for material A, first assign material B as a non-co-product BOM item. 
  3. Assign material B again as a co-product BOM item, save BOM. 
  4. Display the created BOM, indicator BOM is recursive is automatically set for the non-co-product BOM item. 


Scenario 2: 

  1. Material A, B, C and D are used in BOM maintenance. Material C and D are co-products.
  2. Create BOM for material A, assign material C as a co-product BOM item. 
  3. Create BOM for material B, assign material D as co-product BOM item. 
  4. Now try to create BOM for material C, assign material D as a non-co-product BOM item, error 29110 "BOM is recursive" occurs. 

Cause

In BOM maintenance, co-product items (STPO-KZKUP = 'X') are considered as BOM logical header. For co-product scenarios, following rules are followed in recursiveness check. This is as per the SAP standard and which is a requirement for MRP. 

  • If a co-product material (MARA-KZKUP = 'X') is maintained as a co-product BOM item (STPO-KZKUP = 'X'): recursion will be determined if the same co-product material has been maintained again as a non-co-product item. 
  • If a co-product material (MARA-KZKUP = 'X') is maintained as a non-co-product BOM item (STPO-KZKUP = ''): recursion error will be triggered if BOM header level (either the actual header or the logical header) contains co-product material, and the header level material has been used as co-product BOM items. 

 

Scenario 1: 

  • Material B is a co-product in material master, and has been maintained as a non-co-product BOM item under material A. 
  • Material B is being added again as a co-product BOM item under the same BOM, it's considered as logical header level and therefore is recursive with the non-co-product item.
  • Therefore system automatically sets the indicator BOM is recursive for the non-co-product BOM item due to the recursion found. 

Scenario 2: 

  • Material D is a co-product material and is being maintained as a non-co-product BOM item under material C. 
  • The BOM header material C is a co-product material, and has been used as co-product BOM item under material A. 
  • Therefore system will trigger error BOM is recursive. 

Resolution

System behaves as standard.

It's recommended to restructure the BOM to avoid recursion. View STPO_V02 can be used to check if the material has been used as co-product BOM items. 

If it's necessary to maintain BOM in this way, set recursive allowed indicator for such BOM items.

Symptom

SAP system messages in the unix syslog, windows event log and in SAP MMC (SID > instance > Syslog) are always classified with ERROR level, however corresponding messages have more specific log level in SM21 or different alert value in MTE node \[SID]\[INSTANCE NAME]\R3Syslog\ in Rz20.

Note: On Unix the following profile parameters need to be activated

rslg/unix_syslog/facility = LOG_LOCAL0
rslg/unix_syslog/active = true

then in following log files

/var/log/localmessages, /var/log/messages:

you can see all messages classified with the 'err' category, as long as the visibility of the log level is configured in the unix syslog. If you cannot see the log level (info|warn|err|) of the message at all, you need to customize a syslog template for the logging facility first.

Other Terms

R3Syslog

Reason and Prerequisites

This is a programming error. Please note, that SAP Kernel releases lower than 740 implement writting SAP systems messages only into the Windows Event log.

Solution

Please update SAP kernel of your system from DW.SAR.
For an overview of most recent changes in CCMS part of SAP Kernel please see SAP note 1453112.

Symptom

When attempting to upgrade SAP ASE using SAP Host Agent, the following error is logged in the dev_saphostctrl file:

[PID xxxXXXX] execute:  Execute: /bin/csh -c "source /sybase/<SID>/SYBASE.csh >/dev/null && /sybase/<SID>/ASE-16_0/bin/dataserver --sbssav"
[PID xxxXXXX] execute:  Running with pid <xxxXXXX>
[PID xxxXXXX] readOut:  Timeout: -1
[PID xxxXXXX] readOut:  TerminateOnTimeout: false
[PID xxxXXXX] wait:  Process XxxxXx exited with rc=1
[PID xxxXXXX] readOut:  Illegal variable name.
[PID xxxXXXX] readOut:  Pipe has closed
[PID xxxXXXX] readOut:  Process finished
[PID xxxXXXX] ERROR AT <Server> TIME: 2026 Jan 29 08:54:35 DESCRIPTION: Could not identify ASE version LAST OS-ERROR: (0) : Success
[PID xxxXXXX] Library function failed with error: 'Generic error', error message: 'Could not identify ASE version.'
[PID xxxXXXX] *** ERROR => 'Database live update' failed: Could not identify ASE version. [sapdbctrl.cp <port>]

Environment

  • SAP Adaptive Server Enterprise 16.0 and later with Business Suite
  • SAP Host Agent 7.22 PL69

Cause

The variable SYB_SVR was incorrectly configured in the file /sybase/<SID>/SYBASE.csh.

Resolution

  1. Locate the variable SYB_SVR in the file /sybase/<SID>/SYBASE.csh.
  2. Adjust the variable to align with the SAP ASE environment configuration.

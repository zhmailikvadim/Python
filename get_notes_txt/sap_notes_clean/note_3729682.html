Symptom

 When you use transaction EL37 to reverse combined move-out/move-in meter reading (MR) results for a bi-directional device with identical CAR values for both feed-in and consumption registers in table /US4G/MRVOBJREF, the system cancels the MR results but doesn't send a reversal message.

This issue is relevant for the market role: MOS.

Other Terms

US4G, MET, EL37, bi-directional device, MR reversal, /US4G/MRVOBJREF, /US4G/MR_RESEND, CAR values, feed-in register, consumption register

Reason and Prerequisites

The error occurred because of the current standard logic that prevented the system from sending the reversal message when processing combined move-out/move-in MR results with identical CAR values.

Solution

Implement the corrections of this SAP Note or import the relevant support package.

After implementing this SAP Note, the system adds an error message to the EL37 log panel, reminding you to use transaction /US4G/MR_RESEND to send the reversal message. You can verify the fix by checking that the EL37 log panel displays an error message when you reverse combined move-out/move-in MR data, prompting you to use /US4G/MR_RESEND to send the reversal message.

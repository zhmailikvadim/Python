Symptom

After executing employee data replication from SAP SuccessFactors Employee Central to SAP S/4HANA using SAP Master Data Integration (MDI), address information is missing in the target system. Specifically, although the standard configuration is validated and the replication process completes without hard errors, the country-specific address fields remain blank or empty in S/4HANA.

Environment

  • SAP SuccessFactors Employee Central
  • SAP MDI
  • SAP S/4HANA

Cause

This issue occurs due to a configuration mismatch between the global and local address data models in Employee Central.

In the integration logic used by MDI, all fields enabled within a Country-Specific (CSF) homeAddress configuration must also be enabled and defined in the default homeAddress configuration. If a field is active only at the country level but suppressed or missing in the default global settings, the data fails to map correctly during the replication to S/4HANA.

Resolution

To resolve this, ensure parity between your default and country-specific business configurations:

  1. Navigate to Manage Business Configuration (BCUI) in SuccessFactors.
  2. Open the homeAddress element.
  3. Review the fields enabled in your various Country-Specific address formats.
  4. Enable these same fields in the default homeAddress section.
  5. Save the changes.
  6. Perform a OData Metadata Refresh as per KBA 2171588

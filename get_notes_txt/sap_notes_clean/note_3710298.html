Symptom

When using the block function (LFA1-SPERQ) for a supplier, a configured block function that applies to requests for quotations does not work for the Manage Supplier Quotations (F1991) and Manage RFQ (F2049).

Environment

  • SAP S/4HANA
  • SAP Fiori

Reproducing the Issue

  1. Access the Manage RFQ app.
  2. Attempt to create a new RFQ for a supplier with the block function (LFA1-SPERQ) configured.
  3. The app is allowing to create a RFQ with a blocked function set for the supplier. 

Cause

The system considers a bidder blocked only if the LFA1-SPERM or LFM1-SPERM is set to X, or if the supplier's deletion indicator is set. Otherwise, the bidder is added to the RFQ, which can then be published.

Resolution

This is standard behavior.

The apps will check whether LFA1-SPERM or LFM1-SPERM is set, not LFA1-SPERQ. This was added by these two SAP notes:

  • 2928088 
  • 2944504

See Also

  • SAP Note 2944504 - Blocked bidder is getting added to Manage RFQ app
  • SAP Note 2928088 - Blocked bidder can be added to RFQ in Manage purchase requisition app

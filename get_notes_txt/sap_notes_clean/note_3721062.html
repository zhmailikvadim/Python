Symptom

Error:  “HTTP operation failed invoking https://xxx-xxx.xxxxxxxxx.xxxx/xxxxxxx 404 Not Found" Occurs When Sending a Message from Network to an External System

Cause


The HTTP 404 Not Found error occurs when the external system server cannot locate the requested resource. 
This typically happens due to one or more of the following: 

  • Incorrect endpoint URL. 
  • Resource does not exist in the target system. 
  • Service is not deployed or not active in the target system.

Resolution

  • Verify the Endpoint URL maintained under the communication details of the System connection application on the carrier tenant.

  • Review the difference in handling communication details during runtime for different APIs:

  • For Consumer API Freight Tendering and Freight Subcontracting: Maintain Base URL.
    If your API system is configured to accept a POST request at ‘https://carrier-api-system/api/v1/LbnFreightOrderForSubcontracing’, you need to capture the URL in the communication details as ‘https://carrier-api-system/api/v1/’.

  • For Consumer API Freight Booking for Confirmation, Order Tracking, and Request for Planning in External System: Maintain Entire URL.
    If your API system is configured to accept a POST request at ‘https://carrier-api-system/api/v1/FreightBookingRequest’, you need to capture the entire URL in the communication details.

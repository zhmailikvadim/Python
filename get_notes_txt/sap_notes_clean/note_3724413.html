Symptom

You are using the SAP S/4HANA migration cockpit with the "Migrate Data Directly from SAP System" approach.

After data selection completes, you are facing one of the following issues:

  • The expected migration object instances are not selected
  • An error occurs, for example "Migration object XYZ does not exist." 

Other Terms

S/4HANA Migration Cockpit, Select Data, System Upgrade

Reason and Prerequisites

For each migration object, the system generates runtime objects that are used during data selection. The identifier for these runtime objects is derived as follows:

  • The first 20 characters of the migration object’s technical name are taken as the base identifier.
  • The system checks whether a runtime object with this identifier already exists.
  • If it exists, the last two characters are replaced by a two‑digit counter to ensure uniqueness.

In certain scenarios, runtime objects can become inconsistent. Example:

  • Runtime objects for migration object A are generated and the identifiers are stored locally.
  • The system is then upgraded to a higher release, and all runtime objects of your migration projects are cleaned up.
  • Later, a new migration object B is added to the migration project, whose first 20 characters are identical to object A.
  • During generation, the system derives the first 20 characters of the technical name and checks whether the identifier is used by another runtime object. Due to the cleanup, the runtime object is no longer found, and the same identifier is reused for the new object.
  • As a result, two different migration objects end up sharing the same runtime identifier, leading to inconsistent selection behavior or “object does not exist” errors.

Solution

Implement the correction and perform the following steps:

  1. Ensure that the program CNV_PE_SUPPORT_CLEANUP_SEL_OBJ attached to this correction is installed in your system.
  2. Start transaction SE38 and execute program CNV_PE_SUPPORT_CLEANUP_SEL_OBJ to clean up inconsistent identifiers for the affected migration objects.
    • The program also registers new system tasks to force re-generation of the runtime objects in the next step.
  3. Open the SAP S/4HANA Migration Cockpit and restart the “Select Data” activity using the option “New Selection.”
    • The registered system tasks are executed implicitly at the beginning of the data selection.

Symptom

While trying to find the entry count for table COVP, COEP, COSP, COSS, etc. from SE16N giving memory dump in S/4 HANA.

Environment

  • Controlling (CO)
  • SAP ERP Central Component
  • SAP ERP
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA Finance
  • SAP S/4HANA

Reproducing the Issue

1. Run transaction code SE16
2. Enter table name COVP and input the parameters such as controlling area, company code, fiscal year. 

Cause

In SAP S/4HANA, table COVP, COEP, COSP, COSS, etc.   is no longer a physical database table. It is a compatibility view whose data is calculated on-the-fly from table ACDOCA via CDS views.

Because of this, using SE16 / SE16N to read large or unrestricted datasets from COVP is not supported and may result in SQL errors, memory issues, or dumps (such as SQL code: 4).

Resolution

Avoid reading COVP via SE16/SE16N without restrictive selection criteria

If COVP must be used, apply strong filters (e.g. OBJNR, BELNR, GJAHR, etc.)

For analysis of CO line items, select data directly from ACDOCA with suitable selection conditions instead.

This behavior and the recommended alternatives are described in detail in SAP Note 0002185026 and the related notes referenced there.

This SAP note also describes on how to select data directly  from ACDOCA table.

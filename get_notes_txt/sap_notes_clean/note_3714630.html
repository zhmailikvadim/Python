Symptom

You use the transaction MFBF to post a production by lot (WBS element) confirmation in repetitive manufacturing.

You use the transaction CJI3 to display project actual cost for the WBS element. When you display the source document for the REM confirmation, it dumps with runtime error RAISE_EXCEPTION in the program SAPLRKIO.

Other Terms

MFBF, REM confirmation, repetitive manufacturing, production by lot, WBS element, CJI3, short dump, RAISE_EXCEPTION, NOT_FOUND, SAPLRKIO, LRKIOF01, AUFTRAG_LESEN

Reason and Prerequisites

During the processing of 'Production by Lot', no product cost collector exists for the material. Therefore, the system generates a temporary order '%00000000001' in the main memory for the service posting.

Solution

Implement the attached correction instructions.

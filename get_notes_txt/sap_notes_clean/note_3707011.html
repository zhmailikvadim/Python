Symptom


cXML orders appear empty when sent from SAP Ariba to the buyer.

  • The order history displays a success message despite the orders not being sent.
  • The cXML connection URL was tested and confirmed operational.
  • A 404 error was encountered by the support engineer when verifying the cXML connection URL..
  • Error message encountered: "Error while opening connection to the server: java.util.concurrent.ConnectException: Connection timed out: ae-test.newayselectronics.com/23.97.238.186:443".

Reproducing the Issue

  1. cXML orders are sent from SAP Business Network to the buyer.
  2. Order history shows a success message, but the orders are not received.
  3. After test the cXML connection URL to confirm its operational status.

Cause

  • Connection issue due to endpoint availability or network access problem.
  • Error while opening connection to the server: java.util.concurrent.ConnectException: Connection timed out: ae-test.newayselectronics.com/23.97.238.186:443.

Resolution

  1. Verify that the AE test environment is operational at the time of the issue.
  2. Confirm that the sending IP address is whitelisted on the broker's side.
  3. Check for planned downtimes, firewall changes, or maintenance activities that could affect HTTPS connectivity.
  4. Validate the SSL certificate and supported TLS versions for the endpoint.
  5. To include IP addresses in the allow list:

a. Search for the SAP Business Network ID (ANID).  

b. Click User Details.  

c. Click Save Client IPs.  

  • To include IP addresses in the allow list for the buyer's site:

a. In Ariba Spend Management, search for the customer's site.  

b. In the search results, click [customer's name] > Edit Site Profile.  

c. Navigate to Trusted Remote Host IP Addresses.  

d. Click Add.

See Also

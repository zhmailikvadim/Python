Symptom

The Ariba API /buyer/soap/{Ariba Realm ID} is not returning an HTTP 401 error when the user authentication materials are incorrect.

Environment

Web Service

SOAP API

Reproducing the Issue

  • Check for the credentials used by the Ariba realm in SAP Integration Suite
  • Incorrectly maintain the credentials in SAP Integration Suite
  • Trigger outbound Purchase Requisition
  • Go to transaction and approve a purchase requisition

Monitor in SAP Integration Suite

  • Go to SAP Integration Suite Message Monitoring
  • Search for the iflow "S4HANA to Ariba Buying Combined Passthrough"
  • In the iFlow the call to the Ariba realm ended with HTTP 200 and the returned SOAP message with a <Faultsting> value of "AuthenticationPasswordHandler: Authentication materials are incorrect"

Cause

  • This behavior is by design. Once a request successfully reaches the SAP Ariba application server, it returns HTTP 200 (OK), with any authentication/authorization or validation issues conveyed in the response body.
  • If a request fails at the web server or gateway level, it returns an appropriate HTTP error code (e.g., 401, 402, or 403). Therefore, not receiving a 401 in this scenario is expected and working as designed.

Resolution

The system is working as designed. Only requests that pass initial authentication/flow at the web server are forwarded to the application server. Once a request reaches the application server, it returns HTTP 200 (OK), any validation or authorization issues are provided in the response body via FaultCode and FaultString.

Requests that fail at the web server/gateway level return appropriate HTTP error codes (e.g., 401 Unauthorized, 402, 403).

It is not feasible in the current design to return HTTP 401 from the application layer after the web server has accepted and forwarded the request.

You need to update your integration to parse the response body on HTTP 200 for FaultCode/FaultString and handle errors accordingly.

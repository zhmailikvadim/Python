Symptom

The SAP Document and Reporting Compliance solution for e-invoice and e-waybill for India using SAP Cloud Integration or SAP Integration Suite (PaaS model), is deprecated from SAP S/4HANA Cloud Public Edition 2508 release (July 31, 2025). As part of this deprecation, the communication scenario SAP_COM_0629: eDocument - India eInvoice Solution Integration is deprecated. 

Environment

  • SAP S/4HANA Cloud Public Edition
  • SAP Document and Reporting Compliance
  • SAP Document and Reporting Compliance, Cloud Edition

Cause

SAP has deprecated the integration flows that use SAP Cloud Integration or SAP Integration Suite (PaaS model) in the 2508 release for SAP S/4HANA Cloud Public Edition (July 31, 2025). 

  • From the 2508 release to the 2602 release (January 2026), these integration flows will remain supported, but no enhancements will be made. However, customers are encouraged to migrate to the already available successor solution, SAP Document and Reporting Compliance, cloud edition. No data transfer is required for this migration. 
  • Starting with the 2608 release, the e-invoice and e-waybill solutions using the PaaS model will cease to function, and SAP will end all support for this solution. 

Important: We encourage you to complete the migration steps to move to the solution available with SAP Document and Reporting Compliance, cloud edition (SKU 8008260) before the end of June 2026 to avoid production interruptions post 2608 release of SAP S/4HANA Cloud Public Edition. 

Resolution

Refer to the SAP Help Portal documentation for activities to be performed by your customer before migrating to the new solution. This migration does not involve any data transfer. 

  • Migration steps for E-Invoice
  • Migration steps for E-Waybill 

​​​​​​​For more information about the new successor solution (available with SAP S/4HANA Cloud Public Edition), refer to the What's New topic on SAP Help Portal. 

  • Electronic Document Processing (Electronic Customer Invoices)
  • Electronic Document Processing (Electronic Waybills) 

Follow Roadmap to know about the planned timelines for SAP Document and Reporting Compliance. 

FAQs: 

Sl.no  

Question  

Answer  

1  

Will the SAP Document and Reporting Compliance solution for e-invoice and e-waybill for India (PaaS model) support be extended beyond 2608 release (tentatively planned for July 2026)?  

No, there are no plans to extend support beyond 2608 release. 

2  

Will customers be able to migrate from the PaaS model to the SaaS model (SAP Document and Reporting Compliance, cloud edition)?  

Yes, there are few activities to be completed before migrating to the new solution. However, the migration does not involve any data transfer.  

  • Migration steps for E-Invoice 

  • Migration steps for E-Waybill 

5  

Can SAP ERP customers also use SAP Document and Reporting Compliance, cloud edition?  

No, it is planned only for SAP S/4HANA Cloud Public Edition and SAP S/4HANA Cloud Private Edition (release 2023 SPS3). This change does not affect SAP ERP customers, and they can continue using the PaaS solution. 

6  

If I am using SAP Document and Reporting Compliance for Europe or LATAM countries, can I use the same license for SAP Document and Reporting Compliance in India?  

Yes, SAP Document and Reporting Compliance licenses are valid across countries. Please contact your Account Executive for detailed licensing information.   

7  

Which GSPs (GST Suvidha Providers) are supported by SAP Document and Reporting Compliance for India?  

Vayana and Excellon are the two supported GSPs.  
For more information about third-party service providers, see Country scenarios with local / third-party providers. 

8  

Is a separate contract required with GSPs (GST Suvidha Providers) for availing their services?  

Yes.  

9  

Can the same GSP (GST Suvidha Provider) that is used for GST reporting also be used for electronic invoices and electronic waybills?  

Yes.  

10 

Is it possible to use a different GSP for electronic invoices and electronic waybills than the one used for GST reporting? 

Yes

Symptom

When reversing settlement for multiple IP documents, the system is expected to create lots based on the settlement key and pass only the IP document data relevant to the lot currently being processed to the BADI /IRM/BADI_IPBB_ALL method SETTLEMENT_BILLING_CREATE, consistent with the behavior during settlement creation. However, the current behavior results in the BADI method receiving data from all selected IP documents instead of only those associated with the specific lot.

This note corrects the behavior by ensuring that the BADI method receives only the IP document data corresponding to the lot being processed.

Other Terms

/IRM/IPBBM, /IRM/IPBB23, /IRM/BADI_IPBB_ALL, SETTLEMENT_BILLING_CREATE, IT_IPBB_DOC, CT_KOMFKGN, Reverse settle, IP document

Reason and Prerequisites

This is caused by program error

Solution

Please implement the correction instructions attached. These corrections are only valid if you have applied the following support packs.

Release 6.3 – Base Package, 6.4 Base Package and Support Pack 1

Use transaction SNOTE to implement the correction instructions specified below and activate the affected program object.

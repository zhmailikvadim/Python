Symptom

With SAP S/4HANA Cloud Private Edition 2025 FPS00 and SAP S/4HANA 2025 FPS00, the following features that are required for the integration of In-House Service with the processes for Inventory Management and Inventory (MM-IM), and Delivery Management (LE-SHP) have been provided:

  • Transfer service objects to another service center (intracompany nransfer)
  • receive service objects using inbound deliveries
  • Post service objects to customer stock (B stock)

The following feature has been updated:

  • Create outbound deliveries when sending the service objects back to the customer

To use these features, specific Customizing settings are required.

Solution

The required Customizing activities are grouped under the following Business Configuration Sets (BC sets):

  • LE_SHP_IHS_INTCO_LOGISTICS (LESHP Customizing for In-House Service(Intco Log))

The following delivery types and output types are delivered with this BC set:

Delivery Type Output Type
LFS2 (Transfer Service Object - Outbound Delivery) LFS (Outbound Delivery for In-House Service)
ELS3 (Transfer Service Object - Inbound Delivery) ELS (Inbound Delivery for In-House Service)
ELS1 (Receive Service Object - Inbound Delivery) ELS (Inbound Delivery for In-House Service)
LFS4 (Send Service Object Back to Customer - Outbound Delivery) LFS (Outbound Delivery for In-House Service)
  • MM_IM_IHR (Movement Types for In-House Service)

Once you have activated the BC sets, make the following settings for in Customizing for Service:

  1. In the Define Profiles for Logistics Integration Customizing activity, create the required logistics profiles and assign the steps required to launch the logistics processes for the service objects in the In-House Service apps.

The movement types have been enhanced so that they can be used for the logistics integration in In-House Service:

Logistics-Related Step Delivery Type Movement Type
Transfer: Create Inbound Delivery ELS3 (Transfer Service Object - Inbound Delivery) 501
Transfer: Create Outbound Delivery LFS2 (Transfer Service Object - Outbound Delivery) 601
Receive from Customer: Create Inbound Delivery ELS1 (Receive Service Object - Inbound Delivery) 501
Send Back to Customer: Create Outbound Delivery LFS4 (Send Service Object Back to Customer - Outbound Delivery) 601
    1. Only revelant for transferring service object to another service center: In the Define Item Categories Customizing activity, assign the logistics profile that you have defined in the Define Profiles for Logistics Integration Customizing activity to the item category that is used for service objects.

    Specific Settings to Post Service Objects to Customer Stock

    Service objects for which the logistics process is launched in customer returns have this special stock indicator Orders on Hand by default. However, you can specify the special stock indicator Customer Stock to be used if the logistics process is launched in customer returns. You do this in the following Customizing activities:

    • Logistical follow-up activity 0027 (Continue In-House Service): Specify Settings for Returns Orders as Follow-Up in Customizing for Service under Transactions->Settings for In-House Service
    • Logistical follow-up activity 0026 (In-House Service): Specify Settings for In-House Service as Follow-Up Activity in Customizing for Sales and Distribution under Sales->Advanced Returns Management->Settings for Repair as Follow-Up Activity

    Note: If you specify the special stock indicator Customer Stock (B stock), the system uses the movement type 431 to transfer the stock from Orders on Hand to Customer Stock once the quality inspection has been performed successfully for the service object and its availability at the service center is set to Received by Service Center.

    More Information

    For more information about implementing the logistics process for In-House Service, see the Customizing documentation in Customizing for Service under In-House Service -> Integrate In-House Service with Inventory and Delivery Management.

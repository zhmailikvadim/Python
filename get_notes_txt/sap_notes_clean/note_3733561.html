Symptom

An issue occurs when attempting to open a custom application from a Field Service Management mobile web container:

  • The web container successfully opens external website
  • The affected application returns HTTP '403 - Forbidden' when accessed through the web container.

Environment

SAP Field Service Management

Reproducing the Issue

  1. Configure a mobile web container to open a cloud‑hosted web application that requires user authentication.
  2. Launch the web container to access the application.
  3. Observe that the application does not load and returns an HTTP 403 – Forbidden error.

Cause

  • Web containers are intended to provide isolated access to external web content.
  • They do not support the embedding of interactive UI components or panels directly into native FSM screens.
  • WebContainers do not forward user authentication, session tokens, or identity information to external applications.
  • Applications that rely on authenticated access (for example, via OAuth, SSO, or platform identity services) may reject the request and return HTTP 403 – Forbidden.
  • Although external applications can be displayed, they cannot function as embedded or context-aware elements of the native FSM UI.

Resolution

  • Do not use web containers for scenarios requiring UI embedding within native FSM screens, as this is not supported by design. 
  • For applications that require authentication or deeper integration, use a Custom Extension instead of a web container.
  • Custom Extensions support fully integrated authentication flows (e.g., OAuth 2.0, IAS/SSO, client credential flows).
  • When the goal is simply to launch external applications from the mobile app, use the supported “Launch External App in FSM Mobile” capability.

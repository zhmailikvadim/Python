Symptom

  • It is unclear how the confirmation types, "Final confirmation" and "Automatic final confirmation," as well as the "Clear open reservations" indicator, affect mass confirmations performed in transaction code COHV.
  • It is unclear even after reading Note 3158588 description. 

Environment

  • SAP ERP Production Planning (PP)
  • SAP R/3
  • SAP R/3 Enterprise
  • SAP ERP Central Component
  • SAP ERP
  • SAP Enhancement package for SAP ERP
  • SAP Enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA, on-premise
  • SAP S/4HANA Cloud Private Edition

Reproducing the Issue

Post confirmation in COHV. No matter if "Switch Confirmation Fields to Changeable" is ticked or not. No matter what "Default Values" is selected. 

Cause

Standard system behavior. 

Resolution

When using transaction code COHV for mass processing, in case the confirmation type "Final confirmation" is selected, any unprocessed open reservations are cleared. "Clear open reservations" indicator is on. 

When using transaction code COHV for mass processing, in case the confirmation type "Automatic final confirmation" is selected, the functionality follows the behavior of "Automatic final confirmation." i.e., if the confirmed quantity is equal to or exceeds the originally requested order quantity, CNF status is set, and any unprocessed open reservations are cleared. "Clear open reservations" indicator is on.  If the confirmed quantity does not reach the originally requested order quantity, PCNF status is set, and unprocessed open reservations are not cleared, "Clear open reservations" indicator is off. 

See Also

Refer to SAP Note: 3158588 - Confirmations using mass processing are always posted as automatic final confirmations 

Keywords

Confirmation type, Time ticket confirmation, Enter order confirmation, mass confirmation

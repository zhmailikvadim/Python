Symptom

Cannot specify Federal Level tax Jurisdiction code individually for each Canada Report.

Environment

  • SAP S/4HANA
  • SAP S/4HANA Cloud Private Edition

Reproducing the Issue

  1. In Document and Reporting Compliance for Canada, open the customizing activity “Specify Federal Level Tax Jurisdiction Code” under the GST/HST or QST configuration path.
  2. Maintain a jurisdiction code for GST/HST and attempt to maintain a different jurisdiction code for QST.
  3. Observe that the change made for GST/HST is reflected for QST as well, preventing separate maintenance per report.

Cause

This is the expected behavior: the “Specify Federal Level Tax Jurisdiction Code” customizing is shared at federal level and updates a single configuration used system-wide; the QST report always uses jurisdiction code “CAQC” and is not controlled by that federal-level setting.

Resolution

  1. Use “Specify Federal Level Tax Jurisdiction Code” to control the GST/HST report only.
  2. If CAQC must be excluded from GST/HST, leave CAQC unflagged (do not mark it) in this federal-level activity.
  3. Do not attempt to maintain separate federal-level jurisdiction settings per report; this is not supported and changes are global by design.
  4. For the QST report, no jurisdiction maintenance is required at federal level; it always reports “CAQC”.
  5. Run both reports and verify that GST/HST reflects the federal-level selection while QST includes “CAQC” as designed.

Symptom

Facing below error after PI/PO System Copy/New Installation:
java.rmi.RemoteException: Unable to start service 'com.sap.aii.af.svc'. Reported reason: Could not initialize class com.sap.engine.httpdsrclient.protocols.instrumented.https.DSRHttpsURLConnection
at com.sap.engine.services.basicadmin.instanceadmin.InstanceAdminImpl.startService.....

Environment

  • SAP NetWeaver
  • SAP Process Integration
  • SAP Process Orchestration

Reproducing the Issue

  1. Perform a PO system copy in the new system.
  2. Attempt to execute Functional Unit Process Integration and Orchestration and its dependent packages.
  3. Observe the error: "Invoked component sap.com/com.sap.xi.ctc.app is not found."
  4. Attempt to start AS Java Service XPI Service: AF Core -> sap.com/com.sap.aii.af.svc.
  5. Observe the error: "java.rmi.RemoteException: Unable to start service 'com.sap.aii.af.svc'. Reported reason: Could not initialize class com.sap.engine.httpdsrclient.protocols.instrumented.https.DSRHttpsURLConnection."

Cause

The issue was caused due to missing LDAP configuration.

Resolution

  1. Verify the LDAP configuration in the system.
  2. Correct the missing LDAP configuration.
  3. Restart the affected services, including AS Java Service XPI Service: AF Core -> sap.com/com.sap.aii.af.svc.
  4. Confirm that the Java application com.sap.xi.ctc.app is running and the dependent services are starting successfully.

Symptom

In the Italy Self eInvoice process, the accounting document is posted successfully. However, no corresponding electronic document is generated in the system, and therefore no entry is available for display or processing in the eDocument Cockpit.

Environment

  • SAP S/4HANA Cloud Public Edition
  • SAP Document and Reporting Compliance, Cloud Edition

Reproducing the Issue

  1. Go to eDocument Cockpit application.
  2. Enter the Source Document number in the search bar.
  3. Execute the search for the document. 
  4. Observe that no corresponding eDocument is created or displayed in the eDocument Cockpit. 

Cause

Incorrect or missing system configurations

Resolution

The eDocument is not displayed in the eDocument Cockpit because it is not created, as these documents are not marked as eInvoice-relevant. This situation typically arises due to missing or incorrect configurations maintained during the system setup.

The link below provides a complete overview of the required configuration and setup:

  • Electronic Supplier Invoices

In the above links, the section “Configuration Options for Key Users” explains the required SSCUI settings and the section "Mapping Fields to the XML File" explains the required value mappings settings, which are crucial to ensure that documents are considered eInvoice-relevant, enabling successful eDocument creation and preventing future submission errors.

Note: For Self Invoice: an entry "X4" must be maintained in the SSCUI "Activate Source Type Documents for Company Code". 

Once the required SSCUI settings, value mappings, and communication arrangements are correctly maintained, the eDocument will be created and displayed in the eDocument Cockpit.

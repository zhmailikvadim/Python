Symptom

 You want to migrate spatial data from a third-party DBMS to SAP HANA and need to find the proper Spatial Reference System (SRS) for your spatial data in SAP HANA.

Other Terms

Spatial data migration, geographic spatial reference systems

Reason and Prerequisites

Types of spatial reference systems

Each geometry in a spatial database is associated with an ID that identifies its spatial reference system. The two major types of spatial reference systems are geographic spatial reference systems and projected spatial reference systems.

Geographic spatial reference systems

In geographic spatial reference systems, the earth’s surface is modelled as a rotating ellipse with a major and a minor axis called spheroid. Coordinates use angular units like degree. Besides an angular unit, those reference systems also define a linear unit, which is meter or foot in most cases. The linear unit is used in distance and area computations; for example, to measure the distance between two points on the earth’s surface.

A prominent example of a geographic spatial reference system is the “World Geodetic System 1984” with ID 4,326. This is the spatial reference system used by GPS.

Projected spatial reference systems

Projected spatial reference systems are based on geographic spatial reference systems, but define a projection on top of the geographic spatial reference system. The projection projects points from the earth’s curved surface onto a flat 2-dimensional map.

Projected spatial reference systems use a linear unit for both coordinates and distance/area computations. The linear unit is meter or foot in most cases.

Prominent examples of projected spatial reference systems are the WGS 84 / Pseudo-Mercator spatial reference system with ID 3,857 and the family of UTM spatial reference systems with IDs 32,601 to 32,660 and 32,701 to 32,760.

Spatial datatypes in SAP HANA and other DBMSs

SAP HANA offers only a single spatial data type (ST_Geometry) while some third-party DBMSs offer two spatial data types, usually named geometry and geography.

SAP HANA decides---based on the type of the spatial reference system (geographic or projected)---whether to perform its computations on a spheroid or in a flat plane.

Databases with two different spatial data types usually only execute computations on the spheroid if the geography data type is used. If the geometry data type is used, these DBMSs interpret the angular unit as linear unit and execute computations in the flat plane.

To mimic this behavior, SAP HANA offers a planar variant of all geographic spatial reference systems. Those spatial reference systems have an SRS ID offset of 1,000,000,000.

Solution

  • If you are migrating from a DBMS that offers only a single spatial data type, then you can use the SRS in SAP HANA with the same ID as in the source DBMS.
  • If you are migrating from a DBMS that distinguishes between a geometry and a geography type, then you must pay special attention when selecting the SRS in SAP HANA.
  • If you are using a geographic spatial reference system in the source DBMS (geographic spatial reference systems use coordinates with an angular unit like degree) and use the geography data type, then you can use the spatial reference system with the same ID in SAP HANA.
  • If you are using a geographic spatial reference system in a source DBMS offering both a geometry and geography type and you use the geometry type, then you must add 1 billion to the ID of the spatial reference system in the source DBMS.
  • If you are using a projected spatial reference system in the source DBMS (projected spatial reference systems use coordinates with a linear unit like meter or foot) and use the geometry datatype, then you can use the spatial reference system with the same ID in SAP HANA.

You can find information about how to create spatial reference systems in HANA at CREATE PREDEFINED SPATIAL REFERENCE SYSTEM IDENTIFIED BY <srs-id> Statement.

Example 1: You are using the geographic spatial reference system with ID 4,326 in a source DBMS that offers a geometry and geography data type, and you are using the geometry datatype. The proper spatial reference system in SAP HANA has the ID 1,000,004,326.

Example 2: You are using the geographic spatial reference system with ID 4,326 in a source DBMS that offers a geometry and geography data type, and you are using the geography datatype. The proper spatial reference system in SAP HANA has the ID 4,326.

Example 3: You are using the projected spatial reference system with ID 3,857 in a source DBMS that offers a geometry and geography data type, and you are using the geometry data type. The proper spatial reference system in SAP HANA has the ID 3,857.

Example 4: You are using the spatial reference system with ID 4,326 in a source DBMS that offers a geometry type only. The proper spatial reference system in SAP HANA has the ID 4,326.

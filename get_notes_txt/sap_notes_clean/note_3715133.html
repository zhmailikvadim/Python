Symptom

  • Opening the Joule in LoB products, Joule welcome page will not display but returns "Internal Server Error" as below:



  • The issue could also be reproduced by directly calling Joule tenant URL https://<subdomain>.<region>.sapdas.cloud.sap (This URL could be fetched in Joule subaccount -> Instances and Subscriptions -> Joule subscription).

  • By collecting the http network trace while reproducing the issue (1990706 - How to capture an HTTP trace using Google Chrome or MS Edge (Chromium) - SAP for Me), it could observe below request failed with 500 Internal Server Error:
    • https://<subdomain>.<region>.sapdas.cloud.sap/login/callback?authType=ias&code=<xxx>&iss=https%3A%2F%2F<IAS tenant endpoint>

Note: The resolution in this KBA may work only when the symptom matches exactly as above mentioned, since the "Internal Server Error" also involves other causes as per the following KBAs:

  • 3698582 - Internal server error when opening Joule - SAP for Me
  • 3668122 - SF integrated Joule teant URL still pointed to the old Joule tenant URL caused error when open Joule in SF - SAP for Me

Environment

  • Joule
  • SAP Cloud Identity Services

Reproducing the Issue

  1. Open the Joule in Lob product / Open the Joule tenant endpoint directly.

  2. User will be redirected to IAS for authentication.

  3. Post authentication, error will display.

Cause

One of the possible causes of this issue is due to "More than 1 user uuids found for the IAS login user". However Joule only accepts single user_uuid value in the received jwt token for login user. If the login user contains multiple values for user_uuid attribute, 500 internal server error will be returned.

Resolution

  1. Go to IAS Admin Console which is associated with Joule subaccount.

  2. Open Joule IAS application named "das-ias (<subaccount name>)".

  3. Navigate to Trust -> Single Sign-On -> Attributes, ensure "user_uuid" attribute is maintained with only one key-pair as "Identity Directory: Global User ID".
    If there are any redundant values, remove them. (It's essential to internally confirm that the extra values do not have any impact on the business flow, such as role collection mapping is relying on this attribute value).
  4. Save the change and test again in fresh session.

Symptom

After activation Rounding Residual at warehouse level. map the ERP reason codes to reasons for movement in Customizing for Extended Warehouse Management under Interfaces > ERP Integration > Goods Movements > Map EWM Reason Code to ERP Reason for Movement.

While posting WT the reason code is not copied 

Environment

  • S/4HANA
  • SAP Extended Warehouse Management (SCM-EWM)
  • SAP Extended Warehouse Management - Interfaces (SCM-EWM-IF)

Reproducing the Issue

  1. Execute Tx SE16
  2. Enter /SCWM/ORDIM_C
  3. Enter Warehouse task 
  4. Check the field Reason

Cause

The standard reading rounding residuals reason code customizing only from t-code /SCWM/DIFF_ANALYZER. 

Resolution

  • The standard reading rounding residuals reason code customizing only from t-code /SCWM/DIFF_ANALYZER. This is the standard t-code where the mapping considered.
  • One must implement the BAdI /scwm/ex_erp_goodsmvt_ext if you want to use the /SCWM/ERP_STOCKCHECK. 

See Also

SAP NOTE 3138922 - Rounding Residuals (aka. "Decimal Dust") - Consulting Note 

Keywords

Reason code, standard, /SCWM/DIFF_ANALYZER, interfaces, Reason for Movement

Symptom

SAP HANA database data backup via backint execution time is taking longer than before. Backup.log also indicates increased execution time.

backup.log

<T1-Timestamp>   BACKUP   SAVE DATA started [<backup_id>]
<T2-Timestamp>   BACKUP   SAVE DATA finished successfully

SAP Note - 1835075 has been followed but no issue is found on SAP HANA side.

Environment

SAP HANA 2.0 Data Backup with Third-Party Backup Tools

Reproducing the Issue

Perform a complete full data backup with backint to a NAS file system destination 

Cause

Issue from NAS storage

Resolution

This is not a SAP HANA DB issue or third party backint tool issue. Restart NAS storage controller may resolve this decreased data backup performance issue.

Keywords

NAS, storage controller

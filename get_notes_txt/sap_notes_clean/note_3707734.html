Symptom

  • SAP Commerce Cloud system is unavailable
  • Persistent 403 error for endpoints due to IP allowlist issues.

**Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.**

Reproducing the Issue

Attempt to access endpoints; observe 403 error responses even though IP is correctly added in IP filter set.

Cause

Apache configuration of IP allowlist misconfiguration resulting in 403 error responses for endpoints.

httpd.conf
Require expr %{SSL_CLIENT_VERIFY} == "SUCCESS"

This condition enforces mutual TLS (mTLS), meaning that every client accessing the endpoint must present a valid client certificate that is trusted by the server. For this to work, a Certificate Authority (CA) must be configured and trusted in the Cloud Portal so Apache can verify incoming client certificates.

Resolution

To resolve this, one of the following approaches is required:

  1. Remove the client-certificate requirement (this section: Require expr %{SSL_CLIENT_VERIFY} == "SUCCESS")
  2. Add CA certificate for Client

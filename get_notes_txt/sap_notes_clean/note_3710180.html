Symptom

Error "Kit price in the header differs from the price in the item" in Purchase Request ABC (ABC represents Purchase Request ID) after adding new item in Sales Order.

Environment

SAP Business ByDesign

Reproducing the Issue

  1. Go to Purchase Requests and Orders work center.
  2. Go to Purchase Requests view.
  3. Search with Purchase Request ABC.
  4. Choose Edit button.
  5. Select Check button.

Error appears "Kit price in the header differs from the price in the item".

Cause

This applies for Third-Party scenarios in which the Purchase Request ABC was created from Sales Order XYZ. This Sales Order contains kit items with fulfillment set to “External". When Purchase Order EFG is created and ordered, a new item is added to the Sales Order. Then in Purchase Requests ABC the line items' net value resets to zero. 

There is some Inconsistency in the system, and the issue is under investigation.

Resolution

Our development team has taken it to the backlog. It will be implemented in the future release based on priority.

Follow the below steps as a workaround,

  1. If any Items needs to be added before the creation of Purchase Order EFG, add items to the Sales Order XYZ and save it.
    Or 
  2. Add all items to the Sales Order XYZ before releasing Sales Order XYZ.

After Purchase Order creation if any Items needs to be added, kindly try to create new Sales Order and create the Respective items.

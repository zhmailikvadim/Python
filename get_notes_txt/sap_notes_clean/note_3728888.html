Symptom

  • The error message indicates "Forbidden access. Ensure the userKey has the right permissions."
  • The issue was observed after saving OAuth2 credentials in the Emarsys application in the SAP Customer Data Platform console

Environment

SAP Customer Data Platform

Reproducing the Issue

  1. Create an API user as OpenID OAuth2.0 type in the SAP Emarsys system
  2. Enter the created credentials (clientID and secret) in the Emarsys application in SAP Customer Data Platform console
  3. After saving the credentials in the Emarsys application, you will encounter the error as follows: Forbidden access. Ensure the userKey has the right permissions

Cause

The OAuth2.0 userKey configured in the Emarsys application in SAP Customer Data Platform console is missing the necessary permissions

Resolution

  1. Review the permissions assigned to the OAuth2 userKey in the SAP Emarsys System.
  2. Refer to the below documentation for guidance on which permissions needs to be assigned to the userkey when integrating with SAP Customer Data Platform: Integration with SAP Customer Data Platform | Connecting Systems.
  3. After updating the permissions, re-enter the OAuth2 credentials in the Emarsys application under SAP Customer Data Platform console.
  4. This should successfully save the OAuth2.0 credentials without any error.

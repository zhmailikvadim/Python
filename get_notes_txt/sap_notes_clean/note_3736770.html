Symptom

  • An error message, "Sorry, it isn't possible to view data since the record was deleted in the meantime," is displayed when manually updating the Job Information History after creating a new Work Order using the Take Action menu in People Profile.
  • The error occurs when a new Work Order is created with a Start Date immediately following the End Date of the previous Work Order, and Job Information is manually updated with an Effective start Date equal to the new Work Order Start Date.
  • Despite the error message, the new record is saved in Job Info.

Environment

SAP SuccessFactors Employee Central: Contingent Workforce

Reproducing the Issue

  1. Search for the user in People Profile.
  2. Use the Take Action menu to create a new Work Order.
  3. Set the Start Date of the new Work Order to immediately follow the End Date of the previous Work Order.
  4. Set the End Date of the new Work Order.
  5. Save the new Work Order.
  6. Manually update Job Information with the Effective start Date equal to the new Work Order Start Date.
  7. Save the Job Information record.
  8. Observe the error message: "Sorry, it isn't possible to view data since the record was deleted in the meantime."
  9. Open the Job Information history and observe that the records are created successfully.

Cause

This is known issue. 

Resolution

  1. The fix for this issue is planned to be deployed on 2H 2025 release (b2511).
  2. For release timelines, please review SAP SuccessFactors Product Release & Road Map Information
  3. Until the fix is delivered, verify the Job Information history after saving to confirm the record exists.

Symptom

You are using aATP check in PP/DS. Execution of Availability Check for LPP leads to dump because of assertion failed in class CL_ATP_RESULT_ITEM method ASSGN_EXT_CONF_2_BM_REQ_W_MAP.

Other Terms

S/4HANA PP/DS, aATP, Characteristic Evaluation, Availability Check, EOGL, LPP

Reason and Prerequisites

The issue stems from the discrepancy between the request timestamp and the resource operating hours in PPDS. While the request is always generated for the end of the day, resource availability varies. This creates a scenario where the product is available earlier than requested. This behavior is inconsistent with aATP logic, which does not permit confirmations to occur prior to the requested date/time.

An extra check for the availability time has been added to address the discrepancy

Solution

Implement the correction instructions.

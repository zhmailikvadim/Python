Symptom

In SAP Document and Reporting Compliance (DRC) the following behavior is observed when Purchase Orders are changed after the initial electronic document is generated or transmitted:

  • The system does not handle changes to electronic Purchase Orders (PO) in a flexible manner after the initial eDocument has been generated or sent.
  • There is no automatic mechanism to generate and transmit structured Order Change messages when a PO is modified.
  • The system does not support automatic regeneration and controlled resending of updated XML documents for modified PO.

Environment

  • SAP S/4HANA Cloud Public Edition
  • SAP Document and Reporting Edition, Cloud Edition
 

Reproducing the Issue

  1. Generate an electronic Purchase Order (PO) via SAP Document and Reporting Compliance (DRC).
  2. Modify the PO after the initial eDocument has been generated or sent.
  3. Attempt to automatically generate and transmit a structured "Order Change" message for the modified PO.
  4. Attempt to regenerate and resend an updated XML document in a controlled and compliant manner.

Resolution

 Currently, SAP does not natively support structured handling of Purchase Order changes within SAP Document and Reporting Compliance.

  1. Order Change message support

    SAP DRC does not currently provide end-to-end support for Peppol BIS Order Change (T114) or similar procurement lifecycle messages.

  2. Electronic PO lifecycle management

    There is no officially published roadmap item specifically addressing electronic PO lifecycle management (changes, cancellations, resubmissions) within DRC in the SAP public roadmap or SAP Notes.

  3. Recommended architecture

    Customers who require full procurement message lifecycle management may consider the following approach:

    • Use DRC mainly for regulatory electronic documents (for example e-invoices).

    • Handle procurement message exchanges (PO changes, cancellations, acknowledgements) through:

      • SAP Ariba Network, or

      • Middleware/integration platforms (e.g. SAP Integration Suite), or

      • Custom integration with Peppol access points.

  4. Reference documentation

    For supported scenarios and compliance capabilities, refer to the SAP Help Portal documentation for:

    • Submitting Electronic Purchase Orders Using SAP Document and Reporting Compliance, Cloud Edition | SAP Help Portal. 

    • Supported Compliance Tasks by Country/Region | SAP Help Portal.

    • Regulatory Change Manager for upcoming regulatory updates - Regulatory Change Manager.

Symptom

System currently allows the user to enter payment date in disbursement order to a date which is before the bank posting area date. In usual business scenarios, the payment date is equal or greater than the bank posting area date.

In rare cases the payment date can be before the bank posting area date. To provide this flexibility, a customizable error message is now provided which can be set to either 'Error' or 'Warning' based on the bank's business scenario. In case of Error, the user will not be allowed to Save or Activate the disbursement order until the payment date is changed to a correct date.

Other Terms

Payment Date, Disbursement Order, BCA_OR_DISB.

Reason and Prerequisites

Program Error.

Solution

It is recommended to import the relevant Support Package for your release.

You can use transaction SNOTE to implement the correction instructions in advance.

If you implement the advance corrections, you must also perform manual activities.

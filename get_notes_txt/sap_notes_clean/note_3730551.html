Symptom

  • The system does not create automatic transportation lanes when Purchasing Info Records (PIR) are transferred from S/4 to OBP.
  • Error message observed: “Subcontracting Prod. Vers. (PRODUCTIONVERSIONID) must refer to an existing PDS”.

Environment

  • SAP Integrated Business Planning for Supply Chain (SCM-IBP)

Reproducing the Issue

  1. Transfer PIR from S/4 to OBP via the integration model.
  2. Check OBP and observe that automatic transportation lanes are not created.

Cause

The PIR integration payload references subcontracting production versions that do not exist in the production data structures, or invalid PIRs are included, causing the message to fail and preventing automatic transportation lane creation.

Resolution

  1. Archive PIRs that are obsolete or have deletion flags set
  2. Restrict the integration model to include only valid PIRs for activation
  3. If the PIRs are valid, ensure the referenced subcontracting production versions are correctly set up:
    1. Verify the objects are part of an Integration Model for PDS
    2. Confirm the PDS exist in the “View Production Data Structures” app

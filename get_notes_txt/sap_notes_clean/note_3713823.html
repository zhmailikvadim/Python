Symptom

When trying to open an existing marketing document by document number, the document cannot be opened with the following error:
No matching records found  'document type' (OXXX) (ODBC -2028)  [Message 131-183]

Environment

SAP Business One

Reproducing the Issue

Scenario:

  1. Go to Sales - A/R → A/R Invoice, set Item/Service Type as Item and create an item type A/R invoice.
  2. Change the A/R Invoice window to Find mode and set Item/Service Type as Service.
  3. In No. field, input the document number of the above A/R Invoice created in step 1.
  4. Click Find button.

Expected behavior:

The A/R Invoice is opened without any error.

Actual behavior:

The A/R Invoice cannot be opened with the follow error:
No matching records found  'A/R Invoice' (OINV) (ODBC -2028)  [Message 131-183]

If trying to find a Service type document under Item type window, the same error will also be displayed.

Resolution

To open an existing Item type document, ensure that Item/Service Type is set to Item in Find mode. To open a Service type document, ensure that Item/Service Type is set to Service in Find mode.  

Keywords

A/R Invoice, Item/Service Type, Item, Service, Find, No., No matching records found

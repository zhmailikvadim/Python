Symptom

In SAP Adaptive Server Enterprise (ASE), a database session appears in status recv sleep with command AWAITING COMMAND, while still holding transaction log resources.
When running:

dbcc sqltext(<spid>)

Environment

SAP ASE 16.0

Cause

This behavior is by design in SAP ASE.

When a session is in recv sleep / AWAITING COMMAND state:

  • The last SQL statement has already finished execution.

  • The server is waiting for the next request from the client.

  • There is no active SQL statement associated with the session.

The command dbcc sqltext(spid) only displays the SQL text of a currently executing statement.
Once execution is complete, the SQL text buffer may be cleared or reused, which explains why dbcc sqltext may return:

  • No output, or

  • Only a short fragment of text.

Despite being idle, the session may still hold transaction log space if the client:

  • Executed a data modification statement (INSERT/UPDATE/DELETE), and

  • Did not issue a COMMIT or ROLLBACK.

In this case, the open transaction remains active, and log space is retained until the transaction is completed or the connection is terminated.

Resolution

There is no database-side fix required.

To prevent transaction log retention:

  • Ensure applications explicitly issue COMMIT or ROLLBACK after DML statements.

  • Avoid leaving database connections idle with open transactions.

  • Review application logic and connection pooling behavior.

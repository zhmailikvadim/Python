Symptom

Privacy by design and default is based on the purposes (describing the reasons) for processing of personal data. Application area "Info Record" (MM-PUR-VM-REC) supports the determination and assignment of purposes to person-related data by the enablement for the Purpose Determination. The Purpose Determination is the central instance between the purpose configuration and the applications to asynchronously determine and update the purpose assignment for business documents.

You can find further details in the SAP Help Portal (https://help.sap.com) by searching for Purpose Determination.

In the following table you can find details about the Purpose Determination enablement delivered in application area "Info Record" (MM-PUR-VM-REC).

Other Terms

S/4HANA Cloud, Data Controller, BP, Business partner, Purpose Determination, MM_EINA, PURCHASINGINFORECORD

Reason and Prerequisites

Purpose agent processing has been enhanced with a predelivered purpose. The structure PURCHASINGINFORECORD_RULES provides the purpose context field "Purchasing Group". A BRF+ workbench tool can be used to configure the purpose.

MM_EINA

Assigned data controller is determined based on the organizational configuration of the purchasing document.

Purpose Determination worklist is triggered during the creation of a purchasing info record and in case of changing a purchasing info record with respect to business partner data - supplier / prior supplier / manufacturer.

The application provides the purpose class CL_MMPUR_PURPOSE_DET_MM_EINA alongside the data category and data controller to determine purposes specifically for each related business process variant.

The purpose determination can be influenced using customer-specific BRF+ rules. The structure: PURCHASINGINFORECORD_RULES provides the purpose context field "Purchasing Group".

Symptom

You are running a system installation of AS Java High-Availability First Cluster Node.
Installatino is failing during setPermissionsFS with following error:


-------------------------------------
[...]
File system export (share) sapmnt does not exist.
Failed action:  with parameters 
Error number 2310 error type WIN32_ERROR
[...]
-------------------------------------

 

Environment

AS Java High-Availability systems on Windows Server

Cause

The reason for the mismatch in the sapmnt directory is an override of the profile parameter SAPGLOBALHOST during Initialization of the SCS instance in the dialog phase.

Within the sapinst_dev log, You can see that the property SAPGLOBALHOST  is set several times to your virtualhost.   

sapinst tries to get the Fileshare “sapmnt” from your virtualhost (\\<virtualhost >\sapmnt) but sapmnt is located on the server sapglobalhost (\\sapglobalhost\sapmnt):

NWInstall.prototype.setPermissionsFS = function(sharePath) {

// […]

var globalHost = this.getProfile().resolveExtended("SAPGLOBALHOST").value();

// […]

var share = mgt.getFSExportHost(globalHost, "sapmnt");

if (share == undefined) {

       installer.writeTrace("mf trace: couldn't get share info");

Resolution

The issue has been already identified and fixed in the newest  verson of SWPM10.

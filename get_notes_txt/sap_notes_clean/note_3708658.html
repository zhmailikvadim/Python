Symptom

While executing the query on a calculation view which is using function MATCH( ) with multiple column arguments , the query fails with the following error:

--------------------------------------------------------------------------------------
Unallowed number of parameters for function "TK_MATCH_FUNCTION".
Should be 2, 3, but was 10

calcEngine search on olapIndex failed
--------------------------------------------------------------------------------------

Environment

SAP HANA, PLATFORM Edition 2.0

Reproducing the Issue

  1. Go to HANA View
  2. Right click View > Select Data Preview
  3.  Prompt Values:-
  4.  Go to Raw Data tab :- 
    Problematic Filter Expression in detail  

Cause

 Some SQL expression involving function "MATCH()" used in the calculation view has IN operation, but it does not need IN operation.

Resolution

Modify the SQL expression like below:

From ->
(not in (match("MEASURE",'CCLM*','CONVFP*','FAFP*','GAAP*','INVFP*','LHFFP *','PAY*','PRSG*','TRFR*')))

To ->
(not match("MEASURE", 'CCLM*','CONVFP*','FAFP*','GAAP*','INVFP*','LHFFP *','PAY*','PRSG*','TRFR*'))

Keywords

MATCH(),  Data Preview, not in (match

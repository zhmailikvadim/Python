Symptom

TSV_TNEW_PAGE_ALLOC_FAILED in SAPTUNE_GEN_BUFF_OBJTS_LCL


Other Terms


DATA=LT_GENERIC_REGIONS


Reason and Prerequisites


The dump occurs inside of the call of FM SAPTUNE_GEN_BUFF_OBJTS_LCL.(eg: table DDFTX).
Despite of recent optimalization of RSSTAT90 and relevant code used the dump still occurs
The root cause is structure RSTBXOBUF64 (CHAR 900,internal length 1800) which allocate big portion of local memory.


Solution


Several scenarios are possible to avoid the dump:


1/


Implement attached correction instruction.


2/
upgrade to kernel 785 and SAP_BASIS 816 or higher.
The fix contain new data type RSTBXOBUF64STR (SSTRING 900,internal length 8) that consume less memory and has been introduced for this purpose.
3/ 
suppress of RSSTAT90 call in ST03.
ST03>Collector and Performance DB>Performance monitor Collector>Execution Times. Here suppress call of RSSTAT90


4/
Deactivation is possible via profile parameter rsdb/staton to "0". (extended table access statistics).


Until the upgrade of kernel and SAP_BASIS there is only workarroud how to fix the current issue.

Symptom

In the process "Receipt of Quotation for MOS Billing (SUP View)" (DE_B_MOSB_QUOT_SUP), the DTD check step 50 - "Do the offered items correspond to the contractual relationship?" (with rejection code A03) using code list E_0208 was not supported. An extension of the check group in the business system with custom implementation was needed.

This issue is relevant for the market role: SUP.

Other Terms

DTD, MOSB_SUP, E_0208, A03, quotation, 15002, CHK_MOSB_QUOTES_DTD_0208_CUS_EXT, /US4G/CL_CHK_MOSB_PROCESS, CHK_QUOT_REF_REQT, WrongQuotationItemForReqQuotMOE

Reason and Prerequisites

Program error and configuration error.

Solution

Implement the corrections of this SAP Note or import the relevant support package.

After implementing this SAP Note, the system supports the DTD check step 50 - "Do the offered items correspond to the contractual relationship?" (with rejection code A03) using code list E_0208 in the process "Receipt of Quotation for MOS Billing (SUP View)".

To extend the check, follow these steps:

1. Open the definition of check CHK_MOSB_QUOTES_DTD_0208_CUS_EXT (E_0208(A03): Check if Quote Offer items deviating from the contractual relationship) in the Fiori app Configure Checks.

2. Select Extend.

3. Assign your own check class and check method.

4. For the implementation of the check method, refer to check method CHK_MOSB_QUOTES_DTD_0208_CUS_EXT of check class /US4G/CL_CHK_MOSB_PROCESS.

When implementing the check logic:

- Implement the logic to check DTD check step 50.
- If the check fails, set the UBD value of E_0208 A03 "WrongQuotationItemForReqQuotMOE" into UBD structure /US4G/UBD_DATA-INFO_EVENT-RESPONSE_STATUS_LIST and set the check result as "REJECT".
- If the check passes, don't set INFO_EVENT data and set the check result as "CONFIRM".

You can verify the fix by checking that the system supports the DTD check step 50 with rejection code A03 using code list E_0208 in the quotation receipt process.

Symptom

During the replication of the TELNR field to IT0006 for employees in Canada or the United States, the system triggers the error message "Telephone number must consist of seven digits" (Message U5044).

Environment

  • SAP SuccessFactors Employee Central Payroll
  • Employee Data Replication

Reproducing the Issue

  1.  Go to SE38
  2.  Select report RP_HRSFEC_PTP_EE_REPLICATION
  3.  Fill the mandatory fields
  4.  Check SLG1 for errors

Cause

There are country-specific validation for IT0006 (method CHECK_TELNR) requires the local phone number (TELNR/PHNNR) to contain exactly seven digits and only numeric characters when an Area Code is provided separately; non-digit characters (for example, hyphens) or an incorrect length trigger the error.

Resolution

To resolve the issue, ensure that adjustments to the phone_number field in the Phone Information portlet comply with country-specific validation rules, including the proper inclusion of the area code within the phone number.
Additionally, if the TELNR field is maintained via a BAdI, review and update the custom implementation to align with the required phone number format and validation criteria.

Symptom

The creation date in SD document flow is incorrect for the already archived accounting document.

Environment

  • Sales And Distribution (SD)
  • SAP ERP Central Component
  • SAP ERP
  • SAP Enhancement package for SAP ERP
  • SAP Enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA
  • SAP S/4HANA Cloud Private Edition

Reproducing the Issue

  1. run transaction VF03
  2. enter a billing document with accounting document
  3. display document flow for the billing document
  4. check the creation date of the accounting document, it is the same as the header entry date
  5. archive the accounting document 
  6. check the creation date again in billing document' document flow, the creation date is changed

Cause

This is the correct system behavior.

Resolution

When displaying the creation date for accounting document in SD document flow, system is looking for the table BKPF field CPUDT(Day On Which Accounting Document Was Entered). Bur for the archived accounting document, there is no BKPF table entry existed. In such case, system will directly use the billing date (VBRK-FKDAT) for this purpose.

Symptom

On HDB, expert function List open db transactions (report /NZT/SHOW_DB_LOCKS_WP) runs very long and may time out. If it finishes, the output may have many identical lines.

If, for example, a single transaction updates <n> rows of table <a>, then the output will include <n> lines with the same information, one line per row lock. However, there is no added value in listing all row locks held by a single transaction on a table.

After implementing this note, the result set will only include one line per table and transaction. In the above example, there would one single row for table <a> in the result set.

Other Terms

downtime minimization tool for SAP S/4HANA, NZDT

Reason and Prerequisites

Any of the following add-on support packages is installed in your system landscape:

  • NZDT 1.0 SP02

Solution

Required Action

In order for this note correction to be effective, the note must be implemented before:

  • expert function List open db transactions (report /NZT/SHOW_DB_LOCKS_WP) is used.

Implement the correction instructions in transaction SNOTE on the following systems of your system landscape:

  • Sender system
  • Receiver system

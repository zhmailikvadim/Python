Symptom

During API Management transport one gets this error:

CTS+ deployment for "AccessCont" failed: "Content deployment for module "AccessControl" to resource "reserved_for_apimgmt_transport" failed: CPI deployment failed: [VIRTUALHOST_NOT_FOUND_ERROR] Error details: Missing configurations; Please contact support team to configure virtual host. Error message: Missing configurations; Please contact support team to configure virtual host. "

Environment

  • SAP Integration Suite
  • API Management

Reproducing the Issue

Transport an API Proxy in API Management.

Cause

Virtual Host needs to be set to default.

Resolution

The key part is VIRTUALHOST_NOT_FOUND_ERROR. In SAP API Management, an API proxy can only be deployed to an environment that has at least one virtual host assigned. Your transport is trying to deploy to a technical environment/resource named reserved_for_apimgmt_transport, but that environment in the target tenant has no virtual host configured. As a result, deployment fails.

Refer to the following guide for detailed steps: Configuring a Default Domain for a Virtual Host

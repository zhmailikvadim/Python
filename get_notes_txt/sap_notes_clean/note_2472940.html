Symptom

How do you setup SDK Python with Kerberos using a DSN connectionstring?

Environment

  • SAP Sybase Software Developer Kit (SDK) 15.7
  • SAP Adaptive Server Enterprise (ASE) Software Developer Kit (SDK) 16.0
  • SAP Adaptive Server Enterprise (ASE) 15.7
  • SAP Adaptive Server Enterprise (ASE) 16.0
  • Open Client Client-Library
  • Adaptive Server Enterprise extension module for Python
  • ctlibrary
  • Python

Resolution

  1. Go to your SDK client install
    cd $SYBASE

  2. Set Sybase environment variables
    source SYBASE.csh

  3. Check your python version
    python -version
    Python 3.4.4 (default, Mar 13 2017, 10:21:15)
    [GCC 4.4.7 20120313 (Red Hat 4.4.7-17)] on linux

  4. Set PYTHONPATH environment variable for your specific python version
    setenv PYTHONPATH $SYBASE/$SYBASE_OCS/python/python34_64r/lib

  5. Go to your python application
    cd $SYBASE/$SYBASE_OCS/sample/python

  6. Modify python application's connectionstring
    conn = sybpydb.connect(dsn='SecurityMechanism=csfkrb5;Kerberos=serverPrinicpal@REALM;MutualAuthentication=true')

    Note: kerberos is equal to your server / service princpal name. By default this is your ASE name. If it isn't your ASE name it will be the same as your RUN_asename file -k option
    $SYBASE/$SYBASE_ASE/bin/diagserver \
    -sasename \
    -i$SYBASE \
    -N$SYBASE/$SYBASE_ASE/sysam/asename.properties \
    -d$SYBASE/data/master.dat \
    -e$SYBASE/$SYBASE_ASE/install/asename.log \
    -c$SYBASE/$SYBASE_ASE/asename.cfg \
    -M$SYBASE/$SYBASE_ASE \
    -kkrbPrincipal@REALM

  7. Generate the user credential
    kinit krbuser@REALM

  8. Set DSQUERY environment variable to your ASE interface name
    setenv DSQUERY asename

  9. Run the application
    python firstapp.py

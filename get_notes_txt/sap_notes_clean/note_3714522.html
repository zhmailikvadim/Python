Symptom

  • When selling shares under Product Type 16A, the system calculates the realized gain/loss using the FIFO method, even though no consumption sequence has been assigned.

Environment

SAP S/4HANA Cloud Public Edition

Cause

The system calculates the realized gain/loss for Product Type 16A using the FIFO method because Product Type 16A has been maintained in the Single Position Management section under "SSCUI 104115 - Define and Assign Differentiations."

Resolution

To switch to the Weighted Average method, follow these steps:

  1. Access the configuration "SSCUI 104115 – Define and Assign Differentiations."
  2. Locate the Single Position Management.
  3. Delete the relevant settings for Product Type 16A.
  4. Save the changes and ensure that no consumption sequence is assigned for Product Type 16A.

Keywords

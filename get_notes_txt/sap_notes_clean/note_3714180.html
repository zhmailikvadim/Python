Symptom

Event Management API returns empty payload using identifiers endpoint

Reproducing the Issue

  1. Execute a GET request to the Event Management API using a standard API user.
  2. Replace the placeholders with valid realm, eventId, token, and API key.
  3. Submit the request below:
    curl --request GET \
      --url 'https://openapi.ariba.com/api/sourcing-event/v2/<environment>/events/identifiers?%24filter=<filterExpression>&%24top=10&%24skip=0&realm=<yourRealm>&user=<yourUser>&passwordAdapter=<yourAdapter>' \
      --header 'Authorization: Bearer <yourOAuthToken>' \
      --header 'accept: application/json' \
      --header 'apiKey: <yourApiKey>'
  4. Observe the response:
    Status code: 200 OK
    Body: Empty payload (no identifiers returned)

Cause

  • The endpoint GET /events/{eventId}/identifiers only supports filtering when executed using the service user aribasystem.
  • When the request is performed using any other user, the filter logic does not apply, resulting in an empty response.

Resolution

To successfully retrieve identifier data, execute the same request using the aribasystem service user.
Example request (the only required change is the user):

curl --request GET \
  --url 'https://<datacenter>.openapi.ariba.com/api/sourcing-event/v2/<environment>/events/identifiers?%24filter=<filterExpression>&%24top=10&%24skip=0&realm=<yourRealm>&user=aribasystem&passwordAdapter=<yourAdapter>' \
  --header 'Authorization: Bearer <yourOAuthToken>' \
  --header 'accept: application/json' \
  --header 'apiKey: <yourApiKey>'

Once the request is executed with aribasystem, the endpoint returns the expected identifier data.

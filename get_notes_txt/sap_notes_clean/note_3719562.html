Symptom

  • A vulnerability in the Apache log4j-jul.jar library was identified during a vulnerability scan on the Data Provisioning Agent.
  • The scan recommended updating the log4j-jul.jar library to version 2.25.3 from the installed version 2.25.0.

Environment

SAP HANA smart data integration

Reproducing the Issue

  1. Perform a vulnerability assessment scan on SAP DP Agent.
  2. Observe that the scan flags the Apache log4j-jul.jar library as vulnerable and recommends updating to version 2.25.3.

Cause

The vulnerability scan detects the Log4j library version bundled with the DP Agent installation. Such scans identify affected components based on file version and presence, without considering product-specific context. 

Resolution

The DP Agent is not exploitable in the context of CVE-2025-68161, as the affected functionality is not used by the product. Therefore, no immediate corrective action is required on DP Agent level.

As a long-term solution, the DP Agent release 2.8.3.0 already includes an updated Log4j version in which this vulnerability is already fixed.

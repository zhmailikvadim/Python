Symptom

Different recognized value for periodic services type project in different ledger

Environment

SAP S/4HANA Cloud Public Edition

Reproducing the Issue

Cause

There could be different time-based method defined for different accounting principal for RA key(for example, SPPC) setting or it could be a replacement rule for the recognition key (SSCUI 104054) maps SPPC to SPPC1 for a given accounting principle (e.g., IFRS), and SPPC1 and SPPC use different time-based methods in SSCUI 102530, which leads to differing recognition time methods across ledgers.

Resolution

  1. Open SSCUI 104054 (replacement rule for recognition key) and verify whether the RA key(e.g., SPPC) is replaced by another key(e.g., SPPC1) for the relevant accounting principle (for example, IFRS).
  2. Open SSCUI 102530 (Maintain Settings for Event-Based Revenue Recognition) and navigate to Recognition key > Company code and accounting principle.
  3. Compare the time-based method settings of the involved recognition keys (e.g., SPPC vs. SPPC1), such as “Financial periods, actual value date” versus “Calendar days of financial periods, actual value date.”
  4. Align the time-based method settings across the affected ledgers/principles to meet the desired recognition approach.

Keywords

event-based revenue recognition, recognition time method, replacement rule, recognition key, SSCUI 104054, SSCUI 102530, SPPC, SPPC1, IFRS, ledger 0L, ledger 2L, periodic services, revenue difference, financial periods, calendar days

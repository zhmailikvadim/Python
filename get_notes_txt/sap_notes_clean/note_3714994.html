Symptom

You are using the ODBC adapter for ABAP (ABAPODBC) to consume remote objects from an ABAP system and are encountering the following or a similar error:

  • Error: (dberror) [403]: internal error: Error opening the cursor for the remote database <REMOTE_SOURCE_NAME> [SAP][ODBCforABAP] (42000) [MESSAGE:GF8]The character ":" is not valid here. A blank may be missing or there may be too  many blanks.: line 1 col 24 in statement <QUERY>

Other Terms

SDA; Federation; HDI; colon; Virtual Table Name; ABAPODBC; ODBC adapter for ABAP

Reason and Prerequisites

Issue number 353487

Reason:

Due to a limitation in ABAP special characters like colon ":" cannot be used as virtual table name.

Affected Releases:

  • SAP HANA 2:
    • Revisions <= 89.01 (SPS08)

Prerequisites:

  • When using the ABAPODBC adapter to connect to an ABAP system via Smart Data Access

Solution

Apply one of the following SAP HANA Revisions:

  • SAP HANA 2:
    • Revisions of SPS08: SAP is currently working on a permanent solution to this issue. This SAP Note will be updated as soon as a revision containing the solution is available
    • or higher

  • or higher

    Workaround

    Apply one of following workarounds:

    • Virtual Table Naming:

      Avoid including a colon (:) in virtual table names.
    • HDI Environment:

      When working in an HDI environment, manage virtual tables in a folder that does not enforce HDI namespaces. This helps prevent issues related to unsupported special characters, such as colons, in object names.

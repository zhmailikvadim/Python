Symptom

  • After upgrading the BW environment non-compounded keys can't be used directly in existing WebI report for particular BEX query.
  • After the BW update explicitly had to choose the compounded value from LOV to receive the requested results.

Other Terms

webi, web intelligence, BW, 4.3, compound, Bex

Reason and Prerequisites

Environment:

  • SAP BusinessObjects Business Intelligence Platform 4.3 Support Package 04 Patch 07
  • BW
  • Windows
  • Web Intelligence

Steps to Reproduce:

  1. Logon to BI launchpad
  2. Open and refresh a Web Intelligence report containing two Queries pointing to two different BEx Queries & two prompts (say X & Y).
  3. In the prompt window through settings enable the search by key setting
  4. Enter the non –compounded key manually.
  5. Observe getting the data in the report level if we enter the non –compounded key manually
  6. Run the second BEX query.
  7. Enter the non –compounded key value manually
  8. Observe no data is getting retrieved

Symptom

When importing hierarchical master data (e.g., Legal Entity, Business Location) from SAP Datasphere into SAP Sustainability Control Tower (SCT), the import succeeds, but the subsequent validation process fails.

The system displays the following error: INVALID_PARNODE_COMBINATION – Both values for Parent Node Type and Parent Node ID need to be filled out or left empty.

This error is observed on records that are intended to be root nodes (the top level of the hierarchy), where the Parent Node ID and Parent Node Type fields are visually blank in the source data within SAP Datasphere.

Environment

  • Sustainability Control Tower
  • Datasphere

Reproducing the Issue

  1. In SAP system, populate the following SCT local tables with valid data:
  • SCT Legal Entity Node (LT)
  • SCT Legal Entity Link (LT)
  • SCT Business Location Node (LT)
  • SCT Business Location Link (LT)
  1. Open Master Data Management in SCT and initiate the import of these entities from SAP Datasphere system.
  2. After the import completes, start the validation process.
  3. Observe the validation error: INVALID_PARNODE_COMBINATION – Both values for Parent Node Type and Parent Node ID need to be filled out or left empty, although both fields are already empty in the respective record.

Cause

This issue is not a product defect but a data modeling conflict between how SAP Datasphere sends empty values and how SAP Sustainability Control Tower expects to receive them.

  • SCT's Requirement for Root Nodes: To correctly identify a record as a root node in a hierarchy, the SCT validation engine strictly requires that the parent-linking fields (Parent Node ID and Parent Node Type) are true NULL values. A NULL value signifies the complete absence of data.

  • SAP Datasphere's Data Handling: SAP Datasphere has a setting that defines how it treats empty cells when exposing data. If this setting is configured to use "Empty String", Datasphere transmits a blank field as a zero-length character ("").

The SCT validation engine interprets this "" character as an actual value, not as an empty field. Because it sees a value in one parent field but not the other (or expects both to be NULL), it triggers the INVALID_PARNODE_COMBINATION error, as its strict parity check fails.

Resolution

Best Practice Recommendation: Before starting, it is highly recommended to clear all existing data for the affected entities from both SCT and SAP Datasphere. This prevents data conflicts from previous failed attempts and ensures a clean import. The steps below incorporate a method for clearing data within Datasphere.

Correct Data Preparation and Import in SAP Datasphere

Objective: To load master data into Datasphere using the correct NULL handling and ensure the table is clean before the load.

  1. Navigate to the Data Builder:

    • In SAP Datasphere, open the Data Builder and select the target local table for your hierarchical data (e.g., SCT Legal Entity Link).

  2. Clear Existing Data (Choose one method):

    • Method A (Recommended): The data deletion can be performed as part of the upload process in the next step by using the Delete Existing Data Before Upload option.
    • Method B (Alternative): Manually clear the table before uploading by selecting the Delete Data from Table button in the table editor.

  3. Configure and Execute the CSV Import (Critical Step):

    • Initiate the upload process by clicking Upload from CSV File.
    • On the configuration screen, perform the following actions in order:
    • a. Enable the checkbox for Delete Existing Data Before Upload (if you chose Method A).
    • b. Locate the dropdown menu labeled Insert missing string value as. This setting is the root cause of the issue.
    • c. Select the NULL option from the dropdown. This ensures that blank cells in your CSV are interpreted as true database NULL values.
    • d. Click Import to execute the upload with the correct settings.

  4. Deploy the Table:

    • After the import process completes successfully, you must click Deploy. This action finalizes the changes and makes the correctly formatted data available for consumption by SCT.

Final Import and Validation in SAP Sustainability Control Tower

Objective: To pull the correctly formatted data from Datasphere into SCT and confirm the resolution.

  1. Trigger a New Import:

    • Return to SAP Sustainability Control Tower.
    • In the Manage Master Data application, trigger a new master data import from SAP Datasphere for the entities you just updated.

  2. Execute Validation:

    • Once the import job is complete, select the imported data and run the Validation process.

  3. Verify Success:

    • Confirm that the INVALID_PARNODE_COMBINATION error is no longer displayed.
    • Proceed to Publish the master data to make it active in the system.

Symptom

  • After OS upgrade SIA goes missing in CCM.
  • Repair succeeds but no SIA appears in CCM.
  • Add node fails with below error:
    • Logging into <hostname>:<port> as user administrator with  authentication... ERROR: The argument has an invalid value  (FWM 02024) ERROR: AddNode - Could not log into CMS "<hostname>:<port>" with the provided credentials. Ensure that the CMS name, port number, and credentials are correct. (FWM 23002)

Environment

  • SAP BusinessObjects Business Intelligence Platform 20xx (2025 & above)
  • SAP BusinessObjects Business Intelligence Platform 4.x (4.2, 4.3 & above)

Reproducing the Issue

  1. Go to CCM 
  2. SIA is missing

Cause

  • Corba SSL was enabled and passphrase.txt file was missing which was causing the issue.

Resolution

  1. Navigate to the <installDir>\SAP BusinessObjects Enterprise XI 4.0\win32_x86
  2. Run: sslconfig.exe -protocol default
  3. Now cd ..\win64_x64
  4. Run: sslconfig.exe -protocol default
  5. Now Try addnode/recreate node

See Also

  • 2460300-Best Practice - Upgrade to a new Operating System (OS) with SAP BusinessObjects Business Intelligence installed.

Keywords

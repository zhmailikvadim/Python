Symptom

Resolution

When a buyer using SAP Ariba Sourcing solutions integrated with ERP using Ariba Network Adaptor for SAP NetWeaver created an RFQ,  the RFQ failed in PI if it included the following special characters in the description of material, material group, or plant:

  • <
  • >
  • &
  • ;
  • ?

The issue was because the mappings on PI was not parsing the special characters properly.

The issue has been resolved by updating the mapping with a new user defined function to parse special characters with appropriate escape sequence.

 

Code changes:

SAP PI changes:

New UDF: formatSpecialCharacters

SCV: AribaStrategicSourcingAdapter.tpz

Namespace: http://ariba.com/xi/ASN/QuoteRequest

Mapping: MM_REQOTE_ORDERS05_ARBA_RFQ_2_CDF_Quote_XSD

 

Updated the field mappings with the new UDF in the following XPaths:

  • /ns0:Messages/ns0:Message1/_-ARBA_-RFQ/IDOC/E1EDP01/_-ARBA_-RFQITEM/PLANT_NAME to /ns0:Messages/ns0:Message1/_-ARBA_-RFQ/IDOC/E1EDP01/_-ARBA_-RFQITEM/PLANT_NAME

  • /ns0:Messages/ns0:Message1/_-ARBA_-RFQ/IDOC/E1EDP01/_-ARBA_-RFQITEM/MATERIAL_GROUP_DESCRIPTION to /ns0:Messages/ns0:Message1/_-ARBA_-RFQ/IDOC/E1EDP01/_-ARBA_-RFQITEM/MATERIAL_GROUP_DESCRIPTION

  • /ns0:Messages/ns0:Message1/_-ARBA_-RFQ/IDOC/E1EDP01/_-ARBA_-RFQITEM/MATERIAL_GROUP_DESCRIPTION to /ns0:Messages/ns0:Message2/cXML/Request/QuoteRequest/QuoteItemOut/ItemDetail/Classification

  • /ns0:Messages/ns0:Message1/_-ARBA_-RFQ/IDOC/E1EDP01/_-ARBA_-RFQITEM/MATERIAL_GROUP_DESCRIPTION to /ns0:Messages/ns0:Message2/cXML/Request/QuoteRequest/QuoteItemOut/ItemDetail/Classification[1]

  • /ns0:Messages/ns0:Message1/_-ARBA_-RFQ/IDOC/E1EDP01/E1EDP19/KTEXT to /ns0:Messages/ns0:Message2/cXML/Request/QuoteRequest/QuoteItemOut/ItemDetail/Description.

  • /ns0:Messages/ns0:Message1/_-ARBA_-RFQ/IDOC/E1EDP01/E1EDP19/KTEXT to /ns0:Messages/ns0:Message1/_-ARBA_-RFQ/IDOC/E1EDP01/E1EDP19/KTEXT

  1. Go to Ariba Connect and log in. If you do not have a User ID and Password for Ariba Connect, contact your Ariba account executive.
  2. On the Home page, click Product Information and Documentation.
    The Resources, Documentation, Known Issues, Release Updates page appears.
  3. Click Cloud Integration -> Resources.
    The Cloud Integration Resources page appears.
  4. In the Integration Tools section, click Integration tools for Ariba Sourcing and Ariba Contract Management.
    The Integration tools for Ariba Sourcing and Ariba Contract Management page appears.

  7. Navigate to the Ariba_Sourcing_SAP_Netweaver_Repository> Ariba_Sourcing_SAP_Netweaver_Repository_V_CI9_SP2_HF25_October_2018 > ximappings folder.
  8. Import the AribaStrategicSourcingAdapter.tpz file into your SAP Process Integration system.

Symptom

When WRO4j is enabled for JavaScript minimization in the storefront, the build or runtime process fails during JavaScript resource processing.

The following exception is reported in the logs while minimizing JavaScript files:

Caused by: ro.isdc.wro.model.resource.processor.support.JSMin$UnterminatedStringLiteralException
 
The error occurs during the processing of JavaScript resources in the responsive addon, preventing successful generation of the minified JavaScript bundle.

Reproducing the Issue

1. Install the CX recipe in a local SAP Commerce Cloud project (for example, version 2211.47) and complete the system initialization.

2. Enable WRO4J and detailed debug logging by adding the following properties to local.properties:

storefront.wro4j.enabled=true

log4j2.logger.wro4j.name=ro.isdc
log4j2.logger.wro4j.level=debug
log4j2.logger.wro4j.appenderRef.stdout.ref=STDOUT

3. Create a JavaScript file named test.js under the following directory:

bin/custom/yb2bacceleratorstorefront/web/webroot/_ui/responsive/common/js

Add a multi-line string definition using ES6 template literals, for example:
 
var test_html_section = `<div class="errorText">
       <div>${textEroor}</div>
</div>`

4. Perform a full project rebuild and start the SAP Commerce Tomcat server.

5. Access the storefront using the following URL in a web browser:

https://localhost:9002/yb2bacceleratorstorefront/?site=powertools 

6. Observe that the JavaScript minimization process fails and the error described in the Symptom section occurs.

Cause

The JSMin tool does not natively support ECMAScript 6 (ES6) syntax, and the maintainer has indicated that ES6 compatibility is not planned. Since JSMin is designed for ES5 syntax, attempting to minify ES6 code may result in errors or incomplete minification.

Resolution

To avoid JavaScript processing and minification issues, multi‑line string definitions should not be used within custom storefront JavaScript files.

For example, replace ES6 template literal–based multi‑line strings such as:

var test_html_section = `<div class="errorText">
        <div>${textEroor}</div>
</div>`;

to

var test_html_section = '<div class="errorText">'
        + '<div>' + textEroor + '</div>'
        + '</div>';

This approach ensures compatibility with JavaScript processors and minifiers that do not fully support ES6 syntax.

Recommendation:
It is strongly recommended to convert all custom JavaScript files to ES5 syntax to prevent build‑time or runtime issues during resource minification and to ensure consistent behavior across environments.

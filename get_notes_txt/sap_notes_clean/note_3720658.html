Symptom

  • PD application server is down or unavailable?
  • User does not have PD client installed or accessible.
  • Unable to check out models through normal client access.
  • Repository database is hosted on a reachable machine but client-based access is blocked.
  • Is it possible to extract models from the Repository without using the client?

Environment

SAP PowerDesigner (PD) 16.7

Cause

While the repository database stores model data, the models are stored in a proprietary format that requires the PD application to properly interpret and extract.

Resolution

  • Extracting a PD model from the repository using SQL alone is not feasible without PD installed.

  • Note: While VBS scripts can be used to check out models, they still require the PD client tool to function.
    Since the core issue is the unavailability of the PD client, VBS scripts do not provide a viable workaround - if the client is accessible, models can already be extracted through the standard interface.

Symptom

  • Data acquisition from S/4HANA via OData is significantly slow
  • High number of requests observed during data load
  • Noticeable latency despite moderate data volume

Environment

  • SAP S/4HANA (OData Services)
  • SAP Analytics Cloud (SAC)

Other Terms

 Gateway, S/4Hana on-premise, BDC

Reason and Prerequisites

Incorrect batch size request handling

Solution

Cause

The issue occurs when an unsupported batch size (e.g., 50,000) is configured.

  • S/4HANA Gateway enforces a maximum supported batch size of ~5,000
  • When SAC is configured with 50,000, this value is not applied directly
  • SAC switches to server-side pagination
  • S/4HANA determines the effective batch size and may reduce it to ~100 records per request

Effect

  • Instead of processing large batches (5,000 records)
  • The system processes many small batches (~100 records)

This results in:

  • Excessive number of network round trips
  • Increased latency per request
  • Overall significant performance degradation

Resolution

  • Configure the batch size in SAC to 5,000 (recommended)
  • Re-run the data acquisition

Recommendation

  • Always use a batch size of ≤ 5,000 for S/4HANA OData sources
  • Avoid configuring higher values (e.g., 50,000), as this triggers inefficient server-side pagination

Additional Information

  • The observed slowness is not due to data volume, but due to inefficient request handling caused by incorrect batch size configuration
  • Proper batch size ensures optimal balance between payload size and number of network calls

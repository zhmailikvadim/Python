Symptom

Once run report MDG_ID_MATCH_SUPP_BP_REL_UPD into target system the key mapping is not created into the source system.

Environment

  • Sales and Distribution (SD)
  • Logistics Execution (LE)
  • SAP ERP Central Component
  • SAP ERP
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA
  • SAP S/4HANA Cloud Private Edition  

Reproducing the Issue

Login into the Target System. 

  1. Goto SE38 Transaction 
  2. Execute the report MDG_ID_MATCH_SUPP_BP_REL_UPD
  3. Enter RFC Destination (Ex. Source system ID - QM7CLNT100) 
  4. Select the radio button Manul Selection / Automatic Selection.
  5. Enter the Business Partner No. 
  6. Hit Execute 
  7. Login into the source system and check the key mapping is not created. 

Cause

  • Error was not solved for the release support package 107 and 108 by note 3144529. 

Resolution

  • Implement the SAP Note 3555386 - Key mapping not updated for suppliers MDG_ID_MATCH_SUPP_BP_REL_UPD and retest the scenario again.

See Also

  • SAP Note 2897171 - Maintenance of Key mapping in Business Partner.

Keywords

MDG_ID_MATCH_SUPP_BP_REL_UPD, KEY MAPPING, SOURCE, TARGET, SUPPLIER.

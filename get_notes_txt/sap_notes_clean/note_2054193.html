Environment

Extension Promotions

Reproducing the Issue

  1. Set up a threshold free gift promotion (on the hybris Management Console).

  2. In the storefront (on Accelerator), add some products to reach the threshold for the free gift promotion.

  3. Do one of the following steps:
    • Go to checkout. You can see that the free gift product was added. Now update the number of gift products to 1 (which is not how a promotion should be added), click update, then the exception appears.
      OR
    • Add items to the cart, check out and log in.
      OR
    • Remove or reduce the amount of products in the cart (not the FREE GIFT), so that the promotion criteria is no longer met.

Cause

https://jira.hybris.com/browse/PRO-144

Resolution

If possible, update to Platform Version 5.0.x because these problems were fixed then. If it is not possible to update, then use the workaround below from the fix. This requires extending the de.hybris.platform.promotions.impl.DefaultPromotionsService and overriding both methods:

PromotionOrderResults updatePromotions(final Collection<PromotionGroupModel> promotionGroups,final AbstractOrderModel order)
PromotionOrderResults updatePromotions(final Collection<PromotionGroupModel> promotionGroups,
            final AbstractOrderModel order, final boolean evaluateRestrictions, final AutoApplyMode productPromotionMode,
            final AutoApplyMode orderPromotionMode, final Date date)
  1. Let's assume that class MyPromotionsService extends DefaultPromotionsService. The implementation would look like the example below:
de.hybris.platform.promotions.impl.MyPromotionsService
public class MyPromotionsService extends DefaultPromotionsService implements PromotionsService
{
    @Override
    public PromotionOrderResults updatePromotions(final Collection<PromotionGroupModel> promotionGroups,
            final AbstractOrderModel order)
    {
        final PromotionOrderResults result = getPromotionsManager().updatePromotions(
                getModelService().getAllSources(promotionGroups, new ArrayList<PromotionGroup>()), getOrder(order));
 
        getModelService().refresh(order);
        return result;
    }
 
    @Override
    public PromotionOrderResults updatePromotions(final Collection<PromotionGroupModel> promotionGroups,
            final AbstractOrderModel order, final boolean evaluateRestrictions, final AutoApplyMode productPromotionMode,
            final AutoApplyMode orderPromotionMode, final Date date)
    {
        final PromotionOrderResults result = getPromotionsManager().updatePromotions(getSessionContext(),
                getModelService().getAllSources(promotionGroups, new ArrayList<PromotionGroup>()), getOrder(order),
                evaluateRestrictions, productPromotionMode, orderPromotionMode, date);
 
        getModelService().refresh(order);
        return result;
    }
}

2. Then add the bean definition to your extension's spring.xml:

<custom_extension>-spring.xml
<alias alias="promotionsService" name="defaultPromotionsService" />
    <bean id="defaultPromotionsService" class="de.hybris.platform.promotions.impl.MyPromotionsService"
        parent="abstractPromotionsService" scope="tenant">
    </bean>

Note: This is just one way to accomplish what you need, you will need to test this *thoroughly* to make sure it suits your needs.

Symptom

The system determines Rule ID from BCM configuration; when a valid Rule ID is found in BCM, the system does not refer to APM configuration.

Environment

  • SAP S/4HANA
  • Payments and Bank Communication

Reproducing the Issue

  1. Open the Fiori app Manage Payment.
  2. Select a payment order.
  3. Click the batch number in the order under the recipient part (Payment Batch/Queue).
  4. Click the batch number again.
  5. Choose Process.
  6. After processing, the system creates an Outbound PO with the batch number in approval.
  7. Observe that APM configuration (PO type assignment with Rule ID) is not considered; the Rule ID is determined from BCM instead.

Cause

As per standard system behavior, Rule ID determination can occur at both APM and BCM levels. However, priority is always given to BCM. If a valid Rule ID is determined in BCM based on its rule conditions, it is used, and the APM- assigned Rule ID is not considered. APM Rule ID is only used if no valid rule is found in BCM.

Resolution

To bypass the BCM rule assignment, the following options can be considered:

1. Adjust BCM rule conditions in a such a way that no valid rule id is determined for specific scenario, allowing APM config to be considered, or
2. Implement a BADI /PF1/BADI_OPO_RELEASE to add custom logic and customize the rule id determination accordingly.

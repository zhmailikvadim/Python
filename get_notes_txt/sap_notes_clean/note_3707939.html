Symptom

After performing a basic import in Employee Central, the manager field for the impacted user remains unchanged and does not update to "No Manager" as specified in the import file.

Environment

  • SuccessFactors Platform
  • SuccessFactors Employee Central

Reproducing the Issue

  1. Prepare a basic import file where the manager field is set to "No Manager".
  2. Perform the basic import in Employee Central.
  3. Verify the manager field for the impacted user post-import.
  4. Observe that the manager field is not updated to "No Manager" and still displays the previous manager ID.

Cause

In Company system and logo settings->[Do not overwrite the Manager field when it contains ‘NO_MANAGER’ value in the user import file] is enabled.

Resolution

  1. Navigate to Company system and logo settings
  2. Disable [Do not overwrite the Manager field when it contains ‘NO_MANAGER’ value in the user import file]

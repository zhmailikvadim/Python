Symptom

How do I validate supplier procurement units against the requisition procurement units?

Reproducing the Issue

 

Cause

The Purchasing Unit (PU) at the header level and the Supplier PU at the line level are different on the requisition without triggering any error, which is leading the selection of incorrect suppliers.

Resolution

To validate selected supplier against purchasing unit, a member of the Customer Administrator group can set to Yes the parameter Application.Procure.ValidateSelectedSupplierAgainstPurchasingUnit.

If the parameter is set to Yes, a document must contain a Supplier that's mapped in the document's purchasing unit. 

If a user tries to submit a document with an invalid Supplier, an error message is displayed.
For example: The requisition cannot be submitted as supplier xxx for line items 1 is not associated with purchasing unit xxx.

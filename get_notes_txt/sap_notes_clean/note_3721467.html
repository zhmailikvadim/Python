Symptom

The client Analysis for Microsoft offers the creation of formulas at runtime. Here another type of dynamic object can be used, a dynamic selection. Depending on the type of formula, there is a en error raised on loading a saved workbook with :

BRAIN (209): Formula (2) contains a formal error

The formula definition has to be at top level of a binary operator.

The displayed data is still correct. The synchronisation of the local formulas is done again after the dynamic selection was successfull, then there is no warning raised from the BW Backend component OLAP.

Other Terms

BICS, bics.zero, local formula using a dynamic selection, BRAIN (209)

Reason and Prerequisites

The synchronisation of local formulas is done before the synchronisation the dynamic selection, this causes the BW backend component OLAP to raise the warning BRAIN (209).

Solution

A local formula can use a dynamic selection, therefore the synchronisation to the BW Backend Component OLAP is done before synchronising the local formula to the BW Backend component OLAP.

Note: BICS is a common library consumed by various client applications. For details of the BICS releases which include this fix, please view the BICS release notes listed in the 'References' section. Client applications which include a specific release of BICS can be determined from Note 2488995 ‘BICS v3 - Delivery Schedule’.

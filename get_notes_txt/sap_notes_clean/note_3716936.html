Symptom

User gets error message when saving changes on running process: "HTTP request failed. User &<CB NUMBER>& is not authorized to update or delete step &<PROCESS STEP ID>&"   

Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.

Environment

SAP Integrated Business Planning

Reproducing the Issue

  1. Login to IBP as a Process step participant,
  2. Open the Manage Processes App,
  3. Navigate to the affected process that is still with "Open" status,
  4. Open the process step to modify the end date of the step, save the changes and the error message appears.

Cause

Process step participant has not enough privilege on given process. 

Resolution

Workaround for running process: make the changes with any of the Process Owners.

For future process run: add the user as Process Step Owner (or Process Owner) in the template and use that template for the next process.

See Also

SAP help document: Process Roles and Authorizations

Symptom

During the Initial Load of Employees from SAP S/4HANA Cloud Public Edition to SAP Sales Cloud Version 2, some employee records do not appear in the Sales Cloud system although the scheduled replication job completes successfully.

Environment

  • SAP S/4HANA Cloud Public Edition
  • SAP Sales Cloud Version 2
  • Employee Replication / Master Data Integration

Reproducing the Issue

  1. Go to Settings
  2. Search and Open Employee view
  3. You expect to see one or more employee not fetched in the query.

Cause

Employees without a Work Assignment in SAP S/4HANA Cloud Public Edition are not eligible for replication to SAP Sales Cloud Version 2.

When analyzing the inbound payload received by Sales Cloud, the WorkAssignment node is missing, indicating that the employee does not have any active work assignment in S/4HANA.
Sales Cloud requires at least one valid work assignment for an employee to be created.

Resolution

1- Maintain a Work Assignment in SAP S/4HANA Cloud

Ensure that the affected employees in S/4HANA Cloud have at least one active Work Assignment.
Only employees with a valid work assignment are replicated to SAP Sales Cloud Version 2.

2- Re-execute the Employee Replication Job in Sales Cloud V2

After maintaining the Work Assignment, trigger the delta replication job.

  1. Go to Settings
  2. Open Communication Configuration
  3. Select Integrate Master Data with SAP S/4HANA
  4. Under Scheduled Asynchronous Outbound Configurations, search for:  Schedule Employee Replication from SAP S/4HANA Cloud Public Edition
  5. In the Action column, click the square icon
  6. Select Immediate Run
  7. Click Save

After the job completes and the messages arrive in Sales Cloud, check again in Employee settings.

The previously missing employees should now be available in the system.

See Also

Employees must have at least one valid Work Assignment with appropriate dates and organizational data.
Employees without Work Assignment are skipped automatically by the mapping service.

Symptom

A security advisory has been received for vulnerabilities in the React Server Components (RSC) used by the Next.js framework.
The vulnerabilities, tracked as CVE‑2025‑55182 and CVE‑2025‑66478 (duplicate), may allow unauthenticated remote code execution (RCE) via HTTP requests.

Questions were raised regarding:

  • Whether applications deployed on SAP BTP, Cloud Foundry environment, are impacted.
  • Whether SAP BTP provides platform‑level mitigation for these vulnerabilities.
  • Whether any updates or patches are required at the platform or application level.
  • Whether the AWS advisory applies to applications hosted on SAP BTP running on AWS infrastructure.

The affected applications were identified as Next.js‑based web applications deployed on SAP BTP (Cloud Foundry) on AWS.

Environment

SAP CAP – node.js runtime

Reproducing the Issue

  1. Deploy a web application built using Next.js on SAP BTP (Cloud Foundry).
  2. Identify the use of React Server components within the Next.js framework.
  3. Verify the version of Next.js used in the application and check if it corresponds to the vulnerable versions mentioned in the CVE advisory (CVE-2025-55182 or CVE-2025-66478).
  4. Confirm the application is associated with AWS‑provided IP addresses as part of the hosting environment.

Cause

The query arises due to possible vulnerabilities in the "React Server" components within the Next.js framework. These vulnerabilities could allow unauthenticated attackers to execute remote code via HTTP requests.

Resolution

From SAP platform side, CVE‑2025‑55182 (React RSC) / CVE‑2025‑66478 (Next.js) are application‑layer RCEs. All SAP BTP CF platform standard services are not impacted.

  • If none of the applications are using React RSC or Next.js App Router and are deployed on BTP, then it will not be impacted.
  • If these applications use a vulnerable version of Next.js, it must be updated to the latest patched version provided by the Next.js team. Updating the application runtime (Next.js version) is the responsibility of the application owner.

As stated in https://aws.amazon.com/it/security/security-bulletins/rss/aws-2025-030/ document: "Customers using managed AWS services are not affected, and no action is required. Customers running an affected version of React or Next.js in their own environments should update to the latest patched versions immediately".

It is important to clarify that AWS managed services are entirely distinct from SAP BTP. While SAP BTP operates on hyperscalers such as AWS, it is independently managed by SAP. As confirmed earlier by the BTP development team, the SAP BTP platform and standard services do not use the vulnerable Next.js.

To obtain additional guidance on the distinction between AWS‑managed services, consultation with AWS Support is advised.

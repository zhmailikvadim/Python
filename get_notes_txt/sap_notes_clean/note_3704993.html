Symptom

  • When trying to edit a financial transaction in FTR_EDIT error message FTR0203 is displayed: "Transaction in company code in release. Processing not permitted".
  • When checking the corresponding workitem in SWIA it is visible that the task for example TS20000223 or TS20000235 was logically deleted, but the top level workflow is set as completed. 

Environment

Treasury and Risk Management (TRM)

Cause

Improper user handling

Resolution

  1. In case of a cancellation the top-level Workflow instance(WS*) should be logically deleted and not the workitem tasks(TS*).
  2. To overcome this inconsistent release status kindly execute program RFTR_RELEASE_WRITE_DEALFLAGS with status A for the corresponding company code and transaction. 

Keywords

FTR0203, rftr_release_write_dealflags

Symptom

The MediaConversionJob has been failed, with errors recorded in the Cloud System Logging. Below are examples of the logged errors: 

..."ERROR","loggerName":"de.hybris.platform.mediaconversion.job.MediaConversionJob","message":"Failed to convert container 'MediaContainerModel (1234567890123@4)' for format '1234567890987'.","thrown":{"commonElementCount":0,"localizedMessage":"Failed to convert container '12345678901_2 to format 85Wx85H.","message":"Failed to convert container '12345678901_2 to format 85Wx85H."...

And:

I/O error on POST request for "http://imageprocessing:8081/convert": Connect to imageprocessing:8081 [imageprocessing/10.245.3.13] failed: Connection refused: imageprocessing/10.245.3.13:8081; nested exception is org.apache.http.conn.HttpHostConnectException: Connect to imageprocessing:8081 [imageprocessing/10.245.3.13] failed: Connection refused: imageprocessing/10.245.3.13:8081

In addition, imageprocessing pod can be seen in OOMKilled status.

Environment

SAP Commerce Cloud 2211.XX

Cause

The issue is related to images with excessively high resolutions, leading to substantial memory usage. There aren't any strict limitations on image resolutions, but based on ImageMagick documentation, below one may find vital information on ImageMagick conversion process: ImageMagick requires:

    • 6 bytes per pixel for all formats utilizing RGB (.jpeg),
    • 8 bytes per pixel for all formats utilizing RGBA (.png),

During MediaConversionJob. For example, a single image with a resolution of 15221px x 15221px in .jpeg format (6 bytes per pixel) will necessitate approximately 1.4GB of RAM when stored as a raw bitmap.

On order to check the resolution, follow the steps below:

  1. Enter the Backoffice.
  2. Head to Multimedia -> Media Containers.
  3. Look for failing media container and view its details.
  4. While in the common tab, view Medias details.
  5. Check the attached image resolution.

Resolution

If the issue is due to resolution, one may resolve the issue by adjusting the resolution. Resizing the image before the MediaConversionJob is essential in order to improve performance.

If the resolution is not the reason of overconsumption, create ticket to product support for further investigation.

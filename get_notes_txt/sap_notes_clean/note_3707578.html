Symptom

Value Date in transaction code KSPI doesn't meet the fiscal year and period.

Image/Data is from SAP internal systems, sample data, or demo systems.  Any resemblance to real data is purely coincidental.

Environment

  • Controlling (CO)
  • SAP R/3
  • SAP R/3 Enterprise 
  • SAP ERP Central Component
  • SAP ERP
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA Finance
  • SAP S/4HANA

Reproducing the Issue

  1. Run t-code KSPI, enter fiscal year 2026, period 1 to 1.
  2. Check result list and observe that the value date does not update to 01.01.2026.

Cause

Value Date for 01.01.2026 is not maintained in t-code OKEV.

Resolution

Run t-code OKEV, double click the fiscal year 2026, enter the Value Date that should be used in KSPI.

Also check the search help of this field.

Symptom

You use transaction ME51N (Create Purchase Requisition) to create a purchase requisition. The program (function module ME_STATISTICS_EBAN_RKO) calls the accounting interface for commitments management (TRWPR-PROCESS = "BANF" TRWPR-EVENT = "OPENITEM"). The accounting interface issues an error message, for example: BP 603 "Item & budget exceeded" or FCO_AVC_CC 201 "The budget tolerance limit of &3% for cost center &2 was reached" for an item of the purchase requisition using the CO message handler (function module MESSAGES_GIVE).

The error message of the item is written to the message log (application log) at document header level and is not assigned to the affected item of the purchase requisition. This is incorrect. Accordingly, the status of the item is not marked as incorrect (error).

Other Terms

MESSAGES_GIVE, BP603, BP604, FCO_AVC_CC201

Reason and Prerequisites

There is a program error.

Solution

Import the relevant Support Package or implement the attached correction instructions.

After you implement the correction instructions, messages from the accounting interface are assigned to the affected item of the purchase requisition if the message is assigned to an item in the CO message handler (MESG-ZEILE).

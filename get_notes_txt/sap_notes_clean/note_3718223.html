Symptom

When I run the ARBCIG_MASTER_DATA_EXPORT program from my SAP ERP system, I see special characters, such as Â and similar, for the content within the exported CSV file.

The data appears correctly in the SAP ERP source system and in the SAP Ariba solution front-end user interface. The issue is only visible within the exported CSV file.

Environment

  • Managed Gateway for Spend&Network
  • SAP Ariba Buying and Invoicing (B&I)
  • SAP Ariba Sourcing (SRC)
  • SAP S/4HANA
  • SAP ERP Central Component (ECC)

Cause

The appearance of special characters in the exported CSV file is an expected outcome of the encoding process that occurs during the integration flow.

Resolution

The special characters do not impact how the data is displayed or used in your SAP ERP system or the SAP Ariba solution user interface. You do not need to perform any action.

Keywords

Ismg, cig, encryption, masterdata, incremental load, full load, importing, exporting, encrypt

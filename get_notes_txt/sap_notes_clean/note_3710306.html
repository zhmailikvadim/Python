Symptom

When using a custom document type for return purchase orders in SAP S/4HANA, the Shipping tab is missing. Purchase orders created with this custom document type do not display shipping data, whereas standard document types (e.g., NB) for the same vendor and material display the Shipping tab correctly.

Environment

SAP S/4HANA Cloud Public Edition

Reproducing the Issue

  1. Create a return purchase order using the custom Z-type document with:

    • Vendor: X

    • Material: X1

  3. Observe that the Shipping tab is not visible in the purchase order.

  4. Create a return purchase order using the standard document type NB with the same vendor and material.

  5. Observe that the Shipping tab is displayed correctly.

Cause

The custom document type Z is not supported for the Shipping tab functionality in SAP S/4HANA Cloud Public Edition. Additionally, the SSCUI “Configure Delivery Type & Availability Check Procedure by Plant” does not permit configuration of a returns delivery type for custom purchase order types.

Resolution

To enable shipping and return delivery processing, use the standard NB return purchase order type, as custom document types cannot be extended to support this functionality in SAP S/4HANA Cloud Public Edition.

Since this functionality is currently not available, submitting a feature request through the SAP Customer Influence portal is recommended. This allows SAP Product Management to evaluate the requirement for potential inclusion in upcoming releases.

For reference, you can find more information on submitting feature requests for SAP S/4HANA Cloud Public Edition here:

2963059 - How to Submit Feature Requests for SAP S/4HANA Cloud Public Edition

Symptom

In Fiori app Find technical object F2072 returned with the message in the application logs of "No changes were made to equipment Message No. IS819" and another error "Work center _KEEP does not exist in plant XXXX".

Environment

SAP ERP Plant Maintenance (PM)
SAP S/4HANA Cloud Private Edition

Reproducing the Issue

1. Launch Fiori launchpad using tcode /n/ui2/flp.
2. Search for Fiori app "Find technical object" or F2072.
3. Under the technical object type, choose equipment and click go to get the results.
4. Select first two line items (Technical object type) of the search and click “ Mass edit technical objects” option at the top.
5. Expand the location and org data from the Mass edit technical object pop up and select the ABC indicator as C-Non Critical and choose save.
6. Keep values, which should allow to keep the existing Work center value(Instead it is taking this keep as the value for Work center and throwing the error)
7. Choose apply so that it creates a job which can check in the application logs
8. Go to the application logs at the top
9. Navigate to the error logs for further details.

Resolution

Please ensure to implement below notes

  1. 3515438 - /UI2/CL_JSON corrections - PL20
  2. 3578204 - F2072: Visibility of the Inspection Lot column and navigation in the Find Technical Objects app
  3. 3549861 - F2072 - Application crashes when user presses the Go button and Mass Edit saves the _KEEP value
  4. and refresh the cache afterwards according to the SAP Note 2319491.

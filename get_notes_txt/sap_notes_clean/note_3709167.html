Reproducing the Issue

  1. Execute the GET method for the specified API URL.
  2. Observe the runtime error 'RAISE_SHORTDUMP' along with the SQL error code 10811, indicating an issue during execution.

Cause

The SQL error arises because the cumulative value of the field is causing an overflow. The target field for a decimal must be sufficiently large to accommodate the SUM. Hence the annotation for the aggregation function SUM needs to be enhanced to specify the appropriate result type.

Resolution

The target field should be expanded by at least two additional places, as truncating it with a cast is not a viable solution.

The issue stems from the return value of the SUM method, which is causing a dump. The retrieved value has a target precision with digits differing from those allowed before the decimal separator, whereas the element's data type only supports a specific scale. To address this, adjust the data type size of the method element in the custom CDS view YY1_<API> (as an example) by using a cast operator.

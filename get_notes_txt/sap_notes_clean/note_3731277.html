Symptom

  • After running the Service Level Prediction (SLP) function in IBP for Inventory, the Predicted Service Level (PSL) displays a non-zero value (for example, around 0.7) even when demand is zero and IOINVENTORYTARGET (input safety stock) is zero.
  • Users expect PSL to be 0 under these conditions.

Environment

SAP Integrated Business Planning (SCM-IBP) for Supply Chain

Cause

  • PSL in a period is computed using TIP (demand over the exposure + input safety stock) and backlog.
  • Even when demand over the exposure and input safety stock are zero, a non-zero backlog can exist (for example from an upstream supplier), which reduces the PSL from 100%, resulting in a non-zero value.
  • Note: the expected behavior with 0 demand over exposure, 0 input safety stock (IOINVENTORYTARGET) and 0 backlog is that PSL = 100%.

Resolution

  • Note that this behavior is by design: PSL accounts for backlog in addition to TIP (demand over the exposure + input safety stock).
  • Be aware that SLP outputs only the PSL; the backlog used in the calculation is not written to a key figure.
  • However, if IOINVENTORYTARGET = RECOMMENDEDSAFETYSTOCK, you can refer to either the OUTGOINGBACKLOGMEAN and/or OUTGOINGSRCTOLOCBACKLOGMEAN to get an idea on the backlog present.

Symptom

You are using CommonCryptoLib 8.6.3 or lower and detect the following issues:

  1. You encounter a crash when attempting to connect via SSF.
  2. When establishing a TLS connection, you see that a full handshake occurs when a session resumption was expected.
  3. When establishing a TLS 1.3 connection, you see unexpected extensions in the EncryptedExtensions message.
  4. When establishing a TLS 1.3 connection, the handshake is aborted when the ServerHello message includes the "supported_groups" extension. 

                Other Terms

                CCL

                Reason and Prerequisites

                1. When using a certificate with an unsupported signature algorithm, a variable was not initialized and attempts to use this variable caused an crash.
                2. When a client which supports TLS 1.3 and 1.2 is trying to resume a session with a server that only supports TLS 1.2, the session ID could not be found to perform session resumption.
                3. The "illegal_parameter" alert was not sent when some unexpected extensions appeared in the EncryptedExtensions message.
                4. If the "supported_groups" extension was recieved in the ServerHello, the handshake was aborted.

                Solution

                Download and install CommonCryptoLib 8.6.4 (or higher). The update includes the following corrections:

                1. An appropriate error code is returned for an unsupported signature algorithm.
                2. Session resumption occurs when expected.
                3. In compliance with RFC 8446, the handshake is aborted with the "illegal_parameter" alert.
                4. In compliance with RFC 8446, the "supported_groups" extension in the ServerHello message is ignored.

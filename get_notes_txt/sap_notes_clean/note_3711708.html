Symptom

MDG EAM 2023, 2024

When you are processing multiple Material BOM objects in a single change request, post activation of the CR some of the MBOM objects are getting replicated to the target system.

There is a wait time of 2 seconds by default in the DRF replication process, in some cases this wait time is not sufficient and causing this replication issue in the system. So increase this wait time according to your requirement in the TVARVC table.

Table Name: TVARVC
Variable name : UGI_WAIT_TIME_MBOM_DISTRIBUTE
Selection type : P
Field value(LOW) : <enter wait time in seconds>

Other Terms

MDG EAM, UGI_WAIT_TIME_MBOM_DISTRIBUTE, TVARVC, /UGI/EAM_MROBOM_MAP_KEY, CL_USMD_DISTRIBUTION_SERVICES

Reason and Prerequisites

Program Error

Solution

Implement this note and perform the manual Activity:

Add the Parameter and set the value

Follow the below steps :


Run t-code STVARV

Click on Create and Add the Parameter name as
"UGI_WAIT_TIME_MBOM_DISTRIBUTE" and set the value as X.

Click on Save

Note : After saving the details, Please verify the table TVARVC for the
below entry

Variable name : UGI_WAIT_TIME_MBOM_DISTRIBUTE
Selection type : P
Field value(LOW) : <enter wait time in seconds>

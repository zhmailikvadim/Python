Symptom

 You want to implement IP-based allow-listing as an additional security layer for applications running on SAP BTP, Cloud Foundry runtime.

Other Terms

IP-based allow-listing, x-cf-true-client-ip, Cloud Foundry, SAP BTP, security filtering, defense-in-depth, network access control, application security, client IP address

Reason and Prerequisites

According to defense-in-depth principles, IP-based filtering can be added on top of the strong foundation of authentication and authorization mechanisms. For comprehensive guidance on security in SAP BTP, see Security.

Solution

SAP BTP, Cloud Foundry runtime provides the x-cf-true-client-ip request header to enable IP-based filtering. This header contains the original client’s IP address (IPv4 or IPv6) and is included in every request sent to applications. When a route-service is configured, the same IP address is reported to both the route-service and the backend application, always pointing to the original client.

Application/Route-Service Logic

Choose your approach:

  • Implement filtering directly in your application logic, or
  • Create a dedicated route-service application that intercepts and filters requests

Implement the filtering logic:

  • Implement a mechanism to load your allow-list of approved IP addresses or IP ranges
  • For each incoming request:
    1. Retrieve the IP address from the x-cf-true-client-ip header
    2. Validate the IP address against your configured allow-list
    3. Allow or deny the request based on the validation result

Security Considerations

  • Never rely on IP-based allow-listing alone; always combine it with strong authentication mechanisms such as mutual TLS (mTLS)
  • Implement a secure mechanism to manage and update the allow-list configuration
  • Follow security best practices and defense-in-depth principles

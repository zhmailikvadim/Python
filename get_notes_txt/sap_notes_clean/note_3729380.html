Symptom

After upgrading to 2025 FPS01, all US4G MCM processes in the MCM_DE cluster fail to run correctly because the BC set /US4G/BC_MCM_PROCESS_CONFIG_2602 is missing from the system.
This issue is relevant for the market roles: SUP, DSO, MOS, PRS.

Other Terms

MCM_DE, /US4G/BC_MCM_PROCESS_CONFIG_2602, BC set, 2025 FPS01, upgrade, master data

Reason and Prerequisites

 The error occurred because the BC set /US4G/BC_MCM_PROCESS_CONFIG_2602 was missing after the upgrade to 2025 FPS01.

Solution

Implement the corrections of this SAP Note or import the relevant support package.
After implementing this SAP Note, all MCM US4G processes run correctly with the restored BC set /US4G/BC_MCM_PROCESS_CONFIG_2602.
You can verify the fix by creating an instance that creates master data in the business system and checking that all processes complete successfully.

Symptom

When executing report /SAPAPO/TS_BATCH_RUN, the output list does not display all the logs: for every object processed, not all the executed macros are listed.

Other Terms

Batch run, duplicated macro, MacroBuilder, language, sm37, spool,

transaction /sapapo/advm, /sapapo/mc8e (planning job), /sapapo/mc8t (job activity)

Reason and Prerequisites

The report is being executed for macros. The error in the spool log is due to different macros in the job activity that have the same macro description.

This is because in the macrobook, there are two different macros saved with the same name for a given logon language. This is incorrect.

Solution

1. Create two text messages as per the manual pre-implementation attached.
2. Implement the attached correction instruction.

To repair the problem:

-> For the existing macrobooks, enter MacroBuilder and change the macro name for one of the duplicated macros used in the planning job's activity. Then check in the job, that the macros have now different names.

-> For the new macros maintained after implementation of the present correction, it won't be possible to exit the MacroBuilder if there are two macros with the same name.

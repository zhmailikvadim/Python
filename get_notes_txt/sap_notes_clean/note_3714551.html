Symptom

A duplicate key dump ocures in method PREP_DATA_FOR_DYNAMIC_TABDET of class /NZT/CL_ANALYSIS_MAINT_EVENT when activity "Get and assign trkorrs from ref system to maintenance event" NZTA2_ASG_REF_TR_TO_ME is executed with option "AddOn transports from time window" and upgrade maintenance event. This is due to a bug in method PREP_DATA_FOR_DYNAMIC_TABDET.

Other Terms

downtime minimization tool for SAP S/4HANA, NZDT

Reason and Prerequisites

Any of the following add-on support packages is installed in your system landscape:

  • NZDT 1.0 SP02
  • NZDT 1.0 SP03

Solution

Implement the correction instructions with transaction SNOTE on the following systems of your system landscape:

  • Central system

In order to take effect this note needs to be implemented before activity NZTA2_ASG_REF_TR_TO_ME is executed.

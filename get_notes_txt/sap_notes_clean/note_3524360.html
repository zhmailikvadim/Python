Symptom

Contains pattern operator (CP or *) is set as the default operator for a variable in the query definition.

By creating a BW Live model on SAC when inserting it on a story or opening it on data analyzer, the * operator does appear as default, but it is greyed out.

When user deletes this operator and run the query, it is impossible to select the * operator again. It will not appear on the dropdown.

Also if the story is saved and reopened the CP operator becomes a '='.

Environment

  • SAP Analytics Cloud
  • SAP Business Warehouse
  • SAP BW4/HANA
  • SAP Netweaver

Reproducing the Issue

  1. On the query definition, set CP as the default operator for a variable
  2. Create a BW Live model on SAC and insert this model into a story or open it on Data Analyzer
  3. Confirm that the * operator does appear on the variable screen but it is greyed out.
  4. If user deletes the variable entry it is not possible to choose * again.

Cause

Contains Pattern Operator is not supported on BW Live Connections.

As per SAP Help: SAP Analytics Cloud Support Matrix for Live Connectivity to SAP NetWeaver BW and SAP BW/4HANA

Variables in BEx QuerySAP Analytics Cloud (BW Live Connection)
Variable Representation : Selection Option Operator Functionality (except Contains Pattern) Yes
Variable Representation : Selection Option Operator Functionality – Contains PatternNo

However, if CP is set as the default operator for a variable in the query definition, SAC will only display it on the first execution, thus making it not possible for user to manually select it.

Resolution

SAC is working as designed.

CP operator is not supported and although the * operator appears on initial load, SAC will request exactly the same value as if the = operator was selected, therefore, the results will be the same with = or *.

Symptom

After performing the goods receipt with serial number, the partner data are deleted from the equipment / serial number (s).

Environment

  • SAP ERP Plant Maintenance (PM)
  • SAP ERP Logistics (LO)
  • SAP R/3
  • SAP R/3 Enterprise
  • SAP ERP Central Component
  • SAP ERP
  • SAP Enhancement package for SAP ERP
  • SAP Enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA, on-premise
  • SAP S/4HANA Cloud Private Edition

Reproducing the Issue

  1. View the equipment / serial number master data in t-code IE03 or IQ03, partner tab. There is/are partner(s) can be seen. 
  2. Perform goods receipt (movement types that have assigned business transaction PMS1, PMS7, PMT5 or PMT7) with these serial numbers in transaction code MIGO, VL02N or some other batch way.
  3. Check the equipment / serial number master data in t-code IE03 or IQ03, partner tab. There were deleted from the equipment / serial number.

Cause

Check the following customizing of the corresponding partner procedure:

  • In transaction VOP2, "Partner mandat." column: Transaction SPRO -> "Plant Maintenance and Customer Service -> Master Data in Plant Maintenance and Customer Service -> Basic Settings -> Partners -> Define Partner Determination Procedure and Partner Function"   
    >> Check if this partner procedure is set as a mandatory partner
  • Transaction SPRO -> "Plant Maintenance and Customer Service -> Master Data in Plant Maintenance and Customer Service -> Basic Settings -> Partners -> Copy Partner Functions to Master and Movement Data"
    >> Check if the "Copy Partner" indicator is set on this partner procedure 

If this partner function is either set as mandatory or with flag 'Copy Partner' set, the system behavior is intended by SAP. See SAP note 734348 for more information.

This program logic has been in place for many years. There is no plan so far to change this logic.

Resolution

Choose one of the following options:

  • Set the partner function to 'not mandatory' and remove the flag 'copy partner'
  • Consider user exit ICSV0007 (EXIT_SAPLIPAS_001) where customer can adapt partner transfer.
  • Modify system in a way that skip the deletion of the equipment partners at goods receipt.

 

Keywords

T156S, OMJJ, ST156S-VRGNG, T156S, VRGNG, Update control

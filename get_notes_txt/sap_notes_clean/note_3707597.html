Symptom

The initial PI document was created for a product that was later processed for recounting. A subsequent PI document was then created and got deleted. As a result, the initial PI document cannot be posted due to a warning message: "Item 000001 cannot be posted due to exceeded tolerances" (Message no. /LIME/PI_CORE124).

Environment

  • SAP S/4HANA 
  • SAP Extended Warehouse Management (SCM-EWM) 
  • SAP Extended Warehouse Management Physical Inventory (SCM-EWM-PI)

Reproducing the Issue

  1. Access transaction /SCWM/PI_PROCESS.
  2. Select warehouse No.
  3. Attempt to process PI document.

Cause

  • Standard behavior: After a recount is performed, the system generates a new Physical Inventory (PI) document.
  • The newly created PI document which is marked with a Deletion Indicator, preventing further processing and visibility in the Difference Analyzer.

Resolution

  1. Create a new Physical Inventory (PI) document.
  2. Perform the count for the new PI document.
  3. Post the inventory differences using the new PI document.
  4. Verify that the differences are visible in the Difference Analyzer and process them accordingly.

Keywords

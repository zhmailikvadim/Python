Symptom

In the Manage Incidents app (F4759), the Group filter field can be hidden using a Metadata Extension (MDE) with an annotation. When the field is hidden, the app fails to open, and a JavaScript error occurs: “Cannot read properties of null (reading 'getTokens').”

Other Terms

EHS, Manage Incident, Fiori, UI5, Metadata Extension, Group Filter, JavaScript Error, getTokens

Reason and Prerequisites

The application logic accesses the Group filter field without checking if it exists.

Solution

A check was added to verify that the Group filter field exists before the system accesses it. After this correction, the app works correctly even when the Group filter is hidden.

The "Support Packages" section lists the Support Packages that contain these corrections.

Alternatively, you can implement the attached correction instructions.

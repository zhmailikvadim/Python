Symptom

  • When creating and aggregating an inbound LTSD for externally procured materials, the Preference Statements tab in the product master is not updated with newly created entries. Only previously existing entries are displayed.
  • The newly created and aggregated inbound LTSD is not considered. Table /SAPSLL/PRPREF remains empty for the relevant period.
  • No new preference entries are created with the consideration of the newly created LTSD.

Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.

Environment

SAP Global Trade Services, edition for SAP HANA

Reproducing the Issue

  1. Create an inbound LTSD for an externally procured product and release it
  2. Run aggregation

Cause

The newly created LTSD is not taken into consideration during aggregation, because of customizing settings.

Resolution

  • If aggregation should work without GR, change the valuation method from 'Worst-Case LLEs from GR and GRs' to a method that does not require GR 'Worst-Case LTSDs and GRs'.

Or

  • Post the relevant GR documents so that aggregation can proceed with valuation method 'Worst-Case LLEs from GR and GRs' (/SAPSLL/TSDCTR-AGG VALUATION METHOD = 'B')
  1. SPRO path:
    SAP Global Trade Services, edition for SAP HANA -> Preference Management -> Supplier's Declarations -> Define Settings for the LTSD Process
  2. Double click on the Admin Unit

Consider the F1 help documentation of the Valuation Method field:

Evaluation Method A

Long-term supplier’s declarations are evaluated in the aggregation as soon as they are released.

Evaluation Method B

Long-term supplier’s declarations are evaluated in the aggregation as soon as they are released and the first goods receipt in the validity period of the long-term supplier’s declaration exists.

Symptom

Future Time Accounts are getting automatically created for new hires even though the auto calendar for the subsequent year is already run before hiring.

Environment

SAP SuccessFactors Employee Central: Time Management

Reproducing the Issue

  1. Login to instance.
  2. Hire a user with all time off related fields.
  3. Check if time accounts are created for current year and future year.
  4. Confirm that the auto calendar for time account creation has already been executed.

Resolution

  • This is an expected behavior. During hiring, system would consider the Account Creation Offset and create the future time accounts if hiring is done after the offset date. 
  • If hiring is done before the offset date, then only current year time accounts would be created during hiring.

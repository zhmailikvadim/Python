Symptom

During the installation of DM SERVER - APPL SERVER 10.1, the setup wizard reaches the prerequisite check step and fails to recognize the Microsoft SQL Server Command Line Utilities software, even though it appears to be correctly installed on the system. The installation cannot proceed.

Other Terms

DM SERVER, APPL SERVER 10.1, prerequisite check, sqlcmd, SQL Server Command Line Utilities, ODBC Driver 17, SQL Command Line Utilities 15, installation failure, setup wizard

Reason and Prerequisites

The prerequisite check fails due to one or more of the following reasons:

  • Multiple versions of sqlcmd.exe are present on the system, causing a conflict during the prerequisite check.
  • An incompatible version of the ODBC Driver and/or SQL Server Command Line Utilities is installed.
  • The prerequisite check requires specifically:
    • ODBC Driver 17
    • SQL Server Command Line Utilities 15

Solution

Follow the steps below to resolve the issue:

Step 1: Check for Existing Installations of sqlcmd.exe

Open a Command Prompt (cmd.exe) and run the following command:

cmd.exe /C where sqlcmd.exe

Expected Output:

  • Only one version/path of sqlcmd.exe should be listed.

If multiple entries are listed:

  • This indicates multiple versions are installed and conflicting.
  • Proceed to Step 2 to uninstall all existing versions before reinstalling the correct ones.

Step 2: Uninstall Existing Components

Uninstall both of the following components via Control Panel → Programs and Features (or Settings → Apps):

  1. Microsoft ODBC Driver for SQL Server (all versions present)
  2. Microsoft SQL Server Command Line Utilities (all versions present)

Important: Ensure that all existing versions of both components are completely removed before proceeding to the next step.

Step 3: Install the Required Versions

Download and install the following components in the order listed:

OrderComponentRequired Version
1 Microsoft ODBC Driver for SQL Server Version 17
2 Microsoft SQL Server Command Line Utilities Version 15

Note: ODBC Driver 17 needs to be installed before SQL Server Command Line Utilities 15, as the Command Line Utilities depend on the ODBC Driver.

Download links are available via the official Microsoft documentation or the Microsoft Download Center.

Step 4: Verify the Installation

After installation, re-run the verification command to confirm only one version is recognized:

cmd.exe /C where sqlcmd.exe

Confirm that only a single path is returned, pointing to the newly installed version.

Step 5: Re-run the DM SERVER - APPL SERVER 10.1 Setup

  • Restart the installation wizard.
  • The prerequisite check should now successfully recognize the SQL Server Command Line Utilities.
  • Proceed with the installation as normal.

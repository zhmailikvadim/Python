Symptom

  • The tabindex=-1 attribute is set on the image, forcing focus when navigating with a screen reader. As a result, the source path for the image is announced by the screen reader instead of meaningful text.
  • Screen readers (e.g., JAWS) cannot read meaningful information from the image. For example, they cannot interpret logos or images that contain text/words, such as a "Welcome".
  • Users hear technical details instead of descriptive text, such as:
    • The image type (e.g., “image”)
    • The image URL or file name
  •  The image receives keyboard focus even though it is intended to be decorative.

Environment

  • SAP SuccessFactors Work Zone
  • SAP Build Work Zone Advanced Edition

Reproducing the Issue

  1. Use a screen reader such as JAWS 2025, NVDA 15.3, or ZoomText on Windows 11 with Chrome (latest version).
  2. Navigate to the logo or image on the webpage.
  3. Observe that the screen reader announces the source path of the image instead of ignoring it or providing meaningful alternative text.

Cause

The root cause of the issue is caused by a conflicting setup of the image element. Currently, the images include a tabindex attribute, which is why they are being read by screen readers even though they are intended to be decorative (alt="")

This creates a problem: An element that should be ignored is still focusable.

As a result, when a screen reader (like JAWS) reaches this image, it can’t find any meaningful text to read. Instead, it reads technical details such as the image type or URL.

Resolution

Our engineering team has developed a code fix, which is scheduled for deployment on April 6, 2026. An image widget that has a defined purpose and includes an alt description will remain readable by screen readers due to the alt attribute.

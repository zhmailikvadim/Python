Symptom

Incorrect discount is applied on A/P Credit Memo when the document is added in the Web Client.

Reproducing the issue

Prerequisites:

1. Israel localization is used.

2. Approval process exists for User A for A/P Credit Memos.

2. The 'Enable Updating the Document Generated/Updated by Approval Process' and the 'Enable Originator to Update the Document Draft in Pending/Approved Status' parameters are selected (Administration → System Initialization → General Settings → BP tab).

 

Scenario:

1. Create an A/P Invoice with User A. 17% Tax is calculated in the document footer, this is correct.

2. Copy the A/P Invoice into A/P Credit Memo and manually change the Tax to 0%. Add the document, the approval is triggered as expected.

3. Approve the document with the Authorizer.

4. Log in with User A to the Web Client, search for the approved draft in Purchasing → Docs in Approval Process.

5. Select the checkbox of the A/P Credit Memo row and click on Add.

6. Click OK to the message: The selected drafts will be added as documents. The status of the drafts will become closed.

 

Expected behavior:

The final A/P Credit Memo is added with 0% Tax and 0% Discount, same as on the approved draft.

 

Actual behavior:

The system creates a new modified Pending A/P Credit Memo draft with the 17% Tax added back to the footer on the document.

The tax value is also added as a Discount on the footer with relevant Discount %. 

The same happens with A/R Credit Memo.

 

Additional Information:

The A/P Credit Memo can be added with correct values in the standard SAP Business One Client.

Solution

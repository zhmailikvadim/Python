Symptom

Number of LCMBIAR file and the size of collective LCMBIAR file continues to grow every time we export the job for the same objects.

Issue is observerd in exporting the 2nd and 3rd job (for dependent objects and primary objects respectively) by using LCM CLI.

Environment

SAP BusinessObjects Business Intelligence Platform 4.1

SAP BusinessObjects Business Intelligence platform 4.2

Reproducing the Issue

  1. Create an LCM Job (properties file) to Export the LCMBIAR file by using LCM CLI. (when doing a full promotion)

  2. Once the Export LCMBIAR file is completed, you can see the .lcmbiar file gets created by that LCM job in the given location.  (the one you mentioned in the export ".properties" file)

  3. Move all the lcmbiar file to destination and import it and it end by with some error message.

  4. Re-export the same jobs with same properties file and this time the number of lcmbiar get increase hence the size get increased.

  5. Also, in export and import lcmbiar scenario, LCMBIAR file get generate in FILESTORE location. please refer KBA 2201163 - LCMBIAR files are not removed from File store after deleting related LCM jobs

Cause

  • Hot backup is enabled

Resolution

  • Disable the hot backup and do a clean export and import again. 

See Also

2201163 - LCMBIAR files are not removed from File store after deleting related LCM jobs

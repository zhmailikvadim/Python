Symptom

In the process "Receipt of SLP/TEP/PSET from DSO (SUP View)" (DE_C_EQS_SLP_RECEIVE_SUP), the system fills the SLP version start date and end date incorrectly, which causes overlapping during value import. Additionally, the system incorrectly determines the limiting constant when fetching data from the respective tables.

This issue affects the market role SUP.

Other Terms

SLP version, TEP, PSET, /US4G/EQS_SLPV, /US4G/PFDEF_HD, /US4G/PFDEF_EL, limiting constant, profile timestamps, overlapping dates

Reason and Prerequisites

The system uses incorrect logic to determine the SLP version dates and limiting constant.

Solution

Implement the corrections of this SAP Note or import the relevant support package.

After implementing this SAP Note, the system correctly updates the SLP version in the table /US4G/EQS_SLPV and correctly fetches the values for the limiting constant based on associated profile timestamps.

Run the process again to apply the corrections.

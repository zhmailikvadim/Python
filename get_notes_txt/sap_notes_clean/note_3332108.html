Symptom

Need to install SAP Intelligent Robotic Process Automation (iRPA) / SAP Build Process Automation (BPA) agent in the windows server where has no access to internet by default.
Want to know network rules that need to be whitelisted at windows server side for the installation and usage.

Environment

  • SAP Intelligent Robotic Process Automation, on-premise edition
  • SAP Build Process Automation, desktop agent

Resolution

1. During agent installation, browser extension will be installed at the same time. Thus the browser extension URL need to be whitelisted. For the exact URL, please refer to KBA:
2796986 - Web Browsers Configuration Checks for SAP Intelligent RPA 

2. Refer to following document to know which URLs need to be whitelisted depending on your landscape:
Regions and API Endpoints Available for the Cloud Foundry Environment
Is it possible to allowlist the IP addresses that are used by SAP Intelligent Robotic Process Automation Factory?

3. Follow below KBA to make relevant configuration:
3024534 - WebSocket communication blocked by Firewall

Note:
In case 3rd party software like zScaler get used, may get issue when agent tries to call endpoint galaxy.irpa.cfapps.<region>.hana.ondemand.com even if above setting have been configured correctly. For such situation, need to engage 3rd party software administrator/support/vendor to further troubleshoot the cause and fix the configuration accordingly.

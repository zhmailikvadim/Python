Symptom

You post an invoice with GR/IR clearing indicator set for a purchase order for external service without service-related invoice verification (S.-Based IV). This Invoice post a WRX line in the FI accounting document. Afterward, you post an extra invoice for the same service and the Invoice amount is posted in a KBS line in the FI accounting document. It is expected that second invoice to generate same follow-on documents.

Environment

  • SAP ERP Central Component
  • SAP ERP
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA

Reproducing the Issue

  1. Create a PO for an external service with indicator "GR-Bsd IV" set and without indicator "S. based IV" set.
  2. Accept the service entry sheet.
  3. Post an invoice for full service amount with "GR/IR Clearing Indicator" set.
  4. Post another invoice for any service amount with " GR/IR Clearing Indicator" set.

Cause

  • This is the standard behavior when using the indicator "GR/IR clearing" in the invoices.
  • The GR/IR Clearing indicator is usually used when no further invoices are expected. When the flag "Clearing Indicator for GR/IR Posting for External Services" is set in the invoice item, for the first time, this will clear the amount fully.

  • While posting the second document in sequence, there will not be any remaining amount in the GR/IR clearing account. Hence system proposes this amount as a difference to be debited from the expense account. As a GR/IR account is not mandatory, this will only exist when there is an amount opened to balance against the GR. 

Resolution

In order to have a different behavior and post a GR/IR account (WRX) in all Invoices, it is required unflag the indicator "Clearing Indicator for GR/IR Posting" at the item level. See more examples in the SAP Help link.

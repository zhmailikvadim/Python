Symptom

  • An Identity Authentication tenant connected to a SuccessFactors source system sent email notifications (password reset/activation) to onboarding users on a specific date.
  • An external onboarding user reported receiving a password reset email from Identity Authentication.
  • The team needs to determine how many onboarding users received these emails for that date.
  • When searching audit logs for entries with action=sendMail and successful status, events from that earlier date are no longer available.

Environment

SAP SuccessFactors Onboarding

Reproducing the Issue

  1. Configure transformations so that onboarding users receive email notifications from Identity Authentication.
  2. On a specific date, Identity Authentication processed jobs that sent password reset/activation emails to onboarding users.
  3. A user reports receipt of the password reset email.
  4. Attempt to audit and count recipients by checking Identity Authentication logs for action=sendMail events for that date.

Cause

Identity Authentication retains email and audit log data only for a limited period and automatically purges older entries; events outside the retention window (including action=sendMail with successful status) are not available for review.

Resolution

  1. Access Identity Authentication logs for the relevant timeframe that is still within the system’s retention window.
  2. Filter the audit log for entries where action=sendMail and status indicates success, and refine further to password reset/activation emails for onboarding users.
  3. Compile the count of matching events to determine the number of emails sent.
  4. If the target date falls outside the log retention window, note that it is not possible to retrieve historical counts because the system has purged those entries by design.
  5. Implement operational practices to preserve history going forward, such as regularly exporting or backing up Identity Authentication audit logs within the retention period for future auditing needs.

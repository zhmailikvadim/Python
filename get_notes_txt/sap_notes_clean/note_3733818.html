Symptom

This SAP Note delivers additional enhancements  for the IMICC solution referred in SAP Note 3513060.

Wage Type Reporter(HTRICM00) is adjusted to accomodate the changes to new payroll solution to support:

  • Juper split due to Workplace/Company Change
  • R&D/Dangerous work(Document codes 29, 32, 35) which split with V0 table VTYPE B.
  • New /DXX wage types for excessive amount.

Other Terms

IMICC, ICMAL

Reason and Prerequisites

This SAP Note is relevant only for payroll solution of Turkyie.

Please make sure that your system is patched up to the following minimum prerequisite Support Package level

The following SAP Note explains why this step is important

3181624 - SAP Notes Implementation Prerequisites for HCM Turkey - SAP ONE Support Launchpad

This SAP Note is a prerequisite for:

  • 3700431 - HRTR: IMICC - Enhancement for Payroll Wage Type Reporter - [3/3] Code

Note  3733817 - HRTR: IMICC - Enhancement for Payroll Wage Type Reporter - [1/3] Function Group is prerequisite for this SAP note

Solution

New domain PTR_ISRDW, and Data Elements PTR_EXCES and PTR_ISRDW is added.

PTR_EXCES and PTR_ISRDW  is added to table T7TRIC02 and View V_T7TRIC02 and maintenance view V_T7TRIC02.

JUPER split is automatically splited and no extra step is needed.

In schema TR00, the Social Security related wage types for R&D employees were shown with additional wage types such as /PIE, /YIE whereas in the new payroll schema TR01 the they are still put into /NEE, /IEE with split used with V0 table with V0 TYPE = B. To print social security wage types related to R&D you need to enter there through Labor Summary Customizing Sheet which can be open through  SAP MENU-> Human Resources -> Payroll -> Europe -> Turkey -> Next Actions -> Independent Period -> Labor Summary OR through transaction SM31 view cluster V_T7TRIC01.  In Labor Summary Customizing Sheet after creating Wage Type Definition , with selected definition clicked on Connection of Wage Types to Wage Group, you need to enter a new wage TYPE (as example, /NEE) Sign, percentage and you need to set checkbox R&D/Dangerous Work column true. This way the /NEE (SGK Employee Share ) for R&D part of the Payroll will be shown in the Wage Type Reporter.

In Schema TR00, the excessive amount is used with /D01, /D02, /D03 ... /D12 wage types for each month , in schema TR01 the it represented with one wage type /DXX and the month information is written to TMU(unit of time field) inRT table with 3 characters where first two character denotes the year and last  character denotes the month = 261(year: 2026 month:1), 262 (year:2026 month: 2), 263, 264, 265, 266, 267, 268, 269, 26A(year: 2026 month:10), 26B(year: 2026 month:11), 26C(year: 2026 month:12). To add the excessive amount for the DXX(261), in Labor Summary Customizing Sheet after creating Wage Type Definition , with selected definition clicked on Connection of Wage Types to Wage Group, you need to enter a new wage TYPE /DXX Sign ++ , percentage 100,000 and set column Excessive Amount Identifier (DevirMAthG) to 1 which is the last character of 261. If you leave the identifier empty it will sum all available /DXX wage types in the payroll result.

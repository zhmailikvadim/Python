Symptom

  • You want to configure SAP Field Service Management so that a technician can access the Planning & Dispatching application but can only see themselves as a technician on the Planning Board.
  • At the same time, you do not want to modify the existing permission groups.

Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.

Environment

  • SAP Field Service Management (FSM)

Reproducing the Issue

  1. Log in to FSM as a technician user.
  2. Open the Planning & Dispatching application.
  3. View the technician list on the Planning Board.

Actual Result

Multiple technicians are visible on the Planning Board.

Expected Result

Only the currently logged-in technician should be visible on the Planning Board.

Cause

The visibility of objects (such as technicians) is controlled by Policy Groups and Permission Groups. By default, the Planning Board shows all technicians allowed by the assigned policies.

Resolution

This requirement can be fulfilled by using a Read Model Filter on the relevant Policy Group.

SAP FSM provides a dedicated read model filter for controlling technician visibility on the Planning Board:

  • VIEW_ALL_PLANNING_BOARD_TECHNICIANS
    Controls which technicians are visible on the Planning Board.

Configuration Example

To restrict visibility so that a user can only see themselves, configure the filter as follows:

{
    "query": {
        "bool": {
            "must": [{
                    "match": {
                        "unifiedPerson.id.raw": "{{currentUser.id}}"
                    }
                }
            ]
        }
    }
}

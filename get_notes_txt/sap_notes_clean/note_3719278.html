Symptom

You plan to enable Microsoft SQL Server feature 'Change Tracking' in your SAP NetWeaver/Microsoft SQL Server 'Enterprise Edition' System and need to know the following information:

  • Is SQL Server 'Change Tracking' supported/validated by SAP?
  • Can 'Change Tracking' be enabled on the SAP NW/SAP ERP source system (EHPx for SAP ERP 6.0 on Microsoft SQL Server 20xx) SAP standard tables, for analytical consumption?

 

Environment

SAP NetWeaver on Microsoft Windows Server OS/Microsoft SQL Server DB 

Cause

SAP does not certify or explicitly test Microsoft SQL Server  'Change Tracking'.

Resolution

Microsoft SQL Server database feature Change Tracking is not part of "SAP NetWeaver/Microsoft SQL Server 'Enterprise Edition'" standard installation, the setup of Change Tracking is not shipped with the SAP/DB SW installation, which means, Change Tracking is not tested by SAP. However, for those customers who plan to set up Change Tracking, what SAP Technical Support can say is that this Microsoft SQL Server feature can be used by the customers after having it first thoroughly tested in their test systems. 

In regards to Change Tracking implementation in SAP NetWeaver (SAP ERP) systems, we are only aware of possible performance issues on the SAP system/database, for that, for those customers who plan to set up Change Tracking, we recommend the customers follow the SAP best practice for non-SAP tested SW applications, that is, thoroughly test Microsoft Change Tracking in your train-dev-qas systems before configuring it in the production system.

For the specific Change Tracking questions customer should open a corresponding Microsoft case.

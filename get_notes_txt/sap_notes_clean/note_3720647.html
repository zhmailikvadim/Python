Symptom

Fiori Spaces & Pages are configured to separate content for user “A” and user “B”.

Within the same space, two different pages are defined:

Page “X” assigned only to role “R1” (for user “A”)

Page “Y” assigned only to role “R2” (for user “B”)

Each page is linked to a separate Business catalog, and the catalogs are authorized only in their respective roles. However, the same app “Z” is authorized in both roles (via the Technical catalog).

Despite this setup, user “B” (with role “R2” only) can see not only page “Y” but also page “X”, including app “Z”.

Environment

SAP S/4HANA

Cause

  • Pages within spaces do not require additional authorization by design.
  • If a user has access to an app through a technical catalog, the app is rendered across all pages within the spaces the user has access to, regardless of specific page-level authorizations.

Resolution

The design concept of SAP Fiori Spaces & Pages requires that, for an app to be visible to a user, the assigned role must contain both:

  • the relevant space, and
  • the corresponding technical catalog.

Technical catalogs represent the master repository for apps, tiles, and target mappings within a functional area. Business catalogs are derived from technical catalogs and define the subset of content exposed to end users via roles.

 

In this scenario, the app "Z” originates from technical catalog “SAP_TC” and is referenced in multiple business catalogs (for example, “Z_BC_X” and “Z_BC_Y”). Since the technical catalog is assigned in the roles, the tile becomes available in both cases. The technical catalog is therefore the authoritative source controlling the app availability.

 

In summary, an app is rendered for a user only if both the space and the corresponding technical catalog are assigned in the user role.

To achieve a separation of content, separate the spaces.

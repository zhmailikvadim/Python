Symptom

  • During the receiving process in the RF environment, the system does not display the expiration date for certain products/batches.
  • The issue occurs when using RF menu 341(Inbound Processes->Receiving of Handling Units->Rec.HU by Delivery) to enter product, quantity, and batch number.
  • The expiration date is displayed for some inbound shipments but not for others.

Environment

  • SAP S/4HANA
  • Extended Warehouse Management

Reproducing the Issue

  1. Log in to RFUI using transaction code /SCWM/RFUI
    Enter the RF login data detail
    Select the following options:
    03 Inbound Processes
    04 Receiving of Handling Units
    01 Rec.HU by Delivery
  2. Enter the product, quantity, and batch number etc.. for the inbound delivery.
  3. Observe that the expiration date is displayed for some products/batches but not for others.

Cause

  • If the batch has classification, the characteristic 'LOBM_VFDAT' is required to populate the Best Before Date (BBD) field. If users have no classification for the batch then the SLED/BBD has to be filled out in the batch master (Tab: Basic Data 1) MSC3N.
  • In cases where the characteristic 'LOBM_VFDAT' is missing, the expiration date is not displayed.

Resolution

  1. Verify if the batch has classification assigned.
  2. If classification is assigned, ensure that the characteristic 'LOBM_VFDAT' is defined and set correctly.
  3. For batches without classification, ensure the SLED/BBD is filled out in the batch master data (Tab: Basic Data 1) using transaction MSC3N.
  4. Test the RF environment to confirm that the expiration date is displayed correctly for all relevant batches.

Symptom

Issue with the standard SAP Fiori application W0164 - “Product and Service Margin” where the KPI data is not being displayed as expected.

Environment

  • Controlling and Profitability Analysis
  • SAP S/4HANA Finance
  • SAP S/4HANA
  • SAP Fiori

Reproducing the Issue

  1. Access the SAP Fiori App W0164 - 'Product and Service Margin'.
  2. Select a Financial Statement Version.
  3. Observe that all or some value fields and key figures are displayed as zero.

    Resolution

    1. Complete the customizing settings for the Financial Statement Version (FS Version) using transaction OB58.

    2. Assign the required semantic tags (e.g., BILL_REV, ACT_COST) to the FS Version using transaction FINSC_FAGL2SEMTA. Semantic tags are representing the value fields and key figures which are shown in the App.

    3. Review if the expected journal entry data is included in the result list of the feeder Query using transaction RSRT. The Object Type controls which objects/account assignment types are considered in the report. It is prefilled with EO= Profitability Segment, SV = Service Order, SC=Service Contract, PR = Project, OR = Maintenance-Centric Service Orders (MCSO). If one of those objects types are deleted the respective data sets of these objects are no longer considered in the report. The field ACDOCA-ACCASTY should match with this filter for line items to be included in the report.

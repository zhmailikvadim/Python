Symptom

Although the Portlets Configuration in the relevant Country Payroll System Configuration have been correctly configured and the Payroll Unified Configuration is correctly configured (when applicable) and the necessary Role-Based Permissions have been correctly assigned, some Payroll Information tiles are not visible in the People Profile such as: Replication Master Data Details, Earning and Deductions, Tax Information and Garnishment and the following Response is visible in the Browser's Developer tools within the Network section:

400 Bad Request

Code: BadRequest
Message: We can’t process this request because payroll is not configured for the country/region of employment. Please check with your Admin.

Environment

  • SAP SuccessFactors Employee Central
  • SAP ERP HCM and SAP S/4HANA UI Integration with SAP SuccessFactors Employee Central
  • SAP SuccessFactors Employee Central Payroll

Reproducing the Issue

  1. Access the Employee's Legacy People Profile and Latest People Profile -> Payroll Section
  2. Observe that some Cards such as Earning and Deductions, Tax Information, and Garnishment tiles are missing and an error is displayed in the Browser's Dev Tools' Network Section.

Cause

The SF System requests the Employee Maintenance Categories Tiles, however, the Employee's Country/Region is currently set to Status 'Inactive' causing the Payroll Gateway to return a 400 Bad Request response, preventing the display of the Payroll Information tiles.

If the Country/Region impacted Object was unexpectedly changed to Status 'Inactive' it is most likely because its Object Definition's Secured field is set to 'No', which allows unintended edits that created an Inactive time slice, when making adjustment on an Employee's Data in their People Profile.

Resolution

To fix this issue, please, perform the following adjustment:

  1. Go to Manage Data -> Country/Region -> Search the impacted Country (set the Include Inactive Records to 'Yes') and delete the time slice on which the Object is in Status 'Inactive' and/or set the Status back to Active.
  2. To avoid this issue in the future, users must be prevented from editing SF Objects' Status, for that, please, follow the instructions provided in the following KBA: 2662925 - Users can 'Edit' Objects Without Permissions From Payment Information, Work Order Information & Benefit Enrollment UI.

if the issue persist, please, ensure the following pre-requisites are met: 

  • If using the Latest People Profile: Payroll | SAP Help Portal
  • If using the Legacy People Profile: Role-Based Permissions for the Legacy People Profile | SAP Help Portal
  • When configuring Mashups for ERP and S/4HANA Systems: UI Integration Using the Payroll Information Page | SAP Help Portal.
  • When configuring Mashups for ECP Systems: Using Payroll Unified Configuration | SAP Help Portal.

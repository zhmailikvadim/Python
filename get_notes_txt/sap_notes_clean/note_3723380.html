Symptom

Display position of characteristics in T-code: KEA5 is different from COPA line item(tab->Characteristic) when logon system in "JA" or "ZH".

for example:

T-code: KEA5
operating concern 

WW111
WW222

However in COPA document=>  tab "characteristic", the characteristics display as following sequence

WW222
--
WW111

Environment

  • Controlling (CO)
  • SAP R/3
  • SAP R/3 Enterprise 4.7
  • SAP ERP Central Component
  • SAP ERP
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA Finance
  • SAP S/4HANA

Cause

The display in KEA5 is sorted by technical fieldname.

The display in KE24N/KE24 (line item) is sorted

  1. By the group (organizational units, customer related characteristics ) and
  2. The text of the characteristic.

This leads to different sort order in JA/ZH and EN (n EN, If most of the texts are empty, this is why the fields appear by technical field name).

Resolution

This is the standard behavior.

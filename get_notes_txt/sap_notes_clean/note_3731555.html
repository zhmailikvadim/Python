Symptom

Invoice submission to KSeF fails with below error:

<exceptionCode>21115</exceptionCode>
<exceptionDescription>Invalid certificate.</exceptionDescription>
<item>An 'Offline KSeF certificate'  cannot be used for authentication.</item>

Environment

  • SAP ERP 
  • SAP S/4HANA 
  • SAP S/4HANA cloud, Private edition
  • SAP Document and reporting compliance cloud edition

Reproducing the Issue

  1. Launch Tx code Edoc_cockpit
  2. Submit any Invoices to KSEF
  3. Observe the error in Error log

Cause

An incorrect KSeF certificate type (offline) was configured for authentication in Manage communication Settings.

Resolution

  1. Please verify in the KSeF portal which certificate types are registered for the pre-production /demo environment. Ensure that an authentication-capable KSeF certificate is used for the authentication process, and not an “offline” certificate.

    This error originates from the KSeF (Polish e-Invoicing) API and indicates that the certificate used for authentication has been rejected because its type is not suitable for the requested API operation.

  2. Check KSeF Certificates - KSeF (National e-Invoice System) -> Typy certyfikatu KSeF(Types of KSeF certificate) for better understanding of the available types of certificate and their purpose.
  3. Replace the certificate with a KSeF type 1 authentication certificate and reconfigure the DRC CE app - Manage Communication.
  4. Refer to the System Configuration guidance in the community blog and to the Poland Process-Specific Settings in the Help Portal to align the certificate setup.
  5. Save the configuration and retest invoice submission to KSeF.

Symptom

  • The implementation of this SAP Note has no impact on your existing functionality.
  • This SAP Note delivers changes that are usually treated as manual activities within SNOTE (for example, packages, ABAP dictionary objects, message classes).
  • After you implement this SAP Note, implement the primary SAP Note 2409434.

Other Terms

Transportation Management, TM Russian Localization, GU-12, GU-27, GU-27-U-VC, GU-38A, GU-29-U-VC, SMGS, Request, Waybill, ETRAN.

Reason and Prerequisites

  • The above mentioned symptoms appear in your system.
  • This SAP Note is relevant only for Russian Federation.
  • TMRU 100 SP03 must be installed

Solution

Use transaction SNOTE to implement the attached correction instructions before you implement primary SAP Note 2409434. By doing so, report NOTE_2409419 will be installed in your system.

  1. Since report /TMRU/NOTE_2409419 has been developed to be executed in English only, we recommend that you log on in English.
  2. In transaction SA38, select report /TMRU/NOTE_2409419 and select “Execute”.
  3. Select "TESTRUN" and execute the report.
  4. Check the Simulation Log and ensure that all the messages have a green traffic light. If there are any warnings and/or error messages that you cannot solve yourself, create a customer incident under the component TM-LOC-RU.
  5. Go back (choose "Back" or F3), select "UPDATE & ACTIVATE", and execute the report again.
  6. The system prompts you to either activate your objects online or in batch mode. The online mode is recommended as this enables you to see any activation issues immediately. Use the batch mode option only if activation in online mode fails with a time-out.
  7. Optional: Go back (choose "Back" or F3), select "GENVIEW", and generate maintenance dialogs.
  8. Within the Change Log, the system displays a list of all objects processed. Ensure that all the displayed messages are warnings or information messages. If there are any warnings and/or error messages that you cannot solve yourself, create a customer incident under the  component TM-LOC-RU. If you have selected to activate the objects in batch mode (see step 6), check the activation log of the job as explained in the message text of the Detailed Log.
  9. Optional: Go back (choose "Back" or F3), select "SHOWLOGS", and execute the report again.
  10. Select the node with the last execution log (according to the date and time) and ensure that there are only information messages (green traffic lights). If there are any warnings and/or error messages that you cannot solve yourself, create a customer incident under the  component TM-LOC-RU.
  11. If any inactive objects are detected during the release of your transports, execute the "UPDATE & ACTIVATE" step again.

Symptom

Program KEA: Create Benefit Requests (RP_PAYDE_KEA_CREATE_NOTIFS) creates correction notifications based on the initial runs, even though there were no retroactive accounting runs in these periods.

Reason and Prerequisites

You use the Payroll Area selection criterion in the Selection Conditions group box. Due to the existing program error, only persons that belong to the payroll area for the selected payroll period were selected. This leads to an irregular person selection due to payroll area changers. However, for a correct KEA notification, all persons that belong to the payroll area or have belonged to the payroll area at some point in the validity period of the reference number (see V_T5D3E_KEA) must be taken into account.

Solution

Implement the attached correction instructions or import the specified support package.

The programming error is corrected so that the person selection period of the logical database PNPCE always extends over the entire period of the reference number.

Necessary correction of initial runs

If you selected the payroll area during your initial runs, the initial notification was probably created on an incorrect person basis. Therefore, start the initial run again for the last period with which you executed an initial run. Existing initial runs are corrected. For periods in which you have already sent a notification to the German Federal Employment Agency (BA), the system issues an error message stating that a newer notification already exists or that an initial run is not possible. You can ignore these messages in this case.

Symptom

When upgrading Data Services from 4.x version to Data Services 2025 Patch 3 on Linux, Adapter Instance cannot be started, please follow the steps for the workaround.

Other Terms

Environment:

  • SAP Data Services 4.2
  • SAP Data Services 4.3
  • SAP Data Services 2025

Steps:

  1. Before upgrading, Check whether the Data Services version is 4.2 or 4.3 with existing adapter instance configured. 
  2. Upgrade to the Data Services 2025 Patch 3 successfully without errors. 
  3. Logon the Data Services Management Console, navigate to Administrator -> Adapter Instance, then found the adapter instance stopped, ERROR: Adapter cannot be started.
  4. In the adapter configuration, while creating new adapter instance, when trying to start, the same error is reported.

Reason and Prerequisites

The root cause of this issue is, since Data Services 2025, the Java version upgraded to 17, and there are some extra crypto lib required, otherwise the java program will stop to run.

Solution

The workaround is adding the sapjce.jar in the adapters class path.

Steps:

  1. Switch to the Adapter Configuration tab, click the instance name to open, in the classpath field, append the ";<LINK_DIR>/ext/jre/lib/sap/sapjce.jar", save and close. 
  2. Start the adapter instance.

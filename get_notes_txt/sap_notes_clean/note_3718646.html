Symptom

When a user fills out MasterInspCharLongText on the backend with a long text for a characteristic it does not appear on the client.

Other Terms

  • SAP Service and Asset Manager
  • SAM
  • SSAM
  • ICMTANGOAMF10-42994
  • Characterisctics

Reason and Prerequisites

Reason:

There can be manual IC in Inspection plan that do not use MICs and the long texts are associated directly to them.

Prerequisites:

  •  BAS or Visual Studio Code with SAM MDK project
  • `patch` command (available on BAS and macOS by default) via terminal or other patching utility
  • Read the guidelines on how to apply the patch and read the appendix section if you receive any error

Solution

There is a new entity introduced to send these long texts for manual IC via payload.

Backend SAPNote 3670035 is required.

To apply the patch, download the attached 3718646.patch file and save it in a known location, preferably adjacent to your SAM.mdkproject.

If you are on Windows, see SAP Note 3533879 for how to install a .patch file in Windows. If you are on a Mac, follow the steps below:

  1. Open a terminal in the root of your SAM metadata (i.e. SAM.mdkproject/metadata or SAM.mdkproject/metadata/base.definitions)
  2. Run the terminal command `patch --no-backup-if-mismatch -p4 < path/to/3718646.patch`. 
  3. Verify in the terminal window that the following files have been modified and listed as patched, as indicated by the console output:
  • patching file Rules/InspectionCharacteristics/Update/InspectionCharacteristicsLongText.js

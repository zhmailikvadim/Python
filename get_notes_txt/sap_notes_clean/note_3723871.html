Symptom

While submitting Self-Billing Invoice from EDOC_COCKPIT, following error has been raised:
Error message SMP_CALL_FAILED – Document Identifier not supported by the receiver

Error seen in Application logs: iFlow Exception at Main iFlow: SMP_CALL_FAILED
Document Identifier: busdox-docid-qns::urn:oasis:names:specification:ubl:schema:xsd:Invoice-2::Invoice##urn:cen.eu:en16931:2017#compliant#urn:fdc:peppol.eu:2017:poacc:selfbilling:3.0::2.1 is not supported by the Receiver with Peppol participant ID XXXX:XXXXXXXXX

Environment

  • SAP ERP
  • SAP S/4HANA
  • SAP S/4HANA Private Cloud
  • SAP Document and Reporting Compliance, cloud edition

Reproducing the Issue

  1. Attempt to process a Self-Billing Invoice with the document identifier: urn:fdc:peppol.eu:2017:poacc:selfbilling:3.0.
  2. The document fails submissions with status "Error at Sender Access point" and you see error as described in the symptom

Cause

During processing, the system performs an SMP lookup to determine which document types are supported by the receiving Peppol participant. If the receiver has not registered support for the document type being sent, the SMP lookup fails, which results in the error SMP_CALL_FAILED and the document cannot be submitted to the receiver.

Resolution

  1. Confirm the document identifier used in the submission. In this scenario the document type is urn:fdc:peppol.eu:2017:poacc:selfbilling:3.0.
  2. Verify the SMP record of the receiving participant to identify which document types are supported. This can be checked using the external tool Peppol Practical - Participant Information
  3. If the document type Self-Billing (poacc:selfbilling:3.0) is not listed in the receiver’s SMP record, coordinate with the receiving party or their Peppol Access Point provider to register support for this document type.
  4. Once the document type is properly registered in the SMP, the document can be resubmitted successfully.

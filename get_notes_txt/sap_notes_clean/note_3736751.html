Symptom

Requirement to remove the Pay Statement section key figures (gross, payout/net, custom key figures, pay date) in the Payroll Information portlet after configuring the Direct Pay Statement.

Environment

  • Product: SAP SuccessFactors Employee Central Payroll
  • EC Payroll Pay Statement

Reproducing the Issue

  1. Configure the Direct Pay Statement and enable the Payroll Information portlet in People Profile.
  2. Implement BAdI HRSFEC_WAGE_TYPE_BADI_DEF and modify methods to return empty component names (rv_component_name) for gross and payout/net key figures.
  3. Open the Payroll Information portlet and then open the Pay Statement overview page via the Pay Statement link.
  4. Observe that values can be hidden in the portlet, but the Pay Statement overview fails with error 301 when component names are removed; returning a blank string only removes numbers and does not hide the section.

Cause

The BAdI HRSFEC_WAGE_TYPE_BADI_DEF is designed to extend/add data rather than restrict or remove standard key figures; removing standard component names causes the Pay Statement service to fail with error 301.

Resolution

  1. Do not remove standard key figure component names via HRSFEC_WAGE_TYPE_BADI_DEF; this breaks the Pay Statement service and leads to error 301.
  2. If acceptable, in the old People Profile use the mask functionality to obscure key figure values in the Payroll Information portlet. Note: this masking is not available in the new People Profile.
  3. As a limited workaround, setting rv_component_name to return a blank value removes displayed numbers while keeping the section visible and preserving Pay Statement loading; it does not hide the section.

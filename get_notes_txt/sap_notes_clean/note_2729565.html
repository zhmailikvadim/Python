Symptom

How to understand the use of the Maximum Storage Capacity Constraints feature for IBP for Inventory.

Environment

SAP Integrated Business Planning (SCM-IBP)

Resolution

  • First, make sure that the Global Configuration settings are correct and that the input key figures needed exist and are configured correctly.
    • See the following Help page:  Inventory Optimization - Maximum Storage Capacity Constraints
  • Before setting a value for the IOMAXINVENTORY key figure, please run both the Multi-stage Inventory Optimization (MSIO) and the Calculate Target Inventory Components (CTIC) functions to get a benchmark for the on-hand stock output key figures (IOAVGONHANDSTOCK, IOTARGETONHANDSTOCK).
  • Some things to keep in mind when running the MSIO and CTIC functions in relation to Maximum Storage Capacity Constraints:
    • The value of IOMAXINVENTORY is a soft constraint and therefore there are instances when IOMAXINVENTORY will not be considered.
    • The algorithm will not respect a value input for IOMAXINVENTORY if PROPAGATEDDEMANDMEAN is a lot higher, independent of how high the MAXINVENTORYVIOLATIONCOSTRATE is.
    • If the RECOMMENDEDSAFETYSTOCK is already 0, no value of IOMAXINVENTORY will have an impact on that node.

Symptom

When reevaluating the jurisdictional control check for a released batch, the Release Decision Trigger ID is stored in the Inspection Lot (IL) field of the jurisdictional control database table (/LSCH/D_JCRESULT). However, the INSPECTION_LOT field (/LSCH/D_JCRESULT-INSPECTION_LOT) uses the QPLOS data element with data type NUMC(12), which accepts only numeric values. As a result, when reevaluation is triggered using a combination of batch, material, and plant that includes alphabetic characters, the system cannot capture the Release Decision Trigger ID in the INSPECTION_LOT field due to the numeric-only restriction on the field. This leads to incomplete data capture for release decision triggers that include alphanumeric identifiers.

Other Terms

LSCH, LSCH_BACHG, LSCH_RVRDT, JC Reevaluation, Batch Changes

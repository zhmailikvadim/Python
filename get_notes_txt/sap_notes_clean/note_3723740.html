Symptom

The write phase of archiving object /PE1/ORD2 exhibits high database CPU consumption and extended execution times.

Reason and Prerequisites

Database performation optimization.

Analysis indicates that the current implementation relies heavily on FOR ALL ENTRIES (FAE) and potentially unbuffered sequential reads. The overhead caused by FAE’s internal result-set de-duplication and multiple database round-trips for related data is significantly impacting the throughput.

Solution

The places where FOR ALL ENTRIES used had been replaced by RANGE TABLE to read information from database.

Please implement the provided Correction or the corresponding Support Package.

Symptom

  • HTML Interpretation on a field is not rendered correctly and is cut off when using a Style with the value "color:invalid".
  • Only text up to the point where the Style attribute is set is visible. The remaining text is cut off and not rendered when using CR for Eclipse runtime.
  • The HTML is correctly rendered in Crystal Reports 2016 and in BI 4.2 SDKs

Other Terms

export, html interpretation, truncated, render, cut off

Reason and Prerequisites

Environment

  • SAP Crystal Reports, version for Eclipse SP23
  • crjava-runtime_12.2.224.zip


Steps to Reproduce

  1. Open SAP Crystal Reports 2016 designer
  2. Create a new report
  3. Create a formula field
  4. Using the Crystal Syntax introduce the following fromula : '<h3>hello<span style="color:invalid">bluetest</span>hello</h3>'
  5. Drag and drop the new created formula into the report and set the Text interpretation into HTML Text
  6. Run the customer application in order to generate the new created the report
  7. Notice the span tag is not interpreted and we do only get the text result : hello The expected result is "hellobluetesthello"


Cause

The values of color should be color_name(red), hex_number(#ff0000), rgb_number(rgb(255,0,0)), inherit.

The root cause of this issue is that “invalid” isn't a correct value of color.

Solution

The behavior of SAP Crystal Reports 2016 Designer is inheriting color from parent element even the color value is illegal. We have aligned Crystal Reports for Eclipse SP24 with this SAP Crystal Reports 2016 behavior, but please note "color:invalid" is still not encouraged.

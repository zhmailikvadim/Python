Symptom

When executing transaction PPCGO, a material movement fails for instance due to an insufficient stock and it is sent to Reprocessing. When reprocessing this entry, an error /SCWM/ERPINTEGRATION 874 occurs.

Other Terms

EWM, CORUAFFW, FLG_ORIG, ewm_sync_post_req, PPC_PROFILE, T437S, POST_SYNC_REM_EWM

Reason and Prerequisites

Prerequisite:

  • In Material Master of the Assembly (MRP 4 view) a PPC profile is assigned, not REM profile.
  • You use JIT EWM (Extended Warehouse Management).
  • You execute transaction PPCGO to process Backflushes.

Reason:

PPCGO uses a "Synchronous Posting to EWM" by default to ensure a real-time EWM stock consumption because it is very much often used together with JIT production supply planning.

On the other hand COGI reprocessing considers setting of an embedded EWM synchronous posting which is a part of the REM profile. In case the Material master uses PPC profile instead, the Synchronoust posting to EWM is not performed which results into the error /SCWM/ERPINTEGRATION 874.

Solution

COGI records created via transaction PPCGO will always be posted Synchronously to EWM.

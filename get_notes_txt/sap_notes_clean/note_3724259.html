Symptom

When attempting to initiate onboarding for Concurrent Employment from the Recruiting module, the onboarding process fails and the candidate becomes stuck in a Suite Mapping error.

The following errors may be observed:

  • "There is already a recruit record for user XXXX. You cannot create another recruit record if one already exists."

  • "You cannot create a job information record for user XXXX before the effective start date in the recruit record."

Additionally:

  • The candidate appears with an ONB Suite Mapping Error in the BPE Process Monitor.

  • Check Tool validations may not show configuration errors.

  • The onboarding process does not proceed successfully.

Environment

SAP SuccessFactors Onboarding

Reproducing the Issue

  1. Configure the Rule to Initiate Onboarding for Concurrent Employment.

  2. Maintain the required mappings in Recruit-to-Hire Data Mapping.

  3. Initiate onboarding for a candidate from Recruiting for a user who already exists in the system.

  4. Attempt to trigger Concurrent Employment onboarding through the recruiting process.

  5. Observe that the onboarding process fails and the candidate appears with a Suite Mapping Error in BPE Process Monitor.

Cause

  • Concurrent Employment onboarding cannot be initiated from Recruiting, Manual Onboarding, or other ATS integrations in Onboarding 2.0. 
  • Concurrent employment must first be created in Employee Central, and onboarding is triggered only after the concurrent employment assignment is created and the rule conditions are satisfied.
  • Attempting to initiate the process directly from Recruiting leads to system errors because a recruit record already exists for the user.

Resolution

    This behavior is expected by design.

    In Onboarding 2.0, concurrent employment onboarding must be initiated through Employee Central, not from Recruiting.

    To correctly trigger onboarding for concurrent employment:

    1. Add the Concurrent Employment assignment in SAP SuccessFactors Employee Central.

    2. Ensure the Start Date of the concurrent assignment is set to a future date.

    3. Verify that the Rule to Initiate Onboarding for Concurrent Employment is configured correctly.

    4. Once the rule conditions are met, the Onboarding process will be triggered automatically.

    Recruiting-initiated onboarding for concurrent employment is not supported in Onboarding 2.0.

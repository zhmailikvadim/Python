Symptom

SAP HANA system showed High CPU usage which leading to performance degradation and resource contention during peak workload periods.

Environment

SAP HANA DATABASE 2.0

Cause

Parameter Default_Statement_Concurrency_limit = '0' was configured at system and database level.

Parameter definition: Restrict the actual degree of parallel execution per connection within a request: For every HANA process the job executor will attempt to limit the number of threads assigned to process a statement to this limit. This is not a hard limit because blocking situation may require to assign additional threads to avoid issues like, e.g. deadlocks.

A value of 0 leaves the concurrency limitation to the job executor which means that the concurrency is not explicitly limited. Without a concurrency cap workload can overwhelm CPU resources.

Resolution

It is recommended to configure a reasonable concurrency limit based on system sizing and workload characteristics.

As per KBA 2600030 Parameter Recommendations in SAP HANA Environments

minimum formula for default_statement_concurrency_limit from MIN(10 %, 12) to MIN(10 %, 8) .

May run the script from HANA_Configuration_Parameters_2.00.040+.txt from SAP note 1969700   SQL Statement Collection for SAP HANA, which will generate the ALTER SYSTEM statement to change the parameter value.

Symptom

When creating or changing a Purchase Order (PO), the header text cannot be edited through the continuous text editor. Instead, the system displays the following message:

Message no. MEPO089 – “Text contains formatting → SAPscript editor”

Environment

  • Materials Management (MM)
  • SAP ERP Central Component
  • SAP ERP
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA
  • SAP S/4HANA Cloud Private Edition

Reproducing the Issue

1.    Execute transaction ME21N/ME22N to create/chnange a PO
2.    Navigate to the Header Text fields
3.    Observe that the header text is greyed out and not editable in the continuous text editor.

Cause

The source text contains characters interpreted as SAPscript formatting commands, which forces the system to switch from the continuous text editor to the SAPscript editor. Examples of such formatting tags include:

    ,,
<H>...</>
<U>...</>
<C>...</>
<R>...</>
<L>...</>
<O>...</>
<S>...</>
<SB>...</>
<NL>
<NP>
&DATE&
&DAY&
&MONTH&
&YEAR&
&NAME_OF_DAY&
&PAGE&

 

When these formatting elements are present, the text is treated as formatted in ME22N, preventing editing in the continuous text editor.

Resolution

To enable editing in the continuous text editor:

•     Remove all SAPscript formatting characters (e.g., , , ) from the source text.

•     Ensure that the header text contains only plain text without embedded formatting commands.

Once the formatting tags are removed, the header text will be editable through the continuous text editor.

Symptom

In the historical retrocession, the system uses an incorrect date for the retrocession of previous versions.

Other Terms

- Historical retrocession

- Retrocession execution date

- Effective date

Reason and Prerequisites

You start historical retrocession for multiple lives without specifying a target date. After a life change, the system does not reset the retrocession date.

Solution

History:

Versions 1-2 from 5/25/2018: New　

Implement the source code corrections.
Support Packages SAPK-70022INMSG and SAPK-80203INMSG correct the problem.　

Solution:

Before retrocession is performed for a new life, the retrocession date must be reset to the original value.

Technical effects:

When you perform historical retrocession for several lives, the system always uses the correct effective dates of the relevant versions.

Manual changes:

You do not have to perform any manual tasks.

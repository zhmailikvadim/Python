Symptom

The Data Dictionary (DDIC) domain SALV_DO_LAYOUT_RESTRICTION contains incorrect descriptions for its fixed values.
Originally, the domain listed the following value options:

  • 1 – Cross-User and User-Specific
  • 2 – Cross User
  • 3 – User-specific

These descriptions do not correspond to the actual behavior implemented in the class interface IF_SALV_C_LAYOUT.

Additionally, several test reports contained commented code referencing outdated or incorrect value meanings. These comments were misleading and have been removed.

Other Terms

ALV; ABAP List Viewer; SAP List Viewer; Grid; List; SALV, SALV_DO_LAYOUT_RESTRICTION, IF_SALV_C_LAYOUT, RESTRICT_NONE, RESTRICT_USER_DEPENDANT, RESTRICT_USER_INDEPENDENT

Reason and Prerequisites

Reason

The value descriptions in domain SALV_DO_LAYOUT_RESTRICTION were not aligned with the semantics defined in interface IF_SALV_C_LAYOUT.
The interface constants correctly represent the behavior of the layout restriction values:

  • RESTRICT_USER_INDEPENDENT = 1 : Cross-User
  • RESTRICT_USER_DEPENDANT = 2  : User-Specific
  • RESTRICT_NONE = 3 : Cross-User and User-Specific

The domain value texts were in conflict with these definitions, although the underlying logic and behavior of the layout restriction mechanism were correct.

Some comments in internal test reports referred to the incorrect value assignments and caused further confusion.

Solution

The fixed values of domain SALV_DO_LAYOUT_RESTRICTION have been corrected to match the definitions used in interface IF_SALV_C_LAYOUT.
The updated value descriptions are:

  • 1 – Cross-User
  • 2 – User-Specific
  • 3 – Cross-User and User-Specific

These changes ensure consistency between the domain metadata and the actual layout restriction logic.

In addition, outdated or misleading comments in several test reports have been removed. No functional changes were made to the underlying ABAP code.
The constants in IF_SALV_C_LAYOUT remain unchanged and continue to accurately represent the layout restriction behavior.

Symptom

In a custom list report app using smart controls, dynamic date interval filters (for example, today, yesterday, last month, last x days) are used.

When saving a selection variant as a tile with a dynamic date interval, only a static counter path can be provided; dynamic date parameters cannot be passed.The tile counter displays static data (for example, it shows the count for the date when the tile was created instead of “today”). The “save as tile” option appears not to support dynamic date intervals for tile counters.

Environment

SAP S/4HANA

Reproducing the Issue

  1. Open a list report using smart controls and set a dynamic date interval filter (for example, today).
  2. Use “Save as Tile” to save the selection variant to the launchpad.
  3. Provide the counter path during tile creation (only static path available).
  4. Add the tile to a group and display it on the launchpad.
  5. Observe that the tile counter remains static and does not reflect the current dynamic date interval.

Cause

By design, “Save as Tile” creates a personalized custom tile treated as static; dynamic tile metadata (including refresh interval and dynamic date handling) is not copied. As a result, the counter remains static and does not update based on dynamic date intervals.

Resolution

Note that this is standard behavior: tiles saved via “Save as Tile” are static and do not support dynamic date intervals for tile counters.

Use a custom dynamic tile instead of user-created tiles:

  1. Create tiles in the Fiori Launchpad Designer using the Dynamic Tile type.
  2. Maintain an appropriate refresh interval to keep the counter updated.
  3. Configure navigation and filter parameters to reflect the intended filters.

For different filter sets, create additional variants using target mapping parameters, startup parameters, and selection/presentation variants, then add these tiles to a technical catalog and expose them via business catalogs.

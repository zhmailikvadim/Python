Symptom

  • You need to restrict access to specific transaction codes (ME13, ME23N, ME2L, ME2M, ME2N) at the vendor/supplier level.
  • The issue pertains to user interface functionality within the Purchase Orders module of SAP ERP.

Environment

  • Materials Management (MM)
  • SAP ERP
  • SAP ERP Central Component
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA

Reproducing the Issue

  1. Attempt to restrict user access to specific vendors in transactions ME13, ME23N, ME2L, ME2M, and ME2N using authorization objects such as F_LFA1_BEK and M_BEST_BSA.
  2. Configure authorization groups (BEGRU) and activities (ACTVT) in SU24 for relevant transactions.
  3. Test the changes by assigning specific authorization groups to vendors and roles.
  4. Observe that the restrictions do not work as expected.

Cause

  • The standard transactions ME13, ME23N, ME2L, ME2M, and ME2N do not trigger authorization checks on the vendor master data.

Resolution

  • transactions ME13, ME23N, ME2L, ME2M, and ME2N do not perform authorization checks on the vendor master data.
  • For report transactions ME2L, ME2M, and ME2N, you can implement custom check logic using the BAdI ME_CHANGE_OUTTAB_CUS. Detailed information about this BAdI can be found in KBA 2455462.
  • Regarding transaction ME23N, the BAdI ME_PROCESS_PO_CUST is available to incorporate custom check logic.
  • For transaction ME13, there is no user exit or BAdI to enhance authorization checks. If you need to control the vendor layout, modifications can be made in the program MM06IO01_MODIFY_SCREEN. However, please note that such modifications fall outside the scope of standard SAP support.

See Also

  • Refer to KBA 2455462

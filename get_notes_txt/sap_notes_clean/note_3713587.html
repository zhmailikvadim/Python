Symptom

You use the manual bank statement, transaction FF67.
In the overview of manual bank statements, the system displays the status "Not to Be Posted" for a line item. This status is unclear.

Other Terms

FF67 SAPLKKBL SAPMF40K text L85

Reason and Prerequisites

Unclear text element SAPMF40K TEXT-L85: 'Not to be posted'. This text is displayed as the status if the line item has not yet been completed with the following status:

  • Posting area 1 is still open (FEBKO-VB1OK initial)
  • Posting area 2 is not to be posted (for example, because it is a one-step posting rule)

'Not to be posted' means that the subledger is not to be posted.

Solution

Implement the correction instructions in your system.
In this case, the item whose posting area 1 is still to be posted but posting area 2 is not to be posted receives the new status "Subledger Not to Be Posted".

Translation

The translation is delivered with the same Support Package as the correction.

You can also correct the translation manually as follows:

Object type [abbreviation]: RPT4

Object Name: SAPMF40K

To translate the texts, call the translation transaction SE63.

  1. Enter /oSE63 in the command field (transaction field) and press Enter.
  2. In the command field, enter the object type ID (see above) and press Enter.
  3. In the Object Name field, enter the object name (see above).
  4. Set the source language to deDE and the target language to enUS (or the desired target language).
  5. Choose Edit.
  6. Enter the relevant translations for the following fields:
    For language enUS:

    TEXT L85 30  
    deDE   Subledger not to be posted
    enUS   Subledger not to be posted
  7. Choose Save to save the translations and Back (F3) to exit the translation editor.

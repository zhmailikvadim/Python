Symptom

After calling method DELETE_ALL_NODES from class CL_ALV_TREE_BASE, the internal m_node_key_count attribute is not reset.

This leads to ever‑growing node keys when rebuilding the ALV Tree.

Other Terms

ALV; ABAP List Viewer; SAP List Viewer; CL_GUI_ALV_TREE; CL_ALV_TREE_BASE; ADD_NODE; ADD_MODEL_NODE; m_node_key_count; node key counter;

Reason and Prerequisites

Reason

The internal counter m_node_key_count is only initialized when a tree object is created.

It is incremented each time a node is added and is not reset by DELETE_ALL_NODES.

Solution

A new optional parameter RESET_NODE_KEYS has been added to DELETE_ALL_NODES of class CL_ALV_TREE_BASE.

  • Default: ABAP_FALSE. Existing behavior unchanged.
  • If set to ABAP_TRUE, the internal node key counter is reset, so the next added node starts again at key 1.

This provides a controlled way to restart key numbering without needing to recreate the ALV Tree instance.

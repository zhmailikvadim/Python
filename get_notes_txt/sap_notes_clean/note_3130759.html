Symptom

This SAP Note provides a practical guide to transporting objects in SAP BW bridge using gCTS, highlighting key concepts and best practices to ensure a smooth and efficient development process.

Solution

In SAP BW bridge, transporting objects is managed via gCTS, the git-enabled Change and Transport System. To transport objects between tenants, they are stored in a central git repository that is provided and maintained by SAP.

Releasing a transport request pushes the objects of that transport request into the central git repository. To synchronize these changes with a target system, a pull operation is performed, which updates the target system with the latest objects from the repository.

These pull operations are executed at the level of a software component. A software component represents a Git repository. You can manage software components and perform pull operations using the Manage Software Components app in the SAP BW Bridge Cockpit.

Creating a software component automatically generates a top-level structure package in ABAP with the same name. However, you cannot directly create BW bridge or ABAP development objects in this structure package. Instead, these objects must be stored in a development package, which must be manually created as a sub-package of the generated structure package.

For a comprehensive overview of the transport process, including key concepts, prerequisites, and a step-by-step setup guide, refer to the blog post:

How to Transport in an SAP BW Bridge System Landscape 

This resource also addresses frequently asked questions and is essential reading before beginning any development work in BW bridge.

Important: Not following the recommendations outlined in the blog can lead to significant overhead and delays in your implementation project.

In most BW bridge implementation projects, a single software component should be sufficient.

Symptom

Data products in SAP Business Data Cloud (BDC) contain personal data replicated from S/4HANA Public Cloud. If access to this personal data has been blocked in S/4HANA Public Cloud, it will NOT be blocked or deleted in SAP Business Data Cloud.

Other Terms

  • Personal data

  • Blocking of personal data

Reason and Prerequisites

Data products designed with data replicated from S/4HANA Public Cloud may contain personal data. If access is blocked in S/4HANA Public Cloud, these data remain accessible in BDC. Applications in SAP Business Data Cloud can still fully access and use these data for analytical purposes, even if access is blocked in S4 HANA Public Cloud.

Information about blocked access to personal data in S/4HANA Public Cloud is not transferred to SAP Business Data Cloud. Currently, SAP Business Data Cloud does not have a function to directly block access to personal data if it was previously blocked in S/4HANA Public Cloud.

Solution

Currently, there is no solution that eliminates the cause described above. Before selecting data products for uploading to BDC, review the types of personal data included once replicated from S/4HANA Public Cloud. This allows you to decide whether to refrain from uploading those data products into SAP Business Data Cloud.

For blocking purpose, data in S/4HANA Public Cloud can be first archived and then deleted on S/4HANA data base tables. When data is deleted in S/4HANA data base tables then those data will be deleted in SAP Business Data Cloud as well. Therefore, data that is still required in SAP Business Data Cloud must not be deleted in S/4HANA Public Cloud.

To evaluate if a data product contains personal data, please look up the data product information provided in the SAP Business Accelerator Hub:

1. Go to “Data products” in the SAP Business Accelerator Hub: Data Product | Data Products | SAP Business Accelerator Hub

2. Select the data product you are about to onboard in SAP Business Data Cloud

3. Click on the hyperlinked name of the data product listed under “Output Ports” > “APIs/Events”

4. Under “API resources” > “API specification” you can download the JSON file which contains all information about the data types that are provided by this data product.

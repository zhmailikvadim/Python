Symptom

  • Direct access to SAP Ariba application URLs works without issues.
  • Switching between SAP Ariba realms within the application works as expected.
  • Launching the same SAP Ariba URLs via the IdP tiles fails.
  • When accessed through the IdP tiles, the flow returns an HTTP 404 (Not Found) error.

Environment

Single sign On

Reproducing the Issue

  • Open IdP tiles page.
  • Access the respective application which is configured with SAP Ariba URLs.
  • 404 error will be thrown.

Cause

IdP tile configuration for the SAP Ariba realm is not configured correctly.

Resolution

When direct link and switching between the realm works without any issue, this behaviour indicates the issue is with the IdP tile configuration for the specific SAP Ariba realm/environment.

Ask your IT/IdP administrators to review configuration with the SAP Ariba tile. Set the launch URL to the exact, working direct SAP Ariba URL for the target realm.Verify SSO settings align with that respective SAP Ariba environment.

Matching the tile configuration to the working direct link typically resolves the 404 error.

Symptom

Runtime dump CONVT_NO_NUMBER occurs during execution of function module /TMAB4/IMP_FREE.

Other Terms

CONVT_NO_NUMBER, /TMAB4/SAPLIMP_RFC_FACADE, /TMAB4/IMP_FREE, MC_ERR_MSG_TYPE_ERROR

Reason and Prerequisites

The dump is caused by a data type mismatch in an assignment statement.

The return code is defined as type INT4 while mc_err_msg_type_error is defined as type E (character type). The system attempts to convert the character value to a numeric field which leads to runtime error CONVT_NO_NUMBER.

Solution

Kindly Implement the correction instructions attached.

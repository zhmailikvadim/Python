Symptom

An error occurs: "Cannot create ABAP session. The number of ABAP sessions has been exceeded." while running a data flow. Task chains/dataflows run successfully when triggered sequentially but encounter issues during parallel execution.

Environment

SAP Datasphere

Reproducing the Issue

  1. Execute parallel data flows or replication flows from public S4/HANA cloud to SAP Data Intelligence.
  2. Monitor the data flow execution in the Data Integration Monitor for errors.
  3. Observe the error message: "Cannot create ABAP session. The number of ABAP sessions has been exceeded."

Cause

When the source is a S4HANA Public cloud, there exists a limitation. Maximum number of ABAP sessions are fixed and cannot be changed. There isn't a workaround available. However, with S4HANA Private Cloud, the maximum number of ABAP sessions can be increased to manage this scenario.

Resolution

  1. Set different scheduling times for each of the data flows or replication flows to balance the load and ensure the system is not overwhelmed.
  2. If using S4HANA Public Cloud, note that the number of ABAP sessions is fixed and cannot be increased.
  3. For S4HANA Private Cloud, consider increasing the number of ABAP sessions to manage parallel executions effectively.

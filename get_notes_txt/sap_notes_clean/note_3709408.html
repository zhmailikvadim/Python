Symptom

You observe incorrect behavior in deal value restriction processing. (Deal cross-order - Returns update table with positive quantity).  For example, when you assign a value restriction amount to a material, the cross-order table is updated correctly during standard order processing.

However, when a return order is created, the system continues to process it like a regular order instead of treating it as a return. As a result, the consumption value is increased rather than reduced. This behavior is incorrect.

This SAP Note corrects the issue to ensure that return orders are handled properly and consumption values are adjusted accordingly.

Other Terms

Deal, Cross order, returns

Reason and Prerequisites

This is caused by program error.

Solution

Please implement the correction instructions attached. These corrections are only valid if you have applied the Support Packs.

Release V4 - 2023V2, 2023 V3 , and 2023 V3 Support Pack 1.

Use transaction SNOTE to implement the correction instructions specified below and activate the affected program object.

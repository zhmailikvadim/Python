Reproducing the Issue

Deploy project to BTP Cloud Foundry

Cause

Most of the mentioned errors or failed requests (flex, LrepConnector and loadFeatures) are not relevant here. 
The root cause error one is the $metadata failed:

Request URL
...../OData service name/$metadata?sap-value-list=none&sap-language=EN
Request Method
GET
Status Code
404 Not Found

For external or public OData, we need to set up Destinations to refer it:

https://services.odata.org/V2/Northwind/Northwind.svc/

If we directly refers it - https://services.odata.org/V2/Northwind/Northwind.svc/, locally or BAS it will be fine, but after deploying to BTP, it will fails.

Resolution

  1. Create a BTP destination for external or public OData, for example named "btpdestnorthwind" for https://services.odata.org/ 

  2. Then in the UI5 project or during creating Fiori Element when inputting OData, use https://btpdestnorthwind.dest/ to access the odata, example:

    from
    https://services.odata.org/V2/Northwind/Northwind.svc/
    to
    https://btpdestnorthwind.dest/V2/Northwind/Northwind.svc/

  3. npm run deploy-config, gives destination name northwind

    ? Destination Name btpdestnorthwind

    xs-app.json will add the destination:

    {
      "welcomeFile": "/index.html",
      "authenticationMethod": "route",
      "routes": [
        {
          "source": "^/V2/(.*)$",
          "target": "/V2/$1",
          "destination": "btpdestnorthwind",
          "authenticationType": "none",
          "csrfProtection": false
        },
        {
          "source": "^/resources/(.*)$",
          "target": "/resources/$1",
          "authenticationType": "none",
          "destination": "ui5"
        },
        {
          "source": "^/test-resources/(.*)$",
          "target": "/test-resources/$1",
          "authenticationType": "none",
          "destination": "ui5"
        },
        {
          "source": "^(.*)$",
          "target": "$1",
          "service": "html5-apps-repo-rt",
          "authenticationType": "xsuaa"
        }
      ]
    }

Symptom

When you run the reprocess function through the Reprocess Object Version workbench (/vta/bdovrps) in background mode with parallel execution and maximum sessions, you expect the system to create the appropriate number of child jobs to process in parallel. However, the system creates only one child job with a single Data Object and concludes the process as successful.

This note addresses the issue and ensures that child jobs are created correctly based on the configured parallel run settings.

Other Terms

/VTA/BDO_VERSION_REPROCESS, OBJECT_VERSION_REPROCESS, PARALLEL_BJOBS_EXECUTE, Data Objects, Background Job

Reason and Prerequisites

This is caused by program error.

For ECC, SAP Note #3636193 is available

Solution

Please implement the correction instructions attached. These corrections are only valid if you have applied the Support Pack(s).

Release V4 - 2023 V2, 2023 V3 or Support Pack 1

Use transaction SNOTE to implement the correction instructions specified below and activate the affected program object.

Symptom

ODATA API 'API_PROCESS_ORDER_2_SRV' contains time value '24:00:00' which leads to import errors in receiving systems.

Reason and Prerequisites

24:00:00 is a valid value within SAP back-end system (see SAP Note 3028240).

Nevertheless some external systems may prefer to receive

  • '23:59:59' of same day 
  • '00:00:00' of next day 

Solution

With this SAP Note you can activate the conversion of '24:00:00' within the outgoing SOAP message, the ODATA GET or the operations to create a production order through the OData API.

To activate conversion start transaction S_ER9_52003822 or SM30 for table PPSOA_TIME_CONV and create a new entry for the relevant service.

The F4 help for service name will show the list of services for which conversion is supported. You need to create an entry for service 'API_PROCESS_ORDER_2_SRV'

The field conversion mode offers 3 options (the texts will only appear when you are logged on in English)

  • " " -> Keep 24:00 (default)
  • "N" -> Move to Next Day 00:00:00
  • "S" -> Move to Same Day 23:59:59

Symptom

When distributing MATQM IDOCs from ERP to EWM using QL11 transaction, the created inspection rule has flag "Cross-Warehouse" automatically set.

Environment

  • SAP S/4HANA
  • SAP Extended Warehouse Management (SCM-EWM)
  • SAP Extended Warehouse Management Quality Management (SCM-EWM-QM)

Reproducing the Issue

  1. On ERP side, execute transaction "QL11";
  2. Input the required / necessary data to be distributed;
  3. Distribute and create the IDOC on EWM side;
  4. Check the created inspection rule;
  5. Cross-Warehouse checkbox is set.

Cause

Standard behavior.

Resolution

All inspection rules created from MATQM IDOC will be set with "Cross-Warehouse" flag, as the storage location data is not considered in this approach.

In order to create non Cross-warehouse insp rules, quality info records can be used for GR (goods receipt) control, or direct creation on EWM side in /SCWM/QRSETUP transaction for general insp rules.

Symptom

During assemble components in POD SFC TOUCH after set value to barcode input field and press 'ENTER' key the value will not be parsed.

Other Terms

POD SFC TOUCH, ENTER hotkey, Barcode, DC500

Reason and Prerequisites

Steps to reproduce:
1. Create BOM with material BOM component.
2. Create SFC with BOM.
3. Open 'POD - SCF Touch' and retrieve SFC.
4. Click 'Assemble'.
5. Choose assembled component.
6. Set Barcode for parsing.
7. Press 'ENTER'.

Expected results: 'External Serial' input field has been field the value from parsed barcode. 

Actual results: 'External Serial' input field is empty.

Solution

The solution is provided in the form of a reverted previous fix for SAP Note 3589546 - Data Collection Entry plugin does not Save values when ENTER key is pressed in a Touch POD. The hotkey global processing has been removed.

Impact

The code change implemented to address the symptom above may impact below functionalities: NC540, STANDALONE_AS_BUILT, STANDALONE_ASB_TAB, STNDALONE_ASB_TABRPT, DC551, DEF_OPER_POD, DEF_VTR_POD, DEF_INTEGRATED_POD, SUPV_WK_ASSIGN, DEF_WC_POD, DEF_SFC_POD, DEF_SHOPWB_POD, MESSAGE_BOARD, TOUCH_OPER_POD, TOUCH_WC_POD, TOUCH_SFC_POD activities.

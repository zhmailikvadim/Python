Symptom

  • During system copy for Netweaver 7.0X version, there are following error in sapinst_dev.log:
    Warning: Warning during command execution:No archive file exists for development component 'DocumentServicesSupportFiles'/'com.adobe'/'Adobe Systems'/'802.<Timestamp>'/'0'. Cannot pack this archive. 
    Missing archives may lead to errors after the packed SDM installation has been extracted later.
    Info: This SDM installation has been packed to D:\2017XXXXX\JAVA\SDM\SDMKIT.JAR
    To unpack this installation again, extract D:\2017XXXXX\JAVA\SDM\SDMKIT.JAR and invoke the SDM command install with parameter extractpackedinst=true
    Executed, but finished with warning(s). Return code: 2
    Error: Executed, but finished with warning(s). Return code: 2
  • You can also find same error in log sdmlog<Timestamp>.log from usr\sap\<SID>\<Instance>\SDM\program\log\

Environment

  • SAP NetWeaver 7.0
  • SAP enhancement package 1 for SAP NetWeaver 7.0
  • SAP enhancement package 2 for SAP NetWeaver 7.0
  • SAP enhancement package 3 for SAP NetWeaver 7.0

Cause

The root cause of the issue is "No archive file exists for development component 'DocumentServicesSupportFiles'/'com.adobe'/'Adobe Systems'/'802.<Timestamp>'/'0'. Cannot pack this archive. it means there is no such deployment file in the SDM repository, the missing deployment file is for component 'com.adobe'  and the location of the file should be:

\usr\sap\<SID>\<Instance>\SDM\root\origin\com.adobe\DocumentServicesSupportFiles\Adobe Systems\0\802.<Timestamp>

The location can be easily found per the deployment component name in the error log. From above example, the component name is 'DocumentServicesSupportFiles'/'com.adobe'/'Adobe Systems'/'802.<Timestamp>'/'0',  so the location should be "\usr\sap\<SID>\<Instance>\SDM\root\origin\com.adobe\DocumentServicesSupportFiles\Adobe Systems\0\802.<Timestamp>".

Above is one example for failed component, if you have different application in the error log, use the similar way to find the location.

Resolution

  • If you have another system which has exact same version and you can find the missing SDA file in the following location from that system, easily it can be copied to the current system to continue upgrade activity.
 \usr\sap\<SID>\<Instance>\SDM\root\origin\com.adobe\DocumentServicesSupportFiles\Adobe Systems\0\802.<Timestamp>
  • If there is no another system which has same version and could find the missing file, check solution in note 762882 to download the SCA file from service Marketplace and extract the missing file from SCA file.

Symptom

The eDocument is stuck in EDOC_COCKPIT with Cancellation Requested status in India GST Reporting Node.

Environment

  • SAP ERP
  • SAP S/4HANA
  • SAP S/4HANA Cloud Private Edition
  • Digital Compliance Service for India

Reproducing the Issue

  1. Execute EDOC_COCKPIT
  2. Go to India GST Reporting Node
  3. Select any document in "Uploaded to DCS" status
  4. Click on Cancel
  5. Click on Get Status.

Cause

How-To

Resolution

This is because the invoice is already there in the GSTN portal. 

When the cancellation is triggered from edoc_cockpit, it reached DCS. In DCS, all the tax values will be 0 for this invoice since the cancel request has reached DCS. The same has to reach the GSTN portal as well.

For this to happen, Create invoice file for this invoice so that it reaches GSTN portal to change the invoice as cancelled/deleted in GSTN portal. Once this invoice file reaches GSTN portal, it will be corrected in GSTN portal and the invoice will be marked as cancelled/deleted in GSTN portal.
This status should be updated back to EDOC_COCKPIT by performing "Get status". Post that the invoice will be in cancelled status.

Refer to the SAP help portal for creation of invoice file: Save Outward Supply Details | SAP Help Portal

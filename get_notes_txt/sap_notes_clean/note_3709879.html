Symptom

  • Replication flow from HANA column table to SAP Datasphere fails during source setup.
  • Error message: "Failed to prepare source replication artifacts due to database error. (SQL error code = 260, SQL error msg = SAP DBTech JDBC: [260]: invalid column name: HAS_PARAMETERS: line 1 col 21 (at pos 20)). Contact the system administrator for the HANA system to address the issue and then resume the replication task."

Environment

  • SAP Datasphere
  • RMS connectivity

Reproducing the Issue

  1. Execute the replication flow.
  2. Observe the error message indicating failure due to invalid column name "HAS_PARAMETERS".

Cause

The HANA version is too old and does not support the "HAS_PARAMETERS" field which was introduced since HANA 2.0 SPS 04, and it's required by the replication flow.

Resolution

  1. Upgrade the HANA instance to a supported version (Hana 2.0 SPS 04 or higher), where the "HAS_PARAMETERS" field is available.
  2. Alternatively, use Data Flows as a workaround for the replication task if upgrading is not feasible.

Symptom

  • Memory continuously grows for non-java servers even when the servers are idle
  • CORBA SSL is enabled on the servers
  • Memory is not released until the servers are restarted
  • Increased load results in faster memory accumulation

Other Terms

TLS, Security, ssl, high, out of memory, CORBA, leak, memory leak, FRS, Input, output, crcache, crproc, eventserver

Reason and Prerequisites

Environment

SAP BusinessObjects Business Intelligence Platform 4.2 SP05

Reproducing the Issue

  1. Enable SSL on all servers in the environment
  2. Restart all servers in the environment
  3. Once memory utilization stabilizes and the system is idle, record the memory usage of BI Servers
  4. Allow the system to idle for a few hours
  5. Observe that memory usage grows over time

Solution

This issue is fixed in the patches listed in the "Support Packages & Patches" section below.

The "Support Packages & Patches" section will be populated with the relevant patch levels once they are released.

For Business Intelligence Platform maintenance schedule and strategy see the Knowledge Base Article 2144559 in References section.


Workarounds:

  • Periodically restart the servers in the environment
  • Disable SSL

Symptom

On the overview page, the KPI header for logged hours and the time capture section display an incorrect aggregated value when there is a time zone difference between the backend and the device. Due to this difference, the local date may vary between the two locations, leading to inconsistent calculation results.

Reason and Prerequisites

Reason: The calculation of logged working hours is performed using the backend time zone instead of the device time zone.

Prerequisites: 

  • BAS or Visual Studio Code with SAM MDK project
  • `patch` command (available on BAS and macOS by default) via terminal or other patching utility
  • Read the guidelines on how to apply the patch and go through the appendix section if you get any errors.

Solution

With the implemented solution, the user device's time zone is taken into account. As a result, the logged working hours are calculated and displayed based on the user’s local time zone, ensuring that daily working hours reflect the actual values for the user’s location.

To apply the patch, download the attached 3732802.patch file and save it in a known location, preferably adjacent to your SAM.mdkproject.

If you are on Windows, see SAP Note 3533879 for how to install a .patch file in Windows. If you are on a Mac, follow the steps below:

  1. Open a terminal in the root of your SAM metadata (i.e. SAM.mdkproject/metadata or SAM.mdkproject/metadata/base.definitions)
  2. Run the terminal command `patch --no-backup-if-mismatch -p4 < path/to/3732802.patch`. 
  3. Verify in the terminal window that the following files have been modified and listed as patched, as indicated by the console output:
  • patching file 'Rules/OverviewPage/KPI/TimeCount.js'
  • patching file 'Rules/OverviewPage/TimeCaptureSection/TimeCaptureSectionTodayHours.js'
  • patching file 'Rules/OverviewPage/TimeCaptureSection/TimeCaptureSectionYesterdayHours.js'

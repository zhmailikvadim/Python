Symptom

I. The report "Přehled zdravotního pojištění" (RPCZPLT0) shows incorrect amount in part "Celkem bez zalozeni ucasti" and "Celkem brutto bez pojištění".

II. Wage type M990 - Forced retro of Tax+SI (CZ: Vynucený přepočet daně) is hard coded in the report.

III. Report is not processing all personal numbers which are connected via IT0031 and IT0121 if they are not all listed in selection screen. Report should always process all personal numbers which belongs together.

Other Terms

HRCZ, RPCZPLT0, "Celkem bez zalozeni ucasti", "Celkem brutto bez pojisteni", M990

Reason and Prerequisites

Problems were considered as program error and modifications.

Solution

I. Displaying of amounts in parts "Celkem bez zalozeni ucasti" and "Celkem brutto bez pojištění" were corrected.

II. New method for determine customer wagte type M990 was introduced.

III. Report determines all personal numbers which belongs together.

Symptom

liveCache functionality to manually adjust the notified and confirmed quantities of a delivery plan.

Other Terms

/SAPAPO/OM_DELIVERY_MESSAGE, goods receipts, delivery plan

Reason and Prerequisites

The total notified or confirmed quantities of a delivery plan need to be adjusted manually to correct inconsistencies.

Solution

Implement this note to add two optional import parameters

  • IV_METHOD_CONF (method to change the confirmed quantity)
  • IV_METHOD_AVIS (method to change the notified quantity)

to the liveCache interface module /SAPAPO/OM_DELIVERY_MESSAGE. These methods default to 'A' (add), i.e. the quantities are added to the existing ones stored in the liveCache. This is the legacy behavior. To perform an absolute quantity change, i.e. to replace the existing quantities with the ones specified, use method 'C' (gc_change) instead.

The liveCache side functionality required for absolute changes is shipped as of liveCache builds 101.26 (see note 2074843) and 100.58 (see note 2074842) respectively. Use of method 'C' is not intended for business purposes and is not supported within a transactional simulation (simsession).

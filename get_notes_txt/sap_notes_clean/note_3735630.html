Symptom

When you access the Pipeline Manager, you notice that custom fields display their technical names instead of their user-friendly labels.

Environment

SAP Sales Cloud Version 2

Reproducing the Issue

  1. Open the Pipeline Manager.
  2. Observe that the custom field's technical name is displayed in the column below the chart instead of the custom field's label.
  3. To confirm, check the custom field within the Extensibility Administration settings (under Opportunity) and notice that the technical name differs from the label, causing the incorrect display.

Cause

The Opportunity and Pipeline entities share the same extensions anchor, which leads to reusing extension fields across both. The label for the field in the Pipeline entity is maintained as the technical name, resulting in the technical name being displayed instead of the intended user-friendly label.

Resolution

The Opportunity and Pipeline entities share the same extensions anchor, which leads to reusing extension fields across both. The label for the field in the Pipeline entity is maintained as the technical name, resulting in the technical name being displayed instead of the intended user-friendly label.

Option 1: Maintain the Label in Extensibility

  1. Navigate to Extensibility Administration > Pipeline > Pipeline.
  2. Open Custom Fields and locate the affected extension field.
  3. Update the label to the desired text.
  4. Save your changes and verify the label in the Pipeline Manager.

After updating the label in extensibility, the Pipeline Manager should now display the correct, user-friendly label for the custom field.

Option 2: Use Language Adaptation

  1. Go to Settings > Language Adaptation.
  2. Select the application “PipelineCard” (sap.crm.pipelineservice.entity.pipeline).
  3. Set both the source and target language (e.g., English).
  4. Search for the field by name.
  5. Update the label in the language text to the desired value.
  6. Save your changes and verify the update in the Pipeline Manager.

After applying the language adaptation changes, the Pipeline Manager should now display the correct, user-friendly label for the custom field.

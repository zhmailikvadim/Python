Symptom

To DRTM Master Data purge a candidate, it must have the value "d", within Data Inspector - Table_USERS_SYSINFO table's "valid" column. Following the below KBA to change the value to "d", even though the job runs successfully, the value won't change.
3141240 - How to change the status of the user from active external suite (e) to Inactive external suite (d) - Onboarding User Purge - SAP for Me 

Environment

SAP SuccessFactors Onboarding

Reproducing the Issue

  1. Following steps in KBA 3141240
  2. The job successfully runs
  3. The value doesn't change to "d"

Cause

One of the reasons this could be, is the fact the candidate is a Legal Entity Transfer For New Employment. For such users the job won't take affect.

Resolution

Need to run a script to change the value manually, please reach out to Product Support.

Symptom

  • Attempting to disable "Require email verification" in Identity Verification for a child site results in an error message: "Error details: Http failure response for https://accounts.eu1.gigya.com/accounts.setPolicies: 0 Unknown Error".
  • The system does not allow overriding parent settings for email verification in child sites when the parent site has enabled email-based Two-Factor Authentication (TFA).

Environment

Customer Data Cloud

Privacy & Safety (Consent, RBA - Risk-Based Authentication)

Reproducing the Issue

  1. Navigate to the Identity Verification settings for the child site within Customer Data Cloud console.
  2. Attempt to disable "Require email verification" under the child site settings.
  3. Save the changes.
  4. Observe the error message: "Error details: Http failure response for https://accounts.eu1.gigya.com/accounts.setPolicies: 0 Unknown Error".

Cause

  • The parent site has enabled email-based TFA, which requires email verification for all sites within the group. This design ensures consistency in TFA enforcement across the group.

Resolution

  1. Confirm that the parent site has email-based TFA enabled.
  2. Understand that email verification cannot be disabled for child sites when email-based TFA is enforced at the parent level.
  3. If overriding parent settings is necessary, switch the TFA verification method to SMS, which allows phone number verification and does not enforce email verification across the group.
  4. Save the changes under Identity Verification settings after switching the verification method.

  • Refer to: SAP Customer Data Cloud documentation on TFA verification methods.

Symptom

Users are not able to see User Role Tab in Manage Career Site Builder. 

Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.

Environment

SAP SuccessFactors Recruiting Marketing

Reproducing the Issue

  1. Navigate to Manage Career Site Builder from Admin Center.
  2. Attempt to locate the User tab > Role.
  3. Observe that the Role tab is not visible.

Cause

CSB Role Based Permission is enabled in CSB and the user is part of a role. 

Resolution

Once a CSB user has is added to a "Role" when Role Base Permission is enabled in CSB, the user will only be able to see only the permission that has been granted to the role (eg. Appearance Settings, Appearance Styles). The user will not be able to see the User Role tab and there is no option to grant users permission to this tab. In order for a user who has been assigned with a role in CSB to see the User Role Tab, the only option is to make the user admin by enabling "Is CSB admin" for the account of the user. 

Note:

  • Changes can only be done by someone who has their account "Is CSB Admin" or Implementation partner who can access CSB from command center.
  • After making changes on the account of users in CSB, CSB admins can manually run Recruiter sync, otherwise, the changes might take effect within the next 24 hours. 

To enable "Is CSB Admin" for a user: 

  1. Go to Career site builder 
  2. Go to Roles>Admin users
  3. Click on the Pencil Icon or Edit User button 
  4. Toggle is CSB Admin and Save

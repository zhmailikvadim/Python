Symptom

  • In the generated XML for Poland KSeF, the field P_9A (net value) does not include amounts derived from statistical pricing conditions.
  • The net value reflected in the XML differs from the expected net value calculated in the billing document.
  • Statistical condition values are not considered during eDocument mapping.

Environment

  • SAP ERP
  • SAP S/4HANA
  • SAP Document and Reporting Compliance Service

Reproducing the Issue

  1. Maintain pricing procedure with statistical condition types contributing to net value.
  2. Create billing document.
  3. Generate eDocument via EDOC_COCKPIT.
  4. Display XML output.
  5. Observe that statistical values are not included in field P_9A.

Cause

Standard mapping logic for Poland KSeF does not consider statistical pricing conditions when determining the net value (P_9A).

Resolution

This is standard system behavior.

Statistical pricing conditions are not considered in the standard mapping logic for field P_9A (net value) in Poland KSeF.

If inclusion of such values is required, this must be handled via enhancement using:

  • BAdI EDOC_ADAPTOR

  • Method SET_OUTPUT_DATA

SAP standard does not provide configuration to include statistical conditions in this mapping.

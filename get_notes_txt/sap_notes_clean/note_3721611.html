Symptom

  • The `X-CSRF-Token` received by the Sender Adapter is not accessible within the iFlow runtime environment.
  • The token is not visible in the message processing log or in the headers available to scripts.

Environment

SAP Cloud Integration

Reproducing the Issue

  1. Enable trace on the iFlow.
  2. Attempt to locate the `X-CSRF-Token` in the message processing log or in the headers available to scripts.

Cause

The `X-CSRF-Token` received by the OData Sender Adapter is consumed and validated at the adapter layer for security purposes and is stripped before the message enters the iFlow.

Resolution

  1. As an alternative approach, configure the client to perform a preliminary GET request with the header `X-CSRF-Token: Fetch`.
  2. Capture the returned `X-CSRF-Token` from the GET request and include it in subsequent modifying requests using a custom header, such as `My-CSRF-Token`.
  3. Access the custom header within the iFlow using scripts, for example, `message.getHeaders().get("My-CSRF-Token")`.

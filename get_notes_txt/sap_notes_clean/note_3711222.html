Symptom

  • Unknown users appear in the SAP BTP Global Account user list.
  • The users were not created by the organization and their origin is unclear.

Environment

SAP BTP, Cloud Foundry runtime and environment

Reproducing the Issue

  1. Access the SAP BTP Global Account.
  2. Go to Security.
  3. Go to user.
  4. Identified several users in  BTP Global Account.

Cause

User did not created and that do not belong to organization. 

Resolution

  1. Access the SAP BTP Global Account.
  2. Go to Security
  3. Choose your custom or external trust configuration.
  4. Choose Edit.
  5. Go to Create Shadow Users on User Logon and remove the checkmark.
  6. Save your changes.
  7. From now on, the User Account and Authentication service doesn’t automatically create shadow users for a certain identity provider during logon.

Note :- 

  • If automatic creation of shadow users is switched off, users must be created explicitly, and only those users can log on.
  • In global account level if using default idp then cannot Switch Off Automatic Creation of Shadow Users on user logon.
  • The user shown under global account are all platform user. For platform user, can't turn off shadow user creation via UI. Need to do it via API Identity Provider Management
    Credentials for the API call can be gained via Managing API Credentials for Calling REST APIs of SAP Authorization and Trust Management Service

Symptom

  • When creating an HTTP destination in a copied client using transaction /SDF/ALM_SETUP, the system raises: "HTTP destination <destination name> is already used by <destination target ALM> !"
  • The target client (for example, client 200) was created via client copy from a previously configured client (for example, client 100).

Environment

  • SAP S/4HANA
  • SAP S/4HANA Cloud Private Edition
  • SAP NetWeaver Application Server for ABAP (7.40 or higher)
  • SAP Business Suite 7

Cause

During the client copy, the Cloud ALM identifier for the source logical system was copied into table /SDF/CALM_ID. When creating HTTP destinations in the new client, the setup detects the existing entry and reports that the destination is already used.

Resolution

  1. In the target client, open table /SDF/CALM_ID in SE16
  2. Identify the entry that still references the source client/logical system and remove the obsolete entry in the target client.
  3. Run /SDF/ALM_SETUP again and create the HTTP destination for the target client.

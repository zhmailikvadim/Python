Symptom

When FSM Activity is created in SAP and the flag /PACG/ECM_CPROP-FSM_PRIO_ACT_DT is active, the Due Date and Earliest Start Date are not transferred correctly to FSM. As a result, the Activity is replicated to FSM without these dates.

Other Terms

FSM, E4C, Connector, Activity, Due Date, Earliest Start Date

Reason and Prerequisites

When the flag /PACG/ECM_CPROP-FSM_PRIO_ACT_DT is active, the system is expected to transfer the Due Date and Earliest Start Date from table /PACG/ECM_ACTI. However, if these fields are empty, Activities created in SAP still apply the date transfer logic based on these empty table entries, which may lead to missing date information in the FSM Activity.

Solution

If the entries in /PACG/ECM_ACTI are empty, the system applies the previous ("old") logic. In this case, option /PACG/ECM_VSCTD-DATES_ALLOC determines the two dates. For Activities created in SAP, the logic based on the Dates Allocation option is always applied. Please, implement this correction instruction.

Symptom

User receives "HTTP Status 500 - org.apache.jasper.JasperException" error when attempting to view a reccuring instance for Platform Search object, when the recurring object is based on a Calander.

Other Terms

java.util.MissingResourceException: Can't find resource for bundle java.util.PropertyResourceBundle, key schedule.calendar.jan

Reason and Prerequisites

Environment:

SAP BusinessObjects Business Intelligence Platform 4.1

Steps to Reproduce:

  • Login to CMC
  • Select Calanders from the drop down list.
  • From the "Manage" menu click on "New" > "New Calendar". Give the Calander a name and save.
  • Right click on the calander and choose set dates. Set a few dates including the current date for the instance to run on. "Save and Close".
  • From the drop down menu, select folders and navigate to Platform Search Scheduling. 
  • Schedule the platform search Scheduling Objects against the calander as a recurring object.
  • Once the child objects appear under the history of the Platform Search Scheduling Object in the history page, Click on "Recurring" to view the recurring object. 
  • Observe the error message.

Cause: 

  • Due to missing string resources in the Application jar there were exceptions that were occuring.

Solution

This issue is fixed in the SPs listed in the "Support Packages & Patches" section below.
The "Support Packages & Patches" section will be populated with the relevant patch levels once they are released.

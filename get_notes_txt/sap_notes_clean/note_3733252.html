Symptom

Enable Print Forms Generation - S-User IDs using for different instances

Environment

 SAP SuccessFactors Onboarding

Reproducing the Issue

Raise a request for the print form using the same S-user/I-user credentials

Cause

P-User/S-User usage query for print forms service

Resolution

  1. Use a dedicated P-User/S-User per customer; the same P-User/S-User cannot be shared across multiple clients because the print forms service (PFS) URL is generated based on the customer tenant ID, DC, and customer ID.
  2. Do not reuse the same P-User/S-User across instances of the same customer (for example, preview and production); use different credentials per instance.
  3. Maintain a 1:1 mapping: each PFS URL requires its own unique P-User/S-User.

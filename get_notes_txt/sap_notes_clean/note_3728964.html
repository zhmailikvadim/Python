Symptom

Facing below error while registering the SLD,

com.sap.aii.utilxi.rtcheck.registertest.RegisterTestServlet.doPost: Exception while performing SLD registration.
Thrown:
com.sap.aii.utilxi.rtcheck.registertest.SLDAccessException: Unable to register the application AdapterFramework with SLD

Environment

SAP NetWeaver Process Integration

Cause

The issue is caused by an inconsistent or incomplete registration of the PI/PO components in the System Landscape Directory (SLD).

Resolution

  1. Log in to the SLD using the URL: `http(s)://host:port/sld`.

  2. Navigate to "Technical Systems" and select the type "Process Integration".

  3. Delete the following entries associated with the affected system:
    • Adapter Engine
    • Domain
    • Integration Directory
    • Integration Repository
    • Integration Server
    • Runtime Workbench (RWB)

  4. Navigate to "Business Systems" in the SLD and delete the entry "INTEGRATION_ENGINE_JAVA_<SID> ".

  5. Access the NetWeaver Administrator (NWA) and navigate to "Configuration → Scenarios → Configuration Wizard".
    • Select "[All Configuration Tasks]" from the dropdown list and according to your system type, re-register PI/PO system components in SLD  using the dedicated configuration wizard in NWA:
      • For Single stack (AS JAVA) System "PI AEX SLD Self Registration".
        For more details refer SAP Note 2034226 - Configuration Wizard: PI AEX SLD Self Registration

      • For Dual stack or Split stack System "PI SLD Self Registration".
        For more details refer SAP Note 1292803 - Configuration Wizard: PI SLD Self Registration
    • Complete the wizard with the necessary configuration and parameters to re-register the PI/PO components in the SLD.
    • Purpose of this automation task is to register the PO system again in the SLD, this is required when the current SLD registration is inconsistent or some configuration changes are needed.

  6. Verify that all six technical systems (XI Adapter Engine, XI Integration Directory, XI Integration Repository, XI Integration Server, XI Runtime Management Server, and XI Domain) are correctly registered in the SLD. 
    For more details refer SAP wiki How to Handle SLD in PI landscape

  7. Refresh the Java Virtual Machine (JVM) cache:
    • Open directory page i.e. http(s)://host:port/dir/start/index.jsp
    • Then navigate to "Administration" section
    • Navigate to "Java Virtual Machine Cache"
    • Select all caches and refresh them on all three tabs (Repository, Directory, Runtime).

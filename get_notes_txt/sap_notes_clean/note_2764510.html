Symptom

DB2 Version 11, RSU level is greater than RSU1612(NFM):

  1. With LTMCFG parameter DataSharingOption=None, and without DataSharingMember parameter in LTMCFG, Replication Agent fails to start.
  2. With Parameter DataSharingOption=Single in LTMCFG, Replication Agent fails to start.

Other Terms

Environment:

SAP HANA SDI 2.0 SP03 Patch 6 (SDI DB2 Mainframe Adapter Version 2.3.6)

SAP HANA SDI 2.0 SP04 (SDI DB2 Mainframe adapter Version 2.4.0)

Steps to reproduce:

There are two possible scenarios:

  1. In a non-Datasharing environment, set DataSharingOption=None and comment out DataSharingMember in LTMCFG parameter for DB2 version 11, RSU level is greater than RSU1612 and start the Replication Agent.
  2. In a Datasharing environment, set DataSharingOption=Single and DataSharingMember=<Db2 Subsystem name> in LTMCFG parameter for DB2 version 11, RSU level is greater than RSU1612 and start the Replication Agent.

Workaround:

In a non-Datasharing environment, set DataSharingOption=None and DataSharingMember=<DB2 subsystem name> parameters in LTMCFG, and start the Replication Agent.

In a Datasharing environment, set DataSharingOption=Multi, comment out DataSharingMember parameter in LTMCFG and start the Replication Agent.

Solution

The development team is investigating this known issue.

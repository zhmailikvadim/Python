Symptom

You have a requirement to restrict access to Enterprise Projects by Project Manager.

Environment

SAP S/4HANA Cloud Public Edition

Reproducing the Issue

  1. Attempt to restrict access to enterprise projects dynamically based on whether a user is the Project Manager via standard business roles.
  2. Observe that no field exists in project-related roles to restrict access by Project Manager.

Cause

  • SAP S/4HANA Cloud Public Edition does not support dynamic restriction of project access based on whether a user is the Project Manager via standard business roles.
  • Project visibility and access are controlled using predefined restriction types (e.g., company code, project attributes) in business roles, and dynamic filtering by Project Manager is not included in the current delivery.

Resolution

  1. Understand that SAP S/4HANA Cloud Public Edition does not support restricting projects dynamically based on whether a user is the Project Manager via standard business roles.
  2. Utilize project access control settings like Public, Restricted, and Private at project creation to manage visibility and edit rights for unassigned users.
  3. Understand that this situation represents a new feature request and not an error in the current delivery of SAP S/4HANA Cloud Public Edition.

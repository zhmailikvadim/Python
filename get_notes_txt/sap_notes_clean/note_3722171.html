Symptom

  • Delta merge operation for table fails to complete due to 2 billion row limitation.
  • You attempt to partition the table. 
  • Partitioning is blocked by delta merge operation on the same table.

Environment

SAP HANA 1.0

Reproducing the Issue

  1. Perform a delta merge operation on the table which is already close to 2 billion records.
  2. Observe that the delta merge operation does not complete.
  3. Terminate the delta merge operation.
  4. Attempt to partition the table using the command: `ALTER TABLE <table_name> PARTITION BY HASH(BATCH_ID) PARTITIONS 10;`.
  5. Observe that the partitioning process triggers a delta merge that does not complete.

Cause

  • The table already contains more than 2 billion records, which exceeds the single table/partition row limitation in SAP HANA.
  • The delta merge operation fails due to the 2 billion record limit.
  • In HANA 1.0 a table partition operation always triggers a delta merge operation before partitioning the table. Refer to Q 38 of SAP Knowledge Base Article 2044468 - FAQ: SAP HANA Partitioning - SAP for Me

Resolution

    In SAP HANA 1.0, the table has to be recreated as per SAP Knowledge Base Article:  2572224 - How-To: Repairing SAP HANA Tables - SAP for Me

Symptom

Unable to configure the SVTB billing type for preliminary billing documents workflow. 

"Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental."

Environment

SAP S/4HANA Cloud Public Edition

Reproducing the Issue

  1. Access the "Manage Preliminary Billing Document Workflow" app.
  2. Select the Preliminary Billing Document.
  3. Check the drop down menu for billing type SVTB.

Cause

This is the standard system behavior. SVTB is the document type for a billing document request, which is the preceding document of a preliminary billing document. The start condition rules such as "Preliminary billing document type is" and "Sales organization of preliminary billing document is" refer to attributes of the preliminary billing document, not of the preceding document (e.g., a billing document request of type SVTB).

Resolution

As a workaround, for the preliminary billing document workflow get triggered based on the preceding billing document request type, implement the cloud BAdI SD_BIL_APM_SET_APPROVAL_REASON and use it to control whether approval is required:

  1. In the BAdI implementation, retrieve the preceding BDR related to the PBD.

  2. Determine the BDR type and evaluate whether the scenario requires approval.

  3. Set the appropriate billing document approval reason based on this evaluation.

  4. Configure the workflow to start based on the billing document approval reason set in the BAdI.

With this approach, the approval workflow is conditionally triggered according to the preceding BDR type, enabling the required control without changing standard behavior.

Symptom

  • When adding a hyperlink in a story and browsing for a data analyzer insight, users cannot see subfolders that are not shared with them. 
  • This behavior differs from browsing for other items, such as stories, where subfolders appear as if they were at the root level. 

Environment

SAP Analytics Cloud

Reproducing the Issue

  1. Log in to the SAC tenant with a user having restricted access to subfolders under the root public folder.
  2. Verify that the folder containing the insight is accessible in the Files section.
  3. Create a story.
  4. Add a widget, select it, and click on "Add Hyperlink" from the options menu.
  5. Choose "Link to Insights".
  6. Click on "Add Insights".
  7. Observe that the folder containing the insight is not visible in the selection pop-up.
  8. Search for the insight by its name in the search bar.
  9. Observe that the insight becomes visible for selection after searching by its name.

Cause

The issue occurs due to incorrect resource filtering logic in the insights selection screen when adding hyperlinks.
Shared subfolders under the root public folder are not properly accounted for unless the root public folder itself is shared with the user.

Resolution

The fix for this issue is scheduled to be included in the SAC Q2 2026 release, which will be deployed in the middle of May.  

Please refer to SAP Note 2888562 for the release calendar.

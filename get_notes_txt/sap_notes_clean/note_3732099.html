Symptom

In backoffice, the "Switch Search Mode" button overlaps with the "Copy Link" button after disabling auto-search.

Reason and Prerequisites

When the "advanced search" widget loads, it misses the relevant socket event sent by the "Collapsible Container", resulting in certain CSS classes not being applied correctly.

Steps to Reproduce

  1. To disable auto-search, add connection-operator="AND" disable-auto-search="true" to the following configurations:
    component="Csticket", type="advanced-search";
    component="Order", type="advanced-search"
  2. Open Backoffice, Navigate to Customer Support Cockpit
  3. Go to Orders
  4. "Switch Search Mode" button will be overlapping with the "Copy Link” button
  5. Click any button in the search area (e.g. "Lock Section")
  6. The issue is resolved after the interaction

Solution

Adjust the loading logic of "advanced search" widget to retrieve the button state directly from the parent widget, instead of relying on socket events. This ensures that the required CSS classes are correctly applied .

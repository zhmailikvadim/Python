Symptom

  • The application operation got failed with blank page
  • The following info can be seen from HTTPWatch trace:
    Framing 'blob:https://<host>/<URL>' violates the following Content Security Policy directive: "default-src 'self'". The request has been blocked. Note that 'frame-src' was not explicitly set, so 'default-src' is used as a fallback.
  • The issue only occurs when accessing the application through SAP Web Dispatcher, no issue when accessing backend system directly

Environment

SAP Web Dispatcher

Reproducing the Issue

Accessing the application through SAP Web Dispatcher 

Cause

Improper CSP header set by SAP Web Dispatcher Modification Handler

Resolution

Allow content from https://<host> in an iframe.

default-src 'self': This directive specifies that only content from the same origin (i.e., the same domain) is allowed to be loaded. Any attempt to load content from a different origin will be blocked.
frame-src: This directive specifies which sources are allowed to be loaded into <iframe> elements. If frame-src is not explicitly set, the default-src directive is used as a fallback.

Adjust SAP Web Dispatcher Modification Handler setting(icm/HTTP/mod_X parameter) and add "frame-src 'self' https://<host>;" to CSP header.

e.g. 
SetResponseHeader Content-Security-Policy <Existing Header>frame-src 'self' https://<host>;"

Keywords

CSP header Blocked

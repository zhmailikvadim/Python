Symptom

  • Alerts from FRUN with SAPSA user 'sapsa' failed to interact with the database (delete/insert) error. 
  • SAP Host Agent with trace level 3 shows:

    • [PID xxxxx] executeQuery:  --- End of Query ---

      [PID xxxxx] aseDatabaseComponentStatus:  skipAliveCheck NULL

      [PID xxxxx] executeQuery:  Isolation level set to 1

      ...

      [PID xxxxx] Thu Feb  5 05:36:33 2026

      [PID xxxxx] ERROR AT sybOpenClient.cpp:fetchResults:2757 DESCRIPTION: Server message: Number 225, Severity 16, State 11, Line 2, Server BSP --- Message string: Cannot run query--referenced object (name NAME NOT RECOVERABLE) dropped during query optimization.  LAST OS-ERROR: (0) : Success

      [PID xxxxx] ERROR AT sybServer.cpp:aseDatabaseComponentStatus:67206 DESCRIPTION: SAPSA user 'sapsa' failed to interact with the database (delete/insert) LAST OS-ERROR: (0) : Success

Environment

  • SAP BusinessObjects Business Intelligence platform 4.3

  • SAP Adaptive Server Enterprise (ASE) 16.0 for Business Suite

     
     

Cause

Error 225 arises due to concurrent execution leading to table drop/create conflicts, specifically with the sapsa_alive_chk table.

Resolution

Issue is planned to be fixed in Hostagent 7.22 PL71. Update Hostagent according to SAP Note 1031096 - Installing Package SAPHOSTAGENT.

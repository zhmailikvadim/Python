Symptom

Exporting R3load aborts with the error message:

(CNV) ERROR: unknown tabclass for table PKRT_LOAD.

Reason and Prerequisites

PKRT_LOAD table is wrongly classified by R3ldctl as a table class "Y". The table class "Y" existed until kernel version 722 and then was removed. R3load version higher than 722 does not support that table class.

The problem appear only in very specific circumstances, namely when the SAP_BASIS is 730 or older (meaning table PKRT_LOAD does still exists), but the kernel for system export is used 742 or higher (meaning R3load does not support class 'Y' anymore). This situation is not possible with SWPM, where the kernel version of the source and target systems matches the SAP_BASIS version (e.g. 730 SAP_BASIS => 721 kernel). It is also not possible with SUM DMO, because DMO does not support SAP_BASIS 730 as a target release, and in newer releases the table PKRT_LOAD was removed. The only currently known scenario, which triggers the error above is the use of SUM Benchmarking Tool, which can be used with any SAP_BASIS level without any limitation, but for the export it uses a modern kernel, e.g. 745, making new tools to process an old DB payload.

Solution

Workaround:

Locate the object description file (STR), which is used for the failing task (refer to the command file of R3load to find the corresponding STR file) and look for table definition PKRT_LOAD. It will look like this:

tab: PKRT_LOAD
att: SSEXC 9 ?N Yc all PKRT_LOAD~0 SSEXC 9

Change the table class from "Yc" to "Tc" and save the STR file. Retry the job.

Fix:

Update R3ldctl with the patch level higher or equal than the patch level specified in the corresponding section of the current note.

Symptom

  1. Unable to open the attachments from application t-codes, facing below error:
  2. Unable to display the attachments from SAP content server in OAAD.
  3. When check the existence as per step 3-7 in KBA2251113 - Check whether a document exists on SAP Content Server database, in table COMPONENTS, the LVALUE column is set to 0 although the SHORT_VALUE or LONG_VALUE column contains data:

Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.

Environment

All R/3 release.
SAP content server 7.54 SP513.

Reproducing the Issue

  1. Go to OAAD.
  2. Go to Technical Search.
  3. Input document ID and repository ID, execute.
  4. Display from content server.
  5. Got error indicates the documents not existing or unable to view.

Cause

This is a program error happen on SAP content server 7.54 SP513.
Reference: 
3602044 - Memory leak and other errors with SP513
4) Some documents are returned with content-length 0. Looking at the COMPONENTS table, the LVALUE column is set to 0 although the SHORT_VALUE or LONG_VALUE column contains data. 

Resolution

Update SAP content server 7.54 to the latest available SP >= 629.

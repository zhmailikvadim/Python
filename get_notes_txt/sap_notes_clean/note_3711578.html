Symptom

  • Query created using the Custom Analytical Queries application.
    When previewing the query, an error message appears: "Service cannot be used".
    No data is displayed.
  • Running the query in the backend transaction code RSRT results in the following error:
    Error message: "Service cannot be used".
    An exception of type CX_RODPS_OBJECT_NOT_FOUND was raised but was not handled locally or declared in a RAISING clause.
    Error details: ODP - <Field_name> does not exist.
    Position in code: Row 116, Include GET_INSTANCE, Program CL_RODPS_ODP.

Environment

  • SAP S/4HANA Cloud Public Edition 2602
  • SAP S/4HANA Cloud Public Edition 2508

Reproducing the Issue

  • Open the Custom Analytical Queries app.
  • Navigate to the query YY1_XYZ.
  • Select the preview option.
  • Observe the error message: "Service cannot be used."
  • Run the same query in RSRT from the backend, and observe the following error:
    Error message: "Service cannot be used."
    An exception of type CX_RODPS_OBJECT_NOT_FOUND was raised but was not handled locally or declared in a RAISING clause.
    Error details: ODP - <Field_name> does not exist.
    Position in code: Row 116, Include GET_INSTANCE, Program CL_RODPS_ODP.

Cause

There is no selection of hierarchy for the field in the Custom Analytical Queries application; however, the corresponding hierarchy coding has been generated in the CDS view.

Resolution

The developers are working on a code correction for the issue.

3711397- 8.16 SP2: CX_RODPS_OBJECT_NOT_FOUND during activation of CDS query with display hierarchy Off

Hotfix 1000040033 on ABAP PLATFORM CLOUD 2602 04 is ready for delivery.
The Release to Host will be on the weekend 13.02.2026-15.02.2026. 
Please note the release date might be subject to change.

Workaround:
Before an official fix is in place, a recommended workaround is to make some minor adjustments in the Custom Analytical Queries app for CDS view YY1_XYZ. Please make the changes for the  field only. Follow the steps below:

  1. In the Field Selection tab, unselect the field and click the Save Draft button at the bottom right corner. Then, select the field again.
  2. In the Display tab, configure the settings for the field.
  3. In the Filters tab, configure the settings for the field.
  4. Check and publish the query again.

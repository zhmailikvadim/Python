Symptom

A query using GROUPING SETS() with GROUPING() or GROUPING_ID() features returns unexpected results.

Reason and Prerequisites

Reason:

Due to a programming error in SAP HANA, GROUPING() and GROUPING_ID() using constant values can be interpreted incorrectly during the optimization phase at compile time. This may lead to unexpected query result.

Affected Releases:

  • SAP HANA 1:
    • Revisions 122.21 - 122.22 (SPS12)

  • SAP HANA 2:
    • Revision 024.07 (SPS02)
    • Revisions 034.00 - 035.00 (SPS03)

Prerequisites:

  • The query uses constant values for GROUPING() or GROUPING_ID() functions
  • GROUPING SETS() and GROUPING(<constant>) or GROUPING_ID(<constant>) are used in the inner query of a nested query
  • The query projects the GROUPING(<constant>) or GROUPING_ID(<constant>) columns from nested query to the outer query.

    For example, the following query contains constants 1 and 2 for GROUPING() function, and projects G1, G2 from the inner select to the outer select:

    SELECT T1.A, T1.C, GA, GB, G1, G2 FROM
    ( SELECT A, SUM(C) AS C, GROUPING(A) AS GA, GROUPING(1) AS G1 FROM T GROUP BY GROUPING SETS (A,1) ) AS T1,
    ( SELECT A, B, SUM(C) AS C, GROUPING(B) AS GB, GROUPING(2) AS G2 FROM T GROUP BY GROUPING SETS (A,B,2) ) AS T2;

Solution

Apply one of the following SAP HANA Revisions:

  • SAP HANA 1:
    • Revisions >= 122.23 (SPS12)

  • SAP HANA 2:
    • Revisions >= 024.08 (SPS02)
    • Revisions >= 036.00 (SPS03)
    • or higher

  • or higher

Workaround:

  • There is no easy workaround without changing the query. You may consider using WITH clause to materialize the result for the inner part of the nested query and use the materialized result afterwards.

    For example, for the given query described in pre-requisites, re-write the query as below:

    WITH
    T1 AS ( SELECT A, SUM(C) AS C, GROUPING(A) AS GA, GROUPING(1) AS G1 FROM T GROUP BY GROUPING SETS (A,1) ),
    T2 AS ( SELECT A, B, SUM(C) AS C, GROUPING(B) AS GB, GROUPING(2) AS G2 FROM T GROUP BY GROUPING SETS (A,B,2) )
    SELECT T1.A, T1.C, GA, GB, G1, G2 FROM T1, T2;

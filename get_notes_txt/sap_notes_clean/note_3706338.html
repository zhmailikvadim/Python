Symptom

It is not possible to create a User Defined Field (UDF) linked to the Payment Terms system object using Data Interface (DI API).

Image/data in this SAP Note is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.

Reproducing the issue

Scenario:

Use the following DI API code to create a UDF which is linked to the Payment Terms system object:

SAPbobsCOM.UserFieldsMD oUserFieldsMD = (SAPbobsCOM.UserFieldsMD)oCompany.GetBusinessObject(BoObjectTypes.oUserFields);
oUserFieldsMD.TableName = "OITM";
oUserFieldsMD.Name = "TEST";
oUserFieldsMD.Description = "TEST";
oUserFieldsMD.Type = BoFieldTypes.db_Numeric;
oUserFieldsMD.EditSize = 6;
oUserFieldsMD.LinkedSystemObject = SAPbobsCOM.UDFLinkedSystemObjectTypesEnum.ulPaymentTerms;
int ADDUDF = oUserFieldsMD.Add();

Expected behavior:

The UDF is created successfully.

Actual behavior:

The UDF is not created because the member ulPaymentTerms for UDFLinkedSystemObjectTypesEnum is not available in DI API.

Solution

SAP intends to provide a feature package or patch to solve the symptom that is described in this SAP Note. The section References below lists the related feature package or patch once they become available. Be aware that References can only be confirmed at feature package or patch release date. SAP delivers feature packages or patches only for selected releases at its own discretion, based on the business impact and the complexity of implementation.

Workaround: 

Create the UDF using SAP Business One client.

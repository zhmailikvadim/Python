Symptom

In the process "New Installation for Feed-In (SUP View)" (DE_B_MPES_NIN_SUP), step 1000-0080 encountered an error during MOS determination, even though MOS data was sent by the DSO.

This issue is relevant for the market role: SUP.

Other Terms

MPES, New Installation, DE_B_MPES_NIN_SUP, /US4G/RP_TRIGGER_GPKE_NIN, MOS determination, table /US4G/DEF_SP, BADI li_badi_susc_process, get_supply_scenario_partners, IS_UBD_POD_DATA

Reason and Prerequisites

The error occurred because table /US4G/DEF_SP was not maintained when no explicit MOS was provided, or BADI li_badi_susc_process -> get_supply_scenario_partners was not enhanced to retrieve the MOS data sent by the DSO.

Solution

Implement the corrections of this SAP Note or import the relevant support package.

After implementing this SAP Note, you can implement your own solution by using the optional importing parameter IS_UBD_POD_DATA. You need to enhance the BADI to determine the MOS sent by the DSO. You can verify the fix by triggering the process using the trigger report /US4G/RP_TRIGGER_GPKE_NIN and checking that step 1000-0080 is processed successfully.

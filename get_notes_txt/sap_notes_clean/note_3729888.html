Symptom

Calling BRCONNECT using OS level command line without arguments failing with following errors:

BR0301E SQL error -1017 at location db_connect-3,SQL statement:
CONNECT system/*******'
ORA-01017: invalid username/password; logon denied
BR0310E Connect to database instance <SID> failed

Environment

  • SAP Systems based on SAP NetWeaver  
  • Oracle Database 

Reproducing the Issue

  1. Open the OS level command line.  
  2. Call BRCONNECT without providing any arguments using the command: brconnect.

Cause

The password of the SYSTEM user was changed and is different from the default password. 

Note:
Without specifying the user (-u switch), by default the BR*tools are connected to database with the SYSTEM user and its default password.

Resolution

Call BRCONNECT by explicitly specifying the user name and password using the option "-u <username>/<password>".
For more information, see point 7 of SAP Note 562863 - FAQ: Logon mechanisms.

Note: 
Basically there are four option to run BR*Tools (including BRCONNECT):
1. without -u (connect as SYSTEM and its default password)
2. -u / (OPS$ user will be used - old OPS$ connect method)
3. -u <user>/<password> (explicitly specifying the user name and password)
4. -u// using the data that stored in the secure storage (Secure Storage on File System - SSFS) 

See Also

SAP Note 562863 - FAQ: Logon mechanisms

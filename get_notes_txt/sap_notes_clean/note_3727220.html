Symptom

  • SFTPAdapterDatastore remote source fails with:
    • System call <LoadLibrary> to load and initialize DLL or shared library functions failed for <libssh2.so>.
    • Please make sure the library is installed and located correctly.
    • This message contains some internal system details which have been hidden for security.
    • If you need to see the full contents of the original message, ask your administrator to assign additional privileges to your account.
    • DATAFLOW: Data flow <DF_341a6_1> is terminated due to error <50012>
  • DPAgent needs libssh2.so library but OS environment is Redhat 9 and this is not available as standard. 

Environment

SAP HANA Smart Data Integration (SDI) 2.8.2.1

Cause

  • Smart Data Integration (SDI) 2.8.2.1 installation needs libssh2.so file on the OS
  • If libssh2.so is not on the OS the installation will be incomplete and cause issues

Resolution

  • Workaround:
    • Soft link libssh2.so in /usr/sap/<SID>/dataprovagent/bin/libssh2.so
      or
    • Install EPEL package on RedHat9 and install Smart Data Integration (SDI)

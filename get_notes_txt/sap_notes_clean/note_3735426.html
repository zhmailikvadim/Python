Symptom

When executing the termination of a contract, you receive the following error: 'Posting lock &1 already exists'.

Other Terms

Message Class BCA_CN_LOCK_PAYMT, Message Number 032, Function Module BCLP_BACN_PAI_BCLP01, Function Module BCLP_BACN_EVENT_DINP2

Reason and Prerequisites

This error occurs because the global variable 'g_tab_cn_lock_paymt_obj' in function module 'BCLP_BACN_PAI_BCLP01' contains multiple identical posting lock ID entries.

Solution

Import the relevant Support Package that is specified for your release. As a manual advance correction, you can implement the attached correction instructions using the transaction SNOTE. There are no manual activities.

Symptom

Integration Center job failed, and all of this page records are rollback, error message:

Exception caught configuring manager permisions during import: DAOException get sysconfig type: MATRIX_REPORT_PERMISSION_MANAGER_SM_AUTO_GRANT_TEAM_TYPE Key: MATRIX_REPORT_PERMISSION_MANAGER_SM_AUTO_GRANT_TEAM_KEY.

User: UPSERT failed for the row with key: . Reason: Exception caught when loading user XXX.. The row will not be processed for subsequent entities

User: Batch upsert failed as no record was successfully upserted.

Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.

Environment

  • SAP SuccessFactors HXM Suite
    • Integration Center

Reproducing the Issue

  1. Inbound Integration Center Job to upsert User entity.
  2. Job failed and event log has below message:

    User: Batch upsert failed: Exception caught when processing UserImport: Transaction already mark for rollback.
    Exception caught configuring manager permisions during import: DAOException get sysconfig type: MATRIX_REPORT_PERMISSION_MANAGER_SM_AUTO_GRANT_TEAM_TYPE Key: MATRIX_REPORT_PERMISSION_MANAGER_SM_AUTO_GRANT_TEAM_KEY.
    . The batch will not be processed for subsequent entities

    User: UPSERT failed for the row with key: . Reason: Exception caught when loading user XXX.. The row will not be processed for subsequent entities

    User: Batch upsert failed as no record was successfully upserted.

Cause

personIdExternal is already existed, since we cannot use it to create other User entity record.

Resolution

1. Use the same "User ID", "Status", "Last Name", "First Name" and "Manager" information, and upload these details to the SF system through the UI import tool.
--> We can determine which user has a data issue., e.g.
The import file has the following error(s):
User: [0001]This UserId [0001] is duplicated with existing person_id_external
User: [0002]This UserId [0002] is duplicated with existing person_id_external
The above error message means personIdExternal 0001 is already used and the userId is not 0001, we can check it under EmpEmployment entity.

2. And then remove these user records from Integration Center source, the related Integration Center Job will run successfully.

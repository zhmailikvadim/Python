Symptom

Error: org.apache.cxf.binding.soap.SoapFault:java.lang.Exception:java.lang.Exception: Destination URL is not configured for  XX_XXX system with message type XXXXXXXXXXX_XX@line 20 in DetermineSender.groovy

Cause

The issue occurs due to an incorrectly configured endpoint for the inbound interface in CPI, which leads to processing failures.

Resolution

  1. Sign into your CPI account
  2. Click Design >Select the Integrations and APIs > Select "SAP S/4HANA Integration with SAP Business Network for Logistics"
  3. Click Artifacts >Select SystemName_MessageType to Destination.
  4. Click Edit >Maintain Value Mappings > ValueMap (Target System, Message Type, RecipientBusinessSystemID: DEFAULT >Destination Url = URL.
  5. Click Save >Deploy

You must set up the endpoints for the inbound interfaces in CPI as follows:

    • TransportationOrderConfirmation_In
    • TransportationOrderQuotationConfirmation_In
    • TransportationOrderChargeElementRequest_In
    • InvoiceRequest_In
    • TransportationEventBulkNotification_In
    • TransportationOrderChargeElementRequest_In
    • AppointmentConfirmation_In
    • TransportationOrderBookingConfirmation_In
    • TransportationOrderGenericRequest_In

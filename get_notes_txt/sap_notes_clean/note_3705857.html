Symptom

User fails to restore BPC Environment in background mode by UJBR with error message "Failed to upload file from source".

Reason and Prerequisites

To guard against directory traversal threats, BPC will validate the file pathname using the logical path ARCHIVLINK_FILE_READ. If no physical path is assigned to this logical path, the server will not permit file reading.

Solution

  1.  Go to TCODE FILE and select the logical path ARCHIVLINK_FILE_READ
  2.  Double click "Assignment of Physical Path to Logical Path
  3.  Set syntax group and physical path for the BPC backup file. For example, "UNIX" and "/tmp/<FILENAME>". See detailed document by pressing F1 in that page.

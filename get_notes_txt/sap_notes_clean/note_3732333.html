Symptom

  • A freight condition is added at the purchase order (PO) header level with an absolute amount.
  • The header freight value is not distributed equally across all items; instead, it is distributed based on each item’s value.
  • Guidance is required on whether a standard setting exists to distribute the header freight equally across items.

Environment

SAP S/4HANA Cloud Public Edition

Reproducing the Issue

  1. Add an absolute freight condition at the PO header with a fixed amount (for example, 100).
  2. Create a purchase order with multiple line items.
  3. Check how the header freight amount is distributed among the items.

Cause

An absolute header condition whose condition type is flagged as a group condition is distributed proportionally to item value; if not flagged as a group condition, the amount is duplicated for each item.

Resolution

  1. Review standard behavior for header conditions:
  • Percentage header conditions apply the same percentage to all items.
  • Absolute header conditions distribute either proportionally (when the condition type is a group condition) or duplicate the amount per item (when it is not a group condition).
  1. To control the distribution of absolute header conditions, maintain the condition type in SSCUI 101094 (group price field):
  • Set Group Condition = true to distribute proportionally by item value.
  • Set Group Condition = blank o to duplicate the header amount for each item.
  1. Note that equal split by item count (for example, evenly dividing a header amount across items regardless of value) is not provided as a standard distribution option.
  2. Save the customizing and test with a PO containing multiple items to confirm the expected distribution.

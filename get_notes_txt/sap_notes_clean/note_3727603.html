Symptom

  • Customer uses a custom field to synchronize contact data from SAP C4C.
  • The field mapping appears correct and displays values in the desired order, but the actual data shows the field values are mixed up.

Environment

SAP Emarsys Customer Engagement

Reproducing the Issue

  1. Navigate to the Management > Field Editor screen.
  2. Locate the custom field and press Edit.
  3. Hover over the arrows next to the values to verify the Choice ID and Sort ID. For example, hovering over the arrow next to a value shows the text at the bottom of the screen: javascript:this.disabled=true;Up(3,1), where "Choice ID = 3" and "Sort ID = 1".
  4. Observe the discrepancy between the expected mapping and the actual mapping of the values.

Cause

At the field mapping the actual mapping is based on the Choice ID, while Sort ID only controls the display order of the fields.

Resolution

1. Add values to the field mapping. Choice IDs are assigned automatically in the order the values are entered.  
2. If you reorder attributes using the arrow icons, the Sort ID updates accordingly, but the Choice IDs remain unchanged and cannot be edited.  
3. To modify Choice ID, delete and re-add values in the Field Editor to ensure the IDs are assigned in the correct order.

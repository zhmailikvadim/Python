Symptom

  • The delivery cost calculation in purchase orders is based on quantity rather than net price.

Environment

  • sAP ERP Central Component
  • SAP ERP
  • SAP R/3
  • SAP R/3 Enterprise
  • SAP enhancement package for SAP CRM, version for SAP HANA
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA
  • SAP S/4HANA Cloud Private Edition

Cause

The condition type is likely configured with a quantity-dependent calculation rule (e.g., 'C' or 'D' in M/06) or is not correctly set to base calculations on net value

Resolution

  1. Check Calculation Rule (T-Code M/06): Ensure the freight condition type is not set to a "per unit" or "quantity-dependent" rule if a percentage of the net value is required.
  2. Configure Pricing Procedure (T-Code M/08): In the calculation schema, ensure the 'From' and 'To' steps correctly reference the base net price for the percentage calculation, rather than the quantity.
  3. Use Fixed Amount (Header Level): If the delivery cost is a flat fee, assign the condition type at the header level to make it independent of the item quantity.
  4. Verify Condition Type Settings: Ensure that the condition type is marked as "Delivery Costs" so it is added to the material valuation (not just a separate expense).

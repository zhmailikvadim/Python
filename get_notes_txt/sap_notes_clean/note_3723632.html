Symptom

When the Latest People Profile feature is activated, all previously uploaded documents disappear. If users attempt to upload a new document, the first upload appears successfully; however, when they try to upload additional documents, only the most recently uploaded file remains visible.

Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.  

Environment

SAP SuccessFactors HCM Core 2511

Reproducing the Issue

  1. Log in to the system
  2. Go to 'People Profile' page, open MDF section
  3. In the left side of the page search for 'Employee Document'
  4. Try to add attachments and click 'Save'
  5. Under 'Employee Documents' the added file will be visible
  6. Now click on 'Edit' and try to add another file and click 'Save'
  7. The system will replace the first document and only the last added file will be visible 
  8. If you click on 'View all' only the last document will be there

Cause

It’s a known UI template generation defect: When a child association has the same technical name as a field (for example, both cust_CSform_child). Because of this name conflict, the auto‑generated UI renders the attachment area as a single‑value control rather than a multi‑attachment list, so only the most recent file is visible.Please refer to below screen shot.

Resolution

The solution is scheduled to be implemented in the upcoming 2605 release.

Date of the release period: 

  • Preview: 2026 April-13
  • Production: 2026 May 15 - 16

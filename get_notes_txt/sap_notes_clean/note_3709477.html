Symptom

How to assign rights to non- admin user to access security editor

Without proper rights user will receive -error "You do not have the right to open the Security Editor for repository [localhost:6400 (user - Enterprise)]". 

Environment

SAP BusinessObjects Business Intelligence Platform 4.X

Reproducing the Issue

  1. Open a session in IDT with affected user
  2. Click on Windows on top
  3. Select security Editor
  4. Receive error

Cause

User do not have proper rights

Resolution

Assign Rights at Application Level:

Navigate to Application in CMC home

  1. Right click on Information Design Tool
  2. Select user "Add Principals"
  3. Add and Assign security
  4. Assign view rights
  5. Go to advanced
  6. Add/Remove Rights > Grant access to " Edit this object"
  7. Under Application section> Information design Tool> Grant "Administrator security profiles" access > Apply > Ok


For Connection to be visible:

Go to CMC > Connection > Manage > Select Top-level Security > All Connections > Add Principals > Add and Assign security > Assign view right > Apply > OK

For universes to be visible:

Go to CMC > Universes > Manage > Select Top level security > All universes > Add Principals > Add and Assign security > Assign view right > Apply > OK

To Insert and change data security profile under universe (security editor) grant below rights:

In > Advanced > Add/Remove rights

Go to System> Universe (Information Design Tool) > Grant All rights under "Specific rights for universe"

Symptom

When trying to maintain new aging information or aging processes on specific product, the validation to consider that the "Validity Start Date" is not in blocked period does not consider the fiscal year variant settings, conducing to a wrong determination of the fiscal period start date (for current or previous fiscal period) and the end user cannot add new aging information or aging processes although the provided "Validity Start Date" is within the open fiscal period.

Other Terms

Manage Aging Information, Fiscal Year Variant, /DRY/I_PlantOpenPeriods

Reason and Prerequisites

Program error

Solution

Version 1 of February 20, 2023: New creation

Install the code correction.

Description of Correction:

When trying to maintain new aging information or aging processes on specific product, the validation to consider that the "Validity Start Date" is not in blocked period considers the fiscal year variant settings and the end user can add new aging information or aging processes without any issues.

Functional Dependencies:

There are no technical dependencies.

Manual Changes:

No manual work is required.

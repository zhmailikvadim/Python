Symptom

User-Defined Fields (UDF) changes (column movement) made in the Form Settings is not saved in Recommendation Report of Payment Wizard.

Reproducing the issue

Scenario:

  1. Under Tools → Customization Tools → User-Defined Fields - Management → Payments create a UDF for the category Payment Wizard - Recommendation Report.
  2. Go to Banking → Payment Wizard and navigate to step 6 Recommendation Report.
  3. Open the Form Settings window, drag and move the UDF created in first step into a new position.
  4. Execute the Payment Wizard run.
  5. Start a new Payment Wizard run and navigate to step 6 Recommendation Report.

Expected behavior:

UDF is displayed at newly set position.

Actual behavior:

UDF is displayed at the end of the Recommendation Report.

Solution

SAP intends to provide a feature package or patch to solve the symptom that is described in this SAP Note. The section References below lists the related feature package or patch once they become available. Be aware that References can only be confirmed at feature package or patch release date.

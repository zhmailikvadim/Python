Symptom

Workflow notifications and purchase order workflow items appear in the launchpad/work zone, but the My Inbox app displays no items.

Environment

  • Fiori UI for My Inbox Application
  • BTP Workzone

Reproducing the Issue

  1. Open the launchpad or work zone and start the My Inbox app.
  2. Observe that the task list is empty although workflow notifications are received.

Cause

The runtime destination used by My Inbox for the TASKPROCESSING OData service is configured with Basic authentication and fixed backend credentials, so tasks are retrieved for that destination user instead of the signed-in user; if that backend user has no tasks, the result is an empty list.

Resolution

  1. Identify the runtime destination used by My Inbox (the destination that points to the IWPGW/TASKPROCESSING OData service) in your launchpad/work zone configuration.
  2. Review the destination’s Authentication setting:
  3. If it is set to Basic and contains hardcoded user credentials, note that My Inbox will show tasks only for that backend user.
  4.  Reconfigure the destination for principal propagation (single sign-on) so each user’s identity is forwarded to the backend and their own tasks are retrieved.  Users will be prompted for credentials
  5. After adjusting the destination, reload My Inbox and verify that the IWPGW/TASKPROCESSING request returns items for the current user.
  6. If any assistance is needed with the setup of principal propagation, the relevant component is BC-MID-SCC

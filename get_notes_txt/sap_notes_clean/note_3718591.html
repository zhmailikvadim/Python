Symptom

After configuring the Principal Propagation to Cloud for Customer, a connection failed message is displayed.

Environment

  • C4C
  • CX AI Toolkit

Cause

One possible cause is that the entity ID of the IAS metadata contains "https://" in front of it, while the base URL of the tenant does not. 

Resolution

The OAuth 2.0 Identity Provider Issuing Entity Name configured on C4C must follow the Entity ID found in the IAS metadata. This metadata can be found by accessing the following URL, substituting the base URL for the IAS tenant: https://XXXXXXXX.accounts.ondemand.com/saml2/metadata

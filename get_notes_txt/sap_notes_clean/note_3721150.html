Symptom

  • In the Fiori App W0139 (Customer Returns - Flexible Analysis), when a reason for rejection is set to a particular item of a customer return, the item gets disappeared from result list.
  • Checking database table VBAP, notice that when VBAP-KOWRR of a return order has value X, the item is not returned from Fiori App F0139.

*Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.

Environment

  • Sales and Distribution (SD)
  • SAP S/4HANA

Reproducing the Issue

  1. Open the Fiori App W0139 (Customer Returns - Flexible Analysis).
  2. Make fields 'Customer Return' and 'Customer Return Item' as Rows axis.
  3. Make a search, observe that expected customer return item is not appearing in the result. 

Cause

  • For this analytical apps, the item can be found only if the Statistical Value Control (VBAP-KOWRR) is empty.
  • VBAP-KOWRR might be set a value from two possible ways:
    • Definition of reason for rejection in T-Code OVAG has a setting 'Statistics':
    • In Item category definition T-Code VOV7, there is a setting 'Statistical Value':

Resolution

It is standard that the item won't be returned in this Fiori App.

If it is expected to see the item in this Fiori App, consider removing the value from VBAP-KOWRR for the document:

  • If VBAP-KOWRR is set because of reason for rejection, use another reason for rejection which has no Statistics value in OVAG.
  • If VBAP-KOWRR is set by the setting from item category, use another item category that has setting 'Statistical Value' unchecked.

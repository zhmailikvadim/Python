Symptom

  • When running partition-related functions in the class CL_MFLE_AMDP, the process fails on SAP HANA Cloud with SQL errors about missing objects.
  • Functions that count entries or read partitions may short dump with CX_SQL_EXCEPTION due to references to a removed monitoring view or column.
  • The issue occurs when determining table partitions (typically during MFLE conversion).

Other Terms

CL_MFLE_AMDP; COUNT_ENTRIES; GET_PARTITIONS; M_DATABASE; SYS.M_CS_PARTITIONS; SYS.TABLES (PARTITION_SPEC); SYS.TABLE_PARTITIONS; PARTITIONED_TABLES; CX_SQL_EXCEPTION; SNOTE

Reason and Prerequisites

The reason is a program error. The code still uses deprecated HANA system artifacts—monitoring view M_CS_PARTITIONS and column PARTITION_SPEC in SYS.TABLES—which are not available on HANA Cloud.

Solution

Apply attached Correction Instruction.

The correction adapts CL_MFLE_AMDP to detect the database version and use supported objects on HANA Cloud (e.g., switch from M_CS_PARTITIONS to TABLE_PARTITIONS and from PARTITION_SPEC to PARTITION_DEFINITION where applicable).

No manual post‑implementation steps are required.

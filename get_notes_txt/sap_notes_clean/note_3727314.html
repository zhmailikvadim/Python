Symptom

During creation of an procurement alternative via transaction CK91N for a co-product, the following error is raised, CKBA067 Material XXXXXX in plant XXXX is not a process material for co-product XXXXXX'.

Disclaimer: "Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental."

Environment

  • Product Cost Planning
  • SAP ERP 
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA
  • SAP S/4HANA Finance

Reproducing the Issue

  1. Open transaction code CK91N.
  2. Input Material: XXXX and Plant: ZZZZ.
  3. Click on Create.
  4. Select Processing Type: Production.
  5. Enter Business Process: YYYY.
  6. Click Continue to encounter error message CKBA067. 

Cause

  • The co-product indicator for Material XXXX in Alternative BOM 6 (Costing BOM) was not set.
  • This omission caused the system to fail to recognize Material YYYY as the process material for co-product Material XXXX, resulting in the error message CKBA067.
  • Without the co-product flag being set in BOM 6, the validation logic in CK91N does not recognize the process material as the cost-driving parent of the co-product, resulting in CKBA067.

Resolution

To resolve, follow the steps below:  

  1. Open transaction CS02 and access Alternative BOM 6 for Material YYYY in Plant ZZZZ.
  2. Locate the line item for Material XXXX and enable the Co-Product indicator in the item details.
  3. Save the BOM.
  4. Open transaction CK91N and create the Apportionment Structure with Material YYYY as the process material and Material XXXX as the co-product.
  5. This should eliminate error CKBA067.
  6. Note: The changes are isolated to Alternative BOM 6 and will not impact the Production BOM (Alternative BOM 1) or any existing MRP and production order processes.

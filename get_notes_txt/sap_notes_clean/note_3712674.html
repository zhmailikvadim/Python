Symptom

  • CAP is running "locally" in BAS
  • When starting service via cds w, BAS automatically exposed a new hostname.
  • Via browser, the access works for example: https://port4004-workspaces-ws-c6v36.us10.trial.applicationstudio.cloud.sap/
    port4004-workspaces-ws-c6v36.us10.trial.applicationstudio.cloud.sap host is exposed by BAP during cds w.
  • curl --request GET --url "https://port4004-workspaces-ws-c6v36.us10.trial.applicationstudio.cloud.sap/odata/v4/employee/Employees"
    returns nothing
  • curl -v -i -L -X GET https://port4004-workspaces-ws-c6v36.us10.trial.applicationstudio.cloud.sap/odata/v4/employee/Employees
    shows error: 401 Unauthorized

Environment

  • SAP Business Application Studio (BAS)
  • BTP
  • CAP

Reproducing the Issue

Browser accesses work : https://port4004-workspaces-ws-c6v36.us10.trial.applicationstudio.cloud.sap/odata/v4/employee/Employees
In BAS curl nothing return or 401 Unauthorized occurs: 

curl --request GET --url "https://port4004-workspaces-ws-c6v36.us10.trial.applicationstudio.cloud.sap/odata/v4/employee/Employees"
=>returns nothing
or
curl -v -i -L -X GET https://port4004-workspaces-ws-c6v36.us10.trial.applicationstudio.cloud.sap/odata/v4/employee/Employees
=> 401 Unauthorized

Cause

BAS exposed new hostname during cds w to browser, and the new hostname is different from BAS's hostname.

Resolution

cds w log shows which host:port the cds is listening, for example:

[cds] - server listening on { url: 'http://localhost:4004' }
[cds] - server v9.7.0 launched in 318 ms
[cds] - [ terminate with ^C ]

So BAS curl, use local listener http://localhost:4004:

change <curl --request GET --url "https://port4004-workspaces-ws-c6v36.us10.trial.applicationstudio.cloud.sap/odata/v4/employee/Employees">
to <curl --request GET --url "http://localhost:4004/odata/v4/employee/Employees"

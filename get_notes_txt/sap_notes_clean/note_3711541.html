Symptom

During the fill process of pricing with CDOTE for single fill from vehicle contract, after running the transaction CMM_OF_PROCESS,
the CDOTE order status shows 'Filled' in the Fiori application 'Manage Commodity Derivative Orders' but shows the message type as 'E' from message class 'CMMFDOF_ORDER_FILL'.

Replication Steps -

- Create a vehicle contract with NFE fixation where document UoM and market UoMs are different and Save.
- Open the transaction CMM_MP_MASS_PRICING and perform the pricing with CDOTE for single fixation. Make sure that the quantity should be distributed in single lot.
- Run the CDOTE cycle to fill the single lot.
- After running the transaction CMM_OF_PROCESS, open the CDOTE Order in the Fiori application 'Manage Commodity Derivative Orders'.
- Message 'Fills have been received but could not be processed yet' displayed because of error 'Wrong Document Quantities from Upstream System'.

Other Terms

Agricultural Contract Management; ACM; CDOTE; Manage Commodity Derivative Orders; CMMFDOF_ORDER_FILL(257)

Reason and Prerequisites

Program Error

Solution

Import the relevant support pack or apply the correction instruction(s).

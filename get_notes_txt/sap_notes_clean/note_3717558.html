Symptom

Upon launching the SAP Convergent Charging Cockpit app, an Error pop up shows up:

Your browser can’t communicate with the Web server.
Please try again or contact the IT administrator.


Environment

SAP Convergent Charging 2023 FPS 3 (SAP CC 2023 FPS3) and later versions

Cause

This kind of error pop up typically occurs due to network latency causing session state verification timeouts.

Resolution

As of SAP CC 2023 FPS 3, new parameters have been added to the Cockpit "User Session Management and Security Settings":

  • http.session.state.poll
  • http.session.state.timeout

If the applications (tiles) are not working, verify that both the Dispatcher and Updater instances are running in your SAP Convergent Charging Core Server.

If the Dispatcher and Updater instances are running but a popup still appears, this is typically caused by network latency affecting session state verification timeouts. To resolve this issue, adjust the timeout configuration in the Cockpit configuration file:

# http.session.state.poll
#   Description:     Interval (in seconds) between HTTP session state checks
#   Default:         30
#   Valid Range:     10 to 90
#
# http.session.state.timeout
#   Description:     Maximum wait time (in seconds) for session state verification
#                    If exceeded, the user will be prompted to retry
#   Default:         3
#   Valid Range:     Greater than 0 and less than http.session.state.poll

#http.session.state.poll=30
#http.session.state.timeout=3

1. Locate and open the Cockpit configuration file
2. Find the http.session.state.timeout parameter
3. Uncomment the line by removing the # symbol
4. Increase the timeout value (e.g., from 3 to 5 or 10) based on your network latency
5. Save the file and restart the Cockpit service

Note: Ensure http.session.state.timeout remains less than http.session.state.poll.

Symptom

In the ABAP report RSTRFCEG, there is a parameter called CHECK_ONLY. The meaning and implications of this parameter may not be clear, leading to questions about its function and how it affects the execution of the report.

Environment

  • Operating System independent
  • Database independent
  • SAP NetWeaver
  • SAP Web Application Server for SAP S/4HANA
  • SAP ABAP Platform 

Resolution

The CHECK_ONLY parameter in the report RSTRFCEG in SAP is typically used to run the report in a test mode. When CHECK_ONLY is set to true, the report performs all the checks and validations as it normally would, but it doesn't make any changes to the database.

This feature is useful when testing the report or when wanting to see the potential impact of running the report without actually executing the changes. It provides a way to ensure the correctness of the report and the expected outcomes before performing actual operations that could modify the data.

To use the CHECK_ONLY parameter, it needs to be set in the selection screen of the report before executing it. If the parameter is set, the report will run in check only or test mode, and no changes will be made to the database.

Keywords

CHECK_ONLY parameter, RSTRFCEG

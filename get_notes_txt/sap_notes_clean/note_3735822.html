Symptom

  • A vendor credit was posted in FI in period 12 (December) and now only impacts period 12 in CO-PA (Margin Analysis).
  • There is a requirement to reflect the impact of this credit across all 12 periods of the fiscal year in CO-PA only, without changing FI postings or CO ledger data.
  • Top-down allocation was created via “Manage Allocations” and executed via “Run Allocations”, but the allocation applied only to December and not to earlier periods.

Reproducing the Issue

  1. Post a vendor credit note in FI in period 12 (December).
  2. Create a top-down allocation in CO-PA (for example, via t-code KE28 or the Fiori apps “Manage Allocations” and “Run Allocations”) with the intention to spread the credit across multiple periods of the fiscal year.
  3. Start the allocation and observe that CO-PA postings are created only for period 12; prior periods remain unchanged.

Cause

In standard, back-posting to previous periods from CO-PA top-down distribution/universal allocation is not supported. Top-down distribution (KE28) is a period-end function that posts per period, and cumulative (YTD) sender posting is not available.

Resolution

  1. Use top-down distribution to allocate within CO-PA (transaction code KE28).
  2. Run KE28 (Top-Down Distribution of Actual Data) to distribute values using reference data; this is a CO-PA-only process and does not call FI or CO postings.
  3. Execute KE28 in period 1, then reverse it into period 2.
  4. Then execute the same KE28 again in period 2, this way, you've debited the values ​​from period 1 and posted them to period 2.

Symptom

The synchronization was working before, but now it fails on the step 'Sync Initialization' > 'Create Signavio and SolMan Object Mapping'. There is no error message shown in the UI and no error message in the synchronization log Excel file in the sync history.

Cause

The public IP addresses for the Inbound gateway for BPMC may have been rotated by the hyperscalar provider. They may now differ from the IP address allow list implemented against your Cloud Connector.  When this is true, then the cloud connector will not be able to open a secure tunnel for BPMC. This means, BPMC will not reach the Solution Manager system.

Resolution

Please check with your network team, if the Cloud Connector is allowed to access certain list of IP-Addresses only. When that is the case, please compare your setup with the current IP address list: 

Europe Frankfurt (EU10)

  • 3.77.249.228
  • 3.124.61.153
  • 18.158.13.88 

US Virginia East (US10)

  • 23.20.251.219
  • 34.196.82.94
  • 50.16.9.86 

For more information please see the chapter Advanced IP-Based Security in our documentation.

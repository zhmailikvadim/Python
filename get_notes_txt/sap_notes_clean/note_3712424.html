Symptom

  • Health Monitoring for HTTP Availability Monitoring URLs does not return ratings for some URLs performing a redirection.

Environment

  • Focused Run (any version)

Reproducing the Issue

  1. Activate in Health Monitoring "HTTP Availability Monitoring" for some URLs
  2. Attempt to retrieve ratings or perform a login for the URLs in point 1
  3. Observe that no ratings are returned.

Cause

HTTP availability monitoring does not support login for URLs with redirection to authentication or login pages (e.g. IAS).

Resolution

  1. Does Availability Monitoring support HTTP availability monitoring for URLs that require a login?
    • YES. It is supported but only if the call of the URL is NOT re-directed (e. g. redirected to an authentication server).
  2. What is needed for a successful HTTP availability setup when there’s a redirection to a login page (IAS) in the middle? Is this supported?
    • As mentioned in 1, in the case the URL call is re-directed (in this case to a login page of an IAS, but it doesn't matter the kind of re-direction), the logon check of an Availability Metric won’t work.

So in order to ensure HTTP Availability Monitoring works, please:

  1. Verify if the URL maintained does not redirect to another page (e.g., authentication server or login page).
  2. Ensure that the HTTP availability monitoring setup is configured for URLs without redirection.

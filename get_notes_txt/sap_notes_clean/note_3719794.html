Symptom

When attempting to perform a "Full" data extraction by resetting the Last Run Date of a download connector via the REST API, the request fails. If you attempt to set a date older than one year from the current date, the system returns an error message:

Error:
Cannot set a last run date which is earlier than 12 months, Please provide a valid date

Cause

By system design, SAP Fieldglass restricts the ability to programmatically reset connector timestamps beyond a 12-month window to ensure system performance and prevent excessive data loads. While administrators can reset the date further back manually in the User Interface (UI), the REST API enforcement prevents automated tools from triggering full historical syncs older than one year.

Resolution

To manage your download synchronization, you must use the correct API method and adhere to the 12-month limitation

  1. Use the Correct API Endpoint.
    • To update the last run date, you must use a POST request to the connectorupdate resource rather than a GET request to the connector itself.
    • Correct Syntax: POST https://{base_url}/api/vc/connectorupdate/{Connector Name}/lastRunDate/{YYYY-MM-DD HH:mm:ss}
  2. Adhere to the 12-Month Limit
    1. Ensure the date provided in your API call is within the last 365 days. If you require a full historical download of all records (including those not modified in over a year):
    2. Manual Reset: A Configuration Manager must manually reset the Last Run Date to a historical date (e.g., 2010-01-01) within the Connector Catalog UI.
    3. Support Intervention: If the UI reset does not meet your needs, contact SAP Fieldglass Support to request a backend update to the last run timestamp for that specific connector.

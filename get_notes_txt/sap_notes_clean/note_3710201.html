Symptom

After upgrading to SAP S/4HANA 2023, customers integrating employee data from SuccessFactors into S/4HANA may face an issue where, in cases of global employment (employees with multiple assignments), the synchronization does not create a separate Business Partner (BP) with vendor role for each employment. Instead, only one BP vendor is created, and multiple personnel numbers (PERNRs) may appear under the Identifiers section of the existing BP.
As a result:

  • The new BP for the second or subsequent employments is not generated.
  • The original BP vendor segment contains data for multiple employments and countries, leading to inconsistencies (e.g., only one set of bank details available, which may cause payment issues with downstream systems like Concur).
  • Expense processing and bank details for additional employments cannot be handled as expected.

Environment

  • SAP S/4HANA on-premise

Reproducing the Issue

  1. Replicate an employee with multiple employments (e.g., for global assignments) from SuccessFactors to S/4HANA using the integration scenario (Mini Master, CPI).
  2. Run the synchronization from employee master data to Business Partner using the report /SHCM/RH_SYNC_BUPA_EMPL_SINGLE or equivalent batch processes.
  3. Observe that only one BP with vendor role is created for the original employment. Additional employment(s) do not get their own BPs, and PERNRs are listed only as identifiers in the existing BP.
  4. Downstream processes (such as Concur expense processing) are affected due to missing BP for the second employment and/or incorrect or missing bank data.

Cause

This situation typically occurs if the vendor segment for a second or subsequent employment was incorrectly merged or manually added to the original BP, or if expected system logic did not create a new BP vendor for each new employment (PERNR). The system expects one BP vendor per employment. When multiple vendor segments are assigned to a single BP, inconsistencies arise, especially after upgrades or manual corrections.

Resolution

  1. Remove incorrect vendor segments:

    • Identify the BP that has vendor segments for multiple employments in transaction BP.
    • Remove any vendor segment(s) corresponding to the personnel numbers that should have their own BP.

  2. Run BUPA synchronization for each employment (PERNR):

    • For each employment (PERNR) that does not have an associated BP vendor, execute the report /SHCM/RH_SYNC_BUPA_EMPL_SINGLE for that PERNR.
    • The report should create a new BP vendor for the specified PERNR. Bank account details and the relevant country-specific data will be assigned to the new BP vendor.

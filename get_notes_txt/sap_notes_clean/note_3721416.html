Symptom

Payment term from the original invoice is not copied to the residual payment document when posted from transactions like F-44, F-32, F-53 or F-28.

Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.

Environment

  • Financial Accounting
  • SAP ERP Central Component
  • SAP ERP
  • SAP R/3 Enterprise
  • SAP Enhancement package for SAP ERP
  • SAP Enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4 HANA
  • SAP S/4 HANA Cloud Private Edition

Reproducing the Issue

1. Open transaction F-44. 
2. Select relevant open items. 
3. Perform residual item clearing.
4. Simulate or post the document. 
5. Open the line item using FB03, FBL1N etc.. 
6. Observe that the payment term is not filled in the vendor/customer line item of the residual payment document.

Cause

The issue occurs due to incorrect configuration. The “Payment Term from Invoice” checkbox is not selected in Customer/Vendor Tolerances (OBA3), which prevents the payment term from being copied into the residual payment document.

Resolution

  1. Go to transaction OBA3 (customer/vendor Tolerances) and select the respective company code.
  2. Select the check box "Payment Term from invoice".
  3. Save the changes.

Once this setting is enabled, the residual payment document will inherit the payment term, baseline date, and net due date from the original invoice.

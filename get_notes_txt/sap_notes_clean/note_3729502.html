Symptom

When trying to access the condition records for MM and SD conditions types in VK12 or VK13, a pop up screen with the following error appears:

System error: Field KOMG-TAX_SITUATION not defined in data dictionary

Environment

  • SAP ERP
  • SAP S/4HANA

Reproducing the Issue

There are different approaches for reproducing this issue:

  • For instance, going to V/07 for BCBS access sequence and trying to see the condition records.
  • Or, going to VK12 to change the condition types like CBS3, IB3S and IB3M with an option involving Tax Situation.

Cause

Missing proper customizing in view V_T681F.

Resolution

1. For ERP Releases:
It is necessary to access the view V_T681F using transaction code SM30.
Once there, filter by the relevant criteria for "Usage" (such as "A" for pricing) and "Application" (such as "TX") and add the field TAX_SITUATION to the field catalog for Pricing Taxes. 

2. For S/4HANA Releases:
It is necessary to access the view V_T681F using transaction code SM30.
Once there, filter by the relevant criteria for "Usage" (such as "A" for pricing) and "Application" (such as "TX") and add the field J_1B_TAX_SITUATION to the field catalog for Pricing Taxes.

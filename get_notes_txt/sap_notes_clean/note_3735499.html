Symptom

Approval details tab is missing for previously created purchase documents after switching from flexible workflow to classic release strategy - sap s/4hana

Environment

  • SAP Fiori
  • SAP S/4HANA all versions
  • SAP S/4HANA Cloud Private Edition

Reproducing the Issue

  1. Deactivate Flexible Workflow for document type and activate the Classic Release Strategy.
  2. Open an existing purchase document that was created under Flexible Workflow.
  3. Observe that the Approval Details tab is no longer visible for that document.

Cause

Switching from Flexible Workflow to a release strategy, the document will no longer be subject to Flexible Workflow and as a result, the associated tab will no longer be visible.

This behavior aligns with the standard functionality of the system.

Resolution

Avoid switching workflows between Flexible Workflow and Classic Release Strategy during document processing.

The workflow log and history for the document are not deleted. They remain stored within the workflow tables.  Details such as the workflow process and approval history can be reviewed. (For relevant tables, refer KBA 3191211 - Flexible Workflow: reports and tables - SAP S/4HANA. )

Symptom

If Pack/Unpack activity is used with activity rules AUTO_START_SFC and AUTO_COMPLETE_SFC set to YES, done quantity in shop order maintenance build status tab is incorrect after container is done and closed.

Other Terms

PK020, incorrect quantity

Reason and Prerequisites

The issue in the symptom is caused by incorrect code in Complete functionality that updates shop order done quantity (QTY).
To reproduce the issue you should do this:

  1. create an operation in operation maintenance
  2. create a standard routing with this single operation
  3. create a material with lot size 1 and assign the routing to it
  4. in activity maintenance, modify PK020 to have activity rules: AUTO_START_SFC – YES AUTO_COMPLETE_SFC – YES
  5. create and release 12 SFCs (all with QTY 1) to the material. SFCs belong to the same shop order
  6. create a container in container maintenance with Max QTY 12
  7. with PK020 activity pack the recently released SFCs to the container
  8. click „Done and Close” to close the container
  9. in shop order maintenance, build status tab, see QTY 78 instead of QTY 12.

Solution

The code of Complete functionality that updates shop order done quantity was corrected to properly handle Pack/Unpack with SFC start/complete.

Impact

The code change done to address the symptom affecting functionality of Pack/Unpack and Complete functionalities.

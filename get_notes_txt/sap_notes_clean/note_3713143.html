Symptom

A subitem created by Alternative-Based Confirmation (ABC) with a sourcing profile for Third-Party Order Processing (TPOP) has an incorrect item category.

Other Terms

ATP; aATP; Third-Party Order Processing; TPOP; Alternative-Based Confirmation; ABC; Substitution Strategy; Remaining Open Quantity Assignment; VOV4; Item Category Assignment

Reason and Prerequisites

The symptom applies if the following conditions are met:

  • The substitution strategy or its follow-up substitution strategy has a sourcing profile maintained.
  • TPOP does not give a confirmation. This can occur, e.g., if an invalid source of supply was found.
  • The setting for remaining open quantity assignment in the app Configure Substitution Strategy prefers a scheduleline created for TPOP.

The reason for the symptom is that schedulelines created for TPOP having item usage ABC4 or ABC5 are filtered out and a scheduleline with ABC2 is used instead for the subitem creation. The customizing for item category assignment (transaction code VOV4) then chooses an incorrect item category.

The correction makes sure that also schedulelines with item usage ABC4 or ABC5 are considered for remaining open quantity assignment in ABC.

Solution

Apply the correction instructions for this note or upgrade to a SAP S/4HANA support package that contains this correction.

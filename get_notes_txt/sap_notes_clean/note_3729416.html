Symptom

When attempting to copy a space in SAP Datasphere, the following error occurs:

Space couldn’t be copied.
We can provide the following information to assist either you or an SAP engineer to solve the issue:

Correlation ID: abcdef1234ghijk5678
HTTP Status: 403 error
Error Code:
Technical Message:
Forbidden

Environment

SAP Datasphere

Cause

The error typically occurs when required prerequisites for copying a space are not fulfilled. The most common causes are:

  • Missing required global privileges
  • Presence of unsupported objects (e.g., Business Builder objects)
  • Space is linked to SAP Analytics Cloud with exposed objects
  • Restrictions based on space storage type

Resolution

1. Verify required privileges

To copy a space and its contents, the user must have a global role with the following privileges:

  • Data Warehouse General (-R------) – Access to SAP Datasphere
  • Spaces (C-------) – Create spaces
  • User (-R------) – Assign users to the new space
  • Spaces (-------M) – Update all spaces and properties
  • Space Files (-------M) – Access objects and data across spaces

The DW Administrator global role, for example, grants these privileges. For more information, see Privileges and Permissions and Standard Roles Delivered with SAP Datasphere. 

2. Check for unsupported objects

Space copy is not supported if the source space contains:

  • Business Builder objects

Remove or relocate such objects before attempting the copy.

3. Check SAP Analytics Cloud integration

If the space is used as storage by an associated SAP Analytics Cloud tenant, then it cannot be copied if any SAP Analytics Cloud objects are exposed (see Exposing Objects for Consumption in SAP Datasphere).

Remove exposed objects or disconnect usage before copying.

4. Validate space storage type

  • Relevant only for spaces with a storage type SAP HANA Database (Disk and In-Memory), and not for SAP HANA Data Lake Files spaces.

Symptom

Is it possible to approve pending nominees who belong to a users’ target population in Talent Pools?

**Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental**

Environment

SAP SuccessFactors Succession Management

Cause

Resolution

Required permission:

User Permissions >Succession Planners > View Talent Pool nominations > Approve/Reject Talent Pool Nominations permission.

Behavior:

If a user with nomination permission but no approval permission, makes a nomination it will go to ‘Pending’ status with no option to approve or reject:

A user with approval permission can then access the Talent Pool and approve or reject as needed:

If a user with both nomination and approval permission makes a nomination it goes automatically to ‘Approved’ status and the buttons will remained grayed out.

If a user has only "View Talent Pool Nominations" permission they will not see nominations which are in the status of "Pending":

Note: Talent pool owners with proper permissions can now see a new To-Do item Approve Talent Pool Nominees on the legacy home page or in the To-Do panel when there are pending nominees to be approved or rejected. This feature was built to notify talent pool owners with proper permissions to approve nominees timely.  At this time no email notifications are sent to the approvers.

Symptom

The SOAP webservice authentication with credentials (username/password) can use two different methods:

  • HTTP Authentication (using the parameters on headers)
  • Body Authentication (sending the fields userName, password, and orgName inside the body content).

Our guide documents both scenarios here.

After the 2504 release, the SOAP logon operation using only HTTP Authentication (headers, with the body content blank) stopped working.

Other Terms

SAP Cloud Integration for data services - 2504 server

Reason and Prerequisites

The issue occurred due to a code change.

Solution

The correction for this server issue is delivered in SAP Cloud Integration for data services version 2504 HF1 (1.0.11.56-HF1).

Workaround

Please continue using the HTTP header authentication as before, but instead of a blank body, pass the userName and password fields inside the body.

Find the sample below in curl format:

curl --location 'https://hcids.us3.hana.ondemand.com/DSoD/webservices?SOAPAction=logon' \
--header 'SOAPAction: function=logon' \
--header 'orgName: your_org_name' \
--header 'dsodEnv: SANDBOX' \
--header 'Content-Type: application/xml' \
--header 'Authorization: ••••••' \
--data '<Envelope xmlns="http://schemas.xmlsoap.org/soap/envelope/">
<Body>
<logonRequest xmlns="http://webservices.dsod.sap.com/">
<userName xmlns="">A</userName>
<password xmlns="">B</password>
</logonRequest>
</Body>
</Envelope>'

The values inside the fields userName and password can be random, meaning they do not need to be the correct credentials, like the sample above showing "A" and "B".

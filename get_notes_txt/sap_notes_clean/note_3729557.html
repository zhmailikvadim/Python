Symptom

When attempting to upload SAR files via "Load Packages from Frontend" in transaction SPAM/SAINT, it ends with error : No response while accessing SAPCAR (possible program termination).

Under dev_rfcXX log, the error appears as:

======> CPIC-CALL: 'ThSAPOCMINIT', communication rc: CM_PARAMETER_ERROR (cmRc=19), taskhandler rc: GW_SNC_START_EXT_DIS (thRc=730)
Not allowed to start non-SNC connections
ABAP Programm: SAPLCRFC (Transaction: SAINT)
User: <user> (Client: 000)

Error RFCIO_ERROR_SYSERROR in /bas/793_REL/src/krn/rfc/abrfcpic.c : 1815
CPIC-CALL: 'ThSAPOCMINIT', communication rc: CM_PARAMETER_ERROR (cmRc=19), taskhandler rc: GW_SNC_START_EXT_DIS (thRc=730)
Not allowed to start non-SNC connections
DEST =%_TCPIP_%1
HOST =localhost
PROG =/usr/sap/<SID>/<Instance>/exe/sapxpg

Environment

  • SPAM/SAINT
  • SAP NetWeaver AS ABAP

Reproducing the Issue

  1. Navigate to transaction SAINT.
  2. Attempt to upload SAR files via option "Load Packages from Frontend".
  3. Ends with error message: "No response while accessing SAPCAR (possible program termination)".

Cause

The issue is caused by the SNC parameter configuration. Specifically, the parameter 'snc/permit_insecure_start' is set to `0`, while the parameter 'snc/enable' is set to '1', which prevents the initiation of non-SNC connections.

Resolution

  1. Validate the SNC parameter settings.
  2. Update the parameter `snc/permit_insecure_start` to `1`.
  3. Restart the system to apply the changes.
  4. After the restart, attempt to upload the SAR files via transaction SAINT again.

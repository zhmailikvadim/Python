Symptom

As documented on SAP Help, Solr 9.5 will be valid until May 21, 2025. For more information check the Solr Version Validity Policy. After this date you will not be able to create builds with Solr 9.5; you must upgrade to Solr 9.7. You may have received a notification indicating the same.

Environment

Commerce Cloud in the Public Cloud

Solr 9.5

Resolution

You must upgrade to Solr 9.7 to be able to continue to create Commerce Cloud builds.

Frequently Asked Questions

Why can I no longer build with Solr 9.5
In line with SAP's security policies, older software versions are no longer supported after a period of time once newer versions become available.

How do I check which version for Solr is currently running?
Go to your endpoint for the Solr Admin. On the main page under "solr-spec" you can see the Solr version.

How will I be affected?
You will no longer be able to create builds containing Solr 9.5. Builds will fail with the following error if they are attempted:

Will my environment be upgraded automatically?
Environments with Commerce already deployed and running will not be upgraded automatically without your intervention. Upgrading to Solr 9.7 requires that you follow the Upgrading Solr Version steps, which requires creating a new build and deployment. If you do not explicitly set a Solr version, version 9.7 will be used by default for builds created after May 21st.

Will my environment stop working after May 21 if I don't upgrade?
Environments with Commerce already deployed and running with Solr 9.5 will continue to function past May 21. Only creating new builds will be blocked.

What steps do I need to perform to upgrade?
You can find the detailed upgrade steps here: https://help.sap.com/docs/SAP_COMMERCE_CLOUD_PUBLIC_CLOUD/aa417173fe4a4ba5a473c93eb730a417/b70eefc75c6a45b5aa00de55b8989ee1.html#procedure

Do I need to perform a deployment with downtime/migration to perform to upgrade?
You can choose the deployment mode which best suits your needs. You do not need to perform a deployment with any specific options such as with downtime or with migration.

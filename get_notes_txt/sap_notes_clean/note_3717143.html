Symptom

While performing offline data import using EH&S OCC, the following error occurs:

Number/date format is not supported:   / DD.MM.YYYY

However, the data format DD.MM.YYYY is a supported format.

Remark: the error message can also display other supported date formats as: MM/DD/YYYY, MM-DD-YYYY, YYYY.MM.DD, YYYY/MM/DD, and YYYY-MM-DD.

"Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental."

Environment

  • SAP ERP
  • SAP S/4HANA
  • SAP S/4HANA Cloud Private Edition

Reproducing the Issue

  1. Create a .dat file using EH&S OCC;
  2. Upload the file to application server using CG3Z;
  3. Run the Specification Import in CG33;
  4. The data is not imported;
  5. Checking the job log with name starting with "EH&S-IMP-SUB*" in SM37, the following error occurs:
    Number/date format is not supported:   / DD.MM.YYYY
  6. The date format in the file is a supported format.

Cause

  • The Decimal Separator +DN missing in the .dat file header (+DN);
  • The error message has the following structure:
    Number/date format is not supported: "+DN line from .dat file header" / "+DF line from .dat file header"
  • The displayed error message means that +DN line from .dat file header is empty, which is not supported.

Resolution

  • Correct the DN field in EH&S OCC, by selecting "Data Import > Settings > Offline Import" and selecting a supported value in the dropdown list for DN field:
  • Create the file again with the correct settings;
  • Repeat the import process.

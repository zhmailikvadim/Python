Symptom

Layout and reports in Reporting Service and Web Client are not applied with General Authorization.

Reproducing the issue

Prerequisites:

Log on to SAP Business One, set No Authorization in General Authorization for Sales - A/R under Report Layout API for a certain user.

Scenario:

Log on to Web Client with the above user, open a Sales Order detail and click Preview.

Expected behavior:

An error message is displayed as follows: You are not authorized to run this report; contact your administrator to update the settings in “Authorizations”.

Actual behavior:

You can preview the report.

Solution

As of SAP Business One 10.0 PL02, the authorization of layouts and reports can be set in General Authorization according to modules.

A user with Full Authorization of a corresponding module can access the Report and Layout in that module; otherwise, no access is available.

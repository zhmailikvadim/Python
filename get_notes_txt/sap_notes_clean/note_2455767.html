How to generate the "Support Logs" from the Diagnostics Agent OS Level:

   1. Open a Terminal in Linux or Command Prompt (CMD) in Windows

   2. Go to the directory "/usr/sap/DAA/SMDAXX/script" in the Diangostics Agent Server

   3. Run the Linux command : "./smdsetup.sh supportlogs" or the Windows command : "smdsetup supportlogs"

   

   4. The logs will be generated and will be found in the directory /usr/sap/DAA/SMDAXX/SMDAgent/supportLogs_<TIMESTAMP>.zip.

Please note the Introscope Agents Log (Not provided with the Diagnostics Agent Support Log) are on the following path:

  • From directory /usr/sap/SID>/<inst>/work of the Managed System if it's a Netweaver java node:
    • dev_server0 (contains the actually used JVM parameters)
    • std_server*.out

How to download the Diagnostics Agent Support Logs from the "Agent Administration" UI

Prerequisite :

  • Your Diagnostics Agent has to be green and correctly connected to SAP Solution Manager to download the logs from the Agent Administration UI
  • Make sure your SAP Solution Manager user has the correct permissions to access the Agent Administration UI
  1. Launch the SM_Workcenter transaction
  2. Click on the "SAP Solution Manager Adminstration" Tab
  3. Click on the "Agents (Administration)" tile
  4. Click on "Agent Admin" button then click on the "All Agents" button
  5. In the new screen that will be opened, click on the "Agent Log Viewer" tab

    6. Select the relevant Diagnostics Agent from the "Select an agent" list and click on "Download Logs":

     7. Click on "Yes, include support logs"

     8. A zip file will be generated including the logs.

     9. Click on "Click here to download the resource"

     10. Download the Diagnostics Agent Support logs to your computer

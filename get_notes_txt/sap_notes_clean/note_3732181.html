Symptom

  • Calling an integration flow (iFlow) HTTP endpoint returns HTTP Status 401 – Unauthorized.
  • OAuth response indicates error "invalid_token".
  • Error description states: "The token is invalid: Jwt token with audience [...] is not issued for these clientIds: [...]."

Environment

  • Cloud Integration
  • SAP Integration Suite

Reproducing the Issue

  1. Create or use an OAuth client from the Process Integration Runtime service with plan "api".
  2. Request an access token with grant type client_credentials.
  3. Call the iFlow HTTP endpoint with the obtained Bearer token.
  4. Observe the response HTTP 401 Unauthorized with "invalid_token" and a JWT audience/clientIds mismatch message.

Cause

  • OAuth credentials from the Process Integration Runtime service plan "api" (used for managing integration content) were used to call an iFlow runtime HTTP endpoint. Tokens from this plan are not issued for the runtime clientIds expected by the iFlow, resulting in an audience/clientIds mismatch and 401 Unauthorized.

Resolution

  1. In the SAP BTP subaccount, create a new Process Integration Runtime service instance with plan "integration-flow".
  2. Assign or ensure the role ESBMessaging.send for the service instance.
  3. Create a service key for this instance and use the provided client id/secret and token URL.
  4. Request an OAuth token using grant type client_credentials from the token URL of the "integration-flow" instance.
  5. Call the iFlow HTTP endpoint using the new Bearer token.
  6. Do not use credentials from the "api" plan for executing iFlows; they are intended for content management APIs.

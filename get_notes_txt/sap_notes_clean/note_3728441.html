Symptom

Approver can not see the timesheet entry in Approve Timesheet V4 application.

Environment

S4HANA

Reproducing the Issue

  • Open Approve Timesheets V4 application
  • No entry is visible. 

Cause

The Approve Timesheet V4 application collect the reportees in CL_HRFIO_EMPLOYEE_API /GET_DIRECT_REPORTS. This use the A012 relationship.

The timesheet entry can not be visible, when

  • approver is not line manager
  • manager is not based on relationship A012
  • approver is project manager

Resolution

The primary point of influence for determining who can approve an item lies within the Approve Timesheet application itself. 

To influence the standard behavior is to implement the Badi HRFIO_B_APPROVE_TIMESHEET: 

  • the method HRFIO_IF_APPROVE_TIMESHEET~READ_REPORTEES_FOR_APPROVER to collect the relevant/possible  employees
  • the method HRFIO_IF_APPROVE_TIMESHEET~ALTER_TIME_RECORDS  should be employed for further selection based on the registered items   

Note:

  • The My Inbox app collect the timesheet entries based on the workitems connected to the user. 
  • The value help for employee search in the Approve Timesheet V4 app use also the relationship (A012). So to make the value help available the method READ_REPORTEES_FOR_APPROVER , to implement only the method ALTER_TIME_RECORDS is not enough.

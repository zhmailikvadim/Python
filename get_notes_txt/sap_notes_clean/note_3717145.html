Symptom

  • Try to configure new method of trusted authentication using 3559381 - Configuring Trusted Authentication in 4.3 SP04 Patch 1000+ & BI 2025 - SAP for Me 
  • Even after following all steps' trusted authentication is not working redirects to login page.
  • Below appears in CMS log entries
    TrustedAuthMgr: secret pushed to https://<HOST>:<PORT>/BOE/CMC/TrustedAuthSetTokenServlet: code 403 

Environment

  • SAP BusinessObjects BI Platform 4.x
  • Supported Apache Tomcat web application server
  • Usually with load balancer URL

Reproducing the Issue

  1. Setup new method of trusted authentication.
  2. Testing with query string fails with this error: Account information not recognized: LDAP Authentication with an empty password is not supported when single sign on is not enabled

Cause

  • Usually if load balancer is configured to be https the individual tomcat servers are left http, then this causes a failure while Establishing Trust for the CMS to communicate with Tomcat (WAS) 

Resolution

  • HTTPS must be configured on the direct web application server (eg Tomcat) that BI is accessible with, for configuration steps, see your Web Application Server documentation. SSL offloading via Load Balancers is not recommended.
  • Please refer to Prerequisites for configuration in Prerequisites for configuration

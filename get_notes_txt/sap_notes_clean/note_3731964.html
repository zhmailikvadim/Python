Symptom

An Autocomplete from Custom Table attribute used as a Quote Item Custom Field fails to evaluate the Preselection Filter per line item. Instead, it caches the values of the first line item and displays them for all subsequent items in the quote, regardless of their unique Product.SystemId.

Environment

SAP SALES CLOUD CPQ

Reproducing the Issue

  1. Create a custom table with 3 columns, for example "ItemID" and "Text" and "null"
  2. Create an autocomplete attribute sourced from a custom table and add this table as source
  3. In the filter, put a tag "ItemID='<*CTX ( Product.SystemId )*>'"
  4. In "Fields, Calculations, Layout", add a field of type attribute and add the recent created attribute
  5. Add the needed permissions in order to the QICF be visible in the quote.
  6. Create 2 products with different system ID's, e.g "1234" and "4321"
  7. In the custom table, add entries, for the "ItemID" add 1234 and add any text e,g "First item".
  8. Repeat the process for another entry, but for "4321" and with text "Second item".
  9. Now in the Quote, add the 2 products and find the QICF
  10. Notice that if the 1st product of the quote is the one with system id 1234, the value for the QICF will be "First Item" for the 2 products.
  11. If you switch the products order, the value for the QICF will be "Second item" for both products.

Cause

Bug in the application.

Resolution

Tentative fix version is SAP CPQ 2608 release.

Keywords

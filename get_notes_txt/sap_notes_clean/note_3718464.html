Symptom

  • Employee migration from ERP/S4H to SF Employee Central fails.
  • Error message in ERP/S4H system: "GET_BUSINESS_SYSTEM_ERROR An error occurred when determining the business system (SLD_API_EXCEPTION) for transformation template ID &1 and employee &2, during data replication in EC system".
  • Error message in CPI: "HTTP operation failed invoking https://[SF url]/odata/v2/upsert?purgeType=full with statusCode: 500".

Environment

  • SAP SuccessFactors Employee Central integration to SAP Business Suite / SAP SuccessFactors Employee Central integration with SAP Human Capital Management for SAP S/4HANA
  • Employee Data migration from SAP ERP HCM to Employee Central
  • Business Integration Builder (BIB)

Reproducing the Issue

  1. Run ECPAO_EMP_EXTRACTION in ERP/S4H system.
  2. Observe replication failure with SLG1 error message: "GET_BUSINESS_SYSTEM_ERROR An error occurred when determining the business system (SLD_API_EXCEPTION) for transformation template ID & and employee &, during data replication in EC system".
  3. Check CPI tenant and observe error: "HTTP operation failed invoking with statusCode: 500".
  4. Verify SF EC system and confirm no corresponding data available.

Cause

  • The error "GET_BUSINESS_SYSTEM_ERROR An error occurred when determining the business system (SLD_API_EXCEPTION)" typically occurs when the Service Consumer lacks a default logical port configuration in transaction SOAMANAGER.

Resolution

  1. Review the implementation guide titled "Enabling the SAP S⁴HANA Outbound Web Services to Set Up the Connection from SAP S⁴HANA to the Middleware".
  2. Ensure that a default logical port is configured within transaction SOAMANAGER.
  3. Adjust the configuration as outlined in the guide to resolve the issue.

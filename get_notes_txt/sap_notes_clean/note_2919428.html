Symptom

The following symptoms can be observed:

  1. com.sap.me.production$SfcStartService#startSfc – this API method has slow running SQL query from RESOURCE_TYPE_RESOURCE table.
  2. com.sap.me.production$CreateSfcService#createSfc - For each created SFC it's checked if there are standards defined in scheduling standards configuration for combinations of the Material, Routing and Operation, and if found then Earned Standards are initialized with these values. 
  3. com.sap.me.production$CreateSfcService#createSfc - “Create SFC on the fly” business logic locks SHOP_ORDER table just to validate that it is still enough quantity in shop order to create passed SFC with provided quantity.
  4. com.sap.me.demand$ShopOrderService#releaseShopOrder - For each created SFC it's checked if there are standards defined in scheduling standards configuration for combinations of the Material, Routing and Operation, and if found then Earned Standards are initialized with these values.
  5. com.sap.me.production$SfcCompleteService#completeSfc – this method should be extended with new properties to allow skip not used logic during SFC Complete.
  6. SFCRoutingCompleteWorker#getSfcStepData, SFCRoutingCompleteWorker#isSfcInCompletePending – these methods use LIKE operator to search in SFC_STEP.OPERATION_BO column, which can be very expensive from DB perspective on large data tables.
  7. com.sap.me.datacollection$DataCollectionService#logSfcDcGroups – LogSfcDCGroupRequest request was extended with stepId property to avoid DB read to find step ID value needed for activity log functionality
  8. com.sap.me.datacollection$DataCollectionService#logSfcDcGroups was optimized to use more optimal API - readSfc method was replaced with sfcStateService.findProcessLotRefByMemberRef
  9. com.sap.me.production$SfcDataService#collectSfcData was optimized to use more optimal API - readSfc method was replaced with sfcStateService.findSfcDataByRef
  10. Added TIMING traces to Start functionality

Other Terms

PAPI, optimization, performance

Reason and Prerequisites

Some performance improvements can be implemented for the methods listed in Symptom section.

Solution

Code changes were done to improve the performance of the listed above methods.

  1. Query was changed to select only one record from RESOURCE_TYPE_RESOURCE table for provided resource and resource type.
  2. skipEarnedStandardsInitialization optional property was added to CreateSfcRequest request object to control earned standard initialization. Default value is “false”.
  3. disableShopOrderSelectForUpdate optional property was added to CreateSfcRequest request object to control lock on SHOP_ORDER table. Default value is “false”.
  4. skipEarnedStandardsInitialization optional property was added to ReleaseShopOrderRequest request object to control earned standard initialization. Default value is “false”.
  5. Extended CompleteSfcRequest request object with new optional properties:
  • disableShopOrderQtyDoneIncrease = true/false. Default value is “false”, will preserve the existing behavior.
  • disableInspection = true/false. Default value is “false”, will preserve the existing behavior.
  1. Replace LIKE operator with “=” for these two methods.
  2. LogSfcDCGroupRequest request was extended with optional stepId property to avoid DB read to find step ID value needed for activity log functionality

More details and usage examples can be found in the attached PDF file.

 

Installation:

  • Deploy SAPMECORE with latest changes to NW AS.
  • Import SAPMECORE to NWDI SC

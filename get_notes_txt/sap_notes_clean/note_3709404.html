Symptom

  • A Modify event is recorded in the auditing database even though the user did not make any changes to the We Intelligence document .

Environment

  • SAP BusinessObjects Business Intelligence Platform 4.x/2025

Reproducing the Issue

  1. Login to BI LaunchPad with a user who does not have rights to modify a Web Intelligence document.
  2. Navigate to Folders.
  3. Right-click on a Web Intelligence document and select Properties.
  4. Attempt to modify the Title, Description, or Keywords, then click Save.
  5. Getting the error: "Sorry, you do not have the right to 'Edit objects' (ID: 6) for 'Document name' (ID: document id). Please contact your administrator if you require this right '"
  6. Running the following query in auditing database "SELECT User_name , object_name , event_type_id from ADS_EVENT where event_type_id = 1007" shows that a Modify event is captured in the auditing database, even though the user did not successfully change the document properties.

Cause

  • This behavior is by design

Resolution

  • This is the expected system behavior. The auditing system logs attempts to modify objects, even when the user does not have the required rights to make changes .

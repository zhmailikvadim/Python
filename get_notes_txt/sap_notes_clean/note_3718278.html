Symptom

When adding information about Purchase Requisition, an error in HEADER is issued.
All the data are the same, just including "PurchaseRequisition": and "PurchaseRequisitionItem": in Item level. 

Environment

  • SAP S4/HANA 2023
  • SAP S4/HANA 2025

Reproducing the Issue

  1. Use the API API_PURCHASEORDER_2 to create a purchase order with with reference to PR.
  2. Error "Header Data are still incorrect" is returned in result.

Cause

Certain PR processes allow creation of only one PO with reference to a single PR.

Resolution

Do not to provide the quantity and unit in Schedule line entity:

 "_PurchaseOrderScheduleLineTP": [

 "ScheduleLineOrderQuantity": ,
 "PurchaseOrderQuantityUnit": 

Keywords

MEPO 002, API_PURCHASEORDER_2, PR, PO, V4 API

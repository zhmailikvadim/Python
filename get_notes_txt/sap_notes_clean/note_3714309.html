Symptom

Previously, the ability to resolve, reject and reopen comments in Signavio was based on the license held by the user.

  • Process Modeler
  • Collaboration Hub
  • Journey Modeler
  • Process Transformation Manager

Resolution

As of Signavio Process Manager Version 26.5.0, the behavior is that anyone who has Write (W) access to the model can resolve, reject and reopen comments.
Write (W) access to the model means the user has the "Editor" role.
Resolving, rejecting and reopening comments no longer depends on what license the user has.

Users have full authority over their own comments, so they can always resolve, reject, reopen, delete, edit their own comment.

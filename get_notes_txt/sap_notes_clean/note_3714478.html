Symptom

Running SUM, at phase MAIN_SHDIMP/SUBMOD_SHD2_RUN/TABIM_POST_SHD, in ST22 dump "Table OCS_COMP_CACHE does not exist on the database with ID 'dbtrtab'" keeps coming.

Environment

SUM

Reproducing the Issue

Repeat phase MAIN_SHDIMP/SUBMOD_SHD2_RUN/TABIM_POST_SHD

Cause

Table "OCS_COMP_CACHE" does not have package "SPAM" assigned

Resolution

1.Reset SUM following SUM guide.

2.Update SPAM to latest version to add package "SPAM" to table "OCS_COMP_CACHE".

3.Restart SUM from scratch.

Keywords

OCS_COMP_CACHE, dbtrtab

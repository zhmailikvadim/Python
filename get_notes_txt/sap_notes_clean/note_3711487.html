Solution

After you implement this SAP Note, the system enables the access key on the Service NF Data tab of incoming NFS-e screen. Here are the key points of this update:

  • Enablement of the Access Key: The system allows the access key to be displayed and some fields modified, which was not previously possible.
  • Check Digit Calculation: This calculation uses module 11 and is influenced by all fields in the NFS-e Key.
  • The General Environment field can be directly edited by the user.
  • The Random Number field is editable and if it contains fewer than 9 digits, the system adds leading zeros.

  • The NFS-e Number field must be entered in the Service NF-e section.

    • If the NFS-e Number has more than 13 digits, only the first 13 are considered in the Access Key section.

    • If the NFS-e Number has fewer than 13 digits, the system adds leading zeros in the Access Key section.

  • The Fed.Regn.Type field is automatically determined based on the CNPJ/CPF Number. When the number corresponds to a CPF, the value is 1, and when it corresponds to a CNPJ, the value is 2.

Customizing

The system adds the AMBGER (NFS-e Generation System), CDV (Check Digit in Access Key), DOCNUM9 (Random Number in Access Key) fields from NFE_NFSE_ACCESS_KEY structure to Field Group 21 (Service NF-e) in the Field Assignment to Field Group (Header) (J_1BAKHV) view. These customizing are delivered via the new report NOTE_3711487.

Use the Note Assistant (SNOTE) to implement the correction instructions. You can find more information about the Note Assistant in SAP Support Portal, under https://support.sap.com/en/my-support/knowledge-base/note-assistant.html.

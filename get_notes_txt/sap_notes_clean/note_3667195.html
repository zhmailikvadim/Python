Symptom

This regression note lists incompatible changes and known regressions for kernel 777 patch level 837.

This SAP Note is updated as soon as new issues are discovered.

Other Terms

Kernel, error list, disp+work, KRNL777PL837, regression, incompatibility, incompatible change

Reason and Prerequisites

This regression note lists incompatible changes and known regressions for kernel 777 patch level 837.

Review this Note before installing this kernel patch. 

  • Incompatible change is a changed behaviour introduced consciously in the course of a necessary adaptation. Your action might be necessary to adjust the system.   
  • Regression is an error that was caused as an unwanted side effect of a previous kernel patch. Subsequent kernel patch will deliver the fix.

There is a regression note for each released kernel PL (for further information see SAP Note 1802333).

Solution

Before installing this kernel patch, check if your installation is affected by any of below listed incompatible changes or known regressions. After the installation, check this SAP Note regularly for updates.

Incompatible Changes

  • 3549556 (Error Create failed during internal communication)
    in PL 721 and higher
  • 3381634 (Correct Decfloat Dec semantics in special cases)
    in PL 623 and higher - also removes not correctly working functions.
  • 3271972 (syntax errors or runtime errors if, as suggested in SAP Note 3149562, you use %_HINTS HDB)
  • in PL 313 and higher
  • 3115048 (sapstartsrv with native Linux systemd support)
    in PL 441 and higher. Also see Note 3139184 (Linux: systemd integration for sapstartsrv and SAP Host Agent) 
  • 3026990 (RFC Logon - New Internal Logon Ticket - Increased Compatibi618ity Level)
    in PL 330 and higher
  • 3016480 (Logon with SNC credential from external RFC client works despite having SNCSYSACL entry configured)
    in PL 323 and higher
  • 2954810 (Error while setting PF status on selection screens)
    in PL 118 and higher (Kernel regression introduced with note 2912350. This issue will not be corrected with a kernel patch. Instead, install ABAP correction of Note 2954810).

Regressions

Regressions are listed in the "References" section. Check them one by one in order to determine:

  • Whether your installation is affected.
  • Whether a workaround exists.
  • Whether a new kernel PL is available to fix the regression.

In downward-compatible kernel (DCK) scenario also check the DCK note 2870484 (Using kernel 7.77 instead of kernel 7.73) for the listed DCK issues.

The list of all changes (error corrections and improvements) delivered in this patch is available the "Content Info" file of the respective kernel archive on the SAP Support Portal.

Symptom

  • SAP HANA data backup runtime is longer than expected
  • Backup completes successfully without errors
  • No indication of failures in backup.log or backint.log

Environment

SAP HANA DB Version 2.0

Cause

In SAP HANA, data backups read persisted pages from the data volume.
The distribution of LOB data can influence backup behavior.
In particular, a high number of LOB objects with relatively small average size may lead to increased read operations and additional metadata overhead.
For general background information about SAP HANA LOBs, refer to SAP KBA 2220627.

Resolution

There is no indication of a malfunction in the SAP HANA backup framework.
The observed runtime can be consistent with the underlying data layout.

The following analysis steps can be considered:

  • analyze LOB distribution using SAP-provided SQL statements (e.g. SAP Note 1969700)
  • review tables with high LOB object counts and small average LOB sizes
  • evaluate from an application perspective whether stored LOB data is required
  • consider housekeeping or archiving options where applicable

See Also

The system view M_TABLE_LOB_FILES can be used to analyze LOB data.

Example:

SELECT SUM(physical_size)
FROM M_TABLE_LOB_FILES
WHERE (physical_size / page_count) < (64 * 1024);
SELECT SUM(physical_size)
FROM M_TABLE_LOB_FILES
WHERE (physical_size / page_count) < (16 * 1024);

For detailed table-level analysis, SAP provides SQL script collections (SAP Note 1969700).

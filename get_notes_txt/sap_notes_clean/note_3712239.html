Symptom

During the activation of a transformation TRFN with SAP HANA runtime for an error of the following type:

SQL error with code '2.048'. See the following SQL message:
=> column store error: fail to create scenario: [34011] Inconsistent calculation mo
=> del;calculationNode (0BW_SRC_INFOSOURCE_1.[IOBJNM].JOIN.[number].FILTER_JOIN): Mappin
=> g for attribute DATEFROM was found morethan once,Details (Errors): - calculatio
=> nNode (0BW_SRC_INFOSOURCE_1.[IOBJNM].JOIN.[number].FILTER_JOIN): Mapping for Attribute
=> e DATEFROM was found more than once. - calculationNode (0BW_SRC_INFOSOURCE_1.
=> [IOBJNM].JOIN.[number].FILTER_JOIN): Mapping forattribute DATETO was found more than on
=> ce.

The prerequisite for this is that the content of a target field is determined in the model using a lookup and the lookup is based on a time-dependent InfoObject.
In addition, DATEFROM and/or DATETO are used explicitly in the model:

Example:
An aDSO is updated in two steps:

RSDS (DataSource) -> TRFN(1) (Transformation) -> TRCS (InfoSource) -> TRFN(2) (Transformation) -> ADSO (DataStore (advanced))

In TRFN(1), the fields are mapped 1:1 as follows:

RSDS   TRCS
FIELD1(key) -> FIELD1(key)
DATEFROM -> DATEFROM
MY_EQUINO -> MY_EQUINO
    BPARTNER (Constant e.g.)


In TRFN(2), the fields are mapped as follows:

TRCS   ADSO  
FIELD1(key) -> FIELD1(key)  
DATEFROM -> 0DATEFROM  
MY_EQUINO (CHAR18),
BPARTNER (CHAR10)		 -> ZBED41TYP(CHAR10) Lookup

Lookup object on ZBED41TYP is: ZBE41CEQP
with the mappings:
Step 1
ZBE41CEQP (Lookup Object) <-> MY_EQUINO (TRFN Source)
0BPARTNER (Lookup Object) <-> BPARTNER (TRFN Source)

Step 2
ZBEC41TYP (Target) <-> ZBEC41TYP (Lookup)

where the lookup IOBJ ZBE41CEQP(CHAR18) is defined as follows:
ZBE41CEQP with 0BPARTNER(CHAR10) as compounding parent
and the time-dependent navigation attribute ZBEC41TYP(CHAR10)

Since a time-dependent IOBJ internally uses the DATEFROM and DATETO fields to specify the time interval, collisions occur during mapping in the join condition.


You can only rename the DATEFROM field in the InfoSource (TRCS), for example: DATEFROM_X
or implement the attached correction.
In both cases, it should be possible to activate the transformation afterwards.

Other Terms

RSDHA, TRFN, 2048, lookup, CL_RSDHA_JOINS, P_ADD_MAPPINS, timedep, datefrom, dateto, 0datefrom, 0dateto

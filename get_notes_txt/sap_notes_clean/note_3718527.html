Symptom

Intermittent messages failing at SuccessFactors receiver adapter with an error "com.sap.gateway.core.ip.component.odata.exception.OsciException: HTTP Request failed with error: Cannot assign requested address: xxx.xxxxx.com:443, cause: java.net.BindException: Cannot assign requested address: xxxx.xxxxx.com:443".

Environment

  • SAP Cloud integration
  • SAP Integration Suite

Reproducing the Issue

  1. Attempt to process messages through interfaces connected to target system xx.xxx.xxx.
  2. Observe intermittent failures in message processing.
  3. Review the error logs for occurrences of the error: "Cannot assign requested address: xxx.xxx.xx:443".

Cause

The error message indicates that there is an issue with the network connections when trying to connect to the SuccessFactors API. 

Resolution

Enable the "Reuse Connection" option in the Successfactors Odata adapter. This allows the system to reuse existing connections instead of creating new ones, which can help prevent exhausting available ports.

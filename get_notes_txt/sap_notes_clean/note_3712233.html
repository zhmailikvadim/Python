Symptom

According to the latest regulations from authorities the non-taxable income for trainers affects also Health insurance base. If any non-taxable income reduces Tax base, the same amount should reduce also base for HI. If HI base after reduction is zero, the employee is considered as non-insured person for HI for such month. He/she must maintain Health insurance with HI agency as self-paid person (if there is no other source of HI contributions paid like another work contract). If employee is uninsured due to zero HI base and has any absence with paid compensation (like Sickness, Family member treatment etc.) he/she has to be reported as insured for days of absence. Announcements to HIA have to be updated to meet abovementioned regulations.

PDF form for HSKCDPP0 has been fixed (previously there was a typo in the text).

Other Terms

HRSK, HSKCALCM, HSKCALC0, SKCZP, SKCTX, HSKCZPV0, HSKCZPL2, HSKCDPP0, /289, /T1U

Reason and Prerequisites

This SAP Note is relevant only for Slovakia (HRSK)

Solution

Source code of payroll functions SKCTX and SKCZP and report HSKCZPL2 was changed to support latest regulations of HI and Taxes in connection to non-taxable income of trainers.

1. For customers having no trainers in the system, there is no change in processing of HI and Taxes. But we recommend to implement the Note also for such customers because it contains also some code clean-up, Log improvements and also updated PDF form of HSKCDPP0.

2. For customers having trainers in the system, there are some changes: 
a) if wage type /289 is not entered in infotype 0014, the person is considered as normal employee (not as trainer) and the calculation and reporting works as usual.

b) if trainers wage type /289 is entered into infotype 0014, such employee is considered as trainer and Health insurance/Tax is calculated automatically using up to 300 EUR as non-taxable income. After the calculation, the amount of really used non-taxable income* is stored in wage type /289 in amount (BETRG) field.

*typically 300 EUR, but the amount could be lower, if total income/HI base before applying non-taxable income is lower than 300 EUR. It could be used up to zero - no negative values are allowed.

If trainers income/HI base was originally higher than 300 EUR then after applying non-taxable income there is still not-null HI base, such employee is still insured for HI and will be reported with lowered base and full insured days. No special announcements are needed.

If trainers income/Hi base is originally lower or equal to 300 EUR then after applying non-taxable income there is zero base for HI - such employee is considered as "not insured" for the whole month. User has to change field KVKZ1 to value 0 in infotype 0253 for such period. Announcements to HIA (report HSKCZPL2) will generate 2/K (2D/K) message at the last day of preceding period and then 2/Z (2D/Z) message at the first day of next period. New parameter "Process messages for trainers acc.to P0014/P0253" (P_TRAIN) must be checked (enabled).

But if there is any absence with paid compensation during the period - field KVKZ1 should not be changed to 0. The days without absence are not insured, but days of absence are insured. User has to add wage type /T1U to the infotype 0014 for period without absence.  For example, if employee has income originally 250 EUR, after applying non-taxable income of trainer it's zero, but if there is an absence (sickness 0400) since 17.01.2026 till 31.01.2026, wage type /T1U must be entered into P0014 with dates 01.01.2026 - 16.01.2026. employee is insured only during absence days and for the rest days he/she is considered as not insured for HI. Report HSKCZPL2 will then generate 2/K (2D/K) one day before /T1U entry.. in this example it's 31.12.2025 - means that HI insurance ends. And then 2/Z (2D/Z) on the first day of absence.. in this example it's 17.01.2026 - means that HI insurance starts again.

Summary:
a) if there is no absence and original income is higher than 300 EUR, no change in P0253-KVKZ1 is needed and wage type /T1U is not needed as well.
b) if there is no absence and original income is lower than (or equal to) 300 EUR, P0253-KVKZ1 should be changed to 0. Wage type /T1U is not needed.
c) If there is absence for the whole month, employee is insured for the whole month: no change in P0253-KVKZ1 is needed and wage type /T1U is not needed as well. 
d) If there is an absence in the part of month, but the rest income for days without absence is higher than 300 EUR, the employee is insured for the whole month: no change in P0253-KVKZ1 is needed and wage type /T1U is not needed as well.
e) If there is an absence in the part of month, but the rest income for days without absence is lower than (or equal to) 300 EUR, the employee is insured only for days of absence: no change in P0253-KVKZ1 is needed but wage type /T1U is needed for days without absence.

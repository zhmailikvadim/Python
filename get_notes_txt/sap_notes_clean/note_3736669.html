Symptom

Symptom

  • Error when accessing the story installed from Intelligence Application.
  • Story may show empty/older data or display "We couldn't connect to your SAP Datasphere tenant. For more information, see our troubleshooting page." in SAP Analytics Cloud

Environment

  • Story Design & Visualizations
  • SAP Business Data Cloud intelligent applications
  • SAP Analytics Cloud
  • SAP Datasphere

Reproducing the Issue

  1. Log in to SAP Analytics Cloud.
  2. Open the installed story from the Intelligence Application, e.g. Working Capital Insights app..
  3. Observe that access may raise an error, or the story opens but shows no data or older data.

Cause

Possible cause:

  • The user was not assigned the DW Consumer role in SAP Datasphere, or that role was not scoped to the intelligent application's space. 
  • There is failed Replication flow in SAP Datasphere.
  • Row-level security (central permissions) for the intelligent application was not configured for the user, resulting in an empty story.

Resolution

  • In SAP Datasphere, assign the DW Consumer role to the affected user. Add the user to the intelligent application’s space used by the story (for example, the space delivered by Working Capital Insights) and scope the DW Consumer role to that space. See Add Users to a Scoped Role in the SAP Datasphere help.
  • Verify the data replication status in SAP Datasphere and monitor flows; resolve any failed stages before re-testing story access. See Monitoring Flows in the SAP Datasphere help. 
  • Configure row-level security for the delivered data by maintaining the central permissions table and adding the required user identifiers according to the application’s security model.See Applying Row-Level Security to Data Delivered through Intelligent Applications in the SAP Datasphere help.

Environment

SAP SuccessFactors Employee Central - Time Off

Reproducing the Issue

  1. Work Schedule has planned hours as say 7:20 which is 7.3333 hours (Decimal would be 0.666, 0.333 etc.)
  2. Time type configuration is:
    • Permitted Fractions For Unit Hour: Requesting in hours and minutes &
    • Allow Requests In Increments Of [Minutes]: 15 (say)
  3. Request a full day absence for the employee using the above time type, the error message says "Please enter an amount that is a multiple of 15 minutes" 
  4. Expected behavior is that the absence should be successful (since Allow Requests In Increments Of [Minutes] is not applicable for full day absence)

Cause

This is a known defect.

Resolution

  • Product engineering is working on the fix.
  • The fix is planned via patch 1H2026p0 which is tentatively planned on 13th April 2026 for Preview & 16th April 2026 for Production environments.
  • This KBA will be updated once the fix date is confirmed.

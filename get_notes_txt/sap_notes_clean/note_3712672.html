Symptom

While configuring the system and User permissions, some users must not be allowed access to certain menu and option within Field Service Management

Environment

SAP Field Service Management

Reproducing the Issue

  1. Access FSM Admin portal
  2. Create a new User Policy Group
  3. Assign some permissions then assign some users to it
  4. Check what options are available for that user in the Web Application

Cause

Certain menus are tied to some specific object permissions within the Policy Group

Resolution

Here's a list of available menus and tabs within the Web Applications of Field Service Management and the respective necessary permissions to access them. If you want to restrict access to entire Apps, check the SAP Note 3379848 - Limiting access to different apps inside the web application.
Please note that ERP Type companies have different permissions applied to company level that will supersede the Permissions on the Policy Group, mainly when it comes to Creating and Deleting objects that are managed from the ERP System

  • Service Call App
    • Skill tab ->CRUD permissions for REQUIREMENT, SKILL and TAG
    • Smartform tab -> CRUD permissions for CHECKLISTINSTANCE, CHECKLISTTEMPLATE and CHECKLISTCATEGORY
    • ReservedMaterial tab -> CRUD permissions for ITEM, WAREHOUSE, RESERVEDMATERIAL and PERSON
    • Attachments tab ->  CRUD permissions for ATTACHMENTS
    • Equipment tab -> CRUD permissions for Equipment and company setting SAP.FSM.SC.Equipment.Mode on mode 1
    • Custom screens -> CRUD permissions for Screen Configuration
    • Custom Fields and Values -> CRUD Permissions for UDFMeta
    • History tab -> AUDIT_LOGS user role
  • Master Data Application  
    • All menus with the exception of Crew can be hidden via the UI Configuration 'Visible' for the corresponding object
    • Crews menu cannot be hidden, the Crew app doesn't support Policy Group restrictions
    • BusinessPartner, ServiceContract, Equipment, Person, Item and Teams object list requires the CRUD permissions for the corresponding Objects to be visible
    • Skills list requires the CRUD permissions for both Skills and Tag objects. Please note that the TAG object represents the Skill template, while the Skill object represents the assignment of that Skill to a Technician. 
  • Setting and Configurations
    • Master Data
      • WorkTimePatterns tab can be hidden by the  UI configuration 'Visible' for WorkTimePatterns
      • Other tabs cannot be hidden, only the internal options based on what they change (EmployeeDepartment to hide the Department section in People Settings for example)
    • Planning and Dispatching
      • Planning Scenarios tab can be hidden by the UI configuration 'Visible' for PlanningScenarios
      • TimeSlotTypes tab can be hidden by both UI configuration 'Visible' for WorkTimeTask and PersonReservationType. With either enabled, the tab is visible. The object corresponding for 'Reservation Types' is PERSONRESERVATIONTYPE and for 'Absence Types' is WORKTIMETASK
      • In General Settings, the tiles cannot be hidden but you can restrict updates to ServiceCallMappings by the Object ServiceCallStatus and Service Regions by the Object Region
      • All tabs except Planning Scenarios will be hidden by the UI configuration 'Visible' for CompanySetting

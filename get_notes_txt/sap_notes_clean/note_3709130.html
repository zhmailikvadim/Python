Symptom

  • Recurring dumps DBSQL_STMNT_TOO_LARGE occur when performing searches in transaction VL06O.
  • The dumps is:
    Category               ABAP programming error
    Runtime Errors         DBSQL_STMNT_TOO_LARGE
    Except.                CX_SY_OPEN_SQL_DB
    ABAP: Program          SAPLV50Q
    Application Component  LE-SHP-GF
  • The ST22 dump happened at form "ENHANCE_POSTAB_BY_TM_DATA" 
    SAPLV50Q                       / LV50QF43
    FORM            / ENHANCE_POSTAB_BY_TM_DATA

Environment

  • Logistics Execution (LE)
  • SAP Enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA

Reproducing the Issue

  1. Log in to the SAP GUI.
  2. Execute transaction VL06O.
  3. Perform searches within the transaction.
  4. Observe recurring dumps and error messages.

Cause

OSS note 3622045 - LE: Correction of Freight Order Not Being Populated for Delivery in Outbound/Inbound Delivery Monitor is missing in the system.

Resolution

Please implement the attached correction instructions in SAP Note 3622045.

See Also

  • Refer to SAP Note 3622045 - LE: Correction of Freight Order Not Being Populated for Delivery in Outbound/Inbound Delivery Monitor.

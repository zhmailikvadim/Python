Symptom

You run a Structure Version that uses a classic BOM with PLMSI. When you make manual changes in a BOM, such as deleting BOM items or reverting ECN changes using the transaction CCUNDO. After a new successful SV transfer by PLMSI you find duplicated BOM items that are not limited by the ECN validity date.

Other Terms

PLMSI, PLM System Integration, Classic BOM, SV, Structure Version

Reason and Prerequisites

Manual changes of the classic BOM, such as BOM items deletion or reverting ECN changes by the transaction CCUNDO, lead to an inconsistent state in PLMSI BOM key mapping tables.

Solution

Implement this SAP Note to correct the issue with duplicated classic BOM items.

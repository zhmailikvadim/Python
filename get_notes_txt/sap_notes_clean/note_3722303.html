Symptom

When executing the Post Processing Event-Based Product Costing application for the current period, if the Goods Issue (GI) was posted in a previous period, the system will generate the following error:

"Posting period &1 &2 not open for variant &4 and ledger &3."

This error occurs because the system attempts to post in the original GI period, which is currently closed in the posting period variant and ledger configuration.

Other Terms

Post Processing Event Based: Product Costing, Production/ Process Order, Production Accounting, Cost Object Controlling, Universal Parallel Accounting, UPA, FINS_PARALLEL_ACCOUNTING_BF，Event-Based, Event-Based Production Cost Posting, 3F0, UPA On, Report R_FCO_RTPC_INSPECTOR

Reason and Prerequisites

The WIP was posted to previous period and current period if the posting date is current period, hence causing the issue, this design has been change in current release

Solution

This fix introduces is to avoid the event based WIP posting in previous period.

  DELETE lt_posting_periods_ml WHERE jahrper < mv_fiscal_year_period.

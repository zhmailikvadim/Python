Symptom

The feature TRVFE has no documentation and the default delivered version of the decision tree delivers a PDF form variant PTRV_STATS_FORM which does not exist in the system.

Reason and Prerequisites

Design

Solution

Implement the Support Package.

The long text of the feature TRVFX will then be changed to: Form Variants for Reports RPRTEF01_PDF and RPRTEF02_PDF

The default delivered version of the decision tree in delivery client will then deliver only PTRV_SEF_FORM for Trip Schema 01 and PTRV_SEAF_FORM for Trip Schema 02. There won't be an Others branch.

The documentation of the feature in PE03 transaction will then contain:

---------------------------------------------------------------------------------------------

TRVFX Form Variants for Reports RPRTEF01_PDF and RPRTEF02_PDF

Object

Feature

Task

Maintain return values for this feature when reports RPRTEF01_PDF and
RPRTEF02_PDF are used.

Use

This feature controls the PDF form variant used by reports RPRTEF01_PDF
and RPRTEF02_PDF. These reports provide PDF alternative to the ABAP List
reports RPRTEF01 (Summarized Form 1 for Travel Expenses) and RPRTEF02
(Summarized Form 2 for Travel Expenses Accounting).

Procedure

This feature allows only the field 'Trip schema' as decision field. This
decision field can have exactly 2 values: value 01 delivers back the
form variant for RPRTEF01_PDF, the value 02 delivers back the form
variant for report RPRTEF02_PDF. There is no finer granularity. Hence no
different form variants for 1 report based on other parameters are
possible.

Note

The delivered PDF form variant templates are PTRV_SEF_FORM for report
RPRTEF00_PDF and PTRV_SEAF_FORM for RPRTEF02_PDF.

Example

The delivered template decision tree returns the form variant
PTRV_SEF_FORM for Trip schema 01 (RPRTEF01_PDF) and PTRV_SEAF_FORM for
Trip schema 02 (RPRTEF02_PDF).

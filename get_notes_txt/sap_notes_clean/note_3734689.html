Symptom

  • Columns such as name and address match are visible in the 'SPL Hits' table of the 'Manage Blocked Partners' app but are absent in the exported Excel file.
  • No specific error message is displayed during export.

Environment

  • SAP Global Trade Services, edition for SAP HANA
  • Sanctioned Party List Screening

Reproducing the Issue

  1. Open the Fiori app 'Manage Blocked Partners'.
  2. Navigate to the 'SPL Hits' table and ensure columns like SPL number, name, and address match are visible.
  3. Use the Export Table function to export the list to Excel.
  4. Open the exported file and observe that columns such as name and address match are missing.

Cause

Missing notes. 

Resolution

  1. Review and implement the corrections provided in SAP Note: 3680743 - Not all columns can be downloaded in 'SPL Hits' table.
  2. If the issue persists, review and implement SAP Note: 3696731 - Not all columns can be downloaded in 'SPL Hits' table and display more 'SPL Hits' by default in 'Manage Blocked Documents' app. 
  3. After implementing the notes, clear relevant caches if applicable, log off and back on, and re-test the export from the 'Manage Blocked Partners' app.
  4. If the problem remains after the above steps, contact support and provide details of the app version and the implemented notes.

Note: Tags can be available after implementing the notes. For tags we have technical limitation to remove it. Tags could also help users to know which words are matched words. They could be easily replaced in Excel.

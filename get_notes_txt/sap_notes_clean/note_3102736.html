Symptom

You are executing a project via the SAP S/4HANA migration cockpit (approach "Migrate Data Directly from SAP System"). You have changed some parameters of the target function (added new signature parameters or added new fields in the structural parameters of your function). You have executed "Synchronize Structures" for your migration object using Migration Object Modeller (transaction LTMOM) and created field mappings for new target function parameters. Then you start one of the activities for your migration object from FIORI UI. It fails with error message 'Failed to generate transformation for transfer method parameter XXX'.

Other Terms

Field mapping, API change, missing parameter

Reason and Prerequisites

Due to a logic gap, the target function change is not properly registered in the framework. As a result, some of the relevant technical objects are not automatically re-generated.

Solution

Implement the attached correction using transaction SNOTE or import the relevant Support Package in the SAP S/4HANA system.

New parameter changes will then be properly registered in the framework, and all required technical objects will automatically be re-generated again.

Note that this correction does not repair your current erroneous object. Create a new project with the same migration object and implement the required field mappings again.

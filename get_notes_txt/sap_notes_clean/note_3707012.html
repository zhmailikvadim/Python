Symptom

When submitting the Poland eDocuments created prior KSeF 2.0 implementation, following error occurs:

Status of the eDocument does not allow the execution of process step PROC_RESP - Message no. EDOCUMENT080

Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.

Environment

  • SAP S/4HANA Cloud Public Edition
  • SAP Document and Reporting Edition, Cloud Edition
 

Reproducing the Issue

  1. eDocument cockpit application
  2. Select Poland KSeF eInvoice eDocument.
  3. Click on Submit.

Cause

  • KSeF 2.0 introduces a schema change from FA(2) to FA(3).

  • As a result, the eDocument process version for PLKSEF is updated from version 1 to version 2.

  • eDocuments created before onboarding the FA(3) schema will have process version 1.

  • These process version 1 eDocuments cannot be submitted to KSeF 2.0.

Resolution

Create and submit new eDocuments which will have process version 2.

NOTE: Before creating the new eDocuments, the SSCUI "Define Process Communication Through Cloud Services" 105552 must be mapped to "Cloud Edition: Standard Integration" as mentioned in the following help portal documentation: Defining Process Communication Through Cloud Services (Electronic Documents only) | SAP Help Portal.

Symptom

After having used the old client copy tool (transactions SCCL, SCC9, SCC5, SCC7, SCC8, SCC3), selection variants in the new client are "inconsistent in the database" and cannot be used.

Other Terms

SLDBV103, client copy, selection variants, VARI

Reason and Prerequisites

Issue in the old client copy tool

Solution

As described in note

2962811 - New Client Copy Tool: General Information

starting from SAP Basis 7.54, there is a new client copy tool, which is recommended to be used from SAP Basis 7.56.

The old client copy tool can still be used up to SAP Basis 7.57. Starting from SAP Basis 7.58, it has been classified as deprecated and should not be used anymore.

Due to a programming error in the old client copy tool, selection variants are incompletely copied to the new/target client: Required entries in table VARI are not present in the new client, which makes it impossible to use the variants.

Please apply note

2996495 - Include VARI table in old local client copy

to correct the issue in the client copy tool.

The inconsistent variants must be transferred from the source client to the new cilent via transport or file download/upload, as the code correction does not repair the inconsistent variants.

See note

3365831 - Transfer/Copy of selection variants between clients/systems - Use Cases, Tools, Additional Information

for more information.

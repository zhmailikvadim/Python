Symptom

During the mass processing of source documents, the system is expected to update the relevant pools and create new pools as required.

However, pools are not being updated during source document processing, and in some cases, the pool is not getting created at all. This leads to incomplete processing and incorrect pool determination.
This note will fix the issue so that pools are correctly updated and created during the mass processing of source documents.

Other Terms

mass process, source documents processing, pool not created, pool not updated, mass processing issue, pool determination, processing failure, /IRM/IPPL, /IRM/IPPL_CLB_UPDATE, Inventory pool

Reason and Prerequisites

This is caused by program error.

if you are in V/4 –2021, 2022, 2022 V2 and 2023 ,Please implement the latest version of SAP Note # 3379781 as pre requisite note.

Solution

Please implement the correction instructions attached. These corrections are only valid if you have applied the Support Packs.

Release V4 -  2021, 2022, 2022 V2, 2023, 2023V2, 2023 V3, and 2023 V3 Support Pack 1.

Use transaction SNOTE to implement the correction instructions specified below and activate the affected program object.

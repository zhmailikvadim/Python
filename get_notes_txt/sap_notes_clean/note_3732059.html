Symptom

  • Program GRCAUD_ACTION_REMINDER does not send reminder emails when the reminder configuration uses a day range (for example, 30–60 days).
  • Reminders are sent when a single day value is configured.
  • No error message is displayed.

Environment

GRC Audit Management 1.6

Reproducing the Issue

  1. Configure action plan reminder settings using a range of days (for example, 30 to 60 days before the deadline).
  2. Run or schedule program GRCAUD_ACTION_REMINDER.
  3. Observe that no reminder emails are sent for action plans with deadlines within the configured range.
  4. Change the configuration to a single day value and run the program again.
  5. Observe that reminder emails are sent as expected.

Cause

  • Range values in the reminder configuration are not supported by design for program GRCAUD_ACTION_REMINDER.

Resolution

  1. Replace any configured day ranges with multiple discrete day values (for example: 30, 7, 1).
  2. Save the configuration.
  3. Run or schedule program GRCAUD_ACTION_REMINDER according to the required notification cycle.
  4. Verify that reminder emails are initiated when action plans approach their deadlines on the specified discrete days.

    Here is a clean, professional solution write-up based on your notes and the SAP screen:

    Solution: Configure Action Plan Reminder Notifications

    To ensure accurate and timely reminder notifications for action plans, follow the steps below:

    1. Configure Discrete Reminder Days

    • Navigate to Set Action Plan Reminder in SAP Audit Management.

    • Instead of using day ranges, maintain multiple discrete day values using Multiple Selection.

    • Example values: 30, 7, 1 days before deadline (and optionally overdue days like 5, 10, 30).

    2. Save Configuration

    • Enter the required values in:

      • Remaining Days (before deadline)

      • Overdue Days (after deadline)

    • Save the configuration as a variant via:
      Goto → Variants → Save as Variant

    3. Schedule Reminder Job

    • Go to transaction SM36.

    • Schedule background job for program:
      GRCAUD_ACTION_REMINDER

    • Assign the saved variant.

    • Set frequency (typically daily) based on notification needs.

    4. Execute / Monitor Job

    • Run the job manually once or allow scheduled execution.

    • Monitor via SM37 to ensure successful execution.

    5. Validate Email Notifications

    • Verify that reminder emails are triggered:

      • On configured discrete days (e.g., 30, 7, 1 days before deadline)

      • On overdue intervals (if maintained)

    • Confirm correct email templates are used:

      • Approaching deadline

      • Overdue action plans

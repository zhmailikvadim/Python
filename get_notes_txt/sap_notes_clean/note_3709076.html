Symptom

You run the CRA: Conceptos Retributivos Abonados (RPC_PAYES_CRA) program for a large number of employees. However, you experience long processing times in database operations under infotype entries on the FORM check_partitioning_needed.

Other Terms

&CRA - PC00_M04_CRA;

For more information, see SAP Note 1071282 - CONSULT: List of tags for Spain notes.

Reason and Prerequisites

Reason

SAP Note 705870 - PNP LDB: Reduction of memory usage provides a general solution for performance improvement for reports based on the PNP Logical Database. However, this solution is not suitable for scenarios with a large number of personnel numbers in the database.

Prerequisites

You have to apply SAP Note 3705566 - UTIL: Class for defining maximum partition size for PNP reports as a prerequisite.

The following Support Package level is the minimum prerequisite:

Release Support Package
S4HCM 102 HR SP 00
S4HCM 101 HR SP 04
S4HCM 100 HR SP 17
SAP_HR 6.08 HR SP C8
SAP_HR 6.04 (EhP6 and EhP7)  HR SP K0

For more information, click on SAP ERP Human Capital Management - Support Package Schedules.

Note:

When you install an SAP Note, some prerequisites might be required. The prerequisites are SAP Notes that have already been released and that change the same context of the current one. Then, even if these prerequisites are not necessarily related to the symptom of this SAP Note, you should implement them due to technical reasons.

Solution

From now on, the RPC_PAYES_CRA program respects the Cantidad máxima por bloque PNP (PNPMS) constant in the Constantes de nómina (V_T511K) view for defining the maximum partition size for PNP reports. See SAP Note 3705566 for more details.

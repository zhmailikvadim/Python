Symptom

You have a MTAR (Multitarget Applications Archive) file and would like to know which part of the source code can be obtained from it.

Environment

  • SAP Business Technology Platform (BTP)
  • SAP Cloud Foundry (CF)
  • Release Independent

Resolution

An MTAR archive follows the JAR file specification. It can contain: 

  • built application artifacts or source bundles (for example, SAPUI5 app dist content, Java artifacts, Node.js bundles).
  • configuration and deployment descriptors (such as mta.yaml / mtad.yaml, xs-security.json, service bindings, and parameters).
  • Node.js modules (JavaScript files, package.json, etc.)
  • Java modules (JAR/WAR files)
  • HTML5/UI5 modules (HTML, JS, CSS resources)
  • CAP services .jar / .js / .cds compiled output
  • Custom logic for microservices etc. 

Whether the MTAR archive contains the complete source code depends on how the project was structured and which paths/modules were included in the MTA build. 
Unzip the file using the command mtar -x <file>.mtar or rename it to .zip and unzip it to check the contents.

NOTE:
It is recommended to use a GIT repository for managing source code (or have a local backup). Tools like SAP Business Application Studio (BAS)/Web IDE are not designed for source control, therefore SAP highly recommends using GIT. 
GIT also allows developers to have code versioning and collaboration of multiple team members on the same project.

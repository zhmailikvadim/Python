High 139222/2026 BRH WLI not loading quickly with Custom Views

Users experience long load times or timeouts when using the My Release Decisions app when the Countries/Regions column is selected as part of the table variant. To resolve this issue, the underlying SQL view stacks were improved to load the required data more efficiently, especially for large datasets.

The changes impact the following areas:

  • My Release Decisions - General data retrieval for the Release Decisions table, especially when the Countries/Regions column is selected as part of the table variant
  • Release Decision Details - Data retrieval for the Countries/Regions section
  • Regulatory Data Check - Data retrieval for the Release Check Details section

The primary focus for (all) test cases should be to verify that there are no functional changes after the hotfix has been deployed. For example, using the My Release Decisions app should show exactly the same content before the hotfix and afterwards (standard regression testing).

Secondary test focus is to check performance improvements for the described features (Countries/Regions and Release Check Details). Only applicable if there was a performance degradation for the respective features in your production environment.

High 144942/2026 Unable to regenerate WLI

Users are unable to regenerate worklist items. When you try to regenerate a worklist item to a different release type, the following error could occur: “A user is not responsible for the selected release type”. This issue is related to the 2511 release, which improved the authorization check for responsibilities within quality departments, particularly during the regeneration of worklist items.
The problem stems from the authorization check mechanism and is due to inconsistent field naming conventions at the database level for material types and material groups. Because of these naming discrepancies, when the system compares material types and material groups, it may evaluate specific values as undefined. This can cause the responsibility check to fail. To resolve this issue, the inconsistent naming has been fixed.

 Test scope should cover the regeneration scenario, either with the same or a different release type. N n/a

n/a n/a Long-running processes in background processing could lead to redundant task executions 

Background worker process management has been stabilized by adding an improved heartbeat implementation, covering scenarios where long-running processes must be executed.

 The change will improve the enhanced change detection in batches with large component trees. A test case should include one or several changes to a batch with a big component tree.  N n/a
n/a n/a Large inbound data payload could lead to degradations in background processing

Message compression has been enabled for all affected processing queues/topics.
Redundant/unnecessary data will be omitted from the message payload during internal processing.

 Test scenarios should include a large inbound data payload leading to delayed processing in SAP Batch Release Hub for Life Sciences before the hotfix. N n/a

Symptom

By setting Enable Inputs ON for an additional field (for example ProfitCenter) in configuration step (IMG) SAP S/4HANA for Group Reporting > Master Data > Define Consolidation Master Data Fields, user takes control over accounting master data in Group Reporting.

This means, if user creates master data in Group Reporting, the same master data must not be created in accounting.

If user created the same master data in accounting, which had already existed in Group Reporting, afterwards runtime error ITAB_DUPLICATE_KEY occurs in class CL_FINCS_ADDLMD_UTILITY at statement

==>>      insert value #( fieldname = iv_fieldname superordfieldvalue = iv_superordfieldvalue accounting_data = lt_acctng_records ) into table mt_acctng_data_buffer.

while importing master data in Import Master Data for Consolidation Fields app. The runtime error's description (in t-code ST22) contains the duplicate entry, for example as follows:

The key components of the duplicate entry have the values "{PRCTR;1111111119;XXX1}"

Environment

  • SAP S/4HANA Cloud, Private Edition
  • Group Reporting

Cause

While creating master data in accounting, there isn't any existence check if the newly created accounting master data already exists in Group Reporting.

Resolution

Run report of SAP Note 3305099 to resolve the runtime error or ask help of SAP Support to do so in a customer case on component FIN-CS-MD.

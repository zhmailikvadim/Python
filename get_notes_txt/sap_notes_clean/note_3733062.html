Symptom

An inspection type with origin 17 (for example: 1704) is attempted to be added to inspection setup of Material Master using BAPI_MATINSPCTRL_SAVEREPLICA.

But below error is returned in table RETURN of the BAPI:

"Field Control of Inspection Lot Creation (Lot Summary) (CONTR_INSP_LOT_CREATE) not supported for inspection type. Leave the field empty"

(message QS397)

"Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental."

Environment

SAP S/4HANA from 2022 onwards

Cause

Value had been entered to field CONTR_INSP_LOT_CREATE of table INSPECTIONCTRL. But this field belongs to field QMAT-CHG, which is not relevant for inspection types with origin 17.

For inspection types with origin 17, lot summary is maintained in an other field: QMAT-CHG_ORIG_17. The corresponding field of the BAPI is: INSPECTIONCTRL-CONTR_INSP_LOT_CREATE_17.

Resolution

Use field INSPECTIONCTRL-CONTR_INSP_LOT_CREATE_17 in the BAPI to maintain lot summary for inspection types with origin 17. 

And leave field INSPECTIONCTRL-CONTR_INSP_LOT_CREATE empty.

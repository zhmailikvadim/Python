Symptom

  • In the user settings dialog, the Date Format cannot be edited when launching apps via SAP Build Work Zone, Standard Edition.
  • The date format field is read-only and automatically derived (for example: MM/DD/YY).
  • Changes maintained in SAP S/4HANA Cloud Public Edition user settings (for example via  SU01) are not reflected in SAP Build Work Zone.

Environment

  •  SAP Build Work Zone, Standard Edition
  •  SAP S/4HANA Cloud Public Edition

Reproducing the Issue

  1. Launch the app from SAP Build Work Zone.
  2. Open User Settings.
  3. Select "Language and Region".
  4. Try to change the date format from MM/DD/YY to DD/MM/YYYY.

Cause

  • In SAP Build Work Zone, date, number, and time formats are controlled by the "Language & Region" set in User Settings; they cannot be configured independently.
  • User defaults maintained in SAP S/4HANA (e.g., SU01) do not override the client-side formatting applied by SAP Build Work Zone.
  • UI5 applications launched from SAP Build Work Zone use the locale provided by Work Zone, not ABAP user-specific formatting settings.

Resolution

Date format customization depends on the launchpad used.

  • For applications accessed via the SAP S/4HANA Fiori Launchpad, refer to SAP KBA 3111668 - How to set up the date format or time format on Fiori Launchpad? - SAP S/4HANA Cloud Public Edition
  • For applications accessed via the SAP Build Work Zone, refer to SAP KBA 3310903 - How to change the date and number format for UI5 applications on SAP Build Work Zone, Standard Edition

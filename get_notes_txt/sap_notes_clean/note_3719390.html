Symptom

Following a system clone activity, the system hostname and IP address of the target system changed.
You are preparing your system rename and your saphostctrl command with the function PrepareDatabaseCopy does not create the relocate.template file 
The following error is received.

Command:

./saphostctrl -function PrepareDatabaseCopy -dbname <DBSID> -dbtype db6 -dbconfdir /tmp/sapdbctrl-config/ -dbinstance db2<dbsid> -copymethod Offline -dostatechange -dbuser <sapsid>adm -dbpass <password of <sapsid>adm user>

====================================================

Error: Command execution failed. : PrepareExecuteBrdb6brt failed. RelocateTemplate: /tmp/sapdbctrl-config//<DBSID>_NODE0000.scr (fault code: 127)

Environment

BC-DB-DB6
BC-INS-SWPM
BC-INS-SRN 

Reproducing the Issue

  • Perform a system rename with SWPM 
  • In the Define Parameters phase of the Database Configuration Information screen;
  • Choose the directory that should contain the configuration files for the DB6 database rename procedure;
  • Click 'Next';
  • The saphostctrl with function PrepareDatabaseCopy and the executable returns with the following error:
        Error: Command execution failed. : PrepareExecuteBrdb6brt failed. RelocateTemplate: /tmp/sapdbctrl-config//<DBSID>_NODE0000.scr (fault code: 127)
        Operation ID: 
  • The following SAP note 1619720 - System Rename for SAP Systems based on SAP NetWeaver - Using Software Provisioning Manager 1.0 has been checked for
        Error: Command execution failed. : PrepareExecuteBrdb6brt failed. RelocateTemplate: /tmp/sapdbctrl-config//<DBSID>_NODE0000.scr (fault code: 127)
        Sections
        Unix only: Error: Command execution failed. : PrepareExecuteBrdb6brt failed. RelocateTemplate: ... (fault code: 127)
        Windows only: Error: Command execution failed. : PrepareExecuteBrdb6brt failed. RelocateTemplate:  (fault code: 127)
  • The issue is not permissions as per the above note.
  • In the /tmp/sapdbctrl-config directory the following files are present
        db2<dbsid>.cfg
        db2<dbsid>.net
  • The following file is missing.
        relocate.template

 

Cause

The relocate template is missing from the /tmp/sapdbctrl-config directory as it has not yet been created.










Resolution

  1. Create the relocate.template as per  SAP note 1657141 - DB6 - Create relocate.template for system rename 
  2. Check /tmp/sapdbctrl-config directory contains the following files.
    The following files should now be present. 
    db2<dbsid>.cfg
    db2<dbsid>.net
    relocate.template 
  3. Re-run SWPM - PrepareDatabaseCopy to complete this phase.

See Also

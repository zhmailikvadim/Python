Symptom

  • Errorlog contains message, "Current process infected with signal 11 (SIGSEGV)", in the module 'xact_rollback'
  • Followed by stack trace which includes the module 'set_repdb_off' 
  • Client process unexpectedly terminated

Environment

  • SAP Adaptive Server Enterprise (ASE) 16.0
  • SP_REPTOSTANDBY <database-name>, 'NONE' was executed on an active system and became victim of a deadlock.

Cause

This issue has been identified as CR 830117.  Its description reads "The message, "Current process infected with signal 11 (SIGSEGV)", in the module 'xact_rollback' together with a stack trace which includes the module 'set_repdb_off' may be reported in the SAP ASE error log when the command SP_REPTOSTANDBY <database-name>, 'NONE' is executed on an active system and is victim of a deadlock."

Resolution

CR 830117 has been fixed.  It will be implemented in:

  • SAP Adaptive Server Enterprise (ASE) 16.0 SP04 PL07
  • SAP Adaptive Server Enterprise (ASE) 16.1 SP00 PL01

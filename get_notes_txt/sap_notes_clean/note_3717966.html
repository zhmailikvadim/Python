Symptom

After upgrading the MDG support package, editing MDG fields and then submitting the change request (CR) causes the CR to hang and triggers a short dump: runtime error MESSAGE_TYPE_X in CL_MDG_MDF_FIELDINFO_LOCAL=>CONSTRUCTOR.

Other Terms

 CL_MDG_MDF_FIELDINFO_LOCAL, MESSAGE_TYPE_X, MDF role adjustments, MDG AI cases, MDG_MDF_GEN_AREAID, areaid, MDG_MDF_ROLES

Reason and Prerequisites

New MDG fields were introduced via background processing. The automatic post-processing to regenerate and synchronize field metadata may not have been triggered or may not have completed as expected.

Prerequisites:
SAP note 3461293 - Not required check in report MDG_MDF_GEN_AREAID

Solution

 Execute following steps:

1. Go to SE38 
2. Run report MDG_MDF_GEN_AREAID
3. Select parameters: application = 30 Master Data Governance Framework and application area = USMD and execute.
5. Run the report for all available MDG data models in your system and selected application = 30 Master Data Governance Framework

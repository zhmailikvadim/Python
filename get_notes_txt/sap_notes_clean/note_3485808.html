Symptom

When maintaining a role in transaction PFCG for ICSM Release 2406, you enter the relevant ICSM transaction(s) in the role menu. After merging the authorization data, you discover that some objects are showing incorrect default authorizations, even though the appropriate authorizations are necessary to execute these transaction(s).

Other Terms

ISLS, SU24, RSU22UPLD

Reason and Prerequisites

The required authorization defaults were not delivered by ICSM for Release 2406.

Solution

You can upload the authorization default values from the files SU22_ERR_20240916_DELIVER and SU22_ERR_20240916_MAKE, which are attached to this note. To do this, use the "Upload" function in the application toolbar of transaction SU24, as outlined in Notes 198598 and 1539556.

Symptom

  • Prompt Descriptions of the List of Values are not retained in the Prompt window, current value only shows Value in Crystal Reports designer.
  • Issue only happens on current value when refresh report again.
  • Description would display again if re-select from Picklist or restore default value. 

Other Terms

SAP Variable, SAP BW (DSL), Description, Current Value.

Reason and Prerequisites

Environment

  • SAP Crystal Reports 2025 Patch 8
  • SAP BW (DSL)

Steps to Reproduce

  1. Create a crystal report based on BEx Query with Parameters defined at BW side.
  2. Refresh the report.
  3. When prompted select any values for the parameter.
  4. Observe that both the Values and Descriptions are visible in the Prompt window.
  5. Refresh the report again with 'Prompt for new parameter values'.
  6. Observe that the previously entered parameters are only showing up as Values (no Descriptions are visible).

Cause

  • This is a known limitation.

Solution

Workaround:

This issue is not fixed. User has to reload the description from picklist again if need to check description:

1. For Single or Multiple values, user could click "OK" direct after picklist dialog prompted.

2. For Hierarchy node value, user needs to expand the hierarchy to show the selected node and then click "OK".

3. For Range value, description could not be reloaded. User needs to reselect from Picklist again.

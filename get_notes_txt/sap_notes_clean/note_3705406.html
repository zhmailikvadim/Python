Symptom

After a mandate's first use, the XML generation for RCUR transactions incorrectly adds 7 days to the posting date instead of 2.

Environment

SAP S/4HANA Public Cloud Edition

Reproducing the Issue

  1. Utilize a mandate with CORE and FRST.
  2. Generate the XML file and observe that the value date is set by adding 5 days to the posting date.
  3. Utilize the same mandate again with RCUR.
  4. Generate the XML file and observe that the value date is set by adding 7 days to the posting date instead of 2.

Cause

The value date is different as expected.

For an RCUR collection, the lead time for submitting a SEPA Core Direct Debit is two days for subsequent use. However, the DMEE calculates the Required Collection Date as Due Date + 7 days, which is correct. This is because any collections due before the FRST collection must take place after that initial collection has been completed.

Resolution

Refer to the SAP community post SEPA Pre-notification and SDD Timelines Functionality for more insights into the SDD timelines logic.

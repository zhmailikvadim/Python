Symptom

In Mail Communication channel observed the below error while DELETION operation:

Exception caught during processing mail message [0]com.microsoft.graph.http.GraphServiceException: Error code: ErrorQuotaExceededOnDelete
Error message: Cannot delete message because the destination folder is out of quota.

DELETE https://graph.microsoft.com:443/...<Exchange URL>
SdkVersion : graph-java/<version>

403 : 
[...]

Environment

  • SAP NetWeaver 
  • Process Integration

Cause

  • The destination mailbox folder (for example, Deleted Items) has reached its storage quota as enforced by microsoft 365/exchange via microsoft graph.
  • 403 Forbidden may also occur due to missing or insufficient permissions for the microsoft graph application used for OAuth.

Resolution

The error message "Cannot delete message because the destination folder is out of quota" indicates that the folder you are trying to delete messages from has reached its storage limit. This error is returned directly by Microsoft Graph API from your Microsoft 365/Azure backend , indicating that the mailbox or destination folder has reached its quota limit. Because this limitation is enforced at the tenant level, it is not caused by the SAP Mail Adapter, you need to check the issue with your Azure/ Microsoft Team.

  1. Verify with the Azure administrator whether the mailbox and the destination folder (such as Deleted Items) have exceeded quota; clean up items or increase the folder/mailbox quota.
  2. If 403 persists, validate and assign the required microsoft graph permissions for the OAuth application as outlined in the referenced KBA 3492888, then retest.

For your reference, Microsoft has documented this exact behavior when mailbox/folder quotas are exceeded:

Microsoft Support: Cannot delete items in a public folder due to quota issues
https://learn.microsoft.com/en-us/troubleshoot/exchange/public-folders/cannot-delete-items-public-folder
Microsoft Q&A: ErrorQuotaExceeded returned by Microsoft Graph API
https://learn.microsoft.com/en-us/answers/questions/2169662/errorquotaexceeded-error-microsoft-graph

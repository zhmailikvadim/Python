Symptom

When an end-user creates a Goods Movement SAP system stores and displays the corresponding material document using a time zone which doesn't consider the time zone that end-user is located. 

Environment

SAP S/4HANA Cloud Public Edition

Reproducing the Issue

For example: (it also happens for any kind of material document generated):

  1. Create and release a Process Order;
  2. Confirm Process Order, checking date and time of entry;
  3. Display material document generated by the process order confirmation and notice that time of entry is not the same.

Cause

This is the standard system design for Material Documents.

For several applications it is required to know the exact system date and time when a document has been posted.

Material Documents system date and time cannot be displayed in the user's local time zone because replacing this information would disrupt other processes, the underlying logic cannot be customized, and SAP Standards does not support time zone conversion based on user account settings.

 

Resolution

To display Material Documents with the user's local date and time, use the "Material Document Overview" app by following these steps:

  1. Open the "Material Document Overview" app;
  2. Enter the material document number in the "Material Document" field;
  3. Tap the gear icon to access "Settings";
  4. Select "Posting Date" and "Created on" and press "OK".

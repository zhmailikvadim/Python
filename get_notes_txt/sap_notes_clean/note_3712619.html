Symptom

  • Does SAP have a Checklist for checking why Manage Launchpad Page does not allow to add apps to the pages?

Environment

S/4HANA

Resolution

----Check list------
 This checklist will help you systematically identify and resolve issues when you cannot add applications to pages using the Manage Launchpad Pages Fiori app. Work through each section step-by-step and check off completed items. 

QUICK START GUIDE: FIORI LAUNCHPAD & CATALOG SETUP

PHASE 1: USER PERMISSIONS

[ ]1.1 Check Roles (Trans: SU01) - Go to 'Roles' tab. Ensure Launchpad Admin roles exist (SAP_BR_ADMINISTRATOR, SAP_FLP_ADMIN) . - List Roles of your User: __________________________________________

[ ]1.1b For the Roles you have above, using SU01 and navigate to the Authorization Data.  Click on the Roles in the user maintenance, then display role details and navigate to Authorization Tab and look for Authorization Objects section.

Search for the following Authorization Object

Use search function to find specific object
Enter authorization object name (e.g., S_SERVICE)
Press search button

Verify Object Details

Double-click on authorization object 
Check field values in detail view
Compare with required values:

SRV_NAME = /UI2/PAGE_BUILDER_PERS
SRV_NAME = /UI2/INTEROP
SRV_NAME = /UI2/TRANSPORT

Table check:

===============================================================
| AUTH OBJECT  | FIELD     | REQUIRED VALUE         | PURPOSE                                                  
===============================================================
| S_SERVICE    | SRV_NAME  | /UI2/PAGE_BUILDER_PERS | Page personalization                     
|-----------------|--------------|--------------------------------|-------------------------
| S_SERVICE    | SRV_NAME  | /UI2/INTEROP                 | FLP Interoperability                      
|-----------------|--------------|--------------------------------|------------------------
| S_SERVICE    | SRV_NAME  | /UI2/TRANSPORT            | Transport functionality                  
|-----------------|--------------|--------------------------------|--------------------------

[ ] 1.2 Verify Catalog: SAP_CORE_BC_UI_FLD and SAP_CORE_BC_UI - Are these catalog assigned to you or assigned in the Launchpad Admin Role above? [ ] Yes [ ] No

[ ] 1.3 Test Access - Can you open "Manage Launchpad Spaces" app? [ ] Yes [ ] No

 

PHASE 2: CATALOG CONFIGURATION

[ ] 2.1 Content Manager (Trans: /UI2/FLPCM_CUST) - Check for "Outdated" or "Changed" statuses. - Problematic Catalogs: ________________________________

[ ] 2.2 App Assignments - Select your catalog. Are the apps you need listed? - Result: [ ] Found [ ] Missing

PHASE 3: SYSTEM SERVICES (OData & ICF)

[ ] 3.1 OData Services (Trans: /IWFND/MAINT_SERVICE) - Verify ACTIVE status for: - /UI2/INTEROP - /UI2/PAGE_BUILDER_PERS - /UI2/TRANSPORT - List any Inactive: ___________________________________

[ ] 3.2 Error Log (Trans: /IWFND/ERROR_LOG) - Check for recent red entries related to UI2 services.

[ ] 3.3 ICF Nodes (Trans: SICF) - Path: default_host/sap/bc/ui2/interop -> Must be ACTIVE.

PHASE 4: CACHE & BROWSER CLEANUP

[ ] 4.1 Browser Refresh - Press F12 > Console Tab (Check for Red Errors). - Clear Browser Cache & Restart Browser.

[ ] 4.2 System Cache (Trans: /UI2/INVALIDATE_GLOBAL_CACHES) - Execute and wait for completion.

[ ] 4.3 Chip Sync (Report: /UI2/CHIP_SYNCHRONIZE_CACHE) - Run in SE38 or via Transaction.

PHASE 5: FINAL VALIDATION

[ ] 5.1 Open "Manage Launchpad Pages"
[ ] 5.2 Edit a Page -> Go to "Catalogs" section.
[ ] 5.3 Drag an app to a section and click SAVE.

-----end checklist------

Based on the above checklist, once you fill it out with the answer, and if issue still occurs, the team needs to provide the necessary reproduction steps and Traces/HAR/Console Log files as explained here:  2505725 How to create HTTP trace in Fiori Launchpad using Google Chrome, Edge or Mozilla Firefox and create a new case.

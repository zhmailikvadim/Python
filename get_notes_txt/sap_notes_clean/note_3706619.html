Symptom

  • An error occurs during a client copy performed
  • The error message is: "column store error: search table error: [2724] Olap temporary data size exceeded 31/32 bit limit[...]"

Environment

SAP HANA, platform edition 2.0

Reproducing the Issue

  1. Perform a client copy from one client to another client
  2. Review the client copy logs
  3. Observe the error message similar to: "column store error: search table error: [2724] Olap temporary data size exceeded 31/32 bit limit[...]".

Cause

The OLAP engine can process a maximum number of 2^31 records. If more records are processed, the error "Olap temporary data size exceeded 31/32 bit limit" is thrown.

Resolution

Instead of using the OLAP engine the HEX engine can be used by applying a statement hint to use the HEX engine instead of the OLAP engine:

  1. In the HANA indexserver tracefile the error is logged. Find the entry "Olap temporary data size exceeded 31/32 bit limit" corresponding to the time when the copy process fails. The statement HASH is logged as well in the indexserver trace.  
  2. ALTER SYSTEM ADD STATEMENT HINT ( USE_HEX_PLAN ) COMMENT 'Hint for client copy' FOR STATEMENT HASH '<use_statement_hash_the_error_message_refers_to>';
  3. Restart the client copy process after applying the hint

See Also

Refer to KBA 2400006

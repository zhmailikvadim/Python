Symptom

1.When creating a material document with movement type 311 in transaction MIGO, the system automatically generates a second line item.

However, when you later change the material document using MIGO, specifically modifying fields such as Goods Recipient, Unloading Point, or Item Text for Item 1—those changes are not updated in the automatically generated Item 2 at the database (table) level.

 

2.The user is also unable to directly edit Item 2 in MIGO, as it is automatically generated by the system during a 311 movement

Environment

  • Materials Management (MM)
  • SAP ERP
  • SAP ERP Central Component
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA

Reproducing the Issue

1.Post a material document in MIGO using movement type 311.

2.Open the posted material document again in MIGO → Change mode.

3.In the Where tab of Item 1, modify fields such as:

Goods Recipient

Unloading Point

Item Text

 

4.Choose Post to save the changes.

5.Observe that the changes are applied only to Item 1, while Item 2 remains unchanged.

Cause

For movement type 311 (Transfer Posting – Storage Location to Storage Location), the system generates two independent line items:

One for the issuing storage location

One for the receiving storage location

The two lines are not linked for field‑level updates. Therefore, the system does not automatically propagate changes from Item 1 to Item 2.

Resolution

  1. Open MIGO and display the material document.

  2. Go to Settings → Default Settings.

  3. Enable “Display Automatic Line in Transfer Posting.”

  4. Once this option is enabled, view and select Line Item 2 in MIGO and edit it directly.

  5. Update fields such as Goods Recipient, Unloading Point, and Item Text individually for each line item.

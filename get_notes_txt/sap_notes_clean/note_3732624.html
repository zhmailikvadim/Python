Symptom

  • Performed all the steps in the documentation of Trusted Authentication, but when trying the Query String, it does not go through the BI LaunchPad login page.

Environment

  • SAP BusinessObjects Business Intelligence Platform 4.3/2025
  • Windows Server

Reproducing the Issue

  1. Enable the Query String as per KBA 3559381 - Configuring Trusted Authentication in 4.3 SP04 Patch 1000+ & BI 2025 - SAP for Me
  2. By trying to run the following link https://<HOST>:<PORT>/BOE/BI?user=<USERNAME> it does not go through the login page.
  3. No error appears in the logs.

Cause

The certificate that was added to the Cacerts.pem received a wrong line feeding (CR LF).

Resolution

  1. Open the Cacerts.pem file in Notepad.
  2. Remove the incorrect certificate and add it with a Linux Line Feeding (LF).
  3. Restart the Tomcat.
  4. Test the Query String.

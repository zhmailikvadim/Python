Symptom

  • Multiple tax jurisdiction values are automatically populated when creating a Business Partner using transaction BP for the country US.
  • Users are unable to determine the correct tax jurisdiction value to select from the multiple options provided.
  • This issue causes batch processing failures as the system cannot decide which tax jurisdiction value to use
  • Error TAX_TXJCD100 can also be raised.

Environment

  • General Functions Jurisdiction Code / Taxes.
  • SAP ERP
  • SAP S/4 Hana

Reproducing the Issue

  1. Create a Business Partner using transaction BP for the country US.
  2. Observe that multiple tax jurisdiction values are automatically populated.
  3. Attempt to select a tax jurisdiction value manually.
  4. Notice that batch processing fails due to the inability to determine a single tax jurisdiction value.

Cause

External tax engine returning multiple tax jurisdiction codes is always a possibility.  In case of online transactions such as transaction BP when this situation occurs the user can manually select one of the several possible tax jurisdiction codes through the selection pop-up, but this will cause an error in case of batch processing.

Resolution

 In the case of batch processing you need to customize the error message that is raised in batch process according to the SAP note 844365.

Symptom

It is not possible to trigger new hire tasks in a sequence. For example: Only when the buddy task is completed, the meeting task is generated.

Environment

SAP SuccessFactors Onboarding

Cause

Product behavior.

Resolution

  • Tasks appear in parallel by design.
  • This means that when the Primary new hire tasks stage is reached, all primary new hire tasks will trigger as per the onboarding program.
  • Workaround could be to separate the tasks by primary and secondary, having the secondary tasks trigger after the primary tasks in the process variant.

  • Enhancement requests can be submitted following KBA 2090228.

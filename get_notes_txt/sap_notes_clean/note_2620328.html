Symptom

An INVOIC SIDOC (FI or MRM) contains attachments and an error occurs within the attachment processing.

If the SIDOC is reprocessed with YDR3 (Posting visible or invisible) after correcting the error, the attachments are stored correctly, but no invoice  document is created.

Other Terms

C2SEDI, INVOIC IN Switch, INVOIC IN FI, INVOIC IN MRM, C2S8000032686, C2S 8000032686, Scanning Solution

Reason and Prerequisites

Program error reported with ticket 8000032686.

Posting visible and invisible is done with a CALL TRANSACTION. If the attachments are stored correctly, a COMMIT WORK is done and the called transaction is stopped.

Solution

Implement the correction instruction.

The attachments are now processed before the CALL TRANSACTION. The COMMIT WORK that leads to the canellation will be avoided.

 

The solution consists of the following notes:

2620328 - YDR3: INVOIC IN FI SIDOCs with status 51 cannot be reprocessed if the attachments are not archived correctly before
2620427 - INVOIC IN Switch: New function module to encapsule PO reference determination functionality

Symptom

  • In the Fiori app "Create Billing Documents", the billing document preview is read-only and the 'Add Text' option is disabled.
  • Text IDs cannot be added or edited before saving the billing document.
  • To add or change texts, the user must first save the billing document and then edit it in Change Billing Document (VF02).
  • No error messages are displayed.

Environment

•Sales And Distribution (SD)
•Logistics Execution (LE)
•SAP R/3 •SAP R/3 Enterprise 4.7
•SAP ERP Central Component
•SAP ERP
•SAP Enhancement package for SAP ERP
•SAP Enhancement package for SAP ERP, version for SAP HANA
•SAP S/4HANA
•S/4HANA Cloud Private Edition

Reproducing the Issue

  1. Open the Fiori app "Create Billing Documents".
  2. Create a billing document from a logistics or service flow.
  3. In the preview, attempt to add or maintain Text IDs before saving.
  4. Observe that the 'Add Text' option is disabled and texts cannot be edited in the preview.

Cause

The Fiori app "Create Billing Documents" invokes "Manage Billing Documents" app to provide a non-editable preview, where most fields, including Text, are not changeable.
This is a system limitation by design; text maintenance before saving is not supported in the preview.

Resolution

  1. Save the billing document in the Create Billing Documents app.
  2. Open the billing document in Change Billing Document (VF02).
  3. Maintain the required Text IDs and save the changes.
  4. If this capability is required in the preview step, submit an improvement request via the Continuous Influence program (refer to the link in See also).

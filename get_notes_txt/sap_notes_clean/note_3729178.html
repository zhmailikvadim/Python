Symptom

It is expected to retrieve the change log for the associated key fields (Customer, Material) of listing/exclusion condition records from the CDPOS table. However, they are not recorded in CDPOS.

Environment

  • Sales And Distribution (SD)
  • SAP ERP Central Component
  • SAP ERP
  • SAP Enhancement package for SAP ERP
  • SAP Enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA
  • SAP S/4HANA Cloud Private Edition

Reproducing the Issue

  1. Maintain a Listing/Exclusion condition record using transaction VB01:
  • Condition Type: B001
  • Access Sequence: B001
  • Condition Table: KOTG001 (Customer / Material)
  • Enter Customer, Material, Valid From, Valid To
  • Save the record

     2.Modify an existing record using VB02:

  • Add a material
  • Save

     3.Verify that change documents are generated:

  • Check CDHDR → Entry is created
  • Check CDPOS → Field-level changes are recorded

     4.Review the content of CDPOS:

  • Only validity date fields are recorded.

Cause

In standard, change history are not activated for listing/exclusion functionality, as described in Note 842773. Therefore, it is not possible to track changes made to listing/exclusion records.

With the introduction of Note 2921923, customers can activate change history for listing/exclusion condition records. Please note that this requires release 617 or higher.

Resolution

Implement SAP Note 2921923 in the system. Then, retrieve the associated key fields (Customer, Material) from the KOTG_HCHG table as described in this note. 

Keywords

listing, exclusion, condition records, CDHDR, CDPOS, KOTG001, validity date, DATAB, DATBI, KOTG_HCHG

Symptom

  • IBP datastore connection test fails with error
  • Error message: "Test failed for the default configuration 'xxx' of datastore: 'xxx': Cannot connect to SAP Applications datastore <> using host <xxx>, client <100>, user <>, and system number <01: CRFC error: RFC_LOGON_FAILURE- No suitable SAP user found for X.509-client certificate[SAP Partner]>. Please make sure the SAP server is running and the login information is correct."

Environment

SAP Cloud Integration for data services

Reproducing the Issue

  1. There is a IBP datastore configuring "Certificate" as Authentication type
  2. When testing connection for the datastore, it has error in above symptom 

Cause

Renewed certificate is not uploaded to the communication user used in the datastore.

Resolution

Refer Defining the Communication Arrangement | SAP Help Portal to import the new certificate to the communication user. 

See Also

  • Defining the Communication Arrangement | SAP Help Portal
  • KBA 3152253 - Certificate-Based Authentication Configuration Steps for WebSocket RFC - SAP for Me - the part of IBP Fiori Application
    • There is a Note: This step of uploading the certificate to the Communication User must be performed again in the future, whenever the client certificate is renewed to extend its validity.

Symptom

Product was partially received and a batch subitem is created. The main item will show as "Not Relevant" for Goods Receipt and the created subitem will be "Completed".

Environment

  • SAP S/4HANA
  • SAP Extended Warehouse Management

Reproducing the Issue

  1. Create a batch subitem for an inbound delivery line item with partial quantity
  2. Post goods receipt and perform putaway for the subitem

Cause

Standard behavior

Resolution

  1. If the remaining quantity belongs to the same batch as an existing batch subitem: increase the quantity of that batch subitem using the appropriate process codes in the inbound delivery
  2. If the remaining quantity belongs to a different batch: create a new batch subitem by selecting the main delivery item, choose Batches > Create batch subitem, and enter the correct batch number and remaining quantity
  3. After adjusting or creating the batch subitem, proceed with goods receipt and putaway for the remaining quantity

See Also

KBA 0003639784 - Batch Split Scenarios in Outbound Delivery Processing

Keywords

PDI, IBD, inbound delivery order, batch subitem, batch split, partially completed.

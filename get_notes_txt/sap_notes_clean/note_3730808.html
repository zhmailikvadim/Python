Symptom

In SAP S/4HANA, you are experiencing data inconsistencies in the table ITMF_D_MD_HIST. For certain material records, the "from date" value is later than the corresponding "to date" value, which is logically incorrect.

Environment

  • SAP ERP
  • SAP S/4HANA
  • SAP S/4HANA Cloud Private Edition
  • SAP Tax Declaration Framework for Brazil
  • SAP Document and Reporting Compliance service

Cause

This data inconsistency is due to a known product issue where changes to material master data are not recorded with the correct date ranges in the ITMF_D_MD_HIST table.

Resolution

To correct this issue, you need to perform a two-step process: implement a series of SAP Notes and then run a correction report.

  1. Implement SAP Notes:
    For system releases prior to S4CORE 102, the following SAP Notes must be implemented in the exact order listed:

    • 2489490
    • 2518928
    • 2542866
    • 2552235
    • 2594928
    • 2615417
    • 2651834
    • 2670598
    • 2704834

  2. Execute Correction Report:
    After all the above SAP Notes have been successfully implemented, execute the program ITMF_MD_HIST_FIX_MATERIAL_DATA. This will correct the existing inconsistent "from date" and "to date" entries in the table.

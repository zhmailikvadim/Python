Symptom

I need to set up a new SSO configuration or update my existing Identity Provider (IdP) details, and I want to avoid manual entry errors when inputting long Entity IDs, Login URLs, and Logout URLs. I have a standard SAML metadata file from my IT team and want to use it to automatically populate the required fields in the SAP Fieldglass configuration wizard.

Cause

Manual entry of SSO parameters is prone to typos, which results in "UnSupported SSO protocol" or "Invalid IdP URL" errors. Using the Metadata Import tool ensures that the technical endpoints and unique identifiers in SAP Fieldglass perfectly match the values generated by your Identity Provider.

Resolution

Configuration Managers can use this tool during the first step of the SSO Setup Wizard:

  1. Log in as a Configuration Manager and select the Single Sign-On tile from the dashboard.
  2. Select Edit (for existing clients) or the tile will open the wizard automatically for new clients.
  3. On the Identity Provider Details page, locate the Identity Provider Metadata Import section.
  4. Select the Identity Provider Metadata Import tool.
  5. Upload the SAML-standard metadata XML file provided by your IdP (e.g., Azure, Okta, ADFS).
  6. The system will automatically extract and prepopulate the following fields:
    1. Identity Provider Entity ID (Issuer)
    2. Identity Provider Login URL
    3. Identity Provider Logout URL
    4. SAML Response Signing Certificate
  7. After the upload, the system auto-refreshes the page to display the updated information.
  8. Review the populated fields to ensure they align with your technical requirements.
  9. Choose Next Step to proceed to the Service Provider configuration.

Note:

  • Manual Overrides: Even after importing, you can still manually edit fields if your specific environment requires a custom URL or identifier.
  • Certificate Handling: The import tool will attempt to load the signing certificate directly from the metadata. If multiple certificates are present, ensure the correct one is active in the SAML Response Signing Certificates section.
  • Format Requirement: The file must be a valid SAML-standard XML metadata file; otherwise, the import will fail.

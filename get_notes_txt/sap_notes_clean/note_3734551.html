Symptom

When creating a billing document referring a sales document, compared with the condition value in sales document, the condition value in billing document is reversed (Sign reversed).

Environment

SAP S/4HANA Cloud Public Edition

Cause

A possible cause is the pricing type in the related copying control is set to F: No redetermination; turn and fix condition values, which means:

  • During copying, the condition values are adjusted based on the scaling of the quantity; using the ratio between the quantity of the subsequent and the preceding document.
  • The condition values are multiplied with -1.
  • After this adjustment, condition values are fixed and not updated again, even if further changes occur.

Resolution

Modifying the pricing type for related copying control from F to a suitable pricing type that does not invert condition signs (according to the business requirement) in the configuration activity "Define Copying Control for Sales Document to Billing Document".

Keywords

Pricing Sign Reversal, Gross value, Copying Control, Pricing Type F, Condition Values Multiplied By -1, SAP S/4HANA Cloud Public Edition.

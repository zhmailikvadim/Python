Symptom

While using the Fiori app – Manage Payments, the application terminates with an error.
The issue results in a short dump, which can be observed in transaction ST22.

The short dump indicates a syntax error occurring during the execution of the Manage Payments app.
Relevant ST22 dump details have been attached for further analysis.

Environment

SAP S/4HANA 2023 

  • S4CORE 106 

  • S4CORE 107 

  • S4CORE 108 

Reproducing the Issue

  • Log on to the SAP S/4HANA system.

  • Launch the Fiori app – Manage Payments.

  • Search for or select an existing payment.

  • Perform the relevant action in the app (for example, open or process the payment).

  • The application terminates with an error.

  • A short dump is generated, which can be verified in transaction ST22 (syntax error).

Cause

  • The issue occurs while using the Fiori app – Manage Payments.

  • The application triggers backend processing in ABAP class /PF1/CL_BCM_INTEGRATION.

  • A syntax error (SYNTAX_ERROR) is raised at runtime.

  • The dump is caused by an unknown data type /PF1/RMC_BCQN_APL_TYPES.

  • This indicates a missing or inconsistent SAP correction (SAP Note / SP level mismatch).

  • As a result, the OData request fails and the app terminates with an error.

Resolution

  • Implement SAP Note 3411297 in the affected system.

  • After implementing the note, re-test the scenario using the Manage Payments Fiori app to verify whether the issue is resolved.

Symptom

After execution employee synchronization, there is SLG1 error: No company is assigned (message number: SUID01 012) of the specific employee

Environment

S/4 HANA OP with HR integration active status

Reproducing the Issue

  1. Assignment a user to the specific employee via example PA30 or other action steps, in order to create the master data in table PA0105 subtype 0001.
  2. Execution report /SHCM/RH_SYNC_BUPA_EMPL_SINGLE of the specific employee
  3. There is SLG1 error: No company is assigned (message number: SUID01 012) 

Cause

During employee synchronization to BP, it is possible to convert a classic user into a business user, according to note 2623813 - User/Business Partner Integration and SAP HCM in S/4 Hana.
However, HCM doesn't have any information about the Company/Location. There is no company address master data from source HR employee master data as design.

According to standard action steps, before conversion a business user, the user is required to create in user module firstly with relevant company address master data.
Otherwise, the user will miss the company address assignment.

When the classic user is missing company address assignment in user module itself, you may find the error of the specific user ID via transaction SU01 as edit mode directly.



Resolution

  1. Please create the company address assignment of the classic user, according the consulting guide information in KBA 3588941 - Company Address is not assigned to BP user in SU01 
  2. Then you can re-synchronize the example employee

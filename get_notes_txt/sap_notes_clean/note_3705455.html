Symptom

Upon executing Result Analysis for WBS element via KKA2, planned costs are appearing in the output despite there are no planned costs available for WBS element.

Environment

  • Product Costing (COPC)
  • SAP ERP 
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA
  • SAP S/4HANA Finance

Reproducing the Issue

  1. Execute Result Analysis for the WBS element.
  2. Observe that planned costs appear in KKA2 despite no planned costs being present at the WBS level.

Cause

The planned costs are calculated inside the user exit EXIT_SAPLKKAG_002 and populated in KKA2.

Resolution

  1. Deactivate the user exit EXIT_SAPLKKAG_002.
    OR 
  2. Do not calculate planed costs if the user exit EXIT_SAPLKKAG_002 is called via KKA1/KKA2/KKA3. This will create planned cost during result analysis.

Keywords

KKA1, KKA2, KKA3, Planned costs, Result Analysis, WBS Element, User-exit, EXIT_SAPLKKAG_002

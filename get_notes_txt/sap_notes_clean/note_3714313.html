Symptom

For EZ on requesting side, there is no allowance to correct an incorrectly assigned PO on an invoice before a supplier invoice is created by enabling manual PO reassignment when the Hub backend status is Ready for Posting. This note allows user to set EZ on requesting side to set EZ on hold from ready for posting and allows PO reassignment.

Other Terms

G-Invoicing EZ Requesting side PO Assignment Hold

Reason and Prerequisites

 Program error

Solution

Implement this note or upgrade to a corresponding support package.

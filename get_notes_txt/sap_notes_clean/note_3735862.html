Symptom

  • When attempting to create folders at the SFTP root “/” fails.
  • An error message is displayed: "Access denied".

Environment

  • SAP Agent Performance Management
  • Agent Performance Management

Reproducing the Issue

  1. Connect to the SFTP server with a user account.
  2. Navigate to the SFTP root “/”.
  3. Attempt to create a new folder (for example, /UAT) at the root level.
  4. Observe that folder creation fails with the error "Access denied".

Cause

The creation of directories at the SFTP root-level folder is not supported.

Resolution

Considering the creation of directories at the SFTP root-level folder is not supported, it's possible to follow the steps below as an alternative:

  1. Identify a writable top-level directory where the user has permissions (for example, an existing environment directory such as /TEST or /PROD).
  2. Under the chosen directory, create parent folders for each environment needed (UAT, PERF, SIT).
  3. Within each environment folder, create subfolders.
  4. Update any jobs or integrations to reference the new subfolder paths.

Symptom

In a newly created table row, there is a field for users to enter a numeric value such as a price or an amount. Since the field is not automatically selected, the user entered value may be appended to the existing value in the field, which causes wrong value being saved.

Other Terms

Auto select, Numeric, Amount, Price, Cellerator, Table.

Reason and Prerequisites

There is a code error that needs to be fixed.

Solution

Please implement the attached Correction Instructions or the mentioned Support Package.

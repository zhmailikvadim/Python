Symptom

According to the official SAP Help documentation, when attempting to configure a Virtual Host or Custom Domain, calling the service URL https://<url-from-service-key>/apiportal/operations/1.0/Configuration.svc/VirtualHostRequests
results in an HTTP 403 (Forbidden) response with the error message in a HTML Page:

"You are almost there, but seem to have missed a step! Please assign the APIPortal.Administrator role to yourself and log in to the API portal application again."

Environment

  • SAP API Management
  • SAP Integration Suite

Cause

The OAuth token that was generated has a different role than the expected "APIManagement.SelfService.Administrator" that is described in here: Accessing API Management APIs Programmatically

Resolution

Please make sure that service key that is used for OAuth token generation is created with:

  • API Management, API portal service
  • apiportal-apiaccess plan
  • APIManagement.SelfService.Administrator roles

For further details and step-by-step guide for the creation please check SAP Help Page: Accessing API Management APIs Programmatically

Symptom

In the Adjustment Hub Reporting app, users lack predefined time period filters (weekly, monthly, quarterly, half-yearly, yearly) that automatically calculate and apply posting date ranges. Manual date entry is required, reducing efficiency.

Other Terms

Adjustment Hub reporting filters; Fiori time period selectors; posting date auto-calculation; periodic reporting filters; SAP S/4HANA adjustment analytics.

Reason and Prerequisites

The standard Smart Filter Bar in the Adjustment Hub Reporting app (Fiori elements-based) does not include out-of-the-box periodic filters with dynamic posting date computation. This requires UI adaptation to add custom filter controls using SAPUI5 Flexibility services or controller extensions.

  • Fiori Launchpad Designer (/UI2/FLPD_CUST) permissions.
  • Transport request for UI changes.
  • UI adaptation mode enabled on the Adjustment Hub Reporting app.

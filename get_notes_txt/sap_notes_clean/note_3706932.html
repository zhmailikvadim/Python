Symptom

 During PMP “Items” parallel processing, you expect the system to process items correctly without errors. However, update termination messages occur on the TS table, and the PMP selection screen displays “Process functions” twice. This indicates that duplicate process functions are being executed, causing parallel processes to attempt simultaneous updates on the same TS records.

This note corrects the behavior by ensuring that process functions are defined and executed only once, allowing parallel processing to run correctly without TS table update terminations.

Other Terms

/IRM/IPCLBM, /IRM/LIPCLBMF0B, Update Termination, Background Job, DBSQL_DUPLICATE_KEY_ERROR, ABAP_PROGRAMMING_ERROR, CX_SY_OPEN_SQL_DB

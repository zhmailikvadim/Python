Symptom

  • SFTP scheduling failure with error message displayed: "[connection error. [Error sending the file to the server.]: [CrystalEnterprise.Sftp]]".
  • The following error appears on the AJS trace:
    "destination_ftp: exception caught while creating destination dir [] on ftp server []. Details: [Error stringres ID [41313]. Error creating directory [//] on SFTP server. SFTP Code: 3]."
    "destination_ftp: exception caught while putting source file [] to ftp server [] as destination file []. Details: [Error stringres ID [41311]. Could not create the sftpHandle for file [] to write to sftp server. SFTP Code: 2]."

Environment

SAP BusinessObjects Business Intelligence (BI) Platform 4.x/2025

Reproducing the Issue

  1. Attempt to schedule Webi reports to write to the SFTP location.
  2. Observe that all schedules fail consistently with the error message: "[connection error. [Error sending the file to the server.]: [CrystalEnterprise.Sftp]]".

Cause

  • Permission Denied (Code 3): The server returned a "Permission Denied" or "General Failure" error when attempting to create the directory].
  • Path Not Found (Code 2): The system could not create the folders, resulting in a missing path when attempting to transfer the file to the specified location.

Resolution

Address the "Permission Denied (Code 3)" error by ensuring the user account has the "Create Directory" (MKD) or appropriate permissions at the root level of the server.

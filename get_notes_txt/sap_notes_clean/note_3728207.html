Symptom

When executing the custom code check with the variant "S4HANA_READINESS_2025", the observation "Syntactically incompatible change of existing functionality" is raised as a priority 1 issue.

Environment

  • SAP Enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA Cloud Private Edition

Reproducing the Issue

  1. Execute the custom code check with the variant "S4HANA_READINESS_2025".
  2. Observe the priority 1 issue raised with the message "Syntactically incompatible change of existing functionality".

Cause

The issue arises because all released APIs have been extended to accommodate material field length changes. Custom code that calls these APIs needs to be checked and adapted to ensure compatibility with the extended fields.

Resolution

  1. Review the custom code that interacts with the released APIs.
  2. Verify that the custom code is calling the APIs with the correct data and parameters.
  3. Refer to the following SAP Note for guidance:

2438131 - Material Number Field Length Extension: Code Adaptions for usages of released RFCs and BAPIs

"Due to the nature of the custom code scans they will only provide a worklist of call locations of such modules but cannot inspect if the relevant function parameters are used. For the same reason, the custom code scans also cannot verify the correct adaption of the call. This is the responsibility of the developer analyzing and adjusting the custom code. "

Symptom

  • An ALV report is used to display data and handle the standard event AFTER_USER_COMMAND for processing after a standard ALV function is executed (e.g., saving an ALV layout).
  • After upgrading to SAP S/4HANA 2023, the parameter E_SAVED is not being passed when the event AFTER_USER_COMMAND is raised.
  • As a result, the event handler does not process the event as expected.

Environment

SAP S/4HANA 2023

SAP_BASIS 758 SP03 

Reproducing the Issue

  1. Run transaction SA38.
  2. Execute the SAP-standard demo report BCALV_TEST_GRID_EVENTS.
  3. Check the checkbox AFTER_USER_COMMAND under Business Events.
  4. Execute.
  5. Save the ALV layout (open the Save layout... dialog and press, fill layout and name and press on Adopt).
  6. Close the first popup (Output).
  7. Review the data in the information message popup.
    Expected result: E_SAVED = X.
    Actual result: E_SAVED = (empty), although the ALV layout has been saved.

Resolution

The parameter E_SAVED is no longer provided by the ALV Grid in newer releases such as SAP BASIS 758. 

This change is intentional and documented in SAP Note 3716499 - ALV: Deprecation of E_SAVED Flag in AFTER_USER_COMMAND event for Grid

In earlier releases, the E_SAVED flag was set in some layout‑related dialog flows, but it was not consistently maintained across all layout scenarios.

With newer ALV layout dialogs used in BASIS 758, this flag is no longer supplied when the event AFTER_USER_COMMAND is raised.

As described in SAP Note 3716499, applications should now rely exclusively on the user command exported by the event.
When a layout is successfully saved, the event AFTER_USER_COMMAND is raised with E_UCOMM = '&SAVE'

This user‑command–based approach is consistent across all current SAP_BASIS releases and is the recommended and supported method for detecting a layout save operation.

For full details, refer to SAP Note 3716499.

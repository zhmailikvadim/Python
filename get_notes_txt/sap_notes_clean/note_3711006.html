Symptom

  • Destination bin is not determined as expected if the inbound delivery is referring to a production order.
  • The material is the By-Product of the order.
  • WT log shows the destination bin is determined from PSA Production Supply Area directly without putaway strategy search.

Environment

SAP S/4HANA Public Cloud

Reproducing the Issue

  1. Create a production order that includes a by-product(the quantity is set as negative at Component page)
  2. Link the material to the operation that has the work center with the PSA.
  3. Do confirmation for the order at APP ID CO11N, inbound delivery is created.
  4. Create putaway task, the destination bin is determined as PSA bin instead of following the putaway strategy.

Cause

The system determines the PSA bin directly as the destination bin when there is a control cycle created for the by-product. This behavior occurs because the by-product is treated as a raw material that needs to be ready for other production orders.

Resolution

  1. Delete the control cycle at APP Change Control Cycle for WM (APP ID LPK2) for the by-product material.
  2. Create a putaway task for the inbound delivery to verify the resolution.

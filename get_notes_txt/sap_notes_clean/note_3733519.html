Symptom

  • In ASE 16.1 updatease tries to run installhasvss script
  • But 'enable HA' is no longer available so this fails:

    /sybase/ase161SP00PL01/ASE-16_1/bin> updatease

    Server: ASE

    Running installmaster script...
    ...
    (output truncated for clarity)
    ...
    The instmsgs.ebf script has been successfully installed.

    Running installhasvss script ...
    Error fetching results for query: 'select value from master..sysconfigures where name = "enable HA"'.
    The installhasvss script did not install correctly.  You can install this script with ISQL.
    Updating SAP Adaptive Server Enterprise failed.

Environment

SAP Adaptive Server Enterprise (ASE) 16.1

Cause

  • From ASE 16.1, ASE_HA related files, settings and installer option have been removed, refer to
    SAP Note 3205602 - Deprecation of ASE HA option
  • However, the updatease utility still tries to run a script related to HA and this fails because the config parameter 'enable HA' is no longer present
  • This issue has been logged under CR 831900.

Resolution

  • The message can be ignored, the installhasvss script is not meant to be run anymore
  • This issue will be addressed in a future ASE release

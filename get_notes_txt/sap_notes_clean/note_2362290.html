Environment

  • SAP SQL Anywhere
  • Microsoft Windows 

Reproducing the Issue

  1. Use a SQL Anywhere environment %SQLANY16% in a configuration file "sample.cfg" to start a server
    # Configuration file for server Elora
-n Elora
-c 10M
"%SQLANY16%\demo.db"
  2. Run dbsrv16 @sample.cfg
  3. You see the error: Cannot access "%SQLANY16%\demo.db": File does not exist
     

Cause

  • Environment variables do not translate to their actual paths in configuration file
  • Therefore, SQL Anywhere is using path that does not exist
  • Additionally, the correct environment variable would have been %SQLANYSAMP16%
  • The version 16 DCX example shows an incorrect usage

Resolution

  • Since environment variables are not supported for configuration files, use direct path instead
  • The DCX example will be fixed (Version 17 DCX is correct)

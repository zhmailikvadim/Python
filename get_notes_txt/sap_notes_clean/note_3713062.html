Symptom

Despite implemented BADI HCMFAB_B_TSH_FUNCTIONAL and extended Maximum number of search help via the Method SET_MAX_SHLP_HITS, 

the maximum 500 Assignment items are visible in the dropdown list. 

 

Environment

HCM Fiori 2.0

Reproducing the Issue

Set the RV_SHLP_MAX_HITS more then 500 and use the dropdown of the Assignments in the My Timesheet V3 app.

Cause

The RV_SHLP_MAX_HITS is valid for the field search help and not for the Assignment dropdown selection.

From the SAPUI5 control point of view it is not possible to show more than 500 items in the dropdown.

Resolution

Our developers follow the UX team suggestion regarding the drop-down:
https://experience.sap.com/fiori-design-web/combo-box/

From a functional point of view it would be very difficult for the user to distinguish the assignment just by title from a list of 500 entries.
There are several ways the assignments can be handled.

  • Via the backend report REP_FIORI_CATS_ASSIGNMNTS, administrator could create them inactive and the user could set as active just the ones they really need.
  • Use the form entry instead of assignments in case the combination of fields are so huge that the user would need more than 500 assignments.

Symptom

During the parallel processing of inbound CIF queues for purchase orders, the system encounters a database deadlock dump, causing the queues to get stuck with a SYSFAIL status. The issue only happens during simultaneous updates and does not occur when the failed queues are processed sequentially one by one.

Reason and Prerequisites

IMPORTANT: A manual activity to create the lock object /ARMSUI/E_MATTSP in transaction SE11 is required prior to implementing the automatic correction instructions. Please refer to the Manual Activity section for detailed steps.

Solution

New configuration parameter SIM14 (Material Timestamp Lock) is introduced to prevent database deadlocks with the following values:

I - Individual Material Lock (Recommended).
G - Generic Table Lock.
Space - Locking logic disabled.

New configuration parameter SIM15 (Material Timestamp Update Interval) is introduced to reduce redundant database writes:
X - Enable 5-second minimum interval between updates.
Space - Update interval logic disabled (Recommended).

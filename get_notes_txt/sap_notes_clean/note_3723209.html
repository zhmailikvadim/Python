Symptom

  • Audit reports (Change/ Read reports) are not generating in DC22.
  • When attempting to generate Audit Reports (Change Audit or Read Audit), the report generation fails in the DC22 environment.

Environment

SAP SuccessFactors HCM Suite

Reproducing the Issue

  1. Log in to the SAP SuccessFactors instance hosted in DC22.
  2. Navigate to "Change Audits Reports"
  3. Select the desired audit report type (Change Audit or Read Audit).
  4. Create New Audit Report (Change / Read Audit )
  5. Define the required filters (for example: date range, user, entity, or other relevant criteria).
  6. Click Generate or Run Report.
  7. Observe that the report generation fails (for example, job failure status, error message, or no report output generated).

Cause

The audit services are currently disrupted in DC22 due to service degradation , impacting the generation of audit reports.

 
 
 

Resolution

NOTE: The issue has now been resolved. Audit services in DC22 (both Preview and Production) have been fully restored.

Audit report jobs have resumed normal processing, and both Change Audit and Read Audit reports can now be generated successfully.

  • Ongoing service degradation was observed for BTP-ALS in the Bahrain region (SF DC22), leading to customer impact related to Audit Report functionality for few days. Now issue is resolved

  • Our Internal team has restored the Audit services in DC 22 )both preview and production). 

    • In case any issues  related to Audit Report generation, please raise a support case under component LOD-SF-PLT-CHA.

Symptom

  • Unable to unconfirm a resource from a professional services project after the resource has completed their task.
  • System does not allow unconfirmation even when planned hours are adjusted to match actual hours.
  • Users are unable to prevent further timesheet entries for resources whose tasks on specific projects are completed.

Environment

SAP S/4HANA Cloud Public Edition

Reproducing the Issue

  1. Open the Professional Customer Projects app.
  2. Navigate to the Team tab.
  3. Select the Resources tab.
  4. Attempt to unconfirm a resource after their task is completed for a specific project.
  5. Observe that the system does not allow the resource to be unconfirmed.


Cause

  • Once a resource has posted actual time confirmations on a project work package, the system keeps the resource assignment confirmed to ensure consistency between planning and actual postings.
  • Even if the planned effort is adjusted to match the actual hours, the system does not allow the resource to be unconfirmed because the assignment is still required for historical time confirmations and project reporting.

Resolution

To prevent further time postings, maintain the resource assignment with an end date corresponding to the last working day on the project.
By setting the assignment end date, the resource remains linked to the project for historical postings, while preventing any further time recording for that work package.

In order to prevent a resource from posting additional time after their task is completed, please try to proceed as follows:

  1. Open the Plan Customer Projects app
  2. Navigate to project ID.
  3. Go to the Resource Planning tab.
  4. Select the relevant resource
  5. Adjust the planned effort distribution:
  6. Set planned hours to 0 for all future periods after the resource’s last working date
  7. Ensure that no remaining effort exists for that resource

Once no planned effort remains in future periods, the system will not allow further time postings for that resource on the project.

Symptom

The read_only status on tables is reverted after a restart of ASE and write operations are possible again.

Furthermore, the status read_only is not properly reflected in the output of sp_help <table_name>.

Other Terms

SAP Adaptive Server Enterprise, SAP ASE, ASE, read_only, sp_chgattribute, CR831569

Reason and Prerequisites

The read_only status in not properly persisted and therefore lost after a restart.

Solution

The problem is reported in CR 831569.

A fix will be available with the following versions of SAP ASE:

  • 16.0 SP04 PL09
  • 16.1 SP00 PL03
  • 16.1 SP01 PL00

For more information on the 'read_only' attribute of a table see the online documentation .

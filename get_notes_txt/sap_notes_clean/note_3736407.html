Symptom

  • Identifier First flow shows only OTP option  after Communications screen-set registration
  • There is no "Register with password" option

Environment

  • SAP Customer Data Cloud
  • Customer Identity

Reproducing the Issue

  1. Register a new email using the Communications screen-set.
  2. Attempt to log in using the Identifier First screen-set.
  3. Observe that only the OTP option is offered.

Cause

  • When a guest account is created it does not have a password and it is not a registered user.
  • In order to complete the registration of a guest user the available option in Identifier First screen is OTP code verification.

Resolution

This is the expected behavior.  

When Identifier First follows the Communications flow (which creates the account with the identifier), the expected completion path is OTP verification, so "Register with password" is not displayed.

Please note that after user completes the registration via the OTP authentication, a password can be add via a Profile Update screen.

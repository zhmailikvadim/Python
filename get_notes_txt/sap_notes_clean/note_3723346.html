Symptom

  • After migration from ECC to S/4HANA, transaction FAGLL03 does not offer all the standard Dynamic Selection fields that were available in ECC transaction FBL3N.
  • Several master-data-related fields that could be selected and filtered via Dynamic Selections in FBL3N are not present in FAGLL03.

Environment

  • SAP R/3
  • SAP ERP
  • SAP ERP Central Component
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA

Reproducing the Issue

  1. Execute transaction FAGLL03 in S/4HANA.
  2. Open Dynamic Selections (button or Shift+F4).
  3. Navigate through the subareas and search for the expected master-data fields.
  4. Observe that certain fields that were available in ECC FBL3N are missing in FAGLL03.

Cause

  • In S/4HANA, FAGLL03 uses a different selection framework and data model compared to ECC FBL3N; not all fields are exposed by default in Dynamic Selections.
  • Dynamic Selections in FAGLL03 are grouped into “subareas,” each backed by specific DDIC structures. Only fields included in those structures are available for selection.
  • There is no standard switch to automatically restore the broader ECC FBL3N Dynamic Selection set after migration.
  • Per SAP design, additional fields must be introduced via enhancement as per stated on Note 945932.

Resolution

  • This is standard system behavior after the migration to S/4HANA.
  • To use those fields again in FAGLL03, they must be added via enhancement; there is no standard switch to bring them back automatically.
  • Follow the steps of SAP Note 945932 for detailed guidance.
  • Important: Only fields available in the underlying data model and supported by the selection logic should be appended.
  • There is no customizing toggle to replicate the ECC FBL3N Dynamic Selection scope; enhancement per Note 945932 is the supported option.

See Also

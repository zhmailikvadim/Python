Symptom

Using Apache reverse proxy with Kapsel application and SAP Mobile Platform 3.0.x.  When attempting to use AppUpdate plugin, URL has contains an extra 0 results in request failing with HTTP 404.

Environment

  • SAP Mobile Platform 3.0.x
  • Kapsel SDK
  • AppUpdate Plugin
  • Apache reverse proxy

Reproducing the Issue

  1. Develope Kapsel Application that uses AppUpdate plugin
  2. Configure Apache reverse proxy for Kapsel Application to SAP Mobile Platform
  3. Attempt to run application using AppUpdate plugin and plugin generates URL with '0' in suffix, example: https://<internet hostname of reverseproxy>:443/app/0/odata/lcm/v1/ListApps?sap-language=en forwards to https://pdirmmo.match.local:8081/0/odata/lcm/v1/ListApps?sap-language=en

  4. For this incoming URL starting with '0' the SAP Mobile Platform host responds with HTTP 404 (not found).

Cause

The extra 0 on the AppUpdate plugin is generated for support of Relay Server FarmID.  Relay Server 16.5.3 and greater is supported with SAP Mobile Platform 3.0 SP06 and above (See SAP Note 1904213). 

Resolution

There are a couple of solutions for this issue:

  1. Add code in the application to check for "0" and replace it with an empty string for AppUpdate plugin.
  2. Modify the reverse proxy to change the directory path.  Example, if currently have "app" for the suffix, create a second directory for the farm ID, example "app2".  Then the generated path will be  ProxyServer:port/app/app2/...

Symptom

An incoming SAP Business One event triggers several scenario steps. The scenario steps are triggered not exactly once, but several times.

Reproducing the issue

Scenario:

  1. In the integration framework, choose Scenarios → Step Design and create a scenario step.
  2. To create the inbound phase, click Inbound and then the Channel button.
  3. Enter the following:
    1. In the Inbound Channel field, select the SAP Business One option.
    2. In the Process Mode filed, select Asynchronous
    3. In the Process Trigger field, select B1Event.
    4. In the Identification Method field, select B1Event.
    5. In the Identifier field, enter 2, 4, 17.
    6. Enter all other required fields and save the settings.
  4. Create a second scenario step.
  5. To create the inbound phase, click the Inbound and then the Channel button.
  6. Enter the following:
    1. In the Inbound Channel field, select the SAP Business One option.
    2. In the Process Mode field, select Asynchronous.
    3. In the Process Trigger field, select B1Event.
    4. In the Identification Method field, select B1Event.
    5. In the Identifier field, enter 4.
    6. Enter all other required fields and save your settings.
  7. To set up and activate the scenario package.
  8. To trigger a SAP Business One event, in SAP Business One, update an item master data record.

Expected Behavior:

The integration framework message log displays two messages: one is for the first scenario step, the other is for the second scenario step.

Actual Behavior:

The integration framework message log displays four messages: two are for the first scenario steps, the other two are for the second scenario step. 

Cause

This is a program error.

 

Solution

Workaround:

  1. Save the attached B1iF_EventDispatcher.zip to the machine where the integration framework is running.
  2. Log on to the integration framework, choose Maintenance → ZIP Import, select the zip file and choose Submit.

SAP intends to provide a patch or patches in order to solve the problem described. The section Reference to Related Notes below will list the specific patches once they become available. The corresponding Info file of the patches in SAP Service Marketplace will also show the SAP Note number. Be aware that these references can only be set at patch release date. SAP will deliver patches only for selected releases at its own discretion, based on the business impact and the complexity of the implementation.

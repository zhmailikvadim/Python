Symptom

You are using SAP Business Technology Platform (SAP BTP. URL: https://cockpit.btp.cloud.sap) and would like to push the JAR file as the app content with an extended startup timeout.

Environment

  • SAP Business Technology Platform (BTP)
  • Cloud Foundry (CF)

Reproducing the Issue

You have an app deployed to CF. The application however starts slowly.

Resolution

The cf push command deploys an app to SAP BTP CF environments. Use the cf push -t option to push the JAR file as the app content with an extended startup timeout.
For example:
cf push app_name -p target/app.jar -t 120

The -t 120 option sets the health check timeout to 120 seconds, giving the app extra time after staging to respond healthily before CF considers it failed (default is shorter, e.g., 60s).

NOTE: SAP does not recommend the addition of '-u none' flag, as this would disable any health checks and unhealthy applications wouldn't be restarted and remain unrecognized. For further detailed information, check: App manifest attribute reference

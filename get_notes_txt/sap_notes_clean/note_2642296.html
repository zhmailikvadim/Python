Symptom

  • Technical Catalog SAP_TC_*_*_BE_APPS is missing or has missing changes in the Fiori Launchpad designer
  • Already follow the note 2269272 - "Reference Lost" Error for FLP Tiles after applying new UI component version

Environment

SAP S/4HANA

Reproducing the Issue

  1. Open the Fiori Launchpad designer
  2. Search the technical catalog, it is missing

Cause

The RFC destination is not maintained correctly

Resolution

  1. Find the relevant Alias for the Catalog:
    1. Open the Fiori app library:https://fioriappslibrary.hana.ondemand.com/sap/fix/externalViewer/#/home
    2. Search the missing catalog name, for example: SAP_TC_ISU_DM_BE_APPS
    3. Select one of the apps and navigate to the configuration tab to find the system alias

  1. Setup or assure a SM59 destination exists for the alias:
    1. Run transaction code SM59 in gateway system
    2. Create an RFC destination called System Alias_RFC pointing to the backend system, in this example, it is S4ISU_RFC
    3. Check current user and do not specify a user in the Log on Tab
  2. Replicate the catalog:
    1. If the catalog is missing:
      1. Call transaction /UI2/APPDESC_GET on the frontend server
        1. In the catalog ID field, enter SAP_*_BC_*. This will select all technical catalogs and their comprised apps
        2. Choose Execute
      2. If this transaction doesn't exist, run the report: /UI2/GET_APP_DESCR_REMOTE_ALL in transaction code SE38
    2. On subsequent replications (for example: after upgrade or after SAP Note application), if tiles are missing from the catalog or have old configuration, use report /UI2/GET_APP_DESCR_REMOTE_DEV, and choose "full replication" mode. Provide the Catalog ID and the Alias found by the previous steps. 

  * Both for hub and embedded systems backend catalogs (*_BE_*) replication is required.

Symptom

  • NOTE_3591743 is executed according to the post-implementation steps of SAP Note 3592673.

  • When running the report in Test Mode, the error message "Delivering entries for table TVDIR is not approved yet" is logged twice.

Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.

Environment

SAP S/4HANA

Cause

  • NOTE_3591743 attempts to deliver entries to table TVDIR.

  • Delivery of table entries to TVDIR is technically not possible.

  • The required entries already exist in the system, either completely or partially.

Resolution

Although NOTE_3591743 cannot deliver changes to table TVDIR, the required entries already exist in the system.

NOTE_3591743 attempts to create the following entries in TVDIR:

Two possible scenarios exist:

Scenario A

  • The required entries already exist in table TVDIR.

  • The entries are identical to the records in up-to-date internal SAP systems (except the timestamps, which are irrelevant).

  • For views/tables FIUAV_RECON_REP1 and FIUAV_RECON_REP2, the entries match the values shown in the screenshot below.

Result:

  • All required changes are already present.

  • No further action is required.

  • The TVDIR error messages can be ignored.

  • Execution of the subsequent steps of NOTE_3591743 can be continued.

 

Scenario B

  • The required entries exist in table TVDIR.

  • A minor difference exists compared to the records in up-to-date internal SAP systems.

  • For FIUAV_RECON_REP2, the value 'X' is missing in the Checkbox column.

 

Resolution:

The missing change must be applied manually and transported to all relevant systems together with the other changes of NOTE_3591743..

  1. Run transaction SE54
  2. Enter table/view FIUAV_RECON_REP2 and choose "Create / Change".
  3. Select a workbench request
  4. From the transaction menu, navigate to Utilities > Table Maintenance Generator
  5. Change the recording routine to “No, or user, recording routine”. 
  6. Save the changes and confirm the warning “You are changing a function group which does not belong to you”.

 

Result:

  • After applying the change, the TVDIR error messages can be ignored.

  • Execution of the remaining steps of NOTE_3591743 can be continued.

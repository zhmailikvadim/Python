Symptom

You upgrade to SAP S/4Hana on Premise 2023 or 2025. The mass activation phase takes very long. Closer inspection of the log shows, that the duration of phase "Calculation of Dependencies" is similar to the "Object Activation" phase. Normally, the phase should be much faster. For example, there were incidents where phase "Calculation of Dependencies" took 8 hours. For a large activation with an "Object Activation" phase of 9 hours, the dependency calculation normally should not take longer than 3 hours.

When monitoring the phase in transaction SM50, you see class cl_dd_ma_reorg_dep_ddls_obj being executed for a long time.

Reason and Prerequisites

Missing indexes on some internal tables.

Solution

Apply this SAP Note or the related Support Package.

For upgrades: Use the latest SUM containing the current "fixbuffer". During the SPDD phase of ACT_UPG, transaction SNOTE can not be used to apply source code changes. The SAP Note could be applied manually in that phase. However, manual implementation is not recommended for this SAP Note. This SAP Note will be updated once the "fixbuffers" for S4Hana on Premise 2023 and 2025 are available.

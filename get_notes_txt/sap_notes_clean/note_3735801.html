Symptom

  • Trying to create a hierarchy (tree structure) in a custom RAP report using native annotations.
  • The report is exposed as an OData V4 service with UI binding.
  • There is a need to display the data as a hierarchical tree based on one field that connects the records.
  • When adding the hierarchy annotation, the following error occurs: "Error in entity 'ZXXX(CDS)': Hierarchy for view ZXXX is not supported".

Environment

SAP S/4HANA 2023

Reproducing the Issue

  1. Connect to the ABAP system in Eclipse.
  2. Create a simple RAP entity and add the annotation @OData.hierarchy.recursiveHierarchy: [{ entity.name: 'XXX'}].
  3. Model a simple hierarchy for the entity.
  4. Create a Service Definition that exposes the entity.
  5. Create an OData V4 Service Binding, activate, and publish the service.

Cause

Missing functionality.

Resolution

  1. Hierarchy support for RAP OData V4 services is not available in on‑premise S/4HANA 2023 and is not planned for to be down ported to it. Either:
    • Upgrade to S/4HANA 2025, where hierarchy functionality is available; OR
    • Alternatively, use the ABAP environment in SAP BTP (recent cloud releases) where hierarchy support is generally available for development and testing.
  2. If hierarchy is required immediately on on‑premise, postpone the use of @OData.hierarchy.recursiveHierarchy until the target release is available or develop on a supported cloud environment.

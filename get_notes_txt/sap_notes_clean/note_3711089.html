Symptom

The termination end date of a user is different in Personnel Administration and in the Business User view under the Application and User Management work center.

Example:
The user is terminated on DD1.MM.YYYY in Personnel Administration, but the Business User account shows an end date of DD2.MM.YYYY. 

Environment

SAP Business ByDesign

Reproducing the Issue

  1. Go to Personnel Administration.
  2. Search for the user.
  3. Open Employment Details and note the termination end date (DD1.MM.YYYY).
  4. Check the Change Log to see when and by whom the termination date was updated.
  5. Go to Application and User Management.
  6. Open Business Users and search for the same user.
  7. Open the Business User in Edit mode.
  8. Observe that the Validity End Date shows DD2.MM.YYYY, which differs from Personnel Administration.
  9. Check the Changes tab to review the update history of the Business User.

    Cause

    • The issue occurred because the Business User was manually inactivated before the employee termination date was updated. Once a Business User is inactive, the system restricts the Validity End Date to the user’s Last Logon Date and does not allow it to be extended to a later termination date.
    • This is Standard System behavior. 

    Resolution

    To verify the user Last Logon Date, kindly follow the steps below:

    1. Go to the Business Analytics Work Center.
    2. Click on Design Reports.
    3. Search for the report “User Logon Details”.
    4. Open the report.
    5. Enter the User ID in the selection criteria.
    6. Execute the report.
    7. Verify the Last Logon Date displayed for the user.

    If the Last Logon Date is DD2.MM.YYYY, then the Business User Valid To date reflecting DD2.MM.YYYY is working as designed.

Symptom

  • SAP-defined default audit policies differ between two HANA Cloud systems.

Environment

  • HANA Cloud Services HANA Cloud

Reproducing the Issue

  1. Compare the audit policies in the Two different database instances.
  2. Observe that the  SAP-defined default audit policies not the same.

Cause

  • The difference in audit policies between the systems is expected due to provisioning differences.
  • The tenant are provisioned with a newer SAP security baseline that included additional default audit policies, while existing tenants, do not automatically receive these newer baseline policies.
  • SAP may introduce baseline or security policy updates regionally in phased rollouts, resulting in differences between regions even when versions match.

Resolution

  1. The differences in audit policies between the database instances are normal and expected.
  2. Understand that SAP-defined default audit policies can vary based on provisioning timeframes and regional rollouts.
  3. No further action is required as these differences do not indicate any issue with the systems.

Keywords

Symptom

I want to include custom header fields in the SOAP message of my web services export tasks, but the default task output only includes standard header fields.

Resolution

  1. Identify the web services export task that needs custom headers.

  2. Generate the WSDL for the task from Integration Manager->Integration Configuration ->Export Task->View WSDL.

  3. Work with your integration or middleware team to modify the XSLT mapping or SOAP header configuration to include the required custom header fields.

  4. Deploy the updated mapping in your middleware:

    • SAP PI/PO: Deploy the updated integration flow or service mapping to the PI runtime engine.

    • SAP CPI (Cloud Integration): Deploy the updated integration flow to the tenant runtime.

    • Other middleware/ESB: Apply the mapping changes and activate the integration flow/service that handles Ariba messages.

  5. Test the integration by sending a sample export from Ariba and verify that the custom header fields appear correctly in the SOAP message received by the external system.

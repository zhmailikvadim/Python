Symptom

When creating a contract with the new MOS contract type CDOC, the contract validation failed if the global indicators for CMDS (billing of conventional meter device) were not set in table /US4G/KME_ENABLE.

This issue is relevant for the market role: MOS.

Other Terms

CDOC, CMDC, Contract Type, /US4G/IF_CTR_VALIDATION~VAL_CTR_HEAD, KME indicator, /US4G/KME_ENABLE, MOSB, conventional meter device, billing

Reason and Prerequisites

Program error.

Solution

Implement the corrections of this SAP Note or import the relevant support package.

After implementing this SAP Note, the system allows contract validation to pass when creating a contract with the new MOS contract type CDOC, even when the global indicators for CMDS are not set in table /US4G/KME_ENABLE. You can verify the fix by creating a contract with contract type CDOC and checking that the validation completes successfully.

Symptom

You are running the FMLA Workbench transaction (PTFMLA) for an employee who had a break in service. For example, an employee who has been first hired in 2005, fired in 2006, and re-hired in 2014. When calculating the seniority of the employee, in order to determine the eligibility for the FMLA Request, you expect the system to consider only the employment after the re-hiring action. However, the standard system always considers all employment period when calculating the seniority.

Other Terms

PTFMLA, T7US_FMLA_RULE, V_T7US_FMLA_RULE, SENIO_DT, SAPMPTFMLA, Seniority

Reason and Prerequisites

Correction of Legal Function

Solution

A new field has been added to the Definition of FMLA Rules table view V_T7US_FMLA_RULE, found in IMG activity in Payroll > Payroll: USA > Absences > Family and Medical Leave Act > FMLA Guidelines. The inclusion of this new field allows you to determine the start date that should be used to determine the seniority of a given employee.

To use the new functionality, you need to perform the following steps:

  • In transaction SM30, maintain Table/View V_T548Y, and create a new Date Type to be used for Seniority Start date for FMLA. This is step is optional, as you can re-use and existing date type for that purpose.
  • In Transaction SM30, maintain Table/View V_T7US_FMLA_RULE. In the Date Type for Seniority field, maintain the date type created/re-used in the previous step
  • For the desired employees, maintain the Date Specifications (0041) Infotype. Create a new entry for the date type created or re-used in the previous steps. Set the earliest date to be used to determine seniority in the Date Field. For the example described in the Sympton section, for example, that date could be 01/01/2014.

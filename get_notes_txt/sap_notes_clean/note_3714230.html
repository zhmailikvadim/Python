Symptom

Symptom

  • ABAP programming error occurs with runtime error BEHAVIOR_ILLEGAL_STATEMENT.
  • Error message: "Statement 'SUBMIT_AND_RETURN' is not allowed with this status."
  • Execution takes place in a transactional context where a BO implementation is active for "R_INSPECTIONLOTTP".
  • The termination occurred in ABAP program or include "LQEEVU21", in "QEEV_SAMPLING_INSTRUCT_PRINT".
  • The main program was a custom program.

Reproducing the Issue

  1. Open transaction SE38.
  2. Execute the custom report.
  3. The report execution results in a dump with the following error:
  • Category: ABAP programming error
  • Runtime Errors: BEHAVIOR_ILLEGAL_STATEMENT
  • ABAP Program: LQEEVU21

Spotted issue within the code:
   Example:
  *       Call Print Report
>>>>            SUBMIT (#####-###) AND RETURN. 

Cause

  • The issue is caused by a RAP / BO transactional context violation.
  • The custom program calls logic that executes the statement "SUBMIT ... AND RETURN" inside an active RAP BO* (R_INSPECTIONLOTTP), which is forbidden in restricted behavior phases.
  • RAP has strict rules that prohibit certain ABAP statements, including SUBMIT, COMMIT WORK, UPDATE TASK, and RFC CALLS, during restricted behavior phases.

Resolution

    This is a consulting issue, since the issue is located in the way the statement "SUBMIT ... AND RETURN" is called in a custom report.
    SAP advice is to debug the program flow to identify where the `SUBMIT ... AND RETURN` statement is being executed.  And ensure that this logic is moved outside the transactional context or replaced with a compliant approach.

    Reproducing the Issue

    1. Cancel a pipeline from the UI.
    2. Observe that the status is set as 'Cancel', but the stop time value is missing in both UI and database.

    Cause

    This is a known issue.

    Resolution

    If the stop time on the pipeline UI are not being generated after you cancel the pipeline, open a ticket with support to confirm the current status of the pipeline.  If the pipeline is marked as Cancel, then it shouldn't cause any problem in the process. 

    The fix for this issue is planned to be deployed on 2603 release.

Symptom

I need to identify why users are receiving 'UnSupported SSO protocol' or 'You are not authorized' errors when attempting to log in via Single Sign-On. I need to access the raw SAML response data and specific system failure codes to coordinate a fix with my internal Identity Provider (IdP) team without having to open a support case.

Cause

SSO authentication failures typically occur when there is a mismatch between the Identity Provider (IdP) settings and the Service Provider (SP) configuration in SAP Fieldglass.

When a user attempts to log in, the system validates the incoming SAML assertion for security and data integrity. If the system detects an invalid digital signature, an expired timestamp (clock skew), or if the user's account has not been explicitly flagged to allow Single Sign-On, the authentication is rejected and a specific error code is recorded in the background.

Without accessing the SSO Login History, administrators only see a generic browser-side error, making it difficult to determine if the issue is a network timeout, a certificate expiration, or a user mapping error.

Resolution

The SSO Login History provides a transparent view of every authentication attempt, allowing you to view detailed error descriptions and raw SAML responses directly from the dashboard.

Step 1: Access the SSO Login History

  1. Log in to SAP Fieldglass as a Configuration Manager.
  2. Select the Single Sign-On tile from the dashboard.
  3. Click on the SSO Login History tab.

Step 2: Identify the Failed Transaction

  1. Review the list of recent login attempts. You can filter by a specific Period (Date Range) or search for a specific Username.
  2. Locate the failed attempt by looking for "Error" in the Result column.

Step 3: View Detailed Error and SAML Data

  1. In the Error Message column, find the hyperlinked error text (e.g., "User is not authorized to use single sign-on...").
  2. Click the linked error message.
  3. A dialog box will open displaying:
    • Full Error Description: A detailed explanation of why the login failed.
    • SAML Response: The raw XML code sent by your Identity Provider (IdP).
    • Note: If the text is truncated in the main view, hovering over the message will also display the full text.

Step 4: Interpret Results for Resolution

Common failure reasons found in the details dialog include:

  • Invalid SAML - Signature verification failed: Ensure the certificate uploaded in Fieldglass matches your IdP's signing certificate.
  • Invalid SAML - SSO assertion expired: Check for clock skew between your server and SAP Fieldglass.
  • User is not authorized: Confirm that the Single Sign-On flag is enabled on the specific user's account profile.

See Also

  • Real-Time Auditing: The history log is updated as soon as a user encounters an error on the client-side login page.
  • Single Error Focus: If multiple errors occur during one attempt, only the first encountered error is recorded in the history.

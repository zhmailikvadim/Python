Symptom

  • Components are incorrectly allocated to operations within sequences when creating a production order.
  • All components are assigned to one operation instead of their respective operations and sequences.

Environment

  • SAP ERP Production Planning (PP)
  • SAP R/3
  • SAP R/3 Enterprise
  • SAP ERP Central Component
  • SAP ERP
  • SAP Enhancement package for SAP ERP
  • SAP Enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA Cloud Private Edition

Reproducing the Issue

  1. Go to transaction CO01.
  2. Enter material & production plant.
  3. Press Enter.
  4. Default total quantity is proposed.
  5. Enter the end date (e.g., 14.06.2026).
  6. Press Enter.
  7. Information pop-up is triggered; press Continue.
  8. Start date is calculated.
  9. Navigate to Components.
  10. All components are assigned to one operation for e.g. 0010.
  11. Access transaction CA03.
  12. Enter material & plant and other relevant details.
  13. Press Enter.
  14. Navigate to allocations.
  15. Observe that components assigned to incorrect operations within sequences.

Note: Issue can be reproduced via different steps too. 

Cause

  • The issue occurs due to the incorrect creation of component allocations for the routing group.
  • Values in fields STLNR_W and STLAL_W in the relevant table for this routing group are blank. In standard, PLMZ table will always have these field values filled. This kind of inconsistency is not possible when component allocations are created via standard process/standard transactions. 

Resolution

  • Delete the incorrect component allocations in the affected routings.
  • Create correct component allocations again via standard transactions/processes in these routings to resolve the issue.

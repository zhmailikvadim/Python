Symptom

Users attempting to use the Groupware feature receive the error message “Access is denied. Check credentials and try again.” However, adding the missed permissions which have not been granted in the Microsoft 365 Admin Center is not possible, and facing error "Try that again using a different browser. We couldn't connect to that service, likely because of settings put in place by your IT team. Open Azure in a different browser to try it again." in Permission view.

Environment

SAP Sales/Service Cloud Version 2

Cause

M365 admin need to grant admin consent using the link below (replace {Organization} with your own organization ID):

https://login.microsoftonline.com/%7BOrganization%7D/v2.0/adminconsent?client_id=b6a5e632-5250-4590-8952-27e5e7b8ce9f&redirect_uri=https://portal.azure.com/TokenAuthorize&scope=https://graph.microsoft.com/User.Read+https://graph.microsoft.com/Mail.ReadWrite+https://graph.microsoft.com/Calendars.ReadWrite+https://graph.microsoft.com/Calendars.ReadWrite.Shared+https://graph.microsoft.com/Mail.ReadWrite.Shared

If you still facing error "... We couldn't connect to that service, likely because of settings put in place by your IT team...", it indicates that this is not a permissions issue, but rather a tenant-level restriction or policy blocking the admin consent flow.

Even with Global Admin rights, this can happen due to the following Possible causes:

  • Conditional Access policies blocking access to Azure Portal/consent endpoints
  • Security defaults or tenant restrictions on user consent
  • Network restrictions (e.g., IP allowlists, firewall, VPN policies)
  • Browser/session-related issues

Resolution

Recommended next steps:

  1. Try opening the consent link in:
    • Incognito / Private window
    • A different browser
  2. Ensure the Global Admin:
    • Is logged into the correct tenant (Organization ID: xxx)
    • Has access to https://portal.azure.com without restrictions
  3. Need IT team to check:
    • Conditional Access policies (especially those targeting Azure / Microsoft Graph)
    • Any restrictions on:
      • login.microsoftonline.com
      • portal.azure.com
  4. As an alternative, the admin can try granting consent directly via:
    Microsoft Entra Admin Center → Enterprise Applications → (application) → Permissions → Grant admin consent

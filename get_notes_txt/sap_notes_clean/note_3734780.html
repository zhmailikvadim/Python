Symptom

  • When reversing a reconnection entry in the disconnection/reconnection document, a short dump occurs.
  • Reconnection read is not flagged as implausible when a read lower than the previous read is entered.
  • The system attempts to call the outbound notification SmartMeterUtilitiesConnectionStatusChangeRequestERPCancellationBulkNotification_Out even though MDUS is not connected.
  • Error code/message observed: EE_SE_MSG 110.

Environment

  • SAP for Utilities as Part of SAP ERP, release IS-UT 600 and higher.
  • SAP S/4HANA, release IS-UT 801 and higher.

Reproducing the Issue

  1. Enter a reconnection read that is lower than the previous read; observe it is not flagged as implausible.
  2. Reverse the reconnection entry in the disconnection/reconnection document in transaction EC86.
  3. Observe a short dump and that the outbound proxy SmartMeterUtilitiesConnectionStatusChangeRequestERPCancellationBulkNotification_Out is called despite no MDUS connection (message EE_SE_MSG 110 may appear).

Cause

  • The outbound proxy SmartMeterUtilitiesConnectionStatusChangeRequestERPCancellationBulkNotification_Out is not configured.
  • The fallback class IF_ES_ISU_AMI_MANUAL_DISC~CANCEL_REC_REQUEST_BLK_NO_OUT (CL_DEF_IM_ISU_AMI_MANUAL_DISC) initiates the outbound proxy when there is no implementation for ISU_AMI_MANUAL_DISCONNECT, which leads to the dump.

Resolution

  1. Verify whether the outbound proxy SmartMeterUtilitiesConnectionStatusChangeRequestERPCancellationBulkNotification_Out is configured. If required for your process, complete the configuration.
  2. If MDUS is not used, implement the methods for ISU_AMI_MANUAL_DISCONNECT with blank code to prevent the fallback class (CL_DEF_IM_ISU_AMI_MANUAL_DISC) from initiating the outbound cancellation proxy.
  3. Retest reversing the reconnection entry.

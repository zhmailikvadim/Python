Symptom

  • A blank page is seen when logging in to BI Launchpad via Windows AD SSO
  • The last HTTP request seen in a browser trace may be:
    http://<Example>/BOE/portal/2512141240/BIPCoreWeb/VintelaServlet
  • In the BILP web application logs, the below error will be seen:

    An error occurred while authentication filtering. while trying to invoke the method java.lang.Object.hashCode() of a null object loaded from local variable 's'
    java.lang.NullPointerException: while trying to invoke the method java.lang.Object.hashCode() of a null object loaded from local variable 's'
    at com.sap.tracelog.util.Anonymizer.hexHash(Anonymizer.java:11)
    at com.crystaldecisions.sdk.framework.internal.SessionMgr.getSession_aroundBody16(SessionMgr.java:1325)

Other Terms

empty page, anonymizer, winad, nullpointer, blank page

Reason and Prerequisites

Environment

  • SAP BusinessObjects Business Intelligence (BI) Platform 4.3 SP04 patch 1500+, 4.3 SP05 patch 300+, BI 2025 patch 600+
  • Windows
  • Linux / Unix 

Steps to Reproduce

  1. Log in to BILP via Windows AD SSO
  2. Observe the errors mentioned in the symptoms

Cause

Tracelog Anonymization code fails when the logon fails. This issue will only occur when the logon would have failed anyway. It prevents the logon page from being shown again - the real reason logon failure must be investigated separately.

Solution

This issue is fixed in the patches listed in the “Support Packages & Patches” section below.
Due to different internal processes, the “Support Packages & Patches” section of the SAP Note will be populated as below:

  • for fixes delivered in “Support Packages”: immediately (even if the “Support Package” is planned to be released several months later),
  • for fixes delivered in “Patches”: ONLY once the “Patch” is released AND available on SMP (not before)

Symptom

When you try to open t-code MD04 or running MRP like via t-code MD02 or MD03,
the error message ''61123: Error determining planning periods for calendar XXX'' occurs.

Environment

  • SAP ERP Production Planning (PP) Release Independent
  • SAP ERP Release Independent
  • SAP ERP Central Component Release Independent
  • SAP Enhancement package for SAP ERP Release Independent
  • SAP S/4HANA, on-premise Release Independent
  • SAP Fiori for SAP ERP Release Independent
  • SAP Fiori for SAP S/4HANA Release Independent
  • SAP S/4HANA Cloud Private Edition Release Independent

Reproducing the Issue

  1. Transaction OMI4: maintain lot size=PK.
  2. Transaction MD25: define planning calendar.
  3. Run MRP or t-code MD04 and observe that error 61123 is raised.

Cause

Possible root cause:

  1. Some MRP element falls outside of the planning calendar validity.

  2. Material has been set period lot-sizing like PK without planning calendar in master data.

  3. The period lot-sizing procedure is configured with a very high “Number of periods” in t-code OMI4.

Resolution

  • For cause 1:
    please extend planning calendar validity in t-code OMI4.
  • For cause 2:
    please enter the planning calendar in t-code MM02.

  • For cause 3:
    please adjust the field value of h “Number of periods” in t-code OMI4.

Note:
In t-code MD04, if there is no shortage(minus available qty), system will not display error message 61123.
Since there is no need to create a new planning proposal, system will not check the period lot size.
Once there is a shortage occurring, error message 61123 will be displayed when opening t-code MD04.

  • Error determining planning periods for calendar XXX.

    Message No. 61123

    Diagnosis

    The system could not determine sufficient planning periods by the planning calendar to group requirements by planning calendar periods for all the requirements planned in the future.

    System Response

    When a material is planned by using the periodical lot size by planning calendar, sufficient periods must be calculated to group requirements by the planning calendar for all requirements.

    Procedure

    Rework the planning calendar with which the material was planned, which means calculate new planning calendar periods.

    To do this, go to Customizing for MRP, under the activity Change PPC periods.

Symptom

When an ALV is initialized with a layout provided by the application (for example, from a selection screen), the layout is displayed correctly.
However, if this layout being loaded was previously set as the default layout already, the system may not recognize it as default internally.

If the user modify the layout and saves it without opening the Layout Administration screen, the layout was being saved without the default flag, causing it to lose its default status unexpectedly.

Other Terms

ALV; ABAP List Viewer; SAP List Viewer; Default Layout; Layout Save; Layout Lost; LTDX; Selection Screen; Initial Layout; Default Flag Lost; Default Cleared;

Reason and Prerequisites

Reason

When a layout name is passed during ALV initialization, the internal layout structure was filled incompletely.

The attributes for the layout including the default flag was not loaded, leading to an incomplete internal state.

Saving the layout in this state overwrites the existing definition and clears the default flag.

Prerequisites

The following Support Package level is the minimum prerequisite:

Release Support Package
SAP_BASIS 816 SP 00
SAP_BASIS 758 SP 02

When you install an SAP Note, some prerequisites might be required. The prerequisites are SAP Notes that have already been released and that change the same context of the current one. Then, even if these prerequisites are not necessarily related to the symptom of this SAP Note, you must implement them due to technical reasons.

Solution

During ALV initialization, the system now fills the provided layout with the corresponding data from LTDX, ensuring the internal structure matches the stored layout definition.

This prevents the unintended loss of the default flag when the layout is saved.

Delivery

As a general rule, SAP recommends that you install a solution by applying a Support Package.

However, if you need to install the solution earlier: Use the Note Assistant (SNOTE) to implement the correction instructions.

Symptom

Wrong accounting data in GL document due to default venture data set in customer or vendor line that would otherwise be split

Other Terms

JVA, Joint Venture Accounting

Reason and Prerequisites

Due to wrong account determination in the split check logic, wrong split results can occur when posting FI or MM documents. In esp., when the GL account is not given in one line handed over to the accounting interface, the default venture data can be set customer or vendor lines that are supposed to be split by the GL splitter.

Solution

Implement the attached code corrections with transaction SNOTE.

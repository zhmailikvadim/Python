Symptom

Users losing access to Embedded Analytics after automatic removal of IAS groups after release 2602 for Incentive Management

Environment

  • SAP Sales Performance Management 
  • Incentive Management on Hyperscalers

Reproducing the Issue

  1. Go to Portal Page.
  2. Search for users with an specific title.
  3. Perform a sync from IM > IAS.
  4. Observe that users with the specific title have their IAS role removed.

Cause

A new setting "Include Expired Title Groups in SCIM API Response" is added to Global Settings > Authentication Settings. When enabled, the system will be returning roles in SCIM API to users who have inherited portal roles from titles that are no longer active. That is, the SCIM API will include roles for users who still have portal roles inherited from inactive titles.

A title is considered "active", if the records have not been deleted, and Participant + Title + Position, are not end-dated in the past, meaning they have an effective end date, in the future.

Resolution

If the system is treating titles as inactive, you can prevent this behavior by following these steps:

  1. Go to the Portal.
  2. Select Global Settings.
  3. Choose Authentication Settings.
  4. Enable the "Include Expired Title Groups in SCIM API Response" option.
  5. Run a read job on IAS.
    • Please note that the first read job run after setting the flag may take longer than usual due to the high demand on user synchronization.

This should ensure that expired title groups are included in the SCIM API response.

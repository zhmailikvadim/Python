Symptom

After upgarding or full installation of BI 4.2 Sp5, the automation framework do not start.

Logs has following details:

com.sap.bong.wfmanager.Activator||E:\Program Files (x86)\SAP BusinessObjects\AdminConsole\Automation Framework\custom\cms_certificate\TrustedPrincipal.conf (The system cannot find the file specified)

Environment

Business Objects Enterprise 4.2 SP05

Reproducing the Issue

1. Make all the configurations as per the BI Admin console configuration guide :

https://help.sap.com/viewer/55dcdd714f614d1ca77af92b79ff5e44/4.2.5/en-US/3aee30f0473d4cb8a9f8667fb2b43d8b.html

2. start the Automation framework and BI agent from the Central configuration manager

3 .Check after sometime, Automation framework has stopped.

Cause

TrustedPrincipal.conf was not accessible or incorrect.

Resolution

1. Stop the  ActiveMQ instance (broker) if it is running. To stop ActiveMQ instance (broker) simply close the command prompt using which you launched the ActiveMQ broker. [ refer document https://help.sap.com/viewer/55dcdd714f614d1ca77af92b79ff5e44/4.2.5/en-US/1457a6755883432d9efd27ba4984b1f9.html - section Starting Automation Framework Services- point 2. to start ActiveMQ Broker]

2. Log into the CMC

3. Navigate to Authentication - Enable the trusted authentication (If it is not enabled earlier)

4. If you have already enabled Trusted authentification - Create new shared secret file and download it

5. Save it to the below location : <Install Dir>\Program Files (x86)\SAP BusinessObjects\AdminConsole\Automation Framework\custom\cms_certificate

6. Also check if the secret path mentioned in the file wfmanager.properties id correctly pointed to the cmc_properties location.

Symptom

This SAP Note contains the required BRFplus files to carry out the output management setup for SAP S/4HANA  within VSS.

Other Terms

VSS, Output management, output control, BRF+, BRFplus, output parameter determination

Solution

1. Import the BRFplus applications

Each XML file you are going to import contains one BRFplus application.
When you start newly with your SAP S/4HANA system, you can pick the according zip file matching your VSS and S4CORE version to begin with.

Note: The import of a BRFplus application from an XML file will overwrite any existing version of this BRFplus application including its content (business rules).
There is no delta handling. Please make sure that you save your current version before you import the attached files.

1.1 Download the attached zip file based on your S4CORE version:

VSS7 / S4CORE108
    SP00 --> BRF_File_VSS7_S4CORE108_SP00.zip

VSS7 / S4CORE109
    SP00 --> BRF_File_VSS7_S4CORE109_SP00.zip

Note: Do not use a zip file from a different release if you do not find a zip file for your current VSS and S4CORE Component version.

1.2 Extract the files to a temporary folder.

1.3 Perform the upload following these steps:

  • Start transaction BRF+.
    Note: If required, personalize your screen (see attachment BRF+_Personalization.jpg), and change the user mode from Simple to Expert.

  • On the Business Rule Framework plus screen, choose Tools > XML Import.
  • On the Business Rule Framework plus – XML Import screen, under File and Transport Request,
    browse for the local *.xml files you want to import. You need to import the files one after the other.
  • In the Customizing Request field, enter an applicable Customizing Request ID.
  • Choose Upload XML File.
  • When prompted, select the Delete Additional Objects After Import option. This ensures that only objects from the XML file remain in the system.
    This is especially important when you want to overwrite an existing BRFplus application.
  • Choose Back to Workbench.

2. Transporting the BRFplus application(s) and business rules

The BRFplus applications of the output parameter determination use storage type 'Customizing'.
This means you can record and transport changes to the application with a customizing request from development to quality to production system.

However, you need to assure that in the source system, customizing changes and the recording of customizing changes are allowed (transaction SCC4).
The business rules of the output parameter determination technically belong to corresponding decision table of the BRFplus application. They can also be transported as Customizing.

2.1 Handling of master data

Technically, business rules are stored in BRFplus decision tables. Each decision table consists of columns which are each assigned to a specific data type.

For the system, it doesn't make any difference if a table column represents Customizing data, master data, or transactional data.
Due to the storage type ‘Customizing’ of the BRFplus application, everything will be recorded as Customizing by default.

However, master data usually cannot be transported and is only maintained in the target system.
So before transporting the BRFplus changes, remove any master data which is not available in the target system.
Double-maintain this content in the target system. Note that the target system needs to allow Customizing changes for you to be able to do this.

Ideally, you can at least transport master data from the quality to the production system, so that you do not need to maintain anything in production.

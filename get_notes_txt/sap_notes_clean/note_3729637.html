Symptom

When I try to access my SAP Ariba Sourcing or SAP Ariba Procurement applications, I receive an "Unauthorized Request" error. This typically happens if I try to use a bookmark or link after closing my previous session. The URL I am using is similar to https://s1-eu.ariba.com/Sourcing/Main/ad/samlAuth/SSOActions?realm=**realmname** or https://s1-eu.ariba.com/Buyer/Main/ad/samlAuth/SSOActions?realm=**realmname**.

Environment

  • SAP Ariba Sourcing
  • SAP Ariba Procurement

Cause

This error can occur when using the Default Assertion Consumer Service (ACS) URL instead of your organization’s specific Enterprise URL. When a bookmarked ACS URL is used directly, it bypasses the standard sign-in process, and the necessary authentication information from the previous session is lost. The Default ACS URL is not designed for direct access or bookmarking.

Resolution

To prevent this error, you should always use your organization’s unique Enterprise URL to access the application.

  1. Identify your organization’s Enterprise URL, which will follow one of these formats:
    • For Sourcing: https://**realmname**.sourcing.ariba.com
    • For Procurement: https://**realmname**.procurement.ariba.com
  2. Replace **realmname** with your company’s specific realm name.
  3. Update any saved bookmarks or links to use the correct Enterprise URL.

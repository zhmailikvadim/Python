Symptom

  • Analytics > Defect Reporting in Cloud ALM has different count than Defect application shows
  • How do Defects set Use this item as a template respond in search results for Defect app
  • Defect Analytics Shows Incorrect Total Defects Count
  • How to see templates in Defects results list?

Environment

SAP CLOUD ALM

Reproducing the Issue

  1. Launch SAP Cloud ALM
  2. Click Implementation > Design, Build, Test, Deploy > Defects
  3. In Project, select the involved project
  4. In Defect Status, select all status values
  5. The search returns X for one user (but returns Y for another user) -> why does one user see more entries?
  6. From Home > Implementation, click Analytics and Reports > Analytics
  7. Click Defect Reporting
  8. In Project, again, select same value from step 3
  9. In Defect Status, select all values including Postponed
  10. Click Table View
  11. In Settings, enable Postponed column
  12. Search results return rows for latest dates and totals on the most recent row add up to Y -> has less than show in in step 7 for one user but the same for a different user

Cause

In Defects Analytics and Defect application itself, the default behavior in search results is that entries flagged as "Use this items as a template", results will exclude that item:

Resolution

To allow the Defect application's Search results to return items that are set as a template:

  1. Click Adapt Filters
  2. Scroll to Template and select the checkbox and click OK
  3. In order to allow both template and non-template Defects in the results, enable both Yes and No (or set according to desired results):

Note: Defects Reporting only shows counts for true Defects.  Any set as a template are not counted as a Defect.

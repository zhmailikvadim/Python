Symptom

  • A customer case was created in a non-SmartRecruiters component, but the issue seems related to SmartRecruiters.
  • An SAP customer contacts SAP regarding an existing SmartRecruiters case (e.g., speed up, escalation, providing additional information).

Environment

SmartRecruiters Product Solutions on PPMS:

  • SFSF SMARTRECRUITERS
  • SFSF SMARTRECRUITERS WINSTON
  • iOS App for SmartRecruiters for SAP SF
  • Android App for SmartRecruiters for SAP SuccessFactors

Reproducing the Issue

You receive a customer case that is not related to the area you support but:

  • Contains SmartRecruiters-related keywords such as Job, Applicants, Sourcing, Templates, Integrations, Onboarding, Reporting.
  • Does not contain identified keywords but when you search you find cases in components starting with MNA-SR*.
  • An SAP customer calls regarding an existing SmartRecruiters case requesting speed up, escalation or providing additional information.

Resolution

  1. Change or Create the Customer Case component with the SmartRecruiters component from the table below (items starting with MNA‑SR). Please note that all of these are customer‑facing components.
  2. If the 2nd level support is required from the customer case, create a Jira item on SmartRecruiters Jira instance to reach out to SmartRecruiters Engineering team – See SAP Knowledge Base Article 3713697.

What is SmartRecruiters?

SAP SmartRecruiters is SAP’s next‑generation talent‑acquisition platform, acquired and integrated into SAP SuccessFactors to replace the older SuccessFactors Recruiting module. It provides an end‑to‑end, AI‑driven hiring system used by thousands of companies worldwide.

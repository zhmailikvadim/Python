Symptom

  • The "HU Delete Assignment" button remains enabled in the Outbound Delivery (OBD) Handling Unit packing screen even after Post Goods Issue (PGI) is completed.
  • It is expected that the "Delete Assignment" button could be disabled in the Outbound Delivery (OBD) packing screen after the Post Goods Issue (PGI) is completed.  

Environment

  • SAP S/4HANA 
  • Logistics - General
  • Handling Unit Management
  • Outbound Delivery packing
  • Basic Functions

Reproducing the Issue

  1. Create an outbound delivery with handling units and post Post Goods Issue (PGI).
  2. Open the delivery’s packing (HU) screen and observe that the “HU Delete Assignment” button remains enabled.
  3. Choose “HU Delete Assignment” to unassign the HU from the delivery.
  4. Attempt to reverse PGI using VL09; the system raises “Handling unit has been changed. No cancellation possible.”

Cause

It is due to a program error in the delivery processing logic. 

Resolution

  1. Implement SAP Note 0003715312 – “Delete HU Assignment button incorrectly active after PGI in component‑group scenario.”
  2. Use the standard note implementation procedure (for example, SNOTE) and transport the change to all relevant systems.
  3. Validate the correction: after PGI the “HU Delete Assignment” button is disabled, preventing HU de-assignment on completed deliveries.

Symptom

Sync to backend is performed for self assignment when Assignment is at operation level work center

Other Terms

  • SAP Service and Asset Manager
  • SAM
  • SSAM
  • Phase Model
  • Assignment Type 6
  • Operation level work center
  • Work Center Assignment
  • ICMTANGOAMF10-43120

Reason and Prerequisites

Reason:

Assignment Type check is missing.

Prerequisites:

  •  BAS or Visual Studio Code with SAM MDK project
  • `patch` command (available on BAS and macOS by default) via terminal or other patching utility
  • Read the guidelines on how to apply the patch and read the appendix section if you receive any error

Solution

The solution is to add validation check to ensure the assignment type is specifically '2' before executing self-assignment logic..

To apply the patch, download the attached 3723104(YYMM).patch file and save it in a known location, preferably adjacent to your SAM.mdkproject
YYMM represents product release version.

If you are on Windows, see SAP Note 3533879 for how to install a .patch file in Windows. If you are on a Mac, follow the steps below:

  1. Open a terminal in the root of your SAM metadata (i.e. SAM.mdkproject/metadata or SAM.mdkproject/metadata/base.definitions)
  2. Run the terminal command `patch --no-backup-if-mismatch -p4 < path/to/3723104(YYMM).patch`. 
  3. Verify in the terminal window that the following file(s) have been modified and listed as patched, as indicated by the console output:
    • modified:   Rules/MobileStatus/OperationSelfAssign.js

Symptom

Discrepancy between the elements listed in XML export and related search.

Environment

  • SAP Sales Performance Management
  • Incentive Management

Reproducing the Issue

  1. Navigate to 'Plan' workspace.
  2. Select a plan and from there using related search navigate to 'Plan Components'.
  3. Now we will see the number of plan components assigned to the plan.
  4. In the XML export file of we could see a different number of plan components assigned to the same plan.

Cause

Situation was caused by difference in queries utilized by the 'Related search' and 'XML export'.

Resolution

1. What is the correct count?
After looking into the situation, it was determined, that the XML export should be showing the correct setup of specific plans.
***In case of doubt, please feel free to reach out to Incentive Management Support to verify the same for your specific scenario without a shadow of the same.

2. Why is the related search showing more data, than the XML export?
The reason for that is complicated and is rooted in the depth of expression of the certain processes. While the XML export is using a method and code, whereby the relations are being pulled in a way, that the results of the process remains on the "surface" level, related search can go deeper and show any elements with any relation to the base element during the target timeframe (set default period).

3. Example (please note that the below is only illustrative and not a from a real scenario):
There is Plan A, which contains Plan Component A, and Plan Component A contains Rule A.
Rule A could contain a reference to Rule B.
Rule A also could contain a reference to Formula B, which contains a reference to Rule B.
Expected, if looking for related Rules for Plan A, would only be to have Rule A returned, but... The system could potentially also return Rule B, due to a secondary reference, or a deeper reference in the system.

Keywords

XML export, Related search, Plan, Plan Component, Discrepancy.

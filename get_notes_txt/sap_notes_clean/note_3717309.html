Symptom

You are using both Product Compliance and Stewardship and the Product Compliance solution in the same system. You use the public API API_COMPANYSUBSTANCE to write company substance data that is provided by an external application. The creation of a company substance fails with an error saying the given internal number is not within the internal / external number range.

Other Terms

CL_EHFND_API_SUBSTANCE; CHECK_INTERNAL_NUMBER; CM_EHFND_API_SUBST/034

Reason and Prerequisites

The creation of a company substance only considers the default external interval 0E of number range object ESN_SUBID. The creation of company substance data fails if you call the API and enter an internal number that matches a different external interval.

Solution

The number ranges configured in the Product Compliance and Stewardship customizing for Specification Types (TCG31) are considered and the creation of a company substance no longer fails.

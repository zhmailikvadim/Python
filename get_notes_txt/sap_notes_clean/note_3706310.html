Symptom

  • MDF data in standard objects (such as Division, Department, Business Unit) are not forward propagated when existing records are changed via import or UI.
  • Data is propagated when a new record is added, both via Import or UI.

Environment

SAP SuccessFactors Employee Central

Reproducing the Issue

Example scenario:

  1. Import the Division object for the first time, including external code, effective date, name, and description.
  2. Import the Division object for the second time, including external code, effective date, and a translatable field - use the same effective date that was already created in the past.
  3. Import the Division object for the third time, including external code, effective date, and parent division. Parent division is deleted from the records imported in the first and second steps.

Cause

Forward propagation works when the record is inserted, but it does not work when the object is edited.

Resolution

   Ensure to update values on new records only. If past records need to be updated, ensure to update all other existing records in the future.

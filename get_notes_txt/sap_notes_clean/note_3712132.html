Symptom

  • Employee ID is not generating as expected during the onboarding process from Recruiting Management (RCM) to Onboarding (ONB).
  • The rule assigned works correctly in Employee Central (EC) but fails for candidates initiated from RCM.

Environment

SAP SuccessFactors Onboarding

Reproducing the Issue

  1. Initiate onboarding for a candidate from RCM.
  2. Provide a start date for the candidate in the format dd/MM/yyyy.
  3. Observe that the Employee ID is not generated as expected based on the assigned rule.

Cause

  • The rule conditions include scenarios for obtaining the date. For example, "Employee Information.Employment Details.Hire Date is on or after XXX". But the system considers an incorrect start date, leading to mismatched conditions during rule evaluation.
  • The locale of the BPE service user was set to English UK.
  • Recruiting provided the date in the format dd/MM/yyyy, but the system interpreted the date in MM/dd/yyyy format due to the locale mismatch.

Resolution

Change the locale of the BPE service user to English US.

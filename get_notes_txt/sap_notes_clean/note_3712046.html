Symptom

A delegatee is able to approve a document for the delegator even after the delegation’s End Date has passed.
In the approval history, the delegatee still appears in the Real User column even though the delegation should have already ended.

Cause

The delegation does not expire exactly at the End Date shown in the configuration. Delegations are removed only when the scheduled task ExpireDelegation runs. This task runs Monday through Saturday around 12 AM PST/PDT. Because of this schedule, two situations can cause the delegation to remain active after the End Date:

Scenario 1 – The End Date Falls on a Sunday

The ExpireDelegation task does not run on Sundays. If the End Date is set to a Sunday, the delegation remains active until the task runs again on Monday around 12 AM PST/PDT.
This allows the delegatee to approve through the delegationon Sunday.

Example:

  1. User A delegates to User B from Monday, January 1 to Sunday, March 31.
  2. The ExpireDelegation task runs on Saturday, March 30, but it does not run on Sunday, March 31.
  3. The delegation will expire only on Monday, April 1st when the task runs again.

Scenario 2 – The End Date Time Occurs After the Task Executes

The task runs at the start of the day (around 12 AM). If the End Date includes a time that, after converting to PST/PDT, occurs after 12 AM, the task will not remove the delegation until the next day.

Example:

  1. User C delegates to User D from Monday, January 1 at 12 PM to Tuesday, January 9 at 12 PM.
  2. The ExpireDelegation task runs on January 9 around 12 AM, before the specified End Date time.
  3. As a result, the delegation is removed only on January 10 around 12 AM.

Resolution

To prevent this, make sure to:

  • Avoid setting the End Date for delegations on Sundays if your business process approvals on Sundays.
  • Adjust the End Date time so that it falls before the ExpireDelegation task. Using our example, if you need the delegation to end on Tuesday, set the End Date to January 8 at 12 PM, which will cause the delegation to expire on January 9 around 12 AM.

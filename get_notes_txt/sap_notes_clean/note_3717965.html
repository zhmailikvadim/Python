Symptom

Failed to determine agents for Workflow while setting up External Determination of Agents for Supplier Invoice Approval in SAP Ariba Invoicing with error as below:

workflow.service.external.determination.of.agents.failed  
Failed to determine agents for Workflow Instance(<instance-id>) of Invoice(<invoice-id>)

Cause

The workflow error occurs due to an incorrect response content type returned by the External Determination of Agents (EDA) service, and not because of an invalid userUuid. While the EDA service returns a JSON structure, it is sent as stringified JSON (text) instead of a proper JSON response. As a result, SAP Ariba Invoicing cannot parse the response, leading to the workflow failure.

Resolution

To resolve this, please ensure that the response header 'content-type' is 'application/json' to SAP Ariba Invoicing to process it.  Once the response is returned as valid JSON with the correct content type, SAP Ariba Invoicing can successfully process the agent determination.

kindly note that either 'email' or 'userUuid' is sufficient, and it’s not mandatory to provide both. Hence, 'userUuid' can be skipped when 'email' is populated. When 'userUuid' is populated, it should be the 'Global User ID' from IAS. More Information regarding structure can be found External Determination of Agents for Supplier Invoice Approval.

Symptom

Estimated Delivery field of some POs in Guided Buying is empty, even though sometimes Expected Delivery Date is displayed correctly in Buying & Invoicing.

Environment

SAP Ariba Guided buying 

Cause

If Delivery Date is set when creating ship notice, and also some estimated delivery dates are set when creating order confirmation, the delivery date will be shown in Estimated delivery section in GB PO view screen.
And if there is no Delivery Date set when creating ship notice, but some estimated delivery dates are set when creating order confirmation, the estimated delivery date which is closest to today including today will be shown in Estimated delivery section in GB PO view screen. 

The empty EstimatedDelivery values are attributed to the following two scenarios:

1. Supplier sent ship notices to inform buyer, however, the latest ship notice lacks a DeliveryDate or all OCLineItem entries contain an expired DeliveryDate.

2. Supplier did not send ship notice to buyer.

Resolution

To determine the Estimated Delivery Date, the system checks as follows:

If the order is a service order:
The system retrieves the most recent receipt whose status is neither "Rejected" nor "Composing". The ServiceEntryDate from this receipt is then used as the Estimated Delivery date.

If the order is a non-service order:
Step1: The system checks if the supplier has performed any ship notice operations.
If such operations exist, the system selects the ship notice with the latest Ship Notice Date and uses its Delivery date as the Estimated Delivery date.

Step 2: If the Estimated Delivery Date remains unset after Step 1, the system will scan all order confirmations (OCs) to identify potential dates and select the OC line item with the earliest Delivery date that is later than the current date.

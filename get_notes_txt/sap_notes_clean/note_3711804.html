Symptom

In transaction "FGRIR_CLEARING", if the box "Use Clearing Criteria" is checked and a non-existing clearing criterion is used, a dump occurs.

The dump occurs in class "CL_FINS_OI_GROUPING_HELPER" (and method "IF_FINS_OI_GROUPING_HELPER~REPLACE_BSEG_WITH_ACDOCA_NAMES").

A non-existing clearing criterion means, it exists in database tables BSEG, BSIS, ... but not in ACDOCA.

Extended open item management just clears in/operates on database table ACDOCA. Therefore, also just clearing criterion could be used that exists in this database table.

After implementing this SAP note, an error message is shown, if there are clearing criterion entered in the customizing that can't be processed by transaction "FGRIR_CLEARING".

It is recommended in the customizing of clearing criterion (transaction "OB74") to create an own entry for each GR/IR GL-account using extended open item management.

Other Terms

FGRIR_CLEARING, FINS_GR_IR_MASS_CLEARING, Extended open item management, Extended OIM, GR/IR, CL_FINS_OI_GROUPING_HELPER

Reason and Prerequisites

Program error

Solution

Please implement the attached correction instruction.

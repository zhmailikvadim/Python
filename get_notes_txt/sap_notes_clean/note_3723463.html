Symptom

  • The system configures FEFO as stock removal rule
  • The picked product has same expiration date but in different bins
  • The requested quantity is more than the bin quantity, system needs to pick from multiple bins
  • These bins have sort sequence in /SCWM/SBST
  • System does not respect the sort sequence

Environment

  • SAP Extended Warehouse Management (SCM-EWM)
  • SAP Extended Warehouse Management Warehouse Order Processing (SCM-EWM-WOP)

Cause

This is normal behavior.

Resolution

In outbound process, the bin sorting doesn't have any effect on the process which source bin should bin picked during the warehouse task creation.

The deciding factor in this case is the sorting fields defined under the following path in SPRO: SCM Extended Warehouse Management -> Good Issue Process -> Specify Stock Removal Rules. In this case is the same expiration date. So system will pick from the bins in the sequence of bin number.

Keywords

Picking warehouse task; FEFO removal rule; batch expiration date; bin sorting sequence; /scwm/sbst

Symptom

When Cost Center master data is replicated from SAP BTP Integration Suite to SAP S/4HANA using the inbound SOAP API, the messages appear as “Delivered” in SRT_MONI, even when the replication fails due to master‑data issues (e.g., Profit Center does not exist).
No error is visible in SRT_MONI or SLG1, yet the Cost Center is not created in KS03.

Other Terms

Cost Center SOAP API, Web Service, SRT_MONI, Application Log, SLG1, Master Data Replication, Profit Center validation.

Reason and Prerequisites

The inbound SOAP API completes technical processing successfully, so SRT_MONI always shows “Delivered”.
However, the business validation errors (such as missing/invalid Profit Center) are not written to the Application Log, because the API implementation does not pass backend validation errors to BAL logging.
As a result, no error logs are generated, even though the Cost Center creation fails.

Solution

Error‑handling was corrected so backend validation failures are now logged properly in SLG1. When replication fails, meaningful error messages are recorded and visible for analysis.

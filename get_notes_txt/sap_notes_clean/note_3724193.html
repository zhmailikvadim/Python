Symptom

When displaying archived billing document in VF07, under the "PO Data" tab, purchase order data (such as Purchase Order Number, Telephone Number) are empty.

The values are displaying when the corresponding sales order has not been archived. Once the sales order is archived, the purchase order data becomes unavailable.

Environment

  • Sales And Distribution (SD)
  • Logistics Execution (LE)
  • SAP R/3
  • SAP R/3 Enterprise 4.7
  • SAP ERP Central Component
  • SAP ERP
  • SAP Enhancement package for SAP ERP
  • SAP Enhancement package for SAP ERP, version for SAP HANA

Reproducing the Issue

  1. Go to VF07, enter billing document number
  2. Select item entry
  3. Check item "PO Data" section

Cause

The details under item "PO Data" tab are derived directly from the corresponding sales order, tables VBAK and VBKD. 

F1 help -> Technical Information:

Purchase order no.VBKD-BSTKD
PO dateVBKD-BSTDK
Pur. ord. typeVBKD-BSARK
NameVBAK-BNAME
Your ReferenceVBKD-IHREZ
TelephoneVBAK-TELF1
... 

When displaying a document from the archive, the system only reads the data and table entries from the archive that belong to this document only. The system does not read any information linked to this document that is also archived.

Resolution

Refer to the Solution part in Note 999547 - No purchase order data when displaying from archive.

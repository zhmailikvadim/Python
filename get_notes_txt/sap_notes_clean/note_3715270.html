Symptom

When you run valuation or translation programs (FAGL_FCV, FAGL_FC_VALUATION, or FAGL_FC_TRANSLATION) in a system with NewGL, it is unclear which amounts the system displays in the output list.

Other Terms

FAGL_FCV; FAGL_FC_VALUATION; FAGL_FC_TRANSLATION; output list; entry view

Reason and Prerequisites

You may find it difficult to compare the results of the output lists of the valuation and translation programs because the system displays different views of the data.

Solution

The system displays the entry view of open items or the G/L balances on assignment level in the output list of the valuation and translation programs when NewGL is used. The entry view contains the data you entered. The G/L view contains the G/L data of ledgers, including account assignments and possible document splitting values.

For technical reasons, the system displays only the entry view in the output list of the valuation program. As a result, some account assignments may not appear or may not be fully displayed.

The system stores valuation differences in the valuation history tables by item (open items) or by G/L account (non-OIM G/L accounts), without account assignments. The system reads these tables and uses them as the basis for the output list display.

When you run a reset, the system consequently reads the valuation history tables to display the output list. A reset of open item valuation shows the entry view of the valuated open items. A reset of the G/L balance valuation or FAGL_FC_TRANSLATION shows the current balances on assignment level.
In the output list no distribution of the valuation difference values across account assignments is displayed.

This is only a display issue. The system posts reset entries correctly, considering all account assignments.

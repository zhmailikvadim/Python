Symptom

  • The balance carry forward was posted to an incorrect account , although corrected to the right account later, the balance still appears in the wrong account when running the FEC report.
  • The issue involves statistical postings generated by the RFOPENPOSTING_FR program 
  • Guidance is required on reversing these statistical postings from the incorrect account.

Reproducing the Issue

  1. Run the RFOPENPOSTING_FR program 
  2. Post the balance carry forward to the wrong account 
  3. Correct the posting to the right account, but observe that the balance still appears in the wrong account when running the FEC report.

Cause

The issue is caused by statistical postings generated by the RFOPENPOSTING_FR program, which incorrectly posted the balance carry forward to the wrong account 

Resolution

  1. Reverse the incorrect statistical postings using transaction FB08 for individual document reversal.
  2. For mass reversal of multiple documents, use transaction F.80.
  3. Refer to SAP Note 3055251- RFOPENPOSTING_FR: Updated Report Documentation for detailed instructions.

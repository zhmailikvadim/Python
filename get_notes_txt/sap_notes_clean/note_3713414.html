Symptom

After upgrading to a new optimization engine version (build date February 2026 or newer), Load Consolidation shows variations in runtime performance (faster or slower execution depending on the scenario).

Other Terms

Load Consolidation, LCO

Reason and Prerequisites

As part of continuous efforts to enhance SAP Transportation Management (TM), improvements in the Load Consolidation functionality have been implemented, specifically in the optimization engine. This enhancement is designed to:

  • Improve solution quality (reduced costs, etc.)
  • Optimize runtime performance (faster processing of consolidation scenarios)

The new optimization engine has undergone testing across a wide range of scenarios to ensure quality standards. However, there is a possibility that some users may experience longer runtimes for Load Consolidation compared to the previous version.

Solution

If you observe slower execution for Load Consolidation after upgrading to a new optimization engine, the automatic runtime setting can be used. Depending on this setting, Load Consolidation is able to reduce the actual runtime automatically depending on the complexity of the scenario.

This functionality is available by setting an optimization extra parameter (see note 578044):

  • Application: TVSO
  • Section: VSO
  • Name: RuntimeAutomatic
  • Switch: String
  • Char.String: (string see below)

Possible string values are:

  • "FASTEST": a valid load plan is returned as fast as possible. The solution might not be the best for complex scenarios but can still be good.
  • "FAST": focus on short runtimes, but the runtime is a bit higher compared to "FASTEST".
  • "BALANCED": balancing runtime and solution quality.
  • "QUALITY": focuses more on solution quality, which results in longer runtimes.

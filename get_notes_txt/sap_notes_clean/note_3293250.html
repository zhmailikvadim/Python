Symptom

Error Screen does not exist dump comes when maintaining a record directly for infotype Additional Personal IDs Info(IT3305). The issue appears either by applying the note 3104091 or upgarde to support package SAPKE608A0 or higher.

Other Terms

DYNPRO_NOT_FOUND, MP330500, IT3305, T777D, Additional Personal IDs Info, Payroll Saudi Arabia, Payroll Qatar

Reason and Prerequisites

The infotype IT3305 is a secondary infotype and should be maintained from infotype Personal IDs (IT0185). The attribute Primary / Secondary Infotype has changed from Additional Infotype for view to Infotype and causing the issue.

This Note is applicable for the HCM Localization in Kingdom of Saudi Arabia / Qatar

Solution

Configuration change performed to correct the behaviour.

As a general rule SAP recommends that you implement the Support Package specified for your release. However, if you need to install the solution earlier, follow the manual steps described in SAP_Note_3293250_-_Attribute update for Additional Personal IDs Infotype.pdf, which is attached to this note.

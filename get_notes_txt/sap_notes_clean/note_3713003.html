Symptom

An error occurs after entering time in My Timesheet Fiori App and attempting to submit. 
The error message is "Personnel number XXXXXXXX does not exist or you do not have authorization".

Environment

HCM Fiori

Reproducing the Issue

1. Log on to Fiori.
2. Open My timesheet Fiori App
4. Input time and click on submit. 

Cause

Testing the same via CAT2 transaction with the same user, there is an error also
"No authorization for personnel number xxxxxxxx using profile XXXXXXXX on xx.xx.xxxx
Message No. LR034".

It is because of missing authorization customizing configs for CATS dummy infotype 0328 and 0316. The infotypes 0316 and 0328 do not actually exist in the system (called 'dummy' infotypes) and are only used to access the HR authorization concept in order to assign authorizations to read, change or approve time sheet data. Those dummy infotypes are maintained in the users authorization profile. The user access to personnel numbers is controlled by those infotypes.

Resolution

Follow the KBA resolution 1824918 Error "No Authorization for personnel number XXXXXXXX using profile XXXXXXXX on key date" in transaction CAT2 or report CATS_APPR_LITE

Keywords

My Timesheet V4, F3074A, My Timesheet V3, F3074

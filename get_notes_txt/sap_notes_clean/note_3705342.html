Symptom

Once the developemnt SPAU TR is inlcuded into the quality system SPAM, the error pops a below:

SPAM error: Cofile XXX.<SID> doesn't contain 'PROJECT = SAP_ADJUST', so it's not marked as Modification Adjustment.

Environment

SPAM

Reproducing the Issue

  1. Perform a package upgrade through SPAM in the Development system.
  2. Capture SPAU changes in a transport request and flag it with SAP_ADJUST.
  3. Validate the transport request in the source system prior to the upgrade.
  4. Observe that the transport request does not contain 'PROJECT = SAP_ADJUST' and is not marked as Modification Adjustment.

Resolution

Ensure that the SPAU transport request is released in the Development system after SPAM/SAINT has been completed.

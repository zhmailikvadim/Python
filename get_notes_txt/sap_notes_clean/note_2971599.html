Symptom

To set a selection variable on T-code FAGLL03, saving the variant and setting the job does not activate the selection variable for the data in dynamic selection.

Disclaimer: "Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental."

Environment

  • SAP ERP
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA

Reproducing the Issue

  1. Access the T-code FAGLL03
  2. Provide the posting date under All items
  3. Provide the posting date in dynamic selection of G/L Account Line Items
  4. Click on save the variant and set Selection variable as "D", option "=" and Name of the variable "Current date -1 days" for posting date as depicted in below

   5. Save the variant, set the job and use the variant, but the Posting Date (G/L Account Line Items) in Dynamic selection does not change the date to Current date - 1 ( one day before the current date ).

Cause

The 'Dynamic Selections' part seen in the FAGLL03, the variant stored is NOT used in the FAGLL03 itself. This is a by-product of the usage of the logical database SDF in the FAGLL03.

But this is ONLY used for the selection of the master data. The 'Dynamic selections' for the FAGLL03 are created in the report itself with specific function modules, and the values that are restricted here are later stored in the 'hidden' selection fields FS_DYNS and FS_NUM (for the count of these).

Resolution

The related 'Selection Variable' logic which is usable in the FBL*N transactions (as these use the logical databases to the full extend) is not usable in the FAGLL03.

The reason why the FAGLL03 needs to build their own dynamic selections is that this transaction also selects the New G/L table structure, in standard FAGLFLEXA or later ACDOCA. But as the customer can also use their own tables here, this had to be created dynamically in the report itself, and so this LDB selections from 'Dynamic Selections' saved in a variant cannot be used here.

Symptom

SAPOSCOL cannot detect virtualized environment for Linux-based application server on Microsoft Azure

Environment

  • Linux on Microsoft Azure platform

Reproducing the Issue

  • -Run the ST06 transaction or run the followingcommand as sidadm on the OS host
    printf 'dump ccm\nquit\n' | /usr/sap/hostctrl/exe/saposcol -d
  • The Virtualization section returned FALSE for Enhanced Monitoring Access
    Virtualization_Configuration\Enhanced Monitoring Access    FALSE   
    Virtualization_Configuration\Enhanced Monitoring Details    Check SAP Monitoring: No Data. Refer to SAP Note 2191498.   
    Virtualization_Configuration\Solution    VIRT_METHOD_MICROSOFT   
    Virtualization_Configuration\Solution Version    AZURE: NO INFO!    AZURE: NO INFO!   
    Virtualization_Configuration\Cloud Provider    Microsoft Azure    

Cause

  • Wrong extension version used on Linux VM hosted on Azure.
  • The OS of the VM is equal or has a release version higher than SUSE SLES 15, Red Hat Enterprise Linux 8.1, Oracle Linux 8.1, which is better compatible with the new version of the SAP VM extension.

Resolution

  1. Make sure SAP Host Agent is running at version 7.21 PL 47 or later, and the VM has a system-managed identity assigned (system-managed identity is checked by Microsoft Support)
  2. Remove the old extension and reinstall the new version with the flag. This could be done by executing the following PowerShell commands on a system with the Azure Powershell module:
    $rgname = "<Azure Subscription Resource Group>"
    $vmname = "<vmname>"
    Remove-AzVMAEMExtension -ResourceGroupName $rgname -VMName $vmname
    Set-AzVMAEMExtension -ResourceGroupName $rgname -VMName $vmname -InstallNewExtension
    Test-AzVMAEMExtension -ResourceGroupName $rgname -VMName $vmname

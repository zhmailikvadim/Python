Symptom

When changing Purchase Order (PO) via ME22N transaction, the final invoice indicator (EKPO-EREKZ) is greyed out and cannot be set if the PO item is deleted, while changing the same PO via MEMASSPO the field can be set even if the item is deleted.

Environment

  • Materials Management (MM)
  • SAP ERP Central Component
  • SAP ERP
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA
  • SAP S/4HANA Cloud Private Edition

Reproducing the Issue

  1. Change PO in ME22N
  2. Line item is deleted and the "Final Invoice" field is not set in the Invoice tab.
  3. Final invoice (EKPO-EREKZ) field is greyed out
  4. Select the same PO item in MEMASSPO
  5. Choose "Final Invoice" select the column, and perform a mass change.
  6. Save and continue.
  7. Display the PO in ME23N and check that the "Final Invoice" field is now set.

Cause

In transaction MEMASSPO, changing certain fields is possible even if the deletion indicator is set on a PO item. This behavior is part of the standard SAP system design. 

Resolution

The field EKPO-EREKZ can be changed via MEMASSPO even if the item line is deleted, because, via ME22N, the system allows the user to unblock/undelete the PO item, set the Final Invoice (EKPO-EREKZ) indicator, save the data and delete the item again afterwards.

This functionality is also reflected in MEMASSPO, where changes to the "Final Invoice" field are allowed despite the deletion indicator being set.

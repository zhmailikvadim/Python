Symptom

  • Planned accrual item types are not visible in the Display Purchase Order Accruals app (F3928).
  • The app functions differently from the SAP GUI transaction code (POACTREE03), where these item types are displayed. 

Environment

  • Financial Accounting
  • SAP Fiori
  • SAP S/4HANA

Reproducing the Issue

  1. Access the "Display Purchase Order Accruals" app (F3928)
  2. Attempt to view planned accrual item types and related calculations 
  3. Observe that planned accrual item types are not displayed.

Cause

Standard system design.

Resolution

  1. The Display Purchase Order Accruals app functions as intended.
  2. This app is designed to display real accrual postings made in FI for the specific object.
  3. In the section 'Accruals Overview,' proposed periodic values can be viewed.
  4. If no proposed values are made for the purchase order, no accruals will be displayed.
  5. Online calculated accruals are displayed in the SAP GUI transaction POACTREE03.
  6. This transaction can be harmonized with FIORI and added as a tile to your Fiori launchpad if needed. 

Keywords

planned accrual item types, display purchase order accruals, POACTREE03, F3928

Symptom

  • Inbound qRFC queue in SMQ2 becomes stuck with a large backlog of messages.
  • Messages for XI inbound processing do not proceed until manual intervention.
  • In SXMB_MONI, Runtime shows DBQUEUE step "DB_ENTRY_QUEUING" with long begin/end timestamps.

Environment

  • SAP ERP
  • Integration Engine

Reproducing the Issue

  1. Open transaction SXMB_MONI and search for the affected inbound XI interface within the relevant date/time range.
  2. Open a message and review the Runtime to observe the DBQUEUE step "DB_ENTRY_QUEUING" with begin/end timestamps spanning an extended period.
  3. Open transaction SMQ2 and check inbound qRFC queues for stuck entries corresponding to the interface.

Cause

  • XI inbound messages were configured with a parallel degree of 1 via SXMS_IECONF parameter EO_INBOUND_PARALLEL_RECEIVER, causing sequential processing and long waiting times; one failing message blocked subsequent ones.
  • XI queues were not registered for automatic restart in SMQR (via report RSXMB_REGISTER_QUEUES), and the system was set to stop the queue on error until manual restart.
  • An inbound proxy/runtime error likely occurred, which paused the queue and led to accumulation.

Resolution

  1. Review SXMS_IECONF and increase the EO_INBOUND_PARALLEL_RECEIVER parallel degree for the affected inbound receiver (for example, to 10) after confirming compatibility with the receiver implementation.
  2. Register XI queues for automatic restart in SMQR by running report RSXMB_REGISTER_QUEUES.

Symptom

An upgrade to S/4HANA was performing and when trying to execute  transactionTBDM the following error message is raised SG813: "No physical path is configured for logical file name FTRM_TCR_MARKETDATA_FF_IMPORT"

Environment

Treasury and Risk Management (TRM)

Cause

Configuration

Resolution

  1. Kindly check the table/view FILECMCUST especially the parameter REJECT_EMPTY_PATH, which is not maintained in the old environment and ON in the S/4 environment.
  2. Please do not adjust this customizing setting as setting REJECT_EMPTY_PATH to OFF will enable to upload files for logical file names for which no physical path name is maintained and impact all transaction codes involving logical file names.
  3. Moreover, please note this change allows that anyone can read/write any file from/to the application server for example through CG3Y and CG3Z which is considered as a security hole.
  4. Therefore, from a security point of view it is recommended to define a physical path for the logical path like for FTRM_TCR_MARKETDATA_FF_IMPORT.
  5. Define a Physical path for the Logical path FTRM_TCR_MARKETDATA_FF_IMPORT - open transaction FILE and in Logical File Path Definition find Logical Path FTRM_TCR_MARKETDATA_FF_IMPORT. Select it and double click 'Assignment of Physical Paths to Logical Path' and define the Physical path.

Keywords

SG813, FTRM_TCR_MARKETDATA_FF_IMPORT, TBDM

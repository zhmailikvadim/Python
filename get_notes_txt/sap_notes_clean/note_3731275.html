Symptom

  • The New Hire Data Review (NHDR) or Final Review steps do not submit the first time and requires resubmission. As a result, creates two job information record in EC Table.


  • Duplicate Record Error received when we submit MPH.

Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.

Environment

SAP SuccessFactors Onboarding

Reproducing the Issue

  1. Complete the NHDR or Final Review step for a candidate
  2. The page does complete
  3. Re-submit 
  4. Process the candidate in MPH
  5. Receiving a duplicate user error

Cause

  • If your instance has WalkMe enabled, it would be interfering with the page redirection when submitting New Hire Data Process due to which the NHDR has to be submitted twice.
  • This was previously a known issue and this has been fixed by Engineering Team.

Resolution

Solution:

  • The Engineering Team has fixed the issue. 
  • All newly initiated candidates will not have any duplicate candidates.
  • Previous workaround of disabling WalkMe is no longer required. WalkMe can be enabled now.

For existing candidates with the error:

  • Create a ticket to Technical Support and share this KBA in the description or subject of the case
  • Provide your approval for script execution to delete the incorrect or duplicate jobInformation/compInfo record for the affected candidate.

for Technical Support Team

  • See internal information for the sample script.

NOTE: If WalkMe is not enabled and you are seeing duplicate jobInformation or compInformation records created at Manage Pending Hire, please refer to 3614135 - Duplicate Record Error received when we submit MPH - Onboarding - SAP for Me

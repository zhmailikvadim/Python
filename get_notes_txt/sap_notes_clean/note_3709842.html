Symptom

When creating an A/R Invoice via Service Layer based on a Sales Order that contains Freight and a document-level Discount (not line-level), the discount is incorrectly re-applied during invoice creation. As a result, the final Discount shown on the Invoice becomes higher than the original discount from the Sales Order, as if the same discount were applied again on top of the already discounted value.

Reproducing the issue

Prerequisites:
You are working with a Brazilian localization database.

1 - Create a Sales Order and Set the data below:

Document header:
Business Partner: C23900 (any BP works)
Discount: 10.00%
Document row:
ItemCode: A00001 (any item works)
Quantity: 1
UnitPrice: 1000
TaxCode: 1915-001 (this tax is 0. do not change the amounts)
Freight Settigns
Freight Name: Frete
TaxCode: 5109-001 (this tax is 0. do not change the amounts)
Net Amount: 100

2 - Logon in Service Layer and create a AR Invoice with the payload below:
{
  "DocType": "dDocument_Items",
  "DocDate": "2026-01-29T00:00:00Z",
  "DocDueDate": "2026-01-29T00:00:00Z",
  "CardCode": "C23900",
  "TaxDate": "2026-01-29T00:00:00Z",
  "DocumentLines": [
    {
      "BaseType": 17,
      "BaseEntry": 184, //change doc entry
      "TaxCode": "1915-001",
      "BaseLine": 0
    }
  ],
  "DocumentAdditionalExpenses": [
    {
      "BaseDocEntry": 184, //change doc entry
      "BaseDocLine": 1, //check correct line num in base Sales Order Freight table
      "TaxCode": "5109-001", 
      "BaseDocType": 17
    }
  ]
}

 

Expected Results:
The AR Invoice is created with 10% Discount at document header, as the Sales Order

 

Actual Results:
The AR Invoice is created with 11,11% Discount at document header, unlike the Sales Order

Symptom

When running an SAP HANA MDS1) query you receive an error message

  • Error code 42014:
    "Caught exception : exception 42111: SQL Error : SQL Error"

1) MDS: MultiDimensional Services; analytical engine in SAP HANA, responsible for processing analytic request fom SAP Analytics clienst such as SAP Analytics Cloud (SAC) or SAP Analysis for Office.

Other Terms

SAP Analytics Cloud, SAC, SAP Analysis Office, SAP Design Studio, InA/MDS

Reason and Prerequisites

Reason:

Due to a programming error in SAP HANA, a SQL statement was missing a string conversion of a field.

Affected Releases:

  • SAP HANA 2.0 database, EPMMDS plugin revision:
    • "Internal Version"  < 1.00.2026.14
  • SAP HANA Cloud Edition
    • < CE 2026.14

Prerequisites:

  • You create a dimension with a keyattribute with an NVARCHAR type.
  • You create a level-based-hierarchy whose last level has a non-NVARCHAR type (e.g. date, integer)
  • In your query you request the dimension with the hierarchy in question active and request additional display attributes.

Workaround:

Either change the type of:

  • last hierarchy level column to NVARCHAR or
  • the dimension key attribute to VARCHAR (if possible)

Solution

Apply one of the following SAP HANA database Revisions:

  • SAP HANA 2.0 database, EPMMDS plugin revision:
    • "Internal Version"  1.00.202614.00 or higher
    • See SAP Note 2444261 for availability of this EPMMDS plugin revision for your SAP HANA revision
  • SAP HANA Cloud Edition
    • CE 2026.14 or higher

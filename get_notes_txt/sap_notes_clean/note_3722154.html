Symptom

In T-code AW01N after Company Code and Asset was added and Enter pressed, a popup will be shown with warning message: AU390 even after executing Asset Recalculation.

Environment

  • FIN Asset Accounting​
  • SAP S/4HANA ​
  • SAP S/4HANA Finance​
  • SAP S/4HANA Cloud Private Edition

Reproducing the Issue

  1. Access AW01N.
  2. Select Company Code and Asset.
  3. Recalculate values using the "Planned Values" tab and click the "Recalculate Depreciation" icon.
  4. Verify that APC transactions and depreciation values appear temporarily in AW01N.
  5. Exit AW01N and re-enter to observe that the values are missing again even after recalculating values in real mode.

Cause

  • Inconsistency during FAGLGVTR execution caused by incorrect balance carryforward values.
  • The APC GL account associated with the asset was incorrectly set up with open-item management activated, which is not supported for asset reconciliation accounts.
    • To check open-item management:
      • tCode FS00 > Create/bank/interest tab;

Resolution

  1. Ensure that the APC GL account associated with the asset does not have the open-item management checkbox activated.
  2. Execute the program FAA_BCF_TRIGGER in SE38 to restart the asset accounting part of the balance carryforward.
  3. Run FAGLGVTR for Company Code to the fiscal year XXXX with the "Reset Central Balance Carry Forward" option to reconstruct the balance carryforward.
  4. Verify that the missing values are correctly displayed in AW01N.

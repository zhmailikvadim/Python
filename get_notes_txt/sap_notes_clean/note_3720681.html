Symptom

OM-IT0001 integration issues have been identified following the implementation of SAP Note 3413629. In response, SAP Note 3572487 was released to address these problems. However, even after applying SAP Note 3572487, the following integration issues persist:

  • When updating S-O relationships (e.g., S-K...) using t-code such as PP01 or PO13, IT0001 is not automatically updated as expected.
  • After executing report RHINTE30 to integrate data from Organizational Management (OM) to Personnel Administration (PA), the message 'No update of infotype 1 necessary!' is displayed, despite existing data inconsistencies between OM and PA.

Environment

  • Human Capital Management (HCM)
  • SAP ERP 6.0
  • SAP ERP Core Component 6.0
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for HANA
  • SAP S/4HANA

Reproducing the Issue

Prerequisites:

  • Integration between OM (Organizational Management) and PA (Personal Administration) is active
  • The SAP Note 3572487  is implemented in the system
  • The IT0001 is enhanced using the include CI_P0001

Example 1:

  1. Run t-code PP01 to update S-O relationship(e.g., S-K...).
  2.  Check the affected employee's record and observe that the IT0001 is not updated.

Example 2:

  1. Run t-code SE38 to execute report RHITNE30.
  2. Specify the personnel number.
  3. Uncheck the checkbox 'Open Personnel number only'.
  4. Execute the report in live run.
  5. The message 'No update of infotype 1 necessary!' is displayed.

Cause

Source code added by SAP Note 3572487 to ensure successful OM-IT0001 integration has been overwritten by custom modification via SPAU, reverting the code of include LRHPRTOP to an older version, which is incorrect.

Resolution

Revert the source code of include LRHPRTOP to the correct version as follows:

  1. Access transaction SE38.
  2. Enter the program name LRHPRTOP.
  3. Navigate to Utilities -> Versions -> Version Management.
  4. Select the desired version (e.g., version 00023).
  5. Click "Retrieve" to generate an inactive version.
  6. Activate the inactive version and the expected source code of LRHPRTOP is below:
  7. Test the OM replication process to confirm successful integration with IT0001.

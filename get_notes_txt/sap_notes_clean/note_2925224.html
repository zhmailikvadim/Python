Symptom

Payment Information RBP limitations

Environment

SAP SuccessFactors Employee Central

Reproducing the Issue

There is a requirment where employees can create new records and modify or overwrite the current record without option to edit history in payment information portlet.

Resolution

Please follow the below Navigation Steps but when you give permission to "correct " option, "View History" permission gets enabled automatically

  • Go to Admin Center. In the Tool Search field, enter Manage Permission Roles.
  • Choose Take action and then Edit for the role you want to edit.
  • Click the Permission button.
  • Click Miscellaneous Permissions, and select the necessary permissions for all the objects related to PaymentInformationV3, where can grant Create, Insert and Correction permission 
  • Click Done, and then Save Changes.

Hence, you cannot provide "correction" permission without "Histroy" permission for Payment Information Portlet. This is expected behaviour

Keywords

Create, Insert, Correction, permission, History, payment information

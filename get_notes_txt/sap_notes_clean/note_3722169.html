Symptom

When checking the use-cases in the transaction /n/SDF/ALM_SETUP or checking the destination in the transaction /n/SDF/ALM_DIAGNOSTIC, there is a warning regarding the G5 certificate in the "Transports" section (03 - Certificate Check), even though this certificate is already correctly maintained in the STRUST transaction:

Certificate recommended: CN=DigiCert RSA4096 Root G5, O="DigiCert, Inc.", C

Environment

  • SAP Cloud ALM
  • SAP S/4HANA - ST-PI 740 SP31 to SP33

Reproducing the Issue

There are two ways to reproduce the issue:

  1. By checking the use-cases:
    1. Log on to the ABAP Managed System.
    2. Execute transaction /n/SDF/ALM_SETUP.
    3. Check the use-cases.
  2. By checking the Cloud ALM Diagnostic transaction:
    1. Execute transaction /n/SDF/ALM_DIAGNOSTIC.
    2. Select the respective Destination.
    3. Click to Execute.

Cause

Code issue.

Resolution

Implement SAP Note 3710503 in the latest version in the Managed System.
To apply the corrections successfully, please follow the steps below:

  1. Implement SAP Note 3710587.
  2. Go to transaction SA38.
    • Run the report/program "/SDF/NOTE_3710503" according to the steps outlined in SAP Note 3710587.
  3. Implement SAP Note 3710503.
  4. Check the results again by using the transaction "/n/SDF/ALM_SETUP" or "/n/SDF/ALM_DIAGNOSTIC".

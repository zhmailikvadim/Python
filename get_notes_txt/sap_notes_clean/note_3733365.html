Symptom

  • The primary job REEDMFICALCULATE completes while EEDM_CALC_DISP and EEDM_CALC_EXEC_TS* background jobs continue to run.
  • Excessive child and sub-child jobs (for example ~100 EEDM_CALC_DISP per run, each starting ~10 EEDM_CALC_EXEC_TS jobs; totals approaching ~1k–4k jobs) cause scheduler sequencing delays.
  • EDM profile calculation runtime is very long.
  • Job chains cannot wait for completion of all subordinate jobs because the primary job ends earlier.
  • The job log may display “*” instead of the number of time slots.

Environment

  • SAP S/4HANA, release IS-UT 801 and higher
  • SAP for Utilities as Part of SAP ERP, release IS-UT 600 and higher

Reproducing the Issue

  1. Schedule REEDMFICALCULATE in background with intervals defined (e.g., 100) and parallelization settings.
  2. Start the job and observe that multiple EEDM_CALC_DISP jobs are started, each launching EEDM_CALC_EXEC_TS* jobs.
  3. Observe that REEDMFICALCULATE completes while the spawned jobs are still running and the scheduler proceeds to the next chain step.
  4. Review the REEDMFICALCULATE job log and note that the number of time slots may appear as “*”.

Cause

  • Because of possible formula hierarchies, the existing calculation triggers have to be split into identical timeslots.
    That means that during hierarchy creation, first ALL time slices of the triggers to be calculated are compared and the time slots are created in a way that all instances either have all or no values in a slot (which is represented in EFORMULA-TIMESLOTNR). 
  • High number of entries in EFORMULAHIER combined with “Number of intervals = 100” leads to many dispatch/execution jobs.
  • REEDMFICALCULATE submits EEDM_CALC_DISP via background (SUBMIT … VIA JOB) and does not wait for those jobs to finish, so the primary job ends independently.
  • Parallelization was configured without specifying batch servers for the defined number of jobs.

Resolution

  1. Align calculation periods across instances: group similar instances using formula groups and split calculations by groups to reduce time-slot fragmentation.
    Optimize parallelization: specify one or more batch servers and set an appropriate number of parallel processes; use several servers to distribute the load.
  2. Restructure job chains: separate the initial steps (including REEDMFICALCULATE) from later dependent steps into two jobs; start the second job with a delay after the first completes to allow background dispatch/execution jobs to finish.
  3. Verify and repair formula hierarchy structures: refer to Note 3044048 (REEDMFI_CHECK_HIERARCHY_CREATE); use Note 3261278 to delete obsolete hierarchy levels if required.
  4. Apply performance improvements relevant to EDM calculation workbench and related tables: refer to 3008588, 2996568, 2978484, 2972340, 2971586, 2971092, 2967624.
  5. If performance is still insufficient, collect and analyze a trace as described in Note 755977.

Symptom

You want to create a unit cost estimate for a sales order in the Create Sales Order app (VA01). When choosing the 'Unit Costing' button in the costing method popup, you receive an error message. The error message indicates that this functionality is not supported and advises you to contact SAP's technical support if you require this feature.

The system displays this error particularly when you use a requirement type that has no automatic derivation of the costing variant configured. That means the cost estimate would have no quantity structure.

Reproducing the issue:

  1. Navigate to Create Sales Order app
  2. Enter the sales order details:
    • Requirement Type without automatic costing variant derivation
    • Material 
  3. In the menu, choose Extras->Costing.
  4. In the popup window for the costing method, choose Unit Costing.
  5. You get an error message indicating that the functionality is not supported.

Other Terms

Unit Costing, Unit Cost Estimate, Sales Order Costing, VA01, Create Sales Order

Reason and Prerequisites

Unit costing for sales orders is not officially supported in SAP S/4HANA Public Cloud Edition. However, due to the system design, it was still technically possible to create a unit cost estimate for a sales order. Please be aware that cost estimates without quantity structure cannot be used in follow-up processes.

Solution

Please use product costing (with quantity structure) instead.

If your business process requires this functionality, please open an incident with SAP technical support on component CO-PC-PCP and describe your business requirement.

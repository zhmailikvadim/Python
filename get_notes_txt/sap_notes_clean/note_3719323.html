Symptom

  • Images in Email Campaigns do not reflect the dimensions specified through the Insert/Edit Image dialog.
  • Although the image appears correctly resized within the editor, the email received displays the image at its original size instead of the adjusted size.
  • The issue occurs when viewing the email in Outlook (classic).

Environment

  • SAP SuccessFactors Recruiting Management
  • SAP SuccessFactors Recruiting Marketing

Reproducing the Issue

  1. Navigate to Recruiting > Marketing.
  2. Create or Edit an email campaign.
  3. Insert an image, and set the Dimensions using the Insert/Edit Image dialog.
  4. Confirm that the resized image appears correctly in the editor.
  5. Click Review Email.
  6. Click Send Test.
  7. Enter the email address and send.
  8. Review the email received.
  9. Observe that the image is displayed at the original size, not the resized dimensions.

Cause

When dimensions are adjusted using the editor, the system generates width and height attributes that include the unit "px" (e.g., width="200px").

Some email clients do not render image dimensions correctly when the "px" unit is included. These clients expect numeric values without units (e.g., width="200").

Resolution

The system behaviour will be enhanced in an upcoming release.

Click on star to bookmark this article to receive updates about this issue. For more information about updates from subscribed KBAs, you can refer to the KB Article 2171560 - How to be notified of new or updated SAP Notes or KBAs.

Workaround

The issue does not occur in many modern or web‑based email clients. To avoid incorrect rendering, use one of the following approaches:

  1. Upload the image at the exact size needed so that no resizing is required within the editor.
  2. Resize the image directly in the editor by dragging its corners. This will generate width and height attributes values without units. However, if the dimensions are adjusted using the Insert/Edit Image dialog afterward, the generated attributes will change again.

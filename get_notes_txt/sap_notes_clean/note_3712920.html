Symptom

  • When editing an SAP Initiated system in IAS to update URLs for the Kingdom of Saudi Arabia (KSA) Private Data Center migration, the masked ariba.applications.api.key and Password credentials are automatically erased, preventing the configuration from being saved.
  • The issue occurs specifically for integrations categorized as SAP Initiated systems in IAS.

Reproducing the Issue

  1. Navigate to the IAS tenant.
  2. Locate the Ariba integration configured as an SAP Initiated system.
  3. Click the Edit button to modify the system's properties, such as the Open API URLs.
  4. Observe that the previously populated and masked fields for ariba.applications.api.key and Password are now empty.

Cause

The issue is caused by a specific behavior within IAS that affects integrations categorized as SAP Initiated systems. When a user clicks Edit on such a system, the user interface automatically erases the masked credentials (ariba.applications.api.key and Password) from the configuration fields. These credentials were provisioned automatically by SAP and are not known to the user, preventing the required URL updates for the KSA DC migration from being saved.

Resolution

  1. Copy the value from the User property within the Target System configuration in the IAS tenant.
  2. Navigate to the SAP Ariba API Developer Portal, Kingdom of Saudi Arabia Private datacenter.
    • This page will only be available once the migration has been completed.
  3. Use the copied User value to search for and locate the application under the Manage section.
  4. From the application details page in the portal
    1. Copy the Application Key visible on the page, corresponding to the ariba.applications.api.key property in IAS.
    2. Generate a new OAuth Secret (Password) by selecting Actions > Generate OAuth Secret > Submit. Copy and store the newly generated secret.
  5. Return to the IAS configuration screen.
  6. Paste the copied Application Key and the newly generated OAuth Secret into their respective fields (ariba.applications.api.key and Password).

Once the steps above are completed, the Open API URLs can be updated and the integration system can be properly saved:

    1. Update the Open API URLs as required for the migration:
      1. Current: api.mn2.ariba.com → New: api.ksaprv.cloud.ariba.com
      2. Current: mn2.openapi.ariba.com → New: openapi.ksaprv.cloud.ariba.com
    2. Click Save to complete the configuration changes.

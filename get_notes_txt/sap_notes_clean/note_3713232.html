Symptom

According to the official SAP Help documentation, when attempting to configure a Virtual Host or Custom Domain, calling the service URL https://<url-from-service-key>/apiportal/operations/1.0/Configuration.svc/VirtualHostRequests results in an HTTP 401 (Unauthorized) response

Environment

  • SAP API Management
  • SAP Integration Suite

Cause

The OAuth token that was generated belongs to a different service instance than the expected one that is described in here: Accessing API Management APIs Programmatically

Resolution

Please make sure that service key that is used for OAuth token generation is created with:

  • API Management, API portal service
  • apiportal-apiaccess plan
  • APIManagement.SelfService.Administrator roles

For further details and step-by-step guide for the creation please check SAP Help Page: Accessing API Management APIs Programmatically

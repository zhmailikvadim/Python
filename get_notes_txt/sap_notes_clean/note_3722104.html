Symptom

When using My Timesheet V4 app either for Time Entry or creating assignment, the value help result is not as expected. For example by entering receiver order related information, getting orders that are not released as well .

Environment

HCM Fiori My Timesheet V4

Reproducing the Issue

  1. Open My Timesheet V4 app
  2. Create a Time Entry or create assignment
  3. Use the value help by a field

Resolution

The CDS entity C_HCMTIMESHTASSGMTTP contains the field associations. 

As an example the 'receiver order' is defined in the entity:

..

association [0..1] to I_OrderBasicVH as _ReceiverOrder on $projection.ReceiverOrder = _ReceiverOrder.OrderID 

...

The I_ORDERBASICVH is not Fiori My Timesheet specific, it come from package VDM_PP_SFC (software component PP-VDM).

Need to check the standard I_ORDERBASICVH via SE16N, whether does it provide the expected result via SE16N or not?

When the result is the same via app and via SE16N:

  • there can be a wrong customer settings connected to this field
  • customer need to use own CDS view in the application

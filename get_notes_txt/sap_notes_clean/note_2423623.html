Symptom

You are using the DBA Cockpit with the database IBM DB2 for Linux, UNIX, and Windows and you have virtualized some empty tables using the function "Convert Empty Tables" on the "Virtual Tables" screen.

When you run the consistency check provided by the DBA Cockpit screen "Diagnostics -> Missing Tables and Indexes", you find that some secondary indexes of virtualized tables are listed in the folder named "Objects missing in the database".

Other Terms

DBACOCKPIT, DB02, ST04, Table Virtualization

Reason and Prerequisites

The problem is caused by a program error.

The DB13 job performing the virtualization of tables fails to generate a CREATE INDEX statement for secondary indexes which are meant to be created only for specific database platforms.

The symptom occurs for certain transparent tables of an SAP system connected to an IBM DB2 for Linux, UNIX, and Windows database. These tables have secondary indexes defined for some but not all database platforms in the data dictionary. This option occurs only very rarely.

Solution

The solution is shipped as part of the Support Package. Alternatively, you can implement the attached correction instruction.

If you have enconuntered this symptom, follow these instructions to overcome the problem:

  1. Make sure that the correction instructions attached to this SAP Note are implemented in your system.
  2. Materialize all virtual tables that are reported to have missing secondary indexes using the function "Materialize" on the "Virtual Tables" screen.
  3. Virtualize the tables again using the function "Convert Empty Tables" on the "Virtual Tables" screen.

After this procedure, the missing secondary indexes for virtualized tables that were reported on the screen "Diagnostics -> Missing Tables and Indexes" should disappear.

Note that the ABAP report RSDB6_REPAIR_VIRTUAL_TABLES described in SAP Note 2186737 shares some of the behavior of the table virtualization DB13 job, which may lead to the same symptom. This report is also corrected with this present SAP Note.

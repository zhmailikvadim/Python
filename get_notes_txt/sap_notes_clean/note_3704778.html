Symptom

  • Data locking settings are transported with the model regardless of deselection of them.

Environment

  • SAP Analytics Cloud
  • SAC Content Network

Reproducing the Issue

  1. Login to SAP Analytics Cloud (SAC).
  2. Navigate to Transport > Export.
  3. Click on "Create New Export".
  4. Select a model that includes data lock and go to the Data Locking type to exclude its data locking.
  5. Click "Next" and input necessary information, then click "Next".
  6. On the Review and Export page, verify that no data locking is present in the package, then click "Export" to create a package.
  7. Once the export is created, select the export.
  8. Click on "Edit Details".
  9. Notice that the excluded dependency from data locking is displayed.

Cause

  • The symptom is by design

Resolution

The private dimensions and data locking objects are hidden in UI by default and are added automatically to the package as they are private dependencies of the model.

If you wish to export only the model, please follow below steps:

  1. Navigate to "System" -> "Administration" in SAP Analytics Cloud.
  2. Turn on the setting "Display Private Dimensions in Content Network".
  3. Save.

This will list private objects in the UI, include only the model in the package and create an export.
The private dependencies will not be auto-included in the package.
If this behavior is no longer needed, turn off the setting.

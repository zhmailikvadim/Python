Symptom

In IDM 8.0, if you create or modify an attribute and if this attribute has Add/Modify attribute event task specified, it might happen that a second modify task is triggered for the attribute modification.

In some cases, if you modify for example MX_PASSWORD, it might result that a second Password Reset tasks is executed to the same repository and the second task might fail, because IDM is trying to set the same password again.

Other Terms

IDM, Identity Management, IDM, Provisioning, Attribute modification

Reason and Prerequisites

You are using SAP Identity Management 8.0 SP06.

Solution

Apply the attached patch provided in the Support Packages & Patches section of this SAP Note.

Make sure all components are updated to the corresponding SP level before applying the patch.

Symptom

You have used MDG-F Consolidation Governance function and created FS Items and you expect to use the FSI Item as Group Account for GL Account(Chart of Account). When you use the value help of Group Account field in OVP of GL Account(Chart of Account), you realize that not all FSI items will be shown in the value help result list.

Environment

  • Master Data Governance on S/4 HANA on-premise 1511
  • Master Data Governance on S/4 HANA on-premise 1610
  • Master Data Governance on S/4 HANA on-premise 1709
  • Master Data Governance on S/4 HANA on-premise 1809
  • Master Data Governance on S/4 HANA on-premise 1909
  • Master Data Governance on S/4 HANA on-premise 2020
  • Master Data Governance on S/4 HANA on-premise 2021
  • Master Data Governance on S/4 HANA on-premise 2022
  • Master Data Governance on S/4 HANA on-premise 2023

Reproducing the Issue

  1. Create an FSI Item in MDG-F.
  2. Create a new GL Account(Chart of Account) in MDG-F.
  3. Open the value help of Group Account field and check the list of account id returned.

Cause

There is standard logic to determine if FS item is valid to be a group account for GL Account.

Resolution

  • Within MDG-F everything will be read from MDG-F staging tables which can be verified with T-code MDG_DATA_MODEL and provide data model value 0G.
  • FSI Items which is know as Group Account is saved in the check table of entity FSI.
  • The Group Account value will respect the consolidation group chart of account configuration in path: SPRO->Financial Accounting->General Ledger Account->Master Data->G/L Accounts->Preparations->Edit Chart of Account List. which means if the GL Chart of Account used is ABC and it has the group chart of account value configured as AB. Then only the FSI Item created with Chart of Account AB will be considered valid for GL Account under chart of account ABC.
  • And there is standard validation also in method CL_MDGF_DQUERY_OVS_FSI->PROVIDE_RESULT_TABLE: The Account type(ACCTYP) should be same for the GL Account and FSI Item and the group account must not be statistical item.

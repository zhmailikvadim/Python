Symptom

Outbound Idocs are not being sent and remain in IDoc status 03.

Other Terms

ASANWEE, ASAAEMEE, IDocs, 03

Reason and Prerequisites

A new authorization check was introduced to function module /ASADEV/ACI_IDOC_PORT_TRIGGER. If there is no authorization, then IDocs were not processed further.

Solution

Implementing this note enhances the authorization check for a global authorization setting.

Implement the note and perform the manual steps:

1. Go to transaction n/ASADEV/ACI_PART.
Create or update the entry with the following value:
- ACI Global Constant = ACI_AUTH_GLOBAL_IDOC_PORT

2. Go to transaction n/ASADEV/ACI_PAR.
Create or update the entry with the following values:
- ACI Global Constant = ACI_AUTH_GLOBAL_IDOC_PORT
- Field Value = X

If no global authorization is required, ensure that users are assigned the necessary authorization for object /ASADEV/RC.

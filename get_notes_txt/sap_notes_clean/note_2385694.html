Symptom

From 31 October 2016 you have to use a new format when sending payment files containing euro payments. This format is ISO 20022 XML.

According to the EU Regulation No. 260/2012 Norway shall use the SEPA payments instruments from 31 October 2016 for payments in EUR currency.

General SAP supports the introduction of SEPA through the subsequent delivery of the XML payment medium formats. This correspondent to the ISO20022 standard system and recommendations of the European Payments Council (EPC). The pain.001.001.003 is recommended for credit transfers.

Currently there are no country specific requirements known for Norway regarding the pain.001.001.003 format.

Other Terms

NO, Norway, CGI, CGI_XML_CT, SEPA Payments, BAdI, FI_CGI_DMEE_EXIT_W_BADI

Reason and Prerequisites

Reason for the implementation is EU regulation of the payment format coming into force latest on 31.10.2016.

Solution

CGI_XML_CT DMEE tree was delivered for pain.001.001.003 format by SAP previously. The collective SAP Note 2253571 contains up to date information about the CGI CT payment format.

Please implement the SAP Notes mentioned within the overview of the SAP Note 2253571. Important to have the latest version as the DMEE tree refers to exit function modules implemented by these SAP Notes.

Version pain.001.001.002 is no longer supported by SAP. Therefore for banks that still support only the older version pain.001.001.002 you need to copy and modify the DMEE tree accordingly in your own project.

Payments originated in HCM

Please confront guidelines attached with the file "Customizing for salary payment.pdf" to this note for the implementation of payments originated in HCM module.

Customer BAdI for SEB and Nordea banks

Bank specific changes are necessary in Customer BAdI, check attached document "CGI_w_Exits_Implementation_Norway.pdf".

Symptom

  • After distributing a program from one tenant to another, the "Set value" block does not retain its configured value in the target tenant.
  • The field and value being set exist in both tenants, and missing dependencies are not the issue.

Environment

SAP Emarsys Customer Engagement

Reproducing the Issue

  1. Create a program in tenant A with a "Set value" block.
  2. Configure the block to set Field X to Value Y.
  3. Ensure the set values exist in both tenants.
  4. Distribute the program to tenant B.
  5. Open the program in tenant B. Observe that the "Set value" block requires the value to be configured again.

Cause

The internal IDs of custom fields and their choice values are tenant-specific, even if the field name and visible value are identical in both tenants.

Resolution

  1. Understand that during program distribution, field structures are transferred but the underlying internal IDs of custom fields and their choice values differ between tenants.
  2. Reconfigure the "Set value" block manually in the target tenant to ensure accurate data mapping.

Keywords

set value block, program distribution, tenant-specific configuration, internal IDs, automation, field structure transfer, manual reconfiguration, data mapping, SAP Emarsys

Symptom

Sales Order Change – Missing F4 Help

In Sales Order Change scenarios, Joule does not display the required F4 help lists when configuration values contain special characters such as " or \.

When asked to change for either delivery block reason/ billing block reason /change of payment terms/ sales document rejection reason  in the sales order , Joule doesn't return List diplay with respective F4 helps .

This issue is observed in the following scenarios:

  • Changing Delivery Block Reason

  • Changing Billing Block Reason

  • Changing Payment Terms

  • Changing Sales Document Rejection Reason

When special characters are present in the code or description of these configurations, Joule fails to return the list display with F4 help.

Sales Order Fulfilment Monitoring (SOFM) – Carousel Not Displayed

In SOFM issue scenarios, Joule fails to load the carousel display (zero cards are shown) for the following issue types:

  1. Delivery Block in Sales Order

  2. Delivery Issue in Sales Order

This occurs when the sales order data contains special characters such as " or \ in:

  • Material descriptions

  • Delivery block reason codes or descriptions

Reason and Prerequisites

In both Sales Order Change and Sales Order Fulfilment Monitoring (SOFM) scenarios, Joule fails to handle special characters (for example " and \) present in backend configuration .

These special characters are not properly escaped while forming JSON responses, resulting in invalid JSON structures. As a consequence:

  • F4 help lists fail to render in Sales Order Change scenarios.

  • Carousel displays fail to load (showing zero cards) in SOFM for delivery block and delivery issue cases.

Reason : Joule does not properly escape special characters such as " and \ while constructing JSON responses from backend configuration and transactional data.

This leads to:

  • F4 help lists not rendering in Sales Order Change scenarios when delivery block reason, billing block reason, payment terms , or rejection reason 'codes/descriptions' contain special characters.

  • Carousel display failures (zero cards shown) in Sales Order Fulfilment Monitoring (SOFM) for delivery block and delivery issue cases when sales order data (for example, material descriptions or delivery block reasons) contains special characters.

Solution

The issue was caused by special characters (for example " and \) present in backend configuration and transactional data not being properly escaped while constructing JSON responses. This resulted in invalid JSON structures, which caused UI rendering failures in Joule.

The implementation corrects JSON handling by ensuring that special characters are properly escaped before forming responses consumed by the Joule UI.

As a result:

  • Sales Order Change scenarios

    • F4 help list rendering is now stable when special characters exist in:

      • Delivery block reason (code and description)

      • Billing block reason

      • Customer payment terms

      • Sales document reason (header level)

      • Sales document rejection reason (item level)

    • The F4 help list is displayed correctly, and sales order updates complete successfully.

  • Sales Order Fulfilment Monitoring (SOFM)

    • Carousel rendering is now supported for the following issue categories when special characters are present in material descriptions or delivery block reasons:

      • SO02 – Delivery Block in Sales Order

      • SO05 – Delivery Issues in Sales Order

    • The carousel now displays the relevant cards instead of failing with zero cards.

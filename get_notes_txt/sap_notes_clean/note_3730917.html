Symptom

When running SUM Toolbox and selecting option "Uptime Table Selection for Downtime Optimized DMO", the following dump occurs: 

Category                     ABAP Programming Error
Runtime Errors            SAPSQL_PARSE_ERROR
Except.                       CX_SY_DYNAMIC_OSQL_SEMANTICS
ABAP Program             RSTBX_DODMO_SELECT
Application Component BC-UPG-TLS-TLA

Error analysis
    An exception has occurred which is explained in more detail below. The
    exception, which is assigned to class 'CX_SY_DYNAMIC_OSQL_SEMANTICS' was not
     caught and
    therefore caused a runtime error. The reason for the exception is:
    The current ABAP program attempted to execute an Open SQL statement
    containing a dynamic entry. The parser returned the following error:
     ""PUTTBX_S4T" is not defined in the ABAP Dictionary as a table, projection
     view, or database view."

Environment

SUM Toolbox(STBX)

Reproducing the Issue

  1. Run SUM Toolbox(STBX) and select "Uptime Table Selection for Downtime Optimized DMO".
  2. Observe that a dump occurs and ST22 shows errors related to table PUTTBX_S4T not being defined in the ABAP Dictionary. 

Cause

The table PUTTBX_S4T exists at the database level but has no active definition in the ABAP Dictionary (no active runtime object)

Resolution

  1. Confirm in SE11 that PUTTBX_S4T is not active and that there is no active runtime object.
  2. Create a backup of the database table PUTTBX_S4T as needed for audit or recovery purposes.
  3. Remove the leftover table at the database level (drop table PUTTBX_S4T). If present in SE11, ensure it is deleted from the dictionary as well.
  4. Rerun the SUM Toolbox(STBX) and select "Uptime Table Selection for Downtime Optimized DMO" and confirm that no dump occurs.

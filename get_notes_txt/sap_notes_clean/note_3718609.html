Symptom

In the "Benefits Eligibility Date Configuration" there are 2 ages configured in the "set duration from base date", but only the tracker for the first age is created. 

Environment

  • SAP SuccessFactors HCM Suite
  • SAP SuccessFactors Employee Central

Reproducing the Issue

  1. Navigate to Benefits Admin Overview
  2. Select Benefits Eligibility object and open the configured option
    • Verify that the trackers should be created for 2 ages, like 21 years and 25 years, for example
  3. Create a new "Evaluate Eligibility Based on Benefit Master Data Change" with Evaluation Scenario "Determine Ineligibility Date for Enrolled Dependents"
  4. Save the configuration
  5. Run the job and observe that only the tracker for the first age is created

      Cause

      The system behavior is designed to create a tracker for the first eligibility age (21 years in this example) if the dependent has not yet reached this age.

      Once the dependent reaches the first eligibility age, the job will process the existing tracker and create a new tracker for the next eligibility age (25 years).

      Resolution

      1. Ensure that the Benefits Eligibility object is correctly configured with the desired age criteria (e.g., 21 and 25 years).
      2. If the dependent has not yet reached the first eligibility age, the tracker for the first age will be created. When the dependent reaches the first eligibility age, the job will process the existing tracker and create a new tracker for the next eligibility age.
      3. If the dependent has already reached the first eligibility age, the tracker for the next eligibility age will be created directly.
      4. For further details, refer to the SAP Help Portal guide on creating Benefits Eligibility objects using standard base dates.

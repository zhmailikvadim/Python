Symptom

This SAP Note delivers functionality to delete simulation data stored in the following four tables:

T5Q_STP2_ER
T5Q_STP2_EE
T5Q_STP2_PAY
T5Q_STP2_LOG

Currently, when the Single Touch Payroll Phase 2 report (TCODE PC00_M13_STP_GEN or PC00_M13_STP_GEN_CE) is executed for an employee in Submit Event or Update Event mode with Test Run and Update Database selected, SAP stores simulation data in these respective tables.

Other Terms

T5Q_STP2_ER, T5Q_STP2_EE, T5Q_STP2_PAY, T5Q_STP2_LOG, Simulation data

Report: RPCSTDQ0

Transaction Code:PC00_M13_STP2_DE

Reason and Prerequisites

Version Reason for Version Creation
     01

 Pilot Release
     02

 General Release for Customers

Solution

Code changes required for deleting simulation data stored in the T5Q_STP2_ER, T5Q_STP2_EE, T5Q_STP2_PAY, and T5Q_STP2_LOG tables have been implemented in the new report RPCSTDQ0, along with the transaction code PC00_M13_STP2_DE.

To get the code corrections immediately,

  • Apply the correction instructions attached to this SAP Note.

 OR

  • Apply the appropriate HR Support Package (HRSP) mentioned in the ‘Support Packages and Patches’ section of this SAP Note.

As a general rule SAP recommends that you implement the Support Package specified for your release. However, if you need to install a solution earlier, use the SAP Note Assistant to implement the correction instruction. You can find more information about the SAP Note Assistant in SAP Support Portal, under SAP Note Assistant.

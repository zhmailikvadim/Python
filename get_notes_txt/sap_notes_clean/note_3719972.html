Symptom

  • After the 2H 2025 release, the Print Form Service (PFS) was migrated to Cloud Foundry.
  • Onboarding/offboarding documents cannot be uploaded in the instance.
  • Error message: "Http Status Code 403; File could not be updated in PFS."
  • Instance refresh has been done recently.

Environment

SAP SuccessFactors onboarding

Reproducing the Issue

  1. Navigate to Admin Center > Onboarding and Offboarding Document Templates.
  2. Attempt to add a document template.
  3. Observe the error: "Http Status Code 403; File could not be updated in PFS."

Cause

The presence of PFS URL and username in provisioning settings, copied from the source instance during the refresh, prevented the migration to Cloud Foundry.

@Support Engineer please see internal memo

Resolution

  1. Raising the Script for resetting the sys config flag.
  2. Disable Print Forms Generation flag and remove the URL, user id and password from provisioning
  3. The PFS migration job will run every 4 hours, recommended to wait for 24 hours.

Symptom

You use transactions that use SADL to generically read only data from the database for which the executing user is authorized. This includes, but is not limited to, ME* transactions, such as ME2L.

However, the required data is displayed incompletely or not at all in the affected transactions.

Reason and Prerequisites

Authorizations with inconsistent data exist.
We refer to the following data constellation as inconsistent:

  • In addition to the full authorization (*) in a field of an authorization, there are other dedicated values for the same field in the same authorization.

Solution

Implement the attached correction instructions or import the specified support package.

Remark about the solution:

Neither transaction PFCG nor PFCGMASSVAL can be used to create such inconsistencies described in the "Reason and Prerequisites" section.

The correct solution is to correct the authorization data, but this is not done automatically with this correction.
To do this, you should save the authorization data of the affected role again and regenerate the profile.

With this correction, the optimization option in the function module SUSR_USER_AUTH_FOR_OBJ_GET now ignores these inconsistencies at runtime and works only with the same data as the PFCG would create.

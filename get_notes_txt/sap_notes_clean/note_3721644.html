Symptom

  • SAC frontend error message: 
    An error occurred while retrieving the data locks for the visible cells.
    Correlation ID: xxxxxx when loading a story. 
  • Console log: Firefly returned with errors  - "Firefly returned with errors " DataRegionLocksProvider .
  • The issue occurs when opening a complex story contains lots of widget with linked analysis.
    The issue does not occur if you create a new test story that only contains a table with the same setting.

Environment

SAP Analytics Cloud

Reproducing the Issue

  1. Open the story.
  2. Select the table.
  3. Enable the data locking from the table menu -> show/hide.
  4. Wait for several minutes and the data locking error will be observed.

Cause

The issue is caused by the complexity of the story configuration. The linked analysis mechanism used extensively in the story contributes to the generation of large filters, resulting in performance and timeout issues when retrieving the data locks.

Resolution

  1. Disable data locking for the problematic table by selecting the "Ignore Data Locks" option. The users will be able to perform manual data entry on the table cells but the locks will still be enforced on publish. 
  2. If disabling data locking is not an option, consider the following:
    • Reduce the number of input controls and remove unused models in the story.
    • Adjust the story configuration to query and retrieve less data from the backend.
    • Use Data Access Control (DAC) to restrict dimension-level access for users.
    • Disable planning capabilities for the table to make all cells read-only.
  3. Review and optimize the story configuration to reduce the complexity and improve performance.

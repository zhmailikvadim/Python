Symptom

After you switch to sending with eXTra 1.5, the following error occurs when you send B2A entries as a job:

<BTRNR> - Company number not in certificate list exists BN<BNRAG>.pse
Job canceled after system exception ERROR_MESSAGE 

As a result, the job terminates.

However, the manual sending of files using the B2A Manager works without problems.

Other Terms

 eXTra 1.5, H99_B2AMANAGER

Reason and Prerequisites

This problem is caused by an old or incorrect B2A entry for a company number that does not exist in the certificate list of your PSE.

This entry causes the error <BTRNR> - Company number not in certificate list previously BN<BNRAG>.pse, which is no longer caught correctly after the conversion and therefore leads to the termination of the job. 

Solution

After you implement the corrections, the error is caught correctly, the job no longer terminates, and the following files can be sent.

Implement the attached correction instructions or import the specified support package. After you import the HR Support Package or implement the correction instructions, you do not have to perform any further follow-on activities.

Optional:

You can identify the problematic B2A entry as follows:

Transaction: SE16

Table name: P01SV_B2ATRNS

Enter the affected document category in the DOCTY field, enter the company numbers <BTRNR> from the error message in the BTRNR field, and choose Execute.

In the B2A Manager (transaction PB2A), then select the period in such a way that the affected B2A entry is displayed. This should have the status New/New.

If the entry is no longer required, you can delete it using the report HR-B2A: Reorganization Program for B2A Application (H99_B2AREORG).

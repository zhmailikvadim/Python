Symptom

  • Login to the CX AI Toolkit fails with HTTP 403 Forbidden error. 
  • The CX AI Toolkit is bound to the wrong identity authentication tenant. 
  • Removing the previous identity authentication connection does not resolve the binding. 



Environment

SAP CX AI Toolkit

Reproducing the Issue

  1. Attempt to sign in to the CX AI Toolkit. 
  2. Observe that authentication fails with HTTP 403 Forbidden. 
  3. Verify that the toolkit shows or uses an identity authentication tenant different from the intended one.

Cause

  • The CX AI Toolkit was bound to the wrong identity authentication tenant, resulting in failed authentication.

Resolution

  1. Confirm the correct identity authentication (IAS) tenant that the CX AI Toolkit must use. 
  2. In the IAS admin console, open the Toolkit OIDC application and navigate to Client Authentication. 
  3. Copy the Client ID and generate a new Client Secret if none is available (record the secret immediately). 
  4. Reconfigure the CX AI Toolkit to use the correct IAS tenant by updating the binding with the provided Client ID and Client Secret. 
  5. Test access to the CX AI Toolkit and verify that login succeeds and the correct IAS tenant is shown.

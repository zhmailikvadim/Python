Symptom

After executing Initialization, Validation, Release, and Approval in Fiori App F-2620:

  • Data is correctly posted and visible in table ACDOCA
  • Data is not visible in table ACDOCC
  • No errors are raised during processing
  • Data inconsistency exists between ACDOCA and ACDOCC

Environment

  • SAP Business Planning and Consolidation Embedded with RTC (Real-Time Consolidation), version for SAP NetWeaver

Reproducing the Issue

  1. Open the Fiori application Fiori App F-2620.
  2. Select a consolidation entity and reporting data.
  3. Execute the following steps:
        Initialization
        Validation
        Release
        Approval

  4. Verify data in table ACDOCA:
        Data is available
        Verify data in table ACDOCC:
        Data is not available

  5. Execute transaction LISTCUBE for the relevant virtual provider:
        Data is not visible

Cause

The issue is caused by an inconsistency in the data model:

  • A required field (e.g., RBUKRS – Company Code) is missing in the underlying BPC Union HANA view
  • The virtual provider cannot correctly retrieve data due to the missing field
  • RTC configuration is not fully aligned (e.g., after system upgrade or transport)

Resolution

  1. Enhance the BPC Union HANA view:
        Add the missing required field(s) (e.g., RBUKRS)
        Activate the HANA view
  2. Verify RTC configuration:
        Ensure relevant settings are correctly maintained
        Transport configuration if required
  3. Re-activate the RTC model:
        Ensures proper mapping between the HANA view and the virtual provider
  4. Validate the correction: 

            Execute transaction LISTCUBE for the virtual provider
            Confirm that data is visible

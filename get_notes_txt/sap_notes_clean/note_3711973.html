Symptom

In Onboarding 1.0, a candidate may have two different User IDs/accounts, such as one created by SAP SuccessFactors and another generated by an external system. When this occurs, there is uncertainty about whether updating or aligning the User IDs could cause issues in the live Onboarding 1.0 environment. Clarification is needed on whether modifying these User IDs may lead to system impact or downstream effects.

Environment

SAP SuccessFactors Onboarding 1.0

Resolution

As confirmed by Document Migration team, updating the BizX system ID or the SHARP user ID in ONB 1.0 does not cause any major functional impact. This update simply informs the system that both records correspond to the same candidate.

After the ID update, all information currently visible in BizX and ONB 1.0 will remain unchanged. The only difference is that the system will now recognize the two records as belonging to the same candidate, ensuring proper data linkage and consistency.

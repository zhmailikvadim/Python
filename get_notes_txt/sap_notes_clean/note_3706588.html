Symptom

The commitment values for a Project / WBS Element are displayed in double the amount compared to the actual value in other apps, after having created a Purchase Requisition / Purchase Order

Environment

SAP S/4HANA Cloud Public Edition

Cause

The affected Purchase Requisitions / Purchase Orders are created using the GL account that has more than one tag related to Budget Availability Control Profile being used for the project (e.g account is assigned more than one of these following tag: ACT_COST, PSQBUDGET, PRSLS_STCK, any other tags that has parent tag as ACT_COST and/or PSQBUDGET...)

Resolution

Ensure that the GL account used for posting PRs/POs contains only 1 tag related to the Budget Availability Control Profile being used for the project.

  • Refer to: Maintain Budget Availability Control Profile for Projects.

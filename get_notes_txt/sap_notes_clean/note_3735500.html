Symptom

  • OAuth refresh tokens saved in User Settings are not refreshed on use, and instead expire after 90 days, even if they were used before 90 days expiry.
  • The below error may be seen:
    Error Message: Error occurred when requesting an OAuth Access Token from Authorization server (for CUID <NAME>), cause: Error returned from OAuth authorization server. [HTTP ResponseCode - 400]: {"error":"invalid_grant","error_description":"AADSTS700082: The refresh token has expired due to inactivity. The token was issued on 2024-09-05T07:12:56.0151498Z and was inactive for 90.00:00:00}

Important: an incomplete correction was delivered with SAP Note 3595099 which doesn't resolve issue and still leads to errors for end-users. Please consider Support Package and Patches from current SAP Note for official solution.

Other Terms

STS, BILP

Reason and Prerequisites

Environment

  • SAP BusinessObjects Business Intelligence Platform 4.3 Support Package 3 and upper versions
  • oAuth 2.0
  • Google Drive / Microsoft OneDrive

Steps to duplicate

  1. Generate a refresh token for a Microsoft OAuth server in the User Settings (SAP BI Launchpad),
  2. Perform OAuth workflows successfully (e.g. schedule a document with destination to OneDrive),
  3. After 90 days, the workflow starts failing,
  4. Generate a new refresh token manually in the User Settings,
  5. The workflow starts working again.

Cause

On Microsoft side, new refresh tokens get created upon every use (https://learn.microsoft.com/en-us/entra/identity-platform/refresh-tokens#token-lifetime).
However the new tokens are not stored in the User settings on SAP Business Intelligence side.

Solution

This issue is fixed in the patches listed in the "Support Packages & Patches" section below.

Due to different internal processes, the "Support Packages & Patches" section of the SAP Note will be populated as below:

  • for fixes delivered in "Support Packages": immediately (even if the "Support Package" is planned to be released several months later),
  • for fixes delivered in "Patches": ONLY once the "Patch" is released AND available on SMP (not before).

For Business Intelligence Platform maintenance schedule and strategy see the Knowledge Base Article 2144559 in References section.

Workaround

End-users should generate a new refresh token in User Settings (within SAP BI Launchpad).

Symptom

  • SD – Condition record for free goods migration loads successfully (100%) in the Migrate Your Data app, but the records do not appear in the Manage Free Goods app.
  • The data is not visible or active in the system after migration.

Environment

SAP S/4HANA Cloud Public Edition

Reproducing the Issue

  1. Open the app Migrate Your Data – Migration Cockpit.

  2. Select or create a migration project.

  3. Choose the migration object SD – Condition record for free goods.

  4. Upload a completed migration template.

  5. Execute Simulation → No errors.

  6. Execute Migration → Status shows 100% successful.

  7. Open the app Manage Free Goods – Sales.

  8. Search for the migrated records.

Result:
The migrated records are not displayed in the app.

Cause

This behavior is standard system design in SAP S/4HANA Cloud Public Edition.

The reason is architectural:

  • The migration object SD – Condition record for free goods writes data directly to the pricing condition persistence layer.
  • The Fiori app Manage Free Goods – Sales reads data via a CDS-based application layer.
  • These two technical layers are not automatically synchronized.
  • As a result, records migrated via the Migration Cockpit are not exposed in the Fiori app.

Additionally:

  • There is currently no public API available for bulk creation of free goods condition records in SAP S/4HANA Cloud Public Edition.
  • The Manage Free Goods – Sales app does not provide a mass upload (spreadsheet import) capability.

Resolution

This behavior is currently standard in SAP S/4HANA Cloud Public Edition.

Available options:

  • Free goods condition records can be created manually using the Manage Free Goods – Sales app.
  • There is currently no supported mass upload function for free goods via:
    • Migration Cockpit (for Fiori visibility),
    • Public API,
    • Spreadsheet import in the app.

If mass maintenance functionality is required, customers can submit an enhancement request via:

SAP Customer Influence Portal

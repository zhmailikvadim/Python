Symptom

  • Uninstallation of a Data Product is blocked when it is referenced by views or analytical models; removal of all dependent artifacts is required before uninstall can proceed
  • After a new column is added in the source system dimension, the installed Data Product does not show the new field
  • Reinstalling the Data Product after uninstall does not display the new fields in the structure
  • Entire data package was deactivated to check for new fields
  • No specific error messages were provided

Environment

  • SAP Datasphere
  • SAP Business Data Cloud

Reproducing the Issue

  1. Install Data Packages from SAP Business Data Cloud Cockpit
  2. Build Analytical models and/or views that consume the installed standard data products.
  3. Add a new column to a source dimension.
  4. Attempt to uninstall the Data Product; uninstall is blocked due to existing dependencies.
  5. Remove dependent artifacts to allow uninstall, then reinstall the Data Product.
  6. Observe that the new field is still not visible; deactivate the Data Packages to verify fields.

Cause

Resolution

Updating custom fields currently requires an uninstall-and-reinstall cycle.

Better Approaches Available Now:

  • For adding custom fields or extending field lengths: You can already use the manual "Update" option without uninstalling.

  • For removing or modifying custom fields: An uninstall is currently required, as the update process doesn't yet support incompatible changes.

Future Improvements: We are planning to improve support for Data Product changes. We cannot yet provide a timeline for this enhancement, as it's still in development.

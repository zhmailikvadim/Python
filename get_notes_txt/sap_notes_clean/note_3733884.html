Symptom

Session times out after 30 minutes in certain applications, despite maintaining higher session timeout parameters in the Manage Launchpad Settings app.

Environment

SAP S/4HANA Cloud Public Edition

Resolution

The session timeout for SAP GUI apps running in the Fiori Launchpad consists of two parts:

  • UI session
  • Back-end session

The UI-side timeout for the SAP Fiori Launchpad is determined by the following parameters in the 'Manage Launchpad Settings' app:

  • SESSION_TIMEOUT_INTERVAL_IN_MINUTES
  • SESSION_TIMEOUT_REMINDER_IN_MINUTES
  • SESSION_TIMEOUT_STOP_DATA_REFRESH

However, there is a fixed timeout of 30 minutes in the ABAP back end for S/4HANA Public Cloud, which applies to 'SAP GUI' and 'Web Dynpro applications'. This value cannot be changed.

Therefore, the application session times out after 30 minutes.

See Also

More information on default configurations can be check in the Security Recommendations | SAP Help Portal (filter by topic: "Session Timeout: UI" in the table).

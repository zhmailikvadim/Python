Symptom

  • Users cannot view signed onboarding documents in People Profile after the new hire is converted to employee and onboarding is archived.
  • The employee can access the onboarding checklist, but no signed documents are available in People Profile.
  • The signature provider is DocuSign.
  • No error message is displayed.

Environment

SAP SuccessFactors Onboarding

Reproducing the Issue

  1. Proxy as a newly hired employee who completed Onboarding 2.0 with DocuSign and has been converted to employee.
  2. Open the employee’s People Profile (Employee Profile).
  3. From All Actions, attempt to open or view onboarding documents/checklist.
  4. Observe that no signed onboarding documents are displayed in People Profile.

Cause

  • When DocuSign is used as the eSignature provider, signed documents are stored in DocuSign, not in SAP SuccessFactors.
  • SAP SuccessFactors stores only document metadata and signature status (for example, completed or pending).
  • As a result, the system does not have access to the finalized signed PDF for display in People Profile.

Resolution

  1. Acknowledge that this is expected behavior: viewing finalized signed PDFs directly in People Profile is not supported when DocuSign is the eSignature provider.
  2. Retrieve signed documents from the DocuSign account (envelope management) or from DocuSign email notifications.
  3. If visibility within SAP SuccessFactors is required, manually upload the signed PDF files to Employee Files.
  4. Alternatively, implement a custom integration that retrieves completed DocuSign documents and stores the signed PDFs in SAP SuccessFactors for viewing.

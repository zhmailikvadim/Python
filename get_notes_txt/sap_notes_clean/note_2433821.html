Symptom

  • The ESP project is using a File/Hadoop CSV Input Adapter to fetch data from a file location on the local hard disk.
  • In the project's CCL code, the "File" property is set to '*.csv', in order to use a wildcard search: ie file = '*.csv'
  • In the directory where the file is located, there are some files whose filenames do not have any extension.
  • The adapter is unexpectedly processing those files whose filenames do not have any extension.

Environment

SAP Event Stream Processor (ESP) 5.1 SP 11

Cause

The ESP Documentation Guide:  SAP Event Stream Processor 5.1 SP11: Adapters Guide → Documentation → File/Hadoop CSV Input Adapter: Managed Mode Properties is missing guidance on how to specify wildcards.

Resolution

To specify wildcards, use a 'regex' pattern. This pattern must include an escape character, "\", before meta characters in order to include them in the pattern string.

For example: 
 
The following ATTACH ADAPTER CCL Statement will select only files that end with ".csv", by setting the File property to ".*\.csv": 
 
ATTACH INPUT ADAPTER File_Hadoop_CSV_Input1 TYPE toolkit_file_csv_input
TO input_csv_schema1_window1
PROPERTIES
 dir = 'C:/csvinputtestdata' ,
 file = '.*\.csv' ,
 dynamicMode = 'dynamicPath';

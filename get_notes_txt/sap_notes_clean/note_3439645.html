Symptom

In transaction ME2ON, the values maintained in multi-selection on search screen are reset to only one value after executing search.

The same behavior can be found with many other MM transactions, i.e. ME2L, ME9F.

Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.

Environment

  • Materials Management (MM)
  • SAP ERP Central Component
  • SAP ERP
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA

Reproducing the Issue

  1. Enter multiple values for selection fields, i.e. Suppliers, Plant, on the selection screen of ME2ON
  2. Press execute (F8)
  3. Display result as normal
  4. When back to selection screen (F3), the entered Suppliers in step 1 are reset to only one value on selection screen automatically

Cause

SAP standard design.

Resolution

To avoid too much memory consumption, reports behind transactions like ME2O, ME2ON, ME9F or ME2L clear all values from the previous run once you leave the selection result screen. Only one value will be saved with the corresponding memory id like mat (material) or lif (supplier).

As workaround, you can use the variant feature to save the values from the selection screen.

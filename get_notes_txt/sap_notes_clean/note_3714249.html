Symptom

  • When configuring a new Forecast Error (FE) profile for Inventory Optimization (IO) in 2602, value help for Lag does not show any Lag values.
  • All this despite the Lag attribute being defined correctly in the Application Mappings app (for HPA-based plan areas) or the Business Meaning is assigned as Lag (for non-HPA plan areas), and the attribute being assigned to the planning area and the proper planning levels, etc, and the Lag master data containing actual values.

Environment

SAP Integrated Business Planning for Supply Chain version 2602

Reproducing the Issue

  1. Create a new Forecast Error profile for the planning area.
  2. Set the calculation level [i.e. Month | Product | Location | Customer Group (MTHPRODLOCCUSTGROUP)].
  3. Define a forecast error group measure.
  4. Select the Sales Key Figure (i.e. I_IOSALES).
  5. Select the Forecast Key Figure (i.e. I_IOLAGFCST).
  6. Select the option to 'Use Lag'.
  7. Select the value help to pick the desired lags, and observe that no data is available for selection.

Cause

The issue was identified as a regression in version 2602, relating to a UI5 plugin.

Resolution

Issue has been fixed and will be in release 2602 HFC05, all systems are scheduled to get it the weekend of Feb 27-28/Mar 1.

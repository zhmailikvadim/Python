Symptom

While trying the TR release in Dialogue mode the Release ended with timeout error

Reproducing the Issue

Try to release the TR in Dialogue mode

Cause

Huge number of objects in the TR

Resolution

  1. Release the transport request from SE01:

a. Navigate to SE01.

b. Enter the transport request number and display it.

c. Click on the transport request.

d. From the menu bar, select "Request/Task > Release > Background".

OR

2.Use the program 'RSWBO011' to release the transport request in background mode:

a. Navigate to SE38.

b. Execute the program 'RSWBO011'.

c. Enter the task/request number.

d. From the menu, select "Execute in Background".

    See Also

    If the TR release still results in a timeout error, consider extending the runtime parameter (if you choose to release in Dialogue mode) value or splitting the TR into smaller parts.

Symptom

You want to play a custom warning sound when an error occurs in a custom UI5 application. But unlike SAP GUI, Fiori has no built-in option to enable system sounds.

Environment

SAPUI5.

Cause

SAPUI5 does not provide a dedicated built-in control specifically for handling sound or audio playback. SAPUI5 is an HTML5 framework for creating cross-platform, enterprise-grade web applications, and its control library is focused on business UI components (tables, forms, dialogs, etc.), not multimedia.

Resolution

Audio playback should be achievable in SAPUI5 by using standard web technologies. Some examples:

  1. Add an audio asset (for example, MP3/WAV) to the application’s web resources so it is accessible at runtime (for example, webapp/audio/alert.mp3).
  2. In the controller or central error handler, use the HTML5 Audio object to play the sound when an error is detected. Example: var audio = new Audio("audio/alert.mp3"); audio.play();
  3. If advanced control is required, consider using the Web Audio API to load and play sounds programmatically.
  4. Test in target browsers and, if needed, initiate audio after a user interaction (click/tap), as browser autoplay policies may prevent sound from starting without prior user action.

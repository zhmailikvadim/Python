Symptom

You use the class-based signature tool for the execution of digital signatures.

The signing process dumps with the runtime error GETWA_NOT_ASSIGNED when you use a simple signature with an external security product for approval in your application.

Other Terms

Signature tool, external security product, SSF, card readers, simple signature, signature strategy, GETWA_NOT_ASSIGNED, CL_DS_RUNTIME=================CP, INCL_LCL_DS_SIGN_IMP, IF_DS_EXT_SEC_PROD~TRIGGER_SIGN_WITH_EXT_SEC_PROD

Reason and Prerequisites

Cause: The issue is caused by an error in the SAP Note 3590548.

Prerequisites: You define the settings for approval using simple signature without digital signature strategy.

Solution

Implement the attached correction instructions.

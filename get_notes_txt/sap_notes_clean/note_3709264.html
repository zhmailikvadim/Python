Symptom

Enrichment and Validation (E&V) checks cannot be skipped during a simulation run.

Other Terms

Enrichment and Validation, Payment Processing

Reason and Prerequisites

Some E&V checks should be called only during the real run (for example, Embargo and Sanctions check).

These E&V checks cannot be skipped during the simulation run.

Solution

A new flag has been introduced in the E&V check customizing.

If the flag is set, the corresponding E&V check will be skipped during the simulation run.

All E&V checks defined in customizing will continue to be executed during the real run, regardless of this flag.

Please implement provided correction instructions or corresponding Support Package.

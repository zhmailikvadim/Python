Symptom

There is a business requirement to integrate SailPoint with SAP Ariba as of following SailPoint documentation:
Integrating SailPoint with SAP Ariba 

Additional Cloud Identity Services (CIS) information is required to finalize the configuration.

Environment

  • Identity Authentication Service (IAS)
  • SAP Cloud Identity Services (CIS)
  • SailPoint
  • SAP Ariba

Resolution

Check the additional SAP Cloud Identity Services related information below related to Connection Settings:

Connecting SailPoint and SAP Ariba > Connection Settings 

  1. Select Connection Settings from the menu.

  2. In Base URL, enter the host URL of the SAP Cloud Identity Services Tenant.

    1. Additional (CIS related) info:
      Find the administrator and the host URL (<BASE_URL>) of the appropriate CIS tenant as of following KBA:
      3035908 - How to find the Identity Authentication / Identity Provisioning Tenant Administrator Information or Tenant URL

    2. The host url looks like: https://<CIS/IAS tenant id>.accounts.ondemand.com (or https://<CIS/IAS tenant id>.accounts.cloud.sap)

  3. In System ID, enter the ID of the SAP IPS Proxy system created for SAP Ariba Applications.

    1. SailPoint page to configure the SAP Identity Provisioning Service (IPS) Proxy System

    2. Additional (CIS related) info: 
      Check SAP Online help also how to create Ariba proxy system

    3. You can get the system ID from the URL. For example, https://<BASE_URL>/admin/#/proxySystems/<System ID>/

  4. In Client ID and Client Secret, enter the SAP IPS technical user credentials as mentioned in Prerequisites.

    1. SailPoint page to configure the Technical User in SAP Identity Provisioning Service (IPS)
    2. Additional (CIS related) info: 
      Check SAP Online help also how to create technical (system) user within CIS tenant

    3. You need a Client ID and Secret for system authentication. 
      Make sure you save your client secret. You won't be able to retrieve it from the system later.

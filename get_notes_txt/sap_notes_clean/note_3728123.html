Symptom

Header-level ShipTo on the PR is getting defaulted from the OnBehalfOf user’s profile.

Resolution

By default, if no header-level ShipTo value is provided in the payload, the system sets it based on the user profile of the OnBehalfOf user. If you need to override this header-level ShipTo value, please include the ShipTo value under the default line items section of the payload that is being imported.

<urn:DefaultLineItem>
    <urn:ShipTo>
        <urn:UniqueName>?</urn:UniqueName>
    </urn:ShipTo>
</urn:DefaultLineItem>

Symptom

  • The "blocked for posting" attribute set for a FS item or for a subitem is ignored and data can be posted
  • The system allows posting, even though this is not expected

Environment

  • SAP S/4HANA Cloud Private Edition
  • Group Reporting

Reproducing the Issue

  1. In app 'Define FS Items' the "Blocked for posting" was marked or in the subitem settings the 'Posting/Entry block' was marked.
  2. Attempt to post e.g. a group journal manual entry using a document type that posts on posting level 01, 0C or 10 OR try to enter reported financial data on posting level 00 (e.g. via flexible upload)
  3. Observe that the system allows the posting despite the FS item or the subitem is blocked for posting.

Cause

  • Generally, the 'Blocked for posting' attribute is only checked/applied, when running a data collection task that posts with a document type on Posting Level 00 (e.g. via flexible upload) or in posting manual journals that use posting levels 01, 0C, or 10.  It is ignored per design for Posting Level space (Release Universal Journal) and posting levels 20 and 30.
  • Moreover the 'Blocked for posting' attribute is ignored for consolidation units for which there was no data entry in earlier periods. This is to allow data entry of opening/starting balances in the 'period of first data submission' for such a consolidation unit.
  • The logic for the determination of the "period of first data submission" checks, whether data has already been posted for the relevant consolidation unit in the current year (previous periods) or in the previous year (all periods), but for such periods also a status in the data monitor has to exist, that is unequal from  6 (initial) and/or 7 (incomplete).
  • This check for the data monitor status is based on table TF261 (Data Monitor Tasks - Status). The status for the consolidation unit in the relevant consolidation version and for all tasks of task category 02 (Data collection tasks) is evaluated. For the 'Blocked for posting' attribute being relevant, the status of these tasks has to be different from 6 (initial) and/or 7 (incomplete).  

Resolution

Example: Posting a manual document onto a certain FS Item in period 02/2026 is possible for a consolidation unit, even though the 'Blocked for Posting' attribute is set.  

  1. Verify the status of data collection tasks in table TF261 for the relevant consolidation unit and relevant periods/ fiscal years.
  2. If there is no entry in period 01/2026 and no entry in any period of the previous year 2025 for this consolidation unit in that version for a data collection task with a status that is unequal 6 or 7, the period 02 2026 is identified as 'period of first data submission' and data entry/posting is allowed.

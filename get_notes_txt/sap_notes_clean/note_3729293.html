Symptom

  • The Create Correspondence Fiori app is fetching the standard form instead of the assigned custom program and custom form for Correspondence type .
  • The issue occurs when using the Create Correspondence Fiori app but not in the SAPGUI when run the program directly in SE38.

Environment

  • SAP R/3
  • SAP S/4HANA

Reproducing the Issue

  1. Assign a custom program and custom form for Correspondence type ABC.
  2. Use the Create Correspondence Fiori app to print the correspondence.
  3. The custom form is not used.

Cause

The system is using the new output management logic, which overrides the legacy configuration in transaction code OB78 for Correspondence type .

Resolution

  1. Delete the settings for Correspondence type ABC in the customizing of "allocate output management for automatic correspondence".
  2. Ensure the system uses the legacy customizing settings in transaction code OB78.
  3. Test the result to confirm the resolution.

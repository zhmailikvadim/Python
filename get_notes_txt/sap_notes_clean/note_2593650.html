Symptom

Upgrade using Software Update Manager (SUM) fails with error "CONVT_NO_NUMBER".

Cause


SUM in phase PREP_EXTENSION/EHP_INCLUSION or UPLOAD_FIRST is retrieving and defining the queue from packages found in downlaods directory which is stuck due to issues while Upload/Check/Disassemble the package.


Resolution


Follow below steps to solve the issue.


1. Reset current SUM following steps from respective SUM guide.
2. Take a backup of <DIR_TRANS>\EPS\in
3. Remove all the files under <DIR_TRANS>\EPS\in
4. Download only the packages that are needed for this upgrade.
5. Extract a new SUM and start the upgrade from beginning.

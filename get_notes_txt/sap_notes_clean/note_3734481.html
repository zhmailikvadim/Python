Symptom

Time difference in the valid from date in Subscription Billing using API.

Environment

  • SAP S/4HANA Public Cloud Edition 
  • SAP Subscription Billing 

Reproducing the Issue

  1. Create a subscription with a valid-from date/time using a local (non-UTC) timezone.
  2. Retrieve the subscription via the Subscriptions API.
  3. Observe that the valid-from value in the API response is returned with a UTC timestamp, which can appear as a different calendar date when compared to the local timezone.

Cause

System standard design.

Resolution

Subscriptions API responses always return dates with UTC timestamps, which can appear shifted when compared to local time.

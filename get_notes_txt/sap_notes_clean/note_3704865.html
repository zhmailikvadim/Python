Symptom

You have installed SAP Product Lifecycle Costing 4.0 SP10 or newer and you want to access the views provided by the product, using either SAP Analytics for Office Client or SAP Analytics Cloud.

Other Terms

Cause

By default, the XSAHAA application is bound to its own container and UAA services. To enable access to PLC analytical content, XSAHAA must be explicitly bound to the PLC database and UAA services and configured accordingly.

Reason and Prerequisites

Environment

  • SAP HANA XS Advanced (XSA)

  • Product Lifecycle Costing (PLC)

  • SAP HANA Analytics Adapter (XSAHAA), on-premise

  • SAP Analytics Cloud and/or SAP Analysis for Microsoft Office

Solution

Prerequisites

  • A SAP Product Lifecycle Costing backend is already deployed and running in an SAP HANA XS Advanced space
  • Access to:
    • XSA Cockpit
    • XS CLI (optional)
  • Required PLC service instances exist:
    • xsac-plc-db-service
    • xsac-plc-uaa-service

Scenario: Initial Installation

Step 1: Download the XSAHAA installation package

  • Open the SAP HANA tools page: https://tools.hana.ondemand.com/#hanatools
  • Navigate to Analytics adapter for SAP HANA extended application services, advanced model
  • Download the file: xsahaa-onprem-x.x.x-*.zip

Step 2: Install XSAHAA in the PLC space

  • Install the ZIP file in the same XSA space where PLC is deployed, using one of the following methods:
    • XS CLI: xs install <zip-file-name>
    • XSA Cockpit → Product Installer

Step 3: Bind PLC services to the xsahaa-java application

  • Open XSA Cockpit
  • Navigate to the PLC installation space
  • Choose Service Instances
  • Bind the following services to the xsahaa-java application:
    • xsac-plc-db-service
    • xsac-plc-uaa-service

Step 4: Remove default XSAHAA bindings

  • Select the xsahaa-java application
  • Open Service Bindings
  • Unbind the following services:
    • xsahaa-container
    • xsahaa-uaa

Step 5: Maintain user-provided variables for xsahaa-java

  • Select xsahaa-java
  • Open User-Provided Variables
  • Create or update the following 4 entries:
JBP_CONFIG_RESOURCE_CONFIGURATION
    • KEY: JBP_CONFIG_RESOURCE_CONFIGURATION
    • VALUE: ['tomcat/webapps/ROOT/META-INF/context.xml': {'xsahaa-hdi-container': 'xsac-plc-db-service'}]
USE_NAMED_USER
    • KEY: USE_NAMED_USER
    • VALUE: false
MTA_SERVICES
    • KEY: MTA_SERVICES
    • VALUE: [ "xsac-plc-db-service", "xsac-plc-uaa-service" ]
PERSONALIZE_JWT
    • KEY: PERSONALIZE_JWT
    • VALUE: true

Step 6: Restage xsahaa-java

  • Open the Overview section
  • Choose Restage
  • Restart the application after restaging is completed

Step 7: Obtain the xsahaa-java application route

  • After the application is running, copy the application route of xsahaa-java
  • The route is required in the next configuration step

Step 8: Configure destination in xsac-plc-web

  • Navigate to the PLC installation space
  • Select the xsac-plc-web application
  • Open User-Provided Variables
  • Edit the existing destinations entry and locate the following destination:

    {
         "forwardAuthToken": true,
         "name": "xsac-plc-xsahaa-destination",
         "timeout": 1200000,
         "url": "http://localhost:65535"
    }

  • Replace the value of the url property with the xsahaa-java application route
  • Save the changes

Step 9: Restage xsac-plc-web

  • Open the Overview section
  • Choose Restage
  • Restart the application after restaging is completed

Validation

  • Log on to SAP Analytics Cloud or SAP Analysis for Microsoft Office
  • Create a connection to the PLC system
  • Verify that PLC analytical views are available and can be queried successfully

Troubleshooting

  • If HTTP 404 errors occur when accessing the xsahaa-java application:
    Perform an additional restage and restart of the application

Additional Information

  • Ensure that all PLC services are in RUNNING state
  • Ensure that no additional bindings to default XSAHAA containers remain
  • JWT personalization must be enabled for proper user context propagation

Scenario: PLC Update with Existing XSAHAA Installation

When PLC is updated in an environment where XSAHAA is already installed, the initial XSAHAA configuration remains unchanged. However, the binding between PLC and XSAHAA is terminated during the upgrade process.

Resolution: Re-execute Step 8 and Step 9 from the "Initial Installation" section to restore the PLC-to-XSAHAA binding.

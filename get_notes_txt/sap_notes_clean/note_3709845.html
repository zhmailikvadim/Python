Symptom

  • User IDs are generated incorrectly compared to the normal setup.
  • Some user IDs are observed to have extended characters. Extensions may include combinations of numbers and letters.

Environment

SAP SuccessFactors Onboarding

Reproducing the Issue

  1. Initiate the onboarding process via Recruiting.
  2. Complete the Data Collection Steps.
  3. Hire the employee into EC via Manage Pending Hires.
  4. Enter a "No Show" status for the employee.
  5. Observe that the user ID changes to include extended characters, as shown in the attached screenshot.

Cause

The ONB2ReportNoShowEventSubscriber is an internal system event handler ONB2.0 - It listens for No Show events and, by design, modifies the username and userId to make them unique by concatenating random characters. This behavior is standard and happens for data integrity and rehire readiness.

When a candidate is marked as a No Show:

  • Archives or anonymizes the original candidate’s userId and username.
  • Adds a random suffix like #A1B2C3 to the userId.
  • Frees up the original userId for potential reuse in future hires.
  • Does this regardless of whether any business rules exist—it’s hardcoded behavior, not customizable via the UI or rules.

This is SAP’s way of:

  • Preventing conflicts with future users (since userId must be unique),
  • Supporting data privacy and rehire scenarios,
  • Ensuring the original userId can be reused if the person is rehired later.

This is done automatically by ONB2ReportNoShowEventSubscriber and cannot be disabled via the UI or standard configuration.

Resolution

Please check any business rule associated with user ID with scenario: Generate Employee ID for Hire/Rehire that might be causing the unknown concatenation of unknown characters.

Symptom

You are facing below MD01N dump: 

Category               ABAP programming error
Runtime Errors         UNCAUGHT_EXCEPTION
Except.                CX_PARAMETER_INVALID_RANGE
ABAP Program           CL_PPH_MRP_ISSUES=============CP

Short Text
    An exception has occurred that was not caught.

What happened?
    The exception of class "CX_PARAMETER_INVALID_RANGE" was triggered but not
     caught anywhere in the
    call hierarchy.

    Since exceptions represent error situations, and this error was not
    adequately responded to, ABAP program "CL_ABAP_TSTMP=================CP" had
     to be terminated.

Error analysis
    An exception has occurred in class "CX_PARAMETER_INVALID_RANGE". This
     exception was not caught
    in procedure "SET_TIMESTAMP" "(METHOD)" or propagated by a RAISING clause.
    Since the caller of the procedure could not have anticipated this
    exception, the current program was terminated.
    The reason for the exception occurring was:
    The reason for the exception is:
    Parameter has invalid value: Parameter TSTMP_IN has invalid value 0.0000000.



Environment

  • SAP ERP Logistics (PP)  
  • SAP S/4 HANA, on premise
  • SAP S/4HANA Cloud Private Edition Release Independent

Reproducing the Issue

  1. Run T-code: MD01N or schedule background job for program "PPH_MRP_START"
  2. Check T-code: ST22 dump

Cause

Missing note connection

Resolution

Please firstly implement SAP Note 3501326 

Keywords

MRP, dump, MRP live, MD01N, TSTMP_IN, UNCAUGHT_EXCEPTION, CL_PPH_MRP_ISSUES, CX_PARAMETER_INVALID_RANGE

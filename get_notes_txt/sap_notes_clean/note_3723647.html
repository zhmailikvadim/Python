Symptom

  • There is a need to edit/configure released workflows
  • The released workflows are not visible in the "Projects" section of SAP Build Process Automation lobby.
  • The workflows can be found on the monitor "Process and Workflow Instances" page on the SAP Build Process Automation tenant

Environment

SAP Build Process Automation

Reproducing the Issue

  1. Open SAP Build Process Automation service.
  2. Go to Lobby -> All Projects.
  3. Try search for the project associated with the workflow

Cause

The workflows were not created as process automation projects on SAP Build Process Automation but are legacy workflows located in SBAS (SAP Business Application Studio).

Resolution

  1. To view the legacy workflows, download the workflow models in JSON format from the Control Tower:
    • Navigate to Lobby -> Control Tower.
    • Go to Environments -> Public Environment.
    • Select "Processes and Workflows".
    • Choose the workflow definition and click the download model button.

      Note: the JSON file cannot be directly uploaded to SAP Business Application Studio (BAS) for editing.
  2. To edit and redeploy the legacy workflows:
    • Option 1: Use the original .mtar project files to edit and redeploy the workflows in SBAS.
    • Option 2: Recreate the workflows as new processes in SAP Build Process Automation.

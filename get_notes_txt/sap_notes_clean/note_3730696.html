Symptom

  • The scheduled benefits auto-enrollment job returns the message "Error while executing Batch ||nul".
  • In some cases, enrollments appear only after several days, not during the initial batch run.
  • Manual “Evaluate Eligibility Based on Benefit Master Data Change” for a single user can succeed to create the enrollment.

Environment

  • SAP SuccessFactors HXM Suite 
  • SAP SuccessFactors Employee Central

Reproducing the Issue

  1. Hire an employee
  2. Allow the daily “Create or Update Benefit Enrollments for Employee Master Data Changes" job to run
  3. Observe that some benefit enrollments fail with "Error while executing Batch ||nul" and may be created only days later

Cause

A technical exception in batch processing can raise a generic "Error while executing Batch ||nul" and mask the per-user error. 

 If this exception occurred for one or more employees being processed within a batch, the job response shows the same failure message for all employees within that batch.

Resolution

The fix will be available in the next release with deployment on April 13th, for Preview, and on May 15th, for Production. 

With the fix, the exception reported will not occur and the job response will provide the correct error message for the failing employees, while allowing the rest of the employees to go through with the enrollment/edit.

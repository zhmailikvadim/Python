Symptom

  • When entre time for different days. The entries are not correctly save in each day and are totalized in the same day .
  • If fill in the "allowance type" field in time entry screen of CAT2 and enter and enter time for one day. The following error appear "Value must be greater than zero for the external allowance (external code: CATS_ZEILE_0)"

Environment

  • SAP S/4HANA Cloud Private Edition 
  • SAP S/4HANA onprem

Reproducing the Issue

The Business Function CA_CATS_H4S4_SF_CTR for the success factors integration it is active and try to use the CAT2 with the fields


SFSF Allowance Type External Code    CATSD-SFSF_ALLOWANCE_TYPE_ECODE
SFSF Allowance Type UUID    CATSD-SFSF_ALLOWANCE_TYPE_UUID
SFSF Time Type External Code    CATSD-SFSF_TIME_TYPE_ECODE
SFSF Time Type UUID    CATSD-SFSF_TIME_TYPE_UUID

Cause

We don't recommend use timesheet in the backend with the SSFF fields as  the transaction CAT2 it is not tested for these scenarios and  we cannot confirm it is working correctly. 

Resolution

Don't use the CAT2 using the fields

SFSF Allowance Type External Code    CATSD-SFSF_ALLOWANCE_TYPE_ECODE
SFSF Allowance Type UUID    CATSD-SFSF_ALLOWANCE_TYPE_UUID
SFSF Time Type External Code    CATSD-SFSF_TIME_TYPE_ECODE
SFSF Time Type UUID    CATSD-SFSF_TIME_TYPE_UUID

Keywords

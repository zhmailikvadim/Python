Symptom

A notification/alert was sent informing that table CIF_CCR_O_IBP and/or CIF_CCR_S_IBP is reaching the maximum of 2 billion records.

Environment

  • SAP Integrated Business Planning (SCM-IBP)
  • SAP ERP
  • SAP S/4HANA

Cause

These are temporary tables that keep the reconciliation job´s name. However, when the user DEBUGIBPCCR is enabled with the debug mode in CFC2, those tables cleanup is skipped. For this reason, it is not recommended to keep this user enabled in debug for a longer period.

Resolution

As informed in the KBA 3442900 - IBP: Analysis of Real-time Integration Reconciliation results, run the report RCIF_CCR_CLEAN_IBP to clear the tables CIF_CCR_O_IBP and/or CIF_CCR_S_IBP and reduce the records.

If it is not necessary, disable the debug for DEBUGIBPCCR, or if it is necessary to keep it enabled, run the report above more often.

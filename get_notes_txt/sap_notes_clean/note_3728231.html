Symptom

  • The workflow to create a session token for a Windows AD user using the RESTful SDK fails with the following error: "Account Information Not Recognized: Active Directory authentication failed to log you on."
  • The same session token is created successfully when using an enterprise user.

Environment

  • SAP BusinessObjects Business Intelligence Platform 4.3/2025
  • Any supported operating system

     

Reproducing the Issue

  1. Try to login with windows AD user using restful SDK.
  2. Get error "Account Information Not Recognized: Active Directory Authentication failed to log you on"

Cause

  • In split deployment in order to windows AD user to login krb5.ini and bsclogin.conf needs to be present in web application server.
  • The files were present in app server but not in web application server.

Resolution

  1. Keep krb5.ini and bsclogin.conf files in web application server.
  2. Make sure you provide correct path of the files in tomcat configuration->java options.

Symptom

During integration of subcontracting Purchase Info Records to SAP IBP, Transportation Lanes are created even though no Production Version is maintained at the relevant Locations. The integration completes successfully without any error or warning messages.

Environment

  • SAP Integrated Business Planning
  • Real Time Integration IBP

Reproducing the Issue

  1. Login to the ERP/S4HANA system.
  2. Execute transaction CFM1 and generate IM.
  3. Execute transaction CFM2 to activate IM.
  4. Execute transaction CFM9 for initial transfer.

Cause

In subcontracting with multiple production versions, SAP IBP auto-creates only one transportation lane, so the other production versions have no lanes and are excluded from planning.

Resolution

  1. Create transportation lanes for each production version so planning can consider them.
    • Fastest: In Manage Master Data, copy the existing lane, change the Production Version (per CMPDS1), and assign a unique ADDLANEID.
    • Alternatively: Upload the lanes via Data Integration Jobs (CSV), or generate them using rule-based master data.
  2. After creating the lanes, re-run planning so all relevant production versions are included.

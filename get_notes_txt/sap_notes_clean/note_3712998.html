Symptom

  • Employees with a daily salary below the minimum economic zone salary are not receiving tax calculations on regular periods.
  • Tax calculation is missing for wage type /440.
  • Legal and fiscal requirements specify that if an employee has a basic salary plus other earnings, tax calculation is mandatory because the gross salary exceeds the minimum economic zone salary.

Environment

  • HR renewal 

  • SAP S/4HANA 

  • SAP ERP  

  • SAP SuccessFactors Employee Central Payroll 

Reproducing the Issue

  1. Navigate to transaction PC00_M32_CALC and run it.
  2. The employee has a basic salary plus other earnings that exceeds the minimum economic zone salary.
  3. Observe that wage type /440 is missing in the output.

Cause

The system does not considers other tax earnings in the calculation check for the creation of the Wage Type /440.

Resolution

Resolution

  • SAP will not change the standard calculation of technical wage type /440, as doing so could introduce side effects for other customers.
  • To meet customer-specific requirements, use enhancement BAdI HR_MX_HMXCISR0 delivered by SAP Note 0002755787 - [MX] Tax exemption for employees with minimum salary - Article 96. This BAdI allows you to influence the values of the variants used in the /440 calculation without modifying standard logic.

Implementation outline

  1. Ensure your system contains the delivery of BAdI HR_MX_HMXCISR0 (apply the relevant HR Support Package/Notes for MX if needed).
  2. Create a customer implementation of BAdI HR_MX_HMXCISR0 in transaction SE19.
  3. Within the BAdI methods provided, adjust the variant values used by the /440 calculation so they reflect your business requirement:
    • Add the value of the other earnings to the basic salary component used in the /440 calculation (i.e., compute basic salary = basic salary + other earnings before /440 is derived).
    • If multiple variants contribute to /440, ensure you update the correct variant(s) and restrict the logic by payroll area, employee group, period, etc., as needed.
  4. Transport the implementation to your test and production systems and validate the results in payroll simulation to confirm /440 reflects the intended base (basic salary plus other earnings).

Outcome

  • The /440 wage type will be calculated using the adjusted variant values (basic salary including other earnings) in line with your scenario.
  • This approach avoids changes to standard and prevents unintended impacts to other customers, while providing the required flexibility through the delivered BAdI.

Symptom

You create a sales order with an item, then update it with a second one. When you check the values of the CUSplit field in the RDR1 table, it has value NULL for the second item, which is invalid.

Reproducing the issue

Prerequisite:

You are using SAP Business One or SAP Business One, version for SAP HANA, 10.0 Feature Package 2102 or higher.

Scenario:

  1. Create a sales order with an item.
  2. Update the sales order with a second item.
  3. Check the CUSplit field in the RDR1 table through a query (select "CUSplit", * from RDR1 where "DocEntry" = x).

Expected behavior:

The CUSplit field in the RDR1 table has value N for the second item. The valid values for the field are N and Y, depending on the Freight Type setting.

Actual behavior:

The CUSplit field in the RDR1 table has value NULL for the second item, which is invalid.

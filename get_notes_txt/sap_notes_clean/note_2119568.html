Symptom

The partner number of a partner is stored with leading zeros in the table IHPA.

As a result, all EAM list transactions return incorrect results if the data selection is restricted via such a partner.

Other Terms

IW38, IW39, IW28, IW29, IW58, IW59, IH06, IL05, IL07, IE05, IH08, IE07, IE37, IE36, TVPA, NRART, TPAR, PARVW, IHPA, IPAR, US, F4_PARNR_F00, PM_PARTNER_LENGTH, PM_PARTNER_MAINTAIN, PARTNER_READ

Reason and Prerequisites

Reason:

There is a program error.

Prerequisites

The partner type defined for the partner role of the partner is US.

The partner contains only numeric values.

The first numeric value is a zero.

The partner contains a maximum of 11 numeric values.

You used the search help to select the partner.

Solution

Implement the correction instructions.

Also implement SAP Note 2188949.

SAP Note 2120724 contains an analysis/correction report for the described inconsistency. (SAP Note 2120724 has the status "Pilot Release". Please open an SAP customer incident in the component PM-WOC-LE so that the SAP Note is released for you - see SAP Note 538778.)

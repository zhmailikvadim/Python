Medium 1490948/2025 Error while generating report in BRH An internal error occurs when you try to generate a release decision report in the My Release Decisions app. To resolve this issue, the service retry mechanism for the Adobe Document Service (ADS) has been improved by adding an exponential backoff strategy. This update helps prevent HTTP 429 errors during high-concurrency PDF generation, improving the overall request reliability. Testing should cover the generation of release decision reports in the My Release Decisions app. N n/a
High 56550/2026 My Release Register Issue

When the Store Release Register Reports setting is turned off in the Manage Release Configurations app, exporting a release register report in the My Release Register app navigates to a URL that throws a transient error.  This issue has been resolved with a code fix.

Testing should cover the export of release register reports in the My Release Register app with your existing configuration setup.

 N n/a
High 199143/2026 Documents attached to PCN-level release checks are not available

In the My PCN Release Decisions app, attachments for PCN-level release checks disappear from the PCN release decision item after a release decision is set.  This issue occurs because the snapshot created when you set the release decision doesn't include attachments for PCN-level release checks. The issue has been resolved by correcting the snapshotting logic.

 Testing should cover adding attachments to the PCN-level release checks in the My PCN Release Decisions app, then verifying that the attachments can be viewed and downloaded after a release decision is set. N n/a
Medium 225891/2026 Countries/Regions are not available when exporting to excel from BRH

When exporting the worklist from the My PCN Release Decisions app, the countries/regions were not visible in the export to Excel, resulting in an empty column. This issue has been fixed, and the export now correctly includes and displays the associated country/region values.

 Testing should cover exporting the worklist from the My PCN Release Decisions app. Ensure that all the details included in the worklist are exported, including the countries/regions. N n/a

The following changes are based on an extended analysis of the current production system behavior regarding the performance of certain features. They have been added to the regular change to stabilize and improve the performance of the application.

Note: The primary focus for (all) test cases should be to verify that there are no functional changes after the regular change has been deployed.
n/a n/a Change history details not visible in user responsibility template In the Manage Release Configurations app under User Responsibility Templates, the change history details are not visible when user responsibilities are deleted. The code fix addresses cases where the old value is missing during a deletion, but the new value is present. It substitutes the old value with the new value to ensure the change log is not empty.  Testing should cover the change log for user responsibility templates in the Manage Release Configurations app. N n/a
n/a n/a SQL error and HTTP 500 when activating business rules A code modification is introduced to resolve the database lock issue that can occur when business rules are activated. Testing should cover the activation of business rules in the My Release Decisions app and the My PCN Release Decisions app. Open worklist items that have inactive business rules and choose the Activate Business Rule button. N n/a
n/a n/a S4-inbound: Crashes due to unauthenticated GET calls  All expected calls from SAP S/4HANA should be authenticated POST calls that push entities and configuration. Restrictions are now in place to ensure this. This is a technical change with no functional impact. There are no specific test recommendations other than standard regression testing. N n/a
n/a n/a Kafka stabilization items

The fail-safe mechanism for publishing messages to Dead Letter Queue (DLQ) topics has been enhanced.  This change leverages the existing "publishMessage" method to make the publishing method generic and extends the iron-kafka binding configuration validity from 90 to 120 days.

 This is a technical change with no functional impact. There are no specific test recommendations other than standard regression testing. N n/a
n/a n/a Avoid duplicate re-processing of messages

The Kafka integration sometimes can't confirm successful message processing to Kafka due to long processing times. This leads to messages being picked up again and results in duplicate processing.

To resolve this duplicate processing, the message processing state is now stored at the database level. Before processing a message, the system checks whether the message was already processed. If it was, the system skips it.

 This is a technical change with no functional impact. There are no specific test recommendations other than standard regression testing. N n/a
n/a n/a BRH Outbound: Additional logging to cover all versioning functional scenarios  Added more internal logs to capture all exception scenarios and the request context. This helps the internal team with investigations when there's an issue sending a release decision to SAP S/4HANA. This is a technical change with no functional impact. There are no specific test recommendations other than standard regression testing. N n/a

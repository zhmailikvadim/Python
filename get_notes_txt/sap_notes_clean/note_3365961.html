Symptom

When the Learning user tries to register in a class, the “The Voucher Number can't be blank.” message returns and the enrollment does not happen.

Environment

SAP SuccessFactors Learning

Reproducing the Issue

  1. The related class is set in the following manner: The class has a price >
  2. The class has some reserved seats >
  3. The reserved seats of this class have no price related >
  4. E.g.: Learning admin side > Access the class > “Seat Reservations” tab > “Price Per Reserved Seat” is zero >
  5. Then, when the user that falls in the reserved seat condition tries to register in the class, the “The Voucher Number can't be blank.” message returns.

Cause

Having the reserved seats without a price is not a problem, they can be created this way. The conflict happens when the “RequireOrderTicketWhenSlotAvailable” setting is “true”.

This setting is found in the Learning admin side > System Administration > Configuration > System Configuration > FINANCIAL.

The “RequireOrderTicketWhenSlotAvailable” setting means: “When set to false, users can register for a class without entering a voucher ID if there is reserved seat available for them. By default, a voucher ID is required. Options: false or true (default).”.

Resolution

While the “RequireOrderTicketWhenSlotAvailable” setting is “true”, the user would require a voucher to register in the class via the reserved seat. If there is no voucher, when the user tries to register, the “The Voucher Number can't be blank.” message returns.

To have a voucher for the reserved seats, a price needs to be set when performing the action of reserving them – the reserved seats cannot be priceless if the “RequireOrderTicketWhenSlotAvailable” setting is “true”.

If the voucher is not necessary for the user registration via the reserved seat, the “RequireOrderTicketWhenSlotAvailable” setting can be set to false and no change in the reserved seat setup would be required.

Symptom

  • OBJECTS_OBJREF_NOT_ASSIGNED_NO dump occurs while performing SPAU activity for SHI3 translations object.
  • Category: ABAP programming error
    Runtime Errors: OBJECTS_OBJREF_NOT_ASSIGNED_NO
    Except.: CX_SY_REF_IS_INITIAL
    ABAP: Program: CL_WB_ABSTRACT_TOOL===========CP
    Application Component: BC-DWB-TOO
  • Short Text: Access using a 'NULL' object reference is not possible.

Environment

  • SAP S/4HANA
  • SAP S/4HANA 2025
  • Installation, Upgrade and Update Tools
  • Note Assistant
  • ABAP Platform

Reproducing the Issue

  1. Navigate to SPAU.
  2. Select "Translations".
  3. Choose "Object Type SHI3".
  4. Confirm all translations.

Cause

  • The function module RC_ACCESS_PERMISSION dumps because the object type SHI3 has no defined WB persistence class.
  • The service class CL_SHI3_WB_TOOL does not redefine the IF_WB_ACCESS~LOCK_OBJECT method, resulting in a null pointer exception.
  • Need to implemented methods IF_WB_ACCESS~LOCK_OBJECT and IF_WB_ACCESS~UNLOCK_OBJECT so that ENQUEUE and DEQUEUE modules are called respectively 

Resolution

  • Implement SAP Note 3701779 - Maintaining SHI3 translations via SPAU results in a runtime error to resolve the issue.

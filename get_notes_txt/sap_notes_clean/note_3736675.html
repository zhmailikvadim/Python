Symptom

  • A background job is scheduled for transaction CM01 (Capacity Planning).
  • The job is successfully executed and shows status Finished in SM37.
  • When checking transaction SP01 (Spool Requests), no output is generated.
  • The expectation is to find capacity planning results (e.g., capacity load) in a spool request or exportable format (e.g., Excel).

Environment

  • SAP ERP Capacity Planning (PP-CRP)
  • SAP R/3
  • SAP R/3 Enterprise
  • SAP ERP Central Component
  • SAP ERP
  • SAP Enhancement package for SAP ERP
  • SAP Enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA, on-premise
  • SAP S/4HANA Cloud Private Edition

Reproducing the Issue

  1. Access transaction CM01
  2. Navigate to: Planning → Background Processing → Start Job
  3. Create a variant and schedule the job (immediate execution)
  4. Monitor the job in SM37 until status = Finished
  5. Check spool output in SP01
  6. No spool request is found

Cause

This is standard system behavior.

Transaction CM01 was designed as a classic SAP GUI-based capacity planning tool, where:

  • Background processing does not generate spool output (SP01)
  • Results are not written to spool requests
  • The transaction uses screen-based (interactive) output logic, not list/spool-based output

Additionally:

  • CM01 supports multiple interactive views (e.g., Standard Overview, Detailed Capacity List, Variable Overview)
  • These views are dynamically rendered and cannot be directly converted into spool output

Resolution

To access the results of background processing in CM01, follow these steps:

  1. Access transaction CM01
  2. Navigate to: Planning → Background Processing → Read Job Results
  3. From the selection screen, choose one of the available views:
    • Standard Overview
    • Detailed Cap. List
    • Variable Overview
  4. The results will be displayed on screen

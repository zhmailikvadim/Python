Symptom

When loading a Purchase Order in Migration Cockpit, the object loading fails with error messages such as:

  • "### - field name does not exist in structure _WA_R_IS_HEADER"
  • "### - field name does not exist in structure _WA_R_IT_ITEMS"
  • "### - field name does not exist in BAPI structure _WA_R_IT_ITEMS"
  • "### - field name does not exist in BAPI structure _WA_R_IS_HEADER"

Where ### represents a field name such as VAL_AVAIL, VAL_OVERBOOK, VAL_TOLERANCE, or others.

Environment

  • Materials Management (MM)
  • SAP S/4HANA
  • Migration Cockpit
  • BAPIs - Purchase order

Reproducing the Issue

  1. Open the Migration Cockpit app.
  2. Select the relevant migration project.
  3. Choose the Purchase Order migration object.
  4. Execute the Prepare step.
  5. System raises errors indicating that specific fields do not exist in the corresponding header or item structures

Cause

The fields referenced in the errors are industry-specific enhancements, such as those delivered for IS-Retail or Public Sector (PPS). These fields are not part of the standard Purchase Order data model.

Although they may appear available for mapping in FM DMCC_MIG_PUR_PO_CREATE, the underlying standard FM BAPI_PO_CREATE1 does not support these industry-specific fields.

As a result, the Migration Cockpit cannot process them and raises the structure validation error.

Resolution

Mapping of non-standard or industry-specific fields is not supported. 

Refer to the SAP Note 3324716, which provides a list of supported extension fields for the Purchase Order migration object.

Although this note is released for SAP S/4HANA Cloud Public Edition, it references the same migration objects used in Private Cloud and On-Premise. Therefore, the same technical limitation applies to these.

In summary, fields and structures that are not part of:

  • The standard Purchase Order object, or
  • The extensions explicitly documented in SAP Note 3324716

are not supported by SAP in this migration scenario.

Symptom

You run the Programa de cálculo de nómina mexicana (HMXCALC0) report for an employee, and the payroll processes customer wage types that has values configured in the following processing classes:

  • Clasificación de las partes exentas para SDI (62) - Any value.
  • Acumulación y registro de CC-nóminas a delimitar -->CYCL.X024 (41) - values 3, 4, 5, 6, 9 and B.

In the scenario described above, the system does not accumulate the customer wage types in the corresponding SDI technical wage types, when processing the HR-MX: Cálculo del salario diario integrado (MXS1) payroll calculation rule.

Example

Customer wage type Amount (ARS) Processing Class 41 Processing Class 62 SDI technical wage type
9XXX 300,000.00 4 L /XXX

In the example described above, the system calculates the /XXX wage type amount with ARS 0.00, while it should be calculated with ARS 300,000.00.

Reason and Prerequisites

This issue occurs as a side effect of SAP Note 3638296 - [MX] Changes: INFONAVIT loan deductions during absences and incapacities 2025, which changed the position of the Gross Input and Storage (X024) payroll calculation rule within the Mexican payroll schema.

This payroll calculation rule is responsible for transferring wage types configured with processing class 41 values listed above from the Internal Table (IT) cluster table to the Result Table (RT) cluster table.

Previously, the X024 was executed before the calling of the MXS1 in the schema. After the SAP Note installation, it is now executed after the MXS1.

Because the MXS1 payroll rule only processes wage types from the RT cluster table through the Process Result Table (PRT) payroll function, wage types with values configured in both processing classes 41 and 62 will not be present in the RT when MXS1 executes, resulting in this issue.

Solution

To resolve this issue, we recommend implementing one of the following approaches:

1. Call the new payroll calculation rule MXSA after the MXS1

If you are certain that no wage types exist in both the IT and RT cluster tables simultaneously when MXS1 is called, you can change the HR-MX: Cálculo de bases y contribuciones IMSS (MXSI) payroll schema to call the HR-MX: Cálculo del salario diario integrado (processamiento en PIT) (MXSA) payroll calculation rule via Process Internal Table (PIT) payroll function, after the current calling of MXS1.

The new payroll calculation rule MXSA is detailed in the Manual Instructions for SAP Note 3734454.pdf file attached to this note.

2. Create a custom payroll calculation rule to generate standard SDI wage type(s) based on each specific wage type from IT cluster table

If you are not confortable with the option 1 or if you want to accumulate only a specific set of wage types from IT in the standard SDI wage types without moving them to RT, you can create your own custom payroll calculation rule to be called via PIT payroll function after the current calling of MXS1.

In this rule, configure each wage type to specify which standard SDI wage type it should be accumulated into.

Symptom

If you use the function module BAL_LOG_EXCEPTION_ADD to add an exception to an application log and this exception has not implemented the interface IF_T100_MESSAGE, the public attributes of the exception object are saved in the parameter table of the application log message. When you do this, the following problems occur:

  • In addition to the public attributes, all constants are also saved. This increases the size of the parameter table unnecessarily.
  • If the exception object has more than 10 public attributes, their names are not unique in the parameter table.
  • If the attribute value is longer than 255 characters, it is truncated.
  • If the class inherits from another exception class and a text ID of the parent class is used when the exception is triggered, the attribute name of the text ID is not saved correctly in the application log message. 

Solution

Implement the correction instructions contained in this SAP Note or import the relevant support package.

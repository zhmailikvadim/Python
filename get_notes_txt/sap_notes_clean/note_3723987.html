Symptom

  • Custom screen sequence overwritten during system upgrade in SAP S/4HANA
  • When add new item for the custom fields, whether there is any rule for item number 
  • Is there any way to avoid the custom screen get overwrite during system upgrade

Environment

  • Sales and Distribution (SD)
  • Logistics Execution (LE)
  • SAP S/4HANA

Reproducing the Issue

For example :

  1. Create a custom screen add assign the custom screen to screen sequence 'CVIV01' using item 2180000 in transaction BUS6.
  2. Upgrade the system to higher version (for example SAP S/4HANA 2023 version).
  3. Observe that the custom screen is overwritten and replaced by SAP standard screen BPTM66.

Cause

  • Item numbers ending with '00', such as 2180000, are part of the SAP namespace and are not secured against overwriting during system upgrades.

Resolution

  1. Ensure that custom entries in transactions BUS3-BUS7 are assigned item numbers within the customer namespace.
  2. Use item numbers where the last two digits are not '00'. For example, change the item number from 2180000 to 2180001.
  3. Verify that the updated item number is correctly assigned to the custom screen sequence in transaction BUS6.
  4. Test the configuration to confirm that the custom screen remains stable after a release/support package upgrade.

Symptom

An export ends with return code 0008. The export log shows the following error message:

2EETW000 error in key analysis of table ... - key '...' is longer than MAX_KEY_LG:172

As a consequence, the export of the concerned table entry is refused.

Reason and Prerequisites

The error occurs when the key of the affected table is longer than 172 characters, and the key is specified in table E071K_STR.

Solution

As of R3trans version 21.08.25, R3trans is able to deal with such oversized keys by automatically converting them to generic selections. This improvement is provided for the following kernel releases:

  • kernel release 918 and higher: from the beginning
  • kernel release 917 as of patch level 2.
  • kernel release 916 as of patch level 64.
  • kernel release 915 as of patch level 27.
  • kernel release 914 as of patch level 42.
  • kernel release 793 as of patch level 325.

As a workaround, you can manually change the transport request by changing these selections to generic selections, then repeat the export.

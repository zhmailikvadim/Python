Symptom

  • I create an Activity Item that is required for Work Order activation.
  • The Persons who have received Work Item field is blank.
  • The Activity Item is configured for Supplier Account.
  • I expect the supplier user to receive the Activity Item so they can complete it before activation.

Cause

  • The Work Order is still in a pending approval status.
  • When When will this be due? is set to Prior to Worker Start, the Activity Item is only distributed after the Work Order is accepted by the supplier.
  • Until acceptance, the Worker is not confirmed to start, so the system does not trigger the agent that sends Activity Items.

Resolution

  1. Accept the Work Order to trigger Activity Item distribution.
  2. Allow the system agent a few minutes to populate the Persons who have received Work Item field.
    • The internal agent runs approximately every four minutes.
If you want the Activity Item to be distributed before Work Order approval or supplier acceptance, re‑create the Activity Item with the When will this be due? setting configured as one of the following:
  • After Work Order Create, or
  • On Work Order Draft

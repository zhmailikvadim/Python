Symptom

The execution of a calculation view results in the following error:

Error: SAP DBTech JDBC: [258]: insufficient privilege: search table error: [2950] excption 306003: Authorization failed in
Calculation Engine. User is not authorized to add analytic privileges to runtime filters.

Environment

  • SAP HANA Database 1.0
  • SAP HANA Database 2.0

Reproducing the Issue

  1. Increase the authorization trace levels on your database using the following command: alter system alter configuration (‘indexserver.ini’,’SYSTEM’) SET 
    (‘trace’,’authorization’)=’info’ with reconfigure;
  2. Execute the Calculation View
  3. An error similar to the following is displayed:

 

  1. Disable tracing using the following command:

alter system alter configuration (‘indexserver.ini’,’SYSTEM’) unset 
(‘trace’,’authorization’) with reconfigure;

Cause

The use of analytic privileges with repository procedures is not currently supported.

Resolution

Refer to SAP Note 2462871 - Error: SAP DBTech JDBC: [258]: insufficient privilege: Could not obtain all relevant analytical privileges for user <User> accessing view <View Name>due to exception: Invalid SQL AP definition for further information.

See Also

SAP Note 1809199 - SAP HANA DB: Debugging user authorization errors

Keywords

Invalid SQL AP definition, exception 306003, Authorization failed, Failed to obtain analytical privilege definition

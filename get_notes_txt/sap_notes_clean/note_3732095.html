Symptom

  • Authorization object Q_CHAR_PRC does not prevent specific user to edit, valuate or close inspection characteristic as expected.
  • The valuate or close button is still editable in T-code QE01/QE02/QE11/QE12 even Q_CHAR_PRC is defined as not allow user to valuate or close.

Environment

  • SAP ERP Quality Management (QM)
  • SAP R/3
  • SAP R/3 Enterprise
  • SAP ERP Central Component
  • SAP ERP
  • SAP Enhancement package for SAP ERP
  • SAP Enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA, on-premise
  • SAP S/4HANA Cloud Private Edition

Reproducing the Issue

  1. Authorization object Q_CHAR_PRC is defined as below, which means the user can only set the inspection characteristic status from 0,1 to 2 (in process),3(valuated) for plant 0001 and operation TEST.
    WERKS          = 0001
    ARBPL           = TEST
    QESTATOLD   = 0,1 
    QESTATNEW  = 2,3
  2. Open QE01 or QE11,
  3. Enter results for an inspection characteristic that is current in status 1.
  4. Choose Valuate, status is changed to 3.
  5. the button 'Close' is editable and click close button, it is status 5 now.
  6. save, the results can be set to status 5 (closed) which is incorrect.

Cause

Authorization object Q_CHAR_PRC is checked in function module QAUT_CHARACTERISTIC_PROC. This function module can only be called when the work center is maintained on the operation of inspection plan.

Resolution

Maintain work center on the inspection operation of the used inspection plan in QP02, so that the Q_CHAR_PRC authorization check is executed and status transitions are restricted as configured.

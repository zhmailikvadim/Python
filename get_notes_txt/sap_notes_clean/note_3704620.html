Symptom

In some cases, during bulk take reading execution, if the user enters a decimal value, the recorded reading is not saved for the corresponding equipment's or functional location's measuring point.

Other Terms

  • SAP Service and Asset Manager
  • SSAM
  • SAM
  • Measuring Point
  • Bulk Take Readings
  • ICMTANGOAMF10-42179

Reason and Prerequisites

Reason: In the current implementation, the recorded value is passed to the MDK as a string instead of a numeric value. In certain regional settings, the MDK cannot correctly process this string-based value, which results in the reading not being saved.

Prerequisites: 

  • BAS or Visual Studio Code with SAM MDK project
  • `patch` command (available on BAS and macOS by default) via terminal or other patching utility
  • Read the guidelines on how to apply the patch and go through the appendix section if you get any errors.

Solution

With the implemented solution, the recorded value is now parsed into a numeric format before being passed to the MDK. As a result, MDK can always correctly create or update the corresponding entity, regardless of the regional settings.

To apply the patch, download the attached 3704620_XXXX.patch file and save it in a known location, preferably adjacent to your SAM.mdkproject.

If you are on Windows, see SAP Note 3533879 for how to install a .patch file in Windows. If you are on a Mac, follow the steps below:

  1. Open a terminal in the root of your SAM metadata (i.e. SAM.mdkproject/metadata or SAM.mdkproject/metadata/base.definitions)
  2. Run the terminal command `patch --no-backup-if-mismatch -p4 < path/to/3704620_XXXX.patch`. 
  3. Verify in the terminal window that the following files have been modified and listed as patched, as indicated by the console output:
  • patching file 'Rules/Measurements/Points/MeasurementDocumentsCreateChangeSet.js'

Symptom

Administrators need to permanently retire security assets such as PGP keys, SSH keys, or certificates within the SAP Fieldglass Configuration Manager when they are compromised, no longer required due to ended vendor relationships, or replaced by newer assets. This guide explains the administrative procedure to change an asset's status to "Revoked," ensuring it can no longer be used for active encryption or decryption tasks across system integrations.

Resolution

Once a security asset is revoked, the action is permanent for encryption and decryption purposes. Ensure you have an alternative active asset if your integrations require continuous uptime.

  1. Log in to SAP Fieldglass as a Configuration Manager user.
  2. Select the Encryption Keys and Certificates tile from the self-service dashboard.
  3. Locate the certificate or key you wish to retire from the list of existing assets.
  4. In the Action column, choose View Details for the corresponding row of the asset you want to revoke.
  5. On the top-right side of the Details tab, click the Revoke button.
  6. Confirm the action. The status of the asset will immediately update to Revoked.

See Also

  • Integration Impact: A revoked certificate cannot be used for standard integration encryption or decryption. Any automated subscription linked to this key will fail until a new active key is associated with the endpoint.
  • Field Level Keys: For field-level encryption, a revoked key can still be used to retrieve and re-encrypt data, but only with explicit authorization involving a manual process.
  • Revocation vs. Deletion: You can only revoke assets that have been generated and activated. Draft or Inactive assets should be deleted or modified instead.

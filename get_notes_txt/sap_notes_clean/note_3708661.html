Symptom

When executing a report based on logical database FTI_TR_POSITIONS with Key Date Reference "Posting date", the book value is not correct, comparing with TPM20 for the posted position relevant flows.

Environment

  • Treasury and Risk Management (TRM)
  • EA-FINSERV
  • S/4CORE
  • SAP ERP
  • SAP ERP Central Component
  • SAP enhancement package for SAP ERP
  • SAP S/4HANA
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA Finance

Cause

The derived business transactions flows are not fixed, which are not selected when executing the report with posting date.

Resolution

  • Fix the DBT flow via Tcode TPM18 for the relevant position.
  • Consider to change the "Status control" setting in customizing "Control Processing of Derived Business Transactions", instead of "F status plan", set it as "T Same status". The derived business transactions always have the same status as the original business transaction. 
    * Please note that the configuration change only has effect for newly created positions.

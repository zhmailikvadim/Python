Symptom

  • Onboarding 1.0 is scheduled for decommissioning later this year.
  • Assistance is required to determine the necessary steps for migration to Onboarding 2.0.
  • Clarification is needed on what data from Onboarding 1.0 should be saved or downloaded to secure all necessary information before deletion.

Environment

SAP SuccessFactors Onboarding 

Cause

Migration data requirements and prerequisites not met.

Resolution

1. Ensure the following role-based permissions are enabled:
       A.    Monitor Scheduled Jobs and Manage Scheduled Jobs permissions under Admin Center > Manager Permission Roles > Administrator Permissions > Admin Center Permissions.
       B. Import and Export permissions under Admin Center > Manager Permission Roles > Administrator Permissions > Onboarding or Offboarding Admin Object Permissions > ONB2LegacyONB1FormAudit.
2. Enable Onboarding and Employee Central modules.
3. Complete the onboarding process for all inflight candidates in Onboarding 1.0.
4. Verify that the Blob Storage System (BSS) migration is completed before initiating the document migration. Note that the migration job will fail if initiated before BSS migration completion.
5. Ensure sufficient storage space within the Document Migration System (DMS) in Onboarding. Check the storage space required for migration under Onboarding 1.0 > Onboarding 1.0 Dashboard > Feature Usage. By default, DMS provides 500 GB of storage space. Optimize storage by cleaning up unnecessary documents or purchase additional storage space if required.
6. Export documents not supported by the Document Management System (DMS) formats (DOC, PDF, CSV, HTM, PPT, XLS, GIF, PNG, JPG, JPEG, HTML, RTF, BMP, XLSX, DOCX, PPTX, TXT, XML, MSG, DOCM, TIF, EML) and save them locally before migration.
7. Ensure at least 10% of total storage space is available in addition to the space required for migration to prevent failure.
8. Temporarily set the Attachment under Provisioning > Company > Document Attachment to "No Limit" to avoid migration errors. Revert this setting after migration.
9. Initiate the migration job to transfer required documents (onboarding documents and historic audit data) from Document Centre (DC) in Onboarding 1.0 to Document Management System (DMS) in Onboarding 2.0.

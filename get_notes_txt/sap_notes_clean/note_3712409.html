Symptom

  • Delivery address is not transferred to purchase requisition (PR) in EBAN table when using the Fiori app "My Purchase Requisitions - New".
  • ADRNR field in EKPO table is blank for purchase orders (POs) created with reference to PRs generated by the new Fiori tile.
  • Issue occurs when selecting "user entry" or "plant" as delivery address type in the Fiori app.

Environment

SAP S/4HANA On Premise/Private Cloud

Reproducing the Issue

  1. Open the Fiori application "My Purchase Requisitions - New".
  2. Create a new purchase requisition (PR).
  3. Select "user entry" or "plant" as the delivery address type.
  4. Observe that the delivery address is not transferred to the EBAN table.
  5. Create a purchase order (PO) with reference to the PR created.
  6. Verify that the ADRNR field in the EKPO table remains blank.

Cause

Its a standard system configuration.

Resolution

  • The Fiori APP Manage Purchase Order doesn´t take the delivery address from Purchase Requisitions. There is not any change on this and was the same flow design on previous releases.
  • In the Fiori App the Delivery Address is redetermined when the Purchase requisition is converted to a Purchase Order. 
  • For keeping the Delivery address from the PR as the desired content, the usage of the App Create Purchase order Advanced is required.
  • The App Assign and Process Purchase Requisitions and Scheduling Purchasing jobs ( automatic conversion of PR) will also keep the PR delivery address in the PO.

Please check the BADI  "MM_PUR_S4_PO_MODIFY_DELIV_ADDR" as a potential valid workaround, where the reference delivery address number (i.e. REFERENCEDELIVERYADDRESSID = EKPO-ADRN2) can be set .

Symptom

When you calculate result-independent conditions, the system creates an account with an incorrect date of receipt.

Reason and Prerequisites

Prerequisites:

  • The total number of postings in the account exceeded the maximum number of postings entered for each account during the calculation.
  • The user starts the calculation in the following dialog programs:

    • Accounting Dialog Basic System

    • Risk Manager Account Dialog

    • Bordereau Basic System

    • Risk Manager Bordereau

    • Loss dialog at treaty or participation level
  • The user starts the calculation with the following background programs:

    • Background program "Transfer from Risk Manager to Basic System" with activated start parameter "Calculate Accounts"

    • Background program "Split, Calculate, Release (Basic System)"

    • Background program "Split, Calculate, Release Parallel to Treaty (Basic System)"

    • Background program "Split, Calculate, Release Parallel to Acct. (Basic System)"
  • The user starts the calculation on a day that differs from the creation date of the account.

This is because the system creates a new account during the calculation if it detects that the maximum number of postings has been exceeded. This account always receives the current date as the date of receipt, which does not correspond to the date of receipt of the calculated account.

Solution

History:
March 13, 2026: New

Troubleshooting Description:
If the above prerequisites are met, the system transfers the date of receipt of the calculated account to the new account.

Symptom

After performing work in Financial Consolidation rich client and logging out, some users experience longer-than-usual login times upon reconnecting, and any previously saved preferences are lost.

Environment

Financial Consolidation 10.1 SP09

Reproducing the Issue

  1. Log in to Financial Consolidation.
  2. Perform some work or make changes within the application (e.g., data entry, task execution...).
  3. log out of the application.
  4. Log in again to SAP Financial Consolidation.
  5. Notice a longer-than-usual connection time.
  6. Navigating back to the previous work section => The previously saved user preferences have been reset.

Cause

The issue is caused by corruption or inconsistencies in the user profile.

Resolution

Perform a “Save As” for the affected user to refresh the user profile.

Keywords

preferences, save, reset, lost, session, login, time, changes, fc, rich client

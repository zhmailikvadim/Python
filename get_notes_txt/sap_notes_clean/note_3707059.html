Symptom

  • Webservice call return HTTP 500, on an API testing tool
  • Error message in Data Services Management Console (DSMC), under Real Time > Status > Real Time Service > Process ID > Trace Log (or Real-Time Job history) is:
    • Transform USARegulatory_AddressCleanse Transform <USARegulatory_AddressCleanse>: <URC0001>: [SuiteLink Initialization Error - Fail to open SuiteLink file /dataservices/DataQuality/reference_data/slknine.lst.

Environment

  • SAP Data Services 4.3 and higher

Cause

  • The address directories for SuiteLink are not downloaded from SAP for Me software center.
  • Enable Suite Link, in the USA Regulatory Address Cleanse (URAC), is set to Yes.

Resolution

SuiteLink directories are only valid for companies in the United States of America, and cannot be used outside of it. And, SuiteLink adds secondary address information to your addresses and it is required for CASS. As of that, there are two solutions for this issue:

  1. If SuiteLink is needed, and it is a USA company, make sure to download from SAP for Me software center, in the Address Directories session, the SuiteLink directories (bear in mind that you have to purchase it)
  2. If there is no need to use SuiteLink, disable this option in URAC
    1. Open SAP Data Services Designer and navigate to the real-time job.
    2. Find the dataflow using URAC and double click it.
    3. Navigate to the options tab and locate the Assignment Options section.
    4. Set Enable SuiteLink to "No" to disable the use of the slknine.lst file.
      1. If the option is already set to "No", change it to "Yes", save all changes, and then revert it back to "No" to overwrite any corrupted parameters.

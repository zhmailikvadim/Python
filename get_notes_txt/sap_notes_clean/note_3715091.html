Symptom

During the JIT Delivery confirmation, the consignment issue order is created with a fixed order type (ED) instead of being determined based on the sales scheduling agreement configuration

Other Terms

NJIT, Next Generation Just-In-Time Processing, JIT Delivery Confirmation, Self Billing, Consignment issue order type

Reason and Prerequisites

Advance development

Solution

Implement the correction instructions of this SAP Note or import the relevant Support Package.

After implementing this note, the consignment issue order type will be determined based on the sales scheduling agreement used during the JIT delivery confirmation.

The system derives the relevant order type from the configuration maintained for the scheduling agreement type and uses it for consignment issue order creation.

If no configuration is maintained, the system continues to use order type 'ED' as a fallback.

This ensures that the consignment issue order is created in line with the configured business settings and avoids inconsistencies in subsequent processing.

Symptom

When an Activity containing Equipment or Functional Location is updated in FSM, and then the corresponding S/4HANA Service Order is updated in the backend system, the Equipment or Functional Location is removed from the associated Service Item.

Other Terms

FSM, E4C, Connector, S4 Service Order, Equipment, Functional location, Activity

Reason and Prerequisites

The issue occurs due to the wrong synchronization logic between FSM and S/4HANA. The issue occurs when Equipment or Functional location is added to the Service Order Item.

Solution

The synchronization logic has been adjusted. A new field "Change Equi" is now available in table /PACG/ECM_VLOTD. When the option "Create Object" is enabled, the Equipment or Functional Location updated in FSM will be replicated to the S/4HANA Service Order or the corresponding Service Item.

Please implement the provided correction instruction to apply this change.

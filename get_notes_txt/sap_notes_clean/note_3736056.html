Symptom

  • Guidance is needed to handle Czech Payroll regulatory reporting (JMHZ) for employee transfers within the same employer (§ 338) in REGZEC.
  • In REGZEC, actions for deregistration/registration (actions 1 and 2) are generated instead of transfer action code 5.
  • The scenario involves configuration and data in REGZEC, it0041, and related components, including dates and employee identifiers.
  • No specific error messages or error codes were mentioned.

Cause

Resolution

  1. For the transferring-out record (original VS):
  2. Create an IT3457 entry with validity equal to the transfer date.
  3. Do not fill the ID PPV field.
  4. Fill the “External VS” field with the target VS to which the employee is being transferred.
  5. Run the REGZEC report; it will generate the Transfer action (code 5) instead of Exit.

  1. For the transferring-in record (new VS):
  2. Create an it3457 entry with validity equal to the transfer date.
  3. Fill the ID PPV field.
  4. Set “Origin differentiation” to EXT-RZC.
  5. Fill the “External VS” field with the source VS from which the employee is transferred.
  6. Run the REGZEC report; it will not generate any action on the receiving side.
  7. When new identifiers (for example, IK-related data) are delivered by the authority via data mailbox, import them using program HCZCJHR0_SO as applicable to support creation of subtypes 1 and 2 in IT3457.

Symptom

• The net value displayed in the billing due list (VF04) differs from the net value displayed in VF01.
• This discrepancy occurs specifically in delivery-related billing scenarios.

Environment

  • SAP S/4HANA
  • SAP S/4HANA Cloud Private Edition

Cause

• The calculation of VKDFS-NETWR is based on each batch to determine the portion of quantity and perform conversion from sales UOM to base UOM (Convert SalesOrderQuantity to BaseQuantityUnit and then multiply the netwr with delivery quantity / sales quantity).
• The rounding differences occur because NETWR is length 15 decimal 2.
• This discrepancy affects only the billing due list and does not impact the actual invoice values generated from VF01 or VF04.
• The system calculates the condition value based on the total delivery quantity for each batch main item, thereby eliminating accumulated rounding differences.

Resolution

1. In include V05IEXIT, specifically in the subroutine USEREXIT_VKDFS_SORTKRI_DEL, modify VKDFS-NETWR according to custom own logic. 
2. Refer to SAP Note 170183 - What to consider when applying modifications.
3. To change the billing due list for existing deliveries, run the standard program RVV05IVB with the specified delivery document number (ensure to restrict to a small number range).
4. Select the "billing index" checkbox.
5. The billing due list (VKDFS) will be deleted and recreated for the selected deliveries, and your user-exit logic will be applied.

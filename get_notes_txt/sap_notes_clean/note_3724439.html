Symptom

You are replicating GL accounts from MDG-F to ECC via SOA channel. SOA payloads of type GeneralLedgerAccountMasterReplicationBulkRequest are sent out from MDG-F including attribute LedgerGroupSpecificClearingIndicator. This information is not interpreted by ECC inbound processing and field SKB1-XLGCLR is not getting updated from it as expected. This attribute can only be updated currently in ECC when using Idoc replication (GLMAST01).

Other Terms

GeneralLedgerAccountMasterReplicationBulkRequest, LedgerGroupSpecificClearingIndicator, GLAcctMstrRplctnReqCoDets, SKB1-XLGCLR, Open Item Management by Ledger Group, SAPPLCO_GLACCT_MSTR_RPLCTN_RE2, SMT mapping CL_FBS_GLACCTMSTRRPLCTNRQ

Reason and Prerequisites

Missing functionality in ECC: The SOA replication inbound processing of GL accounts so far does not support replication of attribute "Ledger Group-Specific Clearing" in ECC releases.

Solution

If possible, apply the corresponding Support Package to the relevant ECC recipient system(s). Alternatively, you can apply the note with SNOTE. Attention: You then need to perform the manual steps from the attached manual correction instruction in addition.
Afterwards, the ECC system can receive and map the LedgerGroupSpecificClearingIndicator field and update SKB1-XLGCLR.

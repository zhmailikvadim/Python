Symptom

  • Value entered in key figure i_adjcapaavailable cannot be saved in PWS or Excel add-in.

Environment

SAP Integrated Business Planning

Reproducing the Issue

1. Type some value in key figure I_ADJCAPAAVAILABLE.
2. The value is cleared after clicking on Save button.

Cause

The resource's UoM for Available Capacity of the key figure belong a different dimension that the I_ADJCAPAAVAILABLE loaded. The UoM conversion can not happen.
Thus the saving failed.

For example:
1. Resource 0010300001_1010_RM20 has EA UoM for Available Capacity.
2. In Target UoM fo Capa, 'S' is selected to load I_ADJCAPAAVAILABLE value.
3. Try to save some value for the key figure with resource 0010300001_1010_RM20, the value can not be saved.

Resolution

The value can be saved or converted when UoM for Available Capacity and Target UoM fo Capa belong to a same dimension.
(For example, 'S' and 'MIN' belong to time dimension, etc.)

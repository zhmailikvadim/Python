Symptom

When converting planned order to production order in Manage Planned Order app, error message "Runtime Error: 'SYSTEM_ILLEGAL_STATEMENT'. The OData request processing has been abnormally terminated. Analyze the runtime error using Feed Reader in ABAP development tools for Eclipse or using transaction /IWFND/ERROR_LOG or /IWBEP/ERROR_LOG or ST22. Create a support ticket on the application component of the OData service for errors in services delivered by SAP. " occurs.

Environment

SAP S/4HANA Cloud Public Edition

Reproducing the Issue

  1. Go to Manage Planned Order app.
  2. Select the Planned Order and click "Convert to" button.
  3. Error message "Runtime Error: 'SYSTEM_ILLEGAL_STATEMENT' "occurs.

Cause

One possible reason is the number ranges for production order have not been correctly assigned to a number group. Therefore, the dump will occur when converting the planned order to production order.

Resolution

Go to Convert Planned Order - to Production Order (CO40) app and convert the planned order with default order type. If error message CO367 occurs, fix the error referring to KBA 3720445 Error Message CO367 "Number range & is not created for object &" occurs in Convert Planned Order - to Production Order app. If not, raise a new case in PP-SFC-2CL component.

Keywords

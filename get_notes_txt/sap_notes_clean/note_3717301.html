Symptom

If in the method AT_SAVE of BAdI WORKORDER_UPDATE an error message is issued and the exception ERROR_WITH_MESSAGE is raised

- message id '<some message ID>' type 'E' number '<some message number>' raising error_with_message 

the message is not shown to the user in the app W0017 "Process Maintenance Order" ("Create Maintenance Order", "Change Maintenance Order"). Instead, the app terminates.

Cause

Due to technical limitations within the Web Dynpro framework, the application is unable to process error messages generated by the BAdI method WORKORDER_UPDATE AT_SAVE. This is because the timing of the BAdI's execution occurs too late in the process for the application to handle these messages effectively. 

Resolution

BAdI EAM_ORDER_FIELD_CONTROL can be utilized to designate the field as required. Additionally, there is another enhancement option available through the BAdI Framework /PLMB/EX_SPI_APPL_ACCESS (Adjustment of Data for Service Provider Access Methods). This allows you to implement checks and error messages for mandatory fields. Please ensure that the filter values are configured appropriately to avoid unnecessary implementation calls.  

For detailed guidance, refer to section 3.6 of the EAM Web UI Implementation Guide (Adapting EAM Web UI), which outlines the steps for this process. Furthermore, additional information on enhancement possibilities within the PLM framework, upon which the app W0017 is built, can be found in the Enhancement section of the SPI framework WIKI.

To understand how an example implementation works, you may review SAP Note 2918840. This example demonstrates the processing of the EAM order component, where the required field PRICE triggers an error message if the user does not provide content for the field. 

Further information can be found in the related KBA 2708958.

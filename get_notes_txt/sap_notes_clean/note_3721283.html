Symptom

The requirement is to maintain multiple VAT Registration Numbers in the BP transaction code.

Environment

  • SAP S/4HANA
  • SAP S/4HANA Cloud Private Edition

Reproducing the Issue

  1. Go to the BP transaction code.
  2. Maintain a VAT Registration Number with the same country code as the BP country.
  3. Repeat the step with another VAT Registration Number.
  4. Observe that error F2140 is triggered.

Cause

In standard only one VAT number can be added to one country. If the country of the BP and the tax country are same, the VAT registration number will be filled in the KNA1-STCEG/ LFA1-STCEG. 

Resolution

Additional VAT registration numbers can only be added if the country is different from the BP country. In this case, the VAT registration number is populated in the KNAS/LFAS tables. If multiple VAT registration numbers are still maintained for a single country, the following error message is triggered:

"VAT reg.no. already exists for country/region &" (F2140)

Please note that if a VAT registration number is entered in a BP linked to both a customer and a vendor, the VAT registration number is assigned to the customer as well as to the vendor. If a different VAT registration number is required for the customer and the vendor, a separate BP must be created.

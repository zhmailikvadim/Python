Symptom

You are using the Fiori app "Migrate Your Data" to migrate data to SAP S/4HANA using the migration approach "Migrate Data Directly from SAP System”.

If you use transaction SCC1 to transport migration projects or migration objects from one client to another client in the same system, rule repository information could be missing from table /LTB/TROWNER. This will cause the actions "Prepare Mapping Tasks", "Simulate Migration" or "Migrate Data" to fail. The system will display error messages such as: "The runtime of the transformation rule has not yet been defined. Initialize it first by generating the runtime object." or "Transformation Rule Repository XXX Not Registered".

Other Terms

Migrate Your Data, Migrate Data Directly from SAP System, Transport, Client copy

Reason and Prerequisites

A transformation rule is always linked to a so-called "repository". While transporting a migration project or migration object, all relevant configuration settings are added to a transport request except the repository information.

This repository information must be registered in the table LTB/TROWNER.

Deployment: S/4HANA 2023 FPS00~FPS05, S/4HANA 2025 FPS01

App: Migrate Your Data - Migration Cockpit

Migration Approach: Migrate Data Directly from SAP System (Direct Transfer)

Solution

Implement the corrections in SAP Note 3690056 and this SAP Note in the SAP S/4HANA system in accordance with the correction instructions.

After the implementation of the correction, activate the transformation rules for your migration project using 'Activate Rules' button in Migration Object Modeler menu. The system will generate the runtime objects for all available transformation rules and will also register the relevant repositories.

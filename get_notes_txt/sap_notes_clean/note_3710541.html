Symptom

Issues occur during the handling of extended table groups used for data replication scenarios:

  1. Extension of NEW_ROWS table groups for cluster tables with COMPRFLAG = P
    Systems may fail to correctly process or replicate data for cluster tables when the NEW_ROWS table groups are extended. This leads to inconsistencies or errors during replication activities involving compressed (P‑type) cluster tables.

  2. Incorrect DDIC generation for the Re‑Replication Table (extended) of pool tables
    In rare cases where re‑replication is triggered based on non-key fields for pool tables, the system generates an incorrect or incomplete DDIC structure for the extended Re‑Replication Table. Since re‑replication based on non-key fields for pool tables is an atypical scenario, the resulting DDIC mismatch can lead to activation errors or runtime issues during replication.

Other Terms

  • S4CORE 109 FPS00 - FPS01

Symptom

You are using F5241 Manage Maintenance Orders. You select all rows of a table (such as operation table or component/service table) including the empty row and click on "Remove". After all rows removed, you click on "Add" and no new empty row is added.

Other Terms

F5241, Manage Maintenance Orders, FioriUI, onAfterDelete

Reason and Prerequisites

Programm not executed properly due to missing empty value check.

Solution

Please implement this note. After implementation, a new empty row will be created automatically after all rows are removed.

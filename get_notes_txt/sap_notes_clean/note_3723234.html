Symptom

FAGL_FCV is run for ledger-specific open items.  The next valuation run is done to reset these ledger-specific open items. In the output list of the reset run, the system displays several ledger-specific open items, all with identical document numbers and amounts.

Reset postings of any other valuated ledger-specific open items are missing.

Further valuation runs on the same selection parameters process one single ledger-specific open item although the first run for these selection parameters processed several ledger-specific open items.

Other Terms

History, valuation

Reason and Prerequisites

The original valuation run valuates all selected ledger-specific open items. However, the reset run resets only the first of these ledger-specific open items. The system updates the valuation history table only for the first ledger-specific open item.

Solution

After implementing this SAP Note, the system correctly considers, resets, and updates all ledger-specific open items in the valuation history table. The system displays all items in the output list.

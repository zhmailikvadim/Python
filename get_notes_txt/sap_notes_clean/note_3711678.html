Symptom

During the posting of concur documents in transaction cte_post_moni (externa tax engine is called via SETI add on) the error:

"Tax entered incorrect (code RS, amount), correct CAD" is being raised in the error log.

Other Terms

SETI, TXINTF

Reason and Prerequisites

In transaction CTE_POST_MONI, when posting documents, an external tax engine is called via SETI.

In ECC, the Concur expense posting–tax calculation was performed through custom code. As a result, the tax engine was invoked exclusively within the custom object, producing the expected behavior: one seller call and one buyer call.

After the upgrade to S/4, the Concur expense posting–tax calculation was also initiated by standard FI code, in addition to the existing custom code call. Technically, the first invocation still occurs within the custom object, but a second invocation now occurs within the standard function module FI_TAX_CALCULATION. This is due to additional standard code introduced in S/4, which triggers the SETI object a second time.

Consequently, while ECC results in only the expected seller and buyer calls, S/4 initiates extra calls because the tax engine is invoked twice.

The primary root cause is the second invocation through the standard function module FI_TAX_CALCULATION.

Solution


To avoid the external call triggered by standard function module code, a check is introduced to restrict FI hook from concur transaction (internally, cte_doc_moni_start).

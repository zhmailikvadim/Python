Symptom

  • Certain job requisitions are not appearing on the Recruiting Marketing (RM) career site.
  • Although a job posting page URL is generated, accessing the link redirects users to the career site home page instead of the job details page.
  • In Command Center, the Automated Process Logs display the following error:
    •  "Job location : 1 : clientJobId : 1234-en_US Exception : Required elements cannot be blank. Ether (job title and job description) or (internal job title) or all 3 must be present along with all required fields.) [cliendJobId:1234-en_US description: ... location:xxxx title:xxxx internal Title :null applyUrl:xxxx ... and for jobs with multilocations, a primary location must be defined, location labels cannot be blank for any location, multiple primary locations cannot be listed ]"

Environment

  • SAP SuccessFactors Recruiting Management
  • SAP SuccessFactors Recruiting Marketing

Reproducing the Issue

  1. Post a job requisition
  2. Open the generated job posting URL.
  3. Observe that instead of opening the job details page, the link redirects to the career site home page.

Cause

When using the Location object (location_obj), one or more selected locations do not have a valid Country/Region maintained.

Resolution

  1. Navigate to Manage Organization, Pay and Job Structures in Admin Center.
  2. From the Search dropdown, select Location (or the customer‑specific label for the Location object).
  3. Search for and open the Location record that is used in the affected job requisition.
  4. In the Business Address section, update the Country/Region field to a valid, ISO-3166 standard country/region name.
If the selected location in the job requisition is a dummy location record or not a real geo location, remove it from the job requisition.

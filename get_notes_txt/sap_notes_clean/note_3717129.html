Symptom

An SAP HANA database in a system replication landscape has failed.

When trying to start the original primary again to make it a secondary, the instance fails to start.

In the service trace files, you see the entry

Inconsistent log segment information (already known LogSegment.........

Environment

SAP HANA Platform Edition with System Replication

Reproducing the Issue

  1. Encounter a failover to secondary site.
  2. Try to start original primary site after re-registering it as a secondary

Cause

The log area on original primary is only reset by re-registering using froce_full_replica parameter when re-registering

Resolution

Reregister the original primary using "Initiate full data shipping" in HANA studio or option --force_full_replica for command "hdbnsutil -sr_register".

This will initialize the persistence and when instance starts it will begin a full data shipping from the current primary. 

The instance will then be able to start.

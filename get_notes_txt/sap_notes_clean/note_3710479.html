Symptom

It is observed that planning file entry is updated differently in case of material phantom and BOM phantom.

  • Material phantom:
    when special procurement key= ''50-phantom'' is set in the material master (T-code MM02), 
    changing the BOM item quantity will update the NETCH flag for both the phantom material itself and the higher-level assembly.
  • BOM phantom:
    when special procurement key= ''50-phantom'' is not set in the material master but only set in the BOM item(T-code CS02),
    changing the BOM item quantity will update the NETCH flag only for the material itself but not for the higher-level assembly.

  • SAP ERP Release Independent

  • SAP Enhancement package for SAP ERP Release Independent

  • SAP Fiori for SAP ERP Release Independent
  • SAP Fiori for SAP S/4HANA Release Independent
  • SAP S/4HANA Cloud Private Edition Release Independent

Reproducing the Issue

  1. Open T-code CS02, change BOM item qty.
  2. Run MRP and check result.
  3. Check T-code MD21,
    observe that the planning file entry is updated differently.



Cause

This is standard behavior due to a known restriction, which is implemented for performance reasons avoiding unnecessary planning file entry updates.

In case SPK=50 is only maintained in the BOM level, then changes made to the BOM of the phantom assembly will not update the planning file entry of the header material.
For the reason that it would be necessary to perform a where-used list check for every change verifying each instance where the material is included as a dummy item in BOMs. 
However, implementing such a check to ensure PFE updated accordingly, would result in significant performance challenges making it unfeasible to proceed with this approach. 



Resolution

Generally, it is recommended to maintain the special procurement key=50 in material master level rather than BOM item level. 
In case using BOM phantom, please manually reset the planning file entry in T-code MD20 or program PPH_SETUP_MRPRECORDS.

*Possible workaround:
BAdI BOM_USE_PROPAGATION_SWITCHES in the ES PPH_MRPRECORD can be used to make the phantom propagation also for BOM phantom flags. 
*CAUTION: however such modification could lead to an increased performance side effect!

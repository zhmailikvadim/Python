Symptom

Requirement is to configure cXML catalogs instead of using OCI interface in Purchasing documents and in Plant Maintenance.

Environment

  • Materials Management (MM)
  • SAP ERP Central Component
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA

Reproducing the Issue

  1. Attempt to create a new Purchase Requisition using a cXML catalog.
  2. Observe that the item data are not transferred, error might be raised.

Cause

Submitting data in cXML format is not supported and there is no way to use it to transfer items from punchout catalogs to MM/PM documents. Function module WSI_IMPORT_DATA cannot handle cXML type.

Resolution

In order to use a catalog in MM/PM the catalog has to support the OCI standards and comply to the naming convention explained in the OCI documentation (NEW_ITEM-<Field name>[<index>]). The fields and field values have to be separated with "=" and the value pairs have to be separated with "&" characters.

When using XML variant of the OCI, the configuration has to be done as described in Part "XML Variant of the OCI" of the same documentation. In this case, the same architecture is used as in the pure HTML variant, this means that the XML data is embedded in an HTML form for the transfer from the catalog to the application.

Keywords

cxml catalog, oci catalog, catalog configuration

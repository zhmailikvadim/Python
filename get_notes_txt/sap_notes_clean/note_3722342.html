Symptom

Up to now, the accruals update (WB2R_UAC / WB2R_UAV / WB2R_UAP) only updates accruals up to the settlement date of the accruals update. If a contract is changed on February 27, a settlement date for February 27 is generated. However, at this point in time, documents (for example, SD billing documents) with the date February 28. These must also be updated, which is not the case up to now. This SAP Note changes the behavior so that the accruals update also determines and updates documents and their accruals up to the next final settlement date. This does not apply to normal delta accruals settlement.

Reason and Prerequisites

This is due to a program error.

Symptom

The Enqueue Replication Server (ERS) is unable to connect to the Standalone Enqueue Server.

The ERS trace file shows errors like "incomplete initial SSL/TLS handshake" and "SSSLRC_CONN_CLOSED".

The Enqueue Server trace files shows errors like "NiBufIIn: invalid data [...] peer=[IP address of ERS]".

Environment

  • SAP NetWeaver ABAP based product
  • Standalone Enqueue Server v1 (ENSAv1) with Enqueue Replication enabled

Cause

Inconsistent configuration of the Secure Internal Server Communication (parameter "system/secure_communication").

The ASCS would be running with the parameter set to OFF, while the ERS would be running with the parameter set to ON.

Resolution

The parameter "system/secure_communication" must have the same value on all instances of the system.

SAP recommends setting it at the DEFAULT.PFL only, to ensure the consistent configuration of this parameter.

Decide whether "system/secure_communication" should be ON or OFF and adjust the profiles accordingly.

Restart the relevant instances, to activate the desired configuration.

For example, set the parameter to OFF at the ERS profiles and restart the ERS, so it does not use the Secure Internal Server Communication (since the ASCS would not be using it, according to the scenario described on the "Symptom" section of this KBA).

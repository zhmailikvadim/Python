Symptom

Milestones tab is missing on the top-level WBS element even marked as billing element in Project Planning.

Environment

SAP S/4HANA Cloud Public Edition

Reproducing the Issue

  1. Open Project Control and navigate to Enterprise Projects.
  2. Go to Project Planning.
  3. Select the top-level WBS element and observe that the Milestones tab is not available.

Cause

Standard system design: the top-level WBS element (used for commercial integration such as sales order and billing plan management) does not provide the Milestones tab.
Milestones are maintained on subordinate WBS elements where planning functions are enabled.

Resolution

In Projects, there is a functional separation between WBS elements:

The top-level WBS element (Billing Element) is used for commercial integration, such as Sales Order creation and Billing plan management.
Milestones, on the other hand, are part of project planning and execution and are therefore maintained on lower-level WBS elements, where planning functions are enabled.

As a result, the Milestones tab is not available on the top-level WBS, but is available on subordinate WBS elements. This is consistent across projects and represents the standard system design.

Milestones maintained on child WBS elements can still be used to:

  • Generate billing plan dates.
  • Control milestone-based billing via the Sales Order

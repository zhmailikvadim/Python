Other Terms

Deadlock, Logging, LogController, J2EE

Reason and Prerequisites

There are tho methods in LogController class..

  1. public synchronized void reset() { .. }
  2. private synchronized boolean addLogNormalInt(Log log) { .. }

.. which are not synchronized correctly. They are synchronized by the current class instance (this).

Solution

In order to avoid the deadlock, the synchronization of the two methods should be made by class literal (getCLass()).

The fix is performed and can be applied by upgrading to the latest version of J2EE. See SP Patch Level section for more details.

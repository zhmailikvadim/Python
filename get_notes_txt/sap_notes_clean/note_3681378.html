Symptom

While updating the Capitalization document, an error occurs: Field cannot be updated (ODBC -1029).

Reproducing the issue

Prerequisites:

Go to Administration → System Initialization → Company Details → Basic Initialization tab and select the Enable Fixed Assets checkbox.

Scenario:

  1. Using SAP Business One client, create a fixed asset item as 'FA01'.
  2. Use the following DI API code to create an A/P Invoice having the fixed asset item 'FA01' on rows:

    SAPbobsCOM.Documents oAPInvoices = (SAPbobsCOM.Documents)oCompany.GetBusinessObject(SAPbobsCOM.BoObjectTypes.oPurchaseInvoices);
    oAPInvoices.CardCode = "V10000";
    oAPInvoices.Comments = "Added via DI";
    oAPInvoices.DocDate = new DateTime(2025, 11, 11);
    oAPInvoices.Lines.ItemCode = "FA01";
    oAPInvoices.Lines.UnitPrice = 134;
    int AddAPInvoice = oAPInvoices.Add();

  3. After the A/P Invoice is successfully added, the system automatically generates a Capitalization document.
  4. Using SAP Business One client, open the generated Capitalization document and attempt to update the value in the Remarks field for its rows.

Expected behavior:

The Capitalization document is updated successfully.

Actual behavior:

An error occurs: Field cannot be updated (ODBC -1029).

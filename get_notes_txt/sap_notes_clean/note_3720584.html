Symptom

Resolved multiple issues related to incorrect change‑detection and data synchronization:

  • Fixed unwanted “unsaved changes” modals that were triggered even when no actual modifications were made to the data.
  • Corrected synchronization logic that could allow restoration of outdated values or values originating from a different invoice.
  • Improved internal state handling to ensure consistent data integrity and prevent accidental reversion to stale or mismatched item data.

Other Terms

ISR-procedure, QR-IBAN, QR-BILL, parking, FBV2

Reason and Prerequisites

These improvements ensure that modals only appear when genuine changes occur and that restored values always correspond to the correct, current invoice context.

Solution

In general, SAP recommends that you implement the Support Package specified for your release. However, if you need to install a solution earlier, use the SAP Note Assistant to implement the correction instruction. You can find more information about the SAP Note Assistant in SAP Support Portal, under SAP Note Assistant.

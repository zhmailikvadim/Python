Symptom

  • Flow discovery has been enabled, and the integration appears active, but no discovered flows between systems are found.
  • Flow objects are not being fetched in LeanIX SAP Discovery facilitated by SAP Cloud ALM.
  • How to verify the integration between SAP LeanIX and SAP Cloud ALM for flow discovery
  • How to identify the integration monitoring messages will discovered as flow objects.

*Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental. 

Environment

  • SAP Cloud ALM
  • SAP LeanIX

Reproducing the Issue

  1. Enable the SAP Cloud ALM Integration Monitoring for the managed SAP components following the Integration monitoring setup guide.
  2. Enable the flow discovery integration between SAP Cloud ALM and SAP LeanIX.
  3. Wait for 24-28 hours to allow flow discovery to detect flow objects.
  4. Check the integration status in LeanIX SAP Discovery, and in the Sync logs for SAP Discovery, it shows 0 flow objects discovered. 

Cause

  • Integration monitoring determine the messages flows with the Transaction ID.
    For example, there is message collected for system A with transaction ID XYZ, and there is message collected for another system B with the same transaction ID, then a flow System A -> System B will be determined by Integration Monitoring. 
  • Only related messages for different systems or services will be determined as flow, and discovered by LeanIX.

Resolution

  1. Verify that "Enable Flow Discovery" is toggled on in SAP Cloud ALM, which means the related messages for different systems will be determined as flow by the Integration Monitoring Alert Calculation job.
    This can be checked via the steps:
    1. Login to SAP Cloud ALM
    2. Go to Integration & Exception Monitoring app under Operation
    3. Click on the  Configuration button
    4.  under the Application settings, the Flow Discovery, 



  2. Make sure there is related messages for different systems/services collected in Integration monitoring.
    This can be checked via the following steps:
    1. Login to SAP Cloud ALM
    2. Go to the Operation, then open Integration & Exception Monitoring application
    3. Select the scope for the services/systems which has integration messages enabled.
    4. Open the messages collected for the target systems, and go to the message details
    5. Under the tab "Related Messages &Exception" check whether there is related messages information for different services/systems.

      In the example blow, the flow S/4HANACloud ->SAPAribaProcurement ->SAPAribaCIG will be determined. 



    6. The related message is detected for the messages with the same Transaction ID, which can be checked under the Technical Correlation in the message details.



      For the IDOC messages in SAP ERP or SAP S/4HANA system, please implement SAP Note 3259248 to activate the Collection of SAP Passport in SAP Cloud ALM for IDoc Messages. 

    7. If there is no related messages in different systems collected, there is no flow detected. 

  3. Make sure the SAP Cloud ALM Service key used in the flow discovery integration has the scopes "ops-im-configure-ui" and "ops-im-display-ui" included.
    This can be checked in SAP Cloud ALM Subaccount in BTP Cockpit:
    1. Login to the BTP Cockpit and go to the SAP Cloud ALM Subaccount
    2. Under the Instances and Subscriptions tab 
    3. find the SAP Cloud ALM API instance for the integration
    4. Click on the 3 dots (action) 
    5. View Parameters

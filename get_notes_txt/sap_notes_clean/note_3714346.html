Symptom

The base amount values in the ELSTER report are rounded decimals.

Reproducing the issue

Prerequisites:

  • You are working in the localization for Germany (DE).
  • The ELSTER add-on is installed and enabled.
  • You are using the latest version of the SAP Business One ELSTER add-on, patch 11 or patch 12.
Scenario:
  1. Go to Financials → Financial Reports → Accounting → Tax → Tax Report.
  2. Create a tax report for a month. Select an A/P or A/R invoice where the base amount decimal part is greater than 0.5, such as 89.99.
  3. Choose the Elster button to open the Elster window.
  4. On the Retrieve tab, choose the Retrieve Data button. You receive a message: The data has been successfully retrieved; it can be transmitted.
  5. Choose the Print Report button.

Expected behavior:

The system truncates the decimal part of the base amount value. The value in the ELSTER report appears as 89.00.

Actual behavior:

The system rounds the decimal part of the base amount value. The value in the ELSTER report appears as 90.00.

Solution

Symptom

I have updated my SSO certificates in the Configuration Manager, but I still cannot log in via SSO. The authentication appears to reach the Identity Provider successfully, but fails when redirecting back to SAP Fieldglass.

Cause

The Identity Provider is sending an attribute name (e.g., SSOID) that does not match the expected attribute name defined in the SAP Fieldglass SSO configuration. If the configuration is set to look for a specific attribute name and the IdP sends a different one (or a generic one), the mapping fails.

Resolution

You must ensure the attribute names in your Identity Provider (IdP) match the "SAML Attribute Name" defined in your SAP Fieldglass configuration.

  1. Log in with your Configuration Manager account.
  2. Navigate to the Single Sign-On tile.
  3. Click Edit on your existing configuration.
    • Note: The specific attribute name may not be visible on the Overview page; you must enter the Edit mode to see it.
  4. Verify the field labeled SAML Attribute Name (e.g., SSOID).
  5. Confirm with your IT/IdP team that the attribute being sent in the SAML assertion exactly matches this name, including case sensitivity.
  6. Click Update and test the login again.

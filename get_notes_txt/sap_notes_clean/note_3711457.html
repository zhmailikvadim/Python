Symptom

  • The "EsigibilitaIVA" field within the Italian e-invoicing XML in the SAP S/4HANA Cloud Public Edition system defaults to "I" (VAT due immediately), regardless of the NATURE_CODE.
  • How to populate the field with "S" (Split Payment)?

Environment

  • SAP S/4HANA Cloud Public Edition
  • SAP Document and Reporting Edition, Cloud Edition
 

Reproducing the Issue

  1. eDocument cockpit application
  2. Select the source document
  3. Display XML
  4. Locate the "EsigibilitaIVA" field within the XML.
  5. Observe that the field defaults to "I" regardless of the NATURE_CODE.
  6. Attempt to configure the tax code or flag the customer as Split Payment relevant.

Cause

  • The "EsigibilitaIVA" field inside "DatiRiepilogo" defaults to "I" because Split Payment is not explicitly activated.
  • The system does not map the field based on the NATURE_CODE.
  • Here is the abbreviation for each value in EsigibilitaIVA field: 

    I - VAT due immediately
    D - VAT due later
    S - Split Payment (VAT split payments)

Resolution

  1. Configure the value mapping for Natura Code (GET_NATURA_CODE) under namespace /EDOIT.
  2. Ensure that the tax code 'S1' is configured for Split Payment.
  3. Verify that the customer is flagged as Split Payment relevant.

Refer to the SAP Help documentation on configuring value mapping for Natura Code (GET_NATURA_CODE): Configuring Value Mapping for Natura Code (GET_NATURA_CODE) (Mandatory) | SAP Help Portal.

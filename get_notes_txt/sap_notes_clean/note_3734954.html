Symptom

When updating SPAM version, get below dump. Many same dumps can be found in ST22.
******************************
The field "GC_CLOUD_GEN_COMP" is unknown, but there is a field with the similar name "GC_PLUGIN_COMP". 
******************************

Environment

SAP_APPL 604 or higher

Reproducing the Issue

Update SPAM version.

Cause

When updating the SPAM version, scheduled application batch job is executed. The job calls function module CIF_CHECK_BGRFC_STATE. Function module CIF_CHECK_BGRFC_STATE calls function OCS_GET_COMPONENT_STATE.

SE37 >> CIF_CHECK_BGRFC_STATE
**************************
* Check if the bgRFC API is installed correctly

  CALL FUNCTION 'OCS_GET_COMPONENT_STATE'
    EXPORTING
      iv_component  = 'SAP_BASIS'
    IMPORTING
      ev_comp_rel   = lv_comp_rel
      ev_comp_level = lv_comp_level
      ev_last_patch = lv_last_patch
    EXCEPTIONS
      OTHERS        = 0.

  IF ( lv_comp_level <= 0013 AND lv_comp_rel = 700 ) OR lv_comp_rel < 700.
    RAISE wrong_bgrfc_version. "At least NW 7.00 SP14 is needed!
  ENDIF.
**************************

During the SPAM version update, FUNCTION OCS_GET_COMPONENT_STATE is locked. Then the batch job calls function module CIF_CHECK_BGRFC_STATE. OCS_GET_COMPONENT_STATE is called by function module CIF_CHECK_BGRFC_STATE.
As the FUNCTION OCS_GET_COMPONENT_STATE is locked, dump occur when function module CIF_CHECK_BGRFC_STATE is called. The job failed, but it will be triggered again soon. Thus, many dumps occurred until the SPAM update completed.

Resolution

As the SPAM version update completed, the dumps will not occur in the future. But it may occur in next SPAM update. Note 3483014 can remove the trigger of FUNCTION OCS_GET_COMPONENT_STATE for function module CIF_CHECK_BGRFC_STATE. To avoid the issue in the future, implement Note 3483014 before starting the SPAM version update.

See Also

3483014 - No need to check SAP_BASIS release any longer at function module CIF_CHECK_BGRFC_STATE -> increase performance of PIR change transfer and other 
For more information about upgrade/update troubleshooting, please refer to System Upgrade & Update Troubleshooting Wiki.

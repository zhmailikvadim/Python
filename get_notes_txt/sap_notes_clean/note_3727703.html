Symptom

  • Web Intelligence processing server crashes when resolving a comment linked to a block containing a dimension with an empty value

Other Terms

webi, webintelligence, WIPS, restart, coreinfo, shut down, unexpected, Termination, crashing, stop, calculation, break, filter

Reason and Prerequisites

Environment:

  • SAP BusinessObjects Business Intelligence platform 4.3 Support Package 05
  • SAP BusinessObjects Business Intelligence platform 2025 Patch 08
  • Web Intelligence 

Steps To Reproduce:

  1. Create a Web Intelligence document with some dimensions that can return empty as one of it's value and measures.
  2. Create a table in the report with multiple dimension columns including the dimensions that returns empty as one of it's value.
  3. Add a break on these dimensions that contains empty values.
  4. Add a comment to a cell in the table that is positioned on a row where the value is empty for both the dimensions.
  5. Apply a filter on the document so that the table return empty values for all the dimensions.
  6. Save and refresh/re-execute the document.
    1.  >> Web Intelligence processing server crashes.

Cause:
  • This issue is caused by a defect in the product.

Solution

This issue is fixed in the patches listed in the "Support Packages & Patches" section below.
Due to different internal processes, the "Support Packages & Patches" section of the SAP Note will be populated as below:

  • for fixes delivered in "Support Packages": immediately (even if the "Support Package" is planned to be released several months later)
  • for fixes delivered in "Patches": ONLY once the "Patch" is released AND available on SMP (not before)

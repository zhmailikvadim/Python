Symptom

  • The PMingLiU（新细明体) font is not displayed correctly in uploaded PowerPoint files within SAP SuccessFactors Work Zone.
  • The rendered font differs from the original file, and this issue is consistent across different folders, workspaces, and browsers. 

Environment

SAP SuccessFactors Work Zone 

SAP Build Work Zone, advanced edition

Reproducing the Issue

  1. Access SAP SuccessFactors Work Zone.
  2. Upload PowerPoint files to any workspace or content area.
  3. Observe that the PMingLiU（新细明体) fonts displayed does not match the original files. 

Cause

The font discrepancy occurs due to a known limitation in LibreOffice, which is unable to accurately interpret localized Chinese font names (e.g., “新细明体 / PMingLiU Headings”) used in PPTX files. This results in font fallback in SAP SuccessFactors Work Zone, causing inconsistent font rendering compared to the original files. 

Resolution

1. Currently, there is no fix available for this issue. It is officially recognized and tracked by LibreOffice, but as an open-source project, no timeline for resolution has been provided. 

2. As a workaround, replace the PMingLiU (新細明體) font in your PowerPoint files with another supported font before uploading to Work Zone. LibreOffice can correctly render other widely used fonts, ensuring consistent display. 

3. We will continue collaborating with the LibreOffice community and update this article if there are any changes to the situation.

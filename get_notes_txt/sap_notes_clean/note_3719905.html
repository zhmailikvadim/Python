Symptom

In the Manage Cash Pools (Version 2) application, when creating a new cash pool, the user selects the Value Help button for either the Cash Pool Name or Authorization Group fields. On the initial load, the value help correctly displays the available data. However, when the user clicks the Go button without entering any search criteria, the system returns “No data found”, even though data exists. The expected behavior is that the previously displayed data (or all available entries) should be shown when no search filters are applied.

Other Terms

F3266A

Reason and Prerequisites

Program error.

Solution

Implement the attached Correction Instruction (CI) or the relevant Support Package.

Symptom

During the processing of Acord messages, the AU check must be skipped manually several times.

Other Terms

XDI

Accounting unit (AU)

ClaimMovement (CM)

TechAccount (TA)

Reason and Prerequisites

Prerequisite: 

A CM message was processed for which the AU check was manually skipped.

The system now processes a TA message that refers to this CM message.

Reason: 

The AU check does not take into account the case that the same check has already been manually skipped in the referenced CM message.

Solution

History: 

February 11, 2026: New

Troubleshooting Description:

In the AU check of a TA message, the system now first checks whether a CM message is referenced when the error occurs.

If this is the case, the event log of this CM message is checked to find out whether the AU check has already been skipped here.

In this case, the error is not triggered for the TA message and only an informational message is delivered.

Technical effects: 

In the scenario described above, an AU check error no longer occurs for a TA message.

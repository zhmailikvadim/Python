Symptom

  • HTTP 500 error encountered on BI Launchpad.
  • stderr.log shows
    • javax.net.ssl|SEVERE|5B|https-openssl-nio-7104-exec-10|2025-12-10 18:07:19.286 GMT|TransportContext.java:369|Fatal (CERTIFICATE_UNKNOWN): sun.security.validator.ValidatorException: PKIX path building failed: 
      sun.security.provider.certpath.SunCertPathBuilderException: unable to find valid certification path to requested target ("throwable" : {  sun.security.validator.ValidatorException: PKIX path building failed: sun.security.provider.certpath.SunCertPathBuilderException: unable to find valid certification path to requested target
      Caused by: sun.security.provider.certpath.SunCertPathBuilderException: unable to find valid certification path to requested target

Environment

  • SAP BusinessObjects Business Intelligence platform 4.3/2025
  • Windows Server
  • Tomcat

Reproducing the Issue

  1. Replace the expired Tomcat certificate with a new certificate.
  2. Sign the new certificate with an internal Certificate Authority (CA).
  3. Attempt to access the BI Launchpad via the HTTPS URL.
  4. Encounter HTTP 500 error.

Cause

  • Internal CA signed certificate not found in the Java Virtual Machine (JVM) truststore (cacerts) used by Tomcat.

Resolution

  1. Retrieve the Tomcat (or load balancer) HTTPS certificate:
    1. Copy the Tomcat (or load balancer) HTTPS URL into the browser address bar.
    2. Press enter to navigate to this URL.
    3. Click where it says 'Not Secure', next to the URL.
    4. Select 'Certificate is not Valid'.
    5. A window should open with the certificate details (Certificate Viewer).
    6. Select the 'Details' tab.
    7. Click 'Export'.
    8. Save the certificate, changing the format to DER Encoded Binary (.der).
    9. Click Save.
    10. Copy the exported certificate to the machine(s) where Tomcat is located.

  2. Import the certificate into the JVM truststore (cacerts):
    1. Open a command prompt.
    2. Navigate to the following location: <InstallDirecotory>\Program Files (x86)\SAP BusinessObjects\SAP BusinessObjects Enterprise XI 4.0\win64_x64\sapjvm\bin
    3. Run the following command:
      keytool -import -noprompt -trustcacerts -alias -file -keystore ../jre/lib/security/cacerts -storepass changeit
      Example:
      keytool -import -noprompt -trustcacerts -alias CTSPM -file c:/ssl/tomcatcert.der -keystore ../jre/lib/security/cacerts -storepass changeit
    4. Restart the Tomcat to apply the changes.

    Note: Ensure that all certificates are created for the server host Fully Qualified Domain Names (FQDNs) or domain names (Common Names) to avoid SSL exceptions for server name mismatches.

Symptom

Unable to access SAP GUI in S/4Hana Public Cloud.

When attempting to access Web GUI in SAP S/4HANA Public Cloud using the URL: /sap/bc/gui/sap/its/webgui

Error message:

"Call /sap/bc/gui/sap/its/webgui/ to transaction was terminated because: No transaction provided. This is not allowed by service application. Check ITS process trace on vhshpalkci_ALK_00 (8) at 20250927065137. Server return code: HTTP 409. Server time: 2025/09/27 06:51:37."

Environment

  • Identity and Access Management
  • SAP S/4HANA Public Cloud

Reproducing the Issue

  1. Attempt to access SAP GUI in S/4Hana Public Cloud.
  2. Encounter the error message: "Call /sap/bc/gui/sap/its/webgui/ to transaction was terminated because: No transaction provided. This is not allowed by service application. Check ITS process trace on vhshpalkci_ALK_00 (8) at 20250927065137. Server return code: HTTP 409. Server time: 2025/09/27 06:51:37."

Cause

The behavior is working as designed.

Resolution

  1. The interface behavior is functioning as designed.
  2. Refer to SAP Note 0002372292 - SAP GUI for HTML: Session aborted due to transaction filter (BAdI SWEBGUI_TX_FILTER implementation) for clarification on the intended functionality of the interface.
  3. Utilize the Fiori launchpad to access business-related applications, as SAP GUI and Web GUI are not enabled in S/4Hana Cloud Public Edition due to product restrictions for stability purposes.

Review the blog titled  What’s different – 3 new key functionalities in SA... - SAP Community for insights into optimizing access to relevant applications.

Symptom

  • The system does not block sales orders for embargo when the partner functions Ship-To and End User have the same business partner, the same country (UA), but different regions (Ship-To region is empty, End User region is 14 for ex.).
  • The expectation is that the sales order should be blocked because one partner address meets the embargo condition (legal regulation, country/region, region).
  • The issue occurs after upgrading to GTSCORE 300, which introduced region as a condition in embargo checks.
  • After implementing an earlier correction version, a short dump occurred during recheck of existing documents and creation of new documents.
  • No specific error message text or error codes were provided.

Environment

  • Product: SAP Global Trade Services, edition for SAP HANA
  • Global Trade Logistics.

Reproducing the Issue

  1. Create a sales order with the same business partner assigned to the Ship-To and End User partner functions.
  2. Save the sales order so it runs to GTS for embargo check; observe that it is not blocked.
  3. Change the Ship-To address to country UA and leave region blank.
  4. Change the End User address to country UA and set region for ex., to 14.
  5. Run the embargo check again; observe the order is not blocked, which is not expected.

Cause

  • A defect in the initial embargo region logic: the original coding did not consider partners having the same country/region, which led to missing embargo blocks.

Resolution

  1. Implement the latest version of SAP Note 3726337 in the affected system.
  2. If an earlier version of the note was already implemented and a short dump occurs during recheck or new document creation, update to the latest version of SAP Note 3726337 and redeploy.
  3. Re-run the embargo checks for sales orders where Ship-To has no region and End User has a specific region to confirm the sales order is correctly blocked.
  4. Recheck previously created documents as needed to ensure consistent behavior.

See Also

Symptom

  • Unable to create a Pareto chart based on a HANA Live connection.
  • When creating a Pareto chart based on an SAP HANA Live connection in SAP Analytics Cloud, the following message is shown: 
      
    "Cannot display pareto chart because the server does not support sorted running total calculation."

Environment

  • SAP Analytics Cloud
  • SAP HANA 2.0

Reproducing the Issue

  1. In SAP Analytics Cloud, create an SAP HANA Live connection.
  2. Create a story.
  3. Add a chart and select a live model based on the SAP HANA Live connection.
  4. Select the chart type: Pareto.
  5. Select a measure and a dimension.
  6. It displays on the pareto chart the message: 
    "Cannot display pareto chart because the server does not support sorted running total calculation."

Cause

  • Pareto charts rely on server-side sorted running total calculations. This functionality is only available from the latest version of SAP HANA versions.
  • Earlier SAP HANA versions used via Live connection do not provide the required capability and lead to the error message.

Resolution

  • The required SAP HANA functionality to perform the calculations required for Pareto chart was introduced in:
    • SAP HANA Cloud 2025.40 (Q4 2025)
    • SAP HANA 2.0 SP07 revision 79.06 and higher
  • To be able to create pareto chart in SAP Analytics Cloud based on an SAP HANA Live connection:

    1. Verify the SAP HANA version behind the Live connection.
         
    2. If the version is below the supported minimum, update SAP HANA version:
      • SAP HANA Cloud 2025.40 (Q4 2025) or higher.
      • SAP HANA 2.0 SP07 rev 2.00.079.06.1754480252 or higher
        • 2.00.079.07.1759314838
        • 2.00.079.08.1766434523
        • 2.00.087.00.1756091668
        • 2.00.088.00.1760424921
        • 2.00.089.00.1766998439
        • 2.00.089.01.1769502981

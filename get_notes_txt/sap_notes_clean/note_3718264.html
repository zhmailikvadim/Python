Symptom

Run DQ job with exporting data quality reports option in a password encrypted Sybase ASE repo will throw error:

Failed to export report 'MATCH DUPLICATE SAMPLE'.  The error is: Logon Error: JZ00L: Login failed.  Examine the SQLWarnings chained to this exception for the reason(s).

Other Terms

This issue is tracked in CM: 002075125800001143402026

Environment:

Data Services, Data Quality Reports, Sybase ASE, Password Encryption

Steps to reproduce:

- Use a Sybase ASE database with password encryption enabled using command: sp_configure 'net password encryption reqd',1

- Create a DS repo with this Sybase ASE db;

- Create a DQ job in this repo;

- Run the DQ job with option "Export Data Quality reports";

- Job itself may finish successfully, but when exporting the reports, it will throw error.

Reason and Prerequisites

The root cause of the issue is due to exporting reports for password encryption enabled Sybase ASE repo is not supported.

Solution

 The correction is delivered in the release(s) listed in Support Packages & Patches section of this SAP Note.

Symptom

  • Payment Medium Variant is created and available after transport but is not automatically assigned to the Payment Medium/House Bank.
  • Manual assignment may be required in each subsequent system

Environment

  • SAP ERP
  • SAP S/4HANA

Reproducing the Issue

  1. Import the applicable transport(s) into the target system.
  2. Verify that the Payment Medium Variant exists in the target system.
  3. Check whether the variant is automatically assigned to the Payment Medium/House Bank in OBPM4.

Cause

By design, selection variants and their assignments are not included automatically in transports and are not automatically assigned in target systems; they must be transported and verified separately.

Resolution

  1. In the source system, run program RSTRANSP (for example via SE38/SA38) to transport selection variants.
  2. On the RSTRANSP selection screen, choose the relevant application, set Program to SAPFPAYM, and select the required selection variants (stored in table DFPAYV).
  3. Add the selected variants to a transport request and release the request.
  4. Import the transport into the target system(s) using the standard transport procedure.
  5. In the target system, open OBPM4 and verify that the transported selection variants are available.

Alternatively, from OBPM4 you can use “Transport to all formats” (F9), which navigates to RSTRANSP to include variants in a transport.

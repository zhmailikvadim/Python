Symptom

  • When manually searching for a Purchase Order in an invoice, the list of available POs is automatically filtered by the Company Code of the invoice
  • On the other hand, the PO list is not filtered by the Supplier of the invoice

Reproducing the Issue

  1. Open Manage Supplier Invoices Centrally
  2. Open an invoice
  3. Go to Reference Objects and choose Purchase Orders to start a manual search.
  4. Observe that the list of POs is associated to the Company Code of the invoice
  5. However, the list of POs belong to multiple suppliers and not only to the supplier of the respective invoice

Resolution

This is an expected behavior at the moment.

We recommend to submit an enhancement request if you like to see this functionality available in future releases.

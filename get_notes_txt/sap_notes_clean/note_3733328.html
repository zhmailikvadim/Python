Symptom

After importing the SAP_BASIS content (SPXX) metadata into the ESR, RFC adapter channels doesn't work and shows the below error,

Exception: com.sap.aii.adapter.rfc.core.config.RfcConfigException: got error from CPA cache while parsing configuration for server:com.sap.aii.af.service.cpa.impl.exception.CPAObjectException: Value with name serverUnicode not available

Environment

  • SAP NetWeaver 7.50
  • SAP Process Integration

Cause

The cause for the issue is the higher metadata version SP2X when compared to the system SP3Y(metadata SPXX> system SP3Y) the metadata is a lot newer than the actual SP of the PI and changes have been made to it in more recent SPs and it will not work on older ones. Always ensure to have the system SP= metadata SP.

Resolution

To resolve the error you can use below 2 steps,

  1. If you need the adapter to start working again on your system then you need to get the SP2X metadata from the software center (XI CONTENT SAP_BASIS 7.50 SP2X).
    You can check the link: Software Center - SAP for Me  
  2. The best solution will be to upgrade your PI system SP to SP3Y as SP2X is old and the RFC adapter also uses the old JCo2.

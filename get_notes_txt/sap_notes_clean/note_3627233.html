Symptom

  • When trying to schedule any document in BILP (BI Launchpad) after upgrade to BI 2025, the following error is seen:
    RWS 00070 Internal Server Error

  • In the BILP Web application logs, searching for 'scheduleform' reveals the following exception:
    General exception caught
    java.lang.ClassCastException: class com.crystaldecisions.sdk.occa.infostore.internal.InfoObject cannot be cast to class com.crystaldecisions.sdk.occa.infostore.IDestinationPlugin (com.crystaldecisions.sdk.occa.infostore.internal.InfoObject and com.crystaldecisions.sdk.occa.infostore.IDestinationPlugin are in unnamed module of loader org.eclipse.osgi.internal.loader.EquinoxClassLoader @5f09d87a)
    at com.sap.bip.rs.publication.DestinationUtil.initPluginObjects(DestinationUtil.java:108)
    at com.sap.bip.rs.publication.DestinationUtil.<init>(DestinationUtil.java:92)
    at com.sap.bip.rs.caf.processor.ScheduleDestinationsProcessor.read(ScheduleDestinationsProcessor.java:142) 

  • In Central Management Console (CMC), when clicking on the Destination tab, the following error may be seen:
    The argument has an invalid value CrystalEnterprise.StreamWork (FWM 02024)

Reason and Prerequisites

Environment

  • SAP BusinessObjects Business Intelligence (BI) Platform 2025
  • Windows
  • Linux / Unix 

Steps to Reproduce

  1. Upgrade to BI 2025
  2. Open BILP
  3. Navigate to a document
  4. Right-click -> Schedule
  5. Observe the RWS 00070 error is seen

Cause

Deprecated components (specifically CrystalEnterprise.Streamwork destination plugin) were not removed properly during the installation.

This may occur more frequently in very large (CMS DB size) environments

Solution

The issue has been forwarded to development for further investigation

Workaround

First - try to run a repair which will rerun the commands to remove the content:
2468513 - Repairing the BI Platform [Best Practices]

If the repair doesn't work, follow the below:

  • ALWAYS ENSURE YOU HAVE A FULL WORKING BACKUP OF YOUR CMS REPOSITORY BEFORE DELETING DEFAULT OBJECTS (DFOs)
  1. Download attached DFOCleanup script onto the system
  2. Open a command line in the directory containing the script
  3. Run the following commands:

    Windows:
    DfoCleanUp.bat -c <CMSName:Port> -p <AdministratorPassword> -cuid AZOfPrN8IT5Fr1rODnpGT6Q
    DfoCleanUp.bat -c <CMSName:Port> -p <AdministratorPassword> -cuid AT.xjxqMgjlPpEBvBlUOAhs
    DfoCleanUp.bat -c <CMSName:Port> -p <AdministratorPassword> -cuid ATmZqy1fNIBAlOO5qgNaSWE
    DfoCleanUp.bat -c <CMSName:Port> -p <AdministratorPassword> -cuid ARb7jByTWddPsGf0jSexBgw
    DfoCleanUp.bat -c <CMSName:Port> -p <AdministratorPassword> -cuid AUQAptxEqPhJkaLOEvsD1Wo

    Linux / Unix:
    DfoCleanUp.sh -c <CMSName:Port> -p <AdministratorPassword> -cuid AZOfPrN8IT5Fr1rODnpGT6Q
    DfoCleanUp.sh -c <CMSName:Port> -p <AdministratorPassword> -cuid AT.xjxqMgjlPpEBvBlUOAhs
    DfoCleanUp.sh -c <CMSName:Port> -p <AdministratorPassword> -cuid ATmZqy1fNIBAlOO5qgNaSWE
    DfoCleanUp.sh -c <CMSName:Port> -p <AdministratorPassword> -cuid ARb7jByTWddPsGf0jSexBgw
    DfoCleanUp.sh     -c <CMSName:Port> -p <AdministratorPassword> -cuid AUQAptxEqPhJkaLOEvsD1Wo

  4. Restart the SIA & Tomcat
  5. Log in to CMC -> Servers
  6. Right click the AJS and click destinations
  7. The destinations should be loaded properly
    In some rare cases, it may be necessary to create a new AJS (not duplicate / clone), since sometimes the links to the Streamwork plugin can be stored in the AJS itself
    If this is the case, be sure to copy the command line and configuration properties from the existing AJS to the new one

Note: If the issue still occurs after the above, ensure in CMC -> Users and groups that the 'Administrator' user has object ID 12, and run a repair following the best practices:
2468513 - Repairing the BI Platform [Best Practices]

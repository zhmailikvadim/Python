Symptom

  • As part of 1H 2026 update now you have possibility to start using the redesigned Payment Information UI within Latest People Profile on Configure Latest People Profile

  • When enabling this feature system will disable Manage Configuration UI from Latest People Profile. System will use Object Definitions rules to make you control of field properties instead.
  • Please note that there is a road map planning to stop using the Configuration UI and starting using only Object Definition configuration in order to customize the Payment Information field properties.

Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.  

Environment

  • SAP SuccessFactors Employee Central
  • SAP SuccessFactors Employee Profile
  • SAP SuccessFactors Metadata Framework 

Resolution

Prerequisites:

In order to start using the Redesigned UI for Payment Information please follow steps:

  • Enable following option under Admin Center -> Configure Latest People Profile -> Personal Data -> Payment Information.Enable the latest Viewing, Editing, and History UI on Full Profile (With Non-configurable UI) 
  • To display the Payment Information card for all users in the Full Profile, you select the Visible on Full Profile under Admin Center -> Configure Latest People Profile  Personal Data ->  Payment Information.
  • Grant users required permission for People Profile Category as explained in guide: Role-Based Permissions for Latest People Profile
  • Grant users required permission for Payment Information as explained in guide: Setting Up Role-Based Permissions

Frequently Asked Questions (FAQs)

QuestionAnswer
Will the UI Rules already configured in system automatically migrate to new Metadata Rules?No, this is still under review but for now if the standard pre-delivered rules provided does not attend your requirements then is needed to manually create and assign the new rules.
Why I cannot select "Required" while creating Metadata business rule?This is because the field is already required in object definition. Metadata data rule will not allow you to manipulate the requirement of field if the field is already set as required.
How to change field placing and grouping on new Redesigned payment information? 

This is not supported in new Redesign. 
Field order will be based on Object Definition however is not possible group fields and mix standard payment information details with the payment information country specific.

How can I configure the text Font and other cosmic aspect of the words in new redesigned payment information version?

Font, size, style  are not supported on Object Definitions configuration hence this will not be possible in new Payment Information UI Design.

Is there any standard Metadata Rules provided by SAP? If so, how can we implement it?

Yes, there are pre-delivered Metadata Rules they are available to export on Software Download Center . Please look for PaymentInfoMetadataRules.zip file under the "EC Payment Information - Rules Configuration Files"  and export it on Admin Center ->  Import and Export Data.

I have configured the Metadata rules however the rules are not working on Hire Screen.

This is currently expected behavior. For now metadata rules will work only in Latest People Profile.

Symptom

ICM - Bulk Changes

Resolution

Overview

The bulk change functionality provides the means to update a particular set of fields on an Entity from one value to another without having to update each record manually.


Making a Bulk Change

To submit a bulk change for a particular entity, Customer for example, go to the Manager Portal and select the Customers tab.

Click on the Customers link and select Search at the bottom of the page.

The screen will return an icon that looks like an arrow at the bottom of the screen


To make the bulk change, select the arrow icon at the bottom of the screen. The Bulk Change pop-up panel will appear as shown


From the bulk change panel the following information can be entered to make the appropriate changes.

  • Record Selection (required): This defines what and how many records should be updated. There are two available options for this selection.
    • Current Page
    • All records matching criteria (there may be a limit)
  • Error Policy (required): This determines how ICM should handle the requested change if the bulk change request fails the validation. This will default to rollback all updates if the validation fails for any one of the records.
  • Update Fields (required): This will list all of the available fields for the entity selected for which the user would like to make the bulk change. Select the field(s) that are to be updated and either double click on the field or drag and drop to the selected panel.
  • Change Comment (optional field): This field should provide a detailed description of the change and reason for the change. It is recommended that this field be considered required. This provides the ability for various people who will be working with these plans to have a description as to the reason / logic behind this configuration.

On this panel, enter the new value for the records that the entity should be updated and click complete. All records will now be updated with this value.

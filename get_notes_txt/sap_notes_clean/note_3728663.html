Symptom

  • In the transportation cockpit for the ocean booking scenario, the overview of all ocean freight bookings does not display the cargo cut-off date and time values in the respective column.
  • The column displays "dd.mm.yyyy" instead of the actual values.
  • When accessing the detail screen of a freight booking, the correct cargo cut-off date and time values are available.

Environment

SAP S/4HANA Cloud Public Edition

Reproducing the Issue

  1. Open the transportation cockpit in the ocean booking scenario.
  2. Navigate to the right-hand side overview of all ocean freight bookings.
  3. Observe the column for cargo cut-off date and time, which displays "dd.mm.yyyy" instead of the actual values.
  4. Click on a freight booking and open the detail screen.
  5. Observe that the correct cargo cut-off date and time values are available in the detail screen.

Cause

The column for cargo cut-off date and time in the overview is configured to display the cut-off cargo date from the first location (pick-up). However, there is an additional column in the field catalog named 'Cargo Cut-Off Date (Port of Loading)' that should be displayed instead.

Resolution

  1. Verify the configuration of the transportation cockpit overview for ocean freight bookings.
  2. Locate the field catalog settings for the overview.
  3. Replace the current column 'Cargo Cut-Off Date' with the column labeled 'Cargo Cut-Off Date (Port of Loading)'.
  4. Save the changes and refresh the transportation cockpit view.
  5. Confirm that the correct cargo cut-off date and time values are displayed in the overview.

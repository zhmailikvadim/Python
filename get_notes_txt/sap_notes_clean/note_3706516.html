Symptom

  • Not able to access Customer Remote Logon Depot
  • Not authorized to access system XXXX - check your authorization profile and the case number and try again
  • No system data found
  • 400 Bad Request
  • No data found for user, customer or system

Environment

SAP for Me > Customer Remote Logon Depot

Reproducing the Issue

  1. User tried to access Customer Remote Logon depot via case email notification
  2. User got error: No System Data Found:

Cause

S-user ID and system are under different customer number.

Resolution

Logon to SAP for Me again using the correct S-user that is from the same customer number where the system is located.

S-user also needs one or both of the following authorizations:

  • Edit My Login Data - It can store login data and display or change login data stored previously under the S-user ID. The passwords entered by other users cannot be seen; however, SAProuter passwords can be overrided, or new Logon Data sets are created .
  • Edit all Login Data  - It have full access to create, change and display all stored login data for all S-users.

Tip: To check the authorizations, go to My Authorizations and Functions. To request authorizations, use the self-service authorization request (see KBA 2891528).

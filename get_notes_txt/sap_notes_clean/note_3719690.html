Symptom

An error occurs during the Account Verification process when comparing the account owner name in cases where no alias names are maintained. The verification process terminates with rejection reason code DS28 - ReturnForTechnicalReason.

Other Terms

VoP, Account Verification, Verification of Payee, Request Agent, /PE1/CL_RA_IV_COMP_BP_NAME.

Reason and Prerequisites

During the comparison of the account holder name with alias names, if no aliases are maintained in the AM/DM system, the RFC returns an informational message indicating that no aliases were found. This informational message is incorrectly interpreted as an error, causing the process to terminate and resulting in rejection reason code DS28 — ReturnForTechnicalReason.

Solution

Implement the correction instruction attached to this SAP Note. Alternatively, you can import the referenced Support Package.

Symptom

  • An error occurred when Adobe Document Services (ADS) was launched.
  • Error message: "com.adobe.ProcessingException: com.adobe.Processing(200101)".
  • Message No.: FPPRUNX001.
  • Logs indicate a processing exception during a "(Validate)" operation.
  • Error details: "Operations: GetSignatures are not allowed on PDFA Input".

Environment

  • Forms Processing

Reproducing the Issue

  1. Launch Adobe Document Services (ADS) hosted on SAP NetWeaver Java.
  2. Perform a "(Validate)" operation on a PDF/A input.
  3. Observe the error message: "com.adobe.ProcessingException: com.adobe.Processing(200101)" with details "Operations: GetSignatures are not allowed on PDFA Input".

Cause

The underlying API of Adobe Document Services (ADS) that creates and manages PDF/A documents does not support the "getSignature" operation.

This operation is recommended for interactive PDFs but not for PDF/A documents. Attempting to use this operation on PDF/A inputs results in a processing exception.

Resolution

  1. Avoid using the "getSignature" operation on PDF/A inputs as it is not supported by the underlying API of Adobe Document Services (ADS)
  2. Use the "getSignature" operation only on interactive PDFs where it is supported.

Symptom

The Revenue Transfer program (Program A) or the Contract Asset/Liability calculation program fails with the error FARR_CONTRACT_MAIN:308: Inv. Distri. Error on Contract &1: Input &2 <> Output &3 on currency &4 or related errors as FARR_CONTRACT_MAIN 309, 310, 320, 775.

 In most cases, this error catches rounding distortions—edge cases where complex distribution logic results in a minor mathematical mismatch between the input and output totals. Because SAP has already released numerous notes to address these specific edge cases, there is a high probability that a fix already exists for your scenario.

Reason and Prerequisites

This error is triggered by a post-invoice distribution check designed to ensure data integrity. Its purpose is to verify that the total value of invoices remains consistent before and after the distribution process. If a discrepancy is detected, the system stops the process to prevent financial distortion.

Solution

 To resolve the error FARR_CONTRACT_MAIN:308, please follow this structured remediation process:

  1. Implement Missing SAP Notes
    Please implement all recommended SAP notes related to invoice distribution corresponding to your system release which are listed below. This ensures that any known mathematical edge case or rounding logic error is patched before you proceed with data correction.

    1. Release 130 (Software Component REVREC) - Notes 3476034, 3540775, 3632469, 3644730, 3677684
    2. Release 103 (Software Component S4CORE) - Notes 3476034, 3540775, 3632469, 3644730, 3677684
    3. Release 104 (Software Component S4CORE) - Notes 3476034, 3521996, 3540775, 3632469, 3644730, 3677684
    4. Release 105 (Software Component S4CORE) - Notes 3167003, 3367082, 3388088, 3476034, 3521996, 3540775, 3632469, 3644730, 3677684
    5. Release 106 (Software Component S4CORE) - Notes 3167003, 3199262, 3367082, 3388088, 3476034, 3521996, 3540775, 3632469, 3644730, 3664487, 3677684
    6. Release 107 (Software Component S4CORE) - Notes 3367082, 3388088, 3476034, 3521996, 3540775, 3632469, 3644730, 3677684
    7. Release 108 (Software Component S4CORE) - Notes 3476034, 3521996, 3540775, 3632469, 3644730, 3677684
    8. Release 109 (Software Component S4CORE) - Notes 3632469, 3644730, 3677684
  2. Reset Calculated Invoice Distribution
    Implement correction report ZFARR_CLEAR_INV_FX_ED included in the attachment of this note.

    After installation, run this report only for the contracts which encountered the FARR_CONTRACT_MAIN error.

    The program run should clears table FARR_INV_FX_ED and flag IS_CALCULATED in FARR_D_INVOICE.
  3. Execute Program A (Revenue Transfer)
    Execute the revenue transfer program (Program A) for the contracts reset in step 2. This will recalculate the revenue distribution on the reset data.
  4. Execute Program B (Calculate Contract Liabilities and Assets)
    Run the calculation of contract assets and liabilities (Program B) for the contracts transfered in step 3.
  5. Ad-hoc Check
    Before final posting, perform a quality check on the database level:
    1. Open table FARR_D_POSTING.
    2. Eeview the entries for a sample of the reprocessed contracts.
    3. Verification: Ensure there are no unexpected exchange rate differences or incorrect revenue rates.
  6. Execute Program C
    Once the data in the sub-ledger is verified and correct, execute the posting program to FI-GL (Program C) for the contracts processed in the previous steps.

Symptom

  • While executing Finance data migration, the system raises error FINS_FI_MIG128 in MUJ & R21 steps.

Environment

  • SAP S/4HANA Finance
  • SAP S/4HANA
  • Financial Accounting (FI)

Reproducing the Issue

  1. FINS_MIG_STATUS

Resolution

  • The FINS_FI_MIG128 error occurs because currency type 31 is configured as the second or third local currency for the company code, but many documents have the secondary local currency field BKPF-HWAE2 = ' '. Please correct these inconsistencies in your source system so that subsequent migration runs do not raise this error.

Recommended checks:

  1. Verify that currency type 31 is correctly assigned as a local currency for the affected company code.
  2. Identify and correct postings where the secondary local currency amount (BKPF-HWAE2) is missing.
  3. Review also the following tables for completeness and consistency:
    • FINSC_CURTYPE: Ensure entries exist for the relevant source currency and currency type.
    • FINSC_001A: Confirm that all required combinations of source currency and currency type are maintained.

Symptom

When executing transaction MIGO, error message CJ895: "System error in program SAPLCKMLMVALPROC (4) -> F1" is triggered.

Environment

  • Controlling (CO)
  • SAP R/3
  • SAP ERP
  • SAP S/4 HANA

Cause

  1. For SAP ECC versions or S/4 HANA 100 (i.e. not S/4 HANA 101 or above), the possible causes of this error include:

    Missing entry in table CKMLMV001.
    Duplicated entry in table CKMLMV_AUX_KALNR, CKMLMV001.

  2. Another possible cause for this error (regardless of system version) is an issue with number range of object KALK. In table CKMLKALNR, the maximum value for field KALNR is set to "X". However, the system retrieves a smaller value than "X" from function module CKMS_KALNR_GET.

Resolution

  1. For the first cause presented, review and follow indications in SAP Note 1668058: "CJ895 error SAPLCKMLMVBUFFER (4) -> F1".
  2. For the second cause presented, in order to resolve this issue, below steps should be taken: 
    1. Check max value (=A) for field KALNR in table CKMLKALNR via transaction code SE16H.
    2. In transaction SNRO for object KALK, adjust NR status to a number which is bigger than A.
    3. In transaction SM56, reset the buffer for object KALK.

See Also

SAP Note 1668058: "CJ895 error SAPLCKMLMVBUFFER (4) -> F1"

Keywords

CJ895, MIGO, SAPLCKMLMVALPROC, SAPLCKMLMVALPROC (4) -> F1, F1, incoming invoice, purchase order, number range object KALK, CKMLKALNR, SE16H, SM56

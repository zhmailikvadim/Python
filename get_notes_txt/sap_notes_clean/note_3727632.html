Symptom

  • The user followed the procedure to activate the Datasphere or Business Data Cloud tenant as per the provided documentation but did not receive the "Welcome to Datasphere" invitation email.
  • The user attempted to log into the Datasphere tenant using their S-User account credentials but was unable to authenticate.
  • The user attempted to activate the tenant in TRUST to use their IAS tenant, but the start page did not redirect to their corporate IDP.

Environment

Business Data Cloud

Datasphere

Reproducing the Issue

  1. Follow the procedure to activate the Datasphere or Business Data Cloud tenant as per the documentation.
  2. Verify that the "Welcome to Datasphere" email is not received.
  3. Attempt to log into the Datasphere tenant using S-User credentials.
  4. Attempt to activate the tenant in TRUST to use the IAS tenant and check if the start page redirects to the corporate IDP.

Resolution

It is not possible to use an s-user account to log into a BDC/Datasphere tenant and SAP does not create users inside customer tenants. The system owner must log in to the tenant using the email registered on provisioning phase and create additional users or by activating a custom Identity Provider (IDP). After at least one Data Warehouse Administrator user exists, that user also can create additional users.

If a user has been created but did not receive the welcome email, they can use the “Forgot Password” option to reset their password.

If the system owner cannot log into the tenant and does not receive the reset password email even after using “Forgot Password”, follow these steps:

  1. Verify the system owner email address associated with the Datasphere or Business Data Cloud tenant.
    You may log a case with SAP to confirm the system owner’s registered email.

  2. If the email address is incorrect, update it using one of the following methods: a. Contact your SAP Account Executive (AE) and request the email change. Refer to SAP Note 2626344 - Who is the SAP Account Executive for my company?
    b. Delete the tenant and recreate it using the correct email address.
    Note: After deletion, the tenant remains in a suspended state for 7 days, and the same hostname cannot be reused during this period.

  3. If the email address is correct, the system owner can request a new password using the “Forgot Password” option and log in using the newly defined password.

Once logged in, the system owner can proceed to create additional users to access the tenant.

Symptom

When creating a Custom Analytical Query and adding hierarchy nodes as filters (for example: Profit Center, Functional Area, or G/L Account), the system displays the corresponding hierarchy parameter as mandatory in the report, even though the hierarchy field was not configured as mandatory in the Custom Analytical Query app. See the following behavior:

  • The report shows both:
    • a) The hierarchy parameter (composite key field)
    • b) The hierarchy node
  • The hierarchy parameter cannot be left blank.
  • Attempting to select a hierarchy node without entering the hierarchy parameter results in an error.
  • In the generated CDS view, the hierarchy binding appears as #PARAMETER.

The expectation is that multiple hierarchy filters could be added without becoming mandatory.

Environment

  • SAP S/4HANA 2023
  • HEC system
  • Custom Analytical Query (Fiori App)
  • Key User Extensibility Tools

Reproducing the Issue

  1. Open the Custom Analytical Query app.
  2. Create or edit a custom query.
  3. Add a hierarchy-based field (e.g., Profit Center Hierarchy Node) as a filter.
  4. Save and execute the query.
  5. The hierarchy parameter appears as mandatory.
  6. Leaving the hierarchy parameter blank results in an error when selecting a node.

Cause

When a hierarchy node is set as User Input, the system requires the composite key fields of the hierarchy.

These composite key fields are represented by parameters (for example, P_FunctionalAreaHierarchy, P_ProfitCenterHierarchy,...).

Parameters are mandatory by default, therefore the hierarchy parameter automatically appears as mandatory in the report.

The underlying dimension (such as Functional Area or Profit Center) itself is not mandatory, only the hierarchy parameter is.

Therefore, this behavior is standard system design.

Resolution

The system behavior is as designed.

When hierarchy node selection is enabled, the hierarchy’s composite key parameter must be entered.

Parameters used for composite key fields are mandatory by default, and Custom Analytical Query does not provide an option to make these hierarchy parameters optional.

Selecting a value for the dimension (e.g., Functional Area) remains optional, but the hierarchy parameter itself cannot be left blank.

=> No configuration change is available in the Custom Analytical Query app to modify this behavior.

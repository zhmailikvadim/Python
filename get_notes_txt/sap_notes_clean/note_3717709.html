Symptom

When resubmitting a claim, the system is expected to process the resubmission and complete it without interruptions.

However, a runtime error occurs while resubmitting claims, which prevents the process from completing successfully. As a result, we are unable to proceed with claim resubmission and continue further processing.
This note will fix the issue so that claims can be resubmitted successfully without encountering runtime errors.

Other Terms

claim resubmission, runtime error, /IRM/GCR27, SQL_CAUGHT_RABAX, Index Claims, VBELN_NO_PACK_SELECT.

Reason and Prerequisites

This is caused by program error

SAP Note#3717736 is available for ECC Release

Solution

Please implement the correction instructions attached. These corrections are only valid if you have applied the Support Packs.

Release V4 -  2022 V2, 2023, 2023V2, 2023 V3 Base Package and 2023 Support Pack 1.

Use transaction SNOTE to implement the correction instructions specified below and activate the affected program object.

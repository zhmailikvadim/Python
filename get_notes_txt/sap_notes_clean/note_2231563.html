Environment

  • Upgrade or update using Software Update Manager (SUM)
  • For Netweaver 6.40 and 7.0 based systems

Reproducing the Issue

Repeat the phase INPUT-USER-CREDENTIALS in SUM.

Cause

Incorrect value lookup/defined for bindHost property of p4 service.

Resolution

  1. The bindHost property must either be set to 0.0.0.0 to allow P4 clients to access all IPs including loopback for that host .  If you do not want to allow access to all IP addresses, bindHost property can be set to individual IPs separated by space including 127.0.0.1. The entry for localhost is required for SUM to connect to P4 port on loopback address.
  2. Kindly follow the solution as mentioned in note : 609603 - Problems with Multiple NICs and SAP J2EE Engine
  3. Restart the instance after changing the parameters and repeat the failing phase in SUM.

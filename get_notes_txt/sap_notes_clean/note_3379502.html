Symptom

After running an archiving job in transaction code SARA for the object CO_COPC, the costing data is not getting archived in the table KEKO.

Environment

  • Controlling (CO)
  • SAP R/3
  • SAP R/3 Enterprise 
  • SAP ERP
  • SAP ERP Central Component
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA

Reproducing the Issue

1. Open transaction code SARA.
2. Execute the archiving job for CO_COPC object.
3. The expected costing data is not getting archived from the table KEKO.

Cause

Please note according to standard system design, it's not possible to archive cost estimates that are still linked to the material master (view Costing 2). For example:

  • Previous cost estimate.
  • Current cost estimate.
  • Future cost estimate.

This is correct and intended system behavior.

Resolution

Please check if the intended archiving data is linked to the material master (Costing View 2). The system won't archive data from table KEKO which is still linked to previous, current or future cost estimates.

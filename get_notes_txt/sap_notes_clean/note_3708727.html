Symptom

  • The intelligent QA functionality in SAP CX AI Toolkit is providing incorrect information when queried.
  • No specific error messages or error codes are mentioned in the case information.

Environment

SAP CX AI Toolkit

Reproducing the Issue

  1. Log in to the SAP CX AI Toolkit.
  2. Go to Intelligent Q&A tab
  3. Ask a question to fetch data from an Excel file stored in OneDrive or Perform broad, ad-hoc database-style queries or analytics across all records or ask count/aggregation related queries
  4. Observe that the system provides incorrect or inconsistent output.

Cause

Intelligent Q&A uses RAG retrieval method, Ad-hoc database-style queries or analytics across all records won't be fetching accurate results as it is not designed for these type of queries

Resolution

Intelligent Q&A - What it can and cannot do

What it can do 

  • Retrieve and answer explicit factual or procedural questions by finding and combining relevant source documents. Example: “How do I reset the X router?” or “What steps does policy Y require for approval?” (provided the connected data sources contain the answers to these questions).
  • Produce short summaries or syntheses that are directly supported by the retrieved records. Example: “Summarize the onboarding steps in these three documents.”
  • Refer to source documents that contain an answer (below the answer, we see source names / links).

What it cannot do 

  • Perform broad, ad-hoc database-style queries or analytics across all records (for example: “How many customers have invoices >30 days?” or “Show me all Q4 sales opportunities and rank them”). RAG retrieves matching documents, it does not run global counts/aggregations unless those totals already exist in the indexed text.
  • “Scan everything” and form independent judgments about datasets. It synthesizes only from the documents it retrieves; it cannot reliably infer facts that aren’t in those documents.
  • Replace structured reporting or BI tools. Complex filtering, joins, and numeric aggregations require a data warehouse or query engine, not plain RAG retrieval.

How to ask good questions 

  • Be explicit and scoped: specify the task, timeframe, or document set (for example: “According to the 2025 policy document, what steps are required to…”).
  • Ask questions that are explicitly answered in your connected sources.
  • For counts/aggregations, use queries or reports in the original data source or a third party reporting tool. You can upload these reports and then ask the Q&A system to reference those results, but it cannot compute them from the raw data source / documents.

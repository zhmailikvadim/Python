Symptom

  • When a new contact is created for an account, the payload from BusinessPartnerERPReplicationOut contains the new contact but does not include phone number or email, as these are part of the BusinessPartnerERPContactAddressReplicationOut payload.
  • The contact's information does not replicate over BusinessPartnerERPContactAddressReplicationOut until the contact is edited at least once.
  • After editing the contact, the payload correctly includes phone number, email, and other data.

Environment

  • Integration of C4C with ERP

Reproducing the Issue

  1. Open an existing account.
  2. Create a new contact for the account.
  3. Check Web Service Message Monitoring: 'BusinessPartnerERPContactAddressReplicationOut' is not sent with the new contact's information (phone, email). 'BusinessPartnerERPReplicationOut' and 'BusinessPartnerERPAddressReplicationOut 'are sent as expected.
  4. Edit the newly created contact, for example, change the phone number.
  5. Check Web Service Message Monitoring again. This time, BusinessPartnerERPContactAddressReplicationOut is sent and the payload contains the updated contact's information.

Cause

This is standard behaviour. A filter must be added to the  'BusinessPartnerERPContactAddressReplicationOut' Communication Arrangement for 'In Preparation' status in order to have these details sent when new contacts are created. 

Resolution

  1. Goto Communication Arrangement Filters under General Settings and click on Add Row.
  2. Select relevant Communication System for Communication ID and the Communication Scenario 'Business Partner Replication to SAP Business Suite'
  3. Add a filter for 'Send Business Partners with In Preparation status.'

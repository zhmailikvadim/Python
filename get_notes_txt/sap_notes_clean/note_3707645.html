Symptom

  • Error is encountered during the upload of a business transaction via the Import Business Transactions application.
  • Records with the unit of measure “KG” are flagged as inconsistent.

"Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental."

Environment

Sustainability Footprint Management

Reproducing the Issue

  1. Attempt to upload business transaction data via the "Import Business Transactions" application.
  2. Include records with the unit of measure “KG”.
  3. Observe the import completing with inconsistencies due to records using unit of measure as "KG". 

Cause

As the "Unit of Measure Source" is using "ISO", the expected ISO code for KG is "KGM" and not "KG".  

Resolution

  1. Confirm the ISO code for the unit of measure “KG” in the "Manage Unit of Measure" application within Sustainability Footprint Management.
  2. Search for the unit of measure “KG” and verify the ISO code. If the "Unit of Measure Source" is "ISO", then the expected ISO code for KG should be "KGM".
  3. Update the transactional data file to replace “KG” with “KGM”.
  4. Re-upload the corrected file via the "Import Business Transactions" application.
  5. Verify that the upload completes successfully without inconsistencies.

Symptom

It is not possible to add a customer user to a tenant. A System Error is reported in SLD Cloud Control Center.

Reproducing the issue

Prerequisites:

You are using Business One Cloud 1.1 Patch Level 10 or lower version.

Scenario 1:

  1. Log on to SLD Cloud Control Center.
  2. In the Tenants section choose a Tenant record.
  3. In the Tenant Details grid, User Management tab, choose the Add button.
  4. Select one or more Customer Users to be added to the Tenant and choose the Save button.

Scenario 2:

  1. Log on to SLD Cloud Control Center.
  2. In the Tenants section choose a Tenant record.
  3. In the Tenant Details grid, User Management tab, choose the Import Users button.
  4. Select one or more Customer Users to be added to the Tenant and choose the Save button.

Expected behavior:

In either scenario, the Customer User is added to the Tenant.

Actual behavior:

You receive the following: System Error: while trying to invoke the method com.sap.businessone.sld.model.B1UserBinding.getId() of a null object returned from com.sap.businessone.sld.model.ExtensionUserPreference.getUserBinding()

Solution

SAP intends to provide a patch or patches in order to solve the problem described. The section References below will list the specific patches once they become available. The corresponding Info file of the patches in SAP Service Marketplace will also show the SAP Note number. Be aware that these references can only be set at patch release date. SAP will deliver patches only for selected releases at its own discretion, based on the business impact and the complexity of the implementation.

Workaround:

SLDDATA database inconsistency is fixed automatically during upgrade to SAP Business One Cloud 1.1 Patch Level 12. For earlier patch level versions, SAP Support can assist with performing a database fix on SLDDATA manually.

Symptom

When creating a new assignment for an account in the SOM UI to a name‑dependent hierarchy (such as FRS), the system may incorrectly raise message USMD1B 234 (“Account is already contained in an interval”).

The issue typically occurs when the target FRS Item has the same name as an item in another FRS hierarchy, even though name‑dependent hierarchies are isolated and should not influence each other.

Other Terms

SOM UI, FRS, Hierarchy assignment, WRITE_HRY_PARENT_PREDEC_NEW

Reason and Prerequisites

Hierarchy assignments are not handled properly and it may happen that changed/new assignment is not sent to write method.

Solution

 Please implement the SAP Note.

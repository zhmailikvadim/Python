Symptom

  • SuccessFactors admin wants to check why an user can see/edit/delete information of another user, despite setting permissions not to do so.
  • SuccessFactors admin wants to validate permissions granted to specific users for a specific permission and a target.

Environment

SAP SuccessFactors HCM Suite

Resolution

With RBP Troubleshooting tool, users/admins can search the roles granted to specific users for a specific permission and a target user. When some users get some permissions on some target users that should not be granted, the administrator can use this tool to find which role grants the permission so they can update the permission settings.

How to use this feature. Please note that the examples below refer to a user X which can see Birthday of user Y:

Go to RBP Troubleshooting page

  1. Select Target Population Search tab
  2. In "Search Criteria":
  3. In "Access User 1" field, search for the user which has the permission granted - user X from the given example 
  4. In "Permission" field, search for the permission that has been granted to this user in a new window - Employee Profile > Date Of Birth(View) permission from the given example
  5. In "Target User" field, search for the user which information can be seen/edited/deleted - user Y from the example above  from the given example
  6. Click on Search

The Results will display Role ID, Role Name and Actions as View Role Assignments.

NOTE: Even if there a role that restricts an user to have permission over another, if there is another role which allows users such permissions, this rule will prevail.

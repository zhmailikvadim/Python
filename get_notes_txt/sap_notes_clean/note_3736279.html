Symptom

When copying plan data to the target ledger for specific record type (e.g 3 planned assessment/distribution) in transaction code FAGLGP52N, the process fails. An error message is raised  "Enter a valid record type" (GU612).

Environment

  • Financial Accounting
  • SAP ERP Central Component
  • SAP ERP
  • SAP S/4HANA Finance
  • SAP S/4HANA
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA

Reproducing the Issue

  1. Start transaction code FAGLGP52N.
  2. Enter values for Source Data and Copy To
  • Ledger
  • Record Type  (e.g 3)
  • Version
  • Company Code 
  • Fiscal Year
  • Posting Period
  • Document Type
  • Processing Option: Test Run

Cause

 As per system design, this transaction is supposed to copy actual or plan data only (record_type 0 and 1)

Resolution

Do not use FAGLGP52N to copy plan data for record type (e.g 3), this record type is not supported by the transaction. As per system design, this transaction is supposed to copy actual or plan data only. When trying to copy record types different from 0 and 1 in NewGL planning functionality, then  another transactions/way than FAGLGP52N.  Use an alternative standard method to create/copy plan data for record types other than 0 and 1, for example, BAPI_FAGL_PLANNING_POST.

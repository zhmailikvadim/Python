Symptom

In BI 4.2 SP9, if a CSV text file stored in the CMS has a file name that is 37 characters or longer, the viewer will not do anything when the OK button is clicked on the query panel to add the objects to the report.

In earlier versions of 4.2, the error "A problem is encountered. The requested action cannot be completed" may appear and the length at which is occurs may be shorter.

Environment

  • SAP Business Intelligence 4.2
  • CSV
  • Web Intelligence

Reproducing the Issue

  1. Create a CSV file with a title of 37 character or more
  2. Upload it into the CMS repository
  3. Create a WebI document using Text as a data source and select the CSV
  4. In 4.2 SP9, when clicking OK on the Custom Data Provider panel, nothing happens.
  5. In earlier versions you may see the error listed above

Cause

The file name is too long.

Resolution

This issue does not occur in BI 4.3

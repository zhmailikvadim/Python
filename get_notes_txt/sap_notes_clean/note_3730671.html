Symptom

Inbound delivery item weights are set to zero when reversing it by VL09
Value for fields LIPS-NTGEW, LIPS-BRGEW, LIPS-VOLUM and LIPS-LGMNG are set to zero after reversing inbound delivery

Environment

  • Sales and Distribution (SD)
  • Logistics Execution (LE)
  • SAP ERP Central Component
  • SAP ERP
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA

Reproducing the Issue

  1. Run transaction VL09
  2. Select the relevant inbound delivery and perform the goods movement reversal
  3. Open the inbound delivery and observe that the item weights are set to zero

Cause

The purchase order linked to the inbound delivery uses a confirmation control key that has a quantity difference profile assigned with the "Completion indicator" active
During reversal, the delivery quantity reduction logic runs leading to item values being reset
For debugging set a breakpoint at form "REDUCE_DELIVERY_QUANTITIES" at program "SAPLV51W" and check if variable "ls_t163q-compl_ind" is set as "X", if so, this is the cause

Resolution

  1. Identify the confirmation control key used by the purchase order(Confirmation tab at ME23N tcode) associated with the affected inbound delivery items
  2. In SPRO, navigate to: Logistics Execution > Goods Receipt Automotive > Confirmations > Set Up Confirmation Control, and locate the confirmation control key to find the assigned quantity difference profile
  3. Go to: Logistics Execution > Goods Receipt Automotive > Confirmations > Set Up Quantity Difference Profiles, open the assigned profile, and review the “Completion indicator”
  4. If inbound delivery items should not be treated as completed on subsequent quantity differences (such as reversals), deactivate the “Completion indicator” in the profile
  5. Save the customizing changes and test again by reversing the goods movement in VL09 and verifying that inbound delivery item weights are no longer set to zero

Symptom

An issue where SFTP Adapter is unable to poll the files from the SFTP server. The other SFTP sender adapter polling from the same SFTP server, and they are working fine. you have checked the file and directory permission as well.
The issue happening for a particular channel.
The files are getting picked when complete name is provided for the file and not getting picket when a regex expression is given.
There has been an OS patch update recently.

Environment

SAP NetWeaver 7.50

Cause

SFTP sender channel was impacted by metadata corruption. Re-creating the channel rebuilt a clean configuration, restoring normal regex behavior.
The channel parameters may have become corrupted or partially unreadable after system-level OS patching.
This causes the SFTP adapter to internally misread fields like file name patterns.

Resolution

Create a new SFTP channel and re-entered all the connection settings manually. Post that, the files should be picked up from SFTP server and processed correctly.

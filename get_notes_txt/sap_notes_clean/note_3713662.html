Symptom

In SAP Cloud Integration, after updating a certificate used for outbound connections, the iflow fails with 401 errors for some time. Afterwards, the iflow starts to work again without any changes.

Environment

  • SAP Cloud Integration
  • SAP Integration Suite

Cause

This is an expected behavior. In the scenarios, where an existing connection is re-used, like for example in outbound connections, the old key is cached within the connection for some time.

Resolution

To make these connections use the new key, you need to restart the respective integration flows. 

This can be done in the tenant WebUI:

  1. Go to "Manage Integration Content" tile; 
  2. Select the affected integration flow
  3. Click on "Restart" to trigger a restart

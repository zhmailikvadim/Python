Symptom

  • In the e.g. Monitor Subcontracting Documents app, the user clicks a purchase order to see the details, but the browser changes from Fiori to WEBGUI. If you click Back button, the browser remains in the WEBGUI application.
  • This can also happen when trying to navigate back to other apps like Manage Material Coverage app.

Environment

  • Materials Management
  • SAP Fiori
  • SAP S/4HANA

Reproducing the Issue

  1. Open any Fiori app related to Purchase Order
  2. Display any PO
  3. The system redirects the user to the Advanced app
  4. Click Back Button
  5. It does not return to the Fiori app

Cause

When the user clicks the purchase order, the system tries to open it in the Manage Purchase Orders app (1º navigation). If the purchase order contains unsupported features, the system will automatically redirect from the Manage Purchase Orders app to the Display Purchase Order - Advanced app (2º navigation).

When the user clicks Back button, the system will redirect to the previous app (i.e. Manage Purchase Orders), not the Monitor Subcontracting Documents app. So the navigation will remain in this loop.

Resolution

To fix this issue, follow the resolution steps mentioned in the KBA below to disable the 'in-place' navigation:
KBA 2403527 - Force WebGUI/WebDynpro apps to open in same window when launching from Fiori Launchpad

After the parameter has been set (and all caches have been cleared), the purchase order detail view should open a new browser tab.
To navigate back to the original App e.g. Monitor Subcontracting Documents app, it's sufficient just to close this new additional tab. It's not possible to navigate back to original App e.g. Monitor Subcontracting Documents app within this new tab.

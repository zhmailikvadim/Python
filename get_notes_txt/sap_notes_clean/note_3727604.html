Symptom

Component: Foundation for EHS Management
Module: Requirements - Permits

The system shows following incorrect behavior.

When you create a new revision of an existing permit, the system displays a short dump in the system. The dump text includes: “... DATREF_NOT_ASSIGNED ... CL_EHFND_REQ_EDIT_STRUCT_TREE=CP ... Access using a NULL data reference is not possible.”

 

Other Terms

ST22, Dump, DATREF_NOT_ASSIGNED, CL_EHFND_REQ_EDIT_STRUCT_TREE, Manage Compliance Requirements

Reason and Prerequisites

A program error in the system causes the symptom.

You find the prerequisites in the relevant correction instructions.

Solution

After implementing this SAP Note, the system creates and displays the new revision of the permit.

Alternatively, you import the corresponding Support Package for the component extension of SAP Environment, Health, and Safety Management or the Feature Package Stack for the SAP S/4HANA system.

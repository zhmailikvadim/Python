Symptom

An Integration Center Job displays "No data" in the Preview screen after successful execution.

Environment

  • SAP SuccessFactors HCM Suite

  • Integration Center

Reproducing the Issue

  1. Build an integration using Integration Center.
  2. Run the job successfully.
  3. Access the integration again and observe the Preview screen displaying "No data".

Cause

The issue occurs due to the default "modified since query mode" set in the time-based filter in Integration Center, which avoids full loads every time. If no record changes occur after the last run, the result set will be empty.

Resolution

This is an expected behavior because by default a modified since query mode is set up in the time-based filter Integration Center to avoid the full loads every time .
The time-based filter works on the last run time. If there are no record changes after the last run, the result set will be empty. 

In order to view the data in the preview. You can do the following

  1. Change the time-based filter to "None" in the Integration Center settings.
  2. Verify the preview screen to ensure data is displayed correctly.

Note* If you change the Modified Since Query Mode from Modified Since to None the job will be querying into full load every run (which is not recommended).

Symptom

Within F2175, selecting a Maintenance Order (or multiple Orders) and choosing the Print option generates a spool request but there is no physical document printed

Environment

  • SAP ERP Plant Maintenance (PM)
  • SAP S/4HANA, on-premise
  • SAP S/4HANA, Cloud Private Edition

Reproducing the Issue

  1. Open F2175
  2. Execute with criteria
  3. Select an Order (or multiple Orders)
  4. Print

Cause

There is no physical print-out when using F2175, this does not function the same as when using IW31/32 from the SAP GUI

Resolution

  • The option to Print from F2175 will begin a job that should be checked by the User within F6065 (Manage Work Packs)
  • There is no physical print-out. The output needs to be checked within F6065
  • if you want to use F2175 to Print, this requires Output Management to be active
  • The Application Log will show the Work Pack that is created and you can then check this within F6065
  • The following SAP Help documentation can be reviewed regarding this process: Print Multiple Orders and Operations | SAP Help Portal

Keywords

F2173, EAM_ORDOP_MONS1, EAM_ORD_MANS1, EAM_WRKPK_MANS1, Output Control

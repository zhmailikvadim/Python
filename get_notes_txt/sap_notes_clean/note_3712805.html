Symptom

  • In designer, when creating a Web Services REST datastore and selecting OAuth2.0 as authorization type, there is no scope option
  • Job fails with error: " HTTP error:<401, Unauthorized Error: { "statusCode": 401, "message": "Unauthorized. Access token is missing, invalid, or expired." }>"

Other Terms

Keywords

WS REST API, datastore, scope, OAuth2.0, access token URL, invalid access token, 401, Unauthorization

Reason and Prerequisites

Environment

SAP Data Services 4.3 Support Package 3

Solution

Reproducing the Issue

  1. Create a Web Services REST datastore and select OAuth2.0 as authorization type
  2. Fill all the required information
  3. Save the datastore successfully
  4. Run job, it fails with error

Solution

In designer UI, the scope option will be added into Web Services REST datastore configuration in the later SAP Data Services release

As workaround to fix the job error, please take below actions:

  1. Modify the WADL file to add the param name "Authorization" as following for the API you want to call in job
    <param name="Authorization" style="header" type="xsd:string" required="true"/>
  2. Reimport function into designer to refresh the function structure
  3. Change the datastore authorization to Basic rather than the OAuth2
  4. Get the access token value either in POSTMAN or defining a separate job to get the access token
  5. In the issue dataflow Query, hardcode the value of Authorization field to the bear access token value you obtained in above step4

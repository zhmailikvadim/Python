Symptom

  • Standard SAP program ECPAO_EE_INB_INVT_ALV does not capture all BIB replication error messages for employee master data.
  • Error messages visible in the Data Replication Monitor (DRM)/SLG1 logs are not available in the report generated by ECPAO_EE_INB_INVT_ALV.

Environment

  • SAP ERP
  • SAP S/4HANA

Reproducing the Issue

  1. Run the standard SAP program ECPAO_EE_INB_INVT_ALV to generate a report on BIB replication errors for employee master data in transaction SE38.
  2. Compare the error messages captured in the report with those visible in the Data Replication Monitor (DRM) or transaction SLG1.
  3. Observe that some error messages visible in DRM or SLG1 are not captured in the report generated by ECPAO_EE_INB_INVT_ALV.

Cause

The report: ECPAO_EE_INB_INVT_ALV extracts its information from the following tables which can be viewed within the SE16 transaction:

  • ECPAO_INVT_ITMSG
  • ECPAO_INVT_INFTY

The table: "ECPAO_INVT_ITMSG" is where the error messages would be stored for the employees after the replication process. 

Not all error messages from the employee replication which can be seen in the SLG1 logs create a message in the above tables. This is because during the replication, only error messages which are linked to an infotype are added to this table: "ECPAO_INVT_ITMSG". For example if the error was to occur when the employees data is getting pre-processed before any infotype is created, then no entries will get created into the "ECPAO_INVT_ITMSG" table and therefore will not appear on the report: "ECPAO_EE_INB_INVT_ALV"

Resolution

Therefore if the error message is not available to view in the report: "ECPAO_EE_INB_INVT_ALV". It would be required to use the SLG1 logs to view the replication error on the ERP / S/4HANA side. Please refer to the help guide documentation regarding analysing the employee masterdata replication process:

  • Monitoring of Employee Master Data and Organizational Assignment Replication

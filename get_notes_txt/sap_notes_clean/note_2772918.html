Symptom

1. The execution of a transaction via SAP GUI for HTML (WEBGUI) does not show full screen content:

If the web browser window is resized, the content is then displayed correctly.

2. Similar issue can also occur where SAPGUI for HTML is called from a secondary framework like Enterprise Portal where the screen size available to it differs when calling SAPGUI for HTML direct


Example screenshot of the SMEN transaciton called from inplace iView. Calling WEBGUI direct in the Browser - there is no issue and the SMEN  transaction displays correctly

Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.

Environment

SAP NetWeaver 7.40 and higher

Reproducing the Issue

Call SAPGUI for HTML/WebGUI

Cause

Calculation of available screen area

Resolution

Implement SAP Note 2761326 (SAP GUI for HTML: Unexpected results or sizes)

See Also

KBA 2755871 - 500 Internal Server Error: obtaining more information about the error

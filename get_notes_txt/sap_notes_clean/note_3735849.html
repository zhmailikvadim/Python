Symptom

  • A read-only RAP List Report requires custom filter logic via @ObjectModel.filter.transformedBy on a filter field.
  • After activating the CDS view, the Service Binding displays an error:
    Error message observed: "Error related to element '<element name>' of entity '<entity name>': Transformation @OBJECTMODEL.FILTER.TRANSFORMEDBY on element <element name> is not allowed i…"

Environment

SAP S/4HANA

Reproducing the Issue

  1. Create a read-only List Report using RAP with a filter field annotated with @ObjectModel.filter.transformedBy.
  2. Activate the consumption CDS view.
  3. Open the Service Binding and check the entity; the error appears indicating the transformation is not allowed.

Cause

The consumption CDS view is analytical (e.g. contains @Aggregation.default: #SUM), and filter exits using @ObjectModel.filter.transformedBy are not supported for analytical views in RAP/SADL, which leads to the “not allowed” error.

Resolution

  1. Verify whether the consumption CDS view is analytical (e.g. it uses @Aggregation.default: #SUM or other analytical semantics):
    1. If the view is analytical:
      • Remove the @ObjectModel.filter.transformedBy annotation from filterable elements, as this feature is not supported for analytical views.
    2. If custom filter logic is mandatory:
      • Refactor to a non-analytical consumption model where @ObjectModel.filter.transformedBy is allowed.
      • Reactivate the CDS view and Service Binding.
  2. Rebuild and test the Service Binding to confirm the error no longer occurs.

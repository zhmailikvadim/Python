Symptom

When using the integration package SAP Sales Cloud Version 2 Integration with SAP CPQ (latest version), an error occurs when attempting to edit the iFlow Create Follow-Up Quote in SAP CPQ from Opportunity.

The error indicates that JMS (Java Message Service) is required. JMS capabilities are not included in the Basic Edition of SAP Cloud Integration.

Environment

  • SAP Sales Cloud Version 2
  • Integration for SAP Sales/Service Cloud
  • SAP Sales Cloud Version 2 Integration with SAP CPQ (latest version)
  • SAP Cloud Integration (part of SAP Integration Suite)

Reproducing the Issue

  1. Deploy or open the integration package SAP Sales Cloud Version 2 Integration with SAP CPQ.
  2. Open the iFlow Create Follow-Up Quote in SAP CPQ from Opportunity.
The system displays an error stating that JMS is required.

Cause

The iFlow uses JMS (Java Message Service) for asynchronous message processing.

JMS functionality in SAP Cloud Integration is only available with:

  • SAP Integration Suite – Standard and Premium Edition, or
  • Consumption-based (CPEA) licensing, with Cloud Integration service enabled.

The Basic Edition does not include JMS message broker capabilities and therefore cannot support CPQ integration scenarios that rely on JMS.

Resolution

Short-Term solution: To enable JMS and use the CPQ follow-up quote iFlow, upgrade the SAP BTP license to one of the following:

  • Integration Suite – Standard Edition, at a minimum.

For more information, refer to:

  • 2903776 - SAP Integration Suite – Service Plans and Upgrade Paths
  • 3386855 - The component JMS Sender with version 1.2 is not supported in SAP Cloud Integration profile due to missing license
  • 3558865- iFlow contains components that require an upgrade to your existing Cloud Integration license
  • [Self-service enablement of Cloud Integration service on consumption-based model - SAP Community]: https://community.sap.com/t5/technology-blog-posts-by-sap/self-service-enablement-of-cloud-integration-service-on-cloud-foundry/ba-p/13404094

Please contact your SAP Account Executive for guidance on licensing or upgrading to the required edition.

Long-Term solution: 

In previous versions of the integration with SAP CPQ, the iFlow included a JMS component to work around limited API capabilities and to ensure stable processing. With recent updates to the CPQ API, this workaround is no longer required.
 
What will Change:
SAP CPQ has introduced enhancements to its API that now allow direct updates to the fields that previously required JMS-based handling.
Thanks to this improvement, the integration can be simplified, and JMS can be safely removed from the iFlow while maintaining full functional support.

SAP is currently updating the affected iFlow to reflect this improvement. The updated version (1.2.1) will:

  • Remove the JMS component
  • Use the enhanced CPQ API for direct field updates
  • Improve transparency and reduce complexity in the integration design

This new version is in the publication pipeline and will appear in the integration package once released.

Symptom

You are looking for guidance on how to configure proxy server settings for your Smart Data Access (SDA) connections.

You may observe that SDA connections fail with below timeout errors or similar error messages:

  • SAP DBTech JDBC: [403]: internal error: Cannot get remote source objects: [SAP AG][LIBODBCHDB SO][HDBODBC] Communication link failure;-10709 Connection failed (RTE:[89009] Socket connect timeout (10000 ms); $host$= (host:port))

Other Terms

SDA; Federation; Proxy; Timeout; Smart Data Access; HANA; IQ; ASE; MaxDB; Oracle; MSSQL; Oracle; DB2; Netezza; GBQ; Teradata;

Reason and Prerequisites

Reason:

In scenarios where proxy servers are used, additional proxy configuration is required for the corresponding ODBC adapter.

Affected Releases:

  • SAP HANA 2:
    • Revisions of SPS00 - SPS08

Prerequisites:

  1. Smart Data Access (SDA) remote connection to any supported remote database
  2. And the local system is configured to use a proxy server.

Solution

To configure proxy server information for Smart Data Access (SDA) connections, please refer to the configuration guidelines provided by the vendor for the required ODBC driver properties.

Additionally, as outlined in SAP Note 2600176, check the vendor's distribution Web site for release notes, software patches, and other relevant information.

 

Related parameters can be configured in one of the following ways:

  • Directly within the remote source definition
  • In the corresponding DSN section of the .odbc.ini file
  • System-wide by setting environment variables, if supported by the ODBC driver

Choose the method that best fits your environment and the capabilities of your ODBC driver. Furthermore, please note the parameter names (for example, "proxyHostname", "proxyPort", etc.) may differ depending on your ODBC driver and vendor documentation.

Always refer to the documentation for the exact parameter names required by your driver and protect sensitive data such as credentials in accordance with your company's security policies.

 

Remote Source Properties

  • [CREATE | ALTER] REMOTE SOURCE "<REMOTE_SOURCE>" ADAPTER <ADAPTER_NAME> CONFIGURATION '<...>;proxyHttp=true;proxyHostname=<host>;proxyPort=<port>;proxyUserName=<user_name>;proxyPassword=<password>';

 

DSN

Add the ODBC driver-specific parameters and values to the corresponding DSN entry inside the ".odbc.ini" file as illustrated below:


[DSN1]
Driver = /path/to/driver
Server = your_server_address
Port = your_port
<...>
proxyHttp=true
proxyHostname=<host>
proxyPort=<port>
proxyUserName=<user_name>
proxyPassword=<password>

 

Environment Variable

Depending on whether the ODBC driver supports this approach, it may also be possible to set environment variables as shown below:

  • export http_proxy=http://<proxy>:<port>
  • export https_proxy=http://<proxy>:<port>
  • export no_proxy="localhost, 127.0.0.1, .googleapis.com"

If credentials are required, you can use the following environment variable configuration:

  • export http_proxy="http://username:password@proxy-url:port"
  • export https_proxy="http://username:password@proxy-url:port"

 

HANA

For HANA remote sources, you can edit the remote source directly and maintain the HANA ODBC Connection Properties as outlined in the SAP HANA Client Interface Programming Reference.

Edit the remote source and maintain the HANA ODBC Connection Properties:

  • ALTER REMOTE SOURCE "<REMOTE_SOURCE>" ADAPTER HANAODBC CONFIGURATION 'ServerNode=<servernode:SQLport>;Driver=libodbcHDB.so;dmlmode=readwrite;encrypt=true;sslValidateCertificate=false;proxyHttp=true;proxyHostname=<host>;proxyPort=<port>;proxyUserName=<user_name>;proxyPassword=<password>';

Alternatively, you can create a DSN entry in the .odbc.ini file and add the required parameters directly to that entry.

 

IQ

The SAP IQ ODBC driver provides the following configuration parameters for SOCKS proxy support:

ProxyHost=<string>
ProxyPort=<integer>
ProxyUsername=<string>
ProxyPassword=<string>

These parameters can be specified in your DSN or connection string to enable connections through a SOCKS proxy server.

For client applications, you specify network protocol options using the CommLinks (LINKS) connection parameter:

CommLinks=tcpip(ProxyHost=<string>;ProxyPort=<integer>;ProxyUsername=<string>;ProxyPassword=<string>;...)

For more details, please refer to the following documentation:

  1. Network Communications Parameters
  2. SAP IQ Connection and Communication Parameters Reference

Oracle

For the majority of the Oracle ODBC drivers, you can specify proxy details directly within the connection properties or DSN entry:

  1. ProxyHost: The hostname or IP address of the proxy server.
  2. ProxyPort: The port used by the proxy server.
  3. ProxyUser: (Optional) The username for proxy authentication.
  4. ProxyPassword: (Optional) The password for proxy authentication.
  5. ProxyMode: (Optional) Specifies the proxy mode.

These parameters can be entered in the DSN configuration wizard or added directly to the connection string:

  • DRIVER={DataDirect 8.0 Oracle Wire Protocol};HostName=123.210.123.210;PortNumber=5439;ProxyHost=123.255.78.90;ProxyPort=1521;...

For the native Oracle ODBC driver using sqlnet.ora and tnsnames.ora configuration files, you can configure an HTTP proxy using Oracle Net configuration files. To enable this, add SQLNET.USE_HTTPS_PROXY=on to your sqlnet.ora file.
Then, within the address section of your database service name entry in tnsnames.ora, specify the proxy hostname and port using the https_proxy and https_proxy_port parameters. For example:

ADB1_high = (description=
(address=
(https_proxy=proxyhostname)
(https_proxy_port=80)
(protocol=tcps)
(port=1522)
(host=adb.example.oraclecloud.com)
)
(connect_data=(service_name=adb1_high.adb.oraclecloud.com))
(security=(ssl_server_dn_match=yes))
)

 

For more details, please refer to the following documentation:

  1. Prepare for Oracle Call Interface, ODBC, and JDBC OCI Connections with Wallets (mTLS) 
  2. Connecting through a proxy server

Google BigQuery (GBQ)

To set connector-wide configuration options on a non-Windows machine:

  1. In a text editor, open the simba.googlebigqueryodbc.ini configuration file.
  2. In the [Driver] section, specify configuration properties as key-value pairs. Start a new line for each key-value pair.
    For example, to specify a proxy server and to log information from the libcurl library, type the following:
    • Proxy=http://<proxy.mycompany.ca>:<port>

In case a proxy is used, please note below:

    • You must make sure that the proxy allows communication to crl.pki.goog and ocsp.pki.goog.

    • Required certificates may have to be copied to the proxy host accordingly. These certificates are always required, because the communication with Google api is always secured.

      • Otherwise following error may occur: error setting certificate verify locations:  CAfile: /etc/pki/tls/certs/ca-bundle.crt CApath: none

      • See also SAP Note 3159682. - SDA - Google BigQuery ODBC Driver Setup

For more information, please refer to the Magnitude Simba Installation and Configuration Guide, which can be found here: ODBC and JDBC drivers for BigQuery.

Teradata

Teradata ODBC Driver, starting with version 20.00, can be configured to use a proxy server when making connections to servers or database nodes. Only HTTP proxy servers are supported.
Proxy server information is passed to Teradata ODBC Driver using connection string parameters. Using OS proxy settings is not supported at this point.

Order of precedence (highest to lowest):

  1. Connection string
  2. Environment variables

Proxy server settings can be passed to Teradata ODBC Driver via the connection string. The following connection string parameters are supported:

  1. HTTP_PROXY
  2. HTTP_PROXY_USER
  3. HTTP_PROXY_PASSWORD

Proxy server information can be passed to Teradata ODBC Driver using environment variables. Set the environment variables before running the Teradata ODBC Driver client application.
The following environment variables are available:

  • [http://][username:password@](host-name|ipv4_address|ipv6_address)[:port]

For more details, please refer to the official documentation on Teradata ODBC Driver Proxy Server Support.

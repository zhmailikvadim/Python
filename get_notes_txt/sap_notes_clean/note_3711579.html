Symptom

Material custom fields cannot be saved by MDG after upgrade to S/4 HANA 2023 or higher release. The issue only occurs for fields under namespace Y* or Z*. For fields under namespace YY* or ZZ*, no issue occurs. 

Environment

  • SAP ERP Logistics (LO)  
  • SAP S/4 HANA on premise 2023 and higher release
  • SAP S/4 HANA Cloud Private Edition 2023 and higher release

Reproducing the Issue

  1. Create/update materials using MDG functionalities. 
  2. Custom fields under Y* namespace or Z* namespace lost their values. 

Cause

The behavior is due to multiple reasons: 

In S/4 HANA 2023 and higher release, MDG changed the interface on material maintenance. Note 3371448 contains more details on the MDG design change.

  • In MDG releases on S/4 HANA 2022 or older, MDG calls classic MMD logic to maintain materials. 
  • In MDG releases on S/4 HANA 2023 or newer, MDG calls product unified API to maintain materials.  

Material custom field namespace usage: 

  • Field namespaces YY* or ZZ* are reserved for customer usage. 
  • Field namespaces Y* or Z* are reserved for SAP standard fields (e.g. there're standard fields like MARA-ZEINR, MARA-ZEIAR etc.).  

The product unified API is a completely new logic different from the classic MMD logic. In the classic MMD logic, material custom fields are updated by enhancement implementations. The classic enhancement implementations on updating material custom fields won't be called in product unified API. Therefore when material being maintained by product unified API, fields under namespace Y* or Z* lost their values. For fields under namespace YY* are ZZ*, as they're expected input for custom fields, product unified API can recognize them and therefore the values can be kept. 

Resolution

With the implementation of SAP Note 3206530, the Enhancement Spot 'ES_CMD_PROD_CUSTOMER_NAMESPACE' and BAdi 'BD_CMD_PROD_CUSTOMER_NAMESPACE' will be available. 

3206530 - Customer Namespace support in Direct Passthrough (Unified API)

Implement BD_CMD_PROD_CUSTOMER_NAMESPACE and provide the Custom Namespace(s) which should be allowed to save into Database. Then product unified API will be able to save the custom fields under the provided namespace. 

If there're custom fields being saved in custom tables, additionally implement SAP Note 3506979 which enables the function to custom tables. 

3506979 - Enhancement to include customer-specific tables from customer-specific namespace

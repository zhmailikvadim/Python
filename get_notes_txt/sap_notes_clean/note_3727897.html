Symptom

  • Calculation views are missing in the after an upgrade.
  • Error message: "Repository: Encountered an error in repository runtime extension; Model inconsistency. Create Scenario failed: ColumnView ECT::_SYS_BIC:Live_Reports.Purchase.Vendor_Master/AV_CHANGE_LOG (t -1) not found (cannot get catalog object)(calculationNode (AV_CHANGE_LOG))".
  • Error code: Inconsistent calculation model (34011).

Environment

S4/Hana 2025
SAP HANA PLATFORM EDITION 2.0

Reproducing the Issue

  1. Perform an upgrade from S/4HANA 2023 to S/4HANA 2025 FPS00.
  2. Attempt to test reports based on calculation views in the mock system.
  3. Observe that calculation views are missing and not activated.

Cause

  • The view AV_CHANGE_LOG is missing in the target system, which prevents the activation of calculation views.
  • During the upgrade, unsynchronized custom SAP repository views are removed due to synchronization behavior between the SAP ABAP system and the SAP HANA database.

Resolution

  1. Go to the source system.
  2. Locate the missing view AV_CHANGE_LOG under the package Live_Reports.Purchase.Vendor_Master.
  3. Export the view AV_CHANGE_LOG from the source system.
  4. Import the view AV_CHANGE_LOG into the target system.
  5. Activate the view in the target system.
  6. To prevent the deletion of custom SAP repository views in future upgrades, synchronize the views from the SAP HANA database to the ABAP system using transaction SCTS_HTA. Once synchronized, these views will not be deleted during upgrades.

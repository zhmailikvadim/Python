Symptom

The Check Project Inconsistency button in Gateway Service Builder (SEGW) results with the following warning messages:

  • Project '<ProjectName>' has been checked; check warnings
    • Model binding
      • Potential loss of data; ABAP type is more restrictive
      • Valid type mapping; not precise, but best possible
      • Conversion error possible; ABAP type is more restrictive

Environment

SAP Gateway (SAP_GWFND)

Reproducing the Issue

  1. Call t-code SEGW
  2. Open a Project
  3. Select the project name
  4. Click on the button Check Project Consistency

Cause

An OData service uses EDM types and not ABAP types for Entity Properties. These types have to be matched/mapped in SAP Gateway.
To check the possible mismatches the developer can use the Check Project Inconsistency button in the Gateway Service Builder (SEGW) transaction.

"Exact correlations between ABAP types and EDM types are not always possible due to their individual facets and the automatic determination of a corresponding ABAP or EDM type might not be a perfect match." To see more: Mappings and the ABAP Type Editor

These warnings might indicate serious future issues, therefore resolving them is an important task of the developer.

Resolution

  1. Expand the Model Binding.
  2. Double click on the given message entry (it'll open the given entity set's property table in SEGW and select the entry which the message is about).
  3. Further expand the message entry which you double clicked on and the check the suggestion.
  4. In the property table there's a column ABAP Type Editor, click on the button of that column in the selected row.
  5. Follow the suggestions of the message, if they're indeed valid and require maintenance. This detail belongs to the business logic therefore it has to be known by the developer of the project as it might depend on the future use cases of the service.
    • For example:
      • the first message says: "Potential loss of data; ABAP type is more restrictive".
      • by expanding it, suggestions like the following can come: Map instead to the data element 'TIMESTAMP (PACKED[15,0])'.
      • by clicking on the ABAP Type Editor button of the given property from the given type the types and facets are visible.
      • By following the suggestions the values have to be changed to TIMESTAMP PACKED with length 15,0.
      • To do that there are multiple approaches:
      1. Set the Determination Mode to Determination by Runtime Object Generator.
      2. If only the given OData service is using this data source then change the data element itself directly.
      3. Create custom conversion exits (see in the Conversion SAP Blogs in the See Also section of this KBA).

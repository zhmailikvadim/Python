Symptom

Deleting the target group using standard program, 'CRM_MKTTG_TG_DELETE', But program is not deleting the target group or members from table ‘CRMD_MKTTG_TG_H’ or ‘CRMD_MKTTG_TG_I’

"Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental."

Environment

  • SAP Customer Relationship Management - All Versions

Reproducing the Issue

  1. Login to SAP CRM WEB UI – MARKETINGPRO Business Role
  2. Select the 'Marketing' work center
  3. Search for the Target Group you want to delete and note down the TG ID
  4. Pick Target Group GUID from table ‘CRMD_MKTTG_TG_H’ by passing on ID – ex: ‘*12345’
  5. Entry Shows the TG GUID, make note of it, ex: 553C83EB6F7A0CC0E100812300A23B522
  6. You can check the BPs or memeber s of TG in table CRMD_MKTTG_TG_I using TG GUID
  7. Now to delete the TG (*12345), Call SE38 and execute the Standard Program ‘CRM_MKTTG_TG_DELETE’ using F8
  8. Now Supply the TG GUID and excute the Program using F8
  9. TG Should had been deleted from the table ‘CRMD_MKTTG_TG_H’, but the Entry still exists in database table

 

Cause

There are two ways to delete TG:

  1. Permanently delete from the Database
  2. Set the DELETE FLAG to True.

SAP has adopted second method as per the standard design.

Resolution

  1. The related items in CRMD_MKTTG_TG_I will be deleted and the DELETE FLAG is set to true in the header table CRMD_MKTTG_TG_H.
  2. If you permanently want to delete the TG from the Database, you can write a report for this.
  3. SAP doesn't have a standard program for deleting the TG from DB, to make the system consistent.

Symptom

Due to a hardware defect, the netCCS GmbH had to move the server for use of the Schufa compact credit report. Currently, the old server is still running, but can fail at any time and is therefore deactivated on 09/16/2016. To ensure that the Schufa compact credit report can still be used as of this point in time, the tasks described below are required. We recommend that you execute the work earlier if possible.

Other Terms

CGsprint, Schufa, natural person credit report, B2B, B2C, Business Line, netCCS, correction, error, web service

Reason and Prerequisites

This SAP Note is valid for the component CGSSB, Version 130_700, patch level 9 and version 220_702, patch level 3.

Solution

Adjustments in CGsprint:

  • For CGSSB, Version 130_700, import the attached transport request AO6K902729 (fiel name CGSSB_AO6K902729.SAR).
  • For CGSSB, Version 220_702, import the attached transport request KV5K904050 (file name CGSSB_KV5K904050.SAR) instead.

Import of the SSL certificate and firewall release:

  • Import the attached SSL certificate to the communicating SAP system (file name www.soasystems.de.crt). You must import the certificate in transaction STRUST to the PSE file "SSL Client (Standard)". Subsequently, use the menu option "Goto" in transaction SMICM for a "soft shutdown" of the ICM.
  • Set up a firewall/proxy release to enable the SAP system to send queries to the target address https://www.soasystems.de:443. You can remove the release for the old address https://h2230340.stratoserver.net:443. When you do this, note that the new address uses a self-signed certficate.

The problem is solved as of CGSSB, Version 130_700, patch level 10 and CGSSB, Version 220_702, patch level 4.

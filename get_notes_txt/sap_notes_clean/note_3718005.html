Symptom

In a self-written program you post operation confirmations to a shop floor order using BAPI BAPI_PRODORDCONF_CREATE_TT.

Even though no error is sent the confirmation is not saved due to warning "Order is a shop floor order. Use the Perform OA app for confirmations."

Other Terms

MPE_EXEC_CONF046, MPE_EXEC_CONF 046

BAPI, Prodcution Order Confirmation, Operation Confirmation

Reason and Prerequisites

Shop floor orders typically must not be confirmed manually.

  • This is since the operation confirmations are automatically created from operation activity execution.
  • Please see note 3707332 for details.

Nevertheless, it is not forbidden to manually post operation confirmations to shop floor orders.

  • This e.g. can be used to do corrections to the actual data.
  • To make the user aware of the fact that this is an exceptional situation warning MPE_EXEC_CONF 046 is sent.

Due to a program error this warning prevents the saving of the confirmation via BAPI.

Solution

Implement the correction instruction.

Symptom

When I perform a User Upload or run the integration add-on from S/4HANA, names containing special characters (e.g., accented letters like á, é, ñ) do not appear correctly in SAP Fieldglass. Instead, they are replaced by incorrect symbols or "garbled" text.

Cause

This issue occurs because the integration connector is not explicitly set to use UTF-8 encoding. When the source file containing special characters is processed using a different encoding standard (like ISO-8859-1), the system fails to translate the character symbols correctly during the data transfer into the Fieldglass database.

Resolution

You must ensure that the connector is configured to use UTF-8 encoding to support international character sets.

  1. Log in as a Configuration Manager user.
  2. Navigate to Admin Configuration > Integration > Connector Catalog.
  3. Search for the impacted connector (e.g., User Upload) and click Edit.
  4. Locate the Encoding setting in the connector configuration wizard.
  5. Set the encoding type to UTF-8.
  6. Complete the wizard and save the changes.
  7. Correct Existing Data: For users already uploaded incorrectly, you can perform a second upload with the new UTF-8 setting or manually edit the names in the UI to resolve the visibility issue.

Symptom

Short dump occurs during authorization check when creating or changing a Production Version in PEO

The dump happens in /bobf/if_lib_auth_draft_active~check_static_authority when calling function module EFG_USER_AUTH_FOR_OBJ_GET

Instead of showing an authorization error message, the process terminates due to an unhandled exception

Other Terms

PV - Production Version

Reason and Prerequisites

Custom authorization logic calls EFG_USER_AUTH_FOR_OBJ_GET without handling its exceptions (USER_NOT_FOUND, NOT_AUTHORIZED, INTERNAL_ERROR)

The function module can raise NOT_AUTHORIZED if YT_USVALUES remains initial (e.g. after internal mapping/filtering), which leads to a runtime error if not caught

Prerequisite: execution of the BOPF draft authorization check with a user lacking required authorizations for C_FVER_WRK (or missing values for ACTVT/WERKS)

Solution

Adjust the custom code to explicitly handle the exceptions of EFG_USER_AUTH_FOR_OBJ_GET using EXCEPTIONS and treat NOT_AUTHORIZED as an authorization failure (add an error message to eo_message instead of dumping)

Optionally, replace the approach with a standard AUTHORITY-CHECK for object C_FVER_WRK (checking ACTVT and WERKS once WERKS is available) to avoid dependency on returned authorization value lists

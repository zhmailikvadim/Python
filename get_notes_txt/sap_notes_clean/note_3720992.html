Symptom

  • No popup displayed
  • Postal Validation not working
  • Processing slow or failing
  • Start task failed
  • Address Validation stopped happening
  • No running or stuck paths causing the duplicate name
  • Logging showing that dropping the task is having a problem
  • Report /FLDQ/AD_REPT_DISP_ERRLOG shows: T101:023 DQM Error Code: T101 Location Code: 023
    • Error Code:  T101 : The call to START TASK returned an error message.
    • Location Code:  023 : CALL_DQ_ENGINE_ADR_VALIDATE method of CL_DQ_ENGINE_HANA_TASK
    • Error returned from Data Quality Server:
    • Error occurred creating task. – cannot use duplicate name of task: <taskname>
  • Report /FLDQ/AD_REPT_DISP_ERRLOG shows: T101:030 DQM Error Code: T101 Location Code: 030
    • Warning:  A DQM for SAP technical error T101 at location 030 has occurred at transaction.
    • Important: Please check your DQM for SAP Server for operational status.
    • Detailed specification of error situation
    • Error Code T101 : The call to START TASK returned an error message.
    • Location Code: 030 : Issue within /FLDQ/AD_GET_VALIDATED_ADDRESS

Environment

Data Quality Management for SAP Solutions powered by HANA

Cause

In this case the error was caused caused by an incorrect paste of the directory path through the host. Specifically, the folder for directory was mistakenly pasted inside the intended directory path, which resulted in an invalid structure.

Resolution

Ensure the path to the reference data is correct.

Note: After ensuring that the paths are correct and the script server is restarted to pick up the changes, , use transaction SM30 to open the Maintenance View for Table:  SIMDQ_TASK_UPDT  Put a check in all the boxes, to ensure that the task plans are rebuilt correctly on the HANA system and the being processing transactions as normal.

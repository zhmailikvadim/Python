Symptom

  • In transaction KOK2, a variant was created to select orders with a released status; however, it returns the message that no order has been selected.
  • Because no order is selected, the KOK4 transaction does not allow the modification of the order status from released to closed.

Environment

  • SAP R/3
  • SAP S/4HANA

Reproducing the Issue

  1. Access transaction KOK2.
  2. Enter Controlling area
  3. Enter Company Code
  4. Enter Order Type
  5. Select Release: X.

Cause

This is a system standard behavior as transaction KOK2 can only process order categories 01 and 02. Orders under category 05 are not supported by this transaction.

Resolution

  1. Utilize transaction KKF6M for product cost collector collective processing as an alternative.
  2. Confirm that the orders being processed belong to supported categories (01 and 02) for transaction KOK2.
  3. If further assistance is required, refer to relevant documentation or contact support.

Keywords

KOK2, KOK4, released status, order category, KKF6M, Product Cost Collector, Master Data

Symptom

In the process "Overtake and Undertake Quantity Process" (DE_MEMI), when INVOIC resending for MeMi is triggered, the process document reopens, but the INVOIC sending step can't run because step 800 (Check Before End Process) is already completed.

This issue is relevant for the market role: DSO.

Other Terms

MeMi, Resend INVOIC, DE_MEMI, DE_MEMI_CHK_BEF_END, Check Before End Process, /IDXMM/CL_UTILITY, REOPEN_PDOC, overtake, undertake

Reason and Prerequisites

Program error.

Solution

Please implement the corrections of this SAP Note by importing BC sets /IDXGL/2_0_CLNT_INDEP_SP17_002 and /IDXMM/CLNT_DEP_18_001 or the relevant Support Package (SP).
After implementing this SAP Note, when INVOIC resending is triggered for MeMi, the system sends the INVOIC successfully.
You need to retest INVOIC resending for MeMi.

You can verify the fix by checking that when INVOIC resending is triggered for MeMi, the INVOIC is sent successfully.

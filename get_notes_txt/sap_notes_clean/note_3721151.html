Symptom

When a PO line is created using the "Copy" function in the App "Create Purchase Order - Advanced," the Functional Area in the "Account Assignment" tab does not refresh automatically upon entering a new WBS element.

Environment

SAP S/4HANA Cloud Public Edition

Reproducing the Issue

  1. Open the App "Create Purchase Order - Advanced."
  2. Create a new PO line using the "Copy" functionality.
  3. Enter the corresponding WBS element.

Cause

The derive functional area only occurs if the field Functional Area (FKBER) is initial. When a PO is copied by referencing it, the system takes over the EKKO, EKPO, and EKKN data into the new PO without deriving new values for the GL account or Functional Area.

Resolution

  1. Verify that the Functional Area field is not initially empty when creating a PO line using the "Copy" function.
  2. Understand that the system behavior is designed to retain the EKKO, EKPO, and EKKN data from the copied PO without deriving new values for the GL account or Functional Area.
  3. If necessary, manually update the Functional Area field after entering the WBS element.

Symptom

Inbound transaction queue process transaction writes to the status log every single time the transaction is run. This becomes problematic when transactions fail and are rerun automatically the log can have too many of the same type of errors. This may not be desirable and supressing the log was not previously possible. This note addresses this issue by exposing a badi to the inbound queue bom class that allows transaction information influence just before the save.

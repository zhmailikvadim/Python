Symptom

  • An error occurs in MM42 or Manage Product APP when logging in via BTP.

Environment

  • SAP ERP Logistics (LO)
  • SAP S/4HANA, on-premise
  • SAP S/4HANA Cloud Private Edition

Reproducing the Issue

  1. Log in via BTP and access the "Manage Products" App or access MM42.
  2. Encounter the error: "%_KEEPALIVE does not exist in Table T133E."
  3. User backs out and restarts the transaction, losing changes.
  4. Attempt to reproduce the error in SAPGUI/WEBGUI, but the error does not occur.

Cause

  • The issue arises due to the absence of the %_KEEPALIVE command after the kernel update.

Resolution

  1. Refer to SAP Note 3616366 - Downport of command %_KEEPALIVE.
  2. Apply the solution provided in the SAP Note to restore the functionality.

See Also

  • Refer to: 3659212 - "Requested function %_keepalive is not available here" error after kernel update

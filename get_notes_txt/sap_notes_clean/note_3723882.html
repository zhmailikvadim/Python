Symptom

Despite configuring your payroll portlet and cards such as Pay Statements, Payroll Replication Details or Payroll Information correctly - employees are unable to access them, even with the required Role Based Permissions in the Latest People Profile.

Environment

  • Payroll Portlet in SuccessFactors Employee Central
  • Employee Central Payroll functionality part of SF

Reproducing the Issue

  1. Access SF as your affected employee
  2. Navigate to a Payroll Portlet in the Latest People Profile
  3. No Payroll cards shown

Cause

In the 2H 2025 Release. There is a missing null check in the REST API while rendering the data. After re-saving the data, the value of a BSI flag in the USA configuration is updated to False, which prevents the NullPointerException and resolves the issue.

The null-check handling will be properly addressed in the 1H 2026 release.

Resolution

  1. Ensure when you access Payroll Unified Configuration there are no errors. 
  2. If Payroll Unified Configuration loads without any errors, select your configured country.
  3. Refresh the configuration by clicking on Save
  4. Retest the scenario to verify if the payroll information block appears correctly on the employee's latest profile.

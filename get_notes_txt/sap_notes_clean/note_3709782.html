Symptom

Portugal SAF-T reports (Billing, E-Fatura, Integrated, Self Billing) finish with error:

  • Error message: "Entry (XX/99991231/VAT/PT/NOR): entered period can't overlap with the official tax rate period".

"XX" means the Tax Code.
"99991231" means the Valid to date.
"VAT" means the Tax Type.
"PT" means the Tax Country Region.
"NOR" means the Official Tax Code.

Environment

SAP S/4HANA Cloud Public Edition 2602 onwards.

Reproducing the Issue

  1. Go to Run Statutory Reports app.
  2. Run Portugal SAF-T Billing / SAF-T Integrated File / SAF-T E-Fatura report.
  3. The run status is Generation Failed.
  4. Observe the error message related to VAT code validation: "Entry (XX/99991231/VAT/PT/NOR): entered period can't overlap with the official tax rate period".

Cause

A new tax validation has been introduced to verify the validity of official tax codes and PT SAF-T tax codes.

Resolution

Official tax percentages valid on each official period has to be maintained in the Map Tax Code to SAF-T Tax Code configuration activity (ID: 106205)

  1. Go to Manage Your Solution app.
  2. Open the Map Tax Code to SAF-T Tax Code configuration activity (ID: 106205).
  3. Enter the Procedure 0TXPT.
  4. Search for the affected Tax Code.
  5. Mark the checkbox of the affected Tax
  6. Click on the Search help (F4) of the Tax Code field next to the Tax type field to identify the tax validity dates.
  7. Take a note of the Start and End date of the active tax.
  8. If the Valid from date of the Tax Code is earlier than the Start date of the official tax code, it is necessary to delimit the old entry and create a new one.
  9. Click "Delimit" button to create a new entry with the correct validity dates.
  10. Enter the Valid from Date. Here, inform the start date taken from the step 6.
  11. Save your Changes.
  12. Run the Portugal SAF-T report again.

Symptom

GST tax amount shall be calculated based on the Retail Sale Price (RSP) for notified goods, with the provision coming into effect from 1 February.
Tax amount = (Retail sale price X tax rate in % of applicable taxes) / (100+ sum of applicable tax rate) 
Note: Below configurations are applicable for scenarios where item base price excludes GST tax amount.   

"Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental."  

Environment

  • SAP S/4HANA Private Cloud
  • SAP S/4HANA On-Premise
  • SAP ERP

Resolution

 
 

Configurations on Purchasing side: 

Step 1: Create an access sequence to identify Tobacco goods and retrieve the relevant condition records that are mentioned in further steps. 

Transaction: M/07 (for Pricing conditions) 

Transaction: OBQ2 (for TAX conditions) 

 

Step 2: Create new condition types for RSP Price in Pricing conditions. Configure the above access sequence created. 

Transaction: M/06 

 ZMRP : To maintain the Retail Sale price (RSP) 

 

 

 

 

 

Step 3: Create new condition types for IGST, CGST, SGST and UGST tax conditions. Configure the above access sequence created. 

Transaction: OBQ1 

ZIIG, ZICG, ZISG, ZIUG : For calculation of tax amounts 

 

 

 

 

Step 4: Configure the Pricing procedure for RSP condition type. Make the condition ‘Statistical’. 

Maintain the Subtotal to ‘C’ ( for S4HANA releases); For ECC releases sample solution provided for Subtotal ‘5’. 

Transaction: M/08 

 

ZMRP : Should be statistical 

 

 

Step 5: Enhance the routine that calculates the Tax amount in ‘Alt Calc Cndn Amnt’ and Base amount in ‘Alt Cndn Base Amnt’. Follow below steps, 

  1. Please implement note 3709536 (pre-requisite) for S4HANA releases.  

Please refer and implement consulting note 3710104 for ECC releases. 

  1. Post implementation, Identify the routine number that needs to be enhanced. Suggested routine is 345. 

  1. Enhance the routine logic with sample code provided below. 

CALL FUNCTION 'J_1BSA_COMPONENT_ACTIVE' 

EXPORTING 

bukrs = komk-bukrs 

component = '**' 

IMPORTING 

active_component = lv_comp 

EXCEPTIONS 

component_not_active = 1 

OTHERS = 2. 

IF lv_comp EQ 'IN'. 

DATA: lv_cond TYPE j_1icondname. 

DATA: lv_taxrate TYPE komv-kwert. 

CONSTANTS mc_10 TYPE kwert VALUE 10. 

CONSTANTS mc_100 TYPE kwert VALUE 10000. 

DATA: lv_rsp_po_kwert TYPE kwert. 

CALL FUNCTION 'J_1I4_GET_KWERT' 

IMPORTING 

rsp_kwert = lv_rsp_po_kwert. 

IF lv_rsp_po_kwert IS NOT INITIAL. 

xkomv-kawrt = lv_rsp_po_kwert. 

ENDIF. 

lv_taxrate = xkomv-kbetr / mc_10. 

SELECT SINGLE cond_name FROM j_1iexcdefn INTO lv_cond WHERE kalsm = komk-kalsm AND kschl = xkomv-kschl. 

CASE lv_cond. 

WHEN 'IGSTAR' OR 'IGSTSOFFAP'. 

xkwert = ( xkomv-kawrt * lv_taxrate ) / ( mc_100 + lv_taxrate ) . 

xkomv-kawrt = xkomv-kawrt - xkwert . 

WHEN 'CGSTAR' OR 'SGSTAR' OR 'UTGSTAR' OR 'CGSTSOFFAP' OR 'SGSTSOFFAP' OR 'UTGSTSOFFAP'. 

xkwert = ( xkomv-kawrt * lv_taxrate ) / ( mc_100 + ( lv_taxrate * 2 ) ). 

xkomv-kawrt = xkomv-kawrt - ( 2 * xkwert ) . 

ENDCASE. 

ENDIF. 

 

Step 6: Configure the Tax procedure for GST condition types 

ZIIG, ZICG, ZISG, ZIUG:  

  1. Maintain the routine that calculates the Tax amount in ‘Alt Calc Cndn Amnt’ and Base amount in ‘Alt Cndn Base Amnt’. Configure 345 (or whichever is enhanced) here. 

Transaction: OBQ3 

 

 

 

 

 

 

Step 7: Maintain the PRICING condition records  

ZMRP: Maintain the retail sale price of the good  

Transaction: MEK1 

 

 

Step 8: Maintain the TAX condition records 

ZIIG - IGST 

 

ZICG - CGST 

 

ZISG - SGST 

 

 

 

 

 

Step 9:  Map new Tax conditions in J_1IEXCDEFN table 

IMG Path: Financial Accounting -> Financial Accounting Global Settings -> Tax on Sales/Purchases -> Basic Settings -> India -> Goods and Services Tax -> General Settings -> Classify Condition Types for Goods and Services Tax

Symptom

  • The SRS log shows error 702:
    E. ERROR #1028 DSI EXEC(105(1)) - neric/dsi/dsiqmint.c(5381) Message from server: Message: 702, State 1, Severity 20 -- 'Memory request for xxxxx bytes exceeds the size of single page of 16384 bytes.
  • The ASE error log reports the same.
    Error: 702, Severity: 20, State: 1
    Memory request for 16899 bytes exceeds the size of single page of 16384 bytes.
  • If the 702 error is set to print a stack trace via sp_altermessage 702, "with_stack","true" it will look like this:
    memalloc
    LeSQLRep::_sqlRepParamSet
LeSQLRep::SqlRepFixMatrix
LeDMLOp::_LeSqlRepLogSupport
LeInsertOp::_LeOpNext
LeEmitNoSndOp::_LeOpNext
LePlanNext
  • The setting for "sql statement replication" is enabled.

Environment

  • SAP Adaptive Server Enterprise (ASE) 16.0
  • RepAgent is enabled

Cause

  • This problem is seen only when the query text is exceeding 16 kilobytes of text.
  • The parameter 'sql statement replication' is enabled.
  • The issue has been logged under SAP CR 831716. Its description reads, "Error 702, "Memory request for xxxxx bytes exceeds the size of single page of 16384 bytes.", may be reported when "sql statement replication" is enabled.".

Resolution

    • This issue has been fixed.
    • The fix will be available starting in SAP Adaptive Server Enterprise (ASE) 16.0 SP04 PL09

    Workaround:

    Disable 'sql statement replication' if the 702 is encountered by a very large sql statement.

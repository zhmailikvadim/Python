Symptom

There is a question concerning the extension of the Work Scheduling view within the 'Manage Product Master Data' app in the SAP S/4HANA Cloud Public Edition system.

In the Public Cloud system, the Work Scheduling view can only be created or extended when at least one field contains a value.

This behavior differs from the MM01 app, where the Work Scheduling view extension is permitted even when no fields have been filled in.

Environment

SAP S/4HANA Cloud Public Edition

Reproducing the Issue

  1. Open the 'Manage Product Master Data' app.

  2. Switch to edit mode.

  3. Navigate to the Work Scheduling view.

  4. Save without entering any values in the fields.

  5. Open the 'Create Production Order' app to create a production order.

  6. The following error message is displayed: Work scheduling data of material & is not maintained (Message no. CO 070)

Resolution

The app does not support specifying views for product master data.  

The maintenance status is automatically determined based on the values entered in the maintained fields to ensure consistent processing by the system.  

Use the MM01 app as a workaround to create the Work Scheduling view without entering any values.

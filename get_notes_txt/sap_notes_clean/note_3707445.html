Symptom

  • Schema mapping is not working for DDL source files during BW/4HANA migration.
  • Error message like: "Artifact '<artifact-name>': moving artifacts of kind 'Entity' to a different schema is not supported".

Environment

SAP HANA Platform Edition

Reproducing the Issue

  1. Transport DDL source files (HANA repository) as part of HANA migration.
  2. Add the source schema to the target HANA system.
  3. Attempt to change the schema name in the target system in DDL source file.
  4. Observe the error message: "Artifact '<artifact-name>': moving artifacts of kind 'Entity' to a different schema is not supported".

Cause

  • SAP HANA Repository–based DDL sources (.hdbdd) have a known limitation: once an entity is created in a schema, it cannot be moved to another schema by simply changing the DDL source.
  • There is no direct "schema rename" functionality for repository-based DDL artifacts.

Resolution

  1. Note that moving artifacts of kind "Entity" to a different schema is not supported in SAP HANA Repository–based DDL sources. This is by design.
  2. Create a new entity in the target schema instead of attempting to move the existing entity from the source schema.

See Also

  • Refer to SAP Note 3194206 - How to rename a default SAP system schema (SAP, SAPSR3, SAPABAP<#>, SAPHANADB, etc.) on SAP HANA - SAP for Me.

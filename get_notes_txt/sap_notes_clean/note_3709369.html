Symptom

  • Errors encountered when executing sp_iqmonCacheDetails or select * from iqmonCacheDetails.
  • Error message: "SELECT returns more than one row SQLCODE=-185, ODBC 3 State='HY000'".
  • The displayed results may be incorrect or incomplete.

Environment

SAP IQ

Reproducing the Issue

  1. Create two tables having different owners with same table name and at least one same HG index name .
    Example:
    Create user1.table1 (a int, b int) ;
    Create user2.table1 (a int, b int) ;
    create HG index idx_hd_a on user1.table1(a) ;
    create HG index idx_hd_a on user2.table1(a) ;

  2. insert a couple rows

  3. select * from user1.table1 where a=5 ;
    select * from user2.table1 where a=5 ;

  4. select * from  iqmonCacheDetails ;

Cause

The issue has been logged under SAP CR 831725. Its description reads, "Error -185 on iqmonCacheDetails". 

Resolution

There is currently no target SAP IQ Service Pack scheduled for this issue (SAP CR 831725). It has been assigned to our Product Group. Please mark this KBA as a Favorite to be automatically notified when the CR status is changed.

Workaround :

  1. Get the sql code of the system procedure sp_iqmonCacheDetails_Update_Columns :
     sp_helptext "dbo.sp_iqmonCacheDetails_Update_Columns" ;

  2. alter procedure to add condition "table_owner"="ownerName" :

            set "objType" = (select "index_type" from "sp_iqindex"() where "index_name" = "indxName" and "table_name" = "tName" and "table_owner"="ownerName")

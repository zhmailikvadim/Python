Symptom

  • After migrating from the Legacy People Profile to the Latest People Profile a section that was previously not visible is now visible.
  • The corresponding card displays the message: “There’s nothing to show yet in the card.”
  • Administrators report that the section was previously restricted via Role-Based Permissions (RBP) and expect it to remain hidden.

Environment

SAP SuccessFactors HCM Suite

Cause

This is expected behavior.

In the Legacy People Profile, access control for sections was managed through:

User Permissions > Employee Views > Employee Views Section

If RBP was not configured for a section, it was not visible to users.

In the Latest People Profile:

  • Categories replace legacy sections.

  • Section-level permissions under Employee Views Section are no longer supported.

  • Access to categories is controlled via the Configure Latest People Profile admin tool.

  • If a category is not explicitly configured with target roles, it becomes visible to all users by default.

Additionally:

  • Role-Based Permissions settings related to Employee Views Section are not included in the migration.

  • Product-specific permissions remain unchanged.

  • Only legacy section-based permissions are discontinued.

As a result, categories that were previously restricted via legacy section permissions may now appear visible after migration, even if no data is available, leading to the message:

“There’s nothing to show yet in the card.”

Resolution

To restrict visibility of the section category in the Latest People Profile define target roles as outlined in Defining Target Roles for a Full Profile Category | SAP Help Portal.

Symptom

In a subcontracting scenario, generating an outbound delivery using the ME2ON transaction code results in the delivery not being created, displaying the message "0 item(s) posted".

 

Environment

  • SAP ERP
  • SAP S/4HANA

Reproducing the Issue

  1. Adjust the movement type 541 in J_1BIM02V from 'X' to 'U'.
  2. Create a purchase order following the guidelines in KBA 2479074.
  3. Attempt to create the outbound delivery using the Subcontracting Cockpit with the ME2ON transaction code.

Cause

The Subcontracting Cockpit works exclusively for Outbound Deliveries. Therefore, Partner configurations must be set to "Ship-To party"; otherwise, the error "0 item(s) posted" will be triggered.

Resolution

    To resolve the issue:

    1. Change the Main Partner ID configuration to WE (Ship-to party) in the "Sales Document Item Category Table" (J_1BSDICA) view.
    2. Apply SAP Note 3707771 - Missing Nota Fiscal For Movements 541 Or 542 When Posting Delivery For Subcontracting Purchase Orders to your environment.

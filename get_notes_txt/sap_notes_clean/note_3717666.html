Symptom

  • PSE Certificate not renewed by /SDF/CALM_CERT_ROTATION job
  • Error "Logon of user <USER> in client <CLIENT NUMBER> failed when starting a step" in /SDF/CALM_CERT_ROTATION job
  • Error  "/SDF/CALM_CERT_ROTATION aborted on: <DATE>" on /n/sdf/alm_setup

Environment

  • SAP Cloud ALM
  • ABAP System - ST-PI 740 SP29 or higher

Reproducing the Issue

  • Access the managed system ABAP
  • Go to /n/sdf/alm_setup and check the error in step 3.
  • Go to /nSM37
    • Search for /SDF/CALM_CERT_ROTATION job
    • Check the log for the current day.
  • Go to /nSTRUST and check the SSLClientSAPCloudALMClientPSE folder.

Cause

Missing authorization.

Resolution

  1. Assign the role SAP_SDF_ALM_MTLS to the user Background user for rotation configured under /n/sdf/alm_setup.
  2. Wait for the next job run after role assignment.
  3. If the issue persists, review the configuration prerequisites in the SAP Cloud ALM Setup and Administration Guide.

See Also

Refer to: SAP Cloud ALM Setup and Administration Guide.

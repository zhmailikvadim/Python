Symptom

  • The following message together with a stack trace is reported in the ASE errorlog:

﻿Invalid OAM page (OAMINVALID_1) stat 4: pageno=4720, sdes_stat=16, pstat=0x101, dbid=5, pptnid=-3604481, pindid=0, expected ptnid=-16580611﻿

  Note: A complete stack trace is available under Attachments.

Environment

SAP Adaptive Server Enterprise (ASE) 15.7 and 16.0

Cause

The issue has been addressed under SAP CR 817535. Its description reads, "In rare circumstances, the message, 'Invalid OAM page (OAMINVALID_1) stat 4: pageno=<value>, sdes_stat=<value>, pstat=<value>, dbid=5, etc.", together with a stack trace that includes the modules 'th_accounting', 'th_sleep_on_lct' and ''terminate_process', may be printed in the SAP ASE error log when a SELECT query that requires a work table to be created is killed.".

Resolution

Upgrade ASE to a release where SAP CR 817535 is fixed:

  • SAP ASE 15.7 SP141 or higher
  • SAP ASE 16.0 SP02 PL09 or higher
  • SAP ASE 16.0 SP03 PL07 or higher

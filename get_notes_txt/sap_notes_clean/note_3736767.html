Symptom

After execution employee synchronization, there is SLG1 error: Tax number category BR2 does not exist (message number: FKBPTAX 003 ) of the specific employee.

Environment

S/4 HANA OP with HR integration active status

Reproducing the Issue

  1. Hire employee with BR country company code.
  2. Define tax configurations under the company code with specific example tax category BR2 via table PTRV_VENDOR_SYNC to generate/update relevant vendor, according to attached consulting guide KBA 3215465 - How to Synchronize Tax number from Employee Master Data to BP/Vendor (LFA1-STCD*)
  3. Synchronization the employee via example report /SHCM/RH_SYNC_BUPA_EMPL_SINGLE
  4. Go to SLG1, you will find error: Tax number category BR2 does not exist (message number: FKBPTAX 003 ) of the specific employee.

Cause

Before starting any employee synchronization to generate relevant tax master data of relevant synchronization vendor BP, the tax category BR2 configuration was not created in table TFKTAXNUMTYPE / view V_TFKTAXNUMTYPE, according to attached note 775919 - Missing tax number categories of business partner

Resolution

Please create or transport the tax category BR2 in table TFKTAXNUMTYPE / view V_TFKTAXNUMTYPE under current S/4 HANA system.
After that, please re-synchronize the employee.

The similar logic also does work for other tax category.

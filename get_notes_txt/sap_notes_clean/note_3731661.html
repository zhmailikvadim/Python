Symptom

You want a Webdynpro Abap application to use a certain theme. But that theme does not load. Instead, the default theme is loaded.

Other Terms

sap-theme; theme

Reason and Prerequisites

The symptom can have mutiple causes. The cause we detail and tackle in this note is a stale shared buffer. The webdynpro abap framework checks at application start the existency of a requested theme and writes the result into a buffer. Once the status of a certain theme is written into buffer, the existence of a specific theme is not checked anymore - it´s status , existent or not , is read from that buffer, although -  in very rare cases and under abnormal cisrcumstances -  it´s status couid have changed to the better ( existent ). Until now, a system restart was required unless you knew how to manually delete the buffer. With this correction the existence of a theme is re-checked and the buffer is updated accordingly in case the theme is flagged as "not existent". One rare situation is for example when there is a problem reading the mime repository.

Solution

Apply the solution either via SNOTE or Support Package of your SAP_BASIS / SAP_UI release.

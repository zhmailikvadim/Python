Symptom

ASE may not eliminate partitions correctly when a composite key is used for table range partitioning.

Environment

SAP Adaptive Server Enterprise (ASE) 16.0 SP02 PL01 but not limited

Reproducing the Issue

  1. Create a table:
     
    create table test (a int,b int) 

  2. Add range partitions:

    alter table test
    partition by range (a, b)
    (partition_1 values <= (0, 4) on segment1,
    partition_2 values <= (0, MAX) on segment1,
    partition_3 values <= (1, 2) on segment1,
    partition_4 values <= (1, MAX) on segment1)

  3. Insert one row of data per each partition

    insert into test values (0,1)
    insert into test values (0,5)
    insert into test values (1,1)
    insert into test values (1,3)

  4. To see the query plan for a query, use:
     
    set showplan on
     
  5. Select data coming from partition_1 and partition_2 only by running this query:
     
    select * from test where a=0
/* showplan output includes :

        STEP 1
        The type of query is SELECT.
        1 operator(s) under root
       |ROOT:EMIT Operator (VA = 1)
       |   |SCAN Operator (VA = 0)
       |   |  FROM TABLE
       |   |  test
       |   |  [ Partitions Used: 3, Eliminated: 3]

       |   |  [ Eliminated Partition ids : 4 5 6 ]
       |   |  [ Used Partition ids : 1 2 3 ]

       |   |  Table Scan.
       |   |  Forward Scan
       ......
 a           b
 ----------- -----------
           0           1
           0           5
    
*/

NOTE: Partition 3 is used, although it should have been eliminated based on the where clause.

Cause

  • This issue has no impact on the actual result of the query, but only on the optimiser performing an unncecessary partition scan. This may potentially lead to non-optimal query performance, but not to the wrong set of rows being returned by ASE.
     
  • This issue is addressed in SAP CR 794368. Its descrioption redas, "Incorrect table partition elimination when a composite key is used for table range partitioning".

Resolution

  • There is currently no target Fix Pack or Service Pack scheduled for SAP CR 794368. It is being reviewed by our Product Group. Subscribe to this KBA to be automatically notified when SAP CR 794368 is fixed.
      
  • Workaround: For the time being, the only known way to avoid this issue is to use single key range paritioning.

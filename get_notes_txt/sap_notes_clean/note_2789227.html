Symptom

When changing the dates on an existing payment or dunning lock in FPE2M the system allows you to have an expiry date smaller than the valid from date.

Other Terms

FPE2M,  t_locks,  loctab[], loftab[], fdate, tdate, MANSP_TDATE, MANSP_LST,

Reason and Prerequisites

The problem is caused by a program error.

Solution

Implement the source code corrections. Make a manual advance correction or import the Support Package that contains the correction.

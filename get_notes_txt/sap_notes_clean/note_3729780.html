Symptom

You are using production orders, process orders, maintenance or service orders or networks. The order or network contains material components with batch managed single discontinuation items.

You enter batches for the items that belong to the single discontinuation group. Then, you perform an availability check.
Even though only part of the requirement quantity is transferred to the follow-up item, the fully requirement quantity is committed for the follow-up item, if available.

Other Terms

CO01, CO02, CO07, CO08, CO10, CO40, CO41, production order
COR1, COR2, COR7, COR8, CORO, process order
IW31, IW32, maintenance order, service order
CN21, CN22, network
Discontinuation control, simple discontinuation, successor, KZAUS, RSART
Availability check, ATP, Available-to-Promise
Batch management, RESB-CHARG

Reason and Prerequisites

The cause of the issue is a program error.

Solution

Implement the source code corrections as described in the correction instructions.

Please bear in mind that the combination of batch splits and simple discontinuation and batch splits is still not supported. Please refer to SAP Note 125364 for more details. The correction only deals with items without batch splits.

Symptom

  • Functional Area cannot be assigned at an individual WBS level during migration using the "Migrate Your Data" app.
  • Functional Area entered at the Project level propagates to all subordinate WBS elements.
  • Functional Area differentiation by characteristics of each WBS under the same Project is not supported.

Environment

SAP S/4HANA Cloud Public Edition

Reproducing the Issue

  1. Attempt to mass-create Enterprise Projects using the "Migrate Your Data" app.
  2. Enter a Functional Area at the Project level.
  3. Observe that the Functional Area propagates to all subordinate WBS elements.

Cause

  • In SAP S/4HANA Cloud, the Functional Area loaded via the "Migrate Your Data – Enterprise Project" migration object is maintained at the project header level and propagates ("inherits") to all WBS elements below it.
  • It is standard system behavior that all WBS elements in the same subtree share the same Functional Area as the project or billing WBS element.
  • There is no supported functionality in the standard migration template to assign different Functional Areas per individual WBS under the same project.

Resolution

  1. Note that it is standard system behavior in SAP S/4HANA Cloud that Functional Area is maintained at the project header level and propagates to all subordinate WBS elements.
  2. Understand that there is no supported way in the standard migration template to assign different Functional Areas per individual WBS under the same project.

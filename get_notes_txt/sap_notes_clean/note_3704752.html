Symptom

  • A process for creating sales orders with a large volume of items using BAPI_SALESORDER_CREATEFROMDAT2 encounters an error when there is incomplete data at the item level.
  • The BAPI returns message V1 555 with a generic text, which does not specify the missing data or the incomplete field.

Environment

  • Sales And Distribution (SD)
  • SAP ERP Central Component
  • SAP ERP
  • SAP Enhancement package for SAP ERP
  • SAP Enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA
  • SAP S/4HANA Cloud Private Edition

Reproducing the Issue

  1. Access transaction SE37.
  2. Run BAPI_SALESORDER_CREATEFROMDAT2.
  3. The BAPI returns message V1 555 with a generic text, which does not specify the missing data or the incomplete field.

Cause

  • BAPI_SALESORDER_CREATEFROMDAT2 is designed to return the message V1 555 with a generic text, which is considered standard behavior.
  • Not all dialog processing functions available in SAP are supported by BAPIs due to the restricted scope of functionality.
  • Note that the behavior described, where the incompletion detailed message appears in VA02 but not in the BAPI, is expected due to the limitations of background processing. For further details, refer to SAP Note 550726 - FAQ: BAPIs in sales - Restrictions.

Resolution

  • As a workaround, consider using BAPI 'SD_SALESDOCUMENT_CREATE' as a workaround, which provides the INCOMPLETE_LOG export table containing incomplete entries (VBUV entries).
  • To identify the table and field names when message V1 555 is raised, read the table INCOMPLETE_LOG to retrieve all incomplete entries in 'SD_SALESDOCUMENT_CREATE'

Symptom

After a release upgrade,  for a Portuguese Company Code, in the productive system,  subsequent entries on table SIPT_VBRK (and/or SIPT_BKPF) do not have a digital signature .

Environment

SAP_APPL all releases.

Reproducing the Issue

  1. Digital signature was already working in a productive environment.
  2. The SAP productive system has been upgraded to a higher version.
  3. The entries at tables SIPT_VBRK and SIPT_BKPF of documents created after the upgrade don´t have a digital signature.

Cause

Each SAP release has a different PSE file.

There are checks  that detect if the PSE file uploaded is the right one or not for productive systems.

Resolution

After the upgrade:

 

  1. The new Product Data of the SAP system has to be properly entered in the SAP Service MarketPlace. (This is the way to notify SAP that the system is now on a higher release). Do not keep obsolete data.
  2. Then, the  productive encryption certificate for the current SID, has to be downloaded from the SAP Service Marketplace.
  3. Report SIPT_SAFT_STORE_PSE has to be run with transaction SE38  to upload the new PSE file (obtained in previous step) in the productive system.
  4. Run reports SIPT_INTERVAL_SIGN_SD, and/or  to SIPT_INTERVAL_SIGN_FI (also with transaction SE38)  to  generate the missing digital signatures.

See Also

The path to download the encryption certificate has changed in the Service Market Place: Note 2151028 released on April 2015 with the current path.

Note 172481 is a collective one that covers System data maintenance topics. (There is a special SAP component called XX-SER-FORME)


Note 1822433  "1S309 SSF kernel error: invalid parameter" outlined the need for the new PSE file after some upgrade.

Notes 1829604 and 1829640 explain other causes for having documents on tables SIPT_VBRK and SIPT_BKPF without a digital signature for all kind of systems (productive or not) not related to the upgrade.

Symptom

You are working with inbound delivery items that are relevant for Quality Inspection.

Multiple inbound delivery items (with the same product, batch, and reference document such as Purchase Order, Process Order, or Production Order) are assigned to the same Quality Inspection Lot. For some of these deliveries, the Goods Receipt has already been posted, and the inspection lot is still open (no usage decision has been made). For other deliveries, the Goods Receipt has not yet been posted, but open warehouse tasks exist.

If you cancel an open warehouse task for one of these deliveries (without Goods Receipt), the system cancels the entire inspection lot, even though it still contains open quantities from the deliveries for which Goods Receipt was already posted.

Other Terms

embedded, Inpection Lot, Inspection Lot Summary, Cancel Putaway

Reason and Prerequisites

 Code error.

Solution

Implement the attached code corrections.

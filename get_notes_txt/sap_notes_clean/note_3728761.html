Symptom

  • A Stock Transfer Requisition was successfully created using the IBP Purchasing OData V4 API, and the POST request returned a 201 – Created response.
  • The Stock Transfer Requisition is not visible in IBP, either in the Projected Stock or Orders app.
  • No corresponding entries were found in IBP OData monitoring or application logs for the successful message.
  • Status check in the API process returned "Processed with Errors".

Environment

  • IBP Master Data and Transactional Data API services

  • SAP Integrated Business Planning (SCM-IBP)

  • SAP Cloud Integration for data services  (CI-DS)

Reproducing the Issue

  1. User either POSTMAN or https://api.sap.com/api/IBP_Purchasing_RAP_ODataService/tryout
  2. Execute GET - GetTransactionID - /IBPPurgDocRootAsyncWrite/SAP__self.GetTransactionID() , 
  3. Execute a POST request using the provided sample payload to create a Stock Transfer Requisition - /IBPPurgDocRootAsyncWrite/SAP__self.InitiateParallelProcess  - this is for JOB.   
  4. Execute POST - /IBPPurgDocRootAsyncWrite - this is for one record, sample testing purpose 
  5. Execute POST - /IBPPurgDocRootAsyncWrite/SAP__self.Commit  - commit the data
  6. Execute GET -  /IBPPurgDocRootAsyncWrite/SAP__self.GetStatus(TransactionID={TransactionID}), to get the Transaction status
  7. Here you can find that transaction status is PROCESSED WITH ERRORS
  8. Check the STR or PR in the Projected Stock or Orders app and note that the Stock Transfer Requisition or PR is not visible.

Cause

  • Executed POST  data is not yet processed into IBP

Resolution

  1. To get more information about an error by using the following standard the GET Request for the transaction ID, you would get the missing information in the payload, you must add those items in the payload and repeat steps from 3 to 6 in the recreate steps.
  2. GET <host>/sap/opu/odata4/ibp/api_purchasing/srvd_a2x/ibp/api_purchasing/0001/IBPPurgDocRootAsyncWrite/SAP__self.GetStatus(TransactionID= 11112222-3333-4444-5555-666677778888 )
  3. Suppose for STR payload, Check in Payload "IBPPurgDocType": "STR_ITM" requires both ship-from and ship-to location
  4. You can view the STR in the Projected stock App

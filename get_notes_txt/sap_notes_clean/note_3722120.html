Symptom

When starting transaction SM35 – Batch Input: Session Overview, the system loads all existing Batch Input sessions stored in table APQI. In systems with a very high number of sessions, this can result in:

  • Long startup times
  • High memory consumption
  • Potential out-of-memory terminations
  • Unresponsive UI behavior

Other Terms

SM35, APQI, BDC Sessions, Performance, OOM, TSV_TNEW_PAGE_ALLOC_FAILED, BDC_MAX_SESSIONS, Message 00753.

Reason and Prerequisites

During the initialization of transaction SM35, the system attempted to load all existing Batch Input sessions from table APQI into memory. This behavior resulted from the fact that SM35 did not previously implement a technical upper bound for the number of sessions to be read and displayed at once.

There are no specific prerequisites for the issue; it can occur in any system with an unusually large population of Batch Input sessions, regardless of release or support package level.

Also remember to delete sessions that are no longer needed. Instructions for this can be found in SAP Notes 147354 and 18307.

Solution

To prevent performance degradation and memory issues, an update has been implemented in the SM35 session overview logic. The correction introduces the following changes:

1. Introduction of a Display Limit for Sessions

A new internal threshold has been implemented: If the total number of sessions in table APQI exceeds 100,000, SM35 will display a maximum of 100,000 sessions by default and the message 00753 is raised to notify the user. This safeguard ensures stable transaction startup and prevents excessive memory usage.

2. New SET/GET Parameter: BDC_MAX_SESSIONS

A new parameter has been added to allow users to configure the maximum number of sessions to be displayed:

  • Parameter name: BDC_MAX_SESSIONS
  • Valid input range: 0 – 999999999 (the value must be entered without separators)
  • If the parameter is maintained in the user profile (SU3), SM35 will use this value directly and the internal limit of 100,000 is not applied.
  • If the parameter is maintained in the user profile and the value remains empty or 0, the check for the internal limit will be deactivated.

3. New Input Field in SM35: "Max. Sessions"

In the "Selection criteria" area of SM35, a new input field has been added:

  • Field name: "Max. Sessions"
  • Purpose: Allows the user to adjust the maximum number of sessions to be displayed at runtime.
  • The field is linked to the SET/GET parameter BDC_MAX_SESSIONS.
  • If no value or 0 is entered, the system applies no limit.

4. New Footer Fields for Visibility and Transparency

The SM35 footer now contains two output fields:

"Sessions found" – Displays the total number of sessions in table APQI, corresponding to the selection criteria (without the "Max. Sessions" limit).
"Displayed sessions" – Displays the final number of sessions shown in the SM35 list (after applying limits or user input).

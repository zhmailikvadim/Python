Symptom

  • When the name property of a role is changed, the updated name does not appear at runtime in the top-level navigation.
  • The issue persists despite clearing the PCD cache and Navigation cache.

Environment

SAP Netweaver Enterprise Portal

Reproducing the Issue

  1. Navigate to Content Administration.
  2. Select Portal Content.
  3. Select the desired role.
  4. Click on properties.
  5. Change the name property.
  6. Save the changes.

Resolution

  1. Verify that the role whose name property has been changed is set as the entry point and that the visible option is enabled.
  2. If the role is not defined as the entry point, review the properties of the Workset within the role. The actual entry point may be assigned to the Workset, and in such cases, the Workset name is what appears in the top-level navigation.

See Also

SAP Help Document on Entry Points.

Symptom

While carrying out the manual instructions in SAP Note to add OTR text, you get error message 'Invalid alias' (technical name SOTR_MESS 504).

Example note like Solution Manager SAP Note 2325006 , CRM Web UI framework SAP Note 2412531 . This can happen to any note which contain the similar steps.

*Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.

Environment

  • Solution Manager 7.2, SP1, SP2 and SP3
  • SAP System with WEBCUIF as one of software components
  • Or any ABAP SAP systems

Reproducing the Issue

1. Log on to your SAP Solution Manager system in English.

2. Call transaction SE38.

3. In the "Program" field, enter BTFR_CREATE_ALIAS_FOR_CONCEPT.

4. Choose "Execute" to run the program.

5. Enter the values in the screenshot according to the SAP Note 

6. error message appears as detailed in Symptom section

    Cause

    Most of the time, the reason is because the Alias already exists. To check that the alias exists:

    1. Go to transaction SOTR_EDIT
    2. Enter the problematic alias, for example AI_CRM_CM_UI/MEANING_OF_ROW or CRM_BSP_WD_APPL_BASICS/SHARE_LINK, click on display
    3. The alias already exists: It looks like this:

    Resolution

    If the Alias already exists - the manual steps in the note does not need to be implemented. It might be necessary to investigate whether the SAP Note is suitable for current system or not.

    The complete reasons for the error ' Invalid alias ' SOTR_MESS 504 is as following. And the following detail explanation will be provided as long text for error message SOTR_MESS 504 in future BC support packages/releases.

    1. The alias already exists.
    2. The alias does not fulfil the naming convention requirement: <package>/<aliasname>.
        The sign  "/" can only be used as separator and cannot be part of the aliasname.
    3. The alias contains invalid signs. 
        Only the following signs are allowed in a alias  $/ABCDEFGHIJKLMNOPQRSTUVWXYZ0123456789_.-    
    4. Invalid package
    5. Alias is too long 
        The alias can have maximum length 50.

Symptom

In the "Assignment Analysis and Test Plan Generation" app, you press the refresh button of test case selection list. Then a server error with HTTP status code 500 occurs. The short text of the error is "The Assert condition was violated". The runtime error analysis in the ABAP back-end server (transaction ST22) shows that a runtime error ASSERTION_FAILED occurred in Web Dynpro Component /SALM/CHECK_BB_ITR_TC (ABAP Program /1BCWDY/0O2TQAD7YBTPWZL0KL9L==CP).

Other Terms

Assignment Analysis and Test Plan Generation, Web Dynpro Component /SALM/CHECK_BB_ITR_TC, ABAP Program /1BCWDY/0O2TQAD7YBTPWZL0KL9L==CP, ASSERTION_FAILED, UPDATE_CNT_TPCK.

Reason and Prerequisites

The problem occurs only if no test plan was created or loaded during the session.

Apply the following source code correction or the corresponding support package.

Symptom

  • Excel imports for transaction or Org. data like Positions data loads run indefinitely when large files are uploaded.
  • Files with 5k+ records remain in a waiting stage for extended periods, exceeding one hour.
  • No errors re notified on the UI for the run.
  • This article suggests the best practices, limits and the planned notification enhancements. 

Environment

  • SAP Sales Performance Management
  • Incentive Management on Hyperscalers

Reproducing the Issue

  1. Load a very large transaction data or Org. data like Positions (20k+ records) file in the corresponding Incentive Management workspaces
  2. Observe the status remaining in a waiting stage for more than one hour.

Cause

The issue is suspected to be caused by the data volume in large files exceeding standard limits for optimal performance.

Resolution

  1. Refer to the Help documentation for recommended limits: "For the best performance, do not use Data Loaders for more than 1000 records per object."
    • Refer the Help section for more details on the Data Loaders
  2. Split large files into smaller chunks (e.g. 1000 records per file) and attempt to upload them individually.
  3. Monitor the runtime of each file upload to ensure successful completion.
  4. For files that run indefinitely, terminate the process and retry with smaller file sizes.
  5. Future updates will include validations and error messages to notify users of extended runtimes or file size limitations.
  6. The proposed changes and notifications are
    1. A message to suggest users to load no more than 1k records in excel spreadsheet in UI side.
    2. A validator to limit maximum 10K records in a single excel spreadsheet
    3. To Stop a job if job runs more than 5 hours and include the message in the response of job status so that users can perform re-importing

Refer the article for updated release timelines for the changes.

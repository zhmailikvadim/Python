Symptom

  • Digital Access consumption is being reported for sales orders created by unattended bots using the standard sales order API.
  • This behavior contradicts the understanding based on the Digital Access Consumption Guide, which states that no additional licensing should be required for SAP RPA usage.

Environment

SAP S/4HANA Cloud Public Edition

Reproducing the Issue

  1. Prepare an Excel file containing Sales Order data (Customer, Material, Quantity, etc.).
  2. Trigger the SAP RPA unattended bot developed in SAP Build Process Automation.
  3. The bot reads the Sales Order data from the Excel file.
  4. The bot calls the standard Sales Order API in SAP S/4HANA Cloud to create the Sales Order.
  5. The Sales Order is successfully created in the system.
  6. After document creation, check License Compliance → Digital Access.
  7. The Sales Order created by the bot appears in the Digital Access consumption report.

Resolution

  • The created by user for the standard API calls is identified as a CC user, which leads to the activity being classified as DA consumption.
  • To avoid counting it as Digital Access consumption, configure the SSCUI '105628 - User Registration for Digital Access' as per the instructions outlined in this documentation : User Registration for Digital Access.

Symptom

When regular execution date of Standing Order is on the same day as last execution date, Function Module FSTB_FUT_PAYM_GET_BY_CONTRACT returns two executions instead of just the last payment. This happens when Standing Order already executed on the first date( or SO has an execution history).

This SAP Note implements the logic to return one execution if the regular execution date is on the same day as last execution date.

Other Terms

CL_BCA_FUTURE_SO_PAYMENTS, First Last Active Indicator, BCA_API_SORD_GETLIST

Reason and Prerequisites

Program Error.

Solution

Import the relevant Support Package for your release. You can use transaction SNOTE to implement the correction instructions relevant for your release in advance.

There are no manual activities.

Symptom

When movement type 335 (sample withdrawal) is posted for stock under inspection, the withdrawn quantity is not deducted from the inspection lot quantity. This leads to a discrepancy between the QM inspection lot and the MM stock situation.

As a result, MD04 shows incorrect available quantities, including negative stock projections.

Environment

  • SAP R/3
  • SAP R/3 Enterprise
  • SAP ERP Central Component
  • SAP ERP
  • SAP Enhancement package for SAP ERP
  • SAP Enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA, on-premise
  • SAP S/4HANA Cloud Private Edition

Reproducing the Issue

  1. Create Purchase Order for material
  2. Post Goods Receipt (movement 101) → inspection lot is created
  3. Verify stock is in Quality Inspection (QI)
  4. Post movement type 335 (sample withdrawal), e.g. via MB1A/MB1B
  5. Display stock in:
    • MMBE → stock remains in QI (partially reserved)
  6. Check MD04
    • Available quantity becomes negative or inconsistent

Cause

  • Movement type 335 updates physical stock but does not reduce the inspection lot quantity.
  • When posted via obsolete transactions MB1A / MB1B, the system:
      • Performs MM posting only
      • Does not ensure full integration with QM processes

  • QM still considers the full inspection lot quantity.
  • MD04 inconsistencies arise due to different logic between QM and MRP.

This leads to a mismatch between QM and MRP views, resulting in incorrect stock.

Resolution

  • Avoid using obsolete transactions like MB1A and MB1B; use transaction MIGO instead.
  • Refer to SAP Note: 2210569 – Obsolete Material Inventory Management Transactions.
  • Avoid manual stock withdrawal (335) used in parallel to QM sampling.

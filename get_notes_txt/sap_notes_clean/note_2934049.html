Symptom

The access control using Assigned Users is maintained when configuring the Communication Component in the Integration Directory. When transporting Communication Component objects which have user info defined under tab Assigned Users in source system, it's observed that the user info is missing post transport in target system.

Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.

Environment

  • PI Release Independent
  • SAP NetWeaver
  • SAP Process Integration

Reproducing the Issue

  1. Maintain user info under tab Assigned Users in Communication Component
  2. Transport Communication Component from source system to target system
  3. Check user info under tab Assigned Users in target system

Cause

This is as per the design of Integration Directory objects, for example:

  • Communication Channels do not carry the URL and login information post transport as described in SAP KBA 1595705 - PI Communication Channel parameters removed after transport.
  • Communication Components do not carry the user information after transporting from source system to target system.

Resolution

Manually update Assigned Users with required information after successful transport.

SAP KBA 1595705 - PI Communication Channel parameters removed after transport

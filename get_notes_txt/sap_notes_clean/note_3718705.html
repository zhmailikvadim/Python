Symptom

  • Transports without retrofit data are observed in the retrofit tool
  • The 'Retrofit Status' under the Transport Management block displays a warning message stating "Transport is without retrofit data"
  • Release Transport Request action shows a red stop sign with no application log
  • SLG1 logs indicate comparison errors with messages such as "object list length changed"

Environment

  • SAP Solution Manager 7.2 
  • Focused Build 7.2

Cause

  • BAdI /salm/retro_release is activated, but the Auto_BW scenario is not configured or enabled

Resolution

  1. Verify that the BAdI /salm/retro_release is active for the Auto_BW scenario.
  2. Navigate to Focused Build SPRO > Retrofit > Activate Retrofit Automation Scenarios.
  3. Add the Auto_BW scenario and activate it.
  4. For transports showing no retrofit data in the retrofit window, create retrofit data using one of the following methods:

a. From the Retrofit window, select "Additional Functions" and create retrofit data.

b. Alternatively, execute the report RETRO_SERVICE_CREATE_RFIT_DATA to generate retrofit data for the affected transports.

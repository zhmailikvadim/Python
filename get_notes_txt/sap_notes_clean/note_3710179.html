Symptom

Canvas Advanced Reporting displays workflow data for some records but not for others, even when the workflows exist. This behavior is inconsistent and cannot be attributed to workflow type, workflow status, or user permissions.

Environment

SAP SuccessFactors HCM Suite

Reproducing the Issue

  1. Open any Canvas Advanced Report that includes workflow‑related tables
  2. Preview the report and observe that some records do not return workflow data, even though workflows exist and are visible in the UI.
  3. Identify other records with similar configuration or workflow status and notice that workflow data appears for those records.
  4. Compare the two sets of records and verify that the discrepancy is not related to workflow type, workflow status, permissions, or employee configuration.

Cause

This issue is a product limitation related to the metadata mapping of workflow tables used by Canvas Advanced Reporting. Canvas Advanced Reporting retrieves workflow data by joining GO_WF_REQUEST.PENDING_ROW_ID with ROW_ID. The limitation arises because PENDING_ROW_ID only contains a valid value while a workflow is still pending. Once a workflow progresses (updated, moved forward, approved, rejected, or completed), the PENDING_ROW_ID no longer matches the corresponding ROW_ID in the related object table. When this mapping no longer aligns the join fails and the workflow data is not returned in Canvas Advanced Reporting, even though the workflow still exists and is visible in the UI. Because the behavior depends only on whether a valid PENDING_ROW_ID match still exists, and not on workflow type, workflow status, or permissions, the results appear inconsistent.

Resolution

This is working as designed and is classified as a framework limitation rather than a defect.

Workaround: To ensure consistent and complete workflow reporting, we recommend using Story Reports. Story Reports use a more modern reporting framework and enhanced data modeling, which allows them to retrieve workflow information more reliably.

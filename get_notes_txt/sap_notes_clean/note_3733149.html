Symptom

  • Error message FB002 ("file could not be opened") occurs when running the program RFEBKA00 (transaction FF_5) in a background job, if the specified file is not present in the application server path.
  • The entire background job terminates if one of the expected files is missing.
  • The intended behavior is for the job step to complete successfully without an error if no file is found.

Environment

  • SAP ERP Central Component
  • SAP ERP

Reproducing the Issue

  1. Schedule RFEBKA00 (transaction FF_5) in background with a variant that specifies a statement file path on the application server.
  2. Ensure that no file exists at the specified path at runtime (or select “Workstation upload” without providing a file).
  3. Start the job and observe that the run terminates with error FB002 “file could not be opened”.

Cause

The program behavior is standard by design: RFEBKA00 processes a single file per run and terminates with error FB002 if the specified file is not found.

Resolution

  1. Acknowledge the standard behavior: RFEBKA00 will end with error FB002 when the specified file is missing.
  2. If your release supports it (ECC 606 or higher), use the multi-file import report FEB_FILE_HANDLING to read files from an application server directory and process them in a loop (refer to the notes in “See also”).

See Also

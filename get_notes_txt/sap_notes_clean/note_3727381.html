Symptom

Blank addresses are being created in the SAP S/4HANA system when users save an address in the CRM UI without providing mandatory details such as house number, street, postal code, or city.  
If users navigate to another tab from the address tab to any recent object, the address gets saved with blank details .

Environment

  • SAP S/4HANA
  • SAP Customer Relationship Management

Reproducing the Issue

  1. Open any BP.
  2. Navigate to address.
  3. Click on "NEW".
  4. Provide only some or one of the mandatory field like Country.
  5. Navigate to another tab from the address tab to any recent object for example a previously opened BP.
  6. Go back to the initial BP. 
  7. The address gets saved with blank details .

Cause

The issue is due to standard system behavior.

Resolution

If the 'Postal Code Required' flag is enabled, a postal code must be entered for a street address. However, this validation is triggered only when at least one street address field is maintained.
The mandatory check is executed when any of the following fields is filled: 

  • Street
  • House Number
  • Street 2 / 3 / 4 / 5
  • Other City

If none of these fields are entered, the system does not trigger the postal code mandatory validation.

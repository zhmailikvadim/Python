Symptom

  • Setting up communication scenario SAP_COM_0647 (Expose Launchpad Content to SAP BTP | SAP Help Portal).
  • Checking logs using Display Launchpad Content Exposure Logs | SAP Help Portal 
  • Error message: "RFC destination with host URL 'portal-service.cfapps.<datacenter>.<domain>' not in table /UI2/CDM3_CCNTGT"

Environment

SAP S/4HANA Cloud Public Edition

Reproducing the Issue

  1. Attempt to exchange content from SAP S/4HANA Cloud to SAP Build Work Zone.
  2. Check logs for errors using Display Launchpad Content Exposure Logs app.
  3. Observe the error message: "RFC destination with host URL 'portal-service.cfapps.<datacenter>.<domain>' not in table /UI2/CDM3_CCNTGT".

Cause

The host name entered in the communication system under the arrangement for communication system SAP_COM_0647 is incorrect.

Resolution

Update the host name in the communication system under the arrangement SAP_COM_0647_LPD_EXPOSURE to portal-service.cfapps.<datacenter>.<domain>.

           

As per Create a Communication System with Inbound and Outbound Users | SAP Help Portal

Procedure > Step 4

In the Host Name field, enter the callback URL, such as portal-service.cfapps.<datacenter>.<domain>. The URL is needed to enable the content change notification. See Set Up Automatic Updates of a Content Provider - SAP S/4HANA.

If you don't want to enable the content change notification, enter a host name of your choice as this is a required field. In this case, however, you will have to trigger the content synchronization manually on the content provider side on SAP BTP.

Symptom

  • The SYSTEMDB chapter is missing in the EarlyWatch Alert report for a HANA database.
  • The EWA report contains data from additional HANA databases if database connections to those remote systems are defined in transaction DBCO.
  • When executing function module /SDF/COLL_MULTI_HANA_DWL with MOD_ID1 = M_DATABASE, only a single HANA_ID entry for the default tenant database is shown.
  • No entry is displayed for the SYSTEMDB connection, even though the connection exists and works correctly.

Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental

Environment

  • SAP Solution Manager 

Reproducing the Issue

  • Verify that additional database connections to remote systems are maintained in transaction DBCO.
  • Execute transaction SE37.
  • Run function module /SDF/COLL_MULTI_HANA_DWL.
  • Enter M_DATABASE in field MOD_ID1.
  • Observe that only one HANA_ID entry is returned for the default tenant database and that no entry exists for the SYSTEMDB connection.

Cause

Resolution

For IBP systems, a secondary HANA database connection is not evaluated. Any non-default HANA DB connection will not be checked if IBP components are installed.

Environment

SAP Solution Manager 7.2

Reproducing the Issue

  1. Access T-code ST22
  2. There will be a dump present with MESSAGE_TYPE_X

Cause

This dump occurs because the houskeeping job runs at the same time as DVM size extractor or multiple extractors are running at the same time.

Resolution

Because this dump occurs due to extractors running at the same time, the only way to avoid this error is to reschedule the extractors to run at different times and thus avoid this issue.

Please note: This Dump can be ignored.  The size extractor runs every 4 hours and downloads data once a week so if it fails, then, provided the DVM extractor is fetching data correctly, it should run correctly the next time.

How to reschedule extractor:
1. Go to the Extractor Framework UI
2. Select Edit
3. Select the button "Expert Mode"
4. Select an extractor to change the frequency of
5. Select the tab "Extractor Detail" and select "Configuration"
6. Change the value in the field "Period". The value maintained in this field is the period in minutes between extractor runs so you can change this field to get the extractor to run again on short notice.
7. Select Save
It is very important that once the extractor runs after you have changed the value in field "Period" you must return the field to its initial value once more.

Symptom

I need to configure my sourcing integration project to bypass SAP Business Network for the India data center. This is required to meet data residency requirements and route documents directly to SAP Ariba Sourcing.

Environment

  • Managed Gateway for Spend&Network
  • SAP Ariba Sourcing (SRC)
  • SAP ERP Central Component (ECC)

Cause

To meet data residency requirements for the India data center, you need to enable direct integration between your SAP ERP system and SAP Ariba Sourcing, which bypasses SAP Business Network. This requires specific configuration in your SAP ERP system and on the Managed Gateway for Spend&Network portal.

Resolution

To enable direct integration follow these three configuration steps:

Step 1: Enable the Direct Integration Parameter in SPRO

  1. In your SAP ERP system, run the SPRO transaction.
  2. Navigate to Buyer > SAP Ariba Strategic Sourcing Suite Integration > General Settings > Set Up the Interface > Enable direct integration bypassing SAP Business Network.
  3. Set the parameter SRC_DIRECT to X.
  4. Save the configuration.

Step 2: Configure the RFC Destination for SAP Ariba Applications

Ensure your RFC destination points directly to the SAP Ariba Sourcing endpoint for the India data center, not the SAP Business Network endpoint.

  1. In the RFC Destination settings (transaction SM59), for your HTTP connection to the Managed Gateway, ensure the Path Prefix is set to /cxf/receiveERPIDOC.
  2. Set the Target Host for the India data center to:
    • integration.in.managedgateway.cloud.sap (for production)
    • test-integration.in.managedgateway.cloud.sap (for test)

Step 3: Set Cross-Reference Parameters in your Managed Gateway for Spend&Network Project

Within your integration project on the Managed Gateway for Spend&Network portal, you enable the specific parameter for the relevant document types.

  1. Log in to the Managed Gateway for Spend&Network portal and edit your sourcing integration project.
  2. Go to the Cross Reference step.
  3. For the document types QuoteRequest, PurchasingContractConfirmation, and QuoteMessageConfirmation, set the parameter Enable System ID in Sourcing Integration Bypassing SAP Business Network to X (or select the checkbox if available).

After you complete these configuration steps, the system routes your sourcing documents directly to SAP Ariba Sourcing in the India data center, successfully bypassing SAP Business Network.

Symptom

  • During the save step of role type creation for customer projects under professional services, the system displays a dialog with the options “Continue with the lock” or “Cancel lock.”
  • Selecting “Continue with the lock” confirms the existing locks and allows the configuration to be saved.
  • Selecting “Cancel lock” closes the dialog, and the configuration changes are not saved.

Environment

SAP S/4HANA Cloud Public Edition

Reproducing the Issue

  1. Perform role type creation for customer projects under professional services in the central business configuration environment.
  2. During the save step, observe the system displaying a dialog with the options “Continue with the lock” or “Cancel lock.”

Cause

This is expected behavior for transport-controlled business configuration.

Resolution

The system message indicates that Continue with the lock will save the configuration while keeping the existing lock active. This is expected behavior for transport-controlled business configuration. The option is safe from a technical save perspective, but it does not release the lock, and further changes to the same configuration object may remain restricted until the related transport is processed in the downstream system.

If the later completion belongs to the same locked configuration object as the Role Type currently being saved, you should not assume that it can still be edited before the current transport import. In that case, the follow-up adjustment would more likely need to be done in a subsequent Edit/change cycle after the lock is released (transport is completed).

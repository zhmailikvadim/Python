Symptom

  • The response header named "Server" is populated with the value "SAP" in all Cloud Integration iFlows.
  • This behavior occurs irrespective of the iFlow or endpoint type, whether it is a direct Cloud Integration URL or an API URL.
  • Concerns regarding risk of "Disclosing the server’s name can help an attacker understand the underlying technology stack and potentially exploit known vulnerabilities associated with that stack". 

Environment

  • Cloud Integration
  • Integration Suite

Reproducing the Issue

  1. Trigger any Cloud Integration iFlow using either a direct CPI URL or an API URL endpoint.
  2. Observe the response headers returned upon successful execution of the iFlow (200 response code).
  3. Confirm that the "Server" header is populated with the value "SAP".

Resolution

There is no security risk here. The endpoint host name reveals more about the server than this generic header with "SAP" value. SAP is a company name and it does not provide any indication of what kind of software solution is running behind.

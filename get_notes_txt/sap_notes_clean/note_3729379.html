Symptom

  • An error occurs when attempting to call Service Cloud V2 USER SCIM APIs using the documentation provided at https://api.sap.com/api/SalesSvcCloudV2_scimUserService/tryout.
  • Error message: { "schemas": ["urn:ietf:params:scim:api:messages:2.0:Error"], "status": "403", "detail": "User not authorized for either the resource access or the operation" }.
  • A technical user with admin access has been created, but the issue persists.

Environment

Product version: SAP Sales Cloud and SAP Service Cloud Version 2 1.0

Reproducing the Issue

  1. Attempt to call Service Cloud V2 USER SCIM APIs using the documentation provided at https://api.sap.com/api/SalesSvcCloudV2_scimUserService/tryout.
  2. Encounter a 403 Authorization Error with the message: "User not authorized for either the resource access or the operation."

Cause

The SCIM API can only be utilized by users with the user Type API_USER.

Resolution

  1. Log in to the system as an admin.
  2. Navigate to Settings > Users > Technical Users.
  3. Check that the user trying to access the SCIM API has the user type API_USER and has read and write access to the Business service ID sap.crm.md.service.employeeService.
  4. See the documentation at https://api.sap.com/api/SalesSvcCloudV2_scimUserService/path/getAllUsers for sample API calls.

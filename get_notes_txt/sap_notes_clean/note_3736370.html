Symptom

In "Manage Project Procurement" app, the field 'Net Value' is determined by the 'Net Price' value within a Purchase Order item, rather than being based on the 'Net Value' in multiple distribution account assignment.

Environment

SAP S/4HANA Cloud Public Edition

Reproducing the Issue

  1. In ME23n or "Manage Purchase Orders" app, you can observe multiple distribution under 'Account Assignment', with different 'Net Value' corresponding to different 'WBS Element'.
  2. However, in "Manage Project Procurement" app, for the same purchase order, the 'Net Value' is displayed based on the 'Net Price' value within the Purchase Order item, rather than reflecting multiple distribution-based values. 

Resolution

Use 'Net Order Value in Document Currency' column which reflects the net value based on account assignment distribution.
You can add this column from Setting icon.

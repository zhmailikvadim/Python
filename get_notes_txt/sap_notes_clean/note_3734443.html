Symptom

In the MPES process cluster, the reports /US4G/CL_CHK_MPES_PROCESS->CHECK_FEEDIN_SCENARIO and /US4G/CL_CHK_MPES_PROCESS->CHECK_TRANCHE_SIZE_FOR_OBS did not correctly recognize newly created contracts and incorrectly assessed the tranche, generating incorrect check results.

This issue is relevant for the market role: DSO.

Other Terms

MPES, CHECK_FEEDIN_SCENARIO, CHECK_TRANCHE_SIZE_FOR_OBS, tranche, PoD, open contracts, scenario 2, feed-in scenario

Reason and Prerequisites

The error occurred because in scenario 2, the system identified the valuation of the open contracts differently, resulting in incorrect identification of open contracts.

Solution

Implement the corrections of this SAP Note or import the relevant support package.

After implementing this SAP Note, the system correctly recognizes subsequent contracts for PoD with and without tranches and sets the test result correctly. You can verify the fix by running the process with a PoD that has an existing contract and checking that the check results are correct.

Symptom

New records are being added to Change Logs and to the AUSR table for several users, when importing a Crystal Report.

Environment

SAP Business One

Reproducing the Issue

Prerequisite:

Go to Administration → System Initialization → Authorizations → General Authorizations, mark one of the users, check for example the Banking under Subject column. Make sure it's not set to Various Authorizations. It can be Full Authorization.

Scenario:

1. Log in to SAP Business One with a superuser and go to Administration → Setup → General → Report and Layout Manager.

2. Press Import and Next and press on Browse to choose any Crystal Report. Under Content Type choose a report and choose the location in the three dots button - for example Banking Reports. Press on Next and Finish.

3. Check the user Change Log of one of the users who was not involve in the import (or check records in table AUSR).

Expected Behavior: 

No records added to Change Log (table AUSR).

Actual Behavior: 

New records were added to Change Log (table AUSR).

Resolution

This is correct system behavior.

When importing a new Crystal Report, table CDPM is getting updated for the related Father column.

After adding this permission of the report to the Authorizations window, due to the newly added record being No Authorization, the father's permission needs to be updated to Various Authorizations, it indicates that its child record permission has multiple types.

As a result, this will affect Authorizations change log of all relevant users. The column ALLOWENCES of OUSR table is updated and any changes to it will be added to the AUSR table for the relevant users.

So, for some users, when importing a report, it will cause the Father record to be updated, and the update operation will be recorded in the AUSR table.

For example: when importing a new report called 'wwc' under Banking Reports, the Father is Banking (39).

Since the Authorization for Banking changed to Various Authorizations after the import, this affected table AUSR for the relevant users.

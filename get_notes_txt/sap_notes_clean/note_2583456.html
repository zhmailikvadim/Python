Symptom

  • Create one WebI document contains more than 2 queries in the WebI, make sure at least two queries need more than 7 minutes to run
  • Error show after running the queries in 3-tier WebI rich client
  • Error message: An internal error occurred while calling 'processDPCommandsEx' API. (Error: ERR_WIS_30270)(WIS 30270)

Other Terms

three, 3, regression, upgrade, migrate, webi, webintelligence,

Reason and Prerequisites

Environment:

  • SAP BusinessObjects Business Intelligence Platform 4.2 (BI4.2)
  • Web Intelligence (WebI)

Steps To Reproduce:

  1. Create a WebI document based on any universe connecting to oracle database.
  2. Select an object as result object.
  3. Open query script viewer.
  4. Check Use custom query script.
    Input below SQL
    SELECT
      time_sleep(420)
    FROM
     DUAL
  5. Add a new query. Input below SQL
    Input below SQL
    SELECT
      time_sleep(420)
    FROM
     DUAL
  6. Add one more new query. Input below SQL
    Input below SQL
    SELECT
      time_sleep(240)
    FROM
     DUAL
  7. Save the WebI document.
  8. Open it in 3-tier WebI rich client (See KBA 1320280 about how to launch 3-tier WebI rich client).
  9. Refresh query.
    => "An internal error occurred while calling 'processDPCommandsEx' API. (Error: ERR_WIS_30270) (WIS 30270)” error shows.

Cause:

In 3-tier (Zabo) Mode, the Webi Rich Client establishes an HTTP connection between itself and the installed application server (ie Tomcat), which will contacts the connection server, on the server side in case of refresh. The connection server hosts all connections to databases, involved in the refreshing of documents.

A possible issue with a such architecture is to have to maintain http connections, in an opened state, which is a problem for windows. To avoid http timeout issue, we have implemented an asynchronous mode which avoid to keep the http connection opened all the time of the refresh. It is configure through a special setting in the “wsk_bridge.xml” file. In this case, WebI Rich Client, when you do a big refresh, will ask to the application server to keep the connection opened between itself and the connection server, as long as the refresh is not ended. WebI rich Client will ask every 3 minutes if the refresh is ended to the app server, and between each “ask”, will close the http connection which avoid http timeout issue.

Solution

  1. Backup and edit wsk_bridge.xml in

    <BO install dir>\SAP BusinessObjects\tomcat\work\Catalina\localhost\BOE\eclipse\plugins\webpath.AnalyticalReporting\web\WEB-INF\classes

    And in

    <BO install dir>\SAP BusinessObjects\tomcat\webapps\BOE\WEB-INF\eclipse\plugins\webpath.AnalyticalReporting\web\WEB-INF\classes

  2. Find string "<bridge name="HTTPRepositoryJob">".
  3. Find string "</bridge>", which corresponds end tag to the element found at step 2.
  4. Insert "<async_methods>" element like below.

    <bridge name="HTTPRepositoryJob">
            <impl type="java" class="com.sap.connectivity.cs.remoting.http.bridge.RepositoryJobImpl" />
            <impl type="COM"  class="BusinessObjects.CSBridge.HttpRepositoryJob.1" />
            <async_methods>
                    <async_method name="Call" enable="yes" />
            </async_methods>
    </bridge>

  5. Save the file and restart the application server.

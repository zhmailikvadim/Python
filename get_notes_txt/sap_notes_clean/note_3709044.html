Symptom

SAP UI5/Fiori applications and OData Gateway services show outdated data, metadata cache issues, or performance degradation due to accumulated client and metadata caches. Users report applications not reflecting recent changes despite deployments. Financial control products recommend daily cache cleanup scheduling.

Other Terms

UI5 cache invalidation; OData metadata cache cleanup; Fiori launchpad cache issues; Gateway $metadata cache staleness; Background job SM37 scheduling; Programs /UI5/APP_INDEX_CALCULATE, /UI2/INVALIDATE_CLIENT_CACHES. Financial Control, FCFPSL, FCCOREUI, Adjustment Hub, Approval Hub, Reporting App, Recon, Subtn, drilldown.

Reason and Prerequisites

Caching improves performance but causes inconsistencies after app updates, transport imports, or model changes. Daily scheduling prevents buildup. Prerequisites: SAP_UI 7.50+ or NetWeaver 7.40+, proper authorizations (SM37, SE38, S_DEVELOP), and execute in gateway/frontend systems. No parameters typically needed for these reports unless model-specific (e.g., /IWBEP/R_MGW_CLEANUP_MD_CACHE)

Solution

Schedule each program daily via SM37 after creating variants in SE38.

Step 1: Create Variant in SE38

  1. Run SE38, enter program (e.g., /UI5/APP_INDEX_CALCULATE).
  2. Click Execute (F8).
  3. Enter defaults if prompted (often none), then Save as variant (e.g., "DAILY_CLEANUP") via Goto > Variants > Save As Variant.

Step 2: Schedule Job in SM36

  1. Run SM36.
  2. Enter Job Name (e.g., "UI5_APP_INDEX_DAILY").
  3. Click Step, select ABAP Program, enter program name and variant.
  4. Set Server (auto) and Immediate print specs if needed.
  5. Save steps, then Start Condition: Choose Daily (e.g., 2:00 AM), select weekdays, save.

Step 3: Release and Monitor in SM37

  1. Run SM37, filter by job name/status Released.
  2. Select job, click Release (or auto if immediate).
  3. Refresh to monitor: Green=success, Red=failed. Check logs via Job > Log.

Symptom

Customers may observe that despite Additional Internal Hire Activities switch being enabled, Onchange rules configured with the rule context “Internal Hire” are not triggered during Crossboarding initiation. Even though similar behavior is expected based on the RCM–EC Internal Hire process.

Environment

SAP SuccessFactors Onboarding

Cause

  • This occurs because the Onchange Rule execution for Crossboarding functionality is controlled by an internal sysconfig.
  • If this sysconfig [not visible on the UI, it is a backend feature] is not enabled, the existing Internal Hire Onchange rules will not be executed during crossboarding initiation. 

Resolution

  • To ensure the expected behavior, customers must enable the “Onchange Rule for Crossboarding” functionality by raising a ticket with the component LOD-SF-OBX-XBD
  • Based on customer feedback requesting consistent behavior with the RCM–EC Internal Hire process, SAP will remove this toggle starting in the 2H 2026 release and enable the functionality by default.

Symptom

The limit which is displayed in transaction F9K3 is outdated and does not reflect the current valid limit

Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.  

Environment

  • FSCM In-House Cash
  • SAP ERP
  • SAP S/4HANA
  • SAP S/4HANA Finance

Reproducing the Issue

  1. Navigate to transaction F9K3
  2. Select correspond bank area
  3. Access the corresponding account number
  4. Observe the internal account limits displayed in the main page
  5. Note that the limits displayed are outdated and do not reflect the current valid limits

Cause

The logic for displaying the limit values in transaction F9K3 is based on the bank area posting date

Resolution

  1. Verify the current posting date for the bank area in T-code F9B1   ( table BKK01D )
  2. If the current posting date is earlier than the validity date of the updated limit, the system will display the previous limit as per the validity
  3. Update the posting date to a date after the validity date of the new limit
  4. Confirm that the updated limit is displayed in transaction F9K3

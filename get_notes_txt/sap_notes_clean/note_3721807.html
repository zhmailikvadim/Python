Symptom

A connection to a directory server need to be established via LDAP Connector but error message "LDAP Server is not active" occurs.
This can be happen via test function "Log On" in T-Code LDAP or in any other application which tries to connect to a directory server via LDAP protocol.
The LDAP message id is LDAPRC081 for that.

Environment

ABAP based systems

Reproducing the Issue

Connect to a directory server by using an incorrect hostname or port or in a landscape where a middleware is used which blocks the connection between ABAP system and directory server.

Cause

   - incorrect hostname
   - incorrect port
   - middleware like firewall is blocking the connection

Resolution

Ensure that the correct host name and port will be used in the LDAP Server configuration on ABAP system.
In addition to that it need to be ensured that the connection between ABAP system and directory server will not be blocked.

Additional infos for checking the issue:

1) If the successor of the LDAP Connector (which is the ICM LDAP Plug-In) is available please check if the connection can be made by using the ICM LDAP Plug-In. 
    Please see SAP Note 2785547 and KBA 3105079 regarding details about the ICM LDAP Plug-In.

    Once activated for the affected LDAP Server the corresponding "Connection Test" button in the LDAP Server definition can be used to perform the connection test.  
   
    In case the connection test fails you will get appropriate ICM related error message details which could be helpful to determine the issue better.

    Info: Whenever the ICM LDAP Plug-In is available it is highly recommended to use it instead of the LDAP Connector.

2) When using a LDAP Connector for the connection:

    a) If the LDAP Connector will be started via T-Code LDAP, then 
        --> Check and confirm via OS level command (on the server where the LDAP Connector runs) if the directory server can be reached successfully. Important: Use the hostname and port for that which you also entered in the LDAP Server configuration.
    b) If the LDAP Connector will be started and used on a dedicated windows host according to SAP Note 2428608 --> scenario 3. then
        --> Check and confirm via OS level command (on the dedicated windows host where the LDAP Connector will be started manually according to SAP Note 2428608 --> scenario 3.) if the directory server can be reached successfully.
              Important: Use the hostname and port for that which you also entered in the LDAP Server configuration.

3) If possible and if a third party LDAP client is available it can also be checked if the connection can be made successfully with this third party LDAP Client with the same host name and client from the server where also the LDAP Connector runs.

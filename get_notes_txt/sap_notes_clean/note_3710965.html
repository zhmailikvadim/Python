Symptom

While replicating plants from S4H to C4C, the error "Code List Mapping For Value XYZ Of Data Type Country Code Missing" is raised in the C4C Webservice message monitoring.

Here XYZ represents the ISO country code sent from S4.  

Environment

  • SAP Cloud for Customer
  • SAP S/4 Hana

Reproducing the Issue

  1. Go to Administrator Work center.
  2. Go to Webservice Message Monitoring.
  3. Select the failed message with relevant service interface.
  4. Navigate to error log , check the error message "Code List Mapping For Value XYZ Of Data Type Country Code Missing".
  5. When you check the payload , you can see the value is sent under CountryCode node which refers to XYZ in the above message.

Cause

The S4 Hana webservice PlantMasterDataReplicationBundleRequest_Out by design uses for country code in the plant address the ISO country alpha-3 code (char 3). Any mapping of sent data should be implemented on the receiver side.

Resolution

In the C4C tenant:

  1.  Go to the Business Configuration work center and click on Overview.
  2.  Search the configuration element Code List Mapping for Integration with External Applications and Solutions.
  3.  Click on Maintain Code List Mapping.
  4.  Under Code List Mapping Definition select CountryCode for Local Data Type Name.
  5.  Under Code List Mapping Rule select Mapping Rule = Local And Remote Codes Are Equal.
  6.  Change the rule to Map Individual Codes.
  7.  Now it is possible to add/edit now the existing mappings for the 3 digit ISO country codes to the C4C 2 digit countries code.

Symptom

For “actual” distribution or assessment allocation cycles: To enable WBS elements or orders as senders in allocation cycles, you can use the report RKALPSAC. However, even after activating the senders with this report, the sender entries may still not appear.

For “plan” distribution or assessment allocation cycles: To make WBS elements available as senders or receivers. In cycle maintenance for planned distribution, planned assessment, periodic reposting in planning, or indirect activity allocation in planning, orders and WBS elements are not available as sender or receiver options.

Environment

Controlling (CO)
SAP R/3
SAP ERP Central Component
SAP ERP
SAP enhancement package for SAP ERP
SAP enhancement package for SAP ERP, version for SAP HANA
SAP S/4HANA
SAP S/4HANA Finance

Cause

For "actual" distribution allocation cycle or "actual" assessment allocation cycle : The more detailed field control settings in transactions KCAU and KCAV block the settings that report RKALPSAC made. 

For “plan” distribution or assessment allocation cycle : Orders and WBS elements are only offered as sender or receiver in the cycle/segment maintenance if the processing version and Cost Center Accounting have an integrated planning within the validity interval of the cycle.

Resolution

For “actual” distribution or assessment allocation cycles:

  • Use transaction KCAU (for assessments) or KCAV (for distributions).
  • Select and double-click the relevant WBS element or order.
  • In the next screen, click the button to restore default values.


For “plan” distribution or assessment allocation cycles:

  • Activate the “Integrated Planning for Cost Centers and Activity-Based Costing” indicator in the fiscal year–dependent version settings for the cycle validity period.
  • In transaction OKEQ: Ensure plan integration is active for Version 0.
  • Check that “Integrated planning with cost centers/business processes” is selected. Activate it if not already enabled.
  • In transaction OKKP: Verify whether Activity-Based Costing is active for the controlling area. Activate it if required.

Note: If plan data is available, you will not be able to change the indicator. So you  will either need to remove the complete plan data or to set the indicator in the version management starting from the next fiscal year (without plan data)

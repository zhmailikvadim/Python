Symptom

  • If you go to Home -> Groups -> select group you want to change- -> press on "add" -> search for LastName in lowercase -> you don't find any results.
  • If you search for LastName in upper case -> you find the user.
  • But on the other hand, the FirstName seems not to be case-sensitive.

Environment

Identity Authentication Service.

Reproducing the Issue

  1. Sign in to the administration console for SAP Cloud Identity Services.
  2. Choose the Groups tile.
  3. Choose a group from the list on the left.
  4. Press the Add button.
  5. Search for the users.

Cause

This search bar queries for firstName, lastName, loginName, email or resourceId and since in most common cases emails follow some pattern starting with the user's firstName and is also lowercased, the search finds results with a lowercased firstName but actually the match is the email.

As an example:




adding this user to a group:

- Search by 'testuser' works since the search matches part of email



- Search by 'Test-U' works since it matches part of First Name (case sensitive)

Resolution

Searches by both firstName and lastName are case sensitive.

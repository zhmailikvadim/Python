Symptom

  • Shortdump ITAB_DUPLICATE_KEY
  • When a work order contains multiple operations that are assigned to the same notification (via a work list). During the synchronization of work order from device, throwing short dump in SAP Backend.

Environment

SAP Work Manager 6.2.0

Reproducing the Issue

  1. Set up multiple operations on a work order that are linked to the same notification (via a work list).
  2. Synchronize work orders to client.
  3. Get the short dump.

Cause

Due to program error, the operation number on the some of the BAPI return tables may not be accurate i.e. the operation number on these particular return tables.

Resolution

Apply SAP Note "2133645 - Shortdump ITAB_DUPLICATE_KEY during Work Order fetch" released for the fix.

Keywords

ITAB_DUPLICATE_KEY, short dump, work order, fetch, operation number

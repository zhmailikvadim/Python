Symptom

An attribute search on a CDS based search model using fuzzy options is not correctly applied. There are two issues:

1) In the special case that the fuzzy treshhold is not provided when constructing a fuzzy attribute search, e.g. via API method CL_ESH_IF_SEARCH_REQUEST->ADD_ATTRIBUTE() - the fuzzy treshhold of 0 was applied - causing too many distinct hits.

2) In general, all attribute searches were applied as phrase searches (having several words, all were only (fuzzy) matched as phrase).

Other Terms

ESH, Enterprise Search

Solution

Apply the attached correction.

Symptom

  • sp_dboption hangs when deploying the sybsecurityarchive script.
  • The auditing subsystem is suspended, as indicated by the message "The auditing subsystem is suspended".

Environment

SAP Adaptive Server Enterprise 16.0

Reproducing the Issue

  1. Deploy the sybsecurityarchive script using the command: `isql -Usapsa -P -SPOD -X -w9999 -i sybsecurityarchive.sql`.
  2. Observe that the script hangs at the sp_dboption section.
  3. Attempt to manually execute sp_dboption for the sybsecurityarchive database.

Cause

  • The issue occurs due to the sysaudits_01 table being full, causing the auditing subsystem to suspend operations.

Resolution

  1. Verify the auditing subsystem status using the command: `sp_audit "status"`.
  2. Clear the sysaudits_01 table using the correct command: `sp_configure 'current audit table', 0, 'with truncate'`.
  3. Ensure the auditing subsystem is running by rechecking with `sp_audit "status"`.
  4. Re-deploy the sybsecurityarchive script using the original command: `isql -Usapsa -P -S<SID> -X -w9999 -i sybsecurityarchive.sql`.
  5. Confirm the script deployment is complete by verifying the following:
  • Execute `sp_helpthreshold` to check threshold configurations.
  • Execute `sp_help sap_audit_thresh_bck` to confirm the procedure creation.

      6. If the issue persists, ensure both sysaudits_01 and sysaudits_02 tables are empty and auditing thresholds are correctly configured.

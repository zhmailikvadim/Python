Symptom

  • ASE errorlog reports infected with signal 11 sequence
  • In the module GaGenericArray::_GaGetPtr

    Current process .. infected with signal 11 (SIGSEGV)
    Address .. (void**GaGenericArray::_GaGetPtr(int&,GaArrayBlock*&)
  • Followed by a stack trace which includes modules

    GaGenericArray::_GaGetPtr
    CgpPop*PopMergeJoin
    CgpPop*PopNlJoin
    CgpPop*PopAppendUnionAll
    CgpPop*PopEmit

Environment

SAP Adaptive Server Enterprise (ASE) 15.7

Cause

  • Instances reported have been against early releases of ASE 15.7 (ESD#1 - ESD#3) with no reports in current and supported releases. 
  • The issue has been identified and logged under SAP CR 713445:
    "A signal 11 at address GaGenericArray::_GaGetPtr can be seen from a select statement which performs a join multiple tables with a a derived table.".

Resolution

  • There is currently no target SAP ASE Service Pack scheduled for SAP CR 713445 . Subscribe to this KBA to be automatically notified when SAP CR 713445 is fixed.

  • It looks like CR 713445 is resolved in a higher 15.7 releases and does not affect more current releases (from ASE 15.7 SP50 & SP100 upwards).

Workaround

  • While there is no particular pattern identified that causes this stacktrace, minor changes to the query or execution methods can prevent it :
    • change an AND for a 'like' clause
    • reduce IN list
    • set merge_join off

See Also

SAP KBA 1904061 for how to check the current status of an ASE CR (Change Request).

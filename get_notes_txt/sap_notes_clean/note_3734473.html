Symptom

This article describes functionality introduced in the May 2026 SAP Fieldglass release

Some Purchase Order Line fields appear hidden, remain visible, or display differently for buyer users during itemized Statement of Work event creation or editing

Reproducing the Issue

This scenario may be encountered when:

  • A buyer user does not see an expected Purchase Order Line field during Statement of Work creation
  • A field is hidden by configuration and its visibility differs across Statement of Work steps or pages
  • A User Defined Segment field remains visible after visibility settings are updated
  • Field visibility changes immediately after dataset configuration changes
  • Labels, tooltips, or information icons appear updated compared to previous behavior

Cause

  • SAP Fieldglass provides buyer administrators with configuration options to control buyer‑side visibility of Purchase Order Line fields used in itemized Statement of Work events
  • On the Data Set Details page, the Hidden on Form Pages option allows selected fields to be suppressed on supported buyer‑facing Statement of Work pages
  • This configuration follows defined scope and limitations and does not apply to all fields or all Statement of Work pages

Resolution

To manage column visibility for buyer users in SAP Fieldglass:

  • Navigate to the Data Set Details page
  • Locate the Hidden on Form Pages column
  • Select the Purchase Order Line fields to hide from buyer users
  • Save the dataset to apply the changes immediately
  • Confirm the field is not part of a User Defined Segment, which cannot be hidden using this setting
  • Verify the Statement of Work page supports column suppression such as the Characteristics step or Edit Event page

Supplier visibility is not affected by buyer‑side column visibility configuration

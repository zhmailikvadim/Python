Symptom

Boolean fields used in Playbook Rules may show unexpected behavior during evaluation.

For example:

  • The condition is not fulfilled when the field appears as FALSE in the UI.
  • Conditions only work after the field is explicitly set.

Environment

  • SAP Sales Cloud Version 2
  • SAP Sales and Service Cloud Version 2

Reproducing the Issue

  1. Create an Opportunity
  2. Observe the boolean field in the UI.
    • The field appears as FALSE (toggle is deactivated).
  3. However, the field value is actually NULL (not maintained).
  4. The Playbook is not assigned to the Opportunity.

Cause

This behavior is expected and is related to how boolean fields and NULL values are handled.

  • When an Opportunity is created, boolean fields are initially NULL (not maintained).
  • NULL is treated as UNKNOWN, not as FALSE.
  • Therefore, expressions such as Field not equal TRUE do not evaluate to TRUE when the value is NULL.

Additionally:

  • Once the field is explicitly set (TRUE or FALSE), it is no longer NULL.
  • After being set, the field will only toggle between:
    • TRUE
    • FALSE

Resolution

When designing Playbook Rules, NULL values must be handled explicitly.

Recommended approaches:

1. Include a NULL condition when the field is not initially maintained:

Example:
To cover all non-TRUE scenarios:

  • Field = FALSE.
  • OR Field IS NULL = TRUE.

2. Alternatively:

  • Ensure the field is always maintained (e.g., Determination/Custom Logic) to avoid NULL value.

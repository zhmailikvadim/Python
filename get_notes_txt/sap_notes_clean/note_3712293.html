Symptom

 The user doesn't have the authorization to run transaction code VF11 to create cancellation invoice, but the cancellation invoice is created.

Environment

  • Sales And Distribution (SD)
  • Logistics Execution (LE)
  • SAP R/3
  • SAP R/3 Enterprise 4.7
  • SAP ERP Central Component
  • SAP ERP
  • SAP Enhancement package for SAP ERP
  • SAP Enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA
  • SAP S/4HANA Cloud Private Edition

Reproducing the Issue

  1. The user doesn't have the authorization to run VF11;
  2. The cancellation invoice can be found in the document flow.

Cause

The cancellation invoice was created using transaction VF01 instead of VF11, which can be verified in the accounting document created by the cancellation invoice : TCode = VF01.

Resolution

In standard,  when using transaction code VF01, adding the documents and going to "selection list", a cancellation billing type is required to be selected on the "Billing Type" field, as the system needs to know what document type such creation is performing from the selection list.  

If no "Billing Type"  is added in transaction code VF01, then go to "Selection List" button, this will allow user to cancel the selected item(s) and this is the standard system behavior for "Selection List".

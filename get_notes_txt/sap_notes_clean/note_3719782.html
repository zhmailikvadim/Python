Symptom

  • During a company database upgrade the Upgrade Wizard returns an error referring to SAP Note 2353878.

  • No database inconsistencies are found when running the query CALL CHECK_TABLE_CONSISTENCY('CHECK', NULL, NULL).

  • Some error messages similar to the below ones are recorded in the upgrade log files:

    Note I Tec Test result: Not completed due to SBOErr 2 in function CUpgradeCheckTableConsistency::DoCheck, error text: SQL statement 'ALTER SYSTEM ALTER configuration ('indexserver.ini', 'SYSTEM') SET ('table_consistency_check', 'enable_automatic_unl

    Err I Tec DBMCdataBase::ExecuteStatement SQLExecute failed. Error: -1, sqlState: S1000, nativeErr: 2, errText:[SAP AG][LIBODBCHDB DLL][HDBODBC] General error;2 general error: Setting changes failed on remote side. # #

Environment

SAP Business One, version for SAP HANA

Reproducing the Issue

  1. As per SAP Business One Administrator’s Guide, version for SAP HANA, install SAP Business One using another database user rather than SYSTEM.
  2. Execute the SAP Business One Upgrade Wizard to start the upgrade process.
  3. The upgrade process failed at the initial stage with the symptoms described above.

Cause

  • SAP Business One Upgrade Wizard was unable to change some settings in the SAP HANA server due to a lack of authorization/privilege for the database user.

Resolution

  • Ensure the required database user privileges (especially INIFILE ADMIN) are properly granted as documented on SAP Business One Administrator’s Guide, version for SAP HANA - chapter Database Privileges for Installing, Upgrading, and Using SAP Business One.

Keywords

database upgrade, upgrade wizard, setting change, upgrade issue, database privileges, upgrade failure.

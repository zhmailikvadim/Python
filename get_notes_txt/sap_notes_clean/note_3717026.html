Symptom

The note addresses the following issues and improvements:

  • Additional Change Volume Check for Impact Analysis
    • A new validation compares the daily change volume of a table with its total number of rows.This check is executed only for tables with more than 1 million changes per day and raises a message only if the number of changed rows exceeds 50% of the total row count.
  • Adjustment to “Export Table Statistics for Impact Analysis”
    • By default, the export now includes only ST10 trace periods without imports. Periods containing imports can still be displayed and exported by selecting the option “Only display periods without import”.
  • HANA only: Table size used in Impact analysis was the used memory size instead of the disk size which is relevant 
  • Fix for Short Dump in “ZDO Preliminary Checks”
    • A runtime error occurring in the “ZDO Preliminary Checks” related to the DB Trigger view has been corrected.

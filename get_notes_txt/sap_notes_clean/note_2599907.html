Symptom

  • Rendering issue in Portal when launching a transaction iView (ITS / WebGUI) using custom theme. Issues after Backend upgrade.

Environment

  • SAP Enterprise Portal 
  • SAP NetWeaver Application Server Java

Reproducing the Issue

  1. Logon to Portal 
  2. Navigate to a transaction iView(ITS /WebGUI)
  3. You will see rendering issues.

Resolution

  1. Test the results unchecking the checkbox for property "Handover Portal Stylesheet" or "Handover Stylesheet"

  2. There are cases when another application in Portal is launching the transaction
    E.g.: UWL item launching a transaction iView(webgui) - In this case check the "Handover Portal Stylesheet" for following iView in the portal content)
    "Portal Users/Standard Portal Users/Universal Worklist/UWL - Launch SAP Transaction" see also SAP note 1725729 - Transaction issue in SAP GUI for HTML (WebGUI) using UWL

  3. Many Custom themes are not supported for ITS/WebGUI, please check following SAP notes from ITS:
  • 314568 - SAP GUI for HTML functionality / Limitations / Sp. Behaviour
  • 2124106 - Themes for Unified Rendering based UI technologies (Web Dynpro ABAP/Java, BSP, SAPGUI for HTML, BI, JSF)

See Also

  •  See also SAP note 1446099 - Application (Web Dynpro or ITS) is not displayed in customer theme.

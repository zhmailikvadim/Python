Symptom

When executing an iFlow using the RFC receiver adapter in SAP Cloud Integration, the message processing fails with the following error:

Environment

  • SAP Integration suite
  • RFC Receiver adapter

Reproducing the Issue

1. Configure an iFlow using the RFC receiver adapter in SAP Cloud Integration. 
2. Execute the iFlow.
3. Observe the message processing failure with the error message indicating backend connection issues. 

Cause

This issue typically occurs due to incorrect configuration between Destination service, and Cloud Connector.

Possible causes include:

  1. The Virtual Host defined in the Destination configuration is not exposed in the SAP Cloud Connector.

  2. The virtual host or port defined in the Cloud Connector does not match the values configured in the destination.

  3. The backend server hostname or system details are incorrectly configured either in the destination or Cloud Connector.

Resolution

  1. Verify if the backend system is exposed in the Cloud Connector.

  2. Verify the Destination configuration.
    - Ensure the parameters match the Cloud Connector configuration
    - The virtual host defined in the Cloud Connector must match the host defined in the destination.

  3. Verify the connection from the Destination using the Check Connection option.
    - If configured correctly, the connection test will return a successful response : "Backend is reachable from the Cloud Connector."


    -  If not configured correctly, a failure error is returned: "Backend is not available in the list of defined system mappings in Cloud connector."

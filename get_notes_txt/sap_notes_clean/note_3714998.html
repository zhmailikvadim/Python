Symptom

Customers are unclear what integration fetures are available and what's not supported in their SAP Commerce Cloud Dynatrace environments. This KBA aims to provide guidance for Dynatrace Monitoring in SAP Commerce Cloud.

Environment

All production environments in SAP Commerce Cloud.

Resolution

Overview

In general, administrative access and customization of configurations within Dynatrace for SAP Commerce Cloud are not supported. SAP aims to enable access to a limited set of features for self-service where possible, but broader configuration changes and direct administrative privileges remain unsupported.

Supported as Self-Service

The following Dynatrace features are available for customers to configure and use directly within their SAP Commerce Cloud environments:

  • Mark Key Requests: Customers can mark key requests in Dynatrace for critical transaction monitoring. For detailed steps, refer to Key Requests.
  • Export Alerts & Metrics via API: Use the Dynatrace API with a personal access token (PAT) to export metrics and alerts for further processing or integration. For detailed steps, refer to Personal Access Token.
  • Create Alerting Profiles & Problem Notification Integrations: Set up alert profiles and integrate Dynatrace problem notifications with AIOps tools such as ServiceNow or PagerDuty. For detailed steps, refer to Alert Integration.
  • Create Dashboards: Explore hundreds of out-of-the-box metrics and pin them to dashboards to build customized system views and reports, or clone any one of the SAP provided standard dashboards and adjust to your needs.

Supported by SAP (Standard Support)

The following integrations and configurations are managed by SAP and require engagement with the SAP support team:

  • Third-Party Service Monitoring (Standalone): SAP Support can enable monitoring of third-party services as standalone service for separate performance baselining and enhanced anomaly detection.
  • Digital Experience Monitoring (DEM) Enablement/Configuration: DEM features (including Real User Monitoring and Synthetic Monitoring) can be enabled and configured by SAP, provided you have purchased the DEM add-on.
  • Monitoring External Components Integrated with SAP Commerce Cloud: Customers have the option to purchase a 3HU (Third Party Host Unit) addon, which enables full-stack monitoring of Commerce Cloud components hosted outside SAP. This addon is intended specifically for systems that are directly related to Commerce Cloud. Because customers have limited access, we do not recommend using this feature for systems that require frequent or complex configuration changes.

Not Supported

The following features are not currently supported in SAP Commerce Cloud’s Dynatrace integration.

  • Logging Feature: Access to logs is handled separately through the Cloud Logging Service (CLS) via the Commerce Cloud Portal. Dynatrace log integration is not available.
  • Admin Access to Dynatrace Tenant: Direct administrative access to the Dynatrace environment is not provided to customers.
  • Custom Alerting/Thresholds/Metrics/Request Attributes: Creating or adjusting custom rules or metrics beyond the standard configuration is not supported.
  • Integration with Customer’s Own Dynatrace Tenant: It is not possible to connect Commerce Cloud Dynatrace tenant to a separate customer-managed Dynatrace tenant. You can achieve a single pane of glass view by exporting data out of SAP managed Dynatrace and ingest into a remote Dynatrace tenant, however, please note that it requires development and hosting of a service outside of SAP Commerce Cloud.
  • Trace Export: Exporting traces from the SAP-managed Dynatrace instance is not currently available.

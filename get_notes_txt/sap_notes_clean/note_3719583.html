Symptom

You use SAP NetWeaver Process Integration and use the report RSXMB_CANCEL_MESSAGES to cancel the processing of messages with errors. You want to delete these messages without archiving them first.

Other Terms

RSXMB_CANCEL_MESSAGES, Archive, Delete, PERSIST_ARCH_MANUAL_CHANGES

Reason and Prerequisites

You use SAP NetWeaver Process Integration as a central integration server or local integration engine in an application system. You use the report RSXMB_CANCEL_MESSAGES to cancel the processing of messages with errors.

In non-productive system clients, you can prevent the archiving of messages that were terminated with report RSXMB_CANCEL_MESSAGES by assigning the value 0 to the configuration parameter ARCHIVE - PERSIST_ARCH_MANUAL_CHANGES. This change must be made in transaction SXMS_IECONF before executing the report.

There is no option to make this setting in the report RSXMB_CANCEL_MESSAGES.

Solution

Implement the attached correction instructions or import the specified support package.

After you implement the corrections, the selection criterion "Reorganization Action" is added to the selection screen of the report. You use this option to specify whether messages whose processing is canceled with this report are to be deleted or archived.

Remarks:

  • Canceling processing is a violation of the "Exactly Once" protocol, which must be documented in productive system clients by archiving. Therefore, this option is preset to "Archive" in productive system clients and cannot be changed. A client is considered productive if it has been assigned the client role "Productive" in transaction SCC4. In non-productive clients, you can select the "Delete" option. In this case, the affected messages are deleted without prior archiving.

  • The correction is delivered with replacement texts in English. The creation of the text element is therefore optional.

  • SAP NetWeaver 7.0 - 7.3 EHP1: For newer Support Packages, the German version of the text elements is already contained in the automatic correction instructions. If you want the output in a different language, translate the text elements in transaction SE63. If there is no translation for the relevant logon language, the replacement texts are output in English.

  • SAP NetWeaver 7.4 and higher: For newer support packages, the English version of the text elements is already contained in the automatic correction instructions. If you want the output in a different language, translate the text elements in transaction SE63. If there is no translation for the relevant logon language, the replacement texts are output in English.

  • For older Support Packages, the text elements are not contained in the automatic correction instructions. For these support packages, the replacement text is output in English. If you want the text elements to be output in a different language, create the text elements manually in the required language. The required steps are described in the manual correction instructions.

  • This correction updates the documentation object of the report RSXMB_CANCEL_MESSAGES.
    The update of the documentation object is contained in the automatic correction instructions only for higher support packages. The documentation is delivered in German using automatic correction instructions. If you want the output in a different language, translate it in transaction SE63. The translation is already contained in the support packages.

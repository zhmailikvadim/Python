Symptom

You declare your imports to the Danish customs system DMS from SAP GTS.

This SAP Note provides sample implementation of BAdI source code to fill the IDoc field "Version" (STDVRS). This field is required for the converter to be able to find the new mappings.
In addition, the IDoc field "EDI Message Type" (STDMES) is filled so that the old SKAT mappings for import can still be processed in the converter. This change is necessary so that an import message can be sent either to the old or to the new customs system in parallel.

This note contains implementation of the same coding as the previous note 3674422, offering you a possibility to use a sample coding provided in a GTS namespace.

Other Terms

/SAPSLL/IF_EX_CUS_ECC~MODIFY_IDOC_PARTNER_PROFILE,

Reason and Prerequisites

Program Error

Solution

Implement the correction instructions.

You can use the method /SAPSLL/IF_BADI_CUS_ECC_N~MODIFY_IDOC_PARTNER_PROFILE of class /SAPSLL/CL_EXM_IM_CUS_ECC for GTS 1100 or class  /SAPSLL/CL_BADI_ECC_CUS_N for all other versions.

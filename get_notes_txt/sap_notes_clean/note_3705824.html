Symptom

An ATP check for ABC Subitems with partial deliveries reduces the requested quantity to the already delivered amount and does a recheck for the remaining amounts.
This works fine if the delivery quantity is not zero in the delivery document.

The sales order item cannot be deleted, and the quantity cannot be reduced to the already delivered amount which is zero. (Item becomes incomplete)
ATP check does not run for the item in question if there is no change on the quantity and in case, we do change the quantity the ATP check runs for the added quantity.

Other Terms

ABC sub items, subitems, ATP, merge, lv_merge_case,

Reason and Prerequisites

missing functionality with ABC solution

Solution

The solution in this case intends to include the item in the merge functionality removing the requirements and running ATP check.
Depending on the ATP result. In case the material plant and the storage location are the same (Merge Case A) we do merge the result with the existing item.
In case the result has a different plant or a storage location we reject the item with the rejection reason 00 and create a new item.

Please implement the attached correction instruction.

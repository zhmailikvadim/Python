Symptom

  • When opening SAP HANA Web-based Development Workbench, the browser shows "403 - Forbidden".

  • When opening SDI Data Provisioning monitors, the browser shows "403 - Forbidden".

  • XS Engine is reachable, but access is denied.

Environment

SAP HANA Smart Data Integration

Reproducing the Issue

  1. Open SAP HANA Web-based Development Workbench under http://<host>:80<instance>/sap/hana/ide

  2. Observe the error "403 - Forbidden"

Cause

One (or more) of the following:

  1. The URL points to the incorrect database (SYSTEMDB instead of the relevant tenant DB). In an MDC system, both the monitors and the Workbench are normally used from the tenant DB. When trying to access then in SYSTEMDB context, XS can respond with 403 because the required content is not available there.

  2. The request is routed to SYSTEMDB because the tenant virtual host name (alias host name) is not configured correctly. Without this hostname mapping, the Web Dispatcher routes the request to SYSTEMDB by default.

  3. The user does not have the required roles:

    • Workbench related roles

    • Monitor related roles

Resolution

  1. Make sure the correct tenant DB is used in the URL (instead of the SYSTEMDB)

  2. Fix tenant host name mapping (so the request goes to the tenant DB, not SYSTEMDB)

    • Relevant SAP Help Portal article: Configure HTTP(S) Access to Tenant Databases via SAP HANA XS Classic

  3. Assign the required roles to the user

    • SAP HANA Web-based Development Workbench: 

      • sap.hana.ide.roles::Developer >> Contains all necessary permissions
      • Reference: SAP Help Portal SAP HANA Web-Based Development Workbench

    • Data Provisioning Monitors:

      • sap.hana.im.dp.monitor.roles::Monitoring >> Access the monitors 
      • sap.hana.im.dp.monitor.roles::Operations  >> Perform administration tasks

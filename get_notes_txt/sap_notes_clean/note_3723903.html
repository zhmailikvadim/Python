Symptom

In your current SAP BW system you use multiple infopackages for a datasource (starting with prefix 0TB) to load data from a transaction banking source system (software components FSAPPL, CB4AHANA or S4FSFND). The infopackages have non-overlapping selection criteria and have been set up to parallelly request and load the data by an application specific parallelization mechanism in the banking source system.

With the automatic conversion to BW4HANA the infopackages for the same datasource are converted into one single data transfer process (DTP) with extraction mode Delta.

Other Terms

BW 7.50, BW4HANA

In-place Conversion, Remote Conversion and Shell Conversion

Transactional Banking, 0TB*

SAP Note 3706178 - When to Use Full Load DTP, Delta, or Delta Initialization with ODP in BW, BW/4HANA

Solution

The following options are possible:

Option 1)

You use the converted DTP with extraction mode Delta.

Option 2)

You do not use the converted DTP, but manually create DTPs with extraction mode Delta. That means you create the same number of DTPs with extraction mode Delta and with the same non-overlapping selection criteria as the infopackages in the SAP BW system.

Option 3)

You do not use the converted DTP, but manually create DTPs with extraction mode Full. That means you create the same number of DTPs with extraction mode Full and with the same non-overlapping selection criteria as the infopackages in the SAP BW system.

The following implications must be taken into account:

Adjustment of Data in Banking Source System Parallelization of Data Transfer Initialization of DTPs History Loading
Option 1)

Control data for extraction in case of timestamp-based datasources.

 Not possible, as there is only one DTP.

The initialization of the delta transfer can be performed as follows:

  • Initialization without data: Implement SAP Note 3715485 in the banking source system and perform the instructions in the solution section to enable the datasources in the banking source system for this.
  • Initialization with data: It can only be performed for change pointer based datasources.

History loading can only be performed for change pointer based datasources. Implement SAP Note 3529938 in the banking source system and perform the instructions in the solution section to (temporarily) switch the extraction mode for a datasource in the banking source system. No change is required in the BW4HANA system.
Option 2) None. Same as before the conversion.
Option 3) No initialization is required due to Full DTPs.

Based on the implications option 3) is recommended.

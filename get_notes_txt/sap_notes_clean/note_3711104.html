Symptom

  • While creating instance during the selection of runtime environment (CF, Kyma, Neo) showing "You don't have permission to work in CF
  • The user is unable to view the subscription and service under the SAP Business Technology Platform (BTP) subaccount.
  • The issue occurs due to missing authorization as a Cloud Foundry (CF) Org Manager.

Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.

Environment

SAP Business Technology Platform 

Reproducing the Issue

  1. Attempt to subscribe to the Forms Service by Adobe free plan in a Cloud Foundry environment.
  2. Observe that the "Create" button is greyed out.
  3. Verify that the user is unable to view the subscription or service under the subaccount.

Cause

The user does not have the necessary authorization as a Cloud Foundry Org Manager, which prevents access to view instances in the SAP Business Technology Platform subaccount.


Resolution

  1. Verify User Role: Confirm that the user is not currently assigned the Cloud Foundry Org Manager role.
  2. Identify a Current Org Manager:
  • Contact the current Org Manager to assist in adding the user to the organization.
  • The Org Manager can add users without assigning any roles initially.
  1. Add User as Org Member:
  • The Org Manager should log in to the BTP Cockpit.
  • Navigate to the organization settings.
  • Add the user as an Org Member without assigning specific roles at this stage.

     I. Assign Org Manager Role (if necessary):

  • If the user needs to manage the organization, the Org Manager can assign the Cloud Foundry Org Manager role after the user has been added as an Org Member.

     II. Re-attempt Access:

  • Once the user has been added as an Org Member, they should be able to view the instances in the subaccount.

     2. Self-Service Method:

  • If the current Org Manager is unavailable and the user is a Subaccount Administrator.
    1. As Subaccount Administrator navigate to the BTP Cockpit, select the relevant Global Account, and open the Subaccount where the problematic Organization is located.
    2. Click on Services then Instances and Subscriptions.
    3. At the Environments section there is an entry called the same as the Organization's name and the Environment of this entry is Cloud Foundry Runtime.
    4. Click on the button ... and select Update
    5. Press Next >
    6. Use the following JSON format to provide the user details:

      1. For adding users with the default sap.ids origin use this format (no need to specify origin):

        {
            "usersToAdd": [
               {
               "email": "john.do@sap.com",
               "id": "I999999"
            }
          ]
        }

      2. For adding users with custom Idp origin use this format:

        {
            "usersToAdd": [
               {
               "email": "john.do@sap.com",
               "id": "I999999",
               "origin": "custom.idp.value"
            }
          ]
        }

      3. It is also possible to add multiple users with and without origin:

        {
            "usersToAdd": [
               {
               "email": "john.do@sap.com",
               "id": "I999999"
            },
            {
               "email": "john.do@sap.com",
               "id": "I999999",
               "origin": "custom.idp"
            }
          ]
        }

      4. To remove users as org admin just change the name of the json struct to “usersToRemove” like so:

        {
            "usersToRemove": [
               {
               "email": "john.do@sap.com",
               "id": "I999999"
            }
          ]
        }

    7. Press Update Instance to complete the process.

    The Update takes a few seconds to complete, and once done, the chosen user will become Org Manager.
    To complete the process, this configuration has to be removed, otherwise there will be errors like “User <Name> is already Org Manager”.
    To remove the configuration – follow the steps above, but replace the text from 6.) with “{}” and update the instance again.

    Notes:

    • Navigate to the Trust Configuration tab of SAP BTP to find the correct Origin Key for the "origin" field.
    • In the ID field, it is possible to use any P-User, S-User or I-User.

Symptom

  • When attempting to create a sales order using BAPI_SALESORDER_CREATEFROMDAT2, there is no field available to register the cost center (VBAK-KOSTL, VBAP-KOSTL).
  • The sales order creation BAPI (BAPI_SALESORDER_CREATEFROMDAT2, SD_SALESDOCUMENT_CREATE) does not support to maintain cost center in input data.
  • BAPI_SALESORDER_CHANGE contains input field BAPISDH1-KOSTL, but it is not mapped to sales order.

Environment

  • Sales and Distribution (SD)
  • SAP ERP
  • SAP ERP Central Component
  • SAP S/4HANA
  • SAP S/4HANA Cloud Private Edition

Reproducing the Issue

  1. Attempt to create a sales order using BAPI_SALESORDER_CREATEFROMDAT2.
  2. Check for fields corresponding to cost center registration (VBAK-KOSTL, VBAP-KOSTL).
  3. Observe that no such fields are available in the BAPI input structures.

Cause

  • It has not been designed to insert or change the cost center with SD BAPI as mentioned in SAP Note 499002.
  • The BAPIs provide the most important fields and processes, these were designed as a simplified tool for sales document processing, and do not include fields for cost center registration (VBAK-KOSTL, VBAP-KOSTL).

Resolution

  1. Refer to SAP Note 143580 for guidance on implementing an enhancement as workaround.
  2. Enhance the BAPI functionality by utilizing the EXTENSIONIN field to include additional fields.
  3. Add a field for cost center input in the EXTENSIONIN structure.
  4. Implement the logic to fill the USEREXIT_MOVE_FIELD_TO_VBAKKOM (MV45AFZB) with VBAK-KOSTL.
  5. Test the enhancement to ensure the cost center is processed correctly.

See Also

  • SAP Note 499002 - Cost center is not processed
  • SAP Note 143580 - Info SD BAPIs and customer enhancement concept
  • Regarding the cost center on sales order item level, refer to KBA 1739319 - Cost center in sales order item.

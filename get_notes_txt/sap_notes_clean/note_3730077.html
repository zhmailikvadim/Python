Symptom

This note deals with such situations, when the response time of a statistic record corresponds to the idle time of the user.

More specifically it is as follows. Let's assume the user finishes some actions, remains idle and starts new actions again after for example N seconds. When the time interval of one of the resulting statistic records (/nSTAD) corresponds to the idle time and its response time is about N seconds, then this is what is handled in this note.

There are no problem specific traces or syslogs.

Other Terms

SNC handshake

Reason and Prerequisites

Program error related to SNC handshake for a SNC-encrypted GUI communication.

If the GUI communication is SNC-encrypted, the SNC layer performs a handshake to refresh security information from time to time. An SNC handshake is triggered during the task handler is passing GUI output to the SNC layer for encryption. This leads to additional communication prior to sending the GUI output. Since the current statistic record is already marked as closed, the internal mechanisms of the task handler close and flush it during the early stage of the SNC handshake. Further stages of the SNC handshake are performed with an inconsistent statistic record, which is discarded at the end. When the user remains idle starting from this point of time, its next GUI action creates a new statistic record, whose response time is calculated from the start time of the discarded records, so that the response time is almost the same as the idle time.

Solution

There is no workaround. Apply a new kernel patch.

The correction is delivered with the kernel archives:

  • Kernel patch - dw.sar
  • SP stack kernel files SAPEXE.SAR and SAPEXEDB.SAR 

The corrections are contained in all patch levels that are equal to or greater than the patch level specified in the "Support Package Patches" section for the relevant kernel release.
If there is no entry there, the patch has not yet been delivered. The patch level will be entered automatically once the patch is available for download. You can subscribe to this SAP Note to be notified when it is updated.

Follow our recommendations for applying a kernel patch:

  1. Import the latest SP stack kernel if it already contains the correction. For a list of current SP Stack Kernels, see SAP Note 2083594 (SAP Kernel Versions and SAP Kernel Patch Levels).
  2. Apply the hotfix only if it is a serious error that is not yet corrected by the latest SP Stack Kernel. 
  3. Check the regression note for the desired patch level before applying the kernel patch. For detailed information, see SAP Note 1802333 (Finding information about regressions in the SAP kernel).
  4. For instructions on how to download and apply a kernel patch, see SAP Note 19466 (Downloading SAP kernel patches).

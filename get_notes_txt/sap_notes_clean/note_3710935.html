Symptom

When I remove or delete a comment in a Purchase Requisition (PR), the system automatically revalidates all line items.

Cause

The behavior occurs when the parameter Application.GuidedBuy.EnableValidationForCommentAttachmentChange is enabled.

When active, any change to comments or attachments in the PR triggers a full revalidation.

When the parameter is disabled, changing or removing comments or attachments does not trigger revalidation.

Resolution

You can check whether this behavior is expected based on the parameter configuration. Sign into your SAP Ariba Procurement site.

  1. Sign into your SAP Ariba Procurement site.
  2. Go to Manage > Core Administration.
  3. Click Parameters.
  4. Search for Application.GuidedBuy.EnableValidationForCommentAttachmentChange.
  5. If the parameter is enabled, the system revalidates the entire PR whenever you remove or delete a comment or attachment.

This parameter is delivered with the value set to “disabled” by default.

If needed, update the parameter value based on your company policy.

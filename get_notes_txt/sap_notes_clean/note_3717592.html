Symptom

ABAP Dictionary mass activation during release upgrades (usually phase ACT_UPG) takes very long. Several hours are taken between the "Final log" phase and the phase before.

Reason and Prerequisites

If there are dependent errors, the causing direct errors have to be found and marked. This calculation takes too long. The bad performance is due to missing indexes on large internal tables.

Solution

Apply this SAP Note or the related Support Package. If there are no activation errors (as expected when performing a productive upgrade), the phase is fast, even without the corrections of this SAP Note.

Attention: During the SPDD phase of ACT_UPG, transaction SNOTE can't be used to apply source code changes. The SAP Note has to be applied manually.

Eventually, this correction will be part of SUM (with a so called "fixbuffer").

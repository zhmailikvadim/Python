Symptom

  • Error 403 Forbidden with message "The request has been blocked by UCON" occurs when a custom Fiori app calls an HTTP service created in ADT.
  • The same apps and services work in development and test systems but fail in the production system.

Environment

SAP BTP ABAP Environment

Reproducing the Issue

  1. Sign in to the production system with a business user.
  2. Start the custom Fiori application that calls an ABAP HTTP service.
  3. Provide any valid input and start the process that triggers the HTTP call from the UI.
  4. Observe that the backend call to the ABAP HTTP service returns 403 Forbidden with "The request has been blocked by UCON".
  5. Repeat the same steps in development or test systems and observe that the call succeeds.

Cause

  • In DEV and TEST the http services are working for the Business Users as the services are generated into the Test Catalog SAP_CORE_BC_EXT_TST which is assigned to several Business Roles in DEV and TEST. For PROD this is not valid. 
  • The services need to be exposed properly, either via communication scenario (for communication users) or via IAM app (for business users).

Resolution

To assign a business user to a business role for your service, you need to create an IAM app, which can then be included into a business catalog, which, in turn, can be assigned to a business role. Please refer to the help documents below:

  • Define IAM app for business service
  • Consuming services in UI

Symptom

When releasing multiple Date Effective MBOMs from Windchill (for example, version 0001 – Valid From 01/27/2026, version 0002 – Valid From 03/24/2026, and version 0003 – Valid From 02/14/2026), the first two versions are processed successfully: the PBOM is released and the QBOM is updated. However, PBOM version 0003, which has a Valid From date earlier than version 0002, fails to release with the error “Change status of the BOM item is not valid.”

Other Terms

Planning BOM in PEO, Planning BOM Release, "Change status of the BOM item is not valid", Consolidated Planning BOM, BOM Save, Change Notice, Maintain Bill of Material.

Reason and Prerequisites

Program error.

Solution

Apply the attached correction instruction.

Symptom

Input any table in SE16N, after press Enter, there's CONVT_NO_NUMBER dump.

Environment

  • SAP R/3
  • SAP R/3 Enterprise 
  • SAP ERP Central Component
  • SAP ERP
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA Finance
  • SAP S/4HANA

Reproducing the Issue

In SE16N, input any table, there's dump.

Category               ABAP programming error
Runtime Errors         CONVT_NO_NUMBER
ABAP: Program          SAPLSE16N
Application Component  CO-OM

Short Text
"10,000,000,000" cannot be interpreted as a number

Cause

system standard behavior

Resolution

Please check below entry in table T811FLAGS.

MANDT

TAB

FIELD

VALMIN

100

SE16H

SELSIZE_MAX

10,000,000,000

The issue arises because the value in the field VALMIN, which is of type Character, is set as "10,000,000,000". Since this value is not a numeric type, the system generates the error CONVT_NO_NUMBER. To resolve this, please update the value on your side to ensure it matches the expected numeric format.

Keywords

SE16N, SE16, SE11, SE16H, table, dump, T811FLAGS

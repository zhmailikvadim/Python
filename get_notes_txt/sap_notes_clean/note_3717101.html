Environment

  • SAP S/4HANA, on-premise Release Independent
  • SAP S/4HANA Cloud Private Edition Release Independent 

Reproducing the Issue

  1. Run MRP Live with transaction MD01N
  2. Execute the planning run for materials
  3. Review the job log or short dump in transaction ST22
  4. Observe termination with runtime error AMDP_EXECUTION_FAILED in method CL_PPS4_MRP_EXTPROC=>EXT_PROC_SCHED_BACK_FRGDT, with database message invalid DATE, TIME or TIMESTAMP value

Cause

During scheduling, the HANA function add_days evaluates dates against the factory calendar to determine whether the resulting date is a working day. If the relevant calendar period or date context is not available, the calculation falls back to the initial date 00000000.

Subsequent MRP calculations subtract Planned delivery time (PLIFZ), Goods receipt processing time (WEBAZ), or safety time to derive the exact date. Any subtraction greater than zero from 00000000 results in an invalid date and triggers the dump (“invalid DATE, TIME or TIMESTAMP value”).

Resolution

To avoid the dump, ensure that all dates used in the scheduling calculation are valid and that the factory calendar can determine a working day.

  1. Identify the materials that caused the dump using the Material Identification option as explained in SAP Note 2668805.
  2. Correct invalid or initial values to ensure valid calendar dates.
  3. Save changes and rerun MRP Live.

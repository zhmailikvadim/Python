Symptom

  • Changes made in the Backoffice are marked as performed by anonymous user.
  • The logged-in user profile in the top-right corner appears as anonymous.

Environment

SAP Commerce Cloud

Cause

The issue may be caused by a custom implementation of the BackofficeAuthenticationProvider because OOTB implementation ensures only the users belonging to the Employee user group can sign in to the Backoffice Framework application. The default value of backoffice.gettingcurrentuser.annonymoususer.verification property is false and it is consistent with the default BackofficeAuthenticationProvider implementation.

Resolution

  1. Review the custom implementation of the BackofficeAuthenticationProvider.
  2. Consider setting the following property to enable anonymous user verification:

backoffice.gettingcurrentuser.annonymoususer.verification=true

See Also

Keywords

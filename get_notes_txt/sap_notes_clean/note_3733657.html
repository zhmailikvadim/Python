Symptom

Update Automatically field of Planned Working Time Replication Period changes from Yes to No after daily processing.

Environment

Time Data Replication

Reproducing the Issue

  1. Open Manage Data.
  2. Navigate to Planned Working Time Replication Period.
  3. Select the affected replication period record.
  4. Observe that the Update Automatically field has reverted from Yes to No.

Cause

  • As a safeguard, the system disables automatic updates when too many Data Replication Proxies accumulate for the user group assigned to the replication period.
  • If the number of proxies for that group exceeds 14 times the number of users in the group, the system switches Update Automatically to No.
  • Groups with 10 users or fewer are exempt from this automatic change.

Resolution

  1. Review admin alerts:
  2. Go to Admin Center > Admin Alerts and locate items for “Planned Working Time Replication Period.”
  3. Clear the proxy backlog:
  4. Replicate existing Data Replication Proxies to the Employee Central Payroll system.
  5. Resolve any replication errors to reduce the accumulated proxies below the safeguard threshold.
  6. Re-enable automatic updates:
  7. In Manage Data, open the affected Planned Working Time Replication Period and set Update Automatically back to Yes.

Symptom

  • Error message "/SCMTMS/SFIR243 - No change strategy defined for FSD &1; cannot execute change process" occurs when attempting to create a Freight Settlement Document (FSD) for additional charges within a Road Freight Order which is already settled.
  • The FSD is not created for the delta amount when additional charges are added to a fully invoiced Road Freight Order.

Environment

  • SAP Transportation Management embedded in S/4HANA 2020 and higher

Reproducing the Issue

  1. Open the FO 
  2. Click on Edit and navigate to the 'Charges' page.
  3. Add a charge line and enter a rate amount. Save the changes.
  4. In the top menu bar, select 'Charges/Settlement' and choose "Create Freight Settlement Document" from the drop-down list.
  5. On the "Create Freight Settlement Document" page, observe the error message "/SCMTMS/SFIR243 - No change strategy defined for FSD &1; cannot execute change process" in the bottom status bar. 

Cause

  • The error occurs because the existing Freight Settlement Documents (FSDs) do not have a Change Process Strategy maintained.
  • The system checks the original FSD change process strategy during the creation process of the new FSD, and ignores the new FSD change strategy setting in the class /SCMTMS/CL_SETT_CHNG_PRO_FSD inside the method /SCMTMS/IF_SETTLEMENT_CHNG_PRO~APPLY_CHANGE_STRATEGY.
  • This validation is part of standard system behavior. 

Resolution

Ensure that a Change Process Strategy is maintained for the Freight Settlement Document (FSD). Or as an alternative, use new examples going forward and close the existing Freight Orders created before the upgrade. You can cancel the existing FSDs and create new FSDs with the correct setup.

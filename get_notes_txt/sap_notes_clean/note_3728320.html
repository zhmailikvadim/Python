Symptom

When migrating a company database from a Microsoft SQL Server to an SAP HANA Server using the migration wizard, test No. 15 completes with a false positive error when the company uses a continuous inventory system (premigration test No. 15 checks for Incorrect Price in the OINM table, in non-continuous stock; for more information, see SAP Note 1588126). The migration process of that particular company database cannot continue.

Reproducing the issue

Prerequisite:

You are using SAP Business One, version for SAP HANA.

Scenario:

  1. Start the migration wizard.
  2. Select Test Only and the source MSSQL.
  3. Select an MSSQL company database which uses a continuous inventory system.
  4. Run the pre-migration tests.

Expected behavior:

Pre-migration test No. 15 does not detect any false positive errors. The migration process can continue.

Actual behavior:

Pre-migration test No. 15 detects false positive errors detecting transactions created in versions higher than 8.8x. The migration process cannot continue.

Solution

SAP intends to provide a patch or patches to solve the problem described. The section References below will list the relevant patches once they become available, and the Related Info content of relevant patches available in the SAP Support Portal will show this SAP Note. Be aware that these references can only be set at patch release date. SAP delivers patches only for selected releases at its own discretion, based on the business impact and the complexity of the implementation.

Workaround:

Use the migration wizard from SAP Business One x.x FPxxxx, version for SAP HANA, to migrate your company database to the SAP HANA server. Then, use the upgrade wizard from SAP Business One x.x FPxxyy, version for SAP HANA to upgrade your company database.

You may contact SAP Support, which can assist in migrating your company database directly with SAP Business One x.x FPxxyy, version for SAP HANA.

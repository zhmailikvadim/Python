Symptom

The custom Logic (custom determination/custom validation/external hook) will not be triggered until the save button is clicked on the Address pop-up screen.

Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.

Environment

  • SAP Sales Cloud Version 2
  • SAP Service Cloud Version 2 
  • SAP Enterprise Service Management 

Reproducing the Issue

  1. Go to Accounts 
  2. Open one account detailed view 
  3. Go to Addresses section 
  4. Click the pencil button for the existing address (Please note this pencil button is hidden by default. Please contact your admin to unhide this action)
  5. This Address popup screen shows up 
  6. While filling data in this screen, the customer logic is not triggered 
  7. After clicking the save button, the corresponding custom logic is triggered 

Resolution

Usually every single field update on the detail screen triggers a backend roundtrip. In scenarios involving integration with other systems like MDG this could trigger a large volume of messages.

To prevent this, the Address pop-up screen is provided. It ensures that only a single round-trip occurs when the user clicks Save. Therefore, the custom logic will only be triggered after clicking the save button.

Symptom

  • The bizLocation SGLN is not populated in outbound US‑DSCSA Item EPCIS messages for auto‑commissioned SSCCs.
  • Parameter SR_USHC_FILL_BIZLOC is active, but bizLocation remains empty.
  • Auto‑commissioning for SSCCs is enabled.
  • Implementing the referenced note did not change the outcome.
  • No error messages are displayed.

Note: To address this, we implemented SAP OSS Note 3348103, which is intended to resolve this behavior. However, the issue persists even after applying the note.

Environment

SAP Advanced Track and Trace for Pharmaceuticals

Reproducing the Issue

  1. Open the shipping event of the delivery.
  2. Select the option to run a rule.
  3. Choose rule type “SR_INFORM_SUCC” and start the rule.
  4. Check the generated US‑DSCSA Item EPCIS message: bizLocation SGLN is missing while readPoint is present.

Cause

The auto‑commissioning event is not stored (persisted) in the system by default; because the corresponding event is not available, the outbound EPCIS message does not include the bizLocation SGLN.

Resolution

  1. In General Customizing, activate event persistence for automatically created events by enabling:
    • AUTOCREATE_EVT_EXT
    • AUTOCREATE_EVT_INT
  2. Ensure SR_USHC_FILL_BIZLOC is active so that, when the original commissioning or aggregation lacks a business location, the system fills bizLocation with the readPoint GLN.
  3. If parameters were maintained after previous serialization, reproduce the scenario and create new serialization data; these settings do not retroactively apply to existing data.
  4. Maintain a valid readPoint GLN during commissioning to allow correct derivation of bizLocation when needed.
  5. Run the rule “SR_INFORM_SUCC” again and verify the US‑DSCSA Item EPCIS message now contains both readPoint and bizLocation SGLNs.

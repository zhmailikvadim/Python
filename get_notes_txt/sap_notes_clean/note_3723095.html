Symptom

Benefit types configured to carry forward balances are not adjusting automatically based on the specified "Balance Carry Forward Date"

Environment

SAP SuccessFactors Employee Central - Global Benefits

Reproducing the Issue

  1. Configure a Reimbursement claim benefit with Balance Carry Forward Setting set to Yes 
  2. In the Benefit Schedule Period, set the Balance Carry Forward date to a specified date. For example, for the Dec 2025 Schedule Period, set the Balance Carry Forward Date as 15th Jan 2026
  3. Navigate to the user's Benefits Overview page
  4. Observe that even after 15th Jan 2026, the balance is still being carry forwarded and therefore the entitlement amount displayed is incorrect (i.e., Balance carry forward from Dec along with the entitlement amount for Jan 2026)

Cause

This is due to a code defect.

Resolution

The issue has been fixed in the 1H 2026 release (JIRA# BEN-21801)

The logic to handle the "Carry Forward" from the schedule period has been updated to ensure that any leftover claimable amount from a period is not carried over if the claim date is past the balance carry forward date of that period.

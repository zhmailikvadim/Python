Symptom

Field defaults are running whenever screens are viewed during form load.  

Environment

  • SAP Sales Performance Management
  • Agent Performance Management

Reproducing the Issue

  1. Log in to the environment.
  2. Navigate to the "Producer Detail Search" page.
  3. Search for any Producer.
  4. Select the record.
  5. In the "Broker Detail" page, click on the computer icon (next to the "Producer ID" field) to view the "Producer Master" screen.
  6. Review the application logs and observe that the custom Field Default for "BrokerId" is running.

Cause

The Engineering team have been able to reproduce this issue locally and determined the root cause in the application code.

Additional context: The root cause is related to the Copy form action present on most forms. When a form with the copy action is loaded, there is logic to preemptively pull the data from the record and stage it to be copied if the action is selected. Most forms just grab the data present on the screen and use it as a base. However, other forms such as BrokerDFI and VendorDFI have code level configuration that prompts the field defaults to be loaded and the resulting values are staged in case the copy action is selected. This behavior is not exclusive to BrokerDFI and VendorDFI.

Resolution

  1. The Engineering team is investigating this behavior under reference code INS-38392 to determine if it is intended or not.
  2. A code correction has been opened to investigate and potentially rework the logic to prevent the field default expressions from running until the copy action is selected.
  3. Note that the field defaults that are executed in these scenarios will not change any data for the current record.
  4. As a workaround, add the "if(!is_process("INFPOS")) return" line to the top of the expression of your Field Default.  For further details, please refer to Field Defaults.

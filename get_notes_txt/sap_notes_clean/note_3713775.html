Symptom

An error occurs when attempting to archive Event-related data without archiving Object-related data first. The error message displayed is: "Event is not archivable. Not archived object exists."

Environment

SAP Corporate Serialization

Reproducing the Issue

  1. Attempt to archive Event-related data using the /CORS/EVT archiving object.
  2. Ensure Object-related data has not been archived using the /CORS/OBJ archiving object.
  3. Observe the error message: "Event is not archivable. Not archived object exists."

Resolution

  1. SAP does not recommend bypassing the standard archiving sequence & dependency checks using a custom approach, as it breaks SAP’s referential integrity model and carries audit and consistency risks.
  2. It is technically possible to archive Objects later, even if Events and Documents are archived first using a custom approach.
  3. Note that bypassing the standard sequence may lead to:
    • Archived Events/Documents becoming historically orphaned.
    • Object lifecycle history in the UI and reports becoming incomplete.
    • End-to-end traceability (Object -> Event -> Document) no longer being guaranteed.

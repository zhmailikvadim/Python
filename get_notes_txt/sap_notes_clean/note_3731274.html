Symptom

BR*Tools operations failing with ORA-01017 using SYSTEM with password

sqlplus system/<password> works fine.

Environment

  • Oracle any version
  • Not restricted to any OS platform

Reproducing the Issue

Run BRBACKUP or BRARCHIVE.

Cause

SYSTEM user lacks SYSOPER privilege.

Resolution

When testing:

sqlplus system/<password> as sysoper it also fails with ORA-01017, even though sqlplus system/<password> connects fine. This means SYSTEM lacks the SYSOPER privilege.

To fix it, run the following steps:

sqlplus / as sysdba
grant SYSOPER to SYSTEM;

Another option is to implement SSFS for BR*Tools, as per note 1764043.

See Also

SAP Note 1764043 - Support for secure storage in BR*Tools
SAP Note 480266 - Problems with SYSDBA/SYSOPER/INTERNAL connect

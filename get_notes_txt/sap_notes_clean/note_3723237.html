Symptom

The SDMI /ACCGO/CL_SDM_ACM_V_SDMNPEL was created in release S/4HANA 2020 and adds additional information to table /ACCGO/T_CAKITM. Once the additional data is provided, field SDM_VERSION is set to version 01.

New entries created in table /ACCGO/T_CAKITM after upgrade to release S/4HANA 2020 or later automatically contain the additional information in table /ACCGO/T_CAKITM and field SDM_VERSION set to version 01. However, certain processes don't set the SDM_VERSION to the correct value which can cause issues during the upgrade with a start release of S/4HANA 2020 or later.

Other Terms

ACM; Agricultural Contract Management; /ACCGO/CL_SDM_ACM_V_SDMNPEL; SDM; SDM version

Reason and Prerequisites

As part of the pre-upgrade checks for an upgrade from S/4HANA 2020 or later the following checks are executed:

  1. It is checked that /ACCGO/CL_SDM_ACM_V_SDMNPEL has been finished. This is the case for upgrades from S/4HANA 2020 or later.
  2. It is checked that table /ACCGO/T_CAKITM  does not contain any entries that would be picked up by a new run of SDMI /ACCGO/CL_SDM_ACM_V_SDMNPEL. Since the value in field SDM_VERSION is not always set to the correct value, this check may fail.

Solution

Implement SAP Note 3723013 and execute the utility provided by this SAP Note. This utility corrects the value in field SDM_VERSION and sets it to the correct value.

Symptom

Cannot import extension package in SAP Business One Extension Manager after you upgrade from SAP Business One 9.0, version for SAP HANA to SAP Business One 9.1, version for SAP HANA.

Reproducing the issue

Scenario:

  1. Upgrade company DB from SAP Business One 9.0, version for SAP HANA to SAP Business One 9.1, version for SAP HANA.
  2. Open SAP Business One Extension Manager and import your extension package.

Expected Result:

Upload extension package successfully.

Actual Result:

Cannot upload extension package. Error occurs: "System error; check log for details"

Note:

The issue cannot be reproduced if the company DB is created by SAP Business One 9.1, version for SAP HANA. 

Cause

Application error.

Solution

SAP intends to provide patches in order to solve described problem.

The section Reference to related Notes below will list the specific Patches when available.

The corresponding Info file of Patches on SAP Service Marketplace will show the correction / SAP Note number also.

Be aware that these references can only be set at Patch release date.

SAP will deliver Patches only for selected Releases at its own discretion, based on the business impact and the complexity of the implementation.

Workaround:

  1. Open SAP HANA studio and check if the table "EXTENSIONS" on Schema "SLDDATA" has any record.
  2. If no record, check whether the type of the "AUXILIARYPROPERTIES" column is "Text".
  3. If the type is "Text", run the following query string on the SQL console to drop and recreate table "EXTENSIONS", then check again if the type of the "AUXILIARYPROPERTIES" column is "CLOB".
  4. Once the type is "CLOB", reopen the extension manager and upload the extension package again.

DROP TABLE Extensions;

Symptom

Data Services job has a script that invokes SQL stored procedure to run a SQL job using the sql() function. When the job runs, it runs the stored procedure and the job on the SQL server successfully, but the following error is reported in the Data Services log:

DBS-070401 ODBC data source error message for operation <SQLExecute>: <[SAP BusinessObjects][ODBC 20101 driver][Microsoft SQL Server] Job '<name of SQ Job>' started successfully.>.

The error only occurs on the Linux box.

Other Terms

Environment:

  • Data Services: 4.2 SP14
  • OS: Linux Redhat/SuSE running the job server

Steps to Reproduce:

Create a Data Services job and add a script. Call the following in the script:

sql('sql server datastore name>', 'EXEC msdb.dbo.sp_start_job N\'syspolicy_purge_history\', @step_name =  N\'Erase Phantom System Health Records.\';');

Note that msdb.dbo.sp_start_job - is a default MS SQL server stored procedure.
In addition, syspolicy_purge_history is a default MS SQL job available under SQL Server Agent > Jobs

This issue is tracked in Corrective Measure 002075125800002837882020.

Reason and Prerequisites

The root cause of this error was a mis-processing of the error code from the underlying Data Direct driver for SQL server.

Solution

The correction is delivered in the releases listed in the Support Packages and Patches section of this SAP Note.

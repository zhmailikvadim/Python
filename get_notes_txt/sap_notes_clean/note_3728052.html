Symptom

You are using the Migration Workbench (MWB) with the MWB: Transfer Monitor (MWBMON). You're using the 'Block Processing Step' to prevent the data transfer from starting immediately after the access plan calculation finishes. However, when the access plan calculation completes, the 'Block Processing Step' flag resets to its initial state instead of maintaining your setting.

Note: This problem only occurs when the access plan is calculated in batch mode.

Other Terms

MWB, Migration Workbench, MWBMON, MWB: Transfer Monitor, Block Processing Step, Access Plan Calculation

Reason and Prerequisites

MWB functionality is used on one of the following releases:

SAP S/4HANA 2023 FPS00 - SPS05
SAP S/4HANA 2025 FPS00 - FPS01

Solution

 Implement the attached correction using transaction SNOTE.

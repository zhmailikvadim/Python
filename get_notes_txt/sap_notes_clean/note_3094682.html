Symptom

System copy of AS ABAP System failed in testDatabaseConnection phase - using 70SWPM10SP Software Provisioning Manager (SAP Note: 1680045).

The logfiles contain the error entries: (/<sapinst_instdir>/.../sapinst_dev.log.log and R3load.exe.log):

sapinst_dev.log:

...

ERROR <timestamp>  (<user>) (startInstallation) [CSiStepExecute.cpp:1104] id=controller.stepExecuted errno=FCO-00011 CSiStepExecute::execute()
The step testDatabaseConnection with step key |NW_Export|ind|ind|ind|ind|0|0|NW_ABAP_Export_Dialog|ind|ind|ind|ind|abapUnload|0|NW_ABAP_Export|ind|ind|ind|ind|2|0|testDatabaseConnection was executed with status ERROR (Last error reported by the step: Process call '/.../sapinst_instdir/NW7##/LM/COPY/ORA/EXP/CENTRAL/AS-ABAP/EXP/R3load -testconnect' exits with error code 12. For details see log file(s) R3load.exe.log).

Environment

All SAP system based on NW 7.0# (<NW 7.40)

70SWPM10SP - Software Provisioning Manager 1.0

Oracle 11g and higher

AnyDB

Cause

The most recent version of SWPM  contains a shipped LOADTOOLS.SAR (with 7.53 R3load), which will be extracted into the sapinst_instdir and used by SWPM Tool.

7.4 R3load (and higher like 7.53) requires configured SSFS for AnyDB - except DB2 (DB6)

SSFS is not set in AS ABAP SAP Systems with lower kernel than 7.40 as standard connection method, because it's mandatory only as of Kernel 7.40.

Resolution

Setup Secure connection of AS ABAP - 1639578  - SSFS as password store for primary database connect 

follow: the SAP Note 1622837

and use Oracle Client 12.1 for Oracle 10g/11g DB together with ABAP SSFS.

(ABAP SSFS requires the written SAP Kernel version as least, it's recommended to use always the latest supported one)

- The extracted R3Tools should be not changed in the sapinst_instdir, because the latest SWPM tool contains always the latest shipped R3Tools from LOADTOOLS.SAR  -

Symptom

Purchase Orders (POs) exist in both Test and Production environments, but no data is retrieved when I use POLineItemFact.

Reproducing the Issue

  1. Configure an Analytical Reporting API job using the view template with document type as POLineItemFact.
  2. Run the job in test or production.
  3. Check the job results and observe that no record is fetched.

Cause

POLineItemFact retrieves data from Spend Visibility, not from the operational transactional PO tables.
If POs are not modeled or available within Spend Visibility, the API will return zero records.

The transactional PO data is instead exposed through SSPPOLineItemFact.

Resolution

To successfully retrieve transactional Purchase Order line item data:

  1. Use the document type SSPPOLineItemFact instead of POLineItemFact.
  2. Submit the API job with the a view template for SSPPOLineItemFact document type.
  3. Retrieve the results using standard Analytical Reporting API polling steps.
  4. The API output will now include the expected PO line item records.

Explanation of data sources

  • SSPPOLineItemFact: Retrieves data from the transactional PO table.
  • POLineItemFact: Retrieves data from Spend Visibility.

Use SSPPOLineItemFact whenever transactional PO data is required.

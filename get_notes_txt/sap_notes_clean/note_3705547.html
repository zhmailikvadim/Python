Symptom

When copying a network from a standard network in CJ20N, a temporary CUOBJ number is incorrectly saved in table AFKO.

Steps to reproduce:

  1. There are two configurable standard networks : Standard Network 1 and Standard Network 2.

  2. Standard Network 1 contains a milestone with functions that trigger the creation of another subnetwork using Standard Network 2 as a template.

  3. In CJ20N, create a new operative network using the Copy Network function and select Standard Network 1 as the template.

    • All activities and milestones are copied.

    • This also triggers the creation of a subnetwork based on Standard Network 2.

  4. Save the network.

    • For Operative Network 1, the CUOBJ is saved correctly in AFKO.

    • However, for the subnetwork (Standard Network 2), the CUOBJ is saved with a temporary number (9999…1).

The subnetwork CUOBJ is saved with a temporary number, which is incorrect.

Other Terms

AFKO, CUOBJ, MLST,CAUFV, SAPLCNMS, LCNMSF28

Reason and Prerequisites

Program error.

Solution

Implement the note corrections.

Symptom

During SUM upgrade, phase SHDINST_DB_PREP fails with the following error in SHDUSRCRE.log:

BR1334E Owner SAPSR3SHD already exists in database

BR1336E Creating database owner SAPSR3SHD failed


When checking the previous retry attempt in the SHDUSRCRE.log or SHDUSRCRE.SAV file, one of the following errors is found:

ORA-01950: no privileges on tablespace 'SYSTEM'

ORA-01031: insufficient privileges

Environment

  • SAP Software Update Manager (SUM)
  • Oracle Database 12c or higher
  • ABAP Stack

Cause

The BR1336E error is a secondary symptom, not the root cause. The actual error sequence is:

  • BRCONNECT successfully creates the shadow user SAPSR3SHD.
  • BRCONNECT then tries to save the shadow user password in the SAPUSER table owned by OPS$<SID>ADM.
  • OPS$<SID>ADM does not have sufficient privileges on the SYSTEM tablespace, causing the INSERT/UPDATE to fail with ORA-01950 or ORA-01031.
  • Because the password step fails, SAPSR3SHD is left behind in the database.
  • When SUM retries the phase, it finds SAPSR3SHD already exists and reports BR1334E/BR1336E.

Resolution

1. Check the SHDUSRCRE.log for the actual error. Look for entries containing ORA-01950 or ORA-01031 before the BR1336E entry.



2. Grant privileges to the OPS$ user:

    sqlplus / as sysdba

    SQL> GRANT UNLIMITED TABLESPACE TO "OPS$<SID>ADM";

    SQL> GRANT RESOURCE TO "OPS$<SID>ADM";

    Replace <SID> with actual system SID (e.g., OPS$T22ADM).



3. Drop the leftover SAPSR3SHD user:

    SQL> DROP USER SAPSR3SHD CASCADE;



4. Verify the user is removed:

   SQL> SELECT username FROM dba_users WHERE username = 'SAPSR3SHD';

   Result Must return: no rows selected



5. Retry the failed SUM phase.

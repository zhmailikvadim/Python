Symptom

• After implementing the solution in Note 3254206 - F0770: Manage Automatic Payments: Automatic SEPA DD Pre-notification, using Option 1 to enable automatic pre-notification in F0770 via the classic framework, the payment run after the DD pre-notification run cannot be displayed in the "Manage Automatic Payments" app.

• The follow-on payment run is only visible in F110, it can't be displayed in the APP manage automatic payments.  

Environment

  • SAP S/4HANA 

Reproducing the Issue

 1. Implement the solution in Note 3254206.
 2. Use Option 1 to enable automatic pre-notification in F0770 via the classic framework.
 3. Run the payment process in the "Manage Automatic Payments" app and observe the result in F110 as below with 2 F110 runs

 First one : DD pre-notification run
 F110
 Direct Debit Prenotifications
 Parameters have been entered
 Payment proposal has been created
 Run for direct debit prenotifications executed.
 Posting orders: 1 generated, 1 completed
 Executed with payment run XX.XX.XX XXXXX

Second one :payment run 
F110
Payments for Direct Debit Prenotif. XX.XX.XX XXXXX
Payment run has been carried out
Posting orders: 1 generated, 1 completed

 4. Observe that the follow-on payment run (The second one) cannot be displayed in the app.
 5. Check F110 to confirm the follow-on payment run is visible there. 

Cause


  • The Fiori app lacks a one
  • to
  • one feature equivalent to SAPGUI.
  • The follow
  • on payment run after the pre
  • notification run cannot be displayed in the "Manage Automatic Payments" app due to system design limitations.

Resolution

 1.To locate the follow-on payment , check the F110 payment run ID in the log within the "Manage Automatic Payments" app. 

2.Display the payment run in F110 with the run date and ID in the log.

Symptom

You execute a service charge settlement in the settlement method 'SIM Simulation Only (Without Subsequent Actual Settlement)' and the settlement type 'MSU Only Settle Master Settlement Units'.

In the report 'Distribution of Master SU to Settlement Unit', the costs of the master settlement unit are not distributed correctly. The consumption for at least one settlement unit is missing. Therefore, the system does not determine the correct reference factor for this settlement unit.

Other Terms

SCS, MSU, RESCSE, RESCSECO, RESCSETN, measuring point, METERFROMRO, PURPOSE

Reason and Prerequisites

  • There are at least two master settlement units with the distribution type 'Meter' in the apportionment rule and a consumption-dependent measurement type.
  • In both master settlement units, the same settlement unit is assigned with the 'Consumption from RO' indicator set.
  • In the apportionment rule tab of the assigned settlement unit, an area-based measurement type is set. There is no consumption-depenedent measurement type set in this settlement unit.
  • SAP Note 3034198 is already implemented in your system.

Solution

The error is corrected in a Support Package.
For an advance correction, implement the attached corrections in your system.

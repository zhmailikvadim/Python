Symptom

When updating a Service Call using Data Interface (DI API), an error occurs: No matching records found (ODBC -2028).

Reproducing the issue

Scenario:

1. Using SAP Business One client, create a new Service Call.
2. Open the newly created Service Call, go to the Activities tab within the Service Call form, and click the Activity button to create a new activity. Update the Service Call after linking the Activity.
3. Using DI API, delete the Activity that is linked to the Service Call. Sample code:

SAPbobsCOM.ActivitiesService oActSrv;
SAPbobsCOM.Activity oAct;
SAPbobsCOM.ActivityParams oParams;
oActSrv = oCompany.GetCompanyService().GetBusinessService(SAPbobsCOM.ServiceTypes.ActivitiesService);
oAct = oActSrv.GetDataInterface(SAPbobsCOM.ActivitiesServiceDataInterfaces.asActivity);
SAPbobsCOM.ActivityParams param = oActSrv.GetDataInterface(SAPbobsCOM.ActivitiesServiceDataInterfaces.asActivityParams);
param.ActivityCode = 44;
oAct = oActSrv.GetActivity(param);
oActSrv.DeleteActivity(param);

4. Using DI API, attempt to update the Service Call. Sample code:

SAPbobsCOM.ServiceCalls oSCLs = (SAPbobsCOM.ServiceCalls)oCompany.GetBusinessObject(SAPbobsCOM.BoObjectTypes.oServiceCalls);
if (oSCLs.GetByKey(51))
{
oSCLs.Subject = "Test for DI API";
int USCLS = oSCLs.Update();
if (USCLS != 0)
{
string ErrorMessage = oCompany.GetLastErrorDescription();
MessageBox.Show(oCompany.GetLastErrorDescription());
}
else
{
MessageBox.Show("Updated!");
}
}

Expected behavior:

The Service Call is updated successfully.

Actual behavior:

An error occurs: No matching records found (ODBC -2028).

Solution

SAP intends to provide a feature package or patch to solve the symptom that is described in this SAP Note. The section References below lists the related feature package or patch once they become available. Be aware that References can only be confirmed at feature package or patch release date. SAP delivers feature packages or patches only for selected releases at its own discretion, based on the business impact and the complexity of implementation.

Workaround: 

  1. Open the affected Service Call in the SAP Business One client.
  2. Make any minor change (e.g., add a space in description field) and save the Service Call through the SAP Business One client.
  3. Update the Service Call via DI API again.

Symptom

How does the system evaluate edit rules when you act as another user?

Why are edit rules based on groups ignoring my group assignments when acting as another user?

Cause

When you act as another user, the system evaluates edit rules using the identity of the user you are acting as. It checks that user permissions and group to compare the edit rule conditions at the moment you edit the document.

Your own groups and permissions do not apply while you act as someone else.

Example Scenario

There is an edit rule configured to retrigger the approval flow when the user is part of the Purchasing Manager group.

The User 1, which is part of this group, then act as User 2, which is not under this group. The rule will not be triggered, as the User 2 will be taken in consideration for this scenario.

Resolution

When editing a document while acting as, always consider the user's roles and groups to determine the edit rule which will be triggered.

When determining the edit rule that was triggered in a document, check the History tab, referring to the User column as the user being checked for the edit rules.

Symptom

An attempt to add a new database to SAP Replication Server through rs_init utilitary fails with "Connection to '<dataserver>'.'<database>' already have a RepAgent"

Environment

  • SAP Replication Server 15.7
  • SAP Replication Server 16.0

Reproducing the Issue

  1. Set and start rep agent to database before adding database through rs_init

    1. sp_configure 'enable rep agent threads', 1

    2. sp_config_rep_agent <dbname1>, 'enable', '<repserver_name>', '<db_maint>', '<db_main_password>'

    3. sp_start_rep_agent <dbname1>

  2. Run rs_init utilitary to add <dbname1> to Replication Server

Cause

When adding a new database with rs_init, Rep Agent should not be previously set manually as this is done automatically by the utilitary.

Resolution

  • Stop Rep Agent in the affected database
    • sp_stop_rep_agent <dbname>
  • Disable Rep Agent for the affected database
    • sp_config_rep_agent <dbname>, 'disable'
  • Re-run rs_init to add the database

Keywords

RepAgent, rs_init, table level replication, database replication, secondary database, primary database

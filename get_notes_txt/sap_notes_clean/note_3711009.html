Symptom

When adding a new row in a document, if using right click → Row Details... function, the Item No. of the first row might be displayed as a wrong number. But after adding the document, the Item No. of the first row is still the correct one.

Reproducing the issue

Scenario:

  1. Create a marketing document, for example, an A/R invoice.
  2. Input the customer code and add a new row with Item No. A00001
  3. Put the cursor in Item No. field of the second row.
  4. Right click in header area and choose Row Details...
  5. In the opened Row Details... - A/R Invoice window, change the value of Item No. to A00002 and close this window.

Expected behavior:

A new row is added with Item No. A00002 and the Item No. of the first row is still A00001.

Actual behavior:

A new row is added with Item No. A00002 and the Item No. of the first row is also displayed as A00002. But after adding the A/R Invoice, the saved Item No. of the first row is A00001 which is correct.

Cause

System Limitation

Solution

This is a system limitation.

Workaround:
Right click in the second row directly and choose Row Details.... Then input the Item No. of this new row.

Symptom

  • When opening a new Work Zone instance, only the following messages are found: "Not Found"
  • When checking the network trace, the following trace can be seen: 
    • POST request to /dynamic_dest/JAM/v1/single_use_tokens completed with status 500 Error while retrieving destination JAM from destination service. HTTP status code: 404

Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.

Environment

  • SAP Build Work Zone, Advanced Edition
  • SAP SuccessFactors Work Zone

Reproducing the Issue

  1. Open Build Work Zone Advanced Edition.
  2. See the "Not Found" message on the Work Zone. 

Cause

There is an incomplete setup prior to setting up Work Zone.

Resolution

  1. Verify whether the configurator has been run as per the help guide: refer to Run the Configurator | SAP Help Portal.
    • To access the Configurator:
      • Access the SAP BTP cockpit, navigate to Services > Instances and Subscriptions, and click the Work Zone link in the Subscriptions table to access the application.
      • Alternatively, modify the Work Zone URL by adding 'dt' in the front part of the URL just before the 'workzone' or 'workzone-hr':
        • Example: https://ts-support.dt.workzone.cfapps.eu10.hana.ondemand.com/sites#Workzone-Config?sap-app-origin-hint=  
  2. Complete the Configurator steps to ensure that your DWS tenant is already created. You may refer to the guide above for the complete guide of how-to.

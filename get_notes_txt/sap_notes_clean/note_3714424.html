Symptom

The "FromDate" in the ODPV table for fixed asset is incorrect after canceling a Capitalization and posting it again with a different "Asset Value Date".

Reproducing the issue

Prerequisites:

  • Fixed Assets solution is enabled from Administration → System Initialization → Company Details → Basic Initialization → Enable Fixed Assets, and Calculate Depreciation By: Day is selected.

Scenario:

  1. Go to Administration → Setup → Financials → Fixed Assets → Depreciation Types, and define a Depreciation Type "X". In the General tab, select PR Temporis Type:Exact Daily Base for Depreciation Convention - Acquisition.
  2. Go to Financials → Fixed Assets → Asset Master Data, and add an asset FA001 with Depreciation Type "X" assigned.
  3. Go to Main Menu → Financials → Fixed Assets → Capitalization, and add a Capitalization for asset FA001 with Asset Value Date 01.01.2026.
  4. Go to Tools → Queries → Query Generator, and input table ODPV. The "Period Start Date" (FromDate field) for the first sub-period is 01.01.2026 for the asset.
  5. Cancel the Capitalization added in step 2.
  6. Add another Capitalization for asset FA001 with Asset Value Date 05.01.2026.
  7. Check the data in ODPV table again.

Actual Result:

The "Period Start Date" (FromDate field) for the first sub-period is still 01.01.2026.

Expected Result:

The "Period Start Date" (FromDate field) for the first sub-period is 05.01.2026.

Symptom

The configurable message "Insufficient authorization in corresponding sales order" (message types V/032 and V/048) cannot be configured successfully.

One or more of the following issues occur:

  • Application area V/ cannot be entered in the maintenance view for system messages.
  • The message type is not visible or selectable during configuration.
  • The message is not triggered during delivery processing, even though it is maintained.

Reason and Prerequisites

The configurable message is technically evaluated during delivery/shipping processing.

Only message types that are classified as delivery/shipping messages are considered by the system. Message types that are maintained solely for payment card processing (application area V/) do not fulfill the required selection criteria and are therefore ignored.

Prerequisites

To use configurable messages V/032 and V/048, the message types must be configured as delivery/shipping system messages. The message type must use message control version L.

Solution

Configure the message types as follows:

  • Maintain message types in table T160M
    • Use transaction SM30 to maintain table T160M.
    • Create or adjust the entries for message types V/032 and V/048.
    • Ensure the message types are classified as delivery/shipping messages.
    • Set the message control version L.
  • Maintain message characteristics in Customizing
    • Go to
      Logistics Execution → Shipping → System Modifications → Specify Characteristics for System Messages → Define the message types of system messages
    • Verify that the message types are active and have the required output behavior (for example, warning or error).
  • Verify configuration
    • After maintaining table T160M, the entries become available for further maintenance.
    • The configurable message is evaluated correctly during delivery processing once the prerequisite is fulfilled.

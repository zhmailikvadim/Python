Symptom

  • The MB5B transaction in SAP S/4HANA does not return materials without stock.
  • The message "No material contained in selection" is displayed in the selection screen.
  • In SAP ECC, the same transaction with identical inputs returns a list with 0 quantity for materials without stock.
  • In MB51, there are no material documents posted for the affected material.

Note: Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.

Environment

  • Materials Management (MM)
  • SAP S/4HANA

Reproducing the Issue

1. Execute the MB5B transaction in SAP S/4HANA with the following parameters:

  • Material
  • Plant
  • Selection date
  • 'Storage loc./batch stock'

2. Observe that the transaction does not return any results and displays the message "No material contained in selection" at the bottom.

3. Execute MB5B with the same selection criteria on an SAP ECC system, the list below is displayed:

4. You notice that in MB51 transaction, there are no material documents posted for that material & plant in the system.

Cause

There was a change of behavior between SAP ECC and SAP S/4HANA for new materials in the system. New materials that do not have any material documents posted in the system are not checked by MB5B transaction, thus, no result list is displayed.

This causes the system to not display a list for zero stock in MB5B, even though it is displayed on stock reports such as MB52 and MMBE.

Resolution

MB5B will display the results after the first goods movement is performed for the material. To work around this change, a dummy goods receipt for initial entry of stock (movement type 561) can be posted and then cancelled in the system to force. After this first goods movement, MB5B will display the results for zero stock for any period of time.

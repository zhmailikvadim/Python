Symptom

This article describes functionality introduced in the May 2026 SAP Fieldglass release

When working with Resource Activity Items in SAP Fieldglass, customers may notice one or more of the following situations:

  • After uploading an attachment, the Activity Item shows Pending Verification instead of Completed
  • A reviewer does not appear to receive a request to review an attachment
  • The reviewer field shows a role name instead of a specific person
  • The status shown in reports does not match the status shown in the user interface
  • The Final Verification Date is not visible
  • Multiple attachments were uploaded, but only one appears during review

Reproducing the Issue

These situations can occur when:

  • An attachment is submitted on a Resource Activity Item that requires review
  • Verification is enabled for the attachment Activity Item
  • Reviewer requests are sent to roles that include multiple users
  • Reports display standard Activity Item status values
  • The verification process is still in progress
  • More than one attachment was uploaded to the Activity Item

Cause

SAP Fieldglass includes an attachment verification workflow for Resource Activity Items of type Add Attachment and Add Attachment with Expiration Date

When this workflow is active:

  • Attachments must be reviewed and approved before the Activity Item is completed
  • One or more reviewers may be required based on how verification is set up
  • Reviewer roles may include multiple users, so the role name is displayed until someone completes the review
  • The user interface shows Pending Verification while reports continue to show Pending
  • The Final Verification Date appears only after all required reviews are completed
  • When multiple attachments exist, reviewers must navigate between them during the review process

This behavior is expected and helps ensure attachments are reviewed before completion

Resolution

Understand the Attachment Verification Process

  • After an attachment is uploaded, the Activity Item moves to Pending Verification
  • One or more reviewers must review and approve the attachment
  • Reviewers approve or reject attachments using the verification window
  • The Activity Item is completed only after all required reviewers approve

Understand Reviewer Display and Status Information

  • A reviewer role name may appear until a specific person completes the review
  • Status indicators show whether verification is completed or still in progress

Understand Status Differences Between the User Interface and Reports

  • Pending Verification in the user interface means a review is still required
  • Reports may continue to show Pending while verification is in progress
  • This difference is expected behavior

Final Verification Date and Multiple Attachments

  • The Final Verification Date appears only after all reviews are completed
  • When more than one attachment is uploaded, reviewers can move between attachments during review

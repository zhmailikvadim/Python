Reproducing the Issue

Running AFAB transaction or depreciation background job.

Cause

There is a program error: when an RFC communication failure occurs at the start of a parallel task, the count of active tasks is not decremented. Consequently, the system eventually reaches the maximum allowable number of tasks internally and cannot initiate new tasks, even when no tasks remain active.

Resolution

  • Apply the corrections delivered by SAP Note 3552361 - AFAB can enter an endless loop in case of RFC error.
  • If the issue persists after implementing the above note, please recreate the issue and provide ST12 traces by following KBA 2511272 - How to schedule ST12 trace for long-running background jobs in Kernel 721 and above. Then, open a case with SAP Support and attach the traces.

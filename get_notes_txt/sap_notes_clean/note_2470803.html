Symptom

For a Third Party Posting program RPURMP00 remittance rule, more than one HR accumulator has been created. The expectation is that after the first accumulator has triggered, any subsequent accumulators will continue to be available and are recognized as valid to be triggered. The actual result is that after the first accumulator triggers successfully, any subsequent accumulators are ignored.

Environment

  • HR renewal 
  • SAP S/4HANA 
  • SAP ERP  
  • SAP SuccessFactors Employee Central Payroll 

Reproducing the Issue

Use transaction code SE38 to create a Third-Party Remittance Posting run (program RPURMP00) that has two or more accumulators for a remittance rule. For example,

HR Accumulator 1:  Positive amounts $0 to $99,999.99   = remit semi-weekly

HR Accumulator 2:  Positive amounts $100,000 and over = remit next business day

If/when the first accumulator criteria is met and triggered successfully, any and all subsequent accumulators are ignored.

Cause

HR Accumulators are used in the system to use the Remittance Rule when the criteria of the accumulator is met. They were not intended to be used in multiple scenarios for the same rule.

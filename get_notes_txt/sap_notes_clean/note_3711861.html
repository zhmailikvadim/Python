Symptom

The supplier issued a XML without CBS/IBS conditions, however, the purchase orders created after Brazil Tax Reform go-live includes CBS/IBS conditions. This discrepancy leads to errors in the EDOC_COCKPIT when processing the eDocument, such as: 

  • Error J1B_NFE660: Expected IBS/CBS tax rate is X but tax rate in XML is Y
  • Error J1B_NFE661: IBS/CBS tax is not available in XML for purchase order item X
  • Error J1B_NFE663: Tax group IBS/CBS from XML is not available in the current Tax Code

**Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.

Environment

  • SAP ERP
  • SAP S/4HANA
  • SAP Document and Reporting Compliance, Inbound Nota Fiscal

Cause

The errors J1B_NFE660, J1B_NFE661, and J1B_NFE663 occur because there is a mismatch between the purchase order item and the XML items. This mismatch is caused by the absence of CBS/IBS conditions in the XML.

Resolution

To resolve issues where purchase orders are created with CBS/IBS conditions but the supplier's XML lacks these conditions, you can modify the following messages from error to warning using transactions OBA5 or OBMSG:

  1. J1B_NFE660: Expected IBS/CBS tax rate is X but the tax rate in XML is Y
  2. J1B_NFE661: IBS/CBS tax is not available in XML for purchase order item X
  3. J1B_NFE663: Tax group IBS/CBS from XML is not available in the current Tax Code

For future Purchase Orders where the supplier’s XML does not include CBS/IBS conditions:

Once condition records have validation dates configured in transaction code FV11:



The IBS/CBS will be calculated on purchase orders based on the "Document Date" and the configured "Valid From" date on FV11. 


To include purchase orders that will remain open after go-live, adjust the "valid from" date for the condition records accordingly.

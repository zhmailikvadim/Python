Symptom

After upgrading SAP BusinessObjects Business Intelligence Platform from 4.2 to 4.3, systems using an external Tomcat configured with a non‑SAP external JVM (JRE/JDK) experience:

  • “500 Internal Server Error” when accessing BI Launchpad or CMS
  • Tomcat failing to initialize SAP cryptographic providers

Environment

  • SAP BusinessObjects BI Platform 4.3
  • External Apache Tomcat 
  • External JVM 

Reproducing the Issue

  • upgrade to BI 4.3 from BI 4.2
  • Access CMC/BI Launchpad
  • Getting error: HTTP 500 – Internal Server Error

Cause

  • SAP BI 4.3 requires SAPJCE, SAP’s Java Cryptography Extension, for secure confidential channel establishment and various security operations.
  • External JVMs do not ship with sapjce.jar, because it is an SAP‑controlled cryptographic component.

Resolution

  1. Open Tomcat Service Configuration.
  2. Navigate to Java tab.
  3. Update Java Virtual Machine to the SAPJVM path above. 
    (<installdir>SAP BusinessObjects Enterprise XI 4.0\win64_x64\sapjvm\jre\bin\server\jvm.dll)
  4. Restart Tomcat and SIA.

Keywords

500, internal error, jvm

Symptom

The freight order CDS view does not populate the location ID when a stock transfer order is processed without an inbound delivery. The CMR form generated from the freight order lacks the ship-to address because the location id is missing.

Environment

 SAP S/4HANA Cloud Public Edition

Resolution

In S/4HANA Public Cloud, stock transfer processes without an inbound delivery require the Freight Order to capture accurate ship-to address information for the CMR (Consignment Note) form. Currently, when a stock transfer order (STO) is created without an inbound delivery, the Freight Order CDS view does not automatically populate the location ID, resulting in incomplete or missing ship-to address details on the CMR form. This gap impacts logistics documentation accuracy, carrier communication, and compliance with transportation regulations.

Current System Behavior: 

When a stock transfer order is integrated with Transportation Management (TM), a freight order is generated. However, if no inbound delivery exists, the system does not derive the receiving location ID from the STO. As a default behavior, the Freight Order CDS view relies on inbound delivery data to populate ship-to addresses. Without an inbound delivery, the location ID remains blank. 

When we look at the data flow, STO is created with receiving plant and storage location.  TM integration creates a freight order, but without an inbound delivery, the receiving location ID is not automatically populated. The CMR form queries the Freight Order CDS view for ship-to address, which is missing.  The system expects the location ID to be derived from the inbound delivery's ship-to party or receiving point, which is absent in this scenario. 

This is a current system design for scope item Stock Transfer with Delivery (BME).
The following restrictions apply:

  • Creation of inbound deliveries is mandatory when using Transportation Management. Stock transfer process without inbound delivery is not supported.
  • Deviating addresses in the stock transport order are not supported.
  • The use of Transportation Management in combination with Warehouse Inbound Processing (3BR) is not supported.

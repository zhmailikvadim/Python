Symptom

When running an Ad‑Hoc report after switching the system language, the report displays no results.

Environment

SAP SuccessFactors HCM Suite

Reproducing the Issue

  1. Create a Ad-Hoc/Table report
  2. Add a filter to a Foundation Object - such as Event Reason or Pay component
  3. Navigate to the report center and run the report
  4. Change the system language and run the same report
  5. Observe that no data is displayed on the report once the language has been changed
  6. Edit the report and note that the Foundation Objects labels (e.g., Event Reason) labels appear in the original language, noy the new system language 

Cause

This is an expected behavior. Foundation object labels (such as Location, Pay Component, Event Reason etc.) are not converted after a locale change in Ad-hoc reports. The blank values are expected if the filter is maintained when the system language is changed. 

Resolution

Remove the Foundation Object filters from the Ad-hoc report and the report should display data accordingly.

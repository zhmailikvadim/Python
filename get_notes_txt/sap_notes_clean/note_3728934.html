Symptom

  • When uploading data using an upload job, if there are master data errors, the system only shows the rejection reason as "MASTERDATA" without specifying which master data value has the issue.
  • Users must manually check all master data values in the upload file against the model to identify the mismatch.

Environment

  • SAP Analytics Cloud

Reproducing the Issue

  1. Upload a flat file using an Upload Job that includes one or more invalid members in a mapped dimension
  2. The records will be rejected
  3. Download the Rejection Summary file
  4. Check the Rejection Reason column.
  5. The rejection reason will appear as “MASTERDATA”

Cause

The current behavior for the "Rejection summary" is by design, and is behaving as expected.

Resolution

You can track any plans to change the current behavior with the below influence portal request:

https://influence.sap.com/sap/ino/#idea/331167

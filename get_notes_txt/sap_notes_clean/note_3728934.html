Symptom

Journal Entries in status Posted are not visible in any of the Verify General Journal Entries versions (F2547 and F2547A) after an upgrade from 1709 or 1809 OP releases.

Some posted Journal Entries were not included in the Silent Data Migration for class CL_SDM_JE_WF_MIGRATION_2.

CDS view P_WorkflowToObject doesn't return any data if you input the ObjectKey of posted document.

Other Terms

Verify General Journal Entries, F2547, F2547A, new version, For Requester, P_WorkflowToObject

Reason and Prerequisites

The field "Step ID" (STEP_ID) in the table SWWWIHEAD wasn't filled in the 1809 OP and older releases and therefore the CDS vie FACVD_VERDOC_WLI were not able to read such workflow data.

Therefore the issue can occur when you upgrade from 1709 or 1809 OP to newer releases.

Journal Entries for SDM are provided by the class CL_SDM_JE_WF_MIGRATION_2 that reads data from CDS view FACVD_VERDOC_WLI.

Solution

Implement the attached Correction Instruction or the mentioned Support Package to enable the affected data for CDS view FACVD_VERDOC_WLI (F2547).

To provide data for the new version of the app (F2547A), execute the Silent Data Migration again. For more details please see the 3592822 .

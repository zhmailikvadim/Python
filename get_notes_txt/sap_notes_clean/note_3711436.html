Symptom

  • Users are unable to access the environment using the pre-existing single sign-on authentication method after converting from ICM to APM SKU.
  • Error message "Error code: 404 — SAP Sales Cloud" appears during the authentication process. 

Environment

  • SAP Sales Performance Management
  • Agent Performance Management

Reproducing the Issue

  1. Attempt to access any of the APM environments using the pre-existing single sign-on.
  2. The "Error code: 404 — SAP Sales Cloud" is displayed on the page. 
  3. The authentication logs displays: 
    1. [INFO:SamlUtils.getActiveIdp] More than one IDP defined in the system. Cannot determine IDP to use.
      [INFO:SamlUtils.getActiveIdp] IDP could not be determined.
      [ERROR:AssertionConsumer.Process] Error 17700667180280006 : Cannot perform SP-initiated Authentication because we could not determine which IdentityProvider to send the Authentication Request to. If there is only one Identity Provider configured in the system, make sure it is Active. 

Cause

During the ICM to APM SKU conversion, the provisioning team adds an Identity Provider record for SAP IAS upon customer request. If the environment already contains an active Identity Provider record, multiple active Identity Provider entries are created in the system and this results in the standard Assertion Consumer URL (without specifying the idpEntityId) failing to function. 

Resolution

  1. To continue using the existing Identity Provider, the following options are available:
    1. If you do not want to use SAP IAS:
      1. Reopen the Identity Provider record for SAP IAS by navigating to the "Identity Provider Search" page, selecting the record and clicking the "Reopen" button.
    2. If you want to use both identity providers:
      1. Use the corresponding Assertion Consumer URL for each IdP, copied from the “SP Initiated Authorization URL” field of the relevant IdentityProvider record.

Symptom

There is a bug in PricingCommandUtil class method "convertToSLCItemDataBean", which converts iteminfo data bean to slc data bean, then this slc data bean is used as a primary bean for further processing. But it seems that while setting data in slc data bean, there are some field values which are not set and hence the calculation is coming wrong. The filed names are: BaseYears, ScaleYears, DaysPerMonth, DaysPerYear, External Rate Value & Pricing Relevant Flag.

Reason and Prerequisites

IItemInfo Bean should be properly populated with the values of Scale Base values

Solution

 Populate SLCItemDataBean with relative values of missing fields mentioned in symptom from IItemInfo

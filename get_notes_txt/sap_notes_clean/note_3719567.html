Symptom

When creating translation for category, if description is not provided, it gives error 'One or more fields in the API request contains invalid data. See the log for more information'.

Environment

SAP SALES CLOUD CPQ

Reproducing the Issue

1. Go to setup.

2. Add a translation to a category, do not provide description.
3. Try to save.
4. It gives error 'One or more fields in the API request contains invalid data. See the log for more information'.

Cause

Bug in application.

Resolution

ETA : CPQ-2605 (tentative).

Workaround: Add some value in the Description field in translation and you will be able to save it.

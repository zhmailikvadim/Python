Symptom

In the Ticket NLP Classification scenario of SAP Cloud for Customer, inbound emails are processed using Machine Learning models that perform: Language detection

Environment

Cloud for Customer

Reproducing the Issue

C4C receive customer emails written in English. However, some emails contain:

  • Name in different language
  • Addresses or location references in non-English languages

Even though the full email itself is English, the presence of 
such linguistic tokens sometimes triggers the language detector to 
incorrectly classify the message as : Spanish/Italian/English or any other language

Cause

According to the Machine Learning documentation for Ticket NLP Classification:

  • The system uses statistical and neural language detection methods.

  • Detection is sensitive to character patterns, name structures, and frequently occurring words associated with a language corpus.

  • Short texts or text with named entities from multiple languages can mislead the classifier.

Resolution

This is expected behavior.
The NLP model used in the system is designed to process only one supported language at a time within the ticket descriptionmultilingual content in a single text is not supported.
When the model detects mixed languages, it becomes unable to reliably determine the primary language for processing. In such cases, the linguistic analysis is impacted, and the model may fail to correctly interpret the text as English.
Therefore, the presence of even a few foreign-language words can cause the model to misidentify or fail to process the language correctly

Symptom

  • A technical query was raised regarding the "Update Contacts" API endpoint (PUT /contact).
  • Questions were specifically about the `create_if_not_exists` functionality and its operational behavior.
  • Inquiry on using a custom field `pwc_club_id` as the unique identifier (`key_id`) for contacts.

Environment

SAP Emarsys Customer Engagement

Reproducing the Issue

  1. Review the API documentation for the "Update Contacts" endpoint (PUT /contact).
  2. Attempt to configure the `create_if_not_exists` functionality within the API call.
  3. Use the custom field `pwc_club_id` as the `key_id` for contact identification.

Resolution

1. The `create_if_not_exists` functionality enables the API call to perform an update or insert operation:  
   1. If a contact matching the provided `key_id` is found, the existing record is updated.  
   2. If no match is found, a new contact record is created using the payload values.  
2. Include mandatory fields in the payload for new contact creation; otherwise, the creation will fail.  
3. Ensure the field used as `key_id` contains unique values to guarantee reliable identification.  
4. Serialize requests per key value to avoid race conditions caused by parallel or simultaneous requests using `create_if_not_exists`.  
5. Verify that the custom field used as `key_id` is indexed for identification. The field `pwc_club_id` is confirmed to be indexed and can be used as the unique identifier.

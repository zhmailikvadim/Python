Symptom

When you use SuccessFactors EC Time, incorrect calculations may occur for tax-exempt bonuses (section 3b EStG).

Other Terms

Payroll function D2010, employee remuneration information, IT2010, Employee Central Time, P2010, XIT, RPCMASD0_PROCESS_T599Y, process_t599y

Reason and Prerequisites

SAP Note 3611193 (Technical note: Adjustment of payroll function D2010) changed the source code. This ensures that the program part previously executed for filling the field BEGUZ in the table ZH is executed only if it was previously transferred initially from the data of the table P2010. The value '000000' was not recognized as initial.

Solution

Implement the attached correction instructions or import the specified support package.

After you import the HR Support Package or implement the correction instructions, you must perform retroactive accounting to correct the affected cases. If incorrect data was still calculated for 2025 due to retroactive accounting in January, this can be corrected for tax purposes in the previous year using payroll for February by opening the tax year (view V_T5D2M).

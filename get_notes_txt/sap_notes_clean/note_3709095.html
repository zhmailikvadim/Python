Symptom

  • Exported data from development site to quality site using respective API keys
  • Created a hosted page on the quality site
  • Hosted page link does not display correctly when pasted in the browser, while generic communications selection works as expected.
  • Error message displayed: "Oops, Something went wrong. Missing required parameter: auth_token".

Environment

  • SAP Customer Data Cloud
  • UI builder
  • Screen-Sets

Reproducing the Issue

  1. Export data from development site using API Key to quality site using API Key.
  2. Create a hosted page on the quality site
  3. Copy the hosted page link and paste it into the browser.

Cause

The hosted page cannot be used on runtime without the "auth_token". This is an expected behavior.

Resolution

  1. Ensure that the hosted page is accessed with the required "auth_token".
  2. To test the page, run the preference flow where the admin sends an email to the user, and the "auth_token" will exist.
  3. Verify that the hosted page link works as expected after the preference flow is executed.

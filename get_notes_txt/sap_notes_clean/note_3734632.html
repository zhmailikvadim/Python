Symptom

The event for expiring certificates under Cloud ALM-Health Monitoring has been configured, but it is observed that delay occurs for triggering Alerts/sending notifications.

Environment

  • SAP Cloud ALM
  • S/4 HANA system/ ABAP systems running on ST-PI SP32 33 and ST-PI 7.58 SP00 

Cause

Beside the threshold settings within SAP Cloud ALM, also the data collector is rating some of the metrics.
So when no threshold setting is defined in SAP Cloud ALM, the data collector based rating would take over.
This was especially used while onboarding the metric content in SAP Cloud ALM but is not required anymore.

Resolution

The developers have removed the rating of ABAP.Certificates. 

Implement the latest version of SAP Note 3639977 will help resolve the issue.

3639977 - Collective corrections as of ST-PI 7.40 SP32 for SAP Cloud ALM

Please note that it may take up to two hours for the changes to become active.
To speed up that process, please disable Health Monitoring for at least 5 minutes and enable it again.

Keywords

delay, health monitoring, notification, alert, expiring certificates

Symptom

On the consumer (client) side the system raises one of the following error messages:

  • Web Service processing error; more details in the web service error log on the provider side.
  • When executing the customized 'OrgStructure Replication Boomi Process', error is raised: Failed processing original documents i1i in the OrgStructureReplication connector: com.boomi.webservices.WebServiceException: Error invoking soap operation.

AND 

On the provider (server) side the error message below is shown by checking the error log in transaction SRT_UTIL in the ABAP system:

SRT: Plain SOAP: Reliable messaging (RM) configured, but no Message ID and no WSRM assertion provided.

"Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental."

Environment

  • SAP to SFSF Org Structure Replication
  • Middleware Integration (HCI or Boomi)
  • ABAP Web Services
  • SAP NetWeaver
  • SAP NetWeaver Application Server for SAP S/4HANA
  • ABAP PLATFORM - Application Server ABAP

Reproducing the Issue

  1. Send the SOAP request in the consumer (client) system.
  2. Check the error log in transaction SRT_UTIL in the provider (server) system.

Cause

Scenario 1 - if Asynchronous Web Service communication is meant to be used:

  • The SOAP Web Service configuration on the client side is incomplete. The ABAP soap service provider (server) uses the Web Services Reliable Messaging (WS-RM) protocol. Whereas the consumer (client) does not send the necessary message ID in the SOAP header element. This element is mandatory for the WS-RM protocol to work, but it is missing from the SOAP header of the request.

Scenario 2 - if Synchronous Web Service communication is meant to be used:

  • The Operation profile is set to Asynchronous reliable. This is usually the case for custom ABAP Web Services.

Resolution

Scenario 1:

To ensure correct Asynchronous processing, Message ID must be provided by the Consumer side.

  • If the Consumer is not an ABAP system, check if the consumer (client) supports the Web Services Reliable Messaging (WS-RM) protocol. The ABAP implementation can read the MessageId from the SOAP header (WS-A or SAP-MessageID protocol) or in case of SAP-RM (for example: Plain SOAP) also the transfer via query parameter is supported. If the SOAP Header is empty, the query parameter must be used.
    • The correct format of the query must be, as example: https://<server>:<port>/<path>/<myservice>?MessageId=989C6E5C-2CC1-11CA-A044-08002B1BB4F5
    • MessageId must be written with uppercase M and I and the MessageId must have exact this format. All other formats are not accepted. More information in section "Interoperability with 3rd party products" of the following SAP Help Portal page:
      Plain SOAP.
  • If the Consumer is an ABAP system, follow these steps to ensure that the correct Message ID and SOAP header elements are sent:
    1. Make sure the corrections of SAP Note 2368550 are implemented in the system, if applicable to the release.
    2. In the ABAP back-end system, run transaction SOAMANAGER and click Web Service Configuration.
    3. Select the correct "Consumer Proxy" and edit or click "Apply Selection", depending on the system release.
    4. Go to "Configurations" tab and edit the Logical Port.
    5. Click the Messaging tab and enter the data as shown below:

 

Note: If it is not possible to set the value SAP RM for field RM protocol, refer to KBA 2495704 - RM Protocol field not working in SOAMANAGER.

Scenario 2:

The Operation profile needs to be set to Synchronous. Please follow the below steps:

  1. Go to transaction SPROXY_START on the affected ABAP system
  2. Switch to the Web Service Definition tab
  3. Display the affected Service Definition
  4. Switch to the Configuration tab and open the affected Operation Profile to perform this change

Afterwards, the following steps also need to be performed:

  1. Rebuild the SOAMANAGER Design Time cache, refer KBA 2772510.
  2. Create a new binding under that same Service Definition. This is the one that will need to be used as of now.

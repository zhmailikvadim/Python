Symptom

When adding component and creating subnetwork in a collective order, error Message C2145 "Subnetwork for component &1 &2 cannot be created" occurs, even though the following causes in error diagnosis are not fulfilled:

The collective order was not completely imported for processing

  • the component was deleted
  • a subtree already exists for the component
  • a goods issue has already been carried out for the component
  • the component has no requirements quantity
  • the component is a split batch record
  • the material status of the component does not permit in-house production
  • the component is
    • a by-product
    • a co-product
    • externally procured
    • a variable-sized item
    • a text item
    • a part to be discontinued
    • a bulk material
    • an intra material
    • a pipeline material

Environment

SAP S/4HANA Cloud Public Edition

 

Reproducing the Issue

  1. Go to Change Production Order app.
  2. Enter a collective order.
  3. Add a component, "Component & & is a direct production assembly Create subnetwork?" dialog occurs.
  4. Click "Yes", error "Subnetwork for component &1 &2 cannot be created (Message no. C2145)" occurs.

Cause

System will check if entire collective order is processed when creating subnetwork for component. If not, which means the value of "Process Entire Collective Order" is empty in coding, then system will prevent the creation of the subnetwork.

Resolution

Make sure entire collective order is processed. There are two ways:

  1. Choose "Collective Order" selection when entering the production order.
  2. Click "Read Total" button in order structure screen to make sure all orders are ticked.

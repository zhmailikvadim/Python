Symptom

Joule feature is visible in the system for onboarding users; however, the functionality does not work as expected.

For affected onboarding users:

  • Users are able to log in to Joule using IAS credentials.

  • Joule prompts related to onboarding (for example, viewing onboarding tasks or checklist) do not return results or fail to respond.

  • Onboarding-related information such as checklist or assigned activities cannot be retrieved through Joule.

This behavior is typically observed for onboarding users even though Joule appears to be correctly enabled in the instance.

Environment

SAP SuccessFactors Onboarding

Reproducing the Issue

  1. Log in as a new hire through the provided link.
  2. Access joule and enter the IAS credentials.
  3. Provide the prompt as "view my buddy/assist new hires/view my onboarding checklist".
  4. Observe that there is no response or the onboarding checklist cannot be accessed or retrieved.

Cause

  • Joule functionality for onboarding users may fail due to an invalid UserUUID, which results in authentication failures when accessing Joule services.
  • This typically occurs when the Global User ID is not populated or does not match between SuccessFactors and IAS.
  • When IAS is not enabled for onboarding external users, the Global User ID is not generated or synchronized. As a result, Joule features will not function correctly.

Resolution

Expected Behavior

Joule functionality for onboarding users requires proper integration between SAP SuccessFactors and Identity Authentication (IAS).

If IAS is not enabled for onboarding external users, the Global User ID is not generated or synchronized. In this scenario, Joule features may be visible in the system but will not function as expected.

This behavior is expected by design.

Recommendation

To enable Joule functionality for onboarding users:

  • Ensure IAS is enabled and integrated for onboarding external users.

  • Verify that the Global User ID is generated and synchronized.

Manual creation of users in IAS does not establish the required backend synchronization and will not enable Joule functionality.

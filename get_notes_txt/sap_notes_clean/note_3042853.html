Symptom

Restrictions related to using the SAP Build Work Zone, standard edition as a central entry point for the SAP S/4HANA On-Premise releases.

Other Terms

Restrictions, SAP Build Work Zone, SAP Launchpad Service, standard edition, Central Fiori Launchpad, cFLP, central entry point, SAP S/4HANA 1610, SAP S/4HANA 1709, SAP S/4HANA 1809, SAP S/4HANA 1909, SAP S/4HANA 2020, SAP S/4HANA 2021, SAP S/4HANA 2022, SAP S/4HANA 2023, S/4HANA Cloud Private Edition

Reason and Prerequisites

Restrictions related to using the SAP Build Work Zone, standard edition as a central entry point for the following SAP S/4HANA On-Premise or SAP S/4HANA Cloud Private Edition.

The integration of SAP Build Work Zone, standard edition with SAP S/4HANA based on Content Federation requires Fiori Frontend Server 6.0 with a minimum SP04. For more information, see SAP Help Portal: Federation of Remote Content Providers

Note: Check this note for updates on a regular basis.

Solution

Integration with SAP Build Work Zone, standard edition is available starting with SAP S/4HANA On-Premise 1809 with using exposure version 1.

There is a exposure version 2 available to expose the content from SAP S/4HANA On-Premise for the integration with SAP Build Work Zone, standard edition. Exposed apps via using exposure version 2 are launched from SAP Build Work Zone in a standalone (new tab) mode within the shell of SAP S/4HANA launchpad, see details in Note 3345119 - Expose launchpad content to SAP BTP - Exposure version 2.

General Restrictions:

  • Integration with the SAP Build Work Zone, standard edition is not available for SAP S/4HANA On-Premise 1511.
  • The integration of SAP Build Work Zone, standard edition with S/4HANA versions 1610 and 1709 are only released for test and evaluation purposes.
  • SAP Build Work Zone, standard edition is no full equivalent of the SAP Fiori launchpad of SAP S/4HANA On-Premise. Its not expected to have all available features in SAP S/4HANA launchpad to be also in SAP Build Work Zone launchpad. For example, below features in S/4HANA launchpad are not supported when using SAP Build Work Zone, standard edition:
      • My Home 
      • Cards created from S/4HANA can not be directly consumed in SAP Build Work Zone launchpad
      • Personalizing the Navigation Bar to pin and reorder the spaces
      • Merging Launchpad Spaces or Pages

Detailed Limitations:

  • Setup and Administration:
    • Mass exposure on SAP S/4HANA side is not recommended due to content size limitations in SAP Build Work Zone.
    • Notifications functionality will not work in 1809 release as it needs SAP_GWFND component 7.54 SP06 and above. For more info refer Note: 3200288
    • My Inbox integration is supported as of UI5 Patch version
      • SAPUI5 1.84.16 for SAP Fiori Front-End Server 2020
      • SAPUI5 1.71.40 for SAP Fiori Front-End Server 6.0 SP04
  • General Usage and End User Limitations:
    • The merged Spaces or Pages in SAP S/4HANA launchpad are not merged in SAP Build Work Zone launchpad.
    • Content from different S/4HANA systems is not merged in SAP Build Work Zone: multiple groups, spaces, pages and apps with identical names might occur.
    • Sorting orders of Spaces or Pages in SAP S/4HANA launchpad might not be reflected in SAP Build Work Zone launchpad.
  • Usage and End User Limitations related to exposure version 1:
    • Built-in support for ticket creation and support related information is not supported when using exposure version 1.
    • SAP CoPilot / CAI integration is not supported when using exposure version 1.
    • Integrating Microsoft Teams (Share: Microsoft Teams) provided in the app is not supported when using exposure version 1.
    • Contact Support is not supported when using exposure version 1.
    • SAP Analytics Cloud based apps are not supported when using exposure version 1.
    • Remote apps (Apps of type URL or URLT) which are integrated into SAP S/4HANA will be opened in a new browser tab/window in SAP S/4HANA shell or other product shell when using exposure version 1.
    • SAP GUI apps which are embedding Advanced Variant Configuration (AVC) or Advanced Available-to-Promise (ATP) Fiori components are not supported when using exposure version 1, for example "Create Sales Orders - VA01".
  • Usage and End User Limitations related to exposure version 2:
    • Apps exposed via exposure version 2 are launched from SAP Build Work Zone in a standalone (new tab) mode within the shell of SAP S/4HANA launchpad. Therefore the full capabilites from S/4HANA launchpad (e.g. Built-in support, SAP CoPilot, user default paramters) are supported when using exposure version 2.
    • Factsheets apps can be exposed via exposure version 2 with implementing Note 3517657.
    • Apps without transaction codes maintained are not exposed with version 2, refering Note 3345119.
    • Fiori Apps shipped with Software Components UIHR002 (HCM apps), UIMDG001 (Master Data Governance), and UITRV001 (Travel Management) are not supported when using exposure version 2.

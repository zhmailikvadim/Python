Symptom

The cost center is changed in the valuation but is not changed in asset accounting, and the depreciation is posted with the old cost center.

Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental. 

Environment

  • Flexible Real Estate Management (RE-FX)
  • Contract and Lease Management
  • SAP ERP Central Component
  • SAP ERP
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA Finance
  • SAP S/4HANA

Reproducing the Issue

  1. The cost center is changed in accordance with SAP Note 3008264.
  2. In the valuation cash flow the cost center changed correctly.
  3. In the asset accounting, the cost center didn't change, and the depreciation was posted with the old cost center. 

Cause

  • If a cost center change is triggered by the valuation, this cost center update on the asset must be enabled which depends on configuration settings in Asset Accounting.
  • This can be checked by opening the asset in change mode with transaction AS02 and proceed to the time-dependent data to verify if it would be possible to change the cost enter directly on the asset.

  • Note above that the field cost center is not editable. If this is not possible to change, then it cannot be changed also from the contract.
  • This is caused by settings in Asset Accounting which prevent the change of the cost center when the asset is activated.
  • If the flag “Used in Balance Sheet” is set in the configuration settings, it is no longer possible to change the account assignment object in the asset master record, once the asset had been capitalized. This configuration can be found in the following path:

TCODE: ORFA - > Integration with general ledger accounting -> Additional account assignment objects -> Activate Account assignment objects

Resolution

There is no error in the system; this is standard behavior in SAP.

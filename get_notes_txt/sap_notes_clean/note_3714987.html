Symptom

In the Employment Information Schedule (EIS) (HNZLPDS0) report, the Child Support code ‘P’ is appearing for both active and terminated employees.

As per IRD requirements, if an employee has been terminated, the Child Support code should be reported as ‘C’ instead of ‘P’.

Other Terms

  • Program: HNZLPDS0
  • Transaction: PC00_M43_LPDS
  • Include: PAPDSNZF
  • Other: Employment Information Schedule

Type of Object Change

Workbench Objects - No
Code - Yes
Schema/Rules - No
Wage Types - No
Customizing Tables – No

Reason and Prerequisites

In program HNZLPDS0, the pay period start date is used to determine the Child Support code for terminated employees.

The current logic evaluates the termination date against the pay period start date (the is, termination date ≤ pay period start date). Due to this condition, terminated employees may still be reported with Child Support code ‘P’ instead of ‘C’.

Solution

Modify program HNZLPDS0 so that the logic to determine the Child Support code for terminated employees evaluates the termination date against the pay period end date (or reporting period date) instead of the pay period start date.

The logic should ensure that if an employee is terminated on or before the reporting period end date, the Child Support code is set to ‘C’. Otherwise, the code should remain ‘P’.

This change ensures compliance with IRD reporting requirements and prevents terminated employees from being incorrectly reported with Child Support code ‘P’.

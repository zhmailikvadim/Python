Symptom

Created the ESEF iXBRL instance and preview file

Downloaded both files 

Looking at the file sizes of these 2 files, the preview file is significantly larger than the instance file. Why this big difference ?

Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental

Environment

Disclosure Management 10.1

Reproducing the Issue

N/A

Cause

The preview file consists of 2 parts

The first, is simply the instance file (as produced by Disclosure Management)

The 2nd part is created by the Workiva previewer plugin (which is embedded in Arelle - called by Disclosure Management when generating the preview)

This references the java-script (ixbrlviewer.js) and then has the information for all tagged facts (i.e. descriptions in multiple languages, anchor information, LEI etc.)

The section is marked by the comments

<!--BEGIN IXBRL VIEWER EXTENSIONS-->

..
<!--END IXBRL VIEWER EXTENSIONS-->

If you delete the lines in between and save the file....

.... you will see that the files are same (similar) size

Ultimately, this is not something Disclosure Management controls.

Some factors that contribute:

1) embedding images. These images will be repeated multiple times in the preview section

Note: you will see lines like this

 "v": "\u003Cspan class=\"dmspan dmtext001\" style=\"font-size:10pt; color:#000f64; left:0pt; top:0pt; line-height:11.33pt;\"\u003E8 \u003C/span\u003E\u003Cdiv class=\"dmdiv\"\u003E\u003Cimg alt=\"[IMAGE]\" class=\"dmimg\" src=\"data:image/png;base64,iVBORw0KGgoAAAANSUhEUgAAClAAAA

2) use of (non standard) fonts

Resolution

Avoid including images in the tagged text blocks OR use the 'embed pictures'=No parameter

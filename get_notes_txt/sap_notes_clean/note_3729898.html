Symptom

Preliminary program DME (report RPCDTAx0, RPCDTCx0, HxxCDTA0) rejects personnel numbers with the following error message 

"Field T_REGUH-ZSPRA must not be blank in the payment run"

Environment

  • Human Capital Management (HCM)
  • SAP R/3
  • SAP ERP Core Component
  • SAP ERP
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP SuccessFactors Employee Central Payroll

Cause

  • The company code language key (SPRAS) in table T001 is not maintained, resulting in T_REGUH-ZSPRA being left blank during the pre-DME run.

Resolution

  1. Maintain the language key for the affected company code in table T001.
  2. To do this, open transaction OX02 (Edit Company Code) and select the relevant company code.
  3. Enter a valid Language key (SPRAS) and save.
  4. Transport the change as required for your landscape.

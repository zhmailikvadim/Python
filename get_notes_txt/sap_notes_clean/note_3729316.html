Symptom

With S_RFC_ADM, users will have access to SM59 to handle RFC Destinations. How to handle actions in EDOC_COCKPIT for India GST Reporting Node without giving access to SM59?

Environment

  • SAP ERP
  • SAP S/4HANA
  • SAP S/4HANA Cloud Private Edition
  • SAP Digital Compliance Service for India

Cause

Configuration Issue

Resolution

The Authorization Object Required is S_RFC_ADM for RFC related tasks. Since we use RFC to connect with the DCS, this Authorization is mandatory.

S_RFC_ADM can have different activity codes:
Activity 01 – Create
Activity 02 – Change
Activity 03 – Display
Activity 06 – Delete
Activity 36 – Extended Maintenance

Above first four activities have an access to SM59.  Using activity “36”,  user’s will not be able to access SM59 but users can execute their transactions(programs) which involved RFCs. In EDOC_COCKPIT, except eDocument creation other all actions required RFCs to connect with DCS. So it’s mandatory to provide S_RFC_ADM with activity "36".

Symptom

You are going to open or route an incident on the component HAN-DB-ENG-KGE as a Knowledge Graph issue is suspected.

Other Terms

Knowledge Graph, Triple Store

Reason and Prerequisites

For effective problem analysis and resolution, please provide the below information in the provided template to SAP HANA Knowledge Graph Development Team.

Solution

There is a base level of information that should be contained in every incident opened on or routed to SAP HANA Knowledge Graph Development Team. Additionally, there is specific information that helps with processing different error types.

Basic Information & System Access

1. General system information of the affected HANA Cloud instance - see also SAP Note 2904098
   1. HANA Version 
   2. Service/Instance ID (UUID like ab12c345-d678-9012-e34f-g567890h12ij) or
   3. SQL Endpoint 

2. Issue reproduction scenario
   1. Expected versus Observed result
   2. Reproduction scenario step-by-step

3. Traces - triplestore trace file(s)

4. System access with a database user having issue reproduction permissions and the role SAP_INTERNAL_HANA_SUPPORT is very helpful.

5. In case of triplestore outage, support colleagues will themselves collect the relevant traces (e.g., crashdump file and triplestore trace file(s)). Please describe if the outage is reproducible or sporadic.

6. Please also provide traces:

  1. Activate traces:

    alter system alter configuration ('triplestore.ini','System') set ('trace', 'triplestore_life_of_query')='debug' with reconfigure;
    alter system alter configuration ('triplestore.ini','System') set ('trace', 'triplestore_executor')='debug' with reconfigure;
    alter system alter configuration ('triplestore.ini','System') set ('trace', 'triplestore_catalog')='debug' with reconfigure;

  2. Reproduce the issue.
  3. Deactivate traces:

    alter system alter configuration ('triplestore.ini','System') unset ('trace', 'triplestore_life_of_query') with reconfigure;
    alter system alter configuration ('triplestore.ini','System') unset ('trace', 'triplestore_executor') with reconfigure;
    alter system alter configuration ('triplestore.ini','System') unset ('trace', 'triplestore_catalog') with reconfigure;

  4. Collect the traces with "triplestore..." in the name.

 7. In case of "unexpected results" issue, please provide these traces below: 

  1. Activate traces:

    alter system alter configuration ('triplestore.ini','System') set ('trace', 'triplestore_life_of_query')='debug' with reconfigure;
    alter system alter configuration ('triplestore.ini','System') set ('trace', 'triplestore_executor')='debug' with reconfigure;
    alter system alter configuration ('triplestore.ini','System') set ('trace', 'triplestore_catalog')='debug' with reconfigure;
    alter system alter configuration ('triplestore.ini','System') set ('trace', 'triplestore_cgr')='debug' with reconfigure;

  2. Reproduce the issue.
  3. Deactivate traces:

    alter system alter configuration ('triplestore.ini','System') unset ('trace', 'triplestore_life_of_query') with reconfigure;
    alter system alter configuration ('triplestore.ini','System') unset ('trace', 'triplestore_executor') with reconfigure;
    alter system alter configuration ('triplestore.ini','System') unset ('trace', 'triplestore_catalog') with reconfigure;
    alter system alter configuration ('triplestore.ini','System') unset ('trace', 'triplestore_cgr') with reconfigure;

  4. Collect the traces with "triplestore..." in the name.

Symptom

When creating custom fields in globalAssignmentInfo (EmpGlobalAssignment) as Visibility=Edit, they are still not shown as Upsertable=true in OData API.

Environment

  • SAP SuccessFactors HCM Suite
    • OData API

Reproducing the Issue

  1. In Manage Business Configuration, create a custom field under globalAssignmentInfo (EmpGlobalAssignment) and set it as Visibility = Edit.
  2. Refresh the OData API metadata.
  3. Open the OData API Data Dictionary, navigate to EmpGlobalAssignment, and observe that the custom field shows sap:upsertable = false.

Cause

Global Assignment object has specific custom fields which can be set as Upsertable=true; other custom fields are not upsertable.

Resolution

The custom fields in Global Assignment object reserved for being Upsertable are:

  • String fields: From customString101 to customString115
  • Date fields: From customDate31 to customDate35
  • Long fields: From customLong21 to customLong25
  • Double fields: From customDouble21 to customDouble25

Please use one of these fields from the list and ensure you refresh the OData API metadata (KBA 2171588 - OData API Metadata Refresh and Export in SuccessFactors Cloud) after enabling them or doing changes.

Symptom

While scheduling an interview on the Interview Scheduling page, when the user enters the interviewer’s name and clicks on the “Find Availability” option, they encounter the following error message:

errorId = 8e770496-3183-45d6-b19e-bca8d4877692
fingerprint = c8b14c7fcc5547344218e56ff377137c9b36bde5
timestamp = 2026-02-25T08:50:21.030+0000
errorMessage = An application error occurred. Please try again later, or notify support (with the error information of errorID=8e770496-3183-45d6-b19e-bca8d4877692, timestamp=2026-02-25T08:50:21.030+0000, and the version information from the current screen) if the issue keeps occurring.
server = 31pc66bcf1046
versionInfo = Release: 2511.20260206090556
Server: 31pc66bcf1046
Timestamp: 2026-02-25T08:42:05.949+0000 

This error appears immediately after attempting to retrieve the interviewer’s availability. 

Environment

SAP SuccessFactors Recruiting Management 

Reproducing the Issue

  1. Navigate to the Interview Scheduling page in SuccessFactors.
  2. Select a candidate for scheduling interview 
  3. Enter the interviewer’s name in the designated field.
  4. Click on the “Find Availability” option.
  5. Observe the error message displayed immediately after attempting to retrieve the interviewer’s availability.

Cause

The "Outlook and Teams for Online Meetings" section in the Work Tech configuration is showing an error.

Resolution

Outlook and Teams for Online Meetings section in the Work Tech configuration is showing an error. The logs, indicate that the issue is due to incorrect details configured in Work Tech. The error message in Work tech states:

"Unable to connect. Please check your Client ID, Client Secret, or Token URL in Security Centre."

This suggests that the connection details need to be verified and corrected under work tech configuration for "Outlook and Teams for Online Meetings" section. Updating the configuration to ensure a successful connection and then performing an OData metadata refresh. Post this the issue shall be sorted.

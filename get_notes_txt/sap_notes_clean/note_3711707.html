Symptom

  • Attempting to update the termination date for a user from the UI results in the error 'Termination date must be later than hire date'.
  • The update from data loaders does not throw any errors.
  • Error is noticed only while using bulk edit option even for a single record.
  • Error is noticed for records with NULL hire date 
  • Error is noticed only when updating the termination date with past or current date.

Environment

  • SAP Sales Performance Management
    • Incentive Management
    • Incentive Management on Hyperscalers

Reproducing the Issue

  1. Create a record with NULL hire date and termination date.
  2. From the list view, select the option 'Edit selected'.
  3. This opens the typical bulk edit window.
  4. From the selection options, select termination date.
  5. Select a past or current date and hit update.
  6. Notice the error 'Termination date must be later than hire date'.

Cause

  • Product behavior. Additional date checks are initiated on updating the termination date from the bulk update option.

Resolution

  1. Code changes are planned via COM-130346.
  2. Tentatively planned for 2603 release.
  3. As a Workaround either update the termination date from the UI by editing the Payee record or by using excel loaders or XDL.

Symptom

In SDCCN, the service definition, for a function module that has a parameter length of exactly 60 characters, ends in syntax error.

In transaction SDCCN, on the Done tab, the log for a data collection task, contains an error

﻿<module name>: report generation < cluster ID number of the module > failed 

The task itself finishes in successful status. The module mentioned in the error message is missing in the service data collected.

Especially on an FRUN system upgraded to ST-PI 740 SP16 this issue occurs with log message:

﻿/SDF/TECHSYS_CONFIGURATION : report generation ( 001380 ) failed﻿

Other Terms

SDCCN, /1CAGTF/WI, 60 CHAR

Reason and Prerequisites

Program error.

You have SAP Solution Tools Plug-In (ST-PI) installed and by means of SDCCN you run Service Definition Refresh task or Request Session Data task that regenerates the reports /1CAGTF/WI*.

Solution

Install the following correction.

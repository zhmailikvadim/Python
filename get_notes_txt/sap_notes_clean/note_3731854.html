Symptom

Have configured a scenario using the OData adapter and during runtime the below error is returned:

"java.lang.RuntimeException: java.lang.RuntimeException: com.sap.aii.adapter.lib.api.OsciException: Caused by:org.apache.olingo.odata2.api.edm.EdmSimpleTypeException: The literal <yyyy-mm-ddThh:mm:ss:dddddddZ> is not formatted properly"

Environment

  • SAP NetWeaver 7.5
  • SAP Process Integration
  • SAP Process Orchestration

Cause

While reading the OData payload, Olingo (the OData library used by OData/SFSF adapters) converts EDM types to Java types. Certain EDM types can be mapped to more than one Java type. EDM types Time, DateTime and DateTimeOffset are mapped by default to java.util.Calendar. However, the Calendar  does not support nanoseconds. Therefore, Olingo performs a check whether nanoseconds are present in the value and if that is the case, it expects the aforementioned EDM types to be mapped to java.sql.Timestamp. In order to override the default mapping for these EDM types, the required mapping has to be specified explicitly.

Resolution

Configure the advanced parameter "TypeMappings" as described in SAP Note: 2524120 - OData library fails to process specific properties.

Keywords

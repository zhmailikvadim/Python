Symptom

During an export using transaction SCC8N (Client Export), the client state appears to change during runtime.

Environment

  • Product: SAP S/4HANA foundation
  • Client Export Behavior

Reproducing the Issue

  1. Perform an export from the source system to the target system using transaction SCC8N (Client Export).
  2. Monitor the client state during runtime and observe any temporary lock/unlock adjustments and subsequent logging behavior in SCC4/T000.

Cause

This is an expected behavior

  1. Note that during SCC8N client export, temporary adjustments to the source client’s lock state can occur by design; the tool manages these changes, and manual client opening is not required.
  2. If SCC4 change logs do not show corresponding entries while table T000 shows updates, enable the “Append table changes” option in the SCC4 change log display to include table-level changes.
  3. Be aware that updates to “Last Changed By” and “Date” for T000 made during client copy/export are not recorded by SCU3 table logging; this is working as designed.

Resolution

If you uncheck 'Lock Source Client', the flow of client export (SCC8N) is as follows:

① Client export starts
② Until the initialization phase or exit is completed ⇒ Source client is unlocked
③ Export phase
④ Client export completes

 

If you check 'Lock Source Client', the flow of client export (SCC8N) is as follows:
① Client export starts
② Initialization phase or exit is completed
③ Until the export phase ⇒ Source client is unlocked
④ Client export completes

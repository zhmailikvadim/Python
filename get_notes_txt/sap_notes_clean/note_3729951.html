Symptom

  • When entering Package Data and selecting Value Help there are several errors like: "Unable to load data. Try again later, or check your SAP Cloud Connector and SAP S/4HANA configuration. " or "Can't load metadata from the S/4HANA system" when opening Define Data Mapping. 
  • Some users receive: “Service /sap/opu/odata/sap/API_GRTRANSACTIONDATA_SRV/$metadata call was terminated because the corresponding service is not available.”
  • Error HTTP 404 with reason “Not Found” for API_GRTRANSACTIONDATA_SRV $metadata.
  • In some cases, related areas (for example, Manage Packages) show empty value fields and errors when selecting values.

Environment

SAP Group Reporting Data Collection (GRDC) connected to SAP S/4HANA Cloud, Private edition

Reproducing the Issue

  1. Open the Enter Package Data application in Cloud Data Collection - Enter Group Reporting Data.
  2. Attempt to work with a package, selecting Value Help for dimensions
  3. Alternatively e.g. Open Define Data Mapping

Cause

  • There was an error regarding the availability of the odata service. For example, the service was inactive in SICF or in/n/IWFND/MAINT_SERVICE 
  • There was a temporary loss in connection
  • There was a misconfiguration in SAP SCC or SAP web dispatcher
  • A form was too complex and resulted in an error.
  • All resulting in a "HTTP 404 method not found" error . The error response was captured and cached by the application for 24 hours per request key, causing persistent failures during this time. 
  • Even after correcting the underlying cause, the error persists.

Resolution

  1. Check the underlying causes:

    Verify in the backend that OData services e.g.  API_GRTRANSACTIONDATA_SRV are active and reachable (for example, via service activation in the backend).
    Check for intermittent connectivity issues between the GRDC tenant and the backend (for example, review application server/ICM traces and connector logs).

  2. If the 24hours period is too long, clear the GRDC caches using the “Synchronize data…” button in app 'Manage Forms' (requires a role that grants GRDC_SYNC_DATA). This refreshes tenant-level cached request data. This is explained in the user guide.  Ask affected users to sign out of all sessions and sign back in after synchronization completes. After the cache reset initial work will be slower as keys and values are loaded into the cache again first time.

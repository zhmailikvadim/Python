Symptom

  • Changes made to the Workspace are not reflecting.
  • When publishing changes using the pencil icon in a workspace with text content, the changes to not take affect and the workspace is still the same as before editing.

Environment

SAP SuccessFactors Work Zone

Reproducing the Issue

  1. Log in to the system.
  2. Navigate to SuccessFactors Work Zone.
  3. Go to a workspace with text content.
  4. Click the pencil icon.
  5. Click Publish.
  6. The changes to not take affect and the workspace is still the same as before editing.
  7. When copying source code and creating a new workspace, it is published as desired.

Cause

The published page contains different language versions on the same page, and the changes are only made in the default language page.

Resolution

The steps below should be taken to address the issue:

Step 1#: Checking the application language:

  1. In top right corner, click on Profile/Avatar.
  2. Select Settings
  3. Select Language and Region (English by default).

Step 2#: Editing the page according to the value in the language and region from Step 1#:

  1. Select the workspace to be edited
  2. Click on Open workpage settings (under the pencil icon)
  3. Click on Translate the workpage title
  4. In top left right below the Header, select the language (the same as language in settings from Step 1#)
  5. Edit the page
  6. Click on Publish

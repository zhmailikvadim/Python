Symptom

  • An issue with the BCP command using ASE was reported. Incorrect character conversion when using the BCP utility with the -Y and -J options resulted in the correct values not being inserted into the ASE Server.
  • Command example:
    bcp pubs2..publishers in datafile -X -S <server> -U <username> -P <password> -c -t <field_terminator> ' -Y -e <errfile> -J <client_charset> 

Environment

  • Adaptive Server Enterprise 16.0 SP04 PL07
  • SAP CTBCP Utility 16.0 SP04 PL07

Cause

The issue occurred because the BCP command used two utilities that do not work together: -Y and -J.

  • -Y “specifies that character-set conversion is disabled in the server, and is instead performed by bcp on the client side when using bcp out.”
  • -J “specifies the character set to use on the client. bcp uses a filter to convert input between <client_charset> and the SAP ASE character set.”

Using both flags simultaneously creates a conflict in how character-set conversion is handled (client-side vs. filtered conversion), which led to the failure.

Resolution

When both flags ('-Y' and '-J') are used simultaneously, a conflict is created in how character‑set conversion is handled. When using BCP, only one of these options should be used to define the character‑set conversion.

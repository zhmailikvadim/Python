Symptom

  • Error during task promotion: "Job promotion failed due to unexpected error. Verify job exists in production and re-promote the job."
  • Task status in Sandbox is NOT promoted. But same task can be found in Production.

*****"Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental." *****

Environment

SAP Cloud Integration for data services

Reproducing the Issue

  1. Attempt to promote a task from Sandbox to Production.
  2. Observe the error message on the screen: "Job promotion failed due to unexpected error. Verify job exists in production and re-promote the job."
  3. Check production environment to confirm the task exists but is not marked as promoted in Sandbox.

Cause

Old SAP server code with promote issues. This old code left sandbox task promote flag not connected with the production same task. Not being able to reproduce again nowadays. Only happens with task promoted very long time(like during 2000) ago. 

Resolution

  1. Request CIDS system administrator to delete the task in Production.
  2. Re-promote the task from Sandbox to Production.
  3. Verify successful promotion and task status in Sandbox and Production.

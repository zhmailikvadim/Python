Symptom

When receiving contact person information from S/4 for a service document (e.g. service order or service contract), the wrong contact person is sent.

Reason and Prerequisites

The problem only occurs if you use two or more partner functions with the partner function category contact person within the same document. In this case the sent contact person will be random.

Solution

Please apply the attached code correction or relevant support package. With this the partner function which is first in alphabetical order will be chosen. This means in particular that SAP partner functions are preferred over customer partner functions. (If there are multiple partners with the same partner function, there is no problem, because always the main partner is chosen.)

Symptom

  • Error 631 occur during the database load process.
  • After taking a fresh dump, the load completed successfully.
  • Error message:

    00:0024:00000:00834:2026/03/12 19:57:54.34 server  Error: 631, Severity: 21, State: 3
    00:0024:00000:00834:2026/03/12 19:57:54.34 server  The length of 348 passed to delete row routine for the row at offset 0 is incorrect on the following page of database '******': Page pointer = 0x0x8448a6000, pageno = 106244865, status = 0x881, ptnid = 410390146, indexid = 0, level = 0, pnextrno = 0. The expected row length is 15. The offset should be no greater than 44.

Environment

  • SAP Adaptive Server Enterprise (ASE) 16.0 SP04 PL06

Reproducing the Issue

  1. Attempt to load the database from a dump file on the target database server.
  2. Observe the occurrence of error 631 during the database load process.

Cause

The issue has been logged under SAP CR 831882. 
Its description reads, "While loading a database dump, a 631 error with state 3 is raised with the message "The length of xxx passed to delete row routine for the row at offset 0 is incorrect on the following page of database '<database name>': Page pointer = <address>, pageno = <page number>, status = <address>, ptnid = <partition ID>, indexid = 0, level = 0, pnextrno = 0. The expected row length is <length>. The offset should be no greater than <offset value>." in the recovery redo phase.".

Resolution

The solution of this issue is currently under investigation.

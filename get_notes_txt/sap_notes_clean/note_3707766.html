Symptom

You are facing issues at the payment distribution with a curtailment, specifically, the refund amount doubles compared to the previous execution of payment distribution.

This problem occurs if : 
- Payment Plan Principle is used
- unpaid billed items or unbilled receivables before calculation date

As a solution, an additional capital is taken into account when calculating the overpayment capabilities.
This note introduces a new capital which is used at the calculation of overpayment capability.

See also note 3708940 - "New calculation base with regard to Effective Capital (PPP)".

Other Terms

TBCA_CAPITAL_DEF, BCA_OBJ_CHK_CASHFLOW_FOR_PD, Refund Loop, OC, Overpayment

Reason and Prerequisites

Program error

Solution

Import the relevant Support Package that is specified for your release.
As a manual advance correction, you can implement the attached correction instructions using transaction SNOTE.

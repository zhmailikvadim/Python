Symptom

  • Changing the POB's Transaction price to negative from positive or vice versa triggers an error message FARR_MSG_CUSTOM042 or FARR_MSG_CUSTOM043
  • Processing the negative transaction price in FARR_RAI_MON triggers an error message FARR_CONTR_LIFECYCLE001.

Environment

  • Revenue Accounting (RA).
  • Financial Accounting (FI).
  • SAP ERP Central Component.
  • SAP ERP.
  • SAP enhancement package for SAP ERP.
  • SAP enhancement package for SAP ERP, version for SAP HANA.
  • SAP S/4HANA.

Reproducing the Issue

  1. When the transaction price on a Performance Obligation (POB) could is changed from positive to negative value and vice versa, then an error message FARR_MSG_CUSTOM042 or FARR_MSG_CUSTOM043 is raised.

    FARR_MSG_CUSTOM042-Transaction price of perf. oblig. &1 cannot change from pos. to neg.

    FARR_MSG_CUSTOM043-Transaction price of perf. oblig. &1 cannot change from neg. to pos.

  2. While processing a Revenue Accounting Item (RAI) in FARR_RAI_MON, an error message  FARR_CONTR_LIFECYCLE001- "XNEGATIVE_ITEM from '' to X is not permitted" is raised.

    FARR_CONTR_LIFECYCLE001:Changing &1 (field &4) from '&2' to '&3' is not permitted.

    Cause

    The error is caused by the system not permitting the change of the transaction price from positive to negative value and vice versa.

    Resolution

    Set the message type for the following messages to I (Information), W (Warning), or S (Success) to bypass the error:

    • FARR_MSG_CUSTOM042

    • FARR_MSG_CUSTOM043

    Steps to follow:

    • Navigate to the following path:
      Go to Transaction Code FARR_IMG → Revenue Accounting → Revenue Accounting Contracts → Change Messages Control

    • Refer to the following SAP Knowledge Base Article (KBA) for detailed instructions:
      KBA 0002512661 - Error: FARR_MSG_CUSTOM*

      This article explains:

      • How to add these messages to the configuration table

      • How to change the message type from Error to Warning

    If the requirement is to allow both positive and negative values:

    • Leave the XNEGATIVE_FLAG field blank

    • Set the relevant FARR_MSG_CUSTOM message type to Success

    Note:
    SAP strongly recommends testing all configuration changes thoroughly in a non-production environment before transporting them to the production system.

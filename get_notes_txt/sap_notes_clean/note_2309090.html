Symptom

When creating a purchase document from Blanket Agreement in company, you receive one of the following error messages:

No matching records found  'Business Place' (OBPL) (ODBC -2028)  [Message 131-183]

No matching records found UICU ODBC 2028 131-183

Or another similar error message.

 

Reproducing the issue

Scenario:

  1. Warehouses 01 and 02 exist in the database.
  2. Set warehouse 01 as Default Warehouse in the Administration → System Initialization → General Settings → Inventory tab.
  3. Item 1 only has warehouse 02 assigned in Inventory → Item Master Data → Inventory Data tab.
  4. Create a Blanket Agreement for Item 1.
  5. Try to copy (procedure Copy To or Copy From) the Blanket Agreement to a purchase document.

Expected Behavior:

You receive the following error message:

Enter a valid value in "Whse" field - xx- Item code  [Message 173-62]

Actual Behavior:

You receive one of the following error messages:

No matching records found  'Business Place' (OBPL) (ODBC -2028)  [Message 131-183]

No matching records found UICU ODBC 2028 131-183

Or another similar error message.

Solution

SAP intends to provide a patch or patches in order to solve the problem described. The section References below will list the specific patches once they become available. The corresponding Info file of the patches in SAP Service Marketplace will also show the SAP Note number. Be aware that these references can only be set at patch release date. SAP will deliver patches only for selected releases at its own discretion, based on the business impact and the complexity of the implementation.

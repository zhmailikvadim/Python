Symptom

  • Creating a Rep Agent Instance
     
  • Ran ra_admin.sh (on Unix) or ra_admin.bat (on Windows) interactively and received the following errors:
     
    • Request upgrading instance <sa> from a previous minor version release.
      ERROR:  No verifiable instance <sa> exists in directory </sybase/rao211/RAX-15_5>.
      ERROR:  Failed to upgrade instance <sa> in directory </sybase/rao211/RAX-15_5>.
      Reason: No verifiable instance <sa> exists in directory </sybase/rao211/RAX-15_5>..

Environment

  • SAP Replication Server Option (RSO) 15.7.1
     
  • SAP Sybase Replication Server 15.7, heterogeneous edition

Reproducing the Issue

  1. Load Rep Agent for Oracle (RAO) in a Linux platform
     
  2. Go to $SYBASE/RAX-15_5/bin 
     
  3. ra_admin.sh -cmyrao -p12345 -usa-toracle

Cause

  • The wrong syntax was used to run 'ra_admin'
     
  • The 'ra_admin' option,  '-u', was used incorrectly to identify the user but instead that option is intended to signify an upgade  
     
  • The syntax that triggered the error: ra_admin -c <instance> -p <port_n umber>, -t <type>, -u <admin_user> -pwd <password>
      •  

Resolution

  1. Issue $SYBASE/RAX-15_5/bin/ra_admin.sh -h[elp] to list all options for ra_admin:

    •     -c                                     Creates a Replication Agent instance with name <instance>.

          -p <port>                          is the socket port for the administration connection.

          -t <type>                          is the type of database RepAgent will connect to. One of: oracle, oraclexs, ibmudb, mssql

                                                  If you are not copying an instance, then you must provide the -t <type> option.

          -f <exist_inst>                  is the existing instance from which to copy a configuration file.  PDS parameters and port number are not duplicated.

           <user id>                         is an admin user of a Replication Agent instance.

          -pwd <password>             is the password of the admin user.

          -vr <resource>                 Validates resource file contents without creating instance.

          -r  <resource>                  Creates instance based on resource file contents.

          -v <instance>                   Verifies a Replication Agent instance with name <instance>.

          -d <instance>                   Deletes a Replication Agent instance with name <instance> if not currently running.

          -u <source_dir>                Upgrades all verifiable Replication Agent instances of a previous major release from the source directory

              <source_dir>                 to the product installation directory.

              <source_instance_dir>   Upgrades a verifiable Replication Agent instance of a previous major release from the instance directory

              <source_instance_dir>   to the product installation directory.

              all                                 Upgrades all verifiable Replication Agent instances of a previous minor release in the product installation directory.

              <instance_name> | instance=<instance_name>

                                                  Upgrades a single verifiable Replication Agent instance named <instance_name> of a previous minor release

                                                  in the product installation directory.

         -k <instance>                    Clears the instance_rand configuration property for the Replication Agent instance with name <instance>.

         -l                                       Lists all Replication Agent instances and if they are running.

         -b                                      Displays the Replication Agent base directory.

         -reset <instance> { -uid <i>user id</i> -pwd <i>password</i> }

                                                  Resets the administrator password for the instance with the name <instance>

         -h[elp]                               Displays this usage message.

        -uid

  2. Rerun ra_admin.sh with the correct option:

    • ra_admin.sh -c <instance> -p <port_number>, -t <type>, -uid<admin_user> -pwd <password>

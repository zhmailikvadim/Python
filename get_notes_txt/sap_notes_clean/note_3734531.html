Symptom

In the process "Device Installation/Replacement and Finalization of Technical Master Data" (APEU_B_MCM_TMD_CHANGE_PROCESS), step 0170 calculates the Business Start Date incorrectly. The system sets the Business Start Date to either the first of the next month or the first of the month after next.

This issue is relevant for the market roles: SUP, DSO, MOS, PRS.

Other Terms

BAdI /US4G/CL_IM_BADI_MCM_DATES, /APEU/IF_BADI_MCM_DATES~GET_BUSINESS_DATE, Business Start Date, Device Installation, MCM_DE, SLP, RLM

Reason and Prerequisites

The system incorrectly calculates the Business Start Date in the BAdI /US4G/CL_IM_BADI_MCM_DATES ->/APEU/IF_BADI_MCM_DATES~GET_BUSINESS_DATE.

Solution

Implement the corrections of this SAP Note or import the relevant support package.

After implementing this SAP Note, the system calculates the Business Start Date correctly and no longer sets it to the first of the next month or the first of the month after next.

You need to run the process again and reprocess step 0170. You can verify the correction by creating an MCM change process (SLP->RLM) and checking that the system calculates the Business Start Date correctly.

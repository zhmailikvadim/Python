Symptom

When you change Key Date to display different hierarchy version, some nodes are displayed in section Not Assigned incorrectly.

This happens when a hrrp node exists as a standard node in one version of hierarchy and then it exists as a Debit/Credit node in another hrrp version. Result is that the node is not assigned correctly after Key Date is changed in the Balance Sheet Income Statement - multidimensional.

Other Terms

Balance Sheet Income Statement, multidimensional, W0161, plus minus nodes, credit debit nodes, contra nodes, hierarchy versions, hrrp, KeyDate

Reason and Prerequisites

Program error.

Solution

Implement the attached Correction Instruction or the mentioned Support Package.

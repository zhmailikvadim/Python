Symptom

  • A Webi report is returning the correct number of records in the query but not displaying all records rows in the report.
  • The report settings have been verified to ensure no result set limitation is applied.
  • Other Webi reports are able to display the correct number of records without issues.

Environment

  • SAP Sales Performance Management
  • Incentive Management on Legacy Oracle

Reproducing the Issue

  1. Login to Incentive Management.
  2. Navigate to Reporting & Analytics.
  3. Locate the Webi report
  4. Open the Webi report and enter the prompt.
  5. Observe that fewer records are displayed.

Cause

WebI was treating records as duplicates and removing them from the view because the report was not retrieving all columns from the table. When only a subset of columns was selected, multiple records appeared identical (duplicates) even though they were distinct records in the full dataset.

Resolution

  1. Add more columns to the report to differentiate the records.
  2. Ensure the additional columns contain unique or distinguishing data.
  3. Verify that WebI recognizes the records as distinct rather than duplicates, allowing all expected rows to display in the report view.

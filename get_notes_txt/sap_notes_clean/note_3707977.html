Symptom

  • A runtime error with the type "GEN_SELECTION_SCREEN_ERROR" occurred during ATC execution.
  • The ATC run completed successfully without any errors, as confirmed in transaction SM37.
  • The short dump was observed in transaction ST22 during the same time window as the ATC execution.
  • The program identified as the cause of the short dump is Z*, which showed no errors in SE38 directly related to the short dump.

Environment

ATC, SAP S/4HANA, SAP NetWeaver Application Server for ABAP

Reproducing the Issue

  1. Execute ATC run
  2. Observe the short dump "GEN_SELECTION_SCREEN_ERROR" in transaction ST22 during the ATC execution.

Cause

  • During the ATC execution, a check required the generation of the programs.
  • The program Z* could not be generated due to an overlap on the selection screen, causing the runtime error "GEN_SELECTION_SCREEN_ERROR".
  • The ATC execution triggered the dump because the program needed to be generated.
  • During the ATC execution a check required the generation of the program Z*. The ATC execution triggered the dump because the program needed to be generated. The program can not be generated due to an overlap on the selection screen. The ATC execution only shows the symptom, the cause of the problem is the code of the report.
  • The dump only effects the results for the program Z*, The dump does not effect the ATC results for other programs.
  • If a check needs information from the ABAP Compiler (for instance "Search problematic statements for result of SELECT/OPEN CURSOR without ORDER BY") the program has to be generated in case the load is not up to date. This was the case with program Z*. This should explain why the the generation was attempted.

Resolution

  1. Regenerate the program Z* via ABAP program RSDBGENA or by activating the object in SE38.
  2. Execute ATC run again.

Symptom

If a user has checked out a document, then no other user can change the document because of check out lock. But it is possible to unlock and override the document by a release of the document from a child branch.

Other Terms

Check Out Lock, Release, Release Check

Reason and Prerequisites

The check out lock is not proofed by the release check.

Solution

Implement the current correction instruction with transaction SNOTE.
The check out lock is then recognized by release check. Independent which user does the release, the release is forbidden if a document is checked out.

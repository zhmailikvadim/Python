Symptom

When using the GR/IR Reconciliation Fiori app, you expect machine learning–based Data Attribute Recommendation (DAR) functionality to provide automatic recommendations and completion of relevant GR/IR attributes. With the introduction of SAP‑RPT‑1 (Relational Prompting), DAR will continue to be supported for existing customers during a migration period; however, DAR will no longer be available for new customer activations after April 1, 2026 (https://discovery-center.cloud.sap/serviceCatalog/data-attribute-recommendation?region=all)

Other Terms

GR/IR Reconciliation, Fiori, DAR, Machine Learning, SAP‑RPT‑1, AI Core, BTP, RFC Destinations, GR/IR ML Setup, F3302

Reason and Prerequisites

With recent innovations in SAP AI Core and the introduction of the SAP‑RPT‑1 scenario, the legacy DAR (Data Attribute Recommendation) service will no longer be used by the GR/IR Reconciliation app for recommendations. Therefore, the app expects the new customer to configure the SAP‑RPT‑1 model scenario on the SAP Business Technology Platform (BTP). The SAP‑RPT‑1 feature is not provisioned automatically. To make the recommendation capabilities available in the GR/IR Reconciliation app, the backend system must run on S/4HANA 2020 (or higher) and the following prerequisites must be fulfilled:

1. Access to an SAP BTP account

  • An SAP BTP subaccount with entitlement to SAP AI Core is required.
  • More information: https://help.sap.com/docs/btp and https://help.sap.com/docs/btp/sap-btp-cockpit/create-subaccount

2. Activation of SAP AI Core

  • Enable SAP AI Core in the chosen subaccount.
  • More information: https://help.sap.com/docs/sap-ai-core

3. Creation and deployment of an SAP‑RPT‑1 scenario

  • Create a scenario in SAP AI Core, deploy SAP‑RPT‑1, and ensure the deployment is active and reachable.
  • More information: https://help.sap.com/docs/sap-ai-core

4. Retrieval of credentials from the AI Core deployment

After deploying, collect the following:

  • OAuth 2.0 Client ID: clientid
  • Client Secret: clientsecret
  • Token Endpoint: url + /oauth/token
  • AI Core API URL: serviceurls.AI_API_URL (host used in SM59)
  • Deployment ID: the unique identifier of your SAP‑RPT‑1 deployment (required for Step 7 Path Suffix; case-sensitive)
  • Resource Group: the resource group of the deployment (often default; case-sensitive)

Save these values for later; they are required for configuration in the ABAP system. Identityzone and identityzoneid are informative and typically not needed for this setup.

5. Setup of OAuth 2.0 client profile

  • Use transaction SOAUTH2_CLIENT. If unavailable, use OA2C_CONFIG.
  • Service Provider Type: DEFAULT
  • Profile: /IWXBE/MGW_MQTT
  • Configuration: <Profile name> (used later in SM59), e.g., OAUTH-PROFILE-RPT-1
  • OAuth 2.0 Client ID: value "clientid" from Step 4 (e.g., sb-a1692eb4-6f3c-442e-9921-d3fea4fa6840!b947642|xsuaa_std!b318061)
  • Token Endpoint: value "url + /oauth/token" from Step 4 (e.g., test-GRIR-finance.authentication.eu12.hana.ondemand.com/oauth/token)
  • Refresh Token Validity: 0 days (select Unlimited)
  • Clock Skew Tolerance: 5 seconds
  • Resource Access Authentication: Header Field
  • Client Authentication tab: Form Fields > Client Secret = value "clientsecret" from Step 4
  • Grant Type: Client Credentials

6. Setup of RFC destinations in transaction SM59

  • Create an HTTP connection to external server (Type G) to connect to SAP AI Core.
  • Destination name: remember it for later (e.g., RFC-RPT-1).
  • Target Host: the host from Step 4 under serviceurls -> AI_API_URL (e.g., api.ai.intprod-eu12.eu-central-1.aws.ml.hana.ondemand.com). Enter it without the http/https prefix.
  • Port: 443
  • Path Prefix: leave empty
  • OAuth Profile: the one created in Step 5 (e.g., OAUTH-PROFILE-RPT-1)
  • SSL: select an appropriate SSL client PSE (e.g., SSL Client (Standard))

7. Execute report FIS_GRIR_ML_DESTINATIONS_SETUP

After the RFC destination is created, run report FIS_GRIR_ML_DESTINATIONS_SETUP, so the Fiori app can detect and use it.

Parameters:

  • RFC Destination: the value from Step 6 (e.g., RFC-RPT-1)
  • Path Suffix: the deployment ID from Step 4 in the format /v2/inference/deployments/<deploymentId> (e.g., /v2/inference/deployments/d7dfe3cf7c2f9ed3). Note: case-sensitive.
  • Resource Group: from Step 4 (commonly default). Note: case-sensitive.
  • Model Size: depending on the model deployed in Step 3, enter 2000 (SAP‑RPT‑1 Small) or 65000 (SAP‑RPT‑1 Large)

Only after these configuration steps are correctly executed, the GR/IR Reconciliation app will be able to use the SAP‑RPT‑1 features.

Security reminder: clientsecret is sensitive. Store it securely, avoid sharing it in channels that may be logged, and rotate it if you suspect exposure.

Migration reminder: If you have successfully migrated to SAP‑RPT‑1, please unschedule any scheduled periodic runs (background jobs) for the DAR reports FIS_GRIR_ML_DAR_TRAIN, FIS_GRIR_ML_DAR_INFERE, and FINS_ML_DAR_MONI.

Solution

With the implementation of this SAP Note 3730452, the GR/IR Reconciliation app supports the SAP‑RPT‑1 scenario, replacing the DAR service for recommendations. This setup provides additional features via SAP AI Core but requires a one‑time technical configuration by customers. Once the AI Core scenario is deployed and connected, the app can fully use SAP‑RPT‑1‑based recommendations. Customers who previously relied on DAR should migrate to the SAP‑RPT‑1 configuration to continue receiving recommendations.

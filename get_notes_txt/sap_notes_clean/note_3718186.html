Symptom

  • The FLP screen appears empty (no tiles or groups visible).
  • The error persists even when explicitly adding ?sap-spaces=false to the URL. 
  • Upon logging into the Fiori Launchpad (FLP), users receive the error message: "Failure - Unable to load groups."
  • Browser Console logs show hundreds of errors related to "CHIP instances" and "PAGE_BUILDER_PERS" service failures.
  • In a Network Trace (HAR file), the start_up OData response might show a conflict where spacesMyhome is false but the framework flag spacesEnabled is still returning true.

Environment

SAP S/4HANA 2021

Cause

There are two primary levels where this failure occurs:

  1. Framework Mismatch (Configuration): The FLP engine is stuck in "Spaces" mode despite global settings being set to "Classic." This happens when user personalization or role attributes override the global SPACES = false setting.

  2. CHIP Catalog Corruption (Database/Code): A known program error in the CHIP catalog cache mechanism causes the system to attempt to insert duplicate primary keys into the /UI2/CHIP_CSTAT or /UI2/CHIP_CPAR tables. This crashes the Page Builder service before it can render the "Classic" tiles.

Resolution

Phase 1: Validate Configuration (The "Stage and Curtains" Logic) Ensure the system is not trying to load the "Spaces Stage" without any content.

  1. Verify Transaction /UI2/FLP_CUS_CONF (and /UI2/FLP_SYS_CONF):

    • Set SPACES = false

    • Set SPACES_MYHOME = false

  2. Check for Role-Based Overrides: Ensure no PFCG roles assigned to the user contain a "Space ID" or "Page ID" in the Menu tab.

Phase 2: Fix CHIP Corruption (The Permanent Solution) If Phase 1 does not resolve the issue, the CHIP catalog is likely corrupted.

  1. Implement SAP Note 3295872 - Fiori Launchpad: Duplicate key in CHIP catalog cache.

    • This note corrects the logic in class /UI2/CL_CHIP_CATALOG_CACHE.

  2. Execute the following reports in order via SE38 to scrub the corrupted data:

    • /UI2/CHIP_SYNCHRONIZE_CACHE (Syncs the CHIP catalog)

    • /UI2/INVALIDATE_GLOBAL_CACHES (Clears server shared memory)

    • /UI2/INVALIDATE_CLIENT_CACHES (Clears user browser cache)

Phase 3: Database Verification (Advanced) If the reports fail, Basis/Security should check technical table /UI2/V_CONFPROP to ensure the SPACES_ENABLEMENT flag is not hard-coded to true at a layer that overrides standard configuration.

Symptom

When I send a QuoteRequest document from my SAP ERP system, it fails in Managed Gateway for Spend&Network. I see the following error message:

Error: Internal Server Error. Please contact support with the Error Reference Number: XXXXXXXXXX for more details. Suggested Action: Document failed in Business Network. Error Code: CIG-PLT-00644. Please review the Support Note 189049 for solution.

Environment

  • Managed Gateway for Spend&Network
  • SAP Ariba Sourcing (SRC)

Cause

You see this error because the matchingType attribute within the <SupplierSelector> tag of the outgoing cXML document is missing or blank. This attribute is mandatory for SAP Ariba Sourcing to determine how to process the Request for Quotation (RFQ) event. This is caused by a missing configuration in the Lookup Table in Managed Gateway for Spend&Network.

Resolution

To resolve this issue, you need to configure the Lookup Table to correctly map the values from your ERP system to the required Ariba values.

  1. Log in to the Managed Gateway for Spend&Network portal.
  2. Navigate to My Configurations > Cross References.
  3. Select the Lookup Table tab.
  4. From the Select Document Type dropdown, choose QuoteRequest and ensure it is added to your project and find the template named QuoteRequestMatchingType.

Now, you must map your ERP system's value to the correct Ariba Value. The standard values are:

  • public: To specify any public supplier.
  • approvedVendorOnly: To specify suppliers that exist on SAP Business Network and are approved.
  • invitationOnly: To specify suppliers that are invited to the RFQ posting.
  1. After adding or correcting the mapping, click Save.
  2. Deploy the changes to your environment.
  3. Resubmit the QuoteRequest document.

Following these steps ensures you correctly populate the matchingType attribute in the cXML, resolving the internal server error.

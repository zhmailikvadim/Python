Symptom

Service contract change process(sold-to party) fails with 'Error making changes in module CRM_ORDER_MAINTAIN' message log when using custom Z code Service Contract transaction type.

Environment

  • Product: SAP S/4HANA Cloud Private Edition
  • S4CRM: Service Contract

Reproducing the Issue

  1. Note 3671438 - Change Process Message Log is displaying unnecessary error messages has been implemented.
  2. Open/Create Service Contracts with custom Z code transaction type.
  3. Release the service contract.
  4. Start the Change Process.
  5. Update new the sold-to party
  6. Run the change process and observe that an error occurs and the change is not applied.

Cause

The custom Z code service contract transaction type used for the document is not maintained in customizing <Settings for the Maintenance Framework (BTMF)->Activate Change Process->CSLT->Define Transaction Types in Process. 

Resolution

  1. In Customizing (SPRO), navigate to: Service → Transactions → Settings for Subscription Transactions → Contract Changes → Settings for the Maintenance Object (BTMF).
  2. Navigate to Activate Change Process.
  3. Select the change process CSLT.
  4. In “Transaction Types in Process”, add the relevant custom service contract transaction type used for the affected contracts.
  5. Save the change.

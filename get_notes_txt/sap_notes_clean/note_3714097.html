Symptom

  • SAP Business One setup (e.g. Integration Framework / Upgrade Wizard) fails to connect to the HANA database although correct credentials are entered.
  • The setup may show a generic login/database message (e.g. “Failed to connect… verify SBOCOMMON…”), but the setup/ODBC logs contain:
    [SAP AG][LIBODBCHDB DLL][HDBODBC] … Communication link failure; -10709 … (RTE:[1000013] Key not valid for use in specified state)
  • The issue can be reproduced using Windows ODBC Data Source Administrator when connecting with TLS/SSL enabled.

Environment

SAP HANA DB Version 2.0

Cause

The reported error occurs during initialization of a TLS/SSL-protected connection using the SAP HANA ODBC driver.

The connection attempt fails before a database session is established and is reported by the SAP HANA ODBC client with the message:

Communication link failure; -10709 (RTE:[1000013] Key not valid for use in specified state)

This indicates that the connection is terminated during the encrypted communication setup on the client side before authentication to the SAP HANA database is completed.

As the failure occurs at the client connection layer, the SAP HANA database server may not record a corresponding SQL connection or application-level error.

Resolution

  • Ensure that the SAP HANA client version used by the SAP Business One setup is supported and up to date.
  • Execute the installation on a supported and fully patched Windows Server system and verify that encrypted connections to SAP HANA can be successfully established using the HANA ODBC driver before running the setup.
  • If the encrypted ODBC test connection already fails with error -10709, the issue occurs on the client host during TLS/SSL connection setup rather than on the SAP HANA database server.
  • After the encrypted ODBC connection succeeds, re-run the SAP Business One setup.

Symptom

  • During the MAIN_SHDRUN/ACT_UPG phase, activation of DDL sources fails for a custom CDS extend (for example, ZZ1_*) and for the standard view C_CNSLDTNFINSTMNTITEMASSGMTTP.
  • SDDLAR returns error “Repair cannot be executed during upgrade” (DDL_REPAIR097).
  • RUTDDLSDEL reports “DDL Source
    • The extend references custom columns that no longer exist in the underlying table.

Environment

  • Product: SAP S/4HANA
  • Custom Fields

Reproducing the Issue

  1. Start an upgrade using SUM and reach the MAIN_SHDRUN/ACT_UPG phase on the shadow instance.
  2. Observe activation failures for a custom CDS extend and dependent standard views (for example, C_CNSLDTNFINSTMNTITEMASSGMTTP).
  3. Try to repair via SDDLAR and get DDL_REPAIR097.
  4. Try to delete the DDL source via RUTDDLSDEL and get DO779 stating the TADIR entry still exists.

Cause

  • Known limitation during multi-release upgrades where some applications moved to a new technology stack, requiring adjustments to generated custom field extensions.
  • The custom CDS extend references custom fields that no longer exist in the base table, causing activation to fail; dependent views then fail as a consequence.

Resolution

  1. Do not delete or modify Custom Fields artifacts in the shadow system; deletion in shadow is not recommended due to system restrictions.
  2. Roll back the upgrade.
  3. In the source system, identify usages of the following views in Custom Fields by checking database table CFD_W_REP_CDS: C_BUSSOLNORDITMLISTRPTANDOBJPG, C_BUSSOLNORDLISTRPTANDOBJPG, C_CNSLDTNFINSTMNTITEMASSGMTTP, C_CNSLDTNFINSTMNTITEMTP_2.
  4. Note the affected custom fields and temporarily disable the usage of these views in those custom fields; save and publish.
  5. Restart the upgrade.
  6. After the upgrade, re-enable the usage of the views in the previously affected custom fields; save and publish.
  7. Preventive checks before future upgrades:
  8. For Custom Fields: ensure the latest notes are applied (for example, 3658670); run report CFD_REGENERATE_IN_ONPREMISE in simulation mode and perform regeneration if simulation reports errors.
  9. For Datasource Extensions: run report CFD_DE_REGENERATE_IN_ONPREMISE in simulation mode and perform regeneration if simulation reports errors.

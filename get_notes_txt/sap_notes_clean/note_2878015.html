Symptom

  • Web Intelligence fails with below errors while trying to create report based on Bex Queries

    “Failed to retrieve the cube for connection XXXX. (Error: INF DSL 00000)”

  • Existing webi reports fails with a same error on alternate refresh.
  • This issue is only observed on 1st refresh, From 2nd refresh it works fine
  • Issue exists in all viewer mode(HTML/Applet/Webi Rich Client)
  • Works fine in Analysis for Office, Analysis edition for OLAP , Query designer and RSRT.

  • Adaptive Processing Server DSL logs show below error

    com.sap.ip.bi.base.application.exceptions.AbortMessageRuntimeException: Termination message sent &#10;ABEND RSBOLAP (000): Program error in class SAPMSSY1 method : UNCAUGHT_EXCEPTION&#10; MSGV1: SAPMSSY1&#10; MSGV3: UNCAUGHT_EXCEPTION&#13;&#10;ABEND BRAIN (299): System error in program SAPLRRH1 and form RRH1_TLEVEL_READ-3-&#10; MSGV1: SAPLRRH1&#10; MSGV2: RRH1_TLEVEL_READ-3-

Other Terms

webi, web Intelligence, SAP BW,bex,failed,cube,DSL 00000,retrieve the cube for connection,BICS,refresh,ABEND BRAIN (299),RRH1_TLEVEL_READ-3

Reason and Prerequisites

Environment:

  • SAP BusinessObjects Business Intelligence Platform 4.2
  • Web Intelligence (Webi)
  • OLAP BICS
  • SAP BW 7.x

Steps To Reproduce:

  1. In the CMC, create an OLAP connection based on an SAP BW system and save it.
  2. From BI Launchpad/Web Intelligence Rich Client, create a new Web Intelligence document based on this OLAP connection.
  3. When the "Select an SAP BW BEx query" window appears, search for a known BEx Query, highlight it and click OK.
  4. Observe below error appears on Screen
    “Failed to retrieve the cube for connection XXXX. (Error: INF DSL 00000)” 

  5. Same issue is observed if you refresh the existing Web Intelligence document for the first time 

Cause:

  • There is some inconsistency for the hierarchy that leads to the error.

Solution

  • Apply note 1984939, execute report RSR_REPAIR_INFOAREA_HIERARCHY in transaction SE38.

Symptom

When converting planned order to production order in Convert Planned Order - to Production Order (CO40) app, error message CO367 "Number range & is not created for object &" occurs.

Environment

SAP S/4HANA Cloud Public Edition

Reproducing the Issue

  1. Go to Convert Planned Order - to Production Order app.
  2. Enter Planned Order and Order type and execute.
  3. Error message CO367 occurs.

Cause

The number range assigned to the production order type is not defined in configuration activity 106461 - Configure Number Range for Order Types. 

Resolution

  1. Check Prod. Sched. Profile in material master -> Work Scheduling tab.
  2. Check default order type in configuration activity 102508 - Define Production Scheduling Profile for Discrete Manufacturing.  
  3. Check the relation between number range and order type in configuration activity 102004 - Define Production Order Types.
  4. Insert the specified number range in configuration activity 106461 - Configure Number Range for Order Types.

Keywords

Symptom

The textSearch method in Class DefaultSolrProductSearchFacade does not return expected result after removing feature toggle "toggle.product.searchResultFilter.enabled" in release 2211-jdk21.5.

Reason and Prerequisites

We've removed the feature toggle "toggle.product.searchResultFilter.enabled" in release 2211-jdk21.5:
https://help.sap.com/docs/SAP_COMMERCE_CLOUD_PUBLIC_CLOUD/251a1313585a498eb998c2f974f4f338/7fba68ad23fc41ce83e8ef5e86fb1491.html


The impact of this change is that after retrieving products from Solr Search, the system will now filter out (remove) any products that cannot be accessed from the database. There are multiple scenarios where product data present in Solr Search might be inaccessible from the database. The common example is when products are returned from Solr Search but cannot be accessed due to search restrictions.


Following the removal of this toggle, we have received complaints from some customers indicating that their business processes were disrupted. Our investigation revealed that not all customers expect or require this filtering behavior.

Solution

Offer a permanent configuration option that allows customers to enable or disable the filtering of products (retrieved from Solr Search) within the textSearch method of Class DefaultSolrProductSearchFacade.

Add Permanent Property and default is true, user can make use of below property to control enable or disable this functionality of search result filter:

commerceservices.product.searchResultFilter.enabled=true

The fix will be available in Commerce version 2211 and in the following patches:

2211-jdk21.10

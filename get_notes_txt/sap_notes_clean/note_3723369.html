Symptom

I am trying to log in via Single Sign-On, but I receive an error message stating 'You are not authorized to use single-sign-on.' My IT team has confirmed the SSO configuration is active, and the SSO Login History in the Configuration Manager shows my login attempts with the specific error: 'User is not authorized to use single sign-on. Please have your SAP Fieldglass administrator check the user account to confirm the Single Sign-On flag is enabled.

Cause

This error occurs when the SAML assertion sent by the Identity Provider (IdP) is technically valid and successfully consumed by SAP Fieldglass, but the specific user account being identified does not have the Single Sign-On authentication flag enabled. Even with a perfect SSO setup at the company level, each individual user must be authorized to use this login method.

Resolution

An administrator must enable the Single Sign-On flag on the impacted user's profile to allow them to bypass standard password authentication.

Step 1: Locate the User Account

  1. Log in to SAP Fieldglass with administrative privileges.
  2. Navigate to Admin > Users.
  3. Search for the user identified in the SSO Login History error log.

Step 2: Enable the SSO Flag

  1. Open the user's profile and select Edit.
  2. Navigate to the Account Setup or Authentication section.
  3. Locate the Single Sign-On checkbox.
  4. Ensure this flag is selected (checked).
  5. Save the changes.

Step 3: Verify Username Mapping

  1. While in the user's profile, ensure the Username field exactly matches the value being sent in the SAML assertion (either the NameID or the mapped Attribute).
  2. If the values do not match, the system will be unable to find the user record and will trigger an authorization failure.

See Also

  • "Both" Authorization Type: If your company is configured with the login type Both (Username/Password and SSO), the user can still log in with a password if the SSO flag is disabled, but they will fail the SSO handshake until the flag is checked.
  • Bulk Updates: For large groups of users, this flag can be enabled in bulk using the User Upload integration connector by setting the SSO field to 'Yes'.

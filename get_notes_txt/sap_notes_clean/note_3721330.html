Symptom

In an ABAP system, an application uses a proxy to create or receive asynchronous XI messages with Quality of Service(QoS) Exactly Once (EO). Processing of XI messages is triggered by scheduled entries in a queue with prefix XBTS or XBTR. You process XI messages with different interfaces in the same queue. If a serious error occurs during processing for an interface, the queue remains and the messages are affected by all interfaces. In the local ABAP proxy system, it is possible to separate the XI messages with QoS EO interface-specifically into separate queues. This function was introduced with the corrections from SAP Notes 18022294 and 1831889. For more information about the configuration, see the SAP Note descriptions. You have used the configuration as described and no separate queue set is generated for the interface.

Other Terms

Queueset, interface, sender, receiver

Reason and Prerequisites

You use SAP Process Integration as a local Integration Engine in an application system. You have the problem described above. This is not a program error, but a configuration error.

Solution

Implement the correction instructions or import the relevant Support Package.

When you implement the corrections, the new report SXMS_PARTNER_WIZARD is implemented.
A configuration for the determination of an interface-specific queue set must be made in various steps. The report SXMS_PARTNER_WIZARD helps to create all configurations step by step in the correct sequence. A queue set with a predefined parallelism is to be generated for each partner ID. A partner ID is generated with the data from the main header of the example XI message and the configuration for the sender queue set or the receiver queue set is generated using the pipeline ID. 

If you execute the report SXMS_PARTNER_WIZARD, you can use the "Documentation" button to find a detailed description of the functions.

You need the following authorization to run the report:

  • Object S_XMB_AUTH
  • Area: CONFIG
  • Activity:
    • 02 (Change mode & display mode)
      This activity is only necessary if a new configuration is to be created.
    • 03 (display mode)

In the standard case, this authorization is contained in the single role SAP_XI_ADMINISTRATOR_ABAP and in the composite roles derived from it.

Symptom

During JIT Delivery Confirmation processing via EDI for requests containing multiple items, some items may be processed successfully and the Delivery Confirmation document is already created, while the remaining items fail during validation or processing.

Although the document is already partially saved successfully, the Message Monitoring application still displays the overall request status as Error.

As a result, the status incorrectly suggests that the complete request should be reprocessed.

Reprocessing such requests can lead to duplicate Consignment issue orders for the already processed items.

Other Terms

NJIT, Next Generation Just-In-Time Processing, JIT Delivery Confirmation, Self Billing, Consignment Issue Order, Message monitoring status

Reason and Prerequisites

Program error

Solution

Implement the correction instructions provided in this SAP Note or import the relevant Support Package to enable the corrected system behavior.

The system behavior has been corrected for partially processed Delivery Confirmation EDI scenarios.

If the Delivery Confirmation is already created successfully for at least one item, the remaining item-level processing issues are now converted from Error messages to Warning messages before they are passed to Message Monitoring.

As a result:

  • The overall monitoring status is displayed as 'Warning'
  • Users can identify that the request is already partially processed
  • Unnecessary reprocessing is avoided
  • Duplicate Delivery Confirmation creation is prevented

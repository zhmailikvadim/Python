Symptom

When creating an external sales order from SAP Sales Cloud Version 2, partner roles maintained on the sales quote are not replicated as expected to the resulting S/4HANA Sales Order. The following issues may occur:

  • The contact person (AP) is present in the outbound payload from Sales Cloud v2 but does not appear in the S/4HANA sales order.
  • A buyer (EK, custom role) maintained on the account/quote may be absent from the outbound payload and therefore not created in S/4HANA.
  • No error messages are raised in Sales Cloud v2 or in the integration flow.

Environment

  • SAP Sales Cloud Version 2
  • SAP Sales/Service Cloud Integration (GROW integration content)
  • S/4HANA Public Cloud

Reproducing the Issue

In Sales Cloud v2, open an account containing:

•    a contact person relationship
•    a buyer (custom role) relationship


1.    Create a Sales Quote, add at least one product, simulate pricing, and create an external sales order.
2.    In S/4HANA, open the resulting sales order and navigate to the Partner section.
3.    Observe that the contact person (AP) and buyer (EK) roles are missing.

Cause

This behavior is due to known limitations in the S/4HANA Public Cloud V2 API:

  1. Partner function support is restricted to customer‑related partner functions.

    • Contact persons, buyers, and other non‑customer partner roles are not supported, because partner functions in V2 are language‑dependent.

  2. Value mapping is not delivered for unsupported partner functions within the GROW integration content.

    • As a result, Sales Cloud v2 cannot successfully map or pass such partner roles to S/4HANA.

  3. For custom roles such as buyer (EK):

    • Missing value mapping in the integration flow
    • Missing external ID mapping for the linked contact

These can prevent the role from appearing in the outbound payload.

Resolution

1. Acknowledge current S/4HANA Public Cloud V2 API limitations

  • Contact person (AP) and other non‑customer roles cannot be replicated to S/4HANA using the V2 API.
  • The GROW integration content does not provide value mapping for these partner functions.
  • These roles will not appear in S/4HANA even if present in the Sales Cloud v2 payload.

2. Plan adoption of the S/4HANA Public Cloud V4 API

The V4 API supports:

  • Language-independent partner functions
  • Replication of contact persons, workforce, and other extended partner roles

Customers should plan an upgrade to the V4‑based integration once available in their landscape.

3. Interim Workaround

Until the V4 API is adopted:

  • Maintain the required partner functions manually in the S/4HANA sales order.

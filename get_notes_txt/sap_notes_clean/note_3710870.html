Symptom

  • When accessing Work Zone on a mobile device, the side navigation bar does not automatically close after selecting a menu that also acts as a folder;
  • Users must manually click the close 'X' button to collapse the navigation bar.

Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.



Environment

  • SAP Build Work Zone Advanced Edition
  • SAP SuccessFactors Work Zone

Reproducing the Issue

  1. Log in to Work Zone via Mobile browser.
  2. Access the side navigation bar.
  3. Select a nested option and click. 
    1. Example: Workspaces, where it expands the other options. Nothing indicates that it has already re-routed the page.
  4. Observe that the page navigates, but the side navigation bar remains open until the close button is clicked manually.

Resolution

This behavior is expected due to the design of the navigation bar and its menu. The top-level navigation (ex: Workspaces) acts as a folder/category, expanding or collapsing upon interaction. On mobile devices, since the navigation bar is on the side and needs to be expanded to see the options, it takes up the whole screen. If the navigation bar closes once a folder is clicked, it won't be able to perform its function of expanding the options (since the side bar itself would be closed). 

Should you think that this behavior can and should be improved, please submit an enhancement request to let the Product Team know -> 2090228 - How to submit enhancement ideas for SAP SuccessFactors Products - SAP for Me

Symptom

When importing the Import User Data (Consolidated File) to change a user's Name , the import completes successfully. 

However, in the UI of different languages, the Name has not been changed.

For example, the site's default locale is English, after the import. the Name is changed in the English UI but is not in the Japanese UI.

Environment

SAP Ariba Core Administration 

Cause

This situation occurs when the user has a translated Name.

The Name field of a User is a Multi-Lingual String (i.e., it stores more than one string for the name, using the Index of the string to determine the language). 

For example, if the site's default locale is English, and a new user is created or an existing user's name is changed in the Japanese UI, the user will be assigned a Japanese translated user Name.

Resolution

In this scenario, the Import User Data (Consolidated File) task will not change the translated Name.

The only way to change the translated Name is through the UI.

Keywords

Update user name, multi-language settings, name change not reflected

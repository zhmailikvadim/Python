Symptom

  • I want to prorate a monthly rate in SAP Fieldglass so that the billed amount reflects only the days or hours a worker actually worked in the month.
  • I currently see that when a monthly Unit of Measure (UOM) rate is used, the full monthly rate is billed even when the worker works only part of the month.

Resolution

To configure a prorated rate, create a calculated rate using system‑provided calculation logic.

This guidance applies to SAP Fieldglass companies using Rates 1.0. Rates 2.0 uses a different rate framework, so the steps below do not apply.

  1. Sign into your SAP Fieldglass account
  2. Click Admin Configuration (gear icon) > Rate
  3. Click New
  4. Select Rate – Calculated as the Type
  5. Select the Unit of Measure (UOM) you prefer for prorating:
    • Day if you want to calculate a daily prorated rate.
    • Hour if you want to derive hourly prorated amounts.
  6. Select System Calculated as the Calculated Type
  7. Choose a Calculation Logic option. Examples include:
    • Base Monthly Rate divided by the number of calendar days
    • Base Monthly Rate divided by the number of Monday through Friday days
    • Base Monthly Rate divided by the number of Worker days in a corporate calendar excluding bank holidays
  8. (Optional) If using Day UOM, tick Use Prorated Day to allow fractional day entries (e.g., 0.5).
    • This option limits workers to entering day‑based decimals instead of hours.
  9. Click Add

Add the new rate to Job Posting Templates so future Job Postings and Work Orders use it automatically.

To add the new rate to existing Work Orders, initiate a Revision.

  • The Creator can add and remove rate codes or rate schedules rule must be set to Yes to add or remove rates during a Work Order Revision.
  • This rule is editable on existing Job Postings via Actions > Edit Job Posting.

Once applied to Work Orders, provide guidance to workers to ensure they select the correct rate when entering time:

  • Use the Monthly rate only when the entire month is worked.
  • Use the Daily or Hourly rate for partial‑month work.
Note: To automate time entry allocation to the correct rate based on your business requirements, SAP can create Time Sheet Rules for your environment. Time Sheet Rules require Rate Schedules.

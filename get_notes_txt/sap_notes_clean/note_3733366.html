Symptom

You create an ELM 5 notification with the data extractor for the uniform wage notification procedure (ELM) (program RPLELMC0) with company-wide work centers for which both the BUR number and the InhouseID were maintained.

In this case, only the tag <BUR-REE-Number> is output in the generated XML. The <InHouseID> is missing. Instead, you want to transfer the InHouseID to the recipient, but you do not have the option to control the system behavior because the system always prioritizes the BUR number.

Other Terms

ELM 5.0, XML, CR_DS_SD_CPY_DSC, work center, Workplace, BUR number, InHouseID, T50BK

Reason and Prerequisites

The official Swissdec ELM XML schema stipulates that either the <Workplace> or the <BUR-REE-Number> must be transferred in the node <InHouseID> (xs:choice). A simultaneous transfer of both values leads to an invalid XML and to the rejection of the notification by the distributor.

In the previous SAP standard, the priority was defined statically: As soon as a BUR number was maintained, this was reported. The InHouseID was only reported if no BUR number existed.

Solution

Import the HR Support Package relevant for your release. If you implement the solution in advance, use the Note Assistant to implement the correction instructions relevant for your release. You can find more information about the Note Assistant on SAP Support Portal at https://support.sap.com/notes.

This SAP Note introduces a new control option. You can now use a new entry in the view HR-B2A: Constants (maintenance view V_T50BK) to specify that the system is to write the <InHouseID> to the XML if the BUR number and InHouseID are maintained at the same time.

Configuration:

To activate this preference, you must maintain the following entry in view V_T50BK (using transaction SM30) (HCM localization = '02'):

Area Doc.Cat. Constant Text Start End CVal
ELM 0005 CPINH Work Center: InHouseID Report Preferred Instead of BUR Number (X = Active) 01.01.1800 31.12.9999 X

Note: The simultaneous display of both tags in the XML is technically not permitted because the official Swissdec XML schema defines these two fields as an "either or" condition (xs:choice).

The InHouseID is a company-internal number or information compilation that defines a work location. The use of the InHouseID requires a previous agreement between the BFS and the company and requires close and direct cooperation with the BFS as part of profiling. The BFS decides which companies are eligible. The BUR number can only be dispensed with if this information is available and of good quality.

Symptom

     During GRDC Data Mapping the following is observed in case the local currency of the Consolidation Unit is equal to the group currency:

  • Amount in local currency is not copied to the amount in group currency during GRDC data mapping.
  • The amount in group currency remains empty.
  • In the GRDC mapping file a mapping for the Output field AmountInLocalCurrency was defined.

Environment

  • SAP Group Reporting Data Collection
  • SAP S/4HANA Cloud Private Edition

Reproducing the Issue

  1. Run a Data Mapping in GRDC for a consolidation unit where local currency of the consolidation unit is equal to the group currency.
  2.  In the GRDC mapping file a mapping for the output field AmountInLocalCurrency was defined. The expectation is, that the amount in group currency is copied from the amount in local currency.
  3. Observe, that the amount in local currency is not copied to the amount in group currency during GRDC data mapping. The amount in group currency remains empty/zero.

Cause

  • Missing SAP Notes
  • Logic for the derivation of the Amount in Group Currency is not clear

Resolution

Implement the following SAP Notes, if valid for your software level:

a. SAP Note 3690338 - GRDC / Import Reported Data: the LC value is not copied into GC value.

b. SAP Note 3676457 - GRDC - zero group currency value is overwritten with non-zero local currency value.

c. SAP Note 3665868 - GRDC: the local currency value overwrites the group currency value.

d. SAP Note 3651351 - GRDC: Data line is not posted when local currency value is 0 and group currency value differs from 0.

After implementing the SAP Notes, perform a new GRDC mapping run.

Verify that the local currency value and group currency value are updated according to the following logic:

When the LC key (local currency of the Cons.Unit) is the same as the GC key (for the Cons.version):


a) When the GRDC mapping Excel file does not contain a mapping for the field AmountInGroupCurrency but only for the AmountInLocalCurrency: -> The GC value is copied from the LC value. 


b) When the GRDC mapping Excel file contains both, a mapping for the fields AmountInLocalCurrency and AmountInGroupCurrency, but the value for the AmountInGroupCurrency is empty or zero: -> The GC value is not copied from the LC value but stays empty/zero.  (Remark: in that scenario a mapping for the field GroupCurrency is obligatory!)

Remark: For the flexible upload the logic is the same:

When the LC key (local currency of the Cons.Unit) is the same as the GC key (for the Cons.version):

 a) When the upload file does not contain the field KSL/AmountInGroupCurrency but only the HSL/AmountInLocalCurrency: -> The GC value is copied from the LC value delivered in the file.

 b) When the uploadfile contains both, the fields HSL/AmountInLocalCurrency and KSL/AmountInGroupCurrency but the value for KSL is empty or zero. -> The GC value is not copied from the LC value but stays empty/zero.

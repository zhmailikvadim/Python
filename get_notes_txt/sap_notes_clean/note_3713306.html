Symptom

  • SAPUI5 HTTP Request Handler compression is off in Personas health check after system update.
  • SAPUI5 HTTP Request Handler indicator shows red status.

Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.

Environment

SAP Screen Personas

Reproducing the Issue

  1. Apply Support Pack update to the Environment.
  2. Navigate to transaction /PERSONAS/ADMIN.
  3. Go to Tools > Health Check > Services Status.
  4. Observe that SAPUI5 HTTP Request Handler compression value is set to "off".

Cause

The red SAPUI5 HTTP Request Handler indicator means that compression is not turned on for service ui5_ui5 at the mentioned path. 

Resolution

  1. Navigate to transaction SICF.
  2. Locate the service "ui5_ui5" at the specified path, and double click on the relevant service.
  3. Change the compression setting to "Yes".
  4. Verify that the SAPUI5 HTTP Request Handler indicator turns green in the Health Check under transaction /PERSONAS/ADMIN.

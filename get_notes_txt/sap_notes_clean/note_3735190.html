Symptom

On an iOS device with the country set to Saudi Arabia and language to English, Date, DateTime and Time fields are displayed using the Hijri calendar.  Dates should be displayed in Gregorian as they are on Android

Other Terms

  • SSAM 2505
  • Localization
  • Calendar
  • MDK <= 25.6.2 

Reason and Prerequisites

Reason:

Issue is caused by a defect in MDK, that does not recognize the correct calendar to use for this combination of Region and Language

Prerequisites:

  •  BAS or Visual Studio Code with SAM MDK project
  • `patch` command (available on BAS and macOS by default) via terminal or other patching utility
  • Read the guidelines on how to apply the patch and read the appendix section if you receive any error

Solution

To apply the patch, download the attached SAPNote-3735190.patch file and save it in a known location, preferably adjacent to your SAM.mdkproject.

If you are on Windows, see SAP Note 3533879 for how to install a .patch file in Windows. If you are on a Mac, follow the steps below:

  1. Open a terminal in the root of your SAM metadata (i.e. SAM.mdkproject/metadata or SAM.mdkproject/metadata/base.definitions)
  2. Run the terminal command `patch --no-backup-if-mismatch -p4 < path/to/Note-Number.patch`. 
  3. Verify in the terminal window that the files have been modified and listed as patched, as indicated by the console output
  4. Bundle and deploy the application

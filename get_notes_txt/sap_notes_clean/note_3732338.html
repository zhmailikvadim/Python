Symptom

In SAP Solution Manager 7.2, LMDB synchronization fails during Infrastructure Preparation with the following error: 

Connection error; check 'LMDB_SYNCDEST1' under SLD Connections (error code: (411) Connect to Hostname<XXX>:Port<XXX> failed: NIECON) 

Environment

SAP Solution Manager 7.2

Reproducing the Issue

Navigate to SOLMAN_SETUP -> Infrastructure Preparation -> Step 1 : LMDB Synchronization

Cause

The error “(411) Connect to [hostname]:[port] failed: NIECONN_REFUSED” indicates that the SAP Solution Manager (LMDB) is unable to establish a connection to the System Landscape Directory (SLD) because the connection is being actively refused. This is typically caused by incorrect destination settings, an inactive SLD service, or network/firewall restrictions.

Resolution

To troubleshoot this type of error please ensure the below point first:

  • Verify if the SLD service is up and running on host [Hostname] and Port Number
  • Check network connectivity between Solution Manager and the SLD host (firewall, port accessibility)
  • Test the RFC/HTTP destination LMDB_SYNCDEST1 in SM59
  • Ensure that the URL /sld/cimom is accessible via browser
  • Confirm if there were any recent changes at OS/network level or SSL/ICM configuration
  • Ensure no firewall is blocking port 50500 between Solman ABAP and SLD Java. Confirm that the hostname Hostname is properly configured.

Additionally you can refer to the KBA 0002406101 – Error message 'Check URL: Cannot reach URL (HTTP code 411: Connect to XXXXX:0 failed' in step 'Define HTTP Connectivity' – SAP Solution Manager 7.2

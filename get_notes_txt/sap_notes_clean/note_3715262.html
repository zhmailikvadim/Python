Symptom

When rehiring a contingent worker using the existing employment, the Original Start Date field is updated with the new hire date instead of retaining the initial hire date.

Environment

SAP SuccessFactors Employee Central - Contingent Worker

Reproducing the Issue

  1. For a contingent worker, the Original Start Date initially reflects the first hire date.
  2. The worker is then terminated and becomes inactive (for example, effective February 28).
  3. When the worker is rehired immediately after termination (for example, with a rehire date of March 1), the Original Start Date does not retain the initial hire date.
  4. Instead, the Original Start Date is updated to the new hire date.

Cause

As per the backend logic, the Original Start Date value is updated based on the Work Order Start Date.

Resolution

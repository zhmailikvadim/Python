Symptom

In transaction SU22, you can deactivate the inheritance for a variant transaction.

Reason and Prerequisites

Program error

Solution

Implement the correction instructions using the Note Assistant or import the relevant support package.

Explanation:

Parameter and variant transactions are specializations of their corresponding base transactions. Therefore, it makes sense to automatically add the authorization defaults of the basic transactions of the role when merging after including parameter transactions or variant transactions in a single role menu to ensure that the parameter or variant transactions are always executed without errors. If inheritance mode is active, the default values of those authorization objects are copied from the basis transaction for which there are no separate default values in the respective parameter or variant transaction.

With regard to deactivating inheritance, parameter transactions and variant transactions behave differently:

 Parameter Transactions

  • Since these transactions always run in their own check context, their default data can be completely independent of the corresponding base transactions. Therefore, the deactivation of the inheritance mode is allowed.

Variant Transactions

  • With the exception of the start authorization check, a variant transaction is executed in the check context of its base transaction. The data transfer from the base transaction is therefore important for authorization objects without their own default values in the variant transaction. For this reason, the inheritance mode for a variant transaction cannot be deactivated.

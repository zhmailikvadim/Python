Symptom

Developers or administrators require a way to programmatically access, retrieve, or interpret View Cluster metadata. While transaction SE54 provides a manual interface for viewing these definitions, there is a lack of documentation on how to extract this information via standard code. 

Environment

  • View Maintenance Tool

Reproducing the Issue

Cause

The method is unknown.

Resolution

To programmatically retrieve the metadata of a View Cluster, use the released function module VIEWCLUSTER_GET_DEFINITION. 

Input:

Pass the technical name of the View Cluster to the import parameter VCLNAME.

Output Parameters:

The function module populates the following parameters with the cluster's configuration details:

ParameterAssociated TypeDescription
VCLDIR_ENTRY (Export)V_VCLDIRHeader Entry
VCLSTRUC_TAB (Tables)V_VCLSTRUCObject Structure
VCLSTDEP_TAB (Tables)V_VCLSTDEPField Dependencies
VCLMF_TAB (Tables)V_VCLMFView Cluster Events

Exceptions:

The following exceptions should be handled to ensure robust error processing:

  • VIEWCLUSTER_NOT_FOUND: The specified View Cluster does not exist.

  • INCOMPLETE_VIEWCLUSTER: The definition of the View Cluster is inconsistent or missing mandatory components.

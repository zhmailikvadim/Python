Symptom

During startup SAP Commerce lists all typesystemnames stored in the system and identify the one that is currently used to determine what data model will be used. If obtained value of current typesystemname is wrongly aiming default it may cause inconsistency between application logic and data model.

Environment

SAP Commerce Cloud

Cause

The current typesystemname value is based on the data in CCV2_TYPESYSTEM_MIGRATIONS db table. It can be found via following SQL query:

select name from [dbo].[CCV2_TYPESYSTEM_MIGRATIONS] where state='current'

Current typesystemname on SAP Commerce Cloud in Public Cloud (CCv2) is being changed during deployments with migrate data however it is never reverted to default (which is original value). The possible explanation why current typesystemname has default value is the change performed manually or via Commerce Migration Toolkit (CMT) which is used for data migration between systems. If the change was performed during CMT activity the column comment of record from CCV2_TYPESYSTEM_MIGRATIONS table contains "Updated by CMT" value. To verify if the typesystemname value was changed to current by CMT following SQL query can be used:

select comment from [dbo].[CCV2_TYPESYSTEM_MIGRATIONS] where state='current' and typesystemname='default'

Resolution

To avoid unexpected reset of current typesystemname the CMT shouldn't be used or its configuration should be adjusted to not override the values in CCV2_TYPESYSTEM_MIGRATIONS table.

Keywords

typesystemname, Commerce Migration Toolkit, CCV2_TYPESYSTEM_MIGRATIONS

Symptom

  • After upgrading to SAP S/4HANA, a change in BOM selection behavior was observed during product costing using transaction CK40N.
  • Previously, the system selected the BOM based on the earliest valid-from date that was less than the costing key date (e.g., BOM A was selected).
  • After the upgrade, the system now selects the BOM with the most recent valid-from date prior to the costing date, resulting in BOM X being selected instead.

Environment

  • Controlling (CO)
  • SAP S/4HANA
  • SAP enhancement package for SAP ERP, version for SAP HANA.
  • SAP S/4 HANA Finance

Reproducing the Issue

1. Execute transaction CK11N.

2. Enter the Material, Plant, and Costing Variant.

3. Execute the cost estimate.

4. Navigate to the Quantity Structure tab in the cost estimate results.

5. Under BOM Data section, observe that Production Version X is selected by the system.

Cause

By standard system logic:

  • If a Production Version is maintained in the Costing 1 tab of material master (MM03), the system directly uses that version.
  • If no production version is specified, the system determines the production version based on the BOM application defined in the costing variant and evaluates the production versions maintained in the material master.

Historically, production version selection was determined using alphanumeric sorting.

With the introduction of SAP Note 2893498 - CK11N: Date not correctly considered during production version selection, the selection logic was changed to determine the production version based on the latest validity date prior to the costing key date.

This “selection by latest validity date” subsequently became the standard system behavior.

This logic is also documented in KBA 3556861 - No production version determined during costing.

Resolution

  • To revert to the previous production version selection logic (alphanumeric sorting) in the system, activate the existing system variable CK_SEL_PRODVERS.
  • This can be enabled via transaction FIN_FLAGS, as described in SAP Note 3097881.
  • Please ensure that the change is thoroughly tested and validated in a quality or test system before transporting it to the production environment.

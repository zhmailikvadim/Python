Symptom

If creating a diagram in Solution Documentation with the SMUDE API or the Process Management API, the diagram version ID attribute is not created in the database.

Other Terms

DIA_VERSION_ID, COLLABDIA, IFACEDIA, PROCDIA, UNIVERSALDIA

Reason and Prerequisites

The cardinality of the attribute DIA_VERSION_ID is 0:1, so it is not a must.
The coding could not distinguish between attribute is saved in the database with initial value 0 or it is not save, this means NULL. But in both cases in the coding it's handled like 0 which is the initial value.

Solution

Please implement the correction instruction of the SAP Note with transaction SNOTE. Then always, independent with which API, the DIA_VERSION_ID attribute with value 0 is saved in the database when creating the diagram.

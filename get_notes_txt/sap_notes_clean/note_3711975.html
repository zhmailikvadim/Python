Symptom

  • While importing the Rule elements (Territory, Formula, Rate tables etc.) via XML import pipeline, the process fails with the error message "NullPointerException: null".
  • Error codes observed in pipeline logs include:
  • TCMP_30609:E: Data import failed {0} because of {1}.
  • Fatal error in runStage! Cause: Error: Failed in processing XML File with exit code 1! Please check the log files for detail.

Environment

  • SAP Sales Performance Management
    • Incentive Management on Hyperscalers

Reproducing the Issue

  1. Create a territory with references to classifiers or Formula with reference to another rule element like fixed value.
  2. Split the territory/Formula (or the created rule element) into multiple versions.
  3. Delete the most recent version of the territory.
  4. Run the import pipeline using the XML file for the same Rule element to create a new version of the same (use the same effective dates as the deleted version).
  5. Observe the error "NullPointerException: null" in the pipeline logs.

Cause

  • Bug in the product: Deleting the version of the rule element does not remove the classifier or rule element references of the deleted version of the rule element.

Resolution

  1. SAP can perform data surgery to manually update the data.
  2. Alternatively, manually create the version from the UI.
  3. Until the bug is patched, refrain from deleting versions in the workspace to avoid encountering this issue.

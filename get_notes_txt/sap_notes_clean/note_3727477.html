Symptom

When the zoom parameter is changed in a run report (whether opened from the list of reports or from a package) in the SAP Financial Consolidation 10.1 legacy web client, the report layout becomes incorrect: text boxes are shifted from their expected positions.

The layout displays correctly only when the zoom parameter is set to 100%.

Other Terms

BFC, BOFC, scale, factor, misalignment, execution, rendering

Reason and Prerequisites

Steps to reproduce:

  1. Connect to the SAP Financial Consolidation legacy web client.
  2. Run a report containing text boxes (from the list of reports or from a package).
  3. Change the zoom level (for example, from 100% to 80%).

Result: The text boxes are shifted to the left, causing the report layout to become misaligned.

Solution

Update your SAP Financial Consolidation server installation to a support package, patch or release where the issue is fixed.

To activate the fix, add the following key with value "true" in the web.config file of your SAP Financial Consolidation legacy website:

<add key="ScheduleZoomRenderingReviewed" value="true" />

See the Support Packages & Patches section for details and available patches.

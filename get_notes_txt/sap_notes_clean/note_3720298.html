Symptom

You are reprocessing the accounts of an OCM RAR contract using transaction FARR_REPR_ACCT or in the Fiori UI. You receive error FARR_FOUNDATION121: Account determination for recognized revenue failed for company code &1. Your BRF+ configuration seems correct and you can identify a rule which should have been picked.

Other Terms

OCM, CCM, account determination, invoice condition, FARR_FOUNDATION121

Reason and Prerequisites

Prerequisite: One of the condition types used in the contract's POBs was also included in the POB's processed invoice RAIs, but the P/L account was either missing there or it was different from the order RAI.

Reason: Prior to this note, the system could accidentally pick up the reference account from processed invoices, which could lead to errors. This behavior was never according to design. OCM account determination only considers invoice condition types if they do not exist in the original order. If the original order contains the condition type, the accounts carried in the invoice are ignored.

This note also corrects another scenario: If the condition type is only carried by invoices, it ensures that only the latest invoice (and it's P/L account) is used during account determination. This change applies to both CCM & OCM.

Solution

Please implement this SAP note.

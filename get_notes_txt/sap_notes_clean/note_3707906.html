Symptom

  • You have successfully run J1AI with the provisional index, and it updated the revaluation area in AW01N (Asset Explorer).
  • When you try to run J1AI with the definitive index, the following error message appears: Error: "No Revaluation Posted for Period MMM / YYYY" when executing J1AI for the same period.

Environment

  • SAP ERP
  • SAP S/4HANA
  • Inflation in Asset Accounting 

Reproducing the Issue

  1. Run J1AI for provisional index.
  2. Check if the run is successful, and the values appear in AW01N (Asset Explorer). 
  3. Then run J1AI for the same period using the definitive index.
  4. The error "No Revaluation Posted for Period MMM / YYYY" appears on executing J1AI.

Cause

If the provisional index values are the same as the definitive values, J1AI will not post anything, as there is nothing to calculate.

Resolution

  • Verify whether the provisional index and the definitive index are the same.
  • If they are the same, J1AI will not perform a revaluation, as there is nothing left to revalue. This is standard behavior.

However:

  • If the provisional index value is incorrect, you must update it to the correct value that was used when you ran J1AI for the first time.
  • The same applies if the definitive index value is incorrect.
  • After updating the values, try running J1AI again and verify whether the run is successful.

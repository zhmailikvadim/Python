Symptom

  • Configurations maintained in Fiori apps do not initiate a transport request prompt to save the changes.
  • Changes are stored locally without initiating a transport request prompt.

Environment

Advanced ATP

Reproducing the Issue

  1. Log in to the Fiori Launchpad
  2. Open any app.
  3. Create a new entry or modify an existing configuration.
  4. Click "Save."
  5. Observe that the data is saved successfully, but no transport selection pop up appears.

Cause

  • The system design does not include functionality to initiate a transport request for customizing changes.
  • Configurations are stored locally, requiring manual replication across relevant systems.

Resolution

  1. Perform the app configuration in all relevant Fiori Frontend Systems individually, as the current system design does not support transport requests for customizing changes.
  2. To propose an enhancement for future SAP product versions, submit an improvement request following the guidance in SAP KBA 0003593361 point 2b.

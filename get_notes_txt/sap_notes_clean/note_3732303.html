Symptom

When navigating between the reconciliation and the drilldown app, the date filter was carrying over old date values from previous selections, causing incorrect data to be displayed in the drilldown app. Same happens also for the navigation back from Subledger to GL Balance View and from GL Details to GL Balances View.

Other Terms

FCCOREUI ; drilldown ; GL ; general ; ledger ; incorrect date ; wrong date

Reason and Prerequisites

Fix is applied that clears the old date values each time the filter is passed to the drilldown app (or within the different levels of the drilldown app) and only the currently selected date range is applied

Solution

Implement the note and all of its prerequisites.

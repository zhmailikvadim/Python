Symptom

"-iqmt" in .stderr file is "3847" even though explicit "-iqmt 4096" is set in params.cfg

--params.cfg
...
-iqmt 4096
...

--.stderr
...
   3847 threads             (     -iqmt )
...

Environment

SAP IQ 16.1 SP05 PL15

Reproducing the Issue

1. start up SAP IQ server with following parameters.

--param.cfg
...
-gm 200
-iqnumbercpus 32
-iqmt 4096
...

2. check "-iqmt" parameter from .stderr file

Cause

  • This is an expected behavior.
  • "-iqmt" can be specified to "4096" but following limitation is applied.
    -iqmt(.stderr) =<  4096 - "-gm" - "-iqnumbercpus" - 17
    3847 = 4096 - 200 - 32 - 17

Resolution

No action is needed as this is an expected behavior.

See Also

-iqmt Database Server Option

Keywords

-iqmt, -gm, -iqnumbercpus, 4096

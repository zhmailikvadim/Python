Symptom

You are processing RF loading and the resource has a default start bin. The worker start the loading and as you have complex loading, the WT confirmation is involved. It will update the resource actual bin, but when the new WO/HU is processed during the RF loading, the system still using the default bin, not the latest and updated actual bin/

Other Terms

start_bin, default_bin, actual_bin

Reason and Prerequisites

The system using buffering to read the resource and the updated actual bin was available on DB, but the buffer was not updated.

Solution

Implement the correction.

Symptom

When generating the DRC Colombia Income Withholding Tax Declaration report (CO_WHT_INCOME_DCL), the output is empty and contains no documents. The business is unable to meet legal reporting requirements as a result. During technical analysis, it is observed that documents are not being selected because the POSTINGDATE field is not filled for the relevant items.

Environment

  • SAP S/4HANA
  • SAP S/4HANA Cloud Private Edition
  • SAP Document and Reporting Compliance service
  • Colombia Withholding Taxes

Cause

The root cause is a configuration setting related to the Withholding Tax Reporting Date.

  1. The Company Code has been configured to use the "Withholding Tax Reporting Date" feature (view V_WTREPDATE).
  2. When this setting is active, the report intentionally ignores the standard document PostingDate.
  3. Instead, the report relies on the WithholdingTaxReportingDate field.
  4. This field was not populated for the documents, as it must be maintained as a separate, preparatory step using the "Manage Withholding Tax Items" Fiori app.

Because the system was configured to look for a specific reporting date that had not been maintained, no documents met the criteria for the report.

Resolution

You have two options to resolve this issue, depending on your business requirements.

Option 1: Use the Withholding Tax Reporting Date Functionality
This is the intended procedure when this configuration is active.

  1. Before running the CO_WHT_INCOME_DCL report, navigate to the Fiori app "Manage Withholding Tax Items."
  2. Use this app to define the specific reporting date for the relevant withholding tax items.
  3. Once the dates are maintained, run the declaration report again. The documents will now be included correctly.

Option 2: Deactivate the Withholding Tax Reporting Date Functionality
If your business process does not require the use of a separate Withholding Tax Reporting Date, you can disable this feature.

  1. Go to the configuration for the Company Code (view V_WTREPDATE).
  2. Deactivate the "Withholding Tax Reporting Date" setting.
  3. With this feature disabled, the report will revert to using the standard PostingDate from the documents.

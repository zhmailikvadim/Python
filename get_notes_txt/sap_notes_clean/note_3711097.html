Symptom

Japanese characters appear garbled when importing business users using a CSV file in the Maintain Business Users app.

Environment

SAP S/4HANA Cloud Public Edition

Reproducing the Issue

  1. Open the Maintain Business Users app.
  2. Select the Import option.
  3. Upload the CSV file containing the required values.
  4. Verify the imported data.

Cause

The issue occurs because the CSV file is encoded in Shift-JIS format instead of UTF-8.

Resolution

In SAP Help Portal - How to Maintain Business Users, it is mentioned that the CSV file needs to be UTF-8-encoded.

See Also

SAP Help Portal - How to Maintain Business Users

Symptom

You can use the interface for FS-RI treaties to deliver and create a retrocession whose assignment is not uniquely specified with regard to assumed treaty section and retrocession treaty section.

Other Terms

Reason and Prerequisites

Prerequisite:

  • The system contains a retrocession assignment for the assumed treaty A/Section 1 and retrocession treaty B/Section 1.
  • The following additional retrocession assignment is delivered via the interface: Assumed treaty A/section “SPACE” and retrocession treaty B/section 1.

Reason:

During interface processing, the system does not check whether retrocession assignments are unique.

Solution

History:

March 11, 2026: New

Troubleshooting Description:

Checks for the uniqueness of retrocession assignments are also performed during the delivery via the interface for FS-RI treaties.

A retrocession assignment between two treaties without entering sections is permitted only if there is no concrete assignment using sections between the treaties. If concrete assignments already exist, you must remove them or, alternatively, make the assignment more specific by entering an assumed or retrocession section number. This ensures that the assignments are unique. If an error occurs, the system issues the following messages:

“Missing uniqueness in retrocession assignments”

“Data record already exists in the retrocession data database table”

Manual activities:

Manual pre-implementation steps are required.

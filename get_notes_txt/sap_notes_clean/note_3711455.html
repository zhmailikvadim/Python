Symptom

After creating an approval workflow and assigning my own user as the Sales Order approver in the "Manage Teams and Responsibilities" app, the Sales Order is not able to be approved.

Environment

SAP S/4HANA Cloud Public Edition

Reproducing the Issue

  1. Create an approval workflow
  2. Set up your own user as the only approver of the document
  3. Create the Sales Order
  4. Now the document is stuck and can't be approved.

Cause

The approval workflow involves two parties: The first one is the user who creates a document that needs to be approved. Depending on the sales document category, this is usually an internal sales representative or a returns and refund clerk. This user is the approval requester.

The second one is the user who approves a document. This is usually a sales manager.

Resolution

Self-approval of Sales Orders in approval workflows is not supported.

To proceed, it is necessary that one user creates the sales order and sends for another user so he can approve the document.

If multi-level approvals is required, it is possible to refer to the help documentation for guidance on configuring the steps to meet this business requirements.

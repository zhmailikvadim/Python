Symptom

In maintenance orders, the Planner Group appears in the order screen, but the AFKO-PLGRP field remains blank.

Environment

  • SAP ERP Plant Maintenance (PM)
  • SAP R/3
  • SAP R/3 Enterprise
  • SAP ERP Central Component
  • SAP ERP
  • SAP Enhancement package for SAP ERP
  • SAP Enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA, on-premise
  • SAP S/4HANA, Cloud Private Edition

Reproducing the Issue

  1. IW31/32 Create or Edit a maintenance order and enter a Planner Group on the order screen; save the order.
  2. SE16 browse table AFKO and display the order by its number; observe that field PLGRP is not filled.

Cause

AFKO-PLGRP represents the Planner Group responsible for the task list. By design, it is only populated when a task list is assigned to the order and a Planner Group is maintained in that task list.

Resolution

 Maintenance orders only fills PLGRP if a task list is included and that task list contains a planner group. The planner group for a maintenance order is stored in table AFIH, field INGPR

  1. Assign a task list to the maintenance order.
  2. Ensure that the Planner Group is maintained in the task list header.
  3. Save the order.
  4. Verify in table AFKO that field PLGRP is now populated.
  5. Note: Entering a Planner Group only on the order screen does not update AFKO-PLGRP without an assigned task list; this is expected behavior.

Symptom

When attempting Single Sign-On to SAP S/4HANA using the SAML 2.0 Bearer Assertion flow for OAuth 2.0, the token endpoint returns:

invalid_grant
Provided authorization grant is invalid. Exception was No user found with * "user@email.com".
For more information, consult the kernel traces or the OAuth 2.0 troubleshooting SAP note 1688545.

The configuration requirement applies to all SAML‑to‑ABAP user‑mapping modes, where * represents the mapping mode expected by the S/4HANA system (e.g., Alias, Email, Logon ID, etc.).

Environment

  • SAP NetWeaver Application Server
  • ABAP Platform
  • SAP S/4HANA

Cause

The SAML assertion sent to SAP S/4HANA contains a user identifier 'email address', but the ABAP system cannot map this value to any existing user.

Resolution

Ensure that the SAML assertion sent by the Identity Provider contains a user identifier that the ABAP system can successfully map to an existing user. The identifier provided in the assertion must match a maintained value in SU01 (User Alias, Email, User ID..), depending on the mapping configuration.

If the SAML assertion is not providing the expected user identifier, contact the Identity Provider administrator to update the IdP configuration so that it sends a valid and mapped value.

If SAML assertion is correct, adjust the mapping rules in transaction SAML2 so that the incoming NameID or attribute is mapped to the correct ABAP user field. Choose the appropriate option such as Unspecified, Email, or another valid identifier, based on the assertion and verify that the corresponding value is maintained for the user in SU01.


To configure this, proceed as follows:

  1. SAML2
  2. Go to Trusted Providers
  3. Select Show OAuth 2.0 Identity Providers
  4. Choose the Identity Provider
  5. Select Edit
  6. Adjust the Supported NameID Formats according to the required identifier


Additional details about user mapping and identity federation can be found in the SAP Help documentation for Identity Federation in AS ABAP.

For SAC scenarios using SAML Bearer Assertion Flow to S/4HANA systems, refer to 2805974 - Error 'You are not authorized to query the remote system. Please ask your administrator to grant you the InA role.' when creating live connection to S4/HANA Cloud in SAP Analytics Cloud (SAC)

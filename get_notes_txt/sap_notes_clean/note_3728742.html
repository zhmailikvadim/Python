Symptom

You need to troubleshoot connection issues or errors from a specific client system (e.g., PO system, external interface) connecting to SAP Web Dispatcher, but:
  • Setting global trace level to 2 (`wdisp/trace_level = 2`) causes performance degradation or system unresponsiveness
  • The production environment cannot tolerate the overhead of global high-level tracing
  • You need detailed trace information for only one specific client IP address or URL pattern
  • Standard trace level 1 logs do not provide sufficient detail to diagnose the issue

Environment

  • SAP ICM / Web Dispatcher
  • Kernel 753+
  • ABAP Platform 2023+

Cause

  • No logs visible at trace level 1 during error occurrence
  • There is a need for a method to enable detailed tracing <<only for specific client connections>> without affecting overall system performance.
  • SAP Web Dispatcher's global trace level parameter (`wdisp/trace_level`) applies to ALL incoming and outgoing connections. There was historically no built-in mechanism to selectively raise the trace level for specific clients, URLs, or connection criteria.

Resolution

SAP introduced the parameter icm/conditional_trace_<xx> starting from ABAP Platform 2023, which enables selective trace elevation based on configurable criteria.
 
The icm/conditional_trace_<xx> parameter allows you to:
  • Raise trace level only for connections matching specific criteria
  • Keep global trace level at 1 while tracing specific clients at level 2 or 3
  • Configure multiple independent conditional trace rules
  • Set automatic expiry for time-limited tracing
  • Filter by client IP, URL, host header, port, HTTP method, or system name
 
Syntax:
icm/conditional_trace_<xx> = [LEVEL=<1|2|3|sap-trace|trace_info|ssl_debug_info>]
                             [, CLIENT_IP=<IP_address>]
                             [, URL=<path_prefix>]
                             [, HOST=<host_header>]
                             [, SERVER_PORT=<port_number>]
                             [, SERVER_IP=<IP_address>]
                             [, FORWARDEDFOR_IP=<IP_address>]
                             [, METHOD=<HTTP_method>]
                             [, ROLE=<SERVER|CLIENT|INTERNAL>]
                             [, SYSTEM=<system_name>]
                             [, VALID_UNTIL=<+seconds|ISO_timestamp>]

Example:

  • icm/conditional_trace_0 = LEVEL=2, URL=/sap/public/ping, CLIENT_IP=10.0.0.1
    Every HTTP request to /sap/public/ping that is sent from the IP address 10.0.0.1 is handled with a raised trace level.
    If URL, HOST, or FORWARDEDFOR_IP is set, the trace level is not raised until the HTTP request header is completely parsed.

  • icm/conditional_trace_0 = LEVEL=2, VALID_UNTIL=+120.
    If that parameter is set in the profile and the process is restarted, ICM/Web Dispatcher will run with trace level 2 for the first two minutes after startup. That is, for example, helpful to analyze issues that occur during startup only.

Note: icm/conditional_trace is a vector parameter that allows you to configure multiple criteria that are independent of each other.

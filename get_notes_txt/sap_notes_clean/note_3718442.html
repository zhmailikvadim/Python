Symptom

In certain scenarios, the workflow description text returned by /DSCSAG/WI_DESCRIPTION_GET may not be displayed completely.
HTML‑based workflow texts containing line‑break or paragraph tags may not be split correctly into individual text fragments.
As a result, parts of the workflow description may not appear in the consuming application.

Other Terms

 PLM-42128, ECTR, Workitem

Reason and Prerequisites

The function module /DSCSAG/WI_DESCRIPTION_GET processes workflow texts by detecting HTML line‑break‑related tags using a regular expression replacement. When the incoming workflow description contains the same elements in uppercase, the expression does not match.

The issue may occur when HTML tags are provided in uppercase (e.g. <P>, <BR>, <TR>, <BODY>).

Solution

The HTML tag detection logic in /DSCSAG/WI_DESCRIPTION_GET was enhanced to support both uppercase and lowercase HTML tags.

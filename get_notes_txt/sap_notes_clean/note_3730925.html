Symptom

Previewing apps from BAS showing the following error messages:

Sorry, a technical error occurred! Please try again later.
{
"message": "HTTP request failed",
"headers": [],
"statusCode": 403,
"statusText": "",
"responseText": "CSRF token validation failed"

While checking the http trace, it's found that in the response header of each HEAD request, the x-Csrf-Token value is a different one. And the POST request then failed with 403 error.

There's no issue while opening the same app which has been deployed in ABAP (for example).

Environment

  • SAP Business Application Studio

Cause

An oversized session cookie named "arbe" exceeded the browser 4096-character limit, preventing the cookie from being set and breaking session stickiness in preview; each HEAD/POST then ran without a consistent session, producing different CSRF tokens.

Resolution

BAS development team is investigating the issue.

Current workaround: Clear all cookies in browser then reload the preview. Alternatively, run the preview in a private/incognito window to start with a clean cookie store.

Keywords

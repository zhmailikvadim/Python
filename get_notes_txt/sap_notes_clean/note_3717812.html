Symptom

A custom header field is configured at Sourcing request and Sourcing Project levels. Why is the value entered for this field changing to blank with the parent Sourcing Request header overview is edited?

Reproducing the Issue

  1. Create the Sourcing request, leave the custom header field blank
  2. Now, while creating Sourcing Project inside the Sourcing Request, a value is entered for the field and created.
  3. If we navigate back to the sourcing project: 
    1. Click Actions >Edit Overview
    2. Either make a change to any other field or save without making any changes
  4. Navigate back to Sourcing Project, Note the value for the field has now become blank.

Resolution

The custom header field has a setting enabled at the field configuration level: Sync in sub-project on update and Sync in sub-project on create. 

The first rule ensures any data changes to the header fields at the Parent project level will be cascaded to the sub-project level.

Due to this, since initially itself the Parent project did not have a value (was left blank), the sub-project header field also get populated with a blank value.

This seems to be as per the standard rule expectation.

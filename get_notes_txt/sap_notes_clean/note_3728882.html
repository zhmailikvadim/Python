Symptom

Screens randomly shrink to smaller sizes after an update.Users are required to use horizontal or vertical scrollbars to view the content. Resizing the screen to fit full screen does not resolve the issue, as subsequent visits to the same applications result in the screen being resized or shrunk again.The issue is observed in SAPUI5 version 1.84.28.

Environment

SAP S4/HANA

Reproducing the Issue

  1. Access the Fiori Launchpad after the update.
  2. Open any application within the Fiori Launchpad.
  3. Observe that the screen shrinks to a smaller size, requiring the use of scrollbars.
  4. Resize the screen to fit full screen.
  5. Revisit the same application and observe that the screen is resized or shrunk again.

Cause

The issue is caused by bugs present in SAPUI5 version 1.84.28, which have been addressed in later patch versions (1.84.50 or higher).

Resolution

  1. Upgrade the SAPUI5 patch version to 1.84.50 or higher.
  2. Follow the steps outlined in KBA ##3155948 - ABAP SAPUI5 patch version update to perform the upgrade.
  3. Ensure that the prerequisites for the upgrade are met.
  4. Note that no kernel update is required, as the baseline version (1.84) remains the same.

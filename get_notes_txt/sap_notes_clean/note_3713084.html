Symptom

This SAP Note addresses the following issue.

1. Incorrect QR Code is stored in the field QR_CODE_ACCEPTED in table EDOPLINV for documents in status Accepted by KSeF (or) Accepted by KSeF (Offline).

Poland, eDocument, eDoc, eInvoicing, electronic invoice, electronic invoicing, Poland eInvoice, Document Compliance, Digital Compliance, DC, B2B, National System of e-Invoices, KSeF, Krajowy System e-Faktur, e-Faktura, faktura ustrukturyzowana, faktura elektroniczna, Electronic Document, Document and Reporting Compliance, electronic document processing, FA(2) Schema Version Code, FA(3) Schema Version Code, QR Code Accepted

Reason and Prerequisites

For a complete prerequisite list, refer to SAP Note 3279206 (Electronic Document Processing for Poland - KSeF: Implementation Overview).

Solution

Date Version Reason

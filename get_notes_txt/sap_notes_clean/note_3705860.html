Symptom

When an attachment is uploaded to a Quality Notification in SAP S/4HANA, the Notification Change Event is not triggered and the update is not sent to SAP Business Network.

Environment

  • SAP S/4HANA

  • Quality Management (QM)

  • Quality Notifications

  • SAP Business Network integration (via CIG)

Cause

In standard SAP S/4HANA, attachments for Quality Notifications are processed through the Generic Object Services (GOS) framework. Attachment creation or modification does not generate a change document for the Quality Notification header or item.

Notification Change Events are raised only when a change document is created. Since attachment-only updates do not create change documents, no Notification Change Event is generated.

There is no standard configuration, SPRO setting, or CIG parameter available to trigger Notification Change Events based solely on attachment creation.

Resolution

This behavior is expected and working as designed in standard SAP S/4HANA.

If attachment-only updates must trigger a Notification Change Event and be sent to SAP Business Network, a customer-specific enhancement is required. Customers can implement a supported enhancement (for example, using BADI_QMEL_SAVE) to programmatically perform a controlled update to the Quality Notification when an attachment is saved. This controlled update creates a change document and enables the Notification Change Event to be generated.

Keywords

Quality Notification, Attachment, Change Document, Notification Change Event, SAP Business Network, CIG, BADI_QMEL_SAVE

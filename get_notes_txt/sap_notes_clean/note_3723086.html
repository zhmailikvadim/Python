Symptom

In the Settle Route Data app, you navigate to the Visits tab and you open, for example, a delivery (outbound delivery) that requires a signature date.

If the user of the front-end launchpad is set to a different time zone than the back-end launchpad, the F4 value help does not open, and if you enter the same signature date as the route departure or arrival date, you get the following error:

"The signature date must be between the departure and the arrival date for the route."

Other Terms

Last Mile Distribution, LMD, Settle Route Data, SRD, F4242, Signature Date, LstMiRteDocumentSignatureDate

Reason and Prerequisites

This issue is caused by a programming error.

The signature date DatePicker's minimum and maximum date constraints were being set without considering time zone differences between the backend and the launchpad time zone. This caused the date validation to fail or allow incorrect dates when launchpad users were in different time zones than the system.

Solution

Please apply the attached correction instructions to resolve the described issue.

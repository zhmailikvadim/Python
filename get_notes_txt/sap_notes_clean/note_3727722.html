Symptom

The OnSave business rules are triggered successfully when the Job Requisition SAVE is executed either through a simple UPSERT via Integration Center or a UI Save.
However, if the SAVE is done via an ODATA API external call, the business rule does not trigger anymore.

Environment

SAP SuccessFactors Recruiting Management 

Resolution

Our developers have already analyzed this issue, and the problem will be fixed in the next b2605 release.
As a workaround, we suggest to use batch upsert instead of single upsert.

Keywords

OnSave rule ODATA

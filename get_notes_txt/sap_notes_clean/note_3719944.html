Symptom

When a new simulation is created and the user manually selects a simulation date in the dialog, the simulation may end up being created for the day before or after the specified simulation date.

Other Terms

simulation, timezone, date/time, F2570

Reason and Prerequisites

The user who creates the simulation resides in a time zone with significant difference compared to the UTC time. Due to a bug in the date conversion logic in these cases the selected date can be incorrectly interpreted as the previous or the next day in the simulation.

Solution

Install the attached correction instructions or the corresponding support package.

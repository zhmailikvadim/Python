Symptom

When creating a Purchase Order (PO) using BAPI_PO_CREATE1, the system does not take the filled order quantity. The order quantity is rounded up, even though there is no rounding profile defined in Material Master.

Environment

  • Materials Management (MM)
  • SAP ERP Central Component
  • SAP ERP
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA

Reproducing the Issue

  1. Create a Purchase Order using function module BAPI_PO_CREATE1, with an order quantity specified in the input field POITEM-QUANTITY. 
  2. Check the PO in ME23N. The order quantity is rounded up.

Cause

Even though no Rounding Profile is assigned in the Material Master, a Rounding Value is maintained (MARC-BSTRF is not empty).
For example, if the Rounding Value is set to '3', this indicates that if no rounding profile is defined in the material master, the system will round up the entered quantity to the next multiple of 3. This calculation happens in the function module MD_ROUNDING_VALUE. 
Thus, if a quantity of 20 is entered in BAPI_PO_CREATE1 to create a Purchase Order, the order quantity will be rounded up to 21.

Resolution

Go to transaction code MM02 to check the MRP1 view of the material. Adjust the Rounding Value according to business needs.

Symptom

  • The system generates error messages based on the type of transaction being performed.
  • No categorization exists for warnings, informational messages, errors, or validations.
  • Messages are requested to be provided module-wise or transaction code-wise.

Environment

ABAP Platform

Reproducing the Issue

  1. Perform a transaction in SAP system.
  2. Enter invalid data or initiate an incorrect transaction.
  3. Observe that the system generates an error message based on the type of transaction being performed.

Resolution

  1. Message texts are stored in table T100 in the SAP system.
  2. To retrieve the messages, a custom program can be developed to read from table T100.
  3. Note that there is no standard functionality to segregate messages module-wise or transaction code-wise.
  4. The field "ARBGB" in table T100 indicates the application area, which can be used for partial categorization.

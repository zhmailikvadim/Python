Symptom

  • Popup of deleted task out of recent items exists after Task is deleted from UI
  • Error message: "Data does not exist"

Environment

  • SAP Sales Cloud and SAP Service Cloud Version 2
 



Reproducing the Issue

  1. Create a task out of a case.
  2. Delete the task.
  3. Open the shortcuts button in the top right. Inside recent items, locate the task marked as "0 mins ago".
  4. Click on the task. It opens a task with blank data.
  5. It opens a task with blank data.

Cause

The task will be present in the recent items but will not contain the data anymore

Resolution

This is the expected standard system behaviour.

Symptom

  • UI rules affect the visibility of the fields in the Workflow Approval page
  • UI rule is working as expected when adding data from Manage Data or on People Profile

Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental

Environment

SAP SuccessFactors HCM 

Reproducing the Issue

  1. Set up UI rules visibility on the Manage Configuration UI for an MDF object.
  2. Go to Employee Profile or Manage Data. 
  3. Verify that the UI rules are working as expected and fields are shown as expected.
  4. Submit an entry. 
  5. Proxy as an approver and check the Workflow Approval page. 
  6. Notice that the fields which was shown via the visibility UI rule are not showing up.

UI Rule of above: 

Cause

There should be an 'else' condition that would catch whatever the opposite of your 'if' and 'else if' condition is. 

Resolution

There should be an 'else' condition that would catch whatever the opposite of your if and else if condition is:

  • Example, you set a visibility=no initially and whenever a field's value is not null, it should be visible. After the 'if' and then', an 'else' condition should catch whatever the opposite of the 'then' is. 
  • The catcher should be 'else', 'else if' is considered a condition statement. 
  • Ensure to put the UI rule on both the onInitUI Rules and onChangeUI Rules (of the field where this field is a dependent of).

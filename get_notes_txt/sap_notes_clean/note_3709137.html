Symptom

  • Inquiry regarding the support for STCDT usage in Customer Vendor Integration (CVI) for Peru and Colombia to maintain synchronization between DFKKBPTAXNUM and KNA1.
  • SAP documentation specifies that STCDT is exclusively for Argentina to distinguish AR1 Tax category sub-categories.
  • SAP Notes 3633078 and 3500310 mention customization of V_TFKTAXNUMTYPE and CVI API support for other countries, including Peru and Colombia.
  • Concern regarding the use of STCDT for tax reporting in Peru and Colombia.

Environment

Customer Vendor Integration – FS-BP part

Reproducing the Issue

  1. Review SAP documentation indicating that STCDT is exclusively for Argentina.
  2. Refer to SAP Notes 3633078 and 3500310, which discuss customization of V_TFKTAXNUMTYPE and CVI API support for other countries, including Peru and Colombia.
  3. Attempt to use STCDT for tax reporting in Peru and Colombia to maintain synchronization between DFKKBPTAXNUM and KNA1 during UAT testing.

Cause

  • STCDT mapping for Peru has been released, allowing CVI mapping for values to KNA1/LFA1 with one-to-one mapping between the STCDT value and the tax category.
  • No changes have been released for Colombia regarding STCDT mapping.

Resolution

  1. Confirm that STCDT mapping for Peru is supported for tax reporting purposes.
  2. During CVI mapping, ensure that values are considered for mapping to KNA1/LFA1 based on the one-to-one mapping between the STCDT value and the tax category.
  3. Refer to the following mapping details for Peru:
  • PE1: BUPA_TAX_NUMBER_CHECK 6 - Peru: RUC Number
  • PE2: BUPA_TAX_NUMBER_CHECK 1 - Peru: National Identity Document (DNI)
  • PE3: BUPA_TAX_NUMBER_CHECK 4 - Peru: Foreign Resident ID (CE)
  • PE4: BUPA_TAX_NUMBER_CHECK 0 - Peru: Non-domiciled Tax Doc (No RUC)
  • PE5: BUPA_TAX_NUMBER_CHECK 7 - Peru: Passport

See Also

  • Refer to SAP Note 3633078 for information on CVI API support for other countries.
  • Refer to SAP Note 3500310 for customization details of V_TFKTAXNUMTYPE.

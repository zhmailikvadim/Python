Symptom

When modifying the production order quantity using the BAPI BAPI_PRODORD_CHANGE, if the new quantity is less than the quantity already in stock, the BAPI automatically adjusts to the stock quantity and returns a success message. However, this results in an error alert on the frontend. The challenge is how to configure the interface to also trigger an appropriate alert when such a discrepancy occurs. 

Environment

  • SAP ERP Production Planning (PP)
  • SAP ERP All versions
  • SAP ERP Central Component All versions
  • SAP R/3 All versions
  • SAP R/3 Enterprise All versions
  • SAP S/4HANA All versions
  • SAP S/4HANA Cloud Private Edition All versions
  • SAP enhancement package for SAP ERP All versions
  • SAP enhancement package for SAP ERP, version for SAP HANA All versions

Reproducing the Issue

Execute BAPI_PRODORD_CHANGE in SE37.

Cause

This is a standard design. 

Resolution

The BAPI is designed to operate as an update task within the system. It processes changes without interruption and does not display any pop-up messages or warnings during execution.

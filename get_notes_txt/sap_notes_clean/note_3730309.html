Symptom

Run validation universal journal task failed although data in ACODCU meets rule conditions

Environment

  • Group Reporting
  • SAP S/4HANA
  • SAP S/4HANA Cloud Private Edition

Reproducing the Issue

  1. Open the Fiori app “Data Monitor” (App ID: CXCD).
  2. Set the Global Parameters (for example: Cons. Group, Cons. Unit, Version, Fiscal Year, Period, Cons. COA).
  3. Start task Universal Journals Validation.
  4. Observe that VUJ task failed although data meets rule conditions.

Cause

  • In the additional characteristics for Release Universal Journal (RUJ), the problematic field is configured as “Aggregate on Release”; RUJ therefore does not transfer this field from accounting to group reporting. As VUJ validates the released data, it does not read this field and the rule checks it as initial, causing the error.

Resolution

  1. Review the additional characteristics configuration used by the Release Universal Journal (RUJ) and confirm whether field defined in condition is set to “Aggregate on Release”.
  2. If the field is aggregated on release, deactivating VUJ rules for this field which is aggregated over on release.

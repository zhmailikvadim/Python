Symptom

When retrieving data from the BPM OData services with a GET request using URIs based on /bpmodata, "null" is returned as a value for a multi-valued recursive Navigation Property, although the property value is not empty in the process context (e.g. in the task input data).

Other Terms

Galaxy, SAP NW BPM, Business Process Management

Reason and Prerequisites

A ‘multi-valued recursive Navigation Property’ can be represented in the following way in the OData service metadata:

<EntityType Name="Employee">
         <Key>
           <PropertyRef Name="EDM_Key" />
         </Key>
         <Property Name="EDM_Key" Type="Edm.String" Nullable="false" />
         <Property Name="firstName" Type="Edm.String" Nullable="true" />
         <Property Name="lastName" Type="Edm.String" Nullable="true" />
         <NavigationProperty Name="colleagues" Relationship="TestNamespace.Employee_colleagues" FromRole="Employee" ToRole="Employee" />
</EntityType>
<Association Name="Employee_colleagues">
        <End Type="TestNamespace.Employee" Multiplicity="1" Role="Employee" />
        <End Type="TestNamespace.Employee" Multiplicity="*" Role="Employee" />
</Association>

In the given example, "colleagues" Navigation Property is a multi-valued property because the EDM Multiplicity of the second End of the ‘Employee_colleagues’ Association is "*". The Navigation Property is also a recursive one because it points to its parent Entity Type, i.e. Employee in our case.

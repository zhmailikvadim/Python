Symptom

Version history:

  • v1 (14.12.2022): Pilot release
  • v2 (21.12.2022): Pilot release - Only 1 DISABLE field is generated, BC Set is added
  • v3 (17.01.2023): Pilot release - GENDERID, MARSTAT, SEXID and ETHNIC fields are empty in XML, DISABLE fields are not correct if there is multiple EE.
  • v4 (27.01.2023): Pilot release - SEXID is empty in the XML file 
  • v5 (31.01.2023): General release

After implementing the SAP Note 3252473 the HESA report generates an empty XML file.

The screen for IT0614 is not displayed correctly by the UK region of the provider, as different fields are required for reporting in some regions of the UK (e.g. Scotland).

Other Terms

RPUHESG2D01, RPUHESG2, HESA 2022/23

Solution

After implementing this SAP Note the XML file will be generated correctly, and the IT0614 is displayed correctly by the region of the provider.

To implement the changes in your system, apply the assigned automatic and manual correction instruction of this SAP Note. Alternatively, apply the HR Support Package indicated for your release under Reference to Support Packages.

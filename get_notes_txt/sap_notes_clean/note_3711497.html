Symptom

When executing the Line-Item Validation task in Group Reporting at group level, the task log remains in a loading state for several minutes and the user session terminates automatically.

However, when the same Line-Item Validation task is executed at consolidation unit (entity) level, the task completes successfully and the task log loads within a few minutes.

Environment

SAP S/4HANA Cloud Public Edition

Reproducing the Issue

  1. Navigate to the "Data Monitor" app. 
  2. Set the required Global Parameters. 
  3. Select the desired group > Click in the LlV GR () taks cell. 
  4. Execute the Line-Item Validation task at the group level.
  5. Wait for several minutes until the error message appears: "A back-end error has caused the application to terminate. Error message returned: HTTP: 500".

Cause

When Line-Item Validation is executed at group level, the system must process line-item data across all consolidation units included in the group.

In scenarios with a large volume of consolidation data, this processing can exceed the technical runtime and UI session limits of SAP S/4HANA Cloud, Public Edition. As a result, the task log cannot be fully loaded and the user session ends automatically.

This behavior is related to system performance and data volume and does not indicate data inconsistency or a functional error in Group Reporting.

Resolution

To avoid session termination, execute the Line-Item Validation task at consolidation unit (entity) level, where the data volume is reduced and the task completes successfully.

No system configuration or backend adjustment is available to extend runtime limits in SAP S/4HANA Cloud, Public Edition.

Improvement requests can be created and voted on by other customers, which may influence future product enhancements.

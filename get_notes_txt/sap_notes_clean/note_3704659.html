Symptom

  1. Logs from the audit report are identified, but the corresponding activities are unclear.
  2. The logs were obtained from Manage > Administration >Intelligent configuration manager > Audit logs.
  3. The event type for the logs is Data modification, and they were exported by creating a report.
  4. Customer wants to know the user details responsible for group assignment in SAP Ariba environment.

Cause

The Logs are generated when the user is added or removed from the groups.

Resolution

Follow below steps to gather Audit logs.

  1. Access Manage > Administration > Intelligent configuration manager > Manage configuration> click on Continue> you will be directed to Intelligent configuration manager page on the next browser tab.
  2. Navigate to Audit tab
  3. Select the event type as Data Modification.
  4. Filter the application as Buyer/Sourcing.
  5. Specify the date range and click apply .


Once the results are displayed

      1.    Review the logs and identify the event type as Data modification
      2.    Locate the document type ariba.user.core.group in the results.
      3.    The Document ID column will show the Group that was updated.
      4.    The Description column will indicate that the object of type                                               ariba.user.core.group was updated.
      5.    In the same row, you will find the user details who performed the action.

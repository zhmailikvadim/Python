Symptom

You are working in the transportation cockpit and using the field catalog feature to be able to define the displayed components (columns) for the lists and hierarchies. The field catalog application also displays the components that are added by the customer to the Enhancement Include of the TOR BO nodes ROOT, STOP, STOP_SUCC, and ITEM_TR. Additionally, it displays the components that are added by customers via a customer-specific append structure to the respective maximum structure (or substructure) of the respective planning list or hierarchy. The display of these customer-specific fields within the lists or hierarchies can be influenced by customers themselves. The default display type of the fields is restricted to:

  • TEXT_VIEW (= standard display type)
  • CHECKBOX
  • INPUT_FIELD
  • DROP_DOWN_LIST_BOX
  • IMAGE

But the PROGRESS_INDICATOR is not support right now.

Other Terms

FC, TC, /SCMTMS/FIELD_CONFIG, /SCMTMS/PLN, user field, customer field, progress_indicator

Reason and Prerequisites

For field catalogs it is not intended to offer all default display types of list or hierarchies. The default display types must fit into the field catalog and the dynamic configuration creation framework.

Solution

Implement the attached correction instructions or import the specified Support Package.

After the implementation of this correction, it is possible to set the display type 'ProgressIndicator' for customer specific fields.

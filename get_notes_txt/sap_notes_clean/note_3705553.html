Symptom

The job FDM_INVOICE_MEMORY is cancelled due to error: "Internal session terminated with a runtime error TSV_TNEW_PAGE_ALLOC_FAILED (see ST22)"


Environment

  • Collections Management
  • S/4HANA

Reproducing the Issue

  1. Job is created for report FDM_INVOICE_MEMORY
  2. Go to SM37 and job is cancelled with the following details: Internal session terminated with a runtime error TSV_TNEW_PAGE_ALLOC_FAILED (see ST22)
  3. Run ST22, the following dump is triggered:

Runtime Errors         TSV_TNEW_PAGE_ALLOC_FAILED
Method:                   IF_FDM_INVOICE_MEMORY~UPDATE_ALL

Cause

The number of entries processed at the highlighted step is too large, leading to resource bottleneck.

Resolution

  •  The report should not be executed with the flag 'Output Details' selected. This flag outputs a list documenting updated invoice information in addition to the application log, which is unnecessary for batch mode execution.
  •  If there is a high volume of data per company code,  separate jobs should be executed. Such a large amount of data cannot be processed in a single job, and multiple parallel runs should be executed:
    • Parallel run can be defined base on the company code , and to include a defined customer range
    • Create multiple job chains such that the distribution is unified.

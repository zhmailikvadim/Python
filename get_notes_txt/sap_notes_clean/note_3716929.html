Symptom

During the mass process creation of an Agreement Request, the revision fields RVNUM (Revision Number) and RVDAT (Revision Date) are being copied into the newly created Agreement Request. Since this is a new request, these revision values should not be copied from the source document.

This SAP Note corrects the logic to clear the Revision Number and Revision Date fields during mass process creation, ensuring that new Agreement Requests are created without inherited revision data.

Other Terms

Revision Date, Revision Number, /IRM/IPBBAME, Mass Process

Reason and Prerequisites

This is caused by Program error.

SAP Note#3716935 is available for V/4 release.

Solution

Please implement the correction instructions attached. These corrections are only valid if you have applied the Support Packs.

Release 6.4 – Base Package & Support Package 1
Release 6.3 – Base Package
Release 6.2 – Base Package
Release 6.1 – Base Package

Use transaction SNOTE to implement the correction instructions specified below and activate the affected program object

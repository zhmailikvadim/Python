Symptom

Error in Analytical Model; unexpected error appears when a hierarchy is selected in an analytic model preview: "Caught exception: exception 2120042: An unexpected error occurred."

When checking the indexserver trace, you see below error:

[75483]{241651}[1485183/-1]<19ac0b49d946b17f3903e3346502753a#428dac6341618da8> 2025-11-26 15:12:20.583515 d mds HierarchyImplSql.cpp(00083) : Instantiate SQL hierarchy with view - schemaName=<schema_name> name=§<hierarchy> with placeholders=[]
[75483]{241651}[1485183/-1]<19ac0b49d946b17f3903e3346502753a#428dac6341618da8> 2025-11-26 15:12:56.642652 E Assure HierarchyImplSql.cpp(00489) : The following assertion failed: aMaxMemberId < 20000000u
aMaxMemberId = 22598369
20000000u = 20000000
Hierarchy has too many nodes
Correlation id is 0x48dfc85342837be6

Environment

SAP HANA 1.0
SAP HANA 2.0
SAP HANA Cloud

Reproducing the Issue

  1. Open the analytic model in the data builder and start the preview.
  2. Set the required model variables (for example, provide a snapshot date).
  3. In the preview, select a hierarchy on a dimension (for example, choose a named hierarchy and enable hierarchical display).
  4. Observe that no data is returned and an unexpected error appears; browser developer tools show: exception 2120042.

Cause

The selected hierarchy contains too many nodes and exceeds an internal limit in the hierarchy engine (assertion on maximum member id), resulting in exception 2120042.

In the example trace, number of members was 22598369 which is greater than the limit of 20000000.

Resolution

The hierarchy simply has too many nodes which need to be reduced. This can be done in a few ways. See below listed steps:

  1. Reduce the number of hierarchy nodes returned at preview time by applying restrictive filters (for example, filter by year, version, or other relevant attributes).
  2. Use a shallower hierarchy level (avoid expanding to the lowest detail level) if possible, to lower the node count.
  3. Split the hierarchy into separate, narrower hierarchies (for example, by brand or year) to keep each below the internal limit.
  4. Remove non-required nodes or prune obsolete hierarchy members where feasible.
  5. Re-run the preview after each change to confirm the error no longer occurs.

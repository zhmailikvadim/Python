Symptom

Customer may encounters the RuntimeException with timeout error message during promotion evaluation. Such as:

  • The rule evaluation has exceeded the configured timeout of 10 seconds. A potentially infinite loop has been detected in the RuleNetworkEvaluator.doRiaNode2. Rule evaluation is being aborted.
  • The rule evaluation has exceeded the configured timeout of 10 seconds. A potentially infinite loop has been detected in the RuleNetworkEvaluator. Rule evaluation is being aborted.

Reason and Prerequisites

Reason:

During promotion evaluation, the Drools libraries will evaluate the promotion based on its algorithm.

However, there is a flaw in the algorithm and may generate directed cycle in their graph, which will lead to an infinite loop in the promotion evaluation.

Current solution is: throw a runtime exception if the promotion evaluation time exceeds the threshold value (property drools.rulenetworkevaluator.time with default value 10 seconds).

Step to reproduce:

Follow this link: https://me.sap.com/notes/3442067 to reproduce this issue.

Solution

When traversing the nodes in the Drools algorithm, detect the directed cycle in the Drools graph.

Stops traversing the nodes and break the loop if the directed cycle is detected.

Correction will be released in: 2211-jdk21.12

Symptom

No change logs are recorded for HRMS data in work centers such as personnel numbers. 

Environment

  • SAP ERP Production Planning (PP)
  • SAP R/3
  • SAP R/3 Enterprise
  • SAP ERP Central Component
  • SAP ERP
  • SAP Enhancement package for SAP ERP
  • SAP Enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA Cloud Private Edition

Reproducing the Issue

  1. Enter t-code CR02.
  2. Press HRMS button from menu or HR assignment button.
  3. Delete HRMS relevant data such as personnel number/person.
  4. Press Save.
  5. In t-code CR10 no change documents are recorded.

Cause

HR management system data change log is not recorded via work center interfaces as per standard behavior. 

Resolution

Work center supports specific database fields for change document creation. Refer to SAP Note 352485 for details on work center relevant fields with change log availability.

One option can be to activate system database logging for HRMS fields in Transaction SE11 under technical settings of the database table. Evaluate logs using Transaction SCU3 and ensure analysis is performed on the database table, not a customizing object.

Note that activating data logging on large and intensively used database tables may cause performance issues; avoid enabling this on a productive system without consulting experts.

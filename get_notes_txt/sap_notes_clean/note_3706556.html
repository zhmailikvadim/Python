Symptom

  • No items are aggregated in the intrastat declaration file althoug it should
  • Items are aggregated for for the wrong characteristics

Environment

  • Sales And Distribution (SD)
  • Logistics Execution (LE)
  • SAP R/3
  • SAP R/3 Enterprise 4.7
  • SAP ERP Central Component
  • SAP ERP
  • SAP Enhancement package for SAP ERP
  • SAP Enhancement package for SAP ERP, version for SAP HANA

Reproducing the Issue

  1. Go to transaction VEB1.
  2. Select the appropriate country.
  3. Ensure that the ‘Aggregation Criteria’ flag (ACFLG) is set correctly. Currently, this flag is either not set or assigned to the incorrect field.

Cause

The configuration for the aggregation criteria is either missing or has been set incorrectly.

Resolution

Definition of the aggregation criteria

  1. To define aggregation criteria in the system:

    1. Go to transaction VEB1.
    2. Select the relevant country.
    3. Specify the required characteristics and set the ‘Aggreg crit.’ (ACFLG) flag accordingly.

Aggregation criterion

  • The Aggregation Criterion indicator determines whether a particular field is used to aggregate data for the specified declaration type. Data aggregation combines multiple document items with the same characteristics into a single item, reducing both the number of items and the overall data volume in your declaration.
  • The values and quantities from the individual items are totaled accordingly. 
  • Data aggregation is typically applied when you generate the final declaration to the authorities, such as when creating the declaration form or output file.

Use

  • Set this indicator on fields you want the system to use as aggregation criteria.
  • If you do not configure a specific criterion in Customizing for the relevant declaration type, the system will automatically utilize standard characteristics in line with the applicable legal regulations.
  • If you choose to define your own characteristics, ensure they comply with the necessary legal requirements.

Example

  • If you want the system to aggregate only records with the same commodity code, select the ‘commodity code’ field (STATWAREN) as the aggregation criterion. To include the business transaction type as an additional aggregation criterion, select the ‘business transaction type’ field (ARTGESCHA) as well.
  • If you prefer to aggregate by material number, add the field ‘MATNR’ and remove the ‘Aggreg crit.’ flag from the commodity code field.

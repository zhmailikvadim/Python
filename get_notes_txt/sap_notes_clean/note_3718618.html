Symptom

You use multiple Calculation Procedures that contain scripts and expect the system to generate unique Classes for each procedure without conflicts. However, the system generates a Class with a suffix derived from the first five characters of the Calculation Procedure name. When multiple Calculation Procedures share the same first five characters, the system generates identical Class suffixes. This leads to naming conflicts and, at times, transport locking issues during development and deployment.

This note corrects the behavior by ensuring that the generated Class names are uniquely derived and do not cause conflicts when multiple Calculation Procedures share similar name prefixes.

Other Terms

/IRM/0GCLEM, Locked, Multiple Transport, Suffixes

Reason and Prerequisites

This is caused by program error

Solution

Please implement the correction instructions. These corrections are only valid if you have applied the following Support Packs.

Release V/4 – 2021, 2022, 2022 V2, 2023, and 2023V2.

Use transaction SNOTE to implement the correction instructions specified below and activate the affected program object

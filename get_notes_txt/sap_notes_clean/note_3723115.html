Symptom

When calculation payroll for trainers before implementation of SAP Note 3720025 the payroll results stored in cluster may contain incorrect HI and Tax bases. SAP Note 3720025 fixes the calculation functions but if the results were already stored to the cluster with incorrect value SAP strongly recommends you to recalculate payroll for all trainers with retro back to January 2026 with "forced reload of base for trainers" mode. Mentioned mode could be enabled via entering wage type /28M into infotype 0014 for period of January 2026 (and/or February 2026, if results for 02/2026 are incorrect as well) with letter "R" filled in field Assignment number (ZUORD). Since value (BETRG) is obligatory for this wage type, enter any value between 0 and 300 - when ZUORD contains R in the first place, the value in BETRG is ignored. It just enables the "forced reload of base for trainers" mode and then in calculation base for HI (/3A2) and base for Tax (/3A6) will be reloaded and stored correctly.

Forced reload of base is not relevant for gross-up run nor for any runs with AS-PARM2 starting with SIN string (used in Public Sector for simulation purposes).

Other Terms

HRSK, HSKCALCM, HSKCALC0, SKCZP, SKCTX, /28M, P0014-ZUORD

Reason and Prerequisites

This SAP Note is relevant only for Slovakia (HRSK)

Solution

Source code of payroll functions SKCZP and SKCTX was updated to process forced reload of base for trainers functionality.

Implementation

  • As a general rule, SAP recommends that you install a solution by applying a Support Package. However, if you need to install a solution earlier, use the Note Assistant to implement the correction instruction. You can find more information about the Note Assistant in SAP Service Marketplace, under service.sap.com/note-assistant.

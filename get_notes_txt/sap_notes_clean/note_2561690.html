Symptom

Poor performance and high CPU consumption because of missing indexes on large table KNVK

Other Terms

HANA, Performance, secondary indexes, KNVK-LIFNR

Reason and Prerequisites

Normally, the existing non-unique secondary indices are not needed on HANA DB anymore. But there might be also some few situations where HANA DB benefits from a secondary index. This is especially true for highly selective queries on non-primary key fields. These queries can be significantly improved by indexes on single fields which are most selective.

Solution

Follow the steps of the Manual Activity or implement Support Package.

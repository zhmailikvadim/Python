Symptom

Error message: "Validity period cannot be adhered to" /SAPAPO/DM_SCHEDFAIL025 will be encountered during product heuristic execution, and the generated planned order is created within the planning time fence, containing incorrect components based on validity dates.

Environment

  • SAP Supply Chain Management
  • SAP Advanced Planning and Optimization

Reproducing the Issue

  1. Go into product view through transaction /SAPAPO/RRP3.
  2. Run Product Heuristic in product view
  3. After the planning run is finished, check in the log, following message will be got:

    Validity period cannot be adhered to

    Message No. /SAPAPO/DM_SCHEDFAIL025

Cause

It is caused by validities logic in PPDS side, which is already noted in section "4. Validities in the APO system" of note 385602 - Validity in APO (Documentation). 

Based on the components' validity periods from the corresponding PDS, after production planning run, a planned order within planning time fence will be generated automatically by system. Usually heuristic is not allowed to create orders inside the planning time fence. If the result of scheduling puts the order inside the planning time fence, it will try to reschedule the order into the future. While scheduling the order after the planning time fence is not possible because that will violate the validity period of the order(defined by PDS), then message “Validity period cannot be adhered to” will be popped up.

Resolution

It's not a problem that following message is raised. It tells that the system tried to schedule the order on a specific date, but it was not possible because the validity period is restrictive.

Validity period cannot be adhered to

Message No. /SAPAPO/DM_SCHEDFAIL025

 

This message could be accepted, and there are two options to deal with it:

  1. Manually reschedule the order to a later date if required.
  2. Change the date of the requirement to a later date and execute the product heuristic once more.

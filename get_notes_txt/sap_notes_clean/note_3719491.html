Symptom

Generally, SAP's advise is, that customer should not be using the pension model wage types but should use customer copies, which would then prevent any SAP deliveries affecting these wage types unnecessarily.

Environment

  • Human Capital Management (HCM)
  • SAP R/3
  • SAP R/3 Enterprise 4.7
  • SAP ERP Core Component
  • SAP ERP
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP SuccessFactors Employee Central integration to SAP Business
  • SAP SuccessFactors Employee Central Payroll

Resolution

In SAP Note 3633452 PY-GB: Pension contribution wage type configuration changes to handle WPBP splits, SAP delivered the following pension wage types:
 
/P0A, /P0O, /P0P, /P0R - Employee pension contributions
/P1A, /P1O, /P1P, /P1R - Employer pension contributions
/P2A, /P2O, /P2P, /P2R - Employee Additional Voluntary contributions (AVC)
/P3A, /P3O, /P3P, /P3R - Employee Free Standing Additional Voluntary Contributions (FSAVC)                                  
/P4A, /P4O, /P4P, /P4R - Employee Tax Free contributions                          
/P5A, /P5O, /P5P, /P5R - Employee Tax Free Additional Voluntary Contributions (AVC)                  
/P6A, /P6O, /P6P, /P6R - Employee pension holiday nominal value       
/P7A, /P7O, /P7P, /P7R - Employer pension holiday nominal value


If you use the model pension wage types for your customer-specific schemes, it is best to save them before the SP implementation and restore the possibly overwritten values (such as evaluation class values) except processing classes 30, 62, 63.

If you have been using value 2 in processing class 48 to classify pension contributions for being NOT paid under a Net Pay Arrangement (Contribution that is NOT paid under a Net Pay Arrangement is a Net Deduction), restore its value.

If by any chance, the model wage types have been overwritten and you are missing the Net Deduction in the RTI it is not possible to correct it retrospectively. You can however use the BADI in the FPS to report the correct figures for the affected month and change the settings before the next months payroll to ensure that the reporting is correct going forward.

Please find more information in our KBA 3642271 PY-GB: FPS Badi

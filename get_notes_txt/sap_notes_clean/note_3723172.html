Symptom

After Data Migration to S/4HANA, there is a duplicate line item in view FAAV_ANEP for the document with AWTYP AS91. It is caused by an inconsistency in the field AWITEM. The relevant document is in both ACDOCA and FAAT_DOC_IT. In ACDOCA, the field AWITEM is empty. Unlike that, there is at least one relevant line item with a non-zero AWITEM in FAAT_DOC_IT. As a consequence, there are duplicate amounts in AA reports based on FAAV_ANEP.

Other Terms

SAP ECC to S/4HANA brownfield conversion, fins_mig_status, Fixed Asset Accounting, MUJ, AWITEM, FAAT_DOC_IT, FAAV_ANEP duplicity, AS91, legacy data transfer, ERLBT, SLALITTYPE 7900

Reason and Prerequisites

This is a program error in the migration step MUJ.

Solution

Implement the attached correction instructions or the relevant Support Pack in the target migration system.

Reset to and repeat from the MUJ step. Consult SAP Notes 2270540 and 2138644 on how to correctly reset and repeat steps.

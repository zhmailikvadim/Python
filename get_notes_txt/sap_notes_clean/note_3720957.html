Symptom

  • Cost condition is redetermined when 'Return Reason' field is changed at 'Returns' tab in Return Order.
  • A new pricing call is performed when 'Return Reason' is changed.

Environment

  • Sales And Distribution (SD)
  • Logistics Execution (LE)
  • SAP R/3
  • SAP R/3 Enterprise 4.7
  • SAP ERP Central Component
  • SAP ERP
  • SAP Enhancement package for SAP ERP
  • SAP Enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA
  • S/4HANA Cloud Private Edition

Reproducing the Issue

  1. Create a return order via VA01 and see that cost condition is available in item conditions tab.
  2. Go back to header and change the "Return Reason" in the Returns tab.
  3. Go to item pricing conditions again and see that condition was removed.

Cause

  • Cost condition is removed due to the standard ARM coding, that triggers a new pricing execution upon return reason change.
  • This new pricing execution is performed with pricing type "G", which redetermines cost conditions.

Resolution

This logic is hardcoded in ARM coding in the following logic:

  • CL_ADVRET_SD_SALES_ITEM_PRICNG
  • IF_BADI_SD_SALES_ITEM_PRICING~ITEM_PRICING_CALL
  • Identify the condition where the return reason change triggers a new pricing execution (cv_call_pricing = 'X').

See Also

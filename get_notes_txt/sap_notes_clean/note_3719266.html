Symptom

Version History

Version 1 & 2 (23.02.2026): Pilot Version

Version 3 (27.02.2026): General Availability (No changes from Pilot Version)

An employee was terminated, and that leaving date was sent to ROS, and passed to NAERSA, then NAERSA issue an APEN with an event type of 'Left Employment Initiation'. Subsequently, if that same employee gets reinstated - not re-hired (i.e. the termination action gets deleted), then ROS and NEARSA are out of step with payroll. The AEPN is 'incorrect' but still must be applied, because it is the current active AEPN. Before the introduction of this SAP Note, Payroll Function P3448 checks that the event type of 'Left Employment Initiation' is only applied to leavers and issues the following rejection in payroll.

HRIE_AE(116) Error; EMPID &1 has 'Left Employment Initiation' AEPN, but has not left!

After the introduction of this SAP Note, Payroll Function P3448 checks that the event type of 'Left Employment Initiation' is usually applied to leavers and issues the following warning in payroll, but importantly still allows the AEPN to be processed. This allows time for ROS and NAERSA to "catch up" with payroll.

HRIE_AE(116) Warning; EMPID &1 has 'Left Employment Initiation' AEPN, but has not left

Other Terms

Payroll Function P3448, Leaver Re-instated

EITHER

Apply the relevant HRSP

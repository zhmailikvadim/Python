Symptom

MOVE_TO_LIT_NOTALLOWED_NODATA dump happens while changing class type 023 with CLMM transaction

Environment

  • SAP ERP Classification (CA)
  • SAP R/3
  • SAP R/3 Enterprise
  • SAP ERP Central Component
  • SAP ERP
  • SAP Enhancement package for SAP ERP
  • SAP Enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA, on-premise
  • SAP S/4HANA Cloud Private Edition

Reproducing the Issue

  1. Start transaction CLMM.
  2. Choose “Set and Change Values”.
  3. Select class type 023 (Batch) and enter the relevant batch class.
  4. In the Characteristics tab, enter the characteristic(s) to be updated.
  5. In the Assignments tab, assign the same batch class.
  6. In the Objects tab, select multiple batch records to be updated.
  7. Choose Continue to run the mass change.
  8. The system terminates with short dump MOVE_TO_LIT_NOTALLOWED_NODATA and no updates are performed.

Cause

For class types whose object table key field has internal format ABAP Type = X (including batch class types 022/023), changing value assignments using CLMM is not supported by design.

Resolution

There are multiple options to choose from:

  1. Maintain batch classification values via the batch master maintenance (for example, the batch maintenance transaction).
  2. For mass updates, use appropriate BAPIs such as BAPI_BATCH_* or BAPI_OBJCL_*.

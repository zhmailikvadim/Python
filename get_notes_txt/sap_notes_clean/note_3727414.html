Symptom

In the process "Receipt of 55691 from Responsible DSO to MP" (DE_C_MDC_55691_DSO2MP_RECV), when a Point of Delivery (PoD) record exists that sent message 55691 with response status A01 through DSO change processing, retriggering the process automatically sends message 55692 with an incorrect response status A02 instead of A01.

 This issue is relevant for the market roles: DSO, SUP, MOS.

Other Terms

MDC, DE_C_MDC_55691_DSO2MP_RECV, DSO change processing, message 55691, RESPONSE_STATUS, DATA_USAGE_PERIOD, /US4G/CL_CHK_MDC_PROC, A01, A02.

Reason and Prerequisites

 The error occurred because the system lacked the functionality to maintain the correct response status when retriggering the DSO change process.

Solution

Implement the corrections of this SAP Note or import the relevant support package.

 After implementing this SAP Note, when you retrigger a PoD record that sent message 55691 with response status A01 through DSO change processing, the system automatically sends message 55692 with the correct response status A01. Please retrigger the DSO change process in the old DSO back-end system and verify the fix by checking that step 1600-0100 runs automatically, the RESPONSE_STATUS field of component DATA_USAGE_PERIOD is set to Accept (A01), and the market partner on the cloud side receives A01 in step 2000 and sends message 55692 with A01 to the old DSO.

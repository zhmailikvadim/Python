Symptom

  • BCF entries in period 0 have already been posted with blank transaction type and derive default subitem(e.g.,00).
  • Subsequent run of realignment generates GRRA entries reversing the subitem 00 and posting in another subitem(default one maintained in CX1S4,e.g.,95) instead.
  • There is inconsistency between the subitems generated by Realignment and BCF.

Environment

  • SAP Fiori for SAP S/4HANA Finance
  • SAP S/4HANA Finance for Group Reporting

Reproducing the Issue

Run realignment in period 001 in YTD mode.

Cause

The BCF data in ACDOCA has an empty Transaction type.

Resolution

Based on the example mentioned by Symptom, The derivation of subitem is different in balance carry forward compared substitution during the year and realignment in case the sender field transaction type is empty. The system uses the default subitem 95 when the substitution runs in a regular period. In BCF, it uses the carry-forward subitem 00 that is assigned to 95 in CX1S4. 

The issue happens only in the special case that the transaction type is empty in a BCF record in ACDOCA. If it cannot be accepted, please make sure that there are no BCF records with empty transaction type.

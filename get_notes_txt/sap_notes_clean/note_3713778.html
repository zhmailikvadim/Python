Symptom

  • Inconsistent error message "Workflow based Approval Process is inactive" raised (W2B2343).
  • Error occurs when adding a manual price condition type in the "Manage Sales Docs with Customer-Expected Price" app (ID: F2713).
  • Users are able to close the error/pop-up and continue working, but the error is confusing as it should not occur.

Environment

  • Sales And Distribution (SD)
  • Fiori
  • SAP S/4HANA
  • SAP S/4HANA Cloud Private Edition

Reproducing the Issue

  1. Access the "Manage Sales Docs with Customer-Expected Price" app (ID: F2713).
  2. Add a manual price condition type to a sales document.
  3. Observe the inconsistent error message "Workflow based Approval Process is inactive" (W2B2343).

Cause

Normally, the error W2B2343 is raised because the "Activate Workflow Approval" option is not selected in customizing (transaction code WB2B_CUS -> Active Component -> Node).

Resolution

  1. Navigate to transaction WB2B_CUS -> Active Component -> Node.
  2. Select the "Activate Workflow Approval" checkbox.
  3. Save the changes.
  4. Retest the scenario in the "Manage Sales Docs with Customer-Expected Price" app (ID: F2713).

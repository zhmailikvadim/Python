Symptom

Potential HTTP request smuggling vulnerability (as described in SAP Note 3720138) reported in SAP Approuter components.

This note applies to SAP Web IDE for SAP HANA, starting from SPS08 patch 3 and higher.

A security concern was raised regarding possible request-response desynchronization between frontend and backend servers.

Other Terms

SAP Web IDE for SAP HANA, @sap/approuter, HTTP request smuggling, request-response desynchronization, security vulnerability, reverse proxy, static resource serving, CVE, CVSS.

Reason and Prerequisites

The described vulnerability requires SAP Approuter to function as a reverse proxy to a backend system, creating a mismatch in how HTTP request boundaries are interpreted.

From SPS08 patch 3 and higher, in the SAP Web IDE for SAP HANA tools component (@sap/approuter), the Approuter serves exclusively as a web server for static resources, without proxying requests to a backend.

As this deployment does not include a reverse proxy setup, the condition necessary for exploiting the vulnerability is not present.

Solution

No action is required. The vulnerability described in SAP Note 3720138 is not applicable to the SAP Web IDE for SAP HANA tools component.

The architectural prerequisites for exploitation do not exist in this scenario; therefore, the vulnerability cannot be triggered and no mitigation is necessary.

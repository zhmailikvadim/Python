Symptom

You are using SAP Process Orchestration(PO) and PO adapter with configured OAuth 2.0 authentication. You observe that your scenario is not working and the following exception in the traces:

Error: java.lang.RuntimeException: com.sap.aii.af.lib.util.UtilException: Key ID <XYZ> already exists in database: com.sap.sql.exception.OpenSQLIntegrityConstraintViolationException: [301]: unique constraint violated: Table(XI_AF_OAUTH_CACHE) with error: unique constraint violation on pos=0 for table ABC::ABCDEF:XI_AF_OAUTH_CACHE, key: KEY_ID=<XYZ> already exists as udiv=xx
OpenSQLExceptionCategories: [NON_TRANSIENT, INTEGRITY_CONSTRAINT_VIOLATION]

Other Terms

 OAuth2.0

Reason and Prerequisites

For performance optimization there is OAuth 2.0 token caching implemented in the PO. Because of a functional error in rare cases there is a database error caused by content constraint violation.

Program error

Solution

Apply the patch matching your Support Package version as listed in the Support Package & Patches section of this  SAP Note. Follow the instructions described in SAP Note 952402.

As a workaround you can disable token caching for the adapter if this option is supported by adapter type used in the scenario.

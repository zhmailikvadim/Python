Symptom

This SAP Note fixes the following unpleasant behaviors in transaction UCON_CHW:

1) The logged calls shown under 'Logged HTTP Allowlist Checks' are not in descending chronological order.

2) Double-click on a cell/row does not update the detailed information shown below the table.

3) Specifically for entry type 04 (CORS): The last logged occurrence of a call covered by the 'No-log-list' is still shown under 'Blocked'.

Solution

Implement this SAP Note for an improvement in the corresponding behavior:

1) Logged calls are shown in descending chronological order, allowing for a quicker overview over the recent blocked calls.

2) Double-click on a cell/row to show detailed information for that entry.

3) Specifically for entry type 04 (CORS): Blocked calls that are now covered by the 'No-log-list' are hidden from the list (still visible and marked as 'covered by no-log-list' under option 'All').

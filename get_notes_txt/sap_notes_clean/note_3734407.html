Symptom

  • Analytical model is not listed as a table/view when accessed via an JDBC/ODBC connection.
  • The same analytical model is accessible and works in SAC.
  • No error messages are shown.

Environment

  • SAP Datasphere

Reproducing the Issue

  1. Deploy an analytical model in the data builder.
  2. Verify the model is accessible in SAC.
  3. Connect to the space/database via JDBC/ODBC and browse tables/views; the analytical model does not appear.

Cause

  • This is expected behavior: only views exposed for consumption are visible via JDBC/ODBC connections.
  • Analytical models are not exposed via SQL/ODBC and should be consumed via OData.

Resolution

  1. Do not expect analytical models to appear in SQL/ODBC catalogs.
  2. Consume the analytical model via OData by configuring an OData connection in the consuming tool.
  3. If SQL/ODBC access is required, expose a view for consumption in the data builder and query that view via ODBC instead of the analytical model.

See Also

refer to: Consuming Data Exposed by SAP Datasphere

"TypeError: cannot read properties of undefined”

Environment

  • SAP S/4HANA 2023
  • Fiori elements OData V4
  • Custom Fields and Logic (App ID: F1481)

Cause

  • In the OData V4 Service Binding metadata for the ValueList of the custom code list field, the property path was not escaped correctly when the custom field identifier contains "/" character.
  • The unescaped slashes were interpreted as navigation, leading to a broken value help binding and no results.

Resolution

  1. Implement SAP Note 3536314.
  2. Clear caches as described in the cache housekeeping KBA 2319491.
  3. Reload the app and retest the scenario.

See Also

  • SAP Note 3536314 - Value help parameter escaping corrections (OData V4)
  • KBA 2319491 - How to clean the cache in SAP Fiori

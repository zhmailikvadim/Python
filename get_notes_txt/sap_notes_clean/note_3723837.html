Symptom

The Impact Analysis reports false‑positive errors for Read‑Only conflicts when the ZDIMPANA.ZIP file contains periods during which regular transports were imported into the productive system.

Reason and Prerequisites

When the tracing data is picked up a rough check detects if transports were imported but does not pick up any details which tables received imports. During SUM execution a transport lock is enforced and the Impact Analysis should reflect this as best as possible.To avoid misleading results in the Impact Analysis, transports should ideally not be included in the trace data. But this is not always possible.

Solution

This note enhances the collection of the tracing data by storing transport information on table level in the ZDIMPANA.ZIP file.

The Impact Analysis will evaluate these entries and detect potential parallel imports on the affected tables.

If a table with imports is detected, the severity of the Read‑Only conflict is automatically reduced from “Critical” to “Warning”.

The message will show an additonal hint that imports occured:

"TABL <Tabname> with imports will become read-only, but has 291,7/291,7(statement/row) changes/day"

Important to note: it can not be detected if a table is changed by daily business activities and imports. Therefore Read‑Only warnings must still be reviewed carefully.

Symptom

The correction report ZPM_NOT_QMDAB_02 of note 2774588 does not fetch the NOCO date from table JCDS. 

Is there any other SAP note which fetches the NOCO date from table like JCDS having status -I0072?

Environment

  • SAP ERP Quality Management (QM)
  • SAP ERP
  • SAP R/3
  • SAP R/3 Enterprise
  • SAP ERP Central Component
  • SAP Enhancement package for SAP ERP
  • SAP Enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA, on-premise
  • SAP S/4HANA Cloud Private Edition

Cause

It is by standard design. 

Resolution

The correction program ZPM_NOT_QMDAB_02 has been specifically developed to address notifications that were completed earlier but still display an initial completion date.

Since the system did not record a completion timestamp at the time, the program relies on the last change date and time (AEDAT / AEZEIT) as the most reliable fallback. This approach aligns with standard SAP logic as outlined in the note.

It is important to note that this logic is intentional because the system does not retroactively store the NOCO timestamp in QMEL when it was initially missed. As a result, the program uses the last change date and time to fill in the missing data. 

To clarify further:

  • The program does not retrieve the actual NOCO date from status management, as the necessary timestamp was not recorded during notification completion.
  • It does not access JCDS or the I0072 status, and the SAP note does not recommend doing so.

There is no other SAP Note available that provides a standard correction tool for retrieving the completion date from JCDS. 

Keywords

ZPM_NOT_QMDAB_02, notification completion date, NOCO, JCDS

Symptom

The following error appears while launching PI tools (ESR and ID).

Java Web Start Error: Could not download resource [l= http://hostname:port/dir/directory/iaik_jsse.jar v=null s=I] 

From system default traces the following error can be observed:

Could not download resource [l=http://hostname:port/dir/directory/iaik_jsse.jar v=null s=I] from any of these urls [http://hostname:port/dir/directory/iaik_jsse.jar] caused byExecutionException: java.io.FileNotFoundException: http://hostname:port/dir/directory/iaik_jsse.jar 

Environment

SAP NetWeaver

SAP Process Orchestration 7.5

Cause

Missing jar files.

Resolution

Navigate to /usr/sap/<SID>/SYS/global/xi/repository_server/javaws/repository and check whether all JARs listed in the repository.jnlp file are present. 
Similarly,  

If they are not present, try the following:

  • Delete the directory /usr/sap/<SID>/SYS/global/xi/repository_server/. After deleting, re-download a new repository.jnlp file. After downloading the new jnlp and before launching navigate to the /usr/sap/<SID>/SYS/global/xi/repository_server/javaws/repository and check whether all JARs are present. You can also open and check the init.info file and it should say "Okay, no problems detected so far." Then you can launch the repository.jnlp file.
  • Alternatively, you can also copy all the contents /usr/sap/<SID>/SYS/global/xi/repository_server/javaws/repository from a system where there are no issues launching to this system. Then you can launch the repository.jnlp file.

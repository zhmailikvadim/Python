Symptom

In SSAM, a priority alert icon appears next to certain notifications depending on the notification priority.

You may observe the following behavior:

  • Multiple priorities are configured in the SAP backend (for example 0, 1, 2, 3, 4).
  • However, the priority icon in the SSAM mobile application appears only for some of them.
  • For example:
    • In a system with priorities 0, 1, 2, 3, 4, the icon appears only for 0, 1, and 2.
    • Notifications with priorities 3 and 4 do not display the priority icon.

Environment

SAP Service and Asset Manager

Reproducing the Issue

  1. Open the SSAM mobile application.
  2. Navigate to the Notifications list.
  3. Open a notification that has a defined priority value.
  4. Observe the priority indicator icon displayed in the notification header.
  5. Compare notifications with different priorities (for example 0, 1, 2, 3, 4).
  6. Result:
    The icon is displayed only for priorities lower than 3.

Cause

The display of the priority indicator icon is controlled by the metadata rule and not by backend configuration.

In the SSAM mobile application, the icon is displayed based on a predefined condition implemented in the application code.

In SSAM metadata, we have Rules/Common/Library/CommonLibrary.js:

    static shouldDisplayPriorityIcon(context, priority) {
        if (priority < 3 || priority === '*') {
            return IsAndroid(context) ? '/SAPAssetManager/Images/high_priority_icon.android.png' : '/SAPAssetManager/Images/high_priority_icon.png';
        }
        return '';
    }

Because of this logic:

  • The icon is displayed only for priorities less than 3.
  • Backend priority configuration does not influence this rule.

Resolution

This behavior is standard application logic in SAP Service and Asset Manager.

If different behavior is required, it would require:

  • Application customization (modification of metadata), or
  • A feature request submitted through SAP Customer Influence.

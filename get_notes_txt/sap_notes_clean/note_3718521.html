Symptom

You use the Object Match functionality in the Data Object Fiori application and expect the input values and suggestion list to refresh correctly when searching matches for different Data Object lines. However, the input values remain the same even when navigating to other Data Object lines. While the process works when handled line by line and followed by an action such as Save or Reprocess after an account update, issues occur when no intermediate action is performed. After the first successful search, suggestions are populated correctly. For a subsequent search with a different (invalid) End Customer, no suggestions are displayed—even though the account exists. The suggestions only appear again after navigating back to the Data Object lines, performing an action such as Reprocess, and then returning to the address match screen, indicating the screen is not refreshing properly and appears to freeze.

This note corrects the behavior by ensuring that input values and suggestion lists are refreshed dynamically for each Data Object line without requiring additional actions.

Other Terms

/VTA/BSESM, DO, Match, Dynamic Refresh, Screen Refresh

Reason and Prerequisites

This is caused by program error

Solution

Please implement the correction instructions. These corrections are only valid if you have applied the following Support Packs.

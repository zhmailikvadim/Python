Symptom

  • ABAP instance doesn't start

  • In the dispatcher trace file (dev_disp):

     "DpHdlDeadWp: Wxx (pid ...) died (severity=0, status=0, action=WP_ROLL_IN_AFTER_DUMP, DP_WP_DUMPED)".

  • The work process trace files (dev_w*) show :

     "***LOG Q0E=> DpSigGenHandler, Exception.  and

  • Call stack references enqueue handling functions like:

    *---->(FAULT) Stack Back Trace <----*
    000000ABA89FEDF0 00007FF74EBDAD04 000000ABA89FF590 disp+work!EnqGetServerType [enxxhead.c (13422)]
    000000ABA89FEFA0 00007FF74EBDAC9A 000000ABA89FF590 disp+work!ThSetEnqName [thEnqueue.c (774)]

    000000ABA89FF320 00007FF74E93DD95 0000000000000001 disp+work!ThAdXMsBuf [thxxmbuf.c (613)]

Environment

  • Product: SAP NetWeaver Application Server for ABAP 
  • ABAP Platform

Cause

The ABAP instance is not configured to connect with the Enqueue server running in the ASCS instance.

For example:

Standalone Enqueue server (ENSA 1) is running and "enque/process_location" was set incorrectly (e.g., "OPTMIZE")
This misconfiguration prevented proper enqueue server selection and caused work processes to terminate during startup.

Resolution

  1. Identify which the Enqueue server is running in the ASCS instance :
    See Classical Central Instance or Standalone Enqueue

  2. Configure the appropriate Instance or DEFAULT profile according to:

    - ENSA 1: 

       enque/process_location = REMOTESA

       See Profile Parameter Settings for the Standalone Enqueue Server

    - ENSA 2: 

       enq/enable=TRUE

       See Parameter Reference of Standalone Enqueue Server 2

  3. Save the profile changes.
  4. Restart the affected ABAP instance.

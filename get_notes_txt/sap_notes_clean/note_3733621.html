Symptom

Federated apps from an on-premise Fiori system with SAPUI5 1.71.77 show SAPUI5 version 1.145.0 when launched in the Workzone site

Environment

  • Product: SAP Build Work Zone, standard edition
  • Core and Runtime: ABAP front-end server with SAPUI5 1.71.77 (SAP_UI 7.54 SP18); new SAPUI5 runtime repository approach (SUI5_CONTENT) in place; site initially loading SAPUI5 cloud version

Reproducing the Issue

  1. Federate (expose) standard Fiori apps from the on‑premise ABAP front-end server to the site.
  2. Open a federated app in the site and check the SAPUI5 version in the runtime info.
  3. Observe that the version shown is 1.145.0(or current version) instead of the expected 1.71.77.

Cause

  • The on‑premise system is in a mixed state after applying a service pack that introduces the new SAPUI5 runtime repository method (SUI5_CONTENT) without the corresponding delivery note for SAPUI5 1.71.77. Required prerequisites (for example, the note introducing the new repository) were not yet applied, which led to incorrect version selection in the site

Resolution

  1. Implement SAP Note 0003677828 - ABAP SAPUI5 1.71.77 on the ABAP front‑end server to deliver the 1.71.77 content via the new runtime repository and pull in required prerequisites (including the new repository enablement).
  2. Re-expose all relevant roles, apps from the on‑premise front‑end server to the site so the site receives the updated content.
  3. Launch the federated apps again in the site and verify the SAPUI5 runtime now matches 1.71.77

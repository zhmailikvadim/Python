Symptom

Availability check on a planned order in product view (transction /SAPAPO/RRP3) terminates with runtime error COMPUTE_FLOAT_ZERODIVIDE and exception CX_SY_ZERODIVIDE wiith short text 'Division by 0 (type F) in program "CL_ATP_GROUP_MANUAL_DISTRIBUTECP"'.

Other Terms

COMPUTE_FLOAT_ZERODIVIDE, CX_SY_ZERODIVIDE, CL_ATP_GROUP_MANUAL_DISTRIBUTECP, CL_ATP_GROUP_MANUAL_DISTRIBUTECM00B, COMPUTE_TOTALS, Bill of Materials, BOM

Reason and Prerequisites

Reason:

  • Program error.

Prerequisites:

  • Planned order item (with a BOM material) has a component material with zero requirement.

Solution

Implement the attached correction instructions.

Symptom

After repointing a Crystal Report from a UNV universe to a UNX universe and saving it to Business Objects Enterprise over the original report, the report displays references to both universes instead of exclusively referencing the UNX universe.

Other Terms

Crystal Reports, retains UNV, repoiting UNX, UNX, UNV.

Reason and Prerequisites

Environment:

  • SAP BusinessObjects BI Platform 4.3/2025
  • Crystal Reports 2020/2025

Steps to Reproduce:

  1. Start with a Crystal Reports 2020 report that uses a UNV universe as a source.
  2. Set the datasource location to point to the equivalent UNX universe.
  3. Save the repointed Crystal Report out to Business Objects Enterprise by overwriting the existing Crystal Report.
  4. Observe that the Crystal Report is still referencing the UNV universe as well as the UNX universe.

Solution

Currently there are two workaround to the situation.

Workaround 1:

Select the option "Save as" and create a new file instead of overwriting an existing one.

Workaround 2:

Run the sample sdk code attached, to clean up the properties after the overwriting.

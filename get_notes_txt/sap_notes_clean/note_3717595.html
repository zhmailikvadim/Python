Symptom

  • Fields from the Work Schedule object, such as Average Hours Per Day, Average Hours Per Week, Average Hours Per Month, Average Hours Per Year, and Average Working Days Per Week, are not available for reporting in Story Reports.
  • These fields are system-calculated values within the Work Schedule configuration and are not exposed as reportable data fields.
  • Story reporting only exposes supported MDF and transactional fields, not system-derived calculation fields.

Environment

  • SAP SuccessFactors Employee Central 
  • Time Off

Reproducing the Issue

  1. Attempt to create a Story Report based on the Work Schedule object.
  2. Search for fields such as Average Hours Per Day, Average Hours Per Week, Average Hours Per Month, Average Hours Per Year, and Average Working Days Per Week in the reporting table.
  3. Observe that these fields are not available in the reporting table.

Cause

Resolution

We Confirm that the fields in question are system-calculated values within the Work Schedule configuration and are not stored as reportable data fields.

Please note that Story reporting only exposes supported MDF and transactional fields, and system-derived calculation fields from Work Schedule are not currently available for reporting.

Refer to the official documentation for further details:

Reporting of Planned Working Time – SAP SuccessFactors Employee Central Time Management

If reporting of this information is a business requirement, submit an enhancement request via the Customer Influence portal.

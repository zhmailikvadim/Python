Symptom

  • When validating the address for a individual customer/account with PO box fields, the validation popup shows errors.
  • Error message: Unable to process your request due to an external service issue.
  • Error message: Cannot parse outbound synchronous message response. The expected content type for message response is application/json.

Environment

  • SAP Service Cloud Version 2
  • SAP Sales cloud version 2

Reproducing the Issue

  1. Create a new individual customer/account and populate the address using PO box fields.
  2. Click the Validate Address button in the creation screen.
  3. Observe that the validation popup reports errors; if not visible, check the browser/network trace for the error details.

Cause

  • When the po box field is used, mandatory fields required by the external address validation service are missing (PO box country and PO box locality (City)). The external service returned a non-JSON error body, which the data connector could not parse.

Resolution

  1. When specifying a PO box, fill all required PO box fields before validating the address: PO box country and PO box city (po_box_locality).
  2. Save the address changes.
  3. Run Validate Address again and confirm that the validation completes without the external service and parsing errors.

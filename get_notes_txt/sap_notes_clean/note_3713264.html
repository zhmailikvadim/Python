Symptom

Condition type VPRS is not determined correctly based on the customizing from material master data in sales order creation via VA01.

Environment

•Sales And Distribution (SD)
•Logistics Execution (LE)
•SAP R/3
•SAP R/3 Enterprise 4.7
•SAP ERP Central Component
•SAP ERP
•SAP Enhancement package for SAP ERP
•SAP Enhancement package for SAP ERP, version for SAP HANA
•SAP S/4HANA

Cause

The issue occurs due to the future price configuration in the material master data. When the price determination date is set to 2025/12/17 or later, the system uses the future price from the material master data, which leads to the error during the determination of the condition type VPRS.

Resolution

In material master data, if pricing control is set to V, condition rate of condition type VPRS is determined from moving price. Otherwise, it is determined from standard price. If condition category is set to S, condition rate of condition type VPRS is determined from standard price, otherwise it is determined from moving price. If future price is maintained, system will compare the future date and pricing date. If future date is less than price date and future date is not maintained as initial, future price will be used.

Reason and Prerequisites

The SAP HANA instance was installed without the necessary preparation steps as documented in SAP Note 2078425 - Troubleshooting note for SAP HANA platform lifecycle management tool hdblcm - SAP for Me section How to prepare SAP HANA packages for installation or update.

The hdblcm installer execution for the Local Secure Store (LSS) fails to verify executable authenticity while registering the APL binaries in the LSS as <sid>adm user. SWPM is only able to execute hdblcm as <sid>adm user. 

Solution

To solve the issue, copy the original SIGNATURE.SMF file from the HANA archive to /usr/sap/<SID>/lss/shared/SIGNATURE.SMF and retry the failing SWPM step.

Additional information about missing signature files found in SAP Note 2078425 - Troubleshooting note for SAP HANA platform lifecycle management tool hdblcm - SAP for Me section Repairing installations with missing signature.

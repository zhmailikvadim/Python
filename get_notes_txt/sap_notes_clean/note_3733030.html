Symptom

This SAP Note provides you with corrections to the Mexico Chart of Accounts Reporting (MX_CHTACCTS_DCL) report in the Run Statutory Reports app for SAP Document and Reporting Compliance. It addresses the SAT Code field reported as blank in the report output.

When you run the MX_CHTACCTS_DCL report, the system incorrectly displays the SAT Code field as blank in the report output when the Financial Statement Version (FSV) Node is '000'.

Other Terms

DRC; Mexico; Electronic Accounting; MX_CHTACCTS_DCL; FSV

Reason and Prerequisites

This SAP Note is relevant only for Mexico.

To implement Electronic Accounting compliance reports, you need to install the Document and Reporting Compliance (DRC) framework. For more information, see Advanced Compliance Reporting.

You have installed the Electronic Accounting compliance reports for Mexico. For more information, see SAP Notes 2548443 or 2598781.

Solution

After implementing this SAP Note, the system correctly populates the SAT Code field with zeros when the FSV Node is '000'.

As a general rule, SAP recommends that you install a solution by applying a Support Package. However, if you need to install a solution earlier, use the Note Assistant to implement the correction instruction. You can find more information about the SAP Note Assistant in the SAP Support Portal, under SAP Note Assistant.

Documentation for SAP S/4HANA for Mexico is available in the system and in the SAP Help Portal. System documentation (such as field help, customizing, messages) is delivered via Support Packages. SAP Help Portal documentation is available at help.sap.com/viewer/s4h_op_mexico.

You can find blog posts with news and updates about the Electronic Accounting compliance reports in the SAP Community (https://blogs.sap.com/): follow the Advanced Compliance Reporting tag to find the posts.

Symptom

After rebooting the SAP Business One server machine, the Electronic Document Service (EDS) dashboard URL becomes inaccessible.

Environment

  • SAP Business One
  • SAP Business One version for HANA

Reproducing the Issue

Prerequisites:

Ensure the EDS dashboard URL is accessible and functioning normally before proceeding.

Scenario: 

Reboot the SAP Business One server machine.

Expected behavior:

The EDS dashboard URL should remain accessible, allowing continued work.

Actual behavior:

The EDS dashboard URL becomes inaccessible and displays the following error message:

{"Error":{"Code":400,"Message":"Could not retrieve OIDC service details from SLD","Description":"Contact Technical Support or System Administrator."}}

Resolution

After rebooting the server machine, restart the following services in the specified order:

On the SAP HANA server

systemctl restart sapb1servertools.service

systemctl restart sapb1servertools-authentication.service

systemctl restart sapb1edfbackend

On the Microsoft SQL Server

SAP Business One Server Tools Service (64-bit)

SAP Business One Server Tools Authentication Service

SAP Business One EDF Back-end

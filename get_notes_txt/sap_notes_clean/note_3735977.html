Symptom

You use SAP NetWeaver Process Integration (Exchange Infrastructure) and the report RSXMB_TABLE_SWITCH_CONTROL. You notice that you do not switch to each PI client on the output screen if the switch procedure is inactive.

Other Terms

Persistence, reorganization, switch, message, XI message, container, SXMS_CONF_SWITCH

Reason and Prerequisites

You use SAP NetWeaver Process Integration (Exchange Infrastructure) and use the report RSXMB_TABLE_SWITCH_CONTROL and the following conditions are met:

  • You have implemented the cross-client view from SAP Note 3393054.
  • The switch procedure is inactive.

You notice that there are no buttons on the output screen to display the number of messages in other PI clients.

Solution

Implement the attached correction instructions or import the specified support package.

After you implement the corrections, pushbuttons for displaying the messages in other PI clients are displayed for the container #1 even if the switch procedure is inactive.

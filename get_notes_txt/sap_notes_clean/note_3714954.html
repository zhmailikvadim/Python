Symptom

As described in regulatory change manager, under Tax treatment of employer paid family and medical leave contributions, this SAP Note provides you with the relevant system changes to address the updates from Business Software, Inc. (BSI) delivered in TaxFactory 11.0 Regulatory Bulletin 189, regarding the new PFML - Employer Pick-Up (139) tax type.

Other Terms

ER Pick-Up; PFML; Tax Type 139; BTXRATE; T5UTX; 10ERP; 10TSA; 10STT.

Solution

From now on, the system allows you to pick-up employee's PFML taxes. To enable this functionality, proceed as follows:

  • Add PFML - Employer Pick-Up (139) tax type into your taxability model.
  • Maintain the PFML Employer Pick-Up tax type selector (10ERP) feature with the Employee's PFML tax types for each tax authority you want to pick-up.
  • Maintain an entry for PFML - Employer Pick-Up (139) tax type for each tax authority in BSI: Company Experience Rate (V_BTXRATE) view with the desired pick-up rate.

If you do not maintain 10ERP and a BTXRATE entry for Tax Type 139, the system does not calculate any PFML - Employer Pick-Up (139) tax type amount.

Example

Consider an employee that receives USD 10,000.00, lives and works in Delaware (DE) tax authority. This employee pays taxes for two PFML tax types: EE PARENTAL LEAVE TAX STATE PLAN (125) and EE FAM CAREGIVER/QE LV TX STATE PLN (129), and rates 0.16% and 0.04%, respectively. In this scenario, you want to pick-up 50% of Employee's 125 and 129 taxes.

  • In 10ERP feature, you choose 125 and 129 tax types for Delaware to return the value "X", and otherwise to return a blank value. Do not use a value other than "X" for the selected tax types.
  • In V_BTXRATE table, you maintain an entry for Delaware and tax type 139 with experience rate as 0.5.

After the configuration is finished and you run payroll, the system automatically recalculates the 125, 129, and 139 tax type rates as follows (which can be checked in Experience Rate (LUD ER:) parameter from BSI interface):

  • 125 Experience rate: 0.0016 * (1 - 0.5) = 0.0008
  • 129 Experience rate: 0.0004 * (1 - 0.5) = 0.0002
  • 139 Experience rate: (0.0016 + 0.0004) * 0.5 = 0.0010

As a consequence, the tax calculation results are:

  • 125 tax result: USD 8.00
  • 129 tax result: USD 2.00
  • 139 tax result: USD 10.00

Note: the calculated Employer Pick-Up contribution amount is included in employee income and wages subject to FIT, FITW, FICA, and FUTA taxes. For more information, see TaxFactory 11 Regulatory Bulletin 189.

PFML tax types rates maintenance in V_BTXRATE and V_T5UTX

  • The system searches first for any experience rate maintained in V_BTXRATE for the selected employee's PFML tax type to apply the pick-up percentage. In case it does not find, the system uses the default rate from Tax ceilings and rates (V_T5UTX) table.
  • If the selected PFML tax type does not have a rate maintained in V_T5UTX or the default rate value does not fit your business needs, a new entry for this tax type must be maintained in V_BTXRATE view. Do not perform any manual update in V_T5UTX.
  • A rate for PFML - Employer Pick-Up (139) tax type must be maintained in V_BTXRATE. Otherwise, the system will not calculate the employer pick-up amount.
  • If the "Experience Rate Zero" option is selected in V_BTXRATE for PFML - Employer Pick-Up (139) tax type, then the system does not adjust any PFML tax types, and the ER Pick-Up calculation will be 0.
  • If the "Experience Rate Zero" option is selected in V_BTXRATE for one of the selected employee's PFML tax type, the system does not adjust the experience rate and ignores this tax type in the PFML - Employer Pick-Up (139) rate calculation.

Self-adjust considerations

  • The Self-Adjust Tax (LUD SA:) parameter for PFML - Employer Pick-Up (139) tax type by default is Self-adjust tax using year-to-date wages and year-to-date taxes (0) value. If you want to use a different value, maintain Tax Authority Self-Adjust Parameter Selector (10TSA) feature. For more information, see SAP Note 3677145 - BSI: Enablement of 10TSA feature for self-adjustment parameter configuration.
  • Do not use PFML - Employer Pick-Up (139) tax type in Customize Tax Types to be self-adjusted for payroll processing (10STT) feature. If tax type 139 is entered in 10STT with either "X" or "S" value, it will be ignored. Use the Self-Adjust parameter through 10TSA feature if you want to perform any self-adjustment calculation.

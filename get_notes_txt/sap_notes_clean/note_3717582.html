Symptom

When attempting to change an S/4HANA Service Opportunity via the OData PATCH method of the API_OPPORTUNITY_SRV service /sap/opu/odata/sap/API_OPPORTUNITY_SRV/A_Opportunity, the service returns HTTP 204 (No Content) without an error message even when the Opportunity is locked by another user. The expected behavior is HTTP 423 (Locked).

The same behavior is observed for the following objects and services:

  • Business Activities — API_ACTIVITY_APPOINTMENT_SRV / A_AppointmentActivity
  • Tasks — API_TASK_SRV / A_CustMgmtTask
  • Leads — API_LEAD_SRV / A_Lead

Other Terms

Opportunity OData API, API_OPPORTUNITY_SRV, A_Opportunity, API_ACTIVITY_APPOINTMENT_SRV, A_AppointmentActivity, API_LEAD_SRV, A_Lead,  API_TASK_SRV, A_CustMgmtTask, Lock handling, HTTP status code, 204, 423.

Reason and Prerequisites

Program error.

Solution

Implement the attached correction instruction.

After applying the correction, the services will return the appropriate lock status response for PATCH requests against locked objects.

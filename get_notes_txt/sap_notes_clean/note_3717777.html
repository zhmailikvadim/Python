Symptom

You use Payroll Switzerland (program RPCALCC0 for SE "Single Employment" or RPCALCC0_CE for CE "Concurrent Employment") and use the HRFORMS remuneration statement SAP_PAYSLIP_CH03.

You create a remuneration statement for an employee with an hourly wage. If the compensation wage types are not calculated using the standard function of the module for the indirect valuation "CHSTD" (table T511, field MODNA), the number field in the remuneration statement is recalculated based on the amount and approach/basis. As a result, there are differences to the original number field of the wage types from the payroll result (cluster table RT).

Other Terms

CE Payroll Switzerland, program RPFOSLC0_03 (transaction PC00_M02_HRF_03), HRFORMS form SAP_PAYSLIP_CH03, class CL_HRPAYCH_HRF0_PAYSLIP_03, method PROCESS_JOINED_STAR

Reason and Prerequisites

Program adjustment

Solution

Import the HR Support Package relevant for your release. If you want to implement the solution in advance, use the Note Assistant to implement the correction instructions relevant for your release. You can find more information about the Note Assistant on SAP Support Portal at https://support.sap.com/notes.

After you implement this SAP Note, the system does not recalculate the number field even if the following conditions are met:

  • According to processing class '72', the wage type has the value '2' ("Evaluate net hourly wage (holiday/public holiday portions) with /207").
  • In Customizing ('V_T5C91', field 'RECOMPUTE_FROM_BETRAG') for the line formatting of the remuneration statement, the option "Do Not Recalculate Approach/Basis and Number" is activated for the relevant wage type.

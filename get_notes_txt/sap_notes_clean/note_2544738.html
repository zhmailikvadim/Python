Symptom

You are following the Post-manual steps described at the document 'Header texts for MX CFDI.pdf', adding the mentioned entries in the specificed Tables, but you are not able to save the entries.

Error example: '“Enter a valid value” Message No. 0002'

Environment

Mexico CFDI version 3.3

Reproducing the Issue

1. Goto transaction SM30;
2. Enter the view name 'IDMX_DI_VTXTID' or 'IDMX_DI_VTXTIDFI' or 'IDMX_DI_TXTOB';
3. Maintain one of the below values:


CNF Confirmation Number VBBK YCNF
UUD UUID(Unique Identifier) VBBK YUUD
LGD Legend VBBK YLGD
PAC RFC of PAC VBBK YPAC

4. Save the entries.

Resolution

1. Go to Transaction 'SE54' and Input the View name;

1.1. Select the option 'Generated Objects' and press 'Create/Change';

1.2. Select at the Header 'Generated Objects' -> 'Change';

1.3. Click on 'Expert Mode';

1.4. Select all checkboxes and regenerate the dialog.

2. Follow the first step and regenerate the table maintenance for the Views 'IDMX_DI_VTXTID', 'IDMX_DI_VTXTIDFI' and 'IDMX_DI_TXTOB';

2.1. If necessary, use the attached screenshot as reference for the authorization group, function or package information from 'IDMX_DI_VTXTID' and 'IDMX_DI_VTXTIDFI'. As for 'IDMX_DI_TXTOB' there is no need for this information, leave it blank.

3. Delete and reapply, from the start, the entries from the manual steps from the Document 'Header texts for MX CFDI.pdf'.

If this dump occurs in your system, then you should perform the following:

Change the version of program 'SAPLIDMX_DI_DET' to the previous version.

The error occurs because in some scenarios after regenerate the views a new version of SAPLIDMX_DI_DET will be generated.

Using transaction SE38 go to the program 'SAPLIDMX_DI_DET', then select utilities > Versions > version managment and activate the previous version of the program 'SAPLIDMX_DI_DET'.

You may check that the previous version contains include 'LIDMX_DI_DETF02' (and others) that are necessary to avoid this abap dump.

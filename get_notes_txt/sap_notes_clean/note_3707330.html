Symptom

An error occurs in My Timesheet (S/4HANA/Version 4) when submitting time. 
There is an ST22 dump in the backend.

Category               ABAP programming error                                                       
Runtime Errors         RAISE_SHORTDUMP                                                              
Except.                CX_SADL_DUMP_APPL_MODEL_ERROR                                                
Development Object     CLAS CL_HRFIO_MY_TIMESHEET                                                   
Application Component  PA-FIO-TS  
..
..
The exception is related to the previous exception                                            
"CX_HRPA_INVALID_INFOTYPE_DB" that occurred in                                               
program "CL_HRPA_INFOTYPE_CONTAINER====CP", in line 140 of include                            
"CL_HRPA_INFOTYPE_CONTAINER====CM005". The reason for this exception                         
was:
Inconsistent data exists in the database for infotype 0001 A for xxxxxxxx

Environment

SAP Fiori for SAP S/4HANA

Reproducing the Issue

1. Login into SAP Fiori.
2. Open "My Timesheet" Fiori App.
3. Enter the time against WBS element, Activity Type, and Attendance/Activity Time, etc. 
4. Click on Submit.

Resolution

The problem is independent from the application. It can be a problem by other report, app also. 

Check the data and secondary infotypes of the referred PERNR. 

By any inconsistency or need a help by this, please create a case in component PA-PA-XX or in the country specific PA-PA-*.

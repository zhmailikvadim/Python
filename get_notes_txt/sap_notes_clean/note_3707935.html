Symptom

  • In the transaction XXX (e.g. ME0M), the output is not displayed in the ALV Grid format as it was in the previous version (S/4HANA 1709).
  • The output appears in a different layout/format compared to the previous version.
  • The expected behavior is for the ME0M output to display in ALV Grid format for all users by default.

Environment

SAP List Viewer

Reproducing the Issue

  1. Access transaction ME0M in the upgraded S/4HANA system.
  2. Execute the transaction to view the output.
  3. Observe that the output is not displayed in the ALV Grid format, but in an alternative layout.

Cause

The user parameter ME_USE_GRID is not set to 'X' in the user profiles. This parameter controls the display format for purchasing reports, including source lists in ME0M, forcing the ALV Grid view instead of the alternative layout.

Resolution

To restore the default ALV Grid display for the ME0M transaction output across all users after the S/4HANA upgrade, set the user parameter ME_USE_GRID to 'X' in each user's profile:

    1. Go to transaction SU3 (Maintain Users Own Data).
    2. Navigate to the Parameters tab.
    3. Add a new entry:
      1. Parameter ID: ME_USE_GRID
      2. Parameter Value: X
    4. Save the changes. The change takes effect immediately for subsequent executions of ME0M. 

To apply this parameter globally to multiple users:

  1. Use transaction SU10 (User Mass Maintenance).
  2. Enter the list of user IDs (or use selection criteria to include all relevant users).
  3. Select Parameters in the maintenance options.
  4. Add ME_USE_GRID with value X.
  5. Execute and save the mass change.

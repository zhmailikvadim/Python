Symptom

Emails sent using the "Manage Purchase Order" app are delivered without attachments. When automatic message scheduling is enabled, the attachments cannot be added to messages in "To Be Output" status.

Environment

SAP S/4HANA Public Cloud

Reproducing the Issue

  1. Create a purchase order using the "Manage Purchase Order" app and attempt to add attachments on the header level
  2. Trigger the output and observe that the attachments are missing
  3. Disable automatic message scheduling and observe that messages are not generated until the purchase order is approved, preventing the addition of attachments before the email is sent.

Cause

Attachments can only be added or modified when the output status is "In Preparation". Once the purchase order is approved, the output status changes to "To Be Output", and the system no longer allows editing or adding attachments to the purchase order. This behavior is in line with the standard system design.

Resolution

To ensure that attachments are included in the email output, please perform the following steps:

    1. Add all required attachments at the item level before submitting the purchase order for approval.

    2. Confirm that the document output status is “In Preparation” prior to adding any attachments.

    3. Disable attachment merging in configuration, if you want the attachment to be included separately:

      • Go to Implementation Activities

      • Search for Define Output Settings for Document Types

      • Deactivate the option to merge attachments with the output document

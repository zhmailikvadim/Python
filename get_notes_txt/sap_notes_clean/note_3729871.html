Symptom

When executing report /IWBEP/R_FIX_IWSV_USOBHASH via transaction /nSE38 to create SUSH entries for IWSV objects, the following error message is displayed:

“Error in hash input”

Environment

SAP Gateway

SAP S/4HANA

Reproducing the Issue

  1. In transaction /nSE38 enter report /IWBEP/R_FIX_IWSV_USOBHASH and choose execute
  2. In the selection screen enter Workbench request
  3. Deselect the Test Mode checkbox
  4. Execute the report

The system raises the following error message: “Error in hash input”

Cause

The issue occurs because the OData service has been transported incorrectly, therefore it is not consistently available in the system.

Although an entry exists for this service in table USOBHASH, this table only stores authorization related data and does not guarantee that the corresponding OData technical objects are present in TADIR table.

An OData service is only considered available in the system if all related technical objects are correctly assigned to a transport request and successfully imported into the target system.

Resolution

Ensure that the transport requests are imported into the target system with all OData technical objects. 

For detailed instructions refer to KBA 3323348 - OData V2 Service Transport & Troubleshooting Guide

In this KBA you can find the collection of R3TR objects that have to be assigned to the package and to the transport request KBA 2424613 - How to assign an OData service from Local Object ($TMP) to a Transport Request (TR)? - SAP Gateway

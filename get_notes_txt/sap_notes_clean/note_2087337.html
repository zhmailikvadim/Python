Symptom

  • The Archive & Print option allows users to create, download and manage archives.
  • How to create Document Archives from Performance Forms or Live Profile and Scorecard Archives from People Profile.

Environment

  • SAP SuccessFactors 360 Reviews
  • SAP SuccessFactors People Analytics
  • SAP SuccessFactors People Profile
  • SAP SuccessFactors Performance Management
  • SAP SuccessFactors Succession Planning

Resolution

Archive & Print

  • To use this feature, it must firstly be activated in Analytics Tab>Classic Reporting in company setting by you integration partner. You may also reach out to Product Support to enable the feature.
  • It also requires you have been granted Reports Permission>Classic Reports permission in RBP by your administrator.

You have the option to archive items for individuals. To do so:

  1. Go to Reports > Classic Reporting > Archive and Print.
  2. Click Create New and select the available archive option.
  3. Continue the process as usual.

Creating a new batch archive or Accessing and printing an existing archive

  1. Go to Reports > Analytics > Classic Reporting
  2. Click Archive & Print:
    1. You will now see a list of any existing archives that you can access to review or print
    2. To view or print a document, click the link for the archive. Download the archive to your desktop
    3. Extract the PDFs you require for printing or viewing.
  3. To create a new archive, click Create New button. You may create archives of Document Archive

When creating an archive, you have various options to define what will be included. Some examples are:

  • For Document Archive:
    1. First choose an archive name
    2. Select the document you want to archive
    3. Pick a date range
    4. Choose which employees to include
    5. Select archive format

    • There is also an option to exclude certain sections from the printed pdfs.
    • For example, if you do not wish to print a Matrix Grid (9 Box) that was otherwise hidden to users click the "Customize Display Options" to open a popup that will provide a list of checkboxes
    • Default display options will be used to generate the archive
      • Optional: Customize Display Options
        1. Click "Create Archive" when you are done
        2. Deselect the checkboxes for the sections you do not wish to print
    • HINT: Try some test examples with just a few forms to see how the printout will be affected by deselecting sections before archiving large batches.
  • You can use a Dynamic Group to create the batch in Archive and Print if multiple users cannot be selected using the filters
  • When using the filters to create the list for the batch, you cannot edit the list or add/remove anyone

Additional Information

  • Archive & Print enables users to archive or print large quantities of documents.
  • This function is useful for anyone who wants to locally store copies of review forms for auditing records or who wants to print a large number of forms quickly.
  1. The system creates PDF copies of forms
  2. The original forms are not removed
  3. Once you submit request it will be scheduled for archive:
    1. Return to Classic Reporting > Archive & Print once notified job is completed
    2. Download and save to a local network to store indefinitely as the system automatically removes completed archives after 7 days
  4. Single PDF file or individual PDF files?
    1. Individual PDFs will provide the most flexibility if you need to email/print specific forms.
    2. It also makes it easier to find copies for specific individuals
    3. A large individual PDF may be problematic if you attempt to email it, as your email system (or the recipients) may have file size limitations
    4. Single PDF may be easier to manager if you only need to print all forms
  5. If you are still not able to find the form you are looking for, it may be in a "Deleted" status
  6. Change the status flag to Deleted and run the search again

Limitations

  • Single PDFs must not exceed 20MB in size
  • You may need to limit search parameters and run multiple batches
  • No batch archive may exceed 500 documents
  • When you exceed the number of allowable documents to archive in a single batch you will receive the following notification alert:

The number of forms matching your search criteria exceeds the current allowed limit

  • To work around this limitation, simply decrease the size of the current archive using the filter options
  • We suggest limiting the archive to a specific division, department or location

Form Content Permission

By default, both the Detailed Document Search and Archive and Print are designed so individuals granted with reporting rights will be able to see form content normally restricted for certain roles within an actual form. This would enable senior managers or administrators to be able to report on content that is otherwise not visible. Some companies may want to disable this default behavior and also enforce the permissions at the template level also in detailed reports.

NOTE: This is a dynamic option and will apply to live forms in real-time.

To do this per template:

  1. Go to Admin Center > Performance Management > Manage Templates > [Select your template] 
  2. From here, General Settings > Advanced Settings > Check the Enforce permissioning for Detailed Document Search & Archive & Print Reporting checkbox.
  3. Click Update Form Template to save the changes.

NOTE: The option can also be enabled via Admin Center > Form Template Settings

This feature will impact PM review forms and 360 forms on the following pages:

  • Classic Reporting > Detailed Document Search > Click form name in result list to display a print preview of a PM/360 form.
  • Classic Reporting > Archive & Print > Create New Document Archive > Click List Documents button > Click form name in result list to display a print preview of a PM/360 form.
  • Classic Reporting > Archive & Print > Create New Document Archive > After report is generated down and open the PDF file which will display PM/360 form.

If this option is enabled and the permission for the Other's Rating Tab is disabled, tab-related information such as comments and ratings will be hidden as well when viewing and printing the PDF output. This will occur on all steps for all roles of the template settings.

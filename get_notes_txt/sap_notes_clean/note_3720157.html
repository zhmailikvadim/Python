Symptom

  • During the ase patch upgrade, it fails with the below message
    Error: Command execution failed. : SAPSR3 credentials are required to recreate CDS objects on ASE 16.0 SP3 and higher. Please follow SAP note 1797040. (fault code: 127)
    Operation ID: *******

        ----- Log messages ----
        Info: saphostcontrol: Executing LiveDatabaseUpdate
        Error: sapdbctrl: SAPSR3 credentials are required to recreate CDS Objects on ASE 16.0 SP3 and higher. Please follow SAP note 1797040.
        Error: saphostcontrol: LiveDatabaseUpdate failed (sapdbctrl exit code = 1)

  •  Upon checking validate secure store command:
    ./saphostctrl -function LiveDatabaseUpdate -dbname SID -dbtype syb -updateoption TASK=VALIDATE_SEC_STORES
      Passwords are not found in global (or) local (or) ase user store

Environment

 
  • SAP Adaptive Server Enterprise (ASE) for Business Suite
  • SAP NetWeaver (NW) - All versions

Cause

 Passwords are not updated in global (or) local (or) ase user store, so upgrade is failing.

Resolution

  • Update the ase user store with the below command:

         /usr/sap/hostctrl/exe/saphostctrl -user sapadm <password> -function SetDatabaseProperty -dbname SID -dbtype syb -dbuser SAPSR3 -dbpass <password> USER=SAPSR3 -updateoption                                    WITH_ASEUSERSTORE=1 

  • And re-run the upgrade command

See Also

2800483 - SYB: Upgrade / Update SAP ASE with SAP Host Agent 7.21 PL43 and later
1797040 - SYB: SAP Host Agent - Using global or local secure storage

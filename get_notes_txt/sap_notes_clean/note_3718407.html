Symptom

During the execution of a DTP or a process chain, the following termination may occur:

The database reports to unknown database object. The error message from the database was: "invalid table name:  Could not find table/view ...

/1BCAMDP/0BW:DAP:TR_... in schema SAPABAPVL: line 1 col 1767 (at pos 1766)".

In particular, the termination occurs if this is a stacked scenario (for example, ADSO -> TRCS -> ADSO) and the first transformation is defined with an SAP HANA runtime and the second transformation is defined with an ABAP runtime.

In addition, the HAAP of the first transformation (ADSO -> TRCS (SAP HANA runtime)) does not exist in the database.

During DTP execution, the select statement is created in such a way that it reads the data using the CalcScenario of the first transformation. The system determines that the same statement no longer exists in the database and a regeneration is triggered. The system does not take into account that one of the two transformations is defined with an ABAP runtime and a new calculation scenario is created and generated for the DTP. However, this has a different name than the name that is used in the select-statement and the above occurs as a result. Error.

After you implement the corrections, the DTP CalcScenario is created only if necessary (ErrorDTP, at least two transformations with SAP HANA runtime, and so on). As a result, the calculation scenario that is also used in the select statement is then regenerated in this case.

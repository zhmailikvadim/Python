Symptom

After upgrading SAP HANA Cloud, Smart Data Access (SDA) remote queries may fail with out‑of‑memory (OOM) errors.

Typical error messages include:

[SAP AG][LIBODBCHDB SO][HDBODBC] General error;403 internal error: Error opening the cursor for the remote database <REMOTE_SOURCE> [SAP AG][LIBODBCHDB SO][HDBODBC] General error;4 cannot allocate enough memory:  [9] Memory allocation failed;exception 1000002: Allocation failed ; $failure_type$=STATEMENT_MEMORY_LIMIT_FROM_GLOBAL_CONFIG;

Other Terms

Federation; SDA; Optimizer; Semi Join; OOM; HANA Cloud; cannot allocate enough memory;

Reason and Prerequisites

With SAP HANA Cloud 2025 QRC4, a new remote SEMI‑JOIN pushdown capability was introduced.

In certain scenarios, this capability may push down an excessive number of join operations to the remote system. This can lead to a suboptimal join order, resulting in significantly increased memory consumption during query execution and ultimately causing OOM errors.

Affected Releases:

  • SAP HANA Cloud:
    • Revisions >= 2025.40.10 (2025 QRC4)

Prerequisite:

  • A Smart Data Access (SDA) setup where the target system is HANA Cloud and the remote source is either another HANA Cloud system or a HANA 2 system.

Solution

The issue is caused by plan volatility in the SQL optimizer. No permanent fix is planned.

To avoid suboptimal execution plans, apply one of the following workarounds:

  • Statement Level

    Add the SQL query level hint 

    • SELECT * FROM "SCHEMA"."TABLE" WITH HINT (NO_REMOTE_SEMI_JOIN);

    Alternatively add the statement to the corresponding SQL plan and clear the plan cache for this plan.

    1. ALTER SYSTEM ADD STATEMENT HINT (NO_REMOTE_SEMI_JOIN) COMMENT 'SNOW Case XXXXXXX' FOR STATEMENT HASH '<STATEMENT_HASH>';

    2. ALTER SYSTEM REMOVE SQL PLAN CACHE WHERE STATEMENT_STRING LIKE '%<STATEMENT_STRING>%';

    3. Or ALTER SYSTEM REMOVE SQL PLAN CACHE ENTRY '<plan_id>';
  • Remote Source Level

    ALTER REMOTE SOURCE <Remote_Source_Name> SET PROPERTY 'CAP_SEMI_JOIN' = 'FALSE';

Remark:

  • Disabling SEMI‑JOIN pushdown at the remote source level affects all remote queries that access this remote source.

Symptom

Once authenticated session expires in 6 hours, how to extend it till 30 days?

Environment

  • SAP ERP
  • SAP S/4HANA
  • SAP S/4HANA Cloud, Private Edition
  • SAP Digital Compliance Service for India

Reproducing the Issue

  • Login to DCS
  • Open "Manage Authentication"
  • Authenticate the GSTIN
  • After 6 hours the status shows "Expired"

Cause

Configuration is not set for Refresh Token

Resolution

Once Authenticated, Session will be active only for 6hours. To continue session up to 30 days, follow below steps.

  1. Refresh token should be on in the Manage Authentication.
  2. You have to enable session expire up to 30 days in the GSTN portal for each GSTIN Number.

Check the same and deploy the latest GSTN public key certificate in keystore. Refer Managing Keystore Entries | SAP Help Portal. Download the latest GSTN Public key certificate from GST Developer Portal | Returns 

If the issue still persists, then raise the same with GSP team to further check with GSTN on this behavior.

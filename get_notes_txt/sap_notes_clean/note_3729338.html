Symptom

During BP Employee-to-BP Employment synchronization in SAP S/4HANA, the system ignores the customized Employment Number Range setting maintained for switch HRALX ENUMB, resulting in incorrect employment number assignment and related errors in the synchronization process. The expected value maintained in system configuration is not applied.

Environment

  • SAP S/4HANA On-Premise
  • SAP S/4HANA Private Cloud Edition

Reproducing the Issue

  1. Perform BP Employee-to-BP Employment synchronization.
  2. Error occurs due to the employment number range not matching the configuration maintained, and the actual number range value is not applied.

Cause

The constructor of class /SHCM/CL_EE_SYNC_CONSTANTS does not read the current setting for switch HRALX ENUMB, causing the system to ignore the customized employment number range configuration.

Resolution

Apply SAP Note 3690420 - "Fixed ignored customizing for Employment number range."
After installing the note, retest the BP Employee-to-BP Employment synchronization process. The system will correctly respect the employment number range setting as maintained in customizing (T77S0).

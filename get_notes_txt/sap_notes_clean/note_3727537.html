Symptom

  • Error when opening custom OWL with a business user.
  • Error message displayed: "UI application /Y1234567Y_MAIN/SRC/Test/Test_OWL.OWL.uicomponent is not in scope".

Environment

  • Business Config. Tools (SAP Business ByDesign).

Reproducing the Issue

  1. Log into the relevant tenant.
  2. Use the provided business user credentials.
  3. Navigate to Work Centre (WC) KSEF -> Monitor.
  4. Attempt to display the list in the Monitor view.
  5. Observe the error message: "UI application /Y1234567Y_MAIN/SRC/Test/Test_OWL.OWL.uicomponent is not in scope".

Cause

  • The OWL is not added to the Assigned Objects collection in the Work Centre View

Resolution

  1. Open the UI Designer or SDK.
  2. Navigate to the component Y1234567Y_TEST_WCVIEW.
  3. Access the properties of the component and locate the Assigned Objects collection.
  4. Add the OWLs to the Assigned Objects collection:
  5. Save the changes.
  6. Activate the updated component.
  7. Verify that the issue is resolved by testing the access for a business user.

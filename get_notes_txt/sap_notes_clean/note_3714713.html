Symptom

In scenarios where program LIPS_FUELLEN_WERK (include SAPFV50P) is executed, errors raised inside the routine are not propagated back to the caller.
The system continues further processing even after a critical error occurs. This leads to:

  • Short dumps
  • Missing application log entries
  • No meaningful error information for the user

This makes it difficult for users to understand the root cause of the issue.

Example error message not returned to the caller:
VL027 – Shipping from shipping point & and plant & is not possible.

Other Terms

GN_DELIVERY_CREATE, OUTBOUND_DELIVERY, NJIT, LE-SHP-GF

Reason and Prerequisites

A program error caused the SY‑SUBRC result of LIPS_FUELLEN_WERK to not be returned to the caller.
Because the caller did not receive the error status, it continued processing instead of stopping, leading to runtime short dumps and missing or incomplete application logs.

Solution

Implement the correction instructions attached to this SAP Note or Upgrade to the latest available release, where the corrected coding is already included.

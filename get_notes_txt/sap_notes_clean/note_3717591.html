Symptom

  • Behavior of "lastModifiedBy" field in the Manage Workflow Request when the workflow is approved  directly by the approver accessing the system
  • Behavior of "lastModifiedBy" field in the Manage Workflow Request when the workflow is approved via  proxy  on behalf of the approver
  • The lastModifiedBy field in the Manage Workflow Request does not change after approvals made directly by the approver
 

Environment

  • SAP SuccessFactors HCM Suite
  • SAP SuccessFactors Metadata framework (MDF)

Reproducing the Issue

Scenario 1: Workflow Approved Directly by the Assigned Approver

1.Login to instance -> Navigate to Manage Workflow Requests.

2. Create or identify a transaction that triggers a workflow (for example, an employee data change requiring approval).

3. Log in as the assigned workflow approver.

4. Access the pending approval request from To-Do or Manage Workflow Requests -> Approve the workflow request.

5.Search for the approved workflow request. Observe the value of the lastModifiedBy field is not updated 

Result:
The lastModifiedBy field does not update to reflect the approver who approved the workflow.

Scenario 2: Workflow Approved via Proxy

1.Login to instance -> Navigate to Manage Workflow Requests.

2. Create or identify a transaction that triggers a workflow (for example, an employee data change requiring approval).

3. Log in as the assigned workflow approver.

4. Access the pending approval request from To-Do or Manage Workflow Requests -> Approve the workflow request.

5.Search for the approved workflow request. Observe the value of the lastModifiedBy field is updated with proper approver details

Result:
The lastModifiedBy field reflects a different behavior compared to direct approval (i.e.,the value of the lastModifiedBy field is updated with proper approver details)

Cause

There is a workflow inconsistency observed when the approver directly approves the workflow. In such cases, the "lastModifiedBy" field does not reflect the approver’s name. This behavior has been identified as a code-related issue. 

Resolution

  • The Engineering team is aware of this issue and is currently investigating it. At this time, we do not have a confirmed timeline for the fix.
  • Please refer to the KBA for further updates. The KBA will be updated once the fix is planned and the timeline is available on the roadmap.

Symptom

The current SAP Note delivers the new SMART PAYE: New RPN Request report (HIECNRR0) that allows you to generate a New RPN Request file in JSON format to be manually uploaded into Revenue Online Service (ROS) when you need to request RPNs for new employees.

Other Terms

HIECNRR0, PC00_M11_NRR

Solution

The new report (HIECNRR0, transaction PC00_M11_NRR) is designed to be run for any group of employees. The other selection fields (i.e. Personnel Number, Company Code, Organisational Unit etc) are available and can be used, in any way you wish, to choose a group of the employees to process.

The processing of the report will already bypass anyone that has had an RPN uploaded into SAP this tax year or are not in the ERN selected (if one has been selected).

 

Processing and Output Options

Tax year: is a mandatory field

Employer's Registered Number (ERN): is an optional field. If the ERN is entered employees who have a different ERN will be by-passed.

Create File: The checkbox is used to create the JSON file.

New RPN Request: ALV layout selection for the ALV created by the report.

 

Report Output

Once the program has been executed there are several selection options available on screen for the user to display. The icon selections are as follows:

New RPN Requests

It lists information about the records included in the New RPN Request file with the following information:

Request Identifier (generated by the report)
Employer's Registered Number
Tax Year
Personnel Number
PPSN
Employment ID
First Name
Last Name
Hire Date

Statistics

Select the tab to see the following statistics:

Number of Employees selected
Number of Employees rejected
Number of Employees bypassed
Number of Employees processed

Person-Specific Messages

Select this tab to see warning and error messages relating to the employee list.


If the Create File option was selected the New RPN Request file will be created as well. If the report is run in background, the folder where the file is saved needs to be specified in transaction FILE. The relevant Logical File Name Definition is HR_IE_DIR_HIECNRR0.

 

The report's documentation will only be delivered in the corresponding SP/CLC. It is attached to this SAP Note in a PDF format (HIECNRR0.pdf) as a reference.

 

 

EITHER

Apply the relevant HRSP

OR

1) All SAP_HR releases: Apply the Correction Instructions of SAP Note 2732768.

2) SAP_HR 608 only: Apply the Correction Instructions of SAP Note 2732932 and run the UDO program NOTE_2732932 in update mode - if you are not on a sufficiently high SAP_BASIS release the aCI-DDIC correction instructions are not applied automatically by SNOTE.

3) All SAP_HR releases: Apply the Correction Instructions of SAP Note 2732933 and run the UDO program NOTE_2732933 in update and in generate mode. In release SAP_HR 608, step 2) will have created all the necessary DDIC changes, but in SAP_HR release 604 and 600 this step will include the creation of all the necessary DDIC changes (i.e. In SAP_HR 604 and SAP_HR 600 the UDO program NOTE_2732933 does more than it does in SAP_HR 608).

4) Apply the correction instructions attached to this SAP Note, via SNOTE and carry out the manual activities described in the Post-Implementation steps.

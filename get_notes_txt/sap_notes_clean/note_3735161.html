Symptom

  • An error occurs when attempting to export content using Content Storage Network: "Export failed due to tenant export capacity exceeded."
  • Storage for exported content shows high usage such as 99.57% usage (2039.27 MB / 2048 MB).

Environment

  • SAP Analytics Cloud (Enterprise)

Reproducing the Issue

  1. Attempt to create an export in SAC using Content Storage Network
  2. Observe the error message: "Export failed due to tenant export capacity exceeded."
  3. Check Content Network Storage usage; it is near or at capacity (for example, 99.57%).

Cause

The allocated storage space for exported content in the Content Network Storage is full.

Resolution

  1. In SAC, navigate to Transport.
  2. Select Export, then open Content Network Storage to review current usage.
  3. Open the My Content folder to view exported content items.
  4. Select items that are no longer needed.
  5. Delete the selected items using the trash icon to free up space.
  6. Confirm that storage usage is reduced below the maximum limit.
  7. Retry the export operation.
  8. For detailed guidance, refer to: KBA 3121178.

Alternatively, you can create a ticket with SAP Support to increase the Content Storage capacity temporarily in order to move large amounts of objects for scenarios where you may be moving replicating content from one tenant to another (example. Migrating from Test to Prod tenant)

Symptom

Calling the app "Git-Enabled CTS" when logged on in languages other than English fails and returns an error message.

Other Terms

BSP Application BC_CTS_GIT, F4158

Reason and Prerequisites

The app cannot be called in some languages because the translation in the translation object with the text key COnrPLATCkThwCbsDjIZxQ== contains erroneous unescaped characters \u 

Solution

The correction for all languages will be shipped with the following support package:

Software Component Release Support Package Support Package Name
SAP_BASIS 816 02 SAPK-81602INSAPBASIS - SAP_BASIS 816: SP 0002 

Please refer to the SAP Support Package Stacks information in the Support Portal for information on the availability of these support packages: https://support.sap.com/en/release-upgrade-maintenance/maintenance-information/schedules-for-maintenance-deliveries.html.

Alternatively, if you are unable to wait for the fix in the above support package or if you are not able to implement the support package, then this can also be fixed manually as follows:

Object type (shortcut): FIOL

Object name: BC_CTS_GIT                    44A551A5C40EDD10D2DE878AB42750AE

Text key: COnrPLATCkThwCbsDjIZxQ== 

Please use transaction SE63 to fix the translated text(s) as follows:

  1. Enter /oSE63 in the command field (transaction field) and press Enter
  2. Enter the object type (see above) in the command field and press Enter
  3. Enter the object name (see above) in the field Object Name
  4. If necessary, change the languages in the Source Language and Target Language fields
  5. Press Edit
  6. Use the key combination Ctrl+F3 to scroll to the object with the corresponding text key (COnrPLATCkThwCbsDjIZxQ== - see above)
  7. Update the translation in the lower text box accordingly, making sure there are no more occurrences of the unescaped character \u in the translation
  8. Choose Object -> Save Active (Shift+F4) to save your change(s)
  9. Choose Back (F3) to leave the translation editor

Note:

Translations performed in a customer system may be overwritten in the following instances:

  • If the object happens to be part of a future support package
  • Upgrade to a new release
  • Import of new language CDs from Support Releases

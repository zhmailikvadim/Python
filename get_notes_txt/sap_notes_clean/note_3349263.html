Symptom

You are getting the The list object is too big to be exported error message when trying to export table data from MARA or any other table from SE16N in format XXL.

Environment

Office Integration
SAP GUI for Windows

Reproducing the Issue


1. Start transaction SE16N
2. Enter MARA
3. Execute
4. In next screen, change view from ALV Grid to Excel in-place

Cause

The table has more than 255 columns.

Resolution

As documented in note 1429136, the limit number of columns is 256.

Important Note:


1) Even though Excel 2007 or higher version supports 16,384 columns but XXL_FULL_API supports only 256 columns.


As alternative you can create XLSX file from internal table.
That means, download the data to Excel via Excel (In Office 2007 XLSX Format).

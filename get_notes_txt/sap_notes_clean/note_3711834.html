Symptom

  • Synchronization issue in SAP SuccessFactors Employee Central where the manager update is not reflected correctly in the hierarchical level of the Org Chart.
  • The manager update in Job Info is correct, but the hierarchical level still shows the previous manager.

Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.

Environment

  • SAP Successfactors HXM Suite 
  • SAP Successfactors Employee Central

Resolution

In some cases, the Org Chart Hierarchy may not update correctly. This article outlines a workaround that forces a full rebuild of the Org Chart Hierarchy by making a temporary change to an admin user using the Basic User Import.

This process is safe when followed as documented and can help resolve issues related to missing or incorrect reporting relationships.

 

Important Notes

    • By default, Super Admin and System Admin users do not have a manager assigned.
    • Assigning (and later removing) a manager for one of these users triggers a full Org Chart Hierarchy rebuild.
    • The rebuild process can take up to 2 hours to complete.

 

Prerequisites

    • Admin access to Employee Export and Import Employee Data
    • At least one user with System Admin or Super Admin permissions

 

Step 1: Export Employees and Identify an Admin User

    1. Navigate to Admin Tools > Employee Export.
    2. Click Export Users.
    3. Open the exported CSV file.
    4. Remove all rows except:
      • The two header rows (rows 1 and 2)
      • One System Admin or Super Admin user with no manager assigned
        (Example: an “Administrator” user with full System Admin permissions)
    5. Confirm the CSV file now contains three rows total.

 

Step 2: Assign a Manager Using Basic Import

    1. Save the updated CSV file to a known location.
    2. Navigate to Admin Tools > Import Employee Data.
    3. Select Basic Import.
    4. Scroll to the bottom of the page and upload the CSV file.
    5. In the CSV file, assign a manager to the selected Super Admin/System Admin user.
      • Example: Assign PSADMIN as the manager for SFADMIN.
    6. Click Import to submit the file.
    7. Allow up to 2 hours for the Org Chart Hierarchy to rebuild.

Note: 2574481 - Basic Import Options

 

Step 3: Revert the Admin User Back to No Manager

Once the Org Chart rebuild has completed:

    1. Reopen the same CSV file used in Step 2.
    2. Update both users so that the Manager field is set to NO Manager.
    3. Import the file again using Basic Import.

 

Step 4: Run a Full HRIS Sync

    1. Run a Full HRIS process to finalize and synchronize all changes.

Note:

  •  2512570 - Creating an HRIS Sync Job in SAP SuccessFactors Via Scheduled Job Manager or Provisioning 
  • Since 2023, all SAP SuccessFactors data centers have transitioned to the UTC time zone, as per release note PLA-8264

Time-Zone:

  • 3140223 – Integrations FAQ: SuccessFactors DataCenter Time Zone as UTC

  • 2089497 – TIMEZONE Column and Supported International Time Zone Codes

Expected Result

    • The Org Chart Hierarchy is fully rebuilt.
    • Reporting relationships display correctly.
    • The Super Admin/System Admin user is returned to its original state with no manager assigned.

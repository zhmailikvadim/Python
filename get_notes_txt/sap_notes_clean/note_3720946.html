Symptom

This SAP Note provides you with updates to the Colombia DIAN 1001 Report Format (CO_DIAN_1001) report in the Run Statutory Reports app for SAP Document and Reporting Compliance. It addresses the consumption of third‑party information when the Offsetting Account Type (GKOAR) and Offsetting Account Number (GKONT) fields are missing in the financial accounting documents table Universal Journal Entry Line Items (ACDOCA).

When you run the CO_DIAN_1001 report, the system incorrectly aggregates the results when the third‑party offsetting information is not maintained in ACDOCA, as the business partner is not considered in the final output.

Other Terms

1001 Report Format, ACDOCA, BSEG, Offsetting, Third-party, GKOAR, GKONT, DRC, Colombia, DIAN, Magnetic Media

Reason and Prerequisites

To implement the Magnetic Media compliance reports solution, you need a separate license for Document and Reporting Compliance (DRC). For more information, contact your SAP key account manager.

You have installed the Magnetic Media compliance reports for Colombia. For more information, check SAP Note 2717885.

This SAP Note is relevant only for Colombia.

You have implemented SAP Note 3704612.

Solution

After implementing this SAP Note, the system reads the offsetting information primarily from the ACDOCA table. If the corresponding field is not maintained in ACDOCA, the system retrieves the offsetting information from the financial accounting documents table Accounting Document Segment (BSEG) instead. This ensures that the report correctly aggregates the data and accurately reflects the appropriate business partner information.

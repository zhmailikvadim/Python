Symptom

A Scope on a composite index (for example, an index on "field1; field2") will not return any result or an unexpected result set using Advantage Data Architect.

Environment

  • SAP Advantage Database Server (All versions)
  • Sybase Advantage Database Server (All versions)
  • Advantage Data Architect

Cause

  • ARC doesn't support this as it would need a different interface for entering the scope values
  • A simple edit box like it uses now would not be sufficient
  • The intention of the scope in ARC is for use/testing on a "simple" index

Resolution

A Scope can still be set outside of ARC on a composite index by using the AdsInitRawKey/AdsBuildRawKey ACE API or client-specific seek functionality (such as Delphi's FindKey or Locate methods).

Symptom

When an external system attempts to call a Fieldglass web service or connect via SFTP, the connection is rejected with an HTTP 403 Forbidden error. The integration logs may specifically state:

Error: access denied due to non-approved IP address.

Cause

This error occurs when the IP Filter feature is enabled for your company in SAP Fieldglass. The system is designed to reject any incoming traffic that does not originate from an IP address explicitly listed in your "Allowed IP Addresses" configuration.

Resolution

You must add the source IP addresses of your integration servers to the allowed list in the Configuration Manager.

  1. Log in with your Configuration Manager account.
  2. Navigate to the Allowed IP Addresses tile.
  3. Click Edit.
  4. Enter the required IP information for both IPV4 and IPV6 as needed:
    • Static: Enter a specific, complete IP address (e.g., 1.2.3.4).
    • Range: Enter a start and end IP separated by a dash (e.g., 1.2.3.4 - 1.2.3.10).
    • Wildcard: Use an asterisk to allow a partial subnet (e.g., 1.2.3.*).
  5. Click Update to save the changes.

Note: If your integration requires SAP Fieldglass to send data to your servers, ensure you have also allow-listed the SAP Fieldglass outbound IP addresses on your internal firewall.

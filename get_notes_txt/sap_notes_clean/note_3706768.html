Symptom

SAML authentication fails when processing encrypted SAML Assertions / NameID / Attributes with one of the following error messages:

  • Index: 0
  • Cannot invoke "org.opensaml.saml.saml2.core.NameID.getValue()" because the return value of "org.opensaml.saml.saml2.core.Subject.getNameID()" is null

Other Terms

SAML, NameID, Attributes, Authentication, Encryption, Encrypted Assertion, XSUAA for XSA, XSA

Reason and Prerequisites

The local Service Provider fails to correctly process encrypted SAML Assertions / NameID / Attributes. This issue is applicable for XSA versions 1.4.0 - 1.4.5.

Solution

Update the XSA System to version 1.4.6.

If updating XSUAA is not possible, a non-recommended workaround is to configure the trusted Identity Provider to send unencrypted SAML Assertions / NameID / Attributes.

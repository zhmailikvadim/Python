Symptom

  • Email addresses displayed in objects such as Accounts or Individual Customers appear as clickable hyperlinks (blue text)
  • When users click the email address:
    • The system triggers an external email client (e.g., Microsoft Outlook)
  • Requirement:
    • Email addresses should remain non-interactive or not trigger any external application

Environment

  • SAP Service Cloud Version 2 1.0
  • SAP Sales Cloud Version 2 1.0 

Reproducing the Issue

  1. Open an individual account or account record that contains an email address.
  2. Observe that the email address appears as a clickable hyperlink.
  3. Click the email address and note that the default email client opens in a separate window.

Cause

  • Email fields are defined with an email semantic type
  • The UI automatically renders these fields as hyperlinks using the mailto: URI scheme
  • When clicked:
    • The browser delegates the action to the default email client configured on the user’s system
  • This is standard browser and UI framework behavior

Resolution

  • This is expected system behavior.
    • There is currently:
      • No configuration to disable the mailto: hyperlink
      • No standard option to render email fields as plain text
      • No way to redirect the click action to a custom mashup or internal email composer
  • Additional Information
    • The mailto: behavior is based on a URI scheme (RFC 6068) handled by the browser and operating system
    • The system does not control which email client is launched
  • Next Steps/Recommendations:
    • If you require the ability to:
      • Disable email hyperlink behavior
      • Redirect clicks to a custom mashup or
      • Define the hyperlink to open the corresponding email feeder related to the entity where the click is executed from
    • Then, please submit a request via the Customer Influence Portal: https://influence.sap.com/go/sales_service_v2_esm

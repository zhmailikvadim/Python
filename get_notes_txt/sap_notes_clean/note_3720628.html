Symptom

  • Any change made in BOM item in source system generates new BOM line items in target system BOM.
  • Idoc message type BOMMAT.

Environment

  • SAP ERP Logistics (LO)
  • SAP R/3
  • SAP R/3 Enterprise
  • SAP ERP Central Component
  • SAP ERP
  • SAP Enhancement package for SAP ERP
  • SAP Enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA Cloud Private Edition

Reproducing the Issue

  1. Change quantity in source system for BOM via t-code CS02.
  2. Execute BD21 using BOMMAT.
  3. Check outbound Idoc in target system using WE02.
  4. Display BOM in target system using CS03.
  5. A new additional item is created rather than the already existed item being changed. 

Cause

  • GUID identification is not activated for ALE/IDOCs, this can be confirmed via checking the STPO-GUIDX value in both source and target system.
  • The Idoc BOM structure differs between systems, causing the target system to interpret changes as new creations rather than updates.

Resolution

  1. Remove duplicated BOM items in the target system via t-code CS02.
  2. Activate GUID identification functionality as described in SAP Note 567351, point 10 and 11:
    1. Generate definite implementations of methods ALE_IDENTIFY_BY_GUID_INBOUND or ALE_IDENTIFY_BY_GUID_OUTBOUND of BAdI BOM_EXIT
    2. Add necessary fields in the Segment E1STPOM for GUID identification as instructed in SAP Note 567351 under "How do I activate the GUID-identification?".
  3. Ensure STPO-GUID values are identical in both source and target systems for all BOM items.

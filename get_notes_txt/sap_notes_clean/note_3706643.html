Symptom

On the selection screen of a report it is possible to create a background job via the command "Program -> Execute in background".

There may be come confusion about the used variant for the job as the user may previously have loaded or saved a selection variant and may expect that the variant is also used in the newly created job.

Reason and Prerequisites

Additional information

Solution

When creating a background job from the selection screen via "Program -> Execute in background" the newly created job will always contain a new temporary job variant. This variant has the following properties:

  • The variant name starts with "&" followed by a sequential number (e.g. &0000000001234)
  • It doesn't matter if the user has previously loaded or saved a normal selection variant, there will always be a new temporary variant created and used in the job
  • All contained dynamical parts, e.g. a variant variable or date function assigned to a parameter, or a default value computed by the program, are NOT transferred to the temporary variant. Instead the current values on the screen are transferred to the new variant.

If it's required or intended to use a specific variant in a job, one of the following tools / flows have to be used:

  • Via transactions SA38 and SE38 (Menu: Program -> Background)
  • Via transaction SM36/SM37 (Following the standard flow to create or change a job)

Symptom

We will do the categorization for a source customizing transport with some objects in.

Ideally, the status goes from "Waiting for Categorization" to "Open".

But the status is changed to "Retrofitted" without any real object change in the target transport in retrofit target system.

Other Terms

Retrofit;  categorization;

Reason and Prerequisites

There is specail customizing transport without object keys. ST-PI cannot fill the object list by keys into the Toc (transport of copy).

CALM backend will set the empty object list Transport request status directly as "Retrofitted".

ST-PI should fix the problem.

Solution

Apply the attached correction instructions or install the corresponding support package.

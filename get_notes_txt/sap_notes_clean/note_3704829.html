Symptom

  • After repointing a report from UNV to UNX, a mismatch in the number of merged dimensions is observed. The count of merged dimensions before repointing is less than after repointing the Web Intelligence report to the .UNX universe.
  • The report does not display data due to missing merged dimensions after repointing to UNX.
  • The universes used in this case are Linked Universes.

Environment

  • SAP BusinessObjects Business Intelligence Platform 4.3 Support Pack 03 Patch 3
  • Information Design Tool (IDT)
  • Web Intelligence (Webi)

Reproducing the Issue

In Information Design Tool (IDT):

  1. Log in and create a new session in Information Design Tool.
  2. Convert the linked .unv universe to .unx universe.

In BI Launchpad:

  1. Log in to BI Launchpad.
  2. Open a Web Intelligence report using the previously used .unv universe as data source.
  3. Check the count of merged dimensions in the Web Intelligence Report.
  4. Repoint the report from UNV to UNX universe using 'Change Source'.
  5. Check the count of merged dimensions again.
    >> The count of merged dimensions before repointing is less than after repointing the Web Intelligence report to the. UNX universe.

Resolution

  1. Perform a Change Source from .UNV to .UNX universe.
  2. Refresh the Web Intelligence Report.
  3. Verify that all merged dimensions are now correctly displayed.
  4. Save the Web Intelligence Report.

Symptom

  • When converting the first ticket layout in the SAP event ticketing system to a Windows Boca layout 3879 (Boca Racecourse Party Race Day VIP without price 2026 Windows), the layout prints completely overlaid in the standard printer setting (Concert Ticket).
  • In the printer setting/paper size "Receipt", the layout is no longer overlaid but is rotated 180 degrees.
  • The font sizes do not match the original ticket layout.

Environment

SAP Event Ticketing

Reproducing the Issue

  1. Navigate to the SAP event ticketing system.
  2. Access the "Master Data" section.
  3. Select "Layouts".
  4. Search for layout ID.

Cause

  • Changing the name from special printer to Windows printer does not yield the expected effect.
  • Additional changes are required to ensure the ticket prints correctly.

Resolution

Several adjustments are necessary for the new Windows layout.

- Rotate the layout to a horizontal position in the layout editor.

- Additionally, in the Boca/Windows settings, under Advanced -> Print Defaults -> Layout -> Orientation, switch to landscape orientation. This setup is required by Boca.

- Design the layout with the DPI that corresponds to the Boca standard. For example, 200 or 300 DPI.

- Adjust the font size to the font sizes provided by Windows.

- The print format "Receipt" is correct if the ticket size is 82.5 mm x 203.2 mm.

Boca does not support rotating SAP Event Ticketing layouts when printing in Windows.

The print direction always remains the same when printing Boca in Windows. The orientation only controls the rotation on the ticket.

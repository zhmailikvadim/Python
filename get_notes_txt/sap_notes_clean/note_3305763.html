Symptom

In a sales order, you have configured an item with a planned/production order profile such that a selection condition creates another sales-relevant position in the bill of material. The sales order has been saved. When you now change this sales order by removing the subitem, the sales order item list responds correctly and removes this item. However, when you re-enter the configuration and the same subitem is created again, an error occurs with the message: "Internal error in communication between configuration and sales doc. CONFIGURATION_PROCESSING.”

Other Terms

AVC, advanced variant configuration, PPO, planned/production order

Reason and Prerequisites

When reactivating the instance, an internal identifier is set incorrectly, preventing the sales order from accessing the correct instance.

Solution

Implement this note or any of the Support Packages containing the note.

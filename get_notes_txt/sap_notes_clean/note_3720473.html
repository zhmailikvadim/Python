Symptom

Inconsistency error 5111 is reported : 

 [ERROR] Finished SQL: CALL CHECK_TABLE_CONSISTENCY('CHECK', NULL, NULL), with Error: "<schema_name>","<table_name>","$uc
_<index_name>$",0,5111,"Inconsistency found: Consistency violated, rowcount = ###, distinct row count = ### (### duplicates)"

Environment

SAP HANA, platform edition 2.0

Reproducing the Issue

CALL CHECK_TABLE_CONSISTENCY('CHECK', NULL, NULL);

Cause

 Duplicate keys in terms of primary key or unique index columns are existing in the table.

Resolution

You can find "<schema_name>","<table_name>" and "<index_name>" in the error messages. 

 [ERROR] Finished SQL: CALL CHECK_TABLE_CONSISTENCY('CHECK', NULL, NULL), with Error: "<schema_name>","<table_name>","$uc
_<index_name>$"

For example:  

 [ERROR] Finished SQL: CALL CHECK_TABLE_CONSISTENCY('CHECK', NULL, NULL), with Error: "SAPHANADB","COKEY","$uc
_COKEY~1$",0,5111,"Inconsistency found: Consistency violated, rowcount = 37641, distinct row count = 37240 (401 duplicates)"

In this example, we can see the <schema_name>="SAPHANADB", <table_name>="COKEY" , <index_name>="COKEY~1". 

Please execute the SQL commands below in SQL editor of Tcode DB02 one by one:

  1. SELECT INDEX_NAME, COLUMN_NAME FROM INDEX_COLUMNS WHERE TABLE_NAME = '<table_name>' and index_name = '<index_name>';
    (You can get column_name list belonging to the unique index in the output of this step)
  2. CREATE TABLE "<table_name>_BACK" LIKE "<table_name>" WITH DATA;
    (Note : This step is used to back up the original table with duplicate data rows. Make sure this step get executed successfully and check the data row count in the new table "<table_name>_BACK" ,
    if the row count in the new table is the same as original table "<table_name>", continue with the next step. ) 

  3. Delete from "<table_name>" WHERE "$rowid$" IN (SELECT ROW_ID FROM (SELECT "$rowid$" ROW_ID, ROW_NUMBER() OVER (PARTITION BY <column_name list belonging to the unique index>) DUP_NO FROM "<table_name>" ) WHERE DUP_NO>1 ) WITH HINT (NO_AGGR_SIMPLIFICATION, NO_GROUPING_SIMPLIFICATION);

  4. CALL CHECK_TABLE_CONSISTENCY('CHECK','<schema_name>','<table_name>');
    (This step is used to check the consistency again after deleting duplicate data rows)

For Example: 

  1. SELECT INDEX_NAME, COLUMN_NAME FROM INDEX_COLUMNS WHERE TABLE_NAME = 'COKEY' and index_name = 'COKEY~1';
  2. CREATE TABLE "COKEY_BACK" LIKE "COKEY" WITH DATA;
  3. Delete from "COKEY" WHERE "$rowid$" IN (SELECT ROW_ID FROM (SELECT "$rowid$" ROW_ID, ROW_NUMBER() OVER (PARTITION BY "MANDT", "HKGRP", "MATNR", "WERKS", "CBRKI", "CBRKE", "CBART", "DRVTP", "RECIND", "BEMOT") DUP_NO FROM "COKEY" ) WHERE DUP_NO>1 ) WITH HINT (NO_AGGR_SIMPLIFICATION, NO_GROUPING_SIMPLIFICATION);
  4. CALL CHECK_TABLE_CONSISTENCY('CHECK','SAPHANADB','COKEY');

Please Note: Following the steps above is the easy way to solve this issue.
However, if the keys are identical, but there are differences in the remaining table column values, it needs to be checked together with application support team by business to identify and confirm the good data rows  and bad data rows.

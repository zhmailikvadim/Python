Symptom

Region field is being cleared from Person record after being set from ERP side.

Environment

SAP Field Service Management

Reproducing the Issue

  1. Update the Employee on ERP side by assigning them to a Region and trigger the update transaction
  2. Go to FSM and look for the corresponding User

Expected behavior:

The Region set should be visible under the Person Record

Actual behavior:

The regions field is empty after the update. There are Audit logs showcasing the regions property being set and immediately after, emptied

Cause

  1. ERPUSER record has priority over EMPLOYEE record.
  2. If the corresponding ERPUSER doesn't have a Region assigned, it will overwrite changes to the EMPLOYEE record

Resolution

  1. Assign the region to the ERPUSER record on ERP.
  2. Resend the user data to FSM.
  3. Verify that the changes propagate correctly to the other Person Records under the same user and maintain the region.

Symptom

  • A dump occurs when attempting to display or submit an XML document in the EDOC_COCKPIT transaction.
  • Error: "OBJECTS_NOT_COMPATIBLE Operand deepStruc2 has type EDO_MX_CFDSUBMIT_REQUEST_TYPE, but operand deepStruc1 has type EDO_MX_CFDSUBMIT_REQUEST_TYPE1" is displayed.

Environment

  • SAP ERP
  • SAP S/4HANA
  • SAP Document and Reporting Compliance service

Reproducing the Issue

  1. Enter transaction EDOC_COCKPIT.
  2. Submit or display the affected eDocument.
  3. Observe that the dump error appears immediately.

Cause

The issue is caused by the use of an obsolete data type, "EDO_MX_CFDSUBMIT_REQUEST_TYPE", which is incompatible with the required type "EDO_MX_CFDSUBMIT_REQUEST_TYPE1".

Resolution

To fix this issue:

  1. Read and install the last version of the following SAP Note 3167422 - eDocument Mexico - Annex 20 V4.0: Customizing for SAP Application Interface Framework (SAP ERP) or 3167423 - eDocument Mexico - Annex 20 V4.0: Customizing for SAP Application Interface Framework (SAP S/4HANA).
  2. Activate the BC sets attached to the SAP notes 3167422/3167423 in Expert Mode
  3. Confirm that you correctly executed the manual steps as stated in topics 1 and 2 ('Define Interface Version' and 'Assign AIF Interfaces and Versions') of PDF file attached in the SAP Note 3167391 - eDocument Mexico - Annex 20 V4.0: Full Solution.

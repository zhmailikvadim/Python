Symptom

  • A spool request is created in SP01, but no physical copy is printed.
  • The spool status in SP01 is -.
  • Warning displayed: "Spool request created without immediate output".
  • When reprinting the spool request from SP01, the document is printed physically.

Environment

  • SAP S/4HANA
  • SAP S/4HANA FOUNDATION
  • SAP ERP
  • ABAP Platform
  • SAP NetWeaver
  • SAP ERP Central Component
  • SAP Solution Manager

Reproducing the Issue

  1. Print a document from the application in use; preview may work and a spool is created.
  2. Observe the warning "Spool request created without immediate output" and that no physical printout occurs.
  3. Observe the spool status - in SP01 for the created spool request.
  4. Re-test printing the spool from SP01 and confirm that a physical copy is produced.

Cause

The status - means that a spool request is created by an application without the print immediately setting being enabled. This means a spool is created in transaction SP01, but there is no output request created to actually send the spool print data to the printer.

Resolution

If immediate output is required, activate the "Print Immediately" option in the print settings when sending the document to print as per the below steps:

  1. Select the transaction the issue occurs in and give print to the document
  2. Select the Output Device you wish to use in the field 'Output Device'
    1. Then select 'Properties' and check the field: Time of print
    2. If it is set to: Print Out Immediately then execute the request to print.
      or
    3. Notice the checkbox with name "Print Immediately' in the pop-up where the Output Device is specified and make sure it is checked.

If the output is not printed out at this point then proceed with the following steps:

  1. Follow the steps above, but do not execute
  2. Double-click on the line 'Time of Printing' (a new pop-up will appear and it is possible to change the options here, for example: Send to SAP Spooler only for now)
  3. Select the edit mode button (a pencil) in the field "field default value"
  4. Choose "print immediately"
  5. Select the continue button – keep selecting the continue buttons on the various windows until reaching the original print menu
  6. At the bottom of the screen a spool number should appear
  7. This Spool should print immediately and the status in SP01 will should be Completed

NOTE: Most applications have internal coding that can overrule the settings above, if it is still not possible to print immediately after changing these settings or if the options above are not applicable to the application used, review the application in question and open an incident under their component.

Symptom

When you run transactions /MOSB/PRICE or /MOSB/MASS_PRICE to adapt a meter operator contract because of a price change, the system determines the incorrect reason "Article ID Change" instead of "Price Change." This occurs when several positions with the same price classification but different article IDs exist in the meter operator contract.
This issue is relevant for the market role: MOS.

Other Terms

MOS Billing, Price Change, Article ID Change, /MOSB/PRICE, /MOSB/MASS_PRICE, /MOSB/RP_CHG_CONTRACT_PRICLS_G, meter operator contract, price classification

Reason and Prerequisites

Program error. The error occurred because the system identifies comparing positions in new time slices and existing time slices based on price classification. However, the same price classification can be determined on different positions.

Solution

Implement the corrections of this SAP Note or import the relevant support package.
After implementing this SAP Note, the system determines the correct reason "Price Change" when you adapt meter operator contracts with price changes.
You can verify the fix by running transactions /MOSB/PRICE or /MOSB/MASS_PRICE on contracts with several positions that have the same price classification but different article IDs, and checking that the system determines the correct reason "Price Change."

Symptom

  • When running Canadian statutory reports (GST/HST, PST, QST) in the "Run Statutory Reports" app, the report generation completes successfully.
  • All amount fields in the generated report display a value of 0.00, even though financial transactions with relevant tax codes exist for the reporting period.
  • Only the tax number is populated; all other values remain 0.00.

Environment

  • SAP S/4HANA
  • SAP S/4HANA Cloud Private Edition

Reproducing the Issue

  1. Go to the "Run Statutory Reports" Fiori app.
  2. Select the Canadian reporting entity and execute the report (e.g., GST/HST Return).
  3. Execute the report for a specific period where relevant transactions exist.
  4. Observe that the report generation is completed successfully, but the report output shows 0.00 in all boxes except the tax number.

Cause

The issue is caused by an incomplete mandatory configuration. The report only includes data from documents whose tax jurisdictions are explicitly selected in the customizing activity "Specify Federal Level Tax Jurisdiction Code". If no jurisdictions are selected in this setting, the report will not consider any documents, resulting in zero values.

Resolution

  1. In customizing, review “Specify Federal Level Tax Jurisdiction Code” for Canada and ensure the Tax Jurisdiction setting is active for federal-level reporting.
  2. Ensure tax jurisdiction determination is configured so that a tax jurisdiction code is populated on accounting documents at posting time.
  3. Verify that documents within the reporting period contain the correct federal-level tax jurisdiction code; post corrective entries if needed for test purposes.
  4. Regenerate the statutory report for the same period and reporting entity and confirm values are populated.

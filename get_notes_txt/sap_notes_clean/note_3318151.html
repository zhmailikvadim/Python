Symptom

When using Seat Reservation in a class with approval process, the user status gets set as Enrolled right after the registration, before the approval request gets approved

Environment

SAP SuccessFactors Learning

Reproducing the Issue

  1. Inside Learning Administration > at a class record > Seat Registrations tab >
  2. Add a Seat Registration option >
  3. At the user side, users entitled to Seat Reservation registers into the class >
  4. The approval process gets triggered >
  5. However, right after the registration the user already appears as Enrolled, before the approval request can be approved >
  6. The approvers will receive the approval request >
  7. The user's registration will be cancelled if the approval request gets denied

Cause

Confirmed by Engineering that this is expected behavior

Resolution

During the registration, the user entitled to use Seat Reservation will have displayed in the UI a communication mentioning their status will be set as Enrolled.

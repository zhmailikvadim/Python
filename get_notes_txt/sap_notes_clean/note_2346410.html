Symptom

When you run Cash Flow Reference Report, you find that Bank Statement transaction is displayed twice in the report.

Reproducing the issue

Prerequisites:

  1. Go to Administration → System Initialization → General Settings → Cash Flow tab and make sure that you have the following settings:
    • Assignment of Cash Flow Line item is set to Mandatory,
    • Default Primary Cash Flow Line Item for Incoming Payment and Outgoing Payment are defined.
    • Assignment in transaction with All Relevant Cash Flow is set to Warning Only.
  2. Go to Administration → System Initialization → Company Details → Basic Initialization tab and make sure that Install Bank Statement Processing option is selected.
  3. Go to Administration → Setup → Banking → Bank Statement Processing → Internal Bank Operation Codes and make sure that you have at least one code defined.

Scenario:

  1. Go to Sales - A/R → A/R Invoice and add a new A/R Invoice document.
  2. Go to Banking → Bank Statements and External Reconciliations → Bank Statement Processing select a bank account and then choose Create New button.
  3. Select Internal Operation Code, enter Incoming Amt - payment Currency and BP Code corresponding to the A/R Invoice and select the Primary Form Item.
  4. Double-click the row created and choose Add Open Documents button.
  5. Select A/R Invoices in the Document field and choose OK.
  6. Select the A/R Invoice created in step 1, choose Update and then the OK button.
  7. Adjust the Ending Balance on the Bank Statement so the Difference is 0.
  8. Mark the row with the A/R Invoice as Cleared/Selected and click the Finalize button.
  9. Close the Bank Statement Details form and go to Financials → Financial Reports → Financial → Cash Flow Reference Report.
  10. Select dates including the Bank Statement date, select All Transactions Relevant to Cash Flow option and choose OK.

Expected behavior:

The Bank Statement transaction is displayed once in the report.

Actual behavior:

The Bank Statement transaction is displayed twice in the report.

Solution

SAP intends to provide a patch or patches in order to solve the problem described. The section Reference to Related Notes below will list the specific patches once they become available. The corresponding Info file of the patches in SAP Service Marketplace will also show the SAP Note number. Be aware that these references can only be set at patch release date. SAP will deliver patches only for selected releases at its own discretion, based on the business impact and the complexity of the implementation.

Symptom

This SAP Note adds the following fields to the provider contract item:

  • Correspondence Recipient 
  • Address of the correspondence recipient.

Correspondence Creation:
During correspondence creation, the system checks whether a correspondence recipient is defined in the contract account, business partner, or provider contract items.
If the same correspondence recipient is defined in all provider contract items, the correspondence is sent to this recipient.
If there is no or different correspondence recipients defined in the provider contract items, the specifications from the contract account are used or, if no recipient is defined there, the correspondence is sent to the standard recipient defined in the business partner.

Example:
All correspondences concerning a provider contract are to be sent to the same business partner. To achieve this, enter the key of this business partner in all currently valid contract items of the provider contract.

Integration with Subscription Billing:
If you use the integration of the provider contract with Subscription Billing and want to transfer the bill recipient of the subscription to the Correspondence Recipient field of the provider contract item, maintain the corresponding setting in the following Customizing activity (view V_TFK_SB_SETTING, TA SM30):
IMG: Contract Accounts Receivable and Payable -> Integration -> SAP Subscription Billing -> Define Specifications for Data Transfer from SAP Subscription Billing

Reason and Prerequisites

Solution

We recommend that you implement the function only by importing the Support Package.

If this is not possible:

  • Implement this SAP Note and then follow the steps described in the manual correction instructions.
  • Integration with Subscription Billing:
    Since IMG nodes in the SAP Customizing Implementation Guide can only be delivered via the Support Package, the setting for transferring the bill recipient from the subscription must be maintained in the view V_TFK_SB_SETTING (transaction SM30).

If you use the provider contract as a second contract in one of the industry solutions of FI-CA, also implement SAP Note 3696568.

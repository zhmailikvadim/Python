Symptom

  • Connection attempt from SAP Data Provisioning Agent (DP Agent) to SAP HANA via JDBC fails with SSL handshake error.
  • Error messages in the framework trace
    • <Timestamp> [ERROR]  DPFramework | JDBCConnector.makeConnection [] - com.sap.db.jdbc.exceptions.JDBCDriverException: SAP DBTech JDBC: Cannot connect to <HANA_Host>:<HANA_Port> [Cannot connect to host <HANA_Host>:<HANA_Port>. Received fatal alert: handshake_failure -813: <Agent_Host>:<Agent_Port> -> <HANA_Host>:<HANA_Port> ConnectionID:0 SessionID:0].
    • <Timestamp> [ERROR]  DPFramework | JDBCConnector.makeConnection [] - com.sap.db.jdbc.exceptions.JDBCDriverException: SAP DBTech JDBC: Cannot connect to <HANA_Host>:<HANA_Port> [Cannot connect to host <HANA_Host>:<HANA_Port>. Received fatal alert: protocol_version  -813: <Agent_Host>:<Agent_Port> -> <HANA_Host>:<HANA_Port> ConnectionID:0 SessionID:0].
  • From the DP Agent host, telnet towards the target HANA host succeeds without error:
    • telnet <HANA_Host> <HANA_Port> 

Environment

SAP HANA smart data integration

Reproducing the Issue

  1. Open DPAgent Configuration Tool.
  2. Select SAP HANA Connection.
  3. Choose the option to connect to SAP HANA based on the current scenario.
  4. Observe the error message indicating handshake failure (-813).

Cause

TLS mismatch between the DP Agent's underlying JRE and the target HANA system.

Resolution

  1. Verify which TLS versions are accepted by the target HANA system
    • openssl s_client -connect <HANA_Host>:<HANA_Port> -servername <HANA_Host> -tls1_1 
    • openssl s_client -connect <HANA_Host>:<HANA_Port> -servername <HANA_Host> -tls1_2
    • openssl s_client -connect <HANA_Host>:<HANA_Port> -servername <HANA_Host> -tls1_3
  2. Append the following entry to <DPAgent_root>/dpagent.ini:
    • -Djdk.tls.client.protocols="<Accepted_TLS_versions>"
    • IMPORTANT: <Accepted_TLS_versions> has to be adjusted based on the outcome of the above openssl commands
    • Example:
      • openssl command with tls1_1 and tls1_2 succeeds while tls1_3 fails
      • Line to add: 
        • -Djdk.tls.client.protocols="TLSv1.1,TLSv1.2 "
  3. Restart the DPAgent and attempt the connection again.

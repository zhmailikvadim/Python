Symptom

It is not possible to create a subsequent correction of an export declaration as part of the message exchange with the German customs system ATLAS AES from the existing customs document. As a result, when you cancel the existing export declaration, the system incorrectly activates the processing proposal "Request Cancellation Subsequently".

Other Terms

ATLAS, AES, EX-nK, subsequent correction, N830, /SAPSLL/PROCA, /SAPSLL/SAPLCUHD_LOAD_IFACE, /SAPSLL/CUS_SGL02E

Reason and Prerequisites

This problem is caused by a program error.

Solution

Implement the attached correction instructions.

After you have implemented the correction instructions, you have the option of creating a subsequent correction during the processing of the customs document. From the menu bar, choose Edit -> Create Correction -> Subsequent Correction.
This function is active if the issue from the customs territory has been confirmed, no investigation request is open, and the customs document is not in the fallback procedure. The master reference number (MRN) of the original export declaration is automatically set in the first item of the subsequent correction as document N830. If you want to preassign the type of declaration for the customs document of the subsequent correction automatically, you can configure this using a corresponding default data rule. The default field H_RCCOR represents the type of correction declaration.

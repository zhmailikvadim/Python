Symptom

SAP Note 3701545 - DaBPV: Automatic activation of CI special rule 06 (exclusion of DaBPV) for each subapplication delivered the subapplication DBP2 (DaBPV: General simulation of CI special rule 06). If this subapplication is activated, the existence of the CI special rule Exclusion DaBPV (06) is simulated for all personnel cases when creating the DaBPV notifications using the report Create DaBPV Outbound Notifications (RP_PAYDE_DBP_CREATE_NOTIFS). However, after you activate the subapplication, two errors occur:

  • For personnel cases to be registered, the warning message Exclusion from DaBPV procedure is not permitted (HRPAYDE_DBP038) is issued.
  • Despite an existing tax ID in the infotype Tax D (0012), personnel cases may be excluded from the DaBPV procedure. Either no registration takes place or a termination is created if the registration has already taken place.

Other Terms

Social insurance, DaBPV, CI special rule 06

Reason and Prerequisites

Program error

Solution

Implement the attached correction instructions or import the specified support package. After you import the HR Support Package or implement the correction instructions, you do not have to perform any further follow-on activities.

The program errors are corrected:

  • The warning messages are no longer issued if subapplication DBP2 is active. Remaining warning messages in the Notification Tool must be set to Completed manually.
  • If a tax ID exists, the personnel cases are no longer excluded from the DaBPV procedure.

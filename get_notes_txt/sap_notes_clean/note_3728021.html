Symptom

When you attempt to take an instance backup or schema backup from the Remote Support Platform (RSP), the backup operation fails with an error.

Reproducing the issue

Scenario:

1. You have an existing SAP Business One 10.0 environment for SAP HANA with RSP configured.

2. Delete previously taken instance backup files directly from the Linux server using the `rm` command via PuTTY (bypassing RSP/SLD interface).

3. Attempt to take an Instance Backup or Schema Backup from RSP.

Expected behavior:

RSP should successfully perform Instance Backup and Schema Backup.

Actual behavior:

Backup fails with an error code 16000 indicating null schema name (while the backup completes successfully on the backend, it returns an error status to the RSP).

Solution

SAP intends to provide a feature package or patch to solve the symptom that is described in this SAP Note. The section References below lists the related feature package or patch once they become available. Be aware that References can only be confirmed at feature package or patch release date. SAP delivers feature packages or patches only for selected releases at its own discretion, based on the business impact and the complexity of implementation.

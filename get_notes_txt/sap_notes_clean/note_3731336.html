Symptom

In transactions IW21/IW22, the Forward popup (Choose Recipient) opens, but clicking the checkmark/Continue after single-selecting a line does not confirm the selection.

The system only accepts the selection when the user double-clicks the Object ID value. The behavior occurs across multiple variants of the popup.

Environment

  • Product: SAP S/4HANA
  • SAP List Viewer

Reproducing the Issue

  1. Go to transaction IW22.
  2. Open any maintenance notification.
  3. Navigate to the Responsibilities tab.
  4. Click the Forward button.
  5. In the Choose Recipient popup, single‑click an Object ID to select it.
  6. Click the checkmark/Continue to confirm the selection (no selection is applied). Double‑clicking the Object ID applies the selection.

Cause

An issue in the ALV (List Viewer) event handling for certain SAP BASIS 7.58 support package levels (SP00–SP01) causes the Continue event (&ONT) to not populate the selected row context, while the double‑click event (&IC1) does. This prevents the application from registering the selection when using the checkmark/Continue. The behavior is addressed by SAP Note 3689365 and delivered from SP02.

Resolution

  1. Verify your SAP BASIS release and support package level for the ALV component (BC‑SRV‑ALV). If you are on 7.58 SP00 or SP01, proceed to step 2.
  2. Update to at least SAP BASIS 7.58 SP02 (preferably the newest available SP level, for example SP04 or later) in line with the ALV patch strategy.
  3. After the update, implement SAP Note 3689365 (if not already included in your installed SP) to ensure the ALV event handling fix is present.
  4. Re-test IW21/IW22 Forward: single‑select a row in the Choose Recipient popup and click the checkmark/Continue. The selection should now be applied without requiring a double‑click.

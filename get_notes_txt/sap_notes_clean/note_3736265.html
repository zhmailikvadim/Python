Symptom

Integration Center job fail with the issue:

Error - We have temporarily blocked the snapshot pagination request from companyUser <companyID>|<user> for all entities until <date> because too many consecutive requests have been attempted without accessing subsequent data. Please retry after you're unblocked and use $skiptoken to fetch the data of subsequent pages, otherwise you will continue getting the first page and be blocked again.

Environment

SAP SuccessFactors HCM

  • Integration Center
  • Odata API

Cause

Multiple Integration Center jobs, started at nearly the same time and using the same API user, each requested snapshot pagination page 1 for large entities within a short window, reaching the company-user level threshold (default 10) for consecutive first-page requests before any $skiptoken requests could arrive to advance pagination and reset the counter.

Resolution

There are some options to avoid the behavior:

  • Avoid multiple Integration Center jobs starting on same time.
  • Stagger Integration Center job start times by 1–2 minutes instead of scheduling many jobs at exactly the same time.
  • Distribute workload across different API users (for example, assign different users to different job groups) to avoid hitting the company-user level threshold across entities.

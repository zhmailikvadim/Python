Symptom

Unexpected slow BW Compression.

Reading the fact table's partitioning information in USER_TAB_PARTITIONS occurs too often.

Other Terms

RSDRDREQDELTAB

Reason and Prerequisites

  • SAP BW on Oracle
  • Flat InfoCube according to SAP Note 2335159 "ORA: Usage of flat InfoCubes".
  • InfoCubes having many empty BW Requests which haven't been compressed yet,
    this may happened after "selective deletion" operations.
  • There are no corresponding DB partitions for many of these BW requests.

Solution

Implement pilot note with /nSNOTE.

To solve the issue please apply the Support Package as mentioned below. In urgent cases you can use the correction instructions.

Symptom

When executing the eSocial event S-1200 an specific wage type is not classified correctly as income or deduction .

"Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental."  

Environment

  • HR renewal 

  • SAP S/4HANA 

  • SAP ERP  

  • SAP SuccessFactors Employee Central Payroll 

Resolution

The customization of Evaluation Class 6 of the wage type is validated to assign the nature of the wage types.

To configure properly, in the view V_512W_O, set Evaluation Class 06 to 01 for Income (Provento) and 02 for Deduction (Desconto):

For more details it is possible to check the GET_WT_NATURE method of the class CL_HRPAYBR_EFDP_PROC_REMUN:

WHEN pbr99_income.
        IF lv_betrg < 0.
          rv_wt_nature = if_hrpaybr_efd_py_constants=>gc_wt_evt_discount.
        ELSE.
          rv_wt_nature = if_hrpaybr_efd_py_constants=>gc_wt_evt_gain.
        ENDIF.

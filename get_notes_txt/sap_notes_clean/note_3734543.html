Symptom

For a custom app created using key user functionality, the option to activate/deactivate the 'IAM App' is not available in the Maintain Business Role app, even when the standard business catalog is assigned via a custom catalog extension.

Environment

SAP S/4HANA Cloud Public Edition

Cause

The 'Activate/Deactivate' 'IAM App' functionality applies only to standard applications, not to custom applications.

Resolution

Custom applications created using key user functionality are not generated as IAM Apps; they are assigned directly to standard business catalogs via a custom catalog extension.

Since custom catalog extensions are not migrated, no IAM App ID is displayed under the IAM Apps tab in the business role. Even if the custom application appears in the 'Display IAM Apps' app, it will not show any connection to the business catalog, and the 'Activate/Deactivate' 'IAM App' functionality is not available for custom applications.

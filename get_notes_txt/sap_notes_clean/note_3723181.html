Symptom

  • SAF-T BG report failed with an empty error message regarding of the CDS View.
  • Runtime error ACM_UNEXPECTED_VALUE encountered in ST22.
  • Error message: "No runtime data for CDS entity C_BG_SAFT* [DDIC: @AccessControl.authorizationCheck: #MANDATORY, Access Control Existence FALSE]".
  • ABAP programming error occurred during execution of an Open SQL select.

Environment

•Financial Accounting (FI)

•SAP S/4HANA Finance

•SAP S/4HANA

•SAP S/4HANA Cloud Private Edition

•SAP Fiori for SAP S/4HANA
 

Reproducing the Issue

  1. Attempt to generate the SAF-T declaration for Bulgaria using the application F1515 Run Statutory Report BG SAFT.
  2. Observe the failure with an empty error message.
  3. Check ST22 for runtime errors and identify the error ACM_UNEXPECTED_VALUE related to the CDS entity C_BG_SAFT*.

Cause

  • The ACM runtime returned an unexpected value during the authorization check for the CDS entity C_BG_SAFT*.

Resolution

  1. Start transaction SACMDCLS.
  2. Enter the name of the CDS entity mentioned in the runtime error as "CDS Entity Name".
  3. Execute the transaction.
  4. If the result set contains at least one access control:

a. Select all found access controls and execute the function "Generate".  

b. If successful, transport the generated access controls (R3TR DCLS) to update the landscape.  

c. If the generation shows errors, review the reported error message to identify the cause.  

d. If the error indicates a syntax issue in the access control, contact the responsible component for further assistance.  

Refer to SAP Note 2567848 for additional guidance.

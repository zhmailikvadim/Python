Symptom

  • In Process Integration (PI) or Process Orchestration (PO), a previously working integration stops working suddenly with error "Peer certificate rejected by ChainVerifier"
  • In this case, the interface is used for obtaining OAuth session IDs from the Salesforce application with hostname login.salesforce.com.
  • The REST Receiver Adapter is involved. 
  • Messages triggered from the ECC fail in the middleware PO production system.
  • Error message: "SOAP: Error occurred: com.sap.engine.interfaces.messaging.api.exception.MessagingException: java.io.IOException: Failed to get the input stream from socket: iaik.security.ssl.SSLCertificateException: Peer certificate rejected by ChainVerifier".
  • No recent changes have been made on the PI/PO side. 

Environment

  • SAP NetWeaver 7.5

Reproducing the Issue

  1. Monitor the message log of the PO production server.
  2. Observe the failure of messages triggered from the ECC production system in the middleware PO production system.
  3. Review the error message: SOAP: Error occurred: com.sap.engine.interfaces.messaging.api.exception.MessagingException: java.io.IOException: Failed to get the input stream from socket: iaik.security.ssl.SSLCertificateException: Peer certificate rejected by ChainVerifier.

Cause

  • The certificate used by the external Salesforce application was either changed or updated recently. 
  • Or a new certificate requirement was added that previously did not exist. 

Resolution

  1. Contact the external application partner and inquire about any recent certificate changes on their side, for the involved hostname - login.salesforce.com in this case, which is a publicly accessible URL.
  2. If a new certificate has been issued, obtain the updated certificate from the external application.
  3. It may also be possible to download the certificate directly from the browser session (if unclear how to do this, use any search engine to check the exact steps for the browser you are using).
  4. Import the new certificate into the relevant Trust Store in the PI/PO system.
  5. Verify the certificate installation and test the interface to ensure proper functionality.

Responsibility: PI/PO Interface Developer team, BASIS team, Security team.

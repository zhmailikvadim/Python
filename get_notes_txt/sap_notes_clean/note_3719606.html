Symptom

Cannot add an A/R Invoice with a Sales BOM using Without Qty Posting feature when the In Stock quantity is 0 in the warehouse.

Reproducing the issue

Prerequisites:

Go to Administration → System Initialization → Document Settings → General tab under For a Sales BOM in Documents, Display: select Price and Total for Parent Item Only.

Scenario:

1. Create a Bill of Materials of BOM Type - Sales, named 'SalesBom' and a component item 'I1' and a warehouse 'Whs1' that is enabled for Bin Locations.

2. Add a Goods Receipt for Item No. = 'I1', Quantity = 1 and Whse = 'Whs1'.

3. Add an A/R Invoice for Item No. 'SalesBom', Quantity = 1 and Whse = 'Whs1'.

4. Create a new A/R Invoice for Item No. 'SalesBom' and select the checkbox Without Qty Posting, try to add the document.

Expected behavior:

Document is added successfully.

Actual behavior:

Error message appears - Remaining quantity for inventory UoM is not zero; item must be fully allocated to bin locations.


*Same issue occurs if trying to duplicate the first A/R Invoice and adding it.

Solution

SAP intends to provide a feature package or patch to solve the symptom that is described in this SAP Note. The section References below lists the related feature package or patch once they become available. Be aware that References can only be confirmed at feature package or patch release date.

Symptom

Customer would like to Send Application factsheet data from LeanIX to Apptio

Resolution

Here is a Step by Step guide to setup and test the integration to Send Application data from LeanIX to Apptio:

Step 1: Initial Setup in Apptio

This is the part where you need support from your Apptio Admin to perform the following:

  1. Create a service user in Apptio Frontdoor application with "Data Link Admin" role 
  2. Grant the service user access to your Apptio environment
  3. Capture the API keyAccess and keySecret during initial user profile creation
  4. Identify your Apptio environment details:
    • loginUrl: https://frontdoor.apptio.com/service/apikeylogin
    • domain name containing the tables to be used
    • environmentName
    • ulsUrl (Uploader Service URL) based on your region:
      • EU: https://datalink-eu.apptio.com/uls/api/v1 
      • US: https://datalink.apptio.com/uls/api/v1

Step 2: Configure Authentication in SAP LeanIX

  1. Navigate to the Administration area in SAP LeanIX 
  2. Go to the Integrations section
  3. Click "Configure" in the Apptio section
  4. Configure the apptioConnectionConfig section with:
    • loginUrl from Step 1 
    • keyAccess and keySecret from Step 1
    • domain and environmentName from Step 1
    • ulsUrl based on your region from Step 1 

Step 3: Data Mapping Configuration

  1. Configure apptioWriteConfigs for Applications table with the following parameters :
    • type: "detailed"
    • host: your Apptio host (e.g., "bizdev-r12.apptio.com")
    • project: "Cost Transparency"
    • tableName: "lxApps"
    • period: "current"
    • overwrite: true

If you would like to send multiple factsheet types in multiple tables in Apptio, you would need to create multiple apptioWriteConfigs with different table names i.e.: "lxApps". "lxBusinessCapabilities"

Step 4: Integration Activation

  1. Set the configuration as active: true in the JSON configuration 
  2. Save the configuration in SAP LeanIX

Step 5: Integration API Setup

The integration API Connector setup is the step where you define which factsheet data you would like to send to Apptio. This is like setting a template for your table to define the Columns that you need from LeanIX. 

  1. Navigate to the Administration area in SAP LeanIX 
  2. Go to the Integration API section
  3. Open the connector with id: lxExportToApptio, then adjust your processors as needed: please note that for each processor , the output content.type value should match the table name that you set in step 3.1
  4. Once your processor is adjusted, you can save the configuration and click run, once finished, you can scroll all the way down under the output log, you should see a result url that you can then copy

Step 6: Integration Test

  1. Go back to the Apptio integration Setup under Integration-> Apptio, and paste the URL from the integration API result into the Data Provider part. 
  2. Click on the Upload Button.
  3. Upon a Successful run, The LxApps table in this example would be created in your Apptio environment      

Once your tests are finalized, you can raise a ticket for the Support team to run the workflow to automatically schedule the integration run for you.

Symptom

When changing the delivery date using fast change in VA02 (Sales Order Change), the system may not determine the shipping point information (TVST). The item schedule line update proceeds, but the shipping point remains initial. This can lead to incomplete or incorrect rescheduling behavior. The issue appears only during the fast‑change process; manual changes typically work as expected.

Other Terms

SAPMV45A, TVST, VBAP, XVBEP, fast change, schedule line processing, delivery date change, shipping point determination, MVERF_AUFBAUEN_WMENG, FV45VFMV_MVERF_AUFBAUEN_WMENG

Reason and Prerequisites

During fast‑change processing, the system does not ensure that shipping point data (TVST) is available before rescheduling logic is executed. If TVST is initial at this moment, the subsequent quantity and date checks may behave incorrectly.

Solution

Install this SAP Note.

Symptom

When you navigate to a Project System hierarchy report or cost element report (report painter reports), for example, the Project System report S_ALR_87013542 “Project Costs - Actual/Commitment/Total/Planned in CO Area Currency” in the Fiori Launchpad, the system does not display the report output even if you passed all required input fields for the selection screen in the navigation. Instead, it displays the starting page, i.e. the selection screen of the report, but not all input fields are filled. You need to manually enter the missing selection parameters and press "Enter" to open the report output page.

Other Terms

S_ALR, report painter, GR55, financial reports, actual cost, plan cost, planned cost, commitment cost, project system, navigation, authorization, ps_rep_navigation, SAP S/4HANA 2025, SAP S/4HANA Cloud Private Edition 2025.

Reason and Prerequisites

This is due to a technical restriction of the classical cost element and hierarchy reports. As of SAP S/4HANA 2025 FPS01, a solution is available for cost element, i.e. report painter reports in Project System.

Hierarchy reports, i.e. reports that were generated based on CJE0 are not supported by this solution due to additional technical restrictions. For such reports, the data is to be entered manually on the selection screen of the transactions.

This solution applies to scenarios where users navigate to a report painter report listed in the table below from Fiori applications using a custom business catalog based on the technical catalog SAP_TC_PS_BE_APPS.

Solution

Following are the possible issues that might be encountered after the upgrade:

1 – Authorization for new transaction PS_REP_NAVIGATION

Problem

In earlier releases, Fiori applications for Project System could navigate to the S_ALR reports directly from the Fiori Launchpad. However, selected parameters (such as Project Definition, WBS Element, Network, Activity, or Material) from the Fiori applications were not automatically transferred to the corresponding S_ALR reports.
As a result, users had to manually re-enter the same parameters after the report opened in the SAP GUI.

Resolution

To overcome this, a new transaction PS_REP_NAVIGATION has been introduced.
It serves as an intermediate navigation layer between Fiori applications and report painter reports.

All users who previously had access to report painter reports and now wish to navigate to these reports from Fiori applications must also be granted authorization for the PS_REP_NAVIGATION transaction.
Without this authorization, navigation from Fiori applications to the corresponding report painter reports is not possible.

The technical catalog SAP_TC_PS_BE_APPS has been updated.

2 – Navigation to customer defined report painter reports from Fiori applications

Problem

When you navigate from the Fiori applications to the custom report painter reports, it does not display the required page. Instead, it displays the starting page of the report, but not all input fields are filled. If you enter the missing parameters and press "Enter", the application calls the page that you expected in the previous step.

Resolution

In case a custom report is configured in the Fiori catalog to directly launch from the Fiori applications, and you want to auto-fill the parameters while navigating to the custom report painter reports for Project System objects (such as Project Definition, WBS Element, Network, Activity, and Material), a similar configuration approach can be applied as implemented in the SAP_TC_PS_BE_APPS technical catalog. Instead of calling the custom transaction directly from the catalog, configure the catalog entry to call PS_REP_NAVIGATION transaction.
Then, maintain the custom transaction as a parameter with the name ‘Transaction’ in the catalog and set the custom transaction code as the default value (see the example below), along with the relevant Project System object parameters.

Symptom

During an upgrade using Software Upgrade Manager (SUM), an error is observed in phase " PREP_EXTRACT/SUMASK".

Error in SUMASK.LOG:


ERROR: DDIC password does not work:
RFC login to system <SID> ashost <Hostname> nr <Instance No.> gwhost <Hostname> gwservice sapgw01 failed with code 1 key RFC_COMMUNICATION_FAILURE: 
LOCATION    CPIC (TCP/IP) on local host with Unicode
ERROR       partner 'Hostname:33xx' not reached
TIME        Tue Nov 23 23:50:52 2021
RELEASE     753
COMPONENT   NI (network interface)
VERSION     40
RC          -10
MODULE      /bas/753_REL/src/base/ni/nixxi.cpp
LINE        3454
DETAIL      NiPConnect2: 192.168.80.17:3301
SYSTEM CALL connect
ERRNO       111
ERRNO TEXT  Connection refused
COUNTER     15
.
Password for user "SYSTEM" is invalid:
The hdbsql connection is not possible due to the following error message: * 10: authentication failed SQLSTATE: 28000

Note: Verified that DDIC password is correct & Gateway is up and running.

Environment

  • Software Update Manager (SUM)
  • SAP Netweaver AS ABAP

Reproducing the Issue

Repeat phase PREP_EXTRACT/SUMASK in SUM.

Cause

SUM is trying to connect to incorrect instance.

Resolution

  • Verify the instance read by SUM in SAPupConsole.log

    Determining DB type from dbms_type='HDB'
    SAP instance number is 'XX'.
    Type of database system is 'HDB'.

  • Check in the directory of /usr/sap/<SID>, if there is unused SAP instance folder listed.
  • Perform the cleanup of the unused instance.
  • Reset current SUM and start SUM from the beginning.

Symptom

When you create a draft A/P credit memo with specific withholding tax and try to convert it to a document through Data Interface (DI) API, it fails with error "Unbalanced Transactions".

Reproducing the issue

Prerequisites:

You are using a company with the India (IN) localization.

Set rate for system currency: 88.

Scenario:

  1. Go to Administration → Setup → Financials → Tax → Withholding Tax, add an eTDS type withholding tax, for example T01.
  2. Create a Vendor type Business Partner (BP), go to Accounting tab → Tax tab → WTax Codes Allowed and enable T01.
  3. Create a GST Tax Invoice type A/P invoice, select the BP, quantity 1, unit price 300, Tax code: IGST18.
  4. Open the Withholding Tax Table, select WT Code: T01, set Taxable Amount to 300.
  5. Copy the A/P invoice to an A/P credit memo and save it as a draft.
  6. Try to convert the draft A/P credit memo to a document through DI API.

Sample code:

oDrafts = (SAPbobsCOM.Documents)oCompany.GetBusinessObject(SAPbobsCOM.BoObjectTypes.oDrafts);
oDrafts.GetByKey(1);
oDrafts.SaveDraftToDocument();

Expected behavior:

The draft A/P credit memo is successfully converted to a document.

Actual behavior:

The draft A/P credit memo conversion fails with error "Unbalanced Transactions".

Solution

SAP intends to provide a feature package or patch to solve the symptom that is described in this SAP Note. The section References below lists the related feature package or patch once they become available. Be aware that References can only be confirmed at feature package or patch release date. SAP delivers feature packages or patches only for selected releases at its own discretion, based on the business impact and the complexity of implementation.

Workaround: 

Convert the draft A/P credit memo to document through SAP Business One application.

Other terms

Unbalance Transaction, A/P invoice, draft A/P credit memo

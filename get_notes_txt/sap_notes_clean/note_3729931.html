Symptom

I upgraded a Contract Template and modified the Contract Workspace which removed the Task History from already closed document tasks. 

This happened when the parameter application.acm.templateupgradefordocumenttask is set to true. 

Reproducing the Issue

  1. Set the application.acm.templateupgradefordocumenttask parameter to Yes.
  2. Create a New Version of a Contract Template.
  3. Modify an existing Contract Workspace so it picks up the upgraded template.
  4. Open an already-closed document task and observe that the Task history section is cleared.

Cause

By design, when the parameter application.acm.templateupgradefordocumenttask is set to Yes, template upgrades update document tasks regardless of status and overwrite Task Fields, which wipes the Task History.

Resolution

  • Note that deleted Task History cannot be restored; it is permanently removed by current application design.
  • To prevent recurrence, set application.acm.templateupgradefordocumenttask to No before performing Template Upgrades so closed document tasks are not updated.
  • Be aware that when the parameter is set to Yes, the following Task Fields are updated from the Upgraded Template: Owner, Comments, Approvers, ApprovalRuleFlowType, Observers, Dependencies, StartWhenDependenciesComplete, ApprovalRequests, EnableNotificationOnComment, NotificationProfileId, DueDateOffset, DueDateFieldPath.

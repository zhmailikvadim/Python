Symptom

After your migration to S/4 HANA, you observe that for some materials the actual cost component split after the migration differs from the actual cost component split before the migration. For these materials, you find two "Migration" line items in transaction CKM3N in category "Receipts" for which no process category is set. The total price and/or exchange rate differences of these line items are zero, but there are non-zero values in individual cost components which add up to zero. The cost component split is the same in both "Migration" line items.

Other Terms

ACTCCS;
migration;
price limiter quantity, PLQTY, XPLREL

Reason and Prerequisites

Program error. The material has a non-zero price limiter quantity in the period but the total price limiter relevant price and/or exchange rate differences are zero. The cost component split of the price limiter relevant price and/or exchange rate differences is not trivial, though.

Solution

Please implement the attached program correction. After implementing the note and repeating the migration, the cost component splits in the two "Migration" line items in CKM3N category "Receipts" for which no process category is set should add up to zero and thus have no impact on the total cumulative actual cost component split of the material.

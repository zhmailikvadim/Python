Symptom

When using the app Create Golden Tax Documents to merge multiple documents under certain materials and exchange rates, the system automatically generates only one line item, but the unit price is split across the merged items. Users may prefer to retain the original unit prices of the line items without splitting them. 

Environment

SAP S/4HANA Cloud Public Edition.

Reproducing the Issue

  1. Open the app Create Golden Tax Documents.
  2. Select several documents.
  3. Click Merge Documents to perform the merge.
  4. The system generates one line item, but the unit price is found to be split evenly.

Cause

This is standard behavior.

Resolution

To retain the original unit prices of the line items without splitting them, the following methods can be considered:

Method 1: Adjust the Merge Group Field
1. Different codes can be assigned to the Merge Ref.Group field (e.g., 1, 2, 3) to create separate merge groups for the line items. 
2. Choose Merge Documents to process the merge. Multiple line items will be generated instead of a single one, and the unit prices will remain intact without being split.

Method 2: Use BAdI for Enhancement
The BAdI IDGTCN_OB_DATASET_PREP_AFT can be used to implement custom logic for document merging.

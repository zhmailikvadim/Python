Reproducing the Issue

  1. Update the Kernel to the latest patch levels
  2. Check that the folder <SUM path>\SUM\abap\shdviews has the appropriate permissions
  3. Rerun the phase

Cause

A Kernel package was used in N:\<SUM path>\SUM\abap\exe\ that is for a different Operating System or Database then your system is running on.

Check that the package you downloaded from Launchpad.

Example:
SAPEXEDB_300-70000602.SAR is for SQL database
SAPEXEDB_300-70000601.SAR is for Oracle database

Resolution

  1. Replace the kernel with the correct Kernel version and release for your Operating System and Database
  2. Repeat the phase in SUM

Symptom

You declare your import declaration in SAP GTS do customs system DMS.

You declare a Previous document (qaldoc SUMA) on item level. You enter unit of measurement for this document, but it does not create any unitCode for <WriteOff> in the <PreviousDocument> element. As a result, you receive error DK1349 saying, that measureUnitType should be provided in 'KGM'.

Other Terms

Denmark, DK, XML, import, /SAPSLL/LLEG_ECC_DATAF56, /SAPSLL/CL_CHECK_DK_IDMS_IE415, CHECK_ITEM_DOCUMENTS, DK1349, SUMA, PreviousDocument

Reason and Prerequisites

The problem is caused by a program error.

Links between <PreviousDocument> elements and SAP GTS fields on item level are listed below.

XML element SAP GTS field name
./PreviousDocument/TypeCode Type (PAPTY)
./PreviousDocument/ID Number (PAPNO)
./PreviousDocument/LineNumeric Item (PAPIT)
./PreviousDocument/WriteOff/QuantityQuantity Depreciation Quantity (PAPQU)
./PreviousDocument/WriteOff/unitCode UoM (LICUM)
./PreviousDocument/WriteOff/Packaging/QuantityQuantity

Quantity (PAPPC)
./PreviousDocument/WriteOff/Packaging/TypeCode Supplement (PAPAD)

Solution

Implement the corrections instructions.

Update your mapping according to point 23 in Note 2406946.

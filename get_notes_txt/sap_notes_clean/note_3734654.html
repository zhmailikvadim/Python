Symptom

  • After creating a numbering scheme and classifying a product using this scheme, the classification and numbering scheme is not displayed in the product data.
  • In the Fiori "Manage Products" app, the numbering scheme and product classification are not visible.
  • Backend tables /SAPSLL/PNTPR and /SAPSLL/PRCLS show that the classification exists.

Environment

  • SAP Global Trade Services, edition for SAP S/4HANA

Reproducing the Issue

  1. Classify a product for a numbering scheme in GTS.
  2. Open the Fiori "Manage Products" app and display the product.
  3. Observe that the numbering scheme and its classification are not displayed.
  4. Cross-check tables /SAPSLL/PNTPR and /SAPSLL/PRCLS (for example via SE16) to verify the classification exists in the backend.

Cause

  • The relevant legal regulation assigned to the numbering scheme is not determined when displaying the product because the determination procedure is set to "Determination using first successful strategy". Determination stops at the the first strategy (e.g. country group), while the legal regulation is configured at country level, so it is not determined for display.

Resolution

Review the determination procedure:

  • Open SM30 and view /SAPSLL/V_T606DR (or use SPRO: Global Trade Services -> General Settings -> Legal Regulations -> Determination Procedure for Active Legal Regulations).

Set the determination rule to "A Determination Using All Strategies" so that determination is performed using all strategies:

  • In /SAPSLL/V_T606DR, change field DLLRG for LECUS to "A".

Alternatively, if the rule must remain as "first successful strategy", activate the relevant legal regulation at the first strategy level so it can be determined:

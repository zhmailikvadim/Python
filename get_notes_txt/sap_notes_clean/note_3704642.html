Symptom

When trying to update SNC Names massively via transaction SNC1 / report RSUSR300, the only standard option for the variable part is either User Name or Alias.
There's a need to use some other attribute as variable part, such as the Email maintained in transaction SU01. 

"Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental."

Environment

  • ABAP platform
  • SAP NetWeaver

Cause

There's no standard feature for custom variable part such as Email currently.

Resolution

The SNC Name can be custom created via BAdI Implementation: BAdI USRACL_BADI_CREATE_PNAME.
The template class CL_USRACL_BADI is provided on SAP Note 1839538 - "RSUSR300: No change documents".

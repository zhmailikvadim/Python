Symptom

When generating the SPED EFD ICMS/IPI file in SAP, the field DT_FIN_OP (Operation End Date) in the K230 record appears blank. This occurs even for production orders that seem to be completed within the reporting period, leading to a file that is non-compliant with Brazilian tax regulations.

Environment

  • SAP ERP
  • SAP S/4HANA
  • SAP S/4HANA Cloud Private Edition
  • SAP Tax Declaration Framework for Brazil
  • SAP Document and Reporting Compliance service
  • SPED EFD ICMS/IPI

Cause

The SAP Tax Declaration Framework (TDF) and the DRC Brazil Statutory Reporting populates the DT_FIN_OP field in the K230 record using the data from the AUFK-IDAT2 (Actual Finish Date) field in the source ERP system.

Although a production order’s status was set to Technically Completed (TECO) within the reporting month (e.g., March 31st), the corresponding AUFK-IDAT2 field was populated with a date in the following month (e.g., April 1st).

Because TDF and DRC correctly reads this source data, it determines the order was not finalized within the reporting period. According to SPED rules, the DT_FIN_OP field is then intentionally left blank. The issue is not a bug in the solutions but rather an inconsistency in the source data from the ERP system.

Resolution

There are two approaches to resolving this issue:

  1. Short-Term Workaround:
    For urgent reporting needs, you can use shadow table /TMF/D_IT_PROD to overwrite the incorrect date. This allows you to manually provide the correct completion date for the specific production order, ensuring the SPED file is generated correctly for the current period.

  2. Long-Term Solution:
    To prevent this issue from recurring, the root cause in the ERP system must be addressed. The team responsible for the CO-OM-OPA (Controlling - Overhead Cost Orders) component should investigate why the AUFK-IDAT2 field is being populated with a date after the actual completion date. Correcting this data population logic in the source system is the permanent solution.

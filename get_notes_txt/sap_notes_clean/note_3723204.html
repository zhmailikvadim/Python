Symptom

  • EKPO table's 'changed on' date field does not reflect the timestamp when a PO line item is deleted
  • You are expecting that the 'changed on' date for deleted purchase order line items to be reflected in the BW extractor.
  • You are seeking alternative methods to capture the 'changed on' date for deleted purchase order line items other than using CDHDR and CDPOS.

Environment

  • Materials Management (MM)
  • SAP ERP Central Component
  • SAP ERP
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA

Reproducing the Issue

  1. Access the EKPO table in SAP ERP.
  2. Delete a purchase order line item.
  3. Verify the 'changed on' date field in the EKPO table.

Cause

Resolution

  1. The deletion of purchase order line items sets the deletion indicator (LOEKZ = 'L') but does not update the 'changed on' timestamp in the EKPO table, as this is standard SAP behavior.
  2. Note that there is no standard SAP field that stores the deletion date of a purchase order line item.
  3. Use change documents (CDHDR/CDPOS) as the reliable source for capturing the deletion date of purchase order line items.
  4. Custom Enhancement in BW Extractor is recommended as it provides the best balance of accuracy, integration with existing processes, and maintainability. This approach leverages SAP's standard enhancement framework while providing the precise deletion timestamps required by the business.

Symptom

  • A custom template fails during data load into CDL
  • Error "SocketException: Connection reset" occurs when submitting the Pipeline REST job.

Environment

  • SAP Sales Performance Management
  • Incentive Management on Hyperscalers
  • Incentive Management on Legacy HANA 

Reproducing the Issue

  1. Load the control file for the data load.
  2. Load the custom template file; observe that the load fails.
  3. The error is related with Error while submitting job to Pipeline: java.net.SocketException: SocketException invoking <TENANT_INFO> Connection reset on the PIPELINE_PREPARE_DATA milestone.

Cause

Submission to the Pipeline REST API (submitJob) returns a SocketException with connection reset, indicating the HTTP connection between the data loader service and the pipeline service was reset (for example, due to network instability, service unavailability/overload, or firewall/proxy interference).

Resolution

  1. Split large data files into smaller batches to reduce pipeline processing time and avoid long-running requests that may exceed timeout limits.
  2. Retry the job after a short interval, as connection reset errors can be intermittent and may succeed on a subsequent run.
  3. Validate the file structure against the expected schema and ensure all required fields are present before reloading.

Symptom

ADO.NET Provider is not listed in the assembly/GAC

Environment

  • SAP Adaptive Server Enterprise (ASE) Software Developer Kit (SDK) 16.0
  • Adaptive Server Enterprise ADO.NET Data Provider 16.0
    • 3.5 Provider (Sybase.AdoNet35.AseClient.dll)
    • 4.0 Provider (Sybase.AdoNet4.AseClient.dll)
  • Windows x64 (64-bit)

Reproducing the Issue

  • Install SDK for SAP ASE 16.0
    • Sybase.AdoNet35.AseClient.dll look for the provider under C:\windows\assembly
    • Sybase.AdoNet4.AseClient.dll look in C:\windows\Microsoft.NET\assembly\GAC_MSIL\Sybase.AdoNet4.AseClient

Cause

  • The SDK 16.0 installer script for the patches inadvertently wasn't checking prerequisites for the 3.5, 4.0 and higher .NET Framework and therefore didn't register.
  • The issue has been identified and logged under SAP CR 780635. Its description reads, "Installer doesn't register ADO.NET Provider".

Resolution

  • This issue has been addressed in:
    • SDK for SAP ASE 16.0 SP01 PL01 and higher
  • Workaround:
    • Run as administrator open command shell and change to the %SYBASE%\DataAccess\ADONET\dll directory and run these commands:
    • For Sybase.AdoNet35.AseClient:
      • AseGacUtility35 /i Sybase.AdoNet35.AseClient.dll
      • Do same for policies if you use them
      • AdoNetRegistrar35
      • Repeat the same under the DataAccess64\ADONET\dll directory
    • For Sybase.AdoNet4.AseClient:
      • AseGacUtility4 /i Sybase.AdoNet4.AseClient.dll
      • Do same for policies if you use them
      • AdoNetRegistrar4
      • Repeat the same under the DataAccess64\ADONET\dll directory
    • To complete the step so you have DDEX ability in Visual Studio (ADO.NET intergrated in the studio) please refer to KBA 1920705 to complete that registration

Symptom

When performing a reversal of activity allocation (KB24N) posted on a sales order, while having a lock in OKP1 active for "Actual overhead (periodic)" for the relevant period, the message handler is not set in RTOHC_DOCUMENT_CREATE and the messages collected cause an error KI806, resulting in the documents not being reversed.

The message handler is set only for specific objects in RTOHC_DOCUMENT_CREATE. For other objects (such as sales order), the handler is not set and messages collected during the creation of the documents are then handled differently. This note ensures the message handler is always set.

Other Terms

RTOHC, Real-time overhead, Event-based overhead, KB24N, RTOHC_DOCUMENT_CREATE, message handler, error message KI806

Reason and Prerequisites

Program error.

Solution

The message handler has to be always set in RTOHC_DOCUMENT_CREATE. Implement this note or apply corresponding SP.

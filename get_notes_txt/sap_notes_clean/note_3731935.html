Symptom

  • Test remote source connection to SAC through CloudDataIntegrationAdapter failed
  • Error: "SAP DBTech JDBC:[403]:internal error: Cannot get remote source objects: [Adapter error] Initializing SSL Context failed.java.io.IOException: Keystore was tampered with, password was incorrect"
  • Detail error in DP agent log.
    "DPFramework | WorkerThread.processRequest [] - com.sap.hana.dp.adapter.sdk.AdapterException: [Adapter error] Initializing SSL Context failed. java.io.IOException: Keystore was tampered with, or password was incorrect Context: Initializing SSL Context failed. java.io.IOException: Keystore was tampered with, or password was incorrect

Environment

SAP HANA smart data integration (SDI)

Reproducing the Issue

  1. Create or open a remote source configured with CloudDataIntegrationAdapter via the data provisioning agent.
  2. Attempt to browse or fetch remote source objects.
  3. Observe the SSL initialization failure with the keystore/password error.

Cause

Incorrect keystore password configuration for the data provisioning agent’s SSL keystore/truststore used, leading to SSL context initialization failure.

Resolution

  1. Go to DP agent installation bin folder and run "./agentcli.sh --setSecureProperty"
  2. Choose the following option and give password : changeit(Default value, you can also use other string value as the password)
    11. Set default Keystore Password (for Cloud Connectivity)
    12. Set Adapter keystore password
    13. Set Adapter Truststore Password
  3. Restart DP agent and try the connection again.

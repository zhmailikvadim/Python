Symptom

While opening the account's detail view/quick view, the validation message shows up.

Environment

  • SAP Sales Cloud Version 2
  • SAP Service Cloud Version 2
  • SAP Enterprise Service Management  

Reproducing the Issue

  1. Go to accounts.
  2. Open any account.
  3. The validation message shows up. 

Resolution

While opening the account detail view/quick view, the check consistency action triggers for this account. This action triggers the custom validations(e.g., external hooks and codelist restriction validations). For custom logic, please note only postHook triggers and preHook doesn't trigger. 

This is designed for the scenario that if the end user creates an account using the quick creation view and a custom info message appears, the same message will appear again in the account detail view after clicking Save and Open button. Please note if account data is created at time t1 and a new custom validation is added later at time t2 that invalidates the existing account, the corresponding custom error messages will appear when the user opens these "invalid" accounts.

This action cannot be disabled for accounts via the setting. Currently, this check consistency action is only introduced for account. It will be introduced for other Business Partners according to the product roadmap.

Symptom

This note comprises several corrections for TM Logistics Integration in the context of intercompany processes.

1. Actual Transfer of Control dates are not (correctly) propagated for incoterms EXW, FCA, DDP, DAP or DPU in case no explicit incoterm location is given.

2. Customizing view /SCMTMS/V_INCVST can contain mappings of incoterms to specific transportation activities at which corresponding Transfer of Controls should happen. There has been no customizing activity for maintaining this view. This activity is introduced with this note. Furthermore, this customizing has only been considered while posting actual Transfer of Control dates from TM towards LE outbound deliveries but not while determining and propagating planned Transfer of Control dates towards LE outbound deliveriies. With this note, the mentioned customizing is also considered in the context of planned Transfer of Control dates.

3. Planned Transfer of Control dates are determined and propagated from TM towards LE outbound deliveries based on stops with the correct incoterm location but sometimes with the wrong stop category (inbound stop vs. outbound stop). Even if the stop category is correct, the planned ToC date is always determined based on the given arrival or departure date, respectively, although other available dates (e.g. loading end) would suit better. With this note also the above mentioned customizing via view /SCMTMS/V_INCVST is considered when it has to be decided which TOR stop and which available date on this stop should be considered.

4. Missing stages in delivery-based freight units in case packages or batch materials are involved.

Other Terms

Transportation Management; Logistics Integration; Intercompany; Transfer of Control Dates; Logistics Execution; Outbound Delivery; /SCMTMS/V_INCVST

Reason and Prerequisites

 Program Error

Solution

Implement the corrections of the attached correction instructions or apply the corresponding support package.

New IMG Activity for maintenance of customizing view '/SCMTMS/V_INCVST' is only available by applying the corresponding support package. If applying only the corresponding correction instructions, /SCMTMS/V_INCVST can be maintained, like before, via transaction SM30.

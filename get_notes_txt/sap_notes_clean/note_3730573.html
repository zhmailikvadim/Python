Symptom

You run the Programa de cálculo de nómina Argentina (HARCALC0) report for an employee in the following scenario:

  • The employee is fired with a termination date after the payroll pay date.
  • You run the regular payroll period for this employee.

In the scenario described above, the system does not liquidate the proration value, when it should identify the employee termination and liquidate the proration.

Reason and Prerequisites

Reason

This issue happens because the system only uses the pay date as limit for checking the employee firing.

Prerequisites

The following Support Package levels are the minimum prerequisites:

Release Support Package
S4HCM 102 HR SP 00
S4HCM 101 HR SP 06
S4HCM 100 HR SP 19
SAP_HR 608 HR SP D0
SAP_HR 604 (EhP6 e EhP7) HR SP K2

For more information, access the SAP ERP Human Capital Management - Support Package Schedules.

When you install an SAP Note, some prerequisites might be required. The prerequisites are SAP Notes that have already been released and that change the same context of the current one. Then, even if these prerequisites are not necessarily related to the symptom of this SAP Note, you must implement them due to technical reasons.

Solution

From now on, the system compares the payroll period end date and the pay date, and uses the later one.

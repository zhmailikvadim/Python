Symptom

After data migration to SAP S/4HANA Finance, table ACDOCA contains wrong Debit/Credit indicators for negative posting items migrated from BSEG.

Affected ACDOCA items have MIG_SOURCE = 'F' and BSTAT = space. The related BSEG item has XNEGP = 'X' indicating a negative posting. The value of ACDOCA-DRCRK is identical to BSEG-SHKZG, but it should be opposite:

  • For a BSEG item with XNEGP = 'X' and SHKZG = 'S', ACDOCA-DRCRK should be 'H', although the ACDOCA-amounts are positive
  • For a BSEG item with XNEGP = 'X' and SHKZG = 'H', ACDOCA-DRCRK should be 'S', although the ACDOCA amounts are negative
  • For normal BSEG items with XNEGP = space, ACDOCA-DRCRK is identical to BSEG-SHKZG (for ACDOCA-MIG_SOURCE = 'F')

ACDOCA items migrated from NewG/L (MIG_SOURCE = 'F') are not affected as the Debit/Credit indicator is taken from FAGLFLEXA in that case.

Solution

Implement the correction instructions.

Execute report FINS_MIG_UJ_MAPPING_GENERATE to re-generate ZCL_FINS_MIG_UJ_HDB_GENERATED.

Repeat the line item migration as described in note 2138644. In case migration is already finished, apply the correction report of note 2327300 to fix the affected ACDOCA records.

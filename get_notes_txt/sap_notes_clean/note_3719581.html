Symptom

While changing the master inspection characteristic using the F4 help in fiori app QP02 (or in SAP GUI for HTML / WEBGUI) a "control not found" error is raised.

An example:

Received "control not found" on batch step #10: {"post":"action/3/wnd[1]/shell/gridRESULT[0]/tbar/btnCOPY"}
ID: 7930346

The complete batch was:
1: {"post":"action/61/wnd[1]/shell/gridRESULT[0]","content":"position=10","logic":"ignore"}
2: {"post":"action/61/wnd[1]/shell/gridRESULT[0]","content":"position=13","logic":"ignore"}
3: {"post":"action/61/wnd[1]/shell/gridRESULT[0]","content":"position=16","logic":"ignore"}
4: {"post":"action/61/wnd[1]/shell/gridRESULT[0]","content":"position=17","logic":"ignore"}
5: {"post":"action/61/wnd[1]/shell/gridRESULT[0]","content":"position=18","logic":"ignore"}
6: {"post":"action/61/wnd[1]/shell/gridRESULT[0]","content":"position=19","logic":"ignore"}
7: {"post":"action/61/wnd[1]/shell/gridRESULT[0]","content":"position=20","logic":"ignore"}
8: {"post":"action/61/wnd[1]/shell/gridRESULT[0]","content":"position=21","logic":"ignore"}
9: {"post":"action/7/wnd[1]/shell/gridRESULT[0]/rowcol/row[26]","logic":"ignore"}
10: {"post":"action/3/wnd[1]/shell/gridRESULT[0]/tbar/btnCOPY"}
11: {"get":"state/ur"}

Such issue does not occur in SAP GUI for Windows (transaction QP02) and this problem occurs since kernel upgrade.

Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.

Environment

  • SAP S/4HANA, on-premise
  • SAP S/4HANA Cloud Private Edition

Reproducing the Issue

  1. Open app 'Change Inspection Plan' (QP02).
  2. Fill Material and Plant (or the Group and Counter).
  3. Select operation.
  4. Click 'Inspection Characteristics'.
  5. Press F4 on 'Master Insp. Charac.'.
  6. Click 'Go'.
  7. Select MasterIC 'LENGTH1'
  8. Click 'OK'.
  9. Error:
    Received "control not found" on batch step #10: {"post":"action/3/wnd[1]/shell/gridRESULT[0]/tbar/btnCOPY"}
    ID: 7930346
    The complete batch was:
    1: {"post":"action/61/wnd[1]/shell/gridRESULT[0]","content":"position=10","logic":"ignore"}
    2: {"post":"action/61/wnd[1]/shell/gridRESULT[0]","content":"position=13","logic":"ignore"}
    ...
    9: {"post":"action/7/wnd[1]/shell/gridRESULT[0]/rowcol/row[26]","logic":"ignore"}
    10: {"post":"action/3/wnd[1]/shell/gridRESULT[0]/tbar/btnCOPY"}
    11: {"get":"state/ur"}

Resolution

Follow the instructions of following SAP Note to resolve the issue:

3708147 - Valuehelp with error on toolbar button.

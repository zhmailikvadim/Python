Symptom

You are trying to create or change a maintenance plan using the Function Module MPLAN_CREATE or MPLAN_CHANGE. You notice that an inconsistent horizon can be maintained, which can lead to unpredictable behavior in Maintenance Plans. This can also occur with incorrectly maintained source file for the Migration Cockpit.

Other Terms

MPLAN_API, Call Horizon, MPLAN_CHECKS, MPLAN_CREATE, MPLAN_CHANGE, DMC

Reason and Prerequisites

This is caused by validation class MPLAN_CHECKS that missed the validation for a specific scenario: when a horizon is provided in days (HORIZ_DAYS) along with the unit (HORIZ_QUALIFIER) selected as percentage.

Solution

Please implement the attached correction instructions or Support Packages.

To correct maintenance plans with inconsistent Call Horizon, please run the attached correction report. For execution steps, please refer to SAP Note 3585235.

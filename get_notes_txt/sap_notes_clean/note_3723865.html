Symptom

You've noticed that changes made to the Main Layout are automatically applied to all page layouts.

Environment

  • SAP Sales Cloud Version 2
  • SAP Service Cloud Version 2

Reproducing the Issue

  1. Navigate to an entity screen (e.g., Account, Case, Opportunity, etc.).
  2. Click on the Profile Icon.
  3. Select "Start Adaptation".
  4. Choose "Main Layout".
  5. Make a change in the layout (e.g., change the field order).
  6. Click on "Apply".
  7. Switch from "Main Layout" to a Custom Page Layout.
  8. Observe that the changes are also visible in the Custom Page Layout.

Cause

The Main Layout is designed for adaptation changes that all users can see. Custom Page Layouts, on the other hand, are intended for specific user customizations.

For example, if you want to add a new field to the account screen that everyone needs, you should add it to the Main Layout, making it available to everyone with just one change. If only one group needs the field, you add it to their specific Page Layout. If all users except one group need the field, you can either add it to all other Page Layouts or add it to the Main Layout and then hide it in the Page Layout for that one group.

This approach saves time because you make changes for everyone at once instead of repeating them in each Page Layout. If a change is accidentally made in the Main Layout, it can be easily undone there.

Resolution

This behavior is expected. The Main Layout is designed to serve as the default template for all Page Layouts, ensuring consistency across all user experiences. Changes made to the Main Layout are reflected in all Page Layouts to streamline adaptations and save time, as it eliminates the need to make identical changes in multiple layouts. If you need to customize specific fields or sections for certain user groups, you should modify their respective Custom Page Layouts instead.

Symptom

When the ‘New Work Order Requires New Employment’ upgrade has been implemented, it requires that a Contingent Worker be rehired on a new employment when they need a new work order.

However, even given this, when going through ‘Add Contingent Worker’ > ‘Accept Match’, the old Work Order’s details are forwarded to the ‘Work Order’ section. This forces manual removal of the old details in order to fill in the new Work Order details, including any attachments.

If the attachment from the old Work Order is kept in the new Work Order details, then when processing a workflow for the hire, the approver receives an error due to the attachment from the old Work Order being included.

Environment

SAP SuccessFactors Employee Central

Reproducing the Issue

  1. Log into environment
  2. Go to 'Upgrade Center' and confirm the 'New Work Order Requires New Employment' upgrade has been completed
  3. Go to 'Add Contingent Worker'
  4. Fill in the details for an existing (inactive) Contingent Worker
  5. Duplicate Match UI shows the existing worker - select 'Accept Match' to accept this worker as a rehire match
  6. The rest of the sections load - upon getting to the 'Work Order' section, the employee's old employment's Work Order details populate

Cause

Current expected behaviour - enhancement under review

Resolution

This is currently the expected behaviour, however, a code correction is under review for enhancement (ECT-267292). This enhancement will adjust the behaviour so that the old Work Order data is not carried forward on rehire with new employment.

This code correction currently has no resolution / fix timeline, but if you provide the code (ECT-267292) to your CSM, they can track the correction and its progress.

Symptom

eDocument Italy XML - the tag <RegimeFiscale> is populated with the value RF18, however it is expected to be populated as RF01, according to the configuration maintained in the SSCUI 102739 (Maintain Additional Parameters).

Environment

  • SAP S/4HANA Cloud Public Edition
  • SAP Document and Reporting Edition, Cloud Edition
 

Reproducing the Issue

  1. Go to the eDocument Cockpit application.
  2. Select the relevant Source Document.
  3. Go to... -> Display/Preview XML.
  4. Open the downloaded file.

The tag <RegimeFiscale> is displayed with the value RF18.

Resolution

In case the value in the tag <RegimeFiscale> requires to be changed in the XML of the eDocuments without modifying the SAP Standard code, follow the below configuration to modify the information within the XML tags -

Badi EDOC_ADAPTOR_CLOUD

  • As an alternative to directly influence the determination of value in the <RegimeFiscale> tag and avoid incorrect information, it is recommended to utilize the BAdI EDOC_ADAPTOR_CLOUD. This approach allows for the desired modifications to be made within the XML information effectively. (Enhancing the Standard XML File for Electronic Documents using the BAdI EDOC_ADAPTOR_CLOUD).
  • Using the Badi EDOC_ADAPTOR_CLOUD is possible to directly change the information maintained in the tags of the XML of the eDocument through a custom logic.

See Also

  • KBA 3578879 - Enhancing the Standard XML File for Electronic Documents using the BAdI EDOC_ADAPTOR_CLOUD.
  • The IREGFI parameter is used to define the tax regime of the supplier, which is a four-digit code starting with RF followed by a number. Here user can configure the IREGFI parameter to set the RegimeFiscale value to RF01 - Configuring Additional Parameters for Company Code | SAP Help Portal.

Symptom

The goods issue posted for the outbound delivery with quantity greater than the quantity in the STO item when overdelivery tolerance maintained as 0.0%.

Environment

SAP S/4HANA Cloud Public Edition

Reproducing the Issue

  1. Create STO (order type UD) via App 'Create Purchase Order - Advanced'
  2. In the Delivery tab for the item, set the Overdelivery tolerance to 0.0%
  3. Create delivery for the STO via App 'My Purchase Orders - Due for Delivery'
  4. Increase the delivery quantity in the outbound delivery and post goods issue
  5. System allows to post the goods issue for the outbound delivery with quantity that exceeds the STO quantity.

Cause

The message for the overdelivery tolerance is determined by the 'Overdelivery' setting in the SSCUI 102974: Define Delivery Item Categories which is set to ' ' i.e.,  No system response.

Resolution

  1. Review the configuration for SSCUI 102974 'Define Delivery Item Categories'.
  2. Locate the field for item category NLN -> 'Quantity' -> 'over-delivery message'.
  3. Change the message settings in SSCUI 102974 to either 'warning' or 'error', depending on business requirements.
  4. Test the configuration to ensure the system displays the appropriate message and prevents over-delivery during goods issue.

See Also

  1. Go to the configuration step: SSCUI 102974 - Define Delivery Item Categories
  2. Select to the item category (NLN for standard STO item) and click on details
  3. Maintain 'Overdelivery' as required by the business:

    Not maintained = "No System Response".
    A = "Display Warning Message".
    B = "Cancel action and display error message".

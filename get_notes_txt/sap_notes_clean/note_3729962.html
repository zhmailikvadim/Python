Symptom

Why are my delivery and shipping costs being created as an unassigned items on my PO-based Invoice?

When importing using means other than the API, the PO Based invoices are getting their header shipping/delivery costs added as unassigned Delivery Cost items.  This stops the automated processing of invoices and requires manual intervention.

Environment

  • Product: SAP Ariba Invoicing
  • CIM Invoice Processing

Reproducing the Issue

  1. Receive a PO-based supplier invoice that includes shipping/freight (unplanned delivery costs) via email into CIM.
  2. CIM recognizes the delivery cost and creates a separate Delivery Cost item on the invoice.
  3. The Delivery Cost item remains unassigned and the invoice does not auto-post until a user manually assigns/applies it.

Cause

For PO-based invoices, shipping/freight captured as unplanned delivery costs are created as a separate Delivery Cost item that is not auto-assigned. This is expected system behavior and conflicts with a no-touch processing goal for emailed invoices.

Resolution

  1. Note that the observed behavior is working as designed: unplanned delivery cost items created from emailed PO invoices require manual assignment before posting.
  2. If automatic handling is required, consider submitting invoices via API and leverage available functionalities to programmatically manage delivery cost items (refer to: CIM – API invoice processing capabilities).
  3. For invoices submitted by email (non-API), request product enhancement to support automatic assignment/processing of unplanned delivery costs (refer to: Note 3633433 – How to create an improvement or enhancement request for SAP Ariba Invoicing).
  4. As a current workaround for emailed invoices, continue manual assignment of the Delivery Cost item to proceed with posting.

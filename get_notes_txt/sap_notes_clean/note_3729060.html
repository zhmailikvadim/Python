Symptom

This SAP Note contains the adjustments of the communication procedure ELDA (P321) to the organizational description for the data exchange with hospitals (KA), issue 11/2025, valid as of 5/1/2026.

As of Version 4: The (prerequisite) SAP Note 3579061 has been added to the manual pre-implementation steps.

Other Terms

KA-ORG version issue 11/2025, ELDA communication, P321, ELDA
EVSNR (EEA insurance number), EKANR (card number) for EKVK data, ANFO, EKVK 
EREIG, ENTBDAT, foreigner clearing, subrogation/recovery

Reason and Prerequisites

Prerequisites:

  • Country Version Austria (AT)
  • Use of the communication procedure ELDA (P321)  
  • SAP Note 3730468

Reason:

Legal change

Solution

Implement the program changes and perform the manual tasks.
You can implement this SAP Note at any time. The changes are not taken into account until the key date 5/1/2026.

In particular, note the following changes:

  • D.19. - EREIG (event type, admission notification)

The value 7 (AUVA Resumption) is no longer valid. If necessary, note this in a BAdI implementation or in the course of a conversion.

  • E.4. - Admission notification

The Delivery Day field (ENTBDAT) becomes a reserve field. This information is no longer transferred. This field still has the type "numeric", which means that only zeros are transferred here.

  • E.13. - Foreign clearing/subrogation/recovery confirmation

There is only one notification from the PPA agency to the state health fund and no more copy to the hospital.

  • E.15. - EHIC data notification
    • The EEA insurance number (field EVSNR) is no longer transferred if the field EFORM = ANFO. The information remains enterable.
    • The following guidelines now apply to the card number of an existing European insurance card (field EKANR):
      • If fewer than 20 characters are entered, they must be filled with leading zeros.
      • Non-numeric characters contained in it must be replaced by 0 (zero). The system issues the new information NV 624 in the attachment to the EDI message.
      • If the card number is garbled, use the EFORM "ANFO".
      • If the card number is longer than 20 characters (not possible in the standard IS-H system), the leading characters must be truncated as far as necessary.

You can find these and other changes in the document Organization Description KA (Data Exchange with Hospitals), issue 11/2025, on the homepage of the Austrian Social Insurance.

Symptom

Your DB user has created a backup schedule in HANA Cockpit. You found the backup is taken on schedule even the user is deactivated manually.

Environment

HANA Cockpit 2.0

Cause

This is an expected behavior. After the backup schedule is created in HANA Cockpit for a target database by the database user, the job is scheduled and saved in the table SCHEDULER_JOBS of the target database. After the user is deactivated, DB connect is forbidden but those jobs scheduled by the user will be executed normally on schedule.

SELECT * FROM "SYS"."SCHEDULER_JOBS";

Resolution

Explicitly delete the backup schedule in HANA Cockpit. Note: If the DB user is deleted in target database, all scheduled jobs created by the user will be deleted as well.

Keywords

scheduled job works after user is deactivated, scheduled job is executed after user is deactivated

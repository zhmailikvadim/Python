Symptom

  • Users shown as LDAP in master..syslogins (status &64=64) can still log in with ASE authentication even when "enable ldap user auth" is 0.
  • sp_displaylogin shows "Authenticate with: AUTH_DEFAULT" for affected logins.
  • @@authmech during successful sessions returns "ase".
  • master..syslogins status values include 480 (ASE+LDAP+PAM+Kerberos), indicating multiple authentication bits set.

Environment

  • SAP Adaptive Server Enterprise (ASE) 16.x
  • SAP Adaptive Server Enterprise (ASE) 15.x

Cause

A historical defect caused "authenticate with ANY" to set an incorrect status value (480) instead of 0, persisting multi-bit authentication flags into later versions.

The issue has been logged under SAP CR 493107. Its description reads, "sp_modifylogin <login>, "authenticate with", "ANY" sets incorrect status value" 
The problem is that if you set the login authentication type to ANY with sp_modifylogin or sp_addlogin, the system incorrectly sets 480 (it should be 0).
(*sp_modifylogin, sp_addlogin not supported in current version)

Resolution

  • CR 493107 was already fixed in ASE 12.5.4 and 15.0.2. 
  • Set any authentication type in the "alter login" command.

    alter login <login name> modify authenticate with ANY
    go

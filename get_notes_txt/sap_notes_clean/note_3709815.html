Symptom

How the price determined for sales order stock transferred from another sales order via movement type, such 413 E.

Environment

SAP S/4HANA Cloud Public Edition

Cause

It is not possible to "transfer" the price with the transfer of the quantity between sales orders.

Resolution

It is not possible to "transfer" the price with the transfer of the quantity between sales orders.
The valuation of the quantity of the target sales order will follow the standard rules that first it looks if there is already a EBEW segment (from previous posting) and if not uses the standard price.

Its the same logic as in standard MTO process in Make-to-Order (MTO).

To achieve to have the same price, customer would need to do manual price change for the sales order stock after transfer posting via app 'Change Material Prices'.

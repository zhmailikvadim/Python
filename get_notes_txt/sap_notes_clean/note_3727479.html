Symptom

Following behavior observed when distribute materials using change pointers.  

  • No master iDoc and communication iDoc created for message type MATMAS in t-code BD21. 
  • Specific MATMAS change pointers in table BDCP2 are never proceed. 
  • No change pointers created in table BDCP2 when update specific material.

Environment

  • SAP ERP Logistics (LO)  
  • SAP R/3
  • SAP R/3 Enterprise
  • SAP ERP Central Component
  • SAP ERP
  • SAP Enhancement package for SAP ERP
  • SAP Enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4 HANA on premise
  • SAP S/4 HANA Cloud Private Edition

Reproducing the Issue

  1. Open t-code BD21. Input message type MATMAS, execute. 
  2. Message "0 master IDocs set up for message type MATMAS" pop up. 
  3. Message "0 communication IDoc(s) generated for message type MATMAS" pop up. 
  4. Checking table BDCP2, change pointers are not being proceed. 

Cause

The material recorded in change pointer is blocked for distribution. Refer to following steps to check if the material has been blocked for ALE distribution. 

  1. Check table BDCP2 and gets material number from change pointer.
  2. Check material basic data 1 view in t-code MM03. The material has cross plant material status maintained.
  3. Check the material status definition in t-code OMS4. Distribution lock set in the material status.

Resolution

Consider one of the following options if the material need to be distributed by change pointers. 

  • Change to another cross plant material status which has no distribution lock set in t-code MM02. 
  • Change the material status customizing and deactivate the distribution lock in t-code OMS4. 

Alternatively, distribution lock is not taken into account in t-code BD10. Consider distribute the material directly if material status cannot be changed.

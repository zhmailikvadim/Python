Symptom

Previously, any user could edit, process, or delete inbound transactions in the Inbound Transaction Monitor without authorization restrictions. All users had unrestricted access to these operations regardless of their assigned roles or permissions. Now, authorization object /SMFND/A01 before enabling access to these functions.

Other Terms

  • Mobile Application Integration Framework
  • Inbound Transaction Monitor
  • /SMFND/A01
  • /SYCLO/ADMIN
  • S4MOBADDON-8331

Reason and Prerequisites

This is new functionality to ensure that only intended users can modify inbound transactions.

Solution

Apply the attached correction instructions. If not already present, update your user's authorization profile to contain /SMFND/A01 with the activities you want access to.

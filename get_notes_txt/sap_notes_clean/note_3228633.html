Symptom

When you prefer to activate SAP S/4HANA SAP Best Practices 2022(on premise) in a merged client (with all client 000 reference settings), you will encounter some activation errors due to already existing client 000 settings. This SAP note Provides step by step guidance on how to overcome these activation errors.

Other Terms

SAP S/4HANA Best practices activation, On premise Best practices activation, 2022 SAP Best Practices activation in a merged client.

Reason and Prerequisites

SAP Best Practices for SAP S/4HANA On premise were developed and tested on a Best Practices client. The Best Practices are first developed for SAP S/4HANA Cloud and once a year, selected scope items from SAP S/4HANA cloud are tested on an on-premise system in a Best Practices client with only limited IMG client reference 000 settings. Therefore, SAP Best Practices On premise versions are always tested on a SAP Best Practices client by default before their release.

In case, you need to configure additional functionality not provided via S/4HANA Best Practices you would need client 000 reference settings for those additional IMG activities. Therefore, you may need a merged client. A merged client has all client 000 reference settings (not limited to whitelist tables) as it is a full copy of client 000.

Since SAP Best Practices were developed and tested on a Best Practices client some errors will show up when you activate it on a merged client. Therefore, the customers need an activation guidance document to overccome these activation errors.

Solution

Activation guidance documents are attached for Germany and USA country solutions for current releases. In general, the activation errors and the resolutions are also applicable for other countries, with minor difference in Germany or the USA. We expect the same documents also help all country Best Practice activations for the same release.

The activation guidance documents for each release of 2022 will be attached to this note.

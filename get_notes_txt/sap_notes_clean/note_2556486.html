Data Protection and Privacy: ﻿How to Configure the Data Object Auditing Based on Snapshot Recording

To simplify the deletion of personal data, SAP recommends that you pay attention to the following technical information.

In SAP Convergent Charging, end-customer data is implemented in Master Data as subscriber accounts and provider contracts. These concepts can be viewed as data containers into which an SAP CC user can store some end-customer information required for the charging process. As an SAP CC user, if you design and customize these containers to store some personal data, note that the Data and Operation Auditing functions of SAP CC may record these containers into the back-end database system as data object snapshots when some modify or delete operations are applied on them. It depends on your SAP CC system configuration, your security policy, and your data protectionand privacy legislation.

When delete operations are performed on subscriber accounts and provider contracts, the containers are physically destroyed from the database systems and from the SAP CC systems. However, some object snapshot records can still persist in the audit data. For avoiding SAP CC to create audit snapshots for subscriber accounts and provider contracts, if they contain personal data, you can configure the system to stop the background auditing. As an SAP CC system administrator, disable the function (see the AUDIT_OBJECT_ACTIVATION system parameter) or change the AUDIT_OBJECT_DOMAIN system parameter by removing the following audit domains: CUSTOMER and CUSTOMER_TABLE. The AUDIT_OBJECT_DOMAIN system parameter is available as of SAP CC 4.1 SP 2.

For more user assistance and information, refer to the SAP CC 5.0 Security Guide about data protection and privacy, personal data, and sensitive personal data.

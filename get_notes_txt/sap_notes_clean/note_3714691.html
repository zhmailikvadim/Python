Symptom

  • When refreshing/Scheduling a BW-based report using non-compounded key values entered through the prompt window, the report returns blank data.
  • The report functions correctly if you select a value from the available list of values
  • The prompt is based on a compound object in the BW system.
     
     

Environment

  • SAP BusinessObjects Business Intelligence platform 4.X
  • Web Intelligence (WEBI)
  • SAP BW 
  • OLAP BICS

Reproducing the Issue

  1. Create a Web Intelligence Document based on a SAP BW OLAP BICS connection.
  2. Make sure the BW query should contain a Characteristic that is compounded
    Eg: <Compounded values> <TEXT (SHORT/MEDIUM/LONG)>
  3. In the Query Panel, drag the compounded object into the Result objects and Query Filter windows. Set the filter type to "Prompt"
  4. Enter the non -compounded value manually instead of selection from list of values 
  5. Run the report.
    >> Observe no data is retrieving in the report level

Cause

  • Compounding is not supported, and DSL looks for Text

Resolution

  • If an object is configured as a compounded object in the BW system, WEBI will always search for compounded keys.

        or 

  • Select the values from list
    or 
  • Enter key or display key value
    or 
  • Remove compounding on BW side

Symptom

When a contract contains multiple line items, the "Approve ACM Trading Contract" application displays the same 'Delivery Period From' and 'Delivery Period To' values across all rows in the list view, even when the line items have different delivery periods.

Replication Steps

1.Create a trading contract in the "Maintain ACM Trading Contracts" application.

2.Add at least two line items with different 'Delivery Period From' and 'Delivery Period To' values for each line item.

3.Save the contract.

4.Open the "Approve ACM Trading Contract" application.

5.Ensure that the 'Delivery Period From' and 'Delivery Period To' columns are visible in the list table.

6.Search for the created contract.

7. The list table displays contract with multiple line items having same 'Delivery Period From' and 'Delivery Period To' dates.

Other Terms

Agricultural Contract Management; ACM; Approve ACM Trading Contract; Delivery Period From; Delivery Period To;

Reason and Prerequisites

Reason:This issue occurs because the key field defined for the list display entity (List Screen Table) in the "Approve ACM Trading Contract" application is 'Contract Number'.As a result, the Fiori UI framework overwrites non-key-related data with the latest record when multiple entries share the same key.

Pre-requisites: UI Note 3711668 must be implemented first. After that, implement UI Note 3706163 to ensure the solution to work as expected.

Solution

Import the relevant support pack or apply the correction instruction(s).

Symptom

The field GENFLAG gets set to X in table TADIR when creating an annotation model in Annotation Models via Service Reference (Type 3) in Service Builder.

Other Terms

Annotation via referenced service, SEGW, Service Builder, Annotation Model

Reason and Prerequisites

The field GENFLAG gets set to X (GENFLAG = X) in table TADIR for annotation model; this indicates that the objects generated cannot be changed/entered in a Transort Request. GENFLAG has to be blank for an annotation model to be transportable.

For new Type 3 ( Annotation models via service reference) projects created via Service Builder, the GENFLAG is set to blank.

For existing projects, follow the below steps to reset the flag:

      a) Navigate to transaction '/IWBEP/REG_VOCAN'.

      b) Find the annotation model for which the value for GENFLAG field needs to be reset

      c) Delete the annotation model in the same transaction

      d) Re-create the annotation model via the same transaction '/IWBEP/REG_VOCAN' ( do not re-generate through service builder, as it will still retain the same value )

      e) Check the entry in the TADIR table.

Symptom

  • Supplier invoices are received in document and reporting compliance, cloud edition, but are not pushed to the business system EDOC_COCKPIT.
  • Error message: “Error occurred during the step, detailed information: The connection to the business system identified by the destination returned the 415 HTTP status code and could not accept the payload. Try again in a few minutes and if the error persists, open a support request.”

Environment

  • SAP ERP
  • SAP S/4HANA
  • SAP S/4HANA Cloud Private Edition
  • SAP Document and Reporting Compliance, Cloud Edition

Reproducing the Issue

  1. Schedule a supplier-invoice pull job in Document and Reporting Compliance, cloud edition.
  2. Observe that documents appear in the cloud edition, but are not pushed to the business system eDocument Cockpit.
  3. In the errors/monitoring app, see the processing step failing with HTTP status code 415.

Cause

Destination Issue

Resolution

This issue generally happens when the Destination used to pull Supplier Invoices is not of GCO_INTERNAL service. Most common mistake that is made is using the PEPPOL Service. Please check the Destination and compare its URL.

  • PEPPOL: http://VirtualHost:VirtualPort/sap/bc/srt/rfc/sap/dcppushservice/200/dcppushservice/pushrequest
  • Other Countries: 
    • For SAP S/4HANA 2021 or lower & SAP ERP: http://VirtualHost:VirtualPort/sap/bc/rest/sap/gco_internal/
    • For SAP S/4HANA 2022 or higher: http://VirtualHost:VirtualPort/sap/bc/http/sap/gco_internal/.

For non-PEPPOL, the destination should have "Other Countries" URL.

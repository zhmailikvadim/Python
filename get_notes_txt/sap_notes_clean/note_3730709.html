Symptom

During the upgrade of SAP Cloud Connector on Windows using the .msi installer, the installation fails in the configuration adoption phase. The installation log shows entries similar to:

CallConfigurator(): Executing configuration adoption needed during upgrade.
ExecuteCommand(): C:\Program Files\Java\sapjvm_8\bin\java.exe -jar configurator.jar -upgrade
Error: Unable to access jarfile configurator.jar
ExecuteCommand(): Execution ended with return code 1.

The upgrade process stops and the installation is not completed.

Environment

SAP Cloud Connector

Reproducing the Issue

  1. Install SAP Cloud Connector on Windows
  2. Retain the existing configuration during uninstall
  3. Start upgrade using the .msi installer
  4. Installer extracts package and restores configuration
  5. During configuration adoption step, installer executes: java -jar configurator.jar -upgrade
  6. Installation fails with error: Unable to access jarfile configurator.jar

Cause

The installer executes the configuration adoption step using a relative path (configurator.jar). The error occurs when the configurator.jar file cannot be accessed in the execution context. This can happen due to one or more of the following:

  • The file configurator.jar is not present in the installation directory (incomplete extraction)
  • The installer is executed from a different working directory and cannot resolve the relative path
  • The file is blocked, removed, or quarantined by antivirus or endpoint security tools
  • Insufficient permissions to access the file
  • Installation executed from a network location instead of a local directory

Resolution

Verify the presence and accessibility of the file configurator.jar in the SAP Cloud Connector installation directory. Perform the following checks:

  1. Confirm that configurator.jar exists in the installation path (e.g. S:\SAP\scc20\)
  2. Run the configuration adoption command manually using absolute path: "<JAVA_HOME>\bin\java.exe" -jar "<SCC_installation_directory>\configurator.jar" -upgrade
  3. Ensure the installer is executed locally on the server (not from network share)
  4. Run the installation with administrative privileges
  5. Check antivirus/EDR tools for any blocked or quarantined files
  6. If the file is missing, re-run the installation or extract the package again

Symptom

You want to enforce special patient visiting regulations in your hospital: Patients in your hospital may only be visited by their relatives in exceptional cases.

Other Terms

Patient visit rules, COVID-19, ISH_NPRV_RNLPFOR3, RNLPFOR3, public list.

Reason and Prerequisites

Consulting.

Solution

With the solution proposed here, you can indicate in the public list that visitors are allowed for patients who are marked accordingly using a case classification. The proposed solution is intended as a suggestion and can be adapted to meet customer-specific requirements.

Preparation in the system:

  • Create a customer-specific case classification. (IMG: Industry Solution Hospital -> Patient Management -> Cases -> Define Case Classification Categories/Define Case Classification Types; the case classification 'PF'/'BE' is used in the example coding)
  • Use transaction SE11 to add a field (type CHAR, length 1) to the structure ISH_PUBLICLIST_ALV with an APPEND. The example coding uses the field name 'XXVISALLOW'.
  • In transaction SE19, create an implementation for the BAdI ISH_NPRV_RNLPFOR3. Copy the ABAP coding from the attachment to the method IF_EX_ISH_NRPV_RNLPFOR3~MODIFY.
  • Add the new field 'XXVISALLOW' to your ALV display variants.

Usage:

  • Enter the customer-specific case classification for patients who can be visited. (Example: 'PF'/'BE')
  • In the public list, an 'X' is displayed in the customer-specific column for patients who have this case classification.
  • The security guard uses the content of the column to decide whether the visitor can be let through.

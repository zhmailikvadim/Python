Symptom

The Foundation Object (FO) / Generic Object (GO) validation doesn't work during Onboarding steps, for example: New Hire Data Review step, Final Review step. 

Environment

SAP SuccessFactors Onboarding

Reproducing the Issue

  1. Initiate onboarding for a new hire.
  2. Navigate to the New Hire Data Review step.
  3. Enter an inactive job classification code for the candidate and submit the New Hire Data Review successfully.
  4. Go to Manage Pending Hires page, click on submit button, validation triggered:

    The selected JobClassification xxxxx is inactive on DD/MM/YYYY. You can either validate the JobClassification on Manage Data page or you can select another JobClassification.

Cause

This is expected behavior: validations based on Foundation Object (FO) / Generic Object (GO) are not evaluated during the Onboarding steps.

Resolution

  1. Create an onSave business rule for the Onboarding Job Information to perform the required validation (for example, check that the selected Job Classification is active on the effective date).
  2. Configure the rule so that it displays an appropriate validation message when the condition is not met.
  3. This rule needs to be added at the corresponding entity on Admin > Manage Business Configuration (jobInfo)
  4. Test by initiating onboarding and verifying that the rule validates the data during the Onboarding step.

Symptom

The Employee Central (EC) Payroll system is unable to calculate leave liability projections.

All Customers ( using EC Payroll, as well as using on-premise solution ) will see the following new objects :

BADI -> HR_AU_ECP_LEAVEPROVISIONS (Available in release SAP_HRCAU 608 , 604, 600 and EA-HRCAU 608).

BADI Implementation HR_AU_ECP_LEAVEPROVISIONS_IMPL (Available in EA-HRCAU 608).

Table HRSFEC_T5QLM (Available in EA-HRCAU 608)

However, the BADI implementation is activated only for customers using EC Payroll solution.

Other Terms

QLVP, QLVPR

Reason and Prerequisites

The Employee Central system is the leading system for time management. The Employee Central Payroll system was not equipped to perform the leave liability calculations, as all leave relevant data was not available to ECP. All leave balance data will now be sent to ECP system and stored here in IT 2010. ECP System has been enhanced to access this data, and process leave liability projections in payroll.

This enhancement will not impact the existing business logic, and hence customers not using EC payroll will not be affected by it.

Solution

Leave liabilities calculcation has now been enabled for the private sector payroll solution. Customers not using EC Payroll will not be impacted by changes in this note.

EC is the leading system for time management. All time snapshots are sent to ECP, and leave balances (entitlements and accruals) are now stored in IT 2010. Payroll has been enhanced to bypass running time simulation, so as to fetch all quota balances from IT 2010 instead. Leave liability calculations are performed using these balances.

Process flow:

Refer attached document - ECP AU Leave Provisions - Flow.doc

In order to achieve the above, following new objects have been created:

BADI Definition HR_AU_ECP_LEAVEPROVISIONS: This BADI allows for overriding the quota balances retrieved from time simulation in ECP system.

BADI Implementation HR_AU_ECP_LEAVEPROVISIONS_IMPL: This is the implementation of above BADI, but is only invoked when the business function for SFEC (HCM_SFEC_TDEC2HR) is activated.

Table HRSFEC_T5QLM : This is a customizing table, that maps quota types to wagetypes. EC does calendar runs, and sends leave balances to ECP, which eventually are stored in IT 2010, in the form of wagetypes. These wagetypes need to be mapped to corresponding quota types in ECP. ECP system has quota configuration available in V_T556A. The required mapping is done in V_HRSFEC_T5QLM (View for table HRSFEC_T5QLM).

Model wagetypes : NA60 (Accruals for Quota type 60 - Annual Leave) and NE60 (Entitlements for Quota type 60 - Annual Leave) have been created. Refer attached document(Model Wagetypes.doc) for wagetype configurations. Similarly two wagetypes need to be created for every quota type in ECP.

New IMG node : Payroll -> Payroll Australia -> Leave Provisions and Payments (Leave Quotas) -> Leave Provisions -> Map EC Leave Balance Info to ECP Wage Types. This new node opens up view V_HRSFEC_T5QLM, that allows for mapping quota types in ECP to corresponding wagetypes that store leave balances in IT 2010.

Some existing objects needed to be modified:

Include program RPCLPVQ1 : Has code for time simulation that is invoked from within Payroll. This needs to be overridden for ECP customers, and so BADI call introduced here.

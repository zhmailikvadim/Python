Symptom

Extended support for Windows Server 2016 ends on January 12, 2027. For more information on the Microsoft Support Lifecycle, see: http://support.microsoft.com/lifecycle. For more information on the Windows Server 2016 lifecycle, see: https://learn.microsoft.com/en-us/lifecycle/products/windows-server-2016.

Some customers signed a Windows Server 2016 custom support agreement with Microsoft to extend support by Microsoft beyond January 12, 2027.

This document describes how SAP supports existing installations after the end of extended support for Windows Server 2016.

Other Terms

Windows Server 2016 Support, End of Life, Microsoft Customer Specific Support Agreement

Reason and Prerequisites

---

Solution

SAP strongly recommends to upgrade all SAP installations that are based on Windows Server 2016 to the most current supported Windows Server version. There are multiple reasons why SAP customers should no longer run SAP software on Windows Server 2016. As of today, Windows Server 2016 is several years old and might have reliability limitations with regard to new hardware and newer Windows operating systems.

As of today, Windows Server 2019 and higher are supported operating systems for almost all SAP software and offers better stability and more reliability than older Windows Server operating systems. For more information about supported operating systems for your SAP product, see the SAP Product Availability Matrix.

SAP continues to work on customer incident messages that are created after January 12, 2027 for SAP systems running on Windows Server 2016.

SAP cannot commit to provide in all cases root cause analysis and solutions for problems reported for installations running on Windows Server 2016. Problems reported by customers running SAP systems on Windows Server 2016 may take significantly longer to be solved.

The following reasons may force the customers to migrate their SAP installations to a new Windows operating system, for example Windows Server 2019 or higher, to get a particular problem analyzed and solved:

  • The reported error is related to the operating system and SAP Support needs help from Microsoft to analyze the root cause: SAP no longer gets support for Windows Server 2016 from Microsoft. In this case, you either need customer specific support from Microsoft or you need to upgrade your system to a supported Windows Server release to get SAP support.
  • The reported error occurs in a third-party component licensed via SAP. For such components, and under the condition that the component version is still supported by the vendor, SAP is the first point of contact. SAP, however, can only provide support to the same extent as the third-party vendor provides support to SAP. If the error is related to the operating system, the vendor needs help from Microsoft to analyze the root cause: The vendor no longer gets support for Windows Server 2016 from Microsoft. In this case, you either need customer-specific support from Microsoft or you need to upgrade your system to a supported Windows Server release to get support.
  • The internal SAP IT Department no longer provides internal test systems based on Windows Server 2016. The existing systems will be shut down for missing security compliance or ending hardware support contracts. After the shutdown, SAP will no longer be able to run quality tests of SAP software on Windows Server 2016 (such as SAP kernels, SWPM or SAP JVM).
  • SAP Development support will no longer be able to reproduce customer problems on Windows Server 2016 for isolated root cause analysis. In this case, root cause analysis has to take place on the customer’s production system. This may cause system downtime.
  • Any testing or troubleshooting done by SAP will no longer be on the same Windows build as the one used by customers. SAP will no longer be able to test new kernel patches with Windows Server 2016 to detect potential incompatibilities.
  • SAP may change current available services (like SAP EarlyWatch, SAP CCMS etc.) by using new features that are no longer supported on a build for Windows Server 2016.
  • SAP will no longer consider Windows Server 2016 when planning new backward release compatible kernels.

In summary, customers are advised to upgrade their SAP platform to Windows Server 2016 or higher. Visit the SAP Product Availability Matrix (Your SAP Product - Technical Release Information - Database Platforms) for more information about supported combinations.

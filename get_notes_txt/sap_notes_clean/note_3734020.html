Symptom

When multiple recurring documents are created using transaction FBD1 (e.g., 10 documents) and subsequently processed through transaction F.14, it is observed that only a subset of these documents (e.g., 3 documents) may be included in the generated batch session. Users may require a report to verify which recurring documents were( Or were NOT) processed in the F.14 batch session. 

Environment

  • Financial Accounting (FI)
  • SAP ERP Central Component
  • SAP ERP and SAP ERP Central Component
  • SAP enhancement packages for SAP ERP
  • SAP S/4HANA
  • SAP S/4HANA Finance
  • SAP S/4HANA Cloud Private Edition
  • SAP Fiori for SAP S/4HANA

Reproducing the Issue

  1. Create multiple recurring documents using FBD1.
  2. Run F.14 to generate a batch input session for recurring documents.
  3. Observe that the batch session includes only a subset of the recurring documents.

Cause

If a recurring document is not processed by F.14 (no batch input session created for it), it was not included by the selection criteria of F.14 (for example, its “Next run on” date was outside the selected range). And there is no standard report available to explicitly verify which specific recurring documents were included in the F.14 batch session. 

Resolution

Consider the following approaches:

  1. Check Relevant Recurring Document Data in F.15:

    • You can review all relevant data for the recurring documents, such as the next run date, last run date, deletion indicator, and number of runs. This information can help indicate how many financial documents have already been posted from each recurring document.

  2. Utilize Table BKPF for Verification:

    • You can use the BKPF table fields, such as DBBLG (Rec.Entry Doc.), DBBLG GJAHR(GJAHR (RecEntry Fiscal Year), and DBBLG BUKRS (RecEntry Company Code), to determine which specific documents were posted for each recurring entry.

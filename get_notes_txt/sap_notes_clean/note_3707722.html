Symptom

The CVA (SAP Code Vulnerability Analyzer) terminates with the short dump OBJECTS_OBJREF_NOT_ASSIGNED in the class CL_SLIN_CLASS_INFO. The "Active Calls/Events" section of the short dump contains the following entries at the top:

METHOD CL_SLIN_CLASS_INFO============CP CL_SLIN_CLASS_INFO============CCIMP
GEN_PROC=>GET
METHOD CL_SLIN_CLASS_INFO============CP CL_SLIN_CLASS_INFO============CCIMP
SLIN_CLASS_INFO=>GET_METH
METHOD CL_SLIN_CLASS_CONTAINER=======CP CL_SLIN_CLASS_CONTAINER=======CCIMP
SLIN_CLASS_CONTAINER=>GET_PROC

Reason and Prerequisites

There is a program error in CVA. The problem occurs in the (very rare) case that the program to be checked contains a class definition and:

  • This class implements a global interface
  • A local interface with the same name is defined within the program
  • The names of the methods in which the global and local interfaces are not identical

Solution

Implement the related SAP_BASIS correction.

Symptom

You receive product versions in SV as Classic BOM, SV as PEO Version BOM, or COIPS as Order Bill of Material requests. These product versions always go to the product version process, which causes unwanted material updates. Product versions should only go to the product version process if they matter for compatibility mode or implicit plant creation.

Examples:

1. If compatibility mode is ON and the product version does not have key mapping, it should go to the product version process to create key mapping. But if key mapping already exists, it should not go.

2. If implicit plant creation is ON and the product version is not extended with a plant, it should go to the product version process. But if the product version already includes the plant, it should not go.

Also, the product version maintenance BAPI runs in the SV as PEO Version BOM process when processing unit of issue is involved, even if this unit of issue is already maintained. Product versions that do not need extension because they already include the unit of issue should not be sent to the BAPI.

Reason and Prerequisites

PLMSI, PLM system integration, SV, COIPS, SVV, Structure Version, Classic BOM, PEO Version BOM, COIPS as Order Bill of Material, compatibility mode, implicit plant creation, PV, Product version

Solution

Implement this SAP Note to ensure product versions are sent to the product version process only when they are relevant for compatibility mode or implicit plant creation.  Also, product versions that already contain SV as PEO Version BOM UoI do not get sent to the maintenance BAPI.

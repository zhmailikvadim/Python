Symptom

  • SAP Cloud ALM Projects flagged with Access Level "Private" or "Restricted" are not included in the results of requests submitted via the SAP Cloud ALM Projects API.
  • Only SAP Cloud ALM Projects with Access Level "Public" are included in the results of SAP Cloud ALM Projects API requests.

Environment

  • SAP Cloud ALM

Reproducing the Issue

  1. Execute a GET request for the SAP Cloud ALM endpoint "/api/calm-projects/v1/projects".
  2. Select a project in the JSON response and take note of its name/ID number.
  3. Logon to SAP Cloud ALM.
  4. Open the Projects and Setup app.
  5. Select the Project from step 2.
    1. The Project will have Access Level "Public".
  6. Change the Access Level to "Private".
  7. Save.
  8. Execute the same GET request for the SAP Cloud ALM endpoint "/api/calm-projects/v1/projects".
  9. The Project selected in step 2 will no longer be in the JSON response. 

Cause

  • The SAP Cloud ALM API service instance only has scopes for access to Public Projects.
  • The SAP Cloud ALM API service instance is missing scopes required to read Private and Restricted/Protected Projects.

Resolution

  1. Logon to the BTP cockpit.
  2. Select the "SAP Cloud ALM" Global account.
  3. Select the "SAP Cloud ALM" Subaccount.
  4. Select the Instances and Subscriptions tab on the left of the page.
  5. Locate the SAP Cloud ALM API service instance which owns the relevant Service Key/Binding being used for API requests.
  6. Click the overflow button (three dots, "...") on the row for the service instance.
  7. Select Update.
  8. Navigate to step 2 in the pop-up.
  9. Add the following OAuth scopes to allow access to non-public projects:
    1. $XSMASTERAPPNAME.calm-api.projects.private.read
    2. $XSMASTERAPPNAME.calm-api.projects.protected.read
  10. Click the update instance button in the pop-up.
  11. The API will now return Projects with Access Level Private and Restricted/Protected.

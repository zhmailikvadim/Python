Symptom

STO scenario: you pack the OD's into a cross-delivery HU and execute the GI posting for it. In ERP side the SPED will trigger the follow up ID's transferring the cross-HU info in the save replica message correctly, however in EWM side only a single delivery will contain the HU info in the HU tab in transaction /SCWM/PRDI, the rest of the deliveries are not displaying this info. If you open the HU content the HU items will be holding the reference to the other inbound deliveries, only the HU is not displayed in those other inbound deliveries in the HU tab.

Other Terms

HU refercence, reference is lost. cross HU reference is lost

Reason and Prerequisites

 Program error

Solution

Implement the correction instruction or the equivalent support package.

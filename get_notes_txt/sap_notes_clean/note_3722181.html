Symptom

Outbound payloads are changing fields to unexpected values at the outbound payload, while I can see the correct value on the preprocess(inbound) payload.

Environment

SAP Integration Suite, managed gateway for spend management and SAP Business Network

Reproducing the Issue

  1. Create a transaction on the SAP Ariba side.
  2. Forward it to the SAP Integration Suite, managed gateway for spend management and SAP Business Network layer.
  3. Verify the outbound file/check values on target system

Cause

This behavior may be caused due to a Custom Mapping at the SAP Integration Suite, managed gateway for spend management and SAP Business Network layer.

Resolution

To check for the existence of a Custom Mapping within you environment you can log into your SAP Integration Suite, managed gateway for spend management and SAP Business Network and follow the path:

  1. Go to: My configurations > Mappings.
  2. (Look for the affected document type).

In the path mentioned above you will be able to check for the existence of the custom mapping(Custom map column) as well as which environments it affects(SystemID column)

In case you are looking to check the mapping behavior with an existing example, you can use the preprocess(inbound) file and copy its content the the Document Validator Tab.

  • Document validator > Copy inbound payload to the source document box > Test.

Following these steps will show every action taken for the mapping of the referenced transaction

After the test is ran, you will be able to see the result list below the source document.

Looking for the step of the custom mapping transformation, there will be a magnifying glass on the step summary column, if the Custom mapping transformation line is not found, it indicates that this payload is not affected by a custom mapping.

When clicking on the magnifying glass you should look for the fields currently being affected on your scenario.
The list on the left side is the mapping of the transaction according to the standard mapping, and on the right side is according to the Custom Mapping currently affecting the document type.
and all values colored are being modified being red the old value(Left side) and green the new value(Right side).

After this validation you can be sure if the unexpected changes are due to the custom mapping, or if the issue is not related to it.

If the Custom Mapping is changing the values, Please connect with your consulting partner to check on the custom logic within the mapping.
If this is an issue that is occurring with the standard mapping, please raise a case to the SAP Integration Suite, managed gateway for spend management and SAP Business Network support.

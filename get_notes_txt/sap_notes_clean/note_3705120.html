Symptom

  • Only the first stage needs to be relevant for shipment cost calculation, but all stages are marked as relevant.

Environment

  • SAP ERP
  • SAP ERP Central Component
  • SAP R/3
  • SAP R/3 Enterprise
  • SAP enhancement package for SAP CRM, version for SAP HANA
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA
  • SAP S/4HANA Cloud Private Edition

Resolution

  1. Run Transaction SPRO with the following path and ensure the relevant costing indicator is set at the desired level (e.g., header, legL

    SPRO> Logistics Execution -> Transportation -> Shipment Costs -> Shipment Cost Document -> Shipment Cost Relevance and Default for Shipment Cost Type. 

  2. In the shipment type definition, define route stages and mark the first stage as relevant for shipment costing. Subsequent stages can be marked for informational purposes or as non-relevant for cost calculation.

  3. Define condition types to capture data from the first stage (e.g., distance, weight, volume).Use the condition technique to set the calculation basis. Create access sequences to find relevant condition records for the first stage's parameters (e.g., planning point, carrier, region/tariff zone).When creating the shipment cost document, ensure the system proposes costs based on these settings for the first stage.

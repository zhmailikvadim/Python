Symptom

systemReplicationStatus.py shows "CONNECTION TIMEOUT" and "Log shipping timeout occurred" for the secondary site. SAP provides dedicated troubleshooting guidance for SAP HANA system replication communication problems.

Environment

SAP HANA DB Version 2.0

Reproducing the Issue

Cause

If "No space left on device (OS error 28)" is observed on the secondary host in SAP HANA trace context, operating system space exhaustion on the secondary site must be considered. SAP documentation for SAP HANA system replication includes secondary-site space problems as a possible cause of log shipping timeout situations.

Resolution

Check the affected secondary host for operating system space exhaustion and free sufficient space on the filesystem used by the impacted SAP HANA path reported in the trace. After the secondary SAP HANA instance is available again, verify whether system replication communication is restored. This article only covers the scenario in which the secondary host shows OS error 28 together with the system replication timeout symptom.

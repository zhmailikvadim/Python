Symptom

You are using the API interface for consumption by production material request instead of the standard UI interfaces. You create your own function which implements the example code that can be found in the documentation of interface /SCWM/IF_API_MFG_CONSUMPTION with or without your extensions. If you are using this, you can face with the following issues:

-The GI document flow link is not written to the PMR item

-Due to the missing document flow entry, the consumption reversal is not possible

-Due to the missing document flow entry, the double consumption is possible

This example code contains a part which must be owerwritten to avoid this kind of faults as the correction -in the solution part- showes you.

Other Terms

PMR, consumption, API, API interface, production material request, Production Material Request

Reason and Prerequisites

The sample code for usage of interface contains incomplete part.

Solution

Original code part in the example documentation:

lo_mfg_cons->post_consumption( EXPORTING iv_whno       = p_whno

                                          iv_mfg_order  = p_mo

                                          it_items_cons = lt_item_cons

                                IMPORTING eo_message    = lo_message

                                          ev_gmdoc      = lv_gmdoc ).

* add messages if any

IF lo_message IS BOUND.

  IF lo_message->check( ) = abap_true.

    lv_error_occured = abap_true.

  ENDIF.

  lo_message->get_messages( IMPORTING et_bapiret = lt_return ).

  APPEND LINES OF lt_return TO lt_return_all.

ENDIF.

* did errors occurs? Or is this only a testrun? if no, then save

IF lv_error_occured = abap_false AND

   p_test           = abap_false.

  CLEAR lo_message.

* call SAVE and also immediately request to do the commit work
* (in case of errors * during save this .
* will then also do a rollback and CLEANUP).

  lo_mfg_cons->save( EXPORTING iv_commit  = abap_true

                     IMPORTING eo_message = lo_message ).

* add messages

  IF lo_message IS BOUND.

    IF lo_message->check( ) = abap_true.

      lv_error_occured = abap_true.

    ENDIF.

    lo_message->get_messages( IMPORTING et_bapiret = lt_return ).

    APPEND LINES OF lt_return TO lt_return_all.

  ENDIF.

ELSE.

* if errors during consumption API call or a testrun, rollback and clear buffers

  ROLLBACK WORK.

  /scwm/cl_tm=>cleanup( ).

  IF lv_error_occured = abap_true.

    CLEAR lv_gmdoc.

  ENDIF.

ENDIF.

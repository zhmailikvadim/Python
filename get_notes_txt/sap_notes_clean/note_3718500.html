Symptom

You are using Composable Storefront. After checking Dynatrace you can notice that for nginx the deep monitoring was not activated (Activation of deep monitoring was unsuccessful). In the Services section of Dynatrace you can see that nginx was Stopped:

Environment

SAP Commerce Cloud, Composable Storefront

Dynatrace

Reproducing the Issue

Check the Dynatrace related to nginx monitoring:

  • Infrastructure Observability -> Technologies & Processes -> Nginx -> nginx jsapps-* -> pick one of the processes
  • Application Observability -> Services -> Apply filter Technology = Nginx -> pick your service -> click on the Nginx tile

Cause

If you follow KBA 3575589 - How to check nginx version on js apps you can notice that currently nginx uses 1.28.2 version which is the latest stable version available. According to Dynatrace docs related to NGINX supported versions this version is not supported at the moment.

Resolution

This version of nginx is not supported at the moment and the monitoring is not available. However, this does not mean that your application is not working as expected. On the contrary, it works as it was before.

Keywords

spartacus, composable storefront, nginx, activation of deep monitoring was unsuccessful

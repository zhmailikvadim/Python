Symptom

  1. There is the need to influence batch status (Restricted, Unrestricted) and stock types (Unrestricted use, Blocked, and so on) during usage decision (UD) of an inspection lot.
    OR
    There is confusion why a specific batch status or stock type was used during usage decision of an inspection lot.
  2. Warning message is raised when the usage decision is posted for a material with active batch status management:
    "QV047 - Change batch status to -unrestricted- based on the usage decision"
    OR
    "QV048 - Change batch status to -restricted- based on the usage decision    "
  3. Error occurs when the usage decision is posted for a material with active batch status management:
    "QV046 - The UD valuation and the batch status deviate from one another"

Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.

Environment

  • SAP ERP Central Component
  • SAP ERP
  • SAP Enhancement package for SAP ERP
  • SAP S/4HANA, on-premise
  • SAP S/4HANA Cloud Private Edition

Resolution

1.) How to influence batch status and stock type during usage decision?

Following customizing path is relevant:

IMG(transaction QCC0 or SPRO)
>Quality Management
>>Quality Inspection
>>>Inspection Lot Completion
>>>>Edit Selected Sets for Usage Decisions

After selecting the relevant plant and selected set combination, it is possible to define UD codes with stock posting possibilities. For more information, see KBAs:

3586353 - In QA11 Proposal button does not work as expected.

2464838 - Stock was not posted automatically after usage decision is made in QA11.

This way a stock posting and batch status change can be triggered automatically.

***

User can also define

  • the stock posting and
  • the batch status

manually during the UD on the tab 'Inspection Lot Stock', by

  • filling the specific stock field and
  • clicking the 'Batch Status' and adjusting the status.

2.) Warnings:

"QV047 - Change batch status to -unrestricted- based on the usage decision"

"Diagnosis
The current batch status is to be changed from "not released" to "released", since the usage decision (batch status proposal) calls for this.
Procedure
If you do not carry out the change to the batch status and want to change the usage decision code instead, select Cancel, and answer the question on teh safety popup with "No". You can now change the usage decision code and save again.
If the change to the batch status is OK, select Enter."

"QV048 - Change batch status to -restricted- based on the usage decision"

"Diagnosis
The current batch status is to be changed from "released" to "not released", since the usage decision (batch status proposal) calls for this.

Procedure
If you do not carry out the change to the batch status and want to change the usage decision code instead, select Cancel, and answer the question on the safety popup with "No". You can now change the usage decision code and save again.

If the change to the batch status is OK, select Enter."

***

Such warnings can occur, when current batch status (visible in transaction MSC3N) differs from the determined batch status during the usage decision. User needs to decide about the next step, as described in the warning messages. Relevant code section to be found in include MQEVAF30, form OK_FCODE_BUCHEN.

3.) Error: "QV046 - The UD valuation and the batch status deviate from one another"
"Diagnosis
Deviating valuations are not allowed for this material.
One of the following deviations has occurred:
- Accepted valuation for the usage decision when the batch status is Not released
or
- Rejected valuation for the usage decision when the batch status is Released
Procedure
Select another usage decision code, or change the batch status."

Relevant SAP Note:

211089 - Change in batch status in usage decision.

"...
The system behaviour should ensure that, in certain circumstances (see below), the (accidental) release of the batch despite the rejection of the usage decision does not take place.
The check for the alternative status is executed if one of the following conditions is met:
- The inspection lot requires approval.
- The digital signature is required in the inspection lot in results recording.
- The digital signature is required in the inspection lot when the usage decision is made.
- The digital signature is required when the physical-sample drawing is acknowledged (as of Release 4.5B).
- The material requires an approved batch record.
The check is carried out only if batch state management is active for the plant in question.
..."

This document also provides a modification to avoid the error, if needed. Relevant code section to be found in include MQEVAF20, form ABWEICHENDER_CHARGENZUSTAND.

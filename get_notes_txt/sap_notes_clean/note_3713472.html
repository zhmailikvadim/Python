Symptom

In SAP Manufacturing Execution where increasing the database's decimal precision beyond six fractional digits (e.g., to 12 digits) causes a "Rounding Necessary" exception during data collection with formula parameters. Although high-precision values are successfully written to the database, attempts to read them back result in errors in POD.

Other Terms

SAP ME, DC500

Reason and Prerequisites

Steps to reproduce:

1. Update the decimal precision to 12 digits for the DC-related tables.

2. Create a Data Collection (DC) with two numeric parameters P10, P20 and one formula parameter P30.

3. Configure parameter P30 with the following script:

delta = P20 - P10;
exit(delta.toPrecision(8));

4. Release an SFC and collect data in any POD with the following values:
P10 = 6.8E-12
P20 = 5.8E-10

5. Click the 'Save' button.

Actual Result: An internal error is thrown in the POD. In the default trace logs, the error RoundingNecessaryException is recorded.

Expected Result: The save operation completes successfully.

6. Open the Data Collection Result report and retrieve the data collected for the SFC.

Actual Result: An internal error is thrown.

Expected Result: The report displays the collected values correctly.

This behavior occurs because, after retrieving BigDecimal values from MS Server and the Hana database, the existing logic trims trailing zeros without considering that the value may be represented in scientific notation.

Solution

The solution is provided in the form of a code correction. The system will use the standard Java stripTrailingZeros() method of the BigDecimal class, which performs the same operation as the previously implemented logic and correctly handles scientific notation

Impact

The code change implemented to address the symptom may impact all HANA and SQL Server database  retrieval queries with Big Decimal columns.

Symptom

Questions include: availability of a native activation-link mechanism; whether the accounts.resetPassword flow can be reused for initial password setup and activation; best practices to create users without a password, send a link, and complete password definition and activation in one step.

Environment

  • SAP Customer Data Cloud
  • Core REST API & Server SDKs (JWT / PHP / Java)
  • UI Builder – Native screensets

Reproducing the Issue

Cause

Resolution

  1. Use Native Screen-Sets (NSS) to implement the password-related flow; the platform provides built-in “forgot password” capabilities that can be leveraged and tested through the UI Builder’s “Native screen-set” tab.
  2. Refer to: Native screensets (NSS).
  3. Refer to: accounts.getNativeScreenSet REST.
  4. In UI Builder, open the Native screen-set tab and test the available options to align the password setup and activation step with your flow.

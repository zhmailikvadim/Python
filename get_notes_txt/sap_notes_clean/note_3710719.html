Symptom

Relevant for VSS 4.0
You are creating, or editing an existing Order in Service Resource Scheduling - To Do Basket. In the Demands table, you open the search-help dialog for Preferred Resource.

  • Make Code nor Model Code was passed into Hierarchy ID value list filters.
  • In change mode – Workshop Location (Hierarchy ID) of an Order was not passed into Preferred Resource list filters.

Relevant for all versions
Functions defined in /DBME/WMA_MENU_CONF → To-Do Basket → Define functions, which have Selection Mode set to 'No order required',
are not executed in To Do Basket and result in error 'No orders found' in console.

This note contains the following versions of SAPUI5 applications:

  • VSSUI 300
    - To Do Basket (/DBME/WMF_TDO) version 1.7.23
  • VSSUI 400
    - To Do Basket (/DBME/WMF_TDO) version 1.8.14
    - UI5 Common Library (/DBME/CMF_LIB) version 1.5.14
    - SRS UI5 Library (/DBME/WMF_LIB) version 1.6.5
  • VSSUI 500
    - To Do Basket (/DBME/WMF_TDO) version 1.9.21
  • VSSUI 600
    - To Do Basket (/DBME/WMF_TDO) version 1.11.11

Other Terms

Srs, to do basket, tdo, vss, order

Solution

Implement the attached note.
Update application index for:

  • /DBME/WMF_TDO
  • /DBME/CMF_LIB
  • /DBME/WMF_LIB

as described in Note 2449635.

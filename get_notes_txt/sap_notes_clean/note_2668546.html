Symptom

No negative amount can be posted to a Journal Entry Account in some special localizations, e.g. Brazil (BR).

Reproducing the issue

Prerequisites:

You are working in a localization where you cannot post a negative amount to a Journal Entry Account, e.g. Brazil.

Scenario:

  1. In Administration → System Initialization → Company Details → Basic Initialization, set System Currency (e.g. Euro) to be different from Local Currency (e.g. Real).
  2. In Administration → System Initialization → Company Details → Basic Initialization, set Exchange Rate to 3,9857.
  3. Add one A/R Invoice which may create a Journal Entry posting with a negative amount.
  4. Open the Journal Entry created by the A/R Invoice, and check the amount posted to the account.

Expected behavior:

No negative amount is posted to the account.

Actual behavior:

There is a negative amount posted to the account.

Note:

This issue occurs on some customer databases only.

Solution

SAP intends to provide a patch or patches in order to solve the problem described. The section Reference to Related Notes below will list the specific patches once they become available. The corresponding Info file of the patches in SAP Service Marketplace will also show the SAP Note number. Be aware that these references can only be set at patch release date. SAP will deliver patches only for selected releases at its own discretion, based on the business impact and the complexity of the implementation.

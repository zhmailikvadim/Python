Symptom

In the process "MDC: Sending of Settlement-Relevant Change from Responsible SUP to DSO as SUP" (DE_B_MDC_SUP_SETTL_REL_RSP_SUP_DSO) for the MDC_SUP cluster, the process gets stuck in step 200 when master data is not complete. Step 200 is executed only once, so if the master data is incomplete, the process does not proceed even after the data is retrieved again in step 300.

This issue is relevant for the market role: SUP.

Other Terms

MDC_SUP, DE_B_MDC_SUP_SETTL_REL_RSP_SUP_DSO, MDC process stuck, master data not complete, 2310, Gas, settlement-relevant change, step 200, step 300

Reason and Prerequisites

The error occurred because step 200 is executed only once. When the system retrieves master data again in step 300, step 200 is not executed a second time, so the process cannot continue if the master data was incomplete initially.

Solution

Implement the corrections of this SAP Note or import the relevant support package.

After implementing this SAP Note, the system executes step 200 again after retrieving master data if it was incomplete in the initial check. The process then proceeds as expected. Activate the BC sets /US4G/3_0_SUP_SP07_NOTE_3717716 and /US4G/3_0_SUP_SP08_NOTE_3717716 as required.
You need to retest the process after implementing the SAP Note. You can verify the fix by checking that step 200 is executed a second time if the initial check for master data completeness fails, and the process continues as expected.

This SAP Notes is relevant for the following role: SUP

BC Set: /US4G/3_0_SUP_SP07_NOTE_3717716 to be activated

1. After implementing the note. Run transaction SCPR20.

2. Input BC set "/US4G/3_0_SUP_SP07_NOTE_3717716" .

3. Click "Activate" button or choose BC Set->Activate option, from Menu

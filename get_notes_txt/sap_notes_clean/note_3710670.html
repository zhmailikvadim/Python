Symptom

You confirm a warehouse task with standard exception code BIDF or BIDP to trigger the creation of an alternative picking WT. The WT is created correctly, but it is not assigned to the current WHO.

Reason and Prerequisites

This issue only occurs after implementing note 3546620. If you are using Process-Oriented Storage Control in this pick Warehouse Order with POSC, but the next process step does not change the destination bin of the pick WT.

Solution

Implement the Correction Instruction.

Symptom

Quick filter disappears after saving a custom filter in entity OWL.

Environment

  • SAP Sales Cloud Version 2
  • SAP Service Cloud Version 2
  • SAP Sales Cloud and SAP Service Cloud Version 2

Reproducing the Issue

  1. Navigate to any entity OWL.
  2. Select filter A.
  3. Verify that there are six quick filters available, for example.
  4. Save a custom filter B.
  5. Observe that now only five quick filters remain visible quick filter C is missing.

Cause

  • Filter A is a custom filter. It was created from standard parent filter D.
  • When a custom filter A is selected and a new filter B is created from it, the standard parent filter D is referenced instead of the custom filter A. As a result, the quick filters associated with the standard filter D are copied, and the quick filter C is not included.

Resolution

This is the standard system behaviour.

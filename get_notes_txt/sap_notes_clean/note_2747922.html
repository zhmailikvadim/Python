Symptom

This note contains below changes in System recommendations 720 SP08   Fiori UI:

1. In NoteOverview page, the Note Version column is now  added

2. Bugfixes : in the table title of NoteOverview sometimes incorrect "updated" count of notes is displayed.

3. NoteOverview Page cannot be opened, in browser's development trace there is error : "Error: sap.ui.core.ValueState.Warning is not a value of the enums sap.ui.core.ValueState."

4. App cannot be opened, in browser's development trace there is error : "TypeError: Cannot read property 'Systemprop' of undefined"

Other Terms

NoteOverview

Reason and Prerequisites

There are different causes for above mentioned issues：design and programming errors, restrictions of UI5 elements .

Solution

Implement  the  below manual activity to you 720 solution manager system to update BSP Application SM_CM_SYSREC to the version 1.8.11.

( The SysRec internal maintenance branch for SP08 has been moved from 1.9 --> 1.8, version 1.8.11 is the newest update of the old version 1.9.72) .

This note only fix different bugs in system recommendations 720 SP08 UI, it does not change the note calculation results. It  does not affect other solution manager modules, for example, ChaRM.

Note: if some user still see the old filter UI even after cleaning the browser cache, it could be that the app index is not up to date. For updating the app index the ABAP report /UI5/APP_INDEX_CALCULATE must run regularly in the backend. For detail please refer to note 2364579.

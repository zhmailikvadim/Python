Symptom

Error message encountered: "<ESM_Tenant>.crm.cloud.sap refused to connect."

Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.

Environment

  • SAP SuccessFactors Employee Central
    • SAP SuccessFactors Employee Central Enterprise Service Management

Reproducing the Issue

  1. Click on ESM - Get HR Support icon
  2. Observe the connection error

Cause

Configuration issue during the implementation on ESM side.

Resolution

  1. Access ESM through direct link.
  2. Navigate to Users and Control > Settings > Cross-Site Request Forgery (CSRF) Settings > Select "None".
  3. Add your SF API URL domain, for example "*.sapsf.eu" in the Settings > Content Security Policy > frame-ancestors and ensure it is enabled.

Note: The domain required in frame-ancestors may be different than above example, it depends on the DataCenter.

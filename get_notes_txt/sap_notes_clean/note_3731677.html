Symptom

  • Previous Payment Information Defaults for Rehired Users in the “Rehire with Old Employment” scenario during the Personal Data Collection step..
  • If the data is removed in Data Review Step, The payment information section on PDC does not Auto-Expand; Users must click “Add Bank Details”.

Environment

SAP SuccessFactors Onboarding

Reproducing the Issue

  1. Set Hire/Rehire Matching Configuration → “Default Compensation Information during Rehire” to “No”.
  2. Process a rehire using “Rehire with Old Employment”.
  3. On Rehire Data Review, prior payment info is removed.
  4. Proceed to the onboardee’s Personal Data Collection (PDC); observe the bank details section requires clicking “Add bank details” to open.

Cause

  • The Hire/Rehire Matching Configuration option applies only to the Rehire Data Review page and not to the Personal Data Collection (PDC) page.
  • As per entity merge logic for “rehire with old employment”, prior payment/compensation details are considered for New Hire Data Review (NHDR). If prior details are removed in Rehire Data Review, PDC will not auto-expand bank details.

Resolution

  1. Acknowledge that “Compensation Information during Rehire = No” only controls the Rehire Data Review page and does not change PDC behavior.
  2. For “rehire with old employment”, if prior payment info is absent after Rehire Data Review, instruct the onboardee to click “Add bank details” on PDC to enter new payment information (auto-expansion is not expected).
  3. There is currently no configuration to disable defaulting on PDC or to force auto-expansion for rehires.
  4. Optionally, consider passing initial defaults from Recruiting to pre-populate PDC where appropriate.

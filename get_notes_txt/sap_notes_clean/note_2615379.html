Symptom

Check Tool: Check that picklists have unique external codes (PicklistUniqueExternalCodes)

This check detects picklists that do not have unique external codes, the ones with duplicate external codes get returned with the number of times they are duplicated.

This article covers general information regarding this particular configuration check "PicklistUniqueExternalCodes"

Environment

Employee Central - 1802

Resolution

This new Check Tool (Check ID = PicklistUniqueExternalCodes) should be leveraged to validate picklists to ensure that there are no Picklist Options with duplicate External Codes (uniqueness check).

If the External Code is not unique, it can cause issues when (for example) changing Employee Status, setting an Event or Synchronizing data to the SuccessFactors Platform.

Check Result: The system will check the legacy Picklists table (not MDF picklists) and it will determine whether any picklist has values that have non-unique external_code values, and it will flag them in the Error result.

Check Solution: We do not recommend having two or more picklist items with the same external code. Each code should be unique. Please correct the picklist by assigning different external_codes to those picklist values.

Note: this only appears for customers who have not had there picklists migrated to MDF picklist.

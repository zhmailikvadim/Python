Symptom

When entering slaughter process data via RAF Slaughter Process Entry, a PDA record is created even though the corresponding slaughter data record is locked by another user. The PDA record should not be created and should instead be rolled back.

Other Terms

IS-MFM, MEAT, RAF Slaughter Process, PDA Record, /SPMEAT/SLDT, /SPDGS/BAPI_PDA_CREATE, SAP Meat and Fish Management by msg.

Reason and Prerequisites

A logic issue in Function Module /SPMEAT/DP_SLP_CHANGE leads to the creation of a PDA record even when the slaughter data record is locked. The recently introduced import parameter I_NO_COMMIT of function module /SPDGS/BAPI_PDA_CREATE is not used correctly, causing the commit to occur although the slaughter data update cannot proceed.

Solution

History:
SAP Note created on 29.01.2026.

Description of Correction:
The function module /SPMEAT/DP_SLP_CHANGE has been modified so that when calling /SPDGS/BAPI_PDA_CREATE, the import parameter I_NO_COMMIT is correctly passed as abap_true. This prevents the PDA record from being locked if the battle record is locked.

Functional Dependencies:
Note 3708724 must be installed.

Manual Changes:
No manual changes are required for end users.

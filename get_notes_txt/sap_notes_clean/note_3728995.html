Symptom

The by-product with the backflush issue method may have incorrect issued quantity due to rounding.

Reproducing the issue

Prerequisite:

In this sample scenario the system is set up with 3 decimals for quantities in the Administration → System Initialization → General Settings → Display tab.

Scenario:

  1. Go to Production → Bill of Materials and create a new bill of materials of type Production with the following inventory items :
    product item 'Parent', and two component items : 'By-Product' and 'Component'.

  2. Go to Production → Production Order to add a new order with the following parameters : 

  • Planned quantity of 'Parent' is 100,

  • Planned Qty of 'Component' is 1.005; Issue Method : Backflush; Base Qty : 0.01; Base Ratio 201/20000,

  • Planned Qty of 'By-Product' is -1.005; Issue Method : Backflush; Base Qty : - 0.01; Base Ratio -201/20000.

  1. Release the Production Order and then add two Receipts from Production, each with a quantity 50 of the 'Parent'.

  2. Verify the Issued quantity of the 'By-Product'.

Expected behavior:

The Issued quantity of the 'By-Product' is -1.005, in line with the Planned Qty and the issued quantity of the 'Component'.

Actual behavior:

The Issued quantity of the By-Product is -1.006, which is not line with the Planned Qty -1.005.

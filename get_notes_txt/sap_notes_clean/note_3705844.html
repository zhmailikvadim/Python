Symptom

You have created a custom field for a particular entity, and when you try filtering by its values in the Data Export, it results in no records in the "Number of records that will be exported" field.

Environment

  • SAP Sales Cloud Version 2
  • SAP Service Cloud Version 2

Reproducing the Issue

1. Click on the User Icon.
2. Go to the Settings > All Settings. 
3. Go to the Extensibility Administration.
4. Select an Entity (e.g. Account, Case, Opportunity etc.).
5. Go to the Custom Fields tab.
6. Create a Custom Field.

7. Navigate to the Settings > All Settings. 
8. Go to the Data Import and Export > Import and Export Data.
9. Click on Export.
10. Select an Entity (e.g. Account, Case, Opportunity etc.).
11. on Apply Filter, click on the "Apply Filter" button.
12. Filter on the Custom Field.
13. This brings no result into the "Number of records that will be exported" field.

Cause

This might happen in two scenarios:

  1. You do not have any data maintained for this Custom Field.
  2. You have not enabled the "filterable" property for this Custom Field in the Extensibility Administration > Field Query Properties.

Resolution

  1. Ensure Data Exists: Verify that there is actual data stored in the custom field.

  2. You need to ensure that you have maintained the "filterable" property for this custom field by following these steps::

    1. Click on the User Icon.
    2. Go to Settings > All Settings.
    3. Navigate to Extensibility Administration.
    4. Select an Entity (e.g., Account, Case, Opportunity, etc.).
    5. Go to the Field Query Properties tab.
    6. Filter by the specific custom field.
    7. Enable the "filterable" property.

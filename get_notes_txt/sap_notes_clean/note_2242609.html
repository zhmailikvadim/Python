Symptom

Components selected via the Filter Table function in the Select Items to Copy window are not copied over to Issue for Production; only an empty row is added.

Reproducing the issue

Scenario:

  1. Create one production order (PO) with 6 components (Item No.1 - Item No.6). Make sure the Issue Method for all components is Manual.
  2. Duplicate the PO to create another 9 POs, and release all of them.
  3. Open an issue for production, choose Production Order, and select all the POs.
  4. In the Select Items to Copy window, right-click and choose Filter Table. Set conditions: Field - Item Number; Rule - In Range; Value - Item No.2 To Value Item No.6).
    Make sure the filter returns about 50 rows.
  5. Choose the LAST 30 rows out of the 50 filtered results and choose OK.

Actual Behavior:

In the Issue for Production window, only one blank row is added.

Expected Behavior:

All the selected items are brought to the Issue for Production row level.

Cause

Application Error.

Solution

SAP intends to provide a patch or patches in order to solve the problem described. The section Reference to Related Notes below will list the specific patches once they become available. The corresponding Info file of the patches in SAP Service Marketplace will also show the SAP Note number. Be aware that these references can only be set at patch release date. SAP will deliver patches only for selected releases at its own discretion, based on the business impact and the complexity of the implementation.

Workaround: Choose the items directly in the Select Items to Copy window, and do not use the Filter Table function.

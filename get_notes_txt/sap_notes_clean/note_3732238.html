Symptom

Security Audit Log data from table RSAU_BUF_DATA or RSAU_LOG need to archived and deleted. For that the corresponding actions were done.
After running these actions one or more of the following issues can be observed:

- The amount of data in the affected table will not be decreased 
- No archive session will be created for the archive runs
- Although the archive job finished successfully nothing happens
- Data which should be archived and deleted are still available when evaluating the SAL logs via RSAU_READ_LOG or SM20 (as of SAP_BASIS 755)

For analyzing such issues from SAP side it is necessary to provide required information. 
The question is what information are required ?

Environment

SAP BASIS 750 SP03 or higher

Resolution

In case the issue need to be checked and analyzed from SAP side please provide the following details:

1) SAP_BASIS release and support package used for the affected system

2) Details about the SAL kernel parameter settings used for the Security Audit Log --> RSAU_CONFIG or SM19 (as of SAP Basis 755) --> section "Parameter"

3) The number of entries in table RSAU_BUF_DATA and RSAU_LOG (as of SAP BASIS 755)

    --> in systems with SAP BASIS < 755 it can be used i.e. the "Number of entries" button in SE16 or DB related tools 
    --> in systems with SAP BASIS 755 or higher it can also be used i.e. T-Code RSAU_SELF_CHECK or SM19/RSAU_CONFIG --> menu "Goto" --> "Check runtime environment"

          >> with the selection criterias "Audit Events"
                                                            and
                                                      "Statistic Event Volume Overview"
                                                            and
                                                      "Check Data in RSAU_BUF_DATA"
                                                      "Check Data in RSAU_LOG"

4) Details from the selection screen of the archiving write report which shows how the SAL data were archived exactly

5) Information about the status of the jobs executed for archiving AND deleting SAL logs from the database

6) Details from job logs of the jobs executed for archiving AND deleting SAL logs from the database

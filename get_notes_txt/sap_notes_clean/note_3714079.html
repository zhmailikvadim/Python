Symptom

After you calculate a result-independent condition in a Risk Manager account with different original currencies and account currencies, the user changes entries that affect the change of the exchange rate. However, you expect the program to not update the amount in the original currency when you save the account.

Other Terms

/MSG/H_FINA1

/MSG/RBU-AB_BETR

/MSG/RP_BCABRREC-DIFFBU

COGS

Reason and Prerequisites

Prerequisite:  In the IMG activity "Edit Defaults for Accounting (Calculation Fields)", the indicator for posting the differences is activated.

Reason:

The program determines the differences between the actual amount and the target amount in the original currency and account currency of the condition posting. The exchange rate change influences the update of the amount in the settlement currency. The difference amount in original currency is zero. The program filters out condition postings with the difference amount equal to zero in the original currency.

Solution

History:

February 10, 2026: Newly created

Troubleshooting Description:

The program filters out only those condition postings that have difference amounts equal to zero in original and account currencies.

Symptom

Within SAP Build Process Automation (SBPA), want to get processor of an Approval form (which only get populated at runtime after task get approved/rejected) and use it in other activity such as Email, but do not know how to achieve it.

"Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental."

Environment

SAP Build Process Automation - SBPA

Resolution

Currently no such out of box setting exists. Can achieve it by using Script Task to create and populate a custom variable. Refer to following details.

1. Create a Custom Variable
Navigate to Process Details -> Variables -> Custom Variables -> Configure and add a new variable of type "String" (In the following image, the variable name is given as "processor").

2. Add a Script Task before the activity (such as Email)  that want to use custom variable defined in previous step.
Open the Script Task editor. Will see a comment block with general usage instructions. Either keep it or remove it.

3. Set the Variable Value
Assign the Approval Form's processor value to custom variable using format as:
$.context.custom.processor = $.usertasks.form_approveRejectLeaveForm_1.last.processor;
Note:
(1) Replace "form_approveRejectLeaveForm_1" with your form correct identifier. To find it, use the left-side menu to access a form field and copy the identifier from the resulting line.
(2) Replace "processor" with your defined custom variable identifier. 
(3) Use "Validate Script" and "Apply" to verify the script is error-free and apply it.

4. Use the Variable in other activity such as Email.
Reference the custom variable (processor) in the Email's body.

5. Repeat for other branches as needed. Release and deploy SBPA project and validate at runtime.
If all are configured correctly, should receive Mail as following where the actually approver's ID get retrieved fine.

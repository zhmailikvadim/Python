Symptom

  • When using HANA system versioned tables via the .hdbsystemversioning plugin of the hdi-deployer, the new database service includes all columns in the plain SQL insert statement, even though they are not provided as data for the insert.
  • Getting the error message: "INSERT, UPDATE and UPSERT are disallowed on the generated field: cannot insert into generated field SYSTEM_VALID_FROM"

Environment

SAP Cloud Application Programming Model

Reproducing the Issue

    'cap-js/hana' produces the following query:

    INSERT INTO xxx (ID, SYSTEM_VALID_FROM, SYSTEM_VALID_TO)

    The query fails with the error message:

    INSERT, UPDATE and UPSERT are disallowed on the generated field: cannot insert into generated field SYSTEM_VALID_FROM.

    Cause

    The valid from and valid to columns are generated fields in HANA system versioned tables and must not be included in the SQL insert statement.

    Resolution

    Add the '@hana.systemversioned' annotation to the columns that need to be ignored during the insert operation.

    Refer to the below example:

    ---------------------------------

    using {edge.hana.versioning as versioning} from '../../db/hana/versioning';

    annotate versioning.versioned with @sql.append: 'WITH SYSTEM VERSIONING HISTORY TABLE edge_hana_versioning_versioned_history';

    annotate versioning.versioned with {
      validFrom @(
        hana.systemversioned,
        sql.append: 'NOT NULL GENERATED ALWAYS AS ROW START'
      );
      validTo   @(
        hana.systemversioned,
        sql.append: 'NOT NULL GENERATED ALWAYS AS ROW END'
      );
      // IMPORTANT: the `data` column must always be the last defined column
      // As the @sql.append is doing a classic bit of SQL injection
      // to include the required period definition for the history table
      data      @(sql.append: ', PERIOD FOR SYSTEM_TIME (validFrom, validTo)')
    }

    --------------------------------------

Symptom

Error "You are not allowed to delete auto-created entries." (Message No. EAM_USER_STATUS021) is raised when attempting to delete a phase control key from the customizing.

Environment

  • SAP R/3
  • SAP R/3 Enterprise
  • SAP ERP Central Component
  • SAP ERP
  • SAP Enhancement package for SAP ERP
  • SAP Enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA, on-premise
  • SAP S/4HANA Cloud Private Edition

Reproducing the Issue

  1. Open SPRO -> Plant Maintenance and Customer Service -> Maintenance and Service Processing -> Fiori Apps for Maintenance Processing -> General Settings -> Define Phase Control Code for Maintenance Orders.
  2. Select an order header type, then navigate to "Define Phase Control Codes".
  3. Delete a phase control key.
  4. Observe error: "You are not allowed to delete auto-created entries." (Message No. EAM_USER_STATUS021).

Cause

Standard behavior: the current customization framework does not allow deletion of auto-created entries (phase control codes/keys) once saved.

Resolution

Using the Validity setting in "Define Phase Control Codes" to manage unused phase control codes instead of deleting the records.

Choose one of the following values to define the validity of the phase control code.

  • Valid: The entry is valid and is displayed in input help. The user can select this valid entry.
  • Deprecated: The entry will no longer be used in the future but is still displayed in input help. The system displays a warning if the user selects this entry in input help.
  • No longer valid: The entry is invalid and is no longer displayed in input help.

If the validity column not available in "Define Phase Control Codes", implement SAP Note 3305405 and 3696509.

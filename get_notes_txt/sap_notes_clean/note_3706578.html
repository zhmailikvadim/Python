Symptom

Error: Account information not recognized: Could not reach CMS '<hostname>:6400'. The CMS on machine '<hostname>' was stopped due to a critical error. (FWM 20030) when logging on to the Central Management Console

Environment

  • SAP BusinessObjects Business Intelligence(BI) platform 4.x
  • Microsoft Windows Server operating system

Reproducing the Issue

  1.  Login to the Central Management Console(CMC) as Administrator user and see the below error

    Account information not recognized: Could not reach CMS '<hostname>:6400'. The CMS on machine '<hostname>' was stopped due to a critical error. (FWM 20030)

  2.  Open Central Configuration Manager (CCM) and able to login to Manage Servers . 


Cause

BOE deployment issue.

Resolution

  1. Stop tomcat
  2. Move BOE folder from install dir>tomcat>webapps to any other location 
  3. Navigate <BusinessObjects Install Dir>\SAP BusinessObjects Enterprise XI 4.0\wdeploy
  4. Open Command prompt
  5. run : Wdeploy tomcat9 -DAPP=BOE deploy
  6. Start tomcat

Symptom

You experience slow system performance in SAP HANA when a high number of statements execute concurrently. The majority of these statements are mapped to the same aggregated workload class. A significant number of threads show contention on an "unnamed Mutex" (as per "M_DEV_CONTEXTS"."WAIT_SYNC_NAME")  with a callstack similar to:

Other Terms

Mutex contention, Workload context, Thread synchronization, Performance bottleneck, High concurrency

Reason and Prerequisites

Issue number 334120

Reason:

Due to a programming error in SAP HANA, when many threads execute statements concurrently and these statements are mapped to the same aggregated workload class with total memory limits configured, the threads are forced to wait for each other during workload association. This unnecessary serialization causes significant performance degradation in high-concurrency scenarios.

Affected Releases:

  • SAP HANA 2:
    • Revisions of SPS00 - SPS06
    • Revisions <= 079.02 (SPS07)
    • Revisions <= 082.00 (SPS08)

Prerequisites:

  • Many threads execute statements concurrently
  • Statements are mapped to the same aggregated workload class
  • Total memory limits are configured for the workload class. See Properties for Workload Classes and Mappings

Solution

Apply one of the following SAP HANA Revisions:

  • SAP HANA 2:
    • Revisions >= 079.03 (SPS07)
    • Revisions >= 083.00 (SPS08)
    • or higher
  • or higher

Workaround:

You may change the workload class type by converting the aggregated workload class to individual workload classes. This reduces the contention as each statement gets its own context.

Symptom

Data is displayed using the ALV Hierarchical Sequential Grid (HS_Grid). Totals are set for the data and the flag 'Display totals lines above the entries' is set in the display tab of the layout variant definition. This causes the total to be displayed above the corresponding rows of data instead of below.

However, if the layout variant is saved, the setting of the flag is not retained. Thus when the layout variant is used again, the totals appear below the data rows instead of above.

Other Terms

ALV; ABAP List Viewer; SAP List Viewer; Grid; List;

Hierarchical Sequential; HS-LIST; HS-GRID; Hierseq

Reason and Prerequisites

Reason

Grid and list displays use different flags to indicate 'Display totals lines above the entries'. Historically, the flag was saved for grid displays only. If an application requires totals above data rows for a list display, it must set the flag directly after which the settings is fixed and cannot be changed by the user.

HS-Grid allows the user to set 'Display totals lines above the entries' as for other grid displays. However, since HS-Grid is an extension of the old HS-List display, it uses the List version of the flag which is not saved.

Note that the flag has no effect on the old Hierarchical Sequential List (HS-List) display; due to the structure of the display, HS-List does not support totals above data rows.

Solution

From now on, the scenario described in the "Symptom" section no longer occurs.

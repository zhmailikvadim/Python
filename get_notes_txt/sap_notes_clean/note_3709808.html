Symptom

When accessing overview page of Integration & Exception Monitoring, "Either service ID or business service ID required" message is received.

Disclaimer: "Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental." 

Environment

SAP Cloud ALM 1.0

SAP Ariba

Reproducing the Issue

  1. Create OAuth credentials and download them from the SAP Ariba Developer Portal.
  2. Set up the endpoint for ARIBA test on SAP Cloud ALM.
  3. Login to SAP Cloud ALM
  4. Click on Integration & Exception Monitoring Overview page

Cause

Data collection was not enabled for a service or system.

Resolution

To activate the monitoring use cases firstly ensure that Integration Architect role has been assigned to configure events, and monitoring filters for managed components.

Then enable/disable the data collection for a service or system as required. To configure data collection:

Open the scope selector by selecting (Select a Scope).
Select Services and Systems in the Services/Systems dropdown.
Choose the services and systems that you want to monitor in the Services/Systems table and select Apply.
The app reloads with the selected scope.
Open the configuration pane by selecting (Configuration).
Navigate to Managed Components Services / System.
Set the data collection switch of the required managed components to ON.

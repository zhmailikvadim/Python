Symptom

In DMS work flow status shows in external link (eg. www.youtube.com) video upload and approval show as State Undefined. Below screen appears:

Environment

  • EP Release Independent
  • SAP NetWeaver
  • SAP Composition Environment

Resolution

  1. This is working as per the design. The approval process when enabled is only valid for actual content within KM. If you create a link (both internal or external), the link itself cannot be made part of the approval process. Only actual content, i.e. documents can be read by the process and return a workflow state. 
  2. Just to be clear, links cannot be included as part of the approval process, only the documents to which they are linked may be. External content outside of KM cannot be included as part of the approval process either. The KM approval process is exclusive to actual documents stored within KM. Links which cannot be approved or rejected will show 'state undefined'. This is correct.
  3. For example, if you create an internal link to a document within KM (an internal link) - you have the option to access either the 'details' or 'details for link'. The 'details for link will return 'state undefined' whereas the 'details' will show the current state of the actual document itself (if the approval process is enabled for it). For an external link such as www.youtube.com , the resource is not within KM so it cannot provide the 'details' and 'details for link' options. Instead, you can only see the details for the actual link. As think cannot be part of the approval process, it will not have a 'state' so it will return 'state undefined'.

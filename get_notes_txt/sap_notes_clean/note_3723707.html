Symptom

  • The error message "Entity ChangeAuditUI_mdfObject_VH is not found. Please contact your system administrator." appears when selecting external code on MDF Change Audit Reports.
  • The issue is fixed running "OData API Metadata Refresh".
  • Few days later, the same error is displayed, requiring to repeat the process often.

Environment

SAP SuccessFactors HCM Suite

Reproducing the Issue

  1. Login as an Admin with access to the Change Audit feature.
  2. Navigate to Change Audit Reports.
  3. Go to the "Create Business Data Report" or "Create Configuration Data Report" tab.
  4. Select "MDF Change History Data" or "Create Configuration Reports".
  5. In the pop-up window, select one object or the affected object.
  6. On External Code, click the link on the right to open the selection.
  7. Observe the error message: "Entity ChangeAuditUI_mdfObject_VH is not found. Please contact your system administrator."

Cause

  • Lacks of permission "Generate Change Audit Reports", generating the metadata without the correct entity in the cache, leading to the error.
  • When a user performs a refresh of OData V2 Metadata, the system checks whether the user has the permission "Generate Change Audit Reports" to decide whether to build the entity ChangeAuditUI_mdfObject_VH.

Resolution

  • The engineering team has identified and isolated the error. An enhancement code correction is currently in the deployment stage. The enhancement will ensures that the system no longer checks the "Generate Change Audit Reports" permission when building the ChangeAuditUI_mdfObject_VH entity during the OData V2 Metadata refresh, thereby preventing the generation of incorrect metadata.

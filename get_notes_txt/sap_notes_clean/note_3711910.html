Symptom

After configuring a deployment scheduled as described on the below SAP Help Portal page, auto deployment is not triggered in the respective managed systems.

 SAP Help Portal : Scheduling Deployments

Environment

SAP Cloud ALM 

Reproducing the Issue

  1. Create a deployment schedule in the Deployment application
  2. Check the status of imports in the managed system

Cause

The import use case is set to a lower frequency than the frequency of the deployment schedule. 

Resolution

As mentioned in the documentation, "you should schedule a higher frequency of the import use case on the managed system than on the deployment schedule in SAP Cloud ALM."

Symptom

  • SAP Build Work Zone system does not appear in the dropdown list during the setup process (Joule Booster - Step 5 Set Up Integrations).
  • The issue persists despite the SAP Build Work Zone system being set up in the subaccount.

*Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental. 

Environment

  • Joule
  • BTP

Reproducing the Issue

  1. Navigate to the subaccount.
  2. Run Joule Booster;
  3. Proceed to the configuration step 5 labeled “Set Up Integrations” under Joule Booster.
  4. Attempt to select the required system from the dropdown list.
  5. Observe that the desired SAP Build Work Zone system is not visible in the dropdown list.

Cause

  1. The SAP Build Work Zone service was not subscribed under the relevant subaccount.
  2. Only an instance of this service is not enough for this Integration, therefore the system will not appear under the options.

Resolution

  1. Navigate to the subaccount used for this Joule integration
  2. Go to Instances and Subscriptions.
  3. Click on "Create" (upper right button).
  4. Select the Service > SAP Build Work Zone, standard edition > Subscriptions - foundation.
  5. Click "Create".
  6. Refresh the page and rerun the setup process.
  7. Verify that the SAP Build Work Zone system is now visible in the dropdown list under the configuration step 5 - Set up Integrations > “Include Systems”.
  8. If unable to create the SAP Build Work Zone subscription, check the configurations under Entitlements.

Symptom

  • You want to import certificate into HANA File-Based PSE to build up trust relationships.
  • Trusted certificate is going to be expired or has been expired. You want import the renewd certificate.

"Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental."

Environment

  • SAP HANA Platform 1.0
  • SAP HANA Platform 2.0

Resolution

You may use SAP Web Dispatcher administration tool or the SAPGENPSE tool to import the certificate into PSE.

  • With SAP Web Dispatcher administration tool
    1. Access to "PSE Management" from Web Dispatcher Administration tools at the following URL: 
      http://<WebServerHost>:80<SAPHANAinstance>/sap/hana/xs/wdisp/admin
      The use logged on should be granted with role "sap.hana.xs.admin.roles::TrustStoreAdministrator"
    2. Select on the PSE file to be managed, e.g. SAPSSLC.pse.
    3. Click on "Import Certificate"
    4. Copy and paste the certificate to be trusted. Then click on "Import"
      e.g. 
      -----BEGIN CERTIFICATE-----
      MIIF+jCCA+KgAwIBAgIQZuygcxRot6VcDQ+V6GVyYDANBgkqhkiG9w0BAQsFADBE
      ...
      -----END CERTIFICATE-----
    5. For SAPSSLS.pse, you may restart webdispatcher service to let it take effect.
      e.g. stop it from HANA studio and let it to be restarted automatically
       
      For others, you can run below command from HANA server with <sid>adm user to make it take effect. Or you can restart the entire HANA database.
      hdbnsutil -reconfig 
  • With SAPGENPSE tool
    1. Log on as <sid>adm user to your HANA server
    2. go to $SECUDIR folder with command: 
      cd $SECUDIR 
    3. Upload certificate file (e.g.trust.crt) to be trusted, which contains text as in step4 above, into $SECUDIR folder and ensure <sid>adm user has access to it.
    4. Run following command to import it into PSE (e.g. SAPSSLC.pse).
      e.g.
      sapgenpse maintain_pk -a trust.crt -p SAPSSLC.pse
    5. You may verify the trusted certificate list with following command.
      e.g. 
      sapgenpse maintain_pk -p SAPSSLC.pse -l
    6. For SAPSSLS.pse, you may restart webdispatcher service to let it take effect.
      For others, you can run below command from HANA server with <sid>adm user to make it take effect. Or you can restart the entire HANA database.
      hdbnsutil -reconfig

Symptom

After using the 'Copy to new target' functionality and selecting the IBP Calculation Scenario for your task that has IBP as the target, your task returns the error:

"An error has occurred. Contact SAP Support and provide error ID ____"

Other Terms

Environment

SAP Cloud integration for data services - Server version 2303 HF1 - WebSocket RFC day 1 issue

Steps to reproduce the issue:

1. Select Project in the 'PROJECT' tab and select 'Create Task'.
2. Name the task and choose a template (for example, 'IBP_KF_Actuals'), then choose 'Next'.
3. Select any source ERP / ECC datastore then choose 'Next'.
4. Select any IBP datastore as the target, then choose 'Save', which creates a new task.
5. Choose the task and select 'Edit'.
6. Select the data flow object, and choose 'Actions' -> 'Copy to new target'.
7. Select 'Import a new target object', then choose 'Calculation Scenario Folder', select one entry, and choose 'Import and copy data flow'.
8. After the step above, the task cannot be used anymore; it always shows an error popup while editing.

Steps to avoid the issue:

1. Select Project in the 'PROJECT' tab and select 'Create Task'.
2. Name the task and choose a template (for example, 'IBP_KF_Actuals'), then choose 'Next'.
3. Select any source ERP / ECC datastore then choose 'Next'.
4. Select any IBP datastore as the target, then choose 'Save', which creates a new task.
5. Choose the task and select 'Edit'.
6. Select "Add Target Object".
7. Select 'Import a new target object', then choose 'Calculation Scenario Folder', select one entry, and choose 'Import and copy data flow'.
8. After the step above, you will see one new error popup with the validation blocking your action = "The table XXXXXXXX can be used only as a source. It cannot be used as a target".

As you can see in bold above, the issue is happening on step 6 when we follow different selections.

Reason and Prerequisites

The Calculation Scenarios cannot be used as Target for IBP datastores. Calculation scenarios can be used to retrieve data (Source) and not to load data (Target).

The validation is not done when we use the 'Copy to new target' and later 'Import a new target object'.

The correct validation is done when we use the "Add Target Object" and later 'Import a new target object'.

Solution

The correction for this server issue is delivered in SAP Cloud Integration for data services version 2306 (1.0.11.50).

Workaround:

Use the selection "Add Target Object" and later 'Import a new target object' to validate if your actions are allowed. Using these steps, the system checks that Calculation Scenarios are not used as the target.

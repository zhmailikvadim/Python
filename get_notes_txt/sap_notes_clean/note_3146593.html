Symptom

You get a bind permission error when you try to start the VDS on a port below 1024 in the main listener of IDM VDS on Linux

The following error is seen in the VDS log: 

"IDS deployment 'main_listener' (port 636) : Permission denied (Bind failed) (local port 0 to address 0.0.0.0, remote port 636 to address 0.0.0.0) 
 Exception: 
 {0}#1#java.lang.Exception: IDS deployment 'main_listener' (port 636) : Permission denied (Bind failed) (local port 0 to address 0.0.0.0, remote port 636 to address 0.0.0.0)
 at com.sap.idm.vds.protocols.ldap.MxLdapListener.execute(MxLdapListener.java:206)
 at com.sap.idm.vds.MVDServer.Serv(MVDServer.java:245)"

Environment

SAP Identity Management 8.0

Resolution

To achieve lower ports in the main listener of the VDS on Linux you need to do one of the following:

  1. Give sudo permission to the sapjvm java binary for the idm user

  2.  Set the CAP_NET_BIND_SERVICE capability to the sapjvm java binary

  3. Use a tool like authbind to grant this access.[]

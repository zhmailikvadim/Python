Symptom

Error on FC HTML5 web interface when trying to access using SAML:

"Could not validate SAML assertion"

In the logs of the html5, this error message appears:
 [thread 78] ERROR Web HTML5 - Cannot redirect after HTTP headers have been sent.

Environment

SAP Financial Consolidation

Reproducing the Issue

1. Configure SAML for FC HTML5 web client

2. Try to connect to the HTML5 web client using SAML

Resolution

Since the issue is with the SAML assertion, this should be related to the IDP configuration on the environment where SAML is configured.

All the mandatory steps described in the below documentation should be performed on BI server side:

https://help.sap.com/docs/SAP_BUSINESSOBJECTS_FINANCIAL_CONSOLIDATION/c01ad0fe4a7145e7a32aa3ffad52a4e3/3a85eb89770b43a1a267c06d152d0839.html

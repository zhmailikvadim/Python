Symptom

When you run the transaction SQ02 and perform syntax check, you get syntax error "The data object 'AD01WIP_DTL_RPT' does not have a component called '/CBAD/BILL_WBS'.".

Other Terms

SQ02, PS_REV_WIP_DETAIL, PS_REV_WIP_REVENUE, PS_REV_WIP_DETAIL_REV

Reason and Prerequisites

Data consistency issue.

Solution

To fix the issue, please follow these steps:

  1. Start transaction SQ02. Enter InfoSet PS_REV_WIP_DETAIL and press the change button. If a popup 'InfoSet not current' appears, press the 'Yes' button to continue.
  2. Press the button 'Data Reading Program' and delete or comment the following source code lines 27 and 28:

27  ranges  /CBAD/BI  for  AD01WIP_DTL_RPT-/CBAD/BILL_WBS.

28  ranges  /CBAD/OR  for  AD01WIP_DTL_RPT-/CBAD/ORIG_OBJNR.

  1. Expand the datafields for structure AD01WIP_DTL_RPT.
  2. Scroll down and select the data fields  AD01WIP_DTL_RPT-/CBAD/ORIG_OBJNR  and AD01WIP_DTL_RPT-/CBAD/BILL_WBS.
  3. Delete these two datafields from the field group (right mouse button).
  4. If you have an error “Text field for field TEXT_AD01WIP_DTL_RPT__CBAD_000 still used in queries” during deletion, navigate back to InfoSet:Initial Screen and run ‘Query Directory’ by clicking ‘Goto-> Query Directory’. Run it, select lines in the list and press Delete Selected Queries. Now repeat steps 3 to 5 and afterwards continue with step 8.
  5. If, when you try to navigate or try to change anything in SQ02, you get a dump (Runtime Errors CALL_FUNCTION_NOT_FOUND, Except. CX_SY_DYN_CALL_ILLEGAL_FUNC, ABAP Program CL_TEXT_IDENTIFIER============CP), follow the instructions contained in note 2403429. Now repeat steps 3 to 5 and afterwards continue with step 8.
  6. Press the generate button to generate the InfoSet.

Please repeat steps for InfoSets PS_REV_WIP_REVENUE and PS_REV_WIP_DETAIL_REV.

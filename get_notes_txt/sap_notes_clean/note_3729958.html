Symptom

  • A custom Data Product was created in SAP Datasphere and shared to SAP Business Data Cloud. It appears active in BDC Catalog but the sharing to SAP Databricks was failed with error message.
  • UI error: 

    Creating a shared object in databricks-xxx. We will notify you when the process is complete.

    An error occurred and the data product has not been shared. Please try again later.

    Could not create share xxx for data product xxx
    The data product could not be shared. Check the data product details page. Correlation ID: xxx
     
  • Backend error: errorCode: 404; message: "No matching HDLFS Federation found for ApiResource ... under TenantMappingFederation ..."

Environment

  • SAP Databricks
  • SAP Business Data Cloud
  • SAP Datasphere

Reproducing the Issue

  1. Open the BDC catalog and locate the custom HDLFS data product that is active.
  2. Choose Share and select the target SAP Databricks system and workspace.
  3. Provide a share name and start the sharing process.
  4. Observe that an error is returned and the share is not created.

Resolution

  1. Remove the SAP Databricks tenant from the Formation in SAP4ME / BDC solution - SAP Help Portal Documentation: Creating SAP Business Data Cloud Formations.
  2. Re-add the same SAP Databricks tenant to the Formation to initiate a full formation synchronization.
  3. Wait until the Formation state is Ready.
  4. Return to the BDC Catalog and share the custom Data Product to the target SAP Databricks workspace again.

Symptom

  • The configuration tool fails to open in a Linux environment.
  • Error message: "Config tool stopped due to fatal error. See trace file for more details."
  • Additional error message: "java.awt.HeadlessException: Can't connect to X11 window server using ':0' as the value of the DISPLAY variable; you may consider checking your DISPLAY environment variable."

Environment

Linux

Reproducing the Issue

  1. Attempt to open the configuration tool in a Linux environment.
  2. Check the DISPLAY environment variable, which is either unset or incorrectly set.
  3. Execute the command `setenv DISPLAY :0` in the Linux shell, which does not resolve the issue.
  4. Attempt to remove the system VM parameter using the console configuration tool (./consoleconfig.sh), but the option to remove the parameter is unavailable.
  5. Verify the sshd_config file settings, confirming that X11Forwarding and AllowTcpForwarding are set to "yes".
  6. Attempt to use X11 forwarding via SSH, but the session indicates that X11 forwarding is disabled or not supported by the server.

Cause

  • The X11 forwarding feature is not properly configured or supported on the server.
  • The required X11-related packages, such as `xorg-x11-xauth`, are missing from the server.

Resolution

1. Verify that X11 forwarding is enabled in the SSH client settings:

  • Right-click on the session in the SSH client.
  • Select "Edit session".
  • Navigate to "Advanced SSH Settings".
  • Ensure that "X11-Forwarding" is selected.

2. Recreate the session profile in the SSH client and confirm that X11 forwarding is enabled.

3. Test X11 forwarding by performing the following steps:

  • Connect to the server using a verbose SSH connection: `ssh -v @`.
  • Check the DISPLAY variable after login: `echo $DISPLAY`.
  • Attempt to start a simple X11 application, such as `xclock`. Install `xclock` if it is missing.

4. Install the required X11-related package `xorg-x11-xauth` on the server. Refer to the documentation for installation instructions:

  • [Configure SSH with X11 Forwarding](https://docs.oracle.com/cd/E89497_01/html/rpm_80_installation/GUID-99715D5C-8549-40F2-B88E-D2D560A46085.htm#Configure-SSH-with-X11-Forwarding).

5. After ensuring X11 forwarding is properly configured, retry opening the configuration tool.

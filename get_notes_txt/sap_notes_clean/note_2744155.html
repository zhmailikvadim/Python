Symptom

You are using Focused Build. In My Defects, you have the following behavior:

1.    The Developer is ready with the fix – the action of Defect Correction (S1TM) “Retest with Transport” starts the transport and set the status of the Defect (S1DM) to “Solution Proposal”

2.    The fix is not successful – the tester sets the status of the Defect (S1DM) to “In Process”

3.    The status of the Defect Correction (S1TM) is still in the status “Transport to Retest”, so the developer is not informed that the fix is not successful.

With this SAP Note:

When the Defect is set to “In Process”, the Defect Correction should automatically go to “Being Corrected”.

Other Terms

Focused Build, Defect, Defect Correction, Status

Reason and Prerequisites

 Missing customizing.

Solution

Please perform the manual activity.

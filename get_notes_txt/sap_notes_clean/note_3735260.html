Symptom

  • Customer is facing an issue with transaction FAGLF101 used for regrouping AP & AR open line items during month-end activities.
  • Posting list displays open vendor line items, but during batch input session processing (SM35), at least one line item is not posted.

Environment

  • SAP S/4HANA
  • SAP S/4HANA Finance

Reproducing the Issue

  1. Start transaction FAGLF101 and select the variant used for AP/AR regrouping .
  2. Run FAGLF101 to generate the posting list and create the batch input session.
  3. Open SM35 and process the created session (foreground or background).
  4. Verify the resulting postings using FB03.

Cause

Missing coding corrections.

Resolution

Implement SAP Note 2894033 and retest FAGLF101 end-to-end (generate posting list, process SM35, verify in FB03).

 SAP note 2894033 - FAGL_CL_REGROUP: Wrong display when posting overflows

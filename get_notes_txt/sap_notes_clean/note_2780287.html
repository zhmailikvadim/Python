Symptom

For Commodity Order Fill upload using FIX(Financial Information eXchange protocol) Message, fill is created and processed even if Order Ticks are in decimal. Order ticks are incorrectly rounded too, during Fill processing. Order Ticks values are passed via Tag 44 (Price) of incoming FIX string.

Other Terms

Fill Upload, Inbound or Incoming FIX Message, Tag 44, Order Request Ticks, CL_CMMFDAI_FIX_MSG_PROCESSOR, SET_FIX_PRICE_TICKS

Reason and Prerequisites

Only relevant for TAS (Trade At Settlement) Commodity Order

Solution

Exception is added to the base Processor class 'CL_CMMFDAI_FIX_MSG_PROCESSOR' and method 'SET_FIX_PRICE_TICKS'. Exception message is 'Number of Order Request Ticks must be an integer value' [Message ID:'CMMFDAI_FIX_MESSAGE' ; Number: '048'].

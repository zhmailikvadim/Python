Symptom

1. With BE note # 3708331 and FE note # 3708372, a new subtab is introduced in the reconciliation app for items in reconciliation status reconciled with breaks. However this subtab is missing in the UI elements configuration of the config where screens can be assigned to user roles.

2. The maintenance of recon filters in the configuration app for reconciliation requires a manual creation of the filters for each reconciliation track. When dealing with a high number of recon filters and/or filter values, this can be a very time consuming activity.

Other Terms

FCCOREUI ; reconciliation app ; status ; reconciled with breaks ; configuration app ; filter ; upload ; rules ; excel ; filter ID

Reason and Prerequisites

Regarding 1. The new subtab reconciled with breaks is added to the UI elements configuration so that the visibility can be defined on a role level based on customer needs. There is also a BE related note # 3708331 and a FE related note # 3708372 that needs to be implemented as well.

Regarding 2. Upload functionality is introduced for recon filter ID in the configuration of the reconciliation app. There is also a BE related note # 3696876 that needs to be implemented as well.

Solution

Implement the note and all of its prerequisites.

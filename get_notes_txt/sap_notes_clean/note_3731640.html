Symptom

This note corrects the returned messages in authorization scenarios of direct debit mandate processing, ensuring that when a user lacks authorization for the requested action, the system does not explicitly indicate missing authorization but instead returns either the appropriate action-specific message or a message stating that the direct debit mandate does not exist.

For a complete overview of the new functionality Direct Debit Mandates, please check consulting note 3445810.

Other Terms

Direct Debit Mandate, DDM, Package FS_TB_DIRECT_DEBIT_MAND_UI

Reason and Prerequisites

Program Error

Solution

Import the relevant Support Package that is specified for your release. As a manual advance correction, you can implement the attached correction instructions using transaction SNOTE.

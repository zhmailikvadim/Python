Symptom

You execute the Deal Workbench and create a new deal. When you navigate to the Organization tab and change the Sales Organization to one for which the business user is not authorized, the system displays an error message stating "You are not authorized to change agreement type / sales area."

However, when you choose Save, the deal is still saved successfully, although it should not be allowed.

This SAP Note corrects the behavior by ensuring that deals cannot be saved when authorization errors occur.

Other Terms

/IRM/GDLM, Deal, Workbench, Organization, Change, Revision, Sales Org, Save, You are not allowed to change agreement type, Sales area, Incorrect, Authorization

Reason and Prerequisites

This is caused by program error.
Note : SAP Note#3712799  is available for V/4 release.

Solution

Please implement the correction instructions attached. These corrections are only valid if you have applied the Support Packs

Release 6.4 – Base Package & Support Pack 1

Use transaction SNOTE to implement the correction instructions specified below and activate the affected program object

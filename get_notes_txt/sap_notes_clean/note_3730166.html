Symptom

  • The default addressing identifier (BT-49) follows this rule hierarchy:
    • If the suffix is ​​present, the addressing ID will be SIREN_Suffix (the suffix takes precedence over the routing code and the SIRET number).
    • If the routing code is present and the SIRET number is active: SIREN_SIRET_RoutingCode
    • If the SIRET number is present and active: SIREN_SIRET
    • If the SIREN number is present: SIREN "
  • Requirement to use SIREN as the addressing identifier even when SIRET is maintained in the business partner data.
  • No specific error messages or error codes are mentioned.

Environment

  • SAP ERP
  • SAP S/4HANA
  • SAP Document Reporting and Compliance service

Cause

The default addressing identifier logic selects SIRET when it is present and active; no Party ID Type override was configured to force SIREN.

Resolution

  1. Configure the Party ID Type:

    1. Go to the Assign Party ID Types to Business Partner Customizing activity (transaction EDOEUBUPAV).
    2. Create a new entry for the specific business partner (customer).
    3. In this entry, set the Party ID Type to FR:SIREN. This setting tells the system to prioritize the SIREN number for this partner when generating electronic documents.

  2. Maintain Business Partner Master Data:

    1. Go to the Business Partner Master Data transaction (BP).
    2. Navigate to the Identification tab and then the Tax Numbers section.
    3. Select the category FR1.
    4. Ensure the SIREN number is correctly entered in the designated field.

By explicitly setting the Party ID type to FR:SIREN for that customer in the configuration, you override the default logic and instruct the system to use the SIREN number as the addressing identifier.

Symptom

If a Payment Distribution (PD) item remains stuck in “In Process” status and no corresponding results exist in the database from previous executions, any attempt to process this item in subsequent Payment Distribution runs fails. Additionally, this failure causes all other PD items within the same package to be skipped, preventing the package from being processed further.

Other Terms

S4TRBK2.0, In Process, PD Item

Reason and Prerequisites

Program Error

Solution

Import the relevant Support Package for your release.

You can use transaction SNOTE to implement the correction instructions relevant for your release in advance.

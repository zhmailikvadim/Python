Symptom

I have configured a customized award message, however, the system is sending the default email instead of the one I was expecting.

Reproducing the Issue

  1. Create a sourcing event 
  2. Once it's in pending selection status, create an awarding scenario
  3. Submit for award... > Customize Award Emails > select the message > Customize
  4. Change the message's content > Save > Done
  5. Select the checkbox Send Emails to Awarded Participants > Award
  6. Observe that the supplier receives the default email instead of the customized award message

Cause

The supplier profile language is not the same as yours and the award message translation was not configured during the customization process, so the system keeps sending the default email.

Resolution

Ensure that translations for customized award messages are manually set up for all required languages during the customization process. To accomplish this, when you select a message to customize, you will notice two buttons labeled Translations - one for the title and another for the message content. 

Click on each button to add the translations accordingly. By doing this, you will ensure that your message reaches recipients in their preferred language.

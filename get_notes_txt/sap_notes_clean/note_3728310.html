Symptom

When the function module BC_STAB_FETCH_LOGS is called in a loop for different tables in one session, this behavior causes excessive authorization checks and rapidly generates a large number of Security Audit Log (SAL) events, effectively flooding the SAL.

Other Terms

BC_STAB_FETCH_LOGS, VIEW_AUTHORITY_CHECK, DBTABLOG, Security Audit Log; SAL; SAL flood; audit log flooding; excessive SAL events; redundant SAL entries;

Reason and Prerequisites

On repeated calls for different tables from function module BC_STAB_FETCH_LOGS within the same session, the function re-executes VIEW_AUTHORITY_CHECK for tables already checked in previous calls. Each authorization check triggers multiple SAL events, leading to rapid event growth (e.g., calling for ~30 tables produces ~1,000 SAL events within a minute).

Solution

Please implement this SAP note.

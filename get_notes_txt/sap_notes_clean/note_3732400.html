Symptom

The following symptom is corrected with this SAP Note:

  • Incorrect update of customer history 
  • In the R4CM credit insurance master data maintenance (transaction /cgs/start), changes are not saved. With the error message "Previous insured person indicator record exists with same date", the user remains on a tab page such as "Insurance Data". 

Other Terms

Ticket 1000007463, master data maintenance, transaction /cgs/start, exception J_1VCX_ERROR_MESSAGE, RAISE EXCEPTION TYPE J_1VCX_ERROR_MESSAGE, message e558(8V), message e558(8V)

Reason and Prerequisites

The error is due to the check that is processed in the background in the PAI module J_1VCL_MAIN_SCR_TAB_INSURANCE~PAI_0220_YGAB. Before saving one or more master data changes, the system performs plausibility checks in the method J_1VIF_KV_GENERAL_SUBSCREEN~PRE_SAVE_PROCESS. This triggers the exception RAISE EXCEPTION TYPE j_1vcx_error_message and prevents the master data changes from being saved. The changes can only be saved when you save again and, as a result, the customer history can be updated correctly.

The prerequisite for implementing the program corrections is R4CM Credit Insurance (software component P1V) Release 5.700 or 5.751H:

  • Patch level 2403 or 2503 for R4CM credit insurance Release 5.700
  • Patch level 2403 or 2503 for R4CM credit insurance  Release 5.751H

Solution

  • Implement the attached correction instructions.

Corrected as of: R4CM credit insurance 5.700 and 5.751H patch level 2603

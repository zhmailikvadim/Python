Symptom

Users are unable to view notification long text in the “Find Maintenance Notification” app (F2071) on the list or detail pages.

Environment

  • SAP ERP Plant Maintenance (PM)
  • SAP S/4HANA, on-premise
  • SAP S/4HANA, Cloud Private Edition

Reproducing the Issue

  1. Open the “Find maintenance notification” app (F2071).
  2. Apply filters for a plant and notification type where a notification with long text exists, or open a known notification.
  3. Observe that the long text is not displayed in the list and/or object page.
  4. Open the same notification in the “Change/Display Maintenance Notification” app and observe that the long text is displayed there.

Cause

Formerly the long text value was calculated via the exit class CL_EAM_NTF_LTXT_EXIT in CDS view C_ObjPgMaintNotification. It was replaced by a [0..1] association to I_TextObjectPlainLongText in the same CDS view. I_TextObjectPlainLongText eventually reads the data from ESH_SR_LTXT. There should be one living long text value in database table ESH_SR_LTXT. This table does not contain any entries for object QMEL so Long Text is not displayed. 

Resolution

  1. Implement SAP Note 3529817 if it has not been done already.
  2. Test the “Find maintenance notification” app (F2071) and confirm that notification long text is displayed.
  3. To get the long text entries in the table ESH_SR_LTXT, run the report ESH_SR_LTXT_REPLICATE in “Update Run” mode (Transaction SE38)

Symptom

  • It takes a long time to call the F4 help.

  • The search help function is performing slowly during its first call.

  • The performance improves on the second call within the same transaction. However, if the transaction is exited and re-entered, the slow performance issue reoccurs.

Environment

  • SAP S/4HANA
  • NetWeaver Enterprise Search

Reproducing the Issue

  1. Open the transactioMM03.
  2. Initiate the search help function.
  3. Observe the slow performance on the first call.
  4. Repeat the search help function within the same transaction and observe improved performance.
  5. Exit and re-enter the transaction, and observe the slow performance issue reoccur

Cause

The search help exit COM_SE_F4_HELP_EXIT is using some ESH-functionality which reads a lot of dictionary metadata causing long runtimes when first using the type-ahead search help. 

Resolution

  1. Implement SAP Note 3516884 - F4 help too slow
  2. After implementation, retest the F4 help in transaction MM03 (or another transaction) to confirm the performance improvement.

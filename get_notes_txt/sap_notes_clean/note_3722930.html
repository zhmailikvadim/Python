Symptom

A case has been reported with “Very High” priority in the HCM Integration SF Product Area, but the SAP support engineer changed the case priority.
This KBA explains the scope of "Very High" priority of HCM Integration SF cases. 

Environment

  • SAP SuccessFactors Employee Central integration to SAP Business Suite
  • SAP SuccessFactors Employee Central integration with SAP Human Capital Management for SAP S/4HANA
  • SAP SuccessFactors Employee Central Payroll

Cause

  • To maintain optimal service levels, SAP assesses the priority of reported cases based on the actual business impact.  
  • The initial “Very High” priority does not always meet SAP’s required criteria.

Resolution

This KBA defines when “Very High” priority is appropriate for HCM Integration SF cases.

“Very High” priority is justified when:

  • The productive connection between SF and ECP/ERP/S4H is totally down. 
    • Replication is not possible for any employee (both last modified and employee-specific queries fail).
  • An imminent production go-live (within the next five working days) is impacted and could cause serious losses if not resolved (if possible, provide us with a document for the project timeline).
    • The issue has not occurred before (testing previously successful).

“Very High” priority is NOT justified when:

  • Test or development connections are down.
  • Pay Statement is not displayed.
  • Integration project timelines are delayed.
  • Only employee-specific replication errors occur (in any system: test, dev, or production).
  • Confirmations are not working.
  • Issues occur in test systems unless an imminent go-live is impacted.
  • Payroll is affected, but the pay date is still several days away (in the next 5 working days).

Very High Priority Requirements:

To raise a “Very High” priority case, the following must be provided:

  • Go-live date if relevant and customer/partner project manager phone numbers.
  • Full Business Impact description.
  • 24×7 contact phone number (English speaking).
  • SP levels for integration components: 
    • PA_SE_IN
    • ECS4HCM
    • EA‑HRRXX
  • Access to all systems involved. Some examples: 
    • Replication issues: SF, CPI (if used), backend (ECP/ERP/S4H).
    • Mashups / Pay Statement: SF, backend (ECP/ERP/S4H), IAS (if used).
    • User creation: SF, ECP, IPS.
  • Clear definition of the issue, steps to reproduce, and expected results.
  • Written approval to replicate the issue in production (if applicable).
  • Timeline showing when the issue started.
    • Any customer actions (upgrades, config changes, transports).
    • Confirmation whether it worked before.

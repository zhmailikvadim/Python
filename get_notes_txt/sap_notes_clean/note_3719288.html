Symptom

  • Integration error occurs in Cloud ALM for Integration Suite during configuration for Integration & Exception monitoring and Alert configuration.
  • Unable to switch on configuration for Integration Suite, API Management, and Eventmesh.
  • Under landscape management, Eventmesh and API Management are active but endpoints cannot be added under eventmesh.
  • For API Management, endpoints from Process Integration Runtime with API Plan and Integration Flow are not accepted.

Environment

Landscape Management Service

Reproducing the Issue

  1. Configure SAP Integration Suite with Cloud ALM for Integration & Exception monitoring and Alert configuration.
  2. Switch on data collection for Health monitoring.
  3. Attempt to switch on configuration for Integration Suite, API Management, and Eventmesh.
  4. Attempt to add endpoints under eventmesh for API Management using Process Integration Runtime with API Plan and Integration Flow.

Cause

  • For SAP Integration Suite service types API Management and Event Mesh, the Integration & Exception Monitoring use case is currently not available.
  • For these service types, only Health Monitoring is supported in SAP Cloud ALM.

Resolution

1. Integration & Exception Monitoring – Data Collection

For SAP Integration Suite service types API Management and Event Mesh, the Integration & Exception Monitoring use case is currently not available.

For these service types, only Health Monitoring is supported in SAP Cloud ALM. Therefore, the option to switch on data collection for Integration & Exception Monitoring will not be available for API Management and Event Mesh. This behaviour is expected.

 

2. API Management – Endpoint Configuration

Regarding the issue when adding endpoints for API Management, please carefully re-follow the setup steps described in the official documentation:

https://help.sap.com/docs/cloud-alm/setup-administration/sap-integration-suite-api-management?locale=en-US

 

3. Event Mesh – Health Monitoring

For Event Mesh, please note that no endpoint configuration is required in Cloud ALM.

If Event Mesh is active under Landscape Management, please firstly check whether there is relevant monitoring data available in your Event Mesh tenant.

It is possible that:

  • The connection is correctly established, but
  • There is currently no operational data to be reported in Health Monitoring.

From the health monitoring content page, this is the available data for Event Mesh: Message Consumers, Active Connections, Broker State, Message Queues, Message Spool Usage.

Symptom

  • Only one user encounters the error "Cannot load query / data provider 'DS_1': No service implementation available" when accessing the Lumira dashboard, while others with identical authorizations experience no issues.
  • Deleting and recreating the problematic user account did not resolve the error, indicating the problem may not be tied to the user profile itself.
  • The error is consistently triggered by the "DS_1" data provider, suggesting a localized configuration or access issue for this specific user.
  • Lumira APS logs reflect below:

Environment

  • SAP BusinessObjects Business Intelligence platform (BI) 4.3 / BI 2025
  • Windows/Linux

Reproducing the Issue

  1. Open the Lumira dashboard via the OpenDocument URL
  2. Logon with the Enterprise User and password of the problematic user
  3. Failed with error:  Cannot load query / data provider "DS_1": No service implementation available 

Cause

  • The password may have expired and needs to be reset.

Resolution

  • After resetting the user’s password via SAP GUI — Lumira dashboard loads without errors.

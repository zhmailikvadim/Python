Symptom

  • ERS indicator should be updated on the Purchase Order which has GR/IR posted in fiscal year.
  • You need activate the ERS indicator without reversing the GR/IR.

Environment

  • Materials Management (MM)
  • SAP ERP Central Component
  • SAP ERP
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA

Reproducing the Issue

  1. Identify a Purchase Order with GR/IR posted in fiscal year.
  2. Attempt to activate the ERS indicator on the Purchase Order without reversing the GR/IR.

Cause

  • In standard SAP functionality, the ERS process runs only at the time of Goods Receipt (GR) posting, and only if the ERS flag is already active on the Purchase Order (PO) item.

Resolution

  • In standard SAP, you cannot retroactively apply ERS to GR/IR documents that were posted before the ERS indicator was set on the PO. The ERS process runs only at the time of GR posting, and only if the ERS flag is already active on the PO item.
  • If the ERS checkbox is not active when the GR is posted, SAP does not create any ERS-relevant record for that GR, so you can't retroactively settle it via MRRL.

  • An option could be reverse and Re-post GRs 
    Technically, you could:

    - Reverse the existing GR (via MBST or MB01 with 102 movement).

    - Activate the ERS flag on the PO.

    - Re-post the GR.

    - Then run MRRL.

    Note: This approach is not recommended unless absolutely necessary, as it impacts inventory, financials, and audit trails.

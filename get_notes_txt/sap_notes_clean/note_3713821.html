Symptom

The Country of origin (COO) in Intrastat report is supposed to be determined from batch master data from batch maintained in inbound delivery, however, it is still determined from vendor info record or material master data. 

Environment

  • SAP S/4HANA
  • SAP S/4HANA Cloud Private Edition
  • International Trade Management - Intrastat 

Reproducing the Issue

  1. Create a purchase order using transaction ME21N.
  2. Create an inbound delivery using transaction VL31N.
  3. Assign a batch to the inbound delivery.
  4. Create Intrastat receipts using transaction MEIS.
  5. Check COO via /n/ecrs/rp_edit.

Cause

  • 'Selection logic' (field T609IS-WEREC) is set with 'B' (Select Invoice Receipts: Goods receipts are not considered at all. Only invoice receipts (VGABE = 2 or 6) are relevant and consolidated in the process and only the PO history of the selected reporting month is considered. ) in 'Define Data Selection Control for Purchasing' for corresponding country. As a result of this setup, inbound deliveries are not included in the selection process.
  • Batch management is active for the delivery item, but it was not specified on the purchase order item. 

Resolution

To report based on delivery's batches, change the Selection Logic to "A" (Select Goods Rcpts; Wait for Inv. Rcpts; Read Deliveries)) instead of the purchase order.

  • Customizing SPRO -> Governance, Risk and Compliance -> Intrastat -> Data Selection -> Define Data Selection Control for Purchasing

Symptom

You use transaction SM19 to maintain filters of recording profiles.
When you save, the system issues, among other things, a message "Filter relevant for x of y users" that does not correspond to your expectation.
If you check the filter again after saving, the quantity statement is correct.

Other Terms

CL_SAL_CONFIG_CHECK, CHECK_FILT_USER, RSAUPROF_HEAD

Reason and Prerequisites

Program error

Solution

Implement the attached corrections using SNOTE or by importing the referenced support package.

Solution Explanation
The check for the user relevance of a filter is performed using an API based on data that has already been saved.
If your filter has been newly created or has been changed regarding the recording status, client selection, or user selection, the check must not yet be performed.

Note: In quantitative analysis, only the users of the current logon client are considered.

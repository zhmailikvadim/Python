Symptom

When I try to convert a Purchase Requisition (PR) from SAP Ariba Buying and Invoicing to a Purchase Order (PO) in my SAP ERP Central Component (ECC) system, I receive an ERP Push Error with the following messages:

Value of service item exceeds allowed value limit External currency amount too big - unable to convert Purchase Order Failed

Environment

  • Managed Gateway for Spend&Network
  • SAP Ariba Buying and Invoicing (B&I)
  • SAP ERP Central Component (ECC)

Cause

The error occurs because the total value of a single service line item in the Purchase Requisition exceeds the maximum character limit allowed for amount fields in the SAP ERP system. This is a standard system limitation, and the specific limit can vary depending on the currency (for example, 11 digits for JPY, KRW, VDN and INR currencies).

Resolution

To resolve this issue, you need to adjust the line items in the Purchase Requisition (PR) so that their value does not exceed the system's limit.

  1. Open the PR that is failing.
  2. Identify the service line item with a value that exceeds the character limit for the amount field in your ECC system.
  3. Split the high-value service line item into two or more separate line items.
  4. Ensure the value of each new line item is within the acceptable limit for your ECC system.
  5. Resubmit the PR for processing.

You can check the specific restrictions for amount fields in the product documentation. For example, certain currencies like Japanese Yen (JPY) may have an 11-digit limit for amount fields:

Restrictions for Purchase Requisitions.

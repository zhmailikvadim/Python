Symptom

  • Compliance checks are being triggered for all warehouses, even for those where the GTS business system is not defined 
  • Error message: "RFC for GTS system could not be determined" /SCWM/NCTS (Transit Procedure: Messages) Message Number_003.
  • Technical error observed in outbound delivery order processing: "Action could not be successfully executed".

Environment

    • SAP S/4HANA
    • SAP Extended Warehouse Management
    • GTS
 



Reproducing the Issue

  1. Configure the GTS system for a specific warehouse number (e.g., ****) using the following path: SPRO -> SAP SCM Extended Warehouse Management -> Extended Warehouse Management -> Interfaces -> GTS Integration -> Define GTS system for warehouse number. warehouse-specific settings are applied in table view `/SCWM/V_T340DGTS.
  2. Trigger a compliance check for outbound delivery orders in embedded EWM.
  3. Observe that compliance checks are triggered for warehouses not activated for GTS compliance.

Cause

Same Document type used whare relevant to legal control has the GTS check box active, causing compliance checks to be triggered for all warehouses.

Resolution

There are two approaches to determine at warehouse level whether a delivery document is relevant for GTS compliance checks:  

1. Review the settings for outbound delivery orders in embedded EWM: :You can utilize a specific document type for deliveries that are relevant for compliance checks. To implement this, it is necessary to customize the mapping of the ERP document type to the corresponding EWM document type.  

  1. Navigate to the IMG path: SCM EWM -> Extended Warehouse Management -> Goods Issue Process -> Outbound Delivery -> Define Document Types for Outbound Delivery Process.
  2. Ensure the GTS check box is only active for relevant warehouses.


2. Alternatively, you can implement the BAdI /SCWM/GTS_CHECK_RELEVANCE_SET. This allows you to decide relevance based on the warehouse number.

Symptom

This SAP Note provides important information and considerations related to the product SAP GRC UIDP Masking for SAP HANA.

Master Note: 3713860

Other Terms

SAPGRC 100, GRCUIDP 100, UISAPGRC 100, SAP GRC UIDP Masking, UI Masking

Reason and Prerequisites

important information and considerations for SAP GRC UIDP Masking for SAP HANA

Solution

Technical Considerations:

This product is available on SAP S/4 HANA Foundation 2025 and SAP S/4HANA 2025 on-premise and private cloud editions.

SAP GUI 

  1. Masking and Blocking are not supported for field(s) where
    • the field is not captured by the UI masking recording tool, and 
    • the technical address (table name - field name/data element) is not available in F1 help.
  2. Masking is not supported for following:
    • Proposed values in dropdown list
    • Debugging
    • SQL command editors
    • Checkboxes & radio buttons
    • HTML based controls/views
    • SAPScript and Smart Forms
    • ALV reports developed using IDA
    • Messages
  3. For certain scenarios where download or print output cannot be masked due to technical constraints, customers can disable these options using Button Disable Configuration. In downloaded Excel files, masked values remain type-compatible; for example, numeric columns are replaced with their initial value, such as “0000.”
  4. In query transaction SQ01, only data element based masking configuration is supported.

WebClient UI 

  1. Masking is supported for UI fields that are part of context node in a UI component. Fields created through custom BSP pages, tags, and HTML elements will not be supported.
  2. Masking is not supported for
    • UI fields or attributes with custom logic implemented in the UI layer (server or client side)
    • Data in attachments
    • Data in customer fact sheet
    • Data in recent items

Web Dynpro ABAP

  1. Masking is supported for the following UI elements
    • Input field
    • Text edit
    • Text view
    • Label
    • Button
    • Checkbox
    • Dropdown by index
    • Table
    • Dropdown by key
    • Dropdown by list box
    • Link to URL
    • Link to action
  2. Proposed values in dropdown lists cannot be masked.
  3. Masking is not supported in print scenarios. Customers can disable the Print button by maintaining the Button Technical Address in the masking configuration.

SAPUI5/SAP Fiori 

  1. Blocking is only supported in Fiori/UI5 applications consuming OData v2
  2. Masking and Blocking are not supported in the following:
    • Primary key fields (these fields are required for navigation and update scenarios. However, it is possible to use these as context attributes in policies)
    • Get stream (BLOB content)
    • Fiori Launchpad hosted in environments other than on-premise or private cloud
  3. For download or print scenarios, fields with data types other than string are replaced with dummy values.
  4. In a few cases, where OData service does not support runtime addition of fields to an entity, additional configuration may be required. In such applications:
    • Properties of types other than string are cleared irrespective of the masking pattern configured
    • Masked fields remain editable, however, any changes made to these values are discarded

Embedded Analytics in S/4HANA

  1. Masking/Blocking is supported only for read-only scenarios
  2. In SAP Analytics Cloud, support is available only for S/4HANA live data connection scenarios
  3. Numeric and date/time fields are masked using dummy values instead of masking characters (for example **), due to type conversion constraints
  4. Masking/Blocking applies only when data is processed through InA protocol / ABAP BICS in S/4HANA embedded analytics and SAC
  5. The following user interfaces are outside the masking/blocking scope:
    • SAP Analysis for Office
    • Lumira
    • SAP BO Web Intelligence
    • List / HTML / ABAP Web / Java Web in RSRT
  6. For ABAC based scenarios, context attributes must be included in the query result

Other Considerations

  1. Data Blocking Features:
    • In SAP GUI, output-based blocking does not apply to classical reports generated using WRITE statements and print forms
    • The total record count displayed reflects the actual number of records, regardless of suppressed records
    • In scenarios where out-of-box blocking is not available (for example, machine-to-machine communication such as RFCs and Web Services), applications can be enhanced to use the UI Data Protection API to implement blocking.
  2. The Reveal on Demand feature is available in SAP GUI and SAP Fiori only.
  3. In create and update scenarios, if a masked field is blank, it appears disabled for input. This can be handled by configuring blank fields to be ignored for masking.
  4. Masking and Blocking are not supported for:
    • Standalone BSP applications (non-SAPUI5 and non-CRM).
    • Standalone HTML applications (non-SAPUI5).
    • Universal WorkLists (UWL).

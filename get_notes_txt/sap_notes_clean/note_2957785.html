Symptom

  • Create a replication definition and received error 15476
    create database replication definition DS_DB_repdef
    with primary at DS.DB 
    replicate ddl
    go
    Msg 15476, Level 12, State 0: Server 'RS': Log transfer from 'DS.DB' is OFF.
    Cannot create database replication definition for this database.
  • The database connection DS.DB was added as replicate
  • Goal is to replicate data out of DS.DB

Environment

SAP Replication Server (SRS) all versions

Resolution

To change the replicate database connection to be  a primary database, the following steps are needed:

  1. Turn on log transfer on the connection in replication server
     
    alter connection to DS.DB set log transfer to on
    go
     
  2. Submit rs_install_primary.sql against the primary database
     
    isql -Usa -S<primary_ASE> -Pxxxx -D<DB> -i $SYBASE/$SYBASE_REP/scripts/rs_install_primary.sql -o output.txt
     
  3. Verify that the script execution is successful in output.txt file.

Keywords

Primary database, MSA

Symptom

  • When two concurrent employments have the same hire date, changing the hire date of one employment via Hire Date Correction leads to missing Secondary Assignment records.
  • In Manage Data > Secondary Assignment: the record for the original hire date is missing; in the record for the new hire date, the other concurrent employment is missing under Secondary Employment.

Environment

SAP SuccessFactors Employee Central

Reproducing the Issue

Employee A has 2 concurrent employments:
Con1: user id = 103366, username = iheine-1, hire date = 04/01/2026
Con2: user id = 103367, username = iheine-2, hire date = 04/01/2026
 
If change hire date of 103367 from 04/01/2026 to 05/01/2026 via Hire date correction, succeeded.
But check Manage data >> secondary assignment, employee A:

1. 04/01/2026 record is missing;
2. in 05/01/2026 record >> Secondary Employment: Con1 (user id = 103366, username = iheine-1) is missing.

Expected Behavior:

04/01/2026 & 05/01/2026 records should exist after the hire date correction. In 04/01/2026 record, there is Con1 under secondary assignment. In 05/01/2026 record, there are Con1 & Con2 under secondary assignment.

Actual Behavior:

04/01/2026 record is missing. In 05/01/2026 record >> Secondary Employment: Con1 (user id = 103366, username = iheine-1) is missing.

Cause

The issue is caused by a gap in the system's handling of secondary assignments when concurrent employments share the same hire date and one hire date is changed using the Hire Date Correction tool.

Resolution

A product fix is planned for the 2H2026 release.

As a temporary workaround, please correct hire date using a Full Purge Import.

Monitor this KBA and product release notes for the delivery of the fix.

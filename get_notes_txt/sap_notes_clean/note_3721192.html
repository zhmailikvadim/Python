Symptom

  • The App "Assign Product to Product Allocation" and the database table "PASQNC_ASSGMT" contain combinations of deleted or phased-out products from the plant, which require cleanup. 
  • Verification is needed regarding the available options in SAP Standard and the existence of a tool or program for an automated cleanup process.

Environment

  • SAP S/4HANA Advanced Available to Promise (aATP)
  • Product Allocation Check

Reproducing the Issue

  1. Open the App "Assign Product to Product Allocation" or check entries on database table "PASQNC_ASSGMT"
  2. Verify that the app and database contain assignments of products that have been deleted or phased-out
  3. Identify options to clean up these obsolete assignments.

Resolution

In the currently available and planned SAP Standard, there exist 2 options that help to actively delete obsolete product allocation sequence assignments in aATP Product Allocation.

  • Available:  SAP Standard Option 1 - App "Assign Product to Product Allocation", AppID: F2120 
    • Open App "Assign Product to Product Allocation" (App ID: F2120).
    • Select relevant Allocation Sequence.
    • Access "EDIT Mode".
    • For single deletions, manually select single entries and press "Delete" button.
    • For mass deletions, download the existing entries as Excel File, fill column "Delete" in the Excel File for the assignments that should be deleted and upload updated data.
    • Save data changes via "Save" button.

  • Available:  SAP Standard Option 2 - API Delete Product Location Assignment
    • Refer to SAP Help Documentation of your product version for API: Delete Product Location Assignment Link.
      • Use the HTTP method POST to call the DeleteSequenceAssignment function.
      • Test API using Transaction /IWFND/GW_CLIENT (SAP Gateway Client).


  • Mid- and Long Term Approach: Recommend Improvements for Future Product Versions
    • An automated option, that offers automated identification and deletion of phased out product assignments is not yet available in SAP Standard.
    • Short term, this option would need to be realized as a customer specific solution attempt.
    • For mid- and long-term approaches, the opportunity might be used to interact with SAP development teams on enhancing SAP standard products. Refer to SAP KBA 3593361 point 2b.

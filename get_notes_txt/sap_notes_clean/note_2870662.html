Symptom

There are 6 decimals represented in some of the fields of the Stage Analysis report despite the decimal settings.

Reproducing the issue

Prerequisites:

  • Project Management is enabled in Administration → System Initialization → Company Details → Basic Initialization.
  • In Administration → System Initialization → General Settings → Display tab, you have set decimal places for Amounts to 2.

Scenario:

  1. Go to Project Management → Project and create a project of Project Type: External, as follows:
    • Enter a BP code of Customer and Vendor type, respectively.
    • Enter a name in the Project Name field of the header.
    • Enter a Financial Project.
    • Enter a date into the Start Date field of the header.
    • On the Stages tab, enter one stage line with a valid Start Date.
    • Choose Add to add the project.
  2. Create an A/R Invoice or A/P Invoice for the BP used in the created Project records as follows:
    • Add the Project field on the row level, and choose the Financial Project used in step 1.
    • Choose Add to create the documents.
  3. Go back to the Project record, and choose Refresh Record from the upper menu bar. Then proceed as follows:
    • Choose Yes to Unassigned documents exist in the system. Do you want to assign them?
    • In the Document Assignment window, assign the invoice to the Stage.
    • Choose Assign.
    • Highlight the Stage line.
    • Expand the Documents section.
  4. Run Project Management → Project Reports → Stage Analysis report.

Expected behavior:

There are 2 decimals represented after the comma in all of the fields.

Actual behavior:

There are 6 decimals represented after the comma in the following fields: Invoiced (A/R), Total (A/R), Total Variance, Accumulated Invoiced (A/R), Accumulated Total (A/R), Acc. Pot. Subproject Amount Total Variance.

Solution

Symptom

If many replication requests are raised during some hierarchy processing, i.e.

  • many cost center changes are being imported by MDG and the auto-replication of the hierarchy is switched on 
  • many manual changes are done in the SET hierarchy in a short time and the auto-replication of the hierarchy is switched on 

there are many background jobs created and it can take a long time to get the hierarchy replicated.

Other Terms

HRRP, Replication, MDG, Master Data Governance, Auto-replication, HRY_REPRELEV

Reason and Prerequisites

Program enhancement

Solution

Apply following correction instructions using transaction SNOTE.

This note brings an enhancement - if there are more replication requests in a short time, they are collected and the replication is done only once for the requested hierarchy. This is achieved by a new ABAP daemon FINS_HRRP_REPLIC_DAEMON. The daemon is activated automatically, but it could be activated manually using the program FINS_HRRP_START_DAEMON delivered by this note too.

Please be aware that the drawback is that the auto-replication is not done immediately, but there will be short delay (~ 10 seconds or more until the replication requests are comming).

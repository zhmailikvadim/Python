Symptom

  • An "ODBC error [42703] is raised saying that the (first) column of my Table does not exist" when trying to connect tables in Data Manager.
  • Data Manager is unable to connect tables with column names containing special characters.
  • Data Manager fails to connect with tables with column names in mixed cases.

Environment

SAP Predictive Analytics 2.5

Reproducing the Issue

  1. Launch SAP Predictive Analytics.
  2. Go to Data Manager > Create a Data Manipulation and click on the Magnifier.
  3. It works as expected (I can view a sample from the targeted table).
  4. Then click next > (fields of the table are correctly detected) > Views (tab) > Data and Statistics.
  5. Refresh and it throws error.

Cause

Issue has been identified due to mixed case or special characters used in column names of tables.

Resolution

When connecting to DBMS, Automated Analytics does not use quotes around table and column names by default. So if they contain mixed cases or special characters including spaces, they can be misinterpreted by the DBMS. To change this behavior, you need to set the CaseSensitive option to "True" as explained in the ODBC Fine Tuning guide.

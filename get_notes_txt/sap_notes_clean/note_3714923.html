Symptom

  • During SAP system installation or system build using Software Provisioning Manager (SWPM) with SAP NetWeaver 7.5 and SAP ASE, the installation stops at step:

        “SAP ASE Database System Parameters”

  • The following message is displayed:
      Use custom ports, at least one port is already occupied.
  • Log extract (sapinst_dev.log):
      SLPGuiEngine::showMessageBoxHandleButtonImpl()
      Use custom ports, at least one port is already occupied.

Operating system checks confirm that the default ASE ports (4901, 4902, 4903, 4904) are not in use.

Environment

  • SAP NetWeaver 7.5

  • SAP Adaptive Server Enterprise (ASE)

  • Software Provisioning Manager (SWPM)

  • Linux Operating System

Cause

The issue is caused by residual data from a previous or incomplete SAP ASE / SWPM installation attempt.

Although no process is actively listening on the default ASE ports (4901–4904), SWPM may detect a port conflict due to:

  • Remaining sapinst state directories

  • Partially created ASE installation directories

  • Obsolete ASE entries in /etc/services

  • Incomplete cleanup after failed installation

The port validation performed by SWPM includes internal checks beyond active OS listeners.

This issue is not related to proxy configuration or network connectivity, as it occurs before any database connection is established.

Resolution

Perform a complete cleanup of previous installation remnants before restarting SWPM:

  1. Verify no ASE-related processes are running:

     
    ps -ef | grep -i ase ps -ef | grep -i sybase

  2. Remove SWPM temporary and state directories:

     
    rm -rf /tmp/sapinst_instdir rm -rf /root/.sapinst

  3. Remove partially created ASE installation directory (only if no productive database exists):

     
    rm -rf /sybase/<SID>

  4. Check /etc/services and remove obsolete ASE-related entries if present.

  5. Start SWPM again from a clean shell and proceed using the default ASE ports.

Symptom

  • Non-working time maintained in the Work Center is not synchronized with the graphical display in the Detailed Scheduling Planning Board when the shift occurs on a past date.

  • CRC2 capacity interval utilization is not correctly displayed in the Detailed Scheduling Planning Board after a shift update for a resource.

Environment

  • Detailed Scheduling Planning Board

Reproducing the Issue

  1. Navigate to CRC2 and input the Work Center and plant.
  2. Go to Capacity -> Intervals and Shifts -> Create a Shift Interval with '0' capacity a few day's ahead.
  3. Execute report /SAPAPO/CRES_CAPACITY_LENGTHEN or update the shift definition in CR02 to trigger resources Time Stream Update.
  4. Access the Detailed Scheduling Planning Board and notice that a previous Non-Working time maintained is now showing as working time in the DS Board.

Cause

  • According to report /SAPAPO/CRES_CAPACITY_LENGTHEN documentation: Manually entered changes to available capacity, such as additional shifts, that lie in the past and outside the newly generated validity period are deleted by the report /SAPAPO/CRES_CAPACITY_LENGTHEN without warning.

  • As report /SAPAPO/CRES_CAPACITY_LENGTHEN updates the Resource Time Stream according to the “Days -” and “Days +” value in Tab APO Resource → Planning Parameters in the Work Center Capacity, it's expected that a shift interval with 0 capacity that is updated to a past date is deleted from Live Cache.

  • The shift remains maintained in CR02 but is deleted from LiveCache, therefore, it's not displayed correctly in the Detailed Scheduling Board.

Resolution

  1. Increase the “Days -” value in Tab APO `Resource` → Planning Parameters in the Work Center Capacity (T-code CR02). (This adjustment allows the system to continue considering past shift intervals in LiveCache as non-working time for a longer period ("Days -" value)).

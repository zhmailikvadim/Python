Symptom

  • Trusted Authentication (Eg SAML, Query string etc) fail 
  • One of the following errors may occur during the login:
    Http Status 500-Internal Server Error on FioriBI/logon.jsp
    Account information not recognized: Enterprise authentication could not log you on. Please make sure your logon information is correct. (FWB 00008) 

Environment

  • SAP BusinessObjects Business Intelligence (BI) Platform 4.x / 2025
  • Windows
  • Linux / Unix 

Reproducing the Issue

  1. Setup Trusted Authentication following:
    3559381 - Configuring Trusted Authentication in 4.3 SP04 Patch 1000+ & BI 2025 
  2. Test it - and observe that it fails

Cause

Trusted Authentication was previously setup using the legacy method (TrustedPrincipal.conf), and the TrustedPrincipal.conf files still exist in the win64_x64/win32_x86 or linux_x64 / linux_x86 folders on the web application server (Eg Tomcat)

When this is in place, Tomcat attempts to use the legacy method of authentication, which won't work with the new method of trusted authentication.

Resolution

  1. Check the configuration of the web application server to know if there is a custom path for the TrustedPrincipal.conf
    The custom path is set via a JAVA environment variable: -Dbobj.trustedauth.home=<folder that contains TrustedPrincipal.conf>
    For Tomcat on Windows, this can be seen via:
    Start | Programs | Tomcat | Tomcat Configuration

    For Tomcat on Linux, this can be seen in:
    <TOMCATIDR>/bin/setenv.sh

    For other web application servers, refer to the documentation where java environment variables are configured

  2. If the java environment variable -Dbobj.trustedauth.home was found, skip to step 3, otherwise:

    On the Tomcat/webapp machine, Navigate to the following directories:
    Windows
    <INSTALLDIR>\SAP BusinessObjects Enterprise XI 4.0\win64_x64
    <INSTALLDIR>\SAP BusinessObjects Enterprise XI 4.0\win32_x86

    Linux:
    <INSTALLDIR>/sap_bobj/enterprise_xi40/linux_x64
    <INSTALLDIR>/sap_bobj/enterprise_xi40/linux_x86
  3. Rename the TrustedPrincipal.conf as TrustedPrincipal.conf.old
  4. Navigate to the INSTALLDIR\tomcat\webapps\BOE\WEB-INF\config\custom\global.properties
    Ensure trusted.auth.shared.secret= value is empty
  5. Restart Tomcat
  6. Re-test Trusted Authentication

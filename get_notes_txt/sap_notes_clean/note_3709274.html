Symptom

• Users encounter a "500 Internal Server Error" and gateway timeout when attempting to run a call on /sap/opu/odata/IBP/MASTER_DATA_API_SRV.  
• Calls to /sap/opu/odata/IBP/PLANNING_DATA_API_SRV, utilizing the same communication arrangement SAP_COM_0720, are successful.  

Environment

SAP Integrated Business Planning (SCM-IBP)

Reproducing the Issue

1. Configure Authorization to Basic Authentication using the communication user and password.  
2. Run a GET request on the endpoint: https://my*****-api.scmibp.ondemand.com/sap/opu/odata/IBP/MASTER_DATA_API_SRV/DW1PRODUCT(PRDID='SAP',PlanningAreaID='SAPIBP1',VersionID='__BASELINE').  
3. Note the error message: "500 Internal Server Error".  

Cause

The global configuration parameter RELEVANT_MDT_FOR_MD_API was configured as '*', necessitating the integration of all available master data types. A large number of master data types in the system caused the dynamically generated metadata to exceed the service and gateway's handling capacity, resulting in a timeout error.

Resolution

1. Modify the global configuration parameter RELEVANT_MDT_FOR_MD_API from '*' to 'SAP(Prefix)*', ensuring alignment with the parameter PLANNINGAREA (SAPIBP1).  
2. Check the functionality of the MASTER_DATA_API_SRV request URL post-adjustment and validate the request URL syntax.  
3. Conduct tests on the updated configuration to verify the resolution of the timeout error.

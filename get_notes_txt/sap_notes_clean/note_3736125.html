Symptom

  • An error occurs when manually syncing a supplier with Qualified status to ERP.
  • Error observed in integration monitoring: Exception_Text Value is longer than the maximum permitted length 30: 'smprocess.status.type.Qualification.aribalifecycle.active.relatedtype.Sup

Reproducing the Issue

  1. Create a supplier record.
  2. Register the supplier.
  3. Qualify the supplier.
  4. Once the supplier reaches Qualified status, manually sync it to ERP.
  5. Check the integration messages (for example, in Tcode SRT_MONI) and observe the error.

Cause

  • Incorrect field configuration in default properties.csv used an overly specific key that resulted in a value exceeding the maximum permitted length of 30 characters during mapping.

Resolution

Review the default_properties.csv used for the qualification status integration.

Replace the incorrect key with the supported generic field for process qualification:

For Process Qualification, set the vendor.supplierGenericCustomField.smprocess.status.type.${smprocess.type}.aribalifecycle.${smprocess.aribalifecycle}.relatedtype.${smprocess.relatedtype}.category.${matrix.category}.region.${matrix.region}.department.${matrix.department} with a default of${smprocess.uniquestatus}_${smprocess.aribalifecyclestatus} in default_properties.csv.

For Legacy Qualification, set the vendor.supplierGenericCustomField.qualification.status.category.${matrix.category}.region.${matrix.region}.department.${matrix.department} with a default of ${qualification.status} in default_properties.csv.

Save and upload the updated configuration in Default properties.csv file , then test by syncing a newly created supplier that reaches Qualified status to confirm the error no longer occurs.

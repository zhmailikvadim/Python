Symptom

A Sign-reclassification task is configured with specific settings for sign-triggered reclassification based on net values greater than 0 in group currency.

For period 006/2026, the YTD GC balance for Cons. Unit 1010 and FS Item 321720 is 1,637,914.12 USD, which is greater than 0. However, the reclassification does not trigger as expected.

Logs show "No data available," indicating the expected reclassification has not occurred.

Log Header Messages:

Sequence 010: Total for currency-based sign-decision is 26,278,235.95 Group Curr.: USD.

Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.

Environment

SAP S/4HANA Cloud Public Edition

Reproducing the Issue

  1. Navigate to the Data Monitor/Consolidation Monitor
  2. Execute Reclassification Task in Test mode.
  3. Observe that the logs display "No data available," and the expected reclassification does not occur.

Cause

The option "Curr. for Sign Dec." set as "GC Only sum in group currency is evaluated."

Resolution

  1. Navigate to the "Settings" tab in the Data Monitor task S1010 configuration.
  2. Locate the option "Curr. for Sign Dec." set as "GC Only sum in group currency is evaluated."
  3. Leave this field empty and re-execute the task.
  4. Verify if the reclassification is triggered as expected.

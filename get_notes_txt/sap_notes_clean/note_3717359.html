Symptom

During a SAP HANA data backup using a Backint interface, the backup stops at a high completion percentage (for example between 80% and 95%).

The SAP HANA database remains online and productive operations continue.

The following entries can be found in the indexserver trace:

[110075] Backup was canceled by user
[2110005] multistream data transfer terminated

No database crash or restart occurs.

The behavior may be interpreted as a database backup failure although the database itself is operational.

This KBA applies if:

  • the database remains online
  • no indexserver crash occurs
  • only the backup operation is canceled
  • the errors 110075 and 2110005 appear together

Environment

SAP HANA DB Version 2.0

Cause

SAP HANA transfers backup data to the Backint interface via streaming mechanisms (for example using operating system pipes).

The SAP HANA database acts as the data producer while the Backint process acts as the consumer of the backup stream.

If the Backint consumer temporarily stops reading from the pipe (for example due to storage, media server, or network processing delays), the pipe buffer becomes full.
When the pipe buffer is full, SAP HANA backup write operations block.

Blocked backup write operations can impact internal persistence operations such as savepoint processing.
In this situation SAP HANA may terminate the backup operation when the backup stream cannot be consumed by the Backint interface.
The cancellation does not indicate a database failure but a blocked backup data transfer. 

Resolution

Verify the Backint consumer and the downstream backup infrastructure.

The SAP HANA database successfully produces the backup stream.
Further analysis must focus on the Backint consumer and the connected backup infrastructure.

Typical checks:

  • Backup tool throughput
  • Storage response times
  • Media server performance
  • Network throughput between database host and backup infrastructure

Coordinate the analysis with the backup vendor or storage team.

See Also

The error message “Backup was canceled by user” does not indicate a manual cancellation.
It indicates that SAP HANA actively terminated the backup to prevent internal blocking caused by an unconsumed backup stream.

The database remains consistent and operational.

No SAP HANA software defect is indicated when the database remains online and only the backup operation is canceled.

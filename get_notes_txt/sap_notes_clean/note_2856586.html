Alternatively, you may have wrong results to a query involving row-store tables and the disjunctive join operator.

Other Terms

SIGSEGV

Reason and Prerequisites

Reason:

Due to a programming error in SAP HANA, fetching data from disjunctive hash join operator applied to multiple row-store tables can lead in rare cases to a crash or wrong results.

Affected Releases:

  • SAP HANA 1:
    • Revisions <=122.30 (SPS12)

Prerequisites:

  • Execution plan of the statement contains operator DISJUNCTIVE HASH JOIN
  • Tables used in the DISJUNCTIVE HASH JOIN operator are row-store tables (can be temporary tables of intermediate results)

Solution

Apply one of the following SAP HANA Revisions:

  • SAP HANA 1:
    • Revisions >= 122.31 (SPS12)

  • or higher

Workaround:

  • You may change the problematic query to avoid using OR operation in the join condition.

  • You may use SQL hints, for example DISJ_JOIN_INTO_UNION, to influence the execution plan if Disjunctive Hash Join is applied to two tables only (see SAP Note 2142945).

  • You may fully disable usage of disjunctive hash join operator by an online INI configuration:
  1. Disable Usage of Disjunctive Hash Join
    ALTER SYSTEM ALTER CONFIGURATION ('indexserver.ini', 'SYSTEM') SET ('sql', 'disjunctive_hash_join_enabled')= 'false' WITH RECONFIGURE;

  2. After disabling the usage of Disjunctive Hash Join, you have to invalidate the plan cache entry for the query causing the crash. 

    • Lookup the statement hash of the failing query.
      E.g. from the crashdump:
      Dump at QueryExecution:
      {
      Join_hashed_disjunctive::do_fetch
      Buffer id: x
      Operator id: x
      }
      ctx update transactionID: -1
      ctx transactionID : x
      ctx connectionID : x
      ctx logical connectionID: x
      ctx statementID : x
      ctx statement hash : <statementhash>
      ctx sqlusername : x
      ctx appusername : x
      [OK]

    • Lookup the plan_id of the query in the SQL Plan Cache using the statement hash in below command:
      SELECT plan_id FROM M_SQL_PLAN_CACHE WHERE STATEMENT_HASH = '<statementhash>';

    • Invalidate plan_id from step 2 using below command:
      ALTER SYSTEM RECOMPILE SQL PLAN CACHE ENTRY <plan_id>;

Disclaimer

The SAP HANA configuration parameter "[sql] disjunctive_hash_join_enabled" is an internal parameter and may only be used in the context of this SAP Note. Using such a parameter requires that the SAP HANA system has the revision specified in the SAP Note. Using internal parameters outside of this given scope is not supported. Please unset this parameter once you have upgraded to a revision containing the permanent solution.

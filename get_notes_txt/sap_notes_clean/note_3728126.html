Symptom

The GIP-MDS uses a new SSL provider for its web services in the M2M.
To continue downloading response files from the GIP-MDS servers, you must install the new SSL certificate.

Other Terms

Machine to Machine (M2M), MTOM, machine 2 machine, certificate

Reason and Prerequisites

The GIP-MDS SSL provider has changed, requiring an updated certificate to maintain connectivity with the web services.

Solution

After implementing this SAP Note, the system supports the new SSL certificate for GIP-MDS connectivity.

This SAP Note includes the new SSL certificate as an attachment.

Before you add the certificate to STRUST, verify that the certificate is not already present in STRUST. If the SECTIGO SSL certificate from the attachment is already in STRUST, no further action is required.

If the certificate is not present in STRUST, follow these steps:

  1. Download the zip file attachment from this SAP Note.
  2. Select the appropriate certificate based on your company's security level and internal guidelines.
  3. Add the selected certificate to STRUST.

Symptom

When selecting a pay component in the employee profile, the Pay component value defined in the pay component's attributes within "Manage Organization, Pay and Job Structures" are not automatically populated into the amount field.

Environment

SAP SuccessFactors Employee Central - Compensation Information

Reproducing the Issue

  1. Navigate to Manage Organization, Pay and Job Structures.
  2. Set the Pay Component Value to a number.
  3. Navigate to the profile on an employee and open Compensation information portlet.
  4. Create/Edit a record and select the Pay component.
  5. Observe that the Amount field is not automatically populated from the Pay component Value in the pay component definition.

Cause

As per the standard design of the Pay component foundation object, the values defined in the Pay component definition within "Manage Organization, Pay and Job Structures" are not automatically populated when entering the pay component in the compensation information of the employee profile.

Resolution

To automatically populate the Amount field when the Pay component is selected, an onchange rule needs to be configured on the “pay-component” field.

  1. Create a New OnChange Rule with the Scenario: Trigger onChange Rules for HRIS Elements, Base Object: Compensation Model.
  2. Add the If-else conditions as required.
  3. Use the Set operation to set the value of the amount field from the Pay Component Value of the Pay component definition
  4. Assign the rule with onChange event on the pay-component field under payComponentRecurring HRIS element.

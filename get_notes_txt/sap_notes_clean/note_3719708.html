Symptom

In the processes "New Installation Process (DSO View)" (DE_C_GPKE_NIN_DSO) and "New Installation Process (DSO View)" (DE_C_MPES_NIN_DSO), the UTILMD business message is missing mandatory information in step 0150, which leads to an APERAK.

Other Terms

MPES, GPKE, New Installation Process, UTILMD, APERAK, diverse data, clarification case, retrieve data

Reason and Prerequisites

The error occurred because the method for RETRIEVE DATA didn't account for the scenario where the ubd_data-diverse field could be initial. When this field is empty, the UTILMD business message isn't filled with mandatory information.

Solution

Implement the corrections of this SAP Note or import the relevant support package.

After implementing this SAP Note, the system creates a clarification case when the required information isn't available, preventing the UTILMD from being sent with incomplete data. You need to retrigger the process. You can verify the fix by checking that empty diverse data creates a clarification case with the error message "Error occurred when reading table" in step 0150, and the process runs without errors.

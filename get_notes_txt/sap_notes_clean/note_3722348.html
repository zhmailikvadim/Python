Symptom

When try to replicate document from CFIN source system to CFIN target system, it will trigger Assertion Failed dump,

Dump information will be like:

Category               ABAP programming error
Runtime Errors         ASSERTION_FAILED
ABAP Program           CL_FDC_ACCDOC_TAX_ITEM_CHECK==CP
Application Component  FI-GL

"ASSERTION_FAILED"
"CL_FDC_ACCDOC_TAX_ITEM_CHECK==CP" bzw. CL_FDC_ACCDOC_TAX_ITEM_CHECK==CM001
"TAX_CHECK_AND_DETERMINE"

Environment

  • Financial Accounting (FI)
  • SAP ERP
  • SAP S/4HANA 
  • SAP S/4HANA Finance
  • SAP S/4HANA Cloud Public Edition
  • SAP S/4HANA Cloud Private Edition

Reproducing the Issue

Try to restart the In-process AIF GUID and check the dump information in transaction ST22.

Cause

For Central Finance Integration, the data from source system must not set XMWST, since the tax has always to be sent completely by the sender.

Resolution

It needs check the related entries in table CFIN_ACCIT in CFIN source system, 

The value of field "XMWST" is expected not set in CFIN scenario (it is not expected to set with "X").

If the value of the field "XMWST" is set incorrectly, it needs consider to remove such value during replication with custom project.

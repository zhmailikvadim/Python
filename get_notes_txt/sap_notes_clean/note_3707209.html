Symptom

  • A technical error occurs when attempting to create a new entry in the Verify General Journal Entries - New Version app (F2547A).
  • The error message displays "a technical error occurred" with status code 500 and exception details. 

Environment

  • Financial Accounting (FI)
  • SAP S/4HANA Finance
  • SAP S/4HANA
  • SAP Fiori for SAP S/4HANA Finance

Reproducing the Issue

  1. Open the Fiori app F2547A.
  2. Select the 'Create' button to initiate a new entry.
  3. Encounter the error message "a technical error occurred" with status code 500 and exception details.

Cause

Error is due to missing target mappings for the user.

Resolution

  1. Verify that the user has the required target mappings assigned.
  2. Ensure the following target mappings are available: 
  3. For the **Upload** functionality: `AccountingDocument-uploadGLDocforSubmit`.
  4. For the **Create (+)** functionality: `AccountingDocument-submitGLDoc`.
  5. Confirm that the standard SAP role SAP_BR_GL_ACCOUNTANT is assigned to the user. This role includes catalogs SAP_TC_FIN_ACC_COMMON and SAP_SFIN_BC_GL_PARKDCPRE, which contain the necessary target mappings.
  6. To simulate the posting process, also assign the target mapping `VerificationJournalEntry-manage`. 
  7. For reviewing the application log after uploading journal entries, include the target mapping `FinanceApplicationLog-showList`
  8. Assign the standard catalog SAP_TC_FIN_ACC_COMMON to the user account to streamline access to these functionalities.

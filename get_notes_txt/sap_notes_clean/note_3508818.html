Symptom

While trying to consume HTML5 Business content across subaccounts you are met with a "Refused to Connect" error, which happens on an authentication URL.

In Dev Tools, you will see the following error:

  • Refused to display ' https://*.authentiction.eu10.hana.ondemand.com/ ' in a frame because it set X-Frame-Options to 'deny'

While on screen you see similar to:

Environment

  • SAP Business Technology Platform
  • SAP Cloud Foundry
  • SAP Build Work Zone, standard

Cause

End-to-End flow has not been set correctly, as per the official documentation.

Resolution

The correct setup from the official documentation must be followed: Federation of HTML5 Business Solutions

Security Headers must be formatted correctly:

  • Security Header: Content-Security-Policy    Value: frame-ancestors https://*.hana.ondemand.com;

PLEASE NOTE:

Both the provider subaccount of the business solution and the consumer subaccounts must be subscribed to the standard (Application) plan of SAP Build Work Zone, standard edition.

See Also

Federation of HTML5 Business Solutions

Restrictions

CSP: frame-ancestors

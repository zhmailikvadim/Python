Symptom

If a search help is assigned to an input field in the screen painter, this Dynpro field is displayed with a combobox icon by default.

It is not recommended to set the possible entries key concurrently to "0 - (Do not show)".

Otherwise, the outcome of this inconsistent state depends on the GUI:

Whereas SAP GUI for Windows renders nevertheless a combo box icon, SAP GUI for HTML and SAP GUI for Java do not.
The search help functionality, however, is surpressed in all cases. In SAP GUI for Windows, this icon still can be used to trigger a process-on-value request.

Other Terms

Matchcodes, search help, combobox

Reason and Prerequisites

Inconsistent attributes of a Dynpro field.

Solution

In design time, make sure that the possible key entry is not "0". In runtime, check if screen-value_help is set to "0" for the specific field in any LOOP AT SCREEN. If the issue appears on table controls, make sure that the control in ABAP is not used in multiple screens.

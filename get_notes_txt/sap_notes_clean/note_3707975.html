Symptom

The standard report "Process Documentation Review" has been configured to include the "IS_CONTROL" field from the HRP5304 table.

Although the field appears on the Personalize tab, it does not display when the report is executed.

Environment

Process Control 12

Reproducing the Issue

  1. Use T-Code SE16 to access table HRP5304 and confirm the "IS_CONTROL" field exists.
  2. Navigate to report configuration using T-Code SPRO.
  3. Add the "IS_CONTROL" field in the report's personalization settings.
  4. Run the report using NWBC.
  5. Check if the "IS_CONTROL" field appears in the report output; note that it may not display despite these steps.

Cause

The field name was missing the prefix "CN_", and the expected value "X" was not set, preventing it from being included in the report output.

Resolution

Please follow the instruction of this KBA:

2497264 - Risk Coverage Report showing the disabled (Process Step) controls

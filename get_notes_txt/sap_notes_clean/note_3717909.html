Symptom

  • JDBC constructor failed to connect to the database.
  • Java truststore validation failure occurred inside the SAP Identity Management runtime.
  • Error occurs when connecting via JDBC to SQL Server with encrypt=true and trustServerCertificate=false.
  • The Java runtime does not trust the SQL Server certificate chain.

Environment

  • SAP Identity Management 8.0

Reproducing the Issue

  1. Attempt to connect SAP Identity Management 8.0 to SQL Server using JDBC with encrypt=true and trustServerCertificate=false.
  2. Observe that the connection fails due to a truststore validation error.

Cause

  • The Java runtime inside SAP Identity Management does not trust the SQL Server certificate chain.
  • When encrypt=true and trustServerCertificate=false, the JDBC driver requires:
  • A valid TLS certificate.
  • Trusted CA in the JVM truststore.
  • Matching hostname.

Resolution

  1. Ensure that the entire certificate chain for the SQL Server is imported into the SAPJVM Keystore used by the SAP Identity Management runtime.
  2. Restart the Java instance after importing the certificate.

Symptom

When using the entity reassignJobReq in an OData API call to reassign an operator the following error occurs: ‘User xxxx is invalid for [[roleName]]’

Environment

SAP SuccessFactors Recruiting Management

Cause

The UserSysId is used to update the operator and not the username.

Resolution

One possible cause is that the UserSysId is being used in the APi call to update the user, it should be the username. As well as this this the HTTP Method should be "GET" and the user should be in the recruiting group tied to the operator field (if applicable).

Symptom

SLG1 error: HCM001 identification Category Type is missing ( message number /SHCM/EMPLOYEE_BL 069 ) occur when execution report /SHCM/RH_SYNC_BUPA_EMPL_SINGLE

Environment

S/4 HANA OP with HR integration active

Reproducing the Issue

  1. Execution report /SHCM/RH_SYNC_BUPA_EMPL_SINGLE of the specific employee
  2. In SLG1, the error: HCM001 identification Category Type is missing ( message number /SHCM/EMPLOYEE_BL 069 ) occur.

Cause

The IDtype HCM001 configuration is required in V_TB039 and V_TB039A.

Resolution

  1. Add Identification Category 'HCM001' via transaction SM30 for view V_TB039 if missing.

    ID Category = HCM001
    Description = Personnel Number (Main Employment)
    ID Unique = X
    Display Only = X
    Only One ID Number = X

  2. Add Identification Type 'HCM001' via transaction SM30 for view V_TB039A if missing.

    ID Type = HCM001
    Description = Personnel Number (Main Employment)
    ID Category = HCM001
    Persons = X

 These two steps are mandatory during employee synchronization to BP as default.

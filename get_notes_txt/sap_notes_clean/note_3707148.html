Symptom

1. Attempt to create a custom group using import/export task fails.
2. Import Group/User Relationships task does not create custom groups but only maps users to existing groups.

Receiving the below error:

Lookup failed for ariba.user.core.Group with lookup keys: UniqueName, Data: {UniqueName=File "AllGroup_Export.csv" (line [ LINE NUMBER ], column UniqueName): '[ GROUP NAME ] '}

Cause

1. Custom group creation cannot be performed via the Import/Export task in standalone sourcing realms.

2. The Import Group/User Relationships task is designed for user-to-group mapping and does not support the creation of new groups.

Resolution

If custom groups need to be created, perform the following:

  • Create the groups manually through the UI. How do I create a new custom group with users?
  • Alternatively, engage with the consulting team to create an Import Custom Group task during deployment or post-go-live.

If the requirement persists, contact the Customer Success Manager (CSM) to coordinate with the Consulting team How do I contact Consulting or the Best Practice Center? for further assistance.

If you are unsure who your Customer Success Manager (CSM) is, you may contact the Customer Interaction Center in your region for assistance.

Additionally, please raise an enhancement request to include the functionality of custom group creation via Import/Export tasks in standalone sourcing realms.

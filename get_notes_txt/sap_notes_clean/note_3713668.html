Symptom

When the "State" field is configured as mandatory, the system enforces validation (error message: "State is a required field.") requiring users to provide a value for the State field, even if the selected country does not have states/regions.

Environment

  • SAP Sales Cloud Version 2
  • SAP Service Cloud Version 2 

Reproducing the Issue

  1. Configure the "State" field as mandatory.
  2. Navigate to Accounts work center.
  3. Click "+" to create a new Account.
  4. Select a country that does not have states (e.g., Antigua and Barbuda).
  5. Attempt to save the account.
  6. Observe the error message "State is a required field," despite the absence of states for the selected country.

Cause

Resolution

To solve this, our recommendation is to avoid setting the "State" field as mandatory through static configuration.

Hence, users should implement conditional validation using Dynamic Properties, which allows the mandatory status to be adjusted based on specific conditions.

Example:

  • If Country = United States, then State is mandatory.
  • If Country = Martinique, then State is not mandatory.

Note that this type of conditional validation is not planned to be delivered as part of the standard product. Dynamic Properties is the recommended approach for this requirement.

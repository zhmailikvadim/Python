Symptom

  • Error returned when calling the Flow Extension halt endpoint for an Advance Ship Notice
  • API response contains the error message:
    ILLEGAL_STATE_TRANSITION : Illegal state transition for event <FlowExtension_ID>
  • Event is not rejected and the expected failed routing status is not sent

Reproducing the Issue

  1. Configure a Flow Extension for Advance Ship Notice with type set to "suspendresume"

  2. Trigger the Flow Extension event

  3. Call the OpenAPI halt endpoint, for example:

    POST /api/flowextension/v1/prod/flowextensions/action/<FlowExtension_ID>/events/<Event_ID>/halt

  4. Observe that the API responds with:

    ILLEGAL_STATE_TRANSITION : Illegal state transition for event <FlowExtension_ID>

Cause

  • The Flow Extension is configured with type "suspendresume"
  • This type does not support event rejection using the halt endpoint
  • Calling the halt action with this configuration results in an invalid state transition

Resolution

  1. Update the Flow Extension configuration file

  2. Change the FlowExtension type from:

    type="suspendresume"

    to:

    type="suspendresumehalt"

  3. Deploy the updated configuration

  4. Trigger a new Flow Extension event

  5. Call the halt endpoint again

After updating the type to "suspendresumehalt", the halt endpoint works as expected and the event can be successfully rejected.

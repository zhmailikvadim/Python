Symptom

Component: Foundation for EHS Management
Module: Transfer of chemical data from EHS specifications

You use program R_EHFND_FILL_CHM_BY_EHS_SUBST to transfer Chemicals from the SAP-ERP-EHS Specification Database (for example, transaction CG02) to an EHS Management system.

When you transfer data, the system displays error message CM_EHFND_CHM 135: "You cannot add more than one hazardous ingredient" in the application log.

This occurs when you maintain the following data in a specification:

  • Property Type of Substance (SAP_EHS_1011_005): The value in Mixture or Pure Substance is set to Substance, and the corresponding phrase is maintained in BAdI BADI_EHFND_CHM_TRANSFER, method IF_EHFND_CHM_TRANSFER~MAP_TO_CHEMICAL_TYPE, as 'S' (specification),
  • Property Hazardous Ingredients: More than one specification exists on the Composition tab, but only one specification has Component Type ACT_AGENT.

Other Terms

Migration, transfer, Hit List, Chemicals, BAdI: Business Add-In, CompType, unique,

Reason and Prerequisites

Reason:
During transfer, the system checks for a single hazardous ingredient, independent of Component Type. If more than one hazardous ingredient is maintained, the system adds the error and does not transfer the specification.

Prerequisites:
For information about the validity of the corrections, see the correction instructions.

Solution

After implementing this SAP Note, the system additionally checks the Component Type during transfer. The system transfers the specification successfully when only one hazardous ingredient with Component Type ACT_AGENT is maintained.

The "Support Packages" section lists the Support Packages that contain these corrections.

Alternatively, you can implement the attached correction instructions.

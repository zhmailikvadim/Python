Symptom

  • Embedded analytics stories using the standard territory account data model fail to load. 
  • Error message: "Server Error: Error [Server]: (#6018) Query does not exist in the query repository." 
  • Issue occurs when loading existing stories or creating new ones. 

Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.

Environment

  • SAP Sales Performance Management
  • Embedded Analytics
  • Territory & Quota on Hyperscalers

Reproducing the Issue

  1. Log into Embedded Analytics (EA). 
  2. Open a story using the territory account data model. 
  3. Observe the error message: "Server Error: Error [Server]: (#6018) Query does not exist in the query repository."

Cause

  • The schema and view were not present on the database side. 
  • This absence caused the query to fail when attempting to load or create analytics stories using the territory account data model.
  • If the data was visible previously: our platform creates optimized data views based on the specific attributes enabled in your account. When new attributes were recently enabled, a required, automated process to refresh these views did not execute correctly. This left the existing views in an invalid state, leading to the observed errors. 

Resolution

Perform following steps to solve the mentioned issue:

  1. Navigate to Territory and Quota. 
  2. Select Territory Admin.
  3. Go to Administration.
  4. Open Application settings.
  5. Perform Refresh SAC Schema Views.

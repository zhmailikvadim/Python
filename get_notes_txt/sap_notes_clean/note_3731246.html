Symptom

When running payroll, the following hard error occurs in payroll function USTAX:

   

"Images/data in this KBA are from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental."

Cause

The system has identified unaccounted wage types from a period that precedes the Earliest master data change date from IT0003.

Resolution

According to SAP Note 1136214 (PAY: Not every unaccounted period processed in USTAX):

"If unaccounted wage type information exists in table BAL or UNB and belongs to a period that precedes the Earliest master data change date (P0003-RRDAT) stored in the Payroll Status infotype (0003), then the simulated payroll run will error out, issuing the following message: Not every unaccounted period is in current for employee xxxxxxxx Please update the earliest MD change date in infotype 0003 to mm/dd/yyyy then rerun payroll for employee xxxxxxxx."

"During the productive payroll run, when the preceding error message is displayed, standard payroll functionality automatically updates the Earliest master data change date (P0003-RRDAT) in the Payroll Status infotype (0003). Therefore, when you re-execute the productive payroll run, the error message is not displayed again, since Infotype 0003 has been updated."

Based on that, this error is specific to test runs and should not be raised on productive payments (when re-running).
If you'd like to also not receive such error on simulation payroll, you should customize constant UPD03 to a nonzero value in table T511K, as detailed in SAP Note 1136214 (PAY: Not every unaccounted period processed in USTAX).

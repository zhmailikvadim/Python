Symptom

  • SAP Contact Center jobs stopped
  • New statistics have not been processed
  • Running a full OLAP process generates the error
    • File system error: A string store or binary store with a compatibility level of '1050' is at the maximum file size of 4 gigabytes.
    • To store additional strings, you can change the StringStoresCompatibilityLevel property of the associated dimension or distinct count measure to '1100' and reprocess.
    • This option is only available on databases with a compatibility level of '1100' or higher. Physical file: \\?\C:\Program Files\Microsoft SQL Server\...

Environment

SAP Contact Center 7.0

Reproducing the Issue

  1. Connect to the Reporting server Analysis Services using MSSQL Management Studio
  2. Select the OLAP in question and run and Process
  3. Error is triggered after some time

Cause

  • 3rd party issue
  • Dimension attributes or partitions that exceed the 4 GB file size limit

Resolution

  • This issue is not an SAP Contact Center matter and needs to be addressed by the database provider.
  • Please check with your DBA and consult the following articles:
    • "Configure String Storage for Dimensions and Partitions"
    • "Compatibility Level of a Multidimensional Database (Analysis Services)"
  • Make sure that proper backups of the databases are performed before making any changes and test in QA or Dev system before changing production
  • If the above does not solve the issue, please contact the provider for further assistance

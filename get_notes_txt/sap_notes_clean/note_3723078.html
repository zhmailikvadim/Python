Symptom

Find Time Records API returns decimal value, but in Manage Time Records app the effective duration is shown as a whole number.

Environment

  • SAP Digital Manufacturing 2602

Cause

The API returns the duration/effectiveDuration/adjustedDuration as a string in the ISO8601 duration format. This format allows fractional numbers in the lowest time unit (second). The API returns a proper mathematical calculation of the duration, with no assumption on ceiling/flooring/rounding the value.

So, for example PT16.666666666S is a valid duration.

There is also no limit for the decimal digits specified in the standard, but Java uses 9 digits.

Manage Time Records UI, however, trims the decimal digits.

Resolution

Consider using a script in a Production Process to handle the returned value as expected in your implementation.

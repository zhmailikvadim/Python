Symptom

After MRP run, system creates PRs without considering the Dynamic safety stock defined by the coverage profile.

Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.

Environment

  • SAP ERP Production Planning (PP) Release Independent
  • SAP ERP Release Independent
  • SAP ERP Central Component Release Independent
  • SAP Enhancement package for SAP ERP Release Independent
  • SAP S/4HANA, on-premise Release Independent
  • SAP Fiori for SAP ERP Release Independent
  • SAP Fiori for SAP S/4HANA Release Independent
  • SAP S/4HANA Cloud Private Edition Release Independent

Reproducing the Issue

  1. Define Coverage Profile using transaction OMIA.
  2. Maintain Coverage Profile in the material master using transaction MM02.
  3. Run MRP and check results.
  4. Purchase requisition is generated without considering dynamic safety stock.

Cause

The issue is caused by the "Number of periods" value defined in transaction OMIA, which restricts the coverage profile to specific periods.

Resolution

Change or remove the value of "Number of periods" accordingly in transaction OMIA, ensure that the coverage profile is applied across the required periods.

Verify the dynamic safety stock calculation using the formula:

  • Dynamic safety qty = average daily requirements * target range
  • Average daily requirements = total requirements in the period / total days in the period
  • Available quantity (after purchase requisition fulfills requirements) is equal to or greater than the dynamic safety quantity.
  • Dynamic Safety Stock only takes into account for future Requirements without considering past Requirements.

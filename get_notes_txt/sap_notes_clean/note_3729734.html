Symptom

Dump CONVT_NO_NUMBER happens in include RPCBU309_VGET after running payroll driver. "Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental."  

Environment

  • Human Capital Management (HCM)
  • SAP ERP
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP SuccessFactors Employee Central Payroll
  • SAP S/4 HANA

Reproducing the Issue

  1. Run payroll driver via t-code SE38.
  2. Dump happens as shown below:
    Category               ABAP Programming Error
    Runtime Errors         CONVT_NO_NUMBER
    Except.                CX_SY_CONVERSION_NO_NUMBER
    ‘XXXX cannot be interpreted as a number’ (XXXX means wage type)
    For example, ‘/565 cannot be interpreted as a number’.

Cause

The configuration of the PCR is incorrect.

Resolution

  1. Go to t-code PE02.
  2. Change the relevant payroll rule, e.g., X04A. For example, WT/565 is configured in payroll rule X04A.
  3. Under top-level node*, change branch note from text ‘/565 carry-over for next per’ to characteristic ****.

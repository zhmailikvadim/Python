Symptom

When line items are added or removed in a sourcing project, inconsistencies arise between the sourcing project and the corresponding purchase requisition.

Cause

Changes made in the sourcing project are not synchronized with the purchase requisition.

Resolution

To avoid inconsistencies, line items in sourcing requests that originate from purchase requisitions are editable, but adding or deleting items is not supported. In the current integration between SAP Ariba Buying and Invoicing and SAP Ariba Sourcing, automatic synchronization is not supported after the sourcing request is created. As a result, adding or deleting line items in the sourcing project can cause inconsistencies in the purchase requisition. SAP recommends that any modification to line items in the sourcing project be followed by manual updates to the originating purchase requisition to avoid inconsistencies.

See Also

SAP recommends updating the original purchase requisition manually if line items in a sourcing project are changed. This is documented in the Help Portal under Creating a Sourcing Request for a Requisition.

Additionally, if your site is integrated with an external system, you can add line items from material master data (Adding Material Master Items to Guided Sourcing Events). If the sourcing request is initiated from a purchase requisition received from another application, it is recommended not to add or delete items or terms from the sourcing request, as doing so may cause inconsistencies and require manual updates to keep both documents in sync.

Symptom

  • After migrating from SAP ERP 6.0 (BW 7.40) to SAP S/4HANA 2023, testing RFC function modules with /SAPDS/FUNCTION_GET displays parameter structures differently than before.
  • For some custom function modules, the result shows a generic entry such as "RSSOURCE LINE" instead of the expected expanded structure.
  • Other function modules tested in the same system display correctly.

Environment

SAP Data Services XI4.3 and higher version

Reproducing the Issue

  1. Log in to the SAP system.
  2. Start transaction SE37 and run /SAPDS/FUNCTION_GET.
  3. Provide a custom RFC-enabled function module that uses TABLES (or table-like) parameters referencing a line type rather than a concrete structure.
  4. Execute and review the returned structure: observe that a generic entry (for example, "RSSOURCE LINE") is shown instead of the fully expanded structure. Compare with another function module that displays correctly.

Cause

  • Missing support for table parameters referencing line types when reading function module metadata caused /SAPDS/FUNCTION_GET to return a generic line-type entry instead of the expanded structure.

Resolution

  1. Apply SAP Note 2178967 - Support for table parameters referencing line types.
  2. After implementing the note, retest in SE37 by running /SAPDS/FUNCTION_GET on the affected function module to confirm the structure is displayed correctly.
  3. If the structure is still not displayed as expected, clear and regenerate relevant metadata caches in the consuming tool (for example, refresh/re-import in the client) and ensure all related DDIC objects are activated, then test again.

See Also

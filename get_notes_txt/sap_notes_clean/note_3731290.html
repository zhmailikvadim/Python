Symptom

You need to populate the /TMF/D_SOFTHOUSE table with Software House information for the SPED EFD-Reinf R-1000 event in the SAP Tax Declaration Framework (TDF) system for Brazil, but are unsure of the correct procedure. 

Environment

  • SAP ERP
  • SAP S/4HANA
  • SAP S/4HANA Cloud Private Edition
  • SAP Tax Declaration Framework for Brazil
  • TDF REINF app
  • TDF Messaging Framework

Cause

The /TMF/D_SOFTHOUSE table is classified as a complementary table. As per standard TDF design, the maintenance and population of complementary and shadow tables are the responsibility of the customer. These tables are intended to supplement or overwrite information from the standard ERP system that is required for fiscal reporting.

Resolution

The primary resolution is for the customer to implement a process to maintain the data in the /TMF/D_SOFTHOUSE table. This could involve a custom solution or integration with a third-party system.

For customers who do not have an automated solution, a workaround is available using standard TDF transactions to generate a spreadsheet template, populate it, and upload the data.

To use the workaround, follow these steps:

  1. Access the transaction /TMF/SHEET_GEN to generate the template.
  2. On the selection screen, provide the following details:
    • Acronym to a Report: REI
    • View Name: REINF_R1000_SOFT_HOUSE
  3. Execute to generate and download the spreadsheet file.
  4. Open the file and populate it with the required Software House information (e.g., CNPJ, Company Name, Contact Name, Phone, Email).
  5. Save the completed spreadsheet.
  6. Access transaction /TMF/IMPEXL_GUI to upload the data from your saved file into the SAP system.

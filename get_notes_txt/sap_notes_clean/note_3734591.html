Symptom

After execution initial employee synchronization, you got SLG1 error: No transfer of interval 0000000001 0000000999 for object BU_PARTNER (message number NR 028)

Example screenshot:

Environment

S/4 HANA OP with HR integration active

Reproducing the Issue

  1. Configuration switch HRALX PNUMB = example value 1 in table T77S0. 
  2. And in BP module following IMG path, you configurated the internal number range from 0000000001 0000000999, but the number 999 was expired

Cause

The number 999 was expired for number range 01.

Resolution

Please extend the number 999 into more than numbers under number range 01 via following IMG path:

SPRO:
  > Cross application component
    > SAP Business Partner
      > Business Partner
        > Basic settings
          > Number ranges and groups 
            > Define number ranges

The same logic also does work for the external number range as well, which is based on how customer configuration under switch HRALX PNUMB/HRALX PSUBG and HRALX ENUMB/HRALX ESUBG in table T77S0. Please also refer to KBA 2792166 - Same number for BP and employee

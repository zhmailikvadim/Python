Symptom

Error: AADSTS700016:The application with the identifier http://realmname.solution-datacenter.ariba.com was not found in the 'realmname' directory. This issue may occur if the application has not been installed by the tenant's administrator or if no user in the tenant has given consent. Additionally, your authentication request might have been sent to the wrong tenant.

Environment

Single sign on

Reproducing the Issue

Access Ariba SSO URL. http://realmname.Solution-Data center.ariba.com

URLs differs with respect to your solution and data center.

Cause

There can be a two possible causes for this.

  1. Although the Identity Provider (IDP) system, such as Azure or other, utilizes the HTTPS identifier URL, the PS-1444 feature that allows Ariba to send SAML Issuer with HTTPS encryption remains disabled on the Ariba side.

  2. The PS-1444 feature is enabled on the Ariba side to send SAML Issuer with HTTPS protocol; however, the HTTPS identifier URL has not yet been configured in the Identity Provider (IDP) system, such as Azure or other.



Resolution

Solution for first cause:

  1. SAP Ariba has transitioned to enforce HTTPS protocol exclusively for all external communications, including SAML Issuer transmission to the your IDP system such as Azure or other.
  2. The PS-1444 feature must be enabled in the SAP Ariba realm to support secure HTTPS-based SAML Issuer communication.
  3. Reach out to SAP Ariba Support to request the activation of the PS-1444 feature in your SAP Ariba realm.

Solution for second cause:

  1. Either reach out to SAP Ariba Support to request the deactivation of the PS-1444 feature in your SAP Ariba realm since HTTPS identifier URL has not yet been configured in IDP system, such as Azure or other.
  2. Or configure HTTPS Identifier URL in your IDP system, such as Azure or other(Best Security Practice)

Symptom

  • In SAP Replication Server (SRS), create connection to Hive
  • Create connection raises following error in SRS errorlog:

    ERROR #13113 USER(sa) - /generic/useful/cm.c(5386)
    Failed to obtain a reference to the indicated Connector.
    See messages from Connector factory for details. Data server make: hive. Connector type: thrift. Data server: DS_HIVE. Database: RDB.
    ERROR #1027 USER(sa) - /generic/useful/cm.c(5386) 
    Open Client Client-Library error: Error: -99999, Severity 4 -- 'Could not load module <SRS_install_dir>/REP-15_5/connector/lib/libsybhive.so. System error: No such file or director'.
    WARNING #15542 USER(sa) - /generic/ddl/ddldb.c(6918) 
    Connection to server <DS_HIVE> failed. Command batch not executed. See the Replication Server error log for details.

Environment

  • SAP Replication Server (SRS) 15.7.1 SP203 and lower for LINUX
  • SAP Replication Server (SRS) 15.7.1 all versions for Unix or WINDOWS

Cause

  • Starting with version 15.7.1 SP204 on Linux, SAP Replication Server supports replication into Apache Hive.
  • Attempting to create a connection to Hive on SRS versions prior to 15.7.1 SP204, or using SRS on Unix or Windows, will cause the ERROR #13113

Resolution

  1. Install or upgrade to SAP Replication Server SP204 or higher for Linux.
  2. Create connection to Hive

Symptom

  • You need to understand the process and logic of purchase order archival activity in SAP ERP.
  • Inquiry regarding the functionality of the archival program RM06EW70, including its logic and sequence.
  • Request for detailed information about the criteria and sequence of the programs RM06EV70, RM06EW70, and RM06ED70.

Environment

  • Materials Management (MM)
  • SAP ERP
  • SAP ERP Central Component
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA

Reproducing the Issue

  1. Execute the program RM06EV70 to identify and validate purchasing documents eligible for archiving.
  2. Run the program RM06EW70 to extract and write purchasing document data to archive files.
  3. Execute the program RM06ED70 to delete archived purchasing documents from active database tables.

Resolution

Programs RM06EV70, RM06EW70, and RM06ED70 are part of the MM_EKKO archiving object for purchasing documents in SAP. Here's the detailed logic and sequence: 

1. RM06EV70 - Preprocessing Program

This program identifies and validates purchasing documents eligible for archiving
Logic:

- Scans purchasing document headers (EKKO table) based on selection criteria
- Validates archiving prerequisites:

Document completion status
Retention period compliance
No open items or dependencies
Deletion indicator (EKKO-LOEKZ) status (see kba 1646578 Preprocessing program is RM06EV47 though write / delete program is RM06EW70 / RM06ED70 - SAP ERP and SAP S/4HANA)


- Creates a list of documents ready for archiving
- Generates preprocessing logs and statistics

Key Validation Checks:

- Purchase orders must be completed or marked for deletion
- All goods receipts and invoice receipts processed
- No outstanding delivery schedules for scheduling agreements
- Archiving retention periods met

2. RM06EW70 - Write Program


This program extracts and writes purchasing document data to archive files [1]
Logic:

- Reads purchasing documents identified in preprocessing
- Extracts complete document data including:

Header data (EKKO)
Item data (EKPO)
Schedule lines (EKET)
Account assignment (EKKN)
Conditions and pricing data
History records (EKBE)
Text data


- Writes data to archive files in compressed format
- Creates archive index entries
- Handles subcontracting reservations (RESB) for complex scenarios [3]

Data Extraction Sequence:

1.- Header information extraction
2.- Item-level data collection
3.- Related document references
4.- Condition records and pricing history
5.- Text and attachment data
6.- Cross-reference table updates

3. RM06ED70 - Delete Program


This program removes archived purchasing documents from active database tables 
Logic:

- Verifies successful archive file creation
- Performs final validation checks
- Deletes data from active tables in reverse dependency order:

Condition records
Account assignments
Schedule lines
Item data
Header data


- Updates cross-reference tables
- Handles complex scenarios like subcontracting components 

Deletion Sequence:

1.- Dependent object validation
2.- Condition and pricing data deletion
3.- Item-level data removal
4.- Schedule line cleanup
5.- Header record deletion
6.- Index table updates

Program Execution Sequence
Phase 1: Preprocessing (RM06EV70)

- Timing: Run first to identify archivable documents
- Output: List of documents ready for archiving
- Prerequisites: Documents must be completed and meet retention criteria

Phase 2: Write Phase (RM06EW70)

- Timing: Run after successful preprocessing
- Input: Document list from preprocessing
- Output: Archive files containing extracted data
- Validation: Ensures data integrity during extraction

Phase 3: Delete Phase (RM06ED70)

- Timing: Run only after successful write phase
- Input: Confirmation of successful archive creation
- Output: Cleaned active database with archived documents removed
- Safety: Includes rollback capabilities if issues detected

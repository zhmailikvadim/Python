Symptom

  • Received an error when importing a transfer file.
  • Error message: Exception in Transfer, cause: [301]: unique constraint violated: "TCMP"."IMPORTMAIN__TRANSFER": line [line number] col [col number] (at pos [position number]): "TCMP"."IMPORTMAIN":"TRANSFER": line [line number] col [column number] (at pos [position number]): "TCMP"."IMPORTTERRITORYACCOUNTLIB":"TRANSFERTERRITORYACCOUNT": line [LINE NUMBER] col (at pos [Column position number]): Table(CSQ_TERRITORYACCOUNT), Index(CSQ_TERRITORYACCOUNT_PK) with error: unique constraint violation on pos=[pos number] for table [...]::TCMP:_SYS_SPLIT_CSQ_TERRITORYACCOUNT~11en, key: $trexexternalkey$=[...];[...];[...];YYYY-MM-DD;YYYY-MM-DD already exists as udiv=[...], CSQ_TERRITORYACCOUNT_PK.
  • Memory usage = [memory usage number]M.

Environment

Territory and Quota

Reproducing the Issue

  1. Access Territory and Quota.
  2. Import Territory Account.

Cause

  • At line ""TCMP"."IMPORTTERRITORYACCOUNTLIB":"TRANSFERTERRITORYACCOUNT": line [line number]" in the file, it encountered account data that already exists in the database.
  • The database has a "unique constraint" rule that prevents duplicate account records.
  • When the system tried to insert this duplicate data, it automatically rejected it and rolled back the entire transaction to maintain data integrity.

Resolution

  1. Review the data file and check lines for duplicate account entries.
  2. Remove the duplicate records from the file or update them with unique values.
  3. Verify data source, ensure the file does not contain previously uploaded data.
  4. Re-upload the file, once duplicates are resolved, the file should process successfully.

Symptom

Reactivation steps for "classic" G/L planning in S/4HANA system have been completed as described in SAP Note 2474069 . However when executing transaction code FAGLGP52N to copy plan data in background mode, the job gets cancelled due to error message No. SFIN_FI004 "Function is not available" 

Diagnosis
You are trying to use a function that is not available in SAP S/4HANA Finance.

System Response
This function cannot be executed.

Environment

  • Financial Accounting
  • SAP ERP Central Component
  • SAP ERP
  • SAP S/4HANA Finance
  • SAP S/4HANA
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA

Reproducing the Issue

  1. Execute transaction code FAGLGP52N.
  2. Enter values in fields Ledger, Record Type, Version, Company Code, Fiscal year, Period.
  3. Main menu -> Program -> Execute in Background     

Cause

The customizing is missing in view FINS_DEPR_OBJECT (Deprecated objects which are still active)

Resolution

  1. Run the transaction code SM30.

  2. Enter the view  FINS_DEPR_OBJECT.

  3. Add the record:

    Object type: PROG
    Object name: FAGL_PLAN_COPY

See Also

See also SAP Note 3131158 - GL Allocations: Background jobs cancelled after reactivating G/L Planning

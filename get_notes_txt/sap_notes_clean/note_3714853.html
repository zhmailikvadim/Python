Symptom

  • An error occurs in SAP PO system when processing files picked from an FTP server location in .xml format encoded in UTF-16 LE.
  • The error message displayed is: "Invalid XML character (Unicode: 0x0) was found in the element content of the document."
  • The issue prevents purchase orders from updating in the SAP system.

Environment

  • Release Independent
  • SAP NetWeaver 
  • SAP Process Integration

Reproducing the Issue

  1. Configure the SAP PO Sender File Adapter to process files from an FTP server location.
  2. Ensure the files are in .xml format and encoded in UTF-16 LE.
  3. Include hidden NULL characters (Unicode 0x0) in the XML content of some files.
  4. Attempt to process these files in SAP PO.

Cause

  • The source system generates XML files in UTF-16 LE with NULL characters or incorrect encoding interpretation.
  • XML specification does not allow Unicode 0x00 characters, and SAP PO strictly rejects such payloads.

Resolution

  1. Best approach: Configure the source system to generate files in UTF-8 encoding to avoid hidden NULL characters and ensure smooth processing.
  2. If changing the source system is not immediately possible:

a. Clean the files to remove any hidden NULL characters before SAP PO picks them up.

b. Ensure FTP transfer is set to Binary mode, and the File Adapter in SAP PO remains configured as UTF-16 LE.

  1. If the error persists after implementing the above recommendations:

a. Collect Default Traces and XPI Inspector traces using Example 40. Add the Message ID of an error message to the retrospective trace, enabling the "SAP System State" option while collecting the trace.

b. Open a service connection as per SAP Note #0001916218, providing an HTTP URL connection and a user with read access to the NWA (e.g., NWA_READONLY role).

c. Update corresponding logon credentials in the Secure Area as per SAP Note #000508140 and mention the System ID for which the connection is being opened.

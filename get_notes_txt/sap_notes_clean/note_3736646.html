Symptom

  • Names in a contact person partner function are not fetched correctly from the source.
  • In some cases, Name 2 is fetched from table ADRC and Name 1 is fetched from table ADRP.
  • In other cases, Name 2 is fetched from table ADRP and Name 1 is fetched from table ADRC.
  • The behavior occurs when copying names from ADRC and ADRP into the contact person partner function in sales documents.

Environment

  • SAP ERP
  • SAP ERP Central Component
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA
  • SAP S/4HANA Cloud Private Edition

Reproducing the Issue

  1. Start transaction VA01.
  2. Create a sales order
  3. Enter a material and a quantity.
  4. Save the order.
  5. Check the contact person partner function and observe the determination of Name 1/Name 2/Name 3/ Name 4 

Cause

The order of name fields in sales documents is determined by the country-specific address layout key configured in transaction OY07. Different address layout keys apply different formatting standards, causing Name 1, Name 2, Name 3 and Name 4 fields to appear in a different order and source.

Resolution

  1. Review the country-specific address layout key assigned in transaction OY07 for the relevant countries. Detailed information about address layout keys can be found in List of Address Layouts | SAP Help Portal.
  2. If a different name mapping is required, assign a different existing layout key or create and assign a custom address layout key for the country in OY07. See SAP Note 839466.

Symptom

  • In profile management, a long “valid value” (IU012) status range contains a sub-interval with “value was interpolated” (IU021), resulting in overlapping statuses for the same period.
  • Overlapping statuses raise the question which status is shown in transactions and which entries exist at table level.

Environment

  • SAP S/4HANA, release IS-UT 801 and higher
  • SAP for Utilities as Part of SAP ERP, release IS-UT 600 and higher

Reproducing the Issue

  1. In transaction EEDM01, display a profile and navigate to its values/status display; observe that status IU012 is displayed even though profile value has also status IU021 within the same period.
  2. Display table EPROFVALSTAT for the same profile/timeframe and observe that multiple statuses can exist for the same time interval, with entries consolidated into time slices for identical consecutive statuses.

Cause

Understand logic of status display in EEDM02 and storage of status in table EPROFVALSTAT.

Resolution

  1. Review and adjust status display customizing for transactions that show profile values: SAP Utilities > Energy Data Management > Profile Management > Display Function > Define System Status Display for Profile Values.
    This defines which status is displayed when multiple statuses are present at the same timestamp.
  2. Understand table logic for EPROFVALSTAT: the system stores statuses as time slices, consolidating consecutive periods with the same status into a single interval; a change creates a new slice. Storage is not per value or per day.
  3. Note that a profile value can have more than one status at the same time. Transaction EEDM02 shows one status according to customizing, while EPROFVALSTAT keeps each concurrent status as separate, overlapping time slices

Keywords

Symptom

Unable to over receive a quantity for a Purchase Order in the App F0843 'Post Goods Receipt for Purchasing Document' / 'Post Goods Receipt for Purchase Order'.

Unable to over receive a quantity for a Process  Order in the App F6352  'Post Goods Receipt for Process Order'

Environment

  • SAP FIORI for S/4HANA
  • SAP ERP
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA

Reproducing the Issue

  1. Create a Purchase Order (PO) where the 'Unlimited' indicator is set OR an 'Over Delivery Tolerance' is set for the line item.
  2. Go to the app F0843 'Post Goods Receipt for Purchasing Document' / 'Post Goods Receipt for Purchase Order'.
  3. Enter the PO number.
  4. Post a goods receipt for the required quantity.
  5. Go back into the app - choose Settings, set the indicator 'Use 0 as default' - it does not allow you to post a Goods Receipt for more quantity.

**************

  1. Create a Process Order where the 'Unlimited' indicator is set OR an 'Over Delivery Tolerance' is set for the line item.
  2. Go to the app F6352  'Post Goods Receipt for Process Order' 
  3. Enter the Process Order number.
  4. Post a goods receipt for the required quantity.
  5. Go back into the app - choose Settings, set the indicator 'Use 0 as default' - it does not allow you to post a Goods Receipt for more quantity.

Cause

Overdelivery and Unlimited Delivery are currently not supported by the app 'Post Receipt for Purchasing Document' / 'Post Goods Receipt for Process Order'

Resolution

As a workaround you can use the webgui tile 'Post Goods Movement' or transaction MIGO in the backend.

If using Post Goods Movement or MIGO, choose Settings > Default Values, select 'Propose All Items' - the quantity is then open for input.

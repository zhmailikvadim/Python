Symptom

  • Dump generated in ST22 when running program RFKORD10_PDF..
  • Runtime error: CALL_FUNCTION_PARM_MISSING.
  • Exception: CX_SY_DYN_CALL_PARAM_MISSING.



Environment

  • SAP S/4HANA

Reproducing the Issue

  1. Start transaction SE38
  2. Run the program RFKORD10_PDF..
  3. Select the variant.
  4. Perform the program.
  5. Observe the runtime error dump in ST22.

Cause

  • The PDF form "xxx" assigned to the program RFKORD10_PDF . The path is as below:

  • In transaction SFP, check the interface of the PDF form "xxx".
  • Click the interface and check the parameter name of the import of the interface.
  • It will be observed that it is not aligned with the importing parameter passed for the PDF form.

Resolution

  1. Assign a form with the correct interface to the program RFKORD10_PDF.
  2. For example,configure the form "F140_ACC_STAT_01" whose interface "FI_RFKORD" has the same importing parameter as passed in RFKORD10_PDF.

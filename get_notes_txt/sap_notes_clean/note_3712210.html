Symptom

  1. When viewing the evaluated data for hierarchy report in the Result tab or Workitem Result tab, a dump occurs during evaluation.

  2. When performing Simulation/Extraction results in a runtime error "GETWA_NOT_ASSIGNED" in the report /CLMDV/DVF_SUBMIT_BGD.

Other Terms

DTV , Data Transition Validation, RFDUMLOO, Dump in evaluation result, Dump in evaluation, GETWA_NOT_ASSIGNED, CONVT_NO_NUMBER, Dump in Simulation, Dump in Extraction

Reason and Prerequisites

Reason:

  1. The issue occurs due to a display-related problem while rendering the evaluated data.

  2. The issue occurs due to unassigned field symbol access.

Prerequisites:

  3657674 - DTV Tool 1.7 delivery

Solution

Implement this note.

After implementation:

  1. Reset the the evaluated data and perform the Run Selected action. Once execution is completed, the data is displayed correctly without any dump.

  2. No more dump while performing Simulate/Extraction.

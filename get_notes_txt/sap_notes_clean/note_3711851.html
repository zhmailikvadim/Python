Symptom

When changing the translation for message keys with tokenized variables e.g., {2} and {3}, apostrophes are not processed and appear incorrectly. For example:

Environment

SAP SuccessFactors HCM Suite

Reproducing the Issue

  1. Log into SuccessFactors
  2. In the global search, type Manage Languages
  3. Select a custom translations link or create a new custom translation
  4. Locate a message key with a token e.g., {2}
  5. Select Modify
  6. Enter a custom translation which contains an apostrophe > Select OK
  7. Navigate to the area of the system where the key is located and the apostrophe is missing

Cause

In message keys with tokenized placeholders e.g., {0} or {1}, the apostrophe (') is interpreted as a special escape character. As a result, the apostrophe is not displayed correctly in the UI.

Resolution

To correctly display an apostrophe in a customized message when the original translation has tokenized placeholders, use one of the following supported solutions:

Solution 1: Use Smart Quote (’)

Replace the standard apostrophe (') with a smart quote (’) which is the same character used in the default en-US translation.

Note: Some EU country keyboards do not support direct input of the smart quote character. In this case, copy the character (’) from the default translation and paste it into the customized message.

Solution 2: Use Double Apostrophes ('')

  • Use two apostrophes ('') wherever a single apostrophe should appear.
  • The first apostrophe acts as an escape character, and the output will display a single apostrophe (').

Symptom

When alter a replication definition using rs_send_repserver_cmd is not reflected in RSSD immediately.

Environment

  • SAP Replication Server (SRS) 16.0
  • SAP Replication Server (SRS) 15.7.1
  • SAP Adaptive Server Enterprise (ASE)

Reproducing the Issue

  1. create a rep. definition
    create replication definition xx_def ...
    go

  2. alter xx_def using rs_send_repserver_cmd

    exec rs_send_repserver_cmd 'alter replication definition "xx_def" ...'
    go

  3. check its status at RSSD

    select c.colname,c.coltype,c.status,c.status&0x08 from rs_columns c, rs_objects o where c.objid=o.objid and o.objname = 'xx_def'
    go

Cause

  • Most replication server commands run asynchronously so they don't be updated something changed to the RSSD immediately.

Resolution

  • After changing some things, check them with delay time
  • Use a dummy table to check.
    -Just after changing some thing, insert a heartbeat data to a dummy table and then check whether it's replicated to target dummy table or not.

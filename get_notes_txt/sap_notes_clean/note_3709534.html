Symptom

SAP Note 3563521 introduced new subtypes for occupational illness (U6 and U7) in IT3672.

These new subtypes are followed by new, separate ELDA notification types. Therefore, a method was introduced in this context to derive an ELDA notification type from a given subtype. An invalid (also initial) subtype raises an exception and short dump.

For applications, this leads to problems that ignore the specifications according to T582A, according to which a subtype must be specified when this infotype is processed.

Other Terms

Personnel Administration Austria, IT3672, occupational illness, accident report, ELDA

Reason and Prerequisites

Program error

Solution

The correction is provided by HR Support Package (via CLC if necessary). You can implement an advance correction by implementing the correction instructions using SNOTE.

With this solution approach, the system no longer triggers an exception, but transfers the "invalid subtype" to the further processing of the check class. From there, the system issues a corresponding error message (using the message handler).

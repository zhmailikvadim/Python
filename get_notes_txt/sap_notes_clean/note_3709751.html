Symptom

A system dump occurs when attempting to create a 3PL Request from the Delivery Proposals view.
The request is not created, and the process terminates unexpectedly.

Environment

SAP Business ByDesign

Reproducing the Issue

  1. Navigate to Third-Party Logistics work center.
  2. Open the Delivery Proposals view.
  3. Search for a relevant delivery proposal.
  4. Select the proposal and choose Create 3PL Request.
  5. The system triggers a dump, and the request is not created.

Cause

  • The issue occurs due to an incorrect Sender Company identification maintained in the Third-Party Logistics Integration Communication Arrangement.
  • When the Sender Company UUID does not match the expected configuration, the system fails during the responsibility determination step, resulting in a runtime exception and system dump.

Resolution

Update the Sender Company details in the Third-Party Logistics Integration communication arrangement.

Steps:

  1. Navigate to the Application and User Management work center.
  2. Open the Communication Arrangements view.
  3. Search for and open Third-Party Logistics Integration.
  4. Go to the Business Data tab.
  5. Correct the Sender Company Identification.
  6. Choose Check Completeness.

Save and Reactivate the communication arrangement.

After correcting and reactivating the communication arrangement, retry creating the 3PL Request.

Symptom

ODN is not filled in ERDK-EXBEL during reversal process for invoices that are created before implementing ODN.

Environment

  • SAP for Utilities as Part of SAP ERP, release IS-UT 600 and higher.
  • SAP S/4HANA, release IS-UT 801 and higher.

Reproducing the Issue

  1. In transaction EA13, select an invoice that was created before ODN was implemented (that is, the original document has no ODN).
  2. Run the reversal and check the reversal invoice header: field ERDK-EXBEL remains empty.

Cause

This is standard system behavior: an ODN is assigned to reversal documents only if the original posting documents for the invoice already contain ODN entries (for example, in table DFKKEXTDOC and/or DFKKEXTDOC2). The use of ODN must originate from the original document, and it cannot be applied to the reversal of a document that did not initially include an ODN.

Resolution

  1. Verify whether the original invoicing/posting documents contain an ODN entry (for example, check table DFKKEXTDOC).
  2. If the original documents have an ODN, the reversal will inherit and populate ODN (ERDK-EXBEL) by design.
  3. If the original documents do not have an ODN, the reversal will not receive an ODN. This is working as designed.
  4. If regulatory requirements mandate ODN on reversals of legacy invoices without ODN, evaluate a project-specific enhancement with your implementation team in FI-CA/IS-U to ensure compliance, as this extends beyond the standard process.

Symptom

The report /SIE/SF_ED_LOG_EX_BACK includes a built-in WAIT UP TO functionality to prevent system overload. However, the wait time is unnecessarily applied even when the previous run of ARCHIVE_LINK_EXPORT_ALL only outputs the message "EDIBP data already archived" without performing any actual archiving.

As a result, jobs take significantly longer to complete (sometimes days) due to excessive waiting between run IDs, even when no data is being processed.

Expected Behavior: The WAIT UP TO P_TIME SECONDS should only be triggered when data is actually moved to the archive. This change would optimize job execution time and prevent unnecessary delays.

Additionally, message 203 in message class was missing, and has been now created with this note.

Other Terms

CIP2SAP, runID, WAITING LOOP, ArchiveLink, YYEDIBP

Reason and Prerequisites

Program design

Solution

Implement the correction instructions

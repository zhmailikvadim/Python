Symptom

To perform an initial stock upload in SAP Extended Warehouse Management, you use a CSV file as the basis.

Other Terms

ISLS, Distribution, Intelligent Clinical Supply Management, Initial stock data transfer, /CTCO/ISU, ISU, /CTCO/R_INITIALSTOCKUPLOAD

Reason and Prerequisites

You use SAP Intelligent Clinical Supply Management.

Solution

Attached to this SAP Note, you'll find a ZIP file containing several CSV files. You can use these files as templates for transferring stock data to a warehouse in transaction /CTCO/ISU.

SAP transaction /CTCO/ISU lets you upload stock for Clinical Trial Products that are serialized with Medication Kit numbers. This transaction enhances the SAP standard transaction /SCWM/ISU. The file formats are still based on the SAP standard transaction /SCWM/ISU, with added features to enter Medication Kit numbers. Therefore, the requirements for the SAP standard transaction /SCWM/ISU also apply to transaction /CTCO/ISU and should be considered.

Note the following:

  • These files are templates that you need to adjust to fit your data. You can modify these files in a spreadsheet.
  • There are several templates because different EWM versions may offer more fields for stock data transfer.
  • Ensure that the file you upload matches your EWM system. Adjust the data format, including the number of fields, field sequence, data types, and maximum field lengths, to align with the structure /SCWM/S_STOCK_UPLOAD_EXT. You can view this structure in transaction SE11.
  • Depending on your country settings, a CSV file may use different separators, such as a comma or semicolon. Your file editing software and the SAP system must use the same separator.
  • We recommend checking the file's correctness with a small example before uploading all the data.
  • Also, check the application help for transaction /CTCO/ISU.
  • A new 'Item Category' = '9' (POSTYPE) is introduced to enable entry of Medication Kit numbers for Clinical Trial Products.
  • The structure /SCWM/S_STOCK_UPLOAD_EXT has been enhanced with additional fields STUDY_ID and PROTOCOL_ID to record 'Study' and 'Protocol ID' information for Clinical Trial Products. You can enter Medication Kit Serial numbers using the field UII ('Unique Item Identifier').
  • Transaction /CTCO/ISU can be used to upload stock of non-Clinical Trial Products that aren't serialized with Medication Kit numbers using the same file format attached in this note. Leave the ICSM-specific fields 'Study', 'Protocol ID', and 'Medication Kit Number' (UII) empty in this case. However, the file content and format must still conform to the specifications mandated by the underlying SAP transaction /SCWM/ISU.
  • Please select the corresponding template based on your ICSM release.

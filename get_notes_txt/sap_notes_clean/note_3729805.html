Symptom

Queue stuck in SMQ2 Transaction with Error Message M7022 - 'RE Quantity moved exceeded by & & &' when trying to post Movement Type 261.

Environment

  • Materials Management (MM)
  • SAP ERP Central Component
  • SAP ERP
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA

Reproducing the Issue

  1. Open Transaction SMQ2.
  2. Select the relevant Queue
  3. Open the LUW and display the Error Details to find Message M7022.

Cause

In Table IT_GOODSMVT_ITEM[], the Goods Movement Item is flagged with the Reversal Indicator - Field Rev.MvT, causing Movement Type 262 to be processed.

For the Reservation Items, the Withdrawn Quantity - Field  ENMNG - is zero, so additional Reversal Movements led to Error Message M7022 - 'RE Quantity moved exceeded by & & &' and Queue can not be processed.

Resolution

  • Please check the Goods Movement posted for the relevant Reservation Item, Material, Plant and SLoc and check whether the Quantities and MvT posted correspond to the Withdrawn Quantity stored in RESB-ENMNG. Confirm withdrawals and reversals already posted.
  • If you identify a data inconsistency in Withdrawal Quantities for the Order/Reservation, please refer to: SAP Note 656469 - Correction report for incorrect withdrawal quantities in order, and apply the recommended correction Report as guided.

See Also

  • Refer to: SAP Note 656469 - Correction report for incorrect withdrawal quantities in order

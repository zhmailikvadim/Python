Symptom

  1. I am trying to setup a Restricted Activity Item attachment as a buyer user but all supplier users are being able to view the attachment, even if they do not have the restricted flag on their profile. 
  2. All supplier users without the correct permissions are viewing a restricted attachment from an Activity Item.

Cause

  1. Activity Item Type is not set to 'Add Attachment' 
  2. Activity Item's Completion Type is not set to 'Add Attachment to Worker' or 'Add Form Attachment'

Resolution

Buyers may have activity items with attachments that need to be visible to certain supplier users who are the actor on the activity item, but it should not be visible to all the users within the supplier company.

To use this functionality:

  • The buyer user must be in a user role with the 'Administrative Permission Activity and Offboarding Activity > Manage' flag enabled.
  • The buyer user must have the flag 'View Restricted Attachments' enabled. Only users with this profile flag will see the new field on activity items.

To configure the Activity Item setup:

  • Buyer users with the above permissions must create or edit activity item(s) for the type 'Add Attachment' with the option 'Add Attachment to Worker' or 'Add Form Attachment'. 
  • The Actor on the activity item from the supplier side must be Supplier Account or Supplier Account Manager.
  • When this criteria is met, the new field 'Restricted Attachment visible to Actor?' displays. Select 'Yes' to invoke the new functionality.
  • Add the attachment to the activity item admin object and set the visibility to 'Restricted'.

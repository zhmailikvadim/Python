Symptom

A work order configured to skip the first approval level when rates are within the defined rate lookup range is still routed to the first approval level.

Environment

Production and Non-Production

Reproducing the Issue

  1. Configure an approval group where the first approval level should be skipped if work order rates fall within a rate lookup range (Rule Engine Variable Work Order rate is outside the rate range specified in the rate grid or rate lookup)
  2. Create a work order with multiple rate categories.
  3. Ensure the custom rate lookup contains only a subset of those rate categories (for example, only the base rate) and is missing other rate categories used on the work order.
  4. Submit the work order for approval and observe that it is routed to the first approval level instead of being skipped.

Cause

The affected approval level on the Approval Group is based on a Rule Engine (Approval Rule Script), which works with the variable Work Order rate is outside the rate range specified in the rate grid or rate lookup as a condition and checks if the Work Order Rates are outside of the Minimum and Maximum Custom Rate Lookup range. The rule evaluates all rate categories on the work order against the custom rate lookup. Because not all of the work order’s rate categories exists in the lookup, the rule interprets the rates as outside the lookup range and triggers the respective approval level.

Resolution

  1. Identify all rate categories used on the work order.
  2. Use connectors Pick List Value Upload or Advanced Pick List Value Upload to add the missing rates to the Custom Lookup considering the Lookup Segment combinations

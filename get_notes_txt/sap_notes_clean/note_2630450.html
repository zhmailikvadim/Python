Symptom

After selecting a level in the Advanced Levels window, the inserted level is not displayed in the workspace unless the user expands the node.

Other Terms

AOLAP, A-OLAP

Reason and Prerequisites

Steps to duplicate:

  1. Create a workspace in Analysis edition for OLAP application,
  2. Use a Microsoft SQL Server Analysis Services data source,
  3. Add a hierarchy that contains many levels in an axis and add any measure,
  4. Using a right-click on the hierarchy, click 'Show Levels', then 'Advanced' option,
  5. By default all the levels are selected,
  6. Unselect all of them and enable only the top one, the workspace displays the selected level as expected,
  7. Go back to the Show Levels -> Advanced window and select the bottom one (so that you have two levels selected, the top and the lowest ones),
  8. The workspace still shows only the top level, whereas both levels should be displayed instantly,
  9. But if you click the '+' icon on the left of a value in the cross table, it expands and displays the second level.

Cause:

When selecting level in the multi level selection dialog, latest low level is not expended.

Solution

As a workaround, the user needs to manually select the options or expands manually the node.

The issue has been fixed in the patches listed in the "Support Packages & Patches" section below.
The "Support Packages & Patches" section will be populated with the relevant patch levels once they are released.

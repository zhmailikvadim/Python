Symptom

During payroll processing, wage type /380 (Remuner.normal p/aportes) exceeds the proportional ceiling calculated in wage type /31M(Tope prop. mensual).

Example:

  • /31M (Proportional ceiling): 1,500,000.00
  • /380 (final result): 1,875,000.00

The system appears to increase the base beyond the expected capped value.

Environment

  • HR renewal 

  • SAP S/4HANA 

  • SAP ERP  

  • SAP SuccessFactors Employee Central Payroll 

Cause

This behavior is caused by the configuration of the constant:

  • SESAD – Compensación de topes de Seguridad Social = 1

When this constant is set to 1, the system allows redistribution of unused Social Security ceiling between different base wage types within the same payroll period.

Information from internal system documentation:

"A payroll constant used to indicate which social insurance base wage types can or cannot exchange the unused parts of the limit with each other.

 

If a calendar month includes different Social Insurance base wage types, you must ensure that the deductions and contributions are calculated using the limit for each base wage type as a maximum.

 

However, you may find that in a particular month a wage type exceeds its limit, while another wage type does fails to reach it. In this case, you may have to transfer the differences between wage types that do not reach their limits to the base wage types that exceed their limit.

 

You can do this by entering the value 1 in the Value field of this constant."

Resolution

If the expected behavior is that each base should remain capped (without redistribution):

  • Set SESAD = 0 in table T511K

Result:

No redistribution of unused limit and /380 remains at proportional value.

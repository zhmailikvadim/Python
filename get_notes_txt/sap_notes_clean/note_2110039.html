Symptom

  • System tables sysprocesses and/or syslogshold show a process (spid) executing a 'dump database' or 'dump tran' command with the '$sysdams_clr 'transaction name (tran_name) or
     
  • The ASE errorlog reports the folliowing error:

Error: 6918 Severity: 21, State: 2
The SYSDAMS page that stores the state of page 0 in database '<dbname>' could not be found.
The option 'allow incremental dumps' will be disabled because ASE cannot continue recording page modifications. Please contact Sybase Technical Support.

Environment

SAP Adaptive Server Enterprise (ASE) 15.7 and 16.0

Cause

This issue has been address by SAP CR 760159. Its description reads: The 6918 error, "The SYSDAMS page that stores the state of page # in database '<dbname>' could not be found. The option 'allow incremental dumps' will be disabled because SAP ASE cannot continue recording page modifications", can be raised if the table SYSDAMS gets the very first extent of an allocation unit allocated .

Resolution

  • Upgrade ASE to a release where SAP CR 760159 is fixed:
    • ASE 15.7 SP130 or higher
    • ASE 16 GA PL02 or higher 
    • ASE 16 SP01 or higher 
  • After upgrading, run the following commands in each database experiencing the problem to rebuild the sysdams table:
    dbcc traceon(3604)
    go
    dbcc dam(database_name,’drop’)
    go
    dbcc dam(database_name,’create’)
    go
     
  • Workaround:  There is a potential workaround which is not straight forward or recommended at this time when releases with the fix are available. If upgrading is not feasible in the short term, please request the workaround steps from SAP Product Support via an case.

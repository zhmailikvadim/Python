Symptom

When executing transaction PC_PAYRESULT in SAP GUI for Windows, the application automatically sets the ALV focus on the latest payroll result.

When accessing the same transaction through Fiori Launchpad using SAP GUI for HTML (WebGUI), the ALV grid initially shows the first payroll result, requiring the user to manually scroll down.

The behavior difference creates inconsistency in pay result navigation between the two UI channels.

Cause

A program issue in SAP GUI for HTML where the ALV Grid focus and vertical scroll position are not correctly set when rendering the PC_PAYRESULT result list.

This prevents the WebGUI from automatically placing focus on the latest payroll result.This issue is not specific of PC_PAYRESULT but a more general thing (basis).

Resolution

Apply the correction instructions or kernel patch delivered through:
SAP Note 3610977 – WebGUI Gridview: Vertical scroll position not correctly set

The note provides the fix ensuring that WebGUI behaves consistently with SAP GUI for Windows by automatically scrolling to the latest payroll result.

If the note is not yet implemented in your system, please:

Implement the kernel patch listed in the note, or schedule the correction as part of your next maintenance cycle.

Symptom

When executing the run "Activity-Based Costing - Production" in actual mode, and in MEK3 there is maintained the price condition "Cost Element Rates" for a cost center and account, the calculated cost is classified as fixed and not as variable.

Other Terms

Activity-Based Costing - Production Actual, MEK3, Variable Costs, Cost Element Rates

Reason and Prerequisites

Program error

Solution

History:

Version 1 of 23 February, 2026: New creation

Version 2 of 3 March, 2026: Add supplement

Install the code correction.

Description of Correction:

When executing the run "Activity-Based Costing - Production" in actual mode, and in MEK3 there is maintained the price condition "Cost Element Rates" for a cost center and account, the calculated cost is classified as variable.

Functional Dependencies:

There are no functional dependencies.

Manual Changes:

There are no manual activities.

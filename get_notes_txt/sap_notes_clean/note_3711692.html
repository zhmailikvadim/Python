Symptom

Applying a Windows Database Bundle Patch with command:

opatch apply

fails at prerequisite check "CheckActiveFilesAndExecutables" with errors similar to:

"Verifying environment and performing prerequisite checks...

Prerequisite check "CheckActiveFilesAndExecutables" failed.
The details are:Following active files/executables/libs are used by ORACLE_HOME :<drive>:\oracle\<database-name>\19.0.0

<drive>:\oracle\<database-name>\19.0.0\bin\oravssmsgus.dll
<drive>:\oracle\<database-name>\19.0.0\ORAEVRUS19.dll
<drive>:\oracle\<database-name>\19.0.0\oui\lib\win64\vcruntime140.dll
UtilSession failed: Prerequisite check "CheckActiveFilesAndExecutables" failed.
Log file location: <drive>:\oracle\<database-name>\19.0.0\cfgtoollogs\opatch\opatch<date and time>.log

OPatch failed with error code = 73"

Other Terms

"Windows Bundle Patch" "opatch apply" CheckActiveFilesAndExecutables  "error code = 73" oravssmsgus.dll ORAEVRUS19.dll vcruntime140.dll tasklist "Process Explorer" "Microsoft Defender" MOS 3075245.1 KB736608 MOS 2753449.1 KB106870

Reason and Prerequisites

The error occurs because there are active non-Oracle processes holding locks on dll files under Oracle home.

Especially Security/Monitoring processes such as Microsoft Defender might still hold the dll files.

Solution

Assuming the Oracle database has already been stopped, in order to determine the processes and fix the OPatch failure follow the steps:

1) Double-check that all SAP applications and client tools that may load Oracle dll files (SQL*Plus, SQL Developer, OEM agents pointing to Oracle home, PowerShell sessions that used Oracle modules) are stopped.

2) Run services.msc as Administrator and double-check all Oracle services tied to Oracle Home are stopped. For example:

- OracleService<ORACLE_SID>
- Oracle<Oracle home Key>TNSListener
- OracleVssWriter<ORACLE_SID>
- OracleJobScheduler<ORACLE_SID>

etc.

2) Identify which processes hold the dll files reported by opatch using the command:

tasklist /m <dll-name>

For example:

tasklist /m oravssmsgus.dll
tasklist /m ORAEVRUS19.dll
tasklist /m vcruntime140.dll

If tasklist return the name of such processes, stop them.

3) Since tasklist command might not identify all processes holding the dll files, use next the utility Process Explorer.

If it is not installed on your system, download the utility Process Explorer from Microsoft site:

https://docs.microsoft.com/en-us/sysinternals/downloads/process-explorer

Install it by unzipping the file ProcessExplorer.zip in a working directory.

Execute it from a Command Prompt started as Administrator:

cd <Process Explorer directory>
procexp.exe

Inside Process Explorer click "Find" -> "Find Handle or DDL" and enter the mentioned dll files names. For example search for:
oravssmsgus.dll
ORAEVRUS19.dll
vcruntime140.dll

Stop the process(es) accessing the dll files.

4) Retry applying the Windows Database Bundle Patch:

opatch apply

Symptom

  • The issue involves the configuration of OIDC OP to include claims in the access token, specifically the retrieval of the email address.
  • Access tokens received directly or via the token exchange endpoint do not include the scopes.
  • Clarification is sought on the conditions under which fields will be included in the access token.

Environment

  • SAP Customer Data Cloud
  • OIDC

Reproducing the Issue

Configure OIDC OP to include claims in the access token for retrieving the email address.

Cause

The access token format in the RP configuration is not set to JWE (encrypted JWT), which is required for receiving claims in the access token.

Resolution

  1. Change the access token format in the RP configuration to JWE.
  2. Ensure that the RP uses custom claims from the OP that are returned inside the access token.
  3. Install the certificate received from the OP at the RP for decryption of JWE.

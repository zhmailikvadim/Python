Symptom

Data is not being updated correctly in the production environment during delta runs because the ONLY_UPDATE_CHANGED_DATA parameter is set to FALSE, causing all fields to be updated instead of only the changed ones.

Environment

  • SAP ERP
  • SAP S/4HANA on-premise
  • SAP S/4HANA private cloud edition
  • SuccessFactors Employee Central

Reproducing the Issue

  1. Use the standard integration PA_SE_IN to provide data.
  2. Observe that during delta loads, the ONLY_UPDATE_CHANGED_DATA parameter is set to FALSE.
  3. Verify that the receiver system is updating all fields instead of only the provided fields.

Cause

The ONLY_UPDATE_CHANGED_DATA parameter is not correctly configured to be TRUE during delta runs, leading to unnecessary updates of all fields instead of only the changed data.

Resolution

From the backend side, the parameter ONLY_UPDATE_CHANGED_DATA in the proxy of the upsert request payload is set to false for the initial load (as the initial load is not intended to update changed data) and to true for delta loads. Based on this value received from the ERP system, the middleware logic determines whether to activate the “suppress identical records” flag.

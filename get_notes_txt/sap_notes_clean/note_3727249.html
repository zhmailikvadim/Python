Symptom

In the app "Monitor Order Progress" the evaluation is called for multiple sales orders.

For each selected sales order, where no requirements exist (any more), a warning message MF/326 is raised

("For sales order & &, no requirements exist in the master plan").

which has to be confirmed separately.

Other Terms

MD4C, CO46, Order Report, Monitor Order Progress, Rejected sales order, MF/326.

Reason and Prerequisites

Bad usability in the app handling in the current version.

Solution

Issue is solved with new release 2608.

The new solution shows the sales orders without requirement collectively in a popup,

which only has to be confirmed once.

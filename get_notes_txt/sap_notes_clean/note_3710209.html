Symptom

  • You want to run the SAP FSM Cloud Connector under a specific Windows user instead of Local System.
  • In Windows Services, you configure the Cloud Connector service to Log On using a specific Windows user.
  • When running the Cloud Connector Configuration Wizard, you enter the password for this Windows user.
  • The configuration wizard displays the error message: “The password provided is invalid.”
  • However, the password is correct and works when entered directly in Windows Services.

In the Cloud Connector Remote Control logs, the following error is logged:

Environment

  • SAP Field Service Management (FSM)
  • SAP Business One (SAP B1)

Reproducing the Issue

  1. Configure the SAP FSM Cloud Connector service to run under a specific Windows user instead of Local System.
  2. Start the Cloud Connector Configuration Wizard.
  3. Enter the password for the configured Windows user.

Actual Result

The configuration wizard reports an invalid password, although the password is correct.

Expected Result

The Cloud Connector should successfully validate the credentials and run under the specified Windows user.

Cause

When configuring the Cloud Connector to run under a specific Windows user, the NetBIOS domain name must be included in the username. If the NetBIOS domain name is not provided, the configuration wizard cannot correctly validate the credentials and returns a misleading “invalid password” error.

Resolution

Use the NetBIOS domain format when configuring the Windows service and during Cloud Connector configuration.

Correct username format: NETBIOSDOMAIN\Username

How to find the NetBIOS domain name via CMD

If you do not know your NetBIOS domain name, you can identify it using one of the following commands: 

  • Option 1: Command Prompt: whoami
  • Option 2: Command Prompt: echo %USERDOMAIN%
  • Option 3: Command Prompt: nltest /dsgetdc:%USERDNSDOMAIN%

Symptom

You are using the SAP Business Technology Platform (SAP BTP), Cloud Logging service (CLS). The service instance was deleted and you would like to restore it.

Environment

  • SAP Business Technology Platform (SAP BTP)
  • SAP Cloud Foundry (CF)
  • SAP Cloud Logging service

Reproducing the Issue

You have created a SAP Cloud Logging Instance through SAP BTP Cockpit instance which was later accidentally deleted. Now you would like to request a restore.

Resolution

Points to keep in mind:

  • For each service-instance, SAP takes a snapshots of contents once per day and are kept for a period of seven days. Afterwards the backup files get deleted automatically.

  • We support following restoration procedures for customers:

    1. Restoring deleted contents: If a customer accidentally deleted contents, which made it into one of the above stated backups, we can bring them back inside the same service-instance.
      This scenario requires the service-instance to still exist.
    2. Restoring contents of a deleted service-instance: If a customer deletes a service-instance entirely, we offer restoration to a newly created instance. In this case the customer has to create a new instance and we restore the contents of the previously deleted service-instance to the new one. Restoring a service instance with the same endpoints and instance ID is not supported. In this scenario we always require a new (or other) service instance in the same subaccount to restore contents. 
  • For more details, check: Backup and Restore Custom Contents.

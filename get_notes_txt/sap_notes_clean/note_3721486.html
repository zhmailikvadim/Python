Symptom

When you reverse a MeMi document whose billable item was calculated based on an old tax code configuration, the tax data of the reversal billable item is inconsistent with the original billable item if the tax code configuration has been updated.

This issue is relevant for the market role: DSO.

Other Terms

MeMi Billing, Reversal MeMi Document, Billable Item, Tax Data Inconsistent, /IDXMM/CL_UTILITY, DETERMINE_BIT_TAX_DATA, DET_BIT_ORIG_BY_MEMI_DOC, Tax Code Configuration

Reason and Prerequisites

The error occurred because the system calculated the tax data of the reversal billable item based on the current tax code configuration instead of using the tax data from the original billable item.

Solution

Implement the corrections of this SAP Note or import the relevant support package.

After implementing this SAP Note, the system ensures that the tax data of the reversal billable item remains consistent with the original billable item, even when the tax code configuration has been updated.

If the incorrect reversal billable item hasn't been invoiced, you can reverse it and re-create the reversal billable item. If the incorrect reversal billable item has been invoiced, you need to reverse the invoice document first so that the posting document is also reversed. Then reverse the billable item and re-create the reversal billable item.

You can verify the fix by reversing a MeMi document whose billable item was calculated based on an old tax code configuration after updating the tax code configuration. Check that the tax data of the reversal billable item is consistent with the original billable item.

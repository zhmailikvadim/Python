Symptom

  • Process orders are deallocated after re-reading master data using COR2 transaction code.
  • Operations lose their scheduling status and are sent to APO/ePPDS with a non-scheduled status.

Environment

  • SAP Supply Chain Management
  • SAP ERP Central Component
  • SAP S/4HANA
  • PP-DS / Production and Process Order

Reproducing the Issue

  1. Open COR2 transaction code.
  2. Navigate to the 'Master Data' tab.
  3. Execute the "Read Master Data" function.
  4. Observe that operations lose their scheduling status and are marked as CRTD.
  5. Save the order.
  6. Verify that the non-scheduled status is sent to APO/ePPDS, resulting in deallocation.

Cause

  • Operation statuses from R/3 or S/4 systems for production or process orders are always sent to APO or ePPDS.
  • When the "Read Master Data" function is executed in COR2, operations are no longer scheduled, and this status is transferred to APO/ePPDS. This is mentione in the SAP Note 456756.

Resolution

  1. Before saving the order in COR2, ensure that operations are scheduled again to avoid sending a non-scheduled status to APO/ePPDS.
  2. If the order has already been saved, reschedule the operations in ePPDS using one of the following methods:
  • Use the Detailed Scheduling Board (/SAPAPO/CDPS0) to select affected orders and click "Reschedule".
  • Use the Production Planning Run (/SAPAPO/CDPSB0) with a reschedule heuristic to reschedule multiple orders in bulk.

SAP Note 456756 refers with more details to alternative possibilities, including:

  • SAP Note 370959 - Always schedule operations of R/3 orders
  • SAP Note 793626 - Maintaining scheduled status in APO

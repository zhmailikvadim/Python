Symptom

An error occurs when attempting to load any file into the system via XDL, displaying the message: "File Type not supported."


Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.

Environment

  • SAP Sales Performance Management
  • Incentive Management on Hyperscalers

Reproducing the Issue

1. Attempt to load the file "<TENANTID>_<FILETYPE_<ENVIRONMENT>_<DATE>.txt" into the system via XDL.
2. Observe the error message: "File Type not supported."

Cause

  • The filename does not adhere to the standard format required by Express Data Loader.
  • This deviation from the expected format results in the "File Type not supported" error message.

Resolution

Change the name of the file for the following format: <FILETYPE>_<DEV or TST or PRD>_<yyyymmdd>_<hhmiss>_<xxxxxx>.<file-suffix>  

  • DEV or TST or PRD indicates the intended target environment (although this is ultimately determined by which drop-box is used to transfer the file).  
  • YYYYMMDD is the date of transmission. 
  • HHMISS is a customer-specified time-stamp (example, 1:42:57pm is "134257"). 
  • XXXXXX is a content tag that will be standardized during the data mapping process. 
  • An example file name therefore is TXST_TST_20060629_01_EUSAP1.txt.gz.gpg.

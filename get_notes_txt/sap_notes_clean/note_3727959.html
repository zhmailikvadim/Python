Symptom

  • The Purchase Order creation interface times out when left idle for an extended period.
  • Users are unable to continue creating a Purchase Order after the interface times out.

Environment

SAP S/4HANA Cloud Public Edition

Reproducing the Issue

  1. Begin creating a purchase order in the system interface.
  2. Leave the interface idle for approximately one hour.
  3. Return to the interface and observe that it has timed out and does not allow continuation of the purchase order creation process.

Cause

The timeout occurs because the SAP Fiori Launchpad session automatically expires after a defined period of inactivity. In SAP S/4HANA Cloud Public Edition, session management is controlled by the system to ensure security and optimal performance.

Typically, the session timeout occurs after approximately 30 minutes of inactivity. If no actions such as clicking, navigation, or data entry are performed during this time, the session expires and the user must log in again.

Resolution

Users are advised to follow these best practices:

  1. Save the Purchase Order frequently while creating or editing it.

  2. Avoid leaving the application idle for extended periods.

  3. If the user needs to step away, save the document before leaving the screen.

  4. If the session times out, log in again and reopen the document if it was previously saved.

Symptom

After you upgrade to SAP Intelligent Clinical Supply Management (ISLS) 400 SP02, the SAP Fiori app "Manage Outbound Deliveries" (F0867A) fails to load. The system returns an HTTP 500 internal server error when you attempt to access the OData service metadata.

The application now calls the OData service `/CTCO/LE_SHP_ODLIST_SRV` instead of the standard service `LE_SHP_OD_LIST_SRV` that was previously used. The metadata call doesn't work, even after you activate the new service in transaction `/IWFND/MAINT_SERVICE`.

As a result, you can't access the Manage Outbound Deliveries application for both clinical and non-clinical supply chain processes.

Other Terms

ISLS, ISLSUI, Distribution, Intelligent Clinical Supply Management, /CTCO/LE_SHP_ODLIST_SRV, /CTCO/MD_CUSTOMER_MASTER_SRV_01

Reason and Prerequisites

 You use SAP Intelligent Clinical Supply Management.

The metadata cache of the redefined OData service isn't synchronized with the standard service after system updates.

Solution

Post-Installation ISLS 400 SP02

1. Cleanup the metadata cache of both standard service and redefined service (note 2644890) for Manage Outbound Deliveries:

  • /sap/opu/odata/CTCO/LE_SHP_ODLIST_SRV
  • /sap/opu/odata/sap/LE_SHP_OD_LIST_SRV

2. Reload metadata cache of the new OData service:

  • /sap/opu/odata/CTCO/LE_SHP_ODLIST_SRV/$metadata

Post-Installation ISLS 500 SP00

1. Cleanup the metadata cache of both standard service and redefined service (note 2644890) for Manage Customer Master Data:

  • /sap/opu/odata/CTCO/MD_CUSTOMER_MASTER_SRV_01

  • /sap/opu/odata/sap/MD_CUSTOMER_MASTER_SRV_01

2. Reload metadata cache of the new OData service:

  • /sap/opu/odata/CTCO/MD_CUSTOMER_MASTER_SRV_01/$metadata

Repeat the cleanup and reload of the metadata cache if the issue occurs again for the OData services mentioned above.

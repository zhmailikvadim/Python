Symptom

When working with a production order, a short dump (MESSAGE_TYPE_X) occurs during the component replacement process in transaction /SPDGS/PDAON.
After selecting a replacement option, the application terminates with a runtime error if the order is locked.

Other Terms

IS-MFM, CORE, PDA, / SPDGS / PDAON, CO02, component replacement, order lock, runtime error, MESSAGE_TYPE_X, exception handling, SAP Meat and Fish Management by msg.

Reason and Prerequisites

The component replacement logic does not properly evaluate or handle the enqueue (lock) status of the production order before executing follow‑up processes.
If the order is locked at the time of the replacement call, an unhandled exception is raised, resulting in a short dump.
The issue can be reproduced only when the production order is locked.

Solution

History:
SAP Note created on 11 February 2026.

Description of Correction:
The logic executed during component replacement in /SPDGS/PDAON is enhanced to ensure stable behavior when a production order is locked:

Lock check is performed before the replacement dialog is opened or a replacement variant is processed.
When a lock is detected, the system now issues a controlled, user‑friendly message instead of triggering a short dump.
Exception handling is added around the relevant replacement‑processing routines to prevent MESSAGE_TYPE_X runtime errors.

This ensures that a locked production order no longer causes a termination during component replacement.

Functional Dependencies:
None.

Manual Changes:
No manual changes are required for end users.

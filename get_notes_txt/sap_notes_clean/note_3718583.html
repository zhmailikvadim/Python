Symptom

This KBA collects the Transportation Management (TM) worklist related possible solution proposals.

Environment

  • SAP S/4HANA
  • SAP Transportation Management

Resolution

  1. In most of the cases the POWL/worklist performance issue is solved after end user updates the "Max. No. of Records" number max to 500. This is also mention in the below consulting note 2343430.
    1. Typical issue is when the "Max No. of Records" set to 0. In this case the program tries to process all the records together, and if there are too many records stored in the databases it can happen that the memory can not handle it. 
    2. Other typical issue is when the "Max. No. of Records" is set to too high. The number of requested POWL results should be restricted to a value that is really needed. In general it does not make much sense to select 10,000 documents in the daily work POWL. 500 documents are often sufficient.
  2. For most worklist performance common reasons, see consulting note 2343430 - POWL: Bad performance in SAP TM POWLs
  3. For TM authorization concept related performance issues, see consulting note 1766730 - TM Authorization Concept
  4. It can also happen that end user tries to display too many columns, meaning all possible columns are selected for display. It is highly recommended to display only the necessary columns. Displaying any new column can increase the complexity of the database queries running in the background, meaning a longer processing time.
  5. Other possible issue could be that system contains too many old documents (e.g. Freight Orders/Freight Bookings) without completed Lifecycle status. These old documents should be set to Completed or Canceled. After then archiving should be done for these documents. See more information in SAP consulting note 2062862 - SAP TM Archiving
  6. Check the following SAP help page about POWL performance
  7. Check the following SAP help page about POWL Automatic Refresh 
  8. Differences of POWL_D* relevant reports, please check the following SAP help page which summarizes all reports from D01 to D10: POWL Reports

Symptom

Refresh Web Intelligence document without including mandatory object prompt below error:

"Query script generation failed. See your administrator. Missing dimension 'Struct.' (key 'BH6K61LIOYVUTX9V5B0LTRU4N') in the list of Result Objects of the Query Specification. It is mandatory because it is a BEx Structure."

Environment

  • SAP BusinessObjects Business Intelligence Platform 4.3
  • Web Intelligence (Webi)
  • BEx Query

Reproducing the Issue

    1. Create a Web Intelligence document using a BEx Query or a UNX universe built on a BEx Query. 
    2. Add a hierarchy dimension to the result panel.
    3. Click Run.
    4. An error message appears (when query stripping is enabled):
      "Query script generation failed. See your administrator. Missing dimension 'City & Country' (key '00O2TGUNFVTTTM308N7EZTSMW') in the list of Result Objects of the Query Specification. It is mandatory because it is a BEx Structure."
    5. Add the mandatory dimension object to the result panel.
    6. Click Run again. The query now refreshes without any error.
    7. Remove or delete the column containing the mandatory object from the report canvas.
    8. Click Refresh.
    9. No error appears this time. The object now shows in bold format inside the Main Panel.

    Cause

    Absence of a mandatory object results in an incomplete query structure.

    Resolution

    • This is by-design behavior, resulting from BW-side design requirements and structural dependencies.
    • Mandatory objects available in the BEX structure need to be included in the query to satisfy the mandate defined at the BW level and to complete the structure at the query level for successful execution.
    • In the absence of mandatory objects defined in the BEX structure, the execution will not match the constraint/mandate and will pause the execution of the query.

Symptom

  • The system does not allow maintaining multiple contact person details for different business processes for the same customer in KR DRC solution.
  • XML files cannot capture distinct contact details based on the business process.

Environment

  • SAP S/4HANA

Reproducing the Issue

  1. Run EDOC_COCKPIT
  2. Choose KR eDocument
  3. Display XML file

Cause

  • KR DRC solution does not provide standard logic to determine different contact persons for different business scenarios.
  • The contact person is read from the business place, so only one contact is considered.

Resolution

Use BADI EDOC_ADAPTOR~SET_OUTPUT_DATA to implement custom logic that changes the contact person information after the standard mapping so the eDocument XML reflects the expected contact per business process.

See Also

refer to: SAP Note 3022898 - Contact person numbers is different from customer/vendor and BP

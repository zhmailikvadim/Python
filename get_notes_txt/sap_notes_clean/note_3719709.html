Symptom

In EAM Simplicity, you change the MRP relevance of a reservation from "From release" to "Immediately" for an opened maintenance order. After assigning a non-stock item in the component overview, the system issues message IW 295 "Relevance to MRP is reset to "Immediately"; see long text". The Purchase requisition has been created successfully but the cost relevancy for the component is set to 'not relevant for Costing'.

The symptom can also be found directly using the method DIALOG in the BAPI BAPI_ALM_ORDER_MAINTAIN.

Other Terms

AUDISP NO_DISP, RESBD, AFVGD, RESBD-AUDISP, AFVGD.AUDISP, AFVGD-NO_DISP, RESBD-NO_DISP, RESB-NO_DISP, FORM AUDISP_CHECK, LCOMDF30, MRP relevance, IBAPI_C_AUDISP_CHECK, EAM order, PM order, BAPI_ALM_ORDER_MAINTAIN, LIBAPI_OF39, IW295, RESBD-SANKA, SANKA, RESBD.

Reason and Prerequisites

Reason:
Program error

As a prerequisite, the MRP relevance must be set to "From Release" for the order type in Customizing.

Solution

Implement the program corrections.

Symptom

You are using the Production Material Request (PMR) for the production supply. You are using the following settings in transaction /SCWM/PSASTAGE:

  • Staging Method: Single-Order Staging
  • Qty Calc. Type: Calculation Based on PMRs (field QTYCALMODE_STAGE)

If you do not maintain a unit in transaction /SCWM/PSASTAGE, the quantity proposal for the PMR item is 0. If you maintain a unit that is not maintained for the product in the PMR item, the system raises a message. Therefore you need to create an entry for each unit that you are using for your components. The products require a product group to find the entry with the correct unit. This results in a high effort to maintain the additional settings.

Other Terms

TCI, TCO, /SCWM/TPSASTAGE, /SCWM/CL_EI_MFG_STAGE_INFO

/SCWM/MFG_STAGING, /SCWM/R_MFG_STAGING

Reason and Prerequisites

Not provided previously

Solution

Implement the attached Correction Instruction (CI) or the relevant Support Package.

After the implementation of the correction instruction it will be sufficient to maintain one row without quantity or unit single order items that use the PMR item quantity as staging quantity. The system will use quantity and unit from the PMR instead of the quantity and unit field in the settings.

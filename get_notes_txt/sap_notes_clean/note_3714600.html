Symptom

In an Adaptation Project for the SAP Fiori app "Perform Operation Activity", the implementations of the UI Controller Hooks in the override section of the Controller Extension of the Worklist Controller are not executed.

The UI Controller Hooks in the Worklist.Controller are defined in such a way that the implementation is inserted into the Worklist Controller source code section and executed at runtime.

In an adaptation project, the extension of the worklist controller takes place as a class-based extension, which supports multi-level extensions and enables the implementation of custom methods for expanding the controller with new functions. Lifecycle hooks, such as "onInit" or other public methods of the worklist controller, can be overriden in the override section of the Controller Extension.

However, the implementation of the UI Controller Hooks in the override section is not recognized at runtime, because the corresponding source code location in the worklist controller is still undefined. The inserting of the UI Controller Hook coding from the override section of the Controller Extension into the Worklist controller, is not working in a Controller Extension.

Other Terms

F2898, MPE_ACT_DETS1, i2d.mpe.activitydetails1, Business Application Studio, BAS, PEO, App Extensibility: Perform Operation Activity (Expert), Object Mixin

Reason and Prerequisites

The implementation of the UI Controller Hooks in the override section of the Controller Extension is not recognized at runtime.

Solution

However, it is still possible to implement the UI Controller Hooks.

Option 1:

In the Controller Extension file, modify the original Worklist controller's prototype, as in the example below. The implementations of the UI Controller Hook functions are above the controller extension implementation. The coding of the UI Controller Hook function will be inserted into the Worklist Controller source code. As a result, the UI Controller Hook is not undefined at runtime and will be executed.

sap.ui.define(   
    [
        'sap/ui/core/mvc/ControllerExtension'  
        ,'i2d/mpe/activitydetails1/controller/Worklist.controller'
    ],
    function (
        ControllerExtension
        ,WorklistController
    ) {
        'use strict';
       
        WorklistController.prototype.<UIControllerHookName> = function (<variableName>) {
            // your custom code for the UI Controller Hook
           
        };
        ...
        return ControllerExtension.extend("app.variant.<ControllerExtensionName>", {
        // // this section allows to extend lifecycle hooks or override public methods of the base controller
            override: {
            //  /**
            //   * Called when a controller is instantiated and its View controls (if available) are already created.
            //   * Can be used to modify the View before it is displayed, to bind event handlers and do other one-time initialization.
            //   * @memberOf app.variant.<ControllerExtensionName>
            //   */
                onInit: function() {
                    
                    //  ...
                },
           }
       });
});

Option 2:

In the controller extension file, overwrite the onInit lifecycle hook. An anonymous function or a custom function can be assigned to the individual UI controller hooks. With this approach, functionality can be moved to custom methods, reused, and overriden in further extensions.

 

sap.ui.define(
    [
        'sap/ui/core/mvc/ControllerExtension'
        ,'sap/ui/core/mvc/OverrideExecution'
    ],
    function (
        ControllerExtension
        ,OverrideExecution
    ) {
        'use strict';
         return ControllerExtension.extend("app.variant.<ControllerExtensionName>", {
             metadata: {
            //  // extension can declare the public methods
            //  // in general methods that start with "_" are private
                methods: {
            //      publicMethod: {
            //          public: true /*default*/ ,
            //          final: false /*default*/ ,
            //          overrideExecution: OverrideExecution.Instead /*default*/
            //      },
            //      finalPublicMethod: {
            //          final: true
            //      },
            //      onMyHook: {
            //          public: true /*default*/ ,
            //          final: false /*default*/ ,
            //          overrideExecution: OverrideExecution.After
            //      },
            //      couldBePrivate: {
            //          public: false
            //      }      
                    //Custom function 1
                    _updateSomething1: {
                        public: false,
                        finally: false,
                        overrideExecution: OverrideExecution.After
                    }, 
                    //Custom function 2
                    _updateSomething2: {
                        public: false,
                        finally: false,
                        overrideExecution: OverrideExecution.Instead
                    }, 
                }
            },
            _updateSomething1: function (oData) {
                
               // Implement cutom coding
            },
            _updateSomething2: function (oData) {
                
               // Implement cutom coding
            },
            // // this section allows to extend lifecycle hooks or override public methods of the base controller
            override: {
            //  /**
            //   * Called when a controller is instantiated and its View controls (if available) are already created.
            //   * Can be used to modify the View before it is displayed, to bind event handlers and do other one-time initialization.
            //   * @memberOf app.variant.<ControllerExtensionName>
            //   */
                
                 onInit: function() {
                    
                     // Assign anonymous function to the controller hook
                     this.getView().getController().<UIControllerHookName1> = function (<variableName>) {
                       
                        // Implement custom coding ....
                       
                        // Execute custom function
                        this._updateSomething1(<variableName>);

                     }.bind(this);
                     // Assign custom function to the controller hook
                     this.getView().getController().<UIControllerHookName2> = this._updateSomething2.bind(this);
                },
            }
        });
    }
);

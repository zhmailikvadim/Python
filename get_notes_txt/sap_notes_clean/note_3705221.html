Symptom

  • In Excel planning view there are typos for the time periods / time buckets, for example, instead of January 2030, it is displayed as January 1930.
  • There are formatting issues, or inconsistencies with the formatting and standard used. 
  • Some Months may be in a different language or different formats may be used inconsistently across the different time periods.

Environment

  • SAP Integrated Business Planning for Supply Chain Management (SCM-IBP)

Cause

Resolution

  1. Open the Data Integration Jobs app and select "Download Template".
  2. Choose "Time Periods" for Data Type.
  3. Select the appropriate time profile from the Time Profile dropdown menu.
  4. Select "With Existing Time Periods" and click "Download".
  5. Open the downloaded CSV file in a text editor (do not use Excel to avoid format changes to the PERIODSTART and PERIODEND fields).
  6. Edit the incorrect values in the PERIODDESC field to reflect the correct year descriptions. Save the file without altering its format.
  7. Return to the Data Integration Jobs app and click the "+" button to create a new job.
  8. Select "Time Periods" for Data Type.
  9. Choose the appropriate time profile from the Time Profile dropdown menu.
  10. Click the "Browse" button and upload the edited CSV file.
  11. Ensure the Operation Type is set to "Update".
  12. Optionally, assign a new name to the "Job Name" field.
  13. Verify that the "Allow Extension Only" switch is turned off.
  14. Click the button in the bottom right corner to execute the upload process.
  15. Log off and log back into the SAP Integrated Business Planning Excel Add-in to ensure the updated time profile is reflected.

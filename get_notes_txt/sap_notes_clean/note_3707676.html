Symptom

The following termination occurs in SELECT_2 of a generated query report when you execute a query or QueryProvider (QPROV):

Category Resource bottleneck
Runtime Errors TSV_TNEW_PAGE_ALLOC_FAILED

and/or

Category Resource bottleneck
Runtime Errors SYSTEM_NO_ROLL

Other Terms

SP,INITIAL SIZE,Resource bottleneck,query,QPROV,Non-cumulative, non-cumulatives, NCUM,SELECT_2

Reason and Prerequisites

The allocated memory size for SP was relatively huge and the available memory is used up while storing the fetched records from the database.
This solution does not completely resolve the resource bottleneck issue but shall reduce the memory allocation size.

To check the initial size, open the generated report from transaction RSRT -> Technical Information -> Display Generated Report, then find 'INITIAL SIZE' and see the initial size of SP.

Solution

To solve the issue please apply the Support Package as mentioned below. In urgent cases you can use the correction instructions.

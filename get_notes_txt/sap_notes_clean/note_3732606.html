Symptom

When WSM8 is executed it does not update site data in MARC for new articles that have existing records in table WLK1.

Environment

  • Product: SAP NetWeaver 7.4
  • Listing

Reproducing the Issue

  1. Execute transaction WSM8
  2. Verify that entries are not created/updated in table MARC for the affected articles and sites.
  3. Review transaction WSPL or SLG1 for any errors during the listing process.

Cause

  • Standard behavior

Resolution

WSM8 will only create an entry in table MARC when no WLK1 entry is present. This is a performance feature of the transaction.

If the creation of the MARC record fails during the initial listing run, the report RWDIFFERENCEMARCWLK1_ENHANCED can be used to create the missing article segments.

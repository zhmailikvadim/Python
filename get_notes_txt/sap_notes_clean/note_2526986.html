During contract creation(TAS relevant) if you have 2 output types pointing to the same message type OICO with immediate processing then the  processing log is updated with the same idoc number for both the output types which is incorrect.

Other Terms


VA41, VA42, OICO, ROIKCALE

Reason and Prerequisites

The coding in ROIKCALE did not consider the case of 2 output types calling the same processing routine back to back.

Solution

Apply this note and test.

NO MANUAL STEPS REQUIRED.

Symptom

You have installed SPAM/SAINT version 0093 and you perform an upgrade or an SAP S/4HANA conversion. The phase EHP_INCLUSION aborts with the error "OCS package ... does not match the current software component vector". The log file SPDA_EHP_INCLUSION.LOG contains log messages like the following:

  • 3 ETN245 The following import prerequisites of OCS Package "SAPK-758AGINUIBAS001" have not been met:
    ...   
  • 3 ETN264   In the (alternatively) Requirement set "02":
  • 3WETN493 Not allowed software component "UIX01CA1" rel."*" is installed
    ...
  • 2WETN543 OCS package "SAPK-758AGINUIBAS001" does not match the current software component vector

Note that the names of the OCS Package, software component, or failing requirement set number might differ.

The software component mentioned in the log ("UIX01CA1" in the above example) is no longer installed in the system, so the finding "Not allowed software component "..." rel."*" is installed" is not correct.

Other Terms

EHP_INCLUSION, SPAM/SAINT version 0093, not allowed software component

Reason and Prerequisites

The software component mentioned as falsely installed, was installed in the past. The queue calculation functionality for the add-on inclusion in an upgrade/conversion contains an error in SPAM/SAINT version 0093, which incorrectly interprets the information that a software component was installed in the past and treats this software component as still installed.

Solution

The problem is solved with SPAM/SAINT version 0094.

To avoid such queue calculation problems, import SPAM/SAINT version 0094 on the source release before you start the upgrade/conversion.

If the error already occurred in phase PREP_EXTENSION/EHP_INCLUSION, import SPAM/SAINT version 0094 and repeat the phase in the SUM UI.

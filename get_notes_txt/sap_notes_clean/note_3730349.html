Symptom

  • After running .\replace-url.ps1 with the new Kyma server URL to migrate the agent from Neo to Kyma, the agent remains offline in Kyma environment
  • The agent status shows red/offline in the agent tab UI.
  • DSOD log shows following error
    Caused by: java.net.SocketTimeoutException: connect timed out (local port <port> to address <IP_address> (<URL>), remote port <port> to address 
  • From the agent host, telnet to the Kyma server on port 443 fails.
  • From the agent host, telnet to the Neo endpoint succeeds.

Environment

SAP Cloud Integration for data services

Reproducing the Issue

  1. Follow Step(6) of KBA 3646380 to move the agent from Neo to Kyma
  2. Observe the agent remains offline in Kyma environment

Cause

Network is blocked between agent and Kyma server

Resolution

  1. Contact network team to unblock the connection between agent and Kyma server
  2. Ensure the telnet succeeds to Kyma server

See Also

Symptom

You want to post a document in a FI transaction and you have added custom extension fields using structure INCL_EEW_COBL. For at least one of these fields you have linked a foreign key table to validate the input of that field. When inserting an invalid value in that field, the transaction raises the runtime error MESSAGE_TYPE_UNKNOWN.

Other Terms

COBL, Coding Block, INCL_EEW_COBL, Value Check, CL_COBL_CHECK_INCL_EEW

Reason and Prerequisites

Program error.

Solution

Implement the attached correction instruction in your system. The short dump should not occur anymore after the note has been implemented.

Symptom

You run the UniEmens (RPCUEMI0) report, and the following symptoms occur:

  1. The <DiffAccreditoEvento> element is generated without the entire amount of lost salary for the illness period, comprised of the wage types configured in the Ampliamento cliente per gruppo sorgente voce retributiva (V_T5F99FX) table view for the UniEmens calcolo della retribuzione (UEPY) subapplication, linked with the Malattia >=7 giorni (DAMAL) field of the Moduli: definizione campo (V_T5F99FF) table view.
  2. In case of multiple instances of the <DifferenzeAccredito> section, the <InfoEvento> section could be generated for a <DifferenzeAccredito> instance to which it doesn't belong.

Other Terms

Transaction: PC00_M15_UECALC

&UNIEMENS_TIME (for more information, see SAP Note 2327917)

Carenza

Reason and Prerequisites

Reason

These issues are side effects caused by the solution delivered by SAP Note 3695910.

  1. The system does not consider the amount of lost salary from the days of illness during the carenza period when calculating the <DiffAccreditoEvento> element's amount.
  2. The system generates the <InfoEvento> section under the first <DifferenzeAccredito> instance only.

Prerequisite

The following Support Package (SP) level is the minimum prerequisite:

ReleaseSupport Package
S4HCM 102 HR SP 00
S4HCM 101 HR SP 05
S4HCM 100 HR SP 18
SAP_HR 6.08 HR SP C9
SAP_HR 6.04 (EhP6 and EhP7) HR SP K1

Solution

  1. From now on, the system considers the entire amount of lost salary from the days of illness in the carenza period when it calculates the <DiffAccreditoEvento> element's amount.
  2. From now on, the system generates the <InfoEvento> section under the appropriate instance of the <DifferenzeAccredito> section.

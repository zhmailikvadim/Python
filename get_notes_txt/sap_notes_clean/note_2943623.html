Symptom

A payment run for payables has been created. However, in some cases the payment proposal includes the credit memos but in other cases it does not.

Also, when edit the payment proposal and manually select the credit memo that did not get assigned, a warning message is raised that the credit memo should be paid individually.

Environment

SAP Business ByDesign

Reproducing the Issue

  1. Go to Payables work center.
  2. Go to Payment Run view.
  3. Create a new payment run for a given supplier.
  4. Once the execution finishes, check the generated payment proposals.

The resulting payment proposals did not include all the credit memos that are expected. For the credit memos not included in the payment proposal, the following warning is raised in the application log of the payment run: 'Proposal discarded; Incoming payment data missing for <supplier ID and name>'

  1. Open one of the payment proposals.
  2. Click View All.
  3. Go to Open Items tab.
  4. If the proposal is Postponed, click Actions -> Reset Postpone.
  5. From the list of open items, assign to this payment proposal the credit memo to which the system has raised the above warning message.

The following warning is raised: 'Credit memo <credit memo ID> should be paid individually'

Cause

In the master data of the supplier account, in the Financial Data tab, there is a flag for "Pay Individually", when this flag is set in the master data and invoices/credit memos are created, the system understands that each invoice is expected to be paid separately.

Via Changes tab, see that this flag has been switched on and off a few times. Therefore, for invoice and credit memos that were posted when this flag was OFF in the master data, the payment run clubs them in the same payment proposal. For invoices/credit memos that got posted when this flag was ON, then the payment run will clear them separately and in the cases of credit memos the warning is raised not proposed by the run.

Resolution

The system works as expected.

Symptom

  • Unable to add the half-monthly (1/2 months) setup in company codes that currently do not have this configuration.
  • Error occurs when attempting to activate the 1/2 monthly setup via the configuration path “SPR0 >> Financial Accounting >> AA >> General Valuation >> Fiscal Year Specification >> Use 1/2 Months in the company code”.
  • The system attempts to activate half months for all company codes that share the same fiscal year variant and the error message lists other company codes as a result.
  • Error message AA344 is issued when assets already exist for the company code.

Environment

  • FIN Asset Accounting
  • SAP ERP
  • SAP ERP Central Component
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA
  • SAP S/4HANA Finance
  • SAP S/4HANA Cloud Private Edition
  • SAP Fiori for SAP S/4HANA Finance,
  • SAP Fiori for SAP S/4HANA,
  • SAP Fiori for SAP S/4HANA Cloud Private Edition

Reproducing the Issue

  1. Go to SPR0 >> Financial Accounting >> AA >> General Valuation >> Fiscal Year Specification >> Use 1/2 Months in the company code.
  2. Attempt to activate the 1/2 Monthly setup for a company code that currently does not have this configuration.
  3. Observe that an error occurs, possibly listing other company codes that share the same fiscal year variant and/or showing message AA344 if assets already exist.

Cause

  • Activation of “Use 1/2 Months” applies to all company codes assigned to the same fiscal year variant; therefore, other company codes are included and listed in the error.
  • Message AA344 is correctly raised when assets already exist for the company code; in this situation, changing half-period usage is not supported in standard to prevent inconsistencies.

Resolution

SAP does not recommend changing a Fiscal Year Variant that has already been assigned to a company code and already has some postings. This may create database inconsistencies in your system.
If you must change the fiscal year, SAP recommends to proceed in accordance with Note 0000672255 by all means before any change in your system. As a result, you can avoid inconsistencies.

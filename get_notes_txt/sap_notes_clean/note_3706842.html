Symptom

Some customers do not want to create an Approvable Edit Rule that grants full editing permission for anyone that can see the approvable.  This is usually defined as follows under the Approvable Edit Rule tab:

  • Other
    • All Are True
    • Edit Without Effects

However, some configurations only want certain groups to edit and not broad access.

Resolution

In order to ensure groups can edit, you will need to add the group(s) to their own edit rule.  For example:

  • Any Are True
    • User in Group[xxxxxxx]
    • User in Group[yyyyyyy]
    • User in Group[zzzzzzzz]

Feel free to add as many groups as you need.  This will then grant users in that group the ability to see the Edit button on the given approvable.

Please note, unless you have the above configuration as an edit rule, creating an edit rule with actions that are only performed during an edit will not work.

Example:

  • All Are True
    • User in Group[xxxxxxx]
    • Field Changed[yyyyyyy]

This condition will return false since changing a field is only allowed during an edit.  However, to first see the Edit button, you must first build a condition that grants that group access.

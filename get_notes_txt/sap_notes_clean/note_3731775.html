Symptom

  • After reporting a no-show or terminating, the employee continues to appear as the occupant of the position in the position org chart.
  • There are no data issues in the Job Information or Employment Details of the affected user, and termination is recorded in the portlet.

Environment

SAP SuccessFactors Employee Central

Reproducing the Issue

  1. Report a no-show for a new hire or rehire.
  2. Open Position Org Chart or Manage Positions and observe the employee still displayed on the position.

Cause

Invalid position records with incorrect end dates exist and can be seen in Data Inspector (Table_GENERIC_OBJECT_T) for the affected position, causing a stale occupant to remain on the position.

Resolution

  1. Use Data Inspector to review the Position object data (Table_GENERIC_OBJECT_T) for the affected position and confirm invalid or inconsistent end dates/history.
  2. Go to Import and Export Data and export the Position object to obtain the current structure and required columns.
  3. Prepare a Position import file that contains the correct, clean history for the affected position.
  4. Import the Position object using Full Purge mode to overwrite the existing position records.
  5. After the import completes, verify in Position Org Chart and Manage Positions that the position no longer shows the no-show employee as occupant.

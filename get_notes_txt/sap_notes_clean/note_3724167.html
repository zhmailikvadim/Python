Symptom

This is an FAQ for SAP security note 3714585 - [ CVE-2026-27685] Insecure Deserialization in SAP NetWeaver Enterprise Portal Administration

Other Terms

Privileged accounts, Transport Application

Solution

You need to implement SAP Note 3714585 - [ CVE-2026-27685] Insecure Deserialization in SAP NetWeaver Enterprise Portal Administration, and would like to clarify some queries before applying the note.

Q1: What can I do if I’m unable to immediately apply the patch or my version of the product is lower than the ones that has a patch?
Answer: Apply 3660659 - [CVE-2025-42944] Security Hardening for Insecure Deserialization in SAP NetWeaver AS Java, to block common deserialization gadget classes at JVM level, preventing many known exploitation chains.

Q2: Is there any workaround or mitigation available if the patch cannot be applied?
Answer: There is no applicable method to isolate or disable the vulnerable endpoint independently of the patch. However, this endpoint is already restricted and can only be accessed by highly privileged administrator users.
Ensure that:

  • All privileged accounts are actively monitored and managed.
  • Administrator credentials are securely stored and not exposed or leaked.

Q3: What are the privileged accounts that should be monitored and managed as ones that can access the vulnerable end point?
Answer: The relevant users that might be influent are ones that are assigned to any of the following

  • UME Group: Administrators
  • UME Role: Administrator
  • Portal Role: pcd:portal_content/administrator/super_admin/super_admin_role
  • Portal Role: pcd:portal_content/administrator/system_admin/system_admin_role
  • Portal Role: pcd:portal_content/administrator/content_admin/content_admin_role

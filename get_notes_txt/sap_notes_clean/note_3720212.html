Symptom

  • Quick Search in the quote list page is working inconsistently across different users.
  • Some users see "contains" criteria while others see "exact" criteria, despite same user type.
  • Searching for specific terms or quote numbers does not yield results for some users.

Environment

SAP CPQ

Reproducing the Issue

  1. Login as a user.
  2. Navigate to the quote list page and select "other quotes".
  3. Search for a specific term or quote number.
  4. Observe that no results are returned for some users, even though the settings are consistent across user type.

Cause

Bug in the application related to the quote list filter configuration and existing saved filters, causing corruption in the quick filter search functionality.

Resolution

  1. Delete all already saved filters on the quote list page.
  2. Perform a hard reload of the quote list page to remove caching.
  3. Recreate filters and save them again.
  4. Note: A code fix is expected in the 2605 release to permanently resolve the issue.

Keywords

quick search, quote list page, filter, inconsistent search, contains criteria, exact criteria, CPQ

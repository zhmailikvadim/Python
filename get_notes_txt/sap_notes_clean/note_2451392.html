Symptom

All the BOE Servers except Central Management Server, Adaptive Processing Server and WebApplication Container Server doesn’t start after applying patch to update BOE to 4.2 SP04.

All BOE server doen't start after enabling SSL in SIA

In Non-windows Platform, FIPS must be enabled if SSL is enabled at SIA

Other Terms

SIA, SSL, CORBA, BI4.2 SP04, Servers failing, SAP Business Objects, FIPS, SSL in SIA

Reason and Prerequisites

On Non-Windows Platform , BOE requires FIPS to be enabled if SSL is configured for BOE Servers in SIA (CORBA).

Solution

In 4.2SP04  for non-windows platforms (Linux,AIX,Solaris) if you want to configure (CORBA) SIA SSL, make sure your servers are running in FIPS mode.

Details:

For a full build 4.2SP04 installation FIPS will be enabled by default.

But in case:

If you are doing a patch update from 4.1 SPx to 4.2 SP04 or if previously you have disabled FIPS manually, in such a case you should enable FIPS beafore enabling SSL in SIA (CORBA)

Steps to enable FIPS

  • Navigate to <InstallDir>/sap_bobj
  • Run ./stopservers
  • Open ccm.config file
  • Add   the text '-fips' from the SIA Node property list
  • Run ./startservers

Note: For windows platform CORBA SSL can still work in non FIPS mode as well

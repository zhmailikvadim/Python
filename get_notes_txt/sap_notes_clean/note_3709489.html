Symptom

After running the Payment program F110 or F111, the payments are not considered for Bank Communication Management (Cross-payment run) and the payment file is created.

Environment

  • Bank Communication Management
  • SAP ERP
  • SAP Enhancement Package for ERP
  • SAP S/4HANA

Reproducing the Issue

  1. Run F110/F111 transaction.
  2. Observe that cross-payment run is not performed.
  3. Verify that the payment file is automatically created without undergoing the BCM process.

Cause

  • Missing configuration in OBPM5 transaction to activate Bank Relationship Management functionality.
    or
  • Enhancement implementation via BADI FIBL_MPAY_RESERVATION is used, excluding certain payments from BCM processing based on criteria such as company code or payment method.

Resolution

  1. Check the configuration in OBPM5 transaction. Ensure that the corresponding entries are maintained to activate Bank Relationship Management functionality, enabling program SAPFPAYM_MERGE to create payment batches.

  2. Verify if BADI implementation FIBL_MPAY_RESERVATION is being used in the system. This BADI excludes certain payments from BCM processing and directly creates payment files without an approval process. Further details can be found in the SAP note 2599466 - BCM: Activation per company code or payment method (BAdI implementation)

Symptom

  • Transport ID meet Active failed during deployment via transport management.
  • Activation failed in the frontend despite being activated (SAP_SYSTEM)  in the backend.
  • Deployment of version actually trigger via auto distribution.

Environment

  • SAP Cloud for Customer
  • SAP Business ByDesign

Reproducing the Issue

  1. Login to the tenant.
  2. Navigate to Transport Management -> All Transport Request -> Transport ID 10 (Solution - Version 111 Assembled).
  3. Check the transport log.
  4. Observe that activation failed.

Cause

Deployment of version actually trigger via auto distribution.

Resolution

As both 'client 001' and 'client 002' are hosted on system `XXX`, they share runtime artifacts, meaning auto distribution from both clients.

Symptom

  • A termination is issued from transaction KOB1 (or similar) with the exception "INTERNAL_PROBLEM".
  • Runtime error: RAISE_EXCEPTION.
  • ABAP program: SAPLKAEP.
  • Application component: CO-OM.

Environment

  • Controlling (CO)
  • SAP R/3
  • SAP R/3 Enterprise 4.7
  • SAP ERP Central Component
  • SAP ERP
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA
  • SAP S/4HANA Finance.

Reproducing the Issue

  1. Access transaction KOB1 or a similar line item report.
  2. Do not directly select any orders at all or include a big range of orders.
  3. Execute the report with archive access.
  4. Observe the termination with the exception "INTERNAL_PROBLEM".

Cause

  • The termination occurs due to excessive data involved in archive processing. Specifically, the selection criteria result in a WHERE statement with a huge amount of conditions, which cannot be processed by the archive access. In general, line item reports are not designed for mass data processing but to access specific details

Resolution

  1. Reduce the data selection spectrum by choosing a smaller range or specific order numbers (or other objects for other line item reports.
  2. Deactivate archive access during transaction execution: Navigate to Extras > Data Source > Deactivate the archive checkbox.
  • Alternatively, deactivate archive access completely as per SAP Note 521604.
  • For mass analysis of costs, use reports based on total table entries, such as transaction S_ALR_87012993.
  • If detailed information is required, navigate from the mass report (S_ALR_87012993) to transaction KOB1 for specific orders or cost elements.

Symptom

  • An error occurs when using Excel Data Loader to upload a file to Position's Workspace. 
  • The error message displayed is: "Value is Required for the Target Compensation Value Field." 

Environment

  • SAP Sales Performance Management
    • Incentive Management on Hyperscalers
    • Incentive Management on Legacy Oracle
    • Incentive Management on Legacy Hana

Reproducing the Issue

  1. Go to Incentive Management Admin.
  2. Navigate to any Workspace that contains fields associated with a Unit Type.
  3. Attempt to update the Unit Type of a field without providing a corresponding value for that field, using either the User Interface (UI) or Excel Data Loaders.
  4. Example:

    • Excel Data Loader scenario:

      • Use the Position Workspace Excel Data Loader template and populate the Unit Type for the Target Compensation Value field, while leaving the Target Compensation Value itself as null.
        Upon uploading the file to the Position workspace, the error described in the Symptoms section is observed.

    • UI scenario :
      • Navigate to the Transaction Workspace and update the Unit Type for GenericNumber1 through the UI without providing or associating a value for GenericNumber1.
        The UI will fail to save the record and will display a validation or exception error.

Cause

In Incentive Management, fields that support Unit Types require a value to be provided whenever a Unit Type is specified. 

Resolution

  1. When updating any field that is associated with a Unit Type, ensure that a valid value is provided at the same time. The Unit Type and its corresponding value must always be populated together.

  2. When using the User Interface (UI), enter a value for the field, selecting or updating the Unit Type.

  3. When using Excel Data Loaders, ensure that both the Value column and the Unit Type column are populated for the same field.

  4. If a value is not required, remove or leave the Unit Type empty as well.

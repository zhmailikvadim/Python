Symptom

  • Upon upgrading to SP15 or higher, MDM communication with iOS devices is lost
  • The following error is observed in the Afaria Message Log: "APN0017: Enqueue request failure, undefined certificate thumbprint: PushType=MDM, Client=<UDID>"

Environment

  • SAP Afaria 7.0 SP15 or later
  • iOS devices

Reproducing the Issue

  1. Upgrade a working Afaria 7.0 SP14 or earlier system to SP15 or later.
  2. Attempt to send an outbound notification such a Lock to an iOS device. The device does not lock, and the error mentioned under Symptom is seen in the logs.

Cause

  • In SP6, two new fields were added to the database table A_IPHONE_SETTINGS: MDMPushCertThumbprint and AppPushCertThumbprint. This was done to avoid conflicts when the APNS certifcates for Mobile Device Management and/or for Custom-Signed Afaria Application where renewed without deleting the old version of the certificate.
  • In SP15, a new feature, APNS Queuing was introduced to streamliine outbound notifications to iOS devices.
  • Occasionally, on upgrading to SP6 or later, the above database values are not populated in the database. APNS queuing depends upon A_IPHONE_SETTINGS.MDMPushCertThumbprint.

Resolution

  • Reinstall the APNS Push Certificate (for Mobile Device Management) and the APNS Push Certificate  (for Custom-Signed Afaria Application) if applicable.
  • If there is a problem in the UI with reinstalling the certificates, please follow these instructions to update the database manually:
    • Using Powershell:
      1. From a command prompt, type "powershell".
      2. Enter the following lines:
        $Thumbprint = (Get-ChildItem -Path Cert:\LocalMachine\My | Where-Object {$_.Subject -match "APSP:"}).Thumbprint;
        Write-Host -Object "$Thumbprint";
        Right-click in the command window and choose Mark.
      3. Left-click, hold and drag to select the thumbprint returned.
      4. Right-click again to copy the thumbprint to the clipboard.
      5. In the SQL Server Management Studio (SQL Server) or Interactive SQL (SQL Anywhere) run the following query:
            UPDATE A_IPHONE_SETTINGS
            SET MDMPushCertThumbprint = '<thumbprint>'
            WHERE TenantID = <tenantid>
        Paste the thumbprint in place of <thumbprint>. If you only have one tenant, the WHERE clause can be omitted.
        In the case of SQL Anywhere, follow this with:
            commit;
      6. If you have an APNS Push Certificate (for Custom-Signed Afaria Application) defined, and the certificate information continues to display incorrectly, enter the following lines in the command prompt:
        $Thumbprint = (Get-ChildItem -Path Cert:\LocalMachine\My | Where-Object {$_.Subject -match "Apple Production IOS Push Services:"}).Thumbprint;
        Write-Host -Object "$Thumbprint";
      7. Left-click, hold and drag to select the thumbprint returned.
      8. Right-click again to copy the thumbprint to the clipboard.
      9. In the SQL Server Management Studio (SQL Server) or Interactive SQL (SQL Anywhere) run the following query:
            UPDATE A_IPHONE_SETTINGS
             SET AppPushCertThumbprint = '<thumbprint>'
            WHERE TenantID = <tenantid>
        Paste the thumbprint in place of <thumbprint>. If you only have one tenant, the WHERE clause can be omitted.
        In the case of SQL Anywhere, follow this with:
            commit;
      10. Restart the Afaria server.

    • Using MMC:
      1. Open the Microsoft Management Console by typing "mmc" in the Run box or a command prompt.
      2. File -> Add/Remove Snap-in
      3. Select Certificates and click Add
      4. Select Computer Account and click Next
      5. Select Local Computer Account and click Finish, then click OK.
      6. Expand the folder Certificates (Local Computer) -> Personal -> Certificates
      7. Locate the certificate that corresponds to the APNS certificate in iOS Notifications. Be sure to match both the name and the expiry date.
      8. Double-click on the certificate.
      9. Under "Details" select Thumbprint.
      10. Copy the 40-character thumbprint to a text editor.
      11. Remove the spaces and copy the resulting string to the clipboard.
      12. In the SQL Server Management Studio (SQL Server) or Interactive SQL (SQL Anywhere) run the following query:
             UPDATE A_IPHONE_SETTINGS 
             SET MDMPushCertThumbprint = '<thumbprint>'
             WHERE TenantID = <tenantid>
        If you only have one tenant, the WHERE clause can be omitted.
      13. If you have an APNS Push Certificate (for Custom-Signed Afaria Application) defined, and the certificate information continues to display incorrectly, it may be necessary to repeat steps 7-11 for the App Push certificate and run the following SQL query:
             UPDATE A_IPHONE_SETTINGS 
             SET AppPushCertThumbprint = '<thumbprint>'
             WHERE TenantID = <tenantid>
        If you only have one tenant, the WHERE clause can be omitted.
      14. Restart the Afaria server.

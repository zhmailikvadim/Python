Symptom

You want to export a signed key pair (private key and certificate) from one SAP Cloud Integration tenant keystore and import it into another tenant. The key pair was previously imported into the source tenant.

Environment

  • SAP Integration Suite
  • SAP Business Technology Platform
  • Cloud Integration

Cause

For security reasons, Cloud Integration does not allow exporting private keys from the tenant keystore. Only public certificates can be downloaded.

Resolution

  • Reuse the original PKCS#12 (P12/PFX) file that was used to import the key pair and import it into the target tenant.
  • If the original file is unavailable, create a new key pair outside CI, obtain the certificate from your CA as needed, package it as PKCS#12, and import it into the target tenant. Ensure partner systems are updated with the new public certificate.

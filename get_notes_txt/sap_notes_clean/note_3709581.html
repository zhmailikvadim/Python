Symptom

During the installation of SAP BusinessObjects BI Platform Server 4.3 SP0 on Windows Server 2022, the System DSN list does not display any DSNs under:

Configure CMS Repository Database → SQL Server (ODBC)

Even when:

  • Microsoft SQL Server 2022 is used
  • ODBC DSN is created successfully
  • DSN connectivity test works
  • Correct credentials and database details are provided

System DSN remains empty/unlisted.



Environment

 

  • Product: SAP BusinessObjects BI Platform Server 4.3 SP0
  • Operating System: Windows Server 2022
  • Database: Microsoft SQL Server 2022 

Reproducing the Issue

 

  • Install Microsoft SQL Server 2022 (16.0.4200.1).
  • Install the same SAP BO 4.3 SP0 patch as used by customer.
  • Create a new empty MSSQL database.
  • Configure a System DSN pointing to this database.
  • Verify successful connectivity via ODBC.
  • Launch BO installation in Full and Custom modes for checking.
  • At the CMS Repository Database configuration step, the System DSN field remained empty.

Cause

The SAP BO 4.3 SP0 patch used is not compatible with MSSQL Server 2022, which prevents the installer from detecting the System DSN.

Resolution

Install a newer version of SAP BO BI Platform that supports Microsoft SQL Server 2022.

Keywords

System DSN is empty , DSN detection failure, Compatibility issue, Microsoft SQL Server 2022

Symptom

  • Existing user groups are unintentionally removed during provisioning in SAP Identity Management (IDM).
  • No changes have been made to SAP standard BAPIs such as BAPI_USER_CREATE1, BAPI_USER_GET_DETAIL, or BAPI_USER_CHANGE.
  • The issue occurs when roles are provisioned to ABAP systems.

Environment

  • Identity Management 8.0

Reproducing the Issue

  1. Provision roles to ABAP systems using SAP IDM.
  2. Observe that existing user groups are removed during the provisioning process.

Cause

  • The standard behavior of SAP IDM when provisioning to ABAP systems is to overwrite all local assignments of roles and privileges in the AS ABAP system with the central assignments.

Resolution

  1. Ensure SAP IDM sends all valid data, including user group information, with each provisioning action.
  2. Review and update the script used for provisioning to include user group data.
  3. Refer to the SAP documentation on restrictions for AS ABAP/SAP Business Suite System Connectors for detailed guidance.
  4. Use reconciliation jobs to synchronize authorization assignments between SAP IDM and local AS ABAP systems.

See Also

  • Refer to: Restrictions for AS ABAP/SAP Business Suite System Connectors.

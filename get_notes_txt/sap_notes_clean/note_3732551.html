Symptom

  • When launching a SAP GUI for HTML (WebGUI) task from the Portal Universal Worklist (UWL), the WebGUI screen remains stuck on the loading splash screen.
  • No error message is displayed in the UI; instead, the screen only shows the three-dot loading indicator indefinitely.
  • The kernel version of the backend system has been upgraded.

Environment

  • SAP Netweaver Enterprise Portal
  • Universal Worklist

Reproducing the Issue

  1. Sign in to the Enterprise Portal.
  2. Open the universal worklist and select a work item that launches a backend transaction via WebGUI.
  3. Observe that the WebGUI loading page appears and does not complete.
  4. Confirm that no error message is displayed.

Cause

When the "Handover Portal Stylesheet" property of the UWL launch iview is enabled, the sap-cssurl parameter is passed from the Portal to the backend WebGUI.
After a backend kernel upgrade, this behavior causes the page to hang when launched from UWL

Resolution

The UWL contains a transaction iView in the portal content that is used to launch backend transactions during portal runtime.
Disable the “Handover portal stylesheet” property in the “UWL – Launch SAP Transaction” iView.
You can follow the steps in the KBA 1725729 - Transaction issue in SAP GUI for HTML (WebGUI) using UWL

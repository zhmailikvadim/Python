Symptom

The mobile user's location is not updated, which is incorporated in the CA_LOCATION_UPDATE feature.

This error may occur if SAP note 3564088 ("oMDO Processing: No data output should be returned when oMDO is not enabled for any feature assigned to") has been applied to your system.

This note provides the configuration correction for SAP Service and Asset Manager versions 2405, 2410, 2505, and 2511.

Other Terms

S4MOBADDON-8164, /SYCLO/MDW00L, FSM

Reason and Prerequisites

Missing Configuration

Solution

Follow the steps to import configuration change to SSAM 2405, 2410, 2505, and/or 2511 mobile application:

  1. Download the attached bin file matching the SSAM version. So, for 2405, download "CA_LOCATION config 2405.bin"
  2. Go to SAP GUI, execute the program /MFND/CORE_CONFIG_IMPORT_PROG in transaction SE38
  3. Provide the filepath in the 'Filename' parameter of the Selection Screen (f4 help for file browser)
  4. You may optionally provide a 'Customizing Request' to capture the changes
  5. Check the 'Test Run' checkbox and execute the program to ensure no errors occur
  6. Successful output should contain the rows modified for each table. All changes are rolled back when 'Test Run' is checked
  7. Return to the Selection Screen
  8. Uncheck 'Test Run' and then execute
  9. Successful output should contain the rows modified for each table and the transport in which the changes were saved to, if provided
  10. Via ConfgPanel->Component Assigments->SAP_SERVICE_ASSET_MANAGER_nnnn-> oMDO Assignment, verify feature ID CA_LOCATION_UPDATE has oMDO SAMnnnn_USER_INFO_ONLINE as both In-Scope and Active

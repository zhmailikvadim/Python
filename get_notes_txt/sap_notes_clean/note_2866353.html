Symptom

The indexserver crashes with a callstack similar to the one below:

[CRASH_STACK] Stacktrace of crash: (0000-00-00 00:00:00 000 Local)
----> Symbolic stack backtrace <----
0: ptime::FederationQuery::appendIDXCHGOutputCols
1: ptime::FederationQuery::execute
2: ptime::FederationQuery::do_open
3: ptime::Table::open
4: ptime::Itab_materializer::do_open
5: ptime::Table::open
[...]

Reason and Prerequisites

Reason:

Due to a programming error in SAP HANA, an internal data structure (itab) does not contain the expected number of columns during join relocation.

Affected Releases:

  • SAP HANA 2:
    • Revisions of SPS00 - SPS01
    • Revisions <= 024.05 (SPS02)
    • Revisions <= 033.00 (SPS03)

Prerequisites:

  • An SQL statement is executed that involves virtual tables as base tables. This statement uses join relocation which means that HANA decides whether or not it might be beneficial that the join is executed on the remote source rather than on the local HANA system.

Solution

Apply one of the following SAP HANA Revisions:

  • SAP HANA 2:
    • Revisions >= 024.06 (SPS02) 
    • Revisions >= 034.00 (SPS03)
    • or higher

  • or higher

Workaround:

  • Disable join relocation by setting the following parameter:

    ALTER SYSTEM ALTER CONFIGURATION ('indexserver.ini', 'SYSTEM') SET ('smart_data_access', 'enable_rid_for_join_relocation') = 'false' WITH RECONFIGURE;

    Starting with HANA 2.0 SPS03 comments should be added:

    ALTER SYSTEM ALTER CONFIGURATION ('indexserver.ini', 'SYSTEM') SET ('smart_data_access', 'enable_rid_for_join_relocation') = 'false' WITH RECONFIGURE COMMENT 'SAP Note 2866353. Revert after upgrade to a revision containing the fix';

Remarks:

  • Disabling join relocation might have a performance impact on statements against remote sources.

Disclaimer

The SAP HANA configuration parameter [smart_data_access] enable_rid_for_join_relocation is an internal parameter and may only be used in the context of this SAP Note. Using such a parameter requires that the SAP HANA system has the revision specified in the SAP Note. Using internal parameters outside of this given scope is not supported. Please unset this parameter once you have upgraded to a revision containing the permanent solution.

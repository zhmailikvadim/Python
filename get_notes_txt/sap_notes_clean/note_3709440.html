Symptom

1. New Attestation Feature

2. Approval Hub Fixes : When Approver was trying to approve the item , approver was unable to do so because of Warning Pop Message on Application.

In the Approval Hub Application, for all the actions & Headers which we were showing in my worklist or in the details page, The entities to be called were derived from the category process configuration.

The issue was that the filtration was happening considering the process & category. This meant that if for any combination of category & process if there were 2 row items, it was fetching the first-row item all the time. Since for any combination there can be 2 sub process.

Other Terms

Financial Control , FCA App, Approval Hub

Solution

The fix has been done to cater the above-mentioned issue by considering the sub process along with category & process. So, this solution eliminates the makes sure that unique record is picked when filtering the category process configuration.

Another Fix has been done to allow approver to approve the item when workflow reaches at its level.

ATTESTATION Feature:

A configurable attestation has been introduced for the Approve and Reject actions in the Approval Hub application. When users trigger either action from the My Worklist or Item Details page, a dialog now displays an attestation message with a checkbox based on backend configuration.

The attestation’s visibility, mandatory behavior, and display text are dynamically controlled via backend settings for each action and item type, ensuring compliance requirements can be enforced without additional frontend changes.

 

New Elements

  • Attestation section added within the Approve and Reject dialogs.
  • Checkbox enabling users to explicitly acknowledge the attestation before proceeding.
  • Dynamic attestation text displayed when configured.
  • Mandatory mark when the attestation is required to be confirmed by the user.

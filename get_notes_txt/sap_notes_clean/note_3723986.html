Symptom

Zero amounts in group and transaction currencies are displayed in draft mode.

After posting, amounts in group and transaction currencies are populated with specific values (e.g., 10).

Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.

Environment

SAP S/4HANA Cloud Public Edition

Reproducing the Issue

  1. Post values only in local currency (LC).
  2. Save the document.
  3. Observe that the same amount is copied to transaction currency (TC) and group currency (GC).
  4. Note that zero amounts are displayed in draft mode for TC and GC.
  5. After posting, amounts in TC and GC are populated with specific values.

Cause

When "Post in LC" is selected, the system logic dictates that LC values are posted in the app Post Group journal entries. 

If "Post in GC" or "Post in TC" is not selected, the input boxes for these currencies are grayed out, preventing new data entry.

If TC equals LC and the option "Post in Transaction Currency" is not selected for the document type, the TC value is derived from the leading currency. Otherwise, the original source TC value is retained during posting.

The same logic applies to group currency.

Resolution

Confirm that the observed behavior aligns with system logic.

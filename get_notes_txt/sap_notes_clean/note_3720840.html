Symptom

  • The issue occurs when extending a material to a new valuation type within the same valuation area in the target system.
  • Field MBEW-MLMAA is not updated in the target system.
  • Error message: "C+039 ML currencies are changed in CO".

Environment

  • SAP ERP Logistics (LO)
  • SAP R/3
  • SAP R/3 Enterprise
  • SAP ERP Central Component
  • SAP ERP
  • SAP Enhancement package for SAP ERP
  • SAP Enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA Cloud Private Edition

Reproducing the Issue

  1. Use transaction code SE16N.
  2. Access table MBEW.
  3. Extend a material to a new valuation type within the same valuation area in the MDG system.
  4. Replicate the material to the target system.
  5. Observe that the field MBEW-MLMAA is not updated in the target system.

Cause

  • Inconsistency in the system configuration or replication logic.

Resolution

  1. Implement SAP Note 3530456.
  2. Verify that the material is extending MBEW-MLMAA according to system settings.
  3. Ensure the necessary system configurations are correctly set to derive MBEW-MLMAA automatically during material replication.

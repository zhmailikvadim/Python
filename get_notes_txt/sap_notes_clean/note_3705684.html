Symptom

An employee has been enrolled in a benefit, but the enrollment is not showing on their profile as expected

Environment

SAP SuccessFactors Employee Central

Reproducing the Issue

  1. Log into environment
  2. Go to an employee's profile and go to their 'Benefits' portlet
  3. See a benefit is not showing as being enrolled as expected

Cause

This issue can occur when the enrollment record is inactive.

Resolution

Go to 'Benefits Admin Overview' > 'Benefit Enrollment' and search for the User's ID, so their benefit enrollments can be found. If it shows 'No Data', set 'Include Inactive Records' to 'Yes' and see if the record can be found (this also confirms the record is inactive). Within the record, there should be an 'effectiveStatus' field, and this may be set to 'Inactive', which means that while the enrollment record exists, it is inactive and not currently applicable.

If an 'effectiveStatus' field cannot be seen, it may be set to be invisible, which can also be the cause of the issue. This can be resolved by going to 'Configure Object Definitions' > 'Object Definition' > 'Benefit Enrolment', selecting 'Take Action' > 'Make Correction' to edit the object, then finding the 'effectiveStatus' field, selecting 'Details' to bring up the field's details, and setting 'Visibility' to 'Editable'. Save the object, and when refreshing the 'Benefit Enrollment' record in 'Benefits Admin Overview', the 'effectiveStatus' field should be visible. Select 'Take Action' > 'Make Correction' on the relevant 'Benefit Enrollment' record for the user and set the 'effectiveStatus' to 'Active', and this should make the enrollment active for the user.

By then refreshing the employee's profile, the benefit enrollment should show for them as expected as the enrollment record is now active.

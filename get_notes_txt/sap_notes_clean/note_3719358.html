Symptom

After the HANA upgrade, system constantly gets alerts with below message:
301 unique constraint violated: "_SYS_STATISTICS"."ALERT_DELTA_MEM_MERGE_FAIL": line 21 col 592 (at pos 2274): Table(STATISTICS_ALERTS_BASE), Index(_SYS_TREE_CS_#152153_#0_#P0) with error: unique constraint violation in self check for table PIN::_SYS_STATISTICS:STATISTICS_ALERTS_BASE...

Other Terms

301 unique constraint violated, delta merge, OOM, ALERT_DELTA_MEM_MERGE_FAIL, STATISTICS_ALERTS_BASE

Reason and Prerequisites

Issue number 353013

Reason:

Due to a programming error in SAP HANA, indexserver tries to insert records to STATISTICS_ALERTS_BASE table with existing index, causing the unique constraint violation.

Affected Releases:

  • SAP HANA 2:
    • Revisions 079.05 - 079.08 (SPS07)
    • Revisions 085.00 - 089.01 (SPS08)

Prerequisites:

  • You recently upgraded HANA to one of the affected releases.

    and

  • Delta merge fails due to OOM or memory shortage. You get non empty result from HOST_DELTA_MERGE_STATISTICS with cs_error = 2484.
    You can check this by executing below query:

    SELECT * FROM "_SYS_STATISTICS".HOST_DELTA_MERGE_STATISTICS WHERE success = 'FALSE' AND type != 'HINT' AND cs_error = 2484;

Solution

Apply one of the following SAP HANA Revisions:

  • SAP HANA 2:
    • Revisions >= 079.09 (SPS07)
    • Revisions of SPS08: SAP is currently working on a permanent solution to this issue. This SAP Note will be updated as soon as a revision containing the solution is available
    • or higher

  • or higher

Remark:

'301 unique constraint violated' alert is caused by internal error while inserting a record to STATISTICS_ALERTS_BASE table.
However, this is triggered by delta merge failure. To resolve fundamental issue, please analyze why the delta merge got failed.

Symptom

It is defined that the setup time got from setup matrix. While the time picked is different from the maintained values in setup matrix.

Environment

  • Product: SAP S/4HANA
  • Production Planning and Detailed Scheduling

Reproducing the Issue

  1. Define setup matrix in the system
  2. Maintain the related master data, including resource, also confirm the resource capacity updated well in live cache
  3. Create planned orders in product view
  4. Confirm all the settings are well maintained
  5. Check about setup transition, finding it is different from the setup operation time of the order

Cause

Setup duration calculation is influenced by the utilization rate defined in the resource’s capacity profile, causing the applied setup times to differ from the values maintained in the setup matrix.

Resolution

Review the capacity profile of the relevant PP/DS resource and verify the utilization rate setting; adjust it to the intended value (for example, 100% for testing) to avoid unintended scaling of setup times. Then reschedule or recalculate the affected orders/operations and compare the resulting setup times to the values maintained in the setup matrix.

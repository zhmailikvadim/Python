Symptom

  • In CPT1/CPT2, the system does not populate the quantity for the receiver activity type during product costing.

  • The CPT2 formula is syntactically correct and executes without errors, but does not produce the expected result.

  • During test costing, the derived plan quantity for the receiver activity type remains zero instead of reflecting the routing operation time.



Environment

  • Financial Accounting (FI)
  • Controlling (CO)
  • SAP R/3
  • SAP R/3 Enterprise 4.7
  • SAP ERP Central Component
  • SAP ERP
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA

Reproducing the Issue

1) Maintain a routing with activity type MANACT, unit of measure hours, and a non-zero operation time.

2) Create or adjust a CPT2 (Costing Template) with:

  • The same sender cost center as used in the routing.
  • A different receiver activity type (e.g. R&D). 

3) Maintain a CPT2 formula that attempts to derive the routing quantity of activity type MANACT.

4) Execute a test costing run.

5) Observe that the derived plan quantity for the receiver activity type R&D is zero, instead of reflecting the routing operation time.

Cause

This behavior is by design.

In CPT1/CPT2, routing quantities cannot be derived or copied from one activity type to another.

Template formulas are evaluated only in the context of the currently processed activity type.

Functions such as ActivityTypePlanQty can read quantities only for the same activity type, and not for a different activity type.

Resolution

  1. Note that CPT1/CPT2 does not support deriving or copying routing quantities between different activity types.
  2. Template formulas are evaluated only for the currently processed activity type.  
  3. If the same quantity is required for multiple activity types, maintain those activity types directly in the routing (for example, assign multiple activity types to the same operation or use separate operations).
  4. Refer to KBA 3514234 for additional guidance on CPT2 template formula limitations.

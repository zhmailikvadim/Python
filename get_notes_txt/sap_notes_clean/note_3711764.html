Symptom

A performance trace shows that the retrieving of the predecessors during PAC database insertion or TQA writing takes a significant part of the runtime.

Other Terms

aATP; Predecessor; Performance

Reason and Prerequisites

The reason is a loop with where condition that uses a key field with an OR condition or a range logic. Both cannot optimize the key access and hence, cause a performance issue.

With this correction, these OR and range statements are avoide and split into several accesses with a complete key access.

Solution

Apply the correction instructions for this note or upgrade to a SAP S/4HANA support package that contains this correction.

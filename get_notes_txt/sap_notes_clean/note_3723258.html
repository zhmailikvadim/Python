Symptom

When activating or deactivating a distribution model in app "Manage Distribution Model" of SAP Business Data Orchestration (BDO, formerly SAP Master Data Orchestration), an error is displayed at the top of the distribution model object page. This SAP Note describes troubleshooting measures which can be taken without involvement of SAP Support.

Other Terms

BDO troubleshooting, model activation, model deactivation

Reason and Prerequisites

When activating or deactivating a distribution model, including saving changes of a previously activated distribution model ("Save and Activate").

Solution

Overview of error messages

  1. No subaccount destination could be found with the name <destination name>.
  2. Error during communication with destination <destination name>. Remote system response: Unauthorized (status code 401).
  3. The remote system returned status code 400 with the following error message: Periodic job generation not executed yet.
  4. The remote system returned status code 400 with the following error message: Job generation issue. Check execution of job R_JR_BTCJOBS_GENERATOR.
  5. The remote system returned status code 400 with the following error message: Outbound implementation <implementation name> does not support change pointers.
  6. The remote system returned status code 400 with the following error message: Continuous distribution not supported for provider API.
  7. The remote system returned status code 400 with the following error message: Activation failed. Execute initial replication via Cost Center Pull Model.

If your error message is not listed above, please open a ticket on component LOD-MDM-ORC with a detailed description what you want to achieve and/or screenshots of the distribution model with the error.

1. No subaccount destination could be found with the name <destination name>.

A BTP destination is referenced in an existing distribution model (especially if it is currently active or has been activated in the past), but the destination was either renamed, deleted or otherwise modified (additional properties changed). As a result, the system targeted by the destination becomes inaccessible for BDO.

Ideally, if the system behind the destination still exists and contains replicated distribution model data, it should be cleaned up to avoid more serious issues caused by leftover distribution model data in the system. To achieve this, the destination has to be restored to its original state (as maintained in the distribution model). More concretely, the destination must be either

  • renamed to its original name, if it was renamed
  • recreated, if it was deleted
  • enriched again with additional properties required by BDO, e.g. "MDOProvider" or "MDOConsumer", if any of these properties were removed.


After that, it should be possible to save/activate/deactivate/delete the distribution model as expected. If it is absolutely not possible to restore the destination for any reason (for example, because the system targeted by the destination no longer exists), a ticket has to be opened on component LOD-MDM-ORC to request the removal of the destination reference from the distribution model.

2. Error during communication with destination <destination name>. Remote system response: Unauthorized (status code 401).

BDO tries to communicate with a system behind a BTP destination, but the system rejects the request with an HTTP status 401 response.

The authentication details maintained in the mentioned destination seem to be incorrect. Check the authentication details maintained for this destination in the SAP BTP Cockpit and compare them to the authentication details maintained in the system you are trying to reach. 

For connections to ABAP systems using authentication type "BasicAuthentication" with user/password combinations, note that a repeated authorization failure will cause the user to become locked. In this case, after correcting the authentication details maintained in your destination, also check the user directly in the ABAP system and manually unlock the user if required. Additionally, if you are attempting to connect to an on-premise ABAP system, this problem may also be caused by a configuration issue in the SAP Cloud Connector. In this case, the option "System Certificate for Logon" should be unchecked (default setting). It may cause authorization errors if the certificate setup is not done correctly.

3. The remote system returned status code 400 with the following error message: Periodic job generation not executed yet.
4. The remote system returned status code 400 with the following error message: Job generation issue. Check execution of job R_JR_BTCJOBS_GENERATOR.

Refer to SAP Note 3625410 - Job scheduling in Business Data Orchestration.

5. The remote system returned status code 400 with the following error message: Outbound implementation <implementation name> does not support change pointers.

The "Recurrence" setting chosen in the distribution model is something other than "Immediate", but only "Immediate" is supported for the specific business object type and mode maintained in the distribution model.

Change the "Recurrence" setting in the distribution model to "Immediate" and reactivate the distribution model.

6. The remote system returned status code 400 with the following error message: Continuous distribution not supported for provider API.

The "Recurrence" setting chosen in the distribution model is "Immediate", but the option "Immediate" is not supported for the specific business object type and mode maintained in the distribution model.

Change the "Recurrence" setting in the distribution model to something other than "Immediate" and reactivate the distribution model.

7. The remote system returned status code 400 with the following error message: Activation failed. Execute initial replication via Cost Center Pull Model.

Provider SAP S/4HANA system is not in sync with SAP Master Data Integration (MDI). Refer to SAP Note 3633215.

Symptom

You are running SAP Landscape Transformation Replication Server and you face the following problem:

You perceive slowlyness in opening Application Logs tab, because of too many old SLT log entries in table BALHDR.

You are not able to resolve the problem with executions of SBAL_DELETE.

You are not able to change the retention time of SLT logs in the system, so that they would accumulate less, for example in table DMC_RT_PARAMS with the parameter "DAYS_OF_LOG".

Other Terms

LT Replication Server, SLT, replication, replicator, logs, realtime, DMIS 2018, DMIS 2020, S/4HANA 2020

Reason and Prerequisites

Unnecessary application logs stored for long time causing undesirable performance bottlenecks.

SAP LT Replication Server is installed on release DMIS 2018 SP04-SP05 or on S/4HANA 2020 FPS00-FPS02.

Solution

Implement the required corrections on the SLT system in accordance with the correction instructions.

Customize the retention period for SLT logs in the system using an entry in table DMC_RT_PARAMS with the parameter "DAYS_OF_LOG".

Replication jobs have to be stopped during SAP Note implementation: no

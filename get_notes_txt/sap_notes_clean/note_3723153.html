Symptom

  • A static entity was created and used to build a ABAP RESTful Application Programming Model with an OData service. One of the attributes was annotated as DefaultSearchElement.
  • Using the search functionality from the preview application resulted in a runtime dump related to a SQL error. The error message displayed was: "The database returned the SQL code 7. The error text is: feature not supported: CONTAINS predicates on non-column table/view

Environment

SAP HANA 1.0, platform edition

SAP HANA, platform edition 2.0

Reproducing the Issue

  • Using the search functionality from application resulted in a runtime dump related to a SQL error. The error message displayed was: "The database returned the SQL code 7. The error text is: feature not supported: CONTAINS predicates on non-column table/view

Cause

  • The issue occurs because the search operation is performed on a row store table or another unsupported database object for the CONTAINS predicate.
  • The CONTAINS predicate is not supported for row tables, calculation views, analytic views, temporary tables, or SQL Script table variables.

Resolution

  1. Verify if the table is a row store table or another unsupported database object for the CONTAINS predicate.
  2. Refer to the SAP HANA Search Developer guide for supported database objects.
  3. If the table is a row store table, consider converting it to a column store table to allow the use of the CONTAINS predicate.
  4. Ensure that the database objects used for the SELECT statements are supported as per SAP HANA Search Developer guide

Symptom

You have a purchase order in which S/4hana output control (a.k.a BRF+ output) is used, but no output could appear in the newly saved document.

For example because the workflow failed and EKKO-PROCSTAT field is not 05 (and not 02) or simply because there is output determinable from ODP Tcode.

When you open such a purchase order and click on messages button then you get this system message: Output request is not available for object "PURCHASE_ORDER", with object key ".." - Message No. APOC_OR_UI_MESSAGES002

Environment

Material Management

  • S/4hana On-Premise / Private Cloud version 2025 and onwards 

Cause

New development around outputs' displaying from 2025 dated version. In case no output in the PO (no entry yet for such PO in APOC_D_OR_ITEM db table), then if you press the messages button the above mentioned system message is designed to appear.

Resolution

From 2025 system version the "print preview" button can be pressed to check such PO's messages screen in which PO no output determined yet.

If after print preview checking you press again the messages button the messages screen can be opened as by then draft output determination triggered with pressing preview button.

Use the software in this sense.

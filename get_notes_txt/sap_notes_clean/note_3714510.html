Symptom

In the process "Start of Supply for Electricity (DSO View)" (DE_B_GPKE_DSO_SOS_REG), when parallel End of Supply and Start of Supply processes occurred, the supply scenario was not up to date. This led to an error when the system sent the request for Deregistration and Termination to the old supplier.

This issue is relevant for the market role: DSO.
This fix also involves an emergency patch - EP 2000011381 on the Cloud side.

Other Terms

DE_B_GPKE_DSO_SOS_REG, 2508, GPKE, BM_READ_UPD_SUSC_REQ, BM_READ_UPD_SUSC_RESP, supply scenario, POD blocked, Deregistration, Termination request, old supplier, parallel processes, Start of Supply, End of Supply

Reason and Prerequisites

The error occurred because the supply scenario wasn't up to date, leading to incorrect processing when the system attempted to send the Deregistration and Termination request to the old supplier.

Solution

Implement the corrections of this SAP Note or import the relevant support package.

After implementing this SAP Note, the system refetches the supply scenario and rechecks whether the point of delivery (POD) is still blocked after the extension step and before sending the Deregistration and Termination request to the old supplier. The BC set /US4G/3_0_DSO_SP07_NOTE_3714510 contains the necessary corrections.

This fix also involves an emergency patch - EP 2000011381 on the Cloud side.

You need to retest the scenario by completing all End of Supply process activities before closing the deadline at step 0740. You can verify the fix by checking that the system sends steps 0720, 0740, and 0760 successfully in a POD blocked scenario.

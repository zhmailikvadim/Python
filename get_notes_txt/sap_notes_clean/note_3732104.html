Symptom

During withdrawal processing, the fee condition is created as a zero-value condition. If the fee condition has already been created manually, the business operation may become stuck due to a duplicate condition error. This note introduces a fix to skip the creation of the zero condition if a manual condition already exists, thereby preventing the error. Additionally, the edit option has been removed for stuck business operations to avoid potential inconsistencies during processing.

Other Terms

Withdrawal;

Reason and Prerequisites

 This problem is caused by a program error.

Solution

Implement the attached program corrections or import the relevant Support Package.

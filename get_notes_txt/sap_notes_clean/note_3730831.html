Symptom

In the process "Receipt of Meter Reading (DSO View)" (DE_B_MET_MR_INFO_DSO), when you receive a meter reading MSCONS message, the system creates a separate meter reading document after step 0200-0040, even when a matched open meter reading order exists.

This issue is relevant for the market roles: SUP, DSO, MOS.

Other Terms

MSCONS, /US4G/CL_UBD_MR_VALUE_01, /US4G/IF_UBD_MR_VALUE~DETERMINE_MR_DOC, meter reading order, meter reading document, Receipt of Meter Reading (DSO View), DE_B_MET_MR_INFO_DSO

Reason and Prerequisites

The error occurred because of an incorrect check in the system.

Solution

Implement the corrections of this SAP Note or import the relevant support package.

After implementing this SAP Note, when you receive a meter reading MSCONS message and a matched open meter reading order exists, the system no longer creates a separate meter reading document. Instead, the system updates the meter reading value on the matched open meter reading order. You can verify the fix by re-inbounding a meter reading MSCONS message or running a new test to confirm that the system updates the existing order instead of creating a separate document.

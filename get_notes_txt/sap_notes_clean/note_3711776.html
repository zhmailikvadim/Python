Symptom

You generate one or more null DSN declarations with the parallelization mode activated. In the result, you see only null DSN declaration(s) generated.
When you check the content of the declaration, you see that the main blocks S10* are missing. Due to this, the declaration is not compliant.

Other Terms

RPLDSNF0

Solution

After implementing this SAP Note, the system generates the blocks S10* when the parallelization is activated and there is only null DSN generated.

These corrections are available with the HR Support Packages referenced below.
If you have an urgent need of these corrections, follow the correction instructions provided.

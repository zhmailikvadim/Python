Symptom

The customer wants to configure the HTTP protocol in the Cloud Portal endpoint or directly access Apache via port 80. 

However, the site automatically returns a 301 response code, redirecting to HTTPS.

Environment

SAP Commerce Cloud

Reproducing the Issue

  1. Set the endpoint with HTTP protocol.
  2. Access the domain over HTTP.
  3. Observe a redirect to HTTPS with 301 Moved Permanently.

Cause

In the /usr/local/apache2/conf/httpd.conf

<Virtualhost xx.xxx.xx.xx:80>

   <Location />

        Redirect permanent https://xxx%{REQUEST_URI}

     </Location>

</Virtualhost>

Traffic to and from the public internet allows the use of both HTTP and HTTPS protocols. However, every time a request would reach the port 80, Commerce automatically redirects it to the port 443. This redirect ensures that data from an insecure custom endpoint reach the Commerce database with an adequate layer of encryption and protects the sensitive information from potential attacks.

By default, port 80 remains disabled, but you can choose to enable its exposure for the environment using the Port 80 exposed checkbox. Disabling this option ensures that all incoming HTTP traffic is blocked, forcing secure communication over HTTPS. This setting applies only to public connections and does not affect VPN connectivity.

Resolution

Symptom

Communication Framework uses bgRFC inbound queues to process sequential execution of CMF Events for business object instance. In case of any error, event is not executed and corresponding bgRFC unit is locked in the queue. Every failed/locked unit should be revised in order to ensure smooth and not interrupted CMF Events processing.

This note delivers new improved way of bgRFC event queue monitoring. It can be executed via transaction code /DBE/CF_RBGMON.

Other Terms

Communication Framework, Event queue, bgRFC, monitoring

Reason and Prerequisites

New functionality delivered.

Solution

Implement the attached correction.

Following new object will be created in the system:

  • Report /DBE/CF_BGRFC_MONITOR
  • Transaction code /DBE/CF_RBGMON
  • Application table /DBE/CF_ALOGXL

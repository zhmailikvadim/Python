Symptom

When using device group mechanism in the Master rule collection to assign a desktop according to the associated device, smartphone frameworkpage or tablet frameworkpage is not loaded for Smartphone or tablet device respectively. Instead any other desktop will be loaded while ignoring the rule based on Device Group/Type.

Environment

  • SAP NetWeaver 7.3 Support Package 8 and Below
  • SAP enhancement package 1 for SAP NetWeaver 7.3 Support Package 5 and Below

Cause

By default, the device group mechanism is disabled in the above versions. And hence the device is not recognised, when a rule based on Device Group/Type maintained in the master rule collection to assign a desktop to the associated device.

Resolution

In order to enable the device group mechanism in the above versions, follow the below steps:

 Access SAP NetWeaver Administrator by adding /NWA to the AS Java URL (hostname:port/nwa) and then go to Configuration Infrastructure  and choose the Application Modules link

 In the Module List area, search for com.sap.portal.desktop.desktopservice

 In the Web Module Details area, select desktopservice.

 In the Portal Service Details area, set the value of the UseDeviceGroups property to true.

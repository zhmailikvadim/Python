Symptom

Why do Duplicate Planned events appear in the GTT tracking timeline for shipments or freight units?

Cause

If the sample extractor is implemented and the system is using a tracking unit scenario (Scenario 02, 03, or 04), duplicate planned events may appear in the tracking timeline—one with a tracking number and one without. This behavior is expected by design.

Resolution

  1. To remove duplicate planned events from the tracking timeline, contact your internal IT team or implementation partner to modify the logic based on your specific business needs.
  2. There are four tracking scenarios available for TM integration, each utilizing its own planned event process class.
  3. Review the configuration in transaction code ZGTT_TRACK_CONF.
  4. If no entity is configured in ZGTT_TRACK_CONF, the system defaults to Scenario 01, which does not require additional configuration.
  5. Refer to the following planned event process classes for guidance:

    Scenario 01: Maintain Reference Document on Freight Order/Freight Booking
  • Freight Order (FO) planned event in class ZCL_GTT_STS_PE_FO_FILLER
  • Freight Booking (FB) planned event in class ZCL_GTT_STS_PE_FB_FILLER
  • Freight Unit (FU) planned event in class ZCL_GTT_STS_PE_FU_FILLER

    Scenario 02: Maintain Tracked Objects on Freight Order/Freight Booking

  • Freight Order (FO) planned event in class ZCL_GTT_STS_PE_TRK_ONFO_FO_FIL
  • Freight Booking (FB) planned event in class ZCL_GTT_STS_PE_TRK_ONFO_FB_FIL
  • Freight Unit (FU) planned event in class ZCL_GTT_STS_PE_TRK_ONFO_FU_FIL
  • Tracking Unit (TU) planned event in class ZCL_GTT_STS_TRK_TU_ON_FO / ZCL_GTT_STS_TRK_TU_ON_FB

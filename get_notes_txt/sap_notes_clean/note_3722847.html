Environment

SAP S/4 HANA

Reproducing the Issue

  1. Open transaction SE38
  2. Perform a complete CALM configuration with report /SDF/CALM_SETUP

Cause

Millions of entries in ARFCSSTATE table, due to the high volume of records the dump TSV_TNEW_PAGE_ALLOC_FAILED is generated.

Resolution

The reported runtime errors in the system occurrs when data collector attempts to read data from the ARFCSSTATE table

Please follow the detailed steps in the following documents, it will help you resolve the issue:

WIKI: Deleting unwanted entries from RFC tables (ARFCSSTATE, ARFCSDATA etc.)

2899366 - Huge entries in table ARFCSDATA

Symptom

Excessive growth of the Media table and associated blob storage leads to:

  • Database overconsumption

  • Slow PDP rendering

  • CronJob and deployment delays

  • Long backup/restore times

  • Increased cloud storage usage

Environment

SAP Commerce Cloud 2211, SAP Commerce Cloud 2211 for JDK21

Reproducing the Issue

  1. Identify top media folders and media types by volume and size

  2. Estimate average images per product

  3. Validate all image formats are required and used on the storefront

  4. Review CronJobs generating large log media

Prioritize cleanup based on storage footprint

Cause

  • Excessive CronJob logs

  • ImpEx import/export media generation

  • Invoice PDF storage without retention

  • EmailMessage, EmailBody, EmailAttachment accumulation

  • Inbound/Outbound integration payload storage

  • Excessive product image formats (unused/duplicate)

  • High number of images per product

Resolution

1. CronJob Logs

  • Disable unnecessary logging

  • Delete historical logs

  • Configure retention using cleanUpLogsJobPerformable

  • Schedule automated cleanup

2. Catalog Sync Logs

  • Implement retention for sync log media

  • Use cleanup utilities (e.g., sanecleanup extension)

3. Media Retention

  • Configure rules via Data Retention Framework

  • Schedule periodic media cleanup jobs

4. Inbound Requests

  • Ensure inboundRequestCleanupCronJob is enabled and executed regularly

5. Email Cleanup

  • Remove EmailMessage, EmailBody, EmailAttachment if not required

  • If using OOTB sendEmailAction, ensure removeSentEmail step is configured

  • Archive externally if audit retention is required

6. Large Blob Deletions

  • purgeDeletedFilesCronJob runs single-threaded and may be slow for large volumes

  • For mass cleanup, implement multithreaded blob deletion strategy

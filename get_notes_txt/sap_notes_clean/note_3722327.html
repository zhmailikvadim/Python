Symptom

  • After switching from other views back to the list view in calibration sessions, the standard field "status" displays incorrect data when scrolling down.
  • The "status" field initially shows "Active" but changes to "Inactive" upon scrolling down the list.

Environment

SAP SuccessFactors Calibration

Reproducing the Issue

  1. Navigate to a calibration session with multiple subjects.
  2. Open the list view and observe the standard field "status" displaying "Active".
  3. Scroll down the list; the "status" field does not update its value.
  4. Switch to another view and then return to the list view.
  5. Scroll down the list again, and observe that the "status" field changes from "Active" to "Inactive".

Cause

This is a Known Issue.

Resolution

The fix for this issue is planned to be deployed on 1H 2026 release.

For release timelines, please review SAP SuccessFactors Product Release & Road Map Information.

Symptom

  • When using the InvoiceRequest_Out (Technical name: CO_EDI_INVOICE_REQUEST_OUT) to automatically send electronic invoices to customers' systems, the "DocumentDate" was not created with the current date (the date the invoice was created). 

Environment

  • SAP S/4HANA
  • SAP S/4HANA Cloud Private Edition

Cause

• The API CO_EDI_INVOICE_REQUEST_OUT uses the DOCUMENT_DATE derived from the billing date VBRK-FKDAT, mapped in the standard class CL_EDI_SD_INVC_MAPPING method MAP_DOCUMENT_DATE.
• The billing date FKDAT is sourced from preceding reference documents, and for order-related billing, it defaults to the sales document proposed billing date (VBKD-FKDAT).
• The billing date (VBRK-FKDAT) may differ from the billing creation date (VBRK-ERDAT), causing discrepancies when sent to the API.

Resolution

1. Modify VBRK-FKDAT to the current date during billing creation using USEREXIT_FILL_VBRK_VBRP (include RV60AFZC).
2. Alternatively, update the mapping in class CL_EDI_SD_INVC_MAPPING method MAP_DOCUMENT_DATE to map DOCUMENT_DATE with VBRK-ERDAT for exporting BILLING_DATE as the creation date.
3. Refer to SAP Note 170183 for considerations when applying modifications.

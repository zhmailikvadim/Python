Symptom

  • Error encountered while trying to connect a custom connector between SAP Customer Data Platform (CDP) and Emarsys (EMS).
  • Error message: "Bad Request. Request returned unsuccessful status code. Response content: {"error":"invalid_client","error_description":"Client authentication failed (e.g., unknown client, no client authentication included, or unsupported authentication method). The OAuth 2.0 Client supports client authentication method 'client_secret_basic', but method 'client_secret_post' was requested. You must configure the OAuth 2.0 client's 'token_endpoint_auth_method' value to accept 'client_secret_post'."}"

Environment

  • SAP Customer Data Platform
  • Connector Studio
  • Emarsys Integration

Reproducing the Issue

  1. Create a custom connector in CDP Connector Studio.
  2. Create a destination application using the custom connector.
  3. Configure all URL and credential information correctly.
  4. Attempt to connect the destination application.
  5. Observe the error message indicating client authentication failure.

Cause

Emarsys expects the authentication method 'client_secret_basic', but CDP sends 'client_secret_post'.

Resolution

  1. Copy and paste the current JSON from the connector studio "Publish" step to a text editor and add the JSON below to the "securitySchemes" object.

    • ```json

      "securitySchemes": {

        "OAuth2 - via Header": {

          "type": "oauth2",

          "flows": {

            "clientCredentials": {

              "scopes": {},

              "tokenUrl": ""

            }

          }

        }

      }

      ```

      • The name should be 'Header', and the tokenUrl can be specified.

  2. Call the endpoint below and pass the updated JSON in the request body to make this change.
    • PUT https://cdp.eu5.gigya.com/api/workspaces/{{workspaceId}}/applibrary/{{customConnectorId}}?userkey={{userKey}}&secret={{secret}}
      • Replace {{customConnectorId}} with the ID of the custom connector, which can be found at the end of the URL when editing a connector.
  3. Verify the change by calling the same endpoint with HTTP GET (no request body):
    • GET https://cdp.eu5.gigya.com/api/workspaces/{{workspaceId}}/applibrary/{{customConnectorId}}?userkey={{userKey}}&secret={{secret}}
  4. Add a new application based on the updated custom connector.

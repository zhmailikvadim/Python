  • Runtime error COMPUTE_INT_PLUS_OVERFLOW occurs during health check execution.

  • Error message: "In the running program 'CL_DMC_HC_CH_LOGTAB_STATUS====CP', the addition (operation '+' or 'ADD') of the values 2147483647 and 1 produced a value larger than 2147483647 or smaller than -2147483648, which led to an integer overflow."

Environment

SAP Landscape Transformation Replication Server (SLT)

Cause

The issue arises due to the threshold values set in table DMC_HC_SETTINGS where CHECK_ID = LOGTAB_STATUS_CHECK. LOGTAB_WARNING_THRESHOLD and LOGTAB_ERROR_THRESHOLD are set to the maximum integer limit (2147483647). During the health check, the system attempts to add '1' to the threshold value to trigger the warning or error, resulting in an integer overflow.

Resolution

  1. Access table DMC_HC_SETTINGS.
  2. Locate the settings CHECK_ID =  LOGTAB_STATUS_CHECK.
  3. Reduce the values of LOGTAB_WARNING_THRESHOLD and LOGTAB_ERROR_THRESHOLD by at least 1 from the maximum integer limit (2147483647).

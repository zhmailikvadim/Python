Symptom

During the Customer List upload, if an incorrect file name is entered, the system exits the upload transaction. As a result, the user must restart the upload transaction again to correct the file name.
This SAP Note resolves the issue by ensuring that the control remains on the same screen even when an invalid file name is entered, allowing the user to correct the file name without exiting the transaction.

Other Terms

Customer List, /IRM/GCUL_UPLOAD, Wrong File Name

Reason and Prerequisites

This is caused by Program error.

Solution

Please implement the correction instructions attached. These corrections are only valid if you have applied the Support Packs.

Release 6.4 – Base Package & Support Package 1
Release 6.3 – Base Package

Use transaction SNOTE to implement the correction instructions specified below and activate the affected program object

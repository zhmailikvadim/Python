Symptom

Trying to create a production order using the Production Order OData API (via POST API_PRODUCTION_ORDER_2_SRV/A_ProductionOrder_2) with external numbering and custom fields leads to an error message: <code>CO/080</code> <message>External number assignment (enter valid order number)</message>

Request:

POST API_PRODUCTION_ORDER_2_SRV/A_ProductionOrder_2

Body example:

{
"ManufacturingOrder" : "<external number>",       <-----------------External order number
"Material" : "FG126",
"ProductionPlant" : "1010",
"ManufacturingOrderType" : "<order type>",     <--------------------------------- Order type  with external numbering active
"ProductionVersion" : "0001",
"TotalQuantity" : "5",
"MfgOrderPlannedEndDate" : "2026-03-31T00:00:00",
"YY1_TFORDList0_ORD" : "Y"  <-------- Custom Field
}

Other Terms

Production Order oData API, API_PRODUCTION_ORDER_2, Custom Fields, External Number

Reason and Prerequisites

External Order ID for the "ManufacturingOrder" is not passed on for order creation.

Solution

Program Correction.

Possible work around:

- First create the order with an external number and then use a PATCH call for adding the custom field.

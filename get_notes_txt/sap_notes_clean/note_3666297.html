Symptom

The line-by-line calculation has been activated using the BAdI BADI_TAX1_XTXIT_SET. However, when posting a document containing one tax code and two or more G/L Accounts through an enjoy transaction, such as FB60, the system posts only one tax line.

Environment

  • Financial Accounting (FI)
  • SAP R/3
  • SAP R/3 Enterprise 4.7
  • SAP ERP Central Component
  • SAP ERP
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
      • SAP S/4HANA

Reproducing the Issue

1. Activate the line-by-line calculation using BADI_TAX1_XIXIT_SET;
2. Post or park a document using an Enjoy transaction (e.g FB60);
3. The line-by-line calculation or entry of tax does not work as expected.

Cause

"The method was provided for specific SAP developments, for this reason, there are transactions or scenarios that do not fully support this. 

Resolution

There are some limitations when the line-by-line calculation is activated using the BAdI BADI_TAX1_XTXIT_SET. For example, if the line-by-line is activated using this BAdI, the line-by-line calculation or entry of tax is not supported in document parking and enjoy transactions, such as FB50, FB60 or FB70.
It means that the BAdI BADI_TAX1-XIXIT_SET is not called in the EnjoySAP transactions and document parking, hence the line-by-line calculation does not work in these scenarios when activated using this method.

Symptom

The auto-population of the shipping point in the sales order is not working even when there is an entry for the Shipping Condition-Loading Group-Plant-Proposed Shipping Point combination in configuration activity 102126-Assign Shipping Points.

Environment

SAP S/4HANA Cloud Public Edition

Cause

The shipping point determination setting is done via configuration activity 102385 - Define Rule for Determination of Shipping Point. There are two options:

  • Storage-Location-Specific Shipping Point Determination
  • Plant-Specific Shipping Point Determination

If the 'Storage-Location-Specific Shipping Point Determination' is used, the system checks the data in configuration activity 102387 - Assign Shipping Points According to Storage Location. If the Storage Location is not maintained in the sales order and there is no entry in this configuration point to accommodate the absence of a Storage Location, the system will not automatically determine the shipping point.

Resolution

Supplement the settings in configuration activity 102387 - Assign Shipping Points According to Storage Location to handle the assignment of shipping points in the absence of a storage location.

Keywords

Shipping point, Shipping Point Determination, 102387, 102126, 102385, Storage location, Storage-Location-Specific, Plant-Specific

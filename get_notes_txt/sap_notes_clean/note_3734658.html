Symptom

  • In Real User Monitoring Analysis view, for some requests the user name appears as "OTHERS" instead of the actual user names.
  • Other requests in the same view display the correct user names.

Disclaimer: "Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental."

Environment

  • SAP Cloud ALM
  • Real User Monitoring

Reproducing the Issue

  1. Open Real User Monitoring and navigate to the Analysis view.
  2. Select a time range that returns more than 1000 requests.

Cause

The Analysis view applies a rank limit of 1000 for filters; when the total number of requests exceeds this limit, only the top 1000 are shown with details and the remaining requests are aggregated as "Others".

Resolution

  1. Reduce the Analysis time range to a shorter interval so that the number of returned requests is below 1000.
  2. Optionally apply additional filters to further narrow results and keep the total under 1000.
  3. Note that increasing the rank limit above 1000 (for example, to 2000 or 3000) is not supported.

Symptom

With the new build experience, either in Edit of View mode, applying A-Z, Z-A or a Custom Order to a measure header correctly applies the selected sort, but:

- The "Sort Options" menu always displays "Default",
- A sort token is not added in the table's header.

Other Terms

UQM, Orca, Custom Sort, MMO, ODE, OVM

Reason and Prerequisites

A measure member order was applied in the historical build experience via the "Edit Member Order..." panel triggered via the builder panel.
For some BW configurations, having set this measure order results in the visual issues experienced with the new build experience.

Solution

This is a known issue currently being investigated.

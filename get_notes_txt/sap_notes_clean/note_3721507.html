Symptom

During Route Assembly, an incorrect Sales Order Item Reference is determined for a Delivery Item.
This situation typically occurs when one or more Delivery Items have no reference to a Sales Order Item.

Additional Information:

New Releases (>= S4COREOP109):
In newer releases, it is permissible to assemble a Delivery Item that has no Sales Order Item Predecessor (see note 3693014).
This situation can occur when an outbound delivery is updated with additional line items that don't exist in the sales order,
such as additional line items for pallets (Handling Unit Packaging Materials) during EWM Integration.
Due to the issue described above, an incorrect Sales Order Item Reference is assigned to the Delivery Item.
This note resolves the issue so that the Sales Order Item Reference remains empty as expected.

Older Releases (< S4COREOP109):
In older releases, the functionality to assemble a Delivery Item without a Sales Order Item Predecessor is not supported.
During assembly of a new route, an error message should be displayed:
"Sales order &1 does not exist." (LMD_ROUTE_HUB/041)
However, due to the issue described above, this expected error message is not always displayed properly, since an incorrect Sales Order Item Reference is assigned.
This note resolves the issue so that the Sales Order Item Reference remains empty and the expected error message is properly displayed.

Symptom

  • When transaction BUP2/BUP3 is executed for the second time, it displays the previous BP instead of the initial screen to pass BP number.

Environment

  • SAP ERP
  • SAP S/4HANA

Reproducing the Issue

  1. Execute transaction BUP2/BUP3 for the first time. The initial screen to pass BP number is displayed.
  2. Pass a BP number and press Enter to navigate into the BP.
  3. Execute /n/BUP2 or /n/BUP3 for the second time. The initial screen to pass BP number is not displayed. Instead, the system navigates to the previous BP.

Cause

By standard design, transactions like BUP2/BUP3 do not clear the BP buffer (they are designed for quicker use compared to transaction BP).

Resolution

Please use the folder button (Shift+F5) to navigate to a different BP when using BUP2/BUP3.

Keywords

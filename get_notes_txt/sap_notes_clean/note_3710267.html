Symptom

When creating or using a lead filter that includes multiple territories (for example Territory A, Territory B, and Territory C), the result list does not display the expected total number of leads.

Environment

  • SAP Sales Cloud and SAP Service Cloud Version 2
  • SAP Sales Cloud Version 2 

Reproducing the Issue

 

  1. Navigate to Menu → Leads.
  2. Open the Advanced Filter.
  3. In the Territory field, select:
    • One parent territory (for example, Territory A)
    • Its sub-territories (for example, Territory B and Territory C)
  4. Ensure that Include Sub-Territories is set to True.
  5. Click Apply.
    • Result: For example, 10 leads are displayed.
  6. Reset the filter.
  7. Search using only Territory B.
    • Result: For example, 2 leads are displayed.
  8. Reset the filter.
  9. Search using only Territory C.
    • Result: For example, 3 leads are displayed.
  10. Reset the filter.
  11. Search using only Territory A.
    • Result: No leads are displayed, as no leads are directly assigned to the parent territory.

Unexpected Behavior-

Users may question why the combined filter returns 10 leads, while:

  • Territory B returns 2 leads
  • Territory C returns 3 leads
  • Territory A returns 0 leads

This creates the perception that 5 additional leads are appearing unexpectedly.

 

Cause

This behavior is due to territory hierarchy evaluation.

When Include Sub-Territories is set to TRUE:

  • The system evaluates the entire hierarchy of the selected parent territory.
  • Leads assigned to sub-territories are returned as part of the parent territory.
  • The filter does not perform a simple arithmetic summation of each selected territory.

As a result, the total number of leads returned may be higher than the visible sum of individually filtered territories.

Resolution

This behavior is working as designed.

To avoid confusion:

  • Mark "Include Sub-Territories" to FALSE when filtering multiple territories independently.
  • Review the territory hierarchy structure if independent lead aggregation is required.

Symptom

  • Icon name/initial letters not aligned between web and mobile versions.
  • This behavior can be noticed across the mobile application
  • This inconsistency can be replicated on both iOS and Android devices.

Environment

SAP SuccessFactors HCM Suite

Reproducing the Issue

  1. Login to SuccessFactors
  2. If the user has no profile picture uploaded, note that the icon name on the web version displays the user's name initial letters
  3. Login to the mobile app with the same user.
  4. Note that the icon name on the mobile app is not consistent with the web version and displays different initial letters.

Cause

This is a known error.

Resolution

A fix for this issue is planned to be deployed in the 1H 2026 release.
For release timelines, please review SAP SuccessFactors Product Release & Road Map Information.

Once fixed, the expected behavior of avatar display priority in the mobile application should be:

  • Use images for avatars uploaded by end users.
  • If end users do not upload their own image as avatar, show initials which have up to three alphabetical characters (A-Z, a-z).
  • Gender-neutral placeholder when no other image is available.

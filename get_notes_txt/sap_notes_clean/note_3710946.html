Symptom

API requests can be used to query the entire OData metadata or a subset of the metadata by using the system query option $metadata.

To query the metadata document for a subset of entity types, specify the subset of entity type in the URI:

https://<API-Server>/odata/v2/Entity/$metadata

For example, the following URI identifies the metadata of the JobApplication entity type:

https://<API-Server>/odata/v2/JobApplication/$metadata

When using the above URI to query JobApplication metadata, picklist fields are not included in the query result even though the fields are upsertable and they are present in OData API Data Dictionary in Admin Center.

Environment

SAP SuccessFactors Recruiting Management

Reproducing the Issue

  1. Navigate to Admin Center > OData API Data Dictionary
  2. Look for JobApplication entity in the dropdown
  3. Confirm that the picklist fields are included in the list of fields
  4. Next, query JobApplication metadata using the following URI: https://<API-Server>/odata/v2/JobApplication/$metadata
  5. The metadata query result doesn't include navigation properties

Cause

Picklist field is a navigation property. Engineering confirmed a limitation where JobApplication, being an MDF object, cannot be used to filter the navigation properties. Therefore, even though field is present in the OData API Data Dictionary, it is not listed in result body of query https://<API-Server>/odata/v2/JobApplication/$metadata.

Resolution

A workaround is to query the complete OData metadata, including navigation properties, using the below URI:

https://<API-Server>/odata/v2/$metadata

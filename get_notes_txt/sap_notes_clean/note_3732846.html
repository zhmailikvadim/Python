Symptom

This SAP Note replaces the following SAP Notes: 1803855, 2247990, 2310627, 2552162, 2661728, 2695239 

The template functions that read costs or quantities from the CO tables COSP, COSS, or COSL currently have two options for determining values:

    1st Determination per Period
    2. Determination over the entire fiscal year (totaling across all periods)

The following functions are not available in the standard system:

  1. There is no option to change the costs or Totals quantities over a certain period interval.
  2. Delivered template functions that read costs or quantities from the database tables COSP or COSS do not provide the option of totaling the values from period 1 up to the current period.
  3. Delivered template functions that read activity quantities from the database table COSL do not provide the option of totaling the values from period 1 to the current period.
  4. For template functions that have a cost center, an activity type, a business process, and/or a statistical key figure as parameters, there is no option to enter the cost center(s) and so on as a group.
    Therefore, fixed values for cost centers, activity types, business processes, and statistical key figures must always be changed in the template definition.
    With the option of a group entry, fixed values can be changed centrally in group maintenance.

Reason and Prerequisites

Missing function

Solution

Implement the correction instructions of this SAP Note.

After you implement the corrections and make the relevant Customizing settings in transaction CTU6 (see below), the following enhanced functions are available:

    1. In many read functions for costs and quantities, in addition to the period-by-period (calculation category 1) and the full-year method (calculation category 2), reading can take place using a period interval within a fiscal year. To do this, you must choose calculation category 3 (Summation Using Period Interval) and enter a start and end period.
    2. In many read functions, you can enter groups as well as single values for cost centers, activity types, business processes, statistical key figures, and cost elements.

              This should primarily improve the maintainability of template maintenance: a change to a cost center, statistical key figure, and so on, can be made centrally using group maintenance.

Customizing the Template Functions for the New Parameters
The procedure is described here as an example for the template function Actual Qty for Activity Type (logical function name: K_ABC_GET_ALLOC_QUANTITY, ABAP function name K_ABC_ALLOC_QUANTITY) from environment 107 (general data):

    1. Call transaction CTU6  and set a filter for environment 107. Navigate to the template function ActQtyAssmtType and call the details by double-clicking.
    2. Go to the "Parameters" tab page. Here you can see the following parameters:

By default, they all have the type I (invisible) and the default value blank. This means that in template maintenance, the new parameters are not visible for the time being and have no influence on existing templates.

You can now open the new parameters for template maintenance for the existing template function or you can also open existing parameters (for example, "Cost Center") to invisible.
You can also create a new template function to use the new parameters. For more information about creating a new template function (also called environment function), see SAP Note 153008.

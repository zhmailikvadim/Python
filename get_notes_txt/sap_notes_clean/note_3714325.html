Symptom

You are using the Universal Journal in your system.

When running the Federal Transaction Register with dynamic selection(s) active, if you specify more than one criterion you receive a Parse Error.

Alternatively, you may receive incorrect (one single line) or no data when using the "Show By Original Document" checkbox. This is due to FOR ALL ENTRIES on FMUSFGA, where at database level there is a limitation enforced on reducing on the basis of primary key. However, when Universal Journal is active, the primary key (GL_SIRID) in FMUSFGA is all zeros since this field does not exist in ACDOCA or BKPF. Therefore elimation by primary key retains a single entry.

Other Terms

Transaction Register, RFFMFG_TRANS_REG, FMFG_E_TRANS_REG, FMUSFGA, FMUSFGA_COMPAT, Show By Original Document, FOR ALL ENTRIES

Reason and Prerequisites

Reason: When Universal Journal is active, FMUSFGA is replaced by a view on ACDOCA and BKPF. Previously, the Transaction Register was accessing this data by programatically joining ACDOCA and BKPF. However, this created challenges in handling the possibilities of dynamic selection.

In addition, using FOR ALL ENTRIES on FMUSFGA on the basis of reference document number yields a single row due to reduction of adjacent duplicates at database level eliminating entries by primary key (GL_SIRID), which is blank because the field does not exist in ACDOCA/BKPF.

Now selection within the Transaction Register is redirected to the compatibility view. 

Prerequisites: US Federal Agency

Solution

Apply the correction instruction.

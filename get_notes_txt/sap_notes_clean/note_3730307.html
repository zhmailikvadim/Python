Symptom

Contact attributes (KNVK-PARH*) not updated when calling CL_MD_BP_MAINTAIN from a custom program

Environment

  • SAP S/4HANA

Reproducing the Issue

  1. From a custom ABAP program, prepare data to update contact (KNVK) attributes PARH1–PARH5 (or similar fields).
  2. Call CL_MD_BP_MAINTAIN=>MAINTAIN to perform the update.
  3. Observe that no error is raised, but the KNVK-PARH* fields are not updated in the database.

Cause

  • The contact’s external number (PARNR) is not filled into buffer via CVI_MAPPER before calling CL_MD_BP_MAINTAIN.

Resolution

  1. Ensure a PARNR is available for each contact to be updated/created:
  2. For existing contacts, use the existing KNVK-PARNR.
  3. For new contacts, generate a PARNR using NUMBER_GET_NEXT for KNVK-PARNR.
  4. Before calling CL_MD_BP_MAINTAIN=>MAINTAIN, call CVI_MAPPER=>SET_GV_EXT_PARNR to fill the contact’s PARNR and PERSON_GUID into buffer.
  5. Call CL_MD_BP_MAINTAIN=>MAINTAIN to update the Business Partner data. Fill I_DATA-CUSTOMER-CENTRAL_DATA-CONTACT with the contact’s PARNR and the attributes to be updated (for example, KNVK-PARH1–PARH5).
  6. Call BAPI_TRANSACTION_COMMIT to persist the changes.
  7. Verify that the KNVK-PARH* fields are updated.

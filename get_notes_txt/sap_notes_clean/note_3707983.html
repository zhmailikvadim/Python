Symptom

In FIORI app F2229 Manage Purchase Requisition - Professional, creating PR item from cross-catalog search does not returning any catalog items although entries are present in table MMPUR_CAT_ITM and can be seen in app "Manage Catalog items".

Environment

  • Materials Management (MM)
  • SAP Fiori 
  • SAP S/4HANA

Cause

Missing code patch and configuration.

Resolution

  1. Followed the solution section in KBA 2680363 - Internal catalog items can't be found in Cross-Catalog Search - SAP S/4HANA - SAP for Me 
  2. Patch Note 3646334 - Create PR - Users are able to see catalogs that are not assigned to them in the filter panel - SAP for Me 
  3. Patch Note 3437849 - Catalog search filters not working in the app My Purchase Requestions (New) - SAP for Me 
  4. Deactivate and reactivate the search connector using transaction /nESH_COCKPIT with Connector id: CD$ALL~ESH_U_CTLG_ITM 

Keywords

F2229, Manage Purchase Requisition, PR, Purchase Requisition, Item, catalog, cross catalog, cross-catalog, search, Material Master, MMPUR_CAT_ITM, Manage Catalog Items

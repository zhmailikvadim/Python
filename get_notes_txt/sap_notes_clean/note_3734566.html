Symptom

  • Admin updated the label value for the label ID 'MOB_LRN_custom_cards' in Manage Languages within BizX for certain locales such as Dutch and French.
  • Updated label values are not displaying correctly in the SAP SuccessFactors Mobile Application. 

Environment

  • SAP SuccessFactors Learning
  • SAP SuccessFactors Mobile Application

Reproducing the Issue

  1. Log in to SuccessFactors. 
  2. Navigate to the Admin Center and search for “Manage Languages”.
  3. Update the label for label ID “MOB_LRN_custom_cards” with translations (e.g., French and Dutch).
  4. Log in using the SuccessFactors Mobile application and navigate to Learning. 
  5. Observe that the translated locale values are not displayed.

Cause

A mobile application defect: the API for custom card configuration did not provide the locale code required by the mobile client, resulting in English being shown instead of the locale-specific label values.

Resolution

The resolution for this issue has been implemented. It is scheduled to be delivered as part of the Mobile 2605 release.

  • The MobTrain 2605 is currently scheduled to be released on May 19, 2026. 
  • For release-related information, please review the attached KBA - 2436294.

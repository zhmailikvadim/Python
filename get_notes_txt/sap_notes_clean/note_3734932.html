Symptom

During claim creation, the system is expected to process valid lines even if some lines contain errors.
However, if a single line contains an error, the system stops the entire claim creation process and does not proceed with the remaining valid lines. This prevents claims from being created for the valid entries.


This note will fix the issue so that claim creation continues for valid lines even when some lines contain errors.

Other Terms

/IRM/GCRM, /IRM/GCRDMM, Document Mapping, Claim Creation, line error, claim processing.

Reason and Prerequisites

This is caused by program error

Solution

Please implement the correction instructions. These corrections are only valid if you have applied the following Support Packs.

Release V/4 – 2021, 2022, 2022 V2, 2023, 2023V2, 2023 V3 and 2023 V3 Support Package 1.

1. Perform the manual pre-implementation steps that are described below
2. Use transaction SNOTE to implement the correction instructions specified below and activate the affected program object

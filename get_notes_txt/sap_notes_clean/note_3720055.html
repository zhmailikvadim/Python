Symptom

  • Error message "ME794 Acknowledgment for PO item XXXXXXXX 0010 contains quantity variance" occurs during ORDRSP IDoc processing.
  • The error is triggered when the confirmed quantity in the IDoc segment does not match the ordered quantity in the purchase order.
  • The purchase order does not have underdelivery or overdelivery tolerance maintained.

Environment

  • Materials Management (MM)
  • SAP ERP Central Component
  • SAP ERP
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA
  • SAP S/4HANA Cloud Private Edition

Reproducing the Issue

  1. Create a purchase order without maintaining underdelivery or overdelivery tolerance.
  2. Send an ORDRSP IDoc for order acknowledgment with a confirmed quantity less than the purchase order quantity.
  3. Process the IDoc using transaction WE19.
  4. Observe the error message "ME794 Acknowledgment for PO item XXXXXXXX 0010 contains quantity variance".

Cause

  • The quantity check occurs in the subroutine QUANTITY_CHECK (include LEINMF2E) for IDoc processing. The subroutine QUANTITY_CHECK is only called for the IDoc scenario, ME22N checks the quantity in a different subroutine CONFIRMATION_QTY_DATE_CHECK
  • The error "ME794" is triggered when the calculated value of `f1 = yekpo-menge * (100 - yekpo-untto) / 100` is less than the confirmed quantity.
  • This is a hardcoded error in the IDoc processing logic, and no Exit/BADI is available to bypass it without code modification.

Resolution

Set the overdelivery and underdelivery tolerance in the purchase order to align with actual business requirements.

Ensure that the confirmed quantity in the IDoc segment matches the ordered quantity in the purchase order.

Refer to SAP Note 456127 - FAQ: Electronic Data Interchange (EDI) in purchasing, specifically questions 17 and 18, for further guidance on EDI configuration and quantity checks.

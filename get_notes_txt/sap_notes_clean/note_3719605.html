Symptom

A TA message with RIC deviations is processed and the corresponding account is created with the status "Release by Batch".

The release batch also performs cost estimates and can change the posted values of the TA message.

Other Terms

XDI

TechAccount (TA)

Result-independent conditions (RIC)

Reason and Prerequisites

Prerequisite: 

The TA message has RIC deviations that are above the tolerance threshold defined in Customizing.

The TA message is then reprocessed and the RIC deviations are accepted manually.

Reason: 

After you manually skip the RIC errors, the TA message is processed by default and the account is assigned the status that is defined in Customizing.

Solution

History:

February 23, 2026: New

Troubleshooting Description:

After you manually accept the RIC errors, the settlement of a TA message is always set to "Pending".

A new information message indicates the tolerance value used.

Technical effects: 

Accounts from TA messages whose RIC deviations were accepted manually are always created with the status "Pending".

Manual activities:

The translation for the new information message must be entered manually in the system.

Symptom

  • Error indicates the number range for the group (technical name: SAMMG) in table VBSK (Group Processing for Sales Document Header) has only 5% remaining.
  • In NRIV for object RV_SAMMG, multiple intervals exist; the interval used for delivery collective processing is nearing exhaustion.
  • Questions arise on whether a new interval (for example “04”) can be assigned, whether assignment is hard-coded to interval “01”, and how to link the interval number to delivery collective processing.

Environment

  • SAP S/4HANA
  • Sales And Distribution (SD)
  • Logistics Execution (LE)
  • SAP R/3
  • SAP R/3 Enterprise 4.7
  • SAP ERP Central Component
  • SAP ERP
  • SAP Enhancement package for SAP ERP
  • SAP Enhancement package for SAP ERP, version for SAP HANA

Reproducing the Issue

  1. Check table NRIV for object RV_SAMMG.
  2. Identify the intervals defined for SAMMG and observe that the active interval used for group processing shows remaining capacity near 5%.
  3. During delivery collective processing, observe an error/warning indicating the group number range is nearly exhausted.

Cause

The number range interval for object RV_SAMMG (group SAMMG) is close to exhaustion.

Resolution

Firstly, the `No.range int.assgt.` is defined within the configuration path in SPRO:
Logistics Execution > Shipping > Grouped Deliveries > Define Groups of Deliveries.

This setup determines the number range assignment for grouped deliveries. 

Secondly, the SNRO transaction offers the functionality to define the range for this assignment. If there is a need to modify the range, it would require adjustments to the `No.range int.assgt.` configuration.

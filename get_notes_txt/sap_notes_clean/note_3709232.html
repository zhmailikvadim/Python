Symptom

While creating the change agreement request, the system does not populate the segment E1KOMG in the generated IDOC.
This note ensures that the required IDoc segment is included for the change agreement request.

User makes changes in the agreement request such as updating dates or other details saves and approves the request and then triggers the outbound message The expectation is that the outbound file should capture and send all the change records made in the agreement request. After saving and posting the agreement request the outbound file is generated however the changes made in the request are not captured. The system is trying to compare the old and new data however due to an error in this process the outbound message is created without the required change records.

This note ensures that the comparison process works correctly so that all changes made in the agreement request are captured and reflected in the outbound message.

Other Terms

E1KOMG, CHANGE REEQUEST IDOC, /IRM/IPAGRM, ,/IRM/IPAGRU

Reason and Prerequisites

This is caused by program error.

For V4, SAP Note #3648260 and #3691301 is available

Please implement the correction instructions attached. These corrections are only valid if you have applied the Support Packs.

Symptom

  • Uncertainty regarding the actual order of columns in the HG index for multicolumn primary keys and multicolumn indexes not part of primary keys.
  • Confusion caused by the SAP IQ documentation stating: "When the second form is used (PRIMARY KEY followed by a list of columns), the primary key is created including the columns in the order in which they are defined, not the order in which they are listed."
  • Questions regarding whether the column_id output from sp_iqpkeys can be relied upon for determining the order of columns in the underlying HG index.
  • Inquiry about whether the column_name output from sp_iqindex and sp_iqhelp procedures can be ignored for determining the column order in the HG index.
  • Need for clarification on the relationship between the order of columns in the table and the order of columns in the HG index for multicolumn primary keys.
  • Request for a method to determine the actual order of columns in multicolumn HG indexes that are not part of a primary key.

Environment

  • RHEL 8
  • SAP IQ 16.1 SP05 PL09

Reproducing the Issue

  1. Create a table with multicolumn primary keys using the CREATE TABLE statement.
  2. Define an HG index on the table.
  3. Use sp_iqpkeys to retrieve the column_id output for the primary key columns.
  4. Use sp_iqindex and sp_iqhelp procedures to retrieve the column_name output for the HG index columns.
  5. Compare the order of columns in the table definition, sp_iqpkeys output, and sp_iqindex/sp_iqhelp outputs.

Cause

Resolution

  1. To determine the actual order of columns in the underlying HG index for multicolumn primary keys, refer to the SYSIDXCOL system view. The SYSIDXCOL.sequence column provides the order of columns defined for a multicolumn index, starting from "0".
  2. Use the following query to retrieve the column order for a specific HG index:

```

SELECT

systable.table_name,

sysindex.index_name,

syscolumn.column_name,

SYSIDXCOL.*

FROM

SYSIDXCOL,

sysindex,

systable,

syscolumn

WHERE

SYSIDXCOL.table_id = sysindex.table_id

AND SYSIDXCOL.index_id = sysindex.index_id

AND systable.table_id = SYSIDXCOL.table_id

AND SYSIDXCOL.column_id = syscolumn.column_id

AND SYSIDXCOL.table_id = syscolumn.table_id

AND sysindex.index_name = ''

ORDER BY

SYSIDXCOL.sequence;

```

  1. Do not rely on the column_name output from sp_iqindex and sp_iqhelp procedures for determining the column order of multicolumn indexes, as they do not provide information about the order of columns.
  2. For multicolumn HG indexes that are not part of a primary key, use the SYSIDXCOL system view or the query provided above to determine the actual order of columns.
  3. Alternatively, the iqunload utility can be used to generate the correct order of columns for multicolumn HG indexes.

Symptom

When selecting custom fields as filters in launch form, it shows "No Items to Display".

Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental. 

Environment

  • SAP SuccessFactors Performance Management
  • SAP SuccessFactors 360 Reviews 

Reproducing the Issue

  1. Access the launch form tool.
  2. Select template, launch Later or recurring, review period and proceed to select employee.
  3. Select "Group of Employees".
  4. Select "Find employees by filters".
  5. Click "Select Criteria".
  6. Select custom fields. (Note: for how to configure custom fields as filter, refer to the steps in KBA 2632183) 
  7. "No items to display" is showed and can not select.

Cause

The user does not have permission to "Mass Create Form Instances (Launch forms now)".

Resolution

Grant the user permission "Mass Create Form Instances (Launch forms now)" and assign the corresponding target population.

Then, the user will be able to view custom field values of the subjects within their target population of creating form.

For example: If a user has "Mass Create Form Instances (Launch forms now)" permission for their direct reports as the target population, they will see the custom field values of those direct reports when selecting custom fields as filters in the launch form tool.

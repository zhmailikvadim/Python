Symptom

  • Reading from SAP system using an ABAP dataflow and loading into SQL Server, when having NULL values, throws an error:
  • [Microsoft][ODBC Driver 18 for SQL Server] Invalid character value for cast specification

Other Terms

Keywords

ABAP dataflow, SAP, NULL, BODS, SQL server

Reason and Prerequisites

Environment

  • SAP Data Services 2025 (14.4.0.165)
  • SAP Application system (ECC)
  • Microsoft SQL Server 2019
  • Microsoft ODBC Driver 18

Reproducing the issue

  • Reading from a SAP table which contains NULL values, using ABAP dataflow, will return the error in the symptom
  • Enabling Convert SAP null to null, on SAP Data Services Designer, when using ABAP dataflow the error persists
  • Running with Execute Preloaded or Generate and Execute does not mitigate the issue

Solution

This issue has been forwarded to Development for further investigation.

Workaround:

  • Use SAP Table Reader instead of an ABAP dataflow
  • Map the columns using function nlv() to convert the NULL values

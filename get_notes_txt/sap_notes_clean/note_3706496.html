Symptom

  • The parameter WDSHOWEXPLANATION in the WDA/FPM Application Configuration was previously set to default to show explanations. After an upgrade/patch, this default behavior has disappeared.
  • The parameter WDSHOWEXPLANATION display unexpectedly starting with SAP_UI 7.55.

Environment

  • Web Dynpro ABAP
  • SAP_UI 7.55 and above
  • SAP S/4 HANA

Reproducing the Issue

  1. Set the parameter WDSHOWEXPLANATION to True in the SAP standard test application WDR_TEST_HELP as default to show explanations.
  2. Observe that the default behavior of showing explanations is no longer present - doesn't result in the display of the QuickHelp anymore 
  3. Defining the parameter for the application itself does not work
  4. Adding the parameter as URL-parameter does not work (although all the personalization data has been reset) 

Cause

  • The global parameter WDSHOWEXPLANATION behaves differently starting with SAP_UI 7.55 due to changes in the code within class CL_WDR_CLIENT_APPLICATION method CONSTRUCTOR.

Resolution

  • Apply SAP Note 3705449 - Application parameter WDSHOWEXPLANATION not working.

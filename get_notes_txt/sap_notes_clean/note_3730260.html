Symptom

  • After upgrading agent, tasks start failing.
  • Error message: "Table for owner was not found in the repository for datastore . Import this table from the external source. If the name is case-sensitive in the database (and not all uppercase), enter the name as it appears in the database and use double-quotation marks around the name to preserve the case."

Environment

SAP Cloud Integration for data services

Resolution

  • Re-promote issue tasks from sandbox to production, rerun tasks in production
  • Or select issue tasks->More Actions->Reset Cache, rerun tasks in production

Keywords

table not found in repository for datastore, ci-ds, data services agent, agent upgrade, repository cache, cached atl, task fails in production, import table

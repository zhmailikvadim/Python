Symptom

When importing a migration manager file, it gets stuck before being 100% completed.

Environment

  • SAP Sales Performance Management
  • Advanced Workflow

Reproducing the Issue

  1. Log in to Advanced Workflow as an Administrator
  2. Go to Setup -> Administration -> Migration Manager -> Import
  3. Select the file to import
  4. Instead of the the the file being imported, it gets stuck before 100%.

Cause

  1. File size: If the file is too big, the migration manager might be seemingly stuck while trying to import a big portion of it.
  2. Data issue: If there is malformed data from the file, issues might happen while trying to import. In this case, even if the migration seems to be in progress, a backend issue might have happened.

Resolution

  1. If the file was exported from another environment, try exporting the file and importing it into target environment again.
  2. If the above doesn't work, run the debug tool alongside the migration to check for error messages or other signs that the migration might have ended too soon:
    1. Go to Setup -> Development -> Debug Tool
    2. Check the boxes for "Migration Manager" and "System".
    3. click "Start recording"
    4. Try the import again.
  3. If in the debug tool there are new entries appearing while the progress seems stuck, it means that the file might be large. Solution recommended is to wait for about 30 minutes and check if the migration has finished. If no new entries have appeared in the debug tool, open a case for SAP Support and provide all the information collected, including the debug tool logs and file used for import.
  4. If in the debug tool it shows the "Migration Finished" message even though the progress percentage is below 100%, this means that there is a data issue in one of the components being imported. Raise a case for SAP Support and provide the information collected.

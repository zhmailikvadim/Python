Symptom


A task that has two or more data flows using the same target table has issues in the Monitor Log.

Example:

  • In BTP Neo, after running this task, you see on the Monitor Log tab the State/Row count/Elapsed time/Absolute time for all the data flows (two or more).
  • In BTP Kyma, after running this task, you see on the Monitor Log tab the State/Row count/Elapsed time/Absolute time just for the latest data flow executed. Previous data flow executions are replaced/overwritten by the latest data flow execution.

Other Terms

SAP Cloud Integration for data services - Kyma Server 2511 issue

Reason and Prerequisites

All the data flows of the task started and completed successfully. When we have the same target table being written multiple times by different data flows, the monitor log is replaced (re-started) during the task execution. This issue is just the "monitor log" visualization and there is no data loss or missing records in the target.

Solution

The SAP Development team is aware of the issue and we plan to fix this in future releases.

Workaround

  • Check the full Monitor log on the agent machine.
  • Look into \log\JobServer_DSoD_Agent\dsod_agent_repo__(sandbox or prod).
  • Search for your job name and find the log for the respective timestamp executed.

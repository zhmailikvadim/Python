General Information

  • Note on the field Length of Stay Intensive in file FALL:
    As in previous years, a time specification is entered for a case if the case has at least one case classification to which the internal classification type P2 - Intensive Care §21 KHEntgG is assigned. The times of the case classification(s) are taken into account, added if necessary, and the total is rounded according to the specifications.
    If you have not yet flagged the treatment times for intensive care for a case with such a case classification, you can use the program RNAP21INT, which can be used to set the case classification retroactively.
    For more information, see the following SAP Notes:
    • 2741015 - DE: §21 KHEntgG - Documentation of intensive care and
    • 2741416 - DE: §21 KHEntgG - Report RNAP21INT for documenting 'intensive care' by case classification.

  • Note on the Intensive Care Bed ID field in file FAB:
    As has already been the case in the last years, it is indicated with 'J' for a department stay if the case has a case classification for the period to which the internal classification type P2 - Intensive Care §21 KHEntgG is assigned. If only part-times of the department stay are flagged as intensive care periods by a corresponding case classification, the FAB entries are adjusted accordingly (split).
    If you have not yet flagged the treatment times for intensive care for a case with such a case classification, you can use the program RNAP21INT, which can be used to set the case classification retroactively.
    For more information, see the following SAP Notes:
    • 2741015 - DE: §21 KHEntgG - Documentation of intensive care and
    • 2741416 - DE: §21 KHEntgG - Report RNAP21INT for documenting 'intensive care' by case classification.

  • Note on the plausibility of treatment times with intensive care:
    Since the §21 agreement explicitly lists procedures that are used to classify and check the periods of intensive care for a treatment case, these procedures should be flagged accordingly in the system.
    For more information, see SAP Note 3696485 - DE: §21 KHEntgG - OPS for detecting 'intensive care' (data year 2026).

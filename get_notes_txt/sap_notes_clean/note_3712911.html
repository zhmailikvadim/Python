Symptom

For building savings contracts with granted election allocation, the (election) allocation status is not reset when an assignment harmful contract change is executed.

Reason and Prerequisites

In the task for updating the assignment data "Update Assignment Data" (BSC_TK_CHNG_UPD_ALC), the assignment status is reset only if the substatus "Assignment" is not initial. The substatus Election Allocation is not taken into account.

Solution

The implementation /IBS/CLBSC_TK_CHNG_UPD_ALC of the above Tasks are adjusted in such a way that the assignment status is reset even if the substatus Choice Assignment is not initial.

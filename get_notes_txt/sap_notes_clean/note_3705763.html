Symptom

  • A Incentive Management Portal restore has been performed;
  • Previous API user from the Portal stopped being able to authenticate via API calls while using Basic Authentication;
  • Status code when calling API:
    • statusCode: 401 content: CPOR_03301: The user <user_name> is not able to authenticate.

Environment

  • SAP Sales Performance Management
  • Incentive Management on Legacy HANA

Reproducing the Issue

  1. Make an API call using the affected user using basic authentication;
  2. Receives the following return message "statusCode: 401 content: CPOR_03301: The user <user_name> is not able to authenticate" from the API client.

Cause

Incentive Management Portal restore caused the password of this user to be changed and or updated.

Resolution

  1. If to make the API calls user is set up to use Basic Auth, due to the restore the user password could have been changed, hence the error of not being to authenticate;
  2. You can reset the Portal user password by following this Help Portal article Resetting User's Password;
  3. If the system is using SSO and it's not able to follow the steps from the Help Portal, please create an SAP Support case. On how to do this, please use this article as reference: KBA 3476274  - How to create a case for Sales Performance Management Support team - Best practices - SAP for Me
  4. On the case make sure to have:
    1. The API user username;
    2. TenantID;
    3. And also reference this article;.

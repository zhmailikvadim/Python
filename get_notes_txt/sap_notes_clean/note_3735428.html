Symptom

When processing an externally replenished, sequenced JIT call, a warehouse task is created during stock transfer to PSA after goods receipt for the inbound delivery. Upon confirmation of the warehouse task, the sequenced JIT call containing single component material is completed successfully, whereas the JIS call containing multiple component materials fails to update, resulting in the JIS call remaining open.

Other Terms

Just-In-Time, JIS, Warehouse Task creation, Warehouse Task confirmation, Stock transfer, Sequenced JIT call, multiple component materials.

Reason and Prerequisites

Program error.

Solution

Implement the Correction Instructions of the Note or import the relevant Support Package.

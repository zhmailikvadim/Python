Symptom

Issue 1: 

While copy-creating an infotype 0220 record with retro active start date, there was an inconsistency between PA30 and PUIT_UI transactions. In PUIT_UI, above scenario display error message: "Retro Fund Code creation/change is not possible", but PA30 allows it.

Issue 2:

While creating a superannuation fund record retroactively from the initial screen of transaction code PA30, the record is created successfully.  Instead it should display the error message: "RETRO Fund Code change not possible".

Example:

Consider an employee whose payroll was run for the first period (01.07.2024 – 31.07.2024), with the payroll 'Accounted to' date of 31.07.2024. When we try to create a fund record effective from 01.07.2024 through the initial screen of PA30, it allows the creation, which is inconsistent.

The expected behavior is that it should not allow the creation of the record and should instead display the error "RETRO Fund Code change not possible".

Symptom

When configuring the client certificate for communication between SAP S/4HANA Cloud Private Edition and SAP Business Data Cloud (BDC), you need to specify the Distinguished Name (DN) subject string. Previously, the Organizational Unit (OU) GUID in the DN varied depending on the region and hyperscaler where SAP Business Data Cloud was provisioned. You had to look up the correct GUID from a region-specific table.

With the introduction of the Global Certificate Service, the OU GUID is now a single fixed value 3c869ade-ce89-4ee1-a2ff-a6e617e56fdf  that applies to all regions and hyperscalers. The region-specific GUID lookup table is no longer required.

Reason and Prerequisites

SAP Business Data Cloud has adopted a Global Certificate Service that centralizes certificate management across all regions and hyperscalers. As a result, the OU field in the DN subject now uses one universal GUID instead of region-specific values.

Prerequisites:

  • An SAP S/4HANA Cloud Private Edition system is provisioned and registered in UCL (Unified Customer Landscape).
  • You have access to the SAP S/4HANA Cloud Private Edition system to configure the client certificate.

Solution

 The DN string for the client certificate subject must adhere to the following pattern:

CN=staging, L=<tenantId>, OU=3c869ade-ce89-4ee1-a2ff-a6e617e56fdf, OU=SAP Cloud Platform Clients, O=SAP SE, C=DE

Parameters:

ParameterDescription
CN Must be set to staging.
L The SAP S/4HANA Cloud Private Edition system ID (<tenantId>) assigned to the tenant by UCL (Unified Customer Landscape).
OU (1st) Fixed value: 3c869ade-ce89-4ee1-a2ff-a6e617e56fdf. This is the global identifier for the SAP Business Data Cloud certificate service. This value applies to all regions and all hyperscalers (AWS, Azure, Google Cloud Platform).
OU (2nd) SAP Cloud Platform Clients (unchanged).
O SAP SE (unchanged).
C DE (unchanged).

Note: In certain cases, the SAP S/4HANA Cloud Private Edition system ID (<tenantId>) might have a $ at the beginning (e.g., $0204398045). You must remove the $ from the beginning and use only the remaining part.

Example:

CN=staging, L=850432635, OU=3c869ade-ce89-4ee1-a2ff-a6e617e56fdf, OU=SAP Cloud Platform Clients, O=SAP SE, C=DE

Important: The parameters in the subject must be entered in the exact sequence as shown in the example above.

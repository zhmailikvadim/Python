Symptom

  • A Proforma Invoice (document category = 'U') is not appearing in the Intrastat report.
  • INTRA_REL flag is not set although goods crossing the border in the EU and it should be relevant for Intrastat reporting. 

Environment

  • SAP S/4HANA
  • International Trade Compliance

Reproducing the Issue

  1. Create a proforma invoice. 
  2. Run transaction VE01. 
  3. Check the Intrastat report for the relevant period in transaction /ECRS/RP_EDIT. 
  4. Observe that the proforma invoice is missing.

Cause

  • Proforma invoices are generally not relevant for Intrastat reporting and are excluded by standard logic (The INTRA_REL flag is empty.).
  • Only specific exceptions apply (for example, subcontracting or certain intercompany country constellations involving SAN MARINO and ITALY).
  • Using proforma invoices can lead to multiple invoices for the same shipment, risking duplicate Intrastat reporting.
  • This validation is performed within the following include LV60AF0I (FORM INTRASTAT_DATA_SET)

Resolution

Avoid using proforma invoices for Intrastat purposes to prevent duplicate reporting. If reporting is required, the following solutions can be implemented:   

  1. Enhance the logic at the end of include `LV60AF0I` to either remove or set the INTRA_REL flag for billing documents. 
  2. Configure a custom copy routine at the item level in transaction VTFL to manage the INTRA_REL flag for billing documents. 
  3. Utilize the BADI_INTRASTAT_SELECTION methods: 
    • CHANGE_DATA to exclude specific documents. 
    • GET_PURCHASE_DOCS to retrieve additional purchasing documents. 
    • GET_BILLING_DOCS to retrieve additional billing documents
      Refer to '3409775 Intrastat: How to Exclude or Include Documents in Intrastat Declaration' for additional guidance.

Symptom

After the push is received the pop-up always says "Service Order" even though it is a regular Work Order. Also, after downloading the object, there is no success or failure pop-up that informs the user about the success or failure of the download.

Other Terms

  • SAP Service and Asset Manager
  • SAM
  • SSAM
  • Push
  • Push Notification
  • Service Order
  • Pop-up
  • Work Order
  • Notification

Reason and Prerequisites

Reason:

  • The function that checks if a user is using an FSM persona was not passing the correct context. Because of this, the function assumes that the pushed order was always a Service Order.
  • Client data was used to fetch the Object Type and its details. It was empty after the download finished. Because of this, no pop-up messages of either success or failure were displayed. 

Prerequisites:

  •  BAS or Visual Studio Code with SAM MDK project
  • `patch` command (available on BAS and macOS by default) via terminal or other patching utility  

Solution

The note will fix the problems as follows:

  • Pass a correct context in the function which fetches the header key so it can be displayed in the push pop-up title.
  • Instead of using client data, use state variables to fetch the object key and other necessary details to show in the pop of success or failure.

For the solution download the attached 3472320.patch file and save it in a known location, preferably adjacent to your SAM.mdkproject.

  1. Open a terminal in the root of your SAM metadata (i.e. SAM.mdkproject/metadata or SAM.mdkproject/metadata/base.definitions)
  2. Run the command `patch --no-backup-if-mismatch -p4 < path/to/3472320.patch` in the terminal. 
  3. Verify the following files have been modified.
  • patching file 'Rules/PushNotifications/PushNotificationsAlertMessage.js'
  • patching file 'Rules/PushNotifications/PushNotificationsDownloadFailureAlertMessage.js'
  • patching file 'Rules/PushNotifications/PushNotificationsDownloadSuccessAlertMessage.js'
  • patching file 'Rules/PushNotifications/PushNotificationsViewEntityNav.js'

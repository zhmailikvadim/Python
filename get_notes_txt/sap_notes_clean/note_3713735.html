Symptom

Adding a new Activity to the Activity List. Use Plan Payroll Activities app to re-create the recurrences. The steps under the newly added activity are missing when checking in Manage Payroll Activities app.

This symptom happens if the newly added Activity is identical with a previously existing Activity assigned to the Activity List.

For example, in Activity List "US Monthly Regular Payroll", there is already a Monitoring Activity for Test Payroll for Employee Group 0001. The recurrence of Jan. 2026 is already created. Now user wants to add the same activity again for Employee Group 0002. After changing the configuration, use Plan Payroll Activities to re-create the recurrence. But in Manage Payroll Activities, the newly added activity does not contain any step.

Other Terms

PCC, Payroll Control Center, Plan Payroll Activities, Add Same Activities, Step Recurrence, Activate Recurrence, HRPY_PCC_PAL_V1, HRPY_PCC_MAL_V1

Reason and Prerequisites

This symptom is caused by a program error.

Solution

 Implement the corrections in your system or import the relevant Support Package.

Symptom

  • You are using the XML Digital Signer adapter in SAP Cloud Integration (SAP Integration Suite) to sign an XML payload.

  • The requirement dictates that the Reference URI in the <ds:Reference> tag must point to a specific element ID (e.g., URI="#voucher").

  • However, the output signature currently generates an empty Reference URI (URI=""), which signifies that the entire document is signed, or the targeted element is not being correctly referenced.

  • When attempting to fix this by adding the CamelXmlSignatureContentReferenceUri header, the processing fails with the following error: javax.xml.crypto.dsig.XMLSignatureException: ... ResourceResolverException: Cannot resolve element with ID [ID_Value]

Environment

SAP Integration Suite

Reproducing the Issue

1. Configure the iFlow to digitally sign the payload.
2. Attempt to populate the URI value in the signature reference field.
3. Deploy and test the iFlow.

Cause

The XML Signer does not recognize the "id" attribute as a valid XML ID Type because it is not explicitly defined in an XSD Schema.

Resolution

To successfully generate a signature with a specific Reference URI, you must provide the XML Signer with both the Reference URI value and the XML Schema (XSD) that defines the ID attribute.

Follow these steps in your Integration Flow:

  1. Upload the XSD Schema:

    • Ensure the XML Schema (e.g., factura.xsd) that defines the structure of your payload (specifically the element with the ID attribute) is uploaded to the Resources tab of your Integration Flow.

  2. Configure the Content Modifier:

    • Add a Content Modifier step immediately before the XML Signer step.

    • Navigate to the Message Header tab and add the following two headers:

    NameValueDescription
    CamelXmlSignatureContentReferenceUri#YourIDValue(e.g., #voucher). This sets the specific Reference URI in the signature output.
    CamelXmlSignatureSchemaResourceUrixsd/YourSchema.xsd(e.g., xsd/factura.xsd). This points to the uploaded schema resource, allowing the adapter to identify the ID attribute correctly.

  3. Deploy and Test:

    • Save and deploy the Integration Flow.

    • The XML Signer will now use the schema to resolve the element ID and populate the <ds:Reference URI="#YourIDValue"> correctly in the output.

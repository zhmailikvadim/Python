Symptom

  • Values in fields such as "Division" and "Business Unit" associated with deleted "Vacant" employees persist in permission groups despite the employees being purged from the system.
  • Incorrect values are displayed alongside correct ones in filters and reports.
  • The users show as status "V" in the data inspector. 

Environment

SAP SuccessFactors Employee Central

Reproducing the Issue

  1. Navigate to "Manage Permission Groups" and create a new group.
  2. Select a category, such as "Division," and search for available options.
  3. Observe that incorrect values associated with deleted employees are still visible alongside valid options.

Cause

  • Engineering identified that the issue arises from "vacant" users whose division values persist in the database.
  • The purge process did not remove certain field values associated with deleted employees from the backend, leading to their persistence in filters and reports.

Resolution

If you have the issue arise, please raise a case with SAP Support team and provide the relevant information so engineering can run a script to remove the field values.

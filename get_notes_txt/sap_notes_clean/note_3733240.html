Symptom

This KBA explains two types of solutions for generating the CPF monthly report when the error "The CPF A/C check digit for" and "is not correct!" appear.

"Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental."

Environment

  • SAP ERP
  • SAP S/4HANA
  • Human Capital Management (HCM)
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP SuccessFactors Employee Central Payroll

Cause

The CPF account number of the employee in the table SV of the payroll result is incorrect.

Resolution

Verify whether SAP Note 3681418 - "Error Message for Incorrect CPF Account No. Cases for CPF Reports" has been implemented in system.

  • If the note is not implemented:
    The error appears when checking the CPF account in the previous month's payroll SV table, and it is not feasible to rerun payroll for this scenario.

    1. Exclude the employee from the file generation and manually submit the information via EZPay.
    2. Correct the account data in IT0185, IT0186, and IT0179.
    3. Implement the note to prevent similar issues from occurring in the future.

  • If the note has been Implemented:

    1. Check the current month's payroll result SV table to see if the CPF account is incorrect.
    2. Correct the account data in IT0185, IT0186, and IT0179.
    3. Rerun payroll to correct the payroll results and then regenerate the CPF monthly report.

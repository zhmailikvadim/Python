Symptom

During execution of a regular payroll run, the system retroactively recalculates an off-cycle payroll result within the same payroll period.

Example:

  • Payroll period: 01/2026

  • Off-cycle bonus paid on: 01.01.2026

  • When running payroll (PC00_MXX_CALC), the off-cycle result is recalculated during the regular payroll run.

Observed behavior:

  • Retro calculation is triggered unexpectedly.

  • Wage types (e.g., /560) may appear as negative.

  • Even after deleting payroll results, retro is triggered again.

  • Entries are recreated in table T52RIC.

Cause

This is standard system behavior in SAP Payroll.

  • When an off-cycle payment date falls within the same payroll period as a regular payroll run, the payroll driver includes it in the calculation.

  • The system recalculates the off-cycle result to ensure:

    • Consistency of payroll results

    • Correct cumulations and balances

Additional details:

  • The recalculation does not generate additional payment, but ensures alignment between off-cycle and regular payroll results.

  • If payroll results are deleted, the system recreates retro triggers via table T52RIC, which stores retroactive accounting indicators.

  • Therefore, retro recalculation reoccurs even after deletion.

Resolution

No correction is required, as this is expected standard behavior.

Key points:

  • The recalculation:

    • Is by design

    • Ensures data consistency

    • Does not result in duplicate payments

If this behavior is not desired:

  • Review business process for off-cycle payments:

    • Avoid dating off-cycle payments within the same payroll period, if recalculation should be avoided.

  • Evaluate whether:

    • Off-cycle payments should be processed in a different period, or

    • Payroll expectations need alignment with SAP standard logic.

Important:

  • Preventing this behavior via technical workaround is not recommended, as it may lead to inconsistencies in payroll results.

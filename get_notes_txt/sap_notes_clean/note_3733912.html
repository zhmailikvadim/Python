Symptom

Transaction Code - /PRA/FP_WORKPLACE

During execution of an ONRR-2014 run steps having sufficiently large data volume to process, parallel process grouping mechanism creates an imbalanced load of Federal Payable records across the parallel groups on the basis of sales date and company code combination. This can lead to improper memory resource utilization along with performance issues.

Other Terms

General keywords: Add-on, IS-Oil, Industry Solution, Oil and Gas Upstream, PRA, Production and Revenue Accounting, Roadmap, IS-PRA-XT
Problem-specific key words: ONRR-2014, Workplace, Federal Payable, FP, Parallel Process, Parallel Group, Sales Date, Company Code, Huge Data Volume, Performance Issue, Performance Improvement, Memory Run Out Issue, Load Balancing, Load Distribution.

Reason and Prerequisites

Reason:

While processing Federal Payable (FP) data or PRA Details data through an ONRR-2014 run, the parallel process determination groups the sales date and company code combination randomly, instead of combining them based on number of FP records per combination. As a result, some parallel groups process a comparatively larger data volume whereas, other parallel groups process a very small dataset. This may cause above explained issue.

Solution

As a part of the solution, the parallel process grouping will be done based on the number of records present per sales date and company code combination to balance the load/record distribution.

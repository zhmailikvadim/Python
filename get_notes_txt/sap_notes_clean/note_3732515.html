Symptom

  • Tasks are visible in the SAP Build Process Automation (SBPA) My Inbox.

  • Tasks are not visible in the SAP Task Center web application or SAP Mobile Start.

  • In Task Center Administration, the connector for SAPBuildPA shows a status of OK.

  • Exporting the Task Center Cache (via the /export endpoint) shows that the principalId in task-users.json is an Email Address, whereas the user’s Global User ID in the readme.txt (or IAS profile) is a UUID string.

Environment

SAP Build Process Automation 1.0

Cause

  • There is an identity mismatch between the Task Provider (SBPA) and the Task Consumer (Task Center).
  • SAP Task Center uses the Global User ID (UUID) provided by SAP Cloud Identity Services to map tasks to users.
  • If SBPA is configured to identify users by their email address, the Task Center matching logic fails because email@domain.com does not equal the UUID-string generated by IAS.

Resolution

o resolve the visibility issue, the identity attributes must be aligned in the Identity Authentication Service (IAS):

  1. Verify Identity Mapping in IAS:

    • Access the IAS Administration Console.

    • Navigate to Applications & Resources > Applications.

    • Check the configurations for both the SAP Build Process Automation and SAP Task Center applications.

    • Under Trust > Attributes, ensure the Subject Name Identifier (or the attribute used for User ID) is mapped to the Global User ID (the UUID), not the mail attribute.

  2. Refresh the Task Center Cache:

    • Once IAS is corrected, the existing "email-based" assignments in the cache must be cleared.

    • Go to Task Center Administration > Connectors.

    • Select the SAPBuildPA connector and trigger Repopulate Task Cache.

Reference (For any issues or questions tied to Global UID or IAM) please contact or open a case under the two components below. 

  • Component BC-IAM-IDS: For queries regarding Global UID generation or Attribute Mapping within the Identity Authentication service.

  • Component BC-IAM-IPS: If users are being synced from an external source (like SuccessFactors or Azure AD) and the UUID is missing.

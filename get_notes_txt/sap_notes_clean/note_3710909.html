Symptom

When creating a determination or validation, the "documentLanguage" field does not display the full list of available languages. Only a subset of languages appears in the dropdown selection.

Environment

  • SAP Service Cloud Version 2
  • SAP Sales Cloud Version 2

Reproducing the Issue

  1. Click the User Icon.
  2. Navigate to Settings > All Settings.
  3. Select Extensibility Administration.
  4. Choose Sales Quote Service > Sales Quote.
  5. Go to the Determinations tab.
  6. Click Create a Determination.
  7. In Condition Block 1, set a filter on the "documentLanguage" (Language Code) field.
  8. Observe that not all languages are listed in the dropdown.

Cause

The "documentLanguage" field only retrieves languages that are marked as active in the "System Language" checkbox within the Language, Region, and Currency configuration.

Example API Call:

https://<Tenant_URL>/sap/c4c/api/v1/i18n-service/languages?$filter=isActive%20eq%20true

Response (Filtered by isActive = true):

{
    "value": [
        {
            "code": "en",
            "languageName": "English",
            "description": "English",
            "isInstalled": true,
            "enabled": true,
            "isActive": true,
            "isCommLanguage": true,
            "isRtl": false
        }
    ]
}

Only languages with "isActive": true are displayed in the selection list.

Resolution

In case you need to add additional languages, you would need to make them active into the "System Language" checkbox within Language, Region, and Currency configuration screen:

  1. Navigate to Settings > All Settings.
  2. Navigate to Language, Region, and Currency settings.
  3. Locate the desired language(s) and enable the "System Language" checkbox.
  4. Save the changes.

                                                   

This will ensure the language appears in the "documentLanguage" field when creating determinations or validations.

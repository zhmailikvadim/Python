Symptom

You use the data extractor for the uniform wage notification procedure (program RPLELMC0) and select version ELM 5.

When you create an ELM annual notification (domains: for example, AHV/ALV, UVG, LAW..) for a person subject to withholding tax, the system may issue the following error message:

Entry does not exist in T5C1O_EX: Canton XX, Postal Code YYYY, municipality ZZZ

Since the municipality number <MunicipalityID> is optional in accordance with the swissdec guidelines for wage data processing in the XML message part of the personal data <Particulars>, the output of an error message is not appropriate here in the case of inconsistent address data.

Other Terms

Payroll Switzerland, ELM ("Uniform Wage Notification Procedure"), ELM 5.0, domain QSG (cross-border employees), class: CL_HR_B2A_CH_ELM_50 (method ELM_GET_PERNR_DATA_YEARLY)

Reason and Prerequisites

Program correction

Solution

Import the HR Support Package relevant for your release. If you have to implement the solution in advance, use the SAP Note Assistant to implement the correction instructions relevant for your release. You can find more information about the SAP Note Assistant on SAP Support Portal at https://support.sap.com/notes.

After you implement this SAP Note, the preassignment of the municipality number (<MunicipalityID>) is removed from the cluster table ST in the standard SAP system for an ELM annual report. The address data is still checked against the Customizing of the table T5C1O_EX of an explicitly set municipality number in the BAdI HRPAYCHELM01.

Symptom

During the process of comparing data, Data Assurance (DA) is reporting the error:

S 20XX-01-21 10:49:26.026 COMPARE ExecutionResultFetch.run@34245 java.lang.IllegalStateException at com.sybase.da.agent.conn.ConnectionManager.getAndRetry(Unknown Source)

How is this resolved?

Environment

SAP Replication Server (SRS) Data Assurance all versions

Cause

Data Assurance has run out of connections to the database.

Resolution

  • The error suggests DA has run out of connections to the database.
  • Make sure the connection pool is configured large enough to cope with the concurrent comparisons:
    config db_connection_pool_size 15.

Symptom

You run the Payroll Driver (USA) (PC00_M10_CALC) transaction for an employee who has an additional withholding amount in the Withholding Info W4/W5 US (0210) infotype, and the following conditions are met:

  • The available gross is insufficient to deduct all employee taxes, including the additional withholding amount.
  • The additional withholding wage type is configured in the system (see SAP Note 1836745 - Additional Withholding Over Arrears) and it has the arrears characteristic set to Deduct what is possible (2) in the Characteristics of Arrears Deductions (V_T51P6) view.

In this scenario, the system incorrectly deducts the full additional withholding amount and adds the full amount to the Deductions not taken (DDNTK) table as if it was not deducted.

Other Terms

PC00_M10_CALC; RPCALCU0; Additional Withholding; IT0210; Extra amount.

Reason and Prerequisites

Reason

This issue is a side effect caused by the solution delivered by SAP Note 3416936 - TAX: Wrong additional withholding amount over arrears when deduction is to be taken as all or nothing.

Prerequisites

The following Support Package level is the minimum prerequisite:

S4HCM 102 HR SP 00
S4HCM 101 HR SP 05

SAP_HR 6.08 HR SP C9

For more information, access the SAP ERP Human Capital Management - Support Package Schedules (SAP_HR) or SAP S/4HANA On Premise Human Capital Management (S4HCM).

When you install an SAP Note, some prerequisites might be required. The prerequisites are SAP Notes that have already been released and that change the same context of the current one. Then, even if these prerequisites are not necessarily related to the symptom of this SAP Note, you must implement them due to technical reasons.

Solution

From now on, the system correctly calculates the additional withholding amount when the available gross is insufficient and properly records only the actual not deducted amount in the DDNTK table.

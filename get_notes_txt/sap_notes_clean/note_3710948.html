Symptom

When executing transaction VL10D (Activities Due for Delivery) for a specific Stock Transfer Orders, no due deliveries are displayed and the system displays the message “List does not contain any data.”

Environment

SAP S/4HANA Cloud Public Edition

Reproducing the Issue

  1. Open the ME21 app (Create Purchase Order).
  2. Select Order Type as Stock Transfer Order.
  3. Enter the required details, such as Vendor/Plant/Material/Quantity.
  4. Save the purchase order.
  5. Execute transaction VL10D.
  6. Enter Selection Criteria such as purchase order number and execute.

Cause

The Stock Transfer Order (STO) cannot appear in VL10 (Activities Due for Delivery) because the Delivery Creation Date in the Shipping tab is not populated. This occurs when the associated purchase order is still “In Approval”, preventing the STO from being released and the delivery date from being automatically set.

Resolution

In order to make the Stock Transfer Order (STO) available for processing in VL10 (Activities Due for Delivery), follow these steps:

  1. Approve the purchase order in order to release the STO.
  2. After release, the system will automatically populate the Delivery Creation Date in the STO’s Shipping tab.
  3. Once the Delivery Creation Date is set, the STO can be processed in VL10 transactions to create deliveries.

Symptom

 
This Note is for documenting the gzip compression capability in the REST API to inform developers about the Accept-Encoding header support and its performance impact on response payload sizes and network efficiency.

Other Terms

GZIP, COMPRESSION, ACCEPT_ENCODING, CONTENT_ENCODING, HTTP_HEADERS, JMETER, PERFORMANCE_TESTING, LOAD_TESTING, NETWORK_EFFICIENCY, BANDWIDTH_OPTIMIZATION

Reason and Prerequisites

Reason: API consumers are transmitting uncompressed responses, consuming a lot more bandwidth than necessary. The Accept-Encoding header support has been added but is undocumented. Documentation update required to enable developers to leverage gzip compression for improved network efficiency.

Prerequisites:

- Performance testing results from JMeter showing compression behavior (POST/PATCH/GET operations)
- Comparative analysis data:
* Average bytes with/without compression
* Response time impact measurements
* Network efficiency metrics
- JMeter tool for compression testing verification (Postman/Hoppscotch not recommended as they send Accept-Encoding by default)
- Sample test data for POST/PATCH operations (5 iterations minimum)
- Knowledge of:
* OpenAPI/Swagger tags section structure
* HTTP header documentation conventions
* gzip compression behavior and trade-offs
- Access to API documentation repository/system for updates
- Understanding of target audience (API developers/consumers)

Solution

1. Updated OpenAPI YAML - Tags Section:
- Added Accept-Encoding header documentation
- Documented gzip compression support and behavior
- Included performance characteristics (98% payload reduction, 10-17ms overhead)

2. Performance Testing Completed:
- JMeter testing: 5 iterations of POST/PATCH/GET operations
- Results: Average bytes reduced to 0.017-0.02x with compression
- Response time increase: 10-17ms (acceptable trade-off, the difference might decrease in the case of bigger networks.)
- Test data attached: Excel sheet with comparative metrics

3. Developer Guidelines Added:
- Recommendation to use Accept-Encoding: gzip for production
- Testing best practices (use JMeter, not Postman/Hoppscotch)
- Network efficiency improvements documented

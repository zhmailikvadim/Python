Symptom

In Transaction SEGW, it is not possible to edit the OData Project 'BILLOFMATERIALV2'.

You see that the 'Delete' button is disabled and on click of 'Change', you receive the Error 'SAP delivered projects cannot be edited in your system'.

Environment

  • SAP ERP Logistics (LO)
  • SAP S/4HANA, on-premise
  • SAP S/4HANA Cloud Private Edition

Cause

Reason: This is a System Behavior

It is prohibited to change the standard SAP Objects and the corrections pertaining to the OData Project are to be applied via SP.

However, it is seldom possible to wait for the next SP after SAP Note Implementation containing the changes in the UI5 Artefacts.

Resolution

In a situation such as this, below are the steps that need to be carried out for immediate consumption of the Solution.

To correct the 'Editing Prohibited' error in SEGW, please follow the below mentioned steps and then perform the manual activity:

1. Go to Transaction SE03 -> Change Object Directory Entries.
2. On the next screen, enter 'IWPR' in the Input field right below the DTEL Data Element and next to the IWPR field, Please enter the OData Project 'BILLOFMATERIALV2'. 'Enter'.
3. Select the checkbox against the newly maintained entry and click on the 'Execute' Button.
4. On the next screen, you will see the Package 'CS_ODATA' and the OData Project 'BILLOFMATERIALV2'.
5. Position the Cursor on the Object 'R3TR IWPR BILLOFMATERIALV2' and click on the 'Change Object directory' Button or Press Shift+F7
6. Change the Original System to be your own System ID and Click the save button.

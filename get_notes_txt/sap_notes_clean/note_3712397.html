Symptom

When running transaction CURTOADV_CREATE, no PDS generation log is displayed during the first execution. Re‑executing the same selection immediately afterward produces the expected log output.

Other Terms

CURTOADV_CREATE, PDS, PPDS_ERP_CUSLNTRTO_CIF_REPORT, integration

Reason and Prerequisites

This behavior occurs after upgrading to SAP S/4HANA Cloud, private edition 2025 SP00. Due to a missing ELSE section in report PPDS_ERP_CUSLNTRTO_CIF_REPORT, the PDS option table was not populated correctly during the initial run. A missing branch in the selection logic prevented the default PDS option entry from being appended. As a result, the first execution could not be completed and produced no log.

Solution

This behavior is corrected by implementing the attached SNOTE correction instruction. The correction adjusts the PDS option handling coding.

Environment

  • OS Independent
  • DB Independent

Reproducing the Issue

  1. Run the Software Provisioning Manager to perform a dual-stack split or a system copy

Cause

  • The java class com.sap.engine.offline.OfflineToolStart contained in the JAR file sap.com~tc~bl~offline_launcher~impl.jar cannot be read.  Either the file is missing or there is insufficient permissions on the JAR file.

Resolution

At first, ensure that the used/started SWPM Service is for the exact release version of the source system Java stack. The Release and component version of NW AS Java stack can be checked using SAP KBA:1757810.

Different SWPM Tools for different NW AS Java stacks are reachable according the SAP Notes:

for AS Java stack based on NW 7.1 and higher: 1680045 - Release Note for Software Provisioning Manager 1.0 (recommended: SWPM 1.0 SP30)

for AS Java stack based on NW 7.0#: 2595196 - Release Note for *70JDS*.SAR of Software Provisioning Manager 1.0 - Covers No Longer Supported Java and Dual-Stack Options and Access to Guides for Software Provisioning Manager 1.0 Java and Dual Stack

 ---

Note: All directory paths are taken from the sample error message in the symptom section

  1. In the command specified in the error message, make sure that the file/directory after “-classpath” exists
    • For example:
      • Execution of UTL tool ‘/usr/sap/<SID>/<instance>/exe/<sapjvm version>/bin/java -classpath /sapmnt/<SID>/global/sltools/sharedlib/sap.com~tc~bl~offline_launcher~impl.jar’
  2. If the file exists, then check the permissions as per KBA 2096129 - Exception in thread "main" java.lang.NoClassDefFoundError: com/sap/engine/offline/OfflineToolStart
  3. If the file/directory does not exist, then the directory needs to be created and the files added.
    • The file/directory is part of the archive delivered with:
      • the JAVA DVD:  
        • /<java dvd location>/DATA_UNITS/JAVA_J2EE_OSINDEP_J2EE_INST/<J2EEINSTALL.SAR or J2EEINSTALL.SDA>
      • Or the SWPM tool: 
        • /<swpm tool location>/JAVA/<NW version>/J2EEINSTALL.SDA
  4. Copy the archive to the path after "-classpath" in the error
    • For example: 
      • /sapmnt/<SID>/global/sltools/sharedlib
  5. Extract the contents of the archive to the folder path
    • For J2EEINSTALL.SAR use SAPCAR.
      • For example:
        • SAPCAR -xvf J2EEINSTALL.SAR
    • For J2EEINSTALL.SDA use winzip or winrar

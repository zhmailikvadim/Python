Symptom

  • In transaction EDOC_COCKPIT, descriptions and button texts are not displayed correctly when logged in with Polish language (PL).
  • Some texts appear blank or missing.

Environment

  • SAP ERP 
  • SAP S/4HANA
  • SAP Document and Reporting Compliance Service

Reproducing the Issue

  1. Log in to the system using language Polish (PL).
  2. Execute transaction EDOC_COCKPIT.
  3. Observe the toolbar and status descriptions.
  4. Some descriptions or texts are missing or displayed as blank.

Cause

The issue occurs due to missing language-dependent entries (translations) in the system tables.

Standard translations are delivered via Support Packages or language deliveries.
If these are not available in the current system level, the corresponding entries remain missing.

Resolution

Maintain the missing language entries manually as a temporary workaround.

  1. Go to transaction SM30.

  2. Maintain entries in the following tables for Polish language PL:

    • EDOFUNCTIONV (eDocument function codes)

    • EDOPROCSTATV (eDocument status texts)

  3. Ensure required function entries (e.g. GET_DRCC_STATUS, RETRY) are maintained for Polish language.

  4. Save and retest in EDOC_COCKPIT.

Translations will be automatically available once delivered via future Support Packages.

For guidance on maintaining missing translations, refer to:

  • SAP Note 2731171 - eDocument Cockpit button labels are blank due to translations

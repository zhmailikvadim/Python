Symptom

During the processing of the PHI data delivered by ELStAM using the report ELStAM PHI: Process Notifications (RP_PAYDE_E2_PKV_PROCESS_NOTIFS), some notifications remain in the status "To Be Processed".

Environment

  • Human Capital Management (HCM)
  • SAP R/3
  • SAP ERP Core Component
  • SAP ERP
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
 

Cause

In certain cases, there may be such an entry in the table V_P01E2_PKV_ST_A for employees whose system ID does not match the currently used system ID.

This constellation typically arises after a system switch, a legacy data transfer, or a migration if historical entries with an earlier system ID remain in the system.

When assigning the inbound notification, the report ELStAM Procedure: Transfer XML Data to ELStAM Table (RPCE2SD0_IN) reads the last outbound notification for a personnel number and transfers its system ID. If the last message was transferred from an earlier system, the described problem occurs.

Solution

  1. Implement SAP Note 3701683 - ELStAM PHI: Notifications are not processed during system switch.
  2. Adjust the source code of the method GET_SUCCESSOR_SYSTEM of the class CL_HRPAYDE_SV_ADMIN in accordance with the source code example in the SAP Note text.
  3. Execute the report RP_PAYDE_E2_PKV_PROCESS_NOTIFS again for the affected personnel numbers.

Symptom

Universal Allocation run was cancelled or failed, but the run still shows with "In process" status in Allocation results.

The run cannot be aborted.

Other Terms

F3548, Universal Allocation, Run Allocation, Allocation results, Run Status, In process

Reason and Prerequisites

In case the allocation run was interupted unexpectedly, it is possible that the background job has finished, but the allocation run is stuck in 'In process' status.

 The run can not be aborted, because the job is no longer active or exists.

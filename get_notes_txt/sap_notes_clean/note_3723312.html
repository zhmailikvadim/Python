Symptom

This article explains the process used to sync FO/GO Objects and Picklists from SuccessFactors to SmartRecruiters.

Environment

SmartRecruiters for SAP SuccessFactors

Resolution

Prerequisites

To access the sync page, you must have the following permissions:

You have the following permission: Administrator Permissions → SmartRecruiters → Data Mapping for SmartRecruiters.

Procedure

Once you’ve decided which SuccessFactors fields are needed in the recruiting process, you need to sync them with SmartRecruiters. To do that, you first need to configure a mapping between the SuccessFactors fields you want to sync and the corresponding SmartRecruiters fields you want to sync them with.

  1. Go to Admin Center → Set Up SmartRecruiters Field Mapping And Data Sync;
  2. Choose Add Fields to add SuccessFactors fields to the mapping. Use the search box to find SuccessFactors fields and select the ones that you want to sync. Choose Add to add them to the mapping;
    • Fields are displayed according to their parent-child relationship in SuccessFactors, with the child field indented under the parent field.  If you select fields with a parent-child relationship, they are created in SmartRecruiters with a dependency relationship. 
  3. For each field mapping, specify the category (Job or Org) of the corresponding SmartRecruiters field that will be created;
  4. To save your changes, choose Save Mapping. On save, the mapping is validated.
  5. If no error messages are displayed, the validation is successful and the Initiate Sync button is displayed.

When the Initiate Sync button is displayed, you can initiate the data sync.

Important Notes:

  • The following field types are supported:
    • Standard and Custom Picklists
    • Legacy Foundation Objects: location, locationGroup, geozone, jobCode, payComponent, payComponentGroup, payGrade, payRange, frequency, eventReason
    • Standard and Custom MDF objects (Generic Objects)
  • Once the data sync is initiated, it can’t be paused or canceled and the field mappings can’t be modified or deleted.
  • For each field mapping, a sync status is displayed:
    • "In Progress" indicates that the sync is still in progress.
    • "Complete" indicates that the sync completed successfully for that
      field.
    • "Failed" indicates that the sync didn't complete successfully for that
      field
  • There's a limit to the number of fields you can add to the mapping. If you've exceeded the limit, you can't save the mapping. As an alternative, deselect some fields in the Set Up SmartRecruiters Field Mapping And Data Sync tool, or delete some fields in SmartRecruiters, then try again. Careful planning is essential to avoid permanent, unnecessary fields.
  • There is no limit to the number of relationship levels that can be transferred to SmartRecruiters.
  • Syncing values from already mapped Objects and Picklists is currently not available, but it will be included in future updates. This means that if a change is made to an Object or Picklist value, the change will not reflect in SmartRecruiters.
  • When synching SuccessFactors fields to SmartRecruiters, if a field with the exact same name already exists in SmartRecruiters, a new field is created with a unique name. The pre-existing SmartRecruiters field will not be updated.
  • Admins should not select fields inside objects that reference other objects. Specifically avoid:
    • Fields within a Foundation Object (FO) that point to another FO or object.
    • Fields within a Generic Object (GO) that point to another FO or GO.
      • These field-level references are not supported for hierarchy modeling and will not function correctly.

Results

For each field mapping that completes successfully, the corresponding fields are created in SmartRecruiters and populated with data values from SuccessFactors HCM. Translations for field labels and field values are also synced to SmartRecruiters. From this point on, the data can only be managed in SuccessFactors HCM. On SmartRecruiters administration screens, the fields are "read-only" and the values can't be changed.

Common Issues:

  • Can't add fields to mapping: If you can't add new fields to the field mapping, check to see if a sync job is currently running. If so, wait for the sync to complete, then try again. You can't add or edit the mapping while a sync is running.
  • Can't save mapping: 
    • A sync is currently running - Check to see if a sync is already running. If so, wait for the sync to complete, then try again.
    • A field category is missing.
    • Too many fields mapped.
    • Parent-Child relationship doesn't match -     
      The parent-child relationship for some of the selected fields doesn't match the SuccessFactors field relationships. Deselect the fields and add them again.
  • Sync Status "Partially Completed" - The sync status Partially completed indicates that the SmartRecruiters fields were created successfully, but not all of the values were synced. The system automatically attempts to run the data sync again.

For other errors, you can also refer to this page.

Symptom

  • A value help defined for a parameter in a CDS view does not work. 
  • The application or service preview does not function when the parameter Value Help is active.
  • The Value help annotations appear exposed in $metadata, but are not interpreted correctly in the UI.

Environment

  • ABAP CDS with @Consumption.valueHelpDefinition
  • OData V4 service binding preview / Fiori elements V4
  • S/4HANA 2023 FPS03

Reproducing the Issue

  1. Create or use an ABAP CDS view with parameters and define a value help via @Consumption.valueHelpDefinition (for example referencing a standard value help entity).
  2. Expose the CDS view in an OData V4 service binding and activate it.
  3. Start the service binding preview or run a Fiori elements application that consumes the service.
  4. Attempt to open or use the value help for the parameter.
  5. Observe that the value help does not open or the application does not work.

Cause

A defect causes the generated ValueListMapping for parameter value helps to set all parameters to ValueListParameterDisplayOnly, leaving no ValueListParameterInOut mapping and preventing the value help from working.

Resolution

  1. Implement SAP Note 3551365 to fix the ValueListMapping generation for parameter value helps.
  2. Reactivate the affected CDS view/service binding and retest the value help in the service binding preview or application.

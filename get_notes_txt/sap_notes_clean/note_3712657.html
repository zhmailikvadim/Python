Symptom

  • MDK client application crashes on iOS 26.2 when attempting to create the client.
  • Error message observed: "dyld[34513]: Symbol not found: _$s6ArcGIS7FeatureC10attributesSDySSs8Sendable_pGvgTj".

Environment

SAP Service and Asset Manager

Reproducing the Issue

  1. Upgrade the MDK version from 24.7.9-002 to MDK 25.9.1.
  2. Use SSAM client based on MDK 25.9.1 for devices running iOS 26.x.
  3. Create the client.
  4. Observe the application crash with the error message: "dyld[34513]: Symbol not found: _$s6ArcGIS7FeatureC10attributesSDySSs8Sendable_pGvgTj".

Resolution

  1. Ensure the base SAM.mdkproject from the corresponding 2511 branding SDK is used.
  2. Verify the MDK version is updated to 25.9.1 as recommended.
  3. Rebuild the client using the correct branding SDK.

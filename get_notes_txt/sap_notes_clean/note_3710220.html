Symptom

  • User received a work item to re-activate a work order, but there is no action to do
  • I received a work item to re-activate a work order, but the work order is already in 'Confirmed' or 'Closed' status

Reproducing the Issue

  1. Create a Work Order (contingent or services)
  2. Activate the Work Order
  3. Edit the Work Order
  4. If the Work Order is not activated as soon the update is done, a work item will be sent to re-activate

Cause

When a Work Order in 'Active' status is edited, the status changes to 'Approved' and the next is step is 'Need to Activate'. Then the work item 're-activate' is sent to the Worker Primary Contact.

Resolution

If you received the work item 're-activate' and there is no action to activate the work order, it means that the work order was already activated by another user.

Once the work order is active, the associated work item should be removed automatically.

If it remains, please contact the support team to request removal as it is not possible to remove the work item from the user interface.

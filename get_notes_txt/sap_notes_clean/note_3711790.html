Symptom

  • Incorrect date format observed in Excel exports after SAP S/4HANA Cloud Public Edition 2602 upgrade.
  • Incorrect date format observed with SAPUI5 1.138 - 1.144
  • The problem occurs when the date, time, datetime patterns are customized (user settings) and the customization differs from the default settings for the user locale. E.g. if a user has a German locale and the customization defines the format YYYY-MM-DD 

For example:

  • Date format is configured in the User Profile settings as dd.mm.yyyy.
  • Date format is displayed in the applications as dd.mm.yyyy.
  • The date format in the exported Excel files display dates as m.dd.yy.

Environment

SAP S/4HANA Cloud Public Edition 2602

SAP Fiori Launchpad

SAP Build Work Zone

SAPUI5 1.138 - 1.144

Reproducing the Issue

  1. Change the date format in the "User profile > Settings > Language and Region" as dd.mm.yyyy.
  2. Export data from SAP application (e.g., Display Line Items in General Ledger) to Excel.
  3. Observe that the exported Excel file displays dates in the format m.dd.yy instead of dd.mm.yyyy.

Cause

During code harmonization, an incorrect API was utilized to retrieve default export settings for date, time, and datetime patterns. This regression caused the system to apply only standard locale settings, ignoring custom format configurations.

This only affects date, time, and datetime patterns in UI5 client exports when custom formats are used. Other components are not impacted.

Resolution

A fix will be delivered with HFC04 (14-15. February) and SAPUI5 1.142.6.

Please note the release date might be subject to change.

                 

Workaround:

Users can manually adjust the date format in the exported Excel file as a temporary workaround.

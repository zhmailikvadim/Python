Symptom

When processing payments via the F110 transaction using an invoice and a credit note, the system correctly calculates the net value by subtracting the credit note from the invoice. However, the bank remittance (TXT file) sent to the bank omits the rebate value (credit note). This omission causes the bank to reject the payment due to inconsistencies between the payment and invoice amounts. 

Environment

SAP S/4HANA Cloud Private Edition

Reproducing the Issue

  1. Run automatic payments with transaction F110 including at least one invoice and one credit memo in the same payment.
  2. Generate the CNAB 240 remittance file via DMEEX.
  3. Check segment 12.3 (discount/abatement) in the output file: the value from the credit memo is not populated.

Cause

By the standard solution design, credit memo deductions are not treated as “discount”; nominal amount (11.3) already includes credit memos, and amount paid (15.3) equals the nominal amount.

Resolution

If the business requires showing the credit memo deduction explicitly in field 12.3, it is necessary to implement a custom calculation, as the standard solution is not designed to do this.

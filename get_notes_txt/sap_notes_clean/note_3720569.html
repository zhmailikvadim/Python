Symptom

In the original setup, the Implementation Status value is set to "P," while the new copy defaults to Implementation Status "I."

Environment

SAP S/4HANA Cloud Public Edition

Reproducing the Issue

  1. Create a company code in the lower system.
  2. Verify the implementation status at the company code level in General Ledger.
  3. Observe that the implementation status remains set to "In Process".

Cause

In the original setup, the implementation status (IMST) value is set to "P," while the new copy defaults to IMST "I". SAP has designed the system to default IMST to "I" to encourage users to manually assign the IMST value to "P" only for G/L accounts that specifically require it. This approach ensures greater accuracy and intentionality when configuring accounts. That is the reason the new Accounts all get IMST "I" 

Resolution

Use app "Manage operating G/L Accounts" to mass change the "Implementation Status". Just select the G/L Accounts and change the "Implementation Status" to "P" via the mass change functionality.

  1. Open the application "Manage operating G/L Accounts".
  2. Select the G/L Accounts.
  3. Change the "Implementation Status" to "P (Productive Use)" via the mass change functionality.
  4. Release the respective customizing transport.
  5. Verify that the G/L accounts are successfully synchronized to the Production environment.

Symptom

Restricted Recruiters should not be able to release their Requisitions on their own, they should be prompted to request approval for the Requisition to be released.

This article discusses the business logic for that process.

Environment

  • SAP E-Recruiting or SAP E-Recruiting for S/4HANA
  • WebDynpro UI

Cause

The release of a Requisition verifies the authorizations of the User doing so. Particularly, it verifies for Authorization Object P_RCF_STAT ("Object Status in E-Recruiting") to see if the User has authorization for object type NB ("Requisition") status 01 ("Released").

If the User does not have authorization for it, instead of giving an authorization error message, the User is redirected to the page for requesting the release of the Requisition.

If the User does have authorization for it, it is able to release the Requisition as a normal non-Restricted Recruiter.

Resolution

If a Restricted Recruiter is able to release a Requisition without being prompted to request approval, verify the Roles assigned to their User to see if any of them grants the authorization to do so.

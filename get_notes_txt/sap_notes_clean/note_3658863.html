Symptom

When importing ItemMasterAML records, only a subset of the data is successfully imported. Although the Master Data payload shows a completed status, many expected records are missing in the Master Data Search tab.

Reproducing the Issue

After importing a ItemMasterAML payload in SAP Ariba Sourcing, check below steps: 

  1. Go to Administration > Master Data Manager
  2. Click Master Data Search
  3. Select the correct External System
  4. Select the Master Data Type ariba.masterdata.ItemMasterAML
  5. Click Search 

Check if there are any missing records. 

Cause

The Material Number field value must be unique. As a result, the system cannot create multiple records using the same Material Number, even if the associated types or specifications differ.

Resolution

Ensure that each record is assigned a unique Material Number. This prevents conflicts during the import process and allows the system to create multiple records successfully.

If records share a common base, you can append a suffix (e.g., _1, _2, etc.) to differentiate each Material Number while maintaining logical grouping.

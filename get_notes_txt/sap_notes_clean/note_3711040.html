Symptom

You use the year-end bonuses (13th monthly wage) in Payroll Switzerland and have already implemented the related SAP Note 3432272.

In the case of a payment of the year-end bonus controlled explicitly using the BAdI HRPAYCH_YEARENDPAYM (CV_FORCED_PAYM = 'X'), the year-end bonus may not be paid that is based on future master data changes (for example, cost center change in the following month).

Other Terms

Payroll Switzerland, 13th monthly wage, BAdI: HRPAYCH_YEARENDPAYM, class: 'CL_HRPAYCH_JAHRESENDZULAGE', method: PROCESS

Reason and Prerequisites

Program adjustment

After you implement SAP Note 3286091, the previous processing of the year-end bonus in the function module HR_CH_JAHRESENDZULAGE was converted by a new processing in the class CL_HRPAYCH_JAHRESENDZULAGE.

Solution

Import the HR Support Package relevant for your release. If you have to implement the solution in advance, use the SAP Note Assistant to implement the correction instructions relevant for your release. You can find more information about the SAP Note Assistant on SAP Support Portal at https://support.sap.com/notes.

After you implement this SAP Note, in the case of an explicitly controlled payment of the year-end bonus (BAdI HRPAYCH_YEARENDPAYM, CV_FORCED_PAYM = 'X'), a proportional payment of the year-end bonus (wage type /484) also takes place if the period (BEGDA/ENDDA) of the structure CS_5C3H_COMPARISON (CHANGING) corresponds exactly to the corresponding fields for the following month (that is, the period (BEGDA/ENDDA) of the structure CS_5C3H_COMPARISON (T5C3H) must correspond exactly to the corresponding fields for the current month and the corresponding fields must be exactly the same as BEGDA/ENDDA the current fields for the control of the current month.

Example "Payroll period 01/2026 with payment of year-end bonus in January"

Parameters Values Remark
CS_5C3H_COMPARISON BEGDA = 01.02.2026
ENDDA = 28.02.2026		 BEGDA/ENDDA corresponds to the following month
All other values of structure 'T5C3H' are initial
CV_FORCED_PAYM 'X' Payment of year-end bonus

If the payments of the year-end bonus triggered explicitly by the BAdI HRPAYCH_YEARENDPAYM result in side effects, for example, in the processing of the year-end bonus in the standard SAP system, these are not subject to SAP standard maintenance.

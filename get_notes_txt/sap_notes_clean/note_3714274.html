Symptom

You are replicating business partners into SAP Cloud for Customer (C4C) but get the following error:

Entry in field UID Number is longer than N characters

Environment

SAP Cloud for Customer

Cause

The problematic part of the payload is as per the following example:

<TaxNumber actionCode="04">
    <CountryCode>CH</CountryCode>
    <TaxIdentificationNumberTypeCode>1</TaxIdentificationNumberTypeCode>
    <PartyTaxID schemeID=" ">CHE123456789</PartyTaxID>
</TaxNumber>

The ERP Tax Number Type is made up of CountryCode and TaxIdentificationNumberTypeCode, so in this case that results in 'CH1'.

Tax Number Type 'CH1' corresponds to attribute 'UID Number' as seen in Business Configuration > General Business Partners > Maintain Tax Number Types. 

Attribute 'UID Number' is regulated by rule 'Tax Number Two Allowed Input Length' within the Tax Attributes section of the relevant country ('CH' in this case).

In this example, UID Number is 'CHE123456789', which has a number of characters greater than what is defined by rule 'Tax Number Two Allowed Input Length', hence the error.

Resolution

Either increase the number of characters accepted by the rule, or else make sure you do not send a UID Number greater than what's allowed by the rule.

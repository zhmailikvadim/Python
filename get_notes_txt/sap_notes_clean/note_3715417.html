Symptom

When you execute the report SI: Get Data from SI Agencies  (RP_PAYDE_B2A_SV_COLLECTOR), longer runtimes occur if data for the notification procedures euBP and/or DaBPV is retrieved.

Reason and Prerequisites

When the retrieved data is saved to the database, the system checks whether this data is already in the system and is therefore obsolete. For the retrieved file numbers, the system reads the database table with the data packages already retrieved in the past that have the same file number. The system then checks whether the data are the same. If this is the case, the newly retrieved data packages are set to the status Obsolete.

In the euBP and DaBPV procedure, the data packages are all transferred with the file number 1. As a result, the system checks for all previously retrieved data packages whether they have the same user data as the currently retrieved data. This leads to longer runtimes.

Solution

Implement the attached correction instructions or import the specified support package. After you import the HR Support Package or implement the correction instructions, you do not have to perform any further follow-on activities.

After you implement the corrections, the check is no longer performed in the euBP and DaBPV procedures. This does not make sense due to the transfer with file number 1. Data collected twice should not have any negative effects in these procedures.

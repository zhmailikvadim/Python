Symptom

  • The Output File Repository Server (OFRS) directory is increasing indefinitely in size in BI 2025
  • Instances which are cleaned up by instance limits or deleted manually do not have their corresponding FRS entries removed
  • In the CMS logs, errors are observed eg:
    assert failure: (TombstoneCleaner.cpp:173). (false : CTombstoneCleaner::CleanupTombstone: Missing SI_PATH on Tombstone.)

Reason and Prerequisites

Environment

  • SAP BusinessObjects Business Intelligence (BI) Platform 2025
  • Windows
  • Linux / Unix 

Steps to Reproduce

  1. Implement instance limits
  2. Schedule a report to run every few minutes
  3. Observe once the report instances reach the instance limit value and start being deleted the corresponding FRS entry is not deleted

Cause

The tombstones that get created for the instances as part of the hot backup process during deletion are being created without a path to the FRS entry, meaning the files are not deleted

Solution

The issue has been forwarded to development for further investigation

Workaround

To ensure deletions of future FRS entries, disable hot backup from CMC -> Settings

To clean up existing FRS entries, use reposcan:
1689163 - Using the Repository Diagnostic Tool (aka reposcan) in BI 4.x - for WINDOWS [How-To]
1752397 - How to use the Repository Diagnostic Tool (aka reposcan) in BI 4.x - for Unix / Linux

Symptom

When you post a payment using payment order lot FP45, a dump occurs with message (>0 577): "In number range &1, correct current number to &2" during receipt number determination.

Other Terms

FP-REPT  , >0, 577, 577(>0), FKK_RECEIPT_0020, FKK_REPT, LFKREPTF01, 577>0, >0577, (>0 577)

Reason and Prerequisites

Reason: Number range 01 for FKK_REPT has been exhausted.

Prerequisites: Receipt management is active.

Solution

The system should not raise an error when a number range interval lacks sufficient free numbers. Instead, it should proceed to the next number range.

Implement the correction instructions for your release or import the relevant Support Package.

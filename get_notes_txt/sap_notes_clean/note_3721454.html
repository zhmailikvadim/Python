Symptom

  • More than 300 missing views /AMR/* observed in transaction DB02.
  • SAP Allocation Management functionality is not utilized in the affected environment.

Environment

  • SAP Customer Activity Repository
  • Allocation Management

Reproducing the Issue

  1. Open transaction code DB02.
  2. Observe the list of missing /AMR/* views displayed on the right side of the interface.

Resolution

For the /AMR/* related errors in DB02, if the Allocation Management CARAB scenario is not being used or if there are no plans to use it then the /AMR/* views will be in an inactive state - you can ignore these missing views.

If Allocation Management will be used, the prerequisite steps need to be performed to activate the necessary /AMR/* views as per the CARAB Installation/Upgrade Guide:

https://help.sap.com/docs/CARAB/5f6a0c50ac324b0395738c1ce0fe14c6/a09d965de8264010924bc8cb8bd52ba3.html?version=LATEST&locale=en-US

Symptom

  • You are using the SAP S/4HANA Manufacturing for planning and scheduling capability of SAP Digital Supply Chain Management, edition for SAP S/4HANA in a side-by-side deployment with an additional SAP S/4HANA or SAP ERP system (DSC Edition).
  • When changing an order in DSC Edition, i.e. using /SAPAPO/RRP3, the order will be published to source ERP system even if there is no entry for distribution definition in /SAPAPO/CP1 for affected order category and location.

Environment

  • SAP S/4HANA Manufacturing for planning and scheduling capability of SAP Digital Supply Chain Management, edition for SAP S/4HANA in a side-by-side deployment with an additional SAP S/4HANA or SAP ERP system (DSC Edition).

Reproducing the Issue

  1. Maintain distribution definition entries in transaction /SAPAPO/CP1 for some locations only, leaving at least one target location without a specific entry.
  2. In /SAPAPO/RRP3, change an existing order for a location that is not explicitly maintained in /SAPAPO/CP1.
  3. Observe that the order change is still published to ERP when the order originated in ERP and a lookup exists (entry in /SAPAPO/CIFLOOKU).
  4. Remove all entries from /SAPAPO/CP1 and repeat an order change in /SAPAPO/RRP3; observe that nothing is published to ERP (system behaves like ePP/DS and checks PP/DS customizing instead).

Cause

  • Missing or incomplete configuration of distribution definition in /SAPAPO/CP1 for DSC Edition publication.

Resolution

  1. In /SAPAPO/CP1, create at least one distribution definition entry to enable DSC Edition publication.
  2. Maintain distribution definition entries per required order category (for example, in‑house production, external procurement) and for every location where publication to ERP is desired.
  3. For new orders created in DSC Edition, ensure a corresponding location-specific entry exists in /SAPAPO/CP1; only orders for those locations will be published.
  4. For orders that originated in ERP and are changed in DSC Edition, be aware they will be published back to the original ERP if a lookup entry exists in table /SAPAPO/CIFLOOKU; if there is at least one entry in /SAPAPO/CP1 but no entry for this order category and location.

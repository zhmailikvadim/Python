Symptom

  • Error message: "Insufficient stock of product &1 in bin &2."
  • It is not possible to scrap the stock at APP Post Goods Movement (APP ID MIGO) when stock type is S with movement type 551 or 555.

Environment

SAP S/4HANA Cloud Public Edition

Reproducing the Issue

  1. Open the app: Post Goods Movement (APP ID MIGO).
  2. Enter Movement Type: 551 or 555, select Goods Issue - Other.
  3. Enter the target material: &1, and input a scrapping quantity.
  4. Enter Issuing Plant and Issuing Storage Location which is WM relevant.
  5. In the Account Assignment tab, enter Cost Center.
  6. Enter Storage Type and Storage Bin.
  7. Click the Post button.
  8. Observe the error message: "Insufficient stock of product &1 in bin &2."

Cause

The movement type "551" - Withdrawal for scrapping from unrestricted-use stock only determines unrestricted stock and movement type "555" - Withdrawal for scrapping from Blocked stock only determines Block stock with stock type B.

Resolution

It is not possible to scrap WM stock with stock type "S" Scrapping via movement type 551 or 555.

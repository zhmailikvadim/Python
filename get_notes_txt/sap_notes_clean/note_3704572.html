Other Terms

-9212, undo_log_execute_error, CopyBlob, Data_ChainSplitSpaceBackwardReadChain 

Reason and Prerequisites

The crash occurs during the rollback of a transaction that modified some large binary objects (LOBs) and remained active during the SAP MaxDB CHECK DATA SNAPSHOT operation.

Solution

The SAP MaxDB database must be restored.
To prevent this issue in the future, either disable CHECK DATA SNAPSHOT (only recommended for a short time) or upgrade SAP MaxDB to version 7.9.11.14 at least. This version disables the removal of unreferenced LOBs. Refer to the relevant SAP note 3704036 - SAP MaxDB 7.9.11 CHECK DATA SNAPSHOT for more details.

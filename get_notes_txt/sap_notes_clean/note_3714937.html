  • Other transactions (including custom coding) can also be affected.
  • The same scenario in SAP GUI for Windows works properly.

Environment

  • SAP S/4HANA
  • ABAP Platform
  • SAP GUI for HTML/Fiori

Reproducing the Issue

  1. Access transaction PC_PAYRESULT in SAP GUI for HTML.
  2. Enter sample data to display the results in ALV table.
  3. Double-click in an entry in the ALV table.
  4. Observe that the system terminates with the runtime error SYSTEM_CORE_DUMPED.

Cause

Programming error in the Kernel code.

Resolution

Upgrade the Kernel as outlined in the following SAP Note:

3715324 - ALV GridView: CORE DUMP after performing some operations like double-click or collaps/expand rows

Environment

SAP Build Process Automation

Reproducing the Issue

Attempt to deploy a project in any environment.

Cause

The required role IRPAOfficer was missing from the user.

Resolution

  • Add the IRPAOfficer role to one of the user's role collections. For detailed guidance on how to do this, please refer to  Add Roles to a Role Collection | SAP Help Portal 

OR

  • Assign the premade role collection ProcessAutomationAdmin to the user. For detailed guidance on how to do this, please refer to  Assign Users to Role Collections | SAP Help Portal. Please be aware that this role collection contains other authorizations that may not be suitable for certain users, therefore the first option above may be a more viable solution.

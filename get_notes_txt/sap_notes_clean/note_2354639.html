Symptom

SAP Business One Service Manager is not starting and/or not responding when you click the settings button in License Manager.

Reproducing the issue

Scenario:

  1. Intel Security True Key Service is running and you open the SAP Business One Service Manager window.
  2. You click the Settings button for License Manager. 

Expected behavior:

License Manager settings window opens.

Actual behavior:

﻿SAP Business One Service Manager window hangs and is not responding. 

 

Solution

  1. Open services.msc and search for Intel Security True Key Service.
  2. Stop this service and try to open Service Manager again. Intel Security True Key Service runs on the port 30000, and because of this, License Manager settings cannot be open.

 

In case if you want to use this service then apply this workaround.

Workaround:

  1. Stop the Intel Security True Key Service from the services.msc.
  2. Open SAP Business One Service Manager → Select License Manager and click Settings.
  3. Change the port number from 30000 to something else for the License Manager, click Set and then Connect.
  4. Change the port for SLD service also from server.xml file in Tomcat folder. You can find the server.xml file here:
    • for version 9.1 : "C:\Program Files (x86)\SAP\SAP Business One ServerTools\System Landscape Directory\tomcat\conf".
    • for version 9.2 : "C:\Program Files (x86)\SAP\SAP Business One ServerTools\Common\tomcat\conf".
  5. Open the Server.xml file and search for the port number 30010, change the port to the same port you use in the License Manager +10 and save the file.
  6. Go to services.msc and restart the Service Landscape Directory (SLD) service.
  7. Now you can start your Intel Security True Key Service again and work with SAP Business One Service Manager.

Other terms

SAP Business One, SAP B1, Intel Security True Key Service, McAfee etc.

Symptom

After initial sync, register the device for Digital signature. After entering the passphrase move to the next screen, where it generates a secret key.

Secret key is not presented on the screen. Client is stuck with a loading indicator going on forever.

Other Terms

  • SAP Service and Asset Manager
  • SAM
  • SSAM
  • ICMTANGOAMF10-42824
  • Digital signature

Reason and Prerequisites

Reason:

The page's OnLoaded rule retrieves the secret key faster than the related field is displayed on the page.

Prerequisites:

  •  BAS or Visual Studio Code with SAM MDK project
  • `patch` command (available on BAS and macOS by default) via terminal or other patching utility
  • Read the guidelines on how to apply the patch and read the appendix section if you receive any error

Solution

To remove the OnLoaded rule and instead use the field's Value property, retrieving the secret key after the control has loaded.

To apply the patch, download the attached 3717460.patch file and save it in a known location, preferably adjacent to your SAM.mdkproject.

If you are on Windows, see SAP Note 3533879 for how to install a .patch file in Windows. If you are on a Mac, follow the steps below:

  1. Open a terminal in the root of your SAM metadata (i.e. SAM.mdkproject/metadata or SAM.mdkproject/metadata/base.definitions)
  2. Run the terminal command `patch --remove-empty-files -p4 < path/to/3717460.patch`. 
  3. Verify in the terminal window that the following files have been modified and listed as patched, as indicated by the console output:
    • patching file 'Actions/OData/DigitalSignature/CreateTOTPDevice.action'
    • patching file 'Pages/DigitalSignature/PasscodeTOTP.page'
    • patching file 'Rules/DigitalSignature/CreateSignatureSuccess.js'
    • patching file 'Rules/DigitalSignature/GetTOTP.js'
    • patching file 'Rules/DigitalSignature/GetTOTPOnloaded.js'
    • patching file 'Rules/DigitalSignature/TOTPDefaultDeviceReadLink.js'

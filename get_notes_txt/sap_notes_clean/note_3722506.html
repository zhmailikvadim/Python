Symptom

In the MET cluster process DE_B_MET_MR_INFO_DVCHG, the system determines an incorrect meter reading order for a received move-out meter reading in step 0030 when the device changes from a normal device to an intelligent metering system (iMS) device. As a result, the system does not post the inbound meter reading.

This issue is relevant for the market roles: SUP, DSO.

Other Terms

 MET, DE_B_MET_MR_INFO_DVCHG, step 0030, /US4G/CL_METERING_UTIL, GET_IMS_DEVICE, intelligent metering system, iMS, meter reading order, move-out, inbound meter reading, device change, SUP, DSO, POD

Reason and Prerequisites

 The error occurred because the system did not set the intelligent metering system flag when the parameterization instance data on the Melo POD was missing.

Solution

Implement the corrections of this SAP Note or import the relevant support package.

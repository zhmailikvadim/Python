Symptom

You have read SAP Note 177718 - Consult. Pasos a realizar arranque en productivo but there is no reference to the estimation of variable variables of previous years before the go-live.

Environment

  • Human Capital Management (HCM)
  • SAP R/3
  • SAP ERP Core Component
  • SAP ERP
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA

Reproducing the Issue

  1. Before the go-live you have performed all the steps explained in SAP Note 177718 - Consult. Pasos a realizar arranque en productivo.
  2. You have loaded wage type /1IC but the variable variables of previous years are not correctly estimated. 

Cause

Having the total amount of variables variables from the previous year stored in wage type /1IC before the first tax estimation will not work because the processing is done at the code below:

*&---------------------------------------------------------------------*
*&      Form  collect_iga_r
*&---------------------------------------------------------------------*
FORM collect_iga_r USING  pv_pernr  TYPE pernr_d
                          pv_begda  TYPE d
                          pv_endda  TYPE d
                          ps_wpbp   TYPE pc205
                          ps_st     TYPE pc226
                          ps_aper   TYPE pc2aper
                          ps_rgdir  TYPE pc261
                 CHANGING ct_iga_rt TYPE pest0_iga_rt_tab
                          ct_rt     TYPE pesu0_rt_tab.

* Updating the RT with the percentages from table T5E37
  PERFORM apply_t5e37_percentages USING pv_pernr
                                        pv_begda
                                        pv_endda
                                        ps_wpbp
                                        ps_st
                                        ps_aper
                               CHANGING ct_rt.

* Generating EIRE wage types again
  PERFORM process_eire USING as-parm1
                             as-parm2
                             ''
                       CHANGING ct_rt.

* Moving generated wage types to the iga_rt
  PERFORM copy_rt_2_iga_rt USING ct_rt
                                 pv_begda
                                 pv_endda
                                 'R'
                                 'X'
                           CHANGING ct_iga_rt.
ENDFORM.                    "collect_iga_r

 

Even if you include wage type /1IC in table T5E37 it will not work because after reading and processing it in PERFORM apply_t5e37_percentages, the program checks if any wage type /1Ix is in table RT and removes it in PERFORM process_eire for a correct behavior.

Resolution

SAP Note 161969 - CONSULT.: Estimación anual de ingresos gravables states:

"Resultados anteriores:
En cada período de cálculo se importan los resultados de nómina correspondientes al mismo período del año ANTERIOR. Para cada CC-nómina de este resultado se comprueba que esté parametrizada en la tabla T5E37 con el campo PERGV marcado. En caso de estar informado el campo PORCE de esta tabla con un porcentaje distinto de 0, se incrementa el importe de la CC-nómina en cuestión con este porcentaje. Luego se generan las CC-nominas /1Ix aplicando la regla EIRE (o la que ha sido pasada como parámetro a la función EIGA0). Los importes obtenidos se suman a los simulados para el período que se está estimando actualmente sin ningún tipo de tratamiento. Si existe la necesidad de procesar estos importes se deberá hacer dentro del ciclo que se pasa como parámetro a la función EIGA0."

In order to correctly perform this estimation you may consider the following options (please notice this represents just some hints, you need to try them and maybe take further actions):

  • Option 1: In table T558A, include the wage type(s) that should generate the total amount recorded in /1IC for each payroll period of the previous year, taking into account the impact this may have on those payroll results.

  • Option 2: In table T558A, replace wage type with the actual wage type(s) and amounts that should have generated the total amount stored in the /1IC concept. This would work if the EIRE rule is called in your estimation scheme.

  • Option 3: Create a Z payroll function in the scheme used to run RPIGA0E0 to read the payroll concepts stored in table T558A in the same way as is done for table T5E37, so that they are not deleted when processing the EIRE rule.

  • Option 4: Create records in infotype 0015 including those wage type(s) that should carry the amount recorded in the /1IC concept for each payroll period of the previous year and ensure that these payroll concepts are configured in table T5E37 with the V_T5E37-INP15 field ticked.

Symptom

When attempting to connect via SSH to xfer-iad.calliduscloud.com, the connection fails with the error “WARNING: REMOTE HOST IDENTIFICATION HAS CHANGED!” followed by “Host key verification failed” and “Connection reset by peer”, indicating a mismatch with the stored host key in /home/CUSTOM_NAME/.ssh/known_hosts (offending entry: line <LINE NUMBER WITH ERROR>).

Resolution

  1. Confirm the change is expected. 
    Treat the message as a potential security risk (possible man-in-the-middle). Verify with the server/network team that the SSH host key for xfer-iad.calliduscloud.com was legitimately changed (e.g., server rebuild, patching, migration).
  2. Back up the current known_hosts file.
    Before making any changes, create a copy of /home/CUSTOM_NAME/.ssh/known_hosts so you can restore it if needed.
  3. Remove the offending entry.
    Edit /home/CUSTOM_NAME/.ssh/known_hosts and delete the specific entry causing the error. Alternatively, remove the entry corresponding to xfer-iad.calliduscloud.com.
  4. Reconnect to the host.
    Attempt the SSH connection again. You should be prompted to accept the new host key.
  5. Validate and accept the new host key.
    Compare the displayed fingerprint with the trusted fingerprint provided by your system administrator/security team. Accept it only if it matches.
  6. Test and confirm successful access.
    Ensure the SSH session completes normally and the warning no longer appears. If automation or scheduled transfers use this host, confirm they also work successfully.

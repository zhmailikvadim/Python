Symptom

You are creating an incoming NFS-e (Nota Fiscal de Serviço Eletrônica) for the city of São Paulo via transaction code MIRO and notice that the NFS-e are being added to table J_1BNFE_ACTIVE.

Environment

  • SAP S/4HANA
  • SAP ERP
  • SAP Document and Reporting Compliance, outbound invoicing option for Brazil (nota fiscal eletronica)
  • Localization Brazil 

Resolution

Implement SAP Note 3704498 - Inbound NFS-e: Correcting the Reversal of Incoming NFS-e in MIRO that corrects this behavior for future NFS-e.

NFS-e created before the SAP Note implementation will be necessary to remove manually the data from table "J_1BNFE_ACTIVE".

See Also

  • SAP Note 3704498 - Inbound NFS-e: Correcting the Reversal of Incoming NFS-e in MIRO

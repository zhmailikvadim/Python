Symptom

The RAR contract has Validation Result 'E' (Error) status. Upon attempting to reprocess the contract, it triggers an error message stating: 'Recognized Cost account cannot be derived from reference account XXX for POB XYZ' – Message No. FARR_CONTRACT_MAIN136. This issue prevents the successful processing of items in RAI monitor.

Environment

    • Revenue Accounting (RAR).
    • Financial Accounting (FI).
    • SAP S/4HANA Simple Financials.

Reproducing the Issue

  1. Run transaction FARR_REPR_CNTR for contract in error status.
  2. Execute the program and observe the error messages mentioned in the symptom section.

Cause

Misalignment between the attributes of the performance obligation and the reference account with the account determination rules in BRF+.

Resolution

  1. In BRF+, review and update the Recognized Cost decision table (FARR_ACCT_DETERMINE_DT_RC_CST) to align the account determination rules with the reference account setup.
  2. Reprocess the affected items in RAI monitor or reprocess contract after applying the necessary adjustments.

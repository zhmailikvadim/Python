Symptom

A product has a characteristic with reference to MARA-MATNR inherited from a 300 class.

During configuration the material number of the KMAT is assigned implicitly to the characteristic as derived by classification. This feature is built in for the reference to MARA-MATNR. Thus, the characteristic value assignment has the author "classification" and is set as class value. In nearly all cases this behaviour is correct. But if a product variant is configured and the configuration to the product variant contains a value which points to the product variants material number, this strong inference may cause a conflict in the IPC. The product variants material number assignment is typically made with author "user" from the product variant maintenance in ERP or S4.

As the configuration of product variants is typically not interactive, the conflict cannot be resolved manually. In other cases manual conflict resolving is possible, but might lead to the unintended effect, that the product variant is replaced by the KMAT.

Other Terms

KMAT, product configuration, IPC, reference characteristic, product variant, conflict, MARA-MATNR, material number

Reason and Prerequisites

The IPC assigns two values to the characteristic - one from the product variants configuration the other as built-in value setting from the IPC referring to the KMAT.

Solution

The value assignment to reference characteristics is made now as default value instead of derived from classification. Thus it can be overwritten if required from the product variant and does not cause a conflict.

Implement the newest patch for the IPC as described in SAP Note 880749 for the support packages 33 onwards.

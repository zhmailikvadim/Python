Symptom

You would like to understand whether your SAP Field Service Management account is compliant with the Superdomain migration and whether any action is required on your side.

SAP has announced the mandatory migration from legacy domains (*.coresuite.com, *.coresystems.net) to the new SAP FSM superdomains.

You are unsure if:

  • Your account is still using legacy domains

  • Your integrations are still using legacy authentication methods

  • You need to take corrective action

Further details about the migration are available in the official documentation:

  • Common Superdomain for SaaS Applications | SAP Help Portal

Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.

Environment

  • SAP Field Service Management (FSM)

Cause

Your account may still be using:

  1. Legacy Domains

    • *.coresuite.com

    • *.coresystems.net

  2. Legacy Authentication Methods

    • erpSecurityToken

    • accountPassword

    • userAccountPassword

If legacy domains or authentication methods are still being used, your account is affected by the Superdomain changes and requires migration.

Resolution

You can easily verify whether action is required.

Step 1 – Log in to FSM Admin

  • Log in as SUPERUSER

  • Open menu: "Legacy Domain & Auth Usage Stats"

This page shows usage from the last 15 days.

Step 2 – Check Both Tables

A.) Legacy Domain Usage Stats

  • Shows usage of old domains (*.coresuite.com, *.coresystems.net)

  • Displays up to 10 entries (URI + Source IP + latest timestamp)

If the table is empty

  • You are compliant
  • No action required

If the table contains entries

  • Old domains are still in use
  • You must migrate to the new FSM domains

B.) Legacy Authentication Usage Stats

  • Shows usage of old authentication methods:

    • erpSecurityToken

    • accountPassword

    • userAccountPassword

If the table is empty

  • You are compliant
  • No action required

If the table contains entries

  • Legacy authentication is still used
  • You must migrate to OAuth2

Step 3 - Final Status Check 

ResultAction Required
Both tables empty✅ No
Domain table has entries❗ Yes – Migrate domains
Auth table has entries❗ Yes – Migrate to OAuth2
Both tables have entries❗ Yes – Migrate domains & OAuth2

Important

  • The expected state is that both tables are empty.
  • If they are empty → Your account is compliant.
  • If you see any records → Action is required on your side.

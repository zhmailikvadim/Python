Symptom

An inbound payment file containing XML Tags in an incorrect sequence is processed in APM. However during outbound payment file generation, the element sequence is not corrected according to the schema definition. The generated XML file retains the same incorrect tag structure as the inbound file.
As a result, the output file does not comply with the bank-required schema format and is rejected during bank validation.

Other Terms

 Output payment XML, incorrect tag sequence, schema compliance, bank file rejection

Reason and Prerequisites

The ordering of XML tags is controlled by a sorting flag FLG_SORT_REQUIRED that determines whether re-sorting is required during XML processing. If this flag is not set, the sorting logic is not executed. As a result, the XML output retains the existing imported order instead of being arranged according to the schema-define sequence.

Solution

Please apply SAP Note or assigned Support Package.

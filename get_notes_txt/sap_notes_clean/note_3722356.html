Symptom

Column values appear misaligned with column headings when executing a Report Painter / Report Writer report via transaction GR55.

Observed behavior:

  • Report output becomes misaligned when the following display options are enabled:

    • Freeze lead column

    • Freeze page header

  • When scrolling horizontally, the column headings remain fixed, but the data rows appear shifted relative to the headings.

  • The issue occurs especially when the report contains many columns and the page width has been increased in the report layout settings.

  • As a result, the displayed figures appear under the wrong column headers, making the report difficult to read.

Environment

  • Controlling (CO)
  • SAP R/3
  • SAP R/3 Enterprise 
  • SAP ERP Central Component
  • SAP ERP
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA Finance
  • SAP S/4HANA

Cause

  • The scroll bar in Report Painter / Report Writer output is controlled only by the report page dimensions defined in the report layout.
  • The scroll behavior is not technically linked to individual report columns or rows. When the page width is extended beyond the visible screen width, the system cannot guarantee alignment between frozen headers and scrolling columns.

Resolution

  • Use the standard layout dimensions instead of increasing the page width in the Page Control section of the report layout settings.

  • Keep the page width aligned with the standard visible screen size.

  • Configure the report to include only the number of columns that can comfortably fit within that standard width.

Symptom

 Configuration scope is getting overwritten by custom when we upload an excel file. Without upload of excel configuration is getting defaulted based on process template

Environment

  • SAP Master Data Governance
  • SAP S/4HANA

Reproducing the Issue

  1. Create a custom Scope in MDCIMG.
  2. Create a Process Template.
  3. In the "Config ID" field of the EDIT step of the Process Template, assign the custom Scope.
  4. Upload a file when creating the process 
  5. Run a test with the new Process Template, changing different fields from those defined in the Scope.

Cause

Standard Behavior

Resolution

  1. When user creates Mass Process without file, user first gets scope selection screen with default configuration. Then on the next screen, user specifies product selection criteria and continues Mass Process.
  2. When user creates Mass Process from the file, file defines the scope and selection criteria. Tables and fields included in the file is the scope and this data is also what is going to be changes during Mass Process. This explains the difference. Please read follow documentation:File Format for File Upload

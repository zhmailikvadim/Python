Symptom

You are creating inbound deliveries based on expected goods receipts for the production order and notice queues(SPE/INB_DELIVERY_SAVEREPLICA) are hanging with error message VL860 - "Different vendors specified for an item".

Reason and Prerequisites

You have implemented the notes 2280370 and 3002158 which changed the system behaviour: The (dummy) vendor for the inbound delivery was earlier determined from the plant data of the (receiving) planning plant (AFPO-PWERK) and now should be determined based on plant data of the executing plant (AFPO-DWERK).  This is the correct standard behaviour.

However you might have old expected goods receipts in EWM which were created with old logic and thus still carried the old vendor information. When these old EGRs were converted into inbound deliveries, the corresponding queues would be hanging with error message VL860.

Solution

Please implement the following correction instructions. This should fix the issue for the old existing EGRs.

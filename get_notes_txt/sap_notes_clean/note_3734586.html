Symptom

When executing IP document accrual migration, the system is expected to process and delete all Bucket tracking link data. However, the system currently deletes only the first item, leaving the remaining entries in the table. This leads to a runtime error during the Undo All posting action.

Additionally, the posting date is defaulted from configuration, whereas it should be derived from the actual posting date of the accrual.

This note addresses the issue by correcting the behavior to ensure that all tracking data is properly processed and deleted, and that the posting date is determined based on the accrual posting date.

Other Terms

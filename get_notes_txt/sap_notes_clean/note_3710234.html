Symptom

The process instance ID plays a key role in uniquely identifying transactions in SAP Document and Reporting Compliance, Cloud Edition. This KBA explains how to identify the process instance ID for outward eDocuments from the business system for troubleshooting and monitoring purposes.

Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.

Environment

  • SAP ERP
  • SAP S/4HANA
  • SAP S/4HANA Cloud Private Edition
  • SAP Document and Reporting Compliance, Cloud Edition

Resolution

For outward eDocuments, the process instance ID in SAP Document and Reporting Compliance, Cloud Edition is the same as the eDocument GUID from the business system. The identification can be completed in two simple steps:

    1. Obtain the eDocument GUID from the business system:
      1. Launch transaction edoc_cockpit
      2. Locate and select the eDocument that is causing the issue or needs to be monitored.
      3. Choose Goto -> History
      4. Select and copy the eDocument GUID
    2. Use this GUID to search for and locate the corresponding transaction in the DRC Cloud Edition application:
      1. Next, launch the corresponding DRC Cloud Edition Application and navigate to 'Resolve Errors' tile:
      2. Use the eDocument GUID copied in step 4 to apply the filter and locate the relevant entry:

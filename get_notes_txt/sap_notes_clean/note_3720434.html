Symptom

We are facing an issue where a linked absence is displayed as “Marked as Original (Original Absence)” when accessed via:

Administer Time UI > Absence > 16–19 Dec 2025 > Common Illness > Related Records

However, when accessing the same leave record through: Manage Data > Employee Time > the field "Original Absence" is maintained as “No.”

Expected Behavior

The system should not display the leave record as “Marked as Original” when the Original Absence field is actually set to “No” in the Employee Time record.

Environment

  • SAP SuccessFactors Employee Central - Time Management

Reproducing the Issue

  1. Navigate to Administer Time UI for the affected user.

  2. Open the leave record: Sick Leave (Dec 16, 2025 – Dec 19, 2025).

  3. Click the link icon under Related Records.

  4. Observe that the leave request (Dec 16, 2025 – Dec 19, 2025) is shown as “Marked as Original” under Original Absence.

  5. Verify in Manage Data > Employee Time that the same record is flagged as Original Absence = No.

Cause

  • When editing an absence and linking it with other absences, the system may return the oldest absence in the group as null under certain conditions. This can result in inconsistent UI behavior.

Conditions Where This Occurs

This issue may occur in the following scenarios:

  • When the oldest absence in the group was created retroactively, or

  • When the oldest absence was linked to the group after newer absences had already been linked.

This behavior does not impact all users and is limited to specific scenarios.

Resolution

Product Engineering is actively investigating and implementing a permanent solution.

Planned Fix

The issue is scheduled to be resolved in the 1H 2026 release cycle:

  • Preview: April 13, 2026

  • Production: May 15–16, 2026

Temporary Workaround

To maintain consistent behavior until the permanent fix is delivered, users may:

  1. Mark the oldest absence in the group as the Original Absence from the popover.

  2. Save the changes.

  3. After saving, update the Original Absence again to the desired record if required.

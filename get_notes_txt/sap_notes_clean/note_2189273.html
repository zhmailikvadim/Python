Symptom

You have extracted financial and management accounting data via the Extraction Run for Financial Data.
When reviewing the extracted results you notice that the Journal report and the Trial Balance report display different total results for the same period.

Environment

SAP Business ByDesign

Reproducing the Issue

  1. In the General Ledger workcenter navigate to the Periodic Tasks: Extraction Run for Financial Data view.
  2. Press New- Extraction Run for Financial Data.
  3. Enter all mandatory data.
  4. In the Reporting Package field, select a reporting package that contains the Journal as well as the Trial Balance report.
  5. After the run has finished without errors open the run with the View button and press Download and save the Zip file locally.
  6. You open the reports in Microsoft Excel where you filter for a specific G/L account.
  7. You notice that the totals do not match.

Cause

The results do not match as the journal report is using default selection parameters: By default the journal report filters only for 010- Operational Postings.

The Trial Balance report has no default selection in regards to the closing steps and therefore displays any kind of postings, such as 015-Late Operational Postings or 020-Closing Entries.

To verify the selection parameters of the journal report

  1. open the log of the extraction run again from the Periodic Tasks: Extraction Run for Financial Data view and press View.
  2. In the Processed Successfully tab, highlight the Journal report.
  3. In the Report Selection and Result sub-tab click the Report Selection Parameter link.
  4. A small text file opens displaying the default setting with the 010-Operational Postings closing step setting.

Resolution

If you need to include any kind of postings for the Journal extraction you can create a new selection variant with either no closing step selection or by selecting further closing steps:

To create a new Selection Variant*:

Pre-Condition: Use a user with full access to design reports in the Business Analytics workcenter. You have to modify the changes in the Business Analytics workcenter. Otherwise the changes will not be visible for other users.

  1. In the Business Analytics workcenter navigate to the Design Reports view.
  2. Select the Journal report, ID: FINGLAU03_Q0001.
  3. Press Edit with Web-browser.
  4. Select for example the Journal for GDPdU view.
  5. In the Edit Selection tab, remove the Closing step 010. Alternatively, depending which postings need to be extracted, you could also add closing steps here, such as 015-Late Operational Postings or 020-Closing Entries.
  6. Press Selection - Save as and enter an appropriate name, e.g. Journal_No_Closing_Steps and Press OK.

*Tip: Modifying the default settings of report selection parameters is not only useful for data extraction. This way you can adjust the selection parameters of any report to your own needs.

You can now select the new selection variant in a customized reporting package**:
**It is not possible to modify the standard reporting packages.

To create a customized reporting package:

  1. In the General Ledger workcenter navigate to the Periodic Tasks: Extraction Run for Financial Data view.
  2. Press Edit Reporting Packages.
  3. If there is no customized reporting package for the GDPdU extraction saved here create a new one: Select the standard GDPdU reporting package, i.e. SAP_DE_GDPDU and press Copy.
  4. In the popup window, enter a package name starting with Z. Then press OK.
  5. Maintain an appropriate description.
  6. In the Details table underneath the first report is the Journal report, ID FINGLAU03_Q0001.
  7. In the Report View ID column, select the FINGLAU03_Q0001_V02- Journal for GDPdU view.
  8. In the Selection Variant ID column, you can now select your new selection variant  Journal_No_Closing_Steps.
  9. Press Save and Close.

You can now select this new customized reporting package for the data extraction run:

  1. In the General Ledger workcenter navigate to the Periodic Tasks: Extraction Run for Financial Data view.
  2. Press New- Extraction Run for Financial Data.
  3. In the Reporting Package field, select your new reporting package.
  4. Maintain all mandatory data.
  5. Then press Start Now to immediately execute the run.

After the run as finished successfully you can check the report selection parameters of the Journal report again:
This time the Report Selection Parameters will display all the selection parameters that you have defined for this report.

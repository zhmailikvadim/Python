Symptom

Message no. FS670 ‘Document & & & already exists in the check file' occurs after executing program RFFOUS_C. "Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental." 

Environment

  • Human Capital Management (HCM)
  • SAP ERP
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP SuccessFactors Employee Central Payroll
  • SAP S/4 HANA

Reproducing the Issue

  1. Run program RFFOUS_C via t-code SE38.
  2. Input program run date and identification feature. For example,
  3. Click 'Execute' button.
  4. Message no. FS670 'Document & & & already exists in the check file' occurs.
    For example, 'Document 00327077 00011 01 already exists in the check file' occurs.

    The format of the data record is as follows:
    • 00327077: Pernr number
    • 00011: Sequence number
    • 01: Assignment of transfers

Cause

This issue is caused by data inconsistency due to unknown reasons. For example, for pernr number 327077 with sequence number 00011, there is an inconsistency entry with void reason ‘00 (Zero Net Check)' compared with '13 (Lost check)' in table PAYR. Basically, for same pernr, each sequence number with same assignment of transfers only has one entry in table PAYR.

Resolution

Create a custom program to delete the inconsistency entry since this is not caused by SAP in standard.

Keywords

FS670, RFFOUS_C, Check, PAYR, Void, Reguh, RFFORI01

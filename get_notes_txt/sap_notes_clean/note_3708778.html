Symptom

A runtime error CALL_FUNCTION_CONFLICT_LENG occurs during execution of functionality that triggers logic from include /SPDGS/LPDX1F04.
The application terminates with exception CX_SY_DYN_CALL_ILLEGAL_TYPE and program flow is interrupted.
The dump indicates that parameter lengths or types do not match the expected signature during a function module call.

Other Terms

IS-MFM, CORE, PDA, CALL_FUNCTION_CONFLICT_LENG, CX_SY_DYN_CALL_ILLEGAL_TYPE, SAP Meat and Fish Management by msg.

Reason and Prerequisites

A field length or type conflict occurs during the dynamic call of a function module inside include /SPDGS/LPDX1F04, which leads to the program termination. The error is caused by incorrect or incomplete parameter typing in a function module call.

Solution

History:
SAP Note created on 28.01.2026.

Version 2: Note for 6.20

Description of Correction:
The parameter mapping in the function module call has been corrected.
All parameters are now passed with the correct ABAP types.
Structures correspond to the required length definitions of the called function module.
All sequences or internal structures required for processing are correctly initialized before the call.
The adjustment to the parameter interface now prevents runtime conflicts and ensures an uninterrupted process flow.

Functional Dependencies:
None.

Manual Changes:
No manual changes are required for end users.

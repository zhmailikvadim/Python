Symptom

In the old GUI in ME21N/ME51N the users can change the size of the tables and the view, to have the help panel or the document search panel.

The change is saved when the user leaves the transaction.

In WEBGUI applications this change does not get saved.

Environment

  • SAP S/4HANA 1809
  • SAP S/4HANA 1909
  • SAP S/4HANA 2020
  • SAP S/4HANA 2021
  • SAP S/4HANA 2022
  • SAP S/4HANA 2023

Reproducing the Issue

  1. Open the WEBGUI application Create Purchase order - Advanced 
  2. Change the view
  3. Leave the application
  4. Return to the application
  5. Notice that the view remains unchanged

Cause

In old transactions when users leave the application a save process changes the ESDUS table, which contains the user's default settings on how the transaction should look like upon opening. 

In the webgui there is a buffered state which only gets saved when an explicit save process is initiated.

Resolution

To make change in the view, the user must go to the Personal Setting and press save there.

This will save the change to the ESDUS table, hence the next time the user goes to the application their defaults will be shown.

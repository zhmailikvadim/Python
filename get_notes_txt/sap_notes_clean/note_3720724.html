Symptom

  • A share was created in Databricks and a corresponding Data Product was published to SAP BDC via SAP BDC Connect.
  • After Delta Share has been deleted manually from Databricks UI, the Data Product remains marked as "Active" in the SAP BDC Catalog
  • Unpublishing the Data Product does not work because the underlying share is no longer found in Databricks

  • How to clean up this orphaned entry manually from the BDC catalog? 

Environment

  • SAP Business Data Cloud (BDC) Connect
  • Databricks Premium Edition

Reproducing the Issue

  1. Create a Delta Share in Enterprise Databricks.
  2. Publish a Data Product to SAP BDC via SAP BDC Connect.
  3. Delete the Delta Share manually from Databricks UI.
  4. Observe that the Data Product remains "Active" in the SAP BDC Catalog.
  5. Attempt manual deletion via the BDC Catalog UI; note that no option exists for deletion.
  6. Attempt removal via the BDC Connect SDK: observe that the removal fails because the underlying share is no longer found in Databricks.

Cause

The underlying Delta Share associated with the Data Product was deleted manually, resulting in an orphaned entry in the SAP BDC Catalog.

Resolution

  1. Recreate a Delta Share with the same name in Databricks Premium Edition.
  2. Reconnect to SAP BDC via Delta Sharing.
  3. Allow SAP BDC to re-extract the metadata.
  4. Delete the Data Product from the SAP BDC Catalog using the standard deletion method.
  5. Delete the recreated Delta Share from Databricks.

Symptom

  • Post OS patching and server restart, all services were observed to be up and running in the SSO system, but users were unable to log in via Single Sign-On (SSO)
  • Secure Login Client displayed "Internal Server Error."
  • AS JAVA NetWeaver Application log:

**** SECURE LOGIN PROFILE LOCK ****
Profile <profile id> locked
REASON: no certificate found for alias <certificate>
** #* SECURE LOGIN PROFILE LOCK ****

    Environment

    • SAP NetWeaver
    • SAP NetWeaver Application Server JAVA
    • Secure Login Server 
    • Secure Login Client

    Reproducing the Issue

    1. Perform maintenance patching activity and restart servers.
    2. Attempt to enroll for a certificate using Secure Login Server profile in Secure Login Client
    3. Observe that all services are running in the SSO system but users are unable to enroll for a certificate due to an "Internal Server Error."
    4. The corresponding Secure Login Server profile was locked.

    Cause

    After the system update, Secure Login Server (SLS) was restarted. During the startup phase, not all dependent resources were immediately available. The required certificates (managed by AS Java) were temporarily unavailable at the time the profile was validated.

    Since SLS performs security dependency checks during startup/runtime, the temporary unavailability of Secure Login Server certificates (specially the User CA) led to the automatic locking of the affected Secure Login Server profile as a safeguard mechanism.

    Resolution

    Once the Secure Login Server certificates became fully available, editing the locked profile, enabling the User CA and unlocking the profile solves the issue.

    See Also

    SAP Note 3245334 - Secure Login Server 3.0 - Monitoring of Authentication Profile Locking

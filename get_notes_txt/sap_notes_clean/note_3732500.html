Symptom

XDL import file was processing successfully but the respective import pipeline is not triggering in IM admin.

Environment

  • SAP Sales Performance Management
    • Incentive Management on Hyperscalers
    • Incentive Management on Legacy HANA 

Reproducing the Issue

  1. Upload an XDL import file through the XDL UI. (Eg: TXST)
  2. The XDL file completes successful.
  3. Observe in IM admin that no import pipeline run starts for the uploaded file.

Cause

Improper file type settings in XDL UI: the Import Type for relevant transactional file types is set to None.

Resolution

  1. Open the XDL UI and navigate to File Type configuration for the affected transactional file types (Eg: TXST).
  2. Check the Import Type setting. If it is set to None, change it to Validate or Validate and Transfer, according to the processing requirement.



  3. Save the changes to the file type configuration.
  4. Upload a new XDL file and confirm that the corresponding import pipeline run starts automatically in IM admin.
  5. Monitor subsequent file arrivals to ensure automatic validate-and-transfer continues without manual intervention.

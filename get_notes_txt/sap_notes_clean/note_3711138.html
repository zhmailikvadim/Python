Symptom

The PATCH operation is supported by the called API backend, but Integration Suite API Management does not automatically list PATCH when you generate an API from this OData service. 

 

Environment

  • SAP Integration Suite
  • API Management

Reproducing the Issue

The Odata endpoint service list does not contain the PATCH operation in Resources tab.

Cause

The “Resources” tab is primarily documentation for the Try-out console; it does not control which HTTP verbs are actually forwarded by the proxy. You can still call PATCH through the proxy, or you can add PATCH as a documented operation manually (or import the API from the API Hub OpenAPI spec so PATCH appears in the UI).

Why you see the discrepancy

  • SAP API Business Hub exposes a curated OpenAPI spec that includes PATCH for partial updates.
  • When you create the API in Integration Suite from the OData metadata, the wizard generates standard CRUD entries (GET/POST/PUT/DELETE) and omits PATCH for OData V2 services, even if the backend supports partial updates.
  • This omission affects the Try-out view only. The proxy can forward PATCH unless you explicitly restrict methods with policies.

Resolution

Options:

Make PATCH visible in APIM “Resources” (for Try-out and documentation)

  1. In Integration Suite API Management, open your API → Configure → Resources.
  2. Add an operation for the relevant resource path and select PATCH as the method; save and publish.
  3. If you use CORS, update your CORS policy to include PATCH in Access-Control-Allow-Methods and allow headers such as x-csrf-token, If-Match, Authorization.
  4. Test with Try-out and Postman via the proxy.

Recreate/import the API from API Business Hub spec

  1. In Discover, import the API OpenAPI from SAP API Business Hub.
  2. Point it to your target endpoint (like S/4HANA system) and publish. The imported spec will include PATCH in the Resources automatically.

Symptom

  • Manual process for removing entries from IUUC_RS_ORDER and IUUC_RS_STATUS

Environment

SAP Landscape Transformation Replication Server

Resolution

Do NOT follow the steps in this KBA UNLESS you have been directed to.

If when stopping load/replication for a table it gets stuck, see steps in KBA about 2661071 - How to stop/start the replication for a table, using 'Reset all Objects and Settings for a Table' function.

The steps below can be applicable for when the table is disappearing from LTRC or when stuck in scheduled status. If it is stuck in scheduled status, then other actions stated in this Guided Answer tree should have been done first.

Different steps to below are required if an error (seen in LTRC -> Application Logs) is responsible for the issue. Go back to the beginning of this guided answer in such a case.

Note: Only delete the entries corresponding to the affected table(s). For example if in MTID 001 the Table SFLIGHT is either disappearing or is stuck in 'Scheduled' in MTID 001, delete the entries where MTID = 001 and Table = SFLIGHT. DO NOT DELETE ALL ENTRIES!

For example, here is table stuck in "Scheduled" Status with 021 as its MTID (configuration ID).

  1. Remove the table from LTRC if it still exists (see KBA 2661071 - How to stop/start the replication for a table).
  2. On SLT server, delete all related records of MTID and its affected table(s) from table IUUC_RS_ORDER in SE16.
  4. On SLT server, delete all related records of MTID and its affected table(s) from table IUUC_RS_STATUS.
  5. On SLT server, find the CONFIG_GUID of MTID from table IUUC_REPL_MAP. DO NOT DELETE FROM IUUC_REPL_MAP. 
  6. Removing the entry from IUUC_RS_ORDER will result in the entry automatically being removed from the IUUC_REPL_ORDER table. If there are obsolete entries in this table (from configurations that have been deleted already), then these can be removed as mentioned in 2693853 - Entries in IUUC_REPL_ORDER for deleted configuration(s).
  7. Replicate the table again (see KBA 2661071 - How to stop/start the replication for a table).

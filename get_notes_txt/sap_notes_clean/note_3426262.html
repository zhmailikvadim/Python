Symptom

User with read access to Visit, should not be allowed to edit and submit Survey for the Visit.

Environment

SAP Cloud for Customer 

Reproducing the Issue

Prerequisite: User has only read access to visit

  1. Go to Visits work center.
  2. Go to Visits work center view.
  3. Search and open any Visit.
  4. Go to Survey facet.
  5. Select and open and Survey.
  6. Answer the questions in the Survey.
  7. Click on Submit.

Expectation: User does not have write access for Visit, so user should not be able to edit and submit the Survey for the visit.

Cause

Visit's access restriction does not control the edit action for Survey questions.

Responding to a Survey is not controlled by any access restriction, hence visit's access restriction does not impact edit/submit action of survey in visit.

 

Resolution

Users, who have read access restriction maintained for visits is allowed to edit, and submit the Survey in a Visit.

This is the current and standard behaviour.

Keywords

Survey read access in Visit, Access to Survey, Surveys in Visit editable.

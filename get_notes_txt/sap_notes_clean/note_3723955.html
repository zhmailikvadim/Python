Symptom

In the process "Preparation and Waiting for Sending of Meter Reading (MOS MaLo View)" (DE_B_MET_PREPARE_SEND_MR_MOSMALO), the original design purpose​ of this process was to monitor​ the sending status of meter reading documents, not to actively trigger the sending.

To improve the higher automation in daily operations, SAP has added a new feature​ to this process in this update: In step 0330, the system can now actively check for and send​ any existing meter reading documents that have not yet been sent.

Other Terms

MSCONS 13017, /US4G/CL_PRST_SEND_EXIST_MR, /US4G/FM_SEND_MR, meter reading documents, MaLo, step 0330, /US4G/3_0_MOS_NOTE_SP08_3723955, DE_B_MET_PREPARE_SEND_MR_MOSMALO, /US4G/MRVOBJREF, /US4G/MREDIRE

Reason and Prerequisites

New features.

Solution

Implement the corrections of this SAP Note or import the relevant support package to enable this enhancement.

Please activate the BC-set /US4G/3_0_MOS_NOTE_SP08_3723955in transaction SCPR20.

After implementation, the system will have the capability to proactively send pending meter reading documents in step 0330.  You can verify the new functionality by checking that the relevant meter reading documents are sent successfully after step 0330 is run.

The proactive send in step 0330 is triggered the first time the process detects that the MR (Meter Reading Document) has not been sent. The check logic for determining the "not sent" status remains consistent with the previous monitoring logic:

  • If no specific receiver is configured, the system checks if table /US4G/MRVOBJREF​ has been updated.
  • If a specific receiver is configured, the system checks if table /US4G/MREDIREF​ has been updated.

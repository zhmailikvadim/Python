Symptom

You're using MDG_KM_MAINTAIN to create Key mapping for Customer with numbers from external system and the customer number of the external system is longer than 10 digits. When you add such value for Business Object Type Customer with Object ID Type "ERP Customer Number", you will get error: ID value entered in Object ID exceeds the maximum length of 10.

Environment

  • Master Data Governance on S/4 HANA on-premise 1511
  • Master Data Governance on S/4 HANA on-premise 1610
  • Master Data Governance on S/4 HANA on-premise 1709
  • Master Data Governance on S/4 HANA on-premise 1809
  • Master Data Governance on S/4 HANA on-premise 1909
  • Master Data Governance on S/4 HANA on-premise 2020
  • Master Data Governance on S/4 HANA on-premise 2021
  • Master Data Governance on S/4 HANA on-premise 2022
  • Master Data Governance on S/4 HANA on-premise 2023

Cause

The error is expected because the object identifier type code “ERP Customer ID(ERP)” corresponds to the attribute KUNNR in table KNA1. This attribute is defined as a character field with length of 10 characters.

Resolution

If you're creating a mapping for a customer from external system with ID longer than 10, you should use Business Object Type Business Partner and the Object ID Type Business partner external ID and the mapping should involve the Business Partner number linked with the ERP customer in SAP system.

Symptom

You have configured HCM Process and Forms. You are using HRPAO_CREATE_ORGUNIT process to create an Organizational Unit. When controlling area is inherited from master cost center and you try changing the personnel area in the form, an error occurs saying "Controlling area XXXX already used for organizational unit YYYYYYYY".

Other Terms

HRPAO_CREATE_ORGUNIT, P&F, CL_HRPAO_OM_GENERIC_SERVICES, CLEAR_KOKRS, Inherited controlling area, Inheritance, Controlling area

Reason and Prerequisites

The issue was due to a coding error in method DO_OPERATIONS of class CL_HRPAO_OM_GENERIC_SERVICES.

Pre-requisite: Please make sure that the changes mentioned in note 2350761 are present before implementing this note.

Solution

Apply the corresponding Support Package to solve the error. Alternatively apply the attached correction instructions of this SAP Note.

Also, kindly perform the manual activity as shown in the note attachment after implementing the note.

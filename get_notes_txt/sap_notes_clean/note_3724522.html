Symptom

The functionality of the "Logs for Order" button in the T-Code CO24 always displays "No log data exists" regardless of the selected order.

Environment

  • SAP ERP Production Planning (PP)
  • SAP ERP All versions
  • SAP ERP Central Component All versions
  • SAP R/3 All versions
  • SAP R/3 Enterprise All versions
  • SAP S/4HANA All versions
  • SAP S/4HANA Cloud Private Edition All versions
  • SAP enhancement package for SAP ERP All versions
  • SAP enhancement package for SAP ERP, version for SAP HANA All versions

Reproducing the Issue

  1. Open T-Code: CO24.
  2. Execute (just leave everything as it is on the selection screen).
  3. Select the entry and push the button "Logs for Order".

Cause

  • The logs for the production/process orders are only available when missing parts are identified during mass processing in T-code: COHV/COHVPI, such as ATP check mass processing or ATP during mass release.
  • Logs are recorded only if the "Save Log" option is enabled in the mass processing settings.
  • Logs are not generated for ATP checks performed outside of T-code: COHV/COHVPI.

Resolution

  1. Verify whether missing parts are being identified during mass processing in T-code: COHV/COHVPI.
  2. Ensure that the "Save Log" option is enabled in the mass processing settings.
  3. Confirm that logs are being recorded during the ATP check within T-code: COHV/COHVPI.

See Also

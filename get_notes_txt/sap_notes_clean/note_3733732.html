Symptom

  • After buyer runs a Work Order Upload with headers Approval Required=False and Supplier Confirmation Required=True, the supplier does not see accept/decline buttons on the work order.
  • Work orders remain in the created state with no supplier acceptance action available.
    • No explicit error message is displayed.

Cause

  • The header combination Approval Required=False and Supplier Confirmation Required=True is not allowed because the settings contradict each other
    • Setting Approval Required to False puts the work order in created status bypassing any configured approval workflow
    • Setting Supplier Confirmation to true puts the Work Order in Created Status sending it to the Supplier to Accept as the final step in the workflow

Resolution

  1. Do not use the header combination Approval Required=False with Supplier Confirmation Required=True in Work Order Uploads.
  2. Use a valid, non-contradictory combination. Commonly, set both Approval Required and Supplier Confirmation Required to True so the work order routes to the supplier for acceptance.
  3. If work orders have already been uploaded with the invalid combination and are stuck in created, reject those work orders and resubmit them using a valid header combination.

Symptom

In the Manage Payments Fiori application, it is currently not possible to view external transaction chain charges (such as intermediary bank fees) directly within the user interface. Although these charges are present in the ISO 20022 XML payload (within ChrgsInf or Chrgs blocks), they are not exposed, requiring manual inspection of the raw XML files to identify fees applied by various agents.

Other Terms

Fiori Manage Payments,Transaction Chain Charges, ChrgsInf.

Reason and Prerequisites

Displaying transaction chain charges was required for easier identification of charges applied to a payment transaction.

Solution

The solution introduces a new UI component and backend logic to asynchronously load and display transaction chain charges.

Functional Enhancements:

  • New Button: A "Transaction chain charges" button is now visible on the Manage Payments item detail screen.
  • Dynamic Display: Clicking the button opens a popover containing a table that accurately lists the Sequence, Agent, Amount, and Currency for all charges found in the XML.
  • Data Handling: The UI handles 1:N cardinality. If no charges exist, the UI provides a graceful empty state notification.

Symptom

You have set "Automatic GR" in configuration activity 102001 - Define Control Key. However, when doing confirmation in Confirm Production Operation (F3069) app or Confirm Production Order Operation (CO11N) app, automatic goods receipt is not triggered immediately, and no material document is created.

"Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental." 

Environment

SAP S/4HANA Cloud Public Edition

Cause

The Auto GR selection is set to "Later in Background Job" in configuration activity 103921 Define Time for Confirmation Processes. With this setting, automatic goods receipt is not posted immediately during confirmation. Instead, the corresponding goods movements are stored as predefined confirmation processes.

Resolution

  • If Auto GR needs to posted during the confirmation, set the Auto GR selection for the process control key with Type 10 (production order) to "Immediately Online" in configuration activity 103921 - Define Time for Confirmation Processes.
  • If it is set as "Later in Background Job", Auto GR should be processed using one of the following options:
    • Manually via the Predefined Confirmation Processes (CO1P) app.
    • Automatically by scheduling a background job via the Schedule Posting of Predefined Confirmation Processes (F5255) app.

Symptom

In the Web Intelligence report, we noticed that the sorting order of the same object differs depending on the universe source based on OLAP Connection.

  • When using the UNV universe, the object values are sorted alphabetically.
  • When using the UNX universe, the values appear in a random order.

The issue lies with the object having text. 

Environment

  • SAP BusinessObjects Business Intelligence Platform 4.x
  • OLAP Connection
  • Universe (.unv or .unx)
  • Web Intelligence (Webi)

Reproducing the Issue

  • Open the Universe Design Tool and create the UNV universe based on the OLAP connection.
  • Open the Information Design Tool and create the UNX universe based on the same OLAP connection.
  • Open BI Launchpad.
  • Create a Web Intelligence report with two queries: Query 1 using the UNX universe and Query 2 using the UNV universe.
  • Drag the same text dimension from both queries into the report.
  • Observe that the object from the UNX universe is sorted randomly, whereas the object from the UNV universe is sorted alphabetically.

Cause

The UNX universe uses a BICS connection to connect to the BW system, and BICS does not apply any specific sorting, sometimes it  implicitly sort the object based on BW keys.

In contrast, the UNV universe uses a BAPI connection to connect to BW, and by default, objects are sorted in ascending order.

Resolution

This is by-design behavior.

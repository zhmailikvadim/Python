Symptom

  • An issue with email validation was reported in the SAP S/4HANA system. Specifically, invalid email formats such as "a@b" were being allowed in the business partner communication section.
  • The parameter SX_SMTP_ADDRESS_CHECK was configured by the customer first to "NO" and later to "OLD," but invalid email formats continued to be accepted by the system.

Environment

  • SAP ERP Central Component
  • SAP ERP
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA
  • SAP S/4HANA Cloud Private Edition

Reproducing the Issue

  • Go to transaction BP.
  • In "Find," select "Persons" and search by number.
  • Add business partner and click "Start."
  • Click twice on the line, navigate to the "Communication" section, add the email "a@b," and attempt to save.

Cause

Configuration missing

Resolution

Phase 1: Analysis

  • SAP Note 1789073 is to be reviewed so that the complete behavior of the involved parameters can be understood.
  • The current validation process is to be tested using examples such as "a@b" to confirm whether the issue persists.
  • Specific business requirements for email format validation are to be documented to ensure alignment with organizational needs.


Phase 2: Configuration

  • Backup documentation of the current SXPARAMS settings is to be created so that the existing configuration is preserved.
  • The parameter SX_SMTP_ADDRESS_CHECK is to be changed from "NO" to "OLD" so that stricter validation is enabled.
  • It is to be noted that this parameter change takes effect immediately and does not require a system restart.


Phase 3: Testing

  • Tests for business partner creation using invalid email formats such as "a@b" and "user@domain" are to be conducted.
  • The system’s handling of valid email formats, including examples such as user@domain.com and firstname.lastname@company.org, is to be verified.
  • All business partner maintenance scenarios are to be thoroughly validated.

The logic provided is used to ensure that email validation is performed by verifying that no special characters or spaces are included. Any email format that adheres to these criteria, such as a@b or a.b@c, is processed successfully. If the input does not meet these conditions, an error message is displayed.

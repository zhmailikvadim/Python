Environment

  • SAP Adaptive Server Enterprise (ASE) 16.0 with Always-on Option (HADR)
  • SAP Adaptive Server Enterprise (ASE) 16.0 with High Availability Disaster Recovery (HADR) for Business Suite

Reproducing the Issue

  • Create multiple logins with TABULATION character in the fullname column :

        declare @FN char(60)
        select @FN = "TEST" + char(9) + "LOGIN"
        create login TEST_LOGIN1 with password MY_PASSWORD fullname @FN
        create login TEST_LOGIN2 with password MY_PASSWORD fullname @FN
        -- fullname with multiple TABULATION
        select @FN = "TEST" + char(9) + "LOGIN" + char(9) + "WITH" + char(9) + "MULTIPLE" + char(9) + "TABULATION"
        create login TEST_LOGIN3 with password MY_PASSWORD fullname @FN
        create login TEST_LOGIN4 with password MY_PASSWORD fullname @FN

  • Drop the test logins after the test:

        drop login TEST_LOGIN1
        drop login TEST_LOGIN2
        drop login TEST_LOGIN3
        drop login TEST_LOGIN4

  • You can use these logins to test the solution proposed in Resolution

Cause

  • The syslogins table (table containing the information about the logins) have the TABULATION character in the column "fullname".
  • The column named "fullname" contains the detailed name of the login.
  • TABULATION is the default character used by BCP to separate column.
  • BCP is the tool used by ASE Server to import/export data using a flat file.
  • BCP is used by sap_materialilzation only for the materialization of master database to export/import the replicated system tables.
    • So, having TABULATION in "fullname" column is causing shift in column during import.
    • Information from the flat file are inserted into a wrong column.
  • This is causing conversion error during import of master..syslogins table.

Resolution

Solution is to remove TABULATION from "fullname" column.

  1. Identify if there are some rows with TABULATION character in the fullname column. 
    The column named "position" will return the position of the first TABULATION character in the fullname:
    select suid, name, position=charindex(char(9), fullname) from master..syslogins
    where charindex(char(9), fullname) > 0
  2. Create a set of command "alter login" for all the login with a TABULATION character in column "fullname" to replace TABULATION character by a BLANK character :
    --Create a work table #ALTER_LOGIN which will contains the logins with
    --fullname having TABULATION character:
    select suid , name, fullname into #ALTER_LOGIN
    from master..syslogins where charindex(char(9), fullname) > 0
  3. Correct the column #ALTER_LOGIN..fullname to replace all the TABULATION by a BLANK:
    update #ALTER_LOGIN
    set fullname = str_replace(fullname, char(9), " ")
    where charindex(char(9), fullname) > 0
  4. Use #ALTER_LOGIN table to create the set of command "alter login" to modify all the logins which have TABULATION in "fullname" column:
    select "COMMAND TO EXECUTE" = "alter login " + name + " modify fullname " + '"' + fullname + '"'
    from #ALTER_LOGIN
  5. To correct the problem, execute all the "alter login" commands which were created in the previous step.
  6. After this, you can execute: sap_materialize auto, SITE1, SITE2, master

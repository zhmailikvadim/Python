Symptom

When running the “Realign Group Reporting Preparation Ledger” job, the job fails with the message: “An error occurred. Please create an incident on component ‘FIN‑CS’.”

ABAP runtime error DATREF_NOT_ASSIGNED occurs in program CL_FINCS_RF_MAIN==============CP with short text: using a 'NULL' data reference is not possible.

Environment

SAP S/4HANA Cloud Public Edition

Reproducing the Issue

  1. Open the Realign Group Reporting Preparation Ledger app.
  2. Maintain the required parameters.
  3. Run the job.
  4. Observe that the job fails with the error message: “An error occurred. Please create an incident on component ‘FIN‑CS’.”

Category               ABAP programming error
Runtime Errors         DATREF_NOT_ASSIGNED
ABAP: Program          CL_FINCS_RF_MAIN==============CP
Application Component  FIN-CS-COR
Date and Time          03/31/2026 11:02:58 (UTC)

 

 Short Text
     Access using a 'NULL' data reference is not possible.

 

 What happened?
     Error in the ABAP application program.

     The current ABAP program "CL_FINCS_RF_MAIN==============CP" had to be
      terminated because it found a
     statement that could not be executed.

Cause

Including multiple consolidation units in the realignment selection where some units have no accounting data leads to a NULL data reference at runtime, causing the job to fail.

Resolution

  1. Identify the consolidation unit(s) that actually have accounting data for the selected ledger and period.
  2. In the Realign Group Reporting Preparation Ledger app, limit the selection to the unit(s) with existing data (avoid selecting units without data).
  3. Run the job for the identified unit(s) and verify it completes successfully.
  4. If realignment is required for additional units, run separate jobs per unit or ensure those units have relevant data before inclusion.

If you have only one units in the schedule job parameters, the issue may come from huge data volume. You need to add more filters or  specify one GL accounts and test it again.

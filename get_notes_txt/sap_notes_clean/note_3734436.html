Symptom

  • An ABAP short dump occurs with runtime error RAISE_SHORTDUMP and exception CX_CSP_WRP.
  • The issue is observed when creating an object in a custom RAP application.
  • A warning in the behavior definition indicates that the specified CDS view is not based on a database table for the "persistent table".
  • The error text informs that "The result of an external API call was not as expected."
  • Termination location: program CL_CSP_WRP_DD_CDS_TABLE_HEADERCP, method IF_CSP_WRP_DD_CDS_TABLE_HEADER~GET_TABLE_HEADER (include CL_CSP_WRP_DD_CDS_TABLE_HEADERCM001).

Environment

SAP S/4HANA Cloud Public Edition

Reproducing the Issue

  1. Open the custom RAP-based application.
  2. Select a business object instance (for example, a sales order line).
  3. Start an action that creates a related document (for example, create freight order).
  4. Confirm creation. The application terminates with an ABAP short dump.

Cause

Using a CDS view as the "persistent table" in the RAP behavior definition of a managed business object is not supported by the RAP Managed Runtime and leads to the dump.

Resolution

  1. Review the RAP behavior definition of the affected business object and locate the "'persistent table" setting.
  2. Replace the CDS view with a suitable database table that will store the persistent data for the business object.
  3. If a database table cannot be provided, implement persistence using "unmanaged save" and integrate it into the managed BO.
  4. Activate and transport the changes.
  5. Retest the creation/use case to confirm the short dump no longer occurs.

Symptom

A transport request is created to transport the printing details in transaction CQ85 or CB85 from one system to another. The transport seems to be successful, there is no error log in transaction SE01. However in the target system in transaction CQ85 or CB85 the output device is different from the one in the source system.

Environment

  • SAP ERP Production Planning (PP)
  • SAP R/3
  • SAP R/3 Enterprise
  • SAP ERP Central Component
  • SAP ERP
  • SAP Enhancement package for SAP ERP
  • SAP Enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA Cloud Private Edition

Reproducing the Issue

  1. After transporting printing details, open transaction CQ85 or CB85 in the source and target system as well.
  2. Go to Printer specification.
  3. Observe that a different output device is displayed in the two systems.

Cause

Different long device name is maintained to the output device with the same short name. It means that in table TSP03D for the same PADEST (Spool: short name for output device) value, a different NAME (Spool: Long device names) value is maintained in the two systems. During the transport the short name is correctly transported. In CQ85 or CB85 the long device name is displayed which is assigned to the relevant short name for output device. This is standard behavior.

Resolution

  1. Open transaction SPAD and review the output device configuration (table TSP03D).
  2. Ensure the intended short name (value saved in T496P-DRDEST) is assigned to the correct output device.

Symptom

  • As per Automatic Hire behavior, the user will not enter MPH so the Ready To Hire status email trigger is not necessary.
  • However, It is not possible to prevent the Ready To Hire status email trigger when using Automatic Hire.

Environment

SAP SuccessFactors Onboarding

Cause

Expected behavior

Resolution

  • The hiring type field which differentiates manual hires and automatic hires is not updated until user is in hired status.
  • As a result, since automatic hires go from Not Started -> Ready To Hire -> Hired Status in the space of seconds, the email will still trigger from Not Started -> Ready To Hire even though they don't actually enter MPH.
  • Since there is no field at this point to differentiate Manual from Automatic hire, there is no way to add a condition to the rule to stop this from triggering for automatic hires.
  • Enhancement requests can be submitted following KBA 2090228.

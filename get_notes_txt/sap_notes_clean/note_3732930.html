Symptom

When changing a billable maintenance order via BAPI_ALM_ORDER_MAINTAIN, the associated service order lock remains active after BAPI_TRANSACTION_COMMIT. The maintenance order lock releases automatically, but the service order lock persists until the session ends. This can be observed in the lock overview (SM12) after the commit completes.

Other Terms

BAPI_ALM_ORDER_MAINTAIN, CRM_ORDER_DEQUEUE, CRM_ORDER_ENQUEUE, ENQUEUE_E_CRM_ORDER, SM12, CL_ASM_UTIL_MCSO_INTEGRATION, CL_EAM_UTIL_MCSO_INTEGRATION

Reason and Prerequisites

The reason is a program error. The service order lock uses scope 3, which requires explicit dequeue, but no mechanism existed to release the lock after the update task receives lock ownership.

Solution

Implement the attached corrections.

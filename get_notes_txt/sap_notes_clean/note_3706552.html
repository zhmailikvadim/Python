Symptom

In T-code MD04, it is observed dependent requirements always inherit WBS element from the parent planned order.
However, you expect the dependent requirements to inherit grouping WBS element which is assigned to MRP group. 

Environment

  • SAP ERP Production Planning (PP) Release Independent
  • SAP ERP Release Independent
  • SAP ERP Central Component Release Independent
  • SAP Enhancement package for SAP ERP Release Independent
  • SAP S/4HANA, on-premise Release Independent
  • SAP Fiori for SAP ERP Release Independent
  • SAP Fiori for SAP S/4HANA Release Independent
  • SAP S/4HANA Cloud Private Edition Release Independent

Reproducing the Issue

  1. T-code CJ20n:
    set indicator ''Grouping WBS element'' =2- Grouping WBS element for selected MRP groups.
  2. T-code GRM5: 
    assign grouping WBS element=1000001 to MRP group=0001.
  3. T-code MM02:
    assign MRP group=0001 to the component material.
  4. Run MRP and check results:
    DepReq inherits WBS element=1000002 from the parent pldord.

Cause

This is standard system design.

Resolution

By standard, DepReq will get WBS element from the parent pldord, instead of grouping WBS assigned to MRP group. 
During BOM explosion, DepReq will have special stock indicator and WBS element inherited from the parent planned order. 
This ensures the parent material and its components are planned within the same planning segment by MRP run.

When creating individual requirements(like sales order, PIR) for the component, 
in such case, the grouping WBS element assigned to MRP group will be considered.

  • Indicator: Grouping WBS element/PRPS-GRPKZ

    This indicator determines whether a WBS element is used as a grouping WBS element.

    The indicator has two characteristic values:

    1 Grouping WBS element for all materials

    2 Grouping WBS element for selected MRP groups

    Use

    You use value 1, if the grouping WBS element should be valid for all material components that are allocated to a WBS element, which is assigned to this grouping WBS element.

    You use value 2, if you want to have different grouping WBS elements for the material components in a WBS element. This could be the case for instance, if you plan electrical and mechanical parts separately.

    Selection of the valid grouping WBS elements is on the basis of the MRP group and the plant for the relevant material.

    Dependencies

    If you have set the Automatic requirements grouping indicator in the project definition, you can only have one WBS element in the project that is the grouping WBS element for all materials (value 1).

Symptom

  • User being deleted in IAS after their status is set to "INACTIVE" in SuccessFactors.
  • User status in IAS should be updated to "INACTIVE" instead of being deleted.

Environment

Identity Provisioning Service (IPS)

Reproducing the Issue

  1. Set a user status to "INACTIVE" in SuccessFactors.
  2. Execute a read/sync job in IPS
  3. Observe that the corresponding users are deleted in IAS.

Cause

The filter in the SuccessFactors source system is set to status eq 'active' for API version 1 or for API version 2 active eq true

Which causes inactive users to be deleted from IAS.

Resolution

  1. Update the filter in the Success Factors source system to include both active and inactive users. Example: status in 'active','inactive' or active eq true or active eq false depending of the API version
  2. Apply the updated filter to the provisioning job in the Identity Provisioning Service (IPS).
  3. Test the updated filter using a read/sync job.

Symptom

  • The wage type /134(Annual Non-perk Amounts) is calculated as double the expected value in the INHRA function.
  • The value fetched from the CRT table is inconsistent with the COA/CLA basis calculation in INHRA function.

Environment

  • Human Capital Management (HCM)
  • SAP R/3
  • SAP ERP Central Component
  • SAP ERP
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • Payroll India

Reproducing the Issue

  1. Go to pc00_m40_calc t-code.
  2. Run the payroll for the issue employee.
  3. Check the value of the /134 wage type(CRT) from the processing details of the INHRA function. Along with that check the value of /134 wage type from CRT table.

         

         

Cause

Coding correction is missing in the Function CALC_133_134_COA_CLA. Incorrect pay-period has been fetched by the system during calculation.

Resolution

  1. Implement the note : 3472696 - Income Tax Deduction at Higher Tax Rate even with correct PAN details using the SNOTE transaction code.

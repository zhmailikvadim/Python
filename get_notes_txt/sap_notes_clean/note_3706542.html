Symptom

  • Custom Opportunity Document Types created in SAP Sales Cloud Version 2 are not visible after migration.  
  • Under Settings → Opportunities → Document Types, only the standard document type OPPT – Opportunity is displayed.  
  • No deletion entries are visible in the Change History/Changes tab for the missing document types. 

Environment

  • SAP Sales Cloud and SAP Service Cloud Version 2 
  • SAP Sales Cloud Version 2 
  • SAP Cloud for Customer core applications

Reproducing the Issue

1. Log in to SAP Sales Cloud Version 2.  
2. Navigate to Settings → Opportunities → Document Types.  
3. Observe that only the standard document type OPPT—Opportunity is displayed.  
4. Notice that custom Opportunity document types are not visible, even though they appear as created in the Changes tab.  

Cause

This behavior occurs due to a re-execution of Business Configuration (BC) migration from Sales Cloud V1 to V2 after custom document types were created in V2.

Detailed Explanation

  1. In the V1 (C4C) tenant, only the standard Opportunity document type exists:
    • OPPT – Opportunity
  2. During the initial V1 → V2 migration, Opportunity Business Configuration is migrated successfully to V2.
    • As expected, only OPPT is available in V2 after migration.
  3. After migration, custom Opportunity document types (for example:Z001,Z002) are created directly in the V2 tenant.
  4. Later, the Opportunity Business Configuration migration is executed again from V1 to V2.
  5. Since:
    • The V1 system does not contain the custom document types (Z0O1, ZO02)
    • Migration is designed to synchronize V2 configuration with V1
    • The re-migration overwrites the V2 configuration, removing all document types that do not exist in V1.
  6. As this is a synchronization overwrite (not a manual deletion), no deletion record appears in the change history. 

Resolution

This is expected behavior of the V1 → V2 Business Configuration migration.

To restore the document types:

  • Recreate the required custom Opportunity document types in the V2 tenant.

Recommendation / Prevention

To avoid this issue in the future:

  1. Do not re-run Opportunity Business Configuration migration from V1 to V2 after:
    • Initial migration is completed
    • Custom document types are created in V2
  2. Perform Business Configuration migration only when alignment with V1 is explicitly required.

Symptom

  • 1KYF_ comment key figure is involved in the aggregation level
  • Error in Analysis for Office when saving planning data: “Database table '/BIC/Z...' is unknown (RS_EXCEPTION-000).”
  • Short dump when saving in planning context: SAPSQL_PARSE_ERROR with CX_SY_DYNAMIC_OSQL_SEMANTICS in SAPLRRSI; parser reports " '/BIC/Z...' is not declared as a table/view in ABAP Dictionary.”
  • Table '/BIC/Z...' does not exist in DDIC or database.

Environment

  • SAP BW/4HANA
  • SAP S/4HANA

Reproducing the Issue

  1. In RSPLAN, run a planning sequence where a specific step with planning function type 0RSPL_DELETE on a standard ADSO, planning enabled. Then try to save the planning buffer. Observe a short dump (SAPSQL_PARSE_ERROR) referring to non-existing '/BIC/Z...' table.
  2. In Analysis for Office, open a workbook that executes the same planning sequence, let the sequence finish, then select Save Data. Observe error: “The database table '/BIC/Z...' is unknown (RS_EXCEPTION-000).”

Cause

  • The specific step of the planning sequence uses type 0RSPL_DELETE on a standard ADSO in planning mode. During save of the after-image planning buffer, this leads to an unexpected code path where dynamic SQL references a generated '/BIC/Z*' table that does not exist, causing the parse error.

Resolution

  1. Replace the planning function type 0RSPL_DELETE with 0RSPL_DELETE_DSO in the planning sequence for the standard ADSO with planning enabled.
  2. Implement below SAP Notes:
    • 3585041 - Error after reset of after-image planbuffer.
    • 3607546 - 1KYF keyfigures not correctly mapped from AI buffer

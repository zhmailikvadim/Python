Symptom

Multiple users cannot access their Time Sheet, blank time sheet displayed for some employees.

Environment

  • SAP SuccessFactors Employee Central - Time Sheet
  • SAP SuccessFactors Time Tracking

Reproducing the Issue

1) Go to the Time Sheet of an employee
2) Blank Time Sheet UI displayed with message "No data"

Cause

First of all, please make sure that you have checked below points:

  1. Compare the browser version and the PC OS version for a user for whom it is working properly and a user who faces blank Time Sheet. 
  2. Ask the user who faces the issue to access the SF Time Sheet from another network environment.
    For example: try it from home if in the office it does not work. If it works somewhere else, this further confirms the issue comes from the network environment.
  3. Ask the user who faces the issue to try it in another PC / Browser of another employee. If it becomes working even with same network environment only after changing the PC/browser, it indicates the issue comes from the browser version or pc OS version. Then the user should upgrade the versions too.
  4. Please check if there was any antivirus update on user's laptop/PC which might block the access to the timesheet view.


If the above points have been checked, please perform an additional step:

5. In the user's browser, please follow the below steps: 
> Open DevTools (F12 or Right-Click>Inspect)
> Go to Network Tab
> Ensure that Disable Cacheis checked.
> Refresh the page and then export the HAR file
>> Optionally check, if the v2/ request appears

From the HAR file, most probably we can see that the HEAD request is failing/problematic, fetching the CSRF token. 

Resolution

For point 5: Our Engineering Team have been working with enhancement to improve the robustness of the time sheet UI rendering.
Product Engineering Team prepared a fix, PTCH-49103. 
This patch is planned to be delivered with P17 patch cycle, meaning:
-- February 10th for Preview
-- February 12th for Production

This KBA will be updated once there is any change on the planned fix deployment date!

Symptom

Report RS_FIND_ISOSMAP_DUPREC was executed and it reported duplicate or double transfer structure names in table RSISOSMAP. 

Report RSAR_RSISOSMAP_REPAIR was executed to correct the duplicate transfer structure but it fails with error:

TSPREFIX for <old LOGSYS name> not found
TSPREFIX for <new LOGSYS name> not found

Environment

SAP Enhancement package 1 for SAP NetWeaver 7.0

Cause

The same transfer structure is associated with two different source systems in table RSISOSMAP.

Resolution

SAP Note "3065275 - 740SP26: 'RSAR_RSISOSMAP_REPAIR' doesn't work for InfoSources which are associated with Orphanned source system" is not applicable to your current support package. Hence, please follow below steps:

  1. Navigate to table RSISOSMAP and search for the affected OLTP object. If the object is an Export Datasource of a DSO, search for 8<DSO_name>.
  2. Check column TRANSTRU, the same transfer structure is associated with two different source systems (LOGSYS column). 
  3. Execute the Function Module RSAR_LOGICAL_SYSTEM_DELETE and delete the obsolete source system with force mode. 
  4. Execute program ZSSCLEANUP for cleaning up entries from tables RSOSOHIE, RSOSOHIET, RSOAPPL, RSOAPPLT, RSSHIPDVERS and RSISOSMAP. This report was introduced with note 0001815374, you can copy the program 'ZSSCLEANUP.TXT' from the Attachment section of this note, use transaction SE38 to create the program in your system.
  5. Execute the reports RS_FIND_ISOSMAP_DUPREC and RSAR_RSISOSMAP_REPAIR again.
  6. Reactivate the affected objects: 
    • For Datasources, use report RSDS_DATASOURCE_ACTIVATE_ALL
    • For Export Datasources, use report RSDG_ODSO_ACTIVATE to reactivate the associated DSO.

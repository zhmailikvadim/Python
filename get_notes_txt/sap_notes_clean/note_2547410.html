Symptom

  • While performing an upgrade with the Software Update Manager (SUM)  the following error occurs during phase PREP_CONFIGURATION/SUBMOD_MIG_GETKRNL/SCANDIR_MIGKERNELS:
  • The following error can be found in the SAPupConsole.log 
Action PREP_CONFIGURATION/SUBMOD_MIG_GETKRNL/SCANDIR_MIGKERNELS, Type error: SAP Kernel requested
No suitable SAP Kernel archive for platform <system platform> was found in the
download directory.
  • In SCANDIRMIGKRNLS.LOG file, you find messages similar to these:
1 ETQ399 Selecting kernel archives for target 'exe_2nd'.
2 ETQ399 Archive '<download directory>/SAPEXEDB_801-10011374.SAR' is not compatible to SAP_BASIS release '740' 
2 ETQ399 Archive '<download directory>/SAPEXEDB_301-70001628.SAR' is for different DB-type 'db6' != requested 'hdb'.
2 ETQ399 Archive '<download directory>/SAPEXE.SAR' is for different codepage 'NUC' (requested 'UC').
1EETQ399 Last error code set is: Archive for second DB is for platform 'linuxx86_64' != local platform
2WETQ399 SAPEXEDB_301-80000698.SAR' has lower patch number '301' than the system kernel ('321')

Environment

Software Update Manager

Cause

The Kernel related files located in the download directory are not suited for the system on which the upgrade is being performed.

The *.SAR files are targeted for system different from the one trying to perform the upgrade.
The differences between the systems targeted by the *.SAR and the actual upgrading system might be:

  • SAP_BASIS release
  • Database type
  • Unicode/non-Unicode codepage
  • Operating system

Resolution

  1. Search for the appropriate Kernel files for your system in SAP Software Center
  2. Download the required files
  3. Remove the .SAR files mentioned as incompatible in SCANDIRMIGKRNLS.LOG from the download directory
  4. Place the freshly downloaded Kernel files in the download directory
  5. Repeat the SUM phase

Note: You can find the kernel versions suitalbe for the target relase in the Product Availablitiy Matrix.

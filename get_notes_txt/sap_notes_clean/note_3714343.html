Symptom

  • Defect catalogs (Catalog type 9) assigned the status "cannot be used" are displayed and available for selection in the Record Production Defects App (F2929).
  • Users are unable to proceed after selecting a defect code group/code with the status "cannot be used".
  • System does not restrict the selection of code groups/codes that are not in the "Released" status.

Environment

  • SAP R/3
  • SAP R/3 Enterprise
  • SAP ERP Central Component
  • SAP ERP
  • SAP Enhancement package for SAP ERP
  • SAP Enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA, on-premise
  • SAP S/4HANA Cloud Private Edition

Reproducing the Issue

  1. Check the configuration for Catalog type 9 and select a code group with the status "cannot be used".
  2. Record a defect in the Record Production Defects App (F2929) using the code group with the status "cannot be used".
  3. Observe that the system displays code groups that are not in the "Released" status.
  4. Attempt to save the defect. The system does not allow saving the defect due to validation errors.

Cause

this is a missing functionality in Record Production Defects App in S4CORE 2021 release which has been corrected from S4CORE 2023 release onwards.

From S4CORE 2023 release onwards you would be seeing an additional column named as "Code Group is Assignable" and "Code is Assignable" in form of checkbox in the F4 help of code group and code which will be marked with 'X' only when the code group/code is with status "Released".
In this way you will get to know which are the releases ones.

Unfortunately this fix is available only from OP2023 release because of complex changes required to achieve this behavior.
I am also attaching the screenshot of our S4CORE 2023 internal system for your reference. So, kindly upgrade to S4CORE 2023 release and then your issue will get resolved.

Resolution

Upgrade to SAP S/4HANA 2023 release, which includes the required fix.

  • From SAP S/4HANA 2023 release, the Record Production Defects App (F2929) introduces an additional column in the F4 help of code groups and codes:
  • "Code Group is Assignable"
  • This column is displayed as a checkbox and marked with 'X' only when the code group/code is in the "Released" status.
  • This functionality ensures that only code groups and codes with the "Released" status are selectable in the app.

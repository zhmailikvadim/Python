Symptom

Msg 2116 when creating a trigger that selects text or image columns from the inserted or deleted trigger tables.

Msg: 2116, Level: 16, State: 1
CREATE TRIGGER failed because selecting from a TEXT, IMAGE, UNITEXT or off-row Java datatype column of the inserted or deleted table is not supported.

Environment

  • Sybase Adaptive Server Enterprise (ASE) 15.5
  • SAP Adaptive Server Enterprise (ASE) 15.7
  • SAP Adaptive Server Enterprise (ASE) 16.0

Reproducing the Issue

1.  create table table_text_propagation_test (id int identity not null, a varchar(255) null, txt text null)
     go

2.  create table table_text_propagation_test_dest (id int identity not null, a varchar(255) null, txt text null)
     go

3.  create trigger i_table_text_propagation_test on table_text_propagation_test for insert as
     insert into table_text_propagation_test_dest (a, txt) select a, txt from inserted
     go

Cause

Starting with ASE 15.0.3 ESD 4, 15.5 ESD 2, and 15.7, ASE started raising error 2116 if the code for a CREATE TRIGGER attempted to reference a LOB (text, image, unitext) column from the trigger's inserted or deleted tables.  Selecting text or image data from the inserted/deleted pseudotables in triggers has never been supported;   the documentation has always stated that it cannot be done although ASE hasn't previously enforced the restriction.  For more details, see https://wiki.scn.sap.com/wiki/display/SYBASE/ASE+Error+2116

Booting ASE with traceflag 1716 (put -T1716 in the dataserver parameter list in the RUN_SERVER file) will give  the same behavior as pre-15.0.3 ESD 4 versions (including the errors and incorrect results when selecting text values from the deleted pseudotable).

Feature request CR 593730 is open to add support for selecting text and image columns from the inserted and deleted tables in triggers.
This functionality was then added in 16.0 SP04 PL04 under CR 827694 when the "functionality group" sp_configure option is set on.
CR 593730 remains open to allow it even when "functionality group" is not active.

Resolution

A Feature Request has been logged under FR 593730. The Feature Request will be considered in terms of future product development when considering possible new features in the next major release. Please note that the Feature Request may or may not be included depending on various factors.

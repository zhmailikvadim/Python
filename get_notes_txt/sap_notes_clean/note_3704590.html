Symptom

An error occurs in the advanced intercompany supplier invoice with the error message MRM_BASE012: "Cannot determine tax code for tax rate X, type , ctry/region YY."

Environment

  • Materials Management (MM)
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA
  • Advanced Intercompany Scenario

Reproducing the Issue

  1. Attempt to create an advanced intercompany supplier invoice
  2. In the App Monitor Value Chain, select the Supplier Invoice
  3. On the Message Log tab, verify the displayed error message and details.

Cause

The error message MRM_BASE 012 — "Cannot determine tax code for tax rate X, type , ctry/region YY" — appears for the SIIC invoice when the system cannot determine a tax code. A tax code is mandatory for posting supplier invoices.

This typically happens when the CIIC billing document has no tax condition. Normally, a tax condition is maintained in the billing document and its tax information is transferred to the SIIC invoice. During SIIC processing the system maps that tax information using the customizing settings to determine the supplier invoice tax code. If the mapping fails and no tax code is found, MRM_BASE 012 is raised. Example: "Cannot determine tax code for tax rate 0.000000, type , ctry/region DE."

Resolution

A. Ensure that the tax condition is maintained in the CIIC billing document (IV2).

B. Alternatively, maintain the following entry in the customizing:

  1. SPRO - Navigate to Materials Management → Logistics Invoice Verification → Advanced Intercompany Processes → Assign Tax Codes for the Transfer of Supplier Invoices.
  2. Maintain the following entry:
Invoicing partyTax TypeTax RateCountryTax Code
  0,000000DEYY

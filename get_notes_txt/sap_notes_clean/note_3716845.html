Symptom

When determining the next valid time using the function module TSTR_GET_NEXT_VALID_TIME, the system may return an incorrect begin/end time or raise exception TSTREAM_NOT_LOADABLE. This can occur when the requested date is outside generated time‑stream years, when segments span multiple days (>24 hours), or when calculating in negative/positive direction around year boundaries.

Other Terms

TTSTR, TTSTRCOMPH, TSTRCOMPID, TSTR_COMPSGTAB

Reason and Prerequisites

The reason is a program error in the algorithm of TSTR_GET_NEXT_VALID_TIME:

  • Segment durations were not reliably translated into absolute date/time for comparison.

Prerequisites: A time stream with generated years stored with valid HANDLE in TTSTR/TTSTRCOMPH, and calls to TSTR_GET_NEXT_VALID_TIME. The input date is not among the generated years. In this edge case, we get an error isntead of returning the exisitng valid segment

Solution

Apply attached Correction Instruction via SNOTE.

The correction:

  • Introduces internal segment structures to collect year‑wise segments and converts component offsets to absolute DATE/TIME using internal form for calulation.
  • Selects available years by HANDLE, imports segment data, and evaluates segments based on IV_DIRECTION to find the correct next/previous point in time which is valid.
  • The new logic never generates missing segments, it only returns existing segments based on the input requirements.

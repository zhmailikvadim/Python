Symptom

  • Error message: "500 Internal Server Error" occurs when attempting to log in to the mobile services cockpit.
  • The issue is observed across all browsers.
  • The error occurs when signing in using the default identity provider with an s-user ID.

Environment

  • SAP Universal ID

Reproducing the Issue

  1. Open the Mobile Services URL example: https://<mobile-service-cockpit>.cfapps.<xxxx>.hana.ondemand.com/cockpit/v1/org/xxxxxxxxx/space/xxxxxx/index.html#/page.home in any browser.
  2. Select "Sign in with default identity provider".
  3. Sign in using the s-user ID.
  4. Receive error message: "500 Internal Server Error" from GET request.

Cause

  • Missing scopes in the UAA settings were identified as the root cause of the issue.

Resolution

  1. Access the UAA Cockpit using the URL: https://uaa.cf.xxx.hana.ondemand.com/.  Example https://uaa.cf.us10.hana.ondemand.com/
  2. Verify the following scope items are checked:
    1. View details of your applications and services.
    2. Push applications to your account and create and bind services.
    3. Verify user permission to manage service instances.
    4. Access your data with scope 'openid'.
  3. Update the settings and retest the login functionality to Mobile Services cockpit.

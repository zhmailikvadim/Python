Symptom

  • I can fill out my worker's information for registration, but I cannot proceed 
  • I only see the option to Sign in, but not to create an account
  • Nothing happens when I click Sign In after trying to register worker
  • Registration page only gives me the option to Sign In

Cause

This is a session-related issue that occurs as part of the system’s design. It typically happens when a user performs two different actions within the same session.

In this scenario, the issue arises when the user first resends the invitation and then the supplier attempts to complete the worker registration within the same session. This leads to a session conflict. 

Resolution

To avoid session issues, we recommend following one of the best practices below:

  1. Logout and start a fresh session
    After resending the invitation, you should completely log out of the application and close the browser tab to terminate the current session. Then, open the registration link received in the email to complete the registration.
  2. Use different browsers for different actions
    Log in and resend the invitation using one browser. Then, copy the URL from the worker registration email and open it in a different browser to complete the registration. This ensures that both actions occur in separate sessions and prevents session conflicts.

Symptom

You would like to clarify what are the steps for SAP Readiness Check execution in non-unicode SAP ERP systems

Environment

SAP ERP 6.0

Resolution

SAP Readiness Check for SAP S/4HANA supports non-Unicode systems. The report is designed to analyze SAP ERP systems 6.0 EHP0-EHP8 regardless of whether it is running on a non-Unicode or Unicode character set, allowing it to identify necessary preparation steps for S/4HANA conversion. 
Just keep in mind the following points for Non-Unicode Systems:

    Purpose: The SAP Readiness Check will identify if your system is non-Unicode and highlight this as a required pre-project action.
    Requirement: While the check works on non-Unicode, 100% Unicode compliance is mandatory for the final conversion to SAP S/4HANA.
    Workflow: You can run the Readiness Check on your current non-Unicode system, use the findings to plan, and then perform a separate Unicode conversion project prior to the actual S/4HANA migration.
    Data Collection: The analysis is executed using the /SDF/RC_START_CHECK report in your SAP ECC system. 

In summary, SAP Readiness Check is designed to help you prepare for a conversion and it will correctly diagnose a non-Unicode system, but that system must be converted to Unicode before moving to S/4HANA.

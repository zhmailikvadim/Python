Symptom

  • The 'sqm_recover_segs' configuration parameter is set to 10.
  • When SAP Replication Server (RepServer) restarts, it takes 20 minutes for the outbound queue to clean up duplicates from the inbound queue. This indicates that RepServer reads more than 10 segments from the inbound queue. 
  • When the inbound queue backlog is fairly high, the DIST thread may be in the duplicate phase for a much longer time after RepServer restarts.

Environment

SAP Replication Server (SRS) 15.7.1 SP102, but not limited to this SP level

Cause

SAP CR 763245 has been filed to address this issue. Its description reads, "Restart RepServer takes long time to clean up duplicates".

Resolution

  • There is currently no target Fix Pack or Service Pack scheduled for SAP CR 763245. It is being reviewed by our Product Group. Subscribe to this KBA to be automatically notified when SAP CR 763245 is fixed.
  • There is no workaround.

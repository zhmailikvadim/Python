Symptom

  • The Joule assistant icon/entry point does not appear in the S/4HANA Cloud Private Edition Fiori Launchpad.
  • The S/4HANA system was auto-discovered and added to the Joule Formation, but the assistant still does not show up in FLP.

Environment

  • Joule.
  • S/4HANA Cloud Private Edition.

Cause

  1. The user is not assigned the role that contains the Fiori business catalog for the Joule plug‑in.
  2. Browser or server caching is preventing updated content from appearing.
  3. Third‑party cookies are blocked by the browser.
  4. Fiori customizing settings conflict with the current configuration.

Resolution

  1. Ensure the user is assigned the "Single Role" that includes the Fiori business catalog for the Joule plug‑in. 
  2. Clear the browser cache. If the issue persists, invalidate the server caches by running program /UI2/INVALIDATE_GLOBAL_CACHES.
  3. Enable third‑party cookies in the browser’s privacy/settings so the plug‑in can load required resources.
  4. Use transaction /UI2/FLC to inspect the launchpad content — first run the check with the "Configuration" option, and if no errors appear, run it again for "Customizing" to identify conflicting items.

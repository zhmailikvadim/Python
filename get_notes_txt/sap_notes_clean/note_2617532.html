Symptom

When you recalculate the depreciation using transaction AFAR, the runtime error SY_ARITHMETIC_OVERFLOW occurs in CL_FAA_DEPRECIATION_CALC after a long runtime.

Other Terms

COMPUTE_INT_PLUS_OVERFLOW dump short dump

Reason and Prerequisites

The error occurs if there are general problems when locking the company code or if you have locked a company code to be processed in the IMG against changes. You can set the company code lock in the IMG under Asset Accounting (New) -> Preparations for Going Live -> Production Startup -> Lock Company Code Against Changes in Asset Accounting.

Solution

Implement the attached correction instructions or import the specified support package.

In future, the system then terminates the recalculation of the depreciation with a corresponding error message.

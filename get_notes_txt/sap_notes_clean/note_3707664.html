Symptom

  • When accessing transaction BP or FPP2/FPP3 and navigating to the "Where-used" tab, Contract Object is not visible.
  • The "Where-Used List" tab displays the Contract Account (CA) but does not display the Contract Object (CO).

Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.

Environment

IS-PS-CA - Public Sector Contract Accounting

FI-CA - Contract Accounts Receivable and Payable

Reproducing the Issue

  1. Log into system.
  2. Call transaction BP or also FPP2/FPP3.
  3. Enter Business Partner (with role MKK).
  4. Access the "Where-Used List" tab.
  5. Observe that Contract account is displayed, but Contract Object is not visible.

Cause

  • The "Where-Used List" tab is designed to display only Contract Account details and CI-specific information, such as provider contracts or prepaid accounts.
  • Application-specific contracts are intentionally separated into a distinct tab.

Resolution

  1. Verify that the "Contract Objects" tab is available in transaction BP/FPP3.
  2. The available Contract Objects will be displayed here.
  3. If the "Contract Objects" tab is missing, ensure the system configuration aligns with the guidance provided in KBA 2941979.

See Also

Refer to: KBA 2941979 - BP/FPP3: Tab Contracts is missing after migration to S/4HANA 1909

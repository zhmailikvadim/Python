Symptom

Error DEFTAX_MX070 is raised when using RFUMSV50 with multiple active lines , specially with tax codes with reverse charge

Environment

  • Financial Accounting (FI)
  • SAP R/3
  • SAP R/3 Enterprise 
  • SAP ERP Central Component
  • SAP ERP
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA Finance
  • SAP S/4HANA
  • SAP Fiori for SAP S/4HANA Finance
  • SAP Fiori

Reproducing the Issue

  1. Run RFUMSV50 for tax code with multiple active lines
  2. Update run option is not selected
  3.  Try to use option to post tax document from ALV
  4. You get error DEFTAX_MX 070 Do not post from ALV documents w/tax codes w/several active conditions 'Target tax code'.

Cause

Incorrect use of program RFUMSV50 to select documents by ALV, SAP does not recommend to use this feature. 

Resolution

1) If you are using deferred tax solution with reverse charge is mandatory to follow customizing as per note:

  •  2011533 - RFUMSV50: Reverse Charge tax code support - SAP for Me

2) See point 17 of main note 1800344:

When you select line item and click Post button the report does not know what is your intention - to create tax document only for this line item or for all line items. You may create inconsistence if you will post tax document for one line. If you have tax codes with several active conditions use always option "Update run" on the selection screen to post tax transfer documents.

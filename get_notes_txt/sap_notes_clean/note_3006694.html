Symptom

After you implement SAP Note 2920873 or import the relevant support packages (or CLCs), the central wage type /2ZT is no longer generated.

This affects the following:

Release 608 support package 86 and subsequent releases/support packages

Release 604 support package 158 and subsequent releases/support packages

Release 600 support package 192 and subsequent releases/support packages

A claim is almost always created in the subsequent months subject to SI after an employee leaves with SI obligation.

Other Terms

ASV00 U

UEKV

Payment in lieu of unused leave

Reason and Prerequisites

SAP Note 2920873 was originally published for a BVA-specific correction.

However, some of the manual tasks for this SAP Note are generally relevant for the function ASV00 U and also have an effect for persons insured by the Austrian health insurance fund (ÖGK).

The SVX branch in the HR feature ASV01 is necessary because - while SI contributions are simulated for the subsequent months under ASV00 - the last simulation step 9999 for ASV01 is called with the parameter SVX.

This was necessary for a BVA-specific enhancement. In general, however, the last simulation step is not the last step of the SI calculation in the schema.

Solution

Note the manual activity for adjusting the HR feature ASV01 in SAP Note 2920873 (for ÖGK as well). Check whether this adjustment is active in your system.

If this HR feature ASV01 was changed by the customer, you must implement the delivered corrections manually in the customer client.

Relevant excerpt from the manual adjustment in SAP Note 2920873 for your information:

Transaction SE11: P03_AMODE (call context for characteristics)

Create the following value for the domain P03_AMODE:

SVX - Simulation Under ASV00

Transaction PE03: ASV01

Add the following specification to the decision tree:

AMODE = SVX -> Return value = 900

DSAPZS

SVF DAMODE

SVF SVX&ASV01=900,

...

*** DAMODE

*** SVX&ASV01=900

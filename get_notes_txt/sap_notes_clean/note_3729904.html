Symptom

 During the YT archiving process, a dump with the following short text is generated: Exception condition "DATA_OBJECT_NOT_SAVED" triggered.

This issue is related to the presence of custom ILM policies for archiving objects.

Other Terms

/SAPYL/CL_YT_A_ROOT->DO_ARCHIVATION, /SAPYL/YT_CHANGE_FIELD_ARCH

Reason and Prerequisites

The dump occurs during empty field Confirmed On timestamp processing for Cancelled YT. During archiving process there is an assignment set for Cancelled YT only: Confirmed On = Changed On.

Solution

Implement this SAP Note

Run the report SE38 /SAPYL/YT_CHANGE_FIELD_ARCH before attempting to archive. It processes Confirmed On = Changed On for Yard Tasks, where archiving has been already started.

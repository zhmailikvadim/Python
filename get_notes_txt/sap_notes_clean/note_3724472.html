Symptom

A third-party/external system throws an error message accusing that a certificate generated in SuccessFactors is in an invalid PEM format.

Environment

SAP SuccessFactors HCM Suite

Reproducing the Issue

  1. Generate a .pem certificate in Security Center or Manage OAuth2 Client Applications;
  2. Attempt to upload the certificate into a third-party system;
  3. The external system returns an error indicating that the PEM format is invalid;

Cause

Although PEM is a standardized format, external systems may required slight variations.
 
Common reasons include:
    1. Encrypted PKCS#8 Private Key
      SuccessFactors certificates has a PKCS#8 encrypted private key (-----BEGIN ENCRYPTED PRIVATE KEY----- ).
      Many external systems do not accept encrypted private keys, requiring instead unencrypted PKCS#8 (-----BEGIN PRIVATE KEY----- ) or unencrypted PKCS#1 (RSA) (-----BEGIN RSA PRIVATE KEY----- ).
      If the system cannot decrypt the key or does not support PKCS#8 encrypted keys, it will reject it, even though the format itself is valid PEM.
          
    2. Missing or Non‑Standard Line Breaks
      Some SAP systems export PEM as one giant line of Base64.
      PEM is technically readable without line breaks, but many tools expect 64‑character wrapped lines.
         
    3. SAP Metadata in the Base64 Content
      The SuccessFactors certificate include metadata appended to the Base64 payload. It ends with IyMjZmFsc2U=which decoded means ###false.
      External systems often treat this as invalid Base64 or invalid PEM.
         
    4. File Structure Requirements Differ Between Systems
      SuccessFactors certificate contains both Private & Public Keys. However some external systems require the private key and certificate (public key) in separate files (key.pem, cert.pem, chain.pem).

    Resolution

    1. Consult the third-party documentation to confirm the exact PEM structure it requires. If needed, contact the third‑party support;
    2. Manually adjust the SuccessFactors‑generated certificate to match the required format;
    3. Save it and upload it;

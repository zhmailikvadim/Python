Symptom

In some cases, the first line of the long text of Work Order Sub-Operation in the database is empty, rather than matching the short text of Work Order Sub-Operation. In other words, for the long text, the FM READ_TEXT_TABLE returns an empty string as the first line instead of the corresponding short text.

In this case, the GET request MyWorkOrderSubOpLongTexts retrieves the long text exactly as stored in the database. As a result, the first line in the SSAM Application is displayed as empty. In contrast, in transactions IW32/IW33, the short text is automatically copied to the first line of the long text.

The SAP Note introduces a new data filter 'COPY_SHORT_TEXT_TO_LONG_TEXT' in the configuration panel. If the new filter is enabled, then in GET requests MyWorkOrderSubOpLongTexts, when the first line of the long text is empty and does not match the short text, the short text is copied to the first line of the long text.

Please note that for S4CORE 106 SP3, no data filter is applied; the functionality is always enabled.

Other Terms

S4MOBADDON-8096, MyWorkOrderSubOpLongTexts, WO Sub-Operations created using the RAP framework, New Data Filter, /MERP/CL_PM_WORKORDER_OD, GET_WORKORDER_LONGTEXT

Reason and Prerequisites

Programming Error.

Solution

Please implement the SAP Note and, if available and required, configure the new data filter 'COPY_SHORT_TEXT_TO_LONG_TEXT' in tr-code /n/SYCLO/CONFIGPANEL -> Section 'OData Mobile Data Object Configuration' -> oMDO Id SAM<XXXX>_WORK_ORDER_GENERIC -> Tab 'Data Filter' -> Operation READ -> Standard Filter, where <XXXX> is your application version, for example, 2505.

Please be aware that although the new filter is visible for oMDO Ids SAM<XXXX>_WORK_ORDER_GENERIC_EAM, enabling it may impact text handling for EAM Work Orders. It is therefore recommended to enable this filter only for SAM<XXXX>_WORK_ORDER_GENERIC.

Please note that for S4CORE 106 SP3, no data filter is applied; the functionality is always enabled.

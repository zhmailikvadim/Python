Symptom

While trying to output a query locked records is not being displayed with message "System could not read any data"

Environment

  • SAP ERP
  • Personnel Information Systems

Reproducing the Issue

Select any query where locked records should be displayed. 

Cause

By default, locked data records are not processed in the query. 

Resolution

  1. In the InfoSet code used by the query/report, set the switch PNP-SW-IGNORELOCKEDRECORDS to N in the INITIALIZATION or START-OF-SELECTION event to include locked data records.
  2. Regenerate the InfoSet and all dependent queries:
    1. In transaction SQ02, choose Goto > Query Directory, enter the InfoSet name, execute, select all related queries, then choose Edit > Generate Program.
    2. Alternatively, in SQ01 choose Query > More Functions > Generate Program. Restart SQ01 before generation to clear buffered versions.
  3. Execute the query.

Symptom

1. Model contains a calculation that references a date dimension

2. The calculation no longer returns data after the default hierarchy of the date dimension changes.

Other Terms

Restrict, Lookup, Iterate, Date, Hierarchy, Calculation, Mode, Default, YTD

Reason and Prerequisites

Some calculations internally reference the default hierarchy of the models date dimension.

For example, ITERATE or RESTRICT functions using a date dimension.

If the default hierarchy of the date dimension is changed this can affect the results of the calculation.
The calculation must be recompiled to correctly reference the new default hierarchy.

This is a known limitation of Time Hierarchy when working with calculations

This issue is currently under investigation by development.

Solution

The calculation can be rebuilt by making a small change to the formula. For example deleting a character and reentering it.

This will cause the internal calculation logic to be rebuilt and reference the new default date hierarchy.

This issue is currently under investigation by development.

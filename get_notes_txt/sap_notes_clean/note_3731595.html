Symptom

When executing the "Check" function in the Manage Allocations app (F3338), the system raises an incorrect error message for cycles that use Country/Region (LAND1) groups or hierarchies on the receiver side:

"The group &1 used in cycle &2 is invalid"

The cycle can still be saved despite this validation message. The error occurs specifically when the group was created as a Global Hierarchy in the Manage Global Hierarchies app.

Other Terms

GA752, LAND1, COBL_LAND1_GP, G_SET_TREE_IMPORT, illegal_field_replacement, set_is_damaged, RK_AL_CYCLE_CHECK, Global Hierarchy, Country/Region, Manage Global Hierarchies, F3338

Reason and Prerequisites

During the cycle check, the function module RK_AL_CYCLE_CHECK calls G_SET_GET_INFO to validate the groups used in the cycle. For Universal Allocation, the runtime structure ACDOC_ALLOC is used for DDIC field resolution. In this structure, the Country/Region field LAND1 is defined with data element COBL_LAND1_GP.

Solution

Implement the following correction instructions or apply relevant support package. The fix adjusts FORM TEST_SET to bypass the field replacement for the Country/Region field when the data element COBL_LAND1_GP is detected, allowing G_SET_GET_INFO to resolve the set using its own definition.

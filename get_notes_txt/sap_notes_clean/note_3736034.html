Symptom

  • CALM is not running collection jobs for a connected S/4HANA ABAP system.
  • In Landscape Management application Runtime status for use cases shows "Error" while configuration status shows "Active".
  • No jobs are visible in SM37 when searching with "*CALM*" for the recent timeframe.
  • Data appears to be collected despite no jobs running.

Environment

  • SAP Cloud ALM

Reproducing the Issue

  1. Logon to S/4HANA ABAP system.
  2. Run the setup transaction /n/SDF/ALM_SETUP
  3. Complete the setup steps and register the managed system 
  4. After go to the Cloud ALM tenant and navigate to Administration=> Landscape Management => Select the system
  5. Under the section "Supported Use-Cases" observe that for some use cases the runtime status shows "Error" while configuration status shows "Active".
  6. In the managed system, check SM37 and search with "*CALM*" within the last seven days; observe that no jobs are found.

Cause

There was a previous connection of the managed system to the cloud tenant using a different destination that was not cleaned up or deregistered correctly. 

Resolution

  1. Stop data collection for the affected managed system in CALM Health Monitoring.
  2. Remove the legacy/old destination related to the previous connection in both the managed system and the cloud tenant configuration, if applicable.
  3. Deregister the existing connection of the managed system from CALM.
  4. Repeat the system connection setup again using the correct destination and register the managed system again with CALM.
  5. Activate the required use cases 
  6. On the managed system, verify in SM37 that jobs with names containing "CALM" are scheduled for the correct client.
  7. On Cloud ALM tenant, check in Landscape Management that the runtime status changes to "Active" and that data collection aligns with scheduled jobs.

Symptom

  • Wage type /616 (arrears from previous financial year) is forming incorrectly in case of cross FY retro payroll run.
  • Gross salary is formed wrongly via INTAX function in case of cross FY retro payroll run.
  • Correct value of /616(Add Gross Salary) is not getting added to /416 wage type.(Gross Salary)

Environment

  • Human Capital Management (HCM)
  • SAP R/3
  • SAP ERP Central Component
  • SAP ERP
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • Payroll India

Reproducing the Issue

  1. Go to PC00_ M40_CALC t-code.
  2. Run payroll for a cross–financial year scenario where arrears has been transferred to wage type /616 from previous financial year.
  3. Observe that gross salary shown does not align with expected values.

Cause

Custom rule which applicable for generation of wage type /616 (for example, rule ZN43) was incorrectly configured. The logic was applied for indicator 'Y' whereas it should be written for indicator 'N', preventing correct inclusion of arrears (/616) into /416 under section 89 handling.

Resolution

  1. Go to PE02 t-code.
  2. Pass your custom rule which has been copied from standard rule IN43.
  3. Click on 'Change' button.
  4. Adjust the custom rule so that the logic for /416 is applicable for indicator 'N' (not for 'Y') in line with standard IN43.

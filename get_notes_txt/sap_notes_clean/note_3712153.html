Symptom

  • How to mass delete of grounded documents in SuccessFactors?
  • Instead of deleting the grounded documents from Manage Document Grounding page manually, how to mass delete multiple grounded documents?

Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.  

Environment

  • SAP SuccessFactors HCM Suite
  • Joule

Resolution

Example:

In SuccessFactors Instance > Manage Document Grounding, there has 9 documents grounded, and user wants to mass delete the 3 of the documents, for other grounded documents, remain the configurations.

Step1: Export VectorContent Generic Object's MDF Data

1)Go to Import and Export Data > "Select Generic Object" choose "VectorContent" 

2) Keep selections as below, click "Export" button, download the the MDF data Zip file from Schedule Job Manager:

Step2: Edit VectorContent.csv file

1) Extract the downloaded zip file > Only open file: VectorContent.csv, you will see all existing 

Kindly Note: Please do not make any changes in any other files in this zip file, to avoid data & configuration impact.

2) Verify the 3 grounded documents which you want to remove the grounding, keep them in the file, Remove other rows which you wish to keep the metadata of the grounded documents.

3) Add "Delete" on the [OPERATOR] column for the files for which you want to remove the grounding and save the file. For example, I want to delete only 3 files so I just have those 3 files in the csv file with "Delete" Operator:



Step3: Validate and Import VectorContent.csv file to mass delete the grounded documents

1) Validate updated VectorContent.csv by go to Import and Export Data > Import Data > Select "VectorContent" > Import the file saved in Step2 > click validate button.

In this you can see 3 are processed and 3 are passed the checks.

2) Once validation is successful, go back to Import and Export Data > Import Data > Select "VectorContent" > Import the file saved in Step2 > click Import button, to delete the 3 grounded documents.

Once the job is completed, we are successfully able to delete the grounded documents.

Note: you can tag those deleted documents again from the Add metadata option after the pipeline gets triggered once again for the Sharepoint site you integrated with SuccessFactors instance.

Symptom

  • Incorrect behavior after choose Stay on the page when clicking close button of browser in Web Intelligence Editing page.
  • After choosing "Stay on the page" option in the dialog, though browser window remains, the session seems to be deleted abnormally.

Other Terms

Webi, session, print, export, edit, stay, leave, page, close, exit, stay on the page, IE 11

Reason and Prerequisites

Environment:

  • SAP BusinessObjects Business Intelligence Platform BI4.2 SP03

Step to reproduce:

  • Scenario 1, this scenario is reproducible in All kinds of browser.
  1. Log on to BI launch pad in Internet explorer 11.
  2. Edit a Web Intelligence document in Web Intelligence HTML interface.
  3. Do some changes like adding a cell, change the table size.
  4. Click the close button of Internet explorer browser.
  5. An warning message comes to let user select to really leave the page or stay on the page, here choose stay on the page.
  6. Wait for 1-2 minutes, click Print button. Notice nothing happens.
  7. Click the Export button, error occurs as Invalid session. Please close your browser and log on again. (WIH 00013)

    If do the above workflow in Web Intelligence Applet interface mode, after click Export button, similar session error occurs as:
    com.businessobjects.sdk.core.CoreException: com.crystaldecisions.sdk.exception.SDKException$OCAFramework: Unable to reconnect to the CMS <CMS:PORT>. The session has been logged off or has expired. (FWM 01002)
  • Scenario 2, this scenario is only reproducible in IE (Internet Explorer) browser.
  1. Log on to BI launch pad in Internet explorer 11.
  2. Edit a Web Intelligence document in Web Intelligence HTML interface.
  3. Do some changes like adding a cell, change the table size.
  4. Click the close button of Internet explorer browser.
  5. An warning message comes to let user select to really leave the page or stay on the page, here choose stay on the page.
  6. Different as the scenario 1, here immediately click Print button.  Notice that the warning message comes again to let user select to really leave the page or stay on the page even here user didn't select to close the page.
  7. In the 6th step, immediately click Export button, choose format and click OK to perform export. Notice that the warning message comes again to let user select to really leave the page or stay on the page even here user didn't select to close the page.

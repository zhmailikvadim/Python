Symptom

When executing the PC00_M37_EFD_EE_SUMM transaction, the following groups are not generated for event S-5002:

  • infoProcJudRub
  • infoRRA
  • infoPgtoExt
  • infoIRComple

This KBA explains why this behavior is expected.

Environment

  • HR renewal

  • SAP S/4HANA

  • SAP ERP

  • SAP SuccessFactors Employee Central Payroll

Reproducing the Issue

  1. Run the PC00_M37_EFD_EE_SUMM transaction for the S-5002 event.
  2. Obtain the XML file returned by eSocial.
  3. Compare the results of the PC00_M37_EFD_EE_SUMM transaction with the detailed information in the XML file.

Resolution

According to the Regulatory Change Manager tool, some groups are not generated in the report.

Regulatory Change Information: Changes: eSocial layout S-1.2 - standard events/framework  

  • S-5002 - withheld income tax
    • The report eSocial: Visualizador de Eventos Totalizadores de empregado (RPU_PAYBR_EFD_EE_SUMM_EVENTS) will be changed work with this new version except for the groups infoProcJudRub, infoRRA, infoPgtoExt and infoIRComplem.

The groups mentioned above are non-standard generated and are already included in event S-1210. Therefore, it is not necessary to include them in event S-5002 as well.

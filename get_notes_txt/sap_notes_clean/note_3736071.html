Symptom

  • Error "Object cannot be created again" happened during implementing Note 3687455

CUS1 OHADOE_ERH_AZ_010: Object cannot be created again. Object ENHS HRPAYDE_IND_EVAL_B_ERH_AZ missing

SHI3 E542F6681897D21194C20060B0686D73: Object cannot be created again. Object CUS0 SIMGOHADSVD45 missing

Environment

  • SAP ERP
  • SAP NetWeaver

Reproducing the Issue

Use transaction SNOTE and attempt to implement note 3687455.

Resolution

  1. Make sure note 3696168 & 3491432 have been implemented if applicable to the system.

SAP NOTE 3696168 - Incorrect validation during the downport of SHI3 objects via SNOTE/SCWB
SAP NOTE 3491432 - SNOTE: Error "Object cannot be created again..." when object creation is in same Note

    2. Re-implement note 3687455. When error "Object cannot be created again" happened again, select the option “No” to continue. 

   

    3. Note 3687455 will end with status “Incompletely implemented,” immediately re-implement note 3687455 again.  The second run should complete without above errors.

    4. Confirm that note 3687455 shows status “Completely implemented.”

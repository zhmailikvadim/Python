Symptom

When there are multiple positions for the same metering location and device classification with billing calculation rule 'Additional', the system randomly selects the position for the total yearly amount instead of using a consistent logic.
This issue is relevant for the market role: MOS.

Other Terms

MOS Billing, Meter Operator Contract, Total Yearly Amount, Billing Rule, Additional, DET_TOTAL_YEARLY_AMOUNT, DET_TOTAL_YEARLY_AMOUNT_DSO, /MOSB/CL_DEF_BADI_CONTRACT_DET, MOSB, metering location, device classification

Reason and Prerequisites

Program error when handling multiple positions with the same metering location and device classification.

Solution

Implement the corrections of this SAP Note or import the relevant support package.
After implementing this SAP Note, the system selects the position with the maximum yearly amount as the position relevant to the total yearly amount when there are multiple positions for the same metering location and device classification with billing calculation rule 'Additional'.
You need to discard the old time slice with the wrong total yearly amount and create a new time slice.
You can verify the fix by checking that the system consistently selects the position with the maximum yearly amount as the position relevant to the total yearly amount when multiple billing positions with billing calculation rule "Additional" exist for the same metering location and device classification.

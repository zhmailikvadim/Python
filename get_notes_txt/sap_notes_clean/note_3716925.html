Symptom

When user selects a Contract in Mass Changes to Purchase Contracts Fiori application in the Purchase Contract Headers tab, and proceeds downloading the contract information, in the Worksheet 1 only the selected contract is appearing, compared with Worksheets 2 and 3, where the whole list of contract items are appearing, not only the ones for the selected contract

Environment

  • SAP Fiori
  • S/4HANA 2023
  • S/4HANA 2022

Reproducing the Issue

  1. Go to Mass Changes to Purchase Contracts fiori app
  2. From the display list from tab Purchase Contract Headers, select one contract
  3. Click on DOWNLOAD button

Cause

  • Standard behavior of application
  • The user has to select on both tabs: Purchase Contract Headers and Purchase Contract Items
  • This is because the Worksheet 1 includes Purchase Contract Header information, and the Worksheets 2 and 3 contains information of the Items

Resolution

When a user wants to download information for only selected Contracts, the Contract has to be selected in Purchase Contracts Header tab, and also the Items from that contract have to be selected in Purchase Contract Items tab.

This can be accomplished too by filtering the result in the application filters using the concrete Contract that is in need to be downloaded for Offline update.

Symptom

The user starts the transaction "Create/Edit/Display RM Account" and releases the account by choosing "Basic System, Calculate, Release". The system issues the error message "Customizing error (default settlement) between CoCd <BUKRS> and CoCd <BUKRS>".

Other Terms

/MSG/H_FINA1

/MSG/R_A_ABR2

309

/MSG/RBC_ABR

Reason and Prerequisites

Prerequisite:
The account is created for a participation with a connecting treaty. The checked company codes use different current account systems.

Reason:
The existing check is too restrictive. In Customizing for Basic System under Edit Defaults for Accounting, the system only checks whether the following fields match for the relevant company codes:

  • Current Account System
  • Intercompany Settlement
  • Note on Release of Account

Solution

History:
February 17, 2026: New


Troubleshooting Description:
The existing check is adjusted. It does not generate an error message for the following combinations:

Variant 1:

  • The "Current Account System" field is different
  • The "Intercompany Settlement" indicator is NOT set for both company codes.
  • The "Note on Account Release" indicator is identical.

Variant 2:

  • The "Current Account System" field is identical
  • The "Intercompany Settlement" indicator is identical.
  • The "Note on Account Release" indicator is identical.

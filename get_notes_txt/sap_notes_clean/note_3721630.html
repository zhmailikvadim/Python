Symptom

When creating an IDoc in the SAP ERP to integrate with the SAP Integration Suite, managed gateway for spend management and SAP Business Network, the IDOC fails with the error:

No Idoc saved in target system (SOAP-HTTP)

An internal error occurred. For error details please check MPL ID

Cause

This issue typically occurs when the IDoc contains unsupported, invalid, or illegal characters, which causes the target system to reject the message. Generally, this issue happens when the SAP ERP integrates with a SAP BTP, private cloud edition tenant, before reaching the SAP Integration Suite, managed gateway for spend management and SAP Business Network.

Resolution

To identify the exact location of the invalid character, the following steps should be performed:

  1. Download the IDoc as an XML: You can extract the IDoc in XML format using the ABAP program RCOD_DOWNLOAD_IDOC_AS_XML in the SAP ERP.
  2. Analyze the IDoc field values:
    1. Go to transaction IDOC.
    2. Select the radio button Analyse IDOC field values and execute.
    3. Enter the affected IDoc number and select the checkbox Also evaluate empty fields.
    4. Click on Output in internal form and execute.
    5. Export the IDoc data into the required format and check for special or junk characters.
    6. Identify and correct invalid characters.
  3. Open the extracted XML in a good XML editor: It will highlight invalid characters and their locations. These characters often appear in text or comment fields, especially when data is copied from external sources like PDF files. 

Once identified, the data can be corrected or manually edited in the IDoc using WE19, and the IDoc can then be reprocessed.

See Also

For more information about the steps to download the IDOC in XML format, refer to the SAP Note 2386785 - Download IDoc payload in XML format.

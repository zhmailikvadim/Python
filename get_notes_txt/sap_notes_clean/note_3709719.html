Symptom

  • When validating an RFC SNC connection in SAP Datasphere for OAuth 2.0 with Principal Propagation, the following error can be seen:
    ABAP connector(Axino) ABAP connection check FAILED: request failed: rc=1, msg="Message: Name or password is incorrect (repeat logon)\nCode: RFC_LOGON_FAILURE"
  • Along with the error above, in the SM50 logs, the following error can be seen:
    ACL entry for SNC-secured server-server link is missing 

"Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental."

Environment

  • SAP Datasphere
  • SAP Connectivity service,
  • SAP NetWeaver Application Server for ABAP

Cause

The SAP Cloud Connector SNC entry in transaction SNC0 is misconfigured.

Resolution

  • Check and correct the SAP Cloud Connector SNC entry in transaction SNC0:
    • The System ID can be any value of your choice, like "SCC".
    • The SNC name should be the same SNC name configured in the "SAP Cloud Connector -> Configuration -> On-Premises -> SNC" section.
    • Do not forget the p: at the beginning of the SNC name. It should follow this format: p:CN=<Your_CN>. If you have more attributes in the SNC name of the SCC aside from the CN, separate them by a comma.
    • The checkboxes for "Entry for RFC activated" and "Entry for certificate activated" should be checked.

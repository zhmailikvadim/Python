Symptom

In SAP NetWeaver Administrator, when you look at the Certificates and Keys: Key Storage plugin, the system shows a yellow warning icon if a certificate is close to expiring.
- This warning is hard‑coded to appear during the last 3 months of the certificate’s life.
- You cannot change this 3‑month threshold.

What this means for you:
- If you use certificates that last longer than 3 months, the warning will only show near the end.
- But if you use certificates that last exactly 3 months (or shorter), the warning will show for the entire time the certificate is valid.

Example: If you install a 3‑month certificate, you will see the yellow warning from day one until it expires.

Environment

Netweaver Application server for java 7.5

Reproducing the Issue

Go to SAP NetWeaver Administrator → Key Storage, create a 3‑month certificate, and observe the yellow warning icon appearing immediately and persisting throughout its validity

Resolution

Update SAP NW AS Java to a release or support package (SP) where the problem is fixed. See the "Validity" and "Support Packages & Patches" sections of the SAP Note 3415653 - Configurable Warning for Keystore Entry Expiration in SAP NW Administrator for details and available patches.
To avoid system incompatibilities, check SAP Note 1974464 - Information on SCA Dependency Analysis for Java download objects.

A new Keystore service property, EntryExpirationWarningWeeks, lets you set how long the warning should be shown.

Default value: 12 weeks (3 months)

Allowed range: 1 to 48 weeks

After changing and saving the value, restart the cluster for changes to take effect.

Tooltips now show more details:

If more than 2 weeks remain, they show weeks left.

If 2 weeks or fewer remain, they show days left until expiry.

To change the value of EntryExpirationWarningWeeks, go to configtool → template-services → keystore. There you can set a new custom value as per your need.

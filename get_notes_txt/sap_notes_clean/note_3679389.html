Symptom

You upload an attachment in Fiori while the server is down and expect the system to display an appropriate error message indicating that the upload has failed. However, instead of showing an upload failure message, the system incorrectly displays messages stating that the price records have been saved, the records have been created, and the agreement request has been saved.

This note corrects the behavior by ensuring that an appropriate “Upload failed” error message is displayed when the server is unavailable during attachment upload.

Other Terms

FIORI, /IRM/GDRM, /IRM/GDR_UIATTACHMENT_PROCESS, /IRM/LIPAGRMF0A, /IRM/IPAGRM, ATTACHMENTS_UI_PROCESS, /IRM/LGDRMF0A, ACTIVE_DOCUMENT_UPLOAD, /IRM/LGDRMF0S, STORED_DOCUMENT_UPLOAD, /IRM/GDR_INFOCUS_SAVE, /IRM/GDR_UIATTACHMENT_PROCESS

Reason and Prerequisites

This is caused by Program error.

Solution

Please implement the correction instructions. These corrections are only valid if you have applied the following Support Packs.

Release V/4 – 2022 V2, 2023, 2023 V2 , 2023 V3 Base Package or Support Pack 1.

1. Perform the manual pre-implementation steps that are described below
2. Use transaction SNOTE to implement the correction instructions specified below and activate the affected program object

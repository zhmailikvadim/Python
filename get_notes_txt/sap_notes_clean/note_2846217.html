Symptom

A 208 error, "tempdb..tablename not found", may be reported when a temporary table, which is referenced by a stored procedure created by a user who is aliased to dbo, is recreated.

Environment

  • SAP Adaptive Server Enterprise (ASE)
  • SAP Adaptive Server Enterprise Cluster Edition (ASE CE)   

Cause


The issue has been logged under SAP CR 812134. Its description reads, "A 208 error, "tempdb..tablename not found", may be reported when a temporary table, which is referenced by a stored procedure created by a user who is aliased to dbo, is recreated."

Resolution

Upgrade ASE to a release where SAP CR 812134 is fixed: SAP ASE 16.0 SP03 PL04 or higher

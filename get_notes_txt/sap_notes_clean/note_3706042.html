Symptom

  • An error occurs when updating a work package end date through the whitelisted API CPD/SC_PROJ_ENGMT_CREATE_UPD_SRV. The error message displayed is: "Resource not found for segment 'WorkPackage'".
  • When the same operation is performed using the Fiori app "Plan Customer Projects," a popup screen appears with two options: "Retain Total Efforts" and "Retain Efforts for the Overlapping Months."
  • Selecting "Retain Total Efforts" results in the same error as the API, while selecting "Retain Efforts for the Overlapping Months" successfully saves the update.

Environment

SAP S/4HANA Cloud Public Edition

Reproducing the Issue

  1. Open the Plan Customer Projects App in the Dev system.
  2. Select a project.
  3. Select the work package and edit the start date to a different end date.
  4. Save the changes. A popup with two options appears:
  • Option 1: Retain Total Efforts (results in an error).
  • Option 2: Retain Efforts for the Overlapping Months (successfully saves the update).

Cause

'Retain Efforts for the Overlapping Months' is not available for CPD/SC_PROJ_ENGMT_CREATE_UPD_SRV API.

Resolution

  1. "Retain Efforts for the Overlapping Months" is only available for API_ENTERPRISE_PROJECT_SRV.
  2. Refer to the documentation for API_ENTERPRISE_PROJECT_SRV to understand its capabilities: Change Period Distribution.

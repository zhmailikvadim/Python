Symptom

When attempting to publish a change project into a target tenant, the following error message may appear:

  • Invalid sequence. Ensure that ABC is published before this project.

It is important to note that "ABC" represents an example of a Change Project ID and could be any relevant identifier.

Environment

  • SAP Sales Cloud Version 2
  • SAP Service Cloud Version 2

Reproducing the Issue

  1. Click on the User Icon.
  2. Navigate to Settings > All Settings.
  3. Proceed to the Change Projects screen.
  4. Search for the specific change project. For instance, you might search for "ABC"
  5. Click on the Publish button.
  6. The following message gets raised:
  • Invalid sequence. Ensure that ABC is published before this project.

Cause

The sequence enforced by the system is determined by the publishing time in the source system. This mechanism ensures that older changes are published before newer ones, thereby preventing any potential overrides. The change project functionality does not calculate logical dependencies between projects. Instead, the sequence is purely based on the chronological order in which the changes were created or modified.

Resolution

To determine the correct publishing sequence, you should refer to the "Previous" change project attribute within the list of change projects. This attribute will provide guidance on the proper order in which the projects should be published.

By ensuring that you are following the correct order of publishing, these errors should not occur.

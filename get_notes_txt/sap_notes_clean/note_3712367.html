Symptom

Symptom

  • You are using the Manage Alerts app in SAP Customer Activity Repository applications bundle 5.0 SPS01, 5.0 SPS02, 5.0 SPS03, 5.0 SPS04, 5.0 SPS05, 5.0 SPS06, 5.0 SPS07, 5.0 SPS08..
  • ROPI (Review Order Plan Items) button remains enabled when multiple alert details are selected, allowing users to trigger ROPI action for multiple alerts simultaneously, which may lead to unexpected behavior or errors.

Other Terms

SAP Customer Activity Repository applications bundle, CARAB, Manage Alerts, Object Page, ROPI (Review Order Plan Items)

Reason and Prerequisites

Program error.

Solution

Implement the intruction attached to this note in your frontend system.

Example Scenario:

An employee XYZ is hired on 01.07.2025 with monthly payroll frequency.

Overtime wagetype 9001, which is configured with Cumulation Type U, is used to record the employee’s overtime payments.

In payroll period 01.2025, the employee has an overtime payment with 10 hours and no amount, whereas in payroll period 02.2025, the employee has an overtime payment of 100 AUD with 10 hours.
After running payroll for both periods, the following CRT details are displayed.

Payroll Period 01.2025

Cum.Typ Wage Type  Number Amount
     U    9001     10     0

Payroll Period 02.2025

Cum.Typ Wage Type  Number Amount
     U     9001     10     0
     U    9001     10   100

In payroll period 02.2025, the entries for wage type 9001 are not merged, which is incorrect.

Other Terms

CRT, QACRT, ACRT

Reason and Prerequisites

Reason for Versioning:

Version 

 Reason for Version Creation 

     01

  General Release for Customers

Solution

A correction has been implemented to address the issue stated in symptom above. After the code changes, in payroll period 02.2025, the entries for wagetype 9001 are merged in CRT, as shown below.

Cum.Typ Wage Type  Number Amount
     U    9001     20     100

 To get the code corrections immediately,

  • Apply the correction instructions attached to this SAP Note.

 OR

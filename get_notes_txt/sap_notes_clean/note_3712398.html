Symptom


1. Field 'Ultimate Destination' (/BEKA/UDU) on ASD Reference tab of Purchase Order is display-only and cannot be edited before sending SA1 message. This might be required for some scenarios.

2. The value of the 'Part Number (PNR)' field is swapped with the value of the 'ManufactureR (MFC)' field in the 'Manage Quotation Requests' app.

3. When attempting to create a Purchase Requisition by clicking the 'Create Purchase Requisition' button in the app 'Manage Quotation Requests', the system returns the error message E101(06) and the Purchase Requisition is not created.

4. When sending multiple UNH segments in a single message, if the first segment has an error, it will get forwarded to the second segment, triggering an error on it.

5. Answering messages don't get displayed as a change message but get answered with a positive message automatically .

6.Quantity update in SAP document doesn´t work for SD.

7.Wrong recipient in ERRNLT

8.SF1-messages with QTY:0 error

 

Solution


1. Make '/BEKA/UDU' field (Ultimate Destination) on ASD Reference tab (screen 0200, '/BEKA/SAPLA_PO_SCREEN_ENH') editable via BAdI hook in 'change_screen' module before 'MODIFY SCREEN', replacing current display-only behavior.

2.The fields descriptions have been corrected in the Metadata Extension, and from now on, they will be accurate.

3. The supplier value passed to BAPI_PR_CREATE was adjusted to include leading zeros, and from now on this issue will no longer occur.

4. Added CLEAR: ct_return in /BEKA/CL_A_AIF_INBOUND_MAP=>PROCESS_INBOUND method, so the error from previous UNH segment wouldn't be forwarded to the next one.

5. Removed the part that was setting to maint from AIF method /BEKA/IF_A_AIF_SPECIFIC~MAP_MESSAGE.

6. Modified the quanitites of table vbep from /BEKA/SD_QT_FIELDS_SET , /BEKA/CL_A_ACT_CRT_SD_QUOT~CREATE_STANDARD_SALESQUOTATION , /BEKA/CL_IM_A_SD_SALES~UPDATE_QT_FIELDS , /BEKA/CL_IM_A_SD_SALES~IF_EX_BADI_SD_SALES~SAVE_DOCUMENT_PREPARE

7. In method /BEKA/IF_A_ACTION_STEP~PROCESS class /BEKA/CL_A_ACT_SEND_ERRNLT:

Line is added 'cs_output_message-asd_s2000m_outb-agency = ms_operation-agency.'

8. A check is added in class: /BEKA/CL_A_AIF_INBOUND_MAP  method: OVERWRITE_PROPERTY that for SF1 message the QTY:0 should not return error.

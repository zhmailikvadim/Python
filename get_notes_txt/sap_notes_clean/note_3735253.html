  • In database alert log the database shutdown and startup with cluster hostname takes more than 3 minutes which is larger than 180 seconds.

Environment

  • SAP ERP
  • Oracle
  • Windows Server 2019 MSCS (Failover Clustering)
  • Oracle Fail Safe for Windows
  • Oracle Database 19c

Reproducing the Issue

  1. Run the AddOracleDbToOracleClusterGroup.ps1 script to add the database Resource into an Oracle Fail Safe (OFS) group on Windows Failover Cluster.
  2. The script attempts to bring the database resource online and reaches the default pending timeout (180 seconds), returning FS-10727/FS-10012/FS-10782/FS-10890.
  3. In the database alert log, the database shutdown and startup with cluster hostname takes more than 3 minutes which is larger than 180 seconds.

Cause

The database stop/start duration exceeds the cluster resource pending timeout (default 180 seconds), leading to a timeout while bringing the resource online. The long restart time was influenced by the database memory configuration.

Resolution

Workaround: Reduce database memory so that shutdown and startup complete within the cluster pending timeout (observed resolution: restart duration decreased below 180 seconds and the add succeeded).

Increase the Windows Failover Cluster PendingTimeout for the database resource to exceed the measured restart time (for example, 300000 ms):

  1. PowerShell: Get-ClusterResource -Name RESOURCE_NAME | Set-ClusterParameter PendingTimeout 300000
  2. Legacy CLI(cmd): cluster.exe res "RESOURCE_NAME" /prop PendingTimeout=300000
  3. Remove the database from cluster group and Re-run the add operation. Verify that it completes within the configured timeout.

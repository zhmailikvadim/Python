Symptom

When customers send reorder JIS Calls, the external sequence number is provided as ‘0000’ to indicate that the call is a reorder. In customer systems, if reorder JIS Calls are sent with a different external JIT Call number, the external sequence number remains initial instead of displaying ‘0000’. As a result, reorder JIS Calls are not clearly distinguishable in the system based on the external sequence number.

Other Terms

Next Generation Just-In-Time processing, NJIT, Manage Packing Group, External Sequence Number

Reason and Prerequisites

 Program error

Solution

Implement the correction instructions included in this SAP Note or import the relevant Support Package to activate the corrected system behavior.

With this update, the system displays the external sequence number as ‘0000’ for reorder JIS Calls when the external sequence number is received as '0000'. This ensures consistent identification of reorder JIS Calls, regardless of the external JIT Call number used

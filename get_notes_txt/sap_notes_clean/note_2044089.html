Symptom

You choose the pushbutton for generating the IBAN from bank details to maintain an IBAN in credit processing (transaction FPTCPRO or FPCRPO). On the IBAN details screen, the data is not ready for input. If you choose "Change IBAN", a dialog box is displayed that asks whether you want to change the IBAN for the existing bank details or you want the entry of a new IBAN to replace the bank details completely. You cannot make a selection because the second option is fixed. After you confirm the dialog box, you navigate back to the credit processing screen.

Other Terms

IBAN, converter, IBAN_REPLACE, MAINTAIN_IBAN_DIAL, IBMA, SAPLIBMA

Reason and Prerequisites

There is a program error. This occurs if you implement the functions from SAP Note 1953552 beforehand.

Solution

Implement this SAP Note. If there is already an IBAN for the classic bank data (technically: there is an entry in the table TIBAN), then this IBAN is only displayed and you can no longer switch to change mode. When you confirm the dialog box, the stored IBAN is transferred to credit processing. If there is no IBAN yet, then an IBAN can be proposed and transferred as usual. The dialog box described above is no longer displayed.

Comment: The table TIBAN is generally no longer updated during credit processing due to the changes from SAP Note 1841002

Symptom

  • An error occurs when attempting to copy a Knowledge Management (KM) document URL into a specific field in the Enterprise Portal using Microsoft Edge, Firefox, and Chrome browsers.
  • In Chrome and Edge, the URL is not copied to the clipboard, preventing it from being pasted into the browser.
  • In Firefox, the following error message is displayed: "Firefox security restriction! Make sure the Firefox parameter signed.applets.codebase_principal_support in about:config is true."
  • The issue is browser-specific, with functionality working in Internet Explorer but not in newer browsers.
  • The problem persists despite clearing portal and browser caches.

Environment

  • SAP NetWeaver Enterprise Portal - Release Independent 
  • Knowledge Management 

Reproducing the Issue

  1. Log in to the Enterprise Portal.
  2. Navigate to Content Administrator -> KM content.
  3. Locate a document and click on the Context menu.
  4. Select "Copy URL to clipboard".
  5. Attempt to paste the URL; observe that the URL is not copied to the clipboard.

Cause

The "Copy URL to Clipboard" functionality relies on legacy browser clipboard access methods that are no longer supported in modern browsers such as Google Chrome, Microsoft Edge, and Mozilla Firefox due to enforced security restrictions.

Resolution

  1. Continue using Internet Explorer Mode in Edge where clipboard functionality is required.
  2. Implement a manual workaround, such as copying the URL from the document properties directly.

Symptom

  • When multiple Just-In-Time Processing (JIT) calls are selected in JITK transaction for background delivery creation, if at least one selected line has no stock or zero delivery quantity, no outbound delivery is created for any of the selected lines.
  • Lines with valid delivery quantity are not processed when one selected line is ineligible.

Environment

  • Product: SAP S/4HANA

Reproducing the Issue

1. Execute transaction JITK.
2. Select multiple JIT call lines.
3. Ensure one line is not eligible (e.g., zero delivery quantity) and others are valid.
4. Execute delivery creation (e.g., background).
5. Observe that no delivery is created for any of the selected lines

Cause

  • In JITK, delivery creation calls RV_DELIVERY_CREATE via JIT03_CREATE_DELIVERY_II and JIT06_CREATE_DELIVERY, and the application processes all selected items collectively.
  • If any selected item is not eligible, the collective process stops and no deliveries are created for any items. This is standard system behavior.

Resolution


Note that this is standard system behavior in JITK and cannot be changed through configuration.

1.Implement a custom enhancement using user exit EXIT_SAPLJIT03_015 to process only valid items.
2.In the exit, before delivery creation, remove non-relevant entries (for example, items with invalid or zero delivery quantity) from internal table PRE_SET_LT.
3.Ensure consistency by also removing the corresponding entries from the related internal tables passed in the exit.

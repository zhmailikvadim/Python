Symptom

Usage of Audit Log Space Allocation Settings available in Provisioning 

Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.

Environment

SAP SuccessFactors HCM Suite

Reproducing the Issue

  1. Access  Provisioning.
  2. Navigate to Company Settings, where you can find the Audit Log Space Allocation setting.

Cause

Obsolete setting in provisioning 

Resolution

The Audit Log Space Allocation setting in Provisioning no longer has a practical impact under the current system design.

Background

Before b2205:

  • The default storage limit for audit reports (Change Audit + Read Audit) was 1 GB.
  • If the storage exceeded 1 GB, customers could request an increase (e.g., up to 5 GB) via the Audit Log Space Allocation setting in Provisioning.

After b2205:

  • The audit log storage limit was increased to 10 GB by default for all customers.
  • This 10 GB limit is now the minimum effective storage, regardless of any previous Provisioning settings.

 

Current Behavior

  • The Audit Log Space Allocation setting still exists in Provisioning.
  • However, modifying this setting does not impact or change the actual audit log storage limit.
  • The system continues to enforce a minimum storage limit of 10 GB.
  • The Audit Log Space Allocation setting is not technically deprecated, but it is no longer meaningful or effective in influencing audit log storage behavior in the current design.
  • Customers do not need to adjust this setting, as the system-managed storage limit (10 GB) applies automatically.

NOTE: There are plans to deprecate and remove this functionality from provisioning in a future roadmap update. This KBA will be revised once the product team confirms the timelines.

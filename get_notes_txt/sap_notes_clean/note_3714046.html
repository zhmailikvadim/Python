Symptom

When activating a best practices solution you face these error messages below:

Installation data upload failed, see longtext

Cannot retrieve metadata for table <table>, it is not part of cust obj.
An exception was raised
Missing maintenance dialog for customizing object <object>

-

When you expand the error message for more details you see the message number: /SMB/DEFINITION045

Environment

S4/HANA release independent

Cause

Wrong Best Practice version is being used.

Resolution

Before activating the new and correct, the wrong version has to be deleted completely first, please check this KBA below:
3721436 - How to delete a wrong Best Practice solution completely before the complete activation?

-

This is just an example for S4/HANA 2023, the naming convention for another versions might be different.

This error message usually comes up when you are using the wrong Best Practices version.

Please double check the version of Best Practice solution is being used for your S4/HANA. For example, for S4/HANA 2023 you must use the Best Practices content for 2023.

But S4/HANA 2023 has four FPS stacks that should be considered: FPS00, FPS01, FPS02 and FPS03
For these four FPS stack, they have two different Best Practices solution versions:

BP_OP_ENTPR_S4HANA2023_<country code>V10 and BP_OP_ENTPR_S4HANA2023_<country code>V11

So, basically the V10 is valid for FPS00 and FPS01 while V11 is valid for FPS02 and FPS03

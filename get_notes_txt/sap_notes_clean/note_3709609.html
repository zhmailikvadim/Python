Symptom

Implement this SAP Note that integrates the BAdI definition FI_PL_KSEF_INVOICE into the Starter Solution FIPL_SAFT (JPK_KR_PD).

The implemented logic for searching for the KSeF number for a given customer invoice is as follows:

The report will try to get the number stored in the ODN number. The report primarily searches for KSeF numbers stored using the official document numbering (ODN) framework.

Other Terms

Poland, PL, Poland SAFT Report, PL SAFT Report, JPK_KR_PD, KSeF BAdI, ODN, Starter Solution, FIPL_SAFT, FI_PL_KSEF_INVOICE

Reason and Prerequisites

This note is relevant for Polish SAFT Starter Solution only.
You are using Polish SAF-T JPK_KR_PD report, transaction FIPL_SAFT.
This note aims to implement the KSeF BAdI integration specifically for the JPK_KR_PD report within Starter Solution.
SAP Note 3689699 must be implemented as a prerequisite.

Solution

Enhancement spot: FI_PL_KSEF_INVOICE
There is one method: IF_EX_FI_PL_KSEF_INVOICE—>GET_KSEF_INVOICE
There will be no standard solution provided, therefore the customer is to implement the retrieval logic if needed.
This is a fallback logic in case ODN did not contain the KSeF ODN —> if ODN is empty —> BAdI.

If the number is not stored in ODN, and the customer implements BAdI FI_PL_KSEF_INVOICE, the BAdI method GET_KSEF_INVOICE will be called. This method will allow the generation of the KSeF number based on the customer logic.

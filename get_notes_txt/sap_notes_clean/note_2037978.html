Symptom

Short dump UNCAUGHT_EXCEPTION CX_CRM_GENIL_MODEL_ERROR when defining a FPM UIBB that uses a custom defined or enhanced MDG entity.

Other Terms

CL_CRM_GENIL_OBJ_MODEL, IF_GENIL_OBJ_MODEL, CL_MDG_BS_GUIBB_FORM, CL_MDG_BS_GUIBB_LIST,

Reason and Prerequisites

Prerequisite 1: The entity contains a customer defined attribute of data type MANDT

Prerequisite 2: You are using the new BOL/GenIL based MDG UI.

Reason: The MDG generic text assistant (class CL_USMD_GENERIC_GENIL_TEXT) tries to generate a text attribute for the client attribute but this fails due to missing coding.

Solution

1. Apply the attached program correction

2. Run program USMD_TXT_FLD_MAP_DELETE (with transaction SE38) and delete buffer table USMD_TXT_FLD_MAP for the MDG data model you are using. With the attached program correction the buffer table will be rebuilt automatically with the correct entries.

Symptom

The Configuration <Extractor Detail> extractors are failing with the error:

  • Authentication did not succeed.

The following picture shows an exanple of affected extractor:

Environment

  • Solution Manager 7.10 and 7.20
  • SAP Solution Manager Diagnostics

Reproducing the Issue

The Configuration Extractors (CCDB extractors) are executed automatically once per day. The extractors can be found in the CCDB as follows:

  1. Transaction CCDB;
  2. Technical Systems tab;
  3. Select affected/target system;
  4. Check column 'Failed Extractors'

Cause

There is a problem with the SM_COLL user, which is the user used to collect the data.

Resolution

The error Authentication did not succeed indicates an authentication issue with user SM_COLL_<Solman_SID> which is used to collect the information from the JAVA managed system. The steps to fix the issue are:

1) Go to step 'Maintain Users' of the Managed System Configuration;
2) Update the user's password and roles using the Managed System Configuration step.

Ensure thatthe user SM_COLL_<Solman_SID> exists with the recommended roles and that its password is correct.

NOTE: Do NOT use the flag 'Accept manually created user without checking role assignments' to simply turn the user status green. When this flag is selected the user is not checked at all and it might lead to an erroneous information about the user status. The flag must be used only if one is sure that the user/password provided are indeed correct.

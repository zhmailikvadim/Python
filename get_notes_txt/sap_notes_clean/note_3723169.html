Symptom

After you implement Note 3526974 and run forecasting in SAP Customer Activity Repository applications bundle 5.0, 5.0 FPS01, 5.0 FPS02, 5.0 FPS03, 5.0 FPS04, 5.0 SPS05, 5.0 SPS06, 5.0 SPS07, 5.0 SPS08, 6.0 SPS01, or 6.0 FPS02, the system creates duplicate records. These duplicate records have a TSTMP_FR (Timestamp From) value that ends in "120000" in the table /DMF/UFC_TS.

Other Terms

SAP Customer Activity Repository applications bundle 5.0 FPS02, Unified Demand Forecast, Modeling, Forecasting, CARAB 5.0, CARAB 5.0 FPS01, CARAB 5.0 FPS02, CARAB 5.0 FPS03, CARAB 5.0 FPS04, CARAB 5.0. SPS05, CARAB 5.0 SPS06, CAR, CARAB, SAP FIORI FOR SAP CARAB, SAP CARAB 5.0 FPS02, SAP CARAB 5.0 FPS03, SAP CARAB 5.0 FPS04, SAP CARAB 5.0 SPS05, SAP CARAB 5.0 SPS06, Modeling End Date, Forecast  Start Date, Demand End Date

Reason and Prerequisites

Side-effect of Note 3526974

Solution

  1. Open the transaction SE38 and create a custom report in your back-end system.
  2. Copy the attached code to the report.
  3. Run the report after selecting Test Mode and verify the number of duplicate records.
  4. Schedule this report to run as a batch job during off-peak hours without selecting Test Mode.

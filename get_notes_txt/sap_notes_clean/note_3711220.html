Symptom

When attempting to create a sales order using the app "Import Sales Order", the creation fails with the error message as "Ship-to party 12345678 not defined for sales area AB12 C3 D4", even though the same sales area is correctly maintained in the master data of ship-to party 12345678.

Environment

SAP S/4HANA Cloud Public Edition

Reproducing the Issue

  1. Open the app "Import Sales Order".
  2. Upload a file with necessary details about the sales order and hit "Import".
  3. The creation fails with error message "Ship-to party 12345678 not defined for sales area AB12 C3 D4".

Cause

In sales document type "OR", the indicator "Propose Item Division from Material" is enabled. Due to this configuration, the division maintained in the sales order header is ignored, and the division defined in the material master is considered for each item.

Resolution

Follow either of the given solutions below:

  1. Option 1:
    1. Navigate to the configuration activity "Configure Sales Document Types (102495)"
    2. Open the document type "OR" and go to the "General Settings" tab.
    3. Disable the "Propose Item Division from Material" option. This adjustment will allow the system to accept the Division from the uploaded file and prevent the Material's Division from being enforced on the document.
  2. Option 2:
    1. Access the Customer Master Data for Ship-to Party 12345678.
    2. Add a new Sales Area with the combination of Sales Organization: AB12, Distribution Channel: C3 and Division: D4

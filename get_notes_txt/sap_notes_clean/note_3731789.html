Symptom

  • Not able to create SMTP node in SCON transaction.
  • the error "Node already exists" is displayed 

Reproducing the Issue

  1. Open transaction SCON.
  2. Attempt to create an SMTP node.
  3. Observe that the system displays the error "Node already exists" and the node does not appear as created.

Cause

 The node already exists in SCOT but is not in an active (In Use) state.

Alternatively, the SXNODES table can be checked to confirm whether the node already exists in the system.

Resolution

Two possible approaches to resolve the issue:

  • In transaction SCOT, locate the existing SMTP node activate it by enabling the “Node in Use” flag.
  • Remove the existing SMTP node in SCOT and recreate the SMTP node in SCON, then verify that it appears and functions as expected.

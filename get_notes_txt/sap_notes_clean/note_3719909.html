Symptom

  • The document which its language is not in English is indexed successfully, but when querying for details, Joule fails to provide information based on the indexed document.

  • However if querying for indexed document that are in English, Joule could return the information.

Environment

  • Joule

  • Document Grounding

Reproducing the Issue

  1. Create a pipeline using the document grounding API (https://mtls.rage.c-<>.kyma.ondemand.com/pipeline/api/v1/pipeline).

  2. Upload non-English documents into the repository, for example, Microsoft Sharepoint or SAP Build Work Zone advanced edition.

  3. Trigger the pipeline update and verify that the document is indexed successfully.

  4. Query the indexed document for specific details by asking the Joule.

  5. Joule cannot retrieve the expected documents.

Cause

  • Joule uses the language settings of the SAP application when a user request is sent. Upon receiving a request through an SAP application, Joule automatically adopts the language configured within that application to ensure a seamless experience.
     
    • If the language is not explicitly set in the SAP application, Joule sets the language based on:

       - The browser settings on PC or Laptop devices

       - The device settings on mobile devices.

    • Note
      If the SAP application, device, or browser language settings are not supported by Joule, Joule will automatically default to English.

    • For more information, refer to Joule Multi Language Support | SAP Help Portal 

  • When Joule searches documents indexed by document grounding, Joule will search for English default documents and documents in the language defined in the settings, supported by Joule. 

Resolution

  1. Verify the Joule language settings of the SAP application, browser, or device used for querying the indexed document.

  2. If the language is not set to match the language of the document (e.g., German for German documents), update the Joule language settings to the appropriate language.

  3. Run the query again after updating the language settings.

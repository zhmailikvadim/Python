Symptom

How to Add "Standard / Custom Events as Planned Events at Shipment Level in Shipment IDoc"?

Reproducing the Issue

Cause

Planned events are generated from the Shipment IDoc. If the required planned event is not included in the IDoc payload, it will not be persisted or displayed in Global Track and Trace.

Resolution

  • If custom events are needed, configure them in the standard extractor and synchronize them back to Transportation Management; refer to: Sample code configuration guide for integration with global track and trace (see “Q2: How to add customized planned event and synchronize this event back to TM?”).

  • Add the required planned event to the extractor configuration so it is generated and sent with the shipment data; refer to: Add/Remove custom planned events.



Note: All of these are standard events (e.g., loading begin, loading end, departure, arrival, unloading begin, unloading end, POD, proof of pickup) and are predefined in the GTT extractor code. No further action is required from your side. If you want to add new planned events, please follow the guide “Add/Remove Custom Planned Events.”

Symptom

  • The naming convention for RFC connections as outlined in SAP Note 2407589 must follow specific guidelines detailed in Chapter 5 of the installation documentation under point 7.7 ("Creating RFC Destinations for liveCache Applications").
  • An RFC connection with the required name is already in use for another application with different rights, making it impossible to assign the required name.
  • The length of the RFC connection name is limited, adding further constraints.

Environment

  • liveCache Applications

Reproducing the Issue

  1. Attempt to create an RFC connection for liveCache applications following the naming convention outlined in SAP Note 2407589.
  2. Observe that the required name is already in use for another application with different rights.
  3. Note the limitation in the length of the RFC connection name.

Cause

  • The liveCache-related RFC name convention is hardcoded in the program code, making it challenging to modify this behavior.

Resolution

  1. Utilize the RFC connection for other components as well, ensuring that the requirements for liveCache outlined in SAP Note 305634 are met.
  2. Add additional authorizations to the user for other component usage, if necessary.
  3. Ensure that the main requirements regarding the RFC name, user, and password remain consistent with the specifications in SAP Note 305634.

See Also

  • Refer to: SAP Note 305634 - RFC Destination for Global Work in SAP liveCache

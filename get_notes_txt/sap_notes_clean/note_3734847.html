Symptom

You select deliveries in /SCWM/MON and use the 'Goods Issue' functionality. It is possible that the GI aborts with error message '/SCWM/GOODS_MOVEMENT080 - HU &1is located on resource &2' (this error is correct if the HU is actually on the resource). If the HU is moved to the GI-ZONE with a warehouse task in a parallel transaction, it is still not possible to post the GI afterwards. If you try to post the GI again, the same error message is raised, because the system reads the buffered data.

Reason and Prerequisites

This issue only occurs after implemeting note 3202677.

Solution

Implement the Correction Instruction.

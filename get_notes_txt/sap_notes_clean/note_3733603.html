Symptom

  • A custom role previously removed from a user's account appears again after the user logs in.
  • No error message or error code is displayed.

Environment

  • Product: SAP LeanIX solutions
  • LeanIX MTM usermanagement

Reproducing the Issue

  1. The user logs in to the workspace.
  2. The previously removed custom role is visible again in the user's assigned roles.

Cause

  • User roles are managed via the SSO identity provider (IDP). Assignment groups in the IDP automatically assign roles upon login. If the user belongs to a role-assignment group, the corresponding role is reassigned to the user.

Resolution

  1. Contact the SSO identity provider administrator and review the user's membership in role-assignment groups that map to workspace roles.
  2. Remove the user from any group that maps to the unwanted role, or adjust group membership to match the intended access.

Symptom

  • When testing the connection in CMC (Central Management Console) -> Applications -> Authorization Server Configuration or generating a token in BILP (BI Launchpad) -> User settings, the following error may be seen:
    An error occurred, kindly retry.
    If issue persists, please contact your administrator
    please close this window.
  • In the APS (Adaptive Processing Server) logs hosting the STS (Security Token Service), the below error may be seen:
    Process HTTP request : https://mtlsauth.microsoft.com/<TENANTID>/oauth2/v2.0/token
    responseCode : '400'
    OAuthImpl: Connection returned with error and response is : : {"error":"invalid_request","error_description":"AADSTS392200: Client certificate is missing from the request. <TRUNCATED>
    OAuthImpl.generateRefreshToken: error during response processing.
    java.lang.RuntimeException: Error returned from OAuth authorization server. [HTTP ResponseCode - 400]: {"error":"invalid_request","error_description":"AADSTS392200: Client certificate is missing from the request. <TRUNCATED>
     at com.businessobjects.pjs.services.sts.oauth.OAuthImpl.processResponse(OAuthImpl.java:1089)
     at com.businessobjects.pjs.services.sts.oauth.OAuthImpl.generateRefreshToken(OAuthImpl.java:254)

Environment

  • SAP BusinessObjects Business Intelligence (BI) Platform 4.x / 2025
  • oAuth 2.0 or OpenID using Microsoft
  • Windows
  • Linux / Unix 

Reproducing the Issue

  1. Configure a Microsoft Authorization server (Eg for OneDrive or OpenID workflows)
  2. Test the connection or generate a token
  3. Observe the errors mentioned in the symptoms

Cause

The token URL was configured using the mtlsauth URL (mutual TLS) which requires BI to send a client certificate with the request. BI does not support this.

Resolution

Change the Token URL in Applications -> Authorization server configuration from:
https://mtlsauth.microsoft.com/<TENANTID>/oauth2/v2.0/token

To:
https://login.microsoftonline.com/<TENANTID>/oauth2/v2.0/token

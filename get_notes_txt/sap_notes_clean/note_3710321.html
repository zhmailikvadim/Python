Symptom

 When creating an Invoice Draft in Service Layer using a Sales Order as the base document, the line quantity is expected to be recalculated based on the payload values. In the payload, the line includes the properties UnitsOfMeasurement and InventoryQuantity. However, during draft creation, the system appears to ignore these payload values and instead uses the quantity from the base Sales Order line as the reference for the calculation.
After reproducing the issue, we observed that if we update the same draft afterwards (PATCH) using the exact same values that were originally sent in the payload, the recalculation is performed correctly and the line quantity is adjusted as expected

Reproducing the issue

  1. Create manually any Sales Order and set the line Quantity as 10.
  2. Login in Service Layer
  3. POST /Drafts using payload below:

{
  "DocType": "dDocument_Items",
  "DocDate": "2026-01-06T00:00:00Z",
  "DocDueDate": "2026-02-17T00:00:00Z",
  "CardCode": "C20000",
  "DocObjectCode": "oInvoices",
  "DocumentLines":
    [
     {
      "BaseType": 17,
      "BaseEntry": 281,
      "BaseLine": 0,
      "InventoryQuantity": 10,
      "UnitsOfMeasurment": 0.248756
     }
    ]
}

Expected Results:
The line Quantity is recalculated to 40.2 based on the Service Layer payload values (UnitsOfMeasurement and InventoryQuantity).

Actual Results:
The line Quantity remains 10 (from the base Sales Order), and only InventoryQuantity is recalculated/adjusted instead of updating the line Quantity accordingly.

Solution

The section References below lists the related feature package or patch once they become available. Be aware that References can only be confirmed at feature package or patch release date. SAP delivers feature packages or patches only for selected releases at its own discretion, based on the business impact and the complexity of implementation.

Workaround:

1 - Create the Draft using only referring the base document:
  "DocumentLines":
    [
     {
      "BaseType": 17,
      "BaseEntry": 281,
      "BaseLine": 0,
     }
    ]

2 - Update the Draf with the desired values:
{
  "DocumentLines":
    [
     {
      "InventoryQuantity": 10,
      "UnitsOfMeasurment": 0.248756
     }
    ]
}

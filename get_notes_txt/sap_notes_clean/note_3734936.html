Symptom

  • When executing transaction FCLM_CR_INTRAM or FCLM_CR_MMRD, the system triggers a short dump.
  • During Reconciliation, the process terminates.
  • Runtime error: AMDP_EXECUTION_FAILED
  • Database procedure involved: CL_FCLM_CR_AMDP=>SYNC_FEBCL_UPDATE or CL_FCLM_MMRD_CR_ADAPTER=>SYNC_FEBCL_UPDATE 
  • SQL error code: 1299

Environment

SAP S/4HANA 1809 onwards

Reproducing the Issue

  1. Execute transaction FCLM_CR_INTRAM or FCLM_CR_MMRD.
  2. The system attempts to run the reconciliation process.
  3. A dump occurs, and the reconciliation stops before completion.
  4. ST22 shows the AMDP failure for class CL_FCLM_CR_AMDP or CL_FCLM_MMRD_CR_ADAPTER.

Cause

  • The number range for object FCLM_CR with number range number 01 was not maintained.
  • This caused the AMDP method SYNC_FEBCL_UPDATE to fail during execution.
  • Missing number interval triggered SQL error 1299 leading to AMDP_EXECUTION_FAILED. 

Resolution

  1. Go to transaction SNRO. 
  2. Enter object name: FCLM_CR. 
  3. Choose Number Ranges → Intervals. 
  4. Maintain and activate interval 01. 
  5. Save changes.
  6. Re-run FCLM_CR_INTRAM or FCLM_CR_MMRD to confirm fix.

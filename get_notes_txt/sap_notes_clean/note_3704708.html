Symptom

In Payroll Australia, when there is a retro over off cycle payroll, no reversion is generated for Leave provision which is incorrect. "Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental."

Environment

  • Human Capital Management (HCM)
  • SAP ERP
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP SuccessFactors Employee Central Payroll
  • SAP S/4HANA

Reproducing the Issue

  1. Run an off cycle payroll.
  2. In the next period, initiate retro over the off cycle. 
    In the below example, retro is trigged in period 03.2025 over off cycle on 20.08.2025.
  3. Check the leave provision of the regular period after off cycle pay, for example period 03.2025. Leave provision reversal is not generated, which is incorrect.

Cause

  • When running the off cycle payroll, the SAP Note 3522188 - Leave Provision Reversal on Off Cycle Payroll Run Non-CE Private Sector is not implemented yet in the system, so no leave provision is generated for off cycle payroll run.
  • When running the normal payroll and retro over off cycle, the SAP Note 3522188, as well as SAP Note 3621220 - Cost Center missing in retro payroll for employees terminated via off cycle, are implemented with manual instruction.
    The manual instruction of SAP Note 3621220 includes schema change to read leave provision from immediate previous period, namely off cycle pay result:

    So system incorrectly read leave provision from existing off cycle pay result without leave provision, and no reversal can be generated.

Resolution

  1. Undo the change of schema QLVP delivered in manual steps of SAP Note 3621220.
  2. Run the normal pay period, which does not have leave provision reversal, again to correct.
  3. Perform the manual change of schema QLVP in manual steps of SAP Note 3621220 again.

See Also

  • SAP Note 3621220 - Cost Center missing in retro payroll for employees terminated via Off Cycle

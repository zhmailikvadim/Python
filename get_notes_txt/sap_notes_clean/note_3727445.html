Symptom

The KBA has a detailed summary of the behavior of the two switches in Onboarding General Settings → Restart Settings:

  • Allow Recruiting Updates
  • Retain Onboarding Information

Environment

SAP SuccessFactors Onboarding

Resolution

1. Allow Recruiting Updates

Purpose:

  • Controls whether changes made in Recruiting (Start Date or Hiring Manager) are synchronized with the Onboarding process.

Behavior:

  • When enabled, the system listens for updates from Recruiting.

  • If a Start Date or Hiring Manager change occurs in Recruiting:

    • The system triggers an automatic restart (if auto-restart is configured).

    • During the restart, User Entities and Person Entities are updated with the latest values from Recruiting.

  • When disabled, any updates made in Recruiting will not be reflected in Onboarding, even if restart mechanisms are configured.

Note: This switch is mandatory for syncing Recruiting updates with Onboarding.

2. Retain Onboarding Information

Purpose:

  • Controls whether existing onboarding data (in pages like New Hire Data Review) should be retained or overwritten when a manual restart is performed.

Behavior:

  • This switch only applies when a manual restart is triggered.

  • It does not affect scenarios where Auto Restart is configured.

Scenario A — Auto Restart Configured

  • If Allow Recruiting Updates = ON and Auto Restart is configured:

    • The system automatically restarts onboarding when Recruiting updates occur.

    • User and Person entities are updated from Recruiting.

    • Retain Onboarding Information switch has no impact in this scenario.

Scenario B — Manual Restart Performed

  • When Auto Restart is NOT configured and a manual restart is triggered:
Retain Onboarding InformationSystem Behavior
EnabledExisting New Hire Data Review (NHDR) information from Onboarding is retained after restart.
DisabledNHDR information is overwritten with the latest data fetched from Recruiting when the restart loads the NHDR page.

NOTE:

  • Both the switches should not be enabled at the same time if Auto-Restart rule is not configured.

Symptom

The freight line of marketing documents does not display correctly in the tax reconciliation report.

Reproducing the issue

Prerequisite:

  • You are working in a localization where Extended Tax Reporting is supported.
  • In Administration → System Initialization → Company Details → Accounting Data, Extended Tax Reporting is selected.
  • In Administration → System Initialization → Company Details → Basic Initialization, Use Perpetual Inventory is selected.
  • On the Administration → System Initialization → Document Settings → General tab, Manage Freight in Documents is selected.
  • In Administration → Setup → General → Freight, define freight FR1 as follows:

        Revenue Account: 8600

        Expense Account: 4600

       Stock/Fixed Asset: Selected

Scenario 1:

  1. Add an Inventory and Purchasing item I001.
  2. Add an A/P invoice with a vendor as follows:

Line 1: Item Code: I001, Quantity: 1, Unit Price: 100, G/L Account: 1200, Tax Code: I1 (10%), Distribute Freight: Yes

  1. Add freight on the document header:

Line 1: FR1, Net Amount: 10, Tax Group: I1 (10%), Stock/Fixed Asset: Yes

  1. Close the Freight window. Now, on the document header, the Tax in total is 11, and the Document Total is 121.
  2. Add the A/P invoice.
  3. Go to Financials → Financial Reports → Accounting → Tax → BAS Report Generation/Tax Report Generation (depending on the localization), select the period when the A/P Invoice was added, and other relevant information, click 'OK'.
    In the Tax Declaration Name form, find Tax Code 'I1' and select the A/P Invoice added in step 5. Click 'Add'.
  4. Go to Financials → Financial Reports → Accounting → Tax → Tax Reconciliation Report, select the Tax Declaration Name that was added in step 6; choose tax code 'I1'. Run the report.

Actual behavior:

The freight line is missing in the report.

Expect behavior:

The freight line is correctly displayed under the G/L account 1200.

Scenario 2:

  1. Add an Inventory and Purchasing item I001.
  2. Add an A/P invoice with a vendor as follows:

       Line 1: Item Code: I001, Quantity: 1, Unit Price: 100, G/L Account: 1200, Tax Code: I1 (10%), Distribute Freight: No

  1. Add freight on the document header:

       Line 1: FR1, Net Amount: 10, Tax Code: I1 (10%), Stock/Fixed Asset: Yes

  1. Close the Freight window. Now, on the document header, the Tax in total is 11, and the Document Total is 121.
  2. Add the A/P invoice.
  3. Go to Financials → Financial Reports → Accounting → Tax → BAS Report Generation/Tax Report Generation(depends on the localization), select the period when the A/P Invoice added in step 5, and other relevant information, click 'OK'.
    In the Tax Declaration Name form, find Tax Code 'I1' and select the A/P Invoice added in step 5. Click 'Add'.
  4. Go to Financials → Financial Reports → Accounting → Tax → Tax Reconciliation Report, select the Tax Declaration Name that was added in step 6; choose tax code 'I1'. Run the report.

Actual behavior:

The freight line is missing in the report.

Expected behavior:

The freight line is correctly displayed under the Freight Expense account 4600.

Note:

This issue also occurs with Service type A/P Invoice.

Solution

SAP intends to provide a feature package or patch to solve the symptom that is described in this SAP Note. The section References below lists the related feature package or patch once they become available. Be aware that References can only be confirmed at feature package or patch release date. SAP delivers feature packages or patches only for selected releases at its own discretion, based on the business impact and the complexity of implementation.

Workaround: temporarily disable Extended Tax Reporting when running the report.

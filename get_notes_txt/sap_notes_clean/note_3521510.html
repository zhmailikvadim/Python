Symptom

When you log into the portal, you don't receive any tasks or items coming from the connector to the provider system. After an overview of the Connections status, the following error for the system connector is observed: "Trying to access row values in a table which has no rows yet". The Status for the connector is "Exception occurred".

The traces show:

"...

com.sap.mw.jco.JCO$Exception: (106) JCO_ERROR_RESOURCE: Trying to access row values in a table which does not have any rows yet
at com.sap.mw.jco.JCO$Record.checkRowCapacity(JCO.java:14173)
at com.sap.mw.jco.JCO$Record.getString(JCO.java:14694)
at com.sap.mw.jco.JCO$Record.getString(JCO.java:16178)
at com.sapportals.connectors.SAPCFConnector.execution.structures.RecordSetWrapper.getString(RecordSetWrapper.java:276)
at com.sap.netweaver.bc.uwl.utils.r3.R3SystemInfo.getTimeZoneStr(R3SystemInfo.java:386)

..."

Other Terms

UWL, Universal Worklist, TTZCU, Time-Zone

Reason and Prerequisites

This issue occurs because of an unhandled exception, when fetching the data for the configured Time-Zone, in the provider system.

Solution

To correct this problem, apply the patch matching your support package version as listed in the Support Packages & Patches section of this SAP Note, and as per the instructions in the SAP NetWeaver Support Package Stack Guide.

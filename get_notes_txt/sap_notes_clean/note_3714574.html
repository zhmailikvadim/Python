Symptom

  1. Error message: "You don't have permission to save this document" appears when attempting to complete onboarding compliance forms in the mobile browser.
  2. Forms are password-protected in the mobile browser but accessible without restrictions in the desktop browser.
  3. Completion of forms via mobile browser is inconsistent.

Environment

  • Accessibility: Mobile browser and desktop browser

Reproducing the Issue

  1. Access the SAP SuccessFactors Onboarding module via a mobile browser.
  2. Attempt to complete a compliance form.
  3. Observe the error message: "You don't have permission to save this document."

Cause

  • The compliance form PDF was uploaded with an owner password, which applies security restrictions preventing edits or saves from non-owner applications, such as mobile browsers or PDF viewers.

Resolution

1. Reconfigure the PDF form's security settings:

a. Open the PDF form in a professional PDF editor, such as Adobe Acrobat.

b. Navigate to the security settings and use the owner password to remove restrictions preventing editing or saving.

c. If the owner password is unknown, recreate the form without security restrictions.

2.Re-upload the unrestricted version of the PDF form to the SAP SuccessFactors Onboarding module:

a. Log in to the SAP SuccessFactors Admin Center.

b. Replace the secured version of the form with the unrestricted version.

3. Confirm permissions:

a. Verify that role-based permissions are correctly assigned to allow new hires to edit and submit compliance forms.

b. Check permission roles in the Admin Center.

4.Use a desktop or laptop as an alternative:

a. Access the SAP SuccessFactors portal via a desktop or laptop browser, which may bypass mobile-specific PDF restrictions.

Symptom

  • In the stock requirements list (T-code: MD04), the sales order segment is not based on the sales order number but on the sales contract.
  • Planned order created under this segment has sales contract as account assignment instead of sales order.

Environment

  • SAP ERP Logistics (PP)  
  • SAP R/3
  • SAP R/3 Enterprise
  • SAP ERP Central Component
  • SAP ERP
  • SAP Enhancement package for SAP ERP
  • SAP Enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4 HANA, on premise
  • SAP S/4HANA Cloud Private Edition Release Independent

Reproducing the Issue

  1. Create sales contract in T-code: VA41.
  2. Create sales order in T-code:VA01 by referring to sales contract.
  3. Run MRP.
  4. Check the planned order created for the sales order in T-code:MD04.

Cause

This behavior is required for the following reason:

  • The sales contract is the account assignment object to which all further data has to be assigned, so it must not be replaced by the sales order if you create an order which refers to this sales contract. 
  • Since you could have started producing or procuring on the sales contract, the assignment would not be correct anymore if in the planned order or the production order there would be the sales order instead of the sales contract.                            

Resolution

The first document you have created is the contract. Therefore also the customer individual segment (blue line in T-code: MD04) carries the document number of the contract.
This also means that all further documents - planned order, production order, ... - will receive the contract as account assignment.

Symptom

  • System and Database are down
  • Event Configuration for "System not responding", "Instance not responding" and "HANA service not running" were configured
  • Alerts for "System not responding" and "Instance not responding" are triggered
  • Alerts for "HANA service not running" are not being triggered even though the database is down.

Environment

  • SAP Cloud ALM Health Monitoring

Reproducing the Issue

  1. In SAP Cloud ALM Health Monitoring configure the Event "HANA Service not running" your SAP S/4HANA system
  2. The Database is shut down
  3. Observe that no alerts are triggered for "HANA Service not running".

Cause

  • Alerts for "HANA Service not running" are not generated because the Health Monitoring system relies on data being sent via PUSH or PULL jobs. If the database is down, the ABAP instance cannot send the data required to trigger the alert.
  • The current design does not support a separate Heartbeat metric for the database, and the system depends on ABAP instance metrics for alert generation.

Resolution

  1. Refer to KBA 3426236 - Monitoring downtimes, outages, and unavailability with SAP Cloud ALM
  2. Refer to KBA 3536164 -  How alerts and emails work in Health Monitoring Application for an explanation of how alerts and emails work in the Health Monitoring application.
  3. If this functionality is critical, submit an enhancement request via the Customer Influence Program to request the development of a database-specific heartbeat metric
  4. Refer to the following links for submitting enhancement requests and reviewing planned functionalities: Customer Influence Campaign for Operations

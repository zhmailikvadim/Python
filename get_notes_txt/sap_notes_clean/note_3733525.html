Symptom

Error FI 313 occurs when clearing at G/L account level in transaction FB1S.

Environment



Reproducing the Issue

  1. Execute transaction FB1S (G/L Account Clearing)
  2. Create a clearing posting at G/L account level
  3. Error FI 313 is triggered

Cause

  • The error FI 313 occurs because no cost center (KOSTL) is transferred to the clearing line item. As a result, no Funds Center (FISTL) can be derived via FMDERIVE from the cost center. 

  • During open item clearing (OP clearing), CO (Controlling) postings from the line item to be cleared are not automatically transferred to the clearing line item. This includes, for example, the cost center (KOSTL). Only balance sheet-relevant postings (such as GSBER, VBUND, etc.) are transferred by default.

Resolution

Option 1: Define a Clearing Rule (As of Version 4.0)

Define a custom clearing rule in the Customizing section "General Ledger - Open Item Clearing" (Table TFAGS, Transaction OBIA). This allows to transfer custom fields (e.g., KOSTL) to clearing line items.

Option 2: Automatic Account Assignment

Enter an automatic account assignment in transaction OKB9.

Option 3: Use Substitution

Use substitution (Transaction OKC9) to automatically populate required fields in clearing line items.

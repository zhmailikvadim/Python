Symptom

You use transaction VL10HU to create an outbound delivery for handling units. These HUs are loosely assigned to a stock transport order. The purchase order has items with the Subcontracting Stock indicator. (SOBKZ = O). A short dump occurs with the error message "GETWA_NOT_ASSIGNED".

Other Terms

SAPLVHUDLVHU, LVHUDLVHUF09, ADD_MM_DATA_BY_ORDER, special stock indicator, SOBKZ, EKPO, EKKN

Reason and Prerequisites

Prerequisite:

The purchase order has a special stock indicator that is not assigned to an account assignment.

Reason:

In the source code, it is assumed that a special stock indicator is always assigned to an account assignment and that an entry therefore exists in the table EKKN.

Solution

Implement the attached correction instructions or import the specified support package.

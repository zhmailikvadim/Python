Symptom

You have activate the FI document summarization and create a parked FI document using an accounting BAPI or a FIORI app. The document originally contained account assignments, which have been deleted in the parked document due to the summarization. When that parked document is posted, then the GL view also does not contain these account assignments. The GL view also only contains the summarized data.

If one of the accounts in the document is a cost element in CO, the posting could also lead to an error in CO because of missing account assignment on that account.

If you are using NewGL document split and one of the summarization fields is used as split criteria, then the posting also could end up in an error in NewGL document splitter.

Other Terms

BAPI, Document parking, FBV0, OBCY, OBCYX, badi_fins_summarize_bseg

Reason and Prerequisites

The parked document will store only the data of the entry view. So when the entry view is summarized, the corresponding information is lost in the parked document and cannot be restored in the document posting process.

Solution

Deactivate the document summarization for these parked documents or implement the attached correciton instruction in your system. After the corrections of this note has been implemented in your system, the FI document summarization will not be processed anymore for documents that are parked in accounting interface.

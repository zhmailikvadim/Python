Symptom

  • A user is successfully logged into SAP SuccessFactors and has initiated a Proxy session to act on behalf of another user.

  • When the user attempts to navigate to another integrated application (e.g., SAP Analytics Cloud, Employee Central Payroll, CAP Application and others) via Principal Propagation, the proxy context is lost.

  • The target application recognizes the logged-in user (the account holder) instead of the proxy user (the person they are acting as).

  • Proxy User session is not maintained during Principal Propagation between applications using IAS as IDP

Environment

  • SAP SuccessFactors HCM Suite

Reproducing the Issue

  1. Log into SAP SuccessFactors as User A.

  2. Use the Proxy Now feature to act as User B.

  3. While in the proxy session, click a link or tile to navigate to a secondary application (e.g., SAC) that uses Principal Propagation via IAS.

  4. Observe that the secondary application logs the user in as User A, not User B.

Cause

This behavior is due to a current architectural limitation within the Identity Authentication Service (IAS).

When IAS acts as the Identity Provider (IDP), it validates and propagates the identity of the authenticated primary user. Currently, IAS does not support the exchange of "Proxy" or "On-behalf-of" tokens between service providers. While the proxy session is active within the source application (SuccessFactors), that specific session state is not currently transferable through the IAS principal propagation framework.

Resolution

Current Status: The system is working as designed. At this time, Principal Propagation via IAS only supports the identity of the logon user.

Workaround:

  • Users must perform actions in the target application using their own authenticated credentials.

  • If a user needs to perform actions as another person in the target application, they must log into that application directly (if supported) or have the necessary permissions assigned to their own ID.

Future Roadmap: There is an active enhancement request currently in the early stages of review by the IAS and Analytics development teams to support proxy scenarios. This article will be updated as the roadmap progresses.

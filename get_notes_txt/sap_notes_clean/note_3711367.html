Symptom

Customers observe directories with names like DM<*>.dir, .dir, or .tmp being created under the serverX/temp directory on the Adobe Document Services (ADS) Java server.
These directories may trigger internal security alerts or be flagged as a potential cyber attack.

Environment

  • SAP NetWeaver AS Java

  • SAP Interactive Forms by Adobe

Resolution

This behavior is expected and standard for Adobe Document Services.

The directories created under the ADS Java instance path (such as server0/temp) are temporary working directories generated by the Adobe rendering engine during runtime. They are used to store intermediate files while ADS processes form requests and generates PDF output. 

These directories are created internally by ADS, not by external commands and they don't indicate any unauthorized access. The directories are typically deleted automatically after processing or when the Java instance is restarted.

Symptom

FSBP_CENTRAL_MEMORY->GET_BUT000_DATA returns that the partner has been changed if it does not exist (neither on the DB nor in memory).

FSBP_CENTRAL_MEMORY->GET_ADDRESSES_BAPI does not return the fact that no address data has changed if the BP does not exist (neither on the DB nor in memory).

Other Terms

-

Reason and Prerequisites

For a BP that does not exist on the DB it was assumed that it's new and thus changed.

For a BP that does not exist on the DB or in memory no category (person/group/org) is returned so neither the person-address coding nor the organization-address coding was called so the flag for the address not having changed was not set.

Solution

Apply this note or the SP.

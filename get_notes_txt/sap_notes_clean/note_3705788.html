Symptom

  • Auto import job was scheduled in the test system without a user request.
  • The job was last changed by "SAP system processing".

Environment

SAP S/4HANA Cloud Public Edition

Reproducing the Issue

  1. Observe the test system's automation import settings.
  2. Identify that the auto import job was scheduled without user intervention.

Cause

The auto import job was set up as part of the default configuration during the initial ATO setup task list execution.

Resolution

  1. Confirm that the auto import job was configured as part of the default settings during the initial ATO setup.
  2. Refer to the SAP Help Portal documentation for expected behavior in new systems:

Transport Automation

Note that the automatic import for the test system runs daily from 1am to 2am (UTC) and applies to all collections.

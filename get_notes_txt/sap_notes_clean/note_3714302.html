Symptom

  • Your SAP system runs on a Virtual Machine which has been setup in the "Alibaba Cloud" public cloud infrastructure.
  • Your SAP system does not have a valid SAP license.
  • In transaction SLICENSE, there is no license key and no active hardware key.
  • In transaction SM21, the syslog contains entries like:

Msg server hardware ID missing (HWID_MSERR)

The hardware ID of the message server is not known in the application server.

HWID_MSERR An error has occured while determining the hardware ID in the message server. System log messages in the message server should give you an indication about the cause of the error.

HWID_PENDING The application server has still not received a hardware ID from the message server. Use the earlier system log entries to check whether the connection to the message server was opened correctly. You can monitor the message server with the message server monitor (transaction SMMS).The dev_w* developer traces contains entries like:

ERROR => SlicLikeyGetHardwareKey: no hwid from MSG server
ERROR => 11 likey_get_validity_dates: Error getting the hardware key. You might find some information in file "dev_ms" on the message server

  • The dev_ms developer trace contains entries like:

NiIRead: raw read for hdl 1 timed out (4000ms)
*** ERROR => NiRawRead NiRc=-5 len=0 [virtinfoxx.c 495]
VirtIMetaDataServiceGetValue: can't read document
*** ERROR => SlicGetFullHwId: invalid parameter. Please check SAP note 2533233. [slichwid.c 1691]
*** ERROR => SlicGetHwId: SlicGetFullHwId failed! [slichwid.c 662]

  • The command 'saplicense -get' returns the following error:

*** SAPLICENSE (Release 7xx) ERROR ***
ERROR: Unexpected function call
DETAILS: NiIHSBufInit: already initialized

  • on SAP kernel 7.93 the hardware key may be generated, but it is unstable and changes frequently, eg. when running "saplicense -get"

Other Terms

license, saplikey, saplicense, SLICENSE, hardware key, VM ID, Instance Id,  Linux, virtual machine, VM, cloud, IaaS, Alibaba Cloud, Aliyun

Reason and Prerequisites

Starting with SAP kernel patch levels listed in SAP note 2533233 the hardware key for the SAP license is no longer based on the MAC address (as in on-premise Linux installations) but on a unique VM ID which is provided as part of the Alibaba Cloud infrastructure, provided by the Alibaba Instance Metadata Service.

If the Instance Metadata Service is set to "Security Hardened" mode, and if your SAP kernel is running on a patch level that is older than the ones listed below, then the above symptoms will occur.

Solution

  • Make sure that the Alibaba Cloud Instance Metadata Service (IMDS) for your VM is enabled.
  • If you decide to set the IMDS mode to "Method 1: Security hardening mode", then you have to update your SAP kernel to one of the following versions:
    • SAP kernel 7.22 EXT PL #1515 or higher
    • SAP kernel 7.53 PL #1539 or higher
    • SAP kernel 7.54 PL #636 or higher
    • SAP kernel 7.93 PL #350 or higher
    • SAP kernel 9.16 PL #89 or higher
    • SAP kernel 9.17 PL #29 or higher
    • SAP kernel 9.18 PL #20 or higher
    • SAP kernel 9.19 PL #5 or higher
  • If you want to set the IMDS mode to "Method 2: Normal mode", then no SAP kernel update is required. The described symptoms do not occur.
  • For details about the IMDS configuration please consult the Alibaba Cloud IMDS documentation.

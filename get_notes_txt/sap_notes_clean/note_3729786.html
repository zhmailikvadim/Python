Symptom

  • When a supervised mentoring program was created with a gender-based key question configured to prefer male mentees, the mentor was matched with a female mentee.
  • No error message is displayed.

Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.

Environment

  • SAP SuccessFactors Succession & Career Development Plan
  • SAP SuccessFactors Mentoring

Reproducing the Issue

  1. Create a supervised mentoring program.
  2. Configure a key question to capture the mentor’s preference for male mentees.
  3. Start the mentor–mentee matching process.
  4. Review the matching results. The mentor may be matched with a mentee who does not meet the configured gender preference.

Cause

A preferred mentor chosen by a mentee during signup overrides key question criteria in the initial matching cycle of supervised mentoring programs, if the mentor has available slots.

Resolution

1. Verify preferred mentor selection: Check whether the mentee selected a mentor during signup (the UI may show a hint such as “Mentee requested this mentor”).

2. Acknowledge expected behavior: In supervised mentoring programs, if a mentee selected a preferred mentor and that mentor has available slots, the matching engine prioritizes this match over question-based criteria.

3. Enforce key question criteria first (if needed): Either wait until the preferred mentor selection cycle ends, or instruct mentees not to choose a preferred mentor during signup.

4. Proceed with question-based matching: After the preferred-mentor cycle ends, run the question-based matching round and ensure key questions are configured correctly, as key question matches are mandatory during this stage.

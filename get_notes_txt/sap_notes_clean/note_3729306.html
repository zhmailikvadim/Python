Symptom

You edit a F110 payment proposal in transaction FBZ0 or via the option "Edit proposal" in transaction F110. There is a successful payment group which contains at least two items. You reallocate the items from this payment group to at least two different new payment groups. All new payment groups will have the same Unique Transaction Reference (UETR). This behavior is incorrect because the reference should be unique for each payment.

Other Terms

FBZ0, F110, REGUH-UETR, F110V-XMITD

Reason and Prerequisites

Program error

You specified that a payment medium should be created when scheduling the payment proposal.

Solution

Implement the attached Correction Instruction (CI) or the relevant Support Package.

Symptom

Accommodate MuleSoft Cloud API Integration to call the US Treasury System by enabling agencies to securely store the Client ID and Client Secret (provided by MuleSoft) in ABAP Secure Storage.

Other Terms

G-Invoicing, MuleSoft Cloud API, Client ID, Client Secret, ABAP Secure Storage, US Treasury System, Secure Store, Certificate

Reason and Prerequisites

Client ID and Client Secret credentials required for MuleSoft Cloud API Integration must be stored securely within the SAP system. Storing these credentials in ABAP Secure Storage ensures they are protected and not exposed in plain text configurations.

Client ID and Client Secret are needed as part of Multi-Factor Authentication Process to connect to any Treasury API instance via MuleSoft.

Solution

Implement the attached correction instructions.

Apply the SAP notes related to G-Invoicing Cloud API in following sequence:

Symptom

The project profile field is locked and cannot be edited after while copying a project.

Environment

SAP S/4HANA Cloud Public Edition

Reproducing the Issue

  1. Navigate to Project Control - Enterprise Project.
  2. Select a project and click on the "Copy" button.
  3. Insert "Project Name" and "Project ID" and click on the "Copy" button.
  4. Observe that in the header, the project profile field is locked and cannot be edited.

Cause

  • When using the "Copy Project" functionality, the system treats the target project as a derived object, not a new free-form creation.
  • The project profile is inherited from the source project.
  • The system locks the project profile field by design.
  • This behavior occurs before the project is technically saved but after the copy logic is triggered.
  • From a technical SAP perspective, the project already exists in memory with a fixed controlling framework, and the project profile is part of that framework.
  • This is intentional SAP behavior and not a bug or missing authorization.

Resolution

If the project profile must change, create a new project manually and select the correct project profile (e.g., Overhead) at the time of creation, then recreate the Work Breakdown Structure (WBS) manually for the new project.

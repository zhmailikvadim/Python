Symptom

You use the function for valuation of real estate contracts. It is unclear how the conversion date is determined for the conversion of the valuation currency into the first local currency or into the second and third local currency.

Other Terms

RECEEP, CTRULE, WWERT, CTRATE, KURSF, right of use asset, exchange rate

SAP contract, IFRS 16, US-GAAP ASC 842, leasing, right of use, leasing liability, contract valuation, IFRS, International Financial Reporting Standards, US GAAP, RE-FX, real estate

Reason and Prerequisites

See also SAP Note 2474141

Solution

Conversion into first local currency:

The valuation posting (RECEEP) posts in the valuation currency, which corresponds to the currency of the assigned conditions. This may differ from the company code currency. For technical reasons, only the asset documents (RoU documents) are included in the first local currency. Therefore, the RoU posting in local currency and the liability posting in the valuation currency are posted in two separate documents.

﻿The translation of the valuation currency into the first local currency is generally carried out in RE-FX on the basis of the currency translation rule . The currency translation rule is defined in the contract in the posting term section "Posting Parameters". The currency translation rule must always be the same for a valuation rule, in particular also for time slices of the posting term.

In Customizing for the currency translation rule, the exchange rate type (e.g. 'M') and the starting date for the currency translation can be stored.

The following settings are relevant in the context of valuation:

Depending on the combination, a different exchange rate can be determined for the display of planned records (for example, for setting CFGEN conversion date = current date) than for the posting (for example, with setting REEP conversion date = posting date). The exchange rate is determined on the conversion date using the defined exchange rate type from table TCURR. The conversion date is stored in the FI document in the field Conversion Date (BKPF-WWERT) and the exchange rate in the field BKPF-KURSF.

REQDATE and ACTDATE refer to the original posting for which the credit memo or subsequent receivable has been generated. If required, this can be translated using the exchange rate of the original posting.

Conversion to the 2nd/3rd local currency (for example, group currency):

For the conversion to 2nd/ 3rd local currency, the following settings are available in FI:

Financial Accounting -> Financial Accounting Global Settings -> Company Code -> Parallel Currencies -> Define Additional Local Currencies

Currency Type: for example, 30 Group Currency or 40 Hard Currency

Valuation , for example, 1 Group Valuation (this has nothing to do with the valuation in RE-FX)

Exchange rate type , for example: M

Base currency for currency conversion:

1 Conversion based on the transaction currency

2 Conversion based on the first local currency

Conversion Date Type:

1 Document Date (BKPF-BLDAT)

2 Posting Date (BKPF-BUDAT)

3 Conversion Date (BKPF-WWERT)

﻿The conversion of the valuation currency into the 2nd/3rd local currency is generally not executed in RE-FX. However, there are the following exceptions for which the conversion is carried out in the valuation context.

CFTYPE Posting Procedure Transaction
1 REAA Asset transaction (for reduction, complete retirement, sublet (CEG, CEH)
1 RETP Only for sublet (CEG, CEH)
2 READ Depreciation only in the case of direct balance sheet postings (without asset integration)

The conversion as part of the valuation posting takes place in the BAdI BADI_REEX_FI_BAPI, implementation BADI_REEX_FI_BAPI_REEP_LC23, class CL_DEF_IM_REEX_FI_BAPI_REEP_23.

For example, during the conversion of depreciation, the system does not determine the exchange rate from table TCURR, but calculates a mixed exchange rate that ensures that the total depreciation in all currencies matches the total of the asset transactions.

BAdIs in the context of currency conversion:

BADI_RECA_CURRENCY_TRANSLATION This BAdI only refers to the conversion to 1st local currency.

TRANSLATE_CURRENCY_BY_OBJNR Object-specific currency conversion
TRANSLATE_CURRENCY_EVAL_PSTNG Currency conversion valuation posting

BADI_REEX_FI_BAPI  This BAdI can be used for conversion into other local currencies (see also standard class CL_DEF_IM_REEX_FI_BAPI_REEP_23).

Symptom

During route scheduling or background processing (e.g., ATP/BOP), the system terminates with an unexpected exception and message VL572. The short dump indicates a failure when calculating working times, triggered in the scheduling preparation logic. The error message displayed is generic (“Unexpected exception”) and does not provide details about the affected calendar or date. The failure occurs when the system attempts to determine start/end time stamps for activities using factory calendars and time streams.

Other Terms

CHECK_TIMEPOINT_IN_WORKTIME, SD_ROUTE_SCHEDULE_PREPARE, TSTR_CALC_TIME, TSTR_SHIFTS_LOG_NO_BREAKS, Time Stream, Factory Calendar, Scheduling, MESSAGE_TYPE_X, VL572, T9510

Reason and Prerequisites

There is a problem with a generated timestream. As it is a serious error, the program was terminated with a shortdump with error VL572. However this error does not provide sufficient information about the cause of the error and in particular the timestream details. Please note that timestream contents can usually be displayed with report TSTRDISP.

Solution

Apply the attached correction instruction. A shortdump will still be generated in case of timestream error but with a new error message T9510 which provides the timestream details.

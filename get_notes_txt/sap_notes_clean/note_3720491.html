Symptom

In packing group processing, component groups belonging to sequence JIT Calls are assigned to slots according to their sequence numbers. If a subsequent delivery has already been created for a component group with a higher sequence number, the system still allows cancellation of a JIT Call with a lower sequence number in the standard process. Cancelling such a JIT Call can lead to an inconsistent sequence order, resulting in incorrect processing of subsequent sequences.

Other Terms

NJIT, Next Generation Just-In-Time Processing, Packing group, Subsequent delivery, Cancel JIS Call

Reason and Prerequisites

Modification

Solution

Implement the correction instructions provided in this SAP Note or import the relevant Support Package to activate the corrected system behaviour.

With this update, the system prevents cancellation of a sequence JIT Call when a subsequent delivery already exists for a higher sequence number. This restriction ensures that the sequence order remains consistent and avoids disruptions in the processing of JIT Calls and their associated deliveries.

Symptom

When distributing HR master data via ALE using transaction PFAL, the system fails to process custom segments (Z1Pnnnn) in extended message types. The error occurs inconsistently - the same variant may work correctly in some cases but fail in others, even within the same system instance.

The customer exit RHALE001 appears to be active in the system, but the custom code is not being executed during the ALE distribution process.

Environment

  • SAP ERP
  • SAP S/4HANA on-premise
  • SAP S/4HANA private cloud edition
  • SAP CRM
  • SAP SRM

Cause

The code inside the customer exit RHALE001 (specifically in the include associated with function exit EXIT_SAPLRHA0_004) has become corrupted. This corruption prevents the exit from being called by the standard ALE distribution programs, even though the enhancement appears to be active in the system. The corruption may cause inconsistent behavior where the exit works for some instances but not for others.

Resolution

To resolve this issue, the customer exit code must be regenerated:

  1. Execute report RSMODACT to reset the modification adjustment status
  2. Manually deactivate the customer exit RHALE001
  3. Manually activate the customer exit RHALE001 again

This process regenerates the internal code and resolves the corruption issue.

After performing these steps, verify that the function exit EXIT_SAPLRHA0_004 in customer enhancement RHALE001 contains the correct assignment for your extended message type, for example:

F_idoc_control-cimtyp = 'ZRMD_AXX'

Where ZRMD_AXX corresponds to your extended message type (e.g., ZHRMD_A09).

For additional information on configuring custom segments in HR-ALE distribution, refer to SAP Note 165699.

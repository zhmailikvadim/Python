Symptom

You would change the Copying Rules configuration for for example a Purchase Order (PO) item long text like Material PO Text from following settings:

Sequence Source object Text Fixed Fixed indicator
0 Purchase Order Material PO text <empty> (Text automatically adopted in target object)
1 Material Master Purchase Order text

* (User can have text adopted in target object)

to following setup:

Sequence Source object Text Fixed Fixed indicator
0 Purchase Order Material PO text * (User can have text adopted in target object)
1 Material Master Purchase Order text

<empty> (Text automatically adopted in target object)Sequence Source object Source Text Fixed indicator

but when you do that any existing PO lose their links to the Material PO Text, so previuosly text appeared on ME23n item long text tab, but now nothing.

Environment

  • Materials Management (MM)
  • SAP ERP Central Component
  • SAP ERP
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA

Cause

The reason for the text not being displayed is, that it’s not linked to the Purchase Order item directly. Before you have made the customizing changes, the fix indicator in the copying rules was set to *. In such a case the system will check for the text maintained in the material master as it is not linked to the Purchase Order yet. You can find the entry for the Material PO text maintained in the material master in table STXH (TDOBJECT = MATERIAL; TDNAME = your material number ; TDID = BEST).
In case the text was not copied to the Purchase Order Item manually by pressing the ‘Adopt Text’ button, the text will not be found anymore when the copying rules changed.

The reason behind this behavior is that system only search for the Material PO Text maintained in the material master if the fix indicator is set to *. When the copying rule is changed to <empty>, the system will search in table STXH for the relevant Material PO Text linked to the Purchase Order Item. When the text is not linked to the PO item directly, it won’t be found and displayed.

To summarize:

- When fix indicator set to *, text will be read from the material master first
- When the text was taken over by user, the text will be saved in STXH and further on read from this table
- When fix indicator is set to space / blank ' ' text will always be read from STXH once the PO was created (only during creation it will still be read from material master)

Resolution

The current system behavior is expected and won’t be changed.

See Also

If you want to keep such text in such POs, then the only option is to change the indicator back to * and adopt the texts manually to the POs

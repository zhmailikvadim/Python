Symptom

Error message KD625 ("Settle the balance from previous year in previous year first" ) occur in CJ88, even though the balance is "0" in T-code: CJI3.

Environment

  • S/4HANA system

Reproducing the Issue

T-code: CJ88
Input WBS element, period/Fiscal year => execute => error KD625 occur.

Cause

The balance of order/WBS element is zero with different cost elements after the last settlement. As each cost element must be settled, error message KD625 is raised as expected. 
This investment measure is technically closed (status TECO). Settlement is checking if WBS element/order values have to be settled to AUC (1. Step) and if AUC values have to be settled to final receivers (2. Step).
In ECC release the settlement checks the total order balance in step 1. then get message KD213, because total balance of WBS/order  is 0.
In S4 release the settlement checks the WBS element/order costs against settlement data (see costs in file COSPA and settlement data in file COSPD in report RKACSHOW). It was implemented with SAP Notes 2629587 & 2599930, because the check of total WBS element/order balance <> 0 is not sufficient in some cases as described in these Notes. 

Resolution

SAP recommend the solution of SAP Note 2725359 (Investment measure settlement: Settlement of sender costs from previous years) to solve this issue. After the order/WBS element has been successfully settled,  the settings of point 2. of the solution in this Note can be cancelled without any problems. 
If customer keep the settings in T-code:OBA5 and table T811FLAGS, the message KD625 will be displayed as a warning in the future for such a case.

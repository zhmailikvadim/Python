Symptom

Values for restricted account is not displayed correctly when there is PresentationSignReversal involved.

Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental  

Environment

  • SAP Analytics Cloud 
  • PresentationSignReversal

Reproducing the Issue

  1. Create a story(table) using a measure model.
  2. Create restricted account using a base measure with INC type in modeler and place it on rows.
  3. Created a calculated account using the restricted account created above and place it on rows.
  4. Now create 2 Restricted measure and place it on columns.
    1. Restricted measure 1 using any restriction on date, for example 2024.
    2. Restricted measure 2 using any restriction on date, for example 2025.
  5. Now create a calculated measure and place it on columns with formula = RM1-RM2 created above.
  6. Observe that the values for restricted account for calculated measure is displayed incorrectly.

Cause

  • This is a by-design behavior
  • In SAC, sign reversal is applied at the end of the formula calculation for a measure, which results in the observed outcome. Achieving the expected behavior in this scenario would require sign reversal to occur at the beginning of the calculation. Adjusting the solve order would not address this limitation. 

Resolution

Both the back-end and SAC team recommend to use restricted account in calculated account.

Symptom

Questions:

  1. How do I create end notifications in the legacy system during a system switch?
  2. How do I create retroactive end notifications if payroll was run for additional months after the system switch?

Environment

  • SAP ERP 6.04 and higher
  • Human Resources / HR / HCM

Reproduce Issue

A system switch is performed and a leaving action is executed for all pension recipients. However, the report RPCZOVD0 does not generate any end notifications that would be required in the paying office procedure.   

Solution

Answers:

  1. Q: How do I create end notifications in the legacy system during a system switch?
    A: In the case of a system switch, an end message must be generated in the legacy system. A pension payment is recognized by the existence of the wage types /102 and /103 in the payroll result. If a payroll result or the wage types /102 or /103 are missing for the selection period with which the report RPCZOVD0 (Create Pension/Capital Payment Notifications) is started, this is interpreted as the end of a pension payment. 

    Therefore, start the report RPCZOVD0 with the subsequent period after the system switch in the legacy system. 
    Ex. System switch takes place on January 1, 2026. The last period that was still settled in the legacy system is 12/2025. The end notifications in ZMV must be generated on December 31, 2025 --> start of RPCZOVD0 with selection period 01/2026.

    (Note: If RPCZOVD0 was inadvertently started in a period with the subsequent period for which payroll has not yet been run, end notifications are generated for all pension recipients. These are reversed again at the next start - after payroll has been run for the relevant month. Therefore, always make sure that the report RPCZOVD0 is started only after the payroll for the selected period.)


  2. Q: How do I create retroactive end notifications if payroll was run for another month after the system switch?
    A: If it only turns out after the productive payroll run that payroll has already been run for this month in the new system, the payroll results in the database prevent the creation of end notifications in the paying office notification procedure in the case of a system switch.
    If no further follow-on activities have taken place, you can use report RPUDEL20 to delete the payroll results.
    Then start RPCZOVD0 with the subsequent period after the system switch (see question and answer 1).

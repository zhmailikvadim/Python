Symptom

During the processing of material or Purchase order documents for LTSD request using transaction /SAPSLL/LTSDREQ_1, the runtime error ASSERTION_FAILED occurs in the class /SAPSLL/CL_LTSD_REQ_CTRL_STDCR, method GET_REQ_PERIOD_BASED_ON_LTSDS.

Among other things, the symptom occurs in the following constellation:

  • The request period of the LTSD request is two years.
  • A request tolerance was defined in the GTS Customizing activity "Preference Management > Vendor Declarations > Define Settings for LTSD Process".
  • For a product, several inbound LTSDs of the same LTSD issuer exist that cover different periods. These LTSDs have different release statuses.

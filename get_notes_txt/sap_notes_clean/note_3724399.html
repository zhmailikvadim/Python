Symptom

When processing a time stream calculation, the system terminates with a runtime error caused by an arithmetic overflow. The issue occurs during the calculation of time intervals when the system attempts to derive seconds from the date difference.

Internally, the function module TSTR_CALC_TIME receives an invalid base date, which results in an excessively large delta value. As a result, date-dependent  calculations may fail.

Other Terms

TSTR_CALC_TIME, TSTR_DB_SAVE, TSTR_INVALIDATE, Time stream, TSTRCOMP, TSTRCOMPH, STS0, STS1

Reason and Prerequisites

The reason is a program error.

The shared buffer for time‑stream structures was used inconsistently, causing outdated or incomplete data to be read during time‑stream loading.

This led to several kind of error like; incomplete generation of time streams or to a missing or invalid base date during delta time calculation, resulting in an arithmetic overflow.

Solution

Apply the attached Correction Instruction.
The correction disables the use of the shared buffer for time‑stream handling to ensure that only consistent data is read from the database.
No manual post‑implementation steps are required.

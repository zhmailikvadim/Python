Symptom

When you execute the report SI: Transfer to SI Procedures  (RP_PAYDE_B2A_SV_UNPACK), the system issues one of the following error messages for data packages with initial user data:

  • Without DB update: "Confirmation of other errors: DATEIKZ= (<DTART>/<BNRAG>/<BNRKK>/000000), HRPAYDEB2A613, Unknown return type (SVRTY=) occurred. (The return types 1 to 5 are known.)" (HRPAYDEB2A 613)

  • With DB update:
    • "Quarantine due to alternative file number (<DTART>/<BNRAG>/<BNRKK>/000000): File number 000000 is not the next file number <DATNR>" (HRPAYDEB2A 607)

      or

    • "File sequence number <DTART>/<BNRKK>/<DATNR> for PA/PTB <WERKS>/<BTRTL> already processed" (HRPAYDEB2A 006).

Reason and Prerequisites

The retrieved data package does not contain any user data. Instead, an error confirmation with a return code that starts with "E", such as E433 - file structure, is not recognizable or does not correspond to the specifications in the PackageHeader.

The reason for this is that the corresponding outbound notification could not be processed by the receiving office.

Solution

Implement the attached correction instructions or import the specified support package. After you import the HR Support Package or implement the correction instructions, you do not have to perform any further follow-on activities.

After you implement the corrections, the system issues a corresponding error message with a long text about how to solve the problem.

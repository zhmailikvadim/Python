Symptom

You create wage statements for employees as a form in accordance with the guidelines of the wage standard ELM as of Version 5.0. To do this, execute the program Wage Statement (as of ELM 5.0) (report: RPLLAWC3_CE, transaction: PC00_M02_LLAW3_CE).

The following problems occur:

  • A company name that is longer than 25 characters cannot be printed completely on the wage statement form.
  • A person ID with more than 8 characters stored in infotype 0709 – Person ID is displayed incompletely on the wage statement form.

Other Terms

Payroll Switzerland, ELM ("Uniform Wage Notification Procedure"), ELM 5.0 (transaction: PC00_M02_LELM0), wage statement as of ELM 5.0, (transaction: PC00_M02_LLAW3_CE), application key LAW ("Wage Statement"), infotype 0709 - Person ID

Reason and Prerequisites

You require the correction described here for productive use only as of Version ELM 5.0.

Solution

Import the HR Support Package relevant for your release. If you have to implement the solution in advance, use the SAP Note Assistant to implement the correction instructions relevant for your release. You can find more information about the SAP Note Assistant on SAP Support Portal at https://support.sap.com/notes.

After you implement this SAP Note, the correction has the following effect:

  • The BAdI HRPAYCHELM06 is also available when creating the wage statement form (see SAP Note "2357746 - ELM 4.0: Override fields in the enterprise description"). If the company name is filled in this BAdI, this value overrides the name defined in the view “ELM: Grouping of Payroll Units for Enterprises” (V_T5C51) or the corresponding Customizing activity. For an implementation example, see the referenced SAP Note 2357746.
  • The person ID stored in infotype 0709 – Person ID is always printed completely on the wage statement form. The label "PersID" is replaced with "ID" if the person ID is longer than 15 characters.

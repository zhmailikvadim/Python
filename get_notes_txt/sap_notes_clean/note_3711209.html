Symptom

You want to use the "Create Equipment Records Using Scanner" feature and specifically leverage the Advanced Configuration option, which requires a ScreenConfiguration.

Your goal is to customize how barcodes (for example, QR codes) are parsed, such as:

  • Splitting a barcode string into multiple field values
  • Using key–value pairs to populate equipment fields automatically

However, after configuring the ScreenConfiguration:

  • The barcode can be scanned successfully and the equipment creation screen opens
  • But the fields you defined are not automatically populated as expected

Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.

Environment

  • SAP Field Service Management (FSM)

Cause

This issue occurs when:

  • The ScreenConfiguration JSON is incomplete or incorrectly configured, and/or
  • The barcode string format does not match the parsing logic defined in the ScreenConfiguration

As a result, the system cannot correctly interpret the scanned barcode and map the values to the intended equipment fields.

Resolution

To correctly use the Advanced Configuration for barcode parsing, both of the following must be aligned:

  1. The barcode format (for example, key–value pairs and separators)
  2. The barcodeParserConfiguration section in the ScreenConfiguration JSON

Please note that the configuration and examples provided in this article are for reference purposes only and are intended to illustrate a best-practice approach. They do not represent a mandatory or complete configuration for all scenarios. For comprehensive guidance refer to documentation: Search for / Create Equipment Records Using Scanner 

Step 1: Extend the ScreenConfiguration

The ScreenConfiguration “Equipment Mobile” must be manually extended. In particular, you need to update the JSON Template and add a barcodeParserConfiguration section.

"barcodeParserConfiguration": {
    "valuesSeparator": ";",
    "keyValueSeparator": ":",
    "fieldsMapping": {
        "eqmsn": "equipment.manufacturerSerialNumber",
        "eqsn": "equipment.serialNumber"
    },
    "fieldsOrder": [
        "equipment.manufacturerSerialNumber",
        "equipment.serialNumber"
    ]
}

Explanation of the Configuration

  • valuesSeparator
    Defines how individual values in the barcode string are separated
    (in this example: ;)
  • keyValueSeparator
    Defines how keys and values are separated
    (in this example: :)
  • fieldsMapping
    Maps barcode keys to equipment fields
    • eqmsn → equipment.manufacturerSerialNumber
    • eqsn → equipment.serialNumber
  • fieldsOrder
    Defines the order in which fields are processed and populated

Step 2: Generate a Barcode That Matches the Configuration

Using the configuration above, the barcode string must follow this structure.

  • eqmsn:Test123;eqsn:Test456;

Step 3: Scan the Barcode on the Mobile App

  1. Open the FSM Mobile App
  2. Navigate to the Equipment section
  3. Tap the plus (+) button to create a new equipment record
  4. Scan the generated barcode / QR code

Step 4: Verify Automatic Field Population

After scanning the barcode:

  • The Manufacturer Serial Number field is automatically populated with Test123
  • The Serial Number field is automatically populated with Test456

Symptom

  • After in-place upgrade to BI 2025, in Central Configuration Manager (CCM), after starting the Server Intelligence Agent (SIA), it immediately goes to stopped state
  • In the upgrade logs, it can be seen that DeployFiles and AddNode fail with error:
     Could not reach CMS '<HOSTNAME>'. Specify the correct host and port and check for network issues, please refer to SAP Knowledge Base Article 2151044 on the SAP Support Portal. (FWM 20030)|Could not reach CMS '<HOSTNAME>'. Specify the correct host and port and check for network issues, please refer to SAP Knowledge Base Article 2151044 on the SAP Support Portal. (FWM 20030)*_*#*

Reason and Prerequisites

Environment

  • SAP BusinessObjects Business Intelligence (BI) Platform 2025
  • Windows
  • Linux / Unix 

Steps to Reproduce

  1. Upgrade from 4.x to BI 2025
  2. Observe the SIA cannot be started post upgrade

Cause

Java 8 -> Java 17 upgrade means there is no JRE directory. AddNode updates the server command lines to point to the new directory structure of sapjvm, due to it's failure the SIA is still pointing to the non-existent JRE directory.

Solution

The issue has been forwarded to development for further investigation

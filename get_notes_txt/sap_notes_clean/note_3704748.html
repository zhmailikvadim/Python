Symptom

Source control login issue in case of login via MII workbench.

Login to the source control fails in case of few special characters present in the password. same user is able to logon to Netweaver and also in NWDS but same password doesn't work in MII workbench source code login screen.

Netweaver logs shows following error.

Error while creating DiManager Instance
[EXCEPTION]
com.sap.di.sdic.exception.DiAuthenticationException: User credentials are invalid or user is denied access
at com.sap.di.sdic.host.impl.SLDHostImpl.createSLDConnection(SLDHostImpl.java:138)

Other Terms

source control, login, mii workbench, password, special character

Reason and Prerequisites

This happens because of few special characters in the password which breaks the password in the login request

Solution

as a workaround for now, Please avoid below characters while creating a password for the source control user.

These characters are : '<', '>', '&', '"', '\'

Symptom

  • SAP Datasphere tenant is unavailable; landing page shows: "An unexpected error has occurred."
  • HTTP Status: 500 error; Error Code: unexpectedError; Technical Message: HANA Database instance is starting, Code: 1891, SQL State: HY000.
  • Disk usage reached 100%, preventing the instance from operating.
  • Monitoring may show overall storage below total capacity while the effective DATA volume is full.

Environment

  • SAP Datasphere

Cause

  • The HANA Cloud instance experienced a full disk condition in a multi-volume setup where LOG and DATA are on separate volumes; effective usable DATA capacity is lower than the total provisioned storage.
  • Continuous growth (for example, system/runtime data and audit logs) consumed the effective DATA volume until it reached full utilization.

Resolution

  1. Increase Datasphere's storage to provide sufficient space. For example, upsize total storage so the effective DATA volume has free capacity.
  2. Initiate the workflow that updates SAP Datasphere so the new storage size is reflected in System Monitor and Tenant Configuration.
  3. Verify the tenant is up and running after the upsize.
  4. Temporarily pause or reduce data-ingesting activities to prevent refilling during recovery.
  5. Plan capacity based on the effective DATA volume (not the total provisioned storage) and closely monitor data growth, including audit logs, system/runtime statistics, and metadata via System Monitor.
  6. System Monitor has reccently included a new tile to reflect the effective DATA capacity considering a multi-volume setup.

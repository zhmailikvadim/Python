Symptom

You process electronic documents for Peru. The system writes an incorrect value for the Discrepancy response description in the electronic document file.

Other Terms

Peru, PE, eDocument, eDoc, Document and Reporting Compliance, discrepancy response, catalog type, Peru electronic documents.

Reason and Prerequisites

Reason: 

This SAP Note is relevant only for Peru.

Prerequisite: 

You have implemented the Electronic Document Processing Framework in your system. Refer to the implementation instructions provided in the SAP Help Portal for SAP ERP or SAP S/4HANA.

For a complete list of all corrections, check the following SAP Notes:

  • 2031941 - eDocument Peru: Installation Overview Note
  • 2651114 - eDocument Peru: Installation Overview Note for SAP S/4HANA

Solution

After implementing this SAP Note, the system sets the Discrepancy response description based on the catalog type in the electronic document file.

SAP recommends that you implement the Support Package specified for your release. However, if you need to install a solution earlier, use the SAP Note Assistant to implement the correction instruction. You can find more information about the SAP Note Assistant in SAP Support Portal, under SAP Note Assistant.

Note on Area Menu and IMG Updates:

Official SAP documentation cites menu paths on the SAP Easy Access screen and technical names of maintenance objects assigned to the SAP Implementation Guide (IMG; transaction SPRO).
Any updates to the menu on SAP Easy Access screen and to the IMG require a given support package to be installed.
If you have not installed the corresponding support package, the updated menu path is not available in your system.
To access the required objects, call the transaction code assigned to area menus or enter the technical name of the maintenance object in the maintenance transaction, such as SM30 for maintenance views. The object names that you need to use are covered in the documentation.

Note about translations:

Translations are only delivered with support packages.

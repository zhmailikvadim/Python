Symptom

  • An error occurs when attempting to transfer measure data from SAP Datasphere into SAP Control Tower.
  • The error message displayed is: <800002> - The import from SAP instance is not possible. Update the SAP Sustainability Control Tower integration content on your SAP instance to the latest version and overwrite the affected tables/views: For the view XXXXX, the field FISCAL_PERIOD has been removed.

Environment

SAP Sustainability Control Tower

Reproducing the Issue

  1. Attempt to transfer data for a certain measure from SAP Datasphere and SAP Control Tower;
  2. Encounter error <800002> indicating issues with SAP integration content and missing field FISCAL_PERIOD.

Cause

The issue may have been caused by absence of the field FISCAL_PERIOD when the SAP SCT package was initially installed in Datasphere.

Resolution

  1. Re-import the latest SAP Sustainability Control Tower integration content, ensuring that the FISCAL_PERIOD field is included in the views and the option "Overwrite objects and data" is selected to align the metadata content;
  2. Confirm that the metadata discrepancies are resolved by reviewing the updated metadata files;
  3. Test the connection to ensure data retrieval is successful and execute the import again.

See Also

SAP Datasphere Integration

Keywords

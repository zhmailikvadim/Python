Symptom

Poor performance in CO_TRANS 

Environment

  • Controlling (CO)
  • SAP ERP
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA
  • SAP S/4HANA Finance

Reproducing the Issue

  1. Run program FINS_CO_TRANS_WRI via transaction code SE38.
  2. Enter the relevant parameter.

Cause

The increased processing time could be due to many reasons like:

  1. Different / increased data basis;
  2. Less available process;
  3. DB resources due to run of other jobs in the same time.

Resolution

  • Please implement the latest versions of the both Notes into your system via transaction SNOTE and check if the performance issue is resolved.

SAP Note 3129861 - CO_TRANS: DUMP TSV_TNEW_PAGE_ALLOC_FAILED with text "No more memory available to add rows to an internal table" for SELECT on COSP or COSS in program FINS_CO_TRANS_WRI

SAP Note 3151595 CO_TRANS: System does not always archive all COEP entries

  • You could also check if the below customization described in the SAP Note 2906925 could improve performance.

"...

- change the profile parameter settings for parameter rsdb/max_blocking_factor (Split factor for FOR ALL ENTRIES queries) (from 50) to the customized value in c_max_block (see Note 2852252) divided by 10 to optimize ACDOCA selection.
It means if you have set value in c_max_block = 100.000, profile parameter rsdb/max_blocking_factor could be set as 10.000.
Please also see Note 48230 for more explanation on this and related profile parameters.

..."

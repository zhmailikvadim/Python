Symptom

Encountering a dump during external document creation when the UBLExtensions section for PODMIOT3 is missing in the UBL payload.

Other Terms

Poland, eDocument, eDoc, eInvoicing, electronic invoice, electronic invoicing, Poland eInvoice, Document Compliance, Digital Compliance, DC, B2B, KSeF, Electronic Document, Document and Reporting Compliance, DRC, FA(2) Schema Version, FA(1) Schema Version, UBL, External Document, External System, Podmiot3, FA(3) Schema Version, Down-Payment Invoice, UBLExtensions.

Reason and Prerequisites

  • Implement the SAP Note 3708079 Electronic Document Processing for Poland - KSeF: External Documents for FA(3) Schema - Phase II
  • For a complete prerequisite list, refer to SAP Note 3279206 (Electronic Document Processing for Poland - KSeF: Implementation Overview).
  • Check the attached sample UBL payloads for this particular case to create the external document invoices.

Solution

Implemented corrections to handle the scenario where UBLExtensions section for PODMIOT3 are not provided, thereby preventing the dump during external document creation.

Note: Implementation of SAP Note 3719368 is optional. It is only required for customers who need to process external documents based on the FA(3) schema. Customers not using this functionality can skip the note installation.

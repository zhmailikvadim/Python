Symptom

  • Setup remote source through HanaAdapterV2
  • Create replication task with filter, the filter is applied to initial load, but not realtime replication, newly update records applied to target HANA even not within the filter condition

Environment

SAP HANA Smart Data Integration

Reproducing the Issue

  1. Use HanaAdapterV2 to create a remote source with the default configuration settings.
  2. Implement a realtime replication task and apply a filter to specify which records should be replicated.
  3. Update records that fall outside the defined filter conditions.
  4. The updated record, despite not meeting the filter conditions, is still replicated to the target HANA system.

Cause

The configuration for remote source is incorrect

Resolution

  1. Reset all the subscriptions under the same remote source
  2. Update remote source to set "Enable Filter in Triggers" to true.
  3. Start the replication again.

Please note: Realtime replication for the same table with different filter is not possible through HanaAdapterV2. If there is such requirement, please use HanaAdapter remote source instead.

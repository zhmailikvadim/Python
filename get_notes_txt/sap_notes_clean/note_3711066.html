Symptom

The application log related to the value chain is currently filled with 999,999 messages.When there are more than 999999 entries for a specifc value chain, it is not possible to process any documents in value chain and system terminates with CX_VCM_APP_LOGGER dump.

Other Terms

CX_VCM_APP_LOGGER, BALHDR, VCM, Purchase Order, Sales Order

Reason and Prerequisites

There is a limitation in the application log, which can store only up to 999,999 messages for a single log instance. When an application log linked to a value chain becomes full, meaning the tally of messages reaches 999,999, the value chain and related objects (such as sales orders, purchase orders, deliveries, etc.) cannot be processed further.

Solution

Please implement the attached correction instructions.

This note provides a report to create a new log to replace the existing one, allowing for the processing of objects associated with a value chain where the current log is full.

However, please note that after creating the new log, the old messages will no longer be visible in the Monitor Value Chain Fiori app. They can still be accessed via SLG1 or SLGD transactions using the chain instance GUID or log handle.

Symptom

Evaluation request created in Manage Templates (App ID F2193) is not displayed for the appraiser user in the "Evaluate Suppliers" FIORI application (ID F1650)

Environment

  • Supplier Evaluation

Reproducing the Issue

  1. Open the "Evaluate Suppliers" FIORI application (ID F1650) with the user, that was earlier assigned as appraiser user to Supplier evaluation request.
  2. Attempt to search for data.

Observe that no data is displayed despite.

Cause

  • Evaluation requests were created but not processed due to jobs scheduled for the report "/SRMSMC/R_SEV_CREA_RESPONSES" not being released.
  • The status of the supplier evaluation requests remained in an incomplete state (status value 02 of field LIFECYCLE_STCD in table "/SRMSMC/D_SEV_RT").

Resolution

  1. Verify that evaluation requests have been created in the table "/SRMSMC/D_SEV_RT".
  2. Check the status of the evaluation requests in the field "LIFECYCLE_STCD" in the table "/SRMSMC/D_SEV_RT".
  • Status value 02 indicates that the evaluation request is incomplete.
  1. Locate the jobs scheduled for the report "/SRMSMC/R_SEV_CREA_RESPONSES" in the SM37 transaction.
  2. Release the jobs to execute the report and process the evaluation requests.
  3. Confirm that the status of the evaluation requests changes to "Submitted" (status value 03).
  4. Log in with the appraiser's user ID and verify that the evaluation requests are displayed in the "Evaluate Suppliers" FIORI application (ID F1650).
  5. Ensure that all jobs scheduled for "/SRMSMC/R_SEV_CREA_RESPONSES" are released and executed.

See Also

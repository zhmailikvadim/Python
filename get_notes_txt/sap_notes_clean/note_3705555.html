Symptom

The KSeF (Krajowy System e-Faktur) number is a mandatory electronic invoice identifier for supplier invoices in Poland. As regulatory requirements evolve, organizations using SAP S/4HANA and SAP S/4HANA Cloud Private Edition must ensure that supplier invoices include the KSeF number and that it is correctly stored within their ERP system.

This article provides a practical guide for customers on how to add and store the Polish KSeF number using standard invoice processing scenarios, like for example BAPI_INCOMINGINVOICE_CREATE1 and the BADI implementation as per SAP Note 3662756 (BADI_MM_SI_ODN_MODIFY).

Environment

  • SAP S/4HANA
  • SAP S/4HANA Cloud Private Edition

Resolution

Step 1: Use Standard Scenario for Supplier Invoice Creation

Start by creating supplier invoices using one of the standard scenarios. The most common automation are interfaces like BAPI_INCOMINGINVOICE_CREATE1, which allows programmatic creation of supplier invoices in the Materials Management (MM) module. This BAPI is widely used for integrating external invoice data into SAP.

When calling the interface, fill in all required invoice details as usual. However, at this point, the KSeF number (a unique identifier for Polish e-invoices) is most likely not a standard field in the structure. Therefore, it needs to be captured and stored using a custom enhancement in the subsequent steps.

Step 2: Implement the BADI_MM_SI_ODN_MODIFY for Overwriting User-Input ODN (SAP Note 3662756)

SAP Note 3662756 introduces a Business Add-In (BADI) that allows to overwrite the user-inputted Official Document Number (ODN) for MM supplier invoices. This enhancement is crucial for scenarios where the KSeF number, received from the external system or supplier, must be accurately reflected in the SAP invoice record.

  1. Review SAP Note 3662756:
  2. Access SAP Support Portal and review the note for prerequisites and implementation steps.
  3. Download and apply the note to the system, if not already in place.
  4. Implement the BADI:
  5. Go to transaction SE18 or SE19 to access the BADI implementation interface.
  6. Locate the relevant BADI as described in SAP Note 3662756 (typically MM_OM_ODN_SUPPL_INV or similar).
  7. Create own implementation or enhance an existing one.
  8. Enhance the BADI Logic:
  9. Within the BADI method, read the KSeF number from incoming data (for example, from custom fields, BAPI extension structures, or a custom table).
  10. Overwrite the ODN in the supplier invoice with the KSeF number according to business rules.
  11. Ensure proper error handling and logging for traceability.

Step 3: Validate and Store the KSeF Number

After implementing the BADI_MM_SI_ODN_MODIFY, every supplier invoice needs to trigger the BADI logic. The KSeF number will be stored as the Official Document Number (ODN) in the supplier invoice record in SAP S/4HANA. The result can be validated by:

  • Checking the invoice document in the SAP GUI or Fiori app for the presence of the correct KSeF number in the ODN field.
  • Running reports or queries on supplier invoices to ensure KSeF numbers are consistently stored and retrievable.

Best Practices and Recommendations

  • Always test BADI implementation in a quality or sandbox environment before deploying to production.
  • Ensure all stakeholders (IT, finance, compliance) are aware of the changes and how to verify KSeF number storage.
  • Document the enhancement for future maintenance and audits.
  • Monitor SAP Notes for potential updates or changes to KSeF requirements or technical implementations.

Conclusion

By leveraging the standard supplier invoice processing scenario (such as BAPI_INCOMINGINVOICE_CREATE1) and implementing the BADI as outlined in SAP Note 3662756, customers can ensure that the Polish KSeF number is accurately stored in SAP S/4HANA or SAP S/4HANA Cloud Private Edition systems. This approach helps maintain compliance with Polish e-invoicing regulations and supports efficient, automated invoice processing.

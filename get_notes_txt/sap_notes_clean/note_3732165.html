Symptom

HANA Client connection using a windows domain user failing with error:

connect via secure store failed, rc = -10104 (Invalid value for KEY, directory search path: *** ERROR => Connect to database failed. rc=-10909. rcSOL=-10104
[dbhdbsql.epp,00000]    *** ERROR => Connect to database failed, rc=-10909, rcSQL=-10104
[ dev tre,00000] SQLCODE : -10104
dev tre,00000] SQLERRTEXT : Invalid value for KEY, directory search path: C:\ProgramData\.hdb\APPLICATIONSERVER\S-l-5-21-2301267037-2\ [ dev tre,00000]    65493447-633617772-32023
no connect possible: "DBMS - HDB    --- SERVER = " PORT = '

Environment

  • SAP HANA Platform
  • SAP HANA Client for Windows

Reproducing the Issue

  1. Change Windows services to run with a domain service account instead of local accounts (for example, service user for the application server).
  2. Update related environment variables for the service and administrator users.
  3. Attempt to connect to the database using R3trans -d or hdbsql -U DEFAULT.
  4. Observe errors about missing connection data for DEFAULT and invalid value for KEY.

Cause

  • The secure store file SSFS_HDB.KEY was missing in the new secure store directory used by the domain service account.
    Confirm by login as domain user and run: hdbuserstore list
  • The "KEY FILE" is not present/defined in the hdbuserstore for the domain service account
  • Perform the same steps for the local account will show a "DATA FILE" and "KEY FILE" path.

Resolution

  1. Set the DEFAULT key for the domain service account using an elevated Command Prompt (Run as Administrator):
    hdbuserstore.exe -u "DOMAIN\SERVICE_USER" SET DEFAULT "host:3<INSTANCE>15" "DB_USER" "PASSWORD"
  2. Login as domain user and run "hdbuserstore list".
  3. The DATA FILE path is displayed. Note this down.
  4. Login as local user account and run "hdbuserstore list" again.
  5. The path to the KEY FILE should no be displayed.
  6. Copy over the SSFS_HDB.KEY file from the path you got in step 5 to the domain user account path you got in step 3

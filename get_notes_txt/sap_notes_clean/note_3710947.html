Symptom

02/20/2026: With the previous version of this SAP Note, the runtime error GETWA_NOT_ASSIGNED occurred if the PHI data is split during the year.

---------------------------------------------

February 18, 2026: In the old version of this SAP Note, the manual pre-implementation steps for creating entries in the views V_T77PAYDE_NP_ST and V_T77PAYDE_NP_SC were missing.

---------------------------------------------

When you execute the report ELStAM PHI: Process Notifications  (RP_PAYDE_E2_PKV_PROCESS_NOTIFS), the system processes several notifications for a personnel number for the same validity period.

For example, you have registered an employee using ELStAM. With the next collection, the response to the registration and a month list with data for the employee are provided. RP_PAYDE_E2_PKV_NOTIFS processes both notifications, whereby the data from the registration list is already obsolete and is overwritten by the newer PHI data of the month list.

Solution

Implement the attached correction instructions or import the specified support package.

After you implement the corrections, the system processes only the latest notification for the validity period. All other notifications for the same validity period are set to the status Obsolete. For these, the following information is displayed in the output log under Person-Specific Messages:

"Notification with GUID <GUID> set to obsolete because newer PHI data exists" (HRPAYDE_E2_PKV 046).

After you import the HR Support Package or implement the correction instructions, you do not have to perform any further follow-on activities.

Symptom

The issue related to delays in launching the configuration in ECC/CRM/S4 scenarios as observed in more detail.
The primary cause appears to be that each instance maintains its own cache region. With the KB launch, only a few cache regions are refreshed, while others are not. As a result, the configuration launch does not consistently retrieve KB data from a refreshed cache region—leading to inconsistent performance. At times it loads quickly, and at other times slowly, as confirmed through the logs.

Other Terms

KB LOADING, SSC

Solution

To optimize performance and reduce latency during the loading process, we can consider the following points:

  1. Refine “/SLCE/*” or “/SLCC/*” calls
    Remove the conditional check for the CRM environment, since only “/SLCC/*” is required. This will eliminate an unnecessary environment-identification call.
  2. Enhance CFG_API_PERF_FIND_KB
    Modify this call to ensure that additional calls related to COMM_CFGPROF and COMM_CFGOCLASS can be removed.
  3. Fix the updateKnowledgebase API
    Ensure the latest KB version effective from the current date is selected and uploaded into the cache, especially when the KB name is provided as an input parameter.

By using the approach described in point number 3, the customer can refresh the KB data across all cache regions by running a scheduled job (as outlined in SAP Note 2714406) during off‑peak hours.
This ensures that the cache already contains updated KB data, resulting in improved configuration loading times.

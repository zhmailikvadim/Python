Symptom

On a Windows platform, a Java agent and supporting software are installed on the customer’s system, and the Java agent is consumed by SAP Analytics Cloud to support the following features:

  • Acquire data from On-Premise SAP BW systems
  • Acquire data from On-Premise SAP BPC MS systems
  • Acquire data from On-Premise SAP Universe
    • SBOP 4.3 requires version 1.0.354 or higher.
  • Acquire data from On-Premise SAP ERP Central Component 6.0
  • Acquire data from On-Premise SAP HANA Views
  • Acquire data from On-Premise JDBC Datasources
  • Acquire data from On-Premise File Server
  • Export data to On-Premise File Server

Reason and Prerequisites

With the 2024.02 release of SAP Analytics Cloud, SAP Analytics Cloud agent’s minimum version remains at 1.0.75, with the following exceptions:

  • If you require data acquisition from SAP ERP / SQL Database, the minimum version is 1.0.99.
  • If you require data acquisition from SAP Universe (UNX), the minimum version is 1.0.355, the SAP Analytics Cloud agent requires additional setup steps, for more details see SAP Note 3262098
  • If you require data acquisition from SAP Business Warehouse (BW), the minimum version is 1.0.353.
  • If you require data acquisition from an SAP HANA view, the minimum version is 1.0.235.
  • If you require data acquisition from File Server, you need to upgrade to version 1.0.248.
  • Note: Although it is not required, you can upgrade to the agent version 1.0.397, no matter what data acquisition type you use. Do this to take advantage of updates made to SAP Analytics Cloud agent.

List of Changes

  • File Server Export: Implement Retry

Solution

  1. Download SAP Analytics Cloud Agent Simple Deployment Kit from SAP Software Downloads.
  2. Unzip the contents to your local file system, and follow the instructions in the README.txt included in the .zip, and the Post-Setup Guide available in the deployment folder.
  3. Please upgrade to this version at the earliest time as this version contains an improvement for better handling of large data volume for BW acquisitions, and please ensure that Tomcat has at least 4GB of memory.

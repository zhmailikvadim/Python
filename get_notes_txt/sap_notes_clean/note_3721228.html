Symptom

A lot of impex related errors would be observed during the data migration, when upgrading from jdk 17 version to jdk 21 resulting in a deployment failure. 

level":"ERROR","loggerName":"de.hybris.ant.taskdefs.AbstractAntPerformable","message":"Error occurred while performing ant task","thrown":{"commonElementCount":0,"localizedMessage":"Impex import failed for : '00006VSI-ImpEx-Import(8806386762229)'","message":"Impex import failed for : '00006VSI-ImpEx-Import(8806386762229)'","name":"de.hybris.platform.core.initialization.SystemSetupException","extendedStackTrace":[{"class":"de.hybris.platform.impex.jalo.ImpExManager","method":"handleImportResultInInitialization"

Environment

CCV2

Reproducing the Issue

  1. Initiate an upgrade deployment from jdk 17 to jdk 21.
  2. The deployment would fail in the data migration step with a lot of impex errors. 

Cause

  • The property `system.setup.create.data.fail.on.error`by default was set to false in jdk 17, where all the impex related errors were unnoticed and ignored, however in jdk 21, the value of the property is set to true by default which in-turn is resulting in the errors causing the deployment to fail. 
  • Please refer the below help doc which talks about the change in the property value mentioned above, 
    https://help.sap.com/docs/SAP_COMMERCE_CLOUD_PUBLIC_CLOUD/75d4c3895cb346008545900bffe851ce/cfe79424eccc478d96b03d7194392e39.html?ai=true

Resolution

  1. Add/set the property system.setup.create.data.fail.on.error=false in hcs admin
  2. Initiate the deployment with migration and it should be successful. 

Important note: Be aware that by setting this property system.setup.create.data.fail.on.erro to false can result in unpredictable data inconsistencies. Ensure you fully understand the implications and exercise extreme caution before applying this configuration in Prod env.

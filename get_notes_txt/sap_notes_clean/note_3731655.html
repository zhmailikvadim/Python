Symptom

I am able to access Core Administration even though I have not been explicitly assigned Core Administration role. 

Cause

The user has been assigned to a user group that grants permissions for Core Administration, resulting in access being inherited through group membership. 

Resolution

The user has been assigned to a user group (e.g., Tax Manager group) that grants permissions associated with Core Administration. 
Review the user’s group assignments in Core Administration → User Manager.

  • Identify any groups that provide Core Administration permissions.
  • Remove the user from such groups if the access is not required.
  • Alternatively, adjust the group permissions to restrict access as per business requirements.

Once the relevant group assignment is removed or updated, the user will no longer have access to Core Administration.

 

Keywords

Groups, user roles, Core Administartion Access.

Symptom

The secondary system fails to register with the primary site during SAP HANA System Replication setup.
Following error can be found in secondary site.

******

unable to contact primary site host <hostname>:4<instance number>06. connection refused (),location=<hostname>:4<instance number>06. Trying old-style port (port offs et +100)...<hostname>:4<instance number>06

unable to contact primary site; to <hostname>b:3<instance number+1>06; original error: connection refused (),location=<hostname>:3<instance number+1>06; 

******

Environment

SAP HANA, platform edition 2.0

Reproducing the Issue

  1. Attempt to register the secondary site for System Replication.
  2. Observe the error message "connection refused" during the registration process.

Cause

Following parameter is set to .local accidently.

global.ini
[system_replication_communication]
listeninterface = .local 

As explained in following guide, the value for this parameter should be set to either ".global" or ".internal". If the replication ports are open but listened on the localhost interface only, they would be unreachable from the secondary site. 
Secure Configuration of the Network Connection 

Resolution

    Unset the incorrect parameter, so that the default value ".global" takes effect.
    e.g.

    alter system alter configuration ('global.ini','system') unset ('system_replication_communication','listeninterface') with reconfigure;

    Otherwise, set it to ".internal" and also set parameter global.ini/[system_replication_hostname_resolution]

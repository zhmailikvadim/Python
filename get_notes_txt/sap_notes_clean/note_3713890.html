Symptom

You may observe the following symptoms:

-the dummy action is active in the database despite all its permissions are inactive

-the generated dummy action rule for the dummy action is active eventhough all its permissions are inactive

Other Terms

ARA, GRC, function maintenance, dummy action, critical permission risk, active inactive flag, status

Reason and Prerequisites

The dummy action was always set to active in the DB regardless of the status of its permissions. Consequently, the generated dummy action rule was also always incorrectly active.

Possible scenarios:
-all permissions are set as inactive for a dummy action -> dummy action and the generated rule must be inactive
-one of the permissions are set as active for a dummy action -> dummy action and the generated rule must be active
-all the permissions are set as active for a dummy action -> dummy action and the generated rule must be active

Solution

Please implement this correction or the equivalent SP.

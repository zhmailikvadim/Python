Symptom

  • Receiving short dump during OAA reservation call.
  • Error message: "A data record is to be inserted even though a data record with the same primary key or same unique secondary key exists.; INSERT on DB table /OAA/ATP_RESV failed.; Unique index 'EID' violated by EXTERNAL_ITEM_ID=<hexadecimal number>."
  • Runtime error: MESSAGE_TYPE_X_TEXT for ABAP program: /BOBF/CL_DAC_DIRECT_UPDATE
  • Short dump details: "The current application has intentionally triggered a termination with a short dump."

Environment

  • Omnichannel Article Availability

Cause

  • The issue occurs due to the reuse of EXTERNAL_ITEM_ID values, which violates the unique index 'EID' in table /OAA/ATP_RESV.

  • Since CARAB 5.0, the OAA /OAA/ATP_RESV_H and /OAA/ATP_RESV_I were unified in table /OAA/ATP_RESV.  This table contains a unique index for field EXTERNAL_ITEM_ID.

Resolution

  1. Ensure that the OAA interface does not reuse EXTERNAL_ITEM_ID values.
  2. Coordinate with the relevant teams to ensure that unique EXTERNAL_ITEM_ID values are always sent and never reused.

Symptom

SUM fails in phase MIGTOOL_PREPROCESSING/SUBMOD_MIG_PRP/EU_CLONE_PRP with error such as:

3EETQ399 Inconsistency found: ABAP table '/BI0/0100000040' does not exist on the database, but in the ABAP Dictionary, cloning will fail.

The same error can happen for other tables such as: /BI0/0200000002, /BI0/0200000003, /BI0/0200000004, /BI0/0200000005, /BI0/0200000006, /BI0/0200000007...

Environment

SAP Business Planning and Consolidation.

Cause

Temporary BW-generated tables from the BPC Application Shell (/BI0/*) exist in the ABAP Dictionary but not on the database, causing inconsistencies detected by SUM during EU_CLONE_PRP.

Resolution

  1. Decide whether the affected /BI0/* tables are needed.
  2. If the tables are not required: delete each affected /BI0/* table in SE11 (ABAP Dictionary). When prompted for the Object Key during deletion, enter the table name and record the change in a transport if applicable.
  3. If the tables are required: recreate the missing database objects in SE14 (Database Utility) for each affected table by choosing the option to create the database object.
  4. After cleanup or recreation, rerun the SUM phase (continue from EU_CLONE_PRP) and confirm the logs no longer report 3EETQ399 inconsistencies.

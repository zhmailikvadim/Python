Symptom

  • Customer is not using SAP's Default SMTP but rather their own MTA Provider
  • As a workaround, the customer resends the email via API call - KBA 3543859 - Email API not sending emails out of SAP Sales Cloud V2 
  • Customers observe the following behavior:
    • Outbound email show Transmission Status = Failed but no actual error message the Outbound Email Monitoring. The email is not delivered to the end recipient. Or
    • Email shows as Sent, but timeline is not updated and recipient does not receive the email

Environment

  • SAP Service Cloud V2
  • SAP Sales Cloud Version 2

Reproducing the Issue

  • Possible scenarios:
    • Email sent normally from SAP Sales or Service Cloud via the UI
    • Email sent out via Email API

Cause

Scenario A: Email Sent via SAP Sales Cloud V2 UI

  • Possible Causes:
    1. Email not successfully processed
      • Internal processing failure before SMTP handover.
      • Errors may not be exposed in the UI.
    2. MTA handover failed (Customer SMTP only)
      • MIME could not be handed over to customer’s SMTP server.
      • No timeline entry is created.
    3. Email did not trigger due to user input issues
      • Missing or incorrect recipient addresses.
      • Incomplete case context.

Scenario B: Email Sent via API (POST to email-service)

  • Possible Causes:
    1. Email interaction not created
      • Payload missing required objects:
        • Case
        • Business Partner
    2. Incorrect transmission status used
      • transmissionStatus: "SEND" used instead of "CREATE", skipping creation logic or vice-versa.
    3. Validation failed but no error surfaced
      • Backend rejects the email interaction silently if structure is invalid or not linked properly.
    4. SMTP handover failed
      • If MIME generation or MTA handover fails, the timeline will not update.

Resolution

  • For UI-based Emails 
    1. Validate Context: Confirm that the recipient and related data (e.g., Case, Business Partner) were valid at the time the email was triggered.
    2. Check Outbound Email Monitoring
      • Status: Sent (no errors)
        → Email MIME was successfully handed over to the MTA provider.
      • Status: Failed (with errors)
        → Review the error details. Search for a relevant KBA or raise a case with SAP Support.
      • Status: Failed (no error, reprocessing ineffective)
        → Raise a case with SAP Support.
    3. If Using a 3rd-Party SMTP Provider
      • Check your MTA logs to confirm email receipt for processing.
      • If no logs are found and no error is displayed in Service Cloud:
        → Raise an SAP Support case with the following:
        • Case ID
        • Timestamp of attempted send
        • User ID
  • For API-based Emails
    1. Validate your API payload
      • Must include:
        • caseReference
        • businessPartner
        • Valid Email channel and direction
      • Use:
        • "transmissionStatus": "CREATE"
          • will write to the timeline, and associate to a BP and related objects based on the payload
          • will not actually sending the email out
          • entry will not be displayed in the Outbound Monitor.
        • "transmissionStatus": "SEND"
          • will write to the timeline, and associate to a BP and related objects based on the payload
          • will send the email out to the end recipient
          • entry will be displayed in the Outbound Monitor
      • Also refer to the following KBAs:
        • KBA 3543859 – Email API not sending emails out of SAP Sales Cloud V2
        • KBA 3472917 – Issues while creating email interactions over the REST API email-service
    2. Check Outbound Monitoring:
      • If not visible, the interaction was never created or failed silently.
    3. Do not reprocess via API unless the issue is resolved:
      • Reprocessing without root cause may cause:
        • Duplicates
        • Corrupt case states 
        • Affect other backend processes
    4. Raise a support ticket with:
      • API payload
      • Timestamp
      • Associated Case ID and related object details
  • Additional Recommendations
    • Monitor SMTP server logs for custom MTA setups.
    • If timeline entry exists but the email wasn’t delivered, the issue is likely with the SMTP provider and beyond SAP’s tracking.
    • SAP is actively evaluating enhancements to improve visibility into final delivery outcomes and error transparency for both UI and API-triggered emails.

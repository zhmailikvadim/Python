Symptom

When executing the standard change document program (RHCDOC_DISPLAY), the system triggers a short dump with error CX_HRBAS_INVALID_INFOTYPE. This typically occurs during the synchronization or reporting process involving infotypes, where the system cannot resolve the correct table associated to a specific infotype.

Environment

  • SAP S/4HANA on-premise
  • SAP S/4HANA private cloud edition
  • SAP ERP
  • SAP CRM
  • SAP SRM

Reproducing the Issue

  1. Execute the standard change document report (for example, RHCDOC_DISPLAY).
  2. The program terminates with a short dump, indicating a problem related to an invalid infotype.
  3. Dump analysis reveals the mismatch between table/view entries (T777D and T77CDOC) for the affected infotype.

Cause

This error is caused by inconsistent customization in the related infotype tables:

  • The view T77CDOC contains an infotype entry that is not present in view T777D.
  • This can happen if the infotype was introduced by an add-on that was later uninstalled, or manual table changes were made without synchronization.
  • As a result, the system is unable to determine the correct table for the infotype when the report runs.

Resolution

To resolve the error, ensure consistency between T77CDOC and T777D as follows:

Option 1: If the Infotype is No Longer in Use

  • Remove the entry for the affected infotype from view T77CDOC using transaction SM30.

Option 2: If the Infotype is Still in Use

  • Ensure that a corresponding entry for the infotype exists in view T777D.
  • If this entry exists in other systems, create or transport the entry into the affected system.
  • Note: SAP does not deliver all infotype entries by default—custom entries may need to be manually maintained.

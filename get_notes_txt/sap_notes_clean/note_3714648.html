Symptom

Using the Matrix Relationship for Position table will result in duplicate values in the report.

Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.

Environment

  • SAP SuccessFactors HCM Suite
    • Story Report

Reproducing the Issue

  1. Use Position table, there are no duplicate values
  2. Add Matrix Relationship for Position table
  3. Preview the query, see that there are duplicated values even when all the rows have the same value

Cause

To address a legal requirement it was decided to include all workflow status values (N, P, and PH) for hard joins.
Because of nature of auto join type, final query returns duplicate records so it was requested to apply only N as default filter, however SAC is not supporting that.
An enhancement request was already created for the behaviour.

Resolution

An ongoing enhancement is being tracked in FPA97-117.
CSM can be contacted to monitor FPA97-117 to track the progress of this enhancement.

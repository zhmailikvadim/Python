    HTTP/1.1 409 
    OData-Version: 4.01
    X-Content-Type-Options: nosniff
    X-XSS-Protection: 0
    Cache-Control: no-cache, no-store, max-age=0, must-revalidate
    Pragma: no-cache
    Expires: 0
    Content-Type: application/json
    Content-Length: 124
    Date: Sun, 08 Feb 2026 07:51:12 GMT
    Connection: close

    {
      "error": {
        "code": "409006",
        "message": "Unique constraint violated when inserting or updating entity 'AdminService.Products'"
      }
    }

  • CDS log shows:

    2026-02-08T07:51:12.684Z  INFO 132110 --- [io-8080-exec-10] c.s.c.a.o.v4.processors.CdsProcessor     : Exception marked the ChangeSet 13 as cancelled: Unique constraint violated when inserting or updating entity 'AdminService.Products' (service 'PersistenceService$Default', event 'CREATE', entity 'AdminService.Products')

Environment

  • CAP Java
  • HANA Cloud

Reproducing the Issue

POST or Create Entity to CAP Service

Cause

Duplicate key violation

Resolution

  1. Check the key definition of relevant entity.
  2. Ensure the POST / Create entity will not viloate the duplicates.

Keywords

HTTP/1.1 409, 409, c.s.c.a.o.v4.processors.CdsProcessor, Unique constraint violated when inserting or updating entity, 409006

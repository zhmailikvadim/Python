Symptom

  • Unable to create a new WBS element after project release when fund and grant fix indicators are active (validity maintained) and planned start date and planned finish dates of the WBS elements are maintained later.
  • Error: "Fund validity does not overlap with WBS element"
  • Error: "Grant validity does not overlap with WBS element"

Environment

SAP S/4HANA Cloud Public Edition

Reproducing the Issue

  1. Release a project.
  2. Ensure fund and grant fix indicators are active and the assigned fund/grant have valid validity periods.
  3. In app F3799 or F3822, create a WBS element without entering Planned Start and Planned Finish during creation.
  4. Save the WBS.
  5. Observe the errors "Fund validity does not overlap with WBS element" and/or "Grant validity does not overlap with WBS element".

Cause

The system behavior is working as designed.

When a WBS element is created with Fund and Grant (with Fix Indicators active, validity maintained), the system must validate that the WBS planned dates fall within the validity period of the assigned funding objects.

If the WBS is created without Planned Start and Finish dates, the system does not yet have a complete time interval to perform this validation logic. As a result, when the validation is triggered later, the system may raise a validity overlap error as the system cannot ensure if the newly created WBS planned start and planned finish fits the Fund and Grant validity.

Resolution

To ensure successful validation, we recommend maintaining the Planned Start and Finish dates at the time of WBS creation, so the system can immediately verify alignment with the Fund and Grant validity periods.

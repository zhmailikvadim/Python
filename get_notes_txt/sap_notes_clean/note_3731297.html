Symptom

In case there is already a workload class created with statement level limits and you would like to use total limits as well, a new parent workload class can be created.


Environment

SAP HANA Database 2.0

Reproducing the Issue

Example:
There is an existing workload class with name "MyWLC" that has 'STATEMENT THREAD LIMIT' set as 4.
This workload class already has a mapping to a user or application you would not like to change.
The only goal is to add total limits, in this case 'TOTAL STATEMENT THREAD LIMIT'.

Resolution

Use the following commands to create the parent workload class and set it as parent of "MyWLC":

CREATE WORKLOAD CLASS "PARENT_MyWLC" SET 'TOTAL STATEMENT THREAD LIMIT' = '16';
ALTER WORKLOAD CLASS "MyWLC" parent "PARENT_MyWLC";

This way "MyWLC" will inherit the 'TOTAL STATEMENT THREAD LIMIT' of 16 from "PARENT_MyWLC".
After this change the workload class will allow only 4 running threads per statement.
At the same time regadless the number of statements running at the same time, maximum 16 running threads will be allowed.

In case the relationship between the parent and child workload class needs to be removed, use the following command:

ALTER WORKLOAD CLASS "MyWLC" parent null;

Similarly above steps can also be used for 'STATEMENT MEMORY LIMIT' and 'TOTAL STATEMENT MEMORY LIMIT' to create total statement memory limit on top of the already set statement specific memory limit.

Symptom

  • Views which query uses a common table expression (CTE) should be dropped and created again when the underlying table has been recreated.
  • SAP PowerDesigner only drops then recreates the table.
  • If the view has not been created with a CTE, it drops the view and the table, then recreates the table and the view.

Environment

  • SAP PowerDesigner (PD) 16.6 SP02
  • IBM DB2 Version 11 for z/OS

Reproducing the Issue

  1. Select File > New Model to open the New Model dialog.
  2. Select Physical Data Model in the left-hand pane.
  3. Select Physical Diagram in the right-hand pane.
  4. Select IBM DB2 Version 11 for z/OS in the list DBMSs.
  5. Click OK.
  6. Click the Table tool from the toolbox to create one table TABLE_1.
  7. Double-click the table to edit it, click the Columns tab to create one column COLUMN_1, and click OK.
  8. Click the View tool from the toolbox to create one view VIEW_1.
  9. Double-click the view to edit it, click the SQL Query tab to enter the following code example:

WITH CTE (COLUMN_1) AS
   (SELECT COLUMN_1 FROM TABLE_1)
SELECT COLUMN_1 FROM CTE

  1. Click OK.
  2. Select File > Save As, and select Archived PDM (bin) or Archived PDM (xml) in the Save As Type list.
  3. Type a file name, and click Save.
  4. Double-click the table to edit it, and click the Columns tab to add a second column COLUMN_2.
  5. Click OK to close the property sheet.
  6. Select Database > Apply Model Changes to Database.
  7. Enter a destination directory and file name for the script file.
  8. Select the Using an archive model radio button to specify how PowerDesigner will determine the changes to apply.
  9. Click the Select File tool to browse to open the archived model saved at step 12.
  10. Select the Always use create statements option, and click OK.
  11. The Database Synchronization window displays the anticipated result. 



    Click OK to accept changes.
  12. Edit the generated file:

--==============================================================
-- DBMS name: IBM DB2 Version 11 for z/OS
--==============================================================

drop table TABLE_1;

create table TABLE_1 (
   COLUMN_1   CHAR(10),
   COLUMN_2   CHAR(10)
);

Cause

The issue has been identified and logged under CR 804995. Its description reads, "DB2: View with a CTE is not recreated when the underlying table is recreated".

Resolution

Upgrade to SAP PD 16.6 SP05 PL02 or higher where SAP CR 804995 is fixed for IBM DB2 Version 11 for z/OS.

See Also

KBA 3434651 - A blank script is generated after modifying the CTE expression of a DB2 view

Symptom

OData API call is failing with following error while performing UPSERT operation on SFOData.TimeAccountDetail entity.

         Invalid property names: TimeAccountDetail/bookingAmount

Environment

SAP SuccessFactors HXM Suite

Employee Central SFAPI & OData Entities

Reproducing the Issue

.

Cause

The OData API is not designed to create time account details with Entitlement or Accrual booking type. 

Resolution

  1. Use the OData API to create time account details with the `bookingType` set to Manual Adjustment.
  2. It is important to note that the OData API is not designed to create time account details for `bookingType` values such as Accrual or Entitlement, as these are calculated outside the system. The Employee Central Time Off module does not support this specific use case.

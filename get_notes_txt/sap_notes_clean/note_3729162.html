Symptom

During the PTP replication of an employee, it was observed that the IT0008-DIVGV field is calculated based on the data in IT0007.

However, this calculation is only triggered when there are changes such as insert, update, or delete operations in IT0008. If there are no changes in IT0008, even if the work schedule rule in IT0007 is modified, the DIVGV field in IT0008 remains unchanged. 

Environment

  • SuccessFactors Employee Central Payroll

Reproducing the Issue

  1. Change the work schedule rule in IT0007 for an employee in EC and trigger a EE master data replication.
  2. Observe that no update occurs in the DIVGV field of IT0008 unless there are changes in IT0008.

Cause

The standard logic does not automatically trigger a split in infotype 0008 when changes occur in infotype 0007. Therefore, the DIVGV field in IT0008 is not updated based on changes in IT0007 unless there is a direct change in IT0008.

Resolution

To address this issue, follow these steps:

Enhance or create a new implementation for BAdI HRSFEC_B_CE_CHANGE_IT0008:

  1. In the BAdI implementation, add logic to detect changes in infotype 0007 that should trigger a corresponding change in infotype 0008.
  2. Create a split in infotype 0008 with the new record containing an initial DIVGV field.

With this enhancement, the standard logic will update the DIVGV field in the new infotype 0008 record based on the changed planned working times in infotype 0007.

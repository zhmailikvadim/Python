Symptom

  • In the FI_MKKIPL archiving object, the SPOOL “Overview of Entries Read and Archived” shows that table DFKK_IPL_CTRL has a much lower “Viewed” count than “Archived”.
  • Other tables (for example, DFKK_IPL_RATES and DFKK_IPL_ITEMS) show identical or aligned “Viewed” and “Archived” counts, while DFKK_IPL_CTRL does not.
  • No error messages or dump codes are displayed.
  • The behavior is observed during productive-mode test runs and is reported as a display/counter mismatch only (no functional issues).

Environment

  • FI-CA (Contract Accounts Receivable and Payable)

Reproducing the Issue

  1. Start program RFKKIPLAR01 (Write Program) for archiving object FI_MKKIPL in a test run.
  2. After completion, open the SPOOL protocol “Overview of Entries Read and Archived”.
  3. Observe that for table DFKK_IPL_CTRL, the “Viewed” number is significantly lower than the “Archived” number, while other related tables show matching counts.

Cause

  • By design, FI_MKKIPL uses DFKK_IPL_RATES as the leading table and has a 1-to-many relationship between DFKK_IPL_CTRL (control) and DFKK_IPL_RATES (rates).
  • Logging counts differ: “Viewed” reflects the number of actual selects from DFKK_IPL_CTRL, while “Archived” reflects the number of times a CTRL entry is written into archive data objects.
  • For performance and data completeness, the CTRL entry is buffered and copied per data object, so the archived count for DFKK_IPL_CTRL can be higher than the viewed/read count and higher than the physical table entry count.

Resolution

  1. Acknowledge that the DFKK_IPL_CTRL counter discrepancy is expected due to the data model and logging logic in FI_MKKIPL.
  2. Interpret the protocol with the understanding that:
    1. “Viewed” counts the number of times CTRL entries are selected.
    2. “Archived” counts the number of times CTRL entries are written per data object (can be multiple times for one CTRL).
    Proceed with archiving operations as functionality is not affected.
  3. If verification is needed, run a small test in RFKKIPLAR01 and trace how “Viewed” and “Archived” are updated for DFKK_IPL_CTRL in relation to DFKK_IPL_RATES and DFKK_IPL_ITEMS.

Symptom

Closed or deleted Purchase Requisitions are integrated to IBP via RTI when running CFM9.

Environment

  • SAP Integrated Business Planning for Supply Chain 

Reproducing the Issue

  1. Create an RTI integration model for purchase requisitions
  2. In the source system, use a purchase requisition where the closed flag is set but the deletion indicator is not set, and an open quantity remains (requested quantity greater than ordered quantity)
  3. Run the initial load/transfer (for example, via the corresponding integration model execution)

Cause

By design, the selection logic transfers purchase requisitions that do not have the deletion indicator set and still contain open quantities, even if they are flagged as closed. This behavior occurs during initial load and change processing (for example, via CFM9/RTI).

Resolution

  1. Integrating closed Purchase Requisitions without a deletion indicator and with open quantity is standard behavior in RTI.
  2. If the intention is to exclude such purchase requisitions from integration, set the deletion indicator on the requisition item or ensure there is no open quantity in the source system.

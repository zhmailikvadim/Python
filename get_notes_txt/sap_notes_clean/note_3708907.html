Symptom

You are trying to connect with OData Adapter using OAuth, for example Microsoft CRM.

Inside your datastore, you selected the following parameters:

  • Authentication Type: OAuth 2.0
  • Grant Type: Client credentials
  • Client ID:  <your private value>
  • Token Endpoint: <your URL from Microsoft>
  • Client Secret: <your private value>
  • Scope: <your CRM url>
  • Default base64 Binary Field Length: 16
  • Depth: 2
  • OData Version: V4
  • Require CSRF Header: No
  • Odata Metadata Header: minimal

After hitting the test connection, you get the following error:

    • Test failed for the default configuration 'Test' of datastore: 'Your_datastore_name':
    • Timed Out, timeout : 250000 expired.. BrokerClient TimedOut for Destination : com.acta.adapter.odata.OdataAdapter.data_2 .

This issue is tracked in Corrective Measure 002075125800000548372026.

Reason and Prerequisites

The SAP Data Services Development / Adapter team has identified the root cause that the metadata - including entities and functions - due to performance intensive logic related to unbound functions metadata.

Solution

For SAP Data Services: The correction is delivered in the release(s) listed in the Support Packages and Patches section of this SAP Note.

For SAP Cloud Integration for data services: The correction for this agent issue is delivered in SAP Cloud Integration for data services version 2603 (1.0.11.6370).

In addition to the fix, you must perform the following step:

In your agentConfiguration.bat or .sh, in the field "Additional Java Launcher Options", add max heap and min heap as -Xms1024m -Xmx2048m.

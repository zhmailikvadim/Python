Symptom

  • Implementation of eDocument kSeF solution for Poland based on SAP Note 3279206.
  • Missing XML tag for displaying DRS tax value per item in the XML structure and PDF.
  • Current available tags (P_9A, P_9B, P_10, KwotaAkcyz) cannot accommodate DRS value.
  • User seeks advice on which XML tag can be used for DRS, considering it is not subject to VAT.

Environment

  • SAP ERP
  • SAP S/4HANA
  • SAP Document and Reporting Compliance Service

Reproducing the Issue

  1. Implement eDocument kSeF solution for Poland based on SAP Note 3279206.
  2. Attempt to display DRS tax value per item in the XML structure and PDF.
  3. Notice that current available tags (P_9A, P_9B, P_10, KwotaAkcyz) cannot accommodate DRS value.

Cause

No standard mapping exists for DRS tax in the desired format within the XML structure.

Resolution

Adjust mapping through custom code via BAdI EDOC_ADAPTOR method SET_OUTPUT_DATA to accommodate DRS tax value display per item in the XML structure and PDF.

See Also

Refer to: SAP Note 3279206 - Electronic Document Processing for Poland - KSeF: Implementation Overview

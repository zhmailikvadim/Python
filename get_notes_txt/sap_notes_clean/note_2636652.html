Symptom

 ASE infected with 11 in alloc_dirtychains

The stack trace contains these functions:
alloc_dirtychains
CgLavaUtils::CgCompFakeTable
CgGenLava::CGConsScanOp
CgpTabScan::_cgpCodeGen

Environment

SAP Adaptive Server Enterprise (ASE) all versions

Reproducing the Issue

  1. Start ASE with traceflag 3608
  2. Execute "select * from sysprocesses"

Cause

  • The sysprocesses system table is an example of a "fake table" that is created on the fly as a worktable in tempdb from memory structures.
  • Other examples include the syslocks, syslisteners, syscurconfigs, sysmonitors, sysengines, syslogshold, syssecmechs, sysoptions, and all the MDA monitoring tables.
  • This infected with 11 error occurs when ASE tries to instantiate a fake table and tempdb is not available for some reason.
  • ASE should raise an expected error message under such circumstances rather than encounter an unexpected segment violation.
  • The issue has been identified and logged under SAP CR 677539 - Its description reads, "ASE infected with 11 in alloc_dirtychains when attempting to select from a "fake table" (system tables created on the fly from memory structures such as sysprocesses, syslisteners, syslocks) while tempdb is not available.". 

Resolution

CR 677539 is fixed in:

  • SAP ASE 16.0 SP03 PL11
  • SAP ASE 16.0 SP04 PL02

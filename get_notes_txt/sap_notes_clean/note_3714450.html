Symptom

 Previously, users were only able to generate reports based on manually selected date ranges. There was no option to generate reports dynamically based on predefined time frequencies such as Weekly, Monthly, Quarterly, Half-yearly, or Yearly.

Other Terms

Previously, users were only able to generate reports based on manually selected date ranges. There was no option to generate reports dynamically based on predefined time frequencies such as Weekly, Monthly, Quarterly, Half-yearly, or Yearly.
Reporting, Adjustment Hub

Reason and Prerequisites

 The report selection screen did not include a Frequency parameter, and the backend logic did not support automatic date range calculation based on frequency values. As a result, users had to manually specify date ranges each time.

Solution

Implement this SAP Note 
The application has been enhanced to support frequency-based report generation.

New Functionality

  • ·Added new filter field: FREQUENCY

Supported values include current period:

  • WEEKLY
  • MONTHLY
  • QUATERLY
  • HALFYEARLY
  • YEARLY

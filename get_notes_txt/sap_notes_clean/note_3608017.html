Symptom

I am getting an error bellow when using the Detail Upload connector, even though the SOW is in Draft status and I can perform this exact same action without any issues through the UI:

Error: SOW Worker Role cannot be uploaded as no rates are defined for the SOW

Reproducing the Issue

  1. Upload a file using the SOW Worker Role Detail Upload connector.
  2. The connector file contains a valid SOW ID and role details.
  3. The upload fails, even though the user can manually assign the same role to the same SOW via the UI without issue.

Cause

This error indicates that the SOW does not yet have any rates associated to it, which are required for the SOW Worker Role upload to succeed. While Fieldglass allows roles to be added via the UI in advance of rate definition (especially in Draft status), the upload connector requires at least one rate to be defined for the SOW before a role can be successfully assigned through an integration upload.

Resolution

Before using the SOW Worker Role Detail Upload connector:

  1. Ensure that at least one rate is defined for the SOW.
    • You can verify this by viewing the SOW in the UI and confirming that a pay rate has been added under the Financials section.

  2. If the SOW was created via SOW Shell Upload, confirm that rate information was included or added afterward.
  3. Once a rate is defined:
    • Retry the SOW Worker Role Detail Upload.
    • The connector should now process the file successfully.

 This requirement applies to connector uploads only. The UI allows for more flexible sequencing during SOW draft creation.

Symptom

The approval process is retriggered when you add an already approved draft document.

Image/data in this SAP Note is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental. 

Environment

SAP Business One

Reproducing the Issue

Prerequisites:

  1. You are working in one of the following localizations: Argentina (AR), Brazil (BR), Canada (CA), Chile (CL), Costa Rica (CR), Guatemala (GT), India (IN), Mexico (MX), United States (US).
  2. You are using the SAP Business One 10.0 FP 2405 version or higher.

Scenario:

  1. In Administration → Approval Procedures → Approval Stages, create an approval stage Approval as below:
    • Authorizer = user "manager"
    • No. of Approvals Required = 1
    • No. of Rejections Required = 1
  2. In Administration → Approval Procedures → Approval Templates, create an approval template Sales Order as below:
    • Originator = user "A001"
    • Documents → Sales Order
    • Stages → Approval
    • Terms → Always
  3. Log on with user "A001" and create a Sales Order.
  4. Choose Tax Code in the row and add the Sales Order.
    The approval procedure is triggered and a draft is added.

       5. Log on with user "manager".

       6. Go to Administration → Setup → Financials → Tax → Sales Tax Codes and find the Tax Code used in the Sales Order draft.

       7. Change one of the Codes in the row setup to one with a different rate and update.
           As a result, the overall Tax Rate is different from the Tax Rate used in the Sales Order draft.

       8. Approve the draft.

       9. Log on with user "A001" and add the approved draft.

Actual behavior:

The approval process is retriggered.

Resolution

This is due to the fact that the tax code rate in the approved draft does not match the current tax code definition. Once the second approval process with the draft with the new Tax Code is approved, the final document can be added successfully.

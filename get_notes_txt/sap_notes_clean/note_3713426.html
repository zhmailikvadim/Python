Symptom

The job /IWFND/METERING_AGG, which previously ran correctly, has started to produce a dump in ST22 with runtime error COMPUTE_INT_PLUS_OVERFLOW

Category               ABAP programming error
Runtime Errors         COMPUTE_INT_PLUS_OVERFLOW
Except.                CX_SY_ARITHMETIC_OVERFLOW
ABAP: Program          /IWFND/CL_METERING_DB=========CP
Application Component  OPU-FND-CS

Environment

SAP Gateway

Resolution

First try to implement the following notes:

  • 3691927 - /IWFND/METERING_AGG is 'Cancelled' with runtime error COMPUTE_INT_PLUS_OVERFLOW (S/4 HANA 2025 SAP_GWFND 816) - SAP for Me
  • 3703202 - Performance issue in /IWFND/R_METERING_AGGREGATE - SAP for Me 

If the issue persists follow these steps:

  1. In the global configuration, set the "Metering (OData V2 only) Keeping Days for Aggregated Metering" to "X" (the default value is 90 days).
  2. For optimal performance across all clients, it is recommended to set the retention period for metering aggregated records to 14 or 30 days.
  3. Therefore, reduce the retention period for aggregated data to 15 days using transaction /IWFND/GLOBAL_CONFIG.

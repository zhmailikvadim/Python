Symptom

The selection of planning file entries (MD21) does not take into account the selection date for time-phased materials planning. This symptom occurs when you work with planning file entries in the table MDVM.

Other Terms

MD21, DBVM, planning file entry, MRP record, material requirements planning, time-phased date, RM61R-RHYTD

Reason and Prerequisites

SAP recommends the use of planning file entries in the table DBVM (that is, at the level of the MRP areas). This enables you to work with planning file entries at the MRP area level, even if you do not actually use MRP areas. The conversion of planning file entries from the table MDVM to the table DBVM is also beneficial if you do not work with MRP areas but need to plan multiple plants or if you work with large datasets (a large number of materials). The conversion of planning file entries to the new table store DBVM does not have any impact on the user or any screens.  For more information on the topic of planning file entries, refer to the entry "Convert Planning File Entries for MRP Areas" in Customizing (transaction OPP1).
The symptom described above can occur only if you work with the former logic (planning file entries in the table MDVM).

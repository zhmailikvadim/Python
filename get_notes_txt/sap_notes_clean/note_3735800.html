Symptom

***WAITING FOR VALIDATION, if proven, change status to Released to Customer and remove this statement***

Making a change to a project structure using transaction like CJ20N or CJ02 results in below timeout dump message:

Runtime Errors         TIME_OUT
ABAP: Program          SAPLCJDW
Application Component  PS-ST-OPR

Environment

  • SAP ERP Project System (PS)
  • SAP R/3
  • SAP R/3 Enterprise
  • SAP ERP Central Component
  • SAP ERP
  • SAP Enhancement package for SAP ERP
  • SAP Enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA, on-premise
  • SAP S/4HANA Cloud Private Edition

Reproducing the Issue

  1. CJ20N
  2. Open Project Structure
  3. Make a change in the project (i.e. changing status)
  4. Save

Cause

Most likely there is issue in project hierarchy which has lead to inconsistency leading to recursive calls when updating project structure. 

Resolution

  1. First run report CNHIERAR to correct inconsistencies relating to project hierarchy. See SAP Note 149698 for further information on this report.
  2. If this doesn't help, run report Z_PRHI (SAP Note 1938289 - CJ11 : Hierarchy inconsistency in the project II). Please note that this report may need to be implemented manually in SE38.

Keywords

CJ20, CJ01, CJ03, TIME_OUT, SAPLCJDW, FCJDWF00_SET_PSTAB_EX, SET_PSTAB_EX, CALL FUNCTION 'CN_LDST_ACTIVATION_CHECK'

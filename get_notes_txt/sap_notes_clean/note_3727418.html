Symptom

In urgent situations, the customer prefers not to restart the server to change the log level. For further investigation, they need to increase the log level to generate more detailed logs for issue tracing. Therefore, would like to modify the log level dynamically using a Groovy script.

"Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental."

Environment

SAP Commerce Cloud

Resolution

The end user can run the below script via HAC

*******

def targetLevel = "INFO";

def level = org.apache.logging.log4j.Level.getLevel(targetLevel);

def configuration = org.apache.logging.log4j.LogManager.getContext(false).getConfiguration();

def existingConfig = configuration.getLoggerConfig("root");

println "Current Log level of root logger is: " + existingConfig.getLevel();

println "Changing level of " + existingConfig + " from " + existingConfig.getLevel() + " to " + targetLevel;

existingConfig.setLevel(level);

org.apache.logging.log4j.LogManager.getContext(false).updateLoggers();

*******

After running successful, the log level will be changed temporarily. If any pods restart, the changes will be reverted back. 

Please verify it on your lower environments before implementing in production environment.

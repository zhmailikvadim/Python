Symptom

When posting the goods issues in a STO scenario via transaction code VL02N, the system raises the error message "8B286 - NF Number has an incorrect format", but the system determines automatically the NF Number format.

Environment

  • SAP S/4HANA
  • SAP ERP
  • Materials Management (MM)
  • Brazil Localization

Reproducing the Issue

  1. Open transaction VL02N (Change Outbound Delivery).
  2. Enter the outbound delivery number.
  3. Press "Post goods issue"

Cause

The NF type used in the process was configured with an incorrect NF model, causing the system to determine model "00" and reject the NF number format.

Resolution

To correct this error message, is necessary to verify the Nota Fiscal Model used for delivery and check if is correctly configured. In return process, is necessary check the NF Type configured in view "J_1BIM02V" for the current Movement Type.

The configuration is made via SPRO following the steps: IMG -> Cross-Application Components -> General Application Functions -> Nota Fiscal -> Define Nota Fiscal Type, check if the Nota Fiscal Model is configured with Model 55.

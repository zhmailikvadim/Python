Symptom

Invoice cannot be settled; UTILMD message contains segment(s) for a device that is not billing-relevant

In the “clear case” situation: generation of pdoc fails with an ABAP runtime error:

Runtime error in /APE/CL_PROC_FACTORY, line 8: Das Objekt konnte nicht erzeugt werden: Die Klasse /US4G/CL_PRST_CHILD_PDOC_DATA existiert nicht.

Environment

SAP IS-U

Reproducing the Issue

  • Case 1:
    1. Trigger UTILMD creation for Pdoc (sending PID 55218)
    2. Review generated UTILMD message; see if segments are present for device even though it should be excluded (not billing relevant)
  • Case 2:
    1. Process “clear case”
    2. Runtime error appears: "/US4G/CL_PRST_CHILD_PDOC_DATA does not exist"

Cause

  • Case 1: Logical or configuration error in Article determination process – Class /US4G/CL_DEF_BADI_ARTICLE_ID (method CALC_METERING_SERVICES_ARTICLE) does not properly exclude devices not marked as billing relevant, possible incorrect BAdI logic or customizing.

  • Case 2: Missing ABAP class /US4G/CL_PRST_CHILD_PDOC_DATA in the customer system, typically due to missing transport or incomplete deployment.

Resolution

  • Case 1:
    • Debug article/device relevance logic, especially in /US4G/CL_DEF_BADI_ARTICLE_ID-CALC_METERING_SERVICES_ARTICLE
    • Check for active BAdI implementations or customizations affecting device inclusion/exclusion.
    • Adjust code or customizing so only billing-relevant devices generate UTILMD segments.
  • Case 2:
    • Use SE24 to check existence of /US4G/CL_PRST_CHILD_PDOC_DATA
    • If missing, import the correct transport request for this class (contact development if needed).
    • After the class is present, re-run the process to confirm issue is resolved.

Keywords

Symptom

While recording results for an inspection lot and completing all its characteristics - for example in transaction QE01 or QE51N - , the operation is closed with a UD (usage decision) code. The UD code has a follow-up action assigned with object type 'O  For Operation Completion' and with a function module. 

The function module should be called according to the 'AftrPst' (Function Module Called After Posting) indicator, however, it does not happen. On the other hand, for "regular" UD follow-up (type 'L  Usage Decision for Inspection Lot) - for example via transaction QA11 - the same function module is executed properly.

Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.

Environment

  • SAP ERP Central Component
  • SAP ERP
  • SAP Enhancement package for SAP ERP
  • SAP S/4HANA, on-premise
  • SAP S/4HANA Cloud Private Edition

Resolution

Indicator 'AftrPst' is not supported for the operational UD process.

See relevant SAP Help 'Completing the Operations in an Inspection Lot' accordingly:

"Use
You can complete the operations in an inspection lot individually and according to requirements. This allows you to document the exact inspection process for an inspection lot. After an operation has been updated, follow-up actions can be triggered automatically, depending on your system settings. For these follow-up actions, the update indicators AfterPost. (after posting) and Sync Post. (synchronous posting) may not be set in Customizing for the Quality Inspection under Define Follow-Up Actions .

Note
Operation completion is purely informative. It does not influence the existing functions.
..."

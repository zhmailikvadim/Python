Symptom

Error "Change not allowed since account balance is not zero" when trying to remove the Alternative Account mapping for a G/L Account in the Manage G/L Account Master Data app.

Environment

SAP S/4HANA Cloud Public Edition

Reproducing the Issue

  1. Open the Manage G/L Account Master Data app.
  2. Use Adapt Filters and search for the desired G/L Account.
  3. In Company Code Assignment, try to remove the Alternative Account of a Company Code.
  4. Enter the required transport request details and try to save.
  5. The system throws the error: "Change not allowed since account balance is not zero".

Cause

The G/L Account balance is checked in Production system as well. Even if there are no postings in lower tenants, the error will be thrown if there are postings in Production.

Resolution

Although this error will occur if G/L Account balance is not zero in both Production and non-Production environments, it is not recommended to change the value for accounts with postings and balances to avoid further inconsistencies.

Keywords

Alternative account, Manage G/L Account Master Data, balance, zero, production

Symptom

After checking the "Info Update" checkbox on purchase order item level, the price of new order doesn't updated to the purchasing info record under conditions as expected, even after running the Schedule Price Update app.

Environment

SAP S/4HANA Cloud Public Edition

Reproducing the Issue

  1. Go to the Create Purchase Order app (ME21N) or Change Purchase Order app (ME22N).
  2. Enter the vendor, purchasing organization, and purchasing group.
  3. Enter a material with an existing purchasing info record (PIR).
  4. Change the net price in the item details.
  5. Check the "Info Update" box.
  6. Save the Purchase Order (PO).
  7. Go to Schedule Update of Price for Purchasing Documents app, and run the job with certain parameter.
  8. The result shows that PIR is updated with new price, however it is not displayed in the purchasing info record under conditions as expected.

Cause

The condition price displayed on PIR won't be affected/changed by any PO updates, no matter this run is executed or not. 

PO only exists as a last document reference, and its price will be automatically proposed when creating the next order add from this PIR (as reference document). 

This is how the "Info Update" check box works in the system.

For this schedule price for update run for PIR, it is used when changes made to the condition validity or condition value are not updated in the purchasing info record tables when a condition is changed. It is used to keep the Procurement documents in sync with condition changes. Not for the condition price update on PIR itself.

Resolution

This is a standard system design.

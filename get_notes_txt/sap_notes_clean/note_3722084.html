Symptom

  • You use Personalization tokens in an Email Campaign.
  • The campaign check triggers errors related to the fields using those tokens, for example:

"Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental." 

Environment

SAP Emarsys Customer Engagement

Reproducing the Issue

Access the campaign and navigate to the "Campaign check" tab

Resolution

  • When using personalization tokens instead of static values, the validation check cannot properly validate the content because actual values are populated only at send time.
  • The errors in the Content Check do not prevent the campaign from sending. 
  • To test tokens, use a Segment or a Contact List. Avoid entering an email address manually because in that case the test email will not have the personalization token value.

Symptom

  • Attempts to update a user with a PATCH request fail, including operations like changing the active status or adding/removing roles. 

  • The API call returns a 400 Bad Request error with the following response body: 
    "schemas": [
    "urn:ietf:params:scim:api:messages:2.0:Error"
    ],
    "status": "400",
    "scimType": "invalidValue",
    "detail": "Request is unparsable, syntactically incorrect, or violates schema."

Environment

SAP Analytics Cloud 2026.2.5

Reproducing the Issue

  1. Using an API client like Postman, attempt to perform a PATCH operation on an existing user to modify an attribute (e.g., assign a new role). 

Notice : The API request fails with the 400 invalidValue error described in the Symptom section. 

Cause

The issue occurs due to case sensitivity between the 'userName' attribute and the 'idpUserId' attribute for the affected user within SAP Analytics Cloud.

Resolution

Correct the 'idpUserId' for the affected user to match the case of the 'userName'. After this one-time fix, subsequent PATCH requests will succeed.

Symptom

When a production order is created using DI API, it can be created successfully even if the series and posting period do not match.

Reproducing the issue

Scenario:

  1. Create a new company, with the Period Code and Period Name set to "2023", and the Sub-Periods to "Months".
  2. Go to Administration → Setup → Financials → Period Indicators. Add two indicators, "202301" and "202302".
  3. Go to Administration → System Initialization → Posting Periods. Update the period indicator of "2023-01" to "202301", and the period indicator of "2023-02" to "202302".
  4. Go to Administration → System Initialization → Document Numbering. Open the series setup for Production Order. Add two series, "202301" with the Period Ind. set to "202301", and "202302" with the Period Ind. set to "202302".
  5. Use the query below to get the series value for "202302". For example, the value is 71.
    1. SELECT Series FROM NNM1 where ObjectCode = 202 and SeriesName = '202302'
  6. Create a bill of materials, and set the Product No. to "I001" and add a child item "I002".
  7. Add a production order using the DI API below.

    1. Dim order As SAPbobsCOM.ProductionOrders

      order = oCompany.GetBusinessObject(SAPbobsCOM.BoObjectTypes.oProductionOrders)

      order.DueDate = DateTime.Parse("2023-01-01")

      order.PostingDate = DateTime.Parse("2023-01-01")

      order.Remarks = "Created Via DI"

      order.Series = 71

      order.ItemNo = "I001"

      order.PlannedQuantity = 1

      order.ProductionOrderType = SAPbobsCOM.BoProductionOrderTypeEnum.bopotStandard

      order.ProductionOrderStatus = SAPbobsCOM.BoProductionOrderStatusEnum.boposPlanned

      order.Warehouse = "01"

      order.Add()

Expected behavior:

You can't create a production order with a document series from incorrect posting period.

Actual behavior:

﻿The production order is created using DI API, and the document series is from incorrect posting period.

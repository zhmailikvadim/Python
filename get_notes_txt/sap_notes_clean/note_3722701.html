Symptom

  • Safety Stock for Supply Volatility (I_SAFETYSTOCKSUPPLYVARIABILITY) is being calculated as 0 in a planning area based on I_SAPIBP2, which is not expected.
  • CV values are loaded into the Key Figure values for Distribution and Production Lead Time CV, and jobs were executed, but no changes occurred.
    • key figures I_DISTRLEADTIMEMEANTLANE, I_DISTRLEADTIMECVTLANE, I_PRODNLEADTIMEMEANPDS, I_PRODNLEADTIMECVPDS
  • Values were reloaded into the same attributes in the Master Data tables (PDSHEADER, TLANELOCATIONPRODUCT), and jobs were executed again, but no changes occurred.

Environment

SAP Integrated Business Planning for Supply Chain (SCM-IBP)

Cause

  • Check that the distribution and/or production quota attribute is being maintained as non-zero value.  The attributes are I_DISTRQUOTATLANE and I_PRODNQUOTAPDS.
  • Null values are treated as 0.
  • If a TLANE and/or PDS has a quota of 0, the value of LT and CV will have no impact on supply variability safety stock. 
  • If any of the output key figures like I_CALCDISTRLEADTIMEMEAN, I_CALCDISTRLEADTIMECV, I_CALCPRODNLEADTIMEMEAN and I_CALCPRODNLEADTIMECV are null, this is an indication the quota is seen as 0
  • If you have the time-dependent quota flag set to 'X' (i.e.  I_DISTRQUOTATLANE, I_PRODNQUOTAPDSKF), you must maintain a value > 0 in the corresponding key figure(s) for all necessary periods.

Resolution

  1. Maintain proper non-null values for the quota attributes.  Again, a null attribute is treated as 0.
  2. Re-run the Inventory Optimization (IO) process to verify the calculation of Safety Stock for Supply Volatility.

NOTE: it is possible that even with non-zero quotas maintained that the Supply Variability Safety Stock can still be calculated as 0 depending on other variability components.

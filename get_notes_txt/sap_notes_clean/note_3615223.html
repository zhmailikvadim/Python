Symptom

  • Implementing the long account number functionality does not reflect in F110.
  • Bank account number length exceeds 18 characters and the remaining characters, are stored in an additional field.

Environment

  • SAP ERP
  • SAP S/4HANA

Reproducing the Issue

  1. Use a bank account number whose length exceeds 18 characters and store it in the Vendor/Customer master data.
  2. Maintain the excess characters in the reference detail field.
  3. Run transaction F110.
  4. Observe that the system does not fetch the excess characters from the reference detail field.

Cause

  • The solution to store long account numbers is to split the account number using another available field in the master data, beside of the field account number. The system will then concatenate the parts of the account number again, if account number has to be represented in the full length, for instance in a DME (Data Media Exchange) payment file
  • The Countries which are supported for the above functionality can be found in the Note - 1585003 - How to enter a long account number
  • The functionality to store long account numbers by splitting them using the reference field is not applicable to all countries.

Resolution

  1. If the country is not listed in Note 1585003, refer to SAP Note 0003189739 - "Long account number in country-specific settings for S4 systems."
  2. Checking rule 0 is a special rule for excessively long account numbers. The "Account Number" field has a length of only 18 characters in some applications. To store a longer account number, you must use an additional field, whereby no validation across the entire account number is possible using the check rules 1 to 8. The check using rule 0 behaves in the same way as rule 1, but the content of the additional field Reference is also taken into account.
  3. The documentation for the functionality "Checking rule 0" can be found attached to the above note.
  4. A solution which works for both ECC and S4 systems, refer to SAP Note 0001037617 - "Customisable conversion of long account number."
  5. Use Business Transaction Events (BTE) 3070 and 3080 to customize the conversion of long account numbers to SAP fields and vice versa.
  6. Activate the use of the reference field as an addition to the account number field by specifying the country code in the event settings (transaction BF34).
  7. Follow the detailed instructions provided in SAP Note 1037617, example No. 2.

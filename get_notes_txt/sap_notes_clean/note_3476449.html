Symptom

You want to send Poland-specific electronic documents from external systems to KSeF through SAP Document and Reporting Compliance.

Other Terms

SAP Document and Reporting Compliance, SAP Document and Reporting Compliance, cloud edition, KSeF, UBL 2.3, DRCce, Electronic Document Processing Framework, Electronic Documents, External System, External Data

Reason and Prerequisites

The preparation of the XML format in the external system doesn't require any prerequisites. Later, when you want to send this XML format to SAP Document and Reporting Compliance, you'll need to implement the latest Electronic Document Processing Framework SAP Notes.

Solution

The Electronic Document Processing Framework has been extended to handle invoices created in external systems. You can send your invoices using the open-source UBL 2.3 schema, which is a generic format.

SAP provides you with a spreadsheet detailing the cardinality of the relevant tags as defined by KSeF. You can use this spreadsheet as a reference for your XML mapping. Poland uses a subset of the tags provided in the UBL 2.3.

For more details on how to fill out these tags, please refer to the attachments in this SAP Note.

SAP Document and Reporting Compliance checks the cardinality specified in the spreadsheet. If mandatory tags are not maintained, the invoice is rejected with errors.

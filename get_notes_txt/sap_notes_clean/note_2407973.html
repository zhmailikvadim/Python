Symptom

You use the Compliance Management in SAP Environmental Compliance 3.0 and you create new permits and requirement sets. If you create a new requirement set below an approval with extended validity date, the extended validity date is not transferred to the new requirement set.

 

Other Terms

﻿xEM, EC, SAP EC, EP, SAP EP, EEM﻿

﻿Emissionsmanagement, Einhaltung von Umweltschutzauflagen, Umweltleistung, Energiemanagement, Compliance-Verwaltung, Genehmigungen, Anforderungsliste﻿, Gültigkeitsdatum

﻿Emission Management, Environmental Compliance, Environmental Performance, Enterprise Energy Management, Compliance Management, Permits, Requirement Sets, Validity Date 

 

Reason and Prerequisites

This symptom is caused by a program error.

You use SAP Environmental Compliance 3.0 with a Support Package or patch level lower than the level specified in the Support Packages & Patches section.

 

Solution

Download the complete solution for your Support Package level using the appropriate link in the Support Packages & Patches section and implement it in your system. When you do this, apply the other required patches as well. For more information, see SAP Note 1974464.

Note: Patches for ﻿Env.Compl. Centr. Services 3.0﻿ require a server restart to become fully active.

If you create a new requirement set below an approval with extended validity date, the extended validity date is automatically transferred to the new requirement set.

Symptom

  • Create PR in ME51N, when copy PR item 10 to create item 20, then delete the material ID from item 20,  valuation price is not cleared.
  • Or create PR item 10 with material, then delete the material ID and maintain an account assignment category for it, the valuation price still from previous material master data and will not be cleared.

Environment

  • Materials Management (MM)
  • SAP ERP Central Component 
  • SAP ERP
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA

Reproducing the Issue

Scenario 1:

  1. Create PR in ME51N, add item line 10 that has material ID, the valuation price will be determined from material master data. Or we can maintain valuation price manually in Valuation tab.
  2. Copy item 10 as PR item 20.
  3. Delete material ID from PR item 20, the valuation price will nor be cleared.

Scenario 2:

  1. Create PR in ME51N with item 10 that has material ID, the valuation price will be determined from material master data. Or we can maintain valuation price manually in Valuation tab.
  2. Delete material ID from item 10, then maintain account assignment category for item 10, the valuation price will not be cleared.

Cause

The Valuation price is only changed for a new Material.
In case copying Material A with Price for example 10.
Later delete Material A , the Valuation price is not cleared for the user to recheck the correctness of the price.
This is system standard behavior.

Resolution

The copy function copies price and line, it is only possible to maintain the price manually once the copied line is entered, to ensure the correct price is accessed.

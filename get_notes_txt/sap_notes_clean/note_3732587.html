Symptom

Unit of Measure field is not sent in Payment Request Export.

Cause

Expected functionality in Generic and Peoplesoft variant sites.

Resolution

Unit of Measure (UOM) is not a part of any of the Payment Request Export task in Ariba Buying and Invoicing for Generic and PeopleSoft variants.

If it is expected by the external system, a custom field can be created to derive it's value from the standard Unit Of Measure field and the same can be made exportable in the Payment Request export. However, please note that this custom field will not send the preferred unit of measure mapping value. This feasibility needs to be determined if can be achieved by any other logic.

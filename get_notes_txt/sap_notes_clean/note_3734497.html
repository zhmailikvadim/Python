Symptom

Emails sent to SAP are not received or processed, even though a telnet test to the SMTP server is successful and returns a “250 OK” response.

"Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental."

Environment

Release Independent

Reproducing the Issue

  1. Send an Email to SAP using external mail server
  2. The email does not reach SOIN
  3. Perform a Telnet test to check the connection
  4. Check the result and is "250 OK" 

Cause

A successful telnet test with response “250 OK” confirms that from SAP side Network connectivity is working, SMTP port is open and reachable but the Mail server not forwarding emails to SAP.

Resolution

This is a mail server issue, in this case is necessary to contact the server administrator to check the following:

  • Verify email routing to SAP
  • Check SMTP relay and forwarding configuration
  • Review security rules, spam filters, and authentication settings
  • Confirm that emails are being sent to the correct SAP inbound address
  • Validate mail server logs for delivery attempts and failures

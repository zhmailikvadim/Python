Symptom

The note is to introduce 'MM Data Browser' tool (Report MM_DATABRW).

This tool eliminates the need to extract tables individually in separate SE16 sessions when working with Material Management purchasing and services data. It consolidates all related tables into a single, unified view, making them immediately available for analysis.

While it does not modify data or resolve issues on its own, it speeds up investigations by presenting all relevant information for a given business document at a glance with just a few clicks. Its purpose is to streamline daily work and incident resolution in the MM area for materials and services documents.

Business Documents currently covered by tool are: - Purchase Orders, Contracts, Purchase Requisitions, Service Entry Sheets, Good Receipts, Invoices.

Functionalities:-

  • Display data from several related tables relevant for analysis of issues for Purchase Orders, Contracts, Purchase Requisitions, Service Entry Sheets, Good Receipts and Invoices in one view.
  • Navigating to the data of related business documents from Purcahse Order.
  • Filter all the data for a particular item.
  • In case of Service Procurement, filter all the data for a particular Service line.
  • In case of hierarchical services, a click on a PACKNO will show the hierarchy of services under that node and all the associated data with one click.

Disclaimer: No issues would be addressed on the report itself, as its a tool for analysis of issues and not a standard product.

Other Terms

MM_DATABRW, Tool, Utility, Data Browser, Goods Receipt, GR, Service Entry Sheet, SES, Purchase Order, PO, Purchase Requisition PR, Contract, Invoice, Material Management, Service Procurement.

Reason and Prerequisites

Prerequisites:

Authorizations needed : SE16 transaction, S_DEVELOP

There is a need for analysis of database tables of MM area on a regular basis for customer incidents in the area of Material Management. The extraction of data manually from SE16 transactions consumes time and repetitive tasks for almost all incidents at several levels like Technical support team Development support team and customers.

Extracting these data also results into session timeouts several times, resulting in redoing the entire process.

The tool would be helpful to reduce the overall time for all the above scenarios to almost negligible, thus help in starting the actual analysis of the issues much faster.

Solution

Execute report MM_DATABRW

Feature Steps Result
Data Display (Report Execution)

Select the Business Object (Purchase Order, Contract, Purchase Requisition, Service Entry Sheet)

Enter the Document No. for the above selected object

Execute

All necessary table data is immediately displayed for all Items/Services

Color Coding:

Yellow  : Key Fields 
Orange : Other important fields for table relationships
Green   : Fields from 'Highlight Columns' filled in selection screen.
Filter data for a Document Item

Click on a value in column EKPO-EBELP to filter data for Purchase Order / Contract.

Click on a value in column EBAN-BNFPO to filter data for Purchase Requisition.

 All the data displayed is only relevant for this particular item
Filter data for Service Lines

Click on a value in column ESLL-PACKNO for Purchase Order / Contract / Service Entry Sheet / Purchase Requisition

All the data displayed is only relevant for this particular service.

If the PACKAGE is 'X' for the PACKNO, it will display data for the services that belongs to its SUB_PACKNO.
Navigate to SES/INV/GR from EKBE table.

Click on a value in column EKBE-BELNR or EKBE_MA-BELNR from any Business Object, where EKBE is displayed in the output.

 The report will display all necessary table data for the corresponding GR / INV / SES

Other Selections: Display Tables

Mention few relevant table names for the selected Business Object from the drop down. Example for Purchase Order type (EKBE ESLL EKPO)

 Only the mentioned tables are displayed, in the same order as they typed.

Other Selections: Highlight Columns (Up to 3)

You could type any columns names up to 3, belonging to any tables.

Example: NETWR MENGE PACKNO

The mentioned columns will start to display immediately after the Key columns, in the tables where they are present.

They will be highlighted in a color different from other columns.
Other Selections: Rows displayed per table

Example: 20

Maximum 20 rows of any table will be displayed on the screen.

If 'Total rows fetched' is 45

A clickable line "------> show more 25 " will start appearing.

On click of "------> show more 25 "  the table will display all 45 rows.
Other Selections: Hide Field Names Checkbox

If checked, the field descriptions are hidden, only technical names are displayed for all tables

If un-checked, both field descriptions and technical names are displayed for all tables.
Other Selections: Hide Empty columns Checkbox

If checked, all columns having no values for any row is hidden.
Other Selections: Hide Empty tables Checkbox

If checked, all tables having no rows is hidden.
Huge Data Handling Enter an item number for document to restrict the time taken for loading. On very huge data, a popup would appear that suggests to enter the Item Number in the initial screen.
Suggestions Use Variants to save all Optional parameter selections for future.

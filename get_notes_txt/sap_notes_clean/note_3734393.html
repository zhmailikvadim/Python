Symptom

  • Unable to launch the Enterprise Services Repository (ESR), Integration Directory (ID), or other PI swing-based tools using Java Web Start.
  • Enquiry about Supported alternative method to launch ESR/ID and other PI tools.

Environment

  • SAP NetWeaver
  • SAP Process Integration

Reproducing the Issue

Steps to collect Openwebstart logs while reproducing the issue:

  • Open "OpenWebStart Settings" Application -> under "Logging" tab -> Tick all checkboxes (including debug level logging), and specify Log Folder
  • Find the logs in the folder specified above, with file name pattern: YYYY-MM-DD_hh_mm_ss.xxx-ows-stagex.log
    Or Zip the whole log folder and attach it to SAP Ticket for analysis.

Cause

As outlined in SAP Note 2677849, All customers are required to use OpenWebStart along with SAPJVM, to launch PI Tools, other alternatives will no longer be supported/maintained. Since Oracle Java 8 which natively supported webstart is no longer allowed and need an alternative way to launch the .jnlp files to access the PI Tools.



Resolution

Steps to configure OpenWebStart:

  1. Installing SAP JVM
    • Visit the download link  https://tools.hana.ondemand.com/#cloud -> under the CLOUD tab page, scroll-down to section search for "SAP JVM" section
    • Download the zip archive for SAP JVM 8.1 to your system as per your Operating System.

  2. Extract the Zip Archive:
    • Locate the downloaded zip archive (e.g., sapjvm-8.1.zip) to any defined path over the system
    • Right-click on the zip archive and select "Extract" to unzip the contents to a folder of your choice.

  3. Setting up JAVA_HOME and PATH on Windows
    • Setting up JAVA_HOME:
      • Open the Start menu and search for "Environment Variables".
      • Click on "Edit the system environment variables" -> Click the "Environment Variables" button.
      • Under "System Variables", click "New" and enter JAVA_HOME as the variable name and the path to the SAP JVM folder as the variable value (e.g., C:\path\to\sapjvm-8.1), where the downloaded SAP JVM was unzipped.
    • Setting up PATH:
      • In the same "Environment Variables" window, find the "Path" variable under "System Variables" and click "Edit".
      • Click "New" and add %JAVA_HOME%\bin  to the list of paths.

    • Additionally, refer SAP Community blog for installation of OpenWebStart on different Operating System: https://community.sap.com/t5/technology-blog-posts-by-members/set-up-sap-pi-integration-builder-using-open-source-java-and-openwebstart/ba-p/14212065

  4. Installing OpenWebStart
    • Install OpenWebStart (download link -> https://openwebstart.com/download/)
    • Run the installer and follow the on-screen instructions as per your Operating System.
    • For macOS ensure to download the macos-x64 binaries and not aach64, as SAP JVM 8 is available for macos-x64 only.

  5. Configuring SAP JVM in OpenWebstart
    • In the Applications Installed over the system search and open "OpenWebStart Settings" application.
    • In the "JVM Manager" section,
      • Click "Remove ALL" to remove any other present JVM/JDK/JRE 
      • click "Add JVM/Add Local" and select the path to your SAP JVM installation.
      • You could observe the SAP AG JVM being loaded in JVM Manager

  6. Clear cache as step outlined in SAP Note 2677849:
    • The openwebstart configuration and cache can be found:
      C:\Users\<your user>\.cache\icedtea-web
      C:\Users\<your user>\.config\icedtea-web
    • Delete this folders if you face issues and re-do the setup

Note: 

  • Under "JVM Manager", under JVM Manager -> "Settings", disable the "update Strategy" by selecting option "Do not download any version" and increase the "delete unused JVMs" time, otherwise another JVM version will be downloaded, which does not work with ESR / DIR which could cause the CORBA error
  • Make sure you are using a Java 8 JVM (preferably SAPJVM) and delete/deactivate all other JVMs in OpenWebStart

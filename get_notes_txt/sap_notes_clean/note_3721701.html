Symptom

  • Unable to configure QUERY_STRING successfully in BI4.3 SP05.
  • QUERY_STRING test URL https://<Tomcat hostname>:<port>/BOE/BI?user=<user name> does not work properly although every configuration step in KB 3559381 has been followed.
  • The following message output in CMS server trace log.
    (TrustedAuthMgr.cpp:88) TrustedAuthMgr::getTrustedSystem: system is trusted: https://<Tomcat hostname>:<port>/BOE/CMC/TrustedAuthSetTokenServlet
    (TrustedAuthMgr.cpp:310) TrustedAuthMgr::CheckData: trusted URL: https://<Tomcat hostname>:<port>/BOE/CMC/TrustedAuthSetTokenServlet
    (CTrustedAuth.cpp:290) CEnterprise::AcceptLogin: SCrypt: failed to decrypt client's request for trusted authentication.
    (TrustedAuthMgr.cpp:335) TrustedAuthMgr::CheckData: Invalid trusted authentication data.

Environment

  • SAP Business Intelligence Platform 4.3 Service Pack 05 Patch300.
  • Windows Server Supported.

Reproducing the Issue

  1. Configure QUERY_STRING using fresh installation method according to KB 3559381.
  2. Test QUERY_STRING using https://<Tomcat hostname>:<port>/BOE/BI?user=<user name>.

Cause

The BI4.3 SP05 installation got failed.

Resolution

Repair the BI4.3 SP05 installation then re-test QUERY_STRING.

Symptom

You are using the Manage Assortments app in SAP CARAB 5.0 FPS03, FPS04, 5.0 SPS05, 5.0 SPS06, 5.0 SPS07, 5.0 SPS08, 6.0 SPS01 or 6.0 FPS02.

You are facing an issue with Number of Locations with Sales(Estimated) KPI, where the count is being calculated incorrectly.

Other Terms

SAP Customer Activity Repository applications bundle, CARAB, Manage Assortments, CARAB 5.0 FPS03, CARAB 5.0 FPS04, CARAB 5.0 SPS05, CARAB 5.0 SPS06, CARAB 5.0 SPS07, CARAB 5.0 SPS08, CARAB 6.0, CARAB 6.0 SPS01, CARAB 6.0 FPS02, CAR, CARAB, SAP CARAB, SAP Assortment Planning, KPI, Estimated, Number of Locations

Reason and Prerequisites

A Program error.

Prerequisites

You have set up the Manage Assortments app as described in the Common Installation Guide (for new installations) or the Common Upgrade Guide (for upgrade scenarios) for SAP Customer Activity Repository applications bundle. You can find both guides on SAP Help Portal at https://help.sap.com/docs/CARAB. Select your version at the top.

Solution

Implement the correction instructions attached to this note in your backend system.

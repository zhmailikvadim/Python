Symptom

Upgrade process in SUM(Software Update Manager) ended with the following error message in the phase MAIN_SHDRUN/RUN_RSPTBFIL_NZDM_CLASSIFY.

**********

A3WESCRR 449 Nametabs of table <table name> cannot be compared

A3WESUPG2 030 Difference between DD and DB. FULL_TEXT DD:  <-> DB: X (index <index name>)

A3WESUPG 007 Unable to determine the table delta between <table name> and <table name>

A4 ESAUS_CLSY 006 ACTION_END SINGLE_TAB_CLASSIFY at <time stamp> (table <table name>).

**********

Environment

SAP ABAP System

Reproducing the Issue

Retry the failed phase.

Cause

The index mentioned in the log file got inconsistencies between DDIC and database.

The inconsistencies could be at indexed field level or the index type(fulltext/normal index).

Resolution

To fix the inconsistency: delete the index mentioned in the error message and create it again in transaction SE14 in the source system. 

Just retry the failed phase in SUM when the index recreation is finished successfully.

If the workbench is locked and the index recreation cannot be performed in transaction SE14, resetting the upgrade is required -- reset the upgrade, recreate the problematic index and then restart the upgrade from scratch.

Symptom

  • Messages are stuck in SXMB_MONI with the status message waiting in queue.
  • Messages are not visible in SRT_MONI and are not delivered to the target systems.
  • When Logical Port is Default is set to true for one system, messages are sent only to the default system and not to other systems.
  • When Logical Port is Default is not set for any system, no messages are sent to any target system.
  • The issue occurs when attempting to send web-service messages to multiple target systems using service groups in SOAMANAGER.

Reproducing the Issue

  1. Configure service groups in SOAMANAGER without setting Logical Port is Default for any system.
  2. Send test messages via DRFOUT to target systems using replication models and outbound implementations.
  3. Observe the following:
  • Messages appear in SXMB_MONI with the status message waiting in queue.
  • Messages do not appear in SRT_MONI.
  • Messages are not delivered to the target systems.

Cause

  • The name of the business system in DRFIMG and the provider system name in SOAMANAGER did not match exactly.
  • Without a match between DRFIMG and SOAMANAGER, the system attempted to send data leads to causing the issue.

Resolution

  1. Ensure that the Business system name in DRFIMG matches the provider system name in SOAMANAGER exactly.
  2. Verify that logical ports are created automatically through service groups and not manually.
  3. Confirm that Logical Port is Default is not set for any system in SOAMANAGER.
  4. Test the configuration by sending messages via DRFOUT to multiple target systems.

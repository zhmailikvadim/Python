Symptom

You create many planned orders in one call of the function module CLOI_CHANGES_UPL_31.

The performance is bad. A performance trace shows that much time is lost at the statement SORT KALCB_TMP.

Other Terms

CLOI_CHANGES_UPL_31, CLOI_PLAF_CREATE
MD_PLANNED_ORDER_CREATE, CX_PLANNED_ORDER_SCHEDULE
CO_ML_PO_CALC_SHEET_COPY, KALCB_TMP
Material quantity calculation, Performance

Reason and Prerequisites

The issue only occurs if many planned orders are created in the same call of CLOI_CHANGES_UPL_31.
The issue only occurs if the routing that is used for scheduling is a master recipe and formulas for material quantity calculation are maintained.

Solution

Implement the correction instruction.

Symptom

It is observed that cookies are missing, not forwarded, or appear to be modified when requests are routed through SAP Cloud Connector (SCC). This may lead to the assumption that SCC is altering cookie handling in requests or responses.

Environment

SAP Cloud Connector, release independent 

Cause

SAP Cloud Connector does not modify cookies. It operates as a transparent tunnel for HTTP and HTTPS communication and forwards all headers, including cookies, without changes.

The only exceptions are the host header and the location header (which may be rewritten if explicitly configured for virtual host to real host mapping). Apart from this, no HTTP headers or payload content are modified by SCC.

If cookies are missing or not behaving as expected, the cause is located outside of SAP Cloud Connector. The backend application may not generate or return the expected cookies, or the client application may not send the required headers or request context.

Resolution

Investigation should focus on the client application to ensure that cookies and required headers are correctly included in requests, and on the backend application to verify whether the appropriate Set-Cookie headers are set in the response.

For scenarios where cookies are not accepted due to differences between virtual and internal domains, refer to SAP Help Portal:  Configure Domain Mappings for Cookies | SAP Help Portal

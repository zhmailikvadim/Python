Symptom

Dataflow component "datasource.write.external.generic" executions to the external service fail with "HTTP Status 401 – Unauthorized".

Uncertainty about how to configure Basic authentication for the datasource.write.external.generic component.

Environment

  • SAP Customer Data Cloud
  • Core ETL / IdentitySync 

Resolution

  • In Customer Data Cloud (CDC) Admin console > Dataflows, select "Properties" in the menu of component datasource.write.external.generic
  • In field "url", enter the URL of the remote endpoint e.g. https://test.com/test/xxxxxx
  • In "headers" section, click Add button to add a new record (headerName and headerValue)
  • On the popup dialog, set field "headerName"="Authorization"
  • Set field "headerValue"=Encode Base64 of the (userKey) and (secretKey) with the colon character in between the (userKey) and (secretKey).
    • Example: (userKey)=123 and (secretKey)=ABC. 
    • Encode this value (123:ABC) in Base64 => "MTIzOkFCQw=="
  • So set field "headerValue"="Basic MTIzOkFCQw=="

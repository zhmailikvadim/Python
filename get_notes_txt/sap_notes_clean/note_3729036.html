Symptom

application component CA-GTF-SAL which is supposedly OBSOLETE and NOT used since many years is to be removed from the application component hierarchy and also from the messaging tool.

Other Terms

mACH change-request tool, TA SBACH04, TA SBAC, TA HIER.

Reason and Prerequisites

Reason: application component CA-GTF-SAL is obsolete and is to be delted.

Prerequisite: 

- Change-request for deletion to be submitted via mACH-Tool, which is done: https://ach.cfapps.eu10.hana.ondemand.com/#/requests/change/show/2cc6cdfb-b6fb-4ef4-8e5e-cc79f8edf22a & request for deletion is maintained in the ABAP-details for the physical removal of the object in the software-layer MIVERPSD.

Solution

Steps:

1. Check for where-used-list of the component CA-GTF-SAL via TA SBAC.

2. no references shall exist other than its existence in the compnent hierarchy such as TA HIER.

3. Select the structure in TA SBACH04 and select the component and 'remove' it from the component-hierarchy tree.

4. Then go to TA SBAC and delete the component.

5. There should be no error message (or error-popup) seen.

6. You can double-check in TA HIER and see if the component is REMOVED and if it does NOT exist all is good.

7. You can also check in TA SBAC and see in the component is removed.

Finally you can release the transport.

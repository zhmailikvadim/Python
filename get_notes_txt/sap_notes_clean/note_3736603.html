Symptom

After upgrading to SAP S/4HANA 2025, Fiori Launchpad applications load in only half of the screen while the other half continues to display the Home Page. This display issue occurs when using custom themes but resolves when switching to standard SAP themes. 

Environment

  • SAP S/4HANA 2025 
  • SAP Fiori Launchpad 

Reproducing the Issue

  1. Open Fiori Launchpad with a custom theme configured
  2. Click on any application tile to launch an app
  3. Observe that the app loads in only half of the page while the Home Page remains visible in the other half
  4. Navigate back to the Home Page and switch to a standard SAP theme (e.g., sap_horizon, sap_quartz)
  5. Reload the page and try opening the same app - verify it now loads in full screen 

Cause

Custom themes created for previous S/4HANA versions are not compatible with the new Fiori shell architecture and CSS framework introduced in S/4HANA2025. Changes in Fiori shell header height and layout structure post-upgrade affect custom theme rendering and app display dimensions. 

Resolution

  1. Rebuild the custom theme using UI Theme Designer following the procedures outlined in KBA 2505163

    - Access UI Theme Designer via transaction /n/UI5/THEME_DESIGNER or direct URL
    - Select your custom theme and open it
    - Navigate to Theme menu and select "Save and Build" (older versions) or "Save & Publish" (newer versions)

  2. Clear browser cache following the steps provided in KBA 2702930

    - Clear cache in all browsers used to access the Fiori Launchpad
    - This ensures the updated theme files are properly loaded

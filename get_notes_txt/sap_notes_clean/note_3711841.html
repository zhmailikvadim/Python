Symptom

When exporting data from an ALV Grid or ALV List to Comma Separated File (*.csv) format, one or more cells may contain text with multiple occurrences of double quotes (").

In the generated CSV file, only the first double quote is escaped for each cell, while subsequent double quotes remain unescaped.

Other Terms

ALV; ABAP List Viewer; SAP List Viewer; Grid; List; Lean Export; LEx; Excel; csv; xlsx; spreadsheet; Export to Excel; open office XML Format;

Reason and Prerequisites

Reason

During CSV generation, character escaping for double quotes was applied only to the first occurrence per cell value. Additional double quotes within the same cell were not properly escaped.

Prerequisites

The following Support Package level is the minimum prerequisite:

Release Support Package
SAP_BASIS 816 SP 00
SAP_BASIS 758 SP 02

When you install an SAP Note, some prerequisites might be required. The prerequisites are SAP Notes that have already been released and that change the same context of the current one. Then, even if these prerequisites are not necessarily related to the symptom of this SAP Note, you must implement them due to technical reasons.

Solution

The CSV export logic has been corrected so that all occurrences of double quotes within a cell value are properly escaped.

As a result, the issue described in the Symptom section no longer occurs when exporting ALV data to CSV format.

Delivery

As a general rule, SAP recommends that you install a solution by applying a Support Package.

However, if you need to install the solution earlier: Use the Note Assistant (SNOTE) to implement the correction instructions.

Learn more at SAP Note Assistant.

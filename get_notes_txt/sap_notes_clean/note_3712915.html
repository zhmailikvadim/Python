Symptom

  • Generating an oauth token in BILP -> User Settings -> Authorization Tokens may fail with error:
    “An error occurred, kindly retry. If the issue persists, please contact your administrator.” 

  • other workflows may also fail, eg STS OLAP SSO connections
  • In the STS logs, the below errors are seen:
    STS: An unexpected error has occurred (FWM 02004) java.security.NoSuchAlgorithmException: Error constructing implementation (algorithm: Default, provider: SunJSSE, class: sun.security.ssl.SSLContextImpl$DefaultSSLContext)
    com.crystaldecisions.sdk.exception.SDKException$Unexpected: An unexpected error has occurred (FWM 02004)
    cause:java.net.SocketException: java.security.NoSuchAlgorithmException: Error constructing implementation (algorithm: Default, provider: SunJSSE, class: sun.security.ssl.SSLContextImpl$DefaultSSLContext)
    Caused by: java.net.SocketException: java.security.NoSuchAlgorithmException: Error constructing implementation (algorithm: Default, provider: SunJSSE, class: sun.security.ssl.SSLContextImpl$DefaultSSLContext)
    Caused by: java.security.NoSuchAlgorithmException: Error constructing implementation (algorithm: Default, provider: SunJSSE, class: sun.security.ssl.SSLContextImpl$DefaultSSLContext)
    Caused by: java.security.KeyManagementException: problem accessing trust store

Environment

  • SAP BusinessObjects Business Intelligence (BI) Platform 4.x / 2025
  • oAuth 2.0 / Security Token Service (STS)
  • Windows
  • Linux / Unix 

Reproducing the Issue

  1. Setup an authorization server for oauth 2.0 workflows
  2. Navigate to BILP -> User Settings -> Authorization Tokens
  3. Observe the error message mentioned in the symptoms is displayed when trying to generate a token 

Cause

The java cacerts file is inaccessible

Resolution

  1. On the OS hosting the Security Token Service (on an APS), navigate to one of the below directories:
    Windows (2025): <INSTALLDIR>\SAP BusinessObjects Enterprise XI 4.0\win64_x64\sapjvm\lib\security
    Windows (4.x): <INSTALLDIR>\SAP BusinessObjects Enterprise XI 4.0\win64_x64\sapjvm\jre\lib\security
    Linux (2025): <INSTALLDIR>/sap_bobj/enterprise_xi40/linux_x64/sapjvm/lib/security
    Linux (4.3): <INSTALLDIR>/sap_bobj/enterprise_xi40/linux_x64/sapjvm/jre/lib/security

  2. Ensure the cacerts file exists and is accessible by the account that the SIA is running as
  3. If the cacerts file does not exist, copy it from a working environment
  4. Ensure the issuer of the certificate for the host being accessed exists in the cacerts file:

    Eg for oauth 2.0 generate token workflow:
    Ensure the issuer certificate for the URL defined as the token endpoint in CMC -> Applications -> Authorized URL configuration is imported into the cacerts truststore
  5. Download the certificate with the below steps

    Copy the URL


    Paste it in a browser & follow the steps in the screenshot to download the certificates





    Ensure to select the root certificate (the first one mentioned)




  6. Place this file on the OS of the machine hosting the STS
  7. Open a command prompt
  8. Navigate to the following location: cd C:\Program Files (x86)\SAP BusinessObjects\SAP BusinessObjects Enterprise XI 4.0\win64_x64\sapjvm\bin
  9. Run
    keytool -import -trustcacerts -alias <uniquealias> -file <PATH TO DOWNLOADED CERTIFICATE> -keystore ../jre/lib/security/cacerts -storepass changeit

    On BI 2025+:
    keytool -import -trustcacerts -alias <uniquealias> -file <PATH TO DOWNLOADED CERTIFICATE> -keystore ../lib/security/cacerts -storepass changeit

  10. Restart the STS service

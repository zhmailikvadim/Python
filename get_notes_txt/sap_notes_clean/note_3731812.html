Symptom

In transaction ABSO, the header amount field is mandatory although adjustments are posted at the line-item level by depreciation area. This is confusing. Therefore, users need to set the header amount optional or skip it entirely and enter amounts only in line items.

Environment

SAP S/4HANA

Reproducing the Issue

  1. Start transaction ABSO.
  2. Enter header details; the header amount is required.
  3. Go to line items and enter the amount for the relevant depreciation area.
  4. Simulate and post.

Cause

Standard behaviour in ABSO: the header amount field is set as a required entry on the standard screen, even though postings are driven by line-item/depreciation-area values. The header field is informational in many cases.

Resolution

  1. Start SHD0 (Transaction Variants).
  2. Create a transaction variant for ABSO (or create/edit the related screen variant).
  3. Record the ABSO posting screen and change the header amount field properties:
  4. Set the field to Optional (remove “required entry”). Optionally hide the field if acceptable for the business process.
  5. Save the variant.
  6. Assign the variant to users:
  7. Either use SHD0 user assignment, or
  8. Use SPRO → IMG → Financial Accounting → Financial Accounting Global Settings → Transaction Variants to assign per your setup.
  9. Test ABSO to confirm the header amount is optional/hidden and posting is driven by line-item depreciation-area entries.

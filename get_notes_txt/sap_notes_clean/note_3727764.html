Symptom

  • No data has been received in Cloud Health Monitoring for S/4HANA system
  • SAP Health Monitoring shows the following message for SAP S/4HANA system:
     [No data] No data received.

Environment

  • Cloud Health Monitoring
  • SAP S/4HANA
  • SAP Cloud ALM

Reproducing the Issue

  1. Log into Cloud Health Monitoring using the appropriate credentials with administrative or monitoring access.
  2. Navigate to the Health Monitoring Dashboard.
  3. Locate the affected system.
  4. Verify the data status and observe that no data has been received.

Resolution

Please follow the steps below:

Steps to Refresh the Health Monitoring Use Case

  1. Log on to the affected SAP S/4HANA system.
  2. Run transaction code /N/SDF/ALM_SETUP.
  3. Click “Activate use‑cases”:


  4. Disable (deactivate) the Health Monitoring use case.
  5. Exit the program /N/SDF/ALM_SETUP.
  6. Wait 10 minutes.
  7. Run /N/SDF/ALM_SETUP again.
  8. Click “Activate use‑cases”.
  9. Re‑enable the Health Monitoring use case.
  10. Exit the program again.
  11. Wait another 10 minutes.


Then refresh the data collection in Cloud ALM:

  1. In SAP Cloud ALM, go to Operations → Health Monitoring
  2. Select the affected system via the scope selector
  3. Open Configuration
  4. Turn off the Data Collection toggle for the system
  5. Wait 5 minutes and turn the toggle on again

Refresh fetching data:

  1. Log on to the affected SAP S/4HANA system
  2. Execute transaction /SDF/ALM_SETUP once more and click “Refresh – Fetch latest use‑case selection from SAP Cloud ALM”:



  3. Wait 5–10 minutes, then check in Cloud ALM Health Monitoring to confirm whether fresh data is arriving for the system.

These steps should reset the Health Monitoring configuration and resolve the interruptions observed earlier.

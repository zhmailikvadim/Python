Symptom

When I perform a Job Posting Template Upload to update existing templates, the transaction finishes with a "Success" status. However, when I check the templates in the UI, specific rules or flags have not been updated as expected. For example, the setting for "Give Active Resources access to all Task Codes" remains unchanged.

Cause

This issue is typically caused by an incorrect column header name in the upload file.

The SAP Fieldglass upload engine requires an exact character match for headers to recognize and map the data to the correct system field. A common error occurs with singular vs. plural phrasing. For instance, if your file uses "Give Active Resource access..." (singular) instead of the required "Give Active Resources access..." (plural), the system will ignore the column entirely during the update process.

Because the rest of the file may be correct, the system processes the record without error, but simply skips the mislabeled column.

Resolution

You must ensure your upload file headers match the system's required template exactly.

Download the Current Template:

  1. Navigate to Admin Configuration > Integration > Upload Data.
  2. Search for the Job Posting Template Upload.
  3. Select the connector and click the Download Template button to get the most recent version of the file structure.

Compare Headers:

  1. Open your upload file and the fresh template side-by-side.
  2. Verify specific rule headers. Ensure "Resource" vs. "Resources" or similar subtle differences are aligned with the official template.

Update and Re-upload:

  1. Correct any mismatched header names in your file.
  2. Re-run the upload transaction.

Verify in UI: Confirm that the rules on the Job Posting Template now reflect the values provided in your corrected file.

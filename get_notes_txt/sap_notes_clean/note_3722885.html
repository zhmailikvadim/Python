Symptom

  • When performing inbound delivery confirmation via RFUI using the handling unit, the process does not work as expected.
  • Error message: "Warehouse process type for distributive putaway is not defined correctly" (/SCWM/RF_DE051).

Environment

  • SAP S/4HANA
  • Extended Warehouse Management

Reproducing the Issue

  1. Perform inbound delivery confirmation via RFUI using the handling unit(03 inbound delivery->03 putaway->01 Putaway using HU).
  2. Input HU number and encounter the error message: "Warehouse process type for distributive putaway is not defined correctly" (/SCWM/RF_DE051).

Cause

  • The warehouse process type for distributive putaway is not defined correctly in the system configuration.

Resolution

  1. Verify and update the warehouse process type configuration as follows:

a. Navigate to SPRO > Extended Warehouse Management > Master Data > Define Warehouse Number Control.

b. Enter the warehouse process type (WPT) for handling unit warehouse tasks for putaway.

  1. Ensure the following settings in SPRO > Extended Warehouse Management > Cross-Process Settings > Define Warehouse Process Type:

a. Set "Storage Control Relevance" to 3 and make sure that the flag "Confirm Immediately" is set for the defined Warehouse process type (WPT). 

b. User need the warehouse process type for handling unit (HU) warehouse tasks to perform the distributive putaway in the radio frequency environment. Here, the system creates a confirmed warehouse task, which places the handling unit on the resource.

It has 8 condition to create the WPT correctly. The following values must be defined when defining the warehouse process type:

Warehouse process type =  1 for putaway
Source storage type = ' '
Source storage bin = ' '
Destination storage type = ' '
Destination storage bin = ' '
Can be confirmed immediately = 'X'
Manual WT not permitted = ' '
Storage control = 3 for not relevant

Symptom

When running aggregation reconstruction for “condition types” KF-based aggregation type, multiple issues are observed. A dump occurs because the “where” clause is incorrectly prepared using “C.” instead of “C~”. Additionally, in the “where” clause, all condition types are combined using “and” logic, even though they represent independent condition records in the CE table. This leads to incorrect data selection and processing during aggregation reconstruction.
This note corrects the behavior by ensuring proper formatting of the “where” clause and correct logical handling of condition types.

Other Terms

aggregation reconstruction issue, condition types KF, where clause error, C dot vs C tilde, condition types logic, CE table issue, dump in aggregation

Reason and Prerequisites

This is caused by program error

Solution

Please implement the correction instructions attached. These corrections are only valid if you have applied the following support packs.

Release 6.1 – Base Package, 6.2 Base Package, 6.3 Base Package, 6.4 Base package or Support Pack 1

Symptom

  • Calling the workflow REST API to create a case returns HTTP 404 Not Found.
  • After obtaining a bearer token and calling the workflow API, the response shows "invalid issuer".
  • After adjusting headers/body, GET on cases works, but POST (create case) returns HTTP 400 Bad Request with: {"error_code":"PROPERTY_NULL","error_messages":["Status not specified."]}.
  • Difficulty accessing or retrieving the expected response from the cases endpoint.

Environment

  • SAP Sales Performance Management
  • Advanced Workflow

Reproducing the Issue

  1. Use a REST client (for example, Postman, Bruno) to call the workflow cases API with basic authentication to list or create cases.
  2. Observe HTTP 404 Not Found on the cases endpoint.
  3. Switch to OAuth 2.0 (client credentials), obtain a bearer token, and call the workflow API; observe an "invalid issuer" response.
  4. After header/body changes, GET requests succeed, but POST to create a case returns HTTP 400 with error_code PROPERTY_NULL and "Status not specified.".

Cause

  • Workflow OIDC/trust configuration was incomplete or inconsistent, causing tokens to be rejected with "invalid issuer" and leading to API failures.
  • The POST request payload to create a case omitted a required property (status), resulting in HTTP 400 Bad Request with error_code PROPERTY_NULL.

Resolution

  1. Creating Client Secret

    1. Go to Applications & Resources → Applications
    2. Search for the OIDC Workflow tenant. The name generally follows the convention of wf_oidc_<TENANT_ID> or WF_<TENANT_ID>_OIDC
    3. Go to the trust tab in the OIDC Workflow tenant 
    1. Go to Client authentication → Secrets → Add
    2. Select all boxes for API Access. You may add any description and expiration date as desired. Click save
    3. The client secret and client ID will appear in the dialog. It's important to save the client secret, as it will not be retrievable in the future. The client ID can be viewed anytime at the secrets page.
Creating API User in Workflow
  1. In the target tenant as an admin, go to Setup → Users → Users
  2. Create new
  3. Provide any name and e-mail
  4. Check the box for administrator
  5. Department and user type may be any. You can use Workflow Administrator and Workflow Administration
  6. In IAS client ID, insert the same client ID as the WFL OIDC application retrieved earlier.
API Calls
Now that the environment is configured, it should be ready to receive authorized API calls. 
Before proceeding to the authentication and calling the API, create a collection and a request in Bruno.
The target URL will be https://https://environment//wpm/api/<api_target>. All possible API calls may be consulted at Workflow API Documentation | SAP Help Portal
 
Getting Bearer Token and Authentication
  1. Go to the request in Bruno
  2. Select the Auth tab
  3. Select OAuth 2.0 and Client Credentials as authentication method.
  4. In access token URL insert the OAuth URL for the IAS environment. It should follow the pattern https://######.accounts.ondemand.com/oauth2/token
  5. In client id insert the same client id retrieved from the WF_OIDC application
  6. In Client Secret insert the secret created in the first step
  7. Click Get Access Token

Symptom

  • Changes made in the FINCS_RRULE transaction code or in Define Reporting Rules app are not saved at all.
  • After attempting to save data or add a new line in the view, the line disappears despite a "data saved" message being displayed.

Environment

Embedded Analytics for S/4Hana Finance for Group Reporting

Reproducing the Issue

  1. Access the "Define Reporting Rules" application for reporting item hierarchy XXX (version YYY) or in transaction code FINCS_RRULE.
  2. Attempt to add a new row.
  3. Save the changes.
  4. Observe that the new row disappears despite the system displaying a "data saved" message.

Cause

Dummy parameter was entered into one of the active field. 

Resolution

  1. Use the drop-down list for every active field when adding a new row in the "Define Reporting Rules" application.
  2. Ensure that no dummy parameters are used.

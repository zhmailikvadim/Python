Symptom

You are performing an Inventory Sampling for WM. When performing the Stock Population via transaction MIS2 you change the upper value limit and Copy it without first confirming the entry by pressing the Enter key. The resulting Classification (and by extension Stratification) only takes the prior (unchanged) upper value limit into account.

Other Terms

upper price limit, SINVT-PREIS_NEU, SINVT-BUANF_NEU, MM07SFD0﻿

Solution

The correction is delivered with the corresponding Support Packages. For an advance correction, please implement the Correction Instructions contained in this note.

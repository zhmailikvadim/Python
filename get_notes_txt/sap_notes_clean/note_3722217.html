Symptom

Error message displayed when trying to enable a service in BTP Cockpit: "You haven't configured your subaccount (for business users) to trust a tenant of SAP Cloud Identity Services using OpenID Connect. SAML trust isn't supported for this service. To subscribe this application, use the Establish Trust option to configure trust between your subaccount and SAP Cloud Identity Services. For more information, see https://help.sap.com/docs/btp/sap-business-technology-platform/establish-trust-and-federation-between-uaa-and-identity-authentication".

Additionally:

  • The subaccount has not been configured to trust a tenant of SAP Cloud Identity Services using OpenID Connect.
  • Only the Default Identity Provider is available, and no Custom Identity Provider was set up.

Environment

  • SAP Business Technology Platform
  • SAP Cloud Identity Services
  • SAP Build Work Zone

Reproducing the Issue

  1. Attempt to enable SAP Build Work Zone subscription.
  2. Observe the error message: "You haven't configured your subaccount (for business users) to trust a tenant of SAP Cloud Identity Services using OpenID Connect. SAML trust isn't supported for this service. To subscribe this application, use the Establish Trust option to configure trust between your subaccount and SAP Cloud Identity Services. For more information, see https://help.sap.com/docs/btp/sap-business-technology-platform/establish-trust-and-federation-betwe".

Cause

Trust was not set up with a Custom Identity Authentication Service (Cloud Identity Service tenant) with OpenID Connect protocol, which is a requirement to enable SAP Build Work Zone.

Resolution

  1. Access the SAP Business Technology Platform (BTP) Cockpit at: https://cockpit.btp.cloud.sap/.
  2. Navigate to the subaccount.
  3. Choose "Security" -> "Trust Configuration".
  4. Select the option Establish Trust.
  5. Choose the desired SAP Cloud Identity Services tenant.
    • The identity providers listed are the SAP Cloud Identity Services tenants associated with the Customer ID.
  6. Click "Next".
  7. Select the domain.
  8. Click "Next".
    • Optionally, change the name and description of the tenant.
    • The origin key can have a maximum of 36 characters. Only use the following characters for the origin key of the trust configuration: aA–zZ, 0–9, - (hyphen), _ (underscore).
  9. Provide a link text for user logon. Refer to: Using Multiple Identity Providers from the Same Subaccount.
  10. Continue by clicking "Next".
  11. Confirm the configuration by clicking "Finish".

Then enable the SAP Build Work Zone subscription again.

See Also

Refer to: SAP Help Documentation: Establish Trust and Federation Between UAA and Identity Authentication

Symptom

The expiry date for a user in the quality department can be set in the past, even if the user has performed actions such as release decisions after the expiry date.

Environment

SAP Batch Release Hub for Life Sciences (BRH).

Reproducing the Issue

  1. Set an expiry date for a user in the Quality Department.
  2. Choose a date in the past, even if the user has performed actions after the chosen Expiry Date.

Cause

This behavior is intentional and was designed to allow immediate removal of a user's ability to release or review batches. Setting "Expires On = Today" may still allow the user to perform actions during the same day.

Resolution

  1. Confirm that the system allows setting Expiry Dates in the past to enable immediate removal of user access.
  2. Verify that the change log records when the Expiry Date was maintained, ensuring compliance visibility.

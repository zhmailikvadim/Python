Symptom

You implemented note 3657609 and/or are having issues with the after import method USMD_TR_AFTER_IMP_EXEC due to an illegal write access to USMD1213 of flag USMD_CR_IS_CLOSED. Please make sure the note is applied before the upgrade.

Other Terms

VC_USMD001

Reason and Prerequisites

Program Error

Solution

Apply the correction instructions or corresponding SP before upgrading. The after import method USMD_TR_AFTER_IMP_EXEC will no longer write to table USMD1213.

Therefore the flag USMD_CR_IS_CLOSED has to be adjusted manually and cannot be set with the after import method. This means that the report USMD_INIT_CR_IS_CLOSED needs to be triggered manually once per system/client combination (if not done already) if there are performance issues with the MDG system or existing data sets that have not been adjusted before.

If you already tried to upgrade and the system upgrade came to a stop because of the write access, you may ignore these writes as they will not lead to inconsistencies if they are not happening. The previous mentioned report can set the flag accordingly (if required).

Symptom

In Guided Sourcing:

  • Is it possible to copy Sourcing Request documents into a Sourcing Project created on the Sourcing Request?
  • Documents are not transfer from the Sourcing Request to the Sourcing Project as they were on Classic Sourcing;
  • How to copy documents from parent project to child project?
  • System does not copy the information entered in the Guided Sourcing Request to the created Guided Sourcing Project. 
  • "Copy all documents from the parent project?" and "Copy all project groups from the parent projects?" are not available on Guided Sourcing when creating a project from another Sourcing Request or Sourcing Project?

Environment

SAP Ariba Sourcing

Cause

In Classic Sourcing we have below options:

  • Copy all documents from the parent project?
  • Copy all project groups from the parent projects?

Which allowed to copy the above objects (documents and/or groups) from Sourcing Request to Sourcing Project.

As of 2408 release, the above option is not available for Guided Sourcing, as copying teams and document while creating sourcing project was not implemented for guided sourcing, as on Guided Sourcing, you have the option to copy content (Teams, Document) predecessor project or from other projects where you can select the needed guided sourcing request/project.

Resolution

Below there is other ways to copy/import the documents or groups to a Guided Sourcing project.

In case you consider that the original feature from Classic Sourcing or some enhancement on below solutions is needed for your business, we invite you to provide your direct feedback to our product team. You can vote on existent or raise a new enhancement request following the process referred on this KBA 3176706.

HOW TO COPY DOCUMENTS

Although, it is not possible to directly copy during the Sourcing Project creation as on Classic Sourcing, you can copy the documents through one of the below methods below:

  • Copy documents from Parent project via Copy Document from Content Finder (only possible to documents)  
    1. Access the Project that you want the groups/teams copied to (imported);
    2. Scroll down to the Documents section;
    3. Click Create -> Document;
    4. Click Copy Document from Content Finder;
    5. Click on tab Current Workspace;
    6. Here it will be displayed the Parent project (Sourcing Request or Project) and its documents, select the desired document to be imported by clicking on the blank space to the left of the Title of the document;
    7. Click Done to copy/import; (only one document can be imported each time)
  • Set the Sourcing Request or Sourcing Project that is the Parent project as also the Predecessor project and then Copy from Predecessor Project(possible both to documents and teams):
    1. Access the Project that you want the groups/teams copied to (imported);
    2. (If not already set Predecessor Project) Expand Project Summary section, scroll down to Linked Project sub-section and click on Edit button;
    3. (If not already set Predecessor Project) On Predecessor field, open the drop-down for the field scroll down to the end and click on Select other on a new page...
    4. (If not already set Predecessor Project) Enter the sourcing request or project name from which you want to copy from, adjust the filters on the left panel to be able to find the desired request/project/event, then select and click Select button;
    5. (If not already set Predecessor Project) Make sure to save Linked Project section, by clicking on the Save button on the section;
    6. Scroll down to the Documents section;
    7. Click Create -> Document;
    8. Click on Copy from Predecessor Project;
    9. Expand the project/request predecessor and select the documents that you want to import/copy;
    10. Click Done.

Currently (as of 2408 release), only Copy from Predecessor Project allows you to import/copy multiple documents at once.

HOW TO COPY GROUPS/TEAMS

To copy Groups/Teams, you are only able to do if you set the the Predecessor Project (steps 2-5 are only needed if you have not setup as predecessor on project creation) and follow steps below:

  1. Access the Project that you want the groups/teams copied to (imported);
  2. (If not already set Predecessor Project) Expand Project Summary section, scroll down to Linked Project sub-section and click on Edit button;
  3. (If not already set Predecessor Project) On Predecessor field, open the drop-down for the field scroll down to the end and click on Select other on a new page...
  4. (If not already set Predecessor Project) Enter the sourcing request or project name from which you want to copy from, adjust the filters on the left panel to be able to find the desired request/project/event, then select and click Select button;
  5. (If not already set Predecessor Project) Make sure to save Linked Project section, by clicking on the Save button on the section;
  6. On the project header, click on the team symbol next to the Name of the project;
  7. On the pop up open, click Import teams from predecessor project;
  8. On the pop up, select project/request by clicking on the blank space to the left of the project name;
  9. Click on the Done button;

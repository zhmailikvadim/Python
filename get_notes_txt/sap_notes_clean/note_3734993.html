Symptom

Understanding the potential impact of introducing new dimension.

  • What areas are likely to be impacted by this change?
  • Whether existing data flows, Open Hub objects, or BAdIs may require adjustments?
  • If adding a dimension to a productive model with large historical data is generally recommended or if there are risks to consider?

Environment

  • SAP BusinessObjects Planning and Consolidation, version for SAP NetWeaver
  • SAP BusinessObjects Planning and Consolidation, version for SAP BW/4HANA

Reproducing the Issue

Add a new dimension to an existing model.

Resolution

  1. Areas impacted:
    From the BPC perspective, adding a new dimension to an existing model will impact all BPC features that rely on the model structure. This includes existing journal templates, comments, audit data storage, business rules, and the overall model data storage.
    It will be needed to validate that data storage and all corresponding BPC data behave as expected after the new dimension is added. Data loading processes and script logic should also be reviewed to ensure they align with the updated model structure.
  2. Recommendation:
    To minimize unexpected issues, it's recommended to create a new model with the new dimension included and gradually replicate the data and business logic into this new model. This approach provides better control and reduces the risk of disruption to the existing operations.

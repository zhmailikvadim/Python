Symptom

A failing RAP‑based request or OData query should be analyzed, and the generated SQL statement executed by the framework has to be extracted for further investigation or root‑cause analysis.
 
"Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental."

Environment

  • SAP S/4HANA

Resolution

Follow the steps below to record a trace and extract the SQL statement.
  1. Start transaction SALV_TRACE. This opens the ALV Trace Tool, which contains the trace activation panel.
  2. In the Trace Activation area, click "Trace On/Off" so that the trace Status becomes "Active".
  3. While the trace is Active reproduce the issue, i.e. execute the exact user action or request that needs to be analyzed. This could be a UI search, RAP query, OData request triggering a dump or an error.
  4. Stop the trace by returning to transaction SALV_TRACE and clicking "Trace On/Off" again so the status changes to "Inactive".
  5. Select "Refresh List" to display the newly generated trace entries.
  6. In the trace list, locate the most recent trace set created during STEP 3 and open it.
  7. In the trace content, identify the failing row (typically displayed as a red line).
    In the row where the Task column contains “SADL Dynamic SQL Executor – SQL SELECT”, double‑click the Input cell in the Data column.
     
  8. The XML Viewer opens for that trace node.
  9. The SQL statement contained in the XML Viewer can be copied for further analysis.

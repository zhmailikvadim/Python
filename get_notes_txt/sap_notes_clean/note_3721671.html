Symptom

Given the following sap-ui-bootstrap script that bootstraps SAPUI5 from a CDN URL using a concrete SAPUI5 version and /sap-ui-cachebuster:
src="<host or destination name>/<UI5 version>/resources/sap-ui-cachebuster/sap-ui-core.js"

  • In some cases, sap-ui-core.js loads successfully, but subsequent framework resources fail to load.
  • In some cases, sap-ui-core.js itself fails to load

Environment

SAPUI5

Cause

UI5 document Variant for Bootstrapping from Content Delivery Network states

"The [SAPUI5 framework] Cache Buster is only needed if you consume SAPUI5 without a concrete version in the URL. When you consume SAPUI5 with the concrete version in the URL, this is not needed, as the content served by that unique URLs will never change and can be cached forever."

Mixing the use of cache-busting framework with a specified version in the URL has never been considered a valid approach. It may have appeared to work in the past by coincidence, but with recent UI5 updates and changes to underlying mechanisms, the service now behaves more strictly.

Resolution

The underlying infrastructure, that serves SAPUI5 resources, may change at any time while ensuring that only documented and valid bootstrap options keep working. Therefore, when consuming SAPUI5 from the CDN URL, ensure that /sap-ui-cachebuster is not used together with a specified SAPUI5 version. Use the bootstrap URL either without /sap-ui-cachebuster or without a concrete SAPUI5 version.

If the HTML that bootstraps SAPUI5 is not controlled by your organization, please contact SAP Support.

Symptom

  • In SAP GRC EAM, after running transaction code GRAC_EAM, the status of one specific FFID is red. 
  • The "Log on" button is greyed out, and no FF user ID is displayed in the "Used by" field, as the FF user ID has already logged off the system. 
  • To resolve the issue, the GRAC_FFSESSION transaction was run, and the Force Logoff action was performed. 
  • The SPM Maintenance job was scheduled, and SM04 was run in the plugin system, but the issue persists. 
  • Additionally, the specific FFID is not locked in the plugin system, as per the analysis conducted.

Environment

  • SAP GRC Access Control 10.0 
  • SAP GRC Access Control 12.0 
  • SAP Access Control for SAP S/4HANA
  • SAP Access Control 12.0 for SAP S/4HANA

Reproducing the Issue

  1. Run transaction code GRAC_EAM.
  2. Launch the Emergency Access Management window.
  3. Verify that the status for FFID is red, and the "Log on" button is greyed out.
  4. Confirm that no FFUSER ID is displayed in the "Used by" field.
  5. Perform resolution steps such as running the FFLOG synchronization job, GRAC_FFSESSION (Force Logoff), and SM04 in the plugin system. If the issue persists, proceed with further troubleshooting.

Cause

  • Known issues with Firefighter session handling have been identified.
  • These issues were resolved in updates released after SP 24.

Resolution

  1.  The issue is with Firefighter session handling.  
     Based on the details provided, there are a couple of known issues that have been addressed in updates following SP24:  
       - SAP Note 0003527725: Force Logoff action in GRAC_FFSESSION ends with an error, leaving the lock in FFID.  
       - SAP Note 0003476222: Firefighter Sessions without actual logons.
  2.  To resolve this, please have your ABAPer implement the SAP Notes mentioned above.  
     Once applied, kindly retest the issue and let us know if further assistance is required.

See Also

  • SAP Note 0003476222: Firefighter Sessions without actual logons. 
  • Contact SAP Support for further assistance if the issue persists after implementing the SAP Notes.

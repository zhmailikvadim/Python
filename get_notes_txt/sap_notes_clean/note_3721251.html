Symptom

In My Timesheet (Fiori 2.0) application, the Value Help (F4) for WBS Element (POSID) and a custom field shows no entries, although valid data exists in the backend.

Other Terms

My Timesheet Fiori 2.0, V3, F3074, HCMFAB_CATS_GET_VALUEHELP1,WBS Element, POSID, Custom Field, Value Help, Search String

Reason and Prerequisites

The issue occurs because:

  • Search string is not populated in FM HCMFAB_CATS_GET_VALUEHELP1, resulting in no values being returned.
  • For certain attributes, the read logic incorrectly accesses index 1 instead of the actual field name, resulting in no matching values being returned.

Solution

Apply the attached correction instructions, alternatively apply corresponding support packages.

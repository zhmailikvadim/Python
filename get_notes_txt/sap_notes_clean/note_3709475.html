Symptom

Query functions let you run queries during pipeline execution, removing the need for custom stored procedures.

Environment

  • SAP Sales Performance Management
  • Incentive Management

Resolution

Overview:

  • Execute queries during pipeline runs without custom stored procedures.
  • Retrieved data can be used by professional services to create custom SQL that extends product functionality.

Common parameters (applicable to all query functions):

  • Name of the query (required)
  • String parameter (optional)
  • String parameter (optional)
  • Value parameter (optional)
  • Value parameter (optional)
  • Boolean parameter (optional)
  • Boolean parameter (optional)
  • Date parameter (optional)
  • Date parameter (optional)
  • Default value to return if the query returns no result or null (optional)

Important notes about the custom SQL query:

  • The query can be placed in a compensation rule.
  • Supports common return types: Value, Number, String, Date, Boolean.
  • Can reference context values (e.g., current period, current transaction, current position, current payee).
  • Can reference user-provided parameters for use in SQL.
  • DML or DDM operations (Update, Delete, Insert, Create, Alter, etc.) are restricted.
  • Not visible in the UI.
  • Times out if it runs longer than 5 seconds.
  • The query is profiled.

Symptom

 In the Fiori application, when a single line item schedule header is added for the first time, the item is creating successfully. However, after deleting it and attempting to add the same schedule again, the item is not being created, As a result, an incorrect balance amount is displayed in the schedule header.

Other Terms

/IRM/IPAGRM, Single Line item Schedule, schedule_header_delete

Reason and Prerequisites

This is caused by Program error.

Solution

Please implement the correction instructions. These corrections are only valid if you have applied the following Support Packs.

Release V/4 – 2021, 2022, 2022 V2, 2023, 2023 V2, 2023 V3 and 2023 V3 Support Package 1

Use transaction SNOTE to implement the correction instructions specified below and activate the affected program object

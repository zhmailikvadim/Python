Symptom

  • Copy transactional data task creates transaction type (RMVCT) entries despite the source data not containing it.
  • Error message: "Value XXX in field 'Transaction type' overwritten with 615. Message No. G01165."

Environment

SAP S/4HANA Finance for Group Reporting

Reproducing the Issue

  1. Use transaction code CXCK to run the copy program.
  2. Observe that the RMVCT (transaction type) is not empty in the copied data.

Cause

Standard behaviour until technical note "3554327 - Sender Field Consistency Only for Non-Initial Sender Fields" is missing. Furthermore the earlier uploaded data (with transaction type) in the target version causes this behavior if technical note 3554327 is already implemented.

Resolution

  1. Implement SAP Note 3554327 - Sender Field Consistency Only for Non-Initial Sender Fields.
  2. Ensure that the target system does not have any RMVCT (transaction type) uploaded data before executing the copy task.
  3. Test the copy functionality in a period where no uploaded data exists in the target system.
  4. If the issue persists, recheck the parameters used for the copy function and ensure no conflicting data exists in the target system.

See Also

  • Refer to: 3554327 - Sender Field Consistency Only for Non-Initial Sender Fields.

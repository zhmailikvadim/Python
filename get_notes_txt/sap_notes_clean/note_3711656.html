Symptom

  • The Job Information field 'Expected return date' is not visible in Legacy People Profile but visible in Latest People Profile.

  • The RBP for 'Expected return date' field is assigned for the target user.

  • The rule trace shows that no rule is setting the visibility of the field in both Latest and Legacy People profile.

  • The user does not have LOA record in Job History.

Environment

SAP SuccessFactors Employee Central

Reproducing the Issue

  1. Go to the Employee Profile of the affected user.
  2. Observe that the Job Information field 'Expected return date' is not visible in Legacy People Profile but visible in Latest People Profile.

Cause

  • This is Expected behavior.

Resolution

  • On the Legacy People Profile, there was some specific logic to completely hide the field if it was empty, i.e. the user was not on LOA. Refer - Job Information | SAP Help Portal that 'This field will only be displayed for leave of absence job history records.' 

  • However, on the Latest People Profile, empty fields are handled a bit differently, since here they are hidden behind the “show empty fields” button. If this is expanded, the Expected Return Date will also be shown if it is empty, following this new logic of handling empty fields on job info, and to be consistent with other fields. 

  • Hence, as per the behavior of Legacy People Profile, the field is not shown as the user is not on LOA. And on the Latest People Profile, the field is shown as blank as per the new logic of handling the empty fields.

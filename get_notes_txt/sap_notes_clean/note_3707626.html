Symptom

How to Set Up the Condition for "Owner Technical ID" and "Created By(Technical ID)"/""Updated By(Technical ID)" for Leads in the Autoflow? The expectation is to create a autoflow rule with the condition: "Owner Technical ID(ownerID)" Not Equal to "Created By (Technical ID)".

Environment

  • SAP Sales Cloud Version 2
  • SAP Sales and Service Cloud Version 2

Reproducing the Issue

  1. Go to Settings ->Business Flow -> Autoflow 
  2. Click + -> Rule
  3. Select the target Service Leads
  4. Select Event Type 
  5. Set the condition: Owner Technical ID(ownerID) Not Equal to Created By (Technical ID)
  6. Go to Leads -> Update/create the leads
  7. This autoflow is triggered unexpectedly when it doesn't meet the condition 

Cause

"Owner" in an lead is an employee whereas "Created By"/"Updated By" is a user. Hence the Technical Ids for employee and user are Not same.

Resolution

Please follow the steps below to set up the condition.

  1. Go to Settings -> Extensibility Administration -> Select the target entity like leads
  2. Go to Reference Fields Usage tab -> change the dropdown list to Autoflow -> Click + 


  3. Add a reference field for employeeId under the "Created By User" and Click OK 




  4. Once the field is added, please refresh the tenant 
  5. Go to Settings ->Business Flow -> Autoflow 
  6. Create a new rule or update the existing 
  7. Create/Update it with the condition: "referenceUsages/createdByUserEmployeeId" Not equal to "Owner Technical ID(Owner/id)"
  8. Click Save and make sure this rule is Active

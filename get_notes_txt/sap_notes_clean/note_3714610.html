Symptom

Manual delegations automatically removed after import in Parent-child environments

Reproducing the Issue

  1. Log into the child realm.
  2. Navigate to Manage > Core Administration
  3. Go to User Manager > Delegations
  4. Create a Delegation.
  5. Enter the Delegator's details (e.g., name and ID).
  6. Enter the Delegatee's details (e.g., name and ID).
  7. Specify the Delegation start and end dates.
  8. Provide a reason for the Delegation.
  9. Save the Delegation.
  10. Go to the parent realm.
  11. Go to Manage > Core Administration
  12. Navigate to Site Manager > Data Import/Export
  13. Run the Import User Delegation task
  14. Observe that the Delegation is removed after replication.

Cause

When the Import User Delegation task is executed, the system processes the file as follows:

  • The imported file is treated as the single source of truth.

  • All existing delegations in the system are reset.

  • New delegations are created exclusively based on the data contained in the import file.

Resolution

This behavior is standard and expected system functionality. During import events, the system overwrites the existing delegation records and reconstructs them strictly according to the contents of the imported file.

In parent–child (multi-variant) environments, an additional behavior must be considered:

After the replication process is completed, if the Import User Delegation task is executed:

  • Delegations that were recreated manually via the UI (e.g., through UP requests or Core Administration) are removed.

  • The import process overwrites all existing delegations and retains only those present in the import file.

As a result, any delegations re-created through the UI will be deleted after replication if delegation updates are subsequently performed via the import process.

This behavior is working as designed. In environments where delegation updates are managed through integration, the import file must always contain the complete and accurate set of required delegations. Any delegations not included in the import file will be removed during the import process.

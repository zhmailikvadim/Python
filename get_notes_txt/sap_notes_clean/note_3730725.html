Symptom

  • You have set up Audit Logging in the SAP Business Technology Platform (SAP BTP. URL: https://cockpit.btp.cloud.sap) Cloud Foundry environment. 
  • The configurations have been done as per SAP KBA ##2637286 : How to trigger Audit Logs for SAP Business Technology Platform (SAP BTP).
  • You would like to know how to check the current audit log data size for SAP BTP subaccounts and Global Accounts so as to calculate the monthly costs etc. and to plan for the future.

Environment

  • SAP Business Technology Platform (BTP)
  • Cloud Foundry environment
  • Neo environment
  • Audit Log Service for SAP BTP

Reproducing the Issue

Trigger Audit Logging in the Cloud Foundry Environment.

Cause

Resolution

To check the current audit log data size for SAP BTP subaccounts and Global Accounts:

  1. Access SAP Business Technology Platform (SAP BTP. URL: https://cockpit.btp.cloud.sap) cockpit.
  2. Open the Global Account and navigate to Usage.
  3. Select the service “Audit Log Service”. Choose the metric “Usage" view the GB of data used. 
  4. Review the Global Account view and use the breakdown by subaccounts to see the stored volume per subaccount. For more details, check: Monitoring Usage and Consumption Costs in Your Global Account.
  5. Alternatively, open the target subaccount. Navigate to the Usage Analytics tab, select “Audit Log Service”, and review the “volume stored” metric for that subaccount. For more details, check: View Subaccount Usage Analytics.

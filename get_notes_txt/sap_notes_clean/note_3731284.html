Symptom

  • A business rule configured to hide a custom list field for the first matrix manager role on a performance management form is not applied; the field remains visible.
  • The same rule logic works when targeting employee or manager roles.
  • The route map step is configured as “all matrix manager” for the relevant step.
  • The business rule condition targets “first matrix manager”.

Environment

  • SAP SuccessFactors Performance & Goals

Reproducing the Issue

  1. Configure the route step for "All Matrix manager"
  2. Set the business rule to hide specific field only for "First matrix manager"
  3. Launch the form and route to All Matrix manager step
  4. Proxy as First matrix manager and navigate to the section containing the target custom field.
  5. Observe that the field remains visible although a rule should hide it for the first matrix manager role.

Cause

The behavior is by design: the route map step set to “all matrix manager” uses role EP, while “first matrix manager” corresponds to role EX.

Resolution

A rule targeting EX does not apply when the active step role is EP. Rule evaluation also targets the step name first and then the step role.

Possible workarounds:

  1. If the requirement is to apply only to the first matrix manager:
  2. Configure the route map step to use the first matrix manager role (EX), or
  3. Include EX together with EP in the same step (C or I step) and place EX as the first role in that step, then retest the rule.
  4. If the requirement is to apply to all matrix managers:
  5. Define the business rule using the form route map “current step” rather than a specific role, then retest.
  6. Ensure the rule references both the correct step name and step role, as rule logic evaluates the step name first and then the role.
  7. If differentiating the first matrix manager within an “all matrix manager” (EP) step is still required beyond the options above, consider submitting an enhancement request.

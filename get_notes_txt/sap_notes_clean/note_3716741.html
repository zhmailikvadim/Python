Symptom

There is info record exist for Material Master. 

When create PR, planned Delivery Time (PLIFZ)  does not come from info record.

Environment

  • Materials Management (MM)
  • SAP ERP Central Component
  • SAP ERP
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA

Reproducing the Issue

  1. Create PR in ME51N,
  2. Set Source determination indicator,
  3. Fill material, quantity and plant, and PIR was determined as source of supply, in this case, planned delivery time is determined from PIR,
  4. Maintain storage location, then planned delivery time will be overwrite from material master(MM03->MRP1 view-> MRP areas button->details).

Cause

When set source of supply, system determined PIR first and planned delivery time is determined from the relevant PIR(EINE-PLIFZ).

Later provide Storage Loc, with this system call FM MD_MRP_AREA_GET.

System will always have to determine all the required information's after any manual input.

This is because manual input value has higher priority.

Resolution

This is system standard design.

Manual input value always has priority.

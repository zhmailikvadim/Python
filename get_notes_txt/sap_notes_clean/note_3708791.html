Symptom

In the Pre-Production and Production environments, SAP Cloud Platform Integration (CPI) is unable to reach the MCM system.

The following error appears in the logs:

"SSL Handshake failed sun.security.validator.ValidatorException: PKIX path building failed: sun.security.provider.certpath.SunCertPathBuilderException: unable to find valid certification path to requested target"

This error indicates a problem related to SSL certificate validation.

Environment

SAP IS-U

Cause

The root cause is an outdated or missing SSL certificate for the MCM service in SAP API Management.
When the MCM SSL certificate is renewed or changed, the new certificate must be uploaded to the API Management system.

Without this, the SSL handshake cannot be completed due to a missing trust path. 

Resolution

  1. Update the SSL certificate for the MCM endpoint in SAP API Management.
  2. After updating, integration should resume. As confirmed in previous cases, this process resolved the issue for the test (pre-prod) environment.
  3. For Production, please ensure the same certificate update is carried out.

Keywords

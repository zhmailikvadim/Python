Symptom

  • Unable to fetch records of MSEG table in the target
  • No rows are displayed in target for the MSEG table.

Environment

  • S/4HANA 2023
  • SAP Landscape Transformation Replication Server (SLT)

Cause

The issue occurs due to a data model change in the source system, where the content of the MSEG table is no longer available in S/4HANA 2023. The relevant data is now stored in the MATDOC table.

Resolution

  1. Stop the subscription for the MSEG table in the SLT
  2. Replace the subscription for the MSEG table with a subscription for the MATDOC table to extract the required data.
  3. Test the replication process in a non-productive environment to ensure the required data is successfully extracted from the MATDOC table.
  4. If the replication is successful in the non-productive environment, implement the changes in the productive system.
  5. Monitor the replication process to ensure proper functionality.

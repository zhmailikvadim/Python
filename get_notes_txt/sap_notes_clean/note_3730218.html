Symptom

Excessive work processes are utilized when COHV performs batch closure of production orders or other processing. It is expected to specify which application server(s) should execute the COHV mass closure process and to set a maximum limit on the number of work processes that can be used. 

Environment

  • SAP ERP Production Planning (PP)
  • SAP R/3
  • SAP R/3 Enterprise
  • SAP ERP Central Component
  • SAP ERP
  • SAP Enhancement package for SAP ERP
  • SAP Enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA, on-premise
  • SAP S/4HANA Cloud Private Edition

Reproducing the Issue

  1. Open transaction COHV and perform mass processing to close production orders in mass.
  2. Observe that a high number of work processes are started, potentially across multiple application servers.

Cause

Default system behavior. 

Resolution

1. In transaction RZ12, create server group dedicated to COHV processing:

Create a group named COWORK-BATCH for dialog processing of COHV.

Create a group named COWORK-DIALOG for background processing of COHV.

2. In transaction RZ12 assign the selected application server(s) to these groups to restrict processing to specific servers. Set the number of work processes to limit the maximum number used for each specific application server. 

3. In transaction COHV  open the Mass Processing – Close tab and set the maximum number of work processes according to your capacity and requirements through "Max. No.of Processes" input field. 

4. Save and test the mass close to verify the expected server selection and work process utilization.

Symptom

  • Server Intelligence Agent (SIA) failing to start via script.
  • When using scripts with net stop/start to start BI services, the error "Access is denied" appears.
  • Manually stopping and starting succeeds.

Environment

  • SAP BusinessObjects Business Intelligence Platform 4.3
  • SAP BusinessObjects Business Intelligence Platform 2025
  • Windows
  • Linux / Unix 

Reproducing the Issue

  1. Run a script that uses net stop/net start BI Services.
  2. Observe the error "Access is denied".
  3. Stop and start the service manually to confirm it works without scripts.

Cause

The service account running the Server Intelligence Agent lacked the "Act as part of the operating system" user right, causing the access denied error when using net stop/start from scripts.

Resolution

  1. Grant the service account running the Server Intelligence Agent the "Act as part of the operating system" user rights under the following:
    • Local Security Policy > Local Policies > User Rights Assignment.
  2. Rerun the start script.

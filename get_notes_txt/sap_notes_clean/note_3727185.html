Symptom

During the posting of a turnover item via the service BankAccountContractProcessingRevenueExpensePostingIn – CreateBankAccountRevenueExpenseEntryAsBulk, when an incorrect account number is passed in the xml payload of the service, it results in the creation of a postprocessing order.

In the "Payload - Financial Services" tab of the display of postprocessing order,  the "display payload" option does not work as intended and shows an empty payload.

Other Terms

/sappo/ppo2, postprocessing desktop, xml payload, MIF_BARE02 , CREATE_BULK_V1, CREATE_BULK_V2

Reason and Prerequisites

Program Error

Solution

Import the relevant Support Package for your release. You can use transaction SNOTE to implement the attached correction instructions in advance.

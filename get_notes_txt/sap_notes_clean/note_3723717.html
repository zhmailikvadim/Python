Symptom

In the Mobile Apps, the warning message shown when booking material consumption that exceed available stock is inconsistent across platforms:


Actual Behavior		Expected Behavior
AndroidiOSWindows
On mobile at the Create Material screen, when the selected quantity exceeding the available stock, the expected warning message does not appear.Warning message visible but not marked redNo warning message, only the quantity number is marked redWarning message visible and marked red

Environment

SAP Field Service Management

Reproducing the Issue

  1. Ensure that the UI permission ALLOW_BOOKING_ONLY_IF_AVAILABLE_ON_STOCK  is enabled in the user policy group
  2. Log in to the mobile app
  3. Select any open activity 
  4. Tap on the 'Create Material' button
  5. Select an item
  6. Navigate to the 'In stock' tab
  7. Choose one and memorize the stock value
  8. Enter a greater number than the stock level as quantity

Expected behavior: a warning message should appear

Actual behavior: There is no warning message, it is possible to book more material than the available amount

Cause

At 'Master Data Management' on the Item screen, the toggle 'inventoryItem' is set as false.

Resolution

  • Set the 'inventoryItem' toggle to true. This adjustment can be performed via 'Master Data Management' or from the backend system (e.g. ERP).
  • Once the 'inventoryItem' property is updated to true, the stock information will be considered when a material consumption is recorded by the technician on mobile.

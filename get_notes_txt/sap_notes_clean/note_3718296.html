Symptom

  • Certain services, such as /sap/bc/bsp/sap/bsp_veri, do not exist in the SICF transaction, however testing the service via URL displays a logon screen instead of a "404 not found" error.
  • Further cancel the logon, error messages observed: "401 Unauthorized" and "net::ERR_INVALID_AUTH_CREDENTIALS".
  • After input the correct logon data, it then returns 404.

Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.

Environment

  • SAP AS ABAP systems

Reproducing the Issue

  1. check "sap/bc/bsp/sap/bsp_veri" >>this service doesn't exists in SICF
  2. Try to launch https://host:port/sap/bc/bsp/sap/bsp_veri?saml2=disabled
  3. Observe that a logon screen appears instead of a "404 not found" error

Cause

As long as a handler list class is maintained in one of the URI, it is considered a valid request. The URI "/sap/bc/bsp/sap/bsp_veri" has a handler list class configured in the node "/sap/bc/bsp". The control is handed to the BSP after a successful logon, and it is the responsibility of BSP to check whether the BSP application exists or not.

Launch https://host:port/sap/bc/bsp?saml2=disabled, results the same behavior.

404 is returned after a successful logon.

Resolution

  1. Verify that the service /sap/bc/bsp/sap/bsp_veri does not exist in the system using the SICF transaction.
  2. Confirm that the handler list class is maintained in the URI /sap/bc/bsp.
  3. Understand that the control is handed to the BSP after a successful logon, and it is the responsibility of BSP to check whether the BAP application exists or not. This is per design.

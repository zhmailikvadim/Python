Symptom

 Fetch Documents returned a "No documents found in online search" message for Work Orders with IDs longer than 10 characters

Other Terms

  • SAP Service and Asset Manager
  • SAM
  • SSAM
  • ICMTANGOAMF10-43431

Reason and Prerequisites

Reason:

The Document Fetch query was applying both ObjectId and OrderId filters for Work Order searches. Since ObjectId(reservation number) is limited to 10 characters, it could not match Work Order IDs longer than 10 characters, causing the query to return no results.

Prerequisites:

  •  BAS or Visual Studio Code with SAM MDK project
  • `patch` command (available on BAS and macOS by default) via terminal or other patching utility
  • Read the guidelines on how to apply the patch and read the appendix section if you receive any error

Solution

The query is updated to omit the ObjectId filter when the Work Order ID exceeds 10 characters and rely on OrderId to retrieve the associated reservation documents.

To apply the patch, download the attached 3731986.patch file and save it in a known location, preferably adjacent to your SAM.mdkproject.

If you are on Windows, see SAP Note 3533879 for how to install a .patch file in Windows. If you are on a Mac, follow the steps below:

  1. Open a terminal in the root of your SAM metadata (i.e. SAM.mdkproject/metadata or SAM.mdkproject/metadata/base.definitions)
  2. Run the terminal command `patch --no-backup-if-mismatch -p4 < path/to/3731986.patch`. 
  3. Verify in the terminal window that the following files have been modified and listed as patched, as indicated by the console output:
  • patching 'Rules/Inventory/Fetch/InventoryObjectsQueryOptions.js'

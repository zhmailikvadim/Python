Symptom

The Identity Provisioning Service (IPS) Job fails with the following error displayed in job logs:

error=HTTP operation failed invoking https://<IAS_tenant_URL>/scim/Bulk with statusCode: 400, Response: {"schemas":["urn:ietf:params:scim:api:messages:2.0:Error"],"status":"400","scimType":"tooMany","detail":"Bulk operations count exceeded the maximum value supported."},

Please note this error could also occur for other application APIs, for example SAC:

error=HTTP operation failed invoking https://<SAC_URL>.analytics.cloud.sap/api/v1/scim/Bulk with statusCode: 413, Response: {"status":"413","detail":"The number of the bulk operation exceeds the maxOperations (100)","schemas":["urn:ietf:params:scim:api:messages:2.0:Error"]}, 

Environment

SAP Identity Provisioning

Resolution

This is a known issue and IPS development support team is working on a fix. As a workaround, please disable the bulk operation by setting the property ias.support.bulk.operation to false in the IAS system or if the error is mentioning SAC API, then set sac.support.bulk.operation to false in the SAC system.

Keywords

Bulk, operation, max, 400, 413, exceeds

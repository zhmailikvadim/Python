Symptom

  • Planner users are unable to access SAP Analytics Cloud stories to view and plan data for Cost Centers they are authorized for via Data Access Control (DAC).
  • When opening affected stories, the Cost Center filter appears empty or does not display the authorized nodes.
  • The same users can upload flat files for the same set of Cost Centers authorized via DAC.
  • The issue is isolated to the story level and does not affect the data model or DAC configuration.

Environment

SAP Analytics Cloud

Reproducing the Issue

  1. Access the SAP Analytics Cloud tenant using a user role equivalent to the affected users.
  2. Navigate to the story.
  3. Click on the Cost Center filter dropdown and observe that only some members are visible, while authorized members are not displayed.
  4. Navigate to the story number 2.
  5. Click on the Cost Center filter dropdown and observe that authorized members are visible for this story.

Cause

The issue is caused by the combination of Data Access Control (DAC) with "Hide Parents" enabled and hierarchical story filters. When the cost center filter is modified to select an intermediate hierarchy node, the access-control evaluation or filter-rendering omits members in the stories, leading to the filter appearing empty for authorized users.

Resolution

  1. Option 1: Set "Hide Parents" to OFF in the dimension configuration to allow leaf nodes to inherit permissions.
  2. Option 2: Ensure explicit read/write access is defined for all relevant nodes in the hierarchy, including intermediate nodes.
  3. Option 3: With "Hide Parents" enabled, select "All members" in the story filter and then manually exclude unwanted members from the filter. This ensures that authorized users can access the relevant data.
  4. If none of the above options are feasible, create a separate hierarchy for cost centers containing only the members required for the story filter.

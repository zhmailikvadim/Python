Symptom

In SAP Plant Connectivity (PCo), Socket Agents may fail to receive notifications or data from connected devices after idle periods.

Affected data does not appear in the PCo Log Viewer, and a restart of the Socket Agent or the device may be required to restore communication.

Other Terms

Plant Connectivity, PCo, Socket Agent, TCP, Connection issue, Idle period

Reason and Prerequisites

In SAP Plant Connectivity (PCo) version 15.5.5.x, Socket Agents use an OS-level socket health check by default to verify active connections. Certain devices, however, require the legacy behavior where a 1-byte NULL (0x00) is sent during idle periods to maintain the connection. Without this legacy check, notifications or data may be missed.

Solution

Install Patch 07 for SAP Plant Connectivity 15.5 with Support Package 5.


Download the current installation program for your release from the SAP Service Marketplace and install it in accordance with the instructions in the installation guide. The current version of the installation guide is available on the PCo documentation page in the SAP Help Portal at https://help.sap.com/pco. You can use the installation program for the PCo patch to update a PCo version that is already installed or to set up a new installation if PCo is not yet installed on the target computer. The installation will install the required Visual Studio C++ runtime environment needed by the .NET Connector.

Before you start the installation, refer also to the installation instructions for PCo 15.5.5 in SAP Note 3480115.

Follow the steps below to configure the legacy behavior of socket agent for connection stability:

1. On the PCo server, navigate to:
    C:\ProgramData\SAP\PCo\Config\Instances

2. Locate the *.exe.config file for the corresponding Socket Agent instance (the one that connects to the printer/device).

3. Open the .exe.config file in a text editor with admin  and, inside the existing <appSettings> section, add/update the following line:
    <add key="UseLegacyNullProbe" value="true" />

4. Save the file and restart the corresponding PCo Agent instance.

Explanation of the configuration:

  • When set to true, the Socket Agent uses the legacy check by sending a 1‑byte NULL value (0x00) on the TCP socket during active connection checks, matching the behavior from legacy PCo, where printers received a NULL every 30 seconds.
  • When set to false or if the key is missing, the Socket Agent uses OS-level socket health checks and does not send any data to the device during the connection check.

After completing these steps, the Socket Agent will perform the legacy NULL probe check, ensuring stable communication with the device during idle periods.

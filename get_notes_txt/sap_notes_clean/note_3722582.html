Symptom

 In the dialog, which was configured from the backend for attestation, it was not showing more than 255 characters for the attestation text.

Other Terms

Attestation test , approval hub

Reason and Prerequisites

The text was being fetched from the ‘AprAttestation’ entity where it was not possible to maintain more than 255 characters.
Must have /FC1/APPR_ATTEST table in your system.

Solution

The backend now uses OTR object to store texts for attestation. The text alias is stored in the main attestation entity instead of the entire text.

The frontend now uses the alias from the main entity to fetch the text for the attestation from OTR.

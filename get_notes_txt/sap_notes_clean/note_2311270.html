Symptom

When you need to export a data table from SAP Business One using the Copy Table functionality, the unauthorized values that should display in Asterisks as **** are still visible.

Reproducing the issue

Scenario:

  1. Set up authorization for a user and disable option to see Last Purchase Price for example in Administration → System Initialization → Authorizations → General Authorizations, under the subject Inventory → Price Lists → Price Lists → Last Purchase Price.
  2. Re-logon to SAP Business One with the user in Step 1.
  3. Open the Item list report in Inventory → Inventory Reports → Items List. The values in column Last Purchase Price will be seen as ****.
  4. Export the data table. Right-click the table, choose Copy Table, and then paste on the excel report. The block values in column Last Purchase Price are now visible.

Expected behavior:

The unauthorized values are viewed as Asterisks when using the function Copy Table.

Actual behavior:

The unauthorized values still appear when copied using the function Copy Table.

Solution

SAP intends to provide a patch or patches in order to solve the problem described. The section Reference to Related Notes below will list the specific patches once they become available. The corresponding Info file of the patches in SAP Service Marketplace will also show the SAP Note number. Be aware that these references can only be set at patch release date. SAP will deliver patches only for selected releases at its own discretion, based on the business impact and the complexity of the implementation.

Note: This issue does not occur for SAP Business One 9.2 PL4.

Workaround:

Export the data via the upper menu path File → Export → MS-EXCEL or via the icon  from the toolbar. The block values in column Last Purchase Price remain invisible.

Symptom

Data Flow and Replication Flow connection (or other connections which us similar technology) failed with credentials error like:

  • Data flows can’t be used. Please re-enter your credentials in order to establish connectivity for data flows.
  • Replication flows can’t be used. Please re-enter your credentials in order to establish connectivity for replication flows.

Also, Data Flow and Replication Flow execution fail with error message like:

  • "Could not fetch credentials from DWC: Could not fetch credentials from DWC: status code 400"
  • [ReplicationFlow] Failed to fetch the data from server. Error in getReplicationFlow. Failed to retrieve replication flow

Environment

SAP Datasphere

[2026.03.06]

RCA finalised.

Root Cause Analysis:
* An internal network component enabled a new authentication method that should have allowed internal services to smoothly switch the authentication scheme
* The activation of the feature resulted in a short timespan where services could use both methods at the same time.
* An internal service blocked this switch of the authentication which led to a failure retrieving data needed by Datasphere Data Flows and Replication Flows
* The situation was not detected during the validation phase because the tests of the internal service had the new authentication scheme already activated and were not testing the situation with both schemes enabled unlike the production cluster.
 
Preventive Measures:
* Tests for the switch of the authentication mechanism have been added for the internal service
* Automatic alerts for the specific failure will be enhanced to react more quickly in similar cases in the future
* Rollout of such features will be safeguarded by additional measures from operations team

Symptom

  • When a candidate's ID number does not comply with the format specifications, after RCM initiates onboarding, the button displays “Initiation Successful,” but the “Business Process” section in the upper-right corner of ONB shows an error.
  • The notification of the error is not intuitive and requires clicking through to view the error.
  • There is no alert mechanism, such as email notifications, to inform responsible personnel about the error.

Environment

  • SAP SuccessFactors onboarding

Reproducing the Issue

  1. Initiate onboarding for a candidate whose ID number does not comply with the format specifications.
  2. Observe that the “Initiation Successful” button is displayed.
  3. Navigate to the “Business Process” section in the upper-right corner of ONB.
  4. Observe that an error is displayed in this section, which is not intuitively notified to responsible personnel.

Cause

  • The issue occurs due to the lack of alternative notification methods for business process errors in the current system functionality. Currently, there is only one standard template for such notifications, and the error notification is not prominently displayed.

Resolution

  1. Confirm that the `ONB Suite mapping error` is enabled and triggered regularly in the instance.
  2. Refer to the attached screenshot for additional details about the configuration and the triggered notifications.
  3. If alternative notification methods are required, such as email alerts to responsible personnel, submit an Enhancement Request to include this functionality in future releases.
  4. For more information on email triggers in onboarding processes, refer to the SAP Help Portal link provided in the "See also" section.

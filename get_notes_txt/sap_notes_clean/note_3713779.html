Symptom

The asynchronous service BusinessPartnerDataManagementBusinessPartnerIn->UPDATE_BP_AS_BULK is not updating the value of an existing IDENTIFICATION Code, whereas insert and delete of identification code are functioning correctly.

Environment

  • SAP Business Partners - Customer Vendor Integration in SAP ERP and SAP S/4HANA
  • SAP ERP
  • SAP ERP Central Component
  • SAP enhancement package for SAP ERP
  • SAP Supplier Relationship Management for SAP ERP
  • SAP S/4HANA

Reproducing the Issue

  1. Access the asynchronous service BusinessPartnerDataManagementBusinessPartnerIn->UPDATE_BP_AS_BULK.
  2. Attempt to update the value of an existing IDENTIFICATION Code
    Error messages encountered:
    Postprocessing order could not be created (010(FSL_ERROR_CONFLICT))
    Postprocessing order could not be created (005(ECH_MSG))
    Error when formatting the data for ECH (014(FSL_UTILITIES))
    SWC SAP_BS_FND does not exist (012(FSL_UTILITIES))
  3. Observe that the update does not work, while insert and delete operations function correctly.

Cause

The id number is a key field of table BUT0ID, which prevents modification of existing identification codes.

Resolution

  1. Delete the existing identification code in the BUT0ID table.
  2. Recreate the identification code with the new value.
  3. Note that it is not technically possible to update the identification code directly due to the design of the table.

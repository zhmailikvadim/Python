Symptom

When creating production order in CO01, different planning plant and production plant were input. When posting goods receipt, the goods receipt will be received in planning plant, not production plant.

Environment

  • SAP ERP Production Planning (PP)
  • SAP R/3
  • SAP R/3 Enterprise
  • SAP ERP Central Component
  • SAP ERP
  • SAP Enhancement package for SAP ERP
  • SAP Enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA, on-premise
  • SAP S/4HANA Cloud Private Edition

Reproducing the Issue

  1. Create a production order in t-code CO01
  2. Input different planning plant and production plant
  3. Save the order in CO01
  4. When posting goods receipt of this order in t-code MIGO, the planning plant value is proposed for field "Plant" under "Where" tab.

Cause

For the different planning plant and production plant of production order, the goods receipt is received to planning plant. If the goods receipt needs to be received in production plant, use same planning plant and production plant in the order creation stage. Because the plant is related to master data and it is unchangeable after order creation in standard. 

Resolution

Standard behavior

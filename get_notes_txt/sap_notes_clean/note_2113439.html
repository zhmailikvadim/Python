Symptom

The replication of HR master data of Success Factors Employee Central  (SFSF EC) cannot be executed because the processing of the replication data terminates with an error.

If you search for errors in the application log (transaction SLG1), you find a message of the following type
- "Personnel number was already accounted. Do not change entry date" (PG038)
The log includes information messages in the language of the technical user for the replication service. The corresponding information message points out that the error has occurred in infotype 0000 (Actions).

Other Terms

Service Interface EmployeeMasterDataReplicationRequest_In,
Message Type EmployeeMasterDataReplicationRequest.
Service Interface EmployeeMasterDataReplicationConfirmation_Out,
Message Type EmployeeMasterDataReplicationConfirmation.

Reason and Prerequisites

The employee data replication service tries to change the infotype 0000 record which is valid at the hire date. But the personnel number is already accounted so no change of that infotype 0000 is allowed.

Solution

PA_SE_IN add on

1. Most probable:  USER ERROR: In the Employee Central you need to system check the oldest job information record. The effective start date must be the same as for the oldest infotype 0000 record in the payroll system.  If this the case you need to change the start date of the oldest job information record as it is not allowed anymore the hire date for an employee which is already accounted.
2. Data migration / Cutomizing change issue: If the job information history is correct check the GDT mapping data base table PAOCFEC_MAP_CC.  Check whether you find there entries like
GDT Name                             Code List ID          GDT Code Value   ERP value
PERSONNEL_EVENT_TYPE         21201                     H                       XY

 

 If this the case check also data base table pa0000 for the personnel number for which the error message is reported in the application log. If the field MASSN in the database table for the oldest begin date you need to either correct the mapping table entry in DB table PAOCFEC_MAP_CC or you need to correct the field massn in the database table pa0000 for the date of the hire.

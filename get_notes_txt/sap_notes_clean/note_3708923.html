Symptom

  • After migrating from SAP ECC to SAP S/4HANA, issues occur in Personnel Time Management processing:

    • Time Management records beyond the go-live date (03.11.2025) are not visible.

    • Records created before migration are visible in SAP S/4HANA but cannot be processed further.

    • Creation of certain absence types is not possible.

    • Absence quotas cannot be created or updated via infotype 2006.

    • Error message displayed during absence creation or quota maintenance:
      “No entries found in table T554Y_REAC”

    • In some cases, absences are posted with the current date instead of the requested date.

    • Issue mainly occurs after transporting ECC customizing to SAP S/4HANA Quality.

Environment

  • SAP S/4HANA 
  • SAP Human Capital Management (HCM) 
  • Personnel Time Management 
  • Time Data Recording and Management

Reproducing the Issue

  1. Perform system conversion or migration from SAP ECC to SAP S/4HANA.

  2. Ensure business function H4S4_1 is activated.

  3. Transport Personnel Time Management customizing from ECC Production to SAP S/4HANA Quality.

  4. Attempt one of the following:

    • Create absences for specific absence types.

    • Maintain absence quotas using infotype 2006.

  5. System raises error “No entries found in table T554Y_REAC” or Time Management processing behaves inconsistently.

Cause

  • With activation of business function H4S4_1, SAP S/4HANA introduces a new collision-check framework for time infotypes.

    Key points:

    • Table T554Y_REAC does not exist in SAP ECC.

    • SAP S/4HANA uses new customizing views:

      • V_T554Y_REAC – Reaction indicator for overlapping time infotypes

      • V_T554Y_MOCOL – Collision grouping

    • Table T554Y_MOCOL has delivery class C.

    • The SAP standard collision group “00 – SAP Standard” is delivered only in client 000.

    • In target clients, V_T554Y_MOCOL may be empty if customizing was not cascaded from client 000.

    • Conversion report RPU_PT_CONV_T554Y fails if entry 00 is missing, resulting in incomplete collision configuration and processing errors.

Resolution

    1. Maintain Collision Group “00 – SAP Standard”

      • Manually create or copy entry 00 in view V_T554Y_MOCOL from client 000 to the affected client.

      • Required if customizing was not automatically cascaded.

    2. Implement SAP Note 3299479

      • Ensures correct handling of collision grouping and prevents recurrence during system copies or upgrades.

    3. Execute Conversion Report

      • Run report RPU_PT_CONV_T554Y after ensuring V_T554Y_MOCOL is correctly maintained.

      • Transfers existing collision rules to the new SAP S/4HANA views.

    4. Verify Customizing

      • IMG path:
        Specify System Reaction to Overlapping Time Infotypes

      • Ensure views V_T554Y_REAC, V_T554Y_MOCOL, and V_T554Y_KLZBI are consistently maintained.

    After completing these steps, Personnel Time Management processing works as expected.

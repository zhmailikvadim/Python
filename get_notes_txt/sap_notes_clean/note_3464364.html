Symptom

When archiving ABAP print lists using SAP ArchiveLink and a correct Japanese device type (that is, with a Japanese 8000 output code page), the system replaces certain double-byte characters (Kanji) with # characters; the problem becomes obvious when the archived lists are displayed.

Environment

SAP NetWeaver release independent

Reproducing the Issue

  1. Execute transaction OAAD
  2. Input report name then execute
  3. Display the report

Cause

The code page for output type used in current scenario does not support Japanese characters.

Resolution

  1. Check the Output Devices used in current scenario then check the output type assigned to it and make sure the code page used by the output type support Japanese characters.
  2. If no output device is specified for the content repository used for output scenario then ARCH will be used as Output Devices by default.
  3. ARCHUTF8 can be used if it is not clear about which Output type should be selected.

Sample steps to update Output type for a Output Devices as below.

  1. Execute tcode SPAD

  2. Input ARCH as Output Devices then click Display

  3. Use change mode then select 「ARCHUTF8」 as the value of Device Type

  4. Save the configuration then try to generate a printlist and confirm the result

Keywords

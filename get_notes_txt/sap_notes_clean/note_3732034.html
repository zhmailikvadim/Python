Symptom

The integration flow is configured with AMQP sender adapter in Cloud Integration, but skips jobs without logs or failure status intermittently. The missed jobs do not generate message processing logs either.

Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.

Environment

  • SAP Business Technology Platform
  • SAP Integration Suite
  • Cloud Integration

Reproducing the Issue

  1. Configure two integration flows: one that receives inbound requests and publishes messages to an Event Mesh queue, and another that consumes from the same queue using the AMQP sender adapter.
  2. Observe that some scheduled executions are skipped: the publishing integration flow completes, but no message processing logs appear for the consuming integration flow.

Cause

The AMQP sender adapter is configured with 'Consume Expired Messages' deactivated and 'Max. Number of Retries' set to 0, which can lead to expired queue messages not being consumed and no retry attempts on transient delivery issues.

Resolution

Review the AMQP sender adapter configuration of the consuming integration flow, activate 'Consume Expired Messages' if messages may expire before consumption and need to be processed. Set 'Max. Number of Retries' from 0 to an appropriate value for the scenario, and verify retry behavior.

For more details, refer to SAP Help document: Configure the AMQP Sender Adapter and SAP Community blog: Cloud Integration: Connecting to Messaging Systems Using the AMQP Adapter (see 'Executing Retries or Not?' part).

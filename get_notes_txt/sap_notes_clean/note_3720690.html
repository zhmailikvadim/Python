Symptom

  • Files are not moving to the Archive or Delete folder via the SFTP sender channel.
  • Files sometimes move to the folders after two or three attempts but occasionally remain in the processing folder.
  • Error messages observed on trying the post processing “Archive”:
    org.apache.camel.component.file.GenericFileOperationFailedException: Cannot rename file from: <Sourcefile.txt>to:  <Archivefile.txt>Caused by: 3: Permission denied.  
  • Error messages observed on trying the post processing “Delete”:
    org.apache.camel.component.file.GenericFileOperationFailedException - Cannot delete file: <Filename.txt>Caused by: 3: Permission denied. 

Environment

  • Cloud Integration
  • SAP Integration Suite

Reproducing the Issue

  1. Place the file on the SFTP folder.
  2. Attempt to move the file via the SFTP sender channel.
  3. Post processing the file was not moved to Archive or Delete

Cause

The issue is caused by permission-related errors on the SFTP server, preventing the creation of directories or renaming files during post-processing actions like "Archive" or "Delete".

Resolution

  1. Verify the permissions of the user accessing the SFTP folder. Ensure the user has sufficient rights to create, rename, and delete files in the specified directories.
  2. Check the folder structure and confirm that subdirectory-level permissions are correctly configured.
  3. Enable verbose/trace logging on the SFTP server to capture detailed command sequences and identify any discrepancies.
  4. Test the scenario with a simplified folder structure and relative paths to isolate the issue.
  5. If the issue persists, open the ticket under LOD-HCI-PI-CON-FTP to update permissions on subdirectory levels as required.

Symptom

  • After upgrading to JDK 21 and deploying to a development environment, generating an access token via Postman fails.
  • Generating the access token works in a local environment.
  • A SAML authentication flow is initiated on the token request to a backoffice API in the development environment.
  • Postman cannot process SAML responses, leading to validation termination.
  • The issue is not observed in a QA environment which is not upgraded to JDK 21 yet where token generation succeeds.

Environment

SAP Commerce Cloud 2211 for JDK21

Reproducing the Issue

  1. Upgrade the application to JDK 21 and deploy to a development environment.
  2. Use Postman to request an OAuth access token against the backoffice API token endpoint.
  3. Observe that the response initiates a SAML authentication flow and the token is not generated, while the same request works in an environment before SAP Commerce Cloud 2211 for JDK21.

Cause

With SAP Commerce Cloud 2211 for JDK21, the legacy oauth2 extension is replaced by the new authorizationserver implementation; an outdated configuration (using oauth2 instead of authorizationserver) can contribute to the issue.

The configuration for oauth(JDK21) is different than the one before the JDK 21 upgrade:

Before JDK 21 upgrade:

Name: oauth2                                                                            
Context path: /authorizationserver

For JDK 21:

Name: authorizationserver
Context path: /authorizationserver

In this release, the oauth2 extension has been removed and replaced with a new OAuth implementation that comes with the new authorizationserver, resourceserver, and oauth2commons extensions. 

Resolution

Change the name from oauth2 to authorizationserver in the configuration from the manifest.json file.

From:

Name: oauth2                                                                            
 Context path: /authorizationserver

To:

Name: authorizationserver
Context path: /authorizationserver

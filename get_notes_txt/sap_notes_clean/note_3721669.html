Symptom

While running the archive object COPA2_*, it’s taking a long time and not generating any archive files. It’s using too much memory.

Environment

  • SAP ERP
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA Finance
  • SAP S/4HANA

Reproducing the Issue

1. Run the COPA2_* archiving object.
2. Observe that the process takes longer time and fails to generate archive files.
3. Note that the system consumes high memory, leading to job cancellations.

Resolution

  • Apply the packages described in SAP Note 0002763549. SAP recommend running the archive period‑based process (select the "Line items only" checkbox).
  • Implement SAP Note 0003414024 to increase the package size beyond 500.
  • Also ensure that SAP Note 0003106995 has been implemented. 
  • Archiving by company code can reduce data volume, implementing SAP Note 0003521709 (COPA2: Archiving per company code in S/4HANA).

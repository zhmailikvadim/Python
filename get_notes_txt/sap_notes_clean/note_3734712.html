Symptom

A HANA table partitioning is failing with : " Could not execute 'ALTER TABLE "SLTECC"."DBERCHZ1" PARTITION BY RANGE ("BELNR") ( PARTITION '000000000000'<= VALUES ...' in 35:41:20.713 hours . 
SAP DBTech JDBC: [2048]: column store error: fail to alter partition: [2593] Error during split/merge operation;exception 70002593: Repartitioning failed Step Split Main Fragment: Error during split/merge operation;Unexpected exception occurred. Error in communication, exception: exception 2111019: A timeout occurred while reading from the channel; $INFO$=timeout occurred; $PARAM$=erroneous SSL channel (since 12mues) 64 with request-id 425580734290715579 {0x00007d5108f72b90: id=64, channel={<NetworkChannelSSLFilter>={<NetworkChannelBase>={this=136130303371288, fd=585, refCnt=1, local=127.0.0.1/49550_tcp, remote=127.0.0.1/30003_tcp, sta"

Environment

SAP HANA 2.0

Resolution

As per 2380176 - FAQ: SAP HANA Database Trace

"This error indicates that a network communication was terminated by a timeout. It is often the consequence of a slow or hanging network request, but in some cases it can be desired to increase the timeout so that an operation can finish without being terminated prematurely.

For standard repartitioning (SAP Note 2044468) the timeout is per default set to 1 day and after that time it may be terminated with a message like:

exception 70002593: Repartitioning failed Step Split Main Fragment: 
Error during split/merge operation;Unexpected exception occurred. 
Error in communication, exception: exception 2111019: 
A timeout occurred while reading from the channel
This timeout can be adjusted with the following parameter (unit: s), e.g. to 10 days:

indexserver.ini -> [partitioning] -> request_timeout = 864000 "

Symptom

The accumulated depreciation recorded for the second partial retirement, which occurred on the same day as the first partial retirement, appears to be lower than the expected value based on the fixed asset configuration.

Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental. 

Environment

SAP Business One

Reproducing the Issue

Prerequisites: 

  • Working with Fixed Assets.
  • Under Administration → System Initialization → Company Details → Basic Initialization tab, the Calculate Depreciation By is set as 'Day'.
  • In Administration → System Initialization → Posting Periods, the 2024 posting period is set from 2024.01.01 to 2024.12.31. 
  • The depreciation type 'DT2', for example, is configured under Administration → Setup → Financials → Fixed Assets → Depreciation Types with the following specifications:
    • Method: Straight Line
    • Calculation Base: Yearly 
    • Calculation Method: APC / Total Useful Life
  • The asset class 'AC1' is configured under Administration → Setup → Financials → Fixed Assets → Asset Classes with the following specifications:
    • Depreciation Type: DT2
    • Useful Life: 36

Scenario:

  1. Navigate to Financials → Fixed Assets → Asset Master Data.
  2. Add a new fixed asset with information specified below:
    • Item No.: FA10
    • Asset Class: AC1
    • Useful life: 36   
  3. Go to Financials → Fixed Assets → Capitalization.
  4. Create a capitalization for the fixed asset with details outlined below:
    • Asset No.: FA10
    • Posting/Document/Asset Value Date: 01.01.2024
    • Total(LC): 3600
    • Quantity: 10
  5. Once the fixed asset capitalization is created, go to Asset Master Data → Fixed Assets → Depreciation tab and verify the monthly depreciation amount, which will be displayed as shown below:
  6. Navigate to Financials → Fixed Assets → Retirement and execute the following: 
  • First partial retirement with details as outlined below:
    • Posting Date: 10.04.2024
    • Quantity: 3 units
    • Total (LC): 500
  • Result: Calculated accumulated depreciation recorded in retirement journal entry is 99.51.

  • Second partial retirement with details as outlines below:
    • Posting Date: 10.04.2024
    • Quantity: 2 units
    • Total (LC): 600
  • Result: Calculated accumulated depreciation recorded in retirement journal entry is 64.55.

Expected behavior:

The second partial retirement created on the same posting date computes and records accumulated depreciation of 66.34 by using the planned depreciation that has been recorded.

Actual behavior:

The second partial retirement, created on the same posting date, calculates and records accumulated depreciation of 64.55 based on the planned depreciation adjusted for the fixed asset(already retired) included in the first partial retirement.

Resolution

This is the current system behavior.

If a fixed asset undergoes multiple retirements recorded in successive steps on the same day, each subsequent retirement considers the asset as already retired based on the first retirement entry for that day. As a result, for the subsequent partial retirement less accumulated depreciation is recorded.

Illustration of accumulated depreciation calculation:

When a partial retirement of 3 units was initially recorded for fixed asset 'FA10', the depreciation amount for subperiod 4 in the Asset Master Data was adjusted from 98.52 to 78.87.

The accumulated depreciation for the remaining 7 units of the fixed asset is calculated as follows:

Accumulated Depreciation at Year Beginning – Accumulated Depreciation Reversed from First partial retirement + Total Depreciation Amounts (Periods 1 through 3) + (Period 4 Depreciation × 10 / 30)

= 0 - 99.51 + (101.81 + 95.24 + 101.81) + (78.87 × 10 / 30) = 225.64

For the 2 units retired in the second partial retirement (occurring on the same date), the accumulated depreciation allocation is calculated as:

225.64 × 2 / 7 = 64.55

Workaround:

When Calculate Depreciation By is configured as 'Day' under Administration → System Initialization → Company Details → Basic Initialization, enter both partial retirements on the final day of the month. Doing so will ensure that the accumulated depreciation value calculated for the second partial retirement achieves the intended outcome.

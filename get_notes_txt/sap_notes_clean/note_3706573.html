Symptom

  • Time unit on the wagetype was changed from days to hours, resulting in an error when entering "1" in CAT2.
  • Error occurs in CAT2 when attempting to input data for the wagetype.

Environment

  • Human Capital Management (HCM)
  • SAP R/3
  • SAP R/3 Enterprise 4.7
  • SAP ERP Core Component
  • SAP ERP
  • SAP enhancement package for SAP ERP
  • SAP S4-HANA 

Reproducing the Issue

  1. Change the time unit on the wagetype from days to hours in V_T511.
  2. Attempt to enter "1" in CAT2 for the modified wagetype.

Cause

  • CAT2 reads the configuration only once and cannot process multiple unit settings.
  • CAT2 does not read all V_T511 entries, including delimited ones, and picks only one valid entry.

Resolution

  1. Review the configuration in V_T511 to ensure the wagetype settings are consistent.
  2. If user exits are enabled in SMOD (CATS*), verify their impact on time entry processing.
  3. Implement the required logic within the CATS0002 user exit to handle multiple unit settings for the wagetype.

See Also

  • Refer to: SAP Note 1863089 - Specific units can not be saved by running transaction CAT2.

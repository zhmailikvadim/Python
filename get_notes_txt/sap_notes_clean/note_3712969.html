Symptom

  • Error "Unknown store number" error occurs in POS Workbench.
  • /POSDW/CORE003 occurs in /n/POSDW/MON0.

Environment

SAP Customer Activity Repository - All Versions

Reproducing the Issue

  1. Execute the transaction code /n/POSDW/MON0.
  2. Open up Sales Movement data for any store/data combination
  3. Observe that the error "Unknown store number" is displayed.

Cause

HANA views are not able to retrieve the required data.

Resolution

To resolve this issue, please ensure that the following has been done:

  1. POS Workbench uses the following HANA views to retrieve master data from the ECC or S/4HANA source system:
    ECC:
    sap.is.retail.ecc/PLANT
    sap.is.retail.ecc/SalesOrganization

    S/4HANA:
    sap.is.retail.s4h/PLANT
    sap.is.retail.s4h/SalesOrganization

    Test if the above HANA views return the correct data via tcode DB02 with the following queries:
    ECC:
    select * from "_SYS_BIC"."sap.is.retail.ecc/PLANT"
    select * from "_SYS_BIC"."sap.is.retail.ecc/SalesOrganization"

    S/4HANA:
    select * from "_SYS_BIC"."sap.is.retail.s4h/PLANT"
    select * from "_SYS_BIC"."sap.is.retail.s4h/SalesOrganization"

    *Make note that the client ID of the data in these views will need to match the working client ID in CAR.*
  2. If an error occurs when querying the HANA views, ensure that the HANA views have been activated correctly via the /CAR/ACIVATE_HTA report.
    https://help.sap.com/docs/CARAB/5f6a0c50ac324b0395738c1ce0fe14c6/385653490cfb4829a0fd1879cd89edab.html?locale=en-US&version=LATEST

    To check the status of the HANA views, go to tcode SCTS_HTA and enter the package according to the source system that is being used:
    ECC:
    sap.is.retail.ecc

    S/4HANA:
    sap.is.retail.s4h

    The packages should all be green/active.  If any upgrade or DB refresh recently, re-activation of these HANA views via the /CAR/ACTIVATE_HTA report is necessary.

  3. If the HANA views query from step #1 do not return any data, check if the corresponding tables have the required data:

    sap.is.retail.XXX/PLANT - Uses the T001W table from the replicated SAP_ECC or SAP_S4H (or schema mapped to) schema.

    sap.is.retail.XXX/SalesOrganization - Uses the TVKO, TVKOT, and T001 tables from the replicated SAP_ECC or SAP_S4H (or schema mapped to) schema.

    You can query these tables via tcode DBO2:

    select * from "SAP_S4H"."T001W"

    *Replace 'SAP_S4H' with the physical schema name that you are using*
  4. If the above steps do not help, please see the following KBA 2780025 - CAR: Unable to access master data from S/4HANA source system

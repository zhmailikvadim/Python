Symptom

After re-synchronize the example employee, the travel bank could not be refreshed of the synchronized employee BP, even the SLG1 status is successful.

Environment

S/4 HANA OP with HR integration active

Reproducing the Issue

  1. Configuration Main bank and travel bank in table PTRV_VENDOR_SYNC under relevant company code via SM30.
  2. Creation the same bank information in infotype 0009 subtype 0 and 2 of the example employee at the same time via example transaction PA30 or other action steps
  3. Synchronization the employee with SLG1 successful status.
  4. Go to transaction BP, only main bank as ID HC01 can be found in tab payment transactions of the specific BP and the travel bank as ID HC02 could not be found.

Cause

That is standard design. The fields in the maintenance view also provide a F1 help. 
If you choose to transfer both subtypes, and both subtypes exist with an equal bank country/region, bank number, and bank account number (LFBK keys), only subtype 0 is transferred.

Resolution

If customer need to get travel bank in BP module as ID HC02, please consider to create the travel bank master data, which is different from main bank in source HR infotype 0009 of the example employee in source HR module.

Symptom

  • The QR code is not displayed correctly in custom PDF outputs using the EDOPLINV-QR_CODE_ACCEPTED table data.
  • The EDOPLINV table has new fields added related to QR codes, but the correct QR code is not generating the expected URL after acceptance.
  • The QR code is being generated as a groovy link instead of the expected URL.

Environment

  • SAP ERP
  • SAP S/4HANA
  • SAP Document and Reporting Compliance Service

Resolution

  • Implement SAP Note 3713084 - Electronic Document Processing for Poland - QR Code (Accepted) Fix.

Note: Before implementing above note, ensure all the pre-requisites notes in sequence order as mentioned in the overview note 3279206 - Electronic Document Processing for Poland - KSeF: Implementation Overview - SAP for Me are implemented first.

  • To fetch the QR code information in the custom Invoice PDF form: The field QR_CODE_ACCEPTED in table EDOPLINV stores the QR Code image as an XSTRING, which contains the Base64-encoded PNG image content.

To retrieve the QR Code PNG image (XSTRING), the following logic can be implemented in the custom development. 

1. Convert the XSTRING (Base64 content) to Base64 STRING
DATA lv_image_base64 TYPE string.
DATA lo_xstr_to_str_conv TYPE REF TO cl_abap_conv_in_ce.

cl_abap_conv_in_ce=>create(
  EXPORTING
    encoding = 'UTF-8'
    input    = ls_edoplinv-qr_code_accepted
  RECEIVING
    conv     = lo_xstr_to_str_conv
).

lo_xstr_to_str_conv->read(
  IMPORTING
    data = lv_image_base64
).

2. Convert the Base64 STRING to XSTRING (PNG image content)
DATA: lv_image_xstring TYPE xstring.

lv_image_xstring = cl_http_utility=>decode_x_base64(
  encoded = lv_image_base64
).


The variable lv_image_xstring will now contain the actual PNG image content of the QR code, which can be used directly in the custom form (Adobe Form/SmartForm) for rendering.

Note: The above code is a sample example that can be used to convert data into QR Code PNG (XSTRING) format.

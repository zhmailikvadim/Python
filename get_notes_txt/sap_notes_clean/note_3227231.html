Symptom

In the ABAP system log (SM21), the system writes the following error:

  • Database error 258 at OPC > insufficient privilege: Not authorized

Or, in German:

  • Database error 258 at OPC > insufficient privilege: Not authorized

The work process trace contains an SQL statement that tries to read from the SAP HANA object "_SYS_REPO"."ACTIVE_OBJECT".

This error message occurs only for systems that use the SAP HANA database and in which the software component SAP_BW is in use. This concerns the creation of metadata for operational data providers (ODPs). This creation of metadata can be started via different actions; for example, when you start the Data Warehousing Workbench (RSA1) or when you model an Open ODS view.

Other Terms

Open ODS view, ODP, operational data provider

Solution

The system issues the error message due to a missing authorization of the SAP<SID> database users in the SAP HANA database. When the metadata is set up, the time stamp is read. This only involves a performance optimization, so if an error occurs, the time stamps are read from the SAP HANA repository, which takes a little big longer. If you want to use this performance optimization, you should grant read authorization for the following object to the SAP<SID> database user: "_SYS_REPO"."ACTIVE_OBJECT".

With SAP Note 2111563, this behavior was improved so that fewer accesses to this object are required.

Symptom

  • During the onboarding of SAP BusinessObjects Data Services into SAP Focused RUN, the Outside Discovery fails with a 500 Internal Server Error.
  • Error message in SLG1:
    •  "Fatal error - package discarded. Root element SAP_InstalledSoftwareComponent.Name='DATASERV', SoftwareElementID='6692620f-18c0-61f5-517c-c9b73bc2d40c', SoftwareElementState=3, TargetOperatingSystem=0, Version='2025' does not exist in the."

Environment

  • SAP Focused RUN

  • SAP Data Services 2025 and its patches:

    • Patch 1 (version 263)

    • Patch 2 (version 339)

Reproducing the Issue

  1. Access the affected host.
  2. Execute the Outside Discovery (OD) command.
  3. Check the error message in the log.

Cause

The "sld_data_services.xml" file contained missing or incorrect data for the DATASERV components.

Resolution

This issue will be fixed in the Patch 3 of the SAP DataServices 2025. 

The workaround below can be applied in this meanwhile:

  1. Upgrade SAP Data Services 2025 to the latest patch level available
  2. Locate and edit the "sld_data_services.xml" file:
    1. Search for the entries the DATASERV components
    2. Change the value of <version> property from "4.4" to "2025".
    3. Save the XML
  3. Restart the SAP Data Services system to ensure the changes take effect.
  4. Re-execute the Outside Discovery command to push the updated XML file to the SAP Focused RUN system
  5. Check that the process is executed with success

1. Introducción

Este KBA está orientado a consultores, usuarios de nómina y equipos de soporte para la correcta configuración, cálculo y mantenimiento de los descuentos de Infonavit en SAP HCM, considerando los cambios legales y técnicos recientes, así como mejores prácticas y errores comunes.

2. Tipos de Descuentos Infonavit y Códigos SAP

Tipo de DescuentoCódigo SAP (Ejemplo)Descripción
Descuento Fijo381Descuento mensual fijo de Infonavit
Descuento por Porcentaje381Descuento calculado sobre salario base
Descuento por UMA381Descuento según veces UMA asignadas
Arrears (Atrasos)389, 38A, 38BAjustes por deducciones no tomadas
Seguro de Vivienda387Cuota bimestral para seguro de vivienda

Diferencias:

  • Fijo: Monto específico descontado cada periodo.
  • Porcentaje: Calculado sobre el salario base integrado.
  • UMA: Multiplicador de la Unidad de Medida y Actualización, actualizado anualmente.
  • Arrears: Si no se puede descontar el total, el saldo pendiente se acumula para el siguiente periodo.
  • Seguro de Vivienda: Cuota bimestral, puede requerir configuración adicional para integración con Arrears.

3. Configuración de Descuentos Infonavit

Tablas relevantes

TablaFinalidad
T512WCatálogo de conceptos de nómina (wage types)
T51P6Parámetros de deducciones (Deduction Not Taken)
T54C1Periodicidad de acumulación para reset de Arrears
T512ZHistorial de copia de conceptos

Reglas y Esquemas

  • MXYN: Esquema principal para el cálculo de Infonavit.
  • XPDD, X024, X025: Reglas para transferir, acumular y validar conceptos.
  • PRDNT, DNTK: Reglas para deducciones no tomadas (Deduction Not Taken).
  • P0370: Payroll function para Infonavit.

Clases de Procesamiento (Processing Class)

  • PC 4: Define si el concepto se acumula en bases.
  • PC 6: Permite que el concepto pase sin procesamiento adicional (valor “6”).
  • PC 41: Determina el tratamiento en la transferencia de IT a RT.

Feature 32INF

Permite definir el comportamiento del sistema ante valores no descontados (arrears), con opciones:

  • 0: Intentar cobrar en el siguiente periodo.
  • 1: Ignorar valores pendientes.
  • T: Consultar la tabla T54C1 para reset periódico.

5. Cálculo del Descuento Infonavit

  • El sistema identifica el tipo de descuento (fijo, porcentaje, UMA).
  • Calcula el monto a descontar según la configuración y días base.
  • Si el salario no es suficiente, el saldo pendiente se acumula en Arrears (/389).
  • En el siguiente periodo, se intenta descontar el saldo acumulado, según la opción configurada en la feature 32INF y la tabla T54C1.

Días Base:

  • Se consideran días calendario menos ausencias/incapacidades según la ley y configuración.
  • Para cuotas patronales, solo se descuentan ausencias.
  • Para créditos, se descuentan ausencias e incapacidades.

6. Particularidades y Casos Especiales

Arrears (Atrasos)

  • Si no es posible descontar el total, el saldo se transfiere a /389.
  • El tratamiento de Arrears depende de la configuración de la feature 32INF y la tabla T54C1.

Cambios Legales y Opciones de Configuración

  • La ley prohíbe deducir ausencias/incapacidades para el cálculo del Infonavit.
  • Opciones de configuración:
    • Opción 1: No limitar el descuento (no recomendado).
    • Opción 2: Limitar al neto y descartar lo no pagado.
    • Opción 3: Limitar al neto y trasladar lo no pagado a periodos siguientes.
    • Opción 4: Igual a la 3, pero con reset periódico de Arrears.

Seguro de Vivienda

  • Concepto 387, cuota bimestral.
  • No está incluido por defecto en la solución de Arrears/reset, pero puede ser configurado por el cliente.

Reporte y Ajustes Manuales

  • El reporte al gobierno (SUA) puede requerir ajustes manuales si no se logra descontar el total.
  • Empresas grandes pueden tener dificultad para realizar ajustes manuales masivos.

Ejemplo 2 – Reset de Arrears Bimestral

  • En enero y febrero, el empleado no puede pagar todo el descuento.
  • Al cierre del bimestre, el saldo pendiente se resetea y no se intenta cobrar en marzo.

Ejemplo 3 – Error de Configuración

  • Un concepto personalizado no tiene configurada la Processing Class 4.
  • Al ejecutar la nómina, aparece el error X025.
  • Solución: Configurar la Processing Class 4 con valor “6” para permitir el paso.

8. Puntos de Atención

Customizaciones y Conceptos Modelo

  • No utilizar conceptos modelo (Mxxx) en productivo.
  • SAP puede modificarlos o eliminarlos sin previo aviso.

Tablas y Reglas

  • Revisar siempre la configuración en T512W, T51P6 y T54C1.
  • Verificar que los conceptos personalizados estén correctamente parametrizados en las Processing Classes.

Debug y Errores Comunes

Historial de Copia

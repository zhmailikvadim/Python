Symptom

The system settings were changed to another timezone, such as "China Shanghai", but the log time of the actual costing run displays the time in a different location (not the Chinese time).

Environment

SAP S/4HANA Cloud Public Edition

Reproducing the Issue

  1. Change the time zone settings to "China Shanghai".
  2. Run the actual costing run
  3. Open the run logs and observe that the timestamps display a different location’s time rather than the expected time.

Cause

The log time is consistent with the system time zone and does not adjust based on the user’s time zone settings; this is by design.

Resolution

This is standard behavior: the actual costing run log displays timestamps in the system time zone, not the user’s time zone.

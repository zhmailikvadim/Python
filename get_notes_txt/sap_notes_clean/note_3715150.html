Symptom

You are using report "Personal Calendar Generation" (RPTGENB0) and you add manually public holidays to infotype 2001 (Absences) with absence type PUB (table T5BP7) or with absence type VAC (table T5BP7) to the personal calendar processing.

You get following issues in the personal calendar processing:

  • The sequence counting after the collision between the public holiday and the illness period leads to a reset of the illness period counter.
  • The sequence counting after the collision between the vacation (as public holiday) and a maternity period leads to a reset of the maternity leave period counter.

This SAP Note solves the issues above.

Other Terms

RPTGENB0, PUB, VAC, MAT

Reason and Prerequisites

This SAP Note is relevant only for Belgium.

Solution

  • As a general rule SAP recommends that you implement the Support Package specified for your release.

  • However, if you need to install a solution earlier, use the SAP Note Assistant to implement the correction instruction.

  • You can find more information about the SAP Note Assistant in SAP Support Portal, under SAP Note Assistant.

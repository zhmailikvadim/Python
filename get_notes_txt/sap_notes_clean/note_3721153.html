Symptom

In PAT Data Collation, there is an attempt to post a STANDARD document with two CSN (Customer Sales: Without Documents) item categories. During the posting process, the first item is fully posted but the second one fails to post due to some errors. There has not even been any SD documents created for the second item. Then, the document is deactivated and when there is an attempt to reverse it, a shortdump with exception “/PAT/CX_DC_WRONG_PARAM” occurs.

In case the second item uses a third-party plant, the reversal attempt triggers the “No material document found for item &1" exception.

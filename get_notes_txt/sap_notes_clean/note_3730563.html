Symptom

  • SES posting with more items through manage service entry sheet take longer time than usual.
  • Running a service entry sheet causes memory consumption up to 2TB.
  • The PRCD_ELEMENTS table is read even though no pricing information is present.

Environment

  • Materials Management (MM)
  • SAP S/4HANA

Reproducing the Issue

  1. Open the Fiori app for managing or viewing a service entry sheet.
  2. Start the service entry sheet.
  3. After entering the inputs, app hangs or running slow.

Cause

Note missing.

Resolution

Make sure that the below notes are implemented in the system.

3513662 - Performance issue when you view 'Purchase Order History' 

3517013 - F2027: Performance issue when you delete Service entry sheet

3531269 - Performance improvement for Lean Service documents

3553573 - Performance improvement while posting invoices for Lean SES.


Implementing these notes will improve the app performance, in case if the performance is not improved as expected, then take a trace in ST12 and create an ticket with component MM-PUR-SVC-SES and share the same to check if there is any other possible potential improvement areas.

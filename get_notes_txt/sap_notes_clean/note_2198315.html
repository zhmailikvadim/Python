Symptom

This SAP Note describes two independent errors when writing the PXA work process trace:

1.) The work process trace contains the following lines: "A PXA: mprotect (unprotect) failed with error: 12 (address: (nil); length: 1000)"

2.) During the initialization of the PXA cache, there is a core dump, and the work process trace contains the following call stack (example for Linux):

             ....

<signal handler called> [0x7f6c5459d850]
(initPxaCache(unsigned char)+0x137)[0x1cd9257]
(ab_pbinit()+0xbfd)[0x1ce0ead]
(ab_init+0x177)[0x77efc9]

            ....

            There are no further entries for the cause of the core dump.

Other Terms

PXA guard page, PXA memory protection, PXA initialization, PXA cache initialization

Reason and Prerequisites

Regarding 1.): When you remove the protection of the PXA guard page, the system does not check correctly whether the memory protection is active.

Regarding 2.): If the requested memory cannot be provided during the initialization of the PXA, the aforementioned core occurs.

Solution

Implement the specified kernel correction. The patch comment is as follows:

"Improvements to the PXA work process trace".

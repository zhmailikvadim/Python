Symptom

In the case of an MS SQL sender system, when hash parallelization for a table is activated in accordance with the information provided in SAP Note 3655372 - Hash-parallelization of delta replay:

  • the trigger code outputs calculated hash values in the range of 0-255
  • the re-replication outputs calculated hash values in the range of 0-255

This range is too small and would lead to problems during the parallelized transfer because:

  • Calculated hash values in the re-replication have a range of 0-16383
  • Access plans for hash-parallelized tables have a range of 0-16383

Other Terms

downtime minimization tool for SAP S/4HANA, NZDT

Reason and Prerequisites

Any of the following add-on support packages is installed in your system landscape:

  • NZDT 1.0 SP03

Solution

  • The trigger code now calculates hash values in the range of 0-16383
  • The re-replication now calculates hash values in the range of 0-16383

Warning

Please do not implement this note during an NZDT cycle where recording has already been started, triggers are active, and the clone was created. It will lead to a mismatch between trigger hashes and re-replication hashes, which will consequently lead to data inconsistencies.

Required Action

In order for this note correction to be effective, the note must be implemented before:

  • Starting a new cycle

Implement the correction instructions in transaction SNOTE on the following systems of your system landscape:

  • Sender system

Symptom

In the Create/Change Maintenance Order transactions (IW31/IW32) on the Malfunction data tab of the header notification, you change the malfunction start/end Date/time and the breakdown duration is not updated. Instead you get the message  IM 476 'Please check length of downtime'.

This is incorrect behavior. Only in case if the Break down duration is changed by the user manually, it shouldn't update the break-down duration based on Malfunction Date /time changes and inform the user by message IM 476.

Other Terms

EAM, PM/CS, Plant Maintenance, IW31, IW32, Maintenance Work Order, RIWO00-EAUSZT, VIQMEL-MSAUS, SAPLIQS0, 7020, ON REQUEST.
IM476.

Reason and Prerequisites

Programm error in the following Notes:

3285783 - Breakdown duration remains when unchecking the checkbox

3290318 - Breakdown duration remains when unchecking the checkbox

The same issue is already fixed for Notifications directly:

3413675 - IW22: Break down duration is not updated correctly when the Malfunction start/End Date/Time is changed.

Solution

Implement this SAP Note or the import of the relevant support package.

Symptom

Description

This KBA provides guidance on the setup, validation, and common troubleshooting steps required to use My Forms (Fiori 2.0) with Employee Central Payroll (ECP).

To ensure My Forms functions correctly, the system must have:

  • the required SAP Fiori and frontend and backend components

  • a correctly configured SAP Fiori Launchpad

  • the My Forms application, including target mapping, roles, and OData services

  • correct Payroll Unified Configuration in Employee Central

This KBA covers commonly required configuration steps and currently known troubleshooting checks.

Environment

  • Fiori 2.0
  • SAP SuccessFactors Employee Central Payroll

Resolution

1. Backend Software Components (Mandatory Prerequisite)

Ensure the required backend software components are installed at a supported level. It is recommended to always use the latest available support package level.

Required components:

    • UIHR002 - Frontend component for HCM Fiori applications

    • GBX01HR5 - Backend payroll component for HCM Fiori applications

The required and latest support package levels can be verified using the relevant KBA 3283299

2. Required Role Assignments (Mandatory)

The following standard roles must be assigned to the employee user:

    • SAP_HCMFAB_BCR_EMPLOYEE_T
      Provides access to Employee Fiori applications, including My Forms.

    • SAP_HCMFAB_MYFORMS_APP
      Provides the required authorizations for My Forms OData services and HR data.

More information on the My Forms application and its configuration can be found in the official Fiori library documentation.

3. Fiori Launchpad Target Mapping (Mandatory)

Verify that the My Forms application is correctly maintained in the SAP Fiori Launchpad Designer .

Required target mapping values:

    • Semantic Object: Employee

    • Action: displayForms

    • Application Type: SAPUI5 Fiori App

    • Application ID: hcm.fab.myforms

    • Standard Catalog: SAP_HCMFAB_BC_EMPLOYEE_T

    • Standard Role: SAP_HCMFAB_BCR_EMPLOYEE_T

If Z catalogs are used, the target mapping must be copied manually from the standard catalog.

If the Fiori Launchpad Designer cannot be accessed, refer to:

    • SAP Note 3011106  Fiori Launchpad Designer returns 403 Forbidden

    • SAP Note 2729557  Standard SAP catalogs and groups missing in Fiori Launchpad Designer

5. Payroll Unified Configuration - Portlet Setup (Mandatory)

The Payroll Information Portlet configuration in Employee Central is mandatory.
My Forms will not appear in Employee Central unless this configuration exists and is active.

Required configuration:

    • Payroll Unified Configuration maintained per country/region

    • Payroll System Configuration set to Active

    • A Payroll Information Portlet exists

    • A URL link to My Forms is configured

Portlet link configuration:

    • Link Type: URL

    • Permission Category: Self Service

    • URL: 
      https://<ECP_HOST>/sap/bc/ui5_ui5/ui2/ushell/shells/abap/FioriLaunchpad.html?sap-ushell-config=headerless#Employee-displayForms

 

6. Required OData Services (Mandatory)

The following OData services must be active and correctly assigned in /IWFND/MAINT_SERVICE:

    • HCMFAB_MYFORMS_SRV - Retrieves forms and PDFs

    • HCMFAB_COMMON_SRV - Required during application startup

Both services must be assigned to the correct system alias.

7. Required Fiori Launchpad Infrastructure Services

Ensure the following SAP Fiori Launchpad infrastructure services are active in tx: SICF:

  • /sap/bc/ui2/start_up
  • /sap/bc/ui2/page_builder_conf
  • /sap/bc/ui2/page_builder_pers
  • /sap/bc/ui2/interop
  • /sap/bc/lrep

LREP must be active for Fiori applications to load correctly.

8. Additional Required Data Services - For Fiori Launchpad to Work

The following services are required for Fiori Launchpad to work. Make sure they are activate in tx: SICF and /n/iwfnd/maint_services and are assigned to the local system alias.

    • /UI2/PAGE_BUILDER_CONF
    • /UI2/PAGE_BUILDER_CUST
    • /UI2/PAGE_BUILDER_PERS/
    • /UI2/TRANSPORT
    • /UI2/INTEROP
    • /UI2/LAUNCHPAD

9. Authorization Validation (Troubleshooting)

If My Forms does not load or appears blank:

Perform an authorization trace using ST01 (not SU53)

Reproduce the My Forms launch while tracing

Check for missing authorizations related to:

    • /UI2/* services
    • HCMFAB_COMMON_SRV
    • HCMFAB_MYFORMS_SRV

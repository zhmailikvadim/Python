Symptom

When attempting to add members to a Target Group, the members are not added despite being selected from the list.

Environment

  • SAP Sales Cloud Version 2
  • SAP Sales Cloud and SAP Service Cloud Version 2

Reproducing the Issue

  1. Log in as user A assigned to Business Role B.
  2. Navigate to Target Group work center.
  3. Open Target Group C.
  4. Attempt to add a member, for example, Account.
  5. In the Account object value selector, select some Accounts and attempt to add them to the Target Group.
  6. Observe that the selected Accounts are not added to the Target Group.

Cause

  • The user is assigned Business Role B.
  • Business Role B includes sap.crm.service.targetGroupService, but does not include sap.crm.service.targetGroupMemberService.
  • The absence of sap.crm.service.targetGroupMemberService in the user's role prevents the addition of members to target groups.

Resolution

  1. Add the sap.crm.service.targetGroupMemberService to the user's role.
  2. Verify that the user can now successfully add members to the target group.

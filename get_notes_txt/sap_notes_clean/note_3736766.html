Symptom

  • After adding a new approval task to a registration template and upgrading existing supplier registration projects to the new template version, the new task shows as Not Started.
  • The process flow/workflow displays as incomplete (dotted lines).

Reproducing the Issue

  1. Create a new registration template version and add a new task to the Update phase.
  2. Perform a template upgrade for existing supplier registration projects to the new template version.
  3. Open an affected supplier registration project that was already in progress or completed.
  4. Observe that the newly added task is in Not Started status and the process flow appears incomplete (dotted lines), while the registration status remains Registered.

Cause

This is currently an expected system behavior that cannot be altered, when a new task is added in a new template version, existing supplier registration projects upgraded to that version keep the task in Not Started until the supplier is reinvited to the relevant phase, and the workflow may appear incomplete even though the registration status remains Registered.

Resolution

  1. Reinvite the supplier to the relevant Update phase of the registration project.
  2. After re invitation, confirm the newly added task is initiated and proceed to complete it.
  3. Complete the task as required to update the workflow visualization.
  4. Note: If no action is taken, the Not Started task and dotted workflow visualization do not affect the Registered status of the supplier registration.

Symptom

Unable to use SAP Default (sap.ids) despite setting the "Available for User Logon" flag being set

Environment

  • SAP Cloud ALM 1.0
  • SAP BTP

Reproducing the Issue

When the "Available for User Logon" flag is set in the Trust Configuration in the SAP Cloud ALM SAP BTP Subaccount it is not offered as a choice between sap default and the customer's IAS Tenant

When the "Available for User Logon" flag is turned off on the custom IAS, it is still used for authentication and sap default is not used

Cause

At End of September 2025 SAP Cloud ALM tenants provisioned since then have been changed from an SaasRegistry/XSUAA logon token based approach to an SMS/IAS logon token based approach. This means, that for tenants that have been provisioned after that, the IAS is indeed mandatory and also (for the time being) cannot be changed.

Also if the customer has completed the SCI migration, after that the default IDP is no longer available.

Resolution

SAP Cloud ALM tenants provisioned since the end of September 2025 will only use the custom IAS tenant.
The IDP sap.ids (sap default) is not intended for customer use and should not be used for authentication of SAP Cloud ALM users.

Keywords

sap ids, sap default, IAS, user Authentication

Symptom

During result recording or inspection lot creation via Manage Work Center Queue (F2821), catalog values maintained in routing (PLMK) do not appear in inspection lot characteristics (QAMV). Fields for additional catalogs (QAMV-KATAB2/AUSWMENGE2 through QAMV-KATAB5/AUSWMENGE5) remain empty although corresponding PLMK entries exist. The symptom occurs for new inspection lots after a feature pack update.

Other Terms

F2821, PLMK, QAMV, R_INSPECTIONLOTTP, AUSWMENGE2, KATAB2, RAP BO

Reason and Prerequisites

The reason is a program error. The RAP behavior definition R_INSPECTIONLOTTP does not map additional selected sets and code groups from PLMK to QAMV fields (KATAB2/AUSWMENGE2 through KATAB5/AUSWMENGE5), which causes missing values during inspection lot processing.

Solution

Install this SAP Note. The correction adds the required mappings so new inspection lots copy additional catalog data from PLMK to the corresponding QAMV fields.

Symptom

 Get error 24 too many open files

Environment

 Linux all versions.

Reproducing the Issue

Normal operation

Cause

  •  The operating system limit for open files has been reached.
  • To see the settings for maximum open files for the OS level, use the following command:
  • # cat /proc/sys/fs/file-max
  • To see the setting for maximum open files for a user, as root use the following commands:
    • # su - <user>
    • $ ulimit -n

Resolution

Use ps -ef command to list the running processes and note the process id (pid)

  • Run cat proc/limits <pid>
  • this will show the ulimits that this process has applied
  • nofile Open file descriptors per user 
  • fs.file‑max Open file descriptors per host /etc/sysctl.conf

System Settings

  • We don't have a recommended number for these parameters.
  • However ok to increase fs.file‑max by say 25% if you have the problem with too many open files.
  • Change the value in.... /etc/sysctl.conf and activate them with the command "sysctl -p".
  • Don't need a reboot.
  • Check new value with
  • cat /etc/sysctl.conf | grep fs.file‑max

User settings

To see the setting for maximum open files for a user, as root issue the following commands:

  • # su - <user>
  • $ ulimit -n

Recommended values by SAP in /etc/security/limits.conf or in a file in the /etc/security/limits.d directory:

  • @sapsys - nofile 1048576
  • @dba    - nofile 1048576
  • @sdba   - nofile 1048576
  • Modifications to the limit files require a logout for the setting to take effect for newly started programs.
  • After re-login you may run the following command to check your current resource limits:
  • bash -c "ulimit -a"

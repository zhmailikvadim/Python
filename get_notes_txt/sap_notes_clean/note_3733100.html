Symptom

You want to reissue a check in the Fiori application F1578 Manage Outgoing Checks.
The supplier of the check has an alternative payee set up. When confirming the reissue, an exception is raised and the process is terminated.
This issue occurs due to the use of CALL SCREEN in the back-end.

Other Terms

F1578, check, reissue, alternative payee, dump, call screen, exception, payees_show_all, call screen

Reason and Prerequisites

Reissue with Alternative Payee was not supported in the F1578 Manage Outgoing Checks.
The CALL SCREEN statement is triggered in the back-end due to the reuse of the FCH7 transaction.

Solution

Implement the attached Correction Instruction or the mentioned Support Package.
Afterwards reindex the application on your frontend (gateway) system as described in SAP Note no. 2227577 and clean the caches according to SAP Note no. 2017117.

IMPORTANT: This is front-end part of the correction, that requires back-end part. If you implement this SAP Note, make sure that SAP Note no. 3733011 is also implemented.

This SAP Note introduces a new combo box for payee selection during the check reissue process in the Fiori application F1578 Manage Outgoing Checks.
Selection is required if an alternative payee is defined for the supplier of the check.

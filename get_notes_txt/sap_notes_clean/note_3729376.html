Symptom

While using the TCode /PE1/AR_ORD2_DISPLAY to display archived Payment Orders, the following behaviour takes place after we open any desired archived payment order:

  • After viewing the Clearing Area SLA from the archived Payment Order's References tab, when we press the back button, we get an error log window that states, payment order could not be read.
  • On closing this error log window, when the PO_EXPERT screen is displayed, there are no Payment Order details displayed on the screen.
  • Further, when we try to get out of this screen, there is a short dump that occurs.

We can also observe the following behaviour across the Payment Items of the Archived Payment Order as well:

  • After viewing the Clearing Area SLA from the Payment item's References tab or after viewing the Route details from the Clearing Information tab, when we press the back button, we get an error log window that states, payment order could not be read.
  • On closing this error log window, when the PO_EXPERT screen is displayed, there are no Payment Order details displayed on the screen.
  • Further, when we try to get out of this screen, there is a short dump that occurs.

Other Terms

Archived Orders, Clearing Area SLA, Route, TCode /PE1/AR_ORD2_DISPLAY, Short dump, DYNPRO_MSG_IN_HELP, Screen: Illegal message.

Reason and Prerequisites

Program error. The correct data source is not used to retrieve the archived Payment Order details when we navigate from the PO_EXPERT Screen to the Clearing SLA Screen or Route Screen and then back. It tries to read the archived Payment Order from the database instead of reading it from archive. As this read will fail, it causes all the above mentioned issues.

Solution

Implement the correction instruction attached to this SAP Note. Alternatively, you can import the referenced Support Package.

Symptom

When DAS appointment is being created from Yard Order in subsequent call of qRFC, it could be processed in later point of time. For some customer's scenarios synchronous DAS creation could be critical.

Other Terms

/SAPYL/CL_YL_DAS_HELPER_BASE=>CALL_DAS_APP_RFC

/SAPYL/CL_YL_DAS_HELPER_BASE=>CREATE_UPDATE_DS_APP

/SAPYL/CL_YL_DAS_HELPER_BASE=>MODIFY_DSAPP_FROM_YR_YO

/SAPYL/CL_YL_DAS_HELPER_BASE=>MODIFY_DSAPP_FROM_YT

/SAPYL/V_DS_RELO

Solution

Implement the the attached note. There is a new feature to run DAS creation within the FM /SAPYL/YO_EXT_COMMUNICATION. It's relevant for internal DAS integration only in the same client. This option to be switched on in the IMG customizing:

Yard Logistics -> Integration -> Integration to Dock Appointment Scheduling or Logistics Business Network -> Maintain Loading Appointment Relevance for Yard Order Type.

By default this approach is switched-off.

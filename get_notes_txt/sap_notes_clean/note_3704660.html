Symptom

There is a requirement to fill the section code once a transaction was settled. 

Environment

Treasury and Risk Management (TRM)

Cause

Missing functionality

Resolution

  • There are two BAdIs for the TRAC-area (transaction code SE18)
    • BAdI TPM_POS_IND_TRAC
    • BAdI TPM_ACCIF_TRAC
  • BAdI TPM_POS_IND_TRAC has the method CHANGE. This method allows to derive the following fields for the position indicator:
    • TPM_AA_REF  Account Assignment Reference: Parallel Valuation Areas
    • BUPLA           Place of Business
    • SECCO           Section Code
    • KOSTL           Cost Center
    • FKBER            Functional Area
  • The position indicator is derived at the creation-event of the transaction. When the transaction is changed or settled the position indicator already exists, hence the BAdI method is not called again.
  • Another possibility is to use BAdI TPM_ACCIF_TRAC with the method ACCIT_EXIT to change the complete posting document.

Keywords

Section code, TPM_POS_IND_TRAC, TPM_ACCIF_TRAC

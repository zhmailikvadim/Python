Symptom

In the standard process, the system creates and navigates to SD billing documents. However, a custom implicit enhancement has been implemented to create and navigate to an SD sales order instead of billing documents. There is now a requirement to have standard BAdI methods to support both the creation and navigation of SD sales orders in backend and Fiori applications. This would help replace the custom enhancement with a more robust and standard approach.

This note provides guidance by suggesting an appropriate approach to enable sales order creation and navigation through supported enhancement frameworks.

Other Terms

sales order navigation, sales order creation, BAdI methods, SD sales order, billing document replacement, implicit enhancement, Fiori integration

Reason and Prerequisites

This is caused by program error

The external posting document navigation from the calculation run will not work , it should be achieved through Badi EP_DOCUMENT_NAVIGATE implementation .

Solution

Please implement the correction instructions attached. These corrections are only valid if you have applied the Support Packs.

Release V/4 - 2023 V2, 2023 V3 and 2023 V3 Support Pack 1

2. Use transaction SNOTE to implement the correction instructions specified below and activate the affected program object.

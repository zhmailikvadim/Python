Symptom

  • Issues during integration between SAP SuccessFactors and SAP Joule via SAP BTP.
  • The destination LPS_SFSF_dt for "design time" is correctly configured with Basic Authentication and working as expected.
  • When migrating the destination from Basic Authentication to Client Certificate Authentication (mTLS), the following error occurs during the refresh of the content channel in SAP Build Work Zone: "<Connectivity/InternalError>: Could not fetch content due to an internal error. Please try again later. If the issue persists, please open a ticket on the connectivity service."

Environment

  • SAP SuccessFactors HCM Suite
  • SAP Joule

Reproducing the Issue

  1. Configure a design-time destination in SAP BTP using ClientCertificateAuthentication.
  2. Create a new channel in SAP Build Work Zone content channel using the configured design-time destination.
  3. Attempt to refresh the content channel.
  4. Observe the error: "<Connectivity/InternalError>: Could not fetch content due to an internal error. Please try again later. If the issue persists, please open a ticket on the connectivity service."

Cause

Certificate Authentication is not currently supported for Joule.

Resolution

At the present time, only basic authentication is supported for SAP Joule authentication.

We understand that SAP SuccessFactor is migrating to Certificate Authentication, and that basic authentication for SF will be deprecate as per Deprecation of Basic Authentication and Third-Party Corporate Identity Provider (IdP) Direct Integration with SAP SuccessFactors | SAP Help Portal.
Due to that, our PMs are working with the engineering team to provide the certificate authentication for Joule. However, this is not yet available and the release date for it is still under discussion.

You can monitor future updates and announcements regarding the authentication functionality at Consume Content from SAP SuccessFactors | SAP Help Portal

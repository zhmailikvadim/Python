Symptom

  • The issue involves a business rule designed to trigger a warning message when an employee edits an existing record.
  • The rule context uses "Data operation is equal to correct" but fails to trigger the warning message.
  • Rule trace shows 'RuleContext.dataOperation' with a value of 'null'.
  • This issue prevents the warning message from being displayed.

Environment

SAP SuccessFactors HCM Suite

Reproducing the Issue

  1. Login into SuccessFactors.
  2. Navigate to the employee profile and select an impacted object with an existing record.
  3. Click on the edit icon from history.
  4. Navigate to the rule execution log and download the rule trace.
  5. Observe that the rule trace displays 'RuleContext.dataOperation' as null.
  6. Due to this, the rule does not trigger the warning message configured in the business rules.

Cause

In the case of the "onChange" scenario, the system is unable to determine the operation type, resulting in 'RuleContext.dataOperation' being null.

Resolution

  • Engineering confirmed that in the "onChange" scenario, the dataOperation function does not work, and this is expected behavior from the system.
  • For other use cases, it is expected to work

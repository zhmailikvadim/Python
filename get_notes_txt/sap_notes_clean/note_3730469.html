Symptom

  • When setting the delivery complete indicator and updating the Latest GR Date in a purchase order item to close it as a partial order, the system raises a warning message “Latest acceptable GR date exceeded by a schedule line” (Message no. 06807).

Environment

SAP S/4HANA Cloud Public Edition

Reproducing the Issue

  1. Open a purchase order item that has one or more schedule lines with delivery dates.
  2. Set the “Deliv. compl.” (Delivery Completed) checkbox for the item.
  3. Enter or update the “Latest GR Date” to a date earlier than at least one schedule line delivery date.
  4. Save the purchase order.
  5. Observe the warning message “Latest acceptable GR date exceeded by a schedule line” (Message no. 06807).

Cause

  • Standard system behavior: the system validates the “Latest GR Date” (EKPO-LEWED) against the item’s schedule line delivery dates (e.g., EINDT) and raises a hard-coded warning (Message Class 06, Number 807) if the Latest GR Date is earlier than any schedule line date.

Resolution

  1. Review the item’s schedule line delivery dates and identify the latest delivery date.
  2. If you want to avoid the warning, set the “Latest GR Date” to a date on or after the latest schedule line delivery date, or adjust the schedule line dates to align with the intended Latest GR Date.
  3. Save the purchase order and proceed with goods receipt posting.
  4. Note: In the public cloud, Message 06 807 is a hard-coded warning and is not available for configuration in SSCUI 102364; changing its message type or suppressing it is not supported.

Symptom

The delivery status in Customer Demand for a Project Stock Order is Finished without cancelling or actual delivery is completed.

Environment

SAP Business ByDesign

Reproducing the Issue

  1. Go to Supply Planning work center
  2. Go to Customer Demand view.
  3. Search for the Project Stock Order: ABC (ABC represents Project Stock Order ID)
  4. In Customer Demand the Project Stock Order has the following status,
    Delivery Status: Finished
    Release Status: Not Released
    Cancellation Status: Not Cancelled

Cause

  • The Delivery Rule for Project Stock Order is maintained as "Single on-time delivery – full quantity".
  • When this delivery rule is selected, the order must be delivered in full quantity on or before the requested delivery date.
  • If the delivery is not completed within the requested date, the system automatically updates the delivery status to "Finished" in the Customer Demand, without any user interaction.

Resolution

For the affected documents, update the Delivery Date in Project Stock Order to a future date.

  1. Go to Project Management work center.
  2. Go to Project Stock Orders view.
  3. Search for the Project Stock Order: ABC (ABC represents Project Stock Order ID)
  4. Choose Edit
  5. Under Basic Data sub-tab, update the Delivery Date to a future date.
  6. Choose Save.

Complete the delivery within the revised date to prevent the system from automatically updating the delivery status to "Finished".

Symptom

- BPC data manager jobs or scripts with smallest data region, still failed due to error: Keyfigure (Signeddata) XXX is invalid due to overflow
- Business does not have needs for large key figure feature (1829297 - Large Key Figure support for Planning and Consolidation NW and BW/4HANA)

Environment

SAP Business Planning and Consolidation 10.0, version for SAP NetWeaver  
SAP Business Planning and Consolidation 10.1, version for SAP NetWeaver 
SAP Business Planning and Consolidation 11.X, version for SAP NetWeaver

Reproducing the Issue

Run Data manager jobs or BPC Scripts
Run Lite Optimize package 

Cause

BPC NW supports up to 13 digits for integer part and 7 digits for decimal part for transaction data.
Some aggregated transactional data values exceed the limit of 13 integers and 7 decimals for the key figure. 

Resolution

  1. How to identify the overflow entries:
    1. DB02: for affected BPC model cube: run SQL scripts in DB02 with keyfigure length condition, for example, "where keyfigure > 9999999999999 or keyfigure < -9999999999999
      order by keyfigure"
    2. LISTCUBE, run Listcube for BPC model and include request ID in the field list 
    3. Input the selection condition based on SQL result in LISTCUBE
  2. Solution:
    1. If these transaction data entries are identified as incorrect by business user, involve your BPC/BW consultants to delete them (physically) from the InfoCube in BW Workbench
    2. The regular BPC scripts or manually input 0 in the affected data region/cell will not help in this case, as BPC does not physically delete the data but adds a reverted value

Symptom

This KBA aims to explain the limitations and the how to reverse a document created from transaction MRKO (Consignment Settlement).

Environment

  • Materials Management (MM)
  • SAP ERP Central Component
  • SAP ERP
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA

Cause

Unlike any other Invoice Verification process, Consignment Settlement (MRKO) is a process which creates a direct posting of an FI document.
This means:

No MM Invoice involved: This makes it impossible to cancel directly from our side using our online transactions, as there is no MM document to reverse.
No direct cancellation available: It is not possible either to cancel the document from FI side directly, as the FI modules will not let you reverse a document type BKPFF (Invoice Verification), raising error F5673 - "Not possible to reverse the document in financial accounting".

Resolution

The only way to "Reverse" a FI document created from MRKO is to:

  1. Reverse the original Material Document in Inventory Management which was originally settled, which should create a new entry in table RKWA.
    How to do the reversal would depend on the type of document created (201K, 101K, Stock transfer...).
    To be sure, create the XX2 version of the original document (202K, 102K, etc).

  2. Once done, use MRKO to settle the reversal document, which will create an FI Credit Note.
    Note: Bear in mind, though, that the posting logic for reversal might be different than the one used for the original document.

  3. Then, the FI documents could be processed to clear each other, if needed.

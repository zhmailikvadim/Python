Symptom

In transaction MD04, it is observed that

  • Some MRP elements are displaying with decimal places.
  • Some MRP elements are displaying without decimal places.

Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.

Environment

  • SAP ERP Production Planning (PP) Release Independent
  • SAP ERP Release Independent
  • SAP ERP Central Component Release Independent
  • SAP Enhancement package for SAP ERP Release Independent
  • SAP S/4HANA, on-premise Release Independent
  • SAP Fiori for SAP ERP Release Independent
  • SAP Fiori for SAP S/4HANA Release Independent
  • SAP S/4HANA Cloud Private Edition Release Independent

Reproducing the Issue

  1. Open transaction MD04 and enter material.
  2. Observe that the data is displayed with decimal places instead of whole numbers.

Cause

This is standard behavior according to base unit settings.

Resolution

In transaction CUNI, you can define the decimal places for the base unit which is used by the material.
In transaction MD04, the displaying of MRP elements depends on the defined decimal places of base unit.

See Also

Number of decimal places for number display/T006-DECAN

Use

This parameter determines the number of decimal places with which this measurement unit is displayed.

  • 0000077525 - Quantity unit conversion in inventory management

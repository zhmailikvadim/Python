Symptom

  • The following error(s) can be observed in the ICM traces in T-code SMICM:

    [<timestamp>] *** WARNING => Client connection to https://<host name>:443 (over HTTP/2) broken. Request with <number> bytes sent <number> seconds ago {root-id=<ID number>} 

  • Network traffic between the source SAP system and BTP application is significant.
  • The issue occurs intermittently.
  • The parameter icm/keep_alive_timeout is set to the default value of 60 seconds.

Environment

  • SAP NetWeaver
  • SAP Web Application Server for SAP S/4HANA
  • ABAP PLATFORM - Application Server ABAP
  • BTP on Cloud Foundry

Reproducing the Issue

Frequently calling SAP BTP application deployed on BTP CF (Cloud Foundry) from an SAP Netweaver system.

Cause

For the BTP applications hosted on BTP CF platform there is a component called HAproxy that acts as a security gateway and load balancer. HAproxy supports HTTP keep-alive allowing client applications to keep the connection open and send a new request over an existing connection. HTTP keep-alive timeout for HAproxy is set to 60 seconds. It means that if the client is not sending a new request within the next 60 seconds, the connection will be terminated by the HAproxy. 

In rare cases, primarily when customer’s SAP system is geographically distant from SAP’s data center resulting in higher network round-trip times, HAproxy may close the connection in exactly same moment when the incoming request from the customer’s on-premise SAP Netweaver system is going out. In this case the request is getting lost.

Resolution

Set the static parameter "icm/keep_alive_timeout" to 50 seconds in Tcode RZ10 for all instance profiles, and then restart all application server instances.

Note: Modifying parameters in transaction RZ11 is not persistent, the change remains effective only until the next restart of the corresponding application server instance.

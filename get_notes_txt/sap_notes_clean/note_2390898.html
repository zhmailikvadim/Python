Symptom

A WCF call is made to a webservice with a long return string and is receiving the following exception. 

Exception has been thrown by the target of an invocation. ---> System.ServiceModel.ProtocolException:

The content type text/xml; charset=ISO-8859-1 of the response message does not match the content type of the binding (text/xml; charset=utf-8). If using a custom encoder, be sure that the IsContentTypeSupported method is implemented properly. The first 1024 bytes of the response were....

Environment

SAP PowerBuilder 12.6 .NET

Cause

The issue has been identified and logged under Change Request, SAP CR 803443.

Resolution

  • SAP CR 803443 has been determined to be not a bug.  ISO-8859-1 is not a Unicode charset, neither PowerBuilder .NET nor WCF support this charset.  
  • The service would need to make enhancements for this to work, for example an enhancement to use customBinding which uses a customized message encoder.  

Keywords

PowerBuilder, .NET, web service, CR 803443, cr803443

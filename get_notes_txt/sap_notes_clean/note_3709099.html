Symptom

When transitioning data mapping definitions in SAP Group Reporting Data Collection (GRDC) to newer OData services, users may encounter validation errors. Legacy mapping files based on older templates may contain field names that have either been renamed or deprecated in the enhanced API structures.

When uploading an outdated mapping file, one of the following error messages is displayed:

  • For Input Columns: "Column <column_name> is used in the mapping file as an input column but doesn't exist in the source table."
  • For Output Columns: "Column <column_name> is used in the mapping file as an output column but doesn't exist or isn't writable in the ACDOCU table."

Other Terms

S/4HANA Finance, SAP S/4HANA Cloud, Data Mapping Templates, Field Mapping Gap, API Versioning, API Deprecation, OData Services, Transformation Rules, Technical Name Changes, Field Migration

Reason and Prerequisites

To ensure a successful migration of data mapping definitions to the enhanced OData services, customers must manually adjust their mapping files and source filters to align with the updated data structures. While the enhanced APIs provide a more comprehensive set of financial attributes, certain legacy fields have been deprecated, while others have undergone technical name changes to comply with the latest S/4HANA Finance data model.

Solution

Proceed as follows to update your GRDC ACDOCA and ACDOCU data mapping files.

1. ACDOCA: From API_JOURNALENTRYITEMBASIC_SRV to API_GLACCOUNTLINEITEM

When updating your ACDOCA data mappings, please apply the following changes:

API_JOURNALENTRYITEMBASIC_SRV

Enhanced API

ControllingDebitCreditCode DebitCreditCode Rename

2. ACDOCU: From API_GRTRANSACTIONDATA_SRV to API_CNSLDTNGRPJRNLITEM

When updating your ACDOCU data mappings, please apply the following changes:

Legacy API

API_GRTRANSACTIONDATA_SRV

Once you have adjusted the field names and removed the deprecated fields as outlined above, you must update your mapping definitions in the GRDC Data Mapping application.

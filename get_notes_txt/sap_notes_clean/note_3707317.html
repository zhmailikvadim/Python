Symptom

  • The issue begins after a DPAgent restart.
  • Error occurs when testing a remote source connection.
  • Error message in framework.trc when restarting the agent:
    [WARN] Activator | Activator.installAndStartJars  - Failed to install <JDBC_driver>.jar as bundle.Could not create generation directory: /opt/sap/dataprovagent/configuration/org.eclipse.osgi/133/0 
  • Error message in framework.trc when testing the remote source:
    [ERROR] DefaultErrorHandler | CamelLogger.log - Failed delivery for (MessageId: <ID> on ExchangeId: <ID>). Exhausted after delivery attempt: 1 caught: com.sap.hana.dp.adapter.sdk.AdapterException: Failed to query schemas.
    com.sap.hana.dp.adapter.sdk.AdapterException: Failed to query schemas. Context: org.apache.commons.dbcp.SQLNestedException: Cannot load JDBC driver class '<JDBC_driver_class>'
  • Attempts to duplicate working connections result in new connections failing, while old ones continue to function despite identical configurations.
  • The correct driver files are placed in <DPA_root>/camel/lib directory.
  • The driver class parameter maintained in the remote source configuration is correct.

Environment

SAP HANA Smart Data Integration

Cause

The DPAgent service was restarted with the incorrect user (root or other incorrect user), preventing the agent from loading the JDBC driver correctly.

Resolution

  1. Apply the procedure outlined in KBA 3189501 - How to clean up the DP Agent environment after it was started as the root user - SAP HANA Smart Data Integration.
  2. Restart the DPAgent with the appropriate user.
  3. Ensure the DPAgent logging level is set to ALL and verify that the adapters are loaded properly during agent startup.
  4. Test the connection again.

Symptom

Error FAA_POST034 "Technical clearing account is not maintained for chart of accounts &1" happens when posting asset transactions from F-02/F-90 in S/4HANA environment.

Environment

  • SAP S/4HANA Finance
  • SAP S/4HANA

Reproducing the Issue

  1. Start transaction F-02;
  2. Enter document and posting dates, type AA, company code, and currency;
  3. Enter posting key 70, a G/L account for asset acquisition, and a relevant transaction type (for example, 100);
  4. Press Enter;
  5. Observe error FAA_POST034 "Technical clearing account is not maintained for chart of accounts";
  6. Optional: In FS00, check the G/L account used; FH191 "Account does not exist in chart of accounts" may appear if the account is not defined for the chart of accounts/company code.

Cause

  • The technical clearing account for integrated asset acquisition is not maintained in Asset Accounting customizing for the relevant chart of accounts.
  • The G/L account entered for posting does not exist in the specified chart of accounts/company code.

For the explanation of why technical account should be used as of S/4HANA, refer to 
KBA 2368280 - FAQ in a S/4HANA implementation in Asset Accounting

Resolution

  1. Go to below IMG path : 
    T-code: ORFA -> Integration with General Ledger Accounting -> Technical Clearing Account for Integrated Asset Acquisition -> Define Technical Clearing Account for Integrated Asset Acquisition
  2. Assign the account used in F-02/F-90 to corresponding chart of depreciation.

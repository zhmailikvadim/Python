Symptom

The following error shows when creating or updating contract:
MEOUT231 "Enter a valid &1 for item &2"

However, the field is not set as required in field selection (olme -> Contract -> Define Screen Layout at Document Level and olme -> Excel-Based Data Exchange -> Define Settings for Excel-Based Data Exchange for excel relevant update). 

The error can be observed when creating or updating the contract via the following options:

  1. BAPI: BAPI_CONTRACT_CREATE or BAPI_CONTRACT_CHANGE.
  2. MASS t-code: MEMASSCONTRACT.
  3. App: Mass Changes to Purchase Contracts by uploading excel file.

Environment

  • Materials Management (MM)
  • SAP ERP Central Component
  • SAP ERP
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA
  • SAP FIORI

Cause

Function authorization is assigned to the user via EFB in SU01. For Function authorization, FieldSelCtrl Rel (T160D-FBREFN) is maintained, in which, the relevant field is set as mandatory. 
Note: FieldSelCtrl Rel (T160D-FBREFN) is not only for PR but also considered during MASS and BAPI scenario. 

Resolution

  1. Remove the parameter EFB from the users (transaction SU01) if the parameter is not required.
  2. Remove field selection key from Function Authorization (transaction OMET) if it is not required.
  3. Set the field to optional for field selection key maintained in OMET (OLME -> Purchase Requisition -> Define Screen Layout at Document Level). 

Keywords

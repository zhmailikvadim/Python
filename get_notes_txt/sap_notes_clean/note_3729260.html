Symptom

  • Error message: FINCS_ACCTG_INTEG030 "Company assigned to consolidation unit XYZ does not exist" appears, when running the Release Universal Journal task in a planning version, when the company (assigned to the company code in Accounting) has another identifier than the consolidation unit.
  • The Release Universal Journal task works fine for the same consolidation unit in a version for actual data data (when Group Reporting Preparation Ledger - GRPL is active).

Environment

  • SAP S/4HANA Cloud Private Edition
  • Group Reporting

Reproducing the Issue

  1. Run the Release Universal Journal (RUJ) task for a planning version (source table is ACDOCP) for a consolidation unit (e.g. 1100) in a planning version.
  2. The company assigned to the company code in Accounting has another ID than the consolidation unit, e.g. a leading "S" (e.g. company-code ID is 1100 and the assigned company ID is S1100).
  3. Observe the error message: "Company assigned to consolidation unit 1100 does not exist"  (Message No. FINCS_ACCTG_INTEG030) during the Release Universal Journal task in a planning version.

Cause

  • For plan data integration, the system requires a strict 1:1 assignment between the consolidation unit and the company. It is not possible to assign the same company to multiple consolidation units like in the GRPL integration scenario for actuals data (Flexible derivation of consolidation units). For the integration within planning consolidation versions, the company (assigned to the relevant company codes) must have the same ID as the consolidation unit. Using another company ID (such as e.g. a leading "S" in the company-ID), is not supported for universal journal integration in planning versions.
  • The same is true for so-called 'classic integration', that is when GRPL (Group Reporting Preparation Ledger) is not active. The company ID has to be the same as the consolidation unit ID.

Resolution

  1. Verify that the company (assigned to the relevant company codes) has the same ID as the consolidation unit.
  2. If the company has another ID (e.g. with a leading "S") and does not match the consolidation unit ID, it is not possible to use the Release Universal Journal task for a planning version.
  3. Alternatively, use other data transfer techniques such as flexible upload or Group Reporting Data Collection (GRDC) BTP apps to transfer data from ACDOCP to ACDOCU.
  4. Refer to the SAP S/4HANA Group Reporting user assistance/online help/documentation for additional guidance on creating and maintaining consolidation units.

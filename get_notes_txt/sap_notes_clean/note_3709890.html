Symptom

After the conversion to eXTra 1.5, outbound processes of the document category ODES - DEUEV immediate notifications are no longer completed automatically, even though confirmations exist and were processed successfully using the report SI: Transfer to SI Procedures (RP_PAYDE_B2A_SV_UNPACK).

Other Terms

ODES, eXTra 1.5, RP_PAYDE_B2A_SV_UNPACK

Solution

Implement the attached correction instructions or import the specified support package.

If there are open outbound processes of the document category ODES, use the report SI: Display Inbound Processes (RP_PAYDE_B2A_SV_LIST) to check whether inbound files exist for the outbound processes.

Start the report SI: Display Inbound Processes (RP_PAYDE_B2A_SV_LIST), select the procedure IDs DSI and V2R, and remove the standard selection in "Status of Data Package".

In the "Links Between Outbound and Inbound Files" table, the "Outbound File Number"  field contains the file number of the outbound notification. Identify the outbound process that matches this file number in the B2A Manager using the "Description of Data Record" field and manually set the outbound process to  the status "OK, Completed".

Symptom

When launching the Release for Delivery app, an error message is displayed stating that no worklist could be generated and when viewing the details of the error message, it states that the HTTP request failed. When then checking the runtime error log using transaction ST22, there's a GETWA_NOT_ASSIGNED error.

Other Terms

ATP; aATP; Release for Delivery; R4D; RFDY; No worklist could be generated; GETWA_NOT_ASSIGNED; CL_ATP_BPS_SCHEDULER_SD=======CP; CL_ATP_BPS_SCHEDULER_SD=======CM00E; DETERMINE_PAST_DATES_ALLOWED

Reason and Prerequisites

The reason for this problematic behavior is, due to a programming error, a missing data retrieval for relevant document type customizing entries during the worklist generation in Release for Delivery.

Solution

Apply the correction instructions for this note or upgrade to a SAP S/4HANA support package that contains this correction.

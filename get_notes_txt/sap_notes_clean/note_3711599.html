Symptom

After performing a system COPY, the database consistency check displays the message "Table CECOEP does not exist in the database" or "Table CECOEJ does not exist in the database". 

Environment

  • Controlling (CO)
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA Finance
  • SAP S/4HANA

Reproducing the Issue

  1. Perform a system COPY
  2. Run a database consistency check
  3. Observe the message "Table CECOEP does not exist in the database" or "Table CECOEJ does not exist in the database"

Cause

CECOEP and CECOEJ are synonyms pointing to tables COEP and COEJ respectively. Upon pressing OK, the structure of CECOEP/CECOEP is visible in the Data Dictionary.

Resolution

  1. Run "RKE_HDB_EXTEND_COEX" with Extend:
    a) CECOEJ is no longer a table on the database but an alias pointing to COEJ.
    b) Account based CO-PA plan data becomes available (e.g., in KE25).
    c) However, the client copy functionality is impacted due to a).

   2. To perform a client COPY, execute "RKE_HDB_EXTEND_COEX" with Reset: 
       a) CECOEJ becomes an empty table on the database.
       b) Account based CO-PA reports will not return any plan data anymore (even though the data remains available in COEJ).
       c) The client copy functionality is restored due to a).

Additionally, SAP Note 0002777744 introduces an automatic check to ensure proper handling. 
If an account based CO-PA plan report is executed and no data is returned, the system verifies
whether `CECOEJ` is an alias. If it is not, step 1 above is automatically performed to resolve the issue.

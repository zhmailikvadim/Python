Symptom

  • Cannot implement "Default-Deny Access Control" in Information Design Tool.
  • When a user has not been granted or denied to see an object in Business Security Profile, it still can see it even after implementing the “Default-Deny Access Control”.

Other Terms

idt, restriction,4.3, deny, IDT

Reason and Prerequisites

Environment

  • SAP BusinessObjects Business Intelligence (BI) Platform 2025 Patch 08
  • SAP BusinessObjects Business Intelligence 4.3 Service Pack 05
  • Web Intelligence

Steps to duplicate

  1. Open IDT
  2. Open Security Editor
  3. Navigate to the desired universe under “Universes/Profiles” (efashion.unx)
  4. In the Universes / Profiles pane,right-click universe name
  5. Select Insert Business Security Profile
  6. Enter the name: Base Access
  7. The profile editor opens with tabs across the top 
  8. Click the Create Query tab
  9. Locate the By Object section (lower portion of the tab)
  10. The universe’s class tree is displayed.
  11. Grant all the classes by selecting them and clicking the grant button.
  12. Leave the Measures class untouched. Do not grant it. Do not deny it.
  13. Click the Display Data tab
  14. Grant the same seven classes listed in step 8 above
  15. Again, leave Measures untouched – do not grant or deny
  16. Click OK to save the Base Access profile
  17. In the Universes / Profiles pane, right-click the universe name
  18. Select Insert Business Security Profile
  19. Enter the name: Measures Access
  20. Click the Create Query tab
  21. In the By Object section, grant only: Measures
  22. Click the Display Data tab
  23. Grant only the Measures class
  24. Click OK to save the Measures Access profile
  25. In the Universes / Profiles pane, select the Base Access profile under the universe
  26. The right-hand pane displays all available users and groups alongside the Assigned Users list
  27. In the available users/groups list, locate the Everyone group
  28. Select Everyone and click the left-pointing arrow to move it into the Assigned Users list
  29. In the Universes / Profiles pane, select the Measures Access profile
  30. In the available users/groups list, locate (userA for example)
  31. In the Universes / Profiles pane, right-click the universe name
  32. Select Change Business Security Profile Priority
  33. Confirm that both profiles appear in the list:
  34. Base Access
  35. Measures Access
  36. Confirm the aggregation option is set to Very Restrictive (the default)
  37. Close the priority dialog
  38. Click the Save icon in the Security Editor main toolbar
  39. All profile definitions and group assignments are persisted to the CMS repository

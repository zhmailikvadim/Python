Symptom

An error occurs while creating an Integration Monitoring endpoint in landscape management for a system of service type 'SAP NetWeaver Application Server Java':

The endpoint couldn't be saved. Check your entry. 

[TIMESTAMP] [500] I/O error on POST request for "https://ops-im-datacollectors-srv/ConnectionTest": Read timed out

Environment

  • SAP Cloud ALM 
  • SAP NetWeaver Application Server Java (SAP Process Integration, PI)

Reproducing the Issue

  1. Open the Landscape Management app
  2. In the scope selector, select your SAP Netweaver AS for Java system
  3. Choose the service name 
  4. On the Endpoints tab, choose Add to create a new endpoint.
  5. Enter a description.
  6. Choose the use case Integration Monitoring.
  7. Enter the Cloud Connector Location ID and Authentication data, then click Save

Cause

The endpoint check verifies that the connected SAP NetWeaver AS Java system has the usage type 'SAP Process Integration' or 'SAP Process Orchestration'.

If you connect a SAP NetWeaver AS Java system with another usage type, the endpoint check fails.

Resolution

Confirm that your system has a supported usage type.

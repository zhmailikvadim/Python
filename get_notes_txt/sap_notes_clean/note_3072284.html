Symptom

An issue occurs with a scenario such as the following.

1. Use the Standard Rate Maintenance activity to assign a standard rate for the combination of a certain resource, material, router, and operation.
2. Set the Quantity to 2, the Duration to 1, and the UoM to Minutes.
3. Create and release a shop order for the material and router with a quantity of 2.
4. Start both of the SFCs at the operation and resource configured in step 1.
5. Wait a few seconds and complete both of the SFCs
6. Simulate both SFCs starting on the previous day, the first SFC completing on the previous day, and the second SFC completing on the current day:
- Manually change the PRODUCTION_LOG.START_DATE_TIME database values for both SFCs to the previous day, keeping the time part the same.
- Manually change the PRODUCTION_LOG.DATE_TIME database value for only the first SFC to the previous day, keeping the time part the same.
- Manually change the SHOP_ORDER.RELEASED_DATE_TIME and SHOP_ORDER.ACTUAL_START_DATE database values for the shop order to the previous day, keeping the time part the same.
7. Open the POD - OEE activity.
8. Select the resource configured in step 1.
9. Select the previous day which corresponds to the simulated completion date of the first SFC set in step 6.
10. Use the default shift since no specific shift was configured.
11. Select the operation configured in step 1.
12. Select the order created in step 3.
13. Turn off Automatic Refresh and click Go.

Expected Result: The Performance rate should be small and much less than 100% since the second SFC was completed the next day.
Actural Result: The Performance rate is over 100%, much higher than the expected value.

Other Terms

insights, wrong

Reason and Prerequisites

When searching for SFCs in the shop order that match the date range, only the first SFC completed on the selected day is included. This results in a small time interval and a large performance rate.

Solution

The solution has been delivered in the form of a code correction.
Changes were made to correctly include all SFCs started on the selected day, even if some of them were completed at a later date.

Impact
The changes affect OEE performance and speed loss calculations.
The following APIs are affected:
- ResourceStatusLogService.findAggregatedMinorStoppage
- ResourceStatusLogService.findAggregatedMinorStoppageByWorkCenter
- ResourceSpeedLossService.calculateAggregatedSpeedLoss
- ResourceSpeedLossService.calculateResourceSpeedLoss

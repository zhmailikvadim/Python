Symptom

Error in transaction QDB1 when maintaining allowed relationship between a sampling procedure with AQL reference and a dynamic modification rule:

Inspection severity & / AQL & not in sampling scheme & - QD883

Seemingly, the configuration is correct, but the error is raised:

Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.

Environment

  • SAP ERP Central Component
  • SAP ERP
  • SAP Enhancement package for SAP ERP
  • SAP S/4HANA, on-premise
  • SAP S/4HANA Cloud Private Edition

Reproducing the Issue

  1. Call transaction QDB1.
  2. Click 'New Entries'.
  3. Fill Procedure and DynModRule.
  4. Press ENTER.
  5. Error is raised:
    QD883 - Inspection severity 006 / AQL 0,400 not in sampling scheme Q89
    then 
    QD872 - The entry will not be accepted unless you correct the error

Cause

Configuration issue.

  1. As per the F1 help documentation of field QDSV-AQLWERT (AQL Value in Sampling Procedure):
    "...An individual sampling procedure is required for each AQL value in the sampling scheme..."
     
  2. When the user is defining the allowed relationship of a sampling procedure and a dynamic modification rule, the system checks, whether the severities in the dynamic modification rule are to be found in the sampling scheme (which is assigned to the sampling procedure).
    As a sampling procedure is required for each AQL value (point 1.) - and it is also assigned to the sampling procedure - the system checks the severities per AQL value.

If an inspection severity / AQL value combination is missing from the sampling scheme, the error in symptom is raised (QD883).

Resolution

Ensure that in the affected sampling scheme each inspection severity - of the to-be-related dynamic modification rule - has the proper AQL value as per the sampling procedure.

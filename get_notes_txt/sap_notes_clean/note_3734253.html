Symptom

When accessing an application through Work Zone, the system consistently prompts for Username and Password when it should be automatically using the existing authentication from WorkZone.

Environment

  • SAP S/4HANA Cloud Public Edition
  • SAP Build Work Zone

Cause

The runtime destination from SAP Build Work Zone to SAP S/4HANA Cloud was configured with Authentication set to 'BasicAuthentication' instead of 'NoAuthentication', causing a login prompt each time when launching the tile.

Resolution

  1. In SAP Build Work Zone, open the runtime destination that points to your SAP S/4HANA Cloud tenant.
  2. Change the destination setting Authentication to 'NoAuthentication' and save.
  3. Verify the destination URL follows the documented pattern for SAP S/4HANA Cloud Public Edition.
  4. Test the Work Zone tile again to confirm it opens without prompting for separate credentials.

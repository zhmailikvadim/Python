Symptom

  • Incorrect DSN: Blocking message during verification with the DSNVAL tool for employees who left before 2026 and for whom there is an additional payment in January 2026.
  • DSN Error: message DSNVAL S21.G00.51.002/CCH-13

Environment

  • France

Cause

  • Payments for employees with terminated contracts are incorrectly transferred to the current payroll period, resulting in a block 51 with remuneration type 002 that is not zero for the current period. Payments made after contract termination must be assigned to the employee's last active period rather than transferred to the current payroll period.

Resolution

In DSN P24V01, blocks 51 of types 001 and 002 with an amount different than 0 were not allowed in the MPD ("mois principal déclaré") for withdrawn employees. Since the remuneration in block 51 of type 001 are transferred in payroll to the current payroll period, there was a routine in RPLDSNF0 that changes the dates of blocks 51 of types 001 and 002 to the contract termination period.

In DSN P25V01, only blocks 51 of type 002 with an amount different than 0 were not allowed in the MPD for withdrawn employee. However, RPLDSNF0 was not changed to allow blocks 51 of type 001 with an amount different than 0 in the MPD for withdrawn employees.

In DSN P26V01, RPLDSNF0 is aligned with the CCH-13 for field 51.002. Blocks 51 of type 001 with an amount different than 0 are allowed in the MPD for withdrawn employees. Blocks 51 of type 002 with an amount different than 0 are not allowed in the MPD for withdrawn employees. 

With decree 2016-1567, it is functionally not possible to have a block 51 of type 002 with an amount different than 0 in the MPD for a withdrawn employee. We have deactivated in dSN P26V01 the routine that changes the dates of both blocks 51 of types 001 and 002 because the remunerations in block 51 with type 002 should be assigned to the employee's last active payroll period and should not be transferred to the current payroll period.

To solve this issue now:

  • Ensure compliance with Decree 2016-1567, which mandates that payments for employees with terminated contracts must be assigned to their last active period.
  • Verify that no wage type used to populate a block 51 with remuneration type 002 appears in the current payroll period for an employee without an active contract during this period.
  • Adapt the payroll schema and DSN configuration to align with regulatory requirements.
  • Refer to SAP Note 2554008 for detailed guidance on implementing the necessary adjustments.

Symptom

When running an SAP HANA MDS1) query you retrieve a result set with empty cells or the data entry is not possible for a cell.

1) MDS: MultiDimensional Services; analytical engine in SAP HANA, responsible for processing analytic request fom SAP Analytics clienst such as SAP Analytics Cloud (SAC) or SAP Analysis for Office.

Other Terms

SAP Analytics Cloud, SAC, SAP Analysis Office, SAP Design Studio, InA/MDS

Reason and Prerequisites

Reason:

Due to a programming error in SAP HANA, dimension members which were only relevant in a context with constant selection, were erroneously considered for axis tuple creation purposes if that dimension was in drill and had an active hierarchy.

Affected Releases:

  • SAP HANA 2.0 database, EPMMDS plugin:
    • "Fullversion EPMMDS" plugin revision: < 1.00.202614.0
  • SAP HANA Cloud Edition: 
    • < CE 2026.14.0

Prerequisites:

  • In an Account Model you create members
    • Measure Member: Selection on a Measure with constant selection with constant dimension "Dimension1"
    • Account Member: Selection on an account with an arbitrary exception aggregation on exception aggregation dimension "Dimension1"
  • "Dimension1" is in drill with a hierarchy. The hierarchy has a hierarchy node "Parent" with a child "Child".
  • The global filter is set to dimension hierarchy node "Child", i.e. "Parent" should not be shown.
  • Node "Parent" is booked and its value has to be retrieved due to the constant selection.

In such a case the node "Parent" was determined to be the parent node of node "Child" in the context of the axis creation, although "Parent" was not part of the result set due to the global filter.

This caused the axis tuples to be in an invalid state. 

Solution

Apply one of the following SAP HANA database Revisions:

  • SAP HANA 2.0 database, EPMMDS plugin:
    • "Fullversion EPMMDS" plugin revision: >= 1.00.202614.0
  • SAP HANA Cloud Edition:
    • >= CE 202514.0
  • SAP Analytics Cloud: >= 2025.28 (QRC3 2026)

Workaround:

Remove the global filter on the dimension hierarchy in question or adjust it in such a way, that the filter of the constant selection is at least as restrictive as the global filter (with regards to that dimension).

In case the measure member is not reused in a formula: Due to the constant selection, the values for all hierarchy nodes of the dimension in question are the same. Consider moving the constant selection measure to another widget where you remove the dimension from drill.

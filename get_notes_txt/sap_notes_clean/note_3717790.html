Symptom

Dump happens when the HR_Timesheet is enabled and PM_Confirmation is disabled. The error occurs when we update the Timesheet of a Technician and sync to SAP.

The dump doesn't appear when PM_Confirmation is enabled and HR_Timesheet is disabled.

Other Terms

SAMXXXX_PM_CONFIRMATION, HR_TIMESHEET, PM_CONFIRMATION, S4MOBADDON-8260.

Reason and Prerequisites

If the HR_Timesheet is active then first we will update the time in CATSDB and then call the PM confirmation API to update CNF status if the final confirmation flag is set. CNF status is always set by the PM Confirmation API for both the options.  SAMXXXX_PM_CONFIRMATION oMDO should be assigned to HR_TIMESHEET in scope.

Solution

Execute program /MFND/CORE_CONFIG_IMPORT_PROG  using BIN file and save to the transport.

Open ConfigPanel ->Component Assignments->Choose your App->click subtab oMDO Assignment->find oMDO SAMXXXX_PM_CONFIRMATION and make it active for  HR_TIMESHEET ( in case you decided disable PM_Confirmation )

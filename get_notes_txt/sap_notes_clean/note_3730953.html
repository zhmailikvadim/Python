Symptom

When completing service confirmation an inbound delivery is created instead of a material document.

Environment

SAP S/4HANA

Reproducing the Issue

  1. Create a service order and release the service order
  2. Create a follow-up service confirmation
  3. Complete the service confirmation

Cause

The storage location used for goods movement is configured as decentralized WMS relevant (or as HU required, or EWM managed). In these scenarios, postings via the confirmation process create a delivery; the material document is only created when the delivery is posted.

Resolution

  1. If decentralized WMS/EWM/HU relevance is intended for the storage location: post the created delivery to generate the material document, for example using transaction VL32N or MIGO.
  2. If a material document must be created immediately from the service confirmation: remove the decentralized WMS (or EWM/HU required) relevance for the affected storage location in customizing. For decentralized WMS, refer to SPRO > Logistics Execution > Decentralized WMS Integration > Central Processing > Application > Activate Decentralized WMS and deactivate the external WMS setting for the warehouse.

Symptom

  • BAPI_ISUPROFILE_UPLOAD: Unable to reset profile value user status.
  • Despite configuring the user status for implausible profile values and implementing the BADI ISU_EDM_PROFILE_CONVERSION with the overwrite flag set in the method IF_EX_ISU_EDM_PROFILE_CONV~SET_FLAG_OVERWRITE_CUSTSTATUS, the profile value status retains the user status.

Environment

  • SAP for Utilities as Part of SAP ERP, release IS-UT 600 and higher
  • SAP S/4HANA, release IS-UT 801 and higher

Reproducing the Issue

  1. Configure the user status for implausible profile values.
  2. Implement BAdI ISU_EDM_PROFILE_CONVERSION method IF_EX_ISU_EDM_PROFILE_CONV~SET_FLAG_OVERWRITE_CUSTSTATUS and set ex_flag_overwrite = abap_true.
  3. Import profile values for example by using BAPI_ISUPROFILE_UPLOAD.
  4. Observe that user status is not deleted.

Cause

  • System configuration.

Resolution

  1. In the BAdI method IF_EX_ISU_EDM_PROFILE_CONV~SET_FLAG_OVERWRITE_CUSTSTATUS, set ex_flag_overwrite = abap_true.
  2. Reset the existing user status for the entire imported time range by executing the action "Delete User Status for Value" (UI26).
    1. Maintain the action UI26 accordingly in the corresponding status profile (transaction OIBS).
    2. Ensure that the action UI26 is allowed for each user status and deletes it.

See Also

  • Refer to: SAP Note 985299 - Profile value import: User status is not reset

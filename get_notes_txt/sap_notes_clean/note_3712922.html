Symptom

  • Error in savereplica message processing, for Outbound Delivery Order creation: No staging area was found for storage section 0001.
  • Message no.: /SCWM/SHP_RCV074
  • Function Module: /SCWM/OUTB_DLV_SAVEREPLICA
  • Queue: DLV
  • Related customizing (using standard values, in case of custom values, check corresponding entries):
  • Warehouse Process Type determination sets Whse Proc. Type "2010" (Stock Removal) for the delivery item, where Control for Stock Removal has entries:
    • Dest.Stor.Type "9020"
    • Destination Bin "GI-ZONE"
  • Storage Type 9020 Provide in Goods Issue is set with Storage Type Role "D"
  • Storage Bin GI-ZONE is created as:
    • Storage Type "9020" (Provide in Goods Issue)
    • Storage Section "0001" (Storage Section for 9020)

Environment

  • SAP Extended Warehouse Management (EWM)
  • SAP S/4HANA

Reproducing the Issue

  1. Create Outbound Delivery in ERP, with attributes which will map a relevant Warehouse Process Type as described in the Symptom to the delivery item.
  2. Distribute the delivery to EWM.
  3. Check queue error in SMQ2.

Cause

  • Missing setup of Customizing "Define Staging Areas" (table /SCWM/TSTAREA).
  • If there is no other staging area determination configuration, system is using the Storage Bins Storage Section as Staging Area.

Resolution

There are two solutions:

  1. Create a new entry in SCM Extended Warehouse Management > Extended Warehouse Management > Master Data > Staging Areas > Define Staging Areas. Example:
    • Staging Area Group: 9020 (or custom, as it has been defined in the system)
    • Staging Area: 0001 (or custom)
    • GR flag empty
    • GI flag set (X)
    • Loading Rule (optional)
    • Staging Area Bin Capacity (optional)
       
      Alternatively, check and activate BC Set /SCWM/WA_STATING_AREA.
       
  2. Clear the Storage Section value (0001 or custom) from GI-ZONE bin definition via transaction /SCWM/LS02 (Edit Storage Bin).

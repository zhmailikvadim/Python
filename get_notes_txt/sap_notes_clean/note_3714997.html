Symptom

  • MDF parent object with no visible fields and a single one-to-one association shows only the association on the Latest People Profile summary card.
  • Summary card for a custom MDF object does not display individual fields. Instead, it shows only the association name and a record count.
  • The summary card shows a count of 1 record instead of field-level details.

"Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental." 

Environment

SAP SuccessFactors HCM Suite

Reproducing the Issue

  1. Create a parent custom object with all fields visibility set as Not Visible.
  2. Then create one single one-to-one association.
  3. The child custom object fields are visible.
  4. Then the summary card shows only the association name and a record count:

Cause

Expected behavior.

Resolution

When a parent MDF object has no visible fields and only a single one-to-one association, the summary card in Latest People Profile displays only the association information and record count. 

However, from a product design perspective, a parent object without visible fields and with only one one-to-one association provides limited value in the UI.

In such scenario, it would be recommended for better usability to transform the current child object into the parent object instead (when there are no meaningful fields at the parent level).

This approach allows relevant fields to be displayed directly on the summary card and improves the overall user experience.

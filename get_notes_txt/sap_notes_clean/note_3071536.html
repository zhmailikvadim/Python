Symptom

You execute Operation Activities / Serialized Products at Operation Activities. The operation's quantities do not get properly updated.

The problem typically occurs if several actions are executed in a short period of time.

Reason and Prerequisites

Program error:

  • The Operations get updated via Confirmations that are posted in a background process which gets triggered when Operation Activities are executed.
  • This background process may run into locking problems. In this case the background process waits and retries a predefined time.
  • If the lock is gone and the background process can start after having waited, global buffers are not refreshed so that outdated data may be considered to calculate the confirmations.

Solution

 Implement the correction instruction.

Symptom

The enterprise number for a personnel area report area changes.

Environment

  • SAP ERP 6.00 and higher
  • Human Resources / HR / HCM

Cause

The workers' compensation association sends the employer a proactive qualified confirmation for which the end of the responsibility is reported in the data module DSSD.

Solution

Complete the following process:

  1. Processing of qualified confirmation from workers' compensation association using report RPCUVAD0_IN

  2. Maintenance of Customizing in the table view V_T596M for the subapplication BG01

    For this, the valid time interval is delimited in such a way that the 'Valid To' date corresponds to the end of the responsibility. 
    Define an AI notification reason with the date for the reported end of the responsibility:
    - UV03 (end of responsibility of an AI agency) if employment relationships remain beyond the reported end of responsibility, or
    - UV05 (hiring the entire company) 

  3. Change the status of the inbound notification in the Administrator List for AI Inbound Notifications (RPCUVLD0_IN) from "To Be Checked" to "Processed Manually" using the "Manually Processed" pushbutton.

  4. Creation of the wage statement with the defined AI notification reason using the report RPCUVBD0_OUT

  5. When the reporting area is continued, the new enterprise number is defined in Customizing in the table view V_T596M for the subapplication JUPR.

  6. A new enterprise number also requires a query to the AI master data service using the report RPCUVAD0_OUT.

  7. Collection and processing of the qualified confirmation from the AI master data service using report RPCUVAD0_IN

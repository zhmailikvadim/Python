Symptom

  • Assignments cannot be released from the Dispatching Board in FSM Web to users with the PARTNER_TECHNICIAN crowd type.
  • The Release button appears to react visually (it also indicates that "All selected assignments have been released"), but the activity remains unreleased.

Environment

SAP Field Service Management 

Reproducing the Issue

  1. Log in to the Shell
  2. Navigate to Planning & Dispatching
  3. Assign any ready-to-plan activity to technician (a PARTNER_TECHNICIAN user)
  4. Click Release on the assigned activity

Actual behavior:

The Release action appears to trigger (the button responds visually and the message “All selected assignments have been released” is displayed), but the activity status does not change to Released

Expected behavior:

The activity status should change to Released immediately after clicking Release, and the assignment should be successfully released to the technician without errors. 

Cause

The issue was potentially caused by an incomplete Single Tenant Crowd configuration. The required business rules for the crowd may either be missing entirely or not configured correctly, which could prevent the system from properly processing the release action, even though the user interface indicates that the action was successful.

Resolution

  1. Review and correctly configure the business rules for the crowd.
  2. Ensure all steps in the "Configure Single Tenant Crowd" SAP Help Portal documentation are applied as described.
    Refer to: Configure Single Tenant Crowd | SAP Help Portal.
  3. Test the release functionality after applying the configuration changes.

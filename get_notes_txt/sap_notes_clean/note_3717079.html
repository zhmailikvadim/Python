Symptom

  • When displaying an attachment from 'Manage products' fiori app this fails with error: "Asset * was not found in Content Repository (KPRO) Message No. /SAPSLL/CORE_PRODUCT399".
  • Screen shot of the message below:

 "Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental."

Environment

  • SAP Business Technology Platform
  • SAP Basis release independent

Cause

HTTPs communication not configured for backend content repository.

Resolution

  1. To resolve this issue HTTPs should be enabled for Content repository LLS_COR_DOC_REP in the backend system, this is where the attachments are stored.

  2. This can be done be displaying the content repository LLS_COR_DOC_REP in transaction OAC0 and following the steps from KBA 2996974 - How to configure HTTPS for a repository.

  3. After executing command HTTPS% the parameters 'HTTPS on frontend' and 'HTTPS on backend' should be set to 'HTTPS required' for the content repository LLS_COR_DOC_REP.

  4. Also the contentserver service and external alias should be configured in transaction SICF as per KBA 2827260 - Configuration steps for /sap/bc/contentserver service.

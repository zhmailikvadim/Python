Symptom

  • Upgrading SAP ECC to Higher Support package than SAP PO.
  • Validating whether SAP PO must always have a higher SPS version than SAP ECC or vice versa is feasible?

Environment

SAP NetWeaver 7.5

Cause

System landscapes may encounter inconsistencies when other SAP systems run on a higher release than PI/PO, or when mandatory post‑upgrade configuration steps are not performed

Resolution

Find below SAP Best Practices:

  • As outlined in SAP Note 1043047, ensure that no system in the landscape runs on a higher release than the PI/PO system. Before upgrading or installing any application, confirm that PI/PO is on the same or higher release level. If needed, upgrade PI/PO first.

  • There are no direct dependencies between SAP ECC and SAP PO (NetWeaver SPS levels). Both systems function independently; however, maintaining release alignment is recommended to avoid potential integration inconsistencies.

  • Software Update Manager (SUM) is the official SAP tool that should be used for performing all upgrade and patching activities in the landscape.

  • After completing a PI/PO SPS upgrade, it is mandatory to execute the required CTC Configuration Tasks (Wizards). -> Point 5( Support Package Stack Update)

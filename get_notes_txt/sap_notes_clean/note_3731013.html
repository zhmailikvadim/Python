Symptom

  1.  You have an VSS order item with:
    a. material which Checking Group for Availability Check is set with flag described as "No PAC" or "No PCC" (field VERPN in table TMVF),
    b. position type which is relevant for ATP check in VSS.
  2. You add additional position with the same material and position type and save the order.
  3. You can observe that for the latter position more than one reservation positions were created.
  4. You cannot book the goods issue for the position with additional reservations or for all positions at once.

Other Terms

VSS order, duplicate reservation, reservation inconsistencies, RESB, /DBE/ORD_AC_AVAIL_CHECK, incorrect reservations, Supply Demand-Based Capability Check, Product Availability Check

Reason and Prerequisites

Program Error.

Solution

Apply the attached note to resolve this issue.

If needed, use the report /DBE/OE_RESERVATION_CHECK introduced by the note 3567192 or VSS 6.0 to check and possibly correct the reservations.
Before running it, please pay attention that this report should be used only by advanced user, its misuse could cause serious damage.
It is recommended to use fix functionality only for individual orders. We recommend first to test fix functionality in quality system (copy of productive system).

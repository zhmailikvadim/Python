Symptom

During the replication of FI documents from Source System to the Central Finance System the following error message occurs:

Message no. F5 567

Period & is not allowed

Environment

  • Financial Accounting (FI)
  • Central Finance (CF)
  • SAP S/4HANA
  • SAP S/4HANA Finance

Reproducing the Issue

Initiate the replication of FI documents from Source System/s to Central Finance as part of the Initial Load or Real Time Replication.

Cause

This error is usually triggered in the ledger transformation scenario. The source document is posted without any ledger (classic GL) or it is posted in a ledger which has a different fiscal year variant than the target ledger.

When the document is replicated to Central Finance the source period does not correspond to the period derived from the target fiscal year variant.

 

Example:

 

Source FYV-K4 (Jan-Dec)

Target FYV-V3 (Apr-March)

 

The source document is posted on 01.06.2020 (period 6) while in Central Finance posting date 01.06.2020 corresponds to period 3.

The same error will be triggered in case of special periods. If the source document is posted on 31.12.2019, but in period 13 and the target Fiscal Year Variant does not allow for special periods (for example FYV K0), then the posting will fail during replication as the target period does not correspond to the source period.

Resolution

Clear the Period (MONAT field in CFIN_ACCIT) in the Central Finance BAdI. When the document is posted, the period will be rederived based on the target Fiscal Year Variant.

Keywords

CFIN, period, fiscal period, fiscal year, allowed

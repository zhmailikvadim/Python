Symptom

There is a rule configured to remove the position from the user based on the event reason in global assignment. However, it is not working as expected. 

Environment

SAP SuccessFactors Employee Central 

Reproducing the Issue

  • Go to user ABC.  
  • Add a Global assignment for the user.
  • The position was removed for the employee and right to return applied. However, as per the rule the position shouldn't be removed. 

Cause

This is an expected behaviour. 

Resolution

  1. Rule execution will happen on the home assignment and not on the host assignment. The rule will be executed during the creation process of a new job information record with the event “Away on Global Assignment”. It is not clear on which job information record the rule should be executed. 
  2. It is recommended to use the standard rule configuration on the implementation guide  Position Management Settings for Right to Return | SAP Help Portal   
  3. This data will be added in implementation guide via Jira ECT-266681. 

Keywords

ECT-266681, Away on Global Assignment, Global assignment rule, event reason, INC18030527

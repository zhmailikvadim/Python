Symptom

When a DEBMAS or CREMAS IDoc with E1KNB5M segments is processed, or during mass updates to dunning data of customers or suppliers, you receive the error message "Customer ...: Default dunning area was deleted for company code ..." resp. "Vendor ...: Default dunning area was deleted for company code ...".

Other Terms

CMD_EI_API 106, VMD_EI_API 106, XD99, XK99, WE19, BD87, IDOC_INPUT_DEBITOR, IDOC_INPUT_CREDITOR, CMD_EI_API_CHECK=>TABLE_KNB5_CHECK, VMD_EI_API_CHECK=>TABLE_LFB5_CHECK

Reason and Prerequisites

There is a program error causing dunning information to be mixed up across company codes in some cases.

Solution

Implement the attached correction instructions.

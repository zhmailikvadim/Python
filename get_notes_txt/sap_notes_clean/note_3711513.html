Symptom

  • Unable to access the external catalog through the "Create Items from External Catalog" functionality.
  • Missing "SessionTransmissionId" parameter.
  • Error message: "Session could not be established. Please try again."

Environment

  • Materials Management (MM)
  • SAP S/4HANA 2022
  • SAP S/4HANA 2023
  • SAP S/4HANA 2025
  • SAP S/4HANA Cloud Private Edition 2023
  • SAP S/4HANA Cloud Private Edition 2025

Reproducing the Issue

  1. Open the Manage Purchase Requisitions Professional app.
  2. Create a new Purchase Requisition.
  3. Attempt to access an external catalog using "Create Items from External Catalog" functionality.
  4. Error appears.

Cause

As of SAP S/4HANA 2022, it is necessary to maintain an RFC destination to the relevant server in table MMPUR_OCI_CONN.

When using Secure OCI, it is expected the catalog provider sends the SessionTransmissionId parameter back to SAP when launching the catalog. However, this will only be fetched by SAP if the RFC connection is maintained in table MMPUR_OCI_CONN.
If this feature is not supported by the catalog provider, it is recommended to not use Secure OCI. 

Resolution

  1. Maintain the required customizing in OA2C_CONFIG. Refer to the documentation: OAuth 2.0 Client for AS ABAP | SAP Help Portal.
  2. Create an RFC Destination.
  3. Maintain the connection in the table MMPUR_OCI_CONN.

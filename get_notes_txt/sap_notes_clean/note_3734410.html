Symptom

  • Calling a classic BAPI from a RAP behavior leads to a dump when the BAPI contains COMMIT WORK.
  • Calling the BAPI in the SAVE phase also dumps due to an AUTHORITY-CHECK inside the BAPI.
  • Wrapping the BAPI call in a class does not prevent the dump.

Environment

SAP S/4HANA 2023

Reproducing the Issue

  1. Implement a RAP business object (managed or unmanaged) and expose operations via OData V2.
  2. From a RAP handler (for example, during SAVE or modify), call a classic BAPI that performs COMMIT WORK or contains an AUTHORITY-CHECK.
  3. Execute the operation and observe that a runtime dump occurs.

Cause

The BAPI issues statements that are not allowed during specific phases of a RAP transaction (for example, COMMIT WORK), causing a dump.

Resolution

  1. Review the BAPI against the RAP usage requirements and ensure it is compatible; Read SAP Community posts provided in the See Also section.
  2. Avoid calling BAPIs that execute COMMIT WORK or other prohibited statements within RAP transactional phases.
  3. If the BAPI cannot be adapted, decouple the call using synchronous RFC (sRFC) so that it runs in its own LUW.
  4. Test the adjusted design to confirm the dump no longer occurs.

Symptom

Unable to retain multiple simultaneous API sessions for a single user because active sessions are terminated prematurely when new sessions are established.

Environment

  • SAP Sales Performance Management
  • Agent Lifecycle Management

Reproducing the Issue

  1. Initiate the API calls A and B from the internal server, from the same User ID. 
  2. Log out from session B.
  3. Observe that the previous session (A) is invalidated. 

Cause

This is the expected behavior due to the design of the system, which assumes one API user per application to prevent accidental session renewal.

Resolution

  1. This behavior is expected, as environments are designed to have one API user per application to prevent accidental session renewal.
  2. If the setting system.api.session.timeout is set to -1, sessions will not expire, and the same session can be reused, without the need of initiating a new one.

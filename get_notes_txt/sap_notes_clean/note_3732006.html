Symptom

In the log file of a document with 'Failed to Push' status in Monitor Document section of Peppol Exchange app, the error message 'Server returned HTTP response code: 404 for URL' is being thrown.

Environment

  • SAP ERP
  • SAP S/4HANA Private Cloud Edition
  • SAP Document Reporting and Compliance service

Cause

The endpoint configured in Cloud Connector or BTP Destination does not match the correct PushRequest path.

Resolution

This issue typically occurs due to:

  • An incorrect URL copied into the Destination in the BTP Subaccount.
  • An incorrect resource path exposed in the Cloud Connector.

To resolve the error, ensure the following settings are maintained:

  • Maintain the Alternative Access URL /pushrequest as described in Step 6 of Help page Configuring Provider Service | SAP Help Portal. After saving the service definition, the system generates a calculated access URL; retain this URL for upcoming steps.

  • When adding a resource to a connection, ensure the URL path follows the structure:
    /sap/bc/srt/rfc/sap/dcppushservice/<your client>/dcppushservice/pushrequest.
    Refer to the note section in Help page Adding Resource to Connection | SAP Help Portal for details.

  • Use the same calculated access URL when creating the destination for the PushRequest, as described in Help page Setting Up Connectivity | SAP Help Portal.

  • Confirm that the backend user maintained in the BTP Destination is assigned to the role:
    SAP_BC_WEBSERVICE_SERVICE_USER

  • Ensure the user has the following required authorization objects:

    • EDO_BUK
    • EDO_PROC
    • S_PROGNAM
    • S_SERVICE
    • S_BTCH_JOB
    • S_BTCH_ADM

For further details on the required values for each authorization object, refer to Help page Setting Up Connectivity | SAP Help Portal.

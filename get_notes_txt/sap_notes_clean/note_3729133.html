Symptom

When changing a planned order's date in F0251A - Manage Material Coverage app,
system does not pop up the warning message "Date 04.03.2026 lies in the past" (Message No. 61887), which is different from t-code MD12.

Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.

Environment

  • SAP ERP Production Planning (PP) Release Independent
  • SAP ERP Release Independent
  • SAP ERP Central Component Release Independent
  • SAP Enhancement package for SAP ERP Release Independent
  • SAP S/4HANA, on-premise Release Independent
  • SAP Fiori for SAP ERP Release Independent
  • SAP Fiori for SAP S/4HANA Release Independent
  • SAP S/4HANA Cloud Private Edition Release Independent

Reproducing the Issue

  1. Open the F0251A - Manage Material Coverage app.
  2. Select material, plant and MRP controller.
  3. Change the planned order date to a new date that cannot be met due to production time.
  4. Verify that the basic end date is not updated and no warning message "Date 04.03.2026 lies in the past" (Message No. 61887) is displayed.

Cause

This is part of the standard system design.

Resolution

The request to display warning message 61887 in F0251A is not supported due to differences in functionality and logic design between the Fiori App and the GUI transaction.

When changing the planned order value in Manage Material Coverage (F0251A), 
the system triggers the BAPI_PLANNEDORDER_CHANGE function module, which operates differently from t-code MD12.

See Also

Tr. SE37: open BAPI_PLANNEDORDER_CHANGE -> FM document -> parameters -> Return

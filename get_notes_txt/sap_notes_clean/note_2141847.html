Symptom

When using SAP Enterprise Project Connection(EPC) 2.0 to update existing project in Microsoft Project Server with data from SAP Project System( SAP PS), the new nodes from SAP PS like WBS, Network, Activities are not created at the correct position in the Microsoft Project Tasks hierarchy.

Other Terms

Enterprise Project Connection, Microsoft Project Server, MPS, EPC, SAP EPC2.0

Reason and Prerequisites

The Microsoft Project Server Task objects were not being placed in the correct position in the Task hierarchy.

Solution

SAP EPC Sample Integration Solution(SIS) for Microsoft Project Server(MPS) field mapping for Task fields: AddPosition and AddAfterTaskUID is fixed.

SAP EPC 2.0 Support Pack 03 (SP03) release includes fix for this issue.

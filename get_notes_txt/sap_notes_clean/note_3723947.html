Symptom

  • Site account have no password associated with it.
  • Duplicate accounts are created with the same email address when performing social login.
  • The accounts are not linked despite having the same email address.

Environment

  • SAP Customer Data Cloud
  • Customer Identities

Reproducing the Issue

  1. Register a user.
  2. Verify the user's email address to complete account verification.
  3. Perform a Social Login using Social Network (e.g. Google) with the same email address used during registration.
  4. Unable to complete link account process as user does not have password.
  5. Observe that two separate accounts exist in the database with the same email:
  • One created via accounts.register.
  • One created via Social Login.

Cause

The accounts are not linked during the Link Account process due to the absence of user action to merge accounts.

Resolution

  1. Use Link Account V2, which supports account linking via OTP (One-Time Password) through SMS or Email for users without passwords.
  2. Refer to the documentation for Link Account V2 for detailed information

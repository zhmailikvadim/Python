Symptom

When creating planned independent requirements in T-code MD61,
error 6P492 'No requirement type exists for material XXX in plant XXX' occurs.

Reproducing the Issue

  1. Open T-code MD61 and fill the Initial screen
  2. Enter
  3. Error 6P492 'No requirement type exists for material XXX in plant XXX' occurs.

Cause

No valid requirements type can be determined when creating PIRs.

Resolution

To resolve the error message, it is required to maintain

  • Requirements type in T-code MD61 - User parameter or
  • maintain Strategy group in Master data T-code MM02 MRP3 view or
  • assign MRP Group in Master data T-code MM02 MRP1 view and assign Strategy to MRP Group: Use transaction OPPU (or go to: Production -> Material Requirements Planning -> MRP Groups -> Assign Strategy Group to MRP Group) to link the created Strategy Group to the MRP Group.

Message No. 6P492

Diagnosis

No valid requirements type could be determined.

This cause of this could be:

1) For Transaction MD61 no requirements parameters (Requirements parameter ...) are maintained.

2) No MRP group in the material master (View Materials planning 1), and

3) No strategy group in the material master (View Materials planning 3) maintained.


System Response

The system determines the valid requirements type from the strategy group that was allocated in the material master record.

Procedure

Check the possible cause of the error and maintain the appropriate function.

Cause number 1 is maintained in the material master record.

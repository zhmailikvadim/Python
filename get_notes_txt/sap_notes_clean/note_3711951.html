Symptom

 You want to upgrade the SAP HANA component "SAP EXTENDED APP SERVICES 1".

Other Terms

 Update SAP HANA extended application services, advanced model, SAP HANA XS advanced model, SAP HANA XS Advanced, XSA

Reason and Prerequisites

This release of SAP HANA extended application services, advanced model contains important security updates and is recommended to be installed by all customers using a previous release:

Compatibility:

  • This release is compatible with SAP HANA 2 SPS05 and SPS07, and SPS08
  • This release requires at least RHEL 8 or SLES 15 SP3, see also SAP Note 3475745.

Download:

  • Download this release from the SAP Support Portal.

Solution

XS Advanced Runtime

Added:

  • The xs users command now shows more information about users, like e-mail address, human readable name and role collections. In addition, you can get information about XS Advanced users maintained by an external Identity Provider.

Fixed:

  • Fixed an issue where the command XSA select-xsa-runtime-db would return with the following error message after restoring a backup that was taken from a system with a different platform (x86, ppc64) even when the XS Advanced version matches up.

    The installed XSA version '1.<x>.<y>.<timestamp a>' is lower than the XSA version '1.<x>.<y>.<timestamp b>' in the selected tenant database. Please install the latest XSA version, first, before re-trying this command. 

  • Fixed an issue where access logs displayed by the xs logs command could have been out of order.
  • Fixed an issue with the login redirect to UAA after configuring an external domain, e.g. with the xs create-domain --external command, without configuring a port explicitely using the --port option. The following error messages would be visible in the browser:

    Corrupted Content Error

    or

    ERR_INVALID_REDIRECT

    As a workaround for previous XS Advanced versions, use the xs cli --port <port> option with the command xs create-domain --external to configure the router port (30<SNR>33 by default) of the external domain explicitely or specify the router port explicitely when configuring an external domain with the XS Advanced Cockpit.

UAA Server

  • Changed the default value in xsuaaserver.ini of property

    [login.yml]
    wantAssertionSigned    =true

  • Implemented comprehensive audit logging for HANA authentication events in the OIDC module, enabling tracking of authentication success and failure events with detailed contextual information.

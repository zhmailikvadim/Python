Symptom

In sales order when try to simulate , it gives the error message  "Code list mapping for value XYZ of data type BusinessTransactionDocumentProcessingTypeCode missing" .

Environment

Integration of C4C with ERP

Reproducing the Issue

  1. Navigate to the Sales Orders work center.
  2. Open an affected sales order
  3. Select Actions > Simulate.
  4. Observe the error message: "Code list mapping for value XYZ of data type BusinessTransactionDocumentProcessingTypeCode missing."

Cause

  • The communication arrangement "Pricing in Sales Quote Replication Scenario in SAP Business Suite" was configured instead of "Pricing in Sales Order Replication Scenario in SAP Business Suite."
  • Previously, a bug allowed the sales order simulation to work by using the available code list mapping, provided it was unique.
  • After the 2511 upgrade, a new standard code list mapping group was delivered, resulting in non-unique code list mapping groups. This caused the system to fail in identifying the correct code list mapping group for sales order pricing calls.

Resolution

  1. Configure the communication arrangement "Pricing in Sales Order Replication Scenario in SAP Business Suite" for sales order pricing calls.
  2. Specify the correct code list mapping group in the newly created communication arrangement.
  3. Test the sales order pricing call to ensure the issue is resolved.

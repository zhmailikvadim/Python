Symptom

Importing a schema into the System Landscape Directory with a name that matches an existing schema doesn't trigger a warning. This means you might accidentally overwrite data.

Reproducing the issue

Scenario:

  1. Access the System Landscape Directory Control Center.
  2. Launch a schema import operation with a name that's already in use.
  3. Destination → Rename Target Schema is unchecked.

System behavior prior to SAP Business One 10.00 FP2605:

The pop-up "The existing schema will be overwritten" is displayed only when a non-selected schema is at risk of being overwritten.

System behavior after upgrade to SAP Business One FP2605:

The pop-up "The existing schema will be overwritten" is displayed when a non-selected schema is at risk of being overwritten.

The pop-up now appears also when you're about to overwrite the selected schema.

Additional information:

This feature is designed to give users greater control over their import processes. This provides an extra layer of confirmation and minimizes the risk of accidental overwriting.

Cause

This pop-up prevents accidental overwrites of existing schemas. It appears when the target schema name conflicts with any existing one, ensuring you're informed before any changes are made.

Solution

System Behavior

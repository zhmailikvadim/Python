Symptom

  • Attempting to delete the current cost estimate by the cost run ID or deleting the current cost estimate is not working in the background mode in app 'Reorganization of Cost Estimates' CKR1.
  • The background processing option is grayed out.
  • When attempting to delete the cost estimate in the foreground mode, the process fails due to the large volume of data, causing the system to terminate the transaction after approximately 10 minutes.

Environment

SAP S/4HANA Cloud Public Edition

Resolution

The functionality to execute 'Reorganize Cost Estimates (CKR1)' in a background mode is not provided in SAP S/4HANA Cloud, Public Edition. This limitation is by design, as the functions "Execute in Background" and "Execute and Print" are excluded from use in cloud scenarios.

For more information about how to submit new Feature Request, check KBA  2963059 - How to Submit Feature Requests for SAP S/4HANA Cloud Public Edition - SAP for Me .

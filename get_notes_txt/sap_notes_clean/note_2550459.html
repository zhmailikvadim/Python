Symptom

During the replication of a new Employee from SFSF EC to SAP ERP or S/4 HANA with BIB one of the following error messages appears in the corresponding application logs (transaction SLG1):

  • "Personnel number 00000000 not permitted" (PBAS_SERVICE 030)
  • "Enter a personnel number" (PG 000)
  • "Personnel number is not within interval for number range XX"(PG034)

Using PTP replication from SF EC to ECP system, next error message is found on SLG1:

  • Feature NUMKR returns internal number range &1 for external PERNR &2 (HRSFEC_PERNR 005)

Environment

  • SAP SuccessFactors Employee Central integration to SAP Business Suite / SAP SuccessFactors Employee Central integration with SAP Human Capital Management for SAP S/4HANA / SAP SuccessFactors Employee Central Payroll
  • Employee Master Data Replication
  • BIB/PTP Replication

Cause

1. When an Employee is distributed from SFSF EC to SAP ERP or S/4 HANA or ECP, the receiving system has to determine whether it has to create a new Personnel Number (PERNR) for that Employee.

2. If it has to create a new PERNR for that Employee, a new 8-digits number will have to be selected for the PERNR to use.

3. Use transaction PA04 or transaction SNUM / object RP_PREL to create the number ranges that will be allowed for the replicated Employees, and whether they are internal or external number ranges.

4. Configure feature NUMKR with transaction PE03 to determine which of the number ranges created will be used for a given Employee.

4.1. At the start of the replication, the values for Company Code (BUKRS), Personnel Area (WERKS), Employee Group (PERSG), Employee Subgroup (PERSK) and Molga are pre-evaluated for the Employee so that Feature NUMKR is evaluated.

4.1.1. Configure tables T500P and T503 appropriately so that those five fields are properly determined.

5. If the number range determined by feature NUMKR is internal, the PERNRs will receive the next number from the sequence of the number range automatically.

6. For BIB replication. If the number range determined by feature NUMKR is external, follow one of the options:

6.1 Starting from release 1911 (PA_SE_IN 100 SP0027), you can use the Assignment ID from EC as the number for the PERNR.
6.2 Else, you have to implement BAdI ECPAO_IN_EXT_PERNR_MAP to select the number for the PERNR.

7. For PTP replication. If the number range determined by feature NUMKR is external:

    • Assignment id could be used to create the pernr if switch SFEC PERNR = 1 on T77S0
    • BAdI HRSFEC_B_CE_EXT_PERNR_MAP could be used to generate external pernr if switch SFEC PERNR = blank

Resolution

Verify the customizing in tcode SNUM and feature NUMKR and make sure that it's consistent with the customizing on replication side:

  • If an internal number range is returned by feature NUMKR, an internal number range should be configured on tx. SNUM and assignment id constant shouldn't be activated on BIB. Using PTP , T77S0 switch SFEC PERNR should be blank.
  • For BIB replication using external number range, see the documentation for the Assignment ID Use of Assignment ID in Employee Central Integration with SAP ERP HCM | SAP Help Portal and remember to activate view V_ECPAO_CONSTANT entry "Using Assignment ID in Employee Central Integration with SAP ERP HCM".
  • For PTP replication using external number range, see the documentation Creating Personnel Number Using the Employee Central Assignment ID.

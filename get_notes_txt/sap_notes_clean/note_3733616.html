Symptom

When you complete a password reset flow, you encounter an error when the system redirects you back to your application (for example, SAP Build Work Zone).

Common scenarios where this occurs:

  • You're on a mobile device and open an application like SAP Build Work Zone. During authentication, you trigger the password reset flow. After you reset your password, SAP Cloud Identity Services attempts to redirect you to your application, and you get an error.
  • You open the password reset link in a different browser than the one where you initiated the Forgot Password flow.
  • You open the password reset link in incognito or private browsing mode.
  • Your original browser session expires, or you close it before you complete the password reset.

Other Terms

IAS, Identity Authentication, approuter, password reset, forgot password, HTTP 400, HTTP 401, unauthorized, JSESSIONID, state parameter, Work Zone, Mobile Start, session cookie

Reason and Prerequisites

An application such as SAP Build Work Zone sets a session cookie during initial access. You then trigger a password reset flow. After you reset your password in a different browser context (different browser, incognito mode, mobile device, or after you close the original browser), SAP Cloud Identity Services attempts to redirect you back to your application. However, the system doesn't have access to the session cookie.

Solution

Product development is evaluating solutions for this issue. Watch our release notes for updates.

Workaround

For End Users

Option 1: For Non-Mobile Scenarios, Use the Same Browser Context

  • Complete the password reset using the same browser where you originally chose Forgot Password.
  • Don't use incognito or private mode if you made the original request in normal mode.

Option 2: Start Fresh After Password Reset

  1. After you successfully reset your password using the email link, close the browser tab that shows the error.
  2. Go directly to your application URL (for example, the Work Zone homepage).
  3. Log in with your new password.

For Administrators

User Communication: Inform users that after they complete a password reset, they should:

  1. Close any error pages.
  2. Go directly to the application URL.
  3. Log in with their new password.

The password reset itself is successful even when the error appears. Only the automatic redirect fails.

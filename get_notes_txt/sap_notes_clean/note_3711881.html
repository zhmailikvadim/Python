Symptom

  • It is not clear how to maintain field BEMEH (supplementary unit) in Fiori application. 
  • Supplementary unit is missing in the intrastat declaration. 

Environment

  • International Trade Management 
  • International Trade Classification
  • SAP S/4 HANA for International Trade
  • SAP S/4HANA Cloud

Reproducing the Issue

When editing a commodity code in the 'Manage Commodity Code' app, the 'First Unit' and 'Second Unit' fields are not displayed.

Cause

The numbering scheme content for the commodity code has not been defined or populated.

Resolution

  1. You need first to define that BEMEH is available in Fiori application
    • Logon to the Fiori launchpad with user BPC_EXPERT. 
    • Go to Manage your solution > Configure your solution 
    • Select Application Area 'Finance' and Sub Application Area 'International Trade Management'
    • Navigate to item Classification'
    • Configure Configuration step 'Define Numbering Scheme Content for Commodity Codes'.
    • Select the relevant numbering scheme content. 
    • Assign the Unit of Measure system.



  2. Then, you can maintain BEMEH in Manage Commodity Codes FIORI app. 
    • Logon to the Fiori launchpad with user TRD_CLS_SPECIALIST.
    • Open app 'Manage commodity Codes'.
    • Select the numbering scheme content. 
    • Edit the commodity code.

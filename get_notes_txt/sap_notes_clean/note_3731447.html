Symptom

Why are approval nodes displayed differently for different users in the Contract Workspace (CW) approval task?

Reproducing the Issue

1.Create a template
2.Go to the Conditions tab and build a condition related to the User Groups
3.Create an approval task and set the Approval Rule Flow Type to Custom
4.Add a rule title and define the reference condition as Not User Groups
5.Add approvers and groups under the action, and set the supervisor as the Owner
6.Create a CW for the template
7.Check the approval task nodes
8.Users who are part of User Groups will not be able to see the approval task nodes.

Cause

At the template level, the approval nodes are built based on conditions related to User Groups and set condition as Not User Group.  Users who are part of a specific group may see different approvers in the approval task node.

Resolution

When creating a condition using User Groups on approval task nodes, ensure the conditions are configured to include the users who need to view the CW approval nodes.  Users who are part of the User Groups may not see the approval nodes correctly due to the conditions defined for the task nodes at the template level.

If users are in different locales and the amount is involved in the approval node conditions, you need to verify the amount calculation used in the approval node. You can enable the parameter Application.Approvable.LookupTableStrictEN_US Setting this parameter to Yes forces the approval process to use the English locale, ensuring that approval flows are generated correctly regardless of the user’s preferred locale.

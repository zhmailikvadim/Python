Symptom

When Partner Communication documents are generated through the mass processing of Calculation Buckets, the system is expected to create the corresponding PC documents based on the processed Transaction Calculation Run documents, with the appropriate items assigned to each document. In some cases, however, the system may generate an incomplete number of PC documents or create documents with an incorrect number of items.

This note addresses the issue by correcting the behavior to ensure that the corresponding PC documents are created successfully with the appropriate items.

Other Terms

/IRM/IPCBCLBPMP, Mass process, Auto PC, Calculation Run, Calculation Bucket, TCR, Creation

Reason and Prerequisites

This is caused by program error.

If you are in 6.1 Base package, 6.2 Base Package, please implement SAP Note # 3217889 as  pre requisite note.

Solution

Please implement the correction instructions attached. These corrections are only valid if you have applied the Support Packs.

Release 6.1 - Base package, 6.2 - Base package, 6.3 - Base package, 6.4 - Base package.

1. Perform the manual pre-implementation steps that are described below.
2. Use transaction SNOTE to implement the correction instructions specified below and activate the affected program object.

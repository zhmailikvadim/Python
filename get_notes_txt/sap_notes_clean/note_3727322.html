Symptom

  • Tags are not being picked correctly when selected from manage templates → custom competency option.
  • When selecting a custom section type in the competency section and adding an attribute linked to multiple tags, the tag selected at the bottom reverts to the tag displayed at the top after saving the template.

Environment

SAP SuccessFactors Performance Management

Reproducing the Issue

  1. Navigate to manage templates and open the performance form 
  2. Go to the competency section.
  3. Select "show advanced options" and click on "custom skill defined".
  4. Open any of the attributes added and select any tag which is located at the bottom.
  5. Save the template.
  6. Reopen the custom skill section and check the tag of the attribute added under the custom option.
  7. The tag automatically changes and displays the tag located at the top of the attribute linked.

Cause

The tag field in manage templates is designed purely as a filter to help locate skills under a specific tag during selection. Once the skills are selected and saved, the tag used for filtering is not stored or captured—it is not part of the template configuration.

Resolution

  1. Understand that the tag field in manage templates is intended solely as a filter for locating skills under specific tags during selection.
  2. Confirm that once the skills are selected and saved, the tag used for filtering is not stored or captured in the template configuration.
  3. Note that when the custom skill section is reopened, only the selected skills remain saved, while the tag associated with the skill reverts to the tag displayed at the top.
  4. The form does not indicate which tag a skill belongs to, and this behavior is functioning as intended within the system's current design.

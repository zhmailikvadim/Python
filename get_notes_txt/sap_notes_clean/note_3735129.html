Symptom

When trying to deploy a custom agent for Joule via Joule Pro Code Dev tools, the following error messages take place:

  • Skipping deployment for capability: <capability_name> includes a custom agent, which cannot be deployed without user consent. Please remove the agent or provide consent via Admin Center and try again.
  • User is not authorized to deploy sap capabilities

Environment

  • Joule
  • Joule Studio
  • Joule Studio CLI

Reproducing the Issue

  1. Attempt to deploy a capability that includes a custom agent using the pro-code tools or CLI.
  2. Observe that the deployment is skipped with a message indicating user consent is required for custom agents, or an authorization error appears and the deployment ends with DEPLOY_FAILED.

Cause

Custom Agent in Joule is currently available as a free promotional feature for a limited time. Due to which, SAP needs explicit confirmation (consent) before Custom Agents can be deployed.

Resolution

In order to provide consent, access Joule Admin Center and go to the Joule Editor for Building Agents tab. Joule Admin Center access and more details on this process are described in: Joule Admin Center.

Symptom

  • UNX generates duplicate schema prefix in SQL when Data Security Profile is applied
  • SQL syntax error when the Web Intelligence report is refreshed

Environment

  • SAP BusinessObjects Business Intelligence Platform 2025 patch 7 (BI 2025)
  • Information Design Tool (IDT)
  • Web Intelligence (Webi)

Reproducing the Issue

  1. A universe with a Data Security object was migrated from BI 4.3
  2. When the Data Security object was applied to a Webi report, the report got a SQL syntax error.
  3. Checking the SQL generated, there was a duplicate schema name. 
    • schema.schema.table instead of schema.table

Cause

In the Data Security Profile, a replacement table was configured that pointed to the same table. 

Resolution

Delete the unnecessary replacement table mapping.  

  1. Log into IDT as Administrator
  2. Open the Security Editor
  3. Edit the Data Security Profile under your universe.
  4. Go to Tables tab and delete the Replacement table as it is identical to the original table. 
  5. Save the changes.
  6. Try the report again.

Symptom

  • Attempt to upgrade from Information Steward repository form 4.2 SP14 to 4.3 SP3 was unsuccessful.
  • Error message in InformationSteward.ISRepositoryUtility_<timestamp>.log states "cannot use duplicate table name: MMT_<object_name>....".

Environment

Upgrade Information Steward from 4.2 to 4.3

Cause

The root cause of the issue is still unknown. The issue happened with combination of manual modified InstallUtility.properties and multiple unsuccessful upgrade installation/repair installation. The key area cause the symptom is there are some repository base tables existing in the database schema while the definitions of those tables are missing in the MMT_Table (one of Information Steward core object in repository).

Resolution

  1. First run the ISRepositoryUtility or Repair installation, if that doesn't fix the issue continue to step 2
  2. Find list of base tables which may cause the issue manually. The easiest way is to use database metadata query to find list of tables in the Information Steward repository schema and outer join with MMT_Table to find those orphan tables with "MMT_" surfix.
  3. Manually, drop or rename the list of issue tables found in above step.
  4. Rerun ISRepositoryUtility and the Information Steward repository should be upgraded successfully.

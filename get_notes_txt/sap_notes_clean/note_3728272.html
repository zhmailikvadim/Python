Symptom

  • GJ19 is executed for company code, Joint Venture from old equity group to new equity group for specific posting dates observe that JVE2 postings are missing. (For example: from 01.01.2021 to 30.11.2025, JVE1 and JVE2 postings are successfully generated for the period 01.11.2021 to 30.11.2025, but JVE2 postings are missing for the period 01.01.2021 to 31.10.2021).
  • Only postings for Equity Group are generated for JVE1 posting, but postings for Equity Group are missing for JVE2.
  • Displaying JV document (transaction code GD23) showed JVE1 without repost JVE2, while documents contained both JVE1 and JVE2.

Environment

  • Joint Venture Accounting (JVA)
  • Financial Accounting (FI)
  • SAP ERP
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA

Reproducing the Issue

  1. Navigate to transaction code GJ19 for company code, Joint Venture, from Equity Group (old) to Equity Group (new) for specific posting dates.
  2. Display JV document (transaction code GD23) observe that it contains only JVE1 without repost JVE2.
  3. Compare with documents which contain both JVE1 and JVE2 postings.

Cause

The issue is caused by a failure in the prior period adjustment job. This failure can be checked in transaction code GJEV under the event where adjustments are processed. Such failures can arise due to system resource constraints, user intervention, or the presence of buffer documents.

Resolution

There are only two possible solutions available to tackle this scenario. SAP recommends using either of the options mentioned below to post manual adjustments

  1. Perform manual adjustments using transaction code GB01:
    • Configure GB01 to enable postings to ledgers 4A and 4C.
    • Use the "post to all ledgers" option or configure specific ledgers for postings.
    • Coordinate with internal Special Ledger consultants for setup assistance.

  2. Alternatively, develop a custom program using the function module JV_GB01_DIRECT_INPUT:
    • Create a custom report to input necessary line items into the function module.
    • The function module will post entries to the JVA ledgers as required.
    • Engage internal technical teams for development and testing.

Similarly for S/4HANA environments:

  • JVA is integrated with ACDOCA, and GB01 is not required.
  • Resolve similar issues via standard FI adjustment postings.

Irrespective of the approach chosen, run it once in the quality/non-production system before preforming the activities in production system to avoid any unnecessary discrepancies.

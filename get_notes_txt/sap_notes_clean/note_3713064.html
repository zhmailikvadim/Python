Symptom

When accessing an application that loads an editable and ALV in WebGUI, you create a new line, and press F4 for a field.

After selecting an entry, the value selected is filled in another unrelated cell.

Other Terms

ALV; ABAP List Viewer; SAP List Viewer; Grid; List; WebGUI; F4;

Reason and Prerequisites

Reason

The inner state/backend of the editable ALV does not have any row by the time of the call, as there was no "Enter" pressed (check_changed_data) yet.
After selecting the F4 value, the backend of the ALV tries to get in sync with the frontend, and the focus is set to the incorrect position.

Prerequisites

The following Support Package level is the minimum prerequisite:

Release Support Package
SAP_BASIS 816 SP 00
SAP_BASIS 758 SP 02

When you install an SAP Note, some prerequisites might be required. The prerequisites are SAP Notes that have already been released and that change the same context of the current one. Then, even if these prerequisites are not necessarily related to the symptom of this SAP Note, you must implement them due to technical reasons.

Solution

From now on, the scenario described in the "Symptom" section no longer occurs.

Delivery

As a general rule, SAP recommends that you install a solution by applying a Support Package.

However, if you need to install the solution earlier: Use the Note Assistant (SNOTE) to implement the correction instructions.

Learn more at SAP Note Assistant.

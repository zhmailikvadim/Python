Symptom

  • The business branch requested information on whether it is possible to activate MFA in a controlled manner to avoid the "big bang effect".
  • The "big bang effect" refers to all users being logged out and forced to register a phone number for MFA during their first login after the rule activation.

Environment

  • Customer Data Cloud
  • Privacy & Safety (Consent, RBA - Risk-Based Authentication)

Reproducing the Issue

  1. Activate an RBA rule concerning MFA.
  2. Observe that all users are logged out and required to register a phone number for MFA during their subsequent login.

Cause

Resolution

  1. Refer to the documentation on creating RBA rules.
  2. Note that creating an RBA rule does not force users to log out.
  3. Users will need to set a phone number or other MFA method during their next login after the rule activation.
  4. Define the scope for an RBA rule at the API Key level to exclude/include specific child sites from the rule.
  5. Refer to the documentation for configuring RBA rules.

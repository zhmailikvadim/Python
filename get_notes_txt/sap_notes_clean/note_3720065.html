Symptom

  • During customer data destruction using FI_ACCRECV_WRI, the message "Program error: Message Missing" is displayed in the spool.
  • The error occurs when attempting to destruct general customer data while SD documents still exist.
  • The message is misleading and does not explicitly indicate the presence of SD documents.

Environment

  • Sales and Distribution (SD)
  • SAP ERP
  • SAP ERP Central Component
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA 
  • SAP S/4HANA Cloud Private Edition

Reproducing the Issue

Execute report FI_ACCRECV_WRI.

Cause

  • The error message "Program error: Message Missing" is displayed because the system does not provide a specific message indicating the presence of SD documents when attempting to destruct general customer data (KNA1).
  • Archiving KNVV without SD validation is not recommended by SAP.

Resolution

  • First the sales area master data (KNVV) must be archived and afterwards the general data of the customer (KNA1).
  • While archiving the sales area master data the system returns the message about existing VBPA entries, what prevents the archiving of the customer.
  • It is not possible to get the message about existing VBPA entries while trying to archive KNA1 table (general data of the customer ) first, system returns the default message "Program error: Message missing ".

Symptom

Your scanning tools report a vulnerability in one or more components of SAP Landscape Management

Reason and Prerequisites

Customers use 3rd party security scan tools on their SAP NetWeaver Java servers and CVEs are reported for components or open source libraries used in the Software Components deployed

Please note, that according to SAP Netweaver Java compatibility matrix for JDK versions  https://me.sap.com/notes/0002768400 the latest JDK compatible is 8. That means any open source component used cannot be updated beyond its maximum version, supporting Java 8.

Solution

CVE description assessment fixed in SP (if applicable) 
CVE-2025-48913 If untrusted users are allowed to configure JMS for Apache CXF, previously they could use RMI or LDAP URLs, potentially leading to code execution capabilities. This interface is now restricted to reject those protocols, removing this possibility. Users are recommended to upgrade to versions 3.6.8, 4.0.9 or 4.1.3, which fix this issue. false positive  
  • CVE-2025-48913 - If untrusted users are allowed to configure JMS for Apache CXF, previously they could use RMI or LDAP URLs, potentially leading to code execution capabilities. This interface is now restricted to reject those protocols, removing this possibility. Users are recommended to upgrade to versions 3.6.8, 4.0.9 or 4.1.3, which fix this issue.
    • affects : cxf-core
    • assement: false positive
      cxf-core itself does not contain the classes, required for JMS configuration, these are contained in cxf-rt-transports-jms, which is not present in SAP Landscape Management. Therefore this vulnerability is not applicable and not exploitable.

Symptom

  • A new SAP Supply Network Collaboration system is installed, or a new Supplier is added to the Due Quantity Calculation Customizing (Determine Due Schedule Line Quantities).
  • No Due Quantity is calculated, therefore the Supplier is unable to create an ASN.
  • Commitment levels are used properly in customizing: Determine Due Schedule Line Quantities.
  • Due Quantity calculation changed after using Acknowledge function.

Environment

SAP Supply Network Collaboration

Reproducing the Issue

  1. Access Supply Network Collaboration Supplier Portal.
  2. Display a Scheduling Agreement Release item.
    1. Directly via Release > Release Details menu.
    2. Via Release > Release Worklist or via Release > Release Overview menu.
  3. Due quantity is zero for all schedule lines.

Cause

The usual problem is that the Default "Relevant ASNs" setting is used in Customizing "Determine Due Schedule Line Quantities". Although the option "(2) Selection Based on Customer Cumulative Quantity (Default)" is the default, it is working only in case Customer Cumulative Quantity is sent with the Release updates and the ASNs, which is not configured usually. Therefore, use the other option "(1) Selection Based on Time Stamps" and test whether this solves the problem.

Resolution

  1. In Transaction SPRO, enter customizing: Supply Network Collaboration > Delivery > Due Quantity Calculation > Determine Due Schedule Line Quantities.
  2. Create a new entry, or change existing entries where "Relevant ASNs" is "(2) Selection Based on Customer Cumulative Quantity (Default)".
  3. Set "Relevant ASNs" to "(1) Selection Based on Time Stamps".

If there is no change in Due Quantity, access the content listed in See Also for other documented problems.

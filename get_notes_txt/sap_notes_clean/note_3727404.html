Symptom

When doing client copy, it is possible to be failed due to too big table size, and composite OOM raised.
Error as following:
Allocation failure type: STATEMENT_MEMORY_LIMIT_FROM_GLOBAL_CONFIG 

Environment

SAP HANA datase

Cause

Single SQL memory usage has been restricted by parameter statement_memory_limit, and it is recommended to set it as following value as explained in KBA 2600030 - Parameter Recommendations in SAP HANA Environments:
MIN(10 % of allocation limit, 30 GB) - MIN(30 % of allocation limit, 500 GB) 

Resolution

  1. You can consider to stop other operations and set statement_memory_limit to 0 to finish the operation. However this may raise system level OOM. So, it is not recommended.
  2. Please refer to following KBAs to process from client copy tool to split big tables:
    2962116 - New Client Copy: Table Splitting Improvements
    3167350 - Meaning of Processing Options/settings for new client copy tools


Keywords

Out of memory, composite OOM, statement_memory_limit, SCC3

Environment

  • SAP NetWeaver
  • SAP S/4HANA 2023

Reproducing the Issue

  1. Schedule the execution of the following standard programs in background mode:
  • /UI5/APP_INDEX_CALCULATE
  • /UI5/UPD_ODATA_METADATA_CACHE
  • /UIF/CLEAN_LREP
  • you can use a system user type with no roles or profiles assigned.

     2. Observe that the programs execute successfully without performing any authorization checks.

Cause

This is an expected behavior.

Resolution

It's by design that the report does not check any authorizations when running.
When scheduling background jobs the user has already the authorization to operate as system administrator.
So, for the batch user itself no further authorization checks are needed.
Reports /UI5/APP_INDEX_CALCULATE & /UI5/UPD_ODATA_METADATA_CACHE are only a regenerative check and generation of cache used for UI5 apps.
Report /UIF/CLEAN_LREP removes expired application states to reduce used disk space.
These reports are intendent to run regularly on to ensure system performance.

Keywords

UI5, authorization check, background execution, foreground execution, standard programs, /UI5/APP_INDEX_CALCULATE, /UI5/UPD_ODATA_METADATA_CACHE, /UIF/CLEAN_LREP, S_SUISUPRT

Symptom

You delete master data of a characteristic in a large number of InfoProviders, some of which have large datasets. For some MUTs, the automatic usage check, which is executed in parallel in background processes, does not contain any information in the job log as to the InfoProvider for which the job generates the where-used list. Message S001 (RSMD_UC_MESSAGES) in the job log does not contain the name of the InfoProvider.

Other Terms

Missing mapping in method CL_RSMD_UT_META_DATA=>GET_COLUMN_FOR_MUT( ) for MUT = 'ADS3'; in BW4, the mapping for MUT = 'IOB2' and MUT = 'IOB3' is also missing

Reason and Prerequisites

There is insufficient information about the current usage check. When master data deletion terminates due to an inconsistent InfoProvider, it is difficult to determine which object caused the termination.

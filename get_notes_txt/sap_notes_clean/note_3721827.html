Symptom

  • In the Latest People Profile, date fields display in an inconsistent date format when using the Czech locale.
  • The day is shown with one digit if less than 10, while the month is always displayed with two digits (leading 0).
  • Example: "1.01.2026" is displayed instead of "1.1.2026" or "01.01.2026".
  • The issue is related to the UI design of SAP SuccessFactors Employee Central.

Environment

SAP SuccessFactors Employee Central

Reproducing the Issue

  1. Access the Latest People Profile using the Czech locale.
  2. Observe the date format in the profile fields.

Cause

Lower case "m" did not parse the variable for month properly in Java.

Resolution

  • SAP Engineering team are investigating the issue for a code fix. 
  • This KBA will be updated with a fix date once provided.

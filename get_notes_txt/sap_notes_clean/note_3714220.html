Symptom

After confirming a process order, the Activity and Suboperation fields are empty in the Monitor Process / Planned Orders (COOISPI) confirmation list, even though the process order confirmation was completed successfully.

Environment

SAP S/4HANA Cloud Public Edition

Cause

The process order was confirmed using a header-level confirmation method.

When confirmations are performed using Confirm Process Order (CORK), the system records a single confirmation for the entire order. Because this confirmation is not linked to a specific phase or operation, the system does not generate Activity and Suboperation information.

This behavior is standard system design and does not indicate a system error.

Resolution

To record and display Phase, Activity, and Suboperation details, confirmations must be executed at the phase level using:

  • Confirm Process Order Phase (COR6N) app

The corresponding Fiori app Confirm Process Order Phase

Header-level confirmations using Confirm Process Order (CORK) app only provide summarized confirmation data and do not support detailed phase-related reporting.

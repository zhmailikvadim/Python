Symptom

  • An SOW Worker has exceeded their cumulative Committed Spend
  • How can I increase it for the one worker?

Cause

Due to configuration Buyer and Supplier can change SOW Worker dates without a revision, editing the worker's dates without a work order revision has caused the Committed Spend not being recalculated since this is an in-line edit which does not go through approval.

Resolution

You can either

Option 1: Enable sub-config Recalculate Committed Spend when SOW Worker Dates are edited 

This will apply to all SOW Worker’s going forward so please note this will be a big change.

OR

Option 2: Perform work order revision with date change

Committed Spend is calculated based on number of days/ hours and rate. If the worker Start Date or End Date(s) are not changed, then the number of days will remain the same as previous revision(s). Hence, the Committed Spend will not be updated.

Symptom

************************************************************

WARNING: This is a CA-FL (Field Logistics) specific note. If you DON'T have

Field Logistics activated in your system, this note does not apply to you.

************************************************************

Currently, the  Return by Product application restricts users to selecting only one base plant at a time in the app settings. This limitation prevents users from processing returns for remote plants associated with different base plants simultaneously. Consequently, return initiation is hindered across both the Initiate Return by Product and Initiate Returns by Order applications.

Other Terms

CA-FL-RET, Field Logistics,Initiate Returns By Product, SAP S/4HANA OP 2023 FPS02, SAP S/4HANA OP 2023 FPS03.

Reason and Prerequisites

 Advance development

Solution

Proposed Solution: > This update removes the "Base Plant" restriction from the application settings, instead displaying all remote plants defined in the Field Logistics configuration. Users can now select multiple remote plants and save the app settings, which are then automatically applied to the plant filter within the Initiate Return by Product application.

Caution: > This introduces a change in application behaviour on existing users. Previously saved application settings may no longer be compatible. Users will likely need to reconfigure their settings by manually selecting their preferred remote plants upon their first login after the update.

To adopt the changes, kindly implement the correction instructions attached with the note.

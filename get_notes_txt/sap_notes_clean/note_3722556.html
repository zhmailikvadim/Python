Symptom

You are using the BAPI Function BAPI_PR_CREATE or BAPI_PR_CHANGE to update customer-specific field extensions. During the BAPI-runtime the fields are not mapped with BAPI_TE_MEREQITEM.

If manually filled, the fields are updated.

Environment

  • BAPIs - Purchase requisition

Cause

Formerly, the customer specific fields were mapped to the BAPI_TE_MEREQITEM. The former Business Add-In was triggered for the structure CI_EBANDB, and those structures were populated irrespective of whether the BAdI was implemented or no.

With the S/4 system releases, the same has been redesigned and Customer-specific fields are no longer visible in the structures BAPI_TE_MEREQITEM / BAPI_TE_MEREQITEMX. Those structures are obsolete. S4/HANA uses different mechanism using structure MMPUR_PR_BAPI_ITEM_EXT. 

Resolution

Please use the PR-Item BAdI MM_PUR_S4_PR_MODIFY_ITEM or the PR-Header BAdI MM_PUR_S4_PR_MODIFY_HEADER respectively. 

See Also

SAP Note 1111291 - Extension processing in BAPI_PR_CREATE / BAPI_PR_CHANGE

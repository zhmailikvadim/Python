Symptom

SNP PPM generated with missing activity/mode/component with SNP PPM generation with lot size report

Other Terms

/SAPAPO/PPM_CONV_PPDS2SNP_310

Reason and Prerequisites

SNP buckets, to which simulation order generated at program runtime using PPDS PPM is apportioned to, are thanks to note 1706189 created only starting from simulation order start date, 00:00:00 local time. It can happen though that simulation order is scheduled by LiveCache to start earlier, as LiveCache schedules orders in UTC time, only. This can happen when local timezone is UTC minus x(1-12) hours.

Solution

************************************************************************

Implement this note only if note 1706189 was implemented in your system before.

************************************************************************

This note reverts changes from note 1706189. Simulation orders are scheduled forwards from Start of Generation Period date (in selection screen of /SAPAPO/PPM_CONV_PPDS2SNP_310) in UTC time, as was the standard design for years. PPDS source of supply used for SNP PPM generation must exist in the system on 00:00:00 (hh:mm:ss) of Start of Generation Period date in UTC time.

Implement this note and regenerate affected SNP PPMs.

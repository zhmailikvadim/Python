Symptom

Input controls stop functioning correctly after a tooltip is displayed that contains a block from another report page in a Web Intelligence document.

Other Terms

webi, webintelligence, filter

Reason and Prerequisites

Environment:

  • SAP BusinessObjects Business Intelligence platform 2025 Patch 03
  • Web Intelligence 

Steps to Reproduce:

  1. Log in to BI Launchpad.
  2. Open a Web Intelligence document that contains:
    - Two Cross Table blocks located on different reports.
    - An advanced tooltip linking the two blocks.
    - A document-level Input Control based on a dimension used in the first report’s block.
  3. Select a value in the Input Control.
  4. Navigate to the report containing the second block.
  5. Hover over a line in the block
    1. >> The tooltip displays correctly.
  6. Navigate back to the original report containing the first block.
    1. >> The Input Control selection is no longer applied, even though the selected value is still shown as selected in the Input Control in the filter's toolbar.

Cause: 

  • This issue is caused by a defect in the product.

Solution

This issue is fixed in the patches listed in the "Support Packages & Patches" section below.
Due to different internal processes, the "Support Packages & Patches" section of the SAP Note will be populated as below:

  • for fixes delivered in "Support Packages": immediately (even if the "Support Package" is planned to be released several months later)
  • for fixes delivered in "Patches": ONLY once the "Patch" is released AND available on SMP (not before)

Symptom

  • In transaction CG02, using the search option "Substance with Values Assigned" together with "Nonempty Characteristics" returns no results although characteristics are populated.
  • The message displayed is: “There is no object in the database that fulfills the criteria.”
  • The behavior occurs across different Value Assignment Types.

Environment

  • SAP ERP
  • SAP S/4HANA
  • SAP S/4HANA Cloud Private Edition

Reproducing the Issue

  1. Start transaction CG02 (Specification Management).
  2. Open the search and select “Substance with Values Assigned”.
  3. Enable the “Nonempty Characteristics” option.
  4. Choose a Value Assignment Type for which characteristics are maintained.
  5. Execute the search and observe that no results are returned and the message “There is no object in the database that fulfills the criteria.” appears in a pop-up.

Cause

Class type 100 (ESTVA) was configured without the “Engineering change mgmt (time-related) for classification” flag, which is required for the “Nonempty Characteristics” search to work correctly.

Resolution

  1. In SPRO, navigate to Cross-Application Components -> Classification System -> Classes -> Define Object Types and Class Types.
  2. Open class type 100 (ESTVA) and enable the checkbox “Engineering change mgmt (time-related) for classification”.
  3. Save and transport the customizing change as required by your change process.
  4. Retest in CG02 using “Substance with Values Assigned” with “Nonempty Characteristics” to confirm that matching specifications are returned.

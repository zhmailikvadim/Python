Symptom

This is a more technical situation, so typically you might not face an issue on application or UI side at all. However, depending on the scenario, it may be of relevance that properties are propagated thoroughly, even when not actively used in current scenarios. In this case, when retrieving an instance of a model (CL_MDC_MODEL) where the given business object type (OTC) reflects a suited business object type code, the so called data habitat can be set as for all other situations. However, the suite model does not propagate this setting to its "intrinsic" single business object type code bases model instance, so when accessing them you might notice a difference between the habitat in which the suite is working and the one used for single BO type models.

Other Terms

MDG, Consolidation & Mass Processing; MDC; object type code; CL_MDC_MODEL_SUITE; if_mdc_data=>gc_habitat; native; domestic; wilderness

Reason and Prerequisites

Program error.

Solution

Apply the attached corrections to your systems.

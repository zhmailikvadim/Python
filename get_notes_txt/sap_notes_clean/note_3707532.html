Symptom

In the supplier request form, the field Department is not getting updated despite the enterprise user submitting the responses.

Reproducing the Issue

  1. Attempt to submit a supplier request form with the field Department filled.
  2. Observe that the field Department does not update, and the department value does not sync to the database.

Cause

The issue occurs due to the presence of a , character in the DepartmentID field. The system does not process the value containing the special character, which prevents synchronization of the department value in the submission API call.

Resolution

  1. Identify the department unique name in the metadata file.
  2. Remove the , character from the DepartmentID.
  3. Resubmit the supplier request form with the corrected DepartmentID.
  4. Verify that the field Department updates correctly and the department value syncs successfully in the /submitSupplierRequest API call.

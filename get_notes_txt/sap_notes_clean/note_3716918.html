Symptom

• After upgrading from S/4HANA 2020 to 2023, the Cancel Billing Document Fiori app and Create Billing Document Fiori app retain previous documents.
• Screens automatically jump to processing screens, impacting navigation.
• VF11 retains the last viewed document, preventing entry of a new document number until users navigate back and forth multiple times. 
• VF01 jumps directly to the processing screen, requiring users to click the back button to access the initial screen for entering delivery document numbers. 
• The issue occurs exclusively in the Production environment. 

Environment

  • Sales And Distribution (SD)
  • SAP Enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA
  • SAP S/4HANA Cloud Private Edition

Reproducing the Issue

1. Go to Manage Sales Order to view a sales order.
2. Navigate to VF01, where the document will populate and automatically jump to the processing screen without allowing entry of delivery documents.
3. Go to Cancel Billing Document, where the last viewed billing document will populate and automatically jump to the next screen without clearing the document number.

Cause

The behavior is related to changes introduced in S/4HANA 2023 and updated SAP Fiori Apps, which streamline the process but differ from prior versions.

Resolution

1. Clear SAP UI5 cache, browser cache, and perform frontend cache invalidation post-upgrade.
2. Inform users about updated behavior requiring back navigation in the apps.
3. Verify that no custom enhancements or modifications interfere with standard app behavior.
4. Apply all relevant SAP Notes for S/4HANA 2023 upgrade and billing document Fiori apps.
5. Refer to SAP Note 0003604446 for specific resolution steps related to VF01 and VF11 behavior.

Symptom

When you create a purchase order, several items are created. During pricing, rebate conditions are determined for several purchase order items.

In subsequent settlement, however, a statistical update of business volume data is performed only for the first purchase order item.

In the detailed statement for the rebate agreement, the system issues the following error message for the relevant purchase order items:

MN 565 - Updating for purchasing document &, item &, incomplete (delivery schedule)

 

This note contains additional corrections for SAP Note 2040794.

Other Terms

ME21, ME21N, volume-based rebate MEB8, detailed statement, komv, document condition, volume-based rebate MN565

Reason and Prerequisites

There is a program error.

Solution

Implement the source code changes in your system as described in the correction instructions, or import the current Support Package.

Other Terms

Webi, Web intelligence, IDT, unx, dfx, print, export

Reason and Prerequisites

Environment:

  • SAP BusinessObjects Business Intelligence Platform 4.3
  • Information Design Tool
  • Web Intelligence

Steps to reproduce:

  1. Launch Information Design Tool and create a local project.
  2. Add a relational connection, and create a data foundation.
  3. In data foundation, go to Parameters and List of Values.
  4. Create a List of Value object
  5. Create a Parameter object, assign the list of value and then check the option "Select only from list"
  6. Save and close the data foundation.
  7. Right click and export it to PDF format. 
  8. Select all items and perform the export.
    1. >>  "Select only from list" as False is always wrong.

  • This issue is caused by a defect in the product

    • for fixes delivered in "Patches": ONLY once the "Patch" is released AND available on SMP (not before)

These additional duties can become relevant for goods that already have another additional duty rate in place. This leads to the consequence that the duty calculation in SAP GTS needs to handle two or multiple additional duties on the line item.

This SAP Nnote provides an overview about the single parts of the solution how these circumstances have to be treated when submitting your declaration to the customs system US ACE.

Other Terms

Calculation, additional duty, additional duties, USA, chapter 99, additional tariff number, section 301, B7，IEEPA

Reason and Prerequisites

Missing functionality

Solution

Please follow the customizing guide first and then implement the listed SAP Notes in sequence to enable the feature. Please note that the customizing guide uses the example of requiring a second additional duty for your reference. If you need a third or more additional duties, please repeat the process you followed for the second one, except for step 2.

1. Define internal duty type for the second additional duty

1.1 Go to IMG path: Global Trade Services --> Customs Management --> Calculation of Customs Duty --> Define Type of Customs Duty

1.2 Select duty type "B105" and copy as 'Z105'

1.3 Save data

2. Define external duty type “USB013”

2.1 Go to IMG path: Global Trade Services --> Customs Management --> Customs Code Lists --> Maintain Customs Code Lists for Legal Regulation

2.2 Choose your legal regulation for US, e.g. ACE, and execute

2.3 Select Customs Code Lists as "Duty Definition" from the dropdown list and click "Maintain" button

2.4 Create new duty type "USB013" and description as "Additional Duty"

2.5 Save data

3. Map internal duty types "B105" and "Z105" to external duty type “USB013”

3.1 Go to IMG path: Global Trade Services --> Customs Management --> Customs Code Lists --> Maintain Customs Code Lists for Legal Regulation

3.2 Choose your legal regulation for US, e.g. ACE, and execute

3.3 Select Customs Code Lists as "Duty Mapping" from the dropdown list and click "Maintain" button

3.4 Create new mapping:

Ext. Duty Type       Duty Type

USB013                 B105

USB013                 Z105

3.5 Save data

4. Prepare Classification Master Data - Maintain Customs Duty Rates

  • If you are using GTS 11.0, please define the duty rate for the second tariff number for duty type "Z105" via T-code /SAPSLL/TLC_CUSB
  • If you are using GTS edition for HANA, please define the duty rate for the second tariff number for duty type "Z105" in the App "Manage Duty Rates" from launchpad.

5. Apply following notes in sequence

  1. 3578219 - US ACE: Enable the second additional tariff number for 'Additional Duty' calculation - DDIC part
  2. 3578218 - US ACE: Enable the second additional tariff number for 'Additional Duty' calculation

Symptom

In the (FAIR-based) process "Create CFS Notification" (/IBS/CAB_REPT_CREATE), a previously generated notification GUID is deleted if a tax ID is missing. As a result, in a subsequent run, no correction or reversal notification, but a new notification is created instead.

Reason and Prerequisites

In the "Execute Validation" business function (/IBS/CAB_REPT_VALIDATE), the reference to a previously created notification is deleted if an error occurs.

Solution

The implementation /IBS/CL_CAB_REPT_VALIDATE of the above BF is corrected.

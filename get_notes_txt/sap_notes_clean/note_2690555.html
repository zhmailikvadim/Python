Symptom

Installation of Support Packs for Vistex V/4 1709 Solutions (Vistex V/4 Solutions are IA, PAC, DMP and DMR) on S/4HANA On-Premise 1709 system.  Technical Add-on component names involved are V4CORE, V4IA, V4PAC, V4DMP, V4DMR.  The customer system may contain all the Add-ons or a subset of the Add-ons.

This SAP Note provides the support pack installation instructions for the following Vistex V/4 1709 solutions.

  • SAP Incentive Administration for S/4HANA by Vistex (IA) helps manage, track, and analyze your incentive programs
  • SAP Paybacks and Chargebacks for S/4HANA by Vistex (PAC) provides End-to-end management of agreements, contracts and pricing for all your claim processes – from initiation to validation, settlement and analysis
  • SAP Promotions and Agreements for S/4HANA by Vistex, a combination of both SAP Incentive Administration by Vistex (IA) and SAP Paybacks and Chargebacks by Vistex (PAC)
  • SAP Data Maintenance - Pricing for S/4HANA by Vistex (DMP) manages complex pricing challenges with ease and reliability – covering everything from agreement initiation to analytics for buy- and sell-side pricing
  • SAP Data Maintenance - Resources for S/4HANA by Vistex (DMR) streamlines the maintenance of material, customer, and vendor master data

2.  High level summary of installation steps

a)  Download the add-on Support Package files

b)  Unpack the downloaded add-on Support Packages onto SAP transport directory

c)  Download and install the latest "tp" and "R3trans" transport tools from SAP Service Marketplace (optional, but recommended by SAP)

d)  Apply the latest SPAM version available at SAP Service Marketplace (recommended by SAP)

e)  Load packages in SPAM:  from the initial menu of SPAM in client 000, follow the menu path "Support Package" -->  "Load Packages"  --> "From application server"

f) Verify settings in SPAM: from the initial menu of SPAM, follow the menu path "Extras" --> "Settings" and on the Import Tab, make sure the option "Never" is set for the "ABAP/Screen Generation" setting

g)  Install all add-ons V/4 1709 Support Packages in a single queue using SPAM transaction code in client 000

h)  Perform post-processing steps

*** NOTE ***  If this is a new V/4 1709 installation or an SAP migration which includes Vistex software, the support packages will be included via Maintenance Planner and therefore only the post-processing steps from this note need to be performed.




3.  General

a. Downloading add-on Support Packages

V4CORE and V4VUI are common add-ons required for all V/4 1709 solutions mentioned below, additionally you also require at least one of the following add-ons for the IA, PAC, DMR and DMP solutions depending on your license agreement.

The V4VUI add-on can be installed either on the S/4HANA 1709 OP back-end system for embedded scenario or on a separate SAP Gateway/Frontend server based on SAP NW 7.52 for central HUB Deployment scenario.  Install the V4VUI support package(s) on the same system where the Add-on V4VUI is installed.

Solution                                                            Add-on name

SAP Incentive Administration (IA)                        V4IA

SAP Paybacks and Chargebacks (PAC)                  V4PAC

SAP Data Maintenance Resources (DMR)               V4DMR

SAP Data Maintenance Pricing (DMP)                    V4DMP

The V/4 1709 Add-on installation packages are available at the software download web site

https://support.sap.com/en/my-support/software-downloads.html

Download the required Add-on support packages related to the V/4 1709 components already installed in the system (e.g. V4CORE, V4IA, V4PAC, V4DMP, V4DMR, V4VUI).  The support packs are available as .PAT files.

-Log on as user
<sid>adm  on UNIX
<SID>OFR  on AS/400
<SID>adm  on Windows

-Switch to the <DIR_EPS_ROOT> directory of your ERP system (usually
/usr/sap/trans/EPS). The <DIR_EPS_ROOT> directory is displayed after you execute the RSPFPAR report on DIR_EPS_ROOT.

-Switch to the higher-level <DIR_EPS_ROOT> directory.

-Unpack any CAR archive files using the SAPCAR statement.  For example:
SAPCAR -xvf /<CD_DIR>/DATA/K-60E01INIRMGLB.CAR.   on UNIX
SAPCAR -xvf /QOPT/<VOLID>/DATA/K-60E01INIRMGLB.CAR.   on AS/400
SAPCAR -xvf <CD_DRIVE>:\DATA\ K-60E01INIRMGLB.CAR.  on Windows

The support pack files with file extensions of .PAT should now be in the <DIR_EPS_ROOT>/in directory.


b. SPAM update
Import the latest SPAM update BEFORE DOWNLOADING and importing the add-on Support Package.


c. Support Package Import
The Support Packages are imported using transaction SPAM. Refer to the corresponding online documentation.  If the add-on component is already installed, follow the step below to apply the available support pack(s) using SPAM.  However, if this is a new installation or SAP migration, the support packages will be included via Maintenance Planner.

-Prior to applying Support Packages, please ensure that you set the option "Never" for the "ABAP/Screen Generation" in the "import queue" settings of SPAM tool in client 000.

-Install Support Packages for all V/4 1709 Add-on components in the same queue.

Any known issues that occur during installation are documented under Step 6. Problems importing the Add-on Support Packages.

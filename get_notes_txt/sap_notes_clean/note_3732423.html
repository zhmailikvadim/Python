Symptom

When attempting to import a SPM template, the system fails with errors indicating unresolved or invalid references to DForms that are not part of the template.
During the import process, the system incorrectly searches for external Form Templates and displays errors such as:
 
A Form Template with a base class name of TestDForm is required by this import, but cannot be found.
A Form Template with a base class name of SourcingRequirementForm is required by this import, but cannot be found.
 

Cause

Changing the template’s Ad hoc form creation limit from No limit to 0 while an ad hoc form draft existed on an inactive template caused form-related DocumentProcessSettings to be included in the export (allDocumentProcessSettings.xml). During import, the system then expects the referenced form template base class in the target environment, resulting in the missing form template errors.

Resolution

  1. In the source template (SPM), Click on the advanced settings
  2. Set the Ad hoc form creation limit back to No limit.
  3. Save the template to apply the change.
  4. Re-export the project template to generate a new ZIP package.
  5. Import the newly exported ZIP into the production environment and verify that the import completes without the missing form template errors.

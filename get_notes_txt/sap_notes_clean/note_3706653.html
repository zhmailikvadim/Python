Symptom

The Purchase Order Quantity field currently supports up to 3 decimal places, causing rounding discrepancies in pricing, valuation, and transactional accuracy.

Environment

SAP S/4HANA Cloud Public Edition

Reproducing the Issue

  1. Open App 'Manage Purchase orders', 'Create Purchase Order - advanced' or transaction ME21n.
  2. Enter in all required header master data and all required item data. 
  3. Attempt to input a value in quantity filed with 4 decimal places.
  4. Observe that the field only supports up to 3 decimal places, resulting in rounding discrepancies.

Cause

Currently has a limitation of supporting only 3 decimal places for the Purchase Order Quantity field.

Resolution

This functionality is currently unavailable in the standard system.

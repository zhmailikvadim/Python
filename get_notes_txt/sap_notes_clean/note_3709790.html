Symptom

MD01n (MRP Live) creates planned orders even if the procurement type has been set "F-external procurement".
By standard design, MD01n should only create purchase requisitions for the externally procured materials.

Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.

Environment

  • SAP ERP Production Planning (PP) Release Independent
  • SAP ERP Release Independent
  • SAP ERP Central Component Release Independent
  • SAP Enhancement package for SAP ERP Release Independent
  • SAP S/4HANA, on-premise Release Independent
  • SAP Fiori for SAP ERP Release Independent
  • SAP Fiori for SAP S/4HANA Release Independent
  • SAP S/4HANA Cloud Private Edition Release Independent

Reproducing the Issue

  1. Execute transaction MD01N.
  2. Check result.


Cause

There are some restrictions in MRP live, not all scenarios are supported by MD01n.
In such case, materials are automatically redirected to classic MRP in MRP Live.

As screenshot, the usage of coverage profile is not supported by MD01N, 
therefore the material is planned by classic MRP and pldord are generated. 
You may check transaction MD_MRP_FORCE_CLASSIC for planning details,
like ''PPH_MRP 154: MRP with coverage profile is planned using classic MRP.''

Resolution

Please pay attention to the restrictions in MRP live referring to note 0001914010.
You may consider to maintain the creation indicator for MRP group/plant accordingly in transaction OPPR/OPPQ.

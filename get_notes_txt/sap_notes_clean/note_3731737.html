Symptom

In transaction PA30, if a change to the work schedule only concerns the Work schedule rule, then an iDoc is not generated. The same happens if an entry in the work schedule is deleted and there is an automatic update of the existing work schedule. The attached fix addresses these cases, and iDocs are created after every change related to FSM.

Other Terms

FSM, E4C, Connector, PA30, Infotype 0007 Work schedule, /PACG/ECM_PERS_WSD, /PACG/ECM_OBJDELREQ

Reason and Prerequisites

Missing logic in /PACG/CL_IM_ECM_HRPADINF, method IF_EX_HRPAD00INFTY~IN_UPDATE.

Solution

Please, implement this correction instruction.

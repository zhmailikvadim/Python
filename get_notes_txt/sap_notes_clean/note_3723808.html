Symptom

  • In Solr Admin (Cloud -> Graph), all cores display as down
  • Commerce/Backoffice operations that depend on Solr fail. Full indexing attempts fail with errors similar to:
    • "Facet search has failed"
    • "Could not find a healthy node to handle the request." (from org.apache.solr.client.solrj.impl.CloudSolrClient)
    • "Re-trying request to collection(s) [...] after stale state error from server."
  • Solr server logs show core load failures on startup, for example:
    • "SolrCore failed to load on startup"
    • "Could not load conf for core <collection>shard1_replica_n1: Can't load schema null: Unknown fieldType 'text_spell<lang>' specified on field spellcheck_<lang>"

Environment

SAP Commerce Cloud

Reproducing the Issue

  1. Prepare the environment

    • Use an SAP Commerce Cloud project configured with SolrCloud (for example, Solr 9.x).
    • Ensure the project is operational before changes (cores green, indexing succeeds).

  2. Introduce an incomplete language configuration in schema.xml

    • Add fields that reference a new spell-check field type, for example:
      • autosuggest_<lang> with type="text_spell_<lang>"
      • spellcheck_<lang> with type="text_spell_<lang>"
    • Do not add the corresponding <fieldType name="text_spell_<lang>">...</fieldType> definition.
    • (Optional) Add fulltext_<lang> and  dynamicField/copyField entries that reference text_<lang> to broaden the mismatch.

  3. (Optional) Update Backoffice configuration

    • In Backoffice, navigate to System -> Search and Navigation -> Solr Facet Search Configuration -> Facet Search Configuration and add/enable the new language without fixing the missing field type in schema.xml.

  4. Deploy the modified Solr configuration

    • Update and upload the affected Solr config set to the cluster used by your collections.

  5. Execute a search-dependent operation

    • Trigger a full indexing job from Backoffice, or execute a storefront/backoffice search that hits the affected collection.

  6. Observe SolrCloud state

    • Open Solr Admin -> Cloud -> Graph and verify that the affected collection’s replicas/cores show as down (orange).

  7. Check application-side errors

    • In SAP Commerce Cloud logs observe errors such as:
      • "Facet search has failed"
      • "Could not find a healthy node to handle the request." (from org.apache.solr.client.solrj.impl.CloudSolrClient)
      • "Re-trying request to collection(s) [...] after stale state error from server."

  8. Confirm the root cause in Solr logs

    • Review Solr server logs for startup/core load failures, for example:
      • "SolrCore failed to load on startup"
      • "Could not load conf for core <collection>shard1_replica_n1: Can't load schema null: Unknown fieldType 'text_spell<lang>' specified on field spellcheck_<lang>"

Cause

A new language was partially introduced in Solr configuration. The schema defines fields such as autosuggest_<lang> and spellcheck_<lang> that reference a missing field type text_spell_<lang>. Because the required <fieldType name="text_spell_<lang>">...</fieldType> block is absent from schema.xml Solr cannot parse the schema and fails to load the affected cores. As a result, the cluster reports no active replicas and all Solr-dependent features fail.

This typically occurs when following a language-enablement guide but omitting one of the schema steps. A KBA that outlines all required steps is KBA 2334769 - How to add a language in Solr, which includes creating both the spell-check field type and the related fields/copyFields.

Resolution

Complete and validate the Solr configuration for the newly added language by ensuring all required elements are present and consistent across schema.xml and solrconfig.xml, and that Backoffice facet configuration aligns with these changes.

1. Update schema.xml for the new language. Ensure the following are defined (adapt the placeholder <lang> to the intended language code):

  • Define the spell-check field type:
    • Add a <fieldType name="text_spell_<lang>" class="solr.TextField" ...> with appropriate analyzers and filters for the language.
  • Define the core text field type:
    • Add <fieldType name="text_<lang>" class="solr.TextField" ...> with analyzers/filters suitable for full-text search in the language.
  • Add fields:
    • autosuggest_<lang> using text_spell_<lang>
    • text_spell_<lang> using text_spell_<lang>
    • fulltext_<lang> using text_<lang>
  • Add copyField rules:
    • autosuggest -> autosuggest_<lang>
    • spellcheck -> spellcheck_<lang>
    • *_text_<lang> -> fulltext_<lang>
  • Add dynamicField rules:
    • *_text_<lang> mapped to text_<lang>
    • *_text_<lang>_mv mapped to text_<lang> (multiValued)

2. Update solrconfig.xml for suggest and spellcheck. Ensure the configuration builds and uses dictionaries for the new language:

  • In the firstSearcher QuerySenderListener, add entries to build:
    • suggest dictionary with qt=/suggest, spellcheck.dictionary=<lang>, spellcheck.build=true
    • spellcheck dictionary with spellcheck.dictionary=<lang>, spellcheck.build=true
  • In the suggest component:
    • Set the dictionary name to <lang>
    • Set field=autosuggest_<lang>
    • Keep or choose an appropriate lookupImpl (e.g., TSTLookup or FSTLookup) per performance needs
  • In the spellcheck component:
    • Set the dictionary name to <lang>
    • Set field=spellcheck_<lang>
    • Ensure buildOnCommit=true and/or buildOnOptimize=true

3. Align Backoffice facet configuration. In Backoffice, navigate to System -> Search and Navigation -> Solr Facet Search Configuration -> Facet Search Configuration:

  • Confirm the language settings include the newly added language.
  • Ensure the facet configuration is consistent with the schema and Solr config entries for <lang>.

4. Indexing and validation

  • Trigger a full indexing run from Backoffice.
  • Confirm that Solr collections are healthy and that search, suggestions, and spellcheck operate correctly for the new language.

Notes

  • The error "Unknown fieldType 'text_spell_<lang>' specified on field spellcheck_<lang>" indicates the text_spell_<lang> field type is missing in schema.xml.
  • All three areas must be updated: schema.xml, solrconfig.xml, and Backoffice facet configuration.
  • Omitting any single step (for example, the text_spell_<lang> field type) can prevent cores from starting correctly and cause "no active replicas found."

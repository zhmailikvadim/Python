Symptom

  • Selecting Data source for a Seamless Planning model the Error message: "Error in OK handler: [object Object]" is raised
  • Network logs show: "Object ... was not found or you do not have permissions to read it."

Environment

  • SAP Analytics Cloud Seamless Planning
  • SAP Datasphere 

Reproducing the Issue

  1. In Datasphere Create fact view in space 'space 1' and expose it for consumption. 
  2. Share it with another space 'space 2'
  3. In sap analytics cloud, open or create a seamless planning model.
  4. Choose to connect external data using the live connection to sap datasphere.
  5. Select a datasphere view that has been shared into the  'space 2' space.
  6. Confirm the selection and observe the error "Error in OK handler: [object Object]".

Cause

Shared objects are not exposed by default for consumption.

Resolution

In target space ( 'space 2') need to model a space local view (proxy object) and then expose this object for consumption.

Symptom

  • CMS service fails to start on one cluster node.
  • CMS runs normally on the other node.
  • In CMC, CMS properties observe "CentralManagementServer is considered failed it has stopped 5 time(s) within 60 minute(s). Please restart this server."

Environment

  • SAP BusinessObjects BI Platform 4.x
  • Windows server OS

Reproducing the Issue

1. Create a new node in the SAP BusinessObjects using an existing CMS

2. Start the SIA on the newly added node

3. Observe that the CMS on this node fails to start

4. Check the CCM, the CMS will show as failed

5. Attempting to manually start the CMS also fails

Cause

The root cause was a service account mismatch between the two cluster nodes:

  • Node A: CMS was configured to run using domain service account.
  • Node B: CMS was configured using a local account

Resolution

1. Open the central configuration manager (CCM) on the node where the CMS is failed

2. Stop (server intelligent agent) SIA

3. Right click the SIA and select properties

4. Navigate to the "Log on as" section

5. Clear the system account option

6. Enter the same domain service account used on the working node

7. Enter the password

8. Click apply, and then ok to save the changes 

9. Start the SIA

Keywords

CMS, failed, cluster.

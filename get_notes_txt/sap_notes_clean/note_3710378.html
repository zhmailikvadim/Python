Symptom

Clarification is needed about how to delete data records from the staging tabled of the MDG-F data model 0G, which uses Flex Mode.

Environment

  • SAP Master Data Governance 7.0

Cause

Data has been created in MDG-F, which is intended to be deleted from the MDG-F UI and the staging tables of the data model 0G as well.

Resolution

MDG-F is using Flex Mode, which means that the inactive and active data are permanently stored in the staging tables of the data model 0G.

Deletion of active data via the MDG-F UI does not delete the active data from the staging tables, but it only sets the flag Obsolete in the active staging record reflecting the logical deletion in the UI. 

Active data cannot be deleted from the staging tables.

Inactive data from not activated Change requests can be deleted and archived from the staging tables as per the SAP Note 2574330.

Symptom

  • How to achieve Cascading in custom MDF objects?
  • How to set field criteria on custom Generic Objects fields?
  • How to limit the display of values in field of custom Generic Objects type based on the value selected on a different custom GO field?

Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental. 

Environment

  • SAP SuccessFactors HXM Suite
  • Metadata Framework (MDF)

Resolution

To limit the value on one Generic Object field, based on the value selected on another Generic Object field, we can make use of Field Criteria in the following Steps.

Example Scenario: From MDF object: cust_Test, selective values in field cust_SubA should be controlled by the value user selected in field cust_A. 

Step1: Create Valid When Association for the two custom MDF Object cust_A and cust_SubA. Refer below a one to many Valid When relationship as shown in the screenshot.

cust_A: value selected from this field cust_A controls what values can be selected in cust_SubA

cust_SubA: selective values are controlled & limited from the value selected in field cust_A

IMPORTANT NOTES：

Child object (Effective dating = FROM PARENT) can’t be used as a reference in another object and its impact to system. 

For both field reference and Valid when association, reference Objects must NOT be a Child Object. 

Step2：Create some sample data for the Object cust_A and also add some records for Object cust_SubA via the Manage Data page. Here, we take "one-one mapping" relationship as example:

Step3: Add the two custom MDF Objects fields in main object: cust_Test.

Below is a main custom MDF object: cust_Test, that has 2 fields of data type = Generic Object: cust_A and cust_SubA.

cust_A: value selected from this field cust_A controls what values can be selected in cust_SubA

cust_SubA: selective values are controlled & limited from the value selected in field cust_A

Step4: In main Object cust_Test, add the required field criteria which will limit the values displayed in cust_SubA based on the value selected in field Cust_A.

You have to add the Field Criteria on the field : cust_SubA, as shown in the below screenshot:

Destination Field Name: cust_A's field name configured in main Object: cust_Test -> cust_A

Source Field Name: taken from[Association Name in cust_SubA].[InternalCode Name in cust_A] = > cust_A_Association.mdfSystemInternalCode

Check Result:

Now if go to manage data > create record of main Object: cust_Test, value from cust_SubA successfully controlled by value selected from cust_A:

Symptom

You run the Payroll Driver (USA) [PC00_M10_CALC transaction | RPCALCU0 report] using the Regular Rate of Pay (FLSA Overtime Valuation) (RGRTE) payroll function and the FLSA Adjustment of Overtime Hours (FAOH) subapplication with the following setup:

  • You have multiple adjustments IDs configured in the Wage Type Contributions to FLSA Overtime Hours (V_T596J_US_FAOH) view to reevaluate the same wage type.
  • The FLSA: Configure WPBP Split rate (FLSAR) configuration option is set as Use higher rate (HIGHER).
  • At least one of the adjustment IDs in V_T596J_US_FAOH view calculated the Regular Rate of Pay (RROP) as zero.

In this scenario, the system incorrectly overstates the reevaluated wage type by the number of adjustment IDs that had the RROP calculated as zero.

Other Terms

PC00_M10_CALC; FLSA; RGRTE; FAOH

Reason and Prerequisites

Reason

This issue is a side effect caused by the solution delivered by SAP Note 3547219 - FLSA: WPBP rate increase configuration for RGRTE FAOH and NDB.

Prerequisites

The following Support Package level is the minimum prerequisite:

S4HCM 102 HR SP 00
S4HCM 101 HR SP 06

SAP_HR 6.08 HR SP D0

For more information, access the SAP ERP Human Capital Management - Support Package Schedules (SAP_HR) or SAP S/4HANA On Premise Human Capital Management (S4HCM).

When you install an SAP Note, some prerequisites might be required. The prerequisites are SAP Notes that have already been released and that change the same context of the current one. Solution

From now on, the system correctly handles the reevaluation with multiple adjustment IDs when the FLSAR configuration option is set to HIGHER.

Symptom

When using the Split Sample Quantity functionality in a decentral EWM system, try to post Goods Receipt Reversal in an inbound delivery. But there is an error "You can't reverse the goods receipt because of existing sample stock. Message no. /SCWM/QINSP 179" and the Reversal is aborted. The delivery remains unchanged.

Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.

Environment

  • SAP S/4HANA

Reproducing the Issue

Scenario: 
1. When the inspection planning is triggered, an Inspection document and inspection lot pair gets created, and sample split quantity is calculated. Posting Change WT is created with Confirmed status to split the sample quantity from the stock. The split quantity receives Inspection ID Type 'S', while the remaining quantity receives 'A'.
2. Try to post Goods Receipt Reversal, but the error "You can't reverse the goods receipt because of existing sample stock. Message no. /SCWM/QINSP 179" is shown.

Cause

If the Split External Sample Quantity functionality activated in the warehouse, and at the time of inspection planning the stock split for the external calculated sample size was made with the Posting Change, the split needs to be cancelled first, before posting the Goods Receipt Reversal.

Resolution

In Warehouse Monitor (transaction /SCWM/MON) -> Inbound -> Documents -> Inbound Delivery search with the EWM inbound delivery number.
1. Post a reversal for the External Sample Split before posting Goods Receipt Reversal. To do this, on the Inbound Delivery Item level select More Methods button > Cancel Split for Ext. Sample.

2. See received message "Posting change reversed".

3. A new Posting Change warehouse task gets created, which posts the sample stock back to the remaining stock, and changes Inspection ID Type 'S' back to 'A' for the split stock.

4. Post Goods Receipt Reversal without any errors.

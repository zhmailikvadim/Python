Symptom

You are running the SUM (Software Update Manager). It fails early during the Checks roadmap with an error like:

  • =========== Pool Table Consistency Check ===========
    Errors in pool tables detected.
    Before you continue, fix the issues below:
    Pool 'ATAB': Calculated maximum key len to 202, have 100.
    Summary: Found 1 tables with erroneous rows!

Note that the pool table in error could be different (ATAB, DVPOOLTEXT, GLSP, etc.)

Reproducing the Issue

Run SUM upgrade

Cause

There is at least one key field in the pool table that is not adhering to the maximum key length restriction set in dictionary

Resolution

Run this query on the database layer:

  • If the pool table in question is the ATAB table, then run the query as is. If it is another pool table, then please change the table in REFNAME field accordingly:
    • select TABNAME, KEYLENGTH from DDNTT where REFNAME = 'ATAB' order by KEYLENGTH Desc

The above output will show you exactly which table in the pool table is not following the maximum key length restriction. In the ATAB example above, there is at least one entry that has a key length > 100. With this information you can proceed with answering these questions with your team:

  • Who owns this table?
  • Reach out to the owners of this table to determine why its key length does not adhere to the pool table's restriction. The handling of this issue needs to be processed by the owners of the table.

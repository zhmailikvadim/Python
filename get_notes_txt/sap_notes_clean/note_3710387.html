Symptom

  • I receive the following errors when approving a paper invoice in SAP Ariba Buying and Invoicing:
    • The approval flow cannot be displayed because the Ariba system found the following errors or missing fields.
    • Line item [Number] ( Value Added Tax - line level ). Accounting. Accounting [#Line Item Number]: Percentage must be positive.
    • Line item [Number] ( Value Added Tax - line level ). Accounting. Accounting [#Line Item Number]: Quantity must be positive.
  •  The invoice does not proceed in the approval flow.

Environment

Invoice Approval Flow in SAP Ariba Buying & Invoicing

Cause

Tax accounting on some line item of paper invoices have negative values in quantity and/or percentage. 

These values violate the validation/integration rules applied to the document and, consequently, trigger the exception that stops approval.

Resolution

  1. Edit line item to remove negative values. The quantity and percentage of tax accounting needs to be corrected according to valid configuration.
  2. If editing is not possible, Reject the invoice and resubmit it after rewriting line item with the correct values.

Keywords

invoice approval error, approval flow exception, invoice resubmission

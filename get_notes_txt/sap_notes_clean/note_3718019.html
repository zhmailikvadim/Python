Symptom

  • Seeking advice regarding purchase order items with multiple cost center assignments across different company codes for purchase order accruals.

Environment

  • SAP S/4HANA Finance
  • SAP S/4HANA
  • Financial Accounting (FI)

Reproducing the Issue

  1. Access transaction ME23N.
  2. Review purchase order.
  3. Observe that the purchase order item is under company code AAAA, but one of the cost center assignments is under a different company code (e.g., BBBB).
  4. In POACTREE03 all the sub accrual item are in the same company code (AAAA)

Cause

  • Cross-company-code purchasing with account assignment objects from different company codes is currently not supported in standard SAP POAC functionality (SAP note 2812479). 
  • The system enforces company code consistency checks between the plant and purchasing organization and the account assignment objects e.g., cost centers or WBS elements.
  • Manual accrual has an Account Assignment feature that can assign offsetting company codes (SAP note 2885625), but not for Purchase Order Accruals.

Resolution

There are two possible ways of solution:

  • Create the PO with a cost center from the same company code and use subsequent intercompany billing for cost allocation.
    OR
  • Implement the BAdI "Change Additional Account Assignments" and set the OFFSTNG_BUKRS field based on the settings in the CSKS table.

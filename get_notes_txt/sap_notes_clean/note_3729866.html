Symptom

Most SAP icons are defined in table ICON using a four character code which is linked to a global constant, e.g. ICON_EXECUTE_OBJECT = "@15@". However, occasionally, some applications use icons with longer codes, e.g. "@B_MATR@" (Investment stock) and since release 758 these are not displayed when assigned to a user defined toolbar button in ALV Grid.

Other Terms

ALV; ABAP List Viewer; SAP List Viewer; Grid; List; Icon; Toolbar

Reason and Prerequisites

Reason

In 758, unsupported access to ALV Grid toolbar events using the ALL INSTANCES addition was removed, see SAP note 3464189. To provide an alternative for developers, new toolbar handling was introduced to provide a API with which all button types may be added to the ALV Grid toolbar. Unfortunately, this handling assumed use of short four character icon codes; as a result the longer codes were ignored.

Prerequisites

The following Support Package level is the minimum prerequisite:

Release Support Package
SAP_BASIS 816 SP 00
SAP_BASIS 758 SP 02

When you install an SAP Note, some prerequisites might be required. The prerequisites are SAP Notes that have already been released and that change the same context of the current one. Then, even if these prerequisites are not necessarily related to the symptom of this SAP Note, you must implement them due to technical reasons.

Solution

1. Internal toolbar handling allows both long and short icon codes to be used so that long icon codes are no longer ignored.

2. The toolbar API methods provide an additonal parameter ICON_LONG which handles both long and short icon codes. This is consistent with the other 'Long' parameters: TEXT_LONG and TOOLTIP_LONG. Parameter ICON used for short icon codes remains available for compatibilty with existing programs, and allows icon constants e.g. ICON_EXECUTE_OBJECT to be used without conversion.

3. The test program SALV_OM_TEST_TABLE_TOOLBAR is modified to allow testing of long icon codes.

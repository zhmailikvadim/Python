Symptom

You are using SAP Service and Asset Manager (SSAM) application with SAP Field Service Management (FSM) integration and require SSAM users to be assigned to different FSM companies

Other Terms

SAP Service and Asset Manager, SSAM, FSM, SAP Field Service Management, MAIF, Mobile Application Integration Framework

Reason and Prerequisites

New Feature - Steps to implement user based FSM company assignment

Solution

This consulting note lists the pre-requisistes and steps to configure User based FSM company assignment when integrating SSAM with FSM.

Until now, SSAM supported the definition of a single FSM company when integrating with SAP FSM. The FSM company name and ID are defined in the MAIF Admistration and Monitoring portal server configuration properties and this configuration is then applicable to the entire application and therefore all the users. With the application of this note, the default FSM company configured in the Administration and Monitoring server properties can be overriden by user specific FSM company assignment using User Parameters. Note that a user can one be assigned to a single FSM company.

Some additional filters have also been delivered for the Customer, Material and S/4HANA Business Partner oMDOs as part of this functionality to facilitate a closer mapping between the SSAM data download filters and the SAP backend and FSM integration. These new filters are not specific to FSM integration. They can be configured just like any other existing filters and therefore not explained in this note.

Pre-requisites for configuring User Based FSM Company assignment:

  1. Install the following notes:
    1. 3715747 - Add Company Code filter to the CUSTOMER oMDO
    2. 3716424 - New filters added to the S/4HANA Service Business Partner oMDO
    3. 3717973 - New filter added to the Material oMDO
    4. 3697555 - PM/CS - Update the Object Link table with FSM Account and Company ID
    5. 3701835 - Update the Object Link table with FSM Account and Company ID for the S/4HANA Service Orders
    6. 3706819 - Support User based company assignment for FSM integration for PM/CS Orders
    7. 3710227 - Support User based company assignment for FSM integration for S/4HANA Service
    8. 3706683 - Support User based company assignment for external worker User creation
    9. 3705676 - Update the USER INFO classes to support FSM Multiple company integration
  2. Ensure that all the steps specified in the documentation for configuring the SSAM to FSM configuration are completed depending on the persona and order types. Note that the default FSM Company Name and ID must still be configured in the Administration and Monitoring server properties.

Configuring User Based FSM Company assignment:

If you are using Plant Maintenance (PM) or Customer Service (CS) orders, define the FSM company name in the /MFND/FSM_COMP_NAME User parameter.

If you are using S/4HANA Service Orders, define the FSM company name in the /MFND/FSM_CMP_NM_S4 User parameter.

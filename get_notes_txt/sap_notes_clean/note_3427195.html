Symptom

Background:

In the transaction tcode ACACTREE02, an accrual object is deactivated with a date in the future period but the expected premature finished posting is not triggerred.

Solution:

The original logic is that the premature finished posting will not be triggerred if the period of the premature finished date is in future. Now the logic is modified that the premature finished posting will be triggerred if the posting period of the premature finished date is in open status.

Other Terms

ACACTREE02, Accrual Engine, Premature finished posting

Reason and Prerequisites

Program Error

Solution

Apply the note or upgrade to the corresponding support package.

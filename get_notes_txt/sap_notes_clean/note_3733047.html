Symptom

QR Code size defined in the Adobe Form template does not reflect in the ZPL output when the document is printed. This is a known issue described in SAP Note 1923734, which can be solved by altering the XDC file. However, as the system is on Public Cloud it is not possible to modify the XDC file via RSPO0022.

Environment

SAP S/4HANA Cloud Public Edition

Reproducing the Issue

  1. Design a form template with a QR code field and generate the output via SAP Forms Service by Adobe.
  2. Print to a Zebra printer through SAP Cloud Print Manager with a Print Queue defined with the ZPL type.
  3. Observe that the printed QR code is significantly smaller/larger.
  4. Attempting to adjust the QR code sizing in the XDC by accessing report RSPO0022 is not possible in the Public Cloud.

Cause

Adobe Designer template (XDP) does not provide a mechanism to modify the QR Code size in ZPL output.

Resolution

Switch the Print Queue in use to a PDF Print Queue. With this setting, the PDFPRINT integrated in CPM would convert the PDF to ZPL; the printed barcode/QR sizing should match the PDF.

See Also

SAP Note 1923734 - QR Code Sizing in ZPL Output of Adobe Document Services

Symptom

Upon trying to login to BTP Cockpit using the Platform Identity Provider, the following error is thrown:

'Invalid IAS tenant URL
The requested IdP could not be used for login. Possible reasons:

  • IdP provided in the URL is not known. Please check the spelling is correct.
  • The provided URL contains multiple

Environment

  • SAP Business Technology Platform
  • SAP Cloud Identity Services

Reproducing the Issue

  1. Attempt to log into the BTP Cockpit using the Platform Identity Provider.
  2. Observe the error message: 'Invalid IAS tenant URL. The requested IdP could not be used for login. Possible reasons: IdP provided in the URL is not known. Please check the spelling is correct. The provided URL contains multiple identity providers. Please correct the URL so that it contains only one identity provider.'

Cause

  • Wrong URL used.
  • Cloud Identity Service included in the URL does not exist.

Resolution

    Enter the following URL to authenticate using the Custom Identity Provider for Platform Users: https://cockpit.btp.cloud.sap/cockpit/?idp=<CloudIdentityService_TenantID>.accounts.ondemand.com

    If the error still happens, validate the prerequisites described at `Establish Trust and Federation of Custom Identity Providers for Platform Users` were met successfully.

    See Also

    • Refer to: `Establish Trust and Federation of Custom Identity Providers for Platform Users`

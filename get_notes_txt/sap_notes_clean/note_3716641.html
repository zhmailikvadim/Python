Symptom

  • This article explains how to use the "Save Unit for Analysis" feature in SAP IBP to remove error-causing units from queue processing and allow subsequent units to proceed.
  • The "Save Unit for Analysis" feature allows users to remove the top-most erroneous unit from a processing queue. By doing so, subsequent units are allowed to execute without being blocked by the error. The saved (erroneous) units are moved to a separate, system-created queue for later analysis and resolution.
  • Caution:
    We recommend that you use this feature with caution and process only the units that are independent from the one with errors, for example, units for sales orders with multiple line items.

Environment

SAP Integrated Business Planning for Supply Chain Management (SCM-IBP)

Resolution

Scenario Example

Suppose you have a queue, e.g. SAP_SCENARIO_1, where the top-most unit is in error status and is blocking three subsequent processable units.

Steps to Save Erroneous Unit:

  1. Identify the Blocked Queue:

    • Open the queue monitor and locate the queue where processing is stalled due to an error on the top unit (e.g., SAP_SCENARIO_1).

  2. Initiate 'Save Unit':

    • Select the top-most erroneous unit.
    • Click the Save Unit for Analysis button.

  3. Result:

    • The erroneous unit is moved to a new, internally named queue, SAVED_{UNIT_ID}.
    • In the monitor, this queue displays the original queue name (e.g., SAP_SCENARIO_1) at the bottom of the list, ordered by the time it was saved.
    • The previously blocked, processable units are now able to execute.

  4. Monitor Saved Queues:

    • Saved units are visible at the bottom of the monitor.

  5. Handling Dependencies:

    • If an erroneous unit is found in multiple queues, saving it will place it in a single saved queue and unblock all associated queues, unless there are additional errors.

  6. Multiple Saves:

    • Re-saving the same erroneous unit from the same queue creates a new SAVED_{UNIT_ID} each time, resulting in multiple saved queues with the same original name.

Additional Information / Notes

  • Un-save/Return Limitation:
    • There is no feature to return a saved unit to its original queue ("unsave" or "re-park"). To process it, resolve the underlying error and restart the unit manually, either at the unit or queue level.
  • Naming Convention:
    • Internally, saved unit queues are labeled SAVED_{UNIT_ID}, but the monitor shows them under the corresponding original queue name.

FAQ

Q: What happens after I save an erroneous unit?
A: The unit is placed in a dedicated saved queue, and processing of the remaining units continues.

Q: Can I move a saved unit back to its original queue?
A: No. After fixing the error, manually restart the unit.

Q: What if the erroneous unit was blocking multiple queues?
A: All affected queues will be unblocked after the unit is saved.

Q: Why do I see multiple saved queues with the same name?
A: Saving different erroneous units from the same queue creates a new saved queue for each unit.

Symptom

  • When opening a custom card, it opens at the end of the card, requiring the user to scroll up to see the start.
  • When opening a custom card, it automatically opens at the location of the first URL in the content.

Environment

SAP SuccessFactors HCM Suite

Reproducing the Issue

  1. Create a custom card that includes a link positioned at the end of the content
  2. Open the custom card on the home page
  3. Observe that the card opens at the bottom of the content, requiring the user to scroll up to see the top.  

Cause

  • When a dialog opens, focus moves to the first appropriate focusable element.
  • If that element isn’t visible, the dialog content scrolls to bring it into view. 

Resolution

  •  By design, the card will open where the first focused element is located (e.g., at the bottom if the URL is placed at the end of the content).  
  • This aligns with established accessibility guidance for dialogs and supports keyboard and assistive technology users.
  • Changing this behaviour would likely conflict with our accessibility requirements.

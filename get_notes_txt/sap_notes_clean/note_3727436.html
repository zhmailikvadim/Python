Symptom

Symptom

You are attempting to use an SQL on Files adapter — either "File" or "Deltasharing" — in a Smart Data Integration (SDI) replication task (.hdbreptask).

However, when deploying the .hdbreptask via Business Application Studio (BAS), the process fails with an unspecified generic error.

Based on the Business Application Studio logs, the following or a similar error message can be observed:

"text": "{\"error\":{\"errorCode\":\"EXECUTION_ERROR\",\"message\":{\"value\":\"An execution error occurred: internal error: \\\"<HDI_USER>\\\".\\\"(DO statement)\\\": line 7 col 9 (at pos 162): not supported for 'deltasharing' adapter\",\"lang\":\"en-US\",\"requestId\":\"XXXXXXXX-XXXX-XXXX-XXXX-XXXXXXXXXXXX\"},\"args\":[\"internal error: \\\"<HDI_USER>\\\".\\\"(DO statement)\\\": line 7 col 9 (at pos 162): not supported for 'deltasharing' adapter\"],\"errorRef\":\"XXXXXXXXXXXXXXXXXXXX\",\"innererror\":{\"code\":403,\"sqlState\":\"HY000\"}}}"

Other Terms

SQL on Files; deltasharing adapter; file adapter; SDI; Replication Task; .hdbreptask; SDA; Federation; HANA Cloud; BAS; Business Application Studio; HDI; flowgraph; .hdbflowgraph

Reason and Prerequisites

The .hdbreptask artifact is primarily a component of Smart Data Integration (SDI). While certain Smart Data Access (SDA) based adapters are supported to some extent — due to partially shared APIs within SAP HANA — SQL on Files is not part of either SDI or SDA. Instead, SQL on Files is documented separately with its own dedicated documentation and is treated as a distinct, independent functionality within SAP HANA.

Consequently, SQL on Files adapters are neither intended nor designed for use with SDI replication tasks (.hdbreptask) or flowgraphs (.hdbflowgraph) and are therefore not supported in this scenario.

Affected Releases:

  • SAP HANA Cloud Edition:
    • Revisions >= 2024.28.0 (2024 QRC3)

Prerequisites:

  1. Deploying a replication task (.hdbreptask) or a flowgraph (.hdbflowgraph) and
  2. using a SQL on Files adapter.

Solution

Workaround:

Instead of using a replication task, we recommend configuring the natively supported Delta Table Replication as detailed in the official documentation and blog post referenced below:

  1. Configuring Delta Table Replication
  2. Accelerating Your Analytics with Delta Table Replication into SAP HANA Cloud

Remarks:

  • As this is an unsupported scenario, we recommend submitting an improvement request as outlined in the following SAP Note 2265872.

Symptom

  • CIF transfer of Purchase Orders fails with SYSFAIL error: SQL error 600 occurred when executing Native SQL.
  • In ST22 there is a short dump DBIF_DSQL2_SQL_ERROR (CX_SY_NATIVE_SQL_ERROR)
    The error 600 occurred in the current database connection "LCA".
    Error Text of Database: "Work rolled back: OmsObjectNotFound: DbpError -28" 

  • In ST22 there is a short dump DBIF_DSQL2_SQL_ERROR (CX_SY_NATIVE_SQL_ERROR)
    SQL error 423 occurred while executing Native SQL.
    17843 in APS_ORDER_CHANGE# liveCache ERROR 17843 in APS_ORDER_CHANGE

Environment

  • SAP ERP and SCM system
  • Release independent

Reproducing the Issue

  1. Transfer a Purchase Order to APO.
  2. Check APO Product view. PO is missing or not updated.
  3. Check SMQ1/SMQ2 for SYSFAIL queue and/or ST22 for runtime error.

Cause

  • Usually the error happens, because one PO item schedule line was changed more than 256 times within the same transaction. For instance this could happen, if there are created one or more deliveries for affected PO with a lot of batch split items.
  • APO liveCache doesn't support it and result is the database error. At first LC returns an error -28002 (invalid OID) reporting the overflow. Afterwards an error -28814 (object not found) is raised. Furthermore it might happen, that in ST22 error messages from LC are not correctly displayed. You will see database error '-28' only.
  • The liveCache error code 17843 indicates that the purchase document to be transferred is extremely large, which is why the liveCache can't process it (due to a given immutable technical restriction). Too many entires in IT_OUTPUTS or IT_SCHEDULE or IT_ACTIVITIES.

Resolution

  • Avoid more than 256 changes of the same PO item schedule line within the same transaction (Batch split is only typical example).
  • If the error happens, then delete invalid queue and solve inconsistencies by deltareport.
  • Unfortunately there is no technical solution available due to unwanted side effects. It has been verified already by development support.

Keywords

liveCache error 28, 28002, 28814, 1028002, 1028814

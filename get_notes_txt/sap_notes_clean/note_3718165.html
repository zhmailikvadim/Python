Symptom

There are several issues with delegations’ apps:

  • Date intervals are not shown correctly
  • It is not possible to change / delete existing delegation
  • There is an error message “User 'xxx' does not have sufficient authorization (object S_WF_SUBST)”
  • Menu item for delegation apps is not shown on NWBC menu

Reason and Prerequisites

There was a change in the basis API, where the data entries now needs to be stored in UTC time zone. Basis API should be used instead of direct access to the DB table.

Solution

Implement corresponding correction instructions.

IMPORTANT!!! Once the note is implemented, you need to review/enhance the role(s) giving the access to the “Central Delegation” and “My delegation” applications. Originally, it was enough to assign the role with authorization object 'GRFN_USER' activity '58'(My Delegation) or 'GRFN_USER' activity '59'(Central Delegation).

Now, the additional authorization needs to be added into the corresponding roles.

My Delegation:

  • 'S_WF_SUBST' activity '03'

Central Delegation:

  • 'S_WF_SUBST' activity '02'
  • 'S_WF_SUBST' activity '03'
  • 'S_WF_SUBST' activity '06'

We have adjusted the standard power user role SAP_GRC_FN_ALL, so if you are using it, please add above mentioned authorized into it. In case you are using its copy, please adjust its copy accordingly.

Symptom

You want to display the diagnosis history for the patient in the diagnosis overview. If you expand the diagnosis history, the system does not display all diagnoses of the patient. Diagnoses of cases from other institutions are missing.

Other Terms

NP61, diagnosis overview, diagnosis history, cross-institution, client-wide, TN00-MANID

Reason and Prerequisites

This is caused by a program error. Due to the correction from SAP Note 3674861, the system incorrectly no longer determines diagnoses from other institutions.

The error described above occurs under the following prerequisites:

  • SAP Note 3674861 IS-H: NP61 - Call takes an extremely long time is implemented in your system.
  • You have activated the client-wide (cross-institution) identification of patients in the relevant client. You make this setting in the Implementation Guide (IMG) activity Maintain Client-Specific Basic Control (SAP Healthcare - Industry-Specific Components for Hospitals -> Basic Settings -> System Parameters -> Maintain Client-Specific Basic Control -> Set Client-Specific Control Parameters).
  • The patient has cases with diagnoses in other institutions.

Solution

Implement the attached correction instructions or import the specified support package.

After you implement the corrections, the system determines diagnoses from all institutions for the diagnosis history again if client-wide identification is active.

The system still only displays the diagnoses in the diagnosis history for which the user has authorization. If the treatment authorization is active in your system, the system also takes into account whether a treatment authorization has been issued for the case. In the diagnosis history, the system only displays diagnoses of cases for which a treatment authorization has been issued.

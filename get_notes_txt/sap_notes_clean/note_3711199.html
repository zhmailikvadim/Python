Symptom

Prerequisites in the system:

  1. It contains a policy with a claim SCHADEN_EINS with a claim header SCHADENKOPF_EINS with a corresponding claim assessment.
  2. The loss SCHADEN_EINS also changes the claim header to SCHADENKOPF_ZWEI with the corresponding claim assessment.
  3. The loss SCHADEN_EINS again switches the loss header back to SCHADENKOPF_EINS. 

When you process the claim from step c), the system creates a technically incorrect version of the claim assessment. When you navigate through this policy, the system issues the error message "Key does not exist". When you check this policy in /MSG/VR_EXPLORER, the system also issues the message "Multiple open versions exist" and "Change category C not allowed".

Solution

History:
February 9, 2026: New　


Troubleshooting Description:
The system does not create any incorrect technical versions if an existing claim header with claim assessment is used.


Technical effects:
If an existing claim header is used with a claim assessment (as described above a) to c), the existing claim assessment of the newly assigned claim header (SCHADENKOPF_EINS from the above example) is merged with the claim assessment of the previous claim header (SCHADENKOPF_ZWEI). As a result, the claim assessment is reopened and the "Assessment Required" indicator is set. If an automatic assessment is performed (depending on the system settings), this process is also run again and, if the assessment is successful, an automatic final claim movement is generated and processed.

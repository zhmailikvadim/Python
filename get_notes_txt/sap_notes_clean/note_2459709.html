Symptom

You are working in a database with the Intrastat functionality enabled. When you run the Intrastat Declaration Wizard for Reporting Period From 2017-02-01 To 2017-02-28, the application in the background does not cover the whole previous month; it covers the date range from 2017-01-01 to 2017-01-28, for Goods Receipt PO (GRPO) and Delivery.

Reproducing the issue

Prerequisite:

Intrastat functionality is enabled in the database (In Administration → System Initialization → Company Details → Basic Initialization tab, the Enable Intrastat checkbox is selected).

Scenario:

  1. Run the Intrastat Declaration Wizard (Financials → Financial Reports → Intrastat → Intrastat Declaration Wizard).
  2. In step 1 of the Intrastat Declaration Wizard, choose the Create New Declaration option.
  3. In step 2 of the Intrastat Declaration Wizard, choose the New Declaration value in the Declaration Type drop-down box. Choose the Reporting Period From 2017-02-01 To 2017-02-28.
  4. Proceed with the Intrastat Declaration Wizard.

Expected behavior:

When executing the wizard, the application-applied date range should cover the whole previous month, from 2017-01-01 to 2017-01-31, for GRPO and Delivery.

Actual behavior:

When executing the wizard, the application applies the date range from 2017-01-01 to 2017-01-28 for GRPO and Delivery.

Solution

SAP intends to provide a patch or patches in order to solve the problem described. The section Reference to Related Notes below will list the specific patches once they become available.

The corresponding Info file of the patches in SAP Service Marketplace will also show the SAP Note number. Be aware that these references can only be set at patch release date.

SAP will deliver patches only for selected releases at its own discretion, based on the business impact and the complexity of the implementation.

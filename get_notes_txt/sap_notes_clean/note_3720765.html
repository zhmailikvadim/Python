Symptom

  • Connection fails when attempting to connect to SFTP server using SSH keys based on the ssh-ed25519 algorithm
  • Error: com.jcraft.jsch.JSchAlgoNegoFailException: Algorithm negotiation fail: algorithmName="server_host_key" jschProposal="ecdsa-sha2-nistp256,ecdsa-sha2-nistp384,ecdsa-sha2-nistp521,rsa-sha2-512,rsa-sha2-256,ssh-rsa,ssh-dss" serverProposal="ssh-ed25519" 

Environment

  • Cloud Integration
  • SAP Integration Suite

Reproducing the Issue

  1. Log in to the SAP Cloud Integration (SCI) tenant.
  2. Navigate to the monitoring section and select "Connectivity Test".
  3. Click "SSH".
  4. Enter the SFTP server hostname and port (e.g., port 22).
  5. Select "None" and check "Enable support for Deprecated Algorithms".
  6. Click "Send".
  7. Observe the error message: com.jcraft.JSchAlgoNegoFailException: Algorithm negotiation fail.

Cause

The algorithm ssh-ed25519 is not currently supported by the SAP Cloud Integration SFTP adapter.

Resolution

  1. SSH keys using the ssh-ed25519 algorithm are not supported at this time. While support for this is part of our future plan.
  2. Cloud Integration SFTP adapter currently supports only the algorithms listed in SAP note: 3079510
  3. Recommend that the SFTP server administrator enable one of the supported algorithms listed in the note to ensure successful connectivity.

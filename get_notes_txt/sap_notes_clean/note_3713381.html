Symptom

In Verify Journal Entries in General Ledger - For Processor (Inbox) - F2728 the entry is not resent for verification in case rejected and resubmitted.

"Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental."

Environment

SAP S/4HANA Cloud Public Edition

Cause

In App Manage Workflow for Journal Entry Verification - in General Ledger (F2720) on Exception Handling tab the Required Action is set to 'Do nothing'

Resolution

In order to have the journal entry posted again:

1. Open App Manage Workflow for Journal Entry Verification - in General Ledger (F2720) 

2. Select workflow

3. Select Workflow step

4. On Exception Handling tab set the required Action to 'Redo General Journal Entry' and the Action Result to 'Restart workflow'.

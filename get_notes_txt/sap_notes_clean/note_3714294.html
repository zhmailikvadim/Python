Symptom

You want to know whether IPv6 can be used by SAP HANA.

Environment

  • SAP HANA PLATFORM EDITION 2.0

Resolution

Up to SAP HANA 2.0 SPS 03 IPv6 is tolerated, but the database itself relies on IPv4, so a basic IPv4 infrastructure needs to be implemented.

Starting with SAP HANA 2.0 SPS 04 IPv6 can be used independent of IPv4 using the following environment variables:

Environment variableDetails
SAP_IPv6_ACTIVE = 1

Activation of IPv6 support

NI_USEIPv6 = 1Switch from IPv4 to IPv6 for the database itself

When literal IPv6 addresses are involved in combination with ports, they should be encapsulated with square brackets in order to properly separate IP address and port, e.g. [2001:DB8::1234]:30003. In context of SAP HANA parameter settings (e.g. in section system_replication_hostname_resolution, SAP Note 2186744) the square brackets must not be used because otherwise the setting may be interpreted as parameter section that is also indicated with square brackets.

Starting with SAP HANA 2.0 SPS 09 the availability of IPv6 is mandatory because the underlying Lightweight Socket Layer (LSL) requires IPv6. If you disable IPv6 (e.g. via Linux kernel parameter disable_ipv6=1) in this context, the following error is reported by SAP HANA (issue number 348220):

System call failed [generic:97]: Address family not supported by protocol

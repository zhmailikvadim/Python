Symptom

  • GOSI not calculating after upgrade
  • /261 and /262 wage types are not generating in system
  • The standard SASI function is not being triggered in the payroll log.

Environment

  • SAP ERP
  • SAP R/3
  • SAP KSA Payroll
  • SAP ERP Saudia Arabia

Reproducing the Issue

  • Go to tcode PC00_M24_CALC
  • Run variant
  • Check RT 
  • SASI function not generated
  • /261 and /262 wage types not generated

Cause

Incorrect configuration in feature 24ERA

Resolution

The feature 24ERA will be used to configure retirement age and SANED age. The return value parameter should be set as one of the following three for all SI groupings in use,

I.  F/XX/S/YY : To be used if the retirement age for the group of employees are fixed.

Example: New graduates hired on or after 3rd of July, 2024. Retirement age is fixed to 65 and SANED is fixed to 58. The SI grouping will be SN and the feature return value will be F/65/S/58. SANED contribution will be stopped irrespective of the IT3252 “No Age Check”

II.  D/S/YY : To be used where the retirement age is calculated dynamically based on the formula provided by GOSI (as part of the new GOSI system) but SANED age is fixed.

Example: An employee with age below 50 Hijri as of 3rd of July, 2024. Retirement age is to calculated dynamically based on GOSI formula but SANED age is fixed to 58. The SI grouping will be SU and the feature return value will be D/S/58. SANED contribution will be stopped irrespective of the IT3252 “No Age Check”

III.  D : To be used where both retirement age and SANED age is calculated dynamically.

Example: An employee with age below 50 Hijri as of 3rd of July, 2024. Retirement age is to calculated dynamically based on GOSI formula and SANED also follows the same. The SI grouping will be SU and the feature return value will be D. SANED contribution will be stopped irrespective of the IT3252 “No Age Check”

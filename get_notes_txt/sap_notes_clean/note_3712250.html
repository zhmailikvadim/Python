Symptom

When attempting to view dispute cases in transaction FBL5N, users encounter a "Read Error" message. This issue arises during the process of accessing or navigating through the dispute cases associated with customer line items. The error message prevents users from successfully retrieving and displaying the details of the dispute cases.

Environment

  • FSCM Dispute Management 
  • SAP ERP
  • SAP Enhancement Package for ERP
  • SAP S/4HANA

Reproducing the Issue

  1. Navigate to transaction FBL5N to view dispute cases.
  2. Observe the "Read Error" message displayed.

Cause

Missing authorization

Resolution

  1. Check if the user is receiving the error message "No authorization to log in via a trusted system (L-RC=1 T-RC=0)." In this case, either the username or the password for logging in is incorrect. For further details, refer to KBA 3452334 No authorization to log on as a Trusted System (L-RC=1 T-RC=0)

  2. Additionally, it is required that the user has the following authorizations to display a Dispute case. Verify the user's authorization using the SU53 transaction for the following objects:

          S_SCMG_CAS Case Management: Case
          S_SCMG_FLN Case Management: Authorization by Field
          S_SCMG_STA Case Management: Status
          S_SCMG_TXT Case Management: Text Notes

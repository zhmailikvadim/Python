Symptom

The QCT / HECQC report shows RED (NG) for the check HANA Configuration Infrastructure with a message similar to:

“Database not found: HANA instance <SID> not found”

The SAP HANA database is running, SQL connectivity to the database is possible, and monitoring tools (for example system monitoring or health checks) show a healthy system state.
The behavior is typically observed after a Disaster Recovery (DR) role switch or takeover where a former standby host becomes the active database host.

Environment

SAP HAN DB Version 2.0

Cause

The check HANA_Configuration_Infrastructure is executed through SQL-based HECQC operations but is requires that the active SAP HANA instance is correctly detected on the host level.

After a DR role change, the host may already provide SQL services while the host-level instance detection may not yet reflect the new active HANA role.
In this state, the framework cannot reliably bind the check execution to the active HANA instance even though the database is operational.
The check therefore reports “Database not found” although the HANA database is available.

This situation may occur temporarily after a DR role switch and does not indicate a SAP HANA configuration problem.

Resolution

No configuration change is required.

Run the QCT / HECQC checks only after the role transition is fully completed and the active HANA instance is consistently recognized on the host level.

Re-execute the QCT report after the system state has stabilized.

Keywords

QCT, HECQC, DR switchover, Database not found, HANA Configuration Infrastructure

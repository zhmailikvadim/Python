Symptom

  • Basic import is processed successfully and no error is found in the import, however the employees are not created or updated in the instance.
  • No user has been created in the system after the import process.
  • No error messages or warnings are displayed during the import job.
  • The user cannot be found in the system after the import.

Environment

SAP SuccessFactors HCM Suite

Reproducing the Issue

  1. Download the standard import template.
  2. Populate all mandatory fields, save the file, upload it, and run the import job.
  3. Observe that the import job finishes with status “Successfully completed”.
  4. Verify that no user is created/updated in the system, no error messages or warnings are displayed, and the user cannot be found in the system after the import.

Cause

This inconsistency is caused by an additional column for user ID manually added to the business configuration.

Resolution

USERID is already in the business configuration by default, so adding a second one may cause import issues.

In this case, it's necessary to amend your business configuration and remove this additional USERID field so any template downloaded from this point will respect the correct columns. To fix this, please follow the steps below:

  1. Go to manage business configuration
  2. Employee profile -> View template -> sysUserDirectorySetting    
  3. Click on Take action -> Make correction
  4. Open the details of the item/id: sysAllUserDirectorySetting
  5. Delete any additional fields created for "USERID" and save it.
  6. After that, download the new employee import template and fill it out with the user details you want to import
  7. Import the file. The user should be successfully created/updated.

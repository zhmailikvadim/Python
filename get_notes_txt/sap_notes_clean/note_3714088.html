Symptom

 

  • Error message AAPO189 occurs during posting  

  • The Asset Master Record and the Cost Center are assigned to the same Profit Center  (e.g. A). 

  • During posting, the system derives a different Profit Center (e.g. B).

  • The posting is blocked with an error message.

Environment

  • FIN Asset Accounting
  • SAP ERP
  • SAP ERP Central Component
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA
  • SAP S/4HANA Finance
  • SAP S/4HANA Cloud Private Edition
  • SAP Fiori for SAP S/4HANA Finance,
  • SAP Fiori for SAP S/4HANA,
  • SAP Fiori for SAP S/4HANA Cloud Private Edition

Reproducing the Issue

 

  1. Create or maintain an Asset Master with Profit Center A.

  2. Ensure that the Identity Indicator for Profit Center is not set in FI-AA customizing.

  3. In the FI-AA error table, set the message type for Profit Center to Error.

  4. Create a Posting (e.g. ML81N) 

    • Use account assignment to an order or cost object

    • which derives a different Profit Center (e.g. B)

  5. Post the document.

  6. The system raises error AAPO189 and blocks the posting.

Cause

 

The error is caused by inconsistent FI-AA customizing for the Profit Center:

  • The Identity Indicator is not set, meaning different Profit Centers are allowed by design.

  • At the same time, the message type is set to “Error” in the FI-AA error table.

  • This combination (non-identical allowed + error message) is not supported and leads to AAPO189.

Note:
In earlier releases, this inconsistency might not have been enforced due to missing or incorrect checks.
After upgrades, the standard system behavior is correctly applied, leading to the error.

Resolution

. Check Profit Center Activation (Transaction ACSET)

  • Go to transaction ACSET

  • Ensure that PRCTR (Profit Center) is activated as an account assignment object:

    • for all valuation areas with real-time posting

  • Activate PRCTR where missing

2. Adjust Message Control for Profit Center (Mandatory)

Customizing path:
FI → Asset Accounting → Integration with General Ledger → Additional Account Assignment Objects → Error Table

  • For Profit Center (PRCTR):

    • Change message type from Error
      → to Warning or No Message

✔ Result:
Postings with a different Profit Center are allowed as intended by the customizing.

3. Segment Handling (Optional, Recommended)

  • The Identity Indicator may remain unchanged

  • Set message type for Segment:

    • from Error → to No Message

✔ Benefit:

  • Enables automatic segment derivation from the asset

  • Avoids inconsistencies in derivation logic

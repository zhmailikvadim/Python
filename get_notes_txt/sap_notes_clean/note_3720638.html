Symptom

  • Two-Factor Authentication (TFA) is not triggered despite setting up the RBA rule when logging in via a private/incognito browser within the scoped site.
  • When the "AND" condition is set, TFA does not trigger unless both device and country change criteria are met during login.
  • When the "OR" condition is set, the TFA verification screen is displayed repeatedly due to the "apikey" condition.

Environment

  • Customer Data Cloud
  • Privacy & Safety (Consent, RBA - Risk-Based Authentication)

Reproducing the Issue

  1. Create an RBA (Risk-Based Authentication) rule for TFA within the Account Rule Set.
  2. Apply the RBA rule to the Default Account Rule Set to enable TFA for all users.
  3. Access the site URL with a private/incognito browser.
  4. Attempt login using an account created within the scoped site and observe TFA is not triggering or TFA screen displayed repeatedly.

Cause

  • When the "AND" condition is set, TFA does not trigger unless both device and country change criteria are met during that login.
  • When the "OR" condition is set, the "apikey" condition causes repeated triggering of the TFA verification screen.

Resolution

  1. Split the RBA rule into two separate rules:

       a. One for device changes.

       b. One for country changes.

    2. Configure each rule to use an "AND" condition with the API keys.

    3. Test the updated configuration to ensure proper functionality.

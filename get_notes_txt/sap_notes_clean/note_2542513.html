Symptom

The SAP NetWeaver Java application server startup fails with error code 2150 with ABAP user store.

/usr/sap/<SID>/<inst_nr>/work/dev_server0:

ERROR => Java node 'server0' terminated with exit code 2150

The reason of the startup is recorded in std_server0.out file:

************************************
CSN Component [BC-JAS-SEC-UME], DC Name [sap.com/com.sap.security.core.ume.service]
com.sap.engine.frame.ServiceException: Start of UME service failed. Check help topic "Start of UME Service Failed".
Technical details: com.sap.conn.jco.JCoException: (103) JCO_ERROR_LOGON_FAILURE:
Initialization of destination UMEBackendConnection failed: Name or password is incorrect (repeat logon)
at com.sap.security.core.server.ume.service.UMEServiceFrame.start(UMEServiceFrame.java:463)

************************************


Also, the below error can be populated:

************************************

CSN Component [BC-JAS-SEC-UME], DC Name [sap.com/com.sap.security.core.ume.service]
com.sap.engine.frame.ServiceException: Start of UME service failed. Check help topic "Start of UME Service Failed". Technical details: com.sap.conn.jco.JCoException: (103) JCO_ERROR_LOGON_FAILURE:

Initialization of destination UMEBackendConnection failed: User is locked; notify the person responsible


************************************

 

Environment

  • SAP NetWeaver AS Java 7.1 and higher
  • UME user store is ABAP.

Reproducing the Issue

In the ABAP server change the communication user password and restart the Java server.

The Java server security service startup fails with the above exception.

Cause

The communication user's logon credentials are invalid.

Resolution

In the ABAP system change the communication user password.

Store the new password in UMEBackendConnection RFC destination.

Steps:

1. Start configtool.bat (Windows) or configtool.sh (Unix/Linux) as <SID>adm user from the folder /usr/sap/<SID>/<inst_nr>/j2ee/configtool.

2. Press the button "switch to configuration editor mode".

3. Press the "edit" button and on the popup window press the "Yes" button.

4. Look for the ABAP server parameters:

  • Expand node Configurations - Destinations - RFC
  • Double click on the affected RFC destination reported by the exception, this time select the Propertysheet UMEBackendConnection
  • The upcoming window shows the RFC parameters of the UMEBackendConnection RFC destination

  • Note the ABAP server logon parameters
    • jco.client.client - client where communication user must be maintained
    • jco.client.user - communication user ID
    • jco.client.ashost - host name of the ABAP server
    • jco.client.sysnr - instance number

5. In ABAP server change the communication user password

  • Start SAPGui, logon to the ABAP server (host ABCDE000AS, client 001, instance number)
  • Start SU01 transaction, search for communication user ID (SAPJSF), press Edit button (small pencil icon)

 

  • Select "Logon Data" tab.
  • Update the password in "New Password" and in "Repeat Password" fields.
  • Save the changes by pressing the small floppy icon.

Note that password status is "Porductive Password".

6. Store the new passwod in configtool for the RFC destination.

  • Double click on jco.client.passwd field. In the popup windows enter new password in custom-value field, press button "Apply custom".

7. Start the Java application server.

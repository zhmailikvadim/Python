Symptom

When checking expensive statements, some statements are confirmed, but may be disappeared afterwards, however some older expensive statements are not overwritten. 

Environment

SAP HANA database

Cause

Expensive statements are port dependent, some are from xsengine and some are from indexserver.
Indexserver often executes more expensive SQL statements, and it may overwrite it's expensive statements but xsengine will not.

Resolution

Please check based on port. It is possible that indexserver generated expensive statements are already overwritten but older xsengine statements still exist in expensive statement list.
This is controlled by parameter(default 30000):

In-memory tracing records

in_memory_tracing_records

30000

Specifies the maximum number of trace records (per service) stored in memory.

It can be confirmed by following SQL:

select port, count(*)
from m_expensive_statements
group by port;

30.003    30.000
30.007    4.525

In this sample you can see 30003 which is usually indexserver has already reached 30000 limitation but xsengine is still not, there are only 4525 entries generated for xsengine. So xsengine will retain much more older expensive SQL statements, this can be confirmed by:

select port, min(start_time)
from m_expensive_statements
group by port;

30.003    20260211144707429656000
30.007    20250716065248600385000

In this sample we can see xsengine statements are retained even in 2026, but the oldest indexserver expensive SQL is on 2026/02/11.

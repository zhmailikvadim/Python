Symptom

  • The SPL offline screening transaction in E4H does not produce results for contractors who are not available in sanctioned party list (SPL) or global trade services (GTS) systems as business partners.
  • An XML file was created based on SAP guidelines (SAP Note 3056442), but the transaction /SAPSLL/SPL_CHSO3 returns the message: "There are no SPL screening results."
  • Relevant error message: Message No. /SAPSLL/CON_SPL_CHK033

Environment

  • Sanctioned Party List Screening

Reproducing the Issue

  1. Access transaction /SAPSLL/SPL_CHSO3.
  2. Enter a valid legal regulation.
  3. Select Group of external log systems.
  4. Specify the path of the source file from the local directory.
  5. Specify the path of the results file (results.xml) from the local directory.
  6. Use any of the three XML file formats available.

Cause

  • The error `/SAPSLL/CON_SPL_CHK033` occurs due to an incorrect field definition of the XML file.
  • For example region field in the file is set to a default placeholder, 'REGION,' which the system truncates to three characters, resulting in `REG`. However, `REG` is not recognized as a valid region.

Resolution

  1. Remove all default placeholders in the XML file.
  2. Recheck the behavior after updating the region field to ensure it is correctly defined.

See Also

Refer to: SAP Note 3056442

Symptom

  • When posting invoices in FB60 (and similar FI transactions), the G/L account is not carried over from the referenced funds reservation/precommitment.
  • The system requires a G/L account entry. 

Cause

  • This is standard system behavior. In FI postings, the G/L account cannot be automatically derived from a funds reservation.
  • A G/L account must always be entered for FI documents because FI uses it to determine field status, tax settings, and other controls.
  • It is therefore not possible to enter only the funds reservation and have the G/L account taken over from it.

Resolution

Options 1:

  1. Create the G/L account as a separate account assignment group and define rules for how the account assignment should be transferred. 
    IMG: Define Rules for Account Assignment Transfer
  2. FMZ1 – make the account assignment non-changeable.
  3. When using FB60, error RE153 (“G/L account in current document not the same as that in the earmarkd fds”) occurs if the G/L account differs from the one in the funds document.
    The message displays the correct G/L account, so a new line could be entered with the correct G/L account.

Option 2:
Alternatively, use transaction F871 to enter invoices. There you can first reference the funds reservation, and the G/L account will be carried over.

Keywords

FMZ1, FMX1, FMY1, Kontierungsübernahme

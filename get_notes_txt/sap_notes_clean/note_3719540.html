Symptom

Wrong Policy Start Date in PFO entries when shifting Start Date for future-dated New Business Application

Other Terms

FS-PM

Basis Line of Business

New Business

Commission Participant

Shift Policy Start Date

P_B_S_POLICY_SHIFT

PFO_GO_00IP

PFO_GZO

/PM0/CL_AB_INT_PFO_2

/PM0/CL_AB_INT_PFO_COM_ON_2

Reason and Prerequisites

Reason:

In new business processing, during the execution of the shift policy start date, the new effective date is not being retrieved from the application journal while updating the cache data. As a result, incorrect date is populated in PFO when the application is released for the future dated policy.

Solution

As a general rule, SAP recommends that you install a solution by applying a Support Package. However, if you need to install a solution earlier, use the Note Assistant to implement the correction instruction. You can find more information about the Note Assistant in Support, under https://support.sap.com/en/my-support/knowledge-base/note-assistant.html.

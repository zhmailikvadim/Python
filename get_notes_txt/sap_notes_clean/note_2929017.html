Symptom

You use a Unified Rendering-based application with a readonly InputField/ComboBox on it that has a custom-styled hover behavior. Since upgrading to Unified Rendering Cumulative Patch 1711, the hover behavior does apply to the readonly field, where it did not apply previously.

Other Terms

Web Dynpro Java, Web Dynpro ABAP, WDJ, WDA, SAP GUI for HTML, UnifiedRendering, Unified Rendering, Lightspeed, InputField, ComboBox, CustomStyles

Reason and Prerequisites

Using WD ABAP starting from 7.02, WD JAVA starting from 7.30 and SAP GUI for HTML all kernel releases.

Solution

As part of regular intense quality- and patch creation tasks, all Unified Rendering fixes will be collected and shipped about every four weeks.

The fix will be shipped with the next "Cumulative patch for Unified Rendering 1804".

Once available it will be delivered for WD ABAP starting from 7.02, WD JAVA starting from 7.30 and SAP GUI for HTML all kernel releases.

Detailed information about the process and delivery dates can be found in SAP note 2500800 - General information about cumulative patches for Unified Rendering. See also section "References".

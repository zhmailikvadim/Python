Symptom

The partition of a table in the BW name space (<bw_prefix> e.g. BIC, BI0 etc.) is reaching the 2 billion records per partition limit.

Environment

  • SAP HANA Platform
  • BW on HANA
  • BW4HANA

Resolution

BW tables with the naming convention
   /<bw_prefix>/F<cube_name>
like
   /BIC/F<cube_name>
are partitioned as explained by the section
   17. How should tables be partitioned in BW environments?
of the note
   2044468 FAQ: SAP HANA Partitioning

In each first-level partition four second-level RANGE partitions are defined on the KEY_<infocube>P which refers to different types of fact table data:

  • 1st range partition: Compressed requests (Compressed deltas)

  • 2nd range partition: Reference points (Initialization records)

  • 3rd range partition: Historic movements (Historic transactions)

  • 4th range partition: Uncompressed requests (Uncompressed requests)

See SAP Note 1548125 -> "HANA Optimized Inventory Cubes" for more details.

The 4th second level range partition of each first level partition contains uncompressed requests.

To reduce the number of the 4th range partition please start compressing the requests.

Please ask your functional BW team to compress the requests.

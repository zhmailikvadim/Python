Symptom

Enforce Numeric In Serial Numbers Only After A Fixed Prefix

Environment

SAP Advanced Track and Trace for Pharmaceuticals

Reproducing the Issue

  1. Create a serialization profile with length 7 and set SN type to alphanumeric; set prefix to "MIN".
  2. Define and activate a serial number range, for example MIN0000–MIN9999.
  3. Generate or request new serial numbers from this profile/range.
  4. Observe that some generated serials contain letters in the last 4 positions (for example, MIN046M).

Cause

The requirement to restrict the trailing characters to numeric-only after a fixed alphanumeric prefix is not supported out of the box; standard alphanumeric generation does not enforce numeric-only suffixes.

Resolution

  1. Create a serialization profile with length 7.
  2. Set SN type to “alphanumeric case sensitive with exclusion list” (optional, to fine-tune allowed characters).
  3. Set the prefix to “MIN”.
  4. Set “No leading zeroes” to false to allow numeric values like 0000–9999.
  5. Create a range definition MIN0000–MIN9999 based on this profile and activate it; close other ranges to prevent unintended assignments.
  6. Implement BAdI /STTP/BADI_SNR_FILTER to filter out any generated serial numbers that contain letters in the last 4 symbols, ensuring only MIN0000–MIN9999 are accepted.

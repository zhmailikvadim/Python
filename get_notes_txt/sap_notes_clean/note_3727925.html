Symptom

During formation creation according with SAP note 3663336, the error below occurs when performing synchronization of the systems. 

Errors occurred while synchronizing. Choose Resynchronize to restart the synchronization process. If the problem persists, report a case in the BPI-SIG-CON-CLM component and provide the formation ID "XXXXXXXXXX-XXXXXXXXXX-XXXXXXXXXX-XXXXXXXXXX" together with the following technical details:

  • Synchronization between SAP Cloud ALM System SAP Cloud ALM - XXXXXXX and connectivity add-on for SAP Signavio solutions System connectivity-add-on-for-SAP-Signavio-solutions-XXXXXXXXXX has failed.
    – connectivity add-on for SAP Signavio solutions System connectivity-add-on-for-SAP-Signavio-solutions-XXXXXXXXXXX returned a technical error for assignment with ID XXXXXXXXXXXXXXXXXXXXXX:"Updating the tenant id or formation id for an existing mapping is not supported"

Environment

SAP Cloud ALM 1.0

SAP Signavio Process Manager

Reproducing the Issue

  1. Open the BTP Global Account of your SAP Cloud ALM tenant.
  2. In the left navigation pane, navigate to System Landscape > Systems and select Create Formation.
  3. Enter a name for your formation and select the Formation Type Integration with SAP Signavio and SAP Cloud ALM. Then select Next Step.
  4. Select your SAP Cloud ALM and connectivity add-on for SAP Signavio solutions systems. Then select Next Step.
  5. Review the selected tenants and select Create.
  6. Wait for the formation to become ready. The systems have been successfully connected.

Cause

 The error message indicates that intent is to connect Cloud ALM to a SAP Signavio (connectivity add-on) tenant which already has Cloud ALM connected, you can't link it again to your SAP Cloud ALM tenant via a BTP formation. 

Resolution

  1. Delete the Formation in BTP.
  2. Without recreating the formation - Re-try sync from SAP Signavio to Cloud ALM.
  3. For more information on how to sync check Manually Syncing BPMN Diagrams to SAP Cloud ALM

If the issue persists, follow these additional steps below.

  1. Make sure the prerequisites mentioned in the SAP Signavio Process Modeler Workspace Admin Guide were followed - SAP Cloud ALM for Implementation | SAP Help Portal 

    Please note that your SAP Signavio workspace and SAP Cloud ALM tenant need to exist under the same business partner ID.

  2. Ensure that the Cloud ALM API entitlement was enabled for the new SAP Cloud ALM setup.
    This is required for Cloud ALM integration scenarios (Step 21 of SAP note 3257720).

  3. Please follow this guide to add the Entitlement of "SAP Cloud ALM API" to your new Cloud ALM subacconut:
    https://help.sap.com/docs/cloud-alm/setup-administration/enabling-sap-cloud-alm-api

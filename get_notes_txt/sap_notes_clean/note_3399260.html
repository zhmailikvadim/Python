Symptom

Size of Directory /hana/log is growing and no obvious reason is visible as replication works fine

Resolution

Perform 2 manual checks:

1. As <sid>adm user: for i in $(find -L /hana/log/<SID> -name 'hdb*' -type d) ; do echo $i ; hdblogdiag seglist $i | grep -oP 'Formatting|Preallocated|Writing|Closed|Truncated|BackedUp|RetainedFree|\bFree\b|Unknown|ClosedIncomplete|FreeIncomplete' | sort | uniq -c ; done

If a permission issue pops up like the one from a Testserver below, proceed with Step 2:

This way you may see some files do not belong to the correct <sid>adm user and group "sapsys". So HANA can't process these Log Segments and the directory will grow in size.
You need to see if the wrong user is a known one and why the permissions got changed. This is not done automatically by HANA.

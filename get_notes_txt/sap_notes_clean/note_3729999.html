Symptom

In SAP HANA MDS1) you observe that blending query data sources are checked instead of the datasources, which defined in the blending context query.

If many queries are blended, SAP Analytics Cloud uses concatenated names, which may exceed the name length limit of 256 characters used in the procedure performing the authorization check. This results in a not authorized error message.

1) MDS: MultiDimensional Services; analytical engine in SAP HANA, responsible for processing analytic request from SAP Analytics clients such as SAP Analytics Cloud (SAC) or SAP Analysis for Office.

Other Terms

SAP Analytics Cloud, SAC, SAP Analysis Office, SAP Design Studio, InA/MDS

Reason and Prerequisites

Reason:

The design of the authorization check assumed, that the datasource definition (ObjectName and AliasName) are identical to the ObjectName used in the datasource key of the blending DefiningContext.

Affected Releases:

  • SAP HANA 2.0 database, EPMMDS plugin:
    • "Fullversion EPMMDS" plugin revision: <= 1.00.2026XX.Y
  • SAP HANA Cloud Edition: 
    • <= CE 2026.XX.Y

Prerequisites:

  • Currently the object level authorization check is used by SAP Analytics Cloud

 

Solution

Apply one of the following SAP HANA database Revisions:

  • SAP HANA 2.0 database, EPMMDS plugin:
    • "Fullversion EPMMDS" plugin revision: >= 1.00.2026XX.Y
  • SAP HANA Cloud Edition:
    • >= CE 2026.XX.Y

Workaround:

  • SAP Analytics Cloud should use the same ObjectName or one which fits to the HANA name size limit of 256 characters (smaller 128 would be even better).

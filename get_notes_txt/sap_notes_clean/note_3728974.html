Symptom

System is down and main issues reported are:

  • Unable to load Oracle client library: libclntsh.so.11.1: cannot open shared object file
  • DlLoadLib()==DLENOACCESS while loading /usr/sap/<SYSNAME>/SYS/exe/run/dboraslib.so
  • Possible invalid settings for DIR_LIBRARY or dbms_type ('ORA')

Cause

  • The application server cannot connect to the database because the required Oracle libraries are either missing or not accessible.
  • The SAP system cannot load the library libclntsh.so.11.1, which is necessary for the oracle remote connection.
  • The oracle client is not installed on this machine or if the oracle client is installed, the problem is with library load path.

Resolution

  1. Install the oracle client based on the SAP Note 819829 
  2. add the path of the oracle client to the environment variable  LD_LIBRARY_PATH (LIBPATH, SHLIB_PATH) for the SAP user <sid>adm.
  3. Restart the SAP system afterwards.

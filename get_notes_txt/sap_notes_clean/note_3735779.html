Symptom

Invalid syntax error occurs when using POST without the correct SCIM body format.

Environment

  • Product: SAP Ariba Buying and Invoicing
  • Core Procurement REST API

Reproducing the Issue

Send POST /Users with an incorrect or incomplete SCIM request body to create a user and notice we receive HTTP 409 conflict.

Cause

HTTP 409 conflict was caused by an incorrect request body that did not meet SCIM requirements resulting in invalid syntax.

Resolution

    For POST user creation, provide a correct SCIM request body that includes required elements:

    • schemas array including urn:ietf:params:scim:schemas:core:2.0:User 
    • unique userName , active, emails , locale/timezone as needed
    • Required headers including x-anId, apiKey, and Authorization Bearer token

Symptom

  • Import validation fails in the Import Collection app with dependency-related messages.
  • Error messages include: "Collection XXXX version XX needs Access Sequence of Condition Technique YY1_XXX XXXX which is not yet imported."

Environment

SAP S/4HANA Cloud Public Edition

Reproducing the Issue

  1. Open the Import Collection app in the target tenant.
  2. Select the transport request.
  3. Choose Manual Import (or Validate).
  4. Review the execution messages showing dependency errors for a missing Sequence of Condition Technique.

Cause

A required "Access Sequence of Condition Technique" is not yet imported in the target system.

Access Sequences are technically classified as Extension Items and due to this technical classification, they are not automatically captured in standard Customizing transport requests. Extension Items must be transported using the Export Software Collection app (Key User Extensibility) instead of standard Transport Requests, which are used for other configuration objects.

 

Resolution

  1. Identify the missing Sequence of Condition Technique from the validation message.
  2. In the source tenant, locate the key user Software Collection that contains this sequence.
  3. Export the Software Collection containing the Access Sequence.
  4. Import the newly exported Software Collection through the standard transport flow into the target system to make the sequence available.
  5. After the sequence is present in target system, return to the Import Collection app and re-run validation/import of the customizing collections.
  6. Confirm that dependency checks pass and the collections import successfully.

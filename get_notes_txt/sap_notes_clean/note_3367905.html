Symptom

Getting a DUMP when performing archive on the SARA transaction.

Exception error MESSAGE_TYPE_X for ABAP Program /SCTM/R_FRD_ARCHIEVE_WRITE .

This is required to clean up the tables that are super crowded and causing memory consumption.

Environment

  • SAP Extended Warehouse Management (SCM-EWM)
  • SAP Extended Warehouse Management Interface TM (SCM-EWM-IF-TM)
  • SAP Transportation Management (SCM-EWM-TM)

Reproducing the Issue

1. SPRO - >Extended Warehouse Management - >Goods Issue Process - >  Transportation Management in EWM - > Freight Documents - > Define Freight Document Type - FRD ( Collective Shipment Outbound ) must be maintained .

2. Execute SARA transaction.

3. Enter Archiving object as TM_FRD.

4. Facing dump when activating the JOB to execute the selected data.

Cause

The error is occurring because the Order Document types (TMFR for FRD; TMSF and TMSP for SHP) are not activated in the table /SCMB/ORDARSC . 

Resolution

Check and correct the below customizing activities, using their IMG activity documentation as reference:

SPRO -> SCM Extended Warehouse Management ->SCM Basis -> Order Document Management -> Activate Settings in Order Document Management -> Activate Order Document Types and Dependent Objects (transaction /SCMB/ODM_ACT_ORTYPE).

 

Keywords

 MESSAGE_TYPE_X ,  /SCTM/R_FRD_ARCHIEVE_WRITE , TM_FRD , Order Document Management , TMFR for FRD; TMSF and TMSP for SHP .

Symptom

When you preview a Web Client UI Extension by using the Visual Studio Code plugin, you may encounter the following error message in the Web Client: "To work in developer mode, launch the extension debug server first."

Reproducing the issue

Prerequisites:

You are developing a Web Client UI extension using the Visual Studio Code plugin.

Scenario:

  1. Open your Web Client UI extension project in Visual Studio Code.
  2. Navigate to the UI extension layout file, open the right-click menu, and select "Preview (Web Client)."
  3. Log in to the Web Client preview webpage.

Expected behavior:

The Web Client UI extension works fine without any errors.

Actual behavior:

An error message appears: "To work in developer mode, launch the extension debug server first."

Solution

SAP intends to provide a feature package or patch to solve the symptom that is described in this SAP Note. The section References below lists the related feature package or patch once they become available. Be aware that References can only be confirmed at feature package or patch release date. SAP delivers feature packages or patches only for selected releases at its own discretion, based on the business impact and the complexity of implementation.

Workaround:

1. Launch a new browser from the command line with the following arguments:
        "--disable-web-security",
        "--disable-features=BlockInsecurePrivateNetworkRequests",
        "--user-data-dir=${workspaceFolder}/.vscode/chrome-dev-profile",
2. Copy the URL to the new browser and refresh.

Symptom

You are running an OPC UA agent with support for structured data type and connect it to an OPC UA server. The UA server is restarted or the connection is interrupted. In case of an interruption the period of interruption exceeds the timeout for keeping the session alive. Afterwards the OPC UA client successfully connects to the server again, but it can no longer read or process structured data types anymore. Only after a restart the agent is capable of processing the structured data types again.

Reason and Prerequisites

When a server is restarted or the connection is interrupted for too long, a new session has to be created. The client caches the information it needs to deserialize or serialize structured data types. When a new session is created, this information is lost.

Solution

This problem is corrected with patch 12 for SAP Plant Connectivity 15.2 with Support Package 03

Download the current installation program for your version of Plant Connectivity 15.2 from the SAP Service Marketplace and install it in accordance with the instructions in the installation guide. The current version of the installation guide is available on the PCo documentation page on the SAP Help Portal under http://help.sap.com/pco. You can use the installation program for a PCo Support Package or patch to update a PCo version that is already installed or to set up a new installation if PCo is not yet installed on the target computer.

Pay particular attention to the following points before you start the installation:

  • In case of an upgrade installation, make sure that all agent instances have been stopped and that the Management Console is closed. After starting the installation, the system issues a warning message if any PCo applications are still running that may impair the upgrade installation process.
  • If the installation program specifies a system restart, perform the restart before you continue with the installation of PCo.
  • We recommend that you also execute a data backup prior to installation (choose Plant Connectivity -> Data Backup).

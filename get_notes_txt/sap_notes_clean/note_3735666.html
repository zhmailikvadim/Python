Symptom

  • Error AC226 occurs when attempting to configure valuation area XX to adopt values from valuation area XX+1 in transaction OABC/OADB
  • The system displays the error message: "Transfer area must be smaller than area XX in ch.of depreciation" (Message no. AC226). 

Environment

  • FIN Asset Accounting​
  • SAP ERP​
  • SAP ERP Central Component​
  • SAP enhancement package for SAP ERP​

Reproducing the Issue

  1. Open transaction OABC.
  2. In the chart of depreciation, select valuation area XX.
  3. In field “Adoption of values from depreciation area” (table T093A, field WRTAFB), enter valuation area XX+1.
  4. Save.
  5. The system issues Message no. AC226: “Transfer area must be smaller than area XX in ch.of depreciation.”

Cause

The classic Asset Accounting version is designed to restrict adopting values from a depreciation area with a higher numerical identifier than the base area, resulting in this limitation.

Resolution

  1. Upgrade to New Asset Accounting to allow adoption of values from the desired depreciation area.
  2. Perform the necessary customization in transaction OABC to configure the adoption relationship accordingly. 

Keywords

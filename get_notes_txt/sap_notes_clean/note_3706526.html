Symptom

This KBA introduces how to use Dynatrace to database worker pool usage. It can be used to verify database performance behavior.

"Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental."

Environment

SAP Commerce Cloud

Resolution

1.Open dynatrace, go to Azure-->select customer's subscription information -> service information will be shown at bottom of the page.

2. Choose the right DB service, currently most of customers have been moved to Hyperscale -> click the service link -> jump into service details page.

3. Choose the instance which you want to check -> the page will go to details

There are several valuable DB information here, the database worker pool percentage mostly is in the page 2, should click the next page button. Then you can find the below information.


Notes: If the database worker pool usage was high or exhausted, the DB connection and related action will be impacted immediately. Please contact SAP support team to check which queries are occupied the connection. If any urgent, you can restart the application servers to release the connection.

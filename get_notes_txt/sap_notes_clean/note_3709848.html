Symptom

In the execution tree, the activity "Delete inherited objects on receiver system" (NZTR1_DELE_LOGTAB_AF_RC_R) contains substep "Restore FIAA validation rule". If the substep was already successfully executed in the first execution it should not be re-executed again.

Other Terms

 downtime minimization tool for SAP S/4HANA

Reason and Prerequisites

You are using any of the following releases or support packages in the systems of your NZDT system landscape:

  • NZDT 1.0 SP03

Solution

The status of the substep 'Restore FIAA validation rule' is stored after the execution.This status is evaluated before running the substep again.If the substep "Restore FIAA validation rule" is executed already, then re-executing the activity "Delete inherited objects on receiver system" will not execute the substep "Restore FIAA validation rule" again.

Implement the automatic correction instructions in transaction SNOTE on the following systems of your NZDT landscape:

  • Central System

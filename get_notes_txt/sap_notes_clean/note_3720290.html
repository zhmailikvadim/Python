Symptom

When viewing the 'Recurring Deduction' portlet, an employee has a Deduction with a Pay Component of 'Percentage' type. In legacy People Profile, this portlet used to show the percentage value. However, with Latest People Profile, the 'amount' that that 'Percentage' equates to is being shown instead. The 'Percentage' value can be seen if you view that specific Deduction record, but the expectation is that the 'Amount/Percentage/Number Of Units' field should also show the 'Percentage' value.

Environment

SAP SuccessFactors HCM

SAP SuccessFactors Employee Central

Reproducing the Issue

  1. Log into environment
  2. Go to example user's profile, and go to the 'Compensation' category
  3. Go to the 'Recurring Deduction' card to see the 'quick' view, or 'View All' to see all deduction records
  4. See the employee has a Deduction record for a Pay Component of 'Percentage' type
  5. See that the 'Amount/Percentage/Number Of Units' field shows a numerical 'Amount' value for the pay component, and not the 'Percentage' that is expected
    1. For example, if the Deduction is intended to be 5% of a Base Salary pay component, and that Base Salary is 1000, then the Deduction would be 5% of this, which is 50.
    2. The 'Amount/Percentage/Number Of Units' field is showing this '50' value instead of the '5%' value as expected

Cause

Expected behaviour for Latest People Profile

Resolution

This is expected behaviour for the Latest People Profile UI - the 'Amount/Percentage/Number Of Units' field will show the calculated 'Amount' of the Deduction, as the 'Amount' and 'Currency' values are shown on the same line now. If you want to confirm the 'Percentage' value of the Deduction, you can select the Deduction record to bring up its specific details, where the 'Percentage' is maintained as a separate field.

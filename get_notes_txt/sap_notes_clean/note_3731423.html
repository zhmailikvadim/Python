Symptom

  • After adding a configuration product, selecting attributes, and entering a quantity, the quantity initially flows to child items when inherit quantity is enabled.

  • Edit the configuration and add another attribute, the quantity on the main configurable item's primary attribute resets to 1.

  • Other attributes retain the originally inherited quantity from the main configuration product.

Environment

SAP CPQ

Reproducing the Issue

  1. Quote 2.0 engine is used.
  2. Create simple product.
  3. Create user selection attribute and add only one value which is referencing product from previous step.
  4. Create parent/child product, add attribute from previous step and mark it as line item and set inherit quantity flag.
  5. Add parent/child product to quote, change quantity - quantity is propagated to child .
  6. Edit item, just click update quote - quantity on root item is preserved, quantity on child item is reset to 1.

Cause

Bug in application.

Resolution

  • The tentative release fix version for this issue is 2608.
  • As a workaround use container attribute instead of user selection whose value references product.

Symptom

Bad performance because of calculation of derived attributes although no derived attributes exist.

Other Terms

Derived Attributes, Changed At, Changed By, Check Results, NODE_CHANGED_AT, NODE_CHANGED_BY, CONTENT_CHK_RESULT

Reason and Prerequisites

The three attributes NODE_CHANGED_AT, NODE_CHANGED_BY and CONTENT_CHK_RESULT are defined as derived attributes, but not handled as derived attributes.
The derived attributes check is done although no other derived attributes exists.

The attribute 'Check Results' (CONTENT_CHK_RESULT) is defined as derived attribute, but is not considered in the derived framework, because this is only used in the Check Framework.
Because of time inconsistencies, the attributes 'Changed At' (NODE_CHANGED_AT,) and 'Changed By' (NODE_CHANGED_BY) are defined as derived attributes but they are not used as derived attributes.
The attributes are updated in the SOLDOC API with SAP Notes
3293560 - 'Changed At' are wrong in Solution Documentation.

Solution

Please implement the current correction instruction with transaction SNOTE.
After that, only if derived attributes other than the attributes Changed At, Changed By or Check Results are defined, the derived attributes framework is running.

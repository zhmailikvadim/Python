Symptom

CDS connectors cannot be activated in ESH_DASHBOARD. A warning is displayed
W    [ESH_ENG_CDSVAL_ES043]    'enabled' set to 'false' - ES-Connector will not be activated 

Environment

  • Product: SAP S/4HANA foundation
  • NetWeaver Enterprise Search

Reproducing the Issue

  1. Start the Enterprise Search Dashboard (transaction ESH_DASHBOARD)
  2. Select the line "Data Sources"
  3. Put the cursor on the respective connector ESH_S_FUNCTLOCATION and select the magic stick
  4. Logs will be displayed :

I    [ESH_ENG_CDS_ABAP058]     Component for reporting issues: ESH_S_FUNCTLOCATION
I    [ESH_ENG_CDS_ABAP058]     Component for reporting issues: PM
I    [ESH_ENG_CDS_ABAP058]     Component for reporting issues: PM
I    [ESH_ENG_CDS_ABAP046]     Validation findings for domain @EnterpriseSearch:
W    [ESH_ENG_CDSVAL_ES043]    'enabled' set to 'false' - ES-Connector will not be activated
I    [ESH_ENG_CDS_ABAP005]     Corresponding DDIC view: ESH_U_FUNCTLOCTN
I    [ESH_ENG_CDS_ABAP006]     Derived connector ID: CD$ALL~ESH_U_FUNCTLOCTN~
I    [ESH_ENG_CDS_ABAP007]     Derived SWC/ model-ID: CDS/ESH_U_FUNCTLOCTN
I    [ESH_ENG_CDS_ABAP055]     Testing of generated ES-connector successful
I    [ESH_SQL_SEARCH068]       Search with 'SP_PRIORITY' is working on CDS entity ESH_S_FUNCTLOCATION
S    [ESH_ENG_CDS_ABAP045]     ESH_S_FUNCTLOCATION successfully activated for Enterprise Search

Cause

Resolution

  1. Start the Enterprise Search Dashboard (transaction ESH_DASHBOARD)
  2. Select the line "Data Sources"
  3. Put the cursor on the respective connector ESH_S_FUNCTLOCATION (search for it)
  4. Choose the action "Data Source details (F6)" above. The detail view of the connector will be opened.
  5. Under "Data Source-level adaptions (local)" choose the action "(In)activate Data Source". The connector will be activated/deactivated accordingly. 

Or 

  1. Please reference to note 3018839 - Activation/Deactivation of CDS Search Connectors 
  2. Note that to create a local MDE the system needs to be open Transaction SE06 (Set System Change Options):
    Global Setting: Modifiable
    Software Component LOCAL (Local Developments (No Automatic Transport)): Modifiable
    Namespace /0CUST/ (Customer Name Range): Modifiable But you can set in SE06->Edit 
    All Namespaces: Modifiable
    All Software Components: Modifiable Transaction SCC4, Client 100:
    Cross-Client Object Changes: Changes to repository and cross-client customizing allowed
  3. Additionally user needs privileges for authorization object S_DEVELOP with   S_DEVELOP-OBJTYPE = ‘DDLX’
       S_DEVELOP-ACTVT = 01
    to be able to create the necessary MDE (R3TR DDLX).

See Also

  • Refer to: Activation/Deactivation of Search Connectors (Note 3018839)

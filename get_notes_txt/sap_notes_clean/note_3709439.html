Symptom

This Note enables customers to add custom disclaimers / attestations for different actions performed in the Approval Hub app. Whenever a user performs one of the following actions, customers can configure custom texts that are displayed in the confirmation popup for each action:

  1. Approve
  2. Reject

The texts can be maintained based on Category, Process, and Subprocess. In addition, different texts can be configured for different positions of the Approve and Reject actions.

For example, separate disclaimer texts can be maintained for the Approve button on the Worklist page and on the Detail page, respectively.

Customer Enhancement  Changes for moving from Step1 to Step2

In class /FC1/BP_I_AH_ALL_MP, the BAdI implementation responsible for handling item changes has been modified in the UPDATE method.

Previously, the BAdI method CHANGE_ITEM_DATA was invoked immediately before the database update of Manual Posting items. This method has now been removed.

As a replacement, a new BAdI method ADD_ITEM_DATA has been introduced and is executed at the beginning of the UPDATE method. This method enables customers to implement their own logic to add items for a Single Manual Posting during the transition from Step 1 to Step 2. When implementing this method, customers must generate the Document ID for the items to allow further processing.

If the ADD_ITEM_DATA BAdI method is implemented, customers should skip the standard logic that adds default items in Step 2 by setting the flag CV_SKIP_STANDARD within the ADD_ITEM_DATA method.

Symptom

  • When using the CDP connector to trigger the event "Create or update a contact (OAuth 2.0)", the contact in Emarsys is not updated.
  • The name field is expected to be updated during contact synchronization, but no changes occur.
  • Using the event "Create or update a contact - Automatic activation" on customer update resolves the issue.

Environment

SAP Customer Data Platform

Reproducing the Issue

  1. Use the CDP connector to trigger the event "Create or update a contact (OAuth 2.0)".
  2. Attempt to update the name field for newsletter in Emarsys.
  3. Observe that the field is not updated.

Cause

Using the field name instead of the corresponding field ID (numeric identifier) defined in Emarsys.

Resolution

  1. Adjust the payload to use the appropriate numeric field ID for the newsletter field instead of the field name.
  2. Ensure that the payload references the correct numeric field ID for "primaryCDPId" if required.
  3. Test the updated payload to confirm that the newsletter field is successfully updated in Emarsys.
  4. Note that the event "Create or update a contact - Automatic activation" works with field names, while the event "Create or update a contact (OAuth 2.0)" requires numeric field IDs.

Symptom

In transaction PFCG, you merge the authorization data of a single role (expert mode "Read Old Status and Compare with New Data" on the "Authorizations" tab page). When you check the updated authorization list, you notice that the merge process has deleted authorizations with the status "Maintained" even though at least one menu application or default variant returns a suitable authorization default.

Other Terms

Role Maintenance, Profile Generator

Reason and Prerequisites

Program error

The deleted authorizations result from inactive authorization defaults that were first activated after inclusion in the role and then copied. The problem does not occur for authorizations copied from active authorization defaults.

Solution

Use transaction SNOTE to implement the correction instructions or import the relevant Support Package.

Explanations

This correction prevents the symptom described above from occurring again during future merge processes, but it cannot automatically repair roles that are already affected by the error. The deleted authorization copies must be restored manually.
The effort required to maintain the field values in the new authorizations depends on whether or not you have saved the authorization data after the symptom occurs:

  • The data has already been saved.
    In this case, you must change the values as usual using the maintenance dialogs of the authorization fields. You can find the values in the deleted authorizations in the change documents for the role (menu function "Utilities" -> "Display Changes" -> Document Type "Authorization Data") by displaying the role in an additional session.
     
  • The data has not yet been saved.
    If you do not already use the ALV-based display of the authorization data, navigate to the authorization list and choose "Utilities -> Settings" from the menu and select the checkbox "Use ALV Tree (Call Authorization Maintenance Again)". With this setting, you execute the expert mode "Read Old Status and Compare with New Data" on the "Authorizations" tab page in change mode to merge the authorizations again. In the updated authorization list, recreate the desired authorization copies. Since the deleted authorizations are displayed in the other half of the screen, you can transfer the old field values to the new authorizations using drag and drop.

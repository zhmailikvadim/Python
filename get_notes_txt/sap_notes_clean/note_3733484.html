Symptom

When attempting to change the document date while posting the TCR, the system allows the modification initially. However, after posting, the document date is automatically reset to the end of the quarter instead of retaining the user-entered value. This leads to incorrect document date being stored in the posted document.


This note corrects the behavior by ensuring that the manually updated document date is retained and not overridden during TCR posting.

Other Terms

TCR document date issue, /irm/ipttm, posting date reset, quarter end adjustment, document date override, TCR posting issue

Reason and Prerequisites

This is caused by program error

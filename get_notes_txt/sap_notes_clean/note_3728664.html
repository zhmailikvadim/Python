Symptom

The current logic fails to correctly display NIP values that contain special characters. As a result, any NIP with these characters is not visualized properly.

Other Terms

Poland, eDocument, eDoc, eInvoicing, electronic invoice, electronic invoicing, Poland eInvoice, Document Compliance, Digital Compliance, B2B, KSeF, Electronic Document, DRC (Document and Reporting Compliance), FA(3), FA(3) Schema Version

Reason and Prerequisites

  • Implement the SAP Note 3320855 Electronic Document Processing for Poland - KSeF: Human Readable Format (HTML) View
  • Implement the SAP Note 3301912 Electronic Document Processing for Poland - KSeF: Mapping and Code Correction I
  • Implement the SAP Note 3369003 Electronic Document Processing for Poland - KSeF: FA(2) Schema Version Code Corrections
  • Implement the SAP Note 3377810 Electronic Document Processing for Poland - KSeF: FA(2) Schema Version Code Corrections II
  • Implement the SAP Note 3698435 Electronic Document Processing for Poland - KSeF: FA(3) Schema - Display PDF
  • Implement the SAP Note 3702483 Electronic Document Processing for Poland - KSeF: FA(3) Schema Version Code Corrections III
  • Implement the SAP Note 3527316 Electronic Document Processing for Poland - KSeF: Code Correction for Wrong VAT number for Podmiot3 XML field
  • Implement the SAP Note 3676968 Electronic Document Processing for Poland - KSeF: FA(3) Schema
  • For a complete prerequisite list, refer to SAP Note 3279206 (Electronic Document Processing for Poland - KSeF: Implementation Overview).

Solution

Date                  Version         Reason                                                                                        
18.03.2026 1 Released for S4CORE 107
19.03.2026 2 Released for customers
23.03.2026 3 Released for S4CORE 107

The updated logic now supports NIPs that contain special characters. When generating the XML, all special characters are automatically removed to ensure the NIP is exported in the correct standardized format.

Symptom

  • In the PH_WHT_BIR2307 PDF output, the company address mapping is not clear.

Environment

  • Financial Accounting (FI)
  • SAP HANA
  • SAP S/4HANA 
  • SAP S/4HANA Finance
  • SAP S/4HANA Cloud Private Edition
  • SAP Fiori for SAP S/4HANA
  • SAP Fiori for SAP S/4HANA Finance
  • SAP Fiori for SAP S/4HANA Cloud Private Edition

Reproducing the Issue

  1. Open the Fiori app "Run Statutory Reports".
  2. Enter Report name: PH_WHT_BIR2307, select the desired report year and reporting period.
  3. Select a run number.
  4. Open the generated PDF and review the company address.

Cause

The report uses a predefined address concatenation order for the payer/company code address as per PH tax authority guidelines: StreetPrefixName (Street 2), AdditionalStreetPrefixName (Street 3), HouseNumber, StreetName (Street), StreetSuffixName (Street 4), CityName.

Resolution

  1. Review how the company code address is maintained in master data: Street 2 (StreetPrefixName), Street 3 (AdditionalStreetPrefixName), House Number, Street (StreetName), Street 4 (StreetSuffixName), City.
  2. Align the address maintenance to the report’s fixed mapping, as the PH_WHT_BIR2307 form concatenates these fields in the specified order to comply with PH guidelines.

Symptom

FHSQL Webi report Instances failing with one of the errors:

  • "Document" with identifier "#####" does not have any inner resources of type "Report map"
  • "The document cannot be loaded (IES 10833)" 
  • The main report works without error, the issue is with the instances
  • The .wid files corresponding to the report are presented in the input (main report) and output (scheduled report) folders
  • The following erros are also found in the end-to-end trace: 
    “ERROR:C3CDB_GroupingVarFactory: GetTokenFormula: FindByKey failed [KC3cdbGroupingVarFactory.cpp;1780] C3CDB_GroupingVarFactory: GetTokenFormula: 0.003 Error=GetTokenFormula: FindByKey failed”
    "webiserver_SERVERNAME.WebIntelligenceProcessingServer2|open o_read : <INSTALL DIR>:\Programs\SAP BusinessObjects\SAP BusinessObjects Enterprise XI 4.0\Data/SERVERNAME\storage\docs\00000007\00000001.wid_\en_US\doc\docInfo_documentid_en_US_en_US.xml"
  • The file docInfo_documentid_en_US_en_US.xml is presented in the file location.

Environment 

  • SAP BusinessObjects Business Intelligence platform 4.3 SP5 P3

Steps to reproduce:

  1. Open webi instance in BI Launchpad (Select the report > History > select an instance);
  2. Gets one of the error messages: "The document cannot be loaded. (IES 10833)" or "Document" with identifier "#####" does not have any inner resources of type "Report map"

Cause

  • This issue is currently under investigation by development.

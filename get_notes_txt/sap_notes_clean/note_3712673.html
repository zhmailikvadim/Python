Symptom

In transaction C223 when doing automatic routing determination error M3648 "Selection of routing N / not successful occurs. 

Environment

  • SAP ERP Logistics (LO)
  • SAP Enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA, on-premise
  • SAP S/4HANA Cloud Private Edition

Reproducing the Issue

  1. Transaction C223
  2. Enter the Material 6743727,
  3. Input Plant
  4. Input new Production Version ID and description.
  5. Enter the validity period and the required BOM data
  6. In the column "Type" (Task List Type), enter 'N' for Standard Routing,
  7. leave the fields for "Group" and "Group Counter" empty to allow automatic routing determination.
  8. Press Enter.
  9. Error message M3648: "Selection of routing N / / not successful"

Cause

In S/4 HANA systems, the field "Routing Selection ID" is no longer available due to a simplification. The documentation of this simplification can be found in SAP Note 0002270240 - S4TWL - Rate and Rough-Cut Planning.

Resolution

As a result, the field APLSL will always be empty in S/4HANA, the program logic no longer considers the Selection ID, even though the field still exists in the database table. This means that it is not possible to revert back to the logic used in ECC. This new logic may present additional steps for the users, but it ultimately provides more consistent results and helps to avoid issues in MRP Run and order scheduling.

Symptom

  • A service named SYSAM is running in the background under (services.msc) provided by Flexera Software LLC in a production application server to support SAP PowerDesigner. Currently it's running in the list of services locally without any association with service account. Can the service can be configured with service account?
  • Try to find a way such that we can assign a service account to SYSAM process

Environment

  • Windows 10 edition
  • Window 8 edition
  • Window 7 edition

Resolution

Use the SC command to assign a service account. SC refers to Service Control.

Example:

  1. Open DOS prompt in "Run as Administrator"
  2. type: sc  config SYSAM type=own obj="support\john" password="1234"
  • What this does is assign an exising account (domain is support, user is john, password is 1234) to process named SYSAM.

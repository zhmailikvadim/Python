Symptom

If you edit a portfolio contract that already has several condition headers (for example, due to a rollover) in the borrower's liability discharge, the system displays the incorrect condition header in IBS FIN.

Reason and Prerequisites

In the method MAP_CML_TO_FIB (/IBS/CLFM_OUTB_CML_FIB_MAP), the system selects the correct condition for the financing based on the CMU date according to the implemented logic.
If a condition header type “01 – Contract” exists for the contract, the logic only considers the condition header type “01 – Contract” and this leads to an error in the case of financing deals after a renewal.

Solution

The logic has been adjusted in the method In the method MAP_CML_TO_FIB (/IBS/CLFM_OUTB_CML_FIB_MAP) so that the condition header types "02 - Contr. Set" and "03 - Per. Contract”.

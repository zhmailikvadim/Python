Symptom

This SAP Note describes an issue that occurs when releasing a Shop Floor Routing, where the system does not permit the creation of a Production Version for multiple assigned materials and MBOMs.

The issue arises under the following conditions:

  • The previous version of the Shop Floor Routing contained a single assigned material and MBOM.

  • A Production Version has been created for the previous version of the Shop Floor Routing.

  • The previous version of the Shop Floor Routing is subsequently revised.

  • New materials and MBOMs are assigned in the subsequent version of the Shop Floor Routing.

  • The subsequent version of the Shop Floor Routing is released.

As a result, the system does not allow the creation of Production Versions for the newly assigned materials and MBOMs.

Other Terms

PP-PEO; Production Version; MSFR4; Shop Floor Routing

Reason and Prerequisites

Program error.

A released Shop Floor Routing and a corresponding Production Version must exist.
One material and one MBOM must be assigned to it.

Solution

Apply the attached correction instructions

The system allows the creation of Production Versions for the newly assigned materials and MBOMs for the following case:

  • The previous version of the Shop Floor Routing contained a single assigned material and MBOM.

  • A Production Version has been created for the previous version of the Shop Floor Routing.

  • The previous version of the Shop Floor Routing is subsequently revised.

  • New materials and MBOMs are assigned in the subsequent version of the Shop Floor Routing.

  • The subsequent version of the Shop Floor Routing is released.

The following additional scenarios must also be considered.

Scenario 1: The current routing version contains more than one Material and MBOM assignment.

  • A new routing version is created.

  • One or all MBOM assignments are removed in the new routing version.

  • The routing is released.

  • The corresponding Production Version from the previous routing version is used during the Production Version update process.

  • The Production Version is updated accordingly based on the latest MBOM assignments maintained in the routing.

The same behavior should apply to the scenario where the current routing version does not contain any MBOM assignment and an MBOM assignment is added in the new routing version.

Scenario 2: The current routing version contains more than one Material and MBOM assignment and previous version does not contain.

  • Set obsolete for current routing version.
  • The corresponded production version will be used during production version changes.
  • And the production version will be updated to contain the latest MBOM assignments.

The same behaviour is happened when current routing version without MBOM assignment and in previous version contains more than one Material and MBOM assignment.

Symptom

There is an extra 0 being generated in the XML file for the phone number between the DDI (Direct-Dial-in) and phone itself for Slovenia based-countries

This could be an intergration scenario for example S/4 HANA

Environment

 SAP Business ByDesign

Cause

Telephone number is broken down in parts which is leading to wrong phone/mobile number for shipping Intercompany Sales Order

Resolution

In ByDesign you have a formatted telephone number for example it shows as  +386 28838193 (this is displayed on the screen). But, internally the telephone number is broken down in parts. This is broken down to the following components:

Country Dialing Code: 386

Subscriber ID: 028838193

The Suscriber ID shows a leading "0", as this is the rule for Slovenia.

Locally the telephone number starts with 01 or so for the area code.

For dialling from abroad (and calculating the formatted international format of the telephone number, the rule is to remove the first "0" (so called trunk digit).

If there is no entry or an initial entry, the telephone number in S/4 HANA will be constructed differently than in ByDesign and therefore show the "0".

The formatting in S/4 Hana follows the same way as in ByDesign:

Country Dialing code + Subscriber ID. From the subscriber ID the trunk digit is cut.

386 + 028838193 => the 0 should be cut.

 In the Business Configuration settings supports the first "0"  that needs to be removed when calling from abroad and the country dialling code will be added: 386

So the formatted number will look like:+386 28838193

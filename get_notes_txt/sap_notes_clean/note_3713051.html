Symptom

When active, the parameter Application.SM.SyncEmptyRepeatableSectionToSM can be cause data loss?

Environment

Prod/Test

Cause

Parameter Application.SM.SyncEmptyRepeatableSectionToSM activated.

Resolution

Yes — this parameter can affect supplier data and downstream integrations.

What the file says

  • Default = No. When a supplier deletes the final instance of a repeatable section in the supplier-facing questionnaire, the supplier management (SM) application retains that final instance and its answers. That prevents the integrated ERP from losing required data (for example, a bank account or tax ID) and avoids integration errors.

  • If you set the parameter to Yes, deleting the final repeatable-section instance in the supplier view removes that instance from the supplier’s profile in SM and synchronizes the deletion to the integrated ERP system. That changes the data seen in SM and the ERP and can create or remove values in integration messages.

Practical impacts to consider

  • Mismatches: With the default (No), the supplier’s questionnaire view and SM/ERP data can diverge (supplier removed it, but SM/ERP still shows it). With Yes, deletions are synchronized and both sides match.

  • Critical fields: Bank account and tax ID entries are explicitly called out — changing the setting can add or remove those values in SM and the ERP and therefore affect payment/ tax processing.

  • Integration messages: Enabling synchronization of deletions changes outbound integration content and may remove previously populated fields in the ERP.

  • Downstream logic: Reports, validations, or ERP processes that expect those fields may be affected.

  • Governance: The parameter is set by SAP Ariba Support, so plan the change with them and with downstream teams.

Recommendations

  • Don’t change this in production without testing. Try the change in a lower environment and compare records and integration messages.

  • Review mappings and ERP requirements for bank/tax fields so you know whether deletions are acceptable.

  • Notify ERP/integration and business teams, schedule a maintenance window, and take backups or exports of affected data.

If you want the setting changed, contact SAP Ariba Support and describe the business/technical rationale and expected impact.

Symptom

  • When enable the latest editing and history UI, the "Add" button must be clicked twice to add a work permit for the fields to appear.
  • When disable the latest editing and history UI, the "Add" button only needs to be clicked once, then the fields appear.

Environment

SAP SuccessFactors Employee Central

Reproducing the Issue

  1. Go to "Configure Latest People Profile" >> General Settings >> Tick "enable the latest editing and history UI for employee central hris entities on full profile" >> Save
  2. Go to Job information of sample user >> Click on Add for Work Permit Info >> Page is displayed as "There’s nothing to show yet
    When there is, you’ll see it here."
  3. Click the "Add" in the upper right corner, then fields appear.

Cause

Confirmed as a bug by the Product Manager team.

Resolution

It will be fixed on 1H2026 release.

Symptom

 You seek information which GTK+ toolkit library version is required when running the graphical 'hdblcmgui' or 'hdbsetup'.

Other Terms

hdblcmgui, hdbsetup

Reason and Prerequisites

 When running 'hdblcmgui' or 'hdbsetup', you observe one of the following errors:

> ./hdblcmgui
SAP HANA Installer Runtime Error:
perl compilation error (loading LCM::Gui::App::LCMGuiApp): cannot show splash screen: Can't load '/hana/shared/SID/hdblcm/instruntime/Wx.so' for module Wx: libgtk-x11-2.0.so.0: cannot open shared object file: No such file or directory at DynaLoader.pm line 206.
 at SDB/Wx/Splash.pm line 45.
BEGIN failed--compilation aborted at SDB/Install/App/Gui.pm line 6.
BEGIN failed--compilation aborted at LCM/Gui/App/LCMGuiApp.pm line 6.

> ./hdblcmgui
SAP HANA Installer Runtime Error:
perl compilation error (loading LCM::Gui::App::LCMGuiApp): 
cannot show splash screen: Can't load '/hana/shared/SID/hdblcm/instruntime/Wx.so' for module Wx: libgtk-3.so.0: cannot open shared object file: No such file or directory at DynaLoader.pm line 206.
 at SDB/Wx/Splash.pm line 45.
BEGIN failed--compilation aborted at SDB/Install/App/Gui.pm line 6.
BEGIN failed--compilation aborted at LCM/Gui/App/LCMGuiApp.pm line 6.

Solution

Those graphical tools require the GTK+ toolkit library in version 2 up to - and including - the following HANA revisions:

  • HANA 2.0 SPS08 revision 88
  • HANA 2.0 SPS07 revision 79.07
  • HANA 2.0 SPS05 (any revision)

Starting with the following HANA revisions, they require the GTK+ toolkit library in version 3:

  • HANA 2.0 SPS08 revision 89
  • HANA 2.0 SPS07 revision 79.08

Please install an appropriate GTK+-library version:

  • On RHEL-systems:
    # yum install gtk<version>

  • On SLES-systems:
    # zypper install libgtk-<version>

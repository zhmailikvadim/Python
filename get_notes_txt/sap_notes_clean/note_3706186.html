Symptom

  • Relational data tables in Emarsys, sourced from SAP Customer Data Platform (CDP) via Direct Data Access (DDA) have disappeared.
  • Relational data segments relying on these tables are no longer functional.
  • Campaigns and automations dependent on these segments are impacted.
  • Historical configurations relying on these tables are broken.

Environment

SAP Customer Data Platform

Reproducing the Issue

  1. Create relational data tables in Emarsys sourced from SAP CDP using Direct Data Access (DDA).
  2. Delete the corresponding Purpose Views in SAP CDP console.
  3. Observe that the linked relational data tables disappear from Emarsys.

Cause

The deletion of Purpose Views in SAP CDP results in the removal of corresponding database views in Direct Data Access (DDA). This causes the linked relational data tables to disappear from Emarsys.

Resolution

  1. Recreate the deleted Purpose Views in SAP CDP using the exact same Direct Data Access (DDA) naming conventions, including:
  • Purpose view name
  • Schema aliases
  • Segment aliases
  • Indicator aliases
  • Processing Purpose used in the deleted Purpose View

      2. Ensure robust change management practices around Purpose Views to prevent accidental deletion or disabling.
      3. Note that disabling a Purpose View also results in the deletion of its corresponding database views. Restoration in this case is simpler—enabling the Purpose View will automatically recreate the associated database views.

Symptom

  • Enterprise authentication could not log you on. Please make sure your logon information is correct. (FWB 00008) after SAML configuration.
  • The error appears after logon on the IDP page.
  • Error messages observed in the CMS logs:

    "TrustedAuthMgr::getTrustedSystem: system not trusted: https://domainURL/BOE/CMC/TrustedAuthSetTokenServlet"
    "TrustedAuthMgr::CheckData: untrusted URL: https://domainURL/BOE/CMC/TrustedAuthSetTokenServlet"
    "TrustedAuthMgr::CheckData: Invalid trusted authentication data"

Environment

SAP BusinessObjects Business Intelligence (BI) Platform 4.x/2025

Reproducing the Issue

  1. Complete the trusted authentication configuration 3559381 - Configuring Trusted Authentication in 4.3 SP04 Patch 1000+ & BI 2025
  2. Complete the SAML configuration 2791348 - How To: Configure Front-End SAML Authentication on BI 4.3 / 2025
  3. Attempt to authenticate using SAML.
  4. Observe the error "Enterprise authentication could not log you on. Please make sure your logon information is correct. (FWB 00008)".

Cause

The URL "https://domainURL/BOE/CMC/TrustedAuthSetTokenServlet" was not configured as trusted in the Authorized URL Configuration in the Central Management Console (CMC).

Resolution

  1. Access the Central Management Console (CMC).
  2. Navigate to Applications -> Authorized URL Configuration.
  3. Add the URL "https://board-prod.tr.daimlertruck.com/BOE/CMC/TrustedAuthSetTokenServlet" to the trusted URLs list.
  4. Save the configuration changes.
  5. Restart the Tomcat server to apply the changes.
  6. Test the SAML authentication functionality to confirm resolution.

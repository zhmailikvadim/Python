Symptom

  • When using name format to generate general displayname in sequence of lastname firstname, there is a space in between.
  • This does not align with Chinese naming conventions.
  • How can this space be removed?  

Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.

Environment

SAP SuccessFactors Employee Central

  • Personal Information
  • General Display Name

Cause

This behavior is by design.  

Resolution

The space could be removed by following workaround.

  • Enable "preferred-name" field in Manage Business Configuration<personalInfo and grant edit permissions to this field.
  • Create a business rule with "Trigger onSave Rules for Hris Elements" scenario and configure it under Manage Business Configuration<personalInfo. Sample rule like below
  • Go to Manage Data, create a new Name Format to retrieve Preferred Name 
  • Configure the Name format in 'Configure Latest People Profile<General Settings' or 'Manage Data<Legal Entity<Name Format for General Display'.
    • The name format configured in 'Configure Latest People Profile<General Settings' will be applied to the entire instance, regardless of country or legal entities.
    • The name formate configured in 'Manage Data<Legal Entity<Name Format for General Display' will be applied to the employees that assign in this legal entity.
    • When name format is configured in both 'Configure Latest People Profile<General Settings' and 'Manage Data<Legal Entity', system will take advantage of the one which is configured in 'Configure Latest People Profile<General Settings'
  • Click "Refresh General Display Name" button in Company System and Logo settings.

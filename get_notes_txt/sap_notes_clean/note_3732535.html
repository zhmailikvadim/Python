Symptom

When you check the model in transaction PEST, the system displays the runtime error DATA_LENGTH_0 with Exception CX_SY_RANGE_OUT_OF_BOUNDS.

Other Terms

PEST, PUST, SCAN, SAPLHRST, HRPY_PROCESS_SCAN_PROG

Reason and Prerequisites

Reason:

The ABAP scan statement returns a value of 0 for variable length "LEN1" when the system performs a dynamic call using literal text, such as:

CALL METHOD ('CL_H4S4_ACCESS_BUSINESS_FUNC')=>('CHECK_H4S4_SFWS_SC_01').

Prerequisites: You have a process model with a step that is defined as a payroll program.

Solution

After implementing this SAP Note, the system correctly handles the variable length in the ABAP scan statement during dynamic method calls. The system uses constant names for the class and method. Runtime error DATA_LENGTH_0 with Exception CX_SY_RANGE_OUT_OF_BOUNDS no longer occurs when you check the process model in transaction PEST.

Symptom

  1. Only 500 items are loaded when executing the query (even if there are more entries maintained in the system). This screen displays maximum 500 records. Use advanced search for specific results.
  2. There is no basic search to get the price for a product, you always have to use the advanced search.
  3. To search a single product, you have to use the advanced search. However the field Product id is not visible in the advanced search until the query is executed.

Environment

SAP Hybris Cloud for Customer

Cause

The prices view is based on some special technical objects due to which we are unable to provide a simple/ basic search.
We understand the pain points and are looking into ways to improve the usability like trying to remove the execute query step before having all fields available.

  1. The list showing only 500 data records this has been implemented for performance reasons, such that the query is executed fast enough. Also, if we increase the data to a higher number of rows displayed, you will still have to filter out the specific price you are looking for. Therefore we have provided the advanced search which can be used to filter out the specific records.
  2. Providing a basic search is a long term topic and we have added this to our backlog for analysis.
  3. Regarding the product ID field being available only after execute query - once we remove the execute query step if the product ID field, or other fields like sales area and so on, if they are part of the first access sequence you have maintained in the system, this field shall appear automatically in the advanced search and the user shall be able to filter down the product price based on these.

Resolution

This is the expected system behavior.

Keywords

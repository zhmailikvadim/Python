Symptom

When running the ATC Check Consistency of Software Component Relations (SCR_CONSISTENCY, Implementation Class CL_SCR_ATC_CONSISTENCY), false positives are reported for objects of type Behavior Definition (object type BDEF).

A false positive ATC finding is raised with the message "<Object Type> <Object Name> of software component <Software Component> is used without declared dependency", although the dependency was already declared.

This happens although you've applied the corresponding note 3687433 already.

Other Terms

Behavior Definition, BDEF, Software Component Relations, ATC, SCR_CONSISTENCY, CL_SCR_ATC_CONSISTENCY

Reason and Prerequisites

The false positive ATC findings occur because Behavior Definitions are not handled correctly by the ATC Check.

As a prerequisite for this note, you've applied the note 3687433 already.

Solution

Implement this note or install support package 2 of software release 2025.

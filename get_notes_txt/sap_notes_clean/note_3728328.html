Symptom

  • An error occurs while sending PDS data from ECC to IBP through Real time integration (RTI)
  • Error message: PDS activity/resource: Duplicate master data records in input data.

Environment

SAP Integrated Business Planning for Supply Chain

Reproducing the Issue

  1. Send a PDS from ECC to IBP through via Real time integration (CFM1/CFM2/CFM9 or CURTO_CREATE_IBP)
  2. Observe the error messages in the Message Monitoring 
  3. Review the errors "Duplicate master data records in input data"

Cause

The issue arises because there are duplicate key/root attribute values in the data being sent to IBP. These duplicate records are being introduced through the custom BAdI implementation.

Resolution

Example:

The Integration profile for PDS activity requires 4 key fields:

ACTID (PDS Activity ID)
LOCNO (Ext.Loc. Number)
MATNR (Ext. Mat. Num. (DB))
VERID (Prod. Version)

When sending the PDS and modifying data via Badi Method go_badi_change_pds->change_pds make sure there are no duplicate entries for ET_SOSACT created considering the key fields.

Example: 

MATNRLOCIDVERIDACTIDSEQNR
MAT1LOC11000101
MAT1LOC11000102

Sending a PDS with fields such as cannot be transferred as all 4 key fields are identical.

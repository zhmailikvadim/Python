Symptom

  • In transaction /SAPAPO/CDPS_CHARMAT or /SAPAPO/CDPSC7 the Input Help (F4) for field Unit of Measure shows the same UoM twice
  • For example, unit 'Hours' could show up as 'H' and 'HR' separately

Environment

  • SAP S/4HANA
  • Production Planning and Detailed Scheduling (PP/DS)

Reproducing the Issue

  1. Navigate to the transaction /SAPAPO/CDPS_CHARMAT.
  2. In the unit field, open the F4 help.
  3. Observe that some units of measure are displayed multiple times

Cause

Standard behavior

Resolution

  • The Input Help shows the Units of Measure maintained for the system in transaction CUNI (BC-SRV-ASF-UOM)
  • The behavior is not PP/DS-specific - if multiple entries are maintained for the same UoM in CUNI, other (completely unrelated) transactions in the system will show duplicates as well

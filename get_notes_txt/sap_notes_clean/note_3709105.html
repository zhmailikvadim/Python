Symptom

  • I want to manually set the date format on my file name in the script section of a new end point 
  • I want to manually change the order of my date on my file name in the script section of a new end point
  • I want to manually use a 12 hour clock on my time stamp of my file name in the script section of a new end point
  • I want to manually use a 24 hour clock on my time stamp of my file name in the script section of a new end point

Resolution

When populating the File name section of a new end point, you can manually enter the datetime format string

 

Example:

My_file_name_${__datetime:yyyyMMdd__HHmmss}.csv

 

The order of the date format can be adjusted by changing the order of the characters (ex. yyyyMMdd vs ddMMyyyy)

  • yyyy represents a 4 character year string and should be lower case
  • yy represents a 2 character year string and should be lower case
  • YYYY represents a 4 character "week year" string and should be upper case and is only different from yyyy on the first/last week of the year if the week is split between two years
  • YY represents a 2 character "week year" string and should be upper case and is only different from yyyy on the first/last week of the year if the week is split between two years
  • MM represents a 2 character month string and should be upper case
  • dd represents a 2 character day string and should be lower case
  • HH represents a 24 hour clock hour string and should be upper case
  • hh represents a 12 hour clock hour string and should be lower case
  • mm represents a 2 character minute string and should be lower case
  • ss represents a 2 character second string and should be lower case
  • EEE represents a 3 character abbreviation of the day of the week and should be upper case

 

Note: It is not recommended to use mm (minutes) or ss (seconds) in the end point format string for a file name of an upload. The minutes and seconds can be in the file name, but should be a wild card (*) in the end point.

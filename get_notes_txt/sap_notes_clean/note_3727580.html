Symptom

  • OAuth token requests from CPI to SAP Commerce return response with 400 code.
  • On SAP Commerce Cloud side following can be found via Dynatrace:
Exception:
org.springframework.security.oauth2.core.OAuth2AuthenticationException
Message:
OAuth 2.0 Parameter: grant_type
 
  • On CPI side following error is shown:
com.sap.gateway.core.ip.component.odata.exception.OsciException Error in retrieving Authorization header IllegalArgumentException: Status code: 400 { "error_description": "OAuth 2.0 Parameter: grant_type", "error": "unsupported_grant_type" }".

Environment

SAP Commerce Cloud

Cause

The potential reason of the error is that the grant type information is being sent as part of the URL instead of part of the request body.

Resolution

  1. Identify OAuth2 Client Credentials used by the affected iFlow in CPI.
  2. Edit credentials by setting grant type to "Send as part of the request body".
  3. Refer to the following KBA for additional potential misconfiguration: https://me.sap.com/notes/3467343/E.

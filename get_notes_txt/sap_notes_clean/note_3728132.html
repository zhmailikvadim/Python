Symptom

This KBA explains the root cause and solution for the generation of incorrect IR8A additional files with negative amounts.

Environment

  • Human Capital Management (HCM)
  • SAP ERP Core Component
  • SAP ERP
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA

Reproducing the Issue

  1. Generate the original IR8A form and update the cluster.
  2. Run program RPCT8AR0_01 for employees again without making any changes to the employee data or income.
  3. Additional forms for employees who have appendix 8A or 8B files are generated, showing the same amounts as the original form but as negative values.

Cause

  • The original form cluster was updated incorrectly with two entries (this can only be checked in runtime debugging mode). The first entry contains only the IR8A main page data, while the second entry contains both the IR8A main page and appendix A/B data.
  • The wage type with D9 configured for evaluation class 7 is still in use.

Resolution

This incorrect original form cluster update behavior was fixed by SAP note 3680783 - IR8A YOA2026: Continuous Enhancement of API Submission and PDF. 

  • To resolve this issue:
    1. Delete the incorrect cluster data by running the program RPDELCR0.
    2. Implement the note 3680783.
    3. Update the cluster for the original form again.
  • If deletion of the original form cluster data is not possible, generate the file with the revised version by following KBA 3580350, which explains how to use the "Generate revised report" checkbox in Singapore IR8A generation.
  • If generating the additional form is still necessary, work with the local team to correct the imported original cluster data in runtime by removing the incorrect entry that contains only the IR8A main page data.

Evaluation class 7 value D9 should no longer be used. The relevant amount will be merged into item "D4 - Pension/Retirement Benefits Accrued from 1993 (Other than CPF Benefits)."

To generate the additional form for this scenario, ensure note 3730249 - IR8A: Correction for Additional File has been implemented.

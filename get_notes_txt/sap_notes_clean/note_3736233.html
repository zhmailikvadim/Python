Symptom

  • During cutback, duplicate entries are created.
  • After reversing a document with FB08 due to duplicate line items, a subsequent GJCB run does not pick up any items for posting.

Environment

  • Joint Venture Accounting (JVA)
  • Financial Accounting (FI)
  • SAP ERP
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA

Reproducing the Issue

  1. Run GJCB using the relevant variant.
  2. Reverse a posted cutback document using FB08 due to duplicate line items.
  3. Rerun GJCB and observe that no items are selected for posting.
  4. Reverse the same document using GJCB_REV.
  5. Run GJCB again and observe that the document is now selectable; however, other newly added items may still not be picked up.

Cause

  • User handling errors led to inconsistencies: the cutback job was interrupted, causing incomplete postings and potential duplicates.
  • Cutback documents were reversed with FB08 instead of GJCB_REV, leaving related entries unreversed and preventing proper selection in subsequent GJCB runs.

Resolution

  1. Use GJCB_REV to reverse cutback documents; do not use FB08 for standard cutback reversals.
  2. Ensure cutback jobs are not interrupted; schedule and monitor jobs to complete successfully.
  3. Consider implementing the program correction in the SAP Note 3618599 to prevent duplicate cutback postings.
  4. If a previous cutback run was interrupted and certain documents are missing in GJCB_REV (for example, only partial postings exist), analyze the related postings and reverse incomplete documents appropriately. Use FB08 only for those incomplete cases where GJCB_REV cannot process them.
  5. For adjustment postings intended for cutback, verify that debit and credit do not net to zero at account and RECID level; otherwise, GJCB will not process the items.
  6. After applying the above, rerun GJCB and confirm that intended items are selected and no duplicates are created.

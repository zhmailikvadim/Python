Symptom

  • For the same HU number, it appears that the Handling Unit was archived, but the Warehouse Task was not archived in logs.
  • Error message can be checked in SLG1 tcode:
    • /SCWM/HU_WM103 - Handling unit archived
    • /SCWM/HU_WM107 - Warehouse task &1 is not archived
  • The HU still exists in both the standard EWM tables and the custom ZARIX table.  

Environment

  • SAP S/4HANA
  • SAP Extended Warehouse Management

Cause

The DCO status of the Delivery Order associated with the Warehouse Task is not 9, which prevents the Warehouse Task from being archived.  

A Warehouse Task can only be archived if all referenced delivery documents (e.g., Inbound Delivery or Outbound Delivery Order) are completed, meaning the DCO (Document Completion) status on the document level must be 9. 

The HU number was successfully archived because it is associated with a different Warehouse Task under a different Delivery, which already has a DCO status of 9.  

Resolution

  1. Ensure that the DCO status of the delivery is updated to 9. 
  2. Once the document is fully completed, perform the WME_TO archiving again.

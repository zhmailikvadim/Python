Symptom

  • System raises error:

    “Unsuccessful: Modifying infotype 2006 with key xxxx – No entry in table T554Y_REAC.”

  • The issue may occur in multiple reports or transactions in Time Management, for example:

    • RPTQTA00 – Generate Absence Quotas / Transfer quotas to IT2006

    • RPTIME00 – Time Evaluation

    • Creating or updating time infotypes such as:

      • IT2001 (Absences)

      • IT2006 (Absence Quotas)

      • Other time-related infotypes

  • Table T554Y_REAC is empty or missing required entries.

  • System operations related to quota transfer, time evaluation, or time data maintenance fail.

  • Issue typically occurs after activating business function H4S4_1 or implementing HR Support Packages.

  • Issue does not occur in systems where the business function is not activated.

Environment

  • SAP S/4HANA 
  • SAP Human Capital Management (HCM) 
  • Personnel Time Management (PT)
  • Time Data Recording and Management

Reproducing the Issue

  1. Activate business function H4S4_1.

  2. Implement latest HR Support Packages.

  3. Execute one of the affected reports (for example PC00_M13_CTO3, RPTQTA00, RPTIME00) or create/update time infotypes (e.g. IT2001, IT2006).

  4. System raises error indicating missing entry in table T554Y_REAC.

  5. Check table T554Y_REAC → table is empty or not maintained.

Cause

  • Required entries are missing in collision table T554Y_REAC.

  • The table is client-specific and not transportable.

  • After activating business function H4S4_1, system configuration was not converted to the new table structure.

Resolution

  • Maintain required entries in customizing view V_T554Y_MOCOL (for example default value “00”, depending on business requirements).

  • Execute report RPU_PT_CONV_T554Y to populate table T554Y_REAC.

  • Run the report separately in each system and client (DEV, QAS, PRD if applicable), since the table is not transportable.

  • Re-test the affected report or infotype processing.

Symptom

You are using the overpayment capability logic during Payment Distribution by executing an overpayment and there are future billing items or uncleared receivables at FICA-system.

In such cases the overpayment capability logic uses a wrong effective capital amount which can lead to negative receivables or a refund loop. 
After implementation of this note, the overpayment capability logic uses a corrected amount for curtailment which takes the uncleared receivables or future billing items into account.
The payment distribution item will be transferred to post processing in such cases.

See also note 3707766 - "New capital for calculation of Overpayment Capability".

Other Terms

Refund Loop, capital 3111, capital 3110, effective capital, curtailment

Reason and Prerequisites

Program error

Solution

Import the relevant Support Package that is specified for your release.
As a manual advance correction, you can implement the attached correction instructions using transaction SNOTE.

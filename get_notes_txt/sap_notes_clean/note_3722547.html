Symptom

  • Crystal Reports using Sub-Reports do not save the prompts while scheduling from BI launchpad
  • Reports fail with 'Information is needed'
  • The issue doesn't occur in CMC. The successful instances ran from CMC works when rescheduled from BI Launchpad.

Environment

  • SAP BusinessObjects Business Intelligence (BI) Platform 4.x / 2025
  • SAP Crystal Reports 2020 / 2025
  • Windows

Reproducing the Issue

  1. Login to BI launchpad.
  2. Select any Crystal Reports containing Sub-reports with prompts.
  3. Right click and select Schedule.
  4. Select "Reports Features" and add the Prompt values. Observe that the prompts are not getting saved.

Cause

The prompt group ID is the same between the different sub report. When the report was created, it's duplicated, and the report name was changed, then inserted to the main report. Since the prompt ID is the same, Prompt page regards them as the same prompt, so merge them into one for input.

Resolution

The report can be fixed in Crystal Reports client tool:

  1. Double click at the subreport, and open the parameter in the sub report
  2. Change something such as value list or value option then revert the change. 
  3. Save the report, new prompt group id would be renewed in the background for the modified subreport.
  4. Do the same changes for any other subreports
  5. Upload the report to the BI Platform 
  6. Schedule it in BI Launchpad

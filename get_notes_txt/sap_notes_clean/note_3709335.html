Symptom

  • Error message appears when trying to create a formula with a field that contains a 'NEWLINE' (line break, enter) in the name in Crystal Reports designer:

    The field name is not known.

  • In Crystal Reports for Enterprise replacing the NEWLINE with a space resolves the issue, but in Crystal Reports it does not recognize the field with a space.

Other Terms

crystal bw new line break enter field name not known

Reason and Prerequisites

Environment

  • SAP Crystal Reports 2025 Patch 7
  • SAP Crystal Reports 2020 SP5 Patch 3
  • SAP BW

Steps to Reproduce

  1. Open a Crystal report.
  2. Create a new formula.
  3. Add a field which contains a 'new line' in the name.
    It looks like this in the formula editor:

    {New field
    test}

  4. Check if the formula is valid. (top left 'x-2' icon or ALT+C)
  5. Error appears: 'The field name is not known.'
  6. Try to use space instead of a line break/enter for the field.
  7. In this case Crystal does not recognise the field.

Cause

  • This is a feature gap between Crystal Reports 20xx and Crystal Reports for Enterprise.

Solution

  • Crystal Reports 20xx does not support line break in formula and has no logic to convert line break to space.
  • As a workaround modify the field name to remove line break and replace it with a space.
  • There is an enhancement request exists and you can vote for this idea via link: https://influence.sap.com/sap/ino/#/idea/368765

Symptom

  • When updating a position or participant record through the UI, the effective end date in CS_GAPOSITION and CS_GAPARTICIPANT is not updated correctly, leading to overlapping effective dates.
  • This issue does not occur when Position records are uploaded using XDL files.

Environment

  • SAP Sales Performance Management
    • Incentive Management on Hyperscalers

Reproducing the Issue


  1. Create a Position or Participant record.
  2. Generate a version for the record.
  3. Modify any generic attribute in the version through the UI.
  4. Observe that two records are created in CS_GAPOSITION or CS_GAPARTICIPANT with overlapping effective dates.

Cause


A defect in the current version of the application.

Resolution

  1. A code correction has been developed to address this issue under COM.
  2. The fix is scheduled for implementation in the 2605 release.
  3. As an interim solution, use XDL templates to edit versions.

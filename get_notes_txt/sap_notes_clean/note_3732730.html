Symptom

SAP Product Lifecycle Costing 4.0 SP9 Hotfix 2 is released and you want to install it or upgrade from a lower version.

Solution

Release Notes/Changelog:

Bug Fixes

  • Corrected an issue where certain calculated fields were not refreshed when caching was used. The calculation logic now consistently populates all required values from AFL results.

  • Resolved a problem with price‑based filtering where valid items were not returned. Backend price handling was aligned with AFL values, and existing data is updated via a post‑install step.

  • Addressed an application crash that occurred when updating project dates to a completely new, non‑overlapping year range. Lifecycle period processing now safely handles removed and missing periods.

  • Improved robustness of Custom Field processing to prevent the system from entering an unrecoverable state if Custom Field generation is interrupted during long‑running operations.

  • Enhanced stability and performance of Custom Field creation by correcting locking and cleanup order, properly clearing temporary caches, and reducing the impact of expensive updates on very large datasets.

  • Fixed database connection leaks in the XSJS layer that could lead to connection pool exhaustion. Connections are now reliably closed after use, improving stability under load.

  • Ensured item tracking is consistently enabled during Custom Field updates, preventing scenarios where tracking could remain unintentionally disabled.

  • Prevented duplicate or incorrect item numbering by stabilizing item creation logic and eliminating timing‑related conflicts.

  • Optimized several metadata update operations that previously caused high memory consumption and long runtimes on large tables.

  • Resolved failures during transportation import in replace mode by preventing conflicts between background cleanup tasks and deployment steps, avoiding deadlocks and connection resets.

  • Fixed a crash on the home screen triggered during navigation when opening calculation versions with filters by safely handling missing data.

  • Eliminated a copy/paste crash that could occur when moving items between different calculations by stabilizing UI refresh timing.

New Installation and Upgrade:

Follow these instructions when installing PLC for the first time or upgrading an existing version.

Note: Whether you’re installing PLC for the first time or upgrading, make sure that your system includes package Glibc-2.29 or higher to ensure that the PLC AFL Library loads properly. The minimum supported version of the SUSE Linux Enterprise Server Distribution (SLES) for this package is SLES 15 SP3.

New Installation of PLC

If you’re installing PLC for the first time, you don’t need to install any previous versions. Install 4.0 SP9 Hotfix 2 and follow the instructions in the Administrator's Guide for SAP Product Lifecycle Costing under Installation Steps and Checks.

Make sure you've installed the following HANA DB and component revisions:

Revisions between and including 2.00.79.02 (SPS07) - 2.00.089.1 (SPS08) for HANA DB 2.0 
HANA Component EPMMDS Rev 79.02 only for HANA 2.0 Rev 79.02 - EPMMDS Rev 89.1 only for HANA 2.0 Rev 89.01
XS Advanced Runtime for SAP HANA, revisions between and including 1.4.0 - 1.4.7 (SAP EXTENDED APP SERVICES 04)

Importing from SAP ERP: If you plan to import data from an SAP ERP system, apply these SAP Notes:

SAP Note 2276456 and SAP Note 2278609 for BoM & Routing import.
SAP Note 2281004 and SAP Note 2281007 for document structure import.

Upgrading PLC

Before starting any upgrade activity check the status of the entries in the installation log.

You can access the installation log in the PLC database table <SCHEMA_NAME>.sap.plc.db::basis.t_installation_log.

If the result of the query: select count(*) from "<SCHEMA_NAME>"."sap.plc.db::basis.t_installation_log" where state not in ('running','started','finished') is not 0 do not start the upgrade and contact PLC support.

If you’re upgrading from PLC 4.x.x to a newer version, please follow the Administrator’s Guide for SAP Product Lifecycle Costing under Upgrade an Existing Installation.

Consider the following about SAP HANA:

When upgrading, if you're already using SAP HANA SPS05 (stable) or SAP HANA SPS06 and XSA revision 1.0.155, you can upgrade to SAP HANA SPS07/SPS08 and to XSA revision 1.4.x

Upgrading your SAP HANA System

If you need to update your SAP HANA system from SAP HANA 1.0 to SAP HANA 2.0, choose the scenario that works best with your operating system:

Note: You can find the complete list of operating systems that support SAP HANA 2.0 in the SAP Note 2235581.

  1. If your operating system supports SAP HANA 2.0 (e.g. SUSE Enterprise Linux version 12.1 or later), do the following:
    • Upgrade your existing SAP HANA installation according to the SAP HANA Server Installation and Update Guide.
    • Upgrade SAP Product Lifecycle Costing using the information provided in the Administrator's Guide for SAP Product Lifecycle Costing under under Upgrade an Existing Installation.
  2. If your operating system does not support SAP HANA 2.0 (e.g. SUSE Enterprise Linux version 12.0 or earlier), do the following:
    • Upgrade your current operating system to a version supported by SAP HANA 2.0. Additional information regarding the update of your current operating system can be found in the SAP HANA Master Guide (SAP HANA 2.0 SPS00). You can find the complete list of operating systems that support SAP HANA 2.0 in SAP Note 2235581.
    • Upgrade the SAP HANA database and SAP Product Lifecycle Costing as described in the first scenario.
  3. If upgrading your current operating system is not possible, you can:
    • Set up a new system with the corresponding operating system.
    • Clone the current SAP HANA installation to the new operating system. For more information see Copy or Clone an SAP HANA System in the SAP HANA Administration Guide.
    • Upgrade the SAP HANA database and SAP Product Lifecycle Costing as described in the first scenario

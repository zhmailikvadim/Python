Symptom

  • The Tomcat Java Options have been changed significantly from their installed values, and resetting them to the default is necessary.

Environment

  • SAP BusinessObjects Business Intelligence platform 2025
  • Windows / linux servers
  • SAP Java Machine 17 or above
  • Apache Tomcat 9.x for BI2025

Cause

  • BI2025 uses SAPJVM 17 version of "SAP Machine 17" which is different from Oracle Java, but based on OPENJDK.
  • SAPJVM used by BI 2025 does NOT have a "jre" folder, so all paths are directly under sapjvm/bin or sapjvm/lib

Resolution

NOTE: Never use configuration files or startup scripts from older versions. Always use the latest current version of these files/scripts, ensuring to update any custom options pertinent to your environments.     

On Windows servers, the default Tomcat settings are as follows:

JAVA Virtual Machine:

C:\Program Files (x86)\SAP BusinessObjects\SAP BusinessObjects Enterprise XI 4.0\win64_x64\sapjvm\bin\server\jvm.dll

JAVA Classpath:

C:\Program Files (x86)\SAP BusinessObjects\tomcat\bin\bootstrap.jar;C:\Program Files (x86)\SAP BusinessObjects\tomcat\bin\tomcat-juli.jar;C:\Program Files (x86)\SAP BusinessObjects\SAP BusinessObjects Enterprise XI 4.0\win64_x64\sapjvm\lib\sapjce.jar

JAVA Options:

-Dfile.encoding=UTF-8
-Dcatalina.base=C:\Program Files (x86)\SAP BusinessObjects\tomcat\
-Dcatalina.home=C:\Program Files (x86)\SAP BusinessObjects\tomcat\
-Dbobj.enterprise.home=C:\Program Files (x86)\SAP BusinessObjects\SAP BusinessObjects Enterprise XI 4.0\
-Xrs
-XX:MaxMetaspaceSize=1024M
-Djava.awt.headless=true
-XX:+HeapDumpOnOutOfMemoryError
-Xlog:gc:C:\Program Files (x86)\SAP BusinessObjects\tomcat\logs\tomcat.gc.log
-XX:+PrintGCDetails
-XX:+UseG1GC
-Djdk.tls.client.protocols=TLSv1.2

Java 9 options:

--add-opens=java.base/java.lang=ALL-UNNAMED
--add-opens=java.base/java.io=ALL-UNNAMED
--add-opens=java.rmi/sun.rmi.transport=ALL-UNNAMED
--add-opens=java.base/java.net=ALL-UNNAMED
--add-opens=java.management/com.sun.jmx.remote.internal=ALL-UNNAMED
--add-opens=java.management/com.sun.jmx.remote.security=ALL-UNNAMED
--add-opens=java.management/com.sun.jmx.remote.util=ALL-UNNAMED
--add-opens=java.base/sun.security.internal.spec=ALL-UNNAMED
--add-opens=java.base/sun.security.x509=ALL-UNNAMED

Default Values:

Initial memory pool: 256
Maximum memory pool: 4096
Thread stack size: 1024

Please note that these are the default settings, and your particular environment may differ or require different configurations to optimize performance, according to KBA 2405536.

On LINUX Servers

  • On default bundled Tomcat, Java options are configured in <install_directory>/sap_bobj/tomcat/bin/bobjenv.sh 
  • On non-bundled custom install Tomcat, Java options can be configured either at the bottom of <tomcat install>/tomcat/bin setenv.sh or catalina.sh 

###The below Tomcat Java options and classpath load Tomcat to start up with essential environment variables and Java options to run BI 2025 webapps with SAP Java 17. The non-highlighted options are default 

# set the JAVA_OPTS for tomcat
JAVA_OPTS="-Dfile.encoding=UTF-8 -Dbobj.enterprise.home=${BOBJEDIR}enterprise_xi40 -Djava.awt.headless=true -Djava.net.preferIPv4Stack=false -Xmx4096m -XX:MaxMetaspaceSize=1024m -XX:+HeapDumpOnOutOfMemoryError -Xlog:gc:${BOBJEDIR}tomcat/logs/tomcat.gc.log -XX:+UseG1GC --add-opens=java.base/java.lang=ALL-UNNAMED --add-opens=java.base/java.io=ALL-UNNAMED --add-opens=java.rmi/sun.rmi.transport=ALL-UNNAMED --add-opens=java.base/java.net=ALL-UNNAMED --add-opens=java.management/com.sun.jmx.remote.internal=ALL-UNNAMED --add-opens=java.management/com.sun.jmx.remote.security=ALL-UNNAMED --add-opens=java.management/com.sun.jmx.remote.util=ALL-UNNAMED --add-opens=java.base/sun.security.internal.spec=ALL-UNNAMED --add-opens=java.base/sun.security.x509=ALL-UNNAMED"
export JAVA_OPTS

### The below loads sapjce crypto libraries for encryption/decryption 
CLASSPATH="$CLASSPATH":$JRE_HOME/lib/sapjce.jar
export CLASSPATH

Note: A sample bobjenv.sh file for BI 2025 is attached. Options can change with version updates.  The attached file does not exist in custom installs, but the options can be helpful. 

Note: The Java 9-Options for add opens may show differently in translated versions -  it could be shown wrong (original: --add-opens, german: --addopen, italian: --add-open, etc.). Please ensure you are using the correct English format else the JVM can throw an unrecognized option error. Always ensure the option is checked in English format according to Java documentation.

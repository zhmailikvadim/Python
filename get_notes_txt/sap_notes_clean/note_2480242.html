Symptom

  • This is SAP Adaptive Server Enterprise (ASE) high-availability and disaster recovery (HADR) system.
  • Have successfully done the failover from Database Node A to Database Node B.
  • When run "sap_host_available" command in Replication Management Agent (RMA), facing the error in SAP Replication Server (SRS) errorlog :  

    User 'DR_admin' writes: '*******************************************************************************'
    User 'DR_admin' writes: '** HADR operation: sap_host_available XXXX,'
    User 'DR_admin' writes: '** Message: is starting'
    User 'DR_admin' writes: '*******************************************************************************'
    ERROR #14098 USER(XXXXX) - neric/exec/execint.c(3148)
    The Replication Server is in hibernation mode.
    User 'DR_admin' writes: '*******************************************************************************'
    User 'DR_admin' writes: '** HADR operation: sap_host_available XXXXX,'
    User 'DR_admin' writes: '** Message: failed'
    User 'DR_admin' writes: '*******************************************************************************'
    ERROR #14098 USER(XXXX) - neric/exec/execint.c(3148)
    The Replication Server is in hibernation mode.

Environment

  • SAP Adaptive Server Enterprise (ASE) 15.7.1 SP13X
  • SAP Adaptive Server Enterprise (ASE) 16.0 SP02 or higher
  • SAP Replication Server (SRS) 15.7.1 SP 13X
  • SAP Replication Server (SRS) 15.7.1 SP304 or higher

Cause

The replication server can not get out of hibernation mode while waiting for suspending connections.

Resolution

  1. Login the replication server
  2. Turn off hibernation mode
    sysadmin hibernate_off
    go
  3. Login RMA
  4. Rerun sap_host_available command
    sap_host_available <primary_logical_host_name>
    go

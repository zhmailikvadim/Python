Symptom

The error "No value mapping found for value X of EC data type NON_REGD_RSN_SAU (Mapping Mode S)" occurs when the system fails to find a corresponding value mapping for the specified data type and mapping mode during PTP replication processing.

Environment

  • SAP SuccessFactors Employee Central Payroll
  • Employee Master data replication

Reproducing the Issue

  1. Run report RP_HRSFEC_PTP_EE_REPLICATION
  2. Select a variant or fill the mandatory entries
  3. Check the logs in SLG1

Cause

The issue arises due to a mismatch in the data type mapping for the field NON_REGD_RSN_SAU in the T77SFEC_CVMAPS table.
Specifically, the field genericString24 is receiving a different data type than expected, leading to inconsistencies in data processing and potential errors during integration or data transfer.
This discrepancy may be caused by incorrect configuration or mapping settings in the table.

Resolution

To resolve this, adjusting the value mapping configuration to include the necessary mapping for the value coming from the generic fields to match with the values contained in the table T77SFEC_CVMAPS for data type  NON_REGD_RSN_SAU.

Or adjust the configuration on T77SFEC_CVPROPS to use the table T77SFEC_CVMAPC instead.

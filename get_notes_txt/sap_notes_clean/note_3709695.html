Symptom

When an application using ALV IDA calls FREE within a process action, the following runtime dump occurs:

Category ABAP programming error
Runtime Errors OBJECTS_OBJREF_NOT_ASSIGNED_NO
Exception CX_SY_REF_IS_INITIAL
ABAP Program CL_SALV_GUI_GRID_CONTROLER_IDACP
Application Component BC-SRV-ALV
Program error Access using a 'ZERO' object reference is not possible.

This happens whenever FREE is called for the ALV IDA instance that is currently being processed.

Other Terms

ALV; ABAP List Viewer; SAP List Viewer; Grid; List; CL_SALV_GUI_GRID_CONTROLER_IDA->ON_PROCESS_ACTION; CL_SALV_GUI_GRID_CONTROLER_IDA->IF_SALV_GUI_GRID_FWK_RECEIVER~PROMPT_NOTIFICATION_ON_CHANGE; MO_GUI_GRID_MODEL; MO_FWK_OBSERVER;

Reason and Prerequisites

Reason

The dump occurs because the ALV IDA object is no longer available after the FREE call.

Once the process action finishes, the grid controller continues with internal post‑processing in order to update the grid state if the action may have modified it.

However, because the grid object was already freed, further access to it results in a null‑reference dump.

Prerequisites

The following Support Package level is the minimum prerequisite:

Release Support Package
SAP_UI 816 SP 00
SAP_UI 758 SP 02

When you install an SAP Note, some prerequisites might be required. The prerequisites are SAP Notes that have already been released and that change the same context of the current one. Then, even if these prerequisites are not necessarily related to the symptom of this SAP Note, you must implement them due to technical reasons.

Solution

The grid controller now skips the remaining post‑processing steps if the ALV IDA object reference is no longer available. This prevents the null‑reference dump from occurring after the object has been freed.

Symptom

When logging into Backoffice with a specific user, the following error appears:

"Could not execute search"

Additionally, the following warning can be found in the OpenSearch logs:

"WARN","loggerName":"com.hybris.backoffice.widgets.advancedsearch.engine.AdvancedSearchEngineController","message":"Current user cannot read type WorkflowAction"

"Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental." 

Environment

SAP Commerce Cloud

Reproducing the Issue

  • Login to Backoffice
  • The error message "Could not execute search" is appearing

Cause

The issue occurs due to missing read permissions for the WorkflowAction type.

The affected user does not have sufficient access rights to perform searches involving workflow-related data, which results in the Backoffice search failure.

 

 

Resolution

Grant the required permissions for the affected user.

After assigning the correct permissions, the error should no longer occur.

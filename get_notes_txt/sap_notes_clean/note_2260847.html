Symptom

  • After restoring the domainlog.db and .log file.
  • SMP Server log :

2015-12-18 09:29:10.748 WARN MMS main [com.sybase.sup.admin.audit.dao.AuditDao]
[] [] Exception occur when log the audit data:
com.sybase.jdbc4.jdbc.SybSQLException: SQL Anywhere Error -141: Table 'audit_record' not found
 at com.sybase.jdbc4.tds.Tds.processEed(Tds.java:4084)

com.sybase.jdbc4.jdbc.SybBatchUpdateException: JZ0BE:
BatchUpdateException: Error occurred while executing batch statement:

SQL Anywhere Error -143: Column 'errorMessage' not found
SQL Anywhere Error -143: Column 'syncgroupId' not found
SQL Anywhere Error -141: Table 'mms_syncgroup_time_info' not found

Environment

  • SAP Mobile Platform 2.3 SP06

Cause

  • The audit_record table is for a new feature in SMP 2.3 SP05, the table will be created during SMP 2.3 SP05 upgrade, if you replaced *.db and *.log with the files saved from a previous version, for example SMP 2.3 SP04, then you will encounter this table missing issue after file replacement.

Resolution

  • C:\SAP\MobilePlatform\Servers\UnwiredServer\config\upgrade-monitordb-tables-asa.sql

The SQL to upgrade the monitordb is saved in this file, you can run it in dbisql.exe ( Interactive SQL ) against the monitordb, this SQL will check if the required table exists or not, if not, it will create the table accordingly.

  • C:\SAP\MobilePlatform\Servers\UnwiredServer\config\upgrade-domainlogging-tables-asa.sql

If you also replaced domainlogdb.db and domainlogdb.log from an old version of SMP, also run upgrade-domainlogging-tables-asa.sql against domainlogdb in dbisql.exe.

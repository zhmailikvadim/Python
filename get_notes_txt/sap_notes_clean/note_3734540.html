Symptom

After implementing Onboarding 2.0, you noticed that onboardees are now replicating to the ERP system through the standard Master Data replication package in Boomi.

Environment

  • SAP SuccessFactors HCM Suite
  • Boomi
    • Packaged Integration: SuccessFactors Employee Central to ERP Employee Data Replication

Cause

Expected behavior.

Resolution

This standard package uses Compound Employee API to retrieve the data from SuccessFactors.

In Compound Employee API, the only filter capable of filtering out onboardees is the hiringNotCompleted filter. For more details about it, refer to:

  • HiringNotCompleted Employment Filter | SAP Help Portal
  • Select Condition HiringNotCompleted | SAP Help Portal

However, this filter was not implemented in the standard Boomi packages, so onboardee records cannot be filtered out in such scenarios.

See Also

Refer to the note below taken from 2684991 - Middleware Content for SuccessFactors Employee Central Integration with ERP (PA_SE_IN Add-On):

  • Only the most important new features and legal changes are provided for the Boomi integration packages, whereas the SAP Cloud Platform Integration content will be supplied with all new features.
  • New integrations, such as the employee master data and organizational assignment bundle replication, are only provided on SAP Cloud Platform Integration. They will not be made available on Boomi.

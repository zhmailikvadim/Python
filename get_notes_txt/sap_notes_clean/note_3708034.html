Symptom

1.When the Single Touch Payroll Phase 2 report (T‑Code PC00_M13_STP2_GEN) is executed, an incorrect tax treatment code is generated for an employee who has an ATO‑approved downward variation of 0%.

2.When processing payroll for an employee whose IT0817 "Salary & Wages Tax %" field contains "0.00", the system calculates the tax incorrectly.

Current Behavior:

When entering 0.00 in IT0817, the Salary & Wages Tax % field remains blank, and the Field P0817‑TAXPC defaults to "999". Because of this default value (999), tax treatment code DVXXXX is not reported. The table below shows IT0817 data along with the corresponding P0817‑TAXPC field value.

Salary & Wages Tax % (IT0817) P0817-TAXPC
         0.00  999
   999
          5  5

Expected Behavior:

When 0.00 is entered in IT0817, the Salary & Wages Tax % field remains 0.00, and Field P0817‑TAXPC is also set to 0.00, as shown in the table below. This results in the correct tax treatment code being applied.

Salary & Wages Tax % (IT0817) P0817-TAXPC
         0.00  0.00
   999
          5  5

Other Terms

RPCST2Q0, RPCST2Q0_CE, Tax treatment code, IT0188, IT0817, MP081730, MP081720, MP081710, RPCTXQ01

Reason and Prerequisites

Reason: This issue occurs because the behavior of the Income Tax Withholding Variation (Infotype 0817) becomes inconsistent when a value of 0.00 is entered. The Salary & Wages Tax % field is left blank instead of storing the entered value, resulting in incorrect data being written to the database. Consequently, the STP2 report derives an incorrect tax treatment code and also calculated tax is incorrect.

Reason for Versioning:

   Version Reason
     01-03 Pilot Release 
     04 General Release for Customers

Solution

Code changes are required to generate the correct tax treatment code for an ATO‑approved downwards variation of 0%. When 0.00 is entered in IT0817, the Salary & Wages Tax % field remains 0.00, ensuring that the correct tax treatment code is reported as well as correct tax calculated.

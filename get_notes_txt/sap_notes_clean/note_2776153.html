Symptom

Analysis Office is not active/visible in Excel when opening a report of type "Microsoft Excel" from BI Launchpad/CMC using Internet Explorer browser.

In the AO traces (recorded by following instructions of SAP note 2129389), we can see the entries below:
---------------------------
RequestComAddInAutomationService called.
Application.UserControl: False
SupportEmbeddedMode.Value: False
COF Startup is getting cancelled (SetStartupCancelled)
---------------------------

The issue does not occur when using other browsers or when opening the report of type "Analysis Workbook" from BI Launchpad/CMC.

Environment

SAP BusinessObjects Analysis, edition for Microsoft Office
SAP BusinessObjects Business Intelligence Platform

Reproducing the Issue

  1. Go to BI Launchpad or CMC in browser Internet Explorer;
  2. Open an Analysis Office report of type "Microsoft Excel"; 
  3. Confirm that the report will open in Excel but Analysis Office will not be activated/visible.

Cause

When opening an Excel file from Internet Explorer, Excel is trying to open the file in embedded mode (also called automated mode).

Microsoft does not support Add-ins if the MS Office is running in embedded mode. This is why the add-in is not activating.

Resolution

By default, Analysis Office is not supported when MS Office is running in embedded mode.

Solution 1

Settings SupportEmbeddedMode and SupportAutomatedOffice can be changed to TRUE (see SAP note 2083067). 
Refer to chapters "Settings for the Analysis Add-in" and "UtilitiesConfiguration" in the Administrator Guide of AO in SAP Help Portal .

Solution 2

Convert the reports stored in BI Launchadp/CMC from "Microsoft Excel" to "Analysis Workbook", referring to SAP note 1888567.

Workaround

As a workaround, a different browser can be used instead of Internet Explorer.

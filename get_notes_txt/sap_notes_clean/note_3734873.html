Symptom

  • The system uses actual overhead on production orders via overhead templates (CPTA).
  • There is a need to retrieve the actual confirmed activity on an order for a specific activity type, period‑based, similar to OrderMaterialQuantityConf for confirmed material quantity.
  • A calculation is required in CPT2 to pick up periodic actual confirmed activity for a given activity type.

Environment

• Controlling (CO)
• SAP ERP Central Component
• SAP ERP
• SAP enhancement package for SAP ERP
• SAP enhancement package for SAP ERP, version for SAP HANA
• SAP S/4HANA
• SAP S/4HANA Finance
• SAP S/4HANA Cloud Private Edition
• SAP Fiori for SAP S/4HANA

Reproducing the Issue

  1. Start transaction CPT2.
  2. In the overhead template, attempt to add a calculation in the Actual Quantity field to retrieve the periodic actual confirmed activity on the order for the target activity type.

Cause

Resolution

  1. In CPT2, define the Actual Quantity formula using the template function AllocatedActualQtyObjects.
  2. Use the logical function K_ABC_GET_ALLOC_ACT_QUAN_OBJ (based on ABAP function K_ABC_ALLOC_QUANTITY) within the formula to retrieve period‑based actual confirmed activity quantities for the required activity type and order/object.
  3. Save the template and run actual overhead calculation to validate that the period‑based confirmed activity is picked up as intended.

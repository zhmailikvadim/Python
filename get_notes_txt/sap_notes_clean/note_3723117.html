Symptom

  • During the online partitioning there is a significant increase in the Data volume
  • While checking the disk space usage as described in the KBA 3293572 - SQL Statement Collection: "HANA_Disks_Overview" report for SAP HANA , it can be seen that the disk growth is related to fragmented disk space

Environment

SAP HANA 1.0, platform edition

SAP HANA, platform edition 2.0

Reproducing the Issue

  1. Perform online partitioning of a large table on the HANA Database
  2. Observe the fragmented disk volume growth in the data volume during the partitioning activity  

Cause

  • Excessive disk space consumption during partitioning is primarily due to fragmented disk space.
  • Online partitioning recreates the complete table, including all partitions, which can result in temporary additional disk space usage many times the size of the table in memory.

Resolution

  1. Reduce the required disk space during the online table partitioning by initiating the HANA automatic data volume reclaim job earlier
  2. Modify the parameter `.ini -> [persistence] -> continuous_reclaim_start_threshold` to a value of 125 (default value is 150).
  3. Refer to KBA 3268013 - SAP HANA Automatic Data Volume Reclaim for detailed information on the parameter

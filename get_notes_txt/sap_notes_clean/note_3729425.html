Symptom

  • Message processing intermittently fails with errors similar to: org.osgi.service.blueprint.container.ServiceUnavailableException: The Blueprint container is being or has been destroyed: (objectClass=com.sap.it.rt.scc.connectivity.ConnectivityServiceProvider)
  • The error can appear at different steps within an integration flow.
  • The objectClass in the message may vary depending on which service was accessed at the moment of the failure.

Environment

  • SAP Integration Suite
  • SAP Cloud Integration

Cause

  • The error indicates that a proxy for an OSGi service created within a blueprint context (the runtime context of an integration flow) was invoked while that context was stopping or had already stopped.
  • This situation can occur during short, controlled runtime restarts that happen as part of normal platform operations (for example, scaling events, rolling updates, or maintenance), when worker processes are restarted and the integration flow’s blueprint context is destroyed and re-created.
  • If a message is being processed exactly during this transition, calls to runtime services can fail with ServiceUnavailableException, and the objectClass in the error identifies the specific service that was momentarily unavailable.

Resolution

This is a transient, infrastructure-related condition during worker restarts. It cannot be fully avoided. Subsequent executions should succeed.

Recommended mitigations:

  • For asynchronous processing (e.g., JMS-based flows), the automatic retry mechanisms handles transient failures.
  • For synchronous scenarios, implement client-side retry.

Symptom

When customer has customized setup for change number status i.e. other statuses than Active(01) and Incative(02) are configured, the maintain BOM application does not perform correctly for edit with change number operation.
Also, when the change number is supposed to be created on the go via edit with change number functionality, again the system behavior is inconsistent.
Since the system always assumes status(01) to be active and (02) to be inactive, the operation itself is terminated with an error message.

Other Terms

Edit BOM with change number, F1813, Edit BOM with CN, T419S, Maintain BOM, Maintain Bills of Material.

Symptom

Payloads triggered by SLP despite no modifications being made to the questions

Cause

If any Generic Custom Fields are maintained in the Default Field Values area of SM Administration—specifically for Category, Region, Department, or qualification/disqualification/preferred status—then changes to those fields are expected to generate a payload from Ariba.

Resolution

If customers do not want the payload to be triggered, they need to remove the values from the Default Field Values file. As a result, any changes to those fields will not trigger a payload.

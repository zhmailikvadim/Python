Symptom

When opening a service order a big amount time is spent in reading the reference objets for the header and all the items. A performance trace shows that significant time is spent in CRM_SERVICE_OS_GET_OBJ_OB and CRM_SERVICE_OS_GET_OBJ_DB.

Reason and Prerequisites

The problem becomes significant if there are many items (e.g. hundreds) with many reference objects.

Solution

Upgrade to FPS2. There is currently no advance correction available.

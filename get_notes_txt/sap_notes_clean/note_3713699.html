Symptom

  • A change in system behavior has been observed regarding exchange rate calculation for documents issued in foreign currency during FI postings.
  • Previously, the conversion was based on the posting date, but it has unexpectedly shifted to being based on the document date.

Environment

  • Financial Accounting (FI)
  • SAP ERP
  • SAP ERP Central Component
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA 
  • SAP S/4HANA Finance
  • SAP S/4HANA Cloud Private Edition

Reproducing the Issue

  1. Simulate document postings using enjoy transactions such as FB60, FB70, FB65, FB75
  2. Currency conversion settings are based on translation date.
  3.  System is supposed to use posting date as translation date but is taking document date instead.

Cause

  • As per standard system always takes "Posting date" as translation date in enjoy transactions.
  •  In FB01 the translation can be entered manually.
  • An active implementation of BADI FI_TRANS_DATE_DERIVE might change the translation date to "document date".

Resolution

  1. Turn off or change the implementation of BADI FI_TRANS_DATE_DERIVE and system will use posting date again.
  2. Post document using FB01 and translation can be manually determined.

Symptom

Report RPRAPA00 just creates a workfile. It contains all the data, which is the basis for report RFBIKR00 to create the batch input session. It can be difficult to determine where the responsibility of RPRAPA00 ends and the responsibility of RFBIKR00 begins . For example for certain fields on the vendor there's an unexpected input or and input with an unexpected value or filled not at all.

This KBA describes how to check the data within the workfile via transaction SXDA_TOOL.

"Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental."

Environment

SAP Travel Management - on premise

no S/4 System!

Cause

It is sometimes not easy to determine when handling cases dealing with employee vendor master data creating/update which component to choose: FI-TV-COS (RPRAPA00) or FI-AP-AP-I (RFBIKR00, Batchinput for XK01, XK02).

Resolution

1. Start RPRAPA00 with the option "Create Workfile only". Remember the Work file name and path:

2. start transaction SXDA_TOOLS. Fill the selections as shown below and click on Display:

3. This takes you to the overview of all employee numbers included in the file: 

4. Every BLF00 (Header Data) entry is for a separate employee number (vendor to create or update). By double clicking on a single line, you see the concrete content, for example on vendor general data (LFA1): 

Here you see now the detailed data saved by RPRAPA00 .

Please mind, that the / is the nodata sign. It means, that batch-input shall create no entry in this field. This is not equal to space. For example a hidden field on the vendor should contain / . Just space would still mean an entry into this field. Hence it would produce an error when the batch input session is processed .

If you find the data at this place to be as expected by you, RPRAPA00 is not responsible for whatsoever behavior later on, which possibly is not expected . Should you want to open a ticket, please choose component FI-AP-AP-I at first with reference to your SXDA_TOOL results.

Symptom

  • An issue occurs while creating a BOM using the function module CSAP_MAT_BOM_CREATE.
  • The system throws the following error: Message ID: 29 Message Number: 851
  • Error Text: Error converting field Unit

Environment

  • SAP ERP Logistics (LO)
  • SAP S/4HANA, on-premise
  • SAP S/4HANA Cloud Private Edition

Reproducing the Issue

  1. Execute the function module CSAP_MAT_BOM_CREATE in SE37.
  2. Enter the Unit of Measure (UOM) in the external format on the single test entry screen.
  3. Run the function module.
  4. Observe that the system throws the error:
  • Message ID: 29
  • Message Number: 851
  • Error Text: Error converting field Unit

Cause

  • The error message 29851 occurs in SE37 as standard behavior (SAP Note 203805).
  • The function modules of function group CSAP are programmed to expect a UOM entry in the external format.
  • When the CSAP function module is called in single test mode (SE37), the UOM is entered on the single test entry screen and is automatically converted from the external format into the internal format.
  • In single test mode, the CSAP module is called with a UOM in the internal format, resulting in a double UOM conversion that leads to the error message 29851.
  • If the CSAP function modules are called from a program, the UOM conversion is carried out correctly, and the issue does not occur.

Resolution

  1. To deactivate the automatic conversion exit in SE37, change the ABAP Editor settings as follows:
  2. Navigate to Utilities -> Settings -> Function Builder
  3. Tick the option "Switch Off Conversion Exits When Testing".

Symptom

  • IAM apps are displayed as "Unnamed Objects" in the column "IAM-App" when the display language is set to German (DE).
  • The issue does not occur for English (EN) display language, where the IAM App text is correctly displayed.
  • Missing translations for specific IAM App IDs, such as "BP_FIJP_UNMAS_TRAN".

Environment

SAP S/4HANA Cloud Public Edition 

Reproducing the Issue

  1. Open the app "Display IAM Apps" in German display language (DE).
  2. Observe that the column "IAM-App" displays "Unnamed Objects" for multiple IAM apps.
  3. Change the display language to English (EN).
  4. Open the app "IAM-Apps" again.
  5. Observe that the IAM App text is correctly displayed for standard objects.

Cause

  • Translation scope for localization-specific texts is limited to English as the source language and the target language specific to the localization (e.g., Japanese, Romanian).

Resolution

The table which contains all IAM apps is translated as individual logical objects of type TLGS, like SIA6*.

For Japanese, the object is

TLGS: SIA6BP_FIJP_UNMAS_TRAN                      02

belongs to application component FI-LOC-FI-JP, which is Japanese localization. Similar another case

TLGS: SIA611001035_TRAN                           02

belongs to the application component FI-LOC-FI-RO. Which is Romanian localization.

All localization specific texts have only been defined as in scope for the source language English and target language is the one for which the localization refers to.

In these two examples both languages Japanese and Romanian translate from English and there is no other language in scope for them.

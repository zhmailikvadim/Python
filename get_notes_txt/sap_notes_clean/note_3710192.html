Symptom

This SAP Note is relevant only for the country version Germany if you use integrated PPA billing.

The receipt of electronic replacement statements (eEB) via KIM Web service with the program RNCEEB_RECEIVE may not work.

Other Terms

eEB, RNCEEB_RECEIVE, TN1TI_KIM2, V_TN1TI_KIM2, eAU

Reason and Prerequisites

Prerequisite:

In the IMG activity SAP Healthcare - Industry-Specific Components for Hospitals -> SAP Ambulatory Care Management -> Treatment -> Outpatient Clinic Forms -> Electronic Outpatient Clinic Forms -> Configure Dispatch, you have maintained entries for different applications for a place of work number. For example, an entry for the Electronic Work Incapacity Certificate (eAU) application and an entry for Electronic Replacement Certificate (eEB).

Reason - Program error

The program eEB - Receive via KIM Web Service (RNCEEB_RECEIVE) does not take the application into account when selecting the entry from the Customizing table. As a result, the program may not determine the entry for the eEB and therefore may determine an incorrect e-mail address for the query of the eEBs.

Solution

Import the specified Support Package or implement the correction instructions for your release.

The program now determines the entry for the eEB (electronic replacement statement).

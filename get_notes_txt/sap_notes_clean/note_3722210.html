Symptom

  • Changes made using Adapt UI are not saved under any transport request (TR), and no prompt appears to assign them to a transport request.
  • How to transport "Adapt UI Personas (Flavor)" to target system?
     

Environment

SAP S/4HANA Cloud Public Edition

Cause

Changes made using "Adapt UI" are not saved under a transport request (TR) and are not assigned to a transport request. These changes are considered "Key User Extensibility Items" and must be transported to the target system via a "Software Collection" using the "Export Software Collection" app.

Resolution

Create a software collection in the Customizing system by following the steps below:

  1. Navigate to the "Export Software Collection" app.
  2. Click on "+".
  3. Enter a name and create the software collection.
  4. Open the software collection and select "Add Items".
  5. Search for the required item (for example, Adapt UI changes and published views are stored as "Personas: Flavor" or "UI Flexibility: Views and Changes").
  6. Select the relevant entry and add it to the collection.
  7. After adding the items, select "Export".

After a successful export, the software collection is available in the test system with the status "Ready for Import" in the test system, use the "Import Collection" app to import the collection and transport the changes to the target system.

Note: Only changes made in "Configuration Activities" (for example, changes made using SSCUI) are saved under a transport request (TR). For such changes, a prompt appears to assign them to a transport request in order to move them to the target system.

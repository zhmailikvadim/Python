Symptom

You have installed a new Release or Support Package (SP) for one of the following products:

  • SAP S/4HANA Utilities for Billing Enhancements of German Energy 2602
  • SAP S/4HANA Utilities for Intelligent Metering of German Energy 2602
  • SAP S/4HANA Utilities Master Data Updates Option for Germany 2602

As the first configuration activity, you need to activate the business configuration (BC) sets uploaded by the installation. These BC sets contain default Customizing settings, and in particular, the configuration for the German Utilities processes.

Note that the activation sequence described below is crucial to avoid inconsistencies in the configuration!

This SAP Note is applicable for the following own roles in the German Utilities market:

  • Meter operator (MOS)
  • Distribution system operator (DSO)
  • Supplier (SUP)
  • Party responsible for settlement (PRS)

Other Terms

US4G, Backend, Business System, Market Communication, MaCo, MaKo, Billing, Metering, Master Data Updates, Switching, MOS, DSO, SUP, PRS, MSB, VNB, BKV

Reason and Prerequisites

You have installed the current Release or Support Package (SP). That is, you have implemented the latest installation packages for the US4G and UIUS4G components.

For more information, see the SAP Note under References below.

Solution

First, perform the steps in section A). Then, depending on your scenario, select one of the sections B1) or B2), and perform the steps in only the selected section.

A) In all cases, regardless of whether you are implementing a new installation, a release upgrade, or an SP upgrade, activate the following BC sets in the given sequence:

These are full BC sets that contain all required configuration settings.

  1. The BC sets of Market Process Management in SAP S/4HANA.
    See: Activation of Business Configuration Sets

  2. Optional: /US4G/BC_ARCH_CLOUD_CONFIG for settings for the archiving of completed cloud documents in the business system.
    See: Remote Archiving in the Business System

  3. Optional: The BC sets of the Measurement Management (MCM) adapter for the German market.
    See: MCM Business Configuration (BC) Sets

B1) For a new installation, activate the following BC sets in the given sequence:

Activate all the listed BC sets. That is, the general ones and the ones for your market role(s). These are full BC sets that in sum contain all required configuration settings.

Before activating the US4G BC sets using SCPR20, activate the below IS-U Switch BC sets as described under this help link. The reason is that these Switch BC sets are set as non-cascading. So, the contents are not deployed to other clients when the business functions are activated.

ISU_TFK003_T
ISU_TECDE_S
ISU_605_AMI_VC_EVENT_MNGMNT_C_1

Before activating the US4G BC sets using SCPR20, set the Deletion Function to Switch On. To do so, call transaction SCPR20 and choose Utilities -> User Settings -> Maint. Transaction. This setting is required because some BC sets contain entries that are deleted and should not be part of the delivery.

/US4G/3_0_NEW_COMM_2602_SP00_p1 for general process settings, part 1
/US4G/3_0_NEW_COMM_2602_SP00_p2 for general process settings, part 2
/US4G/3_0_NEW_DSO_2602_SP00 for processes for the market role DSO
/US4G/3_0_NEW_MOS_2602_SP00 for processes for the market role MOS
/US4G/3_0_NEW_MOS_CI_2602_SP00 for convergent invoicing processes for the market role MOS
/US4G/3_0_NEW_SUP_2602_SP00 for processes for the market role SUP
/US4G/3_0_NEW_PRS_2602_SP00 for processes for the market role PRS

B2) For an upgrade from release 2305 SP08 (shipped on February 13, 2026) to release 2602 (shipped on February 24, 2026) activate the following BC sets in the given sequence:

Activate all the listed BC sets. That is, the general ones and the ones for your market role(s). These are delta BC sets that collate configuration settings over the version history.

Before activating the BC sets using SCPR20, set the Deletion Function to Switch On. To do so, call transaction SCPR20 and choose Utilities -> User Settings -> Maint. Transaction. This setting is required because some BC sets contain entries that are deleted and should not be part of the delivery.

/UIUS4G/3_0_COMM_2602_SP00 for general user interface (UI) settings
/US4G/3_0_COMM_2602_SP00 for general process settings

Symptom

Trace ID error is received when trying to search (lookup) Visit records, which are related to an Account, while using a Custom AI Tool.

Error: There was an error while fetching C4C data. Trace ID: 08f9747b-645a-4475-934e-769b7bfe0709

Environment

  • SAP CX AI Toolkit
  • Sales Cloud version 2

Reproducing the Issue

The error occurs when trying to look up Visits related to an Account in a V2 connected Toolkit environment.

Cause

  • Seems like an issue with the timeline data. The field `objectId` is missing in the timeline. This is unusual as the timeline detail should have `objectId`.
  • The trace id returns a more details message:
    input: search error searching CNS items error getting timeline filter: error generating object ID filter: objectId not found in response item
  • Similar issues were found to be caused by configuration settings in the V2 tenant.

Resolution

Steps to Fix the Issue – in Sales & Service Cloud V2:

1. Check Timeline Settings for Visits:

- Ensure that the Visits events entity is configured to ‘show on Timeline’ for Accounts.

2. Verify Timeline Filters:

- Check if the Visits entity is 'on' for Accounts in the Timeline Filters (under Customer Insights).

For anyone experiencing this problem, ensuring the correct Timeline settings for Visits will resolve the issue. This includes verifying that Visits are configured to ‘show on Timeline’ for Accounts and that the Visits entity is ‘on’ for Accounts in the Timeline Filters.

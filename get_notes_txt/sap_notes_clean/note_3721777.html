Symptom

When you create or change an outbound delivery in VL01N or VL02N, you edit an item field in the item overview and select Save without pressing ENTER. In this situation, follow-up processing during the save sequence uses the previous field value, which can cause incorrect system behavior.

Other Terms

VL01N, VL02N, outbound delivery, save without ENTER, table control, SAPMV50A, MV50AF0L

Reason and Prerequisites

If you choose Save without pressing ENTER, the last value you entered might not transfer from the screen to the internal structures before follow-up processing begins.

Solution

Implement this SAP Note  or the relevant feature package.
This SAP Note adds a check during the save process. If there are pending item changes when you select Save, the system displays a warning message and cancels the save action. You must then save the delivery again.

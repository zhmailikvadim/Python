Symptom

  • Error encountered during server-side rendering (SSR) execution:
ERROR: URL with hostname '127.0.0.1' is not allowed. Please provide a list of allowed hosts in the 'allowedHosts' option in the constructor.
  • SSR request fails and fallback to CSR

Environment

  • SAP Commerce Cloud

Reproducing the Issue

  1. Having SSR enabled and missing yarn.lock file in git repository, trigger a new build then deploy
  2. Observe the following error in SSR logs when rendering a page:

ERROR: URL with hostname '127.0.0.1' is not allowed. Please provide a list of allowed hosts in the 'allowedHosts' option in the constructor.

Cause

@angular/ssr library patch upgrade has introduced a new environment variable NG_ALLOWED_HOSTS that requires whitelisting SSR server hostname.

This impacts both angular 19 and 21 starting respectively from 19.2.21 and v21.1.5. If you didn't commit your yarn.lock file, therefore during Cloud build you will get the latest @angular/ssr library version, and most likely hit this issue. It's recommended to commit your yarn.lock to avoid unexpected behaviours in the Cloud.

Resolution

In your server.ts add the following:

if (!process.env['NG_ALLOWED_HOSTS']) {
  process.env['NG_ALLOWED_HOSTS'] = '127.0.0.1,localhost';
}

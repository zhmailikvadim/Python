Symptom

  • Unable to log in to BTP application which is using IAS as application IDP.
  • Login attempts do not complete successfully, preventing access to application.
  • Error message: "There was an error when authenticating against the external identity provider: 401: { "error": "invalid_client", "error_description": "No matching certificate or jwks_uri found." }".

Environment

  • SAP Business Technology Platform (SAP BTP)

Reproducing the Issue

  1. Attempt to log in to BTP application.
  2. Observe that the login process does not complete successfully, and access to application is not granted.
  3. Error message appears: "There was an error when authenticating against the external identity provider: 401: { "error": "invalid_client", "error_description": "No matching certificate or jwks_uri found." }".

Cause

The URI value in the "Configure Trust by URI" for the IAS application was wrongly maintained or it is empty.

Resolution

1. Open the administrator console of the IAS tenant.

2. Navigate to the tab "Applications and Resources" and select "Applications".

3. Choose the relevant IAS application. Upon accessing BTP application, page will be redirected to the IAS page for authentication, where the IAS application name will be displayed. 

4. Under the "Trust" section, go to "Application APIs" and select "Client Authentication".

5. Edit the "Configure Trust by URI" field and update the URL:https://<subdomain>.authentication.<btp region>.hana.ondemand.com/token_keys

6. Save the changes and verify the updated configuration.

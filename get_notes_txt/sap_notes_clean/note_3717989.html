Symptom

When users try to change the font color in the content field, the text reverts to black as soon as they hit save. This issue is specifically affecting KB Managers, who are unable to get any color changes to stick after publishing.

Environment

SAP SuccessFactors Work Zone

Reproducing the Issue

  1. Open the workspace in the production environment.
  2. Select any knowledge base article.
  3. Attempt to change the font color in the content field to any color.
  4. Save the changes and view the article.
  5. Observe that the font color reverts to black despite the editor showing the updated color prior to saving.

Cause

  • The root cause of the issue was identified by the engineering team as a defect in the template functionality.

Resolution

Our engineering team has a fix ready to go. We’ve scheduled the rollout across all data centers for March 19, 2026.

Symptom

When sending article iDocs using t-code BD10/BD21, communication iDoc is not created for specific receiving system. 

Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental. 

Environment

  • SAP ERP Logistics (LO)  
  • SAP Industry Specific Solutions (IS)
  • SAP R/3
  • SAP R/3 Enterprise
  • SAP ERP Central Component
  • SAP ERP
  • SAP Enhancement package for SAP ERP
  • SAP Enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4 HANA on premise
  • SAP S/4 HANA Cloud Private Edition

Reproducing the Issue

  1. Open t-code BD10, input article number, logical system. Execute and send article. 
  2. Master iDoc successfully created for message type ARTMAS. 
  3. Communication iDoc is not created for message type ARTMAS. Message 0 communication IDoc(s) generated for message type ARTMAS is shown. 

Cause

No distribution model maintained for corresponding sending system, receiving system and the BAPI. Please review distribution model settings in t-code BD64. 

Alternatively, the configuration can also be checked in table TBD06. 

  • In t-code SE16, check tale TBD06.
  • Search if there's an entry exists for corresponding sending system, receiving system and the BAPI (BOBJTYPE = BUS1001001, METHOD = CLONE). 
  • If not, maintain the missing distribution model in t-code BD64. 

Resolution

Maintain distribution model with corresponding sending system, receiving system and BAPI/Object name/interface/Method. Afterwards recreate ARTMAS iDocs in t-code BD10/BD21. 

Refer following steps to maintain distribution model in t-code BD64. 

  • Open t-code BD64 and enter change mode. 
  • First create a model view, specify technical name and description. 
  • Then add BAPI into the model view with following parameter and corresponding sending/receiving system. 
    Obj. name/interface: RetailMaterial
    Method: Clone
  • Make sure distribution model is correctly added, save the configuration.

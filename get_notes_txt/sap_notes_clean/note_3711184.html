Symptom

  • In Web Intelligence (Webi) 'Save' auditing events (1008) use incorrect statuses.
  • The audit event status is always 0, which indicates the object was successfully saved locally
  • This status is 0, even when saving to the CMS DB, which should be status '2' (indicates the object was successfully saved or exported to a repository)

Other Terms

BILP, BI Launchpad, Web Intelligence Rich Client (WIRC)

Reason and Prerequisites

Environment

  • SAP BusinessObjects Business Intelligence (BI) Platform 4.x / 2025
  • Web Intelligence (Webi)
  • Auditing
  • Windows
  • Linux / Unix 

Steps to Reproduce

  1. Create a web intelligence document
  2. Save it to the repository
  3. Observe the auditing event 1008 (Save) is generated, but with a status of 0, when it should be 2

Symptom

When Running report CATS_DA, and using a wildcard (*) in the filter on column "Receiver WBS"  gives Message, WBS element  XXXXX does not exist. 

Environment

  • SAP ERP Project System (PS)
  • SAP R/3
  • SAP R/3 Enterprise
  • SAP ERP Central Component
  • SAP ERP
  • SAP Enhancement package for SAP ERP
  • SAP Enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA, on-premise
  • SAP S/4HANA Cloud Private Edition

Reproducing the Issue

  1. Run CATS_DA.
  2. Enter Date
  3. Enter WBS in field WBS Element of "Receiver Account Assignment"
  4. Execute
  5. Add Column  Receiver WBS Element 
  6. Either select column and click Set Filter
    (Or click the filter icon and add the column) 
  7.  enter in WBS with Wildcard  eg  XXX-XX* 

Message is raised 

Cause

The technical field name of for  Receiver WBS Element  is  CATSDB_ALV-RPROJ 
The field definition of this field leads to domain PS_POSNR which converts the entered string using conversion routing ABPSP.
This does however not allow generic selections
Field PSPID available in the list which leads to conversion routine ABPSN which allows the selection using generics
The difference is due to how WBS-elements are stored in your system. For WBS-elements there is an internal and an external number and the external number can be masked using a coding mask as well.

Resolution

You can not use Wildcard 
You can use  "Range"  option to  select the data  Eg .  00001 - 99999 

Keywords

PRPS-POSID , CONVERSION_EXIT_ABPSP_INPUT' SAPLPSCONV CONVERSION_EXIT_KONPR_INPUT, POSID,  PS_POSID

Symptom

HTML rendering of OZ-1, OZ-2, OZ-3, OZ-6, and FI Register invoices depended on a legacy XSL file. You implemented ABAP-side changes that use XSLT_TOOL templates, affecting XML-to-HTML transformation behavior

Other Terms

XSLT_TOOL, XSLT, XML-to-HTML, Ukraine OZ1/OZ2/OZ3/OZ6, FI invoice register, ABAP Call transformation

Reason and Prerequisites

Standard XSL files were hard to maintain/version; moved to ABAP-managed transformations via XSLT_TOOL to centralize control and reduce external XSL dependencies

Solution

Implement correction instructions using transaction SNOTE or install the Support Package indicated.

Solution delivered within this SAP Note will:

1. Replace external XSL with XSLT_TOOL templates per document type (OZ-1, OZ-2, OZ-3, OZ-6, FI Register);

2. Adjust ABAP code to call CALL TRANSFORMATION <template> on the XML;

3. Create a new ABAP class for CALL Transformation functionality.

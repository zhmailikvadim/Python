Symptom

The "Valid From/To" field is missing when creating a Consolidation Unit Hierarchy.

Attempting to set "Valid From/To" values at the Consolidation Unit level does not affect reporting functionality.

A Consolidation Unit valid from "01.06.2023" to "31.06.2023" is visible in the "Data Analysis" app when using the hierarchy for period = 012.2023 and Key Date = "01.12.2023".

Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.

Environment

SAP S/4HANA Cloud Public Edition

SAP S/4HANA  Private Cloud Edition

Reproducing the Issue

  1. Create a Consolidation Unit type hierarchy.
  2. Observe that the "Valid From/To" field is not available at the hierarchy level.
  3. Assign "Valid From/To" values at the Consolidation Unit level using Note "3670543 - The Valid From/To field in App F2918 Manage Global Hierarchy is missing".
  4. Use the "Group Data Analysis" app to validate the time-dependent hierarchy functionality.
  5. Verify that the Consolidation Unit valid from"01.06.2023" to "31.06.2023" appears in the app when using the hierarchy for period = 012.2023 and Key Date = "01.12.2023".

Cause

When the "Key Date" filter value changes after the hierarchy is applied, the hierarchy "Valid On" date must be manually updated and applied again for both selections to synchronize.

When selecting the hierarchy, the system propose by default the report "Key Date" value as the hierarchy "Valid On" value, but it can be manually changed.

If you later change the "Key Date" field in the report filter, after having selected the hierarchy, you also need to manually change the hierarchy "Valid on" value manually if you want both fields selections to be in sync, it is not happening in an automatic way.

Resolution

  1. Change the key Data in filter
  2. Disable Hierarchy from the report and Select Hierarchy again
  3. Make sure " valid on "  is the same like Key Date , then search hierarchy->  Click it.

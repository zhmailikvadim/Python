Symptom

Tried to check the scheduled backup using the following SQL statements via HANA Studio, but all the statements returned an error.

SELECT * FROM SYS.M_SCHEDULER_JOBS;
SELECT * FROM SYS.SCHEDULER_JOBS;

"SAP DBTech JDBC: [259]: invalid table name: Could not find table/view M_SCHEDULER_JOBS in schema SYS"

Environment

SAP HANA, Platform Edition 2.0

Reproducing the Issue

Run the SQL statements in HANA Studio.

Cause

The current HANA database version is 2.0 SPS05, and the table/views in those SQL statements are available as of HANA 2.0 SPS06.

Resolution

You can check the scheduled backups in HANA Cockpit. 
(Database Overview -> Backup Schedules  -> Click the scheduled backup -> "More details")

In case of SPS05, you may run the SQL statement below in HANA Studio.

SELECT * FROM "sap.hana.xs.admin.jobs.server.db::SCHEDULES"

DROP SCHEDULER JOB command is also available as of SPS06.

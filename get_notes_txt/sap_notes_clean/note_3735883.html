Symptom

  • Unable to execute SQL command: INSERT INTO virtual_table [(column1, column2, ...)] SELECT column1, column2, ... FROM source_table 
  • The virtual table is generated by CamelJDBCAdapter remote source to connect to GoogleBigQuery.
  • The framework.trc trace on the DP agent contains the following error:
    • [ERROR] SDAAdapterDelegate | ExceptionUtil.newAdapterException [] - Failed to rollback transaction. Reason: java.sql.SQLException: [Simba][BigQueryJDBCDriver](100032) Error executing query job. Message: Transaction control statements are supported only in scripts or sessions org.apache.camel.CamelExecutionException: Exception occurred during execution on the exchange: Exchange[<exchange_ID>]

Environment

SAP HANA smart data integration (SDI)

Cause

To work with transactions across multiple queries, need to set EnableSession=1 in the JDBC URL.

Resolution

Add the parameter EnableSession=1 to the BiqQuery JDBC URL in the remote source configuration.

For example.
jdbc:bigquery://https://www.googleapis.com/bigquery/v2:443;ProjectId=<your-project-id>;OAuthType=0;OAuthServiceAcctEmail=<your-service-account>;OAuthPvtKeyPath=<your-keyfile-path>;EnableSession=1

Symptom

As described in SAP ONE Support Launchpad Notification of Legal Change Unified monthly employer report 2025 (JMHZ)

The functionality delivered in this SAP Note is currently under development. It is intended exclusively for customers registered as JMHZ pilot testers, or for the preview of features that may become available for productive use in the future.

These updates are provided:

  • REGZEC (DM - data mining, CS - compose submission, PF - process feedback, SO - special operations)

    • DM: The insurance period in unemployment support data does not account for information generated by the MESHLA report.

    • DM: Unemployment support data is not generated if there is a change in the variable symbol, unless it involves a VS transfer (Action A5) rather than a standard employee hiring.

    • CS: Correction (A4) is not generated for entry action (A1) when hire date is changed.

  • MH

    • Changes in the administrator mode of the Workbench program (HCZCJHM0_WB, transaction PC00_M18_JHM_WB_ADMI).
      The program itself does not change. Only the ALV function called from the summary forms is changed. Now, you can change the status of individual forms. Each change is logged. To see the change, you need to exit the ALV of individual forms.

    • Attribute 10323 (Overpayment or underpayment from annual settlement) is now filled in.

    • The program HCZCJHM0_DM now has a new parameter “Force correction”.
      By default, the program does not create a correction form for a period that has already been reported, if the new form has the same content as the old one. If you check this parameter, the program will create a new form even if the data is the same as before.

    • In the messages column from DZMH, there was an error icon even if there was no error for the form. Now, the error icon is replaced with a warning icon for statuses 4-6 (partially accepted, in processing, or contains minor errors).

Symptom

  • The current default value for the "SP Sign Logout Response" setting in the IDP-initiated Global Logout section is incorrectly set to "No" in the Default Tenant Configurations for HCM Suite and Identity Authentication. This incorrect setting causes the following error during IDP-initiated logout:

HTTP Status 400 – Identity Provider could not process the logout message received.

Environment

  • SuccessFactors HCM Suite
  • Identity Authentication Service (IAS)

Cause

  • The default value of "Require signed single logout messages" in the IAS application for SuccessFactors is set to Checked/True, which would mean that SuccessFactors as an SP must sign the single logout messages, i.e. "SP Sign Logout Response" in SuccessFactors SSO setting must be updated to "Yes" to ensure proper IDP-initiated logout functionality.
  • For a successful IDP initiated logout, the value for "Require signed single logout message" in IDP, and the value of "SP Sign Logout Response" should be exactly the same, i.e. both should be set to True/Yes or both should be set to False/No, any mismatch would cause the issue described. 

Resolution

1. Documentation Update:

  • The Help Portal documentation (Default Tenant Configurations for HCM Suite and Identity Authentication) will be updated to reflect the correct setting ("Yes").
  • The update will be applied in the 2605 version of the topic once the backend changes are completed (expected in April/May-2026).

2. Backend Changes: 

  •  A batch process will be executed to update all existing tenants with the correct setting are planned for April/May-2026.  
  •  This backend process will also be modified to ensure new tenants are provisioned with the correct default value ("Yes") 

Workaround (Temporary Fix)

  • Until the official update is released, "SP Sign Logout Response" can be manually set in SuccessFactors Provisioning to avoid the logout error.

    • If the value for "Require signed single logout message" in IAS is "Yes" (the default value), the set "SP Sign Logout Response" to "Yes" in provisioning
    • If the value for "Require signed single logout message" in IAS is "No", then "SP Sing Logout Response" should be left as "No".
     

Note : The correct setting ensures compliance with SAML SSO standards for IDP-initiated logout.

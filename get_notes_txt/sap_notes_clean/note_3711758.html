Symptom

  • Error message displayed: "Freight Order &1 does not exist" when attempting to post an invoice using MIRO.
  • Issue occurs despite the Freight Order being successfully integrated into ERP and the Service PO being created.

Environment

SAP Transportation Management embedded in S/4HANA 2025 and higher

Reproducing the Issue

  1. Create a Freight Order which is integrated with Lean Service.
  2. Post the Freight Order which would create the subsequent documents like Service PO & SES.
  3. Navigate to MIRO to post the Invoice.
  4. Enter the FO number under "TM ref" Tab.
  5. Attempt to post the invoice.
  6. Observe the error message: "Freight Order does not exist."

Cause

Lean TM documents cannot be invoiced from the TM Reference tab in MIRO. This is the standard system behavior. 

Resolution

  1. In SPRO customizing of Materials Management under Logistics Invoice Verification -> Incoming invoice -> Activate Integration of Transportation Management (TM) -> Click on "New Entries" and Enable "TM (Lean) Active" under "Settings TM Integration" if not enabled already.
  2. Once the required customizing is completed, the Freight Order becomes visible under the “PO Ref.” tab in MIRO. In a Lean Services integration scenario, during invoicing in MIRO, the Freight Order must be invoiced using the “PO Ref.” tab.

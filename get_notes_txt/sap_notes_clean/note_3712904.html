Symptom

  • IBP Job executing Copy Operators fail with "Filter isn't valid. Please correct it" error.
  • Error occurs specifically when the copy operator is executed with a planning filter.
  • The job fails for the executing user but works successfully for the filter owner.
  • Previous executions of the job were successful before the IBP major update.

Environment

  • Copy Operator

Reproducing the Issue

  1. Execute the Copy Operator job in the 2602 release using a planning filter.
  2. Observe that the job fails with the error "Filter isn't valid".
  3. Verify that the filter is not shared with the executing user.

Cause

  • The planning filter ID is not shared with the executing user, resulting in authorization issues.
  • Pre-2602, unshared planning filters could be used with a copy operator profile 
  • As of 2602, This gap is closed and filters need to be shared with the executing user.

Resolution

Ensure the filter is shared with the executing user in the production environment to prevent similar issues.

See Also

Refer to: SAP Integrated Business Planning documentation for filter sharing guidelines.

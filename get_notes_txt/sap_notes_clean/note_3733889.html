Symptom

You are using SAP Business Technology Platform (SAP BTP. URL: https://cockpit.btp.cloud.sap) Application Logging Service for the Cloud Foundry Environment. Filtering by source_type in logs does not return any results in the discover view.

Environment

  • SAP Business Technology Platform (SAP BTP)
  • SAP Cloud Foundry
  • SAP Application Logging Service

Reproducing the Issue

  1. Go to the Kibana discover view.
  2. Add a search/filter criterion on the field source_type.
  3. Observe that no results are returned.

Cause

A restrictive index mapping limited the source_type field to 32 characters, so longer values were not indexed and queries on source_type did not match those documents.

Resolution

  1. Verify whether the source_type values you need to filter on exceed 32 characters.
  2. If they do, be aware that documents with longer source_type values are not searchable by that field due to the current index mapping. This limitation is planned to be fixed soon (planned to at least 256 characters in future).
  3. As a temporary mitigation until the update is available, search using other identifiable fields or free-text queries where applicable instead of filtering on source_type.

    NOTE: SAP recommends using Cloud Logging Service as it provides more enhancements. For further info, check: SAP Cloud Logging service. For more information, check SAP KBA ##3557260: Information about the deprecation of SAP Application Logging Service.

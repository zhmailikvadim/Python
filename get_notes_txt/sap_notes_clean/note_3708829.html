Symptom

Customers may encounter different types of issues while working with Document and Reporting Compliance (DRC) – Statutory Reporting. These issues can occur during report generation, electronic submission, or configuration of reporting entities.

The issues are commonly observed in the following scenarios:

Generation / Runtime Issues

For example:

  • Statutory report generation fails or returns unexpected errors
  • Mass actions on statutory reports cannot be executed
  • Scheduling of report generation or submission fails
  • Output file download is not possible
  • Document attachment or output management is not working

Communication / Submission Issues

For example:

  • Communication arrangement cannot be set up in the Run Statutory Reports app
  • Errors during electronic submission of statutory reports

Configuration Issues

For example:

  • Issues related to SSCUI (Self-Service Configuration User Interface)
  • Business Configuration (BC) Sets or CBC (SAP Central Business Configuration)
  • View cluster–related configuration problems
  • Errors while setting up reporting entities

Other Terms

Document and Reporting Compliance, Statutory Reporting, Components, Generation, Mass action, Output management, configuration, Attachment Services, FI-LOC-SRF-RUN-COM, FI-LOC-SRF-RUN-GNR, FI-LOC-SRF-CFG, FI-LOC-SRF, FI-LOC-SRF-RUN, FI-LOC-SRF-DEF

Reason and Prerequisites

Incidents related to DRC Statutory Reporting were frequently misrouted due to the use of generic components, resulting in multiple reassignment cycles and increased analysis time.

To enable effective routing of Statutory Reporting issues, additional sub-components have been introduced to enable accurate issue categorization and ensure assignment to the correct responsible team. This helps eliminate unnecessary handoffs and reduces analysis time caused by reassignment between teams.

The new sub-category components support faster and more efficient issue resolution, while the existing components continue to remain available for use.

Solution

Assign the incident to the appropriate component based on the nature of the issue, as described below.

There is one umbrella component FI-LOC-SRF-RUN for issues related to the execution of Statutory Reporting runs.

  • For example- Issue related to Phase, Activities, Reporting task, Preview screen not loading etc. will be routed to FI-LOC-SRF-RUN component.

Depending on the exact type of the issue, use one of the following sub-components/New Components for Effective Routing

  • If the customer is attempting to generate a report and encounters an error or is unable to schedule a mass action for report generation, route the incident to FI-LOC-SRF-RUN-GNR.
  • If the report has already been generated and the issue is related to electronic submission or communication, route it to FI-LOC-SRF-RUN-COM.

On more type of issues are there which are related to configuration for example-

  • If the issue is related to configuration (e.g., SSUI, View clusters), or
  • Communication arrangement or eDocument processing fails or
  • Reporting activity not appearing,  Incorrect periodicity of reports etc. will be routed to  FI-LOC-SRF-CFG. 

For any report definition related issues for example-

  • Issues related to report definition, mapping, etc. route it to FI-LOC-SRF-DEF
  • Licensing or metering related issues will be routed to FI-LOC-SRF-DEF (or CA-GTF-CSC-DRC-PRC)

Additional Information

Using the correct component ensures:

  • Accurate and consistent incident routing
  • Reduced reassignment and analysis effort
  • Faster resolution times
  • Improved handling of Statutory Reporting incidents across all releases

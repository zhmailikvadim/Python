Symptom

You import interactions via RFC or a Web service. The import terminates with a runtime error, the interactions are not saved, and there are entries in transaction ST22.

Reason and Prerequisites

Using RFC or a Web service, data can be passed in fields of the ABAP type SSTRING that exceeds the length defined in ABAP because there is no internal type check for this. When the field contents are written to the database, a runtime error occurs because the database can process only the maximum length.

The corrections truncate all contents of SSTRING fields in a table and its subtables to be imported that are longer than the maximum defined length. This increases the processing time. For 10,000 interactions with filled subtables, there is an increased runtime of about 0.5 seconds.

Solution

Implement the attached correction instructions.

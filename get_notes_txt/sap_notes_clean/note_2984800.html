Symptom

This KBA describes a solution for filling the average working hours fields of infotype 0007. This solution can be used starting with release 2H 2020 (EA-HR SP 86 and S4HCM 101 SP14). For release 1H 2020 (EA-HR SP 82), this solution cannot yet be applied. For that reason, the separate KBA 2925305 exists for release 1H 2020 (EA-HR SP 82) that describes a solution for filling the average working hours fields of infotype 0007 based on a BAdI implementation.

The point-to-point replication of planned working time (PWT) allows replicating the planned working times of an employee from the EC system to the EC Payroll system. Via this replication, the planned working times of an employee are stored for each day in infotype 2003 "Substitutions" of EC Payroll. It is thus not necessary to create an actual work schedule in the EC Payroll system, which would also require keeping this work schedule synchronous with the corresponding work schedule in the EC system.

Nonetheless, infotype 0007 "Planned working time" of EC Payroll requires specifying a work schedule rule. It is therefore necessary to define a dummy work schedule in the EC Payroll system. The work schedule rule of this dummy work schedule is then assigned to employees in infotype 0007 via the master data replication from EC to EC Payroll.

The KBAs 2906872 and 2914265 describe how to setup a dummy work schedule in EC Payroll and how to assign it in infotype 0007. The resulting planned working times are then overridden by the planned working times stored in infotype 2003. Thus, when the system determines the planned working times of an employee, for example, via function module HR_PERSONAL_WORK_SCHEDULE, the correct values are returned.

Besides the work schedule rule, infotype 0007 also stores the following additional fields:

  • Employment percentage
  • Part-time work indicator
  • Weekly workdays
  • Daily working hours
  • Weekly working hours
  • Monthly working hours
  • Yearly working hours

The standard logic of infotype 0007 normally does not allow master data replication to fill all these fields from the data in EC. Instead, some of these fields are calculated by the standard logic of infotype 0007 based on the content of the other fields and based on the average working hours stored with the work schedule rule. However, this poses a challenge. The dummy work schedule rule - by its very definition - cannot contain meaningful values for average working hours.

This KBA proposes a solution on how to assure that all fields of infotype 0007 are filled with meaningful values. It can be applied starting with release 2H 2020 (EA-HR SP 86 and S4HCM 101 SP14)), because the logic of infotype 0007 was enhanced with that release. It is now possible to fill all these fields of infotype 0007 through extensibility, if certain conditions are met.

A solution applicable for release 1H 2020 (EA-HR SP 82) and based on an implementation of BAdI HRPAD00INFTYBL is described in the KBA 2925305.

Environment

  • Employee Central Payroll
  • Time management integration scenario

Resolution

Logic for Filling Infotype 0007

Infotype 0007 is filled through the replication of master data from Employee Central. There are different ways to handle this. Please see also the corresponding chapters in the implementation guide "Employee Central Payroll using Point-to-Point Integration".

In the normal logic of infotype 0007, some of the average fields of this infotype cannot be specified directly. Rather, their values are derived from other average fields of this infotype in combination with the average fields specified in the work schedule rule. Exactly which average fields are entered by the user is determined through the feature WRKHR.

Now, in the case of point-to-point replication of planned working time, a dummy work schedule is used, which cannot - by its very definition - contain meaningful average hours. In fact, it is recommended to leave the corresponding average fields empty. The following screenshot highlights the empty average fields of the definition of the dummy work schedule rule. See the KBAs 2906872 and 2914265 for further details.

In an EC Payroll system, the EC system should be the leading system for specifying the average working hours and days. For that reason, the logic of infotype 0007 was enhanced with release 2H 2020 (EA-HR SP 86). This changed logic applies, if the following conditions are met:

  • The system is an EC Payroll system AND
  • In the work schedule rule, all the fields for average working hours and days are initial AND
  • In the infotype 0007, all the fields for average working hours and days are filled through the replication from EC.

If all these conditions are met, then the normal logic of infotype 0007 no longer applies. Instead, the logic of infotype 0007 does not read feature WRKHR and simply accepts the values for the average working hours and days as provided through the replication of this data from EC.

 

Filling the Employment Percentage

The field "Employment percentage" (technical name EMPCT) of infotype 0007 describes the relation between the average working times as specified by infotype 0007 and the average working times as specified by the work schedule rule assigned to infotype 0007. It does not directly correspond to the Full Time Equivalent of an employee as stored in EC.

If a part-time worker is assigned a work schedule rule corresponding to a full-time worker, then the payroll will automatically adjust that part-time workers planned working times. However, there are good reasons not to assign full-time work schedules to part-time workers. Instead, it is better to define work schedule rules specific for part-time workers. And in this case, the field "Employment percentage" will contain 100% even for a part-time worker.

The following example may illustrate why it is better to define work schedules specific for part-time workers. Suppose the work schedule specifies a 5-day week with working hours from 8:00 to 17:00 including some breaks. If the employee works 50%, then the system automatically assumes that the employee works 5 days a week from 8:00 to 12:30 with the breaks adjusted accordingly.

However, these planned working times may not be correct for this part-time worker, as he may not work 5 days a week and may not work in the morning and have the afternoon off. Instead, one part-time worker may work on Mondays and Tuesdays from 8:00 to 17:00 with breaks, and on Wednesdays from 8:00 to 12:00 without breaks. And another part-time worker may work on Wednesdays from 13:00 to 17:00 without breaks and on Thursdays and Fridays from 8:00 to 17:00 with breaks.

If specific work schedules for part-time work are created and assigned to IT0007, then the field “Employment percentage” must be set to 100%, for this field does not make a statement on the employee’s FTE. Rather, it only refers to how the employee works in relation to the work schedule assigned in IT0007.

The above considerations apply even when no planned working time data is replicated from EC to EC Payroll. In case of PWT replication, a dummy work schedule gets assigned in infotype 0007. And this dummy work schedule cannot contain meaningful average working hours. If the conditions apply as described in chapter "Logic for Filling Infotype 0007", then the logic of infotype 0007 sets the field "Employment percentage" to 100%.

 

Filling the Part-time Employee Indicator

As discussed above, the indicator "Part-time employee" needs to be set for a part-time worker even if the employment percentage is 100%. The standard master data replication sets the "Part-time employee" indicator according to the field "Fulltime employee indicator" in the Job Information of EC.

 

Filling the Weekly Workdays

The field "Weekly workdays" is filled through the standard master data replication based on the value stored in the field "Working days per week" in the Job Information of EC.

 

Filling the Working Hours

Infotype 0007 contains four fields with average working hours:

  • Daily working hours
  • Weekly working hours
  • Monthly working hours
  • Yearly working hours

If the conditions described in chapter "Logic for Filling Infotype 0007" are met, then the logic of infotype 0007 accepts for all these fields the values replicated from EC. The feature WRKHR is not used. However, this requires settings in activity "Define Extensibility of Field Assignment" of the IMG.

Filling the Capacity Utilization Level of Infotype 0008

By default, the field "Capacity utilization level" of infotype 0008 is filled with the same value as field "Employment percentage" of infotype 0007. For that reason, the filling of this field is also discussed here.

As explained above, there are good reasons why the field "Capacity utilization level" of infotype 0008 differs from the "Employment percentage" of infotype 0007. This will result in a warning message raised by the standard logic of infotype 0007. However, this warning can be ignored. In this approach, the field "Capacity utilization level" of infotype 0008 is filled based on the field "FTE" in the Job Information of EC.

It is possible that operations within payroll use the values of field "Employment percentage" of infotype 0007 or of field "Capacity utilization level" of infotype 0008. If these two fields contain different values, then care must be taken to assure that data of the correct field is used in these operations.

 

Filling the Working Hours per Payroll Period of infotype 0008

There is a connect between the field "Working Hours per Payroll Period" of infotype 0008 and the average working hours fields of infotype 0007. For that reason, the filling of this field is also discussed here.

This field is calculated by taking into account the employee's payroll period and the relevant average working hour field of infotype 0007. For example, if the employee has a biweekly payroll period, then the weekly working hours of infotype 0007 are multiplied by a factor two to fill the field "Working Hours per Payroll Period" of infotype 0008.

This field can be important in calculations within payroll. For that reason, care must be taken to assure that this field is filled correctly. If the recommendations of this KBA are followed, then this is assured even in the case of using a dummy work schedule due to a PWT replication.

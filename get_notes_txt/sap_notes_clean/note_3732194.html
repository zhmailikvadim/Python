Symptom

This article explains:

  • SuccessFactors – Microsoft Teams integration behavior when multiple Microsoft 365 Teams tenants are connected.
  • Whether the chat window opens with a user in an external account or with the same user’s guest account in the initiator’s tenant.

Environment

  • SAP SuccessFactors HCM Suite
  • SAP SuccessFactors Platform

Reproducing the Issue

  1. Log in to SF as a user.
  2. Search for a target user.
  3. On the target user’s people profile page, select "Microsoft Teams Chat" in the profile header to open a chat window.
  4. Observe it opens the user's external account instead of the guest account.

Cause

This is the current product limitation. SuccessFactors calls Microsoft Teams APIs to generate a chat link and returns it to the UI. There is no configurable parameter to force use of a guest identity versus an external-tenant identity.

Resolution

  • Note that configuring the "Microsoft Teams Chat" action to open a guest-user chat (instead of an external-tenant user chat) is not supported in SuccessFactors.
  • Note that disabling or hiding the "Microsoft Teams Chat" option in People Profile after enabling the integration is not supported. Please refer to: KBA 3527345 - How to disable the Microsoft Teams Chat Option from People Profile and Quick Card After Enabling Teams Integration.

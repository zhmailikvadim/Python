Symptom

You run the Audit tool (transaction USMM) or function module LSO_AUDIT to count LSO (Learning Solution) users for a calendar year, such as 2016. But the main client (such as client 003) of the LSO system doesn't capture any learner. The LSO user number is 0, (Engine id is 7554).

Environment

  • Human Capital Management (HCM)
  • SAP Learning Solution 6.0

Reproducing the Issue

  1. Go to transaction code SE37 to execute function module LSO_AUDIT;
  2. Click the 'Details View/Edit' icon of table 'PERIOD';
  3. Input start date as '01012016' and end date as '31122016', then click Back button;
  4. Execute this FM;
  5. Get the results in RESULT table;
  6. Click the 'Details View/Edit' icon of table RESULT, and the result list shows client 003 has 0 user number. But customer expects to see a correct number bigger than 0 in this client.

Cause

Audit transaction USMM calls LSO application function module LSO_AUDIT to capture the LSO users. This LSO_AUDIT function will check in table T77ZZ to find whether any LSO Harmonization report had been run in this client.

  1. Go to SE16 to search on records where field REPID contains keyword LSO;
  2. If there is record, it means LSO Harmonization report had been run.

    "Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental."

    If T77ZZ contains records for LSO reports, this FM will compare the dates between 'Program Run Date' and the 'Audit Period End Date'.
    If 'Program Run Date' > 'Audit Period End Date', system will stop capturing any LSO user, so the result will be 0.

Example case:
Customer activated LSO on 2016.06.01; Harmonization report was run at this activation stage.
But for a special reason, customer ran the Harmonization report again in 2017.03.01, the table T77ZZ was updated with this new date.
Later when customer ran LSO_AUDIT to calculate how many LSO users were in year 2016, system can't get any result.

It is because the Program Run Date (2017.03.01) > Audit Period End Date (2016.12.31).

Resolution

LSO Harmonization reports are supposed to be run at the beginning of LSO activation. If any report is run after it and cause 0 result in Auditing, please manually remove those LSO related reports records in table T77ZZ.

After deleting *LSO* reports in T77ZZ, the Audit function module can be run successfully to retrieve correct user number.

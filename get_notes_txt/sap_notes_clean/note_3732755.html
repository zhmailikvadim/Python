Symptom

  • When starting transactions EL28 or EL27 via Transaction Launcher in SAP Customer Engagement, the Business Partner ID is not passed to the selection screen.
  • The issue is browser-independent.
  • The issue is replicable without customer enhancement by setting user parameter WCF_IGNORE_ENHANCEMT = A (SU3).

Environment

  • SAP S4CRM for the releases:
    • S/4HANA 1909 
    • S/4HANA 2020
    • S/4HANA 2021
    • S/4HANA 2022
    • S/4HANA 2023 and higher

Reproducing the Issue

  1. Configure an action profile to launch transaction EL28 or EL27 from Customer Engagement via Transaction Launcher.
  2. Map and attempt to pass the Business Partner ID to the transaction’s initial selection screen.
  3. Start the action and observe that the Business Partner parameter is not passed to the selection screen.

    Cause

    • There is no access to the BP from the installation side.
    • The BP is not part of the CDS view CRMS4_IU_C_ILN (SE16N).

    Resolution

    • A separate action must be created for each object.
    • Alternatively, a relation must be used for the BP (Relation/@AttributeName). 
      - In Customizing, navigate to: SAP Utilities > Customer Engagement > General Functions > Define Utilities Business Layer Objects.
      - Select the Business Layer Object UtilitiesInstallation and identify the relation UtilitiesInstallationCurBuilHeaderRel.
      - Map and pass the Business Partner to EL28/EL27 using the value path UtilitiesInstallationCurBuilHeaderRel/@BP_NUMBER.
      - In BL Object BuilHeader, apply structure CRMST_HEADER_OBJECT_BUI and select the attribute BP_NUMBER.
      - Develop and utilize separate Action Profiles for distinct objects as necessary (e.g., one for the BP context and another for the Installation context).

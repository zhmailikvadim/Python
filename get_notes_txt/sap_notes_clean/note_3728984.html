Symptom

  • When trying to create multiple Freight Orders (FOs) through XML service TransportationOrderGenericRequest_In with reference to the same BTD ID, system throws error message:
    • "Business document with base BTD ID $1 and type code &1 already exists" (/SCMTMS/TEND 334)
  • The issue does not occur when creating FOs manually in the "Create Road Freight Orders" app in Fiori and assigning the BTD ID in Document References section.

Environment

  • SAP Transportation Management 9.x
  • SAP S/4HANA 1709 and above releases

Reproducing the Issue

  1. Create a Road Freight Order with reference to an ERP BTD ID
  2. Trigger the FO creation for a new document through XML TransportationOrderGenericRequest_In with the same BTD ID
  3. The XML is not successful and throws error message /SCMTMS/TEND 334

Cause

This is the standard/expected behavior. 

Resolution

  • By definition, the Base Business Transaction Document ID is a unique reference between a transportation order and the base business transaction document.
  • It ensures the new transportation document to be identified at a later stage. When TM receives an update message, the system uses the Base BTD ID to identify that exactly one existing Transportation Order/FO to be changed.
  • The UI and the TOR Generic interface apply identification differently. In the UI, the Base BTD ID is not used as the unique key to identify the FO. The system can still uniquely distinguish each FO because it has additional internal context for identification.

Keywords

Symptom

When you run the transaction (/US4G/MV_BDR) to trigger the BDR process and select the BDR process number hotspot, the system displays an error and the navigation to the Fiori app "Monitor Process Documents" fails. This issue affects the following processes:
- Sending of Business Data Request (Metering) (SUP View) (DE_B_MET_SEND_BDR_SUP)
- Sending of Business Data Request (DSO View) (DE_B_MDC_MET_SEND_BDR_DSO)

This issue is relevant for the market roles: SUP, DSO, MOS.

Other Terms

BDR(Business Data Request), Monitor Process Documents, Fiori app, /US4G/MV_BDR, Class /US4G/CL_MV_BDR, /US4G/CL_PDOC_UI, BM_MET_BDR_SUP, BM_MDC_MET_BDR_DSO, process document number, process document UUID, semantic object, semantic action

Reason and Prerequisites

The error occurred because the navigation destination was incorrect.

Solution

Implement the corrections of this SAP Note or import the relevant support package.

After implementing this SAP Note, the system uses the correct navigation destination, allowing you to navigate successfully to the Fiori app "Monitor Process Documents" when you select the BDR process number hotspot. To verify the fix, run the BDR transaction from the back end, then select the BDR process number hotspot. The navigation to the Fiori app "Monitor Process Documents" works without error.

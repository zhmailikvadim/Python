Symptom

The LOG_TOOL activity, when used without previous Tool Group selection in TOOL_LIST_DISPLAY does not work – the log tool entry data fields are not displayed.

Other Terms

LOG_TOOL

Reason and Prerequisites

Steps to Replicate:

ME 6.1.4. And same issue in ME 15.0.

Create Tool Group TG001 with one Tool Number. Attach to OP1 operation.

Create Tool Group TG002 with one Tool Number. Attach to the same OP1 operation.

Additionally, update the default Operation POD to add LOG_TOOL into Activities group button. Do not use the default "Tool List" button of the POD.

Click LOG_TOOL from Activities button.

Actual result: Log Tool Entry plugin is displayed empty. So, no possibility to select Tool Number or enter anything.

Expected result:

1) the system should find all attached groups;

2) display the first group in LOG_TOOL plugin and wait for the user to Save the entry;

3) when the Tool Number is logged, the system refreshes the plug-in and displays the next Tool Group, and so on till the list of groups is finished.

Solution

Code correction will be available in next patch releases for SAPMECORE 6.1.4, 15.0 SP1 and 15.0 SP 2.

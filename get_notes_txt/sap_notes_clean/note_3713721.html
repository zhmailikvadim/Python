Symptom

Error MH336 "Cannot use conf. class &1, class type &2 for generic article creation" occurs when trying to assign configuration class to merchandise category in t-code WG21/WG22. 

Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental. 

Environment

  • SAP ERP Logistics (LO)  
  • SAP S/4 HANA on premise
  • SAP S/4 HANA Cloud Private Edition

Reproducing the Issue

  1. Open t-code WG21/WG22.
  2. Attempt to assign a configuration class to a merchandise category. 
  3. Error MH336 "Cannot use conf. class &1, class type &2 for generic article creation" occurs. 

Cause

As explained in SAP Note 2352831, generic article has different data models in SAP ERP Retail and SAP S/4 HANA. 

The additional compatibility check functionality is provided to make sure generic article data can be distributed between SAP ERP Retail and SAP S/4 HANA systems. 

The subscreen "Assign Characteristics Profile Corresponding to Configuration Class" to assign configuration class to merchandise category is only shown when compatibility check is active. 

When assigning configuration class in t-code WG21/WG22, system will attempt to assign a characteristics profile with the same name as configuration class to the merchandise category. 

If the characteristics profile does not exist, system will create it automatically and assign it to merchandise category.

If the characteristics profile already exists but it's variant-forming characteristics different from the configuration class, error MH336 occurs. 

Comparing the variant-forming characteristics in the configuration class and the characteristics profile: 

  • Check the configuration class in t-code CL03, go to Char. tab. The configuration class contains 3 variant-forming characteristics. 
  • Check the characteristics profile in t-code CLWG, go to Char. tab. The variant-forming characteristics in characteristics profile must have relevancy indicator = 2. The characteristics profile contains only 2 variant-forming characteristics.
  • The variant-forming characteristics has difference between the characteristics profile and the configuration class. Therefore the configuration class cannot be used to create generic articles. 

Resolution

Whether the error need to be solved depends on if the generic article data and its classification data will be distributed to a SAP ERP Retail system. 

  • If there's no plan to distribute the data to a SAP ERP system, there's no need to deal with the error. 
    • In S/4 HANA systems, configuration class and its variant-forming characteristics are used for the creation of a generic article. There's no need to assign characteristics profile to the merchandise category. 
    • The additional compatibility check functionality can be switched off to prevent unnecessary characteristic profiles during the generic article creation. This can be done in t-code OMT4 by setting message MH334 to no message.  
    • After additional compatibility check functionality is deactivated, the subscreen "Assign Characteristics Profile Corresponding to Configuration Class" won't be shown in t-code WG21/WG22. 
  • If the data will be distributed to a SAP ERP system, the error need to be solved.   
    • In SAP ERP Retail systems, variant-forming characteristics from the merchandise category or a characteristics profile were taken into account for the creation of a generic article. Therefore the characteristics profile must be assigned to merchandise category to make sure data can be correctly distributed. 
    • In characteristics profiles, only the characteristics with relevancy indicator are considered as variant-forming characteristics. In S/4 HANA system, relevancy indicator cannot be manually maintained when maintaining characteristics profiles in t-code CLWE/CLWF. System automatically set this indicator when creating characteristics profiles base on configuration class in t-code WG21/WG22. 
    • If the characteristics profile is already created, but its variant-forming characteristics does not match configuration class, it's necessary the existing characteristics profile in t-code CLWH. The relevant objects having this characteristics profile assigned need to be archived/deleted first. 
    • After the characteristics profile is deleted, please assign the configuration class again in t-code WG21/WG22. Now system will create the characteristics again with the same variant-forming characteristics as the configuration class. 
    • If the existing characteristics profile cannot be deleted for some reason, then it won't be possible to use the configuration class in generic articles. The only workaround is to change to another configuration class.

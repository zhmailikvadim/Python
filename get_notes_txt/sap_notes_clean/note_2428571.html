Symptom

An update of a marketing document does not trigger the Approval process. This despite the Active when Updating Documents checkbox being selected in the Approval Templates - Setup window.

Reproducing the issue

Prerequisites:

Under Administration → Approval Procedures → Approval Templates:

  1. You have created an Approval Template for Purchase Order where the Active and the Active when Updating Documents checkboxes are selected.
  2. Approval Term is based on user queries.
    •  Example:  Select DISTINCT 'TRUE' from OPOR Where $[OPOR.Confirmed] = 'Y'

 Scenario: 

  1. On the Logistics tab, create a Purchase Order with the Approved option deselected. At this stage the Approval process is not launched.
  2. Reopen the Purchase Order added in step 1, select the Approved option on the Logistics tab and Update the document. 

Expected behavior: 

The Approval process is triggered when the document is updated.

Actual behavior: 

The Approval process is not triggered. 

Note: The Approval process is triggered on the Purchase Order with the Approved option selected on the Logistics tab, when the document is in Add mode. Similar behavior also occurs in the Sales Order level.

Solution

This behavior can no longer be reproduced in the version SAP Business One 9.3 PL0.

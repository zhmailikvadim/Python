Symptom

When executing the report RFM_MANAGE_SO_BACKGROUND for mass repricing, duplicate log messages are generated during processing. Due to redundant message logging, the number of collected messages becomes significantly higher than expected (for example, multiple success messages for the same sales order item). As a result, the message log grows excessively, leading to high memory consumption and, in some cases, runtime errors (dumps) during execution.

Other Terms

Re-Pricing, RFM_MANAGE_SO_BACKGROUND

Reason and Prerequisites

This issue is due to a program error.

Solution

Implement the attached corrections using SAP Note Assistant or import the relevant Support Package.

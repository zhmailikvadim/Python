Symptom

When you process collection agency information using transaction FP03I (program RFKKCOPN), the system posts charges and interest documents. The interest and charge amount in the administration data for the submission of receivables (table DFKKCOLL) is adjusted accordingly. However, the amount is not updated in the table DFKKCOLLH.

In addition, the interest and charge amount in the tables DFKKCOLL and DFKKCOLLH has an incorrect value if the system posts several interest documents for the same collection unit at the same time.

Other Terms

FP-COLLAG, collection agency, receivable submission, interest document, charges document, DFKKCOLL, DFKKCOLLH, FP03I, RFKKCOPN

Reason and Prerequisites

Reason:
Program error

Prerequisite:
You use enterprise services for submission to collection agencies (TFK000ES-XESCOLL - Central Settings for Enterprise Services is active). For a collection unit, a pure interest and/or charge request without payment is reported to the collection unit and posted with FP03I.

Solution

Implement the attached correction instructions or import the specified support package.

In the case of a pure interest or charge posting, the system does not create a new line in the table DFKKCOLLH because there is no change to the submitted item (for example, clearing, write-off, reset of clearing, or reversal) and the submission status does not change. As a result, the interest and charge amount is not updated in the last entry of the table DFKKCOLLH. After you import the Support Package or implement the correction instructions, the system adjusts the interest and charge amount in the last entry of the table DFKKCOLLH so that it reflects the current status of the management data in the table DFKKCOLL. The amounts are now updated correctly even if several interest documents are posted at the same time for the same collection unit.

Symptom

When synchronizing invoices between the Kazakhstan Invoice Journal (transaction FIKZREGINV or RPFIKZ_REGINV) and the Kazakhstan ESF (Electronic Invoice Information System), the following warning message appears:

  • An error occurred: E::000 Error when calling SOAP Runtime Functions:. For more information please see Application Log.

In the Application Log (function View Application Log), the following error is displayed:

  • Error when calling SOAP Runtime functions: soap:Server Лимит не может превышать 250 Лимит не может превышать 250
    J_3RF_DDX020

The error message translates to:

  • “The limit cannot exceed 250.”

Environment

  • SAP ERP 6.0
  • SAP S/4HANA

Cause

The issue is caused by a system limitation introduced by the Kazakhstan ESF portal.

The ESF system allows a maximum of 250 invoices per synchronization request (upload or download).

If more than 250 invoices are included in a single request, the ESF portal rejects the request and returns the above error message.

Resolution

To resolve the issue, maintain the synchronization limits in the SAP system.

  1. Go to transaction FIKZ_DI_BF.

  2. Set the following parameters to 250:

    • Limit of Invoices to Download at Once

    • Limit of Invoices to Upload at Once

After the adjustment, the system automatically splits invoices into batches of maximum 250 documents per request.

This ensures compliance with the ESF system limitation and prevents the error from occurring.

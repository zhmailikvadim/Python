Symptom

  • Bad performance of Data Flow run to load data from CDS View to Datasphere Local Table.
  • Extraction of the same table takes several seconds in ABAP.

Environment

  • SAP Datasphere

Reproducing the Issue

  1. Execute the Data Flow to load data from CDS View to Datasphere.
  2. Observe the prolonged execution time.

Cause

Resolution

Reduce the batch size in the Data Flow target table configuration -> Advanced section:

See  Add or Create a Target Table in a Data Flow | SAP Help Portal 

In addtion, Replication and Transformation Flows are now the recommended approach for loading and transforming data in SAP Datasphere. This next-generation capability offers enhanced connectivity, delta-load support, and resilient & scalable execution, while Data Flows will continue to be supported for existing workflows.

See  Use Replication Flows and Transformation Flows to Load and Write Data | SAP Help Portal 

See Also

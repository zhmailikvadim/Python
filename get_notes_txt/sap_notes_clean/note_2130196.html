Symptom

Invoice amount get doubled on Performance Obligation Structure UI when you take the following steps:

1. Create a contract with time-based performance obligation.

2. Create an invoice in one period.

3. Run program Revenue Posting Run in the period.

4. Do some contract change to make revenue changed in this period again, and run program Revenue Posting Run in the period again.

5. Run program Calculate Time-Based Revenue in the period.

6. Suspend the contract and then unsuspend it.

As a result, the invoice amount is doubled.

Other Terms

Revenue Accounting, Performance Obligation Structure UI, Calculate Time-Based Revenue

Reason and Prerequisites

If customer run program Revenue Posting Run before running program Calculate Time-Based Revenue, some contracts change will cause invoice amount doubled on Performance Obligation Structure UI.

It is caused by program error. Program Calculate Time-Based Revenue wrongly deletes the closed reconciliation key entries in table FARR_D_DEFITEM. In fact, the program only needs to request a new open reconciliation key to transfer the revenue.

Solution

Apply this note

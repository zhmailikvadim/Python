Symptom

In the process "Receiving Order for Changing Billing Data" (DE_B_UOP_ORDR_BILL_CHG_RECV), master data is not updated in the system based on confirmed attributes in the order for billing data change. This issue occurs in steps 0220 and 0250.

This issue is relevant for the market role: DSO.

Other Terms

UOP, Master Data Update, Settlement Unit, Grid Usage Billing, Product Code, UBD, /US4G/CL_PRST_MAP_PC_TO_MDX, DE_B_UOP_ORDR_BILL_CHG_RECV

Reason and Prerequisites

The error occurred because the mapping logic from product code to the exact UBD for master data updating was missing.

Solution

Implement the corrections of this SAP Note or import the relevant support package.

After implementing this SAP Note, the system maps the product code into UBD fields for master data updating. The BC set /US4G/3_0_DSO_SP08_NOTE_3730012 contains the necessary corrections. You can verify the fix by checking that master data is updated in the system based on confirmed attributes in the order for billing data change.

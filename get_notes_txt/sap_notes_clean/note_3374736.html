Symptom

This KBA addresses the possible reasons why OLAP cache may not be utilized for a Query over HCPR with a HANA CV parptrovider.

Environment

  • SAP NetWeaver
  • SAP BW/4HANA

Cause

a) Cache validity timestamp was not maintained: HANA CVs contain data outside of BW, therefore BW has no control over the data within or information on the validity of it. As the system cannot connect a validity timestamp (a.k.a "how up-to-date the data in the provider") to the data retrieved, no cache will be generated.

b) Warning "In data currentness 'All data (9)' only local cache can be used" occurs for the Query. This refers to the 'Request Status' property of the Query and this is the default setting.

c) In general, every mapped field in the HCPR and every field used in the Query must be associated with an InfoObject. Otherwise OLAP cache cannot be used.

Resolution

a) For this purpose, program RSR_ENFORCE_CACHE_FOR_HANAVIEW is offered which allows to manually set validity timestamps for these virtual providers, which will in turn allow for cache entry generation accordingly. More details in SAP Note 3055721 - Report setting OLAP cache timestamps for DataSources (using notes 2602920 and 3017657)

  • Note: as this is set by hand, it is important to keep the timestamp up-to-date when new data is available in the HANA CV, otherwise it may happen that outdated data is delivered by the Query. The same program offers a 'Cache delay' option to assist in automation.
  • Note 2: the timestamp is to be set for ODP of the HANA view (<2H*>) and not the HCPR or the Query.

b) Implement SAP Note 3557914 - OLAP Cache not used for HANA view.

Selecting something for the 'Request Status' setting of the Query that differs from the default (e.g 'All InfoProviders up to released status (RQTS0)') also resolves this warning and should allow for cache entry generation for such a Query, where timestamps was already maintained for the HANA view (see Scenario a)). However, after implementing this SAP Note, this change shouldn't be required any longer.

c) Make sure every mapped field in the HCPR and field used in the Query is associated with an InfoObject. This can be done in the modeling tools.

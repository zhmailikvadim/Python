Symptom

You archive actual postings' line items using archiving object COPA2. Some line items are not archived.

Other Terms

ACDOCA, WRTTP, RKEVRK4D, COPA, Archiving, COPA2, S4HANA

Reason and Prerequisites

Records posted as type 'Statistical Actual' can be posted by modification described in SAP Note 3490500.

Solution

Implement the correction instructions.

Please note that program RKEVRK4D is only a 'template program' for the generation of the 'actual' program RK4Dxxxx (xxxx = name of the operating concern), which is called during archiving.

In order to regenerate this program, call function module RKE_GENERATE_ARCHIVE in the test environment and start it for the respective operating concern.
Example: the name of the operating concern is 'S001', then you have to regenerate program RK4DS001.

Steps for generating program RK4DS001:
•    Call Transaction SE37 (Function Builder)
•    Function module: 'RKE_GENERATE_ARCHIVE'
•    Choose 'Single test' (F8)
•    Enter import parameters ERKRS = 'S001' and MODE = '*'
•    Choose 'Execute' (F8).

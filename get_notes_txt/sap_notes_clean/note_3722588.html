Symptom

To use EC-Time, the calculation rule DS3I - Determination of Basic Pay (§3b) Monthly Basis EC-Time is provided as an alternative to the calculation rule DS3C - Determination of Basic Wage (§3b) Monthly Basis used in the standard SAP schema*.

By using the alternative calculation rule DS31 instead of DS3C, the weekly working time required to determine the basic wage can be determined using the infotype Planned Working Time (0007) if the value is not available in the table for the work schedules (T508A). This is the case if a 'dummy' work schedule without hours is stored in table T508A.

(*) D000 Payroll Schema Germany -> DAL0 Factoring and Storage or D100 Payroll Schema Public Sector Germany -> DOAL Factoring and Storage

Other Terms

WOSTD P007

Reason and Prerequisites

When you use EC-Time, a termination occurs in the rule DS3C if no weekly hours can be determined using the work schedule from the table T508A.

As a prerequisite for using the new calculation rule, the enhancement of the payroll function NUM from SAP Note 3716206 - NUM: New parameters for returning the average working hours from infotype 0007 must exist in the SAP system.

Solution

Implement the attached correction instructions or import the specified support package. After you import the HR Support Package or implement the correction instructions, you do not have to perform any further follow-on activities.

If you use EC-Time, replace calculation rule DS3C with calculation rule DS3I if it is not possible or not always to determine the weekly working time using table T508A. Calculation rule DS3I is already commented out (inactive) in the standard SAP schemas DAL0 and DOAL. After reading table T508A, calculation rule DS3I checks whether the weekly working time is <> 0. If the value is 0, the weekly working time is read from the infotype Planned Working Time (0007).

Symptom

When attempting to change certain header fields like the posting date in T-code FBV4 for parked documents, the "Parked Document (CTRL+S)" button is grayed out, preventing the user from saving the changes.

Environment

  • SAP ERP
  • SAP ERP Central Component
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA
  • SAP S/4HANA Finance
  • SAP S/4HANA Cloud Private Edition
  • SAP Fiori for SAP S/4HANA

Reproducing the Issue

  • Navigate to T-code FBV4 and open a parked accounting document.
  • Modify header values such as Posting Date, Period, or Fiscal Year.
  • Attempt to save changes: The "P

Cause

This issue arises due to the user settings in T-code FB00 where the XCMPL indicator ("Save document as completed only") is activated. This setting disables the ability to park documents in all parking transactions, including FBV4. Consequently, any changes made to the header cannot be saved as Parked document. This behavior similarly affects T-codes FBV0 and FBV2. 

Resolution

  1. Access T-code FB00 and open User Parameters for Document Parking.
  2. Set the XCMPL indicator (save document as completed only) to inactive.
  3. Return to T-code FBV4, make changes to the document header (posting date/period/fiscal year), and use the Park option to save the changes.

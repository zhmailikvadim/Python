Symptom

  • When configuring user roles, there are two permissions for Reopen and Reopen Closed Worker
  • What is the difference between Reopen and Reopen Closed Profile Worker permissions

Resolution

Fieldglass user role permissions gives buyer administrators the ability to granularly set permissions relating to reopening worker records

The Reopen Worker permission for the Profile Worker and Worker module:

  • Users with this permission will be able to reopen Profile Workers and Workers that are currently Open but with a future Closed Date

SAP Fieldglass allows users to select a future End Date during the Actions > Close Worker action

Workers that have future end dates selected will remain Open and will be updated to Closed status once the end date has been reached

The Reopen Worker permission gives users the ability to reopen the worker before the actual End Date takes place so that worker will continue to be in Open status

The Reopen Closed Profile Worker permission and the Reopen Closed Worker permission allows for further control over objects in Closed status:

  • Users with this permission will be able to reopen Profile Workers and Workers that are currently in Closed status to Open status
  • Users without this permission will be unable to reopen a Worker or Profile Worker in Closed status

Users who have both permissions will be able to take both actions as outlined above

Symptom

When change entries for change types PAY0000400 and PROF000200 are generated at the same point of delivery (POD) at the same time for the variant GridUsageBillingData and ConsumptionPartition, the system triggers the process "MDC: Change of Grid Usage Billing Data from Responsible DSO to MP (DE_B_MDC_MAIN_SEND_DSO_RESP_GUB)" multiple times instead of once.

This change is relevant for all market roles.

Other Terms

 Change type PAY0000400, PROF000200, GridUsageBillingData, ConsumptionPartition, Variant Combination, MDX framework, POD, Point of Delivery

Reason and Prerequisites

The system didn't support the combination of change types PAY0000400 and PROF000200 for the variant GridUsageBillingData, resulting in multiple process triggers.

Solution

Implement the corrections of this SAP Note or import the relevant support package.

After implementing this SAP Note, when change entries for change types PAY0000400 and PROF000200 are generated at the same POD at the same time, the MDX framework triggers the process DE_B_MDC_MAIN_SEND_DSO_RESP_GUB only once. Based on your system version, import the BC set /US4G/3_0_COMM_2305_SP08_3727441 or /US4G/3_0_COMM_2602_SP00_3727441. You need to retest the master data changes to confirm that the process is triggered only once. You can verify the fix by checking that when change entries for change types PAY0000400 and PROF000200 occur concurrently at the same POD, the process DE_B_MDC_MAIN_SEND_DSO_RESP_GUB is triggered once.

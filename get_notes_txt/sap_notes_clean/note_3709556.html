Symptom

When executing transaction code FAGL_FCV for G/L Acct open items valuation, the posting fails due to error Message No. KI261 "Cost center XXXX/XXXXXX is blocked against revenue postings on YY.YY.YYYY" 

Diagnosis
Cost center XXXX/XXXXXX  in controlling area ZZZZ is locked for revenue postings on YY.YY.YYYY.

Procedure
Assign the account to another cost center or allow cost center XXXX/XXXXXX  to again accept revenue postings.

 

Environment

  • Financial Accounting
  • SAP ERP Central Component
  • SAP ERP
  • SAP S/4HANA Finance
  • SAP S/4HANA
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA

Reproducing the Issue

  1. Execute FAGL_FCV
  2. Enter values in field Company Code, Valuation Area, Valuation Key Date
  3. Enter selection in Open Items tab: G/L Acct

Cause

The cost center is blocked for actual and plan revenues and the affected GL Acct is defined as Cost element category 11 (Revenues). In transaction code KS03, affected Cost Center XXXX/XXXXXX is blocked for controlling area ZZZZ.

Cost Center       XXXX/XXXXXX
Controlling Area  ZZZZ

Lock
[x] Actual Revenues
[x] Plan Revenues

The affected G/L Account & Company Code is defined in transaction code FS00 as Cost element category 11 (Revenues). However FAGL_FCV doesn't raise error KI261 if the affected G/L Account & Company Code is defined as Cost element category 1 (Primary costs/cost-reducing revenues).

Resolution

The system is working as per current system design. The error KI261 is raised depending on the account posted and how it is defined regarding the cost element category.

Symptom

When a document is posted via FB01 or other posting transaction, the error message appears: FF822 'Tax code &1 defined incorrectly -> Rounding reverse charge &2 > 10 (CENT)'. 
It appears when a tax code is categorized for Reverse Charge.

Cause

Example:  following settings in FTXP:

Tax type              V        Input tax

 

Tax Type             Acct Key Tax Percent. Rate Level  From Lvl  Cond. Type

Acquisition Tax Out.    ESA      20.000-          200    100       NLXA
Acquisition Tax  Inc.    ESE      20.000            210    200       NLXV

Resolution

There are two resolutions.

1., based on SAP Note 694001:

Tax Type             Acct Key Tax Percent. Rate Level  From Lvl  Cond. Type

Acquisition Tax Cred ESA       20.000-        200          100          NLXA
Acquisition Tax Deb. ESE       20.000         210          100          NLXV

 

2. based on SAP Note 98181 point 1.:

Tax Type                               Acct Key Tax Percent. Rate Level  From Lvl  Cond. Type

MWVZ Acquisition Tax Out.         ESA        -20.000            200    100       NLXA
Acquisition Tax  Inc.                   ESE      -100.000            210    200       NLXV
(Both sign must be negative in case number 2.)

A Reverse charge tax code must be defined as INPUT tax code.
It is the logic of Reverse Charge tax code.
Defined like Input tax, and extracted from report RFUMSV00 on both side, Input tax line item and Output tax line item

The process is defined in a way that can only deduct input tax that have paid but in a case where import goods or services without VAT, it is needed to put the invoice on the same basis as domestic invoices with VAT, so it can booked with both input and output tax.

To do this use the accounting types ESA and ESE within a single tax code. "A" in this case denotes output tax "Ausgangssteuer" in German and "E" denotes input tax "Eingangssteuer". 
Put the tax percentage ESA against ESE.
Additionally these put either one or two different GL accounts according to the law of the land.
System will then calculate an equal and opposite posting.

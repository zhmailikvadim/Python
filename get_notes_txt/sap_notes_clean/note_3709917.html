Symptom

When saving the generic Job Location in Manage Data the  Job Location Long Format Generic Object is not updated even though country for both is matching.

Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental. 

Environment

SAP SuccessFactors Recruiting Management 

Reproducing the Issue

Pre-requisite: 

  • Country for Job Location Long format and Job Location are both the same.

  1. Go to Manage Data
  2. Select Job Location
  3. Select location entry with valid country value
  4. Note that even after being updated Job Location Long format has no entry.

Cause

Long format is not associated with an existing Job Location when the corresponding JobLocationLongFormat entry is created after the Job Location. Currently, long format is only populated when a Job Location is newly created or when its country is changed. If a Job Location was created before a long format was configured for its country and the user later opens and saves it without changing the country, the long format is not picked up.

Resolution

Engineering will be implementing code changes to link long format to the Job location during the save operation if no existing long format association exists.

The fix will be implemented for the 2H2026 release:

  • 12th October 2026
  • 13th November 2026

Symptom

During an application process the following dump occurs:

Category:                         ABAP programming error
Runtime Errors:               COMPUTE_INT_TIMES_OVERFLOW
Except:                             CX_SY_ARITHMETIC_OVERFLOW
ABAP: Program                SAPLSTS0
Application Component:  BC-SRV-TIM-TS

Environment

All SAP Basis releases

Reproducing the Issue

The dump can occur during any application process which uses a time stream for calculation.

Cause

This problem is caused by inconsistencies in the shared buffer of the application server. 

Resolution

  • Deactivate the buffering of the time streams based on the SAP note 
    3370301 - TSTR_CALC_TIME dumps with runtime error COMPUTE_INT_TIMES_OVERFLOW (exception CX_SY_ARITHMETIC_OVERFLOW)
    (The note do not have to be implemented, it only contains manual steps.)

  • If the issue still persist after then regenerate the time streams based on the SAP Note 3338373 - New report for generation of time streams
    (This note also do not have to be implemented, however if the report does not exist in the system
    then the coding of the report is attached to the SAP note)

    Use the report ZZTSTRREGENERATE without filling the time stream ID or the calendar ID
    and check the "Generation as per Time Stream " option. This will also check the "Invalidate Database" checkbox.

Symptom

You use the functionality 'Quick Overview' from /FRE/RWB transaction to review the calculated Order Proposals and corresponding quantities of the items.

In the scenario where you are using the Add-On for Fresh Products, you realize that the fields Order Date, Delivery Date, and Availability Date are not displayed in the 'Quick Overview' if an Order Split functionality doesn't occur.

Other Terms

FRP, order proposal quantity, traceability, , Order Date, Delivery Date, Availability Date, Intraday functions, Time-Based Scheduling, Time-Based Replenishment, Order Split (not active), CS_EXTENDED_DATA, BAdI /FRE/UI_QUICK_OVERVIEW_LA_BADI GET_DATA

Reason and Prerequisites

Technical dependency only if you do use SAP Forecasting and Replenishment Add-on for Fresh Products.

Although the OP scheduling dates are correctly calculated during FRP processing, the integration between the Add-On for Fresh Products scheduling framework and the Quick Overview is incomplete. As a consequence, the scheduling information is not transferred to the 'Quick Overview' user interface.

The structure CS_EXTENDED_DATA is not populated with the corresponding scheduling fields for the Order Proposals when an Order Split does not occur.

Solution

This SAP Note is relevant when you use Time-Based Scheduling in combination with the SAP Forecasting and Replenishment Add-On for Fresh Products.

After implementing the corresponding correction instruction in this note, the scheduling information in the 'Quick Overview' is consistent with the scheduling logic, improving transparency and traceability for Order Proposal analysis.

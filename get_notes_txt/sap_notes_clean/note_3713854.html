Environment

SAP Adaptive Server Enterprise Always-on (HADR) with SAP Business Suite

Cause

ASE uses CIS connection to connect RMA server, and whenever there is a successful connection since ASE reboot, ASE will save and keep the same login and use this login to connect to remote server (primary RMA in this context) for future connection. 

This scenario could happen during teardown and re-setup HADR while RMA will be up first, by that time role mapping for external login has not been defined. During this timeframe, the monitoring tool (for example DBACockpit) could keep connecting to primary ASE and query RMA proxy tables then it will be redirect to primary RMA, it will succeed because RMA will redirect password authentication to local SRS and local ASE. Even after role mapping for external login is defined, ASE will continue to use sapsa to connect to RMA, so RMA will keep reporting error that it was using sapsa to connect to local SRS for password authentication. 

Resolution

Reboot HADR to clear the connection information

Starting and Stopping the HADR System | SAP Help Portal

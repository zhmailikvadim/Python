Symptom

  • LMDB Data Supplier completeness check shows missing ABAPInstanceKernel
  • The registration of the ABAPInstanceKernel is not working, although you receive "sent successful" after executing the sldreg command

Environment

  • Managed System Operating System: Windows
  • SAP Solution Manager
  • Focused Run 

Reproducing the Issue

  1. Check the related windows system in LMDB which shows ABAPInstanceKernel missing
  2. Check the related windows system in SLD (via -> Adminitration -> Maintenance -> CIM Classes -> Supplied Elements [SLD version >= 7.1]) and check whether ABAPInstanceKernel information are not available there (for more information check KBA 2225446 - "ABAPInstanceKernel Data Supplier is shown as incomplete in LMDB")
  3. On OS level perform the steps mentioned under "Reason and Prerequistes" from SAP Note 1419603 - "Missing ABAP SAP Central Services (ASCS) instance in SLD/LMDB"  (means no manual creation)
  4. On OS you cannot find files dev_sldregs and dev_sldregk in <X>:\usr\sap\<SAPSID>\SYS\global

Cause

The problem is caused by linux binaries in <X>:\usr\sap\<SID>\<Instance>\exe

sapstartsrv calls krnlreg and sldreg without adding the file suffix (exe). This means that if there is "…\exe\krnlreg" and  "…\exe\krnlreg.exe" in the directory, the OS will call the first one – which will fail due to incompatibility with the operating system

This can happen due to linux kernel unzipped into the Windows folder.

Resolution

Either remove or rename the two linux binary files (sldreg and krnlreg) and keep the 2 windows files. Afterwards follow again the steps described in SAP Note 1419603 - "Missing ABAP SAP Central Services (ASCS) instance in SLD/LMDB" in section "Reason and Prerequistes" or KBA 2225446 - "ABAPInstanceKernel Data Supplier is shown as incomplete in LMDB" and check SLD and LMDB again

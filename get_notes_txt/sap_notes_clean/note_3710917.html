Symptom

  • While editing the dependents portlet, Personal Information is missing for past records, like first and last names. 
  • Attempting to save the missing name information results in the error: "You are trying to modify multiple records with the same keys (start-date, person-id-external). Please review the data and try again."
  • Additional observation: Adding a new future effective record with fields filled resolves the issue temporarily. 

Environment

SAP SuccessFactors Employee Central

Reproducing the Issue

  1. Access the profile of an affected user.
  2. Navigate to the Dependents block and open the history.
  3. Observe that the dependent personal information or names are missing.
  4. Attempt to edit the record by entering the first name and last name of the dependent.
  5. Save the changes.
  6. Observe the error message: "You are trying to modify multiple records with the same keys (start-date, person-id-external). Please review the data and try again."
  7. Add a new future effective record with missing fields filled.
  8. Save the record and observe that the fields reappear in the record. While deleting the new record, information appears properly in the history.

Cause

There are data issues in the Personal Information of the dependents.

Resolution

  1. Utilize the Check Tool to identify and correct dirty data in dependent records:
    1. Navigate to Check Tool > System Health.
    2. Open Employee Central Core > Dependents.
    3. Run the check "Time slices of a dependent’s records have start dates earlier than end dates (StartDatesOfDependentRecordsEarlierThanEndDates)."
    4. Export the results to view the personal information of the dependents.
  1. Prepare a file for Personal Information based on the exported results.
  2. Import the corrected details using the Full Purge mode.
  3. Verify the dependents' data to ensure the issue is resolved.

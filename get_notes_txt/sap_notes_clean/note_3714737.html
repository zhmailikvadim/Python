Symptom

When tracing a work process at level 3 for the database (component B), a DBSQL_INTERNAL_ERROR occurs when the kernel module abosqlstmt_prepare is called.

The internal note in the short dump shows the following information:

The termination was triggered in function "checkTokenIndex"
of the SAP kernel, in line ### of the module
 "//bas/GIT/krn/include/dbi/parser/dbi_parser_token_registry.h".
The internal operation just processed is "METH".
Internal mode was started at ###.
"index is not set"

Also in the developer trace of the work process, the output of the CL_OSQL_STATEMENT_FACTORY=>COMPILE( ) tree ends unexpectedly with lines of
------------------ C-STACK ----------------------
...
------------------ c-stack end ------------------

And short dump occurence is also registered:
A  Error Code DBSQL_INTERNAL_ERROR.

Other Terms

DBSQL_INTERNAL_ERROR, "index is not set", abosqlstmt_prepare, checkTokenIndex

Reason and Prerequisites

The problem occurs because the trace printing function attempts to retrieve the token index and token text for node, which does not have a token (for example FIELD_LIST_ENTRY).

Solution

In the patched kernel, if the node has no token, value token="None" is printed:

Example:

<FIELD_LIST_ENTRY token="None" tree_node_type="FIELD_LIST_ENTRY">

Symptom

Change documents related to charge calculation in Freight documents are creating excessive CDPOS entries. For example, the field CALC_DATE_TSTMP (Calculation Timestamp) in ITEMCHARGEELEMENT node is recorded for all charge elements during every recalculation, even though tracking this field may not be business-relevant.

Other Terms

CDPOS, change document, change history, logs, transportation charges, charge calculation, DO, /SCMTMS/TCC_TRNSP_CHRG, CALC_DATE_TSTMP, Calculation Date Timestamp, /SCMTMS/D_TCHRGE, ITEMCHARGEELEMENT, freight order, charge element, field exclusion, BAdI, Enhancement Spot.

Reason and Prerequisites

The change document framework records all field-level changes for Transportation Charges (BO /SCMTMS/TCC_TRNSP_CHRG).

Solution

Implement the Correction Instructions or upgrade to latest support package.

New Objects Created:

Description

/SCMTMS/EI_TCC_CDO_FLD_EXCL
/SCMTMS/BI_TCC_CDO_FLD_EXCL

Default BAdI implementation (inactive by default)

/SCMTMS/CL_TCC_CDO_FLD_EXCL

Implementation class

Default Behavior:

The default BAdI implementation /SCMTMS/EI_TCC_CDO_FLD_EXCL is delivered inactive. When activated, it excludes the field CALC_DATE_TSTMP (Calculation Timestamp) from change document logging for charge elements.

After SAP Note Implementation:

To apply this fix, perform one of the following:

1. Activate the delivered BAdI implementation:

  • Go to transaction SE19
  • Activate Enhancement implementation /SCMTMS/EI_TCC_CDO_FLD_EXCL

2. Create a custom implementation:

  • Create your own BAdI implementation for /SCMTMS/BADI_TCC_CDO_FLD_EXCL
  • Implement method MODIFY_DB_TABLE_DATA to exclude additional fields as per your requirements

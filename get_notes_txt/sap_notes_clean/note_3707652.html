Symptom

  • In SAP for Me, the provision of Datasphere tenant for the SAP Business Data Cloud product package fails or is stuck in "Processing" status.
  • Following error message is displayed, if you expand Solution type entry
    "The host name XXX is already being used. Please create a new solution with another host name."
  • Without expanding, the error message is : 
    Solution provisioning has failed. Please contact customer support.
    This solution also has 1 message on one of its tenants. Please expand the solution and review the tenant notification.
     
     

Environment

  • SAP Datasphere
  • SAP Business Data Cloud

Reproducing the Issue

  1. Attempt to provision a Datasphere tenant with the host name "XXX".
  2. Observe the provisioning fails or is stuck in the "Processing" status with error message: "The host name XXX is already being used. Please create a new solution with another host name.".

Cause

The host name "XXX" has already been used for the BDC tenant or SAC tenant. Host name must be unique for all tenants(BDC, Datasphere, SAC), it is not allowed to have two instances with the same host name.
It is possible that the initial tenant provisioning job failed and the tenant was only partially created and set to be automatically deleted. The deletion process, however, only runs once a day.

Resolution

If you have an existing tenant with If you can see that a tenant already exists with the same name, then delete the failed instance that has the duplicate host name "XXX".
It may be the case that the system itself already set the failed tenant instance to be deleted automatically, but until it is actually deleted, the hostname is considered taken.

The host name can be checked by clicking on the corresponding instance, a panel opens to the right with more details including the data center and hostname for this tenant.

Please also note that it is only possible to reuse the same hostname from the deleted instance after 7 days, as the deleted instance remains in system for 7 days before being permanently deleted. In fact, it could take close to 8 days until the tenant is actually deleted and the hostname is available again. That's because the job that changes the status from "Suspended" to "For Deletion" for those tenants with Suspended status for entire 7 days runs at 00:00 of UTC time. The job that actually deletes the tenants with status "For Deletion" runs every day around 03:00 UTC time. So, if your tenant completes 7 entire days in Status Suspended close to the last job execution (00:00 UTC), then it will only be set to status "For Deletion" on the next run. Therefore, it may take close to 8 days until the hostname is available again.

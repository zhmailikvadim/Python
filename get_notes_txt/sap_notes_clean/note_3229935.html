Symptom

The Avail. to Release field in the Pick Pack and Production Manager window is not updated correctly for an Assembly BoM (Bill of Materials), if the warehouse was changed in the Sales Order.

Reproducing the issue

Prerequisites:

1. Create two items: AA - sales item and BB - inventory item.

2. Go to Production→ Bill of Materials and create Assembly BoM AA with component BB in warehouse 01, quantity 1.

Scenario:

1. Create a Goods Receipt for item BB with two lines: the first for warehouse 01, quantity 4 and the second for warehouse 02, quantity 4.

2. Go to Sales - A/R→  Sales Order and create a new Sales Order with item AA and with quantity 2. Before adding the document, update the warehouse to 02.

As a result, the following message appears: Do you want to set this warehouse for component items as well? Choose "No" to keep the component items' default warehouses defined in the BoM.

Press Yes and add the document.

As a result, the committed quantity of the item BB is updated correctly in warehouse 02.

3. Go to the Inventory→ Pick and Pack→ Pick Pack and Production Manager and choose the Sales Order you created and press Ok.

4. Mark the line of the Sales Order and press 1 in the column To Release. You can see that the Avail. to Release quantity is 4.

Click the Release to Pick List button and generate the pick list.

5. In the Pick Pack and Production Manager window → under Open tab, Check the Avail. to Release quantity.

Expected behavior:

The Avail. to Release quantity is updated to 3.

Actual behavior:

The Avail. to Release quantity is 4.

If you answer No to the question - Do you want to set this warehouse for component items as well? Choose "No" to keep the component items' default warehouses defined in the BoM from and the warehouse, the Avail. to Release Quantity ﻿is updated correct.

Cause

System Limitation

Solution

This is a system limitation.

Workaround:

As a workaround, you can use a Sales BoM instead. The difference is that the Assembly BoM displays the parent item in pick lists, while the Sales BoM displays component items in pick lists. As a result, the quantity in Avail. to Pick is correct for the Sales BoM component.

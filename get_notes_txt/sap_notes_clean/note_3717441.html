Symptom

When the source document (transaction) currency and agreement currency are different, expect the condition value to be converted correctly in the calculation bucket. However, the currencies are not filling properly, which leads to incorrect values being stored and processed to the calculation bucket level.

This note corrects the behavior by ensuring that currency conversion is handled properly when source document and agreement currencies differ.

Other Terms

calculation bucket, currency conversion issue, source document currency, agreement currency, condition value, incorrect conversion

Reason and Prerequisites

This is caused by program error

Please implement SAP Note#3688808 as a Prerequisite note for 2022 V2 and 2023 Release

Solution

Please implement the correction instructions attached. These corrections are only valid if you have applied the Support Packs.

Release V/4 – 2022 V2 and 2023

Use transaction SNOTE to implement the correction instructions specified below and activate the affected program object.

Symptom

You use the Theme Designer with your own application preview, as described in e.g. Adding Applications for Preview Purposes. You intend to use the Control Picker on it, as described in e.g. Adjusting Individual Parameters Using Expert Theming. But the Control Picker is disabled and you can not use it.

Other Terms

UI Theme Designer, Theme Designer, Application Preview, Preview, Control Picker, Element Picker, Picker, Disabled, SAP Build Work Zone Advanced, Work Zone Advanced, SAP Build Work Zone Standard, Work Zone Standard, Work Zone, Fiori Launchpad, Launchpad, Same-Origin, Cross-Origin, Origin

Reason and Prerequisites

The Control Picker inspects the document object model (DOM) of the selected preview and the documents the selected preview embeds. For security reasons, browsers only allow that if the Theme Designer and all embedded documents have the same origin (see MDN: Same-origin policy). Hence, the Theme Designer disables the Control Picker if the selected preview has a different origin or embeds content from a different origin.

Especially launchpads like SAP Build Work Zone, advanced edition or SAP Build Work Zone, standard edition are known for embedding content from different origins. If you used your launchpad as an application preview, even though the launchpad itself was same origin, it is very possible that it embedded cross-origin content and therefore the Control Picker had to be disabled.

Solution

When using your own application preview, make sure that it has the same origin has the Theme Designer, and that it does not embed cross-origin content. You could e.g. create a separate "theming preview" stripped-down work zone site, that does not embed cross-origin content, and use that.

Or you rely on the existing Control and Application Previews provided with the Theme Designer. Especially the Launchpad preview from SAPUI5 Application Previews shows how a launchpad would look like with your theme applied.

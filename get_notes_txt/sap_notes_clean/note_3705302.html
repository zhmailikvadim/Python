Symptom

  • Autosys jobs launching with server group through the exported command .bat file are not triggered as scheduled.
  • Batch job manual execution with server group fails with the error message: "[while trying to invoke the method java.lang.Object.hashCode() of a null object loaded from local variable 'key']".
  • Removing one of the job servers from the server group, the job runs on the server group without issues
  • Putting that job server back in the server group, the server group has an error when executing the job
  • Jobs run successfully on individual servers but fail when executed by the job server group.
  • Error in AL_RWJobLauncher log:
    Launch response:
    (14.3.3.222) 01_19_2026 15:52:19 (9144): Network failure. (BODI-1250100)
    (14.3.3.222) 01_19_2026 15:52:49 (9144): Launching Job and waiting for completion. inet address , GUID <43983efb_00b1_4cb6_97c4_fed2ffe6f943>. (BODI-1250132)
    (14.3.3.222) 01_19_2026 15:52:49 (9144): *** RWJL_EXIT called.
    (14.3.3.222) 01_19_2026 15:52:49 (9144): *** Launching Job <43983efb_00b1_4cb6_97c4_fed2ffe6f943> failed. Error: <>. (BODI-1250142)

Environment

  • SAP Data Services 4.3

Reproducing the Issue

  1. Upgrade from Data Services version 4.2 to 4.3 SP03.
  2. Define a job server group
  3. Go to the repository and select a job.
  4. Execute the job using the server group
  5. Observe the error: "Batch job 'JOB_EDP_PROJ_WBS_INTERFACE' failed [while trying to invoke the method java.lang.Object.hashCode() of a null object loaded from local variable 'key']".
  6. Execute the same job on individual servers, and observe that it runs successfully.

Cause

  • The job server on one of the servers in the server group (SAPCOE1997) is malfunctioning.

Resolution

  1. Remove the problematic job server from the server group.
  2. Create a new job server on that server host.
  3. Associate the new job server with the repositories.
  4. Add the newly created job server to the server group.

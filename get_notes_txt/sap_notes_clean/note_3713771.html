Symptom

Conditions have been maintained in below path for restart but the flexible workflow is not reset based on the change in PR:
OLME -> Purchase Requisition -> Flexible Workflow for Purchase Requisitions -> Manage Conditions to Restart Flexible Workflow for Purchase Requisitions

Environment

  • Materials Management (MM)
  • SAP FIORI
  • SAP S/4HANA

Reproducing the Issue

  1. Maintain conditions for restart.
  2. Change the field maintained in step 1 in PR.

Cause

Conditions to Restart Flexible Workflow only consider PR with creation R and S. 

Resolution

Modify the restart logic using BAdI MM_PUR_S4_PR_WFL_RESTART for creation indicator other than S and R.

See Also

SAP Note 2848446 - Manage Workflow restart for Purchase Requisitions with any creation indicators via BAdI

SAP KBA 3498800 - Fields which trigger a restart of Flexible Workflow for Purchase Requisitions - SAP S/4HANA

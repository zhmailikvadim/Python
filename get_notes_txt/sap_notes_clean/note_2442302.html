Symptom

During execution of report RKD_REGENERATE_ALL_REPORTS you run into dump COMPUTE_INT_TIMES_OVERFLOW. The dump occur in SAPLKYPG ind SET_SWAHL_DWAHL. The dump also occur, in case you run the RKD_REGENERATE_ALL_REPORTS with setting "Regenereate All - Reset Settings".

The reason for the dump is a inconsistancy in report definition. To get rid of the inconsistancy a regeneration using the monitor of the drilldown reporting tool is needed. The usage of RKD_REGENERATE_ALL_REPORTS with setting "Regenereate All - Reset Settings" should also fix the inconsistancy.

Other Terms

KE30, KCR0, KEMO, RKD_REGENERATE_ALL_REPORTS, SET_SWAHL_DWAHL, COMPUTE_INT_TIMES_OVERFLOW, SAPLKYPG

Reason and Prerequisites

There is a program error.

Solution

Please implement the attached correction instructions. After this the dump will no longer occur in case you use setting "Regenereate All - Reset Settings" for report RKD_REGENERATE_ALL_REPORTS.

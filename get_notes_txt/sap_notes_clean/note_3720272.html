Symptom

An error occurs when attempting to establish a tunnel connection between SAP Analytics Cloud (SAC) and SAP Datasphere (DSP) to schedule a SAC story based on a SAP DSP Live Connection.

  • "We couldn't connect to your SAP Datasphere tenant. For more information, see our troubleshooting page.."
  • ''500 Internal Server Error'' recorded in Developer tools.

Environment

  • SAP Analytics Cloud
  • SAP Datasphere

Reproducing the Issue

  1. Navigate to your SAP Analytics Cloud (SAC) URL.
  2. In the SAC interface, go to the Connection tab.
  3. Select the option to create a "New Connection."
  4. Choose "Connect to Live Data" and then select SAP Datasphere.
  5. In the connection type options, select "Tunnel" and provide the necessary SAP Datasphere URL.
  6. Attempt to save the configuration.
  7. Observe the correlation error message preventing the connection from being established.

Resolution

Before troubleshooting, ensure the following prerequisites are met:

  1. SAML IDP Configuration

    • SAC and DSP must connect to the same SAML Identity Provider (IDP).
    • The same NameID attribute must be used in both SAC and DSP.
    • The user must exist in both SAC and DSP.

  2. Data Center & Landscape Alignment

    • Both SAC and DSP tenants must be in the same data center and same landscape.
    • Cross-data center connections are not supported.

  3. UCL Formation Configuration

    • SAC and DSP must be linked via UCL Formation as per the guide:
      • Configure Data Storage for Planning

  4. Scheduling SAC Story with DSP Live Connection

    • If the goal is to schedule a SAC story based on a DSP live connection, refer to:
      • Schedule a SAP Analytics Cloud Story Based on SAP Datasphere Live Connection

If all prerequisites are met but the issue persists, perform the following steps:

Assign the Scoped Role in SAP Datasphere

  1. Log in to the SAP Datasphere tenant.
  2. Navigate to Security > Users & Roles.
  3. Locate the affected user and assign the Scoped Data Warehouse Cloud Consumer role.
    • This role is required for live data connections between SAC and DSP.

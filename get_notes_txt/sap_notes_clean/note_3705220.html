Symptom

  • In the Manage Purchase Contracts app the system does not allow to edit a contract when the activity 02 (edit) is assigned for the user, but activity 01 (create) is not assigned.
  • Without activity 01 assigned to the role, the system shows the authorization error: "No Authorization to update instances of the node of BO".

Environment

  • SAP S/4HANA 2022 onwards
  • SAP Fiori
  • SAP S/4HANA Cloud Private Edition

Reproducing the Issue

1. Access the Manage Purchase Contracts app with a user that has activity 02 assigned to it, but not activity 01
2. Click on the Edit button
3. Get the authorization error

Cause

Standard system behavior.

It is now possible to save the purchase contracts with the button 'save In Preparation'. This option allows saving a draft before confirming the changes in the original document. As a new draft has to be created to enable this feature, the Activity 01 is required to edit the documents.

Resolution

There are 2 options to avoid the error and allow editing the contracts:

a) Assign the activity 01 to the user role.

OR

b) Use the BAdI MM_PUR_S4_CTR_CHECK to implement a custom logic and throw error messages based on specific requirements.

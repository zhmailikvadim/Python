Symptom

Upgrading B2B addon to 2.0 there are performance issues while saving functional profiles in TPM

Other Terms

TPM, Performance, OOM, Funtion Profiles, B2B

Reason and Prerequisites

Issue is caused due to growing tables for every object chnaged

Solution

Issue is resolved by creating 2 DB indexes. Fix will be made available from PO 750 SP 20 onwards.

As a workaround for other SP releases, create 2 indexes on table XI_DIRSYNCADMIN

Index 1: Create an index on 5 columns "CHANGELISTID", "USERNAME", "TYPEID", "ISDELETED", and "OBJECTID" and allow reverse scans

Index 2: Create an index on 3 cloumns "CHANGELISTID", "TYPEID", and "USERNAME" and allow reverse scans

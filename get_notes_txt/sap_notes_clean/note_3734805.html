Symptom

We would like to to send Application Cost data from a table in Apptio to Application factsheets in LeanIX.

Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.

Resolution

Here is a Step by Step guide to setup and test the integration to Send Application Cost data from Apptio to LeanIX

Step 1: Initial Setup in Apptio

This is the part where you need support from your Apptio Admin to perform the following:

  1. Create a service user in Apptio Frontdoor application with "Data Link Admin" role 
  2. Grant the service user access to your Apptio environment
  3. Capture the API keyAccess and keySecret during initial user profile creation
  4. Identify your Apptio environment details:
    • loginUrl: https://frontdoor.apptio.com/service/apikeylogin 
    • domain name containing the tables to be used
    • environmentName
    • ulsUrl (Uploader Service URL) based on your region:
      • EU: https://datalink-eu.apptio.com/uls/api/v1 
      • US: https://datalink.apptio.com/uls/api/v1

Step 2: Configure Authentication in SAP LeanIX

  1. Navigate to the Administration area in SAP LeanIX 
  2. Go to the Integrations section
  3. Click "Configure" in the Apptio section
  4. Configure the apptioConnectionConfig section with:
    • loginUrl from Step 1 
    • keyAccess and keySecret from Step 1
    • domain and environmentName from Step 1
    • ulsUrl based on your region from Step 1 

Step 3: Data Mapping Configuration

Configure apptioReadConfigs for the Applications table you would like to read the data from and Save the configuration. Please note that the url needs to be generated in Apptio and the format used is the tsv format or the legacy tsv format from the tables in TBM Studio.

Please check with your Apptio Admin if you are not sure about the link format.

Step 4: Testing the Apptio download

The aim of this step is to test if the integration is able to initially fetch the data from Apptio

  1. Click on the download button which would download your Apptio table data in a JSON format that is compatible with the integration API input format.
  2. Save the Json output in a text / Code editor to be used for your later tests in the next step

Step 5: Integration API Setup

The integration API Connector setup is the step where you define which Columns from Apptio you would need to send to which factsheet fields in LeanIX.  For this, you need to make sure that your table has at least one ID that is shared between the two systems and that is defined in LeanIX as a field of type externalId. 

  1. Navigate to the Administration area in SAP LeanIX 
  2. Go to the Integration API section
  3. Create a new Connector with the following details:
    1. Connector Type: lxApptio
    2. Connector Id: lxImportFromApptio
    3. Connector Version: 1.0.0
    4. Processing Direction: inbound
    5. Processing Mode: Partial
  4. Once created, you can add your processors as needed.
  5. In this step, you can use your real input data that you downloaded from Apptio in the Input part of the integration API. We recommend to limit your data object to one application object for the initial test

For an example of a Connector that you can use for reference to build your own, check the "See Also" section below  

Once your tests are finalized, you can raise a ticket for the Support team to run the workflow to automatically schedule the integration run for you.

Symptom

  • CAP OData access hitting error : 500 - TimeoutError: ResourceRequest timed out.
  • cds watch call stack shows:

    [odata] - GET /odata/v4/XXXX/YYYY
    [error] - 500 - TimeoutError: ResourceRequest timed out
        at ResourceRequest._fireTimeout (/home/user/projects/../node_modules/generic-pool/lib/ResourceRequest.js:62:17)
        at Timeout.bound (/home/user/projects/../node_modules/generic-pool/lib/ResourceRequest.js:8:15) {
      code: ''
    }

  • OData client side you may see error:

    {
        "error": {
            "message": "ResourceRequest timed out",
            "code": "",
            "@Common.numericSeverity":
        }
    }

Environment

  • CAP Node.js
  • HANA Cloud

Reproducing the Issue

  1. Run CAP OData request which access HANA Cloud.
  2. General configuration:


Cause

HANA Cloud Database is not running.

Resolution

  1. Make sure HANA Cloud Database is up and running.
  2. HANA Cloud -> Manage Configuration -> Connections -> Allowed connections, ensure CAP IP is allowed by HANA Cloud
  3. If the error still occurs, ensure general configuration:

    https://cap.cloud.sap/docs/get-started/troubleshooting#why-are-requests-occasionally-rejected-with-acquiring-client-from-pool-timed-out-or-resourcerequest-timed-out
    https://cap.cloud.sap/docs/node.js/databases#databaseservice-configuration

Symptom

When attempting to log in to the SAP Financial Consolidation Windows client using Windows Authentication, the login fails and the following error is displayed:

"The Windows user could not be authenticated by the external authentication provider."

Environment

Financial Consolidation 10.1

Reproducing the Issue

  1. Launch the windows client application.
  2. Choose Windows authentication for logon and enter credentials.

-> The login fails with the following error:

"The Windows user could not be authenticated by the external authentication provider."

Cause

When a "Windows Account" authentication is used, The Financial Consolidation application will not connect to the BI platform (BOE/IPS) for the authentication but will create a direct connection to the Active Directory of the concerned domain.  For this authentication method to work, no external authentication must be defined in the FC administration console.

Resolution

Make sure that the “External authentication configuration string” is set to “deactivated”.

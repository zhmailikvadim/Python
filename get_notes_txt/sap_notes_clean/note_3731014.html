Symptom

On sites with data isolation enabled and with some specific client configuration, a NullPointerException is thrown when DefaultCustomerEmailResolutionService.getEmailForCustomer() is called.

Reason and Prerequisites

Reason:

The class DefaultCustomerEmailResolutionService has a property multiSiteUidDecorationService declared but:
- No setter method is provided
- The property is not configured in commerceservices-spring.xml

As a result, multiSiteUidDecorationService is null at runtime.


Prerequisites:

The issue occurs when ALL of the following conditions are met:

1. Commerce version 2211.xx and 2211-jdk21.xx installed. Commerce version 2205 will not be effected.

2. On a site with data isolation enabled
3. Customer type is not GUEST and Customer originalUid is null, or Customer type is GUEST and Customer sapGuestUserEmail is null and email cannot be derived from originialUid

Solution

Workaround:

1. In file:
commerceservices/src/de/hybris/platform/commerceservices/customer/impl/DefaultCustomerEmailResolutionService.java

Add the following methods:

protected MultiSiteUidDecorationService getMultiSiteUidDecorationService()
{
return multiSiteUidDecorationService;
}

public void setMultiSiteUidDecorationService(
final MultiSiteUidDecorationService multiSiteUidDecorationService)
{
this.multiSiteUidDecorationService = multiSiteUidDecorationService;
}

2. In file: commerceservices/resources/commerceservices-spring.xml

Update the defaultCustomerEmailResolutionService bean definition to add a property "multiSiteUidDecorationService":

<bean id="defaultCustomerEmailResolutionService"
class="de.hybris.platform.commerceservices.customer.impl.DefaultCustomerEmailResolutionService">
  <property name="configurationService" ref="configurationService"/>
  <property name="multiSiteUidDecorationService" ref="multiSiteUidDecorationService"/>
</bean>




Correction:

The correction will be delivered in the following versions:
- 2211.51
- 2211-jdk21.10

Symptom

In the SAP Ariba Buying and Invoicing that is integrated with SAP ERP or S/4HANA using SAP Integration Suite, managed gateway for spend management and SAP Business Network, when processing previous purchase order information for the same Scheduling Agreement, the system tries to handle a very large amount of data. This results in the system consuming too much memory, which causes it to stop unexpectedly (system dump) and creates locking issues. Goods receipt was not being created/sent to CIG/BN due to which unable to create invoice.

Cause

This issue occurred due to the erroneous code.

Resolution

Now, the code has been enhanced to resolve the problem, please implement this SAP Note. For more information, please see SAP NOTE 3687224.

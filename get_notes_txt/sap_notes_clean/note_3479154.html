Symptom

You use direct TM-EWM integration for transit warehousing (TW). You want to define the package IDs in the very first TW already at TM side, therefore you built the PKG item hierarchies already in the FWO and maintained the PKG IDs for the FWO subPKG items. The discrepancy handling in the FO doesn't work.

Other Terms

TransportationOrderLoadingAppointmentRequest_In, TransportationOrderLoadingAppointmentNotification_In

Reason and Prerequisites

In the standard transit warehouse scenario the very first LDAP request message is an inbound LDAP request message from TM to EWM which contains the PKG items without package IDs and without PKG hierarchies. EWM creates the real HUs and EWM generates the HU numbers. These are sent back to TM as PKG item hierarchy (topHU and subHU) in the LDAP notification message and the hierarchies and package IDs get updated in FU and FO by the LDAP notification message.The sub PKG items have the HU numbers/PKG IDs. The subsequent outbound LDAP request message to the EWM system will already contain the PKG item hierarchy meaning the PKG IDs generated in the EWM system at the beginning are propagated through the chain.

Solution

Don't send PKG hierarchy and package IDs in the very first inbound LDAP request message, because if you do so, the end to end TW process (e.g the discrepancy handling) doesn't work.

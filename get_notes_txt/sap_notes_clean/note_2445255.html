Symptom

For Campaign Automation, and for a workflow that uses task TS14007952 "Send Target Group to Channel" steps are executed as a background job. Job name is always the same (eg. "Target Group to Channel").

Job numbers are created in table "TBTCO" using Job name. Jobs can be cleared out using report "RSBTCDEL2". As a result, it is possible to have a job number that was already used by a campaign automation. This causes the system to raise errors indicating that the job number is already used.

Message class: CRM_MKTTGGRP_EXP
Message number: 304
Message: "Job with name &1 and number &2 already exists"

Other Terms

TBTCO, RSBTCDEL2, CRMD_MKTTG_JOB, Batch Job, Job Name, Campaign Execution, Campaign Automation, BUS2010022, EXECUTE.

Reason and Prerequisites

This is caused by program error.

In release CRM 700 and earlier, method cl_crm_mktpl_functions=>export_tggrp is called that uses old design logic in BOR object BUS2010022, method EXECUTE.

Design change to make the Job name unique by appending the timestamp to the job name.

Solution

Implement the attached correction instructions or import the specified Support Package.

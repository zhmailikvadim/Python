Symptom

The SOAP API CO_EDI_DELIVERY_REQUEST_OUT is not triggering automatically during the creation of outbound delivery

Environment

  • Sales and Distribution (SD)
  • Logistics Execution (LE)
  • SAP ERP Central Component
  • SAP ERP
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA

Reproducing the Issue

  1. Create an outbound delivery
  2. Observe that the SOAP API CO_EDI_DELIVERY_REQUEST_OUT is not triggered automatically
  3. Check the backend triggering routine in the API documentation for CL_EDI_DELIVERY_REQUEST_IN

Cause

  • The SOAP API CO_EDI_DELIVERY_REQUEST_OUT is not designed to trigger automatically during the creation of outbound delivery
  • Specific setups are required, such as output configuration for the ship-to party

Resolution

There are setups needed like output (for ship to party), SOAMANAGER for use it, but this API cannot be triggered automatically.

DeliveryRequest_Out was first of all designed for Sales with Buyers Integration scenario(SAP_COM_0223), therefore bellow setup guides contain the entire scenario.

Read the SAP Note 2964589 and SAP Help, links provided on See Also section which contains Configurations for Outbound Delivery (Notify) Service in SOA Manager.

Go through these documentations and complete the needed setup.

See Also

  • SAP NOTE 2964589 - Set-up Instructions for Integration of Sales Orders and Customer Returns with External Buyers
  • SAP Help - Configuring Outbound Delivery (Notify) Service in SOA Manager

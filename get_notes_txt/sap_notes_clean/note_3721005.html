Symptom

  • Data collection status on the onboarding dashboard shows incorrect status which base on current new hire start date.
  • The discrepancy occurs when the start date is updated during the New Hire Data Review (NHDR) step.

Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.

Environment

SAP SuccessFactors Onboarding 2H 2025

Reproducing the Issue

  1. Initiate onboarding from Recruiting (RCM) successfully; For example, candidate's start date is February 2, 2026.
  2. Log in to Onboarding dashboard, candidate's data collection task status is ''Overdue''.
  3. Begin the New Hire Data Review (NHDR) step.
  4. Update the start date from February 2, 2026, to March 20, 2026, during the NHDR step and submit to complete the step.
  5. Check the onboarding dashboard for the data collection status of onboardees, still displays ''Overdue'', which should be ''On Schedule''.
  6. Go to manage data check the process, go to ToAggregatedDetails, the data collection task due date still the previous start date, February 2, 2026.

Cause

This is a Known Issue.

Resolution

Product Engineering is investigating a solution.

Click on star to bookmark this article to receive updates about this issue. For more information about updates from subscribed KBAs, you can refer to the KB Article 2171560 - How to be notified of new or updated SAP Notes or KBAs.

Workaround

  1. Configure a business rule to check the due date of aggregated details.
  2. Go to "Configure Object Definitions" page.
  3. Select "Object Definition" in the first dropdown menu.
  4. Select "Aggregated Details" in the second dropdown menu.
  5. Under "Save Rules for This Object," select the configured rule, save.
  6. Ensure that the responsible user for the New Hire Data Review task has the necessary Onboarding or Offboarding Object Permissions, Aggregated Details (view/edit permissions) and that the permission role assignment includes the external Onboarding user.

Symptom

As of SCM 7.0 EhP 3 APO can be deployed in two ways: in SCM Server or on top of ERP via add-on.

This note provides information on planning with VC and CDP in the add-on deployment scenario.

Other Terms

Variant Configuration, Characteristic Dependent Planning, batches, APO-on-ERP, add-on

Solution

To use planning with VC/CDP data in the APO add-on the following is required.

  • Install APO SP07, or
  • Install note 2047761 "APO on ERP: Support classes, characteristics, CDP and variant configuration"

Usage of VC in APO-on-ERP

When running in the add-on, CIF transfer for classification data of class type 300 is not required. APO directly reads class 300 related data from ERP.

This leads to the following system behaviour when using VC-planning in the add-on deployment:

  • CIF transfer for classification data in class type 300 is not required
  • APO reads from ERP classification which is assigned to MARA 
  • The APO product master data transaction /SAPAPO/MAT1 does not show the VC relevant class assignments,  which can be viewed on the ERP material master.
    (ERP classification data is assigned via MARA whereas the APO product master display shows APO-assignments via /SAPAPO/MATKEY)
  • Reference characteristics:
    unlike SCM Server, where CIF maps table fields for reference characteristics, APO-on-ERP requires a mapping of reference characteristics procedure code via BADI /SAPAPO/CURTO_CREATE


Usage of CDP in APO-on-ERP

Prerequisite:
To use planning with CDP (and batches) data in the APO add-on please activate the content of BC-set /SAPAPO/APO_ON_ERP_CDP. This will introduce customizing for APO classes 230 and 400.

When running in the add-on, CIF transfer for classification data of class type 230 and 400 is  required. APO directly reads realted data from the APO-only classes of type 230 and 400.

This leads to the following system behaviour when using CDP-planning in the add-on deployment:

  • CIF transfer for classification data in class type 23/300 to type 230/400 is required
  • APO reads from ERP classification which is assigned to /SAPAPO/MATKEY
  • The product master data /SAPAPO/MAT1 shows the CDP relevant class assignments only, assignments to 23/300 can be viewed on the ERP material master.
    (ERP classification data is assigned via MARA whereas the APO product master display shows APO-assignments via /SAPAPO/MATKEY)

Symptom

You are performing the SUM DMOVE2S4, to perform the DB migration to HANA DB on RISE environment. The phase EU_CLONE_MIG_DT_RUN is running for a long time.

Environment

NetWeaver ABAP based systems.

Reproducing the Issue

Monitor the SUM phase EU_CLONE_MIG_DT_RUN.

Cause

In the file "MIGRATE_DT_00059_<package/table>_IMP.run", there is the following information.
 
ProcReadLatencyAvg: 76357.4 ms
ProcWriteLatencyAvg: 209.8 ms
ProcAvgReadSpeed: 0.1 MB/sec # 0.5 GB/h
ProcAvgWriteSpeed: 47.7 MB/sec # 167.6 GB/h
 
(This is just an example.)
 
According to the SUM DMO central Note (example: Note 3620679 - Database Migration Option (DMO) of SUM 2.0 SP25), there is a latency and bandwidth requirements, as below.
 
********************************************************
g) DMO Move to SAP S/4HANA (on Hyperscaler)
* Technical requirements:
    * A low latency with less than 20 ms
    * A high bandwidth of more than 400 MBit/s
********************************************************
 
While according to the above file content, both the read latency and write latency are too high.

Resolution

Please contact your network team to improve the network performance to meet the technical requirements.

Symptom

Tax numbers (e.g., VAT numbers) are missing in the Display Customer List (F2640) Fiori app for customers whose country/region code does not match the leading two digits (ISO code) of the tax number stored. The tax number is visible in the DFKKBPTAXNUM table but does not appear in the Fiori app.

Environment

  • SAP S/4HANA
  • SAP Fiori

Reproducing the Issue

  1. Go to Display Customer List (F2640) Fiori app
  2. Try to display the customer list for a business partner whose tax number is populated in the DFKKBPTAXNUM table.
  3. Observe that the tax number is missing in the display customer list Fiori app.

Cause

The country/region code of the customer in the master data does not match the leading two digits (ISO code) of the tax number stored in the DFKKBPTAXNUM table.

Resolution

  1. Verify the country/region code of the customer in the master data.
  2. Ensure that the leading two digits of the tax number stored in the DFKKBPTAXNUM table align with the ISO code of the customer's country/region code. For example, if the customer's country is "IT", the tax number should start with "IT" (e.g., ITXXXXXXX).
  3. Update the tax number in the master data to match the ISO code if necessary.

Keywords

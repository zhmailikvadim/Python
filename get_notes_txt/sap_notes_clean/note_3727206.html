Symptom

  • A flexible workflow for some MM document is incorrectly routing approvals to the wrong recipient instead of the expected approver based on the configured approval matrix
  • Wrong workflow is being determined
  • Automatic workflow is being determined but another workflow determination is expected
  • You want to analyze why a certain workflow is being determined for an MM document

Environment

  • SAP Fiori
  • SAP S/4 Hana

Reproducing the Issue

You want to analyze why a certain workflow is being determined for an MM document

Cause

Resolution

You can simulate how flexible workflow is triggered for a particular MM document by following the next steps: 

  1. Access transaction SWF_PROCESS_ADMIN.
  2. Select the workflow of the document you are analyzing. You can check the workflow number you are using by checking the workflow log in transaction swia. Please, note that the standard flexible workflows for MM documents are the following:

    Workflow for Supplier Quotation WS00800193
    Workflow for Purchase Order WS00800238
    Workflow for RFQ WS00800302
    Workflow for Purchase Contract WS00800304
    Workflow for Scheduling Agreement WS00800305
    Workflow for Service Entry Sheet WS00800321
    Workflow for Centrally Managed PO WS00800333
    Workflow for Central Purchase Contract WS00800346
    Overall Release of Central PR WS02000434
    Release of Central PR Item WS02000438
    Overall Release of Purchase Requisition WS02000458
    Release of Purchase Requisition Item WS02000471

    You can also check the standard workflow in the following link: https://help.sap.com/docs/SAP_S4HANA_CLOUD/0e602d466b99490187fcbb30d1dc897c/a84badfeaf0445f4bed2a5c9bb0ed9c2.html
  3. Click icon "start simulation"
  4. Enter the document number in Leading object field and click "Start simulation".
  5. In the simulation result you can find the workflows evaluated, the flag for the selected workflow, flags for "status fulfilled" and "conditions fulfilled". Moreover, if you click "Condition exists" icon for each line, you can find a detailed analysis of the starting conditions regarding the simulated workflow triggering for the particular document you have provided.

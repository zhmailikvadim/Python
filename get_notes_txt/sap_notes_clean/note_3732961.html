Symptom

When attempting to save the jco.destination.pool_capacity setting as <blank> in a BTP Destination configuration, the following error message appears:
Error message: Destination '<Destination Name>': jco.destination.pool_capacity cannot be empty.

Environment

SAP Business Technology Platform
Cloud Foundry environment

Reproducing the Issue

1.Logon SAP BTP Cockpit URL(https://cockpit.btp.cloud.sap) .

2.Navigate to the Subaccount -> Connectivity -> Destinations menu.

3.Select the destination which needs to be duplicate.

4.Then click on the duplicate button.

Cause

When destination is duplicated, it is by default taking all the configuration from the previous destination.

Resolution

If destination is created by duplicating any destination, it will automatically take all the configuration from previous destination and if any configuration is missing from previous one,
it will show similar following error:
Error message: Destination '<Destination Name>': jco.destination.pool_capacity cannot be empty. 

To avoid such error, create the destination from scratch/file/service instance, it will create destination with fresh configuration without any by default configurations.

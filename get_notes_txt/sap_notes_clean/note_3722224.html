Symptom

How supplier create Order Confirmation (OC) without entering OC ID, which is mandatory on OC UI ? Why the confirmation tab displayed OC ID as "Confirmation #: Untitled MM/DD/YYYY" ?

Cause

  • For Supply Chain Collaboration, the UserAgent value in the cXML varies based on where the document (OC or Ship Notice) is created:
  1. SCM Supplier → when the OC or Ship Notice is created from the online PO UI
  2. SCM Supplier Web → when the OC or Ship Notice is created from the Workbench
  3. task processer → when the OC or Ship Notice is created through Excel upload/download
  • Each of these three methods for creating an OC or Ship Notice uses its own distinct UI fields.
  • The buyer’s customization applies only to the OC created from the PO UI Create Order Confirmation.

Resolution

<UserAgent>SCMSupplierWeb</UserAgent>

When this user agent is detected in OC CXML, it indicates that the OC was created from the Workbench → Items to Confirm tab using the Confirm Entire Order option. If the supplier does not enter an OC number in the input box before submitting, the system automatically generates the OC with the name format (Untitled MM/DD/YYYY). This is the standard system behavior.
If you want to avoid this default behavior, you can either:

  • Request the supplier to manually enter the OC ID when creating the OC from the Workbench.  Or
  • Instruct the supplier to create the OC only through the PO UI. (where the customization is applied)

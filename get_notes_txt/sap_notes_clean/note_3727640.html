Symptom

  • Error message in IAS indicating that the Family Name is missing when creating a new onboardee record using Add New Hire to Onboarding.
  • Error in provisioning logs in IAS is as follows: "HTTP operation failed invoking *LINK* with statusCode: 400, Response: Missing required attribute in name - [familyName]. "

Environment

SAP SuccessFactors Onboarding

Reproducing the Issue

  1. Use Add New Hire to Onboarding to create a new onboardee record.
  2. Note that last name is entered in at this point.
  3. Submit without providing additional data.
  4. Go to IAS system.
  5. Go to provisioning logs.
  6. See that the most recent log is finished with error.
  7. See the following error - "HTTP operation failed invoking *LINK* with statusCode: 400, Response: Missing required attribute in name - [familyName]."

Cause

A HRIS sync mapping with the Last Name field as the target field was causing the value to disappear.

Resolution

  1. Navigate to Manage Business Configuration within the system.
  2. Locate the HRIS Sync Mapping section.
  3. Identify any mappings with the target field set to "Last Name".
  4. Delete the HRIS sync mappings for the Last Name.
  5. Initiate Onboarding via add new hire to onboarding and see that the user is synced to IAS correctly.

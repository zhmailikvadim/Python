Symptom

On Linux systems, a fresh SDA installation fails due to a timeout, and all subsequent installation attempts continue to fail.


The following symptoms can be observed:

  1. The file SMDAgent/default/configuration/agent/#/master_password is empty and has both read and write permissions set.
  2. The SDA agent logs contain exceptions similar to:

com.sap.solman.agent.core.spring.AbstractSdaFailureAnalyzer org.springframework.beans.factory.BeanCreationException : Error creating bean with name 'jettyServletWebServerFactory' defined in class path resource [org/springframework/boot/autoconfigure/web/servlet/ServletWebServerFactoryConfiguration$EmbeddedJetty.class]: illegal permissions on file
org.springframework.context.ApplicationContextException: Unable to start web server

Reason and Prerequisites

The master_password is generated using system entropy. On some Linux systems, the available entropy is very low. This can have several causes, for example hardware characteristics or limited I/O activity.

Because of the low entropy, generation of the master_password is significantly slowed down, which in turn causes the SDA installation to run into a timeout.

Solution

Remove the existing SDA installation completely, and then perform a fresh installation using the SDA version specified in the Support Packages & Patches section.

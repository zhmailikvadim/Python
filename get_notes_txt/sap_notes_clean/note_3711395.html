Symptom

Check is generated along with the payment file during payment media creation using transaction FBPM (SAPFPAYM) or F110 "Printout".

Environment

  • Financial Accounting (FI)
  • SAP R/3
  • SAP R/3 Enterprise 4.7
  • SAP ERP Central Component
  • SAP ERP
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA
  • SAP S/4HANA Finance

Reproducing the Issue

Execute a payment run using transaction F110 along with checkbox "Create payment medium" or create payment file using transaction FBPM (SAPFPAYM).

Cause

  • The function module FI_CHECK_NUMBER_GET is executed during file creation, and if a check lot is set up for the combination of paying company code, house bank, account ID, and payment method, the check number is drawn automatically.

Resolution

  1. Identify the check lot configuration in transaction FCHI for combination of paying company code, house bank, account ID, and payment method.
  2. Remove payment method from check lot.
  3. Save the changes and re-execute the payment file creation using transaction F110 or FBPM (SAPFPAYM).
  4. Verify that payment file is generated instead of check.

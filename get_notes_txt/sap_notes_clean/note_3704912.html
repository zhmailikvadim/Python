Symptom

Since the automatic latest Webview2 Version 144.xxx from Microsoft the HTML control is not working properly anymore when using "edge" as browser control in SAP GUI for Windows.

The following issues may occur:

1) Clicking links / pressing buttons does not work anymore
2) Content is no longer displayed (e.g blank header in ALV in SM50, blank screen in SE80 dropdown list and RZ11)

There is no issue when using Webview2 Version 143.xxx in the machine or "Internet Explorer" browser control (SAP Logon Options -> Interaction Design -> Control settings -> HTML Control -> Browser Control)

This could happen in any transaction that contains a html-based content.

Environment

  • SAP GUI for Windows release independent
  • SAP NetWeaver
  • SAP NetWeaver Application Server for SAP S/4HANA
  • ABAP PLATFORM - Application Server ABAP

Reproducing the Issue

Many transactions where an HTML control is used (SM50, SBWP, SOST, RZ11, S80 etc..) are affected.

Cause

Microsoft rolled out automatic updates of the WebView2 component to version 144 which introduced two issues in SAP GUI for Windows.

Resolution

Microsoft solved the issue with WebView2 Runtime version 144.0.3719.93 which was published on Friday, 23rd of January 2026.This version of the WebView2 Runtime was already installed automatically on most devices (depending on how the updates are managed in your organization) and therefore, the issue disappeared again for these devices in the meanwhile.

If you did not receive an automated update to this version, you can download it from the following Link:

https://developer.microsoft.com/en-US/microsoft-edge/webview2/?form=MA13LH#downloador
https://developer.microsoft.com/de-de/microsoft-edge/webview2/consumer/?form=MA13LH

The issue is documented on the Microsoft Edge Known Issues page Microsoft Edge known issues | Microsoft Learn. 

If you have problems with the installation, please contact Microsoft directly.

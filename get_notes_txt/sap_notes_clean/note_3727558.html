Symptom

When you use bxml flattening, you do not want the system to display a different number of decimal places for the cell properties VALUE and FORMATTED_VALUE.

Other Terms

BAdI RSMDX_BAPI Cell Property VALUE FORMATTED_VALUE

Reason and Prerequisites

The observed behavior is correct. When the cell property VALUE is displayed, the number of decimal places used by the OLAP processor to calculate values for the relevant BW key figure is used. This number can differ from the number of decimal places defined for the display of this BW key figure. If you do not want this behavior, you can change the behavior after you implement the correction instructions from this SAP Note or import the relevant Support Package.

To do this, you must create an implementation of the BAdI RSMDX_BADI and implement the interface method IF_RSMDX_BADI~FORMATTED_DECIMALS_VALUE_BXML. The exporting parameter E_USE_FORMATTED_DECIMALS must be set to the value 'X' for all MDX cubes for which a behavior change is required. For these MDX cubes, the number of decimal places of the cell property VALUE corresponds to the number of the cell property FORMATTED_VALUE if no technical restrictions prevent this behavior.

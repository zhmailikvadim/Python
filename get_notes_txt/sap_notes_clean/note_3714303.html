Symptom

  • UI custom MDF set in Recruit template is not functioning for External Onboarding Users.
  • The UI does not display the proper name but shows MDF code instead.

Environment

SAP SuccessFactors Onboarding

Reproducing the Issue

  1. Log in as the candidate using provided credentials.
  2. Navigate to Custom MDF object
  3. Observe that the UI displays MDF code instead of the proper name.

Cause

The issue occurred because translations for the UI were not updated, leading to improper display of names and MDF codes.

Resolution

  1. Navigate to Manage Configuration UI.

  2. Search for the object for which the UI details are not populating.

  3. Click on Edit Properties for the selected object.

  4. Next to Title Type User Defined, you will see a small label/icon indicating translations.

  5. Click on this label to view the available translations.

  6. Update the translation with the value that should be visible to the onboardee.

  7. Click on Save to apply the changes.

Symptom

When we create Quality Notification in Ariba and we will receive the error "Unit of measurement XXX is not available".

Environment

  • Managed Gateway for Buyer Business Network SCC

Reproducing the Issue

  1. Create a Quality Notification from Ariba using QualityIssueNotificationMessage.
  2. Check the inbound processing in the target system (for example, via SRT_MONI) and observe that the Quality Notification fails due to an unavailable unit of measure.

Cause

  • The parameter ENABLE_EXTFORMAT_ISO_UOM was not enabled in the target system, so external ISO UOM values were not converted to internal format during inbound processing.

Resolution

  1. Enable the ENABLE_EXTFORMAT_ISO_UOM parameter in the target system to allow conversion of data to external (ISO) format during processing of QualityIssueNotification_Async_In.
  2. Review UOM conversions in Cloud Integration Gateway (CIG) and remove custom mappings for the affected UOM if they convert an ISO-compliant unit to a different code (for example, mapping MTK to m2), to avoid conflicts with external format handling.
  3. Retest the end-to-end scenario: send a Quality Notification from Ariba and verify that the inbound message processes successfully without UOM errors.

See Also

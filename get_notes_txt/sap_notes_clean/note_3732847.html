Symptom

You start a table re-distribution via HANA Cockpit with option BALANCED, or using a SQL command with option BALANCE_BY_TABLE_SIZE_HOSTED.

You observe that the data is not evenly distributed among the worker nodes.

Other Terms

BW, sap.bw.dso, DSO tables, REORG_GENERATE

Reason and Prerequisites

Issue number 349423

Reason:

Due to a programming error in SAP HANA, the redistribution algorithm is impacted by large HASH/RANGE partition tables which can lead to uneven distribution results.

Affected Releases:

  • SAP HANA 2:
    • Revisions of SPS00 - SPS06
    • Revisions <= 079.08 (SPS07)
    • Revisions <= 089.01 (SPS08)

Prerequisites:

  • You have large HASH/RANGE partitioned tables, for example BW DSO tables with first level HASH and second level RANGE partition.
  • Table redistribution is executed via HANA Cockpit (option "BALANCED") or using SQL command with "BALANCE_BY_TABLE_SIZE_HOSTED=>true"

Solution

Apply one of the following SAP HANA Revisions:

  • SAP HANA 2:
    • Revisions >= 079.09 (SPS07)
    • Revisions of SPS08: SAP is currently working on a permanent solution to this issue. This SAP Note will be updated as soon as a revision containing the solution is available
    • or higher

  • or higher

Workaround:

You may consider disable the problematic feature before generating the redistribution plan.

  • steps to be performed as workaround

    ALTER SYSTEM ALTER CONFIGURATION ('indexserver.ini', 'SYSTEM') SET ('table_redist', 'enable_vote_partitions_by_intersec_kpi') = 'false' WITH RECONFIGURE';

  • Please revert the workaround once you have upgraded to a revision containing the permanent solution.

Disclaimer

The SAP HANA configuration parameter [<table_redist>] <enable_vote_partitions_by_intersec_kpi> is an internal parameter and may only be used in the context of this SAP Note. Using such a parameter requires that the SAP HANA system has the revision specified in the SAP Note. Using internal parameters outside of this given scope is not supported. Please unset this parameter once you have upgraded to a revision containing the permanent solution.

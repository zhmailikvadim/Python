Symptom

When you start the process Setup/Replacement of Devices (MOS View) (DE_B_IM_DEV_CHANGE) using one of the following API methods from class /US4G/CL_IM_PROCESS_API, the system doesn't create a process document if an error occurs during the submission of the installation reply. Some customers want the option to trigger a process anyway.

The following API methods are affected:

  • START_DEVICES_INSTALL
  • START_DEVICES_CHANGE
  • START_DEVICES_ADD
  • START_DEVICES_REMOVE 

This issue is relevant for the market role: MOS.

Other Terms

/US4G/CL_IM_PROCESS_API, START_DEVICES_INSTALL, START_DEVICES_CHANGE, START_DEVICES_ADD, START_DEVICES_REMOVE, installation reply, process document, PDOC, kME, mME, IMS

Reason and Prerequisites

The error occurs because the API methods exit without creating the process document when the installation reply encounters an error.

Solution

Implement the corrections from this SAP Note or import the next Support Package.

After you implement this SAP Note, the system introduces a new parameter IV_IGNORE_IR_ERROR_ON_SUBMIT in all four methods. By default, this parameter has the value ABAP_FALSE, which maintains the existing behavior. If you set the parameter to ABAP_TRUE, the methods don't exit, and the system creates the process document with the erroneous installation reply, if requested.

If you need this functionality, adjust your coding and set parameter IV_IGNORE_IR_ERROR_ON_SUBMIT to ABAP_TRUE. You can verify the fix by checking that the system creates a process document when the submission of an installation reply fails in these four methods after you set the parameter in your coding.

Symptom

  • When attempting to delete valuation rule assigned to a contract, error message RECETM022 - 'Postings already exist for valuation rule &1' is shown.
  • Error is present, despite reversing all valuation postings and deleting prior valuation steps.

Environment

SAP S/4HANA Cloud Public Edition

Reproducing the Issue

  1. Open the contract via 'Manage Contract' or 'Valuation Management' Apps.
  2. Attempt to delete valuation rule which was already posted in the past.
  3. Observe error message RECETM022 - 'Postings already exist for valuation rule &1'

Cause

  • The behavior is standard system design: once postings have been created for a valuation rule, the system does not allow deleting that valuation rule from the contract.
  • Even if postings were later reversed, the error message is justified as the postings/reversals are present in the system.
  • Any already posted valuation rules must remain on the contract to ensure process and reporting consistency.

Resolution

  1. Do not delete the valuation rule if postings (even reversed) exist for it.
  2. Instead, invalidate the valuation rule on the contract and lock it to prevent further use.
  3. Assign and activate the correct valuation rule for future processing.
  4. Ensure any incorrect valuation postings have been fully reversed before proceeding with the new valuation rule.

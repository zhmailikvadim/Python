Symptom

You notice that data is missing in the Introscope or Workload Analysis for an entire server of your MDM system, or data is completely missing for all servers.

In the case below, there's a Master Data Management Server, a Master Import Server and a Master Syndication Server, but only the Master Import Server data is shown:

Environment

Solution Manager 7.1

Reproducing the Issue

Just check the Introscope Enterprise Manager, there should be a node for each MDM server (Master Data Management, Master Import Server, Master Syndication Server, etc). If something is missing, there's an issue.

Cause

The NCS configuration is wrong. The property "agent_port" in file ncs.conf is wrong.

Resolution

Each server has its own ncs.conf file. Please ensure that it is adapted in each server.

The file can be found under /<MDM Installation folder>/<server>/config.

You should adapt it according to the value set in file SapAgentConfig.xml, after the managed system configuration is performed. It can be checked as follows:

  1. Open the Agent Administration UI: http://<Solman FQDN>:<port>/smd/AgentAdmin.
  2. Select Application Configuration.
  3. Expand com.sap.smd.agent.application.wilyhost > Application Resources > SapAgentConfig.xml
  4. In the right side of the screen, under Resource Customization, DO NOT FORGETto change the scope to the agent monitoring the MDM host.
  5. Download the customized file.
  6. There will be an action named TCP_<port>. This action points to a destination, which will have a property port setting which port will be used to reach the NCS library in the MDM. This is the port you must set in the ncs.conf file.

Symptom

This KBA contains the important notes and documentation for DMEE to DMEEX migration.

Main goal of migration - have same outputs in DMEEX as there were in DMEE. 

Environment

  • SAP ERP
  • SAP enhancement package for SAP ERP
  • SAP S/4HANA  All versions

Cause

As per note 3370503, transaction DMEE is obsolete from release SAP S/4HANA 2025. 

Resolution

SAP Notes related to DMEE to DMEEX migration: 

  • Deprecation of DME tool: SAP Note "3370503 - S4TWL – DMEE
  • Main Migration Note:  "3435565 - DMEEX - important notes for DMEE to DMEEX migration" Before proceeding with migration, make sure that the latest version of Note is implemented!. Attached to the Note there is a pdf with migration information (taken from SAP Help)

Features available during migration:

To gain the migration process as smooth as possible, there are some useful tools provided in Migration.

  • Migration Manager – will show you status of DME to DMEEX migration of trees on your system. Available from DMEEX initial screen.  

 

  • Migration Assistant –⁠⁠⁠⁠⁠⁠ will show all relevant information on things that were different in DME to DMEEX. It gives structured assistance throughout the migration process and alerts you about potential inconsistencies. Available in the main menu of Migration.

  • Migration Regression Tool –⁠⁠⁠⁠⁠⁠ will allow to retest the scenario simultaneously with both engines at once, the DMEE and DMEEX. It compares output files generated by DMEE and DMEEX and can be re-runed as many times as needed to verify a correct output. Available in the main menu of Migration.

  • Masking – will allow you to exclude nodes from comparison during a migration regression run if needed. ⁠⁠⁠⁠⁠⁠ The system automatically masks nodes that take their values from current time and date to prevent predictable differences during the comparison. Available in the context menu on the tree.

Key take aways for the migration process:

  • A migration format tree is created as a copy of the original DMEE format tree. This migration format tree is only temporary and is deleted once the migration is confirmed or canceled. If you use the Activate button to activate the migration format tree it has no impact on the original DMEE format tree.
  • You can Cancel the migration of your selected format tree at any time before the migration is completed. All test run data and the migration format tree are deleted.
  • Until the migration is Confirmed, the DME is used in the business process. After migration is Confirmed, the DMEEX is used.
  • Migration shall be Confirmed only after your test scenarios are proceeded and the DMEE and DMEEX output file results are the same. Once the output files meet the expectations of your test scenarios, you can choose the Confirm Migration button for your format tree.

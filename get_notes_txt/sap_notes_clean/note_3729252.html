Symptom

  • Reconciliation job in real-time integration fails intermittently.
  • Error messages include: "Data selection in external system failed", "Data comparison failed", and "Job chain execution stopped".
  • The global configuration parameter `RTI_REC_JOB_START_TIMEOUT` with default settings of 60 minutes appears not to be considered correctly.

Environment

  • Reconciliation

Reproducing the Issue

  1. Execute a reconciliation job in the real-time integration module of the system.
  2. Observe intermittent failures of reconciliation jobs due to delays in the corresponding external system.
  3. Error messages such as "Data selection in external system failed", "Data comparison failed", and "Job chain execution stopped" may appear during the process.
  4. Setting global configuration parameter RTI_REC_JOB_START_TIMEOUT does not change this behavior 

Resolution

As of IBP release 2505, the retention time in ECC can be controlled via global configuration parameter RTI_REC_JOB_START_TIMEOUT

To ensure the parameter works correctly, make sure note 3708918 is applied in ECC

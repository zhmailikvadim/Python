Symptom

When the approver accesses the Workflow approval page to view the detailed information, a misalignment occurs between the header field labels and the corresponding content for the custom MDF object.

Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.

Environment

  • SAP SuccessFactors Metadata Framework
  • SAP SuccessFactors HXM Suite

Reproducing the Issue

  1. Login to SF instnace

  2. Proxy as Workflow Approver 

  3. Navigate to the Workflow Requests page.

  4. Click on a custom MDF workflow request to open the approval details.

  5. Scroll to the details section and observe that the header field labels are misaligned with the corresponding content as per below snapshot -

Cause

  • This issue occurs due to a known front-end UI limitation when the layout type is set to Grid in Configure UI.
  • The Grid layout has known display and alignment issues, confirmed by the Product team, and there is currently no plan for a fix in the roadmap.

Resolution

  • The Grid layout has known display and alignment issues, and as confirmed by the Product Team, no fix is planned on the roadmap.
  • We are in process of deprecation of ConfigUI (Legacy UI), and Product Team has no planning to create any enhancement currently on Legacy UI(ConfigUI).

Workaround:

To avoid the UI misalignment issue, Navigate to Configure UI and change the layout type from Grid to "Flow".

The Flow layout does not have this limitation and displays the fields correctly

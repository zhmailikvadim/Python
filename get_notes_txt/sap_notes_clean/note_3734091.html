Symptom

When finalizing a bank statement for a bank with an interim account in Foreign Currency, the Journal Entry generated with the interim account is not correct in System Currency.

Image/data in this SAP Note is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.

Reproducing the issue

Prerequisites:

  • SAP Business One, local BE.

  • Go to Administration → System Initialization → Company Details → Basic Initialization, System Currency(SC) is different from Local Currency(LC).

  • Go to Administration → System Initialization → Company Details → Basic Initialization → Install Bank Statement Processing(BSP), enabled.

  • You have an Internal Operation Code BP - BnkAcc, Posting Method: Business Partner from/to Bank Account. 

  • You have a House Bank Account, in the demo 000-0000011-11, G/L Interim Account of which is set.

Scenario:

  1. Add an all currencies Vendor BP
  2. Go to Banking → Bank Statements and External Reconciliations → Bank Statement Processing, click Create New.
    • Choose Internal Operation Code BP - BnkAcc,
    • Payment Currency (the same as SC), USD,
    • Exchange rate: 5,
    • Outgoing Amt - Payment Currency: 100 USD,
    • BP: BP
  3. Fill the correct Ending Balance and click Finalize.
  4. Open the last Journal Entry newly added by the BSP.

Expected behavior:

The amount in SC is the same as that in FC, 100 USD.

Actual behavior:

The amount in SC is the same as that in LC, 20 USD and a row for exchange rate difference.

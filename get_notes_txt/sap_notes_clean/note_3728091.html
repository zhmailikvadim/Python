Symptom

  • Monthly Gratuity (EOS) provision for to skipped payroll and Subsequent Payroll
  • In scenarios where payroll is skipped for a full month due to employee locked, WT /649 and /650 not generated

Environment

  • Human Capital Management (HCM)
  • SAP ERP Core Component
  • SAP ERP
  • SAP enhancement package for SAP ERP

Reproducing the Issue

  1. Lock the employee so that Payroll is not executed for the month, Say example Nov month
  2. Run the payroll using transaction PC00_MAE_CALC_SIMU for Dec
  3. Payroll run retro from Nov, Check the Payroll result for Nov and Dec
  4. WT /649 and /650 is not generated for Nov

Resolution

  1. During the retro Nov payroll WT /651 will be generated, in Payroll Function AE47 it reads the previous month RT in ORT and update the value in /651 as Carry forward Provision
  2. In Dec payroll the value from /651 is updated in WT /649 and Provision WT /650 is generated during the Payroll
  3. By this standard process during the Locked Payroll Period or not processed payroll period previous month provision amount is carry forward to current payroll period and generated current month Provision amount.

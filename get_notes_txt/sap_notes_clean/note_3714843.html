Symptom

  • Issue occurs after applying a patch in SAP PO.
  • Attempted fix using OSS note 3434836 by adding 'appendLockTimeout' and 'retry attempts' default values and increasing them on the SFTP receiver channel, but the error persists.
  • Two ICOs are running, appending data to generate one CSV file for the SFTP server target.
  • User authorizations for the SFTP target have been verified and are correct.

Environment

Secure File Transfer Protocol Adapter

Reproducing the Issue

NA

Cause

Lack of Locking Feature: The SFTP adapter does not inherently provide a locking mechanism for files in append mode. This means that the successful processing of messages and the integrity of the data written to the SFTP server depend on the server's own locking capabilities

Resolution

1. Apply the patch corresponding to the current Support Package level, following the instructions provided in the NetWeaver Support Package Stack Guide, as referenced in SAP Note 952402.

2. Configure the newly introduced parameters in the Advanced tab of the SFTP receiver channel:

    • appendLockTimeout: Specifies the maximum time (in milliseconds) the system will wait to acquire the file lock. Default value: 2000 ms.
    • retry attempts: Specifies the maximum number of retry attempts to acquire the lock. Default value: 5 attempts.

3. Ensure that the software component SAP_XIAF is updated to version 7.50 SP 27 or higher, as the fix is not available for PIB2BSFTP.

4. Test the configuration to confirm the resolution of the issue.

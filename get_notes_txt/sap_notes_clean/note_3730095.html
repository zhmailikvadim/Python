Symptom

In app Perform Further Valuations (F107) for scope item 2PD (Provision for Doubtful Receivables), the system posts a valuation difference that does not correspond to any individual invoice or sum of open receivables. When analysing customer line items, there is a difference with the open balance, and no postings can be found that directly explain the valuation amount. 

Environment

SAP S/4HANA Public Cloud.

Reproducing the Issue

  • Post a customer invoice in foreign currency.

  • Execute a Foreign Currency Valuation run.

  • Post or simulate valuation results.

  • Execute Perform Further Valuations (F107) for the same customer.

  • Observe that the valuation difference posted in F107 does not match the open receivable amount or any invoice total.

Cause

The valuation difference in F107 does not originate from individual invoices or their aggregated values.

If a Foreign Currency Valuation run has been executed previously, the valuation results are stored at document level in backend table BSBW. These values represent currency valuation differences calculated for the receivable.

During the F107 run, the system reads the existing valuation data (including values stored in BSBW) and incorporates it into the valuation calculation. As a result, the posted valuation difference may reflect prior foreign currency valuation adjustments rather than the current open receivable balance alone.

Therefore, the amount displayed in F107 can originate from the Foreign Currency Valuation process and not from invoice-level amounts.

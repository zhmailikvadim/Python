Symptom

In order to use the new FIN data model "0G", after following the instruction in SAP standard document Deleting Data Model 0F to delete the active version of data model "0F", you have encountered error "Deletion run completed with errors"(Message no. USMD5000) after running Transaction "MDG_DELETE_MODEL" for data model "0F".

Environment

  • SAP Master Data Governance 8.0
  • SAP Master Data Governance 9.0
  • SAP Master Data Governance 9.1
  • SAP Master Data Governance 9.2

Reproducing the Issue

  1. Run Transaction MDG_DELETE_MODEL on SAP GUI.
  2. Enter Data Model "0F" and Execute.
  3. Error message "Deletion run completed with errors"(Message no. USMD0005) is shown in a popup.

Cause

  1. Switch "FIN_MDM_CORE_SFWS_EHP5" is not activated yet.
  2. Cross Client object change is not allowed for the present Client.

Resolution

  1. Please refer to KBA "2475099 - Switch FIN_MDM_CORE_SFWS_EHP5 is inactive" to make sure corresponding Business Functions are activated, so that the mentioned switch will be activated also.
  2. Please run Transaction "SCC4" on your system and select the corresponding Client where you want to delete data model "0F", go to the detail view and change the value of field "Cross-client Object Changes" as "Changes to Repository and cross-client customizing allowed".
  3. If the above mentioned solution cannot fix your problem, please check and debug in report USMD_DELETE_DATA_MODEL. This will help you to narrow down the problem.

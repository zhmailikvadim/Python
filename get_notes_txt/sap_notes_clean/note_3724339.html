Symptom

When revising work orders to change cost center allocations, entering 0 in Hours for Committed Spend field results in cost allocation amounts remaining at 0.

The Refresh Committed Spend and Calculate Total buttons do not update values regardless of percentage allocations entered when switching from one cost center to another.

Reproducing the Issue

  1. Access the work order
  2. Attempt to revise the work order (for example, with the following parameters):
    1. Start date: 01/01/2026
    2. End date: 31/03/2026
    3. Hours for committed spend: Enter 0
  3. Change the cost center allocation from cost center A to cost center B
  4. Confirm the work order revision and observe that the committed spend is not allocated to the new cost center.

Resolution

Cost allocation calculations are directly dependent on the Hours for Committed Spend field value. When 0 hours are entered, the final allocation amount will always calculate to 0, regardless of percentage distributions. The system requires actual committed hour values to perform cost calculations. This is expected behavior.

The contextual help text shown during the revision, is valid only if the adjustment is applied for the same cost center.  

Enter the number of hours to increase or decrease the Cumulative Committed Spend. Use a negative number to decrease it. For no change to the Cumulative Committed Spend, enter zero. If zero is entered, a zero value will display in the Revised Committed Spend/Committed Spend field when viewing the work order revision details; however, the Cumulative Committed Spend will remain unchanged. 

If a new cost center is being used, entering 0 hours does not allocate committed spend to the new cost center, whereas entering the corresponding number of hours allows the system to calculate the committed spend correctly.

If you just want to re-assign the budget from one cost center to another, you would need to revise the affected work order:

  1. Select the same start and end date
  2. Add the same Hours for Committed Spend
  3. Add the Cost center that you would like to be primary
  4. Re-allocate the percentage > Refresh

Symptom

  • Norway receives approximately 90% of domestic supplier invoices as XML (~350 invoices/month).
  • The XML invoices follow the European standard EN16931 and are received through the PEPPOL network, which requires an access point.
  • The user requests SAP's guidance and recommendations for a provider/system compatible with SAP S/4HANA to replicate the current setup by:
    • Receiving PEPPOL EN16931 XML supplier invoices.
    • Conversion of XML invoices.
    • How to import/create the invoices directly in SAP S/4HANA to avoid manual processing.

Environment

  • SAP S/4HANA Cloud Public Edition
  • SAP Document and Reporting Edition, Cloud Edition
 

Reproducing the Issue

  1. Ensure the XML invoices are received through the PEPPOL network.
  2. Attempt to process the XML invoices following the European standard EN16931.
  3. Observe the lack of integration for rendering/converting the XML invoices into a human-readable format and importing them directly into SAP S/4HANA.

Resolution

  1. Utilize SAP Cloud Integration (CPI) to connect to the PEPPOL access point, facilitate XML exchange, and orchestrate endpoints.
  2. Refer to the SAP Help Portal for guidance on integrating Peppol Exchange with SAP S/4HANA Cloud Public Edition - Integrating Peppol Exchange.

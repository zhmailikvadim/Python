Symptom

You have configured an inbound integration job to pick up a file from an SFTP server and move it to a "Processing" or "Completed" folder once finished. Although the integration runs successfully and the records are correctly inserted into SuccessFactors, the source file remains in the original folder.

In many cases, the logs may not show a failure because the data transfer itself was successful, but the final step (moving the file) failed.

Environment

  • SuccessFactors HCM Suite
    • Integration Center

Cause

The most frequent cause is a pathing mismatch between how your SFTP server sees the world and how the Integration Center is trying to navigate it.

SFTP servers often use "Chroot jails," which restrict a user to their own home directory. If you configure your processing folder using an absolute path (one that starts with a forward slash /), the system tries to find that folder starting from the very root of the entire server. Even if the path looks correct to you, the SFTP server may deny this "top-down" request for security reasons, resulting in a "permission denied" error for that specific move action.

Resolution

To fix this, you need to adjust how the Integration Center "points" to your folder. Instead of telling the system exactly where the folder is on the whole server, you tell it where the folder is relative to where the integration is already looking.

  1. Open your Integration: Navigate to the Source Settings tab of your integration job.
  2. Identify the Processing Folder: Look at your current path configuration.
  3. Remove the Leading Slash: Change your path from an absolute format to a relative format.
    1. Instead of: /SuccessFactors/ArchivedFiles
    2. Use: ArchivedFiles (assuming the folder "ArchivedFiles" lives inside your main SFTP directory).
  4. Save and Run: Execute the job again.

By removing the initial /, you are telling the Integration Center to look "inside the current folder" rather than "at the root of the server." This usually clears up the permission conflict, and your SFTP server will allow the file to be moved.

Note: Ensure that the user account used for the SFTP connection has both Read and Write permissions for both the source and the destination folders.

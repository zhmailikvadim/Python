Symptom

When executing the run "Transfer Product Cost Estimates", the balance type materials with zero costs are not transferred with the flag "No Cost Component Split" and for this reason the transfer consider the existing cost estimation.

Other Terms

Transfer Product Cost Estimates, Balance Type Materials, No Cost Component Split

Reason and Prerequisites

Program error

Solution

History:

Version 1 of 26 March, 2026: New creation

Version 2 of 30 March, 2026: Add supplement

Install the code correction.

Description of Correction:

When executing the run "Transfer Product Cost Estimates", the balance type materials with zero costs will be transferred with the flag "No Cost Component Split" to determine the transfer not to consider the existing cost estimation.

Functional Dependencies:

There are no functional dependencies.

Manual Changes:

There are no manual activities

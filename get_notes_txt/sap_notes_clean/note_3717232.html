Symptom

During CtlibController polling, a SIGSEGV in mutex_lock_impl may occur, followed by timeslice errors or suspended processes accessing a remote ASE.
a stack trace containing the functions listed below is output.

ksctsuspend
omni_ct_results
CtlibRemoteAccess::prepareResultSet
CtlibRemoteAccess::startRpcScan
RPCRemoteAccess::startScan
LeScanOp::_LeOpNext
LeOperator::LeOpNext
LeNLJOp::_LeOpNext
LeOperator::LeOpNext
LeUpdateOp::_LeOpNext
LeOperator::LeOpNext
LeEmitNoSndOp::_LeOpNext
LeOperator::LeOpNext

Environment

  • Adaptive Server Enterprise（ASE) 16.0 SP04 PL06 HF1
  • Component Integration Services (CIS)
  • Solaris on Sparc 64bit

Cause

  • ASE uses Open Client CT-Library to access a remote ASE via CIS.
    Within the ASE process, the CtlibController thread, which is one of the OS-native threads, performs network polling.
    In this case, an error occurred when attempting to acquire a mutex during the network polling operation inside the Open Client Net-Library.
    The SIGSEGV occurred in mutex_lock_impl(), and this function is not included in the ASE or Open Client libraries.
  • The issue has been logged under SAP CR 831791. Its description reads, "The message, "Current process infected with signal 11 (SIGSEGV)", in the module 'mutex_lock_impl' together with a stacktrace which includes the module 'sybnet__async_poll' may be reported in the SAP ASE error log when the CtlibController thread performs a poll. Subsequently, a timeslice error may appear or processes accessing to a remote ASE are suspended."

Resolution

  • Please collect the following information when the issue occurs.
    1. A memory dump generated by the SIGSEGV
    2. A memory dump generated by the timeslice error
    3. Run the OS pstack command on the dataserver process 5 times at 10-second intervals
      to collect native OS thread stack traces.
      pstack <dataserver PID>
    4. Run the OS prstat command and collect CPU usage information.
      prstat -c 5 10
    5. Run the OS pmap command once against the dataserver process
      pmap <dataserver PID>

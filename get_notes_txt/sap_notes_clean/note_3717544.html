Symptom

  • During insert jobs
  • ASE returns client error message:

    There are fewer columns in the INSERT statement than the number of values provided

Environment

SAP Adaptive Server Enterprise (ASE) - All versions.

Cause

  • The message is stating that there is a count discrepancy between the insert values provided compared to the number of columns specified.
  • This is usually a syntax / command error

  • In rare circumstances, it has been reported due to a character set mismatch that caused an unexpected end to one of the strings being inserted.
    • This may be seen when using an input script from a host server or ASE server with different character set / locales

Resolution

  • Generally, the command needs to be reviewed and the syntax corrected to get the correct match between insert values and columns specified

  • For the specific situation where there is a character set issue causing this
    • Compare between source & target servers to identify discrepancies in character set and/or locale configurations
      • This issue may be more prevalent between single and multi-byte (UTF) character sets

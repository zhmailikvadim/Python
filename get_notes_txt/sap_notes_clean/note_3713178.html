Symptom

ABAP runtime error RAISE_EXCEPTION occurs when attempting to create two standard cost estimates in parallel for the same material.

Environment

  • Controlling (CO)
  • SAP R/3
  • SAP R/3 Enterprise 
  • SAP ERP
  • SAP ERP Central Component
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA

Reproducing the Issue

  1. Start transaction code CK11N.
  2. Enter the parameters, such as material, plant, costing variant and costing date.
  3. Create the standard cost estimate (do not save yet).
  4. Now, create a new GUI window.
  5. Start transaction code CK11N again.
  6. Enter the exact same parameters from step #2.
  7. Try to create the standard cost estimate.
  8. The system will issue the message: "Costing blocked - Do you want to cancel?".
  9. If the option "No" is chosen, the runtime error RAISE_EXCEPTION is triggered from program SAPLCKDI with the message "Exception condition TABLE_LOCKED triggered".

Cause

  • The root cause of the issue is that no interval has been maintained for the number range object PCPA_PATNR.
  • Please check the number range settings using the transaction code SRNO.

Resolution

  • Please install the SAP Note 3699371.
  • With this note, the interval 01 for the number range object PCPA_PATNR will be created during the first calculation with CK11N (with a range from 1 to 999999).
  • The runtime error will no longer occur.

See Also

  • Refer to: SAP Note 3699371 - CK11N, CK40N, CK51N: Dump durch Laufzeitfehler RAISE_EXCEPTION mit Ausnahme TABLE_LOCKED

Keywords

Short Dump, Runtime Error, Raise_Exception, SAPLCKDI, Costing Blocked, Group Costing, CK11N, Reference Costing, Exception, Condition, Table_Locked.

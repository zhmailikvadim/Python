Symptom

After enabling the Customer Hosted Web Channel script for iOS 26 Privacy settings in the account you observe that the “x” symbol used to close the overlay campaign is not rendered correctly.

Environment

SAP Emarsys Customer Engagement

Reproducing the Issue

  1. Attempt to configure a hosted URL for iOS 26 support as per the provided documentation.
  2. Test an overlay campaign and verify that the “x” symbol used to close the campaign renders correctly.

Cause

The issue is related to the way the wpes6.js file is downloaded from SAP.

Resolution

Use wget or curl to download the wpes6.js from us as follows:

  • wget https://static.scarabresearch.com/wpjs/wpes6.js -O emarsys_wpes6.js

  • curl https://static.scarabresearch.com/wpjs/wpes6.js -o emarsys_wpes6.js

See Also

Please also refer to the following help file:

  • Ensuring Web Channel Campaigns Work with iOS 26
    => Self-Hosted Web Channel Script
    If the script is not downloaded using wgetor curl, the file might not have the right encoding and the x symbol used in overlay campaigns to close them might not render correctly.

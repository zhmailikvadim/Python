Symptom

  • When opening Manage Packages in GRDC, the message appears: "Can't get the global parameters from the SAP S/4HANA system. The SAP S/4HANA backend RFC function module returned the following error: com.sap.cloud.sdk.s4hana.connectivity.exception.AccessDeniedException: 403 Forbidden. Failed to establish a trusted connection to the XXX."
  • Response headers include: www-authenticate: Bearer realm="SAP NetWeaver Application Server [XXX/100]", error="insufficient_scope", error_description="The request requires higher privileges than provided by the access token.", scope="SAP_BC_SRT_SCS_EXT_HTTP".
  • Connectivity trace shows: "OAuth 2.0 Error: The request requires higher privileges than provided by the access token. Detailed information: Required scope API_GRMASTERDATA_SRV_0001 is not contained in access token scope list."
  • Connectivity trace also shows: "OAuth 2.0 Error: The request requires higher privileges than provided by the access token. Detailed information: Required scope SAP_BC_SRT_SCS_EXT_HTTP is not contained in access token scope list."
  • Additional detail: "The requested OAuth 2.0 scope cannot be granted because it is either not assigned to the current user or to the OAuth 2.0 client. ... error: invalid_scope."

Environment

SAP Group Reporting Data Collection connected to SAP S/4HANA Public Cloud

Reproducing the Issue

  1. Open SAP Group Reporting Data Collection.
  2. Choose Manage Packages.

Cause

  • Required OAuth scopes (for example, SAP_BC_SRT_SCS_EXT_HTTP, API_GRMASTERDATA_SRV_0001) are missing from the access token because they are not assigned to the current user and/or the OAuth client.
  • Not all GRDC-relevant business catalogs are included in the business roles assigned to the user (custom role missing GRDC catalogs compared to the template role).

Resolution

  1. Verify that communication scenario SAP_COM_0241 is available in the SAP S/4HANA Public Cloud tenant; if it is missing, activate the related scope item 287 via SAP Cloud Operations.
  2. Compare the user’s custom business role with the template role (for example, SAP_BR_GL_ACCOUNTANT_GRP) and add missing GRDC business catalogs, such as:
  • SAP_FIN_BC_CCON_GRDC_ERD_PC (Group Reporting Data Collection - Enter Data)
  • SAP_FIN_BC_CCON_GRDC_XPD_PC (Group Reporting Data Collection - Export Package Data)
  • SAP_FIN_BC_CCON_GRDC_RDM_PC (Group Reporting Data Collection - Run Data Mapping)

General checks to do:

  1. Ensure the business user has the required role collections and roles on SAP BTP for GRDC so that the OAuth token contains the necessary scopes.
  2. Review identity provider settings and ensure only the intended identity provider is marked for logon.
  3. Configure and validate the GRDC connection to SAP S/4HANA Public Cloud by following the step-by-step guide (Blog in SAP Community)  and template  (word document available in the SAP Documentation) referenced below.
  4. After applying changes, have the user open the “Enter Global Parameters” app in GRDC to set global parameters, then test Manage Packages again.

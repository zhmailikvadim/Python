Symptom

  • API with limit and offset parameters does not return any data, only a status response is received.
  • When the same API request is sent without the limit and offset parameters, the data is returned correctly.

Environment

  • SAP Emarsys Customer Engagement

Reproducing the Issue

  1. Send an API request to the following endpoint: https://api.emarsys.net/api/v3/export/{exportid}/data?offset=0&limit=100
  2. Observe that only the status response is returned without any data.
  3. Send the same API request to the following endpoint without the offset and limit parameters: https://api.emarsys.net/api/v3/export/{exportid}/data
  4. Observe that the data is returned correctly.

Cause

  • The issue occurs because the query parameters in the API request are missing a preceding slash symbol.

Resolution

  1. Add a slash symbol ahead of the query parameters in the API request: https://api.emarsys.net/api/v3/export/{exportid}/data/?limit=100&offset=0
  2. Verify that the data is returned correctly with the updated API request format.

Symptom

The start date of the collection record is calculated differently depending on how the record is created.

Specifically, the start date is calculated differently for automatic, rule-based creation than for direct creation, for example, from the customer line item list.

Other Terms

Start date, record header data, runtime calculation, attachment, ticket 1000007235

Reason and Prerequisites

This problem is caused by a program error.

The prerequisite for implementing the program corrections is one of the following components:

  • CASH, version 200_702, patch level 2503
  • CASH, version 220_751H, patch level 2503

Solution

The calculation of the start date has now been unified. The calculation is now always performed in the same way as for automatic creation.

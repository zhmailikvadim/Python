Symptom

Functional Area is set in FI documents but appears blank in costing-based COPA document when do billing in VF02 in S/4HANA system, but in ECC system, it is populated.  
This issue occurs specifically for FI documents generated from SD billing documents.

Environment

  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA Finance
  • SAP S/4HANA

Reproducing the Issue

1. Create SD billing documents.
2. Trigger FI documents from SD billing documents.
3. Observe that Functional Area is set in FI documents but appears blank in CO profitability documents.

Cause

In general, the Functional Area is used for cost-of-sales accounting. By definition, costing-based CO-PA supports cost-of-sales accounting (costs are posted together with revenues at billing) and does not require the FKBER characteristic; FKBER is normally relevant only for account-based CO-PA.

Historically, Note 410258 was required to transfer FKBER to any type of CO-PA. With S/4HANA, FKBER became a fixed characteristic because account-based CO-PA was emphasized. Customers running only costing-based CO-PA reported poor performance during mass billing because the profitability segment was being updated with FKBER even though it was not needed for costing-based CO-PA.

As a result, Note 2800046 was delivered via support packages to prevent FKBER from being automatically updated in the profitability segment when account-based CO-PA is not active, avoiding the unnecessary updates and associated performance impact.

Resolution

If FKBER value in costing based CO-PA is really necessary, the workaround would be to fill this within second exit of enhancement COPA0005 (read it from T_ACCIT) as this would not have any negative impact on performance.

Symptom

When running transaction MDS_LOAD_COCKPIT to synchronize Vendors to Business Partners (or Customers to Business Partners), the process fails with a SYSFAIL status in SMQ2. The queue monitor displays the following error message:

CREATE DATA: The specified type (/NAKISA/HRP98XX) is not a valid data type
Message no. SR053

The issue can occur either for infotype 9821 or other /NAKISA/HRP98XX infotypes, depending on system customizing.

Environment

  • SAP ERP
  • SAP S/4HANA on-premise
  • SAP S/4HANA private cloud edition

Reproducing the Issue

  1. Go to transaction MDS_LOAD_COCKPIT.
  2. Set up synchronization process (e.g., Vendor to Business Partner).
  3. Fill in required parameters (e.g., use * as a selection).
  4. Execute with background processing enabled.
  5. Open SMQ2 and check the corresponding queue for the process.
  6. Observe SYSFAIL with error message in the Details/Long Text.

Cause

The error occurs because there are custom entries in the view T777D for Nakisa-specific infotypes (/NAKISA/HRP98XX) remaining in the system after the Nakisa add-on has been uninstalled. The corresponding database tables for these infotypes, such as /NAKISA/HRP9821, are no longer present in the system, leading to the error when processes try to access them.

Resolution

To resolve this issue:

  1. Log in to the affected SAP system.
  2. Go to transaction SM30.
  3. Enter view T777D and display the contents.
  4. Search for entries related to infotypes /NAKISA/HRP98XX (e.g., 9821, 9822, 9823, etc.).
  5. Identify the entries where the database table is in the format /NAKISA/HRP98XX.
  6. Delete these entries from view T777D if the corresponding Nakisa add-on and tables have been removed from the system.
  7. Save your changes.
  8. Re-run your synchronization scenario in MDS_LOAD_COCKPIT.

Note:
If the error recurs for a different infotype (e.g., /NAKISA/HRP9822), repeat the steps above to remove any additional obsolete infotype entries related to Nakisa.

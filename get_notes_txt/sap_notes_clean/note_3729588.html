Symptom

  • While posting GR and MIRO, G/L accounts configured to transaction event key Z01 did not update.
  • A Statistical condition type is assigned to pricing procedure and OBYC settings are maintained, but no accounting entries are posted.

Environment

  • Materials Management (MM)
  • SAP ERP Central Component
  • SAP ERP
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA

Reproducing the Issue

  1. Configure a statistical condition type and assign it to pricing procedure.
  2. Maintain OBYC settings for transaction event key.
  3. Create a purchase order with condition type created.
  4. Post GR and MIRO for the purchase order.
  5. Observe that G/L accounts configured to transaction event key are not updated.

Cause

  • The statistical condition type does not generate accounting entries during GR and MIRO. In standard behavior, statistical conditions only display values in the purchase order or invoice and do not post accounting entries.
  • MM is not working in the same way as SD does. The account determination logic in MM is handled differently compared to SD.
  • In SD, the pricing procedure controls revenue conditions, account keys, and posting to G/L (via account determination in FI).
  • In MM, the pricing procedure (calculation schema) is mainly used for purchase orders and invoice verification, but account determination is linked to transaction keys (WRX, PRD, FRE, etc.) and OBYC settings.
  • The "statistical" flag in MM also exists, but it only influences how the condition behaves in pricing, so it does not automatically create FI postings unless the condition type is explicitly linked to an account key / transaction key in MM. 

Resolution

  1. Remove the "Statistical" flag from condition type in the pricing procedure.
  2. Assign an account key (e.g., Z01) to condition type in the pricing procedure.
  3. Ensure that OBYC settings are correctly maintained for the transaction event key.
  4. Test the configuration to confirm that G/L accounts are updated during GR and MIRO.
  5. If condition type must remain statistical, note that standard behavior does not support FI postings for statistical conditions in Materials Management (MM).
  6. For statistical conditions to generate FI postings, custom development or alternative configurations may be required.

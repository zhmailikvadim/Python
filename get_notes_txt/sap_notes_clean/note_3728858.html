Symptom

  • The “Create” button is not visible or not working for incoming eDocuments in EDOC_COCKPIT.
  • Clicking the Create button results in error: “Action is not allowed”
  • The “Submit” button is not available for incoming eDocuments.

Environment

  • SAP ERP 
  • SAP S/4HANA 
  • SAP Document and Reporting Compliance, Cloud edition

Reproducing the Issue

  1. Open transaction EDOC_COCKPIT.
  2. Select incoming eDocument for Poland.
  3. Observe: “Create” button missing or not working “Submit” button not available

Cause

The configuration in the EDOINCOMSOLV table includes the `edoc process` PLINVIN, which prevents the "create incoming invoice" button from functioning correctly.

Resolution

  1. Access the EDOINCOMSOLV table in the system.
  2. Remove the `edoc process` PLINVIN entry.
  3. Maintain only the PLKSEFIN entry for the company code.
  4. Retest the functionality to ensure the "create incoming invoice" button is operational.

Note: Additionally, for incoming eDocuments, the Submit action is not supported in standard.

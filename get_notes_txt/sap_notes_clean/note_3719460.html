Symptom

When executing the report /SHCM/RH_SYNC_BUPA_EMPL_SINGLE to synchronize an employee with a Business Partner (BP) in S/4HANA, the process fails with error message:
Error in validity periods of addresses.
As a result, the Business Partner is not created or updated, preventing HR processes (such as expense reimbursement) dependent on this BP from functioning correctly.

Environment

  • SAP S/4HANA on-premise
  • SAP S/4HANA private cloud edition

Reproducing the Issue

  1. Execute the report /SHCM/RH_SYNC_BUPA_EMPL_SINGLE for an employee to synchronize HR master data to a Business Partner in S/4HANA.
  2. Observe that the report fails with the error message: “Error in validity periods of addresses.”
  3. The synchronization does not complete, and the expected Business Partner is not created or updated.

Cause

This error is typically caused by an inconsistency or invalid entry in the address validity periods of the HR master data. Additionally, recent changes in the address transfer logic for BP synchronization may require corrective action for affected employees. Missing authorizations can also contribute to this error.

Resolution

To resolve the issue, please follow these steps in the specified order:

  1. Execute Report Again:
    Run the /SHCM/RH_SYNC_BUPA_EMPL_SINGLE report again for the affected employee.

  2. Check Authorizations:
    After execution, check transaction SU53 for any missing authorizations. If any authorization issues are detected, resolve them before continuing.

  3. Override Address with Default:
    If no authorization issues are found, execute /SHCM/RH_SYNC_BUPA_EMPL_SINGLE again for the affected employee only, but this time activate the “Override Address with Default” flag/option in the selection screen.

    Note: This step should be performed only once for the affected employee, as it resets the address data according to new logic and resolves issues with address validity.

  4. Further Assistance (If Needed):
    If the problem persists, open a support ticket and provide system access and relevant details for further analysis by SAP Support.

See Also

Symptom

A Purchase requisition line item is deleted.
Still PR 'Open Qty' field is getting values (EXPECT: zero in the same field)

Environment

  • Materials Management (MM)
  • SAP ERP
  • Central Component SAP ERP
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA

Reproducing the Issue

  1. Go to ME53N and open PR and Observe deleted line item.
  2. Go to Quantities / Dates tab in item level and observe open qty

Cause

Open quantity field is not saved in any table. It is calculated in the background when checking the PR.

SAPLMEGUI                      / LMEGUICJT
METHOD          / TRANSPORT_FROM_MODEL (LCL_REQ_3321_VIEW_MM)
      mereq3321_pbo-omeng = l_data-menge - l_data-bsmng.

According to the coding, it is calculated based on PR quantity and order quantity. When PR is deleted, PR quantity is not changed, so open quantity is not changed also. 

Resolution

This is standard behavior.

Keywords

LMEGUICJT, omeng, deleted, ME53N, bsmng, menge

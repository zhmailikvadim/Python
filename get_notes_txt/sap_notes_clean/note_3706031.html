Symptom

- The "New Main Employment" field is displayed in the "All Actions > Employment Details" section for employees, even when there are no concurrent employments associated with the employee.
- The same field does not appear in the "Employment Information" portlet when accessed via the pencil (Edit) icon, indicating inconsistent behavior between UI entry points.
- This issue occurs in the development environment, and the functionality is deemed incorrect as the field should not be visible under these conditions 

- This field is only used when terminating a primary employment, and there are multiple secondary employments to choose as the new primary. It does not make sense in the context of simple Employment Edit where termination and start/end date change is not possible 

Environment

SAP SuccessFactors Employee Central

Reproducing the Issue

1. Open an employee record.
2. Confirm that the employee has no concurrent employments.
3. Navigate to "All Actions > Employment Details" and observe the "New Main Employment" field.
4. Access the "Employment Information" portlet via the pencil (Edit) icon and confirm the field is not displayed.

Cause

The "New Main Employment" field is incorrectly exposed in the "All Actions > Employment Details" UI for employees without concurrent employments. This field is only relevant when terminating a primary employment with multiple secondary employments.

Resolution

Code correction to remove the "New Main Employment" field from "Take Action -> Employment Details" UI is scheduled for 1H 2026  release.

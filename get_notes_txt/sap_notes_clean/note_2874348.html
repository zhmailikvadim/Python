Symptom

When the user is performing a candidate search, an error regarding the user country is displayed. Please, refer to the following:

"Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental."

Environment

SAP SuccessFactors Recruiting Management

Reproducing the Issue

  1. Go to Recruiting → Candidates tab
  2. Search for any candidate
  3. Referred message will be thrown

Cause

As the message says, the user's Country is not configured. In the User Data File, the Country is either blank or not configured properly, therefore the system cannot match the user country and the candidate privacy option selected.

Resolution

You must update the Country value for the affected user. For that, you can follow the next steps:

  1. In Admin Center → Go to Employee Export
  2. Export User File
  3. On the .csv file, update the Country column for the affected user
  4. In Admin Center → Go to Import Employee Data
  5. Select Import Data operation
  6. Entity as Basic Import
  7. Choose the file
  8. Validate Import File Data to perform a validation before uploading the file
  9. Import button to upload the file
  10. To follow the job, you can click on Monitor Job button

Make sure that the changes were applied successfully by checking if the user has access now to the candidate search.

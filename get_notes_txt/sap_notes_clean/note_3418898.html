Symptom

When maintaining Event Linkage Table SWETYPV, the following message raised with a popup 'Table content obsolete':

"The tables of event linkages have had to be extended due to the introduction of classes. As this was not possible with the existing tables for technical reasons, new tables have been introduced and the existing linkages converted. The content of these tables is therefore obsolete and no longer evaluated. 
Use the new maintenance dialogs"

"Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental." 

Environment

  • SAP Business Workflow
  • SAP NetWeaver
  • SAP Web Application Server for SAP S/4 HANA
  • ABAP PLATFORM - Application Server ABAP

Reproducing the Issue

Update table view SWETYPV by SM30 or SE11.

Cause

The event linkage configuration has been enhanced by adding class based events. So the table view SWETYPV is no longer used for the configuration. 

Resolution

Instead of updating the table view, always use configuration transactions SWE2 or SWETYPV for the event linkage configuration.

Keywords

WF_GDPR, SWF_WORKFLOW, WFMSTD, WF_HOWTO, WF_SAP4ME, SWE2, SWE3, SWFDVEVTY2, SWFDVEVTYNOOPT

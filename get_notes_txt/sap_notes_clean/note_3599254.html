Symptom

Negative available stocks are created when moving lower level Handling Unit (sub-HU) while top-HU (Handling Unit) has an open Warehouse Task (WT), although such negative stock is not allowed in the Storage Type (ST) customizing.

Environment

  • SAP Extended Warehouse Management (SCM-EWM);
  • SAP Extended Warehouse Management Warehouse Order Processing (SCM-EWM-WOP);
  • SAP Extended Warehouse Management Work Center (SCM-EWM-WC);
  • SAP S/4HANA;
  • SAP NetWeaver.

Reproducing the Issue

  1. Create a repacking WT for sub-HU while an open WT exists for the top-HU;
  2. Observe that negative stocks are created in /SCWM/MON or even /SCWM/PACK transactions.

Cause

Customizing.

The Warehouse Process Type (WPT) used for the repacking WT allows negative available stocks to exist. On product documentation it is written:
"If negative stock is not allowed in a storage type, set this indicator to override the settings for the storage type."

Therefore, even if the source ST does not allow negative stocks to exist, when using a WPT that allows it, the ST customizing is not considered and EWM will create negative stocks.

This is why the repacking WT could be created for sub-HU while an open WT existed for the top-HU.
If the WPT customizing was also set to not allow negative stocks, error /SCWM/L3725 would have been raised.

Resolution

If the negative stock existence is not foreseen on the business processes, user may change the WPT customizing on path below:

Transaction SPRO
SCM Extended Warehouse Management
Extended Warehouse Management
Cross-Process Settings
Warehouse Task
Define Warehouse Process Type

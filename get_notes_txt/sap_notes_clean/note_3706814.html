Symptom

 During the Deal Amendment process in the CPE Redesign scope, formula conditions are not correctly synchronized with backend SAP contracts (MM/SD) when new items are added or details are copied. Instead of reflecting the specific Deal data, the system incorrectly generates a default formula condition with three terms.

Other Terms

/DMBE/; CPE Redesign/ Deal Amend; Deal Capture; Formula Condition; Data mapper; Document Condition

Reason and Prerequisites

 The CPE Redesign framework must be active, and the Deal Amendment process must be initiated for a contract containing newly added or copied items.

Solution

The method get_document_condition in class /DMBE/CLM_CPE_RDSG_DATA_MAPPER has been updated to improve the fallback logic. If the system cannot find a document condition via the specific contract and item reference, it will now return the document condition from the first item in the list to ensure the deal's configuration is correctly inherited.

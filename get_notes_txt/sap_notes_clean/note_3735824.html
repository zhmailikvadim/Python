Symptom

Access the OLME transaction (Purchase Requisition → Reservation → Define Settings for Creating Reservations), only one movement type can be maintained per requisition scenario.

For requisition scenario “S – Procurement via Self-Service”, movement type 201 is configured; attempting to add another entry for the same scenario results in error “An entry already exists with the same key” (Message No. SV009).

Environment

  • SAP S/4HANA
  • SAP FIORI

Reproducing the Issue

  1. In Customizing, navigate to: SAP Implementation Guide → Materials Management → Purchasing → Purchase Requisition → Reservation → Define Settings for Creating Reservations.
  2. Maintain movement type XXX for requisition scenario “S – Self-Service Procurement”.
  3. Save it.
  4. Add another entry
  5. Attempt to create an additional entry for the same requisition scenario (S – Self-Service Procurement) with a second movement type (for example, YYY); the system displays “An entry already exists with the same key” (Message No. SV009).

Cause

It is technically not possible to maintain multiple movement types for the same requisition scenario in Define Settings for Creating Reservations; the configuration supports only one movement type per scenario.

Resolution

Standard behavior

Symptom

When attempting to export a catalog, the system fails to generate the catalog file. The B&I log displays the following exception:

(util:WARN) [ID2886]: ServerThread.run() caught an exception while running ariba.catalog.admin.server.AsyncBuyerCheckoutRunner.
java.lang.ArrayIndexOutOfBoundsException: Index 2048 out of bounds for length 2048

Reproducing the Issue

  1. Navigate to Core Administration of the parent realm.
  2. Go to Catalog Manager > Catalog.
  3. Search for the target catalog.
  4. Click on the catalog version.
  5. Click the export button (up-arrow, ↑).
  6. Select the CIF 3.0 format and click Export.

Cause

The issue is caused by a defect in a third-party library (Apache Xerces), not the SAP product. Xerces parses XML in 2048-byte chunks. In a very rare boundary case, if a read operation returns exactly 2048 bytes, and the subsequent read returns 0 bytes, an ArrayIndexOutOfBoundsException is triggered.

Resolution

Since this is a known issue in the third-party library, a direct fix from SAP is not possible and must await a library upgrade.

In the meantime, the following workaround is suggested:

Re-import the existing catalog after making minor modifications, such as editing the description of existing items or adding a dummy item.

This increases the overall byte count of the catalog data, which helps avoid the specific byte count scenario that triggers this boundary condition.

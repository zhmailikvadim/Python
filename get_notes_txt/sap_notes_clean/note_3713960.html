Symptom

You are using the F5356 Manage Maintenance Items application and you try to set the End Date Offset greater than the Start Date Offset. Additionally, the inconsistent changes can be saved, without the corresponding error shown.

You notice that during the Maintenance Plan scheduling, these items are generating Maintenance Orders, where the Basic Start Date is later then the Basic Finish Date.

Other Terms

MPLAN_SCHEDULE, End Date Offset, Basic start date, Basic finish date, Final Due Date, IP10, IP30, IP30H, F5356 Manage Maintenance Items

Reason and Prerequisites

Each Maintenance Order has a Final Due Date. If the offsets are wrongly set in the Maintenance Items, the Basic Start/End dates are miscalculated in the generated call objects.

A new validation has to be added, so the End Date Offset cannot be set greater then the Start Date Offset. Additionally, maintenance plans with wrongly set maintenance items should not to be scheduled.

Solution

Please implement the attached correction instructions or Support Packages.

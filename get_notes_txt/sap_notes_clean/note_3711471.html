Symptom

  • When costing with transaction CK11N or CK40N, it's found that the transfer control does not work with Periods field initial. 
  • In the transfer control it's set to use the Current Standard Cost Estimate and Fiscal year is not marked

     "Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental." 

Environment

  • Controlling (CO)
  • SAP R/3
  • SAP R/3 Enterprise 4.7
  • SAP ERP Central Component
  • SAP ERP
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA Finance
  • SAP S/4HANA

Reproducing the Issue

  1. Execute transaction CK11N for costing.

Cause

With Period initial, the system will only search for the standard cost estimate with the current period. In case the BOM has standard cost estimate released in the previous periods, the system will still not use it although it's valid.

Resolution

  1. Please consider to mark the Fiscal year field.
  2. Input some value in the Period field. Also a big value in this field may result in problems in case year-dependent fiscal year variant is in use as KBA 3597187

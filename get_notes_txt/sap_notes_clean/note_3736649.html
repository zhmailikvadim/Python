Symptom

Schedule Synchronization in Data Audit didn't transfer data from table AuditDataTmpAppSet to table AuditDataAppSet, no errors were found when reviewing the logs in Windows Task Scheduler.

Environment

SAP Business Planning and Consolidation 10.1 version for the Microsoft platform

Reproducing the Issue

Enable Schedule Synchronization in Data Audit

Cause

There are too many records in table AuditDataTmpAppSet 

Resolution

The audit data to be moved can be controlled by StagingCode in table [AppSetExtra].[dbo].[AuditHdrApp], the records with the BatchIds where StagingCode <> 0 are going to be moved from AuditDataTmpAppSet to AuditDataAppSet. 

  1. Backup table AuditDataTmpAppSet.
  2. Control the BatchIds to be processed by updating StagingCode in AuditHdrApp to 0 for other records, this will reduce data volume in Schedule Synchronization.
  3. Run Schedule Synchronization.
  4. Repeat step 2&3 until all BatchIds are processed.

Keywords

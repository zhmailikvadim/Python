Symptom

  • During invoice creation, plant and shipping point are being copied from the sales order instead of the delivery document, despite the invoice being delivery-related.
  • Copying requirements and configuration settings appear to be correct, but the system behavior is not as expected.

Environment

  • Sales And Distribution (SD)
  • SAP R/3
  • SAP S/4HANA
  • SAP R/3 Enterprise 4.7
  • SAP ERP Central Component
  • SAP ERP
  • SAP Enhancement package for SAP ERP
  • SAP Enhancement package for SAP ERP, version for SAP HANA

Reproducing the Issue

  1. Ensure the delivery document contains the correct plant and shipping point.
  2. Execute invoice creation (e.g., transaction VF01).
  3. Observe that the plant and shipping point are copied from the sales order instead of the delivery document.

Cause

This behavior is a standard design to pass the plant and shipping plant from the source document which in this case is the sales order. 
The data is passed in the below logic :

 FORM : VBRK_VBRP_FUELLEN    include : LV60AA93       

  MOVE-CORRESPONDING vbkd TO vbrp.
  MOVE-CORRESPONDING vbak TO vbrp.
  MOVE-CORRESPONDING vbap TO vbrp.

Resolution

If your business requires to use different values from the delivery document, please consider implementing your own logic in the user exits present in the program RV60AFZZ or 
you can assign a custom VOFM routine in the copy control configuration.

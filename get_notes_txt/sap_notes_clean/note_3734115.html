Symptom

RMAN-00571: ===========================================================
RMAN-00569: =============== ERROR MESSAGE STACK FOLLOWS ===============
RMAN-00571: ===========================================================
RMAN-03002: failure of restore command at 02/26/2026 21:33:40
RMAN-20242: specification does not match any archived log in the repository

Environment

BRRECOVER 7.40 (41) or later

Reproducing the Issue

Restore recovery of oracle database with brtools using RMAN 

Cause

There is a mis-match between the backup contents and what is physically on disk in the archivelog directory

Resolution

Run the command:
  RMAN> list archivelog all;

  # Compare the output with the contents of the archivelog directory.
  # RMAN-20242 is raised when we try to find a log in the rman repository that does not exist on disk.


If the log files exist somewhere (backup dir, NFS, staging):

RMAN> catalog start with '/path/to/archivelogs/'; 

Then verify:

RMAN> list archivelog all; 

Now restore again:

Symptom

I am performing a data upload (e.g., User Upload or Worker Update Upload) and the transaction completes without errors. however, when I check the records in the SAP Fieldglass User Interface (UI), specific fields (e.g., Email) have not changed and still reflect the old values, even though the new values were provided in the upload file.

Cause

This issue occurs when the Skip on Update option is enabled for a specific field within the connector's column transformation settings.

When this flag is checked, the system is instructed to use the uploaded value only during the creation of a new record. For any subsequent updates to existing records, the system explicitly ignores the value in that column and retains the data already stored in the database. This is often used for fields that should remain static once a record is established, such as unique identifiers or original hire dates.

Resolution

To allow the system to update the field via upload, you must disable the Skip on Update flag in the connector configuration.

  1. Log in as a Configuration Manager user.
  2. Navigate to Connector Catalog.
  3. Search for the impacted connector and click Edit.
  4. Navigate to the Upload Column Transform section (under the Customize header).
  5. Locate the specific column that is not updating (e.g., Email).
  6. Uncheck the Skip on Update box for that column.
  7. Click Update and follow the wizard to the end to save the changes.
  8. Re-verify: Run the upload file again. The system should now process and reflect the updated values in the UI.

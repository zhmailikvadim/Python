Symptom

When displaying a product in transaction /SCWM/MAT1, the system issues a popup prompting to save changes although the user is in Display mode and has not modified any data. The issue occurs only when leaving the transaction from the Unit of Measure (UoM) tab. Internally, a recalculation of net weight occurs, and the system detects a mismatch between the stored value and the recalculated value (usually rounding). This leads to the message “Would you like to save changes?” even though no user‑initiated data change was made.

Other Terms

/SCWM/MAT1, /SAPAPO/MAT1, Unit of Measure, MARM, Weight conversion, LMAT_MASTERF10, Product master in EWM

Reason and Prerequisites

During display of UoM data, the system recalculates the net weight based on the reference unit and conversion factors. Due to rounding to different decimal precision, the recalculated value may differ from the stored value. The system interprets this difference as a data change and incorrectly sets the relevant “changed” indicator. This behavior may occur in SAP S/4HANA releases.

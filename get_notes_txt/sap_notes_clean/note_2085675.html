Symptom

  • On the material list of a production order or process order, you replace a material component with a phantom assembly. When you do this, the bill of material of the phantom assembly is not exploded.
    If you add the phantom assembly as a new component however, the bil of material is exploded.
  • On the component overview of a production order or the material list of a process order, you enter a material component for which the special procurement key 50 (phantom assembly) is defined in the material master. The bill of material explosion is terminated on the dialog box that is displayed. The system issues message CO 223:
    "A valid bill of material could not be found for material [material number]"
    A normal reservation should therefore be created. Instead, the system sets the "Phantom Assembly" indicator and the indicator for the backflush is not determined.
  • If you add a phantom assembly in a network and confirm that it is relevant to procurement (dialog box), the system does not actually create any requirements.
  • If you add a phantom assembly in an EAM order, the system does not determine any planned costs or requirements, which is incorrect.

Other Terms

Reason and Prerequisites

This program error can be noticed only after you have implemented SAP Note 2026989.

Solution

Implement the source code corrections as described in the correction instructions.

Symptom

When this option is set, all newly created indexes will be compressed by default.

_ADVANCED_INDEX_COMPRESSION_OPTIONS = 16

However, the index before setting this option is uncompressed.
Note: This option requires "Oracle Advanced Compression" license.

Environment

  • Oracle database 12.2.0.1
  • Oracle database 18c
  • Oracle database 19c
  • SAP Release independent
  • Operating System independent

Reproducing the Issue

  1. Remove the Oracle parameter "_advanced_index_compression_options16".
  2. Perform index decompression.
  3. Observe the mix of compressed and decompressed indexes in the system.

Resolution

  1. Check for uncompressed indexes using the following SQL query:
    SELECT INDEX_NAME FROM DBA_INDEXES WHERE OWNER = 'SAPSR3' and COMPRESSION = 'DISABLED':

  2. Compress uncompressed indexes:
    ALTER INDEX "<index_name>" REBUILD ONLINE COMPRESS ADVANCED LOW;

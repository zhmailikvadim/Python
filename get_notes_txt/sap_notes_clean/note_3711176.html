Symptom

The ABAP EarlyWatch Alert (EWA) report shows incorrect or misleading alerts related to Database which is not in use by the system, such as:

"Recommendation to upgrade SAP HANA from SPS 05 to SPS 08, although the productive system is already running SPS 07."

The EWA report references HANA database information from an unrelated system database.

Discrepancy observed between the actual system status and the data displayed in the EWA report generated via SAP for Me / SAP Portal.

Environment

SAP S/4HANA 

Solution Manager 7.2

Cause

The EWA report automatically collects service data from all active database connections (DBCON) configured in the system.

If a DBCON exists to an external SAP HANA Database, the HANA Checks framework may incorrectly include: HANA version, SPS level, Compatibility scope findings  from that external database, even though it is not the primary database of the reported system.

Resolution

Step 1: Identify Active Database Connections

  • Log on to the affected system.
  • Open transaction DBCO.
  • Review all active database connections.
  • Identify connections pointing to external or unrelated SAP HANA database systems.

Step 2: Decide Based on Business Requirement
Option A: Database Connection Not Required

If the database connection is no longer needed:

  • Delete the database connection using transaction DBCO.
  • Save the changes.

Result:
The external HANA database will no longer be included in EWA data collection, and related incorrect alerts will not appear in future EWA reports.

Option B: Database Connection Required

If the database connection is required for integration, reporting, or monitoring:

  • Open transaction DBACOCKPIT.
  • Create or maintain an entry for the same database connection.
  • In the Description field, add the keyword NON_EWA_.
  • Save the changes.

Result:
The database connection remains available for business use, but its HANA data is explicitly excluded from EWA HANA Checks.

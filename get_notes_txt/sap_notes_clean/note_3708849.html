Symptom

On Hebrew Display - when creating a UI Template and dragging several User Defined Fields (UDF) to the form itself from the title, the UDF's are displayed incorrectly.

Image/data in this SAP Note is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.

Reproducing the issue

Prerequisites:

 1. Change the language display to Hebrew.

 2. Create 3 UDF's under Marketing Documents Title (Tools → Customization Tools → User Defined Fields Managements), for example Alphanumeric UDF's with the following titles: testB, testC ,testddddddddd.

Scenario:

1. Go to Administration → Utilities → UI Configuration Template and create a new UI Template.

2. In the template drag first the two UDF's testB, testC under the Dates part and save it. When opening the UI template again the UDF's are shown correctly.

3. Drag the third UDF testddddddddd to the right side of those UDF's (in the middle) and save.

4. Open the UI Template again. Sometimes it appears only the second time opening the UI template.

Expected behavior:

The UDF's are shown correctly.

Actual behavior:

The UDF's are shown incorrect. The title of some of the UDF's are missing.

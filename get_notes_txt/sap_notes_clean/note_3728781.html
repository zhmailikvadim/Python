Symptom

In Create Maintenance Request Fiori app (F1511A), opening the Assess Priority dialog displays likelihoods or consequences that have no configured priority mapping. These appear as selectable entries even though no valid priority combination exists.

Other Terms

F1511A, PM-FIO-WOC-MN, MaintPriorityDesc, Plant, NotificationType, OData, $filter, NE operator

Reason and Prerequisites

The reason is a program error. The selection logic filters by Plant and NotificationType only and does not restrict the result set to likelihood and consequence combinations that map to a maintenance priority, which leads to display inconsistencies.

Solution

Install this SAP Note. The correction adds selection logic to exclude combinations that do not have a maintenance priority so only valid likelihood and consequence pairs remain visible.

Symptom

  • A Portfolio Item is linked to a PS Project as Related Object (i.e. DFM link).
  • When the PS project is deleted, also the Portfolio Item is deleted.

Environment

  • SAP Portfolio and Project Management
  • SAP Portfolio and Project Management for SAP S/4HANA
  • SAP S/4HANA
  • SAP S/4HANA Cloud Private Edition 

Reproducing the Issue

  1. Create a portfolio item.
  2. Link a PS project via DFM.
  3. Delete the PS project.
  4. Observe that the Portfolio Item is automatically deleted.

Cause

  • Standard behavior: with DFM synchronization from PS to Portfolio Management active, deleting a PS project initiates DFM synchronization that deletes the linked Portfolio Item.
  • This is governed by the the standard mapping in customizing activity "Define Execution Mode for Synchronization of Objects": Grouping "PD Project Definition" with Operation and Synch Behavior "Delete" is mapped to Execution Mode "Syncrhonous".
  • Note that the separate "Maintain Attributes for Synchronization Scenarios" customizing controls attribute value synchronization only and does not govern deletion of objects.

Resolution

Either of the following approaches will ensure that no DFM synchronization is carried out when deleting the PS project, so the project will be deleted but the item will not:

  • Maintain a customer-specific entry in "Define Customer-Specific Execution Mode for Synchronization" with Object Grouping "PD Project Definition", Operation and Synch Behavior "Delete", and set Execution Mode to blank (no execution). This prevents synchronization on deletion, so the Portfolio Item is not deleted.
  • Alternatively, implement BAdI INM_DFM_EXECUTION_MODE to return a blank execution mode for Object Grouping "PD Project Definition" with Operation "Delete".

Note that after doing the above, the object link to the (no longer existing) PS project will not get deleted in the Portfolio Item. This means that the user must manually delete this link (they can do so in item tab "Relationships" > "Related Objects").

Symptom

For development and test systems, you want to simulate the message exchange using the Custom Message Exchange (CME) integration scenario without a connection to the authorities.

With the introduction of the Custom Message Exchange, no IDocs are used for customs communication between SAP GTS and the authorities in the context of NCTS Phase 5 and Phase 6, as well as Passar export. As a result, this message exchange cannot be simulated for development and test using the simulation tool for IDocs (/SAPSLL/IDOC_SIM).

This SAP Note provides the basic technical tools for the simulation of inbound responses of a customs system via the CME interface. The country-specific function must be enhanced for the specific use for a localization. In the initial delivery, messages for NCTS Phase 5 or Phase 6 available in United Kingdom (GB), Netherlands, and Sweden.

Other countries are in work as of February 2026 and will be supplemented accordingly in this Hinwesi.

Other Terms

Simulator, simulation, CME messages, NCTS

Solution

You can use the Simulate CME Messages application to simulate responses from a customs system to test Customizing or perform integration tests.

Calling the Simulation of CME Messages

To simulate responses from a customs system, call transaction /SAPSLL/CME_SIM.

Control Customs Message Exchange (CME) Simulator

The simulation of CME messages can be controlled using the maintenance view /SAPSLL/V_TCMESS.

Symptom

This SAP Note contains minor adjustments to the new pickup process:

  • Adjustment to long text of messages HRPAYDEB2A 006 "File sequence number &1 for data type &2 already processed"  and HRPAYDEB2A 617 "Error when transferring data to SI (&1) - GUID &2: &3&4".
  • Message HRPAYDEB2A 621 is delivered with an explanatory long text for return codes I006 - No data available, whereby not all communication partners could be reached and I005 - Data is available. If you have entered the previously used message HRPAYDEB2A 001 in the view V_T77PAY00_NTMSG to exclude the messages in the Notification Tool, change the entry to message number 621. The message HRPAYDEB2A 001 should not be excluded using the view V_T77PAY00_NTMSG because this is used as a generic message and any relevant error messages would therefore be suppressed.
  • In the report SI: Display Inbound Processes (RP_PAYDE_B2A_SV_LIST), the standard output is adjusted for a better overview so that the relevant message text from processing is visible directly.
  • For tasks in the Notification Tool in the area  SI/B2A, you can navigate to the report SI: Display Inbound Processes (RP_PAYDE_B2A_SV_LIST) for the following messages:
    • HRPAYDEB2A 610: Confirmation of data record rejection (&1)
    • HRPAYDEB2A 611:  Confirmation of File Rejection (Inbound File:&1)&2&3
    • HRPAYDEB2A 612:  Confirmation of parser error (&1)
    • HRPAYDEB2A 613:  Confirmation of Other Errors: DATEIKZ=&1 (&2)
  • If a technical error occurs when files are sent to the DSRV, the system currently issues error message HRPAYDEB2A 455 ("Error when addressing the DSRV Web service (rvBEA)"). This SAP Note provides a more general description of the message text "Error when addressing the DSRV Web service" (HRPAYDEB2A 455). 
  • The message text "File sequence number <DATNR> for data type <DTART> already processed" (HRPAYDEB2A 006) is changed to "File sequence number <DTART>/<BNRKK>/<DATNR> for PA/PTB <WERKS>/<BTRTL> already processed" (HRPAYDEB2A 006).

Other Terms

Solution

Implement the attached correction instructions or import the specified support package. After you import the HR Support Package or implement the correction instructions, you do not have to perform any further follow-on activities.

Symptom

When setting up the Enterprise Portal, it is necessary to mitigate security risks by protecting it from Clickjacking attacks

Environment

  • SAP NetWeaver Application Server Java
  • SAP Enterprise Portal 

Resolution

DISCLAIMER: Clickjacking Protection is entire framework, and all necessary setting need to be put in place in order to protect the system from this type of attack. The complete description of this framework can be found in SAP Note 2319727 - Clickjacking protection framework in SAP NetWeaver AS ABAP and AS Java 

The process bellow are the manual steps described in Note 2169722 - Whitelist based Clickjacking Framing Protection in Enterprise Portal - which is valid from version 7.1 forward

                 1.Login into the NWA application using the URL http://<host>:<port>/nwa

                 2.In the Search field, type “Application Modules” and click on Go. Below screen will be displayed 

                 3. Click on the Application modules 

                 4. In the Module List table -> Name column enter *dispwda* and click on Enter.

                 5. In the Web Module Details table, navigate to the “Web Dynpro Properties” tab and enter “default” in the “Property Sheet Name” column and click on Enter.

                 6. In the Full Details table, enter “ClickJacking” in the “Name” column and click on Enter. Default value will be true.  Also enter “X-FRAME-OPTIONS” in the “Name” column and click on Enter. Default value will be SAMEORIGIN. 

 

                 7. If the user modifies the default value, make sure to save the configuration by clicking on the “Save” button in the “Web Modules” table.

    • For more information about X-FRAME-OPTIONS values, you can read KBA 1781171 - ClickJacking vulnerability in WebDynpro Java

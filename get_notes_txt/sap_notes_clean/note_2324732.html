Symptom

  • A custom form is defined in a model extension for PdPDM.Column and is used to replace the General tab in the property sheet of the metaclass.
  • Editing any column it does not display the customized General tab, but the one defined in the DBMS resource file under Profile\Column\Forms\General.
  • The target DBMS is ORACLE Version 12c.
  • It does not happen if the target DBMS is ORACLE Version 11g.

"Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental."

Environment

  • SAP PowerDesigner (PD) 16.6
  • ORACLE Version 12c

Reproducing the Issue

  1. Select File > New Model to open the New Model dialog.
  2. Select Physical Data Model in the left-hand pane.
  3. Select Physical Diagram in the right-hand pane.
  4. Select ORACLE Version 12c in the list DBMSs.
  5. Click OK.
  6. Select Model > Extensions to open the List of Extensions.
  7. Click the Add a Row tool and enter a name for the new extension file.
  8. Click the Properties tool to open the new extension file in the Resource Editor.
  9. Right-click the Profile category and select Add Metaclasses.
  10. Select Column and click OK to close the Metaclass Selection dialog.
  11. Right-click the Profile\Column category, and select New > Form.
  12. Choose Replace General Tab in the list of types, and customize the form.





  13. Click OK to close the model extension.
  14. Click OK to close the list of extensions.
  15. Click the Table tool from the toolbox to create one table.
  16. Double-click the table to edit it and click the Columns tab.
  17. Click the Insert a Row or Add a Row tool to create a column.
  18. Click the Properties tool to open the property sheet.

Cause

The issue has been identified and logged under CR 799328.

Resolution

CR 799328 has been solved in SAP PowerDesigner 16.6 SP00 PL02.

Keywords

XEM, XDB, ORA12C, ORA11GR1, 799328, CR799328

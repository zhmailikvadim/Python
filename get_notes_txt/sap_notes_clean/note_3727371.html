Symptom

  • Some values in the story are incorrect because the value of Date filter set in Query Designer is fixed to the date when the story was last saved.
  • The filter value set in Query Designer is "Today"

Environment

  • Embedded usage of SAP SuccessFactors Cloud
  • SAP SuccessFactors HXM Suite (SF)

Reproducing the Issue

  1. Login to the SF instance and navigate to Report Center.
  2. Open the problematic story.
  3. Open network logs and refresh data.
  4. Verify the Payload for the getresponse and confirm the value of Date filter is set to a date last time when the story saved.
  5. Edit the data source of the story.
  6. Verify the Time filter set for the Position table is configured to 'Today'.
  7. Click on Finish and save until returning to the story-level.
  8. Open network logs and refresh data again.
  9. Verify the Payload for the getresponse and confirm the value of Date filter is now set to the current date (correct).

Cause

  • Story-level parameter configuration has "Keep Current Variable Selections" enabled, which causes the DATE_PARAMETER_1 (As Of Date) parameter to persist with the date used during the last save instead of dynamically updating at runtime.

Resolution

  1. Open the story in edit mode.
  2. Access the variable prompting settings by clicking {} in the Data toolbar.
  3. Open the "Set Variables for…" dialog.
  4. Uncheck the option "Keep Current Variable Selections" and select "Clear Variables and use Server-Defined Values".
  5. Save the story.

Keywords

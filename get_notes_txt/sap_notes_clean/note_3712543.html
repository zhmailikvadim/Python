Symptom

During the costing of production or Process orders for product identifiers using transaction /IBPP/PRE_CALC, the runtime error ITAB_DUPLICATE_KEY occurs.

Due to a program error, existing entries are not deleted when changes are made with regard to the value basis. This can lead to duplicates and no longer relevant records in table /SAPSLL/PRPIDPS.

Other Terms

IBPP, preference, calculation, KMAT, ITAB_DUPLICATE_KEY, CL_LLS_RAP_CSTMS_PROD_PREF_IDNCP

Solution

Implement the attached correction instructions.

Use the attached correction report "ZSLL_DEL_OBSOLETE_PRPIDPS.txt" to delete duplicates or data records that are no longer relevant in the table /SAPSLL/PRPRIDPS in the system. Then cost the production/process orders specified in the correction report again.

Notes on the correction report:

The report can be executed in simulation mode. If the report is executed in the foreground, the entries to be deleted from the table /SAPSLL/PRPIDPS are then displayed together with the reference document. The entries are checked using the action "Change Selected Rows". If the product identifier is locked, the corresponding entries are not processed. In this case, check whether transaction /SAPSLL/PRPID_1 (Process GRs with IDs) is currently running. The status of the relevant entries is displayed in red. A corresponding message is displayed in the MESSAGE field.

If you do not perform a simulation, you can save the changes after the "Change Selected Rows" action. If the report is executed in the background, the entries are corrected automatically and saved if possible.

Symptom

  • Do we need to take transaction log backups in ASE with HADR?   
  • Can we turn on "trunc log on chkpt" in databases with HADR?

Environment

  • SAP Adaptive Server Enterprise (ASE) 16.0
  • High Availability Disaster Recovery (HADR)

Resolution

Setting the "trunc log on chkpt" database option to 'false' and taking transaction log backups is recommended for all non-temp databases in production systems where data loss has to be minimized. This is no different for databases under HADR. See SAP ASE 16.0 SP03 HADR User's Guide where all databases have "trunc log on chkpt" set to "false".

Keywords

truncate, checkpoint

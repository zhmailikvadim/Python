Symptom

  • You want to configure the PCC jobs as per your needs for simpler identification.
  • SM37 job names for Payroll Control Center (PCC) jobs need to include payroll identifier prefixes at the beginning.

Environment

  • Human Capital Management (HCM)
  • Payroll Control Center (PCC)
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP SuccessFactors Employee Central

Reproducing the Issue

  1. Logon into Payroll Control Center.
  2. Go to 'My Process' application.
  3. Run PCC steps that create background jobs.
  4. Open SM37 and review the generated job names.
  5. Observe that the job names do not include the required payroll identifier prefixes at the beginning.

Cause

Custom logic needs to be written.

Resolution

  1. Go to PYC_CONF_WB t-code.
  2. Choose 'Step Template' from drop-down menu.
  3. Copy the PCC step run-time class name(mentioned under step logic field) for which the job details (name) must be changed.
  4. Copy the standard class CL_PYC_BPC_REPORT_SIMPLE and create your own custom class.
  5. In the custom class, implement method GET_JOB_NAME and write your own custom logic to update the job name.
  6. Go to SE24 t-code.
  7. Pass the copied step run-time class name.
  8. In method FP3_BPC_DET_RT_GET_LIST, adjust the class referenced by ls_dto-rpt_rt which points to a standard class.(CL_PYC_BPC_REPORT_SIMPLE). Change it to your custom class.

     

     

See Also

PCC jobs name can be easily set from user-id in newly released PCC version: 'Manage Payroll Activity'. No need to write custom logic.

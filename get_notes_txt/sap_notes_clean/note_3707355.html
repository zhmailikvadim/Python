Symptom

Transactional orders (such as sales orders, stock transfer orders, or production orders) are successfully integrated into SAP IBP and are visible in the Projected Stock app.

However:

  • In the Planning Relevant column, these orders are marked as “No”
  • Although the quantities appear in Projected Stock App, they are not shown in standard Key Figures in SAP IBP (Excel add-in or PWS UI)

Environment

  • SAP Supply Chain Management (SCM)
  • SAP Integrated Business Planning (SCM-IBP)

Cause

This behavior is expected by design.

Standard SAP IBP key figures only read planning-relevant transactional data.

Orders marked as Non-Planning-Relevant are therefore excluded from standard key figures.

Resolution

If the transactional data integrated from external source should be considered for planning, review the orders directly in the source system and adjust them to be planning relevant before integration.

If the objective is to integrate these orders as non-planning-relevant (that is, for visibility purposes only and not for planning calculations), the following applies:

  • SAP IBP provides specific order data key figures that can read non-planning-relevant order quantities
  • These key figures are not available in the sample planning area.
  • Therefore, it is necessary to create custom key figures based on non-planning-relevant order data.

For more details, refer to SAP Help documentation under Non-Planning-Relevant Quantities.

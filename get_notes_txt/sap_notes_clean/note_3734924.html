Symptom

Field and value mapping is not defined for POSNR in the customizing Cross-Application Components->PLM System Integration->DM Structure Version to SAP Classic BOM->Map DM Structure Version to SAP Classic BOM->Define Inbound Field Mapping. When the structure version as classic BoM with component and substitute is updated, POSNR is taken from the component's field posNumber. It is incorrect since POSNR should not be taken from posNumber if field and value is not defined for it.

Other Terms

PLMSI, SV, Structure Version, Position Number, POSNR

Solution

Implement this SAP Note.

Environment

  • SAP NetWeaver ABAP based system
  • ABAP Platform based system
  • Secure Internal Server Communication is active ("system/secure_communication = ON")

Cause

The "global" folder ("/usr/sap/SID/SYS/global") is not shared among all servers of the system.

Resolution

The "global" folder ("/usr/sap/SID/SYS/global") must be shared among all servers of the system.

Once this has been fixed, follow these steps on the affected instance:

  1. Confirm that the instance is completely stopped;
  2. Stop the sapstartsrv process with the command "sapcontrol -nr XX -function StopService";
  3. Delete the file "sap_system_pki_instance.pse" located on the $(DIR_INSTANCE)/sec folder;
  4. Start the sapstartsrv process again with the command "sapcontrol -nr XX -function StartService SID";
  5. Start the instance as usual.

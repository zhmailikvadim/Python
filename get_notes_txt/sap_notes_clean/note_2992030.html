Symptom

  • When creating a business partner using transaction BP default field value should appear
  • BADI BUPA_SET_DEFAULT is not called for transaction BP

Environment

  • Central Business Partner application
  • SAP ERP 6.0
  • SAP ERP Central Component 5.0
  • SAP ERP Central Component 6.0
  • SAP enhancement package for SAP ERP
  • SAP Supplier Relationship Management for SAP ERP
  • SAP S/4HANA
  • SAP S/4HANA Private Cloud

Reproducing the Issue

  1. Run transaction BP
  2. Create a new business partner
  3. Default field values are not filled
  4. BADI BUPA_SET_DEFAULT was not called

Cause

BADI BUPA_SET_DEFAULT is used in API scenario and is not called in dialog mode

Resolution

Please use events like ISDAT or ISSTA to provide BP with default values.

  1. Check whether structure BUSDEFAULT provides the needed field

By SAP standard the fields that can be defaulted in the BP screen is limited to the fields provided in the DDIC structure BUSDEFAULT.
Structure BUSDEFAULT can be edited to add a new field if it does not exist. Please keep in mind that this is an enhancement, enhancement category 'Can be enhanced (character-type)':

  1. Create an own function module in customer name space
  2. Within the function module call BUP_BUPA_FIELDVALUES_SET to fill the default value
  3. Save and activate the function module
  4. Run transaction BUS7
  5. Open event ISDAT/ISSTA
  6. Create a new entry to register the function module for application BUP
  7. Save and make sure 'Call' is set to X

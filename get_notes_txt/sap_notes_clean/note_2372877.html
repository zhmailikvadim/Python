Symptom

You are using oDATA Service /sttpec/if_whs_if_od_services~get_parent_object to get data for a sGTIN and its parent. When you use decode_mode > 2 also the lot will be returned. But only for the child not for the parent. When you implement this note also for the parent objects the lot number will be returned. 

Other Terms

GET_PARENT_OBJECT; LOTNO; Lot number; Lot no; Batch number; Batch no;

Reason and Prerequisites

The reason is a coding error.

This note only applies to Advanced Track & Trace for Pharmaceuticals.

Solution

Implement the assigned correction instructions.

Symptom

  • When running report formats 1001, 1005, 1006, or 1007, you observe that values are being multiplied or that multiple Business Partners (BPs) are being assigned to the same document. This can lead to significantly inflated and inaccurate reporting totals.

Environment

  • SAP S/4HANA
  • SAP S/4HANA Cloud Private Edition
  • SAP Document and Reporting Compliance service
  • DRC Colombia
  • Statutory Reporting Framework

Cause

This issue is caused by a combination of two factors:

  1. Incorrect Use of the XREF1 Field: The XREF1 field in the document line item is being populated with a Tax Number for document types where it should not be. The system is designed to use the XREF1 field for the Tax Number only for specific cases like Travel expenses, Cash journal (petty cash), and Fiscal reclassifications, as documented in SAP Note 3128041. When this field is populated for other documents, such as invoices (e.g., document types 'KR', 'DR', 'RE'), the system assigns that document to every BP that has the same Tax Number, causing the multiplication effect.

  2. Incorrect Business Partner Data Setup: The same Tax Number has been assigned to multiple, different Business Partners in the master data. The system does not support the use of the same Tax Number for different BPs.

Resolution

To resolve this issue and prevent it from recurring, follow these steps:

  1. Correct Existing Documents: Identify the affected documents (e.g., document types 'KR', 'DR', 'RE') where the XREF1 field has been incorrectly populated with a Tax Number. You must clear the XREF1 field for these documents.

  2. Correct Business Partner Master Data:

    • Review your Business Partner data to identify all instances where the same Tax Number is assigned to more than one BP.
    • Consolidate this data to ensure each Tax Number is assigned to only one unique Business Partner.
    • Best Practice: If a single partner operates in multiple scenarios (e.g., as both a customer and a vendor), do not create a new BP for each scenario. Instead, create one BP and assign multiple roles to it.

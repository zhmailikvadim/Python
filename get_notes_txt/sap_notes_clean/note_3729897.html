Symptom

When the SQL statements executed by a Fiori application (that is built using RAP) and an error occurred, then the SQL statements are needed for troubleshooting.

"Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental."

Environment

  • SAP S/4HANA 2021 FPS00 or newer
  • SAP S/4HANA Cloud Public Edition
  • SAP BTP ABAP Environment

Resolution

  1. Open the ABAP Cross Trace view in ADT (ABAP Development Tools) using the menu option: Window -> Show View -> Other... -> ABAP Cross Trace.
  2. Right-click on the system where the trace is to be enabled in the ABAP Cross Trace view. (By default the view is opened at the bottom in ADT) Then select Create.
  3. Enter the username that is to be traced.
  4. Select the RAP Runtime Engine (SADL Framework) options, to get SQL statement data. 
  5. Select Activate trace and click OK.
  6. Carry out the actions in the Fiori application that is to be traced.
  7. Right-click the trace configuration in ADT in the ABAP Cross Trace view, and select Deactivate Tracing to stop the trace.
  8. To display the created trace switch to the Trace Results tab in the ABAP Cross Trace view and double-click on the trace that was created.
  9. Navigate to the SQL statements displayed in the upper section. The SQL statement or the result of it will be displayed in the bottom section.

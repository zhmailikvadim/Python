Symptom

During BC Set activation, the following error message is displayed: "Package GLO-EDO-EU-AR-10 does not exist".

Environment

  • SAP ERP
  • SAP S/4HANA
  • SAP Document Reporting and Compliance service

Reproducing the Issue

  1. Attempt to activate the BC Set as mentioned, for example, in SAP Note 3605436.
  2. Observe the error message: "Package GLO-EDO-EU-AR-10 does not exist".

Cause

Missing pre-implementation steps execution from SAP Note 3364293.

Resolution

  1. Refer to SAP Note 3364293 for the detailed pre-implementation steps.
  2. Perform the manual pre-implementation steps as outlined in the note:

a. Create the package 'GLO-EDO-EU-AR-10' using transaction SE80 as follows:

  • Package: GLO-EDO-EU-AR-10
  • Short Description: DRC - Elec Doc Peppol - Peppol Application Response 1.0
  • Application Component: CA-GTF-CSC-EDO-PAP (if not available in your system, use CA-GTF-CSC)
  • Software Component: SAP_APPL
  • Save the package.

b. Open Package APPL.

  • Go to the tab page 'Subpackages'.
  • Choose the push-button 'Add existing package'.
  • Add the package GLO-EDO-EU-AR-10.
  • Save the changes.

c. Go to package GLO-EDO-EU-AR-10 again and open the 'Properties' tab page.

  • Check if the superpackage APPL is assigned there.

After completing the manual pre-implementation steps, proceed to implement SAP Note 3364293 as instructed in the note in case it was not implemented yet.

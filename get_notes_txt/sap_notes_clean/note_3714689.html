Symptom

  • Connection to the Repository via the Proxy fails when PD client 16.7.8.0 runs on Windows 11 and the PD Proxy runs on Windows Server 2012:
       Proxy version incompatible with instance version.
    then
       The repository cannot be installed using a proxy connection.
       Use a direct connection.
    and finally
       The connection failed because the repository has not been installed on this database.
  • It works fine when using a Direct connection.
  • There is no issue when the Proxy is installed on Windows 11.

Environment

SAP PowerDesigner (PD) 16.7 SP08

Reproducing the Issue

  1. Install PD Proxy on Windows Server 2012.
  2. Launch PD Desktop from Windows 11.
  3. Create a Proxy repository definition and attempt connection.

Cause

The issue has been identified and logged under CR 831805. Its description reads, "Error: 'Proxy version incompatible with instance version' occurs when connecting via the Proxy on Win Server 2012".

Resolution

There is currently no target SAP PD 16.x Service Pack scheduled for this issue SAP CR 831805. It is being assigned to our Product Group.

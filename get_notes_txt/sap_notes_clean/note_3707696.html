Symptom

Setting up communication between ECC/S4 and FSM for a new company in the proaxia connector requires some adjustments in customizing. If the new company is maintained in transaction /PACG/ECM_CACC or /PACG/ECM_COMPL and it is not extended in both Message Broker config files, the communication with these Message Brokers cannot be restored, even if errors in customizing are removed or corrected.

Communication statuses with both Message Brokers in both tables /PACG/ECM_AMBS and /PACG/ECM_AMBSH are set to ‘N‘. The logic used in program /PACG/ECM_HEARTBEAT does not account for the scenario when both Message Brokers are unavailable. The attached correction resolves the problem and improves the way connection verification with Message Brokers is performed.

Other Terms

FSM, E4C, Connector, Message Broker, Failover, /PACG/ECM_CACC, /PACG/ECM_COMPL, /PACG/ECM_HEARTBEAT

Reason and Prerequisites

Logic gap in /PACG/ECM_HEARTBEAT program.

Solution

Please, implement this correction instruction.

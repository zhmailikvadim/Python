Symptom

There are different columns in transaction FEBAN and Fiori application Reprocess Bank Statement Items (ID: F1520) regarding posting status. This article explains the relationship.

Environment

Financial Accounting (FI)
SAP Fiori for SAP ERP
SAP S/4HANA
SAP Fiori for SAP S/4HANA Finance
SAP S/4HANA Cloud Private Edition

Reproducing the Issue

Open transaction FEBAN and application Reprocess Bank Statement Items and compare the output.

Cause

Different structure of columns.

Resolution

In transaction FEBAN the first column is Status of Posting in Bank Accounting which displays the following statuses: Red (not posted), Green (posted) and Yellow (posted on account). The second column is Status of Posting in Subledger which displays the following statuses: Red (not posted), Green (posted), Yellow (posted on account) and Gray (no posting in the subledger).

In application Reprocess Bank Statement Items these columns are merged to Posting Status (Bank Statement Item Life Cycle Status) which displays the following statuses: Green (Completed or Set to done), Yellow (Auto-posted on account), Red (Posting not started or Not completed). The status in the app is green if both statuses are green or gray in transaction FEBAN. 

In the app, the second column is Processing Status, this column displays the information of FEBAN's 'In Process' column.

Symptom

The City and State fields are blanked out in the MDG UI when the Tax Jurisdiction field is derived from Vertex.

Environment

SAP Master Data Governance, Central Governance.

Reproducing the Issue

  1. Navigate to the MDG UI, select the "Manage Customer Governance" tile to create a new customer.
  2. Enter the address details.
  3. Press "Enter".
  4. Observe that the Tax Jurisdiction code is derived and the City and State fields are cleared.

      Cause

      The system behavior is correct and aligns with the SAP MDG standard.  SAP MDG standard takes over all values as returned by Vertex.  This approach has been implemented in the SAP MDG standard code to prevent an endless number of calls to Vertex due to deviating data in MDG versus Vertex.

      Resolution

      1. Confirm that SAP MDG derives data based on the results provided by Vertex or any other data provider.
      2. Verify that the notes identified (2918719 and 3509258) address different issues and are already included in the current release.
      3. Understand that the system behavior is as intended by the SAP MDG standard.
      4. Discuss specific data issues with Vertex, as SAP has no influence on the data provided by Vertex.
      5. If customization is required to retain certain data by ignoring results returned from Vertex, implement custom code. Note that every UI round-trip may identify data differences and repeatedly retrieve details from Vertex.
      6. Refer to SAP Note 83020 or SAP Note 2706322 for consulting-related inquiries.

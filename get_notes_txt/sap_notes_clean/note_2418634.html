Symptom

Your system has a secondary SAP HANA node continuously supplied with all data posted on the primary SAP HANA node using a technique called log replay. You want to direct some of the workload related to analytical transactions to the secondary HANA system, in order to relieve pressure on the CPUs of the primary SAP HANA node.

Other Terms

Performance active/active

Reason and Prerequisites

Your system has a secondary SAP HANA node, usually installed as part of the high-availability strategy. Technically you are using SAP HANA system replication using the logreplay_readaccess mode (http://help.sap.com/hana/SAP_HANA_Administration_Guide_en.pdf).

Note that SAP HANA 2 is required to use the described feature.

Solution

As of SAP HANA 2, there is an option to free up CPU resources by using a secondary SAP HANA node. SAP HANA allows transactions to read data not only from the primary SAP HANA node, but also from a secondary SAP HANA node. The secondary SAP HANA node is continuously supplied with all data posted on the primary SAP HANA node using a technique called log replay. The data on the secondary SAP HANA node has a delay of typically a few seconds compared to the primary SAP HANA node, depending on the current workload and other factors.

For analytical transactions that only require read access, reading from the secondary SAP HANA node is a good alternative. It leaves more free CPU resources on the primary SAP HANA node to ensure smooth transactional processing.

With SAP S/4HANA 1709 following transactions and programs have been enabled for read access from the secondary SAP HANA node:

  • Reports RFBILA00, RFSSLD00, RFSUSA00, RFSUSA00_NACC
  • Transaction FAGLB03
  • FAGLFLEXT-based drilldown reports (Transaction FGI0, Report Type 002)

Note that this feature merely enables the read access from the secondary SAP HANA node. The database finally decides, if the query will be executed on the primary or on the secondary SAP HANA node. Routing of queries to the secondary SAP HANA node can be switched off via SAP HANA ini parameters.

For information on analytical apps that are enabled for this option, see SAP Note 2405182.

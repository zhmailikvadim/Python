Symptom

  • Error after running depreciation posting with the “Schedule Asset Accounting jobs” app.
  • OData error message: “The request URI contains an invalid key predicate.”
  • Exception in gateway logs: 
    <Exceptions>
    <_-IWCOR_-CX_DS_URI_NOT_MATCHING>
    <Text>The request URI contains an invalid key predicate.</Text>
    <Source>
    <Program>/IWCOR/CL_DS_URI==============CP</Program>
    <Include>/IWCOR/CL_DS_URI==============CM008</Include>
    <Line>106</Line>
    </Source>
    </_-IWCOR_-CX_DS_URI_NOT_MATCHING>
    </Exceptions> 

Environment

SAP S/4HANA Cloud Private Edition 2023
SAP S/4HANA Cloud Private Edition 2025

Reproducing the Issue

  1. Start depreciation posting using the “Schedule Asset Accounting jobs” app.
  2. Navigate to job run details or parameters for the completed run.
  3. Observe that the app fails and shows “The request URI contains an invalid key predicate.”
  4. Check /IWFND/ERROR_LOG and find exception /IWCOR/CX_DS_URI_NOT_MATCHING for service APJ_JOB_MANAGEMENT_SRV.

Cause

The gateway service metadata cache for APJ_JOB_MANAGEMENT_SRV was not rebuilt after an upgrade, leading to a mismatch in expected key predicates (for example, missing JobRunStep) and resulting in “invalid key predicate” errors.

Resolution

  1. In the backend system, open transaction /IWFND/GW_CLIENT.
  2. Set the Request URI to: /sap/opu/odata/SAP/APJ_JOB_MANAGEMENT_SRV/$metadata and execute.
  3. In the menu, choose: Metadata -> Rebuild Cache -> Current Service.
  4. Retest the “Schedule Asset Accounting jobs” app and verify that the error no longer occurs.

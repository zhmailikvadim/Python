Symptom

  • When generating the SPED ICMS or SPED Contribuições fiscal reports, the system creates multiple lines for the same material code in Register 0200. This behavior occurs for materials that use the Split Valuation functionality, resulting in more than one Item Type (TIPO_ITEM) being associated with the same material (e.g., "Usage and Consumption" and "Fixed Asset"). 
  • This duplication of records contradicts the rules of the SPED practical guide, which requires a single line per item in Register 0200.

Environment

  • SAP ERP
  • SAP S/4HANA
  • SAP S/4HANA Cloud Private Edition
  • SAP Tax Declaration Framework for Brazil
  • SAP Document and Reporting Compliance service
  • SPED EFD ICMS/IPI
  • SPED EFD Contribuições

Cause

This is the standard behavior in the SAP Tax Declaration Framework (TDF) when a material is configured with Split Valuation. The system identifies that the material has multiple usage types (TIPO_ITEM) and, by default, generates a line for each type in Register 0200, reflecting all possible uses of the material.

Resolution

There are two approaches to resolve this issue, depending on the business requirement.

Option 1: Standard Configuration
For a static definition, you can configure which single Item Type should be considered the priority for tax reporting.

  1. Access the transaction /TMF/MAT_CONFI.
  2. In this transaction, define the Item Type (TIPO_ITEM) that should be reported for the material. This choice should be based on the primary use of the material as defined by the company’s fiscal team.
  3. After saving the configuration, generate the SPED files again. The system will now report only the Item Type defined in this transaction, eliminating the duplicate lines in Register 0200.

Option 2: Custom Solution for Dynamic Scenarios
If the business requirement is to dynamically determine the Item Type based on specific logic (e.g., selecting the type with the highest transaction volume in a given month), a custom solution can be developed.

  • This custom solution should be designed to directly populate the complementary table /TMF/D_MAT_CONF.
  • Your custom program will apply your business rules to determine the correct TIPO_ITEM for each material for the reporting period and then insert that data into the /TMF/D_MAT_CONF table. This provides a flexible way to meet complex reporting needs that go beyond static configuration.

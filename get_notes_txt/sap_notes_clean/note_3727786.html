Symptom

  • Additional rows are added at the beginning of exported Excel files (xlsx format) in Web Intelligence reports.
  • Business macros fail due to the presence of additional rows in the exported files.
  • In BI 4.2 SP9 Patch 1, no additional rows were present. After migrating reports to BI 4.3 SP5 Patch3, additional rows are inserted in the Excel output.

Environment

  • SAP BusinessObjects Business Intelligence platform 4.3 SP05 Patch 3
  • Microsoft® Excel® for Microsoft 365 MSO (Version 2511 Build 16.0.19426.20260) 64-bit

Reproducing the Issue

  1. Open a Web Intelligence report in BI 4.3 SP5 Patch3.
  2. Export the report to Excel (xlsx format) manually or schedule an export.
  3. Observe that additional rows are added at the beginning of the exported file.

Cause

  • In BI 4.3 SP5 Patch3, the placement of text objects in the report header leads to the addition of extra rows in the exported Excel file to accommodate the alignment of these objects.

Resolution

  1. Stop the BI server.
  2. Open the Windows registry.
  3. Navigate to the key: `HKEY_LOCAL_MACHINE\SOFTWARE\SAP BusinessObjects\Suite XI 4.0\default\WebIntelligence\Repeng\`.
  4. Select the string value with the value name `xlsAlignCompat`.
  5. Change the value data to "yes".
  6. Save the registry changes.
  7. Restart the BI server.
  8. Clear the browser cache.
  9. If the environment is clustered, repeat the above registry key changes on all nodes.

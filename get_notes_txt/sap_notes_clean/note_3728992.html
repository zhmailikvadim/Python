Symptom

During an upgrade to S/4 HANA 2023, S/4 HANA 2025, BW HANA 2023 or BW HANA 2025, the SUM Phase HANA_DEPLOY_UPG deletes the customer HANA Repo objects which exist only in HANA DB but not in HTA(ABAP). This deletion is unexpected as no customer objects should be deleted during an upgrade.

The following error messages, relevant to the issue, can be found in the log files SAPCCC*_REPO.[SID]:

"A2 ESCTS_HOT 501 Start of deployment to SAP HANA repository: ......
A2 ESCTS_HOT 507 " "
A4 ESCTS_HOT 595XThe following objects exist in HANA only (not in HTA) and will be deleted "

Other Terms

HTA for HANA Repository, HANA Transport for ABAP, HTA synchronize views, SUM upgrade, HANA_DEPLOY_UPG, RDDHANADEPLOYMENT

Reason and Prerequisites

Program error in the HTA HANA deployment report.

Solution

Prevent Deletion of Objects

The correction instructions of this note will prevent deletion of these objects during an upgrade.

To prevent the unexpected deletion of customer HANA repository objects during an upgrade, implement the correction contained in this SAP Note 3728992 directly before the phase HANA_DEPLOY_UPG starts. For information about how to implement an SAP Note during the update procedure, see SAP Note 2207944.

Recovery of Lost Objects

If the upgrade has already been performed without the correction from this note, and customer HANA Repository objects have been deleted, the following recovery procedure can be used.

The deleted objects must still exist in another location — either in a database backup or in another system in the landscape.

Identify a source system:

- Restore a HANA database backup taken before the upgrade into a temporary or separate system, or
- Use an existing system in the landscape where the objects are still present (for example, a development or quality system).

Recover the objects:

1. In the source system, open transaction SCTS_HTA.
2. Synchronize the missing objects from the HANA Repository into ABAP (direction: HANA → ABAP).
3. Create a transport request containing the synchronized objects.
4. Import the transport request into the target (upgraded) system. The HTA import will deploy the previously missing objects to the HANA Repository on the target database.

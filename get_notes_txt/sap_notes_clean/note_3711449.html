Symptom

When a candidate changes the preferred language in their career site profile, the language of previously submitted applications does not update.

Example:
A candidate completes an application in Spanish. Later, the candidate changes the career site language to English. The career site UI and navigation update to English as expected. However, when the candidate views the previously submitted application, the application data remains in Spanish.

Environment

SAP SuccessFactors Recruiting Management 

Resolution

After review and discussions with Product Management, this behavior is confirmed to be expected. The Reimagined Candidate Experience (CX) introduces an enhanced and redesigned application flow that differs from the standard RMK career site, including how language preferences are handled.

Below is a comparison of key language‑related behaviors between CX Apply and the standard RMK site:

  1. Retention of Selected Language During Application
    In CX Apply, when a candidate selects a language on the career site and clicks Apply, the system retains this selected language throughout the entire application flow. This provides a consistent experience and prevents unexpected mid‑process language switches.

  2. Behavior for “My Profile” and “My Applications”
    If the candidate changes the site language and then navigates to My Profile, My Applications, or initiates a new application, CX behaves according to the documented SuccessFactors Recruiting guidelines published in SAP Help Portal.

  3. Viewing Previously Submitted Applications (RMK)
    Applications are always stored and displayed in the original submission language. Even if the candidate later updates their preferred or display language, previously submitted applications will continue to appear in the language in which they were originally completed.

  4. Differences Between CX Apply and Standard RMK Apply
    Language handling in CX Apply is different by design. CX introduces an improved interface, refined usability, and enhanced locale handling to create a more guided and consistent candidate experience. These enhancements result in certain behaviors that differ from the legacy RMK Mobile Apply process.,

Important Note for Customers: The preferred locale behavior in CX Apply is intentionally different from the traditional RMK career site. This design choice has been communicated previously and is aligned with the product direction to deliver a more intuitive and stable candidate experience.

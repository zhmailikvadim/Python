Symptom

  • Standard tile on Count is not presenting data on drill down. e.g. Development Goal Count, Objective Distribution.
  • No data available on tile drill down.

Other tiles that are not count are working correctly:

"Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental."

Environment

  • SAP SuccessFactors HCM Suite
  • Dashboards and Tiles

Cause

This behavior is caused because the user does not have permissions to access the username of the employees that are on the tile. The tile will have to have access to the to the username information so it can calculate the count of the employees. This behavior will happen only when the three switches below are enabled on provisioning.

Row Level Permission (Implicit based on Report Scope)
Field Level Permission (Must be enabled via Provisioning)
Cell Level Permission (Must be enabled via Provisioning)

See KBA 2434120 - Ad Hoc Report Row/Field/Cell Permissions for further information on the switches.

Resolution

To correct this behavior it is required to give permissions for the users running the tile to access the Username information inside "Employee Data".

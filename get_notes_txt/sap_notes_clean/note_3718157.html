Symptom

"Dynamic Selection" is a reusable service in the SAP Basis layer, which includes a self-contained UI. This UI includes of a tree control that allows the user to choose a subset of predefined fields to be used for selection criteria, that can be entered in another part of the UI.

As this UI is available for SAP end users, it has to meet legal accessibility requirements.

So far, the user-chosen fields have been highlighted with a green color. However, this does not meet the accessibility requirements:

  • There must be a readable information about what the used green color means, which is also available for screen readers e.g. JAWS
  • Color-blind persons must be able to work with the UI, to there must be additional information or characteristics
  • In classic SAP GUI themes, the green background color is overlayed by the orange color of the selection.

Reason and Prerequisites

Issue in ABAP code. A correction is required.

Solution

Apply the provided correction or upgrade to the support package level, that is assigned to this note.

The correction effects the following improvements:

  • A new icon indicates per line/field, whether the field is currently chosen or not
  • The icon has a tooltip ("Field displayed", "Field not displayed") respectively, that it may be accessible via mouse hover and for screen readers when focussing the icon
  • The used icons have different shapes, so that color-blind persons are able to distinguish them 
  • The green backgound color is still used / unchanged

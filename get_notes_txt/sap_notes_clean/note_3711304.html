Symptom

Note 2912961 has delivered a new "Delete gaps" functionality in report /LIME/BACKGROUND_DELETE_EXEC. However the runtime of this functionality is too long and if the report is not scheduled in a warehouse downtime (e.g. in case of 24/7 operation warehouses) it can cause update terminations in the productive system.

Other Terms

/LIME/BACKGROUND_DELETE_EXEC

Reason and Prerequisites

performance

Solution

Implement the code changes.

The functionality is rewritten to be more friendly with ongoing warehouse operations, to avoid update terminations. For the "Delete gaps" functionality to work optimally the report should be scheduled to run regularly (e.g. once in month).

It is still highly recommended to run the /LIME/BACKGROUND_DELETE_EXEC report outside peak warehouse activities.

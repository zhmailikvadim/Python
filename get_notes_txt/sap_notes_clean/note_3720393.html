Symptom

  • From SAP IQ 15.x onward, tables can be range-partitioned so that data is stored in specified partitions.
  • However, range partitioning in SAP IQ has several limitations.

Environment

  • SAP IQ all version

Cause

  • Please note that in SAP IQ range partitioning, any additional indexes other than the default Fast Projection (FP) index are created as global indexes.
  • Range partitioning is described in the following document; however, I have added some additional explanatory notes in the Resolution section.
  • SAP IQ Administration: Database - Range Partitions

Resolution

  1. Only a Single Partition Key Is Allowed
    Range partitioning supports only one column as the partition key.
    You cannot define range partitions using multiple columns (for example, date + region).

  2. Maximum Number of Partitions (Up to 1,024)
    The maximum number of range partitions per table is 1,024.

  3. Partitions Are Defined by Upper Bounds Only

    Each partition is defined using VALUES <= (upper_bound).
    Lower bounds are implicitly determined by the previous partition’s upper bound.

    - Be careful with boundary definitions, especially:

    End-of-month dates
    Timestamp columns (inclusive vs. exclusive ranges)

  4. No Benefit Without Partition Key Predicates

    If a query does not include a condition on the partition key, all partitions are scanned.

    - Ensure that application SQL and reporting queries always include predicates on the partition key (for example, date filters).

  5. Non-FP Indexes Are Global

    Fast Projection (FP) indexes are partition-local.
    All non-FP indexes are global and apply to the entire table.

    - Dropping or adding partitions may still incur maintenance overhead for global indexes.

  6. Adding Partitions to Existing Tables Requires Care

    Range partitions can be added later using ALTER TABLE.

    - However, consider the impact on:

    Data redistribution
    Dbspace utilization
    Backup and restore strategies

  7. Dbspace Design Can Increase Operational Complexity

    Each range partition can be placed in a separate dbspace.

    - Advantages

    Data tiering (hot vs. cold data)
    Partition-level maintenance

    - Disadvantages

    Increased number of dbspaces
    Higher operational and storage management complexity

  8. Design with ILM (Information Lifecycle Management) in Mind

    Range partitioning is well suited for:

    Faster backup and restore
    Dropping or archiving old data

    - Design partitions assuming regular DROP PARTITION operations as part of lifecycle management.

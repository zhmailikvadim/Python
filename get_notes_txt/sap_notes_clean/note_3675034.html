Symptom

In the Monitor Receipts Fiori application of Production Planning and Detailed Scheduling (PP/DS), the display of alert information for receipts depends on the type of the corresponding I/O Node. Certain I/O Node types are not relevant for alerts icons determination, and therefore alerts icons are not read for them.

Other Terms

PP/DS, Monitor Receipts Fiori app, Receipts, Requirements, I/O Node Type, Alerts, Alert Icons, /SAPAPO/RRP4, /SAPAPO/LRRP_REQMTSF01, Alert Column Group, Alert Cache Initiator

Reason and Prerequisites

Previously, the logic determining whether alerts icons should be read for a receipt was implemented within the include /SAPAPO/LRRP_REQMTSF01, specifically in the routine GET_DISPLAY_DATA.

This logic was not encapsulated, and therefore, it could not be easily reused in the Monitor Receipts application. As a result, there could be differences in alert behavior between standard PP/DS transactions and Fiori application.

Solution

To enable consistent behavior, the logic responsible for determination whether an I/O Node type is relevant for alerts icons reading has been refactored into a helper class.

This class encapsulates the necessary check for reusing the same determination logic in the Monitor Receipts Fiori application.

Please note: the solution is based on adjustments in SAP Note 3675003, which includes changes to refactor the standard code in /SAPAPO/LRRP_REQMTSF01 (routine GET_DISPLAY_DATA) and the implementation of a new helper class for reuse. As a result, the Monitor Receipts application uses the same check for determination of alerts icons reading as the standard PP/DS transaction /SAPAPO/RRP4.

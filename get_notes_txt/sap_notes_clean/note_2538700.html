This collective SAP Note provides you an overview on the existing SAP Notes/Knowledge base articles (KBA) related to the SAP S/4HANA Migration Cockpit content build for SAP S/4HANA Cloud.

Note
  • This SAP Note covers only topics for SAP S/4HANA Migration Cockpit - File/Staging approach.
  • This SAP Note covers topics only for SAP S/4HANA Cloud based on SAPSCORE.
    • If you are not sure what your SAP S/4HANA Cloud is based on, download a migration object template and check the product version that is within the header information below the migration object name. If you see there "SAPSCORE" followed by a three digit number, then this SAP Note covers your topics. If you see "S4CORE"  followed by a three digit number, then check SAP Note 2537549 - Collective SAP Note and FAQ for SAP S/4HANA Migration cockpit - File/Staging (on premise / S4CORE).

Be aware that this collective SAP Note is updated frequently. So always please check online for updates.

Reason and Prerequisites

You are using the SAP S/4HANA Migration cockpit in SAP S/4HANA Cloud.

This Collective SAP Note contains the most relevant SAP Notes and Knowledge Base Articles (KBA) links to those topics. It might be the other SAP Notes or KBAs exist that solves your problems. Please always search for SAP Notes or KBAs on the following components before you create a ticket:

  • CA-LT-MC: Errors and issues belonging to the SAP S/4HANA Migration Cockpit and the handling of the tool itself
  • CA-GTF-MIG: Issues belonging to the SAP S/4HANA Cloud Data Migration content (template, value mapping)
  • SV-RDS-EIM: Issues related to the rapid data migration for SAP S/4HANA

For errors during simulation or data transfer where the Message ID is not starting w/ "DMC_", please search for SAP Notes/KBAs on other components.

Error messages during simulation or data transfer are coming from the API used for the data transfer. There might be SAP Notes/KBAs existing that describes when this error will be thrown by the API.

Note

Please, always check first the SAP Notes and KBAs before creating a ticket.

This document might mention releases of SAP software that are not yet available. SAP reserves the right to change the scope and the extent of the software to be delivered. Therefore, the comparison with releases not yet delivered must be considered bearing this reservation in mind.

    4 I have created ticket on CA-GTF-MIG why was it forwarded to another component?
    • Errors and issues related to the delivered SAP S/4HANA Cloud data migration content for the migration cockpit are handled either by CA-GTF-MIG (for example wrong mapping) or another component that build the data migration object.
    • Errors related to the migration cockpit tool are handled by CA-LT-MC (for example, Job issues, short dumps, technical or tool handling issues, and so on)
    • The SAP S/4HANA Cloud data migration content uses SAP standard function modules (migration API) to migrate the data. In case the error/issue is related to the used migration API the ticket will be forwarded to the migration API owning component.

    12 Additional information on ISO code conversion
    • You always have to add a valid ISO code in the target value field of the conversion value
    • A valid ISO code is an ISO code that is maintained in the system and fulfills the following criteria:
      1. It is assigned to at least one internal SAP unit of measure
      2. If it is assigned to more than one internal SAP unit of measure, one of this internal SAP unit of measures must have set the flag PRIMARY

Symptom

Restriction in GB/TB size of data that can be copied during the refresh and whether large production database size could block or impact the refresh execution. 

Environment

SAP S/4HANA Cloud Public Edition

Resolution

In SAP S/4HANA Public Cloud Edition – Test Data Refresh (TDR) there is no published hard limit (like X GB or X TB) on the volume of data transferred. The refresh is designed as a full copy of production master & transactional data to the target test system, and the actual transferable volume is governed more by system sizing, landscape, and service scheduling rather than a fixed numeric cap. Existing data in the target tenant is deleted and replaced during refresh.

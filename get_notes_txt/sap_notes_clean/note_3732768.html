Symptom

User is not able to delete the product cost collector in T-code KKF6N due to error message KO451 "There are still unsettled variances for ORD &"

Environment

  • Controlling (CO)
  • SAP R/3
  • SAP R/3 Enterprise 
  • SAP ERP
  • SAP ERP Central Component
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA

Reproducing the Issue

1. Open T-code KKF6N
2. Input Material and Plant
3. Go to menu options 'Edit' > Deletion Flag > Set
4. You can view the message "Mark for deletion cannot be executed". 
5. Select "Display messages" in the pop-up to see error message KO451.

Cause

The cost collector cannot be deleted because it is still relevant for variance settlement to CO-PA, as controlled by the settlement profile settings.

Resolution

Temporarily deactivate the flag "Variances to Costing-Based PA" in the relevant settlement profile (in transaction OKO7). In the long text of error message KO451 you can find the relevant settlement profile. Then you should be able to set the deletion flag in transaction KKF6N.
Afterwards you should set the flag "Variances to Costing-Based PA" in transaction OKO7 again. 
Note: Removing the check mark & deleting cost collector will have no side effects or issues in posting or variance.

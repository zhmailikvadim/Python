Symptom

When trying to access/open custom form related tile, error  "Sorry, there seems to be a system issue. Please try again later. Error code: FRM-0040" is thrown.
This can happen on any custom form related resource (Line item form, Request Form etc.)

Environment

Production/Test, Guided Buying context

Reproducing the Issue

Link tile to Custom form template/design.
Remove Custom form design in Forms Builder.
Try to access linked tile/resource (e.g. Line item form)

Cause

Error thrown (FRM-0040), is tied to Custom Form error - UnableToPersistFormDataException. This happens when metadata for the form is accessed/pulled from Forms microservice and it is not available or there is inconsistency in available details.
Usually this can happen, when Custom Form (design) is created, published and then linked to Guided Buying tile. Afterwards, linked design (Form) is deleted.
This causes GB tile to be linked to incorrect Form ID or Form ID that is actually not Published and therefore not available.

Resolution

In order to fix this, tile needs to be removed and re-added with correct, currently active Form ID reference.
If form re-added is not working, Custom Form should be re-published in the Forms Builder (ideally using REPLACE option) to trigger update on the Form data.

Keywords

FRM-0040, UnableToPersistFormDataException, system issue, FRM-0040, Custom Form error, RFQ Form error

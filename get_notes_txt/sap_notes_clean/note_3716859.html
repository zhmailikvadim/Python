Symptom

You are executing test data refresh pre-checks in the production system, however the job gets canceled with the following error message:

"Canceled, create a new pre-check" 

Environment

SAP S/4HANA Cloud Public Edition

Reproducing the Issue

  1. Open the Check Your System Readiness app.
  2. Click on the New button to create a new pre-check.

After the execution, the error is displayed in the result status.

Cause

The issue occurs when Quality and Production systems are on different release versions due to upgrade cycle parity.
If one system is already upgraded and the other is still on the previous release, the readiness check may fail because:

  • Content versions are not aligned

  • Technical checks detect incompatibility

  • TDR prerequisites are not met

The readiness check requires both systems to be on the same release version to ensure data consistency and compatibility.

Resolution

Run the System Readiness Check only when both Quality and Production systems are on the same release/version level.

Note: Do not schedule readiness checks or TDR during upgrade windows or when systems are on different releases.
Plan readiness execution in the post-upgrade stabilization period.

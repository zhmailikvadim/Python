Symptom

Dependent object INT field type of oData Mobile Data Object Configuration cause conversion error, INT cannot be converted to CHAR.

This configuration fix contains changes to dependent settings of SERVICE ORDER oMDO -> WOOBJECTLIST source entity type for 2410, 2505 and 2511 applications.

Other Terms

SSAM, oData Mobile Data Object Configuration, Dependent Object, Object Key field INT, SERVICE_ORDER, WOOBJECTLIST

Reason and Prerequisites

Dependent object INT field conversion to CHAR fails

Solution

Follow the steps to import configuration change to SSAM 2511, SSAM 2505 and SSAM 2410 mobile application:

  • Download the attached file - XXXX_SO_DepObjList_INT_Fix.bin
  • Go to SAP GUI, execute the program /MFND/CORE_CONFIG_IMPORT_PROG in transaction SE38
  • Provide the filepath in the 'Filename' parameter of the Selection Screen (f4 help for file browser)
  • You may optionally provide a 'Customizing Request' to capture the changes
  • Check the 'Test Run' checkbox and execute the program to ensure no errors occur
  • Successful output should contain the rows modified for each table. All changes are rolled back when 'Test Run' is checked
  • Return to the Selection Screen
  • Uncheck 'Test Run' and then execute
  • Successful output should contain the rows modified for each table and the transport in which the changes were saved to, if provided
  • Go to Configuration Panel, navigate to SAMXXXX_SERVICE_ORDER oMDO class -> Dependent Object tab -> Select source entity type WOOBJECTLIST and Dependent oMDO ID SAMXXXX_EQUIPMENT -> navigate to Origin Object Keys tab -> verify OBKNR - INT4(10) and OBZAE - INT4(10) do not appear in the list. Instead, OBJ_NUM - CHAR(10) and OBJ_COUNTER - CHAR(10) exist.

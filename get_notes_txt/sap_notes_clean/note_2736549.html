Symptom

You are using Portal Services on XSA, and you configured the SUPPORT_INDEPENDENT_APP attribute in your site application with "true" value. 

Other Terms

FLP@XSA, Fiori, site, portal services, xsa

Reason and Prerequisites

Reason: program error.

Prerequisites: You are using Portal Services on XSA.

Solution

The fix is provided in SAP npm module: site-entry 1.9.18.

Workaround: Setting SUPPORT_INDEPENDENT_APP attribute to "false" will enable the functionality of routes plugins for the applications.

This operation is done using the XS CLI (described in https://help.sap.com/viewer/4505d0bdaf4948449b7f7379d24d0f0d/2.0.03/en-US/addd59069e6f444ca6ccc064d131feec.html).

Launch the XS CLI, and login to the relevant system.

First switch to "SAP" space: "xs t -s SAP".

Then perform the following settings: "xs set-env sap-portal-persistency SUPPORT_INDEPENDENT_APP false".

This command will change the environment variable SUPPORT_INDEPENDENT_APP of sap-portal-persistency application to "false".

The "set-env" command is described in the following documentation: https://help.sap.com/viewer/4505d0bdaf4948449b7f7379d24d0f0d/2.0.01/en-US/8bb90c8be912431dab74877a492a3f5a.html

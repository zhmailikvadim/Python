Symptom

Data conversion task "INS_PRCON_2_PRLRG" was not executed successfully. 

  • XPRAS_AIMMRG error encountered during system upgrade.
  • SQL statement for database object "/SAPSLL/PRCON" executed with error.
  • A3EE/SAPSLL/CNVRSN 026 SQL Error Code: "129" 
  • A2PE/SAPSLL/CNVRSN 016 Data conversion task "INS_PRCON_2_PRLRG" was not executed successfully

Environment

  • Global Trade Services (GTS)
  • Global Trade Services / Logistics Services (SLL)
  • SAP Global Trade Services
  • SAP Global Trade Services, Edition for Hana

Reproducing the Issue

  1. Execute the upgrade process for SAP Global Trade Services Edition for SAP HANA.
  2. During the execution phase, observe the system halting at phase "MAIN_NEWBAS/XPRAS_AIMMRG".
  3. Review logs for errors related to "/SAPSLL/XPRA_V8000".

Cause

The issue is occurred due to inconsistency or overlapping data in table "/SAPSLL/PRCON".

Resolution

  1. Use SQL query below in the SQL Editor in transaction DBACOCKPIT to identify duplicate entries in in table "/SAPSLL/PRCON".
  2. Clean up table "/SAPSLL/PRCON" by removing inconsistent or overlapping records.
  3. Retry the XPRA phase of the upgrade process.
    ========================================================================
    SQL Query statement for reference:
    SELECT PRCON.MANDT, PRCON.LGREG, PRCON.GUID_PR, TO_DATS( CAST( UTCTOLOCAL( CAST( PRCON.DATAB AS TIMESTAMP ) , 'CET' ) AS DATE ) )
    FROM "/SAPSLL/PRCON" AS PRCON
    GROUP BY PRCON.MANDT, PRCON.LGREG, PRCON.GUID_PR, TO_DATS( CAST( UTCTOLOCAL( CAST( PRCON.DATAB AS TIMESTAMP ) , 'CET' ) AS DATE ) )
    HAVING COUNT( * ) > 1;
    =========================================================================

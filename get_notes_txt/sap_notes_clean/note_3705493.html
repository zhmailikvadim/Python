Symptom

  • After upgrading from ECC to SAP S/4HANA the condition type calculation logic for the costing sheet of COPA valuation is not functioning as expected.
  • A value field remains zero in the CO-PA document.
  • The issue occurs despite identical configuration settings between the ECC and SAP S/4HANA systems.
  • In Valuation analysis the value for the Value field is calculated, however the 'Value after valuation' field is blank, and it is not transferred to the CO-PA line items.

Environment

  • Controlling and Profitability Analysis
  • SAP S/4HANA Finance
  • SAP S/4HANA

Cause

SAP Note 3288826 is the root cause for the symptom. This Note introduced a change in system behavior which came with the upgrade. 
This is however the correct and desired system behavior.
Zero values of cost components are recognized as valid values, and respective value field gets blocked for other valuation methods in case cost component split could be successfully read in advance. The original issue (the cause for the Note 3288826) was for transaction KE27, but the general logic also affects online valuation.

Resolution

Assign different value fields for valuation with product cost estimate and costing sheet to prevent this symptom.
OR
Alternatively, adjust the sequence of valuation steps in CO-PA Valuation Strategy by placing the valuation step with costing sheet before the valuation with product cost estimate.

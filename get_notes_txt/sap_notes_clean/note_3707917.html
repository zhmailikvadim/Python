Symptom

  • Joule is unable to access the SAP S/4HANA public cloud system to perform actions such as creating a fixed asset.

    • *Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental. 
  • Communication and data viewing are functional, but an error message appears stating, "I am unable to directly create a fixed asset for a laptop as I do not have access to your SAP system."
  • The user has assigned roles on BTP and has the required Business Catalog "SAP_FIN_BC_AA_MDAT_REG_PC" on their S4 system for creating fixed assets.

Environment

Joule

Reproducing the Issue

  1. Attempt to execute actions such as creating a fixed asset for a laptop in the SAP S/4HANA public cloud system via Joule Web Client
  2. Observe the error message stating the lack of access to the SAP system.

Cause

The issue is caused due to the lack of the role "SAP_FIN_BC_AA_MDAT_REG_PC" being correctly assigned to the backend user or synchronized with the IPS job.

Resolution

  1. Remove the "SAP_FIN_BC_AA_MDAT_REG_PC" role from the backend user.
  2. Reassign the "SAP_FIN_BC_AA_MDAT_REG_PC" role to the backend user.
  3. Run the IPS jobs again to synchronize the role with the BWZ.
  4. Verify the IPS job logs for any errors or unusual behavior.
  5. Check if the role now displays correctly in IPS and confirm that the system is functioning as expected.

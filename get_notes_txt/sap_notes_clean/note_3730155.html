Symptom

When calling an OData endpoint URL no data is returned and a SYSTEM_NO_ROLL dump results.

Environment

ABAP RESTful Application Programming Model  (RAP)

Reproducing the Issue

Open the Fiori application which calls the specific Odata query URL end point and notice no data is returned.

When applying a filter such as to shorten a date range, the data is returned as expected without any errors.

Without applying filters no data is returned and a SYSTEM_NO_ROLL or TSV_TNEW_PAGE_ALLOC_FAILED error can be observed.

Cause

When the OData query is executed with the $count option it is found that the queried entity contains several millions records.

GET /sap/opu/odata/sap/<service binding name>/<entity name>/$count

Resolution

Use query paging options such as $top or $skip in order to reduce the number of records and avoid memory execution errors in the database engine.

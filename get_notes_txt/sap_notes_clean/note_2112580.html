Symptom

You run the Reserves Report for an employee(s) who has calculated Social Insurance closing balance. The corresponding Wage Type Code (RSRV_T1 "Social Insurance Contribution for Reserves" in the standard delivery) includes more then one Wage Type, e.g.:

  • J301   01.01.1800 31.12.9999 Reserves for PFR
  • J302   01.01.1800 31.12.9999 Reserves for SI
  • J303   01.01.1800 31.12.9999 Reserves for MedInsFed

In result the closing balance amount is wrong and equals to the last Wage Type sum (in our example - J303), but should be equal to accumulated sum of all included Wage Types.
This issue is also relevant for all DAQ-Form's RSRV "Reserve calculation report" fields which use the functional module HR_RU_DAQ_RSRV_WAGES_RURES, e.g.:

  • RVTPB   PIC for Reserves Incom
  • RVTPE   PIC for Reserves Outcom
  • RVTSB  SIC for Reserves Incom
  • etc.

Other Terms

HRPAYRU_RESERVES, RPCPAYRU_RESERVES

Reason and Prerequisites

This Note is relevant only for Russia.
You have implemented Note 2061483.

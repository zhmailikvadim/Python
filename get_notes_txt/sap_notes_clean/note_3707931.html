Symptom

  • Performance issuefor archiving jobs on program CHANGEDOCU_DEL of object CHANGEDOCU.
  • Database locks observed.

Environment

  • SAP S/4HANA release independent
  • SAP Netweaver release independent

Resolution

  • Resolution 1

    During archiving process, the change documents are deleted from database. To avoid deadlocks, lock entries are made by default lock logic.
    Deactivate this referring to SAP Note 2510646 - CD: Deactivating the lock logic during archiving of change documents.
    The deadlocks are not expected after the switch. If it happens, the lock logic can be switched on again with no issues.

  • Resolution 2

    Check if the table logging is active for table CDHDR, CDPOS, CDPOS_UID or CDPOS_STR from SE11 -> Display the table -> Technical Settings -> General Properties -> Data Changes
    The logging is not recommended for these tables, as they document the changes of the application tables. Massive changes are done to these tables as expected.
    If it's checked, the performance is impacted when accessing these tables.


If this cannot help, collect the ST12 trace on the work process which the deletion job is running on. It can be found from SM51.
The job collection can be finished after some time, it's not necessary to cover the whole runtime of the job.

KBA 2436955 - How to collect and analyze traces using ST12 (single transaction analysis)

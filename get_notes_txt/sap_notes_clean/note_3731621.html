Symptom

When a visit status is set to "In Process", the "Check-Out" button is unavailable and cannot be used to complete the visit.

As a result, the user must manually change the visit status to "Completed" in order to finish the visit.

Environment

  • SAP Sales Cloud Version 2

Reproducing the Issue

  1. Go to the Visits work center.
  2. Open any visit.
  3. Manually set the visit status to "In Process" (without using the Check-In action).
  4. Attempt to perform Check-Out.
  5. Observe that the Check-Out button is disabled/greyed out.

Cause

Check-Out requires a successful Check-In first. Manually setting the status to "In Process" does not run the Check-In logic and does not create the required check-in timestamp (actualCheckInDateTime), so the system does not consider the visit as checked in and disables Check-Out.

This behavior is enforced by Visit rules that require completing Check-In before Check-Out; previously, such validations might not have been in place.

Resolution

  1. Use the Check-In action to start the visit. This records the check-in timestamp required to enable Check-Out.
  2. After a successful Check-In, use the Check-Out action to complete the visit.
  3. Avoid manually setting the status to "In Process" if you intend to use Check-Out, as this bypasses the Check-In logic and keeps Check-Out unavailable.
  4. Review Visit rules in Settings to ensure they align with your desired visit process.

Symptom

  • The Labor Welfare Fund (LWF) deduction is incorrect calculated during payroll processing.
  • LWF technical wage codes were generated in RT for the month for which payroll is run instead of December, contrary to the standard behaviour.

    Disclaimer: Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental. 

Environment

  • SAP ERP Human Capital Management
  • SAP ERP Payroll India
  • SAP R/3
  • SAP R/3 Enterprise
  • SAP ERP
  • SAP ERP Central Component
  • SAP enhancement package for SAP ERP

Reproducing the Issue

  1. Run transaction SE38 and enter program as 'HINCALC0' / Execute transaction PC00_M40_CALC_SIMU.
  2. Verify that LWF technical wage codes are generated in RT for the month for which payroll is run(ex February) instead of December.

Cause

Incorrect customizing in V_T7INU3

Resolution

  1. Make sure to keep first period as '09' (December).
  2. Keep EE Cont Wage Type and ER Cont Wage Type as blank. If wage type is entered here, then it will automatically generate /3W1 entry in RT.

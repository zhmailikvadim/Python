Symptom

The sapinst program was called with parameter "SAPINST_GUI_HOSTNAME=<hostname>".

But the parameter didn't take effect and the webserver hostname didn't change.

Environment

SAP SWPM 

Reproducing the Issue

Run the sapinst program

Cause

The parameter SAPINST_GUI_HOSTNAME only took fully qualified domain names (FQDN). 

Resolution

To resolve the issue, call the program sapinst with a fully qualified domain name (FQDN) set to parameter SAPINST_GUI_HOSTNAME, e.g.,  "SAPINST_GUI_HOSTNAME=<host1.domain1.com>".

Keywords

sapinst, SAPINST_GUI_HOSTNAME, selftest, return code: 3, connecting, failed, override, webserver, hostname

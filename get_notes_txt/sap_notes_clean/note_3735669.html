Symptom

When attempting to access transaction PTMW, it loads endlessly and can sometimes lead to a TIME_OUT dump. 

Environment

  • SAP ERP
  • Time Manager's Workplace

Reproducing the Issue

  1. Start transaction PTMW with the affected user.
  2. Observe the loading pop up and the application does not open

Cause

A previously saved default selection for PTMW loaded an excessive number of employees, causing long load times or termination each time PTMW started for the affected user.

Resolution

  1. Check whether a default selection is saved for the affected user in table SPERS_OBJ (Pers. Obj = <affected user>, fields name = SELID).
  2. Run report RPT_TMW_CLEAR_DEFAULT_SELID in the affected
    1. Enter the affected user in 'User Name'
    2. Check the 'delete Sel.ID' option
    3. Execute
  3. Start PTMW again and verify that it opens without loading the previous default selection.

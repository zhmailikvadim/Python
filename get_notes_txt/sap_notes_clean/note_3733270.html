Symptom

  • In Performance Management forms using Team Goals, the rating field is visible and updated in real time for all goal assignees, including when the form is in en-route status.
  • However, the Actual Achievement field does not update in real time in the en-route form.
  • This creates inconsistent behavior where rating values are exposed while achievement values are not.
  • Customers report this as a data visibility concern, as sensitive rating information becomes visible before the form is completed.

Environment

  • Product: SAP SuccessFactors Performance & Goals

Reproducing the Issue

1. Login and proxy as affected user.
2. Go to PM form and open the en route folder.
3. Open any form and scroll down to the affected goal.
4. Open goal details and observe that "Actual Achievement" is showing as "-" while "Actual Payout (%)" is populated with data. 

Cause

Current system design for team goals with cascade-update set to push-down: updates made by the goal owner are pushed in real time to assignees, and in the latest form experience goal detail fields with read permission are displayed, including in en-route forms. Engineering confirmed this behavior and is planning an enhancement so en-route forms use a snapshot value instead of real-time for the rating field.

Resolution

  • Planned Release Timeline:
    • Preview: b2605p1 (21-Apr-2026)
    • Production: 15-May-2026 (2605 Release)

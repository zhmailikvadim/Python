Symptom

  • When importing a planning model into the SAC system, below error message would appear:
    "Couldn’t import object XXXX: 'Failed to execute GRANT STRUCTURED PRIVILEGE TENANT_2. to TENANT_2."sap.fpa.services::FPA_OBJECTS_ACCESS". Component = DbHelper. Reason = invalid user name: sap.fpa.services::FPA_OBJECTS_ACCESS: line 1 col 117 (at pos 116)"

Environment

  • SAP Analytics Cloud 2026.2.6

Reproducing the Issue

  1. Log onto SAC
  2. Navigate to Transport-> Import
  3. Find out the package containing the planning model and open the Import panel
  4. On the following panel, navigate to the "Import Options" section
  5. Select the "Overwrite Preferences" as "Overwrite objects and data" and do not select "Remove permissions" and "Import Team Settings"
  6. Under "Content", only select the planning model and click on "Import"
  7. After waiting for a while, observe that this import would fail and below error message would appear:
    "Couldn’t import object XXX: 'Failed to execute GRANT STRUCTURED PRIVILEGE TENANT_2. to TENANT_2."sap.fpa.services::FPA_OBJECTS_ACCESS". Component = DbHelper. Reason = invalid user name: sap.fpa.services::FPA_OBJECTS_ACCESS: line 1 col 117 (at pos 116)"

Cause

The issue is caused by a defect in the product.

Resolution

This issue has been addressed in version 2026.2.9.

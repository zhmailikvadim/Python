Symptom

  • When clicking a tile in the SAP Fiori Launchpad (FLP) intended to launch the Customer Engagement (CE) or CRM UI interface, the following behavior occurs:

    • A new window or loading indicator flashes briefly for a fraction of a second.

    • The system immediately returns (falls back) to the Fiori Launchpad home screen.

    • The application fails to load, but no explicit error message is displayed in the UI.

    • The issue persists even if the underlying transaction (e.g., WUI) works correctly when executed directly in SAP GUI for Windows.

Environment

S/4HANA 2023 and others

Cause

  • The issue is caused by a technical conflict between the Fiori Launchpad's web container and "Launcher Transactions."

    1. Launcher Behavior: Transaction WUI (similar to WEBGUI) is a launcher designed to trigger a URL from a SAP GUI for Windows environment.

    2. Recursive Call: When called from a browser-based Fiori Launchpad, the transaction attempts to trigger a web session from within an existing web session. This causes a loop or session conflict.

    3. Framework Fallback: To prevent a browser crash or infinite loop, the Fiori Launchpad framework terminates the navigation and returns the user to the home screen.

Resolution

  • To resolve this issue and maintain a system-agnostic (multi-client/multi-system) configuration, change the tile from a Transaction-based mapping to a URL-based mapping using a Relative Path versus an Absolute Path.
    • What is an Absolution Path for Target mapping?  This is an example:

      ----Absolute Path example (Typical Workaround - Limited Scalability): An absolute path includes the full server address (https://) and often a specific client.--- 

      1) Change the Target Mapping from "Transaction" to "URL" in Fiori.
      2) Maintain the direct URL for the Customer Engagement service: https://<your domain>/sap/bc/bsp/sap/crm_ui_start/default.htm?sap-client=032 

      NOTE: Limitation: The tile configured above is "locked" to the target system and the defined hard coded client (sap-client=xxx) - It will work for that client but, it may cause login or "page not found" errors for other clients (Scenario: If the user is using the same hardcoded absolute path tile for other clients). 
      ----end-------- 
    1. Open the Fiori Launchpad Designer or Launchpad Content Manager.
    2. Locate the Target Mapping for the affected intent.
    3. Change the Application Type from Transaction to URL.
    4. Enter the URL as a Relative path to ensure it works across all systems and clients:
      • URL: /sap/bc/bsp/sap/crm_ui_start/default.htm (This is just an example)
    5. Important: Do not hardcode the hostname or sap-client parameter (See Absolute Path workaround above where it shows limitation). The Launchpad will automatically append the current system's host, port, and the user's active client at runtime.
    6. Set the Navigation Type (NavType) to explace (Open in a new window) to ensure the CRM UI has a full browser context.

Symptom

Is it possible to map custom fields to name2 column in suppliersexport.csv file in Supplier Lifecycle and Performance?

Resolution

The name2 column in the Suppliers export corresponds to vendor.vendorInfo.name2 in SLP. If you map a question/field on the Supplier Registration or Supplier Request to vendor.vendorInfo.name2, that value will populate the name2 column in the generated Suppliers export. It's not possible to use this fields to other purpose with standard functionality.

With feature SM-14213: Enable supplier full name, Ariba splits supplier name into name1–name4 for integration and exports automatically. With this feature is enabled, you may need to decide whether you want to explicitly map a name-related question to vendor.vendorInfo.name2 or rely on the full name splitting functionality. Please note that it is not supported to use mappings for name1–name4 SM-14213: Enable supplier full name feature is enabled.

If you would like to consider further customization options, please contact your CSM, who will be able to help you engage with a consulting resource to assist in building them on a project basis.

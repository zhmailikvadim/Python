Symptom

The standard method APEU/CL_MCM_CONFIG_ACCESS=>GET_MAINT_PLANT does not allow to enhance the implementation as per customer's specific requirement i.e., the BAdI is missing.

Specific use case - APEU/CL_MCM_CONFIG_ACCESS=>GET_MAINT_PLANT  derives the maintenance plant from the connection object (physical structure/functional location), but the customer requires different maintenance plants per division (gas vs. electricity) at device location level while keeping the connection object plant blank (shared connection object for multi-division).

Other Terms

 APEU,2508

Reason and Prerequisites

Enhancement ,Technical Master Data , Maintenance Plant

Solution

Below BADI and method can be used to override the maintenance plant values while determining attributes of technical master data.

BADI : /apeu/badi_mcm_config_access
Method : get_maint_plant

Input parameter  IV_ISU_OBJECT_IDENTIFIER of method GET_MAINT_PLANT would allow customers to identify the IS-U technical master data object type currently under processing.

Possible values:

  •  'CONNOBJ' Connection Object
  •  'DEVICELOC' Device Location
  •  'PREMISE' Premise

Symptom

The Check Tool provides a range of useful self-service options within Onboarding. This KBA focuses on the latest enhancements that significantly speed up issue resolution and reduce the need to rely on Technical Support.

Environment

SAP SuccessFactors Onboarding

Resolution

Here are some of the latest updates in Check Tool:

  1. BPE Process Flow for Onboarding Initiation from Recruiting

    Purpose
    This feature enables the reinitiation of the Onboarding process directly from RCM.

    Applicable Scenarios
    This process can be used in the following situations:

    • The Onboarding process trigger has been deleted or cancelled, while the onboarding initiation status remains marked as successful.

    • Candidate data has been purged, and the corresponding process trigger is no longer available to apply a quick fix.

  2. System has Onboarding Info of employees: ONB2SelectOrphanedOnboardingInfo

    Purpose
    This functionality is used to identify and remove stale or orphaned onboarding data from the system.

    Applicable Scenarios
    This option can be used in the following cases:

    • When the error message “Looks like something went wrong. Error code: 400” occurs while correcting Job Information through History.

    • When a user appears as an onboardee and, as a result, cannot be assigned as a Manager.

    • When, for any reason after hire, the user continues to appear as an onboardee in system logs and certain details are missing from the Employee Profile page.

How to Apply the Above-Mentioned Solution

  1. Navigate to the Check Tool.

  2. Select System Health.

  3. Choose Onboarding as the application.

  4. Ensure that no parameters are selected under Result Type.

  5. Under the Checks section, select the required check and follow the instructions provided in the UI.

Symptom

The following error message appears when a calculation references an account or measure that has the same name as the model calculation:
"A cyclical dependency was found in the measure"

Environment

SAP Analytics Cloud

Reproducing the Issue

  1. Create a model calculation where the calculation name matches the account or measure member name referenced in the formula
    Example:
    1. Measure calculation name as Expenses(same as existing account member) with the formula
      ACCOUNTLOOKUP([FixedCurrency], [Expenses])
    2. Account member name as FixedCurrency(same as existing measure member)  with the formula
      MEASURELOOKUP([Expenses], [FixedCurrency])
  2. Save the model
  3. Create a story using this model and insert a chart widget
  4. In the story, create any type of account or measure calculation:
    1. If an account calculation is created, select the FixedCurrency account member in the calculation and add FixedCurrency as the measure on the axis
    2. If a measure calculation is created, select the Expenses measure member in the calculation and add Expenses as the account on the axis

     5. Click Ok

Cause

The system detects a self-reference because the model calculation name matches the account or measure used in its formula

Resolution

  1. Navigate to the model calculation that uses ACCOUNTLOOKUP or MEASURELOOKUP
  2. Rename the model calculation or account member so that it does not match the referenced account or measure
  3. Return to the story calculation that triggered the error
  4. Edit and re-save the story calculation (it may initially show an error due to the renamed model calculation)
  5. The widget will then display data correctly

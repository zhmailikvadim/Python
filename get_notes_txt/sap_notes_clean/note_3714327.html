Symptom

When you try to add an Inventory Counting for an employee in a Multiple Branch database, you get the error: employee X does not belong to branch Y; select an employee that is assigned the branch.

Reproducing the issue

Pre-rerequisites:

  • Multiple branch functionality is enabled. (field Enable Multiple Branches in Administration →  System Initialization → Company Details, tab Basic Initialization)
  • Employee is not linked to a User and has multiple branches assigned.

Scenario: 

  1. Go to Inventory→ Inventory Transactions → Inventory Counting Transactions → Inventory Counting.
  2. From the header field Inventory Counter, select the option Employee.
  3. Fill in the rest of the necessary fields (Item, Whse, Counted, Counted Qty) and try to add the document.

Expected behavior:

SAP Business One would add the Inventory Counting.

Actual behavior: 

SAP Business One throws the error employee X does not belong to branch Y; select an employee that is assigned the branch and you cannot add the Inventory Counting.

Cause

System Limitation

Solution

This is a system limitation.

Workaround: Add an Inventory Counting for a User instead of an Employee.

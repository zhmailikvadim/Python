Symptom

After a system upgrade or SPS update, you find that expected entries are missing from table TVARVC.

Other Terms

upgrade, SPS update, TVARVC, selection variants

Reason and Prerequisites

Customer issue, Consulting

Solution

The table TVARVC stores custom variant variables that can be used in selection variants. SAP updates or upgrades do not modify or delete entries in TVARVC.

TVARVC is a client-dependent table with delivery class "G" (“Customizing, protected against SAP updates, only INS allowed”). This means that standard SAP upgrade tools such as SUM or SPAM treat TVARVC like any other table of this delivery class. SAP generally does not deliver variant variables, as both the entries and their values are entirely customer‑defined. Consequently, SAP does not normally insert entries into this table.

From an SAP Basis perspective, the only program with write access to TVARVC is transaction STVARV, which is intended for maintaining customer-specific variant variables. There is no SAP program or upgrade process — neither as part of, nor as an extension to update or upgrade procedures — that would modify TVARVC.

Most changes to TVARVC occur under customer responsibility, typically through:

  • Custom reports scheduled to periodically update variant variables and their values
  • Manual maintenance using transaction STVARV
  • Custom transports imported from other systems

To determine the cause of missing or changed entries, it is essential to understand how the system or client was created:

  • Was the system copied from another system? If so, when was the system copy executed?
  • Was a client copy performed, either local or remote? If so, which profile parameters were used?
  • Review the change log for TVARVC using transaction SCU3. Are there log entries showing when the missing entries were created or deleted?
  • Compare TVARVC entries with another client or system using SE16, especially if the missing entries still exist there: Are there not only missing entries but also newly created or modified ones?

Depending on the type and purpose of the variant variable, the required corrective action may involve transporting the missing entries from another system or executing custom reports to recreate or update them.

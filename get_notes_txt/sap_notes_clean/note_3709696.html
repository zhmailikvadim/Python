Symptom

You are using the Run Statutory Reports app to create the Norwegian SAF-T Financial Data report (NO_SAFT_ACCOUNTING) and the following issues occur:

1. The report takes too long to finish.

2. One-Time customers with no VAT (customer ID with "/NA") are repeated in the file with addresses from other one-time customers with the same ID.

3. G/L Accounts IDs in transaction section have leading zeros.

Other Terms

SAF-T, Audit File, ACR, DRC, SAP S/4HANA for compliance reporting, Statutory Reporting, Run Compliance Reports, SAP Document and Reporting Compliance.

Reason and Prerequisites

These issues are due to program errors.

This SAP Note is relevant only for Norway.

Solution

Import the Support Package specified for your release or implement the attached correction instruction.

As a general rule SAP recommends that you implement the Support Package specified for your release. However, if you need to install a solution earlier, use the SAP Note Assistant to implement the correction instruction. You can find more information about the SAP Note Assistant in SAP Support Portal, under SAP Note Assistant.

To implement the solution, you must carry out manual activities and implement the correction instructions that are valid for your release.

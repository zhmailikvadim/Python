Symptom

You've installed SAP BOBI with the default SQLAnywhere database and changed it to another one to be used for CMS and Audit. Now you want to completely remove SQLAnywhere from your server.

Environment

  • BI 4.3/2025
  • SQLAnywhere
  • Windows Server

Cause

SQLAnywhere is not needed anymore.

Resolution

*Perform the steps below only after confirming that the CMS and Audit where correctly moved to the new database and the platform is functioning like expected and all server are running*

  1. Stop the Server Intelligence Agent (SIA) via the Central Configuration Manager (CCM).
  2. Navigate to ODBC 64-bit settings and remove the following system DSNs: BI_4_audit, BI_4_CMS, and Sample SAP SQL Anywhere DSN.
  3. Access the SIA properties and remove the SQL Anywhere dependency under the "Dependencies" tab.
  4. Uninstall SQL Anywhere via "Add or Remove Programs". If prompted to stop running processes, select the option to close them and proceed.
  5. Restart the SIA.

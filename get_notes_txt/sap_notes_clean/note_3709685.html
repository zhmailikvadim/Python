Symptom

  • Error encountered during payroll processing related to aggregation of chapter viii wage type.
  • Incorrect calculation of /444 wage type in the RT table.

Environment

  • Human Capital Management (HCM)
  • SAP R/3
  • SAP ERP Central Component
  • SAP ERP
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • Payroll India

Reproducing the Issue

  1. Run the payroll using the payroll driver for the provided variant.
  2. Check the value of /444 wage type in the RT table.

Cause

  • PAN card details has not been maintained correctly on info-type 185(Subtype : 0002).

  • Coding correction related to SAP note has been missing from the system.

  • Incorrect configuration of pay-date maintained on V_T549S table.

Resolution

  1. Go to PA30. Pass the PERNR details and info-type number as 185(Subtype : 0002).
  2. Maintain PAN card details.
  3. Implement the following SAP Notes to address related issues:
        • 3498413 - Tax deduction for employees with no PAN or inoperative PAN
        • 3472696 - Income Tax Deduction at Higher Tax Rate even with correct PAN details
        • 3504763 - PAN Inoperative: Income Tax Deduction for Employees With Gross Salary Below Exempted Taxable Limit
  4. If still issue is not resolved after implementing the note , execute the transaction SM30.
  5. Pass the view name V_T549S.
  6. Click on the maintain button.
  7. Pass the HCM Localization value as 40.
  8. Click on the position button. Pass correct combination of period parameter and Date modifier value along with current payroll year.
  9. Maintain correct pay-date and save it. As per the standard Pay-date should be on / after 25 th of that pay-period.

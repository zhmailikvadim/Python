Symptom

When displaying the material's basic data is resulting in dump as below:
Category               ABAP programming error
Runtime Errors         SAPSQL_PARSE_ERROR
Except.                CX_SY_DYNAMIC_OSQL_SEMANTICS
ABAP Program           CL_MDM_CLF_LEAN_SERVICES======CP 

Environment

  • SAP Master Data Governance 9.2

Reproducing the Issue

  1. Launch NWBC
  2. Select Material Governance
  3. Search Material
  4. Click Basic Data
  5. Dump is raised

Resolution

  1. Implement the SAP Note : 3689779 - Dump when using characteristics that have check tables 
  2. The dump is caused by the fact that the check table T059M is used for characteristic X, and the table T059M is a pooled table, which is not supported to have join. The method GET_F4_VALUES_CHECK_TABLE of class CL_MDM_CLF_LEAN_SERVICES uses JOIN to connect the value table to its corresponding text table.  The dump occurs when the code execute the SQL statement that contains a join of table T059M and table T059T because JOIN is not allowed for pooled tables, cluster tables, and projection views. Do not use pooled tables as check table for characteristics. 

Keywords

Characteristics, pooled tables, GET_F4_VALUES_CHECK_TABLE, CL_MDM_CLF_LEAN_SERVICES, T059M, T059T

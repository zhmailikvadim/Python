Symptom

The custom logic doesn't work when overriding some Spring shipped beans during their customization when using SAP Commerce Cloud.

Like: ResourceOwnerPasswordTokenGranter custom logic is not being invoked after successful authentication, despite being defined in the Spring XML configuration.

Environment

SAP Commerce Cloud

Reproducing the Issue

  1. Extend ResourceOwnerPasswordTokenGranter and add custom logic after successful authentication.
  2. Define the custom bean in the Spring XML configuration file.
  3. Attempt to invoke the custom logic after authentication.

Cause

The custom logic in ResourceOwnerPasswordTokenGranter is not being invoked due to configuration issues or limitations in Spring bean overriding.

Resolution

Set the property "spring.main.allow-bean-definition-overriding" to "true" in the Spring configuration file to enable overriding of Spring beans.

This property is available since Spring Boot 2.1, and used to control whether Spring allows multiple bean definitions with the same name to coexist in the application context, with the latest definition overriding the previous ones.

   public class CustomResourceOwnerPasswordTokenGranter extends ResourceOwnerPasswordTokenGranter {

       public CustomResourceOwnerPasswordTokenGranter(AuthenticationManager authenticationManager,

                                                      OAuth2RequestFactory requestFactory) {

       @Override
       protected OAuth2Authentication getOAuth2Authentication(ClientDetails client, TokenRequest tokenRequest) {

           System.out.println("Custom logic after authentication");

2. **Define the Custom Bean in Spring Configuration**:
   Define the custom bean in your Spring XML configuration file. Ensure that this configuration file is loaded by SAP Commerce Cloud.

   **custom-spring.xml**:

   <beans xmlns="http://www.springframework.org/schema/beans"
          xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
          xsi:schemaLocation="http://www.springframework.org/schema/beans
                              http://www.springframework.org/schema/beans/spring-beans.xsd">

       <alias name="customResourceOwnerPasswordTokenGranter" alias="resourceOwnerPasswordTokenGranter"/>

       <bean id="customResourceOwnerPasswordTokenGranter"
             class="com.mycompany.custom.CustomResourceOwnerPasswordTokenGranter">
           <constructor-arg ref="authenticationManager"/>
           <constructor-arg ref="defaultAuthorizationServerTokenServices"/>
           <constructor-arg ref="clientDetailsService"/>
           <constructor-arg ref="oauth2RequestFactory"/>
       </bean>
   </beans>


3. **Ensure Spring Configuration is Loaded**:
   Ensure that your custom Spring XML configuration file is loaded by SAP Commerce Cloud. You can do this by adding the configuration file to the "localextensions.xml" file.

   **localextensions.xml**:

   <extensions>
       <extension name="customextension"/>
   </extensions>

   **customextension/resources/customextension-spring.xml**:

   <beans xmlns="http://www.springframework.org/schema/beans"
          xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
          xsi:schemaLocation="http://www.springframework.org/schema/beans
                              http://www.springframework.org/schema/beans/spring-beans.xsd">

       <import resource="classpath:/custom-spring.xml"/>
   </beans>


4. **Set the Property for Bean Overriding**:
   In SAP Commerce Cloud, you can set the property to allow bean definition overriding in the "local.properties" file.

   **local.properties**:

   spring.main.allow-bean-definition-overriding=true

Symptom

  • In an MDF workflow, a mandatory picklist field (Employee Declaration) is selected.

  • The workflow does not display the selected value in the Workflow Details page.

  • Users are unable to proceed or verify the value, even though the field is required.

  • Issue may appear only in certain language settings.

Environment

  • SAP SuccessFactors HCM Suite
  • Metadata Framework (MDF)

Reproducing the Issue

  1. Navigate to Admin Center → Picklist Center.

  2. Open the relevant picklist and maintain a picklist value with a translation only in English (United Kingdom).

  3. Do not maintain any translation for English (United States) for this picklist value.

  4. Navigate to Configure Object Definitions and open the required custom MDF object.

  5. Assign the above picklist to a field in the custom MDF object and save the configuration.

  6. Configure a workflow for this MDF object and ensure the workflow is active.

  7. Log in as a user whose language is set to English (United States).

  8. Create or update a record for the custom MDF object and select the picklist value created in Step 2.

  9. Submit the record so that the workflow is triggered.

  10. Log in as the workflow approver (with language set to US English) and open the workflow approval request.

  11. Observe that the picklist field value is not displayed in the workflow approval screen.

Cause

The en_US translation for the selected value was not maintained.

Resolution

  • Ensure that translations are maintained for all picklist values in the required system languages.
  • Verify that the picklist values contain a valid translation for English (US) (en_US) or any active language used in the instance. Once the missing translation is maintained, the selected picklist value will display correctly in the workflow details.
  • Align the instance and user language settings with the maintained translations to ensure consistent display behavior.

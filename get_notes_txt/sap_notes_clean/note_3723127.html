Symptom

When attempting to access the import queue you an alert pops up with the following:

Transport control program tp ended with error code 0200
Errors: ERROR: see tp's stdout (Standard Out may not be
av

By checking the tp Output on the side button, this error or similar can be seen:

This is tp version <version> (release <release version>, unicode enabled) 
E-/usr/sap/trans/bin/TP_DOMAIN_<SID>.PFL could not be opened.
EXIT
Error in /usr/sap/trans/bin/TP_DOMAIN_<SID>.PFL: transdir not set.

tp returncode summary:

TOOLS: Highest return code of single steps was: 0
ERRORS: Highest tp internal error was: 0208
tp finished with return code: 208
meaning:
  error in transport profile (parameter missing or unknown)

Environment

All SAP Netweaver and S4/HANA versions

Reproducing the Issue

  1. Go to t-code STMS
  2. Import Overview on top menu (red truck icon)
  3. Double click on the desired import queue

Cause

The TMS profile file does not exist or it was deleted somehow.

Resolution

You can check directly if the profile file exist in the directory /usr/sap/trans/bin, there a file called TP_DOMAIN_<SID>.PFL should exist. The <SID> of this file is usually the SID of the domain controller system.

If the file is not there, you have to update the configuration by logging in client 000 of the domain controller and follow these steps:

  1. Go to t-code STMS
  2. System Overview (top menu)
  3. One click on domain controller system
  4. Click on SAP System (button on top menu)
  5. Update Configuration

With this, the profile file will be generated again and the access to import queue is fixed.

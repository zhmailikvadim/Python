Symptom

The electronic wage statement (RPCUVBD0_OUT) generates warning HRPAYDESVUV 056 ("AI data (BBNRUV &1, employee no. &2, year &3) cannot be assigned") for a personnel number.

Environment

  • SAP ERP 6.00 and higher
  • Human Resources / HR / HCM

Reproduce Issue

During the creation of the electronic wage statement (RPCUVBD0_OUT), the system determines data relevant for AI notifications for a personnel number that cannot be assigned to a partial wage statement to be created.

Cause

The warning is generated because the master data comparison for the AI notification procedure was not carried out successfully for a reporting office.

The following prerequisites must be met:

  • Outbound notification to the AI master data service (data module DSAS) with the status "Transferred"
  • Corresponding qualified confirmation (data module DSSD) with the status "Processed" or "Processed Manually" 
  • Assignment of both notifications (outbound and inbound) using a common operation ID
  • An entry corresponding to the personnel number in the payroll result of table V0 for the current Customizing of subapplication BG01

Solution

  1. Use a test run of the report RPCUVAD0_OUT to check whether the query to the AI master data service was created according to the latest status and whether the status "Transferred" exists after the file was created.

  2. Use the Administrator List of Inbound Notification (RPCUVLD0_IN) to check whether a corresponding qualified confirmation of the AI master data was received and processed that listed the required hazard pay scale.
    In test systems without qualified confirmation from the workers' compensation association, you can alternatively use the table T5D32_GT_VALID to check the permitted hazard pay areas.

  3. On the detail screen of the data modules DSAS and DSSD, check whether the outbound notification and inbound notification have the same activity ID.

  4. Check whether the entry in the payroll result of table V0 for personnel number contains both the workers' compensation association, the hazard pay scale, and the AI membership key and whether this data matches the current Customizing in table view V_T596M of subapplication BG01 for the relevant reporting personnel area.
    If the data does not match, update the entry in table V0 by performing retroactive accounting for payroll to the start of the notification period for the corresponding year of the electronic wage statement.

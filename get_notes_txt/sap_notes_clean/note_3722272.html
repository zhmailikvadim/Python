Symptom

When an Supplier Invoice is created through the Automatic Inter Company process (Value Chain Monitor (VCM) app involved), it is noticed that the field Payment Terms (ZTERM) is missing in both the MM and the FI document.

Environment

  • Materials Management (MM)
  • SAP ERP Core Component
  • SAP ERP
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA
 

Cause

  • In the case of the MM document, the payment terms will not be saved once the document is posted.
    This is due to the fact that the field has been designed as an input field.

    More info can be found under KBA: 2523493 - MIR4: Payment terms (ZTERM) and House Bank Account ID (HKTID) not retained


  • In case of the FI document, when creating the invoice from the user interface, the Payment Term information is sent as it is introduced.
    When creating it via background, though, ZTERM it is either mapped into the API or it is retrieved from the purchasing document/vendor master (depending on the scenario, more details under 322430 - MIRO: Default logic for terms of payment)

    There is, though, a specific case that applies to Inter Company invoices:
    The same way that ZTERM is not saved at the MM side, the IC process is not mapping this field from the billing document into the Supplier Invoice modules,. Only the payment terms' information gets mapped (ZBD1T, ZBD1P, ZFBDT, etc).
    This mapping happens under method MAP_PAYMENT_TERMS (CL_EDI_SD_INVC_MAPPING).

    If, though, any of these fields mentioned has any value and is passed into our application, we will not retrieve any new Payment Terms (for example, from the PO in method COMPLETE_DOCUMENT (CL_IVE_B2B_PREPROCESSOR_ABSTR)), as this would create an inconsistency in between the new Payment Terms and the information mapped from the billing document.
    Thus, the payment terms will remain empty for both documents.

Resolution

Both outcomes are a result of the system's design.
If the requirement is to have the field ZTERM filled in the FI document, this can be done via FI substitution (check 42615 - Substitution in FI) or manually via FB02.

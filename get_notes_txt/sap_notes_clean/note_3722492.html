Symptom

This hub KBA collects information about setting up alerts and notifications in various HANA environments. The alert and notification options depend on the target HANA being utilized. See the different scenarios in the "Resolution" section below.

Environment

SAP HANA smart data integration 2.0

Resolution

HANA on-premise

See KBA 3195642 - How to create email notifications for SAP HANA SDI task status changes or failures? for further details.

HANA Cloud

From the SAP HANA Cockpit, select the "Alert Definitions" option from the "Alerts" section. This is where the SDI-related alerts can be configured. The list of alerts that can be configured is identical to the ones in SAP BTP (see next section).

SAP BTP

The following SDI-related alerts exist in SAP BTP:

  • HDB SDI Agent Availability: Determines how long the agent has been inactive.
  • HDB SDI Agent Memory Usage: Monitors the percentage of the agent's total memory usage.
  • HDB SDI Remote Source Applier Delay: Checks for delays in applying change data for a remote source.
  • HDB SDI Remote Source Change Data Time: Monitors the time elapsed since the Data Provisioning Server last received any change data from the source database.
  • HDB SDI Remote Subscription Exception: Monitors recent exceptions in remote subscriptions and remote sources.
  • HDB SDI Remote Subscription Queue Time: Checks how long the remote subscriptions have been in a queued state.

See SAP HANA Cloud Service Database Events for further details.

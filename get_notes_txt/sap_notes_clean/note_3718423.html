Symptom

When executing the "Push to Supply Planning" process with existing Scheduling Agreement and follow-on Sales Order documents, the system cannot delete Scheduling Agreements. Instead, quantities are reduced to match existing Sales Order quantities, triggering multiple cascading issues.

  1. During Scheduling Agreement Updates, "Study ID" and "Protocol ID" header fields may be set to blank. Also Scheduling Agreements may not be properly deleted despite the process execution
  2. In Callback Messages (SAP S/4HANA to BTP Communication):
    • Success messages contain incorrect "Depot ID" values
    • "Protocol ID" field remains unpopulated despite being required for Lean Studies status update
    • When Scheduling Agreements cannot be located due to incomplete request data, a success message is sent instead of an error message
    • Response data shows incomplete information: Sales Area displays "--", Depot and Document Number fields are empty
  3. Upon Multiple "Push to Supply Planning" Executions:
    • Duplicate requests are triggered for identical site groups
    • Scheduling Agreements display with incorrect or missing MRP Area values (including malformed entries such as "()")
    • MRP Area descriptions fail to populate
    • Site groups remain in "In progress" status until the timeout check automatically sets the status to "Processing failed".

Other Terms

Push to Supply Planning, Scheduling Agreement

Reason and Prerequisites

You use SAP Intelligent Clinical Supply Management.

Solution

Implement the attached correction.

Symptom

While copying a company code from a template company code using transaction EC01, the system issues message FC158.

Environment

  • Financial Accounting (FI)
  • Profit Center Accounting (PCA)
  • SAP ERP Central Component
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA Finance
  • SAP S/4HANA

Reproducing the Issue

  1. Start transaction EC01 and choose to copy an existing company code to a new company code.
  2. Use a template company code that is assigned to a controlling area, where classic Profit Center Accounting is active and has a 8A ledger assignment in table T882.
  3. Do not copy / assign the controlling area to the target company code during the copy.
  4. Afterwards the system displays message FC158.

Cause

Template company code has an assignment to ledger 8A (classic Profit Center Accounting) in table T882. Function module FI_ECOP_BUKRS_CHECK detects an inconsistency (ledger assignment without controlling area) and issues message FC158 (based on T881/T882 checks; relevant when currency types like 20/70/90 are active for the ledger).

Resolution

  • Controlling area assignment is intended:
    1. Assign the new company code to the appropriate controlling area or include the controlling area assignment during EC01.
    2. Run transaction SPRO:
      Enterprise Structure >> Assignment >> Controlling >> Assign company code to controlling area

  • Controlling area assignment is not intended:
    1. Remove the ledger-company code assignment that was copied for ledger 8A.
    2. Start transaction GCL2.
    3. Enter OPEN in the transaction code field and press Enter.
    4. Select ledger 8A and open it.
    5. Choose “Assign comp-cd/comp”.
    6. Select the target company code and choose “Remove”.
    7. Save the changes.

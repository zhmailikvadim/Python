Symptom

  • Using Remote Backup feature
  • Fails after upgrade to 16.0 SP04 SP08
  • The command line and Backup Server log report

    Backup Server: 3.32.2.1: Incorrect type for parameter #13 for RPC as_cmd; expected SRVVARCHAR got SRVVARBINARY.

    Backup Server: 5.7.2.7: RPC ('as_cmd') execution failed.
    Msg 8009, Level 16, State 1
    Error encountered by Backup Server.  Please refer to Backup Server messages for details.

Environment

SAP Adaptive Server Enterprise 16.0 SP04 PL08 and above                                                                        
                                                                 

Reproducing the Issue

load database mydb from '/sybase_dumps/mydb.dmp at REMOTE_ASE_BS

Cause

  • The general expectation when using remote backups is that both servers are on the same release level, but this is not a documented requirement.
  • For ASE 16.0 SP04 PL08 and higher, it is not compatible to use a lower ASE version before these releases for Remote Backups. 
    • This issue that stems from the fix referred to in KBA 3192324, which is new in 16.0 SP04 PL08.

Resolution

Ensure both servers in the Remote Backup pair are of release 16.0 SP04 PL08.

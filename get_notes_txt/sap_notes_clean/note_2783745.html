Symptom

This KBA explains following behavior:
In SOLDOC you have Test Cases - Test Documents with Status RELEASED - but when you create a Test Plan by selecting the option "Only Released Documents allowed" under Settings tab, the Test Case Hierarchy shows empty.

Test Plan creation:

Environment

SAP Solution Manager 7.2

Reproducing the Issue

SM_Workcenter - Test Suite - Test Plan Management - Create new Test Plan - choose "Only Released Documents allowed"

Cause

Document Status not marked as "Released"

Resolution

select the Test Case - Test Document in Soldoc, check the Status and Technical Name - in this example the status is "RELEASED" (technical name 0RELEASED).
you need to make sure that the status is marked as "Released" in view V_SMDDOCSTATE
go to SM30 View V_SMDDOCSTATE
the status should have the Released flag

Afterwards generate Test Case Hierarchy again.

See Also

Guided Procedure under SOLMAN_SETUP - Process Management - Customize Document Handling - Define Values for Documents

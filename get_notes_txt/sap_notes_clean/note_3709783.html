Symptom

  • After Implementing TLSv1.2, Server Intelligence Agent is not responding.
  • A blank page is displayed when attempting access the CMC/BILP URL.
  • ODBC Test connection to database fails with error: "Unable to complete login process due to delay in opening server connection".
  • Error: "[Microsoft][ODBC SQL Server Driver][DSNNAME] SSL Security error"
  • Event viewer shows below error:

"CMS is unstable and will shut down immediately. Reason: SAP BusinessObjects Bl platform CMS: Unable to connect to the CMS system database ''<DSNNAME>". Reason: [Microsoft][SQL Server Native Client 11.0] Unable to complete login process due to delay in opening server connection (FWB 00087)"

Environment

  • SAP BusinessObjects Business Intelligence Platform 4.3
  • Windows (OS)
  • Microsoft SQL server 2016, 2017, 2019, 2022

Cause

  • The issue is related to incompatibility of the ODBC driver version for CMS database with TLS1.2 in BusinessObjects 4.3.

Resolution

  1. Upgrade the ODBC driver to supported version which is compatible with Microsoft SQL Server and TLS 1.2.
  2. Refer the 4.3 PAM guide SAP BusinessObjects BI Platform 4.3 Supported platforms (PAM).

  • Refer to: SAP BusinessObjects BI Platform 4.3 Supported platforms (PAM).

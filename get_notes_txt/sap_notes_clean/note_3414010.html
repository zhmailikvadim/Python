Symptom


The problem occurs only on Windows.

Work processes have the status 'running' without further information. The work processes each occupy a complete CPU.

Reason and Prerequisites

﻿﻿﻿﻿﻿﻿﻿There was a memory overflow in the heap. When checking the heap size after a request was processed, the heap check runs in an endless loop.

Solution

As a workaround, the two profile parameters:

rdisp/os_heap_check_frequency = 0
rdisp/os_heap_check_time = 0

can be set. As a result, the heap size is no longer determined and the work process can no longer be blocked. However, this does not resolve the cause of the problem.

With the kernel patch that is visible in the SAP Note, the heap corruption is recognized and the work process is restarted.

Symptom

You are using the function module DMC_MIG_EQUIPMENT for data migration into your S/4 system. You are providing multilingual short texts.

You can find, that the short text in logon language/primary language is not marked correctly in table EQKT.

Other Terms

Migration, ECC, S/4, Migration Modeler LTMOM, Migration Cockpit LTMC.

Equipment, IE01, IE02, IE03, BAPI_EQUI_CREATE.

ITOB, EQUI, EQKT, TXASP, primary language.

ITOBBUFEQ, EQKT_MOVE_TO_WRITE_BUF,

Reason and Prerequisites

Reason:

Program error in ITOB buffer management for table EQKT.

ITOB-buffer management has an issue with EQKT-TXASP when in the same process the equipment is created with short text in creation language and later on another short text in a different laguage is added in change mode. This way processing is used in the data migation function module DMC_MIG_EQUIPMENT.

Prerequisites:

  • Equipment migration from ECC to S/4
  • Equipment has to be created with multilingual short text.
  • Equipment has to be installed into a superior technical object.

Solution

Correction via this note or via the corresponding support/hotfix package.

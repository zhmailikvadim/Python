Symptom

With SAP Kernel 7.45 or 7.49, there is a very slow (over several days or weeks) but constant rise in the memory consumption in the work processes. The individual work processes can be affected differently.

In most cases, the rise is so slow that it is not really noticeable, especially if the system is restarted frequently - weekly, for example. However, without a restart, consumption can reach relatively high values per process over a period of several weeks.

Other Terms

Memory leak

Reason and Prerequisites

Memory allocation error in the source code of the DBSL. Occurs only with SAP Kernel 7.45 and 7.49.

Solution

Workaround:
Restart the SAP system or the individual affected work process.
The memory allocation is them usually normal for several days or weeks. The increase is really slow

To definitively rectify the error, please apply the new DBSL patch.
The problem is solved for:

SAP Kernel 7.45: as of DBSL Patch 213
SAP Kernel 7.49: as of DBSL Patch 6

Symptom

Error occurs when exporting a software collection: 

App for Analytical Queries YY1_xxxx_AQFA depends on Custom Analytical Projection View YY1_yyy.
Dependency to non existing item. Report an incident on component BC-DWB-DIC-VB

Environment

SAP S/4HANA Cloud Public Edition

Resolution

A fix has been implemented and is planned to be delivered with the upcoming release scheduled for 19.04.2026.

As a temporary workaround:

  1. Create a new Query and the corresponding InA Services using the Custom Analytical Queries and Services application.
  2. Create the query directly within the Custom Analytical Queries and Services application to avoid the migration-related transport issue.
  3. Consume the newly created Query and InA Services in the MDA application and include them in a software collection for transport.
  4. Refer to: KBA 2710858 - Custom Analytical Queries created in SAP S/4HANA Cloud Public Edition are not visible in SAP Analytics for detailed steps on creating InA services and applications.
  5. After the release is delivered, re-test transporting migrated queries and retire the workaround if the issue no longer reproduces.

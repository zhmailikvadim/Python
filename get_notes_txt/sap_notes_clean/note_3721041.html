Symptom

  • When attempting to restore a tenant DB using HANA Studio, the process gets stuck at the following phase:

  • The backup.log only contains the RECOVER DATA command, and no further logs are generated afterward.
2026-01-12T16:12:05+05:30  P0065873      19bb1cc773e INFO    RECOVERY RECOVER DATA started [1768214525758]
2026-01-12T16:12:05+05:30  P0065873      19bb1cc773e INFO    RECOVERY command: RECOVER DATA FOR DB0 USING FILE ('/hana/backup/COMPLETE_DATA_BACKUP_xxxx')  CLEAR LOG

  • Attempting to perform the restore using the RECOVER DATA command directly from either SQL Console or hdbsql shows the same behavior, with no further progress.
  • The indexserver trace does not contain any related errors or helpful information.
  • During the ongoing restore, indexserver Runtime dump (RTE) was collected (it may fail due to the indexserver not starting; try a few more times). 
    Analysis of the RTE dump shows the restore thread was stuck at the following call stack:

Environment

SAP HANA, platform edition 2.0

Reproducing the Issue

Attempt to restore the tenant DB in HANA Studio or using the command: RECOVER DATA FOR DB0 USING FILE ('/backup/PRD_BKP/COMPLETE_DATA_BACKUP_xxxx') CLEAR LOG

Cause

Issues with file system.

Resolution

    Unmount and mount the directory containing the backup data:

    For example:

    umount /hana/backup...
    mount /hana/backup...

    If the restore still stuck at the same phase, reboot the Linux OS and attempt the restore again.
    If the issue persists, collect OS system information as per SAP Note 618104 and contact SAP support with the component: BC-OP-LNX.

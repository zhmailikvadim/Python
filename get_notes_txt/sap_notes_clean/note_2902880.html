Symptom

The sample Customizing delivered in client 000 has the following shortcomings:

  1. Until now, the ELM WHT return could be mapped only for one of the two example enterprises using the sample Customizing.
  2. The example values for enterprise numbers have an invalid structure.
  3. The location in the example entry for the work center does not match the postal code. 

 

Update on 10/20/2020:
After you import the HR Support Package specified in this SAP Note, the system may issue the error message "No wages found - notification cannot be created" when you create the WHT notification (see "Solution" section).

Other Terms

Payroll Switzerland
Uniform wage notification procedure (ELM)

Reason and Prerequisites

Sample Customizing

The changes affect only the sample Customizing in client 000. This means that you do not need these changes for productive use.

Solution

Import the HR Support Package specified for your release to update the sample Customizing in client 000.

Content of the change:

  1. Sample Customizing for the WHT return with the example enterprise 0002 (additional payroll units (view V_T5C2Q), institutions (view T5C52), adjustment of characteristic CHFID and assignment to enterprise (view V_T5C51)).
  2. Formally valid values for enterprise numbers of the example enterprises 0001 and 0002 (view V_T5CSL).
  3. A location matching the postal code in the sample entry for the work center (view V_T5CSW).

 

Update on 10/20/2020:
If you have not created a customer-specific working version of the characteristic CHFID before importing the HR Support Package during the setup of the WHT notification, but you have used the standard version productively, the delivered adjustment of the decision tree of the standard characteristic becomes effective in your system after you import the HR Support Package.
In general, this causes unwanted changes in the assignment of employees to payroll units and consequently, the error message mentioned in the "Symptom" when creating the WHT notification: In this case, create a working version or modified version of the characteristic after importing the HR Support Package (or, if possible, earlier) and adjust the decision tree to meet your requirements. Creating a working version protects the decision tree adjusted to your requirements against overwriting or undesired effects after importing future HR Support Packages, since these only overwrite the standard version but not your working version. See also the documentation "Environment of characteristic maintenance".

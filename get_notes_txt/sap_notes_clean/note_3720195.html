Symptom

  • ASE server experiences periodic slow performance.
  • High CPU utilization is observed
  • With contention on rpmctrl_spin spinlock identified
  • Spids are in often COMPILING state

Environment

SAP Adaptive Server Enterprise (ASE) 16.0 SP04 & 16.1

Cause

  • This issue is related to memory allocation and tracking
  • It occurs in heavily loaded servers.
  • It was logged under SAP CR 830660:
     "On a heavily loaded system, sub-optimal performance and very high contention on rpmctrl_spin is observed."

Resolution

  1. Upgrade ASE to a version that includes the enhancements from SAP CR 830660:
    • 16.0 SP04 PL08
    • 16.1 SP00 PL02

  2. Activate these enhancements by switching on trace flag 758
    Refer to KBA 2601033 - How to boot ASE with trace flags 

Keywords

global temporary table , gtt

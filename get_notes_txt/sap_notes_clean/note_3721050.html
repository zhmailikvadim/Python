Symptom

The contact appears twice in the CSV file when importing via contact import, is not updated correctly based in the contact position in the csv file.

Environment

SAP Emarsys Customer Engagement

Reproducing the Issue

  1. Create a CSV file containing the same contact in the first row and in the 10th row.
  2. Import the CSV file via contact import to update the existing contact in the system.
  3. The contact is updated with the field informations of the 1st row and not the 10th row.

Cause

The reason for this is that we are using multi-threads to handle the import in order to speed it up. Therefore, sometimes the contact can be updated with the information of the contact in the 1st row and sometimes with the 10th row, in that example.

Resolution

We recommend avoiding having the same contact twice in the CSV for contact import and using the Contact API to do sequential updates.

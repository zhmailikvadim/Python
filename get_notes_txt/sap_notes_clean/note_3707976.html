Symptom

Object errors happen in upgrade phase PARDIST_ORIG

Two options below are available in SUM UI, as per the log file SAPupConsole.log:

01) - Repeat phase MAIN_SHDRUN/PARDIST_ORIG to continue at the point where it stopped.
02) - Accept errors and repeat phase MAIN_SHDRUN/PARDIST_ORIG.

The meaning of second option is required.

Environment

ABAP System

Reproducing the Issue

  • Upgrade via SUM

Resolution

  • Do NOT select option "Accept errors and repeat phase MAIN_SHDRUN/PARDIST_ORIG" without explicit suggestion from SAP support or SAP Notes for specific objects. It is an option for specific objects in special situation but not an option to ignore all errors in the phase.
  • If activation errors were accepted in ACT_UPG without explicit suggestion from SAP support or SAP standard documents for specific objects, please reset the upgrade and fix the activation errors in ACT_UPG phase. See more details about the meaning accepting error in ACT_UPG phase in Note 1962922.
  • Ignoring errors inappropriately can lead to problems in the future phases that may not be correctable. In such case, there will be no other choice, but resetting the whole upgrade and starting it all over.

See Also

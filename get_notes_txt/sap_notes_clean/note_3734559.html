Symptom

The transport request status shows "Set to outdated" and cannot be imported anymore.

Environment

SAP S/4HANA Cloud Public Edition

Cause

An outdated collection version can’t be imported anymore. The collection version is automatically set as outdated when the release at export is older than the current release according to the following rules:

  • In 2SL: Collection versions of the type Key User Extensibility are set as outdated after one or two releases, depending on the content of the collection. Collection versions of the type Customizing are set as outdated after one release.
  • In 3SL: Collection versions of the type Customizing are set as outdated only after two releases. Collection versions of the type Key User Extensibility are set as outdated after one or two releases, depending on the content of the collection.

Resolution

It is not possible to import a transport created on release n into release n+2 and higher, which means that it is needed to recreate the transport in the development system.

However, it is not possible to recreate CBC transports. In this case, need to create a support incident.

Symptom

The activation of a planning area failed, and errors appear in the Planning Run Profile related to Demand Prioritization Segments/Rules, such as:

  • “Rule for Demand Prioritization, segment X, contains invalid attribute XXX in condition”
  • “Condition is invalid for demand prioritization segment with segment priority”
  • "Condition is invalid for Inventory Holding Cost Rule segment with sequence number X"
  • The '' sort attribute is invalid in the demand prioritization segment with segment priority XXX

Where:

  • The attribute mentioned in the error is a custom attribute (extended master data)
  • Master data values for the attribute exist and appear consistent

However, the attribute is still considered invalid by the Demand Prioritization rule.

Environment

  • SAP Supply Chain Management (SCM)
  • SAP Integrated Business Planning (SCM-IBP)

Reproducing the Issue

Activate planning area

Cause

The custom attribute has not been properly extended to the OBP master data model.

Even if the attribute exists in the planning area and contains data, it cannot be used in Demand Prioritization rule conditions unless master data extension has successfully completed.
This issue can occur if:

  1. Master data extension is not activated for the planning area
  2. Master data extension is currently In Process
  3. Master data extension did not start, is stuck, or failed

Resolution

Case 1 – Master Data Extension Not Activated

As of IBP 2602, master data extension for planning areas is controlled by the configuration parameter:

ENABLE_MASTER_DATA_EXT_FOR_PAREA

  • If the planning area is not included in this parameter, the master data extension will not run.
  • As a result, custom attributes cannot be used in OBP logic such as Demand Prioritization rules.

Ensure your planning area is properly maintained in this parameter.
Once the planning area is maintained in the ENABLE_MASTER_DATA_EXT_FOR_PAREA parameter in Global Configuration app, reactivate the planning area.
The errors in planning run profiles should be resolved.

Case 2 – Master Data Extension “In Process”

If the master data extension status is In Process, the system is still extending master data to the OBP model.

  • This process may take several hours, depending on:
    • Number of planning versions
    • Number of extended attributes

Wait until the process finishes successfully, then recheck the Planning Run Profile.

Case 3 – Master Data Extension Did Not Start or Failed

If the extension process:

  • Did not start
  • Is stuck for several hours
  • Shows status Failed

Then the attribute will remain invalid for Demand Prioritization rules. In this case, follow the steps described in the KBA below to trigger or restart the extension process:

3540868 - IBP – Master Data Extensibility for Order-Based Planning - Application-Specific Statuses - In Process or Failed

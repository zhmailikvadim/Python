Symptom

When SAML is configured in the system, when access WSIL or WSDL link, the link is redirected to SAML link. In SOAMANAGER, SAML is not supported when configure integration scenario. As a result error happens when run pending task. It is needed to disable SAML for WSIL and WSDL.

Environment

  • ABAP Web Services
  • SAP NetWeaver
  • SAP NetWeaver Application Server for SAP S/4HANA
  • ABAP PLATFORM - Application Server ABAP

Reproducing the Issue

  1. SAML is enabled in the provider system of SOAMANAGER
  2. Run pending task for the provider system, error "No WSDLs were found for inbound interface" happens

Cause

SAML is not supported when configure integration scenario in SOAMANAGER. saml2=disabled parameter is also not an option, because it is not possible to modify generated WSDL link.

Resolution

To resolve the issue, maintain external alias in SICF for WSIL and WSDL.

  1. T-code SICF -> Execute -> Press button External Aliases
  2. Select root node "default_host" and press "Create New External Alias" button
  3. In the creation screen, input
    External Alias = /sap/bc/srt/wsil 
    Description 1 = External Alias for WSIL
  4. Go to tab Target, open node /default_host/sap/bc/srt/wsil, double click the node
  5. Press save button to save the change
  6. Press back, in the list, a service named /sap/bc/srt/wsil is visible
  7. Do step 1-6 for WSDL with similar steps


Once above done, run pending task again.

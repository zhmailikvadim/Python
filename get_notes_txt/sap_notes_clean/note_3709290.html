Symptom

  • Cannot rename a task in Sandbox
  • Error message: "Task names must be unique. A task with name 'XXX' already exists."

"Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental." 

Environment

SAP Cloud Integration for data services

Cause

Same task name is used in another project. A unique task name is enforced across the entire Sandbox system, not at the project level.

Resolution

  1. In Sandbox -> PROJECTS, click column 'Name' to enter the task name.
  2. click 'enter', it will show the task under which project.
  3. rename the task as business need.

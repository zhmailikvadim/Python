Symptom

You generate a raw report (Document Type SBR) using WWI and when you open the report using transactions CG50/CG54, the values of Parameter Report Symbols such as EHS_RPRVN1 and EHS_RPRVN2 (Report version) are correctly displayed.

However, when you open the same raw report using DMS transactions such as CV03N/CV04N, you observe that the values of the Parameter Report Symbols are not displayed. Instead the code of the Report Symbols is displayed.

Environment

  • SAP ERP
  • SAP ERP Central Component
  • SAP Enhancement package for SAP ERP
  • SAP ERP Environment, Health and Safety (EHS)

Cause

In transactions CG50/CG54, you see the raw report plus the currently valid parameter values which the system determines (expansion time: 20 Symbol value is application-dependent) using the Function Module C12O_PARAMETER_VALUES_SELECT_N.

The Function Module C12O_PARAMETER_VALUES_SELECT_N is not called when the raw report is displayed using the DMS transactions, that is why the values of parameter symbols such as EHS_RPRVN1 and EHS_RPRVN2 are not displayed.

Resolution

This is the Standard behavior of the system.

For SBR reports, the symbols of symbol types Specification and Descriptor are expanded. All other symbols (symbol type Parameter) for the report refer to data in other SAP components and are only expanded in the second generation stage from which the final report (Document Type SBE) results.

See Also

Refer to the SAP Help documentation about SBR Reports for more information: Report SBR

Symptom

When materials have multiple valuation areas the replication fails during the consolidation process in inbound processing from S/4 to MDG due to the errors below: 

An exception with the type CX_AI_APPLICATION_FAULT was raised, but was not handled locally or declared in a RAISING clause.Exception of class CX_PROXY_FAULT.

An exception with the type CX_AI_APPLICATION_FAULT was raised, but was not handled locally or declared in a RAISING clause.Exception of class CX_SY_NO_HANDLER

An exception with the type CX_SY_OPEN_SQL_DB was raised, but was not handled locally or declared in a RAISING clause.Exception of class CX_SY_NO_HANDLER

A data record is to be inserted even though a data record with the same primary key or same unique secondary key exists.

Exception of class CX_SY_OPEN_SQL_DB

Environment

SAP Master Data Governance-SAP S/4HANA Cloud Private Edition 2023 

Reproducing the Issue

  1. Material gets replicated from S/4 to MDG with multiple plant valuation areas.
  2. Sproxy > ProductMDMBulkReplicateRequest_In
  3. Execute the variant
  4. This will result in inbound processing error in MDG.

Cause

KALNR is part of the key for CKML* tables for which the CMP source and process tables are modeled on. So it cannot be zero or empty.Most of the time the import files are based on existing exports from active materials so the KALNR would be filled. In the SOA implementation, the XML definition for PRODUCTMDMBULKREPLICATEREQUEST does not transport such field and therefore the field mapping defaults all KALNR to zero. This causes the issue when trying to save the source file for consolidation for multi valuation areas.

Resolution

Implement the SAP Note :  3724400 - CMP: Cost Estimate Number (KALNR) is not set during product source record creation 

Keywords

KALNR, MBEW, CKMLHD, CKMLCR, PRODUCTMDMBULKREPLICATEREQUEST, CX_AI_APPLICATION_FAULT, CX_SY_NO_HANDLER, CX_SY_OPEN_SQL_DB

Symptom

The different callers of the subledger codingblock provider (SCP) use different timestamps in the same package processing.
That behavior bypasses the SCP buffer and ends up in multiple times reading the same master data and analytical status values from the database.

Other Terms

Performance scp_chars sdl_timestamp rdl_timestamp

Reason and Prerequisites

You run S/4HANA for financial products subledger release 3.0 with SP02, SP03 or SP04.

In older versions this optimization is not applicable.

There are no errors in the functionality.

Solution

Implement the attached Correction Instructions or import the specified Feature/Support Package.

In some cases IDoc status messages (EDIDS-STATXT) are saved in different languages.

Environment

  • SAP R/3
  • SAP S/4HANA
  • S/4HANA Cloud Private Edition

Reproducing the Issue

  1. Based on your process generate an IDoc.
  2. In se16(n) verify the status text of table EDIDS.
  3. If logon language is e.g. 'DE', status text is saved in 'DE'.
  4. If logon language is  e.g. 'EN', status text is saved in 'EN'.

Cause

The STATXT field in table EDIDS is populated based on the logon language of the user during Idoc generation.
If no language is maintained for the technical user in SU01, the system defaults to the language set in the GUI logon options.

Resolution

  1. Maintain a default logon language for the technical user in SU01. Verify that the SAP GUI logon options do not overwrite the SU01 settings.
  2. Ensure that the technical user logs in with the desired language (e.g., English) to ensure consistent language in the STATXT field.

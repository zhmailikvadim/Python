Symptom

Business Partner Identifiers which are stored in the data model for S/4 for Accounting Integration shall be deleted after a configurable retention time.

Other Terms

Data Protection and Privacy,  DPP

Reason and Prerequisites

Business Partner related information shall be no longer stored as needed for legal reasons. If a contract has been processed successfully in S/4 for Accounting Integration and the End of Retention Period has been reached, the data shall be destroyed. In the release S/4 for HANA for Accounting Integration 1610 no retention concept for the entire data model is available.

Solution

Report /SAI/RDPPDELBP is supplied. This reports provides a duration on the selection-screen with which you can control the deletion of data. Only finally processed data is selected. As an alternative also a key date can be inserted which controls alternatively up to which date the business parter related data in S/4 HANA for accounting integration 1610 shall be destroyed.

If you like to detroy the business partner related data regularly you can plan the jop periodically wiht a predefined retention period.

Apply the correction construction or install support package SP01.

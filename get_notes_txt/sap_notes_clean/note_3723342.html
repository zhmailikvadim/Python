Symptom

I need to set up SSO authentication for my environment using a specific corporate identifier, such as an Employee ID or Email address, that is sent as a custom attribute in the SAML token. This is required when the standard SAML "Subject" (NameID) does not match the unique Username format in my SAP Fieldglass user profiles.

Environment

SSO authentication fails if the Identity Provider (IdP) places the user's unique identifier in a custom attribute while SAP Fieldglass is still configured to look in the default SAML Subject (NameID). If the Attribute Name defined in the Configuration Manager does not exactly match the XML tag sent in the SAML assertion, the system cannot identify the user, resulting in a "User is not authorized" failure.

Cause

SSO authentication fails if the Identity Provider (IdP) places the user's unique identifier in a custom attribute while SAP Fieldglass is still configured to look in the default SAML Subject (NameID). If the Attribute Name defined in the Configuration Manager does not exactly match the XML tag sent in the SAML assertion, the system cannot identify the user, resulting in a "User is not authorized" failure.

Resolution

Configuring Attribute Mapping

  1. Log in as a Configuration Manager and select the Single Sign-On tile.
  2. Select Edit to open the setup wizard.
  3. Progress to the Service Provider Details page.
  4. Locate the SAML Identity Location property and change the selection from Subject to Attribute.
  5. In the Attribute Name field, enter the exact string used by your IdP (e.g., emailaddress or a full URI).
  6. Click Next Step to save and then Activate the configuration.

How-to: SAML Response Audit Guide

Use this guide to extract the exact Attribute Name from a raw XML log to ensure your mapping is 100% accurate.

Step 1: Capture the Raw SAML Response

  1. Navigate to the Single Sign-On tile and select the SSO Login History tab.
  2. Locate a failed attempt and click the linked Error Message.
  3. In the dialog box, select the link to display the full error details and the SAML Response.

Step 2: Identify the Identity Attribute

  1. Scroll through the XML code to the <saml:AttributeStatement> section.
  2. Find the <saml:Attribute> tag containing the user's ID.
  3. Copy the entire string inside the Name property (e.g., http://schemas.xmlsoap.org/ws/2005/05/identity/claims/emailaddress).

Step 3: Apply the Mapping

  1. Paste this exact string into the Attribute Name field in the SSO Setup Wizard.
  2. Save and Activate the configuration.

Important Note
Attribute names are case-sensitive. If your IdP sends a long URI, you must use the complete string found in the XML to avoid "User is not authorized" errors.

Symptom

CpqTableEntryDateModified is displayed in reverse order when the system (computer) date format is set to 12.02. 

Environment

SAP SALES CLOUD CPQ

Reproducing the Issue

1. Set up the local date to be 12.02.
2. Go to any custom table and edit any entry.
3. Save it, check that the custom table CpqTableEntryDateModified is reversed.

Cause

Bug in application.

Resolution

  • There is no workaround.
  • Tentative fix release version CPQ-2605.

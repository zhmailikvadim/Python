Symptom

  • Replication flow to cloud storage provider target (e.g. Amazon S3) encounters the error: "Retrying for initial load partitioning. The source connector has previously accepted the initial partitioning request."
  • Tables in SLT are in "initial load stopped" status.
  • Target setup fails due to the error: "invalid dataset folder: the data folder is not empty."
  • The Object Log shows error: Target setup failed due to the following error: invalid dataset folder: the data folder "XXX" is not empty. Enable the "Delete All Before Loading" option, remove all the files yourself or choose a different empty target. 

Environment

  • Replication Flow Runtime

Reproducing the Issue

  1. Attempt to trigger an initial load for replication flow to cloud storage provider target.
  2. Observe that the replication flow enters a "retrying mode" with the error: "Retrying for initial load partitioning. The source connector has previously accepted the initial partitioning request."
  3. Check the target setup and observe the error: "invalid dataset folder: the data folder is not empty."

Cause

The target table is not empty, and the "Delete All Before Loading" option is not enabled.

As explained in SAP Help:  Configure a Replication Flow | SAP Help Portal 

  • For cloud storage provider targets, Delete All Before Loading must always be set. (If you still try to run a replication flow for an existing target without the Delete All Before Loading option, you get an error message.) When you start the replication run, the system deletes the object completely (data and structure) and re-creates it based on the source data.

Resolution

1. Ensure that the "Delete All Before Loading" option is enabled in the replication flow configuration.

2. If the "Delete All Before Loading" option is not enabled, you can also manually remove all files from the target data folder or choose a different empty target folder.
3. Retest the replication flow after completing the above steps.

Symptom

What happens to the email reminders when we change the start date (due date) during the onboarding process? 

Will the email reminder due date be automatically updated if the start date (due date) is changed? 

Sample Scenario:
When an employee is terminated, the task due date is set to the termination date, and the same date is reflected as the reminder due date in the EmailReminderState Object. However, when the employee’s termination date is subsequently updated, the offboarding process and the associated task due date are correctly recalculated and updated. In contrast, the reminder due date in the EmailReminderState object continues to retain the original termination date and does not get updated. 

Environment

SAP SuccessFactors Onboarding

Resolution

This is expected system behaviour. When the start date or termination date is updated during the hire cycle for any candidate, it will not modify the due date which is set in EmailReminderState Object. Hence, the due date will not get updated automatically.

Workaround:

  1. Go to Manage Data > EmailReminderState
  2. Filter using the Activity id [This can be found in Email Services > Email Status > Find the email template and under Action section, click on details]
  3. Update the dueDate
  4. Click on Save

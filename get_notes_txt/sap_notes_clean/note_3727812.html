Symptom

  • After upgrading SAP Adaptive Server Enterprise  to SAP ASE 16.0 SP04 PL08, customers observe different results when executing the DATEDIFF() built-in function using the hh datepart.

          Example:

      SELECT DATEDIFF(hh,'09:07','09:37')

         Returned result:

         1

         Expected result based on historical ASE behavior:

             0

  • The results differ from the historical ASE implementation where DATEDIFF() counted the number of datepart boundaries crossed.
  • This behavior change may impact existing applications that rely on the previous semantics.

Environment

SAP  ASE 16.0 SP04 PL08

Cause

  • Change Request CR 829885 modified the internal implementation of the DATEDIFF() built-in function.
  • The CR 829885 introduced a new behavior where we modified ASE built-in function datediff() to round the diff value instead of truncation.

Resolution

Currently there is no configuration parameter or trace flag available to restore the previous DATEDIFF() behavior.

A new Change Request may be evaluated to introduce a trace flag or configuration option to revert to the previous behavior.

831874 Revert behavior of datediff function to truncate value instead rounding it.

Keywords

datediff, CR 829885, CR 831874, boundary crossing, rounding behavior,  datetime calculation

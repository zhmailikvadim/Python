Symptom

  • Commodity code (STAWN) maintained in the material master is not fetched into the sales order pricing access.
  • Commodity code field (STAWN) is available in the pricing communication structure and T681F, but the condition record is not fetched in the sales order.
  • Commodity code field is not initialized in the sales order pricing communication structure.

Environment

  • Sales And Distribution (SD)
  • Logistics Execution (LE)
  • SAP ERP Central Component
  • SAP ERP
  • SAP Enhancement package for SAP ERP
  • SAP Enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA
  • S/4HANA Cloud Private Edition

Reproducing the Issue

  1. Maintain the commodity code in the material master.
  2. Maintain VK11 for the condition type where the access is having the commodity code.
  3. Create a sales order.
  4. Navigate to the Item Condition tab in the sales order.
  5. Click on "Analysis" and double-click on the condition type.
  6. Observe that the system does not fetch the correct condition records for the commodity code maintained in the material master into the pricing communication structure.

    Cause

    • In standard SAP ERP functionality, the commodity code (STAWN) does not exist at the sales order item level (VBAP-STAWN).
    • Therefore, this field is not transferred or determined in the item pricing interface structure TKOMP during sales order creation.

    Resolution

    1. As a workaround, utilize the USEREXIT_PRICING_PREPARE_TKOMP available in include MV45AFZZ.
    2. Implement custom logic in USEREXIT_PRICING_PREPARE_TKOMP to fill the content of TKOMP-STAWN based on specific requirements.
    3. Test the implemented solution to ensure the commodity code is correctly fetched into the pricing communication structure during sales order creation.

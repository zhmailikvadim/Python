Symptom

  • When recalculating the plan in foreground after changing the minimum order quantity or allocated quantity, all stores are deleted.
  • The issue occurs during mass changes to minimum order quantity or when modifying allocated quantity for a single store.
  • No error messages or error codes are issued

Environment

SAP Allocation Management

Reproducing the Issue

  1. Create a regular push plan for a mixed display.
  2. Perform a mass change to the minimum order quantity (e.g. change from 1 to 2).
  3. Recalculate the plan in foreground.
  4. Observe that all stores are deleted after recalculation.
  5. Alternatively, modify the allocated quantity for a single store and recalculate the plan.
  6. Observe the same issue where all stores are deleted.

Resolution

  1. Implement the following SAP Notes:

    3557460 - Allocation Management 5.0 :  Stores are removed on plan recalculation for mixed display articles

    3709373 - Error while saving the push regular plan with multiple mixed articles

See Also

Refer to: SAP Note 3557460 and SAP Note 3709373

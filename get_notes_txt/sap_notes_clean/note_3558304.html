Symptom

System throws "SOAP:1027 SRT: Serialization / Deserialization failed" while submitting the eInvoice. 

Environment

  • SAP S/4Hana Public Cloud Edition

Reproducing the Issue

  1. Go to application edocument cockpit.
  2. Give source document number.
  3. Submit the edocument under einvoice node.
  4. Now in application log error "SOAP:1027 SRT: Serialization / Deserialization failed" is displayed.

Cause

GSP is not maintained correctly. 

Resolution

1. Refer the KBA 2906167 IN : eInvoice : "SOAP 1027 SRT: Serialization / Deserialization failed"

2. Ensure that the correct GSP name is maintained in the SSCUID 103588. System uses this information during GSP integration with source system through SAP Cloud Integration. Following values can be maintained under Service Provide Name for respective GSP's:

• For Vayana: Use vayana
• For Excellon: Use excellon
• For Alankit: Use alankit
• For MastersIndia: Use mastersindia
• For Adaequare: Use adaequaregsp
• For BDO: Use bdo

Note: Only SAP certified GSPs is supported. Refer KBA 2889709 - List of GSPs working with SAP for platform “Digital Compliance service for India” integration certification for more information on SAP certified GSPs.

Refer to section Maintaining Service Provider for eInvoice given in help portal documentation link.

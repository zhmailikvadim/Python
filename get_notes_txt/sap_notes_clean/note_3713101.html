Symptom

You are using Planning&Dispatching and encounter one of the followning problems:

  • When planning services or calculating a track you get messages like “Processing Channel 1: missing configuration (inbound destination)” . Reason: Missing authorizations in role /PLCP/PLANNING.
  • The calculated results of a scenario were incomplete: the track information was missing. Reason: Missing authorizations in role /PLCP/EXEC_SCENARIO.

This note changes the authority check for accessing the background processing configuration. It requests authorizations for field 'TABLE' as /PLCE/R_BGPROCESSINGCONFIG of authorization object S_TABU_NAM (instead of /PLCE/R_BGPROCESSINGCONF ; note the missing suffix IG).

Probably you need to adjust the authorization settings for your users. See also the "Solution" section below.

Other Terms

Planning&Dispatching, PLCE, PLCP, /PLCP/CONFIGURATION, /PLCP/EXEC_SCENARIO, /PLCP/PLANNING, /PLCE/R_BGPROCESSINGCONF, /PLCE/R_BGPROCESSINGCONFIG, /PLCE/BP_R_BGPROCESSINGCONF_S

Solution

Implement the corrections instruction.

If you are using the authorization roles delivered with the PLCP-AddOn, you can update it by uploading the roles that come as attachment with this note.

Save the appropriate file to your file system and remember its location.

File PFCG_ROLES_NOTE_3713101_101_758.SAP is valid for AddOn release PLCP 101_758.

File PFCG_ROLES_NOTE_3713101_102_816.SAP is valid for AddOn release PLCP 102_816.

Start transaction PFCG (Role Maintenance).

Choose menu entry Role -> Upload.

Open file PFCG_ROLES_NOTE_3713101_XXX_YYY.SAP  that corresponds to your system release.

A list of 3 roles to be imported is presented. Some roles have Status red light ("Caution"), some are green ("OK"). Red lighted roles exist and will be overwritten. That is the intended action. Go ahead by pressing Enter ("Transfer").

You should get the message that 3 roles were loaded from the file.

Leave the transaction.

If you need these roles in other clients or systems, you can use transactions PFCG or PFCGMASSTRANSPORT to transport all necessary /PLCP/*-roles (/PLCP/CONFIGURATION, /PLCP/EXEC_SCENARIO, /PLCP/PLANNING).

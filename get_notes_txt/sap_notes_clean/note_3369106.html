Symptom

  • There is a need to connect two or more SAP Cloud Connectors to Cloud application.
  • In SAP BTP cockpit under the subaccount, in the left hand sidebar under Connectivity > Cloud Connectors, the same Connector ID is being reported for 2 or more SAP Cloud Connectors
  • One SAP Cloud Connector is added as a system, but when subsequent SAP Cloud Connector(s) are attempted to be connected the error “ 400 – Technical system with system number <SAP Cloud Connector ID> already exists”.

Environment

  • SAP Cloud Connector.

Cause

This can happen when one of two scenarios are executed in the system:

  1. The SAP Cloud Connector was cloned and the clone is a copy of the original therefore they have the same Connector ID
  2. A backup was created in the first SAP Cloud Connector, and then used to restore the configuration of another SAP Cloud Connector, facilitating the transfer of all subaccounts between the two connectors (configuration has the Connector ID)

Resolution

These two scenarios are not supported by the SAP Cloud Connector and will cause this issue with the same Connector ID being reported for both SAP Cloud Connectors.

Each SAP Cloud Connector must have a unique Connector ID.

  1. To fix this problem one (or possibly more) SAP Cloud Connector systems that report the same Connector ID will need to be deleted, so that only one SAP Cloud Connector with that Connector ID exists.
  2. The SAP Cloud Connectors that reported a duplicate Connector ID would need to be recreated from the start.

This way the possibility of two or more SAP Cloud Connectors having the same Connector ID inside the system is eliminated and in doing so each SAP Cloud Connector will report a unique Connector ID.

Symptom

When I post a Goods Issue for a subcontracting Purchase Order (PO) from my SAP S/4HANA system, the transaction fails in the Managed Gateway for Spend&Network. I see the following errors:

  • Schema Validation failed for cXML‑0000‑ReceiptRequest. Suggested Action: Please check the attachment for error details. For implementation guidelines, refer to the cXML specifications in Integration Portal → Resources → Implementation Guides. ErrorCode: CIG‑PLT‑00623.
  • The value '' of attribute 'type' on element 'ReceiptItem' is not valid with respect to its type.

Environment

  • Managed Gateway for Spend&Network
  • SAP S/4HANA

Cause

You have not maintained the movement type (e.g., Goods issue for subcontract order) in the add-on settings for Receipt Requests in your SAP S/4HANA system. As a result, the system does not populate the type attribute in the ReceiptItem element of the outgoing cXML document, causing a schema validation failure in the Managed Gateway for Spend&Network.

Resolution

To resolve this issue, you need to maintain the movement type (for example 541) in the configuration for Receipt Requests in your SAP S/4HANA system.

  1. In your SAP S/4HANA system, run the transaction code SPRO.
  2. Navigate to the path: Integration with Other SAP Components > Managed Gateway for Spend&Network for Buyer > SAP Business Network Integration > Application Specific Settings > Receipt Request.
  3. Click the IMG-Activity for Maintain Parameters for Receipt Request.
  4. On the Change View “Allowed ReceiptRequest Messages”: Overview screen, click New Entries.

Now you should proceed:

  1. Add a new entry with the following details:
    • Movement Type: 541
    • Stock Type: Select the appropriate stock type for this movement (for example, Unrestricted Use). You must confirm this aligns with your business process.
  2. Click Save and assign the change to a transport request.

After you complete this configuration, you can re-trigger the Goods Issue process, and the transaction will process successfully.

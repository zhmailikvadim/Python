Symptom

  • During warehouse physical inventory at WM level in the “Analyze Differences” app, there is no field to enter a reason for difference/reason for movement.
  • The reason field is available in the “Manage Physical Inventory Document” app (IM-managed), but not in the WM-level “Analyze Differences” process.
  • Error message appears when posting differences if the IM movement type requires a reason: “M7325 A reason has to be entered for movement type 712”.

Environment

 SAP S/4HANA Cloud Public Edition

Reproducing the Issue

  1. Create a physical inventory document.
  2. Enter count results.
  3. Confirm count results at the WM level.
  4. In the “Analyze Differences” app, attempt to post differences (with movement types such as 711/712 configured to require a reason in IM).

Cause

  • In standard, the EWM “Analyze Differences” posting does not provide a reason for movement for Difference Analyzer postings, and there is no EWM-side customizing to enable a reason input field. When IM movement types (for example 711/712) are configured as “reason required” in SSCUI 103222, posting fails with error M7325 because no reason can be supplied from EWM.

Resolution

In standard there is no reason for movement maintained for Difference Analyzer postings, there is no customizing to influence this on EWM side.
So currently, there is a Workaround to this issue: To disable the mandatory requirement for the reason for movement.

Change the control for the reason for movement in Customizing so that the reason is not a required entry field for the movement type. To do this, choose the following path

Configure Your Solution> Logistics> Inventory> Goods Movement> Reason for Movement

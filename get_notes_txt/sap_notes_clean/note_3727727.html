Symptom

You observe the decimal values being incorrect in the case of KE30 reports with float fields more than 15 digits.

Environment

  • Controlling and Profitability Analysis
  • SAP R/3
  • SAP R/3 Enterprise 
  • SAP ERP Central Component
  • SAP ERP
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA Finance
  • SAP S/4HANA

Reproducing the Issue

  1. Access the KE30 transaction.
  2. Execute the report with huge amounts of line items (for example in the case of reporting on multiple periods)
  3. Observe the decimal values being incorrect lines in the report output.

Cause

  • The reason for the issue is a rounding error due to some technical restriction of the drilldown reporting tool in KE30, If the report selects a huge amount of data that need to be summarized into the report output.
  • The KE30 drilldown reporting tool uses float fields for storage of the data and for calculation of the output. The float fields can store up to 17 digits for the values. Due to the huge amounts (up to 15 digits are used for the output) and the huge number of selected data lines this can lead to such rounding issues

Resolution

To avoid such rounding issues, to get better performance and to reduce the memory usage during execution of the KE30 report 
it is highly recommended to switch the performance settings (in option tab of KE32) to "Read upon each navigation step" for the specific report. 

  1. Call transaction KE32, double click the specific report, go to the 'Options' and adjust the settings
  2. Switch Reactions if no presummarized data exists to "Execute report" and switch Performance settings to "Read upon each navigation step"
  3. Save the changes.
  4. Execute the report again in KE30 to verify the issue is resolved.

Keywords

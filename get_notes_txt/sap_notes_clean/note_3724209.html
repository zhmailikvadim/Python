Symptom

 

When creating a Legacy Data Transfer (LDT) segment in transaction FAA_CMP or FAA_CMP_LDT, the system raises the following error:

AC187
Transfer date & is more than one year after the closed fiscal year.

Additional observations:

  • The transfer date is within the fiscal year following the last closed fiscal year
    (e.g. last closed fiscal year = 2025, transfer date = 30.04.2026).

  • The LDT segment is not created.

  • Field DATUM remains empty.

  • Tables T093B and T093C appear to contain consistent fiscal year information.

In some cases, the issue may also occur together with:

  • Error FAA_CMP030

  • Errors during fiscal year closing (AJAB)

Environment

  • FIN Asset Accounting
  • SAP ERP
  • SAP ERP Central Component
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA
  • SAP S/4HANA Finance
  • SAP S/4HANA Cloud Private Edition
  • SAP Fiori for SAP S/4HANA Finance,
  • SAP Fiori for SAP S/4HANA,
  • SAP Fiori for SAP S/4HANA Cloud Private Edition

Reproducing the Issue

  1. Open transaction FAA_CMP_LDT (or FAA_CMP)

  2. Switch to edit mode

  3. Create a new LDT segment

  4. Enter a transfer date in the fiscal year following the last closed fiscal year

  5. Save

Result: Error AC187 is raised and the segment is not created.

Cause

The validation for error AC187 during creation of a Legacy Data Transfer segment does not rely solely on the value of T093B-ABGJA (last closed fiscal year).

Instead, the system evaluates the overall fiscal year status in Asset Accounting, including:

  • Legacy Data Transfer configuration (FAA_CMP / FAA_CMP_LDT)

  • Fiscal year closing status in Asset Accounting

  • Ledger-specific fiscal year status

  • Migration-related system status

If these elements are not fully aligned, the system may raise error AC187 even though fiscal year data in T093B and T093C appears consistent.

Resolution

Resolution

Ensure that the fiscal year status is consistent across all relevant components:

  1. Legacy Data Transfer (FAA_CMP / FAA_CMP_LDT)

    • Verify the LDT segment status

    • Check the transfer date

    • Ensure no inconsistent or incomplete segments exist

  2. Table T093B

    • Field ABGJA (last closed fiscal year)

  3. Table T093C

    • Field LGJAHR

  4. Asset Accounting closing status

    • Execute/check AJAB (fiscal year closing)

    • Ensure depreciation posting (AFAB) is completed for all periods

If inconsistencies are detected, align the fiscal year status before creating the Legacy Data Transfer segment.

How to Identify the Issue

This scenario typically has the following characteristics:

  • Error AC187 occurs during LDT segment creation

  • Transfer date appears valid (within next fiscal year)

  • LDT segment is not created and DATUM remains empty

  • Fiscal year data in T093B and T093C appears correct

=> In such cases, the issue is usually caused by inconsistent fiscal year status, not by an invalid transfer date.

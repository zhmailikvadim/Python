Symptom

In the Japanese environment, the time type field in the custom MDF is displayed incorrectly in the latest people profile, showing one minute more than in other languages.

Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.  

Environment

SAP SuccessFactors Employee Profile - Latest People Profile

Reproducing the Issue

1.login Instance

2.View the time type field in the latest profile in both Japanese and English environments.

3. The displayed time is inconsistent.

Cause

This is a Known Issue. 

COMMON_TimeFormat should be HH:mm:ss not HH:MM:ss 

Resolution

The fix for this issue is planned to be deployed on 1H 2026 release.

For release timelines, please refer to SAP SuccessFactors Product Release & Road Map Information.

As a workaround, you can manually add a custom translation in Manage Languages to change the value of COMMON_TimeFormat to=HH:mm:ss

For more details, please refer to the guide：Modifying UI Text in Manage Languages - Overview | SAP Help Portal

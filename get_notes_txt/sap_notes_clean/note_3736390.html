Symptom

  • When posting asset retirement via BAPI_FIXEDASSET_OVRTAKE_POST, error AA660 “Transaction in area & contradicts the net book value rule” appears.
  • Error AA594 “Errors occurred for asset & in fiscal year &” is displayed alongside AA660.

Environment

  • FIN Asset Accounting
  • SAP R/3
  • SAP ERP Central Component
  • SAP ERP
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA Finance
  • SAP S/4HANA
  • SAP S/4HANA Cloud Private Edition

Reproducing the Issue

  1. In SE37, run a test sequence with BAPI_FIXEDASSET_OVRTAKE_POST followed by BAPI_TRANSACTION_COMMIT to post an asset retirement.
  2. Insert data in following tables: KEY, TESTRUN, CUMULATEDVALUES, POSTEDVALUES, POSTINGHEADERS, TRANSACTIONS, PROPORTIONALVALUES.
  3. Execute the sequence and observe that AA660 and AA594 are raised.

Cause

  • An incorrect value is provided in PROPORTIONALVALUES-ORD_DEP_CY (current year ordinary depreciation). This causes the depreciation calculation in the relevant depreciation area to exceed the proportional amount up to the retirement date, resulting in a negative net book value and triggering AA660; AA594 is shown in addition.

Resolution

  1. Review the depreciation key and period control customizing for retirement for the relevant depreciation area(s), considering the actual posting date, and calculate the current-year depreciation up to the retirement.
  2. In the BAPI call (BAPI_FIXEDASSET_OVRTAKE_POST), add one line in PROPORTIONALVALUES for each relevant depreciation area.
  3. For each line, set ORD_DEP_CY to the calculated current-year depreciation up to the retirement.
  4. Repost the retirement using BAPI_FIXEDASSET_OVRTAKE_POST followed by BAPI_TRANSACTION_COMMIT and verify that errors AA660 and AA594 no longer occur.

Symptom

In the Inbound when you receive a temporary error in the NF-e/ CT-e Status Check response (e.g. 108, 109, 999) these status code are not overriding the last stored SEFAZ status code. The corresponding process step is set to error and the user can repeat the action afterwards to receive a final status code by the government.

Other Terms

SAP GRC, Nota Fiscal Electronica, Inbound, NF-e Status Check, CT-e Status Check

Reason and Prerequisites

The program behavior has been changed.

Solution

Implement Support Package 21 for this feature or as a predelivery implement this note.

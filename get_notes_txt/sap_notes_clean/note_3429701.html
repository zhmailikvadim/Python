Symptom

The way the autorenewal process functions in S/4HANA is still unclear and the items are not automatically renewed

Environment

SAP S/4HANA

Cause

AutoRenewal is not enable in the following  cases:

  • Missing setting in the product master data
  • If the service contract item is for ad-hoc billing
  • If any service contract items contain errors
  • If the service contract item is not released
  • If the cancellation date is set for the service contract item

Resolution

The purpose of this KBA is to provide additional information and screenshots to assist customers with this configuration. The SAP help documentation thoroughly explains the process of autorenewal for service contract items. Please check "Autorenewal of Service Contract items" 

  • Customizing settings:


check the documentation linked to customizing Configure Change Process  -> Settings for the Maintenance Framework (BTMF) ->

SC_ITEM_RENEW must be defined.

  • Product Master settings (fiori app Product Master or backend transaction MM02)

Material master offers the field "Contract autorenewal indicator" with the following values. Values Blank and 1 allow  the activation of the Autorenewal process in service contract item.

  • Service items

Depending on the value of AutoRenewal indicator you will see the following value in the Item Detail. If the value is blank, the indicator can be updated as the below picture. If the value is 1, this field appears marked by default (second screen shot) with the default values in periods Contract Extension and Renewal Period. Finally, if the value is 2 this indicator is not displayed.

  • Renewal report :

S/4HANA Service On Premise uses the report CRMS4_SERVICE_CONTRACT_RENEWAL that must be executed in the backend. There is not currently any FIORI app created for this feature. Literally, the program will process all service contract items whose contract end is found within the auto renewal period that has been defined for the service contract item.

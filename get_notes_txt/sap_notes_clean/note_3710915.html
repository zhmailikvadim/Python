Symptom

You have created a Validation or Post Hook external hook that raises error messages. These messages appear even when simply opening an Account, without any attempt to edit it.

Environment

  • SAP Sales Cloud Version 2
  • SAP Service Cloud Version 2

Reproducing the Issue

1. Click on the User Icon.
2. Go to the Settings > All Settings. 
3. Go to the Extensibility Administration.
4. Select Account Service > Account.
5. Go to the Validations tab.
6. Create a Validation.

7. Go to the Account Object Worklist (OWL).
8. Open an Account.
9. The error message is already triggered.

Cause

Custom validations for Account now execute when the Account Details screen is opened.

Since the validation blocks editing without distinguishing between read and write operations, it runs on opening the Account and displays the error message—even if no changes were made.

Resolution

To prevent the error message from appearing when only opening the Account Details screen, modify the Validation rule by adding an AND condition with the following expression:

  • Previous Value: adminData/updatedOn (Changed On)
  • Condition: Is Not Equal To adminData/updatedOn (Changed On)

                                                                                    

Then, save and test by opening one of those defined Accounts (should not show the error) and then attempting an edit (should block).

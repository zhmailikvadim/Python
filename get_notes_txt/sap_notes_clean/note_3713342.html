Symptom

When querying a virtual table using Smart Data Access (SDA) or Smart Data Integration (SDI), you may encounter an error message such as:

 

HANA logs

Remote Source [<RemoteSource>] is not accessible to perform the operation: failed to parse user information from the credential

 

Application logs

C  SQLCODE    : 403
C  SQLERRTEXT : internal error: failed to parse user information from the credential

Other Terms

Federation; SDA; Smart Data Access; SDI; Smart Data Integration; Security; Credentials;

Reason and Prerequisites

Issue number 355224

Reason:

Possible causes include missing secondary or technical user credentials, active remote connections that remain after the remote source is modified, incorrect credential patterns, or other configuration issues.

Affected Releases:

  • SAP HANA 2:
    • All Revisions

Prerequisites:

  1. Smart Data Access or Smart Data Integration is utilized.

Solution

To resolve this issue, please review the following known root causes and ensure that each one has been addressed.

 

(1) Missing Credentials / Secondary Credentials

Ensure that either a technical user or appropriate secondary credentials are properly assigned:

  • SELECT * FROM SYS.CREDENTIALS WHERE PURPOSE = '<RemoteSource>';

The default output, when the same technical user credentials are used for all users, is shown below. This is indicated by the value "NULL" in the "USER_NAME" column:

USER_NAME,COMPONENT,PURPOSE,CREDENTIAL_TYPE,PSE_NAME
NULL,SAPHANAFEDERATION,<RemoteSource>,PASSWORD,NULL

If the column contains a specific user name (or multiple rows with different user names), indicates that secondary credentials are used. If the user you are using to access the remote source is not listed,

it means that remote credentials for that user are missing. If the column displays only a question mark ("?"), this indicates that no technical user credentials are assigned at all. Hence not a single user can access the remote source.

  • For more information, please refer to the "Managing Secondary Credentials" chapter in the SAP HANA Administration Guide for the SAP HANA Platform.

 

(2) Altered Remote Source

If the remote source was modified at any point—for example, if it was previously used with SDA and later switched to an SDI adapter—there may still be old sessions active with outdated or invalid metadata, such as user credentials.

  1. Check if there are any active connections that are using different metadata for the corresponding remote source:
    • SELECT * FROM M_REMOTE_CONNECTIONS WHERE REMOTE_SOURCE_NAME = '<RemoteSource>';
  2. Close these old connections, preferably from the ABAP side. If that is not possible, you can also terminate the sessions directly at the HANA level by executing the following SQL statement, as described in KBA 2092196:
    • ALTER SYSTEM DISCONNECT SESSION '<Connection_ID>';
    • Replace <Connection_ID> with the actual session ID you want to terminate.

 

(3) Incorrect Credentials Pattern

Ensure that credentials are created following the pattern shown below, as specified in the SAP HANA Administration Guide. Use only this sequence when creating credentials or configuring a remote source:

  • "user=testuser;password=mypassword"

 

Remarks:

  • There may be additional causes other than those mentioned above.

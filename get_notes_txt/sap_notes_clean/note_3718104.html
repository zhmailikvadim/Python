Symptom

Clarification on which data and configurations are overwritten and which are preserved in the Test system during a Test Data Refresh (TDR) from Production. There is uncertainty whether configuration, integrations, users, workflows, or documents will be deleted or replaced.

Environment

SAP S/4HANA Cloud Public Edition

Cause

Test Data Refresh copies business data from Production to Test. However, not all configuration, technical content, and tenant-specific objects are included in the transfer.

Resolution

Brief view on the Information note is given below.

Data Preserved in Test as it is (Not Overwritten)

Category

Examples

Configuration

SSCUI/Configuration activity settings (Configuration data), communication arrangements etc.

Extensibility

Custom fields, custom logic, custom business objects etc.

Integration content

IDoc/EDI, AIF data, integration logs etc.

Documents

GOS attachments, DMS documents, SAP Office etc.

Technical logs

Application logs, change documents etc.

Users & HR

Business users, employees, contingent workers etc.

Workflows

Workflow definitions and runtime items etc.

Other

Chart of accounts config, expense reports etc.

Data Overwritten in Test (Copied from Production)

Category

Examples

Business data   

Business partners, materials, customers, vendors etc. 

Transactional data

FI documents, sales orders, purchase orders etc.              

Master data        

GL balances, open items, pricing data etc.                       

Operational data   

Most application tables used for business processing etc.

Note: End user can see the complete list of tables in scope of TDR in the readiness report pdf.

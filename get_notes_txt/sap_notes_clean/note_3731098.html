Symptom

The field name displays with a duplicate percentage symbol (%%) instead of a single %.

  • Duplicate Symbols in Field Name
    • The field name displays %% instead of a single % when the field type is set to Percentage and the name already includes a %.
    • Example: Field named %Override Payout Matrix appears as &%Override Payout Matrix
  • Inconsistent Display After Renaming
    • Renaming a standard or custom percentage field may cause the duplicate symbol to appear until the type or label is adjusted.
  • Exported Reports Show Extra %
    • Exports of the field may also display %%, leading to confusion in reports or dashboards.
  • Behavior Varies by Field Type
    • When the field type is changed from Percentage to Amount, the duplicate symbol disappears, and only a single % appears if included manually.

Environment

  • Product: SAP SuccessFactors Compensation
  • Hierarchy based approvals
  • Executive Review
  • Variable Pay

Reproducing the Issue

  • Create a custom field or rename an existing standard field and include a % in the field name, for example %Override Payout Matrix
  • Set the field type to Percentage.
  • Save the field.
  • Observe that the field name now displays with a duplicate symbol, such as %%Override Payout Matrix, both in the UI and in any exported reports.
  • Optional: Change the field type to Amount or remove the % from the field name. The duplicate symbol disappears, and the field displays correctly with a single % if needed.

Cause

When the field is configured with the Percentage unit and the field label already includes a "%", the Executive Review export adds another "%", resulting in "%%".

Resolution

  1. If the field is configured with Percentage:
  2. 1 Rename the field to remove the "%" from the label (for example, use "Override Payout Matrix").
  3. 2 Export the Executive Review to confirm the label shows a single "%", added automatically by the Percentage unit.
  4. If the field is configured with Amount:
  5. Keep the "%" in the field label (for example, use "%Override Payout Matrix").
  6. Export the Executive Review to confirm the label shows a single "%".
  7. Avoid including a "%" in the label of fields that use the Percentage unit to prevent duplication in exports.

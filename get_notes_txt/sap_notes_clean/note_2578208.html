Symptom

Current behaviour in detail

Up to SAP Lumira Designer release 2.1 SP0 the behaviour is as follows:

1. It is not possible to delete values from planning enabled cells.

Instead the previous value is restored.

This is true both for planning values on key figures and for planning values on dimensions in new lines.

2. The user is unable to detect if a new line is in place where planning values is possible.

The user has to first add a value to a new line. Only then will the value help icons on dimension appear.

Other Terms

Design Studio, Lumira, Spreadsheet, Planning

Reason and Prerequisites

Program error

Solution

Behaviour after the correction

1. It will now be possible to delete values from planning cells.

2. There will always be a new line available, provided the query view allows new lines and the query is set to planning mode.

This way the user will be aware that an input in this line is possible.

The value help icons will also be available upon hover over a cell.

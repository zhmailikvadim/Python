Symptom

Deployment with data migration fails at the "Migrate data" step.

Error logs are observed:

"Impex import failed for : '0000KQ5U-ImpEx-Import(8827780563445)'","message":"Impex import failed for : '0000KQ5U-ImpEx-Import(8827780563445)'"
"Can not resolve any more lines ... Aborting further passes (at pass 2). Finally could not import 1 lines!"
"line 3 at main script: dumped unresolved line ValueLine[unresolvable:Exception : line 34: cannot update 8796093061591 with values ItemAttributeMap[ registry: null, type: <null>, data: {template=true, destinationchannel=DestinationChannel.DEFAULT(DEFAULT(8799691407451))} ] due to [de.hybris.platform.apiregistryservices.interceptors.DestinationTargetValidateInterceptor@66b5dba7]:Changing the destination channel value of the destination target is not allowed, Exception : line 3: cannot update 8796093061591 with values ItemAttributeMap[ registry: null, type: <null>, data: {template=true, destinationchannel=DestinationChannel.DEFAULT(DEFAULT(8799691407451))} ] due to [de.hybris.platform.apiregistryservices.interceptors.DestinationTargetValidateInterceptor@66b5dba7]:Changing the destination channel value of the destination target is not allowed,line 3 at main script,null,HeaderDescriptor[line 2 at main script, insert_update, DestinationTarget, {}, [id, destinationChannel, template] ],{1=ValueEntry('Default_Template'=Default_Template,unresolved=false,ignore=false), 2=ValueEntry(''=DestinationChannel.DEFAULT(DEFAULT(8799691407451)),unresolved=false,ignore=false), 3=ValueEntry('true'=true,unresolved=false,ignore=false)}]"

Environment

SAP Commerce Cloud in the Public Cloud

Reproducing the Issue

  1. Initiate a deployment process.
  2. Select the 'migrate data' mode for deployment.
  3. Observe that the deployment fails at the migrate data step.

Cause

Data inconsistency in DestinationTarget: the destinationChannel attribute of DestinationTarget 'Default_Template' points to a non-existent EnumerationValue.

Execute below flexible search in HAC and it returns 'No data available in table'

SELECT {ev.pk}
FROM {EnumerationValue AS ev}
WHERE {ev.pk} IN (
    {{ SELECT {dt.destinationchannel}
       FROM {DestinationTarget AS dt}
       WHERE {dt.id} = 'Default_Template' }}
)

The below ImpEx tries to insert/update the DestinationTarget and set destinationChannel to DEFAULT. DestinationTargetValidateInterceptor forbids changing destinationChannel on an existing DestinationTarget. With the original destinationChannel referencing a missing EnumerationValue, the comparison detects a change when the ImpEx sets DEFAULT and the interceptor throws “Changing the destination channel value of the destination target is not allowed,” causing the ImpEx (and the migrate data step) to fail.

INSERT_UPDATE DestinationTarget;id[unique=true];destinationChannel(code)[default=DEFAULT];template
;$destination_target;;true

The related code segment:

private void validateDestinationChannel(final DestinationTargetModel destinationTarget) throws DestinationTargetValidationException
{
final DestinationChannel originalValueOfDestinationChannel = destinationTarget.getItemModelContext()
.getOriginalValue(
DestinationTargetModel.DESTINATIONCHANNEL);
if (originalValueOfDestinationChannel != destinationTarget.getDestinationChannel())
{
throw new DestinationTargetValidationException(CHANGING_CHANNEL_ERROR_MESSAGE);
}
}

Resolution

  1. delete DestinationTarget record of Default_Template from backoffice.
  2. reinitiate a deployment process.

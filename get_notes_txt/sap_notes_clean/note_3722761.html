Symptom

  • Scheduling status: deallocated for Manufacturing orders in APO or ePPDS converted from planned orders in R/3 or S/4HANA
  • The operations of the Manufacturing order are marked as unplanned, and the order is not scheduled on the resource after order conversion with S4HANA or R3 tools.

Environment

  • SAP Supply Chain Management
  • SAP S/4HANA
  • APO PPDS or S4HANA ePPDS / Production and Process Order

Reproducing the Issue

  1. Create a planned order with scheduling status: Scheduled in APO or ePPDS, for example with /SAPAPO/RRP3.
  2. Convert the planned order into manufacturing order using transaction MD04, CO02 or COR2 in S/4HANA.
  3. Display the production order in /SAPAPO/RRP2 or its operations in /SAPAPO/CDPS0.
  4. Confirm that the production order and its operations are marked as unplanned.

Cause

It is expected in standard that Manufacturing orders will become deallocated after conversion with R/3 or S/4HANA tools in APO or ePPDS due to the two following factors:

  • Planned orders do not have operations in R/3 or S4HANA, therefore it is not known during conversion there if the operations were scheduled in APO or ePPDS or not.
  • The scheduling status parameter of the PP Planning Procedure event 'Create or Change Production Order in OLTP System' in APO or ePPDS is relevant only after product or quantity related change of an already existing Manufacturing order.

Resolution

SAP recommends to convert Planned Orders with APO or ePPDS tools in the integrated scenario. For different business requirement or having different outcome, refer the below details: 

  • F1 Help of the parameter 'Scheduling Status' in assigned events of a planning procedure in the SPRO Customizing setting for Maintain Planning Procedures can influence the scheduling status after certain events, it says:

"Scheduling Status of an In-House Production Order
Use
In this field, you specify

the scheduling status of a planned order that is recreated in PP/DS, or transferred from an OLTP system
The scheduling status of an in-house production order in PP/DS after a product or quantity related change to the receipt element"

  • in addition the following documents provide more details about the expected way how the system works:

SAP Note 456756 - R/3->APO: Scheduling status for internal production orders
SAP Note 304510 - APO integration and planned order conversion in R/3

  • One possibility - from the references in the above SAP Notes - which can be used as a base for implementing customer specific logic for order integration is explained in:

SAP Note 370959 - Always schedule operations of R/3 orders

  • In specific software releases on specific Service Packs levels, an SAP Note was creating a side effect on the Standard logic, which was corrected after discovery. The following KBA explains the details:

KBA 3441008 - Scheduling status of planned order 'Deallocated' changes to 'scheduled' in /SAPAPO/RRP3 when Customer use the "send to APO" function 

which refers to:

SAP Note with side effect: 2914165 - Content of field 'Scheduling Status' in the Planning Procedure is ignored during order integration - Part II
Correction SAP Note: 3196656 - During production order creation the value of the field 'Scheduling Status' from the planning procedure is used to set the scheduling status of the order

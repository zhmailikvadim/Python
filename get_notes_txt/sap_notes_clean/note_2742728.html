Symptom

In Analysis Office you add a data provider of a HANA system. On this data provider you open the Member Selector on a characteristic of type "DATE" and search for a date. The search does not return any results although the searched date should be available. When putting the date in quotation marks, the Member Selector will return the searched date.

So, for example a search for the date 01.01.2019 will not any result. But a search for "01.01.2019" returns the searched date.

Other Terms

AO; Analysis Office; Member Selector; Filter By Member; HANA; DATE; Search;

Reason and Prerequisites

Program error

Solution

Temporary Workaround:

Please put the searched date in quotation marks. For example: "01.01.2019"

Correction:

After installing the correction it won't be necessary anymore to put the searched date in quotation marks when searching on a characteristic of type "DATE" of a HANA data provider.

The correction will be delivered by a SAP Analysis for Microsoft Office Support Package.
In section "SP Patch Level" you can find information which Support Package contains the respective correction.

If the section "SP Patch Level" is not available or the information about which Support Package contains the correction is missing, this means that this has not been determined, yet. In this case please re-check this note at a later point of time.

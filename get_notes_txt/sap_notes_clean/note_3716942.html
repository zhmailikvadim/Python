Symptom

You use the report RPT_OVT_CONVERT_DRAFT to convert overtime requests in status DRAFT. You encounter any of the following issues:

  • Your system contains an overtime request with multiple versions. The current version is in status DRAFT, and one or more items were deleted between the previous version and the current version. After the successful conversion of this overtime request, the request owner opens it in the Fiori 3.0 app My Overtime Requests, and clicks the "Send" button. The deleted items reappear.
  • You are viewing the report log. If some overtime requests could not be processed because they were locked, the statistics in the log are incorrect: the locked records erroneously count towards the number of records with errors.
  • You are viewing the report log. Successfully converted requests remain locked by the report until you exit the log.

Other Terms

My Overtime Requests, Fiori, F4939A

Reason and Prerequisites

Program error.

Solution

Implement the attached correction instructions or import the relevant Support Package.

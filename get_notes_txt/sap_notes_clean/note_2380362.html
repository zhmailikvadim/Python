Symptom

The Registration of Innovaphone SIP phones fails when TLS is used.

Environment

SAP Contact Center

Reproducing the Issue

  • Your business organization requires the usage of Transport LayerSecurity (TLS) between SAP Contact Center and Innovaphone SIP phones.
  • The server certificate or issuer of it must be trusted by the phone.
  • The host name in the server certificate must match the SIP Proxy name.

Cause

NA

Resolution

  • If neither the issuer of the server certificate or the server certificate itself is trusted by the phone then the TLS handshake will fail and the following error message will be written to the SIP Bridge log file:
    • ERR> SecureContext::ServerHandshake: AccecptSecurityContext failed due to error [0x80090325]
    • The phone's ‘admin web’ will show the untrusted certificates it has received. Select the respective certificate and allow the phone to now trust it.
  • If the server certificate is trusted by the phone either by the issuer or by itself but the configured SIP Proxy value does not match the host name in the certificate (e.g. the certificate contains a DNS name and SIP Proxy field contains an IP-address) then the following error message will be written to the SIP Bridge log file:
    • ERR> SecureContext::Decrypt: DecryptMessage failed due to SEC_I_CONTEXT_EXPIRED
  • You must configure the SIP Proxy field to match the host name in the certificate. If needed, a local DNS A record can be configured on the phone using the web admin pages.

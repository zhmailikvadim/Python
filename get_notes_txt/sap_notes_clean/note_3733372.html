Symptom

The processing of XML documents fails within:

  • The CALL TRANSFORMATION statement with runtime error XSLT_BAD_SOURCE_CONTEXT and short text "XSLT: No valid XML source" and/or exception CX_XSLT_RUNTIME_ERROR with short text "No valid source context supplied" and kernel message "iXML configuration has prohibited the usage of DTDs" in exception attribute S2.
  • The method PARSE( ) of class CL_IXML_PARSER/interface IF_IXML_PARSER with return code <> 0 and NUM_ERRORS( ) > 0, GET_ERROR( INDEX = 0 )->GET_REASON( ) returns "iXML configuration has prohibited the usage of DTDs".

Related error details in SM21 (Syslog): A35 "iXML: DTD parsing attempted but forbidden by configuration"

Other Terms

DTD, Document Type Definition, iXML library, CALL TRANSFORMATION

Reason and Prerequisites

The profile parameter ixml/dtd_restriction is set to prohibited, e.g. in SAP Enterprise Cloud Services AS ABAP systems according to SAP Note 3250501.

The XML document contains a DTD, e.g. <!DOCTYPE foo> or <!DOCTYPE foo SYSTEM_ID "foo.dtd">.

Refer to SAP Note 1594475 for details about possible DoS attacks in programs using iXML.

Solution

The recommended solution is to allow the usage of DTDs locally in the affected application.

If the application is custom code: Change your custom code as described SAP Note 1712860 examples 3, 4, or 5. When using CALL TRANSFORMATION pass the IF_IXML_ISTREAM instance created as described in SAP Note 1712860 example 3 as XML source.

If the application is SAP Standard code: Check the application component for available corrections, e.g. SAP Note 3580805 for component CA-WUI-UI.

If this is not possible or there is no correction available, you might consider to change the setting of profile parameter ixml/dtd_restriction to the kernel default expansion together with ixml/xml_expansion_factor = 10, if permitted by relevant security policies.

Symptom

When trying to add a custom party role created to Appointments Party Schema, the same is not available to be selected.

Environment

  • SAP Sales Cloud Version 2
  • SAP Sales and Service Cloud Version 2

Reproducing the Issue

  1. Click on the User Menu.
  2. Click Settings.
  3. Navigate to All Settings.
  4. Click Party Roles.
  5. Create a new custom party role. 
  6. Save the changes.
  7. Search for Appointments.
  8. Open Party Schema. 
  9. Select any schema.
  10. Click Edit.
  11. Click on the + icon to add a new custom party role.
  12. See the party created is not displayed.

Cause

The custom party role was created using a Category that is not supported within the Appointments party schema. As a result, it cannot be selected. 

Resolution

Please find below the Categories that are currently supported for custom party roles in Appointments. When creating a custom party role, please make sure that one of the above categories are selected, so that the party will be available for selection. 

  • Custom Account Party
  • Custom Contact Party
  • Custom Employee Party

If you need any other Party Category for Appointments, please raise an Influence Request, where it would be possible to follow-up along with our Product Management team, for the updates on the development and inclusion of such functionality in the system: https://influence.sap.com/go/sales_service_v2_esm.

Symptom

  • Fiori Launchpad startup shows delays during loading of static resources and ui5 modules; multiple network roundtrips accumulate.
  • Browser trace/har and stats_fe indicate delays are not caused by abap backend processing but occur during the Launchpad startup phase.

Environment

  • Product: SAP S/4HANA all versions
  • SAP Fiori Launchpad ABAP Services

Reproducing the Issue

  1. Launch the Fiori Launchpad.
  2. Observe that the initial shell loads and then takes additional time while many UI5 modules and static resources are fetched.
  3. Confirm that the main time is spent on frontend resource loading rather than ABAP processing.

Cause

Standard FLP startup behavior that includes loading and initializing UI5 shell components and associated static resources can influence perceived startup time.

The observed pattern (significant ui5/static resource loading during flp startup) is standard product behavior.

Resolution

  1. Reduce the number of tiles and associated target mappings per user to minimize runtime resolution overhead during startup.
  2. Ensure recent performance optimizations are applied by implementing the corrections described in SAP Note 3467887 where applicable.
  3. Review and follow general flp performance recommendations and troubleshooting guidance to identify additional frontend optimizations, including analysis of resource requests and network latency impacts.
  4. If operating in high-latency environments, apply recommended best practices from the referenced guides to limit roundtrips and improve caching.

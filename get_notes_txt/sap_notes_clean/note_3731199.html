Symptom

If real-time overhead calculation is being performed for original costs with multiple ledgers, row numbering ZLENR is being incorrectly reset when nothing is calculated for a specific ledger in K_SURCHARGE_CALCULATE for specific object.

On releases 109 and newer, this correction was part of a development. This note serves as a downport of this developed part of the code to the older releases.

Other Terms

ZLENR, LEDNR, ledger, ledger-specific, real-time overhead, KZPR, RTOHC, BW_REFBZ

Reason and Prerequisites

Program error.

Solution

Implement this note or corresponding SP.

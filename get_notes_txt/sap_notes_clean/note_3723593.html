Symptom

  • The Planning Calendars App does not reflect the configured working/non-working days in the Key Figure (KF) per special periods defined.
  • Special Rules definition at IBP does not match the Special Definition at S4.

Environment

SAP Integrated Business Planning

Reproducing the Issue

  1. Configure working/non-working days via special rules at SCAL in S4.
  2. Run Real-Time Integration (Calendar, UoM, Currency) application job.
  3. Verify the Key Figure using IBP_CALENDAR.
  4. Observe that the configured working/non-working days are not reflected in the Key Figure correctly.

Cause

  • The Real-Time Integration (Calendar, UoM, Currency) job does not support delta deletion; it only adds or updates records in SAP IBP.
  • If the Erase Prev. Integrated Data flag is left unchecked, previously integrated entries remain in IBP and are not removed.
  • To remove entries that no longer exist in the source (e.g., deleted special periods), you must check Erase Prev. Integrated Data. This deletes prior integrated data and keeps only the newly integrated dataset in IBP.
  • Caution: There is no dependency check when deleting data via this option.

Resolution

Enable the 'Erase Prev. Integrated Data' option when running the 'Real-Time Integration (Calendar, UoM, Currency)' application job to ensure previously integrated data is deleted and only newly integrated data is available in SAP IBP.

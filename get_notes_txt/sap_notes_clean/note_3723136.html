Symptom

  • When executing the payment process, the system throws the following error message: XM122 – "Output device LPXY not defined"
  • The error occurs during execution of program IDFIPAYM_SCHED_AF_PAYM_BORD_OM.
  • This issue is specific to Brazil company codes.

Environment

  • Financial Accounting (FI)
  • SAP R/3 Enterprise 4.7
  • SAP ERP Central Component
  • SAP ERP
  • SAP Enhancement package for SAP ERP
  • SAP Enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA

Reproducing the Issue

  1. Execute a payment run in transaction code F110.
  2. After the payment run, the system automatically triggers program IDFIPAYM_SCHED_AF_PAYM_BORD_OM.
  3. The system issues error XM122 – "Output device LPXY not defined" in the logs.

Cause

  • Program IDFIPAYM_SCHED_AF_PAYM_BORD_OM is used for printing borderos after the payment run.
  • The report expects a printer named "LPXY" to be defined in the system.
  • If this printer is not maintained in the spool/output device configuration, the system throws error XM122.

Resolution

If bordero printing is not relevant for your business process, the message can be ignored. As seen in the job log, subsequent processing steps continue successfully.

If bordero printing is required, create printer LPXY by following the below steps: -

  1. Go to transaction SPAD.
  2. Press Enter on your keyboard
  3. Create a new output device named LPXY.
  4. Maintain the required printer settings as per the attachment titled "How to add printer LXPY" .
  5. Save the configuration.
  6. Re-test the payment run.

This will prevent the error message from appearing.

Symptom

If a DSN of nature 04 "Absence" is executed without executing a DSN of nature 01 "Monthly", the last workday in field S21.G00.60.002 might be wrong if the absence is preceded by another absence whose absence category is returning a absence reason code in feature FABS1. This happens only if the absence modifier MOABW is used as decision field in feature FABS1.

Other Terms

RPLDSNF0

Reason and Prerequisites

When the DSN of nature 04 is executed without the DSN of nature 01, a simulation of the payroll results is performed. Table WPBP is however not simulated for absences. As a consequence the absence modifier MOABW used to read feature FABS1 cannot be determined. The last workday of an absence is not shifted if the absence is preceded by another absence whose absence category is returning a absence reason code in feature FABS1.

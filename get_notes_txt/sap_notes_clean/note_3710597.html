Symptom

Difference in foreign currency (FC) amounts in a journal entry of purchase return.

Reproducing the issue

Prerequisites:

  1. In the General Settings define for Display Settings – Prices 4 Decimals.
  2. In the Currencies - Setup define 6 Decimals for USD.

Scenario:

  1. Create a Purchase Order for a Vendor with currency USD.
  • Rate = 31.49
  • First row: Item1, Quantity = 7, Unit Price = 113.300000 USD, Tax Code = Relevant code with 5%.
  • Second row: Item2, Quantity = 9, Unit Price = 353.444600 USD, Tax Code = Relevant code with 5%.
  • Modify the value in Total (Doc) in the second row from 3,181.001400 USD to 3,181.000000 USD.
  1. Create a Goods Receipt PO (GRPO) based on the Purchase Order.
  2. Copy the GRPO to Goods Return and add the document.
  3. Check the values in the posted Journal Entry for the Goods Return document.

Expected behavior:

The values in the Credit (FC) / Debit (FC) are 3974.100000 USD, same as the document value in Total Before Discount.

Actual behavior:

The values in the Credit (FC) / Debit (FC) fields are 3974.100032 USD.

*Same issue occurs if creating a GRPO Cancellation document.

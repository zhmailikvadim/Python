Symptom

 In the Data Object document, updated field values and the determined End Customer are not retained after Reprocess and Save. In the version, after modifying the End Customer and performing Address Match, the Determined End User fields are populated. However, after Reprocess and Save, the determined End User value becomes empty. This note corrects the behavior to retain the updated values after reprocess and save.

Other Terms

DO Reprocess and Save Issue, Determined End User, End Customer empty, Address Match, /VTA/BDOM

Reason and Prerequisites

This is caused by program error

Solution

Please implement the correction instructions attached. These corrections are only valid if you have applied the Support Packs.

Release V/4 - 2021, 2022, 2022 V2, 2023, and 2023 V2

Use transaction SNOTE to implement the correction instructions specified below and activate the affected program object.

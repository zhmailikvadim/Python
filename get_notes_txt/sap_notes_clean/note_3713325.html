Symptom

'Unexpected EOF after xxxxxxxx bytes' error occurred in import log with RC=16.

Environment

SAP_BASIS release dependent.

Cause

The error message 'Unexpected EOF after xxxxxxxx bytes' is an indicative of a destroyed/corrupted data file that R3trans is no longer able to read it. This error occurs if the file is transferred incompletely.

Resolution

Make sure the R3trans and tp is up to date.

A test is available to determine in OS level whether R3trans can read the data file or not: R3trans -l <path of data file> 

If the data file was destroyed, copy the data file from source system OS to target system OS file system(/usr/sap/trans/data).

More information refer to 2050 - R3trans: Data file destroyed .

See Also

2050 - R3trans: Data file destroyed

2201677 - It is always recommended to use the latest tp and R3trans

Keywords

Unexpected EOF after bytes, RC16, data file destroyed, data file corrupted.

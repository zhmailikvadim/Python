Symptom

You are comparing the data from the source system from report RFBILA00 against BW or BW/4HANA data for datasources 0FI_GL_10, 0FI_GL_11, 0FI_GL_12 or generated 3FI_GL_xx_TT

Environment

  • SAP NetWeaver
  • SAP ERP
  • SAP S/4HANA
  • SAP BW/4HANA

Cause

It is only possible to compare RFBILA00 data in BW,BW/4HANA with query 0FIGL_M30/0FIGL_M30_Q0001

Resolution

  • First and all understand where the issue starts happening refer to KBA 1622422 - [BW Central KBA] How to identify/analyze a data load issue and the incorrect record(s) for more details.
  • Compare BI data of transaction LISTCUBE for your info provider with ERP data of transaction SE16, SE16N or SE16H for tables FAGLFLEXT and FAGLFLEXA.
  • In case ADSO or classic DSO is being used, compare the inbound/new table records with the corresponding tables. 

See Also

  • SAP Note 1397649 - New General Ledger: Reconciliation of BI data with ERP data (only totals records)
  • SAP Note 2302508 - Usage of datasources 0FI_GL_10, 0FI_GL_14 in sFIN / S4HANA

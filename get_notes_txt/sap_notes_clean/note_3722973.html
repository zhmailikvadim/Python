Symptom

You write off a document manually in transaction FP04 or using mass write-off in transaction FP04M. The write-off ends with error message:

KI 235 "Account & requires an account assignment relevant to cost accounting"

or

KI 248 "Account & requires a true account assignment relevant to cost accounting".

Other Terms

FP04, FP04M, FP-PAREA-1052, FP-PAREA-1053, FP-PAREA-P030, XCOAC = 1, FP-EVENT-5030, FKK_INHERIT_CO_ACC_ASSIGNMENTS, KI235, KI-235, 235(KI), KI248, KI-248, 248(KI)

Reason and Prerequisites

This problem occurs due to a program error.

The system inherits CO account assignments from the revenue or expense line of the written-off document. However, the system does not find the revenue or expense line because the tax jurisdiction code (field TXJCD) is not filled identically.

Prerequisite:

  • For the write-off or Mass write-off posting area 1052 or 1053 is set up in the way that the CO account assignment is inherited from the document that you write off
  • For the document that you write off:
    • It does not contain a tax code (field MWSKZ)
    • The business partner item contains a tax jurisdiction code (field TXJCD)

    • The G/L item does not contain a tax jurisdiction code (field TXJCD)

Solution

After implementing this SAP Note, the system correctly inherits CO account assignments from the revenue or expense line, regardless of whether the tax jurisdiction code is filled on the G/L items or not.

Implement the attached SAP Note or the relevant Support Package.

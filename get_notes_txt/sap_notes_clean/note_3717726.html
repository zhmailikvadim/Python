Symptom

You run the HR-HK: Periodic MPF contribution report (format: eMPF) (HHKCPFE0) to generate eMPF data. The system reads the department code from the Feature for eMPF Contribution Report HHKCPFE0 (27PFE) using the format D<Department code>.

Previously, you could not configure the system to retrieve the department code from a specific field in infotype Organizational Assignment (0001), such as Personnel Subarea (BTRTL).

Now after you apply this SAP Note, the system supports reading the department code from a specific IT0001 field with the format D<0001-<Field>>.
For example, if you configure D<0001-BTRTL> in feature 27PFE, the system reads field BTRTL from infotype 0001 as the department code.

Note: You must include the angle brackets ("<" and ">") around 0001-<Field> in your configuration.

Other Terms

eMPF, HHKCPFE0, Department code, 27PFE

Reason and Prerequisites

This is an enhancement.

Solution

Do either of the following:

  • Apply the Support Package;
  • Implement the corresponding correction instructions, then carry out the following manual activities: 
    1. In transaction SE38, change the Text Elements for the HHKCPFE0 report.
    2. On the Text Symbols tab page, add the following data:
      Symbol Text
      099 Department Code can not be read because the infotype &1 does not exist.
      100 Department Code can not be read because the component &1 does not exist in the infotype &2.
    3. Save your entries.

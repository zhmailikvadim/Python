Symptom

If you want to use the user interface of the lessee contract to create a new contract version, you can find the corresponding function (or pushbutton) either in the last posted and not reversed contract version of a contract or in the reversed contract version following this version (if one exists). If the most recent contract version, that is, the contract version with the highest version number, does not match the last posted contract version, because this version is e.g. B., the user may first have to "search" for the active contract version.

The adjustments provided with this note ensure that the button to create a new contract version is always  and only offered in the last created contract version. Therefore, if you want to create a new contract version, you can always call the last contract version directly and then find the function for creating a new contract version there. This is opened automatically if the user specifies only the company code and contract number when calling transaction /DTLN/NL02 or /DTLN/NL03 and leaves the "Contract Version" input field empty. The potentially required "search" for the function is therefore omitted.

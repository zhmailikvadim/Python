Symptom

When trying to change the view "VCFIN_SOURCE_SET" in the Source System and the fields: "Start Year - Balances", "Start Year - Documents " and "Period - Documents" ; there is error message:

Customizing entry can’t be changed because recording had already started
FIN_CFIN_MIGRATION023

Environment

  • Financial Accounting (FI)
  • Central Finance (CF)
  • SAP S/4HANA
  • SAP S/4HANA Finance
  • SAP ERP

Reproducing the Issue

  1. Configure the Initial Load in Source System view "VCFIN_SOURCE_SET"
  2. Execute the Initial Load in the Target System
  3. Try to change the view "VCFIN_SOURCE_SET" for a Company Code involved in the Initial Load from Target

Cause

It is part of the standard behavior of the system, to not allow the change in view "VCFIN_SOURCE_SET" and the fields "Start Year - Balances", "Start Year - Documents " and "Period - Documents", once the Initial Load execution was started/done, as those values are used to determine the documents stored in the Staging Tables at Source.

Resolution

The view "VCFIN_SOURCE_SET" and fields "Start Year - Balances", "Start Year - Documents " and "Period - Documents" can only be changed before the Initial Load has been executed, or if the Initial Load has been executed it is necessary to reset in both Source and Target in order to be able to change it those fields again.

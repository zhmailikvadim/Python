Symptom

When CSRF token protection is disabled for custom OData V2 services, observe that:

  • The HTTP header X-Requested-With is mandatory

  • Requests fail if the header is missing

  • However, the value of the header appears to be arbitrary

    • Examples such as "x", "test", "abc" all work

Environment

  • SAP S/4HANA
  • SAP Gateway

Cause

This behavior is intended and by design.

When CSRF protection is disabled:

  • SAP Gateway does not validate the value of header X-Requested-With

  • The system checks only for the presence of the header

Resolution

This is expected System Behavior

  • The header X-Requested-With must be present
  • The value can be any arbitrary string
  • Using "x" is recommended, but not enforced

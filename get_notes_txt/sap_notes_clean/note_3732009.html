Symptom

The cost center text is missing from the PA20/PA30 header. However, when displaying IT0001 of the employee, the text of the cost center is displayed.
"Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental."

Environment

  • Human Capital Management (HCM)
  • SAP ERP
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for HANA
  • SAP S/4HANA
  • SAP S/4HANA Cloud Private Edition
  • SAP SuccessFactors Employee Central Payroll

Reproducing the Issue

  1. Run t-code PA20/PA30.
  2. Input an employee number.
  3. Observe that the cost center text in the PA20/PA30 header is not displayed.
  4. When displaying IT0001, the cost center text is visible.

Cause

  • This is a standard system behavior.
  • The assigned cost center’s validity start date is in the future relative to the current system date, and the PA20/PA30 header reads the text from table CSKS using the system date.

Resolution

In PA20/PA30 Header, the system retrieves the cost center text using the current system date. Conversely, in IT0001, the text is pulled based on the start date of the IT0001. 

Example Scenario:

  • Given system date: April 1, 2026.
  • According to table CSKS, this particular cost center is valid starting from January 1, 2027.

Explanation:

  • PA20/PA30 Header:
    The cost center text fails to display because the system date (April 1, 2026) is before the start validity date of the cost center (January 1, 2027).

  • IT0001:
    The text is visible here because the start date of IT0001 matches the validity date of the cost center (January 1, 2027), allowing it to access the relevant text.

Symptom

Codelist Restrictions - Restricted Document Types do not appear in the QC screen of Sales Quotes but Standard Order (OR) appears in QC screen of Sales Orders

Environment

SAP Sales and Service Cloud version 2

Reproducing the Issue

Prerequisite:
Maintain a codelist restriction for both Sales Quotes and Sales order where the code to restrict is 'Document Type'. Disable 'Standard Quote' for Sales Quotes, disable 'OR' for Sales Orders.

  1. Go to Navigation Menu
  2. Go to Sales quotes
  3. Click on + icon
  4. The document type is not defaulted
  5. Click on the dropdown
  6. You observe that document type 'Standard Quote' is not visible or populated in the QC screen
  7. Now go to Navigation Menu
  8. Go to Sales Orders
  9. Click on + icon
  10. You observe that 'OR' is defaulted even though it is disabled via Codelist restrictions

Cause

Sales Orders do not have the setting to set the default document type. Hence OR is set as default from the backend whereas, Sales Quotes have the setting to set the default document type. We have a column 'Default' under document type settings under Sales Quotes.

Resolution

This is the current system behaviour.

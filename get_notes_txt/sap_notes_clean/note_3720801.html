Symptom

You are using the Analyze Forecast, Adjust Forecast or Manage Demand Influencing Factors app in SAP Customer Activity Repository applications bundle 5.0, 5.0 FPS01, 5.0 FPS02, 5.0 FPS03, 5.0 FPS04, 5.0 SPS05, 5.0 SPS06, 5.0 SPS07, 5.0 SPS08, 6.0 SPS01 or 6.0 FPS02.

You select a product hierarchy node or a location hierarchy node by pasting the ID of the node directly into the corresponding input field. The system does not check the area of responsibility in this case. So, you may end up selecting a node that is not assigned to you.

Reason and Prerequisites

Reason

Program error: Area of responsibility check is not performed when pasting the ID.

Prerequisites

You have set up the Analyze Forecast,  Manage Demand Influencing Factors apps as described in the Common Installation Guide (for new installations) or the Common Upgrade Guide (for upgrade scenarios) for SAP Customer Activity Repository applications bundle 5.0, 5.0 FPS01, 5.0 FPS02, 5.0 FPS03, 5.0 FPS04, 5.0 SPS05, 5.0 SPS06, 5.0 SPS07, 5.0 SPS08, 6.0 SPS01 or 6.0 FPS02. You can find both guides on SAP Help Portal at https://help.sap.com/docs/CARAB. Select your version at the top.

Solution

Implement the attached correction instructions in your back-end system.

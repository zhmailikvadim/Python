Symptom

  • When logging in to BILP (BI Launchpad) or OpenDocument, the below error is seen:
    The argument has an invalid value token
  • The issue only occurs through external URLs (eg LBs, reverse proxies) - direct to Tomcat there is no issue
  • In the HAR traces, a 401 error is seen against the following request eg for OpenDocument
    https://<HOST>/BOE/OpenDocument/<TS>/biprwsproxy/biprws/logon/internal/token 
  • In the OpenDocument / BILP glf traces, the below error is seen:
    Error in LogonResource biprws internal/token at doTokenLogon() - appkind is Not availableError in LogonResource biprws internal/token at doTokenLogon() - serializedSession is Not available
    Error in LogonResource biprws doTokenLogon() 
     at com.sap.bip.rs.exceptions.InvalidEntSessionException.InvalidInput(InvalidEntSessionException.java:63)
     at com.sap.bip.rs.infostore.LogonResource.doTokenLogon(LogonResource.java:601)

Environment

  • SAP BusinessObjects Business Intelligence (BI) Platform 4.x / 2025
  • Windows
  • Linux / Unix 

Reproducing the Issue

  1. Log in to BILP or OpenDocument
  2. Observe the error generated in the symptoms is shown

Cause

The access was being performed to BI through SAP Cloud Connector (SCC), which has an allowlist for headers, a particular header x-sap-AppKind (introduced in recent versions of BI) was not added to the allow list.

Resolution

Add the header x-sap-AppKind to the SAP Cloud Connector header allow list.

Note: Whilst this issue was specific to SAP Cloud Connector, the same issue could occur against any network component that has the capability to block headers.

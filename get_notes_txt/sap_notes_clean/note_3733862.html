Symptom

  • The XDL application displays incorrect job status: files containing errors show “success” instead of “failure.”
  • Failed row counts are populated correctly, but the error log download fails.
  • Error when downloading logs: “Error log file is not available and can not be downloaded: [object Object]”
  • The issue has been present since the beginning and has never worked correctly.

Environment

  • SAP Sales Performance Management
  • Incentive Management on Legacy HANA 
  • Incentive Management on Hyperscalers

Reproducing the Issue

  1. Open Express Data Loader.
  2. Upload a source file that contains incorrect data.
  3. Observe that the job status shows “success” even though errors exist.
  4. Attempt to download the error log for failed rows and note the error.

Cause

A confirmed product bug in the custom inbound process prevents XDL from fetching pipeline errors, leading to an incorrect “success” status and failure to download error logs.

Resolution

The Engineering team is conducting a thorough investigation to propose a permanent fix for this behavior, which is currently targeted to be implemented in the 2606 release. Please find below the expected 2026 default release dates for all versions:

  • SAP Community: SAP SPM 2026 Release Schedule

Please be aware that these dates could potentially change based on internal Engineering processes.

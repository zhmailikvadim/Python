Symptom

Transaction QM01/QM02 is recorded by BDC (Batch Data Communication, transaction SHDB).

It is observed that when maintaining an object (for example: batch, serial number etc.) at tab Assigned objects in the notification, this action is not recorded by BDC.

"Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental."

Environment

  • SAP ERP Quality Management (QM)
  • SAP ERP Central Component
  • SAP ERP
  • SAP Enhancement package for SAP ERP
  • SAP Enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA cloud private edition
  • SAP S/4HANA

Cause

Transaction SHDB (BDC Recorder) can only record input fields that are processed as classic dynpro screen fields. But screen area Assigned objects uses control-based/subscreen logic. The data entered on this screen area is processed internally during save, and not exposed as dynpro field. Therefore, it cannot be recorded or replayed using BDC.

Resolution

Corresponds with standard design.

As an alternative, one object (batch, serial number etc.) can be entered at header level of quality notification. For example, if one batch is required to be entered, this can be done at field VIQMEL-CHARG at screen area Reference object ( screen 005) . And as it is a classic dynro field, this action is captured by BDC recording.

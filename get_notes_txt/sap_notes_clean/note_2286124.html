Symptom

When using the BTF Editor in WebCRM UI you experience that when hitting the delete button sometimes inserts empty rows behind the cursor instead of deleting subsequent characters. Further down in the document you may also notice that hitting the Backspace button has no effect.

Other Terms

line break, resolveBlockInsideInline, OnKeyDown_BTFEditor

Reason and Prerequisites

This is due to the BTF Editor part of the Web UI JavaScript Framework not being able to handle the <p> and/or <div> tags inside <span> elements. Please note that such a scenario is also an invalid markup, since block elements may not be included inside inline lements.

Solution

WARNING: This Note contains JavaScript code.

If you do not have Note 1855979 and 1920638 or equivalent SP installed, then please do not forget to erase your browser cache (temporary internet files) and server cache, and close all local browser instances before restarting the application again.

To delete caches, please check with Note 2050322.

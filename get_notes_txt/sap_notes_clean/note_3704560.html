Symptom

  • Custom Field is missing on closed worker records
  • CF value was provided on record and was not removed by user/upload.
  • The absence of the Custom Field value prevents records from populating in the Closed Worker Report.
  • Worker Download File generates records with blank Custom Field, causing issues in downstream systems.

Reproducing the Issue

  1. Log in to Fieldglass.
  2. Search for Worker ID associated with the closed profile.
  3. Review the details of the profile.
  4. Observe that the Custom Field is missing in the worker details.

Cause

  • The custom field is marked as Personally Identifiable Information (PII)
  • PII values get scrubbed by agents after worker is closed.

Resolution

Values of Personally Identifiable Information (PII) custom fields are no longer available on worker (WK and PW) records, when worker is closed. They also will not populate on Worker Downloads. 

This is by design, so to comply with data protection policies. If you need to retain the CF values after worker closure, you need to update the custom field setup.
This can be done in the Fieldglass UI by an Administrator, or user with relevant permissions

  • Login to Fieldglass
  • Go to Admin Config
  • Go to Custom Fields 
  • Locate the custom field
  • Click Edit 
  • Change the Personally Identifiable Information = No
  • Click Update

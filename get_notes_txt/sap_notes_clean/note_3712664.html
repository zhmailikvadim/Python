Symptom

  • Benefit enrollments are being auto-created for users who are not yet officially hired, including those still in Manage Pending Hire or marked as No Show.
  • Benefit enrollment records are created for onboardees after running the Benefit Master Data Change Job.

Environment

  • SAP SuccessFactors Employee Central - Global Benefits

Reproducing the Issue

    1. Go to Manage Data > Benefit Enrollement.

    2. Enter user ID > verify benefit is created.

    3. Go to Benefit Admin Overview > Events log. Verify if processed successfully.

Cause

The benefit enrollment has been successfully processed for the user as the data aligns with the configuration eligibility defined in the settings.  

Resolution

  1. Ensure that the rule logic does not trigger auto-enrollment for users who are not officially hired.
  2. To exclude onboardee users from the automatic creation of benefits, you will need to configure a Benefit Event Determination rule for Hire.
  3. For reference, you may consult KBA 3562830 - Error while hiring user from Manage Pending Hire > Onboarding . While this KBA does not directly address the issue, the underlying logic remains applicable.

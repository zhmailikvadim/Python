Symptom

  • Getting short dumps SYSTEM_CORE_DUMPED after performing of some operations like double-click on a row or collapse/expand of rows in ALV GridView control.
  • Short Text: "Process terminated by signal 0."
  • Error message: "500 Internal Server Error".
  • The following C-stack might be seen in the dump:
  • |dw_gui.so;0x5974a0;[S](CSAPSTLObjArray<CSapITSGridViewRow*>::operator[](unsigned long)+0x38)[0x7ef|
    |dw_gui.so;0x759b5d;[S](AdpOmrtGridView::getDragDropID() const+0x5d)[0x7efd687bbb5d]               |
    |dw_gui.so;0x7bc202;[S](PGridView::PGridView(_CmgrROTEntry&)+0x170)[0x7efd6881e202]                |
    |dw_gui.so;0x7d607a;[S](PItsContainer::createControl(_CmgrROTEntry&, PControlContainer*)+0xda)[0x7e|
    |dw_gui.so;0x7d6440;[S](POmrtContainer::build()+0x72)[0x7efd68838440]                              |

Environment

  • ABAP Platform
  • SAP GUI for HTML / WebGUI
  • SAP S/4HANA
  • SAP KERNEL 7.93 SP348 or lower
  • SAP KERNEL 9.16 SP088 or lower

Reproducing the Issue

  1. On WebGUI/Fiori, open any transaction that renders ALV Grid.
  2. Perform operations on the ALV Grid (for example, double-click or collapse/expand rows).
  3. Observe a short dump SYSTEM_CORE_DUMPED.

Cause

Programming error.

Resolution

Update the Kernel as per the following SAP Note:

3715324 - ALV GridView: CORE DUMP after performing some operations like double-click or collaps/expand rows

Correction is delivered in the following Kernel releases:

  • SAP KERNEL 7.93 SP349
  • SAP KERNEL 9.16 SP089

Symptom

After HANA database upgrade to HANA 2.0 SPS07 or SPS08, you started observing error messages like below in indexserver trace.

WorkloadClass.h(00106) : The following assertion failed: (wci.getStatementMemoryLimit() == STATEMENT_MEMORY_LIMIT_UNDEFINED)
cannot have statement-level and workload class-level memory limits at the same time. please recreate <workload class name>

Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.

Environment

SAP HANA Platform 2.0

Cause

As the error message indicates, HANA database does not allow the creation of a workload class with both statement-level parameter and workload class-level parameter set at the same time. For example, you cannot set value for both STATEMENT MEMORY LIMIT and TOTAL STATEMENT MEMORY LIMIT, or STATEMENT THREAD LIMIT and TOTAL STATEMENT THREAD LIMIT at the same time for one workload class. This behavior is also described in KBA 3217875 - invalid argument: statement-level properties and application-level properties cannot be configured simultaneously. However, this can happen in rare cases due to insufficient restriction in old HANA versions.

Resolution

Drop and recreate the relevant workload classes.

CREATE WORKLOAD CLASS Statement (Workload Management) | SAP Help Portal
DROP WORKLOAD CLASS Statement (Workload Management) | SAP Help Portal
Managing Workload Classes | SAP Help Portal

Starting from SAP HANA 2.0 SPS 06, you can manage a total and a statement specific value by using Hierarchies of Workload Classes.

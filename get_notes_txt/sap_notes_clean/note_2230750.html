Symptom

  • Search is not available to end users with "An error occurred while searching: Index is not ready. Please try again later (PSS 01004). (PSS 01004)"
  • Documents are not retrieved on search even index files are available
  • The size of Lucene Index Engine folder (<InstallDir>\SAP BusinessObjects Enterprise XI 4.0\Data\PlatformSearchData\Lucene Index Engine) grows excessively (e.g. 50GB, 100GB, 200GB)
  • Number of Indexed Documents value in Platform Search Application Properties page in Central Management Console stays at 0
  • Using Luke in read-only (3rd party tool available on the web) to read the Lucene Index files shows that the Administrator and User kind are in the top ranking objects in the Overview tab (see attached screenshot in Attachments section)

Other Terms

Lucene Index, Platform Search, Search, Folder Size, PSS 01004

Reason and Prerequisites

Environment:

  • SAP BusinessObjects Business Intelligence Platform 4.1 (BI 4.1)

Cause:

It has been identified that User and UserGroup kind of objects can be referenced thousands of times into the Lucene Index Engine, and this causes inability to index, unforeseen growth of the Lucene Index, and unavailability of the Search in BI LaunchPad.

Duplication of Lucene docs happens not only for User and UserGroup infoObjects but also for all other infoObjects that have Multi-lingual property bag.

In such cases, for each supported locale which is present in Multi-Lingual (ML) property bag, a separate Lucene doc is created during indexing that infoObject.

When such infoObjects are updated (for example, parent containing that infoObject has its name changed), its corresponding Lucene docs are not  updated or deleted.

Instead, new Lucene docs are created and inserted to master index. Over a period of time and after lots of updates to the same infoObject, it's end up creating multitude of redundant Lucene docs.

Solution

This issue is fixed in the patches listed in the "Support Packages & Patches" section below.

The "Support Packages & Patches" section will be populated with the relevant patch levels once they are released.

For Business Intelligence Platform maintenance schedule and strategy see the Knowledge Base Article 2144559 in References section.

Workaround

As the fix is not currently available for release at the time this note is published, in order to solve the problem, customers are advised to:

  1. Implement the fix provided in SAP Note 2235653 and disable users and groups indexing
  2. Rebuild the index via Central Management Content > Applications > Platform Search

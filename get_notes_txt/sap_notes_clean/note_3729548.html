Symptom

Connection "SUCCESSFACTORS_ODATA" couldn’t be established. Remote tables cannot be used due to errors in the connection.

  • Error message: "[LGN0013] Authentication failed. We have prevented an attempted login from unauthorized IP: to company ID: with username: (status code = 8)."
  • Error message: "Unable to connect remote source: DPAdapterAccess::Connect: connection to :443 returned status 401 (adapter type: ODataAdapter, source name: ), Code: 5921, SQL State: HY000."

Environment

  • SAP Datasphere
  • SAP SuccessFactors HCM Suite

Cause

The issue is caused by missing IP allow-listing for the DataSphere IPs in the SuccessFactors tenant.

Resolution

Add the specific IPs from DataSphere to the API login exceptions in SuccessFactors.

  1. Navigate to Admin Center
  2. Company Settings
  3. Password & Login Policy Settings
  4. Set API Login
  5. Include the IPs from Datasphere. 

Now, if you don't know the IPs from Datasphere, please refer to the steps below on how to retrieve the IPs from Datsphere: 

  1. Go to Datasphere navigation bar
  2. Click System
  3. Click About
  4. You'll find the IPs in "SAP HANA Cloud NAT IP (egress)" list.

Or, in case the error persists, you can check under the Guide page below, and search for your specific region:

Domains and IP Ranges

Symptom

THIS PROBLEM CAN OCCUR ONLY WITHIN SAP! THE NOTE IS NOT RELEVANT FOR CUSTOMERS, UNLESS AS PRECONDITION FOR OTHER NOTES!

This note is a follow-up on note 2338911!

When creating new requerst/ response attributes in the ES-Modeler, deadlocks can occur if the same attribute has been created and renamed earlier and is still existing. This is because of the GTABKEY-registration via GUID and external ID. The note introduces a mechanism that adds a counter to the attribute ID until no conflicts with GTABKEY-registrations do occur.

Reason and Prerequisites

Program error

Solution

Apply correction instruction

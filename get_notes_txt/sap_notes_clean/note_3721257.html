Symptom

Some application object types are not available in configuration "Manage Application Object Type Activation". When trying to add new entries, the application object types are not in the list.

"Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental."

Environment

  • SAP S/4HANA
  • SAP S/4HANA Cloud Private Edition

Reproducing the Issue

  1. Go to SPRO -> IMG -> Cross-Application Components -> Output Control -> Manage Application Object Type Activation
  2. Some application object types are missing
  3. Click on the "New Entries" button to create them
  4. The application object types are not in the list

Cause

For such application object types, it is not supported to change application default setting based on Activation on Application Object Type Level.

Regarding Manage Application Object Type Activation, see IMG activity documentation via SPRO -> IMG -> Cross-Application Components -> Output Control -> check the IMG activity documentation of Manage Application Object Type Activation:

Resolution

In SM30, check the view APOC_I_OBJ_TYPEV to see if Activation on Application Object Type Level is available for the application object types.

Symptom

  • In the HANA database, a column store table or partition exceeded 2,000,000,000 rows.
  • Error when the limit is exceeded: '2055: SQL message: maximum number of rows per table or partition reached: '
  • You want to know whether the 2 billion limit can be changed.
  • Whether pre-partitioning activities will impact the follow-up system migration.

Environment

  • SAP HANA Database 1.0
  • SAP HANA Database 2.0

Reproducing the Issue

Cause

  • By design, each column store table and each partition in HANA must not contain more than 2 billion records.
  • This limit cannot be changed.

Resolution

  1. Note that the 2 billion row limit for a column store table or partition is fixed and cannot be increased.
  2. Partition affected tables so that each partition remains well below 2 billion rows.
  3. As an alternative or complement, remove or archive data that is no longer required.
  4. Pre-partitioning does not negatively impact system migration as long as the system can start and stop normally.
  5. Before partitioning application-owned tables, verify application-specific recommendations and requirements.
  6. For partitioning FAQs and application-specific guidance, refer to: KBA 2044468 - FAQ: SAP HANA Partitioning.
  7. For data housekeeping options and administration guidance, refer to: KBA 2400024 - How-To: SAP HANA Administration and Monitoring and KBA 2388483 - How-To: Housekeeping for Technical Tables.

Symptom

It is not possible to create confirmation for a service PO in SUS when PO has account assignment U. System gives error message SE518 "No account assignment exists for service line &"

Environment

  • Materials Management (MM)
  • SAP R/3
  • SAP R/3 Enterprise 4.7
  • SAP ERP Central Component
  • SAP ERP
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA

Reproducing the Issue

1. Create service PO with account assignment U.

2. It will be extracted to an XML and sent to SUS system.

3. Do a confirmation in SUS system for that PO. 

4. When the XML is processed to create the SES in the ERP system SE 518 error message is raised.  

 

Cause

SUS imports only basic information about PO's, account assignment is not imported. When you create a confirmation for the PO in SUS no account assignment information will be included in the XML as this information is not available on SUS side. The account assignment will be pulled from the ERP PO during SES creation in the ERP system. Creating SES for such a PO will result in an error message because the system will be not able to determine account assignment details from the PO during XML processing. Error message SE518 "No account assignment exists for service line &" is raised.  During SES creation account assignment is mandatory (even when account assignment U is also an option in the drop down menu in ml81n). This is because FI document will be created when SES is approved and GR 101 is created and for it G/L account, cost center etc. are needed.

Resolution

Basically there are three options to avoid such problematic situation.

  • Do not use PO-s with account assignment U for service PO's
  • Do not process PO's with SUS which have account assignment U. (You can also exclude these PO's from the extractor so they will not be transferred to SUS)
  • Set a predefined account assignment and fill to bapi_entrysheet_create after confirmation is arrived to ERP.

Last two solutions need custom development which needs to be tested.

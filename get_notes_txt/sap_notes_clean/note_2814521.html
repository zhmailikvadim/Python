Symptom

During the maintenance of a role in transaction PFCG, you enter transaction S_ALR_87005471, S_ALR_87005540, S_ALR_87005742, S_ALR_87005757, S_ALR_87005792, S_ALR_87005395, S_ALR_87005808, or S_ALR_87005903 into the role menu. Following the merging of the authorization data, you notice missing or unsuitable authorization default values for the object K_VRGNG, for example, even though an authorization for the execution of the aforementioned transactions is required for the object K_VRGNG.

Other Terms

SU24, SU22, RSU22UPLD
S_ALR_87005471, S_ALR_87005540, S_ALR_87005742, S_ALR_87005757, S_ALR_87005792, S_ALR_87005903, S_ALR_87005395, S_ALR_87005808
K_VRGNG
KSU1, KSU7, KSV1, KSV7, KSC1, KSC7

Reason and Prerequisites

Until now, the required authorization default values have not been provided by SAP.

Solution

The missing data is delivered with the corresponding support package.
You can upload the authorization default values in advance from the attached file SU24_COOMCCAF_S_ALR.TXT. Use the "Upload" function from the toolbar of transaction SU24 in accordance with SAP Note 1539556.

Before importing the file attached to this SAP Note, make sure that SAP Note 2192354 has been implemented in your system.

Symptom

In CO03/COR3/KKBC_ORD, layouts containing total plan cost columns initially show zero until the column is removed and reinserted.

Environment

  • Controlling (CO)
  • SAP ERP
  • SAP ERP Central Component
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA
  • SAP S/4HANA Finance
  • SAP S/4HANA Cloud Private Edition

Reproducing the Issue

  1. Create a production order (CO01) or process order (COR1).
  2. Cost the order.
  3. Release and save the order.
  4. Goto - Costs - Analysis
  5. Choose a layout containing “Total Plan Costs” (for example, 1SAP02 Cost Trend).
  6. Observe total plan costs appear as zero.
  7. Remove and reinsert the “Total Plan Costs” column.
  8. Plan costs are then displayed.

Cause

  • ALV grid issue in SAP_BASIS release 758

Resolution

  1. Implement the correction per SAP Note 3537467 - ALV Grid - After changing layout data of a column are not updated.
  2. SAP recommends to update SAP_BASIS release 758 to at least support package level 4 (SAPK-75804INSAPBASIS).

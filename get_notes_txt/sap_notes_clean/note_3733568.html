Symptom

As described in SAP ONE Support Launchpad Notification of Legal Change Unified monthly employer report 2026 (JMHZ)

The functionality delivered in this SAP Note is currently under development and needs to be updated regularly with latest SAP Notes.

These updates are provided:

REGZEC (DM - data mining, CS - compose submission, PF - process feedback, SO - special operations, INIT - import of IDs to IT3457, XP - XML for ePortal)

  • INIT: Report HCZCJHR0_INIT has been adjusted for case the input data contains a date in the item "insurance relationship to"(Pojistný vztah do), xml tag <PojistnyVztahDo>. A repeat run delimits the validity of the IT3457 record of subtype 2 to this date, if such a record already exists in the infotype.
  • DM: Report is prepared for the new legislation valid from 1.4.2026. It says the employee terminated one day and hired the day after must be handled as not continuous employment.
  • DM, CS, PF: Following recent updates to the ČSSZ technical documentation, attributes 10092–10101 are now also available for foster parents (pěstouni)

MH (DM - data mining, CS - compose submission, WB - workbench)
Corrections that may cause that corrective individual forms will be created for already submitted periods are marked with code abbreviation (DIF)

  • Improvements
    • In the ALV display of individual and summary forms, it is now possible to show attribute IDs via a tooltip—by hovering the cursor over the column header in ALV.
    • In program HCZCJHM0_WB, the ability to navigate to the B2A transaction has been added for cases when you are sending forms to B2A in production.
    • In program HCZCJHM0_WB, an option has been added to display an overview that makes it easier to check whether MH is complete in each period (month). The overview includes only currently valid (status 20) or rejected forms (status 90). The required final state for a fully closed period: all forms in status OK and their count should match the number of PPV for the given VS. Error columns: rejected forms that were sent last. If these are corrective forms, the original forms still apply and are included in the OK column; therefore, OK + Error will be higher than the number of existing PPV. This overview is not intended to control the reporting process and deliberately contains no editing elements.
    • If you need to decrease the value of attribute 10268 using a wage type, assign that wage type to evaluation class (EVCLS) 14 with value O5 “10268 Worked hours deduction”.
    • If you need to decrease the value of attribute 10268 using an absence, you can now do this in table T5T04_03, using the new classification values “10268 Number of hours worked” and use the summation sign “-”.
    • In section PRO, attributes 10453, 10431-4 use infotype 0021 subtype CZ02. Newly, customer can specify different subtype(s) to be considered instead of CZ02 - via table T7CZRE (Parameter area=JHM_PRO; Parameter name=OTHER_PERSONS)
    • It is now possible to suppress the creation of MH forms for selected contract types (infotype 16) using a setting in the new field T5T47N-NOJMHZ. Program HCZCJHM0_DM will suppress the creation of a form for the given month (FOR period) if all infotype 16 records falling into that month refer to T5T47N-NOJMHZ with value “X”.
  • Fixes
    • Logical check 028 corrected according to JMHZ version 1.4.2.
    • If any of the health insurance elements (10371, 10482) is negative, it is set to zero and a warning message appears in the log indicating that this situation occurred. (DIF)
    • Element 10416 (Income of a non-resident, member state org) is now filled only if the type of gainful activity equals Q or S. (DIF)
    • Attribute 10333 Overtime premiums (source = wage types with evaluation class 13 = P1) is no longer automatically added to attribute 10332 Total premiums. (DIF)
    • In section PRZ, for the absence hours in attributes 10275, 10277, 10278, 10280, 10471, the hours of planned shifts on public holidays that overlap these absences are now also added. (DIF)
    • When determining excluded periods in section ELI, for employees with multiple employments, the overlap of excluded periods and insurance across these employments is now taken into account. (DIF)

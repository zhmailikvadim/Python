Symptom

In some situations, it may be necessary to debug and analyze the HTTP request sent from the DRC Cloud Edition push service to effectively troubleshoot the issue. This KBA aims to guide you through that process.

Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.

Environment

  • SAP ERP
  • SAP S/4HANA
  • SAP S/4HANA Cloud Private Edition
  • SAP BTP
  • SAP Document and Reporting Compliance, Cloud Edition

Resolution

  1. Set an external breakpoint for the BTP destination user in the method IF_REST_RESOURCE~POST (class GCO_ECOM_REST_RESOURCE).
  2. Submit a document so the push service sends an HTTP request to business system. You can do it by following one of the options:
    1. Submit a document in eDdocument Cockpit (edoc_cockpit) transaction.
    2. Reprocess a document from the Resolve Errors - Documents and Statutory Reports app in DRC Cloud Edition.
  3. A debugger session will open and it is possible to analyze the flow process in this method.

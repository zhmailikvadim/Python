Symptom

  • After External Warehouse Management (EWM) processes a goods issue for an outbound delivery order item, medication kits might lose their handling unit (HU) assignment in ICSM or be assigned incorrectly. This can block subsequent logistics processing and cause handling unit information to be missing from the Medication Kit Numbers table.
  • After you pack or repack using the standard EWM transaction, medication kits might also lose their handling unit assignment.

Other Terms

ISLS, Intelligent Clinical Supply Management, Distribution, Medkit, HU, EWM, PGI, ODO

Reason and Prerequisites

Inconsistent state between the EWM HU packing hierarchy and ICSM HU persistence after goods issue in EWM.

Prerequisites:

You use SAP Intelligent Clinical Supply Management.

The ODO document ID and item ID quantity are packed into a single lower-level handling unit and aren't split.

Solution

Run report /CTCO/R_HU_REASGN_POST_EWM_GI (includes Test Mode) to validate the EWM GI evidence and HU hierarchy.

  • The report then updates the Warehouse Request MedNos Table with the provided Target HU assignment.

Run report /CTCO/R_HU_REASGN_POST_EWM_GI2 and select one of the following options to validate and update HU information:

  • Based on GI Posted ODO Item - Update Medno and WHR Medno: uses GI HU to update HU information on the Medication number and Warehouse Request MedNos.
  • Based on Medno - Update Medno and WHR Medno: uses free input HU or history HU to update the Medication number and Warehouse Request MedNos.

Both reports log all actions in the application log (SLG1).

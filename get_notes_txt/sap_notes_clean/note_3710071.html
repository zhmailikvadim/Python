Symptom

  • Change logs after IDOC processing show deleted entries followed by recreated entries for classification data.
  • Classification data is deleted and recreated multiple times during IDOC processing.
  • The classification data which is sent to the target system is incorrect

Environment

  • Classification

Reproducing the Issue

  1. enter transaction BD87
  2. post CLFMAS Idoc 

Cause

  • not all fields in the Idoc are filled correctly

Resolution

  • If the IDoc is created outside a standard SAP system, ensure the MSGFN field is correctly populated in every segment. MSGFN is typically the first field in each segment and determines how the receiving system processes that segment.

  • The receiving system interprets MSGFN as follows:

    • 009: Add
    • 004: Change
    • 003: Delete
    • 018: Unchanged (no change)
    • 005: Refresh

  • Processing rules:

    • For 009, 004, 018, and 005: Create the data in the receiving system if it does not exist, or update it if it already exists.
    • For 003: Delete the data in the receiving system.

  • If MSGFN is missing or not set correctly, the system cannot reliably determine whether to insert, update, refresh, or delete the data, and processing may be unpredictable. Always provide the correct MSGFN value in each segment.

  • For segment E1KSSKM, the STATU field must also be populated. STATU indicates whether the class assignment is active or inactive.

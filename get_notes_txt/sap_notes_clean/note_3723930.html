Symptom

The  requirement involves WBS data with both Active and Inactive status.



Environment

  • SAP ERP Project System (PS)
  • SAP R/3
  • SAP R/3 Enterprise
  • SAP ERP Central Component
  • SAP ERP
  • SAP Enhancement package for SAP ERP
  • SAP Enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA, on-premise

Reproducing the Issue

  1. Run DRFOUT.
  2. Enter Replication Model eg PS_WBS_OUT.
  3. Execute.
  4. In  Targe System not all WBS have been created .

Cause

The standard DRF class 'CL_PS_DRF_WBS_OUTB' have method 'READ_COMPLETE_DATA', which filters the data with a Select query with certain conditions which is automatically eliminating the Inactive WBS element from the output.

CDS view is used for internal calculations to decide if WBS can be replicated to target systems using DRF.
The values in the where clause refers to business transactions as described below (you can refer table TJ01T). 

BABS         Complete
KAMV         Manual cost allocation
PSTE          Confirm WBS
PSTS          Partially confirm WBS
QISR          ISR Account Assignment
RFBU          FI: Postings
RFST          FI: Statistical postings
RKL            Actual activity allocation
RKU1          Repost costs
RKU3          Repost CO line items
RMRP          Incoming invoice
RMRU          Confirm order
RMWE         Goods receipt for purch. order

For any particular project element(WBS Element, Project, Network..etc) based on the system status there are some business transactions that would be permitted and not permitted.
The CDS view p_wbsmaxbustransprmssnaggrg will just let us know if the above listed all business transactions are permitted or not on particular object(OBJNR).
If the business transaction permission is ‘1’ then its is permitted, if its ‘3’ then not permitted.

Resolution

The CDS view is used in a query within the method IF_DRF_OUTBOUND~READ_COMPLETE_DATA of the class CL_PS_DRF_WBS_OUTB.
This method will filter and get all the WBS Elements that can be replicated. Here in the query the filter set is as below “p_wbsmaxbustransprmssnaggrg~bustransactypepermission = '1'”.
Indicating that for a particular WBS Element all the listed above business transactions should be permitted then only the WBS will be selected for 

This is standard behavior that DRF doesn't consider the WBS with the mentioned status (I0046  I0076 I0013)

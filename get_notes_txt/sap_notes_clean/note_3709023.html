Symptom

After starting the creation of a service order, the error message below is triggered:

  • "Enter a (default) purchasing organization for plant xxx"

Environment

SAP S/4HANA Cloud Public Edition

Cause

The behavior is standard in SAP S/4HANA Service Management and is related to the tight integration of Service Orders with Procurement. In contrast to Sales Orders, Service Orders validate procurement-relevant data earlier in the process.

When the system determines that a service order item requires procurement (PR/PO), it requires a default Purchasing Organization to be maintained for the plant. If no default purchasing organization can be determined, the system raises the above error.

The default purchasing organization is required for Service Orders but not for Sales Orders because Service Orders includes procurement as an integral part of service execution. Therefore, the system validates the availability of a default purchasing organization for the plant at item level.

Sales Orders do not enforce this validation because procurement there is optional. 

Also, this doesn't work different when purchasing organizations are used for inside and outside vendors, the default purchasing organization is only used as an initial or fallback value. It can still be changed later during Purchase Requisition or Purchase Order processing.

Resolution

The default Purchasing Organization must be maintained per plant because this process determination is plant-based.

The configuration requires that, at least one purchasing organization assigned to a plant must be defined as the standard purchasing organization.

This means that, the default purchasing organization is maintained at plant level. There is no concept of a global default purchasing organization across all plants.

Therefore, for any plant that is used in Service Orders and triggers procurement, a standard purchasing organization must be assigned to that plant.

Maintaining a default purchasing organization for the plant where the error is being triggered will fix the message.

This can be done in the "Set Up Organizational Structure" app in Central Business Configuration or using the configuration activity “Assign Standard Purchasing Organization to Plant”.

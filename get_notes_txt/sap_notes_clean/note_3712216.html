Symptom

The system is incorrectly determining the tax classification TAXK1 and TAXM1 from the TTX1 condition type to the TES1 condition type during the processing of sales orders and invoices.

Environment

SAP S/4HANA Cloud Public Edition

Reproducing the Issue

  1. Configure tax classifications for customer and material for condition type TTX1 and TES1.
  2. Process sales orders and invoices using the configured customer and material.
  3. Observe that TES1 is incorrectly determined using tax classifications TAXK1 and TAXM1 from TTX1 in TES1 condition records.

Cause

Condition type TES1 is using the access sequence TTX1, which uses TAXK1 and TAXM1 as key fields. 

Resolution

  1. Define a new condition table with TAXK2 and TAXM2 as key fields.
  2. Assign this new condition table to the access sequence of the condition type TES1.

Keywords

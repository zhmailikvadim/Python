Symptom

You use BAPI_ALM_ORDER_MAINTAIN to perform a function on a maintenance or service order. Fields have been defined as mandatory and are validated in IW31/32/38. These are not checked by the BAPI

Environment

  • SAP ERP Plant Maintenance (PM)
  • SAP ERP
  • SAP S/4HANA, on-premise
  • SAP S/4HANA, Cloud Private Edition

Reproducing the Issue

Test the BAPI in SE37

Cause

The BAPI does not check for mandatory fields marked in customizing. The customizing setting are checked as part of screen handling (dynpro) logic as used by traditional transactions. BAPI's do not use these screens or dynpro logic

Resolution

The custom program calling the BAPI should validate the fields been passed to the BAPI. It is the responsibility of the calling program to pass all necessary field values

Keywords

SAPLCOIH, RELEASE, BAPI_ALM_NOTIF_CREATE

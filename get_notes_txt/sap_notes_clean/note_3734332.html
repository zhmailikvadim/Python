Symptom

  • In billing, item division (SPART) is derived from the sales order header division for all items, even when items belong to different divisions.
  • If the item-level division is not maintained in table VBRP, all billing items show the same division as the header.
  • G/L account determination results in the same revenue account for all items instead of separate accounts per item division.

Environment

  • Sales and Distribution (SD) 
  • SAP ERP
  • SAP ERP Central Component 
  • SAP enhancement package for SAP ERP 
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA
  • SAP S/4HANA Cloud Private Edition

Reproducing the Issue

  1. Create a preceding sales document where the “Item Division” indicator in VOV8 is not enabled.
  2. Create a sales order with a header division and add multiple items whose material masters are assigned to different divisions.
  3. Create delivery and billing with reference to the sales order.
  4. In the billing document, check item division (VBRP-SPART) and account determination; observe that all items use the header division and the same revenue G/L.

Cause

The sales order was created with reference to a document type in which the “Item Division” indicator (VOV8) was not enabled. As a result, item divisions were taken from the header and copied into billing (standard behavior), causing a single division for all billing items.

Resolution

  1. In transaction VOV8, enable the “Item Division” indicator for all relevant preceding sales document types used as references (for example, quotations).
  2. Verify that the sales order document type also has the “Item Division” indicator enabled in VOV8.
  3. In transaction VKOA verify that GL account determination is correctly maintained for all divisions.
  4. When creating a new document with items from different material divisions, confirm in billing that VBRP-SPART reflects the item’s material division and that G/L determination separates by division.

Symptom

QR code login results in error message: “Invalid QR Code. Login unsuccessful. This profile belongs to another user.”

"Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental."

Environment

  • SAP SuccessFactors HCM Suite
  • SAP SuccessFactors Mobile

Reproducing the Issue

  1. Log into SuccessFactors in browser
  2. Go to Settings > Mobile
  3. Select Activate via Camera
  4. Use the Mobile app to scan the code
  5. Error is displayed on mobile

Cause

This error is shown when multiple user accounts are attempting to log in from the same device.

For example:

  1. User logs into Mobile app with one user account
  2. From the app, the user clicks on “Sign Out”
  3. Then, a different user profile tries to scan the QR code on the same device

Once an account was authenticated from a device, no other accounts can log in, even if the original account has signed out.

Resolution

Option 1: If more user profiles want to use the same mobile device, then the “Deactivate” option must be selected.

Option 2: If the first account that accessed the mobile is known, the admin can go to Mobile Settings and deactivate the user account so the new user can login.

Option 3: Please uninstall and reinstall the SF mobile app. After the reinstall, it is possible to log in with a new user account. Any time if a new account is in use, SF mobile app needs to get reinstalled.

Option 4 (only for Android): Please clear data and cache for SF mobile app. After the cache and data are cleared, it is possible to log in with a new user account.
Since there is not cache clearing capability for iOS devices, this option is not available on iOS.

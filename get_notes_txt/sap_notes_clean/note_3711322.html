Symptom

Several Collective agreements in Sweden require that the Valuation rules for employee absences be determined by the total number of Calendar days of the absence, rather than the number of Working days.

Other Terms

SNAB, Unpaid leave, T5S0Q, KLBEW

Solution

- The standard configuration for absence evaluation in Swedish Payroll allows the Valuation rule changes based on the number of Work days within an absence period.

- This enhancement allows the system to count either Work days or Calendar days when evaluating absence duration against configured days in table T5S0Q.

- This enhancement solution is flexible, as companies may need to apply both Calendar-day and Work-day based rules for different Employee groups within the same organization, depending on the applicable Collective agreements.

     

Procedure of SAP note installation:

- Implement DDIC changes via automatic correction instruction or manually to adjust table T5S0Q and table view V_T5S0Q.

- Execute a Table Maintenance generator for the table view V_T5S0Q to be editable for the maintenance and entering new records. This step is mandatory and must be executed before aplying of automatic correction instructions as a last step.

- Implement automatic correction instructions giving correct behavior to the Maintenance view V_T5S0Q and bringing new method of Unpaid absence leave evaluation in Calendar days.

     

Table view V_T5S0Q custonizing:

- Processing of Unpaid leave using Calendar days together with the ordinary way using Working days is possible, no conflict happen in the Payroll absence evaluation.

- Therefore in case when there is the same KLBEW used for evaluation in Calendar days and also in Working days, then method using Calendar days is prioritized (indeed if value of CAP Grouping of processed Employee meets the customized criteria in T5S0Q).

- Other Employees who doesn't meet the criteria of CAP grouping (T503-TRFKZ) are evaluated standardly by using Working days if such record exists with the same KLBEW.

- Please be aware that records in table view V_T5S0Q are stored in database based on table keys, so records with the same KLBEW are mostly inserted as a pair automatically, distinguished by the flag for Calendar days. If you don't need both records just simply remove one manually.

 

Please note:

Part of the function SNAB responsible for processing of Unpaid leave absences has been refactored completely. Please retest typical scenarios used for Unpaid leave evaluation before aplying this SAP note to productive systems.

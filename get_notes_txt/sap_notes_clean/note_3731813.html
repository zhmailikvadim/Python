Symptom

Unable to post in AIAB and AIBU due to error AW050 (Settlement is not possible for asset).

Environment

SAP S/4HANA

Reproducing the Issue

  1. Open transaction AIAB or AIBU for line item settlement from asset under construction to a final asset.
  2. Attempt to post the settlement.
  3. Observe that the system stops with error AW050 and the posting is not possible.

Cause

The asset master has field ANLA-XOPVW empty, which leads to error AW050 during line item settlement in AIAB/AIBU.

Resolution

  1. Create a new asset with the correct settings in the asset class so that line item settlement is enabled (ANLA-XOPVW not initial).
  2. If the asset must remain linked to a WBS element, maintain the WBS assignment on the new asset before any value transfer.
  3. Transfer values from the old asset to the new asset using inter-asset transfer:
    • Use ABUMN (inter-asset transfer within the same company code) or ABT1N (transfer posting), as applicable.
    •  Transfer both acquisition and accumulated depreciation so historical balances move correctly.
  4. If carrying the WBS lint to the new asset is not necessary, ensure the transfer approach does not violate original project settlement logic and that reporting/audit requirements are met.
  5. Do not attempt to correct the existing asset master for this issue once it is already in use; settlement via AIAB/AIBU for that asset will continue to fail with AW050.

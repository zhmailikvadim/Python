Symptom

When performing an ATP check in a sales order, you notice that some request quantities, which have the MRP element "Shipping Notification," appear in the Review Availability Check Result screen. However, these quantities are not relevant to the specific sales order.

Environment

SAP S/4HANA Cloud Public Edition

Cause

This is a standard system behavior as "Shipping Notifications" will be considered as future supply if the indicator for it is set in configuration activity 106518 - Configure Scope of Advanced Availability Check. The request quantity for  "Shipping Notifications" quantity will be shown in the sales order when the system performs the ATP check.

Resolution

If "Shipping Notifications" should not be considered in the ATP check according to the business scenario, unflag the "Shipping Notifications" indicator in configuration activity 101099 - Configure Scope of Availability Check.

Keywords

OVZ9, Scope of Check, Shipping notification, 101099, Configure Scope of Availability Check, Sales order

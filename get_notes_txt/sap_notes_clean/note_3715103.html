Symptom

The Impact Analysis raises an error message like

TABL <TABNAME>: High Row-change rate <ChangeRate> versus total rows <Total number of rows>; see note 3715103

Other Terms

Software Update Manager, SUM, Impact Analysis, Downtime-optimized Conversion

Reason and Prerequisites

You are performing the Impact Analysis for a Downtime-optimized Conversion with the Software Update Manager and you are receiving an error message during the Impact Analysis

Solution

This message serves as a warning that the affected table is updated very frequently. As a result, the replication triggers that record changes on this table may have an impact on certain business processes, particularly with regard to performance. Because the ratio between the total number of rows and the number of updated rows is relatively high, you should evaluate whether the table can be excluded from uptime handling and instead be transferred during downtime.

Note: It is not mandatory to remove tables from uptime handling when they are flagged by this check. The purpose of this message is to raise awareness and to recommend validating whether uptime handling is appropriate for this table.

The decision depends on several factors:

  • Downtime requirements Database triggers introduce a small performance overhead on the production system, which is usually not noticeable to end users. However, for certain database platforms and in cases of very high update frequency, it may be preferable to avoid update handling for the table and transfer its data during downtime instead.

  • Overall performance requirements of the production system Depending on system utilization, it may be beneficial to exclude tables from uptime handling having a high change rate but a relatively small total number of rows.

Symptom

  • When Cloud ALM/Extension is provisioned on Data Center EU10-004 and trying to utilize Central Monitoring with the Wizard, encountering error:
    Errors occurred while synchronizing. Choose Resynchronize to restart the synchronization process.  If the problem persists, report a case in the SV-CLM-INF-CON component and provide the formation ID "<id>" together with the following technical details
  • For an S/4HANA Cloud Public service in status Active or Inactive, trying to activate Health Monitoring data collection and encountering error "Data is outdated. [No data] No data received
  • Trying to activate Central Monitoring for S/4HANA Cloud Public in the scenario where the service belongs to a customer number in a CCC though the customer number is not the same customer number where Cloud ALM itself is provisioned

Environment

  • SAP CLOUD ALM
  • SAP S/4HANA Cloud Public

Reproducing the Issue

Symptom 2:

  1. Launch SAP Cloud ALM 
  2. Click Operations > Monitoring and Analysis > Health Monitoring
  3. In Scope, select S/4HANA Cloud Public
  4. In Configuration, ensure data collection is on
  5. In Overview, click card for S/4HANA Cloud
  6. In Services/Systems, click the involved service
  7. In Monitoring, see:
    Data is outdated. [No data] No data received 

Cause

In the case the Cloud ALM or extension is provisioned in Data Center EU10-004 or the S/4HANA Cloud Public service was set to status of Active or Inactive prior to Central Monitoring becoming available, there are special configuration steps that must be taken for the connection to be established and data collection to occur in the various supported Operations use cases.

Resolution

There are three scenarios that control which configuration steps should be followed:

 1) Automated Setup in SAP Cloud ALM is possible when the S/4HANA Cloud Public service meets all the following prerequisites: 
- Imported to Landscape Management from SAP Backend (Source = Imported)
- Is in status “New”
- Customer number of S/4HANA Cloud Public service  == customer number of Cloud ALM (Find this in Landscape Management > Configuration > Import of Subscribed SAP Services > Imported Customer Numbers)
- Cloud ALM/extension is not located in Data Center eu10-004

2) Automated Setup with manual pre-steps is used when the S/4HANA Cloud Public service meets all the following prerequisites:
- Imported to Landscape Management from SAP Backend (Source = Imported)
- In status “New”
- Customer number of S/4HANA Cloud Public service is not the same as the customer number of Cloud ALM (Find this in Landscape Management > Configuration > Import of Subscribed SAP Services > Imported Customer Numbers)
- Cloud ALM/extension is not located in Data Center eu10-004

3) Classic Setup for S/4HANA Cloud Public is used when the S/4HANA Cloud Public service meets any of the following prerequisites:
- Manually created in Landscape Management  (OR)
- In status “Active” or “Inactive”  (OR)
- Cloud ALM is located in Data Center eu10-004

Symptom

This document explains the conditions under which the hardware key changes on the VMware environment.

Environment

VMware on Windows Server

Cause

  • The hardware key change is triggered by specific actions in VMware environments, such as moving virtual machines, cloning, restoring snapshots, altering network settings, modifying virtual machine configurations, or updating VMware Tools.

Resolution

In the VMware environment, the hardware key is changed under the following circumstances:

  1. Moving VM and making clones: Moving the VM to another host or creating a clone may change the MAC address or UUID.
  2. Restore Snapshot: Restoring a snapshot may change the hardware key because the environment reverts to its original state.
  3. Change Network Settings: Changing the VM's network settings may change the MAC address.
  4. Change Settings for VM: Changing the VM settings (For example, adding or removing virtual network adapters) may change the hardware key.
  5. Update of VMware Tools: If the VMware Tools is updated or reinstalled, the hardware key may change.


Note:
You might be able to get information about changing your hardware key by looking at VMware's log files (vmware.log). Please contact VMware for more information

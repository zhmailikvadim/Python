Symptom

Screens appear differently when recording transactions through a specific tool (SHDB) compared to executing the transaction directly.

Environment

  • Dynpro and CUA engine
  • SAP NetWeaver - All BASIS releases.

Cause

The behavior of some transactions changes when they are run in batch mode.

Resolution

The SHDB transaction has a "Not a Batch Input Session" option for recordings. Please activate it and verify if the outcome is different.

Keywords

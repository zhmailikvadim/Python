Symptom

1. Resolved the truncation issue if the length of the tracking unit stop ID exceeded 16 characters.

2. Resolved the duplicate planned event issue. If a freight order or a freight booking contains a stop where the same stop ID has multiple stop roles assigned, for example, stop ID ZS01 has both the standard stop role and the LH (Load Handling Substop for Advanced Shipping & Receiving) role—then, when the tracked process is generated and sent to GTT, the following error appears in the MML (Manage Message Logs): “Duplicate planned events XXX are not allowed. Change the message payload to remove the duplication or contact your Model Administrator.”

3. Added the carrier reference document for the scenario "Generate Tracking Unit based on Container Unit". The system now retrieves this data from the following sources in order of precedence:

  1.  "Document References" sub-tab within the Container Unit's "Items" tab.
  2.  "Document References" tab of the corresponding freight booking.

4. Other code enhancements.

Other Terms

Tracking unit stop ID truncated, duplicate planned events, carrier reference document

Solution

Implement the attached correction instructions.

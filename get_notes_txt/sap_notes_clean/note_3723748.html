Symptom

Using transaction TRIP in WebGui: Print button in status bar does not work as expected.

Environment

SAP Travel Management - on premise

Reproducing the Issue

  1. Launch the TRIP transactionvia WebGui.
  2. Click the print icon in the status bar.
  3. "local printer popup" is not opened as expected / opens but is immediately closed.

Cause

Microsoft automatic updates of the WebView2 component to version 144

Resolution

Check the recommendations in KBA 3704912 - Latest Webview2 Version 144.xxx causes problems with SAP GUI HTML control when using edge as browser control in SAP GUI for Windows

Microsoft solved the issue with WebView2 Runtime version 144.0.3719.93

Symptom

• When creating a new condition record with the same validity but a different condition amount in VK11 or VK15, the old condition record is deleted from the A*** table, but the new condition record number is not updated in A*** table. 
• As outlined in the SAP notes 2954021 and 2079822 in standard behavior, when a new condition record is created with the same validity date, the old condition record number typically remains in KONP/KONH but is replaced by the new condition record in the A*** table.

Environment

  • SAP S/4HANA
  • S/4HANA Cloud Private Edition

Reproducing the Issue

1. Create a new condition record with the same validity period but a different condition amount using VK11 or VK15.
2. Observe that the old condition record is deleted from the A*** table. 
3. Verify that the new condition record number is not updated in A*** table. 

Cause

The issue is caused by the function module CACS_VERSIONING_UPDATE, which inadvertently deletes the new condition record.

Resolution

Apply SAP note 2813943, which pertains to cacscondmaint and updates the condition result with a dump DBSQL_DUPLICATE_KEY_ERROR.

Symptom

Customer Odata endpoint has the following table metadata:

    <EntityType Name="Documents">

        <Property Name="createdAt" Type="Edm.DateTimeOffset" Precision="7"/>
        <Property Name="modifiedAt" Type="Edm.DateTimeOffset" Precision="7"/>

SQL SELECT ..., "createdAt", "modifiedAt", ... FROM "Documents" LIMIT 1000

returns json response with Timestamp values
"createdAt": "2025-03-11T14:47:13.741581800Z",
"modifiedAt": "2023-07-04T08:21:28.777473400Z",

DPAdapterSDK::AdapterException is thrown with error "Subsecond out of range while initializing timestamp".

Other Terms

OData, ODataAdapter, SDK, exception, timestamp

Environment:

SAP HANA Cloud Edition CE 2025.40.10

Reason and Prerequisites

The JSON response for the Timestamp columns returned value with 9-digit subsecond
"createdAt": "2025-03-11T14:47:13.741581800Z",
"modifiedAt": "2023-07-04T08:21:28.777473400Z",

DP AdapterSDK Timestamp data type expects subsecond value in range of 0-9999999. Both 741581800 and 777473400 are > 9999999.

Solution

As HANA Timestamp data type only supports fractional seconds with up to 7 digits (FF7), the DP AdapterSDK Timestamp data type will be updated to truncate subsecond values > 9999999.

The correction will be delivered and the release(s) will be listed in Support Packages & Patches section of this SAP Note when available.

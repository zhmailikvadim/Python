Symptom

When executing a mass settlement posting for IP documents, if an error occurs during accounting document creation that triggers a rollback, the system is expected to cancel the update for both the current and all previously processed IP documents within the same lot. However, in the current behavior, only the processing of the document that caused the error is cancelled. IP documents processed earlier in the same lot remain updated as Completely Settled, resulting in data inconsistency.

This note addresses the issue by correcting the rollback handling logic to ensure that, in the event of an error during accounting document creation, the update is cancelled for all IP documents within the affected lot, thereby preventing inconsistent settlement statuses.

Other Terms

/IRM/IPBB_SETTLEMENT_CREATE, /IRM/IPBB23, IP document, Mass processing, Settle, Completely Settled, Posting, FI

Reason and Prerequisites

This is caused by Program error.

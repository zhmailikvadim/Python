Symptom

Need steps to configure trusted authentication using Query String for WACS.

Environment

  • SAP BusinessObjects Business Intelligence (BI) 4.3 SP04 Patch 1000+
  • Windows

Resolution

When you access BI launchpad then restful call takes place and biprws comes in picture. To configure Trusted authentication using WACS you need to update the properties files and check if query string is working.

  1. Navigate to Central Management Console (CMC) -> Applications -> Authorized URL Configuration
  2. Enter a system for each URL that the BI Platform is accessible on (Ensure these URLs are resolvable from the CMS machines)
    For each System, select 'Enable', give a unique name, and enter in the URL, in the format below:
    https://<HOST>:<PORT>/BOE/CMC/TrustedAuthSetTokenServlet
  3. Click 'Save & Close'

Go to <Installation Directory>\SAP BusinessObjects\SAP BusinessObjects Enterprise XI 4.0\warfiles\webapps\BOE\WEB-INF\config\custom

  1. Create a new file, or edit an existing file called:
    global.properties
  2. Add or edit the following properties:
    sso.enabled=true
    trusted.auth.user.retrieval=QUERY_STRING
    trusted.auth.user.param=X-SAP-TRUSTED-USER
    trusted.auth.url=https://<host><domain>:port/BOE/CMC/TrustedAuthSetTokenServlet

Replace the host and domain with your actual host and domain name.

   3. Create a new file, or edit an existing file called:
      FioriBI.properties

   4. Add / edit the following properties:

     authentication.default=<AuthType> 
     sso.supported.types=trustedParameter
     authentication.visible=true

Where <AuthType> is one of:
secWinAD, secSAPR3, secLDAP, secEnterprise
Whichever is the desired authentication type to do the trusted authentication must be configured here

Save the properties file and restart WACS in CMC.

Check if the same properties files are updated under <Installation Directory>\SAP BusinessObjects\SAP BusinessObjects Enterprise XI 4.0\java\pjs\container\work\MySIA.WebApplicationContainerServer\businessobjects\BOE\WEB-INF\config\custom 

If not, update this files in this location and restart WACS again.

Login to the BI Platform using Query string URL-
Example if using QUERY_STRING:
https://<HOST>:<PORT>/BOE/BI?X-SAP-TRUSTED-USER=<USERNAME>
Note: Administrator will not work

Keywords

WACS, trusted authentication, query string, X-SAP-TRUSTED-USER

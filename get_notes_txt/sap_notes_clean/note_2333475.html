Symptom

ct_get_data() stops processing rows leading to rows missing from the result when a warning, such as one that is raised when an untranslatable character is encountered was received.

Environment

  • SAP Sybase Software Developers Kit (SDK) 15.7 SP134, SP136
  • SAP Software Developers Kit (SDK) for Adaptive Server Enterprise (ASE) 16.0 SP02 PL03

Reproducing the Issue

  1. $SYBASE/$SYBASE_OCS/sample/ctlibrary/csr_disp.c
  2. In the program, change the CS_CURSOR_ROWS to 1, instead of 10.
     
    1. retcode = ct_cursor(cmd, CS_CURSOR_ROWS, NULL, CS_UNUSED, NULL, CS_UNUSED, (CS_INT)1);

 

Cause

  • FDS_EED token to result in stopping processing the rows, leading to rows missing from the resultset when a warning was received.
  • The issue has been identified and logged under SAP CR 795291. Its description reads, "CTLIB: ct_get_data() stops processing rows leading to rows missing from the result set when a warning, such as one that is raised when an untranslatable character is encountered, was received".

Resolution

  • This issue has been addressed in:
    • SAP SDK for ASE 15.7 SP137 and higher
    • SAP SDK for ASE 16.0 SP02 PL04 and higher
    • SAP SDK for ASE 16.0 SP03 and higher

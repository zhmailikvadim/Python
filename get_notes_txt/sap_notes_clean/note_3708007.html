Symptom

  • Address validation service does not validate certain addresses and allows them to be saved directly without validation.
  • When a invalid street number ( ex: 123456789)  is entered in the address field, city, region only warning message is shown.

Environment

  • SAP SuccessFactors Employee Central: Home Address

Reproducing the Issue

  1. Go to Manage Data > Address Validation Configuration for Country/Region.
  2. Set "
  3. Enter the address "12345645 ABCS Boronia Heights QLD 4124" or "1 Darter Cl Lowood QLD 4311" in the address field.
  4. Click Save. 
  5. The warning message shown " The city, region, and postal code are valid. However, the system could not identify the street in the address. "
  6. Click Proceed. 
  7. Incorrect address is saved successfully. 

Cause

This is expected behavior and is as per system design. 

Resolution

  1. Before the 2505 release, all messages returned from the DQM service were treated as errors.

  2. In 2505, message handling was improved: for certain messages related to input fields like street, building, house number, or postal code (with info codes in the 3xxx range from the DQM service), The error message shown while saving were changed to warnings, except for one type of message (info code 3090), which remains an error. (guide: Address Validation Messages for 3xxx Series Information Code | SAP Help Portal )

  3. To enhance address validation in the future for such cases, raise an enhancement request via the SAP Influence Idea platform.

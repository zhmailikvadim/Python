Symptom

When attempting to submit a requisition after selecting a specific Supplier/Vendor and SupplierLocation/Contact combination, the system displays the following validation error:

The line item has mismatched supplier and supplier location

The Supplier Location cannot be found or selected through manual search in the requisition. However, in Supplier Manager, the Supplier Location appears to be correctly assigned to the corresponding supplier.

Cause

The SupplierLocation/Contact is not properly mapped/linked to the PartitionedSupplier/Vendor

Resolution

  1. Run the Export Supplier Location Data (Consolidated File) task.

  2. Review the exported file and verify the VendorID value for the LocationID on which the validation error occurs. 

  3. If the VendorID value does not match the value selected in the user interface, the SupplierLocation cannot be used with that PartitionedSupplier.
    SupplierLocations cannot be moved/remapped from one PartitionedSupplier to another. 

    For a workaround, refer to KB0403475 - Error: "Got RuntimeException Supplier Location [123] is linked to supplier [XYZ]" during Import Supplier Location Data (Consolidated file)

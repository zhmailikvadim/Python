Symptom

  • A Purchase Order is generated (automatically or manually) in SAP S/4HANA Cloud based on the awarded quotation.

  • The Purchase Order Number generated in SAP S/4HANA Cloud is not sent back to SAP Ariba Sourcing.

  • The PO Number is missing from the Audit Logs in SAP Ariba Sourcing.

Environment

SAP S/4HANA Cloud Public Edition

Reproducing the Issue

  1. Create an RFQ in SAP S/4HANA Cloud and publish the RFQ to SAP Ariba Sourcing.

  2. Submit bids from the supplier side in SAP Ariba Sourcing.

  3. Award the sourcing event in SAP Ariba Sourcing.

  4. Verify that the award data flows to SAP S/4HANA Cloud and a Purchase Order (PO) is generated.

  5. Check the Audit Logs in the SAP Ariba Sourcing project.

  6. The S/4HANA Purchase Order number is not visible in the logs.

Cause

This behavior is by design for the Scope Item 4BL integration scenario.

  • SAP Ariba Sourcing is utilized strictly for the RFQ and Quotation Award stages.

  • Once the award is decided, the data is transmitted to SAP S/4HANA Cloud to create the follow-on document (Purchase Order).

  • The integration is designed as a one-way hand-off at this stage. The procurement cycle continues entirely within SAP S/4HANA Cloud.

  • There is no standard mechanism or supported workflow in Scope Item 4BL to send the generated Purchase Order number back to SAP Ariba Sourcing for confirmation or audit logging.

Resolution

This is standard system behavior. No further action is required.

  • The generation of the Purchase Order in SAP S/4HANA Cloud marks the completion of the sourcing process initiated in SAP Ariba Sourcing.

  • Tracking of the Purchase Order and subsequent procurement activities (such as Goods Receipt and Invoicing) must be performed within SAP S/4HANA Cloud.

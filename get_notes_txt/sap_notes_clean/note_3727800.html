Symptom

When implementing scope item 1P7 – Cross-Plant Transfer Posting, the Test Script references two plants (for example Plant 6910 and Plant 6920).

However, during the Organizational Structure step in SAP Central Business Configuration (CBC), only one plant is available and the second plant is not present.

Environment

SAP Central Business Configuration

Reproducing the Issue

  1. Go to the Central Business Configuration (CBC).
  2. Navigate to the workspace.
  3. Access the Organizational Structure step.
  4. Observe that only one Plant is created for Company Code when the scope activation completed.

Cause

The Test Script shows two plants only as an example to illustrate the cross-plant business scenario.

Resolution

In SAP Central Business Configuration, typically only one plant is initially available in the Organizational Structure. Additional plants required for specific scenarios must be created manually by the user.

If the scenario requires two plants, create an additional plant in the Organizational Structure step using the Set Up Organizational Structure activity in SAP Central Business Configuration. Follow the documentation Using the Set Up Organizational Structure App | SAP Help Portal on how to create a new Org Unit.

When working in an Evaluation workspace, organizational unit IDs must follow the naming rules:

  • Distribution Channels and Divisions: Create alphanumeric IDs with at least one alphabetic character.

  • Other Organizational Units (for example Plants): Create alphanumeric IDs where the last two characters must be alphabetic.

Example valid IDs:

  • 10AB

  • XYAB

After creating and confirming the additional plant in the workspace in SAP Central Business Configuration and further deployed via activity Deploy Scope and Organizational Structure, the new Plant should be available in the Starter System.

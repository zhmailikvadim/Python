Symptom

When changing the selection options in MD01N menu, MRP Live gets unexpectedly executed without any user confirmation, using navigation profile in MD04.

This behavior does not occur when MD01N is started directly.

Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.

Environment

  • SAP Enhancement package for SAP ERP, version for SAP HANA Release Independent
  • SAP S/4HANA, on-premise Release Independent
  • SAP S/4HANA Cloud Private Edition Release Independent

Reproducing the Issue

  1. Open MD04
  2. Assign a navigation profile (menu: Environment ->  Navigation profile -> Assign).
  3. Start MD01N using the shortcut in the MD04.
  4. Enter the mandatory field(s) on the initial MD01N selection.
  5. on the initial MD01N choose Goto → Selection Screen Help.
  6. Observe that MD01N starts without asking for user confirmation.

Cause

MD01N is launched from the MD04 and the initial screen is skipped whenever possible.

Resolution

In customizing transaction OPP1 (Material Requirements Planning) > Evaluation > Define Navigation Profiles > General transaction calls > set the flag "With initial screen" (V_T444B-ANFBD):

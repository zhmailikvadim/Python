Symptom

It is not possible to restrict the save an Account/Individual Customer when no Sales Data exists and a field (for example: Currency) on Sales Data is set as mandatory using Dynamic Property.

Environment

SAP Sales Cloud Version 2

Reproducing the Issue

  1. Pre-Requisite: Set a field (e.g., Currency) as mandatory using Dynamic Property within Sales Arrangement.
  2. Navigate to any Account/Individual Customer.
  3. Go to the Sales Data tab.
  4. Confirm that no sales data is present.
  5. Attempt to make changes to the Account.
  6. Observe that the save is executed successfully, contrary to the expectation of receiving an error message "Currency is a required field."

Cause

  • This is the expected system behavior of Dynamic Properties for sub-entities like Account/Individual Customer Sales Arrangement.
  • Currency for "Sales data" will become mandatory only if a Sales Data Instance (Sales Organization) already exists or is added.

Resolution

  1. You need to make sure an instance (Sales Org) in Sales Data is already maintained and then if the Currency is empty/not filled, the error will be raised.
  2. If you need a change in feature/functionality, kindly submit your idea using the Influence Portal.
  3. An alternative would be to explore options to achieve this by creating a Validation on Account service using External Hook.

Symptom

Custom fields under Employment Information may not display correctly during the Rehire Data Review (RHDR) step for rehire candidates.

For affected candidates:

  • Values are calculated or maintained in Recruiting and appear correctly in the candidate application.

  • Values are passed through Recruit-to-Hire Data Mapping.

  • However, incorrect or empty values are displayed during the Rehire Data Review step.

In some cases:

  • Values from a previous inactive employment record are displayed instead of the values passed from Recruiting.

  • If the previous employment record contains empty values, the RHDR fields may also appear empty even when Recruiting passes valid values.

Environment

SAP SuccessFactors Onboarding

Reproducing the Issue

  1. Proxy as a Rehire Coordinator.

  2. Navigate to To-Do and select Employment Profile Selection.

  3. Select the candidate and click Start.

  4. The Data Review for Rehire screen opens.

  5. Navigate to Employment Details and observe that custom fields under Employment Information display incorrect or empty values instead of the values passed from Recruiting.

Cause

Known issue identified by the Engineering team concerning custom fields under Employment Info not mapping correctly on the 'Data Review for Rehire' page.

Resolution

This is a recognized product issue currently under investigation by Engineering.

The issue is tracked internally under reference OBX-23577.

Workaround

As a temporary solution, manually correct the custom field values under Employment Information during the Data Review for Rehire step before submission.

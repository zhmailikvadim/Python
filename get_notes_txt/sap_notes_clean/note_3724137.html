Symptom

IAS Session Timeout configuration is handled differently in various scenarios. The most suitable configuration for the business' use-case has to be identified.

Environment

Cloud Identity Services

Cause

The business may require the session timeout setting of IAS or the external IdP to be considered.

Resolution

There are three distinct scenarios of note:

  1. IAS is acting as the IdP:

    The session timeout configuration set in the CIS admin console will be considered as described in Configure Session Timeout | SAP Help Portal.

  2. IAS is acting as a proxy with "Forward All SSO Requests to Corporate IdP" setting enabled:

    IAS session timeout configuration is disregarded. From the service provider side all authentication requests will be sent to the configured IdP, thus solely the session timeout setting of the IdP will be considered.
    More information can be found at: Forward All SSO Requests to Corporate IdP | SAP Help Portal 

  3. IAS is acting as a proxy with "Forward All SSO Requests to Corporate IdP" setting disabled:

    At the initial login of the session IAS will redirect to the IdP to authenticate the user. Afterwards, as long as the IAS session timeout is not reached, IAS will automatically reauthenticate and not forward requests to the IdP. In this instance IAS session timeout configuration is taken into account.
    More information can be found at: Forward All SSO Requests to Corporate IdP | SAP Help Portal

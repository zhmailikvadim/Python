Symptom

Budget data for multiple fiscal years has been uploaded for a project with budget availability control profile timeframe set as Overall. Now, an amount uploaded for a specific fiscal year needs to be removed while keeping the remaining amounts in other fiscal years.

Environment

SAP S/4HANA Cloud Public Edition

Cause

The issue arises due to the configuration in the import file. While the plan data scope correctly marks X on the fiscal year, the system does not actually account for fiscal year data because the budget for the project is managed throughout the project lifecycle. As a result, when attempting to upload data containing only the line with the fiscal year to remove the budget, and the budget for that line is set, the system inadvertently sets the budget data for the entire project as that value, disregarding the fiscal year specified in the plan data scope.

Resolution

Add an additional line to the import file with the appropriate budget amount for the project in a different fiscal year than the one targeted for budget adjustment / removal.

Symptom

  1. Running command "cf html5-list" in SAP Business Application Studio´s terminal fails with error:

    html5-apps-repo service is not in the list of available services

  2. Command "cf plugins" shows version 1.4.6 for the htlm5-plugin.

Environment

  • SAP Business Application Studio
  • CF HTML5 Applications Repository CLI Plugin

Reproducing the Issue

  1. Run command "cf html5-list" in SAP Business Application Studio´s terminal.

    The output shows error:

    html5-apps-repo service is not in the list of available services

  2. Set varaible CF_TRACE to true:

    export CF_TRACE=true

  3. Run command cf html5-list again.

    The log shows request:

    GET /v2/spaces/<space ID>/services HTTP/1.1

    failing with error:

    HTTP/1.1 429 Too Many Requests
    {"description":"Rate Limit of V2 API Exceeded. Please consider using the V3 API","error_code":"CF-RateLimitV2APIExceeded","code":10018}

Cause

  • htlm5-plugin version 1.4.6 uses CF API v2.
  • As stated in SAP Note 3586830, as of 19th Feb 2026, CF API v2 is not available anymore, and CF API v2 requests fail with status code 429 (Too Many Requests) and error message CF-RateLimitV2APIExceeded. The Cloud Foundry API v3 must be used instead.

Resolution

html5-plugin will be updated to version 1.4.9 in BAS on 4th March 2026. This version uses CF API v3.

See Also

SAP Note 3586830 - Update on Cloud Foundry API v2 Deprecation and Removal

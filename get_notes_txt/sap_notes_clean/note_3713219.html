Symptom

While doing SUM upgrade, there is an error in phase MAIN_DTTRANS/SUBMOD_RRC_STOP/RUN_RRC_READYFINAL  as below;

Last generated error message:
Single errors (code <= 8) found in log file 'PHASE_RUN.ELG'

Batchjob RSUPG_RRC_CRR on shadow system failed.
The following errors were detected in the log files:
# Find the detailed information in log 'RRC_READYFINAL.TP6':
A4 ESUPG 665 Report Name ...: RSUPG_RRC_CRR Component: BC-UPG-TLS-TLA (Upgrade Tools for ABAP)
A4 ESCRR 171 WP: Nr. 30, Report 'SAPLSDEB', Client '000', User 'DDIC', Status 'On Hold'
A4 ESCRR 002 "
A4 ESCRR 192 Using 10 of 13 batch processes
A4 ESCRR 002 "
A4 ESCRR_RRC_INF 002 Control Report RSUPG_RRC_CRR started with action READYFINAL at (timestamp)
A2EESCRR_ISU 018 ERROR: there are still 1.679 tasks not in state READY

Environment

  • Software Update Manager (SUM)

Reproducing the Issue

  1. Perform an SPS upgrade using Software Update Manager 2.0
  2. Proceed through the upgrade phases until reaching MAIN_DTTRANS/SUBMOD_RRC_STOP/RUN_RRC_READYFINAL.
  3. Observe that the checks after the phase MAIN_DTTRANS/SUBMOD_RRC_STOP/RUN_RRC_READYFINAL are negative.
  4. Error message is observed: Checks after phase MAIN_DTTRANS/SUBMOD_RRC_STOP/RUN_RRC_READYFINAL were negative! 

Cause

The issue occurs when CRR replication jobs (CL_CRR_AGENT_BATCH) terminate unexpectedly during the table replication process in SUM.

As a result:
The CRR agent batch processes remain in an inconsistent state.
The replication controller is no longer synchronized with the active replication jobs.

Resolution

Follow the procedure described in the SUM Guide section:

“Aborted CRR Agent Batch, Status Black in Phase DOWNCONF_DTTRANS or Error in Phase RUN_RRC_READYFINAL”

The steps involve restarting the CRR replication control process in the shadow system using function module CRR_TRANSFER_CONTROL, stopping the replication jobs, and restarting them with the correct number of replication batches.

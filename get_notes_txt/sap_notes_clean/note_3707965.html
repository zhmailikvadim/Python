Symptom

When opening the DS Planning Board with a chosen time profile, navigating to the Resource Planning table causes the time profile to switch to the profile configured in the Resource Planning profile.

Other Terms

Time profile, DS Planning Board, Resource Planning Table.

Reason and Prerequisites

By design, the Resource Planning table always uses the time profile defined in the Resource Planning profile. It does not inherit or follow the time profile selected on the DS Planning Board, which can result in different time views between the two screens.

Solution

Configure the desired time profile in the Resource Planning profile so the Resource Planning table uses that profile consistently.

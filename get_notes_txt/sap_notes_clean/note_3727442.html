Symptom

From DB02 -> Diagnostics -> Missing Tables and Indexes, can see /BIC/OU* view is in the 'Objects missing in the database' list.

When check the /BIC/OU* view in SE14, it says in the first screen the table does not exist in DB but can see the table in database level, and the corresponding infoprovider (the technical name is view name without the prefix /BIC/OU) is existing in BW or BW4HANA system.

Environment

  • SAP NetWeaver
  • SAP BW/4HANA

Resolution

  1. Make sure the SAP Note 2635913 - "DBDIFF entries for virtual tables, DDIC views and OpenODS views" has been implemented.
  2. Activate the DAP which has the same name as infoprovider using transaction code RSDAP. For multiple /BIC/OU* views, can use report RSDA_MASS_ACTIVATION (SAP Note 2249274) to activate the corresponding DAPs.

Keywords

/BI0/OU, RSDAP, RSDA_MASS_ACTIVATION, Archiving, aDSO, infocube

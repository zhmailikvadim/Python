Symptom

You use the report RPCIPE01 or RPCIPE00 to start the process of posting to accounting.  Line items with tax codes are incorrect and errors like  'Error during calculation of taxes with tax indicator' (3g210) and 'For tax code ..., the key date 00/00/0000 does not fall in any validity period' (3g001) are issued.

Other Terms

tax, txdat, tax date, MWSKZ, posting date, bapi_gl_acc_getdetail, tax_calc_dt_from, calculate_taxes_gross

Reason and Prerequisites

You post to an S/4 HANA Cloud system. Time-dependent tax is active in this system. You post HR line items with tax. The corresponding Payroll Result of table C1 contains already the tax code ( MWSKZ). The tax date needs to be determined for tax calculation.

Solution

The correction ensures that the start date is determined from posting date in case tax code is already available in payroll result.

Please implement the corresponding correction instruction for the corresponding HR Support Package.

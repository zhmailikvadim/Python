Symptom

When Sales Order creation is canceled (in a stateful environment, e.g. in SAP GUI) and another Sales Order is directly created, it can happen that data of basic methods (e.g. PAL Quantity Assignments) is saved without a Sales Order number.

Other Terms

aATP; end of check handling; document number notifier; PAL; Quantity Assignments; Sales Order integration;

Reason and Prerequisites

The reason is that the Document Number notification is not forwarded to basic methods (like PAL) triggered in such a scenario.

This is because the basic method controller used during save is only registered to the document number notifier when it is created.

But, the document number notifier is re-created in each check (e.g. after cancel), while the basic method controller is created only once (in the stateful environment). Hence, it is not re-registered and the new document number notifier cannot forward the sales order number to the basic methods.

Solution

Apply the correction instructions for this note or upgrade to a SAP S/4HANA support package that contains this correction.

Symptom

Hourly rate pay component is showing the annualized amount in the workflow page in mobile.

Environment

Add New Employee Wizard

Reproducing the Issue

  1. Add a new hire and include a pay component with an hourly rate.
  2. Submit the workflow.
  3. Approver views the workflow in the web version and sees the correct hourly rate for the pay component.
  4. Approver views the workflow via mobile, and the hourly rate is displayed as the annualized amount.

Resolution

Engineering has identified this as a defect in mobile and has prepared the solution to the issue and plans to implement the fix in Release 1H2026 (May, 2026).

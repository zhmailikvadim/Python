Symptom

You are either copying to a target document or canceling a reserve invoice and you receive the following error: 

Valuation method of item XXXXX has been changed after base document was added.

Reproducing the issue

Scenario:

  1. Go to Inventory → Item Master Data and add an item; on the Inventory Data tab, select a Valuation Method from the drop-down menu, for example, FIFO.
  2. Go to Purchasing - A/P -> A/P Reserve Invoice and add a reserve invoice with the item added in step 1.
  3. Go to Inventory → Item Master Data and recall the item added in step 1; on the Inventory Data tab, change the Valuation Method, for example, to Moving Average.
  4. Go to Purchasing - A/P -> A/P Reserve Invoice and recall the reserve invoice added in step 3 and try to copy it into a credit memo.

Expected behavior:

In step 3. an error should inform the user that it is not possible to change the Valuation Method because open reserve invoices exist.

Only when the open reserve invoices have been closed, should it be possible to change the Valuation Method.

Actual behavior:

An error occurs: Valuation method of item XXXXX has been changed after base document was added.

The reserve invoice can be neither canceled nor copied to a target document.

 

The same behavior is reproduced in Sales (A/R).

Cause

Application Error

Solution

SAP intends to provide a patch or patches in order to solve the problem described. The section Reference to Related Notes below will list the specific patches once they become available. The corresponding Info file of the patches in SAP Service Marketplace will also show the SAP Note number. Be aware that these references can only be set at patch release date. SAP will deliver patches only for selected releases at its own discretion, based on the business impact and the complexity of the implementation.

Symptom

Attempting to start the coordinator gets an error regarding a device in IQ_SYSTEM_TEMP, possibly as the result of a hardware failure on the device hosting the dbfile. An error similar to this may occur:

DBSPAWN ERROR:  -82 Unable to start specified database: autostarting database failed
Error: I/O Error on file /IQ_devlink/IQ_TEMP3_rdsk-- (hos_bio.cxx 775)
I/O Error on file /IQ_devlink/IQ_TEMP3_rdsk-- (hos_bio.cxx 775)
Database server shutdown due to startup error
Database server stopped at [DATETIME]
Error: I/O Error on file [PATH TO TEMP STORE DEVICE]
I/O Error on file [PATH TO TEMP STORE DEVICE]
Server failed to start

"Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental."

Environment

  • SAP IQ 16.1
  • SAP IQ 16.0
  • SAP IQ 15.4
  • SAP Sybase IQ 15.2

Reproducing the Issue

Attempt to start a Coordinator server with one of the devices underlying a temp store dbfile unavailable.

Cause

A Coordinator requries all temp store devices to be available to start.

Resolution

To replace a damaged dbfile, start the server with -iqnotemp, drop the file, restart the server, and add the dbfile in the coordinator.

  1. Use the -iqnotemp, -iqmpx_ov 1 and -iqmpx_sn 1parameter to start the database.
    The only temporary file operation allowed on a database while running with -iqnotemp is to drop one or more of the temporary file definitions.

  2. Drop the last file in IQ_SYSTEM_TEMP: 
    ALTER DBSPACE IQ_SYSTEM_TEMP DROP FILE filename
  3. Stop the database.

    IF THERE ARE STILL MORE GOOD IQ_SYSTEM_TEMP DBFILES ON THE SERVER THEN PERFORM THIS STEP:
  4. Restart the Coordinator normally and add more IQ_TEMP_STORE dbfiles as required.  The situation is now resolved.

    IF YOU DROPPPED THE LAST IQ_SYSTEM_TEMP DBFILE THEN CONTINUE WITH THESE STEPS:
  5. Start the database in single-node mode with no files to empty the checkpoint log: 
    start_iq @params.cfg mpxtest.db -iqmpx_ov 1
-iqmpx_sn 1 -n mpxnode_c
  6. Add a “dummy” dbfile to the coordinator. You will need to drop this file and add it again in coordinator mode, because files added in single-node mode have a null server ID; they are owned by the coordinator instead of the server that added the file. To add more space to IQ_SYSTEM_MAIN with a new file on a raw device, use syntax like the following:
    PlatformSyntax
    UNIX 
    ALTER DBSPACE IQ_SYSTEM_TEMP ADD FILE tempfile
'/dev/rdsk/c4t600A0B80005A7F5D0000024'
    where tempfile is the logical or chosen logical name of the new dbfile.
    Windows 
    ALTER DBSPACE IQ_SYSTEM_TEMP ADD FILE
tempfile '\\.\PhysicalDrive3'
  7. Stop and restart the server.

  8. Drop the dummy file.

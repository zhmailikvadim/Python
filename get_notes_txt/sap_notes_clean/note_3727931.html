Symptom

A user A creates an equipment by using transaction IE01. An external equipment 'XYZ' number is provided. A second user B tries to create an equipment with the same external equipment number 'XYZ' in parallel. Although the new equipment 'XYZ' is already locked by the first proccess of user A, user B is able to proceed.

The following error message for the second call of IE01 using the same equipment number is missing:

Equipment <xxxxxxxxxxxxxxxxxx> already exist (Message no. IS806)

Other Terms

IE01, IE02, IE03.

SERIALNUMBER_READ, equi_not_found, IS 806, IS806.

Reason and Prerequisites

Reason:

The error handling introduced with note 3727931 is to strict.

Prerequisites:

Note 3727931 is available in your system.

Solution

Correction via this note or the corresponding support package/hotfix.

Symptom

Snapshot replications for HANA SDA based Remote Tables are failing with information or error message:

  • "Replication action cancelled by request fort the table"
  • "Shadow replication task cancelled by request"

Environment

SAP Datasphere

Cause

The replication job completed successfully in the database.
The task failure was caused by an internal monitoring issue that prevented the system from correctly tracking the asynchronous execution.

Resolution

[UPDATE: 12:15pm UTC, Feb 2]

  1. The behavior explained under "Cause" section can be verified by looking up the replication job history or by verifying the data preview.
    Although the following error will be seen "Error: Shadow replication task cancelled by request in the UI", customer may ignore the error, if they can verify that the jobs are indeed completed successfully.
    To verify jobs are completed successfully, check the replication job history or perform data preview for the affected remote table.
  2. If the snapshot replication impacts task chains, customer may modify the task chains with task ports. Please refer to our Help Portal - Select Task Ports in a Task Chain
  3. Our development team is working on the fix.

[UPDATE: 11:50pm UTC, Feb 5]

  1. The fix will be deployed with the Datasphere Wave 2026.04. Refer to SAP Note 2984828 - SAP Datasphere Release & Delivery Schedule

[UPDATE: 11:50pm UTC, Feb 11]

  1. The fix has been deployed to the tenants with Datasphere Wave 2026.04. Refer to SAP Note 2984828 - SAP Datasphere Release & Delivery Schedule

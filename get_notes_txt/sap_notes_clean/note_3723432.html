Symptom

During the phase "MAIN_PREDTCHECKS/SUBMOD_S4H_CHECK_EXEC/RUN_S4H_SIF_CHECK_GET_RESULTS_EXEC" using SUM, the following error occurred:

A4 ESUPG 301 Report Name ...: RSUPG_SIF_CHECK_GET_RESULTS
A4 ESUPG 002
A4 ESUPG 001 --------------------------------------------------------------------------------------------------------------------------------------------
A4 ESUPG 003 Queue ID found: SUM_TRP0020594567_20260304023128
A4 ESUPG 003 Productuion stack number found: 73555000103300011058
A4 ESUPG 003 Target stack number found 73555000103300011058
A2EESUPG 005 Atleast one manual SIF CHECK run is expected between the first run and before Downtime.
A2EESUPG 005 Before Downtime run not found .
A2EESUPG 005 Run SIF CHECKS before downtime through report /SDF/RC_START_CHECK before proceeding to the next phase

Environment

  • Upgrade tools for ABAP(SUM)

Reproducing the Issue

  1. Launch SUM and start system conversion.
  2. Select option "Run Simplification Item Check Manually" during PREP_CONFIGURATION/INITSUBST phase
  3. SUM halts at MAIN_PREDTCHECKS/SUBMOD_S4H_CHECK_EXEC/RUN_S4H_SIF_CHECK_GET_RESULTS_EXEC phase .

Cause

  • The option "Run Simplification Item Check Manually" was selected during the upgrade process.
  • If this option is chosen, SUM will not be responsible for running the Simplification Item Check. 
  • The Simplification Item Check is either not performed, not completed or encountered error.

Resolution

  1. Before you proceed with the conversion, SUM verifies that the check is complete and error-free. If the check isn't complete or errors exist, the conversion process stops.
  2. Ensure that the report /SDF/RC_START_CHECK is executed correctly.
  3. Verify the output of the report /SDF/RC_START_CHECK and address any pending tasks or errors identified in the report.
  4. Review the instructions provided in the report and ensure that all required actions are completed before proceeding with the next phase of the SUM process.

Symptom

  • No physical table determined from the logical file name "example" in transaction code KEFC.
  • Error message GD163.

Environment

  • Contolling(CO) 
  • SAP ERP Central Component
  • SAP ERP
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA Finance
  • SAP S/4HANA

Reproducing the Issue

  1. Open transaction KEFC.
  2. Enter the input parameters
  3. Execute (F8)
  4. Observe the error message

Cause

The default value for data format  of the logical file path should be DAT for the entry "Example" in transaction code file, however it is not.

Resolution

  1. Change the default value for the data format of the logical file path to DAT.

See Also

3532713 - SG 813 "No Physical File Path is configured for Logical File name" while executing KEFC

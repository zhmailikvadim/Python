Symptom

  • Performance issues can arise when displaying the complete list of measure members, if there is a huge amount of them,
  • Some back-end errors can appear when the Custom Order panel opens.
    Examples include but are not limited to:
    - "A custom order cannot be created because of the following issue: Custom dimension members must not be aggregated: 'Version' is not in drill and is not filtered.",
    - "A custom order cannot be created because of the following issue: Unsupported operation: CustomDimension1 is missing",
    - "A custom order cannot be created because of the following issue: Invalid filter: the dimension attribute type has been modified",
    ...

Other Terms

Custom Sort, MMO, UQM

Reason and Prerequisites

It is now possible to create shareable Custom Orders on top of measures with the new build experience, the historical measure member order having been replaced by this more powerful and flexible feature.
Retrieving measure members reused the existing heuristic as for dimensions or accounts, which fetched data in the original design.
SAC can now exploit existing metadata, to pass them directly to the Custom Order panel - therefore skipping the fetching process which could cause slowness or back-end issues.

Solution

  • The performance issue and some back-end errors were addressed in 2026.03
  • Remaining back-end errors are currently being corrected

Symptom

Unable to delete certain characteristics from the UA Top Down Distribution (TDD) template; error "fields could not be changed" (FINS_ONE_ALLOCATION019).

Environment

    • Controlling and Profitability Analysis (CO-PA)
    • SAP S/4HANA
    • SAP S/4HANA Cloud Private Edition
    • SAP Fiori for SAP S/4HANA Finance
    • SAP Fiori for SAP S/4HANA
    • SAP Fiori for SAP S/4HANA Cloud Private Edition

Reproducing the Issue

  1. Open the top‑down distribution (TDD) template configuration in account‑based CO‑PA.
  2. Attempt to remove the PRCTR and SEGMENT characteristics, or change their status from "Retained and Relevant to Reference" to "Retained but Irrelevant to Reference".
  3. Observe the error "fields could not be changed" with message number FINS_ONE_ALLOCATION019.

Cause

Changing or removing certain G/L‑relevant characteristics (profit center PRCTR, functional area FKBER, segment) in TDD processing instructions is restricted by design to avoid inconsistencies in General Ledger postings; cost element changes are similarly restricted.

Resolution

  1. Do not remove or reclassify PRCTR and SEGMENT (and similarly FKBER, BURKS & Account) in TDD processing instructions; this is restricted by design.
  2. To distribute values across these characteristics, use TDD as intended: in the cycle maintenance, maintain the reference base for PRCTR/SEGMENT to map sender to receiver values.
  3. Run the TDD cycle and verify that values are distributed as required while original company codes remain retained in posted documents.

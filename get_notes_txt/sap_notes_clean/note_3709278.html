Symptom

You have received a notification email advising of the switch in OData Provisioning [NEO] Root Certificate from DigiCert Global Root G2 to DigiCert Global Root

Other Terms

DigiCert, Certificate Update, Certificate Renewal, OData Provisioning, Root CA G3, Root Certificate

Reason and Prerequisites

Environment
SAP ODATA Provisioning (NEO)

Cause
As part of our ongoing commitment to security and trust, SAP is updating the Root Certificate Authority (CA) used for certificates in ODATA Provisioning from DigiCert Global Root G2 to DigiCert Global Root G3. For more information, refer to the following SAP Note: 3666687.

Also, refer to the root certificate migration schedule provided in the See Also section.

Impact
After the update, client systems that do not trust the new Root certificate may encounter SSL certificate validation errors when accessing ODATA Provisioning endpoints. This may result in failed access to the OData services hosted on OData Provisioning.

Solution

Action Required

Customers accessing ODATA Provisioning endpoints from any client systems must:

  • Check trust: Ensure your client system trusts the Root CA: “DigiCert Global Root G3”.
  • Update if necessary: If this certificate is not present, you must download and add it to your client system’s trust store before the migration date to avoid service disruptions.

FAQ:

1. Are web browsers affected?
Web browsers connecting to the ODATA Provisioning Administration UI will generally remain unaffected, as the new Root CA is included in the latest versions of all major browsers and up-to-date operating systems. However, if your organization manages certificates manually, utilizes legacy browser versions, or maintains its own restricted trust store, you must manually add the new DigiCert Global Root G3 certificate to ensure continued access.

2. Do I need to trust Intermediate or Leaf certificates?
No. SAP recommends trusting only the Root CA.

3. Is there an impact on the Cloud Connector scenario?
Impact is not anticipated. The DigiCert Global Root G3 root certificate (used for SAP BTP Neo endpoints for Cloud Connector connectivity) is widely included in standard JVM distributions. As long as the Cloud Connector is running on a supported JVM version, the certificate should already be present. For legacy or highly restricted environments, we recommend a manual verification of the trust store.

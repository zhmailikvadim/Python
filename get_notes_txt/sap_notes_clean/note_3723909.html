Symptom

  • A 403 Forbidden error is triggered in the browser console during file upload via the FileSet OData entity in the Fiori app.
  • HTTP 403 Forbidden error occurs when uploading a file using the FileSet entity in the Fiori application.

Environment

  • SAP S/4HANA Private Cloud 
  • SAP S/4HANA On-Premise

Reproducing the Issue

  1. Log into the Fiori Launchpad and open any Fiori app that has File upload option
  2. Click on the app and upload the file. 

Cause

HTTP 403 Forbidden error occurs because the external network blocks the file request from reaching the SAP S/4HANA system.

Resolution

  • Reproduce the issue using the internal (direct) host instead of the external host.
  • Analyze the result:
    • If the file upload is successful using the internal host, the issue is related to the external host configuration.
    • If the file upload fails using the internal host as well, raise an incident under component OPU-GW-COR.
  • If the issue is confirmed to be with the external host:
    • For systems provisioned via SAP RISE, create an incident under component PCO-OPS, as the external host may be blocking the file upload request.
    • If the external host is managed by your internal network team, coordinate with them to review firewall, proxy, or network restrictions that may be blocking the file upload.

Symptom

Performance Issue while approver doing leave Rejection/Approval. It is taking time to do the action. 

The requests can be approved/rejected. It can occur by Fiori or WDA based Leave approval.

Environment

  • SAP S/4HANA 
  • SAP ERP 6.0
  • SAP Fiori 2.0 for HCM 

Reproducing the Issue

Approve a leave request.

Cause

It can occur, when the Duet BADI implementation is active by default, which causes the issue.

Resolution

The objective of Duet product BADi is to transfer the status of the leave request to Duet (to integrate into MS Office applications). 
Non Duet customers does not need default active implementation. 

  • Refer to SAP Note 1565933 for instructions on deactivating the Duet leave management BADI implementation.
  • Specifically, deactivate the BAdI implementation 'OSP_PT_GEN_REQ_INTER'.
  • Ensure that the Unified Key Mapping (UKM) integration scenario "DUET" is not active, allowing the system to skip storing key mappings, which prevents locking and expedites the leave request process.

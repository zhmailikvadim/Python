Symptom

Displaying an object link of type QM notification (for example QMQMEL) from a document may return the error DO | 410 in function module /DSCSAG/OBJECT_SAP_DISPLAY.

The issue may occur when displaying QM notification object links (for example CMQMEL, NMQMEL, PMQMEL, QMQMEL, SMQMEL) via function module /DSCSAG/OBJECT_SAP_DISPLAY.

Other Terms

(PLM-41927), /DSCSAG/OBJECT_SAP_DISPLAY, QMEL

Reason and Prerequisites

The function module reads the table name for the object type from field TCLO-OBTAB and uses it to determine field properties.
For certain QM notification object types, this field does not contain a suitable table name, which may lead to an error during the field information lookup.

Solution

Function module /DSCSAG/OBJECT_SAP_DISPLAY has been adjusted.
For QM notification object types, the header table QMEL is used to determine field information instead of the value from TCLO-OBTAB.

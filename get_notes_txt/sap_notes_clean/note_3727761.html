Symptom

During the TTRQ-mapping of serial numbers in the LE-TM integration a runtime error occurs. The issue is observed when serial numbers contain significant leading zeros as defined in transaction OLZSN. More details about the setting can be found in consulting note 3101810.

Other Terms

Logistics Integration; Serial Numbers; SN; Alpha Conversion; CX_SY_ITAB_DUPLICATE_KEY; /SCMTMS/CL_LE_TM_MAP_TTRQ=====CP

Reason and Prerequisites

SAP Transportation Management (TM) does not support significant leading zeros for serial numbers so far. When a numeric value is transferred, an automatic alpha conversion is applied. The runtime error arises when the system is configured with significant leading zeros and the LE document contains serial numbers which result into duplicates after applying the conversion logic.

Important:
The note will prevent the runtime error, but the alpha conversion of numeric serial numbers will still take place. TM will therefore not be able to differentiate serial numbers whose uniqueness depends solely on leading zeros.

If differentiation is required, it is recommended to:

  • Introduce at least one alphanumeric character in the serial number as mentioned in the Note given before
  • This can also be done just for logistics integration to TM using BAdI /SCMTMS/BADI_LI_BTD_CHG (method for adjusting serial numbers before mapping)

Solution

Implement the correction instruction attached to this Note or install the corresponding support package.

Symptom

The use of the BAdI 'Define Conditions to Restart Flexible Workflow for Purchase Requisitions(MM_PUR_S4_PR_WFL_RESTART) and detailed information about parameters and a sample code.

Environment

SAP S/4HANA Cloud Public Edition

Resolution

This Business Add-In (BAdI) is used in the MM-PUR-REQ component. Critical changes made to the existing purchase requisitions will restart the approval workflow. This BAdI can be used to enable or disable the restart of the approval workflow in purchase requisitions and to overwrite the configurations defined in configuration step Manage Conditions to Restart Flexible Workflow.

Note:

  • The configuration Manage Conditions to Restart Flexible Workflow supports only the purchase requisitions with creation indicators as ‘S’ and ‘R’. However, this BAdI can be used to enable or disable the restart of workflow for purchase requisitions with any type of creation indicator.
  • Custom fields are not supported in this BAdI. Note that the custom fields are not currently populated even though they appear in the BAdI interface.

The BAdI interface method CHECK_PR_WFL_RESTART has the following parameters:

Import Parameters:

  • PURCHASEREQITEMSPREV- This parameter provides the Purchase Requisition Item data before the change
  • PURCHASEREQITEMSCURR- This parameter provides the Purchase Requisition Item data after the change
  • ADDRESSPREV- This parameter provides the Purchase Requisition Address data before the change
  • ADDRESSCURR- This parameter provides the Purchase Requisition Address data after the change
  • ACCASIGNPREV- This parameter provides the Purchase Requisition Account Assignment data before the change
  • ACCASIGNCURR- This parameter provides the Purchase Requisition Account Assignment data after the change

 Changing Parameters:

RESTARTWORKFLOW: This parameter is used to overwrite the restart of approval process in purchase requisitions. Setting the value as ‘X’ for this parameter will restart the approval workflow and setting the value as ‘ ’ will prevent the restart of the approval workflow.

Example:

Suppose the requirement is to prevent restart of the approval workflow in purchase requisitions for a particular plant. E.g. plant as 'ABCD'. The following code snippet can be used in the Custom Fields and Logic app to implement this:

IF PURCHASEREQITEMSCURR-PLANT = 'ABCD'. "Purchase requisition plant

    CLEAR RESTARTWORKFLOW.

ENDIF.

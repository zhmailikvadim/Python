Symptom

After migrating/switching to a new Java Virtual Machine (JVM), the Cloud Connector fails to start, and the user receives an error message indicating issues with the configured cipher suites.

Ex. The browser shows:

ERR_SSL_CLIENT_AUTH_NO_COMMON_ALGORITHMS

Environment

SAP Cloud Connector 

Reproducing the Issue

Access the SAP Cloud Connector using the browser.

Cause

The set of supported ciphers may differ between the old and new JVM. If the selected ciphers are not supported by the new JVM, the Cloud Connector will fail to start.

Resolution

For release below 2.17.0,

  • Navigate to the Cloud Connector installation directory. [ex. C:\SAP\scc20]
  • Open the conf/server.xml file using a text editor.
  • Adjust the list of cipher suites manually to include only those supported by the new JVM. 
  • Restart the Cloud Connector.

For 2.17.0 and above,

  • Navigate to the Cloud Connector installation directory. [ex. C:\SAP\scc20]
  • From the OS level, run the "resetCiphers.bat"(Windows) or "resetCiphers.sh"(Linux) script.
  • Restart the Cloud Connector.

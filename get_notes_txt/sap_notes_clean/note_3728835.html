Symptom

  • KSeF ID is required for reporting or payment processing.
  • The KSeF ID is not available in accounting tables such as ACDOCA (e.g. field SGTXT).
  • No direct link is visible between accounting documents and KSeF ID.

Environment

  • SAP ERP
  • SAP S/4HANA
  • SAP S/4HANA Cloud Private Edition
  • SAP Document and Reporting Compliance, Cloud Edition

Reproducing the Issue

  1. Process eDocument for Poland (KSeF).
  2. KSeF ID is generated after submission.
  3. Check accounting tables (e.g. ACDOCA).
  4. Observe that KSeF ID is not stored in fields such as SGTXT.

Cause

KSeF ID is not stored in accounting tables (e.g. ACDOCA) as part of the standard integration.

The KSeF ID is maintained in eDocument-specific tables.

Resolution

KSeF ID information is available in table EDOPLINV.

Standard SAP functionality does not populate the KSeF ID into accounting fields such as ACDOCA-SGTXT.

If the KSeF ID is required in accounting documents or reports, this must be handled via custom enhancement.

Additional Information

  • The table EDOPLINV contains KSeF-related data for Poland eDocuments.

  • Standard SAP does not provide automatic linkage of KSeF ID to FI line item fields.

  • Customers can use enhancement techniques to make the KSeF ID available in reporting structures if required.

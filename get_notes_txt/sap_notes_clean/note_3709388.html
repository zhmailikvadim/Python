Symptom

  • Pricing conditions, which are not marked as 'Relevant for Accounting' (V_T683S-IS_ACCT_DETN_RELEVANT), does not have account key mapped in the pricing procedure, are incorrectly included in account determination during billing document processing.
  • This inclusion results in accounting errors.
  • The error message associated with this issue is "Account determination error".

Environment

  • Sales And Distribution (SD)
  • SAP R/3
  • SAP S/4HANA
  • SAP R/3 Enterprise 4.7
  • SAP ERP Central Component
  • SAP ERP
  • SAP Enhancement package for SAP ERP
  • SAP Enhancement package for SAP ERP, version for SAP HANA

Reproducing the Issue

  1. Access transaction VF02.
  2. Create a billing document.
  3. Release the billing document to accounting.
  4. Observe that pricing conditions not marked as 'Relevant for Accounting' and not mapped with ACCT key are included in account determination, leading to accounting errors.

Cause

If a Condition is NOT statistical (KSTAT = X) and NOT Inactive (KINAK not AKLMXZY and take part in the net value calculation it should be sent to FI side, if not during accounting document creation (FM RV_ACCOUNTING_DOCUMENT_CREATE), there would be value imbalance, and an error would be triggered (RW033). Even if ACCT Key/ GL acct is not mapped in Pricing pricing non-statistical condition would be considered for account determination.

When a condition is set as statistical and V_T683S-IS_ACCT_DETN_RELEVANT field is set in respective pricing procedure, this condition is relevant for account determination and could be seen CO document. 


Resolution

The condition type needs to be set statistical, and so during account determination this condition type would not be considered. If the condition type value considered in net value calculation, then the condition type must be handled through account determination.

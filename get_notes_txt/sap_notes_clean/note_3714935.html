Symptom

When trying to call Retrieve BOM API with the returned values from SFC Detail API you're facing the following error:

"type": "problemDetail.type.org.springframework.web.method.annotation.MethodArgumentTypeMismatchException",
"title": "problemDetail.title.org.springframework.web.method.annotation.MethodArgumentTypeMismatchException",
"status": 400,
"detail": "problemDetail.org.springframework.beans.TypeMismatchException",
"instance": "/api/v1/boms"

Environment

  • SAP Digital Manufacturing 2602 and higher.

Reproducing the Issue

  1. Use the Retrieve SFC Detail.
  2. Observe that the returned BOM type is "SHOPORDERBOM".
  3. Use the Retrieve BOM service with the input BOM type "SHOPORDERBOM".
  4. Notice that the service requires the input BOM type to be "SHOP_ORDER", causing inconsistency.

Cause

SFC Detail API returns the BOM type "SHOPORDERBOM" while the Retrieve BOM API expects "SHOP_ORDER", creating a BOM-type mismatch that requires additional scripting to map between the two formats.

Resolution

  • Currently, this behavior is working as designed.
  • On Production Process Design app you shall need an additional script task for mapping this data is necessary.

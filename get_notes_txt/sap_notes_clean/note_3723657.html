Symptom

During the creation or update of a JIT call, an external status may be provided via EDI. The system then attempts to determine a new internal status based on the action control logic, using the combination of the current internal status, the triggered action, and the received external status. If no action control entry exists for the specific external status, the system determines an internal status solely based on the combination of the current internal status and the action triggered, which may lead to an unintended internal status being assigned.

Other Terms

NJIT, Next Generation Just-In-Time Processing, JIT Call EDI Inbound processing, Action Control, External status

Reason and Prerequisites

Program error

Solution

Implement the correction instructions of this SAP Note or import the relevant Support Package.

After applying the correction, the system evaluates the action control using the combination of the current internal status, the triggered action, and a blank external status when no matching entry exists for the external status received via EDI. If no valid action control entry can be determined based on this logic, the system issues an appropriate error message.

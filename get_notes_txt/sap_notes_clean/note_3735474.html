Symptom

  • The SaaS Discovery inbox shows an incomplete application list; expected applications are missing.
  • Applications configured for SSO via Entra ID or Okta do not appear in SaaS Discovery.
  • No errors are shown in the SaaS Discovery sync log.

Environment

  • Product: SAP LeanIX
  • LeanIX SaaS Discovery integration Entra ID or Okta

Reproducing the Issue

  1. Open Inventory and navigate to Inventory tools > Discovery Inbox > SaaS Discovery.
  2. Review detected applications and observe that certain expected applications are not listed.
  3. Open the SaaS Discovery sync log and confirm that no errors are reported.

Cause

1. Custom (Non-Gallery) Applications
Applications created in Entra ID or Okta as custom (non-gallery) apps are not recognized by the LeanIX internal reference catalog. SaaS Discovery relies on Gallery template application IDs for identification. As a result, applications that are not based on these templates cannot be matched and will not be discovered.

There are limited exceptions: certain Microsoft applications may still be discovered even if they are configured as custom (non-gallery) applications.

2. Applications Outside of SaaS Catalog Scope
Some applications in your Entra ID or Okta environment may not meet the criteria for inclusion in the LeanIX SaaS catalog. These out-of-scope applications are intentionally excluded and will not appear in SaaS Discovery. You can find more about out of scope applications in our documentation: https://help.sap.com/docs/leanix/ea/applications-in-reference-catalog#applications-out-of-scope

Resolution

Resolution (Entra ID)
1. In Entra ID, review each missing application.
2. Verify whether it was created from a Gallery template (check the “Created from Gallery” property).
3. If the application is a custom (non-gallery) app:
- Create or switch to the corresponding Entra ID Gallery application
- Configure Single Sign-On (SSO) within the gallery-based application

Resolution (Okta)
1. In the Okta Admin Console, navigate to Applications → Applications and review each missing application.
2. Open the application and determine whether it was added from the Okta Integration Network (OIN) or created as a custom app.
3. If the application is a custom (non-OIN) app:
- Search for the same application in the Okta Integration Network (OIN)
- If available, add it as a new OIN application
- Configure Single Sign-On (SSO) within the OIN integration
- If applicable, manually reassign users and configurations from the custom app to the OIN-based application.

4. Trigger or wait for the next SaaS Discovery synchronization and recheck the Discovery Inbox.
5. Note: Custom/non‑gallery apps are currently not discovered by design (with limited exceptions). If a Gallery‑based app still does not appear, contact support for further investigation.

Symptom

SAP Data Services Real-Time Job that uses SAP HANA 2.X as Target fails with the following error message in the job error log:

  • "DATAFLOW <data flow name> None of the configurations for the specified target table <target table name> contain a database match <HANA 2.x> for the current datastore <datastore name>. Please resave the loader <loader name> with <HANA 2.x>configuration. For optimal results, always migrate data flows with associated datastore information. NOTE: If you migrated data flows without associated datastore information, import the necessary datastore from the source and add a new datastore configuration to match the new environment. When adding a new datastore configuration, set the "Use values from" option under "Values for table targets and SQL transforms" in the "Create New Configuration" window to use the configuration from which you migrated. You can make the new datastore configuration your default or use system configuration to instruct Data Services to use the new datastore configuration. For more migration information, see the Data Services Advanced Development and Migration Guide. 

Environment

  • SAP Data Services 4.x
  • SAP Data Services 2025
  • SAP HANA 2.x

Cause

There was an SAP Data Services migration via atl files.

Resolution

  1. Open the Target configuration in SAP Data Services Designer.
  2. Navigate to "Target"
    • Check and uncheck the "Make Port" option 
    • Execute the Real Time Again

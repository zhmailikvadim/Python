Symptom

You use SAP ACM and the universal prescription in document category version 3 to issue electronic prescriptions.

If you select "Not Specified" as the copayment status for an ePrescription-enabled order, the ePrescription is still displayed as subject to charges.

Other Terms

N3KVUNIRP, copayment, charge obligation

Reason and Prerequisites

Program error

Solution

In the copayment status, "Not Specified" can only be set in the following constellations:

For PHI insured persons or self-payers
Generally for private prescriptions
For ePrescription-enabled orders for these constellations, no entry is made for the copayment status in the preview and in the electronic order data record if the copayment status "Not Specified" is set.

Import the specified Support Packages to solve the problem. You can implement collective correction 3716853 in advance.

Symptom

You experience a high memory foot print of the CDM exposure report /UI2/CL_CDM3_EXP_PUBLISHER or background jobs. Depending on the system configuration the high memory consumption may even lead to program termination if it runs out of memory.

Other Terms

-

Reason and Prerequisites

During the calculation of the CDM representation of the exposed launchpad content all objects in all languages will be loaded into memory to traverse relations between the objects and ensure overall consistency of the dataset. Depending on the number of exposed roles and installed languages this could lead to very high memory consumption.

Solution

With the correction the memory consumption is reduced by avoiding the buffering of language dependent texts. This can reduced the memory footprint by about 50% depending on the actual exposure context. Still the number of exposed roles will drive the total memory consumption and should be kept as small as possible.

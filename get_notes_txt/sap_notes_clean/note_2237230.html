Symptom

 During preparations for a database recovery, when executing Backup List or Backup Check Access, the catalog-coordinator* indexserver hangs in shutdown and is no longer responding.

Other Terms

Multitenant Database Container, Backup, Recovery, High Isolation

Reason and Prerequisites

You are using SAP HANA with a revision <= 102.01.
You are using SAP HANA multitenant database containers, with high database isolation.

Due to the high isolation, inter-process communication is authorized using SSL handshakes. In rare cases, it may happen that a service tries to do this SSL handshake with itself, leading to a deadlock.

Solution

The program error is fixed with SAP HANA revision 102.02 (currently scheduled for Nov 5th, 2015).

Workaround: Manually abort/kill the hanging indexserver process, and continue with the recovery process.

Remarks:

*This word replaces an insensitive term that had been used in an older version of this SAP Note.

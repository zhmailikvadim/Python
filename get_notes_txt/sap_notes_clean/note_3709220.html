Symptom

  • Integrate Joule with SAP Products, SAP Build Work Zone, advanced edition is used
  • When configure the Identity Provisioning service to provision identities and their authorizations to SAP Build Work Zone, advanced edition, IPS job failed with error:
    error=HTTP operation failed invoking https://portal-service.cfapps.<region>.hana.ondemand.com/api/v1/scim/Users with statusCode: 404, Response: 404 Not Found: Requested route ('portal-service.cfapps.<region>.hana.ondemand.com') does not exist

Environment

  • SAP Joule
  • SAP Build Work Zone, advanced edition
  • SAP Cloud Identity Provisioning

Reproducing the Issue

  1. Run Joule Booster to integrate Joule with SAP Products, select SAP Build Work Zone, advanced edition.
  2. Login to SAP Cloud Identity Service to configure source and target system.
  3. For target system, select type as "SAP Build Work Zone, advanced edition"
  4. Run IPS job, observe the job failure with the error messages mentioned in the symptom section.

Cause

The issue occurs because the incorrect target system type was selected in the IPS configuration.

The target system type should be "SAP Build Work Zone, standard edition" when integrating business content from remote business providers to SAP Build Work Zone, advanced edition.

Resolution

To resolve the issue, follow these steps:

  1. Add a new target system for "SAP Build Work Zone, advanced edition", Choose "SAP Build Work Zone, standard edition" as target system type.
  2. Refer to Integrating Your SAP Solution for relevant SAP Products to configure required properties, please pay attention to below properties:
    URL: <Value of the portal-service property of the SAP Build Work Zone, advanced edition service key>
    User: <Value of the clientid property of the SAP Build Work Zone, advanced edition service key service key>
    Password: <Value of the clientsecret property of the SAP Build Work Zone, advanced edition service key>
    OAuth2TokenServiceURL: <Value of the UAA URL property of the SAP Build Work Zone, advanced edition service key + /oauth/token>

Symptom

When connecting to an ODBC datastore the below error can be seen when either editing the datastore or running jobs that use it.

ODBC call for data source failed: <[Drivertype][ODBC lib] Driver Manager Message file not found. Please check for the value of InstallDir in your odbc.ini.>. Notify Customer Support.

Environment

All available SAP Data Services versions

Cause

  • The issue appears to be caused by inconsistencies in the configuration of the odbc.ini and ds_odbcinst.ini files, specifically the headers not matching between these files.

Resolution

Edit the ds_odbcinst.ini file to ensure its header matches the header for the driver configuration in the odbc.ini file.

For example:

  • odbc.ini header: [DRIVERHEADER]
  • ds_odbcinst.ini header: [Teradata Database ODBC Driver 17.20]
  • Update the ds_odbcinst.ini header to [DRIVERHEADER].

Important: Perform the edit via $LINK_DIR/bin/. Open the ds_odbcinst.ini file and change its name to match the header in the odbc.ini file.
                   Restart the SAP Data Services application to apply the changes.

Symptom

When you run GL account balance transactions, such as FAGLB03, FS10N there is below dump happens:

----------------------------------------------------------------------------

Category ABAP programming error

Runtime Errors RAISE_EXCEPTION

ABAP Program FAGL_ACCOUNT_BALANCE

Application Component FI-GL

 

A RAISE statement in program "CL_ABAP_TYPEDESCR=============CP" has raised

exception condition "TYPE_NOT_FOUND".

ABAP Variable Value (Char.) V0 ACDOCA-COST_ELEM
----------------------------------------------------------------------------

Environment

  • Financial Accounting (FI)
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA Finance
  • SAP S/4HANA

Reproducing the Issue

Execute transaction FAGLB03.

Resolution

The dump is caused by your additional field COST_ELEM in the following IMG path:

=======================

Financial Accounting

  General Ledger Accounting

    Information System

      Define Balance Display
=======================

 

The COST_ELEM is with the S/4HANA NOT a field anymore that can be used in the balance display,

as this field was merged with the G/L account (CO and FI use the same table to store the data).

Therefore this value must be removed in the related above IMG path.

Keywords

FD10N, FK10N, CI_COBL, DDIC_TYPE_INCONSISTENCY, ACDOCA, FAGL_ACCOUNT_BALANCE, RAISE_EXCEPTION

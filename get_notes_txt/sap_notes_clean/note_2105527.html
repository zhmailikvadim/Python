Symptom

After the automatic upload of credit card receipts in a travel plan / travel request, the trip will still have the schema PL. However when you attempt to create the expense report based on this plan / request in the WDA application, you are not asked to change the travel schema to a real expense schema. This leads to error message "Enter a trip schema when creating a trip".

Other Terms

FITE_EXPENSES, Webdynpro, schema,

Reason and Prerequisites

Design

Solution

Apply the note or the support package. Then you will always be asked to change the trip schema if it is currently PL.

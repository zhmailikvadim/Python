Symptom

You are using the SAP Cloud Logging service to ingest via Cloud Foundry Runtime and would like to understand how to restrict user access to namespace-level logs in the OpenSearch Dashboard.

Environment

  • SAP Business Technology Platform (SAP BTP)
  • SAP Cloud Foundry (CF)
  • SAP Cloud Logging service

Reproducing the Issue

Try to fetch any log in OpenSearch Dashboard. 

Resolution

SAP Cloud Logging service is released with default index patterns and dashboards. The steps are:

  • Assign the needed read-only Role to Users in OpenSearch.
  • Create a role mapping.
  • Assign additional permissions required for accessing dashboards or index patterns:
    **********************************************************
    PUT _opendistro/_security/api/rolesmapping/global_tenant_read_access{  "backend_roles": ["cloud-logging-read-only"]}PUT _opendistro/_security/api/rolesmapping/readall{  "backend_roles": ["cloud-logging-read-only"]}
    Cloud Logging is provided with default index patterns and dashboards. If you cannot see them, something went wrong with the permissions.
    **********************************************************

    For more details, check: SAP Cloud Logging Service: Configuration of a read-only user.

    NOTE:
  • For more fine-grained permissions, check: OpenSearch Permissions.
  • To check the access assigned to a user, click your user ID on the top right and select: "View Roles and Identities".

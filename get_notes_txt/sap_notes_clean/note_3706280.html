Symptom

You create wage statements for employees with multiple payroll (Concurrent Employment, CE) as a form in accordance with the guidelines of the wage standard ELM as of Version 5.0. To do this, execute the program Wage Statement (as of ELM 5.0) (RPLLAWC3_CE, transaction: PC00_M02_LLAW3_CE).

If you create wage statements for employees for whom retroactive accounting to the previous year took place in the notification year, the additional sheet is not printed. This is the case if only correction amounts from the previous year exist for the data generation of the additional sheet.

Note: The program Data Extractor for Uniform Wage Notification Procedure (ELM) (report: RPLELMC0, transaction: PC00_M02_LELM0) in version ELM 5.0 Document Category: '0005' - "ELM 5.0 - PIV/EIV") works correctly because in this case, no amounts are reported that correspond to the additional sheet.

Other Terms

Payroll Switzerland; wage statement (ELM 5.0); domain: LAW; class: CL_HRPAYCH_ELM50_DAQ_LAW; methods: CE_GET_OUTPUT, CE_CONDENSE_OUTPUT, CE_SAVE_COMMONDATA

Reason and Prerequisites

The described change is required for productive use as of ELM Version 5.0. It is only required if you use multiple payroll (Concurrent Employment, CE).

Solution

Import the HR Support Package relevant for your release. If you have to implement the solution earlier, use the SAP Note Assistant to implement the correction instructions relevant for your release. You can find more information about the SAP Note Assistant on SAP Support Portal at https://support.sap.com/notes.

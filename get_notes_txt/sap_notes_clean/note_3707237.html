Symptom

Executing a CSA policy - PS_PW_COMP, check id - 1.20.01 1.08 (few other check Ids related to HANA Database parameters). It is reporting "No Data Found"

Environment

SAP Focused run

Reproducing the Issue

Eexecute CSA policy - PS_PW_COMP, check id

Cause

Attribute DB_REPL_OP_MODE:!LOGREPLAY is missing in LMDB

Resolution

Ensure that you use `system_attributes="DB_REPL_OP_MODE:!LOGREPLAY"` in the policy configuration. This will allow the system to bypass the secondary databases effectively.  

For further guidance, you can review the publicly available policy documentation at the following link:  https://raw.githubusercontent.com/SAP-samples/frun-csa-policies-best-practices/main/NotesPolicies/HANA/HANASecNotes_2023-11.xml

 Within this document, you will find an example illustrating the correct placement of the string:  `system_attributes="DB_REPL_OP_MODE:!LOGREPLAY"`

In LMDB, set the value of the attrubute DB_REPL_OP_MODE to logreplay for both systemdb and tenantdb entries of secondary db.

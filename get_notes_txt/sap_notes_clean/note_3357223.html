Symptom

When running a VBS script to fetch the last modification date (ModificationDateInRepository) of an object immediately after establishing a connection, the value returned differs when the script is executed a second time. This should not be the case.

Environment

SAP PowerDesigner 16.7 SP07

Reproducing the Issue

1. Connect to a repository containing one Physical Data Model at the root.
2. Execute the following VBS via Tools > Execute Commands > Edit/Run Script twice.

Dim oRepository, oRepoDoc
Set oRepository = RepositoryConnection
For Each oRepoDoc in oRepository.ChildObjects
   Dim sName, sClassName, dtModDateTime
   sName        = oRepoDoc.Name
   sClassName   = oRepoDoc.ClassName
   If sClassName="Physical Data Model" Then
      If oRepoDoc.HasAttribute("ModificationDateInRepository") Then
         dtModDateTime = oRepoDoc.GetAttribute("ModificationDateInRepository")
      End If
      Output oRepoDoc.Name & vbTab & dtModDateTime
   End If
Next

Note: The modification date does not change when executing the following SQL:

select   O.NAME as 'Object Name'
,        O.OMDT as 'Modification Date'
from     PMOBJT O
join     PMRLSH R on R.TRGT=O.OBJT and R.PRNT = 1
join     PMOBJT F on F.OBJT=R.SRCE
where    O.LVRS = 1
and      O.CLSS = 1564 -- Physical Data Models
and      F.NAME = 'Repository Root'
order by O.NAME

Cause

The SAP CR 829274 has been logged to address this new feature request. Its description reads, "VBS: Value of ModificationDateInRepository is wrong".

Resolution

There is currently no target SAP PowerDesigner 16.7 SP07. XX Service Pack scheduled for this issue SAP CR 829274 . It is being assigned to our Product Group.

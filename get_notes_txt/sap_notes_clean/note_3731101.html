Symptom

When double-clicking an entry of type “Customizing Entries for Business Add-In Implementations” in Transport Organizer, the following error occurs:

Incorrect internal lock argument:   0
Message No. TK312

Diagnosis
The request locks for lock object  (sample length 0 and lock key ) cannot be selected.

Reason:
Object = "Space" or
Key = "Space" or
Sample length is greater than 64
System Response
The function was cancelled.

Environment

  • SAP_BASIS 758 until SP06
  • SAP_BASIS 816 until SP02

Reproducing the Issue

  1. Open transaction SE10 (Transport Organizer).
  2. Expand a transport request and locate an entry of type “Customizing Entries for Business Add-In Implementations” (logical object type SXCC).
  3. Double-click the entry.
  4. Observe error “Incorrect internal lock argument: 0” (TK312).

Cause

The generic Workbench display tool attempts a lock check with an empty lock key for SXCC (BAdI activation) entries, which are non-lockable; this leads to TK312 and a misleading error.

Logical transport object type SXCC is not integrated into the Workbench generic display tool; these entries represent modification-free activations performed via the customizing tool, not modifiable implementations. Direct navigation from SE10 is not supported and there is no one-to-one mapping to a specific customizing entry.

Resolution

  1. Implement SAP Note 3721791 to remove the misleading TK312 error in the generic display tool.
  2. Re-test double-click on the SXCC entry in SE10 and confirm the lock argument error no longer appears.
  3. Note the product limitation: direct navigation or detailed display for SXCC (BAdI activation) entries is not supported in the Workbench generic display tool.
  4. Use the respective customizing tool that was used to activate the BAdI to review or change the activation. Be aware that there is no one-to-one navigation from the transport entry to a specific customizing node.

Symptom

Duplicate time confirmations are being posted in the backend when performing time confirmations using SSAM 2310.

SAP note below are applied:

  • SAP Note 0003300038 - SAP Asset Manager - Final confirmation gets duplicated when supervisor cancels add confirmation time
  • SAP Note 0003563050 - SAP Service & Asset Manager: Double Confirmations Created when creating a Final Confirmation for an Operation

Environment

SAP Service and Asset Manager 2310 (or other versions)

Reproducing the Issue

Case 1:

  1. Select any work order.
  2. Complete the operation and click "Done."
  3. Sync the app.
  4. Observe that the time confirmation is posted twice in the backend.

Case 2:

  1. Start one operation.
  2. Pause the operation.
  3. Record the time for the operation.
  4. Do it with multiple operations randomly time will get recorded twice.

Case 3. 

  1. Start one operation.
  2. Pause the operation.
  3. Record the time for the operation.
  4. Kill the app before sync completed.
  5. Once again open the app and do sync. Time will be recorded twice.

Cause

Resolution

Update the offline settings in the BTP Mobile Services configuration as follows:

  1. Navigate to the BTPms cockpit and locate the mobile app.
  2. Choose "Offline Access" and edit the configuration for `DEST_SAMXXXX_PPROP`.
  3. Enter the following details:
  • `check_repeatable_requests=Y`
  • `repeatable_exclude_status=300,301,302,303,304,305,306,307,308,400,401,403,404,500,501,502,503,504`
  • `send_original_repeatability=N`
  • `send_standard_repeatability=N`

Save the changes and ensure the updated configuration is applied.

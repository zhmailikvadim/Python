Symptom

When I run a worker download connector, specific custom fields (e.g., Assignment Category) display outdated or incorrect values for workers assigned to an SOW. While the correct, updated values are visible in the SAP Fieldglass User Interface (UI), the download file continues to show values from a previous or original SOW revision.

Cause

By system design, standard worker-based connectors are often configured to fetch data from the specific SOW Revision that was in effect at the moment the worker was originally submitted or onboarded.

If the worker was onboarded under Revision 0, the integration pulls the custom field values present on that revision. Subsequent updates made to those fields in Revision 1, 2, or later will be reflected in the UI, but the connector logic continues to reference the "original" data associated with that worker's initial record.

Resolution

This is the intended default behavior for standard worker downloads. To align the download output with the latest data seen in the UI, you must request a technical modification to your connector.

  1. Verify SOW Revision: Open the worker's record and check the SOW Worker Details section. Note the "SOW Revision in effect at the time the Worker was submitted" and compare its values against the latest revision.
  2. Submit a Modification Request: If your business requires the connector to pull from the latest active revision instead of the original one, you must submit a Connector Modification Request to SAP Support.
  3. Consulting Review: SAP engineers will review the underlying SQL logic of your specific connector (e.g., reviewing expressions like isnull(ptrcf4.value, ptrcf3.value)) to adapt the "custom feature" of the download to target the most recent revision.
  4. Verification: Once the logic is updated, test the download to ensure the expected values (e.g., Vendor.Managed Project instead of Temp) are now being captured.

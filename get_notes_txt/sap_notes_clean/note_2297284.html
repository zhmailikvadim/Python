Symptom

A mass locking of all transactions codes are needed in order to fulfill a specific business requirement (e.g. the system is being audited, or it's needed to lock the system to end-users), but it is not possible.

Environment

NetWeaver Release-independent

Cause

This is actually a system standard behavior, not an error.

Resolution

SM01 is meant to be used to restrict access to specific transaction codes while changes in the system are being done. The massive lock is not permitted in SM01 as it can damage the system.

If a mass lock is carried in all the transactions of a system, no one will be able to revert the process. In case an audition requires the system to not be used during a timeframe, there are other means to inform the users to not connect to the system.

A better way to do such maintainance is to lock the users from login on the system in a timeframe. This can be done by:

  • Locking the client;
  • Locking users/group of users in SU01/SU10;
  • Advising end users system won't be available during a timeframe;

For more information, check with the Security team the best option that fits in your case.

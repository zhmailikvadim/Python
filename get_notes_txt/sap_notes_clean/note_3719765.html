Symptom

There is a need to provision the employeeId from Microsoft Entra ID and assign to the user's loginName in Identity Authentication, using Identity Provisioning. 

Environment

  • Identity Provisioning
  • Microsoft Entra ID
  • Identity Authentication

Resolution

1. In the source system side:

  • In the property aad.user.attributes, append employeeId as a value
  • Change the sourcePath for the following mapping
    {
    "sourcePath": "$.employeeId",
    "targetPath": "$.userName" 
    }

2. Run a new job, and the employeeId will be provisioned from Entra ID to Identity Authentication.

Symptom

  • After running the upgrade for either IPS or BI the SIA (Server Intelligence Agent) and associated servers do not start
  • After starting, it goes immediately to stopped state

Other Terms

 Keywords

IPS 2025, IPS not starting, IPS upgrade failed, IPS SIA not starting, servers not starting IPS 2025

Reason and Prerequisites

Environment

  • SAP BusinessObjects Business Intelligence (BI) Platform 2025
  • SAP Information Platform Service 2025
  • Linux and Windows

Reproducing the Issue

  1. Run the upgrade to IPS / BI 2025
  2. Try to start the servers
  3. Servers failed to start
  4. Can't login to CMC

Cause

The cause is currently under investigation by Development.

Solution

The issue is currently under investigation by Development.

A few workarounds exist to help resolve this situation.

Workaround #1 

A few java options in the SIA configuration were not updated from Java 8 to Java 17 convention, which may have caused SIA to not startup -

  1. Go to the sap_bobj folder and take a backup of ccm.confg
  2. Remove these parameters from ccm.config [note: these are no longer valid in Java 17 which is current Java version used 2025 and higher]
    "-XX:+PrintGCTimeStamps"
    "-XX:+PrintGCDetails"
    "-XX:LogGcMaxFileCount=3"
    "-XX:LogGcMaxFileSize=5m"
    "-XtraceFile=/usr/sap/SAP/sap_bobj/logging/server_jvm_@PID.log" 
    GCHistoryFilename=/usr/sap/SAP/sap_bobj/logging/server_gc.prf"
    -Xloggc*:file=/usr/sap/SAP/sap_bobj/logging/server01_jvm_%p.log:tags,time,uptime,level:filecount=3,filesize=5M"

Workaround #2
 Create a new node with default servers using the temporary CMS option.

  • Linux:1571993 - How to a add a new node in BOE 4.x and BI 2025 UNIX Environment - SAP for Me
  • Windows:To add a node on Windows | SAP Help Portal

NOTE:  After restarting SIA, If the APS still does not startup, please refer to SAP Note 3571941 - Cloned or Newly Created Adaptive Processing Server (APS) Server Fails to Start in Upgrade Scenario

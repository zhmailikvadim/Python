Symptom

Issue encountered while performing customization steps from SAP Note 3710009 - Electronic Document Processing for Poland - KSeF: Offline PDF Generation and Technical Corrections as below:

"Entry PL_KSEF_SEND_REQUEST 0004 does not exist in EDOINTVERSION "

Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.

Environment

  • SAP ERP
  • SAP S/4HANA
  • SAP S/4HANA Cloud Private Edition
  • SAP Document and Reporting Compliance, Cloud Edition

Reproducing the Issue

  1. Access transaction code SM30.
  2. Enter View EDOINTAIFV.
  3. Input entries for Interface PL_KSEF_SEND_REQUEST.

Cause

The issue arises due to incorrect sequencing of the customization activities.

Resolution

Follow the corrected sequence for customization activities as outlined below:

  1. Maintaining eDocument Interfaces Version.
  2. Assign AIF Interface and Version to ERP Interface.
  3. Interface ID Determination for Process Step.
  4. Process Step Determination for Interface ID.
  5. Assignment of eDocument Status to Process Status.
  6. Adding Billing Header Text ID for PL Offline’s Original Document Number.
  7. Enabling PL Offline’s Original Document Number for FI Document.

Refer to the updated SAP Note 3710009 latest version, which includes the corrected sequence from the attachment section named as SAP Note 3710009 Customization Steps.pdf

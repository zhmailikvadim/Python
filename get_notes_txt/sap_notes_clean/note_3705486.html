Symptom

In SAP Service & Asset Manager, users can enter fractional values for inspection characteristics. The number of fraction digits is not consistent between the backend and the client application, resulting in different decimal precision being displayed or accepted.

Other Terms

  • SSAM
  • Inspection Characteristics
  • Fractional value
  • Decimals

Reason and Prerequisites

 Reason

The previous implementation of this logic did not enforce the configured number of decimals by allowing trailing zeros to be trimmed. It also did not consistently respect the formatting template (e.g. thousands separator)

Prerequisites

  •  BAS or Visual Studio Code with SAM MDK project
  • `patch` command (available on BAS and macOS by default) via terminal or other patching utility
  • Read the guidelines on how to apply the patch and read the appendix section if you receive any error

Solution

This patch standardizes decimal rendering and makes number format respect both configured decimal places (NumOfDecimal) and formatting template rules (comma vs. grouping)

To apply the patch, download the attached 3705486.patch file and save it in a known location, preferably adjacent to your SAM.mdkproject.

If you are on Windows, see SAP Note 3533879 for how to install a .patch file in Windows. If you are on a Mac, follow the steps below:

  1. Open a terminal in the root of your SAM metadata (i.e. SAM.mdkproject/metadata or SAM.mdkproject/metadata/base.definitions)
  2. Run the terminal command `patch --no-backup-if-mismatch -p4 < path/to/3705486.patch`. 
  3. Verify in the terminal window that the following files have been modified and listed as patched, as indicated by the console output:
  • patching src/metadata/definitions/Rules/Classification/Characteristics/CharacteristicValue.js

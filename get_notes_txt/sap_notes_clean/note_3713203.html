Symptom

  • The values of WRPL-MINBE and WRPL-EISBE are not synchronized after MARC-MINBE or MARC-EISBE were changed if MARC-DISMM remains unchanged.
  • Synchronization occurs only when MARC-DISMM is modified.
  • There is an explicit condition for this behavior in the function module SET_WRPL_LIKE_MARC.

Environment

  • Retail
  • SAP ERP
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA
  • SAP S/4HANA Cloud Private Edition

Reproducing the Issue

  1. Ensure material exists for the same distribution center in both tables MARC and WRPL.
  2. Change the value of MARC-MINBE or MARC-EISBE and save. Observe that there is no synchronisation with WRPL.
  3. Change the value of MARC-DISMM and save. Observe that the WRPL-entry takes the value of all three fields.

Cause

  • The MRP type RP should not be used for Distribution Centers.
  • RP type W means "store replenishment" which is done on  store level (Plant category B).

Resolution

There are other FMs that can be used to update WRPL:

  • WRPL_PREPARE_POSTING: checks on reference plant level difference and sets WRPL data accordingly.
  • WRPL_SET_UPD_INS_TAB: updates the global table for update processing.
  • WRPL_POST: posts the new WRPL data.

Symptom

Cross-Plant Configurable Material (MARA-SATNR) or Plant Specific Configurable Material (MARC-STDPD) not updated during the inbound process of MATMAS iDoc, even though the iDoc contains data for the relevant fields.  

Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental. 

Environment

  • SAP ERP Logistics (LO)  
  • SAP R/3
  • SAP R/3 Enterprise
  • SAP ERP Central Component
  • SAP ERP
  • SAP Enhancement package for SAP ERP
  • SAP Enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4 HANA on premise
  • SAP S/4 HANA Cloud Private Edition

Reproducing the Issue

  1. Process MATMAS iDoc inbound in t-code BD87.
  2. IDoc successfully proceed, but configurable material is not updated. 
  3. Check the iDoc data, field SATNR is contained in segment E1MARAM. However, the change is not reflected in the material. 

Cause

When material is maintained by background processing functionalities like iDoc/ALE, configurable material is only updated if configuration data is present.  

The inbound iDoc does not contain any configuration data, therefore configurable material is not updated.  

An iDoc containing configuration data should include the following segments:  

E1CUCFG                        CU: Configuration data

    E1CUINS                        CU: Instance data
    E1CUVAL                        CU: Characteristic valuation
    E1CUCOM                        Master configuration profile configuration parameter

Resolution

Review the material data maintenance and iDoc outbound procedure. 

  1. Ensure the material is configured as a variant in the source system. 
    • Verify in t-code MM03 that the variant indicator is activated and the "Configure variant" button is available.  
    • If the variant indicator is inactive and the "Configure variant" button is grayed out in t-code MM03, refer to KBA 1931917 and complete the configuration.  
      1931917 - 'Specify a valid configurable material' (M3544) issued when trying to configure a Material Variant
  2. After the material is correctly configured as a variant, ensure MATMAS iDoc is correctly created in the outbound system.   
    • Distribute the material in t-code BD10 with the "Send material in full" indicator activated.  
    • Check the created iDoc in t-code BD87 and verify that the segments for configuration data are correctly generated.

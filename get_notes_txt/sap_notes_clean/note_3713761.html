Symptom

When downloading the operational report data from Fiori, instead to downloading data in an accurate row, data is showing in an extra additional row with space is coming in the downloaded file. This is due to the values contains the ‘#’ init. This leads to issue across multiple file formats (.TXT, .XLS, and .CSV) and results in extra blank row or spaces appearing in the downloaded reports.

Other Terms

/VIZI/OPER, /VTA/CL_BFTP_SERVICES, EXPORT_ROW_PROCESS, Extra blanks row, Extra spaces, Operational report, Exporting file, download file

Reason and Prerequisites

This is caused by program error.

If you are 2022 V2, Please download and implement the latest version of SAP Note # 3380333 as pre requisite note.

Solution

Please implement the correction instructions attached. These corrections are only valid if you have applied the Support Packs.

Release V4 - 2022 V2, 2023, 2023V2, 2023 V3 and 2023 V3 Support Pack 1.

Use transaction SNOTE to implement the correction instructions specified below and activate the affected program object.

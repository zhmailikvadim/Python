Symptom

  • You would like to disable page locking on a table to improve concurency
  • You would like to avoid deadlocks on data pages

Environment

  • Netweaver running SQL Server - all supported version.

Cause

  • Inserts are taking long time
  • Concurency problems
  • Deadlocks
  • Performance problems due to blocking

Resolution

Solution 1 using Microsoft Management Studio :

  1. Expand your <Database> -> 'Tables' tree
  2. Find a specific table
  3. Expand indexes
  4. Right click on an index and choose 'Properties'
  5. Select 'Options'
  6. Uncheck 'Use page locks when accessing the index'
  7. Repeat for all indexes

Solution 2 using 'Query':

  1. In Microsoft Management Studio open a 'New Query' window
  2. Make sure to be connected to your database as database administrator.
  3. Run following : EXEC sp_indexoption <table>, <option> , <true|false> e.g. :
    EXEC sp_indexoption [REPOLOAD], "DisAllowPageLocks", "TRUE"

Note : This change can be done while database is use. When table is in use server may return e.g.:
"Lock request time out period exceeded. (Microsoft SQL Server, Error: 1222)" error.
When this happens, just retry the same operation later when table is in use (not locked).

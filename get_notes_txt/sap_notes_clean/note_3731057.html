Symptom

One or more application communication assemblies (characterized by the naming convention /namespace/APP_name_of_the_ca) are not assigned to the default communication assembly (CA) configured in the system as their respective parent CA. This configuration may lead to unexpected behavior of the UCON framework and incorrect representation of statistical data in transaction UCONPHTL (function modules are shown as unassigned to any CA, although they are assigned to an application CA).

Reason and Prerequisites

This situation may occur, e.g., through a change in configuration from a local to a transportable scenario without resetting the UCON framework first. Other root causes can include the transport of the configuration from a different system, while UCON was already set up.

Solution

This SAP Note introduces the report RS_UCON_UPDATE_DEFAULT_CA that assigns all application CAs (/namespace/APP_name_of_the_ca) as child CA to the default CA currently configured in the system.

Symptom

A workspace in SAP Central Business Configuration is deployed to the Development Tenant instead of the intended Customizing Tenant and the requirement may include Tenant reprovisioning or decoupling of deployment target from Development workspace.

Environment

  • SAP Central Business Configuration

Cause

In a 3-system landscape, the Development System contains two tenants with distinct purposes:

  • Development Tenant
    • Used for developer extensibility (ABAP environment)
  • Customizing Tenant
    • Used and designated for business configuration and SAP Central Business Configuration deployments for the main implementation project.

Resolution

When a workspace in SAP Central Business Configuration is deployed to the Development Tenant instead of the intended Customizing Tenant, this does not inherently require reassigning the deployment target, from the workspace, or performing a tenant reprovisioning.

If the Customizing Tenant is already assigned to a separate implementation workspace intended for the main project and configuration activities are still ongoing, the situation can be addressed by proceeding with that workspace, as it represents the correct target for business process configuration.

The deployment executed in the Development Tenant does not impact or block the usage of the Customizing Tenant. Therefore, corrective actions such as target reassignment or tenant reset are typically not necessary in this context.

If no deployment has been executed in a given workspace, it remains possible to delete the workspace and restart the setup regardless of the assigned tenant. Additional details are available in KBA 3381588 - - How to delete the Workspace in SAP Central Business Configuration.

Furthermore, since the Development Tenant is designed for extensibility and development use cases, the existing configuration may still be evaluated for reuse depending on the project scope and does not inherently justify reprovisioning. If reprovisioning is still considered, refer to KBA 3401017 - FAQs about S/4HANA Public Cloud System Reprovisioning for further details.

Tenant reset or reprovisioning should be treated as high-impact actions and are generally only applicable in scenarios where the tenant content is no longer usable or where a clean system state is strictly required.

Symptom

  • The approval workflow submission pop-up does not display the approval user name. Instead, it only shows the role name.
  • Users cannot see who the approval request is being routed to, causing confusion and reducing transparency in the workflow approval process.
  • Behavior deviates from previous UI functionality where the manager name was displayed correctly.

Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.

Environment

  • SAP SuccessFactors HCM Suite
  • SAP SuccessFactors Employee Central
    • Workflow

Reproducing the Issue

  1. Navigate to "Personal Information".
  2. Edit any personal information field with the current date.
  3. Submit the change.
  4. Observe the approval workflow pop-up.
  5. Approval name is missing only the role name is shown.

Cause

The user does not have the "User Search" permission, which prevents the system from retrieving and displaying the Manager's name in the workflow pop-up.

Resolution

Grant the "User Search" permission and reproduce the steps again.

See Also

Note: The behavior does not occur in the Legacy People Profile. It only occurs in the Latest People Profile when the option "Enable the latest Editing and History UI for Employee Central HRIS entities on Full Profile" is enabled.

Access:

  1. Config Latest People Profile
  2. General settings

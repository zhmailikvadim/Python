Symptom

During loading to a DSO, serialization effects occur in the table RSDVENQUEUE.

As a result, parallel loading processes are executed one after the other and the load performance is significantly reduced.

Other Terms

DataStore Object (advanced), ADSO, NCUM, non-cumulative, non-cumulatives, validity table

Reason and Prerequisites

To update the validity table, a lock is set on the table RSDVENQUEUE. This lock is released again with the next commit. Since the lock occurs before the actual update of the DSO table, the lock is held for an extremely long time.

As a result of this change, the lock (and the update of the validity table) is executed only following writing to the inbound queue. This means that the lock is held for only a short period of time and parallel write processes are not serialized as heavily.

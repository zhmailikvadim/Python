Symptom

When you execute the report  SI: Transfer to SI Procedures (RP_PAYDE_B2A_SV_UNPACK), the system issues the error message

"An exception with the type CX_SXML_PARSE_ERROR occurred, but was not handled locally or declared in a RAISING clause.".

Other Terms

 eXTra 1.5, RP_PAYDE_B2A_SV_UNPACK, CX_SXML_PARSE_ERROR

Reason and Prerequisites

The message can occur for various reasons:

  • The affected file was read from the quarantine of the previous reports RPCSVPD0/RPCSVPD0_IN. When you do this, the system did not transfer the user data correctly.
  • The communication server incorrectly provided the data as a GZIP file instead of uncompressed.

Solution

 Implement the attached correction instructions or import the specified support package. After you import the HR Support Package or implement the correction instructions, you do not have to perform any further follow-on activities.

The incorrect user data is then corrected the next time the report  SI: Transfer to SI Procedures (RP_PAYDE_B2A_SV_UNPACK) is run.

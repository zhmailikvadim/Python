Symptom

When changing the position for an employee in Job Information, system will trigger onChange rule configured in position field to sync position data to Job Information. If the new field value is the same as previous one, the field is cleared.

This happens when "Configure Latest People Profile<General Settings<Enable the latest Editing and History UI for Employee Central HRIS entities on Full Profile" is enabled.

Environment

SAP SuccessFactors Employee Central

  • Job Information
  • Latest Editing UI

Reproducing the Issue

  1. Make sure "Enable the latest Editing and History UI for Employee Central HRIS entities on Full Profile" is enabled in Configure Latest People Profile<General Settings
  2. Go to Employee Profile
  3. Change Position in Job Information by clicking pencil icon
  4. Choose a position which business unit is the same as current value
  5. After executing onChange rule, the business unit field value is cleared.

Cause

This happens when

  1. "Configure Latest People Profile<General Settings<Enable the latest Editing and History UI for Employee Central HRIS entities on Full Profile" is enabled
  2. Associations are configured in Configure Object Definition
  3. Field Criteria are not maintained in Manage Business Configuration

Resolution

In order to solve the issue, please first run Check tool to find out the fields that are missing field criteria.

  • Check Tool<System Health<Application: Employee Central Core; Area: Association<Necessary field criteria exist.(MissingFieldCriteriaCheck) <Run  

Maintain Field Criteria settings according to the MissingFieldCriteriaCheck result.

For details about how to configure association and field criteria, please refer to KBA 2287729 - Employee Central - Object Association Field Criteria FAQ

Symptom

Report CNSETTLRULE is not displaying data correctly eg Receiver field 

Environment

  • SAP ERP Project System (PS)
  • SAP R/3
  • SAP R/3 Enterprise
  • SAP ERP Central Component
  • SAP ERP
  • SAP Enhancement package for SAP ERP
  • SAP Enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA, on-premise
  • SAP S/4HANA Cloud Private Edition

Reproducing the Issue

  1. Start transaction CNSETTLRULE.
  2. Enter the affected project number and display the results.
  3. Review a sender WBS at a higher level: the receiver appears as the same WBS instead of the intended subordinate WBS.
  4. Review another sender WBS: the expected receiver (for example, an external object) appears with an incorrectly formatted/parsed value.
  5. In CJ20N, open the same WBS elements and compare their settlement rules; they differ from the CNSETTLRULE output.

Cause

Database inconsistency: missing REC_OBJNR1 in table COBRB for the affected WBS settlement rule records.

Resolution

  1. Run report RKSRULE_REC_OBJNR1 to populate missing REC_OBJNR1 entries in table COBRB for settlement rule records.
  2. Run CNSETTLRULE again and verify that the receiver information now matches the settlement rules maintained in CJ20N.
  3. If needed, check table COBRB to confirm that field REC_OBJNR1 is now filled for the affected records.
    Or 
  4. Report  ZKOREC_OBJNR1f rom the note 640530.

See Also

  • refer to: SAP Note 138572 - RKSRULE_REC_OBJNR1: Completing missing receiver object numbers

Symptom

The modification posting logic from SAP Note 2041370 to perform the G/L balance valuation by multiple runs per key date without the requisite of a reset run.
After enabling the 'Allow Multiple Value Assignment' option, the valuation program proposal is recalculating the total balance for the non open item managed accounts instead of using only the amount associated with new documents posted after the last remeasurement/revaluation. 

Environment

  • Financial Accounting
  • SAP ERP Central Component
  • SAP ERP
  • SAP S/4HANA Finance
  • SAP S/4HANA
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA

Cause

According to SAP Note 2041370 , activating this logic does not provide a delta valuation logic similar to delta valuation logic for open item valuation. Instead it provides a kind of delta valuation effect for G/L balance accounts. Therefore when doing multiple valuation runs for the same key date, previous balance valuations will be reversed  and then the complete balance will be newly valuated. So the difference between these values represents the valuation of the newly posted documents (those which were not included within the first valuation run). 

Resolution

This is expected behaviour for this modified logic in the foreign currency valuation program. 

See Also

More details about the modification logic can be found in SAP Note 2042602 FAGL_FCV: Activate G/L Balance Multiple Valuation Option

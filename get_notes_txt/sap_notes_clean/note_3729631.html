Symptom

  • In result of optimized report VA05, it is expected to have a specific column sorted by default without using ALV SORT function explicitly or setting a layout with sort fields. 
  • It is not clear which is the default sorted fields in standard in VA05 result list.

*Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.

Environment

  • Sales And Distribution (SD)
  • SAP ERP Central Component
  • SAP ERP
  • SAP S/4HANA
  • SAP S/4HANA Cloud Private Edition

Reproducing the Issue

  1. Access transaction VA05.
  2. Input some filter values, and execute the report.

Resolution

  • By default, the report makes descending sort on field VBELN when fetching data from database table. If the selected fields include VBAP-POSNR, it will also be sorted ascending at the same time. This logic is hard-coded. 
  • If it is expected to have other specific fields get sorted by default before result list is returned to the end user so that end users don't need to use ALV SORT function manually, or don't need to update and save a ALV layout setting, custom codes have to be implemented somewhere. For example, method POST_PROCESSING_MASS of BADI BADI_SDOC_WRAPPER_MASS. Reach your local developers regarding the details about how and where to implement codes exactly. 
    *Notice: Custom codes belong to consulting topic and system modification which are out of SAP Support scope( see SAP Note 381348 and SAP Note 83020), customer's  local development team or consultants are responsible for this part. 

See Also

  • This SAP Note introduces BADIs that can be used to enhance optimized reports:
    1780163 - Report optimization: Access to further tables
  • Technically, the hard-codes to add SORT to SELECT statement is in CL_SDOC_SELECT_VAX5_HDB->CREATE_QUERY
  • ALV SORT function is to select a column, click SORT icons.
    ALV layout settings can start by clicking 'Change Layout' icon and then update fields in 'Sort Order' tab.

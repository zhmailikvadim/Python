Symptom

The automatic line with the Tax Account in the Manual Journal Entry is not immediately after the G/L Acct/ BP Code line that contains the Tax Group or Tax Code

    Environment

    SAP Business One

    Reproducing the Issue

    Prerequisites:

    Company localizations where Automatic Tax is available in Manual Journal Entry

    Scenario:

    1. Go to Financials → Journal Entry.
    2. In the header of the document ensure that Automatic Tax is checked.
    3. In the first line select G/L Acct/ BP Code, enter Debit or Credit value and select a Tax Group or Tax Code -depending on localization. Line one.
    4. Verify that the application adds a new line with the Tax Account associated to the Tax Group or Tax Code, with a value in Debit or Credit, immediately after. Line two.
    5. Add a new line with another G/L Acct/ BP Code and either leave the application calculate the balancing Debit or Credit value or enter any value. Line three.
    6. Go back to the line one and change the Tax Group or Tax Code.

    Expected behavior:

    • The line that contains the changed Tax Account remains in the same position but with different Tax Account and values. In the example, Line two.
    • The rest of the lines remain unchanged in value and position. In the example, line three.

    Actual behavior:

    • The line with the new Tax Account is displaced to the last line in the form. In this example line three.
    • The rest of the lines are moved one line upwards. In this example line three becomes line two.

    Cause

    The general mechanism of insertion in the form Manual Journal Entry:

    • Adds any new record line after the last record already present in the form.
    • When a line is deleted, the following lines move on position upwards

    The automatic line or lines created by the Automatic Tax functionality are no exception to this rule.

    Resolution

    This is the correct system behavior.

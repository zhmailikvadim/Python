Symptom

The Manage Commercial Location Groups app provides enhanced capabilities for creating and managing energy communities by grouping Commercial Locations. The application enables both operational and future‑dated planning of participants, ensuring seamless integration with downstream systems such as SAP S/4HANA.

Other Terms

Manage Commercial Location Groups, Energy Sharing, Consumer Participant, Producer Participants, Formula

Reason and Prerequisites

New Feature Development

Solution

Manage Commercial Location Groups Functional Capabilities - 

1. Creation and Classification of Commercial Location Groups

  • Users can create Commercial Location Groups and assign: 
    • Division
    • Define Mapping for Commercial Group Categories - (Config Table - /APEU/V_CG_CATEG )
    • IMG Path: Measurement Concept Management Adapter >> Basic MCM Configuration and Mapping >> Define Mapping for Commercial Group Categories.
  • The Define Commercial Location Types allows classification of the group (e.g., tenant electricity, shared building supply) ( Config Table - /APEU/V_CG_TYPE ).
    • IMG Path: Measurement Concept Management Adapter >> Basic MCM Configuration and Mapping >> Define Commercial Location Types.
  • In /APEU/V_LOCCAT, you define the various location categories, used in the SAP Cloud for Utilities Foundation MCM component for metering location and market location. Moreover, you assign the location categories to the location ID.
    • IMG Path: Measurement Concept Management Adapter >> Basic MCM Configuration and Mapping >> Define Location Category Codes.
    • An Aggregation Point of Delivery (PoD) represents a group of PoDs within the same Commercial Group. It serves as the central PoD to which the data of other PoDs in the group is associated.
  • Categories can be extended or customized using an API‑based configuration.

2. Participant Management

  • Participants can be added or removed with time-slice precision.
  • Participants can be maintained as: 
    • Consumers
    • Producers

3. Administration Data Entry

Users can maintain:

  • Validity period
  • Order code
  • Optional data: 
    • Installation operator
    • Energy community address

4. Validation and Submission Process

  • Process ID: APEU_B_MCM_CG_VALIDATION_PROCESS
  • The Submit action finalizes the current state of the Commercial Location Group.
  • The user can extend the customer implementation step to implement there own validations.
  • By default, validation success is passed. If the user wishes to overwrite via another validation they need to reset the validation success flag.
  • The user can implement their own country specific process through the country specific market process.
  • Submission triggers a validation event, which can be used in SAP S/4HANA to: 
    • Confirm participant eligibility,
    • Determine permissible switch‑in dates,
    • Return validations including potential error messages.

5. After update event successfully completed below action would be performed.

  • Process ID: APEU_B_MCM_CG_UPDATE_PROCESS
  • Update POD Additional Attributes - Usage context, Commercial group identifer, Aggregation POD name, Dormant Indicator.
  • Update formula into formula application table.
  • The user can extend the customer implementation step to implement there own master data updates.
  • The user can implement their own country specific process through the country specific market process.

6. Formula Explosion and System Integration

  • Upon successful validation:
    • A formula explosion ( Application Table - /APEU/INST_FORMU, VALID_FROM & VALID_TO added as new fields ) is triggered for all related Commercial Locations.
    • Updated formulas are then transferred to connected operational systems via an outbound event.
  • Planned future changes are also captured and reflected in the formulas for the relevant effective dates. ( Application Table - /APEU/INST_FORMU ).

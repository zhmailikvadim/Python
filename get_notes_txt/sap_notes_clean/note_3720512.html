Symptom

In the Administrator List for eAU Notifications (RP_PAYDE_EAU_LIST), there are many confirmations with the status To Be Checked for the same absence. The reason for this is that the program Create eAU Requests (RP_PAYDE_EAU_CREATE_NOTIFS) creates requests for an absence even though there are still confirmations with the status To Be Checked that refer to this absence.

Reason and Prerequisites

There is an eAU-relevant absence for which an eAU request was created. The confirmation received from the health insurance fund has a period that is completely after the absence. The confirmation has the status To Be Checked.

Example:

  • eAU-Relevant Absence (Infotype 2001): 01/22/2026 - 01/22/2026
  • eAU request with request key date January 22, 2026
  • Certificate received from health insurance fund: January 23, 2026 - January 24, 2026

Solution

Implement the attached correction instructions or import the specified support package.

The programming error is corrected so that in the constellation mentioned above, eAU requests are only created again in the future if the confirmation to be checked has been processed manually. Existing confirmations in the administrator list must be processed manually. The behavior described in SAP Note 3658866 (eAU: Notification creator - creation of queries despite confirmations with the status "To Be Checked")  is retained.

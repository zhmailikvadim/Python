Symptom

Report RPLDSNF0 is not working anymore after release PV2 of the DSN or after the implementation of note 2120783. An unexpected error interrupts the process of bloc S21.G00.30.

Solution

There are two possibilities:

  • Implement the attached correction instructions or the corresponding HR-SP or CLC.
  • As an alternative you can make some entries in following tables:
    • In transaction SE16N, enter two new lines in table T5FDSNPADEF with following values
      • Parameter 30_018_FIELD, text: Champ de l'infotype 0105 pour S21.G00.30.018
      • Parameter 30_018_SUBTY, text: Subtype de l'infotype 0105 pour S21.G00.30.018
    • In transaction SM30 enter two new lines for view V_T5FDSNPAVAL:
      • Parameter 30_018_FIELD should have value USRID_LONG
      • Parameter 30_018_SUBTY should have value FRMA

Symptom

  • When picking is performed using the Fiori app Process Picking Request (F4643), subsequent attempts to modify the related sales order are blocked.
  • The system raises error message "cannot change; store picking request in process or completed for order &1 - Message no. RFM_ST_PICK_ORD006" when trying to change the sales order after picking has started.

Environment

  • Product: SAP Enterprise Cloud Services
  • In-Store Merchandise and Inventory Management - Fiori Apps

Reproducing the Issue

  1. Start picking for a sales order using the Process Picking Request app
  2. Attempt to modify the sales order (for example, using BAPI_SALESORDER_CHANGE).
  3. Observe that the system prevents the change and issues the error RFM_ST_PICK_ORD 006.

Cause

Standard behavior

Resolution

Once the picking process has started, the system restricts changes to the sales order to avoid inconsistencies. 

For example:

  • A customer places an order for 5 items and a picking request is generated
  • A store associate starts processing the request
  • If during this picking process, the customer adds new items, it could lead to data mismatches.

Until picking begins, however, there is no restriction, the customer may freely adjust the sales order (add or remove items) before that point.

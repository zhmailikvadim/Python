Symptom

  • Wage types are not displayed in the Payroll Journal report (transaction PC00_M**_CLJN).

  • Results configured in transaction PE51 (HR Forms) are not shown in the totals section.

  • Wage types are visible at employee level, but missing in totals display.

  • Z-forms (copies of standard forms, e.g. MXJS) do not show information during report execution.

  • Wage Types added manually in the form window are not reflected in totals.

Cause

The issue is caused by missing configuration in the payroll journal excerpt form:

  • The identifiers (wage type elements) used in the totals display are not included in the cumulation identifier.

  • The totals display in Payroll Journal only processes identifiers that are:

    • Defined in the form, and

    • Included in the cumulation identifier configuration.

As per SAP standard documentation:

All identifiers specified for totals must also be included in the cumulation identifier.

If this linkage is missing, the system cannot aggregate and display the wage types in totals.

Resolution

Maintain the correct configuration in the Payroll Journal form:

Steps

  1. Go to IMG:

     
    Payroll → Payroll: "Your country" → Forms → Payroll Journal → Create payroll journal form
     

  2. Open your Z-form (e.g. copied from MXJS) in PE51.

  3. Navigate to:

    • Create display of totals

    • Excerpt form configuration

  4. Ensure that:

    • All identifiers used in totals are also included in the cumulation identifier
      (e.g. include identifier in ZXJD or relevant cumulation group).

  5. Save and transport the changes.

  6. Re-run report:

     
    PC00_MXX_CLJN
     

  7. Verify that wage types now appear correctly in totals.

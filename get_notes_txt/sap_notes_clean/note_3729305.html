Symptom

  • The asset quantity does not decrease to "0" after performing a full retirement using transaction ABAVN.
  • The retirement document does not display any quantity in the line item.
  • No error messages or warnings are displayed during the retirement process.

Environment

  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA Finance
  • SAP S/4HANA

Reproducing the Issue

  1. Transaction ABAVN.
  2. Input asset number, document date, posting date, and asset value date.
  3. Input posting period, document type, and transaction type in the additional details section.
  4. Leave the partial retirement fields (amount, percentage, quantity) empty to perform a full retirement.
  5. Simulate the document and click "Post".
  6. Open transaction AS03 to check the asset master record.
  7. Observe that the asset quantity remains unchanged and is not updated to "0".

Cause

This is standard system design:

  • For full retirements, the system updates the quantity to "0" only when no partial retirement information (amount, percentage, or quantity) is entered in transaction ABAVN.
  • For partial retirements, the system updates the quantity only if the quantity to be retired is explicitly specified by the user.
  • In this case, the retirement for the asset in question was processed as a partial retirement with an explicitly entered retirement amount, which does not trigger the system to update the quantity.

Resolution

  1. Ensure that no partial retirement information (amount, percentage, or quantity) is entered in transaction ABAVN when performing a full retirement.
  2. If the quantity update is required for a partial retirement, explicitly specify the quantity to be retired in transaction ABAVN.
  3. If the quantity remains incorrect due to prior postings, manually adjust the quantity in the asset master record using transaction AS02 (Change Asset Master Record).

Symptom

You transfer material master data between different systems using Product SOAP API.

The payload contains Advanced Planning-specific section '<PlantSCMAdvancedPlanning>' on plant level.

In specific situations, this section can cause mapping issues in the receiving system, resulting in error message 'The value '' is not in the value range of the XML schema type 'decimal' or it does not meet the specified limitations'.
The affected field name is 'MaxDaysOfSupplyPenaltyAmount'.

Other Terms

MARC_APS_EXT, ProductMDMBulkReplicateRequest_Out, ProductMDMBulkReplicateRequest_In, MDM_PRD_PRODUCT_MDMPLANT_SCMAD, MAX_DAYS_OF_SUPPLY_PENALTY_AMO, PlantSCMAdvancedPlanning, MaxDaysOfSupplyPenaltyAmount, MAXDOS_PEN

Reason and Prerequisites

The table 'MARC_APS_EXT' containing Advanced Planning related data on plant level is empty for a specific material. This can happen if 'Advanced Planning' is not active for the corresponding material in this specific plant.

Due to a program error, the SOAP service is transferring a blank value for field 'MAXDOS_PEN', although this value is invalid based on the field definition.

Solution

Implement the attached Correction Instruction or the mentioned Support Package.

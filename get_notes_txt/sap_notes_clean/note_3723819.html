Symptom

  1. In Poland KSeF 2.0 for the FA(3) schema, the eDocument version should be Version 2. However, it is currently being displayed as Version 1, as illustrated in the image below
  2. FA3 schema requires version 2 for integration using HTTP RFC connection, while older integration using logical port is deprecated. 

Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.

Environment

  • SAP ERP
  • SAP S/4HANA
  • SAP Document and Reporting Compliance Service

Reproducing the Issue

  1. Post an invoice and create a eDocument. 
  2. Navigate to the eDocument history for the posted invoice. 
  3. Verify that the eDocument version is displayed as version 1 instead of the expected version

Resolution

  1. Confirm the configuration maintained in the SM30 View EDOCOMPPROCCOMV for the company code and cloud service. 
  2. Ensure the view includes the eDocument process PLKSEF and PLKSEFIN.
  3. Refer the SAP Help Portal for comprehensive instructions on defining process communication through cloud services.
    • SAP ECC: Defining Process Communication Through Cloud Services (Mandatory) | SAP Help Portal 
    • SAP S4 HANA: Defining Process Communication Through Cloud Services (Mandatory) | SAP Help Portal 

     4. Retest the issue by posting a new invoice and verify whether the eDocument version 2 is created after updating the configuration.

Symptom

This KBA helps to answer additional questions about the security note 3581961.

Solution

Q1: Which customers are affected?

A. All the S/4HANA private cloud on all releases and S/4HANA on-premise on all releases are impacted.

Q2: Is it possible to get the fix for lower SP's?

A. SAP Security note is released for all S4CORE release and support packages available to customers so implement the SAP security note 3581961 in SAP S/4HANA private cloud / SAP S/4HANA on-premise systems.

Q3: Will the upgrade require down time?

A. No.

Q4: Are any other components, other than the component mentioned in the security note, impacted by applying the fix?

No. The software containing the vulnerable program is inactive in SAP S/4HANA and the correction delivered through this security note is within source code only.

Q5: Is there any workaround to fix this vulnerability temporarily?

A. No - The vulnerability is within the source code of an inactive solution and there is no relationship with any other solutions within SAP S/4HANA. There is no workaround or temporary fix for the vulnerability.

Q6: How to verify if the vulnerability is mitigated after applying the patch? How to test after applying the SAP security note?

As vulnerability is within the inactive solution in SAP S/4HANA, you can do the functional validation as such, but however you can just execute transaction SE37 and cross verify to confirm that source code of RFM ‘/SLOAP/GEN_MODULE_REPORT’ is commented.

Q7: Will the upgrade require for S/4HANA systems on lower releases?

A. No. Security note is applicable for all S/4HANA privae cloud / on-premise systems starting from release 102.

Q8: SAP security note 3581961 id applicable for S/4HANA on-premise systems?.

A. Yes

Q9: Does S4CORE need to be on latest release to implement SAP security note 3581961?.

A. No. SAP Security note 3581961 is applicable for all lower S4CORE releases and support packages.

Symptom

Queries about XML tags P_19 and P_19A where The P_19 tag is correctly populated for FI documents but not for SD documents, even though the same tax code is maintained in table EDO_PL_MAPTAXV2V and the P_19N flag is set to true.

 

Environment

  • SAP ERP
  • SAP S/4HANA
  • SAP Document and Reporting Compliance, Cloud Edition
  • SAP S/4HANA cloud, private edition

Reproducing the Issue

  1. Launch Tx code edoc_cockpit.
  2. Select an FI invoice and display the generated XML; observe that tags P_19 and P_19A are populated.
  3. Select an SD invoice with the same tax determination and display the generated XML; observe that tag P_19N may be present and P_19/P_19A may be missing.

Cause

Missing correction Note

Resolution

Implement SAP note 3702483 - Electronic Document Processing for Poland - KSeF: FA(3) Schema Version Code Corrections III which solves the issue

Note : Before implementing the suggested note make sure to follow the sequence mentioned in Overview Guide - 3279206 - Electronic Document Processing for Poland - KSeF: Implementation Overview

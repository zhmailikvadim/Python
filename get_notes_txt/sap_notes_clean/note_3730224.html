Symptom

A contact person can be set to obsolete even if the same contact is still designated as the main contact for an account.

Saving the contact after setting it to obsolete succeeds without any warning or error.

Environment

SAP Cloud for Customer

Reproducing the Issue

  1. Open an account.
  2. Go to the Contacts tab.
  3. Select a contact and set it as the main contact (if not already set).
  4. Open that contact’s record.
  5. Set the contact status to obsolete.
  6. Save. The save completes without any warning, and the contact remains the main contact of the account.

Cause

This is standard system behavior; there is no built-in validation preventing a main contact from being set to obsolete.

Resolution

If a validation or warning is required, submit an enhancement request via the Customer Influence site: Customer Influence for SAP Cloud for Customer.

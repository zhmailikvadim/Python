Symptom

  • Create a Live Office report based on an universe query with Date object as query filter
  • No date returns in Live office when set operator to Between and set operand of start date as Prompt

Other Terms

LO, unexpect, wrong, incorrect, bi4, bo4, not work

Reason and Prerequisites

Environment:

  • SAP BusinessObjects Business Intelligence platform 4.2 Support Package 03 Patch 5 (BI4.2 Patch 3.5)
  • SAP BusinessObjects Live Office 4.2 Support Package 03 Patch 5

Steps to Reproduce:

  1. Logon to Live Office
  2. Click Universe Query to insert a query using universe query
  3. Select a universe with Date type object
  4. Add an date type object to Filter Objects panel
  5. Set operator to Between
  6. Select  Prompt as Operand for start date
  7. Select Constant,  List of Values or Prompt as Operand for end date
  8. Run query
    1. >> No data are returned

Cause:

  • The order of operands is lost when it is sent by the Web Service.
  • The issue is, Const, list of values and prompts are already sent in the same order : Const and List of values first and then prompt, whatever the position which has been defined by the user in the user interface. It means if user define in Live Office a query with a between A and B condition. If A is a prompt and B is a const or a list of value, the sql which is sent to the database will be something like "between B and A" rather than "between A and B". It's a problem here because if A < B, the query returns an empty result.

Solution

A fix will be delivered for Live Office (client side) to keep the order defined by the user, end to end, from the Live offie application to the database request.

This issue is fixed in the patches listed in the "Support Packages & Patches" section below.
The "Support Packages & Patches" section will be populated with the relevant patch levels once they are released.

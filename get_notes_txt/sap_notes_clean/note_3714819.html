Symptom

  • Integration team encounters an error regarding ciphersuites when setting up a JMS connection to IBM MQ (Websphere) with SSL/TLS.
  • Non-SSL connection works fine, but SSL/TLS connection fails.
  • Testing with openssl using cipher AES256-GCM-SHA384 yields positive responses from both SAP PO and IBM MQ.
  • After deploying the required IBM MQ JMS libraries (com.ibm.mq.allclient.jar and org.json.jar), connection to IBM MQ fails entirely, including non-SSL connections.

Environment

  • SAP NetWeaver 7.5
  • SAP Process Integration
  • SAP Process Orchestration

Reproducing the Issue

  1. Attempt to set up a JMS connection to IBM MQ (Websphere) with SSL/TLS.
  2. Deploy the required IBM MQ JMS libraries: com.ibm.mq.allclient.jar and org.json.jar.
  3. Test the connection using openssl with cipher AES256-GCM-SHA384.
  4. Observe that SSL/TLS connection fails with an error regarding ciphersuites.
  5. Observe that non-SSL connection also fails after deploying the libraries.

Resolution

  1. Verify SSL Cipher Suite Configuration:

    • Ensure that the SSL Cipher Suite parameter is set correctly. For IBM MQ, the recommended value is: 
      TLS_RSA_WITH_AES_128_CBC_SHA256
    • This is equivalent to SSL_RSA_WITH_AES_128_CBC_SHA256.
  2. Create and Configure SSL Trust Store:

    • Use the keytool utility to create an SSL Trust Store.
    • Import the server certificate into the Trust Store.
    • Specify the Trust Store location and password in the Additional Java Launcher Options: 
      -Djavax.net.ssl.trustStore=<path_to_truststore>
-Djavax.net.ssl.trustStorePassword=<truststore_password>
  3. Configure Keystore for Two-Way SSL (if applicable):

    • If two-way SSL is required, also specify the JMS Adapter's keystore location and password: 
      -Djavax.net.ssl.keyStore=<path_to_keystore>
-Djavax.net.ssl.keyStorePassword=<keystore_password>
  4. Update JCE Unlimited Strength Jurisdiction Policy Files:

    • Download and install the JCE Unlimited Strength Jurisdiction Policy Files.
    • Place them in the appropriate directory:
      • For Windows: %LINK_DIR%\ext\jre\lib\security
      • For Linux/Unix: $LINK_DIR/ext/jre/lib/security
  5. Deploy JMS Client Library:

    • Ensure that the JMS client library is deployed in the PI/PO system as per SAP Note 1138877.
  6. Configure Additional Parameters in Channel:

    • In the channel configuration user interface, navigate to the Advanced Tab and add the following parameters: 
      mq.ssl = true
ssl.cipher.suite = <your_cipher_suite>
  7. Check Compatibility:

    • Ensure that your environment is compatible with TLS 1.2, which requires JVM 8. This means that your PI/PO system must be on NetWeaver 7.50 or higher.

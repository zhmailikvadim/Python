Symptom

The project is in started status however user can release project invoice request and a customer invoice. The expectation is that a project invoice request and a customer invoice should only be created when the project status is "Released".

Environment

SAP Business ByDesign

Reproducing the Issue

  1. Go to the Project Management work center
  2. Go to the Project Invoice Requests view
  3. Select All project Invoice requests
  4. Search based on relevant Project 
  5. The project invoice requests were created for a Started project

Cause

Resolution

There is a difference between the statuses Started and Released.

When Project Status is Started, the Possible Actions are Release & Stop (Unstop). 

This status is only applicable for a project header. Once started, the project header can no longer be deleted. It also becomes an account assignment object and all project tasks can be released individually. The Started status is not hierarchical, which means that it does not affect the underlying structure. Also, it cannot be reverted.

When Project status is Released and possible actions are ; 

Complete (Revoke Completion)
Complete and Close (Undo Closing)
Stop (Unstop)

Project header or project tasks with this status become accounting objects. The status Released is hierarchical, meaning that all subsequent tasks are affected and automatically converted into them. For example, if a project task is set to Released, all its subtasks are automatically set to Released as well. Please note that Released and Started are the only two statuses that cannot be revoked.

So the integration between the project header and other business areas is enabled when you change the project status to Started. This means that invoicing is possible when the project status is Started.

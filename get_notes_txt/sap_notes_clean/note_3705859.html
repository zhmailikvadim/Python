Symptom

  • The issue pertains to the configuration of ONB 2.0 for assigning the responsible group for the New Hire Data Validation step using the rule ONB2_NewHireDataReview.
  • In ONB 1.0, the validation step is assigned to the Generalist role via the field vpOfStaffingName in Recruiting.
  • The assignment behavior from ONB 1.0 cannot be replicated in ONB 2.0.
  • Attempts to use standard assignment options like Roles, Users, and Dynamic Groups have not been successful.
  • Confirmation is sought on whether the Generalist role or the role from the HR Manager relationship can be mapped to the validation step in ONB 2.0.

Environment

SAP SF ONBOARDING

Reproducing the Issue

  1. Configure ONB 2.0 for assigning the responsible group for the New Hire Data Validation step using the rule ONB2_NewHireDataReview.
  2. Attempt to replicate the assignment behavior from ONB 1.0, where the validation step is assigned to the Generalist role via the field vpOfStaffingName in Recruiting.
  3. Use standard assignment options available in ONB 2.0, such as Roles, Users, and Dynamic Groups, to assign the responsible group.
  4. Observe that the desired assignment behavior is not achieved.

Cause

The transition from Onboarding 1.0 to Onboarding 2.0 removed the functionality to assign activities directly from the Super Admin side. Onboarding 2.0 operates on a framework of rule-based configurations and role-based permissions, which does not natively support the replication of the assignment behavior from ONB 1.0.

Resolution

  1. Review the guide provided in the link below for detailed instructions on how to assign New Hire steps to the appropriate users effectively:

a href="https://help.sap.com/docs/successfactors-onboarding/implementing-onboarding/assigning-new-hire-data-review-task-to-responsible-group" Assigning New Hire Data Review Task to Responsible Group /a  

  1. Explore alternative configurations within the ONB 2.0 framework to achieve the desired assignment behavior.
  2. If the required functionality is not supported natively, consider submitting a feature request or enhancement request to SAP for future updates.

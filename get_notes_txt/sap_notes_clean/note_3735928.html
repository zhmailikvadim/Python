Symptom

When reversing a calculation run document, the system throws an error during billing document creation. This issue occurs due to incorrect field mapping to the "order type," which leads to invalid document type determination during reversal processing. As a result, the billing document cannot be created successfully.
This note corrects the behavior by ensuring that the document type mapping is handled correctly so that reversal of calculation run documents proceeds without errors.

Other Terms

/IRM/IPTTM, calcrun reversal error, billing document creation issue, document type mapping, order type mapping, reversal failure, calculation run document

Reason and Prerequisites

This is caused by program error

Solution

Please implement the correction instructions attached. These corrections are only valid if you have applied the following support packs.

Release V4 - 2021, 2022, 2022 V2, 2023, 2023 V2, 2023 V3 and 2023 V3 Support Pack 1

Use transaction SNOTE to implement the correction instructions specified below and activate the affected program object.

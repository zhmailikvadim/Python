Symptom

  • In SAP Solution Manager Interface & Connection Monitoring, the E2E Alert Analysis Report attached to the alert email notification only shows the count of qRFC entries. The report does not display the detailed qRFC queue names.
  • It is expected to see the specific qRFC queue names included in the report, instead of only the numerical count.

Environment

  • SAP Solution Manager 7.2
  • Interface and Connection Monitoring
  • qRFC Monitoring
  • E2E Alert Analysis Report

Reproducing the Issue

  1. Log in to SAP Solution Manager 7.2.
  2. Navigate to Interface and Connection Monitoring in Technical Monitoring.
  3. Configure qRFC monitoring for the managed system.
  4. Ensure that alerts are generated for qRFC queues.
  5. Trigger or wait for an alert notification to be sent via email.
  6. Open the email notification and download the E2E Alert Analysis Report attachment.
  7. Check the qRFC monitoring details in the report, it displays only the number of qRFC entries, but the individual qRFC queue names are not shown.

Cause

This behavior occurs due to the standard design of SAP Solution Manager.

The E2E Alert Analysis Report only provides aggregated information, such as the count of qRFC entries, and does not include detailed qRFC queue names.

Resolution

The E2E Alert Analysis Report does not support displaying qRFC queue names in the report attachment.

Only the total number of qRFC entries is provided in the report as per standard SAP Solution Manager functionality.

If detailed qRFC queue information is required, it must be checked directly in the monitoring tools, such as:

  • qRFC monitoring transactions in the managed system
  • Interface and Connection Monitoring UI in Solution Manager

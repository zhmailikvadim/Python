Symptom

Static content of SAPUI5 runtime and Fiori apps are cached and can be displayed without authentication. For example, the result of the following requests can be directly displayed without authentication:

https://<host>:<port>/sap/bc/ui5_ui5/ui2/ushell/resources/~<cache token>~/sap-ui-version.json
https://<host>:<port>/sap/bc/ui5_ui5/ui2/ushell/resources/~<cache token>~/sap/ushell/renderers/fiori2/search/searchComponent/manifest.json
https://<host>:<port>/sap/bc/ui5_ui5/sap/<app name>/~<cache token>~5/manifest.json 

Environment

SAP S/4HANA

Reproducing the Issue

  1. Open a web browser
  2. Enter the following URL: https://<host>:<port>/sap/bc/ui5_ui5/ui2/ushell/resources/~<cache token>~/sap-ui-version.json
  3. Observe that the version information is displayed without requiring user login

Cause

Resolution

The static content of SAPUI5 runtime and Fiori apps is stored in the ABAP ICM cache for better performance, which does not require authentication. 

This behavior is not considered a security issue as only static JavaScript content is stored in the cache.

The cached content does not contain confidential business data and does not allow manipulation of data.

Symptom

When trying to release a project item linked to an existing project, the status changes from "Released" to "In Process" automatically and displays the following errors.

  • An orchestration error occurred.
  • Project &1 cannot be linked as it is not in planning stage.

Environment

SAP S/4HANA Cloud Public Edition

Cause

Checked the stage of the linked project using the "Plan Customer Projects" app (F0719). When linking an existing project ID with a project item in a solution order, only projects in the "In Planning" stage can be selected. If a project in a stage other than "In Planning" is linked, the system triggers a validation error during forward orchestration, causing the status to revert to "In Process".

Resolution

  • Ensure that the project linked to the solution order item is in the "In Planning" stage.
  • Avoid linking projects that are not in the "In Planning" stage to the solution order item.
  • Verify the project stage before linking it to the solution order item.
  • If the project stage is changed to "In Execution" after linking, ensure that the item is released from the solution order before creating a billing item.

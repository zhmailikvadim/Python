Symptom

 In the process "Start of Supply for Gas (SUP View)" (DE_B_GELIGAS_SOS_SUP), the system does not set the check result to "Confirm" in method CHECK_CONFIRM_REJ of class /US4G/CL_CHK_GPKE_PROCESS when the info event code is missing in the process step data. This issue is relevant for the market role: SUP.

Other Terms

/US4G/CL_CHK_GPKE_PROCESS, CHECK_CONFIRM_REJ, DE_B_GELIGAS_SOS_SUP, GPKE_SUP, Start of Supply for Gas, process step, info event code, check result, SUP

Reason and Prerequisites

The error occurred because the code checked if sy-subrc = 0 instead of checking if <ls_info_event> is assigned. As a result, when there is no info event code, the check result is not set to "Confirm" as expected.

Solution

Implement the corrections of this SAP Note or import the relevant support package.
After implementing this SAP Note, the system sets the check result to "Confirm" in method CHECK_CONFIRM_REJ of class /US4G/CL_CHK_GPKE_PROCESS when there is no info event code in the process step data. You can verify the fix by triggering the "Start of Supply for Gas (SUP View)" process (DE_B_GELIGAS_SOS_SUP) and confirming that the check result is set to "Confirm" when the info event code is missing.

Symptom

  • When attempting to implement a treeview hierarchy in a RAP composition tree (three-node business object), it results in syntax errors on an older ABAP release.
    Error message: '"link action" is not supported in this ABAP release.'

  • There is an uncertainty whether combining a RAP composition tree and a treeview is possible.

Environment

SAP BTP ABAP Environment

Reproducing the Issue

  1. Define a RAP business object with three composition levels (root -> child -> grandchild) and expose it via OData V4.
  2. Extend the grandchild entity to support a treeview by adding parent/child associations and behavior including link/unlink actions and instance hierarchy.
  3. Activate the behavior definition (e.g. on an ABAP 7.93 system in ADT).
  4. Observe syntax errors, including '"link action" is not supported in this ABAP release.' and missing support for instance hierarchy syntax.

Cause

The RAP treeview feature and related behavior syntax (link/unlink actions and instance hierarchy) are not available in ABAP 7.93. Full support is available starting with ABAP 9.16.

Resolution

  • Verify the ABAP platform release of the target system.
    1. If the system is on ABAP 7.93 or an older release:
      • Acknowledge that RAP treeview and link/unlink syntax are not supported. Implementing the treeview is not possible on this release.
      • Use an alternative approach (e.g. a non-treeview UI or a classic application) until the system is upgraded.
    2. If the system is on ABAP 9.16 or higher release:
      • Implement the RAP treeview using recursive hierarchy annotations and instance hierarchy/behavior as described in the official guides.

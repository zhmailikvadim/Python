Symptom

  • When sharing data between a 16bit DOS application (i.e. Clipper) and a 32bit application (i.e. Delphi), Advantage Error 7008 is returned when opening a table
  • Error 7008: Advantage Server file open failure

Environment

  • Sybase Advantage Database Server 9.x
  • Sybase Advantage Database Server 10.x
  • SAP Advantage Database Server 11.x

Cause

Since Clipper forces the path to the 8.3 DOS naming convention and 32 bit applications use long filenames, the server may believe that two different files are being opened.

Resolution

Place the data in a directory with the 8.3 DOS naming convention.

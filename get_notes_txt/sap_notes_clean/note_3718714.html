Symptom

I need to expand an existing Segmented Pick List custom field (e.g., from 3 segments to 4) by merging it with another existing custom field. I must ensure that downstream integrations, such as the Worker Security Single Line Download, continue to function correctly during and after the transition.

Cause

Directly adding a new segment to an existing Segmented Pick List custom field is generally not supported if the field already contains data. To maintain data integrity and ensure integration stability, a phased transition approach is required where a new field is created and data is migrated from the old configuration.

Resolution

Follow these steps to transition from your old custom field setup to a new, merged Segmented Pick List:

  1. Create the New Field: Define a new Segmented Pick List custom field with the desired number of segments (e.g., 4 segments).
  2. Maintain Both Fields: Keep both the "old" and "new" fields active temporarily. This allows you to map and upload the new data while the old data is still available for reference.
  3. Verify Integration Mapping:
    1. If your integration only downloads a specific segment (e.g., the Job Code segment), verify that the new field is mapped to the same integration output.
    2. As long as the specific segment name remains identical in the new field, changing the surrounding segments will not affect the integration download.
  4. Data Migration:
    1. Use the Download Data tool to extract existing worker records with the old field values.
    2. Use the Upload Data tool to populate the new merged field for all active records.
  5. Decommission the Old Field: Once the new field is fully populated and verified in your downstream systems, you can safely remove the old custom fields.

Column Transformation Guide (Quick Setup)

If your file uses old segment headers, use Upload Column Transformation to map them to the new field automatically:

  1. Navigate to Connector Catalog > Open your Worker upload connector > Click Edit.
  2. Go to Upload Column Transform > + Add Native Fields.
  3. Add entries to map your file headers to the new system segments:
    • Standard Column Name: [New 4-Segment Field Name]
    • File Column Name: [Old 3-Segment Field Name]
  4. Click Update and save.

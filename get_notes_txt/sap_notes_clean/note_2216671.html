Symptom

When executing the CREATE INDEX statement, the following error is occurred.

Example: Create an index TEST~Z1 on columns "MANDT", "BNAME" and "BCODE".

Environment

  • SAP release independent 
  • Oracle release independent

Reproducing the Issue

This error can be reproduced in several circumstances, example:

  • System copy like in import phase
  • System upgrade like in upgrade phase PARCONV_UPG,  ACT_UPG, etc
  • Create a new index via SE11 or SE14 or sqlplus

Cause

This error is caused by a CREATE INDEX statement specified a column that is already indexed. A single column may be indexed only once. Additional indexes may be created on the column if it is used as a portion of a concatenated index, that is, if the index consists of multiple columns.

Resolution

1. Please check the existing indexes belong to the effected table. 

select owner, index_name from dba_indexes where table_name='<table_name>';

2. Then, please find out the column on which these indexes are created.

select index_owner, index_name, column_name from dba_ind_columns where index_name ='<index_name>';

3. If the index with the same column name exists, please drop the related index and proceed.

drop index sap<sid>."<index_name>";

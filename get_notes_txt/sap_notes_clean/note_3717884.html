Symptom

During the Agreement search, the WORKLIST_FILL Badi method is not being triggered due to an empty filter value. The Agreement Type is not being passed correctly to the Badi call, resulting in the method not executing.

This SAP Note corrects the filter value handling and ensures that the Agreement Type is passed properly, thereby triggering the WORKLIST_FILL Badi method during Agreement search.

Other Terms

Worklist Fill, Filter Value, Agreement Search, /IRM/IPAGRM

Reason and Prerequisites

This is caused by program error

For V4, SAP Note #3717847 is available

Solution

Please implement the correction instructions attached. These corrections are only valid if you have applied the following support packs.

Release 6.4 Base Package and Support Pack 1

Use transaction SNOTE to implement the correction instructions specified below and activate the affected program object.

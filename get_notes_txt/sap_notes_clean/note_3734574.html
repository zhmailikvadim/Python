Symptom

Updating the Access Control setting in the Overview tab from No restrictions – Accessible to all users to Private to Team Members saves successfully and the template can be published. After logging out and logging back in, the Access Control setting automatically reverts to No restrictions – Accessible to all users.

Reproducing the Issue

  1. Open an SM process project template.
  2. In the Overview tab, change Access Control from No restrictions – Accessible to all users to Private to Team Members.
  3. Save and publish the template.
  4. Log out of the application.
  5. Log back in and open the same template.
  6. Observe that Access Control has reverted to No restrictions – Accessible to all users.

Cause

Access control settings on process project templates are not supported. In SM process projects, creation permissions are governed by user group membership and project roles, so the template-level Access Control setting does not persist.

Resolution

Don't set access control or apply conditions to process project templates themselves. In process projects, the ability to create a project from the template is controlled by a combination of membership in the SM Modular Questionnaire Manager global user group and a project group in the template that has the Process Initiator role. Access control or condition settings on the template itself aren't supported.

Symptom

We're submitting a requisition (PR) when our enterprise resource planning (ERP) external system that checks the budgets is down, which should send back ERP Push Errors to SAP Ariba Procurement on the real time budget check (RTBC) integration, however, the PR still goes to Submitted status when it should stay in Composing status.

Reproducing the Issue

  1. Create a PR with at least 1 line item.
  2. Submit the PR.
  3. The PR is now in Submitted status.

Cause

The ERP Push Error sent upon PR submission is missing the PR number in the <Id> tag of the cXML response for the RTBC ERP Push Error.

Example:

<?xml version="1.0" encoding="UTF-8"?>
<soap:Envelope xmlns:soap="http://schemas.xmlsoap.org/soap/envelope/">
    <soap:Header/>
    <soap:Body>
        <RequisitionRealTimeBudgetExportReply xmlns="urn:Ariba:Buyer:vrealm_12345">
            <RequisitionError_RequisitionErrorImport_Item>
                <item>
                    <Date>1234-56-78T90:12:34.567+89:01</Date>
                    <ErrorCode>ABC_Error_23</ErrorCode>
                    <ErrorColumn xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance" xsi:nil="true"/>
                    <ErrorDistributionNumber xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance" xsi:nil="true"/>
                    <ErrorLineNumber>1</ErrorLineNumber>
                    <ns2:ErrorMessage xmlns:ns2="urn:Ariba:Buyer:vrealm_12345">External system is down. Please raise a ticket</ns2:ErrorMessage>
                    <ErrorTable xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance" xsi:nil="true"/>
                    <ErrorValue xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance" xsi:nil="true"/>
                    <Id>Server Error</Id>
                </item>
            </RequisitionError_RequisitionErrorImport_Item>
        </RequisitionRealTimeBudgetExportReply>
    </soap:Body>
</soap:Envelope>

Resolution

Ensure the Id field is populated with SAP Ariba Procurement P2O/I's PR number. For example, the following is the corrected cXML from the Cause:

<?xml version="1.0" encoding="UTF-8"?>
<soap:Envelope xmlns:soap="http://schemas.xmlsoap.org/soap/envelope/">
    <soap:Header/>
    <soap:Body>
        <RequisitionRealTimeBudgetExportReply xmlns="urn:Ariba:Buyer:vrealm_12345">
            <RequisitionError_RequisitionErrorImport_Item>
                <item>
                    <Date>1234-56-78T90:12:34.567+89:01</Date>
                    <ErrorCode>ABC_Error_23</ErrorCode>
                    <ErrorColumn xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance" xsi:nil="true"/>
                    <ErrorDistributionNumber xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance" xsi:nil="true"/>
                    <ErrorLineNumber>1</ErrorLineNumber>
                    <ns2:ErrorMessage xmlns:ns2="urn:Ariba:Buyer:vrealm_12345">External system is down. Please raise a ticket</ns2:ErrorMessage>
                    <ErrorTable xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance" xsi:nil="true"/>
                    <ErrorValue xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance" xsi:nil="true"/>
                    <Id>PR1234</Id>
                </item>
            </RequisitionError_RequisitionErrorImport_Item>
        </RequisitionRealTimeBudgetExportReply>
    </soap:Body>
</soap:Envelope>

And as you can see, while there's ns2 in the ErrorMessage cXML tag which seems out of place, however, correcting the Id tag makes this cXML work as a response.

Symptom

The following errors occur when processing infotype 0267 via the renewal screen:

  • When creating a new record, the starting date is suggested based on the payroll control record. For this  the payroll area (ABKRS) is taken from the first 0001 infotype record, instead of the 0001 record corresponding to the sy-datum.
  • The values ​​of Polish specific fields (such as Insurance title code and insurance fields) are lost after saving the record.

Other Terms

Renewal IT0267, CL_HRPA_INFOTYPE_0267_PL, CL_HRPA_UI_CONVERT_0267_PL, ZUORD.

Reason and Prerequisites

Program error.

Solution

The date handling error has been corrected in the CL_HRPA_INFOTYPE_0267_PL class.

The following two table settings resolve the field values ​​disappearing error:

  • 1 record has been deleted from maintenance view V_T588MFPROPS:
INFTY ITVERS SUBTY SNAME FNAME
0267 46   P0267 ZUORD
  • 1 record has been added to maintenance view V_T588MUIFPROPS
INFTY CONTEXT VERSIONID SUBTY STRUCTURE_NAME FIELD_NAME FIELD_ATTRIBUTE
0267   46   HCMT_BSP_PA_PL_R0267 ZUORD READONLY

Please apply the correction by importing the Support Package specified for your release, or by using the Note Assistant. If you implement the change with SNOTE, maintain the two tables manually.

Symptom

The indexserver crashes with a crashstack like this one:

[CRASH_STACK] Stacktrace of crash: (0000-00-00 00:00:00 000 Local)
----> Symbolic stack backtrace <----
0: ptime::ViewQueryRewritter::case_join_set_and_check
1: ptime::ViewQueryRewritter::table_case_joined
2: ptime::ViewQueryRewritter::from_block
3: ptime::QueryExecutor::select_list_from_view_query
4: ptime::QueryExecutor::rewrite_view_subquery
[...]

Other Terms

signal 11, SIGSEGV, CREATE VIEW, CTE, Common Table Expression, CASE JOIN

Reason and Prerequisites

Issue number 347612

Reason:

Due to a programming error in SAP HANA, a table-case-joined table within a WITH clause in a statement is not handled correctly, leading the indexserver to crash.

Affected Releases:

  • SAP HANA 2:
    • Revisions of SPS00 - SPS06
    • Revisions <= 079.06 (SPS07)
    • Revisions <= 087.00 (SPS08)

Prerequisites:

  • CREATE VIEW statement using "SELECT *" to include all columns, is executed on top of a Common Table Expression (CTE) with CASE JOIN clause, e.g.

    CREATE VIEW V_CTE AS
    WITH CTE AS (SELECT A.I, V, DD.II, DD.VV FROM MYTAB1 AS A
    LEFT OUTER MANY TO ONE CASE JOIN
    WHEN A.V = 'abc' THEN RETURN (I AS II, V AS VV) FROM TAB2 M ON A.V = M.V
    END DD) SELECT * FROM CTE WITH DDL ONLY;

Solution

Apply one of the following SAP HANA Revisions:

  • SAP HANA 2:
    • Revisions >= 079.07 (SPS07)
    • Revisions >= 088.00 (SPS08)
    • or higher

  • or higher

Workaround:

  • Use explicit column name/s instead of asterisk, e.g.

    CREATE VIEW V_CTE AS
    WITH CTE AS (SELECT A.I, V, DD.II, DD.VV FROM MYTAB1 AS A
    LEFT OUTER MANY TO ONE CASE JOIN
    WHEN A.V = 'abc' THEN RETURN (I AS II, V AS VV) FROM MYTAB2 M ON A.V = M.V
    END DD) SELECT CTE.I, CTE.V, CTE.II, CTE.VV FROM CTE WITH DDL ONLY;

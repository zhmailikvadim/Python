Symptom

Requests to the theming service return HTTP 429: too many requests.

Other Terms

UI Theme Designer, theming serivce; theming-runtime; theming-repository; themeroot; sap-themes; themelist; Cloud Foudry; 429; too many requests

Reason and Prerequisites

Rate limiting restricts the number of requests that tenants can make within a specific time window. This helps maintain service stability. A tenant-specific limit is enforced. This limit is identical for all tenants and cannot be increased. You can't adjust it by changing plans. Please note that for each end user, theme files are typically loaded once and cached by the browser. This reduces the actual load on the server. For more information, see Rate Limitation.

Solution

The theming service is scaled for large customers, but traffic spikes should be avoided. If you can predict a peak, batch and pace the rollout — for example, when you enable a new end-user app with custom theming and notify users, send emails in batches to subsets. You can send these batches hourly or daily. Avoid conducting load tests unless you coordinate and receive approval in advance. If you have questions about planned usage peaks or find that the production limit is insufficient, submit a ticket under the CA-UI2-THD-CF component.

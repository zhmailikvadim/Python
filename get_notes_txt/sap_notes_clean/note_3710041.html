Symptom

When running report /APEU/RP_MCM_MIGRATION_POST for multiple MCM instances, migration runtime increases significantly due to huge volume data in related actor internal table.

For large data volumes, testing and productive migration may not be feasible.

Other Terms

APEU, /APEU/RP_MCM_MIGRATION_POST, Actor Data, UCOM

Reason and Prerequisites

APEU, 2508

The internal table used to collect actor data is not cleared per instance, leading to growing data volume and increased runtime.

Solution

Please implement the attached the correction instruction.

The lt_ucom_actors internal table is cleared after processing actor data for each MCM instance.

This prevents data accumulation across instances and restores stable migration runtime.

Symptom

Other Terms

RMI_P4, Multiple IP

Reason and Prerequisites

When AS Java tries to connect a stub object through P4 based on the object's connection profiles, it iterates through all available profiles and tries with each one. Some connection profiles cannot be used for P4 connections because they include IP addresses which are not visible or available. In this case, the P4 call waits until a connection timeout refuses it and then AS Java proceeds with the next profile. Sometimes, there is an already existing P4 connection, but AS Java does not recognize it until iteration reaches the corresponding profile. In order to optimize the process of opening a P4 connection, we introduce a patch which checks if there's an already established connection to any of the available connection profiles and reuses it.

This patch does not enhance the process of opening an initial connection. It speeds up the process of connection reuse.

Solution

  • Update your AS Java to a release where this optimization is introduced. See the SP Patch Level section for available downloads.

Solution

Please refer to the attachment IT0442.pdf in the note 2850097, the details of where the new field appears have been shown there.

The validations on the new field are as follows.

1. This field will be hidden unless the begin date for the IT0442 - subtype COMP is greater than or equal to 06.04.2020.

2. The field "Zero EM" is display only when the value of CO2 is greater than 50.

3. The field "Zero EM" is editable if the value of CO2 is less than or equal to 50.

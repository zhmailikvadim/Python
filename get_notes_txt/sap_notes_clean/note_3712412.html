Symptom

Error while confirming the Physical Inventory Count Task via Microsoft excel sheet.

 "Error: item invalid; product/identified stock/logistics unit mismatch"

Environment

SAP Business ByDesign

Reproducing the Issue

  1. Go to Physical Inventory work center.
  2. Go to Task Control view.
  3. Select the task ABC. (ABC represents the task ID)
  4. Choose "Confirm with Microsoft Excel" button
  5. Open the Microsoft excel sheet which is downloaded.
  6. Go to SAP Business ByDesign and log on.
  7. Enter the quantities and necessary details.
  8. Choose 'Save data to' button.
  9. After the data is fully processed, the error occurs "Error: item invalid; product/identified stock/logistics unit mismatch".

Cause

  • The Microsoft Excel template used to create the PI Count Document does not have the logistic areas arranged in order, causing the error while updating the Count Task via Microsoft Excel.
  • For example, the file contains logistic areas in this sequence: LOGAREA1 in row 1, LOGAREA2 in row 2, LOGAREA1 in row 3 & LOGAREA2 in row 4.
  • The Excel file should list logistic areas grouped and sorted, such as: LOGAREA1 in rows 1 and 2 & LOGAREA2 in rows 3 and 4.

(LOGAREA1 and LOGAREA2 represents logistics area IDs)

Resolution

For the affected documents complete the existing Count Task and PI Count Document by following the steps below,

Steps:

  1. Go to Physical Inventory work center.
  2. Go to Task Control view.
  3. Search task ABC. (ABC represents the task ID)
  4. For all line items, enter any dummy quantity or select the Zero Quantity checkbox.
  5. Choose Finish All Items.
  6. Open relevant PI Count Document.
  7. Go to the Count Items tab.
  8. Reject all line items.

By following these steps, the current Count Task and PI Count Document get completed without any changes to inventory quantities.

NOTE: After this, create a new PI Count document, ensuring that the logistic areas are sorted correctly in the Excel file.

Symptom

In app W0161 - Balance sheet/income statement - multidimensional, the "0A" suffix shows in Financial Statement Leaf Items (e.g., YCOA00160000000A) 

"Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental."

Environment

SAP S/4HANA Cloud Public Edition

Reproducing the Issue

  1. Access the "Balance Sheet/Income Statement - Multidimensional (W0161)" Fiori app.
  2. Observe that "0A" suffix are displayed after the account numbers in the 'Financial Statement Leaf Item' field, (e.g., in YCOA00160000000A)

Cause

SAP automatically adds these suffixes during Global Hierarchy Activation to differentiate between nodes and accounts that might have the same base values.
The application determines the prefix based on the debit / credit settings. The "0A" specifically indicates this is a debit and credit node (both positions allowed), while "1A" would indicate debit-only and "2A" credit-only. This is part of the Financial Statement Item characteristic (0GLACCEXT) key structure that includes chart of accounts, account number, and debit/credit position indicators. 

Resolution

    The "0A" suffix is a standard SAP system behavior and cannot be removed, as it is essential for ensuring data accuracy and system integrity within financial statement hierarchies.

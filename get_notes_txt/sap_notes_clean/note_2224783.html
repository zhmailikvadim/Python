Symptom

After click on total expander to collapse all subtotal result groups GridView control becomes empty. This happens if original GridView was scrolled beyond visible extent. After GridView collapse scroll position is not validated.

The issue can be recreated using Personas SL client with following scenario:

Scenario:
1. Go to SE38 transaction code.
2. Set program name to ‘BCALV_GRID_10’ and press F8 to execute the report.
3. Again click on execute button or press F8 to get the output
4. Select ‘Plane Type’ column and choose ‘Sort in Ascending order’ or ‘Sort in descending order’ button available in grid toolbar.
5. Select ‘Airfare’ column and click on ‘Total’ button available in grid toolbar.
6. Now select both the ‘Plane Type’ and ‘Airfare’ columns and click on ‘Subtotals’ button.
7. Scroll to the bottom of the table.
8. Click on Totals icon to collapse data.
Expected: All data rows collapsed. Visible only subtotal and total data. 
Result: Empty grid.

Other Terms

GridView - ALV grid control.

Total expander - UI element in ALV GridView which is used to collapse/expand total or subtotal groups.

Solution

To resolve this issue please do the following:

Determine the patch number of the SAPEXE SAR package with the highest patch number for your kernel version on the SAP Service Marketplace (SMP). If the SAPEXE patch number is higher than or equal to the highest patchlevel number you see in the section Support Packages & Patches of this SAP note for your SAP kernel version, you should download and install the SAPEXE SAR package. Otherwise, download and install the most recent DW.SAR archive.

If there is no entry with the patch number yet, please re-check this SAP note at a later point in time. The patch number level will be added automatically once the corresponding patch is available on SAP Service Marketplace.

You can find up to date information about the status of the patch delivery in the following SAP notes:

SAP kernel 745: 2199308

SAP kernel 742: 2098894

SAP kernel 722: 2198998

FAQ:
Expected delivery dates of SAP kernel patches

  • SAP Kernel patches are usually delivered weekly or bi-weekly on Friday depending on the release. From the patchlevel of latest delivered patch you can roughly conclude when the patch you need will become available.

Downloading SAP Kernel patches

  • All patches are available in the Software Center on SAP Service Marketplace (http://service.sap.com/patches). Detailed information on the exact location of the patches can be taken from note 563161. SAP Note 330793 explains how to download patches from SAP Service Marketplace.

Installation of SAP kernel patches

  • SAP note 19466 explains how install SAP Kernel patches.

Symptom

Financial transactions (Commodity Swaps/Commodity Forwards) used in SAP Commodity Price Risk Hedge Accounting are not displayed in the Trader’s Order Cockpit, despite being properly assigned to valid hedge specifications with currently released status.

Affected transactions remain assigned to superseded hedge specifications and become invisible to traders, impacting hedge position monitoring and risk management oversight.

Other Terms

Commodity Price Risk Hedge Accounting, Trader's Order Cockpit, Superseeded Hedge Specification, Period Mismatch

Reason and Prerequisites

When hedge specifications are superseded during the hedge accounting lifecycle, financial transactions may become "orphaned" in the Trader’s Order Cockpit due to periodicity mismatches between original and current hedge specifications.

Technical Background:

  1. Financial transactions are initially assigned to a hedge specification with specific periodicity
  2. New hedge specification creation with identical key parameters (Company Code, Hedge Book, DCS ID, Direction, MIC Code) automatically supersedes the previous specification
  3. Periodicity differences prevent proper mapping of existing transactions to the new period structure
  4. Transactions remain assigned to superseded specifications but lose visibility in the cockpit

Periodicity Downscaling Restrictions:

The system prevents the following downscaling scenarios to maintain hedge accounting integrity:

  • Half-Year (Target Quota) → Quarter-Based (Hedge Book)
  • Year (Target Quota) → Half-Year (Hedge Book)
  • Year (Target Quota) → Quarter-Based (Hedge Book)

Prerequisites:

  • SAP Commodity Price Risk Hedge Accounting is active
  • Multiple hedge specifications exist with same key parameters
  • Periodicity differences between superseded and current specifications
  • Financial transactions assigned to superseded hedge specifications

Solution

Apply the correction instructions provided within this note. After that, an upscaling in the Hedge Specification periodicities will get supported in the Trader's Order Cockpit.

The code corrections applies additional validation during Hedge Specification creation to check the period category downscaling and to block the creation if such situation happens. Additionally to the period category validation

system verifies if there are future financial transactions assigned in Traders Order Cockpit as the downscaling of periodicity is possible when all financial transactions of the Hedge Specifications are settled for the past.

When creating new hedge specifications, consider periodicity impact on existing financial transaction assignments to minimize superseding conflicts.

Please note: Message text translation must be performed manually for non-English system languages (see separate correction instruction).

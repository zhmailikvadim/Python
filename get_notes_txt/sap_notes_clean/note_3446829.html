Symptom

One of the below dumps occurs after start task reported data validation:

  • Category               ABAP programming error
    Runtime Errors         AMDP_DBPROC_CREATE_FAILED
    Except.                CX_AMDP_DBPROC_CREATE_FAILED
    ABAP Program           /1VEC/

or 

  • Category               ABAP runtime error                                                           
    Runtime Errors         AMDP_VERSION_MISMATCH                                                        
    Except.                CX_AMDP_VERSION_MISMATCH  

Environment

SAP S/4HANA Finance for Group Reporting (from edition 2020) 

Reproducing the Issue

  1. Create validation rule with (e.g.) 12 operands in the left side in "Define validation rules" app.
  2. Assign it to a method
  3. Assign the method for consolidation unit or consolidation group
  4. Run the validation task for unit or group

Cause

There are more than 10 operands in the left side by one of the validation rule that is checked during the reported data validation task. 

Resolution

  • Create maximum 10 operands in left side
  • If more than 10 FSitems are needed to check during validation then assign these FSitems to operand(s) by "Operand Expression Editor" in application "Define Validation Rules". More than 10 FSitems can be added to one operand.

Keywords

Define validation rules, dump, reported data validation, F2627

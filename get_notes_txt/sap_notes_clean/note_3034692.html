Symptom

Error "Party role Employee Responsible is not supported for party role category Administrator Party" raised when replicating Contracts from External System to C4C.

Environment

  • SAP Cloud for Customer
  • SAP ERP

Reproducing the Issue

  1. Go to Web Service Message Monitoring under Administrator Work center.
  2. Search for the ContractReplicationFromExtIn Service Interface with application error.

Cause

The error is being raised due to incorrect Code List Mapping for Contract Replication.

Resolution

There is a special Code List Mapping Group required for Contract Party Role Code - Contract Replication and Pricing Mapping Group.

This is to map the local code Employee Responsible (39) to ERP Sales Representative (Y1) and Contract Administrator (1122) to the ERP Employee Responsible (ZM). You need to maintain the remaining codes under PartyRoleCode(Namespace - http://sap.com/xi/AP/Common/GDT) with the new group created. When you select the mapping group 'Contract Replication and Pricing Mapping Group' ensure that you add the base group SAP on premise integration so that the roll back reference happens for the remaining code list mappings.

The new mapping group has to be added to the Communication Arrangement "Contract Replication from and to SAP Business Suite".

Keywords

Party Role, Employee Responsible, Administrator Party, Contracts, ZM, ERP Sales Representative (Y1), Code

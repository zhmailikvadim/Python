Symptom

  • The Master Log table for the application table PRCD_ELEMENTS reached the maximum number of records allowed, resulting in extraction failure.
  • Error in BW: "Change data capture of .... is broken" (Message no. RSODP_ABAP_CDS341).
  • Error in Source System: "Logging table for application table ... has too many records" (Message No. DHCDC_CORE033).
  • Diagnosis: Logging table for ... contained 2,031,817,928 records. Possible causes include:
  • Observer job running but unable to copy changes to subscriber logging tables.

Environment

  • SAP S/4HANA;
  • BW Business Warehouse;
  • BW/4 HANA ;
  • SAP NETWEAVER.

Reproducing the Issue

  1. Execute DTP with no Data Transfer;
  2. Schedule delta to run hourly;
  3. Observe error within 12 hours post-initialization;
  4. Error messages encountered.

Cause

Logging table for application table ... exceeded the record limit due to failure in observer job or inability to copy changes to subscriber logging tables.

Resolution

  1. Unsubscribe the object and subscribe it again to reset the logging table;
  2. Check the SAP_ABA component and implement SAP Note 3619127 - ABAP Data Integration - Transfer job does not process large CDS views in the source system.
  3. Perform re-initialization of the affected object;
  4. Verify the results post-implementation and re-initialization.

See Also

SAP Note 0003619127 - ABAP Data Integration - Transfer job does not process large CDS views.

Symptom

  • During BW/4HANA Remote Conversion, a package needs to be deactivated for some reason. 
  • Run activity 'Verification of Active Packages' and it reports the following error:
    • "Package no. &1 is active in Sender system; de-activate it"
    • "Package no. &1 is active in Receiver system; de-activate it" 

Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.

Resolution

Before deactivating the package, check if there is any task list still open for the package and finalize it using STC02 transaction. After that, the package can be deactivated. 

There are two ways to deactivate the package:

  1. Check existing packages list for a remote conversion in RSB4HCONV
    1. Select the line corresponding to the package you want to deactivate;
    2. Click SHIFT+F2 or the "Deactivate the package" button.

  2. Create a new package via RSB4HCONV or enter a package via RSB4HCONV or CNVMBTPCT
    1. Run activity 'Verification of Active Packages' or check the error message in the log, it should look like this:


    2. Go back to process tree, select the activity and click on the Troubleshooting button in the Process Monitor header menu:

       
    3. Select the third item on the list and click on the button Execute activity:  


    4. Go back to process tree and check the log of the activity again, it should look like this: 
       

    5. Go back to process tree and run the activity again. It should finish successfully now: 




IMPORTANT: Once a package is deactivated, there is no way of reactivating it, so the progress on that run will be lost and a new run with the same scope will have to be created.

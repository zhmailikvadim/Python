Symptom

An Approver can approve or reject his or her own request using the SAP Fiori Access Approver app in violation of End User Personalization settings. The problem only exists when using the SAP Fiori app .

Other Terms

Approver approves  or rejects own request, End User Personalization for stage, EUP, Stage Configuration,  Approve/Reject own requests, SAP Fiori Approver app

Reason and Prerequisites

The SAP Fior app ignore the Approve/Reject Own Requests in End User Personalization.

Companies can decide whether to allow approvers to approve or reject their own requests. This decision is set in Customizing as follows:

  1. From the SAP Reference IMG , choose Governance, Risk, and Compliance -> Access Control -> User Provisioning -> Maintain End User Personalization.
  2. Under Maintain EUP Fields, make an entry in the field Approve/Reject Own Requests. The Visible setting is No:
    • USER  (or field is blank) - During request approval, if the User ID on the access request = the Login ID of the approver, the system issues an error message.
    • USER AND REQUESTOR - During request approval, if the User ID or the Requestor ID on the access request = the Login ID of the approver, the system issues an error message.
  3. If the Visible setting is Yes: the system makes no validation checks

Solution

The SAP Fiori Access Approver app abides by the Stage configuration settings in End User Personalization. To resolve the problem, implement the note.

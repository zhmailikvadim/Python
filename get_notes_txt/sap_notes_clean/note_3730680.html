Symptom

You are planning to license Data Quality Management (DQM) together with SAP Sales Cloud / Service Cloud Version 2.
However, the following points related to DQM unit consumption are unclear:

  • Whether DQM units can be triggered only for prospects and not for customers, and whether such configuration options exist.
  • Whether DQM units are consumed when Business Partners (BPs) are created automatically from an external system (e.g., portal via SAP CPI).

Environment

SAP Sales Cloud Version 2
SAP Service Cloud Version 2
Data Quality Management (DQM)

Cause

A DQM unit is charged each time a user triggers a DQM action, such as:

  • Clicking Validate Address
  • Selecting a suggestion from the type‑ahead address search

Customers typically want to control when DQM is invoked and whether there are configuration options to limit its usage to specific business partner types (e.g., prospects only).

Resolution

DQM units are consumed only when DQM functionality is explicitly triggered within the Sales Cloud UI, such as:

  • Validate Address action
  • Type‑ahead address suggestion selection

Filtering DQM Usage by BP Type

DQM can be configured through dynamic properties so that the feature is enabled only for specific BP categories (for example, Prospects).
However, DQM is still triggered whenever the user interacts with address validation or type‑ahead features where they are available.

1. If a user types an address and selects an option from the type‑ahead suggestions, do units get consumed?

Answer:
Yes. Selecting an address suggestion in the type‑ahead field always consumes DQM units.
The suggested address always overwrites the currently maintained data—regardless of whether the field was initially empty or prefilled.

Important:
Standard address fields must not be hidden, as doing so is not supported and may disrupt standard processes.

2. If Accounts or Individual Accounts are created automatically via an external source (e.g., CPI from an external portal), are DQM units consumed?

Answer:
No. If the BP is created in the background through an external interface, DQM is not invoked automatically, and therefore no DQM units are consumed.

At present, there is no API-based DQM trigger available during external data loads.
Once SAP introduces API-based address validation in the future, DQM units will be consumed if the external process explicitly invokes DQM for address standardization.

See Also

As summary, DQM units are only consumed when user-triggered within the UI (Validate Address or Type-Ahead).Usage can be limited via dynamic properties (e.g., only for Prospects).External/automated BP creation does not consume units, unless a future API-based DQM function is intentionally invoked.

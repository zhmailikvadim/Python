Symptom

You use the Online validation service to validate VAT Number of business partners against the VIES service. There might be a lot of low-quality data in the system, especially in the Development and Test systems. Validation of such low-quality data might cause issues during testing and causing a lot of traffic to the VIES service, even violating the VIES's Terms of use.

Other Terms

 OVS, Online Validation Service, VIES, EU

Reason and Prerequisites

Requesting validation of obviously invalid VAT numbers might lead to IP address ban by the VIES service.  
Activate the country-specific validation in the IMG activity "Field Checks for Countries/Regions" (transaction code OY17). The VAT Number check is activated when the checkbox "Other data" is checked.

Solution

The rules on VAT Number structure for each EU country is a public information and it is possible to check the length, check digit, etc. Those rules are already a part of the SAP system. 
A new result code has been introduced when a VAT Number doesn't pass such offline validation. There is no need to validate VAT Number having incorrect length etc.

Complete set of result codes can be found in the SAP Note 3085488. Implementation overview for the Online Validation Service and the VIES validation can be found in the SAP Note 3291504.

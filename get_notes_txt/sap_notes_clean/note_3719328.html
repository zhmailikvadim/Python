Symptom

After implementing SAP Note  3639546 and executing report NOTE_3639546 (manual post-implementation steps) via transaction SA38, the SM30 view FIMXV_RGN_CLFN is not generated, even after running the report in Test Run and then Update & Activate mode.

Environment

  • SAP ERP
  • SAP S/4 HANA

Reproducing the Issue

  1. Implement SAP Note 3639546.
  2. Open transaction SA38.
  3. Enter report NOTE_3639546 and execute it.
  4. Select Test Run and execute the report.
  5. If no errors are returned, choose Back.
  6. Select Update & Activate and execute the report.
  7. Open transaction SM30 and check for view FIMXV_RGN_CLFN.
  8. The view FIMXV_RGN_CLFN is not generated / not available in SM30.

Cause

Objects from manual step in SAP Note "3639546 - DRC - Mexico Reporting: New View for Border Region Customizing" are delivered in a later SAP Note from overview 2986139 - DRC - Mexico VAT Reporting: Frequently Asked Questions (FAQ) and Recommendations, more precisely at SAP Note "3651522 - DRC - Mexico Reporting: Post-Implementation Objects for SAP Note 3639546"

Resolution

  1. Execute transaction SE54
  2. Enter the FIMXV_RGN_CLFN in field Table/View
  3. In frame Edit Table/View choose radio-button Generated Objects
  4. Press button Create/Change
  5. Navigate over the menu Generated Objects -> Change F7
  6. In screen Recreate Generation Elements: Exert Mode press icon Select All (F6)
  7. Press icon Continue (Enter)
  8. Request completed without errors appears in the status line

Symptom

You want to transfer stock segments using movement type 309. The product stock is in one segment and needs to be moved to another, but the transfer posting transaction does not move stock between segments.

Environment

  • S/4HANA Retail & Fashion Management (LO-RFM) 
  • SAP ERP Central Component
  • SAP ERP
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA
  • SAP S/4HANA Cloud Private Edition

Reproducing the Issue

  1. In transaction MIGO, choose Transfer Posting with movement type 309.
  2. Select the Material and Batch.
  3. Transfer from stock segment XXXX to stock segment YYYY.

Cause

The batch has a unique stock segment value, and changing it is not allowed per the standard design.

This standard behavior prevents inconsistencies between stock segment values, the batch, and related documents in the system.

Resolution

We cannot support 309 movement type scenarios to change the stock segment value for the same batch.

Keywords

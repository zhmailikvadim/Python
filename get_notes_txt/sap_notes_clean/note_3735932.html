Symptom

When the DSO sends an electricity INVOIC to a supplier, the system raises error message /US4G/PROC_BILLING158 ("Due date can be maximum 10 WD after document date").
This issue is relevant for the market role: DSO.

Other Terms

GUB, INVOIC, REDISND1, /US4G/PROC_BILLING158, /US4G/CL_UBD_INVDOC, VAL_INVDOC_HEAD_FOR_BB, VAL_INVDOC_HEAD_FOR_GU, electricity, supplier, due date, document date

Reason and Prerequisites

Program error. The check logic for due date and document date plus 10 working days was incorrectly applied to electricity INVOIC messages. This validation should only apply to gas INVOIC messages.

Solution

Implement the corrections of this SAP Note or import the relevant support package.
After implementing this SAP Note, the system no longer raises error message /US4G/PROC_BILLING158 when you send electricity INVOIC messages to suppliers.
You can verify the fix by checking that the error message /US4G/PROC_BILLING158 ("Due date can be maximum 10 WD after document date") is not raised when you send electricity INVOIC messages to suppliers.

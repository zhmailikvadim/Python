Symptom

After maintaining source list, MRP doesn't consider the rounding value or rounding profile in info record.

Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.

Environment

  • SAP S/4HANA, on-premise
  • SAP S/4HANA Cloud Private Edition

Reproducing the Issue

  1. Create info record in ME11 and maintain rounding.
  2. Maintain source list in ME01.
  3. Run MRP and check the result.

Cause

This issue only exist after updating sourcing logic with note 3233269. And it can be solved by deleting source list and set auto_sourcing indicator in info record.

After implementing note 3233269, MRP reads sourcing data from view IMRPPROSOS. Source list is considered as a separate. System reads info record firstly and use it to check corresponding source list. If the source list is valid and can be used to replace info record, system copy following data from info record to source list and then use source list to calculate sourcing. Rounding is not considered.

  • Planned delivery time
  • GR processing time
  • Vendor
  • Production version(when info record category is subcontracting)

If there's no source list and auto_sourcing is active, MRP will directly read rounding from info record to create receipts.

Resolution

Delete the source list and set auto_sourcing indicator in ME12, then MRP will consider info record's rounding.

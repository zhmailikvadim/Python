Symptom

In SAP Service and Asset Manager any non-stock components added to a Work Order are missing their description on the Parts list view. The description is present, however, on the Part details page.

Additionally, when viewing parts from an operation, the object cell contains the operation number and description on every cell. On smaller form factors, this takes up unecessary space as every part will contain the exact same information.

Other Terms

  • SAP Service and Asset Manager
  • SSAM
  • ICMTANGOAMF10-42950
  • PartsListView
  • Parts
  • Non-Stock

Reason and Prerequisites

Details page and List View pages were using different properties to display the title; on the list view screen the non-stock description field is just not filled.

Solution

To apply the patch, download the attached 3719725-XXXX.patch file corresponding to your version and save it in a known location, preferably adjacent to your SAM.mdkproject.

If you are on Windows, see SAP Note 3533879 for how to install a .patch file in Windows. If you are on a Mac, follow the steps below:

  1. Open a terminal in the root of your SAM metadata (i.e. SAM.mdkproject/metadata or SAM.mdkproject/metadata/base.definitions)
  2. Run the terminal command `patch --no-backup-if-mismatch -p4 < path/to/3719725-XXXX.patch`. 
  3. Verify in the terminal window that the following files have been modified and listed as patched, as indicated by the console output:
  • patching file 'Pages/Fragments/Parts/PartsObjectCardCollection.fragment'
  • patching file 'Pages/Parts/PartsListView.page'
  • patching file 'Rules/Parts/PartLibrary.js'
  • patching file 'Rules/Parts/PartsFootnoteDisplay.js'
  • patching file 'i18n/i18n.properties'

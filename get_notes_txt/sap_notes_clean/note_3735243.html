Symptom

  • When opening the translation popup for a behavior, the behavior name in the current login locale is automatically and incorrectly replaced by the value from the behavior’s original creation locale.
  • Even after clicking Cancel, the incorrect name remains. If the user then edits another field (for example, the description) or changes the status, the Save action becomes enabled and saving persists the unintended name for the current locale.

Environment

SAP SuccessFactors Talent Intelligence Hub

Reproducing the Issue

  1. Set the login locale to Locale A, for example, UK.
  2. Create a behavior in manage TIH in UK locale and maintain a translation for Locale B, for example Japanese.
  3. Switch the login locale to Locale B, Japanese in this case, and edit the abovementioned behavior.
  4. Open the translation dialog; it defaults to the behavior’s original creation locale A, UK in this case.
  5. Observe that the behavior name in the background for the current locale (Locale B, Japanese in this case) is overwritten by the value from Locale A, UK in this case.
  6. Click Cancel; the overwritten value remains.
  7. Change another field (for example, the description) or change the behavior status; the Save action becomes enabled.
  8. Click Save. The behavior name for the current login locale (Locale B, UK in this example) is saved with the unintended value - translations maintained for Locale A, UK in this case.

Cause

This is a Known Issue.

Resolution

The fix for this issue is planned to be deployed on 1H 2026 release.

For release timelines, please review SAP SuccessFactors Product Release & Road Map Information.

Workaround

No workaround is available.

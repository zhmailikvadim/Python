Symptom

  • LDAP manual login and SSO fails after updating user Common Names (cn) in LDAP.
  • New user is being created under the new cn and not mapping to the existing user.

Environment

  • SAP BusinessObjects Business Intelligence Platform 4.3
  • SAP BusinessObjects Business Intelligence Platform 2025
  • Windows
  • Linux / Unix 

Reproducing the Issue

  1. Change user Common Names through LDAP.
  2. Perform alias update to LDAP users and groups in BI CMC.
  3. Observe new users being created.

Cause

  • LDAP import mapping uses the cn by default.

Resolution

  • Open LDAP Configuration Wizard by clicking on any of the fields.
  • On the "LDAP Server Type" page, select "Attribute Mapping"
  • Change the "User Name" mapping field to something that corresponds to the Account Name of users on BI, for example: sAMAccountName for Microsoft Active Directory.

Symptom

  • Transaction Amount and Transaction Currency, marked as sensitive, are still visible in Inbound and Outbound Payment Orders in the Manage Payments app.

  • Payment Items, though marked as sensitive, are displayed in the app.

  • Expected masking/hiding of sensitive fields (Transaction Amount, Transaction Currency) is not applied.

  • Issue occurs after loading the payment file in XML and viewing Inbound/Outbound POs.

Environment

SAP S/4HANA 2023

Reproducing the Issue

  1. Load a payment file in XML format.

  2. Open the Manage Payments app.

  3. View Inbound and Outbound Payment Orders.

  4. Observe that for Outbound POs, Transaction Amount and Transaction Currency are visible.

Expected Result:

  • Transaction Amount and Transaction Currency should be masked/hidden as per sensitivity settings.

  • Payment Items marked as sensitive should also not be visible.

Cause

Sensitive fields (Transaction Amount, Transaction Currency, and Payment Items) are not being correctly masked in the Manage Payments app despite configuration, likely due to a missing or misapplied masking logic in the app’s display layer for Inbound and Outbound Payment Orders.

Resolution

  • The visibility of Transaction Amount and Transaction Currency in the Manage Payments app is due to the user having full authorization for object /PF1/DISP with /PF1/DATAT = *.

  • To restrict and mask this sensitive data, the security team should remove or appropriately limit the /PF1/DISP authorization for the user.

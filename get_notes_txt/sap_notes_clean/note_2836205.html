Symptom

Auto enrollment job for Insurance benefit is checking only the Default values for Insurance Plan and Coverage. There are other insurance plans and coverages available in the benefit, but the auto enrollment job only checks for default values.

Environment

SAP SuccessFactors Employee Central: Global Benefit

Reproducing the Issue

Run auto enrollment job for a benefit having different insurance plan and see that job is checking only default Insurance plans. In case the Employee is not eligible for the Default Plan or coverage, the enrollment is not done.

Cause

This is expected behavior. Auto enrollment jobs for Benefits will only check for the default values configured in the benefit. This behavior is expected for the old auto enrollment jobs, the tracker based jobs and the automation processes involving ISC.

Resolution

If the requirement is to have different insurance plans or coverage other than the default, then it's necessary to create individual benefits for each Insurance Plan, and define the eligibility rule in the "Eligibility Rule For Coverage".

You can visit our Customer Community Portal and check the idea '341125' with an enhancement focused on this scenario and vote if it makes sense for your requirement or, if you prefer, you can raise another enhancement request as per KBA 2090228 to have this functionality considered in future development cycles

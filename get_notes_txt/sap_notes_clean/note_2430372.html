Symptom

You use SAP Environmental Compliance 3.0 and edit tasks in the overview. You notice that some tasks no longer have any corresponding task templates.

 

Other Terms

﻿xEM, EC, SAP EC, EP, SAP EP, EEM﻿

﻿Emissionsmanagement, Einhaltung von Umweltschutzauflagen, Umweltleistung, Energiemanagement, Übersicht, Aufgabenvorlage﻿

﻿Emission Management, Environmental Compliance, Environmental Performance, Enterprise Energy Management, Dashboard, Task Template

 

Reason and Prerequisites

This symptom is caused by a program error.

The task templates have been deleted, but the related tasks were not deleted.

You use SAP Environmental Compliance 3.0 with a support package level or patch level lower than the level specified in the Support Packages & Patches section.

 

Solution

Download the complete solution for your support package level using the appropriate link in the Support Packages & Patches section and implement it in your system. When you do this, apply the other required patches as well. For more information, see SAP Note 1974464.

Note: Patches for ﻿Env.Compl. Centr. Services 3.0﻿ require a server restart to become fully active.

Note: Corrections have been provided in the interim that require manual tasks. Check the manual tasks of the following SAP Notes and carry them out if needed.

Current Support Package level SAP Notes
Support Package SP20 2297746, 2368673, 2370485, 2389521
Support Package SP21 2297746, 2368673, 2370485
Support Package SP22 2297746, 2370485

 

Execute the new data migration "﻿Delete Tasks with deleted Templates". This data migration deletes all tasks that have no valid task template. The log of the data migration lists all deletion actions.

If the affected tasks are of the type "If another task is completed" (Workflow Tasks) or of the type "When Parent Task Is Generated" (Sub-Tasks), the data migration returns an error. You must then first manually delete all dependent tasks so that the migration can delete the tasks. Execute the data migration "﻿Delete Tasks with deleted Templates﻿" again after you have deleted the dependent tasks.

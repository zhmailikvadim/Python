Symptom

  • During accounts.socialLogin with a social provider, if no matching account exists, the platform automatically creates a new account/UID.
  • The desired behaviour is to stop the flow for unknown identities and return an error instead of registering a new account.
  • Inquiry whether this can be achieved by intercepting the onBeforeAccountRegister event with an extension.
  • No specific error message or error code is observed; the issue is the automatic registration itself.

Environment

  • SAP Customer Data Cloud
  • RaaS (Screen-Sets, Site Policies, Schema)
  • Social login

Reproducing the Issue

  1. Initiate accounts.socialLogin with a social provider (for example, Google or Facebook).
  2. Use a social identity that is not linked to any existing account and has no matching email in the database.
  3. Observe that a new account/UID is automatically created instead of stopping the process.

Cause

  • The default platform behaviour creates a new account when a social login does not match an existing account or linked identity.
  • There is no Admin Console configuration option to disable automatic account creation for social logins.

Resolution

  1. Implement a CDC Extension that handles the onBeforeAccountRegister hook.
  2. In the extension, allow normal processing when a social identity matches an existing account or a linked identity (login proceeds without intervention).
  3. When the hook fires for a prospective new account, perform an accounts.search using the email returned by the social provider to check for an existing account.
  4. If an account is found for the email, allow registration to proceed so the platform can handle the account linking flow.
  5. If no matching account is found, return a cancellation response in the extension to stop registration and surface an appropriate error to the client.
  6. Note that there is no Admin Console switch for this behaviour; the extension is the recommended and supported approach.

Symptom

Following the SAP Note 2800483 to upgrade the ASE server. But the upgrade is failed with below error message:

ERROR AT sybServer.cpp:removeDirectory:1090 TIME: 2026 Mar 05 08:41:31 DESCRIPTION: Failed to remove file: F:\sybase\<SID>\sapdbctrl-config\tmp_install\1622929.tmp LAST OS-ERROR: (5) : Access is denied.
SAP_DB_LiveUpdate:  LiveUpdate failed
ERROR => 'Database live update' failed: Installation failed with rc=-1. Please check F:\sybase\<SID>\sapdbctrl-config\sybInst.4156.out. [sapdbctrl.cp 5795]

1. From the ASE suite.log, Below error messages occurred:

    NONFATAL_ERROR - class com.sybase.ia.common.InstallMergeModuleStatus FatalInstallException Adaptive Server Enterprise: FAILED.  See log file F:\sybase\<SID>\log\ase.log for detail
    LastScreen.init(): install status is FATAL_ERROR:  The installation failed.

2. From ASE.log, below error messages occurred:

   ERROR - Flexeraak8: F:\sybase\<SID>\data\empty - Access Denied

   
    
 

Environment

  • SAP Adaptive Server Enterprise (ASE) 16.0 for Business Suite
  • SAP NetWeaver (NW) - All versions.
  • Windows OS

Cause

Due to empty file created as a folder (ERROR - Flexeraak8: F:\sybase\<SID>\data\empty where empty is a folder name not a file)

Resolution

  • The solution for this issue is to remove the empty folder from the path:
      F:\sybase\<SID>\data\empty
  • Then copied the empty file from another working system like Development or Test systems
  • Re-run the upgrade process. 

See Also

Note 2800483 - SYB: Upgrade / Update SAP ASE with SAP Host Agent 7.21 PL43 and later

Keywords

Installation Failed, Fatal error, Live update, Access denied, Installation failed with rc=-1, sybInst

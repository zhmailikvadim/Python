Symptom

When I run the program ARBCIG_MASTER_DATA_EXPORT in SAP to send the SupplierConsolidated.csv file to SAP Ariba Buying and Invoicing via Managed Gateway for Spend&Network, I see the error This value already exists. Duplicates are not allowed which is visible in Ariba Buying and Invoicing only. 

Environment

SAP Ariba Buying and Invoicing

Reproducing the Issue

  1. Open your SAP environment
  2. Go to transaction SE38
  3. Execute program ARBCIG_MASTER_DATA_EXPORT
  4. Choose Solution Procure-to-Pay or Procure-to-Order
  5. Within Choose Master Data to Export, tick the checkbox Supplier
  6. Click Execute

Cause

Missing configuration of the ARBCIG_SYSTIDMAP table in SAP ERP, therefore the vendor ID gets incorrectly sent in the SupplierConsolidated.csv file (in column SystemID) and the error This value already exists. Duplicates are not allowed happens in SAP Ariba Buying and Invoicing.

Resolution

Maintain the table ARBCIG_SYSTIDMAP in SAP ERP according to this documentation: About Transferring Parameters and Filters in the Maintenance Tables to Other SAP System Landscapes

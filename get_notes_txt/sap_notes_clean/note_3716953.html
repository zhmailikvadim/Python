Symptom

  • When workers are created using the Identity Management Entry Source, a user ID along with additional related records are automatically generated, even for worker types where these creations are not required or desired.
  • Error message encountered: "User-driven identity management is active in the system. You cannot create a worker through this channel because workers are nor created using the Identity Management System." WFD_WP1_MSG138.

"Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental."

Environment

SAP S/4HANA Cloud Public Edition

Reproducing the Issue

  1. Use the Identity Management Entry Source to provision a worker.
  2. Observe that both a user ID and additional records are created automatically.

Cause

It is as per SAP Standard Behavior. The Identity Management Entry Source is configured to automatically create user IDs and additional records during worker provisioning, which cannot be disabled for specific workers.

Resolution

  • Manually create workers without user IDs using the "Manage Workforce" application.
  • Alternatively, utilize the CSV Import option to create workers without user IDs.

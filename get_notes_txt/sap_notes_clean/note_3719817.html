Symptom

I am performing a data upload (e.g., Cost Center Upload) which completes with a "Success" status. However, when I check the records in the SAP Fieldglass User Interface (UI), specific fields such as the Description remain blank or reflect old data, even though the values were present in my upload file.

Cause

This issue occurs when the native Fieldglass labels have been renamed using Custom Text (e.g., renaming Cost Center to Cost Object).

When labels are customized, the standard upload engine may ignore the native header name in your file (like Cost Center Description) because it no longer "recognizes" that name as a valid mapping for the customized object. Since the header is not recognized, the system simply skips that column without triggering a failure, leading to missing data in the UI.

Resolution

You must map your upload file headers to the customized system fields using Upload Column Transformation.

Log in with your Configuration Manager account.

  1. Log in with your Configuration Manager account.
  2. Navigate to Connector Catalog.
  3. Search for the impacted connector and click Edit.
  4. Go to the Upload Column Transform section under the Customize header.
  5. Click + Add Native Fields.
  6. Enter the mapping details:
    • Standard Column Name: Select the native name (e.g., Cost Center Description).
    • File Column Name: Enter the exact header name used in your upload file.
  7. Click Update and follow the wizard to save.
  8. Re-run the upload file to verify the data now appears in the UI.

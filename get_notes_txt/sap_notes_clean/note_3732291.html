Symptom

Unable to activate Form Template in 'Output Parameter Determination' app due to the following errors:

  • Columns must be adapted to change of result data object APOC_S_BRF_DT_FORM_TMPL.
  • Error: DT_FORM_TEMPLATE: Incompatible result column PURCHASEORDERTYPE.
  • Error: DT_FORM_TEMPLATE: Please adjust the table settings.

Environment

 SAP S/4HANA Cloud Public Edition

Reproducing the Issue

  1. Open the 'Output Parameter Determination'(OPD) app.
  2. Select Show Rules for Purchase Order and navigate to the determination step Form Template.
  3. Attempt to add the “Document Type” column via Table Settings in 'Result column', then choose Check and Activate.
  4. The system throws the error: "Columns must be adapted to change of result data object APOC_S_BRF_DT_FORM_TMPL".

Cause

Result columns in DT_FORM_TEMPLATE do not align with the expected structure APOC_S_BRF_DT_FORM_TMPL (supports only Form Template and Form Language), and additional fields like PURCHASEORDERTYPE cause the error.

Resolution

  1. Open the 'Output Parameter Determination' (OPD) app and edit the decision table.
  2. Go to Table Settings and review the 'Result columns.'
  3. Remove any unsupported result columns (retain only Form Template and Form Language columns).
  4. Check and activate the decision table.
  5. Then add the required field under 'Condition Columns', and perform Check and Activate again to resolve the error.

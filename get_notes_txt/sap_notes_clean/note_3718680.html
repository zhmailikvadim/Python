Symptom

Authorization Error is received while sharing data from Enterprise Databricks on Azure to SAP BDC:

failed to execute initial load: SQL Error 129 - transaction rolled back by an internal error: REST API: requestType=Blob_GetBlob, statusCode=403, msg=[{"error":{"code":"AuthorizationFailure","message":"This request is not authorized to perform this operation.\nRequestId:c79a45a4-401f-0069-777e-992d20000000\nTime:2026-02-09T04:39:19.8552456Z"}}] was the body; [x-ms-error-code, AuthorizationFailure] among the headers; Request ID info: x-ms-request-id=c79a45a4-401f-0069-777e-992d20000000. Invalid parquet file <file> msg=Failed to read file metadata (statement no: 0).

Environment

  • SAP Business Data Cloud
  • SAP Business Data Cloud Connect
  • Enterprise Databricks

Cause

BDC-Datasphere is not authorized to read Parquet files of the shared Delta table that is stored on the Azure storage container. This could be related to network policy restrictions that may be configured on the Azure storage account.

Firewall rules are missing the allowlisting of SAP HANA Database VNets.

Resolution

See SAP Note 3726123 - SAP BDC Connect for Databricks - Authorization Failure with Firewall-Protected Azure Storage

Keywords

BDC Connect, Azure, Databricks, AuthorizationFailure, statusCode=403, parquet file, authorization

Symptom

  • Error "OAuth access_token not found from Destination: ######## (HTTP/1.1 400 Bad request)" when trying to Register an on-premise system to SAP Cloud ALM 
  • Error "OAuth access_token not found from Destination :######## (HTTP/1.1 401 Unauthorized)" in on-premise managed systems registered to SAP Cloud ALM

Environment

SAP Cloud ALM 

Reproducing the Issue

  1. Logon to the managed system 
  2. Execute T-code /n/sdf/alm_setup 
  3. Enter all the required details as per document : Setup Managed Components (Services & Systems)
  4. Execute the Step 3 : Enter background user and register system 
  5. Registration fails with the Error message displayed

Cause

  • Incorrect ALM Destination entered 
  • Missing ST-PI Notes and Authorizations 

Resolution

For error : OAuth access_token not found from Destination: ######## (HTTP/1.1 400 Bad request) - Ensure that the correct ALM Target Destination is entered and that the Connection exists in SM59 transaction 

For error : OAuth access_token not found from Destination :######## (HTTP/1.1 401 Unauthorized) - Ensure that the relevant ST-PI notes are implemented and the necessary authorizations are provided for the Background user. Follow the resolution steps provided in the KBA 3316324 - Error "OAuth access_token not found from Destination :######## (HTTP/1.1 401 Unauthorized)" in on-premise managed systems registered to SAP Cloud ALM

Keywords

HTTP, 400, Bad request, Unauthorized, 401, OAuth access_token not found from Destination

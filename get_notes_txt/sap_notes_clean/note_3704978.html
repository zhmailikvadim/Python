Symptom

  • During data replication from SAP S/4HANA to Middleware, the field MSGFN is defaulted to the value "005" regardless of the action (create/change/delete) performed for a cost center.
  • The system is unable to identify the correct action during replication.

Environment

  • SAP S/4 HANA

Reproducing the Issue

  1. Create a cost center using transaction code KS01.
  2. Check the COSMAS IDOC in transaction WE02.
  3. Verify the field in the data segment of the generated IDOC before and after any update to cost center.

Resolution

The reason why we use the initial Message function number in FM MASTERIDOC_CREATE_SMD_COSMAS / FORM CONVERT_CDCHGID_TO_MSGFN: 
C_MSGFN_R =  '005'  "Refresh/Replace" is that COSMAS IDoc's do not support any delta changes in the inbound system (the logic is the similar as if you transport the cost center master data)

But the inbound FMIDOC_INPUT_COSTCENTER IDOC (Initial Processing: IDOC Use for Object Cost Center) executes the complete refresh of the inbound data:

  • First ALL cost center intervals from inbound are deleted 
  • Then created from the scratch corresponding to the incoming IDoc.

External systems don't need the information about creation/update as the data are refreshed anyway and will be undated responding to the data stored in IDoc.

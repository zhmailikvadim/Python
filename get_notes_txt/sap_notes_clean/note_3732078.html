Symptom

  • It is expected to enable the F4 functionality with movement types and reason codes in LI21 transaction during write-off.
  • Reason field in LI21 is visible, but F4 search help for reason codes does not initially work.
  • After enabling f4, selecting a reason code leads to error: "Entry 9001 does not exist in T157D (check entry)".

Environment

  • Product: SAP S/4HANA
  • Warehouse Movements

Reproducing the Issue

  1. Start transaction LI21.
  2. Enter a warehouse number and storage type (for example, 999 for differences), then display items to be cleared.
  3. Select a line and choose Write Off.
  4. Place the cursor in the Reason field and press F4 to open the value help.
  5. Select a reason code (for example, associated with movement types 711/712) and save.
  6. Observe that the system raises: "Entry 9001 does not exist in T157D (check entry)".

Cause

  • The selected reason code is checked against table T157D with BWART (movement type) blank, i.e., the system validates BWART = '' and GRUND = . If only entries with BWART = 711/712 exist, the check fails and raises the T157D error.

Resolution

  1. Configure reason codes for goods movements:
  2. 1. Go to SPRO → Materials Management → Inventory Management and Physical Inventory → Movement Types → Reason for Goods Movements (transaction OMB_CONTROL_REASON).
  3. 2. Maintain the list of reason codes and their texts.
  4. 3. Set the input status for the Reason field (optional/required/suppressed) per movement type as needed.
  5. Make the Reason for Movement mandatory for relevant movement types:
  6. 1. Open transaction OMJJ.
  7. 2. For the relevant movement types (for example, 711/712), set “Reason for Movement” to required.
  8. Maintain T157D so LI21 checks pass:
  9. 1. In the same “Reason for Goods Movements” customizing, create/ensure entries in T157D for each used reason code with BWART left blank (BWART = '', GRUND = ).
  10. 2. Keep existing entries tied to specific movement types if required; do not delete them unless your processes require it.
  11. Test in LI21:
  12. 1. Open LI21, perform Write Off, use F4 on Reason, select the desired reason, and save to confirm the error no longer appears.

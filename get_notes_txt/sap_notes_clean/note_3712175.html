Symptom

After a successful SAP HANA System Replication takeover, XS or OData services are not reachable and return HTTP 404.
The System Replication status (hdbnsutil -sr_state) shows Primary/Secondary in sync with no replication errors.

Environment

SAP HANA DB Version 2.0

Cause

SAP HANA System Replication status does not by itself confirm consistency of XS-related configuration between sites.
INI/profile parameters (for example xsengine.ini and other XS-related configuration settings) are local to each host and are not automatically replicated between primary and secondary systems.
If XS-related configuration differs between sites (for example xsengine.ini settings such as http_url), XS services may not be reachable after takeover although System Replication itself is healthy.

Resolution

Ensure that xsengine.ini and relevant XS-related configuration entries are identical on both primary and secondary hosts before patching or failover testing.
After aligning the configuration, perform a controlled takeover test and validate XS/OData access on the active system.
Optionally, use SAP-provided mechanisms to check INI parameter consistency prior to maintenance activities.

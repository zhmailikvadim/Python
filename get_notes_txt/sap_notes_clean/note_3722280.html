Symptom

Monthly valuation period results are calculated correctly in the time sheet but are not transferred to the TOIL time account as expected. 

Environment

SAP SuccessFactors Time Tracking

Reproducing the Issue

  1. Create the following time valuations to calculate the difference between recorded hours and actual planned working hours monthly.
    1. Store planned working time in a daily time container REB_Daily TC. 
    2. Store recorded working time in a daily container BLUE_DAILY_TC. 
    3. Use a monthly Time Container to store the overtime and post it to the TOIL Account. 
  2. Create working time in the time sheet and the monthly Time Container stores the correct result. 
  3. Observe that the Time Valuation Result is not transferred to the TOIL time account.

Cause

This is expected behavior as of 2H 2025 release.
The TOIL function is not yet supported in the second-level time valuation (Asynchronous Flexible Period Time Valuations influenced by Time Containers).

Resolution

The calculated value of a time type group can be reported by creating a Story report for the object Time Container.

The TOIL function is planned to be delivered in the next release, but the final implementation date is not guaranteed.

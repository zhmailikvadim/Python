Symptom

Error connecting AO to BW when starting the connection from the AO add-in. The error in the symptom can be seen in the AO trace files or when connection attempt is without SNC. When using SNC, the error may be masked by the "Couldn't establish safe connection to system XXX" message.

Environment

  • SAP Analysis for Microsoft Office
  • SAP Analysis for Microsoft Office, edition for SAP Analytics Cloud

Reproducing the Issue

  1. Open Analysis add-in in the client machine.
  2. Attempt to connect to BW system (via Insert Data Source or via Open Workbook)
  3. Select the BW system and click to connect (with SSO or user and password). Get the error.

Cause

Missing target hostname in the XML file used by AO.

Resolution

This issue occurs when the XML file that AO is using to retrieve the BW systems (SAPUILandscape.xml) is not correctly maintained and the affected system entry is missing the hostname to where it should connect. It is even possible that the XML file contains the required tags with the message server, but these tags are not linked to any other tag in the XML file.

This can be confirmed by looking at the connection definition in SAP Logon, when right-clicking the connection and choosing Properties. There, the Message Server field is empty as per below screen shot:

If SAP Logon is installed in the same machine where AO is running, then the recommendation is to manually maintain the message server name in the screen above, save and test again in AO. If no SAP Logon is installed, the XML file can be manually edited, although in this case extra caution is necessary as it cannot contain syntax errors.

Symptom

  • The header information is blank in line item display transactions FBL*N or FAGLL03 when using SAP GUI control setting Edge (based on Chromium). 
  • The issue could also occur in other transactions.

    "Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental." 

Environment

  • SAP ERP
  • SAP enhancement package for SAP ERP
  • SAP S/4HANA        All versions

Reproducing the Issue

  1. Execute one of the FBL*N transactions.
  2. Observe that the header information is blank.

Cause

The issue is caused by the implementation of SAP note 3357158.
There was a change in 'REPORT_TEXT_EDIT' , 'REPORT_TEXT_PRINT'. The FBL*N transactions and FAGLL03 use these over the FILITEXTS function group to display the header data.

Resolution

  1. Refer to SAP Knowledge Base Article 3704912  : "Latest Webview2 Version 144.xxx causes problems with SAP GUI HTML control when using edge as browser control in SAP GUI for Windows".
  2. Follow the recommendations provided in the SAP Knowledge Base Article to address the issue of "Empty Header rows" in SAP GUI transactions.

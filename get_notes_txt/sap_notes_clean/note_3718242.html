Symptom

XSA Upgrade fails due to:

com.sap.xs2rt.installation.util.InstallationException: Updating the Product Installer application failed: Execution of command 'deploy /hana/shared/<SID>/xs/installdata/apps/product-installer/product-installer.mtar' failed.
...
Caused by: java.lang.RuntimeException: Execution of command 'deploy /hana/shared/<SID>/xs/installdata/apps/product-installer/product-installer.mtar' failed.
...
Program terminated with exit code 1.

Environment

XS Advanced Runtime / XS Controller

Reproducing the Issue

  1. Perform an XSA upgrade.
  2. Observe the failure during the upgrade process with error messages such as "Updating the Product Installer application failed" and "Program terminated with exit code 1".

Cause

Duplicate deploy-service instances running on the same machine.

Resolution

The solution is to delete the excess deploy-service instances.

The steps are:

  1. As <SID>ADM user on the HANA system, call XSA stop
  2. Delete all remaining deploy-service processes:
    • e.g. with kill -9 128535 , 3264152
  3. Make sure, no deploy-service process is running any more
  4. Call XSA start
  5. Log-in at XSA with XSA_ADMIN
    • e.g. with xs login
  6. Make sure that deploy-service has only one instance. For this:
    1. Call xs target -s SAP;
    2. Then xs scale deploy-service -i 1
  7. Repeat the upgrade after the above requirements are met.

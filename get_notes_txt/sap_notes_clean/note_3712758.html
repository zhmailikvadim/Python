Symptom

When define forecast profile in MP80, not all the setting in MM02 can be pre-defined with initial value.

Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.

Environment

  • SAP ERP Production Planning (PP)
  • SAP R/3
  • SAP R/3 Enterprise
  • SAP ERP Central Component
  • SAP ERP
  • SAP enhancement package for SAP ERP
  • SAP Enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA, on-premise
  • SAP S/4HANA Cloud Private Edition

Reproducing the Issue

Run MP80 and create forecast profile.

Cause

This is standard behavior of MP80, not all the setting in MM02 can be pre-defined with forecast profile. The following picture contains all the value can be defined in MP80 and default value in MM02 when expanding Forecast tab to the material.

Besides, the Weighting group can't be maintained with following setting together:

  • Model selection
  • Selection procedure
  • Param.optimization
  • Optimization level
  • Alpha factor
  • Beta factor
  • Gamma factor

Resolution

Maintain the value manually in MM02.

Keywords

Forecast profile, MP80, MM02, MAD, Weighting group, Alpha factor, Beta factor, Gamma factor, Delta factor

Symptom

After SSL for system replication has been configured on the meta data channel according to the documentation, the secondary site Nameserver is unable to get meta data from the Nameserver of the primary system.
In the trace files there are error messages like the following:

[16502]{-1}[-1/-1] 2015-10-07 01:44:37.971729 e TrexNet
Responder.cpp(01876) : can't accept: (ssl: internal error)

OR

no.300013  (Crypto/SSL/CommonCrypto/Engine.cpp:516)

    SSL handshake failed: SSL error [536871970]: ???, General error: 0x20000422 | SAPCRYPTOLIB | SSL_accept

SSL API error

0xa060023c | SSL | ssl3_get_record

Other Terms

System Replication, SSL, Meta Data Channel

Reason and Prerequisites

The problem occurs on all revisions of HANA SP7, SP8 and SP9.
As a prerequisite on primary and secondary site the following configuration parameter has already been set in global.ini as part of the SSL configuration:

[communication]
ssl = on

Solution

To make ssl communication on the meta data channel between primary and secondary site nameservers work properly, the following configuration parameters have to be in global.ini on primary and secondary site:

[system_replication_communication]
ssl = on

[communication]
ssl = on

To enable SSL for data/log shipping connection also, the configuration parameter [system_replication_communication]/enable_ssl has to be set be set to on, source or target
(please refer to the documentation for details).

Please note, that starting from SP10 setting the parameter [system_replication_communication]/ssl is not required anymore.

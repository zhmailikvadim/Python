Symptom

When releasing Plant maintenance work order to generate Inspection Lot, system is giving ABAP Dump with below message:

Category               ABAP programming error
Runtime Errors         MESSAGE_TYPE_X
ABAP Program           SAPLCQTA
Application Component  QM
Message class....... Q0
Number.............. 130
Variable 1..........
Variable 2.......... QLOSE
Variable 3..........
Variable 4.......... " "

"MESSAGE_TYPE_X"
"SAPLCQTA" bzw. LCQTAU24
"CQTA_NUMBER_GET_NEXT"

The termination occurred in ABAP program or include "SAPLCQTA", in  "CQTA_NUMBER_GET_NEXT". The main program was "SAPLCOIH".

In the source code, the termination point is in line 38 of include "LCQTAU24".include "LCQTAU24".

   37     WHEN 1.
>>>>>       MESSAGE X130(Q0) WITH  NR_RANGE_NR OBJECT.
   39 *   Nummernkreisintervall &1 zu Nummernkreisobjekt &2 nicht gefunden.

Environment

  • SAP ERP Plant Maintenance (PM)
  • SAP R/3
  • SAP R/3 Enterprise
  • SAP ERP Central Component
  • SAP ERP
  • SAP Enhancement package for SAP ERP
  • SAP Enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA, on-premise

Reproducing the Issue

Release a PM order from IW32 or order Fio App, inspection lot is supposed to generate but dump arise. 

Cause

The number range group is not maintained or maintained incorrect for PM origin inspection lot. 

Resolution

Please check number range assignment via customizings: 

Quality Management
  -> Quality Inspection
       ->Inspection Lot Creation
          * Define Number Ranges for Inspection Lots
 Number range and number range groups are defined. 

Then check

Quality Management
  -> Quality Inspection
       ->Inspection Lot Creation
            * Maintain Inspection Lot Origins and Assign Inspection Types
if certain number range groups is maintained for plant maintenance origin inspection lot (in standard origin is 14 for PM). 

Keywords

IW32; QA09; QLOSE; number range group

Symptom

Invoices submitted through the email channel are successfully received by SAP Ariba Invoicing, but they do not appear in the invoice list for the logged‑in user.
The same invoices appear normally when uploaded directly through the UI.

There are no processing errors, and invoice creation itself works as expected when using the UI.

Reproducing the Issue

  1. Supplier sends an invoice via email to the configured SAP Ariba Invoicing email endpoint.
  2. SAP Ariba Invoicing sends a successful receipt confirmation for the email.
  3. The invoice does not appear in the "Manage Supplier Invoices Centrally" application for the end user.
  4. The same invoice is then resubmitted via a new email (not forwarded), but still does not appear.
  5. When the user uploads the same invoice file manually through the SAP Ariba Invoicing UI, the invoice is processed and appears normally.

Cause

The invoices do not have a company code determined during email ingestion.
In SAP Ariba Invoicing, visibility of invoices is controlled by Data Restrictions configured in the "Manage Data Restriction" application.

Resolution

To validate and resolve the issue:

  1. Assign the affected user to the following default role collections in SAP BTP:
    • CIM_user_all
    • CIM_Processor_All
    • CIM_Uploader_All
  2. Remove any custom role collections that enforce narrower data restrictions (e.g., restrictions by company code only).
  3. Have the user log out and log in again, then refresh the SAP Ariba Invoicing application.
  4. Verify whether the previously missing invoices now appear in the invoice list.
  5. Once visibility is confirmed, choose one of the following long‑term options:
    • Keep the user assigned to the ALL role collections;
    • Or, adjust the custom data restriction settings to allow visibility of invoices without company code;
    • Or, correct/improve company code determination during email ingestion to avoid future gaps.

If the visibility issue is resolved after assigning the ALL role collections, it confirms that data restriction configuration caused the problem.

Symptom

  • System is not allowing to change stop sequence in freight order.
  • Error messages encountered:
    • Could not synchronize location sequence of assigned requirement document
      Message no. /SCMTMS/PLN_VSR_OL 509
    • Planning operation stopped. No data changed
      Message no. /SCMTMS/PLN_VSR_OL 488 

Environment

  • SAP S/4HANA;
  • Transportation Management, Freight Order Management, Freight Order.

Reproducing the Issue

  1. Open Fiori app;
  2. Navigate to Freight Order and enter the Freight Order number;
  3. Fill sequence as Plant1 > Plant2 > Plant1;
  4. Error messages are raised.

Cause

  • Standard behavior;
  • The issue arises because Plant1 is set as both a Source and a Destination Location. As a result, adjusting the sequence for Plant1 isn't feasible due to a Fixed Unit (FU) originating from Plant2 and destined for Plant1, necessitating that Plant2 always precedes Plant1 in the sequence.

Resolution

  1. Review the configuration path: SPRO → SAP Customizing Implementation Guide → Logistics - General → Define Plant;
  2. To resolve this, you can manually insert Plant1 as an additional location. By doing so, and reassigning the FUs so that each location contains only either the Origin or the Destination FUs, you can then adjust the sequence for each location freely;
  3. Note that adjustments to configurations are not within the scope of SAP Support Team and require consultation with the SAP Consulting Team.
  4. Refer to discussions in the SAP Community for similar scenarios and insights.

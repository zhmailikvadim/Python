Symptom

  • In the Manage Reporting Scopes application, it is not possible to edit the core definitions of a reporting scope that has a status of Active or Released.
  • When entering edit mode for an active or released scope, functions to add or remove Scope Items (such as measures) or to change the Organizational Boundary are not available or are disabled.
  • The system does not appear to allow significant modifications to a reporting scope after it has been released.

Environment

SAP Sustainability Control Tower

Reproducing the Issue

  1. Navigate to the Manage Reporting Scopes application.
  2. Create and configure a new reporting scope.
  3. Use the Release button, allowing the scope's status to become Active or Released.
  4. Select the same reporting scope and enter edit mode.
  5. Observe that it is not possible to add new Scope Items or change the Organizational Boundary.

Cause

This behavior is by design and is a fundamental governance feature of SAP Sustainability Control Tower, as documented in the official SAP Help Portal.

Once a reporting scope is released, its core structure is locked to ensure data consistency and maintain a clear audit trail. According to the Reporting Scope Lifecycle, allowing major changes to an Active scope would compromise the integrity of historical data collected under that specific version.

Resolution

To make significant changes to a released or active reporting scope (such as adding scope items or changing the organizational structure), the standard and recommended procedure is to create a new version of the scope.

Prerequisite: The user must have the SUS_SCT_Manage_Reporting_Scope_Manager role assigned.

As described in the "Updating Scopes" section of the product documentation, there are two options. For this scenario, the correct option is Create New Version.

  1. Navigate to the Manage Reporting Scopes application.
  2. Select the Active or Released reporting scope that needs to be changed.
  3. Click the Create New Version button.
    • Note: In some versions or contexts, this may involve first using the Withdraw button on the current scope, and then using the Copy button to create the new editable version.
  4. A new draft version of the scope will be created, pre-populated with all settings from the previous version.
  5. This new draft is fully editable. Make the required changes, such as adding or removing Scope Items.
  6. Save and Release the new version. It will become the new Active scope for its validity period, while the old version is preserved for historical auditing.

Symptom

  • Device-related data (equipment, serial number, material) is not displayed in the service notification created in the Web Client Interaction Center.
  • The function module ISU_IL_SERVICE_NOTIF_GET returns no equipment for the affected notification.
  • In the backend, the notification shows no equipment in the relevant maintenance objects.

Environment

  • Customer Relationship Management (CRM) 
  • Interaction Center

Reproducing the Issue

  1. In the backend system, identify a service notification that exhibits missing device/equipment data.
  2. In the ABAP Function Builder (transaction SE37), test funtion module ISU_IL_SERVICE_NOTIF_GET with the affected notification number and execute.
  3. Observe that the function result does not return equipment/serial/material.
  4. In the Web Client Interaction Center, open or create the corresponding service notification and observe that device-related fields are blank.

Cause

  • The notification has no equipment maintained in the backend (for example, no entry in view VIQMEL and field EQUNR in table QMIH). ISU_IL_SERVICE_NOTIF_GET reads the equipment from these sources, so nothing can be returned or displayed if they are empty.

Resolution

  1. Verify equipment assignment for the affected notification in the backend by checking view VIQMEL and table QMIH (field EQUNR).
  2. If equipment is missing, assign the correct equipment to the notification using the appropriate business process or master data maintenance so that QMIH-EQUNR is populated.
  3. For comparison, identify a working notification where equipment is maintained and confirm that ISU_IL_SERVICE_NOTIF_GET returns equipment for it.
  4. Re-test in SE37 by running ISU_IL_SERVICE_NOTIF_GET with the corrected notification and verify equipment is returned.
  5. Open the service notification in the Web Client Interaction Center and confirm that equipment, serial number, and material now display.

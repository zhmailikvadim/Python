Symptom

Implement this SAP Note that integrates the BAdI definition FI_PL_KSEF_INVOICE into DRC Private Cloud Report PL_SAFT_KR_PD.

The implemented logic for searching for the KSeF number for a given customer invoice is as follows: The report will try to get the number stored in the ODN number. The report primarily searches for KSeF numbers stored using the official document numbering (ODN) framework.

Other Terms

Poland, PL, Poland SAFT Report, PL SAFT Report, JPK_KR_PD, KSeF BAdI, ODN, DRC, FI_PL_KSEF_INVOICE, PL_SAFT_KR_PD

Reason and Prerequisites

This note is relevant for Polish SAFT Private Cloud only.
This note aims to implement the KSeF BAdI integration specifically for the PL_SAFT_KR_PD report DRC Private Cloud.
SAP Note 3689699 must be implemented as a prerequisite.

Solution

Additional logic added in CL_SAFT_KR_PD_KSEF->GET_KSEF_NUMBERS

Enhancement spot: FI_PL_KSEF_INVOICE

There is one method: IF_EX_FI_PL_KSEF_INVOICE—>GET_KSEF_INVOICE
There will be no standard solution provided, therefore the customer is to implement the retrieval logic if needed.
This is a fallback logic in case ODN did not contain the KSeF ODN —> if ODN is empty —> BAdI.

If the number is not stored in ODN, and the customer implements BAdI FI_PL_KSEF_INVOICE, the BAdI method GET_KSEF_INVOICE will be called. This method will allow the generation of the KSeF number based on the customer logic.

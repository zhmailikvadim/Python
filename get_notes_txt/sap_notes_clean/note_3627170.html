Symptom

You receive this error in the inbound message when replicating sales orders into C4C:

Role category Other Party is not allowed for party role Operation Originating; please remove.

Environment

SAP Cloud for Customer

Reproducing the Issue

  1. Go to the Administrator work center.
  2. General Settings.
  3. Click on Web Service Message Monitoring under System Administration.
  4. Select the affected message and click on View/Error Log.
  5. Select tab Error Context.
  6. The error is displayed.

Cause

You are sending to C4C a role code under the wrong category.

For example, under Business Configuration > Party Role Definition, you have role 'Operation Originating' assigned to Party Category 'Sales Employee Party'.

But you are sending that role code under section <ItemCustomDefinedParty> in the payload. Hence the error is expected.

Resolution

You need to send the role code under section <ItemEmployeeResponsibleParty> in the payload, i.e. so that it matches the Party Category for the role code under Party Role Definition.

Keywords

Party Role Code, Party Role Definition, Role category, RoleCode, CustomerOrderReplicationIn

Symptom

CDI real-time replication stops replicating data for one or more subscriptions without any error message. The affected subscription thread produces no further log activity, while all other subscriptions in the same remote source session continue operating normally. The issue persists indefinitely until the DPAgent is restarted or the affected remote source is suspended and resumed.

In the DPAgent logs, the last entry for the affected subscription is a subscription lookup URI log line such as:

URI = https://<host>/api/v1/dataexport/administration/Subscriptions?$filter=ExternalID eq '<subscription_id>'

No "Response wait time" log follows this line, and no further activity is logged for that subscription.

Other Terms

Replication, CDI, Cloud Data Integration, SDI, hanging, subscription lookup, timeout, SAC

Environment

  • SAP HANA SDI 2.0 SP07
  • SAP HANA SDI 2.0 SP08

Steps to Reproduce

  • Start a CDI real-time replication task with multiple subscriptions
  • Under transient poor network conditions, a subscription lookup request is sent to the SAC CDI administration service but receives no complete response
  • The affected subscription thread silently blocks indefinitely with no timeout, no error, and no retry
  • All other subscriptions continue replicating normally

Reason and Prerequisites

The root cause is a gap in the client-side timeout handling for subscription lookup requests during transient poor network conditions. The CDI Adapter was unable to detect and recover from a stalled HTTP request, causing it to wait for the lookup response indefinitely. This is confirmed by the presence of a SocketTimeoutException for another subscription around the same time in the logs, indicating network degradation at that point.

Solution

A subscription lookup timeout has been introduced on the DPAgent CDI Adapter side. Each subscription lookup request is now executed with a total duration limit, enabling the CDI Adapter to
detect and recover from a stalled request and retry to overcome transient network issues. A new DPAgent property has been introduced to control the timeout:

- Property name: cdi.subscriptionLookupTimeout
- Unit: seconds
- Default value: 60

If the default is not suitable for a specific deployment, it can be adjusted by adding cdi.subscriptionLookupTimeout=<value> to the DPAgent ini configuration file.

This correction is delivered in the release(s) listed in the Support Packages and Patches section of this SAP Note.

Symptom

The program analyses goods receipt/invoice receipt (GR/IR) clearing accounts at a specified key date, and generates adjustment postings if necessary.
Adjustment postings are created per company code, GR/IR clearing account, reconciliation account, and business area.

Environment

  • Financial Accounting (FI)
  • SAP R/3
  • SAP R/3 Enterprise 
  • SAP ERP Central Component
  • SAP ERP
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA Finance
  • SAP S/4HANA

Cause

Explain the detail regrouping logic for F.19.

Resolution

  • The separation of the total posting amount into several document lines or documents occurs in terms of the original accounting assignments from the selected GR/IR. 
  • Based on the selected GR/IR items the program determines the original split assignments which consists of the split assignments fields AUFNR, KOSTL and PRCTR.
  • The combined posting of these assignments represent the aggregation level on which the GR/IR item amounts are collected per PO item reference. It means, in the processing loop the program is considering more assignment entities, not only the Profit Center. So it is depending on the detail data constellation of the selected items regarding how many different assignment combinations are existing and thus building separate aggregation groups.
  • At this point it is not yet possible to determine which of the original assignments will actually get relevant and populated in the final GR/IR regrouping posting line, because this will finally depend e.g. on the field status settings of the target accounts, the target account type etc. According to the group amount on PO item reference grouping level the debit/credit items of the group are assigned to target account determination GNB or BNG. The  resulting GR/IR regrouping posting lines which are created based on the aggregated amounts do not receive all of the original assignments, but mainly the Profit Center from the original assignment combination level. According to the debit or credit aggregation level amount the resulting posting line is specified with posting key 40 or 50.

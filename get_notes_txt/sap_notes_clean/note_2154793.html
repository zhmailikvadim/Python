Symptom

Simple  SELECT TOP number 'string' FROM TABLE ORDER BY COLUMN could cause fatal abort on df_Table::SCIdxExistForOrderedLeaf

Excerpt of stack trace on Linux:

 
    **  Command text:  

select top 2000  'a_string'     from lineitem    order by l_orderkey


df_Table::SCIdxExistForOrderedLeaf
vp_OrderedProjection::Prepare
vp_Cursor::PrepareProjection
dfo_VerticalLeafCursor::PrepareProjection
dfo_VerticalLeafCursor::PrepareChildrenAndProjection
dfo_VerticalLeafCursor::PrepareOpus
dfo::PrepareChildren
dfo_RowLimit::PrepareOpus

Environment

SAP IQ 16 any SP.

This issue could be encountered on other IQ previous versions.

Reproducing the Issue

Run on iqdemo

  • This select top including a column in the select list run fine:

select top 2000  l_orderkey  from lineitem order by l_orderkey;

  • This select top on s string aborts:

select top 2000  'a_string'  from lineitem order by l_orderkey;

 

 

Cause

This issue has been addressed in SAP CR 782552.

Resolution

Upgrade IQ to a release where SAP CR 782552 is fixed

  • IQ16 SP 8 PL 36 or higher
  • IQ 16 SP10 PL 03 or higher

This workaround can be used to disable the Ordered Leaf optimization:

set temporary option dml_options2 = 16777216

If necessary, set this option public, it wouldn't adversely affect the performance of most queries.

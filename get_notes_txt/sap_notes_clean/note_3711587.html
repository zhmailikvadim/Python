Symptom

Why does the Catalog search not return result for languages other than the ones in which the catalog is published in Ariba Buying Base edition?

Reproducing the Issue

Upload catalogs in Ariba Buying Base edition in English and try to search for it after changing the default site locale to another language (say French). 

Cause

When a user performs a search the Catalog Connectivity Service initiates a search call to search3.0 where catalogs are published and indexed using the catalog provider interface, the Catalog connectivity service depending on user's locale. It will pass the 2 digit language code for e.g., JP for Japanese to search3.0. Search 3.0 will return the catalog items which are published in JP language.

If there are not catalog item published in Japanese, it will not return results and catalog connectivity service will not show any items.


Resolution

Catalog connectivity service does not do language or localization of the catalog items. To resolve the issue, please publish catalog items in Ariba Buyer Catalog Manager in all the languages which users would be using to access the catalog items.

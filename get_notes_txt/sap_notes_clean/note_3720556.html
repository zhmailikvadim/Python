Symptom

  • When selecting a character from right to left in SAP GUI for HTML and releasing the mouse over the current cell, copying and pasting the text results in an extra line before the characters.
  • In the console tab, a similar entry to the following may be seen:

LS.js(xxx): processSemanticEvent : Semantic event "SapTableCS:CopyToClipboardRequest" occurred, {"Id":"M0:46:1","CurrentSelectionText":"\n11:00:00","Type":"COPY"} {"noEnqueue":true} (LS:Info)

"Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental." 

Environment

  • SAP GUI for HTML

Reproducing the Issue

  1. Access an app in SAP GUI for HTML/Fiori Launchpad.
  2. Navigate to a text field which locates in multiple table controls.
  3. Select the text from right to left, ensuring the mouse pointer is over the current cell when releasing or putting a cursor a bit more over to a cell on the left.
  4. Copy and paste the selected text to another location.
  5. An extra line is observed before the copied characters.

Cause

When the selection gets triggered over the edges of neighbored elements, sometimes the browser would rebuild the structure using \t or \n keys. When using a mouse, it is difficult not to select an HTML element nearby.

Resolution

  • Try to control the cursor to stay within the cell when selecting the content.
  • Avoid using the mouse, click inside the cell and press "Ctrl A" to select the content.

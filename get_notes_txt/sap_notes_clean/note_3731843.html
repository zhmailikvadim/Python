Symptom

Facing error ME 140 - "No print-relevant changes to document XXX exist-print preview & Output status not generated" in purchase order when trying to take a print preview of a Purchase Order for which all line items have been deleted.

 

Environment

Materials Management (MM)
SAP ERP Central Component
SAP ERP
SAP enhancement package for SAP ERP
SAP enhancement package for SAP ERP, version for SAP HANA
SAP S/4HANA

Reproducing the Issue

1. Open purchase order through ME23N.
2. Click print preview.
3. Try to take a print preview of a Purchase Order for which all line items have been deleted.

 

Cause

Standard system design.

Resolution

When all line items are deleted from a Purchase Order, the system re-determines output because the document structure changed. Even though all items are removed, the message routine runs and attempts to create output control entries. However, since the form’s print logic expects item data to generate a meaningful document, no print preview is available when there are no lines to render. This behavior is consistent with standard SAP functionality. So error message ME 140 - "No print-relevant changes to document XXX exist-print preview & Output status not generated" will trigger in purchase order when you try to take a print preview of a Purchase Order for which all line items have been deleted. Messaging is triggered when print-relevant changes occur, but print preview depends on the presence of printable content. So here no print preview when line items are deleted is behaving in standard way. If you require print preview when no line item exists, you have to do customization based on your requirement, this is not available in standard process. If you require deleted item data to be shared within external communication, you have to customize the  output program and output form based on your requirement. 

Keywords

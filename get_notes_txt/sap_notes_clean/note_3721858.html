Symptom

DYNPRO_FIELD_CONVERSION dump in program SAPMV45A occurs when a negative amount is entered for a condition type.

Environment

  • Sales And Distribution (SD)
  • Logistics Execution (LE)
  • SAP ERP Central Component
  • SAP ERP
  • SAP Enhancement package for SAP ERP
  • SAP Enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA
  • SAP S/4HANA Cloud Private Edition

Reproducing the Issue

  1. Open the document.
  2. Go to the Conditions tab.
  3. Enter a negative amount in the Amount column for a condition type.
  4. The dump is triggered.

Cause

The technical cause of the system’s behavior is as follows: The respective dynpro field 'target value' is defined as a positive field.

As a result, when a negative amount is entered directly in the Item Overview tab, the system automatically rejects it and requires a positive amount, which leads to the displayed error message.

When the negative amount is entered in the Conditions tab, the system accepts it. This is because the pricing functionality is a general service capable of processing negative values.

However, when returning to the previous screen, the system attempts to transfer the negative pricing result into the positive-only dynpro field. Since this scenario cannot be processed by the system, a dump is generated.

Resolution

Adapt the customizing of the condition type via V/06:

  • Set the "Plus/Minus" flag to "A Positive".
  • This setting will automatically convert negative input into positive values, preventing the dump.

Symptom

Customer'd like to send request from API pod/node to a service in the backgroundprocessing node.

The curl command used to access the background processing service results in Error message: "403 forbidden error".

Environment

SAP Commerce Cloud

Reproducing the Issue

Run the following groovy script from the script console:

```groovy

import java.net.*

import java.io.*

def url = new URL("https://backgroundprocessing.xxxxxxxx-public.model-t.cc.commerce.ondemand.com/sgwshealthcheck/healthcheck/status")

def connection = url.openConnection()

connection.setRequestMethod("GET")

connection.setConnectTimeout(5000)

connection.setReadTimeout(5000)

def responseCode = connection.responseCode

def responseText = connection.inputStream.getText("UTF-8")

println "Response Code: "

println "Response Body:\\n"

```

--Result: Observe the 403 forbidden error response.

Cause

The IP of the API proxy added does not work as the source IP for the request, which caused the 403 error. 

Resolution

Add node CIDR and pod CIDR to the allow list in the IP filter of targeting service (backgroundprocessing as above sample)

The node CIDR and pod CIDR could be found in Dyantrace as below:

Node CIDR: Kubernetes-> AutomationEngine ->Nodes -> [NodeId]: Internal IP Address
Pod CIDR: Kubernetes-> AutomationEngine ->NameSpace ->......-> Pods -> [PodId]: IP Address

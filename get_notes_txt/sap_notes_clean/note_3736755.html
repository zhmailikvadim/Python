Symptom

After a requisition is split into multiple purchase orders, the interface shows the same ERP PO number for more than one purchase order.

Cause

ERP is sending the same ERPPONumber in its response for different purchase orders, while the Ariba application has generated unique internal purchase order identifiers in system.

Resolution

Capture Order response from ERP to Ariba for split purchase orders and confirm the ERPPONumber value provided for each response. If identical ERPPONumber values are being returned for different purchase orders, check the ERP configuration so that a unique ERP PO number is assigned and returned for each purchase order created from a split requisition.

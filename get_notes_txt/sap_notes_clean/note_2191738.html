Symptom

You get the error message FGL_PRE_CHECK 030 (Field ** of index table ** is missing in table **. See SAP Note 2191738) in the S/4HANA Pre-Transition Check for FIN, executed before the upgrade to S/4HANA.

Other Terms

FGL_PRE_CHECK030, BSIS, BSAS, BSIK, BSAK, BSID, BSAD, FAGLBSIS, FAGLBSAS.

Solution

Before the Technical Upgrade

  1. Add each missing index table field to the table BKPF (if it is a header field) or to the table BSEG/BSEG_ADD (if it is a line item field).
  2. Develop and execute a program to fill these fields in BKPF/BSEG/BSEG_ADD from the index tables.

If the field length of the index table field is greater than 10 characters, this procedure is only possible in case BSEG is not a cluster table anymore. In contrast to Transparent Tables, Cluster Tables restrict the field length to 10 characters.
During technical upgrade BSEG is tranformed into a Tranparent Table. As the error message in the Pre-Transistion check is skippable, you can add the fields after the technical upgrade and follow the steps described in the following section.
Steps to be performed in order to skip a check are described in the blog accessible via the first link in note 2399707. If error message is not skippable, corrections contained in note 2716937 are possibly not implemented in your system.

After the Technical Upgrade

If you skip the error in your ERP system before the technical upgrade, the new views (see below) cannot be activated since the necessary fields are not available in the underlying tables. In this case you need to perform the following steps:

  1. Add the missing index table fields to the table BKPF (if it is a header field) or to the table BSEG/BSEG_ADD (if it is a line item field).
  2. Manually activate the views BSIS, BSAS, BSIK, etc. that could not be activated as part of the technical upgrade.
  3. Develop and execute a program to fill these fields in BKPF/BSEG/BSEG_ADD from the *_BCK tables.

Background

The index tables BSIS, BSAS, BSIK etc. get renamed from BSIS_BCK, BSAS_BCK, BSIK_BCK etc. during upgrade.

Tables BSIS, BSAS, BSIK etc. are then replaced with views BSIS, BSAS, BSIK etc., also known as compatibility views or equally-named views.

These views select data from tables BKPF/BSEG/BSEG_ADD. All fields added by the customer or partner to BSIS, BSAS, BSIK etc. must also exist in BKPF or BSEG/BSEG_ADD, otherwise it is not possible to create these views and the upgrade runs into errors.

The step "Develop and execute a program to fill these fields in BKPF/BSEG/BSEG_ADD" of the solution proposal is necessary to prevent a data loss.

Symptom

  • Users are being automatically logged out from BI Launchpad or the Central Management Console (CMC) after a brief period of inactivity.
  • The session expires and either shows the message ‘Your session has expired’ or redirects them back to the login page unexpectedly.
  • Due to the short timeout duration, users are unable to work continuously without frequent interruptions.

Environment

  • SAP BusinessObjects Business Intelligence Platform 4.x/20XX
  • Weblogic 12 or 14

Reproducing the Issue

  • Log in to the CMC or BI Launchpad.
  • Keep the session idle for around 20 minutes without any activity.
  • After 20 minutes of inactivity, the session automatically times out and logs the user out or redirects to the login page.

Cause

  • The issue occurs because the default session timeout value configured in the BI Launchpad or CMC web application is set to 20 minutes.
  • When users remain inactive beyond this configured duration, the application automatically ends the session.

Resolution

  • Navigate to the BOE WAR files directory, for example:
    E:\Program Files (x86)\SAP BusinessObjects\SAP BusinessObjects Enterprise XI 4.0\warfiles\webapps\BOE\WEB-INF
  • Create a backup of the web.xml file, then open the original file using a text editor such as Notepad. Search for the following entry:

    <session-config>
        <session-timeout>20</session-timeout>
    </session-config>

  • Update the session timeout value to the required duration.

    <session-config>
        <session-timeout>50</session-timeout>
    </session-config>

  • After making the changes, redeploy the WAR files using wdeploy to apply the updated configuration.

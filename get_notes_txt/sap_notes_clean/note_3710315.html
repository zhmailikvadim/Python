Symptom

  • The completion due SLA is marked as completed when the case status is set to "Customer Action".
  • The expectation is that the SLA should only be marked as completed when the case is set to "Completed" status.

Environment

SAP Service Cloud Version 2

Reproducing the Issue

  1. Navigate to cases.
  2. Create a case.
  3. Open the case and set its status to "Customer Action".
  4. Navigate to the Service Level (SL) sections.
  5. Observe that the completion due SLA is marked as completed, despite the case not being in "Completed" status.

Cause

For the Completion Due SLA, the completion date is set when the case is moved to "Requestor Action". This occurs when the case transitions to a status whose assignment status is mapped to "Requestor Action". The mapping between case status and assignment status can be verified in the Status dictionary settings UI.

Resolution

  1. Verify the mapping between case status and assignment status in the Status dictionary settings UI.
  2. Based on the use case, create a custom status that maps to the required lifecycle and assignment status.

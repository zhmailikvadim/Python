Symptom

Symptom

  • All schedules are running 1 hour later than normal time
  • Server time and instance time has a difference of 1 hour
  • Schedule gets delayed due to the time difference

Other Terms

export TZ, EST+5EDT, 1 hour later

Reason and Prerequisites

Environment:

  • SAP Business Intelligence BI 4.1 SP04
  • SAP Business Intelligence BI 4.1 SP05
  • Redhat Linux 6

Steps to Reproduce:

  1. Login to a Linux environment running BI 4.1 SP04 or SP05 using Putty session
  2. Set the time zone on the server using folloiwng command:
    export TZ='EST+5EDT,M3.2.0/2,M11.1.0/2'
  3. Run the following command to check if the TZ has been set properly:
    env | grep TZ
  4. Restart Tomcat and SIA
  5. Once both Tomcat and SIA has started, login to BI Launchpad or CMC and check that the preference is set to Local to web server
  6. Go to Report Samples > Feature Samples folder
  7. Right click on Drilldown report and click Schedule
  8. Go to Recurrence and select Once
  9. Run Date command on the Putty session and you will find that the instance time shown in Recurrence is 1 hour late.
  10. For e.g. If current time on server is 7am instance time shows 6am

Solution

  • Workaround:
  • Change the Current Time Zone under Locales and Time Zones in BILaunchpad or CMC Preferences to "(GMT-5:00,DST)Eastern Time US & Canada"
  • Logout and Login to BI Launchpad or CMC
  • Schedule the report again and it will show correct time

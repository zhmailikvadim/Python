Symptom

Is it possible to edit the Company code on an active term in a Published Contract Workspace (CW)?

Cause

The Company code (Any External integrated field with Compliance contact) was either added incorrectly or left blank during contract creation. 

Resolution

No, Company Code cannot be added or edited on an active term in a Published CW, even if it was left empty by mistake.

it's not possible because:

  • Company Code is a contract compliance / header-level field
  • Once the Contract Workspace is Published, all active terms are locked
  • SAP Ariba does not allow updates to compliance-critical fields on active terms
  • Export / import, mass update, or backend edits are not supported for this scenario

So unfortunately:

  • Not via export/import
  • Not via UI
  • Not via Tech Support backend actions

This is standard system behavior by design.

This is expected system behavior. Any change would require contract amendment or recreation, depending on business requirements.

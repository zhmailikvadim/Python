Symptom

Use "Edit Accrual Objects - Accrual Engine" upload template does not have fields, for example MATNR

When adding MATNR to CI_ACESOBJ_ASSGMT_ATTRIBUTES, activation fails with "Field 'MATNR' in table 'ACES_POSTINGS' is specified twice" (DT963).

Environment

  • SAP S/4HANA Finance
  • SAP S/4HANA
  • Financial Accounting (FI)

Reproducing the Issue

  1. Open the app "Edit Accrual Objects - Accrual Engine".
  2. Use the upload template and observe that Material field is not available.
  3. Enhance structure ACESOBJ_ASSGMT_ATTRIBUTES via customer include CI_ACESOBJ_ASSGMT_ATTRIBUTES to add field MATNR and activate.
  4. Observe the activation error that the field MATNR in table ACES_POSTINGS is specified twice (DT963).

Cause

Adding the standard field MATNR to the customer include CI_ACESOBJ_ASSGMT_ATTRIBUTES leads to a duplicate definition because MATNR already exists in structure ACES_POSTINGS. Customizing includes and appends must use customer namespace fields, therefore the system raises DT963 as expected.

Resolution

  1. Do not add MATNR directly to CI_ACESOBJ_ASSGMT_ATTRIBUTES. Instead, add a customer namespace field (for example, ZZMATNR of type MATNR) to the include.
  2. Implement BAdI BADI_ACE_DOCUMENT_ASGNMT_CUST (enhancement spot ES_ACE_DOCUMENT_CUST) to map the customer field (for example, ZZMATNR) to the corresponding document line-item field (for example, MATNR) during posting.

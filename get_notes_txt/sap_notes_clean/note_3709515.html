Symptom

After system upgrade, the PackingInstructionNumber field in Dangerous Goods Logistics Packing Instructions (table EHDGMD_LPI) may be empty for existing records, preventing proper cross-system integration scenarios where packing numbers with business semantics are required instead of system-specific UUIDs.

Other Terms

 DG - Dangerous Goods

Reason and Prerequisites

Prior to the enhancement, Dangerous Goods Logistics Packing Instructions used only system UUIDs (PackingInstructionSystemUUID) to reference packing instructions. To support remote integration scenarios where Product Compliance is used with multiple logistics servers, the system now requires packing instruction numbers that carry business semantics and can be used across systems.

Prerequisites:

  • Migration applies to all clients except client '000'
  • Source data must exist in table PACKKP with valid packing instruction numbers (POBJID)

Solution

The migration is executed automatically during system upgrade via the Software Delivery Manager Infrastructure (SDMI) framework. The migration performs the following steps:

  1. Data Selection:

    • Identifies all Dangerous Goods Logistics Packing Instructions where PackingInstructionNumber is initial (empty)
    • Uses CDS view P_DNGRSGDSLGSPAINSNMIGRATION to track the status for the need of the migration

  2. Data Derivation:

    • For each record with a valid PackingInstructionSystemUUID:
      • Looks up the corresponding packing instruction number (POBJID) from table PACKKP
      • Matches using PACKNR = PackingInstructionSystemUUID
    • Populates the PackingInstructionNumber field with the retrieved value

  3. Data Update:

    • Updates table EHDGMD_LPI using indicator-based UPDATE (only PackingInstructionNumber field is modified)
    • Processes records in packages of 100,000 entries for optimal performance
    • Marks migrated records with status '01' (done)

  4. Execution Modes:

    • Test mode: Performs migration with automatic rollback for validation
    • Productive mode: Commits migrated data to the database

Business Impact During Migration Runtime:

The migration has minimal business impact:

  1. My Packaged Dangerous Goods App: No impact, the application does not have usage of the migrated field.

  2. New Data Created During Migration: Any new packing instructions created during migration runtime will already contain the packing instruction number and do not require migration.

  3. System Performance: Migration processes data in packages to minimize performance impact. The migraiton is really lightweight updating only one field and selecting from one reference table.

  4. DG Checks: Checks on packing instructions will be downported with 2025 FPS2, so there were no checks being executed.

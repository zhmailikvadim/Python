Symptom

In Risk Manager, you use the "Status Change" function to create a period intersection for a policy section period. The system issues the error message "Duplicate entries for LUD (policy section <POS_ID> from <START_DT> to <END_DT>)".

Other Terms

/MSG/H_RISK1

/MSG/H_PROCESS_MSG01

926

Reason and Prerequisites

Prerequisite:

There are LUD entries in the affected policy section period.

Reason:

During processing, local tables are not initialized. As a result, the check has incorrect data and the system issues the described error message.

 

Solution

History:

March 4, 2026: New

09. March 2026: Release 809 added to SAP Note validity

Description of the troubleshooting/technical effect:

Initializing the local tables prevents the check from accessing incorrect data. When you split a policy section period, the system no longer issues an error message regarding duplicate LUD entries.

Symptom

You have deactivated some CVA and SLIN checks in transaction SCI ("Manage of" -> "Message Priorities") or in "ATC Configuration".

Despite deactivation, the CVA or SLIN checks are still executed; only the results are suppressed before display.

Other Terms

CVA, SAP Code Vulnerability Analyzer, Extended Program Check

Reason and Prerequisites

Performance improvement

Solution

If technically possible, deactivated CVA/SLIN checks are not executed.

Important Notes:

1. If you reactivate previously deactivated CVA/SLIN checks, you should also clean the SLIN result cache using program RSLIN_DELETE_CACHE. Therefore, it is not recommended to frequently deactivate and activate checks and clean up caches.

2. Some SLIN checkers can issue several different messages. Deactivating only one of them does not improve performance, because the corresponding CVA/SLIN checker still has to run.

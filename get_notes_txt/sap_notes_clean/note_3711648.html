Symptom

After a kernel update, developer trace entries may report ABAP programs as "out of date" and request regeneration.

Developer trace contains entries such as:

Statement with Statement Id: stmt { * } is out of date. Regenerate program: PROGRAMNAME.
B  How to regenerate see note  2934377.B

Environment

  • SAP_BASIS 740
  • SAP_BASIS 750 to 755
  • KRNL64UC 7.53
  • KRNL64UC 7.73
  • KERNEL 7.53
  • KERNEL 7.73
  • KERNEL 7.77 and subsequent

Cause

SAP Note 2883577 introduced that a trace entry is written for obsolete program loads. Under certain circumstances, a large number of duplicate entries may appear in the trace.

Resolution

To reduce the trace entries, proceed as follows:

  • Systems up to SAP_BASIS 740 SP04:
  1. Apply the kernel patch specified in SAP Note 2934377.
  2. Execute SGEN.
  3. Refresh PXA buffers on all application servers so the new load version is used:
    • On each application server, enter /$PXA in the OK-code field, or
    • Restart the application server(s).
  • Systems on SAP_BASIS 740 SP05 and later:
  1. Apply the kernel patch specified in SAP Note 2934377 and implement it.
  2. Execute program RSGENERATESQLPROGRAMS.
    • If the report does not list programs that could not be examined or that caused errors, a single execution is sufficient.
    • If the report lists a small number of problematic programs, generate those programs manually in SE38 by choosing Program → Generate.
  3. Refresh PXA buffers on all application servers:
    • On each application server, enter /$PXA in the OK-code field, or
    • Restart the application server(s).

Important: since the programs are regenerated when executing RSGENERATESQLPROGRAMS, transactions may terminate because the Program-Loads used have been updated. For this reason, we recommend that you execute the report at a time when the system load is low.

Symptom

This is an FAQ for SAP security note: 3593356 - [CVE-2026-0510] Obsolete Encryption Algorithm Used in NW AS Java UME User Mapping

Environment

SAP NetWeaver Application Server for Java

Resolution

Below you can find the questions and answers related to the SAP Security Note 3593356 - [CVE-2026-0510] Obsolete Encryption Algorithm Used in NW AS Java UME User Mapping.

Q1: If the UME User mapping is not configured, then do we have to apply the solution mentioned in the note?
Answer: No, If UME User Mapping is not configured in the system, then there is no need to apply the solution as the system will not be affected by the vulnerability.

Q2: Which SAP systems are affected?
Answer: All NetWeaver AS Java systems, including Enterprise Portal (EP), Process Orchestration (PO), MII etc. are affected.

Q3: Which versions of SAP NetWeaver AS Java are impacted? 
Answer: All versions and SP's are impacted till 7.50 SP31. This includes all older versions which are out of standard maintenance e.g. 7.0X, 7.1X, 7.2, 7.3X, 7.4 etc.

Q4: Is it possible to get the fix for lower SP's (e.g. SP30) and NetWeaver versions (NW AS Java 7.40)?
Answer: No, the fix is available only for 7.50 and above SP 32 as described in Support Packages & Patches section of the Security Note 3593356.
Developer has flow security guide where it is described where they should down port the fix. In this situation there is 24 month strategy i.e. they will downport in SPs that were released in last two years (last 24 month) 

Q5: Is there any workaround in lower SP levels, where the fix is not implemented?
Answer: No, there is a no workaround.

Q6: As per Security Note 3593356 there are three different JAVA components (J2EE ENGINE SERVERCORE 7.50 , ENGINEAPI 7.50 , UME ADMINISTRATION 7.50 ) that needs to be patched. Is there any need to update dependencies of mentioned components to close a security vulnerability?
Answer: Yes, All three mentioned components should be upgraded to a higher SP version with all dependencies.

Q7: Is Solman system affected by this vulnerability? For Eg. the SolMan is currently running on SP20, and JAVA part has a base SP of 29. So, is there need to update solman system?
Answer: Yes, the correction of note 3593356 starts with AS Java SP32 which covers SolMan SP21. So, the way out would be to upgrade SolMan to SP21.
Please refer to KBA:
3534229 - SAP Solution Manager 7.2 SP Stack 21: recommended corrections
Please do not update any individual software component to higher SP level without SAP confirmation. When you plan to upgrade SAP Solution Manager system, you need to upgrade both ABAP and Java systems to the same SPS level. See more about SOLMAN versioning in KBA: 2771041 - SAP Solution Manager 7.2: Support Package stack levels. This will help clarify the compatibility and upgrade requirements.

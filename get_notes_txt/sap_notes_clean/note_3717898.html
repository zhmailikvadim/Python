Symptom

In the process "Receipt of Delivery Note for Grid Usage Billing (SUP View)" (DE_B_GUB_SUP_DELV_NOTE), two issues occurred in steps 140, 130-0080, and 132-0080:
1. The system inserted duplicate delivery note entries with incorrect status 'CLARI' into table /US4G/EADOC for the same process.
2. The system triggered duplicate meter reading complaint processes when a DTD recheck was triggered.

This issue is relevant for the market role: SUP.

Other Terms

Grid Usage Billing, Delivery Note, Duplicate Entries, /US4G/EADOC, /US4G/CL_CHK_GUB_SUP_PROCESS, UPD_EADOC_FOR_DELIVERY_NOTE, CHK_DN_MR_COMP_PROC_TRIG, DTD recheck, E_0456, energy amount document

Reason and Prerequisites

Program error.
The errors occurred because:
1. The program incorrectly identified the current step, preventing the system from identifying the correct energy amount document status to update the existing entries.
2. The check logic for determining whether a meter reading complaint process had been triggered was incorrect, preventing the system from recognizing the triggered meter reading complaint process.

Solution

Implement the corrections of this SAP Note or import the relevant support package.

After implementing this SAP Note, the system finds the existing delivery note entries and updates their status correctly without inserting duplicate entries into table /US4G/EADOC. Additionally, the system no longer triggers duplicate meter reading complaint processes.
You can verify the fix by checking that no duplicate delivery note entries are inserted into table /US4G/EADOC for the same delivery note receiving process and that no duplicate meter reading complaint processes are triggered by the delivery note receiving process.

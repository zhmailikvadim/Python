Reason and Prerequisites

Reason:

Due to a programming error in SAP HANA, the System Replication target site did not handle log segments correctly if encryption was enabled and the log segment size was configured differently between the different System Replication sites.

Affected Releases:

  • SAP HANA 2.0 database:
    • Revisions of SPS00
    • Revisions of SPS01
    • Revisions <= 024.00 (SPS02)

Prerequisites:

  • SAP HANA System Replication is used in any operation mode (see SAP HANA Administration Guide).

  • Log encryption is enabled (see System View M_ENCRYPTION_OVERVIEW).

  • Different log segment sizes are configured on the System Replication sites with the configuration parameter:
    global.ini/<service>.ini
    [persistence]
    log_segment_size_mb

Solution

Apply one of the following SAP HANA database Revisions:

  • SAP HANA 2.0 database:
    • Revisions >= 024.01 (SPS02)
    • or higher

  • or higher

Workaround:

  • As a workaround, you can align the log segment size in the different System Replication sites.
    The parameter below should have the same value, both in global.ini and in all <service>.ini files, in all System Replication sites:

    [persistence]
    log_segment_size_mb

  • If you adjust a configuration value, please make sure to perform a reconfiguration of the database where the configuration value was changed using the "hdbnsutil -reconfig" OS command.

  • If you are currently affected by this issue, adjust the log segment as described above and re-register for SAP HANA System Replication.
    Follow the registration steps for the System Replication Target Site as described in the "Setting Up SAP HANA System Replication" in the SAP HANA Administration Guide. A regular registration without using the --force_full_replica option is sufficient.

Remarks:

  • When performing the registration as part of the workaround, usually only the redo logs have to be shipped over.
    In case that the log retention is disabled, or the maximum retention size was reached before the workaround was applied, SAP HANA will automatically perform a full data shipment after re-registering.

  • If you are using the full sync option, the primary site will be blocked in case you encounter the issue described here.

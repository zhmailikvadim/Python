To support negative amounts for returns, this SAP Note delivers an enhancement of the SAP Application Interface Framework (AIF) value mapping ORDER_REASON by adding an additional true/false attribute named “Return”. If a correction invoice has an order reason for which the “Return” flag is set to “true” in the value mapping, the credit lines in this correction invoice will get a negative quantity and a positive price. To update the ORDER_REASON value mapping, you must activate the BC Sets attached as ZIP file ( ZEDO_PL_KSEF_AIF_7.zip ). The ZIP file contains:

  • ZEDO_PL_KSEF_AIF_S4_7.bcs relevant for S4CORE component
  • ZEDO_PL_KSEF_AIF_7.bcs relevant for SAP_APPL component

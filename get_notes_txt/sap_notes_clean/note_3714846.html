Symptom

In S/4HANA, you implemented notes in your development system including DDLs related to the balance carryforward (BCF). The names of the DDLs start with FGL_BCF* or V_FGL_BCF*; corresponding view names in DDIC start with FGLV_BCF*. When the transport is released to your quality and/or production system, you get activation errors during import.

Example 1:

DDL Source FGL_BCF_BS could not be activated
(E - The column wwxyz_pa is unknown)

-> CO-PA characteristic WWXYZ_PA exists in your development system in ACDOCA but not in quality or production.

Example 2:

DDL Source FGL_BCF_PROT could not be activated
(E - The column zz_example_jei is unknown)

-> Custom field ZZ_EXAMPLE_JEI exists in your development system in ACDOCA but not in quality or production.

Other Terms

FGL_BCF V_FGL_BCF CVE-2026-0501 3687749 FAGLGVTR FGLV_BCF 3700593 FGL_MIG_REGENERATE_CDS_BCF FGL_MIG_REGENERATE_CDS_GL FGL_MIG_REGENERATE_CDS

Reason and Prerequisites

Mismatch of extensibility fields in the S/4HANA system landscape leads to syntax errors in generated CDS views.

Solution

The CDS views related to the BCF are partially generated to include extensibility fields in ACDOCA. If you have ongoing developments based on the extensibility options described in note 2453614, there can be a mismatch of extensibility fields between the development system and quality/production. In case the CDS views are transported in generated state, this usually leads to activation errors during the import of the transport request.

There are 2 options to resolve this:

Option 1:

Use this option if the transport was already imported. Execute report FGL_MIG_REGENERATE_CDS_BCF without option 'Reset to Delivery'. This step has to be performed in each system in which the transport gets imported. Note that you still get an activation error during import in each system and this step corrects the views after the import.

Option 2:

Use this option before releasing the transport in development. Execute report FGL_MIG_REGENERATE_CDS_BCF with option 'Reset to Delivery'. Note that in some scenarios (mainly older basis releases) DDIC dependencies prevent the complete reset of the views in the first execution of the report; If errors come up in the first execution of FGL_MIG_REGENERATE_CDS_BCF with option 'Reset to Delivery', repeat this step a few times.

This step transport the CDS views without extensibility fields and activation errors during import are avoided.

Note that regenerating FGL_MIG_REGENERATE_CDS_BCF without option 'Reset to Delivery' is not required in the target systems since the BCF will ensure the CDS views are correctly generated during runtime.

Remark:

If the views are inconsistent in quality and/or production, e.g. resulting from manual correction attempts, but are consistent in development, you can use report FINS_PUT_DDLS_INTO_REQUEST to add all dependent and Financials related CDS views to a new transport request. To avoid potential activation issues, regenerate all views with FGL_MIG_REGENERATE_CDS_GL with option 'Reset to Delivery' before releasing this transport request.

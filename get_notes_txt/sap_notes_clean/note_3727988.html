Symptom

 When you enter a reconnection result in transaction EC86, the system incorrectly triggers the process "Processing of Reconnection Order (DSO View)" (DE_B_GPKE_DISC_RCON_ORD_DSO). The system should only trigger the process "Reconnection Result (DSO View)" (DE_B_GPKE_DISC_RCRS_DSO_OUT).

Other Terms

GPKE, EC86, SWE2, RECONENTERED, DISACTION, /US4G/CL_MDR_BOR_TRIGG_PDOC->FILL_GENERIC_DATA, reconnection result, DE_B_GPKE_DISC_RCRS_DSO_OUT

Reason and Prerequisites

The error occurred because an incorrect entry was added in transaction SWE2 for the MDR event 'RECONENTERED'.

Solution

Implement the corrections of this SAP Note or import the relevant support package.

Before you activate the BC set /US4G/2_0_DSO_SP08_NOTE_3727988, enable the deletion function. To do this, go to transaction SCPR20 and choose Utilities → User Settings. On the Maint.Transaction tab, select the Switch On checkbox under Deletion Function. Then, activate the BC set.

After implementing this SAP Note, the system no longer triggers the process DE_B_GPKE_DISC_RCON_ORD_DSO when you enter a reconnection result in transaction EC86. You can verify the fix by entering a reconnection result in transaction EC86 and checking that only the process DE_B_GPKE_DISC_RCRS_DSO_OUT is triggered, and the reconnection order process is not triggered.

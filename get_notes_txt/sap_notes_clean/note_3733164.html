Symptom

The header is not showing the correct username of the employee in the header when they have more than one employment. When navigating to other employment, the username displayed next to the name, stays the same.

"Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental."

Environment

  • SAP SuccessFactors HCM Suite
  • SAP SuccessFactors Employee Central

Reproducing the Issue

  1. Look for the ID displayed next to the user's name.
  2. Switch to another employment.
  3. Username displayed is the same.

Cause

In the legacy experience, the Username was displayed next to the person's display name on the profile header. In the Latest People Profile, this is change to the person's login name (Notable Differences in Existing Features).

If the company is using Enable log in with old user name in case of a rehire with a new employment in Company System and Logo Settings, the user will have the name login name for the employments, which means the display in the profile will be of this same login name for all the employments. 

If this feature is not being used and the user has a unique login name, each employment will display a different login name next to the name.

Resolution

As per system design, the Standard element "Username" is not available for selection in Configure Latest People profile > Configure Header Fields.

An enhancement request can be raised for this and it will be evaluated for future releases (learn more at 2090228 - How to submit enhancement ideas for SAP SuccessFactors Products).

An alternative is to ensure the users have a unique login name per employment. The login name can be changed in Manage Login Accounts (2859043 - Manage Login Accounts tool).

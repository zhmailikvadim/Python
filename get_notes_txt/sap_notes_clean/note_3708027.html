Symptom

  • The Switch BC Sets are visible in SCPR3 but not in SCPR20, preventing activation.
  • Activation transaction option in SCPR3 is disabled.
  • The Switch BC Set has the property "Non-Cascading" .

Environment

  • Business Configuration Sets

Cause

If you activate industry solutions with business function sets, the system automatically activates the Switch BC Sets that contain the customizing settings for the assigned business functions in the background. This process activates the customizing settings required for the industry solution in your system. 

Standard design. Switch BC Sets may contain:

  • Cross-client customizing
  • Client-specific customizing and Switch BC Set contents must be cascaded
  • Client-specific customizing and Switch BC Set contents must not be cascaded; That is, the contents are only imported into client 000

Resolution

  • As mentioned in the Activating Switch BC Sets document, the system automatically activates all relevant switch BC Sets when switching on business functions by default. So, normally there is no need for activating any individual switch BC Sets. 
  • If the Switch BC Sets needs to be activated manually, the Switch BC Set with the property "Non-Cascading" can be only activated in the reference client (e.g., client 000). Use transaction SCPRSACT to activate the Switch BC Set manually in the relevant target client.

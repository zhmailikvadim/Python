Symptom

When opening a query in a List View via Web Client, the following error occurs:

Reproducing the issue

Prerequisites:

Working with Web Client.

Scenario:

Go to Analytics → User Defined Query → save the query below, for example:

SELECT T0.[ItemCode], T0.[ItemName], T0.[OnHand],

T0.[ReorderQty] AS [מלאי_מומלץ],
T0.[MinLevel],
T0.[ReorderQty] - T0.[OnHand] [כמות_מומלצת_לרכישה]
FROM OITM T0
WHERE T0.[ReorderQty] > T0.[OnHand]

Expected behavior:

Field names can contain only Unicode letters, digits, and underscores.

Actual behavior:

When running a query as above that includes a Hebrew field name (this affects multiple languages, including Hebrew, Arabic, and Chinese, etc.) or alias and clicking on "Open List View," the Web Client returns an error:

Solution

SAP intends to provide a feature package or patch to solve the symptom that is described in this SAP Note. The section References below lists the related feature package or patch once they become available. Be aware that References can only be confirmed at feature package or patch release date. SAP delivers feature packages or patches only for selected releases at its own discretion, based on the business impact and the complexity of implementation.

 

Workaround:

Keep the SQL UDQ in English characters, in the Web Client UDQ "Columns" section the important thing is that the "Name" field will be only in English. Then there is no problem changing the text Description field to another language. If this is done, the UDQ will validate and will open the List View correctly, displaying the foreign language Description as the column text.

Symptom

In the Change Audit Document for Time Sheet, the time offset value is displayed as a numeric string without a colon (for example, 100, 130, 200), instead of standard time zone format like UTC+1:00 or UTC+1:30.



"Image/data in this KBA is from SAP internal system, sample data, or demo system. Any resemblance to real data is purely coincidental."

Environment

SAP SuccessFactors Time Tracking

Cause

The audit log currently shows the raw offset value without formatting.
Although values such as 100, 130, 530 actually represent UTC+1:00, 1:30, 5:30, the system does not insert the colon during display.

Resolution

At present, this behavior is expected based on the current system design.
There is no configuration setting to change how the offset value is rendered in the audit log.

Planned Fix
Product Management has confirmed that a change is planned:
The system will update the audit log to display time offsets using the standard UTC format (e.g., UTC+1, UTC+1:30, UTC+5:30) instead of raw values such as 100, 130, or 530.
This enhancement is scheduled for 1H 2026. For release timelines, please review SAP SuccessFactors Product Release & Road Map Information.

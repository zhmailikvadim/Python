Symptom

Block selection and Copy to Clipboard are implemented by default for each C-Table control that is embedded in a WebDynpro application.

The C-Table Block Selection allows the user to easily copy any selected area of a C-Table to the clipboard. The copied data can then be pasted into any other application such as a text editor, spreadsheet, or email application.

Blocks can be selected in several ways which are described in note 3164952.

Selected cells are clearly highligted.

When the end user presses the standard operating system 'Copy to Clipboard' key combination (e.g. [Ctrl]+[C] in Microsoft Windows), the values of the currently selected table cells are retrieved from the back-end system and concatenated into a long string that is copied to the clipboard. ​The copied values are concatenated into a long string separated by the HorizontalTab character. A new line is created with the Linefeed character.

The Copy to Clipboard implementation is text-based. This means that the content of each selected cell is copied as text, regardless of whether the cell contains a character or a numeric value. Even for a graphical representation, meaningful text is copied.

The following general rules apply:

  • Copy the text shown in a table.
  • Copy only the text if there is an image and text.
  • Copy image tooltip or description or filename if only an image is displayed in the cell.
  • Always copy plain text without formatting.
  • Copy a meaningful value of a graphical Cell Editor.
  • If there are several cell editors in a cell, copy the value for each cell editor and concatenate their text values separated by commas.


The following gives an overview on what is copied per Cell Editor:

Cell Editor

Value copied

Button

The button text is copied.
If no button text is displayed, the button image tooltip/filename is copied.

Caption

The text is copied.
If no text is displayed, the image tooltip/filename is copied.

Checkbox

The checked state is copied and represented by a Unicode symbol font.
In addition the text is copied.

CheckboxTriState

The checked state is copied and represented by a Unicode symbol font.
In addition the text is copied.

DropdownByIndex

The selected item text is copied.

DropdownByKey

The selected item text is copied.

DropdownListBox

The selected item text is copied.

FileDownload

The text is copied.
If no text is displayed, the image tooltip/filename is copied.
The actual file of the FileDownload is not copied.

FileUpload

Not supported

FormattedTextView

 

Just plain text is copied. All HTML markups as well as any control characters are removed.

GraphicalValueComparison

If available the marker value, the max value, and the text is copied in the format:
marker value / max value  text

Image

The image tooltip is copied.
If not available the image filename is copied instead.

InputField

The value is copied as text.

LinkToAction

The link text is copied.
If no link text is displayed, the image tooltip/filename is copied.

LinkToUrl

The link text is copied (the referenced URL is not copied).
If no link text is displayed, the image tooltip/filename is copied.

ProgressIndicator

The display value is copied.
If no display value is shown, then the percent value is copied as text.

RadioButton

The checked state is copied and represented by a Unicode symbol font.
In addition the text is copied.

RatingIndicator

The rating value and the max value are copied in the following format:
rating value/max value *

TextEdit

The plain text is copied. All control characters are removed.

TextView

The text is copied.

ThresholdSlider

The value showing the position of the slide is copied.

ToggleButton

The button text is copied.
If no text is available the image tooltip/filename is copied. The toggle state is not included.

Dependent on the table cell type, additional information might be displayed in the selected table cell:

Cell Type

Value copied

Symbol Cell

Symbol is also copied as Unicode symbol font.

Hierarchical Cell

Hierarchy state icon and the hierarchy level indentation is also copied.

MultiEditor Cell

The value of each cell editor is copied and separated by comma.

Special features of the C-Table are also supported:

Cell Type

Value copied

Fixed Rows

The contents of fixed rows are also copied.

Merged Cells

Cells in a row that are next to each other can be merged.
Only the content of the leftmost cell is copied. The contents of the merged cells is copied as blank.

Grouped Cells

Cells in a column that are next to each other can be grouped.
The content of a grouped cell is copied as blank.

Other Terms

WebDynpro ABAP, CTable control, block selection, copy to clipboard, <CTRL><C>

Reason and Prerequisites

A WebDynpro application is using a Client Table (C-Table) control. The C-Table control does not have implemented an application specific Copy to Clipboard functionality, but is suing the WebDynpro default implementation.

Symptom

Cloud Portal was migrated from BTP Neo to Kyma runtime in March 2026. As part of this migration, the API token capability is replaced with a new Technical User Management feature for improved security and manageability.

Cause

Within the migration, the following changes were implemented:

Previous Method - API Token Management:

  • Required creating API tokens
  • Required S or P users from external accounts.sap.com

New Method - Technical Users:

  • Integrated into Cloud Portal User Management
  • Uses client credentials authentication (Client ID, Client Secret, Resource, Token Endpoint)
  • Streamlined process within the Cloud Portal interface

Impact:

CI/CD pipelines and automation scripts using Cloud Portal API needs to be updated to use Technical User credentials instead of API tokens.

Resolution

To utilize the Technical User management feature, please follow these steps:

Step 1: Create a Technical User

  1. Log into the Cloud Portal.
  2. From the side navigation of the Cloud Portal, select User Management.
  3. Select Technical Users.
  4. Select Create.
  5. Set Permissions for the technical user:
    • Role: Choose between CUSTOMER_SYS_ADMIN or CUSTOMER_DEVELOPER depending on your needs.
    • Environment Access: Select the environments this technical user can access. Environment selection depends on the environments available in your subscription.
  6. Select Create to proceed.

Step 2: Save Your Credentials

IMPORTANT: After creation, a popup will display your technical user credentials. This is a one-time display only. If you navigate away without saving these credentials, they cannot be recovered and will require you to create a new technical user.

Copy and securely store the credentials immediately:

  • Token Endpoint URL
  • Client ID (name)
  • Client Secret
  • Resource

Step 3: Use Your Technical User

Once you’ve created your technical user and saved the credentials, you can authenticate using both CLI and direct API methods:

  • Method 1: Using CLI Commands
    1. The command line interface is packaged as a ZIP file. Search for CX COMM CLOUD MAN CLI at SAP Software Downloads.
    2. Configure the CLI with your technical user credentials using the appropriate configuration commands:

      sapccm config set client-id {CLIENT_ID_VALUE}

      sapccm config set client-secret {CLIENT_SECRET_VALUE}

      sapccm config set token-url {TOKEN_URL_VALUE}

      The CLI will then automatically:
      1.  Retrieve the bearer token using your credentials
      2. Handle token management in the background
      3. Allow your existing CLI commands to work normally

  • Method 2: Using Direct API Access

    For direct API usage, you’ll need to handle token retrieval manually:

    1.  First, obtain a bearer token:

      curl <<token_endpoint>> -X POST \

      -H 'Content-Type: application/x-www-form-urlencoded' \

      -H 'Accept: application/json' \

      -d "client_id=<<client_id>>" \

      -d "client_secret=<<client_secret>>" \

      --data-urlencode 'grant_type=client_credentials' \

      --data-urlencode 'resource=<<resource>>'

      The response will look like this:

      {

      "access_token":"...",

      "token_type":"Bearer",

      "expires_in":3600

      }

    2. You can then use the bearer token for API calls. The new header is: x-approuter-authorization.
      For an example cURL command:

      curl https://portalapi.commerce.ondemand.com/v2/subscriptions/{subscriptionCode}/environments \

      -H "x-approuter-authorization: Bearer <<access_token>>"

Step 4: Verify Functionality

Confirm that your technical user can:

  • Successfully authenticate using the client credentials
  • Perform an API call to get a list of environments
  • Perform a CLI command to get a list of environments

Symptom

During a tenant recovery or system refresh (for example using
RECOVER DATA … USING BACKINT), the recovery fails with the error:

recovery could not be completed:
Error during recovery: Found service neither active nor starting for recovery:
DBID <n>@<host>:<port>

The recovery does not proceed, although the backup itself is accessible and no BACKINT or media errors are reported.

Environment

SAP HANA DB Version 2.0

Cause

In the analyzed scenario, the tenant-specific ScriptServer was not available on the target tenant.

During recovery, SAP HANA expects all required tenant services to be available. If a required service is missing, the recovery process fails with the above error.

Resolution

Enable the missing service for the affected tenant on SYSTEMDB, restart the tenant, and re-run the recovery.

Example for enabling the ScriptServer:

ALTER SYSTEM STOP DATABASE <tenant_name>;
ALTER DATABASE <tenant_name> ADD 'scriptserver';
ALTER SYSTEM START DATABASE <tenant_name>;

Verify that the service is running:

SELECT DATABASE_NAME, SERVICE_NAME, PORT, SQL_PORT
  FROM SYS_DATABASES.M_SERVICES
 WHERE DATABASE_NAME = '<tenant_name>'
 ORDER BY SERVICE_NAME, PORT;

After the required service is running, repeat the tenant recovery using the same backup.

The procedure for enabling the ScriptServer for a tenant is documented in the SAP Help Portal.

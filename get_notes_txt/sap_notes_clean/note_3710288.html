Symptom

When running report /SDF/RC_ALM_COLLECT_DATA and starting Analysis, the scheduled job fails with message

Job started
Step 001 started (Program /SDF/RC_ALM_COLLECT_DATA, Variant &0000000000001, Username <>user)
>> RC call API to start collecting Solution Manager usage data.
Starting collection for scenario Process Management
# Starting collection for capability Branch
# Finished collection for capability Branch
# Starting collection for capability Library Management
RC call ALM data collection API failed
Job was aborted after system exception ERROR_MESSAGE

Environment

  • SAP Solution Manager 7.2

Reproducing the Issue

  1. Execute /SDF/RC_ALM_COLLECT_DATA

Cause

Table AGS_TBOM_BASEIDX is very huge

Resolution

Check if the latest version of 3236443 - SAP Readiness Check for SAP Cloud ALM has already been implemented 72 or higher.

If the issue persist, please identified the table by looking at SM50 while the collection was running. 

If the issue occured reading table AGS_TBOM_BASEIDX, please follow the Note 3673118

See Also

  • SAP NOTE  3125933 - Internal Server error when opening the Library Generation Cockpit.
  • SAP NOTE  3673118 - TBOM Content is Huge - BPCA TBOM Tables Growing Rapidly

Keywords

/SDF/RC_ALM_COLLECT_DATA, AGS_TBOM_BASEIDX, TBOM, RC call ALM data collection API failed, capability Library Management, Database returned SQL code 4, error text: cannot allocate enough memory,

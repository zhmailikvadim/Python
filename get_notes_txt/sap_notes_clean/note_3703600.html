Symptom

  • Unable to link Google Drive to the SAP Business Intelligence Platform environment.
  • After successfully logging into the Google Drive page, a 404 page is encountered when attempting to retrieve the token.
  • Encounter a HTTP 500 error when opening the biprws URL.

Environment

  • SAP Business Intelligence Platform environment 4.x/2025

Reproducing the Issue

  1. Configure Google Drive as a data source 3211882 - How to use Google Drive as data source when creating a new Web Intelligence document?
  2. Generate a token from BI Launchpad settings and log into the app.
  3. Encounter a 404 page when attempting to retrieve the token.
  4. Encounter a http 500 error when opening the biprws URL.

Cause

The issue was caused by a deployment error in the web application on the Tomcat server.

Resolution

  1. Redeploy the web application on the Tomcat server.
    • For Windows-based deployments, refer to the SAP Knowledge Article 3477871 - How to redeploy BI Platform biprws on Tomcat? - Windows.
    • For Linux-based deployments, refer to the SAP Knowledge Article 3165685 - How to redeploy BI Platform related web applications on Tomcat? - Linux/Unix.
    • Ensure to modify the command parameter `-DAPP=BOE` to `-DAPP=` as per the specific application being redeployed.
  2. Recreate the Google Drive authorization server configuration in the CMC.

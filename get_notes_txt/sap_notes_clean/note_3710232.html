Symptom

 Records in Tables /ACCGO/T_CAS_CAI and /ACCGO/T_APPDATA are blank with SDM_VERSION field

Other Terms

/ACCGO/T_CAS_CAI, /ACCGO/T_APPDATA, SDM_VERSION field, Blank SDM Version

Reason and Prerequisites

In few instances the tables /ACCGO/T_CAS_CAI and /ACCGO/T_APPDATA are having blank SDM version

Solution

The Utility Program is designed to accommodate changes required to update the SDM_VERSION field in database tables where the field is currently blank. The migration classes /ACCGO/CL_SDM_ACM_T_CAS_CAI and /ACCGO/CL_SDM_ACM_T_APPDATA are used to update their respective tables.

The program updates the SDM_VERSION value to ‘01’ based on the user’s input selection. Users also have the option to execute the program in Test Mode.

This is a one-time activity performed in the system to update the SDM_VERSION fields using silent migration, where records are updated as a background task.

After the update is completed, application logs are generated showing the total number of records updated in both tables, along with the execution time.

Once the note is implemented, Execute the program: /ACCGO/ACM_SDM_VERS_UPD_UTIL with the default parameters for 'Name of migration class'.

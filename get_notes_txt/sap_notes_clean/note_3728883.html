Symptom

Links to Fiori apps in the user manual lead to a specific system (Q-system).Requirement to create universal links to Fiori apps that dynamically adapt to the currently opened system and client.

Environment

SAP S4/HANA

Reproducing the Issue

  1. Create a link to a Fiori app using semantic (intent-based) navigation links (e.g., #SemanticObject-action?param=value).
  2. Attempt to open the link in a browser (e.g., Chrome).
  3. Observe that the link does not open or function as expected when clicked or pasted into the browser's address bar.

Cause

Resolution

  1. Use semantic (intent-based) navigation links instead of system-specific URLs.
  • Semantic links should only include the shell hash of the app (e.g., #SemanticObject-action?param=value).
  • Avoid hardcoding hostnames, system IDs, or clients in the links.

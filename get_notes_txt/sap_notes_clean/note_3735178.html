Symptom

During the Snowflake configuration, it is not possible to use DSConnectionManager because it does not have a proxy option.

Environment

  • SAP Data Services 4.x
  • Linux
  • Snowflake

Cause

Proxy settings are missing from the DSConnectionManager configuration steps for Snowflake.

Resolution

Try to temporarily connect without a proxy (validate this option with network team) and add the proxy later as per KBA 3652769 - "How to configure proxy information in Snowflake datastore for SAP Data Services?".

If temporarily not using a proxy is not an option, perform the following:

  1. Manually create a DSN in $ODBCINI with the proxy option. Example:
    • [snowflake_example]
      Driver=<driver folder>/<driver name>.so
      UID=<user>
      PWD=<password>
      SERVER=<server address>
      PORT=443
      DATABASE=<database>
      SCHEMA=<schema>
      WAREHOUSE=<wherehouse>
      ROLE=<role>
      Proxy=http://<proxy>:<port>
  2. Manually add the same DSN into $LINK_DIR/bin/ds_odbc.ini pointing to unixODBC:
    • [snowflake_example]
      Driver=/usr/lib64/libodbc.so
      #Driver=<unixODBC so file>

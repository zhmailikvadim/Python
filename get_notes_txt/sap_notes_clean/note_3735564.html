Symptom

After enabling notification profile for the task, system is not sending the completion email to the project owners even though the task has been marked completed.

Reproducing the Issue

  1. Configure the notification profile to trigger notification post task completion.
  2. Complete the task.
  3. Check the mail box. Completion email did not received by the owner.

Resolution

When we customize the notification profile in any task, the email that triggers for the completion action is picked from the project email template Task - Task complete notification sent when a task is complete. The Notification Profile determines the recipients. This is a scheduled batch notification.
As the task name says, it is a scheduled batch notification. That means system will not trigger the completion notification immediately after you mark it complete, instead, this notification will only be triggered based on the scheduled job defined in the backend. This is an expected system behavior.

Symptom

  • Communication System Determination does not behave as expected after activation and configuration.
  • Outbound messages are either always sent regardless of conditions or not initiated at all

Environment

  • SAP Sales Cloud Version 2
  • SAP Service Cloud Version 2

Reproducing the Issue

  1. Open Integration Settings and navigate to Communication System Determination.
  2. Activate rules for Configured interface
  3. Select example customers and perform a minor update on the header data (to trigger replication)
  4. Verify whether the update initiates the message outbound.

Cause

  • Incorrect identifier was used in determination rules for the sales organization; salesOrganizationId (salesArrangements/salesOrganizationId) requires the technical UUID, not the display ID.

Resolution

  1. Use technical UUIDs for salesOrganizationId in Communication System Determination:
    1. Retrieve the salesOrganizationId UUID from the relevant API entity (for example, account/salesArrangements) and not the display ID.
    2. Update the rule conditions to compare against these UUID values.
  2. Validate rule attributes and operators:
    1. For routing by object type, consider attributes such as isContactPerson, isAccount, or roles/role as needed.
    2. Test “equal” versus “contains” based on the actual value format of the chosen attribute.
    3. Validate the Details are correct on the underlying API call
  3. Ensure the target endpoint exists and is reachable to avoid 404:
    1. In the Communication Configuration for the custom route, confirm the configured HTTP path matches an active and deployed endpoint in the integration platform. Activate and deploy the iFlow endpoint and verify authentication/authorization if applicable.
    2. Re-test outbound initiation and confirm successful message delivery in monitoring.
    3. Ensure the communication system is active
  4. If multiple communication systems are used for different targets, enable both configurations and ensure that each rule group points to the correct communication system and endpoint.
  5. A Default condition can be defined, where no condition is specified, and all or specific communication systems are selected.
  6. Re-run the reproduction steps and confirm messages initiate only for records matching the conditions.

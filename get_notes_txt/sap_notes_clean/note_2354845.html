Symptom

After installation of SAP BusinessObjects Business Intelligence platform 4.2, when user login to CMC and navigate to Events, under 'User Notification Events' to create a new user notification events option is not visible as well as icon to create a new user notification events is not visible.

Environment

  • SAP BusinessObjects Business Intelligence platform 4.2
  • Windows server

Reproducing the Issue

  1. Install BI platform 4.2 SP02.
  2. Login to CMC as Administrator.
  3. Open Events Select "User Notification Events".
  4. Right Click "User Notification Events" -->New.
  5. The option to create new event is not visible.

Cause

The DFO 'BusinessObjects_ClientAction_NotificationEvent_Create_dfo.xml' is missing/corrupted in the CMS database.

Resolution

  1. Navigate to <<Installation directory of Business Objects>>\SAP BusinessObjects Enterprise XI 4.0\dfo and backup the file 'BusinessObjects_ClientAction_NotificationEvent_Create_dfo.xml'.
  2. Download the DFO cleanup utility which is attached to the KBA or using the KBA:-2145849 - HOWTO: Cleanup default objects (DFOs) from the CMS repository 
  3. Open the command prompt and navigate to the folder where you have kept the DFO cleanup utility and run the below command.
  4. Dfocleanup.bat -f BusinessObjects_ClientAction_NotificationEvent_Create_dfo.xml -c localhost
  5. Provide administrator credentials and allow the tool to cleanup the dfo file.
  6. Stop the Server Intelligent Agent (SIA).
  7. Place the 'BusinessObjects_ClientAction_NotificationEvent_Create_dfo.xml' in the <Install dir of BI>\enterprise xi 4.0\packages folder.
  8. Restart SIA.
  9. Clear the cache of the 'Web Application server'.
  10. Restart the 'Web Application Server'.

See Also

2145849 - HOWTO: Cleanup default objects (DFOs) from the CMS repository

Very High Internal Issue Due to a coding issue in an error handling routine of the product's background processing logic, messages could get lost due to unavailability or communication issues with the message broker backing service. The affected error handling routine was corrected. In consequence, message loss is prevented as messages will be retried from the persistence layer in the event of the described error scenario. General regression testing is suggested. No specific testing is possible as the issue was located in an error handling routine, which is not involved during normal product usage (that is, outside of an exceptional erroneous infrastructure situation).

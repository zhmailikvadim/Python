Symptom

  • When using Maintenance Planner to plan the SAP S/4HANA 2023 update to SP04, Maintenance Planner appears to have made an error in the calculation of the target patch levels. The generated stack does not include all patch levels required for SP04, and some components are proposed below the required levels—specifically UIS4HOP1 and UIAPFI70.

Environment

SAP S/4HANA 2023

Reproducing the Issue

  1. Use the maintenance planner to plan an S/4 HANA 2023 update to SP4.
  2. Observe that not all relevant patch levels for SP4 are selected, specifically for components UIS4HOP1 and UIAPFI70.

Cause

Resolution

  1. Confirm that SAP Fiori for SAP S/4HANA 2023 – UI for SAP S/4HANA SP03 is included in the target.
  2. Ensure that the minimum supported version for SAP Fiori for S/4HANA 2023 is SPS03 when planning an update to S/4 HANA 2023 SP4.
  3. Review the updated Release Information Note for SAP Fiori for SAP S/4HANA 2023, which states:
  • Support Package Stack 03 of SAP FIORI FOR SAP S/4HANA 2023 on the frontend requires Feature Package Stack 03 or higher of SAP S/4HANA 2023 in the backend (and vice versa).
  • Different SP stack levels of SAP FIORI FOR SAP S/4HANA 2023 and SAP S/4HANA 2023 can be combined if the minimum of Support Package Stack 03 is reached on both systems.
  • For example: Selecting Support Package Stack 04 for SAP S/4HANA 2023 in Maintenance Planner would as a default only select the minimum of Support Package Stack 03 for SAP FIORI FOR SAP S/4HANA 2023, depending on the overall calculation.

See Also

  • Refer to: SAP Note 3336823 - SAP FIORI FOR SAP S/4HANA 2023: Release Information Note

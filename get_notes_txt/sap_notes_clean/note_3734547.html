Symptom

 After performing manual clearing, the field BSEG-AGZEI (clearing item number) reflects the same value across the clearing item and all associated cleared items in table BSEG, which may seem unexpected to users.

Environment

  • Financial Accounting (FI)
  • SAP ERP
  • SAP ERP Central Component
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA
  • SAP S/4HANA Finance
  • SAP FIORI for SAP S/4HANA
  • SAP S/4HANA Cloud Private Edition

Reproducing the Issue

  1. Perform a manual clearing transaction in the SAP system.
  2. Examine the BSEG table entries for BSEG-AGZEI values related to the clearing document.
  3. Observe that BSEG-AGZEI shows the same value across both the clearing item and all cleared items.

Cause

 The BSEG-AGZEI field serves as a clearing reference and is standardized to hold the same value for both the clearing item and all associated cleared items. This behavior is intentionally designed and expected. The field does not correspond to the BSEG-BUZEI (line item number) of the clearing item. 

For this example showed in picture: since all the cleared items were cleared with the same item which is 2000000693/0001/2025 002,  it is correct for the clearing item and cleared items to have identical AGZEI values. 

Resolution

This is the correct and standard behavior of the SAP system.

  • It is important to note that there is no direct relationship between BSEG-AGZEI and BSEG-BUZEI of the clearing item.
  • BSEG-AGZEI provides a clearing reference number which is consistently updated across clearing and cleared items.
  • To determine the relation between a clearing item and its cleared items, you can refer to table BSE_CLR. Key fields here include the clearing document number, company code, and fiscal year. The field BSE_CLR-AGBUZ represents the item number of the clearing item.

See Also

Please note that The AGZEI field exists in BSEG and BSE_CLR, it is not saved in ACDOCA for avoiding duplicating information among tables.

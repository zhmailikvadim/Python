Symptom

Encounter errors in red when following the steps below to reset SYSTEM user password as per Reset the SYSTEM User's Password | SAP Help Portal:

  1. Shut down the instance by executing the following command:
    /usr/sap/<SID>/HDB<instance>/exe/sapcontrol -nr <instance> -function StopSystem HDB

  2. In a new session, start the name server by executing the following commands:
    • /usr/sap/<SID>/HDB<instance>/hdbenv.sh
    • /usr/sap/<SID>/HDB<instance>/exe/hdbnameserver

  3. In a new session, start the compile server by executing the following commands:
    • /usr/sap/<SID>/HDB<instance>/hdbenv.sh
    • /usr/sap/<SID>/HDB<instance>/exe/hdbcompileserver

  4. In a new session, start the index server by executing the following commands:
    • /usr/sap/<SID>/HDB<instance>/hdbenv.sh
    • /usr/sap/<SID>/HDB<instance>/exe/hdbindexserver -resetUserSystem

    Note: In a scale-out system, you only need to execute the commands on the master index server.
  5. Encounter error:

    starting interacttive mode for resetting user SYSTEM...
    unclean shutdown of service instance with pid XXXXX.
    Aborted

  6. Try to start the system:

    /usr/sap/<SID>/HDB<instance>/exe/sapcontrol -nr <instance> -function StartSystem HDB

  7. Encounter error:

    An instance of this service is already running.
    You should consider checking if the process with the id XXXX is running
    and of type hdbnameserver. If you are sure that there is no instance alive
    remove the file /usr/sap/<SID>/HDBXX/<hostname>/lock/hdbnameserver@32001.pid
    and restart.

Environment

SAP HANA Platform Edition

Reproducing the Issue

Run /usr/sap/<SID>/HDB20/exe/hdbindexserver -resetUserSystem as <sid>adm.

Cause

A recent shutdown of the HANA database has not been completed successfully.

Resolution

  • Restart the HANA DB, the lock file should be deleted & re-created again.

  • In case it does not work, check and kill the PID in the trace seen in symptoms:

An instance of this service is already running.
You should consider checking if the process with the id XXXX is running and of type hdbnameserver. 
If you are sure that there is no instance alive remove the file /usr/sap/<SID>/HDBXX/<hostname>/lock/hdbnameserver@32001.pid and restart

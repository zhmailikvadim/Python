Symptom

When creating a order Without Charge order in VA01 and entering an item with blocked material, the Order Without Charge is still being created even though the sales status is set to "B" block. However, the expectation is that the system should block the Order Without Charge with error message "V1028."

Environment

  • Sales And Distribution (SD)
  • SAP ERP Central Component
  • SAP ERP
  • SAP Enhancement package for SAP ERP
  • SAP Enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA
  • SAP S/4HANA Cloud Private Edition

Reproducing the Issue

  1. Assign a block status to a material on the material master (MM02 transaction).
  2. Create an Order Without Charge in VA01, input an item with the blocked material.
  3. The Order Without Charge order is still being created even though the sales status is set to "B" block.

Cause

The creation of a sales order is not controlled by the material status alone, but rather by the customizing of the sales status.

Set the sales status for the relevant SD Document to 'B' (Error message) as needed. For example:

  • Set "Sales Order" (TVMS-SPVBC) to "B" for Order (SD Document Category 'C')
  • Set "Free Delivery" (TVMS-SPVBI) to "B" for Order Without Charge (SD Document Category 'C')
  • Set "Credit Memo Req" (TVMS-SPVBK) to "B" for Credit Memo Request (SD Document Category 'K')

If the sales status for the relevant SD Document is not correctly set to 'B', for example, if only "Sales Order" is set to 'B' but "Free Delivery" is not, Order Without Charge orders can still be created, even though it is believed that the sales status has been set to 'B'.

Resolution

  1. Confirm the SD Document Category assigned to the order type using transaction VOV8.
  2. Update the sales status configuration as needed.

The Sales statuses can be defined in the IMG at:

SPRO → Logistics - General → Material Master → Settings for Key Fields → Data Relevant to Sales and Distribution → Define Sales Statuses.

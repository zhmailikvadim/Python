Symptom

In the SAP Fiori app "Monitor EQS Time Series Versions," when you export MaLos contained in the time series to a spreadsheet along with the accumulated quantity, some consumption quantities are displayed incorrectly with decimal notation.

Other Terms

EQS, Monitor EQS Time Series Versions, MaLo, consumption, spreadsheet, export, floating-point type values, decimal values, accumulated quantity, time series

Reason and Prerequisites

The error occurred because in the spreadsheet, when floating-point values exceed 15 digits in length, the value is considered as General format (character) instead of a numeric or floating-point value. This caused the system to fail to convert the consumption value downloaded from the app.

Solution

Implement the corrections in this SAP Note or import the relevant support package.

After implementing this SAP Note, the system downloads and displays all consumption values correctly in the exported spreadsheet.

You need to download the consumptions from market locations. You can verify the fix by checking that all consumption values are displayed in a single format according to the notation for decimal values.

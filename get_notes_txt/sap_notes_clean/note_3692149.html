Symptom

Below is a clear, step-by-step article explaining how to manually calculate arrears for a wage type in SAP Payroll, using the logic of the Wage Type Reporter and most of the payroll reports like payslip.

(H99CWTR0, PC00_M99_CWTR)

It is written for functional consultants, payroll analysts, and anyone who wants to understand how the system derives differences period-by-period.

Environment

  • Human Capital Management (HCM)
  • SAP R/3
  • SAP ERP Core Component
  • SAP ERP
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA

Reproducing the Issue

Due to change in master data, configuration, time evaluation, or another payroll input employee's past payroll is opened for correction

and arrears in SAP Payroll appear (whenever a wage type is retroactively recalculated).

Cause

Arrears in SAP Payroll appear whenever a wage type is retroactively recalculated. This happens when an employee's past payroll is opened for correction — because of a change in master data, configuration, time evaluation, or another payroll input. SAP compares the old payroll result stored in the cluster with the new recalculated value. The difference becomes an arrear.

Understanding these differences manually helps validate payroll results, troubleshoot anomalies, and explain the calculations to auditors. This article explains how to replicate SAP's arrears logic by hand, using a simple example first and then a real-world case

Resolution

1. Key Concepts

1.1 Payroll Period vs. Retro Payroll Run

Every payroll run processes the current period, but may also process one or more retro periods if prior-period master data changed.

This leads to several “versions” of the same payroll period.

1.2 Sequence Number (SEQNO)

Every time a period is recalculated, SAP assigns a new sequence number.

A higher SEQNO means a more recent recalculation.

Example SEQNO table (simplified):

SEQNOFor-periodIn-periodAmount
101.202501.20251000
201.202502.20251100
301.20250320251200

Here period 01.2025 has been recalculated 3 times.

1.3 Arrears Logic

The arrear for a wage type is:

(Amount in new recalculation) – (Amount from previous stored period result)

SAP performs this comparison for each period separately.

2. Manual Calculation Example (Simplified)

Let’s take the data:

SEQNOFor-periodIn-periodAmount
101.202501.20251000
201.202502.20251100
302.202502.20251100
401.202503.20251200
502.202503.20251200
603.202503.20251200

2.1 Arrears for 02.2025

Compare SEQNO 3 (new) with SEQNO 2 (previous value for that period):

               • New amount: 1100

               • Old amount: 1000

Arrear = 1100 – 1000 = 100 EUR

2.2 Arrears for 03.2025

Two periods (01.2025 and 02.2025) changed in this run:

a. For 01.2025:

               • New amount (SEQNO 4): 1200

               • Previous amount (SEQNO 2): 1100

Arrear = 1200 – 1100 = 100 EUR

b. For 02.2025:

               • New amount (SEQNO 5): 1200

               • Previous amount (SEQNO 3): 1100

Arrear = 1200 – 1100 = 100 EUR

Total arrears posted in 03.2025 = 200 EUR

3. Manual Calculation Procedure — General Formula

To calculate arrears manually:

Step 1 — Identify all periods recalculated

Look at cluster table RT or WPBP with different SEQNOs.

Step 2 — Group entries by For-period

For each period, gather all SEQNO entries.

Step 3 — Sort SEQNO ascending

Lowest = original stored payroll

Highest = latest recalculation

Step 4 — Compare the last two SEQNO amounts

Arrear for a period =

Current amount – Previous amount

Step 5 — Sum arrears for all periods recalculated in the same payroll run

Those arrears are posted in the In-period (usually the current payroll period).

 

4. Summary

To manually reproduce SAP Payroll arrears calculations:

               1. Identify the wage type values across all SEQNO recalculations.

               2. For each period, sort SEQNOs chronologically.

               3. Compare newest and previous SEQNO amounts.

               4. The difference is the arrear posted in the current payroll period.

               5. Repeat for every period recalculated in the run.

With these steps, you can analyze any wage type’s arrears exactly as the Wage Type Reporter displays them.

Symptom

You use Transaction Manager of Treasury and Risk Management. You have created a security position with a security ID with product type 01A, position that is handled with single position management (lot accounting). A corporate action with category Manually generated where the same security ID is provided in both Security Referenced and Security to Transfer to. Also an implementation for BAdI TPM_CORPORATE_ACTION was created in the system. The corporate action is being posted by executing transaction FWKB. Also during the execution the Inflow or Outflow Quantity is changed in the screen Manual Quantity Adjustment (as the BAdI TPM_CORPORATE_ACTION was implemented). In such a case the system raises a runtime error OBJECTS_OBJREF_NOT_ASSIGNED with the following details:

"CX_SY_REF_IS_INITIAL"
"CL_TRANSACTION_TRL============CM03E"
"DETERMINE_SOURCE_POSITION"

Other Terms

FWK0, FWKB, stock, 01A, Manually generated, lot accounting, TPM_CORPORATE_ACTION, OBJECTS_OBJREF_NOT_ASSIGNED, DETERMINE_SOURCE_POSITION

Reason and Prerequisites

Program error

Solution

Implement the correction instructions or import the corresponding Support Package.

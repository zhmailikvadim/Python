Symptom

Manage governance process stuck at activation due to the dump 
Category               ABAP programming error
Runtime Errors         GETWA_NOT_ASSIGNED
ABAP: Program          CL_MDC_DATA===================CP
Application Component  CA-MDG-CMP-AF 

Environment

SAP Master Data Governance-Consolidation and Mass Processing

Reproducing the Issue

  1. Launch Fiori Launchpad
  2. Access app Manage Governance processes-Business Partners
  3. Check the process
  4. Process is stuck at SBGRFCMON with dump

Resolution

  • Implement the SAP Notes
    3483052 - CMP Resolve Concurrent Changes: Fix multi parent entities
    3434274 - Handling of active records in concurrent changes
    3360834 - Resolve Concurrent Changes: Conflicting field name
    3467005 - CMP: Long text data causes error MDC_MODEL 065 during activation step
    3495055 - MDG CMP: Fix Resolve Concurrent Changes for created Longtexts in parallel
    3678521 - CMP: "Enter IBAN" error during activation step of the MDG, Mass Processing application
  • Ensure there is no custom code in the system, if there is custom code check method IF_MDC_DATA~APPEND_ACTIVE_RECORDS (ZCL_MDC_DATA_BP) and ensure there is only 1 call to the method.

 

 

Keywords

MDC_QUEUE_, SBGRFCMON, Dump, GETWA_NOT_ASSIGNED, CL_MDC_DATA, IF_MDC_DATA~RESOLVE_CONCURRENT_CHANGES, IF_MDC_DATA~APPEND_ACTIVE_RECORDS, MDC, CMP

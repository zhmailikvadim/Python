Symptom

User is getting system error while trying to end the collaboration
Caused by: ariba.util.core.WrapperRuntimeException: ariba.util.core.FatalAssertionException: Failed EndCollaboration auto-approval.
Msg: ApprovalHookFailed, Id: null, Token: -1,ErrCode: (missing argument)[########] has been deleted.
 at ariba.collaboration.core.Collaboration.endCollaboration(Collaboration.java:758)

Cause

 [########]  is likely a user name. The user mentioned has been deactivated in the realm.

Resolution

You have the following two options to end the collaboration:

  1. Reactivate the user mentioned.
  2. Reassign the requisition to a new user and then complete the collaboration
    1. Click Manage > Approvable Rule Assignments
    2. Select the desired approvable type (Ex: Requisitions or Invoice, etc)
    3. Select the Preparer / Requester / Approver that you would want to assign the approvable to 
      • Note:  You cannot reassign the approver if the approval node is a group (Customer Administrator, Purchasing Manager, etc.).  This will only work for individual approvers.
    4. Click Documents > Reassign (you can select one or multiple documents)
    5. Enter any comments in the Comments box and click Save button

See Also

For more details about how to reassign a requisition to a new user.

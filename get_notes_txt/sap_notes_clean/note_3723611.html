Symptom

You use Extended Notifications for SAP Business Workflow to inform users via email about workitems in their inbox.
You encounter the following system dump:

Category ABAP programming error
Runtime Errors CREATE_OBJECT_CLASS_NOT_FOUND
Except. CX_SY_CREATE_OBJECT_ERROR
ABAP: Program CL_SWN_DELIVERY_MAN===========CP
Application Component BC-BMT-WFM

Other Terms

CL_CRM_CHM_NOTIFICATION_USER, SWNCONFIG, SWN_SELSEN

Reason and Prerequisites

There are entries configured in transaction SWNCONFIG for a user handler class that do not exist anymore in the system.
Make sure that you use only user handler classes in the "Category" section that still exist and replace entries where the class does not exist anymore.

The dump can also be caused by old obsolete entries in the "Handler Assignment" section that are SAP delivered. CRM Class CL_CRM_CHM_NOTIFICATION_USER is an example, that causes these dumps especially after a system upgrade in none CRM systems.
Delete these entries in your development system and transport the changes through your system landscape.

Solution

Implement the support package or the attached correction instructions.
See the "Reason and Prerequisite" section and remove obsolete entries to user handler classes that do not exist anymore.

After implementing the correction the standard user handler class 'CL_SWN_USER_STD' is used in case a user handler class does not exist.

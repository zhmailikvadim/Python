Symptom

When attempting to open the "Manage Tax Items" activity, the following error message is displayed: "Sorry, the app couldn’t be opened."

Environment

SAP S/4HANA Cloud Public Edition

Reproducing the Issue

  1. Open the "Run Statutory Reports" application.
  2. Filter by report name and reporting year.
  3. Try to access the Manage Tax Items activity, observe the error message: "Sorry, the app couldn't be opened. The navigation target "#ReportingTask-manageTaxItems?CompanyCode=3000&NewApproachSwitch=%20&_ACTKEY=FA163E491ED91FE0B9D49F4EBBB18153&ReportingCountry=ES&TaxReportingDate=20260101%20-%2020260131&sap-app-origin-hint=&sap-xapp-state=ASNCUQ5SHYN372R6O6MG8J1JENHLZHAYKHKZB3KA" of the app could not be resolved. This can be caused, for example, by an incorrect content configuration, an incorrect or missing role assignment, or because your current device is not supported."

Cause

  • Missing Configuration.
  • Missing Business role assignment.

Resolution

  • To access the activity, the business user must be assigned the business role SAP_BR_GL_ACCOUNTANT_XX, as described in the help documentation: Supported Compliance Tasks by Country/Region
  • To activate the "Document and Reporting Compliance" license features, perform the following steps:
    1. Open 105518 SSCUI ID
    2. Choose New Entries.
    3. Select all Countries/Regions checkbox if you want to include all countries/regions or select the country/region key of the countries/regions for which you want to activate the features.
    4. Save your entries

Note: When assigning authorization roles to users, ensure that both Read and Write access are granted.

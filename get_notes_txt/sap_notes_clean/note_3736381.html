Symptom

Error message FWTR056 "Error assigning update type to flow type &1, sign &2, contract type &3" is issued when creating a financial transaction via Transaction code FTR_CREATE.

Environment

  • Treasury and Risk Management (TRM)
  • SAP ERP
  • SAP ERP Central Component
  • SAP enhancement package for SAP ERP,
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP Financial Supply Chain Management
  • SAP S/4HANA
  • SAP S/4HANA Finance
  • SAP S/4HANA Cloud Private Edition

Cause

Customizing is missing in the step "Assign Flow Types to Update Types".

Resolution

Please verify the configuration at the following SPRO path:
Treasury and Risk Management → Transaction Manager → Money Market (or Foreign Exchange / Securities / Listed Derivatives / OTC Derivatives) → Transaction Management → Update Types → Assign Flow Types to Update Types.

Alternatively, you can access this setting directly via transaction code SM30, for table/view ATFTA.

Within this view, please check the assignments for the relevant Contract Type, Flow Type, and Direction. This configuration determines how specific flow types (such as principal payment, interest, fee, etc.) are mapped to update types based on transaction characteristics.

Function module WTR_DIS_FLOW_TYPE_CONVERT             Determine Update Type for Transaction Flow Type

* read the assigned distribution flow type
  SELECT SINGLE dis_flowtype FROM atfta
                             INTO e_dis_flow_type
                             WHERE rantyp  = i_rantyp
                             AND   sfhazba = i_flow_type
                             AND   ssign   = i_sign.

  IF sy-subrc <> 0 OR e_dis_flow_type IS INITIAL.
    MESSAGE e056(fwtr) WITH i_flow_type i_sign i_rantyp
                       RAISING not_found.

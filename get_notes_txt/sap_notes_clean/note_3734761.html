Symptom

  • AO (Analysis for Office) scheduling fails, when installed on Multiple Nodes
  • AO scheduling fails with : Could not start "<Install_path>:\SAP BusinessObjects\Analysis Precalculation\BiPrecalculation.exe". Details: CreateProcess error=2, The system cannot find the file specified.

Reason and Prerequisites

Environment

  • SAP BusinessObjects Business Intelligence Platform 4.x / 2025
  • All Supported Operating System
  • Analysis for Office (AFO) Addon

Steps to Reproduce

  1. Create a Cluster Environment with 2 or more Nodes
  2. Install Analysis for Office Addon on Both the Nodes.
  3. On Node 1 maintain the Node 1 Precalculation Server Path local to the Node1.
  4. On Node 2 maintain the Node 2 Precalculation Server Path local to the Node2.
  5. While Scheduling Analysis report explicitly select respective AJS with Node 1 and then with Node 2.
  6. Scheduling fails with above error

Cause

The design of the code means that the AO scheduling code reads all of the AJS services hosting the AO Scheduling Service (not just the one the schedule is running on) and sets the path to the BiPrecalculation.exe to the last one read. In an environment where the installation paths are not the same on all nodes, this means some AJS will fail.

Solution

The issue has been forwarded to development for further investigation

Workaround

  1. Start the AJS hosting AO service only on servers which have the same installation path
  2. Ensure the ones installed on machines with a different path are stopped
  3. Re-run the schedule

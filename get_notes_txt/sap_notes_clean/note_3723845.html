Symptom

You are using the Review Order Plan Items app in SAP Customer Activity Repository applications bundle 5.0 FPS03, 5.0 FPS04, 5.0 SPS05, 5.0 SPS06, 5.0 SPS07, 5.0 SPS08, 6.0 SPS01, or 6.0 FPS02. When you select a variant, the data from the variant is not applied to the filter bar. Additionally, the following error message is displayed in the browser console: error during 'afterVariantLoad' event handling - Cannot read properties of undefined (reading 'customFilters').

Other Terms

SAP Customer Activity Repository applications bundle, CARAB, Review Order Plan Items, ROPI

Reason and Prerequisites

Reason

Program error in /ext/controller/ListReportExt.controller.js, function onBeforeVariantSave: the _CUSTOM property of the filter data model is overwritten before the variant is saved.

Prerequisites

You have set up the Review Order Plan Items app as described in the Common Installation Guide (for new installations) or the Common Upgrade Guide (for upgrade scenarios) for SAP Customer Activity Repository applications bundle 5.0 FPS03, 5.0 FPS04, 5.0 SPS05, 5.0 SPS06, 5.0 SPS07, 5.0 SPS08, 6.0 SPS01 or 6.0 FPS02. You can find both guides on SAP Help Portal at https://help.sap.com/docs/CARAB. Select your version at the top.

Solution

Implement the attached correction instructions in your front-end system.

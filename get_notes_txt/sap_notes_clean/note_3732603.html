Symptom

MAIF Inbound queue mechanism has a step in the last win branch which before executing the transaction it will check the queue for a previously failed transaction with the same key. If there are any transactions in the queue with the same key, the inbound queue will mark them as cancelled and process the current last win transaction. Once cancellation is issued generic reason "transaction cancelled" is presented and saved into the status table. This note addresses this issue by providing a better error for the user when transaction is cancelled in the last win scenario.

Symptom

You're using electronic document processing for e-mail submission of invoices for Peppol countries/regions and want to implement the following correction:

  • Cross-border email invoices are now sent using the sender company's country process instead of failing when the customer's country has no email invoicing configured.
  • The check whether a customer is activated for email invoicing now correctly considers the company code and sales organization.

Other Terms

eDocument, eDoc, eInvoicing, e-Invoicing, electronic invoice, electronic invoicing, e-document, e-Document, e-Docs, e-Doc, e-docs, e-doc, Document Compliance, DC, Peppol Exchange, Document and Reporting Compliance, electronic document processing, incoming automation solution

Reason and Prerequisites

This SAP Note is relevant only for Peppol countries/regions with e-mail submission.

Prerequisite: 

You have implemented the electronic document processing framework in your system. Refer to the implementation instructions provided on SAP the Help Portal for SAP ERP or SAP S/4HANA.

You have implemented the following SAP Note:

  • 2783795 - Electronic Document Processing for Peppol Countries/Regions: Implementation Overview
  • 3724581 - Electronic Invoice for Peppol Countries/Regions - DDIC Changes in EDOEUBUPA and EDOEUBUPAV
  • 3712230 - Electronic Invoice for Peppol Countries/Regions - EDOEUBUPAV with Company Code and Sales Organization
  • 3708482 - Electronic Invoice for Peppol Countries/Regions - Corrections

Solution

Import the Support Package specified for your release or implement the attached correction instruction.

SAP recommends that you implement the Support Package specified for your release. However, if you need to install a solution earlier, use the SAP Note Assistant to implement the correction instruction. You can find more information about the SAP Note Assistant in SAP Support Portal, under SAP Note Assistant.

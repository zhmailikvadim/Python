Symptom

You are using SOM in S/4 with API functionality to run different change processes through SOAP API (https://help.sap.com/docs/SAP_S4HANA_ON-PREMISE/e94543741b6142e198a86f908c698978/8c105de6389c4c63b5809299fcab24d2.html?version=LATEST&locale=en-US). When maintaining Grace Period data via API (not UI), the GRACE_END date was not automatically calculated by the date rules. After setting the GRACE_PERIOD duration, GRACE_END remained empty or was not updated.

Other Terms

date rule is not getting triggered, grace_period changed, crm_appt_maintain_single_ow

Reason and Prerequisites

Running this change process via UI is maintaining the GRACE_PERIOD as date via CRM_APPT_MAITNAIN_MULTI_OW which triggers internally CRM_DATES_CALC_MULTIPLE_OW to run all date calculations again. But just updating GRACE_PERIOD via CRM_APPT_MAINTAIN_SINGLE_OW is just starting the rule maintained on GRACE_PERIOD. But the rule is assigned to GRACE_END.

Solution

Implement the corrections attached to the note according your S4CEXT EHP release of upgrade to the version support package version mentioned in the present note.

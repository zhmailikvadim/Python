Symptom

You are using SAP Asset Manager. Any status update posted from mobile client incorrectly triggers the execution of reverse mobile status synchronization logic, which should be executed only if status update is made from SAP GUI or third party programs.

The issue only occurs if mobile status or confirmation oMDO class handler (e.g. /MERP/CL_PM_MOBILE_STATUS_OD, /MERP/CL_PM_CONFIRMATION_OD ) has '/MERP/CL_CORE_OMDO_HNDLER_BASE' defined as super class.

oMDO class handler uses '/MFND/CL_CORE_OMDO_HNDLER_BASE' as super class will not see this issue.

Other Terms

Mobile Status, Confirmation, super class,  /SMERP/CL_PM_MBLSTATUS_TOOLS, IS_CALLED_FROM_MOBILE,  S4MOBADDON-2891

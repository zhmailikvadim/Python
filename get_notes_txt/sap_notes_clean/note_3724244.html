Symptom

Purchase orders for a specific location are sent to suppliers in one language instead of another

Environment

SAP Ariba Buying and Invoicing, Purchase Orders

Reproducing the Issue

  1. Navigate to Manage > Core Administration > Supplier Manager > Suppliers.
  2. Search for the relevant supplier.
  3. Click Actions > Edit next to the supplier.
  4. Open the Partitioned Suppliers tab.
  5. Click Edit for the applicable partitioned supplier.
  6. Go to the Supplier Locations tab.
  7. Click Edit next to the supplier location.
  8. Update the Locale field with the desired language/region.
  9. Click OK twice, then click Save.

Cause

The language for the PO sent to the supplier is determined by the Locale configured for the Supplier Location used on the PO. If the Locale is not defined for the Supplier Location, then the language used for the PO print view is determined by the locale associated with the Country of the Supplier Location.

Resolution

Supplier can update the Locale associated with the Supplier Location. 

For example, a supplier has the country set to Switzerland in their Supplier Location, with the locale configured as French. Because of this locale setting, all POs sent to this supplier are currently generated in French. However, if you would like the POs to be sent in German instead, you need to therefore update the locale in this Supplier Location to German.

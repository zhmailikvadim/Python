Symptom

Deadlock detected on table IBINVALST_SYM.

Environment

Variant Configuration

Cause

Deadlocks may occur on table IBINVALST_SYM when the default immediate V2 update setting for variant configuration statistics is used, leading to concurrent updates on the same records.

Resolution

SAP Note 1548171 describes how to update statistical data to improve performance when accessing material variants. While all options in the note achieve the objective, continuously up-to-date statistics (option 1) are not required. A later update (option 4) has minimal system impact and helps prevent deadlocks. SAP therefore recommends option 4 (collective update). In this approach, statistical data is always collected regardless of which process performs the configuration, and the actual statistics update is executed via report RCU_IBSTAT_UPDATE_STATISTICS.

When this variant is selected, entries in table IBBINDING must be removed per point 1 in the note and replaced as follows.

  • Below entries are delivered by default and need to be deleted in table IBBINDING:

SOMETHING_TO_POST_INST    IBSTAT_UPD_IBINST_OBJ
SOMETHING_TO_POST_VALUE  IBSTAT_UPD_IBINVALST_SYM

Then add following entries in table IBBINDING:

SOMETHING_TO_POST_INST    IBSTAT_UPD_IBINST_OBJ_COLL
SOMETHING_TO_POST_VALUE  IBSTAT_UPD_IBINVALST_SYM_COLL

  • Afterwards, refresh IBBINDING or delete the table buffer.

  • When using the “Collective update” option, schedule report RCU_IBSTAT_UPDATE_STATISTICS to run regularly. The report processes CLBUF records with RELID='CB', which are typically created with this option. Set the job frequency based on CLBUF growth, consult the Basis team for guidance.

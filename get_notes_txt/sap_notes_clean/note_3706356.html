Symptom

  • An error occurs in transaction /SCWM/ADGI when selecting a custom account assignment type.
  • The error message displayed is: "Invalid account assgnment for party entitled to dispose."

Environment

  • SAP S/4HANA
  • Logistics Execution (LE)
  • SAP Extended Warehouse Management (SCM-EWM)

Cause

According to the design, the default selection only takes into account orders of categories 'Internal order', 'CO production order' and 'PP production order'. The idea was to select the order categories which are used by many customers.

Resolution

The standard account assignment category 11 (Order) can be used here, if the user still wants to use the custom account assignment category, implement BAdI /SPE/AAC_IN to extend the selection criteria for the custom account assignment category.

  • Add the relevant order category for the order to table GT_RANGETAB.
  • Refer to the implementation example class /SPE/AAC_IN_EXAM to replace the default selection with the required selection criteria.
  • The value range for the order category can be found in Domain AUFTYP.
  • More information is available in transaction SPRO under IMG path: Logistics Execution -> Extended Warehouse Management Integration -> Cross-Process Settings -> Master Data -> Account Assignments -> BAdI: Account Assignment Determination - Field Definition.

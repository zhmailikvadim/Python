Symptom

  • A query using except clause is consuming large amount of procedure cache
  • The table being queried is a very large table with millions of rows
  • Attempting to kill process consuming procedure cache does not kill immediately
  • A 701 is not  immediately reported
  • If a 701 is later reported, the process consuming memory may resolve it self

Environment

Adaptive Server Enterprise (ASE) 16.0 SP04

Cause

The issue has been addressed under SAP CR 831709. Its description reads, "A very large amount of procedure cache may be consumed by a query performing an EXCEPT or MINUS clause against a table with millions of rows."

No target fix

There is currently no target Fix Pack or Service Pack scheduled for this issue. It is being reviewed by our Product Group.

Resolution

SAP CR 831709 is scheduled to be addressed in a future SAP ASE Service Pack. Please mark this KBA as a Favorite to be automatically notified when the CR is fixed.

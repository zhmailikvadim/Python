Symptom

  • Error message: /BOBF/ATF 158 The upload of file &4 with MIME type &1 is not allowed
  • Unable to upload files to incidents through the Reports/Documents tab with Manage Incident Tile
  • The system does not allow saving incidents after attempting to upload these files.

Environment

  • Incident Management

Reproducing the Issue

  1. Navigate to the Manage Incident App.
  2. Search for an incident an open it.
  3. Go to the Reports/Documents tab and attempt to attach (for example) a PPT or PPTX file with any document type/reference in documents section.
  4. Error /BOBF/ATF158  The upload of file &4 with MIME type &1 is not allowed will occur

Cause

Configuration / Missing customizing 

Resolution

  1. Navigate to the customizing point: "Define MIME Types for Attachments".
  2. To find the needed MIME content type locate it in table SDOKMIME for the corresponding extension (i.e. PPTX)
  3. Add a new entry for the Business Object "EHHSS_INCIDENT" and Node "ROOT".
  4. Enter the MIME type of the web object
    For this example using PPTX: add in column 'MIME type of a Web Object' application/vnd.openxmlformats-officedocument.presentationml.presentation).

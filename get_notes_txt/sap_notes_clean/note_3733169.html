Symptom

System shows error message "Maximum number of items reached in entry view" when trying to post a supplier invoice with more than 1,000 lines.

Environment

S/4HANA Cloud Public Edition

Reproducing the Issue

  1. Open the Create Supplier Invoice app.
  2. Create a supplier invoice with more than 1,000 line items.
  3. Post the invoice and observe the error "Maximum number of items reached in entry view".

Cause

The error occurs because the system enforces a hard limit of 1,000 line items per accounting document (entry view).

Resolution

Check the number of line items in the supplier invoice and ensure it does not exceed 1,000. If the invoice has more than 1,000 line items, split it into multiple invoices/documents so that each document contains 1,000 or fewer items.

Post each resulting invoice/document separately via the app or API.

Note that this item limit is system-defined and cannot be increased by configuration or customization.

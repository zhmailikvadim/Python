Symptom

A customer guide on how to handle a performance issue once detected and what are the required steps and information that need to get collected and provided in the SAP Support ticket.

Environment

  • SAP Build Work Zone, advanced edition
  • SAP Build Work Zone, standard edition

Resolution

  • Performance Incident -Required Information
    • In the event of a performance issue detected by a customer working on the SAP Build Work Zone product, a Customer Support ticket should be opened for the relevant component by its service EP-WZ -*, containing all information that is relevant to the issue:

      - Scenario description
      - Start time
      - Duration of the incident
      - Content size
      - Detailed steps to reproduce
      - All additional information that might be relevant for the investigation

            Furthermore, also provide an HAR file of the request, including sap -statistics:
            - Add sap -statistics
            - Collect a network trace HAR file

  • SAP Statistics

    SAP Performance Statistics

    To obtain the SAP performance statistics, perform one of the following:
    - Add "?sap -statistics=true" at the end of the request URL, regardless of its HTTP method type -GET, POST, PUT, or DELETE.
    - Enter (name=sap sap-statistics, value=true) in the HTTP request header.

        Note that the setting in the URL is treated with higher priority than the setting in the HTTP header.

  • Network Traces – Record as an HAR File

    Record Network Traces Using the Developer Tool
  1. Click Chrome / Edge Menu Menu->More tools tools->Developer tools, or just hit function key F12
  2. Select the „Network“ view (shown below)
  3. Optional -Detach the “Developer tools” window from main window
  4. Stop recording
  5. Clear requests using the clear icon (shown below)
    Note – The browser cache is part of the product design, please ensure not to use incognito mode and that the Disable cache check box is not enabled.
  6. Start your application with the „ sap -statistics=true “ parameter in the URL
  7. First execute the test case without recording, to fill the browser cache
    ▫ Note– Verify there aren’t any recorded round trips in the Developer tools window
  8. Click Start , to start the recording
  9. Perform the required steps to reproduce the issue -wait until all calls are completed
  10. Click Stop , to stop the recording
  11. Right click the “round trips” area and select “ Save all as HAR with content content”
  12. Save the trace as an HAR file with a file name describing the test case
  13. Attach the trace to the SAP support ticket

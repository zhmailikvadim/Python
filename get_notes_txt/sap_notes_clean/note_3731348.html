Symptom

Sometimes, you need to update an activity.
Example Scenario:

  • An activity that was already planned and released has disappeared from both the dispatching board and the Planning List.
  • However, when checking the Query API in FSM Admin, the activity shows an executionStage of "DISPATCHING", but the status is "OPEN" instead of "DRAFT".

Environment

SAP Field Service Management

Reproducing the Issue

  1. Go to the Planning & Dispatching board
  2. Open the Activity List and open the activity
  3. When you check the status, you will see that there is no option to change it from "OPEN"

Cause

Because of a wrong setting in the ERP system, the activity status was changed to 'OPEN'.

Resolution

Update the ACTIVITY status via Data API.

  1. Create a Client in FSM admin

    1. Log into the Admin application of your account (https://{cluster}.fsm.cloud.sap/admin/)

    2. Click on Client on left sidebar

    3. Click on “Create” button

      Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.

    4. Define:
      1. “client-id”
      2. select “Client-Authentication Method”,
      3. select the “User Policy Group”

        Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.

    5. Copy the “client-id” and “client-secret”

      Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.


  2. Encode Client Id and Client Secret
    1. Arrange “Client Id” and “Client Secret”, separated by a colon, into a single line of text like this:

      00017214-myTestClient:f4e23cf5-b2fe-4314-bc2b-fcbcc127220b

    2. Use any offline or online tool to encode this text into Base64 format. Make sure to not include any invisible newline characters at the end. 




    3. The result should look similar to this:

      This is the Authentication Basic header for the OAuth 2 API request.


  3. Request OAuth 2.0 Token

    1. Copy the following command into a terminal application on your local machine and replace the {cluster} placeholder with the cluster of your account and the Base64 encoded value after Authentication Basic with your own value:

      CURL:
      curl --request POST \
        --url       https://{{cluster}}.fsm.cloud.sap/api/oauth2/v2/token \
        --header 'authorization: Basic {{Encoded Client Id and Client Secret}}' \
       --header 'content-type: application/x-www-form-urlencoded' \
       --data grant_type=client_credentials


      Method:     POST
      URL:     https://{cluster}.fsm.cloud.sap/api/oauth2/v2/token

      Headers:
         Authorization: Basic {{Encode Client Id and Client Secret}}
         Content-Type: application/x-www-form-urlencoded

      Body:
      grant_type: client_credentials



    2. The response from the server should look similar to this:


    3. Copy the received token

  4. Retrieve the ACTIVITY
    1. Method:     GET
      URL:    https://{cluster}.fsm.cloud.sap/api/data/v4/{resourceName}/{resourceId}?dtos={resourceName.dtoVersion}

    2. Headers

      Authorization:     Bearer {{token}}
      X-Account-ID:    {{fsm-account-id}}
      X-Company-ID:    {{fsm-company-id}}
      X-Client-ID:    {{any}}
      X-Client-Version:    {{any}}

    3. Params

      CURL:
      curl --request GET \

       --url 'https://de.fsm.cloud.sap/api/data/v4/Activity/71C3F04CDCD94E5BB116D7923AB376E9?dtos=Activity.43' \

      --header 'authorization: Bearer {{token}}' \

      --header 'x-account-id: 98765' \

      --header 'x-client-id: Bruno' \

      --header 'x-client-version: 1' \

       --header 'x-company-id: 123456'

    4. Execute the API request

      The result should looks as following:

    5. The received data you can use for your UPDATE request.


  5. Update Existing ACTIVITY
    1.  Method:     PUT / PATCH
      URL:    https://{cluster}.fsm.cloud.sap/api/data/v4/{resourceName}/{resourceId}?dtos={resourceName.dtoVersion}

    2. Headers

      Authorization:     Bearer {{token}}
      X-Account-ID:    {{fsm-account-id}}
      X-Company-ID:    {{fsm-company-id}}
      X-Client-ID:    {{any}}
      X-Client-Version:    {{any}}

    3. Params

      forceUpdate:     true
      dtos:        {{dto}}.{{version}}


    4. Body – add fields with values you want to update 

       {

      "status": "DRAFT"

       }


      CURL:

      curl --request PATCH \

        --url 'https://de.fsm.cloud.sap/api/data/v4/Activity/71C3F04CDCD94E5BB116D7923AB376E9?dtos=Activity.43&forceUpdate=true' \

        --header 'authorization: Bearer <access_token>' \

        --header 'content-type: application/json' \

        --header 'x-account-id: 98765' \

        --header 'x-client-id: Bruno' \

        --header 'x-client-version: 1' \

        --header 'x-company-id: 123456' \

        --data '{

         "status": "DRAFT"

      }'



    5.  Execute the request


  6. Retrieve the ACTIVITY to check the updated value

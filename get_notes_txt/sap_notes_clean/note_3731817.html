Symptom

Errors FAA_POST210 and FAA_POST203 are rising during value transfer with ABLDT.

Environment

  • SAP S/4HANA

Reproducing the Issue

  1. Start transaction ABLDT/AS92 (value takeover) for an existing asset.
  2. Attempt a disposal/value takeover (movement type 200) on an asset that already has cumulative values (for example, posted APC and depreciation).
  3. Observe that the system raises errors FAA_POST210 and FAA_POST203.

Cause

Value takeover with ABLDT/AS92 is only permitted for empty assets; if the asset already contains cumulative values, the takeover is blocked and errors are issued.

Resolution

  1. Use AS92/ABLDT value takeover only for empty assets (no cumulative values present).
  2. Reverse the already posted transfer values, then start the ABLDT action again.

Symptom

This note includes the following corrections in the S/4 Defense Disconnected system:

Issue:

1. Outbound Delivery creation is processed before all STOs queues for the Outbound Delivery are processed in the inbound side (REM) during Delta Load (DL).

2. An error message regarding missing MBEW entries was raised when the system tried to reprocess the error queues via the report /SMFND/IBQ_TRANS_PROC_PROG.

Resolution: 

1. Code updated to check whether all the STOs mentioned in the delivery (LIPS table) are there in the inbound side (REM) before proceeding with Outbound Delivery create.

2. Outbound Delivery which is processed first initializes an instance of CL_MMIM_CROSS_SYSTEM which is kept alive also during the processing of later queues. Code logic added in Outbound Delivery MDO to reset this instance when the creation of the 6B1 movement  material document went into error in Delta Load in the inbound side (REM).

Other Terms

STO, Outbound Delivery, Stock Transfer Order, S/4 Defense Disconnected

Solution

Implement the corresponding CI as per the note.

Symptom

  • When analyzing an authorization trace, the failed check does not display the content that has been checked. Instead, it shows the object with empty data, making it difficult to resolve authorization issues.

Environment

SAP S/4HANA Cloud Public Edition 2602

Reproducing the Issue

  1. Open the app Display Authorization Trace.
  2. Analyze the authorization trace for a failed check.
  3. Observe that the failed check does not display the content that has been checked. Instead, it shows the object with empty data.

Resolution

This is a known issue, which we already have a prepared a fix for, which will be delivered with Hotfix Collection 6(Mar 14th&15th).

Symptom

  • When users transmits data from the client, it presents a blank transmit screen
  • This blank screen seems to be transmitting data but the users of the device may try again that can cause duplicate transaction
  • Example: When a notification edit was done on the mobile device then a Notification completion (NOCO) can be duplicated
    • This is when the transmit starts, it will fail because the transaction set (posts) and removes the notification then the second set attempts to do the same transaction, but the notification is already completed

Environment

  • SAP Work Manager 6.1
  • SAP Mobile Platform 3.0 (SP04 - WPF client 70.5.4)

Cause

 

  1. AG-24170: Fixed in SMP 3.0 SP05: The WPF client does not perform a transmit if the "Hide" transmission is enabled from the Agentry editor under the action step.
  2. SMPAgentry-3166: WPF Client gets stuck on closing the transmit screen.  - In this issue when the Start + Stop (by pressing stop button), the WPF gets stuck on closing the transmit screen. This particular issue is seen in newer version of SMP 3.0  70.6.7 SDK.
  3. SMPAgentry-1577 - Double Transmit on loss of connectivity (SMP 3.0 SP5). - If after any SMP 3.0 server reboot, any Agentry client devices that were previously set as online lose their connection and when a transmit is initiated will transmit twice - fix in SMP 3.0 SP5.
  4. SMPAgentry-2155 - After the server goes down,the push would not work and the transmit will fail. It will take two transmits for the user to be acknowledged by the system. So with this, the first transmit will fail and the 2nd transmit will re-authenticate to get connected (This could also be why the user have a blank screen).
  5. This is an SAP Work Manager 6.1 bug.

Resolution

Two potential solutions:

  1. Install and upgrade to SMP 3.0 SDK SP07 to get the Agentry client fixes on transmit with blank screens.
  2. Apply SAP note: 2151097 - SAP Work Manager - Duplicate transaction gets created if Transmit fails to connect to the Server.

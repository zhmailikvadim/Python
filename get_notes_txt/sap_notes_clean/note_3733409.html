Symptom

Business Rule action notification does not pick up person data entered in the recipient field (“To”).

Error returned: CA-09: Could not deserialize uploaded object to [NotificationDTO_V10], because it contains invalid JSON syntax or some of the passed data does not respect the required data type.



Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.   

Environment

  • SAP Field Service Management (FSM)
  • FSM Business Rules

Cause

Misconfiguration of the notification action “To” field: usernames or a JSON-like array strings (for example, ["Person1","Person2"]) instead of valid person IDs or a valid JavaScript expression that resolves to person IDs.

Resolution

BR To field should be enter the hardcoded Person IDs separated by a comma or a valid Javascript expression

1] Hardcoded Person IDs

Navigate to Account -> Company -> Person to find person IDs

2] Javascript expression

Created a variable with a key, then used that variable in the person box

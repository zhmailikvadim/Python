Symptom

When creating a sales document using idoc_input_orders which contains header text in segments E1EDKT1 and E1EDKT2. When multiple address handling is active and a different address number is passed from idoc, the header text from idoc is overwritten during the call transaction to VA01 as the flag RV02P-AGUPD is set which normally only occurs for a sold-to change but not for an address number change.

Other Terms

FV45KFAK_VBAK_FUELLEN_TEXTE, RV02P-AGUPD, kopftexte_loeschen_einzeln, LV09CF41, FCODE_ADDR_1000, MV45AF0P_PARTNER_CALL_BACK, lv_adrda_new.

Reason and Prerequisites

If the sold-to party is changed. The new text determination takes place for the complete document (header and items) but this should not occur for a address change.

Program error.

Solution

Please implement the attached correction instruction.

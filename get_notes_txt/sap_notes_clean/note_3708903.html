Symptom

If you subsequently change the commissioning date of an inactive generating unit, the start date is currently adjusted accordingly only for such time-dependent data in the generating unit for which the original start date of the corresponding time slice was identical to the old commissioning date.

Other Terms

Transaction /ENBIL/GU02, COMM_DATE

Reason and Prerequisites

Missing function

Solution

You can implement this SAP Note in your system either manually or by importing the relevant Support Package.

After you implement this SAP Note, the default behavior changes as follows:

Change the commissioning date (COMM_DATE) to the future, for example, 1/1/2024 to 1/1/2026:

  1. Time slices whose start date would be before the adjusted commissioning date of the generating unit are given the new start-up date as the new start date. The end date of the time slice is not adjusted.
  2. If time slices are determined whose start and end date are before the new commissioning date, these time slices are only deleted.
  3. If several time slices are determined for an object, for example, billing models for which the start date is before the new commissioning date, only the most recent time slice is adjusted accordingly. The other time slices are deleted.

Change the commissioning date (COMM_DATE) to the past, for example, 1/1/2026 to 1/1/2024

  1. The date is only adjusted for the time slices whose start date was identical to the original commissioning date.

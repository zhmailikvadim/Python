Symptom

For an inspection lot with physical sample, the field 'Origin Results Data' displayed in T-Code QE03 does not change even after changing the field in T-code QE13.

"Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental."

Environment

  • SAP ERP Quality Management (QM)
  • SAP ERP
  • SAP R/3
  • SAP R/3 Enterprise
  • SAP ERP Central Component
  • SAP Enhancement package for SAP ERP
  • SAP Enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA, on-premise
  • SAP S/4HANA Cloud Private Edition

Reproducing the Issue

  1. For an inspection lot with physical sample, change the field 'Origin Results Data' to for example '03 Batch Valuation' and save in T-Code QE11. 
  2. The field 'Origin Results Data' displayed in T-Code QE03 is '02 previous inspection lot', which is different from that in T-code QE13.

Cause

This is by standard design. 

Resolution

For inspection lots with physical samples, the system always uses the values from QAMV(the characteristic specification from the inspection plan).

The value written into QAMR‑QERGDATH is not the user input, but the value read from QAMV‑QERGDATH.

SAP intentionally overwrites the user‑entered origin to ensure that all sample-level results strictly follow the inspection plan’s definition.

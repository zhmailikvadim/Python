Symptom

The requirement is to add blank characters in a XML tag of the file when generating payment files using app Manage Automatic Payments 

 Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.

Environment

SAP S/4HANA Cloud Public Edition 

Reproducing the Issue

  1. Execute payment run via Manage Automatic Payments
  2. View File generated via Manage Payment Media 
  3. Check XML output where expected spaces are missing or compressed

Cause

In the standard system, blank spaces are not recognized in the payment file structure definitions.

Resolution

Open the XML file in text editor as browser will show formatted file and this is not the representation of the raw generated file. Formatted XML file is condensed for the whitespaces in the XML tag value.

There are four supported methods for introducing blank spaces in XML output:

Method 1: Removal of the length from the element:

  1. Find the problematic element where spaces are condensed.
  2. Remove its length.
  3. Activate the tree and retest

Method 2: Offset approach:

  1. Create element with mapping using atoms. 
  2. Keep the atom handling field blank to use fixed offsets.
  3. Create Atom 1 for the first field.
  4. Create Atom 2 with target offset = ("length of the first atom" + "the number of required spaces.")
  5. Activate the tree and retest.




Example: -

  • "Atom 1" in the screenshot above has length '35'.
  • "Atom 2" is defined with offset '40' as highlighted. 
  • This outputs 5 spaces in-between the nodes.

Method 3: ALT+255 (Windows) or "Option +Space"(MacOS) approach:

  1. Declare a node as a constant and press "ALT+255" (Windows) or "Option +Space" (MacOS) on keyboard for each spaces respectively.
  2. It is important to use a numeric keypad for typing the number 255. It will not work if you use the regular digits keys.
  3. If you don't have numeric keypad, please use On-Screen Keyboard. To switch on numeric keypad in On-screen Keyboard, select Options-->Select "Turn on numeric key pad". Now you can see as below: -





    Ensure that "Filter Keys" are switched on in your system under path Settings-->Ease of access-->Keyboard-->Switch on "Use filter keys". Please note that this path might differ according to the type of operating system you use.

Symptom

When executing an allocation in Fiori App Run Allocations (F3548) in test mode, no period check is performed. Period lock errors like F5903 - "Posting period &1 &2 not open for variant &4 and ledger &3" are only raised when running the allocation in update mode.

Environment

  • SAP Fiori
  • SAP S/4HANA
  • SAP S/4HANA Finance

Reproducing the Issue

  1. Open Fiori App Run Allocations
  2. Perform a test run --> no error is raised
  3. Perform an update run --> period lock errors are raised

Cause

The system does not check period status during test runs as per the system design.

Resolution

The test run does not post any document/data and does not require an open period. From a testing and maintenance perspective, it was intended that the test run should be possible to execute even if the period is closed. This behavior cannot be changed.

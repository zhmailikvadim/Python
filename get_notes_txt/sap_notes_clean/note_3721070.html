Symptom

The Extension Shared Folder field located in Cloud Control Center → Customer Management → Tenants → Tenant Details → Extensions includes extension version information but does not update to reflect the latest extension version.

Reproducing the issue

Prerequisites:

An extension with version 1 is deployed, creating an entry under Cloud Control Center → Customer Management → Tenants → Tenant Details → Extensions → Extension Shared Folder. The path appears as \share\tenantstorage\xxxx\Extension\Company\ExtensionName\version1.

Scenario:

Upgrade the extension from version 1 to a higher version, such as version 2. Then check the field value in the Cloud Control Center.

Expected behavior:

When deploying or upgrading an extension in Cloud Control Center, the extension shared folder path should exclude any version number and should appear as \share\tenantstorage\xxxx\Extension\Company\ExtensionName\.

Actual behavior:

The value remains as \share\tenantstorage\xxxx\Extension\Company\ExtensionName\version1.

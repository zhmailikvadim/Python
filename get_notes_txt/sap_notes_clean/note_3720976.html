Symptom

  • The "Catalogs" library pane appears empty or does not list standard Business Catalogs.
  • When using the "Manage Launchpad Pages" app, the user is unable to add apps to page sections.
  • The "Select Catalog" search returns no results even when catalogs are known to exist in the backend.

Environment

SAP S/4HANA 2023 or others (S4CORE 101-108 / UIS4HOP1 300-900)

Cause

  • Cause 1: Legacy personalization data from the previous SAP version is incompatible with the new S/4HANA 2023 metadata. This causes the UI to crash when looking for old object IDs.
  • Cause 2: The "Manage Launchpad Pages" administration tool is not a standalone app; it is an orchestrator that requires a fully activated backend foundation. The issue is typically caused by an incomplete or skipped execution of the Fiori Foundation Task List.
    • Why STC01 (SAP_FIORI_FOUNDATION_S4) is critical:
      1. Technical Catalog Replication: Standard administrative catalogs (e.g., SAP_CORE_BC_UI_FLD) are not delivered via transport; they are generated locally in each client by this task list. Without them, the "Add App" library has no source data to display.
      2. Service Interoperability: This task list activates the "plumbing" (OData and ICF services like arsrvc_upd_admin) that allows the frontend Page Builder to communicate with the backend Catalog Repository.
    • The "Incomplete Run" of the Task List trap for most users:
      • Users often attempt to save time or avoid perceived risks by selecting only "Page-related" tasks. This results in a "short run" (e.g., 3-5 minutes). A partial run is insufficient. If the core infrastructure tasks are not selected, the administrative apps will remain in a "broken" state despite the task list showing a "Success" status in STC02.
      • Special Note (Consulting/Integrator/Admin Overwrite Concerns): 
        • A common misconception is that running the full SAP_FIORI_FOUNDATION_S4 task list will overwrite custom-developed apps, catalogs, or roles. This is incorrect. The foundation task list is additive. It activates standard SAP services and generates standard SAP catalogs. It does not delete or modify "Z" (custom) namespace objects or existing customer configurations.

Resolution

Solution for Cause 1:

  1. Run transaction /UI2/FLP_DEL_PERS.
  2. Enter the affected User ID(s).
  3. Execute to clear the personalization.
  4. Log in using an Incognito browser window to test.

Solution for Cause 2: Perform the following steps in order. It is recommended to test in a Sandbox (Your Sandbox) before applying to Development (Your Dev System):

  • Step 1: Foundation Activation (Basis Team)
    • [ ] 1.1 Access Task List Manager: Run Transaction STC01.
    • [ ] 1.2 Load Task List: Search for SAP_FIORI_FOUNDATION_S4.
    • [ ] 1.3 Selection Strategy: Select the Full Task List. Do not cherry-pick individual tasks.
    • [ ] 1.4 Force Execution: If tasks show as "identical" or "previously run," uncheck the "Skip" column. This forces the system to re-generate the technical catalogs and re-verify the services.
    • [ ] 1.5 Verify Log: Run Transaction STC02. A successful full run typically takes 20–30 minutes. A run under 5 minutes indicates that critical generation steps were likely skipped.

      CRITICAL DIAGNOSTIC OVERRIDE: > Do not rely solely on the "Green" (Success) status indicator in transactions STC01 or STC02. In S/4HANA 2023 environments, the Fiori Foundation Task List may report a successful completion even if critical replication steps were bypassed due to a failure in a preceding task (e.g., Conversational AI or Web Assistant setup).

      The "1-Second Rule":

      Always verify the Duration of the 'Replicate Technical Catalogs' task.

      If the duration is 0–1 seconds, the task has silently failed to commit data to the database, regardless of the Green status.

      A valid replication for a standard S/4HANA 2023 library typically requires several minutes of active processing. If the total Task List runtime is significantly shorter than expected (e.g., under 10 minutes for a full foundation run), a New Run must be initiated to force a deep-level data commit.

  • Step 2: Technical Verification (Support/Admin)
    • [ ] 2.1 Check ICF Nodes (SICF): Verify path: /sap/bc/ui5_ui5/sap/arsrvc_upd_admin. Status must be Active.
    • [ ] 2.2 Verify Catalog Existence: Run Transaction /UI2/FLPCM_CUST. Under the Catalogs tab, search for the required catalog that is missing.
    • [ ] 2.3 Clear System Caches:
      • Run Transaction /UI2/INVALIDATE_GLOBAL_CACHES (Type: Transaction). - Clears all UI2 caches on the server side. Use this after the Foundation Task List (Transaction and Report names are identical). 
      • Run Report /UI2/CHIP_SYNCHRONIZE_CACHE(Type: Report -> Tcode SE38). - Vital for the "Manage Launchpad Pages" app to see the newly generated chips/catalog
      • Run Report /UI5/APP_INDEX_CALCULATE (Type: Report -> Tcode SE38) for all apps (Updates the app index). 
      • Run Transaction /UI2/INVALIDATE_CLIENT_CACHES (Type: Transaction) - Use this if the issue is only appearing for specific users or in one specific client.
      • Run Transaction /IWFND/CACHE_CLEANUP  (Type: Transaction) - This is for the Gateway Cache. Clears the Odata metadata cache. Run this if the "Add App" service structure changed.
      • Run Transaction /IWBEP/CACHE_CLEANUP (Type: Transaction) - This is for the Backend Odata Cache. Clears the metadata on the backend provider side. 
    • [ ] 2.4 Authorization Check: Run Transaction SU53 for the affected user immediately after a "No Data Found" result. If clean, have Basis run an STC01 (Authorization Trace) during the "Search Catalog" action to ensure S_SERVICE and S_PB_CHIP are not failing silently. 
      • Note: Verify user has S_PB_CHIP and S_SERVICE via SU53.
  • Step 3: Browser & UI Validation (End User)
    • [ ] 3.1 Browser Cleanup: Clear browser cache and restart the session.
      • Mandatory: Perform the final test in an Incognito/InPrivate window to bypass any persistent local metadata (CTRL+SHIFT+N). 
    • [ ] 3.2 Final Test: Access "Manage Launchpad Pages" and confirm the Catalog library is now populated.
    • [ ] 3.3 UI Settings Check: In the Fiori Launchpad, click the User Icon > Settings > Spaces and Pages. Ensure "Use Spaces" is toggled correctly. If catalogs appear in the App Finder but not in Manage Launchpad Pages, verify the user has the SAP_CORE_BC_UI_FLD (Fiori Foundation) role assigned.

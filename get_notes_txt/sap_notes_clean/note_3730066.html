Symptom

  • German umlauts (ä, ö, ü, ß) in an incoming EDIFACT message are converted to special characters during transmission/parsing.
  • The EoG message is rejected with a negative CONTRL.
  • A syntax error is reported in the segment due to invalid characters.

Environment

  • Product: SAP Market Communication for Utilities
  • SAP Market Communication for Utilities

Reproducing the Issue

  1. Prepare an EDIFACT interchange that includes German umlauts (ä, ö, ü, ß) in fields such as NAD.
  2. Transmit the interchange with a declared character set that does not support accented characters (for example, UNOA).
  3. Observe that the umlauts are parsed as special characters such as � at the receiver, resulting in a negative CONTRL citing a syntax error in the NAD segment.

Cause

  • The interchange was transmitted using a character set (such as UNOA) that does not support accented characters. During parsing, umlauts were replaced with �, leading to a syntax error in the NAD segment and a negative CONTRL.

Resolution

  1. Ensure that message content only uses characters supported by the declared EDIFACT character set.
  2. If umlauts must be transmitted, declare and use a character set that supports accented characters instead of UNOA.
  3. Resend the interchange after aligning the character set and content.
  4. Validate that the receiver processes the message without a negative CONTRL.
  5. Refer to: EDIFACT Syntax Rules for allowed character sets and syntax levels.

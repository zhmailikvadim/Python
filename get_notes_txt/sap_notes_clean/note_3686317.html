Symptom

A custom task is configured with a question that uses a radio button field type. However, when the task is accessed by an Onboardee, the radio button options do not appear as expected. Other fields within the same task remain visible and accessible. 

Environment

SAP SuccessFactors Onboarding

Reproducing the Issue

  1. Create a custom task.
  2. Configure one of the questions to use a radio button.
  3. Test the task as a onboardee.
  4. Observe that the radio buttons are not visible as expected.

Cause

The radio buttons do not appear because the field was created using a data type that does not support displaying radio button options in custom tasks.

Resolution

  • Since the fields configured in custom task is dependent on the custom mdf ojbect, you can only use the following data type
  • Note: Only String, Number, Boolean, Decimal, Date, DateTime, Time, Generic Object, User type fields are supported so far.

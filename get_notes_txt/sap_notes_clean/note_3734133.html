Symptom

  • After migrating to S/4HANA, expected default account assignment entries in table TKA3A (transaction OKB9) were not created for some cost elements, which had a default account assignment object (cost center or order) available in the classic release (transaction KA02/KA03)
  • For one cost element with a past validity end date, entries were created.
  • No errors were reported for migration step DAA in FINS_MIG_STATUS.

Environment

  • SAP S/4HANA
  • SAP S/4HANA Finance
  • SAP S/4HANA Cloud Private Edition

Reproducing the Issue

  1. Perform the DAA (Default Account Assignment) migration step and confirm its completion in FINS_MIG_STATUS.
  2. Check table TKA3A via transaction OKB9 for the affected cost elements.
  3. Observe that entries are missing for some cost elements and/or created only for a subset of company codes.
  4. Verify that no errors are shown for DAA in the migration log.

Cause

  • The DAA migration logic protects existing configuration: if an entry for a G/L account already exists in TKA3A (OKB9), the tool skips it to avoid overwriting.
  • Certain cost element categories may be excluded from automatic migration if they were not explicitly mapped in the source scope.
  • To avoid redundant data, DAA creates OKB9 entries only for company codes determined from the default account assignment objects (for example, cost centers or orders) actually used for the given cost element.
  • The migration reads the last known assignment from the source (for example, CSKB) regardless of an end date in the past, to ensure historical postings or reversals do not fail.

Resolution

  1. Verify the affected G/L accounts in FS00 are of type Primary Costs (Type P); non-operating types are not considered for default account assignment.
  2. In transaction OKB9, manually maintain the missing default account assignment entries (table TKA3A) for the affected G/L accounts and relevant company codes.
  3. Do not expect DAA to overwrite existing TKA3A entries; adjust or complete configuration in OKB9 as required.
  4. If entries exist only for some company codes, review which company codes are actually determined by the assigned default objects (cost centers/orders) and maintain additional company codes only if required by the business.

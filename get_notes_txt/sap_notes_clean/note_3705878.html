Symptom

  • The extractor C_CmmdtyVersLogcPrcgDocsDEX, which is based on table CMM_VLOGP, provides incorrect values for Term-to-Document FX conversion.

  • Due to incorrect field mapping, Term-to-Document FX factors are not derived correctly in downstream consumers (e.g. exposure, valuation, or reporting).

  • The fields TermToDocFixedFactorFrom and TermToDocFixedFactorTo were mapped incorrectly, leading to:

    • Incorrect FX conversion results

    • Inconsistent NFE / MTM values

    • Mismatch between valuation and reporting layers

  • The issue is observed during data extraction via C_CmmdtyVersLogcPrcgDocsDEX, even though the source data in CMM_VLOGP is correct.

Other Terms

CMM; ACRA; Advanced Commodity Risk Analytics; Data Extractor; VLOGP; C_CmmdtyVersLogcPrcgDocsDEX; I_VersionedLogcDocument

Reason and Prerequisites

Reason

In the base view I_VersionedLogcDocument, the fields required for Term-to-Document FX conversion were incorrectly mapped.
Specifically:

  • TermToDocFixedFactorFrom was mapped to previous_md_factor_from

  • TermToDocFixedFactorTo was mapped to previous_md_factor_to

However, the correct source fields for Term-to-Document FX conversion are:

  • fx_fix_factor_from_term_to_doc

  • fx_fix_factor_to_term_to_doc

Due to this incorrect mapping, the extractor C_CmmdtyVersLogcPrcgDocsDEX, which consumes I_VersionedLogcDocument, propagated incorrect FX conversion factors.

Prerequisite

  • The issue occurs only when Term-to-Document FX conversion is relevant for the pricing or valuation scenario.

  • The extractor C_CmmdtyVersLogcPrcgDocsDEX is used for data extraction based on CMM_VLOGP.

  • No data inconsistency exists in CMM_VLOGP; the issue is limited to the field mapping in the CDS view.

Environment

Reproducing the Issue

  • Carry out a successful upgrade of the database version
  • Upgrade the instance via command ./db2iupgrade -u db2<sid> db2<sid>
  • Error SQL0551N Authorization ID does not have required authorization is returned for the DB2<SID> user 
    Operation: EXECUTE
    Object: SYSPROC.ENV_GET_REG_VARIABLES

Cause

SQL0551N Authorization ID does not have required authorization
Authorization ID: "DB2<SID>"
Operation: EXECUTE
Object: SYSPROC.ENV_GET_REG_VARIABLES
SQLSTATE=42501

 

Resolution

Resolving the SQL0551N  The statement failed because the authorization ID does not have the required authorization or privilege to perform the operation.  Authorization ID: "DB2<SID>".  Operation: "EXECUTE". Object: "SYSPROC.ENV_GET_REG_VARIABLES".
In the following steps, check if the upgrade user has EXECUTE on SYSPROC.ENV_GET_REG_VARIABLES and grant it if missing. Then verify the grant, rerun db2ckupgrade to confirm the database is upgrade-ready. Once confirmed, rerun the db2iupgrade to upgrade the instance.

Steps:

  1. su - db2<sid>
  2. db2 connect to <SID>
  3. db2 "SELECT GRANTEE, GRANTEETYPE FROM SYSCAT.ROUTINEAUTH WHERE SPECIFICNAME='ENV_GET_REG_VARIABLES'";
    If the user does not exist, run:
  4. db2 "GRANT EXECUTE ON FUNCTION SYSPROC.ENV_GET_REG_VARIABLES TO USER DB2<SID>"
  5. db2 "SELECT GRANTEE, GRANTEETYPE FROM SYSCAT.ROUTINEAUTH WHERE SPECIFICNAME='ENV_GET_REG_VARIABLES'";
  6. Confirm the database is upgrade-ready. See the upgrade guide for full steps.<DB2_SWDIR>/instance/db2ckupgrade –e –l /tmp/upgrade.log
  7. Upgrade the DB2 instance. Refer to the upgrade guide for detailed instructions.<DB2_SWDIR>/instance/db2iupgrade –u db2<dbsid> db2<dbsid>

Symptom

In the processes "Receipt of Energy Amounts (SUP View)" (DE_B_MET_ENERGY_VALUE_SUP) and "Receipt of Energy Amounts (DSO View)" (DE_B_MET_ENERGY_VALUE_DSO), the cloud system does not send an APERAK Z20 message when the OBIS code received from market partner is invalid.

This issue is relevant for the market roles: SUP, DSO.

Other Terms

MSCONS, MET, 13109, Obis Code, APERAK Z20, DE_B_MET_ENERGY_VALUE_SUP, DE_B_MET_ENERGY_VALUE_DSO

Reason and Prerequisites

Enhancement

Solution

Please note that the validation of OBIS for energy amount is customer specified, you need to enhance your own check logic in business system then send the result to cloud, the recommanded steps are the following:

  • Extend the step "Send Response to Supply Scenario Read Request" in the processes DE_B_MET_ENERGY_VALUE_SUP or DE_B_MET_ENERGY_VALUE_DSO.
  • Add a customer-specific check group to validate the OBIS code.
  • If the OBIS code is invalid, populate the following fields in UBD:
  • - /us4g/ubd_data-req_result-req_result_code = 'Error'
  • - /us4g/ubd_data-req_result-req_result_txt = 'Invalid OBIS'
  • The business system then sends this information to the cloud along with the supply scenario response
  • When cloud system receives above result code and result text, it will send an APERAK message with error code Z20.

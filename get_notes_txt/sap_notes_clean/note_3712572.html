Symptom

With SAP Note 3553039 - IT0080: Setting the end date, the end date in the IT0080 record is set to the day before the 9th Birth date of child is set if it was previously a high date, regardless of whether or not a birth type without live birth is maintained in the Maternity Protection/Parental Leave infotype (0080). This SAP Note provides a solution that can be used to prevent the end date from being shifted.

Other Terms

Birth types without live birth, Maternity Protection/Parental Leave infotype, IT0080, T554G

Reason and Prerequisites

From a technical point of view, it was not possible to tell from the birth type whether there is a child, or whether it is a miscarriage, stillbirth, or a termination of pregnancy because the birth types are defined as Customizing in table T554G (birth types).

Solution

Import the Support Package assigned to this SAP Note or implement the advance correction. After you import the HR Support Package or implement the correction instructions, you must perform further follow-on activities:

Setup / Required Activities
Enter the birth types without live birth for the personnel subarea grouping you want to use for attendance/absence types in table T544G_DE_OLG (transaction SM30, view T544G_DE_OLG).

How it Works

For the personnel number processed in the Maternity Protection/Parental Leave infotype (0080), the system first checks whether the value from table T544G_DE_OLG exists in the "Type of Birth" field. In the positive case, the move of the end date in the data record is canceled.

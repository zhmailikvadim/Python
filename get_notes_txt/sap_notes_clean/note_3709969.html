Symptom

When posting is made with ABT1N using a customer defined transfer variant, the transaction type for some areas (for the sending asset as well for the receiving asset) is not the one defined in the transfer variant.

Environment

  • FIN Asset Accounting
  • SAP ERP
  • SAP ERP Central Component
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA
  • SAP S/4HANA Finance
  • SAP S/4HANA Cloud Private Edition
  • SAP Fiori for SAP S/4HANA Finance,
  • SAP Fiori for SAP S/4HANA,
  • SAP Fiori for SAP S/4HANA Cloud Private Edition

Reproducing the Issue

  1. Use transaction ABT1N with a customer-defined transfer variant.
  2. Observe that for certain areas, an acquisition/retirement transaction type is picked instead of the transfer transaction types entered in the transfer variant.

Cause

The issue is related to the configuration of the transfer variant and usage of cross-system depreciation areas.

By definition, an intercompany transfer does use retirement and acquisition transaction types. If transfer transaction types are used (3xx), they only work to areas where cross-system depreciation area is assigned in OADB.

Resolution

  1. Verify the transaction type group assignment based on SAP Note 327088.
  2. Check the configuration in table T093 to ensure proper assignment of cross-system depreciation areas.
  3. Remove non-existing cross-system depreciation areas in OADB or assign them to areas intended.
  4. Test transaction ABT1N again after implementing the changes.

Symptom

Get error “TAXPS cannot be determined in non-tax items” (Message no. FS210) during central finance initial load posting simulation.

Countries are USA, Canada and Puerto Rico only 

Environment

  • SAP S/4HANA Finance
  • SAP S/4HANA

Reproducing the Issue

Start a Central Finance initial load for FI postings.

Cause

For an accounting document that originates from a Central Finance source system, the external tax provider should not be called again when the accounting document is posted in the Central Finance system.

Resolution

Make sure all note listed in note  3523474 - Prevent Repeated Call of External Tax Engine in Central Finance System implemented and manually steps done which mentioned in this note:

***

2. Create entries in the maintenance view

    Transaction SM30

    Table/View V_CFIN_SKIP_ETEC

If you want to deactivate the call of the external tax engine, create an entry for each company code involved and set the indicator "Deactivate".

Only journal entries transferred from a Central Finance source system are affected by this maintenance view. Journal entries posted directly in the target system are not affected.

***

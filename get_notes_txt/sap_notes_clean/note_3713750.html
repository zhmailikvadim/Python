Symptom

When I send a Contract from SAP Ariba Contracts to SAP ERP via the SAP Integration Suite, the managed gateway for spend management and SAP Business Network, the ContractStatusUpdateRequest (CSUR) is not triggered by SAP ERP.

Cause

The CSUR interface isn't correctly configured in SAP ERP.

Resolution

Follow the steps to check if the CSUR interface exists:

SAP ERP transaction code SE38 > Execute program PRG_SRT_SINGLE_CONFIG > search for the object CO_PUR_PURGCONTRMNTCO in the Object Name field> Configurations.

If it exists, the entry should be deleted by clicking the trash bin icon in the Delete column. If it doesn't exist, no action is needed on this tab.

After this, the CSUR interface should be recreated following the steps mentioned in the guide below:

Configuring the Connections to SAP Integration Suite, Managed Gateway for Spend Management and SAP Business Network

See Also

To check if there is an issue with the CSUR interface, follow the steps below:

SAP ERP Transaction Code SE38 > Execute program PRG_SRT_SINGLE_CONFIG > search for the object CO_PUR_PURGCONTRMNTCO in the Object Name field > Configurations > Ping.

The expectation is that the test will generate a successful result.

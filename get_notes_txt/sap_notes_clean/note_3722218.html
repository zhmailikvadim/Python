Symptom

In the DBAcockpit transaction overview, the EXPLAIN function does not work in either the SAP GUI or the Webdnypro following a database upgrade on a lower Basis level, example SAP Basis 701.

Environment

Db2 LUW IBM

Reproducing the Issue

  • Call transaction DBAcockpit
  • Select a statement that you wish to analyse in the EXPLAIN function overview
  • Exception is returned 
    SQL0551N  The statement failed because the authorization ID does not have the required authorization or privilege to perform the operation.  Authorization ID: "<SCHEMA>".  Operation: "DROP FUNCTION". Object: "<SCHEMA>.EXPLAIN_GET_MSGS".  
    The error can also be returned for SAP<SID>, SYSTOOLS or SAPTOOLS.

Cause

SYSPROC.SYSINSTALLOBJECTS is being called as the wrong user with insufficient privileges. 

Resolution

  1. First check the owner of the EXPLAIN_GET_MSGS routine.
    The following is an example of how to capture the schema and owner of the routine.
  2. In the DBAcockpit, go to the SQL Editor and run the following command to get the owner of the EXPLAIN_GET_MSGS routine.
    You will need to replace SCHEMA from the EXCEPTION in the command below where you see <SCHEMA>.

    SELECT ROUTINESCHEMA, ROUTINENAME, OWNER FROM SYSCAT.ROUTINES WHERE ROUTINENAME = 'EXPLAIN_GET_MSGS' AND ROUTINESCHEMA = '<SCHEMA>';
    ===================================================================================================

    ROUTINESCHEMA           ROUTINENAME                        OWNER

    <SCHEMA>                     EXPLAIN_GET_MSGS             <DB2<SID>

  3. If you do not have access to the SQL editor in the DBAcockpit, you can run the command at the OS level as follows
    As the db2<sid> user 
    db2 "SELECT ROUTINESCHEMA, ROUTINENAME, OWNER FROM SYSCAT.ROUTINES WHERE ROUTINENAME = 'EXPLAIN_GET_MSGS' AND ROUTINESCHEMA = '<SCHEMA>'";
    For the above command you need to replace the <SCHEMA> with the actual SCHEMA from the EXCEPTION before you can run the command.
  4. Once you have the owner of  'EXPLAIN_GET_MSGS', then go back to the exception SQL command which was generated by SYSPROC.SYSINSTALLOBJECTS
  5. Ensure that you are logged in as the owner of the EXPLAIN_GET_MSGS', in our example this was the DB2<SID>
    Call the command directly on the OS level. You need to insert the SCHEMA of the 'EXPLAIN_GET_MSGS' before you run the command 
    db2 "CALL SYSPROC.SYSINSTALLOBJECTS((SELECT 'EXPLAIN' FROM SYSIBM.SYSDUMMY1), 'D', CAST(NULL AS VARCHAR(128)), 'SCHEMA')";
  6. Go to the DBAcockpit and try to run an EXPLAIN.
  7. The issue should now be resolved with the above steps.

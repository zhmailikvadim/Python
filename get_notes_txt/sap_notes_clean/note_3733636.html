Symptom

  • A job started for a single contract spends about 70% of runtime in functions such as /MSG/R_AR_S_VTG.
  • During an intermediate save, the processing mode is switched to online.
  • Before the save call, data is read from a buffer.
  • Subsequent contract checks, due to the changed status, read from the database instead of reading buffered data.

Environment

  • Product: SAP Reinsurance Management
  • Reinsurance Basis

Reproducing the Issue

Cause

  • Switching the processing mode to online during an intermediate save leads subsequent contract checks to bypass the buffer and perform database reads, increasing processing time in routines such as /MSG/R_AR_S_VTG.

Resolution

Please implement note 3678488 https://me.sap.com/notes/3678488

See Also

refer to: 2969325 - PERF: Performance improvement in reinsurance account statement

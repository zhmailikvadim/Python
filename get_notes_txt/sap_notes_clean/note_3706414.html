Symptom

The error KW407 "No prices could be determined for object &&&&&" is raised during costing. This issue can also occur with order creation or confirmation posting.

Environment

  • Controlling (CO)
  • SAP R/3
  • SAP R/3 Enterprise 4.7
  • SAP ERP Central Component
  • SAP ERP
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA Finance
  • SAP S/4HANA

Reproducing the Issue

  1. Execute transaction CK11N
  2. Input the Material number, plant and costing variant to execute it

Cause

The system can not determine the price for the activity type. It can occur that in the valuation variant you set the strategy as "Actual Price for the period". But currently there is only planned price maintained via transaction KP26.

Resolution

Please check the valuation variant setting and make sure the system can find a price for the activity type. Also please check the plant-specific setting.

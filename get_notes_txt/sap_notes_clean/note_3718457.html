Symptom

When moving WBS in the structure of a project the message CJ418 'WBS element & can no longer be shifted' is returned 

Environment

  • SAP ERP Project System (PS)
  • SAP R/3
  • SAP R/3 Enterprise
  • SAP ERP Central Component
  • SAP ERP
  • SAP Enhancement package for SAP ERP
  • SAP Enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA, on-premise
  • SAP S/4HANA Cloud Private Edition

Reproducing the Issue

Run CJ20N 
Enter Project /WBS
Move WBS or change structure 
Message CJ418 is raised  

Cause

There is a budget assigned to the WBS element on which there are costs 

Resolution

Costs will have to be removed from the WBS then Re-post any costs  (See  SAP Note 95838) 

The status 'BUDG' will need to be reset in order to change the project hierarchy.

The WBS element may have been budgeted using only ONE "type" of activity of budgeting like "original budget" or "Supplement" or "Return" or "Transfer", then it is possible to reset the status BUDG.


If for example,  the budget is created with CJ30 and added some additional budget using transaction CJ37 (Supplement) it is not possible anymore to reset the status BUDG.

So If you want to reset the status BUDG you need to do the following:

1. Delete your budget of the WBS element within that transaction which you first used to create your budget.

2. Goto transaction OPSX.

Enter your WBS-element.

If you choose "no test run" BUDG will be reset.

Then check the status once more through CJ20N. If the status BUDG is no longer present the hierarchy can be changed for the WBS element.

However
If you have already assign costs/PR/PO against the WBS.
System will not allow you to un-release the budget.
In this situation if you can't reverse the cost booked under WBS, there is one way in standard which should work even when actuals are already existing:

1) turn off the budget check using transaction CJBW

2) delete the budget values from the affected subtrees (the one where the WBS-element to be replaced is at the moment, and the one where it is to be replaced to).

3) unset 'BUDG' status using transaction OPSX

4) relocate WBS-element

5) recreate budget values

6) activate budget check using transaction CJBV (if necessary, as this should be done automatically after 5)).

(Where possible  perform these steps in a test system and verify the results before doing so in a production system.)

Symptom

  • When running FAGLB03 the balance correction entries (DF* documents) generated during S/4 migration are visible and contribute to the total balance.
  • When running FAGLL03 directly for the same selection, the DF* documents are not selected.

Environment

  • Reporting/Analysis/Display
  • SAP S/4HANA

Reproducing the Issue

  1. Execute transaction FAGLB03.
  2. Double-click on the amount to drill-down in transaction FAGLL03.
  3. Amounts in transaction FAGLB03 and FAGLL03 are identical and correct. Result list in transaction FAGLL03 shows DF* documents.
  4. Execute transaction FAGLL03 directly with the same selection criteria. DF* documents are not visible in the result list.

Cause

The current behavior is according to standard delivery settings. DF* documents are respected by FAGLB03 and are visible after drilldown from FAGLB03 to FAGLL03. However, DF* documents are not respected when FAGLL03 is called directly.

Resolution

If your business-specific requirement differs from SAP standard delivery settings and you would like to output DF* documents when transaction FAGLL03 is called directly, set customizing as specified in SAP Note 2857175 - FAGLL03 - Select Records with MigSource U.

  1. Navigate to the menu: System -> User Profile -> User Data -> Tab 'Parameters'.
  2. Set the following parameter:
    Parameter: FAGL_OUTER_JOIN
    Value: ENFORCE
  3. Save the changes.

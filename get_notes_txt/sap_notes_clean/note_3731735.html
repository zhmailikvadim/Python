Symptom

I am encountering a server error when trying to approve a Purchase Requisition from Guided Buying or the classic Buying interface, and the system displays an error reference ID and does not complete the approval action. 

Cause

The issue is caused by incomplete or inconsistent master data, specifically when a Unit of Measure used in the requisition has missing mandatory fields (such as a null or empty Name field). This leads to backend errors during approval processing.

Caused by: java.lang.NullPointerException: Cannot invoke "ariba.base.core.MultiLingualString.getString(java.util.Locale)" because the return value of "ariba.basic.core.UnitOfMeasure.getName()" is null.

Resolution

Verify the Unit of Measure master data used in the affected requisition.

  • Ensure all required fields, especially the Name field, are properly maintained.
  • If discrepancies are found, update the UOM data via:
    • Data import/export Unit of Measure task for bulk correction
  • For future transactions, ensure that all master data fields are correctly maintained to prevent such errors.

Note: If the affected requisition is no longer active (e.g., deleted or deactivated), no further action can be taken on that document, and the fix will apply only to new transactions.

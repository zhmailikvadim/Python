Symptom

There is an onchange rule created to raise a message with error severity.

In legacy people profile the error pops up as expected, however, in latest/full people profile the message appears with a warning icon instead of a red error icon.

Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental

Environment

SAP SuccessFactors Employee Central

Reproducing the Issue

  1. Navigate to a user profile in Latest (Full) People Profile.
  2. Perform a change that runs an onChange business rule configured to raise a message with “Error” severity.
  3. Observe the message displays with a warning-style icon/non‑blocking notification instead of an error icon.

Cause

Having an Error-level validation during onChange is not a valid scenario in Latest (Full) People Profile. This behavior (error pop-up on field change) was only available in Legacy People Profile. In Latest People Profile, onChange validations surface as non‑blocking notifications, which can appear with a warning icon.

Resolution

Move validations that must block saving to onSave rules on the relevant Job Information element/field.

Symptom

You use SAC with a BPC live-connection. A SAC story contains an input-ready query with a dimension of type DATE exposed as a key figure. In this case SAC shows date values in the format defined in the user settings on the BW ABAP server (in the following called 'external format'). The date values can be changed one by one or in a mass operation e.g. one may copy more than one cell into the respective data cells. The system assumes the changed cell values to be in external format, if this is not the case the changes might not be processed, the system sends corresponding messages.

The note improves the conversion of the values from external to internal format even in some cases where the external format according the the user setting is not used, even different similar formats might be used.

Example:

The ABAP internal format of DATE values is YYYYMMDD (Y for year, M for month, D for day, Gregorian Calendar), the external format e.g. might be DD-MM-YYYY. The system then expects changed date values in this format but in the improved version also some other formats are accepted, e.g. DD.MM.YYYY or DD/MM/YYYY: these formats will not lead to conversion errors. But using the above user setting and changing a value using the format YYYY/MM/DD will lead to a conversion error, the system will not accept a value in this format.

Of course, changing data cells of type DATE for a characteristic exposed as key figure, the leading format that should be used is the one defined in the BW ABAP server user setting.

Other Terms

SAC, BPC Embedded, DATE, SID key figure, characteristic as key figure, input-ready query, planning, BPC live-connection, BW InA

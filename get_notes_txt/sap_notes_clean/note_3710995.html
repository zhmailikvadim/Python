Symptom

When a non-primary language is selected, and the user follows a specific sequence of actions (taking the test → failing the test → closing the test screen → tapping the test start button), the screen displays a mix of the primary language and the selected language.

Environment

SAP SuccessFactors Learning

Reproducing the Issue

  1. Select a non-primary language in the SAP SuccessFactors Mobile app.
  2. Take the test.
  3. Fail the test.
  4. Close the test screen.
  5. Tap the test start button.
  6. Observe the screen displaying a mix of the primary language and the selected language.

Cause

The overall exam interface and messaging are determined by the exam language selection. The question text itself is configured by the user and is not influenced by the language selection. Additionally, the language selection is session-specific and does not persist across sessions.

Resolution

  • Engineering team confirmed that this is expected behavior.
  • Understand that the overall exam interface and messaging are controlled by the exam language selection, while the question text is configured independently and not influenced by the language selection.
  • Note that the language selection is session-specific and will default back to the primary language if the user exits the exam and returns later.

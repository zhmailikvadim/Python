Symptom

  1. SAML2 SSO is configured with the following:
    • On ABAP system (transaction SAML2):
      • NameID Format is set to "Unspecified";
      • User ID Mapping Mode is set to "Mapping in USREXTID table, type SA";
    • On Identity Provider (IDP), NameID Format is also set to "Unspecified".

  2. During SAML2 authentication, after IDP redirects back to the ABAP system, there's a "401 Unauthorized" error, a logon screen, or a logon popup.

  3. In a Security Diagnostic Tool trace,  messages such as the below ones can be found:
    • SAML20 SP (client <client number> ): Federation error: No user found for name ID '<nameID>' (format: unspecified). Details: No user found for name ID '<nameID>' (format: unspecified).
    • SAML20 SP (client <client number> ):  Exception raised:
      SAML20  SAML20 CX_SAML20_FEDERATION: No user found for name ID '<nameID>' (format: unspecified). Long text: No user found for name ID '<nameID>' (format: unspecified). 

"Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental."

Environment

  • SAP NetWeaver
  • ABAP platform

Cause

External ID inactive or with incorrect syntax.

Resolution

  1. Go to transaction SM30 > Table/View VUSREXTID > External ID type SA.

  2. Make sure that:
    1. There's an entry for the intended user (User column);

    2. The above entry is Activated;

    3. For Name ID with "Unspecified" format, the External ID needs an specific syntax so that USREXTID table can associate the information with the correct Identity Provider (IDP):

      <trusted_provider>:<name_qualifier>:<name_id>

      or

      <trusted_provider>::<name_id>


      Where:
      • Trusted Provider:
        IDP's ID as shown on:
        • Transaction SAML2 > tab Trusted Providers > Name

          or
        • IDP's metadata file > entityID attribute's value
      • Name Qualifier:
        Optional attribute with a domain string value, provided by the IDP.
        • If available, it can be found on SAML Response's Assertion, within Subject attribute.
        • The parsed SAML Response can be found in the Security Diagnostic Tool trace.
      • Name ID:
        The value that will be mapped with IDP's send Name ID value.

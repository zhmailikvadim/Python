Symptom

  • The webi report is based on a universe based on bex query (.UNV)
  • Two prompts are defined: One at universe level with @prompt and the second at webi query panel level as optional
  • When running the report, if you do not provide a value for the optional prompt, the result is correct
  • If you do provide a value for the optional prompt, the universe level prompt is disregarded resulting in incorrect results displayed

Other Terms

bex, bw, taken into consideration, variable, webintelligence, web intelligence

Reason and Prerequisites

  • SAP BusinessObjects BI4.1 SP5 patch9

WORKFLOW:

  1. Launch UDT
  2. Open the universe based on a bex query
  3. Create an @prompt on the 'Calendar month '
  4. Launch WRC or Bi Launchpad
  5. Create a report based on that universe
  6. Define a prompt in the query panel on another object such as 'Commercial Unit'
  7. Run the query
  8. In the prompt window, Select a value for 'Calendar month', leave the 'Commercial Unit' as blank
  9. Results are OK
  10. Refresh the report
  11. In the prompt window, select a value for both 'Calendar month' and 'Commercial Unit'
  12. Notice the results are incorrect, the 'calendar month' has not been taken into consideration

Solution

  • This issue is currently under investigation by the development team

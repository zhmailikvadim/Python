Symptom

  • Some accounts were updated or created with outdated information and needs to be updated with the old data removed.
  • When using CSV files to import the accounts in bulk with fields updated, blank values are parsed as empty string values.

Environment

  • SAP Customer Data Cloud
  • Dataflows (IdentitySync)

Resolution

To properly import fields and set values to null, use JSON as the file type and switch to the file.parse.json component in the dataflow setup, instead of file.parse.csv component.

Keywords

CDC, Gigya, reset account fields to null, update account fields to null

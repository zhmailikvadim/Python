Symptom

Webhooks for Iventory Transfer are triggering events with the status "Archived", but no requests are sent to the specified webhook URL after managing an Inventory Transfer in SAP Business One client.

Reproducing the issue

  1. Create the Subscription

POST https://service_layer_address:50000/b1s/v2/EventSubscriptions

{
"WebhookID": "StockTransfer",
"WebhookURL": "https://webhook_url",
"AuthenticationType": "None",
"Handshake": "tNO",
"EventCollection": [
{
"BusinessObject": "StockTransfers",
"TransactionType": "Created"
},
{
"BusinessObject": "StockTransfers",
"TransactionType": "Updated"
}
]
}

  1. Create or update an Inventory Transfer in the SAP Business One client

  1. Check the event notifications

GET https://service_layer_address:50000/b1s/v2/EventNotifications

Payload response:

{
"EventID": "d2e80e6d-4d66-4f2d-b77e-a38ff089a1d5",
"SourceDB": "409733",
"Status": "Archived",
"ObjectType": "67",
"BusinessObject": "StockTransfer",
"TransactionType": "U",
"Operation": "Updated",
"FieldsInKey": 1,
"FieldNames": "DocEntry",
"FieldValues": "3",
"UserID": "1",
"ReplayState": "None",
"CreateDate": "2026-03-19T00:00:00Z",
"CreateTime": "05:54:49.306",
"UpdateDate": "2026-03-19T00:00:00Z",
"UpdateTime": "05:54:49.306"
}

 

Expected behavior:

 The notification is triggered but no request is sent to Webhook url.

Actual behavior:

 The notification is triggered and the Webhook url receives the request.

Cause

 Program error.

Solution

SAP intends to provide a feature package or patch to solve the symptom that is described in this SAP Note. The section References below lists the related feature package or patch once they become available. Be aware that References can only be confirmed at feature package or patch release date. SAP delivers feature packages or patches only for selected releases at its own discretion, based on the business impact and the complexity of implementation.

Workaround:

 Manage the Inventory Transfer via Service Layer or Web Client.

Symptom

Below syntax error can be seen after implementing SAP Note 3089413 - [CVE-2023-0014] Capture-replay vulnerability in SAP NetWeaver AS for ABAP and ABAP Platform and transporting it to subsequent systems:

"Program CL_APCRFC_VER_BASE_TEST=======CP, Include CL_APCRFC_VER_BASE_TEST=======CM002: Syntax error in line 000146. 
Method 'SET_CLIENT_CODEPAGE_ACTIVE' does not exist. There is, however, a method with the similar name."

"Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental" 

Environment

  • Remote Function Call (RFC)
  • Note Assistant tool (SNOTE)
  • SAP NetWeaver AS ABAP

Reproducing the Issue

Implement SAP Note 3089413 and transport it to subsequent systems

Cause

The syntax error occurs due to the absence of the method 'SET_CLIENT_CODEPAGE_ACTIVE' in the program CL_APCRFC_VER_BASE_TEST=======CP.

Resolution

Implement SAP Note 3291426 - Avoid Syntax error in CL_APCRFC_VER_BASE_TEST after applying 3089413 to resolve the issue.

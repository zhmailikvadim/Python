Symptom

The Copy To/From feature does not generate a new change log for the base document when creating the target document from the base document, either with full or partial details.

Image/data in this SAP Note is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.

Reproducing the issue

Scenario:

  1. Create a marketing document which serves as a base document. For example: Sales Order, Purchase Order, Inventory Transfer Request etc.
  2. Open the base document created in step 1, update any editable field such as Remarks, Quantity, Unit Price etc., and then click Update.
  3. Go to menu Tools → Change Log... and open the Change Log... window for the base document. Two instance rows are available: one for the original document and second for the changes performed in step 2.
  4. Copy the base document to the target document using the Copy To/From feature, either with partial or full details, and add the document.
  5. After creating the target document, multiple fields or values in the base document are updated automatically. For example: Open Inv. Qty, Open Qty, Delivered Qty, Row Status, Target Type, Document Status, Inventory Status etc.
  6. Open the base document created in step 1. Go to menu Tools → Change Log..., and open the Change Log... window again.

Expected behavior:

A new instance row is displayed in the Change Log... window for the changes performed in step 5.

Actual behavior:

A new instance is not created. Only two instance rows are displayed, as in step 3.

In the Change Log... window of the base document, select the first row, click Show Differences, and open the Differences window.

  • All changes performed during step 5 are displayed along with the previous instance 2 (rows in blue).
  • The Previous Value and New Value data for the Data Version field are inconsistent.

 

The Data Version value is also inconsistent in the Audit Trail Report when the report is run for the base document from Reports → Audit Trail Report.

Cause

System Limitation

Solution

This is a system limitation.

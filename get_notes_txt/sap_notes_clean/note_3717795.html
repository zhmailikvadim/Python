Symptom

When attempting to update an Identity Authentication Service (IAS) user using Identity Directory API, a 400 error occurs.

API Endpoint:
PUT /scim/Users/<SCIM ID> HTTP/1.1

Response Code: 400

“scimType”:“invalidValue”,“detail”:“Both name and value properties must be provided in case of custom attributes attribute.“,”status”:“400”}

Environment

Identity Authentication Service

Reproducing the Issue

  1. Attempt to create or update IAS users using the specified API endpoint.
  2. Observe the error message indicating missing properties for custom attributes.

Cause

The issue occurs due to missing "name" or "value" property in the payload for custom attributes when making the API call. Both "name" and "value" properties must be provided for custom attributes.

Resolution

Review the payload details to ensure that the payload being sent in the API call contains both the "name" and "value" properties for custom attributes. For example, the correct payload format should be:

```

"urn:sap:cloud:scim:schemas:extension:custom:2.0:User": {

"attributes": [

{

"name": "customAttribute1",

"value": "AAAAA"

}

]

}

```

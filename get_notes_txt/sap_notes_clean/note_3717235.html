Symptom

1.) Inconsistent salesRelevant flag values between getConfiguration and getConfiguration(when delta bean is enabled) API for several knowledge bases (KBs).

2.) Inconsistent salesRelevant flag values between getConfiguration and getExternalConfiguration methods for non-classical knowledge bases (KBs) such as FBS_OFFICE.

Other Terms

salesRelevant, getConfiguration, getExternalConfiguration, delta bean, non-classical KB, classical KB, SALES ADT characteristics, FBS_OFFICE, SME_OFFICE, FBS_ADMINISTRATION, SME_ADMINISTRATION, SSCCharacteristicGroupMasterDataPopulator, lastInstance, firstInstance, isSalesRelevant, sales relevance determination, characteristic group visibility

Reason and Prerequisites

Prerequisites:

  • SAP Solutions Configuration (SSC) on-premise system
  • Knowledge bases configured in the system (both classical and non-classical KBs such as FBS_OFFICE, SME_OFFICE)
  • Delta bean feature enabled (for Issue 1)
  • Characteristic groups with SALES ADT characteristics

Reason:

  • When delta bean is enabled, the instance hierarchy may be processed differently, causing the last instance to have different sales relevance properties than the first instance.
  • For non-classical KBs like FBS_OFFICE, the FBS_ADMINISTRATION characteristic group contains SALES ADT characteristics but lacks BOM-related characteristics. Additionally, its instance type is KLAH. This causes: 
  1. ) getConfiguration to return salesRelevant=true (SALES ADT characteristics are present)
  2. ) getExternalConfiguration to return salesRelevant=false

Solution

  • Replaced the reference from lastInstance to firstInstance when determining sales relevance.
  • Replace the conditional logic for setting salesRelevant with a direct reference to the instance's sales relevance value.

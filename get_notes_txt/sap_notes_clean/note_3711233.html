Symptom

When executing program /MERP/CORE_CPMS_APPCREATE_PROG in transaction SE38, the system terminates with a short dump due to a syntax error.

Short Text
    Syntax error in program "/MERP/CL_CORE_DEP_QUEUE_KEY_ODCP

Error in the ABAP application program.

The current ABAP program "/MFND/CL_CORE_OMDO_UTIL=======CP" had to be
 terminated because it found a
statement that could not be executed.
In include "/MERP/CL_CORE_DEP_QUEUE_KEY_ODCO        ", in line 3 of program
 "/MERP/CL_CORE_DEP_QUEUE_KEY_ODCP        ", the following syntax errors
have occurred:
Type "/MERP/CORE_DEP_QUEUE_KEY_BADI" is unknown.

Environment

SAP Service and Asset Manager

Reproducing the Issue

  1. Go to transaction SE38.
  2. Execute program /MERP/CORE_CPMS_APPCREATE_PROG.
  3. The program terminates with a syntax error in program /MERP/CL_CORE_DEP_QUEUE_KEY_ODCP.

Cause

The issue occurs because the BAdI interface /MERP/IF_CORE_DEPQ_KEY_BADI is not activated in the system.

Resolution

Step 1: Activate the BAdI Interface

  1. Go to transaction SE18.
  2. Open the enhancement spot /MERP/CORE_OMDO_DEP_QUEUE_KEY.
  3. Double-click the interface /MERP/IF_CORE_DEPQ_KEY_BADI.
  4. Choose Activate.

Step 2: Re-activate the Dependent Class

  1. Go to transaction SE24.
  2. Open class /MERP/CL_CORE_DEP_QUEUE_KEY_OD.
  3. Activate the class and ensure that the activation finishes without errors.

Step 3: Verify the Fix

  1. Go to transaction SE38.
  2. Re-run program /MERP/CORE_CPMS_APPCREATE_PROG.

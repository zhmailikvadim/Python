Symptom

  • The wage return system generates a new IKV number for each month 

Environment

Netherlands

Cause

The system is unable to locate an end date for the IKV, resulting in the creation of new NUMIV entries each month.

Resolution

  1. Utilize new Infotype 3449 to input relevant details, as the planned end date, to prevent monthly generation of new NUMIV entries.
  2. Maintain IT3449 with the correct IKV number and ensure it is set to prevent further monthly increments. Typically you can use it for Parental Leave, WAZO split. For Transition Allowance as this is about a once time payment you can still use the IT0014/0015
  3. Adapt the payroll schema to include the payroll function P3449 (Import Income Relationship Details).
  4. Rerun payroll after maintaining IT3449 and modifying the schema.
  5. Verify the results in the LAIRW table and the wage return system (e.g., RPCLAVN0).
  6. For cases involving multiple children and new parental leave, create a new record in Infotype with a new IKV and planned end date.

Keywords

IKV number, wage return system, NUMIV entries, parental leave, IT3449, payroll schema, P3449 function, Netherlands, annual tax statements, multiple children, paid parental leave: split type; WAZO

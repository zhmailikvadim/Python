Symptom

In a Revision, when a new promotion is added and the performance period is entered, the promo dates should be populated based on the custom logic implemented in the BAdI method PROMOTION_DEFAULTS_FILL; however, the promo dates are not updated according to the dates changed in this method.

Other Terms

/IRM/IPPQM, document_profile_fill, gs_variables-docpf, promotion_defaults_fill

Reason and Prerequisites

This is caused by Program error.

Please implement SAP Note#3575592 as Pre-requisite.

Solution

Please implement the correction instructions. These corrections are only valid if you have applied the following Support Packs.

Release V/4 – 2023 V3 and 2023 V3 Support Package 1

Use transaction SNOTE to implement the correction instructions specified below and activate the affected program object

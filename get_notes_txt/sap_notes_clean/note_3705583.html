Symptom

A workflow rule configured to send an email notification upon the release of a customer invoice to the processor is not sent to user that created the invoice.

Environment

SAP Business ByDesign

Reproducing the Issue

  1. Navigate to the Application and User Management work center.
  2. Select the Workflow Rules view.
  3. Open the rule XYZ (XYZ represents the name of the workflow rule) .
  4. Open the Rule Invoice Release Notification via the Edit button.
  5. The Basic Data are set up as follows:
    Business Object is Customer Invoice
    Timing is On Every Save
  6. Navigate to the Conditions tab
    The condition is set to Status Equal to Released
  7. Navigate to the Define Actions tab
    The Rule Type is E-Mail and the Recipient Determination is maintained as Processor.

  8. Go to the Customer Invoicing work center.
  9. Select the Invoice Documents view.
  10. Select Customer Invoice ABC (ABC represents the Invoice ID).
  11. The Status is Released.

The Invoice was created by user AAA but the e-mail of the workflow rule was sent to user BBB (AAA represents the user that created the Invoice and user BBB represents the user that approved the Approval task of the Invoice).

Cause

In your system an Approval Process for Customer Invoices is set up, the Invoice ABC is based on a Manual Invoice Request and the Conditions of the Approval Process were met for the Invoice ABC.

The E-Mail for the workflow rule Rule Customer Invoice Release Notification is triggered when the Status of the Invoice changes to Released and is sent to the Processor. The Processor is the user that triggers the Release of the Invoice.

The user AAA created the Invoice but since the approval process was triggered the Invoice was created initially in status Not Released. Since this Status is not set up as a condition in the workflow rule, the e-mail was not triggered and sent to user AAA. Since the approval process was triggered the system created an Approval Task for user BBB. When the user BBB approved the Approval Task the E-Mail notification of the workflow rule was sent to user BBB since this is the processor that triggered the change in the Status of the Invoice to Released. 

When the Approver approves the Task, the System will Release the Invoice and since the Approver is in this case than the person triggering the Release, that user is the Processor and hence the E-Mail will be sent to the person that approved the Approval Task.

Resolution

This is the expected system behavior.

It is not possible to set this up in a way that the E-Mail is sent to the user that created the Invoice initially when the Invoice is released. The only alternative would be that you edit the condition of the workflow rule and add the Status Not Released to ensure that the E-Mail is sent to the processor when the Invoice is created.  

Keywords

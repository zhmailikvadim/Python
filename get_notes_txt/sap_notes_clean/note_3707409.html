  • "Cannot connect to SAP Applications datastore <IBP> using host <IBP host>, client <xxx>, user   <xxx>, and system number <01: CRFC error: RFC_CLOSED- WebSocket Close frame : [SAP Partner]>. Please make sure the SAP server is running and the login information is correct."
  • Issue happens after the migration to KYMA environment.

Environment

SAP Cloud Integration for data services

Cause

Cryptographic keys were shuffled during migration; CI-DS is failing to encrypt/decrypt datastore secrets / authentication such as passwords, resulting in the datastore failure. 

Resolution

 In each Sandbox and Production tenant, create a new Cryptographic Key and remove the keys that previously existed.

  1. Go to the CI-DS Administration tab -> Cryptographic Keys section.
  2. Create a new Cryptographic Key for Sandbox by clicking "New Key" at the right side of the page.
  3. Once the new key is visible in the list, select one of the old keys and choose "Actions -> Revoke" or "Actions -> Delete" (the available action button depends on the status of the key). Repeat this process until all of the old keys have been removed from the list and only the new Active key remains for Sandbox.
  4. Create a new Cryptographic Key for Production by clicking "New Key" at the right side of the page.
  5. Once the new key is visible in the list, select one of the old keys and choose "Actions -> Revoke" or "Actions -> Delete". Repeat this process until all of the old keys have been removed from the list and only the new Active key remains for Production.


The solution might be delayed up to an hour as the cache adjusts to these changes.

In some cases, agent service restart (stop/start) is also needed after waiting an hour and after doing the step 5.

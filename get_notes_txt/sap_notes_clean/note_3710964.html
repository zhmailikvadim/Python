Symptom

The “Default Ord. Ty.” field in the “Define Delivery Types” activity is not editable. Users are unable to create new delivery types in the configuration activity “Define Delivery Types” (ID: 102123), as the “New Entries” button is not available.

Environment

SAP S/4HANA Cloud Public Edition

Reproducing the Issue

  1. Open the Manage Your Solution app or access Central Business Configuration (CBC)
  2. Search for configuration activity 102123 and open it.
  3. Select Define Delivery Types.
  4. Double-click the delivery type LO – Delivery w/o Ref.
  5. Observe that the Default Ord. Ty. field is not editable.

Cause

The inability to edit the “Default Ord. Ty.” field and to create new delivery types in the “Define Delivery Types” activity is standard system behavior by design in SAP S/4HANA Cloud Public Edition. Restrictions are deliberately applied to the underlying customizing view to prevent the creation of new entries in this area. Delivery types are tightly integrated with multiple dependent customizing objects, such as Sales Document Types, Item Determination, and Copying Control, which are not exposed for configuration in SAP S/4HANA Cloud Public Edition. Allowing changes in this area could therefore lead to inconsistencies within the overall logistics configuration.

Resolution

As this functionality is currently unavailable, it is recommended to submit a feature request through the SAP Customer Influence portal. This allows SAP Product Management to evaluate the requirement for potential inclusion in future releases. The submitted request becomes visible to other customers, who can vote on it to increase its priority for future product enhancements.

SAP Customer Influence

Feedback and continued collaboration are greatly appreciated in supporting ongoing improvements to the SAP S/4HANA Cloud solution.

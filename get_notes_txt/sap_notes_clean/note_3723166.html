Symptom

Routing Code and Addressing Line Code are empty in France eDocument XML.

Environment

SAP S/4HANA Cloud Public Edition

Reproducing the Issue

  1. Access eDocument Cockpit app
  2. Select related document
  3. Click on 'Go To' button > Display XML
  4. Empty fields are triggered

Cause

Missing configuration under Business Partner Master Data or Maintain Business Partner app.

Resolution

Regarding the Routing Code and Suffix (Addressing Line Code), find below how these fields are handled in the system.

In the Business Partner (BP) Master Data, under the Identification tab, you can maintain the following parameters:

  • FRADLC – France: Addressing Line Code

  • FRROCO – France: Routing Code

Once these parameters are maintained in the BP Master Data, they are considered active by the system.

In addition, the Suffix can also be maintained in the Billing Document Header Text. This option is typically used in scenarios where multiple suffixes are active for the same Business Partner. Note that when maintained at header level, this information takes priority over the value defined in the Master Data.

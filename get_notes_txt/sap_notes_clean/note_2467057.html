Symptom

  • Message IH047 raised when branching to completion confirmations in IW32/IW33
  • "No object were selected" is raised when trying to view completion confirmations in work orders
  • In IW32/IW33 message IH047 raised in menu path Extras -> Documents for Order -> Completion Confirmations
  • No completion confirmations found in IW32/IW33 although they exist in IW42

Environment

  • SAP ERP Plant Maintenance (PM)
  • SAP R/3
  • SAP R/3 Enterprise
  • SAP ERP Central Component
  • SAP ERP
  • SAP Enhancement package for SAP ERP
  • SAP Enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA, on-premise
  • SAP S/4HANA Cloud Private Edition

Reproducing the Issue

  1. Enter order number in IW32/IW33
  2. Choose menu path Extras -> Documents for Order -> Completion Confirmations
  3. Message IH047 "No objects were selected" is raised
  4. Check IW43 - Confirmations exist but are not shown in IW32/IW33

Cause

When the system tries to access the confirmation documents, an authorisation check is carried out on object I_IWERK. However, in transaction SU22 it can be seen that for transaction IW47, object I_IWERK is set to 'Do No Check'. Whereas for transaction IW32/IW33 the object is set to 'Check' and hence different behaviours occur within IW32/IW33 and IW47.

Resolution

This is standard system behaviour and will not be changed. However the following can be done to adjust the behaviour:

If you want IW32/IW33 to behave as IW47 behaves:

  • Give the users corresponding authorization for authorization object I_IWERK (parameters TCODE and IWERK)
  • Deactivate authorization check I_IWERK for "calling" transaction IW32/33. This can be done via transaction SU23. Adjustments of transaction SU23 (customer specific) have higher priority as adjustments of transaction SU22 (SAP standard)

If you want IW47 to behave as IW32/IW33 behaves:

  • Activate authorization check I_IWERK for "calling" transaction IW47. This can be done via transaction SU23. Adjustments of transaction SU23 (customer specific) have higher priority as adjustments  of transaction SU22 (SAP standard)

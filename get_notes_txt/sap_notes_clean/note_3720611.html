Symptom

You experience one of the following symptoms:

  • In production system (which is locked against the customizing changes), you post document via manual posting task. After saving the data, the system raises a popup "Error in Object Editing" with message "Client XXX has status 'not modifiable'".
  • You open the customizing of a manual posting task in transaction UCWB. The system raises popup to save changes when you are leaving this customizing, even there was no change done.

This issue can mostly happen with manual posting task, but could possibly happen with other task as well.

Other Terms

UGMD 001

Reason and Prerequisites

The issue is caused by program error.

Solution

Implement the attached correction instruction or import the specified Support Package.

Symptom

  • The status of the Purchase Order is shown as "Created" on Step 4 of Sourcing Cockpit, even if the Purchase Order has failed to be created in the back-end system.
  • The Document Number isn't displayed as a link, even if switch SRM_701_LINK_TO_ERP is active.
  • System shows zero in column Total Value.

Environment

  • SAP Supplier Relationship Management (SRM) 7.0 and higher
  • SAP enhancement package for SAP Supplier Relationship Management
  • Classic scenario with SOA

Reproducing the Issue

  1. On Step 1 in Sourcing Cockpit search for a Shopping Cart that belongs to Classic scenario
  2. Create a PO draft for this Shopping Cart on Step 2
  3. Click "Process All Draft" or "Process Selected Drafts" button on Step 3

Cause

By that time you reach Step 4 of SOCO, the PO in back-end system may or may not have been created. The reason for this is that the PO creation via SOA is asynchronous and depends on also queuing mechanism of the system. For example the XML could have been scheduled or the xml confirmation from backend would not have been reached to SRM. Because of this it would not be possible to wait for the system to fetch the details from the back-end system. Therefore from Sourcing cockpit, when Purchaser triggers the backend PO creation and an XML is generated that means the PO creation process has been initiated from SRM. Hence it shows the status as "Created". Since the process in asynchronous, it is beyond control to check whether the generated XML has been successfully processed or not. Because of this reason it is also not possible to display the PO from Step 4, therefore the Document Number is not displayed as link. Moreover it is also not possible to gathered the Total Value of the PO.

Resolution

The system works as per SAP SRM standard design.

Keywords

EBP, SRMSTD, REFRESH, /SAPSRM/CL_CH_WD_DOM_SOCO_GAF4, ls_created_obj-status_description = 'Created'(019)., SOA, asynchronous

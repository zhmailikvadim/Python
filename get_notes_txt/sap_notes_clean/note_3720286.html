Symptom

In the process "UOP: Receiving Order for Changing Billing Data" (DE_B_UOP_ORDR_BILL_CHG_RECV), you cannot redefine steps 0220 and 0250 with your own embedded process.

This issue is relevant for the market role: DSO.

Other Terms

UOP, Master Data Update, Extensibility, MDX Embedded Process, DE_B_UOP_ORDR_BILL_CHG_RECV, BC set

Reason and Prerequisites

Program error. This error occurred because the process step type didn't allow you to extend the steps.

Solution

Implement the corrections of this SAP Note by importing BC set /US4G/3_0_DSO_SP08_NOTE_3720286 or the relevant Support Package (SP).

After implementing this SAP Note, you can redefine steps 0220 and 0250 with your own embedded process.
You can verify the fix by checking that you can successfully redefine steps 0220 and 0250 with your own embedded process.

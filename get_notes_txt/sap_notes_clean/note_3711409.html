Symptom

You have created a custom field with a Default Mapping of Direct Reference and you would like to understand how it behaves during the runtime.

Environment

  • SAP Sales Cloud Version 2
  • SAP Service Cloud Version 2

Reproducing the Issue

Note that this is just an example and it can be applied to various scenarios.

  1. Click on the User Icon.
  2. Navigate to Settings > All Settings.
  3. Proceed to the Extensibility Administration.
  4. Select Account Service > Account.
  5. Go to the Custom Fields tab.
  6. Create a Custom Field (e.g., ABC).
  7. Select Lead Service > Lead.
  8. Navigate to the Custom Fields tab.
  9. Create a Custom Field (e.g., DEF) with a Direct Reference to the field created in the Account service (e.g., ABC).

Resolution

Direct mappings perform a one-time copy of values from the parent record to the child record at the moment of mapping. After that initial copy, the child fields are independent—changes made later on the original source entity (e.g., Account) do not automatically update the corresponding fields on the target entity (i.e., Lead).

For example, suppose you have created a Lead from a particular Account. Those fields have properly been copied from the Account to the Lead. However, if those fields are subsequently changed in the Account, they will not be updated in the Lead. The child fields are separate and do not need to follow the parent value.

The only exception to this behavior is when the Account associated with the Lead itself is changed (for example, the Lead's Account lookup is switched to a different Account). In that case, the Lead's mapped fields are updated from the newly associated Account. This aligns with the documentation phrase about associated entity references, which refers to updates that occur when the mapped entity association changes, not when the original parent record is edited after mapping.

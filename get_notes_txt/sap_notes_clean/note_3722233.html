Symptom

It is not possible to create or update a Purchase Order using API API_PURCHASEORDER_2 when partner functions are included in the request payload.

The system returns error "Method not allowed" and the Purchase Order is not created or updated successfully.

Environment

  • SAP S/4HANA
  • OData APIs - Purchase Order

Reproducing the Issue

  1. Send a POST or PATCH request using OData service API_PURCHASEORDER_2.
  2. Include partner function data in the request payload.
  3. Execute the request.
  4. The system returns an error and the PO is not created, or creates the PO without partner data.

Cause

Currently, the OData service API_PURCHASEORDER_2 does not support the creation or update of partner data in Purchase Orders on any method.

This limitation is documented in the official SAP Help Portal under the API documentation for Purchase Order partners:

SAP Help Portal – Partner (API_PURCHASEORDER_2)

Resolution

This behavior is a known functional limitation of the OData service API_PURCHASEORDER_2. Posting or updating partner functions in Purchase Orders via this API is currently not supported.

If maintenance of partner data is required, it must be performed manually in the Purchase Order application.

Alternatively, other integration channels such as Enterprise Services (ESOA) or relevant BAPIs can be used, as they support the update of partner data in Purchase Orders.

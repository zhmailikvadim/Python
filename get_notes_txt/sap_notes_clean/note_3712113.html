Symptom

During schema import, the following error occurs: "SAP DBTech JDBC: [2048]: column store error: import failed: [30130] Could not import cs_lob attribute;Update lob ids for table "XXXX"."CGEV" failed: Allocation failed $REASON$".

Environment

SAP Business One version for HANA

Reproducing the Issue

Scenario:

  1. Export the Schema on the source system and compress the exported files in the Linux server.
  2. Transfer the compressed file to the target system.
  3. Extract the exported schema on the target system.
  4. Attempt to import the schema.

Expected behavior:

Schema import is successful.

Actual behavior:

Schema import fails with following error:

"SAP DBTech JDBC: [2048]: column store error: import failed: [30130] Could not import cs_lob attribute;Update lob ids for table "XXXX"."CGEV" failed: Allocation failed $REASON$".

Cause

The table CGEV allocates excessive space in the company database, leading to import failure.

Resolution

Refer to SAP Note 3078442 to disable the saving of User-Defined Query logs in the table CGEV. If retaining User-Defined Query-related logs is unnecessary, delete data in the table CGEV. 
Note: Deleting data in the CGEV table does not affect users but will result in the loss of Change Logs for query-related operations (similar to ADOC and ADO1 tables for documents). 

Keywords

import, CGEV, allocation, failed

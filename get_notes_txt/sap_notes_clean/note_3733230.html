Symptom

  • When running the Colombia VAT Declaration report (CO_VAT_DCL) in the "Run Statutory Reports" app, the "Net Tax Payable" amount is displayed incorrectly.
  • The value appears without a decimal separator, resulting in a significantly larger amount than expected (e.g., 6,600,834,210.05 is displayed as 660083421005).
  • Other amounts in the summary, such as "Total Input Tax" and "Total Output Tax," are displayed correctly with decimals.
  • This issue occurs when the company code’s currency is configured with decimal places.

Environment

  • SAP S/4HANA
  • SAP S/4HANA Cloud Private Edition

Reproducing the Issue

  1. Open the Run Statutory Reports app.
  2. Select the Colombia VAT Declaration report (CO_VAT_DCL).
  3. Choose the relevant reporting period and reporting entity, then run the report.
  4. Open the generated activity “Tax Return Details for Colombia”.
  5. Open the document “Tax Return Summary and Details”.
  6. In the “VAT Returns for Sales and Purchase – Summary” section, observe that “Net Tax Payable” is displayed without a decimal separator and does not match the input/output tax totals.

Cause

The standard statutory reports for Colombia are designed with the local requirement that amounts are reported without decimal places. The formatting logic within the report for the "Net Tax Payable" field specifically removes decimal separators to meet this requirement. This causes a discrepancy when the underlying company code currency is configured to use decimal places. 

Resolution

  • This is the standard and expected system behavior for the country-specific report for Colombia. The report is not designed to handle currencies with decimal places for the final declared amounts.
  • The recommended solution is to use the "Generic VAT Report", which provides the necessary flexibility to handle various currency configurations.

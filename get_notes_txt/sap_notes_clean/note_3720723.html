Symptom

When attempting to generate a Single Touch Payroll (STP) file using T‑CODE PC00_M13_STP2_GEN for Australian employees, the process fails and the error message "Cluster Table ACRT is inconsistent" occurs.

Environment

  • Human Capital Management (HCM)
  • SAP ERP
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP SuccessFactors Employee Central Payroll
  • SAP S/4HANA

Reproducing the Issue

  1. Execute T‑CODE PC00_M13_STP2_GEN.
  2. Select the employees and start STP2 file generation.
  3. The system returns the error: "Cluster Table ACRT is inconsistent".

Cause

Resolution

  1. Test run report Migration Utility for STP Phase2 Australia (T-CODE PC00_M13_CLST_MYGL2) for the employee with error and validate the output.
  2. Live run the same report and check the output of STP2 report again.

Note: When the financial year to submit STP2 report is 2025-26, input the financial year in Migration Utility report as 2026.

See Also

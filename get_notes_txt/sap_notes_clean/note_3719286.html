Symptom

The following error occurred while generating RFC destination SAP_UPGRADE_ORIG_SYSTEM in the upgrade phase RFCDEST_CREATE_SHD2ORIG_PWD.

*********

1EETQ235 Call of function module "RFC_DESTINATION_CREATE" via RFC failed (error-status "4")!
2EETQ360 RFC of "RFC_DESTINATION_CREATE" failed:
2EETQ361 code/exception  : 4
2EETQ362 key             :
2EETQ399 00024error during
2EETQ399Xlogon##############################################################

*********

Environment

SAP ABAP System

Reproducing the Issue

Retry the upgrade phase RFCDEST_CREATE_SHD2ORIG_PWD.

Cause

SNC access to the source system triggered by SUM for shadow instance is missing in SNCSYSACL.

Resolution

    To resolve the issue:

    1. Call the transaction SNC0 in the original system.

    2. Add one entry with type "E" for the access from the shadow instance Application Server.

    3. Retry the failed phase again in SUM.

    Keywords

    s4hana upgrade, preprocessing phase, MAIN_SHDRUN, RFCDEST_CREATE_SHD2ORIG_PWD, RFC logoff error, RFC_INVALID_HANDLE, shadow system locked, DIR_PUT, AL11, RZ11, SNC error, snc/enable, SNC0 transaction, SAP GUI logon, SAP S/4HANA, SUM, Software Update Manager, SNCSYSACL.

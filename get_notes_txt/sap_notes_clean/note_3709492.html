Symptom

You are trying to display the document/attachment in the browser.

Other Terms

Connection failed, Browser compatibility, Content Disposition, URL

Reason and Prerequisites

The issue occurs because the URL is accessed directly on the browser and SAP back-end system may not be involved in the communication.

Solution

Knowledge Provider (KPro) stores the documents in the Content Repositories created in the SAPGUI transaction 'OAC0'. These Documents are accessible within the SAP system via KPro APIs. When the documents are to be downloaded/displayed on the Frontend, SAP uses direct communication where KPro generates a URL which can be accessed from the user's frontend/web browser (Chrome/ Edge/ Safari)

When the URLs are used on the frontend, the customer should have access to the Content server host.

When SAP triggers a URL to access content, the browser independently handles the retrieval and processing of that content. The communication occurs directly between the front-end (browser) and the content server, without SAP mediating the interaction.

This means

  • SAP Support Scope: Issues related to accessing or reading content from the content server are managed by SAP support.

  • Customer Responsibility: Problems concerning URL access, network connectivity, or content rendering in the browser after the URL is triggered on the browser fall under the customer's responsibility. Customers should verify these issues with their content server vendor.

Document Display and Compatibility:

The display and handling of documents after the URL is triggered depend entirely on the browser's compatibility and settings, which are beyond SAP's control. Customers experiencing issues with the generation/creation of the URL can contact SAP support, BC-SRV-KPR-CMS component. And once the URL is fired over to the browser, any issues related to URL reachability, network connectivity, browser behaviour, or content rendering are your responsibility.

Content-Disposition:

The HTTP response header Content-Disposition determines how a browser handles returned content.

Reference: https://developer.mozilla.org/en-US/docs/Web/HTTP/Reference/Headers/Content-Disposition

The Content Server can set the HTTP response header Content-Disposition, which instructs the browser how to handle the returned content.

  • Content-Disposition: inline
    Displays the content directly within the webpage or inside an embedded viewer (Default behavior).

  • Content-Disposition: attachment
    Forces the browser to download the file instead of displaying it.

To resolve display-related issues, ensure that the Content Server is sending the appropriate Content-Disposition value in the response.

  • For SAP Content Server, refer to SAP Note 842709 – SAP Content Server: Setting the Content-Disposition Response Header.

  • For OpenText Core Archive, refer to SAP Note 3330802 - Core Archive for SAP Solutions - Core Archive Connector - Parameters CONTENT_DISPOSITION_TYPE and CONTENT_DISPOSITION_EXCEPTION_LIST in ICS.Setup

  • 2774742 - Cross-Site Scripting (XSS) vulnerability in ABAP Server and ABAP Platform

FAQ's:

1. Document cannot be displayed in the SAPGUI HTML Viewer and a connection error occurs. What should be checked?

Answer: If an HTTP-based Content Server is used, first verify whether the Content Server can be reached directly from the frontend browser.

This can be tested by calling the ServerInfo URL in a standard web browser:

<protocol (HTTP/HTTPS)>://<Content Server Host>:<Port>/<Script>?serverInfo&pVersion=0045

Example:

https://www.testcontentserver.com:883/bc/server/000?serverInfo&pVersion=0045

Result Interpretation:

  • If the ServerInfo page is displayed successfully, basic network connectivity to the Content Server is working.
  • If the page cannot be accessed, check network, firewall, proxy, or DNS configuration between the frontend and the Content Server.

2. PDF and PNG files open directly in the browser, but DOCX files are downloaded. Is this expected behaviour?

Answer: Yes. This behaviour is expected and is controlled by the browser.

Most modern browsers provide native inline rendering for file formats such as:

  • PDF
  • PNG
  • JPG

However, DOCX files are not supported for inline display by standard web browsers. As a result, browsers automatically download DOCX files instead of displaying them within the browser window.

This behaviour is determined by the browser’s capabilities and cannot be influenced by SAP.

For details on supported file formats and rendering behaviour, consult the documentation of the respective browser vendor or Microsoft.

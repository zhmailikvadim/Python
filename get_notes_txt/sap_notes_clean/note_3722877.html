Symptom

Multiple ODOs are not completed, a user tries to compete those ODOs with the process code O001 massively via the tcode /scwm/mon, the log shows the message "XXX items where the quantity adjustment did not change the quantity".
However, the user can complete those ODOs with the process code O001 via the tcode /SCWM/PRDO.

Environment

Extended Warehouse Management (EWM)

Reproducing the Issue

  1. Access the tcode /scwm/mon and display ODOs.
  2. Apply the process code O001 for those ODOs massively.
  3. Observe that the system fails to process and update the quantity.
  4. Display ODOs via the tcode /scwm/prdo and apply process code O001, the quantities are updated successfully.

Cause

Missing of notes.

Resolution

Implement the following notes if they are missing in the system.

3245183 - Performance issues during confirmation of pick warehouse tasks for deliveries with many items
3424770 - Warehouse Monitor - errors during adjustment of quantities for items in mass.
3470227 - Messages for Adjust Quantity & Create OD When OD Is Existing.
3505103 - Error "The delivery item already has the status 'Completed'." returned during delivery quantity adjustment with zero quantity.
3465689 - When several line items are selected of an inbound delivery, no process code can be applied.
3341561 - Dump during adjusting quantity of a value item to zero.
3123029 - Faulty call in /SCDL/CL_DLV_ADJUST_QUANTITY~ACT_EXECUTE_ITEM.
3102391 - OPC: serial numbers remain in /SCWM/SERI after adjusting to zero.

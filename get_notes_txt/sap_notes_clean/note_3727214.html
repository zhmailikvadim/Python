Symptom

  • Validation error: The content model does not allow element <attachments> immediately after <additionalInfo>. Expected <additionalInfo> or nothing.” in DRC.
  • Error occurs during XSD validation against the schema `/xsd/generic-push-service.xsd` used by the DCPPUSHSERVICE interface.

Environment

  • SAP ERP
  • SAP S/4HANA
  • SAP Document Reporting and Compliance Service

Reproducing the Issue

  1. Push vendor invoice 20260055 from SAP BTP.
  2. Observe that during the PUSH from DRC to the ERP Central Component, an XSD validation error occurs.
  3. The error message displayed is: "The content model does not allow element `` immediately after ``. Expected `` or nothing."

Cause

The error is caused by an incorrect XML element sequence in the submitted vendor invoice file.

Resolution

  1. Review the XML structure of the vendor invoice file with the vendor.
  2. Ensure that the element order strictly follows the schema definition outlined in `/xsd/generic-push-service.xsd`.
  3. During XSD validation against generic-push-service.xsd, the system detected that the element <attachments> appears immediately after <additionalInfo> inside the <document> node. According to the schema definition, this structure is not allowed. After <additionalInfo>, the schema expects either another <additionalInfo> element or the end of the sequence  not <attachments>.
  4. Update the XML payload to align with the schema definition and reattempt the PUSH operation.

Symptom

Preliminary steps to perform in ECC, before migration/conversion to SAP S/4HANA. So that it can avoid below issues in SAP S/4HANA.

Project or WBS element IDs appear without coding mask in UI.

Error CJ726 or CJ727 raised in below scenarios.

  • Try to change a project via Project Definition Overview app.
  • Use CNMASS to update a WBS or Project Definition.
  • Use BAPI_BUS2054_CHANGE_MULTI or BAPI_PROJECT_MAINTAIN to update a WBS or Project Definition.

Other Terms

Project, WBS Element, coding mask, WBS Element overview app

Reason and Prerequisites

To avoid the initial values of PRPS-POSID_EDIT and PROJ-PSPID_EDIT.

Solution

Prerequisites:

Ensure all SAP Notes relevant for your release are implemented:
- 2493182 (SAP S/4HANA)
- 3115066 (ECC 6.0)

Steps to fill empty PRPS-POSID_EDIT and PROJ-PSPID_EDIT fields:

  1. Go to transaction se38 in the backend ECC.
  2. Run the report "R_FILL_POSID_PSPID_EDIT".
  3. Execute with test checkbox "checked" - as a test run. It will just show the amount of table records that will be modified, when running the report in update mode.
  4. Now, execute the report again with test checkbox "unchecked" - it will actually change the entries.
  5. It will show the number of records successfully modified.

Additional info: If you miss to run the report in ECC, you can run the report in SAP S/4HANA as well once the migration is performed.

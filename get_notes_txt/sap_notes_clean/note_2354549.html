Symptom

You want to log read access to certain personal data.

Other Terms

Data privacy, data protection, sensitive data, RAL, SRALMANAGER, configurations, channels

Reason and Prerequisites

This SAP note is relevant only if the application component FI-CA is being used. This is the case if one of the following Business Function Sets is active (see transaction SFW5):

  • Utilities
  • Insurance
  • Contract A/R + A/P
  • SAP Media
  • Telco
  • Industry Extension Public Sector

In order to achieve read access logging of personal data you are required to use SAP Read Access Logging (RAL) as a framework (see SAP note 2347271 for requirements).

In addition to the technical framework, RAL content (log domains, logging purposes, recordings and/or configurations) which are used by RAL at runtime are required.

Solution

With this note, SAP delivers example content for RAL in the application component FI-CA covering the channels SAPGUI Dynpro, RFC, SAP Gateway (oData/UI5/Fiori) and WebServices.

The example content is based on legal definition of "special categories of personal data". This content is not necessarily complete and customer is requested to check if the example content covers their requirement. The content contains configurations for the following log domains:

  • BANK for Bank Account Data
  • CREDIT_CARD for Payment Card Data
  • PROFILE for Creditworthiness Data 

See SAP note 2363367 for details on how to import the RAL content. Import the attachment which fits to your current release.

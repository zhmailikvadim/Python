Symptom

You are requested to enable and provide ODBC Traces for troubleshooting Smart Data Access issues in SAP HANA to a remote DB2 database.

Other Terms

Federation; DB2CLI;

Reason and Prerequisites

Reason:

In order to further analyze a reported issue for a SDA setup, ODBC traces of the communication between HANA and the remote DB2 database are required.

Solution

Follow the steps below to enable ODBC traces on OS level:

Enable DB2 CLI ODBC Trace

  1. Locate the db2cli.ini file, by default located inside the <sid>adm home directory.
  2. Open the db2cli.ini file in a plain text editor as <sid>adm.
  3. Add the following section to the db2cli.ini file or, if the [COMMON] section exists, append the CLI trace keywords in the following example:
    • ﻿[COMMON]
    • Trace=1
    • TracePathName=path
    • TraceComm=1
    • TraceFlush=1
    • TraceTimeStamp=1﻿
  4. Restart HANA DB to make the change effective.
  5. Reproduce the issue.

 

Disable DB2 CLI ODBC Trace

To disable the ODBC traces, set the parameter "﻿Trace" back to "0" inside db2cli.ini and restart HANA DB afterwards.

  • ﻿[COMMON]
  • Trace=0﻿

 

Remarks:

  • HANA DB restart is required
  • Further details can be found here: Tracing the CLI driver by using the db2cli.ini file

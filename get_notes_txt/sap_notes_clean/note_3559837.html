Symptom

You use SAP ACM, the drug order overview and the universal prescription to manage and issue electronic medication orders.

  • In the medication order overview, you want to use quick filters with a few clicks to see which prescription requests for parenteral preparations have been received or in which electronic orders the signature is pending.
  • In the ePrescription order data record, you want to specify a batch number and an expiration date.
  • With document category version 3 of the universal prescription you wish to assign electronic orders to a pharmacy.

Other Terms

KIM ePrescription, AMVO

Reason and Prerequisites

Functionality enhancement

Solution

This SAP Note provides the following function enhancements:

  1. Quick Filter in the Drug Order Overview

    The following quick filters are available in the drug order overview:

    • My Open Prescription Requests: When you select this filter, the list of orders is filtered to orders that are still to be signed and that have been received using KIM. In addition, the EmR of the logged-on user is transferred to the selection parameters, provided this is not already defined. If an OU is stored in the selection parameters, this is removed so that all his or her open prescription requests are displayed to the user.

    • Open prescription requests of the OU: When this filter is selected, the list of orders is filtered to orders still to be signed that were received using KIM. In addition, the OU from the view from which the order overview was called is transferred into the selection parameters. If an EmR is previously defined in the selection parameters, this is removed so that all open prescription requests are displayed to the user.

    • Open E-orders: When this filter is selected, the list of orders is filtered to orders still to be signed. As a result, the system displays the orders pending for signature to the user.

  2. Specification of batch number and expiration date in ePrescription order data record

    In the universal prescription document category version 3, you can enter the batch number and the expiry date of an order for an ePrescription-enabled order in the detail area.

  3. Assignment of ePrescriptions to a pharmacy

    You can use the version of the universal prescription to determine the redemption route for an ePrescription-enabled order. The standard for each order entered is "Patient". By clicking in the field of the relevant order column, you can use a dialog to change the redemption route. You have the following options for selecting alternative redemption routes:

    • Assignment
    • Direct Assignment

You must enter a KIM e-mail address for the redemption routes Assignment and Direct Assignment. You can enter the KIM mail address as free text. To support the entry of the KIM mail address, the search in the directory service of the TI (VZD) is possible. You can use the search criteria to search for KIM mail addresses in the VZD. The results are displayed at the bottom of the dialog. Similarly, you can select a KIM mail address from the Recent tab.

If you choose Assignment or Direct Assignment, the system sends the corresponding ePrescription to the selected recipient using KIM. When the direct assignment redemption route is selected, the patient is deprived of further decision-making authority over the order. The order is then placed in the TI with the workflow type 169.

Please note: When and whether an assignment or direct assignment of ePrescriptions is legally permitted cannot be checked by the software. It is up to the ordering person whether an assignment/direct assignment is allowed in the current pension context.

The following adjustments must be made or checked in the Reference IMG:

Define the LDAP server for the search in the VZD in Customizing under:

SAP Healthcare - Industry-Specific Components for Hospitals -> SAP Ambulatory Care Management -> Treatment -> Outpatient Clinic Forms -> Electronic Outpatient Clinic Forms -> Configure Electronic Prescription.

You can enter OU-related entries for an institution. If you enter several OUs, you must select an LDAP server for each OU.

The functions are available with the specified Support Packages.

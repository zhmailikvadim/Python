Symptom

Legacy Import Error NullPointerException : Cannot invoke "ariba.base.fields.FieldProperties.getProperties()" because "fp" is null

Resolution

This error arises because of incorrect header name used in Contracts.csv file. This file is highly case-sensitive so the header name should be in the correct syntax as per Standard Legacy Import Format. The header names should be the backend name of the contract workspace fields. Few key points as below : 

  • The backend field name of the field Related ID is RelatedId. 
  • Custom fields will be defined as cus_FieldName.
  • There should be no space at the end of the header name used in Contracts.csv.

You can refer the article link for other Legacy import errors Other Common Errors Related to Legacy Import

See Also

As the backend field names is not available and accessible to end users, please request your Designated Support Contacts (DSC) to submit a Case, and SAP Ariba Customer Support will provide the requested backend field names.

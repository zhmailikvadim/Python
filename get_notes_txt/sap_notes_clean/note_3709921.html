Symptom

When attempting to access New Supplier Profile Summary in SAP Ariba Supplier Lifecycle and Performance, a few users encounter issues accompanied by the following warning:
  • To sign in to the application you have to grant the application access to your browser storage. By pressing Continue, a new popup will appear where you can grant the access. If you block this you will not be able to sign in.
  • You have either blocked access to your browser storage or have never opened the current page before. To continue, please go to your browser settings and reset the embedded content permissions, or open the confirmation page.

Reproducing the Issue

  1. Navigate to the supplier management section in SAP Ariba Supplier Lifecycle and Performance.
  2. Attempt to view the supplier profile.
  3. Observe the error message regarding browser storage access.
  4. Note that the page results in a blank screen.

Cause

This is a browser storage / embedded content / popup permission issue, and end users usually hit it because cookies, local storage, or pop-ups are blocked (often by privacy settings or extensions).

The application needs:

  • Cookies & browser storage (local/session storage)

  • Pop-ups or embedded content permissions

  • Sometimes third-party cookies (if auth is embedded or redirected)

If any of these are blocked, sign-in fails.

Resolution

Few browser specific settings to check and update are as below.

Settings for Microsoft Edge (Chromium) browser.
 
Allow cookies & storage
  1. Open Settings
  2. Go to Privacy, Search and Services.
  3. Click Cookies.
  4. Make sure: Allow sites to save and read cookie data is ON and Block third-party cookies → Turn OFF (or add an exception)
 Reset permissions for your site
  1. Open the app page
  2. Click the lock icon in the address bar
  3. Select Permissions for this site
  4. Click Reset permissions
  5. Reload the page
Allow pop-ups
  1. Open Settings
  2. Go to Privacy, Search and Services.
  3. Click Site Permissions.
  4. Click All Permissions.
  5. Click Popups and Redirects.
  6. Add the application.

Settings for Google Chrome browser.

 
Enable cookies & storage
  1. Open Settings
  2. Go to Privacy and security → Third-party cookies.
  3. Select: Allow third-party cookies.
Reset site permissions
  1. Open the app page
  2. Click lock icon → Site settings
  3. Click Clear data and Reset permissions
  4. Reload
Allow pop-ups
  1. Go to Privacy and security → Site Settings → Pop-ups and redirects
  2. Add your app URL to Allowed

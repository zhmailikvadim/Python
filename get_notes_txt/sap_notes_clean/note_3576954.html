Symptom

  • An error occurs during the ingestion process from the CSV file.
  • Example of an error message displayed: “Column header mismatch. Expected 75 columns but got 74”.
    • This error states that the CSV file contained only 74 headers (representing the fields that are ingested) but there are 75 fields mapped in the SFTP source event.

Environment

  • SAP Customer Data Platform
  • Schedule ingest
  • Connectors

Reproducing the Issue

  1. Create a data structure in the SAP CDP source event "Model" step.
  2. Ensure the total number of fields and the field names in the mapping match those of the CSV file header columns.
  3. Ingest the file(s) to process records.

Cause

  • The error may be due to a mismatch in the number of columns in the data structure and the CSV file.  The system expects X columns that represents the fields in the event, but Y columns are provided.
  • The error can also occur if there are values containing special characters (e.g.: backslash, single double quotes, etc.) but were not properly escaped.

Resolution

  • Review the data model fields and the header columns from the CSV file to ensure there are no discrepancies in the number of columns and field names.
  • Use backslash in front of the special characters to ensure they are properly escaped (e.g.: for backslash \\, single quote \', double quotes \", etc.).

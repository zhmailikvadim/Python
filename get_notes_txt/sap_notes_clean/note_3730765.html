Symptom

  • In Story reports, picklist-based fields (for example, City) from the country-specific Address object display blank values although correct data is visible in the UI.

Environment

  • Story reports
  • Module configuration

Reproducing the Issue

  1. Create a new Story report.
  2. Add the picklist-based address field (for example, City) from country specific address object
  3. Filter to a user with known address data.
  4. preview query and observe the field shows blank, while the same data is present in the UI.

Cause

A special (non-ASCII) character in the picklist Code and/or Legacy Picklist ID used by the address field prevents Story reports from resolving the picklist label, resulting in blank values.

Resolution

  1. In Picklist Center, open the affected picklist used by the address field (for example, the City picklist).
  2. Edit the picklist and update the Code and Legacy Picklist ID to remove special characters; use ASCII-only characters.
  3. Save the picklist changes.
  4. In Provisioning, run “Refresh BIZX Reporting Metadata” to regenerate the reporting calculation views.
  5. Reopen the Story report and validate that the picklist labels now display correctly.

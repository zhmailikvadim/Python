Symptom

  • A user encounters an issue where they cannot log in to any CALM tiles despite having full admin access.
  • The error messages displayed include:
  • [Error: 500] 3:27:14 PM GMT+08:00 - Internal server error[undefined undefined] ERROR_READING_FROM_LMS 3:27:16 PM GMT+08:00
  • The issue was initially reported for one user but later escalated as more users reported similar problems.

Environment

  • SAP Cloud ALM
  • SAP Cloud Identity Services

Reproducing the Issue

  1. Logon to Cloud ALM tenant
  2. Attempt to access any of the Application tiles  
  3. User has full admin access but the following error message gets displayed:
  • [Error: 500] Internal server error[undefined undefined] ERROR_READING_FROM_LMS

Cause

The issue occurs when the logon token of a user contains multiple values for a certain claim, such as the email address. This leads to the processing of the logon token failing with unusual side effects.

Resolution

  1. Review the logon token content in the IAS troubleshooting section. Refer to: IAS Troubleshooting Guide.
  2. Check the SAP BTP subaccount SAP Cloud ALM configuration for multiple email attributes defined under Bundled Apps.
  3. Delete any duplicate or conflicting email attributes from the IAS tenant.
  4. Confirm that users can log in successfully after the adjustments.

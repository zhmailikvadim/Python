Symptom

Despite setting the Date field to display only IDs in the prompt window, the system defaults to displaying both ID and Description when reopening the story.

Environment

SAP Analytics Cloud

Reproducing the Issue

  1. Open a story based on a Datasphere live model that have prompts enabled.
  2. Open the variable selector.
  3. Change the Display setting for any Date field to ID and confirm.
  4. Close and reopen the story.
  5. Open the variable selector for the same date field again and observe that the Display setting reverted to ID and Description.

Cause

  • This behavior is by design.
  • Persistence of the display selection for date-type dimensions in the member/variable selector is not supported.

Resolution

To request a new feature or product enhancement, SAP has defined a new process and tool that is now available to customers called Idea Place: https://influence.sap.com/sap/ino/. Idea Place gives customers the ability to log their own Enhancement Requests and work more directly with our Technology and Development group.

The processes and procedures associated with this site will continue to evolve and enhance over time. Refer to Knowledge Base Article 1515837 for more details.

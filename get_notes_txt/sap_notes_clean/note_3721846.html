Symptom

Replication flow fails with the error:  failed to create virtual table: SQL Error 476 - invalid remote object name: Unable to retrieve remote metadata for <table_name>.

Environment

SAP Datasphere

Cause

The issue was caused by an inconsistency in the database name during the process of publishing the delta share from the source system to the target system.

Resolution

In order to resolve the issue, follow the steps below:

  1. Review the process of publishing the delta share from the source system to the target system to identify any discrepancies in the database name.
  2. Create a new data product from the source system and publish it to the target system.
  3. During the installation of the data product, ensure that the data storage is set to remote.
  4. Verify the replication flow execution and confirm that the data can be virtually read from the source system.

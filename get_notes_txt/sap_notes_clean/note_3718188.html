Symptom

I am attempting to perform a data upload, but the transaction fails with a "Required data is missing" error for a specific field. Although I have confirmed that the necessary data is present in my upload file, the error persists. I noticed that the column header in my file matches the customized label I see in the UI, yet the error message refers to a different, native system name.

Cause

This error occurs when the native labels in the SAP Fieldglass UI have been renamed using Custom Text (e.g., renaming Cost Center to Project or Business Unit to Department).

While the UI displays your preferred name, the underlying connector expects the upload file to use the system's Standard Column Names. If the file uses the customized label instead of the native one, the connector cannot identify the data and triggers a "missing data" error.

Resolution

You must map your customized file headers to the native system fields using Upload Column Transformation.

  1. Log in with your Configuration Manager account.
  2. Navigate to Connector Catalog.
  3. Search for the impacted connector and click Apply Filters.
  4. Open the connector and click Edit.
  5. Go to the Upload Column Transform section under the Customize header.
  6. Click + Add Native Fields.
  7. Enter the mapping details:
    • Standard Column Name: Select the native name mentioned in the error message.
    • File Column Name: Enter the exact header name used in your upload file.
  8. Click Update.
  9. Follow the wizard to the end to save the configuration.

Symptom

In the standard system as currently delivered, you try to recreate the scenarios described in the IRS AATS narratives by following the Cookbooks provided. You can face the following issues:

  • When trying to create the Personal Data infotype, but the SSN used in the scenario is not valid, the system does not allow you to use it.
  • For scenario 7, you are reporting only one employee with the current transmission, but the fields FORM1095CATTACHEDCNT and TOTALFORM1095CALEMEMBERCNT should have the value 103.

Other Terms

Cookbook, AATS, IRS, ACA, Communication scenarios

Reason and Prerequisites

As a prerequisite, you have implemented the SAP Note 2291518 - BEN: Affordable Care Act (ACA) - 11th SAP ACA Update.

Solution

This note will deliver the new methods ADJUST_FORM_COUNT, ADJUST_SSN_B and ADJUST_SSN_C in the class CL_HRBENUS_ACA_AATS_TC to allow to cope with the issues described above. To use them, you will need to redefine them with a custom class, as explained in the SAP Note 2276617.

As a general rule, SAP recommends that you install a solution by applying a Support Package. However, if you need to install a solution earlier, use the Note Assistant (SNOTE) to implement the correction instructions for your release.

You can find more information about the Note Assistant in SAP Service Marketplace, under service.sap.com/note-assistant.

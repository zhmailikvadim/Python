Symptom

Error - Not a Valid XML : Check the payload 

Environment

SAP Ariba Sourcing

Cause

This issue occurs because of either incorrect data or an improperly formatted XML payload sent from the external system.

Resolution

1. Verify that the tenant ID 'AN0XXXX' is accurately configured in the system destination URL. This involves checking if the tenant is active and correctly established in your working environment.

2. Make sure the payload is properly indented. If the data is sent as a single line within the payload, it will not validate according to the standard payload structure.

3. Ensure that the endpoint you are using is correct and matches the environment where the tenant is configured. Using the wrong endpoint can result in errors.

Symptom

You are processing RF Reversal of Consumption by MO (HU). After entering the Manufactoring Order, the system prompts for the HU. The entered HU is a GTL barcode with correct length. However, the system does not respect the GTL barcode prefix and incorrectly issues the error message "Invalid input length".

Other Terms

/SCWM/RF_EN289, /SCWM/RF_EN 289, GTL barcode, MIRVHU, HU truncated

Reason and Prerequisites

GTL barcode handling is missing.

Solution

Implement the attached code changes or import the corresponding support package.

Symptom

  • RE-FX lease assets are migrated following the transition scenario 1 or 2 as described in SAP Note 2544232.
  • When posting the initial valuation step, an external acquisition is posted for the asset instead of just changing the cash flow status of the asset cash flow to "Posted".
  • This is causing duplicate asset acquisition on the lease asset.

Environment

  • Flexible Real Estate Management (RE-FX)
  • Contract and Lease Management
  • SAP ERP Central Component
  • SAP ERP
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA Finance
  • SAP S/4HANA
  • SAP S/4HANA Cloud Private Edition

Reproducing the Issue

  1. Execute transaction RECEEP in update mode for a lease contract with migrated asset according to transition scenario 1 or 2.
  2. Check the asset cash flow afterwards.
  3. An asset acquisition had been posted even though the associated lease asset had been migrated.

Cause

The condition type used to specify the net book value of the transferred lease asset has been assigned the valuation property 'F Net Worth Value Specified' instead of 'M Net Worth Value Specified (Legacy Data).'

Resolution

  1. Assign the valuation property 'M Net Worth Value Specified (Legacy Data).' to the condition used to specify the net book value of the migrated asset.
  2. The configuration setting to assign conditions and valuation properties is available using the following path: Transaction: RECACUST > Balance-Sheet Valuation of Contracts > Condition Valuation Groups

See Also

SAP Note 2544232 - Leasing: Handling transition scenarios

Keywords

RECEEP, Asset, Net Worth Value Specified (Legacy Data), External asset acquisition, Migration, Posted

Symptom

The dropdown list for UDFs on the UDO row is not displaying in browser access. It is visible in the B1 client.

Reproducing the issue

Prerequisites:

 

Scenario:

  1. Go to Tools → Customization Tools → User-Defined Fields - Management, expand "Marketing Documents → Rows" and add a UDF field with valid values. Ensure the title length exceeds 20 characters, such as "Test111111111111111111111111".

  2. Log in to Browser Access.

  3. Go to Sales - A/R → Sales Order, select any customer and item, then check the valid values of the UDF field.

Expected behavior:

The valid values for the UDF field are displaying correctly.

Actual behavior:

The valid values for the UDF field are not displaying.

Symptom

  • In the process of integrating Cloud ALM with SAP Business Process Automation (BPA) for providing Requirement Approval in BPA 
  • Facing issue when put the status to Ready for Approval
  • Console shows the API Key error:
    Configuration error occurred : {"error":{"status":400, "code":"<ID>","severity":"error""message":"ApiKey property (irpa_api_key" was not found in request header or body.", "correlationId":"<ID>"}}
  • Updated the Cloud ALM API instance with the CLOUD ALM  scope parameters and generated the service key
  • Updated the API Key in the landscape management of the BPA service but still getting this error

Environment

SAP CLOUD ALM

Cause

When using a shared environment in BPA, an API key is needed to be sent in the header of the API request.  (Info on the API Key found at: API Keys | SAP Help Portal  -> Context section has info on adding the key.)

Also, must add an Additional Property in the destination for IRPA created in the Cloud ALM subaccount within BTP Cockpit -> see https://help.sap.com/docs/connectivity/sap-btp-connectivity-cf/http-destinations?q=apikey#custom-query-parameters-and-headers and https://help.sap.com/docs/connectivity/sap-btp-connectivity-cf/oauth-client-credentials-authentication > URL.headers.<header-key>.

Resolution

Take the following steps:

  • Check API Keys | SAP Help Portal to create the API key for BPA (for help here, involve the BPA support team)
  • In BTP Cockpit > SAP Cloud ALM's subaccount, modify the destination for BPA to include an Additional Property for URL.headers.api-key (use this name!  See https://help.sap.com/docs/connectivity/sap-btp-connectivity-cf/http-destinations?q=apikey#custom-query-parameters-and-headers and https://help.sap.com/docs/connectivity/sap-btp-connectivity-cf/oauth-client-credentials-authentication)
  • After following these steps, retry to create the requirement and see if the error is resolved or not-> if problem remains, log a support case for SV-CLM-API

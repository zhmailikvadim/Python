Symptom

For a Purchase Orders, an accrual has been approved and posted in a prior period and the accrual document exists in the system with Lifecycle Status (P In process). However the same accrual is still proposed again in the next period in the transactions code ACEPROPOSALRUN, FACRARVWCO, and FACRAAPPRVGL after their respective End of Life 

Environment

  • Financial Accounting
  • SAP ERP Central Component
  • SAP ERP
  • SAP S/4HANA Finance
  • SAP S/4HANA
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA

Reproducing the Issue

  1. Ensure a purchase order accrual was approved and posted in a prior period and that the accrual document exists.
  2. Run ACEPROPOSALRUN for the subsequent period.
  3. Open FACRARVWCO and FACRAAPPRVGL to review the proposed and approval lists.
  4. Observe that the same accrual is proposed again and that the review status is N (Not Reviewed) with Review Type E (Explicit review required).

Cause

All Accrual Objects in Lifecycle Status (P In process) are considered during the ACEPROPOSALRUN process, even after their respective End of Life. This approach ensures that any potential utilization postings for PO Accrual Objects are accounted for. However, once the Final Invoice or Delivery Completed flag is set on the PO, and its Lifecycle Status changes to Premature Finished, the accrual object is excluded from the proposal process.

The engine is designed to handle PO Accrual Objects based on Planned versus Actual Costs, with the expectation that the PO will be fully delivered after the Accrual Object's End of Life. If the object is marked as Prematurely Finished, it is excluded accordingly.
If the approval process is active in the configuration and the proposed accrual amount is not approved by the end of the period, the periodic posting run will balance the accrual object to zero. This mechanism is in place to prevent over-accrual scenarios.

Resolution

Accrual Engine is functioning as designed.

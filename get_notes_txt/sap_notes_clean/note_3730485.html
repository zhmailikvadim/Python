Symptom

Blocking a Schedule Agreement (SA) line item using transaction ME32L automatically blocks the corresponding entry in the Source List (ME01) to prevent procurement, but unblocking the Scheduling Agreement (SA) does not automatically remove the block in the Source List.

Environment

  • Materials Management (MM) 
  • SAP ERP Central Component 
  • SAP ERP 
  • SAP enhancement package for SAP ERP 
  • SAP enhancement package for SAP ERP, version for SAP HANA 
  • SAP S/4HANA All Version

Reproducing the Issue

1. Create the SA in ME31L with all relevant information.
2. Block the SA Line Item in ME32L.
3. Source of Supply in ME01 is automatically blocked.
4. Now revert/remove the block from SA.
5. The system does not remove the block indicator automatically from the Source List.

Cause

When you set a block for a scheduling agreement, the subsequent steps need to be checked, for example:

- Do not use source of supply as * procurement option (field Fixed Source of Supply - FESKZ is cleared).

- Do not use material requirements planning (MRP) for source list records (the Material Planning MRP - AUTET field is cleared).


The system cannot automatically unblock the Source List when unblocking the Schedule Agreement because fields like Fixed Source of Supply (FESKZ) and Material Planning MRP (AUTET) are dependent on specific business processes and decisions.

Resolution

This is standard system behavior. 

System cannot make decision automatically since, for example, value FESKZ, AUTET are dependent on business process. So, manually set is required.

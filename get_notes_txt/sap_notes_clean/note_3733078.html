Symptom

  • Inquiry about the availability of built-in malware or antivirus scanning capabilities in SAP Cloud Integration during runtime file processing or transfer.
  • Clarification requested on whether runtime payloads processed in iFlows (for example, files read from SFTP or sent via HTTP/REST) are scanned by the platform.
  • Confirmation sought on any supported configuration or feature to enable antivirus scanning of payloads before sending files to target systems.
  • Noted that the Malware Scanner feature is applicable only to the Cloud Foundry environment and unavailable in Neo.
  • No error messages or error codes were reported.

Environment

  • Product: Cloud Integration
  • SOAP Adapter

Reproducing the Issue

  1. Process files in an integration flow using runtime adapters such as SFTP or HTTP/REST.
  2. Review adapter/runtime configurations and observe that no built-in antivirus/malware scanning option is available for runtime payloads.
  3. Check the Malware Scanner feature and note it applies to design-time uploads (and is available only in the Cloud Foundry environment), not to runtime message payloads.

Cause

  • Cloud Integration does not provide built-in malware or antivirus scanning for runtime payloads processed in iFlows.
  • The Malware Scanner feature is limited to scanning design-time artifacts during upload and is available only in the Cloud Foundry environment; it is not available in Neo.

Resolution

  1. Acknowledge that automatic antivirus/malware scanning of runtime payloads (inbound or outbound) is not supported in Cloud Integration for Neo or the Cloud Foundry environment.
  2. Use the Malware Scanner only for design-time uploads; note that it is available in the Cloud Foundry environment and not available in Neo.
  3. If runtime malware scanning is a requirement, implement scanning at the source system before files are sent to Cloud Integration.
  4. Alternatively, implement scanning in an external gateway or middleware layer placed before/after Cloud Integration.
  5. Alternatively, leverage storage or services in your architecture that provide built-in malware scanning prior to processing by Cloud Integration.
  6. Refer to the documented scope and limitations for additional guidance.

Symptom

  • ASE errorlog reports signal 11 stack trace in module kpsched():
     
    Current process (0x2d002d) infected with signal 11 (SIGSEGV)
    Address 0x0000000000a9fcc8 (kpsched+0x351)
    kechkfault: engine #: scheduler infected with signal 11 (SIGSEGV)     
     
  • Spid 0 (internal scheduler process) dies, crashing ASE abruptly:

  ueshutdown: exiting
  Abnormal exit detected - ﻿

  • The stack trace will include all or most of the following modules: 
     
    keipanic
    kechkfault
    kisignal
    kpsched
    kpstartproc﻿

Environment

  • SAP Adaptive Server Enterprise (ASE)
  • With kernel mode = process
    • Version 15.5 or earlier only had 'process mode'

Cause

This issue is very rare. It can be seen when switching a runnable process between engines and one of the engines hits a signal 11. It is only seen in process mode.

Note:

  • All releases that only have process kernel mode (15.5 or earlier) are now 'end of life' - see KBA 3504083.
  • From ASE 15.7, threaded kernel mode is the default and SAP recommended way to run

There is a CR open to track this issue, CR 768663.  Its description reads:
"In rare circumstances the signal 11 might occur when try to Switch to running a new process in ASE when running in 'process mode' ".

Resolution

There are no plans to address CR 768663, as the issue only affects 'end of life' versions or running ASE in a non-default and not recommended way.

  • When using ASE 15.5 or earlier:
    • Upgrade a current and supported release using 'threaded mode' : ASE 15.7 or higher

  • When using a current and supported release (15.7 or 16.0):
    • Use the recommended 'threaded' mode

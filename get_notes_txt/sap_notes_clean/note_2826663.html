Environment

  • SAP 16.0 
  • SAPIQ 16.1 SP02 PL15 and above

Cause

The issue has been identified and logged under SAP CR 819864. Its description reads,"IQ 16.1 sp02 pl15, GetCleanPage is retrying to avert a potential deadlock"

Resolution

  • Upgrade to where CR 819864 is fixed:
    • SAP IQ 16.0 SP11 PL27 and above
    • SAP IQ 16.1 SP03 PL09 and above
    • SAP IQ 16.1 SP04 PL05 and above
      

      • 

    

  • Set both options below to disable the deadlock avoidance mechanism. 
    The deadlock would occurred in IQ server for the same set of workload. 
    Server restart is required for change to take effect.

    SET OPTION PUBLIC.CORE_Options151 = 'ON'; 
    SET OPTION PUBLIC.CORE_Options155 = 'OFF ';

    • 

  • 

    The deadlock avoidance gear has been made more robust after analyzing various scenarios that were encountered in the field.
     The new settings are required to prevent a number of possible of deadlocks.
     The warning messages are expected and intended in a proper functioning system.
    
If the deadlock issue can be reproduced, they should collect the stack traces for further analysis.

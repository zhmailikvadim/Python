Symptom

You have a WSDl file from an external server and you want to know whether it can be handled correctly within the ABAP WS Framework (e.g. Proxy Generation, creation of a logical port in transaction SOAMANAGER).

Other Terms

  • Web service
  • SOAMANAGER
  • Proxy generation
  • WSDL analyzer

Solution

You can use report RSSIDL_DESERIALIZE_DEMO to check if the WSDL can be read by the WS Framework and if it contains no limitations. If the WSDL is accepted by ABAP WSDL Library, the report will display the output in the internal SIDL format. If an error occurs during parsing the WSDL, a more detailed description of the error that caused the exception will be displayed.

A better solution is to use the WSDL analyzer in transaction SOAMANAGER->Tools->Analyze WSDL. You can define a WSDL file or URL as input. As a result, The content of the WSDL is displayed and, if existing, also unsupported fragments are displayed.

As WSDLs consisting of several files cannot be uploaded to transaction SOAMANAGER. You can use eport SRT_PRG_WSDL_ANALYZER as it offers the same functionality as the WSDL analyzer in transaction SOAMANAGER, but also handles non-monolithic WSDL files. The report is available in SAP Note 3066726.

Also check the SAP Note 2985721 for using non-monolithic WSDL files for creating logical ports in transaction SOAMANAGER.

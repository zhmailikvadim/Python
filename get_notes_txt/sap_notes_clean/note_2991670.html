Symptom

In an SD document, the system issues message V1 208 for a cost condition as follows: "Condition VPRS: Error in converting exchange rate from USD to USD".
The message does not make sense because the specified currencies do not differ and therefore the execution of a currency translation appears unnecessary.
However, there is actually a failed currency conversion, after which the system issues message V1 208 with the wrong variables.

Other Terms

V1 208, V1208, VPRS, WAVWR, ZWAVWR

Reason and Prerequisites

The message V1 208 cannot be issued in this nonsensical form if the goods clearing value is calculated using the valuation segment in the material master. This is possible only if the condition amount for the cost condition is to be calculated from a specified goods clearing value using backward calculation (ZWAVWR logic in FORM XKOMV_BEWERTEN).
The (failing) currency conversion is performed from the currency of the company code *T001-WAERS to the document currency KOMK-WAERK.
The system issues message V1 208 with the document currency KOMK-WAERK and the local currency KOMK-HWAER. This is incorrect.

Solution

Implement the attached source code changes.

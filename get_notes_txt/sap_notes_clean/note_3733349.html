Symptom

  • Replication of a Central Finance (CFIN) table completes the initial load, but subsequent replication does not start.
  • Error message: “In the source system, the job for the boundary calculation failed.”

Environment

SAP Landscape Transformation Replication Server (SLT)

Reproducing the Issue

  1. In transaction LTRS, open Performance Options > Initial Load Options for the affected replication object.
  2. Set reading type to 5 and set parallelization by setting the Number of Jobs and Number of Records per package.
  3. Start replication of the table.
  4. Observe that the initial load completes successfully, but subsequent replication does not start and the error “In the source system, the job for the boundary calculation failed” is logged.

Cause

The reading type of the replication object was manually changed to 5, whereas it should be 1 for the replication object in CFIN scenario.

Resolution

  1. Verify that the SLT replication content version is correct (refer to: 3196293 - How to check the current Central Finance replication content version on an SLT system).
  2. Follow KBA 3017069 - How to specify/change a reading type for a CFIN replication – SLT( in step 4, open “Replication Object” instead of "Initial Load Object") to open the Replication Object for the affected CFIN table and check the reading type.
  3. If the reading type in the replication object is set to 5, change it back to 1 and save.
  4. Reactivate the replication object as needed, restart replication of the table.

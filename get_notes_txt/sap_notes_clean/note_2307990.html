Symptom

When system calculates the total Ordered Qty in the Inventory Status Report, it only rounds the quantity after summing up all the Ordered Quantity rows instead of rounding quantity for every row and summing up the rounded values. This might cause an incorrect total Ordered Qty to be displayed in the report. The also occurs for the Committed Quantity.

Reproducing the issue

Scenario:

  1. Decimal places setting for quantities is 2 in the General Settings > Display tab.
  2. Create new item NewItem, define Items per Purchase Unit 10.76 under Purchasing Data tab in Item Master Data.
  3. Create one Purchase Order for NewItem, document quantity is 1140.48.
  4. Create another Purchase Order for NewItem, with qty 207.36.
  5. Open the Inventory Status Report for this item. Double-click the row to see the detailed list of relevant documents and note that the first purchase order has Ordered Qty 12271.56 and the second has Ordered Qty 2231.19.                        

Expected Behavior:

  • Inventory Status Report should show total Ordered Qty 14502.75 (=12271.56+2231.19)
  • System should calculate it as sum(round(tblDoc.OpenQty * tbldoc.NumPerMsr), (SELECT QTYDEC FROM OADM))

Actual Behavior:

  • Inventory Status Report shows total Ordered Qty 14502.76.
  • System calculates it as round(sum(tblDoc.OpenQty * tbldoc.NumPerMsr), (SELECT QTYDEC FROM OADM))

Cause

Future Version Candidate (FVC)

Solution

Correction of system behaviour is not feasible in the currently supported versions of SAP Business One. The error has been recorded and is a candidate to be fixed in a future version.

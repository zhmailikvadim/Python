Symptom

After setting up a delegation, only the delegatee receives approval notifications for pending documents.

Reproducing the Issue

  1. Go to User Manager > Delegations
  2. Click Create Delegation
  3. Leave the Continue to notify delegator by email of approval requests checkbox unchecked.
  4. Submit the delegation. The delegatee receives approval notifications, but the delegator does not.

Cause

When you create a delegation through the UI, the NotifyDelegator option is controlled by the checkbox Continue to notify delegator by email of approval requests. If you leave this checkbox unchecked, the system sets NotifyDelegator to No, which prevents the delegator from receiving approval notifications.

When delegations are created through import, the NotifyDelegator column cannot be null. This prevents accidental configuration of delegations where the delegator is unintentionally not notified.

Resolution

When creating delegations through the UI, always make sure the checkbox Continue to notify delegator by email of approval requests is checked. This ensures that both the delegator and the delegatee receive approval notifications.

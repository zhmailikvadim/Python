Symptom

The indexserver service of SAP HANA crashes with a crash stack similar to the following:

[CRASH_STACK] Stacktrace of crash: (0000-00-00 00:00:00 000 Local)
----> Pending exceptions (possible root cause) <----
exception 1: no.2000001 (Basis/IO/FileAccess/impl/SimpleFile.cpp:185)
Generic file error; $condition$=INVALID_DIR_HANDLE != m_DirHandle; $message$=Invalid directory handle!
exception throw location:
1: FileAccess::DirectoryIterator::DirectoryEntry::getFullName
2: ptime::PlanTracerImpl::loadMetadataFileCache
3: ptime::BuiltinProcedure_PLANVIZ_ACTION::loadPlanTraceMetadata
4: ptime::BuiltinProcedure_PLANVIZ_ACTION::execute
[...]

Reason and Prerequisites

Reason:

Due to a programming error in SAP HANA, the indexserver tries to access an empty plan trace.

Affected Releases:

  • SAP HANA 2:
    • Revisions of SPS00 - SPS02
    • Revisions <= 037.01 (SPS03)
    • Revisions <= 041.00 (SPS04)

Prerequisites:

  • The plan trace is loaded. This may also happen implicitly, e.g. when the tenant overview in HANA Cockpit is called. 

Solution

Apply one of the following SAP HANA Revisions:

  • SAP HANA 2:
    • Revisions >= 037.02 (SPS03) 
    • Revisions >= 042.00 (SPS04)
    • or higher

  • or higher

Workaround:

  • Disable the plan trace by setting the following configuration parameter:

    ALTER SYSTEM ALTER CONFIGURATION ('global.ini', 'SYSTEM' ) SET ('performance_analyzer', 'plan_trace_enable') = 'false' WITH RECONFIGURE;

    Starting with HANA 2.0 SPS03 comments should be added:

    ALTER SYSTEM ALTER CONFIGURATION ('global.ini', 'SYSTEM' ) SET ('performance_analyzer', 'plan_trace_enable') = 'false' WITH RECONFIGURE COMMENT 'SAP Note 2866779. Plan trace can be enabled again after upgrade to a revision containing the fix';

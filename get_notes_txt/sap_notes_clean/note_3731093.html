Symptom

You customize Interunit Elimination or Reconciliation task to Save the Report and Do not Display Report. So the original Log is shown at the end of the task execution, but the result in report is persisted. You execute the same task several times from the consolidation m monitor. Then you navigate to Display Interunit Elimination/Reconciliation Results to show the results of the last task execution. You recognize that the result of the last task execution is displayed multiple times.

Furthermore you also recognize that the report header information of the next task execution (for example time of execution) is wrongly taken over from the last task execution. The header information shall be taken from the current execution. This is side effect of the SAP Note 3674856.

Reason and Prerequisites

Program error

Solution

Implement the attached Correction Instruction (CI) or the relevant Support Package.

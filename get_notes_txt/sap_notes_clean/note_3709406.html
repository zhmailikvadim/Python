Symptom

When you convert a planned order into a production or process order, you get a popup to select a production version.
When choosing a new production version the goods receipt storage location (Receiving Location) defined in the production version is not taken over correctly.

The issue only occurs if you set up an automatic reread of master data (BOM Selection = 2) and manual selection of production version, in the order type dependent parameters (OPL8 or COR4).

If you reread master data manually later the goods receipt storage location is correctly taken over.

Other Terms

AFPOD-LGORT, SLOC, V_T399X-FLG_READ, T399X-FLG_READ, V_T399X-PVSEL, T399X-PVSEL, COR7, CO40, MKAL_EXPAND-ALORT, MKAL-ALORT, C223

Reason and Prerequisites

Program error.

Solution

Implement the attached correction.

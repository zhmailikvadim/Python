Symptom

First, you perform component backflush in transaction MFBF. The components are not posted but recorded as backlog items.

Then, you post the backlog items in transaction COGI or MF47.

Finally, when you try to cancel the original document log of component backflush in transaction MF12, the system reacts with error message RM 070:
"Error reading document log information &".

Other Terms

MFBF, repetitive manufacturing, backflush confirmation, backlog, postprocessing record, reversal, cancellation, MF47, COGI, MF12, RM070, RM 070, 070(RM)

Reason and Prerequisites

The issue occurs in a specific scenario where no document log item (BLPP record) exists anymore after successful COGI processing.

Solution

Implement the attached correction instructions.

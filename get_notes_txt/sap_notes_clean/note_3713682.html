Symptom

When running the demand realignment service on demands that have demand category XML_ACTIVE or XML_INACT, and the category has to be changed due to change in stocking on dealer location, the period value is lost in the process of the realignment. It will change to zero in RAWDML table records (and in DEMHIS table in case the demand history is persistent.) When a demand like this is displayed in the Raw Demand SAP GUI UI, the demand category is not displayed.

Other Terms

eSPP, demand realignent, dealer demand realignment, demand category

Reason and Prerequisites

Program error

Solution

Implement this Note containing the code change.

Symptom

  • Error appears when saving Recovery Indicator Manipulation Rules in transaction GJCS.
  • Message No. TK455: "Enter numeric values only."
  • Detailed text includes: "The value '500 ' was entered in the field 'EVENT'. This is not possible because it is a numeric field where only numbers may be entered. The length of the field is 000004. The entry was not accepted."
  • Issue occurs while the system is populating entries in table TKAZE; persists even after manual population.

Environment

  • SAP S/4HANA
  • SAP S/4HANA Finance
  • Financial Accounting (FI)
  • Joint Venture Accounting (JVA)

Reproducing the Issue

  1. Start transaction GJCS.
  2. Open Recovery Indicator Manipulation Rules and create a new rule entry.
  3. Save the customizing to a transport request.
  4. Observe that message TK455 "Enter numeric values only" is issued for the EVENT field.

Cause

A known issue in the population of TKAZE entries for the EVENT field causes an incorrect (non-numeric or incorrectly formatted) value to be written, which leads to message TK455.

Resolution

  1. Refer to KBA 2482224 and ensure all EVENT values are strictly numeric and formatted as required (no spaces or non-digit characters).
  2. Implement SAP Note 3548787 that corrects the population of the EVENT field in TKAZE for this scenario.
  3. If previous manual entries were created, review TKAZE for the relevant rules and correct any non-numeric or improperly formatted EVENT values.
  4. Re-save the Recovery Indicator Manipulation Rules in GJCS and transport the changes.

Symptom

Joule has been integrated with a supported product. However Joule's conversational user interface fails with the error:
**********************************************************
Search Service Exception 
AI Natural Language Search is currently unavailable
**********************************************************
Screenshot:

Environment

  • Joule AI copilot
  • SAP S/4HANA Cloud Public Edition
  • SAP Business Technology Platform (SAP BTP. URL: https://cockpit.btp.cloud.sap) 
  • SAP Cloud Foundry (CF)

Reproducing the Issue

Joule has been integrated with say SAP S/4HANA Cloud Public Edition.
Joule Web Client is launched, and the above stated error occurs.

Cause

Bug identified

Resolution

For further analysis, provide the below traces:

  1. Reproduce the issue and provide the Joule trace as per SAP KBA ##3666014.
    Do include message history in your traces file (.zip) as the KBA suggests.
  2. Attach the Browser network trace and console logs.  
    Before starting the tool, ensure that you clear all the browser cache  (check SAP KBA ##3402492 - How to clear Cache and Cookies of Browser ). Make sure to leave the recording button enabled (red) and the Preserve log and Disable cache checkboxes marked.
    Now start recording from the beginning (logging into Joule) and reproduce the issue. Attach the Network trace (.HAR file) and the browser console log (--> Navigate to console tab, --> Right click and select Save as --> Save file with .log extension) after error reproduction. Detailed steps are listed here:
    SAP KBA ##1990706 - How to capture a HTTP trace using Chrome browser.
    and
    SAP KBA ##2764266 - How to get Network Trace and Console Logs
    Console Logs: --> Navigate to console tab. --> Right click and select Save as. --> Save file with .log extension.
    This trace will have the Correlation ID that would help in further detailed investigation. The correlation identifier uniquely distinguishes each request to the microservice. The client would generate this value and keep it as part of its application logs. In turn the server also logs the same value. The two logs can then be correlated, enabling end-to-end tracing.

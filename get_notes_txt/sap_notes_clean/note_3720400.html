Symptom

Getting a lot of CSRF token validation failed Error 403 in synchronization, for example, finished sending POST request to back end http://xxxxx/sap/opu/odata/MERP/SAP_SRV_ASSET_MANAGER_xxxx/DynamicFormAttachments in 420 ms. HTTP status from the back end is 403 with header: X-CSRF-TOKEN : Required

Environment

SAP Service and Asset Manager 2410 version

Reproducing the Issue

  1. Finish to sending POST request to back end http://xxxxx/sap/opu/odata/MERP/SAP_SRV_ASSET_MANAGER_xxxx/DynamicFormAttachments in 420 ms
  2. HTTP status from the back end is 403 with header: X-CSRF-TOKEN : Required

Cause

Product design

Resolution

  • Cross-Site Request Forgery (CSRF) is a type of attack where an attacker sends malicious requests from one location to another where the victim is authenticated. To prevent this attack, a security token is stored in the user's session and must be included with every modifying operation (such as PUT, POST, DELETE).
  • If the provided token is incorrect or expired, the gateway responds with an HTTP 403 ("Forbidden") status code.
  • The reason for the warning in the logs is that a CSRF token was expired. In such cases the SSAM gets a new token and sets it in a subsequent HTTP request.
  • This behavior is typical in OData implementations. The SSAM will automatically resend the request after updating the token, so that is why the CSRF token message's type is warning, not the error.

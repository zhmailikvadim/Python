Symptom

Administrators need to establish secure authentication for SFTP integrations without relying solely on passwords. SSH keys provide a more robust method for identifying the SAP Fieldglass system when connecting to external customer or vendor servers.

  • This article provides guidance for the following scenarios:
  • Generating a new SSH key pair (public and private) for SFTP authentication.
  • Replacing an existing SFTP password-based login with a more secure SSH key.
  • Managing multiple SSH keys for different environment-specific endpoints.
  • Retrieving the public key to share with internal IT or network teams for server allow-listing.

Resolution

SSH keys are environment-specific and unique to the system in which they are created. You must generate a new key for each environment (e.g., Test/Base and Production).

Step 1: Create and Activate the SSH Key

  1. Log in to SAP Fieldglass as a Configuration Manager user.
  2. Select the Encryption Keys and Certificates tile from the dashboard.
  3. Click New in the top right-hand corner.
  4. In the Asset Category dropdown, select SSH Key.
  5. Enter a unique Asset ID (e.g., SFTP_Production_Key).
  6. Select the Key Purpose: AUTHENTICATION.
  7. Check the box for Auto Generate Key.
  8. Provide a strong Passphrase and confirm it. Note: This passphrase will be required when configuring the endpoint later.
  9. Click Create, then select the Activate button in the top right of the details page.

Step 2: Retrieve the Public Key

  1. On the details page of your active SSH key, navigate to the Details tab.
  2. Locate the Public Key (typically starting with ssh-rsa).
  3. Copy this string or download the key file.
  4. Provide this public key to your IT or server administrator. They must add it to the authorized_keys file on your target SFTP server to allow Fieldglass to connect.

Step 3: Associate the Key with an SFTP Endpoi

  1. Navigate back to the dashboard and select the Endpoints tile.
  2. Click New or edit an existing SFTP category endpoint.
  3. In the SFTP Setup section:
    1. Set the Authentication Type to SSH Key.
    2. Select your newly created SSH Key from the dropdown menu.
    3. Enter the Passphrase you created in Step 1.
  4. Fill out the remaining host and port details.
  5. Click Submit and then Activate the endpoint.
  6. Use the Test Connectivity button to confirm the SSH handshake is successful.

See Also

  • Environment Isolation: A public key created in a Test environment cannot be reused in Production. You must generate a unique key pair for each instance.
  • Missing Dropdown: If the SSH Key dropdown does not appear in your Configuration Manager, please contact SAP Fieldglass Support to enable the system-level variable for your company.
  • Revocation: If an SSH key is compromised, use the Revoke button on the asset page to immediately disable all integrations using that key.

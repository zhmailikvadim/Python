Symptom

  • Replication flow stopped working
  • Observer job shown on DHCDCMON as red/stopped
  • Replication not working between source and target
  • BATCH_USER not authorized to run LTB Job Dispatcher
  • Issue started post security patch on April 2nd

Environment

SAP Datasphere
SAP S/4 HANA

Cause

On April 2nd there was a security patch that restricted authorizations on the BATCH_USER in Client 000

Resolution

  1. Run statement to detect unwanted entries in job queue:
    Select * from "/LTB/JOB_QUEUE" in DBACOCKPIT or the DB level
  2. If there is an entry in /LTB/JOB_QUEUE for client 000 where JOB_NAME = /1DH/OBSERVE_LOGTAB or /1DH/PUSH_CDS_DELTA they should be deleted.
  3. Create case on component PCO-OPS  (since ECS managed S4-PCE is involved) requesting deletion of the same informing result of SQL above and the scenario presented referencing this KBA.
  4. After the entries are removed from /LTB/JOB_QUEUE go to the correct client (not client 000), run transaction DHCDCMON and select 'Dispatcher Job'.

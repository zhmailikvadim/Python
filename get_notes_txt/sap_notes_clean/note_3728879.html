Symptom

In Migrate You Data app (migration cockpit) in Fiori, when attempting to create a new project, Migrate Using Staging Tables is selected but the tool redirects to Migrate Data Directly from SAP System.

Environment

S4HANA

Reproducing the Issue

  1. Open Migrate Your Data app in Fiori
  2. Click on Create
  3. Select Migrate Data Using Staging Tables 
  4. Migrate Data Directly from SAP System appears instead of staging tables

Cause

SAP UI Index was not updated yet.

Resolution

Run the report /UI5/APP_INDEX_CALCULATE as per the SAP KBA below:
2449635 - How to calculate SAPUI5 application index

This will update the index and the creation of a new migration project must be working.

See Also

Please always keep in mind that the note analyzer report (CNV_NOTE_ANALYZER_MC_EXT) must be executed and all notes implemented in the system to avoid unwanted bugs or misfunctions. It is not a pre-requisite to update the index, but it is important to maintain the system with the latest updates:

How to implement the latest corrections released for Migration cockpit | SAP Blogs

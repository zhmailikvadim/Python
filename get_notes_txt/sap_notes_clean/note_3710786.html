Symptom

  • Supplier invoice data is successfully retrieved from the KSeF portal; however, the business system fails to convert the data into Incoming eDocuments.
  • The DRC Cloud Edition job status remains in 'Error' or 'Partially Completed':
  • The drill down 'Run Details' report shows few/all process instance IDs in 'Error' status and the detailed error message shows the following:
    "Problem occurred while transmitting the document to the business system. See details and try to reprocess in the Resolve Errors - Documents and Statutory Reports app. The Cloud Edition received the following message when sending information that comes from the external communication party to your business system with destination XXXXXXXX: Process Instance not accepted for processing by application. The process instance ID is xxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxx, HTTP status code is 422"

  • Inside 'Resolve Error' application, a similar error message is seen:
    "Error occurred during the step, detailed information: The Cloud Edition received the following message when sending information that comes from the external communication party to your business system with destination XXXXXXXX: Process Instance not accepted for processing by application. The process instance ID is xxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxx, HTTP status code is 422"

 Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.

Environment

  • SAP ERP
  • SAP S/4HANA
  • SAP S/4HANA Cloud Private Edition
  • SAP Document and Reporting Compliance, Cloud Edition

Reproducing the Issue

  1. Go to DRC Cloud Edition and launch 'Schedule Jobs' app
  2. Try to configure and run a job of 'Process Flow' type 'Retrieve Documents for E-Invoices Throught KSeF (PL)'

OR,

  1. If there's an existing job with this error, simply go inside it and activate it again

Cause

This issue may occur due to several reasons:

  1. Missing code corrections
  2. Supplier invoice data incompatibility with SAP Proxy Structure and ultimately resulting in a XML to ABAP serialization/deserialization issue

Resolution

  1. Missing code corrections:
    1. Before going ahead with Poland related notes, it is needed to have the prerequisites mentioned in 3672304 - What are the prerequisites for implementing Electronic Document Processing for a country/region (non-Peppol) which uses the integration with Document and Reporting Compliance, cloud edition? - SAP for Me, especially the CA-GTF-CSC-DRC-COM notes which are crucial to handle the incoming request from DRC.
    2. Additionally, follow the series provided in overview note 3279206 - Electronic Document Processing for Poland - KSeF: Implementation Overview - SAP for Me and ensure all SAP notes are implemented.
  2. Supplier invoice data incompatibility:
    Supplier invoice data retrieved from the KSeF portal cannot be converted into Incoming eDocuments when the data/payload does not comply with the proxy structure defined by the KSeF schema (FA3 logical structure information-sheet-on-the-fa-3-logical-structure.pdf). This can be verified by re-activating the job and checking the error logs in transaction SRT_UTIL. The following section lists the identified incompatibilities and SAP’s official stance on each:
    1. Tag DataWytworzeniaFa(Date and time of invoice generation):

      To fix this issue, implement SAP Note 3710822 - Poland KSEF FA3: XML format errors for incoming invoices - SAP for Me by following the exact sequence from the overview note.
      NOTE: This error will be visible even if the note is implemented successfully. But the eDocuments will be created.

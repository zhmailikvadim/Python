Symptom

When an invoice is parked or held with lean services using BAPI_INCOMINGINVOICE_CREATE1 or BAPI_INCOMINGINVOICE_PARK, the line-item data is shown as blank in MIR4.

Environment

  • Materials Management (MM)
  • SAP S/4HANA

Reproducing the Issue

    1. When using BAPI_INCOMINGINVOICE_CREATE1 or BAPI_INCOMINGINVOICE_PARK to park an invoice, add references in the BAPI input tables SELECTPO, SELECTDELIVERY, or SELECTSERVICE_LEAN.
    2. At the same time, the item details tables are not filled.
    3. Execute BAPI to park the invoice.

Cause

  • When using reference documents, the system will only pre-fill the document reference into the parked invoice, without loading any item data.
  • In other words, after the invoice is parked and opened for editing, the reference will be present in the input field.

Resolution

  • This is standard system behavior.

  • To see the item details in the invoice, it is necessary to follow the steps below:
    In MIR4, open the invoice >>> Edit >>> Display/Change (F7) >>> choose Update Allocation
    Then the line-item details will appear in the invoice.

  • For related information, read the consulting Note 3731427 - BAPI: SELECTPO, SELECTDELIVERY, SELECTSERVICE_LEAN parameter documentation.

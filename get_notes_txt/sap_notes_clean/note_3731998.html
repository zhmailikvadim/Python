Symptom

  • A request was made to change the tenant URL domain from analytics.cloud.sap to hcs.cloud.sap for sap analytics cloud.
  • The operations team added the new domain as the primary domain and both domains may resolve to the same tenant.
  • In sap for me under resources, the displayed tenant URL still shows the old analytics.cloud.sap domain instead of the new hcs.cloud.sap domain.

Environment

SAP Analytics Cloud - Administration

Reproducing the Issue

  1. Request a sap analytics cloud tenant URL/domain change from analytics.cloud.sap to hcs.cloud.sap (per the documented process).
  2. After operations confirms the change and sets the new domain as primary, open SAP for Me and navigate to the tenant under Resources.
  3. Observe that the displayed tenant URL still shows the old analytics.cloud.sap domain.

Cause

This is a current limitation: information shown in SAP for Me is aggregated from multiple backend sources and there is no mechanism to control or guarantee which of the tenant’s reachable domains is displayed under Resources.

Resolution

  1. Complete the standard URL/domain change request for sap analytics cloud as documented (for example, via the account team and providing the required acceptance statement).
  2. Be aware that both the old analytics.cloud.sap and the new hcs.cloud.sap domains can reach to the same tenant, and SAP for Me may continue to display the old domain due to aggregation limitations.
  3. Use the new added domain for all operational access and bookmarks.
  4. No further tenant-side action is required; this is a display limitation rather than a functional issue.

Symptom

  • Discrepancy between restricted account and base account when the measure is formula-based.
  • The base account calculates correctly, but the restricted version of the same account returns a different value.
  • The restricted story filter does not pass the account properties into the calculation measure, causing mismatched results.
  • When the account is used as-is, the formula-based measure works correctly.
  • When the account is used as a restricted account, the dimension values from the restricted account are not passed into the measure formula.
  • This behavior used to work before the Q1 patch, and the same formula-based measure functioned properly last year.

Environment

  • SAP Analytics Cloud (Enterprise)

     

Reproducing the Issue

  1. Login to the tenant using Access4 Support User.
  2. Open the story filtered for Profit Center = xxx.
  3. Observe that the account row for Account ID = xxx calculates correctly.
  4. Observe that the restricted version of the same account (Account ID = xxx + Profit Center = xxx) does not produce the same value and appears not to pass the account properties into the calculation measure.

Cause

This issue is currently under investigation by development.

Resolution

This issue is currently under investigation by development.

Symptom

  • The credit value recorded does not match the value determined in G110 register.
  • The discrepancy persists even after implementing the solution suggested in SAP Note 2629974 and configuring the Rounding Type field in the customizing view.

Environment

  • SAP ERP
  • SAP S/4HANA
  • SAP S/4HANA Cloud Private Edition
  • SAP Tax Declaration Framework for Brazil
  • DRC Brazil Statutory Report
  • CIAP

Reproducing the Issue

  1. Implement SAP Note 2629974 and configure the Rounding Type field in the customizing view as suggested.
  2. Generate a SPED EFD ICMS/IPI report.
  3. Observe the discrepancy between the ICMS credit value posted to FI and the credited value calculated for the register G110.;

Cause

The discrepancy between the credit value in the accounting document and the G110 register arises due to different calculation methods and rounding configurations in the system. In CIAP, each installment (CIAP_D_QUOTA-POSSIBLE_CREDIT) is multiplied by the credit factor, and the results are summed to create the accounting document. In SPED, the calculation involves multiplying G110-SOM_PARC (the sum of all installments) by the credit factor, which generates the G110-ICMS_APROP value. Since the credit factor contains more decimal places than FI can process, the difference in calculation order can lead to noticeable discrepancies.

Resolution

  1. Review SAP Note 2629974 for detailed information on rounding discrepancies and possible adjustments.
  2. Configure the Rounding Type field in the customizing view to mitigate the issue.
  3. Discuss with the fiscal team how to address the rounding discrepancies. Manual adjustments in FI can be performed by creating a new document to account for the delta between the accounting and G110 values.

Symptom

 Known restrictions for SAP HANA Automated Predictive Library (APL) 2608

Solution

You can find information about this release in the What's New documentation on the SAP Help Portal.

The following restrictions apply to this release:

  • AFL Performance Considerations
    The AFL generation process is a necessary process, but it can be time consuming, especially where the tables being used contain a large number of columns. It is recommended to avoid generating these wrappers for every single APL function call. If possible, the generated wrappers should be re-used as much as possible. This can be achieved by using the APL Deployment Unit and its stored procedures as opposed to directly calling the APL functions. See the APL Developer Guide for more information.
  • SAP HANA Table and Column Names
    When defining and using table types with APL, blanks, double quotes ("), single quotes ('), tab (\t), and return (\) are not supported in column names: The AFL wrapper generation fails when a table type contains column names with spaces. In procedure mode, this results in the error message "Invalid char (blank) in a field name of <table name>". In direct mode, this results in the following error message: "AFL error exception: AFL error: Internal error : generating parameter string failed."
    When using the RECOMMEND function, the table name for the dataset must be KXLINKS1.
  • Precision Issue for the DECIMAL Data Type
    If you run a function whose input and output datasets have a DECIMAL column, the input value precision might be altered in the output table.
  • Change of Behavior of Padding for CHAR(N) Data Type Columns Starting with SAP HANA 2.0 SPS03
    See SAP Note 2680891.
  • Model Name When Exporting as a Stored Procedure
    When exporting an SAP HANA Automated Predictive Library (APL) model as a stored procedure, the first character of the model name cannot be a number.
  • Certain Geolocation-based Analysis Graph Types are not Supported
    Proximity, Colocation, and Frequent Path analysis graphs are not supported.
  • Statbuilder Cannot be Created with Multiple Input Datasets
    The CREATE_MODEL function cannot create a Statbuilder model if multiple datasets are used.
  • Confusion Matrix and Multinomial Classification Model
    The COMPUTE_CONFUSION_MATRIX function does not support multinomial classification models. It only supports binary classification models (legacy and gradient boosting).
  • Export Profit Curves and Multinomial Classification Model
    The EXPORT_PROFITCURVES function does not support multinomial classification models.
  • Export Apply Code Restrictions
    The EXPORT_APPLY_CODE function does not support time series and social network analysis models. For regression (gradient boosting), binary classification (gradient boosting), and multinomial classification (gradient boosting), only JSON export is supported.
  • Gradient Boosting: interoperability issue between platforms of different endianness
    A gradient boosting model created on an IBM Power Systems (Big Endian) platform can't be reloaded on an Intel or IBM Power Systems (Little Endian) platform. Conversely, a gradient boosting model created on a "Little Endian" platform can't be reloaded on a "Big Endian" platform.
  • ANY Procedure: SQL Data Type Restrictions
    The SQL data types used in the dataset columns are not converted when running an APL function with the ANY procedure. Therefore, they must comply with SAP HANA AFL restrictions. The only data types allowed in datasets are: "INTEGER", "BIGINT", "DOUBLE", "CLOB", "NCLOB", "VARCHAR", "NVARCHAR", "DATE", "TIME", "TIMESTAMP", "SECONDDATE", and "DECIMAL".
  • Time-Series model: slowdown with CREATE_MODEL_AND_TRAIN, TRAIN_MODEL, and FORECAST functions
    A change in the model selection strategy for time-series prevents SAP HANA APL from optimizing the model on shorter horizons than the one requested by the end user. This may increase the training time.

Upgrade from a version prior to 1903 (for HANA 2+):

Due to the change to the software component version of SAP HANA APL 1903, upgrades must follow a specific process:

Note: Make sure that you use the SAP HANA resident lifecycle management tool hdblcm:
/hana/shared/$SAPSYSTEMNAME/hdblcm/hdblcm
$SAPSYSTEMNAME is automatically populated. If $SAPSYSTEMNAME is not set up, use <SID> instead.
Do not use the SAP HANA APL installer’s hdblcm tool.
  1. Run ./hdblcm --action=update_components --component_dirs=<Extract_APL_Folder_Path>/installer --ignore=check_version
  2. Follow the instructions on screen. The software component is sap_afl_sdk_apl. 

Note: Use the normal upgrade procedure to upgrade from a version higher than or equal to 1903 to version 2508.

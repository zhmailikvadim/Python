Symptom

Some or all portfolio Items with "Closed" status are displayed in the "Active Items" dashboard in "Portfolio Management" > "Portfolio Items".

Environment

  • SAP Portfolio and Project Management (PPM)
  • SAP Portfolio and Project Management for SAP S/4HANA (PPM)
  • SAP Portfolio and Project Management in SAP S/4HANA (PPM)

Reproducing the Issue

  1. Access the Portfolio Items dashboard, tab "Active Items".
  2. See that some items with status "Closed" are shown in the dashboard.
  3. Your expectation is that these items should appear in the "Inactive Items" dashboard instead.

Cause

The "Status Group" value in an item controls whether an item appears in the "Active Items" or "Inactive Items" dashboard. The status group which is assigned to the item when saving an item after changing its status is based on the customizing "Define Portfolio Item Types". If an item with status "Closed" appears in tab "Active Items", the likely cause is that in the customizing this status is assigned to an "active" status group, or it was assigned to such a status group in the past (i.e. at the time that the "Closed" status was set).

Technical information: the value of field STATUS_GROUP in table /RPM/ITEM_D_DER (which is the table containing the derived item attributes) controls whether an item appears in the "Active Items" or "Inactive Items" dashboard. Items with the following status group values will not be shown in the "Active Items" dashboard: 0500 Canceled, 0700 Closed, 0800 Rejected, 1000 To be Archived, 1100 Archived

Resolution

  1. Check the customizing for the relevant item type (SPRO > SAP Portfolio and Project Management > Portfolio Management > Global Customizing > Portfolio-Independent Settings): Ensure that the status "Closed" is correctly assigned to status group "0700" (Closed). This should ensure that the correct status group is assigned to an item when the item status is set to "Closed" in future.
  2. With regard to items which were closed or completed in the past (i.e. before correcting the above customizing): saving a change to the item in the frontend triggers an update of the derived item attributes and will therefore bring the status group in line with the current item status in accordance with the current customizing. To avoid having to modify and save each affected item in the Portfolio Management frontend in order to trigger an update of the status group, it should also be possible to trigger this update by running program RPM_PROJECT_SAVE_DERIVED_DATA in transaction SE38 for the affected items.

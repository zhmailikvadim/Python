Symptom

When upgrading the Data Services from 4.2 to Data Services 4.3 SP 3 Patch 10, the DpBridgeAdapter stops to run with error: java.lang.NoClassDefFoundError.

Other Terms

This issue is tracked by CM:002075125800000878952026.

Environment:

  • SAP Data Services 4.2 upgrade to SAP Data Services 4.3 SP 3 Patch 10

Steps:

  1. The original system is with Data Services 4.2 installed. 
  2. Upgrading to the Data Services 4.3 SP 3 Patch 10, then the DPBridgeAdapter cannot be started automatically.

Reason and Prerequisites

The root cause is the tp upgrading which the commons-loggings are not correctly added in the DPBridgeAdapter.

Solution

As a workaroud, please follow the steps below to fix the classpath issue:

  1. Logon the SAP Data Services Management Console. 
  2. Navigate to the Adapters -> Adapter Instance -> Adapter Configuration page.
  3. In the classpath, please added the string ";<LINK_DIR>/ext\lib\commons-logging-1.3.5.jar" in the classpath fied, then save and restart the adapter instance.

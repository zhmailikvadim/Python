Symptom

  • Database error occurs when running a combined query in a Web Intelligence document based on hana universe having “END_SQL = LIMIT XX” parameter set
  • Error message: The following database error occurred: SAP DBTech JDBC: [257] )at 206): sql syntax error: incorrect systax error near “Limit”: Line 14 col 1 (at pos 205). For more information about this error, please refter to SAP knowledge Base Articale 2054721 on the SAP Support Portal. (IES 10901) (Error: INF)
  • Incorrect SQL generated when using combined query

Other Terms

wrong, statement, union, webi, webintelligence, ( ) brackets, parenthesis, missing

Reason and Prerequisites

Environment:

  • SAP BusinessObjects Business Intelligence Platform 4.2 Support Package 03
  • Hana  database
  • Web Intelligence

Steps to Reproduce :

  1. Logon to Information Design Tool (IDT)
  2. Create a JDBC connection to HANA DB and then create a single source data foundation
  3. Create a business layer based on it
  4. Click Parameters and then set END_SQL as "Limit <number>” (For example: Limit 100)
  5. Create a Web Intelligence document based on the universe
  6. Drag some objects to Result Query panel
  7. Check that, in the SQL, "Limit 100" was added to the end of the sql
  8. Run query
  9. The query successfully runs without any error
  10. Got to Query Panel
  11. Click Add a combined query icon to add a combined query
  12. Run query
    1. >> Database error occurs
  13. Open the Query Script viewer, the SQL generated is as below:
    (
    SELECT
    Table__1."SHOP_NAME",
    Table__1."MANAGER"
    FROM
     "EFASHION"."OUTLET_LOOKUP"  Table__1
    UNION
    SELECT
     Table__1."SHOP_NAME",
     Table__1."MANAGER"
    FROM
     "EFASHION"."OUTLET_LOOKUP"  Table__1
    )
    Limit 100
  14. Select Use custom query script, edit the SQL as below
    ----
    (
    SELECT
    Table__1."SHOP_NAME",
     Table__1."MANAGER"
    FROM
     "EFASHION"."OUTLET_LOOKUP"  Table__1)
    UNION
    (SELECT
     Table__1."SHOP_NAME",
     Table__1."MANAGER"
    FROM
     "EFASHION"."OUTLET_LOOKUP"  Table__1
    )
    Limit 100
    ----
  15. Run query
    1. >> The query runs successfully this time

Cause:

  • This issue is caused by a defect in the product.

Solution

This issue is fixed in the patches listed in the "Support Packages & Patches" section below.
The "Support Packages & Patches" section will be populated with the relevant patch levels once they are released.
For Business Intelligence Platform maintenance schedule and strategy see the Knowledge Base Article 2144559 in References section.

Workaround:  

  • Use custom query script
  • Manually add the missing parenthesis in the SQL

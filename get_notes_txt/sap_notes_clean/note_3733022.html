Symptom

For the electronic invoice XML version 5.0, the following adjustments are required:

  • In future, the tax points in the fields unit_mt and unit_tt may no longer be divided by the number of participants. On the other hand, the quantity or Number (quantity) of the service divided by the number of participants. The taxpoint value and the scaling factor (scale_factor_mt and scale_factor_tt) must have the full original value (without division). As before, the number of participants is to be returned in the group size field.
  • For invoices according to XML 5.0, only the field NLEI-LFDTSI is delivered as a session code in XML/invoice printing. As a result, the field "SC" (NLEI-WNDNR) is no longer displayed for services from earlier invoices. Since a session number does not yet exist in the field NLEI-LFDTSI for these services, the default value "01" is transferred for the session.
  • For invoices according to XML 5.0, only the field NLEI-SECTION_CODE is delivered as the specialty (section_code) in XML/invoice printing. As a result, the "Specialty" field (TNKFO-FACHR) of the assigned organizational unit is no longer displayed for services from earlier invoices.

Other Terms

MEDIDATA 4.0, XML standard 5.0, INXML, electronic invoice, EOXML, electronic invoice for external order, invoice printing

Reason and Prerequisites

Prerequisites:

  • Country Version Switzerland (CH) 
  • Use of XML standard 5.0 (for INXML or EOXML or invoice printing)  

Reason:

  • Adjustments to the new version 5.0 for the electronic invoice

Solution

Implement the source code corrections and perform the manual tasks.

The following changes are made:

  • In the case of a group therapy service, only the quantity or Quantity of the service divided by the number of participants.
  • Before 1/1/2026, the old determination logic is used again for the transfer of the session code (session_nr) and the session code is read from the field "SC" (NLEI-WNDNR).
  • Before January 1, 2026, the old determination logic is used again for the transfer of the user department (section_code) and the user department is read from the "User Department" field (TNKFO-FACHR) of the assigned organizational unit.

Symptom

Multiple issues occur in the India localization scenarios across sales and purchasing documents, including incorrect behavior when linking down payments, copying documents, validating excisable items, retaining GST attributes, handling place of supply, serial/batch enablement, copying GST tax invoices, and using freight with non‑GST items.

Reproducing the issue

Scenario 1: Net amount drawing fails when system currency is different than local currency (down payment link)

Prerequisites:

The GST configuration is enabled.

Scenario:

  1. Log in to the SAP Business One client.
  2. Create an A/R down payment invoice for any item with quantity 1, pay it in the Payment Means form, and add the A/R down payment invoice.
  3. Log in to the SAP Business One, Web client.
  4. Create an A/R invoice for the same customer, branch, item, and tax code as in the created A/R down payment invoice and increase the item quantity to two.
  5. Open the Down Payment to Draw form and select the down payment created in step 2.
  6. Set the Net Amount to Draw and choose Update.
  7. Choose Add & View.

Expected behavior:

The A/R invoice is added successfully.

Actual behavior:

An error message appears: There is a difference between the document total and its components. [A/R Invoice – Document Total (SC)].

 

Scenario 2: Copy from a delivery to an A/R invoice fails

Prerequisites:

The GST sequence determination is activated.

Scenario:

  1. Log in to the SAP Business One, Web client.
  2. Open the Deliveries tile.
  3. Select a delivery and choose Copy To → A/R Invoice.

Expected behavior:

The A/R invoice is created successfully.

Actual behavior:

An error message appears: Internal error (-10) occurred.
Copying succeeds only when sequence numbers are unlocked.

 

Scenario 3: Incorrect error message appears for standalone A/R or A/P invoices containing excisable items

Prerequisites:

Excisable item and excisable warehouse are defined.

Scenario:

  1. Create an A/R invoice or an A/R credit memo (standalone).
  2. Select an excisable item and an excisable warehouse.
  3. Add the document.

Expected behavior:

A correct error message is displayed: Cannot create a standalone AP/AR invoice or AP/AR credit memo with excisable items.

Actual behavior:

An incorrect error message appears: There are Non‑excisable Item(s) with CENVAT tax code.

 

Scenario 4 - “Supply Covered Under Sec. 7 of IGST Act” field resets after item selection

Prerequisites:

The GST configuration is enabled.

Scenario:

  1. Open a purchase quotation.
  2. Go to the Tax tab and set the Supply Covered Under Sec. 7 of IGST Act field to Yes.
  3. Add an item.

Expected behavior:

The field Supply Covered Under Sec. 7 of IGST Act remains Yes.

Actual behavior:

The field Supply Covered Under Sec. 7 of IGST Act resets to No.

 

Scenario 5 – The Place of Supply field is not refreshed after deleting the first document row

Prerequisites:

Warehouses 01 and 02 are configured with different states.

Scenario:

  1. Create a purchase request or purchase quotation.
  2. In the first row, add an item with warehouse 01.
  3. In the second row, add an item with warehouse 02.
  4. Delete the first row.

Expected behavior:

The Place of Supply field is refreshed to match the state of warehouse 02.

Actual behavior:

The Place of Supply field remains unchanged with the state of warehouse 01.

 

Scenario 6 - Net amount editing fails in an A/R credit memo created from an A/R invoice

Scenario:

  1. Create an A/R down payment invoice.
  2. Create the corresponding A/R invoice in the web client.
  3. Draw the net amount.
  4. Copy the A/R invoice to an A/R credit memo.
  5. In the Total Down Payment section, set the Net Amount to Draw to a lower amount.
  6. Add the document.

Expected behavior:

The A/R credit memo is added successfully.

Actual behavior:

You cannot add the A/R credit memo after editing the net amount.

 

Scenario 7 - Serial/Batch should be disabled when “Without Qty Posting” is selected

Prerequisites:

A GST Debit Memo transaction type is defined.

Scenario:

  1. Create an A/R invoice and set the Transaction Type to GST Debit Memo.
  2. Select a serial/batch item.
  3. Select the Without Qty Posting checkbox.
  4. Select batch/serial number in the Batch/Serial field.
  5. Add the document.

Expected behavior:

The batch/serial number selection is disabled.

Actual behavior:

Document is added, but the selected batch/serial number is cleared instead of being disabled.

 

Scenario 8 - Copy from A/P invoice to A/P credit memo fails for GST tax invoice

Scenario:

  1. Create an A/P invoice and set the Transaction Type to GST Tax Invoice.
  2. Choose Copy From → A/P Credit Memo.

Expected behavior:

The A/P invoice is successfully copied to an A/P credit memo.

Actual behavior:

The copy operation fails.

 

Scenario 9 - Freight is not applicable for non-GST items in the Bill of Supply

Scenario:

  1. Create a goods receipt with a non‑GST item.
  2. Create an A/R invoice with the Transaction Type of Bill of Supply.
  3. Go to the Contents tab, enter the non‑GST item, set a unit price, and add freight.

Expected behavior:

The freight value is acceptable without any errors.

Actual behavior:

A system error is displayed when entering freight for non‑GST item.

Cause

The issues were caused by inconsistent validation and calculation logic in India localization for sales and purchasing documents, leading to incorrect handling of GST rules, document copying, and field behavior in specific scenarios.

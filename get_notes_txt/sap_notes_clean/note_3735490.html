Symptom

When you process returned debit memos using the (FAIR-based) process "Post Savings Subledger" (/IBS/FEB), the condition GUID of the relevant returned debit memo is not transferred to the "Returned Debit Memos Worklist" table (/IBS/TBSC_RD_WL) for all returned debit memos. As a result, no recollection can take place for the affected returned debits.

Reason and Prerequisites

Depending on the sorting of the actual records to be processed (table /IBS/TBSC_BO_REC), the information to be transferred to the "Returned Debit Memos Worklist" table is not transferred from the returned debit memo record, but from the external charge record or own charges record that does not contain a condition GUID.

Solution

In the method "Write Worklist Table RLS" (/IBS/CL_BSC_CNTRL_FEBEP_POST->SAVE_RD_WL), actual records that do not contain a condition GUID are excluded from the selection of ACTUAL records for transferring the information for the entries in the table "Returned Debit Memos Worklist".

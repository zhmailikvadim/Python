Symptom

User is unable to connect a SAP BTP Subaccount to the SAP Cloud Connector due to the error:

com.sap.scc.servlets.RequestFailureExecution$InvalidConfiguration: An authorization problem occurred when downloading the configuration. Ensure that the subaccount exists and that the user has the appropriate permissions for the subaccount. Consult SAP note 2571763 for possible remedies. Token for <SubaccountID>@<BTP_Region> expiring at <YYYY-MM-DD> <HH:MM:SS> +<TimeZone>
...
Caused by: com.sap.scc.cert.HttpRequestException: SCC handshake failed: 403 — Forbidden

Environment

  • SAP Business Technology Platform
  • SAP Cloud Connector

Cause

The user you are using to download the Authentication Data from the subaccount is missing one of the required Role Collections:

  • Subaccount Administrator
  • Cloud Connector Administrator

Resolution

  1. Navigate to BTP Cockpit
  2. Navigate to the subaccount
  3. Security -> Users
  4. Locate the user you are going to use when downloading the Authentication Data file from the subaccount
  5. Add one of the role collections below, as described at Assign Users to Role Collections:
    • Subaccount Administrator
    • Cloud Connector Administrator

IMPORTANT: If accessing BTP Cockpit via a Custom Identity Provider (the URL contains "?idp=<CustomIdentityProvider>"), then make sure the target user belongs to this Identity Provider in BTP Cockpit -> Security -> Users.

Keywords

SCC, BTP, Authentication Data

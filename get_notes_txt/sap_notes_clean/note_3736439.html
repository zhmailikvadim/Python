Symptom

The system generated a short dump when using system analytical engine, e.g. Fiori report, Analysis for Office:

Category               ABAP programming error                                                       
Runtime Errors         UNCAUGHT_EXCEPTION                                                           
Except.                CX_RSR_X_MESSAGE                                                             
ABAP Program           CL_RSPLS_HDB_SWITCH===========CP   
.....
TEXT                              ENQUEUE_STATUS-02-
.....
I_TABNAME                                                                                         
    RSR_TIME_REL 

Environment

  • SAP ERP
  • SAP BW/4HANA
  • SAP S/4HANA

Cause

The dump could be triggered by multiple users requesting to update the table RSR_TIME_REL simultaneously. The situation may occur after some changes in transaction RSRHIERARCHYVIRT that require updating time relations.

Resolution

Check whether there is any lock exists in transaction SM12 for table RSR_TIME_REL:

  • If there is, check and confirm the user is updating the table RSR_TIME_REL by transaction SM50 or SM66. You may wait for a while. After update is finished, the dump will not happen.
  • If there is no, it means the update was finished before, and the lock was released. So the dump will not happen anymore.

Symptom

If Payment Item Postings are made using BAPI's(from Payment Central system or any other exernal system) and if multiple BAPI calls are batched together before a final commit, then the number of Payment Items posted in the Account Management system and corresponding number of BAnkAccountEntry Events transmitted from the Account Management system does not match.

This leads to duplicate events being triggered for the same Payment Item posting.

Other Terms

Business Event, Bank Account Entry, Payment Item, Business Object, Events in TRBK, Payment Item Posting

Reason and Prerequisites

Program Error

Solution

It is recommended to import the relevant Support Package for your release. You can also use transaction SNOTE to implement the correction instructions.

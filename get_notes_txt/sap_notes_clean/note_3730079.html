Symptom

  • In SAP Cloud Connector, the “Check Internal Host” functionality fails with an SSL/TLS error:
2026-01-29 15:54:32,455 +0100#INFO#com.sap.scc.rt#AccessControl connection checker# #SccEndpointValidator for <host>:<port>  can't retrieve further TLS information witk IAIK
iaik.security.ssl.SSLException: Peer sent alert: Alert Fatal: handshake failure
 
  • The error is logged with reference to:
AccessControl connection checker
SccEndpointValidator
 
  • Productive communication (e.g. via SAP BTP application) works successfully
  • No handshake failure is observed in backend ICM logs for productive requests

Environment

  • SAP Cloud Connector
  • Backend system with TLS configuration (including ECDSA-only or restricted cipher suites)
  • SAP BTP connectivity scenarios

Reproducing the Issue

  1. In Cloud Connector, navigate to Cloud to On-Premise and run “Check Internal Host” for the mapped HTTPS backend.

Cause

  1. The behavior is caused by the use of different TLS/SSL implementations within SAP Cloud Connector:

  • Connection Check (Check Internal Host)
    Uses an internal diagnostic SSL library:
    → IAIK SSL Library

  • Productive Communication
    Uses the standard JVM TLS implementation

     2. The IAIK SSL library has limitations in handling certain TLS configurations, especially environments that rely on:

  • ECDSA-based certificates

  • Restricted cipher suites (e.g. ECDSA-only)

Because of this, the connection checker may fail during the TLS handshake even though the productive communication path is fully functional.

Resolution

  1. This is expected behavior.
  • The connection checker is a diagnostic tool only and does not reflect the exact behavior of productive communication.
  • There is currently no option to configure the connection checker to use the JVM TLS implementation

     2. To validate connectivity, perform a productive test instead:

  • Trigger a request from SAP BTP application to the backend system via Cloud Connector
  • Verify:

    1. Backend ICM logs

    2. Cloud Connector logs

If productive communication is successful, the connection checker error can be safely ignored in this context

     3. Additional information

  • The SAP Cloud Connector built-in certificate generator creates RSA-based certificates only
  • If required, an ECDSA certificate can be generated externally (e.g. via OpenSSL) and imported into Cloud Connector for testing purposes

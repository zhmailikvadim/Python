Symptom

There is a business requirement to perform annual cash concentration, but in the standard configuration of the product does not support annual frequency in FIPRD2.

Environment

  • FSCM In-House Cash
  • SAP ERP
  • SAP S/4HANA
  • SAP S/4HANA Finance

Cause

Standard system design.

Resolution

    In the standard, it is possible to achieve this with the 'period' option in transaction F9K2.
    This option specifies the intervals in which the activity is to be executed. The value relates to the period unit specified (days, weeks, months).

    For example: if period/period unit/key date setting is 12/Month/31 then the process will run every 12 months on the last day of the month.

    In this case the next date will be updated on an annual basis after cash concentration is run, but it will depend on the value of the 'next date' field.

    For example: If the field had a value of 19.01.2025 before the process was run, it will update to 19.01.2026 after it.

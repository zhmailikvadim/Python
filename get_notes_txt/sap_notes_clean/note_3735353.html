Symptom

Mass activation of ABAP Dictionary objects terminates with memory-related short dumps such as:

  • SYSTEM_NO_MEMORY
  • SYSTEM_NO_ROLL
  • STACK_STATE_NO_ROLL_MEMORY

The termination occurs inside the V1 postprocessing logic. Analysis of the short dump reveals an extremely deep call stack (tens of thousands of frames), indicating infinite recursion as the actual cause of memory exhaustion. Increasing memory parameters does not resolve the issue.

Other Terms

RDDMASGL, STACK_STATE_NO_ROLL_MEMORY, SYSTEM_NO_MEMORY, infinite loop, infinite recursion, self-referencing structure, circular structure include, V1 postprocessing, CL_DDCMP_DDLS_V1_POSTPROC, circular dependency, ABAP Dictionary, structure activation

Reason and Prerequisites

During V1 postprocessing, the activation framework recursively follows the include hierarchy of ABAP Dictionary structures. If two or more structures mutually include each other — directly or transitively — this recursion has no exit condition, causing an infinite call stack and eventual memory exhaustion.

Solution

Two separate actions are required to fully resolve this issue.

1. Apply the code correction

This SAP Note corrects the V1 postprocessing logic to detect circular include references and terminate the recursion gracefully. Apply this SAP Note or install the corresponding Support Package.

Once the correction is applied, the activation will fail gracefully with an error message identifying the structures involved in the circular dependency, instead of crashing with a memory dump.

2. Fix the circular structure definition

  1. Identify the affected structures from the error message produced after applying the code correction, or from the short dump call stack.
  2. Inspect the include hierarchy in SE11 to confirm the circular reference.
  3. Break the cycle by removing the include that creates the circular dependency.

Symptom

After a successful SAP HANA System Replication takeover, application URLs (for example HTTP/HTTPS services, XS or OData endpoints) are not reachable or return HTTP errors such as HTTP 404.

The database itself is accessible and System Replication shows a healthy status.

Typical observations:

  • hdbnsutil -sr_state shows primary/secondary in sync
  • Takeover completes without replication errors
  • SQL connections to the database work
  • Only application URL or HTTP-based service is not reachable after takeover

The issue may appear only after failover, patching or maintenance activities.

Environment

SAP HANA DB Version 2.0

Cause

SAP HANA System Replication transfers the database role to the secondary system during takeover and ensures database availability.

After takeover the active database host changes. Client requests must reach the host where the active primary system provides the application service.

If the service access endpoint (for example DNS alias, load balancer or hostname) still directs requests to a different host, the database takeover is successful but the application request may not reach the active database node.

This behavior is consistent with the SAP HANA System Replication architecture described in the SAP HANA System Replication Guide and SAP Note 1999880.

Resolution

The service endpoint should resolve to the host that currently holds the primary database role after a takeover.

Possible approaches:

  • Configure a virtual hostname (virtual IP) that follows the active node
  • Use cluster/HA integration mechanisms (HA/DR provider or cluster software)
  • Configure the load balancer health check to select only the active primary database node

After takeover, verify connectivity:

  1. Check active role
  2. hdbnsutil -sr_state

Verify that the service endpoint resolves to the current primary host and test access to the application URL.

In typical scenarios no SAP HANA database parameter changes are required. The issue is usually related to infrastructure or endpoint routing configuration.

Keywords

System Replication, takeover, HTTP 404, load balancer, virtual IP

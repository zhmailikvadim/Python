Symptom

Inflation/revaluation results are incorrect in subsequent periods. The system calculates revaluation using the original APC or an incorrect area instead of the previously revaluated accumulated values.

Images/data in this KBA are from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental. 

Environment

  • SAP ERP
  • SAP S/4HANA

Reproducing the Issue

  1. Execute transaction J1AI. 
  2. Observe the calculation results for the asset revaluation.
  3. The system calculates the revaluation of the APC based on the original APC value instead of the revaluated APC value.

Cause

This issue is caused by inconsistent configuration in the inflation accounting settings:

  • Revaluation Area Configuration
    The revaluation area is not assigned to the correct summed/base area.
  • Revaluation Key Configuration
    The corresponding revaluation key does not contain the required base area entry.

Resolution

Review the inflation accounting customizing in the following IMG activities:

Revaluation Areas

  • Navigate to SAP Customizing Implementation Guide > Financial Accounting > Asset Accounting > Special Valuation > Revaluation of Fixed Assets > Revaluation for the Balance Sheet > Inflation Accounting > Revaluation Areas > Maintain Revaluation Areas.
  • Ensure that the revaluation area is assigned to the correct summed/base area. 

Revaluation Keys

  • Navigate to SAP Customizing Implementation Guide > Financial Accounting > Asset Accounting > Special Valuation > Revaluation of Fixed Assets > Revaluation of the Balance Sheet > Inflation Accounting > Revaluation Keys > Maintain Revaluation Keys.
  • Ensure the corresponding revaluation key contains the required base area entry.

After correcting both settings, execute transaction J1AI again and confirm that the system calculates revaluation based on the revaluated APC instead of the original APC.

Symptom

Error message when converting Purchase Requisition in Purchase Order.

Environment

SAP S/4HANA Cloud Public Edition

Reproducing the Issue

  1. Open the Process Purchase Requisitions app.
  2. Filter for the PR.
  3. Select the PR item (checkbox).
  4. Click Create Purchase Order.

See the pop-up with the error message: Navigation to this application is not supported.

Cause

The behavior for advanced PO creation is updated 2602 onwards.

Resolution

PO details will open in editable mode before it is ordered in the system. As part of this change a new transaction was added to enable this feature, please follow steps below:

  1. Go to app 'Maintain Business Users'.
  2. Search for the user using Process Purchase Requisitions application.
  3. Go to Assigned business roles.

4. Please check the role which contains SAP_MM_BC_PURDOC_PROCESS_PC, eg in this system ZTEST_PURCHASER (it is possible to check for this in the Business Catalog app).

5. Go to this role and navigate to assigned IAM apps:

6. Make sure there is an entry for ME21N_PR_TRAN maintained with Active status checked (if not add and activate this app manually):





7. Save the changes and refresh the session (logout and login) Navigation is expected to work.

NOTE
Even if the Business Catalog is assigned, missing IAM App dependencies must be activated via "Check Dependencies" in Maintain Business Roles.

Symptom

The Reverse Batches and Serials Trace Report duplicates some batches/serial numbers in higher depths, sometimes leading to out of memory and crashing the application.

Reproducing the issue

Scenario:

1. Go to Inventory → Item Master Data, create item A1, in the General tab, choose Managed by Serial Number on every transaction.
2. Go to Inventory → Item Master Data, create items A2, B2, A3, B3, C3, D3, in General tab, choose Managed by Batch on every transaction for all items.
3. Go to Inventory → Inventory Transactions → Goods Receipt, receive A3, B3, C3, D3, all qty = 1, price = 1, in the Batch Allocation popup form, set batch number A3-01, B3-01, C3-01, D3-01. Update and add the Goods Receipt.

4. Go to Production → Bill of Materials, create the following BOMs:
Parent: A1, Children: A2, B2
Parent: A2, Children: A3, B3
Parent: B2, Children: C3, D3

5. Go to Production → Production Order, create Production Order 1:
Type: Standard, Parent: A2, Children: A3, B3.

6. Click the last record to find Production Order 1, change status from Planned to Released.

7. In Production Order 1, right-click → Issue Component, create Issue for Production for A3, B3. In the  Batch Allocation popup form, click Auto Select to select A3-01, Update, choose the 2nd row, click Auto Select to select B3-01. Update and add the document.

8. In Production Order 1, right-click → Report Completion, create Receipt from Production for A2, in the  Batch Allocation popup form, set batch number A2-01. Update and add the document.

9. Go to Production → Production Order, create Production Order 2:
Type: Standard, Parent: B2, Children: C3, D3.

10. Click the last record to find Production Order 2, change status from Planned to Released.

11. In Production Order 2, right-click → Issue Component, create Issue for Production for C3, D3. In the Batch Allocation popup form, click Auto Select to select C3-01, Update, choose the 2nd row, click Auto Select to select D3-01.  Update and add the document.

12. In Production Order 2, right-click → Report Completion, create Receipt from Production for B2. In the Batch Allocation popup form, set batch number B2-01. Update and add the document.

13. Go to Production → Production Order, create Production Order 3:
Type: Disassembly, Parent: A2, Children: A3, B3.

14. Click the last record to find Production Order 3, change status from Planned to Released.

15. In Production Order 3, right-click → Issue Component, create Issue for Production for A2. In the Batch Allocation popup form, click Auto Select to select A2-01. Update and add the document.

16. In Production Order 3, right-click → Report Completion, create Receipt from Production for A3, B3, in the Batch Allocation popup form, set batch number A3-01, click Yes for the popup message "A3-01 already exists, continue?", Update, choose the second row, set batch number B3-01, click Yes for the popup message "B3-01 already exists, continue?". Update and add the document. 

17. Go to Production → Production Order, create Production Order 4:
Type: Standard, Parent: A2, Children: A3, B3.

18. Click the last record to find Production Order 4, change status from Planned to Released.

19. In Production Order 4, right-click → Issue Component, create Issue for Production for A3, B3. In the Batch Allocation popup form, click Auto Select to select A3-01, Update, choose the second row, click Auto Select to select B3-01. Update and add the document.

20. In Production Order 4, right-click → Report Completion, create Receipt from Production for A2. In the  Batch Allocation popup form, set batch number A2-01, click Yes for the popup message "A2-01 already exists, continue?".  Update and add the document. 

21. Go to Production → Production Order, create Production Order 5:
Type: Standard, Parent: A1, Children: A2, B2;

22. Click the last record to find Production Order 5, change status from Planned to Released.

23. In Production Order 5, right-click → Issue Component, create Issue for Production for A2, B2. In the  Batch Allocation popup form, click Auto Select to select A2-01, Update, choose the second row, click Auto Select to select B2-01. Update and add the document.

24. In Production Order 5, right-click → Report Completion, create Receipt from Production for A1. In the  Serial Allocation popup form, set serial number A1-01. Update and add the document.

25. Go to Inventory → Item Master Data, find A1, right-click Serial Transaction Report, in the Serial Transaction Report, right-click Serial Number A1-01 → Reverse Batches and Serials Trace Report.

Expected behavior:

  1. In depth 2, 3 records listed for A2-01, 2 records listed for B2-01
  2. In depth 3, 3 records listed for A3-01, 3 records listed for B3-01
  3. Only 3 depths exist.

Actual behavior:

  1. In depth 2, 4 records listed for A2-01
  2. In depth 3, 4 records listed for A3-01, 4 records listed for B3-01
  3. Depth 4 exists: 4 records for A2-01 listed
  4. Depth 5 exists, 4 records for A3-01 listed, 4 records for B3-01 listed. 

Solution

SAP intends to provide a feature package or patch to solve the symptom that is described in this SAP Note. The section References below lists the related feature package or patch once they become available. Be aware that References can only be confirmed at feature package or patch release date. SAP delivers feature packages or patches only for selected releases at its own discretion, based on the business impact and the complexity of implementation.

Workaround: 

Do not use Serial Transaction Report → right-click Serial Number A1-01 → Reverse Batches and Serials Trace Report.

Use Inventory → Inventory Report → Reverse Batches and Serials Trace Report to open Selection Criteria and check option Omit Disassembly Transactions to Improve Performance to run the report.

Other terms

SnB, hang, oom, block, shut down

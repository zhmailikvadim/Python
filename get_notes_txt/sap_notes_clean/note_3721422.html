Symptom

While applying Manual Post-Implementation Step 2 after implementing note 3589206 an error occurs:

Invalid BC Set activation component assignments (Message number SCPR286)

Environment

• SAP ERP

• Poland 

Reproducing the Issue

  1. Implement SAP Note 3589206 - PL SAF-T JPK_KR_PD Cross-sol: Technical Updates to Cross-Solution Configuration.
  2. Implement SAP Note 3618417 - JPK_KR_PD cust tables - BC Set adjustment
  3. Make sure no objects are locked in transport
  4. Attempt to activate BC Set HBCS_FIN_LOC_SRF_PL_SAFT_KR_PD using transaction SCPR20.
  5. Observe the error: "BC Sets with invalid software component: S4CORE".

Cause

• SAP Note 3618417 - JPK_KR_PD cust tables - BC Set adjustment has not been implemented yet.

• Certain objects are locked in transport, preventing activation.

• The current system release is for SAP ERP, whereas the BC Set contains the S4CORE software component and release, leading to incompatibility. 

Resolution

Step 1: Ensure that SAP Notes 3589206 and 3618417 are fully implemented.

Step 2: If Step 1 did not help, it has to be set manually:

a. Go to transaction SCPR3.

b. Locate BC Set HBCS_FIN_LOC_SRF_PL_SAFT_KR_PD.

c. Edit the properties to update the software component to SAP_FIN and set the release to match (e.g., SAP_FIN - 618).

d. Activate the BC Set using transaction SCPR20.

Step 3: If Step 2 does not populate the required tables automatically, manually add the data to the tables SNC_PL_FINTAG and SNC_PL_PDTAG.

a. Refer to the Polish government link for official tags and descriptions:

  • Struktury JPK w podatkach dochodowych

Information for FINTAG and PDTAG can be found in Schemat_JPK_KR_PD(1)_v1-1.xsd file, but this information should be in the system after the activation of the BC Sets. 

b. Extract the data from the BC sets or use the above xsd file available in the link.

c. Populate the tables with valid tags and descriptions for all business types from 1 to 7, ensuring validity dates are set appropriately (e.g., 01.01.2025 to 31.12.9999).

d. Verify that the transactions for the PL SAF-T reports are now accessible.

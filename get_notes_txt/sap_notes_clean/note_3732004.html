Symptom

Maintenance items referenced task lists whose headers have system condition values (for example, “not in operation” and “in operation”), but after maintenance order called by maintenance plan, the order header does not reflect a system condition.

Environment

SAP S/4HANA Cloud Public Edition

Reproducing the Issue

  1. Create maintenance task list with system condition value maintained.
  2. Create maintenance plan and maintenance item, assign task list to maintenance item.
  3. Schedule maintenance plan to generate call object maintenance order.
  4. Check the maintenance order header, observe that system condition value is not reflected.

Cause

While defining a maintenance process for a piece of equipment or a functional location, define the target system condition when the maintenance job is carried out by the maintenance technician. If a piece of equipment has to be shut down during maintenance and if shutting down this equipment has an impact on production, this has to be planned upfront and not on the day of the maintenance. This allows to plan the maintenance of the pieces of equipment in an optimal manner.

In the Create Maintenance Plan app, for maintenance plans of the category Maintenance Order, select the appropriate operating system condition value and assign it to a maintenance item. During scheduling, when the maintenance call is called which results in the creation of a call object of type Maintenance Order, the maintenance order header will have the system condition. The system condition can then be used for further business processes wherever it is relevant.

Resolution

Maintain system condition for maintenance item through app Manage Maintenance Items, so the system condition value will reflected in maintenance order.

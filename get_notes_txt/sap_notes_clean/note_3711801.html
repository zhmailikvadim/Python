Symptom

  • The SAP RFC Gateway service (sapgwXX) restarts automatically 
  • The restarts occur randomly and not on all application servers at the same time.
  • No manual restart is performed by administrators.
  • The issue is observed intermittently.

Upon reviewing the dev_rd the following errors are recorded:

  • NiIRead: recv (73: Connection reset by peer)
  • SI_ECONN_BROKEN
  • GwRead failed (NIECONN_BROKEN)
  • invalid ni hdl
  • Followed by:
  • GwStopGateway, gateway stopped.

The source IP address in the logs corresponds to a vulnerability scanning tool (for example, Nessus OS Vulnerability Scanner). 

Environment

  • SAP NetWeaver
  • SAP NetWeaver Application Server for SAP S/4HANA
  • ABAP PLATFORM - Application Server ABAP

Reproducing the Issue

  • Run a vulnerability scan against the SAP application server network.

  • The scan attempts TCP connections on the SAP Gateway port (sapgwXX).

Cause

The SAP Gateway restarts are caused by  vulnerability scans resetting TCP connections on the Gateway port, leading to internal network handle errors and triggering a self-protective Gateway restart. The behavior is random because scans do not target all application servers simultaneously.

Resolution

By default, the RFC Gateway internal port is dynamically assigned, which makes it difficult to control or exclude at the vulnerability scanner level. Configuring a static port instead of a dynamic one allows the port to be explicitly identified and blocked or excluded in the scanner configuration.

A fixed port number can be configured using the parameter gw/internal_port.

To implement this, set
gw/internal_port = <port_number>
in the instance profile and restart the SAP instance for the change to take effect.

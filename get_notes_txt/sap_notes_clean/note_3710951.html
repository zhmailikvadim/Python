Symptom

When a TR document containing a return item is posted, the system should update the corresponding matrix value with a negative amount. However, the amount is currently updated as a positive value.
This note corrects the behavior by ensuring that the return indicator is properly considered during the matrix update.

Other Terms

TR, Transaction register, Return indicator, SHKZG, Matrix, Negative, Post, /IRM/IPMXU

Reason and Prerequisites

This is caused by program error

Solution

Please implement the correction instructions attached. These corrections are only valid if you have applied the Support Packs.

Release V/4 - 2022 V2, 2023, 2023 V2, 2023 V3 and 2023 V3 Support Pack 1

Use transaction SNOTE to implement the correction instructions specified below and activate the affected program object.

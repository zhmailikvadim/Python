Symptom

  • After enabling the "Enhanced People Search" in Company System & Logo Settings and completing the company onboarding process, the enhanced directory search does not display all employee details such as Title, Location, Group, and Business Unit. Only the employee name is visible.
  • The issue occurs when proxying as certain users and performing an advanced search in the directory tab. Only the employee name is visible.
  • All prerequisites and permissions were verified and configured correctly as per SAP Help Documentation.

Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.

Environment

  • SAP SuccessFactors HCM Suite

Reproducing the Issue

  1. Login to the instance.
  2. Navigate to Company System & Logo Settings.
  • Locate the setting on the page called "Enable Enhanced People Search in the Global Header and the Directory Search".
  • Notice the settings have been enabled.

    3. Proxy as user with the user ID provided in the case information.

  • Navigate to Menu -> Company Info.
  • Click on the Directory tab.
  • In the Advanced Search fields, select business unit "Type in the Business Unit" in the Business Unit advanced search field.
  • Select Search.
  • Note that the employee names display in the search results but no values are displayed in the Job Title, Location, Group, and Business Unit columns. The expected result is that the user is able to see these values in the directory search results.

Cause

User does not have permission to view the search result employees' Title, Location, Group, and Business Unit fields.

Resolution

  1. Verify the Role-Based Permissions (RBP) configuration for the affected user.
  2. Use the admin tools "User Role Search" or "RBP Troubleshooting" to view the permission configuration.
  3. Add the appropriate permissions to the affected user's role to allow viewing of the Title, Location, Group, and Business Unit fields in the enhanced directory search.
  4. After updating the permissions, there is some caching time, after that, the user is able to view these fields in the enhanced directory search.

Symptom

The behavior happens when you try to print the requisition, where the PR print only shows one line item, despite the requisition containing two line items or more. It does no affect all requisitions, but some.

Reproducing the Issue

You open the requisition, and click on Print. Once the layout for the print appears you notice that it is not showing all the line item details as you can see on the system. 

Cause

This behavior happens when copying a requisition where the tax details are missing for the line items in the original requisition.

Resolution

To resolve and avoid facing this situation in the future a parameter can be used Application.Purchasing.SetTaxCodeAsNullOnRequisitionCopyWhenTaxDetailsAreMissing.
This parameter determines whether the tax code is set to null in a copied requisition when the tax details are missing for the line items in the original requisition.

When set to Yes, the tax code is set to null in the copied requisition when the tax details are missing for the line items in the original requisition.
When set to No, the tax code from the original requisition is retained for the line items in the copied requisition.

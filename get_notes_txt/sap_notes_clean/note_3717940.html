Symptom

  • After a fresh installation of MSSQL and SAP Data Services unable to register newly created MSSQL local repository in CMC > Data Services: There is a problem to connect to repository: [The TCP/IP connection to the host XX.XX.XX.XX, port 1433 has failed. Error: "Connection refused: connect (local port XXXXX to address 0:0:0:0:0:0:0:0, remote port 1433 to address XX.XX.XX.XX (hostname)). Verify the connection properties. Make sure that an instance of SQL Server is running on the host and accepting TCP/IP connections at the port. Make sure that TCP connections to the port are not blocked by a firewall.". ] 
  • MS SQL Server Management Studio connection to database works
  • Repository creation in Data Services Repository Manager works
  • Repository can be associated with the job server in Data Services Server Manager

Environment

  • SAP Data Services 14.x
  • MSSQL Database
  • Windows

Reproducing the Issue

  1. Create an empty database using MS SQL Server Management Studio
  2. Create repository using Data Services Repository Manager
  3. Associate repository with the job server using Data Services Server Manager
  4. Try register repository in CMC > Data Services

Cause

TCP/IP protocol is disabled

Resolution

  1. Check local port value in  MS SQL Server Management Studio:
  2. SELECT local_tcp_port  FROM sys.dm_exec_connections WHERE session_id = @@SPID;
  3. If the value is NULL, Open SQL Server Configuration Manager > SQL Server Network Configuration > Protocols for MSSQLSERVER
  4. Toggle TCP/IP from Disabled to Enabled using right-click.
  5. Restart MSSQL Server service.

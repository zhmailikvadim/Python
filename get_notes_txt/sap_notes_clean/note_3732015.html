Symptom

  • Users experience an authorization issue in the Manage Maintenance Notifications app where the Edit action is disabled for existing notifications in app Manage Maintenance Notifications.
  • Users assigned with business catalog SAP_EAM_BC_MN_MNG_EMP_PC can create maintenance notifications but cannot update or edit existing notifications.

Environment

SAP S/4HANA Cloud Public Edition

Reproducing the Issue

  1. Open the app Manage Maintenance Notifications.
  2. Select any existing notification.
  3. Observe that the Edit option is disabled.

Cause

Users with business catalog SAP_EAM_BC_MN_MNG_EMP_PC assigned to update notifications only when more information is needed, i.e. the notification must be set to 'Action Required' by a supervisor, and editing is then available to the creator or the person responsible.

Resolution

  1. Assign the business catalog SAP_EAM_BC_MN_MNG_EMP_PC to employees who should create and update but not accept/reject notifications.
  2. For notifications that require employee input, have a supervisor set the notification to Action Required (via Actions → Action Required).
  3. Ensure the employee intended to edit is either the notification creator or is maintained as Person Responsible.

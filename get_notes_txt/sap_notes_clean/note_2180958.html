Symptom

The user runs the closing step of the alternative valuation run.

Transaction CKMLCPAVR
=> Specify a Costing Run
=> Tab strip:  Processing
=> Flow Step: Post Closing
=> Execute

When the closing step is executed, a dump is raised at:

Class:    CL_FINS_ACDOC_ML_UTIL
Method: MOVE_CURRENCY_FIELDS
Line:      90

 *     Get LS_MLIT from Runtime Buffer
      DATA(ls_mlit) = get_mlit_from_acdoc_item( is_acdoc_item ).
      ASSERT ls_mlit IS NOT INITIAL.

*     get the right MLCR-line belonging to the ACDOCA line
      READ TABLE lt_mlcr REFERENCE INTO DATA(lr_mlcr)
           WITH KEY posnr = ls_mlit-posnr
                    bdatj = is_acdoc_item-r_origin_accit->gjahr
                    poper = is_acdoc_item-r_origin_accit->monat
                    curtp = lv_curtp.
      ASSERT sy-subrc = 0.    <== Dump

*     Put the values from MLCR
      lr_acdoc_item_curr->vstprs = lr_mlcr->stprs_old.
      lr_acdoc_item_curr->vpeinh = lr_mlcr->peinh.

 

Further places where a dump can be raised:

When the AVR runs in background mode then a dump can also be raised at:

Class:    CL_FINS_ACDOC_ML_UTIL
Method: GET_MLPP_FROM_ACDOC_ITEM
Line:      28

*   Get the right MLPP from the Run-Time Buffer
    get_mldoc_from_acdoc_item(
      EXPORTING
        is_acdoc_item = is_acdoc_item
      IMPORTING
        et_mlpp       = DATA(lt_mlpp) ).
    READ TABLE lt_mlpp INTO rs_mlpp
         WITH KEY posnr = ls_mlit-posnr
                  bdatj = is_acdoc_item-gjahr
                  poper = is_acdoc_item-monat.
    ASSERT sy-subrc = 0.    <== The Dump occurs here
  ENDMETHOD.

Other Terms

CKMLCPAVR, No Delta Logic, AVR_POSTING_NO_DELTA, AVR

Solution

Apply the attached correction instruction

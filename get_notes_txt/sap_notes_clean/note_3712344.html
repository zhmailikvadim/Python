Symptom

Symptom

When overfill is triggered from Settlement, CPE price fixation evaluation gets triggered from contracts code and it triggers CPE for all the available groups.
This results in degraded performance. Ideally only group 0 evaluation should be triggered.

Other Terms

Agricultural Contract Management; ACM; Advanced CPE; Settlement; Overfill

Reason and Prerequisites

Performance

Solution

Import the relevant support pack or apply the correction instruction(s).

This SAP Note delivers performance improvement for overfill scenarios from settlement. CPE price fixation evaluation
will only be triggered for group 0 now, which will result in overall performance improvement.

Other Terms

STAPE, S4CLD, SF Userid, REF_WA_ID

Reason and Prerequisites

  1. You are posting to an S/4 Public Cloud System, it means V_T77S0 switch S4CLD is set to 'X' and SOAMANAGER configuration is in place. 
  2. You are using SAP Successfactors Employee Central and possibly Employee Central Payroll. Your Employees are created in SFSF EC and replicated to the payroll system. 
  3. You are using account assignment type Q for posting with personal number and therefore set V_T77S0 switch STAPE to 'X' to get a valid PERNR on S/4 Public Cloud System. 
  4. You are using BAdI BADI_HRPP_ACCOUNT_ASSIGNMENT to change the company code as company codes differ between payroll system and S/4 Public Cloud System. 

Solution

Implement the attached correction instruction to avoid the error.

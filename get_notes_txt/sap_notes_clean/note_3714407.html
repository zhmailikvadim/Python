Symptom

When creating a sales order referring a sales order, the freight condition in sales order is copied to the return order but not redetermined.

Environment

SAP S/4HANA Cloud Public Edition

Reproducing the Issue

  1. Create a sales order with a freight condition.
  2. Create a return order referring the sales order.
  3. Check the pricing in the return order.

Cause

The system behaves as designed.

The pricing type in related copying control is set as G-"Redetermines taxes, internal prices, condition category L". As a result, the freight condition, whose condition category is F-Freight, will not be redetermined.

Resolution

The workarounds below can be considered according to the business requirement:

  • To manually update the pricing using the pricing type H-Redetermines Freight; Condition Category L when creating return order.
  • If the freight condition is a custom condition type, it can be considered to modify the condition category of the freight condition from F-Freight to L-Generally new.

Keywords

Return Order, Condition Type, Pricing Type, Copying Control, condition category, CBAR, Freight.

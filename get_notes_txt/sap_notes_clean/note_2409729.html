Symptom

When processing a dimension, the following error is displayed in the BPC Admin Client:

Server Time Out

Upon inspecting the tblLogs table, here is no specific error recorded and only the following entries are displayed:

SystemName: CommonService 

JobName: GetStatusMessage

The messages are similar to the one below:

Client: <param name='ClientID'><ClientID>06bd5f2d-c6de-459e-93c4-72e6cdce6d9c</ClientID></param>
Return: <NewDataSet>
<Table1>
<clientID>06bd5f2d-c6de-459e-93c4-72e6cdce6d9c</clientID>
<messageType>5</messageType>
</Table1>
</NewDataSet>

 

Environment

  • SAP Business Planning and Consolidation version 10.x for Microsoft

Reproducing the Issue

  • Launch the BPC Admin Client.
  • Select a dimension.
  • Select Process Dimension.

The error message is displayed after approximatively 80 seconds or so.

Cause

The private memory limit of the IIS Application Pool associated with the /SAP/BPC application has a configured value larger than zero (default) and that value is smaller than the amount of memory consumed by the applications associated with the application pool.

Resolution

Configure the Private Memory Limit setting of the associated application pool to either zero or a value large enough as to prevent IIS from recycling the application pool during nomal use.

Keywords

application pool, http timeout

Symptom

  • During migration of Customer Master Data from SAP ECC to SAP S/4HANA using the Migrate Your Data application or via migration via the Migration Cockpit, the Customer Master creation date is not retained.
  • In SAP ECC, each Customer Master record has its own original creation date.
  • After migration, in SAP S/4HANA, the creation date for all Customer Masters is overwritten with the current migration date.

Environment

  • SAP S/4HANA
  • SAP S/4HANA Cloud Private Edition

Reproducing the Issue

  1. Perform migration of Customer Master Data from SAP ECC to SAP S/4HANA using the Migrate Your Data application or via the Migration Cockpit .
  2. Verify the creation date of Customer Master records in SAP S/4HANA.
  3. Observe that the creation date for all Customer Masters is updated to the current migration date instead of retaining the original creation date from SAP ECC.

Cause

  • Standard design of SAP S/4HANA Migration Cockpit treats migrated records as "new" entries in the target system, aligning with standard record creation logic.
  • The Business Partner creation process in SAP S/4HANA uses APIs that automatically assign the current system date and time as the creation timestamp for new records.

Resolution

It would be considered SAP standard design, that migrated records are treated as "new" entries in the target system, aligning with standard record creation logic.
This occurs due to the standard Business Partner creation process, where new Business Partner records are generated using APIs that automatically assign the current system date and time as the creation timestamp. The Migrate Your Data application within the SAP S/4HANA Migration Cockpit updates Customer Master creation dates to reflect the migration date.

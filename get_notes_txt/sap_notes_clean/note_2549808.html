Symptom

This Note provides details of the Flexible account Grouping and Account Group Change functionalities in SAP S/4HANA.

Solution

Definition

The account group is a classifying feature within customer and supplier master records. When you create a master data record, an account group is assigned to the Customer or Supplier record as per the Customizing (TBD001/TBC001). You can also use the following features to handle account group assignments for master data records: 

  • Flexible Account Group - A feature that enables you to choose a different account group during master data record creation.
  • Account Group Change - A feature that enables you to change the account group after the initial creation of master data based on user authoriazation.

Use

You can choose account groups via Customer Master and Supplier Master Fiori apps and transaction BP.

  • When you choose an account group during the creation of master data record, it determines:

    • which number range the customer/supplier account number should be.

    • whether the number is assigned by the user or by system.

    • which specifications are necessary or possible in the master record.

  • When you enable Flexible Account Grouping, and choose a different account group while creation of a master data record:
    • The number range assignments get overwritten by the new account group.

  • When the Same Number option is enabled along with Flexible Account Grouping: 
    • Customer/Supplier number wont be taken directly from the BP number, it must be explicitly provided by the user (in case of external numbering). This is because account group can be changed by using the Flexible Account Grouping functionality.

  • When you choose a different account group after the creation of a master data record (account group change):
    • The system verifies whether account group changes are possible with the current user authorization and proceeds.

Structure

You can enable the Flexible Account Grouping feature by selecting the relevant check box using the following Customizing activity:

  • For Customer: Cross Application Components > Master Data Synchronization > Customer/Vendor Itegration > Business Partner Settings > Settings for Customer Integration > Field Assignment for Customer Integration > Assign Keys > Define Number Assignments for Direction BP to Customer

  • For Supplier: Cross Application Components > Master Data Synchronization > Customer/Vendor Itegration > Business Partner Settings > Settings for Vendor Integration > Field Assignment for Vendor Integration > Assign Keys > Define Number Assignment for Direction BP to Vendor.

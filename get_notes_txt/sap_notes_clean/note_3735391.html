Symptom

  • AIF alert is not sent to the users configured in /AIF/CUST -> System Configuration -> Recipients > Assign recipients
  • Or it is not sent to all the users configured in above settings

Environment

  • SAP S/4HANA
  • SAP Application Server for SAP S/4 HANA
  • SAP Application Interface Framework

Reproducing the Issue

  1. An error is triggered in AIF message, the error can be observed in /AIF/ERR
  2. Checking the alert according to note 3212623, no alert ID can be found or alert can be found in t-code ALRTDISP, however alert is not sent to all users under the defined recipient

Cause

By default, only when an user confirmed the existing alert, new alert will be triggered to the user. 

Resolution

If the user expects all alert should be sent, the user can set "Send Always" using Fiori APP "Alert Settings", see here.

The system admin can also add the user as "Send Always" using view /AIF/V_ALRT_UN.

  1. In SM30, open view /AIF/V_ALRT_UN.
  2. Add the relevant users for the affected recipient and set the “Send Always” indicator to ensure notifications are generated for all errors.

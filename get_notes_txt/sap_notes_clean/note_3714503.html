Symptom

  • An application for product configuration is launched from the Fiori launchpad 
  • When the configuration component (for example, Configure Item) is triggered, the page remains empty or shows only a gray placeholder.
  • In the browser Developer Tools (F12, Network tab):
    • No $metadata query is observed for the configuration data, or
    • The returned metadata.xml lacks configuration entities (for example, ConfigurationContext, ConfigurationInstance).
  • Services commonly involved:
    • LO_VCHCLF_INTEGRATION_LEGACY_SRV (used by legacy embedders such as VA02/VA03, MM02/MM03)
    • LO_VCHCLF (rarely used directly)
    • LO_VCHCLF_SIMULATION_SRV (used by Simulate Configuration Models app)

Environment

Advanced Variant Configuration (AVC)

SAP S/4HANA Cloud Public Edition 

SAP S/4HANA Cloud Private Edition 

Cause

  • Cache inconsistency (typically server-side) following role changes, app activation, or system upgrades. Automatic client cache invalidation may not be triggered or may run infrequently, causing the Fiori frontend to use stale metadata from cache.

Resolution

  • In transaction SA38, execute report /UI2/INVALIDATE_CLIENT_CACHES to invalidate client caches.
    • Public cloud: must be executed by SAP Support using an EXTENDED_SUPPORT user.
    • Private cloud: can be executed by customer administrators on the frontend server.

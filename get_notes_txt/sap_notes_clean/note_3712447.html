Symptom

There are no values in the field MANDT (client) using transaction SE16N, for example after migration or copy of data.

Environment

  • Controlling CO
  • SAP ERP
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA
  • SAP S/4HANA Finance

Cause

  • The issue is not related to the data itself but rather to the behavior of SE16N.
  • SE16N does not display the client field as it works with the current client by default.
  • The client field is correctly populated in the database.

Resolution

  1. Confirm that the client field (MANDT) is correctly populated in the database using SQL statements in transaction DBACOCKPIT (folder "Diagnostics" - "SQL editor").
  2. Use SE16 or SQL queries for client-independent views of the data instead of SE16N, as SE16N does not expose the client field.

Symptom

Symptom

On opening Revenue Schedule app, you observe revenue contract column doesn't have hyperlink enabled.

Other Terms

Revenue Schedule app, Fiori, RAR, Revenue Accounting Contract, Revenue Contract, Hyperlink, F3882

Reason and Prerequisites

Coding was missing for target navigation.

Solution

Implement the corrections from note to resolve the issue.

Other Terms

BILaunchpad , Open ID , Custom.jsp , 404 Error , BOETrustValidateFilter

Reason and Prerequisites

Environment

  • SAP BusinessObjects Business Intelligence (BI) Platform 4.x / 2025
  • All supported operating platforms
  • OpenID


Steps to Reproduce:

 Case 1

  1. Stop tomcat.
  2. Navigate to ..\tomcat\webapps\BOE\WEB-INF\internal, then backup and edit global.properties and FioriBI.properties(in 4.2, BILaunchpad.properties).
  3. In global.properties, change the value of precompiled.jsp.files.use to false.
  4. In FioriBI.properties, change the value of redirection.iframe.1.redirectto.url to /custom.jsp.
  5. Navigate to ..\tomcat\webapps\BOE\WEB-INF\eclipse\plugins\webpath.FioriBI\web(in 4.2, webpath.InfoView), customize the custom.jsp file according to your requirements. 
  6. Clear tomcat cache(Only BOE folder is enough) as per 1879431

Case 2

  1. Logon to CMC => Applications => AUth Server
  2. Enable an Auth server for OpenID
  3. Navigate to Home => Authentication => Enterprise =>enable openid and select Auth server
  4. Edit FIoriBI.properties file for call back and restart tomcat 
  5. launch BI Launchpad , notice it redirects to IDP logon page 
  6. Pass the credentials and logon 
  7. Notice it spins and asks for IDP logon again (again and again in a loop)

Solution

This is only a workaround for the subject issues and an upgrade should be done to the fixed releases from the SAP notes as soon as is feasible

  1. Stop Apache Tomcat for BI from Central Configuration Manager (CCM) or Windows Services.
  2. Navigate to:
    <BOE Installation path>\SAP BusinessObjects\tomcat\webapps\BOE\WEB-INF\eclipse\plugins\webpath.FioriBI\web\WEB-INF
  3. Take a backup of the web.xml file (copy and rename it as web_backup.xml).
  4. Open web.xml in a text editor.
  5. Search for the filter named BOETrustValidateFilter.
  6. Inside the filter section, add the following entries after excludeUrl4:

    <init-param>
    <param-name>excludeUrl5</param-name>
    <param-value>/logon/start.do</param-value>
    </init-param>
    <init-param>
    <param-name>excludeUrl6</param-name>
    <param-value>custom.jsp</param-value>
    </init-param>

    NOTE : Ensure the param-name numbers are incremental and not repetitive. For example, if the existing excludeUrl4 is present, the new entry should be numbered excludeUrl5 and so on.

  7. Save the changes to web.xml.

  8. Navigate to <BOE Installation path>\SAP BusinessObjects\tomcat

  9. Rename the work folder to work_backup to clear the Tomcat cache.

  10. Start Apache Tomcat for BI from CCM or Windows Services.

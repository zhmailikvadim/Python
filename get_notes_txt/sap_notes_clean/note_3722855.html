Symptom

You have upgraded the system to S/4 HANA 2023 and you realize that when you create the change request and enter the change request description, the value will be wiped off with no further action. When you enter the description the second time, then the value will not be wiped off.

Environment

  • Master Data Governance on S/4 HANA on-premise 2023

Resolution

Please apply the following note:

  • 3543008 - Dump occurs or value vanishes when entering a value as first action in application running in FIORI Launchpad

Keywords

Change Request Description, Wiped off, Twice

Symptom

In the ContractRequestServices document, the <ProcessingTypeCode> in the outbound payload is not the same value I chose for the Document Type in SAP Ariba Sourcing.

Environment

Managed Gateway for Spend&Network

SAP Ariba Sourcing

Cause

The value MK is being mapped to <ProcessingTypeCode> in the outbound because of a configuration in your Cross References.

There are two ways that the mapping can populate this field:

  1. First the mapping will try to take the value from the Lookup Table taking into consideration the value inside the field <FollowUpDocument category="XXX">, if it is "value" or "quantity", and it will take the value from the respective value in the Lookup Table. If there is no configuration for ContractRequestServices in the Lookup Table it won't be able to get a value.
  2. If the first option fails, then it will take into consideration the field <ContractRequestHeader type="XXX"> and the Parameter in the Cross References for ContractRequestServices. If type is "value", the mapping takes the content from Cross References Parameter ValueContract; if type is "quantity", it takes the content from QuantityContract.

If the value is incorrect, you have a missing or wrong configuration in the Cross References Parameter or Lookup Table.

Resolution

Correct the Cross References by maintaining the Lookup Table according to the Configuration Guide - Setting Up Fields in the Lookup Table and/or the Parameters according to the Configuration Guide - Setting Up Cross References for Contracts.

Symptom

  • You are attempting to push changes from SAP Business Application Studio (BAS) to the remote Git Repository. 
  • However due to this security restrictions that are in place, your network team cannot add the BAS public IPs to the GitHub (for example) IP allow list. 
  • You are seeking other alternative options.

Environment

  • SAP Business Technology Platform (SAP BTP. URL: https://cockpit.btp.cloud.sap) 
  • SAP Business Application Studio (BAS)
  • Cloud Foundry (CF)
  • Git

Reproducing the Issue

Git Source Control is setup.

Resolution

  • SAP's recommendation is to add the BAS IPs to the allow list for seamless communication.
    For example, for GitHub, the steps are listed here:     Adding an allowed IP address.
    The required IP's can be added under: Organizations -> Authentication Security -> Enable IP allow list. 
    NOTE: The steps to add the IP's to the allow list might vary from one flavour of Git to another. Do consult the Git provider documentation for more details.

  • Now if the BAS public IP cannot be added to the GitHub IP allow list, BAS will not be able to access the repository directly. This is because GitHub blocks the connection at the network level, and this restriction cannot be overridden from the BAS side.
    Some alternatives to consider in this situation:

    -> Use your corporate laptop or a VPN (where your organization controls the IP) to clone the repository and push changes.
    -> Fork or mirror the repository into an environment where IP restrictions are not enforced, then work on that version from BAS.
    -> If your Network team supports a proxy or VPN solution that routes BAS traffic through an internal IP, they can configure BAS to connect to GitHub via that allowed network path.
    -> If none of these options align with your security policies, the only remaining approach is to use a local IDE where your IP is already permitted.
    Note that there is no official support for any of the options mentioned above and we recommend allow listing the BAS IPs.

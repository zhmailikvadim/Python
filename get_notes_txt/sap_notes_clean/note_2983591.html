Symptom

While adjusting SPAU or SPDD, objects that no longer exist are displayed.

When try to adjust them it is not possible since the object is already deleted.

Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.

Environment

SAP NetWeaver all versions.

Cause

The tab 'Deleted' in SPAU/SPDD is showing deleted objects which have been modified by the customer. Usually customers can delete the modification logs, especially for deleted objects to confirm that they have been taken care of. 

Resolution

It is possible to delete the old modification logs by transaction SE95_UTIL.

  1. Select "Delete modification logs" and execute (F8).

It is possible to restrict the search in the selection conditions or execute without using any condition.

The "Include in Transport Request" option should always be selected. Only if the objects are exported in a transport request, can the information that the modification logs were deleted be transferred into another system.

But if deleted objects are proxy ones (SPRX), it should not be flagged in transport request.

Look for the affected objects.

2. Select the entry and press button "Delete modification log"(Trash icon).

There will be a pop up window as below.

3. Click Yes.

4. Create a new request or fill in request number to delete modification logs.

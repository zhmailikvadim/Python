Symptom

  • The ICMS amount is not transferred correctly to the CIAP-related calculated fields.
  • In CIAP scenarios, the ICMS value is shown as zero although the source Nota Fiscal contains an ICMS amount.
  • In the calculation flow, the tax amount is lost before reaching the node that fills VL_ICMS.
  • The issue can be observed in adjustment NF-e scenarios used for ICMS credit, especially for documents associated with CFOP 1604.

Environment

  • CIAP Brazil
  • SAP ERP

Reproducing the Issue

  1. Create an NF-e for CIAP credit as an adjustment (finNFe = "3").
  2. Classify the document with a CIAP CFOP (for example, CFOP 1604) and set COD_SIT = 00 (normal operation).
  3. Run tax calculation and generate the EFD ICMS/IPI files.
  4. Check TDF views NF_IMPOSTO_CALCULO (field VL_ICMS) and NF_DOCUMENTO_ITEM_IMPOSTO: the ICMS amount is zero.
  5. Verify EFD ICMS registers C100/C190: the ICMS credit amount is not populated.

Cause

The CIAP credit NF-e was created with COD_SIT = 00 (normal operation). For ICMS credit on CIAP, a special situation is expected (COD_SIT = 06 or 08). With COD_SIT = 00 in this scenario, the system does not carry the ICMS amount to VL_ICMS.

Resolution

  1. Review the document classification for CIAP credit NF-e.
  2. Set COD_SIT to the appropriate special situation for CIAP credit (typically 06 or 08, according to legislation).
  3. Recalculate taxes and regenerate the EFD ICMS/IPI files.
  4. Validate that VL_ICMS in NF_IMPOSTO_CALCULO and the corresponding values in NF_DOCUMENTO_ITEM_IMPOSTO, C100, and C190 now reflect the ICMS credit.

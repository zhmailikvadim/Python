Symptom

  • After adding a new public holiday to the holiday calendar, existing absences overlapping that date are not auto-recalculated.
  • Manually running absence recalculation fails.
  • Error message: “Your requested time off must include at least one workday. Please change the dates you’ve entered accordingly.”

Environment

  • Product: SAP SuccessFactors Employee Central
  • Time Off

Reproducing the Issue

  1. User requests time off for a date range.
  2. A public holiday is added to the holiday calendar for a date within that requested range.
  3. Auto-recalculation for holiday calendar changes runs (or a manual recalculation is performed).
  4. The recalculation fails with the error “Your requested time off must include at least one workday.” under Time Alerts

Cause

The Time Type configuration “Requesting on Non-Working Days Allowed” is set to No; after the holiday update the absence falls entirely on a non-working day, causing recalculation to fail with the “must include at least one workday” error.

Resolution

  1. Navigate to Manage Data > Time Type for the affected time type.
  2. Set “Requesting on Non-Working Days Allowed” to Yes.
  3. Go to Administer Time > Time Alerts > Actions.
  4. Select Recalculate Absences to re-run the recalculation.
  5. After a successful recalculation, optionally revert “Requesting on Non-Working Days Allowed” back to No to align with the original business rule.
  6. Optional (to avoid manual effort in future): Ensure “Automatic Recalculation after Holiday Calendar Changes” is enabled, or use the Time Management Recalculation Event (TMRE) as described in the referenced note.

Symptom

  • An activation issue occurred in the QAS system after transporting changes from the DEV system.
  • Column "WW301_PA" was successfully implemented and functioning in DEV but failed to import into QAS, resulting in its absence in the QAS environment.
  • CO-PA functions are operating normally, but the column "WW301_PA" cannot be activated following the TR import.
  • Error messages encountered during the activation attempt include:
  • DDL Source "FGL_BCF_BS" could not be activated.
  • DDL Source "FGL_BCF_PL" could not be activated.

Environment

  • SAP S/4HANA Finance
  • SAP S/4HANA

Reproducing the Issue

  1. Implement Note 3687749 in the DEV system.
  2. Transport changes to the QAS system.
  3. Attempt to activate column "WW301_PA" in the QAS system after TR import.
  4. Observe activation failure with error messages:
  • DDL Source "FGL_BCF_BS" could not be activated.
  • DDL Source "FGL_BCF_PL" could not be activated.

Cause

No explicit cause mentioned in the case information.

Resolution

  1. Run report FGL_MIG_REGENERATE_CDS_BCF in all target systems where activation errors related to views FGL_BCF* occurred during import.
  2. After regeneration via FGL_MIG_REGENERATE_CDS_BCF, triggering the import again is not necessary as the issue will be resolved.
  3. If it is necessary for the import to run through without error, execute FGL_MIG_REGENERATE_CDS_BCF with the option 'Reset to Delivery' in the development system before releasing the TR to quality.

Symptom

When implementing flexible sales document numbering using prefixes, there's the need to know the system's restrictions for this functionality.

Environment

SAP S/4HANA Cloud Public Edition

Cause

Resolution

Any prefixes defined must conform to the following format:

  • Alphanumeric characters only (A-Z, 0-9, no special characters);
  • Maximum length of five characters;
  • The last character must be a letter (e.g.: 1A, 1234B);
  • Prefixes P, S, PBD and TMP are reserved for system use and cannot be assigned.

Note: The technical maximum length of a sales document number is 10 characters. When a prefix is used, this reduces the length of the possible number range by the length of the prefix. For example, a standard sales document number could be 1234567890, if applying the number range prefix ABC, the format would change to something resembling ABC1234567. This leaves only 7 characters for the purely numeric part of the document number, which is dictated solely by the number range interval.

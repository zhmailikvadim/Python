Symptom

This KBA clarifies the supported directory structure and file handling scope within the standard GoAnywhere (SFTP Dropbox) setup. It addresses customer inquiries regarding:

  • Creation of specific folders and subfolders within the GoAnywhere SFTP Dropbox.
  • Configuration or enablement of automated file transfers from the SAP-managed SFTP dropbox to external/third-party servers.
  • Requests for SAP to configure or manage automatic file movement between customer-created directories inside the GoAnywhere SFTP Dropbox. 

Environment

  • SAP Sales Performance Management
  • Incentive Management on Hyperscalers

Cause

Folder creation in the root directory is restricted by SAP GoAnywhere Architecture. 

Resolution

  • The GoAnywhere setup supports file movement only from the Inbound directory for processing.
  • As per SAP GoAnywhere standard architecture and policy, new folders cannot be created under the root, Inbound, Archive, badfiles or any other standard system directories. If required, custom folders can only be created under the Outbound directory.
  • However, SAP is not responsible for picking up or moving files within custom folders. Any such automation or custom file movement logic would need to be implemented and managed by the customer.
  • Customer can be allowed to create any number of directories under outbound directory.
  • Customer will be having full rights on outbound directory.

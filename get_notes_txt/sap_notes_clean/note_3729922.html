Symptom

You want to generate Peru statutory reports and need to know how to fill the reference for those reports.

Environment

  • SAP S/4HANA
  • SAP S/4HANA Cloud Private Edition

Resolution

Under Peru statutory report, a new field for the reference has been delivered by SAP Note 3709778 - DRC Peru Statutory Reporting: Missing information in Document Reference Field, this new field has more size capacity than the previous XBLNR (16 char) which allows Peru statutory report to incorporate higher reference formats.

This new field is available for "PE_JOURNAL_LEDGER", "PE_INVENTORY_BALANCE_LEDGER", "PE_SALES_LEDGER", and "PE_PURCHASE_LEDGER", and it will be under transactions/apps as below:

  • Create Incoming Invoices (App ID: FB60)
  • Create Outgoing Invoices (App ID: FB70)
  • Create Supplier Invoice - Advanced (App ID: MIRO)
  • Manage Journal Entries (New Version) (App ID: F0717A)
  • Manage Supplier Invoices (App ID: F0859)
  • Post General Journal Entries (App ID: F0718)

  1. For types "50", "51", "52", "53" and "54, the correct format to be used is "TT-AAA-YYYY-CCCCCC", which will be described as below:

    TT - Doc Type
    AAA - Aduana code
    YYYY - Year
    CCCCCCC - Correlated number


  2. For other types, as per example "05", "10", "01", "04" and other, the format for the reference is slightly different "TT-SSSS-NNNNNNN" which will be described as below:

    TT - Doc Type
    SSSS - Series (up to 20 char)
    NNNNN - Doc Number (up to 20 char)


In any of the above formats the "-" is required, as it will be used by the logic as the reference on which the field starts/ends.

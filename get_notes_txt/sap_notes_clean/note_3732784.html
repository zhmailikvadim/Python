Symptom

  • When inserting an application as a link in a blog, reopening the link dialog shows “#” instead of the originally selected application.
  • On reopen, the link dialog defaults to “URL” instead of “Application”.
  • The created link still works when clicked, but the originally linked application is not visible in the dialog for review or re-selection.

Environment

SAP SuccessFactors Work Zone

Reproducing the Issue

  1. Create a new blog and open the Insert Link dialog, then choose “Application”.
  2. Select and insert an application; the link is created and works.
  3. Reopen the link dialog for the same link; it opens with the “URL” setting.
  4. Switch to “Application”; the field is empty and the originally linked application is not shown (only “#” appears).

Cause

The blog editor only reads the link href when re-editing application-type links that are stored with “#” and ignores the embedded application parameters, so it cannot resolve and display the originally selected application.

Resolution

  • The fix for this issue is planned to be deployed with the patch 2603B.
  • For patch timelines, please review SAP SuccessFactors Patches Knowledge Base.

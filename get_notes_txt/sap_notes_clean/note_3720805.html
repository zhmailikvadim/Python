Symptom

  • An error SG813 occurs during the final execution step of the Standard Batch/Direct Input process via the LSMW transaction.
  • The same error is observed in previously created direct input runs.

Environment

  • SAP ERP Logistics (LO)  
  • SAP S/4 HANA on premise
  • SAP S/4 HANA Cloud Private Edition

Reproducing the Issue

  1. Access transaction LSMW.
  2. In the Specify file path step.
  3. Execute the process.
  4. In the last step, Start Direct Input Program, the program is launched with the physical file name ***lsmw.conv, resulting in the error.

Cause

Resolution

Under transaction FILE can you please compare the below configuration is maintained:

Logical File Path Definition -> LO_MD_ROOT -> Assignment of Physical Paths to Logical Path

UNIX Unix compatible

  • Logical path    LO_MD_ROOT
  • Syntax group    UNIX       Unix compatible
  • Physical path   /tmp/<FILENAME>

WINDOWS NT Microsoft Windows NT

  • Logical path    LO_MD_ROOT
  • Syntax group    WINDOWS NT Microsoft Windows NT
  • Physical path   <P=DIR_TEMP>\<FILENAME>

Logical File Name Definition, Cross-Client

MATERIAL_MASTER_BTCI_LOG

  • Name                      MASTER_BTCI_LOG
  • Physical file          
  • Data format          ASC
  • Applicat.area       MM
  • Logical path          LO_MD_ROOT

MATERIAL_MASTER_BTCI_NEW   

  • Name                      MASTER_BTCI_NEW   
  • Physical file          
  • Data format          ASC
  • Applicat.area       MM
  • Logical path          LO_MD_ROOT

Save the settings and retest your scenario

For security reasons the file name has to be hard coded: Read note 1500050 and it's linked notes carefully to resolve your issue.

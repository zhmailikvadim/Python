Symptom

  • Replicate a table or CDS view in LTRC
  • CDS objects are stuck in schedule action: Create Table
  • Not possible to stop this action

Environment

SAP Landscape Transformation Replication Server

Cause

Incorrect object names are being used. (e.g. it should be V_COSP_VIEW instead of COSP, or V_COSP_WDV_10, or V_COSP_WDV_11).

Resolution

  1. Go to Data Provisioning → Analyze Actions for the wrong object, choose ‘Remove the View from Participating Objects’, and also select ‘Apply to Dependent Objects’. 
  2. Start the Data Provisioning process again with the correct CDS view name
  3. Apply SAP Note 3187786 – SLT (DMIS 2018) Central System: Data Provisioning UI allows unsupported actions for CDS views if it's applicable in the SLT system

Keywords

SLT, CDS, schedule

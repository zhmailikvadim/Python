Symptom

While configuring Account Validation under "Maintain Global Provisioning Configuration", drop down values Warning & Error are grayed out and cannot be selected.

Environment

GRC 10.X

Reproducing the Issue

Goto SPRO ==> Governance Risk and Compliance ==> Access Control ==> User Provisioning ==> Maintain Provisioning Setting ==> Maintain Global Provisioning Configuration

Here, try to check the checkbox 'Account Validation Check' and select the value either Error or Warning. System allows clicking on the checkbox. However system does not allow selecting the value Warning or Error.

Resolution

To enable the values of Account Validation (Warning/Error), we need to save the Global Provisioning Configuration setting with checkbox 'Account Validation Check' first. The moment you select the checkbox 'Account Validation Check' and save the setting, system allows choosing the required value of Account Validation. Make sure you save the Global Provisioning Configuration once again after selecting the value of the Account Validation.

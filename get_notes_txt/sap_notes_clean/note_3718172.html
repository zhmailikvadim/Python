Symptom

In the infotype SI Additional Insurance (0079), the fields Contribution Scale and Contribution can both remain empty if the field Basic Pay Scale SI is filled and the employee is not liable to insurance contributions in HI and CI (that is, HI key 0 and CI key 0).


However, if there are several records of infotype 0013 in the validity period of infotype 0079, the system does not recognize an infotype 0013 record with HI/CI key 0.

In this case, the system issues the error message

  • 5D521 - Enter either a contribution scale or a contribution

even though the Basic Rate SI field is filled and the employee has HI/CI key 0. It is not possible to save the infotype 0079 record with empty fields Contribution Scale and Contribution.


Solution

Implement the program corrections.

The corrections are delivered by Support Package. An advance correction is possible (see correction instructions).

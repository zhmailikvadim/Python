Symptom

In FIORI App the affected user is not able to job results for other users.

Environment

  • SAP NetWeaver
  • SAP NetWeaver Application Server for SAP S/4HANA
  • ABAP PLATFORM - Application Server ABAP

Reproducing the Issue

  1. Open the FIORI app.
  2. Attempt to display the job results (job log/spool) for jobs started by another user; the results are not shown.

Cause

On releases prior to SAP S/4HANA 2022 (SAP_BASIS 7.57), the new authorization concept for application jobs is not available; the legacy concept does not support per-user authorization to view other users’ job results and, by design, prevents access to other users’ job logs/spool.

Resolution

The new authorization concept is only supported from S/4HANA 2022 (SAP_BASIS 7.57) on. See SAP Note 3190922.
The 'old' authorization concept does not offer to differentiate for actions on other users' jobs, i.e.
There is no setting possible like:

  • User A can see the result list of the job of user C
  • User B cannot see the result list of the job of user C 

See Also

SAP Note 3190922 - New authorization concept for application jobs

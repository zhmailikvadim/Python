Symptom

Impossible to connect to any target system using an insecure connection, the continue button is grayed out and the following message appears.

"Your system administrator has disabled the possibility of using insecure connections. Therefore you can’t continue with this connection."

Environment

  • SAP Analysis for Microsoft Office
  • SAP Analysis for Microsoft Office, edition for SAP Analytics Cloud

Reproducing the Issue

Try to connect Analysis for Office 2.8 SP11 or above to any target system and get the "Insecure Connection" popup before any connection takes place.

Cause

After AO 2.8 SP11, when any other connection status is detected, then AO behaves according to the value of AO parameter AllowInsecureConnections. When this parameter is set as 'No' or 'NoInsecure'  

Resolution

SAP recommends using secure connections to connect AO to the target systems. An insecure connection could be:

  • a connection to a BW system without SNC (Secure Network Communication) or below SNC quality 3
  • a connection without HTTPS to a BI platform, SAP HANA or SAP Analytics Cloud (SAC)

If you want to continue to user an insecure connection, please change the AO parameter  AllowInsecureConnections to other value than 'No' or 'NoInsecure'.

Symptom

  • Error message in the agent's DSOnPremiseAgent log files: "Failed to execute from jobrunID:##### error: Could not open command file"
  • Cloud-side trace log ends with: "JOB:Returning task to agent (/agent_request)"
  • The temporary file created before upload to a Azure Data Lake file location has not been deleted.

Environment

  • SAP Cloud Integration for Data Services
  • Windows operating system

Cause

An antivirus application was interfering with the agent's operation by blocking the deletion of temporary files created during task execution. This resulted in the agent being unable to proceed with the task, causing it to get stuck.

Resolution

  1. Identify the antivirus or security software causing the issue.
    • Set up a Windows Process Monitor capture, filtered to the agent's file paths.
    • When the error occurs, check the previously captured actions from non-agent related processes that interact with the agent's files.
  2. Configure the antivirus or security software to exclude the agent's directories and processes from scanning or tampering.
    1. Exclude the following processes:
      • al_jobserver.exe
      • al_engine.exe
    2. Exclude the following paths, relative to your agent installation folder:
      • ...\log\JobServer_DSoD_Agent\dsod_agent_repo__prod\
      • ...\log\JobServer_DSoD_Agent\dsod_agent_repo__sandbox\
  3. Verify that the agent service user has Full Control authorization on the following folders, including all subfolders:
    • C:\ProgramData\SAP\DataServicesAgent
    • C:\Program Files\SAP\DataServicesAgent
  4. Restart the agent service in Windows after applying the changes.
  5. Monitor the agent's operation to ensure the issue is resolved.

Symptom

  • In SAP Commerce Cloud 2211, with PBKDF2 encoded passwords and legacy.password.encoding.enabled=true, enabling password.encoding.auto.update.enabled=true correctly re-encodes customer passwords to Argon2 on the next successful login.
  • However, for ASM (Assisted Service Module) logins, passwords remain PBKDF2 and are not re-encoded to Argon2 after login.

Environment

SAP Commerce Cloud

Reproducing the Issue

  1. Use an environment where user (customer and ASM agent) passwords were previously hashed with PBKDF2.
  2. Set the following properties:
    • legacy.password.encoding.enabled=true
    • password.encoding.auto.update.enabled=true
  3. Log in as a customer and observe that the password is re-encoded to Argon2.
  4. Log in as an ASM agent and observe that the password remains PBKDF2.

Cause

  • Historically, ASM authentication could follow a different code path that did not use the core DefaultUserPasswordCheckingStrategy, so the auto-update check controlled by password.encoding.auto.update.enabled was not executed for ASM logins.
  • This behavior was corrected in SAP Commerce Cloud 2211.41. From 2211.41 onward, ASM authentication is wired to use the core authentication flow (for example, via authenticationProvider based on de.hybris.platform.spring.security.CoreAuthenticationProvider), ensuring DefaultUserPasswordCheckingStrategy is invoked and password auto-rehash is applied.
  • If the issue persists on or after 2211.41, it is typically caused by customizations in the ASM area (for example, in assistedservicefacades.impl.DefaultAssistedServiceFacade) that bypass the core authentication path and therefore never trigger authenticationProvider.authenticate, which is required for DefaultUserPasswordCheckingStrategy and the password.encoding.auto.update.enabled check to run.

Resolution

  • Product change: The issue was fixed in SAP Commerce Cloud 2211.41. From 2211.41 onward, OOTB systems re-encode both customer and ASM agent passwords from PBKDF2 to Argon2 on the next successful login when password.encoding.auto.update.enabled=true.
  • If running a version earlier than 2211.41, upgrade to at least 2211.41 to receive the fix.
  • If running 2211.41 or later and the issue persists:
    • Review customizations in the ASM area, especially around assistedservicefacades.impl.DefaultAssistedServiceFacade (and related classes or overrides).
    • Ensure the ASM login flow reaches authenticationProvider.authenticate so that DefaultUserPasswordCheckingStrategy.checkPassword() is executed and password.encoding.auto.update.enabled is evaluated. If the flow bypasses authentication via the provider (for example, token-based or custom direct session setup), password auto-rehash will not be triggered regardless of the property value.
    • Align custom flows with the OOTB behavior so that the core authentication provider is used for ASM logins.

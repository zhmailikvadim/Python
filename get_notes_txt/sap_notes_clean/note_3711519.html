Symptom

When a promotion is deleted in the new revision, the system currently marks it as a soft delete in Fiori. However, the expected behavior is that the promotion should be physically deleted from the system in the new revision.

This note ensures that the promotion is physically deleted from the new revision.

Other Terms

/IRM/IPPQM, promotion_delete_new, promotion_delete_check_new, Promotion Physical delete, /IRM/IPPQRVM

Reason and Prerequisites

This is caused by program error

Please implement SAP Note #3313208 as prerequisite

Solution

Please implement the correction instructions attached. These corrections are only valid if you have applied the following support packs.

Release V4 - 2021 and 2022

Use transaction SNOTE to implement the correction instructions specified below and activate the affected program object.

Symptom

  • Error message KI100 is triggered during Sales Order editing via VA02, despite being set to "switched off" in OBA5.
  • The issue persists even after applying SAP Note 352261.

Environment

  • Controlling (CO)
  • SAP R/3
  • SAP R/3 Enterprise 
  • SAP ERP Central Component
  • SAP ERP
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA

Reproducing the Issue

  1. Run transaction VA02.
  2. Set Order and Enter.
  3. Make changes and hit Enter
  4. Error message KI100 is triggered.

Cause

The error message KI100 is triggered due to the Sales Order being associated with a statistical account assignment object, which bypasses OBA5 settings.

Resolution

  1. Implement SAP Note 354290 to extend suppression of error messages to statistical objects.
  2. Verify that the error message KI100 is marked as "configurable" in table T100S via transaction SM30 or SM31.
  3. Ensure the entry includes Application Area "KI", Message Number "100", Allowed Types (e.g., SIWE for Success/Info/Warning/Error), Standard Type (e.g., E), and Switch Off field set to 'X'.
  4. Retry the steps in VA02 after implementing the note and confirm that the error message is no longer triggered.

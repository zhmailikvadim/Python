Symptom

Running statements using VBKD table. e.g. 
SELECT * FROM "VBKD" WHERE "MANDT" = ? AND "VBELN" = ? AND ( "BSTKD" > ? OR "BSTKD_E" > ? );

has performance degradation or fluctuating runtime.


You check a data collector for the affected statement hash and see similar result to this:

Sometimes a bad plan is used and so the execution time is higher.

Environment

SAP HANA.

Reproducing the Issue

Check the runtime of the statements.

Find the corresponding hash and run data collector against it.

Find the hash as per 3322936 - How to find the SQL statement hash for a SQL statement string .
Run data collector as per 3444934 - How to execute "HANA_SQL_StatementHash_DataCollector" for a specific SQL .

Cause

Selections from VBKD with VBELN and BSTKD conditions can suffer from wrong HEX engine decisions on SAP HANA >= 2.0 SPS 08. In order to make sure that a good execution plan with post filters on BSTKD and BSTKD_M is used, the following statement hint is delivered via SAP Note 2700051:

HEX_TABLE_SCAN_POST_FILTER("VBKD"."BSTKD","VBKD"."BSTKD_E") 

Implement SAP Note 2700051 in order to fix performance issues in this context.

Resolution

Implement the corresponding fix from SAP Note 2700051 - Delivery of SAP HANA Statement Hints.

See Also

2700051 - Delivery of SAP HANA Statement Hints.

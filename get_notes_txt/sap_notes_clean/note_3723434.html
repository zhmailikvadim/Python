Symptom

Issue 1: Transaction-Based Creation (FPP1). When creating a business partner via transaction FPP1 or BP in SAP S/4HANA (Utilities scenario), the GP role CRM000 is not automatically assigned to the newly created business partner.

Issue 2: API/Master Data Generator-Based Creation (BAPI_ISUPARTNER_CREATEFROMDATA) When creating a business partner using the BAPI_ISUPARTNER_CREATEFROMDATA function or the master data generator, the business partner role CRM000 is automatically created and assigned, even when this behavior is not desired or required.

The behavior is not aligned. If CE (Customer Engagement) is used or shall be used - the BP role CRM000 is needed for every affected/used contract partner.

Other Terms

FPP1 MKK BAPI_ISUPARTNER_CREATEFROMDATA FKK_FEATURE_ADMIN Master Data Generator MDT

Reason and Prerequisites

The inconsistent behavior of CRM000 role assignment across different business partner creation methods is caused by:

  • Hard-Coded Logic in BAPI - The static method is_crm_erp_coclient in BAPI_ISUPARTNER_CREATEFROMDATA is hard-coded to always return true, forcing automatic assignment of the CRM000 role regardless of the user's intent.
  • Master Data Generator Behavior - The master data generator in S/4HANA enforce the assignment of the role "CRM000" during GP creation, which operates differently from transaction-based creation methods
  • CE (Customer Engagement) Default Activation - Customer Engagement (CE) is automatically active in S/4HANA systems, causing different role assignment behavior between API-based and dialog-based creation methods
  • Inconsistent Configuration Across Channels - There is no unified, configurable approach to handle CRM000 role assignment across different creation channels (Transaction BP/FPP1, Master Data Generator, C4C, WebUI, APIs)

Solution

Implement the attached Correction Instruction or the mentioned Support Package.

This SAP Note provides a unified configuration option to control the automatic creation of the GP role CRM000 when business partners are created via any channel (Transaction FPP1/BP, Master Data Generator, BAPI_ISUPARTNER_CREATEFROMDATA, C4C, WebUI, or other APIs).

Option A: Enable Automatic CRM000 Role Creation (Default Behavior - recommended)
To ensure the CRM000 role is automatically created and assigned to all new business partners across all creation channels:

Navigate to transaction: FKK_FEATURE_ADMIN
Locate the feature ISU_BP_SUPPRESS_CRM000
Ensure the feature is deactivated/unchecked (default state)

Option B: Completely Suppress Automatic CRM000 Role Creation
To suppress automatic creation across all channels and manage CRM000 role assignment manually:

Navigate to transaction: FKK_FEATURE_ADMIN
Locate the feature ISU_BP_SUPPRESS_CRM000
Activate/check the feature to suppress CRM000 automatic creation
Save the settings
Business partners created via any channel (FPP1, BP, BAPI_ISUPARTNER_CREATEFROMDATA, Master Data Generator, etc.) will not have the CRM000 role automatically assigned
Manual role assignment can be performed using transaction BP or the BP Master Data UI if needed

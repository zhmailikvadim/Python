Symptom

When reading data for processes sometimes the same data is read multiple times. This is caused by duplicated keys in the "semantical keys". (Strictly speaking they are not duplicates, because they differ in some field, but that field is not relevant for reading the requested data. Therefore they return the same data, and are logically duplicates.)

Other Terms

Financial Products Subledger, FPSL, result data

Reason and Prerequisites

Performance trace shows long runtimes in /BA1/CL_AL_BR_ACCESS_RDL* classes.

Solution

Implement this note.

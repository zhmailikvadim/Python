Symptom

When saving a PM order, the creation of FSM Activities fails. The issue occurs when the PM order is created without a customer, and region determination is configured to be based on the geographical region (address region). As a result, the order processing stops and FSM Activities are not created.

Other Terms

FSM, E4C, Connector, PM Order, Region, /PACG/ECM_SC_GR, Address Region, Activity

Reason and Prerequisites

When region determination is configured to use the geographical region (address region), the system attempts to determine the region using customer data. If the PM order does not contain a customer number, the logic fails during region determination and interrupts further processing. To resolve this issue, an additional validation was implemented. If the order does not contain a customer, the system now creates a log entry and skips the region determination step instead of stopping the processing.

A new message was introduced: ‘Region determination requires customer but it is empty in the order &1.’ This message is written to the application log and processing continues.

Solution

Please, implement this correction instruction. Component (S4)PACG200 requires manual post-installation action.

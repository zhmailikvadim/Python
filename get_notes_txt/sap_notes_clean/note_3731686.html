Symptom

In payroll function SKJVS results from Annual tax clearing (report HSKCDPRZ) are taken for payroll processing. This payroll function SKJVS should properly process all Annual tax clearing (RZD) results.

Other Terms

Payroll function SKJVS, Annual tax clearing, RZD, child bonus.

Reason and Prerequisites

This SAP Note is relevant only for Slovakia (HRSK).

Solution

Payroll function SKJVS aligned to process Annual tax clearing results in payroll calculation as shown below.

In general, these combinations can occur:

A) RZD finishes with overpayment (RZD row 22 < 0) or RZD finishes with underpayment (row 22 > 0) but deduction from PY is activated (P0257-TTFB401 = 'X')

Then, these wage types are written into payroll:
/403 - value from row 22 of RZD
/424 - value from row 9 - row 10
/423 - value from row 7 - row 8
/426 - value from row 15

 

B) RZD finishes with underpayment (row 22 > 0) and deduction from PY is NOT activated (P0257-TTFB401 = '')

We calculate 'Positive RZD remainder' (tax bonus, employee premium, mortgage, unadjusted tax) as:

  • Positive RZD remainder = row 11 + (if positive: (row 7 - row 8)) + (if positive: row 15) + row 20

Then, if wage types /424, /423, /426 are POSITIVE, those are written into payroll in a same way as in case A). Also, wage type /413 is used instead of /403:
/413 - value from row 22 of RZD
/424 - value from row 9 - row 10
/423 - value from row 7 - row 8
/426 - value from row 15

If /424, /423, /426 is negative, we first clear/compare it against Positive RZD remainder (from above). If Positive RZD remainder is higher than absolute value of e.g. /424, we keep /424 unchanged (keep full negative value) and just lower Positive RZD remainder by absolute value of /424. If the Positive RZD remainder is lower than absolute value of e.g. /424, we reduce /424 up to the remainer (and write into /424.AMT) and full (original) negative value we store into /424.RTE. Then, we lower Positive RZD remainder by used value and perform the same process with /423 and /426.

 

Example:

RZD has following results (and PY deduction is not applied: P0257-TTFB401 = ''):

r.22(a) 1 218,23 (underpayment)
r.13 -1 430,44 (underpayment)
r.7      -22,00 (underpayment)
r.15     200,00 (overpayment)
r.20       34,21 (overpayment)

Positive RZD remainder = row 11 + (if positive: (row 7 - row 8)) + (if positive: row 15) + row 20 = 0 + 0 + 200 + 34,21 = 234,21

Result in payroll calculation would be:

 WT    Name     RTE     AMT
/413 Roč.zúčt.d.   1 218,23
/424 Daň.bon. -1 430,44   -234,21 
/423 ZP     -22,00        0,00
/426 DB na zapl.ú.      200,00

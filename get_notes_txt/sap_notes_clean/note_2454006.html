Symptom

An error message stating "Cannot find data list library info for -- >getUnderwritingMandatoryFlag" is encountered on trying to add a value to a column from a drop-down that is populated with data from a Data List object.

Environment

  • SAP Product Lifecycle Management for Insurance
  • SAP Product and Quotation Management for Insurance 3.0

Reproducing the Issue

Pre-requisites:

  • You have access to a new FS-PRO instance
  • You have imported the Sample Household Product (or similar product that utilizes the SAP Insurance UWA Product Template)

Steps:

  1. Log in to the FS-PRO instance
  2. Open the Sample Household Product
  3. Navigate to the Underwriting > SAP Underwriting > Underwriting Rules Component object
  4. Add a new row and click on the Mandatory column field to Select a Data List Value
  5. An error mesage is generated:

    Message

    Get data list value failed.
    Error Get data list value failed.

    Root Message:

    java.lang.RuntimeException: Cannot find data list library info for -- >getUnderwritingMandatoryFlag in product [Data List Base, version:1 ]

Cause

This issue is caused by not configuring the right Data List object for the FS-PRO system.

Resolution

DSICLAIMER: "Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental."

The issue is resolved by configuring the right Data List object for the FS_PRO system to use.

  1. Log in to the FS PRO Admin Console.
  2. Navigate to System > Edit Configurations > Configuration > PS > Env > Product Reference (PC Only) > DataListSourceLibrary field.
  3. By default, the DataListSourceLibrary field has the value “Data List Base” but the end user could put a different value reflecting a different Data List object that they want to use.

  4. Populate the Override column with the value “SAP Insurance UWA Data List” as this has the Data List API service “getUnderwritingMandatoryFlag” implemented.

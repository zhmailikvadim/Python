Symptom

When running the program /SAPSLL/CUHD_FS_ARCHIV_CHECK, some documents are identified not archivable due to the fact that the corresponding reference document has already been archived in feeder system.

Environment

  • Global Trade Services (GTS)
  • Global Trade Services / Logistics Services (SLL)
  • SAP Global Trade Services

Reproducing the Issue

  1. Run the program /SAPSLL/CUHD_FS_ARCHIV_CHECK in simulation mode.
  2. Enter document reference number and execute: The document is identified as archivable with pop-up "No data found; change the selection criteria".

Resolution

Consider implementing the BAdi /SAPSLL/BADI_CDOC_FS_ARC to influence the flag 'Document Can Be Archived' (FLDRA).

Keywords

archiving inconsistency, SAP Global Trade Services, /SAPSLL/CUHD_FS_ARCHIV_CHECK, document reference number, feeder system missing, BAdi /SAPSLL/BADI_CDOC_FS_ARC, FLDRA flag, customs document archiving.

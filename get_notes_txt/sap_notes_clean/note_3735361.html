Symptom

In the process "DTD Re-Check Process for Invoice Document" (DE_B_CP_INVOIC_DTD_RE_CHK), grid usage invoice complaint reasons are not removed in step 0170, even after a successful DTD re-check process.
This issue is relevant for the market role: SUP.

Other Terms

DTD Re-check, E_0406, Rejection Error, Complaint Reasons, /US4G/AP_PAYMENT 157, INVMON, Grid Usage Billing, INVOIC

Reason and Prerequisites

Program error.

Solution

Implement the corrections of this SAP Note or import the relevant support package.
After implementing this SAP Note, the system removes invoice complaint reasons when the invoice passes every check in the DTD re-check process.
You need to retrigger the DTD re-check process. After the process document is completed without DTD check error, the invoice can be processed.

You can verify the fix by checking that the invoice can be processed without error in transaction code INVMON.

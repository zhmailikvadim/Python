Symptom

  • Transfer payments received from lockbox in BAI2 format to FI-CA. 

  • Process customer payments received via the BAI2 lockbox format.

  • Import the file via lockbox transaction FLB2 and then execute transaction FPB7.

  • Notice that a payment advice and a payment lot is created.

Environment

FI-CA - Contract Accounts Receivable and Payable  

Reproducing the Issue

  1. Post the BAI2 file using transaction code FLB2 in FI.
  2. Execute transaction code FPB7 to clear the contract line items in FI-CA.
  3. See the payment lot item and it's advice.

Cause

Lockbox functionality available from S4Core104 and higher

Resolution

There is a FI part and a FICA part.

 

1. FI part -->  

Transaction FLB2 will read the BAI2 file, create an advice on the FI side and store EBS data in database FEBEP.

 

    1. Verify if the FEBEP table is correctly created with the advice number in the AVSID field.
    2. Check if the data uploaded from FI is correctly configured. For example, ensure that FLB2 uses invoice search type 2 (XBLNR) or 6 (OPBEL) - as type 1 (BELNR) will not work when FPB7 processes the information.

    c. Use Report RFAVIS10 to display advice

2. FICA part -->

Transaction FPB7 read the EBS data from database FEBEP and advice from FI and process them to FICA data (create a payment advice note and a payment lot)

    1. Automatically create advice on FICA side. Advice can be displayed by transaction FPAV.
    2. This advice YYY is the one used to append in payment lot item in transaction FP05.
    3. Lot xxx created with advice YYY.

Symptom

Error message: "Replication failed. Master data *GL accounts* are duplicates" when attempting to activate the global hierarchy.

Environment

 S/4HANA Cloud Public edition

Reproducing the Issue

  1. Access the Manage Global Hierarchies application.
  2. Attempt to activate the global hierarchy for balance sheet and income statement structure.
  3. Observe the error message: "Replication failed due to duplicate master data"

Cause

The same accounts have been assigned to two different nodes — example: one under Assets and one under Liabilities. This configuration is designed to automatically display the accounts under the appropriate section depending on whether the balance is in debit or credit. If these are not used, the system throws an error indicating a duplicate entry.

Resolution

  1. Ensure that each G/L account is uniquely assigned within the Financial Statement Version (FSV).
  2. Alternatively, use Reclassification Increments or Debit/Credit indicators to allow the same G/L account to be assigned to multiple nodes.
  3. Refer to the following apps for utilizing this functionality: F0708 or W0161.

See Also

  • Refer to SAP Note: 3228465 - Manage Global Hierarchies Cannot remove Reclassification Increment in Financial Statement Version.
  • Refer to SAP Note: 3241696 - Restrictions for FSV used as hierarchies.

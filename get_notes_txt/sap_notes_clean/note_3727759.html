Symptom

SAP Note 3640873 introduced new operation SIGAB, which sets hourly rates for absences, when employee has two types of sick leave in the same month, each with different hourly rate. It reads hourly rate /0I1 with the same absence identifier from the previous period. This approach doesn't work when employee is hired during long-lasting absence. Infotype 2001 field Certified start (PA2001-RMDDA) contains the first day of absence, paid by ZZZS. This date is usually before hire date and previous result doesn't exist. As a consequence, hourly rate is not set correctly regardless of its correct entry in the infotype 0015 wage type /0I1.

Other Terms

SIGAB, /0I1, RMDDA

Reason and Prerequisites

  • Employee is hired during long lasting absence, first day ZZZS is entered in RMDDA.
  • SAP Note 3640873 is implemented.

Solution

Payroll operation SIGAB has been enhanced. If no previous result exists, it searches for the hourly rate in internal table LRT, which should contain manually entered hourly rate in wage type /0I1.

Implementation

  • Generally, SAP recommends that you install a solution by applying a Support Package. However, if you need to install a solution earlier, use the tool SAP Note Assistant (SNOTE) to implement automatic correction instruction from this SAP Note. For more information about the SAP Note Assistant see SAP Support Portal under SAP Note Assistant.

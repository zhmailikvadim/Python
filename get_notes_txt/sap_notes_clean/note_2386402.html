Symptom

When sale order settlement is executed with background job for program RKO7VA88, it fails with error message BL252:

 "Message number 999999 reached. Log is full"

Environment

  • Financial Accounting (FI)
  • Controlling (CO)
  • SAP R/3
  • SAP R/3 Enterprise 4.7
  • SAP ERP Central Component
  • SAP ERP
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA

Reproducing the Issue

  1. Execute SE38 with Program RKO7VA88;
  2. Input "Controlling Area" and Continue;
  3. Input the relevant parameters and then choose menu: Program-> Execute in Background;
  4. In T-code: SM37, it is observed that the job exeucted in step 3 has status 'Canceled';
  5. Error message BL252 is found in the 'Job log' at the bottom.

Cause

This is a technical restriction to have not more than 999.999 messages filled into the system log.

Resolution

  1. T-code SLG2 should be used to delete old log messages. (See SAP Note 195157 Application log: Deletion of logs）
  2. For unnecessary customizable information messages, it can be suppressed in T-code OBA5 > Application area "KD".
  3. Set the flag ‘Check transaction data’ in the selection screen of settlement report RKO7VA88. This will extremely reduce the number of error message KD205. (See SAP Note 197561)

Symptom

You use the Security Audit Log with database recording and copy this system.
In general, the audit log data of the previous system should not exist in the target system. If these exist, you require an initial cleanup. However, there is no standard tool for this.

Other Terms

RSAU_CLEANUP, CL_RSAU_SLO

Reason and Prerequisites

Missing function

Solution

Implement the attached corrections using SNOTE or by importing the referenced support package.

Solution Explanation

You can use the report RSAU_CLEANUP to completely delete the transaction data of the audit log (tables RSAU_BUF_DATA, RSAU_LOG, RSAU_FILES, and RSAUFILES_STAT).
This should be done BEFORE you start productive use. We do not recommend that you use the tool later.

Proceed as follows...

  • Use client 000 for this adjustment
  • Unlock transaction RSAU_CLEANUP using transaction SM01_CUS (this may have to be done twice the first time).
  • Call transaction RSAU_CLEANUP, select the tables to be initialized, and execute the report.
  • The transaction locks itself again immediately after successful execution to prevent the accidental destruction of data.

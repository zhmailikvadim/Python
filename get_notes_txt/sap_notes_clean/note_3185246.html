Symptom

I want to configure auto-receipts to be created in Composing and it should be manually editable. 

Resolution

The parameter Application.Receiving.AllowManualSubmissionOfAutoGeneratedReceipts can be enabled to create auto-receiving receipts in Composing and allow them to be edited manually. Please contact support to enable this parameter.

Once this parameter has been enabled, when an order is closed and then reopened the new receipt with be created in Composing  and the user will be able to edit.

See Also

  1. This parameter will not affect receipts created prior to the change. 
  2. This parameter is effective only when the keep reopened orders in Receiving status after receipt submission (Application.Purchasing.PreventCloseOrderForReceipt) parameter is set to Yes, or the Application.Purchasing.AllowPurchaseOrderLineClose parameter is set to Yes.
  3. This Parameter is not currently compatible with Guided Buying. The newly created manual receipts will not be editable in Guided Buying. If you would like to see this feature added to Guided Buying, please submit an Improvement Request
  4. This parameter cannot be used to automatically receive all manual receipts. Auto receiving should be configured through the Receiving Type by commodity code or Receiving Type by Part number section in the Procure to Pay Manager section within Core administration interface.

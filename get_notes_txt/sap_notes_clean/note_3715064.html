Symptom

When a SAP user is assigned to a role defined in /PACG/ECM_USROL, the corresponding entry is automatically created in table /PACG/ECM_ALV_USERS with the "Loginable" flag enabled.

However, if the user is later unassigned from the role and then reassigned again, the "Loginable" flag is not reactivated. As a result, the user remains non-loginable in the Connector, even though the role assignment has been restored.

Other Terms

FSM, E4C, Connector, User, Role, /PACG/ECM_USROL, /PACG/ECM_ALV_USERS

Reason and Prerequisites

The issue occurred in method USER_ASSIGNMENT_CHANGE of class /PACG/ECM_CL_A_ENH_USER_TRANS. The existing logic correctly set the "Loginable" flag when a user was assigned to a role for the first time. However, it did not properly detect and handle reassignments of previously removed users.

Solution

Please, check for pre-installaton steps (component (S4)PACG200) and in the next step implement this correction instruction.

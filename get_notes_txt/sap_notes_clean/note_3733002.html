Symptom

  • The system creates provisional purchase settlements while the application data being correctly distributed.
  • In a back-to-back movement through a virtual plant that is not included in the contract, the settlement is created as provisional.

Environment

SAP Agricultural Contract Management

Reproducing the Issue

  1. Create a Contract with WB21 T-Code.
  2. Set a discharge plant.
  3. Perform a back-to-back movement through a virtual plant that is not included in the contract.
  4. Create the purchase settlement (/ACCGO/STL_WC - Agricultural Contract Management  -> Contract Settlement -> Settlement Workcenter ) and observe that it is created as provisional.

Cause

The application is marked as no physical receipt (no_ph_rec), which causes settlements to be created provisionally unless explicitly set to final.

Resolution

  1. Verify that the application is flagged as no physical receipt (no_ph_rec).
  2. If a final settlement is required, explicitly execute the process to create the settlement as final or set the settlement indicators accordingly before running settlement, uncheck Provisional Settlement and Subsequent Provisional as required by the process.
  3. Re-run the settlement and confirm that the document is created as final.

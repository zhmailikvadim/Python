Symptom

  • Creating a Snowflake Connection in CMC gets the below error
  • Information Steward job server is installed in Linux
  • A Snowflake datastore is created with same parameters in DS Designer, and the related job can be executed using a windows job servers.
  • In the additional infomration field, the authenticator is specified to be okta

Environment

  • SAP Information Steward 4.3 SP2
  • Linux

Resolution

To connect to a Snowflake database, native connection through the IDP(okta) authenticator is not supported yet.

Instead, please create the DSN in Linux $LINK_DIR/bin/DSConnectionManager.sh, then create an ODBC connection pointing to the Snowflake database

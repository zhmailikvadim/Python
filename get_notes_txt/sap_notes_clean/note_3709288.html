Symptom

  • Task fails in Sandbox.
  • Error message: "Cannot load table with the wrong number of columns. Table has <X> columns while the input schema has <XX> columns. Ensure that the input matches the table schema."

Environment

  • SAP Cloud Integration for data services

Cause

The issue arises due to discrepancies in table structures.

Resolution

  1. Re-import the table structure from datastore.
  2. Reset the cache of the affected task.
  3. If still same error, edit the affected task to delete a random Transform , then re-add it with the same logic to refresh the task metadata.
  4. Save the task and execute it again.

Keywords

task execution failure, mismatched columns, table structure discrepancy, reset cache, refresh metadata, task failure, cids, wrong number, columns

Symptom

When analyzing a SAP HANA system using a SQL trace no SQL statements appear in the trace results. Although the trace is activated successfully, the output remains empty or shows only partial technical information without any actual SQL commands.

Environment

SAP HANA 2.0

Cause

There are two possible reasons why no SQL statements are captured in the SQL trace:

  1. The statement never reached the database.
    The operation may have been processed on the application server or filtered out before being sent to SAP HANA.

  2. The statement did not pass through the session layer because it was executed internally.
    This applies especially to SQLScript procedures. Although a procedure contains multiple SQL statements, the statements inside the procedure body are executed internally by the SAP HANA SQLScript Engine. As a result, they are not recorded by the default SQL trace, which only captures statements executed through the SQL session layer.

Internal SQL Script execution can still be traced by enabling internal tracing, which records statements processed inside the SQL Script Engine.

Resolution

To capture SQL statements that are executed internally within SQLScript procedures, enable internal SQL tracing as described in SAP Note 2119087 – How-To: Configuring SAP HANA Traces.

  1. Open the indexserver.ini configuration.
  2. Navigate to:
    [sqltrace] → internal
  3. Set the parameter internal = true.

Setting this parameter to true allows the SQL trace to record SQL statements executed inside procedure bodies. If set to false, only the procedure calls themselves are traced, but not the internal SQL statements processed by the SQLScript Engine.

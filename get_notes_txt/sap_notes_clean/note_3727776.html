Symptom

  • Schema and table names are not visible or missing in the Data Foundation layer while attempting to set up the connection with Snowflake in Information Design Tool.
  • Test connection is successful in Information Design tool.

Environment

  • SAP BusinessObjects Business Intelligence platform 4.3 & 2025
  • Information Design tool
  • Snowflake

Reproducing the Issue

  1. Open the Information Design Tool on your system.  
  2. In local project, create a new relational connection to Snowflake, either using ODBC or JDBC drivers.  
  3. Create a Data Foundation layer based on this connection.  
  4. Go to the Data Foundation options and click on the dropdown menu for the table selection.  
  5. Observe that the schemas and tables are missing from the list.

Cause

The CLIENT_METADATA_REQUEST_USE_CONNECTION_CTX in Snowflake is a feature or parameter related to how metadata requests are handled, particularly in the context of connection context management. 

The setting CLIENT_METADATA_REQUEST_USE_CONNECTION_CTX is set true in the Snowflake database. This configuration restricts the access of the user to all tables, which makes schemas and tables are not visible during connection setup.

Resolution

To resolve the issue, set CLIENT_METADATA_REQUEST_USE_CONNECTION_CTX to False in the Snowflake database configuration. This allows the client to request metadata information, including schemas and tables, through the connection context.

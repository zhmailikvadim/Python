Symptom

When replicating quotes orders from SAP Commerce Cloud to SAP Cloud for Customer, "Configuration for virtual attribute ** in item QuoteEntry is invalid" error occurred for the corresponding outbound request.

Environment

SAP Commerce Cloud

Reproducing the Issue

  1. Attempt to send production orders from SAP Commerce Cloud to SAP C4C via CPI.
  2. Enable the virtual attribute (e.g.taxItem ) in the Integration Object (IO) within the SAP Commerce Cloud.
  3. Observe that orders fail to flow to SAP C4C, and payloads are not generated due to error "Configuration for virtual attribute taxItem in item QuoteEntry is invalid."

Cause

The virtual attribute script missing could lead to errors for virtual attribute configuration and prevent orders' outbound flow.

Resolution

Review if virtual attribute script is existing in SAP Commerce Cloud following below steps:

  1. If virtual attribute is created through impex, try to follow Creating Virtual Attributes in the Backoffice Integration UI Tool and create in Backoffice. When script is not existing, it's not able to select in the create wizard.
  2. Once found script is missing, insert the required script using impex
    INSERT_UPDATE Script; code[unique=true];content;active[default=true]
  3. Then test the order outbound flow again, it should work.

Symptom

  • Starting from SAP BusinessObjects 4.3, the option of use Embedded Database (Apache Derby) is deprecated.
    • See 2912759 - Derby DB is still used instead of Auditing for Monitoring in Fresh and Update builds of 4.3 
  • After switching Trending database to use Auditing database or other supported database, are the files under the following TrendingDB folder can be safely deleted?
    • Windows: <install folder>\SAP BusinessObjects Enterprise XI 4.0\Data\TrendingDB
    • Linux: <install folder>/sap_bobj/Data/TrendingDB

Environment

SAP BusinessObjects Business Intelligence Platform 4.3 / 2025

Cause

Resolution

The files under Trending database can be safely deleted after the following conditions are met. It is also recommended to save the files to somewhere else for backup purpose.

  1. The switch of trending database to Auditing or other database has been done successfully. For example, in the switched database, the monitoring application tables (MOT_MES_DETAILS, MOT_MES_METRICS, MOT_TREND_DATA, MOT_TREND_DETAILS) has been successfully created with data. Also, in CMC (Central Management Console) -> BI Admin Studio -> Click any watch and show historical view -> confirm history data can be checked without problem.
  2. There is no valuable data existing in the existed Apache Derby files, or the data migration has been already successfully finished from the existed Apache Derby to the switched database.

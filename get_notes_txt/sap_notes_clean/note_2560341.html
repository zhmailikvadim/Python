Symptom

Configured CM anonymous access following note 837898 because there are resources that need to be accessed anonymously. Then found attachment of task (created using "Create Task" button in UWL list) from AdHocSystem can be accessed anonymously when use direct link of the attachment, which is not expected.

Environment

  • EP Release Independent
  • SAP NetWeaver
  • SAP Composition Environment

Reproducing the Issue

  1. Follow note 837898 to configure CM anonymous access.
  2. Create task using "Create Task" button in UWL list and upload attachment to this task.
  3. Access attachment of the task via direct link like <protocol>://<FQDN>:<port>/irj/go/km/docs/workflow/40d1d9fd-80a6-3510-a3bd-c707a8c3aeb1/test.txt

Cause

  • CM Repository "workflow" has other value configured for "Security Manager" other than AclSecurityManager thus cause no way to adjust permission setting of "workflow" folder in "KM Content".
  • Permission setting of "workflow" folder or sub folder / file is not correctly configured.

Resolution

1. Make sure correct settings have been implemented following note 837898.

2. Access System Administration -> System Configuration -> Knowledge Management -> Content Management -> Repository Managers -> CM Repository, then click "workflow" and make sure "Security Manager:" has value AclSecurityManager configured. If it has other value configured, please change to AclSecurityManager and restart system to take change effect.

3. Access Content Administration -> KM Content -> root -> workflow, then choose menu Folder -> Details. In the new popup window, please choose Settings -> Permissions. Then in this page, please adjust the permission as you preferred. If you would like all sub folders and files inherit the same permission settings, please use button "Reset Child Permissions".

  

After above configuration, if access attachment again using direct link, it will show information as following. This is expected behavior. This means resource has been correctly protected against unauthorized users.

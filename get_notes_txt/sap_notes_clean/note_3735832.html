Symptom

  • In transaction SEGW, selecting Check Project Consistency or running Syntax Check on an OData service project results in a short dump.
  • The behavior appears after an upgrade to a 758 or higher release.

Environment

  • SAP S/4HANA
    • 2023 release
    • 2025 release

Reproducing the Issue

  1. Start transaction SEGW.
  2. Open the affected OData service project.
  3. Choose Check Project Consistency or run Syntax Check.
  4. A short dump occurs.

Cause

  • Starting with SAP_BASIS 758, stricter validation of naming conventions is enforced for OData services.
  • The service uses inconsistent naming between Entity Types and Entity Sets (Entity Types do not end with “Type”, or Entity Set names do not match the base name), which leads to a dump during the consistency/syntax check.

Resolution

  1. Review all CDS DDLS artifacts and annotations used by the SEGW project.
  2. For every entity, ensure the @OData.entityType name ends with “Type” and the corresponding @OData.entitySet name uses the same base name without the “Type” suffix (e.g. @OData.entityType.name: 'ActivityType', @OData.entitySet.name: 'Activity').
  3. Apply this naming convention consistently across all DDLS referenced by the service.
  4. Regenerate the SEGW service runtime objects.
  5. If a crash persists during checks even after correcting the naming, implement SAP Note 3622844 and retest.

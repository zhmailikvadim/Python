Symptom

As per KBA:

2549021 - Codes and code groups in search help of a quality notification differs from the values defined in its catalog profile
"...
The following is documented in IMG activity documentation of Define Catalog Profile:

"...Each notification type is assigned a specific catalog profile.
You can also store a catalog profile in the material master. This catalog profile takes precedence over the catalog profile assigned to the notification type..."
..."

It means the catalog profile in material master is primary, while catalog profile in notification type is secondary.

However, after the system update, this system logic seems to be ignored.

Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.

Environment

  • SAP enhancement package 8 for SAP ERP 6.0
  • SAP S/4HANA 2020
  • SAP S/4HANA 2021
  • SAP S/4HANA 2022
  • SAP S/4HANA 2023
  • SAP S/4HANA Cloud Private Edition 2023

Reproducing the Issue

  1. Call transaction QM01 for the affected Notification type.
  2. ENTER.
  3. Fill necessary information, like Material, Plant for mat., Description, and so on.
  4. ENTER.
  5. Goto menu 'Extras > Settings > Catalog Profile > Catalog Profile'
    --> Current and Default catalog profile is mixed.

Cause

Side effect of SAP Note:

3466440 - IW21: BAdI to Change the Catalog Profile Assigned to a Notification not triggered

Resolution

Implement SAP Note:

3692346 - QM01: Incorrect catalog profile determination when material is entered

Symptom

• In transaction code V/06, the 'Plus/mins' indicator (field V_T685A-KNEGA) is set for the condition type. 
• When manually entering a positive condition value in VA01 or VA02, the condition amount remains positive instead of changing to negative as expected.

Environment

  • Sales And Distribution (SD)
  • Logistics Execution (LE)
  • SAP R/3
  • SAP R/3 Enterprise 4.7
  • SAP ERP Central Component
  • SAP ERP
  • SAP Enhancement package for SAP ERP
  • SAP Enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA
  • S/4HANA Cloud Private Edition

Reproducing the Issue

  1. Create a sales order in VA01 or modify a sales order in VA02.
  2. Navigate to the header condition screen.
  3. Manually enter a condition value.
  4. Observe that the condition amount remains positive instead of changing to negative, despite the 'Plus/mins' indicator (field V_T685A-KNEGA) being set to 'X'. 

Cause

The 'Plus/mins' indicator (field V_T685A-KNEGA) is designed to work exclusively with the 'amount' field. 

Resolution

To assign a negative value, input the value in the 'amount' field instead of the 'condition value' field.

See Also

774424 - Plus/minus signs of condition amount and value (KNEGA and SHKZG)

F1 help documentation:

Plus/Minus Sign of the condition Amount
Controls whether the condition results in an amount that is negative (a discount), positive (a surcharge), or whether both positive and negative amounts are possible.
Procedure
If the condition amount can be either positive or negative, leave the field blank. Otherwise, enter one of the indicators that specify plus or minus.

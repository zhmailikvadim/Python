Reason and Prerequisites

Steps to reproduce:

1, Send a IDoc request using Postman to Datahub

2, Verify the datahub log.

Reason for Error :

The Webservice logs are managed by LoggingResourceAspect, which is defined in 'datahub-logging-aop-spring.xml’ . It only supports javax.ws.rs for v1 in jdk17:

<aop:config>
<aop:aspect id="loggingResourceAspect" ref="loggingResource">
<aop:before method="logGet" pointcut="@annotation(javax.ws.rs.GET)"/>
<aop:before method="logPost" pointcut="@annotation(javax.ws.rs.POST)" />
<aop:before method="logDelete" pointcut="@annotation(javax.ws.rs.DELETE)" />
<aop:before method="logPut" pointcut="@annotation(javax.ws.rs.PUT)" />
</aop:aspect>
</aop:config>

That mean, we never provide the webservice log support for v2. Furthermore, with the jdk21 upgrade, we now use jakarta.ws.rs for v1 and Spring webservice framework for v2. Therefore the aop pointcut doesn't work and no logs are printed for neither v1 or v2 in 2211 jdk21.



Solution


We need to add log support for jakarta.ws.rs in v1 and for Spring ws in v2


The correct version will be delivered in the following versions : 2211.50, 2211-jdk21.9*

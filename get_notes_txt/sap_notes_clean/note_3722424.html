Symptom

Error message "HU content was changed - GR cancellation not possible" Message no. /SCWM/GM069 occurs, when attempting to reverse Goods Receipt for an EWM inbound delivery where a SKIP inspection lot was created. The status DCO of the delivery item is not completed, but reversing the Goods Receipt is not possible. The issue does not occur when the inspection is not set to SKIP.

Environment

  • SAP S/4HANA
  • SAP Extended Warehouse Management (SCM-EWM)

Reproducing the Issue

  1. Create an inbound delivery in EWM.
  2. Perform Goods Receipt (after packing).
  3. Verify that the Quality Inspection document is created and the stock is posted to Quality stock type. Confirm that the ERP system returns an Inspection lot number, which is set to SKIP.
  4. Observe that a transfer posting is performed from Quality to Unrestricted stock type in EWM, and the Quality Inspection document reference is removed from the stock.
  5. Ensure that no usage decision has been performed and the status of the delivery item is still DCO = 1 (not completed).
  6. Attempt to reverse the Goods Receipt and observe the error message /SCWM/GM069 stating that the goods receipt cannot be cancelled/reversed due to changes in the handling unit.

Cause

The issue occurs due to the immediate stock posting from Quality to Unrestricted stock resulting from the SKIP inspection. This process removes the Quality Inspection reference and changes the stock type, preventing Goods Receipt Reversal (GRR).

GRR is only possible, if the following conditions exist:
    • GRR is posted on the same level where the GR was originally posted,
    • If the stock is in the original storage bin (otherwise first we need to move the stock back to this storage bin),
    • The attributes of the stock were not changed,
    • It also has to contain the quality reference (there must be no decision/skip on this stock).

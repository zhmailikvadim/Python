Symptom

When replicating Equipment from an SAP S/4HANA system to SAP Field Service Management using the Mobile Application Integration Framework (MAIF), the replication fails. 

Error encountered: {"error":"CM-02","message":"CM-02: The service experienced an unexpected error."



Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental. 

Environment

  • SAP S/4 HANA Private Cloud / On-Premise (S/4 HANA)
  • Mobile Application Integration Framework (MAIF)
  • SAP Field Service Management (FSM)

Reproducing the Issue

  1. Update an Equipment in the SAP S/4HANA system
  2. Check Inbound Transaction Monitoring in MAIF.
  3. Actual Behavior:
  4. The MAIF inbound transaction fails with an CM-02 error, and the Equipment change is not replicated to SAP FSM.
  5. Expected Behavior:
  6. The Equipment should be successfully replicated from SAP S/4HANA to SAP FSM.

Cause

The request was rejected because externalId is being passed as the ID in the PATCH-by-ID API. The API expects a valid ID, not an externalId string.

UI_URL_PATH properties still contain the old values

Resolution

To ensure correct replication, please adjust the following settings as described in the documentation below:

Documentation:
https://help.sap.com/docs/SAP_S4HANA_ON-PREMISE/738a456365c0414faba6426d05fd8674/207ec4f763a8428fb575095a29b3148f.html

Please make sure these three properties are maintained exactly as shown:

  • UI_URL_PATH, ADDRESS → /api/fsm-connector/data/v4/Address
  • UI_URL_PATH, EQUIPMENT → /api/fsm-connector/data/v4/Equipment
  • UI_URL_PATH, FUNLOC → /api/fsm-connector/data/v4/Equipment

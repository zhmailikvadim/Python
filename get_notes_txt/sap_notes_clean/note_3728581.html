Symptom

  • The user synchronization to SAP Identity Authentication Service (IAS) fails.
  • The following error is observed in the Identity Provisioning Service (IPS) job:

status=400, detail=Cannot construct instance of `com.sap.security.iag.idstore.api.external.scim.model.ScimEmail` (although at least one Creator exists): no String-argument constructor/factory method to deserialize from String value ('XXX@email.domain')

Environment

Identity Provisioning Service (IPS)

Reproducing the Issue

  1. Trigger the synchronization process by running an IPS job.

  2. Observe the error.

Cause

The issue arises due to an incorrect email transformation setup in the IAS target system while using "ias.api.version": "2". 

                {
                    "sourcePath": "$.emails[*].value",
                    "preserveArrayWithSingleElement": true,
                    "targetPath": "$.emails"
                },

Resolution

To resolve the synchronization issue, modify the transformation to utilize the Default transformation for SCIM API version 2 as follows:

  1. Edit the IAS target system.

  2. Replace the existing transformation with the following configuration:

          {
            "sourcePath": "$.emails",
            "preserveArrayWithSingleElement": true,
            "targetPath": "$['urn:ietf:params:scim:schemas:extension:sap:2.0:User']['emails']",
            "scope": "createEntity",
            "functions": [
               {
                     "function": "putIfAbsent",
                     "key": "verified",
                     "defaultValue": true
               }
             ]
          },
          {
            "targetPath": "$['urn:ietf:params:scim:schemas:extension:sap:2.0:User']['emails'][*]['type']",
            "type": "remove"
          },
          {
            "sourcePath": "$.emails[*].['value','display','primary']",
            "preserveArrayWithSingleElement": true,
            "targetPath": "$.emails"
          },
     

  3. Save the changes.

  4. Run the Read job.

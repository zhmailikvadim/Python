Symptom

Some doubts may appear while setting up the CTMS when configuring the destination.

There are two methods to maintain the URL in destinations:

Using the names of your org and space:
https://deploy-service.cf.<domain>/slprot/<myorg>/<myspace>/slp 

Using the GUID of your space:
https://deploy-service.cf.<domain>/slprot/<my-space-guid>/slp

-

It is recommended to use the GUID because when using the names of org and space, a typo might occur or the URL encoding is not being used correctly (for example, using %20 for space characters). 

So in general, using the space GUID is recommended.

Environment

SAP Cloud Transport Management

Reproducing the Issue

Set up the CTMS.

Cause

Using the names of org and space, a typo might occur or the URL encoding is not being used correctly (for example, using %20 for space characters). 

Resolution

There are two methods to find the GUID of a space.

1 - To retrieve the GUID of your space, use the Cloud Foundry Command Line Interface (cf CLI). Log on to your org, and execute the following command: cf space <my-space-name> --guid.

or 

2 - You can access your target space and check the browser URL. If you follow the URL from the left to right, you will see these two words: /org/ and /space/.
Right after, the GUID of each will be displayed, so it will be similar to this:

/org/<GUID>/space/<GUID>

So, to maintain your destinations, the GUID after /space/ can be used if needed.

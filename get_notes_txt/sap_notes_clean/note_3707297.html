Symptom

  • The WBS elements under the main hierarchy do not have a 'Person Responsible' or 'Project/WBS Manager' field.
  • This absence causes the Supplier Invoice Workflow to default to the main Project approver, instead of routing to specific cost centers or individuals responsible for the WBS elements.

Environment

SAP S/4HANA Cloud Public Edition

Reproducing the Issue

  1. Open the "Project Control - Enterprise Projects" application.
  2. Enter the project ID in the search bar and select the project.
  3. Navigate to the related application - "Project Planning".
  4. Expand the drop down to view the WBS elements below the main hierarchy and observe that there is no 'Person Responsible' or 'Project/WBS Manager' field.

Cause

  • The 'Person Responsible' or 'Project/WBS Manager' field is not a standard field available in SAP S/4HANA Public Cloud Edition. It is expected system behavior by design.

Resolution

  1. Utilize app extensibility to add a custom field for the responsible person.
  2. Refer to the SAP documentation for guidance on app extensibility:

    SAP Help Portal - SAP S/4HANA Cloud App Extensibility

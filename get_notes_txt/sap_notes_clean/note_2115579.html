Symptom

During condition records maintenance via SAPGUI (transaction /SAPCND/GCM), if there are certain fields moved to header and an error is introduced in the record, then the log button at the 'Item' area and well as 'Header' area display the error icon along with the number of errors. Now if the error is resolved then the log button at the 'Item' area shows that there are no errors but the log button at the 'Header' area is not refreshed and still shows the error icon with number of error messages.

Other Terms

Header Area, condition maintenance, Error messages

Reason and Prerequisites

Program Error

Solution

This issue should not occur from SAP_AP SP 33 onwards. For fixing this issue till SP32 please apply the attached correction instructions.

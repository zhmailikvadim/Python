Symptom

  • Not able to get the data refreshed in Excel when using ODATA to get data from Web Intelligence document
  • The retrieved data from the ODATA call is always the data stored in the current Web Intelligence report

Environment

  • SAP BusinessObjects Business Intelligence Platform 4.3 / 2025
  • Web Intelligence

Reproducing the Issue

  1. Follow the guidance in KBA 3297129 - How to use ODATA Web Service as data source in Microsoft Excel? to configure Excel to consume data from Web Intelligence
  2. After the backend database data changed, try to refresh the data in Excel.
  3. Notice that the data is not changed. It remains to be the data which is stored in the Web Intelligence document.

Cause

Resolution

Starting from Support Pack 04 of SAP BusinessObjects 4.3, a new parameter sap_refreshdata=true is added which can be used for refreshing data first. Therefore, after copying the ODATA Feed URI, in the end you can add the request parameter sap_refreshdata=true and use this in the Excel when building the ODATA call.

E.g. before add the parameter, the ODATA can be: http://<web server>/biprws/raylight/v1/documents/cuid_<cuid>/datamodel/data.svc/re_<reportID>-<elementID>
After add the parameter, the new ODATA can be: http://<web server>/biprws/raylight/v1/documents/cuid_<cuid>/datamodel/data.svc/re_<reportID>-<elementID>?sap_refreshdata=true

Symptom

When posting a document using one of the following RFA functions: Finance Accounts Payable, Finance Accounts Receivable, Finance Accounts Payable Items, Finance Accounts Receivable Items, you find the following inconsistency: the value of Tax Base Amount is equal to the value of Tax Amount.

This is only observed for use cases when the Tax Rate is not equal to zero.

Other Terms

LOD-PER-OP Rel 3.0 SP 27

Reason and Prerequisites

Program error

Solution

In order to get the solution, please implement the automatic correction instructions using transaction SNOTE and perform the manual post-implementation steps.

You will now find these new components on the RFA Rules tab:

 - for Finance Accounts Payable: amount[ 6 ]-amt_base(_long)

 - for Finance Accounts Receivable Rules tab: amount[ 3 ]-amt_base(_long)

 - for Finance Accounts Payable Items and Finance Accounts Receivable Items: amount-amt_base(_long)

For tax related line items, the new component AMT_BASE should be mapped to the value of Tax Base Amount and the component AMT_DOCCUR should be mapped to the value of Tax Amount.

For all the other items of a document, the component AMT_BASE can be left empty.

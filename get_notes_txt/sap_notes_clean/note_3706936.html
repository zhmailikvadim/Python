Symptom

  • The custom widget tab is missing from the customer's SAP Neo tenant.
  • Preventing users from creating, viewing, or managing custom widgets.

Environment

  • SAP Analytics Cloud (Enterprise)
 

Reproducing the Issue

  1. Access the SAC Neo tenant.
  2. Navigate to "Stories" 
  3. Observe the "Custom Widgets" option is unavailable.

Cause

  • The Custom Widget option is controlled by user role permissions in SAC.
  • In the affected NEO tenant, the logged-in user did not have the required role permissions (System Owner, Admin, or a custom role with Custom Widget access enabled).
  • As a result, the Custom Widget menu was not displayed, even though it may have been visible previously.

Resolution

Grant the appropriate permissions to the user by assigning a supported role.

Refer to: Standard Application Roles | SAP Help Portal.

Recommended Steps:

  1. Assign the System Owner or Admin role to the user
    OR
  2. Create a custom role by:
    • Copying an existing role
    • Enabling the Custom Widgets permission
    • Saving the role
  3. Assign the updated role to the affected user.
  4. Log out and log back into SAC to apply the changes.
  5. Verify that the Custom Widget option is now visible and accessible.

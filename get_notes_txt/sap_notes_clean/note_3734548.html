Symptom

  • Fiori applications do not load after a system patch; pages remain blank.
  • Browser console shows 404 errors for UI5 resources.
  • Error: “failed to load '/sap/public/bc/ui5_ui5/1/resources/sap/suite/hcm/commons/library-preload.json': NOT FOUND”
  • Error: “GET /sap/public/bc/ui5_ui5/1/resources/~~/sap/ui/commons/DropDownBox.js 404 (NOT FOUND)”
  • Error: “failed to load 'sap/suite/hcm/commons/library-preload' (failed to load 'sap/suite/hcm/commons/library-preload.js' from /sap/public/bc/ui5_ui5/1/resources/~~/sap/suite/hcm/commons/library-preload.js: 404 - NOT FOUND)”
  • Issue observed across multiple apps, including leave request, payslip, timesheet, and profile apps.

Environment

  • UI5 ABAP delivery tools

Reproducing the Issue

  1. Open any UI5-based Fiori app (for example, leave request, payslip, or timesheet).
  2. Observe that the page stays blank and the application does not render.
  3. Open the browser developer tools console and verify 404 errors for UI5 resources (for example, sap/suite/hcm/commons/library-preload.* or sap/ui/commons/DropDownBox.js).

Cause

Resolution

Apply note 3724943 - HCM resources not found by SAPUI5 applications

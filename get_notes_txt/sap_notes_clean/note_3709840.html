Symptom

  • In the App Maintain bill of material (version 2), the 'selected date' or 'key date' defaults to the BOM header validity date instead of the current date.
  • BOM items are filtered based on the BOM header validity date, displaying items as of that date.
  • This issue does not occur in the app of material (version 1), where the 'selected date' defaults to the current date.

Environment

SAP S/4HANA Cloud Public Edition

Reproducing the Issue

  1. Open the App Maintain bill of material (version 2)).
  2. Select a BOM for a material that has changes with a change master (header level change number) in the past.
  3. Observe that the application opens the "oldest" version of the BOM with the 'selected date' defaulting to the BOM header validity date.
  4. Repeat the same steps in the app of material (version 1) and observe that the 'selected date' defaults to the current date.

Cause

App Maintain bill of material(version 2) only shows header instances with Change number automatically .Items are only exposed in the Object Page and that where show all Change numbers on Items and user can select which change number wants to see. That is why by default its Header linked Change number.

Resolution

It is standard behavior.

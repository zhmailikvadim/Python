Symptom

when processing a large set of records exceeding 100,000 using the standard process template and standard match configuration, the matching results are not displayed for all records and the best record calculation consistently shows 100,000 records.

Environment

SAP Master Data Governance-Consolidation and Mass Processing

Reproducing the Issue

  1. Launch Fiori Launchpad
  2. Select Import Data for Consolidation-Business Partners/Products
  3. Upload a file with large set of data exceeding 100,000 records 
  4. Matching results are not displayed for all records

Resolution

  • To keep the performance of the consolidation process high, for processes containing more than 100.000 records changes are not displayed.
    Review : Match Review under section Constraints
  • For Best Record Calculation step, in each group, only 1 record is marked as best record. Number of groups basically equals number of best records. 

Keywords

Best record, Matching, Best record calculation, Match groups, Match review

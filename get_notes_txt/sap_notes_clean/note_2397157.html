Symptom

How to identify the FP index is a nbit or flat FP index ?

Environment

SAP IQ 16

Cause

As part of the post-migration process, all FP indexes should be rebuilt into NBit using sp_iqrebuildindex(). Customer would like to know how many are rebuilt after the process ?

Resolution

  • Use "sp_iqcolumnmetadata" and see 'Style' column

         Syntax =>    sp_iqcolumnmetadata <[ table.name [, owner-name ] ]>

  • This query list all the flat FP indexes:

    select U.name,table_name,index_name, iqc.cardinality
    from  sysidx I1,  sysiqidx I2, systab T, sysusers U, SYSIDXCOL  c,SYSIQTABCOL iqc
    where T.table_id = I1.table_id
    and T.table_id = c.table_id
    and c.index_id =  I1.index_id
    and iqc.table_id = T.table_id
    and iqc.column_id = c.column_id
    and iqc.is_nbit = 'F' /* flat FP */
    and  (I2.table_id=I1.table_id and I2.index_id = I1.index_id)
    and  T.creator = U.suid
    and  I2.index_type = 'FP'
    ORDER BY U.name;
  • This query list all the nbit FP indexes:

    select U.name,table_name,index_name, iqc.cardinality
    from  sysidx I1,  sysiqidx I2, systab T, sysusers U, SYSIDXCOL  c,SYSIQTABCOL iqc
    where T.table_id = I1.table_id
    and T.table_id = c.table_id
    and c.index_id =  I1.index_id
    and iqc.table_id = T.table_id
    and iqc.column_id = c.column_id
    and iqc.is_nbit = 'T' /* nbit FP */
    and  (I2.table_id=I1.table_id and I2.index_id = I1.index_id)
    and  T.creator = U.suid
    and  I2.index_type = 'FP'
    ORDER BY U.name;

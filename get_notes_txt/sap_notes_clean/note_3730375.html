Symptom

  • T-code  /accgo/em_acc which is for Auto accrual configuration is allowing auto accrual only for document types Goods issue and goods receipt "L" and "G" it is not enabled for "A" sales order and "c" purchase order so that post LDC.
  • Unable to provide document type other than G or L as inputs.

Reproducing the Issue

  1. Go to T-code /accgo/em_acc.
  2. go to change mode.
  3. Try to add entries with company code/expense classes.
  4. Try to give any document type other than G and L.

==> System won't allow to save and will throw error document types other than "G" /"L".  

Resolution

Standard ACM will support only the G and L documents; other documents are not allowed for ACM-specific processes. Hence, the validation works as expected.

Symptom

  • Data Services Designer Client is unable to connect to the repository.
  • Error message: ODBC call for data source <> failed: [Microsoft][ODBC SQL Server Driver][DBNETLIB]SQL server does not exist or access denied. [Microsoft][ODBC SQL Server Driver][DBNETLIB]ConnectionOpen(Connect()). Notify Customer Support. (BODI-1112172)


  • Additional observations:
    • Connection test to the repository from CMS is working
    • The error message in Designer displays one particular data source name, regardless of what is set in CMC
    • Telnet to the DB host from the Designer Client machine on the DB port using HOSTNAME only fails
    • Telnet to the DB host from the Designer Client machine on the DB port using FQDN is successful

Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.

Environment

  • All supported SAP Data Services Client versions
  • All supported Windows operating systems
  • Microsoft SQL Server

Reproducing the Issue

  1. Open Designer Client
  2. CMS authentication successful
  3. Attempt to log in to a repository.
  4. Observe the error message:
    ODBC call for data source < > failed: [Microsoft][ODBC SQL Server Driver][DBNETLIB]SQL server does not exist or access denied. [Microsoft][ODBC SQL Server Driver][DBNETLIB]ConnectionOpen(Connect()). Notify Customer Support. (BODI-1112172).

Cause

  • The client and the DB server are in different networks and the Designer client machine can not resolve the simple hostname for the database
  • The database host is coded in the database during repository creation.
  • Repositories were created using simple hostname rather than FQDN or IP, and machines outside the network are unable to resolve the hostname.

Resolution

  1. Create or recreate the repository via Repository Manager using the Fully Qualified Domain Name (FQDN) of the SQL server host. Note: This method is suitable for new repositories as existing data will be wiped during the process.

  2. Edit the problematic repository and manually update the server details stored in the AL_MACHINE_INFO table of the repository database.
    • Update the RESERVED2 column in the AL_MACHINE_INFO table to reflect the FQDN of the SQL server.
    • Execute the following SQL query for each repository:

      UPDATE AL_MACHINE_INFO
      SET RESERVED2 = '<FQDN>'
      WHERE SERVER_NAME = 'RepoManager';

Symptom

When downloading candidate attachments using the new applicant workbench the user receives the following message 
""The document is too large to be displayed. Download the original file to view it offline." " 

Environment

SAP SuccessFactors Recruiting Management. 

Reproducing the Issue

  1. Open candidate applications
  2. Select the Candidate 
  3. Select Download Documents
  4. Receive message: The document is too large to be displayed. Download the original file to view it offline.

Cause

Total file limit exceeds 1.75GB

Resolution

Scenario 1: Lets say if user has selected 20 applicants and chooses to download all sections and if the memory is going beyond 1.75GB , and lets say for 15 applicants the memory reached 1.75GB then for remaining 5 applicants , all sections will show content but only attachments wont be shown after it crosses 1.75GB and message is shown for those 5 applicants with Resume/CoverLetter/OtherAttachment sections as "The document is too large to be displayed. Download the original file to view it offline."

Scenario 2: Lets say if user has selected 10 applicants and chooses to download all sections, lets say the memory will not crossed 1.75GB for all 10 applicants, but lets say for 4th applicant, the 1 converted attachment that has any 1 page more than 5MB (which means 1 page of original attachment is bigger than 5MB after conversion), then that specific attachment will not be shown the user will see "The document is too large to be displayed. Download the original file to view it offline." message will be shown

The attachment would need to be downloaded in order to view the attachment.

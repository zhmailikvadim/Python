Symptom

  • IP restrictions are not displaying in the CDC Admin Console after adding new IP addresses through the IP Restrictions menu.
  • Example IPs added include: 3.127.xx.xx, 3.72.xx.xx, 3.67.xx.xx.

Environment

  • Customer Data Cloud
  • System (Platform Security, Network)

Reproducing the Issue

  1. Add new IP addresses through the IP Restrictions menu in the CDC Admin Console.
  2. Wait for 5 minutes to verify if the IPs appear in the List Definition.
  3. Check the List Definition under IP Restrictions in the CDC Admin Console to confirm the absence of the added IPs.

Cause

  • The IP addresses in question already exist inside subnets of the List Definition. Subnet entries in the IP restriction list cover all IPs within the CIDR range, and exact IPs falling within the subnet are already allowed or blocked based on the rule.

Resolution

  1. Confirm that the IP addresses are already covered by the existing CIDR ranges in the IP restriction list.
  2. Refer to the mappings provided to verify inclusion:
  • 3.127.xx.xx → 3.124.0.0/14
  • 3.72.xx.xx→ 3.64.0.0/12
  • 3.67.xx.xx → 3.64.0.0/12

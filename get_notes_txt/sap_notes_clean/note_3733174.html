Symptom

  • When initiating a call by clicking a customer’s phone number in the Agent Desktop, the CTI softphone displays "Incoming Call" instead of "Outgoing Call".
  • This occurs even though the call was triggered by the agent.

Environment

  • SAP Service Cloud Version 2 1.0
  • SAP Sales Cloud Version 2 1.0
  • CTI Integration (third-party provider)

Reproducing the Issue

  1. Open the Agent Desktop.
  2. Select a Business Partner (BP).
  3. Click on the phone number.
  4. Observe the CTI window behavior:
    • Call is initiated.
    • CTI incorrectly shows "Incoming Call".

Cause

  • The issue is caused by an incorrect CTI payload sent from the CTI provider to SAP Service Cloud Version 2.
  • Specifically, the call direction attribute is misconfigured or missing, leading the system to interpret the call as inbound instead of outbound.

Resolution

  1. Review the CTI payload sent during outbound call initiation.
  2. Ensure that the payload includes the correct call direction indicator (e.g., outbound).
  3. Validate the payload structure against SAP documentation.
    • For example:
      • "<?xml version='1.0' encoding='utf-8' ?>
            <payload>
                <EventType>OUTBOUND</EventType>
                <Action>NOTIFY</Action><ExternalReferenceID>c230aa65-e9c2-4b31-9f9d-ac06d6da7f1d</ExternalReferenceID>
           </payload>"
    • instead of : 
      • "<?xml version='1.0' encoding='utf-8' ?>
            <payload>
                <EventType>INBOUND</EventType>
                <Action>NOTIFY</Action><ExternalReferenceID>c230aa65-e9c2-4b31-9f9d-ac06d6da7f1d</ExternalReferenceID>
           </payload>"
  4. Coordinate with your CTI provider to correct the payload configuration.
  5. Retest the scenario after adjustments.

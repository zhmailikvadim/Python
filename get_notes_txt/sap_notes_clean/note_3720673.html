Symptom

  • Even after updating to the latest versions, FSM clients are still interacting with older *.coresuite.com domains.

Environment

FSM

Reproducing the Issue

Review the stats to identify traffic still interacting with old domains (*.coresuite.com):

https://eu.fsm.cloud.sap/admin/accounts/<fsm account id>/legacyDomainAuthUsageStats

Cause

The central authentication authority is configured to redirect successful logins to coresuite.com.

Resolution

Collaborate with IT department to ensure all clients using *.coresuite.com domains are updated or reconfigured.

The configuration for providing Authentication is a central configuration point for the Domain used by apps to contact the FSM cloud.

Please consult this documentation pages for instructions on how to migrate the configuration of your Authentication Authority:

  • Open ID Connect:
    Field Service Management - Federated Authentication - Migrate OpenId Configuration to Use New SAP Domain
  • SAML (e.g. Azure AD, MS Entra ID): 
    Field Service Management - Federated Authentication - Migrate Existing SAML Configuration to Use New SAP Domain
  • SAP IPS:
    Field Service Management - Federated Authentication - Migrate IPS Integration to New Super Domain

Symptom

You are using the SAP Forecasting and Replenishment (F&R) interfaces to transfer Switches (Switchover) from SAP ERP to SAP F&R.

You encountered the following scenario:
When more than one records exist in Substitution Assisgnment table (database table WRF_FOLUP_TYP_A) for the same Original Material and Substitute Material, and one of the existing records is already in the past (Expired Assignments).
AND
When CurrentDate was greater than the sum of (Past existing record(s) + Specified day(s) maintained in Customizing Setting), the system always took the Past existing record(s) without considering the remaining Recent record(s)

=> This caused the remaining Recent record(s) not being transferred to SAP F&R correctly via either DeltaLoad or Init.Load.

Rootcause of the issue was due to: the oldest record was always considerred first which result in the remaining Recent record(s) was never considered during Evaluation of Substitution Assignment period.

Other Terms

Switchover, Follow-up, Replacement, Event-controlled, DeltaLoad, Init.Load, FRE01, FRE02, FRE50, include LFRE_SUBST_SWITCHF02, form SWITCH_DATA_PROCESS, CT_SWITCHDATA, LS_ASSIGNMENT-DATE_TO,  PS_CUSTOMIZING-SUBST_FTIME_FUP, PS_CUSTOMIZING-SUBST_FTIME_REP, L_DATE, SY-DATUM, WRF_FOLUP_TYP_A-TRANSFERRED

Reason and Prerequisites

Prerequisites: You utilize the following Customzing: IMG -> Logistics-General -> Substitution Processing -> Activate/Process Substitution Processing -> section Switchover Information.
You set the specific day(s) for switchover information remains in the ERP system for either below scenario(s):
- Follow-Up Time for Replacement Material: Specified days
- Follow-Up time for Follow-up: Specified days

Solution

Implement the corresponding correction instruction.
This correction will recheck the validity of Substitution Assignments during Evaluation of Substitution Assignment period.

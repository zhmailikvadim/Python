Symptom

This KBA explains the additional fund calculation retro run behavior in Singapore payroll.

"Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental."  

Environment

  • Human Capital Management (HCM)
  • SAP ERP Core Component
  • SAP ERP
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA

Resolution

Example:

An employee's basic salary is 5000SGD, and the basic wage type is cumulated to the additional fund base wage type /106. To illustrate the behavior, the example employee has CDAC, SINDA, MBMF, and ECF maintained in IT0181 without considering any actual business scenario.

The original payroll run for period 01.2025 generates additional funds contribution based on /106 in function RADDF as shown below:

The employee's salary increased from 5000 to 7500 effective period 2025.01. Period 2025.01 is retroactively calculated in 2025.02.

For 01 in 02:

  1. The difference between the old base /T06 and the new base /106 is generated in RC40 and saved in /F06.
  2. RC75 replaces the /106 with the old amount.
  3. Function RADDF calculates the additional fund based on the replaced /106 for period 01, generating the same outcome as the original payroll run.

For 02 in 02:

  1. Table LRT is generated in X006.
  2. Difference /F06 is flowed and saved into /106 in RC10.
  3. RADDF calculates the additional fund contribution based on the total /106 amount of 9000 SGD.

Note: The retro run for the additional calculation base negative difference follows the same process as described above.

Symptom

 The current migration report CRMS4_MAINTSRVC_ITMTYP_USG_UPD has following drawbacks

  • No customizing validation - The report updates subitems to RRCO even if appropriate item category customizing is not maintained for them, potentially leading to inconsistencies and errors in the existing service quotations.
  • Lack of Flexibility - The report does not offer customers an option to exclude specific subitems from being updated, even if they do not wish to use RRCO for those subitems.

Other Terms

Service with Advanced Execution, Execution Order Item, Maintenance Service Order, Maintenance Service Quotation, Billable Maintenance Order, Resource Related Cost Items

Reason and Prerequisites

This is due to lack of flexibility and capability report offers.

Solution

Read KBA 3533203, implement the correction instructions and execute the report.

  • Before running the report, you must ensure that all relevant entries in the Item Category Determination Customizing have item category usage RRCO mapped. This means that if the item category usage RRCO is not present in the Item Category Determination Customizing then report will not do anything.
  • The report updates transactional rows in CRMS4D_SERV_I by setting ITEM_CATEGORY_USAGE = ‘RRCO’ after considering the following points:
    1. The item category of the subitem has Item Category Usage = RRCO in Customizing (Define Item Category Determination).
    2. The item category of the subitem passes all the validation checks for RRCO items. For more information on the list of validations, please check the Constraints/Properties section of SAP Help Portal Resource-Related Cost Item (select relevant version and language).
    3. If any of the validation checks fail for a particular item category of the subitem, the program will skip updating ITEM_CATEGORY_USAGE for the affected transactional rows. You can review the messages to identify skipped records and the reason for skipping.

Symptom

You want to withdraw a work item. Up to SP4, when you have withdrawn a work item, the assigned solution documentation was activated to the Production Branch. With SP5, a new behavior has been introduced that prevents the withdrawal of a work item which has changed solution documentation elements.

Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental. 

Environment

SAP Solution Manager 7.2, Focused Build ST-OST 05

Reproducing the Issue

Create a work item and change the assigned solution documentation, e.g. by uploading a technical design document. Now try to withdraw the work item.

Cause

The activation of documentation to the production branch in case the work item is withdrawn doesn’t make sense from functional point of view.Therefore, it must be decided whether the changed solution documentation elements are either deleted or moved to another work item.

Resolution

Open the advanced view of your Work Item (e.g. by clicking on the title in the SAP UI5 application) in the CRM UI. Go to the Solution Documentation assignment block.

1. You can identify the blocking solution documentation elements in the column ‘Change Status’ marked with ‘Changed’. Click on the button ‘Release Check’.

2. The solution documentation is opened. In the first screen, all elements of this work item are shown. To reduce the view to the required information click on the work item title:

3. Now only the changed elements are shown. Right-click on the changed element(s) and decide what you want to do, Discard the changes or move them to another change document (only work items are listed where you are part of, e.g. as developer or tester). Please note: A discard could lead to a complete deletion of the Solution Documentation Element, therefore check also the help text in this link.

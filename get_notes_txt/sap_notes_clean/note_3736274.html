Symptom

  • Payment proposal job log shows Error while Mapping Customs Docmt Data () in GTS.
  • Payment proposal fails due to error message FIPAY_GTS121.

Environment

  • SAP S/4HANA 
  • SAP ERP
  • SAP Global Trade Services (GTS)
  • Sanctioned Party List Screening

Reproducing the Issue

  1. Run F110 in the ERP system to create a payment proposal that triggers the GTS compliance check.
  2. Review the job log for the F110 proposal run.
  3. Observe messages: "technical error in gts" (FIPAY_GTS 120) and "Error while Mapping Customs Docmt Data ()" (FIPAY_GTS 121).

Cause

  • The issue can occur due to incomplete customizing for document number ranges in GTS for payment-related documents (missing current-year interval for object SLL_LEGAL).

Resolution

  1. In GTS customizing (SPRO), navigate to: Global Trade Services > General Settings > Number Ranges > Define Number Ranges for Documents.
  2. Add  the number range (20 and 21) for the current year.

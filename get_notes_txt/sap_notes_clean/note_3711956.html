Symptom

  1. Define a custom field for Portfolio Initiatives and make it visible in the Portfolio Management frontend via customizing "Define Custom Field Configuration" and "Assign Fields to Views/Subviews".
  2. The field is displayed as a dropdown field, but the values of the dropdown are not populated.

Environment

  • SAP Portfolio and Project Management
  • SAP Portfolio and Project Management for SAP S/4HANA
  • SAP S/4HANA
  • SAP S/4HANA Cloud Private Edition 

Reproducing the Issue

  1. Define a custom field for Portfolio Initiatives, assigning a valid search help to the custom field when doing so.
  2. Make the field visible in the Portfolio Management frontend via customizing activities "Define Custom Field Configuration" and "Assign Fields to Views/Subviews".
  3. In the PPM frontend the field is displayed as a dropdown field, however no values are populated for the dropdown field.

Cause

One or more field configuration entries in customizing activity "Define Custom Field Configuration" exist which specify a value for "Activity" (e.g., "Admin", "Read", and "Write") but no entry exists with a blank value for "Activity".

Resolution

Maintain an additional entry for the custom field in "Define Custom Field Configuration" > "UI Configuration" which specifies a blank value for "Activity".

Symptom

There are multiple issues when running the mass geocoding report (/SAPAPO/MASS_GEOCODE_LOC), including:

- geocoordinates are updated even when the report is executed in read-only mode ('Show Existing Geocoordinates' or 'Simulate Geocoordinates' execution option) or when the 'Overwrite Geocoordinates' execution option is selected together with 'Preserve manual coordinates' and manual geocoordinates exist for the location

- discrepancies in showing geocoding results

- it does not show if manual geocoordinates are overwritten

- when specific GC level is selected, the report may update the location header geocoordinates from that specific level instead of the most specific GC level

Other Terms

exclude manual coordinates

Reason and Prerequisites

 Programming error.

Solution

Implement attached correction instructions or apply appropriate Support Package. Corrections of this note corrects the above-mentioned issues and may also correct further issues in the report.

The note should be implemented together with note 3729583 which corrects location legal handler so that manual geocoordinates are not overwritten automatically on location access.

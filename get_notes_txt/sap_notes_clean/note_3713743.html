Symptom

  • Inquiry about the possibility of running a backup database directly to S3 Amazon in SAP IQ.
  • Current setup does not allow backup to shared NFS or FS due to insufficient space for the data e.g DB with (40 TB).

Environment

  • Sybase IQ all versions.

Cause

Resolution

  • It is confirmed that SAP IQ does not natively support S3 buckets for backups. Direct backup to the S3 bucket is not supported by SAP IQ.
  • As a suggestion, customers can backup a database to an s3 bucket one of two main ways:
  1. Direct to the s3 bucket

        a. not supported by IQ, though we did add this to ASE recently;

     2. Use some sort of protocol converter like Fuse drivers. These allows, at the OS level, to create a mount point on the OS that points to an s3 bucket.

        a. SAP played around with these in the past for IQ pushing backups to s3, ADL, and HDLFS. All vie Fuse drivers. Performance was horrible as IQ must now slow down the backup due to the Fuse driver protocol conversion that takes place and the network latency that we add by going over the internet.

        b. It could be done, but isn’t something recommended to do for IQ database sizes.

 

It is better to see a customer backup the IQ database to a local filesystem. Then, have a set of scripts outside of the backup process that copies the files to their s3 storage buckets. Keeps IQ running fast backups but allows them to put the backups on a low cost storage share.

NOTE: Both ways suggested would not be supported by us, and the customer would have to do all by themselves.

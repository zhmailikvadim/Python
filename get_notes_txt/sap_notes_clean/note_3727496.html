Symptom

When attempting to upload an attachment in the FSM Web Application → Expense, the file is accepted and appears to upload in the UI, but cannot be saved, and no error message is displayed.

Environment

SAP Field Service Management

Reproducing the Issue

  1. Log in to the FSM Web Application
  2. Navigate to Planning & Dispatching
  3. Open Time and Material Journal
  4. Create a new Expense or open an existing one
  5. Click on Edit
  6. Add an attachment to the Expense
  7. Save the Expense

Actual behavior:
After saving, the system shows Files (0) saved even though the file was accepted and appeared to upload in the UI.
No error message is displayed.


Expected behavior:
The attachment should be successfully saved and displayed under Files (1) (or the correct number of uploaded attachments).
The system should store the uploaded file without errors.

Cause

The issue was caused by customer-side network and environmental conditions (VPN routing, regional restrictions, or browser interference), which prevented the attachment upload request from being correctly processed even though the UI accepted the file.

Resolution

  • Ensure stable network access, especially when using a VPN
  • Log out and log back into the FSM Web App
  • Clear the browser cache completely or use the FSM Web App in incognito/private mode
  • Test from a different supported browser (e.g., Chrome, Firefox)
  • Disable browser extensions temporarily (e.g., AdBlock, antivirus extensions, download managers)
  • Test without VPN or using a different network connection, depending on the environment

Symptom

You edit the address of a patient. You enter an Irish address with a postal code that does not correspond to the format that is relevant for Ireland. The following symptoms may occur:

  • The system does not issue a corresponding error message in SAP Patient Management.
    When you save the patient, the system issues the following warnings:
    • AM 653 Postal code 12345 must have length 8
    • N3 222 Error when creating FI customer

Or

    • AM 699 No B,G,I,J,L,M,O,Q,S,U,0,1,2,3,4,5,7,8,9 in the postal code
    • N3 222 Error when creating FI customer

The system saves the patient data, but does not create the FI customer or does not change an existing FI customer.

  • When you save the patient, the system displays the customer data in maintenance mode and issues one of the following error messages:
    • AM 653 Postal code 12345 must have length 8
    • AM 699 No B,G,I,J,L,M,O,Q,S,U,0,1,2,3,4,5,7,8,9 in the postal code

You must maintain a correct postal code to be able to save the customer.

Other Terms

Postal code, postal code, address, address, NPLZ, T005, ADDR_POSTAL_CODE_CHECK, POSTAL_CODE_CHECK, AM653, AM 653, AM699, AM 699, N3222, N3 222

Reason and Prerequisites

This is caused by a program error.

If the business function ISH_BP_OM is not active, the system still uses the function module POSTAL_CODE_CHECK to check the format of the postal codes in SAP Patient Management. In this function module, the checks for some countries that require specific templates for the postal code check are missing or the existing checks may not be up-to-date.

However, the new business partner uses the function module ADDR_POSTAL_CODE_CHECK. This contains more check templates for postal codes than POSTAL_CODE_CHECK. Among other things, the template for the country Ireland. For this reason, the system issues the relevant error message during the check in the FI customer if the postal code does not correspond to the correct format. If the business function ISH_BP_OM is active, the system also issues the error messages during the processing of the patient master data in SAP Patient Management if the format of the postal code is incorrect.

Prerequisites:

  • In the Implementation Guide (IMG) activity Set Country-Specific Checks (NetWeaver -> General Settings -> Set Countries), you have defined the currently valid settings for the postal code for the country Ireland (IE):
    • Length 8
    • Checking rule 9 (check against country-specific template)
  • The business function ISH_BP_OM ("SAP Patient Management Rearchitecture BP/OM") is NOT active in your system. This business function is available as of SAP enhancement package 4 for SAP ERP 6.0, Industry Extension Healthcare (IS-H 604).
  • When you save the patient, the system only displays the data of the FI customer if the IS-H system parameter DEB_ERR Display FI Transactions in Case of Error is selected and the FI customer sends an error message.

Solution

Implement the attached correction instructions or import the specified support package.

Ensure that the following SAP Notes are implemented in your system:

  • BC-SRV-ADR 3462785 Length of Irish postal codes changed from 7 to 8
  • BC-SRV-ADR 2690217 Irish Postal Code (Eircode) Validation
  • BC-SRV-ADR 2203055 Eircode: Ireland’s new postal code system

After you implement the corrections, the system uses the function module ADDR_POSTAL_CODE_CHECK to check the format of a postal code in SAP Patient Management even if the business function ISH_BP_OM is not active. As a result, the check for the current format of the postal code for Ireland is now also available, for example.

After you implement the corrections, there are no additional checks in SAP Patient Management. The existing checks only use another function module.

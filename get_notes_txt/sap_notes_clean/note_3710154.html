Symptom

Having created a custom User Role and given all possible permissions, there are still some tabs on the Admin portal that are not visible to the Custom User Role, only to the SUPERUSER standard role

Environment

SAP Field Service Management

Reproducing the Issue

  1. Create a new User Role and mark all the checkboxes for permissions
  2. Assign this User Role to an User
  3. Login to Admin portal or the shell application with this User

Expected behavior: 

The user should see all the tabs that are available for the SUPERUSER role as well

Actual behavior:

The following tabs are missing:

  • User Roles
  • User Statistics
  • Data Subjects
  • Personal Data
  • OpenID Connect Configuration
  • Central Configuration Management
  • Auto-Scheduling
  • Extensions

The Audit Logs tab is visible but the user receives a "Forbidden from server side" error screen.

The Auto-Scheduling tab is visible but the user cannot modify or create the auto-scheduling policies.

Cause

The above mentioned functionalities are dependent exclusively of the SUPERUSER role, and cannot be given access to by a custom User Role.

Resolution

This behavior is currently by design, to limit the access to functionalities that have the potential to completely break and turn the account unusable if misused

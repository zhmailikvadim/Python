Symptom

  • Using a license server
  • Current licenses were updated on SAP Portal and a new licence (.lic) file was generated
  • The reread of the licenses generates an error:

    lmgrd reread

    Reloading licenses for sybase license manager ... lmutil ...
    lmreread failed: Cannot connect to license server system. (-15,29) : SAP ASE
  • SYBASE.log shows the following warning messages at the top of the log :

    (lmgrd) Rereading license file... Requested from sybase@hostname
    (lmgrd) WARNING: 2 license server systems specified in license file.
    (lmgrd) (an even number of server systems is probably a mistake.)
    (lmgrd) You may have duplicated a SERVER line by mistake
    (lmgrd) ...Finished rereading
  • SYBASE.log shows the following warning messages at the end of the log :

    (SYBASE) No features to serve, exiting
    (SYBASE) EXITING DUE TO SIGNAL 36 Exit reason 4
    (lmgrd) SYBASE exited with status 36 (No features to serve)
    (lmgrd) SYBASE daemon found no features. Please correct
    (lmgrd) license file and re-start daemons.
    (lmgrd) This may be due to the fact that you are using
    (lmgrd) a different license file from the one you expect.
    (lmgrd) Check to make sure that:
    (lmgrd) <path_to_lic_file>.lic
    (lmgrd) is the license file you want to use. 

Environment

  • SAP Adaptive Server Enterprise (ASE) - all versions
  • SYSAM
    • License Cluster (3 node) setup

Cause

One of the SERVER lines in the .lic file is missing the port number at he end of the line. For example :

SERVER hostname hostid

Resolution

Amend the SERVER line to add the correct port number at the end. The correct entry would be :

SERVER hostname hostid  portnum

It may help to refer to previous (working) license file, they should be the same.

Keywords

signal 36, status 36

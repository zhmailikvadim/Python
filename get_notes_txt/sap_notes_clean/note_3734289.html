Symptom

  • From SAP BusinessObjects 4.3 SP04, a new feature is added to add the OLAP connection support to SAP Datasphere
  • How can track the HTTP call detail which is sent from BusinessObjects to get data from SAP Datasphere?

Environment

  • SAP BusinessObjects Business Intelligence Platform 4.3 Support Pack 04+
  • Web Intelligence
  • SAP Datasphere Cloud

Resolution

HTTP Toolkit can be used for capturing the HTTP traffic in this scenario 

  1. Download and install HTTP Toolkit:
    https://httptoolkit.com/
  2. Run it (ideally on the same machine as the BI servers), or on a machine to which the BI servers have access
  3. Navigate to the settings:
  4. Download the certificate
  5. Open a command line
  6. Navigate to <BO Installation>\SAP BusinessObjects Enterprise XI 4.0\win64_x64\sapjvm\bin
  7. Run the command:
    keytool -import -alias HttpToolkit -file <the path where the exported fiddler certificate> -storepass changeit -keystore ..\jre\lib\security\cacerts 
  8. Input yes and press Enter when it is asking the confirmation about the import.
  9. Log on to Central Management Console (CMC)
  10. Go to Servers and edit the properties of Adaptive Processing Server (APS).
  11. In the command line. Add the following to make it use fiddler proxy.
    -Dhttps.proxyHost=<HTTP Toolkit host> -Dhttps.proxyPort=8000
    In case of the HTTP toolkit is installed in the same server and port is the default 8000. The added part can be -Dhttps.proxyHost=127.0.0.1 -Dhttps.proxyPort=8000
  12. Save and restart APS.
  13. Now start Web Intelligence, execute the workflow which is involving SAP Datasphere access.
  14. Requests will be logged inside HTTP Client and can be analyzed

    The output cannot be exported in the free version.

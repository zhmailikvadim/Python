Symptom

You run the UniEmens (RPCUEMI0) report, and the following symptoms occur:

  1. The <InfoAggCausaliContrib> section is generated with <CodiceCausale> having value 0058 for illness absences that happened before March 2026.
  2. The following elements are generated with the P- prefix when the protocol number of the paper certificate is indicated in the Descrizione malattia (UMSCH) field from the Assenze (2001) infotype:
    • <InfoAggEvento> element from the <Giorno> section
    • <MotivoEvento> element from the <DifferenzeAccredito> section
    • <IdentMotivoUtilizzoCausale> element from the <InfoAggCausaliContrib> section

Other Terms

Transaction: PC00_M15_UECALC

&UNIEMENS_INFOAGG_ABSENCE (for more information, see SAP Note 2327917)

/7MD, Mese Differito, InfoAgg, Protocollo

Reason and Prerequisites

Reason

This issue is a side effect caused by the solution delivered by SAP Note 3693900 - UNIEMENS: Paid leave for medical examinations and treatment of disabling, oncological and chronic diseases.

  1. The system does not block the generation of illness absences that happened before March 2026 in the <InfoAggCausaliContrib> section.
  2. The system considers the prefix as part of the protocol number to fill the elements mentioned in the Symptom section.

Prerequisite

The following Support Package (SP) level is the minimum prerequisite:

ReleaseSupport Package
S4HCM 102 HR SP 00
S4HCM 101 HR SP 05
S4HCM 100 HR SP 18
SAP_HR 6.08 HR SP C9
SAP_HR 6.04 (EhP6 and EhP7) HR SP K1

Solution

  1. The system now blocks the generation of illness absences that happened before March 2026 in the <InfoAggCausaliContrib> section.
  2. The system generates the relevant elements without the P- prefix when the protocol number of the paper certificate is indicated.

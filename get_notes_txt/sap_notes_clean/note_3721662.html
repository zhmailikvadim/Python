Symptom

While attempting to establish a connection to a target endpoint, the system fails with timeout‑related errors similar to the following:

com.jcraft.jsch.JSchException: java.net.ConnectException: errno: 78 (A remote host did not respond within the timeout period.), error: Connection timed out (port <port number> to address <IP address>)."

or 

Exception Occured: error 78 - A remote host did not respond within the timeout period. (Connection timed out) (local host unknown, remote port <port number>  to address <IP address><FQDN>

The connection attempt does not complete, and the remote host remains unreachable within the defined timeout window.

Environment

  • SAP NetWeaver
  • SAP Process Integration, connectivity add-on
  • SAP Process Integration, business-to-business add-on
  • SAP Process Integration, secure connectivity add-on

Cause

The issue typically occurs due to network‑level constraints or connectivity interruptions preventing successful communication with the target endpoint/remote host resulting in a connection timeout (error 78).
Possible underlying reasons include:

  • Firewall or security group rules blocking the required port.
  • TCP/IP parameter restrictions affecting connectivity.
  • Misconfigured ACL (Access Control List) rules.
  • Network/ Security level changes
  • The target system being unreachable or not listening on the specified port etc.

Resolution

We recommend coordinating closely with your network, firewall/security, and target endpoint teams to review packet captures, firewall logs, and connectivity diagnostics to pinpoint the underlying network‑level/ target endpoint cause.

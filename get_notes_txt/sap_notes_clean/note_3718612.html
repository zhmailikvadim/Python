Symptom

When using a BW query template with New Lines enabled (e.g., NEW_LINES_COUNT = 1) for planning scenarios, the expected error message is not displayed to the user when the result set state indicates an error condition such as SIZE_LIMIT_EXCEEDED

Observed behavior:
- Template with New Lines : Error message is NOT shown even when the result set size limit is exceeded
- Template without New Lines : Error message is correctly shown when the result set size limit is exceeded

Other Terms

BI Java, BEx Java,  BEx, Web,  Planning, New Lines, SIZE_LIMIT_EXCEEDED, Error Message, Pivot Table, Matrix, newLinesCount

Reason and Prerequisites

Program error

Solution

Please deploy the recommeded patch

To solve the issue please apply the Support Package or patch as mentioned below. Due to the synchronized patch delivery no updates for other components both on ABAP and Java side are required.

Symptom

  • Tasks with BW target data store experience intermittent failures
  • Error message may differ depending on customer environment, product versions, and when the failure occurs.
  • Load balancing is configured in BW Target Data Store
  • Connection test in BW Target is successful
  • Sample Error messages from task failure:
SQL submitted to ODBC data source <dsod_agent_repo> resulted in error <[SAP][ODBC Driver][SQL Anywhere]Primary key for table 'AL_BW_REQUEST' is not unique: Primary key value ('''')>. The SQL submitted is <INSERT INTO AL_BW_REQUEST (REQUESTID,TYPE,VALUE,DF_NAME,CREATION_TIME) VALUES ( '', 4 , '123451',  'taskname', '2026-02-04 13:30:12')>
  • Sample error messages from BW InfoPackage Monitor in RSA1
Ex=1 Error when opening an RFC connection (CPIC-CA LL: 'ThSAPOCMINIT', communication rc: CM_ALLOCATE_ FAILURE_RETRY (cmRc=2)).

Environment

  • SAP Cloud Integration for data services
  • SAP BW

Reproducing the Issue

  1. Create a BW Target data store with load balancing enabled
  2. Create a task with a BW target
  3. Run the BW task

Cause

Load Balancing for BW Target is not supported in CI-ds. Unlike SAP Data Services where multiple RFC Servers can be configured, CI-ds has a single internal RFC Server. With BW Load Balancing enabled, BW may attempt to communicate to CI-ds from an application server that the CI-ds RFC Server is not expecting, resulting in a communication failure, timeout, or other type of task failure.

Resolution

In the BW Target Datastore, set load balancing to No, and set the Application Server to a single SAP Application Server.

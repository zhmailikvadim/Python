Symptom

The liveCache installer displays a warning that you are about to install an LCAPPS Package of LCAPPS release 10.0 over an existing package of LCAPPS release 7.0.

Other Terms

liveCache LCAPPS package Installer SCM APO sdbinst sdbsetup sdbupd

Reason and Prerequisites

liveCache (LCAPPS) release 7.0 for SCM 7.00 and SCM 7.01 (SCM without Enhancement Package (EHP) or EHP 1) is not compatible with LCAPPS release 10.0 for SCM 7.02 or later, although it usually uses the same MaxDB versions. An upgrade from release 7.0 to 10.0 is allowed as part of an SCM upgrade only.

As of liveCache build 100.57 the installer checks the existing LCAPPS release and advises against an upgrade from 7.0 to 10.0.

Solution

!! Take this warning very seriously !!

In merely all cases you should abort the installation by accepting the default answer "no", and failing to do so will render your SCM system unusable.

1. Wrong Package

In most cases this warning is displayed, because you downloaded the wrong liveCache package of LCAPPS release 10.0 which is not compatible with SCM 7.00 or SCM 7.01. To determine your SCM release:

  • Log on to the SCM system.
  • In the menu select System > Status and press the "Details" button beneath the product version.
  • Search for component SCMAPO.

If you encountered the installer warning, the release of SCMAPO most certainly is either 700 or 701. For these SCM releases it is mandatory to use LCAPPS release 7.0, instructions to find the corresponding packages are given in note 2037583. If with these instructions you do not find the package for your operating system or for the desired MaxDB version, read the next section for possible reasons.

2. End of Maintenance

SCM 7.00 and 7.01 are out of standard maintenance since 2026, and we do not provide newer liveCache builds of release 7.0 for these SCM releases any longer. The final liveCache version is build 70.53 based on MaxDB 7.9.11.10.

  • Due to technical problems the last version for Linux on x86 is build 70.51 based on MaxDB 7.9.10.12.
  • Windows is no longer supported as of Windows Server 2012, see note 2836399.

Any liveCache packages based on a MaxDB version later than 7.9.11.10 for sure are not compatible with SCM 7.00 or 7.01.

3. Migration to S/4 or SCM Upgrade?

The default strategy to keep your system up-to-date, supported and adopted to newer operating systems is to migrate to S/4 PPDS and other SAP products provided as replacement for SCM functionality.

If you consider it necessary to stay on SCM, you should upgrade to the most recent release 7.14. When doing so you should opt to migrate your liveCache from MaxDB to HANA, as standard maintenance for MaxDB based liveCaches ends in 2027, and no further updates will be provided as of then.

4. Upgrade SCM 7.02 or Later

You should answer "yes" to the installer warning only during the process of upgrading from SCM 7.00 or 7.01 to SCM 7.02 or later and want to stay on a MaxDB based liveCache. You can not perform the liveCache upgrade to release 10.0 in advance, as this release is not compatible to SCM 7.00 or SCM 7.01.

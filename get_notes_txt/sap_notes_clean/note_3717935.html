Symptom

Underlying concepts of queries and reporting based on InfoSets do not fit to new data model in S/4 HANA. The delivered queries have limited use and usability (for example, only Entry View, no GL View, based on logical databases). SAP Financials officially ends support of these InfoSets with the support packages referenced in this note.

This SAP Note deactivates the following InfoSets and the queries based on them:

/SAPQUERY/FIBR
/SAPQUERY/FIBR_AUDIT_01
/SAPQUERY/FIBR_AUDIT_02
/SAPQUERY/FIDD
/SAPQUERY/FIDD_AUDIT_901
/SAPQUERY/FIGL10
/SAPQUERY/FIGL20
/SAPQUERY/FIGL30
/SAPQUERY/FIGL40
/SAPQUERY/FIKD
/SAPQUERY/FIKL_AIS_01
/SAPQUERY/FIKL_AIS_02
/SAPQUERY/FISD

Other Terms

SQ00, SQ01, SQ02

Reason and Prerequisites

The reporting solutions are outdated.

Solution

After implementing this SAP Note, when you run a query based on one of these InfoSets, the system issues the new warning message FAGL_RSC_EN050:

"Functionality is not supported anymore (see SAP Note 3722523)".

If you want to continue using the query exactly as it is, you can switch off the warning message via transaction OBA5. From release S4CORE 110, you will need to implement BADI FAGL_AIS method ALLOW_QUERY and set the parameter CD_ALLOW according to your requirements.

Note that the changes to the InfoSets themselves are only delivered via the referenced Support Packages. The correction instructions of this note only implement some prerequisite changes.

Changing the message type of SFIN_FI050 or implementing the BADI method declares that you accept the limitations and want to continue using the queries without any support or future developments by SAP.

Symptom

You use a TU-based TM-EWM integration.

You configured Early Package Building on the TM documents. Also, on the Process Profile used by the Outbound Delivery Order Document Header, you did not set the flag "Qty Update". In EWM, before the initial LDAP Request is processed, you lowered the quantity of the product - the TM and ERP product items remain with the original quantity. When the LDAP Request is sent to EWM, it fails with the following error:

An error occured when trying to pack stock into planned shipping HU
091(/SCWM/POHU)

Other Terms

PSHU, qty, process code, lowering

Reason and Prerequisites

Program error

Solution

Implement the following Note, or apply the support package

After the Note implementation, the PSHU creation from the LDAP message will not consider the open quantity, it will generate the PSHUs for the given product items as sent by TM.

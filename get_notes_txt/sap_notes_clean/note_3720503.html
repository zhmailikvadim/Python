Symptom

In the process "Receipt of 55126 from Responsible DSO to SUP" (DE_B_MDC_55126_DSO2SUP_RECV), the system incorrectly updates the settlement date (/US4G/SETTL_DATE) for reason 04 in step 1600, even when no change occurred.

This issue is relevant for the market role: SUP.

Other Terms

MDC, DE_B_MDC_55126_DSO2SUP_RECV, /US4G/CL_UBD_DATA_POD, DETERMINE_SETTL_CHG_TYPE, /US4G/SETTL_DATE

Reason and Prerequisites

 The error occurred because of a code issue that caused the system to update the settlement date for reason 04 regardless of whether a change had occurred.

Solution

Implement the corrections of this SAP Note or import the relevant support package.

After implementing this SAP Note, the system updates the settlement date (/US4G/SETTL_DATE) for reason 04 only when a change occurs. You need to revert the settlement date shadow table to the status before the update, reverse the steps from step 1600 onward, and re-run the process. You can verify the fix by checking that the settlement date for reason 04 is updated only when a change occurs.

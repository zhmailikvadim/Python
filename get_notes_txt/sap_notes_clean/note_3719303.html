Symptom

Possibility if Candidate locale could be used as a field for mapping under Recruit to Hire Data Mapping. 

  1. Whether Candidate Locale is expected to be available in the Recruiting Data Model?

  2. If this is a known limitation or defect?

  3. If there is a supported workaround or alternative field to capture candidate locale?

  4. If configuration or provisioning is required to enable this field?

Environment

SAP SuccessFactors Recruiting Management 

Reproducing the Issue

  1. Navigate to the recruit-to-hire data mapping tool and try to map Candidate locale field from Candidate template 
  2. Observe that the field is not available for selection or mapping.

Cause

Candidate locale is a default field designed to determine the language used for email communication with candidates during the recruiting process. It is not explicitly defined or exposed in the Recruiting Data Model or templates.

.

Resolution

  1. Understand that the Candidate Locale field is a default field in the OData API Data Dictionary and is not explicitly defined in the Recruiting Data Model or templates.
  2. The field is not expected to be available in Recruiting Data Model as it is not explicitly defined to be exposed because its sole purpose is to identify the language of communication with candidates via email in recruiting module. 
  3. This is not a known limitation or defect. This is how the system is supposed to function and is designed that way. 
  4. If capturing candidate locale is required for mapping, create a custom field in the relevant template and manually input the locale data.
  5. Ensure the custom field is marked as reportable in Provisioning > Configure Reportable Custom Fields.
  6. Verify the configuration and mapping of the custom field in the recruit-to-hire data mapping tool.

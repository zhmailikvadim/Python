Symptom

You would like to recover a normal backup(without encryption) into target HANA(turn on backup encryption) . You would like to know if you need to turn off the 'backup encryption' on target HANA

Environment

  • SAP HANA Platform Edition 2.0

Cause

If backup encryption is disabled and a backup is taken afterwards, that backup is created without encryption. In this case, no encryption key is required during recovery. This applies only to backups that are created after encryption has been disabled. Any backups that were taken earlier while encryption was enabled remain encrypted and will still require the original encryption root key.

SAP documents this behavior clearly in the SAP HANA backup encryption documentation:
Backup Encryption

Resolution

Disable encryption backup on the target system is not necessary.

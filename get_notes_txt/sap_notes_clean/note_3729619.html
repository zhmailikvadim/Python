Symptom

When posting marketing documents, such as delivery notes, or inventory transaction documents, such as goods receipt, or goods issue, using the Data Interface (DI) API with very small decimal quantities and bin locations, the document is created successfully.

However, when reviewing the bin locations in the SAP Business One client afterwards, some bin locations remain visible and selectable for subsequent documents, even though the system shows the available quantity as 0 and they cannot be allocated because the allocated quantity is rounded to 0.

Image/data in this SAP Note is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.

Reproducing the issue

Prerequisites:

  • A warehouse with bin location management enabled.
  • An item stored in bin locations with available quantities (for the example used below):
    • Bin location AbsEntry 2 with quantity 1.
    • Bin location AbsEntry 3 with quantity 5.
  • Posting data through the DI API uses decimal quantities smaller than the allowed rounding precision set for quantities in the General Settings.

Scenario:

  1. Use the DI API to create a delivery note with very small decimal quantities assigned to bin locations (example in VB.NET):

oDoc = oCompany.GetBusinessObject(SAPbobsCOM.BoObjectTypes.oDeliveryNotes)

oDoc.CardCode = "C20000"
oDoc.DocDate = Now
oDoc.DocDueDate = Now

oDoc.Lines.ItemCode = "A00001"
oDoc.Lines.Quantity = 1
oDoc.Lines.WarehouseCode = "05"
oDoc.Lines.BinAllocations.BinAbsEntry = "2"
oDoc.Lines.BinAllocations.Quantity = "0.000086"
oDoc.Lines.BinAllocations.Add()
oDoc.Lines.BinAllocations.BinAbsEntry = "3"
oDoc.Lines.BinAllocations.Quantity = "0.000004"

oDoc.Add()

  1. Log in to the company using the SAP Business One client.
  2. Check the bin locations for the item.
  3. Attempt to allocate the same bin locations in a new document.

Expected behavior:

The system prevents posting of such documents or rounds the bin allocation quantities according to the decimal precision defined in General Settings.

Actual behavior:

The system allows posting of such documents via the DI API or Service Layer without rounding the bin allocation quantities.

As a result:

  • Bin locations remain visible and appear available for selection in the SAP Business One client.
  • However, they cannot be allocated in subsequent documents because the system rounds the available quantity to 0 based on the defined decimal precision.

Solution

SAP intends to provide a feature package or patch to solve the symptom that is described in this SAP Note. The section References below lists the related feature package or patch once they become available. Be aware that References can only be confirmed at feature package or patch release date. SAP delivers feature packages or patches only for selected releases at its own discretion, based on the business impact and the complexity of implementation.

Workaround: 

Adjust the bin quantities to valid values by posting inventory transactions (goods receipt or goods issue) using the DI API to normalize the quantities, and then post corrective documents to reach the intended values.

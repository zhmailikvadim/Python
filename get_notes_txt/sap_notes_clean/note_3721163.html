Symptom

OKTA‑issued JWT tokens are not supported by Cloud Connector during principal propagation, while tokens from XSUAA, Cloud Identity Service, Azure AD are successfully recognized.

Environment

  • SAP Cloud Platform, partner edition
  • SAP Cloud Connector
  • SAP Cloud Identity Service

Reproducing the Issue

  1. Login to Cloud Connector.
  2. Select a Subaccount.
  3. Navigate to "Cloud To On-Premise".
  4. Select "Principal Propagation".
  5. Click on the refresh button in the trust configuration section.

Resolution

  1. When OKTA is not defined as corporate idp, then okta token cannot be used for principal propagation.
    so configure OKTA as the corporate identity provider, so that its tokens can be used for principal propagation. 
  2. SAP BTP supports principal propagation by forwarding corporate IdP issued tokens via the Connectivity service to the on‑premise system.
  3. For more information, refer: Principal Propagation from SAP BTP Applications to On-Premise Systems Using Corporate IdP Tokens | SAP Help Portal

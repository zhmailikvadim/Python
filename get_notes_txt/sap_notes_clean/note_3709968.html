Symptom

SYSTEM_SHM_AREA_OBSOLETE error occurs when attempting to access a shared objects area that has already been released.

Environment

  • SAP NetWeaver
  • SAP NetWeaver Application Server for SAP S/4HANA
  • ABAP PLATFORM - Application Server ABAP

Reproducing the Issue

  1. Attempt to access the shared objects area CL_ICF_SHM_AREA.
  2. Encounter the error SYSTEM_SHM_AREA_OBSOLETE due to the area being released.

Cause

The issue is caused by a run of the ABAP Garbage Collectors during which the Shared Objects area is connected.

Resolution

Workaround: A tiny workaround would be to change the area properties of shared objects area CL_ICF_SHM_AREA in transaction SHMA. Just activate the "versioning" flag. Afterward, the invalidation of the area is required only once to get the new property active. This change will cause some higher memory consumption (usually twice the size of the shared objects area CL_ICF_SHM_AREA) and may also increase the runtime of the ATTACH_FOR_UPDATE call by a few milliseconds. As long as the shared objects area CL_ICF_SHM_AREA has a size smaller than 10MB this should not cause any performance issue.
Otherwise, the parameter ICF/SHARED_MEMORY_CACHE_OFF can be set to 3 to skip the cache until the system has a new kernel.

Solution: Upgrade the KERNEL as per the SAP Note 3512943 - Shared objects: Garbage Collector for ATTACH_FOR_UPDATE without versioning

Symptom

  • Relation set as mandatory is incorrectly displayed as optional in the UI.
  • The relation label shows the text "(optional)" next to it despite being configured as mandatory.

Environment

  • LeanIX Meta model configuration fields

Reproducing the Issue

  1. Navigate to the fact sheet in the LeanIX workspace.
  2. Locate the relation configured as mandatory.
  3. Observe the UI displaying "(optional)" next to the relation label.

Cause

  • The relation has a weight of 0 in the meta model configuration, which results in it being labeled as "(optional)" in the UI.

Resolution

  1. Access the LeanIX meta model configuration settings.
  2. Locate the affected relation and verify its weight value.
  3. Change the weight of the relation from 0 to 1.
  4. Save the configuration changes.
  5. Refresh the fact sheet UI to confirm the "(optional)" label is removed from the relation.

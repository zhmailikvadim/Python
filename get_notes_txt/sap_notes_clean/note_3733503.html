Symptom

  • The purge job for product master data ends with an error and the product cannot be deleted.
  • In the job log, error messages similar to:
    [Business message] Error while deleting master data <Master Data Type_ID>
    [Technical message] column ambiguously defined: <Attribute_ID>

Environment

SAP Integrated Business Planning (SCM-IBP)

Reproducing the Issue

  1. Schedule the Purge Master Data job with or without the Preview option.
  2. The job fails with the "column ambiguously defined" error.

Cause

Before deleting data from the selected master data type, the job first checks for any dependent master data and deletes that data beforehand. During this process, validation fails if a dependent master data type has multiple assigned attributes referencing the same attribute ID, triggering the error. This creates ambiguity during deletion and ultimately results in a database error.

Resolution

  1. Review the configuration of the dependent master data type and identify any duplicate assignments referencing <Attribute_ID>.
  2. To resolve the duplication:
     - If the dependent master data type is not required, delete it.
     - Alternatively, modify one of the assigned attributes so that it no longer references <Attribute_ID>, thereby removing the ambiguity.
  3. After making the changes, reactivate the relevant planning area to apply them. Then, re-run the Purge Master Data job and verify that it completes successfully.

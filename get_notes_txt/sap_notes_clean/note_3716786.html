Symptom

Using Replication Flow for ODP DataSource with BW Context from On-Premise to SAP Datasphere, the Replication flow report showing the error message: 

-No new delta data available.
-DATA_NOT_READY

AND/OR

The process log of Replication flow it is showing message "DATA_AVAILABLE_NOT_READY_FOR_TRANSFER " and still the load is in progress.

Environment

  • SAP Datasphere
  • S/4HANA On-Premise
  • ABAP-based On-Premise system with DMIS

Reproducing the Issue

  1. The replication flow running for an extended period without completing the initial load.
  2. Check the process log for messages such as "DATA_AVAILABLE_NOT_READY_FOR_TRANSFER".

Cause

  • The source system cannot serve the ODQ queue fast enough, resulting in delays.
  • Lack of periodic ODP daemon jobs required for processing ODQ queues.
  • Failed composite requests in ODQMON impacting data transfer.

Resolution

  1. In the source system, verify that all required ODP daemon background jobs are scheduled and running periodically, as described in SAP Note 3561739 (check in transaction SM37).

  2. Ensure the latest available corrections applied to the source system with the report CNV_NOTE_ANALYZER_DI from the SAP Note 3016862

  3. Ensure that sufficient free background work processes are available in the source system.

  4. Check ODQMON and delete or close any failed or invalid composite requests to avoid blocking the ODQ queues.

  5. Monitor the replication flow again to confirm that data transfer is progressing, If the initial load does not recover after the above checks, reset and restart the replication flow only as a last resort.

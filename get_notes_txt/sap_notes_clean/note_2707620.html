Symptom

During the SAP HANA DB File Naming convention as below, there is no specification about the LENGTH OF THE BACKUP FILE NAME as in the help.sap.com to find.

/hana/backup/SYSTEMDB/
CDB2018-09-07_14-28_databackup_0_1 

PREFIX - CDB ( Complete data backup)
$DATETIME ( DATE TIME)
$SUFFIX ( Default value as it come _databackup_ *)

Environment

SAP HANA DB 1.0
SAP HANA DB 2.0

Resolution

The length depends on the file system. Normally, the name is limited to 512 characters, including the entire path. If the value is exceeded, the SQL command is rejected with an error message.

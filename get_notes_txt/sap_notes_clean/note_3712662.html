Symptom

Starting with 2026 QRC1 (2026.02) the System > Monitor > Trace feature is removed.

Tenant logs that are useful for troubleshooting Data Acquisition workflows are still available under System > Overview.

This note explains how to locate and view those logs using the Alerts tab and Data Analyzer.

Reason and Prerequisites

You have access to System > Overview.

You can open Data Analyzer from System > Overview.

Users who receive an on-screen error toast during Data Acquisition and need more detailed logs.

Solution

View Logs in System Overview 

  1. Navigate to System > Overview
  2. Click the Alerts tab
  3. Click Client
  4. In the "Top 10 Messages, Components and User by Errors" chart, click inside the chart area (blank space) to reveal the three-dots (More Actions) menu.
  5. Choose More Actions > Open Data Analyzer...
    • Select either Open in Current Tab or Open in New Tab
  6. In Data Analyzer, add or enable the following columns to approximate System Monitor Trace with columns Timestamp | Trace Level | User Name | Component | Message | Correlation ID | Detail, add additional columns
    • Add Timestamp | Level | User ID | DETAIL columns
  7. Sort log messages by date.
    • Change Timestamp > Sort Options > Newest to Oldest.
      • Note, not Z-A
  8. Remove or adjust the default Level = ERROR filter to include INFO and DEBUG messages if needed.
  9. System Overview does not support substring filtering exactly like System Monitor Trace.  Use Filter by Member to enter a substring then select matching members from the results to include them in the analysis.

Symptom

  • In the ECC system, the "document type" is a mandatory field in T-code ABIF. 
  • After upgrading from ECC to the S/4HANA system, "document type" is not a mandatory field in T-code ABIFL. If the document type is not maintained in the corresponding transaction type, and the "document type" is not entered during ABIFL input screen, an error 'F5814 document type not defined' will occur.

Environment

  • Financial Accounting (FI)
  • Asset Accounting (FI-AA)
  • SAP S/4HANA Finance
  • SAP S/4HANA

Reproducing the Issue

  1. Run T-code ABIFL.
  2. Input the parameters in mandatory field
  3. Perform simulation.
  4. An error 'F5814 document type not defined‘ is displayed. 

Cause

This is a normal system behavior.

Resolution

With S/4HANA in T-code ABIFL, the document type is now derived from the document type assigned to the transaction type.

Therefore, the input field cannot be mandatory, as doing so would prevent the derivation process.

If derivation is not desired, you can still specify the desired document type in the input screen in T-code ABIFL. 

Keywords

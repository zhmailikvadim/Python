Symptom

CSO confirmed that any employee or employer pension contributions made under Automatic Enrolment (My Future Fund) must be reported in the existing EHECS questions that capture the inclusion or exclusion of employee and employer pension contributions. This applies to wage type /309 for employee contributions and wage type /310 for employer contributions.

Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.

Environment

  • Human Capital Management (HCM)
  • SAP ERP Core Component
  • SAP ERP
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP SuccessFactors Employee Central integration to SAP Business
  • SAP SuccessFactors Employee Central Payroll

Resolution

  • For employee pension, the CSO EHECS requires the Total Wages & Salaries figure to be reported before pension. This would be same for My Future Fund EE, wage type /309. Therefore, /309 has no impact on the CSO EHECS reporting and no additional customising is required by our Irish payroll users.
  • For employer pension, My Future Fund ER Contributions need to be included in the employer pension value in the CSO EHECS report. As a result, the mapping for employer pension must be updated accordingly:
    • Table V_T596J_IE
    • Sub-application EHWP
    • Cumulate /310 into ENJO

  • For users of the SAP standard CSO EHECS report, this configuration should be completed before running the first payroll of 2026.
  • For organisations using custom-developed CSO reporting, the solution should be updated to ensure My Future Fund employer contributions are included in the employer pension reporting field.
 
This configuration is part of the When Paid solution. If the ENJO component is not yet active in a system, it can only be activated from the current payroll period onward and cannot be applied retroactively.

Symptom

When creating an outbound service in ADT or in the Communication Systems Fiori app, the following error message appears:

  • Communication scenario: XXX outbound access XXX does not support multiple CA instances

As a result, it is not possible to create or activate the outbound service.

Additionally, the system may display the error:

  • Communication Scenario XX contains errors and therefore cannot be saved

Environment

  • SAP S/4HANA Private Cloud
  • SAP BTP Cloud

Cause

When an outbound service is added to a communication scenario, the scenario must support only a single Communication Arrangement (CA) in the system.

If the “Allowed Instances” setting in the Communication Scenarios app is configured as “One instance per scenario & communication system”, the system allows multiple Communication Arrangements for the same communication scenario.

However, outbound services do not support multiple CA instances. As a result, errors such as “does not support multiple CA instances” may occur when attempting to create or activate additional Communication Arrangements.

Resolution

Adjust the configuration of the communication scenario in the Communication Scenarios app:

  1. Open the communication scenario.
  2. Navigate to General / Overview → Allowed Instances.
  3. Set the option to “One instance per client.”

This setting ensures that only one active Communication Arrangement can exist for the communication scenario across the entire system, which is the supported configuration for outbound services.

Symptom

While updating SAPUI5 library to version 1.120.41, the report "/UI5/MIME_CLEANUP" was executed as part of manual steps outlined in SAP Note 3704532 to delete obsolete content. 

Version 1.120: Failed during execution with multiple error messages, such as:

"Library 'com.sap.ui5.viz' has an error 'Unable to lock object R3TR SMIM F34EEC19'".

"Library 'com.sap.ui5.ux3' has an error 'Unable to lock object R3TR SMIM F34EEC19'".

"Library 'com.sap.ui5.makit' has an error 'Unable to lock object R3TR SMIM F34EEC'".

The report did not execute successfully for version 1.120.

Environment

SAP S/4HANA

Reproducing the Issue

  1. Update SAPUI5 library to version 1.120.41.
  2. Perform TCI steps as per SAP Note 3704532.
  3. Execute the report "/UI5/MIME_CLEANUP" to delete obsolete content.
  4. Observe the following:
  • Successful deletion for version 1.114 with a fresh transport.
  • Failure for version 1.120 with error messages indicating locked objects.

Cause

  • Some MIME objects were locked in an older transport request, preventing successful execution of the report for version 1.120.

Resolution

  1. Identify the locked MIME objects using the provided transport request IDs.
  2. Release the older transport request where the MIME objects are locked.
  3. Re-execute the report "/UI5/MIME_CLEANUP" for version 1.120.
  4. Refer to the following SAP Knowledge Base Articles for additional guidance:
  • [Object directory entry R3TR XXXXXXXX locked for request/task XXXXX](https://me.sap.com/notes/0002716764)
  • [How to find modifiable transports for specific objects](https://me.sap.com/notes/0002560303)
  • [SE03->Unlock Objects(Expert Tool)](https://me.sap.com/notes/0002284238)

Symptom

For an insured person at the VBL, the insurance number has changed due to a correction of the reported gender. Notifications (annual reports, deregistrations) that were reported to the VBL before the gender correction must now be reversed (for other reasons). The notification creator (report RPCZVMD2) terminates the processing of the personnel case with the following warnings/error messages:

  • Insurance number <insurance number> does not match gender <gender>,
  • The insurance number <insurance number> is not correct.
  • The generated SP notifications of <personnel number> are inconsistent
  • Personnel number is rejected (see previous error/warning messages).

Reason and Prerequisites

This symptom was actually already solved with SAP Note 2838973 - SP reporting: Reversal of notifications after correction of reported gender for VBL. However, this SAP Note does not apply if the new insurance number is entered directly in the infotype VBL/SPI Data (0051) during a gender correction (for example, because the gender correction was reported manually) and not only the incomplete one as the process actually requires.

Solution

Implement the attached correction instructions or import the specified support package. After you import the HR Support Package or implement the correction instructions, you do not have to perform any further follow-on activities. The programming error is corrected.

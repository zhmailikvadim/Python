Symptom

  • The query result in the application does not align with the filter input.
  • No matter the result returns 'No data available', or expected Grid list result, it keeps the same result as the result in last 'Go' search, even after the filter input has been changed. 
  • Users need to exit the application and restart it to retrieve the expected results.

Environment

  • FPM Analytical GUIBB

Cause

  • The issue occurs due to a mistaken buffer value which is used instead of the variable value. As opposed to the buffer the later are assigned correctly.
  • Queries running the exits in a dependency driven mode in general are not affected by the problem.

Resolution

  • Make sure the Support Package is >= the required level for each SAP BW package on BW or AS ABAP system by applying SAP Note SAP Note 3366528 - Customer exit variables delivering wrong hierarchy

Symptom

The following error message occurs in FINSC_LEDGER transaction:

Position for global crcy 10 should be empty (ledger '&&'/'&&' company code '&&&&')' 

Environment

  • Financial Accounting
  • SAP S/4HANA
  • SAP S/4HANA Finance

Reproducing the Issue

  1. Access transaction code FINSC_LEDGER in SAP S/4HANA.
  2. Attempt to configure currency type 10 for Ledger 0L and Company Code 7000.
  3. Observe the error message: FINS_ACDOC_CUST251.

Cause

Currency type 10 is customized as FI integrated currency for Company Code in question and maintained as POSNR in FINS_CUST_LEDGER. As per S/4HANA design, currency types 10 must have blank positions.

Resolution

  1. Verify the configuration for currency type 10 in transaction FINSC_LEDGER.
  2. Ensure that currency type 10 is not maintained in column '2nd FI currency' or '3rd FI currency' in view 'Company Code for the Ledger'.

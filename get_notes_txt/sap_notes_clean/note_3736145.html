Symptom

  • In My Timesheet (S/4HANA / Version 4), users reorder assignments within an assignment group using the up/down arrows.
  • After saving and reopening the assignment group, the assignment sequence sometimes changes unexpectedly.
  • When importing the assignment group into the timesheet, the line order sometimes differs from the last saved order.
  • The behavior is intermittent; sometimes the order remains correct, sometimes it does not.

Environment

  • SAP S/4HANA

Reproducing the Issue

  1. Open the My Timesheets (S/4HANA / Version 4) application
  2. Click on the 'Assignments' tab
  3. Create a number of assignments
  4. Click on the 'Assignment Groups' tab
  5. Create an assignment group and add half of the assignments in the desired order
  6. Change the order using the up and down arrows and save
  7. Go to the Time Entry screen and pull the assignment group in (order likely still matches the last saved order)
  8. Go back to the assignment group and add the remaining assignments and save
  9. Update the assignment group to change the order again
  10. Go to the Time Entry screen and pull the assignment group in (order may differ from the last saved order)

Cause

  • When the number of assignments exceeded 9, the sorting logic failed

Resolution

  1. Implement SAP Note 3724238 in the front-end/gateway system
  2. Clean up the cache using the steps provided in SAP Note 2319491 - How to clean up the cache after applying changes that affect SAP Fiori apps

See Also

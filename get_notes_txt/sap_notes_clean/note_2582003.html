Symptom

You have deleted the master data and you want to load the InfoObject again. The loading of InfoObject fails with error R7105 'The object name is not allowed to be empty'

Other Terms

R7105, Delete master data, RSNDI_MD_DELETE

Reason and Prerequisites

Code needs to be adapted to changes to CHAVL after note 2098968

Solution

In the Start Routine of transformation TRCS 0RT_WRF_TDLOCAS_ATTR_DEL -> IOBJ 0RT_TDLOCAS,

in the data declaration

   CONSTANTS: lc_iobjnm TYPE rsiobjnm VALUE '0RT_TDLOCAS',
             lc_tabval(16) VALUE '/BI0/PPLANT'.

Add constants, so the code looks as following

 CONSTANTS: lc_iobjnm TYPE rsiobjnm VALUE '0RT_TDLOCAS',
             lc_asort TYPE rsiobjnm VALUE '0RT_ASORT',
             lc_date TYPE rsiobjnm VALUE '0RT_DATEF',
             lc_tabval(16) VALUE '/BI0/PPLANT'.

In the data declaration

 DATA: lv_key TYPE rsndi_s_chavl-value.

Add variable so the code looks as following

   DATA: lv_key TYPE rsndi_s_chavl-value,
        lv_record_no    TYPE int4 VALUE 1.

inside the Loop of

   LOOP AT DATA_PACKAGE INTO ls_DATA_PACKAGE.
    ls_del_data-value    = ls_DATA_PACKAGE-rt_asort.
    ls_del_data-value+10 = ls_DATA_PACKAGE-rt_datef.
    ls_del_data-value+18 = ls_DATA_PACKAGE-rt_tdlocas.
    APPEND ls_del_data TO lt_del_data.
  ENDLOOP.

 

Add the statements so the code looks as following

 LOOP AT DATA_PACKAGE INTO ls_DATA_PACKAGE.
    ls_del_data-record_no = lv_record_no.
    ls_del_data-iobjnm    = lc_asort.
    ls_del_data-value    = ls_DATA_PACKAGE-rt_asort.
    APPEND ls_del_data TO lt_del_data.

    ls_del_data-record_no = lv_record_no.
    ls_del_data-iobjnm    = lc_iobjnm.
    ls_del_data-value = ls_DATA_PACKAGE-rt_tdlocas.
    APPEND ls_del_data TO lt_del_data.

    ls_del_data-record_no = lv_record_no.
    ls_del_data-iobjnm    = lc_date.
    ls_del_data-value = ls_DATA_PACKAGE-rt_datef.
    APPEND ls_del_data TO lt_del_data.

    lv_record_no = lv_record_no + 1.
  ENDLOOP.

Alternately, upgrade to the latest support package and install the Transformation again.

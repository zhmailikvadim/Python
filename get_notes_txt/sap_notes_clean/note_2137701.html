Symptom

  • A wrong ALV-List is displayed.
  • The layout appears not to work correctly or as you wish based on your sort variant or your selection parameters when executing an asset report. 
  • You observe that the output in the layout does not show the result that you wish and expect.
  • When displaying a report output you believe that the ALV-List layout is "incorrect" or faulty.
  • For example, a new sort variant may show unexpected and inexplicable results.
  • This system behaviour can be observed in any asset report where a specific layout is expected but is not reflected in the report output.

Environment

  • Financial Accounting (FI)
  • Asset Accounting (FI-AA)
  • SAP R/3 4.6C
  • SAP R/3 Enterprise 4.7
  • SAP ERP Central Component
  • SAP ERP
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA

Reproducing the Issue

  1. Go to any asset accounting report.
  2. Enter all your parameters including any sort variant etc. in the selection screen of the report.
  3. Execute the report. 

Cause

Firstly take note that there are two options for Layout-Settings, that a user can avail from.

In List Viewer (ALV) we have the following two Layout-Settings: 

  1. General accessable ALV-Layouts (for all Users): these ALV-Layouts always start with prefix „/“. From this List you can define a "Default" ALV-Layout. In that case this defaulted ALV-Layout will alwasy be pulled for the ALV-List; the exception to the rule is, if a Display Variant (field name VARIANT) is explicitly entered in the selection screen of a report, such as for example in report RABEST_ALV01.
  2. ALV-Layout only for a specific User: here, every User can create their own ALV-Layout. These ALV-Layouts start without prefix „/“. The user then can from their own ALV-Layouts define a "Default" ALV-Layout. However, in this case, the defaulted ALV-Layout is valid for the authorized User only. Other Users cannot see this self-created List of ALV-Layouts.

A note on the side: a user-specific default ALV-Layout will always take precedence over the general default ALV-Layout.

The root cause of the problem in this example is the default setting for a general accessable ALV-Layout: you find that the indicator "Initial Layout Flag" (field "DS") is set for a layout and as a result the system ignores any settings in a new sort version.

Always remember: if a default entry layout is defined in the SAP List Viewer (general or user-specific), then the system will ALWAYS select the default entry layout in the ALV-List regardless what sort variant or selection parameters you have selected in the selection entry screen in an asset report.

Resolution

  1. Execute the asset report with your selection parameters, which could be a new sort variant for example. If the layout is not as you expect, then....
  2. Check the settings for your ALV List (List Viewer) in the report after you execute it.
  3. Go to: Menu path:
    -> Settings -> Layout -> Layout managment

     4.   If you have set the indicator "initial layout flag" for a layout then you must remove the indicator in field "DS" in the list viewer. For further information check the F1 Help for this indicator.

Remember: as long as an "intial layout flag" (= default setting) for any of the standard or your own layouts is set, the system will ignore any parameter settings in the selection screen, including a specific layout in a new sort variant for example. Only if no default ALV-Layout is selected will the parameters in the initial screen of the report be pulled and be displayed accurately.

Always ensure that no intial default layout exists to dictate an unexpected layout. Only exception to this rule is where you have explicitly entered a display variant in the selection entry screen of the report.

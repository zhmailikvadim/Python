Symptom

Task name starting with an adjective (or any non-verb) is flagged in Convention check rule 3.1

Resolution

To resolve the issue where task names beginning with an adjective (or any non‑verb) are incorrectly flagged under Convention Check Rule 3.1, the system will apply the Verb–Object validation using the following criteria:

  1. Primary Rule – Infinitive or OANC‑Verified Verb
    A task label will be considered compliant if its first word is:

    • An English infinitive verb (“to <verb>”), or
    • A word that appears at least once as a verb in the OANC (Open American National Corpus).

  2. Extended Valid Cases – Non‑Verb First Words Allowed
    Even when the first word is not a verb, the following patterns will also be treated as valid Verb–Object task names:

    • Labels beginning with “re” + verb, e.g., “re think”, “re plan”.
    • Labels beginning with “in case of …”, e.g., “in case of emergency”.
    • Labels following the pattern “<anything>, if …”, e.g., “Report, if delayed”.

  3. Outcome
    These rules ensure accurate identification of valid Verb–Object constructions, reduce false positives, and maintain consistency across task naming conventions.

Note: If your task name complies with all the criteria above but the system still does not behave as expected, please raise a support case, and we will investigate the issue further.

Symptom

  • Successfully replicated a table using SLT, which redirects to a view, without encountering any errors.
  • Retrieved data from the source system's table using T-code SE16, and found discrepancies compared to the data in the target HANA table.

Environment

SAP Landscape Transformation Replication Server (SLT)

S/4 HANA

Reproducing the Issue

  1. In T code LTRC, replicate a table which redirects to a view.
  2. Run SE16 on source system to check the row number or data of the table, compare it with that of the target HANA DB, will find the discrepancies.

Cause

When using SE16, if a table has a redirect view, SE16 will display the data from the redirect view instead of the data from the actual table. If there is no redirect view, it will return the data of the table itself.

Resolution

To verify the actual data of a table that redirects to a view, it is necessary to check the data at the database level. This can be done by running T code such as DB02 > SQL Editor or direct database queries to ensure accuracy and consistency.

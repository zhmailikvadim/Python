Symptom

When you use the Manage Basic Compliance Data - Unpackaged Products app or the Migration Cockpit, you can assign packaged products to an unpackaged product. The system then uses Product Compliance Event Processing (PCEP) to generate a compliance assessment for each packaged product.

If you have assigned a large number of products to an unpackaged product, the system started to create a huge amount of Event - Task - Linkages which could result into an out-of-memory runtime error, and stops processing. Events are set to status failed.

When failed Product Compliance Events are reprocessed again, the system might dump with an our of memory error.

Other Terms

PCEP, Migration Cockpit, Creation of Compliance Assessments

Reason and Prerequisites

Reason: The Product Compliance Event Processing (PCEP) has memory issues when reprocessing failed events.

Solution

You have implemented the latest version of note 3714804.

After implementing this SAP Note, the system optimizes the reprocessing of compliance assignments between PECP events and tasks. This optimization prevents the out-of-memory error.

Symptom

  • An issue occurs when using the Contract Creation API "/ACCGO/CCAK_CRT_WITH_DATA_API" to create a priced contract where the line item quantity is less than the absolute tolerance quantity. The system blocks this behavior.
  • The issue impacts the migration of contracts where one or more line items have an absolute tolerance quantity higher than the available open line item quantity for migration.

Environment

  • S/4 HANA
  • SAP Agricultural Contract Management

Reproducing the Issue

  1. Create a variant to the contract create API '/ACCGO/CCAK_CRT_WITH_DATA_API'.
  2. Keep the line item quantity less than the absolute tolerance quantity.
  3. Error raised 'Absolute tol. value cannot exceed CI qty &1 in Tolerance tab for item &2'

Cause

The API is validating the input data as per API design.

Resolution

To migrate the legacy contract:

  1. Create the contract using the actual quantity as in legacy system.
  2. Cancel the consumed quantity from the created contract, as this action will not impact business data.

This process should migrate the open quantity from legacy system to S4 without impacting business data.

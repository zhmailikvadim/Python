Symptom

  • A new case is created for scope items which were already added or activated.
  • How to check whether a scope item is activated into the target S/4HANA Cloud System in SAP Central Business Configuration

Environment

  • SAP Central Business Configuration
  • SAP S/4HANA Cloud Public Edition

Cause

Adding scope items in CBC does not mean the scope items will be activated into target system, customer should complete Deploy Scope and Organizational Structure to trigger activation.

Resolution

Go to CBC Define Scope App. Choose My Selected Scope. Search the scope item.

  • If the scope item can not be found in My Selected Scope, that means it has not been activated yet.
  • If the scope item can be found in My Selected Scope, follow KBA 3732591 to check the CBC workspace phase. 
    • If your CBC workspace is in "Product-Specific Configuration" phase, that means all scope items in My Selected Scope were activated into target S/4HANA Cloud system which is assigned to the CBC workspace.
    • If your CBC workspace is in "Scope and Organizational Structure" phase, follow KBA 3729632 to find whether there are newly added scope items. If yes, please complete Deploy Scope and Organizational Structure to trigger activation.

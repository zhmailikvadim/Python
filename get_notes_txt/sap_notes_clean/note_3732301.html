Symptom

If field selection key AKTV is set to display for document type then while creating the PO with ME21N and Manage purchase order app, system should allow to change the document type

Environment

  • SAP Fiori
  • SAP S/4HANA all versions
  • SAP S/4HANA Cloud Private Edition

Reproducing the Issue

Maintained the following Configuration:

Field Selection for PO -> PO Document type field set as Display for AKTV [Change]

  1.  In GUI - ME21N - able to select the required Document type
  2. In Fiori - Manage Purchase Order app- the Document type field is in Display mode with default document type NB preselected and not able to select required document type.

Cause

App uses the field selection key AKTV (change) instead of the AKTH (create) when creating a new purchase order 

Resolution

This is standard system design

Keywords

Manage purchase order app, Fiori, ME21N, PO, AKTV, AKTH, Document type, editable, field selection key, F0842A

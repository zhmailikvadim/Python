Symptom

During batch job execution in the program, the batch job log displays error CO 889 "Problem with Parallel Task (RC/Size/Text): 3 1". Although the batch job status is marked as finished, the program does not deliver expected results, leaving materials in T-Code COGI unprocessed.

Environment

  • SAP ERP Production Planning (PP)
  • SAP ERP All versions
  • SAP ERP Central Component All versions
  • SAP R/3 All versions
  • SAP R/3 Enterprise All versions
  • SAP S/4HANA All versions
  • SAP S/4HANA Cloud Private Edition All versions
  • SAP enhancement package for SAP ERP All versions
  • SAP enhancement package for SAP ERP, version for SAP HANA All versions

Cause

The root cause is related to incorrect confirmation parallelization settings in T-Code OPKB, including insufficient configuration of work processes and missing Logon/server group entries.

Resolution

Adjust the confirmation parallelization settings in T-Code OPKB:

  • Increase the "Number of Work Processes That Can Be Used".
  • Increase the "portion size for parallel processes".
  • Maintain a Logon/server group for confirmation parallelization.

If the parallelization settings were configured unintentionally, delete the record in transaction T-Code OPKB to prevent the issue.

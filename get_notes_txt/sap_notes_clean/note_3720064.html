Symptom

When attempting to use "Import Remote Workspace" in SAP HANA Studio, the operation fails with the following error: "Cannot cast java.lang.String to org.eclipse.core.runtime.IPath."

This issue does not occur in SAP HANA Studio version 2.3.85 or lower. 

 "Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental."

Environment

SAP HANA Studio 2.3.86

Reproducing the Issue

  1. Select the "Default" workspace, right-click, and choose "Import Remote Workspace."
  2. Click "Browse" and choose an appropriate directory.
  3. Error: "Cannot cast java.lang.String to org.eclipse.core.runtime.IPath." occurs.

Cause

This is a known issue in SAP HANA Studio.

Resolution

Use SAP HANA Studio version 2.3.85, as this version have been tested and work as expected.
Or as a workaround, follow below steps:

  1. Log off from the HANA system.


  2. Click "Create Repository Workspace".


  3. Choose the system and input proper path for the workspace.


  4.  The workspace is created


  5. Log on to the HANA system


  6.  The packages display correctly.

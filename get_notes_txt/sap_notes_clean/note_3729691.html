Symptom

Error message: "PE error 34 023"/"PE error 34.023"/"PE error 34,023" when executing the planning sequence or planning function in HANA/PAK mode, but it works fine in ABAP mode.

Environment

  • SAP BW/4HANA
  • SAP NetWeaver

Reproducing the Issue

  1. Go to transaction RSPLAN.
  2. Select Planning sequence and execute step-by-step in HANA/PAK mode. The error occurs.
  3. When executing it in trace mode(ABAP runtime), no error.

Cause

  1. To get the details of the error, set the user parameter as below in transaction SU3, and then replicate the issue again.

    RSPLS_HDB_SUPPORT
    HDB_ON
    RS_DEBUGLEVEL
    5
    RSPLS_HDB_CHECK
    HDB_IMMEDIATE
    RSPLS_HDB_PE_TRACE
    Y

  2. After getting the new error, check the HANA alert file in transaction ST04 or DBACOCKPIT. The file is located in under left side of Diagnostics -> Diagnosis Files. The file name is indexserver_*alert.trc.
  3. Open this file, and confirm whether you could see the error as below:
    ....IndexID: BWP::SAPBW4:0BW:BIA:INFOOBJECT_F4 (t -1), QueryMediator failed executing query, reason: olap: merging multi value dicts is not implemented, please make sure the following join cardinality is modeled correctly (did you use 1:1 when the data really is 1:N or N:M?);BwPopJoin1Inwards ....
  4. If there is similar as above, it means there is something issue from the INFOOBJECT. Most of the issue might be caused by system upgrade.

Resolution

  1. Run the program RSDDB_LOGINDEX_CREATE for the characteristic INFOOBJECT. 
  2. Verify the resolution by executing the planning sequence in SAP Hana mode to confirm the error is resolved.

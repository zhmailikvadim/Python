Symptom

In the device change scenario in processes Setup / Replacement of Devices (SUP View) (DE_B_IM_DEV_CHANGE_SUP) and Setup / Replacement of Devices (DSO View)" (DE_B_IM_DEV_CHANGE_DSO), the fallback implementation of BAdI /US4G/BADI_IM_DEV_MGR_PASSIVE doesn't consider the time dependency of table /US4G/DCX_ATTR in method DET_DEV_CAT_METER. As a result, incorrect device category attributes are determined.

This issue is relevant for the market roles SUP and DSO.

Other Terms

IM4G, Device Change, BAdI, /US4G/BADI_IM_DEV_MGR_PASSIVE, /US4G/CL_BADI_IM_DEV_MGR_PASV, DET_DEV_CAT_METER, /US4G/DCX_ATTR, device category attributes, time dependency

Reason and Prerequisites

The error occurs because the fallback implementation is missing the logic to consider the time dependency of table /US4G/DCX_ATTR.

Solution

Implement the corrections from this SAP Note or import the next Support Package.

After you implement this SAP Note, the system considers the time dependency of table /US4G/DCX_ATTR when determining device category attributes. You can verify the fix by checking that the system determines the correct device category attributes.

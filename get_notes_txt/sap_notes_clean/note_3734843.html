Symptom

Configuration transport center is not working when adding objects to bundle, the error shown is blank:

"Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental."

Environment

SAP SuccessFactors HCM Suite

Reproducing the Issue

  1. Create the New Bundle Management.
  2. Go to Manage data.
  3. Search for the object
  4. Click on Take Action > Add to transport Bundle.
  5. A blank error pops up

Cause

There are required permissions missing for the user facing the error.

Resolution

Please ensure the user has all the following permissions: 

  • Access to view and download configurations
  • Configure Object Definitions both in the source and the target tenant
  • Grant either Admin access to MDF OData API permission or the View, and Edit Miscellaneous Permissions for the following MDF objects:
    • ExportConfigRequest
    • TransportConfigRequest
    • ExportConfigNode
    • ImportBundle
    • ImportConfigRequest
    • ImportLogEntry
    • BundleDefinitionConfigNode
    • ConfigBundleDefinition

Symptom

In the InventoryTransactionDocumentsFact view, the FinancialPeriod data is using CreateDate of the transactions to determine the attributes.

Reproducing the issue

Scenario:

You are using the InventoryTransactionDocumentsFact view.

Expected behavior:

The FinancialPeriod data is using DocDate of the transactions to determine the attributes.

Actual behavior:

The FinancialPeriod data is using CreateDate of the transactions to determine the attributes.

Solution

SAP intends to provide a feature package or feature packages to solve the problem described. The section References below lists the relevant feature packages once they become available, and the Related Info content of relevant feature packages available in the SAP Support Portal, show this SAP Note. Be aware that these references can only be set at feature package release date. SAP delivers feature packages only for selected releases at its own discretion, based on the business impact and the complexity of the implementation.

Workaround:

1. In the SAP HANA Studio, navigate to sap\<companydb>\stock\Calculation Views\InventoryTransactionDocumentsFact, and double-click the entry.

2. In the Scenario panel, which displays a tree structure of the calculation view, click the Join_6 node.

3. Go to the Details panel, which displays a list of table fields of the OILM table, find the DocDate field and click the grey icon before it. The icon then becomes orange.

4. Go back to the Scenario panel, click the Projection_5 node. Then in the Details panel, which displays the fields exposed in Join_6, click the icon before the DocDate field as in step 3.

5. Go to the Output panel, double-click Expression under the Filters folder. It will open a dialog.

6. In Expression Editor, replace CreateDate with DocDate. After that, the formula will be "DocDate" >="F_RefDate" and "DocDate" <="T_RefDate". Click OK.

7. Click Save and Activate.

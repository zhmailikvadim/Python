Symptom

When performing multiple meter readings and saving them on the CRM UI, only one meter reading is saved. The meter reading documents are saved to the activity clipboard and are saved later when ending the session, which is why at the end of the session all meter readings are saved correctly.

The system falsely removes entries from a internal meter reading header table, which later is used to loop over the meter reading objects primed to be saved.

Other Terms

CL_IUICMREX_BSPWDCOMPONEN_IMPL, CREATE_NEW_MR, MREADHEADER, SAVE, MR_SAVED

Solution

Please implement the correction instructions or install the corresponding support package.

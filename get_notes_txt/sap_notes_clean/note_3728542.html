Symptom

  • A DELETE query executed in the Sybase IQ database runs for an extended duration and consumes a very high amount of TEMP DBSPACE.
  • TEMP DBSPACE utilization increases significantly, impacting other queries and scheduled reporting jobs.
  • The DELETE query involves two tables, with the following query structure:

```

DELETE FROM "owner"."tab_A" AS A

FROM "owner"."tab_B" AS B

WHERE A.COL1= B.COL1

AND A.COL2 = B.COL2;

```

Environment

SAP IQ 16.1

Cause

  • The root cause of the issue is insufficient join conditions between the tables "tab_A" and "tab_B".
  • The join key columns, "COL1" and "COL2", are not unique in both tables, leading to a many-to-many join scenario.
  • This results in a large cartesian product, generating excessive rows during the join operation and causing high TEMP DBSPACE consumption and prolonged execution time.

Resolution

  1. Contact the application team responsible for creating the DELETE query to review and refine the join conditions.
  2. Ensure that the join key columns "COL1" and "COL2" are unique in both tables to avoid many-to-many joins.
  3. Consider adding additional filtering criteria to the query to reduce the number of rows involved in the join operation.
  4. If applicable, optimize the query execution by utilizing appropriate indexing strategies on the join key columns.
  5. Monitor TEMP DBSPACE usage during query execution and adjust database configurations if necessary.
  6. Refer to the application logs and query plans for further analysis and optimization.

Symptom

  • When I search for rates using a specific currency in Admin > Rates, I see fewer rate card records than I expect.
  • For the records that appear, the minimum and maximum rate values display as 0.00 even though valid rate ranges appear during Job Posting creation.

Cause

You see fewer Rate Codes and 0.00 rate values in Admin > Rates when your organization uses a role‑based rate card. A role‑based rate card varies only by job role and does not change by supplier, site, or worker pay type.
Because the templates can store the minimum and maximum bill rate values directly, you reuse the same Rate Code across many templates. You do not need to create a separate Rate Code for each role or template.

When you reuse a Rate Code:

  • Only a small number of Rate Codes exist for the currency
  • Those Rate Codes do not contain template‑level rate ranges
  • Admin > Rates displays fewer results
  • The Rate Codes that appear show 0.00 for minimum and maximum values
You create separate Rate Codes only when rates vary across multiple dimensions, such as supplier, site, or worker pay type in addition to job level. To support this more complex rate structure, SAP Fieldglass uses:
  1. Rate Codes to capture each variation.
  2. A Rate Group to collect related Rate Codes.
  3. A Rate Grid to map rates to supplier–site combinations.
  4. A Job Posting Template that references the Rate Grid.

This multi‑step configuration produces the large number of populated Rate Codes you expect to see in Admin > Rates when you manage supplier‑ or site‑based rate cards.
When you use a role‑based rate card, the templates already contain the ranges you need, so the system does not require additional Rate Codes.

Resolution

Create a Report to view accurate rate ranges.

  1. Sign into your SAP Fieldglass account.
  2. Click Analytics.
  3. Click Create Report.
  4. Choose Job Posting Template as the module.
  5. Add these fields:
    • Min Template Bill Rate
    • Max Template Bill Rate
    • Minimum Pay Rate (optional)
    • Maximum Pay Rate (optional)
  6. Add a filter for the currency you want to review.
  7. Run the report.

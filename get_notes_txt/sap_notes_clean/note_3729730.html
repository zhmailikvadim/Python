Symptom

  • You need to identify which tables are utilized within specific SAP HANA objects, for example views.
  • When a base object becomes inactive in the HANA repository, it's crucial to identify which objects depend on it to understand the potential impact and plan appropriate remediation steps.

Environment

  • SAP HANA Platform 1.0
  • SAP HANA Platform 2.0

Reproducing the Issue

  • The Base object became inactive status
  • Need to check the object dependencies

Cause

In SAP HANA, objects can have complex dependency relationships. When a base object (such as a table, Calculation view, or procedure) becomes inactive, all dependent objects may be affected, potentially causing application failures or data access issues.

Resolution

Method 1: Using System Views for Database Objects

For database objects like tables, use the SYS.OBJECT_DEPENDENCIES system view:

-- Check which objects depend on a specific base object
SELECT 
    BASE_SCHEMA_NAME,
    BASE_OBJECT_NAME,
    BASE_OBJECT_TYPE,
    DEPENDENT_SCHEMA_NAME,
    DEPENDENT_OBJECT_NAME,
    DEPENDENT_OBJECT_TYPE
FROM "SYS"."OBJECT_DEPENDENCIES"
WHERE BASE_SCHEMA_NAME = 'YOUR_SCHEMA_NAME'
  AND BASE_OBJECT_NAME = 'YOUR_TABLE_NAME';

Replace 'YOUR_SCHEMA_NAME' and 'YOUR_TABLE_NAME' with the actual schema and table names you are investigating. For more detailed information on the OBJECT_DEPENDENCIES system view, you can refer to the SAP Help Portal: OBJECT_DEPENDENCIES System View

Method 2: Using Repository Views for Modeling Objects

For HANA modeling objects (calculation views, attribute views, etc.), query the repository system views:

-- Check dependencies for active modeling objects
SELECT * FROM _SYS_REPO.ACTIVE_OBJECTCROSSREF 
WHERE FROM_OBJECT_NAME = '' 
   OR TO_OBJECT_NAME = '';
-- Check dependencies for inactive modeling objects
SELECT * FROM _SYS_REPO.INACTIVE_OBJECTCROSSREF 
WHERE FROM_OBJECT_NAME = '' 
   OR TO_OBJECT_NAME = '';

Method 3: Using SAP HANA Studio

  • Access the Repository Browser:

    • Open SAP HANA Studio
    • Navigate to the SAP HANA Systems view
    • Expand your system node and the Content node

  • Locate the Object:

    • Navigate to the package containing your inactive object
    • Right-click on the object

  • Check Dependencies:

    • Select "Where-Used" from the context menu to see dependent objects [1]
    • For view elements like input parameters or columns, select the element and choose "References" from the context menu

Method 4: Identifying Broken/Inactive Objects

First, identify which objects are inactive or broken:

-- Query to get list of Broken/Inactive Objects
SELECT 
    "PACKAGE_ID",
    "OBJECT_NAME",
    "OBJECT_SUFFIX",
    "ACTIVATED_AT",
    "ACTIVATED_BY"
FROM "_SYS_REPO"."ACTIVE_OBJECT"
WHERE "OBJECT_STATUS" != 0
  AND "PACKAGE_ID" = '';

Symptom

After completing the onboarding process, when a Request for Correction is triggered at the Final Review step, the system behavior is observed as follows:

  • The Final Review step appears as completed after the correction is submitted.
  • The Personal Data Collection (PDC) step is not triggered.
  • Final Review step is re-trigger again.

Environment

SAP SUCCESSFACTORS ONBOARDING

Reproducing the Issue

  1. Complete the onboarding process up to the Final Review step.
  2. At the Final Review step, submit a Request for Correction.
  3. Observe the process behavior after the correction is submitted:
  • The Personal Data Collection (PDC) step is not triggered again.
  • The Final Review step is triggered again instead.

Cause

This is a system limitation.

Resolution

For the hire type Rehire with Old Employment, the Request for Correction action triggered at the Final Review step does not re-initiate the Personal Data Collection (PDC) step.

The functionality to trigger PDC again via Request for Correction after Final Review is supported only for the following hire types:

  • New Hire
  • Rehire with New Employment

This behavior is not supported for Rehire with Old Employment, hence the PDC step is not triggered again once the Final Review step is completed.

NOTE: 

If this functionality is required for your business process, we recommend submitting an enhancement request via the SAP Influence portal to have this feature considered in future releases.

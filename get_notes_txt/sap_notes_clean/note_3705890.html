Symptom

When deploying projects in SAP Build Cloud Visual Functions, may experience the following:

  • Deployments remain in “Deploying” status and do not complete.
  • The issue affects all Visual Function projects in the same subaccount.
  • Creating, importing, or cloning a project does not resolve the problem—deployments still remain stuck.
  • The deployment does not progress to either Deployed or Failed, even after several hours or days.

Because deployments cannot finish, publishing updates or delivering new versions is not possible.

Environment

 SAP Build Apps

Cause

In rare cases, the deployment process fails because a required database instance cannot be successfully provisioned during deployment.

When this occurs:

  • The deployment workflow waits for the database provisioning step to complete.
  • Since the provisioning does not finish successfully, the deployment remains in “Deploying” indefinitely.
  • Because the shared provisioning step is blocked, all Visual Function deployments in the same subaccount are affected.

This behaviour is not related to customer configuration within the project.

Please note:

  • This condition is rare and typically affects an entire subaccount rather than an individual project.
  • The behaviour does not indicate an issue with the project content or customer-side deployment process.

Resolution

This issue cannot be resolved through customer actions in the SAP Build interface because the underlying cause is a failure during the creation of the HANA instance required by Cloud Visual Functions.

When the HANA instance provisioning step fails, the deployment workflow cannot continue, and all Visual Function deployments for the affected subaccount remain stuck in “Deploying.”

After Support resolves the issue on the backend, new deployments should complete normally.

Keywords

SAP Build, Cloud Visual Functions

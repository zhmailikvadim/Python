Symptom

This SAP Note is relevant only for the country version Germany.

You use your HBA card to sign electronic regulations. Signing is not possible.

Other Terms

Crypt, RSA, ECC, HBA, health professional card, connector, PTV5, PTV6, ECC only, encryption procedure, consumer proxy, CO_ISH_TICERTIFICATE_SERVICE_P, READ_CARD_CERTIFICATE, CRYPT, TI, telematics infrastructure

Reason and Prerequisites

This is caused by a program error.

The method READ_CARD_CERTIFICATE has an optional parameter CRYPT in accordance with the GEMATIK specification. This parameter is not yet contained in the interface of the method READ_CARD_CERTIFICATE  of the consumer proxy CO_ISH_TICERTIFICATE_SERVICE_P. As a result, the system does not return a certificate and therefore signing is also not possible.

Prerequisites:

  • You use the country version Germany.
  • The business function ISH_AMBULATORY is active in your system.
  • You use a connector with PTV5 or lower (firmware version).
  • You use a new HBA that has only one ECC certificate. There is no RSA certificate on the HBA, only an ECC certificate.

Note that if you already use a corrector with PTV6, this error does not occur.

Solution

Implement the attached correction instructions or import the specified support package.

This correction is the prerequisite for the correction of note 3723569 DE: ACM: TI: Error in HBA signature without valid RSA certificate.

If you want to implement the corrections before you import the Support Package, you must also implement the corrections from SAP Note 3723569 after you implement this SAP Note. Note the sequence. Signing is only possible again if both SAP Notes are implemented.

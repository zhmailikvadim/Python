Symptom

When an order transitions to Work Done in the Perform Maintenance Job app, the system assigns the status JBFI (Job Finished). This status should be reset at the operation level when the operation's confirmation is canceled (CNF or PCNF).

Current Issue: Both JIPR and JBFI statuses are maintained at the operation level, which can complicate the cancellation logic.

Other Terms

reset JBFI, JIPR, CNF, PCNF, PMJ, phase enabled order, RMPO, status change. work done, work in process

Reason and Prerequisites

When setting Partial Time Confirmation, JBFI system status is reset if JIPR and CNF was set before.

Solution

Please implement this correction to your system.

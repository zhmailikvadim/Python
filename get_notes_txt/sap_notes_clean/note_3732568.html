Symptom

  • Error messages appear when enabling Data Collection for SAP Integration Suite (Cloud Integration) on Neo, and the Data Collection automatically switches off.

Environment

  • SAP Cloud ALM
  • SAP Integration Suite (Cloud Integration) on Neo

Reproducing the Issue

  1. Log on to SAP Cloud ALM.
  2. Open the Integration & Exception Monitoring app.
  3. Open the configuration menu on the top right.
  4. Select the target SAP Integration Suite service.
  5. Toggle to enable Data Collection.
  6. Observe the errors:
    1. "Error Occured during default activation for category SAP Integration Suite Messages".
    2. "Error Occured during default activation for category Integration Content".
  7. The Data Collection will be automatically disabled.

Cause

  • Prerequisite configuration steps were not completed.
    • The OAuth client used for data collection was missing required authorizations.
    • Endpoints for the SAP Integration Suite (Cloud Integration) service are not created in the SAP Cloud ALM Landscape Management app.

Resolution

  1. Logon to the SAP BTP Global account and Subaccount where the SAP Integration Suite (Cloud Integration) service on Neo is running. 
  2. Select the Security tab on the left of the screen.
  3. Select the Authorizations sub tab.
  4. Verify and assign the required roles/authorizations to the OAuth client used by SAP Cloud ALM:
    1. For Integration & Exception Monitoring, ensure the OAuth client has IntegrationOperationsServer.read and NodeManager.read.
    2. For Health Monitoring, ensure the OAuth client hasHealthCheckMonitoringData.Read.
  5. Logon to SAP Cloud ALM.
  6. Open the Landscape Management app.
  7. Locate the SAP Integration Suite (Cloud Integration) service on Neo and click it.
  8. Click the Endpoints tab.
  9. Click "Add".
  10. Enter the OAuth client data for the SAP Integration Suite (Cloud Integration) service on Neo.
  11. Select the relevant use cases for monitoring.
  12. Perform the connection check.
  13. All checks should pass with status "200 OK".
  14. Save the endpoint
  15. Open the Integration & Exception Monitoring app.
  16. Enable Data Collection again for the SAP Integration Suite (Cloud Integration) service on Neo.
  17. The data collection activation should complete without errors.

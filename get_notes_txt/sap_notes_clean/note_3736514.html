Symptom

Applications show “Requisition Closed” in the application status, but the application system status remains “Open”.

This leads to candidates and applications not being eligible for purge as application is still considered In Progress.

The issue occurs after closing requisitions either manually or via API.

Environment

SAP SuccessFactors Recruiting Management 

Reproducing the Issue

  1. Open Recruiting and select an open job requisition with candidates.
  2. Change the requisition to a closed status and close the requisition (either manually in the UI or via API).
  3. Allow the automatic disposition job moveCandidatesToReqCloseStatus to run.
  4. Check affected applications: the displayed application status is “Requisition Closed” but the Application System Status remains “Open.”

Cause

  • A defect where the automatic disposition of applicants does not update the Application System Status to “Closed” when the requisition is closed. The affected process is handled by the moveCandidatesToReqCloseStatus job. This occurs for requisitions closed via UI and via API.

Resolution

The solution of this issue is currently under investigation. Please follow this KBA for updates.

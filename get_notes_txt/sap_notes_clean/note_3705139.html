Symptom

While creating an audit in transaction PLMD_AUDIT, the context menu (right mouse button click on audit object in tree view) has inactive function under 'Create' like:

  • Assign Question List
  • Audit Question List/Question
  • Audit Questn List/Questn Using Template
  • Import Document

Alternatively, other functions in the menu can be inactive too.

Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.

Environment

  • SAP ERP Central Component
  • SAP ERP
  • SAP Enhancement package for SAP ERP
  • SAP S/4HANA, on-premise
  • SAP S/4HANA Cloud Private Edition

Resolution

1.) In case of an SAP S/4HANA 2023 system, ensure that following SAP Note is implemented in the system:

3426422 - PLMD_AUDIT: action 'Assign Question List' is not enabled

2.) In a non-SAP S/4HANA 2023 system (or if the issue persists after step 1.) check whether there is an active implementation in the affected system for BAdI PLM_AUDIT_AUTH_CHECK.

By default this BAdI has a standard implementation (class: CL_DEF_IM_PLM_AUDIT_AUTH_CHECK, method: IF_EX_PLM_AUDIT_AUTH_CHECK~SET_AUTHORITY_ACTIVITIES), which fills the necessary activities into table ct_activities for audit management authorization processing. A custom implementation can overwrite, clear or not fill this table at all. Ensure that the required activities are passed into the table.

See Also

For possible activities in authorization management related to audits, check 'Value Range' for domain PLM_AUDIT_SUB_ACTVT in transaction SE11, like:

3211 - Assign Question List to Audit

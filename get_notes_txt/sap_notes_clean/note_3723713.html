Symptom

During M20 step 2 errors like below examples are generated:
- FINS_ML_MIG 010 CKMLHD-entries do not exist for following records:
- FINS_ML_MIG 011 MBEW table entry, identified via field KALN1: 000123456789

Errors are caused by side-effect of the SAP Note 3678236.

Other Terms

M10, M20, FINS_ML_MIG010, FINS_ML_MIG011, MIG_XBEW_ML_ACT_FOR_KALNR_AMDP

Reason and Prerequisites

SQL select (detecting materials for future fix) was corrupted by SAP Note 3678236.   
- valuated materials available in MBEW table from plant with material ledger already active and 
- with missing records in tables CKMLHD, CKMLPR, CKMLPP and CKMLCR should be detected 
- must be generated later in M10 step (as missing ML records)

Solution

Implement this SAP Note and rerun from M10 step.

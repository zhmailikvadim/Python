Symptom

When applications try to acquire the value chain lock, the CX_VCM_RT_LOCK exception is thrown because the lock cannot be obtained. If the application waits for the lock to be released, it results in an endless wait. Whereas if it catches the exception, the process terminates with a locking error.

Other Terms

VCM, CX_VCM_RT_LOCK, E100, VCM_RUNTIME, VL10H, SFS_W_VSIT, IC_SALES, ICO_STOCKTRANSFER, MS_IC_SALES, MS_IC_STOCKTRANSFER

Reason and Prerequisites

When Applications initially try to collect all locks, they request the locks with the parameter Collect = "X". Later, the same locks are released using Dequeue without Collect. With this combination, the initially requested lock persists in the system after ENQUEUE_FLUSH. When the same application tries to lock the value chain without Collect, a CX_VCM_RT_LOCK exception is thrown. Due to repeated retry attempts, these locks are never obtained until the session is terminated and hence the program runs forever.

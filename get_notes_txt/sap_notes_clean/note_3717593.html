Symptom

  • There is an issue occurring since last day with ToC synchronization from managed system to Cloud ALM's Features -> was working previously
  • User triggers the ToC creation from Cloud ALM Feature and it is created in managed system but the status remains at Creation Scheduled, even after some time passes
  • Transport Request creation working as expected

Environment

SAP Cloud ALM

Reproducing the Issue

  1. Launch SAP Cloud ALM
  2. Click Implementation > Design, Build, Test, Deploy > Features
  3. In Project, enter involved Project Name
  4. In results list, click a feature which is at point where Transport is created and ready for ToC creation
  5. In Transports, click Copy
  6. In pop-up, pick the TR(s) where ToC should be created and click Copy
  7. Status becomes Creation Scheduled
  8. Logon to managed system and in sm37, check /SDF/CALM_CDM_TR_PROC_CL_DEP-<client> and see the log indicates a Transport of copies is created for the involved TR, but at the end of the log is an error:
    Post request returned with status 500
    Post request failed with status 500
    Post request failed with status 500

Cause

This is a known problem that development is working on with hopes to deliver yet today.

Resolution

The fix has been delivered to resolve the ToC synchronization problem.

Please retest the issue by logging off and logging on to your Cloud ALM tenant.  If the problem remains, do let us know so we can review.

Keywords

CALM, CDM, Transport of Copies

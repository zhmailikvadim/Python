Symptom

Question marks (???) appear alongside the MDF rule error message when copying and pasting time sheet data. Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.  

Environment

  • SAP SuccessFactors HCM Suite
  • SAP SuccessFactors Employee Central-Time Sheet
  • SAP SuccessFactors Time Tracking

Reproducing the Issue

  1. A validation rule is configured on the Employee Time Sheet Entry object, which triggers an error message when certain conditions are met.
  2. Navigate to the employee’s time sheet, copy an existing time sheet entry, and paste it to another date.
  3. After clicking the Paste button, the error message is correctly triggered; however, it appears together with “???”. 

Cause

This is a known error in the current behavior.

Resolution

A fix will be delivered in the future release, please follow this KBA for updates. 

As a workaround, create entries manually in the time sheet UI instead of copy-and-paste, the message will displays correctly without question marks.

Symptom

  • Attempting to expand characteristics for Varieties(ID) via OData returns an error.
  • HTTP status: 400.
  • Error message: Navigation property "characteristics" is not defined in CropOData.Varieties.

Environment

  • Product: SAP Intelligent Agriculture
  • SAP Intelligent Agriculture

Reproducing the Issue

  1. Use the legacy CropService OData endpoint and target the Varieties entity.
  2. Send a request to Varieties(the id) with the query option $expand=characteristics.
  3. Observe that the service responds with HTTP 400 and the message that the "characteristics" navigation property is not defined.

Cause

The request was executed against the legacy CropService, which has been phased out and does not implement the characteristics navigation for Varieties; therefore, $expand=characteristics is not available in that service.

Resolution

  1. Switch from the legacy CropService to the newer CropAndVarietyService.
  2. Use the Varieties entity in CropAndVarietyService and request characteristics via $expand=characteristics as documented.
  3. Test the request to confirm that the characteristics data is returned as expected.

See Also

refer to: CropAndVarietyService API documentation

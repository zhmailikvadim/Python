Symptom

An employee has an attachment available in both Employment Details and Job Information (the same attachment, originating from the termination process). When another user updates a different field in Employment Details without having view permission for the attachment field, the system automatically removes the attachment. No error message or fingerprint is displayed.

Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.

Environment

SAP SuccessFactors Employee Central

Reproducing the Issue

  1. Proxy as an user who does not have permission for attachment field in Employment Details;
  2. Navigate to sample employee profile;
  3. Go to Employment Details and make a dummy change in any field > save the change;
  4. Become self;
  5. Navigate again to the same sample profile;
  6. Observe that the attachment in Employment Details has been removed.

Cause

Known issue.

Resolution

A code correction for the 1H2026 major release is planned, however not yet confirmed.

For the general queries about attachment field in Employment Details, in both Take Action > Employment Details and Employment Details portlet, the Employment Details > Attachment permission is respected (there is no additional check on Job Info > Attachment permission).

If you would like to grant View/Edit attachment capabilities on the Employment specific UIs, in the meantime, please try also giving the admin Employment Details > Attachment permission.

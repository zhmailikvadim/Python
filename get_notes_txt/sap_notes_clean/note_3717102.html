Symptom

You are creating an inbound delivery based on an Expected Goods Receipt document, which contains a document specific conversion factor. The item is created with the base unit of measure as expected, however the unit of measure on the item level can be changed, which is not expected and the change can cause inconsistencies.

Other Terms

UoM, document specific conversion factor, EXT_UOM_IND, D, sc_conv_factor_diff

Reason and Prerequisites

This issue is caused by program error.

Solution

Implement the relevant support pack or the attached corrections.

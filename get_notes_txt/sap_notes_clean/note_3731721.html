Symptom

Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.

  • When releasing a billing document to accounting, the accounting document is not generated.
  • Message displayed: "The document is not relevant for accounting".
  • Header posting status is set to VBRK-RFBSK = D (not relevant for accounting)

Environment

  • Sales And Distribution (SD)
  • SAP R/3
  • SAP R/3 Enterprise 4.7
  • SAP ERP Central Component
  • SAP ERP
  • SAP Enhancement package for SAP ERP
  • SAP Enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA
  • SAP S/4HANA Cloud Private Edition

Reproducing the Issue

  1. Create a billing document.
  2. Attempt to release the billing document to accounting.
  3. Observe the message "The document is not relevant for accounting" and that no accounting document is created.
  4. Check the document header (table VBRK) and confirm that field RFBSK (posting status) is 'D' (not relevant for accounting).

Cause

The reason for this can be that in customizing transaction VOFA for the affected billing type, the field TVFK-KALSMC (Account Determination Procedure for G/L Accounts) is not maintained. During invoice creation, if TVFK-KALSMC is initial, the system sets VBRK-RFBSK = 'D', making the document not relevant for accounting. Consequently, release to accounting will be prevented because of this. 

The related code can be found in FORM VBRK_VBRP_FUELLEN / LV60AA93:

...
  IF tvfk-kalsmc = space.
    vbrk-rfbsk = 'D'.
...

Resolution

  1. Open billing type customizing (transaction VOFA or equivalent IMG path).
  2. Select the affected billing type and maintain the field "Account Determination Procedure for G/L Accounts" (TVFK-KALSMC).
  3. Assign the correct account determination procedure (commonly KOFI00, but necessary to discuss and validate internally).
  4. Save the change.
  5. Retry with a new billing document creation.

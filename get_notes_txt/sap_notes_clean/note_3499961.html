Symptom

Error KO288 issued when trying to change company code in transaction KO02

Environment

  • Controlling (CO)
  • SAP ERP 
  • SAP Enhancement package for SAP ERP
  • SAP Enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA
  • SAP S/4HANA Finance

Cause

system design

Resolution

Please note that when creating an object in CO, the SAP system defaults the currency of the company code to which the object is assigned as the object currency. There is the possibility to specify a different object currency only if the controlling area currency is the same as the company code currency (one Company Code and one Controlling Area).

If the company has more than one company code assigned to the controlling area, which would make it a cross-company code. The currency for controlling area and the company codes may differ for the cost accounting that is cross-company code. But for any object currency(for example Internal Order) for cross-company codes it is only possible to choose that of the assigned company codes currency.

Hence the error message is correctly justified and as designed, the object currency must be equal to the company code currency.

The object currency for a statistical internal order can be different from the company code currency.

Also please check the following SAP WIKI Article :https://wiki.scn.sap.com/wiki/display/ERPFI/Changing+Currency+on+Internal+Order+or+Receiving+Error+KO288

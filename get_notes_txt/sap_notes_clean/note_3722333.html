Symptom

Import an Analytic Model with option 'Deploy after import', the import succeed but deployment of the AM failed with message:
---
The Deployment of Imported content failed on step deploy with error: [Datasphere Repository] Failed to deploy imported content, The deployment of objects in space YYY was unsuccessful, Failed to Deploy "XXX" in "YYY".
---
The import package contains only one object which is the AM, and the AM does not have any dependent object. 
The deployment failure occurs only for specific analytic model. Other analytic models are successfully imported and deployed without issues.

Environment

SAP Datasphere

Reproducing the Issue

  1. Import the package which contains the Analytic Model from the source Datasphere system to the target Datasphere system.
  2. Select the "Deploy after import" option during the import process.
  3. Observe that the import completes successfully, but the deployment of analytic model fails with error message.

Cause

Product error

Resolution

A fix for the issue has been implemented and is planned to be available with SAP Datasphere release 2026.07.
Refer to SAP Note 2984828 for updated deployment information.
Until the fix is available, manually deploy imported models as a workaround.

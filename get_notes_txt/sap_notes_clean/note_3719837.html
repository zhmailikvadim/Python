Symptom

  • In the Generate ESG Reports with AI application, the Delete button for a custom template is disabled or appears "greyed out".
  • An attempt to delete a custom template is not possible.
  • There are questions regarding the lifecycle management of custom templates, such as what occurs when the user who created the template is removed from the tenant.
  • Clarity is needed on whether a custom template can be re-assigned or managed by other users after the original creator has been deactivated.

Environment

Sustainability Control Tower

Reproducing the Issue

  1. Navigate to the Generate ESG Reports with AI app and access the Templates tab.
  2. A new custom template is created.
  3. Navigate to the Reports tab and a new report is generated using the custom template from the previous step.
  4. Return to the Templates tab and select the custom template.
  5. Observe that the Delete button is disabled.

Cause

The observed behavior is by design, based on two core application principles:

  • The system enforces a data dependency check. A custom template cannot be deleted while active reports, generated from that template, exist. This feature protects the lineage and integrity of existing reports.
  • Custom templates are tenant-level objects, not owned by individual users. The Created By field serves for auditing and informational purposes only and does not restrict other authorized users from managing the template.

Resolution

The resolution addresses the two distinct scenarios.

Part 1: Procedure to Delete a Custom Template

To enable the Delete button for a custom template, all reports generated from it must be deleted first.

  1. Prerequisite: The user performing this action must have the SUS_SCT_AI_Reporting_Assistant_Manager role assigned.
  2. Navigate to the Generate ESG Reports with AI app and select the Reports tab.
  3. Identify and select all reports that were created using the custom template intended for deletion.
  4. Use the Delete button to remove each of these reports.
  5. After all associated reports are deleted, navigate back to the Templates tab.
  6. Select the custom template. The Delete button will now be enabled.
  7. Click Delete to permanently remove the template.

Note: SAP-provided standard templates cannot be deleted.

Part 2: Template Management When a User is Deactivated

  • If the user who created a custom template is removed from the tenant, the template is not impacted. It remains fully functional and available in the system.
  • Any other user with the SUS_SCT_AI_Reporting_Assistant_Manager role can edit, use, and delete the template (provided the conditions in Part 1 are met).
  • The functionality to change the Created By or Changed By fields is not available, as this is the standard system behavior.

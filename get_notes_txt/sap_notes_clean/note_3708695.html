Symptom

  • At Screen Configuration 'ChecklistInstance' the Quick Action Bar 'Attachment' has set a visibility expression below but still remains visible on Android, works fine on iOS and Windows Client.
  • Example visibility expression where the selection 'Attachments' should be visible when the template name is not 'TypeA': 
    {
             "visibleExpression": "${checklistInstance.template.name} !== 'TypeA'",
             "visible": true,
             "name": "action.createAttachments"
          },

Environment

SAP Field Service Management

Reproducing the Issue

  1. Log in mobile app
  2. Navigate to any Service Call with a Smartform (according to the visibility expression with template name 'TypeA')
  3. Click on the three dots menu at top-right corner
    Expected Behavior:
  4. The selection "Attachments" is not visible
    Actual Behavior:
  5. The selection "Attachments" is visible

Cause

Android-only behavior requires an extra 'action.createAttachment' entry, this will then allow to hide the Attachment action, this is not needed on iOS or Windows Client.

Resolution

For Android only, add a second 'action.createAttachment' entry and configure its visibility to enable hiding the Attachment action. No change is required for iOS or Windows clients.

Add this expression to the JSON 'ChecklistInstance':
{
         "visibleExpression": "${checklistInstance.template.name} !== 'SmartformTemplateName'",
         "visible": true,
         "name": "action.createAttachments"
      },

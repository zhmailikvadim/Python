Symptom

  • When processing an EDI invoice for transfer prices (RBKP-IVTYP=7), users observe that the invoice cannot be parked.
  • The modification note 501524 has been implemented, creating a small piece of code under user exit EXIT_SAPLMRMH_014 to populate the field RBKPV-RBSTAT with the required status.

Environment

  • Materials Management (MM)
  • SAP ERP Central Component
  • SAP ERP
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA 
  • Industry solution ECC-DIMP
  • SAP S/4HANA

Reproducing the Issue

  1. Process an EDI invoice for transfer prices (RBKP-IVTYP=7).
  2. Attempt to park the invoice.
  3. Observe that the invoice cannot be parked. 

Cause

The feature is not supported.

Resolution

  • Transfer prices are supported in the standard design for IDoc invoices only. Once you park the invoice, the EDI process becomes a online process as it calls the same logic as would posting an online document with transaction MIR7. This is no longer a EDI posting. Because the transfer prices data can be not saved in the MM-invoice within the invoice parking process, as you post the invoice further via another invoice process, the transfer prices data can be not transferred to the follow-on FI DOC.

  • Due to the limitation that transfer price (IVTYP = 7) is not supported in the online process, the parking functionality provided by note 501524 does not apply to transfer price EDI invoices.

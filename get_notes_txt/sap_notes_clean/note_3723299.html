Symptom

Some customers using the Skillsoft OCN integration have reported cases where Learning search results display the learning Entity ID instead of a title. This behavior is caused by a combination of (1) how Skillsoft provides localized titles and (2) how the current Learning search fallback rules are designed.

A change is planned for an upcoming release to improve this behavior. Because of this, we do not recommend altering Skillsoft items or manually adjusting titles, as any workaround may be temporary and could interfere with the new logic once delivered.

Environment

SAP SuccessFactors Learning

Resolution

Current System Behavior

Today, Learning search follows these rules when deciding which title to display:

  1. If the item is available in only one locale, the title for that locale is displayed.
  2. If the item is available in multiple locales and one matches the user’s locale, the title in the user’s locale is displayed.
  3. If the item is available in multiple locales, none match the user’s locale, but the search keyword matches a title in another locale, the system displays the title that contains the keyword.
  4. If the keyword appears in multiple localized titles, and none match the user’s locale, the system displays the Item ID.
    • This happens because there are multiple possible titles that contain the keyword, and the system cannot determine which one to prefer.
  5. If the item is available in multiple locales, none match the user’s locale, and there is no keyword match, the system falls back to displaying the Item ID.

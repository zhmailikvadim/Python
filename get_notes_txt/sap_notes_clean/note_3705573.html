Symptom

  • An alias for the web server used for object links is configured and its URL points to the development system for example.
  • When this customizing is transported to the quality system, the entry still points to development, which is not desired.
  • As a result, opening an object link directs to the wrong system.
  • It is unclear how to transport this customizing correctly to the quality system and later to production.

Environment

  • SAP Portfolio and Project Management
  • SAP Portfolio and Project Management for S/4HANA

Cause

The customizing entries are system and client-specific.

Resolution

  • SAP does not provide a standard method to define this customizing for transport and reuse across systems/clients. 
  • These entries are designed to be maintained directly in each system/client rather than transported.
  • Such kind of entries, similar to 'RFC Destination' and 'Logical System', vary by system/client and therefore cannot be transported in a meaningful way. 
  • Basis team should enable the relevant customizing nodes to allow direct maintenance in every system.

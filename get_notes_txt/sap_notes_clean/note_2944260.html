Symptom

Conversion of Remuneration Code 001 to code 002 is not done automatically in case of no Service hours 001

Environment

  • Human Capital Management (HCM) Payroll Belgium
  • SAP R/3
  • SAP ERP Core Component
  • SAP ERP
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA

Reproducing the Issue

[Enter your Reproducing the Issue here...]

Cause

SAP will not deliver an automatism: when there is no REM code 001, we are not able to know whether this is caused by mistake or this is about a period where no Service code 001 exists. We believe it is customer responsability to do the mapping.

Resolution

SAP doesn't deliver an automatism to convert the REM Code 001 into the 002. Possible solutions:

  • book the payments ( single holiday payment, overtime...) on the last month of the activity
  • via a customer specific wagetype, compensate the payments done on R001 code in SVREM and rebook them on R002

Symptom

 You use the ARMSUI solution with the product switches.

When you refresh the product switches (for example, using the report /ARMSUI/RE_PS_UPDATE), some MRP elements are not taken into account.

Other Terms

 ARMSUI, product switch

Reason and Prerequisites

The reason for this is that the ATP categories that are taken into account for the product switch refresh are currently hard-coded and a category may be missing in the code here.

Solution

Implement the attached SAP Note. The ATP categories to be taken into account can then be maintained on a customer-specific basis in the Customizing table /ARMSUI/T_CIF_PS.

You can find this table in the app "Configure Material Control" under the path "Material Control" -> "Control Center" -> "Integration with PP/DS" -> "General Settings" -> "Define ATP categories for product switch refresh".

This SAP Note delivers a preassignment of this table. After you implement the corrections, check whether categories are still missing in this preassignment or whether you do not need some.

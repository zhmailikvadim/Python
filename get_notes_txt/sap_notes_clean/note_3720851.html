Symptom

  • When replacing a component in the "Manage Material" app (ID F1813), the base unit of measure is not updated automatically
  • Error messages encountered include: "Consistency check when changing BOM item & was not successful" , "Unit of measure & not supported for material & in plant &"

Environment

  • SAP ERP Logistics (LO)
  • SAP R/3
  • SAP R/3 Enterprise
  • SAP ERP Central Component
  • SAP ERP
  • SAP Enhancement package for SAP ERP
  • SAP Enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA
  • SAP S/4HANA Cloud Private Edition

Reproducing the Issue

  1. Enter Fiori App Maintain Bill of Material
  2.  Choose BOM to edit
  4. Click Edit Button
  6. Change component to already existing component with a different UoM
  8.  
  9.  No automatic update from “ to EA is observed
  10.  

Resolution

This has been identified as a bug and corrected in the latest S/4 HANA releases. This behaviour now aligns with standard SAP GUI transactions; for example, in transaction CS02, replacing a component with a different UoM triggers an error. This fix ensures consistent standard behaviour across both Fiori and the SAP GUI.

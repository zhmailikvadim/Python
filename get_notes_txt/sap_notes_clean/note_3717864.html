Symptom

Duplicate transport entries appear in the transport assignment dialog within the Features App.

Environment

  • SAP Cloud ALM 1.0

Reproducing the Issue

  1. Navigate to the Features App and attempt to assign transports.
  2. Observe duplicate entries for the same transports under the transport assignment dialog.

Cause

  • Maintaining two destinations to the same SAP Cloud Transport Management service instance
  • Renaming HTTP destinations to one SAP Cloud Transport Management service instance

Resolution

After a release/import event at CTMS level CALM is getting informed and a entry for the Destination-TR number will be displayed.
So when there are duplicated HTTP destination is created for one CTMS instance or HTTP destination is re-named, from CALM point of view, it will get a second entry for the new destination name and transport.

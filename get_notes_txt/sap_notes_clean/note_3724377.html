Symptom

When you use integration scenarios between Transportation Management and Product Compliance in SAP S/4HANA, the system uses different interface structures for EDI data exchange for the different feature pack stacks for SAP S/4HANA 2025. As a result, data retrieval and processing of data from Product Compliance in Transportation Management processes do not follow a unified pattern across different feature stack packs.

Other Terms

Transportation Management, Product Compliance, Electronic Data Interchange (EDI), Interface

Reason and Prerequisites

The system uses different interface patterns for EDI data exchange between Transportation Management and Product Compliance components, which leads to inconsistent integration behavior for the different service pack stacks for SAP S/4HANA 2025.

Solution

After implementing this SAP Note, the system uses harmonized interfaces for EDI calls between Transportation Management and Product Compliance. The IF_EHFND_TM_INTGR_COMP interface with the GET_EDI_STRUCTURE method and the IF_EHFND_PCLD_DAC interface with the READ_PCLD_DATA_BY_UUIDS method now provide consistent data structure handling across the service pack stacks.

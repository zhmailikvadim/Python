Symptom

  • When exporting a Web Intelligence document to Excel, some rows are merged unexpectedly. The affected rows contain a text field of up to 3000 characters.

Environment

  • SAP BusinessObjects Business Intelligence platform 4.x
  • Web Intelligence (WebI)

Reproducing the Issue

  1. Open any Web Intelligence document that has object with more than 3000 characters.
  2. Export the report to Excel format.
    >> Noticed that some rows in the exported file are merged..

Cause

  • This behavior should not be classified as a defect but rather recognized as an inherent limitation within Microsoft Excel. The application imposes a maximum row height of 409 points (approximately 546 pixels), and any attempt to exceed this threshold is automatically adjusted back to the limit.

Resolution

  • In Excel, the height of a row is limited to a maximum of 409 points, which corresponds to about 546 pixels. Even if you try to set a higher value, Excel will automatically reduce it back to this limit. This means that a cell cannot display unlimited vertical content, and once the maximum height is reached, the visible portion of the text will be cut off. This happens even when text wrapping is enabled or very large font sizes are used, making it important to keep in mind that Excel has a strict cap on how much vertical content can be shown in a single cell.

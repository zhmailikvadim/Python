Symptom

Running the preview on a SQL view having ORDER BY DESC or ASC in the sql does not change the result order

Environment

  • Product: SAP Datasphere

Reproducing the Issue

  1. Create a SQL view in Data Builder that selects a column and includes an ORDER BY clause on that column (for example, ORDER BY DESC and ORDER BY ASC).
  2. Open the Data Preview for the SQL view and observe that the displayed rows are not sorted according to the ORDER BY clause.

Cause

The Data Builder preview ignores ORDER BY clauses in SQL views by design; ORDER BY is applied when the view is consumed by applications or clients, not in the preview.

Resolution

  1. Verify the sort behavior by consuming the SQL view from an analytics client or SQL tool; the ORDER BY clause will be applied when the view is executed by consumers.
  2. If you need a sorted display during preview, use the preview’s UI sort controls rather than relying on the ORDER BY clause in the SQL view.
  3. If sorted preview based on SQL ORDER BY is required as a product capability, consider submitting an enhancement request via the official channel.

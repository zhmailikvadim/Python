Symptom

If you use SAP Content Server Version 6.40 or 6.50, the status OFFLINE can be displayed for a repository during even short periods of inactivity. As a result, subsequent reports initially fail and are successful only in a further run.

The content server trace ﻿CS_Trace﻿ contains error messages like the following:

﻿X-ErrorDescription: "BeginTransaction ContentStorage, PrepareStatement failed[SAP AG][SDBODBC DLL][SAP MaxDB] General error;-10807 Connection down: [3] command timeout                          ;ODBC State S1000; Error Code -10807"﻿

Other Terms

Repository, ODBC driver, command timeout, session timeout, connection

Reason and Prerequisites

You use SAP Content Server Version 6.40 or 6.50.
You use SAP MaxDB Version 7.8.02, 7.9.08, or 7.9.09.

Solution

Use the database studio or SQLCLI to start the following SQL statements in the content server database.

Find out (as the user SAPR3 or SUPERDBA) what MAXTIMEOUT value is set for the user SAPR3:
SELECT * FROM users

If the value of MAXTIMEOUT is set to 30, deactivate this as the user SUPERDBA:
ALTER USER SAPR3 TIMEOUT NULL

(This undesired timeout value can occur if the user was created with the database studio by choosing ﻿Users -> New -> User﻿ from the menu.)

Symptom

  • Error message appears in /N/IWFND/ERROR_LOG transaction: "System alias 'SP1CLNT100' does not exist"
  • Entries in table /IWNGW/NOTIF are picking the incorrect system alias 'xxxx', causing failures in IWFND/ERROR_LOG.

Environment

  • SAP Gateway

Reproducing the Issue

  1. Navigate to transaction /N/IWFND/ERROR_LOG.
  2. Observe the error message: "System alias 'xxxx' does not exist".

Cause

Incorrect system alias 'xxxx' is being populated in the notification channel table /IWNGW/NOTIF instead of 'LOCAL', as per the maintained notification channel configurations.

Resolution

In order to resolve this error delete all the notifications from the system:

1. Login to the system

2. Go to transaction: /n/iwngw/h_clear_notif

3. Select radio button: "Execution mode" & select checkbox: "Wipe notifications persistency".

4. Press Execute.

Note: This will delete all the notification data present in the system. The users who have notifications can still access the notifications via MyInbox App.

Symptom

  • Error FAA_POST206 is triggered during asset transaction in transaction code F-90.
  • The error message states: "Transaction is before the legacy data transfer date".

Environment

  • FIN Asset Accounting
  • SAP S/4HANA
  • SAP S/4HANA Finance
  • SAP S/4HANA Cloud Private Edition

Reproducing the Issue

  1. Execute Transaction - > F-90
  2. Attempt to post the transaction.
  3. Observe the error message FAA_POST206 stating that the transaction is before the legacy data transfer date.

Cause

Example:

In table T093C the legacy data transfer date (field DATUM) = 31.08.2025

The posting and document date is e.g. 30.11.2025 (after the legacy data transfer date)

Even then  error message  FAA_POST206 stating that the transaction is before the legacy data transfer date is raised

The first capitalization date on the asset was e.g. on 07.05.2025

In asset accounting customizing in transaction AO54, or customizing path:

Transactions 
Specify How Default Asset Value Date is Determined 

The rule for invoice receipt without purchase order  primary rule 9 set =  Capitalization date

Therefore in this example the asset values date 07.05.2025 for the posting with transaction  F-90 is derived and this date is before the legacy data transfer date of  31.08.2025

Resolution

  1. Access transaction F-90.
  2. On the second screen, press the "More" button in the item section.
  3. In the popup, manually enter an appropriate asset value date (reference date) for the asset line item.
  4. Ensure the reference date is after the legacy data transfer date 
  5. Save the changes and attempt to post the transaction again.

Keywords

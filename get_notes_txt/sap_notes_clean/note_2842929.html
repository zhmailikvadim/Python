Symptom

Symptom

If you export data from an ALV table in the WEB GUI to Excel in MHTML format, the file name 'export.xls' is displayed in the save dialog. The file type is not filled. 
The error message 'Filter "Excel (*.XLS) has an incorrect format' is displayed.

Other Terms

CL_SALV_DATA_PUBLISHER, DOWNLOAD,
CL_ALV_BDS, CREATE_MASK_FOR_FILEFILTER,
MHTML,

Reason and Prerequisites

The export is started in the SAP GUI for HTML.

When the MHTML format is selected, a file with this format is created.
The MIME type of the file name 'export.XLS' in the ‘Save as’ dialog enables the direct opening of Microsoft Excel for an export in the previously selected MHTML format.
However, the file extension is set to .XLS because this is the best way to ensure that Microsoft Excel starts.

The type of file in the save dialog is used to filter the displayed files.
Because the extension of the export file 'export.XLS’ is not allowed to be changed, this SAP Note changes the displayed type to 'Excel (*.XLS)’
The file contents still correspond to the MHTML format, however.

Solution

Use the Note Assistant to implement the attached correction instructions.

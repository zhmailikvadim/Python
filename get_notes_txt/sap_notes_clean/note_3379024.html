Symptom

You see error below while processing a message using JMS on CPI:

Error DetailsCommit of JMS Transaction not possible due to Error committing - transaction rolled back (Transaction commit failed: (TransactedSessionId:17, Name:f6473c83209c42a984667be8e6a89db9, CorrelationId:8692760, downgrade:false) (((Client name: vsa11680847/30186/01290001/9ACuAcRCRh Local addr: 100.77.93.229 Local port: 45498 Remote addr: b4f2f0e0-3ef0-4aaf-bb87-36a8ea724724.messaging.eu1.emc.messagebroker.cloud.sap Remote port: 55443) - ) com.solacesystems.jcsmp.JCSMPErrorResponseException: 503: Message Consume Failure [Subcode:48]))  

Environment

  • SAP Integration Suite
  • SAP Business Technology Platform
  • Cloud Integration

Cause

This error means that the JMS client has temporarely lost the connection to the JMS broker. A network is not 100% stable. 

Resolution

A message retry in place should solve such problems automatically. Hence please set up message retry.

Symptom

App with Standard view set as default and "Apply Automatically" cannot be loaded when default value is not maintained in the Fiori Launchpad user settings. 

Environment

  • SAP S/4HANA

Reproducing the Issue

  1. Open the Fiori app.
  2. Open the variant (view) management.
  3. Set the standard view as default and enable "Apply Automatically".
  4. Save the changes.
  5. Open user profile > Settings > Default values.
  6. Ensure no default value is maintained for the relevant filter.
  7. Return to the launchpad home page and launch the app again.
  8. Observe that the app remains in a loading state.

Cause

A defect in SAPUI5 Analytical List Page app state handling (IappStateHandler) caused the startup promise to remain unresolved when the standard view was set to "Apply Automatically" and no user default values were provided, resulting in indefinite loading. The fix is included in SAPUI5 1.120.35 and later.

Resolution

  • Update SAPUI5 to version 1.120.35 or higher. It's always recommended to update to the latest patch of a minor version as shown in the 'Latest Patch' column here: https://ui5.sap.com/versionoverview.html
  • As a temporary workaround (until the update is applied), either maintain a suitable user default value for the relevant filter in FLP settings or disable "Apply Automatically" for the default variant.

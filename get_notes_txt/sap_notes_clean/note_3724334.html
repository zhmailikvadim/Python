Symptom

You run the Sistema de Informação de Organização do Estado (SIOE+) (RPC_PAYPT_SIOE) report and the following symptoms occur:

  1. The system displays the employee's birth country instead of the nationality in the Nacionalidade field.
  2. There is no option to change the standard output of the employee eligibility decision for generating the Ficha do Trabalhador and Saída de Trabalhador files.
  3. The logic for calculating the Antiguidade/Tempo de Serviço and Antiguidade na Carreira fields does not accurately account for absences, licenses, and bonus periods.
  4. The BAdI para SIOE+ (HRPAYPT_SIOE) enhancement spot does not have documentation.

Other Terms

Transaction: PC00_M19_SIOE_PLUS

&SIOE (for more information, see SAP Note 2197631)

Reason and Prerequisites

Reason

These issues are side effects caused by the solution delivered by SAP Note 3690959.

  1. The system incorrectly retrieves the País/região de nascimento (P0002-GBLND) field instead of the Nacionalidade (P0002-NATIO) field from the Dados pessoais (0002) infotype when populating the report.
  2. There are no BAdI methods to allow customization of the standard behavior.
  3. The system does not retrieve the calculated time evaluation data from the Cálculo de tempo de serviço (0553) infotype to determine the correct seniority and length of service. 
  4. The solution does not consider this documentation. 

Prerequisite

The following Support Package (SP) level is the minimum prerequisite:

ReleaseSupport Package
S4HCM 102 HR SP 00
S4HCM 101 HR SP 06
S4HCM 100 HR SP 19
SAP_HR 6.08 HR SP D0
SAP_HR 6.04 (EhP6 and EhP7) HR SP K2

SAP Note 3690959 is required.

Solution

  1. From now on, the system reads the nationality of the employee from the P0002-NATIO field.
  2. From now on, the HRPAYPT_SIOE enhancement spot has two new methods:
    • Alterar eligibilidade de geração da Ficha do Trabalhador (CHANGE_WORKER_FILE_ELIGIBILITY) 
    • Alterar eligibilidade de geração da Saída de Trabalhador (CHANGE_WORKER_LEAVE_ELIGIBILTY)

    With these methods, you can change the standard eligibility criteria for generating the Ficha do Trabalhador and Saída de Trabalhador files for each employee individually.

  3. From now on, the system updates the logic for the Antiguidade/Tempo de Serviço and Antiguidade na Carreira fields to consider absences, licenses, and bonus periods by retrieving data from the Cálculo de tempo de serviço (0553) infotype. The fields are mapped as follows:

    • Antiguidade/Tempo de Serviço (where field Processo de cálculo cálculo do tempo de serviço (PROCE) = 'PTPS'):

      • Trabalhador-Antiguidade na AP / tempo de serviço: anos: Retrieves the Resultado do cálculo do tempo de serviço: duração em anos (RESYY) field.

      • Trabalhador-Antiguidade na AP / tempo de serviço: meses: Retrieves the Resultado do cálculo do tempo de serviço: duração em meses (RESMM) field.

      • Antiguidade na AP / tempo de serviço: data de atualização da antiguidade na AP: Retrieves the Início da validade (BEGDA) field.

    • Antiguidade na Carreira (where field PROCE = 'PTCA'):

      • Entidade de origem-Anos: Retrieves the RESYY field.

      • Entidade de origem-meses: Retrieves the RESMM field.

      • Entidade de origem-Data de atualização da antiguidade: Retrieves the BEGDA field.

    The CRULE field of the Processo de cálculo cálculo do tempo de serviço (V_T525P) table view allows you to customize the date output format.
  4. From now on, the documentation for the HRPAYPT_SIOE enhancement spot is available in the system.

Symptom

  • An issue occurs in inbound deliveries created in reference to a Purchase Order with DELVRY07. The field "INCO1" is correctly inherited from the Purchase Order, but the field "Incoterm Location 1" (INCO2) is empty in the delivery header.
  • When the delivery is created with VL31N, INCO2 is correctly inherited (example: 180024109). However, when created via IDoc generation (WE19), INCO2 is empty.

Environment

  • Sales And Distribution (SD)
  • Logistics Execution (LE)
  • SAP S/4HANA
  • SAP S/4HANA Cloud 
  • SAP ERP Central Component
  • SAP ERP
  • SAP Enhancement package for SAP ERP
  • SAP Enhancement package for SAP ERP, version for SAP HANA

Reproducing the Issue

  1. Use an existing Purchase Order for testing.
  2. Generate the IDoc (WE19): In IDoc, ensure INCO1 and INCO2 (as well as other INCO fields) are not filled, so that the values from the Purchase Order can be inherited in the delivery.
  3. Create the delivery. Confirm that INCO1 is inherited from the Purchase Order, but INCO2 is empty.
  4. Note that when the delivery is created with VL31N, INCO2 is correctly inherited.

Cause

  • The creation of the inbound delivery does not support Incoterms 2010 functionality currently.

  • The following documentation explains it:

    2180138 - Incoterms 2010 Enhancements

    Restriction

    It is not recommended to activate this Incoterms 2010 functionality in SD and MM if incoterms data integration takes place between SD/MM and other SAP products that do not support Incoterms 2010 functionality. New Incoterms fields were added to SD and MM as part of these enhancements. Integration gaps may arise when other SAP products do not support these new fields. Make sure that other systems support Incoterms 2010 before activating these enhancements in SD and MM.

    For examples, check if the following SAP products support Incoterms 2010: CRM, Transportation Management, Extended Warehouse Management (EWM), Enterprise Services, Industry Solutions, etc.

    Note also that when this note was released, inbound delivery creations in general did not support Incoterms 2010, with exception of output type SPED generation.

Resolution

  • The creation of the inbound delivery does not support Incoterms 2010 functionality currently.

    Segment E1EDL20 was enhanced with new fields inco2_l and inco3_L, but this segment is used in multiple basic type and mostly in DELVRY0 basic type. This basic type is used in different IDoc message like SHPCON, WHSCON and DESADV. But no development were done to include a mapping of these fields in case of DESADV.

  • As described in SAP Help , the new incoterms 2010 were enhanced for the listed documents and only "Outbound Deliveries" is available in this list for the Logistic Execution area.

  • The nature of your request indicates that it is a development request issue rather than a software bug. Customer Support's under the SAP maintenance agreement, is to handle software or system defects in the standard delivered R/3 product.

    If you consider filing a development request, we recommend that you either check SAP Customer Connection or SAP Continuous Influence programs which focus specifically on improvement requests to SAP products.

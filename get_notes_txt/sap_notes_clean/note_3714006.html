Symptom

Open Connectors (OCN) SharePoint Connector authentication based on Microsoft Access Control Service (ACS) will stop working after April 2nd, 2026 due to Microsoft's retirement of ACS. All the SharePoint connections that use ACS authentication will be impacted.


Note: SharePoint connections using Graph/Azure AD authentication will continue to function, no action will be required.

You are impacted only if your SharePoint element instance is configured to use Azure ACS authentication. To verify:

  1. Navigate to your SharePoint element instance configuration.
  2. Locate the setting: "Use Graph Authentication"
    • false → Azure ACS authentication (Action Required)
    • true → Azure AD OAuth authentication (No Action Required)

After the retirement date, customers may observe the following issues:

  • SharePoint connector authentication failure with ACS
  • Failures in integration scenarios based on SharePoint Connector using ACS authentication

Reason and Prerequisites

Microsoft has announced the end of life (EOL) of Azure Access Control Service (ACS), which directly impacts SharePoint integrations that rely on legacy ACS authentication. More details about the retirement can be found in Microsoft official notification.

Solution

Please take following actions, if you are using Microsoft ACS authentication in SharePoint connector:

  • Migrate from ACS authentication to Azure AD OAuth authentication, which is the recommended way forward from Microsoft. Of course, you can keep using SharePoint connector.
  • Avoid using Microsoft ACS for creating new SharePoint Connector instances.

Movement Guide: Move from ACS to Azure AD OAuth

The SharePoint Connector uses the Azure AD OAuth v1 endpoint:

https://login.microsoftonline.com/common/oauth2/authorize

When using the v1 endpoint for Graph-based authentication, the scope parameter is ignored. The token audience is determined by the 'resource' parameter.

Important behavior:

  • resource= determines the token audience
  • scope= is ignored when using the v1 endpoint for Graph-based authentication
  • Permissions are determined by what is configured in the Azure AD application
  • The scope cannot be configured while creating the element instance

 

Step 1: Register an Application in Azure AD

  1. Go to Azure Portal.
  2. Navigate to Microsoft Entra ID → Manage → App registrations → New registration.
  3. Provide:
    • Name
    • Supported account type
    • Redirect URI: https://auth.cloudelements.io/oauth
  4. Click Register.


Step 2: Create a Client Secret

  1. Navigate to Certificates & Secrets → New client secret.
  2. Copy and securely store:
    • Application (Client) ID
    • Directory (Tenant) ID
    • Client Secret


Step 3: Configure API Permissions
Configure the required SharePoint or Microsoft Graph permissions in the Azure AD application.
Permissions granted in Azure AD determine what access the element instance receives.
Grant admin consent if required.


Step 4: Update SharePoint Element Instance Configuration
Update your existing SharePoint element instance with the following values:

  • Use graph authentication = true
  • API Key = Azure AD Application (Client) ID
  • API Secret = Client Secret

Note: The MS OAuth Scope field is not required for v1 Graph-based authentication

Save the configuration and re-authenticate the element instance.

Symptom

After executing the transaction that generated a report about query, Save button is greyed out in System>list>save output method. 

* Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.

There is a requirement to know if there is any way to enable the save button. 

Environment

SAP S/4HANA

Reproducing the Issue

  1. Execute a transaction that generated by query, like YQPXXXXA.
  2. System>list>save
  3. Save button is greyed out.

Cause

Resolution

  1. Confirm the query name from transaction code. 
  2. In SQ01 query definition, default output format can be defined.

  3. There is a list of output options on the selection screen of the query, where it could be switched to SAP list Viewer or ABAP list.



  4. If there is a requirement that these output options on the selection screen without changing the default, it should switch on the old report structure for the single query: Change mode for the query -> Menu: Extras -> Additional Special Attributes -> [X] Use old report structure.

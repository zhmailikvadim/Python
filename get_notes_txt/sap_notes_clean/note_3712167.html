Symptom

  • Tcode /SAPSLL/PR_CLWB_CUS displays the same product number for all line items in the output list after cancelling an action and returning to the output list screen.
  • Area and Numbering scheme are not displayed in the output list screen.

Environment

  • SAP Global Trade Services, edition for SAP HANA

Reproducing the Issue

  1. Execute transaction code /SAPSLL/PR_CLWB_CUS with a specific numbering scheme.
  2. In the output list screen, observe that Area and Numbering scheme are not displayed in the header/title.
  3. Click on "Classify Products" or "Product Master" button.
  4. Click "Cancel" button to return to the output list screen.
  5. Observe that the same product number is displayed for all entries in the output list.

Cause

  • For the specific numbering scheme, no entry exists in table /SAPSLL/TLEGSU with CTSIM or CTSEX being the numbering scheme.

Resolution

  1. Navigate to SPRO -> SAP Global Trade Services, edition for SAP HANA -> General Settings -> Numbering Schemes -> Assign Numbering Schemes to Legal Regulations.
  2. Make sure the numbering scheme is assigned to legal regulation.

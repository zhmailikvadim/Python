Symptom

  • Execution of DBCC LOG fails with error:

    1>  dbcc log(4,-15)
    2> go
    LOG SCAN DEFINITION:
            Database id : 4
            Backward scan: spid = 15
            Log operation type BEGINXACT (0)
            Forward scan: starting at beginning of log for all log records
            Find all qualifying log records.
    d_log(): xls_startscan() failed - XLS error -3: Invalid condition﻿

  • Second parameter passed as a negative spid value

Environment

  • SAP Adaptive Server Enterprise (ASE) 15.7
  • SAP Adaptive Server Enterprise (ASE) 16.0

Cause

CR 772605

Description: xls_startscan() failed - XLS error -3: Invalid condition" may be reported when executing DBCC LOG to get the log records for a particular SAP ASE process id when passing -<process ID> as the second parameter value of this command.

Fixed in:

  • ASE 15.7 SP132
  • ASE 15.7 SP133
  • ASE 16.0 GA PL05
  • ASE 16.0 SP01
  • ASE 16.0 SP02
  • ASE 16.0 CE SP01

The d_log() routine is incorrectly setting internal bits for both a forward scan and a backward scan of the log.

Resolution

Upgrade to version where CR 772605 is fixed.

Workaround:

Execute DBCC LOG with a negative number of log records.

For example:

﻿dbcc log (dbid, -spid, 0, -1000000)﻿

Symptom

  • Unable to locate the "credit released by" field in the status tables or transaction tables VBAK/LIKP.
  • User releasing sales order or delivery from credit block is not listed in a report or dedicated field.
  • Unable to track whether Credit Representative/Credit Analyst released a sales document or another user.

The below KBA provides explanation for both ERP Classic Credit Management and SAP Credit Management (FSCM).

Environment

  • Sales and Distribution (SD)
  • SAP Credit Management (FSCM)
  • SAP ERP
  • SAP ERP Central Component
  • SAP S/4HANA
  • SAP S/4HANA Cloud Private Edition

Reproducing the Issue

  1. Run a VKM* transaction or DCD to release the sales document from credit block.
  2. Review SD tables (VBAK, LIKP, VBUK) to identify the user who released the sales document. No fields are available for this purpose. 

Cause

There is no dedicated field in SD transaction or status tables to store the user ID, user name who released the credit block.

Resolution

  1. In ERP Classic Credit Management and SAP Credit Management (FSCM) scenarios, the user who released the sales document can be identified from the Change Log.
    • Locate the entry for "Overall status of credit checks changed" with value 'D' (= Released) to view the user ID.

  2. In SAP Credit Management (FSCM), it is possible to use the Documented Credit Decision (DCD) to find the "Release by User" information.
    • Display the DCD of the released document via transaction UKM_CASE or SCASE, and check the Internal Note under the Notes section. It shows the time of credit block release and the user who performed the action.
    • Add the Release/Rejection By column to the list of DCDs (list DCDs in UKM_CASE or SCASE -> Change Layout -> Add column).
    • Access table UKM_DCD_ATTR and review field DCD_COMPLETION_USER for this information.

Keywords

credit released by, VKM*, DCD, change log, FSCM, credit management, DCD_COMPLETION_USER

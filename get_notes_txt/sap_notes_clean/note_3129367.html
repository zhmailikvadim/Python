Symptom

Account settlement calculates wrong charge amounts for the first settlement period. This happens if the following conditions are fulfilled.

  1. There is a price condition that doesn't have date types.
  2. There is a charge condition type that has date types that match the settlement periodicity.
  3. The charge matrix of that charge condition has pro rata calculation or no calculation at all for the subperiod that includes the contract start
  4. The first settlement has been skipped (for instance because of a change of the settlement periodicity during extended account creation)

Other Terms

Matrix, Fee, First, Period.

Reason and Prerequisites

Program error.

Solution

Apply the correction or import the relevant support package.

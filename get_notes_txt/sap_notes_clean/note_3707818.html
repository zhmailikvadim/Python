Symptom

When a user attempts to access the Joule Analytics Center, it fails to load and displays an error message: 

Too little/no data is loaded for Product Usage. Start using Joule now!
Too little/no data is loaded for Scenario Usage. Start using Joule now!
Too little/no data is loaded for Interaction Type Usage. Start using Joule now!

"Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental."  

Environment

  • SAP BTP, Cloud Foundry runtime and environment
  • Joule

Resolution

Listing the reasons for not getting the data in Joule Analytics Center:

  • Joule Analytics Center will only show data when there are 8 or more individual users using Joule. This is due to legal requirements and to prevent employee monitoring and tracking or electronic surveillance.

  • For interaction types of Direct Response and Information Retrieval, there is no associated product or scenario. In this case, there won’t be any data for the related charts when you filter by Direct Response or Information Retrieval.

  • Joule Analytics Center only provides data beginning from the Joule Analytics Center release date. Data previous to this date cannot be retrieved.

Enabling and Using Joule Analytics Center:

  • Joule Analytics Center is a tenant specific dashboard that displays Product Usage, Scenario Usage, Interaction Type Usage and Client Type Usage. For further information, refer to Joule Analytics Center.

  • Joule supports several capability types: transactional, navigational, informational and analytical insights. Using available Joule Capabilities and use cases that are navigational or transactional (which will trigger the Joule analytics center report) determined based on the Integrated SAP Solution.

  • User can navigate to Joule Capabilities documentation to know the use cases and capability types available based on their Integrated SAP Solution with Joule.

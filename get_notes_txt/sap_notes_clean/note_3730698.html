Symptom

You are performing an SAP NetWeaver (SAP ERP) system upgrade or update.
During the 'Delete Backup Tables' phase, SUM process fails and Trouble Ticket log report following message:

-----------------------------------------------------------------------------------------------
An error occurred during the execution of Delete Backup Tables of action configure.
Service com.sap.sdt.j2ee.services.servicesimpl.DMTService failed with the following message:

Could not complete DMT process. Check the DMT.DROP_TABLE.LOG log file.
-----------------------------------------------------------------------------------------------


The  DMT.DROP_TABLE.LOG log file indicates:

Environment

  • SAP Systems based on SAP Netweaver;
  • Microsoft SQL Server 2012 and higher;

Reproducing the Issue

Execute SUM

Cause

The error messages above mean there are database triggers enabled in your MSSQL Server database.


Please be aware that SAP does not support customer-written triggers in SAP NetWeaver on SQL Server systems.
The use of customer written triggers is not supported in an SAP NetWeaver on SQL Server system. 

 

Please have your SAP NW DBA carefully read following SAP Notes:

 1252970 - Triggers on SAP tables

689152 - SQL Triggers and SAP NetWeaver on SQL Server


Triggers that are not delivered with SAP are not supported and SAP does not test installation, system copy and upgrade procedures along with customized Triggers, therefore, they might cause these maintenance options to fail.



Resolution

To fix the issue, before executing SUM you will have to disable your database SQL triggers.

See following note:

2916337 - How to find DDL Triggers in SQL Server by SQL Server Management Studio - NetWeaver

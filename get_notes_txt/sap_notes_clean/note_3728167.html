Symptom

When an account for which verification was triggered does not exist in the DM/AM system, the VoP service returns an incorrect combination of HTTP status code 500 and comparison result NOAP to the caller.

Other Terms

VoP, Verification of Payee, Account Verification, Responder scenario

Reason and Prerequisites

If an account does not exist in the DM/AM system, the ISO rejection reason code AC01 – "IncorrectAccountNumber" is assigned to the corresponding Request Agent used for processing the verification request.

Since rejection reason codes are generally interpreted as errors, AC01 was also treated as an error condition, leading to an HTTP status code 500 being returned.

However, according to the EPC VoP specification, the “incorrect account” scenario must be handled as a valid case. In this case, the service must return:

  • HTTP status code 200
  • comparison result NOAP

For all other rejection reason codes, HTTP status code 500 may still be used along with the appropriate error JSON structure.

Solution

Rejection Reason Code AC01 is excluded from the Reason Codes which cause an error. HTTP status code 200 is returning along with comparison result "NOAP".

Please implement provided correction instructions or corresponding Support Package.

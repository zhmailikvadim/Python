Symptom

This note describes the changes in Student Life Cycle Management(SLCM) -  United Kingdom(UK), for archiving which will enable you to adapt to the simplified archiving and deletion of Student/Business Partner based on SAP Information Lifecycle Management (ILM) as described in SAP Note 200796.

Other Terms

Data Privacy,Information Lifecycle Management, Information Life-Cycle Management, ILM, SAP ERP ,Deletion, Archiving, Personal Data, Student, Business Partner, HRP1782, HRP1783, HRT1783, HRP1785, HRT1785, HRP1786, HRT1786, HRP1795, HRP1796, HRT1796, HRP1933, HRP1934, HRT1934, PIQDB_GB_MODDATA, PIQDB_GB_NEEDS.

Reason and Prerequisites

You are using SLCM-UK Process.The use cases and requirements are derived from the data privacy regulations and the SAP data privacy product standard.

The SLCM-UK process that you are using, has application tables containing personal data and for which no ILM option was primarily defined, derives the requirements and use cases from the data privacy regulations and the SAP data privacy product standard.

Hence it becomes mandatory to archive these personal data and adhere to Data privacy and protection standard

For more information, see SAP Note 2007926

Solution

The new functionality for SLCM-UK, Data Privacy will be available with release:

SAP_APPL 617 SP 08

When a Student/Business Partner is blocked, the application does not show any personal data related to that Student/Bsiness Partner . You cannot display or change any information. You cannot create any new process  with that Student/Business Partner.

For Already Blocked Student/business Partner:

  • You can only archive/delete/destruct records
  • The application will not show any personal data related to the Student/Business Partner
  • You will not be able to create new Business with the blocked Student/Business Partner.

The application supports the ILM framework with below objects in place

  • The application supports the usage of application rule variants to enable the maintenance of residence and retention periods for ILM object HRIQUK_STM
  • For the mapping of application-specific condition fields to application rule variants you need to assign ILM rule groups to application rule variants. ILM rule groups can be defined for the ILM objects of SLCM-UK in transaction IRM_CUST.
  • The retention rules can be maintained for these ILM objects through transaction IRMPOL.
  • Archiving Object created for this fucntionality HRIQUK_STM.

NOTE :

The infostructure HRIQUK_STMD delivered through this note, enables reading of data from the Archived files. It is mandatory to activate the infostructure using the T-code SARJ before initiating the archiving process.

Symptom

This document explains how to create the 'perfect' ticket. A ticket that includes all necessary information so an engineer can provide a valuable response on the first interaction, improving issue clarity and reducing resolution time. 

Environment

  • SAP SuccessFactors Employee Central Integration to SAP Business Suite or S/4HANA
  • SAP SuccessFactors Employee Central Payroll

Resolution

  1. Issue Description:  

The current and expected behavior of the system must be clearly described. When the system does not behave as the customer expects, SAP Support needs detailed information about the customer’s requirements to fully understand the issue and provide an appropriate, tailored solution. See some examples:  

  • If a replication error occurs, include the full error message and the error ID (for example, from transaction SLG1 or the Data Replication Monitor). 

  • If data is not replicated as expected, specify the affected field(s) and provide both the current value and the expected value to be replicated. 

          2. Steps to reproduce: 

    To investigate, SAP Support needs a clear, reproducible set of steps with an example scenario. Provide the report (if any) and a variant that reproduces the problem for one representative employee or object and confirm that the assigned engineer has the customer’s authorization to reproduce the issue in test mode. 

    The steps to reproduce should answer the following questions: 

    • What report is executed to reproduce the issue? 

    • Which example employee/object can be used to test this? 

    • Can the engineer reproduce the provided steps in your system? 

     

           3. Environment:  

    Specify which systems are affected and how they are connected (e.g., EC → middleware → ECP/ERP). 

    For integrations, the backend system is often affected, so provide the ECP/ERP system ID and indicate required access. In case of on-premise systems, ensure the R/3 connection is open and provide credentials via a secure channel or secure area. 

    If the issue occurs in Employee Central (EC); for example, mashup/pay statement errors or data replication proxy issues, provide the EC instance ID and a user to access it as per KBA 2088892. 

     

          4. Other useful information:  

    Any additional details that could be related to the issue will help speed diagnosis and root-cause analysis. Useful items to include:  

    • Screenshots to illustrate the issue 

    • Guide or KBAs followed for reference 

    • Current Support Packages (EA_HR, SAP_HR, PA_SE_IN and/or ECS4HCM) of the system.  

    • Latest changes in the affected system or details of when the issue started 

    • Share how often the issue occurs (always, intermittently, or on a schedule) and any patterns observed. 

    • Business impact or priority (who/what is affected and how critical it is) and ensure the Case Contact assigned in the ticket is correct.  

     

         5. Examples of common scenario and how to report them:  

    1. Replication error in SLG1 or Data Replication Monitor: An error is spotted in SLG1 transaction while executing replication (master data/ org data/ time data):  

    1. Provide the replication error ID and message.  

    2. Provide the employee/object ID example that leads to this error. Provide a variant for the report executed. Confirm that the engineer has authorization to execute this variant for testing purposes.  

    3. Provide ERP/ECP system ID. In the case of an on-premise system, open R/3 connection and maintain credentials in secure area. If the error is for time replication, usually access to EC instance is needed so, provide the EC instance ID and a user to access it as per KBA 2088892. 
     

           B. Issue with the data flow from EC to ERP via CPI or to ERP from CPI: An error appears while executing the query, either in srtutil or in CPI:

    1. Provide the error that appears and the system in which it appears. 

    2. Provide the report(s) for which the connection is not working, and clarify whether this issue occurs in all executions or only when running a specific one (e.g., Last Modified Query). 

    3. Provide the ERP/ECP system ID. In the case of an on-premise system, open the R/3 connection and maintain credentials in secure area. Also provide the CPI URL so the issue can be checked from that side. Finally, clarify the details of the system configuration; for example, whether an SCC is used and how the connection has been set up.  

     

           C. Opening Pay Statement or Mashup in EC doesn’t work as expected, either an error or a blank page appears:  

    1. Provide the error that appears or share the current behavior. Clarify the application (e.g., Direct Pay Statement, Web Dynpro ABAP, IR8A form, etc.) failing and share the IdP being used: either SF IdP or IAS. 

    2. Share the employee to be checked and what application to click on employee’s profile. If proxy needs to be done, share the details of the employee.  

    3. Provide ERP/ECP system ID. In the case of an on-premise system, open R/3 connection and maintain credentials in secure area.  Also, provide the EC instance ID and a user to access it as per KBA 2088892. 

    4. Other useful information in this case are a screenshot of the issue and a SAML-trace as per KBA 2461862.

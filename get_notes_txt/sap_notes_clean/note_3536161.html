Symptom

  • Unable to refresh Web Intelligence document in three tier mode from Web Intelligence rich client tool below error message is observed:
    • "The following database error occurred: (CS) "Job already in use". For information about this error, please refer to SAP Knowledge Base Article 2054721 on the SAP Support Portal. (IES 10901)" 
        

Environment

  • SAP BusinessObjects Business Intelligence Platform 4.x
  • Web Intelligence Rich Client (WRC) in 3-tier mode
  • Web Intelligence

Reproducing the Issue

  1. Launch Web intelligence rich client in three tier mode.
  2. Open a Web Intelligence document and refresh.
  3. The document fails with below error.  
    • "The following database error occurred: (CS) "Job already in use" . For information about this error, please refer to SAP Knowledge Base Article 2054721 on the SAP Support Portal. (IES 10901)"
        

Cause

  • It was discovered during the investigation of this matter that the issue may be arising from the disabling of the 'asynchronous call' for Web Application Server.

Resolution

  1. Stop Tomcat.
  2. Navigate to below mentioned tomcat directory and take backup to tomcat cache, copy all available folder outside BOBJ directory for backup purpose.
    • <tomcatFolder>\work\Catalina
        
  3. Navigate to below directory and take a back-up of 'wstk_bridge.xml' file.
    • <tomcatFolder>\webapps\BOE\WEB-INF\eclipse\plugins\webpath.AnalyticalReporting\web\WEB-INF\classes
        
  4. Navigate to back to below directory and open 'wstk_bridge.xml' file and find insert the highlighted text mentioned below and save the xml file. 
    • File location: <tomcatFolder>\webapps\BOE\WEB-INF\eclipse\plugins\webpath.AnalyticalReporting\web\WEB-INF\classes
    • Entry:
      • <bridge name="HTTPRepositoryJob">
                    <impl type="java" class="com.sap.connectivity.cs.remoting.http.bridge.RepositoryJobImpl" />
                    <impl type="COM"  class="BusinessObjects.CSBridge.HttpRepositoryJob.1" />
                    <async_methods>
                        <async_method name="Call" enable="yes" />
                    </async_methods>
                </bridge> 
        
  5. Now, navigate to below location and delete 'localhost' folder.
    • <tomcatFolder>\work\Catalina
        
  6. Start Tomcat.

Please note below points: 

  • It is required to take backup of suggested files before performing any step.
  • Please implemented above mentioned steps during non-business hours as this steps involves deletion of tomcat cache and cache take times to be regenerated. 
  • If You are having multiple tomcat nodes then steps needs to be performed on each tomcat node.

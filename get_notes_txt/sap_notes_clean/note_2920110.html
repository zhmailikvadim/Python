Symptom

You have used 1709 (or lower) release and had closed action tasks.

As part of your upgrade to 1809 or 1909 release, the closed action tasks were migrated to the new task management. These tasks kept their original workflow templates.

As part of the task migration, you run program R_EHFND_WFF_RESTART_PROCESSES with the Migrate Planned Task Instances option selected. However, it does not process the closed action tasks.

After applying the corrections in this SAP Note, you can rerun the program with the Migrate Planned Task Instances option selected to complete the migration of such tasks.

If you have custom workflow templates with custom steps for either tasks or actions, you need to add these custom steps to attributes MT_EXEC_STEP_TEMPLATE_RANGE (for custom steps executed by the task assignee) and/or MT_APPR_STEP_TEMPLATE_RANGE (for steps executed by the task approver) of class CL_EHFND_TASK_PLAN_MIGRATOR. You can do this via post-exit method for the constructor of the class.

Other Terms

Task Migration

Reason and Prerequisites

Reason: Program error.

Prerequisites: The validity of the correction is maintained in the correction instructions.

Solution

The Support Packages that contain this correction are listed in the "Support Packages" section.
Alternatively, you can implement the correction instructions of this SAP Note.

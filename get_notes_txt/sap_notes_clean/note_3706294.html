Symptom

  • Globe icon for`externalName` field translatable missing in manage Data 
  • Missing Translation Globe icon beside External Name Field for MDF Object Under Manage Data

"Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental."

Environment

  • SAP SuccessFactors HXM Suite
  • Metadata Framework (MDF)
  • SAP SuccessFactors Employee Central

Reproducing the Issue

  1. Go to Manage Data under any MDF Object Definition.
  2. Make corrections or insert a new record.
  3. Select  globe icon in the description field.
  4. Result: Missing : global

Cause

The `externalName` field From Object Definition was changed from Data type: string to translatable, which caused the issue.

Resolution

Create a New UI Via Manage configure UI after changing any MDF Object Field Data Type

Step 1: Review  Example : Object Definition: Career Level

  1. Object Definition: Career Level
  2. Field Name / Database Field Name : externalName 
  3. Label: Career Level Name 
  4. Data Type: Translatable

Step 2: Create a New UI Via Manage configure UI

  1. Navigate to Manage Configuration UI
  2. Create a new UI configuration.
  3. Ensure the following configurations:

Example:

Step 4:  Add new UI Under Default Screen

  • Example: Object Definition: Career Level (`cust_Career_Level`) - Default Screen: BestCareerLevel.

Step 3: Check New UI

  1. Go to Manage Data.
  2. Insert new records 
  3. Select any Object Definition
  4. The globe icon appears beside the `externalName` field in Manage Data

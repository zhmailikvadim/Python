Symptom

After completing the COGS revaluation run(COGS Revaluation document) in T-code: CKMLCP, several COPA characteristics are not updated in the PSG screen of the FI document item..  such as, 

- Customer Group
- Sales Office

However, the original delivery document contains all these characteristics correctly.

Environment

  • SAP ERP Central Component
  • SAP ERP
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA Finance
  • SAP S/4HANA

Reproducing the Issue

  1. Go to T-code: FB03 display the FI document that created from T-code: CKMLCP
  2. Double click the item of FI document => click "PSG" button=>check the characteristics value, several characteristics shows blank.

Cause

In T-code:S_ALR_87008135, movement type group 'CF' is assigned in ML for goods issues for deliveries, i.e. the consumption (and therefore also the revaluation during CKMLCP) will be recorded in ML under GL account only ignoring the original CO account assignment object which is the profitability segment. So no derivation will be trigged. 

Resolution

To get the original profitability segment revaluated please assign movement type group 'CC'. Respective change will affect future goods issue for deliveries and subsequent ML closing; i.e. already existing consumption alternatives will not get affected and therefore this customizing change cannot be used to correct any CKMLCP post closing.  

Furthermore this will of course have negative impact on CKMLCP post closing runtime and ABAP memory requirements. Therefore carefully test this before changing this in production.

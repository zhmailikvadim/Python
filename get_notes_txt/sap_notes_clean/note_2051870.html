Symptom

The error message delivered with SAP Note 1688971 ("Missing termin. for retro acctg before earliest RA period") checks whether a retroactive accounting lock is preventing the system from performing the necessary retroactive accounting. To prevent the payroll program from terminating, you must also enter the basic remuneration of the adjustment amount (model wage type MBV3) manually or enter the cumulation of the basic remuneration (model wage type MBV6) using infotype 0014 (Recurring Payments/Deductions) for each period of the prohibited activities. This entry is insufficient if variable payments or portions of the variable payments for the leave and sick pay premium exist in the assessment period for calculating the prohibited activities, and if the assessment period partly lies before the earliest personal retroactive accounting. As a result, the system can no longer determine the adjustment amount for variable payments and a claim arises.

Other Terms

Adjustment amount

Solution

Import the relevant HR Support Package or proceed in accordance with the correction instructions. See SAP Note 1232082 for the connection between the Support Package and the technical name specified in the SAP Note under "Support Packages". An advance correction is possible.

The correction is controlled using the new subapplication MGBP. The subapplication MGBP is inactive in the standard delivery. You can activate the subapplication as of any date (using the table T596D) or on a differentiated basis using the BAdI HRPAY00_B_APPL_VALIDITY for which a sample implementation is available for the filter 01/MGBP.

If the subapplication MGBP is activated, the system checks whether you manually entered all relevant wage types using infotype 0014 (Recurring Payments/Deductions) for each period of the prohibited activities:

  • For the wage type /6H1 (Cumul. Var. Paymt ProA), you must manually enter the variable payments (sample wage type MBV4) or the cumulation of the variable payments (sample wage type MBV7).
  • For the wage type /6H2 (Cumul. Contr.VP/LSP ProA), you must manually enter the portion of the variable payments for the leave and sick pay premium (sample wage type MBV5) or the cumulation of the portion of the variable payments (sample wage type MBV8) for the leave and sick pay premium.

If this is not the case, payroll terminates due to the missing wage types. The termination occurs only if, due a subsequently set retroactive accounting lock (by means of the payroll control record or the earliest personal retroactive accounting), retroactive accounting is no longer performed for the applicable assessment period for the prohibited activities.

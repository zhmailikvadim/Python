Symptom

  • An error occurs in transaction ME2DP when attempting to process purchase orders.
  • Error message displayed: "FPDP_DOWN_PAYMENTS018 Tax code must remain blank if jurisdiction code is active."
  • The jurisdiction code cannot be removed in the purchase order.

Environment

  • Materials Management (MM)
  • SAP ERP Central Component
  • SAP ERP
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA

Reproducing the Issue

  1. Attempt to process the purchase orders in transaction ME2DP.
  2. Observe the error message: "FPDP_DOWN_PAYMENTS018 Tax code must remain blank if jurisdiction code is active."

Cause

The issue occurs because the system prohibits the use of both tax codes and jurisdiction codes simultaneously when jurisdiction codes are active. This design prevents a double-tax scenario. Additionally, when a company code is configured for jurisdiction tax, the tax jurisdiction code becomes mandatory at the purchase order level and cannot be removed.

Resolution

  • SAP does not support down payment postings with tax jurisdiction codes. 
  • According to the official description: If Tax Jurisdiction Code is active for the Company Code (common in US tax scenarios), Then the tax code field must stay empty when posting Down Payments (FPDP_*) transactions.
  • Because when jurisdiction codes are active, SAP determines taxes exclusively through the tax jurisdiction structure, not the standard tax code (MWSKZ). The system prohibits entering both because it would create a double-tax scenario.This design is consistent across all MM/FI integrations involving jurisdiction‑enabled company codes.
  • The rule applies globally: If jurisdiction codes are active, ME2DP (Down Payment Monitor) cannot use a tax code.
  • it is normal and expected that ME2DP cannot process down payment items with MWSKZ filled when jurisdiction codes are active.

On the other hand you can’t remove the jurisdiction code in the PO because when a Company Code is configured for Jurisdiction Tax (US Sales/Use tax), system requires tax jurisdiction code at PO level and does not allow users to remove it. This is documented behavior in similar jurisdiction‑related issues, such as mandatory tax jurisdiction code requirements in billing

In summary:

You cannot remove the jurisdiction code from the PO because it is mandatory at master-data and configuration level when US tax jurisdiction is active.

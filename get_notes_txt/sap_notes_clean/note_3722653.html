Symptom

Error FAA_POST014 (Posting is not possible (asset XXXX YYYYY-0 does not manage areas to be posted) arises in transaction AR29N (report RAAUFW02) when testing to create a new revaluation area.

Environment

  • FIN Asset Accounting
  • SAP S/4HANA
  • SAP S/4HANA Finance
  • SAP S/4HANA Cloud Private Edition

Reproducing the Issue

  1. Create a new, area with revaluation purpose for the chart of depreciation and the company code.
  2. Run AR29N transaction to add the new area in the assets with the options: 
  • Create Revaluation Area
  • Test Run

    Cause

    In test mode the revaluation areas are not created, therefore error FAA_POST014 is triggered. 

    Resolution

    In order to create a new revaluation area to the assets, please run AR29N with following options:

    • Create Revaluation Area
    • No posting

    and run the transaction in real mode in background. 

    See Also

    KBA 3592126 - Creation of Revaluation Area for Assets for non-capitalized assets.

    Keywords

    FAA_POST014, AR29N, revaluation area, revaluation measure, RAAUFW02

Symptom

Using a Replication flow to export a Datasphere object to SFTP csv file, it results in a split of several files.

Environment

SAP Datasphere

Reproducing the Issue

  1. Configure a Replication Flow to export a Datasphere object to SFTP in CSV format.
  2. Run the Replication Flow and observe that the output is split into multiple CSV files instead of a single one.

Cause

The behavior is by design: Replication Flow currently splits CSV output into multiple files and does not provide a setting to control or enforce a single-file export. The number and size of these files depend on the source table size and structure as well as change frequency (during delta loading).

Resolution

  • Note the current product limitation: controlling the number of files for SFTP CSV exports in Replication Flow is not supported.
  • To request support for single-file exports or related controls, submit an enhancement request via the influence channel: Influence portal for Datasphere.

Symptom

At integrating between SAP Analytics Cloud and SAP S/4HANA systems using S/4HANA Apps, the posted data may have inconsistences for amounts using some currencies. For example, amounts in currency JPY in S/4HANA appear 100 times higher than expected. The issue is arizing when mapping rules in configuration include amount fields linked to particular currency keys which do not have 2 digits decimals representation (the number of decimals is defined in TCURX table). The described issue is relevant for Target Types "Plan Data Line Items" (ACDOCP), "Plan Cost Rates" (ACCOSTRATE) and "Activity Type Cost Rate Component Split" (ATPCRCS).

Other Terms

SAC Pull, SAC_INT_PULL, Manage Financial Plan Data Jobs for SAP Analytics Cloud Integration, Schedule Financial Plan Data Jobs for SAP Analytics Cloud Integration, Plan Data Line Items, ACDOCP, Plan Cost Rates, ACCOSTRATE, Activity Type Cost Rate Component Split, ATPCRCS

Reason and Prerequisites

 Program error. Missing conversion for amounts with regard to number of decimals from TCURX table.

Solution

Upgrade to corresponding SPS or implement attached SAP Note.

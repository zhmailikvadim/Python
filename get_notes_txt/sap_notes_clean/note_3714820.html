Symptom

Currently when adding a concurrent employment for Contingent Workers (via Add Assignment) the Original Start Date field is set to match the Hire Date of the new employment.
In historic Concurrent and Rehired Contingent Worker Employments we can see that the 'Original Start Date' was set to be equal to the Hire Date of the users first employment.

Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental. 

Environment

SAP SuccessFactors - Employee Central 2511 - Contingent Workforce

Reproducing the Issue

  1. Add a Concurrent Employment for a Contingent Worker (Add Assignment)
  2. After the assignment has been added go to the users profile and verify the 'Original Start Date' value maintained.
  3. The 'Original Start Date' on the new employment will be equal to the Hire date for the new employment (Hire Date and Original Start Date will match):



  4. If you go to the profile of a contingent worker with multiple employments where the 2nd employment (concurrent or rehire) was added before the 2511 release the 'Original Start Date' value will be equal to hire date of the first employment:

Cause

Our engineering team identified the previous behaviour for the Original Start Date field as a defect.

When concurrent employment is enabled, rehire always happens on a new employment.
In this case original start date of the employment must be set to Work Order start date.

Resolution

The fix for this defect (ECT-255762) was delivered as part of the 2511 product release.
The 'Original Start Date' for rehired/concurrent Contingent Workers should be equal to the start date of the work order (hire date of the new employment)

Please see the KBA below for more information on the intended behaviour of the 'Original Start Date' field:
2080126 - How do Employment Information > Hire Date and Original Start Date fields work? - Employee Central

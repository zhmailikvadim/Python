Symptom

Whenever the following query runs in Data Consolidator (DC) cleanup, it takes around 70 hours to complete:

 delete from DC_Account where DCAccountID in (select distinct dca.DCAccountID from (DC_Account dca left outer join Accounts a on dca.AccountID = a.account_id and dca.RoutingNumber = a.routing_num) where a.account_id is null)

Each row takes around 14 minutes to delete. DC_Account has cascade deletes to around 13 tables and delete on each row of these tables activates the triggers. Please check if the query can be optimized.

Environment

  • Sybase Corporate Online Banking 8.2
  • Oracle Database 11

Cause

SAP Product Development has reviewed the SQL Query identified and has determined that there is no way to optimize the query.  However, SAP Product Development has also identified that the Data Consolidator (DC) Purge Process is not being run by the customer on a regularly scheduled basis.

Resolution

Ensure that the Data Consolidator (DC) Purge is run on a regular basis.  Ideally, SAP Product Support and SAP Product Development recommend that the DC Purge be run daily to minimize the amount of SQL activity required to clean-up old DC data.

Keywords

Data Consolidator, DC, Cleanup, Oracle, long, running

Symptom

  • How to map corporate business address country field to the employee profile  country field ?
  • Which Employee Central HRIS Element Populates the Country Field in the Employee Profile?

Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.

Environment

  • SAP SuccessFactors HCM Suite
  • SAP SuccessFactors Employee Central -HRIS Sync

Cause

Resolution

The HRIS sync mapping for the Country is Fields Hard-Coded in the system using the Coprorate Address of the Location (Job Information > Location > Corporate Address > Country).

The system synchronizes some Employee Central data based on the hard-coded sync mappings from HRIS fields to standard and userinfo elements.

To avoid unexpected behaviors, don't delete nor duplicate hard-coded sync mappings.

Recommendation

  • We strongly recommend that you do not override hard-coded HRIS Sync mappings in the Succession Data Model. But if your business processes require the override, note the cautions and rules while adding custom HRIS Sync mappings.

Employee central Record:

c

Check hard-coded in the system :HRIS sync From Employee Central ( EC ) to Employee Profile (EP )

Note: The HRIS sync mapping for the Country is hard-coded in the system using the Coprorate Address of the Location

Note:

Another option is to synchronize the "Country of Company" (which is derived from the Country of Registration value defined on the Legal Entity) to the Country field in the Platform, but this need to be configured.

Please check this KBA for more details. 2618726 - Syncing the Country of Company between Employee Central and the SuccessFactors Platform.

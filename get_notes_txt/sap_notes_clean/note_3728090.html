Symptom

In a self-billing scenario, the system creates a consignment issue order from the customer’s Delivery Confirmation, followed by the corresponding delivery and invoice.

During invoicing, the system is expected to validate the JIT Action Control settings. However, even when the configuration maintained in JIT Action Control is incorrect or incomplete, the system still allows the invoice to be created, leading to inconsistencies.

Other Terms

NJIT, Next Generation Just-In-Time Processing, JIT Delivery Confirmation, Self Billing, Billing document, Action Control

Reason and Prerequisites

Program error

Solution

Implement the correction instructions of this SAP Note or import the relevant Support Package.

With this fix:

  • The billing process now performs a mandatory validation of the maintained JIT Action Control configuration when invoicing a consignment delivery created from a JIT Delivery Confirmation.
  • If the configuration is missing, incomplete, or inconsistent, the system prevents invoice creation and issues an appropriate error message.
  • This ensures that billing can only proceed when the JIT‑relevant configuration is valid, thereby avoiding inconsistencies in subsequent self‑billing and JIT processing flows.

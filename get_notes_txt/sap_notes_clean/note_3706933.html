Symptom

Upgrading to build 2.00.079.06 (fa/hana2sp07, “79.06”) can cause existing Flowgraphs to fail execution and causes newly created Flowgraphs to generate invalid code for Aggregation Node outputs. The root symptom is that the Aggregation node’s expression attribute is no longer treated as valid in the Flowgraph editor’s generated code. This is a defect that may block flowgraphs that previously worked.

  • Updated Flowgraphs (created before patching, then updated after patching) fail to execute.
  • New Flowgraphs: when columns are added to nodes upstream of an Aggregation Node, those columns are automatically included in the Aggregation Node’s outputs but the generated expression is faulty (invalid or incorrect).
  • In the Flowgraph editor, generated code no longer includes or uses the AggregationNode/outputs expression attribute as it did in prior versions.
  • Error messages or stack traces in the Flowgraph execution logs indicating invalid expression or missing/ignored expression elements

Other Terms

Flowgraph, Aggregation

Environment:

SAP HANA 2.00.079.06 (fa/hana2sp07)

Steps to Reproduce:

  1. On a system running a version prior to 2.00.079.06.1754480252, create a Flowgraph containing an Aggregation Node. Confirm aggregation output expressions appear in the editor-generated code.
  2. Patch the system to 2.00.079.06.1754480252 (79.06).
  3. Update an existing Flowgraph (e.g., change source/target tables or update synonyms) — note that the Flowgraph can no longer be executed.
  4. Alternatively, create a new Flowgraph on 79.06 and add columns to nodes upstream of the Aggregation Node. Observe that those columns are automatically added to the Aggregation Node outputs and the generated expression is invalid or missing in the generated code.

Solution

Workaround: If the expression attribute is removed, the Flowgraphs run successfully as before.

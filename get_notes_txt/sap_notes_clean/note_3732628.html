Symptom

  • sso configuration test succeeds, but users cannot sign in.
  • using an internal sso link shows the message: "Details: No additional information is available at this time."
  • selecting “choose a different workspace” leads to a page showing: "No webpage was found for the web address".
  • accessing the base subdomain shows available workspaces; selecting one redirects to the legacy workspace selection instead of completing sso.

Environment

  • SAP LeanIX solutions
  • SAP LeanIX MTM Single Sign-on

Reproducing the Issue

  1. Configure a new SSO identity provider for multiple workspaces that reside under a shared subdomain.
  2. Attempt login via an internal SSO link or the base subdomain entry point.
  3. Observe that the SSO test succeeds, but users either receive a generic error or are redirected to workspace selection without completing SSO.

Cause

  • The new identity providers were not associated with the parent subdomain that hosts the workspaces. For subdomain-based setups, IDPs must be added at the subdomain level so users can authenticate. This requires creating the ticket for LeanIX Support, as described in the documentation: https://help.sap.com/docs/leanix/ea/migration-to-self-service-sso-configuration#step-6:-(optional)-request-to-add-idp-connection-to-a-custom-subdomain

Resolution

  • Submit a request to LeanIX Support to add/map the new identity providers to the parent subdomain that hosts the workspaces (not only to individual workspaces).
  • Once the sub-domain is configured, observe the login workflow via the sub-domain URL:
    • You will be redirected to the instance URL with the selection of the IDP listed 
    • Once selected teh IDP, you will be directed to the Workspace selection
    • Once the desired Workspace is selected, you will be logged in.
  • Optionally update identity provider display names under Administration > Authentication and SSO > Identity Provider; note that providers are listed alphabetically.
  • Share direct workspace links with users so they land in a specific workspace after SSO.

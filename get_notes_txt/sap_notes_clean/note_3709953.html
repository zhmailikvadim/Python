Symptom

Comparison of Derivation objects were failing as the logic is written in client side. Due to large data processing it is failing.

There are two buttons introduced such as 'Run Compare' and 'View Comparison'.

On selection of two Data derivations under Subscription, the respective button gets Enabled based on below conditions.

'Run Compare' : If the Data Derivation is not compared between the two selected derivations, then this button gets Enabled. On clicking, it executes the comparison which takes around 30 seconds for 1 Million objects in total and stored in Backend.

'View Comparison': If the Data Derivation comparison is already executed. Then this button can be used to view the Comparison results in new screen (SAP GUI)

If there are more than 1 Million objects, then the Report 'DFS_SUBS_DD_COMPARE' can be used to execute the comparison, as there will be timeout error in frontend.

Other Terms

Disconnected Operations, Manage and Monitor Subscription application, Object based navigation, S4 Defense Disconnected, Data Derivation, Staging

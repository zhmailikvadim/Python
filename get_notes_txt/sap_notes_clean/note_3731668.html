Symptom

  • Archiving large volumes of SFC (Shop Floor Control) data fails or performs poorly
  • High memory consumption during archiving jobs
  • Long-running or stuck archiving processes
  • System instability when processing large archive batches

Environment

  • SAP Manufacturing Execution (SAP ME)

Cause

When a large number of SFC records are queued for archiving, but they are not uniformly aged. Each record has been in the queue for a different duration.

For example:

  • Some SFCs may have been queued for 10 days
  • Others for 200 days
  • Others for 800 days or more than 3000 days

If the system retention period is configured as default (365 days in Days Before Archiving SFC/SO  at System Rule Maintenance) , then a very large dataset becomes eligible for archiving at once.

This results in:

  • Massive data volume processed in a single archiving run
  • High parallel processing load
  • Increased memory consumption
  • Higher likelihood of job failure

Resolution

To improve archiving performance and stability, adjust the retention period in System Rule Maintenance strategically.

Recommended Approach:

Instead of archiving all eligible data at once:

  1. Reduce the retention period incrementally.
  2. Process smaller batches of data.
  3. Run archiving jobs iteratively.

Example: Adjust the retention period from the default value to a higher value that is close to the age of the oldest data. For example, if the oldest SFC has been waiting for 1500 days, set the ‘Days Before Archiving SFC/SO’ parameter to 1400 days. This ensures that the system processes a smaller volume of data, increasing the likelihood of a successful, error-free execution. After the initial run, gradually decrease the value (e.g., to 1300 days) as the oldest remaining data shifts (e.g., to 1399 days), and continue this step-by-step approach to archive the data in manageable batches.

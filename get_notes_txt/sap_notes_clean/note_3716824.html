Symptom

The request to re-read and update NI ACL triggered in the transaction SMGW by the command "Re-read NI ACL globally" in the menu Goto -> Expert functions -> External Security reaches all servers in status "Active", but none of servers in status "Passive".

Other Terms

NI ACL trusted IP addresses

Reason and Prerequisites

Adjustment

Solution

Install the specified SAP_BASIS support package or apply the corresponding correction instructions.

Workaround

An update of NI ACL of servers in status "Passive" is still possible. Switch to that server via /nSM51, start /nSMGW and execute the command "Re-read NI ACL".

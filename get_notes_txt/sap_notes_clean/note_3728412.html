Symptom

  • NPS employer contribution wage types (/3FS – Monthly ER NPS Contribution, /3FT - Annual ER NPS Contribution) are not generated in the hiring month. From next month onwards it has been generated.

Environment

  • Human Capital Management (HCM)
  • SAP R/3
  • SAP ERP Central Component
  • SAP ERP
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • Payroll India

Reproducing the Issue

  1. Go to PA40 t-code.
  2. Hire an employee in mid of the month.
  3. Go to PA30 t-code. Pass the PERNR and infotype 587.
  4. Click on the create button.
  5. Fill all the necessary details along with that tick the check-box applicable for 'Employer NPS' and pass the percentage value.
  6. Run the payroll from the payroll driver for this newly hired employee.
  7. Observe that wage types /3F3 and /3FS are not generated.

Cause

Incorrect data maintenance on info-type 587 level.

Resolution

  1. Go to PA30 t-code.
  2. Pass the pernr details and info-type number as 587.
  3. Click on the overview button.
  4. Select the first entry from bottom.
  5. Modify the start date (begda) of info-type 587. Start-date of the info-type should match with the hiring date.

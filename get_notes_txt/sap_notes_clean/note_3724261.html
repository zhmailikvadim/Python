Symptom

The IDOC is split into different SESs based on the account assignment category. If, during posting, one of the SESs is successfully posted, the table /ASADEV/ACI_TSRF is updated with this value. If later in the process some SES posting fails, the process will roll back, the IDOC status will be changed, but the posted SES will be incorrectly saved in the table /ASADEV/ACI_TSRF. All SESs must be revoked in this case; no entry must exist in the table for this TS ID. The behavior is valid for NO COMMIT posting scenario.

Other Terms

 Class /ASADEV/CL_ACI_SES_MULTI_LINE method _POST_SERVICE_ENTRIES

Reason and Prerequisites

The issue happened in the standard logic

Solution

Implement the note to stop the update of table /ASADEV/ACI_TSRF in case of posting SESs with multi account assignment when one of the SES failed with usage of NO COMMIT posting scenario

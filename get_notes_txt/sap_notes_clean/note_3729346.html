Symptom

  • The Cash Account ID within the XML Tag - ` ` in the CAMT.086 bank fee file contains leading zeros, which impacts its accurate interpretation.
  • The ABA number in the CAMT.086 file is a generic one provided by the bank and may not correspond to the ABA number configured in the House Bank Account. It is unclear whether the Import Bank Services Billing Files Fiori app uses table T028Q or a similar table for cross-mapping.
  • The Receiver ID is missing in the CAMT.086 file, resulting in an error indicating that the Receiver ID is mandatory.

Environment

  • Bank Account Management

Reproducing the Issue

Import the CAMT.086 bank fee file using the standard Fiori App Import Bank Service Billing Files.

Cause

  • The Cash Account ID in the CAMT.086 file contains leading zeros, which impacts its accurate interpretation.
  • The ABA number in the CAMT.086 file does not match the ABA number configured in the House Bank Account.
  • The Receiver ID is a mandatory field in the CAMT.086 file as per the Bank Services Billing (BSB) standard defined under ISO 20022, and its absence causes an error during file processing.

Resolution

To handle leading zeros in the Cash Account ID:

  • Implement the standard enhancement framework provided for Bank Fee Management.
  • Normalize the Cash Account ID during file processing before account matching occurs.
  • Alternatively, use the alternate account number field in table FCLM_BAM_ACLINK2 (field: BNKN2) under the HB connectivity section to maintain the account number without leading zeros.
  • For implicit enhancement, use the following method to handle the leading zero logic:
    Class: `CL_FCLM_BRM_BANKFEE_UTIL`
    Method: `CHECK_ACCOUNT_EXIST`

2. To address the generic ABA number mapping:
Use the enhancement point in the CAMT.086 processing flow to intercept the incoming ABA number, perform custom cross-mapping to the internal ABA number, and replace it before account determination logic is executed.

3. To resolve the missing Receiver ID issue:

  • Contact the bank to provide the Receiver ID in the CAMT.086 file, as it is a mandatory field required by the BSB standard.
  • Note that there are no enhancement spots available for the class `CL_FCLM_BRM_XML_PARSER` and method `CHECK_BSB_FORMAT_MAPPING` to bypass this requirement.

Symptom

When a user account in the Integration Scenario Configuration is assigned with Logon Data and configured to use X.509 certificate authentication—applicable exclusively to HTTPS connections—the Pending Tasks will fail, displaying the errors:

  • User account <Logon Data> is assigned.
    Could not create a valid configuration.

Logical ports are created using an HTTP WSDL, making it more common to configure the username and password in Logon Data Management, as this is the expected method for HTTP connections.

Environment

  • ABAP Web Services
  • SAP NetWeaver
  • SAP NetWeaver Application Server for SAP S/4HANA
  • ABAP PLATFORM - Application Server ABAP
  • SAP S/4HANA Cloud Private Edition

Cause

The access to the WSDL file does not require X.509 certificate authentication. When the WSDL file is retrieved, it is accessed using a username and password. However, logical ports for HTTPS and X.509 certificate authentication can still be configured.

Resolution

If access to the WSDL file using certificate authentication is preferred, the WSDL file can be uploaded, and the X.509 authentication method can be set in the Services Registry within SOAMANAGER. To achieve this, the following steps should be followed:

  1. Go to SOAMANAGER,
  2. Navigate to the Service Registry tab,
  3. Click on the Publication of External WSDL option,
  4. Choose the External checkbox,
  5. Provide the WSDL URL and select X.509 authentication,
  6. Select the required SSL Client PSE.

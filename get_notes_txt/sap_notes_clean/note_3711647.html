Symptom

Error while legacy Import of contracts Of ContractId instances with ='[BaseId]', one has DocumentId [BaseId], but another has (missing argument)

Resolution

To solve this error, check the points below:

  • Open all the CSV file in Notepad and scroll to the bottom of the file. Check to see if there are rows of empty data with just commas. If so, delete all the empty comma rows.
  • This can also happen if a previous load file is used and the previous file had more rows of data than the current file.
  • The Contract ID already exists in Ariba. Please check the Contract ID used in the contracts.csv file to see if that CW ID already exists in Ariba.
  • Do not import the same file used in the past for legacy import, as this tool is not used for updating existing contracts.

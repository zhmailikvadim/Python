Symptom

  • Initiative field values are not populated when using RPM_DX_INITIATIVE to create an initiative using a template.
  • The initiative is created with blank values for fields if the field value is blank in the template, even though the Excel file specifies values for these fields.
  • The same response can be observed in program RPM_DX_ITEM for portfolio items.

Environment

  • SAP Portfolio and Project Management
  • SAP Portfolio and Project Management for SAP S/4HANA
  • SAP S/4HANA
  • SAP S/4HANA Cloud Private Edition 

Reproducing the Issue

  1. Create an initiative using RPM_DX_INITIATIVE while specifying an initiative template.
  2. Ensure the Excel file contains a value for a field.
  3. Ensure the initiative template does not contain a value for the same field.
  4. Observe that the initiative is created with blank value for the field.

Cause

The program response is correct per the program design. If the user creates an initiative by specifying an initiative template, field values should be copied from the template to the initiative (including blank values).

Resolution

Workarounds:

  • Create multiple initiative templates that include the required default values, and select the appropriate template when running RPM_DX_INITIATIVE.
  • Or create the initiative in two steps: in the first step run RPM_DX_INITIATIVE in CREATE mode to create the initiative (this will populate field values based on the initiative template), in the second step run RPM_DX_INITIATIVE in CHANGE mode to update any fields that were left blank by the template.

The same applies to the creation of portfolio items via program RPM_DX_ITEM.

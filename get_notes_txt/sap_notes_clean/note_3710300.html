Symptom

  • After upgrading the HANA instance to a high SPS7 revision or SPS8, backups fail
  • Error message observed: "[447] backup could not be completed, [110164] Encryption root keys backup password doesn't exist. It must be either set in the database or must be given with the backup/recover SQL statement."

Environment

SAP HANA Platform Edition

Reproducing the Issue

  1. Perform an upgrade of SAP HANA systems.
  2. Attempt to perform backups on the upgraded SAP HANA system.
  3. Backups fail with the error message: "[447] backup could not be completed, [110164] Encryption root keys backup password doesn't exist. It must be either set in the database or must be given with the backup/recover SQL statement."

Cause

The activation of encryption parameters is a new default behavior introduced during SAP HANA upgrades for HANA2 SP07 revision 79.02 and above, as per SAP Note 3431564 and SAP Note 3539252 - HANA Backups Protected by Encryption Root Keys Backup Password When Local Secure Store (LSS) is Active

Resolution

    If encryption is required, set the encryption root keys backup password using the following SQL command:

    `ALTER SYSTEM SET ENCRYPTION ROOT KEYS BACKUP PASSWORD '';`

    1. Refer to SAP Note 3571561 for detailed instructions on setting the encryption root keys backup password.
    2. Ensure the password is securely stored in a separate safe location, as it will be required for restoring the secure store content before database recovery.
    3. If encryption is permanently disabled, note that data at rest will not be encrypted, which may pose a security risk. Evaluate the business requirements before making this decision.
    4. For systems using encrypted backups, ensure that the encryption root keys backup password is set and properly configured to avoid backup failures.

    Or 

    If backup encryption is not desired, manually disable the following encryption parameters on the affected SAP HANA systems via the parameters:

    • Backup_Encryption
    • log_encryption
    • persistence_encryption

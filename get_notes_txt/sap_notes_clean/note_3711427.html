Symptom

When adding a new line item to an existing purchasing contract using the App "Mass Changes to Purchase Contracts" functionality, based on data uploaded from an Excel spreadsheet, the system does not assign the specified contract line item number from the Excel sheet.
The system adds the new line item using the next sequential number instead of the designated value maintained in the uploaded Excel sheet.
For example, if the purchase contract contains line items 10, 20, and 30, and line item 55 is specified in the Excel upload, the system creates it as 40 instead of 55.

Environment

  •  SAP Fiori 
  •  SAP S/4HANA All versions 

Reproducing the Issue

  1. Create a contract in Fiori with multiple items. For example, item 10, 20 and 30.
  2. Go to the App "Mass Changes to Purchase Contracts", enter the contract number and click the "Go" button.
  3. Download the contract as an Excel file.
  4. In the Excel file, add a new contract item, specify the contract line item number in the Excel upload file (e.g., 55).
  5. Upload the contract file using the upload option in the application.
  6. Observe that the system creates the new line item using the next sequential number (e.g., 40) instead of the designated value 55 from the Excel sheet.

Cause

Regarding item numbers, the system treats identifiers (e.g., purchase contract number, item number) as fictitious values. This is explained in the SAP Help page: SAP Help Page: Purchase Contract Management 

"The unique value you enter for the identifier in each of the worksheets is temporary and is only used for identifying the corresponding record within the Excel file.
Once the information is uploaded, the identifier for the new record in the system could differ from the value you have entered.". 

The same behavior occurs in the Manage Purchase Contracts App. This reflects standard system design, where the system always adds new contract item numbers sequentially."

Resolution

This is the standard system design of the fiori App, that it always adds new contract item numbers sequentially.

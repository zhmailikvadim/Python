Symptom

When you create a Visit from the Contact TI, the Contacts Business Address is not used as the default address within the Visit.

Environment

SAP Cloud for Customer.

Reproducing the Issue

  1. Go to the Customers work center.
  2. Go to the Accounts view.
  3. Open the Account ABC (ABC represents the Account ID).
  4. Go to the Contact tab.
  5. Open the Contact DEF (DEF represents the Main Contact ID).
  6. Go to the Addresses tab.
  7. Here you can see that the Contacts Business Address differs from the address flagged as main within the Account ABC.
  8. Go to the Visits tab of the Contact DEF.
  9. Fill in all mandatory fields and select Save and Open.
  10. The Visit is created with the main address of the Account and not the Business Address of the Contact.

Cause

This is the expected system behaviour.

Resolution

The system is designed to select the main address of the Account by default. If you like, you could mark the business address as the main address, in which case it would become the default address for the Account.

You can also add the Address Selection field in the Visits header. You can use this field to select any other address for the Visit that you have maintained for the Account.

Keywords

Main Address, Business Address, Contact, Accounts, Visit

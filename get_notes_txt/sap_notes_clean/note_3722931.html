Symptom

  • Error message: "Enter a category of quota compensation Message No. HRTIM00BLPIT0416004" encountered during replication of time account payout from Employee Central (EC) to Employee Central Payroll (ECP).
  • The issue is related to the mapping configuration in table T77SFEC_CVPROPC and T77SFEC_CVMAPC.

Environment

  • PTP Time Data Replication

Reproducing the Issue

  1. Replicate a time account payout from Employee Central (EC) to Employee Central Payroll (ECP).
  2. Observe the error messages:
  • "Enter a category of quota compensation Message No. HRTIM00BLPIT0416004".
  • "No time constraint reaction for 0416 with time constraint class 00".

Cause

  • The field MAPPING_MODE in table T77SFEC_CVPROPC for TIME_COMPENSATION_METHOD is set to 'SI'. This configuration causes the program to look up system table T77SFEC_CVMAPS, which does not contain entries for time compensation methods delivered by SAP, resulting in no mapping being found.
  • The default setting in T77SFEC_CVPROPS for time compensation method for 'IS_COUNTRY_DEPENDENT' is TRUE, while the value SPACE is used for 'IS_COUNTRY_DEPENDENT' in T77SFEC_CVPROPC. This configuration leads the program to look up mapping entries in T77SFEC_CVMAPC with COUNTRY_ISO equal to SPACE, but the relevant entry exists with COUNTRY_ISO equal to 'PT'.

Resolution

  1. Change the field MAPPING_MODE in table T77SFEC_CVPROPC for TIME_COMPENSATION_METHOD from 'SI' to 'C'.
  2. Check the field IS_COUNTRY_DEPENDENT in T77SFEC_CVPROPC. If it is blank do not use ISO code in case it is 'X' use the pertinent ISO code.
  3. Verify the replication process after implementing the changes to ensure the error messages are resolved.

See Also

  • Refer to SAP Note 3293008 for further details on quota compensation deduction and related error messages.

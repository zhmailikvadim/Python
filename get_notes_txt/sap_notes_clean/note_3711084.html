Symptom

The attachment is not saved in Web Client when a Purchase Request is created with approval process.

Reproducing the issue

Prerequisite:

Approval process exists for User A for Purchase Requests.

 

Scenario:

  1. Log in with User A to SAP Business One Web Client, go to Purchasing → Create Purchase Requests.
  2. Create a Purchase Request with an attachment.
  3. Click Add & View. The approval process is triggered as expected, click OK.

 

Expected behavior:

The Purchase Request draft is saved with the relevant attachment.

 

Actual behavior:

The Purchase Request draft is saved without the relevant the attachment.

The same happens on other document types as well such as Sales Order with approval process, for example.

Solution

SAP intends to provide a feature package or patch to solve the symptom that is described in this SAP Note. The section References below lists the related feature package or patch once they become available. Be aware that References can only be confirmed at feature package or patch release date.

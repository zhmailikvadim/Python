Symptom

While connecting application server to HANA database when SSL/TLS is enabled, it throws below error in dev_w* traces-:

C  connect property [ENCRYPT = TRUE]
C  connect property [sslHostNameInCertificate = XXX]
C  connect property [sslCryptoProvider = commoncrypto]
C  connect property [sslKeyStore = Y:\usr\sap\D80\D22\sec\SAPSSLC.pse]
C  connect property [sslTrustStore = Y:\usr\sap\D80\D22\sec\SAPSSLC.pse]
C  *** ERROR => Connect to database failed, rc=1, rcSQL=-10709
 [dbhdbsql.cpp 630]
C  SQLCODE    : -10709
C  SQLERRTEXT : Connection failed (RTE:[300005] SAP crypto lib is not available: SECUDIR unknown or default SECUDIR \
C               not found in environment )
B  ***LOG BV3=> severe db error -10709; work process is stopped [dbsh         1324]
M  ***LOG R19=> ThDbConnect, db_connect () [thxxtool.c   6114]
M  ThIErrHandle: in_ThErrHandle: 1
M  *** ERROR => ThInit: db_connect (step TH_INIT, thRc ERROR-DB-CONNECT_ERROR, action STOP_WP, level 1) [thxxhead.c   2855]

Environment

SAP HANA, platform edition 2.0

Cause

  • SECUDIR environment variable not set properly on operating system level.
  • Older HANA client version.

Resolution

  • Check SECUDIR environment variable: Ensure that the SECUDIR environment variable is set correctly. This variable should point to the directory where the SAP cryptographic libraries are stored. This can be set in operating system's environment variables for the <sid>adm user (refer 1827566 - How to set environment variables for SAP system ) 

  • Make sure to use updated HANA client version. (refer 1906576 - SAP HANA client and server cross-version compatibility )

 

Note-: After making any changes to the environment variables or permissions, restart the SAP instance to ensure the changes take effect.

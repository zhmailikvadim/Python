Symptom

In the app "Display G/L Account Line Items" (Reporting View and Posting View) the DebitCreditCode is not translated. The display is language independent S and H.

Other Terms

C_JournalEntryItemBrowser C_OperationalAcctgDocBrowser  FAC_GLV_GL_ACCOUNT_LINE_ITEMS FAC_EV_GL_ACCOUNT_LINE_ITEMS   DebitCreditCode  DebitCreditCodeName

Reason and Prerequisites

This is no error. The code is always language independent. So we have always the value S or H in the column DebitCreditCode.

With this note we add a column DebitCreditCodeName with the language dependent text. In this new column we find the language dependent values for Debit and Credit.

Solution

The solution is part of feature pack 2. Please insert feature pack 2.

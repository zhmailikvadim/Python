Symptom

When a user edits a locally created reading and the measuring point is exclusively qualitative, the reading value is wrongly updated, which causes sync errors.

Other Terms

  • SAP Service and Asset Manager
  • SSAM
  • SAM
  • Take Readings
  • Qualitive measuring point
  • ICMTANGOAMF10-42527

Reason and Prerequisites

Reason: The presence of the characteristic is not validated during editing, although it is checked during creation.

Prerequisites: 

  • BAS or Visual Studio Code with SAM MDK project
  • `patch` command (available on BAS and macOS by default) via terminal or other patching utility
  • Read the guidelines on how to apply the patch and go through the appendix section if you get any errors.

Solution

With the implemented solution, the existence of the characteristic is also validated when the reading is not under creation.

To apply the patch, download the attached 3709911_XXXX.patch file and save it in a known location, preferably adjacent to your SAM.mdkproject.

If you are on Windows, see SAP Note 3533879 for how to install a .patch file in Windows. If you are on a Mac, follow the steps below:

  1. Open a terminal in the root of your SAM metadata (i.e. SAM.mdkproject/metadata or SAM.mdkproject/metadata/base.definitions)
  2. Run the terminal command `patch --no-backup-if-mismatch -p4 < path/to/3709911_XXXX.patch`. 
  3. Verify in the terminal window that the following files have been modified and listed as patched, as indicated by the console output:
  • patching file 'Rules/Measurements/Points/EDT/Controls/MeasuringPointsEDTValueControl.js'
  • patching file 'Rules/Measurements/Points/EDT/MeasuringPointsEDTOnCommit.js'
  • patching file 'Rules/Measurements/Points/MeasurementDocumentsCreateChangeSet.js'

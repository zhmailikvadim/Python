Symptom

After creating a custom OVP UI configuration by deep‑copying the standard configuration, the communicator status does not turn green and remains red.

Environment

  • SAP S/4HANA
  • SAP Master Data Governance 

Reproducing the Issue

  1. Create a custom OVP UI configuration for BP by deep‑copying the standard configuration and applying a custom prefix (for example, “Z”) and naming adjustments.
  2. Create a component for MDG_BS_GOV_COMMUNICATOR and configure cr Wires according to the custom setup.
  3. Refresh the browser and check the communicator status.
  4. Observe that the communicator status does not turn green and remains red.

Cause

Deep‑copying the delivered MDG Business Partner user interface is not recommended and can lead to inconsistent communicator wiring and status behavior; the issue is not caused by an error in the delivered standard.

Resolution

It is strictly recommended to not deep copy any of the SAP MDG delivered user interfaces for business partners, customers, suppliers and/or contract accounts. If you need to adapt these user interfaces, follow the instructions share in our MDG community for [MDG Extensibility].

Symptom

This SAP Note corrects the following problems:

  • In the Notification Tool (NT), tasks for quarantine cases are not completed automatically even though the package was processed successfully in a later run of the report SI: Transfer to SI Procedures (RP_PAYDE_B2A_SV_UNPACK).
  • In the report SI: Transfer to SI Procedures (RP_PAYDE_B2A_SV_UNPACK), log nodes are not displayed correctly if the same data type is processed for different receiving offices. This is only a display problem. The data packages are processed correctly.

Other Terms

 eXTra 1.5, complete collection, RP_PAYDE_B2A_SV_UNPACK, NT

Solution

Implement the attached correction instructions or import the specified support package.

There is no automatic subsequent processing of tasks that are still open for quarantine cases in the Notification Tool.

If you use the Notification Tool and the area SI/B2A and there are open tasks for quarantine, check whether these can be completed.

Choose the Administrator List in NT pushbutton to access the affected package. If the status of the data package is "Processed", the corresponding task can be set to "Completed".

For packages with the status "To Be Processed" and future tasks for quarantine cases, the task is processed automatically after you implement the corrections as soon as the package has been processed successfully.

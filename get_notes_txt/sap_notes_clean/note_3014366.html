Symptom

You are replicating organizational data from your Employee Central System to SAP ERP HCM.

During this replication you notice one of the following symptoms when replicating Organizational Assignments :

- During update of infotype table 1001 (relationships) unnecssary splitted records are written to the database. Subsequent records that are functionally identical are not getting merged.

- The importing parameter IT_PERNR_MAP for BAdI methods ADAPT_S_TO_OBJECT_RELATION and CREATE_OBJECT_RELATION (BAdI EX_SFIOM_ADAPT_EE_ORG_ASSGNMNT) is not sorted; this can e.g. cause a dump in the matrix manager example implementation.

- The ReplicationTargetSystem which is set in the Middleware is not stored in StagingArea table SFIOM_RPRQ_EE; hence, the Employee Confirmation message does not reach the EC data replication monitor.

- In case the staging area is enqueued and there is an additional access to the staging area the dump "OBJECTS_OBJREF_NOT_ASSIGNED" happens.

You are starting the processing report RH_SFIOM_PROC_ORG_STRUC_RPRQ without specifying any selection parameters, in this case replication requests related to customer-specific Organizational Object Types are not processed.

The processing of Organizational Objects (when started from the viewer transactions or using the processing report  RH_SFIOM_PROC_ORG_STRUC_RPRQ) takes a long time or ends with a shortdump due to memory overflow.

Additionally the events SAP_SFIOM_EE_ORGAS_RPPQ_CREATED and SAP_SFIOM_ORG_STRUC_RPRQ_CREATED are raised now with system client as parameter; this allows client specific event-based scheduling. If you want to restrict the job execution to these clients in which the event has been raised, enter the client number as parameter when defining the start time of the job.

Other Terms

CL_SFIOM_EE_ORG_ASS_REPL_V3

Reason and Prerequisites

There is a program error.

Solution

Import the Support Package related to this SAP Note or implement the relevant correction instructions in your system.

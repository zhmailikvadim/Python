Symptom

  • When a candidate rejects the Data Privacy Statement, their profile access is removed, and the system informs them that all personal data will be completely deleted.
  • If a candidate is disqualified from a job application, their application data remains in the system until the configured retention period expires.
  • This discrepancy raises concerns about compliance, as disqualified candidates should have the same control over their personal data as those who reject the privacy statement.
  • Retaining application data after disqualification without explicit consent or immediate deletion may conflict with GDPR principles of data minimization and the right to erasure.

Environment

SAP SuccessFactors Recruiting Management

Reproducing the Issue

  1. Apply to a job posting in RMK.
  2. Recruiter disqualifies the application from number 1.
  3. Candidate rejects the data privacy in their profile.
  4. Application marked to be anonymized but data remains in the system.

Cause

If a candidate is manually rejected and placed in a custom disqualified status, and their profile is anonymized using the anonymize candidate profile feature, the candidate will remain in the manual disqualification status. Additionally, the application data will only be anonymized after the inactivity period that has been configured under Manage Data.  

Resolution

  1. If the candidate is auto disqualified and the candidate profile is anonymized using the anonymize candidate profile feature, the candidate will be moved to the Delete on Demand by admin status.
  2. If the candidate is manual rejected and in a custom disqualified status and the candidate profile is anonymized using the anonymize candidate profile feature. The candidate will remain in the manual disqualification status.
  3. Review the inactivity period configured under Manage Data to verify the retention settings for disqualified candidates.
  4. Refer to the KBA 2670133 - Anonymize Candidate Profiles Feature - Recruiting Management - SAP for Me for additional configuration details and ensure proper setup of the anonymization functionality.

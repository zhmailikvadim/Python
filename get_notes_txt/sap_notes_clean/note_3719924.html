Symptom

When using Identity Provisioning Service (IPS) with the IAS target system(ias.api.version=1), users are added to a group based on a specified condition. However, users are not removed from the group if they no longer match the condition after the condition is updated.

Environment

Identity Provisioning Service (IPS)

Reproducing the Issue

  1. Set up an IPS job with IAS target transformation using:  
    {
      "constant": "yyyy", 
      "targetPath": "$.groups[0].value"
    }
      
  2. Run the Read job.  
      Users are assigned to the group: yyyy in IAS.
      
  3. Add or change the condition to a new value expecting users to be removed from the group if they no longer satisfy the condition.  
    {
    "condition": "xxxx",
    "constant": "yyyy", 
    "targetPath": "$.groups[0].value"
    }
      
  4. Run the Read job again.
    Users who do not meet the updated condition remain in the group, revealing the issue.

Resolution

1. Add "constant": "" to clear the group set by the previous provisioning.

{
    "constant": "",
    "targetPath": "$.groups[0].value"
},
{
    "condition": "xxxx",
    "constant": "yyyy", 
    "targetPath": "$.groups[0].value"
},

...

2. Execute Resync Job.

In a Resync job, even if there's no changes in source system entities, it will still overwrite all entities in target system.

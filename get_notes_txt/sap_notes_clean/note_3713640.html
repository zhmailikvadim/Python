Symptom

Errors indicating that the password has expired are displayed in Replication UI or in Advanced actions UI.

Environment

SAP Variant Configuration and Pricing     (version independent)

Reproducing the Issue

Steps:

  1. Trigger any replication of pricing or configuration data
  2. The replication does not finish
  3. In path “Monitoring – Replication” or “Monitoring – Advanced actions” of the Administration UI there are errors indicating that the password has expired.
    The displayed errors depend on the source database. Some examples below:
    - Error for Oracle database:
    Failed to establish JDBC connection. Error: ORA-28001: the password has expired
    - Errors for HANA database:
    Error occurred when connecting to Hana database.[SQL error code =10]
    SAP DBTech JDBC: [414]: user is forced to change password: alter password required for user

Cause

The password of the user that connects the database of “Variant Configuration and Pricing” to the source database has expired.

Resolution

The password of the user must be adjusted in the source database as well as in the Data Provisioning Agent of SAP HANA Smart Data Integration (SDI DP Agent).

Steps:

  1. Suspend the remote source 
    - Go to  Monitoring – Advanced Actions
    - Go to tab "remote sources"
    - Select the remote source
    - Click on Suspend
    - If there are two remote sources (used for pricing or configuration) perform the same steps above for both of them.

    Comment
    If this step is not performed the SDI DP Agent might connect with the old password while the password is not yet updated in the SDI DP Agent. This will lock the user!

  2.  Change the password for the relevant user on the source system. SAP recommends to disable any password lifetime checks for the user. This would avoid further password changes.

  3.  The password needs to be updated in the SDI DP Agent for the replication.
    Steps:
    - Start the configuration tool (./agentcli.sh --configAgent) 
    - Use option 4 - Remote Source Credentials to edit the user for the relevant remote source with the new password.

  4. Resume the remote source 
    - Go to  Monitoring – Advanced Actions
    - Go to tab "remote sources"
    - Select the remote source
    - Click on Resume
    - If there are two remote sources (used for pricing or configuration) perform the same steps above for both of them.

  5. Restart the erroneous entries
    Go to Monitoring - Replication
    Select the entry with error related to expired password  and click on “Restart selected”

  6. Trigger the load of pricing and configuration data ( if there is any missing data).

If the source system and source database is operated by SAP Enterprise Cloud Services (ECS) then SAP Enterprise Cloud Services (ECS) needs to be involved. Depending on the scenario, this involvement of ECS would be:
-  in step 2 ( change of password in source system)
-  or in both steps 2 (change in source system) and 3  ( change in the SDI DP Agent).
For more details see SAP Note 3273867:
3273867 - How to use SAP Variant Configuration and Pricing data replication when the source system is managed by SAP Enterprise Cloud Services (ECS)

Symptom

After adjusting parameter rdisp/PG_MAXFS and rdisp/PG_SHM parameter and restarting system, the ST02 shows Page area OnDiak is 0.

Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental. 

Environment

  • ABAP platform
  • SAP NetWeaver 
  • SAP S/4 HANA

Cause

When Paging/Swapping memory is used it will first use the memory that is in the buffer (controlled by rdisp/PG_SHM). If more paging memory is required, then memory from OS disk is requested (up to the value of rdisp/PG_MAXFS).

With both parameters having the same size, page memory will be limited to the buffer size.

In a summary, ST02 Page area's calculation is as following:

  • OnDisk = rdisp/PG_MAXFS*8 - rdisp/PG_SHM*8
  • In Memory = rdisp/PG_SHM*8

Resolution

If both rdisp/PG_SHM and rdisp/PG_MAXFS have the same value, the expectation is that OnDisk for Page area is zero. This is per design.

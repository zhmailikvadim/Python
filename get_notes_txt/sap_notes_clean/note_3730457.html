Symptom

After migrating a higher-numbered Purchase Order, the system continues numbering from that value instead of the previous sequence.

Environment

SAP Business ByDesign

Reproducing the Issue

  1. Go to Purchase Requests and Orders work center.
  2. Go to Purchase Orders view.
  3. Select New Button.
  4. Create a new Purchase Order.

The assigned Purchase Order number follows the migrated number rather than the expected next number from the prior lower range.

Cause

Once a higher number is introduced (for example, via migration), the system automatically advances the number range to maintain uniqueness and ensure audit compliance.

Resolution

This is the Expected System Behavior

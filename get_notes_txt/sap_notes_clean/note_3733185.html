Symptom

  • While using function module BUA_BUPA_UPDATE to update business partner addresses, tables BUT020 and BUT021_FS are updated, but change pointers are not captured.
  • No entries are created in CDHDR/CDPOS for the address update.
  • In CNS_CP_MONITOR, no corresponding change pointer entries are displayed for the address change.
  • Error observed in a related attempt: "Internal error when calling operation module BUA_CHECK_ADVW_VALIDITY; a check table is missing".

Environment

  • SAP S/4HANA
  • SAP S/4HANA Cloud Private Edition

Cause

  • BUA_BUPA_UPDATE and BUA_CHECK_ADVW_VALIDITY have “Not released” status and are not intended for use in custom developments
  • These function modules would not be supported by SAP if they do not work as expected as using them could result in missing change documents and change pointers.

Resolution

For “Not released” function modules (which can also be checked in SE37), we do not recommend using them, as may lead to data inconsistencies.
Instead, SAP recommends using released and supported BAPIs. 

To achieve your business requirement, you could possibly consider the following BAPIs from the following SAP KBA:
2484299 - BAPI list for BP

Address Data
BAPI_BUPA_ADDRESSES_GET: Get all addresses for a BP

BAPI_BUPA_ADDRESS_ADD: Add address

BAPI_BUPA_ADDRESS_CHANGE: Change address

BAPI_BUPA_ADDRESS_GETDETAIL: Get address

BAPI_BUPA_ADDRESS_GET_NUMBERS: Get address numbers

BAPI_BUPA_ADDRESS_REMOVE: Remove address

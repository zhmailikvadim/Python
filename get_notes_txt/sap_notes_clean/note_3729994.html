Symptom

After upgrading the SAP Convergent Charging Core Server system to version 2025 FPS 1, customers may have questions about compatibility with existing Java SDK-based client applications that run on previous Java versions. These Java applications or connectors use and are compiled with the Java Software Development Kit (SDK) of SAP Convergent Charging.

Specific scenarios include:
- SDK client applications that are compiled and run on Java 17
- Integration components that use SAP Convergent Charging Java Libraries, APIs, and SDKs - client classes (AsyncStatefulServiceClient, AsyncBatchServiceClient, etc.)
- Partner systems that connect through the Message TCP or HTTP/HTTPS protocols
- Questions about whether SDK recompilation or code changes are required

Other Terms

SAP Convergent Charging, SCC, Convergent Charging SDK, Core Server, SapMachine, Java compatibility, client-server compatibility, AsyncStatefulServiceClient, AsyncStatefulServiceClient, Message TCP, TLS compatibility, cross-version integration, FPS1

Reason and Prerequisites

Reason:

SAP Convergent Charging Core Server 2025 FPS 1 has been upgraded to SapMachine 25 (Java 25) to provide enhanced performance, security improvements, and long-term platform support.

The Core Server system and SDK clients communicate via network-based protocols (Message TCP, HTTP/HTTPS), which are independent of the Java runtime version used by client applications. This architectural design ensures that bytecode compatibility between different Java versions is not a concern for SDK-to-Server communication.

Prerequisites:

For SDK client applications to connect successfully to Core Server 2025 FPS 1, the following requirements must be met:

1. Java runtime: SDK client applications must run on a Java runtime version that is supported according to the Product Availability Matrix of SAP CC:

- For SAP Convergent Charging versions from 2023 FPS 0 up to and including 2023 FPS 3: SapMachine 17

- For SAP CC 2025 FPS 0: SapMachine 17

- For SAP CC 2025 FPS 1 and higher: SapMachine 25

- SDK client applications based on SDK versions earlier than SAP CC 2023 FPS 0 (which were supported only with SAP JVM 8) are not supported for connection to Core Server 2025 FPS 1.
2. Network connectivity: SDK client applications must have standard network connectivity to the Core Server system (ports and firewall rules as per deployment configuration).

Solution

SDK client applications running on Java 17 or higher are fully compatible with SAP Convergent Charging Core Server 2025 FPS 1.

No Actions Required

Existing SDK-based client applications do not require any of the following:
- Recompilation
- Code changes
- SDK version upgrades
- Java runtime upgrades (if already using Java 17 or higher)

Technical Explanation

An SAP Convergent Charging SDK client application communicates with the Core Server system through wire protocols over TCP/IP networks. The Java bytecode version on the client side and server side operate independently:

- Client side: SDK classes run in the client's JVM (for example, Java 17)
- Server side: Core Server 2025 FPS 1 runs in its own JVM (Java 25)
- Communication: Protocol-based message exchange (language- and version-agnostic)

This is analogous to a web browser connecting to a web server—the client and server can run different software versions without compatibility issues.

Validation Steps

While no changes are required, SAP recommends standard integration testing:

1. Deploy Core Server 2025 FPS 1 in a non-production environment.
2. Connect existing SDK-based client applications.
3. Verify successful connection and TLS handshake if used
4. Execute standard business operations.
5. Monitor logs for any security warnings.

For New Implementations

For new SDK client development, SAP recommends the following:
- Use the latest SDK version available with Core Server 2025 FPS 1 to access new features.
- Run client applications on Java 25 for optimal performance and support.
- Follow current security best practices in the Integration Guide.

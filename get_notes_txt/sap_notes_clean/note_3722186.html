Symptom

When I check my transaction logs in Managed Gateway for Spend&Network, I see that reservation-related transactions, such as DeltaStockRequest and QueryStockRequest, are failing. These transactions are coming from my SAP Ariba Test Parent Realm. Transactions of the same type sent from my SAP Ariba Test Child Realm are processing successfully.

Environment

Managed Gateway for Spend&Network

Reproducing the Issue

  1. You monitor the transaction logs in the Managed Gateway for Spend&Network test environment.
  2. You note that scheduled integration events for DeltaStockRequest and QueryStockRequest are being sent from the SAP Ariba Test Parent Realm.
  3. You observe that these transactions consistently fail.
  4. You also observe that identical transactions sent from the SAP Ariba Test Child Realm have a COMPLETED status.

Cause

You have configured your Managed Gateway for Spend&Network project only for your SAP Ariba Test Child Realm. The transactions are failing because they originate from the SAP Ariba Test Parent Realm, for which you have not configured an active project. Managed Gateway for Spend&Network is working as designed by rejecting transactions from an unconfigured realm.

Resolution

If you need to process these documents from the parent realm, you must create a new, separate project in Managed Gateway for Spend&Network exclusively for the parent realm environment. You can apply the same logic in case you have configured a project for your parent realm and there is no project created for the child realm even though you need to process documents for the child.

As stated in the product documentation, you must assign a realm name for your SAP Ariba solution for each environment. This confirms that projects are realm-specific.

Symptom

  • trans.log shows: “ERROR => Connect to database failed, rc=1, rcSQL=-10709”.
  • “Connection failed: [89001] Cannot resolve host name rc=11001: No such host is known (port 3xx15)”.

Environment

SAP HANA Platform Edition 2.0

Reproducing the Issue

4 ETW000  [dbhdbsql.cpp,00000]  *** ERROR => Connect to database failed, rc=1, rcSQL=-10709
4 ETW000  [     dev trc,00000]  SQLCODE    : -10709
4 ETW000  [     dev trc,00000]  SQLERRTEXT : Connection failed (RTE:[89001] Cannot resolve host name '(hostname)' rc=11001\
4 ETW000  [     dev trc,00000]               : No such host is known. (hostname)
4 ETW000  [    dblink  ,00000]  ***LOG BY2=>sql error -10709 performing CON
4 ETW000  [    dblink  ,00000]  ***LOG BY0=>Connection failed (RTE:[89001] Cannot resolve host name '(hostname)' rc=11001: No such host is known. (hostname)
2EETW169 no connect possible: "DBMS = HDB                              ---  SERVER = '' PORT = ''"

Cause

The Hostname mentioned in DEFAULT.PFL and hdbuserstore List are different.

Resolution

  1. Check the SAP HANA secure store keys with hdbuserstore list and confirm the stored hostname and port are correct.
  2. If the hostname is incorrect, update the key using hdbuserstore set to point to the correct host:port and credentials, then verify again with hdbuserstore list.

Keywords

r3trans not working, trans.log, sqlcode -10709 cannot resolve host name, rc=11001, hdbuserstore List.

Symptom

  • You run Release Batch Import , and select the Enable Automatic Rescheduling option, but there is no job scheduled in SM37
  • All logs show in green, no error is reported
  • You observe a similar scenario when you schedule import jobs with the Administration Cockpit

Environment

Solution Manager 7.2

Reproducing the Issue

  • Run Release Batch Import with option Enable Automatic Rescheduling
  • Run import job from the Administration Cockpit 

Cause

Missing authorizations: Object S_BTCH_NAM with field BTCUNAME value = SM_CHM (or a different user ID)

These are the authorizations checked for scheduling a job with another user ID as owner.

By default, the Release Batch Import background jobs are scheduled for user ID SM_CHM, so anyone that wants to schedule the Release Batch Import job periodically, will need the authorization object S_BTCH_NAM

Resolution

Provide authorizations for authorization object S_BTCH_NAM, field BTCUNAME with value SM_CHM.

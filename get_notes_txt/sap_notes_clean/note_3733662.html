Symptom

Messages processed via ABAP Proxy in PI/PO (e.g., monitored in SXMB_MONI or PI runtime) fail with an error similar to:

<SAP:Category>XIProxy</SAP:Category>
<SAP:Code area="ABAP">INTERFACE_SIGNATURE_ERROR</SAP:Code>
<SAP:P1><Interface_Name></SAP:P1>
<SAP:P2><Message_Type></SAP:P2>
<SAP:Stack>
Information about the method parameters is missing for the interface <Interface_Name>
</SAP:Stack>

Or SOAP fault:

Error during conversion of XI message
...
XIProxy INTERFACE_SIGNATURE_ERROR

The interface name, message type, and namespace may vary depending on the integration scenario.

Environment

  • SAP Process Integration
  • SAP Process Orchestration
  • SAP Netweaver Java 7.5

Cause

This issue occurs when there is a mismatch between the ABAP proxy definition in the backend system and the interface definition in the ESR (Enterprise Services Repository).

Common reasons include:

  • The ABAP proxy is outdated and not aligned with the latest ESR version
  • Changes were made to the interface (e.g., structure, namespace, root element) but the proxy was not regenerated
  • XML payload root tag name or namespace does not exactly match the expected definition
  • Inconsistencies detected in proxy metadata (e.g., after transport or interface update)

As of SAP NetWeaver 7.0 SP14 and higher, strict payload validation is enforced, requiring exact match of structure and namespace.

Resolution

To resolve the issue, synchronize the ABAP proxy with the latest ESR definition:

  1. Go to transaction SPROXY in the backend system
  2. Locate the affected proxy object
  3. Right-click and select Check (or press Ctrl + F2)
  4. Verify if inconsistencies or warnings are reported
  5. Regenerate the ABAP proxy so it reflects the current ESR interface definition
  6. Transport the regenerated proxy if required across systems
  7. Reprocess the failed message in PI/PO

Ensure that:

  • The proxy structure matches the ESR definition
  • The XML payload root tag and namespace are correct

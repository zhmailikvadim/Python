Symptom

During the execution of DTO operations on DataStore objects, a termination occurs with the following error:
RSO_DTO 001 with parameters 'GET_TARGET_LOCATION' and '-1-'

Other Terms

ADSO, DataStore object (advanced), DTO, Data Tiering Optimization

Reason and Prerequisites

The error occurs only for DSOs for which a temperature schema was selected that contains the temperature COLD.
If partitions were already moved to COLD before the RSADMIN parameter DTO_KEEP_EMPTY_ was set, the COLD-ONLY partitions are deleted from the metadata during the execution of the DTO repair. However, the actual data is still stored in the COLD store.

When you access the metadata, the error mentioned above occurs.

Solution

After you implement this SAP Note or import the relevant Support Package, the 'Repair' option of the DTO interface (in the back end, transaction RSOADSODTO or the corresponding tile in the Web UI) repairs this inconsistency. Afterwards, this error should no longer occur. 

  • SAP BW/4HANA 2021
    Import Support Package 14 for SAP BW/4HANA 2021 (SAPK-30014INDW4CORE) into your BW system. The support package is available once SAP Note 3625395 ("SAPBWNews SAP BW/4HANA 2021 SP14"), which describes this SP in more detail, has been released for customers.

  • SAP BW/4HANA 2023
    Import Support Package 09 for SAP BW/4HANA 2023 (SAPK-40009INDW4CORE) into your BW system. The support package is available once the SAP Note  with the short text "SAPBWNews SAP BW/4HANA 2023 SP09", which describes this SP in more detail, has been released for customers.

In urgent cases, you can use the correction instructions as an advance correction.

Before you do this, you must refer to SAP Notes 1668882 and 2248091 for information about transaction SNOTE.

Symptom

When submitting a Non‑PO Invoice to SAP Ariba Buying & Invoicing via cXML, the invoice line item includes:
<Extrinsic name="AccountCategory">K</Extrinsic>
Although the cXML contains valid accounting information, the Account Category (Account Assignement) does not default in the Ariba UI and the expected accounting determination is not triggered.
 

Environment

SAP Ariba Buying & Invoicing (B&I)

Reproducing the Issue

  1. Supplier system sends a Non‑PO Invoice cXML containing correct line‑item accounting fields and an Extrinsic for AccountCategory.
  2. The invoice imports into SAP Ariba Buying & Invoicing without technical errors.
  3. n the UI, the Account Assignment is not defaulted to the value sent (e.g., “K – Cost Center”) under line-item Accounting Details.

Cause

The issue occurs because the ERPCommodityCodeMap.csv (CEME file) used for accounting defaulting is missing the column value for AccountCategory for the relevant commodity mapping.

Although SAP Ariba B&I can process AccountCategory when sent as an Extrinsic, the system still requires a matching CEME entry to trigger the accounting defaulting logic.

If the CEME row has the AccountCategory column empty, Ariba B&I does not select the entry, and therefore the expected accounting defaults are not applied — even if the cXML accounting data is correct.

Resolution

  1. Open the ERPCommodityCodeMap.csv (CEME file) in your realm.
  2. Locate the entry corresponding to the commodity on the invoice line item.
  3. Confirm that the AccountCategory column exists and is populated.
  4. Enter the correct value (e.g., K) for the relevant commodity row.
  5. Re‑upload the corrected CEME file using the standard Data Import process.
  6. Reprocess or re‑submit the invoice.

After updating the CEME file, SAP Ariba B&I can properly trigger the mapping, and the Account Category received in the cXML is correctly recognized by the defaulting logic.

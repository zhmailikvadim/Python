Symptom

When I run the ARBCIG_MASTER_DATA_EXPORT program for master data incremental load in one client of my multi-client SAP ERP system, it deletes the change pointers in the ARBCIG_CHANGEDOC table for all clients. This stops master data changes from other clients from being processed and sent to SAP Ariba solutions, resulting in data discrepancies in SAP Ariba solutions.

Environment

  • Managed Gateway for Spend&Network
  • SAP Ariba Buying and Invoicing (B&I)
  • SAP Ariba Sourcing (SRC)
  • SAP ERP Central Component (ECC)
  • SAP S/4HANA

Reproducing the Issue

  1. In client XXXCLNT100, change any field of a master data object (for example, update a vendor address).
  2. Verify that the ARBCIG_CHANGEDOC table has entries in XXXCLNT100, XXXCLNT200, and other clients in system XXX.
  3. In client XXXCLNT200, run the master data incremental load for that object.
  4. The changes from XXXCLNT100 are sent together with the changes from XXXCLNT200 (and the corresponding change pointers are cleared in all clients).

Cause

The table ARBCIG_CHANGEDOC, which stores the change pointers, is a client-independent table by design and does not contain the field MANDT. The ARBCIG_MASTER_DATA_EXPORT program is designed to clear the change pointers from this table after it runs. In a multi-client environment, this standard behavior results in the deletion of change pointers for all clients, even when you execute the program for only one.

Resolution

This behavior is part of the standard product design, and there is no standard fix. If you require this functionality to work differently in a multi-client environment, you can consider the following options:

  • Submit an enhancement request to have this functionality considered for a future release.
  • Engage your internal consulting or development team to implement a customization to meet your business requirements.

Operating multiple clients in a single production system is not a recommended best practice, and there is no documented limitation stating that master data changes cannot be client-specific.

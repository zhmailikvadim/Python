Symptom

  • Unable to create a dynamic user list in SAP SuccessFactors Work Zone using Division or other attributes.
  • The Division field in Work Zone appears blank in Work Zone.
  • Changing the transformation tab code of the source and Target system does not populate the Division attribute in Work Zone.

Environment

  • SAP SuccessFactors Work Zone
  • Work Zone Advanced edition

Reproducing the Issue

  1. Attempt to create a dynamic user list in SAP SuccessFactors Work Zone using Division or other attributes.
  2. Modify the transformation tab code for the source and Target system.
  3. Observe that the Division field in Work Zone remains blank in Work Zone.

Cause

The required user attributes (for example, division, department, region, or custom fields) were not added to the identity provisioning source system Properties, so these fields were not synchronized to Work Zone user profiles.

Resolution

  1. In the identity provisioning source system, add the required user attributes (for example, division, department, region, and needed custom fields) to the Properties so they are included in synchronization.
  2. Review and update transformation mappings to pass these attributes from the source system to the Work Zone user profile attributes; save the configuration.
  3. Run a user synchronization/provisioning job to update users.
  4. Verify in Work Zone that user profiles now show populated values for the configured attributes.
  5. Create or refresh the Dynamic User List and apply filters based on the now-populated attributes as needed.

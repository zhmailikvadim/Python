or

2. With SPS08 revisions, an incorrect SQL statement that should fail with the error 'inconsistent datatype: invalid function argument' is unexpectedly executing and producing unintended results.

Other Terms

signal 11, SIGSEGV, ESX

Reason and Prerequisites

Issue number 352018

Reason:

Due to a programming error in SAP HANA, when comparing expressions with different data types, the NULLIF function may not be evaluated correctly, potentially leading to an indexserver crash or the execution of an incorrect statement, resulting in unexpected behavior.

Affected Releases:

  • SAP HANA 2:
    • Revisions 079.02 - 079.08 (SPS07)
    • Revisions 082.00 - 089.01 (SPS08)

Prerequisites:

  • You executed an SQL statement which contains a subquery that uses the NULLIF function to compare a fixed-point decimal column and a string value, for example:
    -- "dec" is a fixed-point decimal column in a column table TBL1 
    SELECT * FROM (SELECT DISTINCT * FROM TBL1 WHERE NULLIF(dec, '') > 1);

Solution

Apply one of the following SAP HANA Revisions:

  • SAP HANA 2:
    • Revisions >= 079.09 (SPS07)
    • Revisions of SPS08: SAP is currently working on a permanent solution to this issue. This SAP Note will be updated as soon as a revision containing the solution is available
    • or higher
  • or higher

Workaround:

  • Avoid the usage of NULLIF function as in the prerequisites

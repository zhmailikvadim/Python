Symptom

  • Running transaction LISTCUBE on an Open ODS View with input selections keeps reading without displaying any result.
  • In the source system, transaction ODQMON shows no selection is recorded. Due to this, a long-running extraction reads a very large number of records in the source system.
  • Testing the relevant extractor (RSA3) with the same selections returns a small result set quickly.

Environment

  • SAP BW/4HANA or SAP Netweaver
  • ERP or S/4HANA as the source system

Reproducing the Issue

  1. Run transaction LISTCUBE for an Open ODS View which's based on a Transformation/DataSource (ODP).
  2. Enter filters (for example, fiscal period, currency type, controlling area, cost center) and start the display.
  3. Observe that the query keeps reading without returning data; in ODQMON on the source system, verify that the extraction runs without any selection pushdown.
  4. In source system, execute transaction RSA3 for the corresponding DataSource with the same selections and observe that only a small number of records are returned quickly.

Cause

The Open ODS View is based on a Transformation that has a start routine but no inverse routine. Without an inverse routine, the system cannot translate and push down query selections to the DataSource, causing a full extraction without filters. See SAP Note 1486659 - Inverse Routines in Transformation for Virtual Providers, SPO, SEGR, ODSV

Resolution

  1. Ensure that the fields intended for filtering in the Open ODS View match the type, length, and conversion exit of the corresponding fields in the DataSource.
  2. Verify that the filter fields are enabled for selections in the DataSource.
  3. If the Open ODS View is based on a Transformation (TRFN), confirm that the filter field is identifiable from the TRFN mapping.
  4. If the Transformation includes a start routine, ensure that an inverse routine is implemented. Refer to SAP Note 1486659 for guidance on inverse routines in transformations for virtual providers.
    • For example:
      c_th_fields_inbound = i_th_fields_outbound.
      c_r_selset_inbound = i_r_selset_outbound.
      c_exact = rs_c_true.

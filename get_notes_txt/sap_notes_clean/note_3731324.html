Symptom

While dealing the inactive version of an object as per Note 3396053 - Deal with active object but has inactive version exist, message "You are not currently working on an inactive version of <object name>" occurs and nothing happens. Check the corresponding entry in table DWINACTIV, the user under column "UNAME" is not current log on user.

Environment

ABAP system.

Reproducing the Issue

Activate the object to activate the inactive version

OR

"Utility -> Versions -> Return to active version" to keep the active version

Cause

The inactive version is locked by other user.

Resolution

  1. Create/Validate the user under "UNAME" in table DWINACTIV if the user does not exist or not valid.
  2. Log on with the user under "UNAME" in table DWINACTIV to maintain corresponding objects

See Also

Keywords

DWINACTIV; Inactive version; object lock; Return to active version

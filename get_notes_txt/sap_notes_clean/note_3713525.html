Symptom

You are logged on to the SAP System with a language different than English.

In transaction /SAPAPO/PPO_DPEN, you are in edit mode and select a demand class from the dropdown menu. It is translated to the language you are logged on. When selecting one of the possible values, you're getting the error message 'DB861: your entry "*" is too long. The maximum possible entry is 20.' The reason for that is that the field is defined as 20 characters, but in the language you are logged on, the demand class name is longer than 20 characters.

After confirming the message and saving, the system terminates with ABAP dump GETWA_NOT_ASSIGNED.

Other Terms

/SAPAPO/PPO_DEMAND_CLASS_TEXT, /SAPAPO/PPO_S_DEMAND_PEN_ALV, /SAPAPO/CL_PPO_DEM_PEN_MNTNCE

Reason and Prerequisites

The field for the demand class text is not defined long enough for all languages.

Solution

Implement the attached correction instructions. With this SAP Note, the field length for the demand class text is enhanced to handle the demand class texts in all languages.

Symptom

Scope Items 7VW and/or 7WP cannot be activated during the Define Scope phase in SAP Central Business Configuration (CBC). 

Environment

  • SAP S/4HANA Cloud Public Edition
  • SAP Central Business Configuration

Reproducing the Issue

  1. Go to the SAP Central Business Configuration workspace
  2. Go to Define Scope app
  3. Search for 7WP and 7VW
  4. No results found

Cause

When the workspace does not include Configuration Localization Tool (CLT) countries, scope Items 7VW and 7WP will not be displayed as selectable scope items once they are designed exclusively for CLT. 

Resolution

A. Bill of Exchange Requirement 

In workspaces where the Configuration Localization Tool (CLT) is not used, Scope Item 7VW is not available for activation.

For such scenarios, Bill of Exchange functionality is delivered through Scope Item 1WQ, which is the standard scope item for this business process in non-CLT environments. 

Therefore, when CLT is not enabled, scope Item 1WQ must be activated to support Bill of Exchange processing.

B. Withholding Tax (WHT) Requirement

In workspaces where the Configuration Localization Tool (CLT) is not used, Scope Item 7WP is not available for activation.

Scope Item J60 provides the standard Withholding Tax (WHT) functionality in SAP S/4HANA Cloud. When Scope Item J60 is already activated in the workspace, the system includes the required WHT configuration and processing capabilities.

Therefore, when CLT is not enabled, scope item J60 should be used.

Note: CLT can only be used in Implementation Workspace within SAP Central Business Configuration.

Both scope items 1WQ and J60 are self-service scope items and can be enabled by the user itself using Define Scope app in the SAP Central Business Configuration.

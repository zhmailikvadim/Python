Symptom

  • The "Share capital amount" field of SM30 view EDO_PL_INV_FOTRV has a character limit of 50, while the "invoice footer" field as per KSeF allows a maximum of 3500 characters.
  • The "Share capital amount" value is incorrectly mapped to the "invoice footer" field , causing the field to display only 50 characters instead of the allowed 3500 characters.

"Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental."

Environment

  • SAP ERP
  • SAP S/4HANA
  • SAP Document and Reporting Compliance Service

Reproducing the Issue

  1. Launch Access Transaction code SM30 EDO_PL_INV_FOTRV. 
  2. Enter a value exceeding 50 characters in the "share capital amount" field.
  3. Observe that the system restricts input to 50 characters.

Cause

The standard design for DRC Poland restricts the length of data element EDO_PL_CAPITAL to 50 characters. 

Resolution

  1. The standard design for DRC Poland restricts the length of data element EDO_PL_CAPITAL to 50 characters.
  2. Implement BAdI `EDOC_ADAPTOR`using the method `SET_OUTPUT_DATA` and develop logic to fetch information in XML for values exceeding 50 characters.
  3. To request this feature in the standard product, refer to the KBA 3603192 - How to request a new feature in SAP DRC - Electronic Document Processing(eDocument) Solution? - SAP for Me and access the SAP Customer Influence program to submit a feature request.

Symptom

When attempting to transport requests in CTMS, a request is created manually by uploading a file to source node (usually DEV).
This procedure: Add Files to Import Queues

The expectation is that this request is forwarded to subsequent import queue (usually QAS) but the request remains in state initial and it does not appear in next systems.

Environment

Cloud Transport Management

Reproducing the Issue

  1. Go the import queue of source systems (usually DEV)
  2. Click on 'Add' button
  3. Fill all the required fields and upload the file
  4. Click on 'Ok' button

Cause

The requests are not forwarded automatically when adding them manually, only if the request is sent to source node (DEV) by an application.

Resolution

It is not expected that a request will be imported in a source node because source nodes act as source and it does not have a destination to deploy the content. 
That said, requests added manually to source node (DEV) will not be forwarded automatically to subsequent nodes.

There are some options for the set up:

1 - If the approach will always be upload the file manually, delete de DEV node and keep only quality and productions nodes as DEV node will not act as source here because you are uploading manually. So, instead of adding the file to DEV, add the file directly to QAS (quality).

2 - Import the request in DEV import queue (which will end in fatal and that is expected as the source node does not point to any environment, it acts as a source). Then the request will be forwarded automatically to subsequent import queues (this depends on how the forward mode is configured, please see Create Transport Nodes > Forward mode).

3 - Keep the source node (DEV) and enable the Forward mode, then you can upload the files and forward at your own convenience, please check the further information here: Forward Transport Requests

Symptom

  • IT0591 Nominations info type Nomimee's name is not getting store in NAM01 but it is storing in NAM06
  • Initially, nominee details such as Name, Address, User, Date of User, and Share Percentage were stored sequentially in NAM01, ADD01, REL01, DAT01, SHP01, and Recently, nominee names are being stored randomly in NAM06, NAM07, etc
  • Nominations data in Infotype 591 getting stored under NAM06 instead of NAM01
  • Name not getting saved in NAM01 when using PA30 to maintain IT0591
  • PA0591 Namine name updation
  • Nominations Infotype 0591: Increase in field length for Nominee Address and Guardian Address
  • Nominations Infotype 0591 - Nominee Name field character length increase

Environment

  • Human Capital Management (HCM)
  • SAP ERP Payroll India
  • SAP R/3
  • SAP ERP Core Component
  • SAP ERP
  • SAP enhancement package for SAP ERP

Reproducing the Issue

  1. Login to SAP system and enter T-Code PA30.
  2. Enter Personnel No and info type 0591, then click on "Change."
  3. Update, remove, or add nominee details.
  4. Observe that nominee details are stored NAM06 instead of NAM01

Cause

  • The field length for Name of Nominee was enhanced to allow 80 characters as part of HR Renewal changes. However, new fields were created (NAM06 to NAM10), and data for new records is being stored in these fields instead of the existing ones (NAM01 to NAM05).

Resolution

  • The field length for Name of Nominee was enhanced to allow 80 characters as part of HR Renewal changes.
  • The change the field length was not changed for the existing fields i.e. NAM01 - NAM05, however created new ones and if some new record is created then the data gets stored in NAM06 to NAM10. 
  • Refer SAP Note 0003196827-Nominations Infotype 0591: Increase in field length for Name of the Nominee

Refer to: SAP Note 0003196827 - Increase in field length for Name of the Nominee in IT0591.

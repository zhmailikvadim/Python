Symptom

  • EarlyWatch Alert (EWA) is generated successfully in SAP Solution Manager.
  • EWA report is available in Solution Manager (PDF/Fiori/Word).
  • In SAP Support Portal, the EWA status shows “Not Performed”.
  • Customer assumes EWA is not running or SAP has not received the data.
  • No functional issues are visible in Managed System Configuration (MSC).
  • EWA generation jobs complete without errors.

Environment

  • SAP Solution Manager 7.2
  • Managed Systems: ABAP / Java
  • EarlyWatch Alert (EWA)
  • SAP Support Portal connectivity

Cause

The EWA data was generated successfully in SAP Solution Manager, but the upload/synchronization of service data to SAP Support Portal did not occur or failed.

Common reasons include:

  • SAP Support Backbone connectivity issue.
  • Upload job delay or failure.
  • Temporary network or SAProuter issues.
  • Incomplete synchronization after EWA generation.

As a result, the SAP Support Portal does not receive the EWA data and displays the status as “Not Performed”.

Resolution

Perform the following checks to ensure successful upload of EWA data to SAP Support Portal:

  1. Verify SAP Support Backbone connectivity:
  • Check RFC destinations related to SAP Support Portal.

      2. Ensure that service data upload jobs are:

  • Scheduled.
  • Running successfully.
  • Not canceled or delayed.

      3. Verify that the system appears correctly in LMDB.
      4. Regenerate the EWA via scheduled execution if required.
      5. Allow sufficient time for synchronization with SAP Support Portal.

Once the upload is successful, the EWA status in SAP Support Portal will automatically update from “Not Performed” to “Performed”.

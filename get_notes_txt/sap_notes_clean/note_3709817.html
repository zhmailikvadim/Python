Symptom

  • When the 'Rehire with Old Employment' switch is enabled, a new employee number is generated after the rehire check is completed during the onboarding process.
  • IAS deletes the existing account and creates a new one.
  • A new email is triggered to the user, requiring them to reset their password to log back into IAS.

Environment

SAP SuccessFactors Onboarding

Reproducing the Issue

  1. Enable the 'Rehire with Old Employment' switch
  2. Initiate onboarding for a user who has a future-dated rehire.
  3. Complete the personal data step during onboarding.
  4. Observe that a new employee number is generated, leading to the deletion of the existing IAS account and the creation of a new one.
  5. Verify that an email is triggered to the user, prompting them to reset their password to log back into IAS.

Cause

During IPS provisioning from SF to IAS, each SF user is mapped to a unique IAS user via a global UUID. The error occurs when two SF users (for example, an inactive Onboardee and an active Employee) are assigned the same IAS global UUID. IPS only supports a 1 SF user - 1 IAS user relationship. Attempting to assign the same global UUID to multiple SF users leads to the “Duplicated user resource sap global uuid detected” error. Additionally, filters like sf.user.filter = active eq "true" and userType eq "Employee" prevent IPS from reading inactive or non-employee users, which affects provisioning scenarios involving Onboardees.

Resolution

  1. Simplify SF Hiring Process – Align onboarding and employment records to ensure that each SF user corresponds to a unique IAS user. Avoid scenarios where multiple SF users (inactive Onboardee + active Employee) share the same IAS global UUID.
  2. IPS Job Limitations – Understand that IPS provisioning cannot provision 2 SF users to 1 IAS user. Adjust workflows to maintain a 1:1 mapping between SF users and IAS users.
  3. Filter Configuration – Use SF source system filters (e.g., active eq "true" and userType eq "Employee") to ensure only relevant SF users are read and provisioned to IAS.
  4. Future Considerations – Any attempt to bypass deletion or selectively skip Onboardees using conditions (e.g., $.userType=='ONBOARDEE') is currently not supported in IPS; provisioning logic must respect the 1:1 SF to IAS mapping.

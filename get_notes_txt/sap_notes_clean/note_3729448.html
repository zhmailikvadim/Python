Symptom

  • BAdI EDOC_ADAPTOR implementation is not triggered.
  • Custom logic in method SET_OUTPUT_DATA is not executed.
  • Issue observed after implementing latest notes or switching to KSeF 2.0.

Environment

  • SAP ERP
  • SAP Document and Reporting Compliance, cloud edition

Reproducing the Issue

  1. Implement BAdI EDOC_ADAPTOR.
  2. Maintain filter values (e.g. COUNTRY = PL, GENERIC_FILTER).
  3. Process a Poland KSeF eDocument.
  4. Observe that the BAdI implementation is not triggered.

Cause

This behavior can occur due to one of the following:

  • Incorrect or inconsistent filter values maintained in the BAdI implementation
  • Standard system behavior where the BAdI is not triggered for certain KSeF 2.0 processing scenarios

Resolution

Adjust and verify the filter values maintained in the BAdI implementation.

Ensure the following filter values are correctly maintained:

  • COUNTRY = PL

  • GENERIC_FILTER = INVOICE

If the GENERIC_FILTER is maintained incorrectly (for example, as KSEF), the BAdI implementation will not be triggered.

Retest the scenario after correcting the filter values.

For detailed configuration steps, refer to:

  • SAP Note 3682042 – Electronic Document Processing for Poland - KSeF: FA (3) Application Interface Framework (AIF) BAdI Implementations

Symptom

  • A company code that is active for Document Splitting is not present in table FAGL_SPLIT_ACTC.
  • The same company code appears as active in view V_FAGL_SPLIT_ACT.

Environment

  • Financial Accounting (FI)
  • SAP R/3
  • SAP R/3 Enterprise 
  • SAP ERP Central Component
  • SAP ERP
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA Finance
  • SAP S/4HANA

Reproducing the Issue

  1. Verify that multiple company codes are active in Document Splitting.
  2. Check view V_FAGL_SPLIT_ACT and confirm those company codes are listed as active.
  3. Open table FAGL_SPLIT_ACTC and observe that one of the active company codes is not listed.

Cause

It´s intended design that view V_FAGL_SPLIT_ACT displays all existing company codes including those without an entry in table FAGL_SPLIT_ACTC. This is not an inconsistency. For company codes with active document splitting it doesn´t matter whether there´s an entry in table FAGL_SPLIT_ACTC or not. Only for company codes with deactivated document splitting a table entry with inactive = 'X'  is required. If 'inactive' is cleared later, the table entry will remain in table FAGL_SPLIT_ACTC with inactive = space. However, for company codes in which document splitting had been active from the scratch, no entry is created in table FAGL_SPLIT_ACTC.

Resolution

This is standard system behavior.

Symptom

Implementing a CAP application with destination service and facing issue and errors. Created a long running job using cds.spawn and getting error "message":"Error during request to remote service: Failed to load destination. Caused by: Failed to verify JWT. Caused by: Token is expired."

Environment

SAP Cloud Application Programming Model

Resolution

  • The CAP framework passes the JWT token to the SAP Cloud SDK and the SAP Cloud SDK calls the destination service with the JWT and stores the authentication token of the target system access. The SAP Cloud SDK caches this information. Check https://sap.github.io/cloud-sdk/docs/js/features/connectivity/destinations#destination-service and https://sap.github.io/cloud-sdk/docs/js/features/connectivity/destination-cache  
  • For every request that comes in the SAP Cloud SDK fetches a new token
  • If you are using a long running batch job kind of scenario(eg cds spawn) then user propagation is not the right way to call the destination service and one needs to use a technical user or use the destination service with OAuth2.0 client credential flow so the system does not rely on user propagation JWT token

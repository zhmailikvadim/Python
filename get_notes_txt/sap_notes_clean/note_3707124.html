Symptom

You are using a SAP product based on SAP NetWeaver ABAP with IBM Db2 for i. During a homogeneous system copy with SAVRST, SWPM reports an error with AS4FIXFILE:

Processing completed, however errors occurred. Check joblog for errors in DOFIX jobs.

Job log shows something like:

DOFIX job #2 returned an error, please scan the joblog for CPF9897
CALL PGM(*LIBL/CALLFIX) PARM(R3D85DATA *YES *YES *YES *NO 06 *NO D85OWNER R3D85DATA QSQJRN *ALL ¦ 1), RC = 1.

Scanning the job logs of the worker jobs, one of the DOFIX jobs shows something like the below if ENHHDRSAVF was a save file:

CPF9897 - QDBRTVFD failedwith CPF3C23 on ENHHDRSAVF

Other Terms

AS/400, DB2/400, iSeries, DB4, System i

Reason and Prerequisites

Root cause is a programming error. So far, AS4FIXFILE was not able to handle objects of type which are not database files, i.e. save files. In effect, the right things would have happened, but AS4FIXFILE would return an error.

Solution

Apply the ILE patch mentioned in the Support Packages & Patches section of this SAP Note and retry the command.

Symptom

  • When using the Import Master Data – Business Partners application to load supplier data, processing fails with the error: "Vendor number was not filled during external number assignment".
  • The vendor number (LIFNR) is populated in the import file.
  • External number assignment is active and aligned with the correct number range.
  • No visible conflicts are identified in number ranges or related organizational data.

Environment

  • SAP S/4HANA Cloud Public Edition

Reproducing the Issue

  1. Open  Import Master Data – Business Partners Application.
  2. Upload sample file. 
  3. Create Process. 
  4. Save and start the process.
  5. System will throw an error. 

Cause

Incorrect configurations. 

Resolution

  1. Refer to 2637435 - CMP: Using External Numbering in Consolidation Processes for detailed guidance on how external numbering is determined in consolidation and mass processing.
  2. Review and align external number assignment and number range settings according to the note’s guidance.
  3. Verify the import template maps the vendor number (LIFNR) correctly for external numbering scenarios.
  4. Apply any required adjustments and re-run the Business Partner import.

See Also

Refer to: 2637435 - CMP: Using External Numbering in Consolidation Processes

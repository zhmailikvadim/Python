Symptom

In the Cloud Integration message monitor, the messages are shown as failed with error details similar to the following:

“Outbound processing in endpoint at /xxx/xxx failed with message ‘Fault: Could not generate the XML stream caused by: com.ctc.wstx.exc.WstxUnexpectedCharException: Unexpected character ‘H’ (code 72) in prolog; expected ‘<’ at [row,col {unknown-source}]: [1,1].’, caused by ‘WstxUnexpectedCharException: Unexpected character ‘H’ (code 72) in prolog; expected ‘<’ at [row,col {unknown-source}]: [1,1]’.”

However, when opening the failed message in the design view, the processing appears to be completed, and a different error can be identified at a later step in the integration flow:

Environment

  • SAP Integration Suite
  • SAP Business Technology Platform
  • Cloud Integration

Cause

In the Cloud Integration message monitor, only the final error is displayed in the overview screen. Since the integration scenario is synchronous and a failed response is returned to the sender application, the outbound processing error becomes the last recorded error. As a result, this outbound error is the one shown in the monitor.

Resolution

In such cases, always open the failed MPL in the design view and identify the original error within the IFlow steps. In the example described in the symptom section, the root cause was an authentication issue in the HTTP receiver. The failed logon response then triggered the error observed in the sender adapter.

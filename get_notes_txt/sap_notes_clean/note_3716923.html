Symptom

The incorrect G/L account is used when Advanced G/L account Determination is enabled and the purchase order is created from Purchase Request Report.

Environment

SAP Business One

Reproducing the Issue

Prerequisites:

You are working with German Localization. 

Scenario:

  1. Set up Advanced G/L Account Determination, ensuring rules for German customers and for EU customers. 
  2. The EU Expense Account is set to: 1111.
  3. Go to Purchasing → Purchase Request and create a new document.
  4. Go to Reports → Sales and Purchase → Purchase Request Report and select the created purchase request.
  5. Select vendor Vendor1 (EU vendor, not German).
  6. Create the purchase order.
  7. Observe the G/L account used in the purchase order.

Expected behavior: 

The G/L account used in the purchase order is EU Expense Account set in Advanced G/L Account Determination: 1111

Actual behavior:

The G/L account used in the purchase order is NOT EU Expense Account set in Advanced G/L Account Determination.

Resolution

This is the correct system behavior.

Advanced GL Account Determination only operates when creating or updating the document. Otherwise, information regarding G/L Accounts from the base document will remain and be used in the target document. 

In this case, Purchase Request, the base document, doesn't have the conditions to apply the G/L Account Determination.

The target document: Purchase Order, does have those conditions. However, for the conditions to be applied the agency of the user is required. 

The user has to answer the pop up - Update document rows according to new BP's data? Saying YES and the G/L account will be changed according to the BP.  

This question is missing when the purchase orders are created from the Purchase Request Report.

Symptom

PODs transported across environment via API Import appears without their configured Indicators in the Line Monitor Plugin, while the rest of the POD configuration is present.

"Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental."  

Environment

  • SAP Digital Manufacturing 2602 and higher.

Reproducing the Issue

  1. Export a POD that has Indicators configured using API /export, example:

    2. Import the POD using the exported file via API /import

    3. Indicators configuration will not be copied. Indicators will be unchecked, see example:

 

Cause

  • The indicators selected by the Last Indicator Value plugin are indexed by the select equipment ID, indicator name, and indicator structure path. 
  • DM does not guarantee the uniqueness of the equipment name and instead relies on the equipment ID for uniqueness integrity. 
  • In DM, the same equipment is not guaranteed the same equipment ID across cloud environments.  For this reason, when migrating between cloud environments, a POD using the Last Indicator Value will require re-configuration including the re-selection of indicator values. 

Resolution

  • The behavior reported in this KBA is working as designed.

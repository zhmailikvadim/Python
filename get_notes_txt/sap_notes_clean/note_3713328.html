Symptom

This SAP Note is relevant only for the country version Switzerland (CH).

It contains the following enhancements and adjustments to TARDOC and the outpatient flat rates:

  • Service rule C60 (ISH_CH_CHECK_AMBP_MAPPER) has been redesigned in such a way that changes made to physicians and time stamps for a LKAAT service do not lead to the cancellation of the corresponding TARDOC service, but to a targeted adjustment of physicians and time stamps for TARDOC service.
  • For the service rule C61 (ISH_CH_CHECK_AMBP_INV_MAPPER), an additional query for the rule types C19SEC, C29SEC, and C34 is missing so that services that were created by these service rules are excluded from the "inverse mapping".
  • In the future, the checks performed by service rule C19SEC (ISH_CH_CHECK_TARDOC_MAX) will only be performed for TARDOC services that were performed as of January 1, 2026. During the check for the unit 20 "Division and Day", the division is currently not determined correctly from the referenced main service. As a result, the system may issue an error message, even if the service with this unit 20 exists in another case of the same patient.
  • Service rule C29SEC (ISH_CH_CHECK_TARDOC_MAX_GRP) does not yet take a cross-case check into account.
  • Addition of consideration of benefit groups to the determination of the section code
  • Enhancement of the report RNWCH_SEC_CHECK "IS-H CH: Check/Adjust Services for Existing SectionCode" with an update function so that incorrect or missing section codes for the service can be updated in accordance with the Customizing settings.
  • Addition of the import parameters for case number and test run to the methods CALL_CASEMASTER and CALL_GROUPER of the BAdI ISH_CH_AMBP
  • In the future, the CaseMaster will only be called when outpatient treatments are generated if patient encounters also exist. A transferred case without patient encounters currently leads to an error message by the CaseMaster.

Symptom

  • An error message similar to "IANA time zone ‘UTC’ not found" is displayed when attempting to perform a 'Schedule' action in a Fiori app.
  • HTTP request failed with statusCode: "400" and statusText: "Bad Request".
  • The returned value for timezone-dependent logic is "UTC" instead of the expected "Etc/UTC".

Environment

SAP Fiori Launchpad

Reproducing the Issue

  1. Open a Fiori App such as F2222 (Manage Work Center Utilization)
  2. Attempt to create a schedule
  3. Observe the error message "IANA time zone ‘UTC’ not found"

Cause

The application uses the function `sap.ui.getCore().getConfiguration().getTimezone()` to retrieve the timezone, which returns "UTC". Similarly, `Localization.getTimezone()` also returns "UTC". The expected value is "Etc/UTC". This discrepancy triggers an error in the application when handling timezone-dependent logic.

Resolution

  1. Go to Transaction Code /n/UI2/FLP_CUS_CONF
  2. Set parameter TIME_ZONE_FROM_SERVER_IN_UI5 to 'true'

Please note: /UI2/FLP_CUS_CONF is the client-specific transaction code for Fiori Launchpad parameters.

For cross-client (system-wide) parameters, you can alternatively use /UI2/FLP_SYS_CONF.

See Also

Launchpad Configuration Parameters

Please Note: The documentation incorrectly states the default value for this parameter as 'true', whereas the actual default value is 'false'.

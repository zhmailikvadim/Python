Symptom

  • When making changes to a column within a table on the second or greater pages of a report in the Web Intelligence Applet interface and Web Intelligence Rich Client, the report jumps to the first page.
  • The changes can be to edit the formula of the cell, or edit the format of the numbers.
  • The same issue does not occur in the Web Intelligence HTML interface.

Other Terms

webi, webintelligence, java, jrp, richclient, WRC

Reason and Prerequisites

Environment:

  • SAP BusinessObjects Business Intelligence Platform 4.2 Support Package 07
  • Web Intelligence

Steps to reproduce:

  1. Log on to BI launch pad, and select Applet or Desktop in the Web Intelligence preferences
  2. Open and edit the sample document ZZ_Charting under Folders\Web Intelligence Samples folder
  3. Go to RAW DATA page and go to the last page in Quick display mode
  4. Select Month column cell and edit the formula as =[Month] + "TEST", press Enter key to confirm
  5. Notice the report jumps back to the first page
  6. Now go to the last page again, select column SKU Number, right click the cell and choose Format Number
  7. Select a different format, click OK to confirm
  8. Observe the report jumps back to the first page

Cause:

  • This issue is caused by a defect in the product

Solution

This issue is fixed in the patches listed in the "Support Packages & Patches" section below.

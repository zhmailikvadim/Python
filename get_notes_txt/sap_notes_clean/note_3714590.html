Symptom

In the process "Receipt of EQS Profile Definitions from DSO (SUP View)" (DE_B_EQS_PFDEF_DSO_RECEIVE_SUP), if an entry in the table /US4G/PFDEF_EL is deleted but the corresponding header entry in /US4G/PFDEF_EL exists, importing the profile definition again does not insert the missing entry in /US4G/PFDEF_EL in step 0500.

This issue is relevant for the market role: SUP.

Other Terms

EQS_SUP, DE_B_EQS_PFDEF_DSO_RECEIVE_SUP, /US4G/PFDEF_EL, profile definition, settlement, MaBiS, electricity table

Reason and Prerequisites

The error occurs because of a program error that prevents the system from inserting a missing entry in /US4G/PFDEF_EL when the header entry still exists.

Solution

Implement the corrections of this SAP Note or import the relevant support package.

After implementing this SAP Note, the system inserts the missing entry in /US4G/PFDEF_EL when you import the profile definition again, even if the header entry already exists.

You need to import the complete profile definition again for any entry missing in /US4G/PFDEF_EL. You can verify the fix by checking that, after importing the profile definition, the electricity-related details are inserted in the /US4G/PFDEF_EL table.

Symptom

A Custom Field of type Amount with Currency or Quantity is created using the Custom Fields app and enabled for an Analytical CDS View which has a Header Annotation to ignore propagated annotations:

  • @Metadata.ignorePropagatedAnnotations: true

During publishing, the process fails at DDIC mass activation for the generated CDS Extend with the error (see transaction SCFD_LOG):

  • DDIC Mass Activation: Activation failed
  • OData call failed
  • Could not be published
  • Return Code: 8. Object type: DDLS. Object name: ZZ1_***. C_***-ZZ1_*** reference information missing or data type wrong, see long text

Custom Fields of other data types (for example, Text or Date) can be published successfully in the same business context.

No dumps were encountered in ST22.

Environment

  • SAP S/4HANA (On-Premise / HEC)
  • SAP MDG
  • Custom Fields and Logic (Fiori app)
  • Analytical CDS Views
  • Business Context: Customer Company Code – Core

Reproducing the Issue

  1. Open the MDG Fiori App.
  2. Navigate to the "Custom Fields" application.
  3. Create a custom field with the following details:
    • Business Context: Customer Company Code Core
    • Label: Requested CR Limit
    • Identifier: ReqCRLimit
    • Type: Amount with Currency
  4. Save the custom field.
  5. Attempt to Publish the custom field.

Cause

This issue is caused by a product defect in the generation of CDS Extends for Custom Fields of type Amount or Quantity when applied to Analytical CDS Views that use the annotation:

  • @Metadata.ignorePropagatedAnnotations: true

Due to this annotation, the reference field information is not available on such Analytical CDS Views.

As a result, the generated CDS Extend lacks mandatory reference annotations:

  • @Semantics.amount.currencyCode for Amounts fields
  • @Semantics.quantity.unitOfMeasure for Quantity fields

Without explicit reference information, CDS Extend activation fails.

Resolution

  1. Apply SAP Note 3665468, which provides a fix for the bug affecting custom fields of type "Amount with Currency".
  2. After that, republish the corresponding custom field (e.g., ZZ1_REQCRLIMIT).

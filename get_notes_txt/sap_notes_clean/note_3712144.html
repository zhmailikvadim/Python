Symptom

SAP Smart Business runtime is used to expose custom dashboards. This app is loading in the development environment post upgrade. In system ED6 client 200 - Example: SW Actual Cashflow which is implemented using the Smart business runtime is not loading.

Environment

SAP Smart Business S4H - Run Time

Reproducing the Issue

  1. Login to the Fiori Launchpad and search for the app.
  2. Click on the app.

Cause

  • The ICF node for the ODATA Service was not activated in the system.

Resolution

  1. Open transaction SICF.
  2. Navigate to the path /default_host/sap/opu/odata/iwfnd/.
  3. Open the Gateway Catalog Service node.
  4. Ensure the service is active.
  5. To resolve the drilldown issue, access the following link:

https://sed601.adsdev.swcdev:44300/sap/bc/ui2/flp?sap-client=200&sap-language=EN#AnalyticObject-manage&/DisplayGenericReport/E.1603366602129" Analytic Object Manage

  1. Click on "Edit".
  2. Click on the "Configuration" tab.
  3. Select the dropdown "CashFlow by Liquidity Item". It will work the first time.
  4. Select another dropdown.
  5. Return to the dropdown option "CashFlow by Liquidity Item".
  6. If an error appears stating "[50005] - valueAxis: does not meet the minimum or maximum number of feeds definition", click on the "Settings" icon.
  7. Select "Cashflow" in the "Measure" section and activate it.

See Also

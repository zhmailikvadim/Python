Symptom

When using in standard class CL_LBA_STD_QUERY an exception is raised due to appeding value acitve "X" instead of appending the Hierarchy level.

Mehtod was introduced in 3340054 - LBA: Lightweight BICS API - functional roundups

Other Terms

CL_LBA_STD_QUERY;_SET_REQUESTED_COLUMNS; value acitve "X";include-hierarchy_active instead of include-hierarchy_level

Reason and Prerequisites

the method was assigning acitive "X" for hierarchy instead of giving the node level.

Solution

To solve the issue please apply the Support Package as mentioned below. In urgent cases you can use the correction instructions.

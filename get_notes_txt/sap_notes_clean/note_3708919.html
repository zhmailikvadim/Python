Symptom

  • Changes in the range values for "processing status of SPL screening" (/SAPSLL/ADRCON-/SAPSLL/BLKST) between SAP GTS 11 and SAP GTSE4H. 
  • When User Flag BP as “confirmed block” the status reflected in GTS 11 as  "Confirmed" but in E4H, the BP either “confirmed block” or no block, the Process status is updated as “1- Completed”

Environment

  • Global Trade Services (GTS)
  • Global Trade Services / Logistics Services (SLL)
  • SAP GTS Edition for Hana.

Reproducing the Issue

Perform SPL screening for the blocked BP's 

Cause

This behavior is part of the system's standard design.

Resolution

  • In GTS 11, the processing status 'Confirmed' is used. However, in GTS E4H, we've introduced a new SPL screening status for the 'Confirmed Block'.
  • The 'Confirmed' processing status is replaced by 'Completed', which is applied to the SPL statuses 'System Release' and 'Confirmed Block'.
  • No action is required for these two SPL statuses, hence the processing status is 'Completed'.

Keywords

/SAPSLL/ADRCON-/SAPSLL/BLKST, SPL processing status 'Confirmed', 'completed'.

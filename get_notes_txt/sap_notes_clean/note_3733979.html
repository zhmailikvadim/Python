Symptom

When processing a high volume of inbound CIF queues for the same material simultaneously, the system may experience extended processing times. This performance issue is caused by multiple parallel work processes waiting for the logical lock /ARMSUI/E_MATTSP to be released, which slows down the overall CIF processing.

Reason and Prerequisites

SAP Note 3732449 and SAP Note 3734378 must be implemented prior to this note.

Solution

The material timestamp update can now be decoupled from the main CIF inbound process using asynchronous queued Remote Function Calls (qRFC).

The system generates inbound queues with the pattern Z_<MATID> to ensure serialized processing per material.

This asynchronous processing is active by default. To provide a fallback option, a new configuration parameter SIM16 (Disable Asynchronous Update) has been introduced:
Space - Asynchronous qRFC update is active (Recommended).
X - Revert to the synchronous update.

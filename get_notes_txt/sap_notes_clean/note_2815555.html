Symptom

The Fiori Launchpad shows in a busy/wait state after a tile is opened which opens a new tab. This can occur with the following app types which, depending on configuration, may open in a new tab:

  • URL application
  • SAP GUI for HTML (WebGUI) transaction
  • Web Dynpro application

Environment

  • SAP Fiori Launchpad (ABAP)
  • SAPUI5 1.71.13+

Resolution

To resolve the issue, apply patch 17 of SAPUI5 1.71 or higher. See SAP Note 2856654 - ABAP SAPUI5 1.71 release

Workaround:

It is possible to workaround the issue by specifying in-place navigation for Web Dynpro and WebGUI applications so that they do not open in a new tab. There is no workaround for URL applications, the SAPUI5 version must be patched as indicated above. Please see:

  • (Parameters NAVIGATION_GUI_INPLACE & NAVIGATION_WDA_INPLACE): SAP Help - SAP Fiori Launchpad - Launchpad Configuration Parameters
  • SAP Help - SAP Fiori Launchpad - Setting Parameters in SAP Fiori Customizing

Symptom

The following changes are done:

- Sales/Purchase invoices are re-designed. The customizing for invoices is also changed, to address the reporting requirements. Sales/Purchase invoices are now selected based on Tax Mapping, and are maintained in V_BGSAFT_TAXMAP table. New key field was added for tax mapping - select if the tax is relevant for sales or purchase invoices. Customizing must be updated after implementing this Note.

- Taxes in general ledger entries - relevant for global accounts, can now be mapped in V_BGSAFT_ACCMAP. Tax is reported based on the mapping. If tax percentage is maintained, tax is calculated. If not, amount on the line is reported as tax amount.

- Switched to new schema 1.0.2.

- Possibility to report all products based on parameter was added. Parameter specific to report category BG_SAFT_PRODUCTS_REPORTED can be now maintained.

- Bug fixes and improvements

Other Terms

Bulgaria, BG, SAF-T

Reason and Prerequisites

This SAP Note is only relevant for Bulgaria.

Implement Note 3703754 before implementing this Note.

Solution

As a general rule SAP recommends that you implement the Support Package specified for your release. However, if you need to install a solution earlier, use the SAP Note Assistant to implement the correction instruction.

Symptom

When posting Goods Issue from EWM to ERP, the system always uses the current system date as the posting date. Backdated Goods Issue postings are not supported, which can result in “Posting only possible in periods YYYY/MM and YYYY/MM in company code X” errors when the previous financial period is closed.

Environment

SAP S/4HANA Cloud Public Edition

Reproducing the Issue

  1. Opened the Manage Outbound Delivery Orders app.
  2. Posted an outbound delivery with a Goods Issue date in a prior financial period.
  3. The system automatically applies the current system date as the posting date, and a posting period–related error is displayed.

Cause

This is standard system behavior in SAP S/4HANA Cloud, Public Edition.

When a Goods Issue is posted from EWM to ERP, the accounting posting date is automatically set by the system based on the current date. The date maintained in EWM represents the physical movement of goods and is used only for operational and logistical purposes.

The Actual Goods Movement Date maintained in EWM is intended for logistical tracking purposes only and is not considered for financial posting date determination.

Resolution

Backdated Goods Issue postings are not supported in SAP S/4HANA Cloud, Public Edition and cannot be performed via the frontend.

Backdated postings can only be executed by SAP at the backend. Please open a case under component SCM-EWM-DLP-2CL and share the list of deliveries that need to be posted.

Symptom

  • The salary positioning widget is displaying historical salary data when employees changes to past date.

  • Example: Employee sees historical salary positioning data when changing the date to 31st March 2026, even though the widget should only display current data.

  • Latest People Profile is enabled, when testing same steps in Legacy People Profile portlet does not show salary data

Environment

SAP SuccessFactors Employee Profile

Reproducing the Issue

  1. Navigate to the compensation information section and click "View All."
  2. Observe that the system displays current salary positioning data by default.
  3. Change the date to past date.
  4. Observe that the salary positioning widget displays historical salary data.

Resolution

  • The solution of this issue is currently under investigation with SAP Engineering team. Please follow this KBA for updates.
  • Workaround: remove end-user permission to the Salary Positioning widget for users who should not see history until a fix is available.

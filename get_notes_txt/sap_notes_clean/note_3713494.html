Symptom

If you are using Microsoft SQL Server as database provider with SAP Financial Information Management, you may encounter error when FIM tries to connect to the database.

Other Terms

fim mssql jdbc

Reason and Prerequisites

Starting with SAP Financial Information Management SP18 Patch 13, the embedded Microsoft JDBC Driver has been upgraded to a 10.2+ version.

A security-hardening change was introduced by this driver version : traffic between the Java app and SQL Server is encrypted by default using TLS.

Solution

To avoid blocking situation, SAP Financial Information Management now surfaces two additional settings.

  • Encrypt : allow to switch to previous mode by disabling encryption between SAP Financial Information Management and SQL Server.
  • Trust Server Certificate : to disable validation of the SQL Server Certificate.

Recommendation

In a productive environment, it is recommended to set encryption to on and that the driver validates the server certificate (trust server certificate set to off/false).

This will require import of the SQL Server certificate (or at least Certification Authority) in your Java truststore.

Symptom

The 'Add' button is missing from the 'Learning History' tile in the user's landing page (My Learning). How to make the button to be displayed?

"Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental." 

Environment

SAP SuccessFactors Learning

Reproducing the Issue

  1. Log into HCM > navigate to LMS as user >
  2. The 'My Learning' page opens >
  3. Look at the 'Learning History' tile >
  4. The 'Add' button is missing.

Cause

This 'Add' button is controlled by the user permissions 'Access Add to Learning History' and 'Access Add to Learning History for External History Records'. If the user doesn't have neither of these permissions, it's expected that the 'Add' button won't be displayed at the 'Learning History' tile.

Resolution

It has been confirmed by Product Management team that, within b2405 version onwards, the 'Add' button should be displayed as long as the users have either 'Access Add to Learning History' or 'Access Add to Learning History for External History Records' permissions.

IMPORTANT: Before the b2405 version, the 'Add' button in question was displayed only when user had the 'Access Add to Learning History' permission.

To check the user's role:

  1. Navigate to Learning Administration > People > Users >
  2. Search the required user > open the record >
  3. At the 'User Details' tab, search for the field 'Role' >
  4. Note the user's role.

To check the user's permissions:

  1. Navigate to Learning Administration > System Administration > Security > Role Management >
  2. Search for the user's role > open the record >
  3. At the 'Permissions' tab, click on 'Expand All' >
  4. Search for the required permissions. If they are not there, click on 'add one or more from list' >
  5. Click on 'Expand All' and check the 'Add' box for all permissions to be added >
  6. Hit 'Add' > at the pop-up, click on 'OK'.

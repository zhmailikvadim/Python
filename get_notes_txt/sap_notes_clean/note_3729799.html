Symptom

Procurement Unit field displays all PUs during requisition creation, regardless of the Application.Approvable.OrganizationalVisibilityPartitioning parameter settings. Users assigned to specific PUs are still able to view and select all PUs.

Environment

Vgeneric variant realms only.

Cause

The parameter Application.Purchasing.EditRequisitionPUFieldSGUnfiltered is enabled, which removes the visibility-based filtering for the Procurement Unit field on requisitions; when enabled, users can see all procurement units regardless of their assignments.

Resolution

If the requirement is to display only the PU assigned to the users, the parameter Application.Purchasing.EditRequisitionPUFieldSGUnfiltered must be disabled. This parameter specifies whether the Purchasing Unit field on purchase requisitions can be edited without visibility control restrictions. Regardless of whether the visibility control feature is enabled for a site, users can see the complete list of purchasing units of their organization when they edit the Purchasing Unit field on requisitions.

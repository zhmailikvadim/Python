Symptom

  • The Peru Purchase Ledger report does not display the exchange rate maintained in the document.
  • The report retrieves the exchange rate based on the document date instead of the manually entered exchange rate during posting.

Environment

SAP S/4HANA Cloud Public Edition.

Reproducing the Issue

  1. Access the "Run Statutory Reports" application.
  2. Open the Peru Purchase Ledger report.
  3. Observe that the exchange rate displayed in the report does not match the exchange rate maintained in the document.

Cause

  • The parameter PE_DEACTIVATE_CURRENCY_CONV is not activated for the reporting entity.

Resolution

  1. Navigate to the Setting up your statutory reporting activity ID: 101071.
  2. Select the relevant reporting entity.
  3. Select the report category: PE_PURCHASE_LEDGER
  4. Access the Enter Parameters Specific to Report category.
  5. Activate the parameter PE_DEACTIVATE_CURRENCY_CONV with the value X
  6. Refer to the Help documentation provided in the link below for detailed instructions on activating the parameter.
    Setting Up the Purchase Ledger

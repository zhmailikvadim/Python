Symptom

The following dump happens when extracting data from ST03N

Category               ABAP programming error
Runtime Errors         TIME_OUT
ABAP Program           SAPLSTUD
Application Component  BC-CCM-MON
Date and Time          25.03.2026 05:50:59
----------------------------------------------------------------------------------------------------

----------------------------------------------------------------------------------------------------
|Short Text                                                                                        |
|    Time limit exceeded                                                                           |
----------------------------------------------------------------------------------------------------

----------------------------------------------------------------------------------------------------
|What happened?                                                                                    |
|    Program "SAPLSTUD" has exceeded the maximum uninterrrupted runtime and has                    |
|    therefore been terminated.                                                                    |
----------------------------------------------------------------------------------------------------

Environment

  • SAP NetWeaver ABAP                                        
  • SAP NetWeaver Application Server for SAP S/4HANA    
  • ABAP PLATFORM - Application Server ABAP

Resolution

The dump is caused by the maximum runtime of a program exceeded, which is set by system profile parameter "rdisp/scheduler/prio_[high|normal|low]/max_runtime".

There are 2 options to aovid the dump from happening:
1. Either increase the value for the parameter "rdisp/scheduler/prio_[high|normal|low]/max_runtime"
2. Or read statistics for a smaller time period at each run

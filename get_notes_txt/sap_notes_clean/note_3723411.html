Symptom

There is a requirement to know how to get a list of dependent objects when executing transaction without debugging mode? 

Environment

SAP S/4HANA

Resolution

There are two ways from ABAP runtime measurement perspective.

Method 1:

  1. Start transaction SAT and enter the transaction code.
  2. Make sure the "Evaluate now" flag is set, then choose "Execute".
  3. The transaction is then started and embedded into the runtime measurement framework.
  4. So when the transaction is finished, it should be left via the green or yellow exit button in the standard menu.
  5. After that, SAT starts evaluating the runtime and there will be the involved programs in the "Hit list".

Method 2: 

  1. Open ADT's ABAP profiler
  2. Press ALT-F9, enter your transaction code, press "next", choose "Yes, I need the aggregated call tree".
  3. Close the embedded SAPGUI window when your transaction is finished.
  4. Open "Window" -> "Show View" -> "Other" -> "ABAP Traces".
  5. Double click the newest entry in the list. There are several result tabs.
  6. There will be the same hit list, but also the "Aggregated call tree" which may help get an idea of the program flow.

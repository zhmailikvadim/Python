Symptom


Program RFFOJP_T displays the error message F5263: "The difference is too large for clearing"

Environment

  • SAP R/3
  • SAP S/4HANA

Reproducing the Issue

  1. Run F110 to do the automatic payment
  2. Run the printing to generate the DME file
  3. Error message F5263: "The difference is too large for clearing" occurs.

Cause

  • The configuration is incorrect in the path spro/financial accounting/accounts receivable and accounts payable/business transactions/outgoing payments/automatic outgoing payments/payment media/country specific settings for payment media/domestic bank transfer(Japan)/define accounts for bank charges(Japan).
  • The posting key is set as 40 incorrectly for back subaccout in your system. It should be set as "50".

Resolution

  1. Navigate to the following path: spro/financial accounting/accounts receivable and accounts payable/business transactions/outgoing payments/automatic outgoing payments/payment media/country specific settings for payment media/domestic bank transfer(Japan)/define accounts for bank charges(Japan).
  2. Update the bank key configuration to "50" for the bank subaccount.
  3. Save the changes and verify the configuration by running F110 and printing the DME file again.

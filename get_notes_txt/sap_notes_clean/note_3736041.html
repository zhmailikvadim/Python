Symptom

Automated tests in the “Test Your Processes” app stop at the first or second step with an empty logon screen being displayed.

Environment

SAP S/4HANA Cloud Public Edition

Reproducing the Issue

  1. Configure Identity Authentication Service (IAS) and create a test user for test automation.
  2. Maintain a URL under “Maintain URL for Authentication” in the “Test Your Processes” app while using a SAP IdP (IAS).
  3. Launch an automated test case.
  4. Observe that the tests failing in the first steps with the empty logon screen in the detailed log.

Cause

The “Maintain URL for Authentication” setting is applicable only for non-SAP IdP configurations. When using SAP IdP, maintaining this URL is not necessary. 

Resolution

  1. If SAP IdP is being used, open the “Test Your Processes” app and go to “Maintain URL for Authentication”.
  2. Remove the maintained authentication URL and save.
  3. Run the automated test again, with SAP IdP no authentication URL is required.

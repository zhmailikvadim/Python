Symptom

  • In transaction CM01, capacity backlog from past weeks is displayed under the current week/current date.
  • The expectation is to see backlog remain in the original historical weeks.
  • No error messages are shown.

Environment

  • SAP ERP Capacity Planning (PP-CRP)
  • SAP R/3
  • SAP R/3 Enterprise
  • SAP ERP Central Component
  • SAP ERP
  • SAP Enhancement package for SAP ERP
  • SAP Enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA, on-premise
  • SAP S/4HANA Cloud Private Edition

Reproducing the Issue

  1. Start transaction CM01.
  2. Enter a work center and plant, then choose Continue.
  3. Display the capacity evaluation for current and future workload.
  4. Observe that operations/orders with backlog from previous weeks appear under the current week/current date.

Cause

Standard system behavior: CM01 rolls up backlog of past orders to the current date, so historical backlog is shown in the current period.

Resolution

Transaction CM01 shows backlog from past periods on the current date by design, there is no standard configuration to redistribute backlog to historical weeks.

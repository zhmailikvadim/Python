Symptom

Error: "Bot encountered unknown error from Central Invoice Management System" received from the Automated Report notification when using SAP Build to send email to Central Invoice Management.

Environment

SAP Ariba Invoicing

Reproducing the Issue

  1. Send a supplier invoice email to the configured email address for CIM.
  2. In the Business Process Automation (BPA) of the CIM sub-account, either run the trigger of the 6JX project or use the scheduled job; the emails are then triggered to be processed by the 6JX bot.
  3. An exception email is received with the message "Bot encountered unknown error from Central Invoice Management System eu10.cim.cloud.sap. Contact your Central Invoice Management system administrator."
  4. Checking the traces, in POST call Service we can see the error "irpa_core.error.RequesterError: self signed certificate in certificate chain"

Resolution

Check the certificates in your Windows Trusted Root Certification Authorities to see if you can find any certificate containing "DigiCert Global Root G5". 

You can launch the tool by executing certmgr.msc.

Alternatively, you can use PowerShell, you can execute the following command:Get-ChildItem Cert:\LocalMachine\Root | Where-Object { $_.Subject -like "*DigiCert Global Root G5*" }

If you cannot find the certificate, please download it from https://cacerts.digicert.com/DigiCertTLSRSA4096RootG5.crt.pem?utm_medium=organic&utm_source=google&… and add it to Trusted Certification Authorities > Certificates.

By trusting this certificate, the bot will be able to trust the connection to our server and the error should disappear.

If you want to use the PowerShell to install the certificate, you can, using the following command:Import-Certificate -FilePath "DigiCertGlobalRootG5.crt.pem" -CertStoreLocation "Cert:\LocalMachine\Root"

You might need to adapt the path to the location you downloaded the root certificate from DigiCert.

See Also

The RPA bot will be deprecated, and its successor is the "Configure Inbound Email" feature, as detailed on the SAP Help Portal.

Symptom

No error message appears in transaction ME51N when the Purchase Requisition (PR) quantity exceeds the Material Master Order Quantity defined in transaction MM03.

Environment

  • Materials Management 
  • SAP S/4HANA 
  • SAP ERP
  • SAP S/4HANA Private Cloud 

Reproducing the Issue

  1. Open transaction ME51N.
  2. Enter the required parameters (material, quantity, Plant, Storage Location, MRP Controller etc.) 
  3. Click "Check" or "Save" button
  4. Observe that no error message appears when the entered quantity exceeds the Material Master Lot Size defined in transaction MM03.

Cause

Transaction ME51N does not perform validation against the Lot Size from the Material Master (MM03, MRP1 view) for manual quantity entries. This field influences MRP-generated proposals, where quantities below the Lot Size are automatically rounded up. Manual PR entry in ME51N allows any quantity without error or warning.

Resolution

If validation against the Material Master Lot Size is required during PR creation in ME51N, implement the BAdI ME_PROCESS_REQ_CUST to create a custom check.

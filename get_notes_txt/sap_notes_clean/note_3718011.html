Symptom

Activation of a dictionary object (database table, DDLS, etc...) takes a very long (infinite) time or runs into out of memory errors.

Reason and Prerequisites

This is due to a bug when handling objects with the same name, for example a CDS view (V1) with source name 'A' selecting from a table with name 'A'.
Note that this only occurs, when a select asterisk is involved.

Solution

With this note, the bug is handled and the faulty activation can take place afterwards.

Symptom

  • What format should be used when entering the Security ID?
  • What is the Security ID field used for?
  • How should the Security ID be entered during Job Seeker submission?
  • Why is the Security ID field required?

Cause

The Security ID format is defined by the buyer organization and may vary depending on their internal requirements.

During Job Seeker submission, the system may display formatting instructions directly in the Security ID field. Some buyer organizations also configure automatic Security ID generation, in which case the field may be populated automatically after worker information is entered.

The Security ID is used to uniquely identify workers across assignments and help track worker history within the system.

Resolution

The system displays the required format in a blue tooltip when you hover over the Security ID field. This tooltip indicates the expected format defined by the buyer organization.
During Job Seeker submission, refer to the help text displayed next to the Security ID field. This message indicates the required format defined by the buyer organization.

Important considerations:

  • Follow the exact format shown in the field instructions.
  • If the Security ID is automatically generated, no manual entry is required.
  • If the required format is unclear, contact the buyer or Program Management Office (PMO) for clarification.

Note: SAP Fieldglass Support cannot define or modify Security ID formats, as these are configured by the buyer organization.

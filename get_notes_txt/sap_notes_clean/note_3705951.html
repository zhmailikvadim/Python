Symptom

  • Default view does not reflect per "Manage Global Settings" for other end users
  • Even though "Run Default View on Open" is already checked, the view is not opened by default.

Environment

  • SAP S/4HANA 
  • FPM Analytical GUIBB

Reproducing the Issue

  1. Login with UserA, and change the global setting view to View1, re-fresh the screen to confirm the default view is changed to View1
  2. Login with UserB, the default view is not changed to View1, even though the global setting view is set to View1 already

Cause


The global default view value is only valid for users without personalization. The UserB does have a personalization for the respective application and will therefore see their own defined default variant. The logic is also applied if the user defined default variant is empty which means the Standard Variant will be applied.

Resolution

  1. Check if UserB does have a personalization for the respective application
  2. The personalization can be checked and deleted with Web Dynpro Application WD_ANALYZE_CONFIG_USER. 
    Refer to KBA 3428059 - How to delete users personalization data e.g., Saved Search, Bookmark for FPM applications?

Symptom

You are using the SAP transaction IW31  to create the PM/CS Order. You assigned the priority and check the Final Due Date (FDD).

  1. Order creation in IW31 with priority selected on the initial screen:   If the priority is selected on the initial screen, the system calculates the FDD as today’s date + Final Due Date duration.
  2. Order creation in IW31 without selecting a priority on the initial screen:   If the priority field is left blank initially and selected later in the detail screen, the system calculates the Final Due Date (FDD) as Basic Finish Date (GLTRP) + Final Due Date duration. This is incorrect behaviour. According to documentation "When you select a priority, the final due date is determined. If priority is not selected, then the final due date will not be determined."
    Meaning selected at the moment the order is created, not later in the detail screen. The standart behaviour would be FDD stays blank.

Other Terms

EAM, PM/CS, Plant Maintenance, PM Order, Maintenance Order, IW31, IW32, LACD_DATE, CAUFVD-LACD_DATE, EAMS_LACD_CALCULATION, LACD, Final Due Date, 7300, PRIOK, Priority, LCOIHO1K.

Reason and Prerequisites

Programm error introdused via Note 3569851 - IW31/ IW32 : Final Due Date is set to Current Date in the Work Order.

Solution

Implement the attached correction.

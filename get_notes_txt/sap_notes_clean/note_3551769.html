Symptom

Customers who use HANA Smart Data Quality(SDQ) for Address Validation in DQM for SAP and use the STR_SUPPL1, STR_SUPPL2, STR_SUPPL3, or LOCATION fields for address data may experience unwanted data changes.

Reason and Prerequisites

Background: The ADRC/BUPA address model provides two lines above and two lines below the "Address Line".   These would typically be used for special delivery instructions.   For example, the ficticous address below uses two lines above and two lines below.

Example:

Example Company Name

Quality Dept

ATTN: Quality Manager

1324 Street Name

Any City, XX, 12345-1234

Please deliver to the loading dock

37.423021, -122.083739

DQM for SAP uses the following default field mappings for the HANA SDQ task configurations delivered with the product.

STR_SUPPL1 would have the value of 'Quality Dept' in the example above.   The STR_SUPPL1 field isn't mapped to a transform output field.  This results in the input value passed through to output.

STR_SUPPL2 woudl have the value of 'ATTN: Quality Manager' in the example above.   The STR_SUPPL2 field is mapped to the ADDR_REMAINDER_EXTRA_PMB_FULL transform output field.  The Global Address Engine concatenates the extra data provided and returns it in this transform output.   The length of the STR_SUPPL2 field is 40 characters.   Data loss may occur if the extra data is more than 40 characters long.

The STR_SUPPL3 field would contain ''Please deliver to the loading dock'.   The STR_SUPPL3 field is mapped to the STD_ADDR_POINT_OF_REF1_2 transform output field.   This transform output would contain any point of reference data for the address.   This field is 40 characters long.  Data loss may occur if the extra data is more than 40 characters long.

The LOCATION field would contain '37.423021, -122.083739' in the example above.   The LOCATION field isn't mapped to a transform output field.   This results in the input value passed through to output.

Reasoning:  The reasoning behind the default mapping is it is unusual to use all 4 extra info fields.   Passing the values of STR_SUPPL1 and LOCATION through allows customers to enter data and have it be unaffected by the Global Address Engine.   Mapping STR_SUPPL2 is mapped to ADDR_REMAINDER_EXTRA_PMB_FULL to try to avoid data loss in case customers do use more than one extra info field.   Mapping STR_SUPPL3 to STD_ADDR_POINT_OF_REF1_2 provides a place for Point of Reference information, which is important for delivery in some countries.

Solution

Customers can update the transform output field mapping in the SDQ tasks used by DQM for SAP.  Please see the 'Update Mapping Process' below for details on how to change the mapping.

Potential Change 1:  Customers who are confident the STR_SUPPL* and LOCATION fields don't contain any address data needed by Global Address Cleanse to validate the address could remove the values in the 'TRANSFORM_OUTPUT' field for these records.   This would result in the input values being passed through without being changed by Global Address Cleanse.

Potential Change 2: Customers who experience data loss during the concatendation of the extra data into STR_SUPPL2 could map STR_SUPPL1 to the ADDR_EXTRA_FULL transform output and STR_SUPPL2 to ADDR_REMAINDER_EXTRA_FULL.  This would result in the extra data being split over two fields instead of 1.

Potential Change 3: Customers who would like to add custom logic to field the extra data could implement the /FLDQ/SPOT1_TRANS_PV_POST enhancement spot and or the /FLDQ/SPOT1_TRANS_PV_PRE enhancement spot.

Update Mapping Process: Follow the steps below to update the transform output field mappings.  Please note: Implementing note 3548266 is necessary to see the ADDR_EXTRA_FULL and ADDR_REMAINDER_EXTRA_FULL transform outputs in the list for possible transform outputs.

  1. Maintain the SIMDQ_OUTP_MAP table in SM30.
  2. Limit the data shown to a single task.
    • Select 'By Contents' from the 'Selection' menu.
    • Check the box by 'Service ID' and select the Apply button.
    • Enter the service ID for the task you want to update and select the Apply button.Limit the data shown to a single task.
  3. Update the values in the TRANSFORM_OUTPUT field and select the Save button
  4. Update the SIMDQ_TASK_UPDT table.   This will cause the task to be updated the next time it is run.
    • Maintain the SIMDQ_TASK_UPDT table in SM30
    • Select the UPDATE_DEFINITION checkbox for the appropriate task and select the Save button.

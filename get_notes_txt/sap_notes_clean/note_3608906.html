Symptom

SWPM reports error while installation service "MaxDB Database Configuration on all additional MSCS Nodes" in the step "Bring_SAP_DBTech_Online". In sapinst_dev.log there is follwing information reported:

Execution of the command "C:\Windows\Cluster\SAPDBMSCSMan.exe -o <SID>" finished with return code 1. Output:

18751 ERROR: Bringing resource '<SID>' online failed, rc = 997.

Other Terms

70SWPM,  SAPDBMSCSMan.exe

Reason and Prerequisites

You are using Software Provisioning Manager for the SAP NetWeaver 7.0X-based (70SWPM10SPXXX) installation.

Solution

As long the software provisioning tool is not providing the fix use following workaround:

In MaxDB config directory <drive>:\sapdb\<DBID>\data\config copy the file <SID>.01 to the file <SID> (i.e. SDB.01 to SDB) Afterwards restart SWPM.

For installations of the standalone engines such as liveCache or SAP Content Server consider using the SWPM from the SAP NetWeaver 7.X-based installation (SWPM10XXX) as it has already the correction.

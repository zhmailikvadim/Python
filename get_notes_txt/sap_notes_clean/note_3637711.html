Symptom

During operation of apps like "Maintain External Resources", one is tries to create a resource and bind to an existing SU01 user (classic), either via manual process or via import function.

The task fails while the application log shows the following message:

  • "You are not authorized to change users in group"

Environment

  • SAP S/4HANA Cloud Private Edition
  • SAP S/4HANA OnPrem

Cause

During the assignment to a business partner, a change has to be made to the user. In case this user belongs to a group, additional check is performed, against (S_USER_GRP -> ACTVT = 02), as shown in the error message:

Resolution

The user performing the task must be granted the mentioned authorization (S_USER_GRP -> ACTVT = 02), otherwise the assignment cannot complete, ending in a failed operation.

Keywords

user, permission denied, user group

Symptom

After upgrading SAP HANA Database, an alert is displayed "Alert ID 92: Root keys of persistent services are not properly synchronized".

Environment

SAP HANA Platform Edition 2.0 SPS08 and higher.

Reproducing the Issue

  1. Perform an upgrade for HANA database to 2.0 SPS08.
  2. After upgrade, you may observe the Alert.

Cause

  • For Revisions <= 2.00.084
    This is a false alert, details are documented in SAP Note 3159727 - HANA Alert 92: 'Root keys of persistent services are not properly synchronized' is a False Alert.

  • For Revisions > 2.00.084
    The "Root Keys of Persistent Services are not Properly Synchronized" alert (ID 92) was removed in the code. But you may still observe this alert even after upgrade.
    This is because during the upgrade, there can be a short transient window while statistics/alert definitions are migrated. In that window the legacy alert may still trigger once and then auto-clear. 

Resolution

  1. Verify if the alert still exists by running the following SQL query:
    SELECT * FROM _SYS_STATISTICS.STATISTICS_ALERT_INFORMATION WHERE ALERT_ID = 92;
  2. Check the schedule status using the following query:
    SELECT * FROM _SYS_STATISTICS.STATISTICS_SCHEDULE WHERE ID = 92;
  • If no results are returned from the above queries, the alert has been successfully removed and no further action is required.

  • If results are returned, disable the alert using the following command:
    UPDATE _SYS_STATISTICS.STATISTICS_SCHEDULE SET STATUS = 'Inactive' WHERE ID = '92';

  • SAP Note 3159727 - HANA Alert 92: 'Root keys of persistent services are not properly synchronized' is a False Alert.
  • SAP Note 2445867 - How to handle encryption-related alerts in SAP HANA.

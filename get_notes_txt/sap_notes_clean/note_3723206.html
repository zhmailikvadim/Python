Symptom

The key metric "Credit Limit Usage" shows NaN %, instead of showing the correct values.

Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.

Environment

  • SAP Sales Cloud with Intelligent Sales Addon 
  • SAP Sales Cloud Version 2
  • SAP Service Cloud Version 2 

Reproducing the Issue

  1. Go to Customer -> Open one customer 
  2. Go to Customer Insights Tab 
  3. For key metric "Credit Limit Usage", it shows NaN %

Cause

The standard iflow “Request Credit Amount Data from SAP Business Suite” uses 0000 as the default credit segment ID. If this default value is not used in the S4 system, S4 cannot return the expected data. In addition, the current iflow design supports only one credit segment ID, and multiple credit segment IDs are not yet supported.

Resolution

If there's only one credit segment ID in S4, please follow the steps below. 

  1. Open the standard iflow "Request Credit Amount Data from SAP Business Suite" in the CPI
  2. Click Configure button on the top right 
  3. Go the third tab "More"
  4. For field "DefaultCreditSegmentId", delete 0000 and provide the the target credit segment ID 
  5. Click Save
  6. Redeploy this iflow 
  7. Retest the scenario 

If there're multiple credit segment IDs in S4, please check below workaround.

  1. Copy the standard iflow "Request Credit Amount Data from SAP Business Suite"
  2. Add the custom logic to the copied version to pass the "DefaultCreditSegmentId" dynamically 
  3. Create a custom key metric per the help documentation(Intelligent Sales Add-On for SAP Sales Cloud/SAP Sales Cloud Version 2) to consume the response from this customized iflow

Additionally, support for pre/post block for the standard iflow "Request Credit Amount Data from SAP Business Suite" is included in the product roadmap, but the timeline is not concrete. This KBA will be updated once the timeline is available.

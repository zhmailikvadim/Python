Symptom

Receive the error "Field * must not start with cust. namespace in /x/-namespace" in SCFD_EUI. 

  • * stands for the field name.
  • /x/ stands for the namespace.

Environment

  • SAP S/4HANA
  • SAP S/4HANA Cloud Private Edition

Reproducing the Issue

  1. Access to SCFD_EUI transaction.
  2. Enable the Custom Fields.
  3. Receive error message "Field * must not start with cust. namespace in /x/-namespace".

Cause

The namespace /x/ has been maintained to be used in transaction S_ATO_SETUP. This results in generated object names, which is required to use with this namespace.

However, the created objects do not start with the specified namespace. Therefore, it caused the error message.

Resolution

Choose either of one the below solutions:

  1. Change the name of the custom field * to /x/*. Then enable it via transaction SCFD_EUI. => This is the clean, long-term fix that aligns with the namespace and SCFD_EUI requirements.
  2. Change the namespace in transaction S_ATO_SETUP to namespace that was currently set on the field name. Then, enable the field via transaction SCFD_EUI. After enabling it, reset the maintained namespace to /x/ again in transaction S_ATO_SETUP if needed.

An example, there is error with the field name ZDD_ABC and namespace /ZAA/. Then:

For option 1, change the CF name to /ZAA/ABC.

For option 2, change namespace in S_ATO_SETUP to either ZDD or ZZ, then enable the custom field via transaction SCFD_EUI.

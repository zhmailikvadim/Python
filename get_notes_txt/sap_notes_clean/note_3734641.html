Symptom

  • Email notifications for messages triggered under operation applications show a total number bigger than 20 for "Total Messages" but the "Additional Information" section lists only 20 message entries.
  • The notification list is incomplete, so users cannot view all affected messages directly from the email.

Disclaimer: "Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental."

Environment

  • Cloud ALM
  • operation applications

Reproducing the Issue

  1. Receive an email notification that aggregates more than 20 messages (for example, an IDoc error alert).
  2. In the email’s “Additional Information” block, observe that the total indicates more than 20 messages while only 20 items are listed.

Resolution

This is feature by design, alert notifications display a maximum of 20 message entries.

The full set of messages contributing to the alert is available in the alert inbox under Cloud ALM.

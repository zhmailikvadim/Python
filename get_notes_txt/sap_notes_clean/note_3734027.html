Symptom

In SAP Business One Integration Framework version 2.0.50, when you execute a scenario that calls an internal queue (DBQO) after a Service Layer adapter step, the following error occurs:

"Character not allowed in XPath, ':'."

Reproducing the issue

The error is reproducible when:

  • The internal queue (DBQO) is triggered after a Service Layer adapter.
  • The scenario reads a CSV file through an SLD configuration (for example, step "IAG.getTimeSheetFile").
  • The SLD ID of the SAP Business One demo database is maintained in configurationFile.xml.

If the same internal queue (DBQO) is executed without the Service Layer adapter step, the error does not occur.

Expected Behavior:

No error occurs when calling the internal queue (DBQO).

Actual Behavior:

Error occurs: "Character not allowed in XPath, ':'."

Cause

The issue relates to the parameter “Adjust to Smart Message = True”, which can affect all Version 2 adapters in SAP Business One Integration Framework.

When this parameter is enabled, the message is generated dynamically using a style sheet. During this process, the transformation copies the entire message content without explicitly handling attributes.

In this specific scenario, the Service Layer call returns an HTTP error, which is stored in an attribute named “HTTPmsg”. The presence of special characters (such as “:”) in this attribute leads to an invalid XPath expression, resulting in the error.

Solution

SAP intends to provide a feature package/patch in order to solve the problem described. The section References below will list the specific feature package/patches once they become available. The corresponding Info file of the feature package/patches in SAP Service Marketplace will also show the SAP Note number. Be aware that these references can only be set at feature package/patch release date. SAP will deliver feature package/patches only for selected releases at its own discretion, based on the business impact and the complexity of the implementation.

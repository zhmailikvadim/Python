Symptom

  • On-demand payroll execution is only supported for individual employees (single PERNR) by standard functionality in Payroll Control Center.
  • Require a solution to run on-demand payroll for multiple employees simultaneously, typically for scenarios involving new hires and terminations.

Environment

  • SAP SuccessFactors Employee Central Payroll
  • SAP ERP 
  • Payroll Control Center (PCC)
  • SAP S/4 Hana

Reproducing the Issue

  1. Attempt to execute on-demand payroll for multiple employees simultaneously in Payroll Control Center.
  2. Observe that the standard functionality only allows execution for individual employees (single PERNR).

Cause

Standard functionality in Payroll Control Center is designed to support on-demand payroll execution at an individual employee level for targeted correction and validation during alert resolution.

Resolution

  1. Only Adhoc off cycle can run On-demand payroll.
  2. Note for the off cycle type 'D' (On-demand) only supported paytypes are A, B and C as per the enclosed SAP help documentation.
  3. The solution is not possible for off cycle type ' ' (blank) as it is not supported by Payroll Control Center.

Keywords

On-demand payroll, PCC, multiple employees, batch processing, runtime class, planned off-cycle, pay type blank, regular payroll, customization

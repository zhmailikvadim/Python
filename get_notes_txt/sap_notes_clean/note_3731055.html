Symptom

After transferring an employee between companies and synchronizing the Business Partner (BP) via replication from SuccessFactors Employee Central (SF/EC) to S/4HANA, the FLVN00 (Supplier) role is not present in the BP transaction. Deleting and recreating the BP does not resolve the issue.

Environment

  • SAP S/4HANA on-premise
  • SAP S/4HANA private cloud edition

Reproducing the Issue

  1. Transfer an employee from one company to another via replication from SF/EC to S/4HANA.
  2. Execute the transaction to synchronize the Business Partner.
  3. Check the synchronized Business Partner in transaction BP.
  4. Observe that the FLVN00 (Supplier) role is missing.

Cause

There is an existing vendor for the employee which is not linked to any Business Partner. This causes inconsistency because vendors for HR employees must be linked to a BP in S/4HANA.

Resolution

  1. Check the Vendor in Table LFB1: Verify if there is an existing vendor for the employee.
  2. Delete the Vendor: If possible, delete the vendor (EE11044086) or at least the corresponding entries from tables LFA1 and LFB1. Ensure the BP (EE11044086) is not deleted.
  3. Execute SINGLE Report: Run the SINGLE report again. This should recreate the vendor and link it to the BP (EE11044086).
  4. Manual Link Creation: If deleting the vendor is not possible, manually create the link between the vendor and the BP in table CVI_VEND_LINK. Set VENDOR as the Vendor number (EE11044086) and PARTNER_GUID as the BP GUID taken from table BUT000.

Keywords

SAP SuccessFactors Employee Central, S/4HANA, BP synchronization, vendor, FLVN00, supplier role, inconsistency, table LFB1, table LFA1, table CVI_VEND_LINK, table BUT000

Symptom

Replication of Point of Deliveries (POD) from SAP S/4HANA to SAP Service Cloud Version 2 is not functioning as expected.

Observed behaviors:

  • The integration flow (iFlow) "Replicate Utilities POD Object from S4HANA" is referenced but contains no guidance for initial or delta loads.
  • The iFlow does not appear to be invoked or serviced during replication attempts.
  • No outgoing messages are triggered from SAP S/4HANA.
  • SAP Service Cloud V2 does not provide a dedicated user interface to display POD objects.

Environment

  • Product: SAP Service Cloud Version 2
  • Integration for SAP Sales/Service Cloud

Reproducing the Issue

  1. Configure the Utilities replication scenario from SAP S/4HANA to SAP Service Cloud V2 via SAP Cloud Integration (CPI).
  2. Review the iFlow "Replicate Utilities POD Object from S4HANA" and note the absence of instructions for triggering initial or delta loads.
  3. Attempt POD replication and observe that no outbound messages are generated in SAP S/4HANA.
  4. Check SAP Service Cloud V2 and confirm that there is no dedicated POD UI to view replicated POD data.

Cause

  • SAP Service Cloud Version 2 currently does not provide a dedicated POD user interface. Display capability is planned for future releases.
  • POD data is embedded within Utilities technical objects and premise master data, not replicated through a standalone POD iFlow.
  • Replication relies on event linkage in SAP S/4HANA IS-U rather than a dedicated iFlow trigger.

Resolution

1. Manual / Initial Load

To initiate a full or manual load of Utilities technical objects—including POD information—run the following report in SAP S/4HANA IS‑U: ISU_C4C_TECH_OBJECTS_REPL

This program sends technical object and premise data that contain embedded POD details to SAP Service Cloud V2 via the deployed integration flows.

2. Delta Replication

Ensure that event linkage for Utilities objects is active:

  1. Go to transaction SWE2 in SAP S/4HANA.
  2. Validate that event linkage for relevant Utilities objects (e.g., technical objects, premise updates) is active.
  3. When active, delta changes to POD‑relevant data will be automatically replicated.

3. SAP Service Cloud V2 UI Limitation

  • A dedicated UI to display PODs in SAP Service Cloud Version 2 is currently not available.
  • This feature is targeted for a future roadmap release.
  • Customers can monitor progress in the SAP Road Map Explorer and may submit enhancement requests through the SAP Influence Portal.

4. Integration Setup Notes

Refer to the Integration Guide for SAP Sales and Service Cloud to:

  • Deploy the required Utilities master data iFlows.
  • Maintain necessary outbound service configurations in SAP S/4HANA.
  • Validate communication arrangement and endpoint activation for Utilities replication scenarios.

See Also

  • POD data is replicated only as part of broader technical object and premise replication; a standalone POD request is not currently supported.
  • Absence of the POD UI does not prevent POD data from being stored in SAP Service Cloud V2; it affects only the display capabilities.

Symptom

The PO item quantity is not being extracted and updated as the requested quantity in the Sales Order request when using the Fiori App F4920 "Create Sales Orders – Automatic Extraction."

Environment

  • SAP Enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA
  • SAP S/4HANA Cloud Private Edition

Reproducing the Issue

  1. Access the Fiori app "Create Sales Order – Automatic Extraction."
  2. Click on "Upload File."
  3. Enter the Company Code.
  4. Click "Browse" and upload the PO in PDF format.
  5. A Sales Order Request is generated, and the processing flow indicator turns green once the PO data is extracted and processed.
  6. Review the Sales Order Request and note any missing or extracted data.

Cause

The possible reason is that no UoM has been extracted or the extracted UoM cannot be converted into an internal key.

Resolution

According to related SAP Help article, If no UoM has been extracted or the extracted UoM cannot be converted into an internal key, the system leaves both the requested quantity and UoM empty.

Create Sales Orders - Automatic Extraction | SAP Help Portal

If no UoM has been extracted , please consider following workaround to use BAdI SD_SOR_AUTO_XTRCTN_MODIFY to determine a unit. For this please refer to SAP Documentation:

App Extensibility: Custom Proposal for Sales Order Request Fields (BAdI) | SAP Help Portal

Furthermore, please also make sure the correction code in following SAP Note has already been implemented in your system so that BAdI SD_SOR_AUTO_XTRCTN_MODIFY will work properly:

3379857 - Create Sales Orders – Automatic Extraction App: Requested Quantity Passed to BAdI Despite Failed UoM Determination
3631306 - BADI Based UoM Determination Fails to Set Quantity Field in Sales Order Request.

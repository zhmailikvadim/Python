Symptom

When a MDX inbound update changes the forecast basis type from ForecastBasedOnProfile to ForecastBasedOnValues and simultaneously changes the time series type, the system does not update tables /US4G/SETTL_MALO, ELPASS, or EUFASS.

This issue is relevant for all market roles.

Other Terms

MaCo, Settlement MaLo, Load Profile, /US4G/CL_MDX_UBD_SETTLEMENT

Reason and Prerequisites

Program error

Solution

Implement the corrections of this SAP Note or import the relevant support package.

After implementing this SAP Note, the system correctly processes MDX inbound updates when both forecast basis type and time series type are changed. You can verify the fix by creating a new MDX data container and performing an update. After update, in table /US4G/SETTL_MALO, the system delimits the current entry and creates a new entry with forecast basis type ForecastBasedOnValues. In tables ELPASS and EUFASS, the system delimits the current load profile entries.

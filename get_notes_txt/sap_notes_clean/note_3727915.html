Symptom

When accessing capacity allocation transactions (e.g., CM01) and selecting the Standard Overview, the work centers are not displayed in the expected sequence.

“Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.”

Environment

  • SAP ERP Capacity Planning (PP-CRP)
  • SAP R/3
  • SAP R/3 Enterprise
  • SAP ERP Central Component
  • SAP ERP
  • SAP Enhancement package for SAP ERP
  • SAP Enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA, on-premise
  • SAP S/4HANA Cloud Private Edition

 

Reproducing the Issue

  1. Execute transaction CM01.

  2. Enter the required selection criteria (e.g., Plant and/or Work Centers).

  3. Choose Standard Overview.

  4. Observe that the work centers are not displayed in alphabetical sequence.

Cause

This behavior is standard system design.

The sorting logic in capacity leveling transactions is based on the internal Work Center ID rather than the Work Center name.

Resolution

The sorting logic in capacity leveling transactions such as CM01 is based on the Work Center internal ID (OBJID) rather than the Work Center name (ARBPL).

From a technical perspective, the system retrieves and processes the work centers using internal identifiers. As a result, the display order in the Standard Overview does not follow the alphabetical order of the work center names.

For additional technical reference, the program include MC65AFL1, specifically FORM LIST_KAP_UEBERSICHT, is responsible for building the overview list displayed on the screen. Within this routine, the system processes the following loop:

LOOP AT KUHIECAP.  "<< HERE

  CLEAR KZ_POOLK.
  INDEXHIECAP = SY-TABIX.
* ARBKAPID = KUHIECAP_ART-HKAPID.
  ARBKAPID = KUHIECAP-HKAPID.
* ARBARBID = KUHIECAP_ART-HARBID.
  ARBARBID = KUHIECAP-HARBID.
* KZ_POOLK = KUHIECAP_ART-POOL.

The internal table KUHIECAP contains the work centers sorted by ARBID (OBJID). This structure does not contain a field representing the Work Center name (ARBPL).

Keywords

CM01, CM02, CM03, CM04, CM05, CM06, CM07, capacity leveling, sort, sequence

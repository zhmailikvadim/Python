Symptom

  • In Transport Checks, the Cross-Reference Check summary shows many critical issues, but when opening the details no conflicts show
  • Clicking the Cross-Reference Check sometimes opens the Downgrade Protection details instead of the Cross-Reference Check details.
  • The details pop-up does not correspond to the summary counts.

Disclaimer: "Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental."

Environment

  • Cloud ALM
  • Feature: Transport Checks (Cross-Reference Check and Downgrade Protection) in SAP Cloud ALM

Reproducing the Issue

  1. Go to Features in SAP Cloud ALM.
  2. Select a feature that contains transports.
  3. Open the Transport Checks tab and run the checks.
  4. After completion, click the line showing problems: the details dialog does not match the summary or open the Downgrade Protection result instead.

Cause

  • The feature contains multiple source tenants mapped to the same target tenant.
  • The import transport check supports a 1:1 source-to-target tenant relationship per feature; with multiple sources to one target in a single feature, only one result can be opened and the details may not align with the summary.

Resolution

  1. In Transport Checks, ensure you open the Cross-Reference Check result and not the Downgrade Protection result.
  2. Review the feature’s landscape assignments. If multiple source tenants point to the same target tenant within a single feature, split them so each feature maintains a 1:1 source-to-target tenant relationship.
  3. Rerun Transport Checks for the adjusted feature(s).
  4. Open the Cross-Reference Check result and verify that the details dialog corresponds to the summary counts.

Symptom

OnChange Business Rule to Validate HRIS Element not working as expected when updates are performed through Quick Actions or via Joule. However, the same rule behaves as expected in People Profile.

Environment

  • SAP SuccessFactors Employee Central
  • SAP SuccessFactors HCM Suite

Reproducing the Issue

  1. Configure a Business Rule with scenario Validate HRIS Element
  2. Assign with Event Type "OnChange";
  3. Navigate to People Profile;
  4. Attempt to edit the desired field;
  5. The business rule triggers and HRIS Element is validated;
  6. Go to the Home Page > Quick Actions
  7. Attempt to make the same change from step 4.
  8. Noticed the validation does not occur.

Cause

Validation is not supported via OnChange via Quick Actions and Joule. 

Note: the configured rule will trigger but the conditions will not be met. 

Resolution

As a workaround to achieve the expected behavior:

  1. Create a rule using the Validate HRIS Element scenario.
  2. Configure the IF and THEN conditions as required.
  3. Configure the rule to run with event type OnSave for the HRIS element.

Symptom

In Intercompany Matching and Reconciliation (ICMR), when the data source level filter of the matching method is changed, items that do not meet the new filter criteria remain in table ICADOCM. Running ICADR in the mode "Synchronize Document Deletion from Source" does not mark the unexpected items as deleted in ICADOCM. 

Environment

  • Group Reporting
  • SAP S/4HANA
  • SAP S/4HANA Cloud Private Edition

Reproducing the Issue

  1. Open table ICADOCM and review items rolled in under matching metho.
  2. Run transaction ICADR with matching method in mode "Synchronize Document Deletion from Source".
  3. Observe that targeted items remain in ICADOCM

Cause

ICADR is a tool designed to ensure consistency between rolled-in data and source data. It is not responsible for managing Matching Documents and their line items.

Resolution

This is the standard behavior.

The Manage Matching Documents app F3868 can manage/delete line items from ICADOCM as a whole, group by Matching Document Number.

There's no approach to partially delete line items from ICADOCM. If certain line items need to be deleted, delete the Matching Document as a whole first, then rerun the matching process to roll in the correct line items.

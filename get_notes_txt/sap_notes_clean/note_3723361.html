Symptom

When a custom delivery address is created for the B2BUnit of logged-in user in the SAP Commerce Cloud frontend and selected during checkout, it is not transferred to SAP S/4HANA during order submission.

Instead, the order in SAP S/4HANA is created using the default master data address of the B2BUnit for the user, resulting in an incorrect delivery address.

Environment

  • SAP Commerce Cloud
  • SAP Commerce Cloud, integration extension pack

Reproducing the Issue

  1. Login with the user's email.
  2. Add product to the cart.
  3. On Shipping Address, select the newly created delivery address.
  4. Continue the checkout process and place the order.
  5. Check in the debug log, the request payload being sent to S/4HANA message, the selected address details are not included in the payload.

Cause

Property toggle.saps4omservices.addressReplication.enabled controls if DefaultS4OMRequestPayloadModifierHook is being used for adding address to the request payload only while placing an order. By default it is set to false in saps4omservices extension project.properties file. Thus newly created address will not be included in the payload sending to S/4HANA during order creation.

Resolution

Enable the property toggle.saps4omservices.addressReplication.enabled to true in hcs_common service properties in Cloud Portal.

Be aware, applying the change will cause all application services being rolling restarted.

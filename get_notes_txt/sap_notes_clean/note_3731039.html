Symptom

  • Archiving from a source system to Sybase IQ fails when processing date columns; the same date inserted into a simple local table does not fail.
  • Running a SELECT on the affected table in the target database returns: "Date is invalid (111) -- (dflib/dfe_SACast.cxx 361) SQLCODE=-1006168, ODBC 3 State='HY000'."
  • Some rows contain date values outside the supported range '0001-01-01' to '9999-12-31'.
  • Selecting rows with predicates that isolate invalid dates can also raise the same error.

Environment

SAP IQ16.1 SP05 PL14 

Reproducing the Issue

  1. Archive or load data from the source system into Sybase IQ that includes date/timestamp columns.
  2. Ensure the incoming data contains invalid temporal values (for example, '0000-00-00' or dates earlier than '0001-01-01').
  3. Run a SELECT (for example, SELECT * FROM TABLE_NAME) or a LOAD TABLE into the target table that includes those rows.
  4. Observe the error: "Date is invalid (111) -- (dflib/dfe_SACast.cxx 361) SQLCODE=-1006168, ODBC 3 State='HY000'."

Cause

  • One or more columns contain invalid or out‑of‑range temporal values (for example, dates < '0001‑01‑01'), which cause conversion failures during SELECT or LOAD operations.
  • Sybase IQ does not support the non‑standard "empty" temporal date '0000‑00‑00' or timestamp '0000‑00‑00 00:00:00'.

Resolution

  • There is currently no target SAP IQ Service Pack scheduled for this issue (SAP CR 831884). It has been assigned to our Product Group. Please mark this KBA 0003731039 as a Favorite to be automatically notified when the CR status is changed.
  • There is also existing another Change Request HANADATALAKE-18154 - Enable empty timestamps for HANA-compatible databases for HDLRE.

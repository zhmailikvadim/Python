Symptom

This KBA introduces how remuneration statement variants are determined in payroll driver. "Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental." 

Environment

  • Human Capital Management (HCM)
  • SAP ERP
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP SuccessFactors Employee Central Payroll
  • SAP S/4 HANA

Resolution

There are two scenarios:

Scenario 1 – When PE51 remuneration statement is applied, the variant is selected from variant of program RPCEDT*0/H**CEDT0 (* means country molga and ** means ISO code of the country stored in table T500L) as shown in below example.

  1. Take RPCALCJ0 for Japan as an example. The variant for remuneration statement is ‘ZF04FORM’.
  2. Run program RPCEDTJ0 via t-code SE38.
  3. The same variant can be viewed and edited in program RPCEDTJ0.

Scenario 2 – When HR form is applied, the variant is selected from variant of print program defined in t-code HRFORM as shown in below example.

  1. Take RPCALCU0 for US as an example.  The variant for HR form ‘SAP_PAYSLIP_US’ is ‘TEST’.
  2. Run t-code HRFORMS and Display ‘SAP_PAYSLIP_US’.
  3. Select print program /1PYXXFO/SAP_PAYSLIP_US_PRNT.
  4. Run the print program via t-code SE38.
  5. The same variant can be viewed and edited in the print program.

Keywords

Variant, Remuneration statement, Payslip, Payroll statement, PE51, HRFORMS

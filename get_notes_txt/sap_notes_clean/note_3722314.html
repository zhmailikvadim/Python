Symptom

  • Translations maintained for an analytical query are not displaying for one specific field.
  • Selection parameters and all result attributes are correctly translated except for one field.

Environment

  • Fiori based Query Designer

Reproducing the Issue

  1. Login to the system with logon language set to German (DE).
  2. Navigate to the app 'Custom Analytical Queries'.
  3. Select the specific report which needs to be translated.
  4. Open the report.
  5. Observe that selection parameters and result attributes are correctly translated except for one field.

Cause

  • The fields “Type of Contract” and “Duration of Agreement” are custom-developed fields derived from the custom CDS views ZDF_1CT_DURATION_SRH and ZDF_1CONTRACT_TYPE_SRH.
  • The UI labels for these custom fields are not automatically translated into German.

Resolution

  1. Access transaction code SE63 and select "Short Text".
  2. Navigate to A5 and select "DDLS - CDS Views".
  3. Enter the CDS information and maintain the texts.
  4. Edit the text maintenance for the German language. For example, use "DE" as a prefix to indicate the text in German.
  5. After saving the translations, activate the corresponding CDS view or related DDIC object (e.g., Data Element) with T-Code SE80 or ABAP Development Tools to refresh the metadata buffer and ensure the updated texts are applied at runtime.

See Also

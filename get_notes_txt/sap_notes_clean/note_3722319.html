Symptom

In the Customer List item grid, when sorting is applied and the layout is saved, the default layout is not loaded when the list is open. Additionally, sorting based on the second field does not function as expected.

This SAP Note corrects the layout persistence and multi-level sorting logic, ensuring that, the saved default layout is properly loaded when reopening the list, and Sorting based on the second field works correctly. The correction applies to all similar list applications, including Product List, Flexible Group List, Membership List

Other Terms

/IRM/GCULM, Sorting, Item Grid, Membership, Product List, Flexible Group

Reason and Prerequisites

This is caused by program error

SAP Note#3722328 is available for ECC Release

Solution

Please implement the correction instructions attached. These corrections are only valid if you have applied the Support Packs.

Release V/4 - 2022 V2, 2023, 2023 V2, 2023 V3 and 2023 V3 Support Pack 1

 Use transaction SNOTE to implement the correction instructions specified below and activate the affected program object.

Symptom

A submitted Timesheet becomes stuck in the Approval Paused status.
When attempting to restart the approval process, the following error appears:

Time Sheet is paused. One of the following conditions is true:

  1. Approvers have no approve/reject permission for the roles defined in the approval group Time Sheet Approval Group
  2. There is no approver with enough signature authority to approve this Time Sheet
  3. There are no approvers assigned to the user role

Cause

The Approval Paused status appears when the system cannot identify a valid approver based on workflow rules.
Common causes include:

  • Approvers missing approve/reject permissions for the relevant approval group.
  • No approver with sufficient signature authority.
  • No approvers assigned to the user role in the approval workflow.

However, if all the above conditions are met and the timesheet is still stuck in Approval Paused, a frequent cause is that the timesheet was submitted under an incorrect, outdated, or previous cost center.
Because approval routing relies on cost allocation, using an incorrect cost center results in no valid approver being identified, causing the workflow to pause.

Resolution

Verify that the timesheet is submitted with the correct cost center. If the timesheet was created using an outdated or incorrect cost center, revising and resubmitting it with the correct one resolves the issue and allows the approval workflow to continue.

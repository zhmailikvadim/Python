Symptom

SAP has released Note 3573619 to provide the solution for IR8A file submission for YOA2026. The LC Note delivers the new feature 25BKC in the current Change Request (CR), which maps the Bank Key (from Infotype 0009) to the officially published Bank Code.

However, for special scenarios such as PayNow, maintaining a Bank Code may not be required. Therefore, this SAP Note has adjusted the relevant logic in Singapore Income Tax IR8A (New Report). If the customer requires the Bank Code field to be blank in the file, it can be achieved via the following two solutions:

  1. Non-numeric Bank Key: If the Bank Key maintained in IT0009 is non-numeric (e.g., "PAYNOW"), the system will automatically generate a blank Bank Code in the IR8A file.
  1. Numeric Bank Key: If the Bank Key in IT0009 is numeric but you need the Bank Code to appear as a blank value in the file, please configure the return value as "BLANK" for this Bank Key in Feature 25BKC. The system will then generate a blank Bank Code in the file accordingly.

This change covers both the PATLine file and the API file.

In addition, this SAP Note extends the length of the field Max Employee Records Per File on the IR8A report selection screen to 6 digits.

Other Terms

Solution

Apply the HR Support Package to get this solution.

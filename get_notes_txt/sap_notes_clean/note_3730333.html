Symptom

Unexpected outputs are returned for BLOB columns on FOR JSON or XML when executing with ESX engine.

--Example Scenarios 1--

CREATE TABLE TESTTAB (A BINARY(10), B BLOB);
INSERT INTO TESTTAB VALUES (x'414243', x'414243');

SELECT * FROM TESTTAB FOR JSON WITH HINT (USE_ESX_PLAN);

-- For JSON, the output shows the unexpected result with ESX plan.

[
     {
          "A": "414243",
          "B": "ABC"
     }
]

SELECT * FROM TESTTAB FOR XML WITH HINT (USE_ESX_PLAN);

-- For XML, the output also shows the unexpected result with ESX plan.

<resultset>
     <row>
        <A>414243</A>
        <B>ABC</B>
    </row>
</resultset>

--Example Scenario 2--

CREATE TABLE TESTTAB (A BINARY(10), B BLOB);
INSERT INTO TESTTAB VALUES (x'ABC', x'ABC');

SELECT * FROM TESTTAB FOR JSON WITH HINT (USE_ESX_PLAN);

-- This returns "invalid CESU-8 encoding for Unicode string" error message.

SELECT * FROM TESTTAB FOR XML WITH HINT (USE_ESX_PLAN);

-- This returns "invalid CESU-8 encoding for Unicode string" error message.

Other Terms

JSON, XML, ESX, FOR JSON, FOR XML, BINTOHEX, BLOB

Reason and Prerequisites

Issue number 354769

Reason:

Due to a programming missing in SAP HANA, the evaluation of BINTOHEX for BLOB columns on FOR XML/JSON is not considered in ESX process.

Affected Releases:

  • SAP HANA 2:
    • Revisions of SPS06 - SPS08

Prerequisites:

  • The table has the BLOB column.

AND

  • The statement is with "FOR XML" or "FOR JSON".

AND

  • The execution is handled by ESX engine.

Solution

Apply one of the following SAP HANA Revisions:

  • SAP HANA 2:
    • SAP is currently working on a permanent solution to this issue. This SAP Note will be updated as soon as a revision containing the solution is available
    • or higher

  • or higher

Workaround:

  • Possible workaround would be to make the statement to be executed by HEX engine, by applying USE_HEX_PLAN hint.

--From Example Scenarios 1--

SELECT * FROM TESTTAB FOR JSON WITH HINT (USE_HEX_PLAN);

-- Output shows the expected output.
[
     {
          "A": "414243",
          "B": "414243"
     }
]

SELECT * FROM TESTTAB FOR XML WITH HINT (USE_HEX_PLAN);

-- Output shows the expected output.
<resultset>
    <row>
        <A>414243</A>
        <B>414243</B>
    </row>
</resultset>

--From Example Scenarios 2--

SELECT * FROM TESTTAB FOR JSON WITH HINT (USE_HEX_PLAN);

-- Output shows the expected output.
[
     {
          "A": "0ABC",
          "B": "0ABC"
     }
]

SELECT * FROM TESTTAB FOR XML WITH HINT (USE_HEX_PLAN);

-- Output shows the expected output.
<resultset>
    <row>
        <A>0ABC</A>
        <B>0ABC</B>
    </row>
</resultset>

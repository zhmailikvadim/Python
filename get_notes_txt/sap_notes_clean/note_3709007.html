Symptom

You encounter a problem during the merging of PDF files via class cl_rspo_pdf_merge. It is required to analyze the original documents that have been passed to the PDF merger, but these documents are created dynamically.

Reason and Prerequisites

Problem analysis

Solution

  1. Apply the solution from this note or the support package mentioned herein.
  2. Activate option SAVE_PDF_SPOOL using report RSTXPDF3 (see also note 1481367).
  3. Reproduce the problem.
  4. Deactivate option SAVE_PDF_SPOOL.
  5. You can find spool requests in transaction SP01 with title 'PDF merger: orig. doc. no.'.

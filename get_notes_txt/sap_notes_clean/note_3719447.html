Symptom

  • Cloud Identity Services can be bundled with SAP Business Data Cloud (BDC), SAP Datasphere (DSP) and SAP Analytics Cloud (SAC) through Identity Provider Administration Tool

  • Failed to Validate Login  when trying to configure authentication of bundled SAP Cloud Identity Services Tenant in Identity Provider Administration tool  

  • Error message: "Login unsuccessful. Check your authentication settings and try again."

Environment

  • SAP Business Data Cloud
  • SAP Datasphere
  • SAP Analytics Cloud, Enterprise Edition
  • SAP Cloud Identity Services
  • SAP Identity Authentication Service (IAS) 
  • SAP Identity Provisioning Service (IPS)
  • Identity Provider Administration Tool

Reproducing the Issue

  1. Access the Identity Provider Administration Tool.
  2. Select one tenant and provision an SAP Cloud Identity Services Tenant (SAC Guide | DSP Guide) to create an authentication bundle.
  3. Try to configure authentication (SAC Guide | DSP Guide) to work with the selected tenant. 
  4. In Step 2 Validate Login page, click Validate Login button.

Resolution

  • The selected tenant should appear in the Bundled Applications list on your SAP Cloud Identity Services tenant:
    • Application Name: in format "<Product Name> - <your tenant name>"
      • SAC: SAP Analytics Cloud - <your tenant name>
      • DSP: SAP Datasphere - <your tenant name>
      • BDC: SAP Business Data Cloud - <your tenant name>
    • Protocol Type: OpenID Connect
    • There’s a blue label that says “This application is an SAP application. Some settings are read only”. 
  • Please change the subject name identifier of this bundled application in IAS to match the attribute you selected:
    • USER ID: To map to the SAP Analytics Cloud User ID.
    • Email: To map to the SAP Analytics Cloud Email address
    • Custom Authentication User Mapping: To map to a custom value.
  • Follow the same methodology described in SAP KBA 2487567 to check if they are matching or not.
  • Note that if you have an existing application within same IAS for this tenant (protocol type is SAML 2.0), all changes in this existing application won't take affect for bundling process.

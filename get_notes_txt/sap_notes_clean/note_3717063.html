Symptom

  • Web Intelligence processing server crashes when table with measure formula returns empty values while table options 'Avoid duplicate row aggregation' is enabled and 'Show rows with empty measure values' is disabled

Other Terms

webi, webintelligence, WIPS, restart, coreinfo, shut down, unexpected, Termination, crashing, stop, calculation

Reason and Prerequisites

Environment:

  • SAP BusinessObjects Business Intelligence platform 4.3 Support Package SP04
  • Web Intelligence

Steps To Reproduce:

  1. Log in to BI Launchpad
  2. Create a WebIntelligence document based on "State" and "Sales revenue" from efashion universe.
  3. Create a table with "State" and formula =If([Sales revenue] =0) Then Sum([Sales revenue]) In ([State])
  4. Add a footer to the table with =Sum(If([Sales revenue] =0) Then Sum([Sales revenue]) In ([State]))
  5. Enable the option Avoid duplicate row aggregation and disable Show rows with empty measure values
    1. >> Web Intelligence processing server crashes

Cause:

  • This issue is caused by a defect in the product.

Solution

This issue is fixed in the patches listed in the "Support Packages & Patches" section below.
Due to different internal processes, the "Support Packages & Patches" section of the SAP Note will be populated as below:

  • for fixes delivered in "Support Packages": immediately (even if the "Support Package" is planned to be released several months later)
  • for fixes delivered in "Patches": ONLY once the "Patch" is released AND available on SMP (not before)

For Business Intelligence Platform maintenance schedule and strategy see the Knowledge Base Article 2144559 in References section.

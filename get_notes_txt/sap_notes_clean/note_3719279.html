Symptom

  • An error occurs when exporting ALV list to a spreadsheet
  • Error message: "The list viewer cannot be started on this front end computer".

* Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.    

Environment

SAP S/4HANA

Reproducing the Issue

  1. Execute t-code and get ALV list.
  2. Click export>spreadsheet.
  3. The message shows.

Cause

  • The message is raised if the default format is "Excel - in existing XXL Format".
  • This format is not supported in the WEBGUI environment.

Resolution

The message is only raised if the default format is 'Excel - in existing XXL Format'.

Only with this format data of the ALV table is exported via controller. These are not supported in WEBGUI environment (see SAP Note 2868609). Therefore the issue can only be solved by:

  • Deleting the default format 'Excel - in existing XXL format'. Subsequently message will no longer be raised.
  • Selecting another format e.g. 'Excel - Open Office XML Format (XLSX)' if requested.

To re-display the format dialog and delete the default format, it could select function 'Spreadsheet' from context menu as described in the documentation and SAP Note 1080608 (background information for ALV grids).

Contact user and ask them to delete the default format of function -> Export -> Spreadsheet.

Once this is done, re-test the scenario with the user and message should no longer appear.

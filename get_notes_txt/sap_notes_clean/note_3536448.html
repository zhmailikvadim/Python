Symptom

  • Inability to view PDF files in the Manage Supplier Invoices Centrally application within the SAP S/4HANA Cloud Public Edition.
  • The issue occurs intermittently.

Environment

SAP S/4HANA Cloud Public Edition

Reproducing the Issue

  1. Open the "Manage Supplier Invoices Centrally" application.
  2. Attempt to view a PDF file in an invoice.
  3. Error: "The PDF File could not be loaded"

Console: Error: Underlying iframe was not found in DOM

  

  1. Open PDF Viewer - Embedded - Samples - Demo Kit - SAPUI5 SDK 
  2. Error: "The PDF File could not be loaded"

Cause

The issue you are experiencing is related to a new experimental flag being introduced by Chrome, specifically the "OOPIF for PDF Viewer" flag. You can access this flag by navigating to the URL chrome://flags/#pdf-oopif in your Chrome browser. We have observed that this flag is currently functioning inconsistently, even across different instances of the Chrome browser with the same version.

Please check the status of this flag in your browser. If the selected value is either "Default" or "Enabled," kindly change it to "Disabled" from the dropdown menu. This would solve the problem which you are currently facing.

This change in the behavior has been enforced by Chrome and not in the control of SAPUI5 .

Resolution

  1. Navigate to the URL chrome://flags/#pdf-oopif in the Chrome browser.
  2. Check the status of the "OOPIF for PDF Viewer" flag.
  3. If the selected value is either "Default" or "Enabled", change it to "Disabled".

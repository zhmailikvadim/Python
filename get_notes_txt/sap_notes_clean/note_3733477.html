Symptom

  • ASE errorlog reports (many) error 1204 'out of locks':

    Error: 1204, Severity: 17, State: 2 occurred for User 'user_name'. Client IP address is '10.xx.xx.xx'.
    ASE has run out of LOCKS.
    SQL Text: setuser setuser 'user_name'

  • Causing a signal 11 at address 'setuser':

    Address 0x0x0000000000df26b7 (setuser+0x2da) ...
    Current process (0x42f0484) infected with signal 11 (SIGSEGV)

    with stack functions

    setuser
    s_execute

Environment

SAP Adaptive Server Enterprise (ASE) 16.0 & 16.1

Cause

This is an ASE coding issue where the out of locks situation causes an incorrect pointer in an ASE SDES memory structure which the setuser calls on. This issue is logged under new SAP CR 831897:
"Following a 1204 error, the message, "Current process infected with signal 11 (SIGSEGV)", in the module 'setuser' together with a stack trace may be reported in the SAP ASE error log"

Resolution

  • There is no scheduled release or timeframe for CR 831897. It is currently being reviewed by our Product Group

  • Workaround:
    Preventatively increase number of locks as avoiding running out of locks will prevent the subsequent signal 11.
    In general running out of locks will impact ASE processing so it is better to avoid that situation altogether.

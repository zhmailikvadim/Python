Symptom

**This is the correction of note-3699283 When open service order and confirmation in two sessions, add unplanned item in service confirmation but empty in order**


If you have implemented 3699283 for fixing the inconsistencies between the unplanned item in the service confirmation and in the service order, please implement this note. The note 3699283 may cause potential issue for FM CRMS4_UNPLANNED_ITEM_CREATE_EC is never executed, if CRMS4_UNPLANNED_ITEM_CHECK_EC raises an exception.

The inconsistencies between the unplanned item in the service confirmation and in the service order can be solved by note-3397876 Service Order not locked on unplanned item creation.

Other Terms

Unplanned Item, Service Order, Service Confirmation

Reason and Prerequisites

Program Error

Solution

Implement the correction instruction

Symptom

You have received a notification mail advising you of the switch of SAP Cloud Integration for data services root certificate from "DigiCert Global Root G2" to "DigiCert TLS RSA4096 Root G5", effective February 2026.

  • Please check the tentative / currently planned switch schedule dates for each SAP Cloud Integration for data services Kyma server later on this article.

The purpose of this KBA is to provide clarification on the actions required on the customer side in relation to this activity.

Environment

SAP Cloud Integration for data services - Kyma

Cause

As part of our ongoing commitment to security and trust, SAP is updating the root Certificate Authority (CA) used for load balancer certificates in both design-time and runtime environments.

This update supports the broader adoption of our fifth-generation (G5) root CA and allows additional time for its inclusion in major browser and operating system trust stores, ensuring seamless and trusted connectivity.

Resolution

Am I impacted?

This change potentially impact inbound communication to SAP Cloud Integration for data services tenants.

-------------

The SAP article 3619753 explain this process and timelines for SAP Cloud Integration for data services on the BTP Neo environment.

This SAP article is to inform that the same process, impacts and mandatory actions for customers are also applicable for SAP Cloud Integration for data services on the BTP Kyma environment, the only difference is the timelines.

If your team already performed the steps to be prepared for BTP Neo earlier, you don’t need to perform same steps again for BTP Kyma.

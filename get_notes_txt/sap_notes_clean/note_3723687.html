Symptom

When a contract belonging to a product type (company-code dependent) uses an external number range, and the same number range is used for sample contract, it becomes possible to create a contract using the same number as the sample contract. However, when saving such a contract, a short dump or error occurs.
To prevent this issue, this SAP Note extends the duplicate-loan check to also consider archived and sample loans during contract creation for contracts that use external number ranges.

Other Terms

FVD_DB_VDARL_GET_SNG_FLEX

Reason and Prerequisites

This was not previously provided.

Solution

Implement the attached correction instructions or import the specified Support Package

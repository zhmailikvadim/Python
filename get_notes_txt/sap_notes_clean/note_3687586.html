Symptom

From the Netweaver the message below appears:

  • failed to create symbolic link;

On ADS trace the following is presented:

  • #2.0#2020 12 11 15:01:02:361#+00#Error#System.err#
    #BC-SRV-FP#com.adobe~XMLFormService#C0000000#6945#com.adobe/AdobeDocumentServices#SystemThread[XMLForm.exe Error Reader,5,Dedicated_Application_Thread]#Plain#
    ln: failed to create symbolic link '/usr/sap/SYD/xxx/j2ee/os_libs/adssap/XMLFormService/bin/libaide.so': File exists#

Environment

SAP Business Technology Platform

Cause

  • ADS is not able to access the libs using symbolic link. In fact, ADS usually faces issues with NFS mounts:
    • 2526123 - IFbA: Several ADS issues with NFS mount
    • 2715787 - IFbA: Several ADS issues with NFS mount part II
  • The error message can also be triggered by a known issue described in SAP Note 3700067 if the ADS version is lower than the version listed in the note.

Resolution

  • Apply corresponding ADS SP/Patch mentioned in SAP Note 3700067 - IFbA: Error message - ln: failed to create symbolic link
  • Redeploy the ADS as per 2421452 - How to redeploy the ADS server.
  • If the issue persists, please, open an incident under BC-SRV-FP component.

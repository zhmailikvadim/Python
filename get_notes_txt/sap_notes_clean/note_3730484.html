Symptom

You would like to enable the End Date Field on the Recurring Deduction object. 

"Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental." 

Environment

SuccessFactors Employee Central

Resolution

1. Make the field "End Date" Editable in the "Recurring Deduction Items" Object Definition;

2. Go to "Manage Configuration UI" and open the standard Recurring Deduction UI "RecurringDeductionConfigUI". 

2.1 Hover the cursor on the recurring deduction items fields until the dotted blue line shows;
Note that the End Date field must be created under the recurring deduction items outlined by the dotted line, otherwise, an error will occur.

2.2 Click on the Add Field button. The new field will be added by the end of the row. Click on the pencil Icon besides it and set the "Value Type" to "Resolver", "Value Type Resolver" to "Recurring Items.End Date" and the Label Override. After that you can drag the new field to reorder if necessary.

2.3 If you would like to set the number of fields show in the row and visible in the card, you can click on the Properties and change the visible field count.

3. Save the changes. If you save as a custom UI, it needs to be assigned on the "Default Screen" field of the "Recurring Deduction" object definition.

Symptom

For an inbound inspection process in embedded EWM (IOT4), you complete putaway for the stock item before posting usage decision. When posting the decision, the selected follow-up action has no WPT maintained in EWM-QM customizing. The follow-up queue responsible for updating the stock and removing inspection reference fails in SMQ2 with status "Error during follow-up action; See application log". When double-clicking on the message, no log or error is displayed / visible.

Environment

  • SAP Extended Warehouse Management (SCM-EWM)
  • SAP Extended Warehouse Management Quality Management (SCM-EWM-QM)
  • SAP Extended Warehouse Management Quality Management Follow-up Processing (SCM-EWM-QM-FUP)
  • SAP S/4HANA

Reproducing the Issue

  1. Execute transaction "QA11";
  2. Input the inspection lot number;
  3. Set header decision with the respective follow-up code;
  4. Check "SMQ2", follow-up queue is stuck without any error log.

Cause

Missing Warehouse Process Type determination for the WT creation. As putaway has been completed, WPT cannot be taken from the delivery item anymore.

Resolution

Set a default WPT in SPRO path: "SCM Extended Warehouse Management -> Extended Warehouse Management -> Cross-Process Settings -> Quality Management -> Inspection Results -> Maintain Follow-Up Actions" for the respective follow-up action / Warehouse number combination.

This change will be applied for new cases only. For the already failed queues, please create a SAP Support case for SCM-EWM-QM-FUP component.

Symptom

An error occurs when moving a candidate in the new applicant workbench, resulting in a “403 Error” message. This behavior is observed for roles that have access to only one status. Although the candidate is moved correctly in the system, the user is automatically logged out and needs to log in again to continue working.

The issue is happening consistently across all instances, including test, development, and production environments.

The error details displayed are as follows:

internalMessage=Failed to Get Application Header Info
errorMessage=Service /rest/recruiting/application/v1/applicantSummary?jobRequisitionId=&candidateId=&licationId= gave 403
status=403

externalMessage=You do not have an Application Status Visibility

Environment

SAP SuccessFactors Recruiting Management

Reproducing the Issue

  1. Proxy as an affected user.
  2. Access a job requisition.
  3. Open a candidate application.
  4. Click on “Move Candidate.”
  5. Move the candidate to a different status.
  6. Observe that a “403 Error” message appears.

Cause

This is a Known Issue.



Resolution

The fix for this issue is planned to be deployed on 1H 2026 release.

For release timelines, please review SAP SuccessFactors Product Release & Road Map Information.

This behavior does not block the process. The candidate is successfully moved even though the error message appears. After encountering the message, simply refresh the page and it will no longer be displayed, allowing you to continue working normally.

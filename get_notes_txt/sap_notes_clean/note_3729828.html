Symptom

During Silent Data Migration (SDM), errors in one or more of the following classes are displayed during the SAP Readiness Check or the Simplification Item Check for an S/4HANA upgrade:

  • CL_NGC_CORE_SDM_CAWN
  • CL_NGC_CORE_SDM_CAWN_2
  • CL_NGC_CORE_SDM_CLHIER
  • CL_NGC_CORE_SDM_DATUB_CABN
  • CL_NGC_CORE_SDM_DATUB_CABNT
  • CL_NGC_CORE_SDM_DATUB_CAWN
  • CL_NGC_CORE_SDM_DATUB_CAWNT
  • CL_NGC_CORE_SDM_DATUB_KSML
  • CL_NGC_CORE_SDM_MSEHI_CABN
  • CL_NGC_CORE_SDM_CLF_HDR


Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.

Environment

Classification

Reproducing the Issue

  1. Check the log of report /SDF/RC_START_CHECK or within the BTC Migration Tool or similar
  2. Observe an error (i.e. return code = 8 or return code = 12) for CL_NGC_CORE_SDM* classes


Example for CL_NGC_CORE_SDM_CLF_HDR error in client 100:

Cause

Not all entries have been migrated yet.

Resolution

  • Review the error log in detail to identify the client where the errors occurred.

    • If the affected client is 000 (the standard SAP client), re-run the SDM to migrate the data from client 000. A step-by-step guide for client 000 is available in KBA 3606354.
    • For all other clients, check the relevant table associated with the failing class:



  • For example, if class CL_NGC_CORE_SDM_CLF_HDR reports errors, the relevant table is PSDMCH. Open it in transaction SE16 and check for records where SDM_STATUS is not equal to 9 (9 indicates 'migrated'), if such records exist, they are still pending migration.
    • If their internal numbers start with 000*, they are customer entries, in this case, run report RNGC_CONV_CHECK (of note 2975721) and click the 'Convert' button for each item in the result list to resolve the SDMI errors. 
    • SAP standard entries fall within 9999000000–9999999999 or start with SAP_*, for these, re-run the corresponding SDM (see more in note 2907976, section 30).

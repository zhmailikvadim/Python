Symptom

The error "Do not delete columns; templates already exist for this category" is raised even though no columns were deselected for the category in PHAP_CATALOG or PHAP_CATALOG_PA.

Reproducing the Issue

  1. Run transaction code PHAP_CATALOG or PHAP_CATALOG_PA.
  2. Double-click the category that needs to be configured.
  3. [Optional] The configuration should be carried out without removing any selected columns on the Column tab. 
  4. Press Save button after configuration is done, then the error is raised.

Cause

One or more selected columns lack a definition. Possible reasons include accidental deletion using OOHAP_BASIC or transport-related inconsistencies.

Resolution

Please make sure that for any selected column, its definition must exist in table T77HAP_COL.

Steps below can be followed to check and ensure this.

  1. The first thing is to find out the Category Group ID and Category ID of the problematic category.
    1. Run transaction code SE16.
    2. Input the table name of T77HAP_C_T and press Enter key.
    3. Copy and past the name of the problematical category into the CATEGORY_NAME field and execute.
    4. Then the Category Group ID and Category ID can be confirmed under CAT_GROUP_ID and CATEGORY_ID columns respectively.
  2. Check the selected columns on the DB level for the problematical category.
    1. Run transaction code SE16.
    2. Input the table name of T77HAP_C_COL and press Enter key.
    3. Input the Category Group ID and Category ID found in Step 1 into fields CAT_GROUP_ID and CATEGORY_ID respectively and execute to find out the selected column IDs on the database level.
  3. Check whether any of the selected columns is missing.
    1. Run transaction code OOHAP_BASIC.
    2. Double-click the Column folder on the left side panel.
    3. Check whether all the selected columns found in Step 2 have corresponding definition on the right side.
    4. For any missing column, please redefine them here by using 'New Entries' button.
    5. After the redefinition, please make sure Save button is pressed to save the change.

Additional Information:
If a template exists for a category, items cannot be unselected at the category level. This applies to all tabs and prevents data inconsistencies.

Symptom

  • The fields in the Address Information card of the Latest People Profile are not displayed in the expected order based on the Data Model / Business Configuration settings.
  • The Address Type and Country fields are always displayed as the first two items, regardless of the specified field order configuration.
  • After these two fields, the remaining fields follow the expected order configuration.

Environment

  • SAP SuccessFactors HCM Suite
  • SAP SuccessFactors Employee Central

Reproducing the Issue

Prerequisite: Latest People Profile is enabled.

  1. Navigate to an Employee Profile.
  2. Select the Personal Data category and check the Address Information card.
  3. Observe that the Address Type and Country fields are displayed as the first two items, regardless of the specified field order.

Cause

Currently, this is the by-design behavior of the Address Information card on the Latest People Profile.

Resolution

The Address Type and Country fields are hard-coded to always display as the first two items when viewing the Address Information card on the Latest People Profile. Only the fields afterward will follow the defined order.

If this behavior is not aligned with your business needs, please consider submitting an enhancement idea as per KBA 2090228 - How to Submit Enhancement Ideas for SAP SuccessFactors Products.

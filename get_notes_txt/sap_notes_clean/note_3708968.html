Symptom

  • Inbound Delivery Notification cannot be activated again, but there is no Inbound Delivery (PDI) document for it.
  • There is no /SCDL/IDR_TRANSFER (Business Object Transition Service for Inb. Deliv. Notif.) PPF action on PPF tab.
  • PPF action has a custom Schedule Condition, or standard code is modified with an implicit enhancement, because business requirement is to activate the IDNs manually.
  • There is a Change Documents entry about DDC (Create Delivery Documents) status change from Completed to Not Started at the same time when IDN was set to DRC completed, which indicates that the related Inbound Delivery (PDI) was manually deleted via /SCWM/PRDI.
  • IDN item status tab shows below values:
    • DCO Completion - 1 Not Started
    • DDC Delivery Creation - 1 Not Started
    • DRC Completion Request - 9 Completed
    • DVA Activate - 9 Completed
    • DVE Version - A Active
  • The same is valid for:
    • Outbound Delivery Requests (ODR) and Outbound Delivery Orders (PDOs).
    • Posting Change Requests (POR) and Posting Changes (SPC).

Environment

  • SAP Extended Warehouse Management (EWM)
  • SAP S/4HANA

Reproducing the Issue

  1. Create a delivery for EWM.
  2. Manually activate the Request Document.
  3. Delete the Processing Document in the transaction for Maintaining the document (e.g. /SCWM/PRDI).

Cause

In case of using Request Documents, the system automatically creates the Processing Document (PDI - Inbound Delivery, PDO - Outbound Delivery Order, SPC - Posting Change) if using default settings for PPF action /SCDL/IDR_TRANSFER. If the PPF action /SCDL/IDR_TRANSFER is inactive, or the system is not triggering the action based on the Schedule Condition, then deletion of the Processing Document does not create a new delivery.

Resolution

  • If PPF action /SCDL/IDR_TRANSFER is inactive, activate it in SPPFCADM.
  • If PPF action /SCDL/IDR_TRANSFER is active, but wrongly configured (e.g. Action profile is not configured for the Request Document Type), check configuration.
  • If the Schedule Condition of the PPF action /SCDL/IDR_TRANSFER does not trigger the PPF for the first time, and the Request Document is activated manually, following options are available:
    • Turn off the possibility of the deletion of the Processing Document (refer to KBA 3479589 - Disable or restrict button in EWM Delivery transaction codes due to accidental use),
    • Ensure that the Schedule Condition will succeed after the first activation (e.g. via version field content read in table /SCDL/DB_KEYMAP ("-1" = inactive, "1" = active)) to trigger the Transition Service when deleting the Processing Document.

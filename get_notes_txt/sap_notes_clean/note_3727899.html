Symptom

When upload incoming bank statement files with foreign currency to system:

  • The posting area 1 is posted in account currency by default.
  • The posting area 2 is posted in in foreign currency.
  • But business-specific requirement is to execute the posting in Posting area 2 in account currency instead of foreign currency. 

This article provides a guide on how to achieve this.

"Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental." 

Environment

SAP S/4 HANA Cloud Public Edition.

Reproducing the Issue

Receive bank statement files automatically via SAP Multi-Bank Connectivity, or upload manually via Manage Incoming Payment Files app, observe that the subledger document posted for posting area 2 in foreign currency.

Cause

In posting area 2 (Subledger Accounting), the foreign currency fields are used for posting if they are filled. If this information is not available, postings are made in the account currency in the same way as in posting area 1.

Resolution

To use account currency for posting area 2, create an own mapping by copying the standard delivered format using app Map Format Data for Files from Banks (F5608). Here in copied version, navigate as follow:

Format Mapping Properties - <Mapping> - <Document> - <BkToCstmrStmt> - <Stmt> - <Ntry> - <NtryDtls> - <TxDtls> - <AmtDtls> - <AmountAndCurrencyExchange3> - <InstdAmt> - <CcyXchg> - <CurrencyExchange5> - <XchgRate>.

In <XchgRate> node, scroll to tab Output Specification section, the Output Field is currently defined with value PAYMENT_FILE-ITEMS-EXCHANGERATE (Exchange Rate (Different from Local Currency)). Enter Edit mode and remove content of this Output field. Then save the mapping and activate it for testing.

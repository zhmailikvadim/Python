Symptom

When adding a company code in MDG NWBC UI for an ERP Vendor, you realize that either UIBB of "ERP Vendor Company Code: Withholding Tax" or "ERP Vendor Company Code: Extended Withholding Tax" will be shown as read-only based on different company code assigned.

Environment

  • Master Data Governance on S/4 HANA on-premise 1511
  • Master Data Governance on S/4 HANA on-premise 1610
  • Master Data Governance on S/4 HANA on-premise 1709
  • Master Data Governance on S/4 HANA on-premise 1809
  • Master Data Governance on S/4 HANA on-premise 1909
  • Master Data Governance on S/4 HANA on-premise 2020
  • Master Data Governance on S/4 HANA on-premise 2021
  • Master Data Governance on S/4 HANA on-premise 2022
  • Master Data Governance on S/4 HANA on-premise 2023

Reproducing the Issue

  1. Create an Supplier Change request via Supplier Governance.
  2. Add corresponding vendor role and create ERP vendor
  3. Add corresponding company code and check the property of the mentioned UIBB.

Cause

This is the standard behavior as there is a check in method CL_BS_SP_GUIBB_COMPANY_CODE->IF_FPM_GUIBB_FORM~GET_DATA starting from line 30.

  • Withholding tax data is either maintained as part of the company code data; or, if the extended withholding tax functionality is
    active for the current company code, in an additional table.
  • If the extended functionality is active, the withholding tax fields of the current company code must be read-only.(except: withholding tax country is maintainable for both!).

Resolution

This is standard behavior in MDG and you can check the entry of field WT_NEWWT in table T001 for the company code. If the field is checked, then the company code is enabled, the extended withholding tax will be used. When you check in T-code BP, Tab Vendor: Withholding tax in company code will only show the extended withholding tax(from table LFBW).

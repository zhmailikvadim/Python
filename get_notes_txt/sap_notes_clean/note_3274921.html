Symptom

When processing a scheduling agreement subcontracting item with BAPI_SAG_CHANGE or BAPI_SCHEDULE_MAINTAIN, the following error occurs: Material [] with phantom assembly is not supported for subcontracting. (MEOUT 224).

Environment

  • SAP S/4HANA 2020
  • SAP S/4HANA 2021

Cause

BAPI is no longer supporting Phantom materials as of S4 2020. This coding was changed to align the behavior and limitation with other integration scenarios that uses the BAPI logic as well. The check logic is inserted into include LMEOUTP04 method IS_BOM_VALID.

Resolution

The standard BAPIs can no longer process Scheduling Agreements with Phantom material.

The only alternative is to use frontend transactions like ME31L, ME32L or ME38.

Keywords

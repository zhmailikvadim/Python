Symptom

Datasphere CLI tokens continue to expire causing job errors.

Environment

SAP Datasphere.

Resolution

  • Use Refresh Token Lifetime to extend the Token lifetime. See - Create OAuth2.0 Clients to Authenticate Against SAP Datasphere.

  • To avoid a manual authentication use OAuth with --secrets-file and provide the manually retrieved (command: datasphere secrets show) "access_token" and "refresh_token". See details in - @sap/datasphere-cli: Getting Rid of Passcodes Thanks to OAuth Client Support.
  • There is also a Feature Requests to this regard currently under review: Possibility to extend the token lifetime of OAuth clients. You could also check that and vote it if you need this feature.

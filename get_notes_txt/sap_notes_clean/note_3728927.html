Symptom

You call transaction /SAPAPO/DELETE_STOCK. After you entered some criteria at the selection screen you execute the transaction. The transaction exits without providing any information and you are back at the initial screen which appears also when you start a new GUI Window.

Other Terms

/SAPAPO/DELETE_STOCK

Reason and Prerequisites

  • The S4COREOP release of your S/4HANA system is 108 or 109. With S4COREOP release 108 new transaction /SAPAPO/DELETE_STOCK was delivered for report /SAPAPO/DELETE_STOCK.
  • Transaction type for transaction /SAPAPO/DELETE_STOCK is set to dialog transaction at SE93 transaction. Transaction is created for /SAPAPO/DELETE_STOCK report with a selection screen, but with the incorrect type dialog transaction instead of a report transaction.

Solution

With attached correction instruction the transaction type for transaction /SAPAPO/DELETE_STOCK is changed from dialog transaction to report transaction. After this change has been implemented at your system execution of transaction /SAPAPO/DELETE_STOCK will not exit any longer.

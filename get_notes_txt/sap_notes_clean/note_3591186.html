Symptom

The conversion readiness check for SAP GTS, edition for SAP HANA reports the following error for check ID number 06:

A product is classified several times for the same numbering scheme at the same validity start date

The data model in SAP GTS, edition for SAP HANA uses the date (SAP data type DATS) instead of a time stamp as in earlier versions of SAP GTS to map the validity start date and validity end date of a product classification.

The invalid change to the system time zone during live operation, or the direct writing of product classifications using customer-specific programs have created time stamps that lead to errors during the technical upgrade to SAP GTS, edition for SAP HANA because different time stamps can lead to an identical validity start date during a conversion to a date.

Other Terms

SAP GTS readiness check, /SAPSLL/READINESS_CHK_V8000, check ID #06, /SAPSLL/PRCTS, /SAPSLL/PRCTSC

Solution

Create a local report or a report in your namespace, and copy the program source code that is attached to this SAP Note. Insert the program code below the report statement into your report.

The report handles classifications for a product of the same numbering scheme with the same validity start date as follows:

  • The classification with an earlier valid-to date is deleted; the classification with the longer valid-to date is retained. This is independent of whether the numbers of the two classifications differ or when the classification was created.
  • For classifications whose valid-to dates are identical but have different numbers, the classification that was last created is retained. The classification created earlier is deleted.
    If the classifications were created at the same time, the report does not make any changes, and you must decide yourself which classification can be deleted. You make this decision manually by editing the classification in the product master.
  • For classifications whose valid-to dates are identical and also use the same number, the duplicates are deleted.

Carefully check whether the rules mentioned above can be used as a solution for your company. Execute the report only then, and ideally first in simulation mode, which you can activate on the selection screen of the report. Executing the report in an update run results in the immediate deletion of affected classifications.

You can also use the information that the report outputs on the selection list to manually correct the product classifications by editing the product master.

Symptom

  • In the development sub-organization, the System Administration tab does not show or enable the off-neo migration option.
  • There is uncertainty whether migration is in scope for the development sub-organization, with an expectation that development should migrate first.

Environment

SAP Cloud Integration for data services

Reproducing the Issue

  1. Open a sub-organization (for example, a development tenant) in the cloud integration for data services administration console.
  2. Navigate to Administration > System Administration.
  3. Observe that the off-neo migration option is not available or cannot be enabled.

Cause

  • The affected tenant is a sub-organization (sub-tenant) that is part of a main tenant.

Resolution

  • By design, off-neo migration can only be initiated for the main tenant. Sub-organizations are migrated automatically as part of the main tenant migration, so a separate migration option is not exposed for sub-tenants.
  • Testing with the sandbox environment applies to the main organization only, not to sub-organizations.

Symptom

  • During meter read upload or when entering meter reading results, the standard meter reading validation “98 – Previous Reading for Install/Repl or Sim. Without Entered MR” is executed.
  • The validation appears even though it is not explicitly configured in customizing.
  • The situation occurs when a meter reading order for installation/removal/replacement is still open.

Environment

  • SAP for Utilities as Part of SAP ERP, release IS-UT 600 and higher.
  • SAP S/4HANA, release IS-UT 801 and higher.
  • SAP S/4HANA CLOUD

Reproducing the Issue

  1. Create a periodic meter reading order (MRO) while an MRO for installation/replacement is still open.
  2. Enter the meter reading results and save.

Cause

The behavior is standard: validation check ISU_VALID_SP98_PREV_EABL prevents entering meter reading results when an installation/removal/replacement meter reading order is open; it is delivered by standard notes and is not deactivatable.

Resolution

  1. Do not attempt to deactivate validation 98 (ISU_VALID_SP98_PREV_EABL); this check is standard and cannot be deactivated. The corresponding customizing entry exists only for completeness.
  2. For future processing, perform meter readings for billing-related installations promptly so that subsequent meter reading results can pass plausibility checks and are not blocked by open installation/removal/replacement orders.
  3. In case you have interfering (very old) installation MRO: enter the old installation meter reading order with the next available result in transaction EL28 and set 'Meter reading active' to 0 which makes this read INACTIVE. This prevents the open order from blocking subsequent meter reading results.) >> setting the meter read to 'inactive' means that the meter reading result is not taken into account during billing and extrapolation.

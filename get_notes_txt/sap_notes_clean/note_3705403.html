Symptom

Price condition amounts in billing documents fluctuate because the system recalculates values based on varying exchange rates instead of a fixed USD basis in "Change Billing Documents"(VF02) app.

Environment

SAP S/4HANA Cloud Public Edition

Cause

Condition type is not configured to calculate using a fixed condition currency (USD) in configuration activity "Set Condition Types for Pricing in Sales"(101120).

Resolution

Please follow the steps below to complete the configuration to ensure the pricing condition is calculated using a fixed USD currency in 'Manage Your Solution' app:

  1. Open 'Manage Your Solution' app.
  2. Navigate to 'Configure Your Solution'.
  3. Search for the ID '500092'.
  4. Choose configuration activity '101120 - Set Condition Types for Pricing in Sales'.
  5. Select the required condition type.
  6. Enable the Currency Conversion flag, this makes the system calculate the condition in USD first, then convert to document currency after calculation, keeping USD as the pricing base and preventing exchange-rate calculation differences.

In case while using SAP Central Business Configuration (CBC) :

  1. Log on to the project experience in SAP Central Business Configuration.
  2. Select the relevant workspace.
  3. Navigate to the configurations activity tab.
  4. Search for the configuration activity '101120 - Set Condition Types for Pricing in Sales'.
  5. Select the required condition type.
  6. Enable the Currency Conversion flag, this makes the system calculate the condition in USD first, then convert to document currency after calculation, keeping USD as the pricing base and preventing exchange-rate calculation differences.

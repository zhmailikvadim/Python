Symptom

  • Purchase order created e.g. for company code 0001 and trying to post invoice cross company code 1000.
  • No proper customizing maintained for number ranges.

Environment

  • Materials Management (MM)
  • SAP ERP Central Component
  • SAP ERP
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA

Reproducing the Issue

1. PO created with company code 0001

2. Post invoice via MIRO, select company code 1000

3. Select the posting date e.g. 01.01.2026 in current fiscal year i.e. 2026 and, after filling all the required data simulate it, no error message occurs.

4. Then, change only posting date e.g. 01.12.2025 to previous fiscal year i.e. 2025, keeping the rest of the data the same and, simulate it.

5. Not able to post getting error M8385.

 

Remark:

  • Here, internal or external number range e.g. YY is assigned to company code 0001.
  • And, internal or external number range e.g. XX is assigned to company code 1000.

Cause

  • If external number assignment is defined for company code 0001 and interval XX, external number assignment must also be configured for company code 1000.
  • If internal number assignment is defined for company code 0001 and interval XX, internal number assignment must also be configured for company code 1000.

Resolution

  • Maintain the same number range assignment (internal or external), with the same interval, for the involved company codes.
  • Use the following path to perform the change:
    • Access transaction OLMR > Logistics Invoice Verification > Incoming invoice > Number Assignment > Maintain Number Assignments for Accounting Documents > Number range for document types in FI

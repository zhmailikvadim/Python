Symptom

Error message: "Resource &1 works on WO &2" (Message No. /SCWM/RF_EN140)  is displayed when attempting to unassign a resource from warehouse order in transaction /SCWM/MON.

Environment

SAP S/4HANA Cloud Public Edition

Reproducing the Issue

  1. Create a replenishment 2-step product warehouse task.
  2. Log on to the RF device with the first resource.
  3. Confirm the source step of the warehouse task.
  4. Log out or attempt to continue the task with a second resource.
  5. Observe that error /SCWM/RF_EN140 occurs because the warehouse order remains assigned to the first resource.

Cause

  • This issue occurs due to the standard system design, where resources cannot be unassigned from warehouse orders that are in "In process" status (D) with the follow up warehouse tasks from source Resource to Destination bin.
  • When the resource logged onto RF environment and working on the particular Warehouse Order.

Resolution

  1. WO's containing Warehouse tasks:

    1. It is not possible to unassign the resource from the warehouse order which is in status D (In process).
    2. If the corresponding warehouse task of the warehouse order is a follow up warehouse task and it has any value in the source resource field. Error message "Resource &1 works on WO &2" (Message No. /SCWM/RF_EN140) is displayed in this scenario.
    3. It means that stock is lying in the resource and it needs to be dropped in the destination bin. 
    4. To unassign a resource, either confirm / cancel the relevant warehouse task. 
    5. After confirmation / cancellation of the warehouse task, it is possible to unassign the resource from the warehouse order via " More methods" and click "Unassign from resource". 

    WO's of PI documents:

    1. Ensure that the resource is logged off from /SCWM/RFUI transaction. 
    2. This can be verified in transaction /SCWM/MON.
    3. Resource Management  --> Resource.
    4. Input the resource and execute. 
    5. Check whether the RF Exec. Environment column is Blank.
    6. If RF Exec. Environment column is marked with X, then select the entry & click on more methods --> Log off from resource. 
    7. After this step, Warehouse Order can be assigned to new resource for further processing in /SCWM/RFUI.

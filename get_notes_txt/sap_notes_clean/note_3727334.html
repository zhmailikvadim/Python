Symptom

You run payroll for an employee who turns 65 in the middle of the payroll period. The payroll period includes retroactive accounting with retroactive payments for MPF relevant income.

During the original payroll run, the system calculates the relevant income correctly. When you run payroll for the subsequent payroll period, the system triggers a retroactive run for the period when the employee turns 65. In this retroactive run, the system calculates the relevant income incorrectly.

After implementing this SAP Note, the system calculates the relevant income correctly during both the original and retroactive payroll runs. The relevant income remains consistent across both runs when an employee turns 65 in the middle of the payroll period.

Other Terms

MPF, Mandatory Provident Fund

Reason and Prerequisites

You use the alternative calculation method for MPF relevant income when an employee turns 65. This method is provided in SAP Note 3392492. You use wage type /124 as relevant income.

Solution

Implement the attached Correction Instruction (CI) or the relevant Support Package.

Symptom

  • Field value extracted from the source system to SAP Process Intelligence is truncated.
  • E.g. the variable type of the field is varchar(32) in the source system, but the extracted field value in SAP Process Intelligence is truncated to fewer characters varchar(16)

Reproducing the Issue

  1. Access SAP Process Intelligence.
  2. Navigate to Data Modeling > Process Pipeline > Data View/Transformation preview.
  3. Extract data from the source system with a field type defined as varchar(32).
  4. Observe that the field value is truncated upon extraction.

Resolution

  1. Ensure that you have the necessary SAP Notes installed on you source system
    https://help.sap.com/docs/signavio-process-intelligence/user-guide/required-and-use-case-specific-sap-notes?locale=en-US
  2. Ensure that you have met the necessary requirements for the source system version
    https://help.sap.com/docs/signavio-process-intelligence/user-guide/supported-systems-and-rfc-usage?locale=en-US
  3. Test the extraction process after implementing the SAP Notes and verify if the field value is correctly extracted.
  4. If the issue persists, provide screenshots of the Process Intelligence configurations, including connection type, table, and SQL command, for further analysis.

Symptom

There is a need to set up Bearer Authentication for Identity Directory API.

Environment

SAP Cloud Identity Services

Resolution

  1. Create an IAS System Administrator as per Add System as Administrator. When step #6 is reached, follow the "Secrets" section to generate a Client ID and Client Secret. 

  2. In IAS Administration Console, click into "Applications & Resources" > "Applications" > "Administration Console" > "OpenID Connect Configuration". Search for "Access / ID Token (min)" and extend the bar to the maximum 12 hr. Click Save.

  3. In a REST Client tool (e.g. Postman, Insomnia, Bruno, etc.), set the following up:

    Headers   
    - Key = Content-Type
    - Value = application/x-www-form-urlencoded

    Authentication
    - Basic Auth
    - Username = Client ID from above step 1
    - Password = Client Secret from above step 1

  4. Run POST request:    https://<IAS_tenant>.accounts.ondemand.com/oauth2/token?grant_type=client_credentials

  5. In the response, the bearer token is seen below:

    {
      "access_token": "<long token value>",
      "token_type": "Bearer",
      "expires_in": 43200
    }

Note: The OAuth Bearer Token expires in 12 hours (43200 seconds). If there is a need to run more Identity Directory API requests, then a new OAuth Bearer Token will need to be generate. This means performing the above steps #3 to #6 again.

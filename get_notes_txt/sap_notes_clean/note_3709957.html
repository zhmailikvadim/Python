Symptom

  • A job requisition with 3 openings allowed 4 candidates to be moved to the "Ready to Hire" stage.
  • The system does not restrict moving candidates to "Ready to Hire" even when the number of openings is exceeded.
  • Once the number of openings reaches 0, other candidates are moved to the "Requisition Closed" status, and they cannot be removed from that status.
  • How does the system processes the countdown and whether there is a cutoff for the number of candidates that can be moved to "Ready to Hire."

Environment

SAP SuccessFactors Recruiting Management

Reproducing the Issue

  1. Create a job requisition with a specific number of openings (e.g., 3).
  2. Move more candidates to the "Ready to Hire" stage than the specified number of openings.
  3. Observe that the system allows this action without restriction.

Cause

  • The system design allows moving candidates to "Ready to Hire" status even when the number of openings is exceeded.

Resolution

  1. Configure the Auto-Close feature in the job requisition settings to limit the number of hires. This ensures the requisition automatically closes once the specified number of openings is filled.
  2. When the last opening is filled, a popup appears asking whether to close the requisition. Selecting "Cancel" keeps the requisition open, allowing more candidates to be moved to "Ready to Hire."

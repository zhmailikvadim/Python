Symptom

With release 7.50 a productive usage of a PI "dual stack" is no longer supported. However "Dual Usage type" is supported, ABAP and Java are using different system IDs (SIDs), the Java system is still connected via User Management Engine (UME) to the ABAP system. This means that the Java system cannot start without ABAP. In some case, there might be a business requirement to copy Dual Usage PI 7.5 system. After system copy of both ABAP and JAVA stacks, the new ABAP stack can start, but the new JAVA stack fails to start due to incorrect UME connection.

Environment

  • SAP NetWeaver 7.5
  • SAP Process Integration 7.5 Dual Usage type

Cause

The system copy contains all the settings from the source system. The connection details for the UMEBackendConnection destination are incorrect. For the new JAVA stack after system copy, new UME-ABAP should be used for user management.

Resolution

To resolve this issue, adjust the UME configuration offline to connect to new ABAP system as per SAP Note 1827807 - How to edit the SAPJSF password and UMEBackendConnection details offline for AS Java 7.1+.

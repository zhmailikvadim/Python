Symptom

This Note is relevant for IDXAT2 customers only!

Currently, in MD request process trigger method, when call method to get service provider, MD request date is not used as importing parameter, which may result in getting incorrect service provider.

This Note will solve this problem, when call method '/IDXAT2/CL_UTILITY_SERVICE_ISU=>GET_SERVICE_PROVIDERS_FROM_POD', using MD request date as key date.

Other Terms

IDXAT2, MD, Request, Trigger Method, /IDXAT2/CL_MD_TRIGGER, START_MD_REQ_PROCESS, /IDXAT2/CL_UTILITY_SERVICE_ISU, GET_SERVICE_PROVIDERS_FROM_POD, IV_REQ_DATE, KEY_DATE, SY-DATUM

Reason and Prerequisites

Program error.

Solution

Install the correction of this Note.

Symptom

There is a requirement to transport one or more of the following custom objects between different systems or clients in SAP S/4HANA Private Cloud:

  • Custom objects (e.g. custom CDS views, custom analytical queries)

  • Custom fields

  • Custom Business Configuration objects

However, it is unclear which procedure or tools must be used for the transport.

In some cases, the created transport request is incomplete, certain objects are missing in the target system, or errors occur during import or activation.

Environment

SAP S/4HANA Private Cloud Edition
SAP S/4HANA

Cause

In SAP S/4HANA Private Cloud, extensibility/custom objects must be assigned to a software package and registered for transport before being included in a transport request.

If the apps Configure Software Package (App ID F1590) and Register Extensions for Transport (App ID F1589) are not used, the system does not correctly link the custom objects to the transport layer.

As a result, transporting objects directly without proper package configuration and registration may lead to incomplete or inefficient transports.

Resolution

To correctly transport custom objects in SAP S/4HANA Private Cloud, the following procedure must be followed:

  1. Use the Fiori app Configure Software Package to:
    • Create or assign a dedicated software package for the custom objects.
    • Ensure the correct transport layer is defined.

  2. Use the Fiori app Register Extensions for Transport to:
    • Select and register the relevant extensibility objects (e.g., custom fields, custom business configurations, custom CDS views, custom analytical queries)
    • Assign them to a transport request.

  3. After registration, the transport is managed and released using Transport Organizer (Transaction SE09).

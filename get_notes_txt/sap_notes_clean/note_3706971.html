Symptom

When you work in transaction /n/irm/ipttrecon and select the show all items option followed by change mode, the system locks all related calculation run records. When you exit the screen and return to the selection screen, these locks are not released during exit processing, which leads to records remaining locked unnecessarily.
In addition, when another session is opened and all items are selected and switched to change mode, the system does not immediately display a message indicating that the calculation runs are locked by another user. Instead, the lock message is shown only when an individual calculation run is accessed, and only after a record is explicitly selected. This inconsistent lock handling and messaging causes confusion and prevents smooth parallel processing.
This note corrects the behavior of the above issues.

Other Terms

ipttrecon, calculation run locks, lock not released, change mode, show all items, locked by other user, lock message issue

Reason and Prerequisites

This is cause dby program error

For V4, SAP Note #3706911 is available

Symptom

  • There is a integration from SAP ABAP system to External Systems through Web Service(HTTP/HTTPS). 
  • External System is a HA/DR environment.
  • External System IP address has been changed from A to B, nslooup has returned correct IP(B), however, ICM is still connecting OLD IP(A) which can be observed from ICM level 2 trace.
  • After a few mins(e.g. 10mins), ICM will be connecting the expected IP (B).

Environment

SAP NetWeaver Application Server

Reproducing the Issue

Perform HA/DR test on target system side

Cause

SAP ICM caches NS lookup results, both positive and negative ones, so that subsequent lookup with DNS is avoided because the cached results can be used for performance improvement.
The lifetime of the positive and negative results is 600s by default, and it's configured via the profile parameters "is/hostbuffer_timeout_valid_entry".

Resolution

If DNS failover or DNS load balancing is being used, you may have to regularly invalidate or deactivate this NI cache.

Workaround:
Manually delete the entry from NI host name buffer (t-code:smicm -> Menu:Goto -> Host Name Buffer -> Display)

Solution:
Deactivate the NI cache by setting is/hostbuffer_timeout_valid_entry = 0 parameter

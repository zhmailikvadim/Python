Symptom

  • Unprocessed Logging table records building up for a table in MSSQL
  • Long running program CL_IUUC_CONDENSE_LOGTAB_RECRDSCP 
  • Many changes in the logging table for one record
  • Slow DB commits on logging table in MSSQL
  • Slow DB deletes on logging table in MSSQL

Environment

SAP Landscape Transformation Replication Server

Reproducing the Issue

  1. Replicate a table from MSSQL, DB2 or DB6 using SLT
  2. Make many changes to a single record in the source table in a short period of time
  3. SM66 shows a slow DB commit/read on the logging table with the program CL_IUUC_CONDENSE_LOGTAB_RECRDS

Cause

When there are many changes made to a specific record in the source table this results in many records being added to the logging table, SLT condenses these records meaning it replicates the most recent entry in the logging table for the record and deletes the older entries from the logging table one by one, this is to avoid deadlocks. Because the records are deleted one by one this can lead to long running deletes on the logging table. 

Resolution

If a bulk approach to the condense deletes is preferred follow the solution section of note 3169125 - SLT (DMIS 2011 / DMIS 2018) Source & Central System: Enhance logic for logging tables for MSSQL / DB2 / DB6

Reduce the number of changes made to the source table

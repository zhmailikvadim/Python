Symptom

Additional single role(s) are also added to a user indirectly during single role assignment in SU01.

Environment

  • SAP NetWeaver
  • SAP NetWeaver Application Server for SAP S/4HANA
  • ABAP PLATFORM - Application Server ABAP
  • SAP S/4HANA

Reproducing the Issue

  1. Start transaction code SU01
  2. Go to 'Edit' mode
  3. Assign a single role to a user ID.
  4. Click 'Save'.

Cause

Incorrect entries in the AGR_AGRS table. Note that the origin of the wrong AGR_AGRS entries is unknown, and they are not caused by an SAP standard role maintenance function.

Resolution

  1. In order to remove the invalid single role assignments in AGR_AGRS table, first install the attached report Z_CLEANUP_AGR_AGRS in the role development system and maintain the following administrative attributes before activating:

    Title: Deletion of invalid single role assignments to roles
    Type: Executable program
    Status: Customer Production Program
    Application: Basis
  2. The report execution requires no role selection since all customer roles are checked. Deselecting the 'Test mode' is necessary for deleting invalid role assignments in the database. Multiple executions of the report bear no risk.
  3. After the cleanup; export the roles shown in the result list of the report to the test and productive environments as usual.

Important remark: The report performs no authorization checks and doesn't write change documents since it deletes only invalid data that might not exist.

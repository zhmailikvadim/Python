Symptom

  • 'View' Events are not captured for Design Studio Dashboards.
  • 'View' events (1002) can be seen generated and inserted in the Audit Database
  • All other Supported event types can be audited.
  • 'Retrieve' Event can be noted in DB for each View action performed on Dashboard.

Environment

  • SAP BusinessObjects Design Studio
  • SAP BusinessObjects Business Intelligence Platform 4.x

Reproducing the Issue

  1. Confirm that 'view' event is Checked in Auditing Page.
  2. View a Design Studio Dashboard.
  3. Query the Audit Database to return all 'view' events.

Cause

  • The following dfo associated with Design Studio are present in the 'Install Directory\SAP BusinessObjects\setup\packagestemp' and have not been pushed into the CMS Database:
    1. BusinessObjects_AuditEventInfo2_DesignStudio_dfo_frag.is.xml
    2. BusinessObjects_AuditEventInfo2_DesignStudio_dfo_frag.xml

Resolution

  1. Take a backup of the folder <install directory>/SAP BusinessObjects Enterprise XI 4.0\packages
  2. Stop the Server Intelligence Agent (SIA) on all nodes.
  3. Copy the left over xml files from the <Install Dir>/setup/packagestemp to <install directory>/SAP BusinessObjects Enterprise XI 4.0\packages
  4. Start the SIA.

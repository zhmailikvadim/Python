Symptom

I have suddenly started receiving a high volume of email notifications requesting that I approve timesheets and expenses for people with no connection to me whatsoever. These items do not appear in my personal 'To Do' list within the application, but they are flooding my work email inbox and making it difficult to see relevant communications.

Cause

This behavior is typically caused by an Active Proxy Designation that may be outdated or not visible in the primary user interface.

When a user is assigned as a proxy for another user (the "Principal"), the system automatically sends them copies of all actionable notifications intended for that Principal. This issue often occurs if:

  • Outdated Proxies: A temporary proxy relationship (such as for a vacation) was created but did not correctly deactivate upon reaching its end date.
  • Synchronization Lag: The UI may indicate that the proxy has been removed, but the backend messaging agent still recognizes the active relationship.

Resolution

Administrators must verify the proxy settings for the impacted user and force a refresh of the messaging associations.

  1. Identify the Principal: Review the body of the incorrect emails. They often contain a footer or header stating: "You are receiving this as a proxy for [Principal Name]".
  2. Verify Proxy Settings:
    • Navigate to Admin > Users and search for the Principal user identified in Step 1.
    • Open the user record and select Proxies from the left-hand menu.
  3. Refresh the Association:
    • If the impacted user is listed as a proxy, remove the entry or ensure the End Date is set to a past date.
    • If the user is not visible in the list but still receives emails, add them back as a proxy with an end date of "Yesterday" and save the record. Then, remove them again immediately. This "toggling" action typically forces the backend messaging tables to update.
  4. Escalation: If the notifications continue after these steps, contact SAP Support and reference this article

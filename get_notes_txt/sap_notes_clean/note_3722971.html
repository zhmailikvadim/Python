Symptom

After deleting all remaining confirmations of a user-created alternative for a requirement in the Review Availability Check Result UI, the chosen confirmations table is empty and an error message is displayed stating that the chosen confirmations are not available, even though they are available.

Other Terms

ATP; aATP; RACR; Review Availability Check Result

Reason and Prerequisites

For validating the confimations edited in Review Availability Check Result (RACR), an availability check with the chosen confirmations is performed. This validation requires that at least one confirmation line exists. Therefore, when deleting confirmations in Review Availability Check Result, it is evaluated if there's at least one confirmation line and, if not, a zero confirmation is created for the requested date and the requested combination of product and location. However, due to a programming error, this evaluation for missing confirmation lines is not done for deletions of confirmations from user-created alternatives.

Solution

Apply the correction instructions for this note or upgrade to a SAP S/4HANA support package that contains this correction.

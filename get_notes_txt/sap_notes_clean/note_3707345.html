Symptom

You have created a contract in Ariba, and you are using drag-and-drop or cut-and-paste to amend contract items and change parent-child relationships. The error “ProcessContractStatusUpdateRequestInternalError” is raised when returning to the external system.

Environment

  • SAP Ariba Procurement, cloud edition
  • SAP Ariba Sourcing, cloud edition
  • SAP Ariba Contracts
  • SAP ERP Central Component
  • SAP ERP
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA Finance
  • SAP S/4HANA

Reproducing the Issue

  1. Create the contract. 
  2. Drag and drop or cut and paste items to change the parent of a child item.
  3. Send the contract from SAP Ariba to the external system.
  4. The error “ProcessContractStatusUpdateRequestInternalError” is raised in SAP Ariba.

Cause

The error is raised because the SAP item loses its connection to Ariba. When downloading the item spreadsheet from the contract, the "External System Line Number" column is supposed to be populated, but it is empty. In addition, the SRVMAPKEY field in the ESLL table has duplicate entries for various items, where the SRVMAPKEY of deleted items or child service lines is being reused for new items. Field SRVMAPKEY serves as the document's item ID and must remain unique.

Resolution

SAP does not support dragging and dropping or cutting and pasting items between service lines, whether SAP Ariba is integrated with an external ERP system. The mapping in SAP Ariba is not reflected in the ERP system, which can cause failures due to a sequence mismatch.

The documentation below clarifies the supported environment of Adding Service Items to Contract Line Items Documents:

https://help.sap.com/docs/strategic-sourcing/managing-projects-teams-documents-and-tasks/how-to-add-service-items-to-contract-line-items-documents?ai=true

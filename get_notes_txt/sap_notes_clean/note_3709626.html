Symptom

There is the need to download SM50 traces for troubleshooting Datasphere connection issues.

Environment

  • SAP Connectivity service 2.0
  • SAP Datasphere

Resolution

Reproduce the error to get the latest log entries:

1. Collect the SM50 Traces:

a) Call transaction SM50 (process list).
b) Key combination: F5 (select all), CTRL-Shift-F7 => Dialog box.
c) Set Trace Level = 3 and check the "Security", "ICF" and "Crypto Lib" components.
d) Reproduce the issue.

2. To deactivate the trace, repeat step "a" and "b". In step "b" press button 'default values' and save.
3. To download the traces:

a) Execute transaction SE38.
b) Execute the program RSMON000_DOWNLOAD_TRACES.
c) Select the option to provide only "dialog work processes".
d) Select the option to compress first (as .SAR/.ZIP).
e) Select the save/download button.

3. Attach the resulting .SAR file (which will contain all work process files).


Note: provide the userID used and the timestamp when this was reproduced. See details in Opening and downloading developer traces

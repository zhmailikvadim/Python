Symptom

Historical document data in the transactiondetails table from the last few years is required to be deleted from HAC . Attempts to delete the records using flexible queries in HAC fail due to missing references. 

Environment

SAP Commerce Cloud 2211.45 or higher

Reproducing the Issue

  1. Attempt to delete historical document data (eg: transactiondetails table) from HAC using flexible query.
  2. Observe that the query fails due to missing references.
  3. Proceed to delete the records directly from Solr admin UI.

Cause

The issue arises because transaction details already deleted in the database are still present in Solr, leading to flexible query failure due to missing references.

Resolution

  1. Access Solr admin UI.
  2. Delete the required historical document data directly from Solr admin UI.
  3. Ensure that the deletion plan for large data volumes accounts for socket or connection timeout limits.If information on how the Solr server and indexer configurations are set up is needed, the "KBA/SAP Note - 3089650 - Solr Server and Indexer Configurations in Backoffice"and SAP help portal - Define Search and Indexer Configuration can be checked.

  4. Deletion of a document from the SOLR Admin UI is recommended only when the data is not present in HAC, no backup is available, and it is no longer required. Testing should be performed in a lower environment before it is carried out in the Production environment.

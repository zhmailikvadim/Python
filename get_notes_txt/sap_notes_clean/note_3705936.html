Symptom

HEX queries fail with an error like below:

general error: solution cache is missing, exp=(0, 0), op=HEX SELECT (opId: P:0000): FILTER: 'TABU' = (0, 0) SortList: (0, 0) ASC, (0, 0) ASC, (0, 0) ASC, (0, 0) ASC; $condition$=exists(op)

Note:The error above is a general error when a HEX query fails. Please review the prerequisites below to determine if your issue matches the one described in this SAP Note.

Other Terms

HEX, customizing transport request, S4

Reason and Prerequisites

Issue number 352847

Reason:

Due to a programming error in SAP HANA, the information of the queried field is handled improperly between query operations such as JOIN or UNION ALL.

Affected Releases:

  • SAP HANA 2:
    • Revisions of SPS07
    • Revisions <= 089.00 (SPS08)

Prerequisites:

  • A query has UNION ALL and one of the UNION ALL result returns constant values

  • And the query has JOIN, TOP and ORDER BY operators
  • And the JOIN operator refers the constant value from the UNION ALL
  • And the query is compiled to run on the HEX engine

For example:

SELECT "TAB0".A0 FROM

(SELECT TOP 100 * FROM "TAB0" ORDER BY "TAB0".A0, "TAB0".B0, "TAB0".C0, "TAB0".D0, "TAB0".E0)

LEFT OUTER JOIN

(

SELECT 1 AS "TAB1".A1, "TAB1".B1, "TAB1".C1, "TAB1".D1 FROM "TAB1"

UNION ALL

SELECT 1 AS "TAB2".A1, 2, "TAB2".C1, "TAB2".D1 FROM "TAB2"

) U

ON "TAB0".C0 = U.A1 AND "TAB0".D0 = U.B1 AND "TAB0".E0 = U.C1

GROUP BY "TAB0".A0, "TAB0".B0, "TAB0".C0, "TAB0".D0, "TAB0".E0, U.A1, U.B1, U.C1, U.D1;

Solution

Apply one of the following SAP HANA Revisions:

  • SAP HANA 2:
    • Revisions >= 089.01 (SPS08)
    • or higher

  • or higher

Workaround:

Apply the hint NO_SEMI_JOIN_REDUCTION by using one of the following methods:

  • Use a statement hint, for example:

ALTER SYSTEM ALTER STATEMENT HINT (NO_SEMI_JOIN_REDUCTION) COMMENT 'SAP Note 3657808. Revert after upgrade to a revision containing the fix' FOR <statement_string> or <statement_hash>;

  • Add WITH HINT at the end of the SQL statement, for example:

<statement_ string> WITH HINT(NO_SEMI_JOIN_REDUCTION);

  • Set a configuration parameter, for example:

ALTER SYSTEM ALTER CONFIGURATION ('indexserver.ini', 'SYSTEM') SET ('sql', 'enable_semi_join_reduction') = 'false' WITH RECONFIGURE COMMENT 'SAP Note 3657808. Revert after upgrade to a revision containing the fix';

Note: Setting configuration parameter indexserver.ini [sql] enable_semi_join_reduction impacts all queries in system widely. Please revert the workaround once you have upgraded to a revision containing the permanent solution.

Disclaimer

The SAP HANA configuration parameter indexserver.ini [sql] enable_semi_join_reduction is an internal parameter and may only be used in the context of this SAP Note. Using such a parameter requires that the SAP HANA system has the revision specified in the SAP Note. Using internal parameters outside of this given scope is not supported. Please unset this parameter once you have upgraded to a revision containing the permanent solution.

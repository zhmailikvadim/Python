Symptom

You define a job to delete planning version values with a TSCOPY variant. Due to missing authorization the job might fall into an endless loop and create millions of liveCache locks. If the executing user has no authorization (authorization object: C_APO_FUN, function: C_ZRO) the job should be cancelled with an error message. If deletion of the plannig version values fails TSCOPY should release the lock.

Other Terms

/SAPAPO/RTSCOPY, C_APO_FUN, C_ZRO, LC lock

Reason and Prerequisites

TSCOPY is executed in background by a user who has no authorization for function C_ZRO, and LC lock is active for the planning area.

Solution

Implement this correction.

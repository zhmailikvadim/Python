Symptom

  • Unable to add attachment on a leave request
  • 'Failed to send data. Please try again later' error when adding an attachment on mobile application within benefit claims.
  • 'Failed to load document. Please try again later.' error when viewing an attachment on mobile application within workflow details section.

Environment

  • SAP SuccessFactors Mobile
  • SAP SuccessFactors Employee Central

Reproducing the Issue

  1. On SF mobile app, try to:
    1. attach document in the leave request
    2. select an attachment to open in the workflow details
    3. send an attachment on a benefit claim
  2. The error will appear: "Failed to send data. Please try again later" or "Failed to load document. Please try again later."

Cause

The cause is that it is missing the permissions " AND "OData API Attachment Export" for the user to be able to upload.

The permissions are required as mobile app operates through API call and as such it will respect API permissions.

Resolution

  1. On desktop, access SuccessFactors site
  2. Navigate to Admin Center
  3. Set User Permission > Manage Permission Roles
  4. Find the necessary role to grant the permission
  5. Permissions > Manage Integrations Tools
  6. Grant the following permissions: " AND "OData API Attachment Export"
  7. Save 

You should now be able to manage attachments within SF mobile app.

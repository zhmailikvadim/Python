Symptom

  • Error: The query cannot run because it contains objects that reference incompatible tables.(IES 00008)
    Cause of Error Incompatible combination of tables. (IES 01507)
    occurs when running query in converted unx universe.
  • Creating the same query based on the original unv universe, it works fine.

Environment

  • SAP BusinessObjects Business Intelligence Platform 4.2 / 4.3 / 2025
  • Web Intelligence (WebI)
  • Universe Design Tool (UDT)
  • Information Design Tool (IDT)

Reproducing the Issue

  1. Open Universe Design Tool.
  2. Import the unv universe.
  3. Create new query using two objects .  
  4. Check that SQL generated is correct with open SQL.
  5. Open Information Design Tool.
  6. Convert the same unv universe to unx.
  7. Retrieve the converted unx universe and create new query using same two objects.
  8. Check the SQL or run the query.
  9. Error occurs as symptom.
  10. The same issue may also happen when using change source function to change Web Intelligence query source from unv to the converted unx universe.

Cause

  • In the original unv universe, there are some alias tables which the name has the same name with other tables. 
    E.g. There is one physical table named TableA. And there is one alias table also has the same name TableA.
  • In UNV universe, this duplicate name is possible as the physical table name has been assigned with different owner names.
  • However, in the converted UNX universe, the alias table must have the unique name. Otherwise it may bring un-wanted loops to make the tables to be incompatible.

Resolution

  1. Find the alias tables which have the duplicate names with other tables in unv universe.
  2. Rename the alias table to prevent such kind of duplication.
  3. Save the unv universe and publish to repository.
  4. Use IDT to convert the changed unv universe again and then the issue can be resolved.

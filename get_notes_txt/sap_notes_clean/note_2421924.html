Symptom

Debug Purchase Order integration between ECC and TM.

Environment

  • SAP ERP
  • SAP ERP Central Component
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAPHANA
  • SAP S/4HANA
  • SAP Transportation Management 9.0+

Reproducing the Issue

  1. Set a debug point in CL_SE_PUR_TRQSUITEREQUEST.
  2. Create a new PO or change an existing one.
  3. The debugger is not executed.

Cause

The XML creation is executed as part of a background process for the queue, rather than by the user process.

Resolution

There are two ways to debug the creation of XML messages for purchase order in ECC.

  • Deregister ME23N from the queue according to https://wiki.scn.sap.com/wiki/pages/viewpage.action?pageId=393322598.  This allows the debugger to open when you create or change a PO and save.
  • Alternatively, you can run SE24 and test CL_SE_PUR_TRQSUITEREQUEST.  The following steps describe this procedure:
  1. Ensure the following breakpoints are set in CL_SE_PUR_TRQSUITEREQUEST. 
  2. In SE24, enter CL_SE_PUR_TRQSUITEREQUEST and click Test icon or hit F8. 



  3. In GET_INSTANCE, click Execute Method. 




  4. In PROCESS_OUT, click Execute. 




  5. In IV_PO_NO, enter the PO number.
    In IV_EVENT_NAME, enter CHANGED.



  6. If you have set your debug breakpoints, the debugger should open.




    Note that this method does not commit the result.  In SXI_MONITOR, this message appears with a white flag indicating that it was not committed.  To submit this XML for processing, you need to run SPROXY.  Copy and paste this XML message into the appropriate transaction.

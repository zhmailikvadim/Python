Symptom

The memory allocator Pool/RowTableUpdateAllocator keeps increasing, when MERGE INTO statements are executed against a ROW table.

Other Terms

Pool/RowTableUpdateAllocator, MERGE INTO, ROW TABLE

Reason and Prerequisites

Issue number 340874

Reason:

Due to a programming error in SAP HANA, the memory allocator Pool/RowTableUpdateAllocator is not properly cleared after an execution of MERGE INTO statement against a ROW table.

Affected Releases:

  • SAP HANA 2:
    • Revisions of SPS00 - SPS06
    • Revisions <= 079.08 (SPS07)
    • Revisions <= 089.01 (SPS08)
  • SAP HANA Cloud:
    • Versions of 2025 QRC1 - 2025 QRC2
    • Versions <= 2025.28.30 (2025 QRC3)
    • Versions <= 2025.40.19 (2025 QRC4)
    • Versions <= 2026.2.6 (2026 QRC1)

Prerequisites:

  • MERGE INTO statements are executed against a ROW table.

Solution

Apply one of the following SAP HANA Revisions:

  • SAP HANA 2:
    • Revisions of SPS07: SAP is currently working on a permanent solution to this issue. This SAP Note will be updated as soon as a revision containing the solution is available
    • Revisions of SPS08: SAP is currently working on a permanent solution to this issue. This SAP Note will be updated as soon as a revision containing the solution is available
    • or higher

  • or higher
  • SAP HANA Cloud:
    • Versions >= 2025.28.31 (2025 QRC3)
    • Versions >= 2025.40.20 (2025 QRC4)
    • Versions >= 2026.2.7 (2026 QRC1)
    • or higher

Workaround:

  • Change a table type from ROW to COLUMN by executing the following command:

    ALTER TABLE <SCHEMA>.<ROW_TABLE> ALTER TYPE COLUMN;

Symptom

  • When updating the Service Definition, a below error occurred:
    RFC destination '<RFC Destination name>' is not functional
     
    But the mentioned RFC destination is not a correct one so the Service Definition source RFC destination should be changed.

  • When updating the Service Definition, 'Destination' is displayed with empty value

Environment

ABAP system for which the EWA report is generated

Reproducing the Issue

Transaction SDCCN
-> Menu Utilities
-> Maintain RFC Destinations
-> Select 'Delete and Replace Service Definitions'
-> Continue

Cause

  • In the RFC destinations setting, the source system is pointing the RFC destination which is not the correct one.
  • In the RFC destinations setting, the source system is not configured.

Resolution

  1. Transaction SDCCN
  2. Menu 'Goto'
  3. Settings
  4. Task-specific
  5. Under RFC Destinations, click on the icon which is located on the right side of 'Settings'
  6. Click on the icon "Display <-> Change" (2nd from left) from the left
  7. Click on the icon 'Set Service Definition Source system' (6th from left)
  8. Select 'Destination' and select the correct source from the help menu (F4)
    If the EWA report is generated by Solution Manager, it should be the destination for Solution Manager.
    If the EWA report is generated by SAP, it should be SAP-SUPPORT_PORTAL.

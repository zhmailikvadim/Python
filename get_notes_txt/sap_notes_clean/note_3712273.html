Symptom

This is a correction about incorrect browser focus with regard to the row action column in FPM List ATS UIBB. When the user clicks on an action inside the row actions, it may happen that the focus is somewhere else after it. If the action triggers a popup and the popup is closed afterwards, the application would need to call the method request_focus_on_row_action( ) of the extended controller API (if_fpm_guibb_list->get_data( )) in order to put the focus back on the row action UI element.

Solution

Please apply the correction instruction.

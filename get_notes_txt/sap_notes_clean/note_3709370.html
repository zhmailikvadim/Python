Symptom

You are using the DBA Cockpit to monitor an IBM Db2 for Linux, UNIX, and Windows database containing data-partitioned tables. 
In connection with such tables, you notice two kinds of errors:

  1. On the screen Space -> Top Space Consumers, you see a separate entry for each data partition, i.e. the total size of the data-partitioned table is not displayed. 

  2. On the screen Jobs -> DBA Planing Calendar, you notice that the job log of the job "REORGCHK for All Tables" shows the following error messages:  
    "Reorgchk_All:Table <table schema>.<table name> : Wrong data tablespace"

Other Terms

DBACOCKPIT, DB02, ST04

Reason and Prerequisites

The problem is caused by program errors.

These errors occur only for data-partitioned tables on your Db2 for LUW database.

Solution

The solution is shipped as part of the Support Package. Alternatively, you can implement the attached correction instruction.

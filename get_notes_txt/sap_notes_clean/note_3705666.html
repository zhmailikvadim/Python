Symptom

Idoc processing fails with status 51 in transaction BD87 and following error occurs:

  • Error message C$ 366 "Phrase key is not within valid range".
  • Error message CM_PHRASE_DATA 050  "Cannot create phrase text in source language".

Reproducing the Issue

  1. Navigate to transaction BD87 for already executed Idocs. Or use WE19 to copy an existing Idoc and try to post it again.
  2. Observe that Idocs fail with the error messages "Phrase key is not within valid range" or "Cannot create phrase text in source language" and Status 51 (Application document not posted).

Cause

  • Such errors are caused due to incorrect customizing of the number range for object ESN_PHRID
    OR
  • wrong manual number provided in the Idoc segment E1BP1091PH

Resolution

  1. Check the interval settings in transaction SNUM for object ESN_PHRID.
  2. Verify the number range configuration. Ensure that the system has a correct number range maintained.
  3. Compare and adapt the given number accordingly within the Idoc segment in use (Segment type E1BP1091PH, Field name PHRASE). It must fit to the setting of the number range no. from point 2.

  • Refer to: SAP Note 1146705 - Phrase created without phrase items.

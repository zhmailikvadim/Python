Symptom

During the parallel processing calculation run, incorrect items are being picked and processed.

Additionally, in the Reprice Document Index scenario, the system processes every bucket line instead of restricting the processing only to the relevant Index table bucket lines.
Furthermore, buckets that are locked during parallel sessions are not being unlocked properly, leading to subsequent processing issues.
This note corrects the behavior by ensuring that only the appropriate items are selected during parallel processing, restricting Reprice Document Index processing to the relevant Index table bucket lines, and properly unlocking buckets that were locked in parallel sessions.

Other Terms

Parallel processing issue, transactional calculation run, incorrect items selected, reprice document index, bucket line processing, index table bucket lines, bucket lock issue, unlock buckets, parallel sessions

Reason and Prerequisites

This is caused by Program error.

Please implement SAP Note #3694575, 3700449 and 3706932 as prerequisite notes.

Solution

Please implement the correction instructions attached. These corrections are only valid if you have applied the Support Packs.

Release 6.1 – Base Package

2. Use transaction SNOTE to implement the correction instructions specified below and activate the affected program object

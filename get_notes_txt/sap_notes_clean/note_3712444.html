Symptom

When attempting to rehire a former employee and replicate their data from SuccessFactors Employee Central (EC) to SAP S/4HANA (on-premise or private cloud), customers encounter an error during the user synchronization process. The error occurs when the system attempts to assign a different user ID to an employee who already has a user ID previously assigned and synchronized. Delimiting (ending) the relationship in Infotype 0105, Subtype 0001 does not allow assigning a new user ID, nor does it break the link between the existing user and the associated Business Partner. As a result, the replication fails and the employee cannot be registered with the new user ID.

Environment

  • SAP S/4HANA (on-premise)
  • SAP S/4HANA (private cloud edition)

Reproducing the Issue

  1. In Employee Central, a user is rehired with a new employee ID after a previous termination.
  2. The EC to S/4HANA replication is triggered, attempting to create or update the employee with a new user ID.
  3. The employee already has a previous user ID assignment in Infotype 0105, Subtype 0001, synchronized in a prior replication.
  4. The replication fails with an error indicating that changing the user ID is not permitted, even if the Infotype 0105, Subtype 0001 assignment is delimited.

Cause

Once the synchronization between EC and S/4HANA is executed, the system converts the user ID to a business user and links it to a Business Partner. For auditing and compliance reasons, this connection must remain permanently; it cannot be deleted or overwritten. Therefore, the system prevents assigning a new user ID in subsequent replications for the same employee.

Resolution

Once a user ID is assigned and synchronized to an employee from Employee Central to S/4HANA, this assignment and its link to the Business Partner become permanent and cannot be changed through delimiting Infotype 0105 or any other standard process. Delimiting the Infotype 0105, Subtype 0001 entry does not remove the underlying association; thus, attempts to assign a new user ID fail.

SAP Best Practice:

  • The Employee Central logon username must remain unchanged after it is initially replicated to SAP S/4HANA.
  • Do not attempt to change or overwrite the user ID once synchronization has occurred.

Risk Option:
If you have a business requirement to change the user ID, be aware this action involves risks and is not supported by default for compliance reasons. If the change is strictly necessary and you accept the associated risks, follow the steps outlined in Knowledge Base Article (KBA) 2597146 BP integration: Changing the SAP User ID of an employee in an Employee to Business Partner synchronization scenario. This procedure should only be followed after careful consideration and acceptance of the risk to the audit trail.

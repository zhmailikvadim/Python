Symptom

  • Employees are unable to make corrections to their data during the onboarding process prior to signing documents.
  • The onboarding process requires administrators to restart onboarding if incorrect data is identified.
  • No option exists for employees to modify their data without administrative intervention.

Environment

SAP SuccessFactors Onboarding

Reproducing the Issue

  1. Initiate the onboarding process in SAP SuccessFactors Onboarding.
  2. Provide incorrect personal data during the onboarding data collection phase.
  3. Attempt to modify the incorrect data prior to signing the onboarding documents.
  4. Observe that there is no functionality available for employees to make corrections without restarting the onboarding process.

Cause

The system design does not currently support employee-driven data corrections during onboarding prior to document signing.

Resolution

  1. Confirm that the current system does not allow data corrections by employees during onboarding without administrative intervention.
  2. If a change in system behavior is required, submit an Enhancement Request to SAP for consideration in future releases.
  3. Monitor SAP updates and announcements for potential inclusion of this functionality in upcoming versions.

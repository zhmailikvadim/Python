Symptom

The selection logic in T-code: MFBF when using BOM alternative Group with strategy 2 is unclear.

Environment

  • SAP ERP Logistics (PP)  
  • SAP R/3
  • SAP R/3 Enterprise
  • SAP ERP Central Component
  • SAP ERP
  • SAP Enhancement package for SAP ERP
  • SAP Enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4 HANA, on premise
  • SAP S/4HANA Cloud Private Edition

Reproducing the Issue

  1. Maintain the BOM alternative Group in BOM items in T-code: CS02
  2. Material          Priority    Available Stock
    TESTREM_SD    1                 2
    TESTREM_RD    2                 3
    TESTMRP_RD    3                 7
  3. Post MFBF for header material and check the required component quantity.
  4. When posting MFBF for header material with quantity 2, component TESTREM_SD is selected.
  5. When posting MFBF for header material with quantity 3, component TESTREM_RD is selected.
  6. When posting MFBF for header material with quantity 7, component TESTMRP_RD is selected.
  7. When posting MFBF for header material with quantity 12, component TESTREM_SD is selected even though the quantity does not fully cover the requirement. 

Cause

This is standard behavior.

Resolution

In T-code: MFBF, partial consumption for more alternative items are not foreseen. With an alternative item group and strategy 100% check, if the required quantity can not be covered by any alternative component, system selects the first alternative (by priority) that has any ATP quantity available, even if this quantity does not fully cover the requirement.

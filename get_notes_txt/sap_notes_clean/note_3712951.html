Symptom

Discrepancy in the wording of an business rule when changing the language: it becomes partially translated even though it has translation settings configured.

Environment

SAP SuccessFactors HCM Suite

Reproducing the Issue

  1. Access the system.
  2. Verify that the system language is set to German, for example.
  3. Go to "Configure business rules".
  4. Search for the business rule.
  5. Notice that the first part is translated, but the other part is still displayed in the default language (for example, English).
  6. Go to the "Configure Object Defnitions"
  7. Search for the custom object
  8. Verify that the correct translations exist for both English and German in the object labels.
  9. Therefore, since the system language is set to German, all text should be displayed in German.

Cause

This issue was identified by our engineering team and will be fixed in the 1H2026 release.

 

Resolution

The fix for this issue is planned to be deployed on 1H 2026 release.

For release timelines, please review SAP SuccessFactors Product Release & Road Map Information.

No workaround is available.

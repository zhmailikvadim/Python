Symptom

The security SAP Note 3672622 is to be applied, however, during the implementation, facing the following error message:

Object cannot be created again. Object DOCV UOS_CMOD_ADM missing

Environment

  • SAP NetWeaver
  • SAP S/4 HANA

Reproducing the Issue

Implement SAP Note 3672622.

Cause

Program issue

Resolution

Follow the workaround:

  1. During the first implementation, click “Next” without selecting the “Apply Changes” checkbox for UOS_CMOD_ADM.
  2. After this first pass, the SAP Note will appear as only partially implemented.
  3. Run the implementation a second time; the previously missing object will then be available and can be activated.
  4. After the second pass, the implementation should complete successfully.

The above workaround is supported from:

  • SAP_BASIS 750 SP27 onwards
  • SAP_BASIS 751 SP17 onwards
  • SAP_BASIS 752 SP12 onwards
  • SAP_BASIS 753 SP10 onwards
  • SAP_BASIS 754 SP08 onwards
  • SAP_BASIS 755 SP06 onwards
  • SAP_BASIS 756 SP04 onwards
  • SAP_BASIS 757 SP02 onwards
  • All higher releases of SAP_BASIS

IMPORTANT: SAP Note 3672622 has been re-released. 

For SAP_BASIS releases 700 to 702, 731 and 740, perform the manual steps of SAP Note 3716308 prior to applying the correction instructions.

The authorization object needs to be created manually according to the steps in 3716308 as it cannot be created with correction instructions in these releases.

Implement the Support Package & Patches or the Correction Instructions mentioned in this SAP Security Note.

After implementing the fix, users who previously had unrestricted access to certain text modification actions may be blocked unless they are explicitly assigned the new authorization object S_CMOD_ADM. To avoid disruption, review and update the user roles and authorizations accordingly.

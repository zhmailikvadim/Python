Symptom

While sending inspection setup information (inspection types) to EWM system via transaction QA11, the inspection rule is not created for a material, however generated properly for an other material.

Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.

Environment

  • SAP ERP Central Component
  • SAP ERP
  • SAP Enhancement package for SAP ERP
  • SAP S/4HANA, on-premise
  • SAP S/4HANA Cloud Private Edition

Cause

At time of the IDoc creation with transaction QL11, the inspection type (with lot origin 17) was inactive in the material master QM view - inspection setup. E1BP1001004_QMAT-IND_INSPTYPE_MAT_ACTIVE is empty in the IDoc.

Resolution

  1. Activate the inspection type in the material master - for example via transaction MM02.
     

     
  2. Resend the inspection setup data via QL11.

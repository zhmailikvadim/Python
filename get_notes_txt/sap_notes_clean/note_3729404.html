Symptom

When attempting to create a custom Organizational Management (OM) infotype using transaction PPCI, the infotype is not created successfully. The creation process does not complete even though the required structure (e.g., HRIxxxx) has been created beforehand. This issue occurs regardless of whether entries have been manually maintained in views T777Z and T777D prior to running PPCI, or whether no entries have been maintained at all.

Environment

  • SAP ERP
  • SAP S/4HANA on-premise
  • SAP S/4HANA private cloud edition
  • SAP CRM
  • SAP SRM

Cause

Entries for the custom infotype were manually created in the following views before running transaction PPCI:

  • T778T
  • T777D
  • T777Z

Transaction PPCI is designed to automatically generate the required entries in these views as part of the infotype creation process. If entries already exist in these views, PPCI cannot complete the creation successfully because it expects to create them itself.

Resolution

  1. Go to transaction SM30.
  2. For each of the following views, switch to edit mode and delete the entries corresponding to the custom infotype:
    • T778T
    • T777D
    • T777Z
  3. Ensure that only the custom infotype structure (e.g., HRIxxxx) remains. No other manually created entries for the infotype should exist in the above views.
  4. Re-run transaction PPCI to create the custom infotype. PPCI will now automatically generate all required entries in the views.

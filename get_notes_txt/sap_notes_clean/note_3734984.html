Symptom

When navigating from the App Profit Center Actual Data to the line items in Cost Accounting (F4023), the profit center parameter is either not transferred correctly or not transferred at all. As a result, the line items display data for all profit centers instead of the actual data previously shown in the app.

Environment

SAP S/4HANA Finance, SAP S/4HANA, SAP S/4HANA Cloud Private Edition, SAP Fiori for SAP S/4HANA

Reproducing the Issue

1.Open the APP Profit Center Actual Data.
2.Select any Profit Center and Navigate (JumpTo) to the line items in Cost Accounting (F4023).
3.Observe that the profit center parameter is either not transferred correctly or not transferred at all.

Cause

This is the current system design. The limitation lies in the semantic object GLAccount used by F4023. This semantic object does not support the Profit Center parameter in this navigation scenario. Profit Center transfer is only supported when using the CostCenter semantic object.

Resolution

1.Use F2217 (Display Line Items in General Ledger) instead of F4023 when navigation requires the Profit Center parameter.
2.Refer to the Fiori Apps Library → Implementation Information → Target Mappings for details: F4023 App in Fiori Apps Library (https://fioriappslibrary.hana.ondemand.com/sap/fix/externalViewer/#/detail/Apps('F4023')/S28OP  )

Keywords

Profit Center, F4023, Cost Accounting, Fiori Apps, JumpTo functionality, GLAccount semantic object, Parameter transfer

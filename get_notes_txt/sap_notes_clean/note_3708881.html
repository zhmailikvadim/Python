Symptom

SSO is not working on FC HTML5 and the login and password page is always displayed despite that the SSO configuration is in place on both FC and IPS side.

Environment

  • SAP Financial Consolidation
  • Windows server 2019 and higher

Cause

The application pool user for the FC HTML5 website deployed on IIS does not belong to the “Impersonate a client after authentication” policy. 

Resolution

Until Windows 2019, all the IIS_IUSRS belonged to the "Impersonate a client after authentication" policy.

Since W2019, it’s no longer the case. So, when you enable windows authentication, you have to make sure that your application pool user belongs to the “Impersonate a client after authentication”
policy.

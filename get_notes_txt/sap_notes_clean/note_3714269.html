Symptom

  • Error occurs when attempting to change a user ID in Maintenance (Portal) and employee records in Participants UI (Commissions).
  • SAP Help indicates that only admin users can operate in Maintenance.
  • Admin user can open Maintenance and input old and new User IDs, but an error occurs, and the user ID is not changed.

Environment

Incentive Management

Reproducing the Issue

  1. Create a user ID with access to change user IDs in Maintenance (Portal) and employee records in Participants UI (Commissions).
  2. Create an admin user and assign permission to access Portal - Maintenance.
  3. Open Maintenance with the admin user and input old and new User IDs.
  4. Attempt to change the user ID.
  5. Observe the error message and failure to change the user ID.

Cause

The Administrator role in the system had limited access roles, which did not include the necessary permissions to execute maintenance tasks and manage security principals.

Resolution

  1. Verify that the Administrator role is assigned to the user attempting to update the User IDs in Maintenance (Portal).
  2. Add the following roles to the Administrator role:
  • Execute maintenance tasks
  • Manage Security Principals
  1. After adding the roles, attempt to update the user ID again.
  2. Validate the changes and ensure the error is resolved.

See Also

Refer to: [SAP Note 1234567 - Administrator Role Configuration]

Symptom

You create an agent in SAP Plant Connectivity (PCo) and configure it to connect to an OPC server.

The agent is successfully connected and used in the ERP system via transactions COOPC or COOPC1 for namespace browsing.

However, during namespace browsing in ERP:

  • Only a limited number of tags (for example, 60) are displayed
  • Not all available tags from the OPC server are visible

Other Terms

 Plant Connectivity, PCo, OPC DA, OPC UA, Namespace Browsing, Tag Limit, COOPC, COOPC1

Reason and Prerequisites

Program Error

Solution

Install Patch 07 for SAP Plant Connectivity 15.5 with Support Package 5.

Download the current installation program for your release from the SAP Service Marketplace and install it in accordance with the instructions in the installation guide. The current version of the installation guide is available on the PCo documentation page in the SAP Help Portal at https://help.sap.com/pco. You can use the installation program for the PCo patch to update a PCo version that is already installed or to set up a new installation if PCo is not yet installed on the target computer. The installation will install the required Visual Studio C++ runtime environment needed by the .NET Connector.

Before you start the installation, refer also to the installation instructions for PCo 15.5.5 in SAP Note 3480115.

Follow the steps below to configure the number of tags :

1. Open the corresponding PCo agent .exe.config file located at:
    C:\ProgramData\SAP\PCo\Config\Instances\


2. Add or Update the following entry within the existing <appSettings> section:
   <add key="MaxBrowseTags" value="number of tags which you would like to see in ERP" />


   Example: To display 250 tags, use:
   <add key="MaxBrowseTags" value="250" />

3. Restart the affected PCo Agent(s).

4. Refresh the ERP side (for example, re-open the transaction/screen or re-trigger namespace browsing) and verify that the expected number        of tags are now visible.

NOTE - Please keep value for MaxBrowseTags to limited value to avoid any performance issue.

Symptom

You are using the Manage Assortments app and navigates to Analytics screen in SAP CARAB 5.0 FPS04, 5.0 SPS05, 5.0 FPS06, 5.0 FPS07, 5.0 FPS08, 6.0 SPS01 or 6.0 FPS02.

The inbound process loads some master data (products, locations, hierarchy) and transactional data (syndicated timeseries). It checks to see if the products and locations already exist in the database. If they don’t, then it creates them. However, it doesn't finds the existing products, so when you upload similar data again, it creates the same products in table /DMF/CMP_PROD_XR with new GUIDs.

Other Terms

SAP Customer Activity Repository applications bundle, CARAB, CARAB 5.0 FPS04, CARAB 5.0 SPS05, CARAB 5.0 FPS06, CARAB 5.0 FPS07, CARAB 5.0 FPS08, CARAB 6.0, CARAB 6.0 SPS01, CARAB 6.0 FPS02, CAR, SAP CARAB, SAP Assortment Planning, Analytics

Reason and Prerequisites

Prerequisites:

  • You have set up the Manage Assortments app as described in the Common Installation Guide (for new installations) or the Common Upgrade Guide (for upgrade scenarios) for SAP Customer Activity Repository applications bundle. You can find guides on SAP Help Portal at https://help.sap.com/docs/CARAB. Select your version at the top.

Implement the correction instructions attached to this note.

Symptom

  • The inability to map custom MDF fields with onboarding data of candidates, such as first and last name, in SAP SuccessFactors Onboarding.

Environment

SAP SuccessFactors Onboarding

Reproducing the Issue

  1. Attempt to map custom MDF fields with onboarding data of candidates (e.g., first and last name) in SAP SuccessFactors Onboarding.
  2. Perform tests to create the required mapping scenario.
  3. Verify the configuration and check if the mapping is successful.

Cause

You can use custom MDF objects to collect candidate data, but you cannot directly “map” core fields like first/last name from Onboarding to a custom MDF field in EC.

Resolution

  1. you can use custom MDF objects to collect candidate data, but you cannot directly “map” core fields like first/last name from Onboarding to a custom MDF field in EC.

  2. In ONB 2.0 you can create custom MDF Data Collection objects to capture extra onboarding data (e.g. T‑shirt size, laptop type).

  3. Those custom MDF objects are used in Additional Data Collection steps and can hold fields such as first name, last name, etc., but they are not automatically populated from the core Onboarding person fields unless you build custom logic outside standard mapping.

  4.  it is not possible to map a custom MDF Data Collection object directly to EC custom MDF objects or standard EC portlets via the standard Onboarding‑to‑EC mapping tool.

  5. Core fields like first-name / last-name are mapped from Recruiting → Onboarding → EC via the standard HRIS field mappings (PersonalInfo.first‑name, last‑name), not via a custom MDF “copy.”

Symptom

Users are unable to edit items when logged in Backoffice under a user group that has permissions to edit the corresponding itemtype while another user group assigned doesn't have permissions to edit that itemtype.

Environment

SAP Commerce Cloud

Reproducing the Issue

  1. In backoffice, configure Role A to allow editing one itemtype (e.g.subscription) and Role B to restrict editing subscription itemtype.
  2. Assign the two BackOffice user groups to a single employee.
  3. Log in under Role A and attempt to edit subscription items but found can't edit.

Cause

The behavior is expected.

Resolution

Please refer to User Account as Member of Several Individual User Groups listed examples.
The forth line matches above scenario that one user group denied edit, one user group granted edit and no permission setting for user, the final effective access is denied.

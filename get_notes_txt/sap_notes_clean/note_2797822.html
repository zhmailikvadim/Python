Symptom

You are using SAP Digital Manufacturing for execution (DMe) or SAP Digital Manufacturing for insights (DMi). You want to know what corrections and enhancements were delivered for the solution in previous releases.

Other Terms

SAP Digital Manufacturing for execution (DMe), SAP Digital Manufacturing for insights (DMi), corrections, hotfix, enhancement, 2602

Reason and Prerequisites

This SAP Note is subject to change. Please make sure that you obtain the current version.

Solution

IMPORTANT:

Corrections for SAP Digital Manufacturing for edge computing - 3525857

SAP Digital Manufacturing Production Change Limitation - 3335254

Overview on available patches for the SAP Production Connector can be found in the SAP Note - 3449364

Change Log for SAP Cloud Integration Package "SAP S/4HANA Integration with SAP Digital Manufacturing - 3315636

Corrections delivered for SAP Digital Manufacturing 2602

27/Feb/2026 - SAP Digital Manufacturing for Execution: Order Download - EMERGENCY FIX

The following issue has been resolved: when the payload size is substantial, the order download may temporarily strain system resources, leading to delay in order download performance some cases.

01/Mar/2026 - SAP Digital Manufacturing for Execution: Data Collection

The following issue has been resolved:
When you pressed Enter while logging data collection parameters, the system added an empty line to the parameter of type Text. This issue is now fixed. Pressing Enter no longer creates empty lines.

01/Mar/2026 - SAP Digital Manufacturing for Execution: Data Collection
The following issue has been resolved:
The data collection parameters of type List weren't working when the Data Collection plugin was used in Independent Mode or when the plugin was extended. This issue is now fixed, and the plugin works as expected.

01/Mar/2026 - SAP Quality Issue Resolution: PSP

PSPs were being created with a null user ID in the D1 step. This was due to a letter case mismatch (uppercase/lowercase) between S4 & IAS. This has been fixed so that the email case is handled correctly.

[Date correction!] 08/Mar/2026 - SAP Digital Manufacturing for execution: Manage Non-Conformance

In some scenarios, changes to Non-Conformance master data for dataType objects were not immediately reflected in the Non-Conformance log. Therefore, the caching mechanism for dataType the Nonconformance microservice has been improved to ensure changes are propagated faster across SAP DM components.

15/Mar/2026 – SAP Digital Manufacturing for Execution: Dispatching and Monitoring

An issue was identified in which some production orders downloaded to SAP Digital Manufacturing (DM) did not appear in the Dispatching and Monitoring application.

15/Mar/2026 – SAP Digital Manufacturing for execution: Design Production Processes

In the 2602 release, the system onboards built-in services in new tenants with incorrect IDs instead of their predefined IDs. The Publish MQTT Message service isn't affected. This causes validation to incorrectly identify these services as user-defined when deploying a production process design. If your production processes use the affected services, you might need to reconfigure them before deploying your production process design.

15/Mar/2026 - SAP Digital Manufacturing for Execution: Quality Inspection API

The following issue has been resolved: The paging functionality for the "Get Inspection Single Results" public API was not working as expected. This issue has now been resolved.

15/Mar/2026 - SAP Digital Manufacturing for Execution: Manage BOM and Manage Routing

The processing for the BOM (Bill of Materials) and Routing update events has been updated to ensure that all downstream customers receive accurate, up-to-date modification timestamps. No customer actions are required. This is an internal processing improvement that takes effect automatically. Downstream services will begin receiving the corrected "modifiedDateTime" values in update event payloads without any configuration changes.

15/Mar/2026 - SAP Digital Manufacturing for Execution: SAP HANA Integration Improvements

This fix addresses the service degradation events observed on February 25, 2026, in the EU20 data center (EVB10709980; EV50042512). These incidents resulted from an Azure network downtime that affected SAP HANA and, consequently, certain Digital Manufacturing Component Assembly operations. To prevent recurrence, all synchronous SAP HANA database operations have been replaced with asynchronous equivalents, and implicit connection pooling has been enabled in the SAP HANA client library. These updates improve overall performance and allow operations to complete as expected.

15/Mar/2026 - SAP Digital Manufacturing for Insights: Manage Dashboards app

The following issue has been resolved: Users couldn't create stories in the Manage Dashboards app by connecting to the MDO data source. An authorization error occurred even with correct SAML SSO configuration, preventing the embedded SAC system corresponding to the Digital Manufacturing system from connecting to the MDO data source. The issue is now fixed, allowing the connection to be established and ensuring proper authorization checks.

22/Mar/2026 - Digital Manufacturing for execution: Work List Item Selection in Operation Activity PODs

The following issue has been resolved: A work list item selected in an Operation Activity POD could be automatically unselected and the Main Input field of the Operation Activity POD Selection plugin could be automatically cleared when you'd use the Tab key to move away from the Main Input field. This could happen in specific cases where the Main Input Action ID configuration option of the Operation Activity POD Selection plugin has the Start plugin configured to be executed when you use the Tab or Enter key in the Main Input field of an Operation Activity POD. Now, this fix ensures that, when you enable the Disable SFC Selection by POD Notification option of the Operation Activity POD Selection plugin, the work list item remains selected and the Main Input field isn't automatically cleared in the aforementioned case.

22/Mar/2026 - Digital Manufacturing for execution:

The system displayed an unclear message when non-conformances were logged with a nonexistent SFC multiple times within a short period: "CircuitBreaker is OPEN and does not permit further calls." Now, a clear and accurate message appears: "Issue with calling an external service: SFC doesn't exist."

29/Mar/2026 - Digital Manufacturing for execution: SFC Merge

In the SFC Merge  plugin, when attempting to merge SFCs that had been previously split, on rare occasions the merge operation could not be completed and an error was displayed. The error message incorrectly indicated a split failure, even though the operation being performed was a merge.

This issue has been corrected. Now, given correct values, SFC merge operations should complete successfully.

29/Mar/2026 - Digital Manufacturing for execution: Assemble Components

The assembly service would fail and stop responding after prolonged periods of inactivity. An authentication token used by the background worker would expire, preventing the service from processing orders until it was manually restarted.

The application now implements periodic token refresh to maintain valid authentication during idle periods and includes a new health check endpoint that enables automatic service recovery if the token refresh fails. Assembly services continue operating normally with no interruption during periods of inactivity.

29/Mar/2026 - Digital Manufacturing for execution: Dispatching and Monitoring

The following issue has been resolved: Previously, some production orders downloaded to SAP Digital Manufacturing (DM) did not appear in the Dispatching and Monitoring app. A troubleshooting option will show right missing order associated to work center of the supervisor has now been enabled to show help resolve this problem.

05/Apr/2026 - Quality Issue Resolution: Problem-Solving Processes tiles on the DM Launchpad (fix for US20Data Center only)

In the US20 data center, the SAP Fiori Launchpad tiles for Quality Issue Resolution were not deployed correctly to the Quality and Production environments. The deployment pipeline had an issue causing Problem-Solving Processes (PSP)-specific tiles to be absent after deployment.

This issue has been corrected. The deployment pipeline issue has been resolved for US20 environments. Now, PSP tiles are deployed correctly to both the Quality and Production environments in the US20 data center.

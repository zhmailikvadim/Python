Symptom

  • In SAP Invoice Management by Open Text (VIM), if you are using the Enter Cost Assignment Fiori App and you are trying to approve an invoice that has a missing cost object (CO), a wrong error message is being shown for ot.approve.request03.util.ODataConnector
  • Expected is a meaningful warning like shown in SAPGUI for the same action:
    "Account xxx requires an assignment to a CO Object"

Environment

SAP Invoice Management by OpenText All versions

Resolution

  1. Create an case on SAP component XX-PART-OPT-INV and reference this Knowledge Based Article 2892803
  2. Request in the case that OpenText Support provide the necessary Correction Instruction(CI) VIMI-24715

Keywords

OpenText, Invoice Management, CO object, Enter Cost Assignment, Enter Cost Assignment Fiori App, missing cost object, ot.approve.request03.util.ODataConnector, Account xxx requires an assignment to a CO Object

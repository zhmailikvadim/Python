Symptom

The field c_TipoDeComprobante is not determined when generating an e-document with source SD_INVOICE, causing XML generation to fail. The XML shows the attribute TipoDeComprobante as empty, resulting in an invalid invoice that is rejected by the tax authority, blocking the electronic invoicing process. 

Environment

SAP S/4HANA Cloud Public Edition

Reproducing the Issue

  1. Open the eDocument Cockpit APP.
  2. Search for the e-invoice.
  3. Choose the button Go to - Application Log.
  4. Observe the error messages:
  • "Error interno: Error de generación de XML: Error"Error interno: Error de generación de XML: Errorde serialización de comprobant [EN: "Internal Error: XML Generation Error: Error"Internal Error: XML Generation Error: Verifier Serialization Error]
  • error: '' is not a valid value forc_TipoDeComprobante."

Cause

Missing or incorrect value maintained in the value mapping Configuring Document Types (DOC_TYPE).

Resolution

Based on the value mapping Configuring Document Types (DOC_TYPE) , the system maps the document types that you use in your SAP system to the document types that the SAT has defined and fills in the 'TipoDeComprobante' field in invoice XML files. Therefore, please follow below steps to ensure the correspond values are maintained well.

  1. Go to transaction /AIF/VMAP.

  2. Navigate to Value Mapping → Define Value Mappings.

  3. Select the mapping object DOC_TYPE.

  4. Verify the source values in the mapping entries:

  5. Ensure they correspond to valid SD document categories such as (M= Invoice, N= Credit Memo, O= Debit Memo).

  6. Ensure the target values are aligned with the expected TipodeComprobante values required by the Mexican tax authority, for example: [ I = Ingreso (income), E = Egreso (expense), T = Traslado (transfer)].

  7. Save and activate your changes.

Once done,  please re-run the XML display and verify that TipodeComprobante is now populated correctly.

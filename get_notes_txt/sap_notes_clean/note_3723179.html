Symptom

In the SAP Fiori app "Manage Sales Orders - Version 2", using the "Propose Items" function causes an ASSERTION_FAILED dump in program CL_SD_SLSDOC_BEHAVIOR=========CP.

Other Terms

F3893, FILL_ENTITY_KEY_FROM_SEMKEY, FILL_FAILED_AND_REPORTED, CL_SD_SDOC_BEHAVIOR_UTIL=>FILL_FAILED_AND_REPORTED, RFC Error: The ASSERT condition was violated, A Remote Function Call (RFC) to backend system terminated with a system failure exception, /IWBEP/CM_V4H_RUN016.

Reason and Prerequisites

Via the "Propose Items" function in the SAP Fiori app "Manage Sales Orders - Version 2" you can receive product and quantity proposals to ensure a faster selection of products. However, for the functionality to work correctly, the product proposal data must be consistent with the product master data.

If the product proposal data is inconsistent with the product master data, the system must abort the proposal process. To avoid creating an inconsistent sales order, the abort is reported as an ASSERTION_FAILED dump.

Solution

Correct the product proposal data to resolve the ASSERTION_FAILED dump. In particular, ensure that the data used in the product proposal is valid for creating sales order items.

Possible causes of invalid data include:

  • The unit of measure used in the product proposal is not valid for the product.
  • The unit of measure used in the product proposal is not valid as a sales unit for the product (for example, the material master has the "Variable Sales Unit Not Allowed" flag set, which allows only one specific sales unit).

For information on where and how product proposal data is determined and how it can be updated, see the Dynamic Product Proposal documentation.

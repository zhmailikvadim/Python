Symptom

When using the standard (SAP-delivered) Financial Statement Version (FSV) or migrating it (for example, to Global Hierarchies), errors or inconsistencies occur because certain G/L accounts referenced in the FSV are not present in the customer system. These G/L accounts cannot be found in Manage G/L Account Master Data and may show an implementation status such as Not Used or appear as non-existent.

Environment

S/4HANA Public Cloud.

Cause

SAP delivers standard Financial Statement Versions that are designed to be globally valid and scope-independent.
They may reference the full global chart of accounts, including accounts that are not activated, not scoped, or not created in a specific customer tenant.

In SAP S/4HANA Cloud:

  • Not all preconfigured G/L accounts are instantiated in every customer system.

  • The existence of an account in SAP Best Practices documentation does not guarantee that it exists as master data in the tenant.

  • Financial Statement Versions are structural templates, not a reflection of active master data.

This behavior is by design.

Resolution

The system behaves as expected.

Symptom

  • SDI Integration job XXX failed with the following error logs:
  • Unable to connect to table IBP_VERSION.
  • Cannot find API version information.
  • No valid API components detected.

Environment

  • SAP Integrated Business Planning (SCM-IBP)

  • SAP S/4HANA or SAP ERP, supply chain integration add-on for SAP Integrated Business Planning (SCM-IBP-INT-ECC)

Reproducing the Issue

  1. Run the SDI Application Job,
  2. Check following errors in the Application logs
  3. Error : Unable to connect to table IBP_VERSION

Cause

Since Linux account file and data files are mounted with a service account, the mount did not occur after a weekend patch update. The service account's automated process did not update the password, which caused the failure to mount data files.

Resolution

  1. Update the password manually for the service account to ensure the data files are mounted correctly.
  2. Check the data files are mounted successfully
  3. Execute the SDI Integration Job and confirm that It is finished successfully

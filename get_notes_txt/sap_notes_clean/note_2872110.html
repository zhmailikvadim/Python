Symptom

A user-role on tab "Authorizations" cannot be assigned to a bucket due to message "The check did not produce any hits. Call the search."

Environment

SAP Portfolio and Project Management (PPM)
SAP Portfolio and Project Management for SAP S/4HANA (PPM)
SAP Portfolio and Project Management in SAP S/4HANA (PPM)

Reproducing the Issue

  1. Navigate to Portfolio Management > Portfolio Structure
  2. Select a Portfolio
  3. Open a bucket
  4. Go to tab "Miscellaneous > Authorizations > User Roles"
  5. Button: Add
  6. Set a role in the field "Role" or select a "Role" by search
  7. Click button "Assign"
  8. Result: An Error Message is displayed: "The check did not produce any hits. Call the search."

Cause

The message is caused by a missing authorization (authorization object 'S_USER_AGR') to display a "userrole".

CL_ACO_P_ROLES
SEARCH_ROLES

AUTHORITY-CHECK OBJECT 'S_USER_AGR'
             ID 'ACT_GROUP' FIELD <ls_actgroup>-agr_name
             ID 'ACTVT' FIELD '03'.
    CHECK NOT sy-subrc IS INITIAL.
    DELETE et_single_actgroups.

Resolution

The authorization object 'S_USER_AGR' with activity (ACTVT) "Display" (03) must be assigned to the user by a pfcg-role.

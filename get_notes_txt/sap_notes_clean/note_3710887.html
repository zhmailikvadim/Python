Symptom

You run the Programa de cálculo de nómina Argentina (HARCALC0) report for an employee in the following scenario:

  • The employee has an Sueldo anual complementario (AGUI) off-cycle payroll already calculated.
  • The employee is fired after this run.
  • The F001 payroll runs, and the employee has payroll adjustments that make a SAC refund necessary.
  • The Tope elegibilidad exención SAC (LESAC) constant is configured as 0.00 in the Constantes de nómina (V_T511P) table view.

In the scenario described above, the system incorrectly generates a negative the SAC 1o semestre exento (/4S3) wage type or the SAC 2o semestre exento (/4S5) wage type (depending on the SAC period), when it should generate this negative amount on the SAC 1o semestre gravado (/4S2) wage type or SAC 2o semestre gravado (/4S4) wage type.

Reason and Prerequisites

Reason

This issue happens because the system always generates one of the exempted wage types (/4S3 or /4S5) when the calculated SAC amount is less than the LESAC constant amount.

Prerequisites

The following Support Package levels are the minimum prerequisites:

Release Support Package
S4HCM 102 HR SP 00
S4HCM 101 HR SP 05
S4HCM 100 HR SP 18
SAP_HR 608 HR SP C9
SAP_HR 604 (EhP6 e EhP7) HR SP K1

For more information, access the SAP ERP Human Capital Management - Support Package Schedules.

When you install an SAP Note, some prerequisites might be required. The prerequisites are SAP Notes that have already been released and that change the same context of the current one. Then, even if these prerequisites are not necessarily related to the symptom of this SAP Note, you must implement them due to technical reasons.

Solution

From now on, the LESAC constant is configured with the negative amount of -999,999,999,999.99, preventing the system from generating the exempt wage types.

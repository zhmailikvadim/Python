Symptom

If there is a mutiple bytes character (eg, Japanese Character, Chinese Character) in the Data Services query, some additional configuration is needed, otherwise these characters may not be parsed correctly, error: "Regular expression is invalid: quantifier does not follow a repeatable item".

Other Terms

 Environment:

  • SAP Data Services 4.3 SP3 Patch 10

Steps:

  1. Design a job with MongoAdapter as source.
  2. Add a XML Map, in the query field of XML map, add a string contains mutiple-bytes character, save and execute the job.

Reason and Prerequisites

The root cause of this issue is that in some of the system, the multiple bytes character cannot be parsed correctly with default encoding (code page), as the result, the code page should be changed accordingly.

Solution

Please follow the steps below to customize the code page in the Data Services.

  1. Open the locale Selector, change the Default locale to the code page accordingly, alternatively, the UTF-8 may be chosen. 
  2. Save and restart the 'SAP Data Services' in the services to apply this change.

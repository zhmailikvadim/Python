Symptom

You are using Change Request Management (ChaRM) in SAP Solution Manager 7.2. After you performed cycle switch for your release cycle, you may find go-live date of the successor release cycle is current date instead of the date you planned in release planning management.

Reason and Prerequisites

This is due to a program error.

Solution

Implement the attached corrections into your SAP Solution Manager system by using the Note Assistant (transaction SNOTE), or import the relevant support package stack.

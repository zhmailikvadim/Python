Symptom

While login into Service Layer, an error occurs: 401, Internal error: Get access token error,{\"error\":\"unknown_error\",\"error_description\":\"For more on this error consult the server log.\"}.

Reproducing the issue

Prerequisites:

  • Both Active Directory Domain Services and SAP Business One Authentication Server are enabled under Identity Providers tab in System Landscape Directory (SLD).
  • In SLD, navigate to the Users tab and create an SAP Business One Authentication Server user as 'manager' and set its Password as 'Welcome1!'.
  • Login into SAP Business One client with 'manager' user. The system asks you to change the password for 'manager'. Set the new Password having special characters such as '%iTest1'.

Scenario:

Login into Service Layer with 'manager' user as below:

POST /b1s/v1/Login

{
    "CompanyDB": "405830",
    "Password": "%iTest1",
    "UserName": "manager"
}

Expected behavior:

Login is successful.

Actual behavior:

An error occurs:

{
    "error": {
        "code": 401,
        "message": {
            "lang": "en-us",
            "value": "Internal error: Get access token error,{\"error\":\"unknown_error\",\"error_description\":\"For more on this error consult the server log.\"}."
        }
    }
}

Solution

SAP intends to provide a feature package or patch to solve the symptom that is described in this SAP Note. The section References below lists the related feature package or patch once they become available. Be aware that References can only be confirmed at feature package or patch release date. SAP delivers feature packages or patches only for selected releases at its own discretion, based on the business impact and the complexity of implementation.

Workaround:

Set the password without special characters.

Other terms

unknown_error, error_description, Get access token error

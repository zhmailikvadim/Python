Symptom

You are running the payroll for an employee subject to Employee Occupation Tax (Tax Type 51) and/or Employer Occupation Tax (Tax Type 54) for Denver (Tax Authority CO01).
Even though these Tax Types are included on the Tax Model, the taxes are not calculated. You may also see the following BSI Messages on the payroll log:

2013
Formula for DENVER EMPLOYEE OCCUPATION is a non resident only formula while resident code interface parameter indicates a value of resident. No tax result was generated for this tax.

2013
Formula for DENVER EMPLOYER OCCUPATION is a non resident only formula while resident code interface parameter indicates a value of resident. No tax result was generated for this tax.

Environment

  • HR Renewal
  • SAP S/4HANA
  • SAP ERP
  • SAP SuccessFactors Employee Central Payroll

Reproducing the Issue

1) Run the payroll for an employee subject to Tax Types 51 and 54 for CO01.
2) Check payroll function USTAX.
3) The taxes won't be calculated (/451 and /454 for CO01 will be zero).

Resolution

Tax Types 51 and 54 for Denver are non-resident only formula.
This means that they should be customized on the Tax Model for Work, only.

If they are customized on the Residence and/or Unemployment Tax Combos, it should be the reason for the issue.
Please make sure that these Tax Types are included only on the Work Tax Combo, as the below example shows.

Further guidance on the Tax Model customizing can also be found on the following Support Content: https://help.sap.com/docs/SUPPORT_CONTENT/erphcm/3354684289.html?locale=en-US&version=LATEST

    

Example:

T5UTE
CO01    010    01.01.1800    31.12.9999

T5UTM
10    W    U1    1    01.01.1800    31.12.9999    9

T5UTY
9    051    01.01.1800    31.12.9999    Y
9    054    01.01.1800    31.12.9999    Y

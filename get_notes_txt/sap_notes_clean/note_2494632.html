Symptom

You use the integration between FS-PM and the SAP Claims Management system FS-CM.

In FS-CM, you call transaction ICLEMC01 for creating a claim bundle.

However, the system does not display all coverages that exist in FS-PM.

Other Terms

Basis line of business
Claim bundle
/PM0/ABT_CM_FIND_COVERAGES
/PM0/ABT_CM_POLICY_DETAILS

Reason and Prerequisites

Enhancement

Solution

For an advance correction, implement the source code corrections.

When you execute transaction ICLEMC01, the FS-CM system calls both of the FS-PM function modules /PM0/ABT_CM_POLICY_DETAILS and /PM0/ABT_CM_FIND_COVERAGES for retrieving information.

With this SAP Note, the optional importing parameter IV_EFFECTIVE_DT (effective date) is added to these two FS-PM function modules.

If you fill this parameter in your customer-specific implementation, the system executes this information retrieval for the transferred effective date.

Otherwise, it executes it for the current date (default value = SY-DATUM).

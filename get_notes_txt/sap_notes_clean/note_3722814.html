Symptom

The Assign Stock Movement Buckets to Goods Movement Type and Reason activity allows you to configure the INRE (Rework) bucket value status and corresponding movement types.

Other Terms

ISLS, Intelligent Clinical Supply Management, INRE medkit Status

Reason and Prerequisites

You use SAP Intelligent Clinical Supply Management.

Changes and Validations for "Assign Stock Movement Buckets to Goods Movement Type and Reason":

Path: Life Sciences > SAP Intelligent Clinical Supply Management à Distribution > Medkit Status Mapping  > Assign Stock Movement Buckets to Goods Movement Type and Reason

Required Changes

  1. Remove INRE from Bucket Value Dropdown - INRE (In rework) isn't a bucket value in the Reconcile app. Remove it from the bucket value dropdown list in the SPRO node "Assign Stock Movement Buckets to Goods Movement Type and Reason."
  2. Restrict Movement Types 541 and 542 - Movement types 541 and 542, and their equivalents in the rework process, aren't dependent on this table and have no relevance here. If you enter these movement types, the system raises an error and doesn't allow the entry.
  3. Restrict Bucket Value Combinations for Movement Types 543 and 261 - If you combine movement types 543, 261, or their equivalents with other bucket values, the system raises an error and doesn't allow the entry. These movement types can only be used with the "Destroyed" status.

Solution

Implement the attached correction.

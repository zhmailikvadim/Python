Symptom

  • Pricing conditions are not available when using custom condition tables for PIRs based on Material Group.
  • Custom condition tables were created by copying standard SAP tables 025 and 028, and added to a custom access sequence replicating standard sequence 0002.
  • When creating or modifying PIRs in transactions ME11 or ME12, the system does not propose pricing conditions in the conditions screen, nor does it allow entering or editing a price.
  • Tests indicate that standard tables 025 and 028 work as expected, suggesting the system is ignoring the custom tables despite identical configuration.

Environment

  • Materials Management (MM)
  • SAP ERP Central Component
  • SAP ERP
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA

Reproducing the Issue

  1. Create or modify a PIR in transaction ME11 or ME12 using custom condition tables based on Material Group.
  2. Observe that the system does not propose any pricing conditions in the conditions screen.
  3. Attempt to enter or edit a price, which is not allowed.
  4. Add standard SAP tables 025 and 028 to the same access sequence and observe that pricing conditions work as expected.

Cause

Typically, such issues are related to Purchasing-specific configuration or field control rather than the access sequence itself. 

Resolution

  • Custom condition tables do work for PIRs, including those based on Material Group. Please ensure the following:  

- Condition Type Settings:  
  - Condition type must be PIR-relevant.  
  - Verify the condition type (e.g., PB00-like):  
    - Condition class = Price.  
    - Calculation type = Quantity-based.  
    - Allowed for Purchasing.  
    - No "Manual entries not allowed."  
    - Not statistical.  
    - Confirm inclusion in the Purchasing pricing procedure (e.g., RM0000 or your Z-schema).  

- Access Sequence Checks:  
  - In M/06, ensure the access sequence is assigned to the condition type.  
  - Confirm custom table access is:  
    - Not marked "Exclusive" (unless intended).  
    - Has the correct Requirement routine (usually blank).  
    - Uses Application = M.  
  - Test placing your custom table before 025/028, as standard tables found first will prevent the system from reaching the custom table.  

- SE11 Verification:  
  - Check if the table exists physically and confirm custom tables are generated and active in SE11.  
  - Ensure fields are in the correct order.  
  - Verify client dependency (`MANDT` present).  

  • Additionally, try creating a PIR without material, using only vendor + material group. If conditions are not proposed, table access may not be firing.  

Other checks to consider:  
- Ensure the field catalog is extended for Purchasing (Application M).  
- Verify custom tables are generated and transported correctly.  
- Review access sequence order and exclusive flag settings.  
- Confirm the condition type is fully PIR-compatible.

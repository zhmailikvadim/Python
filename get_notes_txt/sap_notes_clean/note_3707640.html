Symptom

PDF files from ELSTER generating incorrect numbers.

Image/data in this SAP Note is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental. 

Reproducing the issue

Prerequisites:

  1. You are working in the localization for Germany (DE).
  2. The ELSTER add-on is installed and enabled.
  3. You are using latest version of ELSTER SAP Business One Addon, Patch 11.
  4. The following setting is set in Windows:  
Go to Control Panel → Region → Additional Settings, open Customize Format.
 
 
  
  • In the region definition Decimal symbol is “,”
  • Digit grouping symbol is “.”
Scenario:
  1. Go to Financials → Financial Reports → Accounting → Tax → Tax Report.
  2. Create a Tax report for a month.
  3. Click the Elster button to open the Elster window.
  4. In the Retrieve tab, click the Retrieve Data button. You received a message: The data has been successfully retrieved; it can be transmitted.
  5. Click the Print Report button.

Expected behavior:

The amounts printed correctly.

Actual behavior:

The amounts printed are incorrect, moved 2 decimal places. 

Note! This scenario is not reproduced when the following definition is set in Control Panel → Region → Additional Settings, open Customize Format.

  • In the region definition Decimal symbol is “.”
  • Digit grouping symbol is “,”

Warning: changes in regional setting affect the behavior of SAP and not SAP application running in the system, any modifications to these settings are carried out under the supervision of system administration to avoid unintended consequences or disruptions.

Symptom

You use transaction SU53 to analyze failed authorization checks of a user.
Even though you are sure that there must have been failed authorization checks, transaction SU53 does not display anything.

Reason and Prerequisites

The failed authorization checks of all users and all clients of a system are stored in the same ring buffer per server instance.
If the maximum size of the ring buffer is reached, old failed authorization checks are overwritten.
Since the same ring buffer is used for all clients, a large number of failed authorization checks in other clients of the same server instance may have overwritten failed authorization checks of their own client.

Solution

Implement the attached correction instructions or import the specified support package.

Remark about the solution:

This correction provides a new report, RSUSR_SU53_STATISTICS. In the releases in which transaction SEC_SUPPORT exists, it is integrated into the subfolder 'Authorization Management'.

You can use the report to determine the buffer statistics of the SU53 buffer on the current server instance. The ring buffer is instance-specific, which is why the statistics are relevant only for the instance on which SU53 was also analyzed.
These are:

  • Instance Name

  • Maximum Number of Entries

  • Number of Entries Currently Used

  • First entry (according to time zone: UTC)

  • First entry (according to user's time zone)

  • Time of the analysis of the buffer (according to the time zone of the user)

  • The following values are output for each client:
    • Client
    • Number of entries in this client
    • Time stamp of the first entry in this client
    • Time stamp of the last entry in this client

Symptom

Cache Region Optimization and tuning with CML libraries in NetWeaver Application Server Java

Other Terms

CML, CACHE REGION, TTL

Solution

The goal is to gain deeper insights that will allow us to recommend more effective tuning measures for improving overall system performance.
We have implemented the following logs and tracing enhancements in the SSC application:

  • Time-to-Live (TTL) Control for Cache
    A new cache attribute has been added to control the TTL value using the engine parameter timeToLiveForKBInCacheInMin (default: 30 minutes).
  • Duration of Database Load
    A new cache attribute now captures and logs the time taken to load knowledge bases (KBs) from the database.
  • Source Identification for KB Loading
    Added an attribute to identify whether a KB was loaded by a scheduled job or through the standard processing flow.
  • Optional KB Compression in Cache
    Introduced a new engine parameter enableKBStoreInCacheWithCompression (default: false). When enabled, KBs will be stored in the cache in a compressed format. For stable operation, the JVM -Xss parameter must be set to a minimum of 4M to avoid StackOverflowError during compression/decompression of large KBs. We’ve set the default to false for now; the plan is to first validate behavior using the existing approach before evaluating performance with compression enabled.
  • Cache Invalidation Listener
    A new cache invalidation listener has been added to log cache activity events.
    To enable detailed logging, set com.sap.custdev.projects.fbs.slc.cml.util.CacheRegionListener to DEBUG level.

Additionally, com.sap.sce.front.base.Database should be enabled in DEBUG mode to provide insights into various cache-related database operations.

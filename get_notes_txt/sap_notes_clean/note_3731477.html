Symptom

Refreshing a CDS query in AO at the first time, data displayed. However, while recalling variables screen, "No applicable data found" displays for the same CDS query although it should have data. The same behavior exists in Transaction Code RSRT while recalling vairable screen.

Environment

SAP Analysis for Microsoft Office 2.8

Reproducing the Issue

  • Insert a CDS query in AO and refresh the query successfully at the fist time.
  • After successfully refresh, click  "Prompts" to recall the variable screen again, no value change and click "OK" tab.
  • The message "No applicable data found" displays, instead of the expected data result set displays.
  • The same behavior exists in Transaction Code RSRT while recalling variable screen for the same CDS query.

Cause

Program error.

Resolution

Implement the SAP Note 3729695 - Wrong Query Result After Recall Variable Screen when Using Hierarchy with Multiple Key Field Header.

Keywords

Workbook, No data, reset datasource, see data, data is present in backend, resubmit variable values, different, query result, CDS view, Recall variable screen

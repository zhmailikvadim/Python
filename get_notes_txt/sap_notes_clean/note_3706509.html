Symptom

  • An error occurs when attempting to add a new Rater Category in 360 Form Template.
  • The system does not allow saving changes to the template when adding the new category.
  • Receive Error - 'Changes failed. There was an error while performing this operation.'

Environment

SAP SuccessFactors 360 Multi-Rater

Reproducing the Issue

  1. Go to Manage Templates > 360 Multi-Rater
  2. Choose your template
  3. Go to Rater List Section
  4. Attempt to add a new Rater Category
  5. Save changes to the template.
  6. Observe the error preventing the changes from being saved. - 'Changes failed. There was an error while performing this operation.'

Cause

The issue is caused by the presence of msgKeys in the XML configuration for Rater Categories, which leads to the system expecting msgKeys for Categories and throwing errors when they are not provided.

Resolution

  1. Add the Rater Category via the XML configuration and ensure to add a msgKey for the new Rater Category.
    OR
  2. Remove the msgKeys from the XML configuration to allow adding the Category via Manage Templates without errors.

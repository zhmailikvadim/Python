Symptom

Deprovisioning of roles via User Access Review(UAR) is not working for any backend systems.

Environment

SAP GRC Access Control 10.0

  • GRCFND_A V1000 SP16
  • User Access Review (UAR)
  • Access Request Administration (ARQ)

Reproducing the Issue

1. Generate UAR Requests with the UAR Generation Job for the respective connectors.

2. Reviewer performes UAR and marks assignments to be removed.

3. Removals fail in the backend systems.

Cause

Role removal was not handled correctly.

Resolution

Please have your ABAPer apply these two corrections related to role removal post SP16.

2098610 - UAR: Roles marked for removal in UAR request not deprovisioned at the end of request

2126311 - UAM: Inconsistent provisioning if removal and assignment of business roles having common technical role in same request

2057482 - Actions taken in UAR request are not getting saved for roles having same name and different system

That should resolve the issue. If you still face an issue, kindly create an case under the GRC-SAC-UAR Component.

See Also

1732931 - Synchronization jobs that are needed to run to generate UAR requests

Symptom

The payment lot key KEYZ1 is not filled when you use the function module FKK_CD_PAYMENT_CREATE to create a payment lot.

Other Terms

Cash desk, FKK_CD_PAYMENT_CREATE

Reason and Prerequisites

Programm error

The system enhances table DFKKOPKC with the new structure FKKOPKC_POC, which contains the lot name KEYZ1. The MOVE-CORRESPONDING statement clears the content of KEYZ1.

Solution

Import the latest SP or implement the attached correction instructions.

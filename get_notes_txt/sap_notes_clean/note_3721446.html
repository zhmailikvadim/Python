Symptom

  • Users are unable to log in to SAP Emarsys using the initial owner (local admin) account.
  • The login page does not redirect to SAP Identity Authentication Service (SAP IAS), and authentication fails before any SSO redirection occurs.
  • Error message displayed: "Sorry, we could not authenticate you. Try again."
  • Users encounter issues with email verification, preventing access to certain features such as "Management" and "Single Sign-On Setup".
  • The SSO icon is not visible after logging into the portal.

Environment

SAP Emarsys Customer Engagement

Reproducing the Issue

  1. Attempt to access the SAP Emarsys login page using the URL provided for SSO.
  2. Log in using the initial owner (local admin) account credentials.
  3. Observe that the login page does not redirect to SAP IAS and authentication fails.
  4. Attempt to log in using the base URL (https://suite.emarsys.net/) with the activation credentials.
  5. Verify email address using the activation link sent via email.
  6. Observe that email verification fails, and certain features remain inaccessible.

Cause

  • The system attempts to authenticate via SSO before the configuration is complete.
  • The SAP Emarsys tenant is configured to authenticate with an IAS tenant that does not contain the corporate users' identities.

Resolution

  1. Ensure that the SAP IAS application is correctly configured to authenticate with the appropriate IAS tenant.
  2. Verify that the IAS tenant contains the corporate users' identities required for authentication.
  3. Recreate the SSO setup in IAS following the steps outlined in the SAP Help Portal documentation.

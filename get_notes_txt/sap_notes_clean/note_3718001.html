Symptom

Ask HR Agent's actual name is displayed in AskHR for some users in Automated Email Notifications despite the Service Center Configuration object being set to hide the agent name and necessary permissions being granted.

Environment

SAP SuccessFactors HCM Suite

  • Ask HR - Employee Central Service Center (ECSC)

Reproducing the Issue

  1. Log in to SuccessFactors → AskHR.
  2. Navigate to My Tickets and open any ticket that includes email interactions.
  3. Go to the Conversation tab → Note that the agent name appears correctly as "HR Agent."
  4. Open Email Conversation.
  5. Observe that the agent's actual name is displayed instead of "HR Agent."

Cause

The engineering team found out that this is a bug in the system.

  • When emails are sent via automated notifications (as opposed to manually typed by an agent), the system uses a different field (FromPartyName) that is not being masked. The masking logic only covers manually-sent emails.

Resolution

  1. Confirmed as a bug in the system.
  2. Engineering team has created ticket SFSWZ-9026 to address the issue.
  3. Fix will be included in release 2605:
  • Preview: April 13, 2026
  • Production: May 15, 2026

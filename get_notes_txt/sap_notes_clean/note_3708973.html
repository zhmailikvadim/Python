Symptom

When executing Planned Replenishment in transaction /SCWM/REPL with no checkboxes flagged on the input screen, system returns warning:

"No replenishment to execute
Message no. /SCWM/REPL009"

If user flags checkbox "Show Log Immediately", the actual error is displayed:

"Planned Replenishment is not permitted in storage type &1
Message no. /SCWM/REPL004"

Environment

  • SAP Extended Warehouse Management (SCM-EWM);
  • SAP Extended Warehouse Management Warehouse Order Processing (SCM-EWM-WOP);
  • SAP S/4HANA.

Reproducing the Issue

  1. Open transaction /SCWM/REPL;
  2. Fill in the warehouse number and all filtering criteria desired;
  3. Select Show Log Immediately checkbox
  4. Execute
  5. Error /SCWM/REPL004 is shown

Cause

Missing customizing.

Resolution

The Storage Type &1 does not contain an entry on customizing Activate Replenishment Strategies in Storage Types for 'Planned Replenishment'.

Customizing path: SAP Customizing Implementation Guide -> SCM Extended Warehouse Management -> Extended Warehouse Management -> Internal Warehouse Processes -> Replenishment Control -> Activate Replenishment Strategies in Storage Types

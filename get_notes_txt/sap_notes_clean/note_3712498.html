Symptom

You declare your import declaration in SAP GTS to customs system DMS.

You have your goods value customized to be declared in original currency. You declare your goods value in original currency, other then customs currency DKK. You are not using fixed exchange rate, but the <CurrencyExchange> element is created in the XML message. As a result, you receive error CWM10003, pointing to customs value and saying, that the currency could not be converted.

Other Terms

Denmark, XML, /SAPSLL/LLEG_ECC_DATAF56, customsValue, amount, amountForVat, cvrtf, cvrtq, txcur, hd_aar, conversion, exchange rate

Reason and Prerequisites

The problem is caused by program error.

Element <CurrencyExchange> contains the exchange rate that is set in advance for the price of the goods, in a contract between the parties concerned.

By default, this element is not created in SAP GTS. To create this element, you have to maintain following entries for Goods Value (invoice line) on Logistics Costs tab on the header.

'Indicator for Fixed Exchange Rate' (CVRTF) must be set to 'Fixed Exchange Rate' (A) or 'Fixed Exchange Rate (IATA Rate)' (B). If you cannot see these options, maintain them in the /SAPSLL/CVRTF domain.

'Indirect Quoted Exchange Rate' (CVRTQ) must contain the exchange rate.

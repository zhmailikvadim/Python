Symptom

  • App could not be opened due to WAF policy restrictions.
  • Error message: "App could not be opened because the SAP UI5 component of the application could not be loaded."
  • Manifest file is not accessible via the URL containing the parameter "~20260116062015.1739900~".
  • Removing the parameter manually from the URL allows access to the manifest file.

Environment

SAP S/4Hana

Reproducing the Issue

  1. Login to the SAP Fiori Launchpad.
  2. Attempt to open any Fiori app.
  3. Observe the error message indicating that the SAP UI5 component of the application could not be loaded.
  4. Check the console logs and identify that the manifest file is not accessible due to the parameter "~20260116062015.1739900~" in the URL.
  5. Remove the parameter manually from the URL and verify that the link becomes accessible.

Cause

WAF policy restrictions block URLs containing specific parameters, such as "~20260116062015.1739900~", which are cache tokens used by SAP to improve application performance.

Resolution

  1. Adjust the WAF policy to allow URLs containing cache tokens, such as "~20260116062015.1739900~".
  2. Ensure that the WAF rules do not block URLs with special characters in file names, as these are essential for the functionality of SAP Fiori applications.
  3. Refer to SAP documentation for more information on cache token-based URLs and their purpose in improving performance:

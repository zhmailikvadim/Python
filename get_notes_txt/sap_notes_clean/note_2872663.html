Symptom

Starting  from January 1, 2020, a change is valid for the tax advance calculation which is regulated by the article 32 of the legal act of personal income tax, DU1991.0350.

The change applies to those calculations of payroll, and to those months, in which the employee's yearly tax progression counter reaches the threshold of 85 528 PLN. Before the change, until end of year 2019, the lower tax rate of 17% applies to all incomes of such a month. After the change, the lower tax rate 17% applies only to the part of the income under the threshold, and the higher tax rate 32% applies to the part of the income above the threshold.

Other Terms

Payroll schema PLTX, payroll function PLTR, payroll rules PLXA, PLX0, PLXT, wage types /531, /10X, /10Y, table field T7PL81-TXIND, tax progression table T7PL02.

Reason and Prerequisites

Legal change

Solution

The legal change implementation described in this SAP Note follows the assumption that the tax advance wage type /531 can still be a single record of the payroll result table RT, after it is calculated using both tax rates, 17% and 32%. In effect, in the single payroll result for which both tax rates have to be applied, the tax rate shown in the rate field of the wage type /531 is given a value between 17.00 and 32.00, rounded to 2 decimal digits. A more accurate, but still rounded, effective rate is also recorded in the payroll result table ST.

The legal change applies only to those wage types that have the tax attribute T7PL81-TXIND=1, assigned statically with the processing class PC61 or only dynamically in the tax split table ST, in the field ST-TXIND.

The implementation requires the following changes:

  1. the ABAP code was changed in the payroll calculation function PLTR, which has the main purpose to select the right tax rate from the tax progression table T7PL02 for those wage types that require progressive tax (T7PL81-TXIND=1),
     
  2. in the tax subschema PLTX, the function PLTR is now called later, almost just before the payroll rule PLXA, which calculates the tax advance wage type /531,
     
  3. in the tax subschema PLTX, a new payroll rule PLX0 was inserted between the function PLTR and before the rule PLXA, to process the tax rate override wage types of PC62=6 and PC62=7, which were earlier processed by the rule PLXT,
     
  4. the payroll rule PLXA was updated for accuracy of the required effective tax rate.


All the changes of payroll calculation rules and schemas are described in the attached document 2872663_schema_changes.pdf.

Apply the correction by importing the Support Package specified for your release, or by using the Note Assistant (SNOTE). If you apply the changes using SNOTE, perform manually the changes described in the attached document 2872663_schema_changes.pdf. If you find any of the payroll rules locked for editing, consider using the tool program RPUCTF00.

Symptom

When generating rules under Function maintenance at NWBC, error message below is triggered:

  • No risk found for selected functions - Error message 020

Environment

  • SAP Access Control 12.0
  • SAP Access Control 12.0 for SAP S/4HANA

Reproducing the Issue

  1. Log in NWBC
  2. Setup
  3. Functions
  4. Enter on desired function that has special character
  5. Click on Generate Rules
  6. Error is triggered

Cause

Some special characters are invalid and not supported by the application. The naming convention does not have a validation during manual mass upload of functions or Function copy as it uses the same methods and programs from Risk IDs, which accepts some characters, but the rule remains valid for Function IDs.

Resolution

Function IDs only accepts an 8 character long name with the valid characters below:

-/ABCDEFGHIJKLMNOPQRSTUVWXYZ_0123456789

See Also

The naming convention is also valid during direct function ID creating, for further information refer to documents below:

  • SAP Help Portal - Access Control: Creating a Function
  • 3724392 - Error "Invalid characters in the Function ID" during Function creation - GRC

Keywords

sap access control GRC, message no 122, message no 020, invalid characters in the Function ID, HAS_VALID_ID_VALUE, CL_GRAC_SOD_UTILS

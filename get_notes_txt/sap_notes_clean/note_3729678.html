Symptom

Physical inventory item navigation does not follow sort order when counting single item or multiple items sequentially

Other Terms

  • SAP Service and Asset Manager
  • SAM
  • SSAM
  • ICMTANGOAMF10-43323

Reason and Prerequisites

Reason:

Navigation using Previous/Next toolbar buttons becomes incorrect after an item is counted, leading to improper button enablement or navigation to the wrong item and breaking the intended sequence. Additionally, when using the Next button within the edit/count screen to process multiple items sequentially, the navigation does not respect the list's current sort order and filter conditions.

Prerequisites:

  • Apply SAP Note 3721052 
  • BAS or Visual Studio Code with SAM MDK project
  • `patch` command (available on BAS and macOS by default) via terminal or other patching utility
  • Read the guidelines on how to apply the patch and read the appendix section if you receive any error

Solution

Code is modified to detect and respect the list's current sort order and filter conditions during item navigation, ensuring Previous/Next buttons navigate correctly and are properly enabled/disabled based on the items position in sequence regardless of whether theuser is counting one item or multiple items at a time. When counting physical inventory items individually instead if using 'Count All', after all items are counted, both counted and uncounted items are filters are automatically selected to display all items together.

To apply the patch, download the attached 3729678.patch file and save it in a known location, preferably adjacent to your SAM.mdkproject.

If you are on Windows, see SAP Note 3533879 for how to install a .patch file in Windows. If you are on a Mac, follow the steps below:

  1. Open a terminal in the root of your SAM metadata (i.e. SAM.mdkproject/metadata or SAM.mdkproject/metadata/base.definitions)
  2. Run the terminal command `patch --no-backup-if-mismatch -p4 < path/to/3729678.patch`. 
  3. Verify in the terminal window that the following files have been modified and listed as patched, as indicated by the console output:
  • patching 'Pages/Inventory/PhysicalInventory/PhysicalInventoryItemDetails.page'
  • patching 'Rules/Inventory/Item/ItemPageOnLoaded.js'
  • patching 'Rules/Inventory/Item/ItemPageOnReturning.js'
  • patching 'Rules/Inventory/Item/ItemPreviousNext.js'
  • patching 'Rules/Inventory/Item/ItemPreviousNextEnabled.js'
  • patching 'Rules/Inventory/PhysicalInventory/Count/PhysicalInventoryDocItemUpdatePost.js'
  • patching 'Rules/Inventory/PhysicalInventory/PhysicalInventoryCountNavWrapper.js'
  • patching 'Rules/Inventory/PhysicalInventory/PhysicalInventoryItemsOnReturning.js'

Symptom

  • Searching for Recipes or Specifications on the PLM Web UI returns ASSERTION_FAILED dump in program /PLMB/CL_RCP_TOOLS
  • Searching for Specifications or Recipes via transaction CG02 returns the ASSERTION_FAILED dump

Environment

  • SAP ERP Recipe Development on PLM Web UI (PLM-WUI-RCP)
  • SAP R/3
  • SAP R/3 Enterprise
  • SAP ERP Central Component
  • SAP ERP
  • SAP Enhancement package for SAP ERP
  • SAP Enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA, on-premise

Reproducing the Issue

  1. Start transaction CG02
  2. Try searching for Recipes or Specifications
  3. Dump occurs with the following details

"ASSERTION_FAILED"
"/PLMB/CL_RCP_TOOLS============CP" bzw. /PLMB/CL_RCP_TOOLS============CM01J
"GET_RCP_CATEGORY"

at line:
ASSERT sy-subrc IS INITIAL

Cause

  • The Recipe Type of the Recipe or Specification you are searching for is already deleted from the system
  • System is searching for Specifications or Recipes that have their Recipe Type deleted creates inconsistency

Resolution

To correct the inconsistency, possible solutions include:

  1. Create the recipe type again.
  2. Delete the recipes of this recipe type. Therefore you need to mark these recipes for deletion, see field DEL_IND of table /PLMB/RCP_RECIPE.
  3. Mark the recipes of this recipe type for archiving, see field ARCHIVE_IND, and for deletion, see field DEL_IND, of table /PLMB/RCP_RECIPE, and archive the recipes.

To set the mentioned indicator fields in table /PLMB/RCP_RECIPE you may use transaction SE16 or may create a small ABAP report that updates these fields.

Symptom

The "First Acquired On" field is missing during the asset master data migration process.

Environment

SAP S/4HANA Cloud Public Edition.

Reproducing the Issue

  1. Perform asset migration in SAP S/4HANA Cloud Public Edition.
  2. Observe that the "First Acquired On" field is missing in the migration template.

Cause

The system logic defaults the "First Acquired On" date to align with the asset capitalization date entered during migration.

Resolution

The "First Acquired On" date defaults to the same value as the entered capitalization date. If different dates are required for these two fields:

  • Enter the desired "First Acquired On" date as the asset capitalization date during migration.
  • After migration, manually adjust the capitalization date using the app Manage Legacy Assets, as this field can be modified.

Keywords

first acquired on date, asset migration, capitalization date, Manage Legacy Assets

Symptom

  • Enterprise search help is not displayed for quality info records in transaction QI03.
  • The search help points to the object 'QUALITY_INFO_RECORD', which appears to not exist in the system.
  • The search for quality info records worked under TREX but does not function after switching to HANA Embedded Enterprise Search.

Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.

Environment

  • SAP R/3
  • SAP R/3 Enterprise
  • SAP ERP Central Component
  • SAP ERP
  • SAP Enhancement package for SAP ERP
  • SAP Enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA, on-premise
  • SAP S/4HANA Cloud Private Edition

Reproducing the Issue

  1. Navigate to transaction QI03.
  2. Attempt to open the search help for quality info records.
  3. Observe that no enterprise search help is available.

Cause

  • Although a search model for QUALITY_INFO_RECORDS exists in the system, it is delivered only as a classic (TREX-based) Enterprise Search model. SAP does not deliver a CDS-based / HANA Embedded Search model for Quality Info Records.
    As a result, Quality Info Records are not Enterprise Search–enabled on HANA and remain dependent on TREX.

Resolution

Quality Info Records are not supported by HANA Embedded Enterprise Search. Their search capability in ECC/S/4HANA depends on the classic TREX-based Enterprise Search, and therefore search will not function once TREX is switched off.

See Also

  • Refer to SAP Note 2793450 - Enterprise Search Models in Quality Management: Availability of CDS Based Search Models in SAP S/4HANA 1909.

Symptom

Cloning Git with SAP Business Application Studio (BAS) fails with:

*****************************************************
/bin/bash: warning: setlocale: LC_ALL: cannot change locale (en_US.UTF-8)
* warning: setlocale: LC_ALL: cannot change locale (en_US.UTF-8)
remote: The repository owner has an IP allow list enabled, and <<IP Address>> is not permitted to access this repository.
fatal: unable to access <GitHub URL>The requested URL returned error: 403
*****************************************************

Environment

  • SAP Business Technology Platform (SAP BTP. URL: https://cockpit.btp.cloud.sap) 
  • SAP Business Application Studio (BAS)
  • Cloud Foundry (CF)
  • Git

Reproducing the Issue

You are following the document: SAP Business Application Studio Cloning Repositories and the above stated error occurs.
Steps:

  1. Login to BAS dev space
  2. Try to clone GitHub repository with command "git clone https://github.com/..."

Cause

Problems that can lead to such issues:

  1. Incorrect configuration
  2. IP address allow listing issues

Resolution

Check the below points:

  1. Ensure that the BAS IP addresses are allow listed. Go through the below links with your network administrator and ensure that the IPs are allowed to communicate: 

    SAP Business Application Studio Availability 
    and
    SAP KBA ##2418879 - Information about IP ranges / addresses on BTP

  2. Ensure that the Git repository connection is configured correctly. Depending on the connection mechanism, check the below documentation:
    Connecting to a Corporate Git Repository
    Connecting to a Public Git Repository
    Connecting to a Public Git Server that Requires a Certificate

  3. If the issue persists, run the below commands in the Git bash terminal:

    3.1. export GIT_CURL_VERBOSE=true 

    3.2. git clone https://<remote repository URL>

    3.3. export GIT_CURL_VERBOSE=true 

    3.4. curl https://<remote repository URL>
    The above output should shed more light on the error. For more details, check: SAP KBA ##3650647: How to increase GIT connection traces for troubleshooting purposes: SAP BTP BAS.

  4. If for some reason the BAS IP addresses cannot be allow listed, check other alternatives mentioned here: SAP KBA ##3736726: Alternatives if BAS public IPs cannot be added to GitHub IP allow list.

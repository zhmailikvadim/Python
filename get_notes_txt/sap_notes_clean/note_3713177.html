Symptom

  • When ingesting data from CSV file, CDP creates new profiles instead of matching to the existing profiles based on the matching rule.
  • However, matching rule works as expected when testing with the Playground.

Environment

  • SAP Customer Data Platform
  • Identity Resolution
  • Schedule ingest

Cause

The attributes in the event schema model are case-sensitive.

Resolution

Ensure the attribute names from the event model matches the names in the CSV headers exactly, including the letter casing.

Keywords

match rule, profiles not merged, field name, field names, case-sensitivity,  case sensitivity, case sensitive, duplicate profiles

Symptom

When manually assigning a project task to a sales order item, the following error message is displayed:

"Item XX: Project task ABC not valid for company DEF"

(Where XX stands for the line item number, ABC for the project task ID and DEF for the company ID).

Environment

SAP Business ByDesign

Reproducing the Issue

  1. Navigate to the Sales Orders workcenter.
  2. Go to the Sales Orders view.
  3. Search for the relevant Sales Order ID.
  4. Click on Edit.
  5. Click on View All.
  6. Go to the Items tab.
  7. Manually assign the project task ID ABC to line item XX by typing it into the Project Task field.

The error message is triggered.

Cause

The Project Task ABC does not belong to the same company as the Sales Order's company (Seller party). As there is a mismatch between the assigned companies, the issue occurs.

Resolution

Either change the Sales Order's company (Seller) to the same one associated with the relevant Project, or assign a different project task to the sales order's item.

Once the mismatch is corrected, the project task will be valid.

See Also

Symptom

When processing large files in size through SFTP sender adapter, it is observed that message are stuck in processing state, encounter crashes/OOM.

Environment

  • SAP Cloud Integration
  • SAP Integration Suite

Reproducing the Issue

  1. Create a simple iflow & deploy.
  2. Then process a large file in GBs 
  3. Monitor the impact of message processing on the memory.

Cause

Due to memory and heap constraints, the system encounters an out-of-memory exception when processing large files.

Resolution

Following Guidelines have to be followed for large file processing:

  • While processing large file, it is essential to process one large file at a time with no processing steps. If there is a plan to process multiple files, users have to wait till the processing of one file gets finished. If its observed that file processing takes 5 mins, it is required to ensure that the attempt should be in a way that the files are being processed in the interval of 6 minutes.
  • It is advised to keep the flow as simple as possible and process the large files where there are no other critical tasks are running.
  • It is mandatory to run the tests on lower environments to determine the max size of file that can be processed and use that limit on prod environment.
  • It is mandatory to run the iflow when there is no other iflows running at that particular moment.
  • Be clear about the requirement on how much of the file size will be processed and how many number of files will be processed simultaneously and design iflows accordingly.
    • For ex: when the user receives the file via http or any other input, its important to ensure that only limited amount of large files are getting processed.
      • If the limit for tenant to process the maximum file size is 4 gb, and the input to the flows is 500mb file, throughout the application only 8 files can be processed as there are some kind  of rate limiting mechanism.
  • It is advised to set the limit of the file to be processed if the adapter has the option to limit file size.
    • For ex: In case of sftp sender, if the user knows that the limit is 1024 mb on the tenant, in the conditions tab its suggested to put File Size (in MB) as 1024 ensuring that files greater than 1024 will not be processed.

Once these guidelines are followed & if the issue still persists, open a support case in the component LOD-HCI-PI-CON-FTP for further analysis.

Symptom

  • An error message "Service organization &1 is not available" is triggered correctly when a non-existent service organization number is entered.
  • However, the error message persists even after entering a valid service organization number.
  • The workaround to this issue is to delete all items and enter them again, which is time-consuming.

Environment

SAP S/4HANA

Reproducing the Issue

  1. Log onto FIORI.
  2. Create a new service order.
  3. Add an item.
  4. Enter a service organization number that does not exist.
  5. Save the changes.
  6. Now, enter a service organization number that does exist.
  7. Save the changes.
  8. The error message still persists even though the service organization number is now correct.

Cause

The issue happens only if you have enabled Enterprise Organizational Model (SAP Customizing Implementation Guide > Service > Basic Functions > Enterprise Organizational Management > Enable Enterprise Organizational Model).

Resolution

Correction Instruction 3554953.

Keywords

Service Organization, CRM_ORGMAN014, CRM_ORGMAN 014, Messages deletion, Enterprise Organizational Model.

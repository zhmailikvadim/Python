Symptom

Unable to maintain Unit of Measure for Variable-Size Item using odata API API_BILL_OF_MATERIAL_SRV. 

Environment

  • SAP ERP Logistics (LO)  
  • SAP S/4 HANA on premise
  • SAP S/4 HANA Cloud Private Edition

Reproducing the Issue

  1. Attempt to maintain VSI Quantity unit using API_BILL_OF_MATERIAL_SRV. 
  2. There's no property to hole the input. 

Cause

Missing functionality. API API_BILL_OF_MATERIAL_SRV updates BOM based on CDS views. The CDS view for BOM item does not provide input for VSI Quantity unit. 

Resolution

If it's necessary to specify VSI Quantity unit, maintain it using function module CSAP_MAT_BOM_MAINTAIN.  

SAP Note 3701655 must be implemented to ensure VSI Quantity unit can be correctly updated. 

3701655 - Variable size item quantities calculated incorrectly when deriving material BOMs via ECTR

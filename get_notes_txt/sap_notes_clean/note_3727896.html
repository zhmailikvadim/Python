Symptom

When executing the following transactions:

  • PC00_M99_PA03_END – Exit payroll

  • PC00_M99_PA03_RELEA – Release payroll

the system displays the error message:

"This action is not possible for status Payroll correction of payroll area <ABKRS>"
Message no. PG555

The error occurs even when attempting to execute the action using a different payroll area.

Reproducing the Issue

Trying to execute one of the following transactions:  PC00_M99_PA03_END,  PC00_M99_PA03_CORR,  PC00_M99_PA03_CHECK,  PC00_M99_PA03_RELEA 

Cause

The error occurs because the requested payroll control record action is not allowed for the current status of the payroll area stored in table T569V (field STATE).

The ABAP logic in program H99CPAYMENU checks the control record status before allowing the transaction to proceed. Depending on the transaction code, only specific statuses are permitted.

Relevant payroll statuses (T569V-STATE):

StateMeaning
0New
1Released for payroll
2Payroll correction
3Exit payroll
4Check payroll results
9Deleted

The code enforces the following rules:

TransactionPurposeAllowed Status
PC00_M99_PA03_RELEARelease payrollOnly if status ≠ 1
PC00_M99_PA03_CHECKCheck resultsOnly if status = 1
PC00_M99_PA03_CORRCorrectionsOnly if status = 1 or 4
PC00_M99_PA03_ENDExit payrollOnly if status = 1 or 4

If the payroll area is currently in Payroll correction (state = 2) or another incompatible state, the program calls the error routine and raises message PG555.

Therefore, the action fails because the payroll control record is in a status that does not permit the requested operation.

Resolution

Verify the current payroll control record status for the payroll area in transaction PA03 or in table T569V.

Ensure that the payroll control record is moved through the correct sequence of statuses before executing the desired transaction.

Typical payroll status flow:

  1. New (0)

  2. Released for payroll (1) → run payroll

  3. Check payroll results (4)

  4. Corrections (1 or 4) if required

  5. Exit payroll (3)

If the payroll area is currently in Payroll correction (2), complete the correction run and move the payroll control record to an appropriate status (for example Check results (4) or Released (1)) before attempting to execute Exit payroll or other actions.

Once the payroll control record is in a compatible status, the transactions PC00_M99_PA03_RELEA or PC00_M99_PA03_END can be executed successfully.

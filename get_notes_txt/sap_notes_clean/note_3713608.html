Symptom

You group a case for which more than 9999 mechanical ventilation hours were entered. Two symptoms can occur:

  • The ENTL message can be sent due to error message NC 476 Value for mechanical ventilation hours is too large: not sent.
  • The grouping result does not correspond to the expected result.

Example:

You have documented 12000 mechanical ventilation hours for a case.

  • In the EDI workbench for the messages for the case, the system issues error message NC 476 Value for mechanical ventilation hours is too large: 12000.
  • The grouper determines the DRG code A07A, but you expect the DRG code A06A.

Other Terms

NPDRG2, grouping, mechanical ventilation hours

Reason and Prerequisites

Reason:

There is a program error and legal requirements.

Currently, in the country version Germany, a maximum length of 4 is provided for the field of the mechanical ventilation hours when transferring the data in the ENTL message in the segment DAU as part of §301. As a result, it is not possible to transfer more than 9999 mechanical ventilation hours.

In the same way, the program only transfers a maximum of four places to the grouper. However, you can enter a five-digit number of mechanical ventilation hours in the Ventilation field (NFAL-RESPI). The program sends only the first four places to the grouper. This can lead to an incorrect result. For example, the grouper only determines the DRG code A06A if more than 1799 mechanical ventilation hours were performed. If there are fewer (for example, 1240), the grouper determines A07A.

It is very unlikely that more than 9999 mechanical ventilation hours will have to be documented for a case.

Prerequisites:

  • You use the country version Germany.
  • More than 9999 mechanical ventilation hours have been entered for the case. Either by manually maintaining the mechanical ventilation hours in the case data or by entering internal procedures for totaling the mechanical ventilation hours (charge type 73).
  • You use the sample function module ISH_OLE2_3MKS_GRP_DRGPEPP_OUT to group a case with the 3M Kopdip Suite from Solventum. This function module determines the mechanical ventilation hours using the form routine fill_gdrg_pepp_input. The field for the mechanical ventilation hours is only defined with four characters in the parameter gdrg_pepp_record (using the structure ish_gdrg_grouper_1-HMV).
    Alternatively, use another function module for the grouping that also transfers only the first four digits of the mechanical ventilation hours to the grouper.

Solution

Implement the attached correction instructions or import the specified support package.

The field HMV of the structure ISH_GDRG_GROUPER_1 is now defined with five characters. This means that the component type is now CHAR5 instead of CHAR4.

As a result, the sample function module ISH_OLE2_3MKS_GRP_DRGPEPP_OUT automatically transfers the mechanical ventilation hours to the grouper with five digits. This enables the grouper to determine the correct DRG code.

It is still not possible to transfer data using §301. Contact the relevant health insurance fund to transfer this case to the fund in a different way. Automatic transfer is not possible.

If you use a different function module for the grouping, check whether only four places are still transferred to the grouper after the implementation of the correction and adjust the function module accordingly to transfer the mechanical ventilation hours with 5 digits.

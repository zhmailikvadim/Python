Symptom

  • SAP Build apps cannot run as manual steps are not provided.
  • The issue matches the scenario described in note 3457196.
  • Services and other content are active in the subaccounts.
  • SAP Build Code is unable to accommodate the existing configuration and services in the subaccounts.

Environment

SAP Build Apps

Reproducing the Issue

  1. Attempt to run SAP Build apps in the Global Account.

Cause

The issue arises due to the absence of documentation for manual setup of SAP Build apps.

Resolution

  1. Refer to the SAP Manual Steps documented at [Manual Configuration](https://help.sap.com/docs/build-apps/service-guide/manual-configuration)

See Also

Refer to SAP Note 3457196 and 3505324 for additional information.

Keywords

SAP Build Apps, manual steps, subaccounts, note 3457196, note 3505324, configuration, services.

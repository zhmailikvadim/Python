Symptom

On SAPUI5 versions 1.144.0 or higher, Fiori Element V4 apps are consuming the following resources which is not existing, hence the requests failed with 404 not found. If manually change to SAPUI5 version 1.143.2 or lower versions, the issue will not happen (the app will not consume the not existing resources).

  • https://sapui5.hana.ondemand.com/1.144.1/resources/customAction/macrosTable/Action.js
  • https://sapui5.hana.ondemand.com/1.144.1/resources/sap/fe/macros/Action.js

Environment

  • SAPUI5 1.144.0 or higher
  • SAP Build Work Zone

Cause

In Main.view.xml, there's an alias of macrosTable pointing to an unknow file, e.g. xmlns:macrosTable="customAction.macrosTable" 
It worked on previous versions since Fiori Elements was less restrictive on the XML syntax.

Resolution


Please change the alias to point to xmlns:macrosTable="sap.fe.macros.table"

You can find an example in our Fiori Development Portal: https://sapui5.hana.ondemand.com/test-resources/sap/fe/core/fpmExplorer/index.html#/buildingBlocks/table/customTableAction

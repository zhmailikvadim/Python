Symptom

When the DSO sends INVOIC messages to a merged into supplier, the process fails with different error messages depending on the scenario:
- For GUB: Error message /IDEXGE/INVOIC_MSG001 (Product ID could not be determined) is raised.
- For GRDB: Process document DE_INVOIC_DSO (step 30) runs into error with message /IDXGC/IDE_ADD038 (&1 cannot be filled).

This issue is relevant for the market roles: DSO, SUP.

Other Terms

GUB, GRDB, Merged Supplier, INVOIC, 31001, 31011, /IDEXGE/CL_DP_INVOIC_AS_001, INSTANTIATE, /IDEXGE/CL_INVOIC_28B_CL, FILL_SG26_LIN, /IDEXGE/CL_INVOIC_28D_CL, ISU_COMPR_INVOIC_OUT, /IDEXGE/CL_INVOIC_V22, GET_PRODUCT_ID

Reason and Prerequisites

Program error.

Solution

Implement the corrections of this SAP Note or import the relevant support package.

After implementing this SAP Note, the system processes INVOIC messages to merged into suppliers successfully without raising error messages. For GUB scenarios, the system determines the correct product ID. For GRDB scenarios, the process document DE_INVOIC_DSO runs successfully without the field filling error.

You can verify the fix by testing INVOIC sending to a merged supplier and confirming that no error messages are raised during the process.

Symptom

  • Invoices are stuck without IDoc generated for output type.
  • Error during processing: "A message type could not be (uniquely) identified" (Message no. B1128).

Environment

•Sales And Distribution (SD)
•Logistics Execution (LE)
•SAP R/3 •SAP R/3 Enterprise 4.7
•SAP ERP Central Component
•SAP ERP
•SAP Enhancement package for SAP ERP
•SAP Enhancement package for SAP ERP, version for SAP HANA
•SAP S/4HANA
•S/4HANA Cloud Private Edition

Reproducing the Issue

  1. Start transaction vf31.
  2. Select output type to be processed.
  3. Enter the billing document numbers.
  4. Run the processing.

Cause

Duplicate partner profile entries (table EDP12) for affected partner type causes multiple message types to match the same application/output combination, leading to ambiguity and error B1128.

Resolution

  1. Review partner profile/message control entries.
  2. Check table EDP12 (or partner profiles in we20) for affected partner type (for example 'LS'), related to billing output condition.
  3. Ensure that exactly one entry determines a single intended message type (MESTYP) for the above combination.

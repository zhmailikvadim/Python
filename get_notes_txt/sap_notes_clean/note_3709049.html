Symptom

The BAPI FSH_API_PURCHASINGORDER_CHANGE does not validate whether the custom fields contain correct values when a value helper is defined. This behavior differs from the standard transaction ME22N, which performs this validation as expected, following the standard logic.

Environment

  • S/4HANA Retail & Fashion Management (LO-RFM) 
  • SAP ERP Central Component
  • SAP ERP
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA
  • SAP S/4HANA Cloud Private Edition

Reproducing the Issue

  1. Go to transaction SE37
  2. Execute the function module FSH_API_PURCHASINGORDER_CHANGE.
  3. Enter an invalid value in the custom field
  4. Observe the behavior
  5. No validation occurs for this field in the BAPI call.
  6. Compare with ME22N
  7. In transaction ME22N, utilize the same PO that we use in the BAPI call. Enter the same invalid value for the field.
    An error message is raised indicating that the value is not valid, as per standard validation logic.

Cause

Custom field validations in the GUI rely on value help. These validations do not occur automatically when using an API or BAPI.

Resolution

Implement validations for your custom fields using a BAdI.

Use the BAdI ME_BAPI_PO_CUST to embed the custom validation logic.
Leverage the method EXTENSIONIN to handle the custom fields during PO processing, specifically via the interface method IF_EX_ME_BAPI_PO_CREATE_02~EXTENSIONIN.

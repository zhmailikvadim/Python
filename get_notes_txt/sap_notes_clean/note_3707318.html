Symptom

In ‘Billing for Down Payment’ section (Test script scope item 4E9), down payment request is only enabled for work package type "Time and Expenses", and "Fixed Price".

Environment

SAP S/4HANA Cloud Public Edition

Reproducing the Issue

  1. Navigate to the Plan Customer Projects app.
  2. Select the "Billing" tab.
  3. Attempt to manage down payment requests for a work package with Billing Method = Usage-Based Billing.

Cause

  • In SAP S/4HANA Cloud Public Edition, the down payment process (Scope Item 4E9 – Billing for Down Payment) is designed to support only projects with work packages using Time and Expenses or Fixed Price methods.
  • Usage-Based Billing is consumption-driven, and billing amounts are calculated post actual usage records. This dependency prevents the system from linking or reconciling down payments with future usage-based billing documents.

Resolution

  1. As an alternative, create a separate Fixed Price work package exclusively for the advance payment, while maintaining the main project billing as Usage-Based Billing.
  2. Submit an improvement request via SAP Customer Influence to propose enhancements to the system in regards to down payment usage options.
    SAP Customer Influence Improvement Request

  3. KBA 3578819 for detailed information on down payment processing limitations for service contracts in SAP S/4HANA Cloud Public Edition.
  4. KBA 3206294 for guidance on submitting feature requests for SAP S/4HANA Cloud Enterprise Portfolio and Project Management.

Symptom

  • Reports such as MDF Configuration Data and MDF Change History do not show any deletion action performed by a user or API user.
  • The deletion is not traceable in Change Audit reports.

Environment

  • SAP SuccessFactors HCM Suite
  • Metadata Framework (MDF)

Reproducing the Issue

  1. Login to instance
  2. Navigate to Change Audit and generate the “MDF Configuration Data” report for the custom portlet record.
  3. Generate the “MDF Change History” report for the same record.
  4. Review the generated reports.
  5. Observe that no deletion action is recorded for the deleted record.

Cause

This is a known issue for user-based MDF entity(the externalCode field is user type).

Resolution

  • The fix for this issue is planned to be deployed on 1H 2026 release. For release timelines, please review SAP SuccessFactors Product Release & Road Map Information
  • The workaround is typing the externalCode directly, instead of selecting the user in the value help pop up. Then the change audit data is generated in the report.

Symptom

In Customizing, an entry for UTILMD message variant 021 is missing in table /IDEXGE/T_IN_VC (Customizing path: SAP Utilities -> Intercompany Data Exchange -> SAP Intercompany Data Exchange for German Electric Utilities -> Data Exchange Processes -> Assignment of Message Processing Classes -> Assign Class for Inbound Message Variant Versions). As a result, you cannot trigger UTILMD iDocs for this variant.

Other Terms

/IDEXGE/T_IN_VC, SPRO, UTILMD, IDXGC, IDEXDE, IDEXGE, MESCOD, message variant 021, format change, /IDXGC/3_0_UTILMD_NOTE_3717213.bcs

Reason and Prerequisites

The error occurred because of missing Customizing entry.

Solution

Implement the corrections of this SAP Note or import the relevant support package.

After implementing this SAP Note, the system updates table /IDEXGE/T_IN_VC with the required entry, allowing you to trigger UTILMD with message variant 021.

Activate the BC set /IDXGC/3_0_UTILMD_NOTE_3717213.bcs to import the required entries. You can verify the fix by checking that UTILMD is triggered successfully with message variant 021.

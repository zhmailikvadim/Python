Symptom

This KBA will cover how to cancel Service Calls, if necessary.

Environment

SAP Field Service Management

 

Reproducing the Issue

  1. Log in 'Shell'
  2. Navigate to the 'Planning & Dispatching' board
  3. Go to the 'Planning & Dispatching' board
  4. Enter 'Service Call' menu
  5. Open a 'Service Call' from the list
  6. Click on edit
  7. Change the status to 'Cancelled' 

Resolution

Cancel the SERVICE CALL via Data API.

  1. Create a Client in FSM admin

    1. Log into the Admin application of your account (https://{cluster}.fsm.cloud.sap/admin/)

    2. Click on Client on left sidebar

    3. Click on “Create” button

      Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.

    4. Define:
      1. “client-id”
      2. select “Client-Authentication Method”,
      3. select the “User Policy Group”

        Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.

    5. Copy the “client-id” and “client-secret”

      Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.


  2. Encode Client Id and Client Secret
    1. Arrange “Client Id” and “Client Secret”, separated by a colon, into a single line of text like this:

      00017214-myTestClient:f4e23cf5-b2fe-4314-bc2b-fcbcc127220b

    2. Use any offline or online tool to encode this text into Base64 format. Make sure to not include any invisible newline characters at the end. 




    3. The result should look similar to this:

      This is the Authentication Basic header for the OAuth 2 API request.



  3. Request OAuth 2.0 Token

    1. Copy the following command into a terminal application on your local machine and replace the {cluster} placeholder with the cluster of your account and the Base64 encoded value after Authentication Basic with your own value:

      CURL:
      curl --request POST \
        --url https://{{cluster}}.fsm.cloud.sap/api/oauth2/v2/token \
        --header 'authorization: Basic {{Encoded Client Id and Client Secret}}' \
       --header 'content-type: application/x-www-form-urlencoded' \
       --data grant_type=client_credentials


      Method:     POST
      URL:     https://{cluster}.fsm.cloud.sap/api/oauth2/v2/token

      Headers:
         Authorization: Basic {{Encode Client Id and Client Secret}}
         Content-Type: application/x-www-form-urlencoded

      Body:
      grant_type: client_credentials



    2. The response from the server should look similar to this:


    3. Copy the received token



  4. Retrieve the SERVICE CALL
    1. FSM Admin -> Company -> Query API

    2. execute the query:
      SELECT sc.code,  sc.id, sc.statusCode, sc.statusName FROM ServiceCall sc WHERE sc.code = '{{serviceCall_code}}';


      Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.

    3. Copy the Service Call ID

    4. Open the HTTP client for API call

      Method:     GET
      URL:    https://{cluster}.fsm.cloud.sap/api/data/v4/ServiceCall/{ServiceCall-ID}?dtos=ServiceCall.{dtoVersion}

    5. Headers

      Authorization:     Bearer {{token}}
      X-Account-ID:    {{fsm-account-id}}
      X-Company-ID:    {{fsm-company-id}}
      X-Client-ID:    {{any}}
      X-Client-Version:    {{any}}

    6. Params

      CURL:
      curl --request GET \
        --url 'https://{cluster}.fsm.cloud.sap/api/data/v4/ServiceCall/{ServiceCall-ID}?dtos=ServiceCall.{dtoVersion}' \
        --header 'authorization: Bearer {{token}}' \
        --header 'x-account-id: {{fsm-account-id}}' \
        --header 'x-client-id: {{any}}' \
        --header 'x-client-version: {{any}}' \
        --header 'x-company-id: {{fsm-company-id}}'

    7. Execute the API request

      The result should looks as following:

    8. The received data you can use for your next API request.


  5. Cancel the SERVICE CALL
    1. Method:     PUT / PATCH
      URL:           https://{cluster}.fsm.cloud.sap/api/data/v4/ServiceCall/{ServiceCall-ID}?dtos=ServiceCall.{dtoVersion}


    2. Headers

      Authorization:     Bearer {{token}}
      X-Account-ID:    {{fsm-account-id}}
      X-Company-ID:    {{fsm-company-id}}
      X-Client-ID:    {{any}}
      X-Client-Version:    {{any}}

    3. Params

      forceUpdate:     true
      dtos:        {{dto}}.{{version}}


    4. Body – add fields with values you want to update 

       {
              "statusCode": "-4",
              "statusName": "Cancelled"
       }

      CURL:

      curl --request PATCH \
        --url 'https://{cluster}.fsm.cloud.sap/api/data/v4/ServiceCall/{ServiceCall-ID}?dtos=ServiceCall.{dtoVersion}&forceUpdate=true' \
        --header 'authorization: Bearer {{token}}' \
        --header 'content-type: application/json' \
        --header 'x-account-id: {{fsm-account-id}}' \
        --header 'x-client-id: {{any}}' \
        --header 'x-client-version: {{any}}' \
        --header 'x-company-id: {{fsm-company-id}}' \
        --data '{
        "statusCode": "-4",
        "statusName": "Cancelled"
      }'



    5.  Execute the request


  6. Check the API response

    Expected response: "statusCode": "-4",  "statusName": "Cancelled"

Symptom

SQLDriverConnect() / SQLDriverConnectW() may not set DATABASE parameter to OutConnectionString.

  • This problem may occur when InConnectionString of SQLDriverConnect() / SQLDriverConnectW() does not contain DATABASE parameter.
  • From ASE ODBC Driver 15.7 SP133 and 16.0 SP00 PL05, this problem occurs for all data source names (DSNs).
  • From ASE ODBC Driver 15.7 SP140 and 16.0 SP03 PL02, this problem only occurs if data source names (DSNs) contain "db", for example, "dbsource" or "xdbsystem".

Environment

  • SAP Software Developer Kit (SDK) 15.7 SP133 or later
  • SAP Adaptive Server Enterprise (ASE) Software Develope Kit (SDK) 16.0 SP00 PL05 or later
  • Adaptive Server Enterprise ODBC Driver by Sybase
  • Microsoft Windows x86 (32-bit)
  • Microsoft Windows x64 (64-bit)
    • Win 7/8/10, 2008/2008 R2, 2012/2012 R2

Reproducing the Issue

  1. Create a new data source using ASE ODBC driver via ODBC Administrator (32bit) with the following information.
    The DSN needs to contain "db".

Data source name : xdbsystem
Server name : <Host name where your ASE is running>
Server port : <ASE port number>
Database name : <database name>
Login ID : <user name>

  1. Download "Microsoft Data Access Components (MDAC) 2.8 Software Development Kit" and install it.
  2. Run C:\Program Files (x86)\Microsoft Data Access SDK 2.8\Tools\x86\odbct32w.exe.
  3. Change "Argument Display Length" in Buffer Lengths tab.

Choose "Tools" -> "Options...".
Select "Buffer Lengths" tab.
Change "Argument Display Length" to 200.
Click "OK" button.

  1. Allocate environment handle

Choose "Env" -> "SQLAllocHandle" from menu bar.
Select "SQL_HANDLE_ENV=1 (3.0)" as Handle Type
Click "OK" button.

  1. Set environment attribute to specify ODBC version

Choose "Attr" -> "SQLSetEnvAttr" from menu bar.
You don't have to modify any parameter.
Click "OK" button.

  1. Allocate connection handle

Choose "Env" -> "SQLAllocHandle" from menu bar.
Select "SQL_HANDLE_DBC=2 (3.0)" as Handle Type.
Click "OK" button.

  1. Connect

Choose "Connect" -> "SQLDriverConnect" from menu bar.
Choose "<empty string>" as InConnectionString.
Choose "SQL_DRIVER_COMPLETE=1 (1.0)" as DriverCompletion.
Click "OK" button.

A dialog box to select a data source will appear. Choose the DSN "xdbsystem", and click "OK" button.
Then, a dialog box to input the required information for the DSN will appear. Input a password, and click "OK" button.

  1. Check if SQLDriverConnect() succeeds with SQL_SUCCESS_WITH_INFO and OutConnectionString does not contain DATABASE parameter.

Cause

  • The issue has been identified and logged under SAP CR 817256. It's description reads, "SQLDriverConnect() and SQLDriverConnectW() with SQL_DRIVER_COMPLETE may not set DATABASE parameter to OutConnectionString if data source names include "db" in its name."
  • OutConnectionString is set a pointer to a buffer for the completed connection string. Upon successful connection to the target data source, ODBC Driver should set the completed connection string to this buffer.
  • Microsoft Access and Excel rely on the value of OutConnectionString returned from SQLDriverConnectW(), so you may unexpectedly see one or more dialog boxes to input required information for a DSN because of missing DATABASE parameter.

Resolution

  • There is currently no target SAP SDK Service Pack scheduled for this issue (SAP CR 817256). It is being reviewed by our Product Group. Please mark this KBA as a Favorite to be automatically notified when the CR is fixed.
  • Workaround:
    • Use SAP SDK 15.7 SP130 or SDK 16.0 SP00 PL04.
    • Rename DSNs so that they don't contain "db" in their names.
    • Explictly specify DATABASE parameter in InConnectionString when calling SQLDriverConnect() or SQLDriverConnectW() if possible.

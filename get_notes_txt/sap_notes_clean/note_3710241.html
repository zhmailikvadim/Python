Symptom

User updates product data with the MDG, Mass Processing Fiori application. Application fails to load the active area data of the material ledger prices after starting the process. It raises the CX_SY_OPEN_SQL_DB exception and corresponding  "cannot allocate enough memory" error can be found in the system log (SM21 transaction).

Other Terms

I_MaterialValuationPrices, IMATVALPRICES, MBEW, V_MBEW_MD, APPEND_CKMLCR, CL_MDC_DATA_MAT, CMP performance, CKMLCR

Reason and Prerequisites

Program error

Solution

 Implement correction instructions.

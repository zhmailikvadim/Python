Symptom

The warning below is shown in the Enable Gateway Services step after Support Package update.

System Changed. This activity needs to be re-executed

Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.

Environment

SAP Solution Manager 7.2 SP06 and above.

Reproducing the Issue

After the Support Package level of the Solution Manager updated, the warning will be shown in the transaction SOLMAN_SETUP, under the scenario Infrastructure Preparation, in the step Enable Gateway Services

Cause

All Gateway Services under the same Application group should have the same activation status. However, after Support Package update, there could be new Gateway Services added under the same Application group that you previously activated.

Resolution

  • Apply the SAP Notes 2568040, 2582326, 2592942 and 2681010 first if possible.
  • If all Gateway Services have been previously activated, try to activate all Gateway Services again so that new Gateway Services from the Support Package update are included in the activation.
  • Else it is needed to make sure that those new Gateway Services that are required by the existing activated Gateway Services are activated as well.
  • For instance, screenshot below shows that the first two Gateway Services under the Job Monitoring Application already being activated with green status, and after the Support Package update, the third Gateway Service is added under the Job Monitoring Application and is in deactivated status.
    When any one of the checkbox of the 3 Gateway Services is selected , the other two will be selected automatically. 
    It is needed to click the Activate button so that all Gateway Services under the Job Monitoring Application will have the same activation status.

The image below is just an example, and it is needed to check for each of the activated Gateway Services.

  • It is recommended that manually select all activated Gateway Services and activate them again to ensure that the newly added relevant Gateway Services are included in the Activation.
  • If there is still warning status in this step, try to go through a few rounds of manually selecting those activated Gateway Services to ensure that no relevant Gateway Services are accidentally left out.

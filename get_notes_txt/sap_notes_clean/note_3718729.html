Symptom

When I perform a User Upload, many records fail with the following error:

Error:
The selected default OUC / Cost Centre is not associated to the selected default Legal Entity.

This error occurs even when I have verified in the UI that the Cost Center is correctly associated with the Legal Entity I am uploading.

Cause

Cause:

This issue is typically caused by one of two primary factors:

  • Header Name Mismatch: The upload engine expects a specific header name to identify the Cost Center (e.g., Default OUC / Cost Centre). If the header in your file is slightly different (e.g., Default Cost Centre), the system will ignore the column. Consequently, the system attempts to validate the uploaded Legal Entity against the existing Cost Center already on the user's profile, leading to an association error if they do not match.

  • Hidden Data Formatting (Extra Commas): Tools like Microsoft Excel can sometimes insert trailing commas into CSV files. These extra characters can interfere with the system's ability to parse the record correctly, causing valid combinations to fail.

Resolution

To resolve these errors, you must validate your file structure and data formatting.

Step 1: Validate Column Headers

  1. Navigate to Admin Configuration > Integration > Connector Catalog.
  2. Search for the User Upload and view the required technical specifications.
  3. Ensure your file header matches exactly (e.g., Default OUC / Cost Centre).
  4. If the header is incorrect, update your file and re-run the upload.

Step 2: Cleanse Data Formatting

  1. Open your CSV file in a plain text editor (like Notepad or TextEdit) instead of Excel.
  2. Check for extra commas at the end of rows or within data fields.
  3. Remove any unnecessary delimiters and save the file.
  4. Re-upload the cleansed file to verify the fix.

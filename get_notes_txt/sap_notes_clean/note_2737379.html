Symptom

When you perform a forward booking of claimed components, they are not taken into account in the renewal background job using the new function for LRM_862 and the "booked-until" date is not updated.

Other Terms

- Preliminary posting

- Renewal background job

- Claimed

- Claim

- Booked until

- CAP

Reason and Prerequisites

Program error

Solution

History:

Version 1 from 1/4/2019: New
Version 2 from 1/7/2019: Adjusted

Implement the source code corrections.
Support Package SAPK-70020INMSG corrects the problem.　

Solution:

When you perform a forward booking of claimed policy components using the renewal background job, the system also takes into account the claimed components and updates both the "booked-until" date and the CAP.

Technical effects:

There are none.

Manual changes:

You do not have to perform any manual tasks.

Symptom

Integration flows using the AEM adapter experience connection failures with the following error

Error Message: 503: Too Many Connections For Client-Username

This error occurs when:

  • A specific client username has exceeded the maximum number of simultaneous connections allowed by the SAP Integration Suite, advanced event mesh service
  • Connection counts temporarily double during Cloud Integration blue-green deployment
  • Multiple application instances are sharing the same username
  • Service plan connection limits are insufficient for deployment scenarios

Other Terms

  • 503 Error
  • Too Many Connections
  • Client Username Limit
  • AEM Service
  • Connection Pool
  • max-connections-per-client-username

Reason and Prerequisites

The error is triggered by the max-connections-per-client-username setting configured in the client profile associated with your username. This setting limits the number of concurrent connections a single username can maintain across all service types (SMF, Web transport, REST, MQTT, and AMQP combined).

Three Types of 503 Errors

  1. 503: Too Many Connections For Client-Username — Per-username limit exceeded (this error)
  2. 503: Too Many Connections For VPN — Message VPN limit exceeded
  3. 503: Too Many Clients — System-wide broker maximum reached

Common Scenarios:

Blue-Green Deployment Impact
During Cloud Integration blue-green deployment of worker nodes, connection counts may temporarily double due to the nature of the deployment strategy. This occurs because both old and new worker nodes establish connections simultaneously during the switchover phase.

Other Scenarios:

  • Multiple application instances sharing the same username
  • Service plan connection limits not configured to handle deployment scenarios
  • Insufficient connection threshold planning for temporary spikes during deployments

Solution

1. Increase the Connection Limit

Adjust the max-connections-per-client-username setting in the client profile.

Reference: See Configuring Client Profiles in the SAP Integration Suite, advanced event mesh Help documentation.

2. Use Connection Pooling in Integration Flow (AEM Sender Adapter)

Starting with AEM Adapter version 1.4.1, a feature to pool connections on the Sender Adapter end has been introduced, reducing the number of connections created.

Action Required:

  • Upgrade to AEM Adapter version 1.4.1 or higher
  • Configure connection pooling in your integration flow

Reference: Please refer to the 1.4.1 AEM Adapter release notes and documentation.

3. Monitor Connection Usage

Monitor AEM Service events to track connection usage:

  • VPN_CLIENT_USERNAME_CONNECTIONS_HIGH — Warning event when approaching limit
  • VPN_CLIENT_USERNAME_CONNECTIONS_EXCEEDED — Error event when the limit is exceeded

4. Check System Limits (AEM Service)

Be aware of the system-wide connection limits based on your AEM Service edition:

Edition  Maximum Connections
Standard 100 connections
Enterprise Edition Up to 100,000 connections

Important Notes

Plan for Blue-Green Deployments:
Plan your SAP Integration Suite, advanced event mesh service connection thresholds accordingly to avoid JCSMPErrorResponseException: 503: Too Many Connections errors during new node startup.

Proactive Management:

  • Configure connection count alerts on your AEM Service
  • Ensure your service plan's connection limits can accommodate the temporary spike during blue-green deployments
  • Monitor connection metrics regularly to prevent reaching limits
  • Consider using connection pooling to reduce the total number of connections required

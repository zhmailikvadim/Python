Symptom

  • When an operation is deleted from an existing work order, the 'FLAG_DELETE' flag is not updated in table /SMFND/D_OBLNK_H.

  • When an operation in an existing work order is deleted and added again with the same activity/operation number, the external ID changes in SAP Business Network Asset Collaboration (BNAC), but it is not updated in table /SMFND/D_OBLNK_H. As a result, any changes to the new operation are not reflected in BNAC. The documents attached to the new operation also do not get synchronized to BNAC.

    Steps to reproduce the issue:

    1. Change an existing work order with operations in IW32.

    2. Delete an operation and add it again with the same operation number.

    3. Save the work order.

    4. Check table /SMFND/D_OBLNK_H in SE16N. A new external ID is generated in BNAC, while the table /SMFND/D_OBLNK_H still contains the old external ID.

Other Terms

Work order, document, operation

Reason and Prerequisites

Mismatch of external IDs in the object link table for work order operations in SAP ERP (S4/ECC)

Solution

Implement the note or upgrade to the relevant support package.

After implementing the note, all changes and documents attached to the work order operations in ERP (S4/ECC) will be replicated and attached to the work order operations in SAP Business Network Asset Collaboration (BNAC).

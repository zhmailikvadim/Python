Symptom

The current timeline in the Capacity Scheduling Board app (F3951) is displayed in UTC time zone, rather than the local plant time zone.

*Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.

Environment

SAP S/4HANA Cloud Public Edition

Cause

The "current timeline" in the Gantt chart in the application is set to UTC by design.

Since the application can load multiple plants across multiple time zones, each row in the Gantt chart can correspond to a different time zone.

To avoid confusion, the "current timeline" is always shown in UTC time zone and not in any individual time zone.

Resolution

If the current timeline needs to be displayed in the plant time zone, raise a feature request on the Customer Influence Portal.

Refer to KBA 2712608 - How to Submit Feature Requests for Core Logistics and Manufacturing in SAP S/4 HANA Cloud Product for more details

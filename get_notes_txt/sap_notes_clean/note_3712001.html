Symptom

Searching for an entry using a smart filter results in "No results found" even when the searched value exists in the application.

Environment

SAP S/4HANA Cloud Public Edition

Reproducing the Issue

  1. Access the affected app in the Fiori Launchpad.
  2. Enter a value in the smart filter field and click "Go".
  3. Observe that the search ends with the message "No results found", even though the searched value exists in the application.

Cause

In the SAP Fiori UI, the smart filter fields from the smart field bar are case-sensitive. Searching for a value with incorrect case formatting (e.g., 'xyz' instead of 'Xyz') does not return the expected results.

Resolution

  1. Ensure that the value inserted in the filter has the exact same case as the stored user data.
  2. Alternatively, use pattern matching with asterisks (*) around the search terms where supported.

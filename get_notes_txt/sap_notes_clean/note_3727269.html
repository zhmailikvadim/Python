Symptom

You are trying to check any mashup (like Payroll System Information) that requires admin services in SuccessFactors (except Pay Statement) and get the following error message:

  • Error ID: 201
  • Error Message: The system failed to generate the access token.
  • Detailed Exception: { "error":"invalid_scope","error_description":"Requested OAuth 2.0 scope exceeds the scope granted by the resource owner or OAuth 2.0 client. Make sure that both have access to the scopes requested. For more information consult the kernel traces or the OAuth 2.0 trouble shooting SAP note 1688545" }

Environment

  • SAP SuccessFactors Employee Central Payroll
  • Mashup
  • Admin Services

Cause

Either the user that is trying to check this mashup is not allowed (don't have the required roles in SU01, is not set as Admin in SuccessFactors and/or the user mapping is not matching in SAML Trace), or this user doesn't exist in Employee Central Payroll at all.

Resolution

  1. Ensure that all solution steps from SAP Knowledge Base Article 3586976 were followed;
  2. See if Payroll Administration RBP is assigned to the user in SuccessFactors;
  3. Run a SAML Trace through Security Diagnostic Tool to identity which user is getting mapped in the mashup call. For more information about how to take the trace, please check SAP Knowledge Base Article | 2960670 - How to collect SAML trace with the Security Diagnostic Tool for ABAP system;
  4. Check SU01 to see if this user exist. If yes, ensure that the Alias is set according to what is getting mapped (you can check it through the SAML Trace from step #2);
  5. If the user exist, make sure that role SAP_CLOUD_ADMIN_OAUTH is assigned and the profiles for this role were generated. To check if the profiles were generated, after assigning the role, double click on it. If Authorizations and User tab are showing a green circle, then the profiles were generated, otherwise follow the steps below:
    1. In Authorizations tab, click in Display Authorization Data. In the new window that will open, click on Generate (ball icon);
    2. In User tab, click on User Comparison button.

      In case that the tabs are still not showing a green circle, repeat the process.

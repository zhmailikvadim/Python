Symptom

When listing a contract in transaction VA45 in S/4 HANA system, the value of field "order quantity" shows blank.

Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.

Environment

  • Sales and Distribution (SD)
  • SAP S/4HANA

Reproducing the Issue

  1. Check the contract in transaction VA42/VA43, the value of field "order quantity" is 13.



  2. Check the contract in transaction VA45, the value of field "order quantity" is blank instead of 13.

Cause

In transaction VA42/VA43, field "order quantity" has technical name RV45A-KWMENG and its value is calculated during runtime as mentioned in KBA 2489907. However, in transaction VA45, it used the optimized report SD_SALES_DOCUMENT_VA45 to select value from the database tables such as VBAK and VBAP. 

Resolution

This is standard system behavior. Displaying different quantity value by the VA45 report requires modification of the standard program. For more detail about the optimized reports according to your requirement, please refer to the note 1780163.

See Also

The similar scenario in ERP system is described in KBA 2489574 - Field "Order quantity" in VA45 shows different value from the one in VA42/VA43.

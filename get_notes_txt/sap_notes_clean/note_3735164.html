Symptom

When a non-invoice document is uploaded in SAP Ariba Invoicing, the system recognizes it as an invoice.

 

Environment

SAP Ariba Invoicing.

Reproducing the Issue

  1. In SAP Ariba Invoicing, go to "Upload Supplier Invoices Centrally"
  2. Select a company code
  3. Select pdf file that is not an invoice
  4. Click on "Start Upload" wait for the upload to be completed
  5. Go to "Manage Supplier Invoice Centrally"

An invoice is created and set to status "Draft/"Manual Adjustment Required".

 

Resolution

This is expected behavior. SAP Ariba Invoicing is designed to process invoice files. If a non-invoice file is selected, the application treats it as an invoice.

Uploading non-invoice documents through this function is a misuse of the application. 

In SAP Ariba Invoicing all documents uploaded should be invoices.

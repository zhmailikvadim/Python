Symptom

You want to know what JDBC driver version currently supported by Microsoft can be used with SAP ME 15.5.

Other Terms

Microsoft JDBC Driver certification, mssql-jdbc-12.10.1.jre8.jar, MSSQL 2022

Reason and Prerequisites

SAP has certified a newer Microsoft JDBC driver version for usage with SAP Manufacturing Execution 15.5.

Microsoft JDBC Driver version 8.4 has reached its end-of-support date (July 31, 2025), therefore a newer driver version must be used going forward.

This note provides information about the certified Microsoft JDBC driver version that can be used with SAP ME/MECTC/MEINT/MEINTCTC/MESCRIPTS 15.5 when running on Microsoft SQL Server 2022.

Microsoft maintains the driver lifecycle and compatibility matrix here: https://learn.microsoft.com/en-us/sql/connect/jdbc/microsoft-jdbc-driver-for-sql-server-support-matrix. Customers should always verify general platform support and lifecycle status using the Microsoft support matrix.

Solution

The following Microsoft JDBC driver version is certified for use with:

  • SAP ME 15.5
  • SAP MEINT 15.5
  • SAPMECTC 15.5
  • SAPMEINTCTC 15.5
  • SAPMESCRIPTS 15.5
  • Microsoft SQL Server 2022

Certified driver:
mssql-jdbc-12.10.1.jre8.jar (Microsoft JDBC Driver 12.10 for SQL Server)
The driver can be used during:

  • SAP ME installation and database setup
  • SAP ME database migration and upgrade procedures
  • CTC wizards for database configuration and upgrade

The mssql-jdbc-12.10.1.jre8.jar file must be specified as the JDBC driver during execution of the SAP ME Database Setup (Installation) and SAP ME Database and Data Migration (Upgrade) CTC wizards.

Important Notes:

Starting with Microsoft JDBC Driver 10.2, SSL encryption is enabled by default and required by the driver. Therefore, the SQL Server instance must be properly configured for SSL/TLS connections.
If SSL is not configured on the database server, database connectivity tests and CTC wizards will fail.

Ensure that:

  • SQL Server has a valid TLS configuration.
  • Certificates are properly installed.
  • Trust settings are aligned with your security policy.

The current ME behavior aligns with modern security best practices and encourages users to implement SSL.
However, security level is not strictly enforced by ME application after completion of CTC tasks.
Security settings can be changed later at the customer’s initiative and upon acceptance of risks. For example, by adding parameters to the jdbc connection string of the datasources.

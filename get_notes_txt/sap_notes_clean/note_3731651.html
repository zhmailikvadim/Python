Symptom

  • Outbound delivery is created with zero quantity in the delivery quantity field.
  • During goods movement for the delivery, the delivery quantity is shown blank.

Environment

  • SAP R/3
  • SAP R/3 Enterprise
  • SAP ERP
  • SAP ERP Central Component
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4 HANA

Reproducing the Issue

  1. Start the delivery creation from purchasing document or from sales document.
  2. Observe that the created outbound delivery item has 0 quantity and no confirmed quantity.

Cause

  1. Missing correction note
  2. Lock entries are blocking the available stock
  3. Customizing setting

Resolution

  1. Missing correction note
    • Check whether the following correction has been implemented. If not, implement it and test the situation again: SAP Note 3226387 - Delivery created for zero qty
  2. Lock entries are blocking the available stock
    • Check the lock/enqueue overview (for example, transaction SM12) for ATP-related locks for the affected material and location: ATPENQ and ATPENQS4.
    • If obsolete ATP enqueue entries are found (not tied to active business documents), remove the outdated locks in coordination with the responsible business team. Useful information about deleting lock entries can be found in our KBA 1725088 - Discrepancy between transactions Co09 & Co06 
    • Re-run delivery creation and confirm that quantities are now confirmed and populated.
  3. Customizing setting
    • If public cloud used, check KBA 3229596 - Delivery can be created with 0 quantity

Symptom

You have a Non-Hana database system and some of your table accesses to database table USMD_PP_CD are very slow. Such table accesses happen at different MDG applications (e.g. at the display of change documents, the archiving of change requests or the deletion of change requests).

Other Terms

USMD_PP_CD, archiving object USMD_CR

Reason and Prerequisites

Technically, ABAP select statements at table USMD_PP_CD using only the change request as selection condition in the where clause, do not access via a primary index. For such table accesses a secondary index is created via this note. Such an index is automatically created at SAP Hana databes systems which is the reason that the index must only be created for Non-Hana database systems.

Solution

Perform the manual activity of this note or implement the respective support package.

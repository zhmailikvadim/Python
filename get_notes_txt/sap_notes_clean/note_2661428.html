Symptom

("+" masks a single character, "*" masks any character string)

Environment

ABAP system

Reproducing the Issue

Using wildcard characters "*" or "+" in the HIGH(Upper limit) filed of "Range" or "Exclude Range".

Cause

Wildcard characters can not be used for RANGE selection.

This information can also be found in the online help:
Classic Dynpro Programming

"If high is filled, you can use BT (BeTween) and NB (Not Between). These
operators correspond to BETWEEN and NOT BETWEEN that are used to check
if a field belongs to a range. You cannot use wildcard characters."

Resolution

It is a standard behaviour.

DO NOT use wildcard characters in RANGE, instead, ALWAYS use fixed value.

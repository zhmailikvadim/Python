Symptom

  • An ATC check variant configured for clean core compliance is showing multiple findings with the title/message "Detect customer modifications/object is modified."
  • Findings are being reported for objects assigned to custom namespace, resulting in false positives.

Environment

  • SAP S/4HANA
  • ABAP Check Frameworks - ABAP Test Cockpit, Code Inspector

Reproducing the Issue

  1. Access transaction ATC.
  2. Click on "Schedule runs" and schedule an ATC Clean Core check.
  3. Review the results for findings with the title/message "Detect customer modifications/object is modified."

Cause

The current version of the "Detect Customer Modifications" check is older than the specific checks designed for Clean Core compliance. It reports all modifications, including those in custom namespaces, without restriction to objects owned by SAP.

Resolution

  1. Implement SAP Note 3679781 to configure the check to only report Business Modifications.
  2. This configuration should remove the modifications that are irrelevant in the Clean Core context.
  3. Monitor the system after applying the note to ensure the findings align with Clean Core compliance expectations.

See Also

  • SAP Note 2928076 - Detect Customer Modifications Check
  • SAP Note 3679781 - Configuration for Business Modifications

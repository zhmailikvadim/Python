Symptom

  • Error loading tiles in launchpad designer
  • Tiles not displayed in launchpad designer
  • Tiles are visible in the CONF layer but not the CUST layer.

Environment

  • SAP NetWeaver 7.5
  • SAP S/4 Hana
  • SAP Fiori 1.0 or higher

Reproducing the Issue

  1. Log into Fiori launchpad designer
  2. Navigate to a Catalog e.g. - SAP_SD_BC_SALES_ANLYTS_OP
  3. Observe the error, i.e.-

Cause

Catalogs might be outdated or corrupted and have lost the reference

Resolution

  1. Go to the configuration layer by providing the parameter scope=CONF on your designer URL i.e https://<hostname:port>/sap/bc/ui5_ui5/sap/arsrvc_upb_admn/main.html?scope=CONF# 
  2. Look for the same Catalog and confirm that all tiles load successfully without any issue
  3. Navigate back to the customization layer by removing the parameter scope=CONF in the URL
  4. Delete tiles which do not load or delete the entire Catalog
  5. Automatically, new tiles or Catalogs will appear by the replication from the configuration layer

Note:

  • If the same issue is observed in the configuration layer, then the tile or catalog is removed by SAP
  • This only applies for SAP delivered Catalogs because remote Catalogs are not available in the configuration layer

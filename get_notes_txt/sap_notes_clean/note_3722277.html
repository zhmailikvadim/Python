Symptom

When opening the Create Access Request tile in SAP Identity Access Governance, you may encounter the following behavior:

  • A popup message displays: "Error Not Found"
  • After closing the popup, the Application filter in the access request form is blank nothing is listed in filters.

Environment

  • SAP Identity Access Governance (IAG)

Cause

The issue is caused by a browser navigator cache conflict.

Outdated or corrupted cached data stored in the browser interferes with the loading of the Create Access Request tile, preventing applications from being retrieved and displayed correctly. 

Resolution

Step 1 — Clear the browser navigator cache

  1. Close all active SAP IAG sessions.
  2. Open the browser settings and navigate to the cache/browsing data section.
  3. Clear all cached data, including cookies and stored site data.
  4. Close and reopen the browser.
  5. Log back into SAP IAG and reopen the 'Create Access Request' tile.
  6. Verify that the Application filter now displays the expected entries and that no error popup appears.

Step 2 — Verify pending configuration items

After clearing the cache, review SAP Note 3422959 - IAG - Cannot Create Requests in IAG - SAP for Me to confirm that all required configuration steps have been completed in the system.

Pending or incomplete configuration items addressed in that note may contribute to or reproduce the same behavior.

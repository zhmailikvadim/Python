Symptom

You have a user in tenant BBB and the user has select privilege on a table located in another tenant AAA after cross-database access has been configured. You want to grant the select privilege on the same table to another user in BBB and it fails with error: SAP DBTech JDBC: [7]: feature not supported: use of remote database object.

Environment

  • SAP HANA, platform edition 1.0
  • SAP HANA, platform edition 2.0

Cause

For cross-database access, the authorizations are managed via remote identity and it is not technically possible to grant privileges on objects in source database between users in target database.

Resolution

Refer to Cross-Database Authorization in Tenant Databases | SAP Help Portal to create the remote identity.

See Also

3674449 - How to set up SAP HANA Cross-database Access

Keywords

use of remote database object，cannot grant privilege on table in source database

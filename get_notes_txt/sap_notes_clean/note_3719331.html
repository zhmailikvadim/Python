Symptom

  • Suppliers are not inactivating despite being included in the upload file and marked for inactivation
  • No errors are thrown by the connector for the suppliers that remain active
  • Manual inactivation attempts via the Admin Configuration > Supplier > user are unsuccessful, with no error messages displayed

Reproducing the Issue

  1. Attempt to inactivate suppliers using Activate or Inactivate Supplier Upload connector
  2. Verify that the upload file includes the suppliers to be inactivated
  3. Check the connector logs for errors; no errors are displayed for the suppliers that remain active
  4. Attempt to manually inactivate the suppliers via the Admin page > Supplier > user
  5. Observe that the suppliers remain active despite no errors being thrown

Cause

Suppliers cannot be inactivated due to one or more of the following reasons:

  • Suppliers are associated with active admin objects
  • Suppliers are included in Distribution Lists or Approval Groups
  • MSP suppliers cannot be inactivated from either the UI or upload

Resolution

Verify that the suppliers are disassociated from Distribution Lists:

  • Navigate to Admin Configuration > Distribution List
  • Select the relevant Distribution List and click Edit
  • Locate the supplier in question and select Disallowed under the Usage drop-down menu
  • Click Update

Verify that the suppliers are removed from Approval Groups:

  • Navigate to Admin Configuration > Approval Group
  • Select the relevant Approval Group and click Edit
  • Remove the supplier or edit the Role of the approver
  • Click Update
  • Re-upload the file and verify the transaction is successful.

For MSP suppliers, note that they cannot be inactivated via UI or upload

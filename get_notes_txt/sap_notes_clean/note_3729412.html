Symptom

The employee is a new starter and a pensioner. 

Annual pension amount should be filled in the RTI table. If it is not, please follow the below suggestions. 

Environment

  • Human Capital Management (HCM)
  • SAP R/3
  • SAP R/3 Enterprise 4.7
  • SAP ERP Core Component
  • SAP ERP
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP SuccessFactors Employee Central integration to SAP Business
  • SAP SuccessFactors Employee Central Payroll

Resolution

The annual pensions amount is reported just for the new hire pay period and not later. Late new hires will have their pensions reported in the coming month.

The new starters have to have the right tax code source applied in order for it to be correctly reported.

Tax source has limited validations which allow wrong combination of entries to be saved. To resolve this, the following enhancements have been made:

  • For Tax code source = "EMP"
    a. New starter Declaration = 4 then tax code entered must be 0T WK1/Mth1 basis
    b. New starter Declaration = 3 then tax code must be BR, 0T, D0, D1.

          No validation on starter Declaration 1 or 2 added

  • For Tax code source ="EXP"
    a. New starter declaration = 4 then tax code entered must be 0T mth1 basis
    b. No validation added for starter declaration is 1,2 or 3
    Suffix codes "J" and "V" have been removed as they are no longer valid.
  • If the tax code source is PEN and Annual Pension amount is entered, then Starter
    Declaration should be blank.

HMRC have re-introduced the requirement for all NI numbers to have suffixes.
Therefore, an extra validation has been introduced. The last character must be ABDC.
If the tax code source is PEN and Occupational Pension indication is set, annual Pension
amount cannot be 0 or blank.
In addition, the validations for NI number has been enhanced to check for the suffix.

Also make sure, that GBPEN Pensioner Indicator is active. 

This feature is used to configure pensioners. PME02 is the structure for decision making.
This is used by the payroll function to set the Number of Hours worked for Pensioners as ‘Other’.

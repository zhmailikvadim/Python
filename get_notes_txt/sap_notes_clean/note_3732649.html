Symptom

KPI shows wrong value

Environment

 SAP Cloud for Customer core applications 2508

Reproducing the Issue

1) Go to business analytics work center

2) Go to design reports view

3) Open the report and see the key figure value

4) Go to design KPI

4)Open the KPI and click on preview and see the value

The value of the key figure different in report and KPI

Cause

KPI does not consider the formula at each row level in report instead it will take overall value of the dependent field. According standard behavior calculated key figures always calculate based on total value.

For example you created a calculated key figures with formulas as KF1 = A+B , KF2 = MAX0(KF1) , KF2 consider total value of KF1 , if we add row fields in report then at each row level it will show expected answer in report at each row level and if you add values of those individual row result you will get correct value and in KPI it will show different value but if you remove row fields in report  KF2 show value as max0(KF1 total value) which will match KPI value. In KPI even though you add row fields in report KPI Shows the value based on total value. Due to this difference you confuse that KPI showing wrong value but it is showing correct value only.

Resolution

Remove all the row fields from the view in report and compare the value of key figure with KPI value. You will see the same value.

Please note that if you want to create a custom key figure which need to be calculate the values at each row level then it is not possible due to standard behavior. It will show you correct value at each row if you add the row fields to the view. But in KPI it won't consider that as you are expecting.

Symptom

You use actual costing runs which include company codes with multiple fiscal year variants. After running the CKMLCP Settlement step you observe wrong actual costing results.

Other Terms

CKMLCP;
XSHIFTED

Reason and Prerequisites

Due to a program error, the preliminary beginning inventory values and quantities are not included in the actual cumulative stocks of the materials of the costing run. Due to another program error, the total consumption quantities and values of the activity types of the costing run are not determined correctly. Instead, the settlement of the activity types is based on a total consumption quantity and value of zero.

Solution

Please implement the attached program corrections.

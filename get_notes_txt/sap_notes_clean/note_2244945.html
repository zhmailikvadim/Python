Symptom

  • The order of the prompts in Web Intelligence documents is incorrect under some scenarios
  • This occurs when using a "Between" or "Not Between" filter type and changing the "end" filter type to "Prompt" before changing the "start" filter to "Prompt"

Other Terms

webintelligence, webi, query, prompt, list of values, LOV, define, definition, wrong

Reason and Prerequisites

Environment:

  • SAP BusinessObjects Business Intelligence Platform 4.1
  • Web Intelligence

Steps To Reproduce:

  1. In any editor, create a Web Intelligence document based on the eFashion sample universe
  2. Drag the "Year" object into the Result Objects and Query Filter Windows
  3. Set the Query Filter to "Between"
  4. Change the second filter’s type to "Prompt"
  5. Change the first filter’s type to "Prompt"
  6. Click Run Query
    1. >> When the Prompts window appears, the "End" prompt value appears first and the "Start" prompt value appears second
    2. When the "start" filter is edited first, and then the second, the prompt order is as expected (Start first, End second)

Cause:

This issue is caused by a defect in the product.

Solution

This issue is fixed in the patches listed in the "Support Packages & Patches" section below.
The "Support Packages & Patches" section will be populated with the relevant patch levels once they are released.

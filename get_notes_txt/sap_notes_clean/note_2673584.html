Symptom

The analytic engine transforms CDS metadata to (transient) BW metadata. The result of the transformation is buffered. But when using this metadata buffer, it has to be checked, if on the the used CDS view has been updated. This check might be time consuming, because of

  • many selects on DB table DDNTT
  • long runtime (some 100 milli seconds gross time) in method _GET_CURRENT_ODP_MD_TIMESTMP of class CL_RSRTS_METADATA_ODP

Other Terms

CDS, CL_RSRTS_METADATA_ODP, buffer,

Reason and Prerequisites

Check of changes of CDS view is not optimal

Solution

  • SAP BW 7.52
    Implement Support Package 3 for SAP BW 7.52 (SAPK-75203INSAPBW) into your BW system. The Support Package will be available as soon as SAP Note 2613219 with the short text "SAPBWNews 7.52 BW ABAP SP3", which describes this Support Package in more detail, is released for customers.

  • SAP BW 7.53
    Implement Support Package 1 for SAP BW 7.53 (SAPK-75301INSAPBW) into your BW system. The Support Package will be available as soon as SAP Note 2653494 with the short text "SAPBWNews 7.53 BW ABAP SP1", which describes this Support Package in more detail, is released for customers.

In urgent cases you can use the correction instructions.

Before you use the correction instructions, make sure that you check SAP Note 1668882 and SAP Note 2248091 for transaction SNOTE.

This SAP Note might already be available before the Support Package is released. In this case, however, the short text still contains the term "preliminary version".

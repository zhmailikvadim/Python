Symptom

In the process "Receipt of EQS PoD Deactivation for Grid Time Series from nDSO (DSO View)" (DE_B_EQS_POD_DACTV_DSO_RECEIVE_DSO), the system deactivates the Point of Delivery (PoD) even when the DTD check fails. No rejection is sent in this case.

This issue is relevant for the market role: DSO.

Other Terms

EQS, DE_B_EQS_POD_DACTV_DSO_RECEIVE_DSO, US4G, DTD check, PoD deactivation, BC set, Grid Time Series

Reason and Prerequisites

The error occurred because of missing configuration and required code adjustments in the process.

Solution

Implement the corrections of this SAP Note or import the relevant support package.

After implementing this SAP Note, the system does not deactivate the PoD and does not send a rejection if the DTD check fails.

Activate the BC set /US4G/3_0_SP09_DSO_3714557. You can verify the fix by running the process and confirming that, if the DTD check fails, the system does not deactivate the PoD and triggers a rejection. If you are using the Cloud version, ensure the related emergency patch is in place.

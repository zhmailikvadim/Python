Symptom

In the report /US4G/RP_EQS_ASSGN_AUTH, the mandatory field definition prevents correct use of variants. This affects the processes "Sending of Activation of Assignment Authorization (PRS View)" (DE_B_EQS_ASSAUTH_ACTV_SEND_PRS) and "Sending of Deactivation of Assignment Authorization (PRS View)" (DE_B_EQS_ASSAUTH_DACTV_SEND_PRS).

This issue is relevant for the market role: PRS.

Other Terms

EQS, Settlement, MaBiS, Assignment Authorization, TCode /US4G/EQS_ASSGN_AUTH, Settlement Territory, Settlement Unit, Time Series Type, Start Date, End Date

Reason and Prerequisites

The issue occurs because of a program error.

Solution

Implement the corrections of this SAP Note or import the relevant support package.

After implementing this SAP Note, you can create variants for the report /US4G/RP_EQS_ASSGN_AUTH. The system removes the implicit mandatory field checks for the fields "Settlement Territory," "Settlement Unit," "Time Series Type," and "Start Date/End Date."

You need to save variants for the transaction /US4G/EQS_ASSGN_AUTH as required. You can verify the fix by checking that no issue occurs during variant creation.

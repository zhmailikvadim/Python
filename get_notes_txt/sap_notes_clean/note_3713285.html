Symptom

The description of payment terms is not visible in Ariba during data import from SAP using the ARBCIG_MASTER_DATA_EXPORT program. The description field is empty in the source from which payment terms information is fetched.

Environment

  • Materials Management (MM)
  • SAP ERP Central Component
  • SAP ERP
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA
  • SAP S/4HANA Cloud Private Edition

Reproducing the Issue

  1. Import data from ERP to Ariba using the program ARBCIG_MASTER_DATA_EXPORT.
  2. Observe that the description of payment terms is not visible in Ariba.

Cause

Standard system design. The description is missing in the designated table for payment terms.

Resolution

The logic in FM ARBCIG_PAYMENTTERM_EXPORT states that if the account type of the payment term is not 'VENDOR' (KOART NE 'K'), the system fetches the payment term description from table TVZBT.

If the account type is 'VENDOR', the description is fetched from the T052U table.

To avoid this issue in the future:

  1. Maintain the missing payment term descriptions in the designated table to ensure proper data transfer to Ariba:
    1. Account type is "Vendor"
      Maintain the description in T052U
    2. Account type is not Vendor
      Maintain the description in TVZBT

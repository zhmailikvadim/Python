Symptom

The update of the plan costs statistics with transaction MF27 or report RMSERI20 terminates with runtime error 'SAPSQL_ARRAY_INSERT_DUPREC'.

Other Terms

MF27, RMSERI20, SAPSQL_ARRAY_INSERT_DUPREC, CX_SY_OPEN_SQL_DB, SELECT_SESTA

Reason and Prerequisites

Root cause: No locking concept implemented.

Prerequisites: The transaction MF27 or report RMSERI20 is processed in parallel mode.

Solution

1. Carry out the manual activities.

2. Implement the attached correction instructions.

3. Afterwards, the system will react with an error message if running the report RMSERI20 for the same plant in parallel mode.

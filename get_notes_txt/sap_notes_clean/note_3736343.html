Symptom

  • In the One-Time Deduction portlet Overview section, the Unit of Measure field doesn no appear.
  • The Amount / Percentage / Number of Units field displays values with currency even when the selected Pay Component type is “Number”.

Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.

Environment

  • SAP SuccessFactors Employee Central
  • Deductions

Reproducing the Issue

  1. Navigate to an employee profile.
  2. Open the One-Time Deduction portlet.
  3. Create or view a One-Time Deduction record.
  4. Select a Pay Component of type “Number”.
  5. Save the record and view it in the Overview section.
  6. Observe that the Summary card Amount/Percentage/Number of units includes currency and there is no Unit of Measure

Cause

As designed: The Summary card display for One-Time Deductions in Latest People Profile is hard coded and does not show Unit of Measure, and the value field renders with currency.

Resolution

The fields and rendering in the One-Time Deduction Summary card on Latest People Profile are hard coded and cannot be changed via configuration.

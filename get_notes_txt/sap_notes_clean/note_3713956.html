Symptom

This SAP Note is relevant only for the country version Germany if you use integrated PPA billing.

You execute the program RNCEEB_RECEIVE regularly. During the first program run, messages are retrieved and processed correctly. If no new messages were available during a subsequent program run, all subsequent program runs also no longer retrieve any messages, even if new messages are available.

Other Terms

 eEB, NEEB_S

Reason and Prerequisites

Prerequisite:

  • In the IMG activity SAP Healthcare - Industry-Specific Components for Hospitals -> SAP Ambulatory Care Management -> Treatment -> Outpatient Clinic Forms -> Electronic Outpatient Clinic Forms -> Configure Dispatch, you have maintained entries for different applications for a place of work number. For example, an entry for the Electronic Work Incapacity Certificate (eAU) application and an entry for Electronic Replacement Certificate (eEB).

Reason:

  • Program error - if no messages are retrieved, the status entry in table NEEB_S in field TS_EEB_SINCE was set to 0. This incorrect entry prevents a correct program run.

Solution

Implement the attached correction instructions or import the specified support package.

If an incorrect entry already exists in your system, see SAP Note 3712943 for information about deleting the incorrect table entry.

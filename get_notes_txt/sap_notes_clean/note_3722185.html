Symptom

When tries to replicate the data from non-SAP source system to CFIN target system, even it is already maintained the mappings (such as for Trading Partner)

In the transaction /AIF/ERR, there still exists error FB 279,

Message code: FB279

Message text: Company **** is not defined.

Environment

  • Financial Accounting (FI)
  • SAP S/4HANA 
  • SAP S/4HANA Finance
  • SAP S/4HANA Cloud Private Edition

Reproducing the Issue

Try to replicate the data from non-SAP CFIN source system to CFIN target system.

Cause

The mapping entity "COMPANY_ID", which will be used for Trading Partner mapping, is not supported with the Third-Party/External Interface.

Resolution

In the following help document,

MDG Data Mapping

Section: "Mapping Entities Supported by Third-Party System Interfaces to Central Finance"

It describes the scenario for Third-Party/External Interface, there are restrictions for the available / supported Mapping Entities.

In general, the related Mapping Entity for Trading Partner is "COMPANY_ID", which is not in the list for supported ones.

As such, it will not be picked up in standard mapping for the External Interface,,

Because of it not being considered, the mapping does not happen and so the validation on FI side will happen and cause the error message FB279,

  

For such scenario, please discuss with local consultants and consider possible custom solutions.

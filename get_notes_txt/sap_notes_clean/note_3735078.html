Symptom

When attempting to pack a Handling Unit (HU) in an outbound delivery (VL02N), the system issues message HUGENERAL290 and does not allow the HU to be assigned.

Messages occurred - see log (separate session)

Message no. HUGENERAL290

Diagnosis

If there are already 6 sessions open, the system does not issue an error log.

Environment

SAP S/4HANA Cloud Public Edition

Reproducing the Issue

  1. Maintain stock in a HU-managed storage location
  2. Ensure Handling Units exist for the stock
  3. Create an outbound delivery (sales order or STO-based)
  4. Open delivery in VL02N
  5. Attempt to pack or assign an existing HU
  6. Enter HU number and confirm

Cause

The behavior might be caused by an active Handling Unit Physical Inventory document for the storage location. While such an inventory process is open, the system restricts any logistical movements involving the affected HUs.

Resolution

It is necessary to identify whether a Handling Unit Physical Inventory document is active for the storage location. Once identified, the inventory process must be completed by posting any differences and closing the document. After the physical inventory is finalized, the system removes the restriction, and the Handling Units can be successfully packed into the outbound delivery.

Symptom

A specific document number is not displayed in transaction AW01N, although:

  • the document was posted to the same asset as other documents,

  • other postings (for example from later fiscal years) are visible in AW01N,

  • the asset itself can be displayed without errors.

This situation typically occurs when reviewing historical postings and leads to the assumption that a reporting or system issue exists in AW01N.

Environment

  • FIN Asset Accounting
  • SAP ERP
  • SAP ERP Central Component
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA
  • SAP S/4HANA Finance
  • SAP S/4HANA Cloud Private Edition
  • SAP Fiori for SAP S/4HANA Finance,
  • SAP Fiori for SAP S/4HANA,
  • SAP Fiori for SAP S/4HANA Cloud Private Edition

Reproducing the Issue

  1. Open transaction AW01N.

  2. Display an asset with multiple postings across different fiscal years.

  3. Observe that some document numbers (for example from later fiscal years) are displayed.

  4. Observe that a specific document (for example from an earlier fiscal year) is not displayed in AW01N, although it was posted to the same asset.

Cause

AW01N is not a general document display.
The report shows only postings that are Asset Accounting–relevant and were successfully posted further to Asset Accounting.

A document will not be displayed in AW01N if, for example:

  • it was posted in FI but not capitalized or posted further to the asset,

  • it was not Asset Accounting–relevant (posting type not relevant for FI-AA),

  • it exists only in FI and never reached Asset Accounting,

  • it was reversed or posted as a statistical/technical entry,

  • it originates from an older fiscal year (for example before migration or with different customizing).

This behavior is by design.

When is a document displayed in AW01N?
A document is shown in AW01N only if:

  • it was posted further to the asset (for example, capitalized),

  • it was Asset Accounting–relevant,

  • it was not reversed or posted as a purely technical/statistical entry.

In this case, the posting appears as a movement in AW01N.

Resolution

Verify the original posting of the document that is not displayed in AW01N:

  • Check whether the document was capitalized or posted further to Asset Accounting.

  • Check the posting type and its relevance for FI-AA.

  • Check whether the document was reversed.

  • For older fiscal years, consider possible differences in customizing or migration history.

If the document was not Asset Accounting–relevant or not posted further, it is expected that it does not appear in AW01N.

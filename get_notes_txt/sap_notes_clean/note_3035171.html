Symptom

  • Report schedule sent via SFTP is set to XLSX but file delivers as CSV (or vice versa)
  • Report file is corrupted when it gets to my SFTP site

Resolution

When a report is set to deliver as SFTP, some details such as file name and extension, are also maintained through an internal end point

  • End points can be created by Configuration Manager users or by SAP Fieldglass when requested (SFTP delivery request)

If an update is made on the report schedule to change the file extension such as from CSV to XLSX, the change needs to be replicated to the end point, otherwise the file will be likely be corrupted once it arrives to your SFTP site due to conflicting information

To update the file extension for SFTP report schedules:

  • Update the report schedule to the new desired file extension
  • Submit a case to SAP Fieldglass Support. Include your username, report name, report schedule, and what should be the new extension

Symptom

Reusing a handling unit within the same warehouse that already has print history with old content leads to issues. The handling unit was previously in Deleted status in EWM and contains PPF log entries in table /SCWM/HU_PPF. When the same handling unit number is reused with new content, PPF is successfully initiated, but the spool is not generated, and print output is not initiated.  

SLG1 logs show the following messages:  

  • Condition &1 found for action &2
  • Action summarization: Only one action permitted
  • There Is Already At Least One Action

Environment

  • SAP S/4HANA
  • SAP Extended Warehouse Management

Reproducing the Issue

  1. Log on to RFUI.
  2. Enter Warehouse, Resource and Difference Device.
  3. After login, select printer.
  4. Click F1 – Next.
  5. Navigate to RF Menu 41 → Staging Process → Repacking / Consolidation.
  6. Select Work Center and press Enter.
  7. Enter Source HU and press Enter.
  8. Select Pack Material
  9. Enter Destination HU
    On the next screen, click F1 – Pack; a success message “Handling unit created successfully” is displayed.
  10. Press Enter, then click F4 – Close.
  11. Enter handling unit and click Close; the message “Handling unit successfully closed” is displayed.
  12. Observe Successful PPF trigger in SLG1

The Issue may occur in different transaction and other documents printing in EWM.

Cause

Action merging of the PPF is set to ‘EWM: Max. 1 Action for Each Action Definition’.  

Resolution

To resolve the issue change the Action merging of the PPF to ‘EWM: Max. 1 Unproc. Actn for an Actn Def., Do not Del + lock’

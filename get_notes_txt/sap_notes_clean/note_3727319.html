Symptom

Information about Forecast Consumption in background. 

Environment

  • SAP Supply Chain Management (SCM)
  • SAP Supply Chain Management > Advanced Planning and Optimization (SCM-APO)
  • SAP Supply Chain Management > Advanced Planning and Optimization > Forecast Consumption (SCM-APO-FCS-CSP)
  • SAP S/4 Hana with Embedded PP/DS

Reproducing the Issue

In APO: 
SPRO -> APO -> Demand Planning -> Basic Settings -> Consumption -> Set Consumption Type = 'Consumption in Background' 

In S/4HANA: 
SPRO -> Advanced Planning -> Set Consumption Type = 'Consumption in Background' 

Resolution

In background consumption mode, pegareas (IDs for location products) are collected from the sales order CIF queues. These do not trigger forecast consumption updates in this mode. The report /SAPAPO/DM_ALLOCATION_BTC_RFC is responsible for triggering forecast consumption for the collected pegareas, thereby updating the forecast consumption information in the Orders LiveCache.

Within the product view (/SAPAPO/RRP3), the Elements tab displays the forecast consumption situation by reading data directly from the Orders LiveCache. As a result, the Elements tab may show an outdated version of the forecast consumption situation if the report /SAPAPO/DM_ALLOCATION_BTC_RFC has not yet been executed.

On the other hand, the Forecast tab (including both the Overview and Details sub-tabs) displays a simulated forecast consumption situation. This simulation calculates the consumption situation dynamically, ensuring that the information shown is always up to date.

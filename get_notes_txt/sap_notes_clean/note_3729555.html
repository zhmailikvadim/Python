Symptom

  • After importing videos via Workspace Import, video thumbnails (or preview images) displayed in video tiles appear broken, missing, or not rendered correctly. Instead of the expected preview image, users may see a blank placeholder or a broken image icon.
  • Users can still access and play the videos without any issues.

Environment

  • SAP SuccessFactors Work Zone
  • SAP Build Work Zone Advanced Edition

Reproducing the Issue

The video was uploaded into the system via a workspace import using a ZIP file.

Cause

The issue was caused by a bug related to FFmpeg, a third-party component used to extract video thumbnails.

The system previously used a component called FFmpeg to generate video thumbnails on the server side. Due to compliance requirements, this component has been removed, and thumbnails are now typically generated by the browser when a video is viewed.

However, when a workspace is imported, thumbnail generation is handled by a background process instead of the browser. In this case, the system would normally rely on an alternative internal service to generate the thumbnails.

This alternative service is currently disabled, which prevents the system from generating video thumbnails during the workspace import process.

Resolution

Our Engineering team has developed a code fix and is planning to deliver it by 27 May 2026. This correction will apply to newly imported workspaces.
For videos with missing thumbnails in previously imported workspaces, we recommend uploading a new version to trigger automatic thumbnail generation.

Symptom

User has a SAPUI5 application and they wish to use device camera to capture images on Android, iOS, and Windows with Fiori Client and browser.

Environment

  • SAPUI5 1.38.16
  • iOS iPad and iPhone
  • Android tablet and phone
  • Windows desktop and tablet
  • Fiori Client

Resolution

1.  The Fiori Client on iOS and Android uses the Cordova camera plugin to capture images from the camera.  See details at:

https://blogs.sap.com/2017/02/08/appendix-e-non-kapsel-plugins/

2.  On the browser, the issue is the Microsoft x-ms-webview.  SAPUI5 natively does not provide camera functionality and that has to be provided by the plugins of the Kapsel (Fiori Client) and Cordova browser.  The webview of the iOS (UIWebview) and Android (Crosswalk) works with the Cordova Camera plugin that accesses the camera.  On Windows this can be done in Chrome/Edge/Firefox browser via a HTML Tags.  You can review this at the following page that demonstrates how to create screen shots with html5 “video” tag.

https://www.kirupa.com/html5/accessing_your_webcam_in_html5.htm

This works fine in Edge, Firefox, and Chrome.  This does not work on Internet Explorer and in Fiori Client since they use the Microsoft x-ms-webview as x-ms-webview which doesn’t support these tag.  Logically, a user could possibly use deep links to call the browser (ie Chrome), take a picture, store locally, and return to Fiori Client, but this could pose security risks.

Symptom

There is a condition with a BRF+ decision table with maintained entries.
You copy the condition and navigate to DAD, then back (without saving).
The already copied decision table is lost.

Expected behaviour: When navigating to DAD without saving the changes (the creation of the decision table), an error message is expected to be displayed to block navigation and force the user to save the data.

Reason and Prerequisites

Program Error.

Solution

Implement the attached correction instructions or install the corresponding Support Package.

Symptom

While importing material data using the Import Data for Consolidation app, column misalignment occurs when a preceding field is left blank, causing the system to take values from subsequent columns and shift all data, leading to import failure.

Environment

  • SAP Master Data Governance - Consolidation and Mass processing
  • SAP S/4HANA Cloud Public Edition

Reproducing the Issue

  • Login to Fiori.
  • Open the Import Data for Consolidation app.
  • Leave one or more fields blank in preceding columns of file.
  • Upload the file.
  • Observe that:
    • Values shift to adjacent columns.
    • Data becomes misaligned.
    • Import fails.

Cause

This is caused by unused or empty columns in the Excel file being included in the sheet

Resolution

  • Use a correctly generated template file:
    • Download via Export Data app or use the Full Template from the app.
  • Adjust the Excel file before upload:
    • Hide unused columns (e.g., optional columns not maintained).
    • Ensure no unnecessary empty columns exist within the used range.
    • Alternatively, maintain a default value (e.g., “0”) in empty mandatory fields.

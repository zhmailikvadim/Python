Symptom

When adding settings to the CONNECTORS configuration file and saving the changes, the place where some of the settings were originally added changes and it's causing confusion for Learning Administrators.

  • Is there any rule that determines the display order or grouping of rules within the CONNECTORS configuration file?
  • And why is this setting displayed somewhere else when looking at the changes through the 'View Configuration Changes' option?

Environment

SAP SuccessFactors Learning

Reproducing the Issue

  1. Log into HCM > navigate to Learning Administration >
  2. At System Administration > Configuration > System Configuration >
  3. Open the CONNECTORS configuration in edit mode >
  4. Under the existing settings for any connector, add a new line with the setting to be added (for example: for the section that lists the settings for the User Connector, add the setting user.field.transform.data.SUPER.@NULLIFY@=NO_MANAGER) >
  5. Upon 'Apply Changes', note that the new setting added gets rearranged, getting displayed in the section of another connector >
  6. And when looking at the changes through the 'View Configuration Changes' option, then the setting is displayed somewhere else (for example: when saving the setting from step 4 above, it gets places almost at the end of the CONNECTORS configuration. But when looking the 'View Configuration Changes' page, the setting is displayed almost at the top) >
  7. Why does the above behaviors happen?

Cause

As per our Engineering team, there is a logic in place which decides how the connector settings appear after saving them.

Regarding the logic within the CONNECTORS configuration file, after saving the changes made to settings:

  1. Base configurations: Refers to the properties which are already available in the CONNECTORS configuration by default;
  2. Overridden configurations: Refers to the properties which are not available in the CONNECTORS configuration by default, and are added by the customer.

For example, let's consider the property user.field.transform.data.SUPER.@NULLIFY@, which is not present at the CONNECTORS configuration by default. Considering that the setting is getting added close to the existing User Connector settings:

  • user.field.mapping.data.NOTACTIVE=NOTACTIVE
    ....
    user.field.transform.data.SUPER.@NULLIFY@=NO_MANAGER
    ...
    ...
    accountcode.field.mapping.data.REMOVE_ACCT=REMOVE_ACCT (last line of the default CONNECTORS configuration)

After saving the changes, the base configurations are grouped together and the overridden configurations are placed at the end of the CONNECTORS configuration file. And within the overridden configs, the property which was overridden last comes at first. Considering the example, the configurations will get arranged as below:

  • user.field.mapping.data.NOTACTIVE=NOTACTIVE
    ...
    ...
    ...
    accountcode.field.mapping.data.REMOVE_ACCT=REMOVE_ACCT (last line of the default CONNECTORS configuration)
    user.field.transform.data.SUPER.@NULLIFY@=NO_MANAGER

Now, regarding the logic of the 'View Configuration Changes' page, this is the order in which configurations are shown in this page:

  1. Base configurations: Refers to the properties which are already available in the CONNECTORS configuration by default;
  2. Base overridden configuration: Configurations which are available in the CONNECTORS configuration but their mapping has been changed;
  3. Overridden configurations: Refers to the properties which are not available in the CONNECTORS configuration by default and are added by the customer.

For example, let's keep considering the setting user.field.transform.data.SUPER.@NULLIFY@, which is not present at the CONNECTORS configuration by default:

  • user.field.mapping.data.NOTACTIVE=NOTACT (base configuration, being overridden)
    ...
    user.field.transform.data.SUPER.@NULLIFY@=NO_MANAGER
    ...
    ...
    accountcode.field.mapping.data.REMOVE_ACCT=REMOVE_ACCT     (last line of the default CONNECTORS configuration)

After saving the changes and opening the 'View Configuration Changes' page, the order seen will be the following:

  • ...
    ...
    ...
    accountcode.field.mapping.data.REMOVE_ACCT=REMOVE_ACCT (last line of the default CONNECTORS configuration)

    Your Configuration Changes

    user.field.mapping.data.NOTACTIVE=NOTACT (Base overridden configuration)
    user.field.transform.data.SUPER.@NULLIFY@=NO_MANAGER 

Resolution

As per Engineering team, this logic is the application's current expected behavior. To suggest a different system behavior, it would be possible to have an enhancement request created for it (as per KBA 2090228), where the Learning Product Management team can further review and assess the requested functionality to be considered in a future release.

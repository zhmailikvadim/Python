Symptom

The on-premise system mapped in the SAP Cloud Connector (SCC) cannot be reached. This may cause malfunctions in applications that rely on SCC to connect to on-premise systems.

In the SCC traces (ljs_trace.log for versions <2.17 or scc_core.trc for versions ≥2.17), located in ../<SCC_Install_Folder>/log, the following errors may appear:

+0000#ERROR#com.sap.core.connectivity.spi.processing.AbstractProtocolProcessor#tunnelclient-84-2# #Unable to open connection to backend system: java.nio.channels.UnresolvedAddressException

Or

+0000#ERROR#com.sap.core.connectivity.spi.processing.AbstractProtocolProcessor#tunnel-client-86-1#0xac546f25#Unable to open connection to backend system: <internal_hostname>
java.net.UnknownHostException: <internal_hostname>

Environment

SAP Cloud Connector

Reproducing the Issue

  1. Login to Cloud Connector Administration UI (https://<cloud_connector_host>:8443/).
  2. Select the desired "Subaccount".
  3. Navigate to: "Cloud To On-Premise" -> "ACCESS CONTROL".
  4. Add a new system mapping by clicking on button "+" (Add). 
  5. Check the availability of the newly added system.
  6. The check result "Not Reachable" is observed.

Cause

The exception above indicates that the network could not resolve the hostname the Cloud Connector was attempting to connect to. This exception is not raised by the SCC itself; it originates from a lower software layer (the JRE) that provides TCP/IP connectivity APIs to the SCC. According to Java documentation, UnresolvedAddressException is an unchecked exception thrown when an attempt is made to perform a network operation on an unresolved socket address.

In this case, the internal hostname cannot be resolved to an IP address, typically due to a typo or incorrect entry in the maintained hostname details.

Resolution

  1. Verify the hostname for typos and correct it if necessary.

  2. Contact the network or basis team to confirm that the hostname the SCC is trying to connect to is reachable and can be resolved from the SCC server.

Symptom

  • Internal session terminated with runtime error "GETWA_NOT_ASSIGNED".
  • The runtime error occurs during a scheduled billing job.
  • The same dump occurs during collective billing via transaction VF04.
  • The dump is visible in ST22.
  • ST22 points to ABAP program/class CL_SDBIL_VCM_UTIL=============CP and method IF_SDBIL_VCM_UTIL~FILTER_MSICO_DOCUMENTFLOW.

Environment

•Sales And Distribution (SD)
•Logistics Execution (LE)
•SAP R/3 •SAP R/3 Enterprise 4.7
•SAP ERP Central Component
•SAP ERP
•SAP Enhancement package for SAP ERP
•SAP Enhancement package for SAP ERP, version for SAP HANA
•SAP S/4HANA
•S/4HANA Cloud Private Edition

Reproducing the Issue

  1. Start the scheduled billing job (monitor via transaction SM37).
  2. Observe that the job terminates.
  3. Open ST22 and review the dump "GETWA_NOT_ASSIGNED".
  4. Alternatively, run collective billing via transaction VF04.
  5. Observe the same dump "GETWA_NOT_ASSIGNED" in ST22.

Cause

  • After upgrading to S4CORE 109, a required correction is missing. The absence of SAP Note 0003664597 causes the runtime error.

Resolution

  1. Review ST22 to confirm the runtime error "GETWA_NOT_ASSIGNED" in CL_SDBIL_VCM_UTIL and method IF_SDBIL_VCM_UTIL~FILTER_MSICO_DOCUMENTFLOW.
  2. Verify the system is on S4CORE 109 and check whether SAP Note 0003664597 is already implemented.
  3. Implement SAP Note 0003664597 using the standard note implementation procedure (for example, via transaction SNOTE).
  4. Retest the scheduled billing job and collective billing (VF04) to confirm the dump no longer occurs.

See Also

refer to: SAP Note 0003664597

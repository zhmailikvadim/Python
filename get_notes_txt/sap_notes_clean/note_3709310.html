Symptom

  • A runtime dump is triggered when attempting to receive handling units against a production order in transaction /SCWM/RFUI. 
  • The error message displayed is: "Number range interval for handling unit identification is not defined (/SCWM/HU_WM044)".
    Dump:
    Category               ABAP programming error
    Runtime Errors         ASSERTION_FAILED
    ABAP: Program          /SCDL/CL_BO_MANAGEMENT========CP
    Application Component  SCM-EWM-DLP

Environment

  • SAP Extended Warehouse Management
  • SAP S/4HANA

Reproducing the Issue

  1. Access transaction /SCWM/RFUI.
  2. Enter the following details:
    Warehouse Number, Resource and Default Device
  3. Navigate through RF menu options: 3(Inbound Processes)-4(Receiving of Handling Units)-6(Rec. HU by manufacturing Order)-4(Rec. HU by Order).
  4. Select Work Center.
  5. Enter Manufacturing Order.
  6. Attempt to create handling units for the production order.
  7. Observe the dump occurrence with the mentioned error message.

Cause

The packaging material type assigned to the handling units did not have a number range interval defined in the SPRO configuration under "Assign Number Range Intervals to Packaging Material Types".

Resolution

  1. Verify the packaging material type assigned to the handling units.
  2. Navigate to SPRO configuration: "Assign Number Range Intervals to Packaging Material Types".
  3. Assign the appropriate number range interval (e.g., interval 02) to the packaging material type.
  4. Test the receiving process via /SCWM/RFUI transaction to confirm resolution.

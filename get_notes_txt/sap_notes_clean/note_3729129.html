Symptom

When using the "Copy Item Type and Item Category" tool to create new entries, the program automatically generates customizing transport request entries for the newly created item types/categories.However, the tool adds table key entries to the transport by concatenating all table fields while removing necessary trailing spaces.

As a result, the generated table keys do not match the required format for the table entries. Consequently, the new item category/type data fails to be moved to the target systems during the transport process. In the transport request's 'Table Keys' view, the keys appear compressed or shorter than the technical definition of the table fields.

Other Terms

Class: /PAT/CL_COPY_ITEM_TYPE_CAT, Methods: MODIFY_ITEM, SET_REQUEST_DATA, START_OPERATION

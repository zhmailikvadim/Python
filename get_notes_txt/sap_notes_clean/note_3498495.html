Symptom

This document provides a similar logic that can be applied to the extractor in order to extract data multiple times a day using DataSource 0FI_GL_6.

Environment

ERP All Versions

S/4HANA All Versions

Cause

0FI_GL_6 does not support minute-based extraction the same way as 0FI_GL_4 does, although it uses BWFI_AEDA2.

0FI_GL_6 uses a different function module, so it completely ignores BWFINEXT parameter from the table BWOM_SETTINGS.

BWFI_AEDA2 contains the field AETIM, it's not used in Delta extraction for 0FI_GL_6.

But this shouldn't be a problem in practice in case of several delta per day, because normally the data volume (of highly aggregated GLT0_AEDAT) in delta mode should not be too high.

Resolution

To extract delta multiple times daily with 0FI_GL_6, you need to use the parameter 'BWFIOVERLA' in the table BWOM_SETTINGS.

This information is explained in consulting Note: 1012874 - FAQ extractors 0FI_AP_* 0FI_AR_* 0FI_GL_* (except 0FI_GL_10, 0FI_GL_11, 0FI_GL_12, 0FI_GL_14) (Relevant part: 'BWFIOVERLA').

Please note that using this parameter may impact performance, depending on the volume of data being loaded.

We recommend testing it extensively internally to identify any potential performance issues with the new setting.

Symptom

Issue : When location data changed at the top FLOC using transaction IH01, its lower structure changes were not getting synchronized due to change timestamp check at the inbound side of Functional Location.

Solution :

With this NOTE, change timestamp check for FLOC is controlled by a switch maintained in table /ISDFPS/CDFPSK1

Field             : Value
DFPS_KEY     : FLOC_TMSTMP_CHK
DFPS_VALUE : X (When X then date/time stamp check happens. If blank then no date/time stamp check)

Other Terms

Defense disconnected S4, Floc, IH01

Solution

Implement the attached correction instructions provided with the NOTE.

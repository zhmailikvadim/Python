Symptom

For certain intercompany billing processes, the goods movement status of the delivery changes from "C" to "A" after an ICB invoice/credit memo has been created.

Environment

  • Sales And Distribution (SD)
  • Logistics Execution (LE)
  • SAP R/3
  • SAP S/4HANA
  • SAP R/3 Enterprise 4.7
  • SAP ERP Central Component
  • SAP ERP
  • SAP Enhancement package for SAP ERP
  • SAP Enhancement package for SAP ERP, version for SAP HANA

Reproducing the Issue

  1. Create a Returns Delivery, assign a batch, and perform Post Goods Receipt. At this stage, the Goods Movement status is correctly "C" (Completed).
  2. Create an Internal Credit Memo with reference to the Returns Delivery.
  3. Upon checking the Returns Delivery status again, the Goods Movement status has reverted to "A" (Not yet started).

Cause

To ensure that, in the case of a stock transfer, goods movements are not taken into consideration twice, only the goods issue is taken into consideration when creating intercompany billing (document type IV) for deliveries, and only the goods receipt is taken into consideration when creating an internal credit memo (document type IG) in the case of returns deliveries.
When a delivery document contains a mixture of returns items and delivery items, this causes the goods movement status to be processed incorrectly

Resolution

Review and implement note 2986971 :  Goods movement status changed by creation of ICB invoice

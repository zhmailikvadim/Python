Symptom

In some early ECC release, JSON serializer could not be used due to basis component missing. In this case, built-in XML serializer will be used instead.

Solution

Please implement this note to resovle this issue. This Note applies for Central Finance source system.

For example:

Settlement date is 01.01.2005. On this date you run settlement for the period between 31.12.2004 and 01.01.2005. In this case, the following will be considered:

1. Number of days will be based on the date on which interest is calculated, which in this case is 365, as the date is 01.01.2005. Previously, system would have taken the number of days in the previous year. Since the previous year (2004) was a leap year, system would have taken 366 days as the number of days.

2 The balance for interest calculation will be the incoming balance as of 01.01.2005 (no change from the previous calculation method)

3. Rate of interest applicable will be the rate as of 31.12.2004 (no change from the previous method)

Note that the interest calculation method should be act/acty (ISDA)

Other Terms

Leap Year, Russia, Interest Calculation, Act/ActY (ISDA), Accrual, Time Deposit

Symptom

  • When the "Ignore Initial Values" flag is active in transaction /SAPAPO/TSCUBE, input records with data equal to 0 are ignored during data load.
  • Existing values in the target key figure are retained instead of being replaced by 0.
  • Records with values smaller or greater than 0 are loaded as expected.

Environment

SAP Supply Chain Management (SCM)
SAP Advanced Planning and Optimization (SCM-APO)
SAP Enhancement package for SAP Supply Chain Management

Reproducing the Issue

  1. In transaction /SAPAPO/TSCUBE use a variant where flag "Ignore Initial Values" is active in Additional Settings.
  2. Select a combination where the source cube contains zero values for the source key figure.
  3. Run the data load.
  4. Observe that existing values in the planning area target key figure are not replaced by 0; only values smaller or greater than 0 are transferred.

Cause

  • This is as per design. If flag "Ignore Initial Values" is active, zero values can be treated as initial and therefore skipped during data load, so existing target key figure values are retained. 
  • Note: A 0 value in the source cube might be a real 0 value - or maybe the source key figure has no value for the period/combination.
  • Please see the detailed explanation for limitations on BW side in SAP Note 1548325 - Extraction: distinction between zero and initial. It explains limitation for data extraction and reload related to zero and initial values.

Resolution

  1. Before the data load using /SAPAPO/TSCUBE run report /SAPAPO/RTSINIT_KF or transaction /SAPAPO/TSINIT_KF with a variant using the same selection and grouping condition as in transaction /SAPAPO/TSCUBE to initialize the target key figure values in the target period. This will set initial values.
  2. If the target key figure setting does not allow zero ( 0 = initial) then the expected result is achieved. If the key figure setting allows zero (0 <> initial) then there will be initial values instead of the zeros because /SAPAPO/TSINIT_KF also clears 0 values.
  3. If process chains are in use, insert a TSINIT_KF process (it has its own process type) before the data load.

See Also

SAP note 1548325- Extraction: distinction between zero and initial

Keywords

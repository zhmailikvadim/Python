Symptom

When a grid is configured within a nested table in the Questionnaire Painter, and a show/hide rule is configured for the grid and grid columns through Content Display Rule, the Content Display Rule is not respected when the screen is loaded.

  • On initial load, the grid and grid columns will show.
  • On save of screen, the grid container will show, but the grid columns and values will disappear. The values that have been saved in the grid are persisted to the database.
  • On refresh of screen, the grid container, grid columns, and values which were previously saved will show.

Other Terms

grid, showhide, show/hide, disappear, values, hidden, JSONScreenController

Reason and Prerequisites

Prerequisites:

A grid is configured within a nested table in a Questionnaire View for which a Show/Hide rule is configured for the grid and grid columns. Please follow the instructions below to configure if required.

  1. Find any Existing Screen in FS-QUO/FS-IPW, find the corresponding Screen in FS-PRO Product Configurator.
    • Example: Sample Household Product, Household Policy Screen (Sample Household Product -> User Interface -> Data Capture -> eApplication(Producer Policy eApp PQM30) -> eApp Section (Policy Data) -> eApp Screen (Household Policy).
  2. Open the Questionnaire Layout Painter by clicking the Screen Icon and Selecting "Edit". Find a corresponding Questionnaire Layout to modify.
    • Example: HouseHold Policy View PQM30 in Sample Household Product -> User Interface -> Sample Household Policy Model View -> Views -> SAP Household Policy View.
  3. Find an existing Questionnaire to add a grid and 2 or more columns to.
    • Example: Sample Household Product -> User Interface -> Sample Household Policy Model View -> Questions -> SAP Household Policy Questions.
  4. Open the Questionnaire Painter from the View and select the Questionnaire View to modify from the View Dropdown.
    • Example: HouseHold Policy View PQM30.
  5. Add a new row to an existing Questionnaire View. In the new row insert a table with 1 column and 1 row. Within the new table, add a grid created from Step 3.
  6. Add a new row to an existing Questionnaire View. In the new row, insert a radio button for an arbitrary column where the length = 1.
    • Example: ABDAPOLICY.ICC_CD
  7. Save and Build the Questionnaire View. Close the Questionnaire Painter.
  8. In the eApp Screen from Step 2, create or edit the Content Display Rule. Within buildShowString/buildHideString function, add the corresponding code:

    • buildHideString:
      IF ABDAPOLICY.ICC_CD = "0"
      HIDEReturnString = HIDEReturnString + "<GRID_ID>, <COLUMN_ID1>, <COLUMN_ID2>, etc"
      END IF

    • buildShowString:
      IF ABDAPOLICY.ICC_CD = "1"
      SHOWReturnString = SHOWReturnString + "<GRID_ID>, <COLUMN_ID1>, <COLUMN_ID2>, etc"
      END IF
  9. Save and Build the Rule.
  10. Publish and Deploy the Product.

Steps to Reproduce:

  1. Log into FS-QUO/FS-IPW.
  2. Navigate to the eApp screen which was modified in pre-requisite.
  3. Click the new Radio Button to set the value to "Yes". The newly configured grid should show.
  4. Enter Data for the columns in the grid. Click Save on the Screen.

Actual Result: After the Save has completed, the grid container remains but the grid columns and column values disappear to show an empty grid.

Expected Result: After the Save has completed, the grid container, grid columns, grid values should persist on the screen.

Cause: The application code will parse the Questionnaire Metadata to grids within a Table -> Row -> Cell - > Grid configuration. However, the code does not consider the case where a table can be configured within an existing table as a Table -> Row -> Cell -> Table -> Row -> Cell -> Grid.

Solution

Recursively find grids within nested Questionnaire Tables to ensure that all grids are identified regardless of how many nested tables are configured.

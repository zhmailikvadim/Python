Symptom

For secure HTTP access, you may need to generate a CSR (certificate request) file than contains SAN information (Subject Alternate Name) using keytool.

Environment

  • SAP IQ 16
  • SAP IQ 16.1

Resolution

Here are the steps for generating the CSR.  Replace the information passed in parameters with your own relevant information:

  1. Run keytool with the "genkey" parameter.
    $ keytool -genkey -alias XXXXD0074 -keypass nosecret -keyalg RSA -sigalg SHA1withRSA -keystore server.keystore -storepass nosecret -validity 3650 -ext SAN=dns:test.example.com
    What is your first and last name?

    What is the name of your organizational unit?
    [Unknown]: sap
    What is the name of your organization?
    [Unknown]: sap
    What is the name of your City or Locality?
    [Unknown]: sap
    What is the name of your State or Province?
    [Unknown]: sap
    What is the two-letter country code for this unit?
    [Unknown]: sap
    Is CN=sap, OU=sap, O=sap, L=sap, ST=sap, C=sap correct?
    [no]: yes
  2. Run keytool with the "certreq" parameter.
    $ keytool -certreq -alias XXXXD0074 -keystore server.keystore -file cert_request.csr -ext SAN=dns:test.example.com
    Enter keystore password:  [enter password provided above]
  3. See the resulting files generated.
    $ ls
    cert_request.csr server.keystore

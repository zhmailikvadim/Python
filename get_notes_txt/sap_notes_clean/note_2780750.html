Symptom

After installing Dynamic Tiering component on HANA 2.0, log backup using Backint against esserver is not working, whereas Full/Delta Backups are running fine.

Environment

SAP HANA 2.00.033.00
Data Protection for SAP HANA(R) 8.1.0.0

Reproducing the Issue

Issue the log backup using backint against the esserver.

Cause

For SAP HANA 2.0 SPS 02 and later releases, SAP HANA raises alerts when the dynamictiering log area space for point-in-time recovery falls below a low threshold.
The log areais under the directory basepath_logvolume_es/subpath/eslog.
This alerts mean that the area is out of space.

Resolution

To resolve the PITR out-of-space condition, it is required to add the ES_LOG_THRESHOLD_SIZE configuration parameter to esserver.ini.
Please refer to the "Dynamic Tiering Service Properties" in References for details about extended storage configuration properties including this parameter.

If it has been already existed, then

1.Check the value of the ES_LOG_THRESHOLD_SIZE configuration option. If it is set to a much larger value than the typical PITR log volume, try adjusting it to a lower value that can still accommodate the typical PITR log volume.
2.If system replication is turned ON, then the log space size is governed by the value of the logshipping_max_retention_size option. To free up space, lower the value of this option.

If these steps above fail to resolve the out-of-space condition, please refer to "Log Area for Point-in-Time Recovery Out of Space" in References.

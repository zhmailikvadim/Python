Symptom

You create withholding tax accounting for Switzerland using the HR form SAP_QSTC0_CH01 (transaction:  PC00_M02_HRF_QSTC0).

Persons of all payroll units are displayed in a sequential list. The totals are calculated globally across all payroll units.

Other Terms

Withholding tax, WHT accounting, payroll unit, payroll number, HRFORMS, SAP_QSTC0_CH01, CL_HRPAYCH_HRF0_QSTC0, SSL number, ABRNR, T5C2Q, ABREH, ABREINHEIT, page break

Reason and Prerequisites

Up to now, the data structure HRPAYCH_HFQST_S_FORM contains only a flat table SAL (all salary lines of all payroll units) and a global total table SAL_TOT with exactly one record TOTAL_A/B/C. The HRFORMS form outputs this flat table sequentially without grouping by payroll unit.

A grouping level is missing in the form output structure that allows the form to generate separate pages with separate totals for each payroll unit.

Solution

Import the HR Support Package relevant for your release. If you implement the solution in advance, use the Note Assistant to implement the correction instructions relevant for your release. You can find more information about the Note Assistant on the SAP Support Portal under https://support.sap.com/notes.

A new field HRPAYCH_HFQST_S_FORM' is added to the form output structure ABREH_DATA', which groups the salary data and totals per payroll unit. The HRFORMS form is adjusted in such a way that separate pages with the corresponding SSL number, separate detail lines, and separate totals are output for each payroll unit. The SSL number belonging to the payroll unit is determined using the table/view V_T5C2Q.

In addition, this correction enables the multiple selection of payroll units on the selection screen of the print program.

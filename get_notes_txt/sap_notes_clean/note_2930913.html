Symptom

Information on the installation of SAP Best Practices for S/4HANA (on premise) (Singapore) (SGV7)

Other Terms

Best Practices, S/4HANA

Reason and Prerequisites

This note is only relevant for the following SAP Best Practices version: SAP Best Practices for S/4HANA (on premise) (Singapore) (SGV7)

Solution

This SAP Note contains additional information on the installation and
documentation of the SAP Best Practices for S/4HANA (on premise) (Singapore) package

1. Software and Delivery Requirements and Administration guide for SAP S/4HANA solution implementation
Before you start the implementation of the SAP Best Practices for S/4HANA (on premise) (Singapore) package read carefully the following documents:

  • Software and Delivery Requirements documentation: https://support.sap.com/content/dam/SAAP/Sol_Pack/BP_OP_ENTPR/BP_OP_ENTPR_S4HANA2020_07_Requirements_EN_XX.docx

  • Administration Guide for SAP S/4HANA OP Solution Implementation: https://help.sap.com/viewer/S4HANA2020_AdminGuide

2. Installation Preparation

2.1 SAP NOTES
Ensure that the following SAP Notes are implemented in addition to the notes already listed in the Software and Delivery Requirements document or listed in the Administration Guide for SAP S/4HANA OP Solution Implementation or listed in the FPS specific note that is listed in the collective note 2914416 "SAP S/4HANA, on-premise edition 2020 collective note for content activation": None.


2.2 Correction transport
All corrections will be offered as a collective transport request which will be attached to a FPS specific note that is listed in the collective note 2914416 "SAP S/4HANA, on-premise edition 2020 collective note for content activation".


2.3 Installation: Trouble Shooting (errors that might occur during activation)
See the Administration guide for SAP S/4HANA solution implementation for information on general error handling. It is very important to read, understand and follow the generic guidelines on how to handle known activation errors.

Detailed descriptions of workarounds per known activation error can either be found in a FPS specific note that is listed in the collective note 2914416 "SAP S/4HANA, on-premise edition 2020 collective note for content activation" or are attached to this note (see documents 2020_OP_nnn_XX.docx - nnn is the Building Block ID).

3 Usage
3.1 Errors in Activated SAP Best Practices S/4HANA on premise edition (Singapore) package due to Wrong/Missing Settings
See the Administration guide for SAP S/4HANA solution implementation for information on general error handling. It is very important to read, understand and follow the generic guidelines on how to handle known errors.

Detailed descriptions of workarounds per known usage error can either be found in a FPS specific note that is listed in the collective note 2914416 "SAP S/4HANA, on-premise edition 2020 collective note for content activation" or are attached to this note (see documents 2020_OP_nnn_XX.docx - nnn is the Scope Item ID).

3.2 Corrected documentation deliverables
In case we changed any documentation deliverables, they will be attached to this note.

4. SAP Solution Manager 7.2

We recommend downloading the SAP Solution Manager 7.2 content via the SAP Solution Manager 7.2 Download Basket in the SAP Best Practices Explorer (available if you have an S user) by selecting the solution package.

If you choose not to connect the SAP Solution Manager 7.2 with the Internet, you can import the content (JSON) as a local file into SAP Solution Manager. The JSON file is included in the documentation package or you can open the file from the following location (https://support.sap.com/content/dam/SAAP/Sol_Pack/BP_OP_ENTPR/BP_OP_ENTPR_S4HANA2020_07_SAP-BP-SolMan_EN_SG.json) and save it locally.

In case the import fails, you can check SAP Note 2363258 for further instructions.

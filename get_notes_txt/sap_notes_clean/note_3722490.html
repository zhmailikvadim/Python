Symptom

When creating a Vendor Claim from an Agreement Schedule, the manually maintained condition configured in the system is not being determined. As a result, the total amount is not updated in the claim header.
This SAP Note provides the necessary corrections to ensure proper determination of the manual condition and correct calculation of the total amount in the Vendor Claim.

Other Terms

Vendor Claim, Agreement Schedule, /IRM/IPPRASP, Total Amount, Manual Condition

Reason and Prerequisites

This is caused by program error.

Note : SAP Note#3722458 is available for V/4 release.

Solution

Please implement the correction instructions attached. These corrections are only valid if you have applied the Support Packs

Release 6.4 – Base Package & Support Pack 1

Use transaction SNOTE to implement the correction instructions specified below and activate the affected program object

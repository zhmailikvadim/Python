Symptom

Under purchase order item level "confirmations" tab, there is a confirmation line but without inbound delivery number.

Environment

SAP S/4HANA Cloud Public Edition

Reproducing the Issue

  1. Create a purchase order in SAP S/4HANA Cloud Public Edition.
  2. Navigate to the purchase order confirmations tab.
  3. Manually add a confirmation with a specific quantity.
  4. Attempt to locate the inbound delivery number via "display inbound delivery" or other applications but can't find the inbound delivery number.

Cause

Manually adding a confirmation with a specific quantity in the purchase order confirmation tab does not automatically create an inbound delivery. As a result, the inbound delivery field remains empty.

Resolution

  • Option 1: Receive the confirmed quantity through the MIGO transaction.
  • Option 2: Edit the purchase order confirmation tab, change the quantity to 0, and press Enter. This action will delete the confirmation. Afterward, save the purchase order.
    Once the confirmation is removed, use app "create inbound delivery" to create a delivery for the entire quantity.

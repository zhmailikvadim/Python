Symptom

The system sets the field "Earliest personal recalculation date for time evaluation" in infotype 0003 to high date (12/31/9999). This prevents you from making further changes to time data.

The system can also set the field "Earliest personal retroactive accounting date" to high date, which affects payroll retro calculation.

The system sets these dates in infotype 0283.

Other Terms

PA_CALC, PA_TIME, PU22, PRDAT, PRTEV

Reason and Prerequisites

Reason:
The system sets the date to high date (12/31/9999) when the method SET_SMALLEST_DATE of class CL_HRARCH_SMALLEST_DATE_FINDER receives an initial date value.

Prerequisites:
You run archiving with payroll (PA_CALC object) without selecting the option to archive time data, or you run archiving for time data (PA_TIME object).

Solution

Implement the attached correction instructions or implement the relevant HR Support Package for your system.
After implementing this SAP Note, the system correctly handles initial date values during archiving operations.

The correction adds a parameter to the method SET_SMALLEST_DATE to return an initial value as the smallest date.
This behavior is optional to avoid affecting other code that do not want an initial value as the smallest value.

Symptom

  • eDocument generated is not reported in local currency for Colombia eDocuments
  • How to change currency of eDocument generated via eDocument Cockpit app or Manage Electronic Documents app for Colombia.

Image/data in this KBA is from SAP internal systems, sample data or demo systems. Any resemblance to real data is purely coincidental.

Environment

SAP S/4HANA Cloud Public Edition

Reproducing the Issue

  1. Go to eDocument Cockpit app or Manage Electronic Documents app
  2. Search and select related document
  3. Click Display > Display/Preview XML
  4. Download the XML
  5. In XML file the reported currency is not 'COP' (Colombia currency)

Cause

For Colombia, this behaviour is documented in SAP KBA 2805847 - eDocument Colombia Troubleshooting Guide (Topic B103) and must be delivered by the implementation partner via extensibility.

Resolution

As referenced in SAP Help Colombia App Extensibility, there are Business Add-Ins (BAdIs) available for Colombia localization to add, change and/or remove information from files generated by eDocument Cockpit or Manage Electronic Documents apps. Similarly, each localization’s documentation in the SAP Help Portal Country/Region-Specific Functions provides access to the eDocument Cockpit and the BAdIs available for that localization. The following page works as a guide to create Custom Logic implementations, which can be used to implement the BAdI or enhance an existing implementation: SAP Help Creating Implementations.

This SAP KBA provides guidance for modifying currency and values in specific fields of the Invoice XML for Colombia. It is a proof of concept, not an implementation instruction; apart from identifying the relevant BAdI, all coding decisions remain under the scope and responsibility of the implementation partner.

Refer to steps below:

  1. To adapt XML file created for Colombia invoice, use BAdI EDOC_CO_ADAPT_XML_INVOICE;
    a. Note: for PDF modification or change a different type of document, check SAP Help Colombia App Extensibility and evaluate the related BadI for your purpose.
  2. Create a Custom Logic  to determine the value in 'COP'
    a. In this example, 'ACDOCA' table (Journal Entry Item table) was retrieved to get the same info registered in the Journal entries;
  3. Through analyzing the change parameters, modify the value to have the currency as 'COP' and their value in 'COP' according to value determined in the previous step.
    a. Some fields may be modified, but depending on customer implementation, it might need more or fewer fields changed.
    b. Implementation Partner should review the change parameter and determine which needs to be changed and which logic to use for it.

  4. With the following implementation, the XML file for a sale in 'USD' was changed from sending the info in USD (left) to having it in 'COP' (right).

  5. Note that after changing the currency, it's requested to follow SAP KBA 2974488 - Calculation errors in CUFE to have the CUFE recalculated, as the values have changed.

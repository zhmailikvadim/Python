Symptom

Clarification is needed regarding the difference between locking/unlocking and activating/deactivating user statuses in Ariba.

Cause

Resolution

    Locked: Applied when a user forgets their password or is intentionally restricted from logging into the system. Locked users can still be included in assignment groups and configurations.

    Active: When marked inactive, the user is effectively removed from the system's functionality, akin to a soft deletion. Inactive users cannot be utilized in any capacity within the system.

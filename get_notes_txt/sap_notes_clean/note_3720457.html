Symptom

Is it possible to rest the password for non-dialog user type from Access Request?

Environment

Access Control

  • Access Request

Reproducing the Issue

na

Cause

Design and compliances

Resolution

Resetting password for non-dialog user type is not possible through Access Request.

The reason for this is, GRC application is designed to provide services through NWBC for end user's (Dialog User Type) because NWBC requirement is to allow only Dialog Users to login and non-Dialog users are not allowed to login to NWBC. Hence, end user need to login to perform action such as Raise Request service, execute Risk Analysis service, reset password service etc., through NWBC. 

Other than Dialog User's  needs to be maintained by system administrator only for resetting password directly in SU01 but not from GRC application

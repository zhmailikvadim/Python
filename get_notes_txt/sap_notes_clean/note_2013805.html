Symptom

  • Unable to publish documents created in the version of Web Intelligence Rich Client installed via the BI launch pad if Auditing Set Events is not enabled in the Central Management Console (CMC).
  • While Saving to the corporate server, an error message appears:
    • Could not save the document to the repository for the following reason: [repo_proxy 17]
    • SessionFacade::sendAuditEvents - (com.crystaldecisions.sdk.exception.SDKException$EventTypeDisabled:Event Type1008 is disabled (FWM 02073)) [repo_bridge -BridgeSessionFacade::sendAuditEvents] (WIS 30567)”

Other Terms

Web Intelligence Rich Client, publishing

Reason and Prerequisites

Steps to Reproduce:

  1. Log into the CMC.
  2. Go to “Auditing”.
  3. Slide the slider to either Custom or Minimal under “Set Events”.
    Note: If setting to custom, uncheck “Save” option.
  4. Click Save and log out of the CMC.
  5. Login to the Web Intelligence Rich Client installed via the BI launch pad.
    Note: The issue does not occur in Web Intelligence Rich Client installed via the SAP BI Platform CD.
    This Issue occurs with all users once the Save option is removed, including the Administrator.
  6. Create or open a document.
  7. Attempt to save to it to the corporate server, either to My Favorites or Public Folder).
    The following error message appears:
    1. Could not save the document to the repository for the following reason: [repo_proxy 17]
    3. SessionFacade::sendAuditEvents -
      (com.crystaldecisions.sdk.exception.SDKException$EventTypeDisabled:Event Type1008 is disabled (FWM 02073)) [repo_bridge -
      BridgeSessionFacade::sendAuditEvents] (WIS 30567)
  8. Log into the BI launch pad or Import documents via Web Intelligence Rich Client, and see that the report was saved, despite the error message.

The reason for the undesired behavior is an error in the audit event management.

Solution

Update your BI Platform Server and Client installations to an Support Package, Patch or Release where the issue is fixed.

See the Support Packages & Patch section for details and available patches.

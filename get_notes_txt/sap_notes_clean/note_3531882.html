Symptom

When attempting to delete records from the table 'DEPT' in SAP IQ, the operation fails.

(dba)> delete from DEPT
Could not execute statement.
  Cannot delete/truncate CK dba.DEPT.ASIQ_IDX_T1723_I4_HG because of RI violation - FK has non-nulls.
  -- (slib/s_ohcidx.cxx 3803)
  SQLCODE=-1009150, ODBC 3 State="HY000"

Environment

  • SAP IQ 16.x

Reproducing the Issue

  1. Create the table

    In the EMPLOYEE table, the department ID is a foreign key for the DEPT table; each department ID in the EMPLOYEE table corresponds exactly to a department ID in the DEPT table.

    CREATE TABLE DEPT
    (DeptNo int primary key,
     DeptName varchar(20),
     Mgr int );
       
    CREATE TABLE EMPLOYEE
    (EmpNo int primary key,
     DeptNo int references DEPT(DeptNo) on delete restrict,
     LastName varchar(20),
     FirstName varchar(20),
     Salary int);

  2. Insert a Rows

    insert into DEPT values(100,'R & D',501)
    insert into DEPT values(200,'Sales',902)
    insert into DEPT values(300,'Finance',1293)

    insert into EMPLOYEE values(102,100,'Whitney','Fran',45700.000)
    insert into EMPLOYEE values(129,200,'Chin','Philip',38500.000)
    insert into EMPLOYEE values(148,300,'Jordan','Julie',51432.000)

  3. Delete all rows on DEPT

    (dba)> delete from DEPT
    Could not execute statement.
      Cannot delete/truncate CK dba.DEPT.ASIQ_IDX_T1723_I4_HG because of RI violation - FK has non-nulls.
      -- (slib/s_ohcidx.cxx 3803)
      SQLCODE=-1009150, ODBC 3 State="HY000"

Cause

The DEPT table has a foreign key that is being referenced by other table. The error occurs when trying to delete records from this DEPT table without first deleting the corresponding records in the child table.

Resolution

I will explain using the DEPT and EMPLOYEE tables in [Reproducing the Issue].

  1. Identify the child table that reference the DEPT table.

    sp_iqconstraint procedure can lists referential integrity constraints defined using CREATE TABLE or ALTER TABLE for the specified table or column.

    The child table is the EMPLOYEE table.

    Sample
    ---------
    (dba)> sp_iqconstraint DEPT
    table_name table_owner index_name           CKtype     column_name foreign_table_name foreign_table_owner role fk_enforced foreign_index_name   foreign_column_name location
    ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------
    DEPT       dba         ASIQ_IDX_T1723_I4_HG PK         DeptNo      EMPLOYEE           dba                 DEPT Y           ASIQ_IDX_T1724_C2_HG DeptNo              Main

    (1 rows)

  2. Delete the records in the child table that reference the records you want to delete in the DEPT table.

  3. After deleting the corresponding records in the child table, proceed to delete the desired records in the DEPT table.

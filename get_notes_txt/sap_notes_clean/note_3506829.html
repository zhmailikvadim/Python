Symptom

When editing or transferring object dependencies with many codelines the runtime may get unexpectedly long. According to the trace the system spends too much time in the SET_DATES_FOR_CUKN method of class CL_VCH_DEP_VALIDITY.

Other Terms

CUKN, DATUB

Reason and Prerequisites

Too many lines get updated due to a program error

Solution

Implement the corrections from this note or apply the relevant support package

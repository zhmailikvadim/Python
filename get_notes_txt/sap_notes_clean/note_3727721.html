Symptom

Perhaps during partitioned commodity code creation an incorrect ID or adapter source was used.  How can this be fixed?

Resolution

Please submit a case to Ariba Technical Support to perform a back-end fix.  While we cannot fully delete the created partitioned commodity code from your realm's database, Ariba Support can change the unique ID to a value that will not be used.  For example, partitioned commodity code , PCC1, will be changed to PCC1:INACTIVE.  This will free up the unique ID previously used to make the corrections.

Please note, the adapter source cannot be changed.  Ariba Support can only modify the unique ID itself.

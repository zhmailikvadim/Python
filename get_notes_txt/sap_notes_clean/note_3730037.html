Symptom

  • When adding a new item to a purchase order created before a tax change, the old tax code defaults instead of the new one.
  • You need to update a new tax code in an old PO. 
  • You would like to avoid this default old tax code value and new tax code should be suggested by default. 

Environment

  • Materials Management (MM)
  • SAP ERP Central Component
  • SAP ERP
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA

Reproducing the Issue

  1. Open an existing purchase order that was created before the tax code change took effect.
  2. Add a new item to the purchase order.
  3. Observe that the system defaults the old tax code on the new item instead of the new tax code.

Cause

  • The pricing date used for price determination defaults to the purchase order document date when no Price Date Category is set; therefore, the condition technique (for example, condition type NAVS) selects the tax code valid on that historical date, resulting in the old tax code being proposed.

  • When the user adds a new item to purchase order, the program uses the document date (EKKO-BEDAT) for the price determination.

  • The price determination (function module PRCING) determines the purchasing tax condition record for NAVS valid on document date. If you would like to determine the new tax code, you need to make sure that the program uses a pricing date which leads to the new expected tax code

     

Resolution

Possible solutions:

  1. You could set the "Price Date" (EKPO-PRDAT) manually on tab page "Condition Control".
  2. You let the system derive the "Price Date" (EKPO-PRDAT) via the "Price Date Category" (EKPO-MEPRF) on tab page "Condition Control".
  3. You change the "Price Date" (EKPO-PRDAT) via  a customer specific implementation in BAdI ME_PROCESS_PO_CUST method PROCESS_ITEM.
  4. You change the "Price Date" (KOMK-PRDAT) via a customer specific implementation in BAdI ME_PO_PRICING_CUST method PROCESS_KOMK.

Background:

  • "Price Date Category" (EKPO-MEPRF) 

Form MEPO_ITEM_FILL_MEPRF (LMEPOF5M) sets the "Price Date Category" (EKPO-MEPRF) on tab page "Condition Control".
The "Price Date Category" (EKPO-MEPRF) is set based on reference document (e.g. reference purchase order, quotation, contract, info record or vendor master data.
The "Price Date Category" (EKPO-MEPRF) controls how the "Price Date" (EKPO-PRDAT) is set.

Check if field is visible because in customizing.
SAP Customizing Implementation Guide -> Materials Management -> Purchasing -> Purchase Order -> Define Screen Layout at Document Level

Field Selection Key: ME21N 
Selection group: Quantity and Price
Field "Pricing date control"

  • "Price Date" (EKPO-PRDAT) 

In form PREISFINDUNG_PRDAT (MM06EFKO_PREISFINDUNG_PRDAT) the program sets the pricing date for the new purchase order item (EKPO-PRDAT) based on the "Price Date Category" (EKPO-MEPRF). 
In your example the "Price Date Category" (EKPO-MEPRF) is not set because no reference document is involved hence the program sets the "Price Date" (EKPO-PRDAT) based on the purchase order document date (EKKO-BEDAT).

SAP Customizing Implementation Guide -> Materials Management -> Purchasing -> Purchase Order -> Define Screen Layout at Document Level

Field Selection Key: ME21N 
Selection group: Quantity and Price
Field "Pricing date control"

  •  Condition Pricing Date (KOMK-PRSDT)

Function module ME_FILL_KOMK_PO sets the condition pricing date (KOMK-PRSDT) based on the "Price Date" of the purchase order item (EKPO-PRDAT).

The condition pricing date (KOMK-PRSDT) is used for the price determination process. 

The condition pricing date (KOMK-PRSDT) can be changed via  BAdI ME_PO_PRICING_CUST method PROCESS_KOMK.

  •  "Tax Code" (EKPO-MWSKZ)

Form PREISFINDUNG (MM06EFKO_PREISFINDUNG) set the tax code on PO item level (EKPO-MWSZ) based on the purchasing tax condition record for NAVS. The purchasing condition record was determined via the price determination process (function module PRICING).

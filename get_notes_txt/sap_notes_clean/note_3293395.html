Symptom

This SAP Note is relevant for the following roles: DSO, Supplier, and MOS.

For IDXGL SP11 and SP12:

The following objects only provide to select data by the Market Location (MaLo) installation. The Metering Location (MeLo) is missing:

  • Transaction /IDXGL/TOU_ASSIGN
  • Class /IDXGL/CL_TOU_FINDER
    • Method /IDXGL/IF_TOU_FINDER~FIND_REG_BY_INSTL_OBIS
    • Method /IDXGL/IF_TOU_READER~GET_ASSIGN_ALL_DEEP
  • Class /IDXGL/CL_TOU_READER
    • Method /IDXGL/IF_TOU_READER~GET_ASSIGN_REG_BY_INST

For IDXGL SP12:

The note will fix potential syntax errors with regards to parameter IT_INSTALLATIONS_ME of the methods above.

Other Terms

Zählzeiten, Time-of-Use Definitions, ToUD, ZZD, DSO, SUP, MOS

Reason and Prerequisites

Coding issue

Solution

Implement the SAP Note.

If you find that after you have implemented the SAP Note, some field labels are missing in German, proceed as follows. The translations will be shipped with the next Support Packages. Please refer to the information on SAP Support Portal at https://support.sap.com/sp-stacks.

The affected transport objects are:

PgID Object Name
R3TR DTEL /IDXGL/DE_TOU_INSTALLATION_MA
R3TR DTEL /IDXGL/DE_TOU_INSTALLATION_ME

Package: /IDXGL/METERING_TOU

Software component: IDXGL

Application component: IS-U-IDX

If you want to enter the translations manually in advance, you can do so as follows:

  1. Call transaction SE63.
  2. In the command field (where you would normally enter a transaction code), enter DTEL and press ENTER.
  3. Enter the object name (see below) in the Object Name field. 
  4. If required, change the source and/or target languages.
  5. Click Edit.
  6. Enter the translation in the text field below the source text.
  7. Click Save and Back.

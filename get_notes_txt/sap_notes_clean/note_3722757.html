Symptom

You've ran HANA SQL Statement SQL: "HANA_Threads_Callstacks_MiniChecks" (available on SAP Note 1969700) and checkID C1504 was reported:

Check ID  Area   Description                    Call stack details 
C1504      SQL     HEX task finalization    hex::TaskScheduler::finishAndWait

Environment

  • SAP HANA PLATFORM EDITION 2.0 SPS07

Resolution

This operation is performed at the end of parallelized HEX statement execution, the report of checkID C1504 can be an indication for SAP HANA job queueing due to NUMA contention.

If the HEX engine (SAP Note 2570371) suffers from NUMA node contention on SAP HANA <= 2.0 SPS 07, you can consider setting the following SAP HANA parameter to allow stealing from other NUMA nodes:

indexserver.ini -> [hex] -> worker_normal_affinity_fraction = 1

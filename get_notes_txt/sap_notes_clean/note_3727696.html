Symptom

In NAST based output an output type is being used which has transmission medium 7 (Simple Mail). 

Once it is double clicked on it or communication method is checked and then it is returned to the output screen the partner function is changing to 'MA'. 

Environment

  • SAP R/3
  • SAP S/4HANA
  • S/4HANA Cloud Private Edition

Reproducing the Issue

  1. Create a billing document which has an output type with transmission medium 7 (Simple Mail).
  2. In VF02 go to header and check the output screen.
  3. Double click on the questionable output type or mark it and press on communication method.
  4. Return from back to the output screen by pressing 'F3'.
  5. Check the partner role which got changed to 'MA'.

Cause

The system changes the partner role to "MA" when external email address(es) is maintained on the communication method screen for the questionable output type. Partner type "MA" is used to maintain email addresses, while partner type "RE" is associated with a partner number. This is hard coded and is performed in:

FORM FIND_NAST_PARTNER_AND_TYPE (LWMCMF01)

...
 * external SAPOffice address
       NRART = PARTNER_SADR.                                                            < gets value 'MA'
...
     SELECT * FROM TPAR WHERE NRART = NRART.
       EXIT.
     ENDSELECT.
     IF SY-SUBRC = 0.
        P_PARVW = TPAR-PARVW.
     ELSE.
...

 

Resolution

This is the intended system response. If this is disturbing then either a condition record should be create with 'MA' or a modification is required to prevent the system from changing the partner role. In this regard SAP Note 170183 is to be considered.

Symptom

You are using transaction VL02N for a warehouse related delivery. You have created a transfer order, but are reducing picking and delivery quantity aftrerwards. The "Overall WM Status" is reset to "B-In Process" so that Goods Issue cannot be posted.

With SAP note 2494539 the behaviour of transaction VL02N was changed, so that the picking confirmation status was reset, once the picking quantity changes. This also applies to the WM status.

This is changed now, so that for WM related deliveries, the behaviour is as before. "Picking Confirmation Status" and "WM Status"  stay unchanged for warehouse related processes when changing the picking quantity.

Other Terms

VL02N, Overall WM Status,

Reason and Prerequisites

Program Error. Note 2494539 has been implemented before.

Solution

Implement this note

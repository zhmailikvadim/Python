Symptom

  • Maintenance order not created automatically while creating emergency maintenance notification in app Manage Maintenance Notification.
  • Issue not exists while using app Create Maintenance Request or app Create Maintenance Order.

Environment

SAP S/4HANA Cloud Public Edition

Reproducing the Issue

  1. Attempt to create an emergency notification via the "Manage Maintenance Notification" app.
  2. After maintenance notification created, emergency order is not created automatically.

Cause

Problem with notification processing (context: Emergency work) - If, for any reason, order creation fails in the background, the background queues get blocked and hence all later queues got blocked.

Resolution

  1. Enter the app "Monitor bgRFC Queues."
  2. Click on the Inbound Destination 'BGPF' and navigate inside.
  3. Locate the queue with the error status "Queue has execution errors."
  4. Select the first row showing "Unit has execution errors" and click on the "Delete" action at the top. This will delete the blocked queue and restart the process.

Keywords

Emergency order, Automatic creation, Maintenance notification, Manage Maintenance Notification, Create Maintenance Request, bgRFC queues, Blocked queue, Queue execution errors.

Symptom

  • Before an SAP Solution Manager SP level update (often called upgrade), concerns were raised regarding the impact on Change Request Management (ChaRM) operations.
  • A SP level update was performed and issues arise before performing SOLMAN_SETUP mandatory activities.
  • Standard transaction types were modified before the update and unexpected behaviors are reported after it.

Environment

SAP Solution Manager 7.2

Cause

  • The configuration for Change Request Management is not affected by the SP level update, provided mandatory SOLMAN_SETUP activities are completed post-update.
  • Standard transaction types for Change Request Management may be overwritten during the update.
  • If standard transaction types were modified, custom configurations on standard transaction types need to be updated to the customer namespace (Z/Y) using transaction AI_CRM_CPY_PROCTYPE before updating.

Resolution

  1. Ensure completion of all mandatory SOLMAN_SETUP activities post-update.
  2. Avoid working with Change Request Management documents during the update until SOLMAN_SETUP steps are completed.
  3. If standard transaction types were modified, use transaction AI_CRM_CPY_PROCTYPE to update customizations on standard transaction types to the customer namespace (Z/Y) before upgrading as explained in KBA  2686343 - How to use AI_CRM_CPY_PROCTYPE.
  4. Refer to the following documentation for detailed guidance:
    1. SAP Help Initial Setup for the SAP Solution Manager System
    2. SAP Help Release Notes

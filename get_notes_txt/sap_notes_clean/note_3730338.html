Symptom

  • When print preview application document with PDF preview, the output displays multiple extra lines/mess scribbles across the page.
  • The issue only occurs if use SAPGUI browser control "Edge(Based on Chromium)" but not happen when use "Internet Explorer".

Environment

  • SAP ERP all releases
  • SAP NetWeaver all releases
  • SAP S/4HANA all releases
  • ABAP PLATFORM - Application Server ABAP

Reproducing the Issue

The issue happens when print preview the application document. For example:

  1. Open transaction ME23N and display any purchase order.
  2. Choose Print Preview.
  3. Start the PDF preview (for example, via Goto -> PDF Preview or by running PDF!)

Cause

  • The application document contains English texts and use a Chinese bold font in format.

  • Bold Chinese fonts are not available in PDF viewer(e.g. the Chinese language pack of the Adobe Reader does not contain bold Chinese fonts). 
    So, the PDF viewer uses a trick here: To make the characters bold, it switches the text rendering mode from 'Fill text' to 'Fill then stroke text' (using the PDF command 2 Tr according to the PDF specification). 
    This has the effect that each character gets an additional outline which takes care that the character looks bold. Usually, this works perfectly with every PDF viewer. 
    But Microsoft Edge has a bug here: Somehow it does not outline the character, but it outputs the outline with a much larger font size (the lines which you see are the bottom left corner of these outlines).  

Resolution

There are several workarounds for this issue.

  1. Change the GUI Browser Control for HTML to “Internet Explorer” (Interaction Design -> Control Settings->Browser control) to avoid using Edge PDF viewer.
  2. Run report RSTXPDF3 and set parameter CJK_USE_BOLD_IT to Off. Note: bold text will no longer appear bold after this change.

  3. When print application documents,if the output device uses a non-UPE device type like POST2, HPLJ4, SAPWIN, Pls run report RSTXPDF3 set USE_CASCADING,UPE_FONTS_CASC and CJK_EMBED_FONTS to On.

    if uses SWINCF or PDFUC,Pls run report RSTXPDF3 to set UPE_FONTS_CASC and CJK_EMBED_FONTS as On.

    If uses UPE(a device type like POST2, HPLJ4 + URDT), Pls run report RSTXPDF3 to set CJK_EMBED_UPE as On.

     Pls note:The generated PDF file size will increase when use the solution 3.

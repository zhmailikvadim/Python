Symptom

You have an EWM PDI document, with a batch managed item. If you navigate to the packing using tr. /SCWM/PRDI, Menu -> Inbound Delivery -> Follow-On Functions -> Pack, you see an empty handling unit and the delivery item seems to be unpacked, however it should have been packed into the handling unit.

Other Terms

/SCWM/PRDI, /SCWM/RFUI, #WOP_INCON_STOCK_QUAN_MISSING

Reason and Prerequisites

batch update destroys planned stock

Solution

Implement the following notes:

3728633 - Inconsistent Planned Stocks after batch update
3670812 - physical stock disappears after changing batch master data

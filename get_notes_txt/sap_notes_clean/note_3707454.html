Symptom

The alpha factor is not reflected in the forecast calculation.

Environment

  • SAP ERP
  • SAP ERP Central Component
  • SAP Enhancement package for SAP ERP
  • SAP Enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA, on-premise Environment
  • SAP S/4HANA Cloud Private Edition

Reproducing the Issue

  1. Execute transaction MP30 or MM02.
  2. Select Material and Plant.
  3. Using transaction MM02 Navigate to the "Forecast" tab.
  4. Using transaction MP30 click on "Execute Forecast".
  5. Select a period.
  6. Choose the "Constant model" option.
  7. Select the first radio button: "First-order exponential smoothing with Alpha factor".

Cause

  1. With forecasting using alternative historical data (AHD), a forecast can be created based on freely definable data. Forecasting using alternative historical data is therefore not based on past consumption values that were updated in sites during goods issue posting.
    The absence of valid Alternative Historical Data (AHD) records for effected period prevents the forecast calculation from considering this period.

  2. A gap exists when:
    A period has no AHD record at all (missing),
    Or the period is outside the maintained AHD interval.
    The "Gaps in time series" indicator identifies missing AHD records or periods outside the maintained AHD interval, resulting in the system defaulting to the previous forecast value.

Resolution

  1. Ensure that Alternative Historical Data (AHD) is maintained for the required period.
  2. Verify that the "Gaps in time series" indicator is checked to identify missing or invalid AHD records (in transactions MAHD1, MAHD2 Last column "L": Gaps in Time Series).
    This indicator checks whether all required historical periods in the forecast horizon contain valid AHD values.

Symptom

  • Entity PickAndBatchSplitOneItem does not return the batch split line number while running by API_OUTBOUND_DELIVERY_SRV
  • No response body is returned for the operation

Environment

  • Sales and Distribution (SD)
  • Logistics Execution (LE)
  • SAP ERP Central Component
  • SAP ERP
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA

Reproducing the Issue

  1. Access transaction /n/IWFND/GW_CLIENT
  2. Set the method to POST
  3. Execute the API call and execute it:

            "...odata/sap/API_OUTBOUND_DELIVERY_SRV;v=2/PickAndBatchSplitOneItem?DeliveryDocument='......."

    Cause

    Standard behavior.

    The operation does not return a response body, as per documentation:

    - SAP Help Portal - PickAndBatchSplitOneItem

    Resolution

      To retrieve the newly created items, perform a batch request that combines this operation with a read operation

      Keywords

      pickandbatchsplitoneitem, batch split line number, odata api, outbound delivery, api call, http response, no response body, batch request, sap s4hana, delivery document interface, API_OUTBOUND_DELIVERY_SRV

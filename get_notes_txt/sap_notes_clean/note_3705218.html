Symptom

While testing HTTPS connection from SCC, following error is raised from SCC web UI.


====
MessageTLS certificate issue, e.g. certificate validation has failed: Server Certificate Failure: Issuer with subject DN CN=<customer name> is not trusted.
Add this issuer to the Cloud Connector backend trust store.
Exceptioniaik.security.ssl.SSLCertificateException: Peer certificate rejected by ChainVerifier.
====

Environment

  • SAP Cloud Connector(SCC);
  • SAP AS Netweaver ABAP backend system;
  • SAP AS Netweaver Java backend system;

Cause

Missing certificate on the SCC end.

Resolution

Make sure there is a certificate generated and trusting the host in the request (Internal host of the SCC mapping), and this certificate is available either on SCC -> "Configuration" -> "On-Premises" -> "System Certificate", or on SCC -> "Configuration" -> "On-Premises" -> "Backend Trust Store" -> "Allow List".

Symptom

In OPSJ user tried to change an existing Coding Mask  the message Projects/WBS elements with the editing mask & still exist.

Environment

  • SAP ERP Project System (PS)
  • SAP R/3
  • SAP R/3 Enterprise
  • SAP ERP Central Component
  • SAP ERP
  • SAP Enhancement package for SAP ERP
  • SAP Enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA, on-premise
  • SAP S/4HANA Cloud Private Edition

Reproducing the Issue

  1. Run OPSJ.
  2. Select Coding Mask.
  3. Make Changes .

Cause


If you try to delete/create a Coding Mask in customizing for a project ID for which objects already exist, you will get the error message JP011 

Resolution

All such projects or WBS elements must be archived (or physically deleted) before you can delete/create a Coding Mask the way in which you require.
This is explained in SAP Note0000553661 - FAQ 2: Coding masks 

The entries in tables PROJ (project definition), PRPS (WBS elements), PROJS (standard projects) and PRPSS (standard WBS elements),
 and possibly the versions tables (VSPRPS_CN, VSPROJ_CN) need to be deleted before you can delete/create the Coding Mask

Once you have deleted or archived all projects and WBS elements which use the identifier, you should be able to delete/re-create the Coding Mask as required.

Symptom

  • The information for the selected forecast model, best fit algorithm and job execution time differs when checking it in Statistical Forecast Details in Excel Add-in or Planner Workspaces App compared to the Application Job logs:

"Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental."

Environment

  • SAP Integrated Business Planning (SCM-IBP)

Reproducing the Issue

  1. Open the planning view for the given combination.
  2. Navigate to the statistical forecast details and observe the last forecast run details.
  3. Check the last forecast run log for the job.
  4. Compare the forecast details with the statistical forecast target key figure in the planning view.

Cause

  • The executed statistical forecasting job is set up to use model assignments, and the forecast model selected during execution is configured to write results to a different output key figure than the main forecast model defined in the job settings.
  • The key figure added in the planning view is not an output key figure defined in the forecast model configuration.

Resolution

  1. Verify whether the specific combination was forecasted using the main job model or an assigned forecast model.
  2. Ensure the planning view displays the output key figure associated with the model that was used during the forecast execution (either the main model or the assigned model).

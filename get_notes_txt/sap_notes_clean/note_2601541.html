Symptom

The useradmin application returns 1500 users from an LDAP group search even though more users exist in that group.

Environment

SAP Application Server Java

Microsoft Active Directory as UME datasource

Reproducing the Issue

  • In UserAdmin application search for users in a specific LDAP group which contains more than 1500 users.
  • From command line interface search for group users with ldifde command, for example:
    ldifde -v -s <hostname> -d "CN=basis,DC=na,DC=dir,DC=part,DC=com" -p subtree -r "(cn=C:SALES)" -f Groups.txt

Cause

The active directory default MaxValRange limits group search results to 1500.

Refer to Microsoft documentation LDAP Policies.

Resolution

Adjust the MaxValRange limit in the Microsoft Active Directory server.

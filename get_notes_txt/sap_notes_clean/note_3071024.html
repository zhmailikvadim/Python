Symptom

With regard to MGV Billing Monitor, customer is or may be facing the following problems:

1. Consumption Items are not filtered for Consumption Item Class, if customer has defined the Consumption Item Class for their own billing solution, these Consumption Items are also shown in MGV Billing Monitor.
2. Billable Item Class is hardcoded, even if this could be customized by customer.
3. Bad German translation of "INVOIC_STATUS", 1/Open: Translation should be "Offen" instead of "Geöffnet"

Other Terms

MGV Billing Monitor, MGV_BILL_MON, /US4G/MGV_BILL_MON, /US4G/CL_MGV_BILL_MON, /US4G/CL_MGV_BILL_REV, /US4G/CL_MGV_BILL_MON~GET_CIT, /US4G/CL_MGV_BILL_REV~GET_BILLABLE_ITEMS_BILLING

Reason and Prerequisites

 Program error.

Solution

1. Consumption Items are not filtered for Consumption Item Class, if customer has defined the Consumption Item Class for their own billing solution, these Consumption Items are also shown in MGV Billing Monitor.

Solution:

Please implement the correction instruction or apply the higher Support Package.

2. Billable Item Class is hardcoded, even if this could be customized by customer.

Solution:

Please implement the correction instruction or apply the higher Support Package.

3. Bad German translation of "INVOIC_STATUS", 1/Open: Translation should be "Offen" instead of "Geöffnet"

Solution:

The translations will be shipped with the next Support Package. Please refer to the information on SAP Support Portal at https://support.sap.com/sp-stacks.

If you want to enter the translations manually in advance, you can do so as follows:

  1. Call transaction SE63.
  2. In the command field (where you would normally enter a transaction code), enter DOMA and press ENTER.
  3. Enter the object name (/US4G/D_MGV_INVOIC_STATUS_DSP) in the Object Name field.
  4. If required, change the source and/or target languages.
  5. Click Edit.
  6. Double click Domain Value Node
  7. Replace text "Geöffnet" with "Offen"
  8. Click Save and Back.

Symptom

You run the Programa de cálculo de nómina Argentina (HARCALC0) report for an employee in the following scenario:

  • The employee has a monthly periodicity.
  • There is an entry record in the Pago complementario (IT0015) infotype, regarding an additional/bonus payment for the past year.
  • You run the Ajuste de impuesto anual (AJIA) for this employee.

In the scenario described above, the system does not account for wagetypes Porción pror. p/ impuesto (P156) or Monto s/ pror p/ impuesto (N156) in the final AJIA report, when it should.

Reason and Prerequisites

Reason

This issue occurs because wagetypes P156 (Porción pror. p/ impuesto - Prorated portion for tax) and N156 (Monto s/ pror p/ impuesto - Amount without proration for tax) lack the necessary indicators in their treatment class configuration, preventing them from being processed during the AJIA annual adjustment run.

Solution

The system now correctly considers wagetypes P156 (Porción pror. p/ impuesto) and N156 (Monto s/ pror p/ impuesto) during calculation processing for the AJIA report.

This SAP Note delivers configuration updates in the processing class settings for wagetypes P156 and N156, ensuring they are properly included during annual tax adjustment calculations.

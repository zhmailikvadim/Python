Symptom

  • The User is able to create the Joule skill using the Use Joule Studio to Create and Deploy a Joule Skill SAP Developer Tutorials.
  • When deploying the Joule skill or project, the process fails with the following error.

    Deployment Information
    The project "<project or skill name>", version "X.X.X" cannot be deployed by the component "Integration Gateway".
    The project <project or skill name>, version X.X.X cannot be deployed in this environment because the user does not have the required "extensibility_developer" role in Joule. Please contact your administrator to get the necessary permissions.

"Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental." 

Environment

  • SAP BTP, Cloud Foundry runtime and environment
  • Joule
  • SAP Build Automation

Cause

The role "extensibility_developer" is not visible in the subaccount because the corresponding service is not subscribed in the subaccount where the deployment is being attempted.

Resolution

Ensure that users who access Joule Studio are assigned the required role collections for both Joule and SAP Build Process Automation.

Steps to be followed:

  1. In the SAP BTP cockpit, assign the below role collections in the subaccount where SAP Build Process Automation and Joule is subscribed.

    SBPA Roles:

    Role Collection

    Description

    ProcessAutomationAdmin

    Manages the process configuration, permissions, and authorizations within SAP Build Process Automation.

    ProcessAutomationDeveloper

    Manages the creation, editing, and publishing of individual processes and automations within SAP Build Process Automation.

    ProcessAutomationParticipant

    Participates in active processes.

    ProcessAutomationDelegate

    Manages access to the application based on the specific resources granted, such as projects and environments. These permissions enable users to view, edit, or administer the application and allows to supervise the visibility scenario dashboards.

    ProcessAutomationExpert

    Empower business process experts with comprehensive (advanced/premium) operational visibility, analytical process performance insights, and the ability to initiate, work on, and approve tasks, alongside instance-level monitoring and drill-down capabilities for enhanced process performance management.


    Joule Roles:

    Role

    Description

    extensibility_developer

    Develop, compile, and deploy Joule custom capabilities.

    end_user

    Use Joule.

  2. Go to the BTP Cockpit to assign the required role collections to users or groups. Assign roles to users via user groups rather than individual users. Create and manage these groups using the Cloud Identity Services and assign the relevant role collections to the groups. Refer about assigning roles in the SAP BTP Cockpit, see Assigning Role Collections to Users or User Groups. 

  3. Verify that the Joule environment is shared with the user or group that has permission to execute Joule skills. In SAP Build Lobby, go to the Control Tower to share the environment. 

Note: SAP Build Process Automation and Joule subscriptions do not need to be in the same Global account or Subaccount; however, using the same IAS across both subaccounts is mandatory.

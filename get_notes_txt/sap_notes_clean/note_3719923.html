Symptom

In production order, perform staging to create PMR(Production Material Request) on Cloud WM side. There is requirement to close the PMR later on as staging is not needed.
However, changing the requirement quantity to 0 on Production order side cannot change the quantity of PMR accordingly. 

Environment

SAP S/4HANA Cloud Public Edition 

Reproducing the Issue

  1. Create and release a production order.
  2. Perform staging and generate PMR.
  3. Change the requirement quantity of one of the components in the production order to zero.
  4. Observe that the PMR quantity is not updated to zero.

Cause

Components without a requirement quantity are not taken into account in the PMR.

Resolution

  1. Go to the production order component view and change the requirement quantity to zero.
  2. Navigate to the header view of the production order and perform staging again.
  3. Access the PMR and verify that the Deletion flag is set for the item. This can block subsequent process.

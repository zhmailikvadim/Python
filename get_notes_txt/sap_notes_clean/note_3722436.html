Symptom

  • Tile variant of the app "sales volume - check sales order" is not retained when logging in again.
  • Filters applied to the saved tile are lost after logging out and logging back into the system.

Environment

  • Smart Business for S4HANA

Reproducing the Issue

  1. Log in to the system.
  2. Open the app "Sales Volume - Check Sales Order".
  3. Choose the filter "by delivery document" and save it as a tile.
  4. Navigate to "My Home" and open the saved tile; the filter works as expected.
  5. Log off from the system and log in again.
  6. Navigate to "My Home" and open the saved tile; the filter is no longer applied.

Cause

  • Upon installation or upgrade of FES/SAP_UI (e.g., during service pack updates), the configuration parameter APPSTATE_TRANSIENT is set to true or defaults to true.
  • When APPSTATE_TRANSIENT is set to true or no value is maintained, application state data for navigation is stored only in the browser memory (transient), which does not persist after logoff, page reload, or when opening the application in a new tab.

Resolution

  1. Log in to the SAP GUI system.
  2. Execute transaction `/n/UI2/FLP_CUS_CONF`.
  3. Choose "New Entries (Position)".
  4. Enter `APPSTATE_TRANSIENT` in the FLP Property ID field.
  5. Press Enter.
  6. Set the Property Value to `false` (please enter exactly “false” in lowercase).
  7. Save the changes.
  8. Test the application again to confirm whether the filters are retained as expected.

See Also

  • Refer to: SAP Note 3053861 - Filters are not working after reloading the tile which is created through Save as Tile.

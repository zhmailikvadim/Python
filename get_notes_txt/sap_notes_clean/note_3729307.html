Symptom

A simple query against a single table using INDEX RANGE SCAN can block other similar queries against same table with event "read by other session"

The hanging query does not finish regardless how much data needs to be read. It hangs forever. 
It can block other activities like

  • Other queries against the same table
  • Database shutdown for e.g. doing offline backups

Symptoms seen in DBACockpit Session monitor

  • A couple of sessions doing a select single against the same table are waiting for same block with "read by other session"
  • Access path is always an INDEX RANGE SCAN against a multicolumn index
  • The second column of the index is missing from the “where” condition leading into a “full index range scan”

Other Terms

SBP 19.29.0.0.251121 202511, 37653908, AIX, hang, Index range scan

Reason and Prerequisites

Important:  This issue was only seen with AIX platform. It was not reproducible on other platforms.


This issue will be visible after SBP 19.29.0.0.251021 has been applied on AIX platform.
It is caused by 19.29 AIX interim patch for bug 37653908 which is part of SBP 19.29.0.0.251021.

37653908 is included in DBRU 19.30.0.0.260120 (no longer provided as a single fix as like in SBP 19.29.0.0.251021).
SBP 19.30.0.0.260120 202602 is NOT affected by this issue.
SBP versions before SBP 19.29.0.0.251021 202511 are not affected either as they don't include the AIX interim patch for bug 37653908.

Solution

To fix the issue on Oracle SBP 19.29.0.0.251121 installations, please

Rollback 19.29 AIX interim patch for bug 37653908:

1. Stop all database applications, in particular all SAP applications.

2. Shutdown the database instance(s) (In a RAC env: On all nodes)

3. (In a RAC env: For each node of the cluster:) 
   As Oracle software owner, typically Unix user oracle: Use opatch to rollback the patch from your installation home $IHRDBMS:

    env ORACLE_HOME=$IHRDBMS $IHRDBMS/OPatch/opatch rollback -id 37653908

4. Start the database instance(s) (In a RAC env: On all nodes)

5. Start all database applications, in particular all SAP applications.

Alternative

  • Update your SAP database to SBP 19.30.0.0.260120 202602.

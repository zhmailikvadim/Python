Symptom

In transaction /PACG/ECM_CSPSE the relationship is set up as a source of the address of the contact person. This address can be found in table BUT052. However, the outbound idoc created for the contact person contains the incorrect the global address of that business partner. The problem could be found during the change of the address data performed from the FSM side. As a result of the exchange data between the FSM and ECC/S4, the changes on the FSM side were reverted. In the implemented correction the logic for the address selection was adjusted so that it evaluates both sources, compares them, and prioritizes the direct relationship information.

Other Terms

FSM, E4C, Contact, Person, Business, Partner, Communication, Address, Relationship

Reason and Prerequisites

This is caused by incorrect outbound selection procedure.

Solution

Please, implement this correction instruction.

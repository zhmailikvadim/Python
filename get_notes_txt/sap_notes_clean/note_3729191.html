Symptom

  • When accessing SAP HANA platform lifecycle management tool in HANA Studio, you got error like below:

    "Operation hdblcm_detect_system is not defined in SAP Host Agent. Probably configuration hdblcm_detect_system.conf is not properly deployed in exe/operations.d under SAP Host Agent installation on host <hostname>."
    "Client received SOAP Fault from server: Operation not defined: hdblcm_detect_system Please see the server log to find more detail regarding exact cause of the failure. "

  • The Platform lifecycle management tool also could not be accessed via the Web UI at URL http://<hostname>:1128/lmsl/HDBLCM/<SID>/index.html and shows error like "page can't be found".



    "Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental."

Environment

  • SAP HANA, platform edition 1.0
  • SAP HANA, platform edition 2.0

Reproducing the Issue

Access SAP HANA platform lifecycle management on SAP Hana studio.

Cause

  • HDBLCM is not installed
  • Incorrect SAP Host Agent version
  • Misconfiguration on HDBLCM tool or SAP Host Agent

Resolution

Follow the "Resolving problems with hdblcm installation" in 2078425 - Troubleshooting note for SAP HANA platform lifecycle management tool hdblcm to identify the root cause for the problem and resolve it accordingly.

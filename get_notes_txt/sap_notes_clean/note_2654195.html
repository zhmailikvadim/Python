Symptom

How to close the first Material Ledger period after S/4 HANA migration?

Environment

  • Controlling (CO)
  • S/4 HANA

Cause

Before S/4 HANA conversion, some material documents were posted during the migration period.

Resolution

"Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental."

  • As of S/4HANA 1610, there is a new Actual Costing with simplified data model and calculation logic. 
  • A New ML costing cockpit has been designed with a new logic, different steps and different tables.
  • Former tables (CKMLRUNPERIOD, CKMLRUNPLANT ..) have been replaced by table MLRUNLIST.

  • As a consequence, Costing runs created before migration can no longer be used as those are useless.
  • Before system conversion is started, all Material Ledger costing runs, no matter if actual costing (transaction CKMLCP) or alternative valuation run (transaction CKMLCPAVR) need to be finished. Please see in note 2354768 topic ML Actual Costing Cockpit (Transaction CKMLCPAVR). 

If the migration date is 08/07/2018 (Date format DD/MM/YYYY):

  • There should not be created any costing run for period 007/2018.
  • Before executing the Conversion S/4 HANA,  all pending costing runs created in previous periods (last one for period 006/2018) should be closed.
  • After executing the Conversion S/4 HANA: you can create costing runs in period 007/2018.
    The material documents posted in period 007/2018 before the migration (date 08/07/2018) will be processed in the costing run period 007/2018.
    Migration will create MLDOC entries for current period (007/2018) and all previous periods (based on old MLCD) back to 012/2016.
    When you start posting goods movements on S/4 HANA, additional MLDOC entries will be created, also in period 007/2018.
    New CKMLCP is based on MLDOC and it has all required data (data posted on old release as well as data posted in S/4HANA).

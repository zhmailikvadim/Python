Symptom

A user enters an alphanumeric CNPJ using lowercase letters, and the system accepts the entry without displaying any warning or error messages. This occurs even though the official government specification requires that alphanumeric CNPJs use only uppercase letters.

Environment

  • SAP ERP
  • SAP S/4HANA
  • SAP S/4HANA Cloud Private Edition
  • Master Data
  • Localization Brazil

Cause

According to the official guidelines from the Brazilian Federal Revenue (Receita Federal), the new alphanumeric CNPJ format must use only uppercase letters. The use of lowercase letters is considered incorrect data entry. The official government specification can be referenced here: https://www.gov.br/receitafederal/pt-br/centrais-de-conteudo/publicacoes/perguntas-e-respostas/cnpj/cnpj-alfanumerico.pdf

Resolution

This is not a system error. The system is not designed to validate or convert alphanumeric CNPJs entered with lowercase letters. No development adjustments will be made to handle this type of incorrectly entered data, as attempting to do so could introduce more significant problems.

Action: Users must ensure that all alphanumeric CNPJs are entered using only uppercase letters, in accordance with the official standard.

Keywords

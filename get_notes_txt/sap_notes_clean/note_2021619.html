Symptom

As described in SAP Note 441215, some systems erroneously have a user account with a blank user name. It is sometimes referred to as the SPACE user.
SAP Note 441215 provided a suggestion for a customer report (ZSUSR_DELETE_USER_SPACE) to correct this inconsistency in your systems.
You would like to have an adequate function in the standard SAP system.

The existence of the SPACE user described above is responsible for a GETWA_NOT_ASSIGNED short dump in the function module PRGN_USERS_EXISTENCE_BUFFER.
It occurs, for instance, when you run the report PRGN_COMPRESS_TIMES.

Other Terms

SAPLPRGN_AUTH
LPRGN_AUTHU25
<user_buff>

Solution

Implement the attached correction instructions or import the relevant Support Package.

A new text element will be delivered with the Support Package named below. If you implement the correction instructions to install the correction in your system, the new text on the selection screen of RSUSR_CLEANUP_USER_TABLES will initially appear only in English, regardless of your logon language.

Unfortunately, the corresponding supplements to the report documentation can only be delivered in a Support Package, as well. They are therefore described ahead of time in the following sections:

Explanations of the solution

The "Delete the user SPACE" option (technical name: sc_space) is only available if you deactivate the test mode. The report does not make it possible to check in advance whether the user SPACE exists.
This option lets you decide whether you want to delete this user - which erroneously exists - as described in SAP Note 441215. This option is selected by default.
When you activate it, the report deletes the USR02 entry of this user - if it exists - and displays all other occurrences of the user in other tables, together with all inconsistencies, in the results list. You can deal with these entries there, just like you would handle any other inconsistent entries.

Symptom

I receive the following error in Managed Gateway for Spend&Network transaction tracker when sending an Order Request:

Access Control check validation failed for the Authentication. associated to User XXX is not matching with ANXXX defined in the Project Cross-Reference.. Suggested Action: Please verify if the Supplier Cross-reference ID in the payload matches with the project Sender Cross-reference configuration.Please contact support with ErrorCode:CIG-PLT-00639. 

Environment

  • SAP Ariba Buying and Invoicing (B&I)
  • Managed Gateway for Spend&Network
  • SAP S/4HANA

Cause

The issue occurs because the payload sent from SAP S/4HANA is missing the required attribute SenderBusinessSystemID.

Without this attribute, the system cannot correctly validate the sender against the Project Cross-Reference configuration, leading to the authentication validation failure.

Resolution

You verify and correct the payload configuration in SAP S/4HANA.

  1. Review the preprocess payload of the affected document
  2. Check if the attribute SenderBusinessSystemID is present
  3. If the attribute is missing, work with your implementation partner or consulting team to perform the necessary configuration changes, as this involves system setup and payload structure adjustments.
  4. Ensure that the value of SenderBusinessSystemID contains the correct SAP S/4HANA system ID
  5. Confirm that the sender configuration matches the Project Cross-Reference in Managed Gateway for Spend&Network

Symptom

  • Blank page when going to Destinations for OneDrive
  • If you check on WACS using port 6405 (localhost:6405/BOE/BI) works fine, proving the configuration is correct
  • Webapp trace for BI launchpad shows error
    • 2026 02 05 09:40:31.647|HttpSender::sendRequest
      2026 02 05 09:40:31.647|HttpSender::sendRequest; first call
      [GET] https://graph.microsoft.com:443/v1.0/me/drives
      2026 02 05 09:40:31.717|General exception caught
      java.lang.RuntimeException: Error while silently connecting: org.w3c.www.protocol.http.HttpException: Peer certificate rejected by ChainVerifier
  • An SSL trace with IAIK debug records (see SAP KBA 2673775) shows the following messages
    • sl_debug(1): ChainVerifier: No trusted certificate found, rejected.
      ssl_debug(1): Sending alert: Alert Fatal: bad certificate
      ssl_debug(1): Shutting down SSL layer...
      ssl_debug(1): SSLException while handshaking: Peer certificate rejected by ChainVerifier

Environment

  • SAP BusinessObjects BI Platform 4.3
  • SAP NetWeaver

Reproducing the Issue

  1. Login BI Launchpad
  2. Go to Settings -->"Authorization Tokens"
  3. Click "Generate"
  4. Login Microsoft 
  5. See that Token has been generated
  6. Go to "Folders"
  7. Right-click any report --> Schedule
  8. Under Destination select "OneDrive"
  9. Observe that it is not possible to select a OneDrive folder, as none are displayed

Cause

  • Error Message: "ChainVerifier: No trusted certificate found, rejected"
  • Missing Component: certificate not present in NetWeaver Java truststore (in this case was DigiCert Global Root CA)
  • Trust Store Status: Missing root certificate authority for chain validation

Resolution

  1. Get the Certificate that is missing based on the logs
  2. Import it to the SAP Netweaver Trustore
  3. This is the same as documented in this KBA  3076281 - Peer certificate rejected by ChainVerifier - No trusted certificate found, rejected - SAP for Me

Symptom

After adding a Global Assignment via Import or API the job relationships have not been created automatically by the system.

Environment

SAP SuccessFactors - Employee Central

Reproducing the Issue

  1. Import Global Assignment Employment
  2. After Import go to the global assignment profile for the user
  3. View their Job Relationship portlet
  4. Verify that no Job Relationship records have been added automatically by the system

Resolution

Currently position follow-up processes are not triggered for changes from import/API with the global assignment and higher duty event reasons.
If the Global Assignment is added via the UI the Job Relationship records will be created successfully by the system.

Please see this section of the help portal for more information on this:
Help Portal: Position Management - Follow-Up Processes

If you would like to have this functionality included in a future release we would encourage you to consider creating an enhancement request:

Symptom

  • Both delete job and store job are set as 'start automatically' in customizing.
  • The delete job have been executed automatically after write job finished.
  • The session is still pending for delete job, and listing as 'incomplete session'.

Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.

Environment

All R/3 release.

Reproducing the Issue

  1. Start the write job in SARA.
  2. The auto-start delete and store job finished successfully.
  3. If go to SARA->delete, a session is still pending for releasement.

Cause

The 'delete with test variant' was checked in variant for write job.
This will cause the delete job run in test mode and no data will be removed from the system.
The system thinks delete job not be executed yet.

Resolution

Deselect 'delete with test variant' in write job variant.

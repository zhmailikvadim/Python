Symptom

  •  Text rendering in exported PDF file is not the same as in the previous version of Crystal Reports

Environment

  • SAP Crystal Reports 2013
  • SAP Crystal Reports 2016

Reproducing the Issue

  1. Export report in two versions of Crystal Reports from two different environments
  2. Observe results are not the same

Cause

There are multiple potential causes:

  1. Fonts difference
  2. Text rendering dll - usp10.dll 
  3. Printer Driver 
  4. Exporting dll - crxf_pdf.dll
  5. Registry keys

Resolution

  1. Compare the fonts used in the report and installed on the machines (C:/Windows/Fonts), see  2214614 - Incorrect font displayed when exporting a report to PDF format from Crystal Reports, and using an Open Type Font
  2. Open the report and go to Help > About Crystal > More info. Compare the versions of usp10.dll. Note if usp10.dll loaded from Windows location then not possible to replace, but if the usp10.dll loaded from SAP Business Objects folder then it is possible to replace usp10.dll. See: 1853617 - Characters overlap when printing a report from Crystal Reports
  3. Go to Page Setup and compare the printers associated with the report. See:   1835763 - Text truncates when exporting a report to PDF format from Crystal Reports
  4. Go to Help > About Crystal > More info and compare versions of crxf_pdf.dll. Note, replacing the .dll is not supported
  5. There are 4 registry keys in charge of Exporting to PDF. See: 1835763 - Text truncates when exporting a report to PDF format from Crystal Reports, 2469456 - Incorrect characters in PDF export results, if the function urlDecode is used in Crystal Reports

Symptom

  • If the data provider details are previously asked for, then Web Intelligence RESTful Web Service returns no data.

Other Terms

Webi, Raylight, webintelligence, dp, queryplan, dataprovider

Reason and Prerequisites

Environment:

  • SAP BusinessObjects Business Intelligence Platform 4.1
  • SAP BusinessObjects Business Intelligence Platform 4.2

Step To Reproduce:

  1. Request the data of a Web Intelligence query through Web Intelligence RESTful Web Service. For example:

call:

[GET] raylight/v1/documents/9476/dataproviders/DP0/flows/0

Accept: text/xml

response:

<DATA_PROVIDERS ...>
  <DATA_PROVIDER ...>
    <COLUMN ...>  
    ... 
    <COLUMN ...> 
    <ROW>
      <CELL INDEX="0">Texas</CELL> 
      <CELL INDEX="1">Houston</CELL> 
      <CELL INDEX="2">e-Fashion Houston Leighton</CELL>
    </ROW>  
    ...
    <ROW> 
      <CELL INDEX="0">Texas</CELL>  
      <CELL INDEX="1">Houston</CELL>  
      <CELL INDEX="2">e-Fashion Houston</CELL>
    </ROW>
  </DATA_PROVIDER>
</DATA_PROVIDERS>

>> See that data are correctly returned in the <ROW> elements.

  1. Get this data provider details:

call:

[GET] raylight/v1/documents/9476/dataproviders/DP0

Accept: application/xml

response:

<dataprovider ...>
  <id>DP0</id> 
  ...
  <query>...</query>
</dataprovider>

  1. Do again the same call as in 1. :

call:

[GET] raylight/v1/documents/9476/dataproviders/DP0/flows/0

Accept: text/xml

response:

<DATA_PROVIDERS ...>
  <DATA_PROVIDER ...>
    <COLUMN ...>  
    ... 
    <COLUMN ...>  
  </DATA_PROVIDER>
</DATA_PROVIDERS>

>> See that there are no more data.

Cause:

  • Querying the execution plan (computed through a separate call, but also through the data provider details as well), was resetting the data.

Solution

This issue is fixed in the patches listed in the "Support Packages & Patches" section below.
The "Support Packages & Patches" section will be populated with the relevant patch levels once they are released.

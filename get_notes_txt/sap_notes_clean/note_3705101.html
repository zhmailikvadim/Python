Symptom

  • In Manage Footprint Results, aggregated status is marked as "Successful" even when at least one input item has a status of "Error".
  • The error was identified using plausibility checks.

Environment

Sustainability Footprint Management

Reproducing the Issue

  1. Use a product within a production (goods issue to production) without having an opening stock or any goods receipt for the specific product.
  2. Perform a footprint calculation for the production.
  3. Observe that the aggregated status is marked as "Successful" despite one or more input items having a status of "Error".

Cause

The behavior is caused by a coding issue where the aggregated status does not correctly reflect "Error" when at least one input item has an error status.

Resolution

A fix was developed and should reach all tenants by January 30th. Please follow this KBA for updates or create a case for component SUS-PFM.

Keywords

Aggregated Status, Error Status, Production Calculation, Plausibility Checks, Coding Issue, Bug, Input Item Error, Inconsistent Status, Incorrect Status, Sustainability Footprint Management, SFM.

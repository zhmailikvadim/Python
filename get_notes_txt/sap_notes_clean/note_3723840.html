Symptom

  • A Dispute Case is created when there is a mismatch between the expected invoice amount from the Freight Order and the amount submitted by the carrier in the Carrier Invoice.
  • However, when navigating to the Dispute Case in the Fiori app and clicking the "Edit" button, the document does not switch to edit mode. No error message is displayed, and the document remains in display mode.
  • Additionally, in the Dispute Case, the Workflow Status is "Awaiting Approval".

Environment

SAP Transportation Management Embedded in S/4HANA 1709 and above releases

Reproducing the Issue

  1. Create a Freight Order and send the document to the carrier
  2. In the Carrier Collaboration Portal, the carrier submits a Carrier Invoice to SAP TM
  3. Due to a mismatch between the expected invoice value in the Freight Order and the proposed invoice value in the Carrier Invoice, a Dispute Case is created
  4. Navigate to the Dispute Case in the Fiori app
  5. Click on the "Edit" button
  6. The document remains in display mode and cannot be edited

Cause

  • This is a standard system behavior. When the Approval Workflow is enabled in the Settlement Profile, the system triggers workflow WS61200016 to process the Dispute Case.
  • In this scenario, the Dispute Case is sent for approval and only the users assigned as Level-1 Approver and Level-2 Approver in the relevant Organizational Unit are authorized to approve or reject the Dispute.
  • While the Workflow Status is set to "Awaiting Approval", the approval decision is still pending. During this phase, the Dispute Case is locked for changes and cannot be switched to edit mode. Therefore, the Edit button does not activate editing functionality.

Resolution

  • When the Approval Workflow is enabled, the Dispute Case can only be edited after the workflow process has been completed. This occurs once the assigned Level-1 and Level-2 approvers process the Dispute Case and the Workflow Status changes to either "Automatically Approved" or "Completed". Only after reaching one of these statuses the document becomes editable again.
  • If the Approval Workflow should not be used in the business process, the "Approval Workflow" checkbox must be disabled in the corresponding Settlement Profile via the following customizing path:
    • Transaction SPRO -> Transportation Management -> Settlement -> Define Settlement Profile

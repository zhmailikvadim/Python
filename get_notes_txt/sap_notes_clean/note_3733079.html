Symptom

Let's say there is a Quality system and the Production system which have SAP BASIS 758 (where only the new client copy tools has to be used as per SAP note 2962811 - New Client Copy Tool: General Information ) and there is a requirement to refresh the Quality system with the data of Production system. Before the system refresh of Quality system, the user master and authorization data was exported with old client copy tool SCC8.

However, after the system refresh during the import of user master and authorization data neither SCC7 or SCC7N is working:

  • SCC7 is failing as the system provide a pop-up message that the SCC7 is obsolete. 
  • SCC7N is failing with error "Profile not existing".

Environment

  • SAP BASIS 758
  • AS ABAP

Reproducing the Issue

  • Export with SCC8
  • Import with SCC7 or with SCC7N

Cause

  • SCC7 is failing as it is obsolete, because from SAP_BASIS 758 and above, old Client Copy Tools are deprecated as per note 2962811 - New Client Copy Tool: General Information
  • SCC7N is failing as the export was done with old client copy tool SCC8

Resolution

Please use / implement SAP Note 3629789 - Revoke SCC7 transaction for clients exported using SCC8 to allow SCC7 to be executed if you had done the export via SCC8.

See Also

  • 2962811 - New Client Copy Tool: General Information
  • 3629789 - Revoke SCC7 transaction for clients exported using SCC8

Symptom

The user carries out the following steps:

  1. Creates a payment request with a reference document.
  2. Runs a payment proposal for the payment request and enters the payment request into FPRL_ITEM.
  3. Reverses the payment request in F8REV.

The expected result is that if the reference document cannot be reversed, the payment request also cannot be reversed, to keep the data consistent. However the actual result is that even if an error in F8REV shows that the reference document cannot be reversed, the payment request is still reversed, causing inconsistent data.

Other Terms

EPIC, payment request, F8REV, reverse

Reason and Prerequisites

In F8REV, BTE 1011 should be used to check whether or not a reference document can be reversed, instead of 1025.

Solution

Implement this note and add a new entry in FIBF as described under Manual Activities.

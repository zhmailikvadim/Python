Symptom

When validating recoverability using the SAP HANA recovery validation tools, the check fails when the recovery chain involves a Backint-based full data backup, while the validation succeeds when the chain relies on file system based full data backups.

Example command:

python hdbrecovercheck.py checkPointInTimeReachable \
-D <DBNAME> \
-b \
--backintDataParamFile <backint_parameter_file> \
-t "<timestamp>" \
-v

The command executes but fails during the validation of the Backint full data backup.

Example error patterns:

ERROR: hdbrecovercheck.py checkPointInTimeReachable got unexpected error
Error occurred while fetching list of backups

or

ERROR [110203] Not all data could be written:
Expected 4096 but transferred 0

Observations:

  • Recoverability check succeeds when the recovery chain is based on file system full data backups + log backups
  • Recoverability check fails when the recovery chain depends on a Backint full data backup
  • Backup jobs reported by the third-party backup solution appear successful

Environment

SAP HANA DB Version 2.0

Cause

The recoverability validation tools (hdbrecovercheck.py, hdbbackupdiag) verify the backup chain based on the SAP HANA backup catalog and the configured backup interfaces.

If the relevant full data backup was created using Backint, the validation process interacts with the configured Backint interface to access the backup metadata required for validation.

In certain situations, the validation step may fail when the recoverability check attempts to access information through the Backint interface, even though:

  • the backup job is reported as successful by the third-party backup solution
  • the corresponding entry exists in the SAP HANA backup catalog

In such cases the SAP HANA backup catalog and recovery framework may still contain valid entries, but the recoverability validation cannot be completed successfully during the interaction with the Backint interface.

Resolution

To further analyze the situation, execute the recoverability validation using the SAP HANA diagnostic utility hdbbackupdiag and collect the relevant outputs.

Example command (adapt parameters as required for the environment):

hdbbackupdiag \
--check \
-v \
-t -B \
-u "<timestamp>" \
--databaseName <DBNAME> \
--useBackintForCatalog \
--backintCatalogParamFile <backint_parameter_file>

Execute the command as <sid>adm on the SAP HANA host.

Collect and review the following information:

  • Output of the hdbbackupdiag command  
  • SAP HANA trace files generated during the execution  
  • Logs from the configured Backint backup solution  
  • Backint parameter file used for the database configuration  

If the recoverability validation fails only when the recovery chain includes a Backint full data backup, additional analysis of the Backint configuration and the interaction between SAP HANA and the configured Backint interface may be required.

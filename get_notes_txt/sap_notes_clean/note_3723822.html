Symptom

During migration to the new price table FMLT_PRICE, records with identical Cost Component Split data but differing additional key fields (KKZST, KEART, PATNR, DIPA) are not migrated correctly.

Other Terms

FML_PRC_TRANSITION, Price Transition Launchpad, Migration to FMLT_PRICE

Reason and Prerequisites

Previously, during migration to the new price table FMLT_PRICE, records with identical Cost Component Split data were compressed irrespective of their additional key fields values.

Solution

Implement attached correction instructions. Alternatively, install appropriate Support Package mentioned in this SAP Note.

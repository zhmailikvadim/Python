Symptom

The user changes an FGU account in loss management and saves. The system issues the following error messages:

  • Error when accessing the business object layer /MSG/RTWAPLANWK
  • Inconsistency between buffer and parameter in method SET_XTABLE / class /MSG/CL_R_FW_BO_ELEMENT
  • Operation cannot be executed due to data processing errors

Other Terms

/MSG/CX_R_FW_BO_EXCEPTION,

Reason and Prerequisites

Prerequisites:

  • Before saving, the user has already saved a change to the same FGU account.
  • The covering treaty has the nature of treaty “NPOBL” and an active reinstatement.
  • The loss has an FGU account. The amount has an amount that is relevant for the calculation of reinstatement premiums.
  • The loss has a preview.
  • The covering treaty still exists in a loss other than coverage.

Reason: the system does not update a worktable with all available primary key fields. As a result, the system updates the data record of another loss.

Solution

History:
January 30, 2026: Newly created

Troubleshooting Description:
The system updates a work table with all available primary key fields. The correct record is updated. The system saves the change to the FGU account without errors.

Manual activities:

Manual pre-implementation steps are required.

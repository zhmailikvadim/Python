Symptom

When replicating data from HR to SRM, the error message "Determination of connections to object 'S' of 'CP' failed" (Message No. BBP_UM_UI036) occurs. This issue often arises when there is an employee with concurrent employment and one active job along with two withdrawn jobs.

Environment

  • SAP ERP
  • SAP SRM

Reproducing the Issue

  1. Identify the employee with concurrent employment having one active job and two withdrawn jobs.
  2. Attempt to replicate the employee data from the source ECC system to the target SRM system.
  3. Observe the error message "Determination of connections to object 'S' of 'CP' failed" (Message No. BBP_UM_UI036) in the SRM system.

Cause

The error occurs due to missing data in the iDocs that are supposed to create the CP→S relationship in the target system.

Resolution

To resolve the issue, follow these steps:

  1. Execute transaction PFAL in the source ECC system for the affected employee in Update mode. Include the following infotypes:

    • Infotype 0: Actions
    • Infotype 1: Organizational Assignment
    • Infotype 2: Personal Data
    • Infotype 3: Planned Working Time
    • Infotype 6: Addresses

  2. After executing PFAL, verify that the correct data has been sent via iDoc with message type HRMD_ABA.

  3. Ensure there is a successful posting of the application document for the iDoc to confirm that the data has been replicated correctly.

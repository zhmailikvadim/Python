Symptom

A segment with a range of activity types cannot be created.

Environment

  • Controlling (CO)
  • SAP S/4HANA
  • SAP S/4HANA Finance
  • SAP S/4HANA Cloud Private Edition
  • SAP Fiori for SAP S/4HANA

Reproducing the Issue

  1. Navigate to Fiori App Manage Allocations - F3338, receiver details.
  2. There is no "range" option for activity type selection.

Cause

  • The check was added in the S4HANA 2023 release as the Activity type was delivered as default configuration for receiver basis.
  • There is no automatic unit conversion, and input in intervals for Activity type (LSTAR) and Stat. Key figure (STAGR) was disabled due to potential errors and limited use cases where units in such intervals can be added together.

Resolution

The check was added in the S4HANA 2023 release as the Activity type was delivered as default configuration for receiver basis. There is no automatic unit conversion and therefore, input in intervals for Activity type (LSTAR) and Stat. Key figure (STAGR) was disabled. It is prone to errors and there is rarely a case where the units in such interval can be added together.
 
As a standard solution, this will not be changed. It's able to utilize the intervals in a Group - but this needs to be under user's consideration to input only intervals with compatible units. Otherwise the results will not be correct. Same goes for the SKF option as receiver basis selection.

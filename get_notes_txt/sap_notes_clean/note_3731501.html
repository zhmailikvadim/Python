Symptom

The system raises the following error:  "Batch & of material & is already locked by &." in Post Goods Movement app.

Environment

 
SAP S/4HANA Cloud Public Edition



Cause

The error occurs because multiple users attempt to post goods movements to the same batch at the same. This is standard system behavior, designed to ensure data consistency, prevent simultaneous updates, and maintain inventory accuracy.

In SAP S/4HANA Cloud Public Edition, a batch is unique at the material level, meaning a batch number belongs uniquely to one material. The lock object is applied at the material & batch level, restricting parallel processing of the same batch for the same material.

Resolution

The following operational approaches are recommend:

  • Post immediately after batch determination, avoid leaving Post Goods Movement app open after batch proposal.
  • Exit Post Goods Movement app if not posting immediately; the lock is released once the app is exited.
  • Operational alignment between users, coordinate batch usage when working with high-demand materials.

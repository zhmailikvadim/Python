Symptom

  • The Message Monitor overview is very slow to load.
  • After loading, no message data is displayed in the overview.
  • The system is otherwise operational and other functionalities work without delays.
  • No error messages or error codes were provided.

Environment

  • Product: SAP NetWeaver Process Integration, adapters
  • SAP Process Integration (PI) related performance issues

Reproducing the Issue

  1. Start the PI/PO production system.
  2. Navigate to the Message Monitor overview.
  3. Observe that the page loads slowly and no data is displayed.

Cause

  • the Message Monitor is not able to correctly retrieve the message data, which results in an empty overview. At the same time, the system may take longer to load as it attempts to read incomplete or inconsistent entries from the database.

Resolution

Tune messaging resources for the Adapter Engine:

  1. In NWA, open Java System Properties > Services > “XPI Service: SAP XI AF Core”.
  2. Locate the property messaging.connectionDefinition.
  3. Add or adjust the adapter-specific configuration after the global definition to increase consumer threads, for example:

(name=global, messageListener=localejbs/AFWListener, exceptionListener=localejbs/AFWListener, pollInterval=60000, pollAttempts=60, Send.maxConsumers=15, Recv.maxConsumers=15, Call.maxConsumers=15, Rqst.maxConsumers=15)

You may set maxConsumers between 15–20 depending on adapter usage.

     4. Save and restart the system for changes to take effect.

Symptom

When viewing the list of mass change jobs in the Fiori app "Monitor Mass Change of Sales Documents" F5210 you notice that the start date or start time of the job is incorrect.  When compared against the start time of the job in SM37, or when compared against other fields in the List Report such as "Last Changed Date/Time" it can be offset by one or more hours.

Reason and Prerequisites

There is a program error. The date and time are stored in the database in UTC format but the values are converted as if they were generated using the system time zone.

Solution

The solution will be included in the Support Package appropriate for each release specified in this SAP Note. In urgent cases you may implement the attached correction instructions.

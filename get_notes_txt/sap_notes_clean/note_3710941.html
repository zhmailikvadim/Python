Symptom

When attempting to access learning administration, the system authenticates the user using the LMS admin account that matches the BizX User ID rather than the linked account associated with the LMS user record.

For example, for a BizX user with ID '123', an LMS user with ID '123' will necessarily exist. This user is related to LMS Admin ID '1234'. However, when authenticating the user on the learning administration side, the system authenticates the account as LMS Admin ID '123'.

Environment

SAP SuccessFactors Learning

Reproducing the Issue

  1. Log into the system >
  2. Navigate to Learning Administration > People >
  3. Search for the affected User ID >
  4. Verify the related admin account linked to the user record >
  5. Return to the SuccessFactors Home Page >
  6. Access Learning Administration >
  7. Navigate to Preferences in the administration side >
  8. Observe that the system authenticates the administrator based on the matching LMS Admin ID instead of the linked admin account.

Cause

 The system logic prioritizes implicit mapping over explicit mapping when authenticating users on the learning administration side. Implicit mapping checks for a corresponding/matching LMS Admin ID, which overrides the linked admin account (explicit mapping).

Resolution

This is how the system logic operates. An enhancement request would be required for a different system behavior. Alternatively, if needed, users can access the administration side from the 'Learning Administration' option in the 'Dashboards' dropdown menu on their Learning Home Page. When accessing the admin side this way, the system authenticates the account in accordance with the related admin.

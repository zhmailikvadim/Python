Symptom

Other Terms

ESX, leftstr, expression, calculation view

Reason and Prerequisites

Issue number 350911

Reason:

Due to programming error in SAP HANA, the leftstr has not been correctly considered in ESX engine. While the leftstr function should check in bytes, it has checked in characters when running with ESX engine. Hence, even when situations where no rows should be expected, some rows may be returned unexpectedly if it's calculated by leftstr and executed by ESX engine.

For more information, appropriate functions should be well chosen depending on your requests, and regarding functions, please find the information from String Functions.

Affected Releases:

  • SAP HANA 2:
    • Revisions of SPS00 - SPS08

Prerequisites:

  • There are leftstr string functions used in expressions and calculations.

AND

  • Data contains Unicode strings.

AND

  • The logic is executed by ESX engine.

Solution

Apply one of the following SAP HANA Revisions:

  • SAP HANA 2:
    • SAP is currently working on a permanent solution to this issue. This SAP Note will be updated as soon as a revision containing the solution is available
    • or higher

  • or higher

Workaround:

  • Possible workaround would be to apply NO_USE_ESX_PLAN hint.

From the example scenario:

SELECT COL1 FROM TESTVIEW1 WITH HINT (NO_USE_ESX_PLAN); -- returns no rows when not running by ESX engine - which is expected in the scenario.

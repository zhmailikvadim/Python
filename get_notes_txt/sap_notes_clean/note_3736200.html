Symptom

  • When creating a validation row in transaction OKC7, an automatic row appears in transaction OKC9 without any substitution.
  • Double-clicking the automatically created row in OKC9 displays the error Message No. GB560 “Function module called up incorrectly”.

Environment

  • Controlling CO

  • SAP ERP Central Component

  • SAP ERP

  • SAP enhancement package for SAP ERP

  • SAP enhancement package for SAP ERP, version for SAP HANA

  • SAP S/4HANA

  • SAP S/4HANA Finance

  • SAP S/4HANA Cloud Private Edition

  • SAP Fiori for SAP S/4HANA

Reproducing the Issue

  1. Create a validation row in transaction OKC7.
  2. Open transaction OKC9 and observe that a substitution row is automatically created without content.
  3. Double-click the automatically created row in OKC9 and observe error GB560 “Function module called up incorrectly”.
  4. (Delete the row with validation from OKC7 and the blank row in substitution from OKC9 disappear again automatically.)

Cause

  • Transactions OKC7 and OKC9 are maintenance views V_TKAZE and V_TKAZES of the same underlying table TKAZE. Saving a validation in OKC7 creates a corresponding entry visible in OKC9, even if no substitution logic is defined.
  • There are no substitutions for call-up point 2; attempting to open such a placeholder line leads to error GB560 because the validation environment is invoked with an invalid event parameter.

Resolution

  1. Note that the automatic line in transaction OKC9 after maintaining OKC7 is expected due to the shared data model; it does not indicate an active substitution or have other impact.
  2. Do not attempt to maintain or create dummy substitutions for call-up point 2 in OKC9, as substitutions are not available for this call-up point.

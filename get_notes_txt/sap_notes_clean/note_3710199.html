Symptom

  • The product costing is not correct; the total value of the component B is 100x lower than expected.
  • The activity quantity is calculated as 100 EA instead of 1000 EA, leading to incorrect cost estimation.

Environment

  • Controlling (CO)
  • SAP R/3
  • SAP R/3 Enterprise 
  • SAP ERP
  • SAP ERP Central Component
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA

Reproducing the Issue

  1. Run transaction code CK11N.
  2. Review the cost estimate result.
  3. Observe that the component B in the itemization is calculated with a quantity of 100 EA instead of 1000 EA.

Cause

  • Tcode KP27 shows the price unit, but activity quantity is calculated based on the template allocation settings, not from tcode KP27.
  • The ABC template TRP211 in environment 001 was modified, but the corresponding program coding was not regenerated. As a result, the updated quantity of 1000 EA was not reflected in the template evaluation, and the system continued using the previous quantity of 100 EA.

Resolution

  1. Access transaction code CPT2.
  2. Generate the template coding for template TRP211 in environment 001.
  3. Verify that the updated quantity is reflected correctly in the program coding.
  4. Execute transaction CK11N to confirm that the component B displays the correct quantity and total value.

Symptom

If a service order ODATA or SOAP API request is sent to create a service order in released status, the status is not set to released. No error is persisted in the database or reported via API.

Other Terms

Service Order, Status Released, Service Order Creation, ODATA API, SOAP API

Reason and Prerequisites

The reason for this is an available error status in the item level during the status change. The status change event registration is called before certain validations.

The issue happens only in the create case via external API, if you already pass the released status.

Solution

Please apply the attached manual correction instruction.

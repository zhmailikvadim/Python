Symptom

  1. Can a user directly access the APM Tables/views into SAC without building virtual tables and calculation views in HANA DB ?
  2. Is there a guide on how to set up the configuration steps and create virtual tables in HANA DB?
  3. HANA Studio/Web IDE in our architecture. Where is it possible to create the calculation views and implement the business logic /joins?
  4. Which team is usually responsible for creating virtual tables and calculation views in HANA DB (APM or SAC)?

Environment

  • SAP Asset Performance Management

Resolution

  1. Virtual Tables are not a must. APM publishes views for consumption data directly. If there is a need to join data between APM entities or between APM and other applications such as S/4, then the recommendation is to create joins/unions in SAC inside SAC Model.
  2. Not required. However, if there is a plan to create Virtual Objects - the following document provides the steps: Create a Virtual Table.
  3. Joins can be created either in SAC or Datasphere can be used to connect to APM data source and create views joining entities coming in from other applications.
  4. Data can directly be joined in SAC without having to create calculation views, or use Datasphere. However, if virtual tables/calculation views need to be used, these would have to be created by the customer.

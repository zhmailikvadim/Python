Symptom

You carry out a system upgrade with the Software Upgrade Manager (SUM) or the database migration option (SUM DMO).
The phase EU_CLONE_MIG_DT_RUN terminates because R3load import processes fail. A typical import log in the directory SUM/abap/migrate_dt contains error messages like the following:

/usr/sap/B00/SUM/abap/migrate_dt/MIGRATE_DT_00033_IMP.LOG:

(DB) ERROR: DDL statement failed
 (CREATE  INDEX "TZRR_EVTC~ID1" ON "TZRR_EVTC" ( "RESULT_ID" , "XML_ID"  ) )
 DbSlExecute: rc = 99
  (SQL error 260)
  error message returned by DbSl:
invalid column name: XML_ID

Other Terms

i5/OS, iSeries, AS400, OS400, System i, IBM i, SUM, EU_CLONE_MIG_DT_RUN, R3ldctl, DMO

Reason and Prerequisites

The error occurs only on the database platform DB2 on OS400 (DB4). The structure information generated by R3ldctl in the phase EU_CLONE_MIG_DT_PRP and used by R3load in the import, was created using the wrong ABAP Dictionary tables. As a result, inconsistencies occur that are only noticed in the import.

Solution

Download the latest version of the R3ldctl patch from SAP Service Marketplace; at the very least, however, use the version attached to this SAP Note. Replace the R3ldctl in the directory /usr/sap/<SID>/SUM/abap/exe.

To avoid the error, replace the R3ldctl shortly after the target kernel has been extracted.

If the error has already occurred, proceed as follows:

Replace R3ldctl as described above.

Rename the directory /usr/sap/<SID/SUM/abap/migrate_dt.

Repeat the phase EU_CLONE_MIG_DT_RUN with the option "init". The missing directory migrate_dt and this option cause the prepared steps from the phase EU_CLONE_MIG_DT_PRP to be executed again.

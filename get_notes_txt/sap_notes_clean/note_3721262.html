Symptom

When you try to find a technical object in the Maintenance Notification Creation screen using the Basic Text Search, the response time can be slow if the list of technical objects is long.
The technical object search uses multiple fields, including TechnicalObjectDescription and FunctionalLocationLabelName. Since these fields are calculated, they contribute to the slowness.

This SAP Note enables the Metadata Extension for C_TechObjFlatVH. Within this extension, the fields included in the fuzzy search can be modified to improve performance.

Other Terms

C_TechObjFlatVH, MetadataExtension, TechnicalObjectDescription, FunctionalLocationLabelName, C_MaintWorkRequestOverviewTP,
Fuzzy search, TechnicalObject

Reason and Prerequisites

The Technical Object value help responds slowly due to the use of calculated fields. By excluding these fields from the fuzzy search in the Metadata Extension, users can significantly improve the search performance.

Solution

Implement this SAP Note.
Then configure the Metadata Extension of C_TechObjFlatVH, to get better performance.
If you set the fuzzinessThreshold to 1 the search will look for exact matching in the description, which can help with the performance.
On the other hand, if you set the defaultSearchElement attribute to false, the fuzzy search will not include the annotated field in its generated query, which results in better performance. In addition, you can adjust the other parameters of the Search annotation to further optimize performance.

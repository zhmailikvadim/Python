Symptom

The following message shows when displaying the secinfo file via T-code:SMGW ->Goto ->Expert Function ->External Security ->Display (Sec Info):


"no entries in file gw/sec_info"


Even the secinfo file is correctly created under the directory defined by parameter gw/sec_info.

The level 2 work process trace file displays the same information when reproducing this issue:

M  ThGwMonDump: received 33 bytes
M  ThGwMonDump: no entries found in gw/sec_info
M  ThGwMonDump: received 1 lines

Environment

Kernel Version Lower than 720.

Reproducing the Issue

T-code:SMGW ->Goto ->Expert Function ->External Security ->Display (Sec Info)

Resolution

Please change the value of parameter gw/tcp_security to 1.

This parameter could be dynamically changed, no system restart is required.

Keywords

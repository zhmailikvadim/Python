Symptom

Write or Store process is done and delete program should be execute but in the Archive Selection screen sessions can not be selected.

"Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental." 

Environment

Release version independent 

Cause

Prerequisites for executing the deletion program are not met.

If the archive file generated during Write process or the stored file can not be read and verified then archive session can not be deleted, this is standard behavior to prevent data loss.

Scenario 1:

Delete Before Storing have been chosen but the but the archive file generated during Write process can not be accessed. 

Scenario 2:

Storage using ILM is done but there is no enough authorization.

Resolution

1.Check the archive file generated during Write process exist on filesystem level and make sure it can accessed.

2.Assign a proper role which has authorization object S_ILM_SRS with activity 16.

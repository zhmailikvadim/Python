Symptom

  • ALEAUD IDoc transferring from MDG to ECC system via PIPO does not contain the sender IDoc number in E1STATE DOCNUM.
  • The status of the original IDoc in MDG remains at 03 instead of changing to 41.
  • ALEAUD Special Handling is enabled in PIPO, but the original IDoc number is not reflected in MDG.

Environment

SAP NetWeaver 7.50

Cause

  • The issue is related to configurations in the ALEAUD communication channel in PIPO. The destination is not configured correctly.

Resolution

  1. Verify the ALEAUD communication channel configuration in PIPO. Ensure the option "ALEAUD Special Handling Needed" is enabled.
    -  Refer to SAP Note 1786132 - FAQ: 7.5 Java IDoc (IDoc_AAE) Adapter and focus on point 3.2 regarding ALEAUD handling with XI Messages.
  2. Verify ALEAUD destination is pointing to correct system.

Symptom

  • Wingding font is ignored when a Web Intelligence document is exported to PDF.
  • Issue only from BI Launchpad.
  • No issue in Web Intelligence Rich Client
  • Issue with specific RHEL versions, no issue if we use RHEL 6.4
  • Below error message captured in traces:
    FindPhysicalFont  Font not Found(wingding.ttf) in Path :/home/boeuser/BI42/sap_bobj/enterprise_xi40/linux_x64//fonts/ttf/:/home/boeuser/BI42/sap_bobj/enterprise_xi40/linux_x86/crpe/fonts/:/home/boeuser/BI42/sap_bobj/enterprise_xi40/linux_x64/crpe/fonts/:/usr/lib/X11/fonts/TTF/:/usr/share/fonts/ja/TrueType/:/usr/share/fonts/ko/TrueType/:/usr/share/fonts/zh_CN/TrueType/:/usr/share/fonts/zh_TW/TrueType/ [kresovlefontname.cpp;225] 

Other Terms

webi, webintelligence

Reason and Prerequisites

Environment:

  • SAP BusinessObject Business Intelligence 4.2 Support Package 05
  • BI Launchpad
  • RHEL7.2

Steps to Reproduce:

  1. Setup an environment with Wingding font configured
  2. Create a Web Intelligence document
  3. Choose “No Data Source”
  4. Select the report header which is currently displaying ”Report 1”
  5. Set the font to Wingding
  6. Export the Web Intelligence document to PDF format
  7. Open the PDF
    1. >> It still displays text as “Report 1”

Cause:

  • Wingding font is not installed on the Linux server

Solution

Make sure that the font wingdings.ttf is installed on your Linux box, for that please install the Microsoft font package.
Make sure that PHYSICAL attribute of FONTATTRIBUTE tag in fontalias.xml is equal to the name of the font file on your disk.
Restart the WebIntelligenceProcessingServer

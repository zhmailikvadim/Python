Symptom

When I try upload a document on Event and other documents section of my guided sourcing project, I receive the error: Sorry we could not reach the Ariba service.

Cause

The user does not have the necessary role inside the guided sourcing project to do the action.

Resolution

You must have the Active Team Member or Project Owner role to upload documents within the project. If you do not have one of these roles, you will not be able to add files to the project.

To add the roles to the project group you can follow the documentation: How do I edit a project group's roles?. 

Please note if Can owner edit this Project Group rule is set to No, the adjustment can only be made in the template, not in the project.

See Also

If you want to know more about the roles and permissions you can refer to the documentation: What roles can I assign to my project team members?.

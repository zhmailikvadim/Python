Symptom

You are using the DRFOUT transaction for PMAT replication of Sales Prices from S4 to CARAB in SAP Customer Activity Repository applications bundle 6.0 SPS02. You have encountered that the replication is not including Price Change information for all stores despite price changes being set at Store Level, CARAB only receives data for a single store. Existing price records in CARAB are being deleted and replaced by new ones, leaving almost all stores without pricing data.

Other Terms

DDF, DRFOUT, Replication, Price Replication, /DMF/PRC, Store, PMAT

Reason and Prerequisites

There is program error in /DMF/CL_MDIF_PRODUCT_AMDP - UPDATE method.

Solution

Implement the attached backend corrections in your system.

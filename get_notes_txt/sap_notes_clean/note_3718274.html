Symptom

  • Initial load fails with the error: "Preparation steps in SLT failed."
  • Error message: "Failed to create triggers for table /BIC/* with logging table /1CADMC/*"
  • Error message: "SQL1667N The operation failed because the operation is not supported with the type of the specified table. 
  • Error code: SQL_CODE:1667

Environment

SAP on Db2 for LUW
SAP Landscape Transformation Replication Server (SLT)

Reproducing the Issue

  1. Attempt to replicate the table /BIC/* using SLT.
  2. Observe that the replication fails during the preparation steps in SLT.
  3. Check SLG1 logs for the error messages related to trigger creation failure and SQL1667N error code.

Cause

  • The table /BIC/* is a column-organized table.
  • Creating triggers for column-organized tables is not supported in Db2.

Resolution

  1. Confirm that the table /BIC/* is a column-organized table by running the following query:

`SELECT tableorg FROM syscat.tables WHERE tabname = '/BIC/*'`

If the result is "C", the table is column-organized.

  1. Verify the database type by checking the table RSADMIN for Db2 Blu settings.
  2. Avoid creating triggers on column-organized tables in Db2, as this operation is not supported.

See Also

Restrictions for column-organized tables

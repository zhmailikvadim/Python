Symptom

You upload a Price Review for a customer via transaction /IRM/GPRUPL and expect the system to retain the Pricing Unit value provided in the upload file (for example, “0”). When the upload is completed, the system automatically updates the Pricing Unit to “1” instead of keeping the uploaded value. Pricing unit is updated as “1”; ideally, pricing unit should be updated as “0” as we are trying to upload the distributor to the existing Agreement. This results in incorrect pricing data being stored for the customer.

This note corrects the behavior by ensuring that the Pricing Unit value provided in the upload file is retained and not defaulted to “1” during processing.

Other Terms

Download/Upload

Reason and Prerequisites

This is caused by Program Error.

Solution

Please implement the correction instructions. These corrections are only valid if you have applied the following Support Packs.

Release V/4 – 2021, 2022, 2022V, 2023, 2023V2, 2023 V3, and 2023 V3 Support Package 1.

Use transaction SNOTE to implement the correction instructions specified below and activate the affected program object

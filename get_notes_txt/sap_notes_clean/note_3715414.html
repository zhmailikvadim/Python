Symptom

Data Extractor for the Uniform Wage Notification Procedure (Report RPLELMC0), Version ELM 5: Cross-Border Employees France with Special Agreement, that is, the cantons VD, NE, JU, SO, BS, BL, BE, and VS:  

1. Can these be maintained in IT0038 with a pay scale other than SFN?

2. Which domains in RPLELMC0 can be used to report these employees?

3. Domain WHT: Why is the pay scale SFN not visible in the ALV view "Wages"?

Reproduce Issue




Solution

Answer 1:

1. Cross-border employees in France cantons with special agreement, that is, VD, NE, JU, SO, BS, BL BE, and VS, must be assigned in IT0038 with
- WTxLiab = 4
- Rate = SFN 
must be maintained

Conversely, cross-border employees in France cantons without a special agreement can only be
- WTxLiab = 4
- Rate <> (not equal to) SFN 
must be maintained.

--> See Swissdec - Guidelines for Wage Data Processing - Wage Standard CH (ELM) Version 5.0 or Swissdec - Addendum to Guidelines Wage Standard CH (ELM) Version 5.3


Answer 2:

Cross-border employees in France cantons with special agreement:

1.  ..... are first reported with the general domain WHT (monthly declaration).

2. ..... must also be reported with the domain Cross-Border Employees, Application Key = QSTG (Annual Report). 
(Alternatively, the application key of the domain Cross-Border Employees can also remain empty (blank).)

3.  ....., if information about teleworking was also made in infotype IT3441, the domain Cross-Border Employees, AppKey = QGFR (Annual Notification) is also used. 
(The limit value 40% for teleworking according to Swissdec - Addendum to Guidelines Wage Standard CH (ELM) Version 5.3 is NOT checked by the software.)

Note that the address data in IT0006 is also correct: Cross-border employees France must have an address with 'Country/Reg. Key = FR (France).
After you have corrected the master data, also run payroll (RPCALCC0*) if it is still possible for the period.
Otherwise, RPLELMC0 does not process the PERNR or an error message is generated: No wages found - notification cannot be created.



Answer 3:

According to Guidelines for Wage Data Processing - Wage Standard CH (ELM) Version 5.0, 9.5.2:
..."Instead of a WHT pay scale code, the predefined category SFN is assigned for qsP that fall under this special agreement...."

The pay scale code SFN is therefore not equivalent to other pay scale codes and is therefore not displayed in the ALV view "Wages".
The indicator 01 ("specialAgreement" in 4.0, "SFN" as of 5) in the field "vQK.oTariff", on the other hand, displays the cross-border employees France with a special agreement.
(The XML contains the tariff code SFN: <CategoryPredefined>.)

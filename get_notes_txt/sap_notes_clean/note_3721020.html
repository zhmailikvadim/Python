Symptom

When you generate the payment medium for the bank (transaction PC00_Mxx_FFOT), the status of the payment run is "Reserved for external payment run", even though the report pre-DME program like (RPCDTBQ0_CE) has been executed successfully without any error.

Other Terms

DME, Payment, Concurrent employment, CE.

Reason and Prerequisites

Reason: In the CE pre-DME program for separate payment RPCDTBQ0_CE, the system does not update the run status in the table REGUV before you exit the ALV List.

Prerequisites:

You execute the report RPCDTBQ0_CE (Preliminary Data Medium Exchange - for separate payment run). While viewing the ALV List, you execute transaction PC00_Mxx_FFOT or you exit the session.

The problem does not occur when you exit the log in program RPCDTBQ0_CE by choosing Back (F3) before starting PC00_Mxx_FFOT. In this case, the system correctly updates REGUV table and displays the status as "Payment run has been carried out".

Solution

Implement the attached correction instructions or import the relevant HR Support Package for your system.

The corrections in this note ensure that the system updates the status of REGUV before the ALV is displayed.

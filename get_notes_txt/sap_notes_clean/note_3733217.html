Symptom

Submission of R-9000 event returns status "Failed"; other events submit successfully.

Environment

  • SAP S/4HANA
  • SAP S/4HANA Cloud Private Edition

Reproducing the Issue

  1. Navigate to the "Run Compliance Reports" app in Fiori.
  2. Select the relevant report for Brazil EFD-Reinf Exclusion Files.
  3. Generate the file for the R-9000 closing event.
  4. Attempt to submit the event to the tax authorities.
  5. Observe that the submission status updates to "Failed".

Cause

The failure is caused by an incorrect value maintained for the ENVIRONMENTTYPE parameter. An extra space in this parameter’s value in the configuration view SRFVC_RPG_ENT leads to a communication error during submission.

Resolution

  1. Go to the following SPRO path: Cross-Application Components > General Application Functions > Document and Reporting Compliance > Country/Region-Specific Settings > Brazil > Statutory Reporting > EFD-Reinf Exclusion Files > General Settings > Setting Up Your Statutory Reporting.
  2. Select the reporting entity you use to report Brazil EFD-Reinf.
  3. Go to the properties specific to the reporting activity.
  4. Locate the ENVIRONMENTTYPE parameter.
  5. Remove any leading, trailing, or embedded spaces so that the parameter value contains no spaces.
  6. Save the change.

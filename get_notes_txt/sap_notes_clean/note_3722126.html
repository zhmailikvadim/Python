Medium 1392968/2025 Release check shows old values despite config deletion in BRH A release check displayed old release check messages, which were deleted from the configuration prior to the creation of the release decision item. To resolve this issue, the filtering logic for release check messages has been enhanced to ensure release checks always show the valid release check messages. Testing should cover the configuration of release check messages and ensure the business rules return the expected messages for release checks. N n/a
Medium 224587/2026 Error: HTTP code: 413, status reason: Request Entity Too Large, Destination: BP_IHLS

With the classic (now deprecated) setup of Integration Hub for Life Sciences, larger payloads of up to 1 MB were not supported. This restriction has been changed to allow message payloads up to 1 MB. However, payloads that exceed 1 MB still aren't supported in the classic setup and generate errors. Payloads that exceed 1 MB are only supported with the new simplified setup.

Note:

The classic setup of Integration Hub for Life Sciences will be sunset with release 2611 of SAP Batch Release Hub for Life Sciences. If you are still using the classic setup, please migrate to the simplified architecture as soon as possible to avoid any disruption to your batch release processing. For more information, see SAP BTP Account and Integration Hub Tenant.

 Testing should cover the integration between SAP Batch Release Hub for Life Sciences and Integration Hub for Life Sciences. N n/a
Medium 213783/2026 Issue with display of additional batch characteristics in BRH against a CFG batch in My PCN Release Decisions In the My PCN Release Decisions app, the Value column was missing from the Additional Batch Characteristics table, which is displayed on the details page for an assigned batch when additional batch characteristics are available. This issue is now fixed. The Value column configuration has been added to the Additional Batch Characteristics table in the My PCN Release Decisions app. This UI configuration change reuses existing components already used by the My Release Decisions app. No backend changes were required. Testing should cover the integration of additional batch characteristics in the My PCN Release Decisions app. N n/a
Medium 211544/2026 Inspection lot logs not available in monitor data processing tab 

Data sent through SAP Event Mesh didn't fail when activation validation issues occurred. The data was activated despite the validation errors, and no monitoring logs were displayed for the errors. To resolve this issue a manual transaction wrapper has been implemented to wrap staging and activation in a single transaction. This ensures there is an automatic rollback when failures occur.

 This issue is relevant for customers using SAP Event Mesh. Testing should cover the integration between SAP Event Mesh and SAP Batch Release Hub for Life Sciences. The issue specifically occurs when there is a data validation issue (for example, a mismatch between the release quantity and the posted quantity). N n/a
Medium 188800/2026 Release Decision Report - Custom fields not displayed Custom fields were not displayed in release decision reports. This issue has been fixed by adding the missing custom (extension) fields in release check details, including component and batch record review scenarios, by improving the merge and formatting support for batch record review attributes. Testing should cover the creation of release decision reports. N n/a
Medium 220043/2026 Issue with "My Release Decisions" app In the My Release Decisions app, the Release Type filter field did not display all existing release types. This issue has been fixed by changing the UI element from a dropdown to an input field with value help. This change ensures that the Release Type filter displays all the filter values, even when there are a high number, and it also improves the user experience. Testing should cover filtering of release types in the My Release Decisions app. N n/a
High 303372/2026 Issue with records not loading to BRH but sent and logged Batch component records did not appear in the My Release Decisions app although they were sent to SAP Batch Release Hub for Life Sciences and logged. This issue occurred because the component relations cache cleanup was not triggered when component relations were sent nested inside ComponentsActive and activation happened in a separate $batch atomicity group (via /activate). The fix uses composition-level component activations for the cache cleanup, so that newly added components are visible. Testing should cover sending batch component data to SAP Batch Release Hub for Life Sciences, and ensuring the components appear as expected in the My Release Decisions app. N n/a
High 220107/2026 PCN Technical Release for WL Item created with trigger type as GR  A PCN-related release decision trigger for an existing assigned batch with a different release decision trigger type created another batch release decision item. In the My PCN Release Decisions app, this resulted in a PCN release decision item with two entries for the same batch in the Assigned Batches table, each with a different release decision trigger type. Two entries for the same assigned batch are not supported and cause several UI consistency issues. To prevent this issue occurring again, the code has been adjusted to reject any secondary PCN-related release decision trigger for an assigned batch that already exists in the PCN item.  Testing should cover the creation of PCN release decision items and their assigned batches. Medium 245771/2026 PCN Worklist: Batch Summary does not have radio buttons for selection

In the My PCN Release Decisions app, two entries for the same assigned batch were created. This scenario is not supported and caused several UI consistency issues. The code has been adjusted to prevent this issue occurring again. However, if a release decision trigger with the incorrect type is sent initially, the PCN release decision item cannot be used further. A new end-user procedure has been enabled to resolve this scenario: you must cancel all items created by the PCN release process flow for your trigger. Then, send the PCN release decision trigger again to create a new PCN release decision item, and send the correct release decision triggers for the assigned batches.

Note this procedure must be performed immediately after the creation of the PCN release decision items. It does not work if one of the items created by the PCN release process flow is not canceled. For example, this procedure does not work if an item related to the PCN release decision trigger is already completed/confirmed or was previously regenerated.

 Testing should cover the creation of PCN release decision items and their assigned batches. Y

User Guide:

PCN Release Decisions

Updated to explain how to proceed when a release decision trigger for an assigned batch was sent with the incorrect type.

Symptom

  • While adding an SAP S/4HANA Cloud Private Edition System to a formation of type "integration with SAP business data cloud", synchronization fails.
  • Popup shows: Errors occurred while synchronizing. Choose Resynchronize to restart the synchronization process.”
  • Technical message: [BDC-FOS]: genericInputValidation … CSR validation failed; Provided subject: CN=staging, L=123456789, OU=..., OU=SAP Cloud Platform Clients,O=SAP SE,C=DE is invalid; Please correct the following attributes: {L=010001001};”

Environment

  • SAP Business Data Cloud
  • SAP S/4HANA Cloud Private Edition System as the source of the formation

Reproducing the Issue

  1. Run transaction STRUST to create a PSE and generate a CSR in the S/4HANA Cloud Private Edition System.
  2. Add the S/4HANA System to a formation and start synchronization.
  3. Observe the synchronization error indicating CSR validation failed and that L must be corrected.

Cause

The DN (Distinguished Name) string does not follow the required pattern because the L attribute does not match the required S/4HANA Cloud Private Edition System identifier (tenant id).

Resolution

Follow the recommendation in the last sentence of the error message:

Please correct the following attributes: {L=010001001}; " 

Correct the L part (tenant id) in CSR string to the recommended one, e.g: 010001001 (no '$')

The correct L part also can be known in me.sap.com:  Customer Landscape  >>Systems  >>Choose the S/4 system  >>System Details  >>System ID. Detail is as below:

This rule is documented in online below:

https://help.sap.com/docs/business-data-cloud/administering-sap-business-data-cloud/create-pse-and-generate-certificate

The DN string must adhere to the following pattern:

CN=staging, L=<tenantId>, OU=XXXXXXXX-XXXX-XXXX-XXXX-XXXXXXXXXXXXX, OU=SAP Cloud Platform Clients, O=SAP SE, C=DE

<tenantId>

The SAP S/4HANA Cloud Private Edition system ID that is assigned to the tenant by UCL (Unified Customer Landscape).

In certain cases, SAP S/4HANA Cloud Private Edition system ID (<tenantId>) might have a “$” in the beginning, for example, "$0204398045”. You must remove the “$” from the beginning and use only the remaining part.

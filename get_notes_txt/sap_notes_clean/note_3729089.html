Symptom

DT_E_S field has values different than the expected, specially on scenarios involving COD_SIT "01" and/or "07"

Environment

  • SAP Tax Declaration Framework Brazil
  • SAP S/4 HANA

Reproducing the Issue

Generate Brazil EFD ICMS/IPI and validate register C100 result for DT_E_S field.

Cause

Expected behavior according the standard logic considering Extemporaneous scenarios.

Resolution

The standard logic will determine DT_E_S field considering data originated from "J_1BNFDOC" or shadow "/TMF/D_NF_DOC".

Depending on the scenario it will consider "DT_E_S" or "DT_DOC" or even "DT_EXTEMPORANEA", according below logic:

jf( in("COD_SIT",'02','03','04','05'),
    '',
    jf( in("COD_SIT",'01','07') and not( isnull("DT_EXTEMPORANEA") or  "DT_EXTEMPORANEA" = '' or "DT_EXTEMPORANEA" = '00000000'), 
        "DT_EXTEMPORANEA",  
        jf( "DT_EXTEMPORANEA" > '$$P_DT_FIN$$',
            '',
            jf( isnull("DT_E_S") or ("DT_E_S" = '00000000'), 
                "DT_DOC", 
                jf( "DT_E_S"<"DT_DOC" and not( isnull("DT_EXTEMPORANEA") or "DT_EXTEMPORANEA" = '' or "DT_EXTEMPORANEA" = '00000000') ,
                    "DT_EXTEMPORANEA",
                    "DT_E_S"
                )
            )
        )
    )
)

In resume:
if COD_SIT in {02,03,04,05}:
return ' '
else if COD_SIT in {01,07} and DT_EXTEMPORANEA present:
return DT_EXTEMPORANEA
else if DT_EXTEMPORANEA > P_DT_FIN:
return ' '
else:
if DT_E_S missing:
return DT_DOC
else if DT_E_S < DT_DOC and DT_EXTEMPORANEA present:
return DT_EXTEMPORANEA
else:
return DT_E_S

For COD_SIT "01" (Escrituração Extemporanea de Documento Regular) and "07" (Escrituração Extemporanea de Documento Complementar), the mainly value is fetched from "DT_EXTEMPORANEA" which comes from field "DSAIENT" from "J_1BNFDOC" and if DT_E_S result needs to be adjusted, you may use shadow "/TMF/D_NF_DOC", field DT_EXTEMPORANEA directly (or the other fields depending on your scenario).

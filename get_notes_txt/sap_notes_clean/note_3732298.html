Symptom

ShaExecuteOperation fails when a parameter value contains the delimiter character :.

When passing a URL as a parameter, for example:

  • Input parameter: URL = "http://hostname:port"

the value is incorrectly truncated at the : character. As a result, only the part before the first : is processed.

  • Expected: URL = "http://hostname:port"
  • Actual: URL = "http" (the remaining part //hostname:port is ignored because the value is truncated at the delimiter : after http)

This behavior can be observed in the agent log similar to:
Log: com.sap.solman.agent.mai.os.ShaExecuteOperation [executeConfiguredOperation()] Start with CUSTOM_OPERATION_NAME=<Operation_Name>, PARAMETERS={URL=http}, RETURNFORMAT=JSON

Reason and Prerequisites

ShaExecuteOperation parses parameter key–value pairs by splitting on the delimiter character :; if the parameter value itself contains :, the collector incorrectly stops reading the value at the first occurrence of :, so only the substring before the first : is treated as the value and the remaining part is ignored (for example, for URL=http://hostname:port, only http is interpreted as the value while //hostname:port is truncated)

Solution

Upgrade the SDA to at least the versions listed in “Support Packages & Patches”.

In the corrected implementation, the parameter string is parsed so that the first occurrence of : separates key and value. The part before the first : is interpreted as the key, and everything after it is interpreted as the value.

For example, for the input "URL":"http://hostname:port", the key is URL and the value is http://hostname:port.

Additionally, a new option is available to use custom delimiter if required. You can override the default delimiter by appending ,delimiter:<delimiter-char> at the end of the parameter definition. For example:

"URL"="http://hostname:port",delimiter:=

uses = as the delimiter, and the parameter is resolved as Key = URL and Value = http://hostname:port.

Symptom

This notes covers the following issues in the reporting app:

- When creating a report and try to specify a filter, the drop down list only contains up to 100 fields even though there should more fields

- Deletion of mandatory filter criteria is possible

- Switching the filter operators doesn't refresh the entered value in all scenarios

- Missing value help for reconciliation track, reconciliation track variant and reconciliation run ID in various reporting patterns

Other Terms

FCCOREUI ; Reporting app ; refresh ; values ; operator ; filtered field ; low ; high ; filter type ; value ; search ; help

Reason and Prerequisites

Code error

Please note that there is also a BE related note # 3702452, that needs to be implemented as well.

Solution

Implement the note and all of its prerequisites. Please also note that the dependent search help (e.g. display only those run IDs in value help that belong to the recon track that was also specified in the filter) only when editing the filter for already generated reports. When creating a new report, the dependent search help is not possible due to technical restrictions.

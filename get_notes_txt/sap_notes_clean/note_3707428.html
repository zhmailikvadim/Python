Symptom

The following error occurred in the IPS (Identity Provisioning) job log from source system to Ariba target system.
 
error=Could not connect to target host https://<ariba API host>/api/mds/v1/prod Caused by: Unexpected character ('<' (code 60)): expected a valid value (JSON String, Number, Array, Object or token 'null', 'true' or 'false')
 at [Source: REDACTED (`StreamReadFeature.INCLUDE_SOURCE_IN_LOCATION` disabled); line: 1, column: 1],

Environment

Identity Provisioning

Reproducing the Issue

Run the IPS job.

Cause

The property "URL" of the Ariba target system is not configured correctly.

Resolution

As per the document SAP Ariba Applications | SAP Help Portal, the property URL should contain only the host name without the URL path.
For example:

SAP AribaIdentity ProvisioningValues

SCIM API URL

URL

Examples:

  • US: https://openapi.ariba.com

  • Europe: https://eu.openapi.ariba.com

  • UAE: https://mn1.openapi.ariba.com

See Also

SAP Ariba Applications | SAP Help Portal

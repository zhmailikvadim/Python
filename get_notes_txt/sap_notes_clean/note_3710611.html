Symptom

  • Reconciliation keys are transferred in background mode.
  • The spool request indicates that the transfer was successfully executed.
  • Upon validation, the keys are not actually transferred.

Environment

  • Release: S/4HANA 2020 and higher resp. S4CORE 105 and higher
  • FI-CA - Contract Accounts Receivable and Payable

Reproducing the Issue

  1. Start transaction SE38.
  2. Insert report RFKKGL00 and click on 'Execute (F8)' button.
  3. Select an existing variant.
  4. Schedule the run in background via menu: Program => Execute in Background or press F9.
  5. Insert the output device and click on 'Continue' button.
  6. Start the run via the 'Immediate' button and save it.
  7. Start transaction SM37.
  8. Select the relevant job like RFKKGL00.
  9. Click on 'Execute (F8)' button.
  10. Mark the relevant line and click on the 'Spool' button.
  11. Verify that the example key is successfully transferred.
  12. Start transaction FPF3.
  13. Insert the example reconciliation key and press ENTER.
  14. Observe that the status is still 'Transfer to G/L has not yet taken place'.

Cause

  • Improper or outdated variant for report RFKKGL00.
  • Since release S4CORE 105, the Test Run flag has been replaced by option field 'Test Run/Update Run'.

Resolution

  1. Adjust the existing variants for report RFKKGL00.
  2. Ensure that for real transfer runs, the processing option 'Test Run/Update Run' is set to 2 for Real Run.

Symptom

Error message C+606 (Number assignment for ML document is incorrect) appears when performing settlement (program RKO7CO88) in "Parallel Processing" mode.

Environment

  • Controlling (CO)
  • Financial Accounting (FI)
  • SAP R/3
  • SAP R/3 Enterprise 
  • SAP ERP Central Component
  • SAP ERP
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA Finance
  • SAP S/4HANA
  • SAP Fiori for SAP S/4HANA Finance
  • SAP Fiori

Reproducing the Issue

1. Execute program RKO7CO88 with "Parallel Processing" flag active.
2. Error C+606 (Number assignment for ML document is incorrect) is returned.

Cause

Number range buffering issue for ML Documents when running in parallel mode.

Resolution

1. Maintain configurations for number range objects ML-BELEG and CK_MLVNR. For more details, refer to KBA 2746689 - C+606 Number assignment for ML document is incorrect.

2. In transaction SNRO, tab Customizing, maintain Buffering of number range objects ML-BELEG and CK_MLVNR to "Parallel Buffering".

3. Regarding the change to "Parallel Buffering", please refer to the information in the following notes,

  • 1398444 - Buffering the document number assignment for RF_BELEG
  • 504875 - Buffering of number ranges
  • 599157 - Number ranges: New buffering method
  • 840901 - Parallel buffering and pseudo-ascending number assignment
  • 2371023 - Number Range for ML-Document in S/4 Finance On-Premise Edition

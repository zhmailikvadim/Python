Symptom

  • UI adaptations were performed in the app “Manage Sales Orders - Version 2,” creating a custom view for the Items section to add columns in a specific sequence.
  • After transport UI Flexibility: Views and Changes from customizing tenant to test system via a software collection, the changes are not reflected in test system. 

Environment

SAP S/4HANA Cloud Public Edition

Reproducing the Issue

  1. Open the app Manage Sales Orders - Version 2 and start Adapt UI.
  2. Create a custom view for the Items table, add columns, and change the column order.
  3. Attempt to transfer the change via the Export Software Collection.
  4. Import into the test system and verify the view; the view is missing or present without the configured column layout.

Cause

UI Flexibility: App Variants were not transported together with UI Flexibility: Views and Changes.

Resolution

  1. Ensure the view is created and adjusted in Adapt UI (not only via personalization), so the change is recorded as UI Flexibility: Views and Changes.
  2. In the extensibility tooling, verify that both artifacts exist for the app: “UI Flexibility: App Variants” and “UI Flexibility: Views and Changes.”
  3. Add both artifacts to the same software collection (for example, a hotfix), then export the collection version.
  4. Import the exported software collection version into the target (test) system.
  5. Open the app in the target system and confirm the custom view appears with the expected column configuration.
  6. If prior transports missed the App Variant, create a new hotfix that includes both the App Variant and the Views and Changes, then re-import into the target system.

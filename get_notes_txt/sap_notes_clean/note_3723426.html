Symptom

 When a change of settlement and a move-out occur on the same day, the settlement unit cannot be generated to the point of delivery (PoD) correctly. The system is expected to generate a one-day time slice for this special changed date because the new settlement unit starts at the beginning of the day and the settlement ends at the end of the same day.

This issue is relevant for all market roles.

Other Terms

 Settlement, EEDM11, /US4G/CL_IM_EH_EEDMSETTLUI, point of delivery, PoD, time slice, change of settlement, move-out, settlement unit

Reason and Prerequisites

 The error occurred because in method /US4G/CL_IM_EH_EEDMSETTLUI-ADJUST_SETTLUNIT_ASGMT_RECORDS line 55, if the changed date of the master data change (MDC) was equal to or less than the settlement end date, the system removed this record.

Solution

Implement the corrections of this SAP Note or import the relevant support package.

After implementing this SAP Note, the system generates the expected one-day time slice for this special changed date correctly. You can verify the fix by checking that the one-day time slice for the special changed date displays correctly when a change of settlement and a move-out occur on the same day.

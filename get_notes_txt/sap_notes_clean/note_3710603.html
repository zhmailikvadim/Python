Symptom

The "Display Dialog" pushbutton on the screen "Management of Dialog Requests/Dialog Messages" in the B2A Manager (transaction PB2A) does not work.

Details:
After you send an ELM 5 wage notification in the B2A Manager (for example, a statistics notification) and a successful status query, the final recipient can initiate a dialog. In this case, the system displays an entry "Management of Dialog Requests/Dialog Messages" in the data for the process step. You can use this entry to navigate to the "Management of Dialog Requests/Dialog Messages" screen and download the dialog messages with the "Call Dialog" pushbutton and then choose "Display Dialog" to display them or choose "Answer Dialog". In contrast to the other two pushbuttons, the pushbutton for displaying does not work.

Other Terms

Payroll Switzerland, uniform wage notification procedure (ELM), class 'CL_HRPAYCH_ELM50_DIALOG_MSG', B2A constant '.DIAR'

Reason and Prerequisites

This SAP Note is relevant if you use ELM 5 and receive a dialog message from the final recipient.
This SAP Note is not relevant for ELM 4.

Solution

Import the HR Support Package specified for your release or implement the relevant correction instructions.

Workaround (instead of implementing the correction instructions):

In view 'V_T50BK' for (area ELM, document category 0005), create a B2A constant '.DIAR' with constant value 'CL_HRPAYCH_ELM50_DIALOG_MSG'. After you implement the corrections or implement the corrections, you can delete the constant again.

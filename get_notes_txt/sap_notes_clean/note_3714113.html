Symptom

You are creating S4 Hana Service orders with items relevant for procurement. Also you have maintained your custom specific account assigmnet category under SAP Customizing Implementation Guide -> Service -> Transactions -> Settings for Service Transactions -> Integration -> Logistics Integration -> Enterprise Organizational Management -> Define Logistics Scenario for Procurement of Materials and Services. Although of maintaining this customizing, you recognize that hardcoded account assignment category is passed when creating the Purchase Requisition or Purchase order.

In the following scenarios the account assignment category is hardcoded and not taken over from customizing.

- Creation of items belonging to the Advance Shipment scenario ( for both controlling scenario new item-based accounting / assignment via account assignment object e.g. internal oder ). In this case the account assignment category is always set to 'H'.

- Creation of standard procurement items with assignment via account assignment object, the account assignment category is set depending on the corresponding object.

Now with this note the priority will be given to the maintained account assignment category in customizing before setting a default value.

Remark: When using your custom specific account assignment category in the Advance Shipment scenario, you have to make sure that you are using an account assignment category which allows assigning a sales order item. Also when using any other scenario with account assignment manager it has to be ensured, that the account assignment category is allowed for the corresponding accounting object ( e.g. for internal order standard account assignment category = 'F' ).

Other Terms

PR, PO, GR, Integration

Reason and Prerequisites

Hardcode instead of using existing customizing capabilities

Solution

Implement attached correction instruction.

Symptom

When attempting to archive an Agreement that has auto participation, the system does not allow the archiving process to proceed and displays an error stating “Participation exists.” To continue with archiving, deployment codes to be archived. However, these deployment codes are required for future business processes, which causes the archiving process to be blocked.


This SAP Note provides the necessary corrections to handle participant table entries during the calculation run archiving, allowing the Agreement to be archived without requiring the deployment codes to be archived.

Other Terms

Archiving, Participants, Deployment Code, Calculation Run Archiving

Reason and Prerequisites

This is cuased by program error.

Solution

Please implement the correction instructions attached. These corrections are only valid if you have applied the Support Packs.

Release 60E Support Pack 4.

1. Perform the manual pre-implementation steps that are described below.
2. Use transaction SNOTE to implement the correction instructions specified below and activate the affected program object.

Symptom

In Story Reports users are seeing a different required date compared to what they are seeing in Learning for that same user

Environment

SAP SuccessFactors Learning

Reproducing the Issue

  1. Go to Learning Administration
  2. Go to assigned items
  3. See any item with a required date 
  4. Go to Story Reports
  5. See there is a different date and time showing for the required date field for the same user and record

Cause

The Required Date field is converted to UTC in story reports

For example when viewing a user's assigned items in Learning Administration whose timezone preference is in Eastern Standard Time timezone the required date shown here is in Eastern Standard Time but when going to story reports to view this same record you will see the required date now shows a day later as it was converted to UTC time from Eastern Standard Time.

Resolution

This behavior is as per current system design so if you would like to see a difference in this behavior then please submit an ehancement request as per the KBA below:

  • 2090228 - How to submit enhancement ideas for SAP SuccessFactors Products - SAP for Me

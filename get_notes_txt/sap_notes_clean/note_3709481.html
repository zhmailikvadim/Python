Symptom

An issue was identified during the reload of initial data: the activity rule GROUP_DUPLICATED_SFCS is incorrectly created as a Custom Rule for activities SY510 and SY511. It should instead be recognized and generated as a Predefined Rule with its default value set to FALSE.

Other Terms

SAP ME, SY510, SY511

Reason and Prerequisites

Steps to Reproduce

1. Reload initial data in Site Maintenance.

2. Observe the creation of the activity rule GROUP_DUPLICATED_SFCS for SY510 and SY511.

Actual Result: The rule GROUP_DUPLICATED_SFCS is created as a Custom Rule.

Expected Result: The rule should be recognized and created as a Predefined Rule for SY510 and SY511, with its default value remaining FALSE.

 This SAP Note is the follow up of the SAP Note 3480627.

Solution

A code correction has been implemented. After the fix, the activity rule GROUP_DUPLICATED_SFCS will be properly displayed and handled as a Predefined Rule for SY510 and SY511. All activities derived from SY510 or SY511 should be re-created after the site initial reload action.

Impact

The implemented code change affects only the initial data reload functionality and does not influence other system areas.

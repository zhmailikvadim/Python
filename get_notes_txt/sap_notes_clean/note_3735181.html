Symptom

  • Validation issue with characteristic allowed value: the inbound payload provides the characteristic value description instead of the technical value, but the change request is created and can be activated without error.
  • No validation error occurs during inbound SOAP processing or when the data is passed to the change request.
  • In some cases, error message C1 834 appears: "In characteristic &1, the value &2 is not available in class &3."
  • The description value may be written to the classification table instead of being rejected.

Environment

  • SAP S/4HANA
  • SAP Master Data Governance

Reproducing the Issue

  1. Send an inbound SOAP message that includes a classification characteristic where the payload contains the characteristic value description instead of the technical value from the allowed value list.
  2. Monitor the inbound processing and observe that no error is raised and a change request is created.
  3. Approve/activate the change request and observe that no validation error is issued and the description value is persisted.

Cause

  • Lean Classification validation did not reject characteristic value descriptions in the staging area, allowing them to pass into the change request and activation.
  • Inconsistent behavior between payloads can also occur if the change request type used for inbound client maintenance is configured to run validations in step 00 (which is not permitted for inbound processing).

Resolution

  1. Implement SAP Note 3654245 to correct Lean Classification validation so that characteristic value descriptions are rejected in the staging area and change requests with value descriptions as technical values cannot be activated. Refer to: 3654245 - Lean Classification: Validation for characteristic values do not fail for value descriptions
  2. Verify and correct the change request type used for inbound client maintenance so that no validations are executed in step 00:
  • Navigate to MDGIMG -> General Settings -> Process Modeling -> Change Requests -> Configure Properties of Change Request Type.
  • In Enhancements and Checks per Change Request Type, ensure only Basic Check is relevant.
  • In Entity Types per Change Request Type, set Field Properties to No Required Field Check for all entity types.

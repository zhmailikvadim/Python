Symptom

When trying to insert a data source while the setting RecentListSize (Maximum Number of Entries in Recent List) is set to 0, following exception will occur:
"[78a1-2f54-e5] Index was out of range. Must be non-negative and less than the size of the collection. Parameter name: index"

Temporary Workaround:
For a temporary workaround please increase the the value of the setting RecentListSize as described in SAP KBA 3092666.

Other Terms

AO; Analysis Office; Recent List; Recent Data Sources; DataSources; 78a1-2f54-e5

Reason and Prerequisites

Program error

Solution

After installing the correction, the exception won't occur anymore when inserting data sources while the setting RecentListSize is set to 0.

The correction will be delivered by a SAP Analysis for Microsoft Office Support Package.
In section "SP Patch Level" you can find information which Support Package contains the respective correction.

If the section "SP Patch Level" is not available or the information about which Support Package contains the correction is missing, this means that this has not been determined, yet. In this case please re-check this note at a later point of time.

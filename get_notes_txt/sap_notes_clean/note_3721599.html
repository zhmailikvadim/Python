Symptom

  • When generating a list report that is using "Not Equal" operation, the results are returning all values.

Environment

PowerDesigner

Reproducing the Issue

  1. In a new CDM, create and edit an extension.

  2. Create a string ext attribute for the Entity metaclass.

  3. Add the model entity E1 and attribute ATT1.

  4. Set the ext attribute value to 'test' for E1.

  5. Create a second Entity E2 and attribute ATT2.

  6. Create a list report based on extended attribute.

  7. Select some properties (Name,Parent.ExtendedAttribute) when defining column filter.

  8. Specify
    Parent.ExtendedAttribute = test

  9. Click Next
    Only ATT1 is returned.

  10. Go back and modify the condition:
    Parent.ExtendedAttribute Not Equal test
    It returns ATT1 and ATT2. We should have only ATT2.

    Note: No problem with Parent.Comment.

Cause

  • The issue has been identified and logged under CR 831828. Its description reads, "List Report filters not working when using "Not Equal" condition".

Resolution

  • There is currently no target SAP PD 16.7 Service Pack scheduled for this issue SAP CR 831828. It is being assigned to our Product Group.

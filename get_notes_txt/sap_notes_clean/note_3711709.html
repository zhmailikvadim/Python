Symptom

1. The KIM access key is not taken into account for KIM Web service communication with amasys.

2. Prescription requests cannot be read and processed from the amasys mailbox with the service ID "eReception_ParenteralePreparation;Prescription Request".

Other Terms

N3KVMST01 V7, N3KVUNIRP V3

Reason and Prerequisites

Function enhancement & error correction

Solution

1. The KIM access key is taken into account for KIM Web service communication with amasys.
As of amasys 3.31, the KIM access key is copied to the clipboard using a separate function button in amasys and must be inserted and saved in the "KIM Access Key" field in Customizing under "KIM: Set Up Shipping".

Note: The use of the new KIM access key is optional as of amasys 3.31, but mandatory as of amasys 3.33 (SP47), since only the KIM access key for the call of the KIM WS (CO_N2AZ_WS_KIMWSSERVICE) is supported as of this time.

KIM message dispatch with the new KIM access key for DALE-UV, eEB, and the electronic Application for connection rehabilitation is enabled with another SAP Note/SP.

2. The system processes the service IDs "ePrescription_ParenteralePreparation;Prescription Request" and "ePrescription;ParenteralPreparation;1.0" as expected for prescription requests.

Import the specified Support Packages to solve the problem. You can implement the collective correction in advance.

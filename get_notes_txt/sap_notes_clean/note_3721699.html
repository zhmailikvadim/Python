Symptom

  • Editable Data Table (EDT) is used to dynamically display Notification Tasks
  • If no tasks are available, the application shows a loading bar with the message "Waiting for metadata" instead of displaying the EDT header or an empty table. 

Environment

SAM Extension: 2505.0.2

Cause

Product enhancement

Resolution

This behavior may confuse users and negatively affect user adoption, as it appears that the application is still loading data.

To resolve the issue, implement the fix provided in the attached file.

  1. Download and unzip the attached file.
  2. Locate the following directory in your project: SAM/build_workspace/SvcAssetMgr/plugins/extension-EditableDataTable_Plugin-EditableDataTable/platforms/android
  3. Replace the existing .aar file with the .aar file provided in the attachment.
  4. Rebuild the Android application using the command: tns run android
  5. After rebuilding, test the application again to verify that the progress bar no longer appears when there are no notification tasks available.

Note: For SAM Extension 2511.2.0, the fix will be provided as part of the pagination feature. 

Keywords

Editable Data Table, EDT, Waiting for metadata, loading bar, notification task, empty table, SSAM, Android plugin

Symptom

My Robotic Process Automation (RPA) bot, which integrates with SAP Ariba by scraping data directly from the user interface (UI), is failing. The bot often breaks after system updates because HTML elements or field names have changed.

Environment

  • SAP Ariba Procurement
  • SAP Ariba Sourcing

Cause

Using an RPA bot to scrape data from the user interface is not a supported method for integration. The HTML elements in the UI are subject to change during regular system updates, which will cause bots that rely on screen scraping to fail.

Furthermore, SAP Ariba’s contractual terms require that all non-human users (bots) must be registered in the SAP Ariba Developer Portal to ensure safe and compliant usage.

Resolution

To ensure a stable and compliant integration, you must take the following actions:

  1. Migrate your integration method from UI scraping to using the supported APIs available through the SAP Ariba Developer Portal.
  2. Register your bot in the SAP Ariba Developer Portal.
  3. Contact your SAP Account Manager to discuss proper bot implementation strategies.

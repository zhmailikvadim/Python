Symptom

The Business Add-In BAdI: Cost Determination Status (EAM_COST_DETERMINATION_STATUS) allows you to disable the automatic determination of planned and overhead costs while saving maintenance orders. This can help to avoid performance issues while saving large orders with many operations or components, or projects that contain a large number of maintenance orders linked as subnetworks. You can then determine these costs separately, for example in a periodic background job.

Depending on the release level of your system, you experience the following issues:

  • The Business Add-In BAdI: Cost Determination Status (EAM_COST_DETERMINATION_STATUS) is not included in the SAP Implementation Guide.
  • The BAdI documentation does not describe how you can disable the cost calculation for maintenance orders during project-related processing using the Project Planning Board or the Project Builder.

Other Terms

ACDOCP, Costing, Performance, EAM, IMG, SPRO

Reason and Prerequisites

Documentation error

Solution

Use the BAdI Builder transaction (SE18) to display the Business Add-In BAdI: Cost Determination Status (EAM_COST_DETERMINATION_STATUS). You can find the updated documentation in the attachment to this note.

The SAP Implementation Guide and the BAdI documentation are updated with the delivery support packages of this note. If you import the relevant support package, you can access the Business Add-In in Customizing for Plant Maintenance and Customer Service under System Enhancements and Data Transfer -> Business Add-Ins -> BAdI: Cost Determination Status.

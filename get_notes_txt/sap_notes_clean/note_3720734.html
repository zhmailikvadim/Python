Symptom

A mandate initiation request fails with reject reason code MD12 and the message "Mandate reference not unique, mandate could not be created, reason MD12" (Message 051, Message Class /PE1/MSG_RA_MNDT). This occurs in scenarios where both the creation and reinstatement of the mandate are not possible

Other Terms

DDMC, DDI, Direct Debit Mandate, Create Mandate, MD12, MD22, /PE1/MSG_RA_MNDT

Reason and Prerequisites

The issue occurs when a mandate cannot be created or reinstated because the mandate was already cancelled more than 2 months ago. In this specific case, an irrelevant (misleading) error message and an incorrect rejection reason code are provided.

Solution

A new message has been introduced to be used during the mandate creation process when reinstatement is not possible (Message 059, Message Class /PE1/MSG_RA_MNDT).
Additionally, the correct Rejection Reason Code MD22 ("MandateInitiationCancelled") is now provided for this case, instead of the incorrect MD12 ("NotUniqueMandateReference").

Please implement provided correction instructions or corresponding Support Package.

Symptom

  •  Following error message appears when trying to create a new crystal report based on a BEx Query in Crystal Reports 2025:

    Failed to parse data source: Cannot read properties of undefined (reading 'push')
  • Issue only happens when Bex Query is defined with keyfigures that are organized as tree.

Other Terms

CrystalReports, SAP BW (DSL), Bex Query, nested keyfigure

Reason and Prerequisites

Environment

  • SAP Crystal Reports 2025 Patch 8

Steps to Reproduce

  1. Open Crystal Reports 2025 designer.
  2. Choose to create a 'Standard Report'.
  3. Select 'SAP BW (DSL)' option.
  4. Choose a BEx Query with nested keyfigures
  5. Fill out the mandatory parameters if necessary.
  6. Observe error message appears.

Cause

  • Crystal report 2025 uses old API to display nested keyfigure structure as a flat list.

Solution

This issue is fixed in the patches listed in the "Support Packages & Patches" section below.

Due to different internal processes, the "Support Packages & Patches" section of the SAP Note will be populated as below:

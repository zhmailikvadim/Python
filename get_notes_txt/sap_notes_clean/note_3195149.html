Symptom

When web applications calls for *.xsjs call and experience timeout, performance error, or disconnects, we need to enable verbose tracing on xsengine for troubleshooting. 

URL may look like :  https://[hostname]/.../[application name].xsjs/   which indicate it is a web call via the XS classic engine (xsengine). 

Environment

  • HANA 1.0, all revisions.
  • HANA 2.0, all revisions.
  • HANA Cloud editions. 

Resolution

Extract the Database user and Application user from the existing traces,  searching for [application name].xsjs which will reveal details like below : 

[0000]{00000}[-1/-1] yyyy-mm-dd hh:mm:ss i TraceContext     TraceContext.cpp(01269) : UserName=DB_USER, ApplicationUserName=APP_USER, ApplicationName=xs, ApplicationSource=/.../[application name].xsjs..., ClientIp=, StatementHash=, StatementExecutionID=

Then collect the debug trace for this application.
  1. enable user-specific trace with context 'OSS<case_num>'. Please replace the DB user and APP user you used to send the http requests. You'll get the user names in above step.

    ALTER SYSTEM ALTER CONFIGURATION ('global.ini', 'SYSTEM' ) SET
    ('traceprofile_OSS<case_num>', 'sql_user') = 'DB_USER',
    ('traceprofile_OSS<case_num>', 'application_user') = 'APP_USER',
    ('traceprofile_OSS<case_num>', 'maxfilesize') = '200000000',
    ('traceprofile_OSS<case_num>','maxfiles') = '50',
    ('traceprofile_OSS<case_num>', 'XSHttpNetIO') = 'debug' ,
    ('traceprofile_OSS<case_num>', 'XSRequestHandler') = 'debug' ,
    ('traceprofile_OSS<case_num>', 'IpConnection') = 'debug',
    ('traceprofile_OSS<case_num>', 'HttpClient') = 'debug' ,
    ('traceprofile_OSS<case_num>', 'HttpClientAuth') = 'debug' ,
    ('traceprofile_OSS<case_num>', 'xshttpdestination') = 'debug' ,
    ('traceprofile_OSS<case_num>', 'xshttpdestinations') = 'debug'
    WITH RECONFIGURE;

  2. Wait or reproduce the timeout error reoccurrence, and note down the timestamps.
  3. Disable the trace

    ALTER SYSTEM ALTER CONFIGURATION ('global.ini', 'SYSTEM') UNSET ('traceprofile_OSS<case_num>') WITH RECONFIGURE;

After these, you may find additional xsengine traces with name like xsengine_<hostname>.30007_OSS<case_num>.<part number>.trc
Then please attach these traces to the case.

Please note, the trace could increase very fast, please monitor the disk space of the trace directory and avoid disk full situation. And note to disable the trace immediately after issue captured. Otherwise the captured information would be flushed away by increased traces.

Keywords

xsjs, timed out, timeout, HTTP Request failed

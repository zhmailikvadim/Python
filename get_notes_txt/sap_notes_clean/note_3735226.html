Symptom

  • Replication flows are stuck in SAP Datasphere with the message "The source connector has accepted the initial load partitioning request. The next attempt will be after"
  • In the backend, transaction DHCDCMON shows jobs in a not running (red) status.

Environment

SAP Datasphere

Reproducing the Issue

  1. Open the Replication Flow Monitor for the affected replication flow
  2. Start or restart the run
  3. Observe that the run shows the message: "The source connector has accepted the initial load partitioning request. The next attempt will be after ..."
  4. Observe that the data is not being loaded in SAP Datasphere
  5. Go to transaction DHCDCMON in the backend system
  6. Observe that the DHCDCMON transaction shows jobs in a not running (red) status

Cause

Resolution

Implement the corrections in the ABAP-based source system as described in SAP Note 3667882 -  ABAP Data Integration - CDC jobs with an event-based start condition are displayed in DHCDCMON as not running

See Also

SAP Note 3667882 -  ABAP Data Integration - CDC jobs with an event-based start condition are displayed in DHCDCMON as not running

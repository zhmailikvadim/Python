Symptom

When executing BW Query, data is read from Database by packages. Default package size is 20000. That means 20000 records are returned from Database to BW by each package.
When the data column from Database is many times of 20000, there are many packages. It can happen that each package reading takes relatively long, which results in long query execution time.
In such situations, increasing Package Size may shorten the Query execution time.

Environment

  • BW7.5
  • BW4HANA
  • S4HANA

Resolution

Package Size can be changed as following:

  • For BW 7.5 or BW 7.5x or S4HANA: Maintain RSADMIN parameter 'RSR_FETCH_PACKAGESIZE' with a value between 20000 and 500000 using the report 'SAP_RSADMIN_MAINTAIN'. (When maintaining a value out of this range, the setting will be ignored and the default 20000 will be used.)
  • For BW4HANA: Use transaction RSCUSTV35 to set a value for "Package Size" and save. (There is no hard limitation between 20000 and 500000 but the same is recommended.) 

Note: The above two settings are global parameters that affect all BW Queries. A large Package Size increases ABAP memory consumption and may result in ABAP memory shortage.

Implement below SAP note to solve a known program error that Package Size setting does not take effect.
3649747 - Adapting Package size in parallel processing.

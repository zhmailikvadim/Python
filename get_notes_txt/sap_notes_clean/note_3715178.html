Symptom

When the location bundle code (external code of the MCM instance) changed, the system did not create an MDX change entry,thus no bundle change process triggred and no market message sent out.

This issue is relevant for the market role: DSO.

Other Terms

 /UCOM/MDX_PROC_OUT, /UCOM/RP_MDX_PROCESS_OUT, /UCOM/VC_MDX_RST, LBS, LBSHDMA0100, DE_B_MDC_MAIN_SEND_DSO_RESP_LBS, MarketLocationBundleChange, 55175

Reason and Prerequisites

Missing Functionality

Solution

Implement the corrections of this SAP Note or import the relevant support package.

After implementing this SAP Note, the system creates an MDX change entry when the location bundle code (external code of the MCM instance) changes. The BC set /US4G/3_0_COMM_SP08_NOTE_3715178 contains the necessary corrections. You can verify the fix by changing the bundle code (external code of the MCM instance) and checking that an MDX change entry of event 'LBS' and change type 'LBSHDMA0100' appears in transaction /UCOM/MDX_PROC_OUT.

Symptom

Connection properties and client properties are enabled in the virtual host runtime as requested and how to verify the headers and payload.

Environment

  • SAP BTP API Management

Reproducing the Issue

  1. Follow the blog referenced in the case: how to read client certificate details in an API proxy in SAP API Management.
  2. Attempt to enable the connection properties and client properties for the virtual host runtime.
  3. Debug the virtual host runtime using the API Proxy debug functionality to verify the headers and payload.

Resolution

  • Enable the connection properties and client properties for the virtual host runtime as requested.
  • Use the debug functionality in the API Proxy within SAP API Management to verify the headers, payload, and custom variables created by the AssignMessage policies.
  • Perform the following actions during debugging:                   
    • Activate a trace session and connect to the proxy.Review all captured headers and payload information.
    • Verify the custom variables created by the AssignMessage policies.
    • Confirm that the client certificate details were accurately extracted and assigned.

Note that changes to runtime properties may require some time to take effect in Integration Suite.

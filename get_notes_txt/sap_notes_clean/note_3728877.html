Symptom

You are using the Run Statutory Reports app to create the financial audit file (report category NL_AUDIT_FILE), but the report takes too much time to run or ends with the following error:

SQL error on database selection: cannot allocate enough memory: search table error: [9] Memory allocation failed;

Other Terms

Netherlands, NL, Audit File, NL_AUDIT_FILE, ACR, SAP S/4HANA for Advanced Compliance Reporting, DRC, Document and Reporting Compliance, Run Statutory Reports

Reason and Prerequisites

This SAP Note is relevant only for the Netherlands.

Solution

As a general rule, SAP recommends that you install a solution by applying a Support Package. However, if you need to install a solution earlier, use the Note Assistant to implement the correction instruction. You can find more information about the SAP Note Assistant in the SAP Support Portal, under SAP Note Assistant.

To implement the solution, you must carry out manual activities and implement the correction instructions that are valid for your release.

In case of activation errors, you can try to activate the objects contained in this SAP Note by executing the reports below:

  • For DDLS objects, execute the RUTDDLSACTSNOTE report according to the steps available in SAP Note 2607137.
  • For DCLS objects, execute the R_ACMTOOL_DELETE_INACTIVE_V report according to the steps available in SAP Note 2691812.

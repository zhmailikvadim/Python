Symptom

  • The use of `UI.SelectionPresentationVariantType` to display multiple views in a Fiori Elements V4 application results in the display of "(0)" in the tab name of the view on the screen.
  • The "(0)" displayed in the tab name is unnecessary, and there is no option to remove it.
  • In previous versions (1.136.1), the issue does not occur, but it appears in the version used via CDN (1.142.0).
  • Setting `showCounts: false` in `manifest.json` does not hide the number of tab names.

Environment

Fiori Elements V4

Cause

Resolution

It is bug of 1.136.1 , so behavior of 1.136.1 is wrong.
Please use SAPUI5 1.136.12 or later.

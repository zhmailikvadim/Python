Symptom

The Edge web browser control of SAP Business Client displays status messages as a tooltip in the bottom left corner of the browsing area. When hovering a link, that status tooltip contains the target URL. After clicking a link which opens a new tab, that status tooltip erroneously remains in the foreground and covers contents of the new tab / new application. It only disappers after switching back to the original tab. This also happens for stacked canvases in the same tab, for instance if a native SAPGUI transaction is stacked over the FLP home page.

Other Terms

SAP Business Client; SAP BC; NWBC; Edge; Webview2; Tooltip; URL; URI; Statusbar; Status Bar;

Reason and Prerequisites

Program error

Solution

The issue was resolved in SAP Business Client 8.00 PL21 and PL23. However due to the implementation of Microsofts Webview2 web browser control there could be still situations even with PL23, where the tooltip does not disappear correctly again. In such situations please apply SAP Note 3690274: Install SAP Business Client 8.00 PL24 and set in the administrator configuration file (%programdata%\SAP\NWBC\NwbcOptions.xml) the EdgeSetting EnableStatusTooltip to value false.

Symptom

For migration transactions created via transaction code ‘/KJRTAX01/ATMIGINS’, for the transaction type is retirement or transfer retirement and the quantity is not zero, when you run Property Tax Report (RAIDJP_TAX), the retirement quantity of this migration transaction was not displayed correclty in the field 'Retirement: Quantity' of ALV list or the field 'Quantity' of retirement line item list PDF form.

Other Terms

Japan, Property Tax Report, RAIDJP_TAX

Reason and Prerequisites

This is a correction of legal funcion.

The error reason is retirement quantity is not considered for migration transaction in program.

Solution

Apply the latest support package or implement the correction instructions in this note.

Symptom

You might receive the following conflicting error messages for invoices or IR documents based off a contract fixed-fee item:

  • Reference Date must not be after xx/yy/zzzz
  • Reference Date must not be before aa/bb/zzzz

The date, aa/bb/zzzz is after the date, xx/yy/zzzz, so there is no valid date I can select to resolve the issue.

Reproducing the Issue

  1. Submit a contract-based invoice or approve an Invoice Reconciliation (IR).
  2. Encounter error messages related to the Reference Date field.

Cause

This error occurs because the End Date is after the Billing Date (start date) for the contract line item that is being invoiced.
The End Date field is displayed only if the Recurring field is set to Yes. If the Recurring field is set to No, the End Date field is hidden from view.
The Reference Date was left blank. It is a required field.

Resolution

To resolve the issue, have the contract owner take the following steps:

  1. Change or Amend the contract.
  2. Change the contract request document if you use only contract compliance.
  3. Amend the contract workspace document if you use Ariba Contract Management.
  4. Change the Recurring field to Yes. This causes the End Date field to appear.
  5. Change the End Date field to a future date.
  6. Change the Recurring field back to No.
  7. Submit or Publish the contract.

The invoice or IR document based off this contract item no longer displays the error messages related to the Reference Date field.

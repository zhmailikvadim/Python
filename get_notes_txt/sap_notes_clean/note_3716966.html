Symptom

  • When the Display Name value is updated in IAS for a user, it is not updated in Work Zone after successful IPS sync.
  • The Display Name in Work Zone reflects outdated values despite being updated in IAS and confirmed via SCIM API.

Environment

  • SAP Build Work Zone Advanced Edition
  • SAP SuccessFactors Work Zone

Reproducing the Issue

  1. Update the Display Name value for a user in IAS.
  2. Perform a user sync via IPS.
  3. Observe that the sync is successful but the Display Name in Work Zone does not reflect the updated value from IAS.

Cause

The “Use profile information from the SCIM API” is not enabled in Work Zone Admin Console > Feature Enablement > Features. As a result, Work Zone does not consume Display Name value from IAS via the SCIM API during the IPS sync.

Resolution

  1. In Work Zone, go to Admin Console > Feature Enablement > Features. 
  2. Under Integration Features, enable “Use profile information from the SCIM API”. 
  3. Save the changes.

Important: once this feature is enabled, users will no longer be able to manually edit their Display Name, Location, or other profile attributes in Work Zone, as these fields will be managed by the identity provider.

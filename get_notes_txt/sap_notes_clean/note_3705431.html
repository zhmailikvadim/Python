Symptom

Screen Personas flavors created with BDT (Business Data Toolset) transactions don’t work as expected after upgrading to SAP S/4HANA Cloud Public Edition 2602.

Environment

  • Version: SAP S/4HANA Cloud Public Edition 2602.
  • Screen Personas flavors created on transactions built on the BDT framework (for example BP, BCA_BUPA, and FPP1).

Cause

Before SAP S/4HANA Cloud Public Edition 2602, Screen Personas flavors for BDT-based transactions were typically created by enabling BDT mode (parameter ENABLE_BDT_MODE = X) . However, there may be existing flavors that were created with BDT transactions without having enabled BDT mode.

As of SAP S/4HANA Cloud Public Edition 2602, BDT mode is enabled by default, causing incompatibility with those existing flavors.

Resolution

To restore the functionality of existing BDT flavors, explicitly disable BDT mode by setting the parameter DISABLE_BDT_MODE = X.

To disable BDT mode, proceed as follows:

  • Launch "Manage Flavors" app.
  • In the navigation panel on the left, select Settings (under Customizing).
  • Add the following parameter:
    • Parameter Name: DISABLE_BDT_MODE
    • Value: X
  • Save your changes.

Once you set this parameter, existing BDT flavors work as expected.

Symptom

In the standard system as currently delivered, you execute either report Delete Personnel Numbers Completely (RPUDELPN) or Destruction of Personnel Numbers in Live Systems (RPUDELPP) to completely delete personnel numbers, but the system does not delete the personnel number entries from the following tables:

  • T5QABPY
  • T5QABPYFS
  • T5QDWH

Other Terms

Class: CL_IM_HRPAYAU_DELETE_TABLE

Method: IF_EX_HRPAYXX_DELETE_PERNR~DELETE_DATA

Reason and Prerequisites

When a Personnel Number(PERNR) is deleted, the corresponding entries in the following tables are not removed. This behavior is not in line with Data Protection and Privacy design considerations.

  • T5QABPY
  • T5QABPYFS
  • T5QDWH
Note Version Reason for Versioning
       01  General Release

Solution

The required logic has been implemented in method IF_EX_HRPAYXX_DELETE_PERNR~DELETE_DATA to delete the corresponding employee details when a Personnel Number (PERNR) is deleted.

As a general rule SAP recommends that you implement the Support Package specified for your release. However, if you need to install a solution earlier, use the SAP Note Assistant to implement the correction instruction. You can find more information about the SAP Note Assistant in SAP Support Portal, under SAP Note Assistant.

Symptom

During Advanced-CPE(ACPE) adoption(note 3628704), for an existing customer converting data to ACPE as described in note(3631220) , you want to convert an existing custom CPE routine to adopt to a standard CPE routine on a CPE term(table CPE_TERM is updated with the new routine number), then this note provides a BAdi to apply this behavior.

In ACPE implementation, existing custom CPE routines must be re-implemented using the new BAdi approach as described in note 3628704. But if any standard routine fits the behavior and you want to replace the custom CPE routine with a standard routine that fits, then this new BAdi can be implemented to provide the replacement routine mapping.

This BAdi is called during routine conversion step in CPE_CONV_WB(Data conversion workbench). This BAdi currently supports reference date routines and maturity selection routine.

Other Terms

Advanced-CPE data conversion, CPE.

Reason and Prerequisites

This is an additional feature supporting customers to adapt standard CPE routine instead of reimplementing custom CPE routine in ACPE.

Solution

Implement BADI BADI_CPE_DATACONV_CUSTOM_ROUT to provide the mapping from custom to standard CPE routine. 

Note : It is for the customer to test the process to validate there is no impact due to this change. It should be validated that the standard CPE routines works similar to the custom CPE routine you are tring to replace.

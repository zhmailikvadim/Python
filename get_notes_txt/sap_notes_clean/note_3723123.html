Symptom

You are using SAP Datasphere or SAP Data Intelligence to replicate data from ABAP system. You may or may not be using SLT. You notice that the size of the buffer statistics tables like LTRDB_METRICS or LTRDB_STATEVENT keep on increasing as long as the replication is running. If the replication is running for a year, the statistics are not deleted for a year, leading to high volume of data in these tables.

Other Terms

DMIS 2018, SAP LT Replication Server, SLT, Datasphere, Data Intelligence, Replication Flow, RMS, Initial Load, RDB, statistics

Reason and Prerequisites

The ABAP-based source system or the SLT system is using DMIS 2018 SP13 or SP14 or

The ABAP-based source system or the SLT system is using DMIS 2025 SP00.

Solution

Implement the corrections in the SLT system or ABAP-based source system.

IMPORTANT:

For deleting the records, you must first maintain a key-value pair in the table LTBAS_RTNTN using transaction SM30. The key is "RDB_STAT_LONGRUN_CLEANUP" and the value is "X". Only when the key-value pair is defined, the statistics for long running replications will be deleted.

Note: There is another key "RDB_STAT_EVENT_RTNTN_DAYS" in the table LTBAS_RTNTN which determines the number of days the RDB statistics will be retained in the system after the replication is deleted. Now, the same key is also used for deleting statistics of long running replications. If it is not defined, the default value is 7 days. This means that if the "RDB_STAT_LONGRUN_CLEANUP" is enabled, all statistics from tables like LTRDB_METRICS and LTRDB_STATEVENT that are older than 7 days will be deleted. If you explicitly set the value of "RDB_STAT_EVENT_RTNTN_DAYS" to 30, then all statistics older than 30 days will be deleted for long running replication tasks.

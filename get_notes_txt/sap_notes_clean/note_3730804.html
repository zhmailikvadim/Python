Symptom

When the payroll driver for Australia (T-Code: PC00_M13_CALC) is run for an employee, the RESC wage type is generated with an incorrect amount. This has been observed when the employee has WPBP splits.

Example:

Consider an employee XYZ who has WPBP splits in PP 10.2024.

WPBP Split

Period

OTE Amount

Split 1

01.04.2025–21.04.2025

6021.76

Split 2

22.04.2025–30.04.2025

3978.24

Total OTE = 10,000 AUD

During payroll processing, the system calculates RESC based on the latest WPBP split OTE amount (3978.24 AUD), which is incorrect.

Other Terms

RPCSUPQF, RPCSUPQF_GEN, CALCULATE_RESC

Reason and Prerequisites

Correction of legal function

Prerequisites: SAP Note 3660060

Solution

After implementing this SAP Note, the system considers the OTE amounts from all WPBP splits for RESC calculation instead of only the latest split.

To get the changes, import the Support Package specified for your release.

Alternatively, apply the attached correction instructions.

As a general rule, SAP recommends that you implement the Support Package specified for your release. However, if you need to install a solution earlier, use the SAP Note Assistant to implement the correction instruction. You can find more information about the SAP Note Assistant in SAP Support Portal, under SAP Note Assistant.

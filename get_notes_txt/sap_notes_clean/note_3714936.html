Symptom

  • Extra text "No está en uso. Seleccione Español (Spanish)" is displayed beside the country name Español (Mexico) on the "Change Language" and Provisioning's company settings page in SAP SuccessFactors.
  • The text replacement or even the Manage Language tool functionality does not meet the requirement to remove the extra text.

Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.

Environment

SAP SuccessFactors HCM Suite

Reproducing the Issue

  1. Go to Settings.
  2. Co to Change Language.
  3. Observe the extra text "No está en uso. Seleccione Español (Spanish)" displayed beside the country name Español (Mexico).

Or

  1. Go to Manage Language
  2. Observe the extra text "No está en uso. Seleccione Español (Spanish)" displayed beside the country name Español (Mexico).

Cause

  • Configuration: The language names can be customized via the Manage Language tool and are stored in locale en_US. By default, the customization of language names will not display in the tool.

Resolution

  1. Download the English (US) custom locale file from the "Manage Languages" tool.
  2. Locate the key starting with "LOCALE_" in the downloaded file.
  3. Delete the key starting with "LOCALE_" or delete the entire customization of en_US.
  4. Save the file after making the changes.
  5. Recreate an empty customization for en_US.
  6. Upload the modified file back into the "Manage Languages" tool.

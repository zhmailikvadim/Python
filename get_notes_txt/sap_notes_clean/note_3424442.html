Symptom

When loading a requisition_rejection_reason.csv file in Procurement Operations Desk > System Settings > Requisition Denial Reasons with more than five records, the task completes successfully without errors. But then checking the existing records, only five are displayed.

Cause

The Scroll Bar does not show up in this part of the User Interface unless you are intently looking for it.

Resolution

Move your cursor to the far right of the screen and a scroll bar will appear, and you can then see the other records which were successfully loaded.

NB: a valid record in the requisition_rejection_reason.csv is a unique combination of the RejectionReasonCode + RejectionReason. Once a valid record has been created, the value for the RejectionReason cannot be modified. Therefore, if business requires a change to the value of the RejectionReason, a new unique combination must be created to substitute the existing one.

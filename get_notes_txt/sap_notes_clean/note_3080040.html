Symptom

When deploying project from SAP Business Application Studio to on-premise SAP Netweaver, any error occurs. See examples:

  • Failed to deploy Application: <Parse Error>
  •  UnhandledPromiseRejectionWarning: TypeError: Cannot read property 'split' of undefined
        at deploy (/local/npm/lib/node_modules/@sap/abap-deploy/lib/abap-client/deploy.js:59:29)
        at abapDeployCommand (/local/npm/lib/node_modules/@sap/abap-deploy/lib/commands/abap-deploy-flow.js:119:27)
        at process._tickCallback (internal/process/next_tick.js:68:7)
    UnhandledPromiseRejectionWarning: Unhandled promise rejection. This error originated either by throwing inside of an async function without a catch block, or by rejecting a promise which was not handled with .catch(). (rejection id: 3)
    [DEP0018] DeprecationWarning: Unhandled promise rejections are deprecated. In the future, promise rejections that are not handled will terminate the Node.js process with a non-zero exit code.

Environment

SAP Business Application Studio

SAP Netweaver ABAP Gateway

Reproducing the Issue

Deploy HTML5 project from SAP Business Application Studio to on-premise SAP Netweaver ABAP Gateway using "abap-deploy" command.

Cause

 The command "abap-deploy" is deprecated.

Resolution

Use "npm run deploy" tool to deploy Fiori application to ABAP instead of "abap-deploy" as per the following documentation:

https://help.sap.com/viewer/17d50220bcd848aa854c9c182d65b699/Latest/en-US/1b7a3be8d99c45aead90528ef472af37.html

Symptom

  • Error message "You were not automatically logged on to the remote data source. Please ask your administrator to verify the OAuth configuration. " would appear after accessing the direct live datasphere story
  • When accessing the direct live datasphere story, the authentication popup may be stuck at https://assets.sapanalytics.cloud/production/oauth/pkce-redirect.html?code=
  • In console log, error message "Failed to fetch data (Correlation Id: undefined), devMessage: Error [IOLayer]: (#1018) AUTHENTICATION_TIMEOUT
    Error [Protocol]: (#73) No response object given, clientStatusCode: 1018, serverStatusCode: 0" could be found out

Environment

  • SAP Analytics Cloud 2026.2.5

Reproducing the Issue

  1. Log onto SAC
  2. Access the direct live data connection to Datasphere story
  3. Observe that the authentication popup would hang on custom IdP logon URL or https://assets.sapanalytics.cloud/production/oauth/pkce-redirect.html?code= 
  4. After waiting for a while, the above authentication popup would disappear and the error message "You were not automatically logged on to the remote data source. Please ask your administrator to verify the OAuth configuration. " would appear

Cause

The issue is caused by a defect in the product.

Resolution

This issue has been addressed in Client Version 2026.2.7

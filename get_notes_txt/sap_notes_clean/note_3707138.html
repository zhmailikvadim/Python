Symptom

DELTA‑T (Transit/NCTS) is the system provided by the French customs authorities for handling transit procedures. The French customs authority converts the connection to DELTA‑T for electronic data exchange (EDI) from the previous system MAREVA to GUN2 (Guichet Unique National – Version 2).

This SAP Note describes the steps required to request access to DELTA‑T.

Information about the complete technical changeover of the transit procedure to GUN2 is described in SAP Note 3708127.

Other Terms

France, FR, DELTA-T, shipping, UCC

Solution

The steps listed here are a summary from the publicly available documentation of the Delta-T ZIP archive PROJET_Package_cso_certif_deltat_phase-6.zip (path: PROJET_Package_cso_certif_deltat_phase 6 > GUN)

1. Create an account in the customs portal

If you do not already have one, create a user account in the following customs portals:

  • Production environment: https://douane.gouv.fr/
  • Test environment:         https://form.douane.gouv.fr/

2. Agreement to Use Customs API

Relevant documents:

  • edi-douane-via-api-processus-1-contrat-utilisation-public (1).pdf
    • Description of the steps to conclude the user agreement
  • DocOperateurs_APIpourEDIDouane_versioncompletediffusionrestreinteV1.9.pdf
    • Technical documentation for requesting and creating an API account (contains a link to the form formulaire-administrateur-douane_0.pdf)

After signing the agreement, send the contract along with the account names from step 1 to: certification-edi@douane.finances.gouv.fr. This is a prerequisite for the release for the creation of an API account.

Note: If an API account for DELTA‑IE already exists, the use of DELTA‑T must also be requested. The existing API account must contain an entry API_DELTA_T in which the URL and token can be stored. Contact certification-edi@douane.finances.gouv.fr in this regard.

3. Applying for an API Account

  1. Once you have received authorization to create an API account, you create it in the relevant customs portal. To do this, follow the steps in document DocOperateurs_APIpourEDIDouane_versioncompletediffusionrestreinteV1.9.pdf from the ZIP archive mentioned above.
    The ID and password of the API account are the access data to the customs system and must be defined in SEEBURGER BIS.
  2. After the account has been created and activated by customs (step 3.2.2), you must set up a callback address on the SEEBURGER BIS and define a Authentication Token. The callback address is the address from which the customs system can reach the SEEBURGER BIS in order to be able to transmit asynchronous response messages. Ensure that the address can be accessed from port 443 from the following IP addresses: 185.24.184.203, 185.24.185.203, 185.24.186.203, 185.24.187.203. Have the customs confirm the IP addresses.
    The Authentication Token can be any character string. This is added by the customs system in the header of the response messages and validated by SEEBURGER BIS.
  3. Once this data has been saved successfully, the API account is fully set up and can be used for communication.

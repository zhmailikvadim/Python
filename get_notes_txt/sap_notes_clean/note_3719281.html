Symptom

SAP Note 3677587 (PY-GB: Utility Classes 21 - Work Address) introduced a new feature GBWRK to define an employee's Workplace Address for a group of employees (or a single employee).

That solution included a field in the decision tree of feature GBWRK (IT0016-WRKPL) that should not have been included. Not only is the field IT0016-WRKPL not available in the GB screens of the Contract Details infotype (00016), but IT0016-WRKPL is not a suitable field for customers to use in a feature's decision tree - even if it was on the GB screens.

IT0016-WRKPL uses data element PAD_ORT01 – which makes the field case sensitive and free format, as such it does not have an F4 help (i.e. there is no customising table behind it that you can use to define fixed values, because the field is free format). The Data Element is used elsewhere as “City”. Any field that is case sensitive and free format (i.e. without an F4 help) is a poor choice of field to have as a decision tree field in a feature. Across different infotype records the same "City" can be misspelt by users (or use the wrong case or mixed case) and then the “City” won't be correctly identified by the feature GBWRK. Finally, the data element PAD_ORT01 is under the remit of International Cross Applications HR team and so the PY-GB team cannot change the format of the field (or whether it has an F4 help or not). As the Data Element PAD_ORT01 is widely used in multiple infotypes and a multitude of places, it would not be changed by the International Cross Applications HR team, even if the PY-GB team requested a change. If you want to use IT0016-WRKPL, you will have to not only add the field to the GB screens as a modification, but also have to work around the values being free format. As an alternative, the features BADI (HR_GB_FEATURES ->EXIT_FEATURES3) does allow you to use any field or any logic to derive Workplace Address.

This SAP Note removes IT0016-WRKPL from the decision tree of the feature GBWRK (i.e. removes IT0016-WRKPL from the structure PMEGO) and updates the documentation of the feature GBWRK to reference this SAP Note.

Other Terms

GBWRK, IT0016-WRKPL, PMEGO

Solution

Apply the relevant HRSP. These changes are not being delivered via SNOTE. If you wish to implement the changes ahead of the relevant HRSP, simply stop using IT0016-WRKPL in your decision tree of the feature GBWRK.

Symptom

When WPBP splits exist in the last month of a contract that has ended, the system generates incorrect blocks 53 with unit code 40. The sum of the number of days in blocks 53 unit code 40 exceeds the number of contract days in the month.

This issue also occurs when you perform a complementary pay for the last month of a contract that has ended with WPBP splits. In this case, the system also generates an unexpected block 51 of type 001 (RD for re-distribution) with the re-distribution of the remuneration of the contract's last active period. The system already declares this remuneration in block 51 of type 001 (RDF for re-distribution and fusion) with the retroaccounted difference.

After you send your DSN, you receive a Urssaf CRM 119 after 4 hours with the following anomaly:
"Une anomalie est détectée lorsque le nombre de jours calendaires de la période d'emploi pris en compte dans le calcul du plafond de Sécurité sociale (53.002 rattaché au 53.001 type 001, 53.003 unité de mesure 40) est supérieur au nombre de jour de présence du contrat du salarié dans le mois (40.001, 62.001).
Ce contrôle ne s'applique pas :
- aux contrats rattachés à une base assujettie plafonnée (78.001 type 02) ayant une date de début antérieure au 1er jour du mois contrôlé,
- aux individus en multi-contrats."

It is unclear whether the MSA performs a similar check. You have then to check your DSN for the incorrect blocks.

The January 2026 payroll results are correct.

 When you process a retroactive departure for an employee who has multiple WPBP splits in the departure month, the system declares block 53 'cancel and replace' entries twice. The system declares these entries once under parent block 51 type 001 'A' cancel and 'RD' redistribute, and again under parent block 51 type 001 'D' Delta or 'DF' merged delta.

11

When you hire an employee in month M under contract 00002 with start and end dates in month M-1, the system contains no relevant compensation or activities in the REM/ACTI cluster tables for month M. When you rehire this employee in month M+1 under contract 00003 with a start date on the first day of month M, the system declares the activity for month M+1 twice in the monthly DSN for MPD M+1. The system displays one entry under block 51/type 001 for the new contract 00003 and another entry under block 51/type 001 for the former contract 00002. The system should declare the activity only once under the new contract 00003.

Other Terms

DSN, WPBP, CRM, MSA

Reason and Prerequisites

WPBP splits exist in the last active period of a contract that has ended.

Solution

After implementing this SAP Note, the system does not generate the incorrect blocks 51 of type 001 (RD) and blocks 53 with unit code 40 described above.

Check whether you received CRM 119 with the above anomaly for the DSN you already sent. Check whether you have unexpected blocks 51 type 001 (RD) and blocks 53 with unit code 40. If you have these blocks, implement this SAP Note immediately and run the DSN again. For the DSN you already sent, send a DSN of type "cancel-and-replace".

If you do not correct these errors in the January 2026 DSN, correcting them in the February 2026 DSN becomes complicated. You need to create technical wage types to create a fictitious difference with the January 2026 payroll results, which are correct.

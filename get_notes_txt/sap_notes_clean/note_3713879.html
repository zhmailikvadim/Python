Symptom

Encountered an Error:ConfirmationRequest is not updating line item included in the previous confirmation request when attempting to send a new order confirmation (OC) to update the previous.

Reproducing the Issue

Submit a cXML order confirmation with type='update' for a purchase order that previously had all line items confirmed in the previous order confirmation with type="new".

When submitting it exclude any line item from the update document version as compared to the previous version and observe the error message with failure of the order confirmation.

Cause

When an order confirmation update is sent referencing a previous version without including all the original line items, a validation failure occurs.

This typically happens when the confirmation is sent from an external system as a cXML file to your SAP Business Network account.

The validation process requires all original line items to be present in updates, even if some items have already been confirmed and shipped, to prevent mismatches.

Resolution

Send a new order confirmation from your external system that includes all line items as of the previous order confirmation even if some line items are confirmed or shipped.

Note: Alternatively, you may create a new online confirmation directly from your SAP Business Network account, ensuring the inclusion of all line items.

Symptom

  • Objects are not getting mapped correctly during change source in BI 2025 when new column is added at the beginning or middle of existing column in updated SQL. 
  • Objects map correctly if a new column is added at the last of existing columns in the updated SQL.

Other Terms

fhsql, bi2025, change source, webi, new column, new object, middle, first, FH-SQL

Reason and Prerequisites

Environment:

  • SAP BusinessObjects Business Intelligence 2025 Patch 07
  • Web Intelligence
  • FH-SQL

Reproducing the Issue:

  1. Log in to BI Launchpad.
  2. Create a new Web Intelligence document using FH-SQL in BI 2025 using efashion connection.
    • SELECT
    • T.'Mth',
    • T.`Sales_revenue`
    • FROM
    • Agg_yr_qt_mt_mn_wk_rg_cy_sn_sr_qt_ma T
  3. Perform the Change Source to update the SQL. And add a new column in the beginning of the SQL, here Year column.
    • SELECT
    • T.'Yr',
    • T.'Mth',
    • T.`Sales_revenue`
    • FROM
    • Agg_yr_qt_mt_mn_wk_rg_cy_sn_sr_qt_ma T
  4. Notice in object mapping window the objects are not mapped correctly.
  5. Cancel the workflow and re-perform the same now instead of adding new column in the beginning of the SQL add the new column after existing column at last.
    • SELECT
    • T.'Mth',
    • T.`Sales_revenue`,
    • T.'Yr'
    • FROM
    • Agg_yr_qt_mt_mn_wk_rg_cy_sn_sr_qt_ma T
  6. Notice now the objects maps correctly.

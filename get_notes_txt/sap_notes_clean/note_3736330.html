Symptom

  • Handling unit inconsistency prevents the goods receipt from being posted for an inbound delivery.
  • Error message: HUGENERAL 327 during goods receipt in VL32N.
  • Error "HU items' material posting is inconsistent" is issued in VL32N.

Environment

  • Logistics General (LO)
  • SAP R/3
  • SAP R/3 Enterprise 4.7
  • SAP ERP Central Component
  • SAP ERP
  • SAP Enhancement package for SAP ERP
  • SAP Enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA

Reproducing the Issue

  1. Start transaction VL32N and enter the inbound delivery.
  2. Choose Post Goods Receipt.
  3. The system raises error HUGENERAL 327 and the posting does not complete.

Cause

  • The purchase order uses account assignment category ‘K’ (value-only update, for example to a cost center). Such documents are not logistics relevant and do not post stock to a storage location; therefore, using handling units and processing deliveries for these items is not intended in standard. This leads to a handling unit status inconsistency and error HUGENERAL 327 during goods receipt.

Resolution

For scenarios that require handling units and goods receipt via VL32N, ensure the underlying purchase order does not use account assignment category ‘K’. Use a purchase order with a blank account assignment so that the process is logistics relevant and stock is posted to a storage location.

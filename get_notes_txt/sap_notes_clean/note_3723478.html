Symptom

For Oracle field type (TIMESTAMP WITH TIME ZONE) and (INTERVAL DAY TO SECOND), they are not supported in CR JDBC driver.

Other Terms

Oracle, field type, interval, jdbc

Reason and Prerequisites

Environment:

  • SAP Crystal Reports 2020 Support Package 4

Steps to Reproduce:

  1. Open SAP Crystal Reports 2020 Designer.
  2. Create a new connection using Databricks database along with JDBC as middleware driver.
  3. Add the JDBC connection details (put non-admin DB user) as the connection string and click finished. 
  4. Add a table which contains a field which type is  (TIMESTAMP WITH TIME ZONE) or (INTERVAL DAY TO SECOND), "Not implemented" error will prompt

Cause:

This db type is not supported.

Solution

This issue is fixed in the patches listed in the "Support Packages & Patches" section below.

Due to different internal processes, the "Support Packages & Patches" section of the SAP Note will be populated as below:

               - for fixes delivered in "Support Packages": immediately (even if the "Support Package" is planned to be released several months later)

               - for fixes delivered in "Patches":  ONLY once the "Patch" is released AND available on SAP ONE Support Launchpad (not before)

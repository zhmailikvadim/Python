Symptom

If Payment Medium Workbench skips introductory payment(s) of Payment Run, either because the amount(s) exceed(s) PMW Format's limit defined in the customizing or when PMW Event 31 resp. 30 module raises exception NO_PAYMENT, the being generated output file can get split due to wrongly called PMW Event 25.

Other Terms

PMW, Payment Medium Workbench, FI_PAYM_MEDIUM_WRITE, skipped payment, Event 25, file split, Event 30, Event 31, exception NO_PAYMENT, amount exceeds format's limit, BFIBL02240, BFIBL02 240, 240(BFIBL02)

Reason and Prerequisites

Program error.

Solution

Implement attached program correction or import relevant Support Package.

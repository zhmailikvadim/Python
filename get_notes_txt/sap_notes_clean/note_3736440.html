Symptom

  • Gmail recipients of Onboarding Emails see the header display a long XML/error code instead of the intended image.
  • The issue appears limited to Gmail mailboxes.
  • In some cases, downloading the email locally results in normal display.

Environment

SAP SuccessFactors Onboarding

Reproducing the Issue

  1. Configure and send an Onboarding Email via Email Services.
  2. Receive the message in a Gmail mailbox.
  3. Observe that the email header does not render the image and instead shows XML/error code.

Cause

Improper mailbox setting at the recipient.

Resolution

Some mailboxes automatically load all images (eg. the Apple Inbox), while others are more protective and only load for senders that are on the Safe Senders List. 

Here are the options that are displayed when right-clicking on such email message.

Explicitly mention to the recipient that the Sender should added to the Safe Sender List.

SAP Community Link: Emailing/sending error – SAP Community.

If there are questions about mailbox settings, contact the recipient’s local IT for help.

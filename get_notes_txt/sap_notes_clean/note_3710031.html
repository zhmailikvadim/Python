Symptom

  • Saving Analysis for Office (AFO) workbook to Business Intelligence server creates dsws files in the Input File Repository Server (FRS) folder.
  • These temporary files are not automatically removed, eventually filling up disk space.

Other Terms

dsws, AO, AFO, Analysis for Office, workbook, report, temporary, temp, FRS, Input FRS, File Store, delete, remove, auto, automatically.

Reason and Prerequisites

Environment:

  • SAP BusinessObjects Business Intelligence Platform (BI) 4.x
  • Supported SAP Analysis for Office 

Steps to Reproduce:

  1. log in to AFO.
  2. Create a workbook. 
  3. Save a workbook on the SAP Businessobjects BI platform.
  4. Notice the generation of multiple dsws files in Input FRS folder.
  5. These files are not automatically deleted even after the hot backup period (4 hours by default) ends.

Cause: 

The cause is currently under investigation by SAP Development.

Solution

The issue is currently under investigation by SAP Development.

Workaround:

Disable Hot Backup from CMC > Settings

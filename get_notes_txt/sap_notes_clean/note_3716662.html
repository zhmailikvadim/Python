Symptom

You want to use the vipReason field in the Reference Field Usage for Case from the Contact Person, but it is not available.

Environment

  • SAP Sales Cloud Version 2
  • SAP Service Cloud Version 2

Reproducing the Issue

  1. Navigate to Settings > All Settings.
  2. Go to the Extensibility Administration.
  3. Select Case Service > Case.
  4. Go to the Reference Field Usage.
  5. Click Create.
  6. You cannot find the vipReason field under the Contact options.

Cause

The vipReason field is intentionally excluded from the contact entity for reference usage due to the complexity of contact relationships. Here is a detailed explanation:

A contact can have multiple "Is Contact Person For" relationships, and the VIP information can vary for each of these relationships. For example:

  • Contact ABC works for Company 1 with the VIP code "managing director."
  • The same Contact ABC works for Company 2 (default Contact Person relationship) where he is not a VIP.

In this scenario, the contact person's root entity VIP value would be empty because it reflects the default "Is Contact Person For" relationship. If a case is created involving Contact ABC and Company 1, the correct VIP reference value should be "managing director," but the system would show it as empty due to the default relationship.

Because of this complexity and potential for inaccurate data representation, our development team has excluded all relationship-specific contact root attributes, including the vipReason field, from reference usage in the Case entity.

Resolution

There is no direct resolution to include the vipReason field due to the above-mentioned complexities. You may consider alternative approaches to capture and manage vipReason information, such as custom fields.

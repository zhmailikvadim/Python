Symptom

You open the Functional Location List in IL05 or IH06. You display Classification data for these Functional Locations. You select objects that show Class Data. You change master data and you save it. When the application returns to the ALV list, the classification section is empty. The classification data appears again only after you run a manual list refresh with F8 Update.

Other Terms

IL05, IH06, ALV Grid Control, RIIFLO20, FCODES_WITH_MARK_L, Classification Buffer, Characteristic

Reason and Prerequisites

This is due to a program error.
The list refresh updates only parts of the output after you save. The classification buffer and its display logic do not run again in the return flow PBO. A missing trigger in FCODES_WITH_MARK_L stop the classification section from rebuilding until you run a manual refresh.

Solution

Implement this SAP Note to align the handling of the classification to add the missing refresh trigger in FCODES_WITH_MARK_L so the system rebuilds the classification data when you return from maintenance. No manual post implementation activities are required.

This Note was created by implementing parts of Note 1656798 in the PBO.

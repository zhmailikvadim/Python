Symptom

After importing Personal Information data, Permission Group membership is incorrect because the associated users are not resolved for DG filters, whereas UI updates work correctly.

Environment

SAP SuccessFactors Employee Central

Reproducing the Issue

  1. Create a Permission Group using a DG filter from a Personal Information field.
  2. Update a test user’s Personal Information data via import.
  3. Verify the Permission Group, user is missing.
  4. Update the same field via the UI and verify, user is included correctly.

Cause

When Personal Information data is imported, the system does not populate the required user_sys_id in the Personal Information table. Permission Groups rely on this ID to identify users, so when it is missing, the group may not display the correct users. Updates made directly through the Personal Information UI are not affected and display correctly.

Resolution

  • The engineering team plans to enhance Permission Group behavior so that it retrieves user_sys_id from the Employment Information table instead of the Personal Information table. 
  • This enhancement is tracked under ECT-257214 and is tentatively planned for the 1H2026 release, though the timeline may change. 

Workaround: You can update data for impacted users via the UI or use DG filter fields from other HRIS elements, such as Job Information.

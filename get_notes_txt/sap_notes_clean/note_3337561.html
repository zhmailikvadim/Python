Symptom

Error in "Run ABAP reports" phase of system copy in SWPM.

(Last error reported by the step: The JVM reports an exception during execution of class ( com.sap.sdt.ins.component.nw_ci_instance_abap_reports.RunRSWBOINS ) and function executeStepClass. DETAILS: The reported error message from JVM is: java.lang.InternalError: java.io.FileNotFoundException: \C:\Users\xxxxx\AppData\Local\Temp\2\sapinst_exe.xxxx.xxxxxxxxxx\jre\lib\ext\sunec.jar

Checking on filesystem above file indeed is missing.

Environment

AS ABAP system

Reproducing the Issue

Run the Software Provisioning Manager (SWPM) tool to perform an Installation / System Copy

Cause

SAPinst 749 and higher is shipped together with SAP JVM (Java Virtual Machine) binaries. During starting SWPM tool, it extracts itself into temporary directory %userprofile%\.sapinst\sapinst_exe.xxxx.xxxxxxxxxx, and prepares JVM files in

Initialization of 'SAPINST_JRE_HOME':
    Property 'SAPINST_JRE_HOME' is C:\Users\xxxxx\AppData\Local\Temp\2\sapinst_exe.xxxx.xxxxxxxxxx\jre
    Environment variable 'SAPINST_JRE_HOME' is C:\Users\xxxxx\AppData\Local\Temp\2\sapinst_exe.xxxx.xxxxxxxxxx\jre

Resolution

If due to some reason the file is missing on filesystem, download JVM package and extract to get the missing file, and put in path as per the error message.

For example C:\Users\xxxxx\AppData\Local\Temp\2\sapinst_exe.xxxx.xxxxxxxxxx\jre\lib\ext\sunec.jar

Close SWPM's UI, check its process stopped on OS level, then restart SWPM tool which should extracts itself into a new temporary directory %userprofile%\.sapinst\sapinst_exe.xxxx.xxxxxxxxxx, and prepares JVM files again.

Symptom

  • When attempting to create or maintain a work schedule using transaction PT01, the system raises the following error:
    Message No. TK430
    “The system administrator has set your logon client to the 'not modifiable' status. Client-specific objects cannot be changed in this client.”

  • The transaction is terminated and no changes can be saved.

Environment

  • SAP S/4HANA 
  • SAP Human Capital Management (HCM) 
  • Personnel Time Management 
  • Time Data Recording and Management

Reproducing the Issue

  1. Log into the affected system/client.

  2. Execute transaction PT01.

  3. Attempt to create or modify a work schedule.

  4. The system issues error TK430 and stops processing.

Cause

  • The client is configured as “not modifiable” in table T000 (typically a productive client).

  • Customizing changes (including work schedule generation) are not allowed in such clients by design.

  • This is a standard system protection mechanism to prevent direct changes in productive environments.

Resolution

The error TK430 is expected system behavior when the client is set to “not modifiable” (typically a productive client maintained in table T000).

  • Customizing activities such as work schedule generation in PT01 cannot be performed in such a client.

  • Maintain and generate the work schedules in a customizing or development client instead.

  • Transport the changes to the productive client using the Change and Transport System (CTS).

This restriction is part of the standard SAP design to protect productive environments. Direct changes in a productive client are not recommended and not supported.

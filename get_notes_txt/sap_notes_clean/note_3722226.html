Symptom

In the Latest People Profile, employees can view sections that they do not have permissions to, which appear with "No Data".

Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.

Environment

  • SAP SuccessFactors HCM Suite
  • SAP SuccessFactors Employee Central

Reproducing the Issue

  1. Go to Employee Profile
  2. Select the category from the left menu (in this example we will use Job Data)
  3. Click View All on the card (in this example we will use Job Information)



  4. See sections showing with 'No Data' even though the employee does not have permissions to any fields in these areas. (in this example we will use Position and Time Information)

Cause

This is working as per system design.

Resolution

The sections are hard coded and not controlled by Role Based Permissions. Currently, no configurations are present to alter them.

If the user does not have any permissions to the element data, the section will appear as 'No Data' on the card View All.

In case this is important to your business, we encourage you to submit an idea for SAP SuccessFactors Products to have this functionality considered within future development cycles, please consider opening an enhancement request. For detailed steps on How to Submit Ideas for SAP SuccessFactors Products, please refer to KBA 2090228.

Symptom

In all of the sudden it's perceived that the SAP Cloud Connector(SCC) is down.

Environment

  • SAP Cloud Connector(SCC);
  • Operating System Linux/Windows based.

Resolution

1. From the Operating System level, validate if SCC process is up and running with following OS commands:
 
 - ps aux | grep -i scc           <linux>
 - netstat -an | grep 8443    <linux>
 - tasklist | find "SCC"           <windows>
 - netstat -an | find "8443"   <windows>

2. If the results above confirm SCC process is running, validate what's the error returned from the browser https://<SCC-host/IP-Address>:8443;

3. Set SCC trace loggers to DEBUG directly from SCC OS level configuration file, as per KBA 2679740;

4. Restart SCC process from OS level, collect either ljs_Trace.log, scc_core.trc or scc_services.log (windows) and validate if there is any particular error raised from the trace at the time SCC web UI is launched.

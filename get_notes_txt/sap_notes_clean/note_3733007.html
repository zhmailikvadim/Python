Symptom

/SPE/CREATE_STO_FOR_DIRDLV queue stuck on ERP SMQ2 with error: 'ERP item category cannot be determined (Message No. /SPE/DIRDLV010)'.

Environment

  • SAP S/4HANA
  • SAP ERP Central Component (ECC)
  • SAP Service Parts Management (LO-SPM)
  • SAP Extended Warehouse Management (SCM-EWM)

Reproducing the Issue

  1. Open transaction SMQ2.
  2. Run the queue overview.
  3. Search for the relevant queue for direct outbound delivery creation.
  4. Open the queue entry to see that processing stopped with message /SPE/DIRDLV010.

Cause

  • The receiving plant has no purchasing organization assigned in customizing.
  • Multiple purchasing organizations are assigned to the receiving plant, but no default purchasing organization is maintained at plant level. In the direct outbound scenario the system tries to select the default purchasing organization for the receiving plant; if none is found, it raises /SPE/DIRDLV010.

Resolution

  1. Check the assignment of purchasing organizations to the receiving plant in customizing (Assign purchasing organization to plant).
    IMG: Enterprise Structure >Assignment >materials Management > Assign purchasing organization to plant .
  2. If no purchasing organization is assigned, assign the appropriate purchasing organization to the receiving plant.
  3. If multiple purchasing organizations are assigned to the receiving plant, maintain the default purchasing  organization for the plant 
  4. For example, in transaction OMKI; stored at plant level.
  5. Save the configuration and retry the direct outbound delivery creation, or reprocess the stuck queue in SMQ2.

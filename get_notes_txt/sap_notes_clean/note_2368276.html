Symptom

During the calculation of the annual special payment in TVoeD or TV-L, you want to specify the assessment rate manually using the infotype 0265 ("Special Regulations") with the subtype 03 ("Special Payment").

This is required for various constellations, for example, in the case of a mid-year forced special payment with a later re-entry in the same year, if different assessment rates apply for both periods according to the table. Another scenario is a harmful leave in combination with a standard pay increase, for which the amount of the annual special payment is to remain unchanged. This reduces the assessment rate in the table T5DPBS0E to compensate for the increase of the special payment. In the case of a harmful leave, the employee does not receive the increased pay scale table remuneration, but due to the reduced assessment rate, his special payment would be reduced retroactively. This can also be prevented using the new field in infotype 0265.

Solution

Import the Support Package assigned to this SAP Note or implement the advance corrections. If you implement an advance correction, also refer to the manual pre-implementation steps.

In both cases (import of the Support Package or advance correction), you should then update the table T588M. In this case, you must ensure that the new field for entering the assessment rate is displayed only on the subtype 03, but not in the subtypes 01 or 02. For this, the line with GRP 022 should be set to "Hide" for the variable keys 01 and 02, for the variable key 03 however, to "Standard Setting".

Symptom

  • Deleting email address or phone number in SAP S/4HANA is not synced to corresponding customer profile in SAP Customer Data Platform (CDP) via the standard business partner person source event.
  • After the source no longer contains an email, the contact in the platform still retains the previous primary email after ingestion.
  • Unable to reset the profile's email or phone number to null using the business partner person source event.

Environment

SAP Customer Data Platform

Reproducing the Issue

  1. Ingest a business partner person with an email or phone number using the SAP S/4HANA "Get Business Partner Person in real time" source event; a contact is created with primaryEmail/primaryPhone.
  2. Remove the email or phone from the business partner person in S/4HANA; the updated business partner person without email/phone is ingested via the same connector to CDP.
  3. Observe that the contact is updated but the previous primaryEmail/primaryPhone values remain and are not deleted.

Resolution

The primaryEmail and primaryPhone fields which are mapped by the default mapping are calculated fields. This means that the event will take the primary email and primary phone values which are address independent. If no such values exist, then the first email and phone values which are associated with the business partner will be taken.

There is a workaround to delete email address or phones by making the application configuration and mapping modifications below.

  1. In the application configuration, set null handling to "Allow null and overwrite".
  2. For the primary email mapping (repeat for primary phone as needed), add a conditional mapping with the "exists" condition so that when a value exists, it is mapped normally.
  3. Add a second mapping to the same target attribute to create an "else" condition.
    ->
  4. On the "else" condition, apply a transformation with the default value set to "Null" to force deletion when the source does not provide a value.
    ->
  5. Save the mapping.

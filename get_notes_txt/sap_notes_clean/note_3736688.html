Symptom

  • Duplicate email addresses exist among users in Incentive Management
  • The IM to IAS sync job fails or produces inconsistent results due to duplicate emails
  • Administrators are unable to modify user email addresses through the portal
  • Database surgery was considered but API approach is more efficient and safer

Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.

Environment

  • SAP Sales Performance Management
  • Incentive Management on Hyperscalers

Resolution

  1. Do run a GET for all of the existing users: 
    GET https://<TENANTID>.app.commissions.cloud.sap/usersvc/portal/services/scim/v2/Users/
  2. Identify you user and filter your endpoint:
    GET https://<TENANTID>.app.commissions.cloud.sap/usersvc/portal/services/scim/v2/Users/92
  3. Check the same user in the Portal as well 
  4. Copy the response from GET and open a new tab. Now perform a PUT method. At the new tab, paste the response to the body as JSON format. Modify the email address. Please note that, do not use the USERID, instead USERNAME to perform the email address modification.
    PUT https://<TENANTID>.app.commissions.cloud.sap/usersvc/portal/services/scim/v2/Users/Portal_user_test1 

  5. Now check user on the portal 

Note: If the PUT not works with the provided body, check the reference page, and copy the PUT example, and replace the details. API's only work if you provide all of the required details.

Symptom

You want to exclude purchase requisition commitments from availability control and prevent CO commitment updates for purchase requisitions (PReqs), while still allowing commitment updates for purchase orders.

Environment

SAP ERP
SAP S/4HANA

Cause

Consulting

Resolution

If you want to have proper CO availability control checks on PO level only you can suppress the creation/consideration of PR commitment. There exist two methods to achieve this.

Method 1: suppress creation of PR commitment
You can suppress the creation of PR commitment via implementing the correction contained in SAP Note 0001307755.
This means: PR commitment will not be created and stored in the corresponding tables (COOI, COSP, RPSCO etc.), hence the PR commitment will also not be considered in CO availability control and in reporting.

Method 2: suppress consideration of PR commitment in CO availability control only
You can suppress the consideration of PR commitment in CO availability control only via implementing the user exit as described in SAP Note 0000955107
This means: PR commitment will be created and stored in the corresponding tables (COOI, COSP, RPSCO etc.), the PR commitment will not be considered in CO availability control (i.e. it doesn't increase the assigned value in CO availability control) but it will still be considered in reporting (Commitment + Assigned values).

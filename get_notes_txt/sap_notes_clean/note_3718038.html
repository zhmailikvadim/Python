Symptom

For persons who enter a continuous semiretirement model as of 2026 and who receive additional benefits or overtime, the calculation of the wage compensation is incorrect.

Reason and Prerequisites

You have implemented SAP Note 3691430. This only introduced the adjustment of the replacement rate and the change of the model names. However, an adjustment of the wage compensation calculation was missing.

Solution

The correction is provided by HR Support Package (via CLC if necessary).
You can implement an advance correction by implementing the automatic correction instructions using SNOTE.

Details

As can be found on the AMS website, in the case of continuous semiretirement agreements whose term begins as of 1/1/2026 or later, overtime hours and additional work hours are no longer taken into account in both the lower value and the upper value.

For this reason, wage type /TOG is not used as the upper value when calculating the wage compensation for this person group, but wage type /TUG.

This results in the new formula: Wage Compensation /TLA = ( /TUG - /TUA ) / 2.

There is no change for continuous semiretirement agreements whose term started before January 1, 2026 and for block time models.

Retroactive accounting should be performed accordingly for persons with continuous semiretirement agreements as of 1/1/2026.

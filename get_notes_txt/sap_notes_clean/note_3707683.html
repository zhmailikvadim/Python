Symptom

  • Issue occurs while running the Joule Booster after completing all required steps as per Joule Onboarding Guide.
  • When re-running the Joule booster in an SAP BTP subaccount where Joule has already been configured, the booster execution fails with the following error message.

    Error message encountered: "Could not include system das-application-ias to the formation because it conflicts with the following rules:
    System das-application-ias can be part of only one formation of type Integration with Joule. This system is already part of another formation."

Environment

  • SAP BTP, Cloud Foundry runtime and environment
  • Joule

Cause

  • A single Joule subscription supports integration with multiple different SAP products but can only integrate with one tenant per product.
  • Data isolation is enforced at the tenant level, preventing multiple tenants of the same product from being part of the same Joule formation.

Resolution

  • To avoid the unique constraint violation, ensure that only one system/tenant per SAP product is added to a Joule formation.
  • If integration with multiple tenants of the same product is required, separate Joule subscriptions and formations must be used. 
  • If Joule has already been integrated into one product (with a Joule Formation in place), integrating Joule into another product type does not require running the Joule Booster. Instead, use the 'Include System' option in the existing Formation, as outlined in the referenced documentation Include or Exclude a System in the Joule Formation.
  • Please also take note of the Joule constraints outlined in the referenced link Constraints for Joule.

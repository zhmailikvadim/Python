Symptom

  • BAPI_HU_CREATE creates one single item instead of multiple identical items despite definition of multiple items.
  • The material item shares the same characteristics such as material number, batch, stock category, plant/location etc. 

Environment

  • SAP S/4HANA
  • Logistics Execution (LE)
  • SAP R/3
  • SAP R/3 Enterprise 4.7
  • SAP ERP Central Component
  • SAP ERP
  • SAP Enhancement package for SAP ERP
  • SAP Enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA Cloud Private Edition

Reproducing the Issue

For example:

  1. Using transaction SE37. 
  2. Execute BAPI_HU_CREATE. 
  3. Create sufficient test data in the Test Data Directory.
  4. Define 2 or more material items with the same identical characteristics. 
  5. Observe the result:  one line item is created instead of separate line items for identical items.

Cause

When the stock characteristics are identical, there can naturally be automatic accumulation of a packaging material line item when multiple of the same materials are required. 

Resolution

  1. This is correct system behavior for auto-packing. 
  2. If separate items are required this can be achieve via modification.
  3. Otherwise, depending on your process the items can be forcibly separated in manual intervention in the likes of VL02N.

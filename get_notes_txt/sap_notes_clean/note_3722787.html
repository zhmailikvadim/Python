Symptom

You wish to set up time valuation logic that checks for core time adherence

"Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental." 

Environment

  • Successfactors Employee Central
  • Time Tracking

Resolution

Below you can find a simplified example, you can adjust accordingly depending on core hours or days that should or should not count.

  1. Use a valuation of the type 'Create Time Record' that generates a time type group containing you core hours.


  2. Next, use valuation of type 'Deduct Group from Input Groups' to remove deduct the working times from your core hours group.
    The Threshold group in the below examples contains all time types that count towards  the core hours


  3. Assign these valuations to the time recording profile. If your working time has a gap within the core hours, the UI will now show a warning message

Keywords

valuation, overlap, gaps, cover

Symptom

  • The effective end date of the last version of a position cannot be updated via API due to system constraints.
  • Attempts to update the effective end date using three different approaches (PUT, DELETE, and creating a new position version) did not yield the expected results.

Environment

  • SAP Sales Performance Management
  • Incentive Management on Hypercalers

Reproducing the Issue

  1. Attempt to update the last position using the API endpoint '/positions' (PUT) with parameters 'ruleElementOwnerSeq', 'effectiveStartDate', and 'effectiveEndDate(today)'.
  2. Attempt to create a new "last" position using the API endpoint '/positions(ID)/versions' (PUT) with parameters `effectiveStartDate` and `effectiveEndDate(today)'.
  3. Attempt to delete the position using the API endpoint '/Position(ID)'  (DELETE) with parameters 'effectiveStartDate( tomorrow, newly created)' and 'effectiveEndDate(endDate 2200, newly created)'.

Cause

The inability to update the effective end date of the last version of a position via API is due to system constraints that prevent modification of the effective end date when it is part of the key.

Resolution

  1. Use the API endpoint '{{baseUrl}}/mtsvc/tcmp/rest/v2/positions(:id)/versions (POST)' to update the effective end date of the last position version.
  2. Include the following parameters in the body of the API call to end date a position:

    [{
    "effectiveStartDate": "YYYY-MM-DDT00:00:00.000+00:00",
    "effectiveEndDate": "YYYY-MM-DDT00:00:00.000+00:00"
    }]

    Remember to replace `YYYY-MM-DD` with the desired start and end dates.
  3. If multiple positions need to be updated, include additional periods in the body as follows:

    [    {
    "effectiveStartDate": "YYYY-MM-DDT00:00:00.000+00:00",
    "effectiveEndDate": "YYYY-MM-DDT00:00:00.000+00:00"
    },
    {
    "effectiveStartDate": "YYYY-MM-DDT00:00:00.000+00:00",
    "effectiveEndDate": "YYYY-MM-DDT00:00:00.000+00:00"
    }]
  4. Ensure that the periods are correctly specified to avoid creating additional versions.
  5. If calculation results exist for the position, the update may not be applicable due to system restrictions. In such cases, alternative approaches may need to be explored.

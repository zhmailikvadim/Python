Symptom

There is an expectation that document numbers for picking or warehouse-related processes follow distinct number ranges based on specific organizational units (such as Company Code, Plant, or Sales Organization).

Environment

SAP S/4HANA Cloud Public Edition

Reproducing the Issue

  1. Create outbound delivery documents for different Company Codes (e.g., 1000 and 2000).
  2. Perform picking for these deliveries.
  3. Observe the generated picking document numbers.
  4. Notice that:
    - The number range does not vary based on Company Code or Plant.
    - All documents follow the same number range interval defined at document type level.

Cause

In SAP S/4HANA Cloud Public Edition, the number range logic for logistics execution documents (including picking and warehouse-related documents) is not designed to be driven by organizational units such as Company Code, Plant, or Sales Organization. Number ranges are determined at document type level, and there is no standard configuration or extensibility available to influence this dynamically based on organizational data.

Resolution

This behavior is in line with the standard design of SAP S/4HANA Cloud Public Edition.

Number ranges for logistics execution documents, including picking and warehouse-related documents, are defined at document type level and are not determined based on organizational units such as Company Code, Plant, or Sales Organization. As a result, the system does not support the assignment of different number ranges depending on organizational data.

For requirements related to organizational differentiation, this is typically handled through alternative approaches. For example, it may involve capturing the relevant organizational unit separately (such as Company Code, potentially derived from plant or delivery data using available extensibility options), or handling the relationship at reporting level. 

Using standard analytical tools (such as Custom Analytical Queries), picking or warehouse documents can be combined with delivery and organizational data (e.g. plant and Company Code), allowing traceability without changing the document numbering logic.

If it is essential for the organizational unit to be reflected directly in the document number, this would require a product enhancement, as this capability is not available in the current standard functionality. In such cases, you are encouraged to submit an improvement request via the SAP Customer Influence portal, where enhancement requests can be proposed and voted on by other customers.

For guidance on how to submit a request, please refer to the following SAP Note:
3206294 – How to Submit Feature Requests for SAP S/4HANA Cloud Enterprise Portfolio and Project Management
https://me.sap.com/notes/3206294/en-US

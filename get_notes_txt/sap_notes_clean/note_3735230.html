Symptom

  • Using the standard sales area Copy function in MDG application, the target sales area is not populated for MVKESALES (sales data), Grouping data is copied, but MVKESALES remains empty or the source sales area row is updated again instead of the target. 

Environment

  • SAP S/4HANA
  • SAP Master Data Governance

Reproducing the Issue

  1. Open MDG Material UI and create a material; enter the relevant material type so that defaults are populated.
  2. Select a material with an existing sales area (source).
  3. Use the toolbar action 'Copy' to copy the sales area from the source to a different target sales area.
  4. Observe that grouping values are copied, but MVKESALES (sales data) for the target sales area is not created/populated, or the source sales area row is updated again.

Cause

  • A BRF+ derivation rule for entity MVKESALES is active. During write, derived data is present and the application ignores the copied sales data, so MVKESALES for the target sales area is not created/populated.

Resolution

Please implement below SAP Note to resolve this issue

  • SAP Note 3623431 - if_usmd_gov_api~set_message_container does not set message container correctly.

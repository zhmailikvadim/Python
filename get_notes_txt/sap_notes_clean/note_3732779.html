Symptom

In configuration SSCUI 102530 - Maintain Settings for Event-Based Revenue Recognition, Error FINS_REV_REC_URR013 "Select 'No IFRS15 processing' as Billable Determination Method" raised.

Environment

SAP S/4HANA Cloud Public Edition

Reproducing the Issue

  1. In customizing for Revenue Recognition Key for projects, set the bundling scenario to “Per Solution Order” and enable “Use Revenue Contracts”.
  2. Maintain a billable determination method for one accounting principle while another principle is active in Event-Based Revenue Recognition but not relevant for Contract-Based Revenue Recognition.
  3. Save the settings. The system raises: "Select 'No IFRS15 processing' as Billable Determination Method" (FINS_REV_REC_URR013).

Cause

If one of the Accounting Principal, for example USGP or IFRS is not relevant for CBRR, the billable method should be maintained as 'No IFRS Processing'.
This can happen as follows:
You may have activated both accounting principle IFRS and USGP in EBRR, but in the meantime, only one accounting principle is relevant CBRR. For example, you activate 3KK for IFRS, but not for USGP.  In that case,  the billable method shall be maintained as 'No IFRS Processing'. 

Resolution

  1. Open SSCUI 102530 – Maintain Settings for Event-Based Revenue Recognition.
  2. For the Revenue Recognition Key used for projects, select the accounting principle that is not relevant for Contract-Based Revenue Recognition.
  3. Set the Billable Determination Method to “No IFRS Processing” (leave it empty).
  4. Ensure the accounting principle that is relevant for processing has at least one supported billable determination method maintained (for example, Billable equals Invoice, Billable follows Percentage of Completion, or Billable at completion).
  5. Save the changes, then return to the key and enable “Use Revenue Contracts” and set bundling scenario “Per Solution Order” again to confirm the error is resolved.

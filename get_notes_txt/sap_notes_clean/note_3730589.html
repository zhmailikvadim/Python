Symptom

  • During Load Data Capture (LDC) for a Multiple Buy Sell (MBS) scenario, the load event is processed successfully but the unload event cannot be released.
  • The system selects the correct event location for the unload event but picks the wrong customer.
  • Encountered an error during LDC processing when executing the Multiple Buy Sell scenario for Intracompany, Sales, Purchase, Sale contract sequence.
  • Load event processed successfully however the unload event failed with an error: ‘Customer not valid’ and unable to release the unload event due to this error.

Environment

SAP S/4HANA

Reproducing the Issue

  1. Create an MBS chain where pegging includes two consecutive legs on the same side (for example, sales-to-sales or purchase-to-purchase).
  2. Create LDC based on this pegging and process the load event successfully.
  3. Attempt to release the unload event.
  4. Observe that the unload event picks an incorrect customer and displays the warning "Customer not valid".

Resolution

  1. Review the nomination pegging for the MBS chain used by LDC  it shouldn't be two consecutive purchase legs or two consecutive sales legs pegged to each other.  
  2. Ensure the pegging alternates between purchase and sales legs; remove any sales-to-sales or purchase-to-purchase pegging.

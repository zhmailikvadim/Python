Symptom

  • An inconsistency occurs in the monthly accumulation logic for Withholding Income Tax (IRRF) under law 15.270 during vendor invoice posting.
  • The system does not considering accumulated amounts when calculating the IRRF when processing multiple invoices within the same month.
  • A user with SAP_ALL authorization manages to post the invoices correctly.

Environment

  • SAP S/4HANA Cloud Private Edition
  • SAP S/4HANA
  • Brazil Withholding Taxes

Reproducing the Issue

  1. Process multiple vendor invoices within the same month.
  2. Observe the incorrect calculation of IRRF, considering accumulated amounts.

Cause

The new code delivered for S/4HANA to calculate the accumulated base uses CDS views I_OPERATIONALACCTGDOCITEM and I_SUPPLIER, in methods GET_ACCUMULATED_BASE_AMOUNT (CL_FIWTBR_REDUCN_IRRF_CALCTR) and IS_NATURAL_PERSON, respectively.

This requires specific authorizations that were not granted to the user. In particular, authorization objects F_ACDOCA_C and B_BUP_DCPD are required for the calculation to work correctly.

Resolution

  1. After the error occurring, check at SU53 if there are any failed authorization check.
  2. Ensure the user creating the document has the required authorization object F_ACDOCA_C and B_BUP_DCPD.

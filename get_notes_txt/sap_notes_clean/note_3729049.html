Symptom

  • When attempting to run an OData-based import job from SAP Datasphere into an SAP Analytics Cloud (SAC) Planning Model, the job fails or completes with significant rejections.

  • Error Message: "Null value is not allowed for single column dimensions."

  • The issue started occurring immediately following a tenant update to SAP Datasphere (late January 2026).

  • Thousands of rows are rejected, and the data preview in SAC may not explicitly show the null values if they are generated via associations.

Environment

  • SAP Analytics Cloud 2026.2.5  

  • SAP Datasphere 2026.5.1 

Reproducing the Issue

  1. Log in to the SAP Analytics Cloud tenant.

  2. Open a Planning Model that receives data from SAP Datasphere via an OData connection.

  3. Navigate to the Data Management tab and select an existing Import Job

  4. Run the import job.

  5. Observe the Data Timeline; notice a high volume of rejected rows.

  6. Download the Rejection Summary and observe the error: "Null value is not allowed for single column dimensions" for dimensions where data is joined via Associations in the source Datasphere view.

Cause

  • A recent update in SAP Datasphere modified the metadata handling of Association Joins exposed via OData. Fields that were previously handled as empty strings are now arriving as true NULL values.

  • Since SAC Planning models require every dimension to have a valid Member ID for every record, the system rejects rows containing NULL values unless a default mapping or handling instruction is provided.

Resolution

There are two primary methods to resolve this, depending on the desired scope of the fix:

Method 1: Connection-Level Handling

This is the most efficient fix as it covers all dimensions within the job.

  1. Open the affected Import Job in SAC.

  2. Navigate to Import Settings.

  3. Enable the switch: "Import Empty Member ID Cells".

  4. Save and Run the job.

Method 2: Manual Mapping

Use this if you only want to allow nulls for specific dimensions.

  1. Open the Mapping (Map to Model) step of the import job.

  2. Select the affected Dimension (e.g., Afdeling).

  3. In the right-side panel, locate the "Map Unmapped Values to" or "Default Value" field.

  4. Enter the hash symbol (#).

Performance Optimization

To reduce high turnaround times:

    1. Go to the Pagination Settings in the Import Job.

    2. Manually enter the Key Fields from the source view to enable Server-Side Paging.

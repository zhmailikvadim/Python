Symptom

  • Replication fails with the error: "Problem during invoking SuccessFactors Web service. SOAP faultCode S:Server, faultString SFAPI Domain Error!, errorCode INVALID_SFQL, errorMessage: Invalid SFQL! Error: You do not have the permission for segment onboardinginfo."
  • The issue occurs when the onboardinginfo template is not enabled in the Transformation template configuration.
  • The error is triggered for clients who do not have the onboardinginfo template enabled in the ERP system/template configuration.

" Important Note":  Preview upgrade on April 13th and Production upgrade on May 16th. The workaround to use CE API Admin Permission keeps relevant until the fix is solved. 

Environment

  • Product: SAP SuccessFactors Employee Central integration to SAP Business Suite
  • Compound Employee API

Reproducing the Issue

  1. Run the employee replication for a client where the onboardinginfo template is not enabled in the ERP/template configuration.
  2. Observe the error: "Problem during invoking SuccessFactors Web service. SOAP faultCode S:Server, faultString SFAPI Domain Error!, errorCode INVALID_SFQL, errorMessage: Invalid SFQL! Error: You do not have the permission for segment onboardinginfo."

Cause

Changes in OSS note 0003683821 added the onboardinginfo portlet by default to the CompoundEmployee selection query, which causes permission errors for clients without the onboardinginfo template enabled.

Resolution

  1. As a temporary workaround, grant Full Access to CompoundEmployee API for the affected API user to bypass segment-level permission checks for onboardinginfo. Refer to: Granting permissions for full access to CompoundEmployee API.
  2. Review guidance on segment-level permissions to understand the impact when onboardinginfo is included in queries. Refer to: Granting permissions for segment access to CompoundEmployee API.
  3. Refer to: 2807662 - EE EC to ERP: Replication query error for related query/field validation scenarios.
  4. Refer to: 3332844 - Employee Replication Error: "Invalid SFQL! Error: 'xxxxxx' must be specified in the SELECT clause" for additional SFQL guidance.
  5. A product fix from the CE API side is in progress. Keep the workaround active and follow this KBA for updates; revert to segment-based permissions once the fix is available.
  6. Preview upgrade on April 13th and Production upgrade on May 16th. The workaround to use CE API Admin Permission keeps relevant until the fix is solved.

Symptom

  • During the migration of interfaces from SAP PO to BTP Integration Suite, the XML payload output remains the same, but the BTP-IS AMQP output appears differently in the ActiveMQ server message.
  • Extra text is added above the actual text/XML payload, causing data processing failure at the ActiveMQ server.
  • Error message observed in ActiveMQ server while processing the payload delivered by AMQP adapter:
    "System.InvalidCastException: Unable to cast object of type 'Apache.NMS.ActiveMQ.Commands.ActiveMQBytesMessage' to type 'Apache.NMS.ActiveMQ.Commands.ActiveMQTextMessage'."

Environment

  • Cloud Integration
  • SAP Integration Suite

Cause

The message type error: "Unable to cast object of type 'ActiveMQBytesMessage' to type 'ActiveMQTextMessage'" happens because the consumer is trying to read the message as text, but ActiveMQ is actually receiving it as binary. 

Resolution

  1. Ensure that the message being passed to the AMQP adapter is plain UTF-8 text.
  2.  If any Groovy script or mapping step converts the payload into bytes or Base64, CPI will automatically send it as a BytesMessage instead of a TextMessage.
  3. If ActiveMQ supports decoding, encode the XML payload before sending it to AMQP to prevent interference from AMQP headers.
  4. Alternatively, consider using the JMS adapter instead of the AMQP adapter, as the JMS adapter sends only the XML body without AMQP headers.

Symptom

When additional nodes are added using the Service Layer Controller, their status initially appears as Init Ok. However, these nodes do not process any incoming requests. After some time, their status may change to Init Err.

Reproducing the issue

Prerequisite:

SAP Business One 10.0, version for SAP HANA SP2511 or higher installed

Scenario:

  1. Access the Service Layer Controller at https://<SLServer>:40005/ServiceLayerController/.
  2. Navigate to the Node Management section.
  3. Click Add.
  4. Specify the Port From and Port Count, then confirm by clicking Yes.
  5. Send test requests through the Load Balancer or monitor the Service Layer activity.

Expected behavior:

New Service Layer nodes are successfully added, their status is Init Ok, corresponding services are created and running on the host, and the nodes are able to process incoming requests.

Actual behavior:

When additional nodes are added using the Service Layer Controller, their status initially appears as Init Ok. However, these nodes do not process any incoming requests. After some time, their status may change to Init Err.

When checking the operating system, only processes related to nodes created during the initial installation are present. Additionally, when verifying the systemd units (for example by running command systemctl list-units or reviewing /etc/systemd/system), no services corresponding to the newly added nodes can be found.

The Load Balancer reports the following error in the Service Layer error log:

  • (111)Connection refused: AH00957: http: attempt to connect to 127.0.0.1:50005 (localhost:50005) failed
  • AH00959: ap_proxy_connect_backend disabling worker for (localhost:50005) for 60s

Solution

SAP intends to provide a feature package or patch to solve the symptom that is described in this SAP Note. The section References below lists the related feature package or patch once they become available. Be aware that References can only be confirmed at feature package or patch release date. SAP delivers feature packages or patches only for selected releases at its own discretion, based on the business impact and the complexity of implementation.

Workaround: 

Remove any newly added nodes via the Service Layer Controller and run the Reconfiguration to add new nodes via the installer.

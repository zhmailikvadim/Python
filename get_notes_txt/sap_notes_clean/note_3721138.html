Symptom

In SAP EarlyWatch Alert (EWA) the following warning appears:

Number of collectors with retention times < 42 days = 4

The customer verifies the SAP HANA configuration and executes:

SELECT s.ID,
       o.NAME AS OBJECT_NAME,
       s.RETENTION_DAYS_DEFAULT
FROM _SYS_STATISTICS.STATISTICS_SCHEDULE s
JOIN _SYS_STATISTICS.STATISTICS_OBJECTS o
  ON s.ID = o.ID
WHERE s.RETENTION_DAYS_DEFAULT < 42
ORDER BY s.RETENTION_DAYS_DEFAULT ASC;

The SQL query returns multiple statistics collectors with retention times below 42 days, which does not match the value reported in the EarlyWatch Alert.

Customer Interpretation

  • SAP HANA monitoring misconfiguration
  • Statistics server malfunction
  • Assumption that disabling collector 5008 is required

Environment

SAP HANA DB Version 2.0

Cause

The SQL query lists all statistics server collectors defined in SAP HANA.

The EarlyWatch Alert calculation is based on SAP monitoring content and therefore may not correspond to the complete list of collectors returned by the SQL query.

SAP HANA delivers several monitoring collectors with relatively short retention periods (for example 4 or 21 days) to control the growth of monitoring history data stored in schema _SYS_STATISTICS.

As a result, the number reported in the EarlyWatch Alert can differ from the number of collectors returned by the SQL query.

The EarlyWatch Alert finding is informational and does not by itself indicate a database malfunction or incorrect SAP HANA configuration.

The retention period configured for statistics collectors mainly affects the amount of monitoring history data stored in the _SYS_STATISTICS schema and does not impact database runtime behavior.

Resolution

No corrective action in SAP HANA is required from a database perspective.

Do not disable collector 5008 unless the specific BW Direct Extractor Connection (DXC) scenario described in SAP KBA 2493696 applies.

Changing retention settings is optional and only affects the amount of stored monitoring history data.

Symptom

When maintaining a Duration/Offset Values in an input field, the system may terminate with a runtime error. The dump occurs after saving and entering invalid characters (for example “//”) into Additional Duration or Additional Offset fields.

Instead of prompting for a valid value, the application terminates unexpectedly.

The issue is reproducible when the invalid entry is processed by the duration/offset conversion routine.

Other Terms

CONVERSION_EXIT_TSTRN_INPUT
CONVERSION_EXIT_TSTRN_OUTPUT
TSTRDURAN
Duration conversion routine

Reason and Prerequisites

The reason is a program error.

Invalid input values are not correctly handled within the underlying conversion routine, causing an exception during type conversion.

This leads to a runtime dump instead of returning an initial value.

Solution

Apply the attached Correction Instruction.

The correction enhances error handling in the conversion routine so that invalid duration/offset entries no longer trigger a dump but return a safe, initial output value.

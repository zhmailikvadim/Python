Symptom

When executing DRFOUT and trying to replicate the legacy org model, it fails with the error message “Missing authorizations for reading the database table 'CRMC_FSM_ORGC_MP'.” 

Environment

  • SAP Field Service Management
  • S4 On-Premise via CPI Connector

Reproducing the Issue

  1. Enter Replication Model into DRFOUT
  2. Fill Outbound Implementation along with Filter
  3. Press the button "Execute"
  • Expected: Org Level successfully replicated 
  • Actual: Error "Missing Authorizations for reading the database table 'CRMC_FSM_ORGC_MP'"

Cause

In PFCG Tcode, the role "SAP_S4C_SERV_FSM_ORG" is not generated.

Resolution

In PFCG Tcode, click on "Generate" Button & relevant role "SAP_S4C_SERV_FSM_ORG" would be generated

Keywords

PFCG, CRMC_FSM_ORGC_MP, DRFOUT, Missing Authorizations, generation

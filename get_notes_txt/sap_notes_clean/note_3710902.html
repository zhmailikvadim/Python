Symptom

  • In the SAP EarlyWatch Alert (EWA) report for an SAP ASE system, the Backup Strategy check shows a yellow warning.

    The EWA report displays messages such as:

    “We were unable to read your Database Backup Strategy information.”

    “There is less than 1 month of data available for analysis.”

  • Although database backups are being executed on the ASE system, no backup history data is displayed in the EWA Workbench under Database Administration.
  • The function module `/SDF/SYB_BACKUP_HIST_NEW` responsible for reading the ASE backup history returns no data.

Environment

SAP Solution Manager 7.2

Reproducing the Issue

  1. Navigate to Database Administration → Backup Strategy in the EWA report.
  2. Observe a yellow rating with a warning indicating missing or insufficient backup history data.
  3. Open the EWA Workbench and check Download Data → Database System → ASE.
  4. No backup history data is available.
  5. Execute function module /SDF/SYB_BACKUP_HIST_NEW in transaction SE37.
  6. The function module returns no results.

Cause

  • The issue is caused by a coding defect in the function module `/SDF/SYB_BACKUP_HIST_NEW` in the current ST-PI version, which prevents the ASE backup history from being collected.

Resolution

  1. Apply SAP Note 3587475 to correct the coding defect in the function module `/SDF/SYB_BACKUP_HIST_NEW`.
  2. Confirm that the backup history data is collected correctly and the warning is resolved in the next EWA report.

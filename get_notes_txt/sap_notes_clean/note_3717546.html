Symptom

DRTM retention purge timeframe -system uses the previous Termination event and not the more recent No-Show event.

Environment

SAP SuccessFactors Onboarding 

Cause

Expected Behavior

Resolution

As per the standard behavior while performing the Data Retention Management Retention time for the master data purge is calculated from the inactive user's termination date.

Because it overrides the configured retention times for all other types of data, the retention time for the DRTM Master Data purge should always be longer than the retention time for any other DRTM purge request types.

The master data purge is a full purge of data across the SAP SuccessFactors HCM suite.
When setting up a DRTM Master Data purge request, you cannot select which purge objects or types of data to include.

Hence, the system will use the previous Termination event and not the more recent No-Show event.

Symptom

You have checked table T5UTX in your system and noticed a wrong ceiling and/or rate kept for a Tax Type.

   

"Images/data in this KBA are from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental."

Environment

  • HR Renewal
  • SAP S/4 Hana
  • SAP ERP 
  • SAP SuccessFactors Employee Central Payroll

Reproducing the Issue

1) Open transaction SM30 and access view V_T5UTX.
2) Check an entry.
3) The ceiling and/or rate of this entry is incorrect based on your expectations.

Resolution

Table T5UTX is updated by the Sync Tool (report RPUPAYUS_SYNC_TAX_DATA).

When this report is run, the BSI data is collected and written on table T5UTX. On this process, no data is expected to be changed by SAP.

So, the first thing to be checked is the rate and ceiling kept on the BSI Portal.

For that, kindly access your BSI Portal, then go to "User Data Queries > Wage Limits By Authority And Tax Type" and search for the specific Tax Type.

  

Example:

 By checking this information, there are two possibilities:

   

1) The rate and/or ceiling is incorrect on the BSI Portal

On this case, the information should be updated on the BSI Portal first. After that, the Sync Tool needs to be re-run and the information should be corrected in table T5UTX.

If you encounter this situation, kindly open a case to PY-US-BSI component, attaching a screenshot of the above information taken from the BSI Portal. SAP Support Team will engage BSI team to review the information from the BSI Portal.

  

2) The rate and/or ceiling is correct on the BSI Portal

On this situation, it can be something with the Sync Tool execution.

Firstly, please run it in productive mode and check the log, looking for any error raised. If there’s an error, it should be the reason why T5UTX has not been updated and, therefore, has an entry with a wrong rate/ceiling.

If you find an error and don’t know how to solve it (or don’t see any error raised), please report a case to PY-US-BSI component, attaching a screenshot from the BSI Portal and the error received (if that's the case). SAP Support Team will troubleshoot the scenario.

 

Important: Table T5UTX is updated with date from last year, current year and next year. Entries outside this range will not be updated.

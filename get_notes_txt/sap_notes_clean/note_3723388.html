Symptom

Supported item category has been assigned to contract document type other than MK via T-code OLME -> Contract -> Define document Types for Contract -> Choose contract type (other than MK), then maintain the relevant item categories. 
However, when creating a new contract with this document type via Manage Purchase Contracts app, the assigned item category does not show in the drop list.

- The Fiori app 'Manage Purchase contracts' does not display the category 'W- Material group' when creating a Purchase contract, although this option is visible in the GUI app. 
- The issue impacts the ability to create contracts for Material groups where Plant or Service cost is not known in advance.

Environment

  • Materials Management (MM)
  • SAP S/4HANA
  • SAP FIORI

Reproducing the Issue

  1. T-code OLME -> Contract -> Define document Types for Contract -> Choose contract type (other than MK), then maintain the relevant item categories ('Standard' and 'Material Group', or subcontracting and consignment as of S/4HANA 2022) 
  2. Create a contract in Manage Purchase Contracts app with the document type used in step 1. 
  3. Select item category but the expected category does not show.

Cause

The current design is that when creating contract in Manage Purchase Contracts app, if there is no information filled on item level, system will show item categories assigned to document type MK as default item categories. Then, once any information is filled on item level, for example short text, system will redetermine the item categories. 

Resolution

As a workaround, fill item short text in header screen first, then system will redetermine item category based on document type.

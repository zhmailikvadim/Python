Symptom

In the Delivery Sequence Group configuration, when a shipping storage location that is EWM‑managed is selected, the system automatically determines and populates the corresponding EWM warehouse number. However, if the user subsequently changes the shipping storage location to an IM‑managed storage location, the EWM warehouse number is expected to be cleared automatically.

In the standard system, this clearing did not occur, and the previously determined EWM warehouse number remained populated even after selecting an IM‑managed storage location. This resulted in inconsistent configuration data within the Delivery Sequence Group.

Other Terms

NJIT, Next Generation Just-In Time processing, JIT, Delivery Sequence Group, EWM Warehouse, Storage Location

Reason and Prerequisites

Program Error

Solution

Implement the correction instructions provided in this SAP Note or import the relevant Support Package to activate the corrected behavior.

With this update, the system automatically clears the EWM warehouse number when the shipping storage location is changed from an EWM‑managed location to an IM‑managed location in the Delivery Sequence Group. This ensures that the configuration data remains consistent and reflects the correct warehouse‑management method for the selected shipping storage location.

Symptom

We are unable to perform mass changes of prices on purchase contracts using the app "Mass Changes to Purchase Contracts F2669" through the offline method (Excel file upload).

After downloading the contract file, updating the new price values in the Excel file, and uploading it to run the job, the job log shows successful processing with no errors. However, the new price is not updated at the line item and condition level in the contract.

Environment

  • S/4HANA
  • SAP Fiori

Reproducing the Issue

  1. Download the file for a specific contract and save it locally.
  2. Update new price values in the Excel file under the tabs "Purchase Contract Item" and "Purchase Contract Item Pricing".
  3. Upload the price on excel tab 'Purchase Contract Item Price'
  4. Verify the job log, which shows "Quantity Contract [contract number] changed" and "Processing of Excel completed" without errors.
  5. Check the contract; observe that the new price is not updated at the line item and condition level.

Cause

Customization. 

Resolution

To allow changes to condition records, update the following configuration:

  1. Navigate to SAP Customizing Implementation Guide -> Materials Management -> Purchasing -> Excel-Based Data Exchange -> Define Settings for Excel-Based Data Exchange.
    • Execute the configuration. Select the document 'EXL_OPS_CTR' and click on the worksheet.
    • Select the worksheet 'WS_3' and click on 'Columns'.
    • Locate the field 'CONDITIONVALIDITYENDDATE' and remove the 'Read-only' flag.
    • Save it.

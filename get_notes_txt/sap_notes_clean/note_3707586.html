Symptom

When creating a Purchase Requisition (PR) in the Fiori Manage Purchase Requisition - Professional app, the values for the fields 'Street 2' and 'Street 3' are not saved in the 'Delivery Address' tab.

However, in t-code ME51N, the fields 'Street 2' and 'Street 3' are correctly updated.

Environment

  • Materials Management (MM)
  • SAP Fiori 
  • SAP S/4HANA
  • SAP S/4HANA Cloud Private Edition

Reproducing the Issue

  1. Access the Manage Purchase Requisition Professional app
  2. Click on the 'Create' button and create an item
  3. Navigate to the "Delivery Address" tab
  4. Select the address type as "User Entry"
  5. Enter values for the fields "Street 2" and "Street 3"
  6. Save the purchase requisition
  7. Observe that the data entered in "Street 2" and "Street 3" is not retained.

Cause

System is working as designed. The fields "Street 2" and "Street 3" are not available in the Manage Purchase Requisitions - Professional app. 

Resolution

Use the Create Purchase Requisition - Advanced app to save the values of the fields 'Street 2' and 'Street 3'.

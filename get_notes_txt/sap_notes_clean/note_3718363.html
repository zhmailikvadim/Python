Symptom

  • Queue stuck with error "Goods issue has already been posted for delivery".
  • Error message displayed: "Message No. VL602".

Environment

  • SAP S/4HANA
  • SAP ERP Central Component (ECC)
  • SAP Service Parts Management (LO-SPM)
  • Logistics Execution (LE)
  • SAP Extended Warehouse Management (SCM-EWM)

Reproducing the Issue

  1. Open transaction SMQ2.
  2. Execute the transaction.
  3. Search for the queue name using Ctrl + F.
  4. Double-click on the queue name twice.

Cause

Inconsistent status caused by manual execution of VL_COMPLETE.

Resolution

  1. Reverse the changes performed by VL_COMPLETE by executing report RVDELSTA for the delivery without the simulation flag.
  2. Verify that the Goods Movement (GM) status has been reversed.
  3. Go to SMQ2, select the queue, validate the data and try to process the Queue.
  4. Check the GM Status at the Delivery Level.

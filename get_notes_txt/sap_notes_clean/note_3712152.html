Symptom

Canceling a fixed asset document does not automatically reconcile related G/L accounts; both the original and reversal transactions appear in the Account Balance window when checkbox Display Unreconciled Transactions Only is selected.

Environment

SAP Business One

Reproducing the Issue

Prerequisites:

Working with Fixed Assets.

Scenario:

  1. Navigate to Financials → Fixed Assets → Capitalization to create a fixed asset capitalization.
  2. Add the Capitalization document.
  3. Open the added capitalization, right-click, and select Cancel to reverse it on the original posting date.
  4. Open the original capitalization entry, then navigate to the G/L account and access the Account Balance window.
  5. Select Display Unreconciled Trans. Only checkbox to filter the transactions in Account Balance window.

Expected behavior:

Both the original and its reversal journal entry row are automatically reconciled upon document cancellation and are not listed as unreconciled in the Account Balance window. 

Actual behavior:

Both the original and its reversal journal entry row are not automatically reconciled upon document cancellation and are listed as unreconciled in the Account Balance window.

Resolution

This is standard behavior for fixed asset transactions involving G/L accounts, as automatic reconciliation is not performed for G/L accounts associated with both the original document and its cancellation.

Currently, G/L account rows are not automatically reconciled by the system.

Similar working is noted for the below fixed assets documents and their cancellation:

  • Capitalization 
  • Capitalization Credit Memo 
  • Manual Depreciation 
  • Depreciation Run

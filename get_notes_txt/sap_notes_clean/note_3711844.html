Symptom

The Auto Renewal process will be set up for main- and subitems in subscription contract. Once the auto renewal period is reached and the auto renewal action will be exceuted with report RSPPFPROCESS there is an error message CRM_CANCEL(436) with text 'Cancellation request must be after the Activation time period' getting raised.

Other Terms

RSPPFPROCESS; ZZCONTRACT_AUTO_RENEWAL; AR_EXTENSION_AR_PERIOD

Reason and Prerequisites

The issue is only occuring in case the subitem will be processed before the main item in the auto renewal action and the date profiles are the same in main- and subitem. However as this could happen and the AR_EXTENSION duration will not be inherited from main- to subitem this duration needs to be fetched from main item once the subitem is processed.

Solution

Correction Instruction.

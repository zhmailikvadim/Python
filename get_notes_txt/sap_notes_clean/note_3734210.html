Symptom

A connection test for DRC Cloud Edition RFC in SM59 returns following error:
*** ERROR => SSL handshake with xxx:443 failed: SSSLRC_CONN_CLOSED (-10) was raised

Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.

Environment

  • SAP ERP
  • SAP S/4HANA
  • SAP S/4HANA Cloud Private Edition
  • SAP Document and Reporting Compliance, Cloud Edition

Reproducing the Issue

  1. Launch SM59
  2. Go inside RFC destination used for DRC Cloud Edition
  3. Perform a connectivity test

Cause

SSL handshake failure may occur due to missing or disabled TLS SNI configuration in the system profile parameters.

Resolution

  1. Check if the TLS extension was enabled, as follows:
    • Access the Maintain Profile Parameters (RZ11) transaction.
    • In the Parameters Name field, enter the following: icm/HTTPS/client_sni_enabled
    • Choose Display.
    • Under the Value of Profile Parameter icm/HTTPS/client_sni_enabled section, the following sources must have TRUE as the Result Value:
      • Default profile.
      • Instance profile.
      • Current profile.

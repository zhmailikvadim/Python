Symptom

When you run the DIAN350 Withholding Tax Return report for Colombia using the "Run Statutory Reports" app, the report generation completes but shows no data in the output. You have verified that there are relevant withholding tax documents for the selected company code and reporting period in tables like WITH_ITEM or P_STRPWHLDGTAXITEMLOG, but these documents are not being selected by the report.

Environment

SAP S/4HANA

Reproducing the Issue

  1. Open Run Statutory Reports and select the DIAN350 report for Colombia.
  2. Select the relevant reporting entity and period.
  3. Start the run and review the Legal Reporting/Data section: it shows “No data,” although withholding documents exist.

Cause

The root cause of this issue is missing configuration for the withholding tax codes. The DIAN350 report’s logic requires an Income Type to be assigned to the relevant withholding tax codes. If this "Income Type" is not maintained, the system’s selection logic will exclude the corresponding documents from the report, resulting in an empty output.

Resolution

  1. Review the Colombia withholding tax report configuration for DIAN350 in Customizing/ACR settings.
  2. Maintain the income type assignment for all withholding tax codes used by the relevant documents, according to the official implementation guidance.
  3. Save the configuration and ensure that it transports to the target client/system if required.
  4. Start a new DIAN350 run for the same reporting period.
  5. Verify that entries are now written to ACCT_SRF_REP_ITM and that the Legal Reporting section displays the expected documents.

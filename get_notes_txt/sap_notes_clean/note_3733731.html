Symptom

When I change a record in my custom Z-table, it fails to trigger the creation of a new data record in the change document table (ARBCIG_CHANGEDOC). This prevents the incremental data load from my SAP ERP Central Component (ECC) to SAP Ariba Buying and Invoicing (B&I) or SAP Ariba Sourcing (SRC) via the SAP Integration Suite, Managed Gateway for Spend&Network.

Environment

  • Managed Gateway for Spend&Network
  • SAP Ariba Buying and Invoicing (B&I)
  • SAP Ariba Sourcing (SRC)
  • SAP ERP Central Component (ECC)

Cause

You are experiencing this issue because the ARBCIG_CHANGEDOC table and the incremental load functionality within the SAP Integration Suite, Managed Gateway for Spend&Network add-on only work with standard SAP objects. Standard SAP mechanisms, such as change pointers you configure via transaction SWEC for specific business objects (for example, ASSET, COSTCENTER, VENDOR), manage the triggers for these change events. The standard product functionality does not support creating records in the ARBCIG_CHANGEDOC table based on changes to a custom Z-table.

Resolution

  1. To trigger change document creation from a custom Z-table, you must create a custom development.
  2. Implement a custom Business Add-In (BAdI) or another enhancement to capture the change event from your Z-table and then programmatically populate the required change document tables.

Since this involves custom development and solution design, it falls outside the scope of standard product support. You should engage with SAP Professional Services or a certified implementation partner to assist with designing and building this custom solution.

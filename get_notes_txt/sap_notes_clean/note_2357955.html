Symptom

  • In the Upgrade Management Tool (UMT), the 'Apply Time Filter' option is not applied for personal content such as that in user inboxes or personal folders - it is still shown on the Summary screen

Environment

  • SAP BusinessObjects Business Intelligence Platform 4.2 (BI 4.2)

Reproducing the Issue

  1. Open the UMT
  2. Enter an SAP BusinessObjects Enterprise XI 3.x system
  3. Enter an SAP BusinessObjects Business Intelligence Platform 4.2 system
  4. Click 'Next'
  5. Select 'Apply Time Filter'
  6. Select the required time span
  7. Click 'Next'

Content from the public folders is filtered correctly but personal content from outside the apply time filter is shown in the Summary page.

Cause

The personal content is calculated as a dependency of the users however content in the public folders is chosen explicitly on the object selection page

Resolution

This is expected behaviour. 

Only content which is selected on the Object Selection page has the time filter applied.  Anything that is calculated as a dependency does not.
Content from the Public Folder is automatically brought across when its parent folder is selected and will therefore be filtered as expected.  Personal content is brought across as a dependency of the user and therefore does not.

From the 'Help' in the UMT tool:

The filter selected in the Select Filter page is applicable only for selecting the objects in the Object Selection page. However, this filter selection is not applicable for dependency calculation in the Summary page.

For example, perform the following:

  1. In Select Filter page, select the object type as WebI report.
  2. In the Object Selection page, select the WebI report, and choose the Export this object and its dependencies.

Irrespective of the filter that you choose in the Select Filter page, the Summary page displays the universe and connection along with the WebI report. This is due to the dependency rule.

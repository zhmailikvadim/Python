Symptom

Service Call Scheduling related data cannot be retrieved using Service Layer or DI API.

Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.

Environment

SAP Business One

Reproducing the Issue

  • Use the following HTTP request to retrieve the Service Call Scheduling related data using Service Layer:

    GET /b1s/v1/ServiceCalls(20)

OR

  • Use the following DI API code to retrieve the Service Call Scheduling related data:

    ServiceCalls oServiceCall = (ServiceCalls)oCompany.GetBusinessObject(BoObjectTypes.oServiceCalls);
    oCompany.XmlExportType = BoXmlExportTypes.xet_ExportImportMode;
    oServiceCall.GetByKey(20);
    oServiceCall.SaveXML(@"E:\TESTServiceCall.xml");

Expected behavior:

Service Call Scheduling related data is successfully retrieved.

Actual behavior:

Service Call Scheduling related data is not retrieved as ServiceCallSchedulings collection/object is returned empty.

Resolution

To retrieve the Service Call Scheduling related data, kindly make sure the following:

  1. The option Enable Scheduling of Multiple Tickets is selected on the Per Document tab of Document Settings form for Service Call (follow path Administration → System Initialization → Document Settings → Per Document tab → select Service Call from the Document drop-down).
  2. Define a Scheduling for the Service Call.

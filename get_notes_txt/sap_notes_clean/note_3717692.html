Symptom

In order to set an item as 'Completed' in a service order, a service confirmation, marked as 'Final Confirmation', is created. For the service order item to be marked as completed, its final service confirmation also needs to be in status 'Completed'. However, it's not possible to complete the confirmation due to error:

  • "Item &1: Requested quantity &2 exceeds the available quantity 0.000."

The item in a service order is of type non-stock, so it has to be procured. Before trying to create this final service confirmation, the following actions were taken:

  • Cancellation of already created service confirmations
  • Revocation of approvals
  • Deletion of service entry sheets
  • Recreation of service entry sheets

Environment

SAP S/4HANA Cloud Public Edition

Cause

The error message indicates that the available quantity for this service item has been exceeded. This is not related to stock, but rather to the lifecycle of the service order item.

The sequence of cancelling confirmations, revoking approvals, and modifying service entry sheets has left the original item with zero remaining quantity for confirmation. Therefore, the system correctly keeps the item in 'Open'/'Released' status and prevents completion due to the validation error.

Ideally, the 'Final Confirmation' indicator is set only once to close the loop on a service order item. It cannot be applied multiple times to different confirmations.

Resolution

Since the current item quantity is consumed, it's recommended to create a new item in the service order for the concerned products. Then, it's possible to process the Service Entry Sheet and Confirmation against this new item.

Alternatively, it's advised to recreate the entire document chain.

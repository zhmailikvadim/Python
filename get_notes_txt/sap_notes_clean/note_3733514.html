Symptom

  • Central Management Server (CMS) commit speed for Crystal Reports is bottlenecked / limited at around 8 commits a second, when using 10 parallel threads continuously comitting
  • This issue may be seen when scheduling a large number of individual instances
  • This issue only happens in environments (Eg BI Platform or CR server) containing an OEM license
  • When using a non-OEM license in the same environment, the commit speed was 110 commits a second

Reason and Prerequisites

Environment

  • SAP BusinessObjects Business Intelligence (BI) Platform 4.3 / 2025
  • Crystal Server 2020 / 2025
  • Windows
  • Linux / Unix 

Steps to Reproduce

  1. Using an SDK that commits Crystal Reports (eg using Infostore.Schedule function in java SDK) in parallel, eg 500 schedules by 10 threads
  2. Observe the commit speed gets limited to around 8 commits a second

Cause 

The presence of an OEM license introduces additional checks on the license, which increases contention and slows down the overall commit speed.

To determine whether an OEM license is used, the BI Support Tool can be used:
2138275 - How to generate a Landscape Analysis Report for SAP Support using SAP BI Platform Support Tool version 2.x

The license key report outputs the flag contained on the license, the specific flag relevant is:
SI_OEM_KEYCODE_PRESENT 1

If this flag is not '1', then this issue will not affect the environment

Solution

The issue has been forwarded to development for further investigation

Symptom

The MESSAGE statement as well as the kernel part extracting information for ABAP shortdumps use a kernel function that reads the attribute values of the last ABAP exception raised. This kernel function writes beyond the memory area provided by its callers if the respective attribute value is longer than that memory area. This destroys the memory object located behind that memory area, which might cause the ABAP processor to fail unexpectedly later on. The issue is known to cause SYSTEM_CORE_DUMPED shortdumps, but might also result in yet unknown errors that are difficult to analyze.

Solution

Install a kernel that contains the correction identified by patch label "Extracting exception attributes destroys memory".

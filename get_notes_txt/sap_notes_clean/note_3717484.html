Symptom

SAP program /SAPPSSRM/CLOSEOUT does not display backend errors. When running the program on SRM side to close Purchase Orders, success message is displayed on the SRM side, indicating that the Purchase Order has been successfully closed, while the document ends up in an error status on the S/4 side. The error is visible in transaction SLG1 but not displayed in /SAPPSSRM/CLOSEOUT, resulting in an inconsistent status information between SRM and S/4.

Environment

  • Procurement for Public Sector

Reproducing the Issue

  1. Execute the closing process using SAP program /SAPPSSRM/CLOSEOUT.
  2. Observe that the program performs several checks on the SRM side and returns an "OK" message indicating successful closure.
  3. Verify the backend processing in S/4, where an error is raised during validations performed by the function module B46B_DPO_TRANSFER.
  4. Check transaction SLG1 to view the error, which is not displayed in /SAPPSSRM/CLOSEOUT.

Cause

SAP program /SAPPSSRM/CLOSEOUT is designed to run locally. The logic is to call the local Function 'BBP_PD_PO_ITEM_GETDETAIL' and read documents, check them in SRM and make closing activities where possible.

The end of that process is to call the Function BBP_PD_PO_UPDATE. That is also a local Function.
So everything is done locally.

The Function B46B_DPO_TRANSFER is only called asynchronously, not in the frame of the program. 

Resolution

Missing functionality. In order to solve the issue, there should be a simulation with the help of the transfer Function B46B_DPO_TRANSFER calling the ECC BAPI BAPI_PO_CHANGE in test mode.
We do not have the above built-in in the current design of the program.

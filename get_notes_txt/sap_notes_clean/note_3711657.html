Symptom

The SAP Note contains the following two new enhancements:

1. New BAdI BADI_FSTB_CAS_PA_CANCELLATION (interface IF_FSTB_CAS_PA_CANCELLATION, enhancement spot ES_FSTB_CAS_PA_PROCESSING):

The BAdI is called during the cancellation of payment arrangements by mass run Cancel Payment Arrangements (report RFSTB_CAS_CANCEL_PAYM_ARRGMT) after receiving the data of the payment arrangements to be cancelled for a switch-out request. It identifies the standing orders and the direct debit mandates from the switch-from account. Forward orders and payment templates are not cancelled.

The following methods are available:

  • DETERMINE_STANDING_ORDER: Determines which standing order from the switch-from account is to be cancelled.
  • DETERMINE_DD_MAND: Determines which direct debit mandate from the switch-from account is to be cancelled.

The fallback class CL_FSTB_CAS_PA_CANCELLATION is available which identifies payment arrangements to be cancelled, as follows:

  • Standing order: From all standing orders of the switch-from account, the standing order is identified by the recipient account (bank key and account number) and the reference number of creation.
  • Direct debit mandate: The direct debit mandate is identified by the payer account (switch-from account), the mandate ID and the payee ID.

2. Include the reference number of creation to identidy a standing order to be cancelled from switch-from account:

After receiving the data of the payment arrangements to be cancelled for a switch-out request, the mass run Cancel Payment Arrangements (report RFSTB_CAS_CANCEL_PAYM_ARRGMT) identifies and cancels the standing orders based on the recipient account details. If multiple standing orders exist for the same recipient, an error is thrown.

With this SAP Note, the determination logic is enhanced so that, in addition to the recipient bank details, the reference number of creation (data element BCA_DTE_SO_REFNO_CREATE) is also considered to accurately identify a standing order that needs to be cancelled. If multiple standing orders exist for the same recipient account and for the same reference number of creation, an error is thrown.

The new logic is included in the method DETERMINE_STANDING_ORDER of fallback class CL_FSTB_CAS_PA_CANCELLATION.

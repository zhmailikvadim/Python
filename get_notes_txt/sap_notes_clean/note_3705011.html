Symptom

  • An error occurs when attempting to store data in the form of blobs in new Azure Storage Accounts due to enhanced network security requirements.
  • Connections to the storage accounts need to be whitelisted, but the necessary IPs/Service Tags for whitelisting are unclear.
  • The issue is critical as the affected systems are essential to business operations.
  • The error message observed when pointing to the new storage is: "com.microsoft.azure.storage.request: This request is not authorized to perform this operation."
  • The issue occurs when moving to a new infrastructure with enhanced network security.

Environment

  • SAP Customer Data Cloud
  • Dataflow

Reproducing the Issue

  1. Create a test Dataflow that reads from the Azure Storage.
  2. Point the test Dataflow to the older Azure Storage and observe a Success status with the message: "Blob downloaded successfully. blobName = text.json.. blobPrefix = ."
  3. Point the test Dataflow to the new Azure Storage and observe the error message: "com.microsoft.azure.storage.request: This request is not authorized to perform this operation."

Cause

Enhanced network security requirements in the new infrastructure necessitate whitelisting connections to the storage accounts.

Resolution

  • Create a new dataflow with the "datasource.write.azure.component" as the first dataflow step and use the same credentials.
  • This will allow verification of whether the connection succeeds or if the same error occurs.
  • If the connection succeeds, it indicates that the credentials have write access but may lack the necessary permissions for read actions.
  • Review Azure credentials and external resources:
  • Refer to: Azure Storage authentication and access control.
  • Refer to: Troubleshooting Azure Storage access issues.
  • Allowlist the necessary IP addresses from the following list:
  • Refer to: Allowlisting SAP Customer Data Cloud IP Addresses | SAP Help Portal.

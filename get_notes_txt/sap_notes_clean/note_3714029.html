Symptom

  • Running the data extract pipeline from the API results in failure.
  • Whereas running the same pipeline from the UI completes successfully without errors.
  • Error message observed in console logs: "Error: dataExtractsFileType cannot be null. Please provide a valid fileType!"

Environment

  • SAP Sales Performance Management
    • Incentive Management on Hyperscalers

Reproducing the Issue

  1. Set up the service account for API runs.
  2. Trigger the data extract pipeline without including the filetype reference in the API call.
  3. Observe that the pipeline is triggered but fails immediately.

Cause

  • Invalid API call: The API call does not include the required filetype reference or the Tags used were invalid in the request body.

Resolution

  1. Review the API call payload and ensure the correct tag for referencing the filetype is used.
  2. Replace 'runDataExtractsFileType' with 'dataExtractsFileType' in the API body while referencing the Filetype for the DataExtracts stage Pipeline.
  3. Refer to the API accelerator documentation for proper tag usage: Incentive Management APIs (Kubernetes)
  4. Retry the API call with the updated payload including the correct filetype reference.

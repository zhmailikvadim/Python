Documentation

All guides for SAP Enterprise Threat Detection are available at http://help.sap.com/sapetd.

SAP Enterprise Threat Detection 2.0 SP07 PL06 provides corrections for the following issues:

Forensic Lab UI

  • The fields Privilege Name, Privilege Type and Privilege Grantee Type were sometimes not filled when transferring User Change Log from ABAP Log Extractor (New).

Streaming

  • Payloads sent to the Log Collector by the ABAP Log Extractor (New) were not properly parsed in some cases. This could cause several issues like wrong timestamps being parsed.
  • One or several Log Collector queues remained full for a long time, which meant that Log Collector couldn't send data to the remote endpoint fast enough. Now the Log Collector's HTTPSender supports multithreading, which improves performance.
  • A new configuration in the Normalizer Processing Settings allows to save original logs (sent from new ABAP Log Provider) in a short form. This allows to reduce memory consumption of Original Logs in Kafka, Hot and Warm storages. Example configuration:

  •        <UseShortFormatForOriginalLogs>

                                                    <Enabled>true</Enabled>

                                                    <LogType>BusinessTransactionLog</LogType>

                                                    <LogType>...</LogType>

            </UseShortFormatForOriginalLogs>

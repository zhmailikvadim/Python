Symptom

During JIT Delivery Confirmation processing, the system creates consignment issue orders on an item-by-item basis. If the sales scheduling agreement for one of the items is locked, the system processes only the remaining items and creates consignment issue orders for those unlocked items. The items linked to the locked sales scheduling agreement are skipped.

After unlocking and reprocessing the JIT Delivery Confirmation, the system attempts to create orders for all items again. Because the EDI process supports only creation and not updates, items that were already processed earlier result in duplicate consignment issue orders.

Other Terms

NJIT, Next Generation Just-In-Time Processing, JIT Delivery Confirmation, Self Billing, Consignment issue order

Reason and Prerequisites

Program error

Solution

Implement the correction instructions provided in this SAP Note or import the relevant Support Package to enable the corrected system behavior.

With this update, JIT Delivery Confirmation processing is handled using an all‑or‑none approach within a single LUW. If any item in the confirmation fails due to a locked sales scheduling agreement, the system does not process any items. This prevents partial processing and eliminates the possibility of creating duplicate consignment issue orders. After resolving the issue (by unlocking the sales scheduling agreement), the customer can reprocess the entire confirmation safely.

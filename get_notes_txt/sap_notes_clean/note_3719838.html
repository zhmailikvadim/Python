Symptom

With the introduction of the new Harmonized Planing Area (I_SAPIBP2), SAP IBP adapted the integration strategy using the following technologies:

  • Real-Time Integration (RTI) for the integration of master and order data from SAP ECC or S/4HANA (on-premise and private cloud)
  • SAP Cloud Integration (CI) for the integration of additional aggregated data from SAP ECC or S/4HANA (on-premise and private cloud), and flexible integration with other SAP, legacy and 3rd-party systems

The above integration technologies are strongly recommended.

Solution

In April 2026, new IBP license SKUs will be introduced following and extending SAP IBP's integration strategy: The new license SKUs will not include the entitlement to use SAP Cloud Integration for data services (CI-DS) and the CI-DS infrastructure will not be provided to new customers. A separate CI license is needed, as the CI entitlement is not included in the new IBP license SKUs.

The effects on customers are outlined below:

For new customers

  • CI-DS-based integration can't be used for new customers
  • CI-based integration can be used to integrate aggregated/flexible data (incl. master data) into all IBP planning areas (SAPIBP1, I_SAPIBP2, SAP4, SAP7F,...) connecting SAP ECC or S/4HANA (on-premise and private cloud), other SAP, legacy and 3rd-party systems
  • RTI-based integration can be used to integrate master data and order data into I_SAPIBP2 and SAP7F connecting SAP ECC or S/4HANA (on-premise and private cloud) systems
  • The combination of RTI and CI is strongly recommended for the integration with the Harmonized Planning Area (I_SAPIBP2)

For existing customers

  • There are no plans to sunset/discontinue the integration using CI-DS
  • CI-DS will move to maintenance mode and will be supported with bug fixes and security related enhancements only. No new CI-DS features will be delivered in the future
  • CI-DS may still be used after the migration to the Harmonized Planning Area. Adaptations will be needed
  • CI-DS migration to the BTP-Kyma runtime environment is needed. More information: https://help.sap.com/docs/cloud-integration-for-data-services/tenant-migration-guide

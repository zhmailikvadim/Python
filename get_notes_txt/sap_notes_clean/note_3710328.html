Symptom

  • Error message encountered: "Could not read any integration flow properties with URL".
  • Issue occurs when setting up the "Check Configuration for Integration of SAP S4HANA with SAP Field Service Management" artifact within the integration from S/4HANA to FSM.

Environment

SAP Field Service Management 1.0

Reproducing the Issue

  1. Set up the "Check Configuration for Integration of SAP S4HANA with SAP Field Service Management" artifact within the integration from S/4HANA to FSM.
  2. Navigate to Monitor > Monitor Message Processing > All Integration Flows.
  3. Choose Configuration Check Result under attachments for the integration flow.
  4. Observe the error message: "Could not read any integration flow properties with URL".

Cause

  • The Cloud Integration host URL and tenant management URL are not the same in the subaccount.
  • The integration flow uses standard Cloud Integration OData APIs, which require a valid Cloud Integration tenant management URL.

Resolution

  1. Verify that the Cloud Integration host URL and tenant management URL are correctly configured in the subaccount.
  2. Ensure the Cloud Integration Management URL follows the correct structure in Cloud Foundry: `https://.it-<...>.cfapps.sap.hana.ondemand.com`.
  3. Additionally, you can find the valid Cloud Integration management URL when you create the service keys on top of Process Runtime Integration instance with API plan in your subaccount.
  4. Replace the incorrect Cloud Integration Host parameter value with the correct Cloud Integration Management URL in the integration flow.
  5. Redeploy the integration flow after updating the URL.

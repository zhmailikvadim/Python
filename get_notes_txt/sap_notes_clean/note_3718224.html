Symptom

  • SAP HANA log volume reaching 95% full, potentially leading to system instability.
  • Approximately 1300 log segments in truncated state.
  • Error messages observed in backup logs:
  • "BackupExe_LogBackup.cpp(xxx) : Report backup failure: exception 1: no.xxx (Basis/Structures/StructureEnumHandler.hpp:114) TID: xxx"
  • "String '30' doesn't match enum type."

Environment

SAP HANA PLATFORM EDITION 2.0

Reproducing the Issue

  1. Observe the SAP HANA log volume reaching 95% full.
  2. Check the status of log segments using the following SQL command:

```sql  

SELECT b.host, b.service_name, a.state, COUNT(*)  

FROM "PUBLIC"."M_LOG_SEGMENTS" a  

JOIN "PUBLIC"."M_SERVICES" b ON (a.host = b.host AND a.port = b.port)  

GROUP BY b.host, b.service_name, a.state;  

```  

  1. Verify the log segments are in a truncated state.
  2. Review the backup logs for errors indicating "String '30' doesn't match enum type."

Cause

  • The parameter `log_backup_interval_mode` is incorrectly configured with the value '30'. This prevents log segment backups from being triggered, causing log segments to accumulate in a truncated state.

Resolution

  1. Change the parameter `log_backup_interval_mode` to its default value 'immediate'.
  2. No system restart is required for the changes to take effect.
  3. Refer to the SAP HANA documentation for further details on the `log_backup_interval_mode` parameter:

SAP HANA Documentation - log_backup_interval_mode.  

4.Verify the configuration changes and ensure log backups are triggered correctly.

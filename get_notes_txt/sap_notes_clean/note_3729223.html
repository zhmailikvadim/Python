Symptom

When submitting an eDocument from "India GST Reporting Node" to DCS, there is an error "Process could not be determined".

Environment

  • SAP ERP
  • SAP S/4HANA
  • SAP S/4HANA Cloud, Private Edition
  • SAP Digital Compliance Service for India

Reproducing the Issue

  • Execute EDOC_COCKPIT
  • Navigate to node "India GST Reporting"
  • Select the eDocument
  • Click on Submit

Cause

Incorrect Posting.

Resolution

Relevant section was not determined ending in process could not be determined error. In standard, the process determination will happen in the class CL_EDOC_MAP_IN_GSTR2, method GET_GSTR2_FI_PROCESS (for incoming side) and CL_EDOC_MAP_IN_GSTR1, method GET_GSTR1_FI_PROCESS (for outgoing invoice).

For outgoing, the process indicators are:

0 - B2B Invoices
1 - B2C Invoices
2 - Credit/Debit Notes
3 - Export Invoices
4 - Advance Tax/Down Payment docs
5 - Tax Paid

For incoming, the process indicators are:

0 - B2B Invoices
1 - B2BUR Invoices
2 - Credit/Debit Notes
3 - Import of Goods
4 - Import of Service
5 - Advance Tax/Down Payment docs
6 - Tax Paid
7 - GST Journal Voucher
8 - ISD Invoices 

The process is determined based on the BSEG table. Every indicator will be set after checking certain parameters. If the determination is not happening properly, this means that the issue is with the posting. With the help of an ABAPer, analyze the above code and take an action.

If from the standard process the required process is not determined, make use of the BAdI EDOC_IN_MAP for implementing custom logic to tweak the process determination and change C_PROCESS as per business requirement. For already posted documents, BAdI can be used as mentioned above. For new postings, check with the posting team or raise a case in FI-LOC-SD-IN or FI-LOC-MM-IN with the findings from the ABAPer.

Symptom

  • Additional items were added into an inbound delivery.
  • This addition items have no reference to the purchase order.
  • With EHP7, LIPS-KZBEW = 'blank' in the additional line items with movement type 511.
  • After the upgrade to EHP8, LIPS-KZBEW = 'L' in the additional line items with movement type 511.  
  • The following error message popped up during the post goods receipt for the inbound delivery "M7226 - Update control of movement type is incorrect (entry 511 X X _ L)".

Environment

  • Sales and Distribution (SD)
  • Logistics Execution (LE)
  • SAP R/3 Enterprise 
  • SAP ERP Central Component
  • SAP ERP
  • SAP enhancement package for SAP ERP

Reproducing the Issue

  1. Execute VL32N.
  2. Add additional line items into an inbound delivery.
  3. Save the inbound delivery.
  4. Observe LIPS-KZBEW is different between EHP7 and EHP8.
  5. Post GR via VL32N.

Cause

  • The SAP note responsible for the difference between the two releases is SAP Note 0002530413 - error message M7226 when posting GR additional delivery item. This note is the one that has changed the creation these additional items in the delivery from LIPS-KZBEW empty to "L".
  • The direct use of Inventory Management Movement Types such as 511 has never been allowed in VL32N. The presence of this movement type in an inbound delivery is only supported when the posting is made in MIGO and, because there is WM involvement, the system creates a delivery and not a material document. In that case the delivery is created with LIPS-KZBEW empty and the reception can be made. But adding an item in VL32N with a movement type such as 511 is normally not supported. It was only allowed because there was a program error. When the program error was corrected with the SAP Note 0002530413 - error message M7226 when posting GR additional delivery item, the use of the movement type 511 was no longer possible. 

Resolution

Implement SAP Note - 3647012 - VL32N : error message L9029

Keywords

LIPS-KZBEW, KZBEW, 511, T156SC, M7226, M7 226, T156X, update control, movement type

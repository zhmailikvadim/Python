Symptom

  • An issue occurs with EAN/GTIN configuration in SAP s/4hana cloud private edition.
  • Error /CDBASIS/MSG898 occurs when running /STTPEC/INT_MAT.

Environment

SAP Advanced Track and Trace for Pharmaceuticals

Cause

  • Misunderstanding of ean/gtin behavior: w4es “ean length” controls only the display, while leading zeros are not persisted in ean11.
  • Existing EANs and GTINs can become duplicates when leading zeros are ignored, resulting in the same stored value across categories.

Resolution

  1. Check for and remove duplicate numbers across EAN categories where leading zeros are the only difference, as per standard guidance.
  2. Configure the GTIN number range to 10000000000000 through 99999999999999.
  3. Use the correct EAN/GTIN customizing and do not rely on leading zeros for uniqueness because EAN11 does not store them.
  4. Avoid assigning the same EAN value under multiple EAN categories if this leads to duplicates once leading zeros are ignored.

Note: For more detail, follow the SAP Note - 2716224 (Before Conversion or Upgrade > Check and Remove Duplicate Numbers section).

Symptom

  • No prompt is played when call is reaching IVR
  • The following messages appear at logs:
    • CEM:WRN> FILE_PLAY failed in MRS : {'INFO': 'NO_MRS_SLOT', '_CMD': 'PLAY_FILE', '_EVT': 'FILE_PLAYED', 'FileName': 'C:\\Users\...\\Welcome message.wav', '_DATA': 'ERROR', 'CALL_ID': '3A0B94A5BEBAFE459696F90667260CBD', '_PEER_ID': 'CallDispatcher:XXXX_VU'}
    • CallDispatcher: Cannot handle MRS command, since no slot reserved for MRS gw {540C45FF-88E7-4DBD-A903-0AA985BC13D7}

Environment

SAP Contact Center 7.0

Reproducing the Issue

Perform a call to an IVR number

Cause

  • Missing configuration at Media Routing Server (MRS) or module linkage
  • Not enough MRS slots configured for prompts

Resolution

  1. Ensure that the custom prompt file format is correct as per KBA 2911443
  2. Follow the "Linking Media Routing Server (MRS) to Other Modules" configurations for System Configuration (SC)
  3. Increase "Maximum Number of Simultaneous Prompts" field.
    • Check "SAP Contact Center introductory training - part 2" document (pages 31-35)
  4. Reset Virtual Units where the Media Routing Server "MRS" and  Call Dispatcher "CD" are running
  5. If after the above changes you still get the same errors in the logs, then check the RTP Packet Length variables
    1. Connect to the Infrastructure Administrator
    2. Connect to the desired HAC instance
    3. Navigate to the Virtual Unit (VU) where the Call Dispatcher is installed (do this from the Virtual Unit Level)
    4. Check the Variables for Call Dispatcher block and ensure the RTP Packet Length in MS is set correctly
      • The default value should be 20 or 30 ms
      • You need to ensure this matches the same value defined at your Gateway
      • Check the "SAP Contact Center Introductory Training IA" for more

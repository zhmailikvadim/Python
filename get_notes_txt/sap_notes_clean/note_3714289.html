Symptom

When creating an Advanced intercompany Sales Order document, the error message "The CO account assignment object belongs to company code XXXX, not YYYY" is appearing.

Environment

SAP S/4HANA Cloud Public Edition

Reproducing the Issue

  1. Go to Create Sales Orders app.
  2. Select the document type OR
  3. Enter all the necessary information to create the Sales Order
  4. In the field company code, select a company code "x"
  5. In the field "Plant", select an element which belongs to a different company code.
  6. See the error message appears.

Cause

In advanced intercompany sales order processing, the fields "WBS Element" "Plant" and "Company code" must all belong to the same element.

In case the WBS Element and the Company code of the sales order are equal, but the Plant doesn't belong to that same company code, the error message will be triggered.

Resolution

All 3 fields must belong to the same company code, once this is done the error message will not be triggered.

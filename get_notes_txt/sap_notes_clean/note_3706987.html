Symptom

Although the parameter max_trace_file_size is set, the automatic file rotation for backup.log and backint.log does not work. As a result, both files continue to grow indefinitely.

Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental. 

Environment

SAP HANA Platform

Resolution

The parameter must be set in the global.ini of the SYSTEMDB.
Setting the parameter in a tenant database is not sufficient.

The configuration can be verified, for example, via SSH / terminal:

  1. Log on to the HANA system as <sid>adm

  2. Navigate directly to the SYSTEMDB configuration directory by executing:

     
    cdcoc 

  3. Check whether the parameter is set:

     
    grep max * 

The output should list the configured parameter in global.ini, for example:

hs2adm@hanasingle2:/usr/sap/HS2/home> cdcoc

hs2adm@hanasingle2:/usr/sap/HS2/SYS/global/hdb/custom/config> grep max *
global.ini:# 2026-01-20 09:52:22.223717 setting [backup]/max_trace_file_size="100" (was "")
global.ini:max_trace_file_size = 100

To reduce an already oversized backup.log or backint.log file, please follow the instructions in SAP Note 2797078 – How to configure the size of the HANA Backup files backup.log and backint.log.

Symptom

When creating an outbound delivery from a sales order, the route that was manually changed in the sales order is re-determined in the delivery instead of being copied.

Environment

SAP S/4HANA Cloud Public Edition

Reproducing the Issue

  1. Create a sales order.
  2. Manually change the route in the sales order to a different route.
  3. Create the outbound delivery from the sales order and check the route in the delivery.

Cause

In configuration activity "Define New Route Determination By Delivery Type" (101106), the outbound delivery type (for example, LF) is set to one of the bellow:

  • A - New route determination without check
  • B - New route determination with check
  • C - New route determination for inbound deliveries

So the route is re-determined during delivery creation, overwriting the sales order’s manually maintained route.

Resolution

  1. Open configuration activity “New Route Determination By Delivery Type” (101106).
  2. Identify the delivery type used for outbound delivery (for example, LF).
  3. Change the setting to blank (No new route determination).
  4. Save the configuration.
  5. Create a new sales order, adjust the route as needed, and create the outbound delivery to confirm the route is now copied from the sales order and not re-determined.

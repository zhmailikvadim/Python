Symptom

You created a Dynamic Property to either hide or make the "Title" field non-updatable in the Contact screen, but it does not work as expected.

Environment

  • SAP Sales Cloud Version 2
  • SAP Service Cloud Version 2

Reproducing the Issue

  1. Click on the User Icon.
  2. Navigate to Settings > All Settings.
  3. Go to Extensibility Administration.
  4. Select Contact Service > Contact Person.
  5. Open the Dynamic Properties tab.
  6. Create a Dynamic Property rule.
  7. Open the Contact Object Worklist (OWL).
  8. Select and open a Contact.
  9. The "Title" field remains visible or editable despite the Dynamic Property configuration.

Cause

The issue occurs because the wrong technical field name was used for the "Title" field.

  • The correct technical field name for "Title" is "formOfAddress".
  • The "formOfAddress" field was not included in the Dynamic Property rule, causing the "Title" field to remain editable or visible.

Reference: You can verify the field name in the API response:

https://<Tenant URL>/sap/c4c/api/v1/repository-service/uiviews/sap.crm.md.contactpersonservice.uiview.contactDetail

Example JSON response:

{
    "name": "formOfAddress",
    "labelTextId": "sap.crm.common.businessPartnerService.title",
    "label": "Title",
    "binding": "formOfAddress"
}
 

Resolution

For resolving it, add the "formOfAddress" to the Dynamic Property rule. Once configured correctly, the "Title" field will be read-only or hidden based on the defined conditions.

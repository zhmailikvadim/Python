Symptom

  • Button 'Settlement Rule' in app 'Project Control - Enterprise Projects' is missing.
  • This behavior occurs for Project Profile 'Project With Revenue'.
  • This change has been observed since 2602 upgrade.

Environment

  • SAP S/4HANA Cloud Public Edition

Reproducing the Issue

  1. Open the Project Control - Enterprise Projects app.
  2. Create a new project and save it.
  3. Try to add a settlement rule: the Settlement Rule button is not available.

Cause

Change in behavior introduced with release 2602: with the “Profitability Segment Assignment for Projects with Revenue” feature, the Settlement Rule button is disabled and unavailable for projects with revenue.

Resolution

Please refer to What's New document Profitability Segment Assignment for Projects with Revenue (page 772) which describes this change in behavior in 2602.

With this feature, for projects with revenue, you can now assign a profitability segment to the project or to a WBS element in the hierarchy, if either of them has been marked as a billing element.
To add the profitability segment, use the Assign link available for the new Profitability Segment field that has been added to the Control tab.

With this new feature, the Settlement Rule button will be disabled and unavailable for revenue projects.

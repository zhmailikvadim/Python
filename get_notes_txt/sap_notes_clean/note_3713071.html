Symptom

  • The rate component (e.g., percentage) appears lower than the value set in the rate configuration
  • The percentage changes after a Work Order Revision is created
  • The value changes when the worker’s presented amount or pay rate changes

Resolution

The system uses the rate component's Component Limit to ensure that the calculated monetary amount does not exceed the maximum allowed. Even if the rate is configured with a fixed percentage (e.g., 17%),
the system may adjust it if applying that percentage to the worker’s presented amount would exceed the component limit.

Example:

  • Component Limit: 1258.00
  • Presented Amount: 7500
  • Admin Value: 17%

17% of 7500 = 1275, which exceeds the limit. To stay within the limit, the system recalculates:

Adjusted Percentage = (Component Limit / Presented Amount) × 100

In this example: (1258 / 7500) × 100 = 16.77%

This is the value displayed on the Work Order Revision. When the presented amount changes, the percentage is recalculated accordingly, and this is a normal system behavior.

Symptom

When running an integration subscription, the transaction fails with the following error message in the Connector Activity:

Error:
Document:Multiple files found.. Multiple files found with the given file name match pattern. Please try changing match pattern to match single file name.

Cause

This issue occurs when the GET command script in the SFTP Endpoint configuration is too broad.

By design, an integration subscription is built to process exactly one file per execution cycle. If the file search pattern (e.g., using a wildcard like *) matches more than one file in the specified host directory, the system cannot determine which file is the correct one to process and triggers this error to prevent data corruption or duplicate processing. Common triggers include:

  • Residual Files: Older files from previous days were not moved or deleted from the SFTP folder.
  • Hidden/Temp Files: Your external system may be generating temporary or hidden files in the same directory that match the naming pattern.
  • Timestamp Overlap: The pattern (e.g., ${__datetime:yyyyMMdd}) matches multiple files uploaded within the same day.

Resolution

Because the endpoint script is managed by SAP, you must ensure the host directory is clear of extra files or request a script update from SAP Support.

Step 1: Cleanse the SFTP Directory (Self-Service)

  1. Connect to your SFTP server directly using a client like WinSCP or FileZilla.
  2. Review the directory specified in your endpoint configuration.
  3. Ensure that once a file is processed, it is moved to an Archive folder or deleted to prevent it from being picked up in the next run.
  4. Remove any hidden or temporary files that may be matching the naming pattern.

Step 2: Request a Script Update (SAP Support)

  1. If you require a more specific naming pattern (e.g., adding hour/minute variables) to ensure unique matches, you must contact SAP Fieldglass Support.
  2. Open a support case and provide the Endpoint Name and the specific Transaction ID of the failure.
  3. Provide the desired new naming convention (e.g., requesting the addition of ${__datetime:yyyyMMddHHmm} to the GET command).
  4. SAP Integration Engineers will update the script on your behalf and confirm once the pattern has been narrowed.

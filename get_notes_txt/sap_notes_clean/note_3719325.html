Symptom

You are wondering how to check the daily peak memory usage in HANA database.

Environment

  • SAP HANA Platform Edition 1.0 
  • SAP HANA Platform Edition 2.0 

Resolution

1. Daily peak memory utilization can be check via DBACOCKPIT/DB02 -> Performance -> Load History ->Database Used Memory.

DBACOCKPIT/DB02 fetches data from the view M_LOAD_HISTORY_HOST.
With the default settings (1 sample every 10 seconds and 100,000 sample entries), the history retention is approximately 11 to 12 days.

2. The following SQL statement can be used to check more historical data.

SELECT

TO_DATE(SERVER_TIMESTAMP) AS DAY,

MAX(MEMORY_USED) / 1024 / 1024 / 1024 AS PEAK_USED_MEMORY_GB

FROM _SYS_STATISTICS.HOST_LOAD_HISTORY_HOST

GROUP BY TO_DATE(SERVER_TIMESTAMP)

ORDER BY DAY;

The above SQL statement retrieves data from the view HOST_LOAD_HISTORY_HOST.
By default, the retention period for this view is 42 days.

The two methods retrieve data from different views: M_LOAD_HISTORY_HOST and HOST_LOAD_HISTORY_HOST.
Although the data from M_LOAD_HISTORY_HOST is synchronized to HOST_LOAD_HISTORY_HOST by the statistics server, not all data is synchronized.
As a result, the outputs may differ slightly, but there should not be any significant differences.

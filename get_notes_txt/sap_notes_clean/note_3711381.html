Symptom

  • The "Assign Responsible Group for Onboarding Rehire Check" rule is evaluated to 'NULL' and does not assign the corresponding group configured in the rule when using Job Information - Position as a condition.
  • The rehire check rule does not assign the rehire verification task to the responsible group, although the conditions were satisfied.

Environment

SAP SuccessFactors Onboarding

Reproducing the Issue

  1. Configure the "Assign Responsible Group for Onboarding Rehire Check" rule condition based on onboardee's Position 
  2. Assign the rule in Manage Data > Onboarding Configuration 
  3. Initiate Onboarding for a rehire
  4. When the rehire verification task is triggered, it is not assigned to the expected group configured in the rule

Cause

The rule is being evaluated as 'null' because the BPE Service User does not have proper permissions granted for the Position object.

  • In Configure Object Definition, the Position Object is assigned to Permission Category 'Miscellaneous Permissions', but it is not granted for the OnboardingBpeAdmin role.
  • In Check Tool > System Health > Rehire coordinator rules exist and are valid, there is an issue found.

Resolution

Ensure that the below permissions are Granted:

  • Permission Category: Manage Onboarding or Offboarding
  • Permissions granted:
    • Restart Onboarding Permission
    • Access New Hire Data Permission
    • Employment Profile Selection API Permission
  • Permission Category: Employee Central Effective Dated Entities
    • View Current, Edit/Insert is granted for the Job Information fields - Company, Division, Location

Aside from making sure that the Position field is populated and passed from Recruiting, ensure that permissions are granted to the BPE Service User.

  • Please add the Miscellaneous Permissions for the RBP role OnboardingBpeAdmin (or whatever is the Permission Category assigned in Configure Object Definitions under Security)
  • Execute an RBP Refresh (Provisioning)
  • Run the check in the Check Tool > System Health > Rehire coordinator rules exist and are valid
  • There should be no more issue found in the result.

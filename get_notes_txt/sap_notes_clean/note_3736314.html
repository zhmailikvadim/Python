Symptom

When running the "Synchronize Users from SuccessFactors HCM to SmartRecruiters", no users are being created in SmartRecruiters, even when status is job successfully completed.

Environment

SmartRecruiters for SAP SuccessFactors

Reproducing the Issue

  1. Go to Admin Center → Scheduled Job Manager;
  2. Find the scheduled job Synchronize Users from SuccessFactors HCM to SmartRecruiters;
  3. Select View Details;
  4. Verify that job is successfully completed, but there's no success count for users creation.

Cause

The "status" field is not configured in the Succession Data Model, preventing the sync job from identifying and creating valid users.

Resolution

Configure the "status" field in the Succession Data Model. After this configuration is applied, the sync job will begin creating users in SmartRecruiters.

However, please note that the system uses a watermark/checkpoint mechanism to track sync progress. Users that failed to sync previously will not be retried during delta sync operations unless the watermark is reset. To resolve this:

  • Make a dummy change to the affected users in the User Data File (UDF)
  • The next delta sync job will capture these users

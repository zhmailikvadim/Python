Symptom

The SAP Business One client encounters an unhandled exception when launched using RunAs command

Environment

SAP Business One

Reproducing the Issue

Prerequisites:

Enable the Single Sign-On and run the SAP Business One Client with RunAs command. 

Scenario:

  • Enable the Identity Provider
  • Run the SAP Business One Client with a specific RunAs command below using your username:

“%SystemRoot%\System32\runas.exe” /user:%USERDNSDOMAIN%\R%USERNAME% “%ProgramFiles%\SAP\SAP Business One\SAP Business One.exe”

Expect Behavior:

SAP Business One Client login successfully without terminating or with an error Unhandled Exception.

Actual Behavior:

The SAP Business One Client unexpectedly terminates with an "Unhandled Exception" error.

Cause

SAP Business One client terminates unexpectedly when calling the CEF function CefBrowserHost::CreateBrowserSync. After upgrading the CEF library to address a security issue, this function can no longer be called from the command line.

Resolution

Alternative Solution: 

Run SAP Business One using a PowerShell script.

Follow these steps: 

  1. Create a new file named run_sap_business_one.ps1 and save it in a convenient location, such as C:\.
  2. Copy the content below into run_sap_business_one.ps1. Adjust the command to match your environment such as file path, username, etc.
  3. Open PowerShell. Execute the script by entering its full path: C:\run_sap_business_one.ps1

# Save the below script as "run_sap_business_one.ps1"

$credential = Get-Credential -UserName "Enter the username" -Message "Enter the password for the user"
Start-Process -FilePath "C:\User\Output.x64\SAP Business One.exe" -Credential $credential -NoNewWindow

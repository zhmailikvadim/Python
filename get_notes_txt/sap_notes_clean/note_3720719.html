Symptom

Asset value date on subsequent acquisition postings defaults to the capitalization date of the asset master record instead of being derived from the document date or posting date.

Environment

SAP S/4HANA

Reproducing the Issue

A capitalized fixed asset received multiple subsequent acquisition postings.

Upon review, the asset value date derived from none of the expected sources: neither the document date nor the posting date. 

Instead, it was found to match the original capitalization date of the asset master record.

Cause

A substitution rule is active in the system at the document line item level, targeting the asset value date field (BZDAT).

The substitution logic, stored in table TABVO, replaces the value of BZDAT during the posting process before the standard derivation from the document date or posting date can take effect.

As a result, the system does not read BLDAT or BUDAT as the source for the asset value date; instead, the substitution populates BZDAT with the capitalization date of the fixed asset master record (field AKTIV from table ANLA),

which is why all subsequent acquisition postings consistently reflect that date regardless of the document or posting date entered by the user. 

When the system processes a ZUSO (standard acquisition) posting, and TABVO is stored with 09 as primary rule, it first applies it, which means it will use the capitalization date already recorded in the asset master record (field ANLA-AKTIV, "Capitalized on") as the asset value date.

This has a critical implication: if a capitalization date was manually pre-entered in the asset master before the acquisition posting, that date will be used as the asset value date in place of the document date or posting date on the FI document.

The asset value date then feeds directly into the ordinary depreciation start date calculation (via the period control method of the depreciation key).

Resolution

System design

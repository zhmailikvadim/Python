Environment

  • SAP Financial Consolidation (FC) 10.1

Cause

It seems that the system made an attempt to insert a record with a duplicate (unique) key. This error will also be generated if an existing record is updated to generate a duplicate (unique) key. Typically the cause of this behavior is a duplicate primary key.

Resolution

To resolve the problem, please follow the following steps:

1. Stop the Finance datasource in the Administration console.
2. Copy both tables for safety reasons:
     select * into ct_identifier_pool_save from ct_identifier_pool
     select * into ct_recycled_identifier_save from ct_recycled_identifier
3. Run the following two queries:
     Delete from ct_identifier_pool
     Delete from ct_recycled_identifier
4. Restart the datasource in the Administration console.

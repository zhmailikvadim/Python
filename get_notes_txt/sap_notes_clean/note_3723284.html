Symptom

  • After reversing several documents, the reversal flag and reversal document number were correctly updated in the BKPF table but not in the VICAPROCESSID table.
  • The reversal indicator remains blank in the VICAPROCESSID table

Environment

  • Flexible Real Estate Management (RE-FX)
  • Contract and Lease Management
  • SAP ERP 
  • SAP ERP Central Component
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA Finance
  • SAP S/4HANA
  • SAP S/4HANA Cloud Private Edition

Reproducing the Issue

  1. Reverse several documents using transaction RERAPPRV that were posted by the periodic posting process with transaction RERAPP
  2. Verify the reversal flag and reversal document number in the BKPF table of the reversed accounting documents
  3. Check the VICAPROCESSID table to verify that the posting process is not marked as "reversed"

Cause

The reversal process with transaction RERAPPRV was not executed using the selection by posting process ID. Instead, other selection criteria were applied, which resulted in the reversal process including documents from multiple posting processes. 

The posting process is marked as reversed in the VICAPROCESSID table only if the reversal with transaction RERAPPRV is processed with selection by the process ID.

Resolution

  1. Ensure that the reversal process is executed using transaction RERAPPRV with selection by the process ID.
  2. Verify that the reversal flag and reversal document number are correctly updated in the BKPF table.
  3. Confirm that the reversal indicator is updated in the VICAPROCESSID table for the reversed process ID.

Keywords

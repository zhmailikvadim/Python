Symptom

  • Alerts are triggered and visible in Alert Inbox, but email are sent only once 
  • Monitoring is functioning correctly, and recipient lists are present

Environment

  • SAP Focused Run

Reproducing the Issue

  1. Upgrade SAP Focused Run system 
  2. Enable job and automation monitoring or any other monitoring application.
  3. Observe that alerts are triggered once despite monitoring functioning correctly, first mail is sent then no additional mail is
  4. Configuration is correct, recipient lists being present

Cause

This article explains the standard behavior of alert grouping in SAP Focused Run monitoring applications and provides guidance on enabling atomic alerts for scenarios requiring individual notifications for each alert. It also includes a warning about the implications of enabling atomic alerts.

Resolution

1. Standard Behavior: Alert Grouping

  • Alert Generation: When monitoring is enabled in SAP Focused Run, metrics are continuously evaluated. If a metric breaches its defined threshold (e.g., yellow or red status), an alert is generated.
  • Notification Process:
    • For the first instance of an alert, an email or SMS notification is sent to the configured recipients.
    • Subsequent alerts for the same metric are grouped into an alert group, and no additional notifications are sent for these grouped alerts.
  • Closing the Alert Group:
    • To close the alert group and enable new notifications, the metric must return to a normal state (green or grey status).
    • Once the alert group is closed, if the metric breaches the threshold again (yellow or red status), a new alert is generated, and a new notification is sent.
  • Manual Confirmation:
    • Alerts can be manually confirmed by the user. After confirmation, if the next data collection reports a yellow or red status for the metric, a new alert is generated, and a notification is sent.

2. Enabling Atomic Alerts

  • What are Atomic Alerts?
    • Atomic alerts are individual, standalone alerts generated for each occurrence of a metric breach. They are not grouped, meaning every alert is treated independently.

  • Behavior of Atomic Alerts
    • Each alert triggers its own notification (email/SMS) to the configured recipients, regardless of the metric's previous status.
    • Atomic alerts are useful for scenarios where every occurrence of an issue needs to be tracked and notified separately.
  • How to Enable Atomic Alerts
    • To enable atomic alerts, you can configure the monitoring settings to disable alert grouping for specific metrics. This ensures that each alert is treated independently and generates a notification.

3. Warning: Impact of Enabling Atomic Alerts

  • When the default alert grouping behavior is changed to "Atomic", every alert occurrence will trigger downstream actions such as email or SMS notifications.
  • For example, in job monitoring, consecutive alert occurrences for a job (with the same rating, e.g., red or yellow) will no longer be grouped. Instead, every alert occurrence will result in a new entry in the alert list and trigger a notification.
  • Important Note: Enabling atomic alerts may lead to a high volume of notifications, especially in scenarios where frequent alerts are generated. It is recommended to carefully evaluate the need for atomic alerts before enabling this option.

Key Points:

  • The standard behavior in SAP Focused Run is to group alerts for the same metric into an alert group, sending notifications only for the first alert in the group.
  • Atomic alerts can be enabled for scenarios requiring individual notifications for each alert, removing the alert grouping functionality.
  • Enabling atomic alerts will result in notifications for every alert occurrence, which may lead to a high volume of emails or SMS messages.

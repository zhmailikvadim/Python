Symptom

  • A Team cannot be deleted or changed in the new team design interface of SAP Analytics Cloud.
  • The team is visible only in the new design interface, not in the former design interface, causing uncertainty about which interface displays accurate information.
  • Attempts to delete the team result in the error message: "Object TEAM:t.A:xxxxx was not found or you do not have permissions to read it. Correlation ID: xxxxx"
  • Attempts to change the team result in the system appearing to work without completing the action.
  • The issue persists even with system administrator authorizations, system ownership, and no license issues.

Environment

SAC Administration

Reproducing the Issue

  1. Access SAP Analytics Cloud and navigate to Security > Teams. Ensure the current design experience of teams is being used, not the former one.
  2. Search for the team.
  3. Attempt to edit the team. The screen keeps loading, and the error appears in network logs:
  • "Object TEAM:t.A:xxxx was not found or you do not have permissions to read it."
  1. Reload the page and attempt to delete the team. The error message in the UI states:
  • "Object TEAM:t.A:xxx was not found or you do not have permissions to read it. Correlation ID: xxx"
  1. Switch to the former interface in Teams, search for the team, and confirm it is not visible there.

Resolution

Please create a case under LOD-ANA-LS so the Development Team can clear any erroneous entries in the backend.

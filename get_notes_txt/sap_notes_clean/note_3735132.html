Symptom

  • The task for Pull Data from SAP Analytics Cloud is not visible in the Data Monitor
  • The feature "FINCS_PULL_SAC_DATA" cannot be found in the Activate Features configuration activity
  • Uncertainty whether the SAC integration functionality is activated

Environment

SAP S/4HANA Cloud Public Edition

Reproducing the Issue

  1. Navigate to the Activate Features configuration activity.
  2. Search for FINCS_PULL_SAC_DATA; the feature is not listed.

Cause

The functionality to pull plan data from SAP Analytics Cloud is available as part of the standard scope and does not require manual activation via the Activate Features configuration activity.

The task for pulling data from SAP Analytics Cloud is defined via configuration but will only appear in the Data Monitor once it is assigned to a Task Group.

If this assignment is not maintained, the task will not be visible in the Data Monitor.

Resolution

To make the task available in the Data Monitor, please follow the steps below:

1. Define the Task for SAC Data Pull

  1. Open the configuration activity.
  2. Go to Define Tasks for Pull Data from SAP Analytics Cloud
  3. Create or review the task definition for pulling plan data


2. Assign the Task to a Task Group

  1. Open the configuration activity.
  2. Go to Define Task Group.
  3. Select the Task Group used in your Data Monitor.
  4. Assign the previously defined SAC pull task to this Task Group.
  5. Save your changes.


3. Verify Task in Data Monitor

  1. Open the Data Monitor app
  2. Select the required Global Parameters.
  3. The task will now be displayed using the name defined in the configuration activity and can be executed.

Additional Information:

The availability of the configuration activity "Define Tasks for Pull Data from SAP Analytics Cloud "and the job template "SAP_FIN_CS_PULL_DATA_FROM_SAC (Pull Data from SAP Analytics Cloud)" in the "Schedule Jobs for Consolidation Tasks" app confirms that the functionality is already available in the system.

The feature "FINCS_PULL_SAC_DATA" is no longer required to be activated separately in recent releases.

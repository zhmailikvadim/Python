Symptom

  • Unable to trigger flexible workflow for return purchase order due to inability to maintain negative value in step condition.
  • Error message: "Enter a value greater than zero for the condition 'Total net amount of purchase order is less than or'".

Environment

  • SAP S/4HANA

Cause

  • The flexible workflow will trigger for a return purchase order when the value is negative. However, the standard application does not allow negative values to be set in the workflow condition 'Total net amount of purchase order is less than or equal to'.

Resolution

  1. In the Manage Workflows for Purchase Order App, flexible workflow will allow a value of 0 in the 'Total net amount of purchase order is less than or equal to' field to be configured and activated. However it will not allow activation of negative values (e.g. -500Euro). 
  2. An option may be to code the BADI SWF_WORKFLOW_CONDITION_DEF and BADI SWF_WORKFLOW_CONDITION_EVAL to add and compare a custom condition that will trigger the workflow. 

See Also

  • SAP Help for the BADI SWF_WORKFLOW_CONDITION_DEF & SWF_WORKFLOW_CONDITION_EVAL.

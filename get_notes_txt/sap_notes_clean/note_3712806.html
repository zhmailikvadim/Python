Symptom

• Planned orders created in APO are not updating back into APO from ERP(ECC) Order number.  
• Orders Queue stuck in the inbound queue of the APO system at transaction code SMQ2.  
• The order number displayed in ECC already exists in APO, associated with another material.  
• Error message in SMQ2: A table entry already exists with this secondary key. Error code: /SAPAPO/OM_ERROR215. Return code: 215 (om_sql_duplicate_2nd_key). 

Environment

  • SAP ERP system 
  • SAP Supply Chain Management module
  • Side-by-side system configuration 

Reproducing the Issue

1. Use transaction code /SAPAPO/RRP3 to change mode.  
2. Create a new planned order by entering the required details, such as quantity and dates, and save the order.  
3. Refresh the screen and observe that the order number remains the same as created in the APO system.  
4. Navigate to transaction code SMQ2 in the same system and observe the entry stuck in the inbound queue with error code /SAPAPO/OM_ERROR215.  
5. Use transaction code /SAPAPO/CCR. Select the planned-order object and press Enter. Error /SAPAPO/CIF156 occurred for the created planned order (Order &1 — number mismatch: &2 (APO) <-> &3).

Cause

  • The root cause is an out-of-sync system copy / non-simultaneous copy of systems and number-range counters.
  • APO contains orders with higher numbers due to its state being copied later or used longer, while ERP order number-range counter stayed at a lower value from an older copy.
  • When APO sends new orders to ERP, ERP uses its own “next number” to create the order. If that number is already present in APO or otherwise already used, creation fails or results in duplicate-number conflicts.

Resolution

  1. Identify the highest order number currently present in APO for the relevant order types. 
  2. Confirm the current number range value in ERP for the corresponding order types using transaction code SNUM. 
  3. Update the ERP Number Range Status to a value exceeding the highest number in APO using transaction code SNUM.

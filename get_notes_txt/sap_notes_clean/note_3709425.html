Symptom

  • In a CX Flow using a C4C Custom Connector enrichment, the $select query parameter is ignored, causing the GET request to return the full object instead of the selected fields.

Environment

SAP Customer Data Platform

Reproducing the Issue

  1. Create a Custom Connector to SAP Cloud for Customer (C4C) with a GET operation.
  2. Configure a $select query parameter in the connector as a constant or default value.
  3. Create a CX Flow that triggers this connector as an enrichment action.
  4. Execute the CX Flow for a test profile.
  5. Check the outbound request in logs or monitoring.

Result:
The outgoing GET request does not include the $select parameter, and C4C returns the full object.

Cause

CX Flows do not support constant values for query or path parameters (such as $select) when configured directly in the Custom Connector.
These parameters must be provided as runtime variables during CX Flow execution.

Resolution

Configure the query parameter (for example, $select) as a runtime variable and provide its value in the CX Flow action, not as a constant in the Custom Connector:

  1. Create a request parameter variable for the query parameter.

  2. Map the variable to the query parameter name (for example, $select).

  3. Set the variable scope to Action.

  4. Define the parameter value in the CX Flow action configuration.

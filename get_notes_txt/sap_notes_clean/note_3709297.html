Symptom

  • When creating an interest rate instrument transaction (product category 550), Repayment method "Installment repayment" or "Annuity repament" is selected. In conditions tab, Final repayment condition is added automatically, e.g. Condition type 1120.
  • Despite deleting the final repayment condition, final repayment flow still appears in the cash flow tab after saving and making further changes to the transaction.

Environment

  • Treasury and Risk Management (TRM)
  • EA-FINSERV
  • S/4CORE
  • SAP ERP
  • SAP ERP Central Component
  • SAP enhancement package for SAP ERP
  • SAP S/4HANA
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA Finance

Reproducing the Issue

  1. Create a transaction with product category 550.
  2. Set Repayment method as "Installment repayment" or "Annuity repament", then delete the Final repayment condition in Conditions.
  3. Save the transaction.
  4. Make further changes to the transaction and verify that Final repayment flow persists in the cash flow tab.

Cause

As designed.

Resolution

When the Repayment Method is set as 'Installment Repayment' or 'Annuity repayment', the system not only creates an installment/annuity repayment condition, but also a final repayment condition. This is intended and part of the design.  

Thus the transaction has both an Installment Repayment (or Annuity Repayment) as the main repayment condition and the Final Repayment as a secondary (fitting) repayment condition.

In such a case, some of the final repayment condition attributes, such as the due date and due date day shift values, are automatically set by the system to ensure that the remaining balance of the contract is zero.

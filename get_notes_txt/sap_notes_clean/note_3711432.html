Symptom

  • Some users are unable to see values in the query results/front-end reports for apps.

  • Error messages displayed:
    • Position in code >> Row: 16 Inc: CREATE_CHARACTERISTIC_INSTANCE Prog: CL_RSBOLAP_SELECTOR_BASE
    • You do not have the authorization for component <query technical name>
    • No status change necessary
    • CL_RSBOLAP_SELECTION_OBJECT >> Row: Inc: 46 Prog: IF_RSBOLAP_SELECTOR~INITIALIZE_VARIABLE_

Environment

  • SAP BW
  • SAP BW/4HANA
  • SAP Netweaver

Reproducing the Issue

  1. Run the query in RSRT or a specific front-end tool (e.g. Analysis for Office, Fiori apps)
  2. After selected the variables, the issue occurs.

Cause

  • Missing BASIS authorization for authorization object S_RS_COMP or S_RS_COMP1.

Resolution

  1. Reproduce the error;
  2. Access SU53 to check the authorization errors occurring to the user affected;
  3. Assign the value to the respective Field(s) and Object(s) highlighted in SU53 log, to the user, via normal BASIS process.

See Also

Refer to: SAP Help Portal | OLAP Authorizations

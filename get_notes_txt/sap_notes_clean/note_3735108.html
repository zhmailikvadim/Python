Symptom

  • A vulnerability (CVE-2025-48976) is reported by a vulnerability assessment scan for a .jar under com.sap.aii.igw.commons.api.lib/lib/commons-fileupload-commons-fileupload-1.5.jar.

 Path              : /usr/sap/<SID>/J00/j2ee/cluster/bin/ext/com.sap.aii.igw.commons.api.lib/lib/commons-fileupload-commons-fileupload-1.5.jar
 Installed version : 1.5
 Fixed version     : 1.6

  • The finding is associated with Integration Gateway functionality.
  • Enquiry on mitigation plan or effect of Vulnerability in PI/PO System

Environment

  • SAP NetWeaver
  • SAP Process Integration

Reproducing the Issue

  1. Run a vulnerability assessment scan on the Application Server Java installation.
  2. Observe that the scan flags for CVE-2025-48976

Cause

The Apache Commons FileUpload 1.5 library bundled within the Integration Gateway (com.sap.aii.igw.*) libraries is present on the filesystem, which leads the scanner to flag CVE-2025-48976.

The above vulnerability present in the commons-fileupload-commons-fileupload-1.5.jar means that in some function/feature/method present in this .jar is having this security gap. Now just because a file is .jar is present in the AS Java does not imply that SAP PO is using that functionality of that .jar and if it is not using this functionality (Cloud Integration Content), such CVE have Zero impact on the system and you need not to worry about fixing this, and could ignore it.

The path of this .jar contains com.sap.aii.igw which means this .jar is related to IGW (Integration Gateway) functionality of SAP PO.

Resolution

  1. Verify whether Cloud Integration Content (Integration Gateway) is enabled/used (for example, iFlows from Cloud Integration executed in the on-premise PI/PO system).
  2. If Cloud Integration Content is not enabled/used, there is no functional exposure; the CVE can be ignored. On older 7.50 support packages, NWA may show com.sap.aii.igw.* applications as “stopped” when the feature is not enabled.
  3. If Cloud Integration Content is enabled and mitigation is required, disable the functionality by following steps outlined in SAP Note: 2197483- Cloud Integration Content Enablement.

Note: SAP will update the impacted jar file to a version which does not have the vulnerability in the upcoming SP35 in case you wish to plan for the upgrade

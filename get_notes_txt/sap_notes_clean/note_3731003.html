Symptom

  • A node.js application experienced crashes due to out-of-memory conditions previously.
  • You would like to check the feasibility of monitoring memory usage and configuring alerts so that you are informed in advance.

Environment

  • Cloud Foundry runtime
  • BTP

Cause

Resolution

You can setup the monitoring notifications through the SAP Alert Notification Service;

Create and configure SAP Alert Notification service:

1. Provision an instance of SAP Alert Notification for SAP BTP in the relevant subaccount/space.

2. Define subscriptions with event sources for the Cloud Foundry runtime metrics.

3. Configure conditions for threshold-based alerts, for example:

Memory utilization approaching critical limits (for example, >70%, >85%, >95%).

  • CPU utilization above defined thresholds.
  • Instance health changes (start/stop/crash/restart).
  • Response time/latency exceeding acceptable levels.

You can also use the Application Autoscaler to scale the application temporarily if the memory load is high.

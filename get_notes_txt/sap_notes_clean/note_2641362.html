Symptom

You use HRFORMS to display payslip for a period with retro and the address shown is not current one.

Environment

  • Human Capital Management (HCM)
  • SAP R/3
  • SAP ERP Core Component
  • SAP ERP
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA

Reproducing the Issue

View payslip of an employee with retro calculation on payroll and a change on It0006.

Cause

This is because a lack of customization on the form used.

Resolution

"Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental."

First please check SAP Note 3535319 - HRFORMS: Wrong address being printed due to wrong sorting (DIM_EMPLOYEE) when there are retroes is implemented on your system to assure the DIM_EPMLOYEE is correctly shorted. 

Then on the customization it can be configure the BEGIN DATE and ENDDATE as condition to choose which address to print. On the following screenshot it appear an example of configuration for BEGDA and ENDDA. This has to be customized as per customer needs.

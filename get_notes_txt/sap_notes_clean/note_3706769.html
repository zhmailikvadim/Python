Symptom

This SAP Note addresses the installation of Production Connector 3.0 for SAP Digital Manufacturing.

Other Terms

Manufacturing, DMC, OPC, OPC DA, OPC HDA, OPC UA, OSISoft, PI, WebSocket, machine model, shop floor designer, production process designer, production connectivity model, 2602

Reason and Prerequisites

This SAP Note provides updated installation instructions for Production Connector 3.0 and describes known problems for the installation.

Solution

Production Connector 3.0 is compatible with SAP Digital Manufacturing, release 2602, and its subsequent releases.

Restrictions

The Production Connector cannot support shop floor systems of the type GE Proficy Historian because no runtime components compatible with .NET 10.0 are available for these source systems. You cannot install the Production Connector if you are using Proficy Historian shop floor systems in the integration of SAP Digital Manufacturing with Plant Connectivity.

If you want to use structured data types in the environment of an OPC UA shop floor system, note the supported data types and the restrictions that are documented in SAP Note 3461876.

Technical Prerequisites for the Installation

A detailed installation guide is available on SAP Help Portal. The Production Connector requires two .NET 10.0 runtime components to run.

  • .NET Desktop Runtime 10.0 (x86)
  • ASP.NET Core Runtime 10.0 (x86)

You can download these runtime components from the Microsoft Web site, if required: https://dotnet.microsoft.com/en-us/download/dotnet/10.0. It is important that you use the x86 versions of these runtime components because the Production Connector applications run as 32-bit programs. If you have installed the runtime components, they are usually kept up to date using Windows Update if the machine has a connection to a Microsoft Update Server. The following operating systems support the Production Connector:

  • Windows 11 (x64)
  • Windows Server 2016 (x64) (with restrictions, see below)
  • Windows Server 2019 (x64)
  • Windows Server 2022 (x64)
  • Windows Server 2025 (x64)

If you install the Production Connector on the Windows Server 2016 operating system, you must perform manual steps when generating the cloud to Production Connector certificate as described in SAP Note 3194709. We therefore recommend that you install on a newer Windows server version.

For more detailed information about the release and maintenance strategy for the Production Connector, see SAP Note 3255564.

Ensure that the Windows operating system on which you want to install the Production Connector has the latest status and that you supply it with updates regularly. Usually, Microsoft delivers security updates for supported operating system versions every second Tuesday of the month ("Patch Tuesday"). SAP recommends that you install these updates promptly and asks you to  keep the .NET runtime components up to date.

JSON Web Token (JWT) Validation Setup

The validation of JWT is mandatory as of release 2.0 of the Production Connector. Cloud services will not work without the corresponding configuration. Therefore, before you start an upgrade, maintain the JWT validation parameters in the source release of the upgrade already. This prevents any existing cloud integration being interrupted by the upgrade to release 3.0 due to the missing parameters for JWT validation. Since you can maintain the JWT parameters as of release 1.2 of the Production Connector, you can only upgrade directly to release 3.0 from release 1.2 or higher. The installer will end the upgrade installation if these parameters are missing. For more information about the installation prerequisites for JWT validation, see SAP Note 3400382.

Migration of the Cloud Integration of SAP Plant Connectivity

If you have previously used SAP Plant Connectivity (PCo) to integrate your manufacturing systems with SAP Digital Manufacturing, you have to perform the following steps to migrate to the Production Connector 3.0:

  1. If you have a PCo release lower than 15.5.4, you must first upgrade to the latest patch of PCo 15.5.4.
  2. From PCo 15.5.4, you must first upgrade to the latest patch of Production Connector 2.7. The direct upgrade from PCo to the Production Connector 3.0 is not supported.
  3. From the Production Connector 2.7, you can upgrade to the Production Connector 3.0.

After a successful migration, the cloud integration is run via the Production Connector while it is deactivated in PCo. For more information about the prerequisites for migrating from SAP Plant Connectivity to the Production Connector and for joint operation of PCo and the Production Connector on one computer, see SAP Note 3223128.

SAP prompts you to perform the migration of SAP Plant Connectivity now because the support has ended for the cloud integration component of Plant Connectivity. SAP no longer provides corrections for this component.

Running the Installation

Download the installation program for Production Connector 3.0 from SAP Software Center and install it in accordance with the instructions in the installation guide. If a newer version with a higher support package level or patch level is already available in the Software Center, use this instead. The installation files of the Production Connector are always cumulative and contain all previously published support packages and patches.

You can use the SAP setup installation server to distribute a Production Connector reference installation from a central server to remote computers. For more information about using the SAP setup installation server, see the installation guide for the Production Connector and the SAP setup guide, which can be extracted from any installation file. To do so, run the following command in a command prompt:

<Installation executable> /extract :"<target directory>"

After unpacking, you can find the installation guide directly in the target directory in PDF format.

For dialog installations, start the installation by launching the installation program. The installation scope includes only a single component that is preselected automatically. Choose the  installation directory or accept the proposal from the installation dialog.

Bear in mind the following points regarding subsequent patch installations:

  • Ensure that, in dialog mode, you have stopped all agent instances. After you start the installation in dialog mode, the system issues a warning message if Production Connector agent instances are still running that may impair the upgrade installation process. The installer will automatically stop any other Production Connector application in the installation process. If you perform a remote installation using the installation server, the installation process closes the Production Connector applications that are currently running on the target computer without further warning. Once the installation is complete, the agent instances that were previously running are started again automatically. You can also reproduce this behavior for a local installation without the use of the installation server by starting the installation using a command prompt and entering the following command:

    start /wait <executable installation file> "/silent" 

    If you are using PowerShell, the command is

    & .\<executable installation file> /silent | Out-Null

  • If the installation program specifies a pending system restart, perform the restart before you continue with the installation of the Production Connector.

The Help Portal page https://help.sap.com/docs/PRODUCTION_CONNECTOR contains links to other guides, the feature scope description, and the release notes. You can use the Product Availability Matrix to branch directly to the download section to download the installation file.


Information About the Installation and the Required Post-Implementation Steps

Starting the Control Center in Display Mode

As of release 2.1, the Control Center starts in display mode by default, even if you have Windows administrator rights. In display mode, users without administrator rights now also have the option of starting and stopping agent instances. If you need to update the configuration of cloud integration, you must explicitly start the Control Center as an administrator. For more information about the changes when using the Control Center, see the release notes.

Migration from Plant Connectivity

If you have installed the Production Connector on a computer on which cloud integration was previously configured via SAP Plant Connectivity, the cloud integration should have been automatically migrated to the Production Connector during the installation. You can verify the success of the migration by calling the Control Center of the Production Connector after the installation. If the migration failed, the system issues a corresponding message. In that case, follow the instructions displayed in the Control Center.

If the migration was successful, the agent instances transferred from SAP Plant Connectivity appear in the Control Center and you do not receive any further messages. Open the Configure Production Connectivity app in SAP Digital Manufacturing and select the Plant Connectivity/Production Connector installation belonging to the current machine. It should appear there as "connected". Now choose the Refresh pushbutton and the installation should correctly identify itself as the Production Connector with version 3.0. After this step, you have successfully completed the migration.

After successful migration, you can delete the now inactive cloud configuration elements from SAP Plant Connectivity by choosing Edit -> Remove Migrated Cloud Configuration in the PCo Management Console. You can then continue to use PCo to create local configurations or you can uninstall it.

For more details about the migration from SAP Plant Connectivity to the Production Connector, see SAP Note 3223128.

Setup of the JSON Web Token Validation

After the installation of Production Connector 3.0, complete or check the parameters for the validation of the JSON Web Tokens (JWT). This validation ensures that the Production Connector only processes service requests from the cloud and rejects requests from unauthorized sources. Cloud services will not work without the parameters for JWT validation. For more information, see the application help, which you can call from the Cloud Integration -> JWT Validation tab page of the Control Center.

Changes to Certificate Validation

As of release 2.1, the logic of the certificate validation for Web-based communication in the Production Connector has been adjusted to the logic of the Microsoft certificate validation. You must now store the issuer certificates in the folder for issuer certificates. Additionally, store the root certificate of the certificate chain in the folder for trusted certificates. Self-signed certificates must be stored in the folder for trusted certificates as well.

If you use scenarios in which tag values are written back to a server via a production process in SAP Digital Manufacturing, after an upgrade to Production Connector 3.0, you should check whether the server certificate configured in the Production Connector is stored in the certificate folders in accordance with the rules above. In this scenario, the C:\ProgramData\SAP\ProdCon\CertificateStores\TrustedCertificates folder is the default folder for trusted certificates, and the C:\ProgramData\SAP\ProdCon\CertificateStores\TrustedIssuers folder is the default folder for issuer certificates.

Checking the Integrity of the Installation

The Security Guide in the section Verifying the Integrity of the Installation describes how you can check the integrity of the installation by evaluating the signatures. However, the installation contains the assembly Microsoft.Win32.TaskScheduler.dll from a third-party vendor. This assembly is not digitally signed. If you want to check the authenticity of this assembly in the installed Production Connector version, you can proceed as follows:

  • Open a PowerShell window in Administrator mode.
  • Switch to the system directory of the Production Connector installation (typically C:\Program Files (x86)\SAP\Production Connector\System).
  • Enter the following command to create an SHA256 hash:

Get-FileHash .\Microsoft.Win32.TaskScheduler.dll

The hash should have the following value in the Production Connector releases 3.0.0:

  • 81488FB823E2CCDFAE2C35AE75DEC2E695091332137C360100501F1D8F68F190

Error Handling

The cloud services and access to the configuration are provided by the main service, which is set up as a Windows service that starts automatically after system restarts. During installation, the system automatically searches for a free port for the configuration services. If the port is still blocked by a third-party application or network rules, the installation cannot be completed successfully. In particular, the required migration of the configuration cannot be performed. In this case, the system issues corresponding error messages during the installation. To successfully complete the installation, follow the instructions in the "Troubleshooting of Installation Errors" section in the Installation Guide, which you can find on SAP Help Portal for the Production Connector (https://help.sap.com/docs/PRODUCTION_CONNECTOR).

The agent instances migrated from Plant Connectivity may not write to the event logs assigned to them. If this is the case in your system, restart the computer on which the Production Connector is installed. Implementing at least patch 14 for SAP Plant Connectivity 15.5.4 before the migration ensures that event logs are not deleted unintentionally (see SAP Note 3294841).

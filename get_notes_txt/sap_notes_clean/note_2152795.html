Symptom

From PowerBuilder 12.5 client, trying to connect to EAServer 6.3.1 via SSL with the EAServer's Test Certificate, and the following security error returns:
 "SCL Session Connect:Protocol layer error.:SSL protocol server is not authenticated."

Environment

  • SY EAServer 6.3.1 (EBF 19113)
  • SAP PowerBuilder 12.5.2

Reproducing the Issue

1.  Start EAServer 6.3.1
2.  From a PowerBuilder, use the following code to try to connect to EAServer 6.3.1 and it will result in the ssl protocol is not authenticated error:

SSLServiceProvider sp
getcontextservice("SSLServiceProvider", sp)
sp.setglobalproperty("QOP","sybpks_domestic)
sp.setglobalproperty("pin","sybase")

// connect to easerver using iiops
in_connect.userid = <username>
in_connect.password = <password>
in_connect.driver = "jaguar"
in_connect.application = <application_name>
in_connect.location = "iiops:// + <servername> + ":" + <portnumber>

in_connect.ConnectToServer()

Cause

The issue was with how the certificate was generated.

Resolution

The following commands were used to generate a new certificate and import the certificate to sybcert.db.

//Generate the certificate and follow the prompts
keytool -alias <certificatename> -genkey -keyalg rsa -keysize 1024 -keystore <path_to_easerver>\Repository\Security\keystore.jks


// self-sign the certificate
keytool -selfcert -alias <certificatename> -validity 1825 -keystore <path_to_easerver>\Repository\Security\keystore.jks

// Export the certificate
keytool -export -alias <certificatename> -file <filename>.cert -keystore <path_to_easerver>\Repository\Security\keystore.jks


// Import the certificate into the trustore
keytool -import -trustcacerts -alias <certificatename> -file <filename>.cert -keystore <path_to_easerver>\Repository\Security\truststore.jks

// For a PowerBuilder or c++ client, import to certificate store
<path_to_easerver>\bin\sc-tool -import -file <filename>.cert -keypass <password> -storepass <password>
<path_to_easerver>\bin\sc-tool -trusted -alias "<OrgName>'s CA" -trustedCerts true

(NOTE:   The sc-tool uses the Orgname + "'s CA".   There may be more than 1 space between the OrgName and CA.  Use the following command to verify the syntax:  sc-tool -list -storepass <password>)

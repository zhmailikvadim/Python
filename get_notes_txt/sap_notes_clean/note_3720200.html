Symptom

  • An error occurs when attempting to retrieve user in the Onboarding custom MDF.
  • Attachments uploaded by candidates in Recruiting (RCM) are not populating in Onboarding (ONB) as intended.
  • When attempting to insert data via Manage Data, the system fails to identify user.
  • The issue impacts the ongoing iteration process.

Environment

SAP SuccessFactors Onboarding

Reproducing the Issue

  1. Open Manage data and click on create new.
  2. Type RCMtoONBDocumentRNH and provide onboarding candidate employee ID (e.g., 80016560).
  3. Click on Save.

Cause

Recruiting-Onboarding integration mapping does not support the mapping/flow of attachment type fields.

Resolution

  1. Confirm that Recruiting-Onboarding integration mapping currently does not support the mapping/flow of attachment type fields.
  2. Recommend submitting specific scenarios and business requirements on the Influence Portal to allow the Product Management team to analyze and consider implementing this functionality in future releases.
  3. Refer to the following articles for further information:
  • Refer to: 2090228 - How to Submit Ideas for SAP SuccessFactors Products.
  • Refer to: 2894427 - Is it Possible to Map Field Type Attachment From Recruiting to Onboarding? - Onboarding.

Symptom

Validation on tax number; errors “Tax code is not valid” and “Number of digits is not valid” appear when maintaining or saving an account tax number.

Environment

SAP Cloud for Customer core applications

Reproducing the Issue

  1. Go to the Account work center.
  2. Open any existing account or create a new account.
  3. Navigate to the Tax Numbers tab.
  4. Click Add to create a new tax number entry.
  5. Select Country: XXXXXX and Tax Number Type: Registration Number (XXXXX).
  6. Enter a tax number value, for example: 123456789000.
  7. Click Save and observe the errors “Tax code is not valid” or “Number of digits is not valid”.

Cause

  • The local tax number check (“Tax field Check Enabled”) is enabled in Countries/Region > Tax Attributes for the country, which enforces country-specific validation and leads to format/length errors for the entered value.

Resolution

  1. Open the Business Configuration work center.
  2. Go to Implementation Projects.
  3. Open the Activity List and choose the Fine-Tune area.
  4. Search for and open “Countries/Region”.
  5. Choose “Edit Countries/Region”.
  6. Select the country record for France.
  7. Open the Tax Attributes tab and set “Tax field Check Enabled” to No.
  8. Save the changes and retest the account tax number entry.
  9. Note: When this setting is No, the system uses the tax number maintained in the connected ERP system; the tax number cannot be modified in the cloud system.
  10. Note: Follow-up activities (for example, invoicing) will use the ERP-maintained tax number across integrated processes.

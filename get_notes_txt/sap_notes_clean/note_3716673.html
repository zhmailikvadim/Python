Symptom

  • Data discrepancy issue observed in Story Report between e.g. Numeric Point Chart and Bar Chart, while using the same data source.
  • Same fields and Filters are applied to the different chart types
  • The total shown in the Numeric Point Chart is different from the totals of the Chart values (when manually summed)

    Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.

Environment

SAP SuccessFactors HCM Suite

Reproducing the Issue

  1. Navigate to Report Centre
  2. Edit Story report
  3. Copy the e.g. Numeric Point widget.
  4. Paste it to the same Story Canvas page.
  5. Adjust the chart type from Numeric Point to Bar/Column.  Select a Dimension for this new chart type
  6. Observe that the number of totals displayed is not consistent between the two charts.

Cause

Insufficient Dimension Context for the Calculation.

Resolution

  1. Add the Dimension used in the bar charts as Dimension Context for the Calculated Dimension. e.g. Select Business Unit from the list if the bar chart uses Business Unit as Dimension.
  2. Ensure that the dimension context of the calculated measure aligns with the bar chart's configuration to achieve consistency in the total values displayed across different chart types.

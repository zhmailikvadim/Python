Symptom

In class CL_IM_CO_MES_INTEGRATION, the method DETERMINE_DISTRIBUTION_STATUS is activated. However, after the final confirmation (CNF) for the production order, the status "Distributed" (DISB) still changes to "Distribution Completed" (DISD).

Environment

  • SAP ERP Production Planning (PP)
  • SAP R/3
  • SAP R/3 Enterprise
  • SAP ERP Central Component
  • SAP ERP
  • SAP Enhancement package for SAP ERP
  • SAP Enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA, on-premise
  • SAP S/4HANA Cloud Private Edition

Reproducing the Issue

  1. Create a production order.
  2. Activate the method DETERMINE_DISTRIBUTION_STATUS.
  3. Post a final confirmation for this order.
  4. Observe that the status 'DISB' changes to 'DISD'.

Cause

The status DISB indicates that the order is distributed to a system where confirmations are sent from. After a final confirmation (CNF), the status DISB switches to DISD, meaning it is no longer relevant for the subsystem.

The method DETERMINE_DISTRIBUTION_STATUS is designed to reset the order status to "DISB" if the status changes from "CNF" to "PCNF" after a cancellation. It cannot prevent the status from changing to 'DISD' from 'DISB'.

Resolution

This is standard behavior, not a bug.

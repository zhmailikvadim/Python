Symptom

After upgrading, you get a NoSuchMethodError exception thrown by your custom action's references to the XMLDataType class.  This class is exposed by the BLS SDK package for custom action development.  The affected methods here are:

public com.sun.org.apache.xpath.internal.objects.XObject evalWithError(String sXPath) throws Exception;

public com.sun.org.apache.xpath.internal.objects.XObject eval(String sXPath);

Reason and Prerequisites

The cause for this that MII uses the Xalan library for XML processing in the SP5 service pack, as opposed to using default Oracle/Sun JDK classes.

Solution

There are two solutions here.  There will be a 14.0 SP5 patch to revert this change and restore the original method signature.  The eval and evalWithError methods will be deprecated.  You can wait for this patch (see the related notes), or you can implement the following steps.

-------------------

To update your code to use the Xalan objects, make sure the Xalan.jar is available to the classpath of the action.  Also, update your references and import the Xalan versions of these classes.

Old package/class:

import com.sun.org.apache.xpath.internal.CachedXPathAPI;

import com.sun.org.apache.xpath.internal.objects.XObject;

New package/class:

import org.apache.xpath.CachedXPathAPI;

import org.apache.xpath.objects.XObject;

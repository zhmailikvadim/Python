Symptom

During Service Entry Sheet (SES), the error Message No. M7036 not displaying even after the confirmation control exists in the PO.

Environment

  • Materials Management (MM)
  • SAP ERP
  • SAP S4/HANA 

Reproducing the Issue

  1. Create the service PO. 

  2. Check under confirmation tab in the item details, confirmation key is maintained i.e.. EKPO-BSTAE is having a value. 

  3. Try to create SES in ML81N by providing the created PO number. 

  4. System is allowing to create SES, when it should not allow. 

Cause

Since the confirmation control key is maintained at item level (and not at service level) this functionality cannot be used in the same way as for normal materials.  

Resolution

This is the intended design, confirmation control key is not considered during SES creation.

See Also

A possible workaround could be user-exit "EXIT_SAPLMLSR_010", It has the importing parameter EKPO(-BSTAE). Kindly implement own custom logic to display an error message when posting the Service Entry Sheet (SES) for a service purchase order (PO).

Keywords

PO, SES, Confirmation control key, user exit, ML81N, M7036

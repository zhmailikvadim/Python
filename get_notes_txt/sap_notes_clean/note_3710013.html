Symptom

  1. The interface user from CPI to FMS was locked due to incorrect logons.
  2. All FMS inbound interfaces failed due to the locked user.
  3. The issue is intermittent, and the credentials work fine after unlocking.

Environment

  • SAP Integration Suite
  • Cloud Integration

Reproducing the Issue

  1. Observe that the interface user from CPI to FMS is locked due to incorrect logons.
  2. Review SM20/SM21 logs to identify multiple incorrect password attempts originating from the Cloud Connector host.
  3. Unlock the user manually in FMS and confirm that the credentials work fine after unlocking.

Resolution

Change the user credentials in the backend system to prevent the locking issue.

If the issue persists, share the below details to BC-MID-SCC team:

  1. Set log settings in the Cloud Connector to DEBUG for Cloud Connector Loggers, Information for Other Loggers, and enable Payload Trace/Tunnel Traffic Trace flag for the affected subaccount.

  2. Replicate the issue and collect the following logs from the Cloud Connector:
  • ljs_trace.log/scc_core.trc
  • traffic_trace_<subaccount>_on_<region>.trc/tunnel_traffic_<subaccount>_on_<region>.trc
  • Timestamp of replication
  • Provide screenshots of subaccount information, ACCESS CONTROL mapping details, and Check availability details from SCC.
  • If principal propagation is used, provide the certificates maintained in SCC and level 3 ICM logs from SMICM.

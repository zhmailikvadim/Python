Symptom

  • The following error occurs intermittently when pressing the 'Connection Test' button on an SAP application datastore in both the Sandbox and Production environments:

    Test failed for the default configuration '<datastore configuration>' of datastore: '<datsatore name>':
    Cannot connect to SAP Applications datastore <> using host <<application server>>, client <<client number>>, user <<username>>, and system number <00: CRFC error: RFC_COMMUNICATION_FAILURE-
    LOCATION CPIC (TCP/IP) on local host with Unicode
    ERROR partner '<application server>:<port number>' not reached
    TIME <timestamp>
    RELEASE 753
    COMPONENT NI (network interface)
    VERSION 40
    RC -10
    MODULE D:/depot/bas/753_REL/src/base/ni/nixxi.cpp
    LINE 3458
    DETAIL NiPConnect2: <application server>:<port number>
    SYSTEM CALL connect
    ERRNO 10060
    ERRNO TEXT WSAETIMEDOUT: Connection timed out
    COUNTER 10
    [SAP Partner]>. Please make sure the SAP server is running and the login information is correct.

  • The connection test succeeds a few seconds later.
  • The DS Agent and the SAP system are hosted on the same machine.

Environment

  • SAP Cloud Integration for Data Services 1.0.11.xxxx
  • Windows OS

Reproducing the Issue

  1. Login to CPI-DS organization.
  2. Go to "Datastores" tab.
  3. Select the SAP application datastore.
  4. Press "Test Connection with Default Configuration" button.

Cause

Antivirus software may interfere with the connection test.

Resolution

Deactivate the antivirus software on the DS Agent host and try again. If the issue persists and the Windows host cannot resolve to itself, contact Microsoft Support.

Keywords

LOD-HCI-DS, CI-DS, CPI-DS, database, connection, timeout, RFC_COMMUNICATION_FAILURE

Symptom

In the process "MOS Billing: Receipt of Price Catalog (Receiver View)" (DE_B_MOSB_PCAT_SUP), step 0020 "Request for Price Catalog Data" contains a check in method /US4G/CL_PROC_STEP_PRICAT -> ENHANCE_STEP_DATA that verifies whether the predecessor document exists. You cannot adjust this check to meet your requirements.

This issue is relevant for the market role: SUP.

Other Terms

Price Catalog, /US4G/CL_PROC_STEP_PRICAT, /US4G/CL_UBD_DATA_MOSB_PRICAT, GET_DATA_FOR_QUERY, /US4G/CL_UBD_MOSB_PRICAT, /US4G/IF_UBD_MOSB_PRICAT~QUERY, BO_UBD_MOSB_PRICAT, MOSB, predecessor document

Reason and Prerequisites

Insufficient customization options.

Solution

Implement the corrections of this SAP Note or import the relevant support package.


After implementing this SAP Note, the system provides code changes in /US4G/CL_PROC_STEP_PRICAT -> ENHANCE_STEP_DATA that allow you to customize the price catalog retrieval logic.
If you need to implement your own price catalog retrieval logic to affect step 0020 "Request for Price Catalog Data" in process DE_B_MOSB_PCAT_SUP, follow these steps:


1. Open the Fiori App "Configure Business Objects"
2. Select and open business object BO_UBD_MOSB_PRICAT
3. Select "Extend" and enter your own Data Container Class, such as 'ZCL_UBD_DATA_MOSB_PRICAT', then select OK

  • for class ZCL_UBD_DATA_MOSB_PRICAT, ensure it inherits from class /US4G/CL_UBD_DATA_MOSB_PRICAT
  • redefine method GET_UBD_INSTANCE to instantiate with class ZCL_UBD_MOSB_PRICAT.
    • for class ZCL_UBD_MOSB_PRICAT, ensure it inherits from class /US4G/CL_UBD_DATA_MOSB_PRICAT
      • redefine method /US4G/IF_UBD_MOSB_PRICAT~QUERY to fulfill your business requirement

You can verify the fix by testing your customized price catalog retrieval logic in step 0020 of the process.

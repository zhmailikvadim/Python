Symptom

You use transaction FWZZ to create security master data for a bond. On the Basic Data tab, you enter a nominal value. If the nominal value is greater than 1,000, the system uses the entered amount for the cash flow simulation. However, the system uses the default amount of 1,000 for the effective interest rate calculation instead of your entered value.

Other Terms

FWZZ, EIR, effective interest rate

Reason and Prerequisites

A programming error causes this issue.

Solution

After implementing this SAP Note, the system uses your entered nominal value for both the cash flow simulation and the effective interest rate calculation when the nominal value is greater than 1,000.

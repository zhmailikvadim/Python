Symptom

This SAP Note provides you with corrections for the withholding income tax (known locally as Imposto de Renda Retido na Fonte, IRRF) calculation when the Tax Code contains the percentage subject to tax lower than 100% in invoice posting.

When you post an invoice, no IRRF is calculated even though the gross base amount exceeds the minimum amount for retention or the calculated tax value is lower than expected.

Other Terms

BR, Brazil, 15270/2025, IRRF, IRRF Reduction, New IRRF

Reason and Prerequisites

This SAP Note is relevant only for Brazil.

For all S/4HANA, SAP_FIN, and SAP_APPL releases, you have implemented the following SAP Notes:

  • 3696029 - Brazil Withholding Tax Calculation - New Customizing Activity for IRRF Amount Reduction
  • 3697535 - Brazil Withholding Tax Calculation - Calculation of Withholding Income Tax (IRRF) Reduction
  • 3705756 - Brazil Withholding Tax Calculation – Calculation Error in Withholding Income Tax (IRRF) Accumulation Amount by Withholding Tax Type and Code

For S/4HANA 109 to 105, SAP_FIN 617 and 618, and SAP_APPL 606 releases, you have implemented the following SAP Notes:

  • 3700214 - Brazil Withholding Tax Calculation - Calculation Error in Withholding Income Tax (IRRF) Accumulation Amount
  • 3701949 - Brazil Withholding Tax Calculation - Calculation Error in Withholding Income Tax (IRRF) Amount
  • 3704495 - Brazil Withholding Tax Calculation - Withholding Income Tax (IRRF) Accumulation by Withholding Tax Type and Code

Solution

After implementing this SAP Note, the system correctly calculates the gross base amount. As a result, the withholding income tax (IRRF) amount is correctly calculated for the documents.

For more information about the development delivered for the Legal Change, see the entry for this SAP Note in the What's New topic on SAP Help Portal.

  • SAP S/4HANA: Reduce Withholding Income Tax for Natural Persons
  • SAP ERP: Reduce Withholding Income Tax for Natural Persons

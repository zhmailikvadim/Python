Symptom

When starting ASE, the boot fails and the error log prints as below: 

  • kernel Either the config parameter 'use security services' is set to 0, or ASE does not support use of external security mechanisms on this platform. The Security Control Layer will not be initialized. No external security mechanisms will be supported.
  • kernel ERROR: FIPS mode is enabled but FIPS Libraries are not available.

Environment

  • SAP Adaptive Server Enterprise (ASE)
  • SAP Adaptive Server Enterprise (ASE) 16.0

Cause

  1. This error is rare as in most cases ASE should adjust the 'configuration parameters' automatically.
  2. When changing a configuration parameter on a running ASE
  • FIPS login password encryption = 1
  • use security services = DEFAULT 

Resolution

The Sybase server is able to start, after setting the below two parameters in <SID>.cfg (\sybase\<SID>\<SID>.cfg) file.

[Security Related]

  • FIPS login password encryption = 0
  • use security services = 0

Note: Make sure to take the backup of <SID>.cfg file before making any changes. ex: <SID>_bkp.cfg

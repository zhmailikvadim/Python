Symptom

  • Emails are not processed successfully by the mail receiver interface.
  • Message processing fails with the following error: "An invalid XML character (Unicode: 0x3) was found in the element content."

Environment

  • SAP NetWeaver
  • SAP Process Integration

Cause

The issue occurs because the email payload contains a non-printable control character (Unicode 0x03, ETX) within the XML content. XML specifications do not allow control characters in the range 0x00–0x1F (except tab, line feed, and carriage return), causing the XML parser to fail.

Resolution

  1. Validate the failing payload using an XML validator or a text editor capable of displaying hidden/control characters.
  2. Locate the invalid control character (Unicode 0x03) in the XML payload, often appearing as an invisible character before closing HTML tags (for example, immediately before </div> in the email body).
  3. Remove or sanitize the invalid character from the payload before XML parsing.
  4. Request users or automated workflows to send emails in text/plain to ensure XML compliance and prevent insertion of unsupported characters.

Please note: This behavior is not a PI/PO product defect. It is a standard XML format requirement, and messages containing invalid XML characters cannot be processed by any XML-compliant parser.

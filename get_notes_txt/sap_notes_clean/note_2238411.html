Symptom

You find it's impossible to add users to an environment - after you've added the users and you find the users disappear from the Security->Users screen on BPC Web Administration. It's not possible to assign users to team or profile either.

On the ABAP server side, the report UJE_VERIFY_SECURITY seems to be completely broken. If you run report UJE_VERIFY_SECURITY_DATA to check security data, it always says that "You are not authorized to maintain authorizations". But in fact this is not true because the BPC service user is assigned to role SAP_BPC_SERVICE and the user who is running UJE_VERIFY_SECURITY_DATA has SAP_ALL authorization. The report may also complain that "Agr ZBPC_XYZ has incorrect value". If you use transaction PFCG to check the mentioned role, you may find the authorizations tab page is flagged with red icon, and the status of the authorization profile is "No authorization data exists". If you continue to run report UJE_VERIFY_SECURITY_DATA with option "Fix security data if has error", it will say that the role is fixed. But when you run it again to check security data, it will even report that the role does not exist.

Other Terms

PFCG_ORGFIELD_CREATE, $RSINFOAREA

Reason and Prerequisites

This is because you've activated the "Organizational levels" popup for transaction PFCG. For example, you may have defined field RSINFOAREA as organizational level field.

Solution

Perform the actions below to fix the issue:

  • Apply the program correction instructions of this SAP Note, or upgrade to the corresponding Support Package, and
  • Assign one more authorization to the BPC service user: S_USER_VAL (AUTH_FIELD:*; AUTH_VALUE:*; OBJECT:$ORGVALUE$). This authorization can be tailored to more specific authorizations based on your real situation, see SAP Note 495282.

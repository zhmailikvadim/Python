Symptom

  • When a transportation unit is linked to multiple deliveries with varying routes assigned in ECC during delivery creation, the system generates the warning message "Loading with unclear route; No sequence proposal (/SCWM/RF_DE235)".
  • The system can also display the error message "Handling unit is not scheduled (/SCWM/RF_DE014)" afterwards.

Environment

  • SAP S/4HANA
  • SAP Extended Warehouse Management

Reproducing the Issue

  1. Open the transaction code /SCWM/RFUI.
  2. Log into the system (Warehouse, Resource, and Device).
  3. Follow the menu path: Outbound Process -> Loading -> Loading by Transportation Unit.
  4. Enter Transportation Unit and press Enter.
  5. Press Enter again.
  6. Observe the error message: "loading with unclear route; No sequence proposal" (/SCWM/RF_DE235).

Cause

Standard system behavior.

Resolution

Concerning the loading sequence logic:

  • If multiple different routes exist within the deliveries, the loading sequence will be cleared.
  • If there is at most one specified route and other entries have an initial route, the loading sequence will be maintained.

When multiple routes are present, it is challenging to determine the real-world stopping order. As the Radio Frequency (RF) framework is not designed to evaluate feasibility or handle transportation management, it cannot dictate loading positions. Consequently, the loading sequence is cleared. RF assesses the given routes, matches them against expectations, and sets the loading sequence accordingly. This is the standard design and widely accepted by the customer base, hence modifications are not feasible. If differing routes exist on deliveries compared to the transportation unit route, check if a new route encompassing all stops can be established and if route determination supports this for the deliveries. When varying routes are assigned to deliveries, the loading sequence will be cleared, and this outcome is not changeable.

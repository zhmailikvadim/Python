Symptom

  • Program terminated in remote system XXX: 208:42000 cannot be interpreted as a number
  • Logging table does not exist in the Database

Environment

SAP Landscape Transformation Replication Server

Reproducing the Issue

  1. Replicate a table
  2. Error Program terminated in remote system XXX: 208:42000 cannot be interpreted as a number shows in the application logs
  3. Logon to the source system
  4. Run transaction SE11
  5. Enter the logging table
  6. An error appears stating the Table does not exist in the database

Cause

The logging table does not exist in the database

Resolution

  1. Stop the replication for the table 2661071 - How to stop/start the replication for a table - SLT 
  2. If the logging table still exists in SE11 and gives the error "Table does not exist in the database" perform the below steps, if not skip to step 3 - 
    1. Run transaction SE37 on the source
    2. Enter the Function Module DDIF_OBJECT_DELETE
    3. Enter the type TABL 
    4. Enter the logging table
    5. Execute
  3. Reload the table

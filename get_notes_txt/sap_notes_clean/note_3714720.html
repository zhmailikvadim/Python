Symptom

This SAP Note is relevant only for the country versions Germany, Switzerland, and Austria.

The long fields are not active in your system, the additional fields are active. As a result, the House Number field is available as an additional field in the patient address. You use the GetDetail method of the Patient business object (BAPI_PATIENT_GETDETAIL) to determine the patient data. The system does not return only the street in any field in table ADDRESSES for the patient's address or for the patient's additional address. The STR_NO field in the ADDRESSES structure always contains the street and house number. The field STREET_LONG is not filled because the long fields are not active. The HOUSE_NO field is filled if the house number has been maintained in this field.

A field is missing that contains only the street in this constellation (long fields are not active, additional fields are active).

Other Terms

BAPI_PATIENT_GETDETAIL, TN00, IS-H: Client Control Parameters, Additional Fields, Add. Fields, EXTFIELDS_ACTIVE, long fields, LONG_PAT_ADR, STR_NO, HOUSE_NO, STREET_LONG, street, house number

Reason and Prerequisites

Prerequisite:

You use one of the following country versions:

  • Germany (DE)
  • Switzerland (CH)
  • Austria (AT)

In the Customizing activity Maintain Client-Specific Basic Control (SAP Healthcare - Industry-Specific Components for Hospitals -> Basic Settings -> System Parameters -> Maintain Client-Specific Basic Control -> Set Client-Specific Control Parameters), you have maintained both parameters for your client as follows:

  • Long fields Patient and Patient Address: not selected
  • Addnl Fields Active: Selected

Reason:

Missing function

A field is missing that contains only the street in this constellation (long fields are not active, additional fields are active). The field STREET_LONG is only filled if the long fields are active.

Solution

Implement the attached correction instructions or import the specified support package.

Afterwards, the method GetDetail of the business object Patient (BAPI_PATIENT_GETDETAIL) returns the street name without house number in the export parameter ADDRESSES in the constellation described above (long fields: not active, additional fields: active) for the patient address or for the other patient address in the field STREET_LONG. You can find the house number in the HOUSE_NO field as before. The STR_NO field is still filled with the street and house number.
CAUTION: If the house number was not entered in the House Number field in the patient's address, but in the Street field, the STREET_LONG field also contains the house number in this constellation.

Please note:

This change only affects the patient address and the other patient address, not the addresses of next of kin, employers, or other addresses that are stored in the system for the patient in their country version.

For these addresses, the street name with house number can only be found in the table ADDRESSES as before in the field STR_NO, since you cannot enter the street and house number for these addresses in separate fields on the interface of SAP Patient Management.

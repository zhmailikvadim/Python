Symptom

  • Getting an error "Account & for company code & is not defined" GM110 in FAGLGA4B.

Environment

  • Financial Accounting (FI)
  • SAP ERP
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA Finance
  • SAP S/4HANA

Reproducing the Issue

  1. Execute transaction code FAGLGA4B
  2. Enter mandatory parameters

Cause

  • Secondary cost elements are relevant only in CO, not in FI. Since NewG/L is an FI function, allocating secondary cost elements is not permitted

Resolution

Please note that the transactions FAGLGAXX pertain to NewG/L allocations, which permit only pure FI accounts in exit 07 (master data validation exit) and do not allow secondary cost elements. For instance, account TEST123 is classified as type 'S', indicating it is a secondary cost element account, and therefore, it cannot be utilized in NewG/L allocations. Such accounts are suitable for CO allocations, but not for NewG/L operations where they are disregarded.

Additionally, this explanation is supported by the KBA 3244556 - FAGLGA35 - Error GM110 - Account for company code is not defined. Although this KBA references transaction FAGLGA35, the information is relevant and applicable to all NewG/L allocation transactions.

Therefore, this restriction is inherent and unavoidable due to a specific validation in the master data, which verifies the account type used in NewGL allocations. This behavior is consistent with the intended system design and functionality.

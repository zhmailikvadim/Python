Symptom

The Next navigation button is disabled when the Physical Inventory item list is sorted in descending order

Other Terms

  • SAP Service and Asset Manager
  • SAM
  • SSAM
  • ICMTANGOAMF10-42706

Reason and Prerequisites

Reason:

The Next button enablement logic was incorrect, and the item query intended to display results in descending order didn't include the required sort order

Prerequisites:

  •  BAS or Visual Studio Code with SAM MDK project
  • `patch` command (available on BAS and macOS by default) via terminal or other patching utility
  • Read the guidelines on how to apply the patch and read the appendix section if you receive any error

Solution

Fixed the Next button enablement logic and fixed the sort oredr in item query intended to display results in descending order

 

To apply the patch, download the attached 3721052.patch file and save it in a known location, preferably adjacent to your SAM.mdkproject.

If you are on Windows, see SAP Note 3533879 for how to install a .patch file in Windows. If you are on a Mac, follow the steps below:

  1. Open a terminal in the root of your SAM metadata (i.e. SAM.mdkproject/metadata or SAM.mdkproject/metadata/base.definitions)
  2. Run the terminal command `patch --no-backup-if-mismatch -p4 < path/to/3721052.patch`. 
  3. Verify in the terminal window that the following files have been modified and listed as patched, as indicated by the console output:
  • patching 'Rules/Inventory/Item/ItemPreviousNext.js'
  • patching 'Rules/Inventory/PhysicalInventory/PhysicalInventoryItemDetailsQuery.js'

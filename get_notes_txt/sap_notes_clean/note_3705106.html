Symptom

This SAP Note provides updates to ensure continued connectivity with the eSocial platform following the certificate update to a new security standard, in compliance with the eSocial Certificate Update regulation published by the Brazilian Government on December 11, 2025.

The eSocial system is transitioning to certificates issued by the international Certification Authority, Sectigo, which impacts the communication channel between employers' systems and the eSocial platform.

Other Terms

eSocial; Certificate Update; Security Standard; Sectigo; TLS; HTTPS; Root Certificate; Intermediate Certificate; Sectigo Public Server Authentication Root R4; Sectigo Public Server Authentication CA OV R36

Reason and Prerequisites

Reason

The eSocial system requires an update to its security certificate standard to maintain secure HTTPS (TLS) connections. The new certificates are issued by Sectigo Certification Authority and must be installed in application servers' trust anchors to ensure continued connectivity with the eSocial platform.

This legal change is mandatory and affects systems that communicate with eSocial.

Solution

From now on, the system supports the new eSocial security certificate standard using Sectigo Certification Authority certificates.

To maintain successful HTTPS (TLS) connections with the eSocial platform, the following certificate chains must be installed in the application servers' trust anchors:

Root Certificate: Sectigo Public Server Authentication Root R4

Intermediate Certificate: Sectigo Public Server Authentication CA OV R36

The certificate update is effective from:

- January 12, 2026 for the production restricted environment

- June 2026 for the production environment (specific date to be confirmed by authorities)

Manual Activities (also in the attachment)

The following manual activities must be performed by system administrators:

  1. Download the new Sectigo Certification Authority certificates as follows:
    1. Root Certificate: Sectigo Public Server Authentication Root R4
      1. Download URL: https://crt.sh/?d=4256644734
      2. File name: Sectigo Public Server Authentication Root R4.crt
    2. Intermediate Certificate: Sectigo Public Server Authentication CA OV R36
      1. Download URL: https://crt.sh/?d=4267304698
      2. File name: Sectigo Public Server Authentication CA OV R36.crt
  1. Update eSocial SSL certificates in Trust Manager (STRUST) transaction:
    1. Access the STRUST transaction.
    2. Select the eSocial SSL connection that you want to update.
    3. Remove the old certificates:
      • Locate and remove the following certificates:
      • CN=Autoridade Certificadora Raiz Brasileira
      • CN=Autoridade Certificadora do SERPRO SSLv1
    4. Add the new certificates:
      • Click on Certificado → Importar.
      • Select the new certificate file for the production or restricted environment service and confirm.
      • Once imported, click on the Incluir na lis.certificados button.
      • Ensure the certificate now appears in the Lista de Certificado field.
    5. Save your changes.
  1. Test the connection with eSocial environment:
    1. Access transaction SM59.
    2. Select the HTTP connection for eSocial.
    3. Click on Teste conexão button to verify the certificate installation and connectivity with eSocial services.

Note:

All steps should be performed in accordance with the instructions provided in SAP Notes 2514483 and 2585709. Please ensure you review these SAP Notes thoroughly, especially the attached Digital_certificates_v4.pdf for more detailed guidance.

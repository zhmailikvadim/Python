Symptom

When executing the liability calculation for a contract created via Optimized Contract Management (OCM), the program may encounter an error. Despite the error, posting records for the affected contract are still stored in the posting table.

Depending on the specific nature of the error, certain records—such as exchange rate differences for costs—may be missing. In these instances, the remaining records are inconsistent and the balance does not equal zero.

This SAP Note serves as a follow-up to SAP Note 3487752. While the original note addressed the same underlying issue, this update provides a fix for a specific gap identified in the previous implementation.

Other Terms

FARR_CONTRACT_LIABILITY, OCM

Reason and Prerequisites

Program error

Solution

Implement the correction instruction or corresponding service pack

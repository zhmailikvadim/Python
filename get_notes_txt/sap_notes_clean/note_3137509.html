Symptom

You opened the ESS for an employee and noticed that a W-4 (IT0210) record is missing, even though the employee has a valid record:

 

"Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental."

Environment

  • HR Renewal
  • SAP S/4HANA
  • SAP ERP
  • SAP SuccessFactors Employee Central Payroll

Resolution

There are a few possibilities that may cause this issue. Therefore, please check the following:

   

1. W-4 Indicator

The W-4 Indicator of the given Tax Authority should allow the record to be showed on ESS.

Currently, this is allowed for W-4 Indicators 1, 2, 3, 5, 7 and 8.

To check the W-4 Indicator of the given Tax Authority, please refer to view V_T5UTZ (or view V_T5UTZC_W4_OVR in case you have an override).

  

2. Tables Maintenance

An entry should exist on views V_T588MFPROPC, V_T7XSSPERSUBTY and V_T7XSSPERSUBTYP for the Tax Authority whose W-4 record is missing on ESS.

You can find instructions of some of this customization on the FAQ number 6 of this blog.

Exception: It is not necessary to maintain view V_T588MFPROPC for Federal level in particular, as the Federal W-4 is already part of the standard delivery on ESS.

    

3. Infotype Maintenance

The employee trying to edit the state W-4 record must have one record for that state in at least one of these infotypes: IT0006, IT0207, IT0208 or IT0209.

For IT0006 there is an additional configuration delivered by note 2488866 (IT0210: Address subtype for W-4 creation using feature ADDRS), which allows only customized subtypes of IT0006 to be considered for W-4 records.

         

4. Business Function

For the ESS application to work properly, a prerequisite is to make sure the Business Function HCM_ESS_WDA_1 is activated on the system.
This case ben validated by transaction SFW5.

For further details on Business Function HCM_ESS_WDA_1, refer to the SAP Help, note 1921872 (ESS: W-4 Tax Withholding Form displays incorrect dates) and note 1580327 (ESS: W-4 Tax Withholding Exemption Texts in ABAP WebDynpro).

         

5. Custom View

The previous recommendations are the main points that may cause the issue.

In case your issue persists, please make sure that you are indeed using the standard delivery of the ESS view, as custom modifications/enhancements may also be the root cause.

See Also

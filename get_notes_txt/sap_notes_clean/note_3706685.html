Symptom

As part of SAP Integration Suite, managed gateway for spend management and SAP Business Network solution migration from SAP BTP NEO to SAP BTP Cloud Foundry, we want to know steps to add new  cloud foundry subaccounts in SAP Cloud Connector. 

Cause

SAP Business Technology Platform (BTP) has announced the end-of-life for SAP BTP NEO.  As such, applications such as Integration Suite, managed gateway for spend management and SAP Business Network must migrate to SAP BTP Cloud Foundry (CF).

Resolution

1.    Based on Managed Gateway for Spend & Network Data Center you are using, locate the subaccount details in corresponding section of article: Managed Gateway for Spend & Network - Migration from SAP BTP NEO to SAP BTP Cloud Foundry

2.    Login to you SAP Cloud Connector application where you have configured subaccounts to connect to Managed Gateway for Spend & Network application.

3.    Navigate to Home > Connector > Add Subaccount > Configure manually > Next

4.    Fill Region, Subaccount looking into article mentioned in point 1 above. Display Name can be of your choice.

5.    Login E-Mail enter following: $SAP-CP-SSO-PASSCODE$

6.    Password value is a passcode value generated when you login to the One-time Passcode URL mentioned in above article next to the subaccount you are adding.

    • Login E-Mail to use while login to One-time Passcode URL is your puser email which you see in your Managed Gateway for Spend & Network portal > My Configurations > Authorization and use corresponding puser password.

7.    Location ID value should remain same as your current prod or test connection. Finish

8.    You get message saying: Destination configurations on cloud side may have to be adapted when changing or setting location ID of Cloud Connector. Click OK

9.   Maintain Cloud to On-Premises in in SAP Cloud Connector settings for the newly added subaccounts to match those of current ones related to Managed Gateway.

10.  Repeat steps 1 -10 for next subaccount.

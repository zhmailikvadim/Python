Symptom

When using the Manage Purchased Product Footprints application, an error "Unit of measure mismatch" is displayed. This occurs when attempting to map an emission factor (e.g., with a Unit of Measure of KG) to a purchased product that has a different base Unit of Measure (UoM), such as PC (Piece) or ST.

The error may be confusing if a conversion rule from the product's base UoM to an intermediate unit (e.g., PC to G - Grams) already exists in the product master data. The system does not automatically perform a chained conversion (e.g., from PC to G and then to KG) and fails the mapping.

Environment

Sustainability Footprint Management

Reproducing the Issue

The error occurs when the system lacks a direct conversion path between a product's unit and an emission factor's unit.

1. Prerequisite Data Setup:

  • Define a product with a base Unit of Measure (UoM) that is not a standard mass unit (e.g., PC for Piece).
  • In the product's master data, create a quantity conversion rule from this UoM to an intermediate mass unit, like Grams (G). For example: 1 PC = 50 G.
  • Intentionally do not create a direct conversion rule from PC to KG.

2. Execution Steps:

  • Open the Manage Purchased Product Footprints application.
  • Initiate a new mapping by clicking Create.
  • Select the product defined in the setup above.
  • Choose an emission factor where the UoM is KG.

3. Expected Result:

  • The system will fail to create the mapping and will display the error: "Unit of measure mismatch".

Cause

This behavior is by design within the SAP Sustainability Footprint Management calculation engine. The root causes are:

  1. No Support for Chained (Transitive) Conversions: The system requires a single, direct conversion rule from the product's base UoM to the emission factor's UoM. It cannot link multiple conversion rules together (e.g., PC → G → KG). The system validation looks for a direct PC → KG rule and, if not found, triggers the error.

  2. Missing Prerequisite UoMs (Configuration Issue): In some scenarios, the error cannot be resolved because the required Units of Measure (either the source or target UoM) are not configured in the tenant. This is a configuration prerequisite that must be solved before the conversion rule can be created.

Resolution

To resolve this issue, a direct conversion rule must be created in the product master data. The process involves two steps:

Step 1: Verify Unit of Measure Availability Ensure that both the source UoM (e.g., PC) and the target UoM (e.g., KG) exist in the system. If they are missing, they must be configured in the tenant first.

Step 2: Create the Direct Conversion Rule via Master Data Import

  1. Calculate the direct conversion factor from the source UoM to the target UoM.

    • Example: If 1 PC of a product weighs 50 grams, the direct conversion factor to kilograms is 0.05 (since 50 / 1000 = 0.05).

  2. Navigate to the Import Master Data application and select the Product template.

  3. In the downloaded file, go to the QuantityConversions tab.

  4. Add a new row for the product with the direct conversion rule. Using the example above, the entry would be:

Product IDUoM of Source QuantityTarget QuantityUoM of Target quantity
[Your_Product_ID]PC0.05KG
  1. Save the template and import it back into the system using the Import Master Data app.

Once the import is complete, the system will recognize the direct conversion rule. The mapping in the Manage Purchased Product Footprints application will now succeed without the "Unit of measure mismatch" error.

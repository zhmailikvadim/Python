Symptom

While creating a PO in ME21N account assignment category is automatically filled.

Environment

  • Materials Management (MM)
  • SAP ERP Central Component
  • SAP ERP
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA

Reproducing the Issue

1. Create a PO with ME21N fill all the details except account assignment category
2. Save the PO
3. (Any) Account assignment category will be automatically filled.

Cause

  • The Business Function LOG_MM_OM_1 in ERP Operations is active in the system.
  • When the 'Real-Time Consumption' indicator (EINE-CO_ORDER) is set in the subcontracting info record, the system checks the table IAOM_OM_CSC for a default account assignment category for the PO order type.
  • If a default account assignment category exists, it is automatically assigned to the PO item.

Resolution

Navigate to: T-code SPRO -> Materials Management -> Outsourced Manufacturing -> Define Controlling Scenario

Here, you can verify if the account assignment category 'F' is maintained for the PO order type they are using. If you does not wish for the account assignment 'F' to be populated for this PO type, you could consider removing this setting.

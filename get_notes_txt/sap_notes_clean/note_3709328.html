Symptom

You need to delete/cleanup the expired entries from the SRMPROTOCOL table.

Other Terms

SRMPROTOCOL, SRM_PROTOCOL_DELETE, expired, cleanup, delete.

Reason and Prerequisites

This SRM_PROTOCOL_DELETE report will help cleanup expired logs from the table SRMPROTOCOL.

Solution

Kindly implement the SAP Note or upgrade to the relevant support pack.

The delivered report includes the following selection parameters, referenced from SRMPROTOCOL:

      1. POID Directory ID: POID ID whose logs are to be deleted.

      2. User: User ID that created the log.

      3. Activity Id: Identifies the activity of the log.

      4. Element Type ID: Specifies the SPS ID.

      5. Protect NOT_DEL = X: When checked, excludes protocol entries flagged as protected (NOT_DEL = 'X') from deletion.

      6. Test Run – When selected, the program runs in simulation mode and no logs are deleted.

      7. Delete Size – Defines the maximum number of logs deleted per processing block.

Based on the inputs above, the deletion is performed for the logs whose expiry date is lesser than the date on which this report is processed.

Please make sure that user has authorization for modify the table SRMPROTOCOL (Authorization Object S_TABU_NAM Activity '02').

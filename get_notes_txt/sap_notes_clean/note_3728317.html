Symptom

You are using the Instance Secure Store in the File System (Instance SSFS) as Server-Side Secure Store. This means that the SQL query returns the result 'Instance SSFS is active':

select  (case when (select  count(*) from "KEY_MANAGEMENT_CONFIGURATIONS") > 0 then 'LSS is active'  else 'Instance SSFS is active' end) from dummy;

Furthermore SYS.M_FEATURE_USAGE returns result IS_DEPRECATED = TRUE:

SELECT IS_DEPRECATED  FROM SYS.M_FEATURE_USAGE WHERE COMPONENT_NAME = 'SECURITY' AND FEATURE_NAME = 'ROOT KEY MANAGEMENT SSFS';

Reason and Prerequisites

As of SAP HANA 2 SPS09, the SAP HANA Instance Secure Store in the File System (Instance SSFS) as Server-Side Secure Store has been deprecated for SAP HANA.

 

Its successor is the Local Secure Store (LSS)

2764368 - Release Information for the SAP Local Secure Store

LSS supports the following security features, which Instance SSFS **NOT** supports:

  • Use of external key management systems for HYOK and BYOK scenarios 

  • Encryption algorithm with mode CTR for BACKUP, LOG, PERSISTENCE and APPLICATION root keys.

  • SAP HANA backups are self-contained with LSS. No additional root key backups will be required anymore (SAP Note 3539252)

Prerequisites:

  • You are using the Instance Secure Store in the File System (Instance SSFS) as Server-Side Secure Store. 

Solution

Transition from Instance Secure Store in the File System (Instance SSFS) to Local Secure Store (LSS) as SAP HANA Server-Side Secure Store.

  1. Follow the procedure in Activate the Local Secure Store (LSS)

  2. Verification: The SQL query returns 'LSS is active':

select  (case when (select  count(*) from "KEY_MANAGEMENT_CONFIGURATIONS") > 0 then 'LSS is active'  else 'Instance SSFS is active' end) from dummy;

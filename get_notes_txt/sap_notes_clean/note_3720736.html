Symptom

SWPM fails with error :

Command CRTR3SYS SID(<SID>) GLOBALHOST(*LOCAL) NUCUC(*YES) failed. Return: 65280; CPF0006

Where <SID> is the system ID of the SAP system eg: PRD

Environment

.

Reproducing the Issue

Run SWPM tool

Resolution

Run the failing command at OS level and check joblog for details (DSPJOBLOG)

1) Logon as SAPIUSR

2) Run command ADDLIBLE SAP<SID>IND   

3) CRTR3SYS SID(<SID>) GLOBALHOST(*LOCAL) NUCUC(*YES)

(Where <SID> is the system ID of the SAP system eg: PRD)

4) Check joblog with DSPJOBLOG,  then hit F10 key, then shift + F6 key 

5) Page up until you find the error.

Previous customer cases were due to error  Object already exists. Object is /usr/sap/trans
__________________________________________________________


The symbolic link /usr/sap/trans exists on your system and it should point to /sapmnt/trans

If it does not, then run the command below to rename the symbolic link

RNM OBJ('/usr/sap/trans') NEWOBJ('/usr/sap/trans.old') 


Afterwards, please retry the step in SWPM.

If there are still issues, please open a support case with component BC-INS-AS4

Symptom

The object type Position with the A003 relationship (Org Unit) is being delimited according to the A008 Person relationship dates, which is not the desired behavior.

Environment

  • SAP ERP
  • SAP S/4HANA on-premise
  • SAP S/4HANA private cloud edition
  • SuccessFactors Employee Central

Reproducing the Issue

  1. The employee is assigned to position X.
  2. When the employee’s end‑of‑employment date is changed, the A008 relationship in PP01 becomes restricted according to the EoE dates.
  3. Additionally, other relationship types (e.g., A003 OrgUnit) are also being limited according to these dates.

Cause

The system is customized so that the relationship A003 is updated via both employee replication and organization replication, which can cause issues. Specifically, the flag SFSFI OANSO is empty (no value), meaning the relationship position-org unit 003 is being updated by both replications at the same time.

Resolution

  1. Go to transaction SPRO -> IMG guide -> Personnel management -> Integration with SuccessFactors Employee Central -> Business Integration Builder -> Organizational Data Integration -> Restrict Relations Created When Replicating Organizational Assignments.
  2. Set the flag SFSFI OANSO to X in the relevant system.
  3. For more detailed information, refer to the guide: Excluding position relations from organizational assignment replication

Symptom

  • When creating a custom field using the Custom Field Application, the field is not available in the configuration Define Copy Behaviour of Fields in Material Master (SSCUI 10649).
  • This issue limits the usability of reference material and reference sites in Retail for maintaining master data after go-live.
  • Attempting to add the missing entry in table T130F through configuration results in display mode only.
  • Error occurs when searching for or entering the field custom field.

Environment

SAP S/4HANA Cloud Public Edition

Reproducing the Issue

  1. Log in to the S/4HANA Central Business Configuration.
  2. Navigate to General Business Data
  3. Select Products
  4. Select Materials
  5. Open Central Configuration for material master : Define Copy Behaviour of Fields in Material Master (ID 10649).
  6. Select New entry.
  7. Enter the custom field.
  8. Or, search for the field custom field in the list.
  9. Observe the error.

Entry Custom Field does not exist in T130F (check entry)

Alternate steps:

  1. Log in to the S/4HANA Central Business Configuration.
  2. Navigate to General Business Data
  3. Select Products
  4. Select Materials
  5. Open Central Configuration for material master : Assign field to field selection groups
  6. It is ready only

Cause

  • Custom fields are not registered in the central field table T130F for Material Master Maintenance.
  • Missing entries in the copy behavior configuration table T130F with proper reference indicator (KZREF).
  • Field selection group assignment is not configured for custom fields.

Resolution

  1. The field needs to be maintained in table T130F before it can appear in the configuration Define Copy Behaviour of Fields in Material Master.
  2. Note that the configuration Define Copy Behaviour of Fields in Material Master is specific to only Retail, as mentioned in the configuration help:


  3. Understand that in S/4HANA Public Cloud, custom fields are created using the Custom Field Application and are not available in the classical MM01/MM02/MM03 apps.
  4. Refer to the recommendation to use "Manage Product Master Data" instead, which supports custom fields using extensibility but does not allow the addition of custom fields to table T130F.
    https://help.sap.com/docs/SAP_S4HANA_CLOUD/f86dc2eb1f8b48c880a7607213104b27/577c65a371a543d68a36285…
    Section Add Custom Fields using Extensibility
  5. The BADI's for product master CMD_PRODUCT_VALIDATE / BD_CMD_PROD_DATA_API_CHECK_2 are for generating errors / warnings hence, it is not possible to influence field contents .
  6. Check out this SAP KBA -  2963059 - How to Submit Feature Requests for SAP S/4HANA Cloud Product to know more about how to submit feature request in Customer Influence.

Symptom

  • Professional Tax (PTax) calculation is showing a discrepancy in the system.
  • For the month of February, the system calculated the total PTax amount as 2600, whereas the expected yearly amount should be 2500.
  • WT /422 for February showing 2600, for March the calculation appears correct and shows 2500.

Environment

  • Human Capital Management (HCM)
  • SAP ERP Core Component
  • SAP ERP
  • SAP enhancement package for SAP ERP

Reproducing the Issue

  1. Hire an employee in Monthly PTAX Deduction state say for example in State Maharashtra or Karnataka
  2. Execute the payroll calculation using transaction code PC00_M40_CALC.
  3. Review the Professional Tax (PTax) calculation for the month of February.
  4. Observe that the system calculates the total PTax amount as 2600 instead of the expected yearly amount of 2500.
  5. For March the calculation appears correct and shows 2500.

Resolution

The system takes the actual Professional Tax (PTax) deductions for the already executed period and projects the current month's PTax amount from the remaining pay periods to calculate the annual PTax deduction amount (/422).

For February, the system calculates the value as follows:

Actual value from April to January: 2000 (200 x 10 months) + 300 (Current month's value) + 300 (Projection of the current month) = Total: 2600

For March, the system adjusts the calculation:

Actual value from April to February: 2300 + 200 (Current month's value) = Total: 2500

This behavior is standard functionality of the system during projection. The system automatically adjusts the values based on actual deductions and performs annual projections to calculate the annual income tax value.

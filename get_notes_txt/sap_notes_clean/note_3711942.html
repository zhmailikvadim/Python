Symptom

In report RSDBSPJS table VARINUM is accessed when a background job is scheduled to create a temporary selection variant (starting with "&").

In case of rare severe system insufficiencies, e.g. a DB connection error with respect to a secondary connection, an SQL exception may occur, which cannot be handled properly in the program flow.

The previous implementation, which just raises the received SQL exception, is not the best choice, as there is no one to handle the exception and the effected shortdump RAISE_EXCEPTION does not reflect that the situation cannot be handled and the program must be aborted necessarily.

Reason and Prerequisites

Issue in ABAP code. A correction is required.

Solution

Apply the provided correction or upgrade to the support package level, that is assigned to this note. After that, a raise shortdump is triggered.

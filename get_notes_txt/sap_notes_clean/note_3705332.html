Symptom

Access a custom field, and check a tab (such as User Interface, Analytics, etc.) but there is no specific CDS View to enable.

Environment

  • SAP S/4HANA
  • SAP S/4HANA Cloud Private Edition

Reproducing the Issue

  1. Access the "Custom Fields" application.
  2. Locate a custom field created with the related business context.
  3. Select a tab (such as User Interface or Analytics) then could not find a specific CDS View to enable.

Cause

The OData Model for that specific CDS View is not enabled.

Resolution

Check if CDS View is extended from which business context:

  1. Go to transaction SE80.
  2. Select Development Object.
  3. Search for that specific CDS View with Object type "Extensibility CDS Context", then click on the result.
  4. Select Support of Custom Fields, if there is a tick in "Is Invisible" box, this means the view is currently hidden. 
  5. Then, check "Related Business Context" to see if the listed business context(s) is the same with the one created for the Custom Field.

On SAP GUI, please follow below steps to check for OData Model:

  1. Access to transaction SCFD_REGISTRY.
  2. Access tab "Extensible CDS View", then search for the specific CDS View.
  3. If there is a result, select it and click on the button "Where Used". => The Object name is the OData Model.
  4. Access the tab "Extensible OData Model" in the same transaction, then search for the OData Model.
  5. It will display a cross in either of 3 columns (UI Report / Form / API) => This is the tab to enable the OData Model on the Custom Field.
    • UI Report -> User Interface
    • Form -> Form Templates
    • API -> SOAP APIs

On the Custom Field application, please follow below steps to enable the OData Model:

  1. Access to the Custom Field with related business context that needs to be extended with the CDS View.
  2. Select the relevant tab in above step 5 on SAP GUI, then enable the relevant OData Model.
  3. After enabling, the CDS View should be displayed in the Custom Field to extend.

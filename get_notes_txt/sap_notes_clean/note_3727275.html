Symptom

When creating routings with operations referencing from reference operation set using Engineering Workbench functionalities, the PLPO-WERKS field is not populated. 

The issue occurs specifically when using Engineering Workbench functions like t-code CEWB or BAPI BAPI_ROUTING_CREATE instead of classic CA0X t-codes.  

Environment

  • SAP ERP Logistics (LO)  
  • SAP R/3
  • SAP R/3 Enterprise
  • SAP ERP Central Component
  • SAP ERP
  • SAP Enhancement package for SAP ERP
  • SAP Enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4 HANA on premise
  • SAP S/4 HANA Cloud Private Edition

Reproducing the Issue

  1. Create routing operations referencing from reference operation set using Engineering Workbench functions. 
  2. Field PLPO-WERKS is not updated. 

Cause

Engineering Workbench functions process data differently compared to classic CA0X transaction codes.

Resolution

The system behaves as standard, and no action is required. 

The behavior difference does not impact SAP standard functionalities since the referenced operations are fetched directly from the reference operation set. 

If the PLPO-WERKS field needs to be populated, consider creating a BDC recording for t-code CA0X instead of using the Engineering Workbench function modules.

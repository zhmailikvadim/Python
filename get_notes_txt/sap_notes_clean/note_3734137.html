Symptom

When using transaction FB02 to edit a parked vendor invoice, the customer number entered in the G/L account line item is removed after the system displays warning message F5284, "VAT registration number was transferred from customer master record."

Environment

  • Financial Accounting (FI)
  • SAP ERP
  • SAP ERP Central Component
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA
  • SAP S/4HANA Finance
  • SAP S/4HANA Cloud Private Edition
  • SAP Fiori for SAP S/4HANA

Reproducing the Issue

  • Go to T-code: FB02 for a parked vendor invoice and select "Edit Document."
  • Access the G/L account line item and click on "More Data."
  • Input a Customer number along with a Reporting Country.
  • Press Enter; the system triggers warning F5284 stating, “VAT registration number was transferred from customer master record.”
  • Upon confirming the warning, the previously entered Customer number in the G/L line item is erased and no longer visible.

Cause

This behavior is a result of SAP's standard design. The Customer field in "Additional Data" for G/L line items in T-code:FB02 is intended exclusively for deriving the VAT Registration Number using the Reporting Country. The entered Customer number is not stored at the line-item level and won't be displayed post derivation. 

Resolution

This is the expected system behavior. If it is necessary to store the customer number in FI G/L line items, users may consider implementing a custom FI substitution to populate the field BSEG-KUNNR. However, per SAP Note 842318, the field KUNNR is not officially released for substitution purposes. Any custom solution enabling substitution for this field should be approached with caution, thoroughly tested in a non-production environment, and implemented at the user's own risk and discretion.

Symptom

When running Supply Planning Run or Projected Stock app in a planning area with Characteristic-Based Planning (CBP) enabled in Order-Based Planning (OBP), the forecast quantities fail to be read.

The following error is raised:

  • Reading Forecast with CBP attributes requires multi-KF setting enabled.

In Application Mapping App, the forecast key figure is currently assigned to the single forecast field "Forecast" and not to any of the multiple forecast fields (for example, "Forecast (PR1)", "Forecast (PR2)", etc.).

Reproducing the Issue

  • SAP Supply Chain Management (SCM)
  • SAP Integrated Business Planning (SCM-IBP)

Cause

As described in CBP restriction note 3422877, automatic consumption during the supply planning run is not supported for Characteristic-Based Planning (CBP).

Therefore, if a key figure is assigned to the “Forecast” field in Application Mapping (single forecast), the system raises an error when reading forecast data in CBP-enabled scenarios.

Resolution

A multiple key figure configuration must be used. In Application Mapping, configure the forecast as follows:

  • To use a single forecast, assign the key figure to "Forecast" field.
  • To use multiple forecast, assign Key Figures to "Forecast (PR1)", "Forecast (PR2)", and so on…, instead of "Forecast".

Important:

  • The "Forecast" field (single key figure) and "Forecast (PRx)" fields (multiple key figures) cannot be used simultaneously.
  • Using the multiple key figure setup disables automatic consumption. Forecast consumption by sales orders can instead be handled using the IBP_CONSUMPTION function for simplified calculation.

Symptom

When trying the view details in Native Storage Extension application of HANA Cockpit, you get error like below:

Data Request Failed
Error: Internal Server error
com.sap.db.jdbc.exceptions.RTETimeoutException: Data receive failed (socket timeout) 

or
Data Request Failed Error: Internal Server Error Data receive failed; socket timeout [null].: com.sap.db.jdbc.exceptions.RTETimeoutException: Data receive failed; socket timeout [null].

Environment

HANA Cockpit 2.0

Cause

The performance of NSE-related functions is highly dependent on the size of the HANA database. It may take time to fetch the details and triggers the timeout.

Resolution

  1. Open Cockpit Manager.
  2. Click "Cockpit Settings" on the left menu, then select "Connections".
  3. Click "Edit" button at the bottom, then change the "Standard database connection timeout period (seconds)" to an appropriate value, such as 200 seconds or higher.
  4. Click "Save" button at the bottom, then refresh the Cockpit page after saving.

Note: It may take a while for the setting to take effect.

Keywords

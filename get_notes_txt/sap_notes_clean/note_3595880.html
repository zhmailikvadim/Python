Symptom

The following symptom is identified when you run the HR-PT: gerador do processamento da folha de pagamento (RPCALCP0) report:

  • You run a retrocalculation for a period in a previous year and there is a payroll period split in January of the year for which you are triggering the calculation. While recalculating the payroll periods, the Nº atribuição variável (V0ZNR) of the wage types that propagate the differences from previous years to the in-period is increased twice during the split January periods, leading to reporting inconsistencies as the reference year for the adjustment is one year prior to what it should be. 

Other Terms

Transaction: PC00_M19_CALC

(for more information, see SAP Note 2197631)

Reason and Prerequisites

Reason

This issue is caused by the Regra de cálculo para pessoal (PR06) rule because it is not prepared to correctly propagate previous years adjustments in case of payroll split in January.

Prerequisite

The following Support Package (SP) level is the minimum prerequisite:

ReleaseSupport Package
S4HCM 101 HR SP 00
S4HCM 100 HR SP 07
SAP_HR 6.00 HR SP M4
SAP_HR 6.04 HR SP J0
SAP_HR 6.08 HR SP B8

Solution

From now on, the PR06 rule is modified so that it increases the V0ZNR only in the first partial period of the split payroll result of January.

Example:

Proceed with the following steps to reproduce the symptom mentioned in this SAP Note.

Description

CALC: Retroactive changes for previous year run in January of following year with payroll split in January. 

Preparation/Background

1. Hire an employee on 01/01/2024 with salary EUR 2000.

2. Change employer starting from January 15th 2025 to trigger a payroll split.  

3. Run payroll until 01/2025.

4. Change employee's salary to EUR 3000 starting from 12/2024.

Execution

1. Execute payroll for 02/2025 with retroactive accounting to 12/2024. 

Result

After the execution of step 1, the following wage types must be generated

1. In the result table of 12/2024:

Wage type Amount (EUR)
/106 2000
/A06 1000

2. In the result table of the first split of 01/2025:

Wage type Amount (EUR) V0ZNR
/106 1304.35  
/Z06 1000 01

3. In the result table of the second split of 01/2025:

Wage type Amount (EUR) V0ZNR
/106 1695.65  
/Z06 1000 01

4. In the result table of 02/2025:

Wage type Amount (EUR) V0ZNR
/106 3000  
/107 1000 01


Note: These are the expected results once the SAP Note is installed.

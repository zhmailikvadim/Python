Symptom

When building Composable Storefront the build fails and in the logs you could see:

INFO d.h.c.e.NpmrcGenerator [main] Generating npmrc file in /opt/workspace/build/customer-project/js-storefront/spartacusstore/.npmrc to enable repositories [spartacus-dev]
...
INFO d.h.c.e.NpmrcGenerator [main] Repository spartacus-dev in known, url: https://73554900100900004337.dev.npmsrv.base.repositories.cloud.sap
...
INFO d.h.c.h.PrintOutputHandler [Thread-29] sterr: error Couldn't find any versions for "@spartacus/asm" that matches "~221121.1.0"
...
ERROR d.h.c.e.Executor [main] [FAILED] - Command yarn failed on retry 1 of 5, waiting 5 seconds before next attempt
de.hybris.ccv2.BuilderException: Executing of external command failed with exit code: 1

The same can happen with other versions, e.g.

INFO d.h.c.h.PrintOutputHandler [Thread-29] sterr: error Couldn't find any versions for "@spartacus/asm" that matches "~221121.2.0"

Environment

SAP Commerce Cloud

Cause

The cause of the issue is the usage of the:

"enabledRepositories": ["spartacus-dev"]

inside js-storefront/manifest.json in your repository. This can happen if you copy sample repository that is mentioned on help page: SAP Commerce Cloud Sample Repository.

Resolution

Please check help page: Add Applications to JavaScript Storefronts, especially point 7:

Enable SAP Commerce Cloud, composable storefront by adding the following data to the manifest file

{
    "applications": [
        {
            "name": "yourAppName",
            "path": "yourAppName",
            .
            .
            "enabledRepositories": [
                "spartacus-6-cdn"
            ]
        },
     ...
    ]
}

In order to resolve the issue pleas echange enabledRepositories in your js-storefront/manifest.json from spartacus-dev to spartacus-6-cdn.

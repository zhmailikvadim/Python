Symptom

Integrated asset acquisition in Materials Management does not fill ZEKKN field (Sequential Number of Account Assignment) in the APC account but it is filled in the Technical Clearing Account for Integrated Asset Acquisition account.

 

 

 

Environment

SAP S/4 HANA

Resolution

With new Asset Accounting in S/4HANA, integrated postings are split by the system into the operational part and the valuating part.
The integrated asset acquisition from Material Management via MIGO now only hits a special technical clearing account, called Technical Clearing Account for Integrated Asset Acquisition

In a second step the program will create additional, ledger-group-specific postings that transfer the value from the clearing account to the final asset APC account.

From a technical perspective, the system generates ledger-group-specific documents for each accounting principle.

Sequential number from the purchasing document ZEKKN can be find in the posting on the Technical Clearing Account for Integrated Asset Acquisition account. Only this document is processed by MIGO.

The ledger-group-specific postings however, which finally transfer the value to the asset APC account, get internally created by Asset Accounting and are not processed by MIGO.

These ledger-group-specific postings never receive the ZEKKN value! As this transfer is done by Asset Accounting itself, we can only update fields which are available in Asset Accounting and ZEKKN is not one of them.

Therefore, with new FI-AA posting logic it is not possible to get the ZEKKN field posted in the ledger-specific documents that post on the asset APC account.

Symptom

  • Subsequent booking of incorrect customer orders with transaction code KE2B is not possible.
  • When a customer order entry (KEKF) is activated, a faulty customer order cannot be reposted into the CE9 table with transaction type A using KE2B.

Environment

  • Product: SAP S/4HANA
  • Profitability Analysis
  • SAP FIORI
  • SAP R/3
  • SAP R/3 Enterprise
  • SAP ERP Central Component
  • SAP ERP
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA Finance

Reproducing the Issue

  1. Run transaction code KE2B for a specific order.
  2. There is no update preformed

Cause

KE2B actually only works for the cost-based profitability analysis.

Resolution

  1. For the combined profitability analysis, transaction code KE4T should be used.

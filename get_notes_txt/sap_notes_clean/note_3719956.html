Symptom

In Validation Rule configuration V2(hrpy_pcc_cvr_v2), the check message 'Alert dependency "" is not allowed' displayed, but the problematic alert dependency does not display under section 'Dependencies'.

Therefore this error cannot be resolved in the application.

When migrating validation rule v1 to validation rule V2, the migration report (PYT_MIGRATE_VR) reports error "Dependency (XXX) does not exist."

Other Terms

PCC, hrpy_pcc_ctl_v1, hrpy_pcc_ctl_v2

Reason and Prerequisites

For Validation Rule configuration V2(hrpy_pcc_cvr_v2), the invalid alert dependency (Validation rule) will not display under section 'Dependencies', but it will included in the consistency check.

For Validation Rule configuration V1(hrpy_pcc_cvr_v1), there is no consistency check for the dependencies.

Solution

Import the related Support Package or implement the attached correction instruction according to your software component.

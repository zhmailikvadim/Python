Symptom

After enabling the option "Dynamic User Creation" under Step 3 of System->Administration->Security, users are not being created when they logon for the first time, but instead get the error "Unable to authenticate your credentials"

Environment

  • SAP Analytics Cloud 2026.2.6

Reproducing the Issue

  1. Enable Dynamic User Creation option
  2. Attempt to logon as a new user
  3. Get error "Unable to authenticate your credentials"

Cause

This can happen if there are any default roles set to use a concurrent license when the tenant no longer has any concurrent licenses available.

Resolution

To resolve the issue, go to security->Roles, and for each role that has been designated as a default role do the following:

  1. Click the gear option in the upper right and uncheck the option "Use as default role"
  2. Save the role
  3. Click the gear option in the upper right and check the option "Use as default role" back on
  4. Save the role

This will change the role back to a named user license as concurrent licenses are not available on the tenant to be assigned to.

Symptom

Since the controller of the bill correction does not get initialized upon screen navigation the controller saves the last state of the searchpage radio button, causing issues when identifying the correct searchpage view to display. When navigating to the bill correction the view now always starts with the crm searchpage as default.

Reason and Prerequisites

CL_IUICBICO_SEARCHPAGE, WD_CREATE_CONTEXT, SEARCH_TYPE

Solution

Please implement the correction instructions or install the corresponding support package.

Symptom

Error appears when opening the manage pending hires onboarding page with the following error - [COE0019] MPH2ONB1Entity: duplicated property name jobInfo_department.

Environment

SAP SuccessFactors Onboarding

Reproducing the Issue

  1. Go to the Manage Pending Hires page.
  2. In the dropdown, go to the onboarding section.
  3. Observe that the page fails to load and returns with the error [COE0019] MPH2ONB1Entity: duplicated property name jobInfo_department.

Cause

A duplicate “department” field (jobInfo_department) is configured in the Manage Pending Hires Column Configuration (MPHColumnConfiguration), causing a duplicated property in MPH2ONB1Entity.

Resolution

  1. Open the Manage Pending Hires column configuration (MPHColumnConfiguration) in tmanage data.
  2. In the dropdown, select onboardingCandidate.
  3. Remove the duplicate department entry so that only one JobInfo > Department remains.
  4. Save the configuration.
  5. Reopen the Manage Pending Hires page and confirm the error no longer occurs.

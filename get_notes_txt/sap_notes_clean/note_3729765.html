Symptom

  • A Portfolio Item has no active decision points.
  • All item decision points show blank status.
  • It is not possible to change the status of any decision point in the item.

Environment

  • SAP Portfolio and Project Management
  • SAP Portfolio and Project Management for SAP S/4HANA
  • SAP S/4HANA
  • SAP S/4HANA Cloud Private Edition 

Reproducing the Issue

  1. Open an affected Portfolio Item.
  2. Observe that there are no active decision points. All decision points have a blank status.
  3. It is not possible to set a status in any decision point.

Cause

There are currently no know causes in Portfolio Management application for this inconsistent state.

Resolution

Consistency in the affected items can be restored as follows:

  1. Access the Portfolio Management frontend with an authorized user (with "Admin" ACL authorization for the Portfolio Items).
  2. Open the affected portfolio item.
  3. Open the last decision point.
  4. Press the button "Revert to Previous Decision Point" and confirm the popup.
  5. Open the newly activated (second last) decision point.
  6. Press the button "Revert to Previous Decision Point" and confirm the popup.
  7. Repeat these steps until the first decision point becomes active.
  8. The first decision point can now be processed per the standard business process.

Note: 

A different scenario is: there are no active decision point in an item, but the decision points have a valid status. If a portfolio item has this state, running program RPM_ITEM_CONSOLIDATE for the item with flag "Active decision point GUID" will update the "active" flag for decision points in accordance with the decision point status.

Symptom

  • Customer is evaluating whether Microsoft 365 (M365) SMTP servers can be used for inbound and outbound email communication in SAP Service Cloud Version 2.
  • Clarification requested regarding compatibility
  • No explicit confirmation found in public documentation

Environment

  • SAP Service Cloud Version 2 1.0
  • SAP Sales Cloud Version 2 1.0

Cause

  • SMTP integration depends on alignment between authentication methods supported by both systems
  • Microsoft has introduced changes in Exchange Online security policies, moving toward modern authentication (OAuth 2.0) and reducing support for Basic Authentication

Resolution

Current Behavior

  • SAP Service Cloud Version 2 supports SMTP connectivity using Basic Authentication
  • M365 environments may allow SMTP usage if Basic Authentication is enabled at tenant level

Important Consideration

  • Microsoft’s direction toward modern authentication standards may impact SMTP configurations that rely on Basic Authentication
  • This is part of a broader industry-wide security evolution, not specific to a single platform

Recommended Approach

  • Customers are advised to:
    • Verify authentication capabilities within their M365 tenant
      • For M365, please check the "Updated Exchange Online SMTP AUTH Basic Authentication Deprecation Timeline"
    • Consider other SMTP services compatible with their organization’s security policies
    • Align email infrastructure with supported authentication mechanisms across systems

Product Direction

  • SAP continuously evaluates enhancements to support modern authentication methods, including OAuth 2.0, to ensure alignment with evolving industry standards
  • Customers can share feedback or track related enhancements via the Customer Influence Portal: Emails: SMTPAUTH over TLS as Authentication Method

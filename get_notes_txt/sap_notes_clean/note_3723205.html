Symptom

  • During the activation of a booster in SAP BTP Global Account, the system does not find the correct IAS tenant.
  • The expected IAS tenant is correctly linked in the subaccount's Trust Configuration but is not being recognized during the booster activation process.

Environment

Provisioning

Reproducing the Issue

  1. Navigate to SAP BTP Global Account.
  2. Search for the booster and click on "Start".
  3. In Step 2, select the desired subaccount and proceed with the next steps.
  4. Observe that the booster does not find the correct IAS tenant.

Cause

The issue occurs due to the incorrect configuration of the "Customer records" field in the IAS Admin Cockpit under "Applications & Resources".

Resolution

  1. Access the Cloud Identity Services Admin Cockpit.
  2. Navigate to "Applications & Resources".
  3. Select the relevant tenant.
  4. Go to "Tenant Information".
  5. In the "Additional Information" section, add the correct Customer ID in the "Customer records" field.
  6. Verify that the booster now recognizes the correct IAS tenant.

Setting: Configure additional customer records to make this tenant visible and selectable in tools like SAP BTP Cockpit and SCI Tenant.

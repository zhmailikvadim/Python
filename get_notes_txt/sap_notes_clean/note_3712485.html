Symptom

You are running Report R_MMPUR_RPO_START_PROCESSING to create purchase orders. When running the report, it does not create purchase orders while processing data from the RPO_D_ITEM_BUF table. Entries are created in the RPO_ITEM_D_MAP table without purchase order numbers. When checking log in Tcode SLG1 you see Error message MMPUR_RPO_MESSAGES044 - "Item with hash key XXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXX and material YYYYYYY for purchase order is obsolete" displayed.

Environment

  • SAP S/4HANA
  • SAP F&R-SAP ERP Interfaces

Cause

The validation of whether order list items are considered obsolete is configured in the customizing settings under: SPRO -> Materials Management -> Purchasing -> Order Optimizing -> Grouping of Order List Items into Purchase Orders -> Define Control Characteristics. 

The validation is performed based on the Control Code 0CBR, where the Buffer Time for Order List Items is defined in minutes. By default, the ORDERING_TIME in the RPO_D_ITEM_BUF table is set to the beginning of the day in UTC time. When the report R_MMPUR_RPO_START_PROCESSING is executed at a later time than ORDERING_TIME + Buffer Time for Order List Items, the system evaluates the items as obsolete and issues the error message MMPUR_RPO_MESSAGES044.

Resolution

To avoid the error message you can resolve the issue by choosing one of the possibilities mentioned bellow:

  1. Increase the Buffer Time for Order List Items in the customizing settings;
  2. Modify the ORDERING_TIME via BADI_MMPUR_RPO_GRP (Enhancement of Business Logic in Grouping API) to set ORDERING_TIME to a later time, ensuring items are not evaluated as obsolete when the report R_MMPUR_RPO_START_PROCESSING is executed;
  3. Implement customer-specific logic for processing obsolete Order List Items using the BADI_MMPUR_RPO_PROC methods PROCESS_ITEMS_FOR_RELEASED_PO or PROCESS_ITEMS_OBSOLETE_TRANSFER.

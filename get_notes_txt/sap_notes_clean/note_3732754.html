Symptom

You implemented the BAdI /SCMTMS/BADI_LI_BTD_CHG~ADAPT_LI_INPUT_EWM and cleared all warehouse data in order to skip the update from EWM to TM in Adv. SR integration. Despite clearing the input data, the integration method continues processing resulting into a runtime error instead of terminating early.

Other Terms

Adv. SR; EWM-TM Integration; Direct EWM-TM Integration;

Reason and Prerequisites

Due to a programming error, the integration method does not immediately terminate when warehouse input data is cleared in the BAdI /SCMTMS/BADI_LI_BTD_CHG~ADAPT_LI_INPUT_EWM. Instead, the method continues processing, which leads to a runtime error during subsequent operations.

Solution

Implement the attached correction instruction or install the corresponding support package. Afterwards, when input data is cleared in the BAdI, the integration method will terminate immediately without further processing, preventing the runtime error.

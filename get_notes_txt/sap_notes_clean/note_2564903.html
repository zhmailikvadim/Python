Symptom

SAML2LoginModule is enabled in SAP NetWeaver Administrator (NWA).

While trying to connect a Knowledge Management (KM) repository (e.g. https://<host>:<port>/irj/go/km/docs/ ) with WinSCP as a WebDav Client, it is not working as long as SAML2LoginModule is active.

In WinSCP only blank screen is displayed.

Removing SAML2LoginModule, the KM content is displayed in WinSCP.

Environment

  • EP Release Independent
  • SAP NetWeaver
  • SAP Composition Environment

Reproducing the Issue

SAML2LoginModule is enabled in SAP NetWeaver Administrator (NWA) (https://<host>:<port>/nwa/auth):

The "Enable WEbDAV server" is selected in WebDAV Protocol setting in System Adminsitration -> System Configuration -> Knowledge Management -> Content Management -> Protocols -> Show Advanced Option -> WebDAV Protocol

While connecting to KM content in WinSCP, a blank screen is displayed, KM content is not available:

After removing the SAML2LoginModule from "ticket" configuration in NWA, the KM content is available in WinSCP.

Cause

The current latest WinSCP version (5.11.2) can not handle ?saml2=disabled option.

Resolution

1. Disable the SAML2LoginModule in NWA if continuing to use WinSCP as the WebDAV client for KM content

2. Another option would be to use SAP Portal Drive. The "?saml2=disabled" option is handled in SAP Portal Drive 6.0 Patch1 and above.
    The parameter ”?saml2=disabled" must be added manually to each Portal URL, for each connection item in Portal Drive during setup of connection to corresponding Portal.

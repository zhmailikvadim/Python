Symptom

  • Archiving write for FI_TF_GLF ends with runtime error DBSQL_STMNT_TOO_LARGE in transaction ST22.
  • Short text: “Open SQL command is too big.”
  • Exception: CX_SY_OPEN_SQL_DB.
  • ABAP program: CL_FML_AEXT_ARCHIVING=========CP (also referenced: CL_FML_AEXT_ARCHIVING=========CM00N).

Environment

  • Controlling (CO)
  • SAP S/4 HANA 2021.
  • SAP S/4 HANA 2022.
  • SAP S/4 HANA 2023.

Reproducing the Issue

  1. Check ST22 for a dump in program CL_FML_AEXT_ARCHIVING=========CP with error DBSQL_STMNT_TOO_LARGE during FI_TF_GLF archiving.
  2. Run the archiving report FI_RF_GLF_WRI for object FI_TF_GLF using a variant that includes Material Ledger–related accounts and observe the runtime error.

Cause

The Open SQL statement becomes too large during selection in CL_FML_AEXT_ARCHIVING (SELECT_CKMLHD) when processing large Material Ledger–related data ranges in the FI_TF_GLF write phase.

Resolution

  1. Implement the correction provided in SAP Note 3669814 “FI_TF_GLF_WRI: Acdoca archiving dumps on memory allocation limit”. 
  2. If implementing the above SAP Note doesn't resolve your issue, please contact SAP Support to ask for authorization for pilot note 3367888 “Runtime error ‘SAPSQL_STMNT_TOO_LARGE’ when archiving”.

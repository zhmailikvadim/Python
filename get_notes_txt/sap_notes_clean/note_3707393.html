Symptom

Custom condition types added in Purchasing are not displayed in the Purchasing Info Record.

Environment

SAP S/4HANA Cloud Public Edition

Reproducing the Issue

  1. Add new custom condition types in purchasing.
  2. Attempt to view the condition types in the purchasing info record using transaction ME11.
  3. Observe that the condition types are not displayed.

Cause

The "Set Calculation Schema" for the defined procedure for the newly added condition types is not maintained correctly.

Resolution

  1. Verify that the "Set Calculation Schema" for the defined procedure for the newly added condition types is maintained correctly.
  2. Ensure that the manual column in the control data for the procedure is configured appropriately.

See Also

Refer to: 3484191 - Conditions missing/ Net price empty in Manage Purchasing Info Records app - SAP for Me

Symptom

  • When pasting a value that starts with an asterisk (*) into a value help criteria field, the input is converted into a search token and the field cannot be clicked into again to make additional entries or changes.
  • The behavior reproduces in classic GUI transactions accessed via WEBGUI and FLP; it is not observed in SAP GUI for Windows.

Environment

WebGUI / SAP GUI for HTML

Reproducing the Issue

  1. Open the value help tab in any transaction on WebGUI.
  2. Paste a value beginning with an asterisk (for example, *TEST) into any field.
  3. Observe that the value becomes a search token and the field cannot be edited directly by clicking into it. The token shows like this:

         

Cause

  • This is the new feature as of the correction in 3537645 - Paste to valuehelp criterion
  • Intended behavior of the value help criteria section: pasting a value starting with "*" converts the entry into a token and switches the search logic to "contains".

Resolution

  1. To modify the criterion after tokenization, open the search options dialog for the affected input (use the options icon next to the field) and edit the token there.
  2. If exact matching is required, change the search logic from "contains" to "equals to" in the search options dialog.
  3. If inline editing is needed, remove the token and re-enter a value without a leading asterisk (*), then adjust the search logic as desired.

Symptom

When migrating SAP Business One hosted on the Microsoft SQL Server to SAP HANA, you find the migration fails, and find errors in the logs containing the following: inserted value too large for column.

Reproducing the issue

Prerequisites:

There are certain fields in the Microsoft SQL Server database that contain no values.

Scenario:

You are using the migration tool to migrate SAP Business One hosted on the Microsoft SQL Server to SAP HANA.

Expected behavior:

The migration runs successfully.

Actual behavior:

The migration fails, and there are errors in the logs containing the following: inserted value too large for column.

Solution

SAP intends to provide a feature package or patch to solve the symptom that is described in this SAP Note. The section References below lists the related feature package or patch once they become available. Be aware that References can only be confirmed at feature package or patch release date. SAP delivers feature packages or patches only for selected releases at its own discretion, based on the business impact and the complexity of implementation.

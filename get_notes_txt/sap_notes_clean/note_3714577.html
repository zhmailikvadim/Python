Symptom

The working directory of a locally called external command is not the 'work' directory any more. The 'work' directory is the directory, which is contained in the profile parameter DIR_HOME.
The new behavior is not a bug, because a certain working directory has never been guaranteed. Nevertheless many customer applications rely on this.

Other Terms

working directory, SNC

Reason and Prerequisites

The change of the working directory was a side effect of a correction.

Solution

This note delivers a code change in the external commands framework. The framework now uses the new RFC API for local calls of external commands. Apart from technical advantages it has the consequence, that the working directory is the 'work' directory again.

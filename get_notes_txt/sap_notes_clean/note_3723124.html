Symptom

When changing the plant in a Purchase Order item, the Company Code and Controlling Area in the Account Assignment tab update correctly.

However EKKN - IMKEY still shows the old value

Environment

•Materials Management (MM)
•SAP ERP Central Component
•SAP ERP
•SAP enhancement package for SAP ERP
•SAP enhancement package for SAP ERP, version for SAP HANA
•SAP S/4HANA

Reproducing the Issue

1. Create a Purchase Order with:

      Company Code = "XXXX"

      Plant = "XXXX"

      Assign Real Estate Object, Profit Center, and Business Entity.

2. Change Plant manually to "YYYY"

3. Observe EKKN - IMKEY still shows "XXXX".

(PATH: In PO   item detail >> account assignment tab >> click on real estate object >> select Bus Entity>> search options(F4) >> Company code is "XXXX"

Cause

This is standard SAP system behavior.

Resolution

The behavior is standard within SAP systems. Specifically, Real Estate objects are not dynamically re-derived and remain independent of changes made to the PO plant. These objects are validated only at the time of assignment and retain their original company code unless explicitly deleted and reassigned. Consequently, changing the plant updates the PO company code but does not automatically re-derive the Real Estate object, which can lead to mismatches if adjustments are made manually without first deleting the Real Estate object.  

Use one of the following approaches:  

  • Edit the Real Estate object:  
    1. Navigate to ME22N.  
    2. At the item level, delete the Real Estate object completely.
    3. Save the changes.
    4. Re-enter the Real Estate object, Business Entity, and Profit Center.
    5. Save again. 
  • Cancel and recreate the item (recommended for a cleaner resolution):
    1. If feasible, delete the PO item entirely.
    2. Recreate the item, ensuring the correct plant is assigned first.
    3. Then, assign the Real Estate object.

Additionally, to prevent similar mismatches in the future, consider implementing a validation in `Badi ME_PROCESS_PO_CUST, method check` based on the business requirement.

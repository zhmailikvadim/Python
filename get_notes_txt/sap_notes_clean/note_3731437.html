Symptom

  • Abbreviations for time units (e.g., “22h 30m”) are used for course duration in the learning library.
  • Screen readers announce the abbreviations literally (e.g., “twenty-two h thirty m”) instead of expanded, meaningful text.
  • Users must interpret the shorthand to understand the duration, which can reduce clarity for screen reader users and those unfamiliar with the abbreviations.

Environment

  • SAP SuccessFactors Learning

Reproducing the Issue

  1. Access the Access SuccessFactors system.
  2. Navigate to Learning > Library > Browse all > Find learning.
  3. Observe the course duration displayed using abbreviated time units (e.g., “22h 30m”).
  4. Use a screen reader to read the course duration and note that abbreviations are read literally.

Cause

This issue has been identified as a bug when using screen readers.

Resolution

The backend code has been fixed by engineering. The fix version is scheduled to be delivered as part of the 1H 2026 release. 

For product-related information, please review the SuccessFactors Product Release & Road Map

Symptom

1 ) You have identified a negative stock quantity in the inventory and you try to reduce the quantity further (while still keeping it negative) using the MI10/MI04 transaction.

2)  You wish to post negative Quantity count using MI10/MI04 transaction system does not allow and prompts the below error

     Specify positive values only

       Message No. 00126

Environment

  • Materials Management (MM)
  • SAP ERP
  • SAP ERP Central Component
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA

Reproducing the Issue

  1. Attempt to post a negative quantity using the MI10 transaction.
  2. Encounter an error because MI10 does not allow the entry of negative numbers.

Cause

Negative stock was allowed at the material master level/storage location level, leading to the creation of a material document with negative stock .

SAP standard system validation prevents counting or reversing stocks in the negative range during physical inventory processes

Resolution

This behavior is standard SAP design.
Physical Inventory transactions do not allow entering negative counted quantities, even if the book inventory is negative.
SAP requires that the counted quantity always represents the physically available stock, which must be a non‑negative value.
To resolve the issue:  Enter only positive or zero physical counted quantities

Keywords

MI10, Physical Inventory, MI04, PI, Negative Quantity count , material document.

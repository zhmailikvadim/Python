Symptom

In EAM Simplicity, you create a new functional location with a template of an existing functional location. You copy the template from the corresponding search help. However, the system incorrectly issues error message IL100 ("Functional location & does not exist").

Other Terms

W0029, technical object, TPLNR, STRNO, CONVERSION_EXIT_TPLNR_INPUT, CONVERSION_EXIT_TPLNR_OUTPUT, IL, 100, 100(IL)

Reason and Prerequisites

The reason and prerequisite is that you use the alternative labeling for functional locations.

Solution

There is a program error.

Note: If the automatic correction instructions deactivate the window WD_VALUE_HELP of the Web Dynpro component EAMS_WDC_ESH_VALUE_HELP_2, activate it again manually afterwards.

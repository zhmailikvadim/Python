Symptom

  • There's a huge number of entries in table /IMA/EDIDS 
  • The housekeeping job SAP_FRN_AIM_HOUSEKEEPING (program /IMA/HOUSEKEEPING_NEW) is not deleting records from the table /IMA/EDIDS
  • The housekeeping job spool confirms that no records are found or deleted from /IMA/EDIDS.

Environment

  • SAP Focused Run 500

Reproducing the Issue

CHECK 1:

  1. Go to SE16 -> Table /IMA/EDIDS
  2. Click on "Number of Entries" -> Huge number of entries displayed

CHECK 2:

  1. Execute the housekeeping job SAP_FRN_AIM_HOUSEKEEPING (program /IMA/HOUSEKEEPING_NEW).
  2. Observe that no records are deleted from /IMA/EDIDS.

Cause

There are multiple documents that have not reached Final or Non Final Status since very long.

Resolution

The resolution of this problem needs, beforehand, a detailed analysis as it will require to locate all the documents that are being collected (and, therefore, changed) constantly without allowing reaching "Final" or "Non-Final" statuses for the established Housekeeping periods. Below example indicates how to proceed for a specific document behaving this way, but this procedure would need to be applied to all the documents behaving likewise.

EXAMPLE:

In table /IMA/HK_CAT_STAT we can see which statuses are defined as FINAL or as NON_FINAL for and every type of document. In our case, for IDCO we have status FINAL = SUCCESSFUL and NON_FINAL = ERROR or INTERMEDIATE

At the same time, in table /IMA/HK_REN_APP shows the retention periods for each document and for each status. In our case, for IDOCs, the retention period for FINAL status is 30 (column RETENTION _PERIOD) and the retention period for NON_FINAL status (column RETENTION_PERIOD_NF) is 15.

With these 2 tables we can see then, that the only way for the Housekeeping jobs to get rid of the entries from /IMA/EDIDS for IDOCs is:

  • That the specific IDOC document has been in FINAL status (meaning, with SUCCESSFUL status) for 30 days

OR

  • That the specific IDOC document has been in NON_FINAL status (meaning, with ERROR|INTERMEDIATE status) for 15 days

Having specified this, let's focus on a specific document (622518). In /IMA/EDIDC we can see the latest time a document (in this case an IDOC) has been collected. As this table reflects the last data collection and the document collection is triggered once the document has been modified/changed, it also indicates (with a minor time difference) when the document was changed. In this case you can see the IDOC 622518 had its latest collection on 4th Mar at 11:22:44. We can also see that the status for the document is 29 meaning "Entry in outbound table not found" (which is neither SUCCESSFUL or ERROR|INTERMEDIATE)

This information gives a clear indication: the document is being collected so it's been changed/modified recently without a FINAL or NON_FINAL status, so it can't be deleted (as it is still a relevant document).

Nevertheless, and here comes the problem, when looking in table /IMA/EDIDS, we can see the history of collections for this document and the document has been collected (and, therefore, changing) constantly since April 2025 (almost a complete year). As the document, since very long time ago (in this case since April 2025) has been changing constantly and, therefore, there hasn’t be any time where the status Final (SUCCESSFUL) has persisted for 30 days or a Non Final Status (ERROR|INTERMEDIATE) has persisted for 15 days, then the document is not deleted (as there are constant changes and Final or NonFinal statuses are not reached for the established Housekeeping periods) and the table /IMA/EDIDS is growing with the different data collections.

So if there are multiple documents with this behaviour, then the number of entries in /IMA/EDIDS could reach huge numbers. You can see that a single document (like the one used in this example) has created almost 27000 entries in a year. This is not a bad behavior of Advance Integration Monitoring, so the actions need to be focused on reducing the retention periods or deleting the entries from /IMA/EDIDS manually.

POSSIBLE ACTIONS: 

  1. Adjust (reduce) the retention periods for the different statuses (FINAL and/or NON FINAL) for the affected document type in table /IMA/HK_REN_APP 
  2. Delete the unwanted old entries directly from the table /IMA/EDIDS manually

Communication with other customs authorities

If you use SAP Customs Management for communication with customs authorities and use SEEBURGER AG Business Integration Server (BIS) for electronic communication with the customs systems, message conversions and workflows must be installed on BIS.

For further details, see SAP Note 1241609.

Contact your consultant with regard to conversions in other supported countries.

 

If you use processing with the customs server in the Netherlands, you have to use message conversions that were created after November 13, 2016 in the converter.

For further information, see SAP Note 2386176 ("AGS3: Transfer of seals in change messages (SEQID)".

Preference management

If you issue customer-based long-term supplier’s declarations (transaction /SAPSLL/PRE_VDO_701) or re-issue them (transaction /SAPSLL/PRE_VDO_702), you have to convert your worklist for customer-based long-term supplier’s declarations (transaction /SAPSLL/PRE_WLO_001) so that you can continue to issue them. This applies only if a worklist exists when you import the support package. To do this, use the report /SAPSLL/PRE_VDO_FILL_VDOWLV, which is provided in Support Package 16. Alternatively, you can also proceed as described in SAP Note 2793346, section II, "Worklist for long-term vendor declarations for customer's purpose". 

If you upgrade from an SAP GTS release lower than SAP GTS 10.0 to SAP GTS Release 10.1, you have to execute the program /SAPSLL/TPREFSK_CLONE using transaction SE38 after you import Support Package 01.

Before you import the support package, ensure to the greatest extent possible that the majority of entries in the table /SAPSLL/PRFPMRP is archived with the archiving object PRFCALC.

Important

If you use sanctioned party list screening for employees, you must first import the support package and then copy the program z_ftoa_conv.txt provided with SAP Note 1683200 to a local program in your system and execute it. Otherwise, the existing assignments of your employees to the foreign trade organizational units can no longer be interpreted properly.

Note that SAP Note 2513767 results in a bug that causes all SPL entities to be marked as changed after you upload the XML file. As a result, many false positive results for business partners that you manually released are generated again during a future SPL delta check. It is extremely time-consuming to check these again. Note this problem if you manually implement SAP Note 2513767 or an upgrade of the GTS system to the higher version GTS 11.0 with SP11 or SP12. The problem is corrected with SAP Note 2573362.

Standardization of transport data for US customs declarations

The use of the fields on the Transport tab has been standardized for US customs declarations. For more information, see SAP Note 2214185. Following the import of the support package, fill in the fields as described in this SAP Note so that the correct data is sent to the customs systems during outbound processing. Documents created earlier for which messages are still to be sent must be adjusted accordingly. Messages that have already been sent remain unchanged and are unaffected by the changes.

Depreciation of licenses, letters of credit, and securities

If you use the value depreciation or quantity depreciation for licenses or the value depreciation for letters of credit and securities, you must implement SAP Notes 2231154 and 2234114. Otherwise, incorrect depreciation posting runs or runtime errors may occur.

 

API for creating a customs bill of lading

a)

Support Package 16 changes the interface of the API for creating customs bills of lading (name: /SAPSLL/API_6800_CFB_SYNCH). Refer to SAP Note 2805640 for more details. It indicates that you have to execute the migration program Z_PZ_MIGRATE_VESSEL_NAME_CFB.

Check customer-defined programs that call the API /SAPSLL/API_6800_CFB_SYNCH in SAP GTS or from an external system. Check customer-defined programs that use the structure /SAPSLL/API6800_HDR_CFB_S. The length of the field VESSEL_NAME is extended from 20 to 35 characters. Check whether your source code can handle the changed field length. Check any customer-specific implementations of the method CHANGE_INPUT_PARAMETERS of the interface /SAPSLL/IF_EX_BADI_CFB.

b)

Support Package 16 changes the interface of the API for creating customs bills of lading (name: /SAPSLL/API_6800_CFB_SYNCH). Refer to SAP Note 2807400 for more details.

Check customer-defined programs that call the API /SAPSLL/API_6800_CFB_SYNCH in SAP GTS or from an external system. Check customer-defined programs that use the structure /SAPSLL/API6800_HDR_CFB_S. The length of the following fields has been extended from 15 or 20 characters to 35 characters:

Name in API Data element Length previously Length now
SBOLNO /SAPSLL/SBOLNO 20 35
BOLNO /SAPSLL/BOLNO 20 35
MASTER_BOLNO /SAPSLL/MBOLNO 20 35
AWBNO /SAPSLL/AWBNO 15 35
MASTER_AWBNO /SAPSLL/MAWBNO 15 35
TRUCK_NUMBER /SAPSLL/TRCKNO 15 35

Check whether your source code works with the changed field lengths. Check any customer-specific implementations of the method CHANGE_INPUT_PARAMETERS of the interface /SAPSLL/IF_EX_BADI_CFB.

 

If you use the US FTZ solution, take the following into account:

This SP adds the fields "BOM Explosion Indicator Necessary" (/SAPSLL/BOMEX_R3, /SAPSLL/BOMEX) and "Prod. Ref. Number for BOM from Feeder System" (/SAPSLL/BOMRF_R3, /SAPSLL/BOMRF) to the API /SAPSLL/API_6800_CUWL_SYNCH (through SAP Note 2609653). In addition, implement SAP Note 2609654 in the feeder system. This SAP Note enables the transfer of goods receipts of finished products to SAP GTS. Note that you must set the new "Goods Receipt from Production (FTZ)" indicator for goods movements that constitute a goods receipt on the basis of a production order in the view /SAPSLL/V_TLER3B for the application level MM0C.

 

Classification distribution

If you distribute commodity codes to the feeder systems, see SAP Note 2831295. With Support Package 17, you can also distribute tariff numbers because the database has been enhanced for this purpose. After you import the Support Package, you must execute the program /SAPSLL/TCOTS_STCTS_EXC_SET using transaction SE38.

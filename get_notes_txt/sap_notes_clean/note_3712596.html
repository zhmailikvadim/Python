Symptom

 Enables Client ID and Client Secret stored in ABAP Secure Storage to be passed as HTTP Headers in API Calls to the US Treasury System via MuleSoft Cloud API Integration.

Other Terms

G-Invoicing, MuleSoft Cloud API, Client ID, Client Secret, HTTP Headers, API Call, ABAP Secure Storage, US Treasury System, Certificate

Reason and Prerequisites

MuleSoft Cloud API Integration requires the Client ID and Client Secret to be passed as HTTP Headers for authenticated API calls to the US Treasury System. These credentials must be retrieved from ABAP Secure Storage and included in every API request.

Prerequisites:

  • 3712598 – G-Invoicing Cloud API – Avoid Trailing Slashes in the Group API Call URI
  • 3716012 – G-Invoicing Cloud API – Attachment Deletion URI to be Derived
  • 3712600 – G-Invoicing Cloud API – Secure Storage of Client ID and Client Secret

Solution

Implement the attached correction instructions.

Apply the SAP notes related to G-Invoicing Cloud API in following sequence:

NoteTitle
3712598 G-Invoicing Cloud API – Avoid Trailing Slashes in the Group API Call URI
3716012 G-Invoicing Cloud API – Attachment Deletion URI to be Derived

3712601 G-Invoicing Cloud API – MuleSoft Integration

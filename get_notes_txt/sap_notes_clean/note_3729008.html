Symptom

This SAP Note provides you with corrections to the Mexico Account Balance Reporting - Month 13 (MX_ACCT_BAL_P13_REP) report in the Run Statutory Reports app for SAP Document and Reporting Compliance.

When you download the "XML file output for G/L Account Balances" from the Mexico Account Balance Reporting - Month 13 report, the system displays the current period in the filename instead of period 13.

Other Terms

 DRC, Mexico, Electronic Accounting, MX_ACCT_BAL_REP, MX_ACCT_BAL_P13_REP, G/L Account, Alternative G/L Account, Country Chart of Accounts

Reason and Prerequisites

This SAP Note is relevant only for Mexico.

To implement Electronic Accounting compliance reports, you need to install the Document and Reporting Compliance (DRC) framework. For more information, see Advanced Compliance Reporting.

You have installed the Electronic Accounting compliance reports for Mexico. For more information, see SAP Notes 2548443 or 2598781.

You have implemented previous notes listed in SAP Note 2812643.

Solution

After implementing this SAP Note, the system correctly displays the filename with period 13 when you download the "XML file output for G/L Account Balances" from the Mexico Account Balance Reporting - Month 13 report.

As a general rule, SAP recommends that you install a solution by applying a Support Package. However, if you need to install a solution earlier, use the Note Assistant to implement the correction instruction. You can find more information about the SAP Note Assistant in the SAP Support Portal, under SAP Note Assistant.

To implement the solution, you must carry out manual activities and implement the correction instructions that are valid for your release.

You can try to activate the objects contained in this SAP Note executing the reports below:

  • For DDLS objects, execute the RUTDDLSACTSNOTE report according to the steps available in SAP Note 2607137.
  • For DCLS objects, execute the R_ACMTOOL_DELETE_INACTIVE_V report according to the steps available in SAP Note 2691812.

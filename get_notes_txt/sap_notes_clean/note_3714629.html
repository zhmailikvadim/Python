Symptom

While converting an MM SRV PO to Lean PO using conversion report "Schedule Purchase Order Conversion",  error "Change indicators could not be uniquely determined in material mapping" is generated. This happens only when there are items with only Limits and no services.

E.g.

We have MMSRV PO as below:

Line 10  item category D(Service) - No services, have limits.

Line 20 Item Category D(service) - Have service and limits.

Error is getting generated due to the first item (Item 10).

Other Terms

S4CORE, MM-PUR-SVC-SES, Tool to Convert MMSRV POs to Lean POs, CL_MMSRV_PURDOC_MIGRATION, Lean Services, Schedule Purchase Order Conversion, F8760, Brown Field Scenario

Reason and Prerequisites

Program Error

Solution

As general rule, SAP recommends that you install a solution by applying a Support Package. However, if you need to install a solution earlier, use the Note Assistant to implement the correction instruction. You can find more information about the Note Assistant in SAP Service Marketplace, under service.sap.com/note-assistant.

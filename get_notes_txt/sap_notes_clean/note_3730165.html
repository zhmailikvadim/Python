Symptom

An output item was generated for a sales document, e.g.: an ORDER_CONFIRMATION is generated for a sales order, and it can be seen in the related applications. However, the document is not stored in the configured CMIS repository.

Environment

SAP S/4HANA Cloud Public Edition

Cause

In SAP S/4HANA Cloud Public Edition, the attachment service uses the DMS and GOS document management frameworks, which use different logic to handle the attachments:

  • For DMS based attachments, the attachments uploaded from a business application are associated to a Document Info Record (DIR). The DIR contains the actual documents (or attachments) and the metadata about the documents.
    • This is where the sales document data and attachments are stored.
  • For GOS based attachments, the uploaded attachments are directly linked to the SAP business objects.
    • This is where outputs are stored.

A visual representation of the architecture can be found here: SAP CMIS Profile | SAP Help Portal

Resolution

In the CMIS repository, the documents generated from outputs are stored in the folder /Knowledge Provider/SOMU_PHIO.

For harmonizing document storage, there's a feature request raised for it already, if it matches the business necessity, vote and follow it for further updates from the product management team. Harmonize approach of Output Management documents and attachments for external CMIS repositories | Customer Influence Portal

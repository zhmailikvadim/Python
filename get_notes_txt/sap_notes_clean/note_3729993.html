Symptom

While attempting to connect to a remote FTP server with a channel that's configured to use FTPS protocol, the FTP connectivity is failing.

Recent changes might've been implemented like importing new certificate into your PO system.

While performing a channel ping to test the connection, errors such as the one below can be observed: "FTP connection to host * and port * cannot be created"

Environment

SAP PO 7.5

Cause

This is a network level issue likely caused by blocked Ip address and/or port is closed from Firewall.

Resolution

 The Network Team should be involved to verify the below:

  • Firewall rules: Ensure that the outbound traffic from the PI/PO server to the FTP server (on the required IP/port, e.g., port 21 or 22 depending on FTP/SFTP) is allowed.
  • Routing configuration: Verify that the PI/PO server can reach the FTP server’s IP address over the network without restrictions.
  • Intermediate firewalls or proxies: Make sure that no restrictions or drops are applied on the traffic in between.

Keywords

FTP connection to host and port cannot be created

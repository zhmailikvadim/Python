Symptom

  • Purge Key Figure Data application job is scheduled with check box Delete Planning Objects selected, but planning object is not deleted.
  • Job log displays following warning messages:
    Warning Deletion of planning objects may not be possible.
    Warning To delete planning objects, add the following versions: <VersionID_1>, <VersionID_2>
  • When the "Preview" option is selected during a simulation run of the purge job, qualified planning objects are not listed.

Environment

SAP Integrated Business Planning (SCM-IBP)

Reproducing the Issue

  1. Go to Application Job app, click on + to schedule a new job and select 'Purge Key Figure Data' template.

  2. Enter parameters such as Planning Area, Version, Base Planning Level.

  3. Select either Older Than (Period Offset) or Later Than (Period Offset).

  4. Enter value for the parameter Period Offset.

  5. You may or may not select Planning Filter.

  6. Select check box Delete Planning Objects.

  7. Simulate or Run the job.

Cause

Planning objects may be used in several versions at the same time. Unless a version has version-specific master data, it typically shares the planning objects with the base version. One or more versions that use the same planning objects are missing in the selection. Therefore, deleting the planning objects may not be possible.

Resolution

While scheduling the purge key figure data job, if the selected base planning level has key figures which are selected in user‑defined versions which are not version‑specific, make sure to include all those versions and the _BASELINE when scheduling the job.

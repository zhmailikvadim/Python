Symptom

  • When attempting to migrate SAP Webi reports from an OLAP (HANA On-premise) connection to an OLAP (HANA InA) connection using SAP BO Web Services, the API call to update the connection returns a 404 error. Other standard API calls (e.g., connection list, document list) are functioning as expected.
  • error_code": "WSR 00102","message": "The definition of this BW connection is incomplete \"<ID>\". Please specify a valid ID string."
  • GET http://<HOST>:8080/biprws/raylight/v1/documents/<ID>/dataproviders/mappings?originDataproviderIDs=DP5&targetDatasourceId=<TARGETID>;
    Result: 404 Error

Environment

  • SAP BusinessObjects Business Intelligence Platform 4.x / BI2025

Cause

Incorrect usage of the API

Missing parameters in the URL if it's a BEX Query or HANA View. The targetDatasourceId parameter has to have a specific BEX Query or HANA view, it can't include only an ID of a connection

Resolution

Specify the specific BEX Query or HANA view in the URL:
http://<HOST>:8080/biprws/raylight/v1/documents/<ID>/dataproviders/mappings?originDataproviderIDs=DP5&targetDatasourceId=<TARGETCONNECTIONID>;<HANAVIEW or BEXQUERY>

Eg:
http://<HOST>:8080/biprws/raylight/v1/documents/<ID>/dataproviders/mappings?originDataproviderIDs=DP5&targetDatasourceId=5678;TEST_HANAVIEW

For more details see:
https://help.sap.com/docs/SAP_BUSINESSOBJECTS_WEB_INTELLIGENCE/0ef63ba725cd41a8ab4a69c226ec6b07/7da08f306f701014aaab767bb0e91070.html?q=targetdatasourceid

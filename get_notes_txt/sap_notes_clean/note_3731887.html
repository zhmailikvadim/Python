Symptom

When running RPFIBR_MIGRATE_CREDIT_CONTROLS, the system issues error "Enter a valid Branch for Asset XXXX in Company Code XXXX".

Environment

  • SAP S/4HANA Cloud Private Edition
  • SAP S/4HANA

Reproducing the Issue

Start the backend program RPFIBR_MIGRATE_CREDIT_CONTROLS to migrate tax credits and the error "Enter a valid Branch for Asset XXXX in Company Code XXXX" appears during processing.

Cause

  • Missing configuration for company code and branch in the Define Organizational Structure (ORGSTR) customizing in the client where the report is run.
  • Cost center categories linked to the assets not maintained in “Define Production Cost Center Relevant for Credit Control” (/TMF/D_CCPRCSTCT) customizing.
  • Inconsistent LAST_CREDIT_RECOVERED_DATE in the migration template (fiscal period does not exist in the system for the given context), causing lookup failures.

Resolution

  • Maintain the Define Organizational Structure (ORGSTR) for all relevant company codes and branches in the same client where you run RPFIBR_MIGRATE_CREDIT_CONTROLS. SPRO path: Financial Accounting > General Ledger Accounting > Periodic Processing > Report > Statutory Reporting: Brazil > Company Settings > Define Organizational Structure.
  • In customizing “Define Production Cost Center Relevant for Credit Control” (/TMF/D_CCPRCSTCT), add all cost center categories that are used by assets participating in PIS/COFINS credit control.
  • Review the migration template and correct LAST_CREDIT_RECOVERED_DATE so that the fiscal year/period exists and is valid for the asset/company/branch context in your system.
  • Save the changes and run RPFIBR_MIGRATE_CREDIT_CONTROLS again.

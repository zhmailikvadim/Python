Symptom

After creating and activating a new Portugal FI ATCUD series with a validation code, the first FI invoice or credit note generates ATCUD = “0”. Subsequently, the series is no longer displayed in GLO_PT_SVC, although it remains active at table level, and the validation code is cleared from SIPT_NUMST_FI, preventing correct ATCUD generation for subsequent documents.

Environment

  • SAP ERP
  • SAP S/4HANA
  • SAP S/4HANA Private Cloud Edition

Reproducing the Issue

  1. Create and activate a new Portugal FI ATCUD series (e.g. F16D / F18E) with a validation code using GLO_PT_SVC.
  2. Post the first FI document (invoice or credit note) using a document origin that results in BKPF-AWTYP = BKPFF.
  3. Check the generated document and observe ATCUD = “0”.
  4. Reopen GLO_PT_SVC and note that the series is no longer visible.
  5. Verify in table SIPT_NUMST_FI that fields REG_ATCUD_ID and REG_INVOICE_TYPE are cleared.

Cause

The first FI documents of the newly created Portugal series were posted with BKPF-AWTYP = BKPFF, which is not recognized as a valid initial FI document by the ATCUD process. As a result, the system ignores the series, clears the validation code from SIPT_NUMST_FI, removes the series from GLO_PT_SVC, and generates ATCUD = “0”.

 

Resolution

Ensure that the first FI document of each new Portugal ATCUD series is posted with a valid FI document origin supported by the ATCUD process. Implement the relevant SAP Notes for Portugal digital signature and ATCUD handling. If the issue has already occurred, create a new unique series (Document Type + Fiscal Year + Number Range + Company Code + Series ID) and use it for subsequent FI invoices and credit notes.

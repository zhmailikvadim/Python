Symptom

  • In transaction MB26, an outbound delivery is successfully created for a partial quantity against a reservation / PM order.
  • When attempting to create another delivery for the remaining open quantity of the same reservation/PM order, the system does not allow the creation of a second delivery (item not proposed/blocked).
  • Error GOODSMOVEMENT008 is raised with diagnosis: "No goods movements can be selected".

Environment

  • Materials Management (MM)
  • Inventory Management
  • SAP ERP Central Component
  • SAP ERP
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA

Reproducing the Issue

  1. Start transaction MB26.
  2. Enter a reservation number and plant.
  3. Create an outbound delivery for a partial quantity against the reservation/PM order.
  4. Attempt to create a second delivery for the remaining open quantity of the same reservation/PM order.

Cause

Reservation items that have open deliveries are excluded from the MB26 pick list; while a delivery remains open, the system does not propose the item, which can lead to GOODSMOVEMENT008.

Resolution

  1. Check whether there is an open outbound delivery for the reservation item.
  2. Post the goods issue and complete/close the existing outbound delivery.
  3. Return to MB26 after posting and pick the remaining open quantity; the reservation item should now be proposed again.
  4. Create the subsequent delivery for the remaining quantity, if required.

See Also

Symptom

Activation of SAP Note 3596394 fails with error “Data object <FS_MAPPING_ATTR> has no component KEEP_IN_SYNC” in method UPDATE_CHG_NMBR_ATTRIBUTES.

Other Terms

PLM-CR, ECN, Change Record, Integration of Change Management, 3578782, Keep In Sync, KEEP_IN_SYNC, 3596394, Activation Error, UPDATE_CHG_NMBR_ATTRIBUTES

Reason and Prerequisites

Field "KEEP_IN_SYNC" is not delivered by SAP Note 3596394. It is delivered by the backend changes in SAP Note 3578782. If SAP Note 3578782 is not implemented, the system does not contain the required structural enhancement, leading to the mentioned activation error.

Solution

Continue implementing the next SAP Note 3578782 (Back End Changes) after encountering the mentioned error.

Recommended procedure:

  1. Keep the implementation of SAP Note 3596394 in the state as it is.
  2. Start implementation of SAP Note 3578782, you may still get activation error here.
  3. Keep the implementation of SAP Note 3578782 in the same state.
  4. Activate SAP Notes one-by-one.
  5. Go to transaction code SNOTE and select SAP Note 3596394 and perform following steps using menu SAP Note -> Activate SAP Note.
  6. After successful activation of SAP Note 3596394, repeat the same step #5 for the remaining SAP Note 3578782 and activation of all objects should be successful.

Symptom

After BADI FTR_CUSTOMER_EXTENT(Open TR-TM: Customer Enhancement) defined and start transaction code TX01.

Find that in the method: EVT_APPLICATION_START has been activated because the breakpoint stops in the debugger, but in the Method: EVT_TRANSACTION_SAVE_READY, debugging with a breakpoint does not stop at the target location and the Method itself is not called.

Environment

Treasury and Risk Management (TRM)

  • SAP R/3
  • SAP R/3 Enterprise 
  • SAP ERP Central Component
  • SAP ERP
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA Finance
  • SAP S/4HANA

Reproducing the Issue

Implement BADI FTR_CUSTOMER_EXTENT, and set a breakpoint to trigger Method: EVT_TRANSACTION_SAVE_READY.

Resolution

Events such as EVT_TRANSACTION_SAVE_CHECK and EVT_TRANSACTION_SAVE_READY are only called if you have previously set up user-defined tab enhancements within EVT_APPLICATION_START.

Therefore, you must add the following codes within your own implementation of EVT_APPLICATION_START, for example:

 CALL FUNCTION 'FTR_CUSTOM_BADI_SAMPLE_START'.

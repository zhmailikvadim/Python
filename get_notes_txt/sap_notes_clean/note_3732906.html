Symptom

During execution of:

hdbnsutil -sr_takeover

the following error is returned:

this is not a system replication site, cannot takeover

In the system replication setup phase or prior to the takeover attempt, additional messages may be observed in nameserver traces such as:

  • Could not reach any host of site
  • nameserver communication error; internal error(5500)

System replication status may be reported as ACTIVE. In parallel, communication-related errors may be present in nameserver traces.

Environment

SAP HANA DB Version 2.0

Cause

The takeover command can only be executed on a system that is recognized by SAP HANA as a valid secondary system replication site.

If the system is not recognized as a valid secondary system replication site at runtime, the takeover command will fail with this error.

Resolution

Ensure that the system replication is correctly configured and that no communication-related errors are present before executing takeover.

Recommended checks:

  • Verify system replication state using:
    hdbnsutil -sr_state
  • Verify replication status using:
    SELECT * FROM SYS.M_SYSTEM_REPLICATION;
  • Ensure that system replication status is ACTIVE and no communication-related errors are present in nameserver traces

If errors are observed, resolve the reported issues and verify the system replication state again before retrying the takeover.

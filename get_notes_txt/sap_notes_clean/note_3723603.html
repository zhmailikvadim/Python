Symptom

  • Configuring a Microsoft SQL Server Data Source, in DSConnectionManager for a SSL connection, fails with:
    • Unable to dynamically load client DLL DAopenssl30.so. [SAP BODS][ODBC SQL Server Wire Protocol driver]SSL Initialization failure

Environment

  • SAP Data Services 4.3
  • SSL connection to Microsoft SQL Server
  • Azure SQL
  • Linux Job Server

Reproducing the Issue

  1. Run DSConnectionManager.sh from cd $LINK_DIR/bin
  2. Try to add a new Microsoft SQL Server via Data Direct
  3. Fails with the error in the symptom

Cause

The DataDirect driver bundled with SAP Data Services 4.3 and higher, does not support SSL, which is mandatory for SSL connection (Azure SQL).

Resolution

For support with SSL, a Microsoft SQL Server driver has to be downloaded/installed and used, instead of the DataDirect drivers. And the unixODBC driver manager has to be used instead of the DataDirect driver manager.

  • To use the unixODBC driver manager for the Data Source:
    1. Navigate to cd $LINK_DIR/bin and open ds_odbc.ini file to modify it (e.g vi ds_odbc.ini)
    2. Copy the sample entry needed, and add to the end of the file
    3. Change the DSN name (inside []) to the one that will be used (e.g [TEST], and point the driver to the unixODBC driver
      1. Driver=/usr/lib64/libodbc.so
  • Create the Microsoft SQL Server DSN in the $ODBCINI file:
    1. Open $ODBCINI file to modify it (e.g vi $ODCBINI)
    2. [TEST]
      Driver=/opt/microsoft/msodbcsqlxx/lib64/libmsodbcsql-xx.x.so (point to the MS SQL Server driver)
      Server=IPaddress_or_FQDN,Port (port is optional)
      Address= network address of the server running (if needed)
      Database=database_name
      Encrypt=yes
      TrustServerCertificate=yes
      1. For more ODBC connection string keywords, please check Microsoft ODBC documentation

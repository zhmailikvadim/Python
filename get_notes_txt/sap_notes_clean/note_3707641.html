Symptom

With SAP S/4HANA Cloud Private Edition 2025 FPS01 and SAP S/4HANA 2025 FPS01, you can perform intercompany transfers for service objects and perform in-house service processes within intercompany service orders.

If you use In-House Service in a release earlier than SAP S/4HANA Cloud Private Edition 2025 FPS01 or SAP S/4HANA 2025 FPS01, specific Customizing settings are required in Customizing for Service.

Solution

You need to make the following settings Customizing for Service.

Partner processing

To enable entries in the End Customer field, you must make settings in the following Customizing activities:

  1. Define Partner Functions

Create an entry for the end customer and assign the function category End Customer to it

  1. Define Access Sequences

Create an access sequence to copy the entry from the Sold-To Party field to the End Customer field.

Make the following settings for your access sequence in the Individual Accesses view:

  • Dialog sequence: 10
  • Source: Current Partner
  • Wait Until Source is Available checkbox: Selected
  • Mapping/Definition for Partner Being Searched checkbox: Selected
  • Partner function category: Sold-To Party
  • Usage in source: Customer Relationship Management
  1. Define Partner Determination Procedure in Service
  • Assign the partner function End Customer (CRM) to the procedure REPA2021 (In-House Services).
  • Assign the access sequence that you have defined in step 2 to the partner function and make the following settings:
    • Minimum: 1
    • Maximum: 1

The end customer is displayed in the following apps:

  • In the Manage In-House Services app, on the In-House Service Details screen
  • In the Manage In-House Service Objects app, on the In-House Service Objects Details screen

This field shows the business partner that originally sent the object for service and typically owns it. This business partner matches the sold-to party specified for the original in-house service. In the intercompany in-house service, the sold-to party is the business partner of the sales organization that launched the intercompany process.

For more information, see the Customizing documentation in Service under In-House Service->Define Partner Processing.

Text management

The following settings are required to copy entries that have been made in the text fields of the original in-house service to the text fields in the intercompany in-house service:

Text object: CRM_ORDERH

  • Definition of Procedure view for text determination procedure REPA2020 (Texts for In-House Service)
    • Text type: R001 (Customer Instruction)
    • Access sequence: R001
  • Definition of Access Sequence view for access sequence R001
    • Sequence: 0001
    • Copy from: CRM_ORDERI
    • Assigned function module: CRMS4_REPA_COPY_TEXT_TO_HEADER
    • Continue checkbox: Not selected

Text object: CRM_ORDERI

  • Definition of Procedure view for text determination procedure REPA2022 (Texts for In-House Service Objects)
    • Text type: R001 (Customer Instruction)
      Access sequence: R001
    • Text type: R002 (Examination Results)
      Access sequence: R001
  • Definition of Access Sequence view for access sequence R001:
    • Text type: R001
      Sequence: 0001
      Copy from: CRM_ORDERH
      Assigned function module: CRMS4_REPA_COPY_TEXT_TO_ITEM
      Continue checkbox: Selected
    • Text type: R002
      Sequence: 0002
      Copy from: CRM_ORDERH
      Assigned function module: CRMS4_REPA_COPY_TEXT_TO_ITEM
      Continue checkbox: Selected
    • Text type: R001
      Sequence: 0003
      Copy from: CRM_ORDERI
      Assigned function module: CRMS4_REPA_COPY_TXT_TO_ITEM_IC
      Continue checkbox: Not selected
    • Text type: R002
      Sequence: 0004
      Copy from: CRM_ORDERI
      Assigned function module: CRMS4_REPA_COPY_TXT_TO_ITEM_IC
      Continue checkbox: Not selected

For more information, see the Customizing documentation in Service under In-House Service->Define Text Management.

Specific settings for the intercompany transfer for service objects: PPF (Post Processing Framework) actions

You perform actions to create follow-up service transactions and documents, or to add service objects to existing service transactions in the Manage In-House Service Objects app under Decide for service objects related to the original in-house service.

For service objects related to intercompany in-house services, it is not intended to create follow-up service transactions and documents or to add service objects to existing service transactions. Instead, you do this for the service objects related to the original in-house service.

The parameter IS_ INTERCOMPANY in the scheduling conditions that are maintained in the Define Conditions Customizing activity is used to specify if such actions for the action profiles REPA2021 (Actions for In-House Services) and REPA2022 (Actions for In-House Service Objects) are provided for intercompany in-house services or not. The following values apply to this parameter:

  • No value: The action is provided for in-house services and intercompany in-house services.
  • ` `: The action is not provided if the in-house service is an intercompany in-house service. We recommend using this value.
  • ’X':  The action is provided only for intercompany in-house services.

For more information, see the Customizing documentation in Service under In-House Service->Define Actions.

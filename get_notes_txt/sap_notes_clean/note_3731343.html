Symptom

While using SAP Service and Asset Manager application, user is configured with variant and the variant configuration influences some change to standard delivered mobile app parameter e.g., 'ABC' delivered with value as 'X' in standard. In variant configuration, it is overidden to 'Z'. Though the variant is active, new parameter value isn't determined using the variant configuration. Determined value is from standard configuration.

Other Terms

S4MOBADDON-8404, Variant Configuration, /SYCLO/CL_CORE_MDO_TOOLS, /MFND/CL_CORE_ODATA2_PROXY_MGR, /MFND/CL_CORE_OMDO_HNDLER_BASE, /MFND/CORE_REPEATABILITY_UTIL

Reason and Prerequisites

Reason: If the user is assigned with variant, it should be applied in the below cases:

  • Reading mobile app parameter value using utility class/method /SYCLO/CL_CORE_MDO_TOOLS~READ_MAPP_PARAM_VALUE
  • Evaluate Repeatability is supported using utility class/method /MFND/CORE_REPEATABILITY_UTIL~EVAL_REPEATABILITY_SUPPORTED
  • Determine the usage of HDB Hints /MFND/CL_CORE_OMDO_HNDLER_BASE~SET_DATABASE_HINTS
  • Evaluate if tombstone should be skipped in case of shared data between multiple oMDOs /MFND/CL_CORE_OMDO_HNDLER_BASE~READ_SKIP_TOMBSTONE_PARAM
  • Load inactive user dependency list to generate tombstone /MFND/CL_CORE_OMDO_HNDLER_BASE~LOAD_INACTIVE_DEP_KEYLIST
  • Auto purge client state jobs /MFND/CL_CORE_ODATA2_PROXY_MGR~SCHEDULE_CLIENT_STAT_PURGE_JOB
  • Auto purge user dependency /MFND/CL_CORE_ODATA2_PROXY_MGR~INIT_USR_DEPLIST
  • oMDO feature check in proxy class /MFND/CL_CORE_ODATA2_PROXY_MGR~PROCESS_REQUEST

Solution

Please apply this note in SAP system.

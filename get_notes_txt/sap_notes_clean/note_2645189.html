Symptom

You are trying to create a custom query in the sales organization object value selector field, but you are not able to save it.

Environment

SAP Cloud for Customer

Reproducing the Issue

  1. Go to the Sales work center
  2. Go to the Sales Quotes or Sales Orders view
  3. Select New
  4. Go to the field Sales Organization and open the object value selector
  5. In the sales organization popup, select Show Advanced Filter
  6. Formulate the query with the available fields
  7. Select Save Query

You will see that the query created is not saved.

Cause

Sales area determination for sales quotes is enabled in scoping.

Resolution

Sales area determination business option allows the user to add through the selection of the sales organization, its corresponding distribution channel and division valid for the account and the employee.
It will be defaulted if the sales area matches from the employee and the account.
If multiple matching assignments exist, the user can select the appropriate one.
If an employee is not assigned to a sales area, the determination will consider account’s sales data only

So while it is enabled, user cannot limit the sales organization and the same can be verified by following the steps below

  1. Go to the Business Configuration work center
  2. Go to the Implementation Projects view
  3. Identify the relevant project and select it
  4. Select Edit Project Scope
  5. Navigate to step 4, Questions, using the guided activity
  6. Select the element Sales
  7. Go to the sub element New Business and select Sales Quotes
  8. Under Questions for Sales Quotes, find Sales Area Determination for Sales Quotes

Here you will see that the question, "Do you want to enable users to default and specify, per sales quote, a predetermined combination of sales org., distribution channel and division?" is in scope.

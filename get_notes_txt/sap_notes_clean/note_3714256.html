Symptom

In SAP NetWeaver, the B2B Log Viewer does not display any logs when accessing the monitor, even when the filter timeframe is extended up to one year. When attempting to view logs, the following exception is displayed in the B2B Log Viewer: com.sap.tc.webdynpro.model.ejb.exception.EJBModelExecuteException.

Environment

  • SAP NetWeaver
  • SAP Process Integration

Reproducing the Issue

  1. Access the B2B Log Viewer in SAP NetWeaver.
  2. Attempt to view logs by setting the filter to 1 year timeframe.
  3. Observe that no logs are displayed in the monitor.
  4. Exception message appears: com.sap.tc.webdynpro.model.ejb.exception.EJBModelExecuteException.

Cause

Resolution

  1. Verify that B2B logging is enabled: Navigate to NWA -> Configuration -> Infrastructure -> Java System Properties -> Services -> XPI Service: Messaging System. Ensure B2B logging properties are enabled.
  2. Enable AS2 anonymous logging: In the same service, check the parameter b2b.log.anonymous. Set the value to ON if AS2 anonymous logins are used: Refer to SAP Note 2261553 – No logs created in B2B Log Viewer for AS2 anonymous logins.
  3. Persist duration time: In the same service, review the parameter messaging.log.RetainTime. Increase the retention time if the current value is too low for operational or historical analysis. Refer to the SAP article Persist duration time at the Adapter Engine for synchronous and asynchronous messages: Persist duration time at the Adapter Engine for synchronous and asynchronous messages
  4. Confirm user authorizations: Ensure the user has the appropriate roles for B2B monitoring and log access in SAP NetWeaver.
  5. Update all B2B-related components: update B2B components to the latest supported versions. Follow the guidance in KBA 2484216 – Deployment of B2B components.

Symptom

You use the report Upload File and Processing ELDA Sick Leave Notification (RPCKSBA0_B2A) to upload a sick leave statement. The values of individual fields are missing.

In particular, this becomes clear if no unique personnel number was found during consolidation and data for the analysis is missing in the B2A Manager (H99_B2AMANAGER).

Other Terms

HRPAYAT_KSB060, MODUL, HRPAYAT_T5A1M_MODUL, Hatschek, VONA, NAME

Reason and Prerequisites

If a customer-specific conversion was defined for the ELDA notification type KB in the view T5A1M, the data was processed incorrectly. Instead of performing the customer-specific conversion for the relevant field, the relevant field was not transferred to further processing. As a result, the data of the field was missing.

Solution

The correction is provided by HR Support Package (via CLC if necessary).
You can implement an advance correction by implementing the automatic correction instructions using SNOTE.

Import Version 3 from February 25, 2026. In the previous version, the error did not appear in the B2A Manager.

Details

Since the KSB files are already provided in the restricted code page of ELDA, it is not assumed that an additional conversion is required for the upload to an SAP system.
Up to now, the function of the customer-specific conversion for KSB fields was not supported and is not introduced with this.

After you implement this SAP Note, the system issues error message HRPAYAT_KSB 060 ("Customer-specific conversion is not supported") in the case of customer-specific conversions defined for KSB. This should avoid any missing data and point out that the customizing needs to be adjusted accordingly.

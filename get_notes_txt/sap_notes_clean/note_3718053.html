Symptom

  • Newly added payees are missing from the 'tcmp.cs_padimension' table, causing data retrieval issues for custom reporting procedures.
  • The issue occurred after migrating to HANA, during the first instance of adding new payees post go live in the updated system.

Environment

  • SAP Sales Performance Management
    • Incentive Management on Hyperscalers
    • Incentive Management on Legacy HANA 

Reproducing the Issue


  1. Create a Crystal report in Oracle system by querying the 'cs_padimension' table.
  2. Migrate the application from Oracle to HANA as part of the Titan project.
  3. Add new payees to the HANA system.
  4. Run the Embedded Analytics report to view payees.
  5. Verify that the newly added payees are missing from the report.

Cause

The issue occurs because the migration to HANA replaced the 'cs_padimension' table populated by ODSReportGen pipeline in Oracle with the 'csa_padimension' table in the HANA ETL pipeline.

Resolution

  1. Modify the custom code in Embedded Analytics reports to reference the 'csa_padimension' table instead of the 'cs_padimension' table.
  2. Test the changes to confirm that newly added payees are now visible in the Embedded Analytics report.

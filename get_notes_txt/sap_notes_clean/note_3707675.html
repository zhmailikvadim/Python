Symptom

After creating a sales order in the Advanced Intercompany Sales (5D2) process, the purchase order is not created automatically. In the "Monitor Value Chains" (F4854) app, there is no purchase order in the document process.

Environment

SAP S/4HANA Cloud Public Edition

Cause

The confirmed quantity of the sales order item is 0. If items do not have the confirmed quantity, the corresponding PO will not be created automatically.

Resolution

Check the material inventory and make sure the item confirmed quantity is not 0 so that the auto creation for PO can be triggered.

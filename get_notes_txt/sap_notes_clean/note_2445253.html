Symptom

  • SAP Transportation Management with TM-ERP Shipment Integration. 
  • ERP Shipment is created from a Freight Order (FO) or Fright Booking (FB). 
  • In transaction ST22, short dump  "ASSERTION_FAILED"  "/SCMTMS/CL_TOR_HELPER_ERP=====CP" bzw. /SCMTMS/CL_TOR_HELPER_ERP=====CM006  "CHECK_TOR_FO" is generated, where the identified program is /SCMTMS/PROCESS_TRIGGER_BGD
  • Shipment for the FO or FB is not created. 

Environment

  • SAP Transportation Management 9.1
  • SAP Transportation Management 9.2
  • SAP Transportation Management 9.3
  • SAP Transportation Management 9.4 

Reproducing the Issue

  1. Run /SCMTMS/PROCESS_TRIGGERS_BGD.
  2. System dumps with error message:  "ASSERTION_FAILED"  "/SCMTMS/CL_TOR_HELPER_ERP=====CP" bzw. /SCMTMS/CL_TOR_HELPER_ERP=====CM006  "CHECK_TOR_FO"

Cause

ASSERT is thrown in /SCMTMS/CL_TOR_HELPER_ERP CHECK_TOR_FO method where an inconsistency is detected - either the FO/FB does not exist or it does not contain the expected STOP or ITEM information.

Confirm that the target FO exists.  If the FO was deleted before /SCMTMS/PROCESS_TRIGGER_BGD is run, then the system will dump.

To check if the FO exists:

  1. In /SCMTMS/D_TRIGHD, view the triggers that are causing the dump.  
  2. Copy the target business object ID.
  3. In /bobf/test_ui, enter /SCMTMS/TOR and search for the target business object ID in the ROOT ID field.
  4. If the object does not appear, then it does not exist.

If there are too many entries in the table /SCMTMS/D_TRIGHD, examine the dump for any identifying TOR ROOT ID.  Alternatively, you can set a breakpoint in /SCMTMS/PROCESS_TRIGGER_BGD and go through the triggers to see which one causes the dump.  User may wish to enter selection criteria in the batch report to narrow down the number of triggers.

Resolution

Identify relevant trigger entry, determine the problematic FO/FB and investigate whether the trigger is still relevant:

  1. In table /SCMTMS/D_TRIGHD, identify the trigger entries that led to the short dump. 
  2. Copy the target business object ID. 
  3. In transaction /BOBF/TEST_UI, for business object /SCMTMS/TOR, search ROOT for the target business object ID. 
  4. Ensure the FO/FB exists and, if it does exist, that they carry shipment-relevant ITEM_TR and/or shipment-relevant STOPs. 

There may be circumstances where in between the creation of the trigger entry and the processing of the trigger entry, the FO/FB may have been modified to be no longer relevant for ERP Shipment creation. If this is the case, the trigger may be cleared from /SCMTMS/D_TRIGHD.

Consider the circumstances that caused the FO to be deleted before the triggers were run.  If the FO was deleted intentionally and the triggers are no longer necessary, then the triggers may be cleared from /SCMTMS/D_TRIGHD.

See Also

For SAP TM 9.5 and above a new trigger concept has been introduced and /SCMTMS/PROCESS_TRIGGER_BGD is obsolete and is not used.  See SAP Note 2526415 for more info.

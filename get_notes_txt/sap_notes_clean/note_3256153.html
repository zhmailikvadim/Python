Symptom

In SM21 error similiar  to.

e=28 semget(3004439,20,2016) (28: No space left on device)

Also can see SM21 error.

Maximum number of permitted terminal connections reached.

Environment

SAP on SUN Solaris.

Cause

Problem with SUN kernel parameters.

Resolution

Try these parameters.

project.max-shm-ids=(priv,1024,deny)

set semsys:seminfo_semmni=1024

Make sure the project is defined for all the instances not just one.

Check note.

724713 - Parameter Settings for Oracle Solaris 10 and above

See also...

https://wiki.scn.sap.com/wiki/pages/viewpage.action?pageId=375030464

 

In some case, the error still happens after parameter change, please do the following
- completely stop the SAP instance
- stop the sapstartsrv processes as well
- verify that no process are running with <sid>adm (ie: ps -ef | grep <sid>adm)
- login again with <sid>adm
- start the sapstartsrv
- start the SAP instance

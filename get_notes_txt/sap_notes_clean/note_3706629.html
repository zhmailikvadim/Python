Symptom

An SAP HANA high availability node restarts unexpectedly. SAP HANA traces show normal secondary operation with log replay and synchronization, followed by a clean restart. No SAP HANA crash, panic, out-of-memory condition, or other SAP HANA software error is recorded.

Environment

SAP HANA DB Version 2.0

Cause

The observed restart is consistent with a system replication takeover triggered outside the SAP HANA database layer, for example by cluster management software or a manual administrative action.

Resolution

This behavior does not by itself indicate an SAP HANA software issue. To identify the trigger of the takeover, analyze operating system and cluster management logs for the relevant timeframe.

For general background on SAP HANA system replication, refer to SAP Note 1999880 – SAP HANA System Replication – FAQ.

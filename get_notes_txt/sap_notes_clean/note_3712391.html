Symptom

When creating a Master Agreement using the Mass Extension Workbench, the system incorrectly adds scale data to the Target Master Agreement. This issue occurs even though the source document does not contain any scale information.

As a result, the Target Master Agreement is created with unintended scales that were not maintained or expected based on the source document data.

Other Terms

/IRM/IPPQMAME, /IRM/IPPQ_MASS_EXT_PROCESS, conditions_copy, /IRM/GCLE_KONM_READ, /IRM/GCLE_KONW_READ

Reason and Prerequisites

This is caused by program error.

Please implement the latest version of SAP Note # 3652430 as pre requisite note.

Solution

Please implement the correction instructions attached. These corrections are only valid if you have applied the Support Packs.


Release V4 - 2022 V2, 2023, 2023V2, 2023 V3 and 2023 V3 Support Pack 1.

Use transaction SNOTE to implement the correction instructions specified below and activate the affected program object.

Symptom

Program S3VBRKRL is being performed to reload archived billing documents. The reload is aborted and ends with error FG216.

Environment

  • SAP R/3
  • SAP S/4HANA
  • S/4HANA Cloud Private Edition

Reproducing the Issue

In S3VBRKRL add the documents to be reloaded and execute it.

Cause

If in table ADMI_CRIT e.g. for archiving object SD_VBRK flag is applied for XCRIT_RELO and an euro conversion got performed after the archiving run of of the required documents, FG216 is issued in:

    ADMI_CRIT-OBJECT     = ADMI_RUN-OBJECT.

    ADMI_CRIT-XCONT_RELO = ' '.
  ENDIF.

    IF ADMI_CRIT-XCONT_RELO = 'X'.

                 AND CONV_DATE > ARCHIVE_DATE.

    ELSE.

If field critical reloading of archived files (ADMI_CRIT-XCRIT_RELO) is marked the respective archiving object is seen as critical with regard to previous currency conversions. In such a case when a reloading transaction refers to data archived before the last currency conversion is started then the process is terminated.

Resolution

Review the settings in transaction HOSH of the questionable archiving object, especially with regard to XCRIT_RELO flag in the ADMI_CRIT table.

Ensure that the indicator is appropriately configured to allow reloading transactions without causing issues. If this flag is not set, archiving is considered non-critical. In this case, the respective application logic must ensure that the reloading transaction does not cause any issues. 

Please note that reloading of archived documents should only be performed in exceptional cases with caution see SAP note 491506.

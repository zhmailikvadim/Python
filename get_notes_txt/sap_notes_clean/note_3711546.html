Symptom

When working with multiple monitors configured with different DPI settings, and Excel is set to "Optimize for best appearance" under Excel Options → General → When using multiple displays, you may encounter display issues in Analysis for Office.

If you move Excel to a secondary monitor and initiate a drag & drop action from the Analysis for Office Task Pane, the crosstab may not be rendered correctly. Specifically:

  • The crosstab overlay used to indicate the drop area may appear at an incorrect position.
  • The overlay may also be incorrectly scaled, resulting in a zoomed‑in or zoomed‑out appearance depending on the monitor's DPI.

When dragging a characteristic onto the crosstab, the drop location markers may be displayed in the wrong location or may not appear at all.

Other Terms

AO; Analysis Office; Drag; Dragging; Drag & Drop; Overlay; Drop Zone; Multi-Display; Scaling; Indicator; Ghosting

Reason and Prerequisites

Program error

Solution

After applying the correction, the crosstab overlay, drop marker positioning, and scaling behavior function correctly when performing drag & drop operations from the Analysis for Office Task Pane in multi‑monitor environments with differing DPI values.

Please be aware that this correction depends on the same prerequisites described in SAP Note 3248397.

Limitations

The following known limitations of the drag & drop functionality remain unchanged:

  • Incorrect scaling in Excel compatibility mode
    When Excel is configured to run in “Optimize for compatibility” mode (Excel Options → General → When using multiple displays → Optimize for compatibility), the crosstab overlay may not scale correctly in multi‑monitor setups with different DPI settings.

  • Incorrect marker positioning when using Excel Split View
    When the Excel Split feature is enabled (Excel Ribbon → View → Window → Split), the drop location markers may appear in the wrong position or may not appear at all.

The correction will be delivered by a SAP Analysis for Microsoft Office Support Package.
In section "SP Patch Level" you can find information which Support Package contains the respective correction.

If the section "SP Patch Level" is not available or the information about which Support Package contains the correction is missing, this means that this has not been determined yet. In this case please re-check this note at a later point of time.

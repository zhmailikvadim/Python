Symptom

Questionable value in "current consumption" field that can be found in transaction ES32 and EL28 

Environment

  • SAP for Utilities as Part of SAP ERP, release IS-UT 600 and higher.
  • SAP S/4HANA, release IS-UT 801 and higher.

Reproducing the Issue

  1. Start transaction EL28 or ES32 and display a meter reading result for an installation.
  2. Navigate to the layout where the field "current consumption" is shown.
  3. Compare the value of this layout field with the "current consumption" shown in transaction EL31 page 1.
  4. Observe that the layout field value differs from the dynamic consumption shown on EL31 page 1.

Cause

  • The field labeled "current consumption" in the layout belongs to Quantity Determination (technical field EABLD_EL31-QNT).
  • It is not the same as the dynamically calculated current consumption (EABLD-CONSMPT) displayed on EL31 page 1, which reflects the difference between the current and previous meter readings.

Resolution

  1. Do not use the layout field "current consumption" (EABLD_EL31-QNT) unless the Quantity Determination functionality is intentionally in use.
  2. Remove the "current consumption" (EABLD_EL31-QNT) field from the EL31/EL28 layout if Quantity Determination is not required.
  3. To view the actual current consumption, open EL31 page 1 and refer to the dynamically calculated field EABLD-CONSMPT.
  4. For quantity used in billing/reading to be billed, rely on the dynamic determination based on the difference between current and previous readings, considering register factor and decimal places.

Keywords

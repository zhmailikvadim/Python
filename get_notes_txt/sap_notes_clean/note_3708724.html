Symptom

When calling the BAPI /SPDGS/BAPI_PDA_CREATE, it is not possible to deactivate the COMMIT.
This can lead to inconsistent data if multiple application steps must be executed within a single overarching logical unit of work, but the BAPI triggers an internal COMMIT prematurely.

Other Terms

IS-MFM, CORE, PDA, COMMIT, /SPDGS/BAPI_PDA_CREATE, SAP Meat and Fish Management by msg.

Reason and Prerequisites

The BAPI /SPDGS/BAPI_PDA_CREATE performs an unconditional COMMIT WORK internally.
The interface provides a flag controlling the COMMIT behavior, but the flag is not evaluated during execution, resulting in the COMMIT always being executed regardless of the caller’s settings.

Solution

History:
Note created on 28.01.2026.

Description of Correction:
The logic of /SPDGS/BAPI_PDA_CREATE is corrected so that the COMMIT‑control flag is properly evaluated.
If the flag “no COMMIT” is set by the calling program, the BAPI no longer executes an internal COMMIT WORK.
This ensures that the caller retains full control over the logical unit of work and can bundle several dependent operations safely before triggering the final COMMIT.

Functional Dependencies:
None.

Manual Changes:
No manual changes are required for end users.

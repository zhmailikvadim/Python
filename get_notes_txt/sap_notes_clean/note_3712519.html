Symptom

When importing any Datastore Table, the Catalog is imported, causing a mismatch for the table when using System Configuration.

Other Terms

BODS, DS, BD, Sys, Config, Owner, Source, Target, env, design, hosts, aliases

Reason and Prerequisites

Environment:

  • SAP Data Services 2025 and above
  • Databricks

Steps to Reproduce:

  1. Create a Databricks Datastore with two Configurations.
  2. From the Local Object Library, import by name any table from this Datastore.
  3. Table is imported with Catalog name from one of the environments.
  4. Design a Job which uses this table as Source/Target.
  5. Unable to switch System Configurations due the Catalog information.

Solution

This issue has been forwarded to Development for further investigation

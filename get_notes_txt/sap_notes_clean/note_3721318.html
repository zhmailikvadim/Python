Symptom

In SSAM version 2405 qualitative values in checklists are not updated properly after initial selection. Yes/No pairs or other values selected from a list cannot be changed after a value is selected.

Reason and Prerequisites

Reason:

The updated values were sent to store into the local database, but the related Navigation Properties was not changed accordingly, which revert the change. Both of the list view and table view in the "Record Results" screen were affected. 
In addition two other issue were found. In list view the "Extension" value was incorrectly fetched, which also blocked the update process.
In the table view the qualitative checklist items which were not selected were incorrectly handled, this issue also blocked the process, where this case presents.

Prerequisites:

  • BAS or Visual Studio Code with SAM MDK project
  • `patch` command (available on BAS and macOS by default) via terminal or other patching utility
  • Read the guidelines on how to apply the patch and read the appendix section if you receive any error

Solution

After updating the navigation properties the checlist value changes were saved properly in both list and table view. 
The "extension" object fetching was fixed.
The checklist items which were not selected are skipped now during processing.

To apply the patch, download the attached 3721318.patch file and save it in a known location, preferably adjacent to your SAM.mdkproject.

If you are on Windows, see SAP Note 3533879 for how to install a .patch file in Windows. If you are on a Mac, follow the steps below:

  1. Open a terminal in the root of your SAM metadata (i.e. SAM.mdkproject/metadata or SAM.mdkproject/metadata/base.definitions)
  2. Run the terminal command `patch --no-backup-if-mismatch -p4 < path/to/3721318.patch`. 
  3. Verify in the terminal window that the following files have been modified and listed as patched, as indicated by the console output:

  • patching file 'Rules/InspectionCharacteristics/Update/InspectionCharacteristicsFDCUpdateDone.js'

  • patching file 'Rules/InspectionCharacteristics/Update/InspectionCharacteristicsUpdateDoneEDT.js'

Symptom

Monitoring for AIF messages or interfaces is configured via Integration & Exception Monitoring (IEM) in SAP Focused Run (FRUN) but no data is displayed for it.

Environment

  • Product: SAP Focused Run
  • FRUN: Advanced Integration Monitoring (AIM)

Reproducing the Issue

  1. Configure AIF categories for a managed system in IEM and activate the setup.
  2. Open IEM and check the AIF category for the managed system; observe that no AIF data is collected or displayed.

Cause

  • Prerequisites are not fulfilled
  • Authorization is missing
  • No data exists

Resolution

Check whether below prerequisites were fulfilled

  1. The monitored S/4HANA system should run on:
    • SAP_BASIS 7.54 (and higher)
      and
    • SAP_ABA 75E (and higher)
  2. In the monitored S/4HANA system, the data should exists in /n/AIF/ERR.
  3. In the monitored S/4HANA system, the setup of AIF should be done using the transaction /AIF/IFMON.
  4. As mentioned in the note 2450740, the SDA user(SDAGENT_<FRUNSID>) of the S/4HANA system should have below authorization objects:
    • Authorization Object: /AIF/FRUN: Activity=33
    • Authorization Object: S_RFC: RFC_NAME=/AIF/FRUN_API_EXT, /AIF/GET_FRUN_DATA; Activity=16
      Note: Above authorization objects are not delivered by ST-PI. Z-role should be created and assigned to SDA user of S/4HANA system.

See Also

  • SAP Note 2450740 -  Roles to authorize access in managed systems to collect data for FRUN
  • Expert Portal: SAP AIF monitoring with Focused Run

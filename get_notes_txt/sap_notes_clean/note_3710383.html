Symptom

While creating and outbound delivery - for example from a sales order - the expected inspection lot is not created in one scenario, while - in exact same system - the lot is created in an other case, properly.

The material master QM view is maintained properly: the necessary inspection type - for example with origin 10 - is assigned to the material.

Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.

Environment

  • SAP ERP Central Component
  • SAP ERP
  • SAP Enhancement package for SAP ERP
  • SAP S/4HANA, on-premise
  • SAP S/4HANA Cloud Private Edition

Reproducing the Issue

Example scenario:

  1. Call transaction VA02 for an order.
  2. Goto menu 'Sales Document > Deliver'.
  3. Click SAVE.
  4. An outbound delivery is created, but in one case lot is triggered, while in other cases, no lot is generated along with the delivery creation.

Cause

Missing configuration in customizing.

Resolution

Ensure that following QM customizing contains the relevant 'Origin - DelivCat - InspType' assignment:

IMG(transaction QCC0 or SPRO)
>Quality Management
>>Quality Inspection
>>>Inspection Lot Creation
>>>>Inspection at Shipping
>>>>>Inspection at Shipping (Composite Activity)
--> Assign Inspection Type to Delivery Category

For example:

if a lot with origin 10, inspection type 10 is expected for delivery category 'S', ensure such assignment exists.

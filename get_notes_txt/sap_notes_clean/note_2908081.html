Symptom

The correspondent business scenario, from the technical point of view, requires an SAP Analytics Cloud Story to be built.
It is wanted to use GEO map with Custom Choropleth layer in order to leverage from the benefiits of creating Self-Created Region Characteristics with Self-Uploaded Shapes, that are origanized in Self-Defined hierarchical Arrangement of the Regions.

In case of customer shapes, the main source of Geo data is the shapes itselfs, that have been created in for example ArcGIS. The shapes are needed to be uploaded to the Business Document Service (BDS). it is needed to be done transaction RSDIOBJGISMAINT.

Shapedata comes in (at least) triples:

Region.dbf: DBase file contains a table with Attributes describing Regions
Region.shp: Shapefile containing all geographic Shapes
Region.shx: Link between Shape and DBase File

All the transformations and geographical calculations happen in HANA. So for processing the SPATIAL engine of HANA is used, this means that to have a good perfromance, we want all the GEO data in GEO format, so HANA can work with it right away. Therefore after the shapes are uploaded to the Business Document Service, the Geo data is still needed to be transformed to the HANA Spatial format by selecting option "Update Geotable used during reporting". During the update process, error occurs:

Error Update Task for geotable cancelled due to errors.
Information Shapefile ga_test_shape_ga_nav1.shp has been loaded.
Error invalid column name: <"nav.attr.">: line 1 col 8 (at pos 7)
Error Reading from .dbf file failed due to missing fields.

Environment

  • SAP Analytics Cloud 2020.7 and upwords
  • SAP BW/4HANA 2.0 SP4

Reproducing the Issue

1., Go to the BW Modeling Tools and select one of the navigation attribute of your Geo Characteristic that is in Geo Type 2 (Shapes).
2., Select Actual Upload Shapes. Execution redirects to SAPGUI(Transaction RSDIOBJGISMAINT)
3., Click go and upload the shp, dbf and shx files
4., Select Action „Update Geotable“
=> Error occurs

Cause

DBase File (.dbf) needs to be enhanced by Field containing corresponding characteristic values. (or more if compounded).
Field Names need to be Name of Characteristic. The error occurs, because the DBase File misses the used navigation attribute name and therefore the shapes are not mapped to certain characteristic members of the navgation attribute.

Resolution

It‘s possible to download the char. values in DBase File Format via Geo Action „Geodata Download(everyth.)“ in the BW Modeling Tools. It is possible to edit the file with Microsoft Access (available through MS Office 365).

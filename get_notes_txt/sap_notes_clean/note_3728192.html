Symptom

  • Users cannot see available values in the prompt dropdown/search results
  • Prompt search returns empty results despite data being present in the backend database.

Environment

  • BO Version: SAP BusinessObjects BI Platform 4.3
  • Any Supported Operating System

Reproducing the Issue

  1. Log in to BI launchpad
  2. Open a Web Intelligence report
  3. Refresh the repor
  4. In the prompt window, search for any data using a date value. (Example: Start Date: 01/01/2026 End Date: 15/01/2026)
  5. You will notice that you will not receive any data for provided date range even if data is available in the backend database

Cause

  • In Web Intelligence Processing Server (WIPS), List of Value (LOV) cache option was Enabled i.e. LOV is cached in the following path:
    <BO Install Directory>\SAP BusinessObjects Enterprise XI 4.0\Data\<server_port>\storage\universes\<CUID of the universe>

Resolution

  1. Navigate to Central management Console (CMC) -> Servers -> Web Intelligence Processing Server (WIPS) -> Properties
  2. Under Information Engine Services, uncheck the option 'Enable LOV Cache'.
  3. Click on 'Save' and restart the server.

Keywords

4.3,webi, lov, universe, cache, refresh, upgrade

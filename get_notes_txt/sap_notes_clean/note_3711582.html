Symptom

A new profit center with the same name but different validity period was created. For example, profit center hierarchy X with validity period 01/01/2000 - 31/12/2024 and profit center hierarchy X with validity period 01/01/2025 - 31/12/9999 both exists. It is necessary to view data for 2024 using the older validity period version of the hierarchy X.

Environment

SAP S/4HANA Cloud Public Edition

Reproducing the Issue

  1. Change the profit center hierarchy.
  2. Attempt to generate past-year actual reports using the previous fiscal year's profit center hierarchy.
  3. Observe that the reports are generated using the new profit center hierarchy instead.

Cause

The issue is caused by the time-dependent nature of the profit-center hierarchy. Specifically, the key date filter in the report app Profit Center - Plan/Actual determines which hierarchy version is selected for the report.

Resolution

  1. Set the key date filter in the report app Profit Center - Plan/Actual to a date within the validity period of the desired hierarchy version (e.g., 30/12/2024 for the older hierarchy version).
  2. Click "Go" to update the report based on the selected key date filter.
  3. Right-click on the Profit Center column and select "Select Hierarchy".
  4. Use the search functionality in the popup to locate and select the hierarchy version that is valid for the chosen key date.

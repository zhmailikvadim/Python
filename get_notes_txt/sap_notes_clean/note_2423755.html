Symptom

CBTA recording stops at the login screen of the System Under Test.

Investigation.zip reports the following error:

ERROR | MainThread          | TestCreationWizardRecordingStep    | Unable to start the recording!

or

ERROR   : Thread01 (MainThread) : GuiScriptingControl : AuthenticateUser method: Unable to login into the system.System.Runtime.InteropServices.COMException (0x80010105): The server threw an exception. (Exception from HRESULT: 0x80010105 (RPC_E_SERVERFAULT))
at System.RuntimeType.ForwardCallToInvokeMember(String memberName, BindingFlags flags, Object target, Int32[] aWrapperTypes, MessageData& msgData)
at SAPFEWSELib.ISapSessionTarget.get_ActiveWindow()
at GuiScripting.Spy.GuiScriptingControl.AuthenticateUser(GuiSession gSession, SapGuiCredentials cred)

Environment

  • Solution Manager 7.2
  • Component Based Testing and Automation (CBTA).

Reproducing the Issue

  1. Navigate to TCE (Test Script) and edit.
  2. Launch CBTA to record the script which stops at the login screen of the System Under Test (SUT).

Cause

Issue related to to Scripting parameters in the System Under Test (SUT).

Resolution

  1. Please validate the system parameters in RZ11 is set with following values:
    + sapgui/nwbc_scripting: is * to FALSE
    + sapgui/user_scripting: is * to TRUE
    + sapgui/user_scripting_disable_recording: is * to FALSE
    + sapgui/user_scripting_force_notification: is * to FALSE
    + sapgui/user_scripting_per_user: is * to FALSE
    + sapgui/user_scripting_set_readonly: is * to FALSE
  1. In order to make a permanant change, you need to makesure the values are set via RZ10,
    - Start transaction RZ10
    - For Profile, select Default
    - Select Extended Maintenance, click on Change.
    - Click Parameter.
    -
    Type Parameter name <parameter_name>
    -
    Type Parameter value <parameter_value>
    -     Click on Green Arrow where you will be prompted to Save the changes in profile.
    - Select 'YES' when prompted to activate the profile.

Symptom

An invoice is posted matching the goods receipt, and, GR-Bsd IV is set. For the invoice item, the blocking 'Quality' is set, and the invoice is blocked. Even though the material has been released from quality inspection through MIGO, the MRBR transaction is not releasing the invoice using the option "Release Automatically".

Environment

  • SAP ERP Central Component
  • SAP ERP
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA

Reproducing the Issue

  1. For the PO/SA, create a GR into blocked stock, using movement type 103.
  2. Then, create a GR from the blocked stock, using movement type 105. Here, the stock type applied is '2 - Quality Inspection'.
  3. Post the invoice referencing the movement type 103.
  4. Remove the material from the quality inspection.
  5. Try to release the invoice using the Release Automatically option via MRBR. It won't work, as the block is still active for the invoice.

Cause

  1. With movement types 103 and 105, during MIRO processing, system will derive the reference document from the movement type 103.
  2. Quality inspection will be made from the movement 103 instead of 105.
  3. Movement type 103 is normally moved to restricted stock, and movement 105 is from restricted to un-restricted stock.
  4. Due to above logic, if the user have done the Quality Inspection during movement type 105 rather on movement type 103, then, the current Invoice payment block remains - as the invoice refers to movement type 103.

Resolution

In the MRBR transaction, release the invoice using the 'Release Manually' option.

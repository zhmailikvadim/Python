Symptom

When running the consolidation tool for active records of business partner for finding duplicates and best records calculation, system is showing too many matched records for a small pool of data and while opening the match group it is consuming a lot of memory and resulting in dump

Category               ABAP programming error
Runtime Errors         TIME_OUT
ABAP Program           CL_SQL_STATEMENT==============CP
Application Component  BC-DB-DBI

Environment

  • SAP Master Data Governance-Consolidation and Mass Processing

Reproducing the Issue

  1. Launch Fiori Launchpad
  2. Initiate a process for consolidation of active records
  3. Select the template as 'Consolidation of active records'
  4. Start the process and select the selection criteria as equal to and provide those attached BP numbers
  5. Continue till Matching step.
  6. Open Match groups
  7. Dump is raised

Cause

  • If the custom ruleset includes "Identification" rule.  In the BUT0ID table, tens of thousands of records have the same IDs. This results in millions of matching combinations in the MDC_FUZZY_RES table.

Resolution

  • It is recommended to start with a smaller ruleset and adopt a step-by-step approach when adding new rules. Specifically:  

    - Add one rule at a time and test the ruleset to ensure it functions as expected.  
    - Gradually incorporate additional rules, testing the ruleset after each addition.  

    This methodical approach will help identify any failing rules and streamline the troubleshooting process. 

  • When creating rules, you should also remember that indirect matches are also included in the same group, e.g. BP_A matches BP_B by one of IDs but does not match BP_C, however, BP_C matches BP_B by some other ID; in such case, BP_A and BP_C match indirectly and are included in the same group as BP_B.

See Also

  • Review SAP Note 3620928 - CMP: Table MDC_PACKAGES_PRC contains incorrect number of processed matching step packages, MDC_FUZZY_RES

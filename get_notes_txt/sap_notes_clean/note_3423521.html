Symptom

  • Create a remote source through FileAdapter and create a Virtual table
  • Select through the virtual table failed with error: "Remote execution error Exception while processing request "SDA Response for FileAdapter of type SUCCESS_MSG responding to FEDERATION_GET_NEXT" for query ***"
  • From framework.trc file, following error showed
    "DPFramework | AsyncHandler.processStreamingExecuteStatement [] - AH(***): com.sap.hana.dp.adapter.sdk.AdapterException: Exception while processing request "SDA Response for FileAdapter of type SUCCESS_MSG responding to FEDERATION_GET_NEXT" Context: java.lang.NegativeArraySizeException: -1073741824
                                    at com.sap.hana.dp.fileadapter.CharBuffer.<init>(CharBuffer.java:17)
                                    at com.sap.hana.dp.fileadapter.tableloaders.TableLoaderWithAllFilters.getNextRowData(TableLoaderWithAllFilters.java:197)
                                    at com.sap.hana.dp.adapter.sdk.adapterbase.BaseAdapterClass.getNext(BaseAdapterClass.java:1026)
                                    at com.sap.hana.dp.adapter.sdk.decorator.AdapterCDCDelegate.getNext(AdapterCDCDelegate.java:118)
                                    at com.sap.hana.dp.adapter.framework.core.AsyncHandler.fetchRows(AsyncHandler.java:493)"

Environment

SAP HANA Smart Data Integration

Cause

framework.maxDataSize in DP agent configuration value is too large

Resolution

  1. Suspend capture for all the remote sources if there is live subscriptions for the DP agent
  2. Stop DP agent
  3. Go to edit dpagentconfig.ini file under DP agent installation directory to revert the parameter value to default as below
    framework.maxDataSize=10MB
  4. Restart DP agent
  5. Resume capture for all the remote source if you suspend in step 1

Keywords

DP agent, FileAdapter, remote source, NegativeArraySizeException

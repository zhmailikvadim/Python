Symptom

WorkOrder Details > Operation Card > Overflow menu > Record Time - Workorder, Operation and Work center fields are editable

Other Terms

  • SAP Service and Asset Manager
  • SAM
  • SSAM
  • Record Time
  • Add Time
  • CATS
  • Operation Card
  • Overflow Menu

Reason and Prerequisites

Reason:

Record Time from operation card was referring to Time sheets library used for Home >Time Sheets > Add, which caused issue in processing non editable fields.

Prerequisites:

  •  BAS or Visual Studio Code with SAM MDK project
  • `patch` command (available on BAS and macOS by default) via terminal or other patching utility
  • Read the guidelines on how to apply the patch and read the appendix section if you receive any error

Solution

ObjectCardTimeCreate has been updated to use TimeSheetEntryCreateForWONav action.

To apply the patch, download the attached 3552069(YYMM).patch file and save it in a known location, preferably adjacent to your SAM.mdkproject
YYMM represents product release version.

If you are on Windows, see SAP Note 3533879 for how to install a .patch file in Windows. If you are on Mac/Web(BAS), follow the steps below:

  1. Open a terminal in the root of your SAM metadata (i.e. SAM.mdkproject/metadata or SAM.mdkproject/metadata/base.definitions)
  2. Run the terminal command `patch --no-backup-if-mismatch -p4 < path/to/3552069(YYMM).patch`.
  3. Validate that changes have been applied to the following files:
    • modified file 'Rules/OverviewPage/MyWorkSection/ObjectCardTimeCreate.js'

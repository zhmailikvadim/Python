Symptom

On a MDF Object (e.g Position) when using a conditional association for a translatable field and a custom UI configuration, when switching languages the translation are being incorrectly overwritten with value of other language

Environment

SAP SuccessFactors Employee Central

Reproducing the Issue

  1. Configure a field of type translatable on position object that contains a conditional association and custom UI.
  2. Insert a record on position and populate translations.
  3. When we change system language E.g. From English UK to German
  4. Proceed to insert a new record with no changes. 
  5. German value is overwritten by the value for the language below German in translations

Cause

SAP Engineering team have confirmed this as a defect.

Resolution

SAP Engineering team has confirmed this will be fixed in 1H 2026 release, which is scheduled for deployment in preview on April 13, 2026, and in production on May 15-16, 2026.

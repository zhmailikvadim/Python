Symptom

As described in SAP ONE Support Launchpad Notification of Legal Change Unified monthly employer report 2025 (JMHZ)

The functionality delivered in this SAP Note is currently under development. It is intended exclusively for customers registered as JMHZ pilot testers, or for the preview of features that may become available for productive use in the future.

These updates are provided:

  • REGZEC (DM - data mining, CS - compose submission, PF - process feedback, SO - special operations)

    • DM: New BAdI HR_CZ_JHR_JOB_POSITION_NAME has been implemented to retrieve the position name when Organizational Management (OM) is not active.

    • DM: The birth name, place of birth, and country are mandatory fields. The system validates these fields, and if they are not populated, message 057 is triggered.

    • PF: When IKMPSV is not known and it's received first time as feedback for A1 then IKMPSV isn't set correctly in internal DB tables. This will cause preparation of correction (A4) message.
  • MH

    • Adjustments to logical checks:

      • 042 – changed from Error to Warning
      • 065 – removed
      • 081 – removed
      • 099 – the check is not performed for deferred income
      • 243 – check extended to include fields 10307, 10309, 10310
      • 331 – added
    • Implementation of a new methodological rule for deferred income, scenario 2 (2 forms for one period) – the “Primary PPV” attribute is no longer populated in the form for deferred income.

    • For forms of type “Deferred income,” the value “O” is now saved in field T7CZJHM01-ROW_TYPE. Previously, forms were not distinguished and the field value was always “I”.

    • Changes in the administrator mode of the Workbench program (HCZCJHM0_WB, transaction PC00_M18_JHM_WB_ADMI).
      The program itself does not change. The ALV function called from drill-downs to individual and summary forms is changed.

      • Individual forms

        • It is now possible to change the status of individual forms. The change is logged. To make the change visible, you need to leave the individual-forms ALV.

        • It is now possible to display an overview of changes made to the given form by both the “Change status” and “Change XML” actions.

      • Summary forms
        • It is now possible to display an overview of changes made to the given form by both the “Change status” and “Change XML” actions.
    • Sending JMHZ/MH now automatically prepares and sends a processing request = JMHZ/DZMH.
    • The ALV for displaying individual and summary forms now shows messages from DIS and DZMH separately.

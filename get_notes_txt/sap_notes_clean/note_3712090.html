Symptom

  • An error occurs when attempting to add a custom field (Z field) in the VBAP table using the Custom Fields & Logic App (CFL).
  • While checking VBAP table, it will show "Partially active"
  • Then observe the error message displayed are: 
    • "Table change not possible. There is a DB trigger on the table."
    • "Table VBAP (Statements could not be generated)"

Environment

  • Custom Fields app
  • SAP S/4HANA 2023

Reproducing the Issue

  1. Open the CFL app.
  2. Add custom field to table VBAP.
  3. Publish the custom field.
  4. Check VBAP; it will show "Partially active."
  5. Press on "Check" and observe the error messages.

Cause

The issue occurs because there are database triggers which exist on an SAP table. However, these triggers are not known to the SAP system.

This is a correct behavior and it works as designed. Database triggers are not created or managed by SAP. If a table has database triggers which are unknown by SAP, table change on ABAP level is prohibited.

Resolution

1. Make sure to complete setting up the S_ATO first. Follow the steps outlined in the following SAP Notes:

  • Refer to: 0002283716 - Key User Application is not configured (for this SAP Note, follow the resolution steps for ABAP-based on-premise products).
  • Refer to: 0002807979 - How to setup Adaptation Transport Organizer for S/4HANA On Premise or Private Cloud system via S_ATO_SETUP.

2. After completing the setup, refer to the following SAP Knowledge Base Article (KBA) to resolve the error "Table change not possible. There is a DB trigger on the table":

  • Refer to: 0002284776 - Table change not possible. There is a DB trigger on the table.

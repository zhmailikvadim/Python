Symptom

Symptom

Exchange object key register modules are used to store saved exchange objects in memory to avoid duplicate exchange triggers in a particular LUW session. The buffer refresh is not properly handled with application specific modules when the transaction updates are called one after another without re-launch of a particular transaction.

Other Terms

ERPADDON-361, Exchange

Reason and Prerequisites

Programming Error

Solution

Applied attached Correction Instruction.

NOTE: This note is dependent on the following core note, please install this note before applying these corrections.

2090353 - Refresh xChange Cache Tables at the end of each xChange Processing LUW

 

1) In the Payroll (HIECALC0), the Late New Starter periods all have the pay date of the period in which the employee is first paid (i.e. the IN period pay date). This means that the checks on structure IETAX were performed in all the Late New Starter periods if the employee is paid for the first time after the SMART PAYE Go Live date. The most commonly issued payroll rejection was HRIESP(021).

Employment Id is mandatory. Please run Employee List (HIECEEL0)

This note changes the pay date handling, so that the checks on structure IETAX are only performed in the Late New Starter periods if the Late New Starter period has a pay date after the SMART PAYE Go Live date (according the T549S).

2) In the Payroll Submission (HIECPAY0), the Late New Starter periods are all treated as "N"ormal submissions and in most cases they don't have much data. The fields that do contain data are as follows; ER's PRSI'able Pay, PRSI Class, PRSI Sub-Class, PRSI Weeks, Address (for employees without a PPSN) and Start Date. Each of these have been aggregated into the record submitted for the period in which the employee is first paid (i.e. the IN period).

This note changes the handling of Late New Starters in the Payroll Submission, so that only the period in which the employee is first paid (i.e. the IN period) is submitted and all the other periods are simply aggregated into that one submission.

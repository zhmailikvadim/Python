Symptom

During mass release of settlement documents via transaction WAB_RELEASE, the system encounters a short dump (MESSAGE_TYPE_X) in program /TXINTF/SAPLHOOK_FI, causing termination of the settlement release process.

Other Terms

 SETI, TXINTF

Reason and Prerequisites

While processing pricing data, field CS_KOMK-TRTYP remained empty despite being assigned from memory in program SAPLWLF1. Table T180 contained no corresponding entry, resulting in the transaction type not being populated. As this field is required for subsequent buffer validation, the process was skipped, leading to a runtime dump.

Solution

A new configuration entry has been maintained in transaction SM30 under view V_T180 with the following values:

Transaction Code (TCODE): WAB_RELEASE
Program: SAPLWLF1
Transaction Type: H
Activity Category: V

All other fields are left blank.

Additionally, new validation logic has been introduced to support the mass release process. If the settlement document is being processed through the WAB_RELEASE transaction and the settlement document type is available, the system will automatically refer to the corresponding entry maintained in view V_T180 to derive the required transaction type.This ensures that the transaction type field is properly populated for subsequent validation steps, allowing the settlement release process to proceed without interruption.

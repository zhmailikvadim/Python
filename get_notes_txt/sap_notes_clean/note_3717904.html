Symptom

  • An error occurs during the "Define dunning configuration" step when attempting to save changes.
  • The error message displayed is: "No primary table is defined for Customizing Object."
  • Prior to the error below messages are displayed:
    • Dunning levels: 4 not maintained; old dunning level is saved, Message no. FT367
    • Dunning level data has still not been entered, Message no. FT307

"Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental." 

Environment

SAP S/4HANA Cloud Public Edition

Reproducing the Issue

  1. Navigate to the "Define Dunning Procedures" (SSCUI 102302) activity.
  2. Attempt to configure the dunning procedure without entering dunning levels.
  3. Click the Save button.
  4. Observe the error message: "No primary table is defined for Customizing Object."

Cause

The error occurs due to missing preceding configuration values in the dunning setup process.

Resolution

  • Ensure all required dunning levels are maintained and saved during the configuration process.
    1. Click on Dunning levels button on the screen
    2. Maintain the dunning levels and parameters.
    3. Go back and Save the dunning procedure

  • Verify preceding configuration activities as outlined in the SAP Help Portal under "Configure Dunning Process Flows".
  • Refer to the Knowledge Base Article 3550088 - Interest Calculation, Dunning and Correspondence overview / how-to scenarios for additional guidance on interest calculation, dunning, and correspondence scenarios.

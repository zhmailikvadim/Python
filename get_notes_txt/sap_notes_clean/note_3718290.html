Symptom

When a sample certificate generated in SAP Cloud Connector is imported into transaction CERTRULE in the backend system, the following issue occurs:

  • The mapped email address is reported as ambiguous.

As a result, certificate-based user mapping cannot be completed successfully.

Environment

SAP Cloud Connector 2.16 - 2.18

Reproducing the Issue

  1. A sample certificate is generated in SAP Cloud Connector.
  2. The certificate is imported into transaction CERTRULE in the backend ABAP system.
  3. The system reports that the mapped email address is ambiguous.

Cause

Multiple user accounts exist in transaction SU01 with the same email address maintained。

Since certificate mapping in CERTRULE uses the email attribute to identify a user, the system cannot uniquely determine which user should be mapped.

Resolution

  1. Execute transaction SU01 in the backend system.
  2. Search for users using the email address referenced in the certificate.
  3. Ensure that the email address is assigned to only one user.
  4. Remove or adjust duplicate email entries if necessary.
  5. Repeat the certificate mapping in CERTRULE.

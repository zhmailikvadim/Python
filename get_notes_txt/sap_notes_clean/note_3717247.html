Symptom

The HASH entries in the /PACG/ECM_AHASH table are always created at the company level. Therefore, if you pass an organization-level ID instead of a company ID to the check_object_already_sent method, you'll need to determine the correct company ID before proceeding. Currently, if you pass an organization-level ID to the check_object_already_sent method, it's used directly in the selection, so the result may be incorrect. That is, if an object (equipment or functional location) has already been sent, the selection won't be able to find any corresponding entry. This will prevent some data from being added to or generated from iDoc documents.

Other Terms

FSM, E4C, Connector, /PACG/ECM_AHASH, Organizational level, Equipment, Functional location, Parent Id

Reason and Prerequisites

Incorrect logic in class /PACG/ECM_CL_X_UTIL2, method CHECK_OBJECT_ALREADY_SENT.

Solution

Please, implement this correction instruction.

Symptom

When the pair-ship is active and picking is executed using a load carrier, the system creates two PDA records instead of one.

Other Terms

IS-MFM, CORE, RAF Application Core, Picking, PDA Management, Pair-Ship, Load Carrier, SAP Meat and Fish Management by msg.

Reason and Prerequisites

This issue occurs in systems where the new PDA Management is active, but the legacy pair-ship logic is still executed.

Solution

History:
Note created on 07.07.2025.

Description of Correction:
The correction ensures that when the new PDA Management is active, the legacy pair-ship logic is bypassed to prevent duplicate PDA record creation.

Functional Dependencies:
This correction is relevant only if the new PDA Management functionality is activated in the system.

Manual Changes:
No manual changes are required.

Symptom

In transaction "ME32L" where the system displays an error message "Net Price must be greater than 0," when attempting to save a new schedule line. 

Environment

  • Materials Management (MM)
  • SAP S4/HANA 

Reproducing the Issue

  1. Enter transaction ME32L.

  2. Provide scheduling agreement number.

  3. Try to add another line item by providing the details in material, target quantity, and net price.  

  4. After clicking enter, save the scheduling agreement.   

  5. Error appears: - "Net Price must be greater than 0 - 06218"   

Cause

In table PRCD_ELEMENTS, entry for previous items in SA does not exist. Due to this, when checking the item conditions screen against these items or making changes to this SA, the system re-checks the price, but no price can be determined, then error 06218 shows. In addition, the table PRCD_ELEMENTS is only valid in S/4HANA release. In ECC releases, the table used for document conditions is KONV.

Resolution

Perform the price determination manually with clicking the update button with option B in item conditions screen against those items whose entry does not exist in PRCD_ELEMENTS so that system can fetch the valid condition properly. This is only valid if the scheduling is ‘time-independent’.

Keywords

Scheduling agreement, PRCD_ELEMENTS, net price, 06218, item conditions, price, time-independent

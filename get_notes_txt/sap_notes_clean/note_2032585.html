Symptom

  • In IDT connecting to ECC system and using abap func module to supply data to IDT Universe We have added a new table output on ECC abap side but IDT is not recognizing these changes.
  • Refresh Structure on the data foundation does not detect new table in abap func module output.
  • ABAP Function Modules do not update new columns

Environment

  • SAP BusinessObjects Business Intelligence platform 4.x
  • Information Design Tool (IDT)

Reproducing the Issue

  1. Create JCO Connection to SAP ECC ERP Data source
  2. The Data Source is an ABAP Function Module
  3. Create a  Multi-Source Data Foundation
  4. Generate a Business Layer and publish.
  5. Add a New field to the ABAP Function Module in ERP
  6. Retrieve the Unx Universe
  7. Select the Data Foundation
  8. go To Refresh Structure
  9. New objects are not detected.
  10. In the Connection select view data and new columns are visible.

Cause

The Data Foundation is looking at the Cached universe properties. The Multi-source universe goes  to get the metadata from the BI platform, then to the federation engine and then to the ERP machine.  Whereas a Single Source universe does directly to the data source.

Resolution

Perform a Stop and Start of the Adaptive Processing Server to clear the cache.

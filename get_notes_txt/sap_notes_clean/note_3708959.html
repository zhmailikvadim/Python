Symptom

The Concur API /detokenizer/v4/company/{company_id}/creditcard/{token_id} fails with error 431 Request Header Fields Too Large.

Environment

  • SAP ERP
  • SAP ERP integration with SAP Concur solutions
  • SAP S/4HANA
  • SAP S/4HANA Cloud Private Edition
  • SAP S/4HANA integration with SAP Concur solutions

Reproducing the Issue

Check Connection at CTE_SETUP >> System >> Check Connection.

- OR -

You can try to fetch the credit card number from Concur using the mentioned API - test report CTE_FIN_GET_CCARD_NR.

Cause

This error has been identified until this moment only in environments where SAP middleware CPI is used in-between SAP S/4HANA and Concur.

The Detokenization API already has a large URI so depending on the amount of scopes assigned for the APP (Client ID) in Concur, the JWT token size exceeds the maximum permitted size of the HTTP Request Header.

Therefore, this is cause by Detokenization API peculiarity + amount of scopes for the Client ID + usage of CPI in CTE_SETUP.

Resolution

Reduce the amount of scopes assigned to the Client ID in Concur to the less number of scopes possible.

If reducing is not possible, segregate the tasks in using multiple Client IDs.

For example: one APP for master data export such as Employees, Cost Center, Internal Orders, Project WBS Elements, and so on... and another for the Financial Posting integration only.

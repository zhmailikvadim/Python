Symptom

When attempting to load cell values into a multidimensional lookup table (MDLT) with return type set to "Date" using the "PLMC_Template", the process fails and displays the error message: "The imported cell of Lookup Table, <MDLTNAME>, has invalid date format."

Environment

  • SAP Sales Performance Management
  • Incentive Management

Reproducing the Issue

  1. Upload the "PLMC_Template" file filled out for the multidimensional lookup table (MDLT).
  2. Observe the error message: "The imported cell of Lookup Table, <MDLTNAME>, has invalid date format."

Cause

Check to see if there is the correct Date format in the dimension fields is the correct format of DD-MMM-YYYY.

Resolution

  1. Open the file that caused the error.
  2. Update the DIMXLOW and DIMXHIGH date values in the file to the correct format: DD-MMM-YYYY.
    1. Ex: DIM0LOW = 01-AUG-2026 | DIM0HIGH = 31-AUG-2026
  3. Re-upload the corrected file using the Express Data Loader UI.
  4. Refer to the attached "PLMC_Template.xlsx" for the Mapping and Validation section, which details the proper date format.
  5. If the issue persists after correcting the date format, contact support for further assistance.

Symptom

The Logon Data Assignments shows blank in Create new service group based connection in Integration Scenario Configuration for Central Configuration.

It can be checked in Step 2.5 Create Integration Scenario Configuration when implementing SAP Note 3489604 - Customs Message Exchange: Configuration Guide for setting up communication.

Environment

  • Global Trade Services (GTS)
  • Global Trade Services / Logistics Services (SLL)
  • SAP Global Trade Services 10.1
  • TRANSPORTATION MANAGEMENT (TM)
  • ABAP PLATFORM

Reproducing the Issue

  1. From the SOAMANAGER landing page navigate to Integration Scenario Configuration
  2. Provide a meaningful name and description and choose Create
  3. Create a new Service Group Connection 
  4. Assign IBC References
  5. Assign Service Groups
  6. Define Matching Interfaces
  7. Once the service-group-based connection is created, select it, and choose Logon Data Assignments

Cause

The Logon Data type created in Logon Data Management is not Template type.

Resolution

Check the Logon data created in Logon Data Management, which should be Template type, not Local.

Keywords

Logon Data Assignments, Central Configuration, Integration Scenario Configuration, Service Groups

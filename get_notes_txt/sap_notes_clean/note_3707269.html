Symptom

  • When exporting a package containing only folders, the "Share Team" setting initially appears as "On".
  • After the export process is completed, the "Share Team" setting automatically changes to "Off". 

Environment

SAP Analytics Cloud

Reproducing the Issue

  1. Log on to SAC
  2. Navigate to Transport --> Export --> Content Network Storage.
  3. Click on New Export --> Select only ABC folder from Public (without dependencies) --> Next.
  4. Enter the Name, Description, Details.
  5. Enable "Share Team" setting.
  6. Click on Next --> Export.
  7. Wait until the .package is created.
  8. Click on the .package.

Notice : The "Share Team" setting is off.

Cause

Each object supports two types of access permissions:

  • Native permissions: Permissions explicitly assigned to the object itself
  • Inherited permissions: Permissions inherited from the parent folder

Currently, Content Network supports transport of native permissions only.

If a folder has all its permissions inherited from the parent folder, and no native permissions are defined on it. As a result, when the folder is exported, no team share settings are included in the package.

Resolution

Workaround : Import the package into the destination system and manually share the folder with the respective teams after import.

Symptom

Since the TU capacity is usually calculated in a subsequent step (synchronization), the system issues an error message too late to prevent the assignment or loading. For this reason, exceeding the permitted capacities does not lead to an error, but only to a warning in the log. 
However, the message prevents the correct calculation of the TU capacity, which leads to incorrect loading weights/loading volumes.
If is loaded with warehouse tasks, this means that the checkout can no longer be canceled because the cancellation is prevented by exceeding the permitted capacity.

Reason and Prerequisites

Program error

Solution

Only the first capacity that exceeds the permitted capacity is written to the log, but the loaded values are calculated correctly. When the checkout is canceled, the maximum permitted capacity is deleted to enable a reverse booking.

Implement the attached correction instructions.

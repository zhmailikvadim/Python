Symptom

  • In SAP Incentive Management integrated with SAP Cloud Identity Services - Identity Authentication, activation (onboarding) emails are not sent to users created via Identity Provisioning.
  • Users may expect newly provisioned users to automatically receive activation emails. However, this behavior is controlled through target system transformation settings in Identity Provisioning.
  • The configuration responsible for triggering activation emails is not explicitly enabled by default, which can lead to confusion regarding how to enable or disable onboarding email notifications. As a result:
  • Newly created users do not receive activation emails
  • There is limited visibility on how email notifications are controlled
  • Users may be unaware of the required transformation setting (sendMail) in the IAS target configuration

    Environment

    • SAP Sales Performance Management 
    • Incentive Management on Hyperscalers 
    • Incentive Management on Legacy HANA 

    Cause

    • In SAP Cloud Identity Services - Identity Authentication for SAP Incentive Management, the activation (onboarding) email is controlled by the sendMail attribute in the target system transformation configuration.
    • By default, during user provisioning from SAP Incentive Management via Identity Provisioning, the sendMail parameter is either:
    • Set to "False", or Not explicitly maintained in the transformation mapping
    • As a result, the provisioning process creates users without triggering activation emails.
    • Since this behavior is governed at the transformation level and not exposed as a visible setting in the standard UI, users may be unaware that email notifications are disabled by default. 
       
       

      Resolution

      To enable activation (onboarding) emails for users provisioned from SAP Incentive Management to SAP Cloud Identity Services - Identity Authentication, the sendMail parameter must be enabled in the target system transformation configuration.

      Follow these steps:

      1. Navigate to:
        Identity Authentication → Identity Provisioning → Target Systems
      2. Select the relevant IAS target system.
      3. Go to the Transformations tab.
      4. Switch to JSON Editor and choose Edit.
      5. Locate the following transformation entry:
        {
           "constant": "false",
            "targetPath": "$.sendMail",
            "scope": "createEntity"
        }
      6. Change the value of "constant" from  "false" to "true"
        {
            "constant": "true",
            "targetPath": "$.sendMail",
            "scope": "createEntity"
        }
      7. If needed, this setting can be reverted by changing the value back to "false" to disable activation emails.
      8. please refer to the detailed help article in the see Also section

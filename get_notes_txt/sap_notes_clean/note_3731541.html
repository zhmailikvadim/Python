Symptom

  • FF logon is successful, and the session is active. 
  • RFC is not responding when attempting to log in to the respective system.

Environment

  • SAP Access Control 12.0
  • SAP Access Control 10.1

Reproducing the Issue

  1. Execute GRAC_EAM
  2. Select FFID
  3. click on logon
  4. Provide Reason codes
  5. click on logon. 

Cause

  • rfc/reject_expired_passwd RZ11 parameter is set to 1 by the default profile which rejects all incoming RFC connection with initial password.
  • Firefighters are always have initial password, hence the RFC connection seems rejected.

Resolution

Follow the suggested steps in your system to resolve the issue.

  1. Login into Plug-In system (ECC/S4)
  2. Go to Transaction RZ11
  3. Enter the Parameter: rfc/reject_expired_password
  4. Change this parameter to 0 (default) value, to allow Firefighters to be logon.
  5. It is only applicable if the Firefighter ID is Dialog or Communication user.

Note: The default and suggested user type is Service type for FFID. If that is configured, then this RZ parameter does not apply. 

Explanation:

  1. 0 -> RFC Logins allowed even if password is expired
  2. 1 -> RFC logins rejected if password is expired 
  3. The scenario is if the FFID User type is Dialog or Communication and rfc/reject_expired_passwd is set to 1, then Firefighter Logon will not works due to the FFID has Initial password which is rejected by the RZ parameter.
  4. The solution is to change the FFID user type to Service (suggested and preferable way). If the user type need to be Dialog, then change the rfc/reject_expired_passwd to 0 will be the solution.

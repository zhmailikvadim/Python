Symptom

Error: "com.crystaldecisions.sdk.exception.SDKException$PluginNotFoundAtCMS: The Enterpise plug-in does not exist in the CMS (FWM 02017)" while running a .jsp file

Environment

SAP BusinessObjects Business Intelligence Platform 4.2

Reproducing the Issue

  1. Try to run any .jsp file which contains this snippet of code:

 final String BO_AUTH_TYPE = "Enterprise";

enterpriseSession = CrystalEnterprise.getSessionMgr().logon(BO_USER_NAME, BO_PASSWORD, BO_CMS_NAME, BO_AUTH_TYPE);

  1. Error message "com.crystaldecisions.sdk.exception.SDKException$PluginNotFoundAtCMS: The Enterpise plug-in does not exist in the CMS (FWM 02017)" appears.

Cause

The error message is caused by the wrong Plugin name "Enterprise" assigned to the string  “BO_AUTH_TYPE”.

Resolution

Change the Plugin name from "Enterprise" to "secEnterprise" in “BO_AUTH_TYPE”.

Keywords

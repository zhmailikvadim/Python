Symptom

When creating a parameterized view using CREATE VIEW statement with CASE JOIN condition in it, it throws error: invalid column name: "[Column Name]"

But the column is present in the base view or table.

  • Create statement example: 
    CREATE VIEW VIEW_NAME( IN P_PARAM NVARCHAR(000042) ) AS 
SELECT
	 F1.COL1,
	 F1.COL2,
	 VIEW_3.COL10,
	 F1.COL3
FROM F1( P_PARAM => :P_PARAM )
LEFT OUTER MANY TO ONE CASE JOIN 
	WHEN F1.PLANCODE = 'A' 
	THEN RETURN ( 
		VIEW_1.COL_A,
		VIEW_1.COL_B,
		VIEW_1.COL_C,
		VIEW_1.COL_D
	) 
	FROM VIEW_1
	ON ( F1.KEY1 = VIEW_1.KEY1 
		AND F1.KEY2 = VIEW_1.KEY2 
		AND F1.KEY6 = VIEW_1.KEY6 ) 
	WHEN F1.PLANCODE = 'P' 
	THEN RETURN ( 
		VIEW_2.COL_A,
		VIEW_2.COL_K
	) 
	FROM VIEW_2
	ON ( F1.KEY7 = VIEW_2.KEY7 
		AND F1.KEY6 = VIEW_2.KEY6 ) 
END AS CASE_JOIN_RESULT
LEFT OUTER MANY TO ONE JOIN VIEW_3
ON ( F1.KEY6 = VIEW_3.KEY6 
	AND F1.KEY8 = VIEW_3.KEY8 )

Other Terms

Invalid column name,  Syntax Error

Reason and Prerequisites

Issue number 347890

Reason:

Due to a programming error in SAP HANA, creating a parameter view using CREATE VIEW statement with CASE JOIN condition would result in preprocessor rules not properly propagated, resulting in it throws 'invalid column name' error even when the column exists.

Affected Releases:

  • SAP HANA 2:
    • Revisions of SPS00 - SPS06
    • Revisions <= 079.07 (SPS07)
    • Revisions <= 087.00 (SPS08)

Prerequisites:

  • View is created using CREATE VIEW statement.

Solution

Apply one of the following SAP HANA Revisions:

  • SAP HANA 2:
    • Revisions >= 079.08 (SPS07)
    • Revisions >= 088.00 (SPS08)
    • or higher

Workaround:

  • Until the database is upgraded to a revision containing the fix, please change the CASE JOIN order in the create statement, to place the case join at the last position in the join order.

    • Workaround statement example: 
      CREATE VIEW VIEW_NAME( IN P_PARAM NVARCHAR(000042) ) AS 
SELECT
	 F1.COL1,
	 F1.COL2,
	 VIEW_3.COL10,
	 F1.COL3
FROM F1( P_PARAM => :P_PARAM )
LEFT OUTER MANY TO ONE JOIN VIEW_3
ON ( F1.KEY6 = VIEW_3.KEY6 
	AND F1.KEY8 = VIEW_3.KEY8 )
      LEFT OUTER MANY TO ONE CASE JOIN 
	WHEN F1.PLANCODE = 'A' 
	THEN RETURN ( 
		VIEW_1.COL_A,
		VIEW_1.COL_B,
		VIEW_1.COL_C,
		VIEW_1.COL_D
	) 
	FROM VIEW_1
	ON ( F1.KEY1 = VIEW_1.KEY1 
		AND F1.KEY2 = VIEW_1.KEY2 
		AND F1.KEY6 = VIEW_1.KEY6 ) 
	WHEN F1.PLANCODE = 'P' 
	THEN RETURN ( 
		VIEW_2.COL_A,
		VIEW_2.COL_K
	) 
	FROM VIEW_2
	ON ( F1.KEY7 = VIEW_2.KEY7 
		AND F1.KEY6 = VIEW_2.KEY6 ) 
END AS CASE_JOIN_RESULT

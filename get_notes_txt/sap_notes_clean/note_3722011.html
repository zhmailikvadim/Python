Symptom

Message processing fails intermittently during outbound HTTP/OData calls with errors similar to:

  • com.sap.gateway.core.ip.component.odata.exception.OsciException HTTP Request failed with error: Cannot assign requested address Cause: java.net.BindException: Cannot assign requested address

or

  • error 99 - Cannot assign requested address (Address not available) java.net.NoRouteToHostException

The issue typically:

  • Occurs randomly under high load.
  • Happens when a large number of outbound connections are created in a short time period.
  • May temporarily resolve without changes.

Environment

  • SAP Integration Suite
  • SAP Business Technology Platform
  • Cloud Integration

Cause

The error “Cannot assign requested address” indicates exhaustion of available ephemeral (source) ports at operating system level.

When a high number of outbound HTTP connections are created in a short time:

  • Many TCP connections remain open or in TIME_WAIT state.
  • The OS runs out of available source ports.
  • New connections cannot be established.
  • The runtime throws java.net.BindException.

This is a platform-level networking limitation and not related to a specific backend system.

Resolution

1. Enable Connection Reuse:

  • Enable “Reuse Connection” (HTTP Keep-Alive / connection pooling) in the HTTP or OData adapter settings. If the option is not available, update the adapter to the latest version as per 3260097 - How to update Cloud Integration components' version.
  • Ensure connection reuse is enabled in all iFlows connecting to the same receiver system.
  • See 3381958 - How to enable Reuse Connection in CPI for ODATA and SuccessFactors ODATA adapter for further details.

2. Reduce Parallelism / Optimize Load:

  • Review the integration design along with CPI internal developers to:
    • Reduce batch sizes.
    • Limit parallel processing.
    • Avoid excessive short-lived connections.
    • Review retry mechanisms.
    • Follow best practices for handling large message volumes.
  • See Manage Large Batch Sizes for further details.

3. CPI runtime VM restart:

If connections remain stuck and the issue continues:

  • Create a ticket in LOD-HCI-PI-OPS component in order a restart of the runtime/VM nodes can be performed.
  • Provide the Cloud Integration URL and a 15–20 minute maintenance window.

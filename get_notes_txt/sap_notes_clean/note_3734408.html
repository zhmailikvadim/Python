Symptom

Search Help values are not populated for a custom field added to an app and when opening the Value Help for the custom field, no entries are displayed and the following error is observed in console:

"TypeError: cannot read properties of undefined”

Environment

  • SAP S/4HANA 2023
  • Fiori elements OData V4
  • Custom Fields and Logic (App ID: F1481)

Cause

  • In the OData V4 Service Binding metadata for the ValueList of the custom code list field, the property path was not escaped correctly when the custom field identifier contains "/" character.
  • The unescaped slashes were interpreted as navigation, leading to a broken value help binding and no results.

Resolution

  1. Implement SAP Note 3536314.
  2. Clear caches as described in the cache housekeeping KBA 2319491.
  3. Reload the app and retest the scenario.

See Also

  • KBA 2319491 - How to clean the cache in SAP Fiori

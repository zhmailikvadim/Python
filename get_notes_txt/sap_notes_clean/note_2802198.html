Symptom

When you run the "Programa de cálculo de nómina Argentina" (HARCALC0) report, the Cálculo de aportes y contribuciones a la Seguridad Social (ARSES) payroll function may incorrectly calculate the Welfare (Obra Social) withholding in the following scenario:

  • The employee is part time
  • The theoretical remuneration is higher than the employee's remuneration
  • The employee has a SAC payment during the semester

In this scenario, the Imp.tope compl.O.Soc. (/374) wage type may incorrectly consider the T.Parc SAC. Proporc. (/3C4) wage type accumulated of the semester in its generation. This may lead to an inconsistent base to the Welfare withholding calculation.

Other Terms

&SSOCIAL (further information see SAP Note 1152671)

PC00_M29_CALC; /361; /362; /363

The following Support Package level is the minimum prerequisite:

ReleaseSupport Package

Solution

Version Date Comments
2 August 19, 2019 Released for customer

From now on, the generation of the /374 wage type will not consider the accumulated /3C4 wage types of the semester, unless there is a Importe bruto SAC p/seg. (/115) wage type in the period.

As general rule, SAP recommends that you install a solution by applying a Support Package. However, if you need to install the solution earlier, proceed as follows:

    1. Use Note Assistant (SNOTE) to implement the correction instructions

      You can find more information about the Note Assistant in SAP Support Portal, under support.sap.com/kb-incidents/notes/assistant.html.

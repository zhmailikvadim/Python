Symptom

Mapping is applied to Line item custom form in Guided Buying to map value(s) to custom fields on Requisition. Mapping is not working and value is not mapped/transferred. Instead we get NULL or Not specified value on Requisition.

Reproducing the Issue

Create GB Custom Form and use Application Mapping on one of the fields.
Map value to Custom field on Requisition

Cause

Custom fields on Requisition are not supported for mapping from Line Item forms in Guided Buying. Supported fields are only ones on Line-item level.

Resolution

Per documentation custom fields are supported for application mapping on Line item forms. However, this is fully supported only for Line item level fields and not for Requisition-level fields.
For example cus_myField on Requisition object is not supported for mapping, but same mapping to field on LineItem level is supported and can be used.

This is caused by the mapping limitations as current design allows only mapping to Line Item level fields and some Header-level fields which are listed in Mapping Custom fields documenation as "standard line item fields".

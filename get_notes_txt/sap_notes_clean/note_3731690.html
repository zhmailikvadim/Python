Symptom

IBP Excel Add-in advanced filtering in master data not working as expected; shows "No data found; Check your filter, or ask your administrator to review the permissions assigned to you."

Environment

  • SAP Integrated Business Planning
  • SAP IBP, add-in for Microsoft Excel (User Interface)

Reproducing the Issue

  1. Log into a planning area via the SAP IBP, add-in for Microsoft Excel.
  2. Create a master data workbook with Product on one tab and Location Product on another tab.
  3. Apply simple filters: for Product, filter by an attribute value (for example, a model year); for Location Product, filter by a matching Product ID.
  4. Confirm the selected Product exists in Location Product and the attribute value matches.
  5. On the Location Product tab, remove the simple filter and apply an Advanced filter using the Product attribute (for example, model year).
  6. Observe the message: "No data found; Check your filter, or ask your administrator to review the permissions assigned to you."

Cause

The global configuration parameter PLAN_VIEW | EXCEL_FILTER_LOGIC was not set for advanced filtering logic (set to 1), so evaluation did not use existing planning objects or modeled master data relationships.

Resolution

  1. Ensure the SAP IBP, add-in for Microsoft Excel is version 2508.2.0 or newer.
  2. In Global Configuration, set PLAN_VIEW | EXCEL_FILTER_LOGIC to 2 to evaluate based on existing planning objects or modeled master data relationships.
  3. Save the change, refresh the workbook, and reapply the Advanced filter on the master data tab.
  4. Verify that data is now returned as expected when using Advanced filtering.

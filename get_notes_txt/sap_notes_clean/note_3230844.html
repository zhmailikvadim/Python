Symptom

This SAP Note informs you about all restrictions in this release.

Other Terms

Release restrictions, SAP S/4HANA 2022, SAP S/4HANA 2022 FPS01

Reason and Prerequisites

This SAP Note provides information about the restrictions that exist for the SAP S/4HANA 2022 release.

For general release information on SAP S/4HANA 2022 refer to SAP Note 3145277.

Note: This SAP Note is subject to change. Check this SAP Note for changes on a regular basis. All important changes are documented in section "Important Changes after Release of SAP S/4HANA 2022".

Solution

General Restrictions:

  • In SAP S/4HANA the maximum length of the material number has been extended to 40 characters. Long material number-related restrictions are described in SAP Note 2233100.
  • In SAP S/4HANA currency amount fields with a field length between 9-22 including 2 decimals have been extended to 23 digits including 2 decimals. Related restrictions are described in SAP Note 2601956.
  • The extensibility framework is currently not integrated in the privacy by default functionality of S/4HANA. Please be aware, when you use this functionality for processing of personal data you might also need to provide this integration as part of your enhancements.
  • The released information for add-ons can be found in SAP Note 2214409.
  • The attached Business Functions are not released with SAP S/4HANA 2022.       
    • It is not permitted to switch them on, even if it is technically possible (see attachment). Additional information regarding Always-Off and Always-On Business Functions can be found in SAP Notes 2240359 and 2240360.
    • If a Business Function was switched on in the start release system, but defined as always off in SAP S/4HANA, then a system conversion is not yet possible with this release at that current point in time.
    • If a Business Function was switched off in the start release system, but defined as always on in the target release, then the Business Function will be automatically activated during the conversion
  • Restrictions for ABAP Platform 2022 as part of SAP S/4HANA 2022 are described in SAP Note 3225504.
  • Restrictions for Java components as part of SAP S/4HANA 2022 are described in SAP Note 3225521.
  • For CoPilot skills available with S/4HANA 2022 and related restrictions refer to SAP Note 3224632.
  • Restrictions apply to the usage of the CDS View Replication functionality; for details refer to SAP Note 2977657.

Conversion Restrictions:

Restrictions for system conversions to SAP S/4HANA 2022:

  • The conversion from source releases below SAP ERP 6.0 EHP 5 (SAP_APPL 605) is supported for systems containing vendors with assigned contact but you need to ensure to update to the following minimum SP levels upfront.
    In addition, you need to implement corrective SAP Note 2383051 and further corrections into the system. For this purpose, please get in contact with SAP via customer incident on component AP-MD-BF-SYN.
    The minimum SP levels are: 
      1. SAP ERP 6.0 SP22
      2. EHP2 FOR SAP ERP 6.0 SP10
      3. EHP3 FOR SAP ERP 6.0 SP09
      4. EHP4 to EHP8 FOR SAP ERP 6.0 No minimum SP required.

   This is not relevant for systems which do not have vendors with assigned contacts. You can also verify for existence of records in table KNVK where LIFNR has non-initial values (KNVK-LIFNR <> ‘’).

  • Restrictions in conversions for DIMP-LAMA: see SAP Note 2384347.
  • The following add-ons cannot be converted to SAP S/4HANA 2022 in one step but require an intermediate upgrade step:
    • The Israeli Annexing Transport solution (see SAP Note 2298073 for more details)
    • SRM_SERVER 550/SRM_PLUS 550 (see SAP Note 2251946 for more details)
    • SAP Financial Closing cockpit 1.0 (see SAP Note 2513315 for more details)
    • SAP Revenue Accounting and Reporting 1.0, 1.1, 1.2 (see SAP Note 3214045 for more details)
  • For restrictions concerning Cost Of Goods Manufactured (COGM) please see SAP Note 2270414.
  • For restrictions concerning New Inventory Management Data Model, see SAP Note 2493434.

Performance Restrictions:

  • Performance restrictions for SAP S/4HANA 2022 can be found in SAP Note 3254104.

Core Solution Restrictions:

  • Restrictions related to Responsibility Management: You as data controller are responsible for ensuring that the data used in responsibility definitions in teams are managed in accordance with any applicable legal requirements or business needs, such as data protection legislation or data life cycle requirements. Responsibility definitions in the Fiori app "Manage Teams and Responsibilities" are currently not integrated in privacy by default functionality. Therefore, the responsibility definitions in teams should not be used for the processing of personal data if this processing falls under any applicable data protection legislation.
  • Information regarding Fiori application 'Manage Product Master' (Master Data Maintenance – Product Master): in the Fiori application Manage Product Master for valuation and supply planning you must maintain field values of length less than nine for price fields. This is the field length that is supported until the S/4HANA 2021 release. The system will not process the values in these fields in case you specify a value that is nine or more than nine digits.
  • Restrictions for "Multiple Address Handling in SD Documents Using SAP Business Partner" are described in SAP Note 3081750.
  • For Procurement / app Manage Supplier Confirmations the following restrictions apply:
    • Only documents of type NB (standard purchase order) or direct copies with the same settings as NB are supported.
    • Only the item categories Standard (blank) and E (Enhanced Limit) or direct copies with the same settings can be selected.
    • The Manage Supplier Confirmations app cannot be used for the intercompany stock transfer process.
  • Since SAP S/4HANA 2021 FPS2, in Plant Maintenance Order/Work Order a new functionality was added, which allows end users to activate mandatory checks on component fields via the configuration (SPRO: Plant Maintenance and Customer Service -> Maintenance and Service Orders -> EAM Message Control). With this end users get info/warning/error messages if certain fields are not maintained (Storage Location, Goods Recipient, Unloading Point, Recipient Location and Quantity). The issuing of messages does not work for components added to the PMO through a tasklist, an API or the order copy functionality.

Symptom

This note includes the following corrections in the S/4 Defense Disconnected system:

Issue: When BAPI_OUTB_DELIVERY_CREATE_STO is executed to create an outbound delivery, it triggers the EFI event for the outbound delivery before COMMIT WORK is called. As a result, a queue entry is created with empty outbound delivery data.

Resolution: To resolve this issue, queue creation will be skipped when both the header (LIKP) and item (LIPS) data are empty.

Other Terms

STO, Outbound Delivery, BAPI_OUTB_DELIVERY_CREATE_STO, EFI, S/4 Defense Disconnected.

Solution

Implement the corresponding CI as per the note.

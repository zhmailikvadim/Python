Symptom

Error KT206 – "Maintain the control indicators correctly" occurs when posting controlling or billing documents.
The "All Currencies" indicator in transaction OKKP is inactive for controlling area.

Environment

SAP ERP, SAP S/4HANA, SAP S/4HANA Finance

Reproducing the Issue

Open transaction OKKP for the affected controlling area.
Notice that the "All Currencies" indicator is inactive.
Attempt to activate it for the current fiscal year.
The system returns KT206.

Cause

For effected Co Area “All Currencies” flag (TKA00-ALLEW) has been deactivated.

Resolution

  • Once the flag ‘All Currencies’ (TKA00-ALLEW) has been turned off it will not be possible to turn it back on.
  • If transaction data already exists, you cannot change this indicator. To do so could lead to program errors and data inconsistencies (for example, when planning or when settling orders).
  • Deactivating the indicator is always possible. You cannot reactivate it during the ongoing fiscal year, but you can in the following year. Activation during a fiscal year with existing data is not allowed.
  • If there are values in the controlling area or on orders, you should only make changes in exceptional cases. In some circumstances, changing the indicator involves data conversion.
  • Only CO-OM-CCA is active:
    Deactivating the indicator is always possible. If you deactivate it, you cannot reactivate it until the next fiscal year. Activation during the current fiscal year with existing data is not possible.
  • CO objects with validity periods that continue beyond the current fiscal year are active (such as orders):
    If you want to reactivate the indicator, you must first make sure that the existing values are also converted in the field groups for the transaction currency and object currency. Otherwise, the System creates inconsistencies. Deactivation also results in inconsistencies because orders, for example, can no longer be settled to zero.
  • If you require support from SAP on data conversion, see Note 90240 .

Keywords

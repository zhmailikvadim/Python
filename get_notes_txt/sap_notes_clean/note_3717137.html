Symptom

  • Input address is new development
  • Address assigns to a incorrect address
  • Unexpected Canada assignment

Environment

  • SAP Data Services
  • HANA Smart Data Quality

Reproducing the Issue

Enter a Canadian address that is new development.  

Cause

The Canada post website has a address classification of Proposed Site code also known as module code or CMB's that are include  in the data that is seen on their web site within two weeks of their creation even though  the actual address are not occupied yet.  These address are not included in the dataset that is released to SAP for redistribution as these are still marked as proposed addresses as they should not be considered deliverable at this time.  

Resolution

Addresses that are not occupied are not valid Point of Call addresses in Canada until they are occupied.  They are are not expected to be included in the data until they are occupied.

Once the data is no longer marked as proposed, the data will then be included in the next months release, which will then allow the address to assign correctly.

Symptom

An error occurs when using sorting with Key Figures has a member in A-OLAP.

The error is "com.businessobjects.multidimensional.services.InvalidTupleForSortException: Tuple sorting of [0FISCYEAR:MEMBER/0FISCYEAR/ZZ2004] is not supported.".

Other Terms

AOLAP, BI, BOE

Reason and Prerequisites

Steps to duplicate

  1. Log into BI Launchpad,
  2. Run the application Analysis edition for OLAP,
  3. Create a workspace, based on any data source,
  4. Add any dimension for sorting into Columns axis,
  5. Add key figures which has a member measure and a dimension into Row axis,
  6. Try to sort with dimension in the Columns axis,
  7. Error appears: com.businessobjects.multidimensional.services.InvalidTupleForSortException: Tuple sorting of [*****:MEMBER/*****/*****] is not supported.

Cause

Even Key Figures has a member and when selecting the Key Figures, the member is not selected in 'SelectionState' of the Key Figures.

Solution

The issue has been fixed in the patches listed in the "Support Packages & Patches" section below.
The "Support Packages & Patches" section will be populated with the relevant patch levels once they are released.
For Business Intelligence Platform maintenance schedule and strategy, see the Knowledge Base Article 2144559 in References section.

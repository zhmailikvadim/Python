Symptom

You call the method retrieve_by_association of service manager to retrieve an instance from the target node of a cross association. The instance does not exist, and you are not receiving any data back. Afterward, the instance is created in a different session.

You are trying to read the instance again, but you still might not get it.

Other Terms

BOPF, Retrieve_by_association, Dirty Read

Reason and Prerequisites

Additional development

In case you want to get the result from the method retrieve_by_association, you have to set the input parameter iv_invalidate_cache to abap_true.

Solution

If you want to correct the problem in advance, implement the source code correction.

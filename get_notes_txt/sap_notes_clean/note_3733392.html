Symptom

Once the eDocuments are submitted,

  • The status remains as 'Acknowledged by Service' / 'Acknowledged by DRCC'
  • The status does not change/update automatically even after refreshing the screen

Note: The screenshot shown here is from the Poland Electronic Document Processing solution and is provided purely as an example. This KBA is applicable to all country versions of the Electronic Document Processing solution that utilize DRC Cloud Edition. 

Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.

Environment

  • SAP ERP
  • SAP S/4HANA
  • SAP S/4HANA Cloud Private Edition
  • SAP BTP
  • SAP Document and Reporting Compliance, Cloud Edition

Reproducing the Issue

  1. Launch transaction edoc_cockpit
  2. Submit an eDocument
  3. Refresh the screen and notice that the eDocument status does not change

Cause

The push service is not working.

Resolution

There are several possible reasons for this issue. Please review each item below to help identify the root cause and take the appropriate action.

1. The connection from the DRC cloud edition to your business system (push service) is not working properly. Review the error logs in 'Resolve Error' application and check corresponding KBAs from TSG 3720886  -> Topic B1.

2. The HTTP requests from the push service might not be reaching the SAP S/4HANA or SAP ERP system. To solve that, perform the following:

    • Access the AS Instances of SAP System (SM51) transaction.
    • Select all application server instances shown in the list.
    • Increase the trace level to 3:
      • In the top menu, select Goto > Trace > Instance Trace > Change Trace.
      • Enter trace level "3" and save your entries.
        Note: Do not keep trace level to 2 and 3 for more than 1 minutes in a system. Keeping it open for very long can cause performance issue.
    • Clear the traces (it will make easier to analyze the logs).
      • In the top menu, select Goto > Trace > Instance Trace > Reset Trace.
    • Submit a document in order to make the Push service send an HTTP request to SAP S/4HANA or SAP ERP system. You can do it by following one of the options:
      1. Submit a document in the eDocument Cockpit (edoc_cockpit) transaction.
      2. Reprocess a document from the Resolve Errors - Documents and Statutory Reports app in DRC Cloud Edition.
    • Access the AS Instances of SAP System (SM51) transaction.
      • Select one application server instance.
      • In the top menu, access Goto > Monitors > ICM Monitor.
        • Now, you are in the ICM monitor for the selected Application Server Instance.
          • In the top menu, access Goto > Trace File > Display All.
          • Search for 'gco_internal'.
            1. If there are no entries on the log, the push service requests are not reaching SAP S/4HANA or SAP ERP system. Some issues exist in the BTP or Cloud Connector configuration. 
            2. If there are only entries with the value '/gco_internal/v1'
              1. There may be some missing configuration, as described in 1B.
              2. Another possibility is a short dump in the system. See details in the ABAP Runtime Errors (ST22) transaction for the BTP destination user. 
            3. If there are entries with the value '/gco_internal/v1' and '/gco_internal/v1/processInstance/status': 
              1. Check if country-specific Edocuments notes are implemented.
              2. Additionally, you can debug the system to check the error. Check KBA 3733393 - How to debug an HTTP request coming from the DRC Cloud Edition push service in SAP S/4HANA or SAP ERP system?.
    • Decrease the trace level to 1:
      • Access the AS Instances of SAP System (SM51) transaction.
      • Select all apllication server instances shown in the list.
      • Decrease the trace level to 1:
        • In the top menu, select Goto > Trace > Instance Trace > Change Trace.
        • Enter trace level "1" and save your entries.

3. Check the authorization objects for the technical user set in the SAP BTP destination, as follows:

    • SAP S/4HANA: Check user authorization described in the Setting Up Connectivity.
    • SAP ERP: Check user authorization described in the Setting Up Connectivity.
    • (Valid for SAP S/4HANA 2020 or higher version) Check the Background Remote Function Call (bgRFC) configuration, as follows:
      1. Make sure you are not missing the Creation of Inbound Destination.
      2. Inbound GCO_ECOM bgRFC Units are not processed in the bgRFC Monitor (SBGRFCMON) transaction, as follows:
        1. User authorization S_RFC described in the Setting Up Connectivity.
        2. Verify the Supervisor Dest.:
          • Go to SBGRFCCONF transaction.
          • Verify in the tab "Define Supervisor Dest." the <destination name>.
          • Go to SM59 transaction and find the <destination name>.
          • In the Logon & Security tab, confirm if the User in this tab exists, and verify:
            • Confirm with menu Utilities if the tests is working:
              • Connection test.
              • Authorization test.

Symptom

When creating a new territory with a similar name to a previous territory the following error appears:

"Failed to save bean class "class com.callidus.quotaterritory.beans.Territory" of lifetimeId"

Environment

  • SAP Sales Performance Management
  • Territory & Quota on HANA
  • Territory & Quota on Legacy Oracle
  • Territory & Quota on Hyperscalers

Cause

When creating a new territory with a similar name of a previous territory that was deleted (Reached its effective end date) you may face the error below:

"Failed to save bean class "class com.callidus.quotaterritory.beans.Territory" of lifetimeId"

This error is due to the new territory having an Effective Start Date equal to the Effective Start Date of the previous territory.

Resolution

Ex: 

A Roll up territory named Test1 has an Effective Start Date of 03/01/2026 and an Effective End Date of 03/10/2026

When creating a new territory, the Effective Start Date will be automatically pulled from the parent, which is 03/01/2026.

In order to fix this, the new territory needs to be created as a New Version, using the example above, we use the following steps below:

  1. Create a new Territory
  2. Fill out the Details such as Name, Territory Type, etc.
    1. In this example, the name will be Test1 and the Territory Type will be Rep Level Territory
  3. Click "Save As"
  4. Click "Save as New version"
  5. Set the Start Date to one day after the End Date of the previous Territory;
    1. With this example, the Effective Start date would be 03/11/2026
  6. Set End Date to your desired End Date. 
  7. Click Create 

Following these steps, a new Territory will be successfully created with a name equal to the previous deleted Territory. 

Should any errors appear during the creation of the New version, please contact SAP Support for further assistance, referencing this Knowledge Base Article.

Symptom

This Hub KBA centralizes information about the "Cannot load JDBC driver class" error and provides a checklist for troubleshooting the issue.

  • In non-Datasphere environments:
    • Remote source using the CamelJdbcAdapter does not work.
    • Connection test is unsuccessful.
  • Datasphere environments:
    • "Generic JDBC" connection using the Data Provisioning Agent does not work.
    • Connection validation is unsuccessful.
  • Error in framework.trc log file:
    • Cannot load JDBC driver class '<JDBC_driver_class>'
      • <JDBC_driver_class> is the value given in the "Driver Class" parameter of the remote source/connection configuration.

Environment

SAP HANA Smart Data Integration

Cause

The agent cannot load the JDBC driver class given in the remote source/connection configuration. There can be various causes for this error; the list of possible fixed is collected in the "Resolution" section below.

Resolution

Scenario 1: the JAR file provided in the <DPA_root>/camel/lib folder is incorrect or not in the correct location.

  • KBA 3023664 - Error: "SAP DBTech JDBC: [403]: internal error: Cannot get remote source objects:could not get tables" while testing remote source connection through CamelJDBCAdapter
  • KBA 3287123 - Error: "Cannot load JDBC driver class 'io.trino.jdbc.TrinoDriver'" when testing connection through CamelJDBCAdapter
    • NOTE: The Camel JDBC Adapter is compatible only with JDBC driver JAR files that are packaged as OSGi bundles.
  • KBA 3103913 - Error: "Cannot get remote source objects: Could not get tables" while testing remote source connection through CamelJdbcAdapter - SAP HANA Smart Data Integration
    • NOTE: Check the Product Availability Matrix for the correct driver version.

Scenario 2: the driver class given in the remote source / connection configuration is incorrect.

  • KBA 3347062 - How to identify the JDBC driver class for the driver of CamelJdbcAdapter

Scenario 3: DPAgent was changed to use CamelDataSource instead of BasicDataSource. A manual update of certain XML files and a refresh of the adapter configuration might be required.

  • SAP Note 3435601 - [CamelJdbcAdapter, CamelInformixAdapter] Cannot load JDBC driver class after upgraded to DPAgent 2.7.3.1
  • SAP Note 3423054 - [CamelJdbcAdapter] Cannot load JDBC driver class 'org.apache.hive.jdbc.HiveDriver'
  • SAP Note 3437951 - [CamelJdbcAdapter] Cannot load JDBC driver class after upgrade to DPAgent 2.7.3.1 or later
  • KBA 3536462 - Error: "Cannot load JDBC driver class <'DriverClass'>" for remote source created through CamelJdbcAdapter - SAP HANA Smart Data Integration

Scenario 4: DPAgent was restarted using the incorrect user.

  • KBA 3707317 - [CamelJdbcAdapter] "Cannot load JDBC driver class '<JDBC_driver_class>'" error after agent restart - SAP HANA Smart Data Integration

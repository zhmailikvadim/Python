Symptom

  • The Web transactions are no longer triggered/launched from the SAP GUI after Microsoft Edge WebView2 runtime client update to version 146.0.3856.62
  • The browser opens and immediately closes.
  • A blank Simple Browser Window opens.
  • WebView2 window closes due to certain actions in ABAP backend sessions

Environment

  • SAP GUI for Windows 
  • Edge (based on Chromium) selected as browser control in the SAP Logon options (default option)
  • ABAP PLATFORM - Application Server ABAP

Reproducing the Issue

  1. Log on your backend system using the SAP GUI for Windows with Edge browser control selected in the SAP Logon options.
  2. Call any Web transaction such as NWBC, SOAMANAGER, FLP, NWBC... from the SAP GUI OK Field.
  3. Edge does not open or a blank page is opened. Or in some cases the browser page opens then closes quickly.

Cause

Microsoft Edge update to version 146.0.3856.62.

Resolution

#####Issue is under investigation - KBA will be updated daily######

Workarounds available:

1. From Microsoft:

Revert to a previous Microsoft Webview2 fixed release (E.g: 145.0.3800.97 or prior version) https://developer.microsoft.com/en-us/microsoft-edge/webview2/?form=MA13LH

Loaded Microsoft EdgeWebView2 files are located in the following folder:  C:\Program Files (x86)\Microsoft\EdgeWebView\Application

NB: Please note that the only supported SAP GUI version is the SAP GUI 8.00. More info in SAP Note 147519 - Maintenance strategy / deadlines for SAP GUI for Windows / SAP GUI for Java.

2. From SAP:

Users can create favorites from Easy Access Menu screen after login into the system. Right click on favorites -> "add other objects" -> "web address or file" -> put the target URL there as shown below:

In the example below the entry refers to Fiori Launchpad. Thus, users will be able to access the URL directly from the Favorites instead of launching /n/UI2/FLP t-code. It works because this will call CL_GUI_FRONTEND_SERVICES=>EXECUTE to execute a URL. SAP GUI for Windows transfers the URL directly to the operating system and lets the operating system decide which helper application is suitable to open this URL, opening the configured default browser. The differences between methods of opening web content is explained in SAP Note 2913405 - SAP GUI for Windows: Dependencies to browsers / browser controls (Check section about Methods to display web content in or by SAP GUI for Windows.)

After a admin define all favorites URLs as above, they can download it via Download to PC/Upload from PC features as outlined in the SAP Note 456728 - "Easy Access: download/upload of favorites" and share with other users or they can transfer it to the role menu as outlined in SAP Note 1792527 - "PFCG: Transfer of Easy Access favorites to role menu".

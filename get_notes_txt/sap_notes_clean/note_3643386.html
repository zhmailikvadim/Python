Symptom

New builds and previously working builds for SAP Commerce Cloud with Datahub Starts to fail with:

"de.hybris.ccv2.datahub.task.DuplicateExtensionFileException: Provided more than one file for extension: extension_name"

Environment

SAP Commerce Cloud

Reproducing the Issue

Starting on August of 2025 try to build a Commerce Cloud with duplicated declaration of a Datahub extension on the Datahub manifest.

Cause

On August of 2025 a new builder version for cloud portal was released, starting on this update the Datahub build process was improved to avoiding building the same extension two times.

Resolution

  1. Download the build logs
  2. Look for the duplicated extension name provided on the error:

    "de.hybris.ccv2.datahub.task.DuplicateExtensionFileException: Provided more than one file for extension: extension_name"

  3. Search for duplicated declarations for the "extension_name" in your manifest, localextensions and Datahub manifest.
  4. Delete any duplicated declarations for the extensions, ensuring an unique declaration.
  5. Retry the build.

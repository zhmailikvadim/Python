Symptom

In UK Crossboarding scenarios, the Starter Checklist in SuccessFactors Onboarding Compliance incorrectly displays the Employment Start Date as the employee’s original Hire Date rather than the Transfer Effective Date.

In the UK Starter Checklist compliance form, when “Do you have form P45?” is selected as No, the employee personal details are auto populated with the Employment Start Date derived from the Hire Date instead of the Transfer Effective Date.

Environment

SAP SuccessFactors Onboarding

Reproducing the Issue

  1. Login as the candidate.
  2. In the UK Starter Checklist compliance form, when “Do you have form P45?” is selected as No.
  3. The employee personal details are auto populated with the Employment Start Date derived from the Hire Date instead of the Transfer Effective Date. 

Cause

  • This is a Known Issue.
  • Static Mapping to Hire Date: The form pulls the date from employmentInfo.startDate, which always reflects the original hire date and does not adjust for crossboarding.
  • Identifying the Process-Type is missing : Onboarding Compliance forms applies the same logic to all scenarios—hire, rehire, transfer, and internal move—resulting in incorrect start dates for rehire and cross boarding processes.
  • Identifying Job Information Dates: Relevant dates such as: positionEntryDate, jobEntryDate, eventEffectiveDate are not used. Lot more when to use which field has to be determined.
  • Mapping Gap from EC to MDF : The Starter Checklist relies on an MDF field for Start Date, but due to missing process logic, it always receives the Hire Date.

Resolution

  • The fix for this issue is planned to be deployed with the patch 2H 2026 [P3]. For patch timelines, please review SAP SuccessFactors Patches Knowledge Base
  • Product Engineering is investigating a solution.
  • Click on star to bookmark this article to receive updates about this issue. For more information about updates from subscribed KBAs, you can refer to the KB Article 2171560 - How to be notified of new or updated SAP Notes or KBAs.

Workaround

  • The only available workaround is to use the override functionality to adjust the standard mapping. Customer can temporarily map an alternative EC field to override the Start Date on the Starter Checklist. It must be manually maintained. 
  • The only available workaround is to use the override functionality to adjust the standard mapping. This is the approach we generally recommend to customers who need to modify predefined mappings or have specific requirements for reporting values in Compliance forms.

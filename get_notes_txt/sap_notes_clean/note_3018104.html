Symptom

  • How to check which inspection lot origin is assigned to a specific movement type?
  • How to check which inspection types are assigned to a specific inspection lot origin?

Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.

Environment

  • SAP ERP Quality Management (QM)
  • SAP R/3
  • SAP R/3 Enterprise
  • SAP ERP Central Component
  • SAP ERP
  • SAP Enhancement package for SAP ERP
  • SAP Enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA, on-premise

Cause

It would be necessary to know this information - for example - due to inspection lot creation.

Resolution

  1. Call transaction OMJJ.
  2. Select 'Movement type' and CONTINUE.
     

     
  3. Enter affected 'Movement type' (101 in the example).
     

     
  4. Double-click 'Update control / WM movement types' in the tree view.
     

     
  5. Select the relevant movement type combination and go to 'Details' (or scroll through each entry by selecting them all).
     

     
  6. The inspection lot origin is visible in the screen area 'Internal control (not changeable)'.
     

     
    It is also possible that an inspection lot is not foreseen for a movement type ('Inspect.lot origin is empty):
     

     
  7. Call transaction QCC0 or SPRO.
  8. Navigate to path:
    IMG
    >Quality Management
    >>Quality Inspection
    >>>Inspection Lot Creation
    >>>>Maintain Inspection Lot Origins and Assign Inspection Types
  9. Select relevant inspection lot origin and double-click 'Inspection types for the origin'.
     

     
  10. Existing inspection types are listed here for the lot origin.
     

     
    IMPORTANT:
    An inspection type with variant '01' must exist for each inspection lot origin. The system chooses this inspection type, if no control specifies otherwise. System can even trigger a DUMP, if variant '01' is missing, see KBA 2463649 - DUMP ASSERTION_FAILED in QA07 transaction (RQAAAS10).

Symptom

In some scenarios, customers may prefer to enter asset-related invoices through MIRO instead of using transaction F‑90. This article explains how to activate and use MIRO to post invoices directly to an asset, including entering the KSeF (Official Document Number) required for Poland.

Environment

  • SAP S/4HANA
  • SAP S/4HANA Private Cloud Edition

Resolution

1. Enable Asset Posting via MIRO

Make sure that the SAP system is configured to allow asset postings through MIRO.

This functionality can be activated by following SAP’s guidance: SAP Help Portal – Material Accounts & Assets

 

2. Post the Asset Invoice in MIRO Step-by-step

Step 1 – Open the Invoice in MIRO

Go to MIRO – Enter Incoming Invoice.

 

Step 2 – Enter Basic Invoice Details

Fill in:

  • Invoice date
  • Posting date
  • Vendor
  • Total amount

 

Step 3 – Assign the Asset

Instead of referencing a Purchase Order:

  • Select Asset as the account assignment
  • Leave PO number and purchasing fields blank

 

Step 4 – Enter KSeF / ODN Number

For Polish invoices:

  1. Navigate to Additional Fiscal Data.
  2. Enter:
    • Official Document Number Type (e.g., KSeF)
    • Official Document Number Value
    • Legal timestamp if required

This ensures local compliance and proper document tracking.

3. Backend Storage of the KSeF Number

The KSeF/ODN number is automatically stored in the backend in the ODN tables used by SAP.

Example backend entry includes:

  • Document number
  • ODN type (e.g., ZZPLMA)
  • ODN value
  • Timestamp

This confirms that the invoice is recorded with its official document number for audit and reporting.

Summary

Using MIRO as an alternative to F‑90 is fully supported when:

  • Asset posting via MIRO is enabled
  • The asset is entered directly in the invoice
  • The KSeF/ODN number is added through Additional Fiscal Data

This method simplifies invoice handling while ensuring compliance and proper backend traceability.

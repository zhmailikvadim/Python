Symptom

I run a payment terms data import task, and the import fails because required fields are missing in the CSV file.

Resolution

  1. Sign in to Ariba Administrator.

  2. Navigate to Manage Core Administration > Data Import/Export.

  3. Download the template or exported file for Import Payment Terms Data (Consolidated File).

  4. Open the file and verify that all required columns are present and populated, such as:

    • UniqueName

    • Name

    • PayInDays

  5. If any required column is missing or blank, add the column header exactly as defined and enter valid values for each row.

  6. Save the file and rerun the Import Payment Terms Data (Consolidated File) task.

  7. Verify that the payment terms appear correctly in Payment Terms after the import completes.

Symptom

When using a region that uses a comma (,) as the decimal separator, the Parts Issue quantity, Item Issue, Receive and Transfer quantities are displayed with a period (.) as the decimal separator when a fractional quantity is pre-populated.  This leads to an error when attempting to save the transaction

Other Terms

  • SSAM 2410
  • 2505
  • 2511
  • Parts Issue
  • Inventory
  • Receiving
  • Transfer
  • ICMTANGOAMF10-43621

Reason and Prerequisites

Reason:

Calculated inital value was returned using toString() instead of using localization utilities

Prerequisites:

  •  BAS or Visual Studio Code with SAM MDK project
  • `patch` command (available on BAS and macOS by default) via terminal or other patching utility
  • Read the guidelines on how to apply the patch and read the appendix section if you receive any error

 

Solution

To apply the patch, download the attached SAPNote-3735771-XXXX.patch file for your version of SSAM and save it in a known location, preferably adjacent to your SAM.mdkproject.  Use the 2410 file for SSAM 2410 and the 25xx file for SSAM 2505 and 2511.

If you are on Windows, see SAP Note 3533879 for how to install a .patch file in Windows. If you are on a Mac, follow the steps below:

  1. Open a terminal in the root of your SAM metadata (i.e. SAM.mdkproject/metadata or SAM.mdkproject/metadata/base.definitions)
  2. Run the terminal command `patch --no-backup-if-mismatch -p4 < path/to/Note-Number.patch`. 
  3. Verify in the terminal window that the following files have been modified and listed as patched, as indicated by the console output:

patching file 'Rules/Inventory/InboundOrOutbound/ValidateInboundOrOutboundDelivery.js'

patching file 'Rules/Inventory/IssueOrReceipt/GetReceivedQuantityDefaultValue.js'

patching file 'Rules/Inventory/PurchaseOrder/MaterialDocItemQtyAndUOM.js'

patching file 'Rules/Inventory/Validation/ValidateIssueOrReceipt.js'

patching file 'Rules/Parts/PartLibrary.js'

4. Publish and deploy the modified application

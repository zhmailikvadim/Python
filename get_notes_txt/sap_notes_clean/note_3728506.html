Symptom

Trying to sort a dynamically calculated virtual element in the CDS view implemented via an ABAP virtual element class results in "Error message: CX_SADL_ELEMENT_OP_NOT_IMPLMTD".

Environment

SAP S/4HANA

Reproducing the Issue

  1. Implement a virtual element in a CDS view using the annotation `@ObjectModel.virtualElementCalculatedBy: 'ABAP:'`.
  2. Populate the virtual element dynamically at runtime using the interface `IF_SADL_EXIT_CALC_ELEMENT_READ`.
  3. Attempt to apply sorting (ascending/descending) on the virtual element field in a Fiori application.
  4. Observe that sorting does not work due to the field being dynamically calculated and not persisted in the database.

Cause

Sorting and filtering requests are pushed down to the database layer. Virtual elements calculated dynamically through ABAP logic are not represented in the database and therefore cannot support sorting.

Resolution

  • Sorting is not supported for virtual elements calculated dynamically through ABAP logic and not derived from a single database field.
  • If the calculation of the virtual element is based on other non-virtual fields and is monotonically increasing or decreasing relative to the source fields, sorting requests can be delegated to the calculation source fields.
  • If the calculation does not meet the monotonic condition, annotate the calculated element with `@ObjectModel.sort.enabled:false` to disable sorting.
  • Refer to the documentation on implementing sorting of virtual elements for additional guidance: Implementing Sorting of Virtual Elements

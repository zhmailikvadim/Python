Symptom

  • Error "Carrier/vendor doesn’t exist in order" during invoice posting.
  • Message No. /SCMTMS/SFIR340 has been raised during invoice posting.

Environment

SAP S/4HANA 2025 and Higher

Reproducing the Issue

  1. Start transaction MIRO.
  2. Enter the reference document for freight settlement and attempt to post the invoice.
  3. Observe error /SCMTMS/SFIR340

Cause

  • The BAdI implementation /SCMTMS/MRM_FC_START (Enhance MRM_FINAL_CHECK) is active in an embedded S/4HANA scenario.
  • This implementation is intended only for sidecar deployments and leads to the error check “carrier/vendor doesn’t exist in order”.

Resolution

  1. Check whether BAdI /SCMTMS/MRM_FC_START (Enhance MRM_FINAL_CHECK) is active.
  2. Deactivate the BAdI implementation in embedded S/4HANA TM scenarios (it is designed only for sidecar use).
  3. Retest MIRO by posting the carrier invoice with reference to the TM freight order.
  4. Confirm that the invoice posts without error /SCMTMS/SFIR340.

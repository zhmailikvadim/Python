Symptom

  • Error message 'Description "&1" already exists' when the same description is used for a different Likelihood definition

Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.

Environment

  • SAP S/4HANA

Reproducing the Issue

  1. Enter the customizing with SPRO and navigate to Environment, Health and Safety - Health and Safety Management - Risk Analysis - Risk Matrix - Specify Likelihood
  2. Attempt to create two different likelihood entries with the same description (column Description (XS)).
  3. Observe that the system prevents duplicate descriptions and displays the error message: CM_EHFND_COMN051 'Description "&1" already exists'

Cause

The system behavior is designed to ensure that the likelihood name is unique in the "Graphical Risk Matrix".

Resolution

  1. Use a more meaningful and clear description for each likelihood entry instead of duplicating descriptions.
  2. If a change in system behavior is required, refer to SAP Note https://me.sap.com/notes/11 and utilize the SAP customer influence session to submit an improvement request.

Symptom

  1. When trying to schedule reports to email destination using Office 365 SMTP server, the job fails with the below error:
    • server error. [CrystalEnterprise.Smtp]: [SSL negotiation has failed during setup of the SMTP server connection, please ensure your SSL configuration is correct on both the client and the server.] 
  2. No changes were made to the BI environment, the issue occurred unexpectedly after March 22, 2026.

Environment

  • SAP BusinessObjects Business Intelligence platform 4.x
  • All supported system

Reproducing the Issue

  1. Log on CMC/BI Launchpad as Administrator
  2. Try to schedule any report to Email destination
  3. The job fails with the below error:
    • server error. [CrystalEnterprise.Smtp]: [SSL negotiation has failed during setup of the SMTP server connection, please ensure your SSL configuration is correct on both the client and the server.] 

Cause

  • This is a known Microsoft issue: SMTP-based scheduling jobs have been failing since March 22, 2026. For more details, see: 
    • Trust DigiCert Global Root G2 Certificate Authority to Avoid Exchange Online Email Disruption | Mic…

Resolution

  1. Download the certificate from: https://cacerts.digicert.com/DigiCertGlobalRootG2.crt
  2. Double click the certificate to open it
  3. Select 'Details' tab, then click 'Copy to file'
  4. In the pop-up window, click 'Next'
  5. Select the export format as 'Base-64 encoded X.509 (.CER)'
  6. Specify the file name, then click 'Next'
  7. In the next window, click 'Finish' to export the certificate
  8. Rename the downloaded certificate to certificate.crt
  9. Copy the certificate.crt into both 32 and 64 bit directories on the BI Server(s):

    Windows:
    <INSTALLDIR>\SAP BusinessObjects Enterprise XI 4.0\win32_x86\
    <INSTALLDIR>\SAP BusinessObjects Enterprise XI 4.0\win64_x64\

    Linux:
    <INSTALLDIR>/sap_bobj/enterprise_xi40/linux_x64
    <INSTALLDIR>/sap_bobj/enterprise_xi40/linux_x86 
  10. Restart all Adaptive Job Servers

Please note: If on BI 4.3 SP3 or higher version, the certificate.crt must be converted to certificate.pse, see the steps in:

  • 2540416 - How to set up SMTP over SSL, BI 4.2 SP04+

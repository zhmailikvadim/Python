Symptom

Once a batch/serial managed item is used as a component of a production order, the item cannot be removed from the item master data, even if it is deleted from the production order.

Reproducing the issue

Scenario:

  1. Create a new item (item A) and set Manage Item by to Batches or Serial Numbers.
  2. Open an existing production order in which the Status field is Planned or Released.
  3. Add item A as one of the components and choose Update.
  4. Right-click the row of item A and choose Delete Row, then choose Update.
  5. Open item A, right-click and choose Remove. The system displays the following message: "Removal of an item cannot be reversed. Do you want to continue?". Choose OK.

Expected behavior:

Item A is deleted successfully.

No record is generated in the OITL table from step 2 and step 3.

Actual behavior:

The following error message is displayed: You cannot remove item; linked transactions exist.

Two records are generated in the OITL table from step 2 and step 3.

Cause

System limitation

Solution

This is a system limitation.

Workaround:

Set such items as Inactive (Item Master Data → General Tab).

Symptom

  • During processing of the TransportationOrderGenericRequest_In (TOGR_IN) inbound service, the indicator transportationStageListCompleteTransmissionIndicator="true" is not respected and the system builds its own stages.
  • The behavior occurs for pre-carriage freight orders and ocean bookings.
  • Consolidation settings do not change the outcome.
  • The inbound XML contains a single TransportationStage, but the created freight booking/order has multiple stages.

Environment

  • SAP S/4HANA 2021 and above releases

Reproducing the Issue

  1. Prepare a TransportationOrderGenericRequest_In payload with transportationStageListCompleteTransmissionIndicator set to "true" and exactly one transportation stage defined.
  2. Trigger the XML service through SPROXY.
  3. Send the request to create a freight booking/order.
  4. Open the created document and review the stages; observe that additional stages are generated instead of using only the provided single stage.

Cause

  • In the External Planning scenario, the field transportationStageListCompleteTransmissionIndicator is not evaluated because External Planning currently supports only one staged Freight Unit (FU).
  • Restrictions include: External planning with stage splits is not supported; only freight orders with the sequence type of stages Defined and Linear are supported.

Resolution

  1. Note that there is no standard solution for enforcing the provided stage list via TOGR_IN in the External Planning scenario due to current functional restrictions.
  2. If the business requirement is to use only the stages provided in the inbound XML, implement an enhancement via BAdI /SCMTMS/TOR_SE_TOR_GN_REQ after the Update Handler (OMUH) execution to adjust the generated modification list (MT_MOD) before the final MODIFY/save.
  3. Retest the inbound processing and verify that the resulting stages align with the payload.

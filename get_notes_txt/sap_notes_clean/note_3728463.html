Symptom

  • Tax Simplifier statement is not accessible from Successfactors.
  • 'Web Dynpro application does not exist error' is coming while opening the app.
  • 500 Internal error is coming while opening TSS application although from SE80 it is opening correctly while testing.
  • Error message comes "Operation not supported for incompatible data objects"

Environment

  • Human Capital Management (HCM)
  • SAP R/3
  • SAP ERP Central Component
  • SAP ERP
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • Payroll India

Reproducing the Issue

  1. Logon to SF portal.
  2. Try to open Tax Simplifier statement.

     

Cause

  • Business Functions related to TSS has not been activated on SFW5.
  • 40SCD feature value has not been maintained correctly.
  • PIN_SS_RGDIR structure is not holding correct field details.
  • Incorrect URL configuration on Payroll Unified Configuration for Tax Simplifier Statement.

Resolution

  1. Go to SFW5 t-code. Check the business function : HCM_LOC_CI_26 , HCM_LOC_CI_36 activated or not.
  2. Go to PE03 t-code.
  3. Pass the feature name as 40SCD.
  4. Maintain the value for Country grouping '40' as : IN00/HR_IN_SSITP_2010 (if you have activated your customized schema , then it might be : ZN00/HR_IN_SSITP_2010)
  5. Go to SE11 t-code.
  6. Add the field : BTN_DWNLD_PDF (type : char30) on the structure: PIN_SS_RGDIR.
  7. Go to Admin Center.
  8. Select 'Payroll Unified Configuration'. Select India on the left panel.
  9. Under Statements, choose Add Link. The Statements dialog box displays.
  10. Enter the link name in the Link Name field. Format will be : https://:<SAP Payroll System Name>/sap/bc/webdynpro/sap/hress_a_rep_in_ssitp?sap-client=<SAP Client Number>&sap-language=EN&COUNTRYGROUPING=40&sap-wd-configId=HRESS_AC_REP_IN_SSITP#
  11. Click on Save button.

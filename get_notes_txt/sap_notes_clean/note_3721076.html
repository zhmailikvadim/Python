Symptom

When submitting or displaying documents in eDocument Cockpit app, the error 'SMP_CALL_FAILED/The Receiver with Peppol participant ID XXXX is not registered to the Peppol network.' is triggered in Application Log.

Environment

SAP S/4HANA Cloud Public Edition.

Reproducing the Issue

  1. Access eDocument Cockpit app
  2. Select related document
  3. Go to Display > option to display PDF is not available

Cause

The error reported indicates a problem with sending electronic documents to a participant in the Peppol network, specifically related to the Participant ID mentioned in message. The error code 'SMP_CALL_FAILED' suggests a failure in the Service Metadata Publisher (SMP) call, which is a crucial part of the Peppol document exchange process. The error text specifies that the document identifier is not supported by the receiver with this Peppol participant ID. 

Service Metadata Locator (SML): The SML is a central component in the Peppol eDelivery Network that helps locate the Service Metadata Publisher (SMP) for a given participant. It acts as a directory to find where the metadata for a participant is published. In productive environments, participants must be registered in the SML to be able to receive documents.

Service Metadata Publisher (SMK): The SMK is used in test environments and serves as a decentralized registry for technical metadata. It allows participants to test their setups before going live. For test tenants, the receiver should be registered in the SMK.

Resolution

These validations address the root cause of the SMP_CALL_FAILED error related to unsupported document identifiers.

Verify Participant Registration
Confirm that participant ID is the intended receiver.

  • In productive environments, validate registration in the Service Metadata Locator (SML).
  • In test environments, validate registration in the Service Metadata Publisher (SMK).

The participant’s receiving capabilities and supported document types can be verified using the Peppol Public Lookup tool Helger.

Confirm Environment
Ensure the tenant type is correctly identified.

  • Test tenants require registration in SMK.
  • Productive tenants require registration in SML.

Validate Document Type Support
Check whether the document type and document identifier being sent are supported by the receiver. The error indicates that the document identifier is not enabled for the participant. Confirm the supported document types via the Helger Webservice to see the document types accepted by the participant.

Review Receiver and Master Data
If registration and document types appear correct, verify the receiver’s Peppol configuration.
In test scenarios (for example, Peppol Practical – Participant Information), ensure the vendor master data enterprise number matches the one maintained for the test participant (for example, Crossroads Bank for Enterprises number).

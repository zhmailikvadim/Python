Symptom

In app "Monitor Product Allocation Order Items" (reachable via corresponding cards of app"Product Allocation Overview") in some cases previous order items (which have an latest delivery date equal to the first date in filter field "Delivery Date Range") are displayed, if the time zone of the allocation object is not equal to the time zone of the user in Fiori Launchpad.

Other Terms

Product Allocation, advanced Available-To-Promise, aATP, F3474

Solution

Implement the attached correction instruction.

After implementation of the note the new app coding will be operative after batch job /UI5/APP_INDEX_CALCULATE is executed.

If you cannot wait until execution of this batch job, you may start the corresponding program /UI5/APP_INDEX_CALCULATE manually, enter MONPALORDITMS1 in tab strip "Single SAPUI5 Repository Only" and press execute button.

Please clean up your browser cache.

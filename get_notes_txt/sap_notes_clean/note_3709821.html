Symptom

  • Scheduled program start and end dates cannot be updated to future dates. Date reverts back to the current date upon saving changes.
  • The issue occurs specifically in Japanese locale with certain date patterns where Japanese characters for months are used (e.g., 1月 for January).

Environment

SAP SuccessFactors Learning

Reproducing the Issue

  1. Login to SAP SuccessFactors Learning Administration with locale set to Japanese.
  2. Open a scheduled program or create a new one.
  3. Edit the agenda and select future dates for the start and end dates.
  4. Save the changes.
  5. Observe that the selected dates revert back to the current date.

Resolution

LMS engineering team has finalized the fix which is scheduled to be delivered in 1H2026 release.

Additional details are available in the following links: Product Release & Road Map Information 

Workaround:

  • Switch to a different locale or change Japanese locale date pattern that does not use Japanese characters for months.
    • Date patterns that are confirmed to work, such as:
      • DD/MM/YYYY
      • M/D/YYYY
    • Avoid using date patterns that are known to cause issues, such as:
      • yyyy/MMM/d
      • MMM-dd-yyyy
      • dd/MM/yyyy

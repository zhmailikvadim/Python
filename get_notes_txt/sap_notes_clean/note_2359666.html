Symptom

Transaction KO8G is executed to perform a settlement, for example, to a production order, but the system displays the error message KD233. However, the same process works fine in KO88.

 

Environment

  • Financial Accounting (FI)
  • Controlling (CO)
  • SAP R/3
  • SAP R/3 Enterprise 4.7
  • SAP ERP Central Component
  • SAP ERP
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA

Reproducing the Issue

<Execute> the settlement proccess via KO8G.

Cause

The reason is that Transaction KO8G does not allow processing of orders category 4 and 10. This is because KO8G is a transaction made for massive processing of internal orders.

For example, is informed a order range from 1000 to 2000, KO8G will ONLY select orders within that range that are indeed internal orders and not the production orders in that range. 
In case there is no internal orders in the informed range the system should trigger the Error Message KD233.

Single processing in KO88 is possible for Production Orders, because the system will assume that the user actually wants to process the Production Order that he entered.

Resolution

Since the transaction CO88 is made for Production Orders. If more restricted selection criteria are required (e.g., at the level of single order numbers), the program RKO7CO88 is recommended for use in the background via Transaction SE38.
For RKO7CO88 a selection variant can be saved, and it behaves similar to transaction CO88, but with more restricted selection.

Symptom

  • The Product characteristic in CO-PA is blank during document posting.
  • KEDR derivation rules fail because they depend on the Product characteristic.

Environment

•Controlling (CO) 
•SAP ERP Central Component
•SAP ERP
•SAP enhancement package for SAP ERP
•SAP enhancement package for SAP ERP, version for SAP HANA
•SAP S/4HANA Finance
•SAP S/4HANA
•SAP S/4HANA Cloud Private Edition

Reproducing the Issue

  1. In Transportation Management, open a Freight Order.
  2. Create and post a Freight Settlement Document (supplier invoice).
  3. In the GUI, open the supplier invoice posting in Financials and identify the generated accounting document.
  4. Run transaction KE24 for the relevant operating concern and controlling area.
  5. Use the accounting document as reference and execute the report.
  6. Observe that the Product field in the profitability segment is not derived.

Cause

  • For freight expense postings created from settlement documents, the material number is not populated on the FI line (for G/L account postings without stock relevance). As a result, the Product characteristic in CO-PA remains blank. This is standard behavior as documented for settlement documents.

Resolution

  1. Review the standard behavior: refer to 3152786 - Material number missing in accounting document of settlement document.
  2. If Product must be derived in CO-PA for these postings, implement enhancement logic to supply the material:
    1. Implement BAdI WLF_ACC_ENHANCEMENT to populate ACCIT-MATNR (for example, derive from the related delivery or source document). 2. Test by reposting a Freight Settlement Document and verify the CO-PA line in KE24.3 .If CO-PA determination needs to be explicitly enabled for the Agency Business Document, implement BAdI WLF_DOCUMENT_EXT and set CHECK_COPA_RELEVANT (parameter C_RE_DETERMINATION_RELEVANT = X).4. Re-test end-to-end posting and confirm the Product characteristic is derived in CO-PA and KEDR rules execute as expected.

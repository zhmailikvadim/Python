Symptom

Activation error "The partner function &1 is not available in procedure" occurs when performing a PPL rebase or merge.

Environment

SAP S/4HANA Cloud Public Edition

Reproducing the Issue

  1. Open the configuration activity "Set Up Partner Determination for Sales Document Header (ID: 101288)";
  2. Maintain the source function in the "Partner Functions in Procedure" step;
  3. Rebase the changes in the PL system or merge them into the mainline system.

Cause

The validation of the source function verifies the selected procedure in the "Partner Determination Procedures" step. Typically, this process works correctly during the fine-tuning. However, in a PPL activation scenario, the selected procedure may appear empty because no changes are detected in this step during the activation. As a result, the "Partner Determination Procedures" step is excluded from the activation item.

Resolution

This issue will be fixed by CE2608. Before the permanent fix, use the workaround following the below steps:

  1. Manually mark the activation item as completed.
  2. In the target system, manually maintain the partner function &1 with the source function &2 in the configuration activity "Set Up Partner Determination for Sales Document Header (ID: 101288)".

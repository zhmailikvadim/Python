Symptom

Payment order remains in status "export completed - pending confirmation(228)" or "export completed(230)" and can't be finalized because some payment items are still in status "conversion completed(39)".

Other Terms

/PF1/FIN_OUT_ORD - Finalize Pending Outgoing Orders

Reason and Prerequisites

During the export process for large payment orders, the status transition of payment items is executed in parallel.

If the payment file is sent and the pain.002 (status notification) is received before the internal process finishes updating all payment items, the technical status of the payment items may be overwritten back to "conversion completed(39)".

This occurs because the main export process keeps a lock at the object level. During the pain.002 import, this lock is checked only at the end of the process, after the external status has already been updated in the payment items. This can lead to an overlap, where two processes update the same payment items at the same time.

Solution

Code correction to check object list lock at the earliest during processing of status notification file to prevent concurrent updates.

Please implement SAP Note or assigned Support Package.

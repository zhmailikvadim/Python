Symptom

When an SOW revision is approved in SAP Fieldglass and pushed to SAP Ariba, it is intermittently rejected with various custom validation errors. These errors often relate to accounting fields such as Cost Center, Account, Project, or TaskID.

Error:
Cannot update and submit Requisition PRXXXX. Exception while submitting: (Please select Cost Center XXXX and Account YYYY since you’ve chosen to Capitalize.)

Upon investigation, you confirm that the values in the Fieldglass SOW and the resulting integration payload are actually correct and should pass validation.

Cause

This issue typically represents a race condition or synchronization lag. In some instances, the Ariba system triggers custom validation logic (AML) before it has fully "read" or updated all custom field values provided in the incoming payload. Because the validation occurs on the "old" state of the requisition before the "new" payload data is applied, it throws a false positive error.

Resolution

While SAP Engineering is working on a permanent solution to ensure data is fully committed before validation (referencing internal task TM-18003), you can resolve these specific failures by resubmitting the document.

  1. Verify Data: Open the SOW in SAP Fieldglass and confirm all Cost Allocation and Custom Field values are correct.
  2. Trigger Repush: In most cases, simply resubmitting the SOW revision from Fieldglass will resolve the error.
  3. Manual Resubmission: If the requisition remains in a "Composing" status in Ariba with the error, have the Ariba user or requester resubmit the existing PR without changes.
  4. Confirm Success: On the second attempt, the Ariba system usually processes the identical payload successfully, moving the PR to "Ordered" status.

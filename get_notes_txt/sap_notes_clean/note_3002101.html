Symptom

  • Views are loaded successfully but don't go to replication phase automatically
  • Load Statistics shows all portions of the view are loaded, but it still shows in Initial Load in Process part
  • View is loaded successfully but it's not listed in Load Statistics
  • Data Transfer monitor shows the view is in initial load phase
  • Data Transfer monitor doesn't list the view
  • Master controller job keeps canceling due to a dump:

    Category ABAP programming error
    Runtime Errors UNCAUGHT_EXCEPTION
    Except. CX_DMC_RFC_CONNECTION
    ABAP Program CL_IUUC_DB_CONN_RCV===========CP
    Application Component CA-EUR
    Date and Time 18.06.2020 04:24:03

    Short Text
    An exception has occurred that was not caught.

    What happened?
    Exception 'CX_DMC_RFC_CONNECTION' was raised, but it was not caught anywhere

Environment

SAP Landscape Transformation replication server

Cause

The Replication Master Controller Jobs try to read the DDIC Information of the view ZMKDBW_IMPMKTPL. Therefore the RFC Function Module "DD_VIEW_GET" is used, but this Function Call ends with an SYSTEM_ERROR, because the UCON-Framework rejects the execution. Because Function Module DD_VIEW_GET is missing in the Logging Phase of the UCON activation.

Resolution

Please register the Function Module "DD_VIEW_GET" in the UCON Framework in SLT. You can find the UCON documentation here: https://help.sap.com/viewer/1ca554ffe75a4d44a7bb882b5454236f/7.5.6/en-US/ab35e1c69f744d69a4fcf4ca93284e0c.html

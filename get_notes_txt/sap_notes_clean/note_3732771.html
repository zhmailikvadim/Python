Symptom

  • Sales order is blocked in GTS when the system is in PT language, but released when the system language is English.
  • There is an error message indicating that ''SAP GTS: document is temporarily blocked for Trade Compliance checks" (Message No. ZSD_GTS006).''

Environment

  • Global Trade Services (GTS)
  • Global Trade Services / Logistics Services (SLL)
  • SAP Global Trade Services
  • GTS Edition for Hana.

Reproducing the Issue

  1. Log on with the user interface language set to PT (or another non-EN language).
  2. In the source ERP, start VL01N and create a delivery from an existing sales order.
  3. Observe that the process is blocked in GTS and the message "SAP GTS: document is temporarily blocked for Trade Compliance checks" (Message No. ZSD_GTS006) is shown.

Cause

The GTS system does not have the user logon language (for example, PT) installed. When the logon language is not EN or DE and is not installed in GTS, the document transfer to GTS fails, resulting in a temporary block for trade compliance checks.

Resolution

  1. Install the required logon language (for example, PT) in the GTS system using transaction SMLT. For more details, please refer to 3156438 - SMLT: Language installation - all steps - SAP for Me  and 330104 - Transportrequests used for language delivery // SRx - SAP for Me.
  2. After the language installation completes, repeat the delivery creation and confirm that the document transfers to GTS without being blocked.

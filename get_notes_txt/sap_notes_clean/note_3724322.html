Symptom

The [[VIEW_LINK]] tag in workflow action email notifications directs approvers and contributors to the workflow details page instead of the subject workflow user People Profile page.

Environment

SAP SuccessFactors Employee Central

Reproducing the Issue

  1. Approve a workflow.
  2. The initiator receives a Workflow Action Approval Notification email containing a link.
  3. The link directs to the approval page instead of the People Profile page

Cause

  •  The observed behavior is expected system functionality for initiators and contributors.
  •  When these users click the [[view_link]] tag in the Workflow Action Notification email, they are redirected to the Workflow Details page because they are active participants in the workflow process.

 

When “Redirect CC Users to Approval Page” = No on Workflow Configuration Page (Default setting)

  • CC users are redirected based on workflow type:
    • User‑based Workflows
      • CC users are redirected to the People Profile page of the subject user.
    • Non‑user MDF Workflows
      • Redirects to the Manage Data page for MDF objects (e.g., Departments, other MDF entities).
      • For Position‑related workflows, redirects to Manage Positions.
    • Foundation Object (FO) Workflows
      • Redirects to the specific Foundation Object page.

 

When “Redirect CC Users to Approval Page” = Yes

  • CC users are always redirected to the Workflow Details page, regardless of workflow type

 

 

Resolution

If this behavior is not ideal for your business needs. Please consider opening an enhancement request to have this feature introduced. The design team will consider all enhancement requests raised in the community and will consider this request as an option for a future release. Please see the attached KBA 2090228 - How to Submit Ideas for SAP SuccessFactors Products on how to do this.

Symptom

After posting the first step of a sales order related Stock in Transit (SIT) process you get only partial GI using transaction VLPODL for some delivery items. This inconsistency is related to customizing change of the schedule line category after first GI. VLPOD gives in such case the error message M7096: 'Goods movement not possible with mvmt type 687'.

Other Terms

Movement types, 687, 601T, VLMOVE 103, M7 096, TVEP, NU

Reason and Prerequisites

Customizing error not correctly handled.

Solution

Implement attached correction instruction of this note. Afterwards, new error VLMOVE103: 'Customizing changes in Stock in Transit processes are not allowed' will be given.

Symptom

The option "Do Not Replicate Archiving Activities to Target System" in the LTRC administration data is greyed out and cannot be selected, even in edit mode or after deactivating the connection.

Environment

SAP Landscape Transformation Replication Server (SLT)

Reproducing the Issue

  1. Open transaction LTRC and navigate to the configuration’s Administration Data.
  2. Locate the option "Do Not Replicate Archiving Activities to Target System".
  3. Attempt to enable the option in edit mode and observe it is greyed out.
  4. Deactivate the connection and attempt again; the option remains greyed out.

Cause

By design, the "Do Not Replicate Archiving Activities to Target System" setting can only be defined during configuration creation. Additional source-side settings are created at that time to ensure consistency, so changing this flag in an existing configuration is not supported.

Resolution

  1. Note that changing "Do Not Replicate Archiving Activities to Target System" for an existing SLT configuration is not supported.
  2. Create a new SLT configuration and select "Do Not Replicate Archiving Activities to Target System" during creation.
  3. Validate replication with the new configuration to ensure expected handling of archiving activities.
  4. If the parameter is not visible at creation time, refer to the related note in the See also section.

Symptom

Can the MTU be set below 9000 on Linux servers running SAP HANA Database?

Environment

SAP HANA, platform edition

Reproducing the Issue

Cause

  • Inconsistent MTU settings across network components can cause packet drops when some devices cannot handle the configured MTU, leading to communication errors or hanging threads.

Resolution

  1. Align MTU settings consistently across all relevant network components end-to-end (hosts, switches, routers, SD‑WAN) to avoid packet fragmentation or drops.
  2. If intermediate network components cannot reliably handle jumbo frames, reduce the server MTU below 9000 to match the network path (for example, align with the SD‑WAN MTU).
  3. For remote or inter–data center connections where compatibility is uncertain, consider using MTU = 1500 to be on the safe side.
  4. After changes, validate connectivity and application behavior by testing affected connections and monitoring for errors such as -10807/[89006]/“recv failed rc=104”.
  5. If communication errors or hangs persist, review network timeout settings and refer to the linked notes for symptoms like hanging threads or connection timeouts/refusals.

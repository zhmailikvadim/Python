Symptom

The technical job SAP_UCON_STATISTIC_RECORDS does not retrieve any data. Checking the job log you can see that the job tries to read data far in the past, this data is no longer present in the system.

Other Terms

SAP_UCON_STATISTIC_RECORDS, no data

Reason and Prerequisites

The technical job SAP_UCON_STATISTIC_RECORDS does not retrieve any data, because the job tries to read statistical data too far in the past.
This can happen if the technical job was started once and stopped, it has been disabled or failing for a long time.
The job tries to read the data from the timestamp of the last successful run up to now.

Solution

Implement the attached correction instruction to provide the possibility to reset the technical job.
To reset the job and read the actual data ignoring the data from the past.
You can use the UCON Cockpit and via the menu Operations -> Statistical Data -> Reset Technical Job to reset the technical job that it starts reading the actual data.
If the menu entry is missing you can start the report
- RS_UCON_RESET_TECH_JOB for releases SAP Netweaver 740, SAP Netweaver 740, SAP Netweaver 750 and SAP Netweaver 752.
- RS_UCON_RESET_BATCH_JOB for all other releases supporting the UCON framework.

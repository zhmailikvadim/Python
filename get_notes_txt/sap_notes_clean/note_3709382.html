Symptom

FSM Cloud Connector allows manual sending of master data for Functional Location from SAP ECC/S4 to FSM. One of the program options is "Send Subhierarchy", which, when enabled, is intended to transfer functional locations and equipment belonging to the structure of the selected functional location.

However, even when this option is enabled, the components belonging to the structure of the functional location being sent are not transferred as expected.

Other Terms

FSM, E4C, Connector, Equipment, Functional Location, /PACG/ECM_TRIG_TELO, Subhierarchy

Reason and Prerequisites

The issue was caused by inconsistent selection logic during subhierarchy processing

When processing, the logic collected child functional locations and subsequent selection continued only if the sub-functional locations were found.
In scenarios where the original functional location had only equipment assigned beneath deeper hierarchy levels, they were not considered in further processing.

Solution

Please, implement this correction instruction.

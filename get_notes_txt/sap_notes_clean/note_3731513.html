Symptom

Inconsistent behavior for parent-child picklists after enabling Enable the latest Editing and History UI for Employee Central HRIS entities on Full Profile in Configure Latest People Profile. 

Environment

SAP SuccessFactors Employee Profile

Reproducing the Issue

  1. Ensure the “Enable the latest Editing and History UI for Employee Central HRIS entities” is enabled in Configure Latest People Profile.
  2. Open an employee profile and go to All actions -> Change Job and Compensation info.
  3. Select Change Job with an effective date.
  4. Change the field example you see parent field (for example, Job Classification) and observe the child field (for example, Job level) is populated correctly.
  5. In the same employee profile, use the Job information card pencil edit.
  6. Change the same parent field is set and observe the child field is cleared (left blank).

Cause

In PPX (Latest People Profile edit), child fields are reset whenever the parent field changes, even if the previous child value remains valid. The “Change Job and Compensation info” action still uses PP3(Legacy), which does not reset the child if it remains valid. This difference leads to the observed discrepancy.

Resolution

This is the current expected design in Latest People Profile (PPX); a product enhancement has been requested without a committed timeline.

Workarounds:

1. Manually reselect the appropriate child field value after changing the parent in pencil edit.
2. Disable the “Enable the latest Editing and History UI for Employee Central HRIS entities” in Configure Latest People Profile to use the (PP3)legacy behavior for editing.

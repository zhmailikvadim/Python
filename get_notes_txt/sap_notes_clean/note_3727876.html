Symptom

  • In Crystal Reports for Enterprise 4.3, reports exported to "Word (RTF)" format have random spaces removed between words.
  • The issue occurs when exporting reports through Crystal Reports for Enterprise client tool.
  • Exporting to PDF works as expected.

Environment

  • Crystal Reports for Enterprise 4.X.
  • SAP BusinessObjects Business Intelligence platform 4.X.
  • Windows OS

Reproducing the Issue

  1. Run a report in Crystal Reports for Enterprise.
  2. Export the report to "Microsoft Word (RTF)" format.
  3. Observe that spaces are removed between words in the exported document.

Resolution

Add the font files (.ttf) that are being used in the report in the following path:

  • C:\Program Files (x86)\SAP BusinessObjects\Crystal Reports for Enterprise XI 4.0\jre\lib\fonts

If the fonts are the default ones, which are already installed in Windows, you can copy the font files from the following directory:

  • C:\Windows\fonts

To avoid future issues, you can copy all the fonts from the Windows directory and paste them in the Crystal one, that way all Windows default fonts will work as expected.

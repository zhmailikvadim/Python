Symptom

  • A custom card doesn't show the second level child object on "View All" page in the Latest People Profile
  • Unable to see the child object of a child object in the UI
  • How to manually add a object's third layer in a Parent > Child > Child association in ConfigUI page?

Environment

SAP SuccessFactors HCM Suite

Cause

By default, only the parent and first level child fields are configured for use out of the box. The second level composite child fields should be manually configured in the ConfigUI.

Resolution

  1. Go to Manage Configuration UI page and search for the object's default screen
  2. Click on Add Group
  3. In the new section, click on Edit Properties
  4. Set Is Collection to yes and select the second level object as Collection Resolver
  5. Enter the Title to define the section label
  6. When adding fields, set the "Value Type" as "Resolver"
  7. Then, set the required field under "Value Type Resolver" and define the "Label Override"
  8. Repeat the steps 6 and 7 until you have added all required fields

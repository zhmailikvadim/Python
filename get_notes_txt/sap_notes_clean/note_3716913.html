Symptom

  • Invoices from a specific punchout catalog supplier do not apply accounting splits as expected when posted via SAP Ariba Buying and Invoicing.
  • Accounting splits function correctly for other suppliers and invoices posted via SAP Business Network (SBN).
  • Parameter Application.Procure.CopySplitAccountingsToChildren is set to YES.

Cause

The parameter Application.Procure.RespectSupplierOrOpenICSAccounting is set to YES. This configuration ensures that accounting details provided by the supplier in the cXML are retained. If the supplier does not explicitly define accounting splits in the cXML, the system carries over the purchase order splits to the invoice.

Resolution

  1. Verify the parameter  Application.Procure.RespectSupplierOrOpenICSAccounting is set to YES.
  2. Ensure whether the supplier includes explicit accounting details for splits in the cXML file when submitting invoices.
  3. If the supplier does not provide accounting splits, the system will automatically apply the purchase order splits to the invoice.
  4. Refer to the Retain standard accounting info on invoices from Ariba Network | SAP Help Portal  for further configuration details regarding retaining standard accounting information on invoices from the Ariba Network.

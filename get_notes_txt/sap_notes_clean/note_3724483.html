Symptom

  • Deep Link configured in Configure Custom Navigation for the custom MDF card of a specific user is redirecting to the profile (PPX) of the logged in user. 
  • Deep Link configured in Configure Custom Navigation for a custom MDF card not working when the Latest People Profile is enabled. 
  • The Deep Link will work without issue when the Menu Location in Configure Custom Navigation is set to "Action Search" but will not work when set to "Top Navigation". 

Environment

SAP SuccessFactors HCM 

Reproducing the Issue

  1. Configure a Deep Link for a specific user's MDF card for the Latest People Profile (PPX) per Deep Links to Full Profile Categories and Cards
  2. Go to Configure Custom Navigation.
  3. Add a new Custom Navigation, using the Deep Link as the URL, and set the Menu Location to "Top Navigation".
  4. When you try to access from Top Navigation it redirects you to the profile of the logged in user rather the MDF card of the user the Deep Link was configured for.
  5. Go back to Configure Custom Navigation and add a Menu Location for the same item e.g. Action Search. 
  6. Search for the "Menu Item Label" in Action Search. 
  7. You are brought to the MDF card of the user the Deep Link was configured for.
  8. Similarly when you enter the Deep Link directly into the Address Bar while logged in it works without issue.

Cause

This issue has been confirmed in the latest Shell Bar, but only in preview mode. WEF-169181 has been created to address it. The fix will be available in release 2605, as the latest Shellbar is not currently in production.

Engineering have investigated the issue and identified that the issue is only present in preview instances where the Latest Shell Bar is being used. The Latest Shell Bar has not ye been deployed to production instances. 

Resolution

Fix to be deployed during 1H2026 release.

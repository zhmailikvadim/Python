Symptom

When implementing a note, there is no option to create a new TR to store the note, instead, the note is stored in an existing TR.

Environment

Snote

Reproducing the Issue

Implement a note via Snote

Cause

The user who implement the note is the same user of existing TR (modifiable status, not release yet).

Resolution

It is by design that if the user who is implementing note, is the same user of an existing modifiable TR, then the note will be stored in the existing modifiable TR.

Keywords

note, existing TR

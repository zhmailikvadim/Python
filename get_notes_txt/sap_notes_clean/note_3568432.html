Symptom

You are executing test plans or post-upgrade test plans, and notice that the TDC data variant is not picked correctly in test plan execution.

Alternatively, you can see that the TDC data variant is not working after the upgrade.

Environment

SAP S/4HANA Cloud Public Edition

Cause

Old test plans will not be automatically updated after the system upgrade.

Resolution

If a test data container data variant is created and the Test Process is later modified, the test data container previously assigned to the test plan will not be updated.

The changes will only be reflected once a new test plan data variant is created. This can be done by following the steps below:

  1. Open the "Test Your Processes - Manage Test Plans" app.
  2. Locate the test plan and open it.
  3. Navigate to the variants section.
  4. Click on the + button to create a new test plan data variant.

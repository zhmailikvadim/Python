Symptom

Error saying that "The Leave Must not Start before XX/XX/XXXX date"

Environment

  • UIHR002
  • GBX01HR5
  • SAP enhancement package 7 for SAP ERP 6.0
  • SAP enhancement package 8 for SAP ERP 6.0
  • S4/HANA all releases 

Reproducing the Issue

When you are trying to apply leave the system it is getting the error Also happen in the portal but not in PA30.

Cause

This is because the customizing table V_T554S_WEB  customization of the "Processing in the Past"

Here you Specifies the periods in which employees are permitted to create, change, or delete an absence of this type retroactively, that is, when the start date of the absence is in the past.


  1. Note
    You specify which actions an employee is permitted to carry out in the System Response area.

    The past is defined as the day before the system date. You can enter information for the current day in the Start Date Is the Current Day area.

Use

You have the following options for specifying periods:

  • Processing Permitted to Recalculation Limit
    Employees can process absences of this type, provided that the start date is not before the earliest retroactive accounting date for payroll that was determined dynamically by the system. This limit is set so that the absences can be stored in the system without problems.
  • Processing Not Permitted
    Employees are not permitted to process any absences of this type if the start date is in the past.
  • Retroactive Processing Permitted in Period
    You can use this function to restrict the period in which employees can process absences retroactively. This means that you can prevent employees from triggering unwanted retroactive payroll runs through processing their absences.

Example

A employee wants to change an absence retroactively on Tuesday, March 15, 2005. The system calculates the retroactive periods as follows:

 

Period Start date of leave no earlier than
 
1 day Monday, March 14, 2005
1 week Tuesday, March 8, 2005
1 month Tuesday, February 15, 2005
1 year Monday, March 15, 2004

 

Resolution

Check the retro of the table v_t554s_web for the type leave that you are using and confirm this is updated with the correct option that you need

Symptom

You are a customer of the new Group Reporting logic and want to import consolidation units.

You obtain the following error:

"Fiscal year variant of company codes in accounting and version XXX differ"

Other Terms

T001, FINSC_LEDGER, Company Code, Consolidation Unit, PAPM, TF00

Reason and Prerequisites

One of this consolidation unit, say XYZ is assigned to a company code XYZ. This company code is assigned to company XYZ. For this company a ledger customizing has been done in FINSC_LEDGER. Beside the company code XYZ there are additional company codes assigned to company XYZ. For these company codes as that are not used for accounting no ledger configuration is present. The import check searches for these company codes in the ledger configuration but cannot find them and thus a fiscal year variant.

There is no fallback to T001.

Solution

Implement the correction instruction of this SAP Note or install the corresponding support package.

Note the correction adapts:

  • Having a set of company codes with the identical company assigned (scenario described above)
  • Having a parallel ledger (non-leading) or an extension ledger with a subset of the company codes assigned
  • Checks on local currency, group currency, functional currency, and fiscal year variant are only performed based on the assigned company codes to the ledger and no longer on the superset of all company codes with the same assigned company

Note:

This correction introduces a translation relevant text. As texts are only transslated in a support package it is therefore only visible in english.

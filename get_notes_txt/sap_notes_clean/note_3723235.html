Symptom

When trying to approve a workflow for adding a new assignment for a contingent worker (added via 'Actions' > 'Add Assignment') the system raises an error message preventing the workflow approval:
"No permission to access the attached file xxxx. Attachment in field xxxx does not belong to the user xxxx."

Environment

SAP SuccessFactors Employee Central - 2511

Reproducing the Issue

  1. Go to the profile of a Contingent Worker
  2. Click the 'Actions' button
  3. Select 'Add Assignment'
  4. Fill the details for the new assignment and click submit
  5. Click 'Submit' on the workflow pop-up
  6. Proxy as the approver and try to approve the workflow for the new contingent employment
  7. Error message appears:
    "No permission to access the attached file xxxx. Attachment in field xxxx does not belong to the user xxxx."

Resolution

Our engineering team have confirmed that this error is occurring due to a validation for the attachment field which is skipped on hire workflows but in concurrent employment enabled instances, the validation is still being triggered for the 'Add Assignment' action.

A fix has been created and will be deployed as part of the 1H 2026 release.

Symptom

If there's an open quantity in order and a partial delivery was created, the system allows a second delivery to be created as long as the first delivery hasn't had goods issue posted yet.

Environment

  • Sales And Distribution (SD)
  • Logistics Execution (LE)
  • SAP ERP Central Component
  • SAP ERP
  • SAP Enhancement package for SAP ERP
  • SAP Enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA

Reproducing the Issue

  1. Create one sales order with 10 quantity and get the credit card authorization;
  2. Create first delivery for this sales order with partial quantity, for example 5;
  3. Try to create second delivery but not saved, the credit check is passed in such case;
  4. Do PGI for the first delivery;
  5. Try to create second delivery again, now the second delivery will be credit blocked due to  insufficient payment card authorization.

Cause

The deliveries do not contain the credit card data, but the system checks the authorizations in the sales order.

During the check in the delivery the system takes into consideration all existing billing documents and deliveries with processed PGI.

It means, that the deliveries with PGI status not "C" will not be taken into consideration, when the check is done from the delivery. 

Resolution

This is the correct system behavior. If you need to restrict the creation of such deliveries, you would need to modify the standard system logic.

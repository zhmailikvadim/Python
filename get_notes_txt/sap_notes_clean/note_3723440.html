  • Syntax error dumps occur replicating a table.

    What happened?

  • Error in LTRC:
    Creation of Migration object Z_<TABLE_NAME>_<MTID> failed
    Error when reading from ABAP Dictionary for table <TABLE_NAME> (RFC destination ****)
    DDIC activation failed for one of the structures listed below (Table <TABLE_NAME>):
    Log ****** could not be found

Environment

SAP Landscape Transformation Replication Server (SLT)

Cause

Running jobs on the wrong client or have refreshed the SLT system, which makes objects in table DMC_STRUC_ID have with the same IDENT value

Resolution

  1. Run the SLT jobs in the correct client:
    2879868    Administrating LTRC with users on different clients causing issues - SAP Landscape Transformation Replication Server
  2. Have all configurations de-activated
  3. Go to LTRC> Expert Functions-> 'Reset Load and Replication Status' and 'Reset Status of Tables and Synonyms' (leave table name empty to do all in config, and do this for each MT_ID)
  4. Expert Functions-> Delete Load /Replication Objects-> Leave table name empty to do all table in config (do this on all MT_IDs)
  5. Delete all entries in table DMC_STRUC_ID as these values are invalid due to duplicates IDENTS across different objects.  If an SQL session is being used, reset the table buffer with $SYNC
  6. Start the SLT configurations, 1 at a time.  SLT will create all objects when it detects that they are missing.
  7. When regeneration of objects is completed, new entries will be created in the DMC_STRUC_ID table, with all IDENT values being unique.

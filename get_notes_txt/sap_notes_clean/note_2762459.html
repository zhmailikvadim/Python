Symptom

The corrections in this note have been retracted. The functionality described is available from ABAP release 754 onwards.

The ATC check "Search problematic statements for result of SELECT/OPEN CURSOR without ORDER BY" (technical name CL_CI_TEST_NO_ORDER_BY) does not report findings for the following code pattern:

> SELECT * FROM table INTO wa.

> MOVE wa-component TO wa_2.

> APPEND wa_2.

> ENDSELECT.

> READ TABLE wa_2 BINARY SEARCH.

Other Terms

Code Inspector, S/4 HANA Custom Code Checks, ORDER BY

Reason and Prerequisites

Missing functionality

Solution

Upgrade your system to an ABAP release 7.54 or greater.

Symptom

In fiori launchpad, one observes that there can be tiles with same name but the semantic object is different.

Environment

SAP Fiori Launchpad

Reproducing the Issue

  1. Add all semantic objects of one application to the Fiori Launchpad.
  2. Observe that duplicate tiles with the same name appear on the launchpad.

Cause

Semantic objects of one application were added, resulting in duplicate tiles with identical names. This can happen for certain applications that have multiple semantic objects, e.g. "Create Maintenance Item" IP04, "Change Maintenance Item " IP05, "Change Maintenance Items - List View" IP17.

Resolution

You can simply delete the unwanted tiles, but if you want to preserve them, you can distinguish the tiles by adding a subtitle to the tile using the Launchpad Designer:

  1. Access the Launchpad Designer using transaction code '/n/ui2/flpcm_cust'.
  2. Select the tile to adjust and add a subtitle.

Please note that: Modifying the tile definition will break the tile reference only (Not the Target mapping).

Changing it will make that any future change that SAP deliver for the tile definition itself(like the tile title) will not be visible for you.

Breaking the tile reference in this case will not impact in any functionality related aspect.

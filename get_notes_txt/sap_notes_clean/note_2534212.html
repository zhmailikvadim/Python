Symptom

Shop Order Schedule Report does not display data when Shop Order Specific Routing used.

Other Terms

Shop Order with order specific Routing.

Reason and Prerequisites

Steps to reproduce:

1)  Send ProductionOrderUpdateRequest with <meint:erpOrderType>MAKE_TO_ORDER</meint:erpOrderType>.

2)  In ME, go to Shop Order Report - retrieve that shop order, and select Shop Order Schedule Report.

Actual Result: Message 'Your query returned no data'.

Expected Result: Shop  Order Schedule shows steps, scheduled start/complete times for each routing step.

The issue occures because scheduling information is not updated with correct routing type for shop order specific routings (type 'H').

Solution

The source code has been corrected to update Routing type for scheduling records if it detected as Order Specific.

Impact: Web service ERPProductionOrderService.

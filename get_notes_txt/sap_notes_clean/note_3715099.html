Symptom

  • The Server Intelligence Agent (SIA) is not created during installation.
  • Creating an SIA with the same name as previously used fails.
  • Creating an SIA with a different name succeeds, but login to the CMS is not possible.

Environment

  • SAP BusinessObjects Business Intelligence (BI) 4.x
  • All supported Windows OS.

Reproducing the Issue

  1. Uninstall BI/IPS manually using KBA 1691555 - Manually remove SAP BI 4.x from Windows Server [How-To] [Video]
  2. Keep the CMS DB content
  3. Install BI/IPS using the existing CMS DB
  4. Notice that SIA is not created after installation 
  5. Trying to create SIA with the same name fails
  6. Login to CMC after creating new SIA with new name fails

Cause

The installation was done on an existing CMS DB schema with data in it.

Resolution

  1. Uninstall the BI/IPS manually again as per the KBA 1691555 - Manually remove SAP BI 4.x from Windows Server [How-To] [Video]
  2. Clear the CMS DB making sure that the schema is blank
  3. Perform the installation as per the best practices on a blank/new CMS DB Schema following steps in KBA 1952120 - Best practices & pre-requisites on Windows while Install/Update/Patching BI

Symptom

  • User is unable to set the "maxrows" parameter for a QB Query API call to override the system.api.query.max.rows default value of 200 rows in Agent Performance Management.
  • User expects that the "maxrows" parameter should allow the return of more than the system.api.query.max.rows option set value, up to the system limitation of 10,000 rows.

Environment

  • SAP Sales Performance Management
  • Agent Performance Management

Reproducing the Issue

  1. Perform a GET request on a defined QBQuery.
  2. Example: /ICM/REST/query/run?qbqueryid=a-qbquery-id&maxrows=7500.

Cause

This is expected application behavior.

Resolution

    The option system.api.query.max.rows defines the number of max rows to be returned via API call, as it is the override - you cannot return more rows than the value that is set for system.api.query.max.rows.

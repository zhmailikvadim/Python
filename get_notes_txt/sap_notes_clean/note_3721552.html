Symptom

Unexpected runtime behavior of statement "LOOP AT itab INTO wa"

Other Terms

KERNELCOR,  Internal Table, LOOP, runtime monitor, SRTM

Reason and Prerequisites

This patch extends SAP Note 107547 (ITAB - runtime monitor check ITAB_FREED_IN_LOOP).

While SAP Note SAP Note 107547 introduced a runtime monitor check to detect when an internal table is overwritten in the body of a LOOP statement, this patch provides the profile parameter abap/itab_chk_on_exchg_in_loop to enable extended diagnostics or force a specific runtime error when such a situation is detected.

Solution

The correction is delivered with the kernel archives:

  • Hotfix file DISP+WORK
  • SP Stack Kernel  files SAPEXE.SAR and SAPEXEDB.SAR 

The correction is contained in all patch levels that are equal to or greater than the patch level specified in the "Support Package Patches" section for the relevant kernel release.
If there is no entry there, the patch has not yet been delivered. The patch level will be entered automatically once the patch is available for download. You can subscribe to this SAP Note to be notified when it is updated.

Follow our recommendations for applying a kernel patch:

  1. Import the latest SP stack kernel if it already contains the correction. For a list of current SP stack kernels, see SAP Note 2083594 (SAP Kernel Versions and SAP Kernel Patch Levels).
  2. Apply the hotfix only in the case of a serious error that is not yet corrected by the latest SP stack kernel. 
  3. Check the regression note for the desired patch level before applying the kernel patch. For detailed information, see SAP Note 1802333 (Finding information about regressions in the SAP kernel).
  4. For instructions on how to download and apply a kernel patch, see SAP Note 19466 (Downloading SAP kernel patches).

For detailed information about our recommendations, see the following strategy document: Update Strategy for the Kernel of the Application Server ABAP in On Premise Landscapes.

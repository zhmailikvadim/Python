Symptom

  • In the first step of the form in the Latest Experience of Performance Forms, the manager can see the employee's rating.
  • The manager has write access to the "Item Rating" but does not have access to other ratings (e.g., "Subject Rating") in this step.
  • The employee's rating cannot be hidden in the first step while the manager's rating is active, regardless of configuration changes.
  • This behavior does not occur in the Legacy Experience of Performance Forms, where the employee's ratings are not visible to the manager in the first step.

    **Image/data is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental** 

Environment

SAP SuccessFactors Performance & Goals

Reproducing the Issue

  1. Log in to the system.
  2. Proxy as the manager of the affected user.
  3. Launch a form from the performance inbox.
  4. In the first step of the form, observe that the manager can see the employee's rating in the Competency section.
  5. Navigate to "Manage Templates" in the configuration of the form.
  6. Check the field permissions for the respective sections.
  7. Switch the form to the Legacy view and verify that the employee's ratings are not visible to the manager in the first step of the route map.

Cause

  • The issue arises due to the permissions around “Ratings from Others” and the Gap Analysis functionality in the Latest Form Experience.
  • For the Goals, Competency and Skill section:
    A user can view the gap in goal, skill, and competency sections when either of the following conditions is met:

    • The user can view the gap in Gap Analysis and has permission for Ratings from Others.

    • The user can't view the gap in Gap Analysis, but EZ Rater is used and show-gap isn't set to false

Resolution

1) Go to General Settings.
2) Navigate to other's rating option.
3) "none" permission needs to be given for Rating from Others Rating option or Gap analysis button permission.

Symptom

  1. User is getting a warning in logs since build 2026.04 while performing TRUNCATE/VACUUM:

    Warning: An error occurred. Updating statistics for Local Table (File) has been stopped.

    View details: {"code":359, "message":"string is too long: \"<space>\".\"<LOCAL_TABLE>": line 133 col 2 (at pos 4424): Assigned value for parameter or variable \"STATISTIC_RESULTS\" exceeds maximum length ..."}

  2. Also after performing TRUNCATE , even though all records are deleted, the count of "Number of Active Records" is not 0 in local table monitor

Environment

Datasphere Wave 2026.04

Reproducing the Issue

  1. Navigate to Data Integration Monitor -> Local Table (file) monitor.
  2. Open any local table (file) and perform TRUNCATE/VACUUM operation.

Resolution

[Last Update on 8:00pm UTC, Feb 9]

  • Truncate or Vacuum operations are working fine, but the statistics cannot be collected due to a bug.
  • Development team is working on the fix. The team is planning to deliver the fix with Datasphere Wave 2026.05. Refer to SAP Note 2984828 - SAP Datasphere Release & Delivery Schedule.
  • Merge task can be run correcting the table size statistics in the monitor.

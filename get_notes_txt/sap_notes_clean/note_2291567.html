Symptom

In SAP Customer Checkout monitor, you find the receipt is Released with a Doc ID. However, you cannot find the document in SAP Business One.

Reproducing the issue

Scenario:

  1. Cashier creates a Receipt in SAP Customer Checkout.
  2. In the SAP Customer Checkout Monitor, you see that the Receipt has status Released and an ID is present in the column B1 Doc ID.
  3. User searches for the Receipt in SAP Business One using the B1 Doc ID, but the receipt is not found in SAP Business One.

Expected Behavior:

In the SAP Customer Checkout Monitor, the receipt is seen with status Inconsistent and an error message detailing what caused the receipt to be rejected by SAP Business One is displayed in SAP Customer Checkout Monitor.

Actual Behavior:

This behavior occurs only if the Receipt creation in SAP Business One was rejected in the B1i transaction commit phase. In the SAP Customer Checkout Monitor, the receipt is seen with status Released and B1 Doc ID is present.

Cause

If an exception is raised during a B1i transaction commit phase, the DI calls are rolled back, but the calls to SAP Customer Checkout Monitor Database are not rolled back. When an issue is found during a commit phase, the Error handler for the SAP Customer Checkout integration package is not called. Integration framework implements the 2PC (2-phase-commit) protocol and it allows individual error handling steps to be defined that are called in case of errors. In case of errors that occur in the second phase (the commit phase) however, those error handling steps are not called, but default error handling is applied.

Solution

Integration framework version 1.22.5 and higher include the correction.

Workaround for integration framework version 1.22.2, 1.22.3, and 1.22.4:

  1. Follow the steps mentioned in the SAP Note 2292409. This will update the B1if framework so that the scenario specific error handler will be called.
  2. Now you must upgrade Integration scenario package sap.CustomerCheckout and  sap.CustomerCheckoUI  to version 3.0.15 or higher.
    1. If you are running the Release 1.0 SP03 or Release 2.0 SP00 of SAP Customer Checkout, take the integration scenario package from the SAP Note 2163033.

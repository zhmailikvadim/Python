Symptom

  • Error response contains code KALC_VIA_MSGS/000 with message “Allocation cycle does not exist”.

Cause

The error KALC_VIA_MSGS/000 (“Allocation cycle does not exist”) typically occurs when the app does not find the generated VIA result objects / hierarchy headers for application FI-CO. In other words, the app does not simply read “existing cycles” from configuration tables—it expects the cycle to have produced the backend result objects (via the standard execution framework).

Based on this, the most likely possibilities are:

  1. Cycle execution path
    The cycle may exist and even be executed manually, but if it was not executed via the standard scheduled overhead accounting job template, the VIA result objects may not be generated, and the results app will not list the cycle.
  2. Execution context mismatch
    The job might have been executed, but with a different controlling area / fiscal year / period / version than the one used when launching the results app, causing the result objects not to be found.
  3. Backend result generation not persisted
    The job finishes, but the expected result artifacts are not created/updated (this is what we want to verify next in your system using live data).

Resolution

  • To be able to use the App F5854 successfully, the prerequisite is that the backend execution has generated the corresponding result context for the same controlling area / fiscal year / period combination. The detailed reason why the specific execution did not lead to a consumable result context would need to be checked on the backend execution/result-generation side rather than in the Fiori app itself.

  • These apps are designed to display generated allocation results; they do not themselves create or repair the underlying VIA result objects.

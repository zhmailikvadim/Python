Symptom

After updating the Kerberos keys and tickets using AES256 The error below occurs. 

-------- CHECK keytab ----------------------------------------------------------
Checking key version number and key in keytab /usr/sap/SID/etc/krb5_hdb.keytab for SPN hdb/server ...FAILED
    ERROR:   Key version numbers match, but keys in local keytab /usr/sap/SID/etc/krb5_hdb.keytab and AD are different.
      COMMAND: $DIR_EXECUTABLE/krb5/bin/kvno -k /usr/sap/SID/etc/krb5_hdb.keytab hdb/server@domain
      RETURN CODE: 1
      STDERR: "hdb/server@domain: kvno = 3, keytab entry invalid
kvno: Wrong principal in request while decrypting ticket for hdb/server@domain"
--------------------------------------------------------------------------------

Environment

  • SAP HANA Database 1.0
  • SAP HANA Database 2.0

Cause

The keytab configuration is incorrect.

Resolution

To solve this problem it's important to follow the steps below:

  • Execute the command:
    •  ktpass -out krb5_XXX.keytab -princ SAP/Kerberosxxx@server.domain -mapUser Svc_SAPKerbXXX@server.domain -crypto AES256-SHA1 -pType KRB5_NT_PRINCIPAL -pass "<password>" -mapOp set to generate the krb5_xxx.keytab.
    • The krb5_XXX.keytab generated will be use on both HANA and ABAP sides.
  • After running the command ktpass , all SPNs on the AD side should have been deleted, so it's necessary to add them all again as next::
    • setspn -S hdb/server.domain\Svc_SAPKerbxxx
    • setspn -S hdb/server.domain\Svc_SAPKerbxxx
    • setspn -S HTTP/serve. domain\Svc_SAPKerbxxx
    • ...
  • Once all SPNs are properly configured, place the keytab in /usr/sap/<SID>/home/etc/ as krb5_hdb.conf, aligned with the environment variable and the krb5_hdb.conf in the same directory, ensuring the HANA setup functions correctly.

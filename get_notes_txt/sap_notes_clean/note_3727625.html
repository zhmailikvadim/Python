Symptom

The symptoms of this might be broad. In general there will be issues with locked/enqueued workitems. Therefore you may recognize stall lock entries for table SWWWIHEAD. Further for example events (like the restart event in flexible workflow) might not be processed correctly. If the event trace (SWEL) is turned on, you will also see the failing event there.

Other Terms

Flexible workflow, events, restart, custom attributes updates, cl_swf_utl_run_usr_attr_chg, MC 601, Requested object E_WORKITEM is currently locked by user '', SM12

Reason and Prerequisites

You or the application uses cl_swf_utl_run_usr_attr_chg to update custom attributes. Afterwards you make directly or indirectly changes to the running workflow instance. Here, due to a program error, locks for work items are not handled properly.

Solution

Implement this SAP note or install the corresponding support package.

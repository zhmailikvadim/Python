Symptom

The issue occurs in SAP Business Client (NWBC), where the application crashes when rendering a tab containing a custom PDI mashup with PDF content. It typically happens after opening multiple invoice PDFs (around four or more) and switching between tabs while they are still loading. This behavior is specific to NWBC and cannot be reproduced in standard web browsers.

Environment

  • NetWeaver Business Client Runtime Environment
  • NetWeaver Business Client for HTML
  • SAP Business Client
  • SAP NetWeaver
  • SAP Web Application Server for SAP S/4HANA
  • ABAP PLATFORM - Application Server ABAP
  • SAP Cloud for Customer

Reproducing the Issue

  1. Start the SAP Business Client (NWBC) desktop application.
  2. Navigate in the C4C UI e.g. Contacts view.
  3. Search for a contact and open the related individual customer record.
  4. Click in the action to open the invoice PDF (current mashup opens it in a new tab/window).
  5. Open multiple invoice PDFs (four or more) in separate tabs and switch between the tabs while the PDFs are still loading.
  6. Observe that NWBC displays "The browser rendering the content of tab '?' crashed".

Cause

A bug in the Edge Browser causes the WebView2 process stack used by the SAP Business Client control to crash when the application opens PDFs in new tabs/windows from the custom mashup.

Resolution

The solution is to update the C4C Custom Mashup to embed the PDF within an <iframe> instead of opening it in a new URL.

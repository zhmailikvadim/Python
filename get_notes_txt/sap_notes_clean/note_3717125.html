Symptom

Unable to save the Replacement scope definitions when using the import method Clean and replace a subset of data. When reopening the import settings, the last Replacement scope definition is always deleted.

Environment

SAP Analytics Cloud Enterprise 2026

Reproducing the Issue

  1. Access the affected model.
  2. Select a new import job, and edit the import settings via the "data source" icon.
  3. Select the method "Clean and replace a subset of data" and add 3 Replacement Scope Definitions.
  4. Hit save and refresh the page.
  5. Open the Import Settings via the "data source" icon once again, but notice that the last Replacement Scope Definition has been deleted.
  6. The same setting, when done via More (3 dots) > Edit > Import settings, is saved correctly.

Cause

Dimensions in Replacement Scope Definition were not mapped to source columns, causing the import setting to be considered incomplete and resulting in scope loss upon reopening.

Resolution

To save a dimension in the replacement scope, the dimension must first be mapped to a source column. This is because the import setting is considered incomplete when there are still unmapped dimensions, so the replacement scope does not consider those dimensions when we reopen the import settings.

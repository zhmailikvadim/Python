Symptom

  • User is unable to login to Ariba.
  • Error message displayed: "Identity Provider could not process the authentication request received. Delete your browser cache and stored cookies, and restart your browser. If you still experience issues after doing this, please contact your administrator."

Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.  

Environment

Identity Authentication Service

Reproducing the Issue

  1. Attempt to login to Ariba.
  2. Observe that the login prompt does not appear.
  3. Error message is displayed: "Identity Provider could not process the authentication request received. Delete your browser cache and stored cookies, and restart your browser. If you still experience issues after doing this, please contact your administrator."

Cause

The URL from the SAML request is different from the Entity ID.

Resolution

Check if the URL from the Entity ID and SAML2 authentication request are matching.

  1. Inside IAS Administration Console, go to Monitoring & Reporting -> Troubleshooting Logs
  2. Search the correlation ID from the error (as per screenshot below)
  3. Find the URL on an error message like "Identity Provider could not process SAML2 authentication request sent over POST due to configuration error. No trusted Service Provider configuration with name [http://<xxxxx>-<xxxxx>.ariba.com] found Correlation ID: 12345678-1234-1234-1234-123456789876 ".
  4. Finally, compare and correct the URL on Entity ID.
    A common difference found is the SAML URL starting with http:// and the Entity ID starting with https://

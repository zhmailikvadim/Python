Symptom

When attempting to cancel a goods movement document in an Advance Intercompany Stock Transfer scenario, the cancellation fails because the posting date used for the cancellation corresponds to the posting date of the original material document rather than the current date. If the posting date of the original document falls within a closed period, one of the following error messages is issued:

  • M7053: “Posting only possible in periods XX and XX in company code XX”

  • M7037: “In plant &, post to current period &/&”

Other Terms

 M7 037, M7-037, M7/053,  Cancel Goods Issue, Reverse GI with Posting Date, Posting Date,

Reason and Prerequisites

Posting date of the cancellation document is in the past and the previous period is closed.

Solution

Use the Business Add-In (BAdI) MMIM_CREATE_GM_POSTING_DATE (method IF_MMIM_CREATE_GM_POSTING_DATE~CHANGE) to adjust the posting date during cancellation via the VCM monitor.

Please refer to the following manual implementation to create the BAdI implementation, including copying the sample code:

Steps to Create and Activate the BAdI Implementation

  1. Go to transaction SE18.

  2. Enter BAdI name MMIM_CREATE_GM_POSTING_DATE and choose Display.

  3. Position the cursor on Implementation, then right mouse-click and select:
    Create Enhancement Implementation

    • Enhancement Implementation: ZZ_MM_POSTING_DATE

    • Short Text: ZZ_MM_POSTING_DATE

  4. Click Continue and save the enhancement implementation.

  5. Position the cursor again on Implementation, then right mouse-click and select:
    Create BAdI Implementation

    • BAdI Implementation: ZZ_POSTING_DATE_IMP

    • Description: ZZ_POSTING_DATE_Implementation

    • Implementing Class: ZZ_POSTING_DATE_IMP

  6. Press Enter, then click Copy Sample Class.

  7. Replace the implementation of METHOD if_mmim_create_gm_posting_date~change with the code from the attachment "IF_MMIM_CREATE_GM_POSTING_DATE.txt".

  8. Activate the BAdI implementation.

If serial numbers are involved, the posting date must be the current date or the same as, or later than, the Date of Last Goods Movement (see SAP Note 2515305). In this case, you can adjust the implementation by uncommenting out the following statement so that the posting date is set to the system date:

           postingdate = cl_abap_context_info=>get_system_date( ).

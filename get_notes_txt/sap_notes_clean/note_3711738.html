Symptom

Your usual ABAP transports regularly execute the steps 'Versioning' and 'Generation'.

Your ABAP server has a lot of batch processes available and you are looking for a way to reduce import times.

Reason and Prerequisites

Usually all import steps of a transport request are executed in sequential order.

But this is not needed for the steps 'Generation' and 'Versioning'. Once the 'Main Import' step has been executed, they can be processed independently of the other steps.

Remark:

This is true up to kernel release 9.18. Starting from that release, 'Versioning' can no longer be executed independently of other steps. It can have a dependency on the transport step 'After-Import-Method'. So from 9.18, only the step 'Generation' can be executed independently.

Solution

The import behavior described above can be used when you have installed at least one of these tp versions or higher:

9.16: 381.699.95

7.93: 381.587.93

7.89: 381.576.93

See note 19466 on how to download and install a new tp version.

Additionally, you need to configure the tp transport profile parameter ASYNCHRONOUS_STEPS with the step letter which should be executed asynchronously.

E.g., configure

ASYNCHRONOUS_STEPS = 'VvG'

in kernel release 7.93 for the asynchronous execution of 'Versioning', 'Versioning after Import', and 'Generation'

Or configure

ASYNCHRONOUS_STEPS = 'G'

in kernel release 9.16 or higher for the asynchronous execution of 'Generation'

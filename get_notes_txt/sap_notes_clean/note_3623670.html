Symptom

In Fiori, open the agreement and search for an entry where the settlement partner field contains an invalid value. The system is expected to throw an error, as the settlement partner maintained is not valid. In the backend system, performing the same action correctly triggers an 'Invalid Partner' error and prevents the agreement from being saved. The same behavior is expected in Fiori.
However, in the current Fiori behavior, the system allows the agreement to be saved without displaying any error message for the invalid partner.

This note ensures that the system will trigger the appropriate error message when an invalid partner is maintained.

Other Terms

Agreement, Settlement Partner, /IRM/IPAGRM

Reason and Prerequisites

This is caused by program error

SAP Note#3623714 is available for ECC release.

Solution

Please implement the correction instructions. These corrections are only valid if you have applied the following Support Pack(s).

Release V/4 – 2021, 2022, 2022 V2, 2023, 2023V2, and 2023 V3.

Use transaction SNOTE to implement the correction instructions specified below and activate the affected program object

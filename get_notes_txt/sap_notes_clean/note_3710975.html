Symptom

You have created a Dynamic Property rule to designate the "Sales Area" field as mandatory within the Account service, but this rule is not functioning correctly. Despite configuring the Dynamic Property, the "Sales Area" field does not become mandatory when expected.

Environment

  • SAP Sales Cloud Version 2
  • SAP Service Cloud Version 2

Reproducing the Issue

1. Click on the User Icon.
2. Go to Settings > All Settings. 
3. Go to the Extensibility Administration.
4. Select Account Service > Account Team Member.
5. Go to the Dynamic Properties tab.
6. Create a Dynamic Property.

7. Go to the Account Object Worklist (OWL).
8. Open an Account.
9. Navigate to the Account Team Members.
10. Click to Add.
11. The Sales Area does not become mandatory.

Cause

The current system configuration does not support the application of Dynamic Properties to make the "Sales Area" field required. This limitation arises because the "Sales Area" field in the user interface represents a combination of three backend fields: Sales Organization, Distribution Channel, and Division. This behavior in the UI layer complicates the application of dynamic properties directly to the "Sales Area" field.

Resolution

To ensure that the "Sales Area" is marked as required, it is recommended to implement an External Hook. The idea would be using a postHook external hook for raising errors whenever the combination of three fields are not filled (i.e., meaning no Sales Area) and hence blocking instance creation, This approach provides more flexible validation capabilities and can effectively handle the dependencies involving the three backend fields. By utilizing an External Hook, you can achieve the desired behavior and mandate the "Sales Area" field as needed.

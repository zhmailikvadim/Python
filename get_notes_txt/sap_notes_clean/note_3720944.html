Symptom

  • When checking the finished goods in transaction CKM3, under the "Production" section, cost contributions from raw material Valuation Types are NOT consumed in the production order in the goods issue (movement type 261) document.
  • Is it standard system behavior for the Material Ledger to associate non-consumed Valuation Types with a Finished Good in CKM3?

Environment

  • Controlling (CO)
  • SAP R/3
  • SAP R/3 Enterprise 4.7
  • SAP ERP Central Component
  • SAP ERP
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA Finance
  • SAP S/4HANA

Reproducing the Issue

  1. Enter transaction CKM3
  2. Click receipts and production in the dropdown.
  3. Under the Production section, cost contributions from raw material Valuation Types are NOT consumed in the production order in the goods issue (movement type 261) document.

Cause

  • The valuation type is not used as a key characteristic for the procurement processes of the actual BOM. As a result, the calculation is performed on all valuation types for the material together, rather than on the valuation type level.

Resolution

For production of batch-valuated, the SAP best practice recommendation is setting the controlling level on the individual by setting the Parameter DETAILED_CONTROLLING_BATCHES in table CKMLMVADMIN. See SAP Note 1320758 Controlling level - order-specific. 

If setting parameters all new orders, can create afterwards get their own production process which will completely separate the actual BOM of different batches of the same product. If using batch-valuated materials also in subcontracting process, the solution is different. It will require to have to activate the valuation type (batch) as characteristics in the subcontracting process. For details see SAP Note 1051224 Valuation type as characteristic in procurement process.

As explained in SAP Note 1320758, there are 2 options on how to assign the values for BADI_PROD_CONTROLLING_LEVEL in plant:

  1. Set CMLMVADMIN global key DETAILED_CONTROLLING_BATCHES, afterwards the controlling level will be automatically set to the individual order in case of batch valuated products.
  2. Or if required to only have this logic for a certain plant or order type etc., option to use BAdI BADI_PROD_CONTROLLING_LEVEL to make the decision based on the input parameters (as described in SAP Note 1320758), for example only set parameter CX_CONTROLLING_PER_ORDER = X for orders from plant.

Symptom

  • An error occurs during Passkey registration in an Android application integrated with SAP Customer Data Cloud.
  • The operation returns error code 400096.
  • The error appears within a web screenset.

Environment

  • SAP Customer Data Cloud
  • RaaS
  • Mobile SDK
  • Android

Reproducing the Issue

  1. Open an Android application that integrates SAP Customer Data Cloud and uses a web screenset for authentication.
  2. Initiate Passkey registration from the web screenset
  3. Observe that the flow fails and returns error code 400096.

Cause

The issue is related to the FIDO usage tied to web screen-sets, which is not supported on Android.

Resolution

  1. Do not initiate Passkey (FIDO) flows from web screensets on Android, as they are not supported.
  2. Implement Passkey authentication using the native Android SDK methods provided by SAP Customer Data Cloud.
  3. Refer to the SDK guidance and important notes, and update the app to use native Passkey APIs.
  4. Test Passkey registration again using the native SDK to confirm the error 400096 no longer occurs.

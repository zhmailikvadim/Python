Symptom

You have a PO Claim Check scenario where you  use SCPMergingBean. When incoming the payloads have XML prolog, the PI messages fail with MAPPING_ERROR with an exception message similar to:
com.sap.aii.utilxi.misc.api.BaseRuntimeException: The processing instruction target matching "[xX][mM][lL]" is not allowed.
at com.sap.aii.mappingtool.tf7.rt.xparser.MTSaxHandler.run(MTSaxHandler.java:259)

You can see in the monitors that the (large) input payload for the mapping has XML prolog.

Other Terms

"[xX][mM][lL]" is not allowed, Claim Check, SCP, SCPMergingBean

Reason and Prerequisites

The issue is caused by a program logic error in SCP application in PI.

Solution

The issue is resolved by source code correction. Apply the patch corresponding to your PI/PO versions and SP level as per the "Support Packages & Patches" section of this note.

Symptom

After posting the goods receipt, the account assignment type field appears blank in the resulting journal entry.

Environment

SAP S/4HANA Cloud Public Edition

Reproducing the Issue

  1. Post a goods receipt for a purchase order.
  2. Open App Manage Journal Entries - New Version (F0717A) and display the created journal entry line items.
  3. Observe that the Account Assignment Type (ACCASTY) field is blank although the WBS element is set as Required Entry in the Additional account assignment group in Define Field Status Variants Configuration Activity.

Cause

The posting line is made to a balance sheet G/L account. The Account Assignment Type (ACDOCA-ACCASTY) is only filled for postings to primary or secondary cost elements and remains empty for balance sheet accounts, even if a WBS element or cost center is entered.

Resolution

  1. Identify whether the affected journal entry line is posted to a balance sheet G/L account.
  2. Acknowledge that ACCASTY is populated only for postings to primary or secondary cost elements; for balance sheet accounts it remains blank by design.
  3. If Controlling reporting requires ACCASTY to be filled, adjust account determination or posting so that the relevant line posts to an appropriate primary or secondary cost element G/L account.
  4. Post a test goods receipt and verify in Manage Journal Entries that ACCASTY is now populated for the cost element line.

See Also

Manage Journal Entries (F0717A), Manage G/L Account Master Data (F0731A)

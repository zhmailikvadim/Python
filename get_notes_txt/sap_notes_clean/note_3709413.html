Symptom

The SDATE feature is currently considered for severance payment, Lump Sum D length-of-service calculation, and unused long service leave calculation in Termination Organiser. However, the SDATE feature should be used only for severance payment and Lump Sum D length-of-service calculation. If SDATE is not maintained, the QDATE feature should be used instead for these calculations.

Long service leave calculation should always use the date retrieved from the Date Specifications (IT0041) infotype based on the QDATE feature.

Other Terms

 Function Module:HRTO_AU_CALC_SERVICE_DTLS

Reason and Prerequisites

SDATE defines severance date types in IT0041, while QDATE defines the Long Service Leave (LSL) start date via IT0041.Whereas if SDATE is maintained it is considered for severance payment, Lump Sum D length-of-service calculation, and also for unused long service leave calculation.

However QDATE feature should be used for unused long service leave calculation.

Note Version

Reason for version

01 & 02

General Release

Solution

The HRTO_AU_CALC_SERVICE_DTLS function module has been modified to ensure the system correctly considers the SDATE/QDATE feature for payment calculation.

As a general rule SAP recommends that you implement the Support Package specified for your release. However, if you need to install a solution earlier, use the SAP Note Assistant to implement the correction instruction. You can find more information about the SAP Note Assistant in SAP Support Portal, under SAP Note Assistant.

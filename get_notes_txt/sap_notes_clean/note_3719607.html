Symptom

  • The initial Issuer in IAS Admin Console-> Tenant Settings -> OpenID Connect Configuration was <IAS tenant ID>.accounts.cloud.sap or <IAS tenant ID>.accounts.ondemand.com; but it was changed to https://<IAS tenant ID>.accounts.cloud.sap or https://<IAS tenant ID>.accounts.ondemand.com
  • There is a requirement to reverse the Issuer to the non-compliant value <IAS tenant ID>.accounts.cloud.sap or <IAS tenant ID>.accounts.ondemand.com. However, the non-compliant value does not appear in the selection list

Environment

SAP Cloud Identity Services

Cause

The previous issuer value was removed from the tenant settings options after the change, which prevents rollback functionality.

Resolution

Please refer to the following steps to reset the Issuer:

  1. Access Admin Console in Browser
  2. Navigate to Tenant Settings -> OpenID Connect Configuration
  3. Open Developer Tools(F12 for Google Chrome) and go to the Network Tab of the Developer Tools
  4. Make "dummy" change to enable the Save button (e.g. change Domain for Browser Flows to another option then back to default)
  5. Click on the Save button
  6. Locate the request (.../service/settings/oidcConfig/?BasicAuthn=disabled) in the Network tab of the Developer Tools and right click on it; then Go to Copy-> Copy as fetch in the information after the right click
  7. Go to the Console Tab of the Developer Tools
  8. Paste the Fetch into the console (Pasting might need to be allowed first)
  9. Edit the oidcName property in the body of the pasted request to remove the https:// prefix
  10. Click Enter button on the keyboard to send the request
  11. The Issuer is now reset to the non-compliant one

Symptom

When printing multiple revision data using the user function, the pricing data is not populated correctly in the custom report. For the second revision record, the pricing data from the first record is still retained after calling the function module /IRM/IPPQ_INFOCUS_READ, resulting in incorrect pricing information being displayed.

This note ensures that the correct data is exported from the function module /IRM/IPPQ_INFOCUS_READ and displayed properly.

Other Terms

/IRM/IPPQ_INFOCUS_READ, /IRM/IPPQM, /IRM/IPPQMAM, IRM/IPPQRVM, ET_RULES, Multiple Revisions pricing data, /IRM/IF_IPPQ_USER_FUNCTIONS~EXECUTE

Reason and Prerequisites

This is caused by program error

For V4, SAP Note #3704997 is available

Solution

Please implement the correction instructions attached. These corrections are only valid if you have applied the following support packs.

Release 6.3 – Base Package and 6.4 Base package

Use transaction SNOTE to implement the correction instructions specified below and activate the affected program object.

Symptom

During schema import, the following warning message is displayed:

"java.sql.SQLWarning: general warning: Variables with the same name as a parameter are not allowed in main block: CURRENCY"

Environment

SAP Business One, version for SAP HANA

Reproducing the Issue

Scenario:

  1. Export the Schema on the source system and compress the exported files on the Linux server.
  2. Transfer the compressed file to the target system.
  3. Extract the exported schema on the target system.
  4. Attempt to import the schema.

Expected result:

The schema import is successful without any warning message.

Actual result:

The schema import is successful, but the following warning message is displayed:

"java.sql.SQLWarning: general warning: Variables with the same name as a parameter are not allowed in main block: CURRENCY"

Cause

The warning occurs when variables declared in the main block have the same names as input or output parameters, causing conflicts during evaluation.

Resolution

This warning message does not impact the functionality of the system.

          The warning appears if one attempts to create a procedure or function, or execute an anonymous block, where one or more variables declared in the main block have the same names as input or output parameters.

          Example:

          CREATE PROCEDURE proc1 (IN abc INT) AS
          BEGIN
            DECLARE abc INT = 1;
            ...
          END;

          In this case, the parameter 'abc' cannot be evaluated inside the procedure because it is hidden by the variable 'abc'. Similarly, output parameters or function return values cannot be set from within a function, procedure, or anonymous block if they are hidden by variables.

          Enable debug database tracing as documented in SAP Note 2134959, run the import again to identify the object causing this warning, and adjust the procedure accordingly.

          Keywords

          import, warning, Variables, CURRENCY

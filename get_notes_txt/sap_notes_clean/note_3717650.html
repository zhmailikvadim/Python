Symptom

  • The database backup in file system is failing and shows an error similar to the one below in HANA Studio:

    Could not back up database XXX@XXX
    [447] backup could not be completed, Error during stream send: Error: multistream data transfer 
    terminated during send after transferring XXXXXXXXXXXX instead of XXXXXXXXXXXX bytes, 
    rc=$sysrc$: $sysmsg$
  • The Backup has been triggered for the file system.
  • The indexserver trace additionally reports the error as shown below:
     ERROR in async. file transfer: exception  1: no.2000008  (Basis/IO/FileAccess/impl/LocalFileCompletionThread.cpp:338) TID: 768XXX
        Error during asynchronous file transfer (async_write), rc=28: No space left on device; 

Environment

SAP HANA, platform edition 2.0

Reproducing the Issue

Initiate the HANA data backup

Cause

A disk full situation can be identified by reviewing the indexserver trace or executing df -h from the operating system level, which should reveal that the backup mount is close to full or lacks adequate space for the backup.



Resolution

Either free up the space on the backup mount following the guidance in 2096851 - Backup related Housekeeping using HANA Studio, or add more disk storage.

To determine the expected size of the next data backup, you can also refer to the M_BACKUP_SIZE_ESTIMATIONS system view.



Remarks:

  • Do NOT delete log files at OS level as this can render the HANA database inconsistent and unusable and will result in the need for a database recovery;
  • Alternatively to Deletes backups from the backup catalog using BACKUP CATALOG DELETE Statement

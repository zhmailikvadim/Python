Symptom

  • Silent Data Migration (SDMI) classes for Portfolio Management application display status "initial" in SDM_MON.
  • The following classes are affected:
  • /S4PPM/CL_SDM_PORT_D_ACF_01
  • /S4PPM/CL_SDM_BUCKET_D_ACF_01
  • /S4PPM/CL_SDM_INM_INIT_ACF_01
  • /S4PPM/CL_SDM_ITEM_D_ACF_01

Environment

  • SAP S/4HANA
  • SAP S/4HANA Cloud Private Edition 

Reproducing the Issue

  1. Access transaction SDM_MON.
  2. Observe that the SDMI following classes have status "initial" in one or more clients:
  • /S4PPM/CL_SDM_PORT_D_ACF_01
  • /S4PPM/CL_SDM_BUCKET_D_ACF_01
  • /S4PPM/CL_SDM_INM_INIT_ACF_01
  • /S4PPM/CL_SDM_ITEM_D_ACF_01

Cause

Resolution

Program R_SDM_ONLINE_EXEC can be used to start a SDMI migration class manually.

  • If the migration is completed successfully by doing so, the observed issue is solved.
  • If the migration fails again, the program may provide additional information as to why the class did not complete successfully.

Due to dependencies between the Portfolio Management objects, if the migration classes are executed via program R_SDM_ONLINE_EXEC, this must be done in the following order (in each case, ensure one class has completed successfully before triggering the next one):

  1. /S4PPM/CL_SDM_PORT_D_ACF_01
  2. /S4PPM/CL_SDM_BUCKET_D_ACF_01
  3. /S4PPM/CL_SDM_INM_INIT_ACF_01
  4. /S4PPM/CL_SDM_ITEM_D_ACF_01

See Also

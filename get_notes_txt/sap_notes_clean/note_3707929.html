Symptom

  • errorcode=-4221 encountered during database connectivity between sap java application and db2 database.
  • Error message: "Couldn't connect to DB com.sap.sql.log.OpenSQLException: Exception of type com.ibm.db2.jcc.am.user caught: [jcc][1068][10625][4.33.32] Caught java.security.user while encrypting data. See attached Throwable for details. ERRORCODE=-4221, SQLSTATE=null."

Environment

SAP on Db2 for LUW

Cause

With DB2 11.5.9.0 and version 4.33 of the IBM Data Server Driver for JDBC and SQLJ (db2jcc4.jar), the default security mechanism is encrypted user ID and password security.

Resolution

Try the following workaround of reverting to the previous(11.5.8 and before) default security mechanism for userID and password:

Add "securityMechanism=3" to your connection string using the sap configtool.

  1. Open the configtool located at: /usr/sap/<SID>/.../j2ee/configtool/configtool.sh
  2. Got to "Secure Store > Connection Pools" and you should find URL/connection string like this:  "jdbc:db2://<hostname>:<port>/<SID>:currentSchema=<SCHEMA>;<param_1>=<value>;<param_2>=<value>;"
  3. Add string "securityMechanism=3;" at the end of the connection string. Add extra ';' at the begining to separate it from previous parameter config. For example: "jdbc:db2://<hostname>:<port>/<SID>:currentSchema=<SCHEMA>;<param_1>=<value>;<param_2>=<value>;securityMechanism=3;"

After that, restart the server, the issue can be resolved.

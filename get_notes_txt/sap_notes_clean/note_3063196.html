Symptom

You work with the universal prescription N3KVUNIRP Version 1 as part of discharge management.
If you change the patient data or the insurance relationship in the personal data field, the system automatically updates the selected physician without user intervention.

Other Terms

PPA Sample, Admission/Visit, Ins./Ins. Provider Selection

Reason and Prerequisites

Program error

Solution

This SAP Note changes the system behavior as follows:

The selected physician is prevented from being updated if changes are made to the personal data field.

The following logic is defined for the use of the universal prescription in discharge management:

The system presets the physician number in the LANR field for the default physician. If there is no valid physician number, the field remains empty.
When you change the preset physician, the "Person List: Initial Screen" dialog with the "Physician" and "Employee" checkboxes is selected for the physician selection.
When the physician is transferred, the system checks for a valid LANR.
In discharge management, the field remains empty if there is no valid LANR.
If the physician has several valid LANR, these are offered for selection.
When you change the patient data or the insurance relationship, the system does NOT change the selected physician.

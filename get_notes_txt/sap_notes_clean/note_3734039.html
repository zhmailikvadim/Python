Symptom

When analyzing classic BAdIs ME_CHECK_ALL_ITEMS and ME_DEFINE_CALCTYPE, the interface IF_BADI_INTERFACE is not present/assigned.

Environment

  • Materials Management (MM)
  • SAP ERP
  • SAP enhancement package for SAP ERP
  • SAP S/4HANA
  • SAP S/4HANA Cloud Private Edition

Cause

Classic BAdIs do not necessarily use IF_BADI_INTERFACE.

SAP often provides a generated BAdI interface with naming patterns such as:

IF_EX_<BADI_NAME> (SAP standard)

ZIF_EX_<BADI_NAME> (customer namespace, if applicable)

Some classic BAdIs are implemented with internal mechanisms (including function-module based logic) and the generated interface may not inherit from IF_BADI_INTERFACE. This behavior is standard within SAP.

New BAdIs (Enhancement Framework) always has an explicit interface that inherits from IF_BADI_INTERFACE.

Resolution

SAP-recommended programmatic approach (stable / forward-compatible)

Avoid direct table reads of BAdI metadata (storage differs across technologies/releases). Use SAP runtime/definition classes instead:

  • Classic BAdIs: use CL_BADI_DEF
  • New BAdIs (Enhancement Framework): use CL_ENH_BADI_RUNTIME

These APIs read the underlying metadata correctly regardless of where/how SAP stores it in a given release.

Keywords

ME_CHECK_ALL_ITEMS, ME_DEFINE_CALCTYPE, IF_BADI_INTERFACE, interface missing

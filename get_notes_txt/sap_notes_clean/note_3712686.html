Symptom

After migrating system to S/4HANA, there is issue during dependency executing in S/4HANA system, and you find table CUEXN is empty in system.

The possible symptom could be:

  • Dump in CU51/CU50:
    Category               ABAP programming error
    Runtime Errors         RAISE_EXCEPTION
    ABAP: Program          SAPLCUPM
    Application Component  LO-VC
  • Different behavior is observed during BOM explosion in ECC & S/4HANA system.

Environment

  • SAP ERP Logistics (LO)
  • SAP S/4HANA
  • SAP Enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA Cloud Private Edition

Cause

In S/4HANA the compilation is stored in a different table (CUEXN instead of CUEX).
This table does not exist on ERP ECC.
When upgrading system into S/4HANA, report MIGRATE_CUEX_TO_CUEXN is run to migrate CUEX to CUEXN.

Reason of why the table is not migrated is currently unknown.

Resolution

Execute report RCU_COMPILE_DEPENDENCY to compile the dependencies in system.

Keywords

CS_BOM_EXPLOSION, cukrx_read_codes_from_db_hist,  LCUKRFRE

Symptom

  • Customer wants to reduce the provision of roles and authorizations to Key Users to a minimum, only assign the Business Role SAP_BR_EXTENSIBILITY_SPEC and PFCG role SAP_NW_APS_EXT_CFL_APP
  • When trying to publish a custom field or data source extension, receive the error "Could not be published"

Environment

SAP S/4HANA 1909 and above version

Reproducing the Issue

  1. Assign the Business Role SAP_BR_EXTENSIBILITY_SPEC and PFCG role SAP_NW_APS_EXT_CFL_APP to key users
  2. Logging the Custom fields and logic fiori application
  3. Create a new custom field
  4. Publish and fail with error message "Could not be published"

Cause

  • The role SAP_NW_APS_EXT_CFL_APP contains authorizations to the Custom Fields app and for maintenance of the CFDF repository object. However for publishing there are additional authorizations required to generate other repository objects (CDS extends, data elements, etc.), which are contained in role SAP_NW_APS_EXT_CFD
  • Customers don't want to provide development authorizations, reduce the required authorizations to key users, so the generation role SAP_NW_APS_EXT_CFD should be assigned to the job user

Resolution

  1. Run transaction code S_ATO_SETUP
  2. Click the "Register User for Job" button under the section "Key User App specific settings"
  3. Leave the table name as "ATO_PK_JOB_USER" and click "Maintain" button
  4. Create or update the job user with item type CFDF value (Assign the PFCG role SAP_NW_APS_EXT_CFD)

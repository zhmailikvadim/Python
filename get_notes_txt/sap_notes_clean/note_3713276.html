Symptom

The ME2L report displays incorrect values in the "Still to be Invoiced (val.)" field for service PO.

Environment

  • Materials Management (MM)
  • SAP ERP Central Component
  • SAP ERP
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA
  • SAP S/4HANA Cloud Private Edition

Reproducing the Issue

  1. T-code ME21N: Create PO with item category as D, the PO can include service line or limit.
  2. Make sure the 'Goods receipt' value is NOT set, only set the 'Invoice Receipt' flag.
  3. T-code MIRO: Post invoice for the PO.
  4. T-code ME2N: The "Still to be Invoiced (val.)" is not calculated correctly.

Cause

  • PO was created with item category D, and non-deductible amount is maintained under the Conditions tab.
  • PO's 'Net Price' is not including the non-deductible tax amount. However, the invoice value in the PO history is including the non-deductible tax amount.
  • By this reason, in transaction ME2N, to ensure the 'Still to be invoiced' value is calculated correctly, the system excludes the non-deductible tax amount from the invoice value.

    The calculation for "Still to be invoiced" value is as follows:
    Still to be invoiced value = PO net price - Invoice value (excluding the non-deductible tax amount)
                                        = PO net price - Invoice value * PO net price / (PO net price + PO non-deductible tax amount)

Resolution

This is the standard system design.

Symptom

You use BAPI_PRODORD_CHANGE to change a production order. You provide a storage location in the parameter ORDERDATA-STORAGE_LOCATION and also set ORDERDATAX-STORAGE_LOCATION.
The MRP area is not determined from the storage location.

The same symptom occurs when you change a process order using BAPI_PROCORD_CHANGE.

Other Terms

Change order, BAPI
BAPI_PRODORD_CHANGE, production order
BAPI_PROCORD_CHANGE, process order
MRP area, AFPO-BERID

Reason and Prerequisites

The MRP area is not determined from the specified storage location.

Solution

After you implement the corrections, the system determines the MRP area from the storage location.
The corresponding MRP controller is determined as well. However, if an MRP controller is transferred in the parameter ORDERDATA-MRP_CONTROLLER during the BAPI call, it has priority.

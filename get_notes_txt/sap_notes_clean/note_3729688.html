Symptom

Catalog items created in the "Manage Catalog Items" app are not displayed in transaction ME51N. Instead, a blank screen appears when launching the catalog. 

Environment

  • SAP S/4HANA
  • SAP S/4HANA Cloud Private Edition

Reproducing the Issue

  1. Go to transaction code ME51N
  2. Click the "Catalog" button available in the item overview
  3. Choose any Web Service ID
  4. Observe that the screen appears blank instead of displaying the catalog items.

Cause

There are two types of catalogs:

  • Internal catalogs: These catalogs do not have an URL, so they can't be "launched". Catalogs items are created in the Manage Catalog Items app and they can be added to the cart using the Cross-Catalog Search feature.
    Examples of internal catalogs: MATERIALMASTER, S4H_GLOBALCATALOG, HUB_EXTRATION.

  • External catalogs: also known as punch-out catalogs, they are provided by some Third-Party company (e.g. SAP Ariba) and can be accessed as a website. It is possible to access the catalog (even outside of the SAP environment) and add items from there. 

Resolution

Only external catalogs should be used in ME51N transaction. Ensure that catalog call structure parameters are configured correctly in transaction code OLME: Navigate to OLME > Purchasing > Environment Data > Web Services: ID and Description.

For internal catalogs, use the Cross-Catalog Search feature available in native Fiori apps (e.g., My Purchase Requisitions New, Manage Purchase Requisitions Professional) as outlined in KBA 2680363.

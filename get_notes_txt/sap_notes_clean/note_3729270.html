Symptom

An user creates an inspection rule from the inspection type in ERP side via QL11 transaction. The created inspection rule is inconsistent with the inspection type setting in MM03's QM view. 'Lot summary' and 'Insp. Ctrl' are different. For example the inspection type contains lot summary as '2' but created inspection rule has Insp. Ctrl as ' '.

Environment

  • SAP Extended Warehouse Management (SCM-EWM)
  • SAP Extended Warehouse Management Quality Management (SCM-EWM-QM)
  • Decentralized EWM

Reproducing the Issue

  1. Maintain an inspection type in MM03 for a material
  2. Use QL11 and create an inspection rule in EWM
  3. Check created rule and the field Insp. Ctrl is inconsistent with Lot summary in MM03

Cause

The functionality is designed to use the EWM customizing to determine the inspection lot summary settings. As a result such inspection rule is a cross-warehouse rule which is considered to be valid for multiple warehouses. Oppositely, ERP can have different settings on plant level for the same inspection type and material. Therefore, such inconsistency happens.
Such inspection rule's Insp. Ctrl is set based on EWM customizing: SCM Extended Warehouse Management-> Cross-Process Settings-> Quality Management-> Quality Management for Decentralized EWM-> Basics and Integration-> Communication with an External QM System-> Define Insp. Rule Settings for Inbound Inspections (IOT 4)/Define Insp. Rule Settings for Whse-Internal Insp. (IOT 5). It will not consider the lot summary set in the ERP material master but directly use the setting here.

Resolution

There is currently no standard solution for this part. A possible resolution will be enhancing local types lty_e1bp1001004_qmat of /SCWM/CL_QIR_INPUT_MATQM at the end with a new field for example "qdoc_cntrl TYPE char1". With an additional enhancement in class /SCWM/CL_QIR_INPUT_MATQM method GET_DEFAULT_VALUES_BY_INSPTYPE to use in this case is_qmat-qdoc_cntrl to update the relevant fields in rt_default_values.

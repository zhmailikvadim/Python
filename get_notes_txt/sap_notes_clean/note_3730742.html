Symptom

  • When running a Data Manager package in SAP BPC, no records are loaded into the target Infoprovider.

    Observed Behavior:
    Record count: 0
    Accept count: 0
    Reject count: 0
    Skip count: 0
    Package status: WARNING

Environment

  • SAP Business Planning and Consolidation 10.1, version for SAP NetWeaver

Reproducing the Issue

  1. Open SAP BPC and navigate to Data Manager.
  2. Select or create a Data Manager package that loads data from a source cube.
  3. Maintain a selection/filter on a cube field (e.g., FISC_PERIOD).
  4. Set the filter to a value that does not exist in the source data (e.g., FISC_PERIOD = 1).
  5. Save and execute the Data Manager package.
  6. Review the package execution log.

Cause

The Data Manager package uses a selection/filter on a cube field with a value that does not exist in the source data.

Resolution

  1. Open the Data Manager Package in SAP BPC.
  2. Navigate to the Source Data / Selection settings.
  3. Modify the filter: Update the filter to match actual source values (e.g., FISC_PERIOD = 0) or Remove the filter to load all available records.
  4. Save the changes.
  5. Re-run the package.
  6. Verify that records are now being loaded successfully.

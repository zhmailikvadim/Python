Symptom

In the DmfA/DmfAPPL-Report (RPCDMFB0) the amount of deduction code 3102 (elderly employees) is not correct when the birthdate data has been changed in infotype 0002(Personal Data).

Other Terms

DMFA, RPCDMFB0, deduction 3102, PAYR, FM 'HR_BE_PAY_HAS_AGE_QUARTER_END', IT0002, T5F99FC, birthdate.

Reason and Prerequisites

The function module 'HR_BE_PAY_HAS_AGE_QUARTER_END' is reading the wrong birthdate data if the birthdate has been changed and there are more than one records in infotype 0002.

This is a program error.

Solution

As a general rule, SAP recommends that you install a solution by applying a Support Package. However, if you need to install a solution earlier, use the Note Assistant to implement the correction instruction. You can find more information about the Note Assistant in SAP Service Marketplace, under service.sap.com/note-assistant.

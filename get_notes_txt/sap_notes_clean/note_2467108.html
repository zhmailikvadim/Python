Symptom

  • You are using the Web Callback feature in SAP Contact Center and verified the Last Result in the "LIST OF CALLBACKS CREATED IN THE SYSTEM" is showing the status of "Connected", yet the callback is still active:
  • Callback calls are redialing even when the call was originally connected before. Your expectation is that the callback should not redial

Environment

SAP Contact Center 7.0

Reproducing the Issue

  1. Make a test call to a queue configured to have a queue callback or create a web callback request (see Configuring Web Callback)
  2. Logon to CDT and to the CallBack queue configured
  3. Call is offered to the agent
  4. Agent picks accepts, then CDT dials out to the customer number
  5. Customer answers the call and hangs up
  6. After some time, the same call is alerted to the same or other agents serving that that CallBack queue

Cause

  • Missing configuration
  • This is the standard behaviour when no CallBack Script has been attached to the CallBack Queue

Resolution

  • Callback queues require a corresponding mandatory Callback Script
  • Since a callback script is mandatory in the callback queue, agents who use IP desk phones or serve in queues as external agents cannot log on to callback queues.
  • This is true for WebCallback and Queue Callback features
  • To use the callback function in your system, please follow the steps outlined in the "Queue Callback" online help, under the "Configuration" sub-title:

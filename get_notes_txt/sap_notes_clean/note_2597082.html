Symptom

  • During an upgrade, ASE hits a timeslice at address "read"
  • With a stack trace including module dmpld__read_dump_record
  • ASE log records of note are:
2018/01/22 17:13:07.78  The dump history file '/opt/sybase/ASE-16_0/dumphist' is being updated from version '1.0' to version '1.1'. The previous file will be renamed as ....
2018/01/22 17:14:48.00  timeslice -1002, current process infected at 0x7f8ef20947fd (read+0x2d)

with a stacktrace including functions:

(null)+0x7f8ef2095850()
ufread
ufgets
dmpld__read_dump_record

Environment

SAP Adaptive Server Enterprise (ASE) 15.7 & 16.0

Cause

  • The timeslice is linked to the upgrade of the dumphist file
    • The timeslice is 100 seconds after dump history file message (100.2 seconds = 1002 clock ticks as reported by the "timeslice -1002")
    • The dmpld__read_dump_record function is for reading the dumphist file.
  • ASE is making a call on the OS to read the dumphist file.
  • This call does not return within the maximum time ASE allows (cpu grace time) and ASE terminated this process with timeslice.
  • The cause of this error is outside of the ASE domain, as the OS can't complete the ASE request to access the file.
    • Most likely there is a (temporary) glitch regarding the dumphist file.

Resolution

There are no effects known from this issue, so it can be ignored.

Symptom

Inbound delivery is created in ECC, but cannot be find in EWM. A stuck queue is found in SMQ2 with the following errors messages:

"Could not create header data delivery &1, system &2, Message No. /SCWM/ERPINTEGRATION012
&1 does not lie in a defined number range interval for this HU, Message No. /SCWM/HU_WM085
Error in the message handling unit level, Message No. /SCDL/ADAPTER_MP005".

Environment

  • SAP Extended Warehouse Management
  • SAP S/4HANA

Reproducing the Issue

  1. Go to tcode SMQ2
  2. Search for the Inbound Queue name
  3. Double click on the queue name to check the error

Cause

The HU number does not lie within any external number range defined for the corresponding warehouse number in EWM.

To address this, run tcode SNUM to display the Object /SCWM/HUID. Then, navigate to "Interval Editing" and display Intervals with the warehouse number.

Resolution

  1. Maintain the number range for HU identification in customizing with the following path: tcode SPRO -> IMG -> SCM Extended Warehouse Management -> Extended Warehouse Management -> Cross-Process Settings -> Handling Units -> External Identification -> Define Number Range for Handling Unit Identification.
  2. After updating the number range as required, execute the stuck queue and check if the delivery is successfully replicated to EWM.

Keywords

Inbound Delivery, SMQ2, /SCWM/HU_WM085, HU number range, /SCWM/ERPINTEGRATION012, /SCDL/ADAPTER_MP005, SNUM, object /SCWM/HUID

Symptom

  •  SAPUI5 application deployment to on-premise ABAP repository fails during pipeline execution when README.md files are present in the webapp folder
  • Repository rejection due to unsupported file type in deployment package
  • BAS Deploy log shows the following error 
     *** IGNORE *** : README.md (Not uploaded as binary/text type is unknown: Adjust content of files .UI5RepositoryTextFiles and .Ui5RepositoryBinaryFiles)

Environment

SAP Business Application Studio

Reproducing the Issue

  1. Launch the Business Application Studio
  2. Create the Fiori app and add Readme.md file in it 
  3. Deploy the file. 

Cause

On-premise ABAP repository only accepts specific deployable application artifacts and enforces strict file type validation during deployment process

Resolution

  1. Identify the README.md file and any other documentation currently stored within the webapp folder structure.

  2. Move the README.md file out of the webapp folder and into the project root directory (one level up).

  3. Verify that the webapp folder now contains only supported web assets (*.html, *.js, *.css, *.json, *.properties, *.xml, and images).

  4. Run the deployment pipeline again to ensure the changes are applied correctly.

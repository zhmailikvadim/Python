Symptom

Replication with both "connection set dynamic_sql"  and "connection autocorrection”

This may result in the following errors after table UPDATE command :

  ERROR #5216 DSI EXEC(103(1) MY_ASE_Site2.testdb) - eneric/dsi/dsiutil.c(546)
  The interface function 'RCIDescribeParam' returns FAIL for database 'MY_ASE_Site2.testdb'. See messages from the interface function for more information.

  ERROR #1027 DSI EXEC(103(1) MY_ASE_Site2.testdb) - eneric/dsi/dsiutil.c(546)
  Open Client Client-Library error: Error: 16843163, Severity 1 -- 'ct_describe(): user api layer: external error: This routine cannot be called when the command structure is idle.'.

Environment

SAP Replication Server (SRS) 16.0

Reproducing the Issue

SIMPLIFIED REPRO EXAMPLE

  1. create replication definition MY_TEST_TABLE ...
    with dynamic sql
    go
  2. alter replication definition MY_TEST_TABLE ...
    set autocorrection on
    for MY_TEST_TABLE_SUB
    with replicate at  MY_ASE_Site2.testdb
    go
  3. alter connection to MY_ASE_Site2.testdb
    set dynamic_sql to 'on'
    go
  4. UPDATE MY_TEST_TABLE ...
    go

Cause

The issue has been identified and logged under SAP CR 831818. Its description reads, "DSI was shutdown due to error 5216 with autocorrection and dynamic_sql 'on'"

Resolution

There is currently no target SRS Service Pack scheduled for this issue (SAP CR 831818). It is being assigned to our Product Group. Please mark this KBA as a Favorite to be automatically notified when the CR status is changed. 

See Also

KBA 2171560 about how to be notified of new or updated SAP Notes or KBAs in the ONE Support Launchpad.

Symptom

You're attempting to create a rule in the Personal Data Collection step to have the Country/Region field automatically populated based on the selected Company. However, You're unable to select the Country/Region field when configuring the rule.

Environment

  • SAP SuccessFactors Onboarding
  • SAP SuccessFactors Employee Central

Reproducing the Issue

  1. Log in to BizX.

  2. Navigate to Configure Business Rules.

  3. Create a new rule under Employee Central Core > Trigger onSave Rules for HRIS Elements.

  4. Build the rule based on Global Information.

  5. When attempting to set the value for the Country field, the Country/Region field is unavailable for selection because it is greyed out.

Cause

Expected behaviour.

Resolution

In Onboarding or Employee Central, it is not possible to set the Country/Region field using a business rule.

The field is populated based on hardcoded logic that defaults the value according to the country maintained on the Legal Entity.

To ensure the correct country is populated:

  1. Navigate to Manage Data > Legal Entity.

  2. Select any legal entity and verify that the Country is maintained.

  3. During hiring, when this legal entity is selected, the Country/Region field will be automatically populated.

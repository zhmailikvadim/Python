Symptom

In production orders with batch-managed components, the system allows the order to be set to Released status even though no batch has been assigned to one or more components.

Environment

SAP S/4HANA Cloud Public Edition

Reproducing the Issue

  1. Release a production order in the system.
  2. Proceed with batch determination for materials in the order.
  3. Observe that for certain materials, the system fails to assign a batch, leaving the remaining quantity unassigned.
  4. Verify that the 261 movement type is not executed, resulting in unrecorded consumption quantities.

Cause

This behavior is standard system logic.

Order release checks material-level availability based on the availability check configuration. It does not guarantee successful batch-level assignment.

Batch determination depends on:

  • Batch determination strategy.
  • Selection criteria and class characteristics.
  • Condition records.
  • Available stock at batch level.
  • Batch split configuration.

If no batch matches the strategy criteria, or the Checking Control configuration allows release without mandatory batch assignment the system will:

  • Release the production order.
  • Leave component quantities without batch assignment.
  • Not execute movement type 261 for unassigned quantities.

Material availability check and batch determination are technically independent processes.

Resolution

If the business requirement is to prevent release when batch assignment is missing, this can be controlled via configuration.

  1. Go to Configuration Activity/SSCUI: Define Checking Control (ID 102447).
  2. Select the relevant Order Type and Plant combination.
  3. In the section Batch Assignment, select: Value 3 – No release if batch assignment is missing.

Important:

If the “Batch Assignment” field is not available, the Checking Control was created without batch control enabled.

A new Checking Control entry must be created to activate batch assignment control.

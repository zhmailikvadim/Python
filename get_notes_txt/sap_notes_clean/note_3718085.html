Symptom

  • Sudden change in the database hardware key.
  • License expiration noted in HANA Studio

Environment

SAP HANA Database

Reproducing the Issue

  1. Logon screen went down.
  2. Check the systems from OS level; they are up and running fine.
  3. Open HANA Studio and observe that the license has expired.
  4. Consult the backup team; backups are failing due to license expiration.
  5. Check the hardware key and observe that it has changed.

Cause

  • The root cause of the issue is most likely a system identity change, such as a system rename, which can cause the hardware key to change unexpectedly.
  • The hardware key change leads to the expiration of the license.

Resolution

  1. Check for System Identity Changes:

    • Verify if any system identity change (such as a system rename) was performed recently. Hardware keys often change when the system’s identity is altered.

  2. Analyze Relevant Files:

    • Review the following files to investigate the cause of the hardware key change:

      • sldreg.xml: This file contains system registration details and can indicate if there were any changes to the system.

      • hdblcm trace files: These trace files can provide information about configuration changes that might have triggered the hardware key change.

  3. Resolve the License Issue:

    • After identifying the cause, work with SAP account team or support to refresh or reassign the license to match the new hardware key.

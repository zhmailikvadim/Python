Symptom

Transport data from Business Partner role “CRM010” (Carrier) is not replicated to the target system when using DRF/standard outbound implementation.

Environment

  • SAP S/4HANA

Reproducing the Issue

  1. In the source system, maintain Transport Data for a Business Partner with role CRM010 (Carrier).
  2. Start DRFOUT and select a replication model that uses outbound implementation 986_3 for Business Partner replication.
  3. Execute the replication for the Business Partner.
  4. Review SRT_MONI message and payload transport-specific fields (for example, service level code) are not present in payload.

Cause

  • The standard BP replication does not support carrier-/transportation-specific attributes from the Transport Data tab (for example, service level codes). These fields are not part of the supported data set and therefore are not included in the outbound payload.

Resolution

  • Based on the current design, replication of the requested carrier-/transportation-specific attributes is not supported.
  • This is outlined in the documentation provided in the application help section, which you can access here,
  • [https://help.sap.com/docs/SAP_S4HANA_ON-PREMISE/44e06f22436c43e582db6ccd5250e29b/f4edce62151b421bbb45d4c92a98b9fc.html?locale=en-US&version=2023.001]
  • Additionally, SAP Note 2221398 clarifies that any fields not explicitly mentioned in the replication files are not supported. The note states: "Any field that is not mentioned at all in any of the files is not supported."

See Also

  • Refer to: SAP Note 2221398 - (Un-)Supported Fields in Data Model

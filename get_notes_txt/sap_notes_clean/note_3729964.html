Symptom

One of the following may be observed and a tool is needed for troubleshooting this:

  • Changes made via BAPIs (i.e. BAPI_SALESORDER_CHANGE or BAPI_SALESORDER_CREATEFROMDAT2) are not updated in the configuration or an inconsistency occurs (CFG_IS_INCONSISTENT_OR_INCOMPLETE, message no. CU013)
  • Components or items are not selected during BOM explosion
  • Unexpected or incorrect results in CU50



 

Environment

Variant Configuration

Cause

Resolution

To help to understand the system behavior dependencies can be traced with the help of transaction CUTRACE. This tool records the configuration logic at runtime and logs the sequence and outcome of object dependencies like preconditions, selection conditions, procedures, constraints.

Key benefits:

  • Identifies why a value was set or not set and which dependency caused it.
  • Reveals why components or items were selected/excluded (or not found).
  • This may clarify system behavior and serve as a starting point for troubleshooting. 

Steps:

  1. Start transaction CUTRACE.
  2. In Settings, enable the 'More Detailed' option.
  3. Select the relevant trace areas, optionally set a filter for a specific dependency.
  4. Turn the trace on.
  5. In a parallel session, reproduce the issue (e.g. run CU50, perform BOM explosion, or execute the relevant BAPI scenario).
  6. Turn the trace off.
  7. Review the results in 'Trace List'

Symptom

  • It is not possible to add aggregation annotations in a custom entity CDS (RAP) view to enable charts with visual filters in a Fiori app.
  • When an aggregation annotation (for example @Aggregation.default) is added to an element, the Service Binding activation fails with the error "Aggregate functions are not allowed in transactional views".

Environment

SAP S/4HANA release independent

Reproducing the Issue

  1. Create or use a RAP service based on a custom entity CDS view that is transactional (I-/C-layer with behavior definition).
  2. Add aggregation annotations (for example @Aggregation.default) to elements intended for charting.
  3. Activate or preview the Service Binding and observe the error "Aggregate functions are not allowed in transactional views".

Cause

Transactional enabled entities (I- and C-layer) cannot contain aggregation because aggregated elements have no persisted representation during create/update/delete; aggregation (read) and transactional enablement are mutually exclusive.

Resolution

  1. Remove all aggregation annotations from transactional CDS views (I-/C-layer) that support create, update, or delete operations.
  2. Create a separate non-transactional analytical CDS view for reporting and charting, and add the required aggregation annotations (for example @Aggregation.default) there.
  3. Expose the analytical (non-transactional) CDS view via a separate OData entity and bind it to the list/chart or analytical sections of the app.
  4. Use the transactional (non-analytical) CDS view exclusively for create/update/delete operations, for example in detail pages.
  5. Verify that the analytical OData V2 entity generated for the aggregated view uses a generated ID key as documented, and adjust UI bindings accordingly.

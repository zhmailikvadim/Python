Symptom

Developers can see and select cloud foundry orgs/spaces from all subaccounts, not just the subaccount where the BAS subscription is done when targeting Cloud Foundry when deploying applications

Environment

  • Cloud Foundry Environment
  • SAP Business Application Studio

Reproducing the Issue

  1. In sap business application studio, start “Deploy to Cloud Foundry” for an HTML5 application from a project in a DEV subaccount.
  2. Log in to Cloud Foundry (via SSO or CF login).
  3. In the target selection dialog, observe that all CF orgs/spaces where the user has roles are listed, including all spaces, and can be selected for deployment.

Cause

Space visibility during deployment is determined by the Cloud Foundry user’s assigned org/space roles; the tool lists all orgs/spaces granted to the logged-in CF identity and there is no feature to restrict visibility by subaccount.

Resolution

Remove the user’s Cloud Foundry authorizations in the spaces they should not be able to deploy to (for example, Space Developer/Manager or equivalent roles). The available subaccount, org and space is bound with the user who logged into CF either directly via CF CLI or via BAS. This means the only way to restrict the access to is remove the users authorizations from the orgs/spaces.

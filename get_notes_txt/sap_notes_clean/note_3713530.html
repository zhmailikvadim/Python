Symptom

You perform an upgrade from SAP ERP EHP6 or higher to a SAP S/4HANA 2025 or higher. In phase ACT_UPG you get error messages such as:

  • Activation of table type "/1LT/SHDW_T0000000000000000213"
  • Row type "/1LT/SHDW_S0000000000000000213" is not active or does not exist
  • Table type "/1LT/SHDW_T0000000000000000213" was not activated

Note that the numbers may vary, but the name of the row types start with "/1LT/SHDW_S".

With release SAP S/4HANA 2025 or higher, it is no longer possible to ignore any errors during phase ACT_UPG.

Other Terms

DMIS, MWB

Reason and Prerequisites

MWB / DMIS functionality is used on one of the following releases:

  • DMIS 2011 SP25 - SP28
  • DMIS 2018 SP11 - SP14
  • DMIS 2020 SP07 - SP10
  • DMIS 2025 SP00
  • SAP S/4HANA 2017 FPS00 - SPS17
  • SAP S/4HANA 2018 FPS00 - SPS15
  • SAP S/4HANA 2019 FPS00 - SPS13
  • SAP S/4HANA 2020 FPS00 - SPS11
  • SAP S/4HANA 2021 FPS00 - SPS09
  • SAP S/4HANA 2022 FPS00 - SPS07
  • SAP S/4HANA 2023 FPS00 - SPS05
  • SAP S/4HANA 2025 FPS00 - FPS01

Solution

Implement the attached correction using transaction SNOTE.

Execute the report DMC_CLEANUP_SHADOW_DDIC_INC.

Note that you must perform this step in your source system before the upgrade. If you perform this step afterwards, you may have to restart part of the upgrade process.
If you continue to encounter issues with objects in the /1LT/ namespace during the upgrade process after running the report, please contact SAP for support using component BC-UPG-RDM.

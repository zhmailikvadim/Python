Symptom

Error occurs in SRT_MONI code, when loading instruction to YL (Yard Logistics) is triggered:

No mapping for ext. ref. doc. &1 to &2 item ref. doc. found
/SAPYL/YARD_ORDER550

(If the error message contains number 73, it means Outbound Delivery.)

Environment

  • SAP Yard Logistics
  • SAP Yard Logistics for S/4HANA

Reproducing the Issue

  1. FO (Freight Order) is created in TM (Transportation Management) system.
  2. Planning steps are finished. The XML message is created.
  3. The XML is stuck with the mentioned error message.

Cause

Missing customization

Resolution

  1. Go to SPRO -> Yard Logistics ->Integration ->Integration to SAP Transportation Management ->Integration to SAP TM specific for LDAP ->Map External Reference Documents to Yard Request Item Reference Documents 
  2. Maintain the entry for the mapping for the objects mentioned in the error message.

Keywords

024(BS_SOA_COMMON), 550(/SAPYL/YARD_ORDER), http://sap.com/xi/SAPGlobal20/Global, proxy, Error in the customer Inbound BADI implementation: An exception was raised, /SAPYL/YARD_ORDER 550

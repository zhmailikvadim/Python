Symptom

You are using HANA System Replication in operation mode delta_datashipping and observe that the delta data shipment frequency decreases.

Additionally, in a trace of a service with a big persistence, like indexserver, you notice a long time gap between the following trace lines:
[0000]{-1}[-1/-1] 0000-00-00 00:00:00.000000 i PersistenceManag PersistentSpaceImpl.cpp : loadContainerDirectories - done.
[0000]{-1}[-1/-1] 0000-00-00 00:00:00.000000 i Logger LoggerImpl.cpp : primaryBackupFinished(0x00000000000)

Other Terms

HSR

Reason and Prerequisites

Reason:

Due to a unfavorable implementation of page checks after a delta data shipment arrived on the secondary site, shipped pages were unnecessarily read from disk.

Affected Releases:

  • SAP HANA 1.0 database:
    • Revisions of SPS07 - SPS11
    • Revisions <= 122.09 (SPS12)

  • SAP HANA 2.0 database:
    • Revisions <= 002.01 (SPS00)
    • Revisions <= 011.00 (SPS00)

Prerequisites:

  • You are using SAP HANA System Replication with operation mode delta_datashipping.

Solution

Apply one of the following SAP HANA database Revisions:

  • SAP HANA 1.0 database:
    • Revisions >= 122.10 (SPS12)
    • or higher
  • SAP HANA 2.0 database:
    • Revisions >= 002.02 (SPS00)
    • Revisions >= 012.00 (SPS01) 
    • or higher

  • or higher

and

set the following internal configuration parameter on all sites and restart the secondary site afterwards:

global.ini
[system_replication]
enable_cache_pages_for_preload_version_check = true

Workaround:

  • If possible, you can consider to switch your operation mode to logreplay. Please be informed, that on older Revisions you might encounter a hanging logreplay on secondary site. For details please refer to SAP Note 2425682.

Remarks:

  • Enabling the page cache for the preload version check on secondary site using the previously described configuration parameter can yield to a higher memory consumption on secondary site.
  • The issue described in this SAP Note does not affect the data consistency of the secondary site.

Disclaimer

The SAP HANA configuration parameter [system_replication] enable_cache_pages_for_preload_version_check is an internal parameter and may only be used in the context of this SAP Note. Using such a parameter requires that the SAP HANA system has the revision specified in the SAP Note, the described error situation has occurred. Using internal parameters outside of this given scope is not supported.

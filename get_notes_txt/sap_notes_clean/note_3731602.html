Symptom

During the parallel program "Renewal of a Portfolio", external locks occur at life level in certain constellations. This occurs in particular if several policies belong to the same life and several of these policies belong at the same time in the above program are to be processed and processed in different packages of parallel processing. In these cases, the system issues the message "Life <LIFE NUMBER> is being processed by user <USER ID> since <DATE TIME>".

Other Terms

/MSG/VR_BO_INTERFACE-->003

Solution

History:

March 26, 2026: New　

Troubleshooting Description:

The system ensures that life data is loaded during the "Renewal of a Portfolio" program in "Read/Change" mode. Life data is not saved during the program "Renewal of a Portfolio" and therefore an exclusive lock as in the previous mode "Read/Change/Save" is not required.

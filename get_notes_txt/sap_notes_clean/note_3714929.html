Symptom

When trying to login SAP application via IAS and MS Entra ID, unexpected email address shown in Logon screen of MS Entra ID.

Environment

Identity Authentication

Reproducing the Issue

Login SAP application via IAS and MS Entra ID

Cause

The login hint from IAS is not handled by MS Entra ID correctly and mapped to unexpected email address.

Resolution

To avoid this issue, there are following options. 

  • Solution 1:
    Change the Login Hint configuration in the configuration of corporate IdP in IAS to "User Attributes=>Email" instead of the default configuration "User Input" and make sure the Login Hint is always email address. 
    • This solution requires the user present in IAS.

  • Solution 2:
    Disable the Login Hint configuration in the configuration of corporate IdP in IAS. For the details of the configuration, please check the document Configure Login Hint Parameter.
    • This solution requires the user to enter their email address on the IAS login screen and then enter it again after being redirected to corporate IdP.

  • Solution 3:
    Adjust the corporate IdP configuration and force to input user name every time.
    1. Create system as administrator:  Add System as Administrator 
    2. Execute PATCH https://<IAS tenant>/IdentityProviders/v1/<Corporate IDP ID>

      with body:

      {
                      "operations": [
                                      {
                                                      "op": "replace",
                                                      "path": "/automaticRedirect",
                                                      "value": false
                                      }
                      ]
      }

      • Identity Provider ID - can be get from Admin Console

                   *For more details about the API for the corporate IdP configuration, please check the document SAP Business Accelerator Hub 

Keywords

IAS, Entra ID, Azure AD, unexpected email address.

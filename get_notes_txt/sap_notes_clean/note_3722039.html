Symptom

  • Users are able to logon to application through IAS but they do not have permissions to run their job
  • Users are assigned to Application groups in IAS tenant but you cannot find these groups inside the jwtPayload in Troubleshooting Log files or in SAML attributes sent to applications in a SAML trace

Environment

SAP Cloud Identity Services

Cause

Application specific Groups are filtered out with latest update.

Resolution

When using application groups, it is required to set in the IAS Application attributes,  groups as Application Groups

  1. Sign in to the administration console for SAP Cloud Identity Services.
  2. Under Applications and Resources, choose the Applications tile.
  3. Choose the application
  4. Choose the Trust tab.
  5. Under SINGLE SIGN-ON, choose Attributes
  6. Set groups as follows

When the application is inhered and you cannot add another "groups" attribute as self-defined attributes, you could create it in the parent application.

Also, check if the required groups are related to the application which needs to send the groups. If they are not related:

1.    Assign the groups to the application ("All Groups" will work)
2.    Groups belong to parent application. Assign the application to parent application. Add mapping "groups" -> "Application Groups" at parent application
3.    Remove the application from the required group(s)

As reference: Configuring User Attributes from the Identity Directory | SAP Help Portal

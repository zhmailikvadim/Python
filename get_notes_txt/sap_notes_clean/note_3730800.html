Symptom

  • When calling the SAP BTP OAuth token endpoint " https://<subdomain>.authentication.<region>.hana.ondemand.com/oauth/token" from a third-party system. The request fails with an error: PKIX path building failed: sun.security.provider.certpath.SunCertPathBuilderException:unable to find valid certification path to requested target
  • This issue occurs during OAuth2 token generation using the client credentials grant.
  • The same endpoint works successfully when tested using tools such as Postman, but fails when invoked from the third-party system.

Environment

  • SAP Business Technology Platform

Cause

The third-party system does not trust the SSL certificate chain of the SAP BTP authentication endpoint due to a missing or outdated root/intermediate certificate in its trust store. As a result, the TLS handshake fails and the request is terminated before OAuth processing begins.

Resolution

  1. The issue is caused by missing TLS trust on the third-party system side and is not related to OAuth or BTP Configuration.
  2. Ensure that the third-party system trusts the SSL certificate of the BTP authentication endpoint by importing the required root certificate into its trust store (JVM or OS level): DigiCert TLS RSA4096 Root G5
  3. Existing root certificates (such as DigiCert Global Root G2 or G3) do not need to be removed. If already present; the new root can be added alongside the existing trusted certificates.

For updates on root certificate changes in SAP BTP (Cloud Foundry), refer to KBA 3566727- Root Certificate Replacement in the SAP BTP, Cloud Foundry Environment

Symptom

During valuation posting (RECEEP) of contracts containing high object volumes (e.g., ~50 objects), the system experiences timeouts or performance degradation.

Other Terms

RECN, performance, Dump, response time, currency translation, TRANSLATE_TO_LOCAL_CURRENCY

Reason and Prerequisites

You use Asset Accounting integration with depreciation area in parallel currency.
Large contracts with many objects exceed the optimal processing capacity of RECEEP.
In particular, the currency conversion process of valuation cash flows requires multiple exchange rate retrievals and calculations, adding computational overhead to valuation processing.

Solution

 
Split large contracts into several smaller contracts of ~20 objects in order to reduce the object volume per valuation run.

Follow guidance of  915080 - Consulting note: Contract dialog (performance).

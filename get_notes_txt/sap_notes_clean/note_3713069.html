Symptom

You create new subitem for carrying forward the equity FS items, retained earnings and net income. 
The subitem you create is with for example prefix 6**. 

On running the balance carryforward, you notice that the system has carryforward the amount on the subitem with prefix 9** as set in SAP S/4HANA Cloud system standard and not on your created subitem 6**. 

You expect SAP to change the subitem derivation for the retained earnings in the activity Specify FS Items to be Carried Forward so that subitem 6** is automatically selected instead of 9**.

Environment

SAP S/4HANA Cloud

Reproducing the Issue

  1. In the Data Monitor app, run balance carryforward task.
  2. In Group Data Analysis app, look for FS Item Retained Earnings and see that subitem 900 has been the one derived.  

Cause

For S/4HANA Cloud system, the activity Specify FS Items to be Carried Forward is not available for users nor for SAP to make any change in subitems derivation.

Resolution

FS Items with certain item roles are automatically reclassified to the FS Item assigned to the role S‑RETAINED‑EARNING, and subitem 900 is set during this process. This is the designed system behavior of the activity Specify FS Items to be Carried Forward.

It is not possible for S/4Hana Cloud users nor for SAP to make any change in Specify FS Items to be Carried Forward activity.

Symptom

  • Errors occur when attempting to add objects to the central repository
  • The issue is not limited to the specific objects that are being added to central repository
  • Error messages: "Version control operation 'Add to Central Repository' failed. Repository <> does not contain object <> for type<>(BODI-1260123)"
  • Error messages: "Version control operation 'Add to Central Repository' failed. Sybase SQL Server <> error message for operation<ct_results>: <Sybase Server message number <12104> State<1>Severity<16> From Server<>:Message Text is: A LOCK TABLE command is not allowed outside the scope of a transaction on table 'AL_PROJECTS' in the database.>(BODI-1260123)"

Environment

  • SAP Data Services
  • Sybase

Reproducing the Issue

  1. Login to designer.
  2. Confirm the central repository is active.
  3. In the local object library, locate an object.
  4. Right-click on the object and select the option "Add to Central Repository" with "Object and dependents".
  5. Observe the error message "repository does not contain object ".
  6. Retry the same step and observe the error message "Sybase SQL Server <> error message for operation<ct_results>".

Resolution

  1. Create a new central repository using repository manager tool
  2. Register the newly created central repository in CMC
  3. In designer, test adding objects to the newly created central repository

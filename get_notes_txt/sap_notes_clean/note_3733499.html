Symptom

  • An error occurs when attempting to import a table from an S/4HANA system to a DS datastore.
  • The issue appears after the target S/4HANA system was upgraded to the 2025 release.
  • Cannot retrieve table metadata - Table does not exist: RFC CallReceive error <Function /SAPDS/TABLE_IMPORT: RFC_ABAP_MESSAGE, Incompatible Call Rejected, see note 2295840; Called Incompatible Function :/SAPDS/TABLE_IMPORTCPROG:)- Incompatible Call Rejected, see note 2295840; Called Incompatible Function :/SAPDS/TABLE_IMPORTCPROG:al_engineDEST:<appserver>[SAP NWRFC <version>][SAP Partner <basis version>][<SID>][<FQDN app server>][<username>][4103]>.

Environment

  • SAP Data Services 4.3
  • SAP Cloud Integration for data services

Reproducing the Issue

  1. Start Data Services Designer.
  2. Open the S/4HANA datastore.
  3. Attempt to import the metadata for any table from the S/4 system.

Cause

  • Compatibility issue: Data Services 4.3 is not certified with S/4HANA 2025 (yet). KBA 3222345 - SAP Data Services 4.3 and SAP Information Steward 4.3 compatibility with SAP S/4HANA
  • The remote-enabled function module /SAPDS/TABLE_IMPORT is on the blocklist (see RS_RFC_BLACKLIST_CUSTOM and UCON checks), and /SAPDS/TABLE_IMPORTCPROG does not exist in the system SE37. The system calls the non-existing FM from the server side that causes the problem.

Resolution

  1. Implement the correction of the Support Packages and Patches section of the SAP Note: 3391325 - Data Services: added size category for table import
  2. Run the program RS_RFC_BLACKLIST_CUSTOM from SE38.
  3. Unblock the function module that is responsible for the import to the DS side: /SAPDS/TABLE_IMPORT
  4. The user that runs the program RS_RFC_BLACKLIST_CUSTOM should have the following authority
    AUTHORITY OBJECT : S_UCON_ADM
    ID: UCON_TYPE : FIELD : BLRC
    ID: UCON_NAME : DUMMY
    ID: ACTVT : FIELD : 01, 02, 03, 06
    Provide the above mentioned authority to the user to overcome the error message  "custom entry cannot be read"
  5. Try to import the table again without facing any issues.

KERNEL Patch is not needed!

See Also

  • SAP Note 3391325 - Data Services: added size category for table import

Symptom

  • The user has admin access in IAS tenant and also the role "Manage Identity Provisioning" assigned to him and the error message appears continuously in the browser when trying to login to IPS tenant.
  • The IPS tenant is in the SAP Cloud Identity infrastructure.

Environment

  • Identity Authentication
  • Identity Provisioning

Reproducing the Issue

  1. Log in to the IPS tenant in the SAP Cloud Identity Infrastructure.
  2. Error message appears.

Cause

IAS admin console has corporate IDP setup. 

Resolution

Follow either of the following resolutions:

  • Turn off the corporate IDP for the Identity Authentication Admin Console application.
  • Configure Identity Federation in your Corporate IDP to allow users from IAS to log in to IPS as mentioned in Configure Identity Federation

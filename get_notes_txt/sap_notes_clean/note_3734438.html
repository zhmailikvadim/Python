Symptom

When replicating Installed Base data from an external SAP CRM system to SAP Cloud for Customer (C4C) using the InstalledBaseMassReplicationIn service, some messages fail with the error: “Enter a valid customer name.”

Environment

SAP Cloud for Customer

Cause

When the error does not occur, the External ID matches the Internal ID, allowing the system to successfully identify the customer.

When the error occurs, the External ID differs from the Internal ID. In these cases, the system attempts to retrieve the Internal ID using the External ID, which results in the error message.

Resolution

To prevent this issue, the C4C Internal ID must be included in the incoming Installed Base payload. Specifically, the <PartyID> element must contain the Internal ID maintained in C4C, not the external CRM ID.

Please note that ID mapping is not supported for Installed Base replication, therefore, the system requires the C4C Internal ID to be provided directly in the payload.

This behavior differs from other objects, such as Sales Orders and Quotes, where ID mapping is available.

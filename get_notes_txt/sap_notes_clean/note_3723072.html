Symptom

When navigating through a multi-step wizard, users may encounter redundant or incorrect data in Step2.

Specifically, if a user navigates back to Step 1 (from a later step or the Summary screen) to change the Process or Category , the UI incorrectly appends the new details to the existing data from the previousselection. This results in an accumulated list of details rather than a refreshed view relevant to thecurrent selection.

Other Terms

 Adjustment hub , MP single , Manual Adjustment

Reason and Prerequisites

The issue is caused by a missing refresh logic in the client-side state management. The UI controllerwas configured to append new data to the internal model/buffer of the second step whenever a changeoccurred in the first step, rather than overwriting or clearing the existing state.

Solution

A logic correction has been implemented in the client-side controller.
Validation:
Added conditional checks to ensure only data mapped to the currently selected Process and Category is processed for the second step.
Data Handling:
The UI logic now explicitly clears or overwrites the buffer for Step 2 before populatingit with new details. This ensures the view is always synchronized with the user's most recent selectionin Step 1.

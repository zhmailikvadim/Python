Symptom

Important Points To Be Considered While Integrating SAP Sales Cloud Version 2 and SAP CPQ

Environment

SAP Sales Cloud Version 2
SAP Sales Cloud Version 2 Integration with SAP CPQ

Resolution

Consider these important points while integrating SAP Sales Cloud V2 with SAP CPQ

  1. CPQ Integration is delivered to be used in conjunction with S4HANA Integration.
  2. Initial Load of Master Data from S4HANA to Version 2 to be completed or You atleast update the External IDs of all master data used in opportunities with S4 System ID reference in External ID Mapping. 
  3. This means that there is no direct ID mapping between SAP CPQ and SAP Sales Cloud Version 2 but only indirectly via SAP S/4HANA. Therefore, the IDs from SAP S/4HANA are used in the integration between SAP CPQ and SAP Sales Cloud Version 2.
  4. If any external ids are not retrieved during transfer , the error  "Outbound Synchronous request mapping failed. Check Integration Message Monitor for more details." is triggered
  5. In Sales Cloud Version 2, Opportunity to External Sales Quote communication configuration should be unique(External Quote creation in CPQ + S4HANA is not allowed). You get error "Missing unique configuration for external quote communication." if you try to create quote with multiple configurations.

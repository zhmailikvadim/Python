Symptom

  • Manual move-out reversal using the scenario generator results in exceptions that corrupt scenario data, making it unusable.
  • When the simulate flag is used during scenario generator initialization, data checks are insufficient and the mode is ineffective.

Environment

SAP Market Communication for Utilities

Reproducing the Issue

  1. Launch the scenario generator to perform a manual move-out reversal.
  2. Initialize the run with the simulate flag enabled.
  3. An exception occurs during processing and the resulting scenario data becomes corrupted/unusable.

Cause

The scenario generator writes directly to persistent data, has no simulate-and-rollback mode, and does not use ABAP rollback; the simulate flag performs minimal checks, so exceptions can leave partially written, corrupted scenario data

Resolution

  1. Note that a simulate-and-rollback mode for the scenario generator is not provided.
  2. Use the supported Billing Simulation framework, which runs only on temporary runtime data and makes no database changes.
  3. Access Billing Simulation via: Utilities → Contract Billing → Billing Simulation.
  4. Alternatively, configure your own simulation scenarios via: SPRO → Utilities → Contract Billing → Billing Execution → Simulation Scenarios.
  5. Do not rely on ABAP rollback mechanisms (for example, ROLLBACK WORK) to safeguard scenario generator processing.

See Also

  • Community discussion: ROLLBACK WORK and simulation considerations

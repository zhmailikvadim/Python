Symptom

  • The Process Graph app does not display any data at all or does not display recent data in the SAP BW/4HANA cockpit.
  • JSON response of the call /sap/bw4/v1/monitoring/processNetwork?from=...&to=.... delivers empty result.
  • Tables/CDS Views below either does not return any records or they just have very old records:
    • RSSTAT
    • RSSTAT_REF
    • RSSTAT_REF_IPRO
  • Table RSPCRECENTRUN does not have the fields below filled (for all records it is 0):
    • AVG_CPU
    • AVG_DB
    • AVG_ABAP_MEM
    • AVG_DB_MEM

Environment

  • SAP BW/4HANA

Reproducing the Issue

  1. Open the SAP BW/4HANA Cockpit
  2. Navigate to the Process Graph app under Monitoring
  3. Wait until it loads and/or extend the Date range to be checked
  4. The expected result is something like this:

Cause

  • RSPC_WATCHDOG is not active.
    This can be checked in transaction ODQ_TQ_MONITOR by searching for "External ID" RSPC_RECENT_RUN_CACHE:

    • If the status is "empty" or the "Last Execution" is too old or this item is not even there in ODQ_TQ_MONITOR, then this is an indication, that the RSPC_WATCHDOG is not running.
  • One (or more) of the Daemons below in transaction SMDAEMON is not existing:
    • S_GSM_CO_APPLICATION_DAEMON
    • S_GSM_CO_INSTANCE_DAEMON
    • S_GSM_CO_SYSTEM_DAEMON
    • S_KSR_CO_INSTANCE_DAEMON
    • S_SMN_CO_INSTANCE_DAEMON
    • CL_ODQ_QUEUED_TASK_DAEMON

Resolution

  1. Th S_GSM_*, S_KSR_* and S_SMN_* Daemons are created when the Activating the Data Collection for Technical Monitoring Apps is performed. Ensure that this process has been followed.
  2. Go to transaction SMDAEMON
    1. Menu GOTO --> System wide list
    2. Menu --> ABAP Daemon --> Check daemon table

      If the following pop-up returns, you may remove inactive entries:
      "Daemon DB table entries for inactive AS instance [...]"
  3. If the RSPC_WATCHDOG is not running:
    1. Open SE24 and enter the class CL_RSPC_WATCHDOG and push F8
    2. Execute the method SCHEDULE
  4. If in SMDAEMON the Daemon CL_ODQ_QUEUED_TASK_DAEMON is missing:
    1. Run the program ODQ_TQ_RESUME_TASK_QUEUE. After a short time, the daemon should be created and the data collection should begin.
  5. After some time the corresponding fields of RSPCRECENTRUN begin to be filled.

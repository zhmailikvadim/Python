Symptom

  • The report generated using DRC Argentina AR_VAT_PRINT for VAT Purchases does not contain any data at field "Authorization Code" from the PDF file generated.

Environment

  • SAP S/4HANA
  • SAP S/4HANA Cloud Private Edition
  • DRC Argentina
  • Statutory Reporting

Reproducing the Issue

  1. Run the "Run Statutory Reporting" application.
  2. Run AR_VAT_PRINT print
  3. Choose "VAT Purchase" and download the report.
  4. Notice the missing CAE information at field "Authorization Code" 

Cause

The "Authorization Code" number from VAT Purchase report is fetched from table J_1APACK1 at CDS view I_AR_SupplierCAI. This table is filled using transaction J1APAC01 "Create/Change Print Authorization Code".

VAT Purchase report does not have the same behavior from VAT Sales report. VAT Sales consumes data for this field from EDOARINV (eDocuments) or J_1ACAE (Argentina Electronic Invoice) at CDS view P_AR_VATPrintSalesDetail.

Resolution

  1. Run transaction J1APAC01.
  2. Inform the Supplier from the document that is missing the CAE number.
  3. Fill the information required.

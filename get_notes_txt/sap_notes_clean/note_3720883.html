Symptom

  • Your system's SPAM/SAINT version is on version 93
  • You are running SUM (Software Update Manager), and it fails at EHP_INCLUSION phase.
  • The phase fails during the queue calculation for UIBAS001 758:
    • 3 ETN262    Calculating queue part for add-on "UIBAS001" rel. "758"                                                        
      3 ETN490 Use the Add-On installation package "SAPK-758AGINUIBAS001" (type &4"AOI") for Add-On &2"UIBAS001" rel.&3"758"      
      3 ETN245 The following import prerequisites of OCS Package "SAPK-758AGINUIBAS001" have not been met:                        
      3 ETN264   In the (alternatively) Requirement set "01":                                                                    
      3WETN492 Required Add-On component "UIX01CA1" rel."200" is not installed                                                    
      3 ETN264   In the (alternatively) Requirement set "02":                                                                    
      3WETN493 Not allowed software component "UIX01CA1" rel."*" is installed                                                     
      2WETN543 OCS package "SAPK-758AGINUIBAS001" does not match the current software component vector   
  • The error can occur for other packages as well if the add-on failing pre-requisite check was retrofitted to another add-on in a previous upgrade.

Resolution

SPAM/SAINT version 93 unintentionally introduced a regression while addressing a separate queue calculation issue.

A fix is being introduced in SPAM/SAINT version 94. Please monitor for its release here:

  • https://me.sap.com/softwarecenter/template/products/_APP=00200682500000001943&_EVENT=DISPHIER&HEADER=Y&FUNCTIONBAR=N&EVENT=TREE&TMPL=01200615320100001425&V=MAINT&TA=ACTUAL&PAGE=SEARCH

In the meantime, please wait for SPAM/SAINT version 94 to be released around early March. If an urgent solution is required, please create a case with BC-UPG-OCS.

Symptom

We have maintained the DRFF Filter in MDG to a third-party system. The requirement is to replicate only customer account group which is included and other account groups are excluded in DRFF, but on create/change the system still generates IDocs to the target system for those excluded account groups (OILDEB & OILCRE).

Environment

  • SAP S/4HANA Cloud Private Edition 2023

Cause

MDG Data Replication Framework (DRF/DRFF) does not support the IS-OIL-specific IDoc message types OILDEB and OILCRE. DRFF filters apply to supported standard IDocs (DEBMAS/CREMAS) only, so DRFF include/exclude conditions do not control OILDEB/OILCRE generation.

Resolution

  1. Please do not rely on DRFF filters for OILDEB/OILCRE; these message types are not supported by MDG DRF filtering.
  2. If account-group-based DRF filtering is required within MDG, switch replication to the supported standard IDoc types DEBMAS (customer) and CREMAS (supplier) as per the official configuration guidance.
  3. If OILDEB/OILCRE must remain in use, implement filtering in the ALE/IDoc distribution layer instead of DRFF, for example:
  • Use the Distribution Model (BD64) to control which message types are sent to which logical systems and to apply object-level filters where applicable.
  • Maintain Partner Profile outbound parameters (WE20) to control outbound processing and receiver determination.
  • Add custom outbound enhancement logic to suppress IDoc creation based on criteria such as account group (for example, allow only ).
  • Test and confirm that only the intended account groups generate outbound IDocs after applying the chosen approach.

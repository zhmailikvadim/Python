Symptom

When running MRP to create schedule lines, the quantity in the schedule line (order unit) is incorrect while the base unit quantity in MD04 is correct.

Environment

  • SAP S/4HANA Cloud Public Edition

Reproducing the Issue

  1. In the material master, maintain an alternative unit of measure with a conversion factor (for example, base unit M2 and alternative unit KG with a defined conversion).
  2. In the scheduling agreement, set the order unit to the alternative unit (for example, KG).
  3. Run MRP to create schedule lines, then compare:
  • MD04 shows correct quantities in the base unit.
  • The schedule line quantity in the order unit is not calculated according to the conversion factor maintained in the material master.

Cause

  • The alternative unit-of-measure conversion in the material master was changed after the scheduling agreement item was created.
  • The scheduling agreement item retained the old conversion values (EKPO-UMREZ/UMREN), and MRP reads the conversion factor from the scheduling agreement item master data rather than the updated material master.
  • As a result, schedule line quantities were calculated using the outdated conversion factor.
  • Changes to the material’s unit-of-measure conversion do not update existing scheduling agreement items automatically.

Resolution

  1. Verify the current unit-of-measure conversion in the material master.
  2. Check the existing scheduling agreement item and note its stored conversion values.
  3. Create the scheduling agreement item again (or create a new item) so that it pulls the latest unit-of-measure conversion from the material master.
  4. Deactivate or close the old item to prevent further planning against outdated conversion data.
  5. Run MRP again to generate schedule lines.
  6. Validate that the schedule line quantity in the order unit now aligns with the material master conversion and matches the base unit quantity shown in MD04.

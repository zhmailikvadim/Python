Symptom

  • During the settlement process in SAP S/4HANA, the "Currencies/ Ldgs" indicator in the settlement rule is deleted after running transaction KO88.
  • This issue occurs when the "C - Freely defined currencies" option is used for the "Currencies/ Ldgs" indicator in the settlement profile after the implementation of SAP Note 3225180.

Environment

  • SAP S/4HANA

  • SAP S/4HANA Finance

Reproducing the Issue

  1. In the settlement profile, set the "Currencies/ Ldgs" indicator to "C - Freely defined currencies".
  2. Create a new order with settlement rule (COBRA-ADDCURR = 'C').
  3. Run the settlement using transaction KO88.

Cause

The modification of SAP Note 3225180 can not be used (does not work) with periodic production orders = production orders with settlement rule tape PER and Results Analysis key.  The problem is that WIP calculation is not supported for additional currencies. But in periodic Controlling (WIP at target costs), the settled costs to material must be reduced by the WIP values for each currency. This value flow does therefore not support additional currencies. 

Resolution

Periodic production Controlling ( = settlement rule type PER in combination with RA key for WIP at target costs) is not possible for the additional currencies because target WIP calculation is not supported for additional currencies.

  • Process orders could also be configured without RA key (no WIP calculation)

or

  • with FUL rules and WIP at actual costs. 

It depends on user's business scenarios and requirements if this is possible.

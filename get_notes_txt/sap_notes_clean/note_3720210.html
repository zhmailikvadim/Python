Symptom

You use the Cross Reference UI profile and access the search help (F4) on the Customer field, expecting the value help to function normally. On the F4 help screen, the Customer field itself also has F4 enabled. When you click on this field, it triggers the same search help again, causing a recursive loop. However, this issue does not occur in the Claim UI profile for the same search help.

This note corrects the behavior by preventing the recursive triggering of the search help in the Cross Reference UI profile.

Other Terms

F4, Search Help, XREF

Reason and Prerequisites

 This is caused by Program Error.

Solution

Please implement the correction instructions. These corrections are only valid if you have applied the following Support Packs.

Release V/4 – 2023V2, 2023 V3, and 2023 V3 Support Package 1.

Use transaction SNOTE to implement the correction instructions specified below and activate the affected program object

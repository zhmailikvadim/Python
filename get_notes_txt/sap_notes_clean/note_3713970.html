Symptom

  • Users can't sync after a 'UnifiedPerson' entry is created via an API request.
  • A synchronization error appears on mobile.

Environment

SAP Field Service Management

Reproducing the Issue

  1. Create UnifiedPerson via Data API
    POST - https://{{cluster}}.fsm.cloud.sap/api/data/v4/UnifiedPerson?account={{accountName}}&clientIdentifier=COR_CON_NONE&dtos=UnifiedPerson.11&company={{companyName}}
  2. Set a random value for the refId field. 
    If refId is not set in the API call, the following error appears: CA-249: Field [refId] is not valid [must not be null].​
    The refId must be included in the API request. This field is required by the UnifiedPerson DTO. 
  3. send the POST request
  4. Log in with the affected user in the mobile app.
  5. Sync the app.

Actual behavior:
A synchronization issue occurs on mobile. A sync error is shown.

Expected behavior:
The mobile app should sync successfully without any synchronization errors.

Cause

If the refId differs from the id, a synchronization issue occurs.

Check in Query API:
SELECT p.id, p.refId, p.type, p.userName, p.code FROM Person p WHERE p.type = 'ERPUSER' AND p.userName = '...';
SELECT up.id, up.refId, up.types, up.userName, up.code FROM UnifiedPerson up WHERE up.userName = '...'; 


Entries in the Person table (for ERPUSER: id and refId) are inconsistent, please refer the SAP Help Portal official documentation about 'Person - UnifiedPerson'.

Resolution

  • Set both refId and id to the same value in the request body.
  • Sample:
    "refId": "0123456789ABCDEF0123456789ABCDEF",
    "id": "0123456789ABCDEF0123456789ABCDEF"
     

See Also

SAP Help Portal official documentation about Person - UnifiedPerson, UnifiedPerson DTO and Data API.

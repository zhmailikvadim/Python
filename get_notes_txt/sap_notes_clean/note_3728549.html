Symptom

An error message 'Customer XXXX is not defined in company code XXXX' appears in AIF during Sales Document replication from the source system to the Central Finance system.

This customer XXXX is the payer of this Sales Document and all the process of this SO has no error in source system.

Other Terms

Central Finance, Accounting Views, Sales Document, Payer, F5(102)

Reason and Prerequisites

This is a program error.

Solution

Implement this note in the Central Finance system.

To follow the SD standard process, the new logic is:

For the payer, the validation will check against Customer Master Data, technically, the customer must exist in the KNA1 table.

Symptom

Specific products with defined filters are not replicated as expected when attempting to initialize and replicate only certain product data.

Multiple mock loads of product data result in all products being replicated regardless of filters applied. 

"Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental."

Environment

SAP S/4HANA Cloud Public Edition

Reproducing the Issue

  1. Use app: Replicate by Replication Model
  2. Select Replication model:  XXXXXX
  3. Click Define Filter Criteria
  4. Change Main Filter: MDM:FILTER FOR Product Service 
  5. Change Segment filter: Segment filter for sales (MDM_SALE_F) 
  6. Replicate the replication object. 
  7. Check the Message Monitor for the products being replicated. 
  8. Issue: Products are not replicated following the filter set in Define Filter Criteria.  

Cause

Incorrect configuration of System Filter in the Communication Arrangement.  

There are two values for System Filter in the app Communication Arrangement:

  • Empty: System Filter inactive -> The filter set in Define Filter Criteria is applied.
  • X: System Filter active -> The filter set in Define Filter Criteria is not applied.

Resolution

  1. Open the app Communication Arrangement;
  2. Select the 'Empty' option in the System Filter;

Then the filter set in Define Filter Criteria will work as expected.

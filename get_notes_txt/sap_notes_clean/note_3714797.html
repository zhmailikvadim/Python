Symptom

When adding a Purchase Invoice based on a GRPO, the "Consider Quantity" is not updated correctly as sent in the DI API code.

Reproducing the issue

Prerequisites:

Fixed Assets has been enabled on the Administration → System Initialization → Company Details → Basic Initialization tab.

Scenario:

  1. Go to Modules → Financials → Fixed Assets → Asset Master Data and add new Fixed Asset Master Data (for example: ItemFA). 
  2. Go to Modules → Purchasing - A/P → Goods Receipt PO (GRPO) and add a GRPO for Fixed Asset Master Data added in step 1. Don't select the checkbox for Consider Quantity at line level in GRPO.
  3. Add an A/P Invoice based on the GRPO added in step 2 using DI API or Service Layer. For example:

    SAPbobsCOM.Documents oPurchaseInvoices = (SAPbobsCOM.Documents)oCompany.GetBusinessObject(SAPbobsCOM.BoObjectTypes.oPurchaseInvoices);
                oPurchaseInvoices.CardCode = "V10000";
                oPurchaseInvoices.Lines.BaseEntry = 477;
                oPurchaseInvoices.Lines.BaseType = 20;
                oPurchaseInvoices.Lines.BaseLine = 0;
                oPurchaseInvoices.Lines.ConsiderQuantity = BoYesNoEnum.tYES;
                int RetVal = oPurchaseInvoices.Add();

Expected behavior:

The Consider Quantity at the row level in the A/P Invoice is selected.

Actual behavior:

The Consider Quantity at the row level in the A/P Invoice is not selected.

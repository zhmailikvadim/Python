Symptom

  • Validation logic is not triggered during the Move-In process using the Move Integration Service.
  • Custom validation rules (e.g., from post-exit hooks) are not raising errors as expected.
  • Example Scenario:
    • A move-in record is created for a premise that lacks required master data (e.g., Premise Type is blank).
    • The expectation is for a validation error message to appear on the first step after opening the Move-In facet.
    • However, no error message is shown, and the process continues without validation.

Environment

  • SAP Service Cloud Version 2 1.0
  • SAP Sales Cloud Version 2 1.0
  • Service Cloud V2 Utilities Addon

Reproducing the Issue

  1. Navigate to Extensibility > Extensibility Administration > Move Integration Service.
  2. Select the Move process and configure a validation (e.g., raise an error if Premise Type is blank).
  3. Open any test Premise record where the Premise Type is intentionally left empty.
  4. Initiate a Move-In process for this premise.
    • Expectation: A validation error message should appear on the first step after opening the Move-In facet.
    • Actual Result: No error message is shown, and the process continues without validation interruption.

Resolution

  • Post-validation logic and determination hooks are currently not supported in the Move-In/Move-Out process via the Move Integration Service in SAP Service Cloud Version 2.
  • Validation and error handling for such business rules must be implemented in the backend system (e.g., S/4HANA).
    • Please refer to KBA 3639443 - Extending the Move Process from S/4HANA in SAP Service Cloud V2 Utilities Add-on 
  • If validation support from the Service Cloud V2 side is required for Move-In/Move-Out processes, this represents a new requirement.
    • Customers are encouraged to submit it through the SAP Customer Influence Program.
    • For all roadmap features, please visit SAP Roadmap Explorer.

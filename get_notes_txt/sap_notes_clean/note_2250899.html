Symptom

  • In FIM when logged in with an Active Directory authentication, saving the FC datastore failed with the error bellow :

Error : Failed while calling the "Finance " system to perform the 'getreferencevaluefromtable'action . Failed to invoke services --> Exception Hresult :0X80041618

  • The same problem does not occur when connecting with Enterprise authentication

Environment

  • SAP Financial Consolidation 10.1
  • SAP Financial Information Management 10

 

Reproducing the Issue

  • Connect to IPS with an AD authentication: The user connect directly with its login name and without specifying the domain name
  • Connect to FC: The user connect to FC with specifying the domain name : Domain_name \login
  • Connect to FIM with an AD authentication: The user connect directly with its login name and without specifying the domain name
  • Once connected to FIM , the AD user try to save the FC datastore

 

Cause

A bad definition of the AD user on FC application is the root cause of the issue

Resolution

Change the definition of the user under FC like :

  • login : username
  • Alias : SecWinAD : Domain_name \ username

Keywords

Hresult , webservice , run , job , FC, AD, Users

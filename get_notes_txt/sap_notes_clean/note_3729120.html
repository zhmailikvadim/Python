Symptom

  • When running a custom ReportForm, the "Please wait.. Generating report" message will be displayed.  
  • The application logs indicate:  
    [ERROR:ReportResource.runReport] [RequestIdentifier: 65570402223851] Query canceled because execution took longer than the allowed time.
    [EXCEPTION--An Exception has occurred--]
java.sql.SQLTimeoutException: The query has timed out.
  at net.sourceforge.jtds.jdbc.TdsCore.wait(TdsCore.java:4131)
  at net.sourceforge.jtds.jdbc.TdsCore.executeSQL(TdsCore.java:1086)
  at net.sourceforge.jtds.jdbc.JtdsStatement.executeSQLQuery(JtdsStatement.java:493)
  at net.sourceforge.jtds.jdbc.JtdsStatement.executeQuery(JtdsStatement.java:1427)
  at com.mchange.v2.c3p0.impl.NewProxyStatement.executeQuery(NewProxyStatement.java:3 

Environment

  • SAP Sales Performance Management
  • Agent Lifecycle Management

Reproducing the Issue

  1. Access the environment;
  2. Search for the custom report;
  3. Click "Run";
  4. The "Please wait.. Generating report" message will be displayed;
  5. Upon navigating to the application and processing logs, the following Exceptions are displayed:  
    [ERROR:ReportResource.runReport][RequestIdentifier: 65570402223851] Query canceled because execution took longer than the allowed time.
    [EXCEPTION--An Exception has occurred--]
java.sql.SQLTimeoutException: The query has timed out.

Cause

This issue is caused due to code issues, where connections are left open on the database after the connection timeout.  

Resolution

  1. The Engineering team is conducting further investigation to address this matter under reference code INS-38556.
  2. The target to have the fix implemented is the 2605 release.
  3. To validate further details regarding the 2026 release schedule, please refer to: SAP SPM 2026 Release Schedule.
  4. Please note that these dates are subject to change at the discretion of our Engineering team.

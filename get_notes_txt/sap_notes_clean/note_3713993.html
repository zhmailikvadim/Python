Symptom

In the process "Registration of Meter Operation Service for Electricity (New MOS View)" (DE_B_WIM_REGMOS_MOSN), step 0130, the system does not create a BPEM case when the Point of Delivery (PoD) creation fails. The exception is caught but not raised again, so no BPEM case is generated.

This issue is relevant for the market role: MOS.

Other Terms

WIM, DE_B_WIM_REGMOS_MOSN, /US4G/CL_PRST_MOSN_REG_CRT_POD, registration process

Reason and Prerequisites

The error occurred because the system catches the exception during PoD creation but does not raise it again, which prevents the creation of a BPEM case.

Solution

Implement the corrections of this SAP Note or import the relevant support package.
After implementing this SAP Note, the system creates a BPEM case with a solution repeat step and completes the step if PoD creation fails. Activate the BC set /US4G/3_0_MOS_SP07_NOTE_3713993 using transaction SCPR20. You can verify the fix by retesting the scenario: if PoD creation fails, the system generates a BPEM case.

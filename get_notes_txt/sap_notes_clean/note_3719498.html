Symptom

  • Multi Action task scheduled using the calendar fails with the error: "Member does not exist in the dimension or selected hierarchy, or its hidden," despite the members existing.
  • The Multi Action executes successfully when triggered manually in the story.

Environment

  • SAP Analytics Cloud 2026 2.5
  • Multi Actions

Reproducing the Issue

  1. Open the calendar and create a Multi Action task.
  2. Check the Data Action monitor after the scheduled job triggers.
  3. You will see the error "member [member id] does not exist in the dimension or selected hierarchy, or it is hidden"

Cause

Issue occurs if one or more dimensions are set to "hide parents" in the "Rights / Access" section of the dimension definition in the model.

Resolution

The issue will be resolved in an upcoming patch release.

In the meanwhile you can work around this issue by unchecking the "Hide Parents" option in the model dimensions.

Symptom

  • In the cleanup on the sender, central, and receiver system:
    • Deletion of function groups and function modules is sometimes inconsistent
    • If function modules touched by the cleanup are registered with UCON, the performance is negatively impacted and the cleanup becomes very slow

Other Terms

downtime minimization tool for SAP S/4HANA, NZDT

Reason and Prerequisites

Any of the following add-on support packages is installed in your system landscape:

  • NZDT 1.0 SP03
  • NZDT 1.0 SP02

Solution

  • Deletion of function groups and function modules will be performed consistently
  • If function modules touched by the cleanup are registered with UCON, they will first be de-registered in order to maintain high cleanup performance

Required Action

In order for this note correction to be effective, the note must be implemented before:

  • Either executing any of the following execution tree activities for the cleanup:
    • NZTR1_DELE_NZDT_OBJ_S
    • NZTR1_DELE_NZDT_OBJ
    • NZTR1_DELE_NZDT_OBJ_R
  • Or executing any of the following cleanup programs locally:
    • /NZT/CLEANUP_SENDER
    • /NZT/CLEANUP_CENTRAL
    • /NZT/CLEANUP_RECEIVER

Implement the correction instructions in transaction SNOTE on the following systems of your system landscape:

  • Sender system
  • Central system
  • Receiver system

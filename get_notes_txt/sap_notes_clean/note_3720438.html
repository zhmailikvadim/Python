Symptom

When maintaining SSCUI "Specify Selected FS Items for Automatic Posting" in SAP S/4HANA Cloud Public Edition, the following error occurs when saving changes:

"Deletion not possible for record created by SAP"
Message no. /CFG/CF_SSC_FRW123

This error may occur when changing Debit or Credit subitems in sections such as:

  • ANI, B/S
  • ANI, I/S
  • Deferred Tax, B/S
  • Deferred Tax, I/S

Environment

SAP S/4HANA Cloud Public Edition

Reproducing the Issue

  1. Log into Customizing CBC. 
  2. Go to section configuration and search for IMG Activity "Specify Selected FS Items for Automatic Posting".
  3. Open IMG Activity "Specify Selected FS Items for Automatic Posting".
  4. Change Debit Subitem in section "ANI, B/S" to XYZ (XYZ represents the Subitem).
  5. Change Credit Subitem in section "ANI, B/S" to XYZ.
  6. Press the Save button.
  7. Error message: "Deletion not possible for record created by SAP" will appear.

Cause

The issue is caused by inconsistent configuration of the Financial Statement (FS) Items assigned to the Item Roles maintained in SSCUI Specify Selected FS Items for Automatic Posting.

The Breakdown Category assigned to the FS Item determines:

  • Allowed Subitem Categories
  • Allowed Subitems
  • Sub-assignment structure

If the Breakdown Category does not support the required Subitem Category, the automatic posting configuration becomes inconsistent and the system may reject the changes with error "/CFG/CF_SSC_FRW123".

Typical examples include FS Items assigned to Item Roles such as:

  • S-ANI-BS
  • S-ANI-PL
  • S-DEF-TAX-AST
  • S-DEF-TAX-PL

Additional Information:

This issue may also occur if the FS Items assigned to Item Roles such as S-ANI-PL or S-DEF-TAX-PL use a Breakdown Category without any active breakdown dimension (for example no Subitem and no Partner defined).

In such cases, the automatic posting configuration becomes inconsistent and the system may issue error. Using a Breakdown Category where at least one breakdown dimension is defined (for example optional Partner) resolves the issue.

Resolution

Verify and correct the FS Item configuration:

  1. Open the app Define Financial Statement Items.
  2. Identify the FS Items assigned to Item Roles such as:
    S-ANI-BS 
    S-ANI-PL 
    S-DEF-TAX-AST 
    S-DEF-TAX-PL
  3. Verify that the FS Items use a Breakdown Category that supports the required Subitem Category.
  4. Ensure that the required subitems are allowed for the FS Items.
  5. Save the FS Item changes.
  6. Open SSCUI "Specify Selected FS Items for Automatic Posting" again.
  7. Maintain the required Debit and Credit subitems.
  8. Save the configuration.

After correcting the FS Item configuration, the changes can be saved successfully without error

The Breakdown Category determines the allowed sub-assignments and subitems for an FS Item.

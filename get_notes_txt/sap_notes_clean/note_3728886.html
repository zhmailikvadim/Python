Symptom

  • Multiple buckets with the same ID can be created.
  • No error message is displayed when creating a new portfolio bucket with the same ID as an existing bucket.

Environment

  • SAP Portfolio and Project Management
  • SAP Portfolio and Project Management for SAP S/4HANA
  • SAP S/4HANA
  • SAP S/4HANA Cloud Private Edition 

Reproducing the Issue

  1. Create a new portfolio bucket with the same ID as a bucket that already exists.
  2. Observe that no error message is displayed, and the bucket is successfully created.

Cause

  • The default program response per the application design is not to enforce the uniqueness of bucket IDs.

Resolution

A different program response can be achieved by maintaining a value for global setting 0007 / 0051 "Bucket ID check" in the customizing activity "Override Default Global Settings":

SPRO
> Portfolio and Project Management 
 > Portfolio Management
  > Global Customizing
   > Global Settings
    > Override Default Global Settings.

Configure the global setting as follows:

  • ' ': No verification for external ID is performed. Allows duplication of bucket IDs within and across portfolios. (This is the default setting.)
  • 'L': Verification of external ID is performed within the portfolio. Does not allow the creation of buckets with the same external ID within the same portfolio. External ID duplication is allowed across portfolios.
  • 'G': Verification of external ID is performed across portfolios. Does not allow the creation of buckets with the same external ID across portfolios. External ID duplication is not allowed.

Symptom

  • A bookmark registered (e.g., in the Trial Balance application) is not automatically added to the favorites of other users when initiated by a different user.
  • Previously, bookmarks were automatically registered in the favorites of other users upon being created.
  • As a result, the bookmark is not displayed by default, and user operations differ from previous behavior.

Environment

FPM Analytical GUIBB

Reproducing the Issue

  1. User A registers a bookmark in the Trial Balance application.
  2. User B initiates the Trial Balance application.
  3. The bookmark registered by User A is not added to the favorites of User B.

Cause

The behavior has changed due to a design update introduced with SAP_BW 758. In this new design, if a bookmark is saved as public, it needs to be manually set as a favorite for the other users themselves in the manage dialog before it becomes visible to them.

The design change version can be checked by database table FPM_SAVED_SEARCH, take Trial Balance application as example:


MANDT         000
USER ID       *
ROLE ID
CONFIG ID     FIS_FPM_GRID_GENERIC_TRIAL1
CONFIG TYPE   0
CONFIG VAR
APP KEY       2CCFITRIALBALQ0001

...

VERSION       1

version 1 means the bookmark follows the new design, 
version 0 means the bookmark follows the old design.

Resolution

 Newly created public bookmarks needs to be set to favorite by each user via the manage dialog setting.

Symptom

This KBA describes the restrictions for the integration scenarios of Multiple different Work Zone editions with Joule

Environment

  • SAP Build Work Zone, standard edition
  • SAP Build Work Zone, advanced edition
  • SAP Start
  • SAP Joule

Resolution

  • You can only integrate ONE Joule instance with ONE SAP Build Work Zone instance (one edition).

  • In case of two SAP Build Work Zone subscriptions (different editions) in the same BTP Subaccount:
    e.g. One for Build Work Zone, standard edition and one for Build Work Zone, advanced edition, then only ONE of them can be integrated with Joule Subscription.

  • If you have a subscription to both SAP Build Work Zone standard edition and SAP Build Work Zone advanced edition on the same Subaccount, you might notice that the after integrating Joule with one edition, the Joule site setting becomes enabled on the other edition. Please note, that running Joule on the other edition is not supported and might yield unexpected results.

  • To integrate Joule with both SAP Build Work Zone, standard and advanced editions, you can only create an additional subscription to Joule service in another separate BTP Subaccount and use one edition per BTP Subaccount as always.

  • You cannot include multiple Work Zone instances to the same Joule setup (Should fail the Formation creation with a clear error)

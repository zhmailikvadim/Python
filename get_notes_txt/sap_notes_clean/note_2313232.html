Symptom

Due to FS-RI functions, the FS-RI posting table my be empty during the release of accounts or may contain fewer postings than FS-CD items are created (for example, execute treaty calculation rule). However, you expect that there is always an FS-RI posting for the FS-CD item.

Reason and Prerequisites

Program error

Solution

History:

Version 1 from 5/3/2016: New　

Implement the source code corrections.
The problem is solved with Support Package SAPK-70012INMSG.　

Solution:

The system checks whether a corresponding posting exists in FS-RI before the claim reference is used. If no postings exist, no claim reference must be transferred to FS-CD.

Technical effects:

There are none.

Manual changes:

You do not have to perform any manual tasks.

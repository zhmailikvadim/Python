Symptom

  • Error message is raised during processing of Poland KSeF eDocuments:
    "Status of the eDocument does not allow the execution of process step GET_DRCSTR"

  • The issue occurs when attempting one of the following actions in EDOC_COCKPIT:

    • Execute Get Status

    • Resubmit a rejected eDocument

  • The system does not allow further processing of the eDocument despite the document being in rejected status.

Environment

  • SAP ERP
  • SAP S/4HANA
  • SAP Document and Reporting Compliance, cloud edition

Reproducing the Issue

  1. Attempt to resubmit a rejected eDocument from the cockpit.
  2. Trigger the "Get status" process step.
  3. Observe the error message: "Status of the eDocument does not allow the execution of process step GET_DRCSTR".

Cause

Electronic documents created before the delivery of resubmission functionality do not contain the required process steps in the process framework.

As a result, when attempting to execute actions such as Get Status (GET_DRCSTR) or resubmission, the system determines that the process step is not applicable for the current document status and raises the error.

Resolution

This is standard system behavior.

  • Resubmission and retry functionalities are supported only for electronic documents created after the feature delivery (January 23, 2026).

  • Electronic documents created before this date cannot be resubmitted.

For such older documents:

  • Correct the source data if required.

  • Create and submit a new electronic document.

To use resubmission functionality, ensure the system is updated with the relevant feature delivery and test with newly created documents.

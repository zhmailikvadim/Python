Symptom

Source is getting deleted on selecting alternative used functionality in Recipe with multiple stages

Environment

  •  SAP ERP Recipe Development on PLM Web UI (PLM-WUI-RCP)
  • SAP S4/HANA

Reproducing the Issue

  1. Create a multi-stage recipe which has 3 stages for example
  2. Navigate to the Formula tab
  3. Make the 1st and 2nd stages as input to the 3rd stage
  4. Save the Recipe
  5. Update the input stream of the 2nd stage as alternativee to the 1st stage - do this by unticking the alternative used check box
  6. Update the item number
  7. The source of the 2nd stage is getting removed

Cause

  • If there are 2 items with the same item number from the same stage, then only one of them can be used in the process, that is why the "alternative used" option exists.
  • In the case you untick the check box and then tick another entry with the same item number, the source will switch back for the last entered number while the other blanks out.

Resolution

This is the intended design of the system. This functionality may be requested as a product improvement, note 11 contains the details if it is desired to use these options.

Symptom

When load data into DP planning book, COM error 40053 happens:

  • Error for COM routine using application program (return code 40,053). (Message No. /SAPAPO/OM 102)
  • Creation of relationship has failed. (Message No. /SAPAPO/OM_TS 053)

"Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental."

Environment

  • SAP Supply Chain Management (SCM)
  • SAP Advanced Planning and Optimization (SCM-APO)
  • SAP enhancement package for SAP Supply Chain Management
  • SAP Supply Chain Management Demand Planning (SCM-APO-FCS)

Reproducing the Issue

  1. Go to transaction /SAPAPO/SDP94
  2. Select corresponding Planning Book and Data View
  3. Use Selection profile or Selection window to load data

Cause

COM error 40053 means "Double relation between CVCs are not allowed".
When an aggregate time series was created or a relation generated between two time series, one or more relations was specified twice.

Resolution

  1. Execute report /SAPAPO/TS_PSTRU_TOOL in t-code SE38.
  2. Select radio button "Mast.Plng Obj.Struc." and input Master Planning Object Structure (MPOS) name.
  3. Select radio button "Check duplicate entries"
  4. Push "Execute" button, and in the next screen, push "Execute" button directly.
  5. The result list is the duplicated entries.
    *Notice: If the CVCs are not really duplicate, you can push "Internal Format" button, you'll find duplicate GUIDs for the CVCs.
  6. System cannot correct such inconsistency automatically. You have to decide which CVC should be deleted / recreated, and do it manually.
    *Notice: Deletion of CVC will also delete time series data of it. Please ensure data backup and restore before and after deletion.
  7. If there's active fixed aggregate for this MPOS, perform the same as above for the fixed MPOS.

Keywords

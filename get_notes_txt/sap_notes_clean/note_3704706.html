Symptom

When you try to add an A/P invoice based on landed costs, the following error message appears:

"Costs or Customs amount exceeds sum total of Costs or Customs Balance Due in all based landed costs; account code 1795"

Reproducing the issue

Scenario:

  1. Create a non-inventory Item 'p1'. In the Item Master Data, clear the 'Inventory Item' checkbox. Set G/L Accounts By to 'Warehouse'. In the Warehouses Setup, set Expense Account to 3400.
  2. Go to Administration → Setup → Purchasing → Landed Costs and select an existing landed cost. Set Landed Costs Alloc. Account to 1795.
  3. Go to Administration → Setup → Inventory → Customs Groups. Set Customs Allocation Account to 1795 (same as in Landed Costs) and Customs Expense Account to 3400 (same as in Warehouses Setup Expense Account).
  4. Go to Purchasing - A/P → Goods Receipt PO, select vendor V10000, add item 'p1', set necessary item line fields, and choose Add.
  5. Go to Purchasing - A/P → Landed Costs, set Vendor to V10000 and Broker to V1010. Choose Copy From → Goods Receipt PO, and select the goods receipt PO created in step 4.
  6. Set Customs Value on the line to 20.
  7. On the Costs tab, select the landed cost maintained in step 2. Set Amount to 15, and choose Add.
  8. Go to Purchasing - A/P → A/P Invoice. Set Vendor to V1010, choose Copy From → Landed Costs, select the landed costs you created, and choose Add.

Expected behavior:

The system adds the A/P invoice successfully.

Actual behavior:

The system displays an error message "Costs or Customs amount exceeds sum total of Costs or Customs Balance Due in all based landed costs; account code 1795".

Symptom

When running the year-end CPF refund report (transaction PC00_M25_NCCPY, report RPCCPYR0_01), employees who have split contribution wage types in their payroll results may show incorrect CPF contribution totals. This occurs when the same wage type appears multiple times with different split indicators.

The report displays only the last occurrence amount instead of the sum of all splits. For example, if an employee has wage type /305 (Total CPF Employee Contribution) split into SGD 619.35 and SGD 580.65, the report shows only SGD 580.65 instead of the correct total of SGD 1,200.00.

Other Terms

Report RPCCPYR0_01, PC00_M25_NCCPY, Split Wage Type, Year-end CPF Report

Reason and Prerequisites

This is a program error.

Solution

You could apply HR Support Package or implement instructions in this SAP Note.

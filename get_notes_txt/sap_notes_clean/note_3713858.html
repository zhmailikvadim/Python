Symptom

When attempting to run the report /SHCM/RH_SYNC_BUPA_EMPL_SINGLE for Employee/Business Partner Synchronization in SAP S/4HANA, the execution fails with a short dump (category: LIST_TOPOFPAGE_OVERFLOW). This typically occurs when a large number of employees or a long list of company codes are selected for processing in a single run.

Environment

  • SAP S/4HANA on-premise
  • SAP S/4HANA private cloud edition

Reproducing the Issue

  1. Open the report /SHCM/RH_SYNC_BUPA_EMPL_SINGLE.
  2. Select a large number of employees and/or include a long list of company codes in the selection criteria.
  3. Execute the report.
  4. Observe that the system generates a LIST_TOPOFPAGE_OVERFLOW short dump.

Cause

The report /SHCM/RH_SYNC_BUPA_EMPL_SINGLE is designed primarily for initial synchronization (all employees need to be synced for the first time) or for manual synchronization of one/few employees. Attempting to process a large volume of employees and/or many company codes in one execution leads to a high memory and list processing demand, resulting in a LIST_TOPOFPAGE_OVERFLOW short dump.

Resolution

  • For initial synchronization involving many employees, run the report in several smaller batches (e.g., process fewer company codes per job) or enable the parallelization feature to distribute the load across multiple jobs.
  • For regular, daily synchronization (delta processing of only recently changed employees), utilize the dedicated report /SHCM/RH_SYNC_BUPA_FROM_EMPL instead, which is optimized for ongoing, incremental synchronization.
  • Avoid running /SHCM/RH_SYNC_BUPA_EMPL_SINGLE as a mass or delta program for large data volumes.

Symptom

When posting a claim, the system should handle claims with a zero amount and allow posting where applicable.However, an error is raised stating that posting cannot be performed when attempting to post a claim with a zero amount. This prevents the claim from being posted and interrupts further processing.

This note will fix the issue so that claims with zero amount are processed correctly during posting.

Other Terms

zero amount claim, claim posting error, posting not allowed, /IRM/GCRM, zero value posting, SAVE+POST, /IRM/GCRU

Reason and Prerequisites

This is caused by program error.

SAP Note #3720807 is available for ECC Releases.

Solution

Please implement the correction instructions attached. These corrections are only valid if you have applied the Support Packs.

Release V4 - 2022 V2, 2023, 2023V2, 2023 V3 and 2023 V3 Support Pack 1.

Use transaction SNOTE to implement the correction instructions specified below and activate the affected program object.

Symptom

  • A Per Position Comp&Pay was ran for a period.
  • The position on this pipeline generated credits and deposits.
  • User has X credits visible in the UI, but the system generates 2X commissions. (double).
  • Each credit should generate only 1 commission.

Environment

  • SAP SuccessFactors Incentive Management
  • Incentive Management

Reproducing the Issue

  1. Run the Compensation and Pay pipeline for a period in Per Position Mode.
  2. Select the position chose on Position's workspace.
  3. By the Related search, verify credits in the UI for the affected period.
    1. Should see X.
  4. Check the commissions generated
    1. Expected X, but it's showing 2X (double).

Cause

  • Two pipeline runs were executed on the same day for the same position.
  • Duplicate credits were created during the pipeline runs, leading to duplicate commissions.

Resolution

  1. Reset the classification for the affected period to remove duplicate credits.
  2. Run the pipeline again with advanced logs enabled for Verbose and SQL logging to verify the issue resolution.
  3. If the issue persists, perform a reset from payment for the affected period and rerun the Compensation and Pay pipeline.
  4. Confirm that the number of credits and commissions generated matches the expected values.

Keywords

Symptom

  • While performing Goods Issue against Production Order (movement type 261), system allows raw material consumption exceeding the quantity required in the Order.
  • No error or restriction is triggered when Over-Issue occurs.

Environment

SAP S/4HANA CLOUD PUBLIC EDITION

Reproducing the Issue

  1. Create a production order (CO01) with the required quantity.
  2. Ensure raw material components are maintained in the order with specific planned quantities.
  3. Release the Production Order.
  4. Open the App Post Goods Movement (MIGO).
  5. Select Goods Issue → Order and ensure movement type is 261.
  6. Enter the Production Order ID.
  7. Enter a quantity above the Required Quantity as maintained in the Order.
  8. Execute the Goods Issue.
  9. System allows posting without issuing an error or a warning message.

Cause

Configuration: The message 'M7 362 (SSCUI 103897) is currently set to not trigger any message. It can either be set to 'E' for error or 'W' for Warning.

Resolution

  1. Access SSCUI 103897.
  2. Scroll down to the message M7 362 ‘& exceeded by &&’.
  3. From the category drop-down box, select ‘E’ if the system should return an error or 'W' if only a warning should be triggered. 
  4. Click Save.

If set to 'Error', processing will be stopped by this error message when goods issue quantity exceeds requirement quantity.
If set to 'Warning', user will be notified of the exceeded quantity compared to the requirement quantity and the user can decide to rectify the quantity or continue the Goods Issue with the exceeded Quantity.

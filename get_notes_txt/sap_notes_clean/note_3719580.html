Symptom

  • Whether the session expires automatically after 30 minutes regardless of activity or if it expires only when there is no activity for 30 minutes 
  • Inquiry about changing the default session expiration time and the maximum configurable value recommended by SAP
  • Questions related to OAuth client properties such as accessTokenValiditySeconds, refreshTokenValiditySeconds, authorizationCodeValiditySeconds, and oidcTokenValiditySeconds

Environment

SAP Commerce Cloud 2211, SAP Commerce Cloud 2211 for JDK21

Resolution

  1. The Commerce implementation is based on the Spring Authorization Server, where the `accessTokenTimeToLive` parameter does not have a hard-coded upper limit within the library itself
  2. Constraints are imposed by the `java.time.Duration` data type. Commerce translates the property `authserver.accessToken.timeToLive.seconds` into a value in `Duration.SECONDS` without additional limitations
  3. It is recommended to use a refresh-token flow:
  • Issue short-lived access tokens (up to 300 seconds) together with long-lived refresh tokens
  • When an access token expires, obtain a new access token in the background using the refresh token
  • This approach allows the use of self-contained JWT access tokens issued by Commerce and avoids per-request database validation
  • Using long-lived, self-contained JWT access tokens can pose significant security risks

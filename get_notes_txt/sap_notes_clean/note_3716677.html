Symptom

  • The scenario is integration of MFG DM and QM, MFG record inspection results and send it to ERP QM system to complete the result recording (call BAPI BAPI_INSPOPER_RECORDRESULTS), but MFG system receives error message from ERP system during this process.
  • in MFG DM with the message: "Collaboration execution failed due to the external error: Defects occurred during confirmation; Defect class 000000 of characteristic w/confirmation no. XXXXX not defined."

Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.

Environment

  • SAP ERP Quality Management (QM)
  • SAP R/3
  • SAP R/3 Enterprise
  • SAP ERP Central Component
  • SAP ERP
  • SAP Enhancement package for SAP ERP
  • SAP Enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA, on-premise
  • SAP S/4HANA Cloud Private Edition

Cause

The issue is caused by the wrong sampling procedure used in the routing/inspection plan. The sampling procedure is designed for inspection points, but the routing/inspection plan is not defined for inspection point.

  1. In the payload of the MFG DM system, the value of inspection plan/operation/characteristic is specified.
  2. Open the inspection plan or routing in T-code QP03 (inspection plan ) or CA03 (routing) refer to the payload of MFG DM system.
  3. On head level of the inspection plan, the inspection point is not defined.
  4. On the inspection characteristic level, it can show which sampling procedure are used.
    In T-code QDV3, open the sampling procedure, all the below 3 highlight options are for inspection points.

Resolution

This is incorrect usage of sample procedure in the inspection plan.

To ensure the consistency of the inspection plan, please change another sampling procedure which is defined as 'without inspection points' on the characteristic of the inspection plan or assign an inspection point on the header of inspection plan in T-code QP02.

Kindly note the changes will not reflect on existing inspection lot, please create new inspection lots and check the system behavior again.

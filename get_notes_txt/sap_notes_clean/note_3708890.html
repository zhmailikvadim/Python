Symptom

During the migration of a HANA Server Database to another server, a backup was created to disk and imported on the new server, in the process there was a loss of information from the backint backup catalog.

There is a need to perform a manual backup to disk without losing the backup catalog information.

Environment

SAP HANA, platform edition

Resolution

No supported method exists to manually include or merge BACKINT catalog information into a disk-based backup.

Notes:

  • During migration and import, the backup catalog is not transferred.
  • Recovery from a disk-based backup restores only the catalog entries that existed at the time the backup was created.

Recommended approach for workaround:

  • Perform a full SAP HANA system copy using HANA backup/recovery.
  • Recover the system database first, then each tenant database.
  • Install and configure the same BACKINT vendor library on the target system before starting recovery.

Outcome:

  • After recovery, the internal backup catalog reflects the state at the time of the source backup.

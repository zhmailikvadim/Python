Symptom

Solr full index failed with below error:

Cause

The StackOverflowError appears because getAllSupercategories() is getting called recursively until the stack gets filled up. This can happen if you have categories with circular references. For example you have set Category A as a subcategory of Category B, B a subcategory of C, and C a subcategory of A etc. Or you have set Category A as supercatogry of itself.

Resolution

Check category structure and delete the circular reference.

We could use below groovy script in hac to check if there's category set as mentioned.

import de.hybris.platform.servicelayer.search.FlexibleSearchQuery

import de.hybris.platform.category.impl.DefaultCategoryService

import de.hybris.platform.category.model.CategoryModel

 

DefaultCategoryService.metaClass.checkCyclic = { final CategoryModel category ->

    if (category == null) {

        print "Parameter 'category' was null."

    }

    return checkCyclicInternal(category, new HashSet<CategoryModel>(Collections.singleton(category)))

}

DefaultCategoryService.metaClass.checkCyclicInternal = { final CategoryModel category, final Set<CategoryModel> controlSet ->

    final Collection<CategoryModel> superCategories = category.getSupercategories()

    if (superCategories != null && !superCategories.isEmpty()) {

        for (final CategoryModel parent : superCategories) {

            if (notVisited(parent, controlSet)) {

                for (List<CategoryModel> _ : checkCyclicInternal(parent, controlSet)) {

                }

                markNotVisited(parent, controlSet)

            }

            else

            {

                println 'path cycle found for category: [' + category.getCode() + ']'

            }

        }

   }

}

defaultFlexibleSearchService.search(new FlexibleSearchQuery("SELECT {PK} FROM {Category}")).getResult().each {

    defaultCategoryService.checkCyclic(it)

}

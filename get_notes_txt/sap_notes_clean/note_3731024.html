Symptom

  • Wrong Functional Area is picked during CATS time sheet entry. 
  • When a time sheet is created by copying from a reference line, the Functional Area from the reference entry is copied to the new entry instead of updating to match the changed receiving object.
  • This leads to incorrect Cost of Sale reporting.
  • Issue observed when changing from a Network Activity Example: (FA “XX”) to a Cost Center (FA “YY”); saved entry keeps the reference Functional Area. 

Environment

  • Human Capital Management (HCM)
  • SAP ERP Project System (PS)
  • SAP R/3
  • SAP R/3 Enterprise
  • SAP ERP Central Component
  • SAP ERP
  • SAP Enhancement package for SAP ERP
  • SAP Enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA, on-premise
  • SAP S/4HANA, cloud edition

Reproducing the Issue

  1. Start transaction CAT2
  2. Copy a time entry from a reference line (for example, a Network Activity) with Functional Area “XX”.
  3. Change the receiving object to a Cost Center with Functional Area “YY”.
  4. Save the time sheet. the Functional Area remains from the reference line instead of updating to the receiver’s Functional Area.

Cause

Standard CAT2 defaulting behavior: values are set only when fields are initial and are not automatically refreshed on later changes. When copying and then changing the receiver, the Functional Area field is not cleared, so no redetermination occurs 

Resolution

  1. Review the standard defaulting logic for CAT2 to confirm that fields are not automatically refreshed after copy/change (refer to:  SAP Note 333884 and KBA 2459244).
  2. If redetermination of Functional Area is required when the receiver changes in CAT2, implement user exit CATS0002 (“Supplement Recorded Data”) to clear and/or rederive the Functional Area on change.
  3. Test in CAT2 by copying a line, changing the receiver, and saving to verify the Functional Area updates as desired through the exit.
  4. If historical entries must be corrected, optionally use report ZCATSCNELO to delete affected CATS records and re-enter them as needed (refer to: SAP Note 623693 & also please refer to the mentioned KBA 1904878).

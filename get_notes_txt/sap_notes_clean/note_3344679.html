Symptom

You use the dunning program (F150), interest program (FINT, FINTAP, RFDZIS00_PDF, RFKZIS00_PDF), or FI correspondence with printing technique PDF and e-mail dispatch. 
You have programmed that e-mails are to be signed: c_finaa-mail_sign = 'X'.

The following symptoms occur:

  • The e-mails are sent successfully, but are not signed.
  • The system issues an inexplicable message, for example, BL208: "No log found in the database" 
  • During live dunning printout, the dunning data is not updated. You notice this as follows:
    • In the status of the dunning run, the number of printed dunning notices does not match the number of dunning notices to be sent.
    • In the master record of the business partner, "Last Dunning Notice" and "Dunning Level" have not been updated.
    • In the dunned documents, "Last Dunning Notice" and "Dunning Level" have not been updated.
    • The dunning notice is not contained in the dunning history of the business partners. 

You expect message MSINT602: "No signatures for mails possible in this system" and the successful sending of the e-mail without a signature. The dunning data must be updated during live print.

Reason and Prerequisites

The error occurs if it was impossible to sign e-mails in the system (Lc_sendreq->Set_sign( 'X' ).in SEND_ONE).

Program error: The routine SEND_ONE from the include LSEND_PDFF01 returns an error message in ls_message, even though the e-mail is sent successfully (without signature).

Solution

Source code corrections

  • An item data is inserted into DBM order without pressing ENTER
  • Any usage of function keys (Back, Exit, Cancel, Document Overview, Action Log, Status Overview etc.) will trigger event ITEM_NEW which in fact inserts an item into DBM order

Other Terms

/DBM/ORDER01, /DBM/ORDER02, ITEM_NEW

Symptom

  • Read access to model(s) and the story is required to view the data.
  • Read access to models grants visibility in the folder structure. 
  • Not sharing the private folder does not prevent discovery; the model can still be found by searching the model name. 
  • Clicking a model opens it in the modeler view, allowing data preview and visibility of the model structure.

Environment

SAP Analytics Cloud (2026.2.10)

Reproducing the Issue

  1. Login to SAC system (with a user that has "Read" access to (one or more) model(s) used for story viewing).
  2. Navigate to the relevant folder or use 'Search tab' to locate the model by name.
  3. Observe that the model is visible.
  4. Click the model and observe it opens in the modeler, where data preview and the model structure are visible.

Cause

Feature limitation (by design): When "Read" access is granted to a model, the model remains discoverable (including via search) and can be opened in the Modeler in read mode; hiding the model or disabling model opening is not supported yet is SAC. 

Resolution

An enhancement request (ER) to enable hiding model objects from search and the modeler for "Read" access, has been created: Enhancement of current read access to hide model objects from search and modeler.

"SAP is committed to reviewing and responding to the ideas submitted on a regular basis. The ideas with the most votes will receive a higher level of attention for response and consideration. The processes and procedures associated with this site will continue to evolve and enhance over time. 
Refer to Knowledge Base Article:2424376 - How do you create an enhancement request and provide feedback for SAP Analytics Cloud? for more details"

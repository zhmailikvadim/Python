Symptom

With Release 7.0, assumed fields were added to the account dialog in the Basic System (treaty, section, accounting year, and accounting period). However, only the assumed treaty and the assumed section are ready for input for retrocession treaties, inward posting year and period are only displayed.

The fields for inward posting year and period can be made ready for input using the BAdI method /MSG/R_A_ADDON_BADI~FIELD_MODIFICATION_1120. If you make only entries in the two fields for inward posting year and/or period, these are lost.

Other Terms

Assumed fields

Program error

Solution

History:

6/11/2015: New

Implement the source code corrections.
Support Package SAPK-70006INMSG corrects the problem.

Solution and technical effects:

The system behavior was adjusted in such a way that user entries in the two fields for inward posting period and/or year are retained.

Note that after you open the fields using the BAdI method, you must ensure that this data is not overwritten by a default value in the standard system. For this, the BAdI method /MSG/R_A_ADDON_BADI~PREALLOCATE_ASSUMED_FIELDS already exists.

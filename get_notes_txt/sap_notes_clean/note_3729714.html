Symptom

  • The TM-defined loading sequence (e.g., sequence of loads/stops/consignments) is not visible in EWM Monitor or RF UI in an Advanced Shipping and Receiving (ASR) scenario.
  • In TM Freight Order -> Load Plan, the "Loading Sequence" column may display the value "0".
  • TM Load Plan attributes, including the loading sequence, are not transferred to EWM in the standard ASR integration.

Environment

  • SAP Transportation Management embedded in SAP S/4HANA
  • SAP Extended Warehouse Management in SAP S/4HANA

Reproducing the Issue

  1. Go to "Edit Freight Order"
  2. Enter the Freight Order.
  3. Go to the "Items" tab.
  4. Change the item hierarchy to "Load Plan".
  5. Create the Load Plan.
  6. Observe that the "Loading Sequence" column displays the value "0".
  7. (Optional) Check EWM Monitor or RF - loading sequence is not available.

Cause

  • The standard advanced shipping and receiving integration model does not transfer TM Load Plan attributes, including the loading sequence, to EWM. This is expected system behavior in the current ASR integration scope.

Resolution

  • In TM, the loading sequence is part of the Freight Order Load Plan; however, it is not automatically determined in standard and may appear as “0”.
  • In ASR integration, TM Load Plan attributes (including loading sequence) are not transferred to EWM. This is standard system behavior.
  • The TM planned loading sequence is supported only via TM loading list printout and is not supported in EWM RF or Monitor-based execution in ASR scenarios.
  • Refer to SAP Note ##3423370 for feature comparison between ASR and TU-based TM-EWM integration.
  • There is no standard configuration or BAdI available in ASR to enable loading sequence visibility in EWM.
  • If required, a custom solution can be developed to transfer and display the loading sequence in EWM. This may require cross-application enhancements.

Note : BAdIs referenced in KBA ##2769244 are applicable only for TU-based (LDAP) integration and are not available in ASR scenarios.

Symptom

The Workflow API endpoint is the same URL used to log in to Workflow. However, in some cases, this URL might not be visible.

Environment

  • SAP Sales Performance Management
  • Advanced Workflow

Resolution

  1. If the log in was executed to Workflow directly, the API endpoint is the URL displayed in the browser.
  2. If the log in was executed via proxy from Sales Performance Home or Sales Portal, the URL might not be the true Workflow URL, as the browser shows still the Sales Portal URL. To discover the true URL:
    1. Collect the HAR logs while going from Sales Portal to Workflow, as described in: SAP For Me: 3503700 - How to to collect HAR (HTTP Archive) files for support - SAP for Me 
    2. Check for the URL that appears at the time of the logon. If needed, search for the keyword "/wpm" in the browser's network tools. All workflow URLs are located in the /wpm path.
  3. The API endpoint is the URL captured in one of the steps above plus the path /api appended after /wpm.

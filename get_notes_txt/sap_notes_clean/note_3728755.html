Symptom

  • Attempting to delete plant text in English using the "start mass maintenance - Product" app results in the deletion of German text instead.
  • The issue occurs without any errors or warnings displayed by the application.
  • The deletion flag 'D' was added in the first column for the English text, and the file was uploaded to the mass maintenance app.
  • Exporting plant texts via the "Export Master Data - Products" App confirms that German text was deleted instead of English text.

Environment

SAP S/4HANA Cloud Public Edition

Reproducing the Issue

  1. Log-in to S/4HANA Fiori App
  2. Select Export Master Data - Products App
  3. Export using filter option for required Plant 
  4. Now, select Start Mass Maintenance - Product App
  5. Upload this file to the application with Action Code D for deletion or required line item
  6. Click on Start and proceed to the next screen
  7. Continue the process until completion, noting that no errors or warnings are displayed.
  8. Export plant texts via the Export Master Data - Products App and observe that German text has been deleted instead of English text.

Cause

The issue is caused by the system design.

Resolution

  1. The language key for the plant is set to DE, which restricts edit of Product Plant Text  to the language defined by the Plant Language Key.
  2. Note that the Language Key for Plant is a single selection and not a multi-selection view, allowing only one language to be selected as per standard design.
  3. To request a change in functionality, refer to the Influence Forum for submitting feature requests.
  • Refer to: 2963059 - How to Submit Feature Requests for SAP S/4HANA Cloud Product.

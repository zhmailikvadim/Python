Symptom

Option 1)
You create a pro forma billing document for a delivery in the feeder system. The system creates a customs export declaration in SAP GTS. You send an electronic export message to the US customs server and receive a registration number.
You do not complete the pro forma.
You change the delivery by adding, deleting, or changing delivery items.
You create a second pro forma billing document for the same delivery.
You expect the system to update the existing customs declaration and retain the registration number.
Until now, you have had to cancel the customs declaration so that the system could create a new customs declaration, and the registration number was not retained by the system.

Option 2)
You create a pro forma billing document for two deliveries in the feeder system. The system creates a customs export declaration in SAP GTS. You send an electronic export message to the US customs server and receive a registration number.
You do not complete the pro forma.
You create a second pro forma billing document for both deliveries and specify a third delivery.
You expect the system to update the existing customs declaration and retain the registration number.
Until now, you have had to cancel the customs declaration so that the system could create a new customs declaration, and the registration number was not retained by the system.

Option 3)
You create a pro forma billing document for two deliveries in the feeder system. The system creates a customs export declaration in SAP GTS. You send an electronic export message to the US customs server and receive a registration number.
You do not complete the pro forma.
You create a second pro forma billing document for one of the two deliveries.
You expect the system to update the existing customs declaration and retain the registration number.
Until now, you have had to cancel the customs declaration so that the system could create a new customs declaration, and the registration number was not retained by the system.

Other Terms

Export corrections, USA, X12, proprietary format, M0410, X.12, ITN

Reason and Prerequisites

New functionality

Solution

Implement the following SAP Notes:

  • 3083340 - ABAP Dictionary
  • 3083342 - Customizing
  • 3087586 - Constant include
  • 3094759 - API for creation/changing a customs export declaration
  • 3085276 - API for change management
  • 3087428 - Customizing and default data
  • 3095222 - Customizing
  • 3091426 - Inbound processing
  • 3087585 - Message function does not take new cancellation message into account
  • 3091572 - Acceptance message after sending of cancellation message always cancels customs declaration
  • 3087973 - US-specific data retrieval is not processed for new electronic messages
  • 3087974 - US-specific incompleteness check is not processed for new electronic message
  • 3088006 - US-specific data retrieval is not processed for new electronic messages
  • 3091426 - Inbound processing does not delete COCO document
  • 3093378 - Incoming message does not set a status value upon acceptance of cancellation
  • 3094340 - OITN number is not filled for change message

Symptom

When you run transaction code /US4G/MEMI_WRITEOFF (program: /US4G/RP_MEMI_WRITE_OFF) with mode 'Write-Off Doc. per MeMi Doc.', only the first MeMi document is written off. The other MeMi documents remain in their previous status because the whole FICA document is cleared.

This issue is relevant for the market role: DSO.

Other Terms

/US4G/MEMI_WRITEOFF, /US4G/RP_MEMI_WRITE_OFF, FICA document, MeMi document, write-off, status 89

Reason and Prerequisites

Program error.

Solution

Implement the corrections of this SAP Note or import the relevant support package.

After implementing this SAP Note, the system writes off MeMi documents one by one as intended.
You can verify the fix by checking that every selected MeMi document for write-off can be processed to status 89.

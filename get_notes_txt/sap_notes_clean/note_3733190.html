Symptom

After creating a Sales Contract and approving then, even though the field "Overall Status" in the Sales Contract was changed to "Completed", the CDS View I_SDProcessStatus in the Customer Data Browser app still shows the status as "Open"

Environment

SAP S/4HANA Cloud Public Edition 2602

Reproducing the Issue

  1. Open an existing sales order using the "Change Sales Documents" app.
  2. Create a new order item linked to a quantity contract using the 'Create with Reference' button, the 'Outline Agreement popup', or 'Assign Contract', and save the order. Keep the "Change Sales Documents" app open.
  3. In a new window, add a new last item to the quantity contract.
  4. Return to the original window in the step 2, continue editing the sales order by creating a new order item referencing the newly added contract item using 'Create with Reference', the 'Outline Agreement popup', or 'Assign Contract', and save the order.

Cause

This is a known error already being fixed by SAP Development teams. 

A correction is planned in release 2608.

This fix is expected to reach customer Prod systems on August 19.

Please note that SAP may make changes to the release content without prior notice. You can find the latest information on the S/4HANA upgrade schedule at the Upgrade & Maintenance Schedule page.

Resolution

To avoid this issue, follow the instructions below: 

  1. Add a new dummy contract item in the contract and save it.
    • The overall status in CDs view will be updated.
  2. Delete the new dummy contract item and save.

    Before proceeding with the correction delivery, close the Change Sales Documents window used after step 2.

    Open a new window to create a new order item that references the newly added item in the quantity contract.

    This Workaround is for existing contract items whose overall status is not correct in CDs view I_SALESCONTRACTITEM in the Customer Data Browser app.

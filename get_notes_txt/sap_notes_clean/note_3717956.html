Symptom

During the process of eInvoice PUSH operation in PEPPOL DRC Network, the system raises the following error: "<Message ID> => Error at PUSH operation, MPL: <MPL ID>". Additionally, the following message is displayed: "com.sap.it.xml.validator.exception.XSDSchemaValidationException: Validation failed for: /xsd/generic-push-service.xsd errors: [org.xml.sax.SAXParseException: The content "DRC-Destination-Prod" of element >system< does not match the required simple type. Value "DRC-Destination-Prod" contravenes the maxLength facet "10" of the type Q{}systemType, Line : <The exact position in the XML payload where the error occurred>]. Exchange[<Exchange ID>]"

Environment

  • SAP S/4HANA Cloud Public Edition
  • SAP Document and Reporting Compliance, Cloud Edition

Reproducing the Issue

  1. Open SAP BTP Sub Account.
  2. Navigate to Subscriptions.
  3. Access the Application.
  4. Select PEPPOL Exchange.
  5. Open Monitor Documents.
  6. Select the document.
  7. The error is displayed. 

Cause

The unique name that identifies the business system exceeds 10 characters in the Destination configuration of the SAP BTP subaccount.

Resolution

  • If the PUSH operation fails due to an XSDSchemaValidationException indicating that the system name has exceeds the allowed length, ensure that the unique business system name maintained in the Destination configuration of the SAP BTP subaccount does not exceed 10 characters. 
  • Detailed instructions on setting up connectivity is explained in the following help documentation: Setting Up Connectivity.

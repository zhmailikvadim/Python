Symptom

While recording results for inspection lots with inspection points, an unexpected invalidation takes place for each characteristic results, except the ones in the last sample / inspection point.

The results recording can take place directly - via transaction QE51N, QE11, and so on -, via BAPI BAPI_INSPOPER_RECORDRESULTS, via MFG-DM integration, and so on.

The attribute '/  Invalid result' can be seen in table QASR, field ATTRIBUT. The user might see following value: 'Result not current (invalid)'.

Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.

Environment

  • SAP ERP Central Component
  • SAP ERP
  • SAP Enhancement package for SAP ERP
  • SAP S/4HANA, on-premise
  • SAP S/4HANA Cloud Private Edition

Cause

Configuration issue.

Resolution

To avoid such invalidation of results, deactivate the indicator 'Last Inspected Sample Is Valid' (V_TQ79-LAST_VALID) in following customizing path for the affected inspection point identifier:

IMG(transaction QCC0 or SPRO)
>Quality Management
>>Quality Planning
>>>Inspection Planning
>>>>General
>>>>>Define Inspection Points

Example:

***

To find the relevant inspection point identifier for an inspection lot, call transaction SE16 for table QALS, where

  • PRUEFLOS is the 'Inspection Lot Number', and
  • SLWBEZ is the 'Identifier for Inspection Point Field Combination'.

Symptom

  • Managers are unable to change the due date for instructor-led courses in the new manager view.
  • Manager cannot locate the course in the team view to adjust the due date for the user even though the course was manager-assigned as mandatory with a due date.
  • Due to this users are receiving the expiration notifications even though the class is still not due as the due date at the item level is not changed.

Images/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.

Environment

SAP SuccessFactors Learning

Reproducing the Issue

  1. Assign an instructor-led training (ILT) course to a user as a required course from the "My Team" view.
  2. Log into the instance as the user and verify the item appears under the "Required" section.
  3. Schedule a class to the above mentioned ILT course
  4. Now self-enroll as a user to the class created above.
  5. Navigate to the "My Team" view as the manager.
  6. Attempt to change the due date for the required course under "Change Due Dates" in the "My Team" view. Observe that the required item does not appear.

Cause

As per the PM confirmation this is expected behavior as per current application design. 

Resolution

The change due date functionality only applies to changes at the course (item) level, not at the class level, which is why when a class is associated to the mandatory item, the manager cannot change the due date.

Best Practise

If you are planning to have a scheduled class for an ILT course, then plan upfront that while assigning the course to the user please make sure that the required date of the item is set for a long time so that you can then have classes scheduled accordingly within the required date.

Work Around

In order to stop the overdue (or) expiration notifications being sent to users & managers, from the learning admin side please uncheck the boxes as highlighted in the image below, for both the users & managers(at the individual item level) & click on "Apply Changes" to save the changes.

This would stop the notifications being sent to the users & managers for the specific item.

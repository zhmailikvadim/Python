Symptom

To comply with General Data Protection Regulation (GDPR) policy, personal data should be processed lawfully, transparently, and for a specific purpose.

Once that purpose is fulfilled and the data is no longer required, it should be deleted.

Other Terms

GDPR, User-Related Data, Background Process, Delete User-Related Data from WIP/ODS, Aging

Reason and Prerequisites

User ID is stored in some SAP ME database tables. Also User ID is used for Labor logging and it is treated as personal data.

To comply with GDPR, automatic data deletion mechanism is required so that data gets automatically erased after a retention period is reached.

Until the retention period is reached information needs to be available for traceability requirements, A&D or medical FDA requirements.

Solution

The solution is provided in the form of a code change.

A new background process (Delete User-Related Data from WIP/ODS) is implemented to delete user-related data after the retention period (WIP/ODS Retention Period) is reached.

User-related data includes the following:

  1. Any records in WIP and ODS schemas containing User ID reference (USER_ID or USER_BO).
  2. SAP ME user records if the following conditions are met:
    • Corresponding SAP NetWeaver UME record is deleted.
    • There are no records in ACTIVITY_LOG and AR_ACTIVITY_LOG tables created during the retention period and containing this User ID.  

The background process can be enabled and maintained only at Global (*) site by users with administrators permissions.

By default, the retention period is set to 30 years (10950 days) in a new ODS rule (WIP/ODS Retention Period). It can be maintained only at Global (*) site.

NOTE: To make the rule available, execute CTC tasks "SAP ME Configuration for Upgrade" or do reload initial data on created sites.

CAUTION: Do not make the retention period too short. It will cause data inconsistency. See recommendations section for details.

Recommendations:

  • Backup your databases regularly.
  • Archive your data regularly. 
  • Make archiving period shorter than WIP/ODS retention period.

Impact

The code change made to address the symptom above may impact the following:

  • Background Process Execution
  • ODS Rule Maintenance 
  • Production and Master data containing User ID information

Database records with User ID will be deleted, but related (child) records can remain untouched, as they don't have User ID reference.Typically, the child records share the same creation date or are dated within a small number of days while they are Work In Progress. Records containing a user ID are transactional not master data.
Providing the customer sets the deletion threshold to be greater than the oldest needed record plus a small buffer margin, data integrity of all actively used / reported records will be fully maintained.
Even if a small number of records very close to the cut point become disassociated with deleted related records it will not have any impact on records not at the margin, so risk is minimal providing the parameter is set correctly.

Labor Logging Personal data is movied to ODS storage by ODS Labor Log background job and then will be deleted by Delete User-Related Data from WIP/ODS background job.

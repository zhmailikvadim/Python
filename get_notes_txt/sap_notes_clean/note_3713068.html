Symptom

  • The PDF visualization in the Inbound Solution is not working when displaying through EDOC_COCKPIT but is functioning correctly when using the Fiori App "Manage Electronic Documents".
  • Error message displayed in EDOC_COCKPIT: "The action is not allowed for any of the selected eDocuments".
  • Message No. EDOCUMENT094.

Environment

  • SAP S/4HANA
  • SAP ERP
  • SAP Document and Reporting Compliance Service
  • Document Compliance Poland

Reproducing the Issue

  1. Access EDOC_COCKPIT.
  2. Select the respective source key.
  3. Attempt to display the PDF.
  4. Observe the error message: "The action is not allowed for any of the selected eDocuments" (Message No. EDOCUMENT094).

Cause

Missing PLKSEFIN entry in the view SM30 EDOCOMPPROCCOMV.

Resolution

  1. Update Configuration: Ensure that the PLKSEFIN entry is properly maintained in the view SM30 EDOCOMPPROCCOMV as described in Defining Process Communication Through Cloud Services | SAP Help Portal 
  2. Retest Scenario: After updating the configuration, create a new supplier invoice and test the scenario again to confirm that the PDF displays correctly.
  3. Important Note: PDF functionality will only apply to newly created eDocuments following the configuration update. Supplier invoices (eDocuments) previously created using version 1 will not display the PDF.

See Also

Refer to SAP Help: Defining Process Communication Through Cloud Services.

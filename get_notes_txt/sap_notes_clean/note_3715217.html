Symptom

After upgrading to a new SAP_BASIS Support Package or after applying note 3576856 some external commands do not work as before. The cause is that external commands do not have the working directory DIR_HOME anymore (i.e. the directory, which is contained in the profile parameter DIR_HOME).

Moreover, this note improves the solution given by note 3669653.

Other Terms

External commands, SM49, SM69

Reason and Prerequisites

The described symptom is a side effect of some improvements. This is actually not a bug. It has never been guaranteed that external commands have a certain working directory. But some customer developments relied on the old behaviour and, for example, called external commands which created or accessed files without path names.

Solution

If you are affected by the issue, please apply this note. If your SAP_BASIS release is 8.16, please apply the note 3714577.
After that, DIR_HOME is again the working directory. Another condition for DIR_HOME being the working directory of external commands is, that the profile parameter rfc/use_gwstart is set to 0.
This is the case in most systems.

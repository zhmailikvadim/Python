Symptom

 This Knowledge Base Article (KBA) provides guidance on setting up and verifying a Defined Benefit (DB) plan within monthly payroll processes and Year-End Adjustment (YEA) in Payroll Japan.

Environment

  • Human Capital Management (HCM)
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for HANA
  • SAP S/4HANA
  • SAP S/4HANA Cloud Private Edition
  • SAP SuccessFactors Employee Central Payroll

Resolution

To implement and validate a DB plan in monthly payroll and YEA, follow these steps:

  1.  Create Your Own DB Plan: 
    Navigate to the SAP Reference IMG (SPRO) by following this path: Personnel Management -> Retirement Benefit System (JP) -> Corporate Pension Plan
    For instance, create a plan named "ZXXX."
  2. Assign the calculation basis type. For this example, use "Fixed amount. SPRO -> Personnel Management -> Retirement Benefit System (JP) -> Corporate Pension Plan -> Basis Payment Calculation -> Define Calculation Basis Parameters
  3. Maintain IT0889 Record for DB Plan:ZXXX
  4. Execute Monthly Payroll:  After execution, verify that wage types /Q31 and /Q32 are generated in the results table (RT) without any deductions in monthly payroll.
  5. Check in PC_PAYRESULT to ensure the generation of the table JP_CPT, which stores the DB plan's information.
  6. Run Year-End Adjustment (YEA) in the end Verify that deduction wage types, such as /YS5, are generated in the results, indicating the deductions was applied in YEA process.

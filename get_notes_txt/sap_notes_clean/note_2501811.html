Symptom

When user taps on the Open button under the context menu in Twitter section of the Design Studio Dashboard, it does not open the URL in SAP BI Mobile App 6.6.3

“Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental."

Environment

  • SAP BusinessObjects BI Platform 4.1 Support Pack 7 Version: 14.1.7.1853
  • Design Studio Dashboard
  • SAP BusinessObjects Mobile App 6.6.3

Reproducing the Issue

  1. Create a Design Studio Dashboard having Twitter links based on Rapid Mockup components.
  2. Login to SAP BI Mobile App 6.6.3 and open the Design Studio dashboard.
  3. Once the dashboard is open, tap ON the slider under the Twitter tab on the right side.
  4. Now, press and hold on any name under the Twitter section till the context menu pops up.
  5. Tap on the Open button.
    Observation: Ideally it should open the Twitter url or page within the SAP BI Mobile App (as seen in version 6.5.16) but post upgrade to SAP BI Mobile App 6.6.3, it does not open the Twitter url or page.

Resolution

  1. Please login to CMC and navigate to Applications -> SAP BusinessObjects Mobile -> Client Properties.
  2. Add the parameter: feature.enable.legacy.webview
  3. Set the value to "true".
  4. Click "Save and Close". 



  5. Log out of SAP BI Mobile App and log back in again. Perform the same workflow.

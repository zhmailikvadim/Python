Symptom

It's observed that the 'mapping of project id' has trailing zeros for migration object 'Enterprise Project', even though it was not given in the template.

Environment

SAP S/4HANA Cloud Public Edition

Reproducing the Issue

  1. Open application Migrate Your Data.
  2. Select the respective migration project.
  3. Select the migration object 'Enterprise Project'.
  4. Goto mapping task.
  5. Check 'Mapping of project ID'.

Cause

  • Project IDs starting with "P*" uses the coding mask /000000000-0000. 
  • The issue occurs in the mapping task due to the configuration maintained in the code masking (SSCUI: 102881).

Resolution

  • The trailing zeros are determined by the configuration defined in SSCUI '102881 – Define Project Coding Mask.' Accordingly, zeros are appended to the Project ID Mapping based on this setup.
  • Therefore, ensure the configuration is maintained correctly in line with the business requirements.

Note: This has no impact on the migration. The value maintained in the relevant template will appear exactly the same in the target application.

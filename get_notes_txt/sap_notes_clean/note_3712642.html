Symptom

MAIF meter replacement process does not copy rate data from the old device to the new device. This note addresses this issue in two ways, one it introduces default mechanism to copy the simple rate data which consists of rate fact group, rate type and periodic consumption from the old meter to the new meter. This option needs to be enabled by following the post installation instructions. Should a more in depth mechanism be needed a new Badi has been introduced as part of this note which allows for custom logic be executed so the appropriate rate data is added into the ISU structures. The badi name is /MISU/DEVICE_RATE_DATA.

Other Terms

MAIF, Meter, Meter Replacement, Badi /MISU/DEVICE_RATE_DATA, rate data

Reason and Prerequisites

Enhancement to existing functionality

Solution

Apply the attached correction instructions, subsequently the post installation manual instructions

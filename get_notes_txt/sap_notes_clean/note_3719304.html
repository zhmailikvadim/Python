Symptom

  • Interview slots displayed as available in interview Scheduling page do not match the actual availability in Outlook Calendar for the selected interviewer. 
  • Scheduled interviews are not being removed from the interviewer's Outlook Calendar when deleted in Interview Scheduling.

Environment

SAP SuccessFactors Recruiting Management 

Reproducing the Issue

  1. Try scheduling an interview in Interview Scheduling page.
  2. Observe that the availability slots shown for interviewers do not match their actual availability in Outlook Calendar for selected interviewers.
  3. Delete a scheduled interview.
  4. Verify that the corresponding meeting remains in the interviewer's Outlook Calendar despite being deleted.

Cause

Partial migration made to Work Tech Configuration. The legacy configuration for Outlook integration under "Set up Interview Scheduling Outlook Integration" has not been completely removed, causing conflicts with the migrated Work Tech Configuration.

Resolution

  1. Navigate to the “Set Up Interview Scheduling Outlook Integration” page in SAP SuccessFactors.
  2. Remove all populated details from the page.
  3. Ensure that the “Establish Outlook Integration for Interview Scheduling” option is unchecked.
  4. Turn off the currently selected Exchange Online option and save the changes.
  5. Perform an OData Metadata Refresh.
  6. Navigate to “Manage Recruiting Settings” and access the “Interview Settings” section.
  7. Test the connection. If an error occurs due to missing service account configuration, proceed to the next step.
  8. Configure the service account details correctly under Work Tech Configuration and establish a successful connection.
  9. Perform another OData Metadata Refresh.
  10. Return to “Manage Recruiting Settings” → “Interview Settings” section.
  11. Check the option “Enable Outlook Integration” and save the changes.
  12. Perform one more OData Metadata Refresh.
  13. Retry the interview scheduling process to confirm resolution.

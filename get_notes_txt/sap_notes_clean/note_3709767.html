Symptom

  • Following the guidelines of KBA 3524757 Standard Texts are included in the print form but they are not transferred to the external system in the EDI message.
  • The configuration for mapping outbound texts does not include the standard text type. 

Environment

SAP S/4Hana Public Cloud Edition 

Reproducing the Issue

  1. Configure standard texts (TEXT ST) in configuration node 'Define Texts for Purchase Order' (SSCUI ID 102270).
  2. Attempt to include the standard text type in 'Define Mapping for Outbound Texts for Integration with Suppliers' (SSCUI ID 102290).
  3. Validate the configuration and observe that the standard text type is not included in the EDI payload.

Cause

Standard Texts are currently not supported for EDI transmission  

Resolution

It is possible to include header and item texts F01 F06 etc in the EDI transmission but not standard texts.
The XML Text ID can have mapping with only one Text ID from the P.O
Example  
1) Include text type shipping instructions 'F06' in the configuration node 'Define Texts for Purchase Orders' (SSCUI 102270)    
2) Include text type shipping instructions 'F06' in the configuration node 'Define Mapping for Outbound Texts for Integration with Suppliers' (SSCUI 102290)

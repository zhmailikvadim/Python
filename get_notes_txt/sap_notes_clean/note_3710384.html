Symptom

When running VF04 without filters, certain SD documents appear in the list. Let's assume that documents appear with Sales Organization (VKORG) 'AA00'.

When this Sales Organization filter is added (i.e. VKORG = 'AA00') on the VF04 selection screen as an extra selection criteria, the documents already do not appear after execution.

Environment

  • Sales And Distribution (SD)
  • SAP R/3
  • SAP R/3 Enterprise 4.7
  • SAP ERP Central Component
  • SAP ERP
  • SAP Enhancement package for SAP ERP
  • SAP Enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA
  • S/4HANA Cloud Private Edition

Reproducing the Issue

  1. Open transaction vf04.
  2. Run VF04 without any filters. SD documents appear in the list, e.g. with Sales Organization 'AA00'.
  3. Apply filter Sales Organization 'AA00'.
  4. The list is empty, and the SD document is not displayed.

Cause

The possible reason is due to custom code / userexit, being executed during the VF04 process. In form VKDFS_DATA_DISPLAY (SDBILLDL), there is a userexit possibility: 

  CALL FUNCTION 'SD_VF04_CALL_CUSTOMER_FUNCTION'
    TABLES
      c_vkdfif = p_gt_fvkdfi
    EXCEPTIONS

If the content of the above table is being modified there, then end result of VF04 will change accordingly.

Resolution

  1. Identify if custom code / userexit is being executed during the VF04 process.
  2. Review the logic within the userexit to determine if it modifies relevant tables (p_gt_fvkdfi, c_vkdfif, gt_fvkdfi).
  3. Adjust the userexit if necessary.

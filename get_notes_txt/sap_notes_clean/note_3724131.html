Symptom

When creating or copying a Basic Pay (IT0008) record in the HRPAO_PAOM_MASTERDATA application or via Employee Action, changing the Percent of Capacity (BSGRD) value causes the Indirect Valuation Flag (INDBW) to be cleared after saving, even though the wage type amount is saved correctly. This issue only occurs when Dynamic Processing Rules are enabled for IT0008.

Other Terms

Basic Pay, Dynamic Processing Rule, DE, DPR, IT0008, INDBW, indirect valuation flag blank, Percent of capacity, BSGRD

Reason and Prerequisites

Program Error.

Solution

Implement the Correction Instructions attached to this note or upgrade to relevant support pack.

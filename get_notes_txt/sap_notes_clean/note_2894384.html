Symptom

  • Using the RESTful Web Service to copy a Web Intelligence document, a "[1]" might be wrongly added to the name of the copy if it shares the first characters of another document (even if the name in its entirety is different).

Other Terms

Webi, webintelligence, Raylight, numbering, bracket

Reason and Prerequisites

Environment

  • SAP BusinessObjects Business Intelligence Platform 4.2 Support Package 07
  • Web Intelligence 
  • Raylight

Reproducing the Issue

  1. Pick a Web Intelligence document you want to copy, for example "My document (Original)" whose id is 1234.
  2. Copy it using Web Intelligence RESTful Web Service by choosing a destination name which is a leading substring of the original one:

call:

[POST] raylight/v1/documents?sourceId=1234

Accept: application/xml

request body:

<document>
    <name>My document</name>
</document>

response body:

<success>
    <message>The resource of type "Document" with identifier "5678" has been successfully created.</message>
    <id>5678</id>
</success>

  1. Get the details of the newly created Web Intelligence document:

call:

[GET] raylight/v1/documents/5678

Accept: application/xml

response body:

<document>
    <id>5678</id>
    ...
    <name>My document[1]</name>
    ... 
</document>

>> "[1]" has been appended to the user chosen name even if no "My document" Web Intelligence document exists in the same folder.

Cause

  • This issue is caused by a defect in the product.

Solution

This issue is fixed in the patches listed in the "Support Packages & Patches" section below.
Due to different internal processes, the "Support Packages & Patches" section of the SAP Note will be populated as below:

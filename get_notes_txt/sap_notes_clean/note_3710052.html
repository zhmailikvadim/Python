Symptom

For the personal number assigned to the work center which comes from the plant for which the object SERVICECALL is not defined in /PACG/ECM_PLAS, the region determined based on that work center is only added to the IDoc with EMPLOYEE data. In the idoc for the ERPUSER the region data is missing. As on the FSM side the information for the ERPUSER has a priority over the EMPLOYEE one, then the information about the region is not displayed there.

Other Terms

FSM, E4C, Connector, Work Center, /PACG/ECM_PLAS, /PACG/ECM_P_GR, /PACG/ECM_IS_REGIONDATA

Reason and Prerequisites

This is caused by missing 'iv_compid' parameter in method IDOC_OUTPUT of class /PACG/ECM_CL_S_IDOC_O_EMPLOYEE.

Solution

Please, implement this correction instruction.

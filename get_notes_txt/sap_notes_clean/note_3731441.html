Symptom

  • Error message 3G203 "Account with the following search key was not found".
  • This error starts to appear after migrating from an older release to S4HANA / H4S4 when with the same scenario, the error did not use to happen in the older system.

Cause

  • This issue started to appear when zero-value line items were included in documents as part of the standard behavior in H4S4. In certain cases, wage types aggregate to zero at the symbolic account level. Historically, these did not require an assigned G/L account, as they were only relevant for ensuring overall posting balance.
  • However, once zero-balance line items are displayed, the system now expects every symbolic account to have a corresponding G/L account. As a result, organizations must either maintain G/L account assignments for these cases or activate the switch to suppress zero-balance line items. 

Resolution

Please refer to note 0003464949, specifically the last question. You have three possible solutions:

  1. Set switch POST/PZERO to avoid zero line items
  2. Delimit the V_T52EL and V_T52EZ entry (handle with care, as it could lead to posting imbalance)
  3. Define an account for the symbolic account in the receiving system

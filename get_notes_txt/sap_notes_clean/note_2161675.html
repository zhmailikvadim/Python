Symptom

When trying to setup availability monitoring for TREX using the ccms_config.py script with option --grmg you run into following issues :
1)  The script failed and don't generate the XML scenario file for upload
2)  the generated file inconsistent and can not be uploaded by TREX GRMG server   or
by hand in tc. GRMG  (in this case error “No component found - no GRMG_COMPONENTS entry made” displayed )
The issue can be only reproduced on TREX 710 REV 58 - REV 63

Other Terms

TREXgrmg.py ccms SAPCCMSR GRMG_COMPONENTS

Reason and Prerequisites

this is the bug in the TREX functionality

Solution

Issue is resolved with the TREX 710 REV 64 . As workaround please replace the ccms_config.py with the attached one

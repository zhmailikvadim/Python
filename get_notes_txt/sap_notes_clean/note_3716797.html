Symptom

You have one of the following two symptoms:

1.  The runtime error CONVT_OVERFLOW exception CX_SY_CONVERSION_OVERFLOW occurs in the class FI_TAX_SERVICES2 and the method ADJUST_EXCHANGE_RATE.

2. You notice that the default value for the tax rate (on the tax screen when you enter the FIN document) matches the translation ratios of the exchange rate type from the BTE and not the exchange rate type of the document. However, during the calculation of the tax amount in local currency using the tax rate, the exchange rate type of the document is used. As a result, the calculated tax amount is incorrect by a factor of 10 or 100 or....

Other Terms

  SG121, HWSTE, LWSTE, LWBAS

Reason and Prerequisites

In the global parameters of a company code, you have specified that the tax items are translated into the local currency with a different exchange rate for postings in foreign currency with the parameter value T001-TXKRS = 1 (manual exchange rate entry possible).

You use the business transaction Event (BTE) 2051 to determine a proposal for this alternative tax rate (BKPF-TXKRS). Either with your own implementation and use the method adjust_exchange_rate (SAP Note 2982727) or with the delivered standard function FI_TAX_RATE_DETERMINE_HU. In this, you do not evaluate the normal exchange rate type of the document, but a different exchange rate type. For example, in FI_TAX_RATE_DETERMINE_HU, the exchange rate type T005-KURST that is defined in the settings for the country. You use different translation ratios for the two exchange rate types, for example, once 1:1 and once 100:1.

For one of the two exchange rate types, the translation ratios are different depending on which currency is used for the calculation, for example: EUR -> HUF with factor 1:1 but HUF->EUR with factor 100:1.

They have an inverted rate. This is usually the case for exchange rate type EURX, for example.

If you use different exchange rate types in SAP FIN, these should always have identical translation ratios. You receive this recommendation in message SG-121 in the currency settings.

You have implemented SAP Note 3617396 in your system.

Solution

Implement the attached correction instructions.

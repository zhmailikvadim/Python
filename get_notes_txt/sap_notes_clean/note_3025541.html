Symptom

The country version Switzerland currently supports ELM (standardized wage notification procedure) for ELM Version 4.0 only.
You require information about the content and availability of the planned changes for ELM Version 5.0.

Other Terms

Payroll Switzerland
Employee Central Payroll (ECP) / SAP SuccessFactors
Standardized wage notification procedure (ELM)

Reason and Prerequisites

This SAP Note provides information about the planning and delivery of the program changes required to transfer wage data in accordance with the new Version 5.0 of the uniform wage notification procedure (ELM) in the HCM country version Switzerland.

This SAP Note is currently relevant for you only if you want to find out about the changes planned in the future.

Details:

As of today (March 8, 2021), you do not need to take any urgent action: Swissdec is planning on being able to receive productive wage notifications in accordance with Version 5.0 only as of 2022 at the earliest. From this date onward, it is intending to allow parallel operation with the currently supported Version 4.0 for a transition period. Swissdec is planning for the final version of the complete specification required for certification (including certification test cases) to be available as of Q2/2021 at the earliest.

As a customer, you will still be able to perform a transfer using Version 4.0 after the formal validity period of the existing ELM 4.0 product certification for SAP has expired1. In consultation with Swissdec, instead of working toward recertification of the old standard 4.0, we are aiming to provide a new development and achieve certification for Version 5.0 as a next step.

We are planning to continually update this SAP Note with details of changes/further developments in the planning and partial delivery of (possibly testable but not productively usable) updates until a certified and productively usable status is achieved.

Update on 2/21/2022:

The complete specification required for certification, including certification test cases for ELM V5.0, is finally available in the final version in accordance with information from swissdec dated February 4, 2022.
We are currently planning to sign the certification agreement with swissdec in the first half of 2022 with the aim of completing the certification for ELM 5.0 by the end of Q2/2023, so that you can implement and test probably as of Q3/2023, and report productively with ELM 5.0 as of January 1, 2024.
This planning can stilll shift in both directions, depending on the general resource situation/certification availability at swissdec, our implementation progress, and any other changing framework conditions, such as short-term legal changes.

Update on 8/29/2023:

  • The SAP HCM localization Switzerland has been ELM 5 certified since calendar week 34/2023 (onPremise and Cloud products (EC, ECPay)).
    This formally entitles you to participate in the ELM 5.0 wage notification procedure with the SAP software.
  • With related SAP Note 3367097, you receive the electronic ELM 5.0 transmitter certificate issued by swissdec for SAP.
    This also enables you technically (together with the other extensive program changes) to send a correctly signed ELM 5.0 notification (as well as status requests, result requests, and responses to dialog requests) to the distributor and to decrypt the received responses.
    SAP Note 3353448 contains rudimentary information about how to configure the transmitter in advance.
  • With the previously published ELM 5.0 SAP Notes, you mainly receive the status used for the certification tests.
    The list of SAP Notes in the "Solution" section under "Previous Partial Deliveries" has also been enhanced with this SAP Note update. 
  • However, we still expect post-implementation steps/subsequent deliveries:
    • This week (CW35, 2023), swissdec plans updates to the wage standard description ELM 5.0, which may still require subsequent deliveries (in particular, in the areas of cross-border employees, wage statements, and concurrent employment).
    • We are currently working on an update of the Implementation Guide with a separate new chapter for ELM 5.0, which you can use to set up ELM 5.0 completely (planned SAP Note number: 3276736). We plan to provide you with the next HR Sync Package at the latest (planned availability according to HR SP Schedule: 11/9/2023).
  • We therefore generally recommend that you schedule the import of the next HR Sync Package and then configure ELM 5.0 using the implementation guide that will then be available.

    Due to the scope and incomplete documentation, we do not recommend that you make use of the theoretical option to implement and set up ELM 5.0 with SAP Notes. However, if you want to test in advance, you should use a current HR Support Package so that most of the ELM 5.0 SAP Notes are already contained. You must also expect that you may have to identify missing Customizing entries and master data using the explanations in the SAP Note texts (or even using the debugger) as long as the IMG is not yet complete (that is, only possible for technically very experienced persons). 

  • Note on "Minor-Version" ELM 5.1: As described in the document "Addition to Guidelines for Wage Standard CH (ELM) Version 5.1" of swissdec, the difference to the "Major-Version" ELM 5.0 or ELM 5 with regard to the electronic notification consists only in two additional fields in the annual notification of the domain AHV/ALV ("AHV Payroll"). These fields are to be transferred for the first time as of 1/1/2024 and are related to the revision "AHV21": 

    • AHV Basis (XML tag <AHV-AVS-BaseSalary>)
    • Information that the exemption amount was waived (XML tag <WaiveOfPensionDeduct>)

    We are planning to deliver this adjustment (among other changes required as of 2024 as part of AHV21, in particular: Payroll) with the year-end legal change 2023/24 (see SAP Note 3310430). 
    Note the following: For Minor-versions as opposed to Major-versions, swissdec provides only for tests of the new features, but not for a new complete certification (also see the document "Concept Release Process for Standards" of swissdec).
    In particular, the transmitter certificate that has been technically set up and used for signature and decryption for ELM 5.0 remains the same, as well as the transmitted certificate number for all Minor releases of ELM 5.0 (including ELM 5.1).

Update on 11/22/2023:

The program changes for Version 5.1 / AHV21 and final enhancements to the Implementation Guide for ELM 5 will be contained in the HR Support Package for the year-end legal change Switzerland 2023/2024 (see List of HR Support Packages in the "Symptom" section of SAP Note 3310430).

The new Implementation Guide for ELM 5 is available only by HR Support Package:

    • The HR Sync Package specified in SAP Note 3276736 (available as of 11/9/2023) contains most of the program changes and a version of the ELM 5 IMG that is already sufficiently complete for configuration.
    • In addition to the year-end legal change, the HR Support Package specified in SAP Note 3310430 (expected to be available as of 12/7/2023) contains other program changes required to start with ELM 5.1 (for AHV21, for example) and slight improvements to the ELM 5 IMG for customers who already use ELM 4.0 (such as additional comments if you can skip an IMG step for ELM 5 if ELM 4.0 is already configured).

For simple configuration of ELM 5, we therefore recommend that you import at least the HR Sync Package specified in SAP Note 3276736 (better still, import the subsequent HR Support Package in accordance with SAP Note 3310430) and then use the new ELM 5 IMG to set it up. (Path: "Payroll: Switzerland -> Communication with Authorities (B2A) -> Uniform Wage Notification Procedures (ELM 5.0)"): 
If you import only up to the HR Sync Package, use transaction SNOTE to implement at least the program changes for ELM 5 contained in the subsequent package.

It may also be technically possible to use lower HR Support Package levels for ELM 5, but due to the scope of the SAP Notes to be implemented then, this is not recommended and there are related restrictions (no IMG, missing optional elements for usability such as ALV layouts) and potentially technical problems during the implementation of this SAP Note.

We plan to link later SAP Notes as part of maintenance for ELM 5 to this SAP Note as a reference.

Consulting note 3392567 ("ELM 5.0: Upgrade ELM 4.0 to ELM 5.x - Description of Changes") contains an overview of the changes between ELM 4.0 and ELM 5 from a functional point of view.

Solution

Planned content

Certification of ELM 5.0 and required changes and enhancements in the system as of and including SAP_HR 6.04.
For SAP_HR 6.00 or lower, an update to at least SAP_HR 6.04 is required (see SAP Note 2955338).

We expect the following changes:

    • B2A Manager: New document category for ELM 5.0 with updated process control 
    • Adjustments in the transmitter to enable transfers also to be made in accordance with the technical specification for ELM 5.0.
    • New function to enable "dialog messages" to be swapped with the end recipients during the notification process.
    • Extensive adjustments and enhancements to master data, Customizing, payroll, data retrieval, and XML generation for the following domains:
      • AHV/ALV
      • UVG
      • KTG
      • UVGZ
      • FAK
      • LAW/tax
      • WHT
      • Cross-border employees (new domain as of ELM 5.0)
      • Statistics

Comments:

      • ﻿As before, the domain BVG is not supported with ELM 5.0.
      • The new guidelines and specifications for ELM 5.0 can be viewed at www.swissdec.ch.

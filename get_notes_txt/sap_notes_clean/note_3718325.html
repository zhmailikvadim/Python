Symptom

When saving data after uploading or deleting an attachment in APP F2345(Manage Usage Decisions), the message "Data was saved" is displayed. However, the document does not actually get saved or deleted, and the inspection lot remains unchanged with respect to the attachment status.

Environment

SAP S/4HANA 2022 

SAP S/4HANA 2023 

SAP S/4HANA 2025

Reproducing the Issue

  1. After uploading or deleting an attachment in APP F2345(Manage Usage Decisions), after saving, the message "Data was saved" is displayed. 
  2. However, the document does not actually get saved or deleted, and the inspection lot remains unchanged with respect to the attachment status. 

Cause

Incomplete customizing and program error.

Resolution

In S/4 CORE 2022, it is needed to implement the note 3717167 in backend system, and after that follow the below customizing steps.

Whereas in 2023, note implementation is not required, but the below customizing needs to be in place.

  1. Go to SM30 transaction --> Enter the view 'V_AO_TYPE_DEF' & click on Display --> Maintain the below entry
    Attachment Object Type 'INSPECTIONLOT', SAP Object Type 'InspectionLot', TCLO Object 'QALS' & Mark the 'Is HDM Adopted' Flag(Checkbox) as 'X' --> Save it. In case if the entry already exists then check if the Is HDM Adopted' Flag(Checkbox) is marked as 'X' or not. If not then please mark it.


  2.  Go to SPRO transaction --> ABAP Platform --> Application Server --> Business Management --> SAP Object Type Repository --> SAP Object Types --> Maintain Object Representation --> Create the below 2 entries
    i SAP Object Type 'InspectionLot', Rep type 'BOR ID', SAP Object Representation 'BUS2045'
    ii. SAP Object Type 'InspectionLot', Rep type 'Key Fields of Objects', SAP Object Representation 'QALS'
    After that Save it. In case if the entry already exists then check if the above fields are filled correctly or not If not then update it. 

  3. Go to SPRO transaction --> ABAP Platform --> Application Server --> Basis Services --> Harmonized Document Management --> Activate HDM for Attachment Object Type --> Create an Entry with Attachment Object Type 'INSPECTIONLOT' & mark the 'Activate' checkbox --> Save it. In case if the entry already exists then check if the Activate checkbox is marked as 'X' or not. If not then please mark it. 
  4. Go to SPRO transaction --> ABAP Platform --> Application Server --> Basis Services --> Harmonized Document Management --> Activate HDM for Attachment Object Type --> Select the 'INSPECTIONLOT' line item & click on 'Activate Document Frameworks' --> Maintain an entry with Attachment Object as 'INSPECTIONLOT' & Document Framework as 'Document Management System' --> Save it. If an entry already exists then check if the deactivate checkbox is marked as 'X'. If yes then unmark it. 


  5. Go to SPRO Transaction --> Quality Management --> Basic Settings --> Maintain Settings at Client Level --> Under the Further Setting --> Mark the 'Active' checkbox entry for the entry for Business Process 'Quality Inspection' & Switch 'Document Link to Insp. Lot' --> Save it.

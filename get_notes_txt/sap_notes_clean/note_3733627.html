Symptom

  • Customer complains that the Fields migrated from V1 do not have Data Format into V2.
  • In Extensibility Administration > Case Service > Case > Custom Fields, migrated String or Boolean fields display an empty Field Format.
  • Repository metadata for affected String fields shows dataType "STRING" without a dataFormat entry.
  • No error messages are displayed.

Environment

  • SAP Cloud for Customer
  • SAP Sales Cloud and SAP Service Cloud Version 2

Reproducing the Issue

  1. Select the user icon.
  2. Open Settings > All Settings.
  3. Navigate to Extensibility Administration.
  4. Select extensible entity, for example, Case Service > Case.
  5. Open the Custom Fields tab.
  6. Locate a migrated custom field of type String or Boolean.
  7. Observe that Field Format is empty.

Cause

  • In V1, String fields do not have a Field Format attribute.
  • Boolean fields do not have a Field Format option in V2 in general.

Resolution

  • String fields with empty Field Format is equivalent to dataType=STRING with dataFormat=STRING; no action is required for functionality.
  • For Boolean fields, no Field Format is expected; this is by design.
  • When creating new String custom fields directly in V2, if no Field Format is selected, String will be defaulted.

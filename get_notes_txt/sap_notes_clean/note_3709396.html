Symptom

Instance key in a S4 is created with leading zero(example:INST-0123) .When this Instance is migrated to cloud, cloud stores this instance key without leading zero(for same example it will be INST-123).This creates confusion with a different id between cloud and S4.

Other Terms

/APEU, /UCOM, /APEU/RP_MCM_MIGRATION_PST_F01, MCM, DER, Migration, Leading zero, INST-0*

Reason and Prerequisites

Program error.

Release(2602)

Solution

Added validation to explicitly detect instance keys with leading zeros and do not migrate instance to cloud.

Please implement the attached the correction instruction.

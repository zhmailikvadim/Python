Symptom

The pricing result of an order item contains conditions that use the maximum condition value functionality.
This item also has a billing plan. Billing takes place for a date in the billing plan.
The net value (NETWR) of the generated billing item corresponds to the value to be billed.
The tax (MWSBP) for the generated billing item has also been calculated correctly.

However, the condition values (KWERT) of the individual conditions in the billing item do not correspond to expectations.
They do not correspond to the condition values of the order item scaled with the factor "Value to be billed / net value of order item".
The condition values of the conditions that use the maximum condition value functionality are null.
The condition values of the other conditions have been scaled with the factor "Value to be billed / (net value of order item minus condition values of conditions included in the net value that use the maximum condition value functionality)".

The incorrect behavior does not occur if the maximum condition value has already been reached for the conditions that use the maximum condition value functionality (KMXWR = 'X').


Example


Pricing result for order item:

Condition KWERT Currency Comment
Price  960.00  EUR  
Surcharge  140.00  EUR   
Discount A -200.00  EUR  (with maximum condition value, KMXWR = 'Y')
Discount B -100.00  EUR   
Net value (NETWR)  800.00  EUR  (total of active non-statistical conditions)
Tax 19% (MWSBP)  152.00  EUR  (calculated from NETWR)
Statistical condition 1  600.00  EUR   
Statistical condition 2  600.00  EUR  (with maximum condition value, KMXWR = 'Y')


The value of the billing plan date to be billed is EUR 200.00.
Based on the scaling factor of EUR 200.00 / EUR 800 = EUR 0.25, the following pricing result is expected for the billing item:

Condition KWERT Currency Note
Price  240.00  EUR = EUR 960.00 * 0.25 and fixed with KSTEU = 'E'
Surcharge 35.00  EUR  = EUR 140.00 * 0.25 and fixed with KSTEU = 'E'
Discount A -50.00  EUR  = EUR -200.00 * 0.25 and fixed with KSTEU = 'E'
Discount B -25.00  EUR  = EUR -100.00 * 0.25 and fixed with KSTEU = 'E'
Net value (NETWR)  200.00  EUR  (total of active non-statistical conditions)
Tax 19% (MWSBP) 38.00  EUR  (KSTEU = 'D', calculated from NETWR)
Statistical condition 1  150.00  EUR  = EUR 600.00 * 0.25 and fixed with KSTEU = 'E'
Statistical condition 2  150.00  EUR  = EUR 600.00 * 0.25 and fixed with KSTEU = 'E'


The system incorrectly returns the following pricing result:

Condition KWERT Currency Note
Price  192.00  EUR = EUR 960.00 * 0.2 and fixed with KSTEU = 'E'
Surcharge 28.00  EUR  = EUR 140.00 * 0.2 and fixed with KSTEU = 'E'
Discount A 0.00  EUR  = EUR 0.00 * 0.2 and fixed with KSTEU = 'E'
Discount B -20.00  EUR  = EUR -100.00 * 0.2 and fixed with KSTEU = 'E'
Net value (NETWR)  200.00  EUR  (total of active non-statistical conditions)
Tax 19% (MWSBP) 38.00  EUR  (KSTEU = 'D', calculated from net value)
Statistical condition 1  120.00  EUR  = EUR 600.00 * 0.2 and fixed with KSTEU = 'E'
Statistical condition 2 0.00  EUR  = EUR 0.00 * 0.2 and fixed with KSTEU = 'E'

In the copy process, the conditions with the maximum condition value "Discount A" and "Statistical condition 2" were set to zero. As a result, the net value of the order item (EUR 1,000) and thus the scaling factor (EUR 200.00 / EUR 1,000.00 = EUR 0.2) were calculated incorrectly.

Other Terms

Billing plan, maximum condition value,
AMOUNT, AMOUNT_RULE, KOUPD, KMXWR, KMXAW,
PRICING_COPY

Reason and Prerequisites

(1) Conditions that use the maximum condition value functionality are copied to the internal table TKOMV in the copy process (function module PRICING_COPY) with fixing KSTEU = 'E' if the maximum condition value has already been reached (KMXWR = 'X') and they have not already been fixed (KSTEU NA 'EFH'). This is necessary so that the limitation is not lost in the generated billing document as a result of a revaluation (see SAP Note 615370 Section C).
If the maximum condition value has not yet been reached (KMXWR = 'Y') and the condition value is not fixed (KSTEU NA 'EF'), the condition value is set to zero and transferred to TKOMV without fixing. The subsequent revaluation of the condition in the generated billing document calculates the condition value correctly again.

(2) In the case of the billing of a billing plan date, the system additionally calculates the net value in the function module PRICING_COPY based on the TKOMV condition values. All condition values transferred to the internal table TKOMV (except taxes) are then scaled with the factor "Value to be billed / calculated net value of order item" and fixed with KSTEU = 'E'.
Conditions with KMXWR = 'Y' already have a condition value of zero after (1). Due to the initial condition value, they can no longer receive the correct condition value due to scaling. If such conditions are also relevant for the net value, the net value is also calculated incorrectly, which means that the condition values of other conditions are also scaled incorrectly.

Solution

If the copy process is carried out for the billing of a billing plan date, the condition values of conditions that use the maximum condition value functionality must not be set to zero.

Implement the attached changes in the source code.

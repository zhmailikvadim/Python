Symptom

It is possible to save an S/4HANA Service business transaction without User Status even if it is assigned with a status profile in the customizing. This situation can occur, for example, when a user is authorized to Create a business transaction but is not authorized to Create/Change its statuses (authorization objects B_USERSTAT & B_USERST_T).

Other Terms

Status Profile, Save, Missing Status, B_USERSTAT, B_USERST_T, DB inconsistency.

Reason and Prerequisites

This SAP Note is a follow-up to Note 3498719.

The system should prevent users from saving an S/4HANA Service business transaction without a user status when a status profile is assigned in Customizing, in order to avoid database inconsistencies. If such transactions are saved, they cannot be edited or processed further.

There is one exception: this restriction does not apply if the user statuses defined in the profile do not trigger any business transaction (that is, the Trans. column in the status profile is empty).

Solution

Implement the attached correction instruction.

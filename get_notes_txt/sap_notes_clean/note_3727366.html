Symptom

  • Custom Analytical List Page (ALP) Fiori apps fail to open in the Quality system.
  • Error message: "App could not be opened because the SAP UI5 component of the application could not be loaded."
  • Error message: "Failed to load UI5 component for navigation intent '#Z-analyze'."
  • Error message: "No PresentationVariant found for qualifier: ChartPresentationVariant."
  • Error message: "Adding object 'component' with duplicate id 'application-Z-analyze-component'."

Environment

  • Fiori elements v2

Reproducing the Issue

  1. Open HTTP remote connection to the test system.
  2. Navigate to the Fiori Launchpad.
  3. Attempt to open the ALP Fiori app 
  4. Observe the error messages mentioned in the Symptom section.

Cause

  • The Analytical List Page (ALP) app fails to load due to missing annotations in the OData service metadata, specifically the "PresentationVariant" and "LineItem" annotations required for rendering charts and tables.
  • A misconfiguration caused a crash during back-navigation due to an undefined variable in the NavigationController.js file, leading to a "duplicate ID" error when attempting to re-open the app.

Resolution

The issue was fixed, and it will be released in the upcoming version 1.108.51 of of SAPUI5 in the end of March 2026

Symptom

When user attempts to access the My Inbox from the SBPA, the following error message is displayed:

Error message: "An issue occurred while connecting to service TASKPROCESSING. Please contact your system administrator with the following information: Error when connecting to service with URL: bpmworkflowruntime/v1/tcm/"

Internal logs show these errors:

Log-dbg.js:xxx 2XXX-xx-xx xx:xx:xx.xxxxx Cannot load meta data for service bpmworkflowruntime/v1/tcm/ - 403 () - HTTP request failed
<?xml version="1.0" ?><error xmlns="http://schemas.microsoft.com/ado/xxxxxx/xx/dataservices/metadata"><code>TASK_USER_NOT_AUTHORIZED</code><message xml:lang="en-US">An error occurred: Access to tasks is forbidden</message></error> com.sap.spa.inbox
u @ Log-dbg.js:xxx

Environment

SAP Build Process Automation

Cause

Missing role assignments for users, specifically the roles required to access and modify workflow contexts and task instances.

Resolution

  • User roles must be assigned through the workflow capability service instance. Authorizations for technical access are granted to OAuth clients using the OAuth 2.0 Client Credentials grant.
  • The workflow capability runtime is an SAP Business Technology Platform (SAP BTP) service that must be explicitly subscribed to. When subscribed in an SAP BTP subaccount, a corresponding service instance of the workflow capability runtime is automatically created. This service instance is then used to manage user roles and client authorizations required for workflow execution .

  • Roles for Accessing the Workflow Capability Runtime:

    • WorkflowContextAdmin (global role)  - Permission to partially modify or completely override the workflow context of a workflow instance
    • contextAdminUsers - Permission to retrieve the context of a task instance
    • contextAdminGroups - Permission to partially modify the context of a task instance

  • Above roles grant permissions such as the ability to partially modify or override the workflow context of a workflow instance, retrieve the context of a task instance, and partially modify the context of a task instance. Please read Authorization Configuration about the roles and configuration for Workflow Capability Runtime.

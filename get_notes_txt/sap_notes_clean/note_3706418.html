Symptom

A user tries to create an Ad-hoc warehouse task via tcode /n/SCWM/ADHU or /n/SCWM/ADPROD, the error "Error Could not determine ERP stock data for" happens, the message no. is /SCWM/ERPINTEGRATION105.

Environment

  • SAP Extended Warehouse Management (SCM-EWM)
  • SAP Extended Warehouse Management Warehouse Order Management (SCM-EWM-WOP)

Reproducing the Issue

Create an Ad-hoc warehouse task via tcode /n/SCWM/ADHU or /n/SCWM/ADPROD.

Cause

The destination storage type of the WT has "Availability Group" maintained in the customizing "Define Storage Type" under SCM Extended Warehouse Management -> Extended Warehouse Management -> Master Data.
However, there is no entry maintained for the combination of the "Plant", "EWM warehouse number" and "Availability Group" in the customizing "Map Storage Locations from ERP System to EWM" under SCM Extended Warehouse Management -> Extended Warehouse Management -> Interfaces -> ERP Integration -> Goods Movements for the combination of the "Plant", "EWM warehouse number" and "Availability Group".

Resolution

Move the stocks to a storage type with "Availability Group" as blank.
If the WT must be created with the issued storage type, create a case with the component SCM-EWM-WOP.

Keywords

Could not determine ERP stock data for; /SCWM/ERPINTEGRATION105; /SCWM/TMAPSTLOC; Availability Group.

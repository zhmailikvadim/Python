Symptom

In SAP HANA MDS1) you observe, that a hierarchy parent node value is incorrectly calculated in the context of a calculated measure, if unbooked readmode is enabled.

If you fully expand the parent node, then its value is shown correctly.

1) MDS: MultiDimensional Services; analytical engine in SAP HANA, responsible for processing analytic request fom SAP Analytics clienst such as SAP Analytics Cloud (SAC) or SAP Analysis for Office.

Other Terms

SAP Analytics Cloud, SAC, SAP Analysis Office, SAP Design Studio, InA/MDS

Reason and Prerequisites

Reason:

Due to a programming error in SAP HANA, the hierarchy was not internally drilled and cells of hidden children (which should have referenced constant selection values) not created, causing visible parent nodes to not be correctly aggregated along the hierarchy (i.e. the exception aggregation along the hierarchy for the formula in question).

Affected Releases:

  • SAP HANA 2.0 database, EPMMDS plugin:
    • "Fullversion EPMMDS" plugin revision: < 1.00.202614.00
  • SAP HANA Cloud Edition: 
    • < CE 2026.14.0

Prerequisites:

In a model you create the following:

  • Dimension A with a hierarchy
  • Calculated Measure CM1 with an exception aggregation (e.g. SUM) over dimension A and a formula using operands with different constant selection contexts, e.g. the following two
    • Calculated Measure M2 with constant selection on dimension B with a filter on value B_BOOKED.
    • Measure M3

The following entry exists in the datasource:

Dimension A Dimension B Measure M2 Measure M3
A_LEAF B_BOOKED 1 1

When modelling the query you:

  • put dimension A into drill with the hierarchy with readmode "unbooked values"
  • have a global filter
    • on a parent node A_PARENT (with child A_LEAF) of the hierarchy of dimension A (no drill/expansion to A_LEAF)
    • on a dimension value B_UNBOOKED of dimension B
  • have selected CM1

The value for calculated measure CM1 and node A_PARENT is then shown to be NULL (empty cell) although it should be 1, as the exception aggregation should have aggregated the value from A_LEAF.

Solution

Apply one of the following SAP HANA database Revisions:

  • SAP HANA 2.0 database, EPMMDS plugin:
    • "Fullversion EPMMDS" plugin revision: >= 1.00.202614.0
  • SAP HANA Cloud Edition:
    • >= CE 2026.14.0

Workaround:

Add an additional constant calculated measure CCM4 (e.g. "0"). Then add CCM4 as operand to CM1 by a simple addition.

Symptom

In transaction COOISPI, the system processes many XStep trees and then updates changes by running the XStep reference posting (FM CMX_XS_REF_POST) repeatedly. The system ends processing with short dumps "DBSQL_STMNT_TOO_LARGE" and "DBSQL_ARGUMENT_ERROR".

Other Terms

COOISPI, CMX_XS_REF_POST, ICMXXSIMP08, IF_CMX_XS_AGENT~SAVE, LCL_CMX_XS_REF_AGENT, LCL_CMX_XS_INSTR_AGENT, m_object_txt_mod, m_object_txt_del

Reason and Prerequisites

The reason is a program error. The logic does not refresh the multilingual text buffer tables (m_object_txt_mod, m_object_txt_del) in IF_CMX_XS_AGENT~SAVE for the reference and instruction agents in include ICMXXSIMP08, so IT_OBJECT_TXT_MOD and IT_OBJECT_TXT_DEL accumulate across repeated runs of CMX_XS_REF_POST.

Solution

Install this SAP Note. The SAP Note adds the missing refresh of m_object_txt_mod and m_object_txt_del in include ICMXXSIMP08 for the reference and instruction agents.

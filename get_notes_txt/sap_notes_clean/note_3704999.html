Symptom

  • The test connection to a CI-DS datastore connecting to a Database fails with one of the following error messages:
    • ODBC call <SQLDriverConnect> for data source <XXXXX> failed: <[SAP AG][LIBODBCHDB SO][HDBODBC] Communication link failure;-10709 Connection failed (RTE:[89013] Socket closed by peer {<Agent_IP>:<port> -> <Server_IP>:443} (<Agent_IP>:<port> -> <Server_hostname>:443))>. Notify Customer Support.
    • ODBC call <SQLDriverConnect> for data source <XXXXX> failed: <[SAP AG][LIBODBCHDB SO][HDBODBC] General error;-10719 Connect failed (invalid SERVERNODE 'XXXXX')>. Notify Customer Support.
  • Creating a DSN through DSConnectionManager.sh returns the same error.
  • The agent machine has a proxy.

Environment

SAP Cloud Integration for data services

Reproducing the Issue

  1. Launch a CI-DS agent machine configured with a proxy.
  2. Run $LINK_DIR/DSConnectionManager.sh.
  3. Configure a DSN to a HANA DB. The last step will attempt a connection test and fail.

Cause

Without the proxy details, the connection between the agent machine and the ODBC server cannot be established:

  • When using ODBC, DSConnectionManager.sh does not provide the option to maintain the machine proxy details. 
  • When not using ODBC, the CI-DS datastore does not include the options to maintain the machine proxy details. 

Resolution

Create the DSN manually and include the machine proxy details. For instance, when creating the DSN for a HANA Cloud DB with SSL:

  1. Add the following entries to $LINK_DIR/bin/ds_odbc.ini:

    1. [<DSN_NAME>]
      servernode=<HANA_Hostname>:443
      encrypt=TRUE
      sslValidateCertificate=FALSE
      Driver=<path_to_HANA_driver>/hdbclient/libodbcHDB.so
      sslMinProtocolVersion=TLS12
      sslMaxProtocolVersion=MAX
      proxyHttp=TRUE
      proxyHostname=<proxy_hostname>
      proxyPort=<proxy_port>

  2. Save the file. 
  3. Test the connection from within the Agent Machine terminal by running this command from within $LINK_DIR/bin: 
    1. isql <DSN_NAME> <User> <Password> -v
  4. Test the connection from CI-DS: 
    1. Create the datastore in CI-DS.
    2. Maintain "Use Data Source(ODBC):" as Yes.
    3. Maintain "ODBC data source name" with the DSN name created in step 1.1
    4. Fill in the remaining datastore details. Save it. Test the connection.

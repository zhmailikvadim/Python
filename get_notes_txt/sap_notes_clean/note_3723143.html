Symptom

  • Health Monitoring shows different Heap Memory than user sees on managed system
  • What is the source of Health Monitoring's Heap Memory for S/4HANA?
  • Is setting for ST02 used in determining the Heap Memory?

Environment

SAP Cloud ALM

Reproducing the Issue

  1. Logon to R/3 for the managed system 
  2. Enter transaction RZ11
  3. In Parameter, enter abap/heap_area_total and see Current Value is 16000000000
  4. Enter transaction ST02
  5. Under SAP Memory section, double click  Ext. global memory
  6. In abap/heap_area_total, adjust it to 30,000,000,000
  7. Allow time to pass and collect Health Monitoring data
  8. Launch SAP Cloud ALM
  9. Click Operations > Monitoring and Analysis > Health Monitoring
  10. In Scope, select the involved managed system
  11. In Overview, click managed system's card
  12. In ABAP Resources, click Heap Memory
  13. See there are several instances there showing Heap Memory is 16 GB for each -> expecting 30 GB based on ST02 setting

Resolution

Health Monitoring's Heap Memory is based on the Instance Profile Parameter abap/heap_area_total value.

Note: Development has discussed internally whether it would be better to use the "dynamic" Heap Memory Limit as defined in RSMEMORY or maybe additional possibilities.  As of now, the decision is to keep the limit based on the Profile Parameter because of the following reasons:

  1. The profile parameter is a validated configuration of an ABAP Instance, while manipulation with RSMEMORY (and others) are user driven and not necessarily validated
  2. When manually overwriting the Heap Memory limit, it can cause consequence for the system health

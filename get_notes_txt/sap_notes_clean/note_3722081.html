Symptom

While working with a quality notification - for example in transaction QM02 - user enters an exclamation mark '!' at the front of a character field - like 'Task Text' or 'Responsible'. At notification save the field values are cleared / deleted.

Environment

  • SAP ERP Central Component
  • SAP ERP
  • SAP Enhancement package for SAP ERP
  • SAP S/4HANA, on-premise
  • SAP S/4HANA Cloud Private Edition

Reproducing the Issue

Example scenario:

  1. Call transaction QM02 for a notification.
  2. Press ENTER.
  3. Navigate to tab 'Tasks'.
  4. Enter an exclamation mark ( ! ) before the existing 'Task Text' (or the 'Responsible' or any other character field).
    - before: Test
    - after: !Test
  5. SAVE.
  6. 'Task Text' (or the 'Responsible' or any other character-type) field is cleared.

Resolution

The exclamation mark '!' is a special character, also referred as 'reset' or 'undo' character during screen processing. It clears the field value.

Read following SAP Note for more details:

734353 - Exclamation point (!) deletes a field in the first position.

Symptom

Issue occurs while checking collective availability for planned orders and throws an error "Incorrect Setup of table ATPMATX".

Environment

SAP S/4HANA Cloud Public Edition

Reproducing the Issue

  1. Create a planned order for the material which has availability check assigned with custom availability checking group assigned to it.
  2. Go to "Change Planned Order" app (MD12) and check the created planned order.
  3. Go to the line item which has the material under the particular planned order.
  4. Observe error message "Incorrect Setup of table ATPMATX" during the process of checking availability triggered.

Cause

  • Inconsistent or incomplete entries in table ATPMATX for custom availability checking group causing rejection of availability check configuration.
  • Custom checking group not properly configured.

Resolution

  • Create a custom checking group aligning with the settings of SP or SR standard checking group and make necessary adjustments with respect to scope of check. Assign the new checking group to the relevant materials.
  • If a custom checking group is already used, change the setting of the custom checking group to “Cumulation of confirmed quantity when created and changed” (recommended to avoid inconsistencies).
  • Alternatively, delete the existing custom checking group and create a new one by copying from the SP/SR checking group. Adjust the scope of check based on business requirements and test the process to ensure proper functionality. 
  • The custom checking group should not be set to "No Cumulation." This setting means that confirmed quantities are not cumulated, and only stock is considered during the ATP check. Consequently, the system checks availability against confirmed quantities, which can lead to inconsistencies in the ATP logic

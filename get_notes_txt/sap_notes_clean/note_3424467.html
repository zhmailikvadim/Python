Symptom

When you clear document with transaction like F-53, you find that even you have define the cost center with OKB9, they are getting changed to the expense GL cost center from invoice. 

Environment

  • SAP R/3
  • SAP R/3 Enterprise
  • SAP ERP
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA Finance

Reproducing the Issue

1.OKB9 define the cost center

2.F-53 to clear the document.

3.FB03 to display the document.

Cause

The FM K_COBL_CHECK from CO is already receiving the unexpected cost center so OKB9 will not be performed.

In FORM BUCHUNGSZEILE_MERKEN from program SAPMF05A, BSEG-KOSTL is already filled by the document splitting.

Based on your settings in customizing view V_FAGL_SPLIT_FL2 column "RealExchRateDif" for field "KOSTL Cost Center". Since this setting is activated the Cost Center is transferred from the cleared item to the exchange rate difference item.

Resolution

If you don't want the Cost Center to be transferred from the cleared item you can deactivate the transfer of this field in customizing view V_FAGL_SPLIT_FL2 through transaction SM30.
Please be aware that such a change will affect all future clearings.

In case you do changes this must be checked and tested in before moving to production. If you decide to change the customizing this will effect only new documents. For old documents this will not have any effect.

And please do not do this in your PRD system directly!

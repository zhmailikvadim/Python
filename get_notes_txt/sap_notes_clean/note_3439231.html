Symptom

Some "Planned End Date" in Sourcing Request via RFQ cXML is 1 day before of "QuoteRequestHeader closeDate" in cXML in Japanese Customer environment.

Cause

"PlannedEndDate date" is set the closeDate in PST when parameter Application.AQS.UseTimezoneFromUserRequestTimezone is set as "No".

Resolution

With Application.AQS.UseTimezoneFromUserRequestTimezone parameter set "Yes", it will consider as calendar date.

Keywords

UseTimezoneFromUserRequestTimezone; closeDate; Planned End Date

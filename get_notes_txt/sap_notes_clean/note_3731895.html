Symptom

How do I identify whether a bid was submitted directly by a supplier or as a surrogate bid using the Event Management API?

Cause

The Event Management API does not expose a dedicated field in supplierInvitations or related endpoints to flag bids submitted as surrogate bids, although the system tracks this information.

Resolution

There are two alternatives you can follow:

  1. Use Event Management API audit logs:
    1. Call GET /events/{eventId}/auditLogs.
    2. Parse the audit log entries (equivalent to the event’s Log tab in the UI) to identify actions indicating that a surrogate bid was submitted.
  2. Use Analytical Reporting API with SurrogateFlag:
    1. Call GET /api/analytics-reporting-details/v1/prod/views/SupplierParticipationFactSystemView with appropriate filters (for example, updatedDateFrom/updatedDateTo).
    2. Inspect the SurrogateFlag boolean field to determine if a surrogate bid was placed for a supplier.
    3. Note: Data from the Analytical Reporting API is not real-time and typically has ~24 hours latency due to scheduled data loads.

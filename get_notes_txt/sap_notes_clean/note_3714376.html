Symptom

  • Cannot create derived table by using WITH statement after upgrading to SAP BusinessObjects 2025
  • Error occurs in validation as:
        >> Error parsing the expression: "WITH". The WITH clause is not allowed in derived tables. Use the @DerivedTable function if you want to include derived tables.. (IES 01028)
  • The same derived table works well in some older version.

Other Terms

webi, webintelligence, CTE, hana, oracle, upgrade

Reason and Prerequisites

Environment:

  • SAP BusinessObjects Business Intelligence platform 4.3 Support Package 04
  • SAP BusinessObjects Business Intelligence platform 4.3 Support Package 05
  • SAP BusinessObjects Business Intelligence Platform 2025
  • Information Design Tool
  • Web Intelligence

Steps to reproduce:

  1. Log on to Information Design tool.
  2. Create a new data foundation based on a relational HANA database connection.
  3. Insert a new derived table. Using SQL contains WITH like the following example:
    (use real table existing in database)

    WITH
    TEMP_TABLE as (
    SELECT
    Table__1."AAA",
    Table__1."BBB",
    Table__1."CCC"
    FROM
    "EMP_TEST" Table__1

    )
    Select * FROM TEMP_TABLE

  4. Click Validate button.
  5. ERROR occurs
    1. >> Error parsing the expression: "WITH". The WITH clause is not allowed in derived tables. Use the @DerivedTable function if you want to include derived tables.. (IES 01028) 

Cause:

This issue is caused by a defect in the product.

Solution

This issue is fixed in the patches listed in the "Support Packages & Patches" section below.
Due to different internal processes, the "Support Packages & Patches" section of the SAP Note will be populated as below:

  • for fixes delivered in "Support Packages": immediately (even if the "Support Package" is planned to be released several months later)
  • for fixes delivered in "Patches": ONLY once the "Patch" is released AND available on SMP (not before)

Symptom

You run a non-belgian company code with a (perhaps among others) belgian tax identification number. You want to create a (belgian) VAT return declaration for the belgian tax authorities.
In the declaration, the wrong country (code) is used in a certain field of the output file. The belgian tax authorities only except the value 'BE', independent of where your company is located .

Other Terms

RFUMSV00, FOTV, VAT return

Reason and Prerequisites

The country (code) is taken from the company address, instead of the declaration itself.
Anyway, the format is used for Belgium only, and BE is the only value which is appropriate. So, a constant can be used in the corresponding DMEE trees.

Solution

The correction will be delivered via support packages. For preliniary correction perform the following manual corrections.

Start transaction DMEE and do the following changes for the three DMEE trees listed (if they exist in your system).

UMS3/BE_EVAT_RET_XML:
Mark the node 'VATSENDING - DECLARER - COUNTRY'.
On the Attributes Tab, change the Mapping Procedure to 'Constant'. Then, on the Source Tab, enter the Constant 'BE' (without any ', just BE).

UMS3/BE_FOTV_RET_XML:
Mark the node 'ns2:VATConsignment - ns2:VATDeclaration - ns2:Declarant - CountryCode'
On the Attributes Tab, change the Mapping Procedure to 'Constant'. Then, on the Source Tab, enter the Constant 'BE' (without any ', just BE).

UMS3/BE_FOTV_SPEC_RET:
Mark the node 'ns2:VAT629Consignment - ns2:VAT629Declaration - ns2:Declarant - CountryCode'.
On the Attributes Tab, change the Mapping Procedure to 'Constant'. Then, on the Source Tab, enter the Constant 'BE' (without any ', just BE).

Symptom

  • SOW work order was revised, but there was no work item to approve it
  • The revised SOW is in status Approved, but it never went through the approval process

Cause

Statement of Work (SOW) rules may cause an approval to be unnecessary for a revision. These workflow rules are defined on the SOW template or chosen when the SOW is created

Resolution

Select the Details tab on the SOW to review the rules chosen when the SOW was created

  • When the SOW Revision rule SOW Revision approval required only when Maximum Budget is increased is set to yes, an approval is not required and a work item will not be created
  • When the SOW Worker Rule Require Approvals on SOW Workers added during SOW Response and SOW Revision Response is set to no, an approval is is not required and a work item will not be created

Rules set on the SOW Template or on the SOW Type by buyer admin

  1. If you do not see the above rules on the SOW Details tab, the buyer administrator set them via the SOW Template or SOW Type
  2. Request the buyer Program Management Office (PMO) to review the SOW template and the SOW Type for these two rules to determine if a work item should be created

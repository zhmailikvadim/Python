Symptom

If checks for user environment fail, the error message of the startdb or stopdb script is too unspecific.

Other Terms

SAP Adaptive Server Enterprise, SAP ASE, SAP Sybase ASE

Reason and Prerequisites

The environment check can fail after a system copy with system rename, if SRS (replication) was activated.

In this case, after the system copy, the setup HADR tool needs to called to setup replication again. If this is omitted the user environment with respect to the replication setup can be inconsistent.

 

Solution

Use startdb/stopdb according to this note.

Symptom

I have configured my download connector format to XML via the Configuration Manager. However, when I manually download the file from the Integration Audit Trail or the Download Data page, the file is saved with a .csv extension.

Cause

This is the expected and designed behavior of the SAP Fieldglass UI.

  • Manual Downloads: When you retrieve a file through the UI, the system defaults to a .csv extension for the download action.
  • External Calls: When the connector is triggered externally (e.g., via BTP or an API call), the file conversion occurs at a post-processor level. The system will successfully send the file in the requested XML format to the external endpoint.

Resolution

No configuration change is required as the integration will function correctly during automated runs. To verify the format manually, follow these steps:

  1. Download the file from the Integration Audit Trail.
  2. If the file has a .csv extension, right-click the file and select Open with > Notepad (or any text editor).
  3. Confirm that the internal content is structured as XML code.
  4. Perform an end-to-end test using your external system (e.g., BTP) to confirm the received file format is correct.

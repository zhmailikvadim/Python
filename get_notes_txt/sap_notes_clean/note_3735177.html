Symptom

There is a requirement to update the existing Identity Provider (IdP) to a new one used for SAML Console Login(Federated Console Login).
Is it possible to maintain the operational status of the existing IdP while configuring and testing the new IdP to avoid interruptions during the testing phase.

Environment

  • SAP Customer Data CLoud
  • Federated Console login

Resolution

If you have a test partner, the new IdP configurations can be configured and tested in the test partner before applying to the production partner so that the existing SAML Console login is not disturbed.

If you do not have a test partner, then only way to test new IdP configurations is by replacing the existing IdP configurations with the new one.

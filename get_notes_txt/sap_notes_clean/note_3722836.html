Symptom

  • Requirement to design a table with dynamic attributes in the main window in smartform creation.
  • The number of columns in the table is not fixed and may vary dynamically.

Environment

Smart Forms

Reproducing the Issue

  1. Attempt to design a smartform and create table with dynamic attributes in the main window.
  2. Set up a table where the number of columns is not fixed and may vary dynamically.

Cause

Resolution

The tables width and columns, which have been created in SmartForms (for example with the Table Painter) are fix and can not be changed during runtime.

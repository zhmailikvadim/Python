Symptom

In the DO Data Model, data mapping currently works only between Parent–Child sections. When mappings are maintained from other initial version sections, the data is not being mapped to V2.


In this scenario, the Source Header section is mapped to the Target Schedule section, and the Source Schedule section is also mapped to the Target Schedule section. However, the data from the Source Schedule section is not getting mapped to the Target Schedule section in V2. This note corrects the mapping logic to ensure proper data transfer across all maintained section mappings.

Other Terms

Section Mapping Issue, Parent Child Mapping, Source to Target Mapping, /VTA/BDOM

Reason and Prerequisites

This is caused by program error

Solution

Please implement the correction instructions attached. These corrections are only valid if you have applied the following support packs.

Release V4 - 2022 V2, 2023, 2023 V2 and 2023 V3 or Support Pack 1

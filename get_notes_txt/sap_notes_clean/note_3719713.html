Symptom

SAP BusinessObjects Live Data Connect is a component that enables SAP Analytics Cloud to leverage existing WebI documents and relational BI universes created with the information design tool (UNX).

SAP BusinessObjects Live Data Connect is an application embarking its own application server and is used by SAP Analytics Cloud when you are accessing live data on SAP universes and WebI documents.

The following instructions create an SAP Analytics Cloud model based on a relational universe or WebI document without replicating data in the cloud:

  1. Connect to SAP BusinessObjects BI Platform and get the list of relational universes and WebI documents

  2. Select a universe or WebI document, run the Query Builder and create a query by selecting objects

  3. Retrieve the metadata and save your model

  4. Reuse your model for SAP Analytics Cloud stories

Reason and Prerequisites

The SAP BusinessObjects Live Data Connect 2025 release is the sole version supporting SAP BusinessObjects 2025. If you upgrade this version from SAP BusinessObjects Live Data Connect 3.4.4 the same configuration file could be kept and reused for version 2025.

Make sure that you have the following elements before installing SAP BusinessObjects Live Data Connect 2025.

  • An SAP BusinessObjects Business Intelligence 2025 installation

  • Microsoft Visual C++ 2015 Redistribuable Packages.

  • The SAP BusinessObjects Live Data Connect 2025 component available on the SAP Support Portal.

Installing SAP BusinessObjects Live Data Connect on the same computer as SAP BusinessObjects Business Intelligence is not supported, so make sure you have a dedicated computer for your SAP BusinessObjects Live Data Connect setup.

If you are using SAML authentication, when configuring CORS (Cross-Origin Resource sharing), make sure your SAML identity provider URL is included in the list of allowed origins. Refer to the documentation for more information.

New features in LDC 2025 (Release date: March 1st, 2026)

  • Support of SAP BusinessObjects 2025

Known Restrictions

SAP BusinessObjects Live Data Connect 2025 has below restrictions:

  • The query builder doesn't support multi-columns list of values.
  • It does not support universe delegated measures.
  • It does not support universe hierarchical list of values.
  • It does not support SAP Analytics Cloud linked dimensions.
  • It is not possible to create a model on universe containing cascading prompts or prompts depending on contexts.
  • Creating calculated dimensions from a chart is not supported.
  • Sort: LDC does not support custom order.
  • Rank: "Apply to each dimension" is not supported when measure relies on IF 
  • HANA input parameters or variables on currencies or units displays keys instead of the descriptions.
  • HANA multi-colomns are not supported
  • HANA dynamic default variable value on dates is not supported
  • It is not possible to compose IF with LIKE i.e. IF( LIKE(), , )
  • Live Universe Connectivity is not supported from SAP Analytics Cloud mobile application
  • When connecting to a clustered BI Platform, you must choose one CMS as the entry point for LDC and reference it under the boe.nameserver parameter.
  • When using calculated measures on a Live Universe Connectivity datasource, make sure the connection language is set to EN (english) to avoid misinterpretation of decimal separators.
  • Connecting to LDC from the same machine (localhost) is not supported. Your LDC server should always be running on a different computer than your web browser.
  • In the member selector, in Optimized Design Experience stories, search is not supported for hierarchical presentations.

You'll find the complete list of limitations here.

Known Issues

  • Showing the key instead of the description in list of values of index aware prompt
  • Manual entry is allowed for constrained prompts
  • Partial refresh is silent for charts
  • Not able to work with live connectivity on universe after killing user session in the Central Management Console of SAP BI Platform.
  • High precision numeric measures generate an error
  • In some cases (eg when the query panel is maximized), double clicking a predefined filter in the query panel may result in it being added twice to the query. You should either remove the incorrectly added predefined filter, or use drag and drop to add it to your query.
  • In some cases, expand and collapse icons may appear incorrectly in SAP Analytics Cloud tables.
  • Default values for calculated/dynamic variables are evaluated only once, when the story is created. Reopening the story does not trigger reevaluating the values for these variables.
  • In optimized mode, for tables, only the crosstab is supported, other layouts trigger calculations that break.
  • Histogram charts do not render on Live Universe Connectivity datasources.
  • Conditional aggregation does not work on Live Universe Connectivity datasources.
  • Scheduling of Live Universe Connection based documents does not work.
  • In Optimized Design Experience stories, applying a measure-based story filter on Live Universe Connection charts breaks them.
  • In Optimized Design Experience stories, recursive rank and break grouping in sort may behave incorrectly in tables.

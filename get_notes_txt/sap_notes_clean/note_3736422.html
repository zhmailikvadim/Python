Symptom

You observe that items reviewed for additional recovery amounts that do not qualify are not being removed from the ADM Index table. As a result, the table continues to grow, causing the ADM Batch Job to run progressively longer over time and eventually time out. It is determined that the Index table is not being cleared appropriately.

This not corrects the issue by ensuring that non-qualifying and outdated items are automatically cleared from the ADM Index table, preventing performance degradation of the ADM Batch Job.

Other Terms

/IRM/IPG_SDINDEX_MAINTAIN, Automated Data Maintenance, Performance, Long Runtime

Reason and Prerequisites

This is caused by program error

Solution

Please implement the correction instructions attached. These corrections are only valid if you have applied the Support Packs.

Release V/4 - 2022 V2 and 2023

Use transaction SNOTE to implement the correction instructions specified below and activate the affected program object.

Symptom

  • Users encounter unexplained values in transaction MCSJ related to customer purchasing infostructures.
  • The issue arises when attempting to analyze purchasing data using infostructure S612.
  • Users seek a method to delete all existing values and rebuild them for accurate analysis.

Environment

  • Materials Management (MM)
  • SAP ERP Central Component
  • SAP ERP
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA

Reproducing the Issue

  1. Access transaction MCSJ in SAP ERP.
  2. Attempt to analyze purchasing data using infostructure S612.
  3. Observe unexplained values in the analysis results.

Cause

Standard SAP Infostructures: 001-500 is  reserved for SAP-delivered standard infostructures, it should not be modified directly.

Customer Infostructures: 501-999:  is available for customer-specific implementations 

MCSJ serves as the entry point for accessing customer-defined purchasing infostructures (501-999 range),  displaying custom purchasing analysis reports and executing customer-specific purchasing evaluations.

As a best Practices for Customer Infostructures:

  1. Namespace Compliance: Always use numbers 501-999 for customer infostructures
  2. Documentation: Maintain clear documentation of custom infostructure purposes
  3. Authorization Strategy: Implement appropriate authorization controls for sensitive purchasing data
  4. Performance Optimization: Design infostructures with appropriate indexing for large data volumes

Resolution

To delete all values from your customer purchasing infostructure and rebuild them, follow this approach:

1.- Delete Existing Infostructure Data

Use transaction OLIX (Delete Data from Infostructures):

Navigate to OLIX
Enter Parameters:

Infostructure: Your customer infostructure number (501-999 range)
Version: 000 (main version)
Application: 2 (Purchasing)


Select Deletion Options:

- Choose appropriate date ranges if needed
- Confirm deletion of all data

Important: Always delete the data first before rebuilding to avoid duplication issues

2.- Rebuild Infostructure Data
After successful deletion, rebuild using the appropriate restructuring transactions:
For Purchasing Infostructures:

OLI7: Statistical setup for purchase orders
OLI9: Statistical setup for invoices (if applicable)

Execution Steps:

- Run OLI7 with version 000 to rebuild purchase order data
- Run OLI9 with version 000 to rebuild invoice data (if your infostructure includes invoice information)
- Monitor Progress: These transactions can take considerable time depending on data volume

3: Verification
After rebuilding:

  • Check MCSJ to verify data appears correctly
    - Run test reports to ensure values are accurate
    - Compare totals with source documents to validate completeness

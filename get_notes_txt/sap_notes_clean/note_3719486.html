Symptom

The communication protocol between the SAP GUI and the dispatcher allows GUI responses to be discarded. However this is fatal for SNC encrypted GUI communication with flow control. A missing package raises the following error in SNC processor in the work process:

M ThSncIn: process input data of rq_id 356252889 at 11010058 with length 664
N *** ERROR => SncPFrameIn()==SNCERR_GSSAPI [D:/depot/bas/78 4458]
N GSS-API(maj): An earlier expected token has not yet been received

In the dispatcher trace dev_disp the following trace can be seen:

DpRTmPrepareReq: multiple send of T7_U77_M3: discard outdated response

Other Terms

multiple send

Reason and Prerequisites

Program error

Solution

There is no workaround.

Apply a new kernel patch. The fix is secured with the new dynamic profile parameter rdisp/allow_discard_gui_response of type boolean with values 0, 1, on, off and so on. Its initial value "0" activates the fix. In case of unexpected problems, the parameter can be switched to "1" to restore the behavior, which was active before the patch.

The correction is delivered with the kernel archives:

  • Kernel patch - dw.sar
  • SP stack kernel files SAPEXE.SAR and SAPEXEDB.SAR

The corrections are contained in all patch levels that are equal to or greater than the patch level specified in the "Support Package Patches" section for the relevant kernel release.
If there is no entry there, the patch has not yet been delivered. The patch level will be entered automatically once the patch is available for download. You can subscribe to this SAP Note to be notified when it is updated.

Follow our recommendations for applying a kernel patch:

  1. Import the latest SP stack kernel if it already contains the correction. For a list of current SP Stack Kernels, see SAP Note 2083594 (SAP Kernel Versions and SAP Kernel Patch Levels).
  2. Apply the hotfix only if it is a serious error that is not yet corrected by the latest SP Stack Kernel. 
  3. Check the regression note for the desired patch level before applying the kernel patch. For detailed information, see SAP Note 1802333 (Finding information about regressions in the SAP kernel).
  4. For instructions on how to download and apply a kernel patch, see SAP Note 19466 (Downloading SAP kernel patches).

For detailed information about our recommendations, see the strategy document: Update Strategy for the Kernel of the Application Server ABAP in On Premise Landscapes.

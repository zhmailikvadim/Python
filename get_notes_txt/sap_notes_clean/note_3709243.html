Symptom

  • The receiving profit center option is missing when creating a new SKF value in the Fiori App "Manage Statistical Key Figure Values".
  • Only the receiving cost center option is available to create SKF in the app.

Environment

SAP S/4HANA Cloud Public Edition

Cause

The system is designed to utilize the receiving cost center as the receiver for SKF values. The system automatically copies the values set for the cost center to its associated profit center.

Resolution

Under the context of Profit Center Distribution, when SKF is utilized, the Profit Center SKF values are automatically derived based on the posted SKF cost center values. Within the app Manage Statistical Key Figure Values, users can post SKF values for a cost center, and these values are subsequently associated with the profit center linked to that cost center. 

This setup will use the cost center associated with the profit center as the receiver instead of specifying the receiver profit center directly. The system will then copy the posted value defined for the cost center to its corresponding profit center.

Symptom

Despite activating the workflow and ensuring all necessary preconditions are maintained, the approval/reject request for 'Preliminary Billing Documents' is not triggering to the authorized user as expected.

Environment

SAP S/4HANA Cloud Public Edition

Reproducing the Issue

  1. Create workflow in "Manage Workflow for Preliminary Billing Documents".
  2. Activate the workflow.
  3. Create a preliminary billing document via "Create Preliminary Billing Document" app.
  4. System does not trigger workflow.
  5. The workflow fails to notify the approver in the "My Inbox" app.

Cause

The workflow does not notify the approver in the "My Inbox" app because the triggering mechanism for "Preliminary Billing Documents" is linked to the finalization step. The approval request is initiated only when the billing clerk, acting as the approval requester, manually finalizes and saves the document.

If the document is saved as a draft or without setting the finalization status, the workflow is not triggered, and no notification is sent to the approver.

Resolution

After the "Preliminary Billing Documents" is created, ensure the preliminary billing document is finalized using the "Finalize" button in the "Create Preliminary Billing Document" app as the approval request is only initiated when the billing clerk, acting as the approval requester, manually finalizes and saves the document.

Symptom

  • When a technician creates an emergency notification from the SSAM application, the plant value is picked up from the first value of Plant from the Syclo Admin Panel (User Attribute) instead of the IWK parameter for the user.
  • This issue occurs only for a few users, while the rest receive the correct Plant value picked up from the IWK parameter.

 "Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental"

Environment

SAP Service and Asset Manager all versions

Reproducing the Issue

  1. Log in to the SSAM application using the credentials of a user experiencing the issue.
  2. Create an emergency notification from client
  3. Observe that the plant value is picked up from the first value of Plant from the Syclo Admin Panel instead of the IWK parameter for the user.
  4. Synchronize the data and verify that the Maintenance Plant and Operation values are the same as those of the notification.

Cause

Resolution

  1. Run tcode: '/n/syclo/configpanel'
  2. Open oData Mobile Data Object Configuration
  3. Expand corresponding application SAP_SERVICE_ASSET_MANAGER_XXXX
  4. SAMxxxx_NOTIFICATION_GENERIC -> Data Filter section -> Operation Read -> Data Distribution -> PLANPLANT*
  5. Deactivate the Mobile User Attribute rule.

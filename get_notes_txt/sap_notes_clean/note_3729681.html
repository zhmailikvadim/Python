Symptom

  • Data does not refresh after changing filter values in the multi-dimensional report.
  • Users receive the message "No Data" from the query, even though data corresponding to the filter is available but not returned.
  • Dragging the splitter between the fields list on the left and the table output on the right re-triggers the query and displays the correct data.

Environment

FPM Analytical GUIBB

Reproducing the Issue

  1. Launch the multi-dimensional report with the default variant.
  2. Enter a value in the filter bar field and click "Go".
  3. Observe the message "No Data" from the query, even though data corresponding to the filter is available but not returned.
  4. Drag the splitter between the fields list on the left and the table output on the right to re-trigger the query.
  5. Observe that the data is returned and displayed correctly.

Cause

  • Program error

Resolution

    • Implement note 3710836 - Result Set / Grid empty when executing query after default bookmark loaded.

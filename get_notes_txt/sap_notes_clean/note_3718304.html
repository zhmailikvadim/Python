Symptom

I synchronize my SAP system with Managed Gateway for Spend&Network (ISMG) and complete all configuration steps for the RFQ process according to SAP documentation.

When I create or save an RFQ in transaction ME41:

  • The Output Type triggers correctly.

  • The IDoc generates successfully.

  • I can see the IDoc in transaction WE02.

  • I see successful processing logs in transaction SRT_MONI.

However:

  • I do not see the document in the ISMG.

  • The system does not transmit anything to ISMG.

  • When I reprocess the IDoc, the system prompts for the P-User.

  • After I enter the P-User, the system executes the sending process, but the document still does not appear in ISMG.

Environment

  • Managed Gateway for Spend&Network
  • SAP ERP Central Component (ECC)
  • SAP S/4HANA

Cause

The receiver port is not correct in your SPRO configuration 

Even though the IDoc generates successfully and appears in WE02 and SRT_MONI, the system does not forward the message to ISMG if the receiver port or port definition is missing or incorrectly maintained.

As a result, the IDoc remains within the SAP system and does not reach to ISMG.

Resolution

You must verify and complete the receiver port configuration.

Follow these steps in sequence:

  1. Configure the receiver port according to the guide Configuring the Receiver Port | SAP Help Portal 

  2. Create the port definition according to the guide Creating Port Definition for SAP Integration Suite, Managed Gateway for Spend Management and SAP Business Network | SAP Help Portal 

After completing the configuration:

  1. Re-test the RFQ creation in transaction ME41.

  2. Monitor the IDoc in transaction WE02.

  3. Verify that the document appears in the SAP Integration Suite, Managed Gateway for Spend&Network and SAP Business Network.

If the issue persists, verify:

  • The partner profile configuration in transaction WE20.

  • The correct assignment of the receiver port.

  • The logical port configuration in SOAMANAGER, if applicable.

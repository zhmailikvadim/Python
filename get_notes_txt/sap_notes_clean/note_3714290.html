Symptom

When customer tries to run Story Report for Growth Portfolio shows WSM instead of PM_Forms.

Environment

SAP SuccessFactors Talent Intelligence Hub and Job Profiles/Job Description Manager/Job Profile Builder/Center of Capabilities

HCM Suite

Reproducing the Issue

  • Login to customer instance
  • Go to Reporting > Run Growth_Portfolio Story Report
  • Go to the Sample affected  
  • Source Type Shows WSM it should be PM_Forms

Cause

The user-level update related to competency "mark as passionate" in Growth Portfolio was performed after the PM_Forms rating.

Resolution

This is expected behavior as per design.

Keywords

Growth Portfolio Story Report, Source Type, Story Report

Symptom

You are running the Cálculo de nómina española (RPCALCE0) report for an employee that has both the BI cuenta ajena (/151) and the BI consejeros (/153) wage types. The tax percentage being applied to the /153 wage type might be wrong if the tax percentage calculated by the Datos fiscales (0062) infotype is higher than the tax percentage customized for key Consejeros y Administradores (E) in the Claves y subclaves de percepción (V_T5E41) view.

Other Terms

&CALC; IRPF (for more information, see SAP Note 1071282)

Reason and Prerequisites

The tax percentage defined for key E in the V_T5E41 view should be applied to the /153 wage type.

The following Support Package level is the minimum prerequisite:

Release Support Package
6.08 HR SP 13
6.04 HR SP 85
6.00 HR SP B9

(For more information, click on "HR Support Packages Schedule" in support.sap.com/maintenance-schedule)

Solution

Corrections to source code of HR_E_READ_TAX_PERCENTAGE function module have been provided to solve this issue.

From now on the tax percentage applied to the /153 wage type will always respect the customizing from the V_T5E41 view, unless the Clave percep. (CPER1) field from 0062 infotype has value Consejeros y administradores (E) and the tax percentage was manually set.

Version Date Reason

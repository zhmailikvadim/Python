Symptom

You've maintained multiple Planning Versions for a PDS.

After You've run Product Heuristic in /SAPAPO/RRP3, you've noticed that in the Log that PDSs where selected that where not in the specific Planning Version that you are working with currently.

When checking in /SAPAPO/CURTO_SIMU, the PDS for the specific planning version is not found.

Other Terms

 PDS, planning version, simulation version, /SAPAPO/CURTO_SIMU, /SAPAPO/RRP3

Reason and Prerequisites

All PDSs are first integrated with Planning Version “ALL” assigned by default (Planning Version 000). When creating an order, the PDS with Planning version “all” is considered valid to be selected. When using planning tools, system will first try to assign a valid PDS for the specific planning version, if no PDS is found the PDS with Planning version (All) will be selected. This is a valid scenario.

Solution

The Solution lies in the SAP Note 1700617:

"The correction creates method LOAD_ACTIVE_VERSION in class /SAPAPO/CL_PDS_BUFFER. Its interface contains exactly one return parameter (EV_RETURN) that controls whether the PDS of the active version should be loaded or not. The parameter will be set to 'X' per default. I.e. the PDS of the active version will be loaded.

If you do not want to load the PDS of the active version you have the opportunity to enhance this method. It is recommended to use the implicit enhancement point at the end of the method. Within the enhancement just clear parameter EV_RETURN to avoid loading of the active version's PDS.

This solution is not a modification. It replaces SAP Note 1695104."

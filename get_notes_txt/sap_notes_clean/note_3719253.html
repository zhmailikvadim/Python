Symptom

In ASR scenario, during setting the handling execution status to Checked Out or Departed at the FO outbound stop, or receiving such events reported by external event web service message, the system behavior does not automatically update the status to "Load End" at stop level even all items at current stop are loaded.

  • Error message: Event Loading End must be reported manually or via a service and can't be derived." (Message No. /SCMTMS/TOR_EV_PROC 026)

Environment

SAP Transportation Management embedded in SAP S/4HANA 2020 and subsequent releases

Reproducing the Issue

  1. Set up TM-EWM integration via Advanced Shipping and Receiving (ASR).
  2. Perform loading in Extended Warehouse Management (EWM).
  3. Ensure "loading end for item" is reported in ASR FO.
  4. Proceed to the "Check-Out" step in TM or via an external service.
  5. Observe the error message indicating that the "loading end" status is missing.

Cause

In the ASR outbound scenario, the Handling Execution Status at the FO stop (warehouse loading stop) must be set to Loaded (i.e., the stop-level “Loading End” event must be reported) before subsequent events such as Check-Out can be reported.

The system behavior is intentional: when a stop’s Handling Execution Status is set to Loaded, the mechanism automatically removes any FO items that do not have the Loaded status from the ASR FO. After the stop-level Loaded event is reported, no additional items can be loaded onto the truck.

Therefore, succeeding events cannot be reported at the stop until the Loading End event is first reported at the warehouse loading stop after the trucker confirms the truck load status at stop level is completed.

Resolution

Please ensure 'Loading End' event at the warehouse loading stop is reported manually or via a service before reporting the succeeding events.

For instance, user can use the Handling Execution Status option Set Stop to Loading Completed to set the status of the warehouse loading stop to Loaded in ASR FO Overview screen.

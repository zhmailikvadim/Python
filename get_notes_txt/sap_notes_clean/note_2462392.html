Symptom

After upgrade of a NetWeaver BW system to SAP_BW 7.50 SP04 or higher, you notice errors with Open ODS Views during query execution, or when activating a Composite Provider that hasOpen ODS Views as PartProvider. The Open ODS View has integer-like Keyfigures with SUM-aggregation.

The error message when activating a Composite Provider is similar to: "Error: Datatype fixed18.0(8) of <attrbute_name> does not match datatype double(8) of attribute <attribute_name> in node <node_name>".

Other Terms

Open ODS View

Reason and Prerequisites

The Data Type Mapping for integer-like keyfigures with SUM-aggregation has been changed with 7.50 SP04. In older releases, the ABAP Type FLTP has been used, which corresponds to HANA Column Store Type double. With the introduction of the ABAP Type INT8, the mapping has been changed to BIGINT (HANA Column Store Type: fixed(18,0).

Solution

The Calculation Scenarios of Open ODS Views need to be recreated. There are two options:

  1. Report RSDDB_INDEX_CREATE_MASS with option "Create Views for all (FBPA) Open ODS Views" and uncheck option "Skip object if it already exists". Then the CalculationScenarios/ColumnViews of all Open ODS Views will be recreated.
  2. Report RSDDB_LOGINDEX_CREATE can be used to recreate the CalculationScenario/ColumnViews for single InfoProviders.

If the Open ODS Views are consumed within Composite Providers, then the Composite Providers should be re-activated afterwards.

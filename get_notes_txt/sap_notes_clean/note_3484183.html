Symptom

  • There is a requirement to increase/decrease the timeout values for a SOAP Adapter channel.
  • The following exception occurs intermittently
    Error in call over HTTP: HTTP 0 null
    [EXCEPTION]
    java.net.SocketTimeoutException: Read timed out (local port XXX to address XXX, remote port XXX to address XXX
    at java.net.SocketInputStream.socketRead0(Native Method)...

Environment

SAP NetWeaver 7.5

Resolution

For SOAP Sender Channel - it's recommended to maintain the parameter "syncTimeout" under module "CallSapAdapter" as per SAP KBA 0001621719 - Timeout parameters in SOAP Sender/Receiver Adapter. The default value is 300000 msec (5 minutes).

For SOAP Receiver Channel - If XMBWS.Timeout is set, it will behave as read timeout as per SAP KBA 0001621719. However, it's recommended to use new timeout parameter "XMBWS.ConnectTimeout" and "XMBWS.ReadTimeout" as per SAP Note 2294811 SOAP receiver adapter terminates synchronous session, ensure both connect and read timeouts are covered properly. The default value for "XMBWS.ConnectTimeout" 10000msec (i.e., 10 seconds) and the default value for "XMBWS.ReadTimeout" is 300000 msec (5 minutes). 

==

Note:

  1. Since PI 7.1, there is no property "xiadapter.outbound.timeout.default" under NWA -> Configuration -> Infrastructure -> JAVA System Properties -> Services -> XPI Adapter: XI.
  2. Property "xiadapter.inbound.timeout.default" still exists under service "XPI Adapter: XI", but it's only for inbound messages received in a classical scenario, not ICO scenario and not affecting outbound messages.

Symptom

This is a guidance to understand how multiple active implementations for BAdI HRESS_PAYSLIP_BADI_DEF works and how to handle it.

Environment

  • SAP SuccessFactors Employee Central Payroll
  • Direct Pay Statement

Cause

In the most recent versions of EA-HR support package, a change in the processing flow is now considering every single active implementation for a single BAdI. This is expected and it suppose to be like this.

Resolution

  1. Open all implementations for BAdI HRESS_PAYSLIP_BADI_DEF;
  2. Identify the implementation that should be the primary one;
  3. Deactivate all other implementations except the primary one;
  4. If functionalities from other implementations are required, consolidate them into a single implementation.

Keywords

Direct Pay Statement, SuccessFactors, BAdI, HRESS_PAYSLIP_BADI_DEF, HRSP, EA-HR, implementation, enhancement.

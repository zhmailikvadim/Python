Symptom

When accessing email settings in Email Services, a warning message display as 'Warning: Unable to find your email settings'. 

Environment

SAP SuccessFactors Onboarding

Reproducing the Issue

  • Search for Email Services
  • Select Email Settings
  • Encounter message: 'Warning: Unable to find your email settings'

Cause

  • This issue generally occurs in newly provisioned instances
  • The issue can also occur if necessary permissions like Email Framework, configure object definitions, etc is not granted. 
  • The issue is caused due to absence of defaultSettings in EmailSettings object in Manage Data

Resolution

  1. Please try and run the MDF Object Definition to DB Sync Job and then perform an OData API Metadata refresh and check if the warning message is still triggering. 
  2. Check if necessary permissions for Email Framework, configure object definitions, etc is granted. If not, please provide them and check again in Email Settings. 
  3. Even after performing the above two checks, the issue occurs, then: 


    • Go to Manage Data
    • Search for EmailSettings object in first dropdown and search for defaultSettings (defaultSettings) in second dropdown.
    • If the defaultSettings is not appearing, then you need to re-configure this object. 
    • You can re-configure this object by using the code as defaultSettings, isCustomLayout as No, customLayout as default and leave the footer blank. (If, in case the code is blank, you can refer to Configure Object Definitions and enter EmailSettings. Now change the visibility of externalCode to Editable from Read Only. You can later change this visibility back to read only after the issue is resolved) 
    • In the layout section, choose code as default and add the layout as an HTML Email Template for layout and placeholders. You can copy the same template from the lower environment and paste it here. Similarly, choose custom customLayout and add the HTML Email Template copied from the lower environment.  
    • Save these configurations and perform an OData API Metadata refresh and check if the issue still persists.

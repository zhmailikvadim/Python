Symptom

This SAP Note provides the optimizations the following address and message optimizations for the Singapore Income Tax IR8A (New Report):

Employee's address:

1.Remove the postal code from address information.

Since the postal code is a separate field in the generated file, it should not be included in the address fields to avoid duplication.

2.Modify the CO (Care Of) address determination logic.

With the new optimization, if the Care of field in infotype 0006 is maintained, the Singapore local address will be recognized as a CO address.

3.Fix the issue where the field may be duplicated in Address Line 1 and Address Line 2 in the file.

4.For myTax files: clear the Country/Region field when the employee’s address is a local Singapore address.

As the myTax file template does not provide a Singapore option for this optional field, the field shall be submitted blank for local address.

Message optimizations:

1. Change the message 'Has no employment' to 'Employee has no employment for the current period', and change the message type from 'Error' to 'Warning'.

2. Add a new message when Appendix 8A records exceed the limit of 10.

3. Add a new message when Appendix 8B records exceed the limit of 30.

For employees affected by items 2 and 3, it is recommended to exclude them from the current variant and submit manually.

In addition, this SAP Note includes an enhancement to optimize the scenario where IC Type cannot be retrieved due to the absence of the NRIC table in the payroll result.

Some customers may have no NRIC table in their payroll results due to their custom payroll scheme. With the new change, the report will read IC Type from IT0185 in such cases.

Other Terms

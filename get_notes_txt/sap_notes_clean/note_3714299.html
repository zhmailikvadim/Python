Symptom

  • An error occurs during the posting of a physical inventory document.
  • The error message "GETWA_NOT_ASSIGNED" appears when attempting to post the document from the transaction /SCWM/PI_PROCESS.
  • The issue arises when adding a new material to a storage bin during the counting process, where the material physically exists but has never been recorded in the system for that bin.
  • The error was observed during testing of the physical inventory process.

Environment

  • SAP Extended Warehouse Management (SCM-EWM)
  • SAP Extended Warehouse Management Physical inventory (SCM-EWM-PI)

 



Reproducing the Issue

  1. Create a physical inventory (PI) document for storage bin counting (HL).
  2. Enter count results for materials already existing in the PI bin.
  3. Add a new material to the bin, specifying its stock type, owner, and quantity.
  4. Attempt to post the PI document from the transaction /SCWM/PI_PROCESS.

Cause

Program error. 

Resolution

Please implement below notes in the effected system to avoid the issue. 

  1. 3571027 - Dump when posting an inventory document.
  2. 3552063 - /SCWM/PI_PROCESS: Dump "ASSERTION_FAILED" when processing PI document in transaction on the press of Enter.

Keywords

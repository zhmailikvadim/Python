Symptom

When generating the SPED EFD ICMS/IPI report (SPED Fiscal) using either SAP Tax Declaration Framework (TDF) or SAP Document and Reporting Compliance (DRC) for Brazil, the file fails validation. The validator returns an error related to record 0221, such as:

"The value must exist in the COD_ITEM field of a parent record 0200 with TIPO_ITEM filled with ‘00 – Merchandise for Resale’ and that has a single child record 0221 with COD_ITEM_ATOMICO equal to the COD_ITEM field of the parent record."

Further analysis shows that the system is generating multiple 0221 child records for a single 0200 parent record. This often occurs when a material classified for resale (TIPO_ITEM = '00') has an associated Bill of Materials (BOM) that the system incorrectly processes, leading to multiple component entries in record 0221.

Environment

  • SAP ERP
  • SAP S/4HANA
  • SAP S/4HANA Cloud Private Edition
  • SAP Tax Declaration Framework for Brazil
  • SAP Document and Reporting Compliance service
  • SPED EFD ICMS IPI

Cause

The root cause is an inconsistency in the material’s master data configuration across different plants within the source ERP or S/4HANA system. The SPED file generation logic in both TDF and DRC Brazil expects a material to have a consistent classification and usage type (which determines the TIPO_ITEM) in all plants where it is active.

When this consistency is broken, the report generation can fail. Common scenarios include:

  • A material is classified as "Resale" (TIPO_ITEM = '00') in one plant but has a Bill of Materials (BOM) in another. The system may try to explode the BOM, which is invalid for a simple resale item, thus creating multiple 0221 records.
  • A material is correctly set up as a "Finished Product" (TIPO_ITEM = '04') in its production plant but is incorrectly classified as a "Resale" item (TIPO_ITEM = '00') in other distribution plants.

This conflict in classification leads to incorrect data generation for records 0200 and 0221. The TIPO_ITEM field is derived from various material master fields (e.g., MARC-KZKUP, MBEW-MTUSE, T134-MTREF), and discrepancies in these fields for the same material across plants will cause this error. More information about TIPO_ITEM field, check KBA 2419759.  

Resolution

To resolve this issue, the material must be treated with a consistent classification across all plants for reporting. This can be accomplished using one of the following two methods:

1. Align Material Master Data (Recommended)

This is the preferred solution as it corrects the data at its source.

  • Identify the material code (COD_ITEM) that is causing the validation error.
  • Review its master data settings across all relevant plants in your ERP or S/4HANA system.
  • Correct the configuration to ensure the material has a single, unified purpose. For example, if a material is produced and has a BOM, its classification should reflect that consistently and not be set to "Resale".

2. Use Tool-Specific Overwrite Configuration

If changing the source master data is not feasible, you can use a solution-specific configuration to enforce consistency for reporting purposes.

  • Go to transaction /TMF/MAT_CONFI.
  • Create an entry to overwrite the TIPO_ITEM for the problematic material. This will override the inconsistent values from the ERP master data during report generation.

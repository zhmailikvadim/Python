Symptom

You wish to enhance PMMO Distribution so that a different Cost Element is used to post the cost to the assigned WBS element (or Pegging breakpoint) than was used to post the original order cost against the group WBS element/order.

However, when you do that, the Fiori app "Cost Distribution" assumes that the two Cost Elements are the same, as in the standard implementation, and shows you the original Cost Element, obtained from the credit posting which offsets the original cost against the group WBS/order, instead of the Cost Element from the debit posting to the operative WBS element/breakpoint.

Other Terms

F7381, PMMO_Distribution, RPMMO_DISTRIBUTION, kstar, acdoca-racct, Account Number, co_beknz, Financial WBS Element

Reason and Prerequisites

This is functionality that was unforeseen during the original development.

Solution

Implement this SAP Note.

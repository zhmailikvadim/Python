Symptom

During the transfer of an Application to a Contract, an error occurs if the Business Partner (BP) is not in Released status. However, creating a Contract directly does not trigger this error, resulting in an inconsistency in the system behavior.
To prevent this inconsistency, a new parameter C_FLG_NO_BP_CHECK_STATUS_CHG has been introduced in the BAdI FVD_MASTERDATA, method SET_GLOBAL_FLAGS.
On implementation of this BAdI and when this parameter is set in the method, it allows customers to control or disable the BP not released check on status change For e.g. during the Application-to-Contract transfer process.

Other Terms

TB_NO_BP_CHECK_STATUS_CHANGE, FN45, IF_EX_FVD_MASTERDATA

Reason and Prerequisites

 This was not previously provided.

Solution

 Implement the attached correction instructions or import the specified Support Package.

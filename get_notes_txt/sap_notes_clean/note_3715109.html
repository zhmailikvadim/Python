Symptom

After you reprocess/reprice the bucket date, you find that the old record retains with zero values instead of deleting the record

(shows deleted record data with zero values)

  • For example:
    • 1. Goto /irm/ipbrclbm and give bucket number nnnn as input and execute
    • 2. Reprice any document in the bucket and run the aggregation reconstruction.
    • 3. Again goto bucket nnnn and navigate to X901 aggregation table.
    • 4. filter with the repriced document and you will find two rows i.e one with old data and one with new data.

Other Terms

/IRM/IPCLBM, /IRM/IPCLBCU, Claims, IPBRCLBM, Bucket, X901, Aggregation

Reason and Prerequisites

This is caused by program error

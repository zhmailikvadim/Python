Symptom

During the import of a Transport Request, the ABAP Dictionary activation fails with the below error:

1EEMC744 You do not have authorization for the database modification

    Environment

    • SAP NetWeaver - all BASIS releases.
    • ABAP Dictionary Activation and Conversion.

    Reproducing the Issue

    1. Start the import of a transport request in TMS for ABAP.
    2. Monitor the import phases until ABAP Dictionary activation runs.
    3. Observe that the import fails with message MC744/EEMC744 "You do not have authorization for the database modification" and that DDIC objects are not activated.

    Cause

    Missing authorization for database modification during transport import.

    Resolution

    1. Ensure the import user has the required ABAP Workbench authorization S_DEVELOP for DDIC activation as indicated by SU53 transaction, for example:
      • OBJTYPE: VIEW and/or TABL (as applicable).
      • ACTVT: 02 (Change) and/or 40 (Generate/Activate).
      • Object/package fields (such as DEVCLASS/OBJNAME) matching the transported objects.
    2. Create or adjust a custom role with these exact values and assign it to the import user.
    3. Repeat the transport import and verify that ABAP Dictionary objects activate without errors.

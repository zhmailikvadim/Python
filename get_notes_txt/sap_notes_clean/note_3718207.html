Symptom

Business require to assign a repository to storage category SOMU, the repository is not available to assign in OACT.

Cause

The storage category SOMU does not accept a document area.

Resolution

If the error faced is 'SOMU is unknown', follow the KBA 2840359 instead.

In transaction OACT, assigning a repository to category SOMU requires the repository "Document Area" field to be left blank.
This conflicts with ArchiveLink documents, which require the repository "Document Area" to be set to "ArchiveLink."

Because of this, the same repository cannot be used for both SOMU and ArchiveLink documents.

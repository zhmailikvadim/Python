Symptom

  • You have set up Audit Logging in the SAP Business Technology Platform (SAP BTP. URL: https://cockpit.btp.cloud.sap) Cloud Foundry environment. 
  • The configurations have been done as per SAP KBA ##2637286 : How to trigger Audit Logs for SAP Business Technology Platform (SAP BTP).
  • You would like to know whether it is possible to archive the generated Audit Logs.

Environment

  • SAP Business Technology Platform (BTP)
  • Cloud Foundry environment
  • Neo environment
  • Audit Log Service for SAP BTP Cloud foundry environment

Reproducing the Issue

Trigger Audit Logging in the Cloud Foundry Environment.

Cause

Product limitation; unsupported as of now.

Resolution

  • SAP does not provide a service by which the Audit logs can be archived. 
  • By default, the audit log data stored for the account will be retained for 90 days, after which it will be deleted. 
  • In case there is a desire to retain the generated data, customers can using the API , download the content and store it in another location (this is customer specific and SAP does not provide this service).
  • It is also possible to enable Audit Log service, premium edition, and configure a custom retention period longer than the free 90 days. For details, see Audit Log Retention for the Cloud Foundry Environment and SAP KBA ##3419922 - How to request for Audit Log Service retention period increase.

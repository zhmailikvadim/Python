Symptom

  • When processing a DDA file in transaction FF.5, vendor invoices with Withholding Tax (WHT) on payment are not updated.
  • In the DDA report (DDA_BR), these invoices appear with an error status ('E').
  • The error is caused by a document amount mismatch, as the net amount in the DDA file differs from the gross amount on the invoice in the SAP system.

Environment

  • Financials - Electronic Bank Statement
  • SAP ERP
  • SAP S/4HANA
  • Localization Brazil

Reproducing the Issue

  1. Post a vendor invoice that is subject to Withholding Tax (WHT) on payment.
  2. Receive a DDA file from the bank where the payment amount for this invoice is the net value (gross invoice amount minus WHT).
  3. Process this file using transaction FF.5.
  4. Run the DDA report via transaction DDA_BR.
  5. The system will display the corresponding invoice with an error status ('E'), indicating that the amount in the file does not match the amount on the invoice.

Cause

This is the standard and expected system behavior. The Brazilian banking authority, FEBRABAN, has not defined a mandatory standard for whether DDA files should contain the gross amount (before tax) or the net amount (after tax).

Due to this lack of a standard, the DDA process is designed to prevent incorrect invoice updates by performing a strict validation check:

  • The system automatically compares the CNPJ, Due Date, and Document Amount from the DDA file against the vendor invoice.
  • An exact match across all three fields is required for the system to automatically associate the barcode.
  • In cases of Withholding Tax (WHT), the gross amount on the SAP invoice does not match the net amount in the bank’s DDA file.
  • This amount mismatch correctly causes the validation to fail, flagging the item with an error ('E') for manual review.

Resolution

The designed solution is to use the manual post-processing function within the DDA report.

  1. Run transaction DDA_BR to display the list of DDA items.
  2. Identify and select the line item that has the error status ('E').
  3. Click the Post Processing button.

This action signals to the system to proceed with the update despite the amount mismatch. The system will then associate the barcode with the vendor invoice, and the status icon will change from red to green.

For customers who wish to automate this process and avoid the manual step, a custom development is possible.

    • An enhancement can be implemented at the enhancement point RFFOBR_DDA_POINT within the standard report RFFOBR_DDA.
    • Within this enhancement, custom ABAP code can be written to identify the correct invoice and instruct the system to bypass the amount check, allowing the update to complete automatically during post-processing.

Symptom

  • After completing Personal Data Collection step, the E-Signature step does not Initiate because of the form errors.
  • Error displayed: Invalid option are set for[CheckBox1]
  • Document Template Mapping Preview shows the value appears to pass but it does not populate on the form.

Environment

SAP SuccessFactors Onboarding

Reproducing the Issue

  1. Go to BPE to-do.
  2. Observe the Error message: Invalid option are set for[CheckBox1].

Cause

  • The form contains a nested checkbox structure (a checkbox placed under another checkbox), which is not supported.
  • The Required property is set to “No” for all related checkbox fields.
  • Export values configured for the checkboxes are misaligned (contain special characters and/or do not exactly match the picklist option values).

Resolution

  1. Remove the nested checkbox structure so that each checkbox is placed at the same level (No checkbox under another checkbox).
  2. Set the Required property appropriately for the checkbox fields (do not leave all related checkbox fields as “No” if user input is expected).
  3. Review each checkbox’s export value and ensure: No special characters are used in the export value. and also, the export value exactly matches the corresponding picklist option value (same spelling, spaces, capitalization, and formatting).
  4. Save the form in Adobe Acrobat and Re-upload it to the Document Template.
  5. Use Document Template Mapping Preview to validate that the correct values populate each checkbox.
  6. Re-test the onboarding process and confirm the E-Signature step initiates without errors.

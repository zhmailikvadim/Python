Symptom

Runtime errors: TSV_TNEW_PAGE_ALLOC_FAILED while extracting from 0CO_OM* 

"TSV_TNEW_PAGE_ALLOC_FAILED"
"SAPLPSBF" or LPSBFU05
"PSBW_SELECT_ACTIVITY"

or 

"TSV_TNEW_PAGE_ALLOC_FAILED" " "
"SAPLPSBS" or "LPSBSU09"
"PSBW_GET_OBJNR_FOR_NWA" 

Environment

  • S/4HANA
  • SAP ERP

Reproducing the Issue

  1. Attempt to extract data from the 0CO_OM* datasources in BW from ECC/ERP or S/4HANA.
  2. Observe runtime errors: TSV_TNEW_PAGE_ALLOC_FAILED.

Resolution

1. Increase the Heap memory or EM according to basis Note 2180736 - TSV_TNEW_PAGE_ALLOC_FAILED;
Or
Increase the frequency of the delta extaction to make sure every time the data size to be processed it not quite high; 

2. Set parameter NOOBJPRESL in BWOM_SETTINGS for the affected DataSource

    • Purpose: Avoid object pre-selection to reduce memory footprint.
    • How:
      • Maintain table BWOM_SETTINGS (e.g., via SM30) and create/update an entry for the specific DataSource.
      • Parameter name: NOOBJPRESL
      • Value: X
    • Reference: SAP Note 2159875 (for explanation; implement it if applicable).

      Use object-relevant selections to avoid full-table reads

      • Suggested fields: KOKRS (Controlling Area), POSID (WBS Element), PSPID (Project Definition).
      • Do not run the data extraction with completely empty selections on large datasets.

3. Performance tuning (MAXSIZE). Some CO-OM DataSources use parameter MAXSIZE from table BWOM_SETTINGS to control data package size.

    • Reference: SAP Note 2546578.
    • Action:
      • Maintain BWOM_SETTINGS for the affected DataSource.
      • Parameter name: MAXSIZE.
      • Recommended value: between 10,000 and 100,000.
      • Set this parameter if it is currently empty.
    • Important considerations:
      • Not all CO-OM DataSources honor MAXSIZE.
      • The number of records per data package may differ from the raw MAXSIZE due to calculation logic (see SAP Note 417307).
      • Package sizes can vary significantly between packages by design (see SAP Note 413992; FYI only, no correction needed).

4. While using Service API (SAPI), follow the instructions per KBA 2941940 - [BW Central KBA] Troubleshooting SAP Service API performance or configuration issues (Data Packages, Work Processes, RFC Connections, Partner Profiles) too.

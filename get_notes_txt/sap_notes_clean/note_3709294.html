Symptom

In certain scenarios, authorization checks implemented via BAdIs (DOCUMENT_AUTH01 / DOCUMENT_AUTH02) are not executed consistently when document originals are opened from SAP ECTR.

This behavior can occur if customer logic relies on application-specific activities (for example 53 – Display application start) within these exits.

Standard authorization checks for document originals are performed.

Other Terms

CDESK_SRV_AUTH_CHECK, CDESK_SRV_GET_FILES, ACM , Activity 53, Acticity 52

Reason and Prerequisites

Function module CDESK_SRV_AUTH_CHECK differentiates between general document activities (for example 03 – Display document info) and application-related activities (for example 53 – Display application start).

The BAdI methods are executed only for the general activity (lv_activity), while application-related activities (lv_activity_dok) are not passed to these exits.

As a result, BAdI-specific logic that explicitly evaluates application activities may not be triggered in all use cases.

Solution

To allow consistent execution of customer exits in well-defined scenarios, the authorization check was adjusted in a controlled and backward-compatible way:

  • CDESK_SRV_AUTH_CHECK was enhanced to optionally execute BAdIs for application-related activities (52 / 53)
  • A new input parameter IV_CHECK_BADI_APPL was introduced to control this behavior explicitly
  • This parameter is set to true only in the context of opening document originals:
    • CDESK_SRV_GET_FILE_URL
    • CDESK_SRV_GET_FILES

All other usages of CDESK_SRV_AUTH_CHECK remain unchanged, ensuring existing customer implementations are not affected.

Symptom

Business rule "Get Next Possible Date for Changes" is used, but it always get error "Back dated change is not possible" when editing the corresponding information in SF no matter the date is before or after the payroll area next possible date for changes for the payroll area in ECP systems.

Environment

SAP SuccessFactors Employee Central Payroll

Reproducing the Issue

  1. Business rule "Get Next Possible Date for Changes" is used, for example in payment information.
  2. Change payment information from a date in SF.
  3. Error "Back dated change is not possible" is shown no matter the date is before or after the next possible date for changes for the payroll area in the ECP system.

Cause

The rule function returns 01/01/1900 so all changes are not possible.

  • The pay group or the user ID isn’t valid.
  • The OAuth configuration is missing or the payroll system URL/Client ID is missing in the Payroll System Configuration object.
  • The SSO is not working.

Resolution

  • Make sure the pay group and the user ID is valid.
  • Check the OAuth configuration. Refer to Using OAuth 2.0 to Integrate Employee Central and Employee Central Payroll
  • Maintain the payroll system URL/Client ID in Payroll System Configuration.
  • The business rule can be used for both ADM and ESS. Make sure the SSO between SF and ECP is working for the ADM/ESS user. You can collect SAML trace to troubleshoot the SSO issues. Refer to 2960670 - How to collect SAML trace with the Security Diagnostic Tool for ABAP system.
  • You can set an external breakpoint for the ADM/ESS user in class CL_HRSFEC_PAYCTRL_REC_DPC_EXT method PAYROLLCONTROLRE_GET_ENTITY or GET_PAYROLL_CTRL_REC_DETAILS.

Symptom

HTTP 403 error when calling POST /jobs. Error: “The service requested is not available”

Reproducing the Issue

  1. Prepare a REST request for creating a Sourcing reporting job using the following endpoint:
    POST https://openapi.ariba.com/api/sourcing-reporting-job/v2/<env>/jobs?realm=<REALM>

  2. Include valid headers (apikey: <API_KEY>, Content-Type: application/json).

  3. Use a request body similar to:
    {
      "viewTemplateName": "<VIEW_TEMPLATE_NAME>",
      "filters": {
        "LastModifiedFrom": "<ISO_UTC_TIMESTAMP>",
        "LastModifiedTo":   "<ISO_UTC_TIMESTAMP>"
      }
    }

  4. Execute the request.

  5. Observed result: HTTP 403, message “The service requested is not available”.

Cause

  • The request used the /v2 endpoint, which is valid only for Operational Reporting for Procurement.
  • Operational Reporting for Sourcing currently only supports version 1 (/v1).
  • Using the Procurement request as a template and cloning it can inadvertently introduce the wrong version, causing a 403 response.

Resolution

  1. Use the correct endpoint for Operational Reporting for Sourcing:
    POST https://openapi.ariba.com/api/sourcing-reporting-job/v1/<env>/jobs?realm=<REALM>

  2. Keep all required headers and authentication details.

  3. Send the same request body as before.

  4. The job submission should now complete successfully when using /v1.

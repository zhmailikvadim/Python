Symptom

You run the Year End Reporting, Canada (PC00_M07_CCYR) transaction to generate a T4, T4A or T4A-NR and the system generates the PDF form with only the employee/recipient's last name in capital letters.

Example:

Employee Name: Employee From Canada

Employee/Recipient's Name in PDF form: CANADA Employee F

Other Terms

RPCYERK0; CL_HRPAYCA_YE_FORM_HANDLER; CL_HRPAYCA_YE_FORM_HANDLER_LFO; FILL_EMPLOYEE_NAME.

Reason and Prerequisites

Reason

The system only converts the last name to capital letters.

Publication RC4120 - Employers’ Guide Filing the T4 slip and Summary, released by the CRA, states "Enter all in capital letters the employee’s last name followed by the employee’s first name and initials". The fillable PDFs released by the CRA also automatically capitalize the employee's / recipient's first name, last name and initials in the T4/T4A slips respectively.

Prerequisites

This SAP Note follows the same minimum Support Package level as described in SAP Note 3656002 - Main SAP Note for Canadian year-end 2025.

For more information, access the SAP ERP Human Capital Management - Support Package Schedules (SAP_HR) or SAP S/4HANA On Premise Human Capital Management (S4HCM).

When you install an SAP Note, some prerequisites might be required. The prerequisites are SAP Notes that have already been released and that change the same context of the current one. Then, even if these prerequisites are not necessarily related to the symptom of this SAP Note, you must implement them due to technical reasons.

Solution

From now on, the system generates the T4, T4A or T4A-NR PDF form with the employee/recipient's last, first and middle initial name in capital letters.

Example:

Employee Name: Employee From Canada

Employee/Recipient's Name in PDF form: CANADA EMPLOYEE F

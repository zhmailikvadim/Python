Symptom

When the scrap value/Inventory Numbers field is left blank or set to 0 in the migration template, the scrap value/Inventory Numbers of the asset subnumber is still inherited from the main asset during migration using the Migration Cockpit – Fixed Asset Master Data.

Environment

SAP S/4HANA Cloud Public Edition

Reproducing the Issue

  1. Prepare a migration file using the Migration Cockpit - Fixed Asset Master Data template.
  2. Set the scrap value/Inventory Numbers field for the subnumber asset to 0 or leave it blank in the template.
  3. Execute the migration process.
  4. Verify the migrated data in the target application 'Manage Fixed Assets'.
  5. Observe that the subnumber asset inherits the scrap value/Inventory Numbers from the main asset number instead of reflecting the intended value (0 or blank).

Cause

The design logic of the Migration Cockpit aligns with the Fiori app behavior, where sub-assets inherit values from the main asset if the corresponding fields are left blank or set to 0 during migration.

Resolution

Use the API Fixed Asset – Master Data to bulk create or change the scrap value/Inventory Numbers for fixed assets.

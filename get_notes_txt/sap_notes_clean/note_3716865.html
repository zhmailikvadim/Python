Symptom

  • The issue occurs during the transaction FP09 for return lot closing process.
  • An error occurs when attempting to close a return lot containing payment documents from multiple company codes.
  • The error message displayed is: >2 668 "A returns lot is only allowed to contain one company code".

Environment

FI-CA - Contract Accounts Receivable and Payable  

Reproducing the Issue

  1. Create a new Return Lot in transaction FP09.
  2. Assign company code xxx to the Return Lot header.
  3. Include two payment documents in the Return Lot item level - one with company code xxx and one with company code yyy.
  4. Attempt to close the Return Lot.
  5. Observe the error message >2 668 "A returns lot is only allowed to contain one company code".

Cause

The error >2 668 arises because the return lot contains payment documents from multiple company codes, which violates the system's design requirement.

Resolution

The functionality is operating as intended. Return Lots are designed to contain only one company code.

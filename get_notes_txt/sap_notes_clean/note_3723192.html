Symptom

  • Delivery date variance fields (KBED1–KBED5) in infostructure S012 show only 0 after refreshing/rebuilding.
  • GR value is updated in transaction MC$4 immediately after posting, but delivery date variance is not calculated.

Environment

  • Materials Management (MM)
  • SAP ERP Central Component
  • SAP ERP
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA

Reproducing the Issue

  1. Refresh infostructure S012 by referring to SAP Note 64636.
  2. Enter a vendor and period in the MC$4 selection screen.
  3. Observe that GR values are displayed, but delivery date variance fields (KBED1–KBED5) remain 0.

Cause

  • Delivery date variance fields in LIS (S012) are statistical fields that only get updated via real-time posting (from PO/GR/Invoice movements).
  • Historical variances cannot be recalculated during a rebuild, as they require comparing historical planned vs. actual dates.
  • Delivery variance is calculated only for goods GR, not for services, as services use service entry sheets and acceptance dates instead of goods movement events.

Resolution

  1. Confirm that KBED1–KBED5 fields exist in structure S012 using transaction OMOS (Field catalog LIS).
  2. Maintain settings for relevant purchasing organizations in the navigation path: Logistics - General -> Logistics Information System -> Logistics Data Warehouse -> Updating -> Updating Control -> Settings: Purchasing -> Change Period Split for On-Time Delivery Performance.
  3. Post a new PO and Goods Receipt where the delivery date differs from the planned date. Verify if transaction MC$K shows a non-zero variance.
  4. In standard practice and system logic, delivery date variance is calculated only for goods GR, not for services. For services, you would need custom KPIs or enhancements if you want to track timeliness against planned service dates.

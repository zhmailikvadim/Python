Symptom

Correct company code is not fetched for inbound SWIFT MT300 / MT320 in Treasury Correspondence in case there are multiple company codes for the same BIC / SWIFT Code. In such cases it is seen the inbound SWIFT contains tags 87(option A or D or J) and(or) tag 83(option A or J) that contains company code details like NAME , ADDRESS, BIC Code etc. The correct company code is not fetched since there is not enough data in SWIFT tags to indentify a unique company code.

Other Terms

BADI_TCORF_FORMAT_MULTI_OPTION

Reason and Prerequisites

This is a program error.

Solution

1. Implement the attached correction instructions provided in the SAP Note via SNOTE transaction.

2. Once the above is done, create a custom implementation of BADI - BADI_TCORF_FORMAT_MULTI_OPTION.

3. Develop a custom logic in method - IF_TCORF_BADI_FORMAT_MULTI~CHANGE_PAYLOAD to fetch the correct company code. Inside this method, importing parameter IO_MESSAGE contains the SWIFT file contents in structured format. Changing parameter CR_PAYLOAD contains derived SWIFT data in TCORY_CODATA format. Incorrect company code that is derived after execution of mapping rules, needs to be modified with correct company code in this changing parameter CR_PAYLOAD.

4. You can make use of database tables TZBZ , T001 and others to develop a logic to fetch the correct company code from tags 87J and(or) 83J in the inbound SWIFT.

5. Make sure the codebase in method IF_TCORF_BADI_FORMAT_MULTI~CHECK_CONTENT in standard implementation(EI_TCORI_SWIFT_FILE) of BADI is copied to the custom implementation of the same method as well.

6. For the custom BADI implementation maintain suitable Filters Values as maintained for the standard implementation.

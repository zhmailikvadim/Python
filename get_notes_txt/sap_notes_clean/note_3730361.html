Symptom

OpenID Connect (OIDC) Authentication is missing features such as being able to select a specific OIDC configuration (besides the Default Interactive Configuration) for interactive OIDC Logon, Token Exchange support for SAP Cloud, a more flexible Redirect URL handling for multi-hostname system setups or an improved OIDCTokenRequested cookie handling.

Other Terms

OpenID Connect, OIDC, OIDC Logon, OIDC Bearer, ID Token, JSON Web Token, JWT, authentication, reauthentication, logon, login, single sign-on, SSO, security, OIDCTokenRequested, SOIDC, SOIDC_ANALYZER, SOIDC_SESSIONS, Fiori, FLP, Token Exchange

Solution

OpenID Connect (OIDC) in ABAP

With this delivery we enhance the existing support for OpenID Connect (OIDC) authentication for ICF services, WebSocket RFC, and JCo.

Please note, that I will not duplicate the extensive configuration and troubleshooting guide I wrote in SAP Note 3111813. That SAP Note will always be kept up to date and reflect the latest available features, their configuration, and how to troubleshoot OIDC authentication. Therefore, if the following brief description is not sufficient, please consult the OpenID Connect (OIDC): Troubleshooting Note 3111813 for more extensive information.

Find here a list of all the features delivered with OpenID Connect (OIDC) up to and including this OIDC Feature Set 6 with a short introductory description:

  • (F1) User propagation via an OIDC ID Token (JWT), described in the following as logon method 'OIDC Bearer' for ICF services.
  • (F2) Interactive standard-compliant OpenID Connect Authentication, described in the following as logon method 'OIDC Logon' for ICF services.
  • (F3) Ease of configuration by optionally supporting the automated download of the OpenID Provider metadata (OIDC discovery document) and its JSON Web Key Set (JWKS).
  • (F4) Interactive single logout (SLO) via the Front-Channel.
  • (F5) Interactive single logout (SLO) via the Back-Channel.
  • (F6) Fiori Launchpad support for interactive OIDC authentication by preserving the URL Anchor Fragments (https://fiorilaunchpad.sap.com/sites#Shell-home).
  • (F7) Additional ID Token signature algorithms (RS384, RS512).
  • (F8) Selected additional claims for the user mapping besides the standard claim 'sub'.
  • (F9) Proof Key for Code Exchange (PKCE) support during interactive OIDC (OIDC Logon).
  • (F10) Configurable / custom claim for the user mapping in addition to the standard claim 'sub' and the selected additional claims 'email', 'oid', and 'user_name'.
  • (F11) Enhanced supportability of OpenID Connect via the new transaction SOIDC_ANALYZER. 
  • (F12) A report which can be scheduled to regularly update an OIDC Configuration's key material with the latest JWKS from the OpenID Provider. 
  • (F13) The possibility to configure for each OIDC Configuration a local proxy which you might need if your ABAP system requires an HTTP proxy for outgoing HTTP connectivity. 
  • (F14) For OIDC Bearer, certificate-bound tokens can be properly validated via the new 'cnf' claim check. SAP Note 3455890 must be applied additionally. 
  • (F15) The usability/clarity of transaction SOIDC has been significantly improved to dynamically hide non-activated properties in the OIDC configuration. 
  • (F16) An OIDC configuration can be activated and de-activated in transaction SOIDC. 
  • (F17) Logout Token Validation for Back-Channel single logout evaluates configured non-standard User Mapping Claims.
  • (F18) Transaction SOIDC_ANALYZER supports the creation and validation of custom tokens.
  • (F19) The download of the JWKS in transaction SOIDC no longer directly replaces the configuration's keys but highlights the differences.
  • (F20) A correction was provided for Feature F6 which caused Syslog Events and Security Audit Log entries. 
  • (F21) A correction was provided for Feature F13 which prevented the Local Proxy usage for new OIDC configurations prior to saving for the first time.
  • (F22) The HTTP Client can select specific OIDC Configurations (besides the Default Interactive Configuration) for interactive OIDC Logon.
  • (F23) Option to auto-update an OIDC configuration's key material.
  • (F24) Usage of configured SSL ID in all outgoing HTTP calls from the ABAP system to the OpenID Provider (download of metadata/discovery, JWKS, exchange of Authorization Code into ID Token).
  • (F25) Transaction SOIDC_SESSIONS and Token Exchange Support for OAuth Client to support SAP Cloud scenarios.
  • (F26) Flexible Redirect URL handling for multi-hostname system setups.
  • (F27) Improved OIDCTokenRequested cookie handling.
  • (F28) An alternative to the Default Provider Metadata URL in OIDC Discovery (determined from the Issuer URL), a new option allows to download metadata from a Custom Provider Metadata URL.
  • (F29) Improved idempotency in the OIDC Configuration API ensures that a database update is only triggered when the OIDC Configuration was actually changed.
  • (F30) API Protection support through OIDC Bearer Token authentication for selected SAP scenarios. 

The initial support for OIDC authentication (Features F1, F2, and F3) was started in SAP BASIS 7.56 SP1 as described with SAP Note 3105025.

This was extended by additionally offering support for Front-Channel Single Logout (Feature F4) in SAP BASIS 7.57 (OpenID Connect Feature Set 1).

OpenID Connect Feature Set 2 provides the aforementioned features F5 to F10 (and in case of the OIDC Downport the features F1 to F10) and the corresponding SAP Note 3313726 describes the corresponding SPs for each release from SAP BASIS 7.58 down to SAP BASIS 7.52. including the kernel dependencies. Please note that ABAP and kernel are separate entities which can be updated in the system, and OIDC will only work if both (ABAP and kernel) are new enough.

OpenID Connect Feature Set 3 provided by SAP Note 3452750 contains the features F11 to F15.

OpenID Connect Feature Set 4 provided by SAP Note 3585020 contains the features F16 to F21.

SAP Note 3594613 contains feature F22.

OpenID Connect Feature Set 5 provided by SAP Note 3632887 contains the features F23 to F27.

OpenID Connect Feature Set 6 provided by this SAP Note contains the features F28 to F30.

Symptom

When you reprice the calculation bucket, you expect the updated rate to be reflected in the pools as well. However, after repricing, the new rate is not updated in the pools even though the calculation buckets are updated correctly. This leads to inconsistencies between the calculation bucket data and the pool values.
This note corrects the behavior by ensuring that the updated rate is properly reflected in the pools after repricing the calculation bucket.

Other Terms

calculation bucket reprice, pool update issue, new rate not reflected, bucket repricing, pool inconsistency, rate update issue

Reason and Prerequisites

This is caused by program error.

Solution

Please implement the correction instructions. These corrections are only valid if you have applied the following Support Packs.

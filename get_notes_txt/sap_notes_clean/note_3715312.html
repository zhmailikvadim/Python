Symptom

In the nJJIT integration scenario, where an outbound delivery is created from a Supply-to-Customer JIT call with component‑group–based processing, the Delete Assignment button in VL02N remains enabled even though goods movement for the outbound delivery is already completed.

Other Terms

NJIT, HU, Packing, VL02n, Delete HU

Reason and Prerequisites

The issue occurred due to a program error in the delivery processing logic.

Solution

Implement the correction instructions attached to this SAP Note or upgrade to the latest available release/support package, where the corrected code is already included.

Symptom

  • Users are unable to export transactions from the Transaction Workspace in SAP Incentive Management.
  • The export process appears to loop indefinitely, and subsequent attempts to export display the message: "The export has already started! Please wait until the process is complete."
  • Users can only initiate a new export after closing and reopening SAP Commissions.
  • The issue occurs regardless of the export parameters entered.
  • Customer's browser language is not in English.

Environment

  • SAP Sales Performance Management
  • Incentive Management

Reproducing the Issue

  1. Set the browser language to German.
  2. Navigate to Incentive Management Admin in the environment.
  3. Go to Preferences > Customization.
  4. Open Calculation > Transactions.
  5. Change the Custom Value of any attribute to include non-English special character (e.g., ä, ö, ü, ß, á, ó, é).
  6. Save the change.
  7. Return to Calculations > Transactions workspace.
  8. Choose Export Data > CSV/Excel.
  9. Select "Export all displayed records from Current Page to Last Page" and click Export.
  10. Observe that the export fails, and subsequent attempts indicate that a download is still in progress, preventing new exports.

Cause

The issue is caused by a bug in the Data Loader functionality, which fails to handle non-English special characters in the Transaction Custom Value field.

Resolution

The fix for this issue is planned to be deployed on 1H 2026 release.

For release timelines, please review SAP SuccessFactors Product Release & Road Map Information.

Workaround

If the filename contains non-English characters, adjust the browser language or the user's language preference.

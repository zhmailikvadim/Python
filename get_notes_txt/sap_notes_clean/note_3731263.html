Symptom

You are using Universal Parallel Accounting or the new Material Price Table FMLT_PRICE via Business Function FINS_FMLT_PRICE_TABLE_BF. You are using the transaction CKME (Release Planned Price Changes) to release planned price changes in your system. You want to perform the price release with parallel processing by entering a server group and number of parallel processes. The material and/or material prices are inconsistent after the price change or the price release performs very slow.

Other Terms

 CKME, FMLT_PRICE

Reason and Prerequisites

Program error

Solution

Corrected with support package. Implement the attached correction instruction for an advanced correction.

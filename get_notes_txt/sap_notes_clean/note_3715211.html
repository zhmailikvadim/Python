Symptom

When processing multiple delivery proposals for the same predecessor document where delivery creation fails, the system does not properly reset the BOPF Buffer variant (BV), leading to inconsistent buffer variant state and further operations failing with BV-related errors.

Other Terms

Delivery Creation failed; BOPF; Buffer Variant reset failure; /SCMTMS/CL_TM_LE_INTEGRATION; DLVCR_EXEC_MAIN; Only active variant can be saved; /SCMTMS/CL_BUFVAR_DISPATCHER=>/BOBF/IF_FRW_BUFFER~DO_SAVE

Reason and Prerequisites

The reason is a program error in method DLVCR_EXEC_MAIN of class /SCMTMS/CL_TM_LE_INTEGRATION. When delivery creation fails, the system runs into a return statement and does not reach the coding where the failed state is set (ev_dlvcreation_failed = abap_true). As a consequence, the Buffer Variant is not reset correctly from the calling logic.

Solution

Implement the corrections of the attached correction instructions or apply the corresponding support package.

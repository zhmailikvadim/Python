Symptom

Symptom

There is situation where while INSTALLING qTest, on the step of Elasticsearch you face the below issue after entering the port number for ES. 

"An error occurred: PKIX path building failed: sun.security.provider.certpath.SunCertPathBuilderExecption: unable to find valid certification path to requested target"

Environment

SAP Enterprise Performance Testing by Tricentis

Resolution

Check the folloowng knowledge article from Tricentis:

An error occurred: PKIX path building failed: sun.security.cerpath.SunCertPathBuilderExecption:.....

Keywords

qTest, keystore, elasticsearch, certificate, communicate

Environment

SAP Promotion Management, SAP CARAB

Reproducing the Issue

  1. Import offers to SAP CAR using the function module /DMF/OPIF_OFFER_INBOUND.
  2. Ensure the logical system is assigned to the offers.
  3. Verify that the offers are saved in the database with the status "Released."
  4. Transfer button is not enabled

Cause

The logic which prevents to transfer these imported offers is in class: /PRM/CL_OFFER_PWL_ACT_TRANSFERCP method: SET_CONTROL_PROPERTIES. The root cause why the transfer button is not enabled is due to the value of offer_tcd which is 02 (imported - complete). 

Resolution

The coding in SET_CONTROL_PROPERTIES method allows only offer_tcd 01 and empty for transfer, therefore it is not supported to transfer these offers in standard system.

Keywords

offer transfer, SAP PMR, ERP synchronization, /DMF/OPIF_OFFER_INBOUND, master data system, offer_tcd, SAP Promotion Management, logical system SSDCLNT110, GK POS distribution, Released status, SET_CONTROL_PROPERTIES, /PRM/CL_OFFER_PWL_ACT_TRANSFERCP

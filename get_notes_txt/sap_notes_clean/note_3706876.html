Symptom

Currently, it's not possible to control, trough the BI platform, which paths can be selected when scheduling to SFTP, making it possible for a user to schedule to any path the user has access to.

Environment

  • SAP BusinessObjects Business Intelligence Platform 4.x
  • SAP BusinessObjects Business Intelligence Platform 2025

Reproducing the Issue

  1. Select a report
  2. Right Click > Schedule
  3. On destination, select SFTP
  4. Uncheck the default path
  5. Now you're able to customize the path the file will be schedule to

Cause

Currently, SFTP destination security is expected to be done on SFTP side, as BI does not provide additional security measures in this category.

Resolution

There is an improvement request made for this that is currently being addressed by the development team. The link for the same is on the "See Also" section just below.

WORKAROUND:

  • Give the user access to only a specific folder in it's home directory. This way only that folder can receive schedules, as other paths will be blocked and will result in an error in the schedule.
  • Another option is to use a chrooted ssh user to prevent unwanted path or file interactions.

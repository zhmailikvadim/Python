Symptom

After a new implementation, upgrade, company rollout, or related system change, the FSM Connector functionality works only partially.

Typical observations include:

  • /PACG* IDocs are generated and processed successfully in ECC/S4.

  • No errors are visible in IDoc processing.

  • However, results are not fully visible in FSM.

Example

  • Service orders are replicated and visible on the FSM Planning Board.

  • Related activities are missing in FSM.

  • Object changes appear to be only partially synchronized.

Environment

  • SAP Field Service Management (FSM)
  • Proaxia Connector (PACG)
  • ECC, S/4 HANA

Reproducing the Issue

Expected behavior: 

  • Communication between ECC/S4 and FSM works correctly.

  • All relevant objects and changes are replicated completely.

  • Customized connector functionality operates as defined.

  • All related objects (e.g., service orders and activities) are visible in FSM.

Actual behavior: 

  • Partial functionality is observed.

  • /PACG* IDocs are successfully generated and processed.

  • No technical errors are reported.

  • Some replicated data or dependent objects are missing in FSM.

Cause

The issue is typically caused by one of the following:

  1. Customizing tables in ECC/S4 are not properly updated.

  2. Post-implementation steps were not done.

  3. The FSM Connector setup program was not (re)done after:

    • A new installation

    • An upgrade

    • A new company rollout

    • SAP Note implementation (if required)

Most communication and functional inconsistencies in the FSM Connector are caused by missing execution of the setup program.

Resolution

Run Setup Program

Run transaction: /PACG/ECM_SETUP

This program must be run in the following scenarios:

  • New installation of the FSM Connector

  • Implementation of a new company

  • Connector upgrade (Add-On or Service Pack implementation)

  • SAP Note implementation (if manual steps indicate it)

 

Important: Multiple Logical Systems / Companies

If multiple companies use different combinations of:

  • Logical system

  • Distribution model

  • Partner profile

The setup program must be run separately for each combination (triple) of these parameters.

If each company uses its own dedicated logical system, distribution model, and partner profile, run the setup program individually for each configuration.

Verification

To verify whether the setup program was run correctly:

  1. Run transaction:/PACG/ECM_VERSION

  2. Compare the displayed value of Setup Version with:

    • The latest version stored in table:/PACG/ECS_SETUP

If the Setup Version in /PACG/ECM_VERSION does not match the latest entry in /PACG/ECS_SETUP, run/PACG/ECM_SETUP.

Additional Information

  • /PACG/ECM_SETUP – FSM Connector Setup Program

  • /PACG/ECM_VERSION – Display FSM Connector Version

Refer to the FSM Cloud Connector documentation (Confluence) for detailed setup and post-implementation instructions.

Summary

If IDocs are processed successfully but data is only partially visible in FSM, the most likely root cause is that the setup program /PACG/ECM_SETUP was not run after installation, upgrade, or system changes.

Re-running the setup program typically resolves the issue.

Symptom

  • Activity does not automatically inherit the address from the equipment, even though the equipment has an address assigned.
  • The address must be manually associated by accessing the activity and selecting the option "Activity Equipment".



    "Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental."  

Environment

  • SAP Field Service Management (FSM)

Reproducing the Issue

  1. Assign an address to equipment
  2. Create an activity and associate it with the equipment.
  3. Actual Behavior:
    Observe that the activity does not have an address derived from the activity equipment and display the Address origin as "User Defined"
  4. Expected Behavior:
    Activity address should derived from the activity equipment and display the Address origin as "Activity Equipment"

Cause

  • Refer to the documentation Address Resolution Logic | SAP Help Portal for details on the activity address resolution process
  • The address resolution logic fails when the activity address and the equipment address are not the same, resulting in a blank address displayed in the frontend.

Resolution

  1. Verify that the activity.equipment has an address and follows the resolution logic step 3: Address of the equipment linked to the activity, if it exists (Activity.equipment -> Equipment <- Address.object.objectId).
  2. Use the Query API to check the activity and equipment address details.
    • select a.object.objectId, a.address, a.code, a.id, a.businessPartner, a.equipment from Activity a where a.code = 'ABC';
    • select a.id, a.externalId, a.object.objectId, a.object.objectType, a.street, a.streetNo, a.zipCode, a.city, a.location.latitude, a.location.longitude  from Address a where a.id = 'YYYYYYYYYYY';
  3. Ensure a.equipment matches the a.object.objectId

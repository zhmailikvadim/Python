Symptom

  • The Tricentis Authentication Service was configured with an SSL certificate using an 8192-bit key length (https://docs.tricentis.com/tosca-2025.1/en-us/content/tosca_server/tricentis_service_configuration.htm#configure-tricentis-authentication-service)
  • While working with the Authentication Service (User Administration), issues are observed such as the inability to see any users or errors when creating user groups

Environment

SAP Enterprise Continuous Testing by Tricentis

Cause

This issue is caused by a defect in a third-party component provided by Microsoft that does not properly handle SSL certificates with very large key sizes (such as 8192-bit). The status of the issue can be tracked here: https://github.com/AzureAD/azure-activedirectory-identitymodel-extensions-for-dotnet/issues/3407

Resolution

A bug fix has been identified and is being tracked here: https://github.com/AzureAD/azure-activedirectory-identitymodel-extensions-for-dotnet/issues/3407

SAP and Tricentis partner is planning to provide the fix via a patch/release. At this time, we cannot provide a definitive release date of the patch/release with the fix. Once the improvement is available, we will update this Knowledge Base Article (KBA) accordingly.

Please follow this KBA for the updates.

In the meantime, we recommend the following workaround to address the issue:

  1. Create a new certificate with a key length of 2048. Detailed certificate requirements can be found here.
  2. Configure the Tricentis Authentication Service to use this certificate. Instructions for this configuration are available here.

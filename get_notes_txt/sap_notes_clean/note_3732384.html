Symptom

An error “DBSQL_DUPLICATE_KEY_ERROR” occurs when attempting to upload and valuate a non-operated check through the check input in PRA table /PRA/CI_PPD. This issue prevents the successful import of the check input data.

Environment

  • SAP S/4HANA 2023
  • SAP S/4 HANA ON-PREMISE

Reproducing the Issue

  1. Go to transaction code /n/PRA/CI_PPD and click on 'Create.'
  2. Enter the header information from the screenshot titled 'Check' and hit Enter.
  3. In the spreadsheet template, select the cells you want to import from the Upload Template tab and copy them.
  4. Use the Import button to import the data from your clipboard.
  5. Save within the transaction.
  6. The dump error occurs.

Resolution

During the upload of check input data, the system stopped with dump error because some rows in the uploaded file are generating duplicate line numbers. In the table, each record in the upload must have a unique combination of three fields:

  • Remitter
  • Check Number
  • Line Number

In the current upload, the Line Number starts again from 0001 after row 100.000 due to default length of the field. Because the Remitter and Check Number remain the same, this creates duplicate records, and it triggers the error preventing inconsistent data.

As a workaround you should upload the data in Excel format with up to 100,000 rows per file. If the dataset contains more than 100,000 rows, you will have to split the data into more than one Excel file and upload them separately. 

The length of the fields are standard defined by the PROFOG consensus. 

Keywords

PRA/CI_PPD, key ABAP dump when importing data, PRA check input data.

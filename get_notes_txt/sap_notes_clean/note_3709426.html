Symptom

  • An error is encountered when a Business Partner (BP) is attempted to be created in SAP S/4HANA after the system conversion from ECC to S/4HANA.
  • The error message “BP already exists” is displayed.
  • The BP number indicated in the error message is confirmed to exist, and it is able to be viewed in transaction BP. 

Environment

  • SAP ERP Central Component
  • SAP ERP
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA
  • SAP S/4HANA Cloud Private Edition

Reproducing the Issue

  • Enter transaction BP.
  • Select grouping as Sold to party.
  • Enter all details with BP role as customer or vendor.
  • Save the details.

Cause

The NR status is not being updated during the migration process.

Resolution

For the first time after migration, the NR status should be updated manually as a post‑migration step, after which it will be automated by the system for all succeeding entries. This is required because, in most migration templates or segments in LTMC, the option to update the NR status is not provided. 

The respective customer/vendor table (KNA1/LFA1) should be checked to determine the highest customer/vendor number that has been assigned/created. The NR status should then be updated with the next exceeding number. Once the NR status has been set, the preceding manual customer/vendor creation will be adopted automatically.

For example, if the number range interval is 1–100 and the highest vendor number assigned/created is 78, the NR status should be updated with 79.

Additionally, 
Use transaction SM56 to check current number range buffer entries and their status.
Refresh the Number Range Buffer using below path.

Navigation Path : SM56 → Number Range Buffer → refresh

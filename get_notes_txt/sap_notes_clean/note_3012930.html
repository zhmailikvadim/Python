Symptom

During step Reassignment you get a dump with error message: "ASSERTION_FAILED". This error occurs in method CL_FAGL_OBJ_TYPE_APAR -> R_SPLIT_MANUALLY_P, because for a document a currency type did not exist at step GENERATION. In the meantime (especially during an S/4 migration) the currency type has been added to the documents, but it is still missing in the reorganization tables.

Solution

The corrections of this note, do not solve the problem. Those documents with missing currency types still cannot be reassigned by the reorganization tool. But as the dump is avoided, you are able to identify which documents are affected by the problem. Each document has to be handled individually.

Implement the program corrections to avoid the dump.

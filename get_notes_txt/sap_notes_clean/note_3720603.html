Symptom

In the process "Start of Supply for Gas (New SUP View)" (DE_B_GELIGAS_SOS_SUP), step 1500-0100 treats the values 0.000 and 0,000 as different in the comparison logic. This affects the business message BM_GPKE_SUP_SOS_INFO_PROC_END and the field GRID_USAGE_COMMUNITY_DISCOUNT.

This issue is relevant for all market roles.

Other Terms

GPKE_SUP, DE_B_GELIGAS_SOS_SUP, MDX data container list, /UCOM/CL_MDX_CONVERSION_RULE->CONV_OF_QTY, conversion group QTY_FORMAT, BM_GPKE_SUP_SOS_INFO_PROC_END, GRID_USAGE_COMMUNITY_DISCOUNT, decimal comparison

Reason and Prerequisites

The error occurred because the comparison logic between the system value and the inbound value didn't handle the special case where decimal values use different separators (0.000 versus 0,000).

Solution

Implement the corrections of this SAP Note or import the relevant support package.

After implementing this SAP Note, the system treats the values 0.000 and 0,000 as equal in the MDX data container list. You need to retrigger the process to run the MDX inbound update process. You can verify the fix by checking that the data container list treats the values 0.00 and 0,00 as equal.

Symptom

When maintaining User-Defined Texts (UDTs) in SAP EHS, the formatting of long texts appears inconsistent between transaction CG02 and the WEBUI Display/Change Specification application.

The following symptoms may be observed:

  • Line breaks (CR/LF) entered in CG02 are not displayed in the WEBUI.
  • Text maintained in CG02 may appear as a single continuous block in the WEBUI.
  • HTML formatting (for example <br>, <b>, <i>) maintained in the WEBUI is displayed as plain text in CG02.
  • The same UDT content appears differently depending on whether it is viewed in CG02 or in the WEBUI app.
  • Formatting performed in one UI is not interpreted by the other.

Environment

  • SAP ERP Specifications for Recipe Development on PLM Web UI (PLM-WUI-RCP-SPE)
  • SAP S/4HANA Cloud Private Edition
  • PLM system integration for SAP ERP 

Reproducing the Issue

  1. Open CG02 in the SAP GUI for Windows.
  2. Navigate to a Specification and open a User-Defined Text (UDT).
  3. Enter long text with line breaks (CR/LF) in the SAPscript/ITF editor.
  4. Save the text.
  5. Open the same Specification in the WEBUI Display/Change Specification app.
  6. Navigate to the corresponding UDT and display the text.
  7. Observe that line breaks from CG02 are not shown in WEBUI.
  8. In WEBUI, maintain text using HTML formatting (for example <br>).
  9. Save the text.
  10. Display the same UDT again in CG02 and observe that HTML tags appear as plain text.

Cause

This logic is as designed.

CG02 and the WEBUI Specification application use different text technologies, and no automatic conversion is implemented between them.

  • CG02 stores long texts in SAPscript/ITF format, using CR/LF line breaks.
  • The WEBUI renders text content as HTML, interpreting HTML tags and ignoring SAPscript-style CR/LF formatting.
  • ITF and HTML are technically incompatible and do not translate 1:1.
  • The WEBUI application is not intended to be a full successor to CG02, and complete feature parity is not in scope for Standard Maintenance.

Due to these technical differences, formatting inconsistencies might occur.

Resolution

To avoid inconsistent formatting results:

  1. Use a single interface (CG02 or WEBUI) consistently for maintaining UDT long texts.
  2. Avoid mixed maintenance between SAPscript/ITF and HTML environments.

Currently, no changes to the system are planned. However, you may submit a Customer Influence request to propose this enhancement.

Symptom

You have collected the plan decision recorder trace with the format "indexserver_<host>.<port>.sqlplan.<counter>.trc", when you want to check the trace content and followed the steps in guide Plan Decision Recorder | SAP Help Portal , it's failed to display the result correctly. 

Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.

Environment

  • SAP HANA Database 2.0

Reproducing the Issue

  • With SAP HANA >= 2.00.079.01 and CE 2023.Q3.28 the SQL plan decision recorder is available to record the recompilation state.
  • The trace can be enabled by configuring the parameters listed in 36. How can important optimizer decisions (e.g. for specific execution engines) be determined? (KBA 2000002).
  • For HEX engine, to record the detail reason for why HEX engine is rejected to be used, the parameter "indexserver.ini -> [sql_plan_decision_recorder] -> enable_hex_rejection_reason" is necessary to be enabled with TRUE.

Resolution

Besides the steps listed in the Guide, the trace can also be viewed by using the Excel tool with following steps:

  1. Download the collected "indexserver_<host>.<port>.sqlplan.<counter>.trc" trace file from Diagnosis file from DBACOCKPIT and modify the file suffix with "txt".



  2. Open the Excel and navigate to Data tab and select the option of "From Text/CSV", then select the sqlplan trace file downloaded in step1.



  3. Keep the default option and click "Load" to display the result.

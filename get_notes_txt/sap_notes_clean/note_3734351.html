Symptom

  • SAP Note 2899150 sets searchMode=alphanum for fields with conversion exit ALPHA, but this does not work for NUMC fields without an explicit ALPHA conversion.
  • In CL_SADL_ENTITY_CDS->READ_ENTITY_DATA, NUMC fields are not treated as having ALPHA conversion unless the DDIC domain maintains the ALPHA conversion exit.

Environment

  • ABAP Platform
  • SAP S/4HANA

Reproducing the Issue

  1. Use a SADL-based CDS/OData exposure that includes a NUMC-typed field whose DDIC domain does not explicitly maintain the ALPHA conversion exit.
  2. Perform a search on the NUMC field and observe that alphanum handling (e.g., with leading zeros) is not applied, resulting in incorrect or unexpected search matches.

Resolution

  1. Implement SAP Note 3648692 to enable processing of search on all NUMC fields with searchMode=alphanum.
  2. After implementation, retest the affected SADL-based search scenarios on NUMC fields to confirm expected behavior.
  3. If the note cannot be implemented immediately, as a temporary workaround maintain the ALPHA conversion routine explicitly on the DDIC domain of the affected NUMC fields and verify search results.

Symptom

When using the Export Customizing Transports application, clicking the "Update Dependencies" button does not show the list of dependent transport requests in the "Dependencies" tab in the app immediately after the button is clicked. 

The "Dependencies" tab in the app is not auto-refreshed immediately after the button is clicked.

Environment

SAP S/4HANA Cloud Public Edition 2602

Reproducing the Issue

  1. Go to the Export Customizing Transports app.
  2. Open a transport request that is available to be released.
  3. Click the "Update Dependencies" button.
  4. Expected: Dependent transport requests are listed in “Dependencies” tab.
    Actual: On clicking the “Update Dependencies” the dependent transport requests are not listed, the dependency list is not updated and appears empty or unchanged even though the respective transport request has dependent transport requests.
  5. Refresh the screen (manual browser refresh).
  6. The list of dependent transport requests is then displayed.

Resolution

A fix addressing the auto refresh for the Update Dependencies action has been implemented in the code base and will be included in the SAP S/4HANA Cloud Public Edition 2608 release. Customers running releases prior to 2608 should apply the documented workaround until their tenant is upgraded to 2608.

               

Workaround:

If the dependency list is not shown immediately after clicking "Update Dependencies," perform the following step: Refresh the app

In the Export Customizing Transports app, after clicking "Update Dependencies" and seeing no results, manually refresh the browser (F5) to verify if there are any dependent transport requests.

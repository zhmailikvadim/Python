Symptom

The OAuthV2 policy generates a token response that includes certain properties that are not RFC-compliant.

Environment

  • SAP Integration Suite
  • API Management

Cause

SAP API Management’s built-in OAuthV2 policy is documented as returning a few non-RFC-compliant fields by default (notably token_type: "BearerToken" and expires_in as a string), and its refresh-token expiry error payload also doesn’t match the RFC shape.

Resolution

If the prefix BearerToken is modified to Bearer, then the token should successfully verified.

Given the note in the documentation, the compliant fix is to add a transformation policy after OAuthV2, e.g. AssignMessage policy or JavaScript, to rewrite the response.

1) Make token response compliant (token_type, expires_in)

You want to transform:

  • "token_type":"BearerToken" → "token_type":"Bearer"
  • "expires_in":"3600" → "expires_in":3600 (number)

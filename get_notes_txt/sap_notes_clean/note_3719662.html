Symptom

An MM supplier invoice is posted with a maintained Tax Reporting Date and: Document Date ≠ Posting Date

When the document is reversed in the same posting period, the system clears the Tax Reporting Date in the original invoice.
As a consequence:
•    The original document no longer contains the Tax Reporting Date.
•    The value is missing in statutory tax reporting (e.g. DRC).

Environment

SAP S/4HANA Cloud Public Edition

Cause

This behavior is caused by a system defect. During the reversal process, the system incorrectly clears the Tax Reporting Date in the original document although the value should remain unchanged.

Resolution

A correction for this issue is planned for delivery on 02.03.2026. After the fix is implemented, the Tax Reporting Date will be retained in the original document when the reversal is performed in the same posting period.

Until the correction is available, the Tax Reporting Date can be restored for reporting purposes by using the Manage Tax Items app.

The process:

  1. Run the relevant DRC report, open Manage Tax Items, select the affected tax item, and manually maintain the correct Tax Reporting Date.
  2. After saving the changes and re-running the report, the updated value will be considered in the reporting output.

This workaround updates the reporting data only and does not change the original MM invoice document.

Symptom

Facing below error in PO Mail Channel:

Error in retrieving mailbox size: com.microsoft.graph.http.GraphServiceException: Error code: ErrorItemNotFound
Error message: The specified object was not found in the store., Default folder Root not found.

GET https://graph.microsoft.com:443/v1.0/users/<EMAIL ID>/mailFolders?%24top=50
SdkVersion : graph-java/<version>

404 :
[...]

Environment

  • SAP Process Integration
  • SAP Process Orchestration

Cause

Missing permissions on the Microsoft Graph API side in the Azure configuration.

Resolution

  1. Ensure that the necessary read permissions are configured on the Microsoft Graph API in the Azure configuration.
  2. Refer KBA: 3492888 and define the required permissions.
  3. Perform a full cache refresh using the following URL <host>:<port>/CPACache/refresh?mode=full.
  4. Generate a new refresh token and ensure it is updated on both the PO and server (Azure).

Additionally,

  • Use Graph Explorer/ Postman to check and authenticate the request
  • Cross check your configuration as per SAP Blog: Step By Step PI\PO Mail Adapter OAuth 2.0 Configuration with Office365
    Recreate a brand new Mail sender channel with latest SAP Basis Content version on SAP PI 7.5. After refreshing the token with Microsoft authorization code and adding redirect URL, then check the end to end integrated flow.
  • Check for configured ports and requirements as per SAP NW version in request syntax.

Symptom

  • SAML authentication fails for BI Launchpad.
  • Logon attempts result in a looping behavior between logon.jsp and main.jsp until the maximum allowed number of sessions is reached.
  • Error message displayed: "An error occurred, kindly retry. If issue persists, please contact your administrator. Please close this window."
  • Multiple BI Launchpad sessions are created when checked in CMC > sessions due to automatic login attempts.

Other Terms

SAML, SSO, loop, redirect, infinite loop, endless

Reason and Prerequisites

  • SAP BusinessObjects Business Intelligence (BI) Platform 4.x / 2025
  • SAML

Steps to reproduce: 

  1. Ensure SAML authentication is configured and functioning correctly in previous BI versions.
  2. Upgrade to eg BI 4.3 Support Pack 5 Patch 3.
  3. Attempt to log in to BI Launchpad using SAML authentication.
  4. Observe that multiple concurrent sessions are created while the login attempt retires are made.
  5. Once reached the max limit of concurrent sessions, fails with :"An error occurred, kindly retry. If issue persists, please contact your administrator. Please close this window."

Cause

This issue only happens when using the external (Standalone) biprws for BOE, which is no longer the default

Solution

This issue is fixed in the patches listed in the "Support Packages & Patches" section below.

The "Support Packages & Patches" section will be populated with the relevant patch levels once they are released.

For Business Intelligence Platform maintenance schedule and strategy see the Knowledge Base Article 2144559 in References section.

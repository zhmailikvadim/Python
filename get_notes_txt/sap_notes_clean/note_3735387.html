Symptom

Not able to hide the  custom fields which are added in Enjoy transactions even suppress the field using transactions OB41 and OBC4.

Environment

  • SAP R/3
  • SAP ERP Central Component
  • SAP ERP
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA Finance
  • SAP S/4HANA

Reproducing the Issue

  1. Add a custom field to the FB50 standard screen at the line item level.
  2. Then suppress the field using transactions OB41 and OBC4.
  3. Notice the field is still visible in the screen 

Cause

In the coding, the CI_COBL custom fields not assigned to any screen variant in ENJOY transactions are automatically allocated to the generated fields ACGL_ITEM_GEN-CHAR1~ ACGL_ITEM_GEN-CHAR5, causing the field status not to work.

This is the standard behavior in ENJOY transactions, and it cannot be changed.

Resolution

If you do not want to display certain fields in ENJOY transactions, you must register a new screen variant in transaction SHD0 and set the fields 'ACGL_ITEM_GEN-GEN_CH*' to be hidden.

Set this screen variant as the standard variant for all ENJOY transactions you are using.

Instructions for registering screen variants are detailed in KBA 1910030. Note that all ENJOY transactions such as FB50, FB60, FB70, FV50, FV60, and FV70 use the same program screen (SAPLFSKB '0100') for G/L line items.

Symptom

Customers report that users with hybrid authentication profiles (social login + site password) receive 403007 Permission Denied errors when logging in via social providers (Facebook, Apple, Google) if the linked local password has expired.

This prevents the application from retrieving account information (getAccountInfo) and triggers automatic logout by the SDK.

Example Error Message:

errorCode: 403007
errorMessage: "Permission Denied"

Cause

The SDK and backend enforce the site’s password expiration policy for all accounts with local passwords, including hybrid social accounts. Social login does not bypass the local password check in these SDK versions.

Resolution

  1. Backend Social Login Flow

    • Authenticate users with the social provider token via your backend.

    • Call CDC REST API accounts.socialLogin directly.

    • Inspect the response for expired local passwords and route users to a password reset/change flow.

  2. Adjust Password Policy

    • Remove or extend password expiration for hybrid users if possible.

  3. Passwordless / OTP Migration

    • Convert hybrid users to passwordless or OTP flows to avoid local password validation.

Proposed Feature Request:

  • Add SDK support to allow social login to bypass expired local password checks, or provide a callback to intercept internal validation before logout.

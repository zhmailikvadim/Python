Symptom

Japanese file (Encoding- SHIFT-JIS) not readable

Environment

SAP S/4HANA

SAP S/4HANA Private Edition 

SAP S/4HANA Cloud Public Edition

Reproducing the Issue

  1. Fiori App Manage Incoming Payment Files
  2. Click on Import
  3. Select Payment file for Advanced Payment Management
  4. Choose Converter ID and Upload Files
  5. Check Incoming Payment file Log and Object list Number
  6. In app Manage Payments  select Object List Number and show payload

    Note this is only example, principles in kba apply to all integrations that have Incoming direction.

Cause

Currently only UTF-8 is supported.

Resolution

  • If any other encoding should be read by the engine (such as e.g. Shift-JIS), it must be converted to UTF8 prior upload. 

  • Convert the Shift-JIS encoded file to UTF-8 and then import the file into the Test Incoming Format Mappings app

Symptom

There is a requirement to configure a custom Infotype and have it available in SAP SuccessFactors via the Mashup feature. 

Environment

  • SAP SuccessFactors 
  • SAP ERP 
  • HR Renewal (Personnel & Organization)

Reproducing the Issue

Resolution

The mashup feature allows custom infotypes from SAP ECP to be displayed and maintained directly within SuccessFactors Employee Central interface. This integration enables seamless data management without switching between systems.

It is based on using the application HRPAO_PAOM_MASTERDATA. Taking the example of IT9000 (Germany), maintain view cluster VC_T77PAOMDCF to register the custom infotype for mashup usage:

  • Create entries for infotype 9000 and assign the relevant country/region and actions (create, display, change) as required.
  • Ensure the infotype is enabled for the HRPAO_PAOM_MASTERDATA application context.

Prerequisites

  • Administrative access to both SAP ECP system and SuccessFactors Employee Central
  • Technical knowledge of infotype structures and Web Dynpro components
  • Custom infotype 9000 already created and activated in the ECP system
  • Standard mashup functionality already configured and working

Step-by-Step Configuration

1. Custom Infotype Setup
  1. Verify Custom Infotype Creation: Ensure infotype 9000 is properly created using transaction PM01 in the ECP system
  2. Confirm Infotype Structure: Navigate to SE11 and verify the PS structure (PS9000) contains all required fields
  3. Validate Technical Characteristics: Check transaction PM01 → Technical Characteristics to ensure proper configuration
  4. Test Standard Functionality: Use transaction PA30 to verify the infotype works correctly in the backend system
2. Identify Mashup Components
  1. Access Existing Mashup: Navigate to the working mashup screen in SuccessFactors Employee Central
  2. Get Technical Information: Right-click on the mashup screen and select 'Technical Help' to identify:
    • Web Dynpro application: HRPAO_PAOM_MASTERDATA
    • FPM component: FPM_FORM_UIBB_GL2
    • Component configuration pattern (e.g., HRPAO_DTL_FORM_IT0020_DE for IT0020)
3. BOL Model Configuration
  1. Access GENIL Model Browser: Go to transaction GENIL_MODEL_BROWSER
  2. Enter Component: Input HRPAD and click Display
  3. Create Abstract Structure: Under Abstract Structure, create a new structure HCMT_BSP_PA_DE_R9000 for infotype 9000
  4. Add Customer Include: Within the structure, add customer include CI_DE_R9000 containing the custom fields from infotype 9000
  5. Activate Structure: Save and activate the BOL structure
  6. Validate Model: Run report HRPAD_R_EDIT_MODEL to check for any errors in the HRPAD model
4. Component Configuration Creation
  1. Access SE80: Login to the ECP system and navigate to transaction SE80
  2. Open FPM Component: Navigate to Web Dynpro Component → FPM_FORM_UIBB_GL2 → Component Configurations
  3. Create New Configuration: Right-click and create a new component configuration named HRPAO_DTL_FORM_IT9000_DE
  4. Configure Feeder Class: Set the feeder class parameters:
    • BOL Component: HRPAD
    • Object Name: IT9000
5. UI Enhancement Configuration
  1. Start Component Configurator: Double-click the new component configuration and click 'Start Configurator'
  2. Access Enhancement Mode: In the web browser, click 'Other Functions' → 'Enhance'
  3. Create Field Groups: Add the custom fields by creating appropriate field groups based on screen requirements
  4. Configure Field Properties: Set field attributes, labels, and validation rules as needed
  5. Save Enhancement: Save the UI enhancement configuration
    6. SuccessFactors Integration Setup
    1. Configure Payroll System: In SuccessFactors Admin Center, navigate to Manage Data → Payroll System Configuration
    2. Add Infotype Link: Under Portlets section, add a new link with:
      • Infotype Number: 9000
      • Ensure the related infotype web service is activated
    3. Set Service Configuration: Configure the mashup service to point to the custom infotype

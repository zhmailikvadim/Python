Symptom

You are using the Monitor Capacity Utilization Fiori app. You select a resource to navigate to the Capacity Utilization Object page. However, you notice that the list of operations and shifts displays no data though shifts are maintained in PP/DS and assigned as capacity variants to the resource.

Environment

  • Fiori Advanced Planning Capacity Utilization 
  • UIS4HOP1

Reproducing the Issue

Attempt to view shifts or operations data for a resource in the Monitor Capacity Utilization app.

Cause

The resource settings do not meet the requirements for the Monitor Capacity Utilization app. Specifically, the backend logic checks whether a resource is relevant for external capacity. When a resource is not marked as relevant for external capacity, the app intentionally does not render the shifts and operations sections.

Resolution

  1. Verify the resource settings in the work center/resource master data.
  2. Update the resource to be flagged as relevant for external capacity.
  3. Ensure the associated configuration is consistent with this setting.
  4. After the adjustment, confirm that the shifts and operations sections are displayed as expected.
  5. If the resource is intended for internal capacity only, no changes are required as the current behavior is correct by design.

Symptom

  • Users appear in a table report with employee status Retired but are missing from the functional license count.
  • The steps from “Measuring Functional User Usage in Employee Central” were followed, but wrong count is displayed on SAP For Me or other external reports.

Environment

SAP SuccessFactors Employee Central

Reproducing the Issue

  1. Go to Report Center.
  2. Select Table Report → Single Domain → Person and Employment Info (as of Date).
  3. Provide a report name and description.
  4. Under People > Refine Criteria > Other Filters, select Include inactive users.
  5. Under Columns, add the required columns.
  6. Under Filters > Job Information > Employee Status > By Selection, choose Retired, then select Finished.
  7. Preview, save, and run the report, then download the results.
  8. Compare this list to the functional user usage report and observe that retirees are not counted as functional users.

Cause

Users with employee status Retired may not be included in the functional user usage measurement for visualization, although functional licenses are required for them.

Resolution

Engineering team will develop a report that consistently show the number of pensioners, which can be considered for functional license measurement. The enhancement still does not have a date confirmed, please follow this KBA for updates.

Workaround:

  1. As a temporary measure, use an Ad Hoc Table Report (Person and Employment Info – as of Date) with Include inactive users enabled and filter Employee Status = Retired to identify and monitor the affected population.
  2. In Picklist Center, verify that the “employee-status” picklist includes the Retiree value and is correctly configured.

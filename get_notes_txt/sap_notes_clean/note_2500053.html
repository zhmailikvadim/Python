Symptom

Accounting records with USER only; PROGRAM and/or TRANSACTION missing

Other Terms

DB2 accounting

Reason and Prerequisites

SAP kernel changed behaviour. From 7.42 onwards the SAP kernel provides in some cases USER only at transaction end.
Old implementation:
Accounting/WLM record gets written at transaction start and end if one of the following parameter gets provided by the SAP kernel:
USER, PROGRAM, TRANSACTION.
New implementation:
At transaction start the accounting records gets written if one of the following parameter gets provided by the SAP kernel:
USER, PROGRAM, TRANSACTION.
At transaction end the accounting records gets written if either USER and PROGRAM gets provided or USER and TRANSACTION.

Solution

Please apply fix as described in this note.

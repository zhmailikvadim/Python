Symptom

System Monitoring Instance Local RFC Availability might report a false positive in some rare cases. i.e. although the system is available availability alerts are raised.
This problem only occurs when RFC Ping authentication is used.

When this occurs, the Diagnostics Agent logs contains the following error:

[EXCEPTION]

at com.sap.abap.rfc.function .RfcFunction.execute ( RfcFunction.java:63 )
at com.sap.smd .mai.model.collector .SAPPingRfcCollector.executePing( SAPPingRfcCollector.java:111 )
...

Other Terms

RFC, Ping, Monitoring, SysMon, JCO, NullPointerException

Reason and Prerequisites

A defect in the JCO client pool prevent from a proper reuse of the JCO connection.

Solution

Workaround to be considered is to configure the Ping to not use the authentication. If workaround cannot be applied for security purpose, then deploying the fix is required.

Deploy the LM-SERVICE Software Component, mentioned in the "Support Packages & Patches" section.
Before proceeding with the deployment, pay attention to the general recommendations of SAP note 1483508.

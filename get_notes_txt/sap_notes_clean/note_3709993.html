Symptom

1. SAP Note 3669174 delivered support for Non-taxable income of sport trainers. Feedback received after first month of usage says, that there is a possibility to improve the solution. Originally the Note 3669174 says:
"Real non-taxable income is automatically calculated and the rest income is considered as assessment base for taxation (/106). Really used value is stored as Rate (RTE) field of WT /289 in the final RT table while entered limit is still stored in Amount (AMT) field."
It should be changed vice versa to allow better usage of wage type reporter and other usual tools for wage type processing.
This SAP Note changes the fields usage so after implementation of this SAP Note the really (calculated and) used value of non-taxable income is stored in Amount (AMT) field and the entered limit is now stored in Rate (RTE) field of /289 in final RT table.

2. If any non-taxable income is used in calculation of tax, the same amount should lower also Health insurance base used for HI contributions calculation. There is no need to enter any additional data for HI, the wage type /289 is now used both for tax and HI calculation with the same rules/principles.

3. Rate for Health insurance for special income (dividends) has risen from 14% to 15% since January 1st, 2026. This SAP Note brings the change of constant PZPOS value in T511K with proper validity date.

Other Terms

HRSK, HSKCALCM, HSKCALC0, HSKCREC0, SKCTX, SKCZP, /289, T511K-PZPOS

Reason and Prerequisites

This SAP Note is relevant only for Slovakia (HRSK)

This is a follow-up Note for SAP Note 3669174

Solution

1. Source code of payroll function SKCTX was changed to save entered non-taxable income limit to the RTE field and really used non-taxable income to the AMT field of wage type /289 in final RT table.

2. Source code of payroll function SKCZP was updated to calculate also lowered HI base in case the employee is a sport trainer and has any not-null non-taxable income of trainer accounted during taxation. Health insurance report HSKCZPV0 reports lowered HI base in such case.

3. This SAP Note delivers the change of constant PZPOS value in T511K with proper validity date. Since 01.01.2026 the valid value is 15.

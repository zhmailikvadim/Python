Symptom

MII application  cannot be started. The error message in the default traces is the following:

Error in starting application [sap.com/xapps~xmii~ear]
...

Caused by: com.sap.engine.services.servlets_jsp.server.exceptions.WebDeploymentException: [Cannot load servlet [JEE_jsp_XMII__Menu_<NUMBER>]. Error is: [java.lang.NoClassDefFoundError: JEE_jsp_XMII__menu_<NUMBER> (wrong name: JEE_jsp_XMII__Menu_<NUMBER>) 

Environment

SAP Manufacturing Integration and Intelligence

Reproducing the Issue

Not applicable.

Cause

The issue is related to case sensitivity in the JSP-generated servlet class name. Specifically, the system is searching for `JEE_jsp_XMII__menu_<NUMBER>` (lowercase) but instead encounters `JEE_jsp_XMII__Menu_<NUMBER>` (uppercase), which results in a ClassDefNotFoundError.
Additionally, the error message suggests that this is a deployment issue.

Resolution

Follow the below steps:

1.Navigate to the folder containing the files `JEE_jsp_XMII__Menu_<NUMBER>.class` and `JEE_jsp_XMII__Menu_<NUMBER>.java`. This folder should be located at:  j2ee\cluster\apps\sap.com\u00apps~xmii~ear\servlet_jsp

2. Rename the files as follows:  
   - `JEE_jsp_XMII__Menu_<NUMBER>.class` → `JEE_jsp_XMII__Menu_<NUMBER>.class.bak`  
   - `JEE_jsp_XMII__Menu_<NUMBER>.java` → `JEE_jsp_XMII__Menu_<NUMBER>. java.bak`  

3. Restart NWA server.

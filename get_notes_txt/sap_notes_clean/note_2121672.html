Symptom

You are facing one of the following problems where you can change the status of an activity although the status of the corresponding change request should prevent this:

  1. You created a change request and assigned an activity of category 'Task'. Then the status of the change request is set to 'On Hold'. If you open the assigned activity of category 'Task' and switch to edit mode, it is still possible to change the status of the task.
  2. You created a change request and assigned an activity of category 'Approval', 'Notification', 'Structure Node' or 'Task'. The status of the assigned activity is set to 'On Hold'. Then the status of the corresponding change request is set to 'Rejected' or 'Not Applicable'. If you open the assigned activity and switch to edit mode, it is still possible to 'Remove Hold' and then perform further status changes for the activity.

Other Terms

Change Request

Activity

Reason and Prerequisites

Reason:

Program error

Prerequisites:

1. An activity of category 'Taks' was created and the status of the corresponding change request is 'On Hold'.

2. An activity was created and its status was set to 'On Hold'. Afterwards, the status of the corresponding change request is set to 'Rejected' or 'Not Applicable'.

Solution

Apply the correction of this SAP note.

Symptom

When I perform a data upload (such as a Profile Worker Update Upload) and leave a custom field column empty in my CSV file, the field does not become blank in the system. Instead, the system ignores the empty cell and retains the value from the previous version of the record.

Cause

By default, the SAP Fieldglass upload engine is designed to ignore fields where no value is explicitly provided to prevent accidental data loss. If a cell is left empty, the system assumes no change is intended for that specific field and preserves the existing data.

Resolution

To explicitly set a custom field to a blank value during an upload, you must use a specific reserved string in your upload file.

  1. Open your upload file (e.g., CSV or Excel).
  2. Locate the column for the custom field you wish to clear.
  3. Enter the exact string [Blank] (including the square brackets) into the cell for the relevant records.
  4. Save and run the upload again.
  5. Verify that the new record revision now shows an empty value for that custom field.

Alternative Method:
For the Custom Field Update Upload specifically, if a field is included in the file header but the cell is left empty, the system will set that value to blank. However, for most other standard connectors, the [Blank] string is required.

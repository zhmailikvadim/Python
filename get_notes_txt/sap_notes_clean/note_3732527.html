Symptom

Data inconsistency in iPad and iPhone: users observe inconsistent behavior for Notifications and Work Orders (a work order can be completed on iPad without completing the header notification, while this is not the case on iPhone).

Environment

  • SAP Service and Asset Manager
  • iOS devices: iPad and iPhone
  • MDK Client 25.6.3
  • SSAM Plugin SDK & Branding 2505.0.1

Reproducing the Issue

  1. Sign in to the mobile app on both an iPad and an iPhone using the same user.
  2. Ensure both devices run MDK Client 25.6.3 with SSAM Plugin SDK & Branding 2505.0.1.
  3. On the iPad, complete a Work Order without completing the header notification.
  4. Perform synchronization on both devices (potentially at the same time).
  5. Observe that the completion behavior differs between iPad and iPhone, resulting in inconsistent data between devices.

Cause

Concurrent synchronization across multiple devices under the same user is allowed by design and can lead to discrepancies between device datasets and completion behavior.

Resolution

  1. Review and configure the relevant run-time parameters as described in Note 2713969 for program /SYCLO/CORE_RUNTIME_PARAM_PROG to manage user session handling and synchronization behavior across devices, specifically USE_DEFAULT_DEVICE_ID.
  2. Where applicable, adjust parameters to limit or prevent simultaneous synchronization for the same user across multiple devices to maintain consistent processing of Notifications and Work Orders.
  3. Validate the configuration in a test environment and confirm that work order and notification completion behavior is consistent across iPad and iPhone before moving to production.

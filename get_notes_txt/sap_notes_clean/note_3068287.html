Symptom

Errors in tomcat log:
15-Jun-2021 17:40:19.082 INFO [main] org.apache.catalina.startup.Catalina.start Server startup in 500126 ms
15-Jun-2021 17:40:19.090 SEVERE [main] org.apache.catalina.core.StandardServer.await StandardServer.await: create[localhost:8005]: 
 java.net.BindException: errno: 68 (The socket name is not available on this system.), error: Bind failed (local port 0 to address 0.0.0.0, remote port 8005 to address [some ip address] (localhost.[some.domain.name]))
Followed by
15-Jun-2021 17:40:19.100 INFO [main] org.apache.coyote.AbstractProtocol.pause Pausing ProtocolHandler ["http-nio-8080"]
15-Jun-2021 17:40:19.104 INFO [main] org.apache.coyote.AbstractProtocol.pause Pausing ProtocolHandler ["https-jsse-nio-8443"]

And then

15-Jun-2021 17:40:19.592 INFO [main] org.apache.coyote.AbstractProtocol.stop Stopping ProtocolHandler ["https-jsse-nio-8443"]
15-Jun-2021 17:40:19.594 INFO [main] org.apache.coyote.AbstractProtocol.destroy Destroying ProtocolHandler ["http-nio-8080"]
15-Jun-2021 17:40:19.595 INFO [main] org.apache.coyote.AbstractProtocol.destroy Destroying ProtocolHandler ["https-jsse-nio-8443"]

Environment

This can occur on any OS

This will affect all versions of tomcat

In the case that inspired this article it was:

  • AIX 7.1
  • BusinessObjects 4.2 SP4 P7
  • Tomcat 8.5.13

Reproducing the Issue

When tomcat starts it goes through a sequence of steps.
1) It takes over some memory
2) It opens the ports for listening defined on the Connector lines in the server.xml (8080 for example) and begins queuing user requests (browser spins during this time)
3) It processes the web applications in the webapps folders and populates the work folder
4) It then opens the shutdown port (8005 by default) on localhost
5) It begins handling user requests (browser displays content)

If tomcat cannot do any of these steps it fails at that stage.

In the case of the error above, the problem was that localhost was resolving to a non local ip address. It was failing at step 4 on the list above.  That 'SEVERE' error line in the above section should look like this:

29-Jan-2020 11:34:21.260 SEVERE [main] org.apache.catalina.core.StandardServer.await StandardServer.await: create[localhost:8005]: 
 java.net.BindException: errno: 67 (The socket name is already in use.), error: Bind failed (local port 0 to address 0.0.0.0, remote port 8005 to address 127.0.0.1 (localhost))

Or even better, not mentioned at all because this doesn't normally show up in the catalina.out log. The log entries for starting up normally end after the 'Server startup in xxx ms' line.

localhost was not being properly resolved to it's normal ip address of 127.0.0.1

This was further confirmed by simply doing a ping to localhost at the command line:
ping localhost
PING localhost.company.domain.com (192.168.x.x) 56(84) bytes of data.

That should simply say this instead:
ping localhost
PING localhost (127.0.0.1) 56(84) bytes of data.

Because of this tomcat could not open the shutdown port and complete it's startup.

Cause

The root cause in this case was that localhost was being incorrectly resolved by name resolution.
1) the company domain was being appended to localhost
2) DNS was being used before hosts file was being used (normally the other way around)
3) The localhost.company.domain.com was resolving to an address that did not exist on this host (should be 127.0.0.1)

localhost is a critical part of communications on the host and this has detrimental impact on overall host communications in various ways outside of BusinessObjects and Tomcat.

Resolution

Network or system administrator needs to correct the resolution of localhost.  What is needed to do this depends on how they run their network.

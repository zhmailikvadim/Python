Symptom

  • Mid Month PTAX Region transfer from one state to another, PTAX calculated for both the state
  • PTAX calculated for both the State for Mid Month PTAX Region change
  • Karnataka employee transferred to Assam during Mid month, PTAX calculated for both Karnataka and Assam
  • The payroll deduction amounts are calculated separately for each state based on the respective state legality.

Environment

  • Human Capital Management (HCM)
  • SAP ERP Core Component
  • SAP ERP
  • SAP enhancement package for SAP ERP

Reproducing the Issue

  1. Hire employee in a PTAX Region say example Karnataka
  2. After executing Payroll for some period do mid month transfer to another state say example Assam
  3. Run the payroll calculation usingT-code: PC00_M40_CALC_SIMU
  4. Observe the PTax deduction for employees who transferred from Karnataka to Assam in Middle of the month in INPTX function.
  5. PTAX is deducted for both the states

Cause

Resolution

  1. When an employee transferred in middle of the month, for example in Jun, the payroll system deducts PTAX for both state if the employee eligible for PTAX deduction for the state and PTAX Basis for Karnataka and Assam fall under the PTAX Slab.
  2. The deduction is based on the PTAX slab rate applicable to the employee's state (example Karnataka or Assam).
  4. In this example employee is transferred from Karnatak to Assam on 16-Jun, based on the Transfer the PTAX Basis is split to Karnataka and Assam based on the number of working days in that state, if the PTAX Basis (Proj Basis) fall between the Slab, then that particular slab amount is deducted for that state.
  6. Karnataka PTAX Basis amount is 31,250 which fall under 200 Rs Slab, So it has deducted Rs 200 for Karnataka, Similarly for Assam PTAX Basis amount is 31,250 which fall under 205 Rs Slab amount, So it has deducted Rs 205 for Assam
  7. This behavior is standard functionality of the system and complies with legal requirements.

Symptom

Enduser gets following error *sometimes* while connecting to IQ.
-------------
(-102) [08004] [Sybase][ODBC Driver][SQL Anywhere]Database server connection limit exceeded
Connection 0 failed using DRIVER={SQL Anywhere 12}; CommLinks=tcpip(HOST=...;ServerPort=..);uid=dba;DatabaseName=SAPIQ

***LOG BY2=> sql error -102 performing CON [dbds 1143]
***LOG BY0=> [IQ Error SQL-102:08004][SQL Anywhere]Database server connection limit exceeded [dbds 1143]
***LOG BY1=> sql error -102 [dbacds 1823]
-------------

IQ .cfg file has -gm 100 so IQ is configured for 100 connections.

Environment

  • SAP IQ 16 SP08+
  • Any Platform

Cause

The majority of IQ connections were from Solution manager via a user called SOLMAN_DBA and they are idle.

Resolution

  • Temporary workaround
    • Enduser had high -ti value in SAPIQDB.cfg.
    • With high -ti value like 4400 ( which is more than 70+ hours )
      Following may be happen
      • Solution manager connection ( user SOLMAN_DBA ) comes in
      • Does some work but not disconnect. leaves it upto IQ to cleanup based on idle timeout.
      • Since this values is high, all such connections simply stay there even if they are not truly doing anything.
      • At some point of time, there may be 100 such connections based on how many connections were there is last 70+ hours
        If they were < 100 , you will not see this problem, If they were near 100, you will see the problem.
    • In this scenario, SOLMAN_DBA created 4 connections every 6-7 minutes, so setting -ti to 60 helped working around the problem temporarilly

  • It was further discovered that applying following dbsl patch brought solution manager connections under control

    721_EXT build

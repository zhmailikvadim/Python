Symptom

The SAP BusinessObjects repository may contain inconsistencies between the CMS database and the File Repository Servers (FRS).
These issues may manifest as:

  • Missing objects or orphaned files
  • Objects referencing non‑existent FRS content
  • CMS metadata issues (ACLs, principals, submitter IDs)
  • Runtime errors when users access or schedule BI content

To diagnose or repair these inconsistencies, SAP provides the Repository Diagnostic Tool (RDT) also known as reposcan.exe.

Environment

  • SAP BusinessObjects Business Intelligence Platform 4.x
  • SAP BusinessObjects Business Intelligence Platform 2025

Resolution

What the Repository Diagnostic Tool Does

The Repository Diagnostic Tool (RDT) scans the SAP BusinessObjects repository for inconsistencies.
Depending on the parameters used, the tool can:

  1. ✅ Diagnose and log inconsistencies
  2. ✅ Repair inconsistencies and log the corrective actions taken

The scan result is stored in:

  • Repo_Scan_yyyy_mm_dd_hh_mm_ss.xml
    → Lists all inconsistencies found during the scan.

If the repair option (-repair) is enabled, the tool also generates:

  • Repo_Repair_yyyy_mm_dd_hh_mm_ss.xml
    → Shows all objects repaired and any orphaned FRS files deleted.
    → Lists inconsistencies that could not be repaired.

HTML Report Rendering

An XSL stylesheet is included by default and is automatically associated with the XML output, enabling the scan report to be displayed in a readable HTML format when opened in a browser.

The XSL file is located in:

  • Windows

<INSTALLDIR>\SAP BusinessObjects Enterprise XI 4.0\reposcan
  • UNIX
./sap_bobj/enterprise_xi40/reposcan

Symptom

When executing the Preliminary DME Program using transaction PC00_M99_FPAYM, the program fails to generate a file. It displays the error message "No Payment Media Created in Format &XXXXX" (Message no. BFIBL02159). This typically happens when the "Proposal run" checkbox is selected for a payroll payment run.

Environment

  • Financial Accounting (FI)
  • SAP R/3
  • SAP ERP Central Component
  • SAP ERP
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA

Reproducing the Issue

1. Transaction PC00_M99_FPAYM

2. Execute with run no XXXXXP , Date

3. Error message "No Payment Media Created in Format &XXXXX" (Message no. BFIBL02159)

Cause

This error occurs because the process for payroll payments in PC00_M99_FPAYM is fundamentally incompatible with the 'Proposal run' option. By design, the Preliminary DME Program evaluates final payroll results to create live payment data. This is different from other payment transactions (like F110) that have a separate proposal step.

The program creates a payment run where the "Identification Feature" is system-generated and ends with the letter 'P' to signify a payroll run. For these runs, the system automatically sets the proposal field REGUH-XVORL to SPACE, indicating a live run. When you select the 'Proposal run' checkbox, the program searches for records where REGUH-XVORL equals 'X'. Since this condition can never be met in this process, the program finds no data and issues the error.

Resolution

If you tick the checkbox ‘Proposal run’, the program expects that the value of the field ‘XVORL’ of the corresponding record in the REGUH table to be equal to ‘X’, which means that the payment run is a proposal. Nevertheless, based on the ‘P’ at the end of the identifier you provided, this payment run refers to payroll, in which case it is not possible to create a proposal, as the records of REGU* tables are filled automatically and the value of the field ‘XVORL’ is set to SPACE, which indicate live run.

The Preliminary DME Program creates a payment run that contains payment data and these payment data are directly saved in the following tables: REGUP, REGUH and REGUV. The data in these tables are then used for generating a payment medium file. Unfortunately, the Preliminary DME Program offers no way to create a proposal run the same as in transaction F110/F111. This is the reason why you are only able to generate a payment medium file without ticked ‘Proposal run’.

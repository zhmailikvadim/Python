Symptom

You executed a sync job in IAS/IPS and checking the provisioning log for skipped/failed entities, but the downloaded log file does not show any records.

This KBA will guide you to enable the option in IAS/IPS for capturing skipped/failed entities in the provisioning log.

Environment

SAP Sales Performance Management

  • Incentive Management

Reproducing the Issue

  1. Login to IAS.
  2. Go to Identity Provisioning > Source System.


  3. Select the system for which you want to execute the sync job.
  4. Select 'Jobs'.


  5. Select the 'Read Job' and click on 'Run Now'.


  6. Go to Identity Provisioning > Provisioning Logs.


  7. Search for the recently executed job and check the logs.
  8. Notice that there are some skipped or failed entities in the statistics.


  9. Download the log file.
  10. Notice that the log file does not show any skipped or failed entities but shows the following error message.
    • 'There are no skipped entities for this job execution or ips.trace.skipped.entity property is not enabled.'

Cause

IPS trace options are disabled in the source system.

Resolution

  1. Go to Identity Provisioning > Source System.
  2. Select the source system for which you executed the sync/read job.
  3. Go to 'Properties'.


  4. Here you will find the IPS trace options.
  5. Enable the 'ips.trace.skipped.entity' and 'ips.trace.skipped.entity.content' properties.


  6. Re-run the job. Now you will be able to download the provisioning log that shows the skipped entities in the log file.

Symptom

02/16/2026: The SAP Note text under "Solution" has been adjusted and enhanced with an example.

------------------------------------------------------------

When you execute the report SI: Transfer to SI Procedures  (RP_PAYDE_B2A_SV_UNPACK), the system issues one of the following messages:

  • "Quarantine due to alternative file number (<DTART>  /<BNRAG>/<BNRKK>/<DATNR>): File number <DATNR> is not the next file number <DATNR>" (HRPAYDEB2A 607)
  • "File sequence number <DATNR> for data type <DTART> already processed" (HRPAYDEB2A 006)

When you do this, the displayed file number differs from the file number in the user data. You can use the Display User Data  pushbutton to display the user data in the log. The file number is in the preprocessing record of the file.

Other Terms

eXTra 1.5, complete collection, RP_PAYDE_B2A_SV_UNPACK, RP_PAYDE_B2A_SV_COLLECTOR

Reason and Prerequisites

The counter in the file name in the outer XML (the displayed file number) differs from the file number of the user data. With the conversion to eXTra 1.5/SOAP, the system unwittingly switched from the file number in the user data to the counter in the outer XML to update the V_T5D4U.

  • For message 607, the counter in the outer XML is larger than the file number in the user data.

  • For message 006, the counter in the outer XML is smaller than the file number in the user data.

Solution

Implement the attached correction instructions or import the specified support package.

After you implement the corrections, the file number of the user data is now used again as the file number/file counter for V_T5D4U when you execute the report SI: Get Data from SI Agencies (RP_PAYDE_B2A_SV_COLLECTOR).

If affected data packages still have the status To Be Processed, the file counters of these data packages are corrected automatically the next time report RP_PAYDE_B2A_SV_UNPACK is run. If you are affected by the problem, the system issues the following information in the output log:

"File number adjusted from <DATEINUMMER_ALT> to <DATEINUMMER_NEU>" (HRPAYDEB2A 619)

After you implement this SAP Note, if the system issues the message "File sequence number <DATNR> for data class <DTART> already processed" (006), proceed as follows:

  1. Adjust the file counter in V_T5D4U so that the data packages can be processed with the message. To do this, temporarily delete the existing entries from V_T5D4U.
  2. Execute the report RP_PAYDE_B2A_SV_UNPACK again so that the data package is processed with the message.
  3. Then enter the temporarily deleted entries with the file numbers back to the last processed file number in V_T5D4U.

Example:

The data package with the message "File sequence number <DATNR> for data class <DTART> already processed" (006) has the file number 201.
In V_T5D4U, the file counter is already set to 205.

    1. Copy the entries with the file counters 201 to 205 from V_T5D4U and delete them temporarily.
    2. Execute RP_PAYDE_B2A_SV_UNPACK again. The data package with the file number 201 is now processed.
    3. Then enter the temporarily deleted entries with the file numbers 202 to 205 in V_T5D4U again. 

After you implement this SAP Note, if you receive the message with the number 607, wait two more days to determine whether the quarantine resolves itself. Otherwise, maintain the missing file numbers in V_T5D4U.

If you have not saved the output log, you can use transaction SE16 to select the affected data packages in the database table PC01B2A_SV_P_HI. To do this, maintain the value 619 in the MSGNO  field.

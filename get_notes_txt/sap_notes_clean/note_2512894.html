Symptom

Storing set point group on a Routing with Step ID leads to SetPointGroupNotFoundForProductionException.

Environment

  • SAP Manufacturing Execution (MFG-ME)
  • SAP Manufacturing Execution 15.0
  • SAP Manufacturing Execution 15.1

Reproducing the Issue

  1. Create Material.
  2. Create Operation.
  3. Create linear Routing.
  4. Create Resource.
  5. Create and release Shop Order.
  6. Create Set Point Group with "releasable" status, specify Set Point Parameter, Data Type, Set Point String Value, Material, Routing, Step ID, Resource, Shop Order.
  7. Call SetPointProductionService.useSetPointGroup API request.

Cause

The current API: useSetPointGroup does not allow to pass Router and RouterStepId. It only allows to pass in Operation, Resource, Material and Identifier. Passing in Operation, which is placed at a Router at a specific Step will be handled as two different objects like Operation and Router+StepId (even though that operation is placed at that particular router and that particular step).

Resolution

System works as designed. Possible workarounds depending on scenarios:

  1. Attach SetPointGroup to: Resource, Material, Operation and ShopOrder. And then while using useSetPointGroup API, pass in Resource, Material, Operation and ShopOrder (along with Site).
  2. Attach SetPointGroup to: Resource, Material and ShopOrder. And then while using useSetPointGroup API, pass in Resource, Material and ShopOrder (along with Site).

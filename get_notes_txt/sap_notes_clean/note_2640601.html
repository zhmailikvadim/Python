Symptom

When using 100% quality inspection with multiple steps in a routing, the system incorrectly changes SFC to Complete Pending status at quality inspection step when 'DISPLAY_PLUGIN_100QI' rule is set to 'NO'. The issue is occurs in standard Work Center and Work Center Touch pods.

Steps to Reproduce:

  1. Configure SAP ME for 100% Quality Inspection. (See Integration of Quality Inspection Lot section in SAP ME online help.)
  2. In SAP ME Activity Maintenance for activity QM030, set 'DISPLAY_PLUGIN_100QI' rule to 'NO'.
  3. Configure ECC production order having 2 step routing with 100% sampling and data collection occurring at each routing step. Send to SAP ME.
  4. Release the production order in SAP ME.
  5. In Work Center pod, start SFC at first operation.
  6. Click Quality Inpection button.

Expected:

  • The SFC remains in Active status and data can be collected.
  • The SFC can be processed at the second step in the routing.

Actual:

  • The system hangs momentarily then places the SFC in Complete Pending status and data cannot be collected.
  • The SFC cannot be processed at the second step in the routing.

Other Terms

SAPMEINT, ECC, 100% quality inspection, QM030

Reason and Prerequisites

Programming error.

Solution

The fix is implemented in a form of code change. To apply the solution, deploy the latest patch version of SAPMECORE component. 
 
 Impact 
 
 The code change done to address the symptom above is in the SAPMECORE component and isolated to InpspectionLotPlugin and CompletePluginGUIService modules.

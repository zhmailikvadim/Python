Symptom

You want to create a top-level package (i.e. a package without super package), e.g. in software component HOME, with a package type other than "Structure Package". However, this is not possible because the package type "Structure Package" is preselected and cannot be changed.

Other Terms

Package Builder, Package Editor, Package Wizard, SE21

Reason and Prerequisites

The reason for this is that the control logic is more strict than in the previous releases. Since the package type "Structure Package" was not enforced for top-level packages in non-cloud software component types in the past, this is an incompatible change that is revoked by this note. However, it is still recommended that a package hierarchy starts with a structure package like described here.

As a prerequisite, your system was already upgraded to support package 1, or you have implemented the SAP note 3613719 and already. Additionally, the SAP note 3706845 is required.

Solution

Apply the attached correction or install support package 2.

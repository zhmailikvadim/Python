Symptom

When you try to edit Tax Rates under transaction FTXP, some fields are grayed out while others are available to edit.

Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.

Environment

  • SAP ERP
  • SAP S/4HANA
  • SAP S/4HANA Cloud Private Edition

Reproducing the Issue

  1. Start transaction FTXP
  2. Select tax code
  3. Enter tax jurisdiction code related to Canada
  4. Open edit mode
  5. Tax rate fields grayed out

Cause

Expected behavior due to jurisdiction code levels defined in table TTXD: rates must be maintained at the appropriate jurisdiction level.

On the above example, Federal taxes is locked while using "Tax Jurisdiction" for region and unlocked while using the "Tax Jurisdiction" for federal GST (CABC and CA00 respectively).

This will be based on details from table TTXD along with the TAx Jurisdiction parameters as example below:

Considering examples CA00 and CABC the system considers the level as:

  • CA00 - Level 1, as only the first 02 blocks are non-zero.

  • CABC - level 2, as first 02 and second 02 blocks are non-zero

This is considered on FTXP transaction in order to determine which tax rate is available to adjust, crossing details with A053/A003 tables as well.

Resolution

Tax rate field is "level" dependent and the Tax Jurisdiction selected will impact on it being available to edit or not.

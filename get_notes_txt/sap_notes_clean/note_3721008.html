Symptom

When tags are added from the Application page, they sync to the candidate profile successfully. However, the addition of tags is not recorded as the latest update in the candidate profile as the last modified date of the remains unchanged.

Environment

SAP SuccessFactors Recruiting Management

Reproducing the Issue

1. Access any candidate application.  
2. Add a tag to the candidate profile from the Application page.  
3. Verify that the tag is visible in the candidate profile tags.  
4. Check the candidate profile history and confirm that the tag addition is reflected.  
5. Query the candidate profile's last modified date using Integration Center or API.  
6. Observe that the last modified date does not update and reflect the current date.
7. Add the tag directly to the candidate profile via candidate search or "View Profile" from the Application page.  
8. Query the candidate profile's last modified date using API and confirm that the date reflects the tag addition.  

Cause

This behavior is expected and it is specific to tags.

Resolution

Engineering and Product has reviewed the functionality and confirmed that the last modified date does not update when tags are added within the application page.

To ensure the last modified date is updated:

  • Users should add tags directly within the candidate profile. This action will automatically update the last modified date to the current date.
  • Alternatively, users can click "View Profile" from the Application page to access the candidate profile page and add the tag there.

Symptom

No commitment value is displayed for purchase document in KSB2 or in any CO reporting when the cost center is the account assignment and message RW011 is issued.

Environment

  • Controlling (CO)
  • SAP R/3
  • SAP R/3 Enterprise 4.7
  • SAP ERP Central Component
  • SAP ERP
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA

Reproducing the Issue

  1. Access SAP transaction KSB2.
  2. Enter the purchase document number.
  3. Observe that no commitment value appears/ message RW011 is issued when AC commitment document is displayed

Cause

  • The "Commitment Update" is set to "inactive" for the cost center in the controlling area, preventing the program from updating the purchasing commitment in the CO tables.

OR

  • The "Lock Indicator for Commitment Update" is set to "X" in transaction KS03 on the "Control" tab.

OR

  • The cost element used is a balance sheet account (without "Record account assignment" flagged).

Resolution

  1. Ensure that the commitment update is active for the controlling area associated with the cost center in transaction OKKP.

  2. Ensure that the “Commitments” indicator is not set to lock in the Control tab of the cost center in transaction KS03.

  3. Ensure that the cost element is a P&L account, in the case of a balance sheet account, the “Record account assignment” indicator should be active (Cost Element Type 90) in FS00/KA02.

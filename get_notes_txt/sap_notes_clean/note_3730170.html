Users require guidance on common functional queries regarding the Cost Center Review Booklet app, specifically concerning period selection, hierarchy displays, and filtering logic for G/L accounts and cost centers.

Environment

SAP S/4HANA Cloud Public Edition

Reproducing the Issue

Cause

The Cost Center Review Booklet utilizes a specialized multidimensional interface that differs from traditional grid-based reports. This leads to questions regarding how to modify the default hierarchical views or include specific data sets.

Resolution

1. Selecting Data for Multiple Periods

To report on more than a single period, follow these steps:

  • Locate the Filter icon in the right-side toolbar.
  • Find the Fiscal Year Period field.
  • Use the value help to select multiple individual periods or a range.

2. Disabling Cost Center or G/L Account Hierarchies

If a flat list is preferred over a hierarchical tree structure:

  • Right-click on the Cost Center or G/L Account column header.
  • Navigate to Dimension > Properties.
  • In the Hierarchies tab of the pop-up window, deselect the Display hierarchically checkbox.

3. Displaying Data Without Cost Center Assignment

The Cost Center Review Booklet is specifically designed for cost center-centric reporting. Therefore, the system is pre-filtered to ensure a cost center is always present.

  • While page filters allow switching between Real Assignment and Statistical Assignment, it is not possible to display data where the cost center field is empty.
  • This design choice ensures data consistency and prevents the misleading results often found in broader reports where no preselection exists.

4. Viewing All Cost Centers and G/L Accounts via Filters

To ensure the report encompasses all available master data:

  • Cost Center Hierarchy: This is preset to the Standard Hierarchy by default. Since the Standard Hierarchy includes every cost center defined in the controlling area, no further adjustment is needed to see all cost centers.
  • G/L Account Hierarchy: To include all G/L accounts, select the Not Assigned option within the G/L account hierarchy filter.

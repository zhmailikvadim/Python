Symptom

Selective deletion on a DSO fails with a SYNTAX_ERROR short dump. 
In the short dump source code, the following statement can be found:

* #CP-SUPPRESS: ZUK:tear it down
    DELETE REPORT l_sel_repid.




Environment

SAP NetWeaver 7.5

Cause

The affected DSO does not have a description maintained.
This causes an inconsistency during runtime, resulting in a SYNTAX_ERROR short dump.

When performing a check on the DSO in transaction RSA1 (DataStore Object > Check), the following warning message is displayed:

DataStore Object <DSO name>: Specify a description
Message no. RSDODSO121 

Resolution

  • Go to RSA1
  • Open the affected DSO
  • Switch to change mode
  • Maintain a value in the description field
  • Save and activate the DSO
  • Retry the selective deletion

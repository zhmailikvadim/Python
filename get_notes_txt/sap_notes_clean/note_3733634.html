Symptom

  • While configuring Custom Rules in Trading Partner Management (TPM), the user cannot find options for:
    • EDIFACT
    • ASC X12
  • Only XML-based formats are available for rule creation.

Environment

  • Cloud Integration
  • SAP Integration Suite

Cause

  • Trading Partner Management currently supports Custom Rules only for XML-based type systems.
  • Supported type systems include examples such as:
    • GS1 XML
    • IDoc
    • SOAP-based formats
  • EDI flat-file formats like EDIFACT and ASC X12 are not supported for Custom Rules.
  • This is a standard product limitation (missing functionality).

Resolution

  • This behavior is expected and cannot be changed via configuration.
  • There is no workaround to enable EDIFACT or ASC X12 in Custom Rules.

Recommended approaches:

  • Use XML-based message formats when working with Custom Rules.
  • Convert EDI flat files (EDIFACT / X12) to XML before rule processing.
  • Implement logic in integration flows (e.g., iFlows) instead of TPM Custom Rules.
  • Monitor SAP roadmap updates for future support of EDI flat-file Custom Rules.

Symptom

External client tools (for example Microsoft Power BI, ODBC, JDBC or hdbcli applications) are unable to connect to SAP HANA Cloud.

The ODBC test connection fails with errors similar to:

SQLSTATE: 08S01
Native error: -10709
Communication link failure
rc=10061: No connection could be made because the target machine actively refused it

The connection attempt is typically made to a direct IP address or to a hostname that does not correspond to the SQL endpoint of the SAP HANA Cloud database instance.

The SAP HANA database itself is running and accessible from SAP applications, but external database connectivity fails.

Environment

SAP HANA DB Version 2.0

Cause

The client connection does not use the SQL endpoint assigned to the SAP HANA Cloud database instance.

SAP HANA Cloud database connections from external client tools (for example ODBC, JDBC, Power BI or hdbcli) must use the SQL endpoint assigned to the SAP HANA Cloud database instance.

If the client connects using a different hostname or endpoint than the SQL endpoint provided for the instance, the connection attempt may fail with errors such as:

Connection failed (RTE:[89006] System call 'connect' failed, rc=10061: No connection could be made because the target machine actively refused it)

Resolution

Retrieve the SQL endpoint of the SAP HANA Cloud database instance in the SAP BTP Cockpit.

SAP BTP Cockpit → SAP HANA Cloud → Database → Connectivity → SQL Endpoint

Configure the client connection (Power BI, ODBC, JDBC, hdbcli or other SAP HANA client tools) using the SQL endpoint provided for the database instance.

Do not replace the SQL endpoint with direct IP addresses or alternative hostnames.

After correcting the connection endpoint, verify that the SAP HANA database user used by the client has the required privileges and retry the connection.

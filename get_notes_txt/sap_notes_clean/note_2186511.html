Symptom

The handling of different archiving objects is inconsistent. This SAP Note is part of a series of SAP Notes for optimizing data archiving across different applications and concerns the archiving object QM_QMEL. For more information about this, see SAP Note 577847.

Other Terms

Archiving, ADK, transaction SARA, archiving object QM_QMEL, maximum size in MB, commit counter, RQARCQMV, RQARCQMA, Customizing settings QM_QMEL IRM_CHECK_CUST, Information Lifecycle Management, ILM, ILM object QM_QMEL, termination_flag, suspension

Reason and Prerequisites

a. Time reference fields and condition fields of the ILM object for the archiving object QM_QMEL in IRM Customizing are insufficiently defined and assigned.

b. The write program of the archiving object QM_QMEL does not support the suspension concept. This means that the write phase of an archiving run for the object QM_QMEL cannot be suspended.

c. The maximum size in MB does not correspond to the standard recommendations.

d. No binding selection block for restrictions exists in the write program RQARCQMA and the preprocessing program RQARCQMV.

e. For QM_QMEL, hints are entered in AOBJ Customizing.

See SAP Note 577847.

Solution

a. Time reference fields and condition fields of the ILM object for the archiving object QM_QMEL in IRM Customizing are correctly defined and assigned.

b. If you want to use the function described, proceed as follows:            
    1. Implement the attached source code corrections in the program RQARCQMA.
    2. With transaction AOBJ for the archiving object QM_QMEL, set the indicator that allows suspensions.

c. As a result of the corrections, in the Customizing (transaction AOBJ) for the archiving object QM_QMEL, the maximum number of data objects is set to the default value 0, the commit counter is set to 1, and they are transferred in the case of new deliveries.

d. The selection block for restrictions in the write program RQARCQMA and the preprocessing program RQARCQMV is enhanced.

e. The hints for QM_QMEL are removed from AOBJ Customizing.

Symptom

When you manually add a phantom item in a planned order, you find that the phantom item is not exploded automatically. Additionally, when you convert the planned order to a production order, the phantom item shows as a normal item, and the "phantom" indicator is not marked.

Environment

SAP S/4HANA Cloud Public Edition

Cause

This is standard system behavior.

The phantom item will be exploded during the MRP run, however, when manually added to a planned order, the system will not check whether the material is a phantom item and will treat it as a normal item.

Resolution

If a phantom item needs to be included in a planned order, ensure it is maintained in the BOM.
If the phantom item needs to be temporarily added, manually add it in the production order.

Keywords

BOM, Phantom item, Phantom assembly, Production order, Dummy assembly, Manually added, Automatically exploded, Planned order

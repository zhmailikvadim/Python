Symptom

A storage bin is blocked for putaway or stock removal, and once it is searched in /SCWM/ADGI, this block is not displayed and do not prevent Goods Issue (GI) posting.

Environment

  • SAP Extended Warehouse Management (SCM-EWM)
  • SAP Extended Warehouse Management Goods Movement (SCM-EWM-GM)
  • SAP Extended Warehouse Management Warehouse Order Processing (SCM-EWM-WOP)
  • SAP S/4HANA

Reproducing the Issue

  1. Block a storage bin for putaway or stock removal;
  2. Search for this storage bin in /SCWM/ADGI;
  3. Try to post the stock;
  4. No error will be raised and posting will not be prevented.

Cause

The blocks are intended only for Warehouse Task (WT) creation. It is not intended that these blocks are used to prevent posting changes or GI postings in the EWM standard. 

Resolution

If the requirement is to prevent the posting from storage bins that are blocked in /SCWM/ADGI, BAdI /SCWM/EX_CORE_GM_STOCK can be implemented to read the storage bin and raise an error/exception, preventing this posting.

Symptom

After Google Chrome/Edge upgrade, the following error appears when accessing external catalogs: 

"The connection is blocked because it was initiated by a public page to connect to devices or servers on your private network."

Environment

  • Materials Management (MM)
  • SAP S/4HANA
  • SAP S/4HANA Cloud Private Edition

Reproducing the Issue

This is just one example, but the same error can occur in different applications:

  1. Open the Create Purchase Order - Advanced app
  2. Click Catalogs icon
  3. Select any external catalog 
  4. Error is shown

Cause

Starting with Chrome 142 and Edge 143, new security restrictions were implemented that prevent public HTTPS pages from accessing private network resources.

Refer to SAP Note 3683742 for more details.

Resolution

  • Temporary workarounds:
    • Option 1: Administrators can set the LocalNetworkAccessRestrictionsTemporaryOptOut policy to temporarily reverse the changes in browser behavior.
    • Option 2: Visit chrome://flags/#local-network-access-check (or edge://flags/#local-network-access-check if you are using Edge browser) and disable "Local Network Access Checks." This temporary measure is effective until Chrome 152 (August 2026).
    • Option 3: Add external domains such as *.publicdomain.com in the browser settings (Chrome: chrome://settings/content/localNetworkAccess; Edge: edge://settings/privacy/sitePermissions/allPermissions/localNetworkAccess) within the "Allowed to connect to any device on your local network" section.

      After changing the settings, save and restart your browser.

  • Long-term solutions:
    SAP is monitoring official updates and will update this document as soon as new solutions become available.

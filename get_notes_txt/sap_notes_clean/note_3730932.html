Symptom

The documentation of BAdI SWF_FORWARD_AGENTS_GET_CUSTOM explains, that the list of potential owners can be influenced by an own implementation at action forward in My Inbox. This documentation is misleading, because the customer gets the impression that the value help will show only those agents, which are allowed as recipient. This is not the case. The custom implementation is called at save time. Therefore the end user is able to choose a recipient, which will not be accepted at save time.

Other Terms

 Documentation, My Inbox, Forward

Reason and Prerequisites

The value help of users in My Inbox is context free and is used for different actions (e. g. forward, create substitution). The taskprocessing service does not know that the user list is requested for the forwarding action of a task instance. Therefore the provided user list cannot be restricted and the check is triggered only at save time.

Solution

Install the corresponding support package or implement the assigned correction instruction.

With the correction instruction the documentation will be updated in original language. It does not contain any coding changes.

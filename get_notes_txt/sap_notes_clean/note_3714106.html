Symptom

SAP Customer Checkout + SAP Customer Checkout manager 2.0 FP21 PL23 can be used for fresh installation also for upgrading from previous versions.

Other Terms

 SAP Customer Checkout + SAP Customer Checkout manager 2.0 FP21 PL23

Reason and Prerequisites

 SAP Customer Checkout + SAP Customer Checkout manager 2.0 FP21 PL23

Solution

SAP Customer Checkout + SAP Customer Checkout manager 2.0 FP21 PL23 following issues/features have been fixed or added.

SAP Customer Checkout: Pepper PlugIn

  • Symptom: Payment on terminal issue with a different currency than requested by the POS. A customer had to pay approximately 250 EUR but chose to pay in a foreign currency (e.g., Argentine Pesos). On the terminal, ARS was selected, and approximately 400,000 ARS were charged. The POS processed this as EUR because it was unaware of the currency change. Currently, the currency from the transaction result is not mapped. Since the amount differed from the initial request, the payment item was updated to the new numerical value. If no exchange rate is configured for the foreign currency, the POS should not update the amount but should instead retain the original amount if a currency change is detected.

  • Fix: Fixed the incorrect amount maintenance after paying with a different currency on the terminal.

SAP Customer Checkout: Pepper PlugIn

  • Symptom: To resolve payment terminal issues with multiple card payments for a single receipt, an option was introduced to configure a delay before each card payment. However, a delay should only occur between two card payments, not before the first one.

  • Fix: Adapted the delay between card payments in the plugin configuration to ensure a delay is only added between two card payments, but not before the first one.

SAP Customer Checkout: Cashing-up report

  • Symptom: Some printed cashing-up reports show number of PoS systems: 0.

  • Fix: Fixed a bug that caused some cashing-up reports to show the number of POS systems as 0 when Business Day mode is set.

SAP Customer Checkout: Performance

  • Symptom: For large datasets (e.g., from 2016 to 2026), the optimization strategy performs well. However, for small timeframes (e.g., triggering a Cash-In from the POS for the current state only), the strategy leads to excessive overhead and is too slow.

  • Fix: Improved the performance of POS-triggered Cash-In operations.

SAP Customer Checkout: US local tax mode

  • Symptom: A rounding amount of 0.01 penny is calculated and displayed on the receipt.

  • Fix: Fixed rounding issues in US local tax mode.

Customer Checkout: Quantity type code

  • Symptom: The quantity type code for selection materials was missing from the selection structure export.

  • Fix: The quantity type code for selection materials is now included in the selection structure export.

SAP Customer Checkout: TSE/DSFINV-K

  • Symptom: When a "No Sale" receipt is posted from the POS, the signature in the Receipts app in the SAP Customer Checkout manager is empty, and the transaction end time in the archive is null.

  • Fix: Fixed the issue where the TSE signature was empty in the Receipts app and the transaction end time was null (Archive -> Integration Suite) for "No Sale" receipts.

SAP Customer Checkout / SAP Customer Checkout manager: TSE/DSFINV-K

  • Symptom: In the DSFinV-K export, sales items from canceled receipts are not displayed in lines.csv.

  • Fix: Sales items from canceled receipts are now shown in lines.csv during DSFinV-K exports.

SAP Customer Checkout / SAP Customer Checkout manager: TSE/DSFINV-K

  • Symptom: Print-outs for Cash-in/Cash-out and Pay-in/Pay-out should contain a QR code and the same information as a standard sales receipt.

  • Fix: Added QR code printing for Cash-in / Cash-out and Pay-in / Pay-out transactions.

SAP Customer Checkout / SAP Customer Checkout manager: TSE/DSFINV-K

  • Symptom: Debtor items (payment on credit) are currently handled like payment items. For DSFinV-K exports, they should be handled as sales items.

  • Fix: Fixed issues with payment on credit in the DSFinV-K export.

SAP Customer Checkout / SAP Customer Checkout manager: TSE/DSFINV-K

  • Symptom: The generated DSFinV-K ZIP archive contains a file called parameters.txt. This file is generated by default by the jobs framework and causes issues when read by services that validate the archive.

  • Fix: Removed the parameters.txt file from the DSFinV-K ZIP archive.

SAP Customer Checkout /SAP Customer Checkout manager: Discount

  • Symptom: In some scenarios involving total amount recalculations ("flops"), the discount is inconsistently removed.

  • Fix: Discount calculation is now working as expected.

SAP Customer Checkout / SAP Customer Checkout manager: Coupon influencer scenario

  • Symptom: If multiple SAP Customer Checkout systems request an influencer coupon simultaneously, all POS systems receive the same coupon instance. This causes the local assignment check to falsely succeed, resulting in all systems assigning the same coupon.

  • Fix: In the coupon influencer scenario, a separate new instance is now created for every coupon request instead of returning the first valid instance.

SAP Customer Checkout manager: MS SQL Exception

  • Symptom: In MS SQL databases, TIMESTAMP is considered a row version rather than a date/time. When CAST(null as TIMESTAMP) is used for a column of type Date, the database throws the following exception: Operand type clash: timestamp is incompatible with datetime2.

  • Fix: Fixed the database exception occurring in MS SQL when the archive communication arrangement is active and "Attach DSFinV-K file" is enabled for the Sales Summary outbound service.

SAP Customer Checkout manager: Article Export

  • Symptom: When an article had multiple price lists removed and one new price list added, filtering the export by the single price list still showed the article linked to the removed lists.

  • Fix: Fixed the issue where prices that had already been removed from an article were still being exported.

New Features Added

SAP Customer Checkout: Encryption Key Management

  • Added functionality to specify custom root encryption key 

SAP Customer Checkout: French Localization

  • Added additional print option for sales line/items in hardware settings
  • Automatic configuration when switching country to France or French territory:
    • Print PoS system and group: ON
    • Cashier info: Print cashier ID only
    • Sales person info: Print sales person ID only
    • Sales item/line quantity info: Print number of sales lines
  • Enhanced settings for existing SAP Customer Checkout installations 

SAP Customer Checkout: Pepper PlugIn Enhancements

  • Added info popups to cashier for customer/merchant receipt signatures and ID verification requirements
  • Implemented automatic "repeat last receipt" transaction (Code 201) for Netherlands credit card terminal requirements

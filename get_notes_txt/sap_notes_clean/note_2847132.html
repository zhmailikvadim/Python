Symptom

  • CPI-DS Tasks that consume REST web services. By transporting these services into the productive environment of CPI-DS, you are unable to change the services endpoint.
  • You have already changed the associated Datastore to use a new WADL file containing the correct address but the job still fails.

Environment

SAP Cloud Platform Integration for data services

Cause

When a datastore is promoted, the metadata stays the same regardless of further changes to the datastore. Part of that metadata is the endpoint URL for the REST service.

Resolution

Instead of updating the endpoint URL, change the Dynamic Base URL. This datastore parameter is specifically designed for runtime adjustments.

In documentation:

  • Dynamic Base URL: The base URL comprised of the protocol, server name, port number, and path of the service that listens to RESTful web service requests.You can obtain this information from the WADL file under the element resource and attribute base. This is useful when you have multiple servers serving the same resources and you want to dynamically create the URL during the call.

When you mouse over this field in the datastore, the description is more clear:

  • If the base URL is different from the one defined in WADL. Note: The URL is only processed at runtime.

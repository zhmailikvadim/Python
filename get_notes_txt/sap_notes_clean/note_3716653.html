Symptom

Information stored in the Commerce Cloud database must be retrieved or altered. There is no way to access or modify this data via other methods. This can apply to table data, indexes, table structure and more.

Environment

Commerce Cloud in the Public Cloud

Resolution

*Warning, directly modifying Commerce data on a database level is not generally recommended, only proceed with the below steps if you are confident you will not break any Commerce functionality*

Whenever possible, items should be modified programmatically via the Service Layer and Commerce logic. Direct SQL should only be used if the former is not possible. Directly altering database data risks breaking relations between items. 

SQL can be executed 2 ways from the Administration Console:

1) Console > Flexible Search > SQL Query

When altering data, ensure the "Rollback" toggle is set to "Commit".
This interface is limited in the SQL that is allowed, many statements may get rejected for not being valid SQL. For these situations an alternate option via scripting can be used.

2) Console > Scripting Languages

When altering data, ensure the "Rollback" toggle is set to "Commit".

The following Groovy code snippet can be used; the SQL statement can be altered to perform other operations:

import de.hybris.platform.core.Registry
conn = Registry.getCurrentTenant().getDataSource().getConnection()
stmt = conn.createStatement()
ddlQuery = "TRUNCATE TABLE customType" //replace with the required SQL statement
stmt.executeUpdate(ddlQuery)

Check the "Result" tab if results are expected to be returned.
Check the "Stack trace" for any errors. 
The Commerce user account has limited permissions, some SQL statements may not be allowed to be executed.

Keep in mind, this completely bypasses the Commerce item caching and invalidation mechanism. Items modified this way may still be retained in the local cache of this or other Commerce pods, leading to stale or invalid data. To ensure the cache is cleared, all Commerce pods should be restarted afterwards to ensure the cache is cleared on all pods and that current data is loaded from the database.

If you wish to run a query and export that data to a file, please check KBA 3723895.

Keywords

SQL Query, Modification, DDL, DML

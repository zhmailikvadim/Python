Symptom

  • A custom MDF object is experiencing an issue where the direct manager's comment field does not function correctly.
  • When the direct manager fills the comment field, an error message is triggered, indicating an incorrect validation rule.
  • The issue occurs during the approval workflow process whose request is pending with the direct manager.
  • The error persists even after the comment is added by the direct manager.

Environment

  • SAP SuccessFactors HCM Suite

Reproducing the Issue

  1. Login to the SAP SuccessFactors instance.
  2. Navigate to the custom MDF object via approving request in homepage
  3. Assign the request to the direct manager.
  4. Direct manager fills the comment field and attempts to approve the workflow.
  5. Observe the error message triggered despite the comment being filled.

Error: Please Update the Comment field and approve the workflow

Cause

The "General Comment" box is part of the default workflow and cannot be customized or included in the onSave rules to make it mandatory. The error occurs because the onSave rule requires comments from specific approvers, and the workflow fails if these fields are left blank.

Resolution

  1. Confirm that the "General Comment" box is part of the default workflow and cannot be customized or included in the onSave rules. (Red box)
  2. Ensure that the approver's comment field specific to an approver is filled to meet the onSave rule requirements. (Green box)
  3. If the error persists, create validation rules for specific approvers to ensure their respective comment fields are filled.
  4. Test the workflow after implementing the validation rules to confirm functionality.

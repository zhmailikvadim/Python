Symptom

  • For some candidates, the country field in the recruiting v2 table report displays "-1".
  • The issue persists even though the country field is not configured to accept "-1" as a valid value.
  • The audit history indicates that the candidates initially had blank country values, which were later changed to "-1".
  • The issue is observed when generating reports with a filter for the country field set to "-1".

Environment

SAP SuccessFactors Recruiting Management

Reproducing the Issue

  1. Create a recruiting v2 table report with columns - candidate ID and Country.
  2. Add a filter for the Country field with the value "-1".
  3. Preview the report.

Cause

  • The country code can be defaulted to -1 in DB and same is reports because of following reasons,

    1. Invalid/missing country data
    2. No matching country found in the picklist.

Resolution

  1. Verify the configured country list in the system against the standard guide to identify missing countries.
  2. Add the missing countries to the configured country list in the system.
  3. Update the configured country list to include all valid country values.

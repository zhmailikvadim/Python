Symptom

  • The availability of specific BADIs/user exits for different system versions is unclear.
  • Clarification is needed to identify the compatibility of BADIs/user exits with SAP S/4HANA Private Cloud, SAP S/4HANA Public Cloud, GTS11 with ERP or S/4HANA feeder system, GTS E4H with ERP or S/4HANA feeder system, and ERP.

Environment

  • SAP S/4 HANA for International Trade
  • Intrastat Processing
  • SAP S/4HANA Public Cloud
  • SAP S/4HANA Private Cloud
  • GTS E4H (edition for HANA)
  • GTS 11.0
  • ERP

Resolution

The following Badi's/customer exits are available for implementing custom requirements:


S/4HANA Private Cloud

  • BADI_INTRASTAT_SELECTION
    • Method: CHANGE_DATA - Change Data for Intrastat Declaration
    • Method: GET_PURCHASE_DOCS - Get additional purchasing documents
    • Method: GET_BILLING_DOCS - Get additional billing documents


S/4HANA Public Cloud (3 different badis are available)

  • INTRASTAT_CHANGE_DATA
    • Method: CHANGE_DATA - Change Data for Intrastat Declaration
  • INTRASTAT_GET_BILLING_DOCS
    • Method: GET_BILLING_DOCS - Get additional billing documents
  • INTRASTAT_GET_PURCHASE_DOCS
    • Method: GET_PURCHASE_DOCS - Get additional purchasing documents

GTS11 with ERP or S/4HANA feeder system (badi is available in the feeder system)

  • /SAPSLL/ECR_EXTRACT
    • Method: IS_SD_CHANGE_MAPPING - Mapping for SD Issues and SD Receipts (Returns)
    • Method: IS_MM_CHANGE_MAPPING - Mapping for MM Receipts
    • Method: IS_SD_IB_CHANGE_MAPPING - Mapping for Intercompany Billing (SD) as MM Transactions
    • Method: IS_MM_RET_CHANGE_MAPPING - Mapping for MM Issues (Returns) 

GTS E4H (edition for HANA) with ERP or S/4HANA feeder system (badi is available in the feeder system)

  • /SAPSLL/INTRASTAT_SELECTION
    • Method: CHANGE_DATA - Change Data for Intrastat Declaration

ERP

  • EXIT_SAPLV50G_001:
    • Influence data selection and modify document header and item information during the declaration process. Documents can be included/excluded.
  • EXIT_SAPLV50G_002:
    • Modify data immediately before it is written to the intermediate file (VEIAV table).


Note:
Implementation of a user exit is considered a modification and falls under consulting scope, not SAP product support. For more details, please refer to the note 170183 on considerations when applying modifications.

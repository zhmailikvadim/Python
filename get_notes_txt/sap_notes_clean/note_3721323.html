Symptom

  • In SAP Web UI (WebDynpro), if an image file, e.g. PNG, is stored within a Document Info Record, users may wish to view it with SAP 3D Visual Enterprise Viewer.
  • You are trying to implement the Integration of Viewer for DMU Viewing (PLM Web UI)
    • Ref: https://help.sap.com/docs/SAP_ERP/0f3e26f224d9419688b3d25d7c2e46fe/491404533b8a8c4ae10000000a441470.html

Environment

  • SAP 3D Visual Enterprise Viewer 9.0

Reproducing the Issue

  • In SAP Web UI, when selected the image file opens in the Browser instead of SAP 3D Visual Enterprise Viewer, preventing markup or redlining functionality.
  • In SAP GUI, example (CV02N), the process works as intended — double-clicking the original opens the image in SAP 3D Visual Enterprise Viewer, allowing markup and redlining functionality.

Cause

  • All browsers have deprecated support for ActiveX plugins. Consequently, the SAP 3D Visual Enterprise Viewer, previously available as an ActiveX plugin, will no longer function within PLM Web Dynpro applications.

Resolution

Possible Workaround:

  • You can use Microsoft Edge, which offers an IE compatibility mode that supports ActiveX. Detailed instructions can be found in SAP Note: https://me.sap.com/notes/3198562/E

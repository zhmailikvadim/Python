Symptom

Organizations operating globally may need SAP systems to support multiple time zones within a single instance to ensure accurate local timestamping, scheduling, and reporting.

Environment

Release Independent 

Resolution

Its not possible to have multiple time zone for a single instance in SAP, but the users can have personal time zones

  1. Go to transaction SU01
  2. Select the user 
  3. Set a personal time zone as per the necessity 

The system always use first the SU01 personal time zone, in case it don't exist, then the system use STZAC time zone. 

Keywords

SU01, time zone, personal time zone, multiple time zones

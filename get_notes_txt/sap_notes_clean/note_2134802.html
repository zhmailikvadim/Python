Symptom

Files created by SQL Anywhere in /tmp directory are set with 0777 permission.

Environment

SQL Anywhere version 11 and later running on Unix based system.

Resolution

By default SQL Anywhere creates all temporary files it needs for processing in /tmp , which has full read-write permission for everyone.

Starting in version 11 these permission can be restricted by usign SATMP environment variable.

  • create directory , e.g. "/tmp/mytmp" with full permission for the creator only e.g:
    mdir /tmp/mytmp
    chmod 700 /tmp/mytmp
    export SATMP="/tmp/mytmp"
  • restart database server

Note: To use this feature, the directory specified with SATMP environment variable must *not* be one of the "standard locations" listed here:
- /tmp
- /tmp/.SQLAnywhere
- the value of the TMP environment variable, if set
- the value of the TMPDIR environment variable, if set
- the value of the TEMP environment variable, if set
- a symbolic link pointing to any of the above

Also, to make this change permanent it's recommended to update sa_config##.sh (sa_config.csh) file so this variable is set every time SQL Anywhere environment is set.

Symptom

  • SAP ASE error log file from file system level SID.log is filled up quickly.
  • How to clear contents of SAP ASE log files while SAP ASE is running.

Environment

  • SAP Adaptive Server Enterprise (ASE) 15.7 and 16.0 for Business Suite
  • SAP NetWeaver (NW) - All versions

Cause

The available space in the file system has been exhausted.

Resolution

  • Firstly shutdown the ASE Dataserver.
  • And then rename the existing ASE error log file from file system.
     ex: Sybase/<SID>/ASE-16_0/install/<SID>_04022026.log
  • After renaming the existing error log file then start the ASE Dataserver, which will create the new ASE error log in Sybase/<SID>/ASE-16_0/install/<SID>.log as default. 
  • Note: Do not delete/move the current  <SID>.log while SAP ASE is running.

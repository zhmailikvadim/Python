Symptom

  • The system does not support the assignment of a temporary user ID during the recruitment process.
  • The system cannot assign a temporary user ID (e.g., 1001_1) during recruitment and replace it with a real user ID (e.g., 1001) upon completion of the hiring process.

Environment

SAP SuccessFactors Recruiting Management

Cause

Resolution

  1. The current system design does not support the functionality to assign a temporary user ID during recruitment and replace it with a real user ID upon hiring.
  2. To request this feature, refer to KBA 2090228 - How to submit enhancement ideas for SAP SuccessFactors Products.

See Also

  • Refer to: 2090228 - How to submit enhancement ideas for SAP SuccessFactors Products

Keywords

temporary user id, recruitment process, candidate management, onboarding, user id assignment, hiring process, enhancement request, system design limitation

Symptom

When a change process with change order will be triggered for a contract on the (same) start date then there is a runtime error type ASSERTION_FAILED triggered in CL_CRM_BILLPLAN_BILLREQ_I_INTGCP.

Other Terms

ISTA; billing plan; billreq_i; create follow_up

Reason and Prerequisites

The issue is only occuring if note 3690705 is implemented. In case its the same day change then there is no need for re-triggering the billing plan as nothing was changed.

Solution

Correction Instruction.

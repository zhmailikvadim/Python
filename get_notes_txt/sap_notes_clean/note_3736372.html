Symptom

The provisioning process fails when attempting to create a new user through the IPS Proxy system(Ariba). 

Error Response:

{
  "schemas": [
    "urn:ietf:params:scim:api:messages:2.0:Error"
  ],
  "detail": "Failed to create entity. HTTP operation failed invoking https://eu.openapi.ariba.com/api/mds/v1/prod/scim/Users with statusCode: 400 and response: {\"status\":400,\"detail\":\"Error while processing request.\",\"schemas\":[\"urn:ietf:params:scim:api:messages:2.0:Error\"],\"scimType\":\"invalidSyntax\"}",
  "status": 400
}

Environment

  • Identity Provisioning Service (IPS)
  • Identity Authentication Service (IAS)
  • Identity Access Governance (IAG)
  • Ariba

Reproducing the Issue

  1. Set up the integration flow IAS → IAG (request) → Identity Provisioning (Ariba proxy) → Ariba.
  2. Trigger the job from IAG to provision users from IAS to Ariba.
  3. Observe in Ariba that the users are not created.

Cause

The failure occurs because the required fields are not included in the payload as mandated by the Ariba API. This results in a syntax error being reported by the API.

Resolution

Ensure that all the necessary fields are properly included in the API request payload as required by the Ariba API. Below is an example of the correct odm structure including the necessary fields, such as displayId and companyCodeId:

"urn:ietf:params:scim:schemas:extension:sap.odm:2.0:User": {
    "companyCode": [
      {
        "company": "XXX",
        "displayId": "XXX",
        "primary": true
      }
    ],
    "costCenter": [
      {
        "costCenterId": "XXX",
        "companyCodeId": "XXX",
        "primary": true
      }
    ],
    "purchasingOrganization": [
      {
        "displayId": "XXX",
        "primary": true
      }
    ]
  },

Symptom

At the time of the Client copy, transaction log is Full and processing is aborted.

Environment

SQL Server in SAP NetWeaver Products

Cause

Database may require a very large transaction log size because the Client copy performs several large DML commands (delete/update/insert) in a single transaction.

Resolution

The only solution is to be sure that transaction log has Auto Expand set and you have enough disk space available.

For more information please refer to the following SAP notes:
421644 - SQL error 9002: Transaction log is full
363018 - File management for SQL Server

As a workaround, please perform the following:

  1. Perform frequent Transaction Log Backups while the client copy is taking place.
  2. Backup the database, put the database in Simple mode, perform the Client copy, put the database back to Full mode then perform a Full backup again.
  3. Please make sure there is no multiple transaction log files for a SQL Server database.

Symptom

  • Multiple SQL Anywhere repositories show intermittent job failures at different intervals and dates due to network instability.
  • It is necessary to increase the connection timeout, but there is no option in SAP Data Services Server Manager.
  • The jobs fail with the following errors:
    • ODBC call <SQLDriverConnect> to connect database <DS_PROD_SQL> failed with error: <[SAP][ODBC Driver][SQL Anywhere]Connection
      error: Timeout occurred while waiting for connection response>. (...)
    • REP-100108: Initialization Cannot perform operation on repository <Current Repository> because no connection has been opened.

IMPORTANT: SAP Data Services requires a stable connection to the repository. Adjust the timeout only if there is no other option. First, always discuss with the network team to improve latency and identify the reason for the network issue.

Environment

  • SAP Data Services
  • SQL Anywhere as repository

Cause

Timeout issue at network level leading to ODBC connection timeout to the SQL Anywhere repository.

Resolution

  1. Create an ODBC DSN for each affected SQL Anywhere repository connection.
  2. In the DSN, add the connection parameter AppConnTimeout in the Advanced tab and Additional connection parameters section. For example, AppConnTimeout=120
  3. In Data Services Server Manager, recreate the repository definitions to point to the newly created DSN so the AppConnTimeout parameter is used.

See Also

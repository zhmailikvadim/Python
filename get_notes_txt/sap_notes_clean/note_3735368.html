Symptom

  • When an existing custom task is deleted and customer creates a new one, the Program still points to old one. 
  • You need to manually go the Onboarding or offboarding program and delete old custom task and link to a new one. 

Environment

SAP SuccessFactors Onboarding 1H 2026

Reproducing the Issue

  1. Create a custom task and assign it to Onboarding Program. 
  2. Delete the custom task created in step 1 and create a new custom task
  3. The Onboarding program still references to the old custom Task (created in step 1)

Cause

This is a Known Issue.

Resolution

The fix for this issue is planned to be deployed on 2H 2026 release.

For release timelines, please review SAP SuccessFactors Product Release & Road Map Information.

Workaround

Go to the Onboarding program and manually delete the old custom task.

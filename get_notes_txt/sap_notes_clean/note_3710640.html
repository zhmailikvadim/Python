Symptom

After UD, the follow-up action fails to process the stocks in EWM, or it is not possible to create IOT5 inspection for a stock item due to error:
"Mixed storage for storage type &1 (bin &2) failed due to insp. ID check" (/SCWM/HU_WM166)

"Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental."

Environment

SAP Extended Warehouse Management

SAP S/4HANA

Cause

In the storage type where the stocks are located, the customizing checkbox "Mixed Insp.Docs" (in non-S/4 EWM) / "No Mx Insp Docs" (in S/4 EWM) is flagged, and at the time of the decision processing or IOT5 creation, there are more than one stock items with the same product/batch in the storage bin or top HU.

With this customizing flag active, it is not allowed to have stocks with Inspection Document /Inspection lot and without  Inspection Document /Inspection lot reference in the same Available quantity location. If the "Level of Available Quantity" (AVQLVL) is on bin level, it means that the whole bin can either have no inspection reference or all stocks must have the same inspection reference. It will not be possible to perform a posting change from or to Inspection Document / inspection lot in that bin.
If "Level of Available Quantity" (AVQLVL) is on HU, that same check only applies for the top HU (including sub HUs).

See the storage type customizing at SAP Customizing Implementation Guide -> SCM Extended Warehouse Management -> Extended Warehouse Management -> Master Data -> Define Storage Type:

Resolution

In order to facilitate the stock postings, see the below options:

  • move the stocks to another storage bin where there is no other stock item, to ensure that one stock item gets processed
  • move stocks (if there are more items to process) to another storage type, where customizing allows multiple inspection references in one bin
  • adjust the storage type customizing to allow multiple inspection references in the same storage bin:
    • in S/4 EWM: SAP Customizing Implementation Guide -> SCM Extended Warehouse Management -> Extended Warehouse Management -> Master Data -> Define Storage Types


    • in non-S/4 EWM: SAP Customizing Implementation Guide -> SCM Extended Warehouse Management -> Extended Warehouse Management -> Master Data -> Define Storage Types

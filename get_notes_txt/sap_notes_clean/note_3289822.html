Symptom

In the DSN, you notice the following unexpected behavior: an S21.G00.40.040 field is wrongly filled because of a retroactive contract deletion in retro, for a complete payroll period (one month).

Other Terms

DSN payroll calculation, RPLDSNF0, FCTT, bloc 40.

Reason and Prerequisites

  • This error is caused by the generation of technical entry 99xx in the CTT table where some data is replaced by the current data, and the data from the original deleted CTT entry isn't kept for this field S21.G00.40.040 ("Code risque accident du travail") 
  • Entry 99xx is triggered by a retroactive change of contracts.
  • 99xx is a technical entry created to trace the deletion of a CTT line, following changes in the validity of the contract period.

Solution

Since this error occurs during the generation of the CTT table (payroll execution), to fix the problem in the DSN declaration:

  1. Implement this SAP Note.  
  2. Run the payroll again.

Implement the attached correction instructions or the corresponding HR-SP or CLC.

The correction enclosed within this SAP Note will be included in HR Support Packages referenced below. If you need the correction before the HRSP is available, you may apply the attached correction instructions.
It should be emphasized however, that correction instructions are only a short-term solution, not a substitute for the Support Package.

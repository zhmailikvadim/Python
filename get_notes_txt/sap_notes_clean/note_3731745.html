Symptom

When submitting the Poland eDocuments to KSEF, following error occurs:

"ApacheError<ErrorDetails> <iFlow_ID>PL_KSeF_E-Invoicing_-_Open_Interactive_Session</iFlow_ID> <CamelFailureRouteId>Process_1</Camel org.apache.camel.component.ahc.AhcOperationFailedException: HTTP operation failed invoking .../v2/sessions/online with statusCode: 403"

Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.

Environment

  • SAP ERP
  • SAP S/4HANA
  • SAP Document and Reporting Compliance, Cloud Edition
  • SAP S/4HANA cloud, Private edition

Reproducing the Issue

  1. Launch Tx code eDocument Cockpit.
  2. Select the Poland KSeF eInvoice node and submit the invoice.

Cause

Missing permissions/authorization for the certificate used.

Resolution

  1. Verify with KSeF that the certificate used for KSeF 2.0 is correctly authorized for the taxpayer and the targeted environment (demo or production), and that it has the required scope/permissions to open an online session.
  • Check the authorization objects for the technical user set in the SAP BTP destination, as follows:
    • If you have SAP S/4HANA: Check user authorization described in the Setting Up Connectivity | SAP Help Portal .
    • If you have SAP ERP: Check user authorization described in the Setting Up Connectivity | SAP Help Portal .

      2. Ensure the certificate presented in Manage communication settings matches the one authorized in KSeF

      3. After correcting permissions or configuration, retest in eDocument Cockpit: Submit the document, then choose More -> Get Service Status.

      4. Refer to  KSeF 2.0: Certificate & Authorization for Establishing Connection with the Cloud Edition  for detailed requirements.

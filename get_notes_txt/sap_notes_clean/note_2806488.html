Symptom

Updating any field in marketing documents causes the error "Source file does not exist  , '\'  [Message 3721-1]"

Reproducing the issue

Prerequisites:

You have completed the following steps:

  1. Go to Administration → System Initialization → General Settings → Path tab, and enter a valid Attachment path.
  2. Go to Administration → System Initialization → Print Preferences → General tab, and select the following checkboxes:
    • When Adding Marketing Document
    • Use Attachments Folder as Default Path to Export PDF
    • Attach Exported PDF to Marketing Document.
  3. Go to Administration → System Initialization → Print Preferences → Per Document tab and select Sales Order; then select the Export to PDF checkbox.
  4. Go to Administration → Setup → General → Report and Layout Manager.
    1. On the List tab, go to Sales - A/R → Sales Order → Sales Order (Items).
    2. Go to the Printing Seqs tab and create a new sequence with at least 2 layouts (either PLD or Crystal Reports).
    3. Set this sequence as default for the Sales Order document.

Scenario:

  1. Open a Sales Order, specify customer and item details, and do not attach any file on the Attachments tab. Click the Add button.
  2. In the message box that opens with the message "You cannot change this document after you have added it. Continue?", click the YES button.
  3. Browse to the created order and check the Attachments folder.
  4. Try to update any field in the order.

Expected behavior:

Step 3 - The PDF is attached on the Attachments tab.

Step 4 - The Sales Order is updated successfully.

Actual behavior:

Step 3 - The PDF is not attached.

Step 4 - The Sales Order is not updated and an error appears:  "Source file does not exist , '\'  [Message 3721-1]".

NOTE:

The issue happens with different document types.

If only one layout is used for the document, there is no issue.

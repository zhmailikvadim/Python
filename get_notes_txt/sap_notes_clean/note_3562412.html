Symptom

There is a need for more information regarding of table partitioning of the table CE9XXXX_H which is used for combined profitability analysis (cPA).

Environment

  • Controlling (CO)
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA Finance
  • SAP S/4HANA

Reproducing the Issue

  1. The system is using combined profitability analysis (cPA).
  2. There is a lot of entries in this table, for example 1.4 billion entries which can be viewed in transaction SE16.

Cause

There is a need to partition the table as it has grown large.

Resolution

The read accesses in CO-PA always occurs via CE9xxxx_H outer join CE9XXXX_IV union CE9XXXX_H outer join CE9XXXX_IC with selective selection on CE9XXXX_H, i.e. content-based partitioning on CE9XXXX_H should be the focus. Depending on the data structure, GJAHR (fiscal year) is usually a good choice here, as most of the time data is read from one GJAHR. The other tables CE9XXXX_IV, CE9XXXX_IQ and CE9XXXX_IC are then always selected via join using the primary key, i.e. here a good choice is to partition using a hash on the "technical" primary key. Other fields do not really make sense in these here either, as generally read in multiple currencies or multiple cost elements from CE9XXX_IV, CE9XXXX_IC and CE9XXXX_IQ. 

See Also

Ensure that the table CE9xxxx_H does not exceed 2 Billion line items. If this is the case, then it is best to partition using GJAHR or PERIO, sometime BUKRS is helpful (depends on the data). Ideally, the partitions are 1.5 Billion items in size

CE9 archiving is also an option (see note 3110511 ).

Symptom

  • An error occurs in deadline step 0610 (waiting for MeLo change) in process DE_B_WIM_MPM_REQ_MOS when the system tries to determine the deadline.
  • Error message: "CL_ABAP_TSTMP, TSTMP 0.0000000 can't be converted."
  • In step class /US4G/CL_PRST_QUOT_TECHCHG_DDL, method GET_REFERENCE_TIMESTAMP reads structure ls_step_start_data-/us4g/ubd_data-diverse[ 1 ]-data, but it is not filled, resulting in the error.

Environment

  • Product: SAP S/4HANA Utilities for German billing enhancements
  • SAP Market Communication for Utilities
  • SAP S/4HANA 2023

Reproducing the Issue

  1. Open the process document for process DE_B_WIM_MPM_REQ_MOS.
  2. Navigate to step 0610 (deadline for MeLo change).
  3. Observe that the step status is "error" and the log contains "CL_ABAP_TSTMP, TSTMP 0.0000000 can't be converted."

Cause

The UBD object expected by /US4G/CL_PRST_QUOT_TECHCHG_DDL=>GET_REFERENCE_TIMESTAMP (ls_step_start_data-/us4g/ubd_data-diverse[ 1 ]-data) is missing or not filled, causing deadline timestamp determination to use an initial value that cannot be converted.

Resolution

Review the UBD data to ensure the existence of the object at the location /us4g/ubd_data-diverse[ 1 ]-data.

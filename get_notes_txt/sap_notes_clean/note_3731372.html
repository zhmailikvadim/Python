Symptom

  • Access to the SAP Business Technology Platform (SAP BTP Cockpit URL: https://cockpit.btp.cloud.sap) using SAP Cloud Identity Services (IAS tenant).
  • Although the required permissions have been assigned, several cockpit sections - such as Service Instances, Cloud Foundry, and others - are not accessible.
  • Other users who authenticated via the SAP ID service (and have the same permissions) are able to access without issues.

Environment

  • SAP Business Technology Platform (SAP BTP)
  • Cloud Foundry (CF)
  • SAP Cloud Identity Services

Reproducing the Issue

  1. Log on to the SAP BTP cockpit as a user from the custom identity provider (for example: https://emea.cockpit.btp.cloud.sap/cockpit/?idp=hemhm.accounts.ondemand.com). 
  2. Navigate to the Global Account and open the subaccount.
  3. Affected users see no spaces and are not listed as org members, while other users with the same roles (who reside in the SAP ID Service) can see them.

Cause

Usually such issues occur when the Corporate IdP does not send the needed attributes in the SAML assertions.

Resolution

Check the below points:

  1. Correctly map the user attributes provided by the corporate identity provider to the attributes required by SAP BTP.
    Ensure that the SAML assertion includes the “mail” attribute (not just “email”), as this is required for user name mapping.
    For more details, refer to: SAP KBA 2964274: Establish Trust and Federation for Cloud Foundry Platform Users for Custom Identity Providers
    and Map User Attributes from a Corporate Identity Provider for Platform Users | SAP Help Portal.
  2. OIDC Token Attributes in IAS
    In IAS, confirm that the OIDC token received by the Cockpit contains the expected user attributes, such as “mail” and (if applicable) “email”.
    IAS troubleshooting tools can be used to log and inspect tokens. Refer to: Logging OpenID Connect (OIDC) Tokens | SAP Help Portal.

  3. Conditional Authentication Configuration (if enabled)
    If SAP Cloud Identity Services Conditional Authentication is configured, ensure that the required attributes are included in the JWT token.
    This situation may occur when:

    • The Default Identity Provider is set to the Corporate IdP, and
    • Identity Federation is also configured with the Corporate IdP

    and the the configured attributes may not be automatically included in the JWT token.
    If both configurations are required, Enriched Assertion Attributes must be configured for the Corporate IdP so that the necessary user attributes are included in the JWT token.
    For more details, refer to: Enrich Assertion Attributes Coming from Corporate IdP | SAP Help Portal.

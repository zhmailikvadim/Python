Symptom

During implement a TCI note by SNOTE, below screen is displayed:

   Phase RUN_SPDD_?: Information on Objects Modified in System

If this screen is exited without any action, message "The transport based corr. Instructions require a modification adjustment" (Message No. TN838) is displayed.

The implementation terminated with message: Failed to execute current sub-queue. SAP Noe cannot be processed further.

The TCI note remains status "Can be implemented" which means it is not implemented yet. The transport used for this note and TCI package cannot be released neither.

*Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.

Environment

  • SAP S/4HANA
  • SAP ERP
  • SAP NetWeaver

Reproducing the Issue

  1. Implement SAP Note by SNOTE which is a TCI type note.
  2. The implementation stops in Phase RUN_SPDD_? step

Cause

This step "Phase RUN_SPDD_?" cannot be skipped. SPDD adjustments must be performed before continuing the TCI package import, otherwise there is risk of data loss.

Resolution

If the screen in Symptom is already exit, it is possible to re-enter this screen in SPAM with below steps:

  1. In transaction SPAM, open Goto > Status > Queue to view the import stop details.
  2. Click “Call SPDD” button to enter t-code SPDD and perform all necessary SPDD adjustments
  3. If SPDD adjustments are already completed, choose “Confirm Adjustment” button.
  4. Continue the import by selecting “Import Queue” in SPAM and wait until the package import finishes successfully.
  5. Return to t-code SNOTE and continue the implementation of the note that stopped.

For instruction of how to perform SPDD adjustments, refer to Adjusting Repository and ABAP Dictionary Objects for more information.

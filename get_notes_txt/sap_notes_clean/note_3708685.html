Symptom

  • Postal code and country fields are not loaded into the database in the correct order when using TQACX/TQGAC templates.
  • The issue occurs due to the incorrect order of fields in the older template files.

Environment

  • SAP Sales Performance Management
  • Territory & Quota on HANA
  • Territory & Quota on Legacy Oracle

Reproducing the Issue

  1. Use the TQACX/TQGAC templates to load data.
  2. Ensure the template contains the fields country and postal code.
  3. Load the template into the system.
  4. Verify the data in the database; observe that the postal code and country fields are reversed.

Cause

The issue arises due to an incorrect order of fields in the template files. The template specifies the order as country followed by postal code, whereas the system expects postal code followed by country.

Resolution

  1. New template files will be released to reflect the correct order of fields: postal code followed by country.
  2. For users continuing to use the old template files, reverse the order of the fields in the template before loading data into the system.

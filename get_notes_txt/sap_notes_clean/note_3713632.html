Symptom

Pods start to fail after a property change or deployment with the following exception:

Context hybris Global Context Factory couldn't be created correctly due to, Error creating bean with name 'defaultAdaptiveSearchCacheRegion' defined in class path resource [global-adaptivesearch-spring.xml]: Unexpected exception during bean creation; nested exception is org.springframework.beans.factory.BeanExpressionException: Expression parsing failed; nested exception is org.springframework.expression.ParseException: Expression [#{2000 * ( T(java.lang.Runtime).runtime.maxMemory() / (1024 * 1024 * 1024 * 4.0 ) )] @0: No ending suffix '}' for expression starting at character 0: #{2000 * ( T(java.lang.Runtime).runtime.maxMemory() / (1024 * 1024 * 1024 * 4.0 ) )","message":"Context hybris Global Context Factory couldn't be created correctly due to, Error creating bean with name 'defaultAdaptiveSearchCacheRegion' defined in class path resource [global-adaptivesearch-spring.xml]: Unexpected exception during bean creation; nested exception is org.springframework.beans.factory.BeanExpressionException: Expression parsing failed; nested exception is org.springframework.expression.ParseException: Expression [#{2000 * ( T(java.lang.Runtime).runtime.maxMemory() / (1024 * 1024 * 1024 * 4.0 ) )] @0: No ending suffix '}' for expression starting at character 0: #{2000 * ( T(java.lang.Runtime).runtime.maxMemory() / (1024 * 1024 * 1024 * 4.0 ) )

Environment

SAP Commerce Cloud

Reproducing the Issue

Adjust a region cache property for any region using an expression with a typo just like below one:

regioncache.adaptivesearchregion.size=#{2000 * ( T(java.lang.Runtime).runtime.maxMemory() / (1024 * 1024 * 1024 * 4.0 ) )

Apply this property in the service desired to reproduce the issue in cloud portal, then click on "Apply Configuration". Once property is deployed in config map, the exception described in the symptom will prevent pod to come up.

Cause

Cache region implementation uses Spring configuration for region cache that uses Ehcache, just like the example below:

example-spring.xml

<bean name="productCacheRegion" class="de.hybris.platform.regioncache.region.impl.DefaultCacheRegion">                  
  <constructor-arg name="name" value="productCacheRegion" />
  <constructor-arg name="maxEntries" value="50000" />
  <property name="handledTypes">
    <array>
      <value>1</value>
    </array>
  </property>
</bean>
                
<bean name="manualCacheRegion" class="de.hybris.platform.regioncache.region.impl.DefaultCacheRegion">
  <constructor-arg name="name" value="manualCacheRegion" />
  <constructor-arg name="maxEntries" value="50000" />
  <property name="handledTypes">
    <array>
    </array>
  </property>
</bean>

A cache region has the following properties:

  • name: unique region name

  • maxEntries: cache region size

  • evictionPolicy: LRU, LFU, and FIFO

All of these three can be defined using properties. As an example, for size one can use expressions, if the expression has a typo, then spring will fail to parse the field.

Resolution

Once the exception is identified, is important to understand which region cache is causing the issue by looking at the bean name described in the log:

Error creating bean with name 'defaultAdaptiveSearchCacheRegion' defined in class path resource [global-adaptivesearch-spring.xml]

In the case above, problematic region cache is adaptiveSearch. Once one has identified it, look for the properties used to manipulate this region, in this case something similar can be seen in Cloud Portal:

regioncache.adaptivesearchregion.size=#{2000 * ( T(java.lang.Runtime).runtime.maxMemory() / (1024 * 1024 * 1024 * 4.0 ) )

As observed, the expression is not properly closed, and the correct syntax should be:

regioncache.adaptivesearchregion.size=#{2000 * ( T(java.lang.Runtime).runtime.maxMemory() / (1024 * 1024 * 1024 * 4.0 ) ) }

Correct it on Cloud Portal, apply configuration and then wait for pod to restart.

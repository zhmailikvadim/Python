Symptom

  • After upgrading SAP Data Services to a version compatible with Red Hat 9, the Data Services services fail to start.
  • When executing the svrcfg command, the following error occurs:
    svrcfg: error while loading shared libraries: libnsl.so.2: cannot open shared object file: No such file or directory

Environment

  • SAP Data Services 4.x
  • Operating System: Red Hat Enterprise Linux 9 (RHEL 9)

Reproducing the Issue

  1. Upgrade SAP Data Services to a version compatible with RHEL9.
  2. Execute:
    ./svrcfg
  3. The error occurs.

Cause

  • The issue is caused by a missing legacy library dependency.
  • libnsl.so.2 was deprecated and removed in RHEL 9.
  • SAP Data Services binaries still depend on this library.
  • As a result, executables such as svrcfg cannot start.

Resolution

  1. Install Required Compatibility Package:
    Contact the system administrator and install the libnsl.so.2 library.
  2. Create Required Symbolic Link:
    Navigate to the library directory and create the following link:
    cd /usr/lib64
    ln -s libnsl.so.3 libnsl.so.2
  3. Source Data Services Environment:
    Navigate to:
    <DS_INSTALL_DIR>/DataServices/bin
    Then execute:
    source ./al_env_sh
  4. Verify the Fix:
    Run:
    ./svrcfg
  5. Result:
    • svrcfg executes without errors.
    • SAP Data Services starts successfully.

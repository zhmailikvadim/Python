Environment

  • SAP Adaptive Server Enterprise (ASE) 16.0 SP04 PL06 HF1 with Always-on Option (HADR) and below Versions.

Reproducing the Issue

# add some blank in resource file.

Eg:

setup_site=PRIM     $
PRIM.ase_host_name=ssl-site0        $
ase_sa_user=sa            $

Run removehadr.sh -R <resource_file>

This was addressed under SAP CR 829300 "The removehadr.sh script will fail to connect to servers if the blank space is added before or after property in the response file."

Cause

If the blank space is added before or after property in the response file the removehadr.sh script will fail to connect to servers.

Resolution

Upgrade ASE/RS to a release where SAP CR 829300 is fixed:

  • 16.0 SP04 PL07 or higher
  • 16.1 SP00 PL00 or higher

Remove the blank spaces from resource file.

Eg:

setup_site=PRIM $
PRIM.ase_host_name=ssl-site0$
ase_sa_user=sa$

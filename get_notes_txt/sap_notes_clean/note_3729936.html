Symptom

You are executing a BOP run for your STOs. You notice, that the Delivery Date for the requested schedule line is changed to the date of the confirmed line. This is not expected. When executing an ATP check, you do not see this behavior. Only the commited date is changed for the requested line, but the delivery date stays the same.

This is only true for new schedule line handling for STOs.

Other Terms

Stock Tranport Orders, Backorderprocessing, gATP

Reason and Prerequisites

The reason is a program error.

Solution

Implement the attached correction instruction or install the corresponding support package.

Symptom

  • An error occurs when attempting to activate more than one implementation in BADI FDM_PARTNER_MAPPING  
  • Error message ENHANCEMENT194 is issued
  • Additionally Runtime error RAISE_EXCEPTION could be issued.

"Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental."

Environment

  • Collections Management 
  • SAP ERP

  • SAP enhancement package for SAP ERP

  • SAP S/4HANA All versions

Reproducing the Issue

  1. Attempt to activate the BADI several implementations of BADI FDM_PARTNER_MAPPING
  2. Error message is issued "Could not activate the Business Add-In implementation FDM_XXXX_YYYY. Message no. ENHANCEMENT194".

Cause

The BADI is defined as a Non-multiple use BADI, which means only one implementation can be activated at a time.

Resolution

  1. Navigate to SPRO and follow the path: Financial Supply Chain Management -> Collections Management -> Integration with Accounts Receivable Accounting -> Master Data Distribution -> Activate/Deactivate Standard Implementation for Several FI Systems -> Mapping in FI-AR.
  2. Alternatively, access SE18 and check the BADI Name: FDM_PARTNER_MAPPING.
  3. Ensure that only one implementation is active in the system.
  4. Deactivate any additional implementations if more than one is active.
  5. Activate the required implementation.

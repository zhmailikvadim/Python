Symptom

  • You have price that need to be updated in settlement document.
  • The settlement document that are marked as posting status = P (Incomplete)
  • You execute transaction codes WAB_PRICING and WLFMP, however both the transaction are not determining prices. During the execution of WAB_PRICING, within the processing control box under the posting status field, one of the possible values is the posting status "P". 
  • While in WZR1 - Pricing update with C it gets determined correctly.

Environment

  • Condition Contract Maintenance
  • Settlement Management
  • Agency Business
  • Condition Contract Management
  • Condition Contract Settlement
  • SAP S/4 HANA

Reproducing the Issue

  1. WAB_PRICING
  2. Enter the settlement document and execute.
  3. It gives a success message however the repricing is not performed.

Cause

Mass pricing is not allowed when the document status is 'P' (incomplete). Status 'P' indicates that no distribution has been completed.

Resolution

You can use the transaction code WZR2 to update the price, but this doesn't apply to mass pricing.

For a more "mass oriented" approach using WAB_PRICING, you might consider implementing the Badi CL_WLF_CALL_ADDITIONAL_DATA=>CHECK_NEW_PRICING_ALLOWED_DOC to add custom logic allowing repricing for the posting status "P".

Keywords

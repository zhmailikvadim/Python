Symptom

Try to create an IDoc of any type and send it to the target system.

Environment

  • SAP Basis ALE
  • SAP NetWeaver
  • SAP Web Application Server for SAP S/4 HANA
  • ABAP PLATFORM - Application Server ABAP

Reproducing the Issue

Create an IDoc of any type and send it to the target system.

Cause

When the IDoc is created, two messages are displayed, like the samples below:

10 master IDocs set up for message type CLFMAS
and
0 communication IDoc(s) generated for message type CLFMAS

The message type CLFMAS is just a sample, it can be also any other message type.

Resolution

Maintain the missing entry for the recipient system with the message type from the error message in the Distribution Model (transaction BD64).
For further informations also read KBAs:

  • 2464346 - Information on BD64 - How is filtering working in the standard in the Distribution Model,
  • 2553610 - Status 29: Could not determine recipients for message type.

The message type in this case is simply a sample for a missing entry. The same error can be raised for any other message type. The resolution is the same, independent of the message type.

Keywords

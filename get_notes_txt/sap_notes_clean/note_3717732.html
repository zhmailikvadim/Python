Symptom

When I view my SAP Ariba dashboard, the HCP Portlet containing my SAP Business Technology Platform application is blank or fails to load. When I check the browser’s developer tools, I see that the authentication endpoint is returning an HTTP response header of X-Frame-Options: DENY.

Environment

  • SAP Ariba Procurement
  • SAP Business Technology Platform
  • SAP Identity Authentication Service

Cause

The application embedded in the dashboard portlet is protected by SAP Identity Authentication Service (IAS). The IAS authentication endpoint sends an X-Frame-Options: DENY header, which is a security feature that instructs the browser to prevent the page from being displayed inside an iframe. Since the SAP Ariba dashboard uses an iframe to display the portlet, the browser correctly blocks the sign-in page from rendering. This security control helps prevent clickjacking attacks and cannot be disabled.

Resolution

To resolve this, you must authenticate your session outside of the SAP Ariba dashboard frame.

  1. In a new browser tab, sign in directly to the application.
  2. Once your session is established, return to the SAP Ariba dashboard.
  3. Refresh the page. The HCP Portlet will now load correctly.

Keywords

HCP, portlet, cloud, dashboard, iframe

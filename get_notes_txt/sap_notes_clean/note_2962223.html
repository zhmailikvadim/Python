Symptom

  • There is a high memory usage value on the allocators:

"Pool/RowEngine/QueryExecution/SearchAlloc"
 or
"Pool/RowEngine/QueryExecution".

  • The high value can be reported in a Memory Overview script output (HANA_Memory_Overview_* script from SQLStatements in SAP note 1969700) or can appear in an indexserver trace.

    e.g.
    HANA heap memory (used);      100;      100 (sfgdfd);       60 (Pool/RowEngine/QueryExecution/SearchAlloc)

Environment

  • SAP HANA platform edition 1.0
  • SAP HANA platform edition 2.0

Cause

The following allocators are not optimized:

  • Pool/RowEngine/QueryExecution/SearchAlloc

  • Pool/RowEngine/QueryExecution

Resolution

Guidelines on how to manage high memory usage for these allocators are outlined in SAP Note 1999997 FAQ: SAP HANA Memory - under search terms "Pool/RowEngine/QueryExecution/SearchAlloc" or "Pool/RowEngine/QueryExecution".

Having high memory usage for these allocators does not normally require any changes or optimizations unless there is a critical situation.

The FAQ explains how to optimize statements related to these allocators and also how to perform further analysis on these statements.

There are some known issues as well with these allocators growing to an unmanageable size, these are outlined in the Memory FAQ as well.

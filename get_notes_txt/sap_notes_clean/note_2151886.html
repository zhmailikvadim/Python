Symptom

Null value in SFC_STEP.QTY_COMPLETE_PENDING. When trying to assign such SFCs to a new routing/operation via Change Production or RMA order the following error appears: nested exception is: java.lang.NullPointerException: while trying to invoke the method java.math.BigDecimal.compareTo(java.math.BigDecimal) of a null object loaded from local variable 'qtyCompletePending'.

Other Terms

Qty Complete Pending, Archive, Unarchive

Reason and Prerequisites

If SFC was archived in ME version prior to 5.0, it may have QTY_COMPLETE_PENDING=NULL in table AR_SFC_STEP in archive schema and unarchiving of such SFC will lead to null value in WIP schema as well.

Solution

The following SQL Script should be executed:

Oracle WIP:

UPDATE SFC_STEP SET QTY_COMPLETE_PENDING='0' where QTY_COMPLETE_PENDING is null;
COMMIT;

Oracle ODS:

UPDATE AR_SFC_STEP SET QTY_COMPLETE_PENDING='0' where QTY_COMPLETE_PENDING is null;
COMMIT;

SQL Server WIP:

UPDATE SFC_STEP SET QTY_COMPLETE_PENDING='0' where QTY_COMPLETE_PENDING is null
go

SQL Server ODS:

UPDATE AR_SFC_STEP SET QTY_COMPLETE_PENDING='0' where QTY_COMPLETE_PENDING is null
go

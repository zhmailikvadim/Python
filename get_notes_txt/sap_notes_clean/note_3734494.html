Symptom

  • How to implement value mapping in web service
  • In this example, we implemented value mapping in the receiver system for the BP Role.

Environment

  • Sales and Distribution (SD)
  • Logistics Execution (LE)
  • SAP ERP Central Component
  • SAP ERP
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA

Resolution

For example, in source system the BP has role FLCU01, and we want to create BP role XXX in target system.

  • Transaction VMIMG -> Maintain Value Mapping
    • For object type 'Data Element' and Global Data Type 'MDG_FND_PARTY_ROLE_CODE', click on button 'Display Mapping Relationships' -> Edit Mode -> New Entries
    • Specify a mapping ID, List Agency ID as source business system, List ID as 'MDG_FND_PARTY_ROLE_CODE'. Select the line and double click on 'Define Value Mapping' in left pane.
      • Specify Internal code value as 'XXX' (BP role for Customer in receiver system), External Code Value as BP Customer Role (FLCU01) from source system.
      • Important: The value mapping must be defined even if corresponding BP role in receiver and sender system is the same. For example Contact Person role BUP001.
  • Transaction VMIMG -> Assign Code Lists to Elements and Systems
    • Create entry for object type 'Data Element' and Global Data Type 'MDG_FND_PARTY_ROLE_CODE'. Specify Business System, List ID, List Agency ID.

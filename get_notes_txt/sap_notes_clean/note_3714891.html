Symptom

When pulling data in T-CODE PB2A for STP submission, there is below error:

  • Error code: SBR.GEN.AUTH.004
  • Message text: Check the ABN in the credential matches the business/agent details (eg ABN or agent number) in the transmission.

Environment

  • Human Capital Management (HCM)
  • SAP ERP
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP SuccessFactors Employee Central Payroll
  • SAP S/4HANA

Reproducing the Issue

  1. Attempt to pull data from ATO using the T-CODE PB2A.
  2. Observe the error message.

Cause

The ABN for the M2M credential does not match the ABN in STP submission.

Resolution

Check whether the issue happens in EVTE (non-PROD) or PROD system.

  • When the issue occurs in the EVTE (non-PROD) system, verify that the EVTE ABN is 14088411787 and ensure the EVTE M2M certificate attached in SAP Note 3471361 is being used.
  • When the issue occurs in the PROD system, check whether the access and permissions between intermediary ABN and subsidiary ABN is maintained correctly in ATO access manager. Check Access Manager - Business Appointments.pdf which is available in Australian Payroll workzone for detail.

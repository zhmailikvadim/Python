Symptom

The System DB Status shows as Failed after you try to deploy the System Database onto a DB instance directly from the System Landscape Directory Control Center.

Image/data in this SAP Note is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.

Reproducing the issue

Prerequisites:

  • You are using SAP Business One 10.0 version lower than FP 2608.
  • You have installed SLD, License Manager, and SLD Agent.
  • You intend to use Microsoft SQL Server 2022 as your database instance to store the SBO-COMMON database.

Scenario:

  1. Log in to SLD Control Center. 
  2. Navigate to the Global Settings tab and define the CD Repository and Central Log Directory.
  3. Navigate to the DB Instances and Companies tab and add a new DB Instance with MSSQL 2022.
  4. Press Deploy, select System Database, and press Next.
  5. press Start.

Expected behavior:

The System Database deploys successfully.

Actual behavior:

The System Database fails to deploy.

Cause

Program Error

Solution

SAP intends to provide a patch or patches to solve the problem described. The section References below will list the relevant patches once they become available, and the Related Info content of relevant patches available in the SAP Support Portal will show this SAP Note. Be aware that these references can only be set at patch release date. SAP delivers patches only for selected releases at its own discretion, based on the business impact and the complexity of the implementation.

Workaround:

Use the Setup Wizard to install SBO-COMMON (Repository).

Other terms

SBO-COMMON; Centralized Deployment

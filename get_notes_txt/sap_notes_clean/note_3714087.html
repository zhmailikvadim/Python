Symptom

This SAP Note is to determine random segmentation key instead of default segmentation key('01').

Other Terms

/PE1/CL_BPE_T_SEGKEY, READ

Reason and Prerequisites

If additional servers are added to parallel generators without a segmentation key configured, the existing logic always assigns the default segmentation key “01.” This results in a major processing imbalance, as a large number of transactions are routed to the default segmentation key (01).

Solution

Implement the correction instruction attached to this SAP Note. Alternatively, you can import the referenced Support Package.

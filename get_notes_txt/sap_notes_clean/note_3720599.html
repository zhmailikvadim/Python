Symptom

During installation of an older SAP HANA 2.0 revision using hdblcm, the installer stops with a Local Secure Store compatibility requirement similar to:

Selected component SAP HANA Database is not compatible with SAP HANA Local Secure Store version '<current_version>'.
The SAP HANA Database '<revision>' requires SAP HANA Local Secure Store version <required_version>.

The required LSS version cannot be found in SAP Software Center.

The installation is being performed in preparation for a system copy, migration, or go-live, and the target system is intended to match the source system revision.

Customers assume the same SAP HANA patch revision must be installed to restore a database backup from the source system.

Environment

SAP HANA DB Version 2.0

Cause

Each SAP HANA revision is delivered together with a specific Local Secure Store (LSS) version as part of a delivery stack.
Older SAP HANA revisions may require an LSS version that is no longer included in currently distributed installation media.

The SAP HANA installer performs a compatibility check between SAP HANA and LSS components and prevents installation of outdated component combinations.

The behavior is expected and by design. 

The situation typically occurs because it is incorrectly assumed that a database backup can only be restored to a target system running the exact same SAP HANA patch revision as the source system.

Resolution

For SAP HANA recovery scenarios, the target system must run a SAP HANA version that is compatible with the backup according to SAP HANA recovery rules.

In general, restoring a backup to the same or a higher SAP HANA version is supported, provided the version compatibility requirements are fulfilled.

Therefore:

  1. Install a supported SAP HANA revision available in SAP Software Center.
  2. Installing the identical SAP HANA patch revision as the source system is not required for database recovery scenarios.
  3. Proceed with the database recovery on the target system using the backup from the source system.

The required Local Secure Store version is delivered together with the corresponding SAP HANA installation media and no separate LSS download is necessary.

Symptom

  • Error during creation of DDIC sources of Open Hub Destination ZOHTEST

          Message no. RSO435

  • Error when activating Open Hub Destination ZOHTEST

          Message no. RSO404

Environment

  • Any Business Warehouse release, pre-SAPKW74001 on a non-HANA database platform.
  • SAP Netweaver BW.

Reproducing the Issue

  1. Create the Openhub destination in transaction RSA1, filling in all the required fields listed, including the 'create Transformation and DTP' flag
    (Please note that you need to fill BOTH the 'Open Hub Destination' field and the 'Description' field when creating the Openhub, failure to do this is enough to cause the above Activation errors).



  2. Save the Destination Type as 'Database Table', selecting the 'Technical Key' flag.



  3. Save the Destination and activate the Open Hub Destination.

  4. The first message box will not show any error.

     

  5. However, the second message box (log display) will show the following errors...

Cause

Cause A: This functionality, creation of an Open Hub destination of Destination Type 'Database Table' using the 'DB Connection' field, only works in BW 7.3 on a HANA database platform.

Cause B: Any other reason that violates a database constrain.

Resolution

For cause A, there are two options:

  1. Upgrade to BW 7.4. This release allows you to create Openhub Destinations of the Destination Type 'Database Table' to all other standard database platforms.
  2. Set up a Third-Party Open Hub Destination on BW 7.x, please see the following link for details: http://help.sap.com/saphelp_nw70/helpdata/en/43/7a69d9f3897103e10000000a1553f7/frameset.htm

For cause B:

  1. Go to SE11 and search for the database table related to the OHD object;
  2. Open it and click on Check (this will reveal the error, but the final solution escapes the scope of this KBA).

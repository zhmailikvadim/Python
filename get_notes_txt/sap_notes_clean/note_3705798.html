Symptom

When updating the product via Manage Mass Process-Products app, we are getting the exception at edit step:

"An exception with the type CX_SY_OPEN_SQL_DB was raised, but was not handled locally or declared in a RAISING clause"

Environment

  • SAP Master Data Governance-Consolidation and Mass Processing

Reproducing the Issue

  1. Launch Fiori Launchpad
  2. Click "Manage Mass Processes Products" app
  3. Click  the "Create Process" button
  4. Select the Process Template drop down and click on Save
  5. Select Configuration from drop down and hit continue
  6. In Step Configuration (Selection), enter values and hit Start
  7. Error is raised

Resolution

  • Implement the SAP Notes :
    3710241 - CMP: Memory allocation error when reading active material ledger data
    3571275 - Performance of read from i_materialvaluationprices

Keywords

CX_SY_OPEN_SQL_DB, I_MATERIALVALUATIONPRICES, IMATVALPRICES, CL_MDC_DATA_MAT, APPEND_CKMLCR, MBEW

Symptom

You work with inspection lot origin 17 and you can experience that the quantity update is missing for the field QALS-LMENGE03 (Quantity Posted to Sample) in case of UD (EWM inspection) with follow-up action code sample consumption.

Other Terms

IOT4 decentral embedded EWM FUINT 9 LMENGE03

Reason and Prerequisites

Missing functionality

The handling of the field LMENGE03 in DB table QALS is missing in the EWM-QM / ERP-QM integrated scenario.

Solution

Implement the attached code corrections.

Attention!!!

If you are working in embedded EWM and you don't have generated version for IOT4 then you don't need to create a new IOT version otherwise please read it further.

The additional data (LMENGE03) for the inspection lot is activated through the generation of a new inspection object type (IOT) version. Afterwards, you must adjust the inspection object type-dependent Customizing and if you are using inspection rules then copy the existing check rules to the new inspection object type version.

All inspection document / lot has to be completed before generating a new IOT version in the production system! Otherwise the system won't be able to find the old documents which can cause issues during the GR (lot summary) process and the decision process because of the mismatching IOT versions. For more details please check the KBA 2866278 - How-to generate new IOT version in EWM.

Symptom

Export to Spreadsheet functionality for ESR objects fails in NWDS due to Apache poi jar upgradation.Open source apache poi library version 3.8  was reported to have security issues. Hence the library was updated to 3.17 and corresponding changes are performed in the code to make it compatible with the new library.

Other Terms

ESR, NWDS , Export to spreadsheet

Reason and Prerequisites

NWDS runtime does not support OOXML-based POI APIs required for .xlsx generation for version 3.17.

Solution

The issue is resolved by code changes to use a supported Apache POI workbook format.

To get this correction available in your Support package you need to apply the patch matching to your support package version as per the instructions in the NetWeaver Support Package Stack guide which can be found on the SAP Service Marketplace as described in SAP Note 952402.

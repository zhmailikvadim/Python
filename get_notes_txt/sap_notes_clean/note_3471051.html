Symptom

When generating an xml or exporting an electronic report an error occurs: Cannot call electronic file manager. Contact technical support.FD.Runtime.FlowRunTimeException: @3: Input data size too big. Size limit is 0 bytes

Environment

SAP Business One

Reproducing the Issue

Prerequisites:

You are working with electronic documents.

Scenario:

You are generating an xml or exporting an electronic report.

Expected behavior:

Message appears: Operation completed successfully

Actual behavior:

Error appears: Cannot call electronic file manager. Contact technical support.FD.Runtime.FlowRunTimeException: @3: Input data size too big. Size limit is 0 bytes

Resolution

This is correct system behavior. The user has two options:

Login into SAP Business One using Run as Administrator.

OR

  1. Go to the path 'C:\Program Files\SAP\SAP Business One'.
  2. Check the default value for  'key="MaxInputFileSize" value="100000000".
  3. If the max value is the correct one
    1. Remove the folder
    2. Shut down SAP Business One 
    3. Enter SAP Business One using Run as Administrator.

Symptom

  • You created virtual systems in TMS.

  • You activated use case task 'Transports: Read Landscape' with transaction code /n/sdf/alm_setup in your domain controller system and client 000. 

  • Diagnostic job /SDF/CALM_CDM_DIAGNOSTIC has run.

  • However, these virtual systems are not visible in SAP Cloud ALM > Project and setup > System Groups .

Environment

  • SAP Cloud ALM 1.0 

Reproducing the Issue

  1. Access the SAP Cloud ALM launchpad;
  2. Attempt to select the virtual system from 'projects and setup' tile > 'system groups' tab.

Cause

Virtual systems are designed to act as placeholders for real production systems that will be implemented later.
As such, they are not displayed under the system group for selection.

Resolution

Virtual system will not be shown under the system group for selection. 
Check First steps to work with SAP Cloud ALM Deployment scenario for SAP ABAP systems (7.40 or higher) 

  • 3.2. Handling Virtual systems

When virtual systems are included in a TMS route, in the Feature the target tenant for the virtual system:client will get a grey color as it is not possible to trigger any import-deploy on a virtual non existing system

Symptom

You use Production Operation List (POL) integration and notice these symptoms:

1. Field Value Mapping (FVM) for the component target structure
You configure FVM with a component target structure and extension fields as the source structure. The value from the extension field in the DM message does not set the component value in SAP. The expected result is that the value from the extension field in the DM message sets the component value in SAP.

2. POL FVM mapping for DESCRIPTION field
You configure POL FVM with the target structure CPS_TASK_LIST_MAINT_OPR or CPS_TASK_LIST_MAINT_SOPR and the target field DESCRIPTION. The value from FVM does not map to the description of the POL routing’s operation or suboperation in SAP.
Actual result: The value from FVM is not mapped to the description of the POL routing's operation/suboperation in SAP.
Expected result: The value from FVM is mapped to the description of the POL routing's operation/suboperation in SAP.

3. Non-writable fields in F4 help
Fields that you cannot write through POL FVM mapping appear in the target list in F4 help for the target structures CPS_TASK_LIST_MAINT_OPR and CPS_TASK_LIST_MAINT_SOPR.

Other Terms

PLMSI, PLM system integration, POL, Components, FVM

Solution

Implement this SAP Note to fix the Field Value Mapping behavior in Production Operation List integration.

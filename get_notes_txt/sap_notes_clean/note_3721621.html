Symptom

You update your system to be compliant with Brazilian Instrução Normativa nº 2229, published on October 15, 2024, which changes the CNPJ format from numeric (NUMC) to character (CHAR) in eSocial events.

However, you get syntax and runtime errors, or the generation of unnecessary rectification events along the eSocial framework, related to eSocial BAdI implementations or calls.

Other Terms

eSocial; CNPJ; Instrução Normativa 2229; BAdI; CL_HRPAYBR_INSC_NUM_NORMALIZER; NUMC; CHAR14; Event rectification

Reason and Prerequisites

Reason:

After the implementation of the relevant SAP Notes, all CNPJ fields had their types converted from NUMC to CHAR and this change affects BAdI interface method signatures.

Therefore, you need to check every BAdI implementation and adjust them as described in the Solution section.

Prerequisites:

You have implemented SAP Note 3700999 - eSocial - Alphanumeric CNPJ - Changes in standard events.

Solution

This SAP Note provides guidance on how to adapt your BAdI implementations to comply with Instrução Normativa nº 2229/2024 and avoid unnecessary rectifications due to CNPJ format changes.

Recommended Actions

1. Review all your custom BAdI implementations related to eSocial events.
2. Identify BAdIs that process CNPJ values.
3. Be familiar with the standard method CL_HRPAYBR_INSC_NUM_NORMALIZER=>STANDARDIZE( iv_insc_num ) if any BAdI changes are necessary. Check the attachment How_to_Format_INSC_NUM_to_SAP_Expected_Format.pdf for the implementation examples and the BAdIs that may be affected.
4. Test your implementations to ensure proper CNPJ format conversion.

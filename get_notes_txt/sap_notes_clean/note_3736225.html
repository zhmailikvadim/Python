Symptom

As described in SAP ONE Support Launchpad Notification of Legal Change Unified monthly employer report 2026 (JMHZ)

The functionality delivered in this SAP Note needs to be updated regularly with latest SAP Notes.

These updates are provided:

REGZEC (DM - data mining, CS - compose submission, PF - process feedback, SO - special operations, INIT - import of IDs to IT3457, XP - XML for ePortal)

  • DM, XP: New BAdI hr_cz_jhr_job_storno has been defined. Implement the BAdI if the action storno is needed (employee does not start to work) but for some business reason you are not going to delete them from the Personal master data (for example the integration with Success Factors is switched on)

  • DM, XP: Insurance period is determined for the last two years. For example, for an employee being on Maternity leave, it is not enough because there are no insured periods in the last two years. The insured period is determined from the hiring date to the termination date in such a case.

MH (DM - data mining, CS - compose submission, WB - workbench)
Corrections that may cause that corrective individual forms will be created for already submitted periods are marked with code abbreviation (DIF )

  • Fixes
    • After installing SAP Notes 3733567 and 3733568, the option to edit XML in ADMIN mode of program HCZCJHM0_WB did not work.
    • Program HCZCJHM0_DM was generating a false positive message, "No PPV found with an activity type permitted for the primary PPV," in cases where the PPV was inactive and an infotype 121 existed that referenced only itself. The message has been removed for this situation.
    • When creating the MH for 03/2026, you received message no. 192 stating that the XML contains attribute 10319, "Employee requested annual reconciliation." The attribute has now been removed from the filing for month 3.
    • The logical check MZD 028, which reported a false positive error after installing SAP Note 3733568, has been fixed.
    • For PPV employees working in hazardous conditions, error message PRZ 282 appeared if they had not worked any time (attribute 10268 equal to zero). The program was incorrectly populating attribute 10274. From now on, in this situation the program does not populate attribute 10274. (DIF)
    • When determining the place of work from the organizational assignment using method 2, the T5T1L record with TXTYP=SP was incorrectly used instead of the correct TXTYP=CZMV. Corrected to CZMV. (DIF)

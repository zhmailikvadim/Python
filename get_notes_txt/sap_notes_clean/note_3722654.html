Symptom

In the existing technical implementations of Tank Planning and the maintenance of so called Product Storage Definitions no checkpoint groups have been defined. Therefore it is not possible to extend the existing features by additional analysis options based on such Checkpoint Group definitions.

Other Terms

Tank Planning, PSD, /SAPAPO/STORDEF, Product Flow

Reason and Prerequisites

This SAP Note does not provide any new functions for the tank planning functions in PP/DS in SAP S/4HANA. Or for maintaining the so-called product storage definitions. It serves only as a container for new technical objects to be made available in lower release versions for future program adjustments. These new objects provide the basis for further technical enhancements with the aim of an optimized error analysis.
This SAP Note does not have to be implemented manually using transaction SNOTE. The changes implemented by this SAP Note will probably never be identified as technical prerequisites for implementing other dependent SAP Notes. Therefore, the implementation should usually be done via support package upgrade. Only in exceptional cases where code elements are introduced with these new checkpoint groups can a manual implementation be required.

Solution

Do not implement the changes of this SAP Note without real technical requirement. The implementation should usually be done by upgrading the corresponding support package according to your system installation. In case you have to create the checkpoint groups manually according to the manual correction instructions use package /SAPAPO/STORDEF for the assignment of the new objects.

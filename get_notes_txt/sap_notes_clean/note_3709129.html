Symptom

Indexserver crashes with a crash stack similar to the below:

[CRASH_STACK] Stacktrace of crash: (0000-00-00 00:00:00 000 Local)
----> Symbolic stack backtrace <----
0: SQLDBC::SQLDBC_PreparedStatement::prepare
1: SQLDBC::SQLDBC_Catalog::SQLColumns
2: QLColumns_Enc
3: SQLColumnsW
5: federation::OdmInternalService::do_odbc
[...]

Other Terms

ODBC, Client, Federation, SDA, Low Memory

Reason and Prerequisites

Issue number 352886

Reason:

Due to a programming error in SAP HANA, HANA ODBC driver of SDA could not handle the operations of remote queries if a small memory allocation failed.

Affected Releases:

  • SAP HANA Client:
    • Revisions <= 2.27.19
  • SAP HANA 2:
    • Revisions <= 079.08 (SPS07)
    • Revisions <= 089.00 (SPS08)

Prerequisites:

  • HANA is extremely low in memory globally or 
  • Statement memory limit is reached.
  • And SDA is configured between HANA databases.
  • And ODBC function to access the remote HANA DB is called.(ex. CREATE VIRTUAL TABLE)

Solution

Apply one of the following SAP HANA Revisions:

  • SAP HANA Client: Client can be downloaded and installed separately.
    • Revisions >= 2.27.23

or

  • SAP HANA 2: Patched client revision is included.
    • Revisions of SPS07: SAP is currently working on a permanent solution to this issue. This SAP Note will be updated as soon as a revision containing the solution is available
    • Revisions of SPS08: SAP is currently working on a permanent solution to this issue. This SAP Note will be updated as soon as a revision containing the solution is available
    • or higher

Symptom

  • Error message displayed: “Error setting up RFC connection to DEVCLNT704” Message No. /SCWM/ERPINTEGRATION104.

Environment

  • EWM Interface
  • RFC

Reproducing the Issue

  1. Access transaction SPRO.
  2. Navigate to the IMG path: SCM Extended Warehouse Management -> Extended Warehouse Management -> Interfaces -> ERP Integration -> Goods Movements -> Define Customer-Specific Movement Types -> Communication EWM -> ERP: Customer Movement Types.
  3. Create a custom movement type mapping it to a standard one.
  4. Attempt to save the entry.
  5. Observe the error message: “Error setting up RFC connection to DEVCLNT704” Message No. /SCWM/ERPINTEGRATION104.

Cause

  • Multiple own business systems have been configured for EWM system, which is not supported.
  • Remote function module /SPE/GET_T156 in ERP is blacklisted

Resolution

  1. Verify the configuration in the SPRO path: SCM Extended Warehouse Management -> Extended Warehouse Management -> Interfaces -> ERP Integration -> General Settings -> Define Own Business System.
  2. Ensure that only one valid business system is defined for a single system installation.
  3. Maintain the business system mapping to logical systems in the SPRO IMG path: Define Business System.
  4. In the decentral scenario, ensure the ERP Logical system is mapped to an ERP Business System.
  5. If the issue persists, check the parameter `ucon/rfc/check_blacklist` in transaction RZ11. If the blocklist check is not required, set the parameter to 0 to deactivate the check.
  6. Alternatively, override the blocklist by following the steps in KBA 0003530014 - Override the blocklist with report RS_RFC_BLACKLIST_CUSTOM.
  7. Refer to SAP Note 0002295840 - Outbound / Inbound calls from external to RFC FM are blocked when the FM is blocklisted and the UCON-Check is active for additional information.

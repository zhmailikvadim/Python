Symptom

Change pointers were set to processed even though there was an error during document preparation for extraction.

In case of BADI implementation, it was not possible to clear the standard document created from the function module /ASADEV/ACI_FG_PO_ATTACHMENTS.

Other Terms

 /ASADEV/ACI_FG_PO_ATTACHMENTS

Reason and Prerequisites

Change pointers should remain unprocessed in case of error.

Solution

New parameter is now introduced in badi spot CONTENT_CHANGE, lv_clr_output, where if this is set, it clears the output that function module /ASADEV/ACI_FG_PO_ATTACHMENTS determined and is not added in payload.

Symptom

When performing asset retirement using transaction F-92, the system issues the following error message:

AAPO190
“Account assignment object 'Functional Area for Investment' differs from that in the asset master record.”

The same process may work in ECC but fails in SAP S/4HANA.

Environment

  • FIN Asset Accounting
  • SAP S/4HANA
  • SAP S/4HANA Finance
  • SAP S/4HANA Cloud Private Edition

Reproducing the Issue

 

  1. Execute transaction F-92.

  2. Enter required header data (Company Code, Posting Date, etc.).

  3. Enter customer and clearing line items.

  4. Select the Asset Retirement indicator.

  5. Enter the asset number and proceed with posting.

  6. System issues error AAPO190.

Cause

In SAP S/4HANA, stricter consistency checks are applied for Account Assignment Objects (such as Functional Area – FKBER).

During posting, the system derives a Functional Area (for example from the Cost Center or Profit Center).

If the Functional Area field is suppressed in the asset master screen layout, no value is stored in the asset master record.

During posting, S/4HANA compares:

  • Derived Functional Area (from account assignment)

  • Functional Area stored in the asset master

If the asset master value is blank and the derived value is populated, the system detects a mismatch and triggers error AAPO190.

Although customizing may be identical in ECC and S/4HANA, the runtime validation logic in S/4HANA is stricter due to Universal Journal integration.

Resolution

 

Depending on business requirements, one of the following solutions can be applied:

Recommended Solution (Best Practice)

  1. Adjust the screen layout for the asset master:
    Asset Accounting → Master Data → Screen Layout → Define Screen Layout for Asset Master Data

  2. Make field group 89 – Functional Area visible (Optional).

  3. Maintain the Functional Area in the asset master (transaction AS02).

  4. Repeat the posting.

This ensures consistency between master data and posting and is the recommended audit-compliant solution.

Alternative Solutions

  • Remove the “Agreement Indicator” (Acct Assignment Object Same in Asset Master and Posting) for Functional Area.

  • Change the message control from Error to Warning.

Note: These options reduce system consistency checks and may allow differences between posting and master data.

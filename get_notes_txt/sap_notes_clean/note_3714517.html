Symptom

In transaction /SPMEAT/PE12, when selecting a disassembly order and material, the system displays change IDs that are not permitted according to the material master data.
Despite being disallowed, these change IDs appear in the selection list and can be chosen.
Selecting such a non‑allowed change ID triggers an error message indicating that no corresponding production entry exists.

Other Terms

IS-MFM, MEAT, Change ID, Classification, /SPMEAT/PE12, Material Master, Disassembly Process, SAP Meat and Fish Management by msg.

Reason and Prerequisites

Due to missing validation logic in the GUI change‑ID list, the system does not filter out change IDs that are not allowed based on the material master data.
As a result, disallowed IDs remain selectable even though subsequent processing will reject them.

Solution

History:
SAP Note created on 11 February 2026.

Description of Correction:
The selection logic for the Change ID list in transaction /SPMEAT/PE12 has been enhanced.
The corrected implementation ensures that:
Only allowed change IDs from the material master data are displayed.
Non‑allowed change IDs are suppressed from the user interface and cannot be selected.
This prevents the error that previously appeared when a user selected a disallowed ID.


Functional Dependencies:
None.

Manual Changes:
No manual changes are required for end users.

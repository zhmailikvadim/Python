Symptom

 ERROR: Error while resolving username ahpcrypt rc=111: Connection refusedException: exception  1: no.2050001  (MultiDB/impl/DatabaseIsolationAPI.cpp:521) TID: 21091    
 System error: $msg$, rc=13: Permission denied

Environment

  • SAP Business Technology Platform
  • XS Advanced Runtime (XSA)
  • HANA

Reproducing the Issue

  1. Patch operating system [OS]. [It is worth noting that OS upgrades can sometimes alter security-related configurations, such as user roles or permissions]
  2. Attempt to start the HANA database.
  3. Observe that the XSA-Services (hdbxscontroller ,hdbxsexecagent hdbxsuaaserver) are not starting.
  4. Check the XSA Controller Log and <SID hanadb> XSA diagnose logs for the following error:  ERROR: Error while resolving username ahpcrypt rc=111: Connection refusedException: exception  1: no.2050001  (MultiDB/impl/DatabaseIsolationAPI.cpp:521) TID: 21091 System error: $msg$, rc=13: Permission denied
  5. Check if LSS is in use - if this is unknown, the Daemon file will contain something similar to: Program parameters of <localsecurestore>
    name = Local Secure Store
    executable = localsecurestore 

Cause

  • Missing or incomplete configuration of the user <sid>crypt - see: LDAP User Authentication.
  • It is worth noting that OS upgrades can sometimes alter security-related configurations, such as user roles or permissions.

Resolution

  1. Refer to SAP Note 3244349 XSA can not start up since: Error while resolving username rc=111: Connection refusedException System error: $msg$, rc=13: Permission denied, for detailed solution suggestions.
  2. Ensure that the operating system users are correctly associated with the appropriate group and have the necessary permissions to access secure storage. Specifically, verify the permissions for the users mentioned in the error messages.

If Local Secure Store (LSS) is not required in the environment, consider uninstalling it. Refer to KBA 2933804 for instructions on how to uninstall the HANA Local Secure Store (LSS).

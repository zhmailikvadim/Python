Symptom

When monitoring subscription activity in SAP Fieldglass, a transaction displays a status of document:OK; delivery:ERROR.

Error:
Delivery: Error while delivery.. An exception occurred while processing the file(s) in Fieldglass. Please contact SAP Fieldglass support.

Cause

This behavior is as designed for subscriptions utilizing API-1.0.

Unlike other versions, the API-1.0 subscription process waits for the upload connector to finish processing the data. If the file is successfully retrieved from the source (Document: OK) but contains normal data errors (e.g., invalid IDs, formatting issues, or duplicate records), the delivery status is set to ERROR to alert the administrator that the data was not fully consumed.

Resolution

You must review the Upload Audit Trail to identify and correct the specific data errors within your file.

  1. Sign into your SAP Fieldglass account.
  2. Click Admin Configuration > Integration > Subscription Activity.
  3. Enter the Name of your subscription and click Apply Filters.
  4. Click on the Transaction ID that resulted in the delivery error.
  5. Locate the Server Message or navigate to the Upload Audit Trail for the associated connector.
  6. Review the records marked as failed (e.g., "Object Id is invalid" or "Duplicate of record").
  7. Correct the data in your source file and re-upload or wait for the next scheduled subscription run.

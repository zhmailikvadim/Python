Symptom

Triggering a consistent snapshot on AWS Instances results in "Create snapshot from volume failed: Tag (Name) is already defined in resource id <volume id>".

Other Terms

SAP LaMa, Enterprise Edition, AWS, Cloud, Snapshot, Volumes, Amazon Web Services, Instances, Tag, Name

Reason and Prerequisites

Triggering a consistent snapshot on AWS instances with volumes and the volumes contain tags with the Key "Name" is causing the snapshot process to fail.

Examples:

  • Volume has tag with key "Name".
  • You have implemented the enterprise edition of SAP Landscape Management 3.0 SP 15 PL01 or lower

Solution

You have the following options:

  • Implement SAP Landscape Management 3.0 SP15 Patch02.
    For more information about implementing support packages and patches, see https://help.sap.com/viewer/lama_instguide.
  • Workaround: Before triggering the consistent snapshot, remove the tag with key "Name" from all EBS volumes to snapshot in AWS.

    Related Information

    • For more information about the enterprise edition, see SAP Help Portal at http://help.sap.com/lama.

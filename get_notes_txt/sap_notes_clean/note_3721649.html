Symptom

The "Take Readings" option is being controlled by the Work Order Edit authorization check. This is incorrect as the readings are tied to Technical objects.

Other Terms

  • Phase Model
  • Take Readings
  • Supervisor Technician

Reason and Prerequisites

  • BAS/Visual Studio Code with SAM MDK Project
  • `patch` command (available on BAS and macOS by default) via terminal or other patching utility
  • Read the guidelines on how to apply the patch and read the appendix section if you receive any error

Solution

The note fixes this issue by removing the workorder edit authorization check while taking readings.

To apply the patch, download the attached 3721649_YYMM.patch file and save it in a known location, preferably adjacent to your SAM.mdkproject
YYMM represents product release version. 

If you are on Windows, see SAP Note 3533879 for how to install a .patch file in Windows. If you are on a Mac, follow the steps below:

  1. Open a terminal in the root of your SAM metadata (i.e. SAM.mdkproject/metadata or SAM.mdkproject/metadata/base.definitions)
  2. Run the terminal command `patch --no-backup-if-mismatch -p4 < path/to/3721649_YYMM.patch`. 
  3. Verify in the terminal window that the following files have been modified and listed as patched, as indicated by the console output:
    • modified:   Rules/Measurements/Points/MeasuringPointsTakeReadingsIsVisible.js

Symptom

When running a GRDC mapping run, the Run Status of the Mapping run is 'Error' and in the traceability log file there are unexpected error messages raised like

  • Error message "FINCS_DTX-014: Task &3 doesn't exist for document type &2 version &1 &4."
  • Error message "G00-063: Task &1 is not contained in task group &2 of dimension &3" 
  • Error message "G00-095: No processing because task &1 cannot be executed for any/all CGs/CUs"

When checking the configuration and/or the status in the data monitor for the relevant execution parameters, you cannot find the root cause for the errors.

Environment

SAP S/4HANA Cloud Private Edition, SAP Group Reporting Data Collection

Reproducing the Issue

  1. Execute a GRDC Data Mapping run in the Run Data Mapping app.
  2. In the Mapping Definition a Mapping File is used that includes a mapping where e.g. the target dimension 'Consolidation version' is specified. For example it is specified that the Output dimension 'Consolidation Version' shall have the fixed value "Y10"
  3. Observe in the 'Job Run Overview' tab in the Traceability Log file in column 'Mapped Data' under 'Generated Values' that the expected Consolidation Version "Y10" is shown.
  4. Observe in the Run Status for the relevant Mapping run in app 'Run Data Mapping' that the status is 'Error'. The version shown there is the expected one "Y10".
  5. Observe error messages such as FINCS_DTX-014, G00-063 or G00-095 in the traceability log file. When checking the details for that error messages related to the target consolidation version "Y10", the root cause for these errors cannot be found, because for example the relevant task has an editable status in the data monitor in the relevant consolidation version and in the relevant year/period.
  6. When opening the task log in the 'Task Logs' app for the relevant Data Mapping run, however it can be observed in the header section, that another consolidation version (e.g. "ACT") is shown. In the Log Items section, the consolidation version "Y10" is shown. The same information can be seen in the SRT_MONI transaction, when looking at the message (payload) details.

Cause

  • The issue occurs due to a discrepancy between the global parameters set by/for the business user who runs the Data Mapping and the global parameters being used in the Mapping definition.
  • For example the business user has set the consolidation version "ACT" in the global parameters but in the Mapping Definition the target dimension Consolidation Version "Y10" is defined.
  • The Receiver Interface FinancialConsolidationReportedFinancialDataBulkIn will evaluate the global parameters being set in the global parameters of the business user to e.g. check if the task status is editable.

Resolution

  1. Verify the global parameters sent from/during the GRDC mapping run via the 'Task Logs' app or via the transaction SRT_MONI. These global parameters are responsible for the error situation.
  2. Ensure that the global parameters for the user are set correctly (in the app 'Enter Global Parameters / CXGP' ), matching to the values that correspond to the data rows, such as version "Y10".
  3. Re-execute the GRDC mapping run after correcting the global parameters.
  4. Confirm that the mapping operation completes successfully without errors.

Keep in mind: The global parameters from the business user are checked when using Principal Propagation. In case Basic Authentication is used, the global parameters for the technical user (defined in the destinations S4HanaCConsoEndpoint*) are checked.

Symptom

  • hdbwebdispatcher service shows status RED (Stopped) and does not start.
  • While checking the Web Dispatcher traces it is possible to find the following entry

    webdispatcher_<hostname>.30006.000.trc:

    [00000]{-1}[-1/-1] 0000-00-00 00:00:00.000000 e webdispatcher    webdispatcherbackendconfig.cpp(00000) : Invalid value in "public_urls": <parameter value>

Environment

SAP HANA Database

Reproducing the Issue

  1. Start the SAP HANA Database.
  2. Observe the hdbwebdispatcher service turns RED (Stopped) shortly after startup.
  3. Check the Web Dispatcher trace and find the reported errors indicating an invalid value in "public_urls".

Cause

This situation may occur when a parameter is configured in an incorrect section of the xsengine.ini configuration file.

In this case, the parameter "root_page" was placed within the "public_urls" section, whereas its correct section is "httpserver", resulting in the Web Dispatcher service failing to start properly.

Resolution

To properly resolve this issue, it is necessary to remove any incorrectly configured parameters from the affected session.

If a misconfigured value belongs in a different section of the configuration file, its correct placement can be determined by consulting the parameter configuration reference in the following article:

SAP Help Portal Configuration Parameters SPS 08

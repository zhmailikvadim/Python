Symptom

Even though you can create profit centers with valid from date 01.01.1900 in KE51, the interval check while using the BAPIs BAPI_PROFITCENTER_CREATE and BAPI_PROFITCENTER_CHANGE is performed against the valid from date 01.01.1950.

Other Terms

LRKPMTOP, PROFITCENTER_CREATE, PROFITCENTER_CHANGE, mindatum

Reason and Prerequisites

The valid from date was changed from 01.01.1950 to 01.01.1900, but the corresponding BAPI variable for the interval check had not been adjusted yet.

Solution

Implement the attached coding correction or the respective correction instruction.

Once the note is implemented, the system will be consistent in the valid from date for master data maintenance with KE51/KE52 as well as with the BAPIs BAPI_PROFITCENTER_CREATE and BAPI_PROFITCENTER_CHANGE.

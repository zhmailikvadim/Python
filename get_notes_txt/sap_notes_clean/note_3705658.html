Symptom

When a user who is an approver in an active or pending workflow is permanently purged using System Identifier Purge, the approver in the workflow is automatically replaced with System user/Technical user (system user v4admin).

Environment

  • SAP SuccessFactors Employee Central
    • SAP SuccessFactors Employee Central Workflows

Cause

A purged user cannot remain as an approver in a workflow. Therefore, the system replaces the deleted approver based on predefined internal logic.

Resolution

This behavior is expected.

  • When a manager who is serving as an approver is permanently purged using System Identifier Purge, the system automatically assigns v4admin as the approver in all active or pending workflows.
  • This fallback mechanism is hard coded and cannot be configured or modified.
  • The replacement is necessary to prevent workflow corruption or errors that could occur if a purged user ID remained referenced in the workflow step.
  • This behavior is applicable only for workflows routed through the Manager relationship. It does not apply to workflows routed using Dynamic Groups.
  • After the replacement occurs, the workflow remains accessible and can continue to be processed or reassigned by administrators.

Keywords

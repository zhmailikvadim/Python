Symptom

  • For certain countries, ADR2-TELNR_LONG removes the leading 0 from the telephone number.
  • Want to understand which configuration affects this behavior.

Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.

Environment

  • Sales and Distribution (SD)
  • Logistics Execution (LE)
  • SAP ERP Central Component
  • SAP ERP
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA
  • SAP NetWeaver

Reproducing the Issue

  1. Maintain a telephone number in BP as '0123456'.
  2. Verify the telephone number stored in ADR2-TELNR_LONG as '+49123456'.
  3. Observe that the leading digit '0' is removed.

Cause

  • ADR2-TELNR_LONG : Complete Number: Dialing Code+Number+Extension

The content of this field is automatically calculated by the system based on the telephone number and country code components.

  • The customization in t-code OYC4 affects how the telephone number is stored in the field ADR2-TELNR_LONG.

    Tel. To :
    International Dialling Code for Telephone/Fax
    The international dialing code is the number for a country or region in accordance with the ITU-T standard. Example: The USA has the international dialing code 1.

    Digit del. :

    Digit to be deleted for calls from abroad
    To dial some numbers from abroad, the first digit of the local dialing code must be omitted.

    Example: 06227-34-0
    (local dialing code for Walldorf and the SAP number)
    If you dial this number from outside Germany, you must dial 49 for Germany and omit the leading zero of the local dialing code. In this case enter a zero in this field.

    For some other countries you must enter the number 9 (for example, Spain).

    For destination countries for which the leading digit does not need to be omitted (for example, France), leave this field empty.

Resolution

In this sample, in t-code OYC4, the field "Digit del." is set to 0, so observe that the leading digit '0' is removed.

In case, that the number stars with '+', then the prexif is NOT check with T005k table and number is automatically stored into database.

Symptom

During the creation of VZ01 notifications for the "Digital Pension Overview" using the report RentUeG: Create Notifications (RP_PAYDE_RUE_CREATE_NOTIFS), data is deleted in the value table (P01RUE_VZ01_WERT) and the standard notifications table (P01RUE_VZ01_PDF) if no

  • annuity payments,
  • One-time amounts
  • Both or Right to Choose 

exist.

Other Terms

CL_HRPAYDE_RUE_DATABASE_ACCESS->IF_HRPAYDE_RUE_DATABASE_ACCESS~DELETE_NOTIF_DB, P01RUE_VZ01_WERT, P01RUE_VZ01_PDF,

Reason and Prerequisites

Before new notifications are written, any existing old notifications with the status "New" are deleted. A value range table with the relevant GUIDs is used to delete the tables P01RUE_VZ01_WERT and P01RUE_VZ01_PDF.

For notifications without

  • Pension payments in table P01RUE_VZ01_B02B,
  • One-time amounts in table P01RUE_VZ01_B02C,
  • Both or right to choose in table P01RUE_VZ01_B02D   

this value range table is initial. As a result, all entries in the tables P01RUE_VZ01_WERT and P01RUE_VZ01_PDF are then deleted. - Also value specifications and standard notifications of other personnel numbers.

Solution

Implement the attached correction instructions or import the specified support package.

After you implement the corrections, the system deletes values from the tables P01RUE_VZ01_WERT and P01RUE_VZ01_PDF only if relevant GUIDs exist.

After you import the HR Support Package or implement the correction instructions, you must perform the following activities.

Create the VZ01 notifications for the "Digital Pension Overview" using the report RentUeG: Create Notifications (RP_PAYDE_RUE_CREATE_NOTIFS).

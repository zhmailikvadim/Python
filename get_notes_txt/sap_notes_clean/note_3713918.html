Symptom

During a posting containing multiple company codes dump ASSERTION_FAILED appears in CL_FINS_ACDOC_POSTING_EVENTS=>IF_FINS_ACDOC_POSTING_FRAMEWRK~POST, because a document number does not exist in mt_acdoc_number, but it exists in in mt_acdoc_projected.

Other Terms

CFIN

Reason and Prerequisites

In case more than one company code is processed there can be the situation in which two company codes exist with the exact same identical document number. In case one of the documents fulfills criteria of creating a header for BSTAT 'U' and the other does not fulfill this criteria, currently no header is created, as items are checked per document number only. The company code informaton must be considered here, too.

Solution

Please implement the corrections or implement the corresponding support package.

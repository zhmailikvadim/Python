Symptom

When changing the item quantity in a service order created as a follow up transaction of a solution order, the pricing is not re-determined for the item.

However, when doing the same in a standalone service order (not created from solution order), the pricing is redetermined automatically.

Environment

SAP S/4HANA Cloud Public Edition

Cause

For service orders created as follow-up transactions of solution orders, certain restrictions apply. So it's not possible to change any commercial data defined in the solution order for the service order.

It's not possible to edit the following data defined for the service order:

  • Sold-to party
  • Service contract
  • Pricing
  • Item Category
  • Organizational data related to sales

The item quantity is a changeable field, however, since the pricing cannot be changed, then this quantity change doesn't redetermine the pricing. This behavior is the expected.

Resolution

If it's necessary to add more quantity of an item or add a new item and also sum the price of this item to the net value, it's possible to add unplanned items to a service order that references a solution order item. Unplanned items are assigned to the sold-to party and sales organization defined in solution order. However, if needed, it's possible to change the bill-to party, payer, payment terms, and the ship-to party defined for the item.

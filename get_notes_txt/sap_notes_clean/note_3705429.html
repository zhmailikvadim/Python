Symptom

  • When transferring a Focus PDS for WBS element using CURTOADV_CRT_FOCUS, an error occurs related to the corresponding WBS element.
  • Error message: /SAPAPO/CURTO 201 - Account assignment object for WBS element &1 not found.
  • Message No. CURTO1009.

Environment

  • SAP S/4 Hana with Embedded PP/DS
  • Release independent

Reproducing the Issue

  1. Execute transaction CURTOADV_CRT_FOCUS for WBS element
  2. Check for error message /SAPAPO/CURTO 201 - Account assignment object for WBS element &1 not found.
  3. Observe Message No. CURTO1009.

Cause

  • Account assignment object does not exist in ePP/DS in table /SAPAPO/ACCKEY.
  • When creating a Focus PDS for WBS element using CURTOADV_CRT_FOCUS the corresponding account assignment object (WBS element) must be available in PP/DS. 
  • The direct Transfer of WSB Elements is not realized in SAP Standard for S/4 Hana with ePPDS. 

Resolution

  • In order to get transferred a Focus PDS for WBS element correctly, the corresponding account assignment object (WBS element) must be available in PP/DS.
  • This can be achieved by assigning the WBS element to a network at S/4 System and there is an active Integration model for this network to transfer the network to PP/DS.
  • Also the transfer of any orders/planned independent requirement will lead to the creation of an accounting object with WBS element.
  • If there has been transferred a planned independent requirement, there was created the corresponding account assignment object as an entry in table /SAPAPO/ACCKEY but only with the internal object number from S/4 Hana (field PRPS-PSPNR).

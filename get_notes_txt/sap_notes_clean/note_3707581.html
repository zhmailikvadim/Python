Symptom

  • Master data or transaction data changed, transfer the changes to IBP system through RTI. However, no changes reflected on IBP.
  • There is no inbound queue in bgRFC monitor.
  • There is no inbound message in 'Message Monitoring' app.

For example, 

Change plant name and transfer it to IBP by integrate model. However Location name doesn't change on IBP.

Environment

  • SAP ERP
  • SAP S/4HANA
  • SAP Integrated Business Planning

Reproducing the Issue

  1. Change master data or transaction data.. 
  2. Activating the integration model.
  3. Check the mapped master data on IBP side, for example location name in IBP. The location name remains unchanged.
  4. No active queue in bgRFC Monitor.
  5. No inbound messages observed in the Message Monitoring app.

Cause

Since there is no inbound queue in IBP, data doesn't transfer to IBP at all.

Make sure configuration has been set up properly on source system. Check IBP RFC destination system configuration on /SM59.

Do 'Authorization Check' for IBP system Connection, there is 'Name or password is incorrect (repeat logon)' error. 

The RFC destination system logon user has been locked. 

Resolution

Unlock the IBP RFC destination system logon user in the Source system.

  • Reset the password of IBP RFC destination system logon user by following SAP Note 3390056.

In the Source system, navigate to transaction /SM59 and check IBP RFC destination system settings.

  • On the 'Special Option' tab, use 'Fast serializer'.
  • Refer to SAP Note 3110007 for managing RTI transactional data queues.

See Also

  • Refer to SAP Note 3110007 - IBP Real-time Integration: Information/Restrictions.
  • Refer to SAP Note 3390056 - IBP: RTI error 'Name or password is incorrect (repeat logon)'.

Keywords

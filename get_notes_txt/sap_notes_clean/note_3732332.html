Symptom

You are trying to use the REST API POST method for on account-service or individual-account-service to create External IDs but the IDs sent in the request payload are not considered

Environment

SAP Sales & Service Cloud Version 2

Reproducing the Issue

  1. Use the Customer or Individual Customer Service on the public API package: sap sales/service cloud v2 rest api (Try Out) or call the same REST API from a client tool.
  2. Send a POST request to create an Individual Customer including an externalIds array.
  3. Observe that the creation succeeds but the response does not contain the External ID and it is not stored.

Cause

  • By design, the externalIds collection for Individual Customer is not creatable (creatable=false in the API metadata). External IDs cannot be set during POST creation.
  • The related fields (External Customer ID and External Business Partner ID) are configured as not creatable and not updatable; they are intended to be populated by standard replication from the connected system.
  • The standard process is: create the Customer or Individual Customer, send it to the connected system, then a confirmation message returns and sets the External ID in Sales Cloud. Corrections are allowed only via the External ID Mapping UI to cover rare cases.

Resolution

To create External IDs , use the standard replication services from S4HANA or ERP. Alternatively, Data Impex tool can also be used to create or update the external id's in bulk mode.

Symptom

  • Importing preferred supplier data with preferred supplier level 6 or 7 fails.
  • The system returns the error message: "Invalid Level".
  • Importing preferred supplier levels 1 to 5 works without error.

Environment

  • Qualification/Disqualification/Preferred: Preferred

Reproducing the Issue

  1. Import preferred supplier data for a qualified supplier using preferred supplier level 6 or 7.
  2. Observe that the system displays the error "Invalid Level" and does not import the record.

Cause

Only five preferred supplier levels are supported. System only considers uniquename between 0 to 5 as a valid level. If uniquename is anything other than 0 to 5, Invalid level is thrown.

Resolution

While Import Preferred Supplier Levels do not limit the number of entries that you create, only maintain supported range of 1 to 5.

  1. Manage > Administration > Site Manager > Data Import/Export
  2. Choose Export tab and search for Export Preferred Supplier Levels
  3. Download the PreferredSupplierLevelExport.csv and keep records with uniquename other than 1 to 5. Save the file
  4. Choose Import tab and search for Import Preferred Supplier Levels
  5. Click Import, select the Delete operation and browse the PreferredSupplierLevelExport.csv file
  6. Click Run

Run the preferred supplier data import with the level value between 1 to 5. If you are keeping the level as 0, success message is seen but no preferred entry is created as by default 0 is considered as Not Preferred.

  1. Manage > SM Administration > select file type as Supplier preferred list data
  2. Choose and Upload the csv file which has level value set for suppliers between 1 to 5.

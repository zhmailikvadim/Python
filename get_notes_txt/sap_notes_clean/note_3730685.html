Symptom

You are replicating products from S4 to SSCv2 and faces the following error on web message monitoring:

"Product ID XXXXXXXX must be outside the internal number range ABC to XYZ."

Environment

  • SAP Sales Cloud Version 2 Integration with SAP ERP / SAP S4HANA.
  • SAP Service Cloud Version 2 Integration with SAP ERP / SAP S4HANA.

Reproducing the Issue

In SSCv2,

  1. Go to "Settings".
  2. Go to "All Settings" tab.
  3. Under Integration tab, go to "Message Monitoring".
  4. Search for the message ID: xxxxxxxxxxx.
  5. Check the error message 
    {
        "code": "inbound-data-connector.132",
        "message": "Failed to process the inbound message.",
        "details": [
            {
                "code": "400",
                "message": "{\"error\":{\"message\":\"Product ID XXXXXXXX must be outside the internal number range ABC to XYZ.\",\"code\":\"sap.crm.md.service.productService.productMessage.displayIdInsideInternalNumberRange\",\"target\":\"{displayId}\",\"details\":[{\"message\":\"Enter an ID for the Product.\",\"target\":\"{displayId}\",\"code\":\"sap.crm.md.service.productService.productMessage.displayIdIsInitial\"}]}}"
            }
        ],
    }

Cause

When you choose to have same product id in both systems , you need to adjust the internal product number range. The numeric numbers which are sent under receiverDisplayId (to be considered as internal number of V2) are falling within the internal number range maintained.

Resolution

We recommend aligning this range with the number range group of products being sent from S4HANA.
After review it, the replication process should function as expected. 

Keywords

receiverDisplayId, product replicaiton.

Symptom

There is high disk usage on ABSTRACT_SQL_PLAN_ENTRIES_ table.

For example, you can see the high disk usage in the results of HANA_Tables_LargestTables_* SQL statement. 

SCHEMA_NAMETABLE_NAMERECORDSDISK_GBMEM_GBLOB_DISK_GB
_SYS_PLAN_STABILITYABSTRACT_SQL_PLAN_ENTRIES_2162981062.919.911062.89

Environment

  • SAP HANA Platform Edition 2.0 

Cause

Plan stability is used and it uses high disk space. 

Resolution

1. Identify for what table's it's enabled.
You can do that checking either HANA Monitoring View ABSTRACT_SQL_PLANS or running HANA SQL Statement SQL: "HANA_SQL_PlanStability_AbstractSQLPlans" (available via SAP Note 0001969700).

The column COMPILATION_OPTIONS of monitoring view M_SQL_PLAN_CACHE contains ABSTRACT SQL PLAN if an abstract SQL plan is used.


2. Then you can either remove the abstract plan specifying a filter using command:

ALTER SYSTEM REMOVE ABSTRACT SQL PLAN FILTER '<filter>' -- Remove existing abstract SQL plan


3. Once in a while as a maintenance you can remove all plans using command:  

ALTER SYSTEM REMOVE ABSTRACT SQL PLAN ALL --Remove all existing abstract SQL plans

Before removing any entries please discuss with the Team on your side that enabled the feature in case some abstract execution plans should be kept.

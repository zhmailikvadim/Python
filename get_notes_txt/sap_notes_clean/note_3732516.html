Symptom

  • Differences between totals in transaction code KE30 and transaction code FBL3N/Fiori app F2217 for COGS when using the same account intervals and the same posting period.
  • The KE30 total for COGS is lower than the total shown in FBL3N/Fiori app id F2217 for the same selection.

Environment

  • Product: SAP S/4HANA
  • Profitability Analysis
  • SAP FIORI
  • SAP R/3
  • SAP R/3 Enterprise
  • SAP ERP Central Component
  • SAP ERP
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA Finance

Reproducing the Issue

  1. Run KE30 report for a the target company code and period.
  2. In FBL3N or Fiori app F2217 (Display Line Items in General Ledger), select the same cost element intervals and the same period as in KE30, then compare the COGS totals with KE30.

Cause

  • KE30 selects only CO-PA line items; technically, items where the object number (OBJNR) represents CO-PA (for example, starting with “EO”).
  • FBL3N/F2217 select all FI line items by default, including additional line items that are not CO-PA objects. These additional items cause the difference when totals are compared with KE30.

Resolution

  1. In FBL3N/F2217, restrict the selection to line items that correspond to CO-PA objects (for example, filter by OBJNR pattern used for CO-PA such as a prefix “EO”) so that the FI selection aligns with the KE30 CO-PA scope.

Symptom

When running the J1AI report process for an asset, the system issues error message 8Z712 and the revaluation does not complete.

Environment

  • SAP ERP
  • SAP S/4HANA
  • SAP S/4HANA Cloud Private Edition
  • Inflation Management

Reproducing the Issue

  1. Execute the J1AI report for a target period.
  2. Select an asset whose Asset Date (base date) is later than the target period (for example, an asset created/decapitalized with a base date in a later month).
  3. Run the report.
  4. Error 8Z712 appears.

Cause

The issue occurs because the period you are trying to process in J1AI is earlier than the asset’s Asset Date (base date / J1AI asset date).

  • In this situation, the asset is not considered valid for revaluation in that earlier period, so J1AI does not calculate the inflation adjustment and may request a valid asset value date.
  • Additionally, if the Last Revaluation Date maintained in the inflation method is later than the J1AI asset date, the system can also raise error 8Z712 (Last Revaluation Date later than the J1AI asset date).

Resolution

  1. Verify the asset’s Asset Date (base date / J1AI asset date) in the asset master.
  2. Ensure the period you want to process in J1AI is not earlier than the asset’s base date.
  3. If you must process an earlier period, adjust the asset base date according to business rules; otherwise, run J1AI only for periods on/after the base date.
  4. Check the inflation method used by the asset, and confirm the Last Revaluation Date is not later than the asset’s J1AI asset date.
  5. Update either the Last Revaluation Date or the asset base date so both dates are aligned with the period being processed.
  6. Run J1AI again and validate that the inflation adjustment is calculated correctly.

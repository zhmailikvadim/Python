Symptom

  • This KBA outlines what stale sessions are and why running table statistics is critical in SAP SuccessFactors Agent Performance Management. 
  • Database performance issues can often be traced to two common factors: stale sessions and outdated table statistics. Understanding what these are, why they occur, and how to manage them is critical to maintaining optimal database performance and query efficiency. 

Environment

  • SAP SuccessFactors Sales Performance Management
    • Agent Performance Management

Resolution

  • What Are Stale Sessions?
    • Stale sessions are database connections that remain open but are no longer actively performing work. Although idle, they may still consume resources.
  • Why and When They Occur
    • Applications not closing connections properly
    • Abandoned or long-running transactions
    • Network interruptions or application crashes
    • After application restarts or long periods of inactivity
  • Impact
    • Unnecessary resource consumption
    • Increased contention and session exhaustion
    • Overall performance degradation
  • What Are Database Statistics?
    • Database statistics describe the data distribution within tables and indexes. The query optimizer uses these statistics to determine the most efficient execution plan.
  • When Statistics Become Stale?
    • Large data inserts, updates, or deletes
    • Bulk loads or batch processing
    • Frequent data changes throughout the day
  • When to Run Statistics?
    • Although database statistics are typically scheduled to run daily at a specific time, it is still recommended to run statistics during the day when performance issues are observed. Statistics can become stale due to ongoing data changes, which may cause the optimizer to select inefficient execution plans and result in longer-running queries.
  • Importance
    • Improves query performance and consistency
    • Prevents inefficient execution plans
    • Helps reduce long-running queries and resource contention

Proactively managing stale sessions and ensuring database statistics are kept up to date—both through scheduled jobs and ad hoc runs when needed—are key best practices for maintaining database performance and reliability

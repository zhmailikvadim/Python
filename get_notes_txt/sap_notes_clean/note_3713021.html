Symptom

There are large number of entries in the situations queue ( sit2_eng_q / sit2_inst_q ) with "On Hold" status. There are delays with processing of situation types or with situation instance operations (e.g. closing an instance).

Other Terms

Situation Handling, situations, Manage Situation Types - Extended, Situation Handling extended framework

Reason and Prerequisites

Prerequisites :

  1. You are using the Extended Framework for Situation Handling.
  2. You are using event-based triggers

Reason : 
When event-based situation triggers are used, the respective business applications can sometimes raise events multiple times (e.g. when items of a header object are modified, an event signaling a change on the header object might be raised for each item instead of just once). These events are then exact duplicates (same object, same event). A similar pattern can also occur when users repeatedly press the close or assign button for a situation instance (before the UI refreshes). Previously, the Situation Handling framework would still process all of these requests, which could result in request piling up processing queues for Situation Handling, and thus delaying further processing of situation types or instance operations.

The optimization in this note improves overall processing by filtering out duplicated entries within a short window of time.

Solution

Apply this note to improve the processing of situation types or instance operations, when there are duplicated events within a short period of time.

Symptom

  • When using the Request maintenance app (F1511), after selecting the Notification list and choosing a Notification, the edit and delete options are not displaying on screen 
  • However, when the paged is refreshed, the edit and delete options are displayed.

 

Environment

  • Plant Maintenance (PM)

Reproducing the Issue

  1. Open the app Request maintenance 
  2. Click on Notification List
  3. Click on notification
  4. Edit and delete options are not available
  5. Refresh screen - The options are now available

Cause

Expected System behaviour

Resolution

  • When the Request maintenance app is opened, the Notification is in 'Create' mode.
  • If the 'Notification List' is selected, the Notification cannot be changed as the system is still in 'Create' mode for the initial Notification, so the 'Edit' and 'Delete' options are not available.
  • When refreshing the screen, the first Notification in 'Create' mode is forgotten and the system opens the second Notification which has been selected from the 'Notification List' again (this can be seen in the URL with the respective notification number).
  • Therefore. the 'Edit' and 'Delete' options are then available.

Keywords

F1511, Notification List, Edit, Delete, Refresh

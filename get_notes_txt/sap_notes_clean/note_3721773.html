Symptom

When using the function module HR_PAD_HIRE_EMPLOYEE to hire multiple employees, Message no. 00058  "Entry xxxxxxxx does not exist in T528B (check entry)" is encountered during the processing of the second employee. Notably, the first employee is hired successfully without any errors.

Environment

  • Human Capital Management (HCM)
  • SAP ERP
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for HANA
  • SAP S/4HANA
  • SAP S/4HANA Cloud Private Edition

Reproducing the Issue

  1. Execute the custom program using function module HR_PAD_HIRE_EMPLOYEE.
  2. Encounter Message no. 00058 'Entry xxxxxxxx does not exist in T528B (check entry)'.

Cause

  • The custom program uses a combination of function modules—specifically, HR_PAD_HIRE_EMPLOYEE and HR_INFOTYPE_OPERATION/HR_MAINTAIN_MASTERDATA—to manage infotypes, which leads to compatibility issues. For instance, after executing HR_PAD_HIRE_EMPLOYEE and performing a commit, HR_INFOTYPE_OPERATION is used to independently create another infotype, e.g. IT0105.
  • This is problematic because HR_PAD_HIRE_EMPLOYEE utilizes the new/decoupled infotype framework, whereas HR_INFOTYPE_OPERATION and HR_MAINTAIN_MASTERDATA rely on the old infotype framework. Mixing these frameworks within a single program is not supported and causes conflicts.
  • From a technical standpoint, when processing the first employee, the context is set to "LGCY" (legacy) in the class CL_HRPA_INFOTYPE_FACTORY. This context indicates the use of the old infotype framework. Consequently, when processing the second employee through the custom program, within the T582ITVCLAS_READ method of the same class, the logic class for IT0001 is incorrectly overwritten by CL_HRPA_INFTY_0001_CE due to the legacy context setting. This results in an improper framework application and potential data handling errors.

Resolution

To accommodate the specific requirement of creating an infotype separately after executing the function module HR_PAD_HIRE_EMPLOYEE within a custom program, the function module HR_CONTROL_INFTY_OPERATION should be used to create the target infotype within the custom program. The function module HR_CONTROL_INFTY_OPERATION supports both the old and new infotype frameworks.

To utilize the new infotype framework within HR_CONTROL_INFTY_OPERATION, the following configurations are necessary:

  1. Run t-code SM30 to change table T77S0 configuration:

    • Set the switch CCURE PC_UI ("PC UI Main Switch") to 'X'.
    • Additionally, set one of the following switches to 'X':
      • CCURE MAINS ("Concurrent Employment Main Switch")
      • CCURE GLEMP ("Management of Global Employees")
      • CCURE GLOPY ("Payroll: Global Employee")

  2. Run t-code SM30 to change table T582ITVCLAS configuration:

    • Ensure the relevant infotype is set to completely decoupled by assigning NITF_ADM = '3' ("Permitted in All Circumstances").

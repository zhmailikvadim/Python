Symptom

  • Garbled characters appear when printing PDF
  • Install the Monotype Fonts for Printing package from Mono Type Fonts for Printing to the windows system as described in the following KBA:
    Note 2964063 - How to use mono type font printing for PDFPRINT

Environment

  • Product: SAP S/4HANA foundation
  • Print and Output Management

Reproducing the Issue

When printing the PDF directly from SAP system the print out looks different with preview.

Cause

The font like  "AdobeSongStd-Light" is embedded for Chinese characters.
The font is slightly lighter than others, that may cause the low print quality on particular printers. 

Resolution

1. Modify form template to use other bolder fonts e.g. AdobeHeitiStd-Regular

2. Modify font mappings in file FontSubstitution.xml which can be found in C:\Program Files (x86)\Common Files\SAP Shared\Fonts.
Some further restrictions by using this option:
a. Font "AdobeSongStd-Light" is now embedded in PDF, need to remove Embed option first at ADS server in file 'custom_xfa.xci'
b. SubstitutionFont need to be imported to the same folder above
c. Windows restart is required(SAPGUI machine) to see the change taking effect

Symptom

When executing the Post Credit-Risk-Based Impairment job in Schedule General Ledger Jobs, the run fails with the error message:

'Unable to get tax rate of journal entry &1/&2. Please open a ticket.'

The error may occur for one or multiple journal entries and prevents successful completion of the impairment run.

Environment

SAP S/4HANA Public Cloud.

Cause

The error occurs when a journal entry selected for credit-risk-based impairment contains a tax code but has no corresponding tax line / tax base data.

When a tax code is present on a document line, the impairment program must determine a tax rate to decide whether valuation should be performed on a net or gross basis. This tax rate is derived from the document’s tax data. If no tax base or tax amount exists, the system cannot determine a valid tax rate and terminates the run with this error.

Typical scenarios include, but are not limited to:

  • Migrated open items where VAT was settled in a legacy system and no tax line was migrated

  • Zero-amount invoices where no tax line is generated

  • Documents posted with tax codes but without tax calculation

  • Documents that are not suitable for impairment but are still selected by the run

This behavior is expected by design.

Resolution

Ensure that documents selected for credit-risk-based impairment are tax-consistent:

  • Documents that require impairment must either

    • contain a valid tax code and corresponding tax line data, or

    • contain no tax code at all.

  • Documents that should not be impaired (for example, clearing documents or other non-credit-risk-relevant items) should be excluded from the impairment run (for example, via account selection or semantic tagging).

If the affected documents are migrated open items where tax was already settled in a legacy system and no tax line exists by design, there is currently no standard way to process these documents in the credit-risk-based impairment run. In this case, the only supported options are to exclude the documents from the run or request a product enhancement.

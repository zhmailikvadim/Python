Symptom

System experience slow performance during RF (Radio Frequency) transactions in SAP EWM while processing Warehouse Orders. It is not clear which backend processing step or Function Module is causing the delay during RFUI execution. Standard traces do not clearly identify which RF processing steps are responsible for the long runtime.

Environment

  • S/4 HANA
  • SAP Extended Warehouse Management Radio Frequency (SCM-EWM-RF)

Resolution

To identify slow-running Function Modules during RF processing, use the RF performance measurement tool as follows:

  1. Activate the RF performance tool:
    Execute transaction /SCWM/RSRC and activate the flag "RF performance measurement"


  2. Execute the RF transaction:
    Log in to RFUI and process a Warehouse Order (WO).

  3. Run the performance analysis:
    Execute transaction ST13 and enter PERF_TOOL and execute.

    Enter EWM_RF_ANALYSIS and execute
    On the next screen, press button "Details"


    Review the list of executed Function Modules and their runtime.

This tool highlights only the slow-performing Function Modules, allowing you to identify performance bottlenecks in RF processing and focus analysis on specific backend logic.

Note: The tool is designed to identify slow RF processing steps and does not trace every activity.

Keywords

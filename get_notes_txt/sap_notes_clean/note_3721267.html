Symptom

  • The issue occurs when creating an application using the standard project template in the managed app router.
  • Error message: ValidationError: Format validation failed (Route references unknown destination "destName").
  • The application is created successfully but fails to run from the run configuration.

Environment

SAP Business Application Studio

Reproducing the Issue

  1. Open SAP Business Application Studio (BAS).
  2. Create a new React application using the standard project template: Managed app router.
  3. Attempt to run the application from the run configuration.
  4. Observe the error message: ValidationError: Format validation failed (Route references unknown destination "destName").

Cause

The xs-app.json file references a destination that is not properly configured in the destination service.

Resolution

To resolve the format validation error, follow the below steps:

    1. Open the Project Explorer in BAS.
    2. Navigate to the UI module project folder.
    3. Edit the .env1 file and update it using the following format:  

      destinations=[{"name":"<destination_name>","url":"https://<destination_name>.dest","proxyHost":" http://127.0.0.1","proxyPort":"8887"}]

      destination_name: Use the exact destination name associated with your service as define in the cockpit in your SAP BTP account

    4. Save the `.env1` file.
    5. Refresh the Run Configuration panel or reload the browser.
    6. Run the application.

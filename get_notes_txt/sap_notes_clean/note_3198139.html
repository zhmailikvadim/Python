Symptom

Incorrect Local Tax ID are copied to purchase orders or purchase quotations, which are created based on sales orders or sales quotations.

Reproducing the issue

Scenario:

  1. Add a sales order or sales quotation with customer related Local Tax ID (Sales Order window, Accounting tab).
  2. Use the Procurement Confirmation wizard to create a purchase quotation based on the sales order or sales quotation created in step 1.

Expected behavior:

Local Tax ID maintained in the sales order or sales quotation are not copied to the purchaser order or purchase quotation, Local Tax ID field (Purchase Quotation window, Accounting tab). Local Tax ID field in the generated purchase documents should retrieve local tax ID defined in the vendor.

Actual behavior:

Local Tax ID maintained in the sales order or sales quotation are copied to the purchase order or purchase quotation, Local Tax ID field (Purchase Quotation window, Accounting tab).

Solution

SAP intends to provide a feature package or patch to solve the symptom that is described in this SAP Note. The section References below lists the related feature package or patch once they become available. Be aware that References can only be confirmed at feature package or patch release date. SAP delivers feature packages or patches only for selected releases at its own discretion, based on the business impact and the complexity of implementation.

Workaround:

Manually correct the local tax ID in the purchase order and purchase quotation documents, which are created based on the sales order or sales quotations.

Symptom

SAP Product Lifecycle Costing 4.0 SP10 Hotfix 1 is released and you want to install it or upgrade from a lower version.

Solution

Release Notes/Changelog:

Bug Fixes:

  • An issue in collaborative mode that could cause incorrect item ordering after adding, deleting, or moving items and then saving the changes, has been resolved.
  • Performance has been significantly improved when moving assemblies with deep structures within the same calculation version.
  • To avoid displaying incorrect differences, the Compare view now treats falsy values (null, false, "", and 0) as equivalent.
  • The costing sheet and component split are no longer removed from the UI when updating master data for selected items, ensuring calculation version master data remains unchanged and selections stay valid.
  • A warning is now shown in the status bar on save to inform users that changes to the root item cannot be saved in collaborative mode.
  • The lazy loading mechanism was corrected to prevent all items from being loaded at once, which previously caused the application to become unresponsive.
  • Price Components are no longer incorrectly displayed in the Compare view; the column is now hidden from the grid and cannot be dragged and dropped from the side panel.
  • Item loading now proceeds correctly even if one of the requests (details, master data, or calculated values) fails, as long as the others complete successfully.
  • Filter operators such as starts with, contains, and ends with now work correctly for custom formula fields (CFFs) that do not allow free‑text values when opening calculation versions with predefined filters.
  • Sorting by predecessor was stabilized to prevent invalid item positions and application crashes when some items are filtered out.
  • Mass change logic was optimized to avoid unnecessary updates and recalculations; changes are now applied at once and refreshed afterward, without triggering multiple recalculation requests.
  • The master data update process was fixed to complete successfully without executing unnecessary validations.
  • Comparing calculation versions no longer results in loading errors or warnings that could leave the view stuck in a loading state.
  • Validation errors triggered by deleted custom fields were removed, allowing the save operation to complete successfuly.
  • The master data warning icon now is hidden once all master data timestamps are are up to date and identical to the one defined on the calculation version header.
  • Running the transportation tool in replace mode no longer removes all custom fields from layouts; fields present in both source and target systems are preserved.
  • Repeated merge conflicts in collaborative mode, caused by simultaneous deletions and insertions by different users, are now properly resolved on save.
  • A resource leak in the Master Data Replication tool was fixed by ensuring database connections are properly released.
  • Change tracking was corrected to ensure the recorded user always reflects the one who actually performed the save action.
  • Mass change operations are now permitted with the read instance privilege, aligning the behavior with item update authorization rules.

Analytics Integration

  • Hana Analytics Adapter was removed from PLC shipment. It should be integraded following the steps mentioned in note:  https://me.sap.com/notes/3704865 

New Installation and Upgrade

Follow these instructions when installing PLC for the first time or upgrading an existing version.

Note: Whether you’re installing PLC for the first time or upgrading, make sure that your system includes package Glibc-2.29 or higher to ensure that the PLC AFL Library loads properly. The minimum supported version of the SUSE Linux Enterprise Server Distribution (SLES) for this package is SLES 15 SP3.

New Installation of PLC

If you’re installing PLC for the first time, you don’t need to install any previous versions. Install 4.0 SP10 Hotfix 1 and follow the instructions in the Administrator's Guide for SAP Product Lifecycle Costing under Installation Steps and Checks.

Make sure you've installed the following HANA DB and component revisions:

  • Revisions between and including 2.00.79.02 (SPS07) - 2.00.088.0 (SPS08) for HANA DB 2.0 
  • HANA Component EPMMDS Rev 79.02 only for HANA 2.0 Rev 79.02 - EPMMDS Rev 88.0 only for HANA 2.0 Rev 88.00
  • XS Advanced Runtime for SAP HANA, revisions between and including 1.2.4 - 1.4.5 (SAP EXTENDED APP SERVICES 02, SAP EXTENDED APP SERVICES 03 or SAP EXTENDED APP SERVICES 04)

Importing from SAP ERP: If you plan to import data from an SAP ERP system, apply these SAP Notes:

  • SAP Note 2276456 and SAP Note 2278609 for BoM & Routing import.
  • SAP Note 2281004 and SAP Note 2281007 for document structure import.

Upgrading PLC

Before starting any upgrade activity check the status of the entries in the installation log.

You can access the installation log in the PLC database table <SCHEMA_NAME>.sap.plc.db::basis.t_installation_log.

If the result of the query: select count(*) from "<SCHEMA_NAME>"."sap.plc.db::basis.t_installation_log" where state not in ('running','started','finished') is not 0 do not start the upgrade and contact PLC support.

If you’re upgrading from PLC 4.x.x to a newer version, please follow the Administrator’s Guide for SAP Product Lifecycle Costing under Upgrade an Existing Installation.

Consider the following about SAP HANA:

  • When upgrading, if you're already using SAP HANA SPS05 (stable) or SAP HANA SPS06 and XSA revision 1.0.155, you can upgrade to SAP HANA SPS07/SPS08 and to XSA revision 1.2.x/ 1.3.x.

Upgrading your SAP HANA System

If you need to update your SAP HANA system from SAP HANA 1.0 to SAP HANA 2.0, choose the scenario that works best with your operating system:

Note: You can find the complete list of operating systems that support SAP HANA 2.0 in the SAP Note 2235581.

  1. If your operating system supports SAP HANA 2.0 (e.g. SUSE Enterprise Linux version 12.1 or later), do the following:
    • Upgrade your existing SAP HANA installation according to the SAP HANA Server Installation and Update Guide.
    • Upgrade SAP Product Lifecycle Costing using the information provided in the Administrator's Guide for SAP Product Lifecycle Costing under under Upgrade an Existing Installation.
  2. If your operating system does not support SAP HANA 2.0 (e.g. SUSE Enterprise Linux version 12.0 or earlier), do the following:
    • Upgrade your current operating system to a version supported by SAP HANA 2.0. Additional information regarding the update of your current operating system can be found in the SAP HANA Master Guide (SAP HANA 2.0 SPS00). You can find the complete list of operating systems that support SAP HANA 2.0 in SAP Note 2235581.
    • Upgrade the SAP HANA database and SAP Product Lifecycle Costing as described in the first scenario.
  3. If upgrading your current operating system is not possible, you can:
    • Set up a new system with the corresponding operating system.
    • Clone the current SAP HANA installation to the new operating system. For more information see Copy or Clone an SAP HANA System in the SAP HANA Administration Guide.
    • Upgrade the SAP HANA database and SAP Product Lifecycle Costing as described in the first scenario

Symptom

  • During S/4HANA conversion in SUM PREP_EXTENSION/IS_SELECT, add-ons are shown with status UNDECIDED: SRMNXP01, SRM_SERVER.
  • SUM offers “Upgrade” or “Delete” via SAINT packages but requires corresponding SAINT/ACP packages to proceed.
  • If the selection is Upgrade, the error message is: “The program has not found a SAINT package that fits to add-on SRM_SERVER.”

Environment

SUM 2.0

Reproducing the Issue

Repeat SUM.

Cause

The SRM components (SRM_SERVER, SRMNXP01) cannot be deleted via SAINT; they must be planned for deletion in Maintenance Planner so that the stack.xml contains the to-be-deleted flag and SUM removes them during conversion. (refer to SAP Note 2251946)

Resolution

Check the stack xml file, the SRM product should have the deletion flag as below, if not, then plan the new stack xml file without any manually removing the SRM product, Maintenance Planner will calculate the target action automatically.

==

<to-be-deleted>
<application-system creationclassname="SAP_XXXystem" system-number="">
...
<initial-state>
<product>
<caption>SRM 7.0 EHP4 ON ERP 6.0 EHP8</caption>
<name>SAP SRM ON ERP</name>
<version>704 ON 608</version>
<ppms-number>73554900100900000506</ppms-number>
<vendor>sap.com</vendor>
</product>
<software-feature selectively-installable="N">
<caption>SRM 7.0 EHP4 ON ERP 6.0 EHP8: SRM Server</caption>
<name>SAP SRM ON ERP</name>
<version>704 ON 608</version>
<vendor>sap.com</vendor>
<ppms-number>735549001009000005061</ppms-number>
</software-feature>
<software-component-entry>
<caption>SRM SERVER 7.14</caption>
<name>SRM_SERVER</name>
<version>714</version>
<vendor>sap.com</vendor>
<ppms-number>67838200100200021494</ppms-number>
...
<software-component-entry>
<caption>SRMNXP01 150</caption>
<name>SRMNXP01</name>
<version>150</version>
<vendor>sap.com</vendor>
<ppms-number>73554900100200002193</ppms-number>
</software-component-entry>
</initial-state>
<target-state/>
</to-be-deleted>
</sp-stacks>
===

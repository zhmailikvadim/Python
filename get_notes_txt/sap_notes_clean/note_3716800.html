Symptom

During automatic tendering, when the carrier is configured to receive the tender via email the system does not create a “Request to Carrier” entry in the Communication History tab. This entry is generated only when the tender is transmitted via EDI (XML).

When the carrier responds by email, the system processes the response after the RFQ update, and a “Confirmation by Carrier” entry is created in the Communication History.

Environment

  • SAP S/4HANA 1709+

Reproducing the Issue

  1. Open Freight Order
  2. Click on Edit
  3. Click on Subcontracting
  4. Select Automatic Tendering
  5. Select any existing Tendering Profile and perform the action
  6. Check the Communication History Tab, you will notice that Request to carrier is not shown for Email Tendering

Cause

Standard Behavior

Resolution

As per SAP standard behavior the Communication History of a Freight Order is not designed to record the outgoing tender email. Therefore, when a carrier receives the tender via email “Request to Carrier” entry won't get created in the Communication History tab.

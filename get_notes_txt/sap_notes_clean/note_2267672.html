Symptom

Execute the T-code: FTR_EDIT, change the end term of interest rate instrument to extend it to future date, the cash flows of the deal did not update correctly as the modification.

Environment

  • Financial Accounting (FI) 
  • SAP R/3 
  • SAP R/3 Enterprise 4.7 
  • SAP ERP Central Component 
  • SAP ERP 
  • SAP enhancement package for SAP ERP 
  • SAP enhancement package for SAP ERP, version for SAP HANA

Reproducing the Issue

  1. Create a transaction of interest rate instrument by T-code: FTR_CREATE, set the “Repayment structure”, “Repayment form” as “Final repayment”, and “Frequency” as “Manual input”;
  2. Save the transaction;
  3. Then edit the transaction of interest rate instrument by T-code: FTR_EDIT, modify the term of the deal to extend it a future date;
  4. Check the flows of the transaction, will find that they are not updated.

Cause

The issue is that the 'Frequency' in the 'Repayment Structure' has been saved as 'Manual input'. This means that the 'Repayment Structure' is never synchronized with the data from other parts. Everything is manually input and the system will never try to do so. 

Resolution

  1. Execute T-code: FTR_EDIT then choose the deal;
  2. Click on ‘Change’ button;
  3. Change the 'Frequency' in the 'Repayment structure' back to 'At end of term', this will give the indication to the system that the repayment structure is not manually input anymore;
  4. Change the value in the field 'End' in the 'Term' section.
  5. It will update the 'End term' as well as 'Due Date' of final repayment, and the nominal interest flows will be also generated.

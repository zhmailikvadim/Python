Symptom

An email notification indicates that an API certificate for a service provider is about to expire.

Environment

SAP Cloud Identity Services

Resolution

  1. Click on the link to the expiring API certificate provided at the bottom of the email. It should end with "/trust/password".
  2. Next to "Certificates", click on "Add".
  3. Generate, Upload, or Insert a certificate depending on the business scenario.
  4. Refer to SAP Help for further details: Configure Certificates for API Authentication.

    Notes:

    • There is no need to follow SAP Note 3562650 - Renewal of SAP Cloud Identity signing certificates for SAP S/4HANA Cloud Public Edition and SAP IBP. This SAP Note is only relevant for the tenant's Signing Certificate, and not for an Application's API certificate.
    • If the Application in question is a bundled SAP application, such as S/4HANA Cloud, IBP, or SuccessFactors, and there is a need to know how to obtain their renewed certificates, open a Case under the respective Application's component:
      • For S/4HANA Cloud: XX-S4C-OPR-INC.
      • For IBP: SCM-IBP-OPS-INC.
      • For SuccessFactors: LOD-SF-PLT-IAS.

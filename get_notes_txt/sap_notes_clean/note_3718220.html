Symptom

A key user attempts to use the Adapt UI functionality to embed a BTP custom application as a subsection content (e.g., in the Manage Sourcing Projects app, ID F4861).

  • When navigating to Advanced Settings, the Sandbox Parameters (such as allow-scripts, allow-same-origin, etc.) are missing or not visible.

  • The issue is environment-specific; these parameters are visible in one subaccount (e.g., Development) but missing in another (e.g., Test) despite the user having identical roles.

Environment

SAP S/4HANA 2023

Cause

  • Framework Delta: Although both systems appear to be on the 1.120 branch, the version in the faulty environment (1.120.0) is a legacy build from October 2023.

  • Feature Availability: The capability to manage sandbox settings for iFrame-embedded content via UI Flexibility was introduced in later builds. The working environment (1.120.39) uses a 2025 cloud-aligned build that includes these backported advanced adaptation features.

  • ATO Status: In many cases, the affected environment also lacks a configured Adaptation Transport Organizer (ATO), which is required for full UI flexibility management.

Resolution

  1. Upgrade SAPUI5: Update the SAPUI5 framework in the affected environment to match the working environment version (1.120.39) or higher.

    • Note: SAP officially recommends SAPUI5 version 1.124 or higher for full support of these specific sandbox parameters.

  2. Enable ATO: Ensure the Adaptation Transport Organizer is correctly set up in the target system using transaction S_ATO_SETUP.

  3. Sync Cloud Update Cycles: Align the RISE Private Cloud update schedules to ensure feature parity across the Development, Test, and Production landscapes.

Symptom

  • Job is failed when target table is enabled with option: bulkload mode: truncate.
  • Error message:
    Data flow XX|Loader XX SQL submitted to ODBC data source <XX> resulted in error <[DataDirect][ODBC lib] You are not licensed to use this ODBC driver with the DataDirect ODBC Driver Manager under the license you have purchased.Thank you for your cooperation.>. The SQL submitted is <DELETE FROM `XX`.`XX`  WHERE 1=1>.

Environment

  • SAP Data Services

Reproducing the Issue

  1. Create a job with table DB table set with bulkload mode: truncate.
  2. Run job fails with above error.

Cause

Target DB user doesn't have requested authentication.

Resolution

  1. When bulkload mode: truncate is used, DS job will sent 'Delete' SQL command to target DB firstly.
  2. Make sure the DB user maintained in target DB datastore has authentication to run 'Delete' command in target DB side.

Keywords

odbc driver error, sap data services, delete command error, bulkload truncate, ds

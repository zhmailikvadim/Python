Resolution

  1. Client Copy Tool

    Client copy tool can be used to copy data between clients of same or different systems. The data that is copied is defined with the help of profile with which the client copy tool is executed. Based on the profile selected, application/customizing/user master data are copied during the execution of client copy tool.

  2. Types of Client Copy 

    Client copy can be performed in three different ways based on the requirement.

    2.1. Local Client Copy 

    Using local client copy, data can be copied between two clients of the same systems. SCCLN is the transaction for local client copy. Execution can be triggered by providing source and target client values along with profile name.

    2.2. Export/Import Client Copy 

    Using Export/Import client copy, data can be copied between different systems. SCC8N is the transaction for client export. SCC7N is the transaction for client import.


    2.3  Remote Client Copy 

    Using Remote client copy, data can be copied between different systems. SCC9N is the transaction for remote client copy.

  4. Follow-up / Next Steps
     
    • For profiles where the data must be retained in target, the data should be populated into shadow table before client copy, so that the same can be inserted back to target without any data loss during client copy.
    • PCA refactoring is required where the business users are converted to classic users for profiles where the application data will be removed during client copy. Hence, retaining business user data in USR21 table and other address tables can lead to inconsistency.
  5. Transactions 
    • SCCLN – Local client copy
    • SCC8N – Client export
    • SCC7N – Client Import
    • SCC9N – Remote client copy
    • SCC3 – Client tools log analysis

      The transactions SCCLN/SCC8N/SCC7N/SCC9N are supported from 752 and above.

      Caution:

      • Please take back up of data before client copy.
      • For doing remote client copy using profiles SAP_APPL/SAP_APPX, please follow the below steps:
      1. Perform user master export from target system (SAP_USER profile)
      2. Execute the remote client copy using SAP_APPL/SAP_APPX
      3. Import back the user master data which was exported in step A.
  6. Tips 
  • Important SAP Notes related to client copy issues. 2825603-Issues with address master data after client copy
  • After a note implementation in both Source and Target Systems, client copy needs to be triggered again.
  • Client Copy execution status and logs – Transaction SCC3 can be used to monitor the status of client copy execution. Overall status of the execution and status on specific steps can be viewed here.

    If the processing status is finished and exit status (Initialization/Analysis Phase/Postprocessing Phase) don’t have any error, that indicates the client copy executed successfully. Processed table details as part of address can be viewed in “processed Tables’ tab.

    If the status has any errors, the tabs ‘General Messages’ and ‘Exit Messages’ can be used to check the details.



    7. FAQ 
    7.1. What are the steps to be followed before client copy? 

    Please take back up of data before client copy.

    For doing remote client copy using profiles SAP_APPL/SAP_APPX, please follow the below steps:

    1. Perform user master export from target system (SAP_USER profile)
    2. Execute the remote client copy using SAP_APPL/SAP_APPX
    3. Import back the user master data which was exported in step A.

    7.2. Why should we take backup of the data before doing client copy? 

Sometimes due to unforeseen circumstances data might get corrupted so it is best practice to have backup of data which can be used to restore system.

7.3. Can we retrieve the lost data after client copy? 

If a backup has been taken before executing Client Copy, then it can be used to restore the data, if there is no backup data of the target system then we cannot retrieve lost data.


7.4. When should we implement the Note? 

Notes should be implemented in both source and target systems before starting the client copy process.


7.5. Why Business Users conversion to classic? 

Application data will be affected during the client copy process. Hence retaining the users as business users will lead to inconsistent data in the system. Hence the conversion of business users to classical users are required.


7.6. How to get the converted business users be reconverted? 

Classic users can be converted to business users with the help of HCM_Sync_Reports - /SHCM/RH_SYNC_BUPA_EMPL_SINGLE or /SHCM/RH_SYNC_BUPA_FROM_EMPL.

7.8. Why user master related address data being deleted with Profiles like SAP_APPL & SAP_APPX? How to handle it? 

For doing remote client copy using profiles SAP_APPL/SAP_APPX, please follow the below steps:

    1. Perform user master export from target system (SAP_USER profile)
    2. Execute the remote client copy using SAP_APPL/SAP_APPX
    3. Import back the user master data which was exported in step A.

Symptom

When the Standardized Data Validation task is run at consolidation group level, the system shows the error message: "Rule Group not assigned for [consolidation unit]".

A custom selection excludes certain consolidation units from validation rule groups, and validation behaves as expected when run at individual consolidation unit level.

At consolidation group level, the task still attempts to validate excluded units and fails with the above message.

Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.

Environment

SAP S/4HANA Cloud Public Edition

Reproducing the Issue

  1. Configure a validation rule group assignment using a custom selection that excludes specific consolidation units from both Reported Data Validation and Standardized Data Validation rule groups.
  2. In the Data Monitor, run the Standardized Data Validation task at a consolidation group level that contains both included and excluded consolidation units.
  3. Observe that the validation fails with the message "Rule Group not assigned for [consolidation unit]" for units not assigned to the rule group.

Cause

Current application logic: when validation is run at consolidation group level, all subordinate consolidation units must have applicable rule groups assigned. Partial exclusions in the rule-group assignment are not supported at group level.

Resolution

If you need to run validation at consolidation group level, assign an appropriate validation rule group to all subordinate consolidation units so that no unit is left unassigned.

Or you can leave the Consolidation unit selection empty.

Alternatively, run the validation at individual consolidation unit level only for the intended units to respect the exclusion.

For product enhancement, submit an improvement request so that group-level validation can optionally skip excluded units: refer to: Customer Influence for SAP S/4HANA Cloud.

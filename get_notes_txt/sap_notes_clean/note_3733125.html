Symptom

  • The system is unable to open the WBS screen from Project Control - Enterprise Projects after clicking the “>” (drill-in) icon for a WBS element.
  • Error message: “Sorry, the app couldn't be opened. The navigation target ‘#EnterpriseProject-processProjectElement?...’ of the app could not be resolved. This can be caused, for example, by an incorrect content configuration, an incorrect or missing role assignment, or because your current device is not supported.”

Environment

  • SAP S/4HANA Cloud Public Edition

Reproducing the Issue

  1. Open the Project Control - Enterprise Projects app.
  2. Open Project Planning for an Enterprise Project.
  3. In the work breakdown structure, click the “>” (drill-in) icon for a WBS element.
  4. Observe that the app does not open and the error appears: “Sorry, the app couldn't be opened… The navigation target ‘#EnterpriseProject-processProjectElement?...’ could not be resolved.”

Cause

The user is missing a required business role that provides access to the “WBS Element” app (available via the Project Financial Controller or Project Manager business role templates).

Resolution

To resolve this issue, ensure the user is assigned an appropriate business role that grants access to the WBS Element application.

  1. Assign one of the following business roles to the user (based on business requirements):

    • Project Financial Controller (SAP_BR_PROJ_FIN_CONTROLLER)
    • Project Manager (SAP_BR_PROJECTMANAGER)

  2. If required, create a custom business role using one of the templates above.
  3. Assign the business role to the user via the Maintain Business Roles / Identity and Access Management application.
  4. Ask the user to log off and log on again (or clear browser cache) to refresh authorizations.
  5. Reopen Project Control - Enterprise Projects, navigate to Project Planning, and click the “>” icon on a WBS element to verify that the WBS Element app opens successfully.

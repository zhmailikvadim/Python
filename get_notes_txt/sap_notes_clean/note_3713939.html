Symptom

Updates of data in the SAP S/4HANA Cloud Private Edition system are no longer reflected in SAP Business Data Cloud.

You find an error message like the following in SAP Cloud Connector:

  • An error occurred when trying to connect to <some_number>@cf.sap.hana.ondemand.com. See 'Log and Trace Files' and in particular scc_core.trc for details. Certificate has expired

Other Terms

certificate, expired, renewal, replication

Reason and Prerequisites

The certificate used for the integration between SAP Cloud Connector and SAP Business Data Cloud has expired.

Additionally, before certificate expiry, SAP Cloud Connector will present a warning about an upcoming expiry.

Solution

SAP Cloud Connector version 2.18 and higher provides functionality for automatic certificate renewal. See SAP Knowledge Base Article 3632133 for how to enable this functionality.
We have sent this notification to all customers using SAP Business Data Cloud. If you have already enabled automatic certificate renewal, no action is required.

According to SAP Note 3302250, a regular update of SAP Cloud Connector is required to stay in the range of supported SAP Cloud Connector versions. Additionally, SAP Cloud Connector will notify users about new SAP Cloud Connector versions. When updating SAP Cloud Connector in the context of SAP Business Data Cloud, it is advisable to upgrade to the latest released version.

If you are running an SAP Cloud Connector version that is supporting additional integrations that you cannot upgrade for some reason, you can run a dedicated SAP Cloud Connector deployment specifically for SAP Business Data Cloud and SAP S/4HANA Cloud Private Edition integration. However, such a change requires dismantling the existing "Integration with SAP Business Data Cloud" UCL formation. Please contact SAP Support to reset the integration before re-creating the UCL formation.

The same is true for already expired certificates, so re-creating the UCL formation is required as well.

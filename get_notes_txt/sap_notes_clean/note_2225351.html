Symptom

Since the import of the last Support Package for CGsprint, newly retrieved credit reports from Creditreform no longer have an endorsement end date. The date is displayed correctly only when a second inquiry is called.

Other Terms

CGsprint, Creditreform, supplement, supplement end, monitoring, business information

Reason and Prerequisites

This problem is caused by a program error. This SAP Note is valid for the component CGSCD 130_700, patch level 5.

Solution

Import the attached transport request AO6K902195 (file name CGSCD_AO6K902195.SAR).

Then proceed as follows to determine the affected credit reports:

    1. Use transaction SE16 to open the table /CGS/T_MONITOR.
    2. Restrict the selection using the following criteria:
      • AGNCY (credit agency): CREFO
      • PRDCT (information products): PRTY-1, PRTY-2, PRTY-1002, PRTY-1101, PRTY-1102, PRTY-1199
      • GBTYP (Monitoring Type): Double-Click --> Single Value --> OK (Restrict to Empty Value)
    3. Execute the selection
    4. From the results list, copy all values from the "REPID (ID Number)" column to the clipboard.

Perform the repair as follows:

  1. Call transaction /CGS/CUST.
  2. Double-click the entry "Utilities" --> "Data Reorganization" --> "General: Rebuild Report Tables".
  3. Make the following restrictions:
    • Credit Agency: CREFO
    • ID Number: List from Clipboard
    • Products: PRTY-1, PRTY-2, PTRY-1002, PRTY-1101, PRTY-1102, PRTY-1199
    • Action to be performed: Select only "Update Monitoring Status of Reports" here
    • Deactivate Test Run
  4. Use F9 to start the report in the background.

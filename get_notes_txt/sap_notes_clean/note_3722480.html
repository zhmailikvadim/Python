Symptom

You observe unexpected amounts being reported in the Monthly income declaration (DMR) (RPCMIDP0).

Often, the root cause of inconsistent or unexpected values is related to customer-specific configuration, especially in:

  1. BAdI HR_PT_MID_RP
  2. Customer customizing in table view V_T596J
  3. Substitution selection (PPT_SUBS flag)

These configurations may lead to unexpected results in the report.

Environment

  • HR renewal
  • SAP S/4HANA
  • SAP ERP
  • SAP SuccessFactors Employee Central Payroll

Resolution

  1. Check the BAdI HR_PT_MID_RP implementation

    Ensure that the custom logic in the BAdI is not altering standard calculation values. Disable the implementation temporarily for comparison if needed.

  2. Review customer entries in T596J

    As said in note 1810804:

    "You can customize the wage types according to your needs on view V_T596J (customer entries). In case there are wage types customized for a cumulation wage type on view V_T596J, the system does not read the customizing for that same accumulation wage type from view V_T596I."

    Therefore, make sure that all standard wage types defined in T596I for each accumulation wage type are also present in T596J. Missing wage types cause unexpected accumulation results.

  3. Confirm substitution configuration

    The standard execution requires “Gerar Declarações” (PPT_SUBS) flag to be selected. Please ensure this flag is always marked when running the report.

Symptom

  • Necessary permission assigned but login name still missing in People search bracket
  • Hide username is turned off but Login name are still not showing up

Environment

SAP Successfactors HCM Suite

Reproducing the Issue

  1. Log in to the SAP SuccessFactors instance using the support access credentials.
  2. Navigate to the global search functionality.
  3. Search for an employee's name.
  4. Observe that the username is not displayed in brackets alongside the employee's name.

Cause

Missing `Login Name permission` in the Role-Based Permission (RBP) configuration.

Resolution

Note: The Enhanced People Search replaces the username with the login name. However, it still adheres to the tenant setting that controls the removal of both the login name and username from search results. 

  1. Navigate to the Role-Based Permission (RBP) settings in the SAP SuccessFactors instance.
  2. Grant the permission: User Permissions > Account Data > Login name
  3. Save the changes.
  4. Allow some time for the cache to refresh.
  5. Verify the global search functionality to ensure the Login name is displayed in brackets alongside the employee's name.

Important: About New Account Data Permissions

These permissions are associated with a new API enhancement for `Account data`, which is not yet publicly available to customers. Our engineering team is actively reviewing the situation to determine if it is possible to hide the table to avoid confusion. For more information you can monitor this KBA for further updates.

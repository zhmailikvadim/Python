Symptom

I need to carry out tests of the invoicing process in Mexico for CFDI with specific TAX IDs that cannot be deactivated from the production environment.

Reproducing the Issue

If the TAX is not registered in the desired environment when the supplier is uploading the XML, this error occurs: Error: "ExtDoc-3 Tax IDs in the document do not match your tax configuration in SAP Business Network".

Cause

That error is expected by the system, the TAX ID must be configured with both the corresponding ID on the buyer’s side and the corresponding ID on the supplier’s side. 

More details of the process can be verified in the guide: Tax Invoicing in Mexico.

Resolution

With reference to maintaining the TAX ID in both environments Test and Prod, this action is not possible, the TAX ID must be in a single environment.

To request the removal of the TAX ID from one environment and to place it in another, you will need to contact our support team.

See Also

You can open an enhancement request so that it may be taken into consideration in future updates that the TAX can function actively in both environments.

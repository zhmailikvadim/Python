Symptom

  • SI-CHECK item "SI2: FIN_AA" reports: "Fiscal year is not yet closed - See note: 2618023".
  • When performing fiscal year closing in Asset Accounting, the system displays: "Depreciation area XX is already closed for the year." - AC424

Environment

  • FIN Asset Accounting​
  • SAP S/4HANA ​
  • SAP S/4HANA Finance​
  • SAP S/4HANA Cloud Private Edition

Reproducing the Issue

  1. Run SI checks using program /SDF/RC_START_CHECK or via SUM and review check item "SI2: FIN_AA".
  2. Observe the message: "Fiscal year is not yet closed - See note: 2618023".
  3. Perform fiscal year closing for Asset Accounting for the affected company code.
  4. 4. Observe the error: "Depreciation area 03 is already closed for the year."
  5. In OAAR, check valuation areas for the company code and note that mentioned area is missing.
  6. Check tables T093B and T093D and note missing entries for mentioned area in the company code. 

Cause

Valuation area XX is missing in table T093B for the company code, which prevents fiscal year closure.

Resolution

  • If there's no asset yet created:
  1. In transaction OAOB, delete the assignment of the affected company code to the chart of depreciation, then save.
  2. Reassign the same company code to the chart of depreciation, then save.
  3. Verify that the system created the missing entries for the company code in tables T093B and T093D (including area 03).
  4. Re-run the SI check (SI2: FIN_AA) and the Asset Accounting fiscal year closing to confirm the issue is resolved.
  5. Perform these steps first in a test system before implementing in production.
  • If you already have assets created in specific company code, please contact SAP Support.

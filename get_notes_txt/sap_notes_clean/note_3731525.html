Symptom

You are in the BTP FrontEnd environment and use the Create Maintenance Request app (F1511A).  Once a draft is created and you navigate back to app F1511A, a popup shows a My Drafts button option.  Its purpose is to navigate to My Maintenance Requests app (F4513).  When you click My Drafts of the popup the navigation does not occur.  Sometimes it keeps looping and no information shows up.  Sometimes the navigation seems successful and sometimes the navigation takes back to the original app F1511A.  Debugging shows that an unexpected call of onCreateWorkRequestMatched occurs during navigation in the BTP environment.

Other Terms

Create Maintenance Request, F1511A, My Maintenance Requests, F4513, CreateMaintenanceRequest.controller.js, onCreateWorkRequestMatched, CrossApplicationNavigation, MaintenanceWorkRequest‑manage, MaintenanceWorkRequest‑create, loadDraft, BTP

Reason and Prerequisites

The reason is a programming error. The navigation logic triggers an unintended callback in the handler onCreateWorkRequestMatched in the BTP environment, which leads to inconsistencies in the internal navigation flow.

Solution

Implement this SAP Note in FrontEnd system to correct the navigation sequence so that the app runs the expected target without the unintended callback.

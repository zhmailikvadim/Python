Symptom

When Attempting to set initiateIASMigration to false in Learning Administration returns the error: "Identity Authentication is already active. Configuration updates are not allowed".

Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.

Environment

SAP SuccessFactors Learning

Reproducing the Issue

  1. Access the Learning Administration
  2. Navigate to System Administration > Configuration > System Configuration > SAP CLOUD IDENTITY SERVICES.
  3. Set the "initiateIASMigration" configuration to "false" error pops up

Cause

Initiate SAP Cloud Identity Services Identity Authentication Service Integration has already been initiated in the tenant. By design, the Upgrade Center task is removed and configuration updates to the migration flag are blocked once IAS is active.

Resolution

  1. Access the Bizx environment.
  2. Navigate to the Upgrade Center.
  3. Attempt to locate the "Initiate SAP Cloud Identity Services Identity Authentication Service Integration" option 
  4. Check its already Upgraded

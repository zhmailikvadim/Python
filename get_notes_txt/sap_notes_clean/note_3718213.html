Symptom

If you want to reassign a visit with assigned diagnoses in the Case Revision function and do not specify a target case, the system issues (unjustified) error messages. The reassignment and the creation of a new case still work, but the movement to be reassigned is not canceled for the source case. In addition, the diagnoses are not transferred.

Other Terms

Case revision, ISH_AT_SCOX_XDOK, commit

Reason and Prerequisites

Prerequisites:

  • Country Version AT (Austria)
  • Case Revision - Reassign a movement with diagnosis without specifying a target case

Reason:

Program error

Solution

The adjustments are available for the Support Packages of the specified releases. If you require the changes in advance, implement the correction instructions relevant for your system.

Case revision is performed correctly.

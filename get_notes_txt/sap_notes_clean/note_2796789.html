Symptom

Some IDocs show status "Distribution Unconfirmed" in SCUL

"Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental."

Environment

Release independent 

Cause

There are a couple of causes for this issue:

  • Insufficient resources to process the IDoc in the child system
  • Issue with technical setup (partner profiles etc.)
  • Missing authorizations for the communication user
  • Outbound parameter is set for batch processing but no job is scheduled, or authorization problem with the job.
  • In child system, the outbound qRFC scheduler is lacking of resource.

Resolution

Check the points below:


  • Insufficient resources to process the IDoc in the child system

    In case the child system's inbound processing of IDocs is set to start immediately in WE20 in the partner profiles, every IDoc that is sent needs at least one work process available in the receiving system.
    Otherwise the IDoc is not processed.
    In case a large number of IDocs sent simultaneously,  set the inbound processing to 'Trigger by Background Program' and schedule the periodic job RBDAPP01 as per SAP Note 399271.


  • Issue with technical setup

    Check the SAP Help Portal - Setup of Central User Administration.
    Also check the partner profiles in BD64 in the central system.

  • Missing authorizations for the communication user

    Make sure that the RFC destination in SM59 has a specific user set in the logon tab, with the naming convention CUA_<SID>_<CLNT>.
    If the setting "current user" is configured in the logon tab, the authorizations of the users will be different for every user, resulting in sporadic authorization errors.
    The user CUA_<SID>_<CLNT> must have specific authorizations. Provide the following authorizations according to the SAP Help Portal - Set Up Field Distribution Parameters and SAP Note 2000585 (for child systems as of release 731):

    • Copy of role SAP_BC_USR_CUA_SETUP_CLIENT with generated profile
    • Copy of role SAP_BC_USR_CUA_CLIENT with generated profile.
    • As of release 731, copy of role SAP_BC_USR_CUA_731_CLNT_SETUP. Maintain the object S_IDOCPART in the role with EDI_PRT = LS and EDI_PRN = <logical name of CUA central system>.

  • Outbound parameter is set for batch processing but no job is scheduled, or authorization problem with the job. 
    Schedule the periodic job RBDAPP01 as per SAP Note 399271 and check the job log for authorization errors.

  • In child system, the return tasks are waiting in the queue in SM58. QOUT Scheduler in SMQS has status RES_LACK. Check KBA 1970757.

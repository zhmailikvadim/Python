Symptom

For processes instantiated from the templates:

  • MDC Template Main: Responsible to Authorized (Responsible View) (DE_B_MDC_MAIN_SEND_RESP_TEMPLATE)
  • MDC Template: Responsible to Authorized (Responsible View) (DE_B_MDC_SEND_RESP_TEMPLATE)

When a BPEM Clarification Case occurs, the system creates a child process document in S/4. However, the corresponding main process document does not contain a link to this child document. The two are related only via the same Additional Business Object (ABO) and not through Linked Documents.

Expected Behavior

  • The main process document should list the newly created child process under Linked Documents, enabling direct navigation.
  • Add the common access reference of the cloud child process document in the 'Additional Header Data' section of the S4 main process document header.
  • The child process document should reference back to the main process (clear parent–child relationship).

Actual Behavior

  • The main and child processes share the same ABO, but Linked Documents are not generated, preventing navigation via document links.

Impact

  • Loss of end-to-end traceability.
  • Users cannot efficiently navigate from the main to the child process; miss the parent–child relationship.

Other Terms

Main process document, Child process document, Linked documents

Reason and Prerequisites

The error occurred because the system lacks a implemented function to link the main and child process documents via Linked Documents.

Solution

Apply the corrections from the relevant SAP Note or import the corresponding support package.

Behavior After the Fix
When a child process document is created in an MDC process, the system automatically links the main process document and the child process document via Linked Documents.

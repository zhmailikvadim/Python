Symptom

User built custom Fiori Client using SAP Mobile Platform SDK 3.0 SP11 and cannot save pdf attachement to iBooks in the screenshot below.

Standard Fiori Client 1.6.3 downloaded from Apple Store does not experience the same issue and works correcly for iOS mobile device in the screenshot below.

Environment

  • SAP Mobile Platform SDK 3.0 SP11
  • Fiori Client 1.6.3
  • iOS mobile device

Reproducing the Issue

  1. Install SAP Mobile Platform SDK 3.0 SP11 environment
  2. Build custom Fiori Cient in accordance with http://scn.sap.com/docs/DOC-56080
  3. Login the Fiori Client and download a PDF attachment
  4. Select the attachment link and open it as PDF document
  5. Press the bottom button and "Save PDF to iBooks" is not visible in the popup screen

Cause

The Kapsel Printer Plugin was missing from the SMP SDK environment and the shared option was not enabled in the browser.

Note: The save button is enabled in the standard version of Fiori client 1.6.3 in the Apple Store.

Resolution

This issue is scheduled to be addressed in SMP SDK 3.0 SP12 . However, this is on a best endeavour basis and not a guarantee of delivery in this release. The target Fix Pack/Server Pack may change without prior notification. The workaround is we can upgrade SMP SDK environment to v3.0 SP11 PL01 because this function is enabled temporarily in this version. However, it has been out of our support area due to this feature is not released in SMP SDK officially.

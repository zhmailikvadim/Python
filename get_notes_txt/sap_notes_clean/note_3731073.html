Symptom

When attempting to edit an installation in ES31, the Time Zone field is greyed out and not editable.

Environment

  • SAP for Utilities as Part of SAP ERP, release IS-UT 600 and higher. 
  • SAP S/4HANA, release IS-UT 801 and higher.

Reproducing the Issue

  1. Open transaction ES31 and display the installation.
  2. Navigate to "Billing/Meter Reading Control" and attempt to change the Time Zone.
  3. Observe that the Time Zone field is greyed out and cannot be edited.

Cause

The installation already has billing documents. By design, the system prevents changing the time zone after billing to avoid inconsistencies (especially for RTP where profile values depend on the installation’s time zone).

Resolution

  1. Verify whether billing documents exist for the installation.
  2. If the installation has not been billed yet, change the time zone in ES31 before any billing documents are created.
  3. If billing documents already exist, acknowledge that the Time Zone field is locked by design and cannot be changed in ES31.
  4. If a correction is still required, perform a controlled data change to field EANL-ETIMEZONE via a report or transaction SE16. Please note, this approach is strongly discouraged due to potential inconsistencies (for example, reversals with rebilling) and must be thoroughly tested in a non-production system with alignment across billing and EDM processes.

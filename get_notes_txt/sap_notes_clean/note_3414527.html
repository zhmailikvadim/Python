Symptom

Solr 9 is officially supported as of 2211.16. However, when the customer is trying to upgrade, the below error emerges during the deployment.

[java] java.lang.reflect.InvocationTargetException
[java]     at java.base/jdk.internal.reflect.NativeMethodAccessorImpl.invoke0(Native Method)
...
[java] Caused by: java.lang.NoClassDefFoundError: org/apache/solr/client/solrj/impl/HttpSolrClient$RemoteSolrException
[java]     at java.base/java.lang.Class.forName0(Native Method)
... 
[java] Caused by: java.lang.ClassNotFoundException: org.apache.solr.client.solrj.impl.HttpSolrClient$RemoteSolrException

Environment

SAP Commerce Cloud 2211

Cause

This issue could be introduced by different root causes:

1. Due to API changes from Solr 8 to 9, as mentioned in Solr Changes in 2211.16:

You may have to modify your custom code if it used the deprecated classes that were removed from the Solr 9 client. This applies even if you're using Solr 8 because the Solr 9 client is used to support both Solr 8 and 9. For more information, refer to the Deprecations and Removals section in the Solr Reference Guide.

2. The issue environment is pinned to an old AppDef (lower than 2310-14) which does not include the changes made to SolrCloudBackup to make it compatible with the Solr 9 client.

Resolution

If the issue is caused by API changes, please adjust your custom code to utiliz the latest API.

If the issue is caused by the pinning of the AppDef of your environment (which officially is not a supported option), please create a support ticket to unpin your system to use the latest AppDef version. Then re-build and re-deploy the system.

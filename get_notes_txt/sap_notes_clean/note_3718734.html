Symptom

When trying to access SSCUI/Configuration Activity "Specify Scheduling Parameters" ID 102813, the following error messages are raised: 

  • No Authorization to start the UI for transaction S_ALR_87007268

Environment

  • SAP Central Business Configuration (CBC)
  • SAP S/4HANA Cloud Public Edition

Reproducing the Issue

  1. Navigate to the CBC workspace.
  2. Click on the "Configuration Activities" tab.
  3. Search for the Configuration Activity ID 102813.
  4. Click on the entry for the activity.
  5. Click the "Go to Activity" button.
  6. Observe the error message: "No Authorization to start the UI for transaction S_ALR_87007268".

Cause

CAUSE 1 - Missing Business Catalog.

or

CAUSE 2 - SSCUI is specialized for certain countries.

Resolution

CAUSE 1 - Missing Business Catalog

The issue occurs because the business catalog SAP_CA_BC_IC_LND_PP_PI0_PC is delivered with scope item BJ8. Therefore, please ensure that scope item BJ8 is activated in your system to enable access to SSCUI 102813. If you require assistance activating the scope item, please refer to KBA 3122430.

Additionally, the business catalog SAP_CA_BC_IC_LND_PP_PI0_PC must be assigned to the BPC_EXPERT business role, or to the corresponding role currently in use.

To allow editing, verify that the "Write Access" field is set to "Unrestricted" for the BPC_EXPERT role, or for the equivalent role in use.

CAUSE 2 - Country Restriction

The SSCUI is available only for the following countries: AE,AR,AT,AU,BE,BG,BR,BW,BZ,CA,CH,CL,CN,CO,CZ,DE,DK,DO,EG,ES,FI,FR,GB,GR,HK,HR,HU,ID,IE,IL,IN,IT,JP,KR,KW,KZ,LU,MX,MY,NA,NL,NO,NZ,OM,PE,PH,PL,PT,QA,RO,RS,RU,SA,SE,SG,SI,SK,TH,TR,TW,UA,US,VN,ZA.

The SSCUI is not available for countries not listed above.

If neither Cause 1 nor Cause 2 resolves the issue, please create a support case under component PP-PI-POR-2CL and provide the affected CBC user, as described in KBA 2838282.

Symptom

While you install/upgrade the delivered MDG Solutions extension software MDG EAM 2024 on S/4 HANA 2025,

Installation Issues : 

Issue 1. Installation ends with an error during DDIC activations .
Error shown as for Example DDL Source /UGI/C_MDQLTYEBOMGENRESULT could not be activated(Package : /UGI/MDQ_DDL, SW-comp : UGI, Appl-Comp: CA-MDG-ADQ)

Post Installation Issues : 

Issue 2. Syntax error in UGIGEO class /UGIGEO/CL_MDG_EAM_GEO_FEEDER and /UGIGEO/CL_MDG_GEF

        Error message : Program /UGIGEO/CL_MDG_EAM_GEO_FEEDER=CP, Include /UGIGEO/CL_MDG_EAM_GEO_FEEDER=CM007: Syntax error in line 000079 The type 'P(8)' of lv_timestamp' is not compatible with type 'UTCLONG' of <'LS_GEOM_DATA>-GEF_D
        Error message : Program /UGIGEO/CL_MDG_GEF=========CP, Include /UGIGEO/CL_MDG_GEF=========CM00D: Syntax error in line 000021 No values passed to the mandatory parameter 'IV_BO_ID'.

Issue 3. Syntax error in UGI7 class /UGI7/EAMWB_MDL_ENTITY_TAB

       Error message : The program /UGI7/EAMWB_MDL_ENTITY_TAB====CP , Include /UGI7/EAMWB_MDL_ENTITY_TAB====CM002 : Syntax error in line 00012. the type " TABLE OF TY_S_UIBB" of 'LT_UIBB' is not compatible with the type 'TABLE_OF_TYPE_S_UIBB' of

Issue 4 :When you are trying to add MDQ services via /IWFND/MAINT_SERVICE t-code post installation,  adding the services ends up with error and you are unable to add the MDQ services.

Other Terms

MDG EAM 2024 on HANA 2025 installation/upgrade errors

Reason and Prerequisites

Due to change in underline DDIC's by SAP as part of  S4 HANA 2025.

Solution

Step1 : Using ADT tool, Perform the manual steps mentioned in the attached document [EAM 2024_on_S4HANA2025_Errors_Resolution_steps.pdf] after SAINT/SUM ends with DDIC activation.

Step 2 : You need to save the relevant changes to TR [ do not activate the objects] as per the project guideline.

Step 3 : Continue with SAINT/SUM process for successful installation/upgrade.

Step 4 : Consider Importing the TR's [ refer to zip file and co/data files -Filename :TRs Files for MDG EAM 2024 on S4HANA 2025-Manual Import Post Installation-Upgrade.zip] into the system after successful installation/upgrade as per your installed component list.
                              UGI/UGIGEO -> Issue 2 -> Import K900129 and R900129 files
                              UGI7-> Issue 3 -> Import K900137 and R900137 files

Step 5 : To address the Issue 4 : Consider importing the TR into the system . To get the relevant co/data files, Please follow the steps mentioned below.

                                           Step 5a: Download all the .7z files[R900150.7z_001->R900150.7z_006 and K900139.rar] into one location file directory on your computer.

                                           Step 5b :Right-click on any of the 7z files that are a part of the collection and click on the "Extract here" or "Extract to folder" option in the pop-up menu. Your 7z application will load up and begin decompressing all the files.

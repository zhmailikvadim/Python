Symptom

Project creation fails with the warning message: "The planned finish is outside the planned time frame of the project."

Environment

SAP S/4HANA Cloud Public Edition

Reproducing the Issue

  1. Navigate to the Project Control - Enterprise Projects app in SAP S/4HANA Cloud Public Edition.
  2. Enter a project ID, and select the option to copy the project.
  3. Enter the project name and set the planned start date.
  4. Proceed with the project creation process.
  5. Observe the warning message: "The planned finish is outside the planned time frame of the project."
  6. Click the "Create" button to bypass the warning message.

Cause

  • SAP S/4HANA Cloud Public Edition enforces an internal validation that restricts project dates to a maximum period of approximately 30 years from the current date.
  • During the copy operation, the system applies date validations that automatically truncate dates exceeding the allowed limit.

Resolution

  1. Adjust the planned finish date of the original project to a more realistic value, such as within the range of approximately 30 years ahead to the maximum, before initiating the copy operation.
  2. Proceed with the project copy operation after ensuring the planned finish date adheres to the system's validation limits.
  3. Note that this is a warning message instead of an error message, it will not prevent you to create or copy a project, once clicking in 'Create' button the system bypass the warning message and project is created.

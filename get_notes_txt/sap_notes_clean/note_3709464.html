Symptom

iFlows are failing in Cloud Integration due to connectivity issues with the SFTP server, error received is:

org.apache.camel.component.file.GenericFileOperationFailedException: Cannot connect to sap-sftp://xxxxxxxx:22, cause: java.io.IOException: End of IO Stream Read 

Environment

  • Cloud Integration
  • SAP Integration Suite

Reproducing the Issue

  • Do SSH Connectivity Test from Cloud Integration to SFTP server and/or
  • Run iFlow with SFTP adapter connecting to a particular SFTP server

Cause

SSH initialization is failing due to the absence of a shared cipher suite.  

Resolution

Ensure that the server-side configuration includes a compatible cipher suite for SSH communication. 

Cloud Integration currently supports the following ciphers for SSH (SFTP) communication:

blowfish-cbc
3des-cbc
aes128-cbc
aes192-cbc
aes256-cbc
aes128-ctr
aes192-ctr
aes256-ctr
3des-ctr
arcfour
arcfour128
arcfour256

***Note*** As per SAP Help: SFTP Adapter "Please note that the ciphers listed above can change in the future. New ciphers can be added and existing ones can be removed in case of security weaknesses. In such cases, you will have to change the ciphers on the SFTP server and reconfigure the integration flows that contain SFTP adapter. SAP will inform customers, if necessary"

See also KBA 3698079: Cipher Suite Supported by SFTP adapter in SAP Cloud Integration (CPI)

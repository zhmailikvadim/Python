Symptom

You have used a search string or a user exit to fill the BDC fields in FEBEP.
You use the BDC account type (FKOA*) to determine in which posting area and in which line the field (FNAM*) is to be filled with the value (FVAL*).

If you now post the item, the corresponding fields are not filled in the generated clearing item or in the line on account, even though it is BDC Account Type.

Other Terms

FEBEP-FNAM1, FEBEP-FVAL1, FEBEP-FKOA1, FEBEP-FNAM2, FEBEP-FVAL2, FEBEP-FKOA2, FEBEP-FNAM3, FEBEP-FVAL3, FEBEP-FKOA3 
FB05

Reason and Prerequisites

It is a clearing transaction in the relevant posting area (posting indicator in the posting rule is 4,5,7 or 8).  In this case, the BDC fields cannot be transferred to the clearing item.
Technically, this assignment is not possible because transaction FB05 is called during clearing and the clearing item is therefore not filled by the account statement. No BDC field movement is supported for the clearing line item itself.

If no clearing takes place, but a posting on account takes place, the system response is the same: In the line item on account, the BDC fields are not filled.

Solution

There are the following alternatives for changing the line items of a line item on account:
- Use the Business Add-In (BADI) with the definition name FEB_BADI  (s. SAP Note 494777, point 4) to specify corresponding values in the table FTPOST.

When trying to clone a project from there corporate GitHub repository, the developer is receiving an error message and the pull request fails. 

Environment

  • SAP Business Application Studio.
  • GitHub.

Reproducing the Issue

  1. From the Business Application Studio getting started page.
  2. Select clone from Git.
  3. User logs in with their Auth Token.
  4. Select the project to clone.
  5. Error is then thrown LC_All: Cannot change location.

 

 

Cause

  • During the personal token creation, the Customer account had to be authorized for Users id.
  • The Users was using personnel Auth Token not associated with the corporate Git Authorization.  
  • Use a terminal command git clone https://<USERNAME>:<Auth_TOKEN>@github.com/Customers_Name_Services/<REPO>.git

Resolution

  • Ensure the User ID is associated with the Companies privet GitHub.
  • Ensue the User ID is using an authorized token associated to the company's GitHub.

Keywords

BAS, GitHub, SAP Business Application Studio

Symptom

Requirement is to configure the JMS Sender channel using EOIO Quality of service.
While processing the messages few of them get processed and few got stuck and not reached the PO system.

Environment

  • SAP NetWeaver 7.50
  • SAP Process Integration

Cause

The issue is that the SAP JMS Provider proactively pushes messages to consumers. Messages are then retrieved by the consumers when receive() method is called. This behavior—pushing messages to the consumer—is unusual and unexpected and all other JMS providers do not operate this way. In the case of EOIO, a consumer is created per server node; however, the message polling job is executed on only one server node, and only the consumer on that node is checked for messages. The remaining consumers—or, the other server node—will still hold messages, which will not be read until the channel is stopped.

Resolution

To resolve this issue you can use the below 2 workarounds,

  1. You should use the additional property "Parameter Name: UseOldClusterLock, Parameter Value: true"
    --> Go to advanced mode--> additional parameters and set the above parameter to true.
  2. or the other option is using Message Listener instead of Message Poll Job

See Also

SAP Note: 1595929 - JMS Sender channel EOIO QoS fails with Message Listener
SAP Help page: Configuring the Sender JMS Adapter

EOIO, JMS, Message Listener, Clusterlock

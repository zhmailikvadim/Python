Symptom

In the process "Sending of Non-Settlement-Relevant Change from Responsible SUP to DSO as SUP (11109/55109/44109)" (DE_B_MDC_SUP_N_SETTL_REL_RSP_SUP_DSO), step 0200 doesn't apply changes made through the BAdI /US4G/BADI_MDC_PROCESS. When you implement the BAdI to add missing scenarios to the process step data, the system doesn't update the step data with these changes.

This issue is relevant for the market role: SUP.

Other Terms

BAdI, /US4G/BADI_MDC_PROCESS, /US4G/CL_CHK_MDC_PROCESS, CHECK_DATA_COMPL_FOR_SENDING, UBD data, step data, PDoc.

Reason and Prerequisites

The error occurred because the check method /US4G/CL_CHK_MDC_PROCESS-CHECK_DATA_COMPL_FOR_SENDING didn't include logic to update the step data after calling the BAdI method.

Solution

Implement the corrections of this SAP Note or import the relevant support package.

After implementing this SAP Note, the system saves the updated UBD data after running the check method /US4G/CL_CHK_MDC_PROCESS-CHECK_DATA_COMPL_FOR_SENDING when you update the step UBD data through the BAdI /US4G/BADI_MDC_PROCESS. You need to re-run step 0200 and the following steps in the process DE_B_MDC_SUP_N_SETTL_REL_RSP_SUP_DSO. You can verify the fix by checking that the changed UBD data appears in the step 0200 additional data in the PDoc of the process DE_B_MDC_SUP_N_SETTL_REL_RSP_SUP_DSO.

Symptom

STO inbound delivery goods receipt posting queue stuck with error "Deficit of PL Stock in transit prev. &1 &2" (Message No. M7021).

Environment

  • SAP Extended Warehouse Management (SCM-EWM)
  • SAP ERP Central Component (ECC)
  • SAP Extended Warehouse Management Interface (SCM-EWM-IF)
  • Interface ERP Delivery Processing

Reproducing the Issue

  1. Go to VL33N and check the GM status of the delivery.
  2. Go to /SCWM/PRDI and check the GR status of the delivery.
  3. Go to SMQ2 and check the queue.

Cause

Queue is trying to post Goods Receipt in a previous period for which stock does not exist.

Resolution

  1. Post the GR with the current date.
  2. Access transaction SU01.
  3. Assign parameter 'DLV_POSTING_DATE' = 'X' (Uppercase).
  4. 'X' will update the posting date to the current one.

Symptom

  • When an administrator attempts to configure the destination (URL/target page) for a UI5 card in the SAP Work Zone Admin Console, mouse clicks on the destination field are ignored. Keyboard navigation (select the field and use the up/down arrows) still works, indicating that the underlying functionality is intact but the mouse event handling is broken. 

Environment

  • SAP WorkZone Advanced Edition
  • SAP SuccessFactors Workzone

Reproducing the Issue

  1. Log in to SAP Work Zone as a Global Administrator.
  2. Navigate to Admin Console → Content → Cards.
  3. Open an existing card or create a new one.
  4. In the card preview, locate the Destination section.
  5. Click the drop-down arrow to try and select a new destination with your mouse.
  6. Notice that the selection clicked on with the mouse does not capture the update.

Cause

The issue was caused by a defective version (2.10.0) of UI5 Web Components.

Resolution

Fix will be set on 1H2026.10 (2603 SalesDemo) 

See Also

Symptom

During the migration process, on the end transaction step, the system tries saving the utilities sales contracts.

In case of the migration with historical data using class CL_CRMS4_IU_CM_PFP_API_MIGCREH the contracts have already been saved.

CL_CRMS4_IU_CM_PFP_API_MIGCREA relies on the use of attribute FCONTRACT to save the contracts, which is empty and would results in a shortdump.

Other Terms

CL_CRMS4_IU_CM_PFP_API_MIGCREH, END_TRANSACTION_SPEC

Solution

Please implement the correction instructions or install the corresponding support package.

Symptom

There are two records being created in the Job Information of the users upon updating a position in Edit mode in the Position org Chart:

  • Fields in the position are changed and the Higher level Position is changed.
  • Just the Higher-level Position is changed

Environment

SAP SuccessFactors Employee Central: Position Management

Resolution

The below details the expected behavior for both scenarios:

Change field defined in rule and higher-level position:

  • One record is created for the changes to the position field attributes which are synced via the Position to Job Information Sync Rule.
  • The second record is to sync the manager change. The manager change are synced from the change to the Higher-Level Position on the position is not carried out by the Position to Job Information Sync Rule.
  • This action is carried out by a background job which will sync the manager, this requires a second record to be created.

Change just Higher-level position:

  • One record will be created for the manager change.
  • When this manager change record, is created it will validate the job information data against the corresponding position fields defined in the business rule. If there is a discrepancy in the data, the system will create a second record to update the job information fields to have the same values as the position fields. 
  • When the manager change record is created, if there is any onSave business rules which will change any field values, the system will create a second record to update these field values. 
  • If there is both a discrepancy in the data and any onSave rules to update data, both of these actions will be carried out in the second record created for field value changes.

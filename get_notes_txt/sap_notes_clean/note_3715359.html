  • Syntax error thrown when using CamelJdbcAdapter and issuing SQL commands that include functions.
  • The syntax error may vary depending on the source database, but can be similar to the following:

Environment

SAP HANA smart data integration 2.0

Reproducing the Issue

  1. Use CamelJdbcAdapter to create the remote source.
  2. Execute a query including SQL functions.
  3. Observe the syntax error and query failure in the logs.

Cause

The SQL function rewrite in the CamelJdbcAdapter does not fit the requirements of the source database. The rewrite logic needs to be adjusted to match the source database syntax.

Resolution

  1. Open and edit the <DPA_root>/camel/samples/sample-jdbc-dialect.xml file. This file contains the data type and function rewrites of the CamelJdbcAdapter. NOTE: Changing anything here will apply globally to all remote sources using the CamelJdbcAdapter.
  2. Replace the function rewrites with a syntax that is accepted by the source database.
  3. Restart the agent to apply the changes.
  4. Test the query to confirm resolution.

Below are a few examples.

  • Google BigQuery SQL does not accept the :: cast operator and uses INT64 for all integer sizes and FLOAT64 for real/double; it uses NUMERIC/BIGNUMERIC for decimal values. See KBA 3716041 - [CamelJdbcAdapter] Syntax errors when connecting to Google BigQuery - SAP HANA Smart Data Integration for further details.
    • A well-known issue is the various ways in which different databases treat TOP/LIMIT clauses. See the following documentation:
      • KBA 3490916 - Error "Syntax error at or near '100'" while previewing data through VT table - SAP HANA Smart Data Integration
      • KBA 3566022 - [CamelJDBCAdapter] "DB2 SQL Error: SQLCODE=-104, SQLSTATE=42601, SQLERRMC" when previewing Remote Table from DB2 in Datasphere - SAP HANA Smart Data Integration
      • KBA 3406001 - Cannot preview an imported remote table with SQL syntax error - SAP HANA Smart Data Integration
      • SAP Note 3440425 - [CamelJdbcAdapter] Source system doesn't support TOP in the query SQL
      • SAP Note 3131458 - CamelInformixAdapter does not support TOP operator on the virtual table

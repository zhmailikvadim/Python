Symptom

When using the Import Allocation Data app (F5596), the following behavior is observed:

1. Open the app and click Go.
2. Select a file name and click Download Template.
3. Choose Download Template with Values.
4. Select one or more allocation cycles, then click Cancel.
5. Navigate to a different file name.

Result: The allocation cycle selection from the previous file name is still visible/retained — the UI state is not reset when navigating to a new file.

Expected Result: Upon navigating to a different file name, the allocation cycle selection should be cleared and no previously selected values should be displayed.

Other Terms

universal allocations, UA, F5596, import allocation data - values

Reason and Prerequisites

Program error.

Solution

Apply the attached correction or implement the respective support package.

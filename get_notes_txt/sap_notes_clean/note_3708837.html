Symptom

  • In custom MDF object field defined as User data type.
  • Data is missing for this particular field in the Story report.

Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.

Environment

  • Data Modelling - Module & Entity Data Issues

Reproducing the Issue

  1. Create a new Story report.
  2. Select fields from the User Sub-object within the affected Custom MDF object.
  3. Click on Preview Query.
  4. No data is displayed.

Cause

The custom field is of data type User, and through a rule, a string field is passed into it.

Resolution

In the configuration, a business rule was created where a String field was passed into a field of Data type User, resulting in a data issue in the Story report

Change the data type of Custom field from User data type to String.

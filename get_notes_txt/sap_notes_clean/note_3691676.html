Symptom

An application uses SALV OM to display data using a custom GUI which includes added functions. The application obtains a reference to the FUNCTIONS object prior to setting the custom GUI with method TABLE-> SET_SCREEN STATUS and uses the method FUNCTIONS-> SET_VISIBLE( ) to set some of the added functions invisible.

When the data is first displayed, the GUI shows all added functions, including those that were set invisible.

After a user chooses one of the added functions from the GUI, the system resets and the visibility of the added functions is set as expected.

Other Terms

ALV; ABAP List Viewer; SAP List Viewer; Grid; List; SALV OM

Reason and Prerequisites

Reason

When a reference to the FUNCTIONS object is first requested, the function codes of the GUI are retrieved and stored as an internal list. If this is done before method TABLE-> SET_SCREEN STATUS is called, the function codes of the standard GUI are retrieved which will not include functions added to the custom GUI. When method TABLE-> SET_SCREEN STATUS is called, the internal list of functions is not refreshed, so any attempt to set the visibility of an added function will fail and all added functions will be visible.

When a user chooses one of the added functions, the function is not recognised and SALV re-builds the internal list based on the custom GUI that was set by method TABLE-> SET_SCREEN STATUS. From this time the visibility of the added functions is set as expected.

Prerequisites

The following Support Package level is the minimum prerequisite:

Release Support Package
SAP_BASIS 816 SP 00
SAP_BASIS 758 SP 02

When you install an SAP Note, some prerequisites might be required. The prerequisites are SAP Notes that have already been released and that change the same context of the current one. Then, even if these prerequisites are not necessarily related to the symptom of this SAP Note, you must implement them due to technical reasons.

Solution

From now on, the system will re-build the internal function list whenever method TABLE->SET_SCREEN STATUS is called to change the active GUI.

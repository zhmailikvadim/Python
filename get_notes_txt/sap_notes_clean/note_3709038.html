Symptom

  • Cloud ALM always displays a minimum of 3 work processes in the metric 'Workprocesses On Hold'
  • No work processes or a lower number are on hold in the managed system(transaction SM50).

Environment

SAP Cloud ALM

Reproducing the Issue

  1. Access Cloud ALM.
  2. Navigate to Health Monitoring.
  3. Observe that the system displays a certain work processes on hold.
  4. Check the managed system (transaction SM50) and confirm no work processes are on hold or there is a lower value.

Cause

  • The behavior is expected and is not related to system load or performance issues.

Resolution

Some of the Work Processes shown as “On Hold” are triggered by SAP Cloud ALM Health Monitoring itself.

SAP Cloud ALM Health Monitoring collects a wide range of technical metrics across ABAP, SAP HANA, and the operating system.
To ensure efficient and timely data collection, the framework uses parallelized data collectors.

While these collectors are running, the affected work processes may temporarily switch to the status “On Hold” until the respective data collection step is completed.
Since SAP Cloud ALM Health Monitoring captures a snapshot of the current work process state, these internal monitoring activities are also included in the reported data.
As a result, the monitoring collectors can slightly influence the work process status they report.

Although some process-related metrics could theoretically be shifted to the SAP Host Agent to reduce involvement of the ABAP stack, the remaining collectors would still lead to a comparable number of work processes being in “On Hold” status.
We continue to evaluate possible optimizations; however, the system is currently functioning as designed.

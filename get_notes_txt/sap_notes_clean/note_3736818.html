Symptom

  • ABAP Test Cockpit Check shows "Execution status" Completed, but "Check Result" shows empty in SAP Cloud ALM Feature "Transport Checks" area.
  • Error in the /SDF/CALM_CDM_TR_SUB_CHECK job: 
    • Exception code 6 raised from /SDF/TEAP_ENVI_ANA during transport check process. Message No. /SDF/CALM_CDM_MSG099
  • Error "Exception code 6 raised from during ATC check. Empty transport request detected" in log of job /SDF/CALM_CDM_EXPORT_SUB_CHECK when doing Perform ATC Check from Feature

Environment

SAP Cloud ALM 

Reproducing the Issue

  1. Logon to SAP Cloud ALM.
  2. Select the Features app.
  3. Open a Feature with some Transport Requests that require checks.
  4. Scroll to the Transport Checks area.
  5. Select Export tab.
  6. Click in "Perform ATC checks" if at least there is a transport is status Modifiable
    Note: The ABAP Test Cockpit (ATC) is a tool for doing static and dynamic quality checking of ABAP code and associated repository objects, it is run in the Transport Organizer during development, initially in smaller development units like an ABAP class and then in bigger development units, like a package or task or transport request, to find and clean up defects.
  7. ABAP Test Cockpit Check shows Scheduled under "Execution status" column.
  8. In the indicated development system:client check in sm37 for the periodic job /SDF/CALM_CDM_EXPORT_CHECK that triggers job /SDF/CALM_CDM_EXPORT_SUB_CHECK when there is a Perform ATC checks run at Feature level

Resolution

Prerequisites for Export Checks

Ensure that the configuration is completed in your development system:client where the workbench transports are being created as per the instructions in the SAP Help portal Transport Checks - Prerequisites for Export Checks:

  • You've installed ST-PI 740 SP 31 or ST-PI 740 SP 30 with SAP Note 3569187, 3569295. For ST-PI 740 SP 31 you've also installed SAP Note 3621761
  • SAP Note 3656149 - SAP Cloud ALM - CDM: Common issue fix for Export Check is implemented
  • Your SAP_BASIS version is at least 740 SP13.
  • You've configured the ABAP Test Cockpit (ATC) . For more information, see Quality Checking with the ABAP Test Cockpit (ATC). Run transaction ATC, then select Basic Settings:
  • Note: You've set the Behavior on Release of the Code Inspector to Disable "Code Inspector" as test driver. 

  • In Global Customizing, you can set whether the ATC check should be performed at Task or Workbench Request release. To change the Global Customizing, run transaction SE03 and select Global Customizing (Transport Organizer):

  • In your development system DEV:XXX you've activated use case task Transports: Export Checks (client-specific) on development system all clients XXX on which you want to perform an export check, where XXX is the client where the transport is created.
  • You have the Project Admin, Project Lead, Project Member, or Developer role.
  • At least one CTS-managed workbench transport in status Modifiable is assigned to your feature. Customizing transports aren't supported.
  • Note: If you're not able to perform transport checks in the Features app, check whether the diagnostics job /SDF/CALM_CDM_DIAGNOSTICS has already been performed today. If not, you can either wait for the next run or repeat scheduling immediately of the last finished run with transaction SE38.
  • Note: Please check SAP Note 2270689 - Remote Analysis (for source system), ensure that the corresponding Note for your source system release is implemented and manual post-implementation activities done (execute report RS_ABAP_SETUP_ANALYSIS). For example, SAP Note 3373034 - Remote Analysis (for source system. Release 7.58 New) and then run report RS_ABAP_SETUP_ANALYSIS in development system.

Specific Errors for Export checks

  1. Error in job /SDF/CALM_CDM_EXPORT_SUB_CHECK: "Object checks options configured in SE03 are not supported for ATC check" (/SDF/CALM_CDM_MSG125)
    • Refer to KBA 3633554 - Perform ATC Check from Feature "Transport Checks - Export" fails with "Exception Raised from ST-PI backend function module..." 
  2. Error in job /SDF/CALM_CDM_EXPORT_SUB_CHECK: "Read configuration from TR_READ_CHECK_SEVERITY: ATC check is switched off" (/SDF/CALM_CDM_MSG126)
    • Refer to KBA 3633554 - Perform ATC Check from Feature "Transport Checks - Export" fails with "Exception Raised from ST-PI backend function module..." 
  3. Error in job /SDF/CALM_CDM_EXPORT_SUB_CHECK: Job RFC(XXXXX-YYYYYY):<timestamp>: Client process aborted with RFC system error Function module "RS_ABAP_GET_INTFS_INCL_INTF_E" SATC_RUN_TIME303
    • Refer to SAP Note 2270689 - Remote Analysis (for source system), ensure that the corresponding Note for your source system release is implemented and report RS_ABAP_SETUP_ANALYSIS is executed (manual post-implementation tasks).
  4. Error in job /SDF/CALM_CDM_EXPORT_SUB_CHECK: "Exception code 6 raised from during ATC check. Empty transport request detected" 
    • Please make sure that Note 3656149 - SAP Cloud ALM - CDM:Common issue fix for Export Check is implemented in development system.

Transport check - Execution Status:

  1. If "Execution status" shows Completed and "Check Result" shows:
    • A green icon means no conflicts found. 
    • If your result shows a warning, yellow icon, or critical error, red icon, you can select the check to navigate to the Transport Checks app to get more details.
  1. If "Execution status" shows Scheduled means that job /SDF/CALM_CDM_EXPORT_CHECK is still running in DEV:XXX. Note that it is not possible to perform a check again as long as it is in status Scheduled.
  2. If "Execution status" shows Failed if job /SDF/CALM_CDM_EXPORT_CHECK is running for long time.
  3. If "Execution status" shows Canceled if the connection from the managed system to the SAP Cloud ALM tenant is broken so that the job hasn't been picked up by the managed system within 8 hours.

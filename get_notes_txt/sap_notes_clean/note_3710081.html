Symptom

  • Not able to create settlement rules for non billing WBS Elements in "Project Planning" app post the CE2602 upgrade.
  • Previously created settlement rules for non billing WBS elements cannot be found in the "Project Planning" app post the CE2602 upgrade.
  • This is only observed with enterprise projects that have a project profile of 'Project with Revenue'.

Environment

SAP S/4HANA Cloud Public Edition CE2602 and later

Reproducing the Issue

  1. Open a project has project profile of 'Project with Revenue' in the app "Project Control - Enterprise Projects".
  2. Navigate to a a WBS element which is not a billing element.
  3. Notice that it is not possible to view/create settlement rules.

Cause

Post the CE2602 release the system design will not allow to create settlement rules for non billing WBS elements in the "Project Planning" app.

Resolution

As a workaround:

  • To display existing settlement rules for non billing WBS elements: The app 'Manage Settlement Rules - Projects' can be used.
  • To create settlement rules for non billing elements: This has not been allowed since CE2602. However, our development team is considering enhancing the profitability segment attributes in a future release.

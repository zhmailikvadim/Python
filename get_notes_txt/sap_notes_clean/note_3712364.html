Symptom

When you execute the report ELStAM Procedure: Transfer XML Data to ELStAM Table - Distributed Reporting  (RPCE2SD0_IN_VR), the system issues the following error message:

"No pers. no. for ID no. <IDNR> and ER tax no. <AGSTNR> on <KEY DATE>"  (HRPAYDEE2 408).

Reason and Prerequisites

  • You have not selected the Assignment Attempt in All RFCs  checkbox on the selection screen. 
  • There are several messages for the identification number in different systems/clients.
  • For one of these systems/clients, you have not maintained the RFC destination on the selection screen. This could be because it is an obsolete system or client.

Solution

Implement the attached correction instructions or import the specified support package.

After you implement the corrections, the assignment is correct again and takes into account only the RFC destinations maintained on the selection screen.

You can use the report Manage ELStAM Data (RPCE2ZD0) to assign all unassigned basic ELStAM  notifications.

You can use RPCE2SD0_IN_VR with the checkbox Reassign ELStAM PHI  to assign all unassigned PHI notifications.

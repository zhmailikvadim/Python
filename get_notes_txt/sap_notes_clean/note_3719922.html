Symptom

In the process "Calculation of Energy Amounts (MOS MaLo)" (DE_B_MET_ENERGY_AMOUNT_CALC_MOSMALO), step 0055 "Deadline for Sending Energy Amount with User Decision (DECI)" created a BPEM case with category WM15. After you selected "Execute the solution," the BPEM case remained in "In Process" status.

This issue is relevant for the market roles: MOS

Other Terms

MET, BM_AGGR_ENERGY, WM15, /US4G/CL_EXCP_PROC_SOLVE_WIM, energy amount calculation, deadline, user decision, DECI, DACT, check result, external exception code

Reason and Prerequisites

The error occurred because the exception configuration was incorrect.

Solution

Implement the corrections of this SAP Note or import the relevant support package.

After implementing this SAP Note, step 0055 creates a BPEM case with the correct external exception code and check result.

For existing BPEM cases in step 0055, select "Re-process the current process step." The system reverses the old step 0055 and generates a new step 0055 with "Deadline Active (DACT)" status. When you run the process, the new step 0055 displays "User Decision" status. The system confirms the old BPEM case and generates a new BPEM case with two solutions: "Cancel the process document" and "Re-process the current process step." The new BPEM case includes check results EAC_MR_VALUES_COMPLETED or EAC_MR_VALUES_UNCOMPLETED.

You can verify the fix by checking that after you select "Re-process the current process step" for existing BPEM cases in step 0055, the system reverses the old step 0055, generates a new step 0055 with "Deadline Active (DACT)" status, and after running the process, the new step 0055 displays "User Decision" status. Verify that the system confirms the old BPEM case and generates a new BPEM case with two solutions and the appropriate check results.

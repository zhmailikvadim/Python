Symptom

After adjusting the partition granularity for ADSO with dynamic partitioning, a transport is created to carry the changes to the other systems, but it fails with the following error:

Remodeling is not possible
Part.Granularity 0CALYEAR to 0CALWEEK not allowed if some part. already moved to cold
Error when activating DataStore-Object &1

If the ADSO has "Deleted" partitions, the error will also be triggered, as "Deleted" partitions are interpreted as being in Cold Store. 

Environment

SAP BW/4HANA

Cause

  • Change in the partition granularity for dynamic partitioning is not allowed when some of the partitions have been already moved to the cold store. 
  • This check was introduced with SAP Note 3550165 - Change in partition granularity for Dynamic partitioning not allowed when partitions moved to cold store

Resolution

If Deleted partition exists, clean up the partition using the Data Tiering Maintenance app or process chain type Cleanup Archive. After that, the Deleted partition should not be displayed anymore in the DTO temperature administration. Check the following link for further details Deleting Data.

For Deleted and Cold partitions, another option is to change the partition temperature from Deleted/Cold to Warm by following below steps:

  1. Go to Data Tiering Maintenance (transaction RSOADSODTO);
  2. Change the planned temperature from "Deleted"/"Cold" to "Warm";
  3. Click on "Execute" button and execute the change with "Repair and Adjust" option.

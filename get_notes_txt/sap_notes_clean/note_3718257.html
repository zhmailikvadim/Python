Symptom

The error message "A partner breakdown is missing for item $110001" appears during the execution of the Consolidation of Investments task in update run mode.

The error is associated with statistical FS items in Consolidation of Investments.

Environment

SAP S/4HANA Cloud Public Edition

Reproducing the Issue

  1. open the Consolidation Monitor app.
  2. Execute task 2101 Consolidation of Investments (activity-based) in update run mode.
  3. Observe the error message "A partner breakdown is missing for item &&&" appearing multiple times for statistical FS items.

Cause

Breakdown categories for statistical FS items do not include partner unit subassignment, leading to the error message "A partner breakdown is missing for item $110001".

Resolution

CoI check was explicitly done in transaction CX6C3 for private customers and was recommended to execute the check before each COI execution. 

As of 2602  version, system implemented a part of CoI check during the CoI task execution automatically.

The configuration for generated statistical items will be checked during the execution.

Such inconsistencies in the stat items might have no immediate impact on CoI but for some CoI activities when the values saved in the statistical items are used e.g. total divestiture, indirect increase/decrease etc it will impact the result and in many times it is too late to fix the data in previous years where periods are closed.

Currently the customer must open an incident and SAP will execute the cleanup report and see any inconsistencies in previous years if additional correction is needed.

The check also gives other errors which can be corrected by end users. Only for generated statistical items and generated breakdown categories are done by the cleanup report.

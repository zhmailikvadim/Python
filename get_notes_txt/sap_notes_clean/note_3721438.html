Symptom

You create a posting run where subsequent retroactive accountings occur in periods for which the posting runs concerned have already been archived, the system does not read the related index results (P-Reslts from PPOIX/PPOPX) from the archive correctly and therfreo does not reverse the previous entries.

Other Terms

RPCIPE00, PPOIX, PPOPX, PAR6, PARC, LPAR6F01, perform in_loop, delete buffer, gt_pcalac_buf, runid

Reason and Prerequisites

The buffer is not corretly cleared and the runid belongs to former Archive handle.

Solution

Please implement the assigned correction instruction or apply the corresponding HR Support Package.

Symptom

The function module G_GLU1_ITEMIZE_AMOUNTS splits table lines with very high amounts into multiple lines with a maximum amount of 9999999999999 (thirteen digits).

Other Terms

G_GLU1_ITEMIZE_AMOUNTS , TSV_TNEW_PAGE_ALLOC_FAILED , memory overflow , GLU1

Solution

After applying this note, the function module G_GLU1_ITEMIZE_AMOUNTS has a new optional parameter called "max_value" Type P.

If you supply this parameter, you can change the maximum amount for the split. Please note that the parameter can't be smaller than '99999999.99' (ten digits).

 

However, if you choose a small maximum amount, it is very likely that you will face the following dump:

Category Resource Shortage
Runtime Errors TSV_TNEW_PAGE_ALLOC_FAILED

Short Text
No more memory available to add rows to an internal table.

In this case, you need to increase the parameter "max_value".

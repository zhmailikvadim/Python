Symptom

Users attempting self-registration in SAP Cloud Identity Services receive a success message stating
   "An email with a link to activate your account has been sent to..."
but no activation email is delivered when an account with that email address already exists in the system.

Environment

SAP Cloud Identity Services

Reproducing the Issue

Prerequisites:

  • SAP Cloud Identity Services is configured with 'Public' User Application Access option enabled for self-registration
  • User has previously completed self-registration and has an active account in the system

Scenario 1: Self-registration attempt for the same application

  1. User navigates to the application login page
  2. User clicks on the self-registration option
  3. User fills out the registration form with an email address that already exists in the system
  4. User submits the form
  5. System displays:
       "An email with a link to activate your account has been sent to..."
  6. No activation email is received

Scenario 2: Self-registration attempt for a different application

  1. User navigates to a different application that uses the same Identity Authentication service
  2. User is unaware that both applications share the same identity authenticator
  3. User attempts self-registration using the same email address
  4. User fills out the registration form
  5. User submits the form
  6. System displays:
       "An email with a link to activate your account has been sent to..."
  7. No activation email is received

Note: If the email address is unknown to SAP Cloud Identity Services, the self-registration process works as expected, including the activation email delivery and subsequent login steps.

Cause

The system is designed not to notify visitors if an account with the provided email already exists for security reasons.

Resolution

  • The system is intentionally designed to prevent sending activation emails if an account with the same email address already exists. This is a security measure to avoid unauthorized notifications.
  • Users are recommended to initiate the "Forgot Password" process to verify whether an account already exists and reset the password if necessary.
  • To improve user experience while maintaining security best practices, the success message displayed after a self-registration attempt can be customized.
    This can be done in the Administration Console under the key `registerSuccess.emailSent`.
    Detailed instructions for changing tenant texts can be found in the documentation at Change Tenant Texts via Administration Console.
  • Below are some examples of alternative messages one can consider during customization (using a placeholder {0} for the email address):
    • Option 1 (Conditional):
      "If this email address is not already registered, an activation link has been sent to {0}. Please check your inbox. If you don’t receive an email within a few minutes, you may already have an account. Try using 'Forgot Password' instead."
    • Option 2 (Balanced):
      "An activation email has been sent to {0} if this address is eligible for registration. Note: If an account already exists with this email, no email will be sent. Please use 'Forgot Password' to access your existing account."
    • Option 3 (Action-Oriented):
      "Check your inbox at {0} for an activation link. Didn’t receive an email after 5 minutes? You may already have an account—click 'Forgot Password' to recover access."

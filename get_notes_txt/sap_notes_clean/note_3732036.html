Environment

SAP Data Services 4.x

Reproducing the Issue

  1. Start the Data Services server configuration utility (./svrcfg).
  2. Choose option 2.
  3. Choose option r to remove a bound repository from the job server.
  4. The process fails with CON-120307 and "general error;10 authentication failed".

Cause

  • The bound repository schema had not been upgraded to Data Services 4.3, leading to connection/update failures when the 4.3 server attempted to access it during removal.
  • The repository user is locked on Database side.
  • The repository user credentials were changed on the database side but were not updated in Data Services via svrcfg.

Resolution

  1. Verify repository version:
  2. Use Repository Manager or command line to check the repository version (for example, repoman -v).
  3. If the version is lower than 4.3, upgrade the repository to 4.3.
  4. Update stored repository credentials in the job server before removal:
    1. Source the DS environment (for example, ". ./al_env.sh" in $LINK_DIR/bin).
    2. Run ./svrcfg > 2 > u, and enter the current database password for the repository.
  5. Remove the repository binding:
          Run ./svrcfg > 2 > r, choose the target repository.
  6. Check and make sure the database user is not locked during the whole process.

Workaround:
If removal still fails and the repository is no longer in use, perform a controlled manual unbind:

  1. Back up DSConfig.txt.
  2. Remove the section corresponding to the target repository from DSConfig.txt and renumber subsequent [JobServer_x@@n] entries sequentially.
  3. Save the file and restart the Data Services services.
  4. Retest and confirm the job server no longer lists the removed repository.

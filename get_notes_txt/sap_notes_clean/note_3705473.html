Symptom

  • Mobile number entered via 'add candidate', 'employee referral', or the career site CDC widget is not populated on candidate profiles in the new candidate experience.
  • The issue occurs when recruiters add candidates via 'Add Candidate' functionality, where other fields like first name, last name, email, and country are populated, but the mobile number is missing.

Environment

SAP SuccessFactors Recruiting Management

Reproducing the Issue

  1. Login as Employee and navigate to Careers – Refer a friend.
  2. Enter required information and click on Submit.
  3. Login as the candidate and review Profile Information; the mobile number (cellPhone) is missing.
  4. Recruiter creates a candidate profile via 'Add Candidate' or other methods like 'Employee Referral'.
  5. Candidate resets their password and logs into the career site for the first time.
  6. CDC widget appears and prompts the candidate to complete the phone number field.
  7. Candidate provides the phone number and continues login to access their candidate profile.
  8. Upon accessing the candidate profile, the mobile number field remains empty, and the previously entered value is removed.

Cause

The issue is due to the phone country code missing in the information shared during the referral process. The CDC invalidates the number and stores nothing, as it expects the phone number along with the country code.

Resolution

Engineering team has identified the root cause and developed a code fix for the issue. The fix will be implemented as part of the 1H 2026 Release.

  • Preview release date: April 13th, 2026
  • Production release date: May 15th - 16th, 2026

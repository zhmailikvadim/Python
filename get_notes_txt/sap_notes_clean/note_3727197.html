Symptom

When an Adv. Shipping and Receiving notification received to update a TM document like a consigment order to change for example the purchase order number for a product or change the external delivery note number, the corresponding freight units were not unassigend or assigend correctly. This issue happens in cases with nested package hierarchie and for the top level package a package id was maintained in the Adv. Shipping and Receiving notification. After updating the TM document, the top level package still refered to the unassigend freight unit and in the corresponding TM document no delivery creation was possible anymore.

Other Terms

Adv. Shipping and Receiving notification, Consigment order, ASN message, Change External Delivery Note Number, Change Purchase Document

Reason and Prerequisites

Program error

Solution

Implement the attached correction instruction or install the corresponding Support Package.

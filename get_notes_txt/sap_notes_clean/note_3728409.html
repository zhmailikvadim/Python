Symptom

The transport of process chain variants of type "Remote Process Chain" terminates with error: 

Start of the after-import method RS_RSPC_AFTER_IMPORT for object type(s) RSPC ( )

Check results for process REMOTE ILV_&1 in chain &2:
Error in remote call of destination &3: RFC destination &3 does not exist.

A warning message with further information can be found in the transport log (transaction SE01):

Remote chain ILV_&1: Destination &3 does not exist. Please maintain.
Message No. RSPC073

Diagnosis
In the definition of the remote process chain ILV_&1, destination

&3, in which process chain &2 is supposed to be executed, has been entered.

However, this destination does not exist.

System Response
The function will continue to run.

Procedure
Either create destination &3 or maintain variant ILV_&1 manually and enter a different destination.

Make sure that process chain &2 and destination &4 exist in the target system for this destination.

Environment

  • SAP BW/4HANA
  • SAP NetWeaver 7.x

Cause

Even if RSLOGSYSMAP is maintained correctly, it does not influence remote process chains. Remote chains reference a single, fixed logical system name across the entire landscape, so the entries in RSLOGSYSMAP are not evaluated for those calls and therefore have no effect.

Furthermore, there is no other mapping available to map the destinations in the remote chain during transport.

Resolution

  1. Define a consistently named RFC destination (for example, "remotechain") that points to the target system where the remote process chain will run.
  2. Use this same destination name in every BW landscape system (development, quality, production), but configure each one to point to the correct corresponding remote system for that landscape tier.

Example:

  1. Development: BWD → RFC destination "remotechain" points to RED
  2. Quality: BWQ → RFC destination "remotechain" points to REQ
  3. Production: BWP → RFC destination "remotechain" points to REP

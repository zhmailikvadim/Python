Symptom

How to secure the sensitive information (such as logon credentials, ClientId and Secret .etc) exposed in the JDBC URL when using CamelJdbcAdapter remote source.

Environment

SAP HANA smart data integration (SDI)

Cause

Since DP agent 2.8.1, there is a feature that provides security connection properties in credential entry, eliminating the need to append sensitive information to JDBC URL explicitly. 

Resolution

1. Refer to note 3605148 - [CamelJdbcAdapter] Add a new credential entry which can specify additional security connection properties since DPAgent 2.8.1

2. Run the following command:
    ./agentcli.sh --configAgent --> 4. Remote Source Credentials --> 2. Add Credentials --> Enter Remote Source Name: <your_remote_source_name> --> Enter User Name: db_security_properties --> Enter Password:[key1]=[value1];[key2]=[value2];...

Example: For OAuth Machine-to-Machine (M2M) authentication to connect to Databricks via CamelJdbcAdapter, use the following format:
   Password: OAuth2ClientId=xxxxxxxx;OAuth2Secret=xxxxxxxxx 

See Also

3605148 - [CamelJdbcAdapter] Add a new credential entry which can specify additional security connection properties since DPAgent 2.8.1

Symptom

A limited number of incorrect or unwanted records were loaded into HR Mini-Master-related tables (such as HRP1000, HRP1001, PA0000, PA0001) due to issues such as errors in number range definition. This prevents further data loading or testing activities in the HR Mini Master area. There is a need to safely remove these records from the tables without risking data inconsistencies or resorting to manual database deletions.

Environment

  • SAP ERP
  • SAP S/4HANA on-premise
  • SAP S/4HANA private cloud edition
  • SAP CRM
  • SAP SRM

Reproducing the Issue

  1. Due to a configuration or process error (for example, with number range assignment), unwanted records are imported into HR Mini-Master tables.
  2. Subsequent processes or data loads are blocked or result in errors because of these incorrect records.

Cause

Unintended data creation in HR-related tables, typically due to configuration errors (e.g., number range issues), leads to the presence of erroneous records that must be deleted to proceed with HR Mini Master processes and testing cycles.

Resolution

For PA tables (e.g., PA0000, PA0001):*
There is no standard tool to delete records in PA* tables directly at the database level, as these store infotype data and manual deletions can cause inconsistencies. The following recommended approaches are available:

  • PA30/PA40 transactions: Use for individual record deletion via the SAP GUI.
  • LSMW (Legacy System Migration Workbench): Use for mass deletions.

Note: It is not recommended to delete data directly from the database tables as this can result in data discrepancies and inconsistent application behavior.

For HRP tables (e.g., HRP1000, HRP1001):*

  • Report RHRHDC00: Use this report to selectively delete individual records by object and infotype/subtype. This allows granular deletions.
  • Report RHRHDL00: Use this report to delete all entries for specified objects and infotype/subtype (mass deletion).

See Also

Refer to SAP Note 2667088 - Reports for removing HRP1000 and HRP1001 data.

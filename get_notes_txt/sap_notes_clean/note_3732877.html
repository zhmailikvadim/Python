Symptom

  • Unable to copy all contents  to a destination folder in SAC
  • The operation fails after about 5 minutes with error "502 Bad Gateway", resulting in a partial copy.

Environment

  • SAP Analytics Cloud (Enterprise)

Reproducing the Issue

  1. In Files, select the source folder that contains many subfolders and objects (for example, under My Files or Public).
  2. Choose Copy, select the destination folder, and confirm.
  3. The copy operation fails after several minutes with error 502 Bad Gateway, and only part of the content appears in the destination.

Cause

  • Too many objects are included in a single folder copy operation, which leads to a 502 Bad Gateway response.
  • Large folder copies can exceed the processing window and trigger a 502 via the gateway. 
  • Copying a folder may take too long, causing the front proxy to time out, resulting in 502 — consistent with general 502 behavior such as upstream server delays or overloads.

Resolution

  1. Split the copy into multiple smaller batches (copy subfolders or groups of objects instead of the entire folder at once).
  2. Run each batch separately and wait for completion before starting the next to reduce the likelihood of timeouts and 502 errors.
  3. If moving content between tenants, consider using Content Network Storage (export/import) instead of copying large folders in a single operation.

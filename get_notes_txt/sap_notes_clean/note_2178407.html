Symptom

When running Cálculo de nómina española (RPCALCE0) report on the Pagas Extras (cálculo parte de derecho) (SPA) payroll function for an employee with hiring date set on Datos de Fecha (0041) infotype different than the Hiring event date from Medidas (0000) infotype,  SPA wrongly generates the number of days for Indemnización finiquitos (M717) and Ind. despido improcedente (M719) wage types.

Other Terms

&CALC; (further information see note 1071282)

Reason and Prerequisites

The issue is caused by a program error.
The Datos de Fecha for a date type relevant for the T530C configuration was not considered.


The following Support Package level is the minimum prerequisite:

Release Support Package
4.70 HR SP E8

(For more information, click on "R/3 HR Support Package Schedule" in www.service.sap.com/ocs-schedules.)

Solution

Corrections to source code of RPCSPAE5 include have been provided to solve this issue.

From now on whenever the employee has an date on 0041 infotype, that is also configured to M717/M719, and this date is different than the entry date on Medidas (0000) infotype, it will be calculated the M717 and M719 according to the criteria of SAP Note 1711630.

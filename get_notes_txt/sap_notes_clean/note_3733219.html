Symptom

SAP configured bundles are shared with customer sites. You can share the configuration bundles you created with multiple customer sites that are in the same data center. 

Resolution

Use this procedure to share configuration bundles with customer sites (to apply in a different site) using the Intelligent Configuration Manager workspace.

Prerequisites

1.You must be a member of the Partner Administrator group and Customer Administrator group.

2.You must use the Sourcing and Contracts site for this functionality.


You can follow the below steps to share configuration bundles with customer sites

1.Go to the Intelligent Configuration Manager workspace. For detailed steps, see Accessing Intelligent Configuration Manager Workspace.

2.From the menu bar, choose Bundles > Manage bundles.

3.From the Manage bundles page, the Bundles created section lists all preconfigured bundles created in this site. These can be shared with multiple customer sites.

Optionally, use the full-text search criteria in the search  field to search for the bundle.

4.Choose a bundle row with the Created status that you want to share by selecting to open and view the bundle details page.

5.From the Shared with customer sites section, select Add to add a new customer site to share the bundle with.

6.In the Add customer box, update the following required fields:

  • Ariba Network ID: Enter the Ariba Network ID, also called ANID. Make sure to include the System ID for child sites. For example, CHILD1-AN012345-T, AN034567-CHILD1, where CHILD-1 is the System ID.

  • Customer label: Enter the customer site details to identify the customer site uniquely. For example, if you've multiple sites associated with the customer that need some identification.

  • Confirm testing: Select the checkbox to confirm your testing and verification of the bundle.

7.Select Add to add a customer site.

Symptom

  • An error occurs when attempting to create a new Fiori application inside a CAP project using SAP Studio:
  • When selecting "Metadata" as the data source and choosing the "app" folder as the target location for the Fiori app, the error message states:

    The selected project folder is part of an existing CAP project. This application must use the local CAP project as the data source.

Environment

  • Tools for developing SAP Fiori applications
  • SAP Business Application Studio (BAS)

Reproducing the Issue

  1. Open SAP Studio and navigate to the CAP project.
  2. Attempt to create a new Fiori application.
  3. Select "Metadata" as the data source.
  4. On the subsequent screen, choose the "app" folder as the target location for the Fiori application.
  5. Observe the error message stating:
    "The selected project folder is part of an existing CAP project. This application must use the local CAP project as the data source."

Cause

  • The error occurs due to a new enhancement that enforces the use of the local CAP project as the data source for applications within CAP project structures.

Resolution

Using the tooling to generate an HTML5 application within a CAP project structure while selecting a data source that is not a CAP data source—is currently not supported.
This restriction has been in place since 2024. It was introduced because SAP observed cases where developers generated unrelated projects inside the CAP workspace and then expected the CAP deployment model (MTA/CAP build) to work without additional restructuring or configuration. 

For reference, the following backlog items tracked the change:

  • https://github.com/SAP/open-ux-tools/pull/2428

  • https://github.com/SAP/open-ux-tools/issues/2456

If you are looking for working CAP reference setups, including different approuter configurations, please refer to:
https://github.com/SAP-samples/fiori-tools-samples/tree/main/cap

If you are using a monorepo structure, one supported approach is to separate projects into subfolders with their own independent deployment configurations.
If your goal is to develop standalone HTML5 apps that consume CAP OData services, using an SAP BTP destination can also be a viable option.
The following guide provides an example:
https://github.com/SAP-samples/fiori-tools-samples/blob/main/cap/destination/README.md

Symptom

SAP instance can not be started and following error is found in the work process trace.

  • dev_w*

    C *** ERROR => unable to load DB2 library '/usr/sap/<SID>/<instance name>/db6_clidriver/lib/libdb2.so'.
    Reason: Unable to find library '/usr/sap/<SID>/<instance name>/exe/db6_clidriver/lib/libdb2.so'. 

R3trans finishes successfully.

  • trans.log

    Info: successfully loaded DB2 library '/usr/sap/<SID>/SYS/global/db6/<platform>/db6_clidriver/lib/libdb2.so' 


CLI driver in global directory is not successfully copied to instance directory by sapcpe.

  • sapcpe.log

    *************************************************
    source: /usr/sap/<SID>/SYS/exe/run
    target: /usr/sap/<SID>/<instance name>/exe
    *************************************************

    *** WARNING: ActualizeLocalFile: there is a problem to copy <src> because uid of <src> is 0
    , 2, No such file or directory, [sapxxcpe.c:2497]
    *** WARNING: ActualizeLocalFile: sapcpe skip copying <src>, because it is not able to set the right file mode to the target file
    , 2, No such file or directory, [sapxxcpe.c:2498]
    *** WARNING: ActualizeLocalFile: copy <src> to <dst> with root permissions
    , 2, No such file or directory, [sapxxcpe.c:2499]

Environment

SAP on IBM Db2 for Linux, UNIX, and Windows

Reproducing the Issue

Run command: sapcontrol -nr <instance_number> -function Start

Cause

Possible reason: Improper permission on SAP directory "/usr/sap/<SID>/SYS/exe/run"

Resolution

Make sure "/usr/sap/<SID>/SYS/exe/run" is owned by <sid>adm user, instead of root user.

The permission could be corrected with chown command.

For example: chown <sid>adm:sapsys /usr/sap/<SID>/SYS/exe/run

Keywords

load, DB2 library, sapcpe, root, permission

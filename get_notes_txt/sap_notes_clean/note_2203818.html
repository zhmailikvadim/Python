Symptom

After execution of ODSProductionLogETL and ODSNonconformanceETL scripts, records in ODS_TIME_DIM table are inconsistent:

  • Some records of GRANULARITY = SHIFT have NULL value in ODS_PRODUCTION_SHIFT and ODS_DAY_CLASS columns. Some of these records have GMT_BEGIN_DATE_TIME = GMT_END_DATE_TIME, and ODS_BEGIN_DATE_TIME = ODS_END_DATE_TIME.
  • PRODUCTION_LOG records of production shifts that span the midnight (for exmaple, 22:00-06:00) sometimes show 2 records - the first is 22:00-00:00 and the second is 00:00-06:00, but sometimes they show 3 records - 22:00-00:00, 00:00-06:00 and 22:00-06:00. However, it is expected to be just one record of 20:00-06:00.
  • Some records in ODS_TIME_DIM of GRANULARITY = SHIFT that have NULL value for the ODS_PRODUCTION_SHIFT and ODS_DAY_CLASS whereas the respective records in ODS_PRODUCTION_LOG table have DAY_CLASS and PRODUCTION_SHIFT columns filled in correctly.
  • Some records in ODS_TIME_DIM of GRANULARITY = SHIFT that have NULL value for the ODS_PRODUCTION_SHIFT and ODS_DAY_CLASS whereas the respective records in ODS_NC_DATA table have DAY_CLASS and PRODUCTION_SHIFT columns filled in correctly.
  • When Summer time is in effect, WIP.PRODUCTION_DATE.PRODUCTION_DATE value is one hour less than it should be.

Other Terms

SAP ME Scripts, ODS processing

Reason and Prerequisites

The root cause is that some scenarios were not covered by the original design.

Solution

The fix is implemented in a form of code change. See Manual Activity section for additional instructions.

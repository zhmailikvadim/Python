Symptom

While creating the agreement from master agreement postings, the user is populating custom field values in the BADI method DEFAULTS_FILL. Although the values are being assigned during the execution of the BADI, they are not getting saved in the agreement header table after the agreement is created.

This note ensures that the values populated in the DEFAULTS_FILL BADI will be saved correctly in the respective table.

Other Terms

Defaults_fill, /IRM/IPAGR_CREATE, /IRM/IPAGRM, Master Agreement Postings, /IRM/IPBBASP

Reason and Prerequisites

This is caused by program error.
Note : SAP Note# 3714680 is available for ECC release.

Note : Please implement SAP Note#3685025 as a Pre requisite note

Solution

Please implement the correction instructions. These corrections are only valid if you have applied the following Support Packs.

Release V/4 – 2023 V2, 2023 V3 and Support Pack 1

Use transaction SNOTE to implement the correction instructions specified below and activate the affected program object

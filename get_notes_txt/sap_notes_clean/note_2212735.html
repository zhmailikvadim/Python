Symptom

  • When a datawindow that includes a text field with a CR and/or LF included in the data is saved to a xlsx file, the following error is seen when it is opened with Excel 2007:

Excel found unreadable content in filename. Do you want to recover the contents of the workbook? If you trust the source of this workbook, click Yes.

  • The error did not occur with PB 12.6 GA and does not occur when using Office 2010 or 2013.  

Environment

  • SAP PowerBuilder (PB) 12.6 build 4058 and 4081
  • Microsoft Office 2007

Reproducing the Issue

  1. Using PB 12.6, Office 2007 and a datawindow with a text column with embedded CR/LF in the data, save the datawindow as an XLSX! file with the following code.
    1. Dw_1.SaveAs("savedname.xlsx", XLSX!, false)
  2. Open the resulting xlsx file with Excel 2007.   

Cause

The issue has been identified and logged under CR 789193.

Resolution

This CR 789193 is fixed in PowerBuilder 12.6 PL06 Build 4088 and higher.

Keywords

IM, InfoMaker, CR789193, CR 789193

Symptom

When querying the master data and organizational assignment replication, from EC to your S/4HANA system - it fails with the following error in CPI: "Fault: Multicast processing failed for number 0. Exchange[XXX]. Caused by: [org.apache.cxf.binding.soap.SoapFault - Authorization missing for service 'http://sap.com/xi/PASEIN EmployeeMasterDataAndOrgAssignmentBundleReplicationRequest_In', operation 'EmployeeMasterDataAndOrgAssignmentBundleReplicationRequest';

Environment

  • SAP SuccessFactors Employee Central integration with SAP Human Capital Management for SAP S/4HANA
  • Employee Master Data Replication
  • ECS4HCM Support Package

Reproducing the Issue

  1. Attempt to replicate employee master data and organizational assignments from Employee Central to the S/4HANA system.
  2. Observe the replication failure with the error: Authorization missing for service 'http://sap.com/xi/PASEIN EmployeeMasterDataAndOrgAssignmentBundleReplicationRequest_In' in CPI.

Cause

The user specified in CPI, that coincides with the user in the S/4HANA system, was not configured as a type B (System User) and/or the required roles have not been appropriately assigned to them.

Resolution

  1. Ensure the this user is configured in SU01 as a type B (System User)
  2. Assign the role SAP_HR_ECPAO_WEBSERVICES to the user.
  3. Note: you may have copied this standard role and created your own custom role, if so - please ensure that they have access to the following:
    1. Web Services: EmployeeMasterDataAndOrgAssignmentBundleReplicationRequest_In and EmployeeMasterDataAndOrgAssignmentPushNotification_In
    2. Tables: ECPAO_QRY_ADM, ECPAO_QRY_ADMDT and ECPAO_RPRQ_EE

For more information, please refer to the S/4HANA Help Guides: Granting Permissions in SAP S∕4HANA to Replicate Employee Master Data and Organizational Assignments from Employee Central | SAP Help Portal

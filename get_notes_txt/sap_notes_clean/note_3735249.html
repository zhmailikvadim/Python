Symptom

This article describes functionality introduced in the May 2026 SAP Fieldglass release

You may notice that a Security ID (SID) generated in SAP Fieldglass looks different than expected

This is most noticeable when

  • A name includes accented letters or characters from international alphabets
  • A name includes hyphens apostrophes spaces or punctuation
  • A newly created worker has a different‑looking Security ID than similar workers created in the past

Reproducing the Issue

This behavior may occur when

  • A worker, job seeker, or contractor name includes accented or international characters
  • A name contains hyphens, apostrophes, spaces, or periods
  • A Security ID is generated automatically during onboarding or profile creation
  • Newly generated Security IDs are compared with older Security IDs created previously

Cause

SAP Fieldglass uses updated global name handling rules when generating Security IDs

These rules help ensure

  • Consistent Security ID creation across countries and languages
  • Reliable behavior for international workforces
  • Predictable Security ID generation across the SAP Fieldglass application

Only the Security ID follows these global rules
The worker name is always stored and displayed exactly as entered

Resolution

This behavior is expected and does not require any action

SAP Fieldglass now applies consistent global standards when generating Security IDs

What to Expect

  • Accented or international characters are converted to standard letter equivalents for the Security ID
  • Hyphens spaces apostrophes and periods do not affect the Security ID
  • Some special symbols are not included in the Security ID
  • Newly generated Security IDs may look different from those created in the past
  • Your existing setup for automated Security ID generation continues to work as designed

If your organization uses Security IDs in downstream systems or reports ensure those processes do not rely on historical formatting patterns

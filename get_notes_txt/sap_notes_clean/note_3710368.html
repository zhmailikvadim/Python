Symptom

When attempting to delete a custom field created in the Custom Fields and Logic app for business context /IAM/ISSUE_ROOT, the system raises an error indicating that the field cannot be deleted due to existing usages.

The Custom Fields and Logic UI displays a message: "The selected field(s) cannot be removed from the business context due to the following existing usage(s). Please remove the usages and try again."

Also, the structure is non-editable to remove the field as per KBA 3600064 - Custom Field cannot be deleted due to external usages - SAP for Me suggestions.

Even after:

  • Executing report SDDIC_DDLS_INDEX_ALL_SOURCES following SAP Note 3137922 - Custom Fields fail to be Published/Deleted due to dependencies on own generated objects - SAP for Me,
  • Removing external usages in BRF+/FDT functions in all relevant clients (e.g., clients 400 and 500),
  • Ensuring that the custom field is no longer referenced in generated classes or BRF+ objects, 
  • Checking where‑used data shows no remaining relevant references.

the deletion in the Custom Fields app still fails.

Then both Custom Fields and Logic UI and transaction SCFD-LOG display message:

  • Create a task for transport request <transport_request>.

After creating a task for the transport request <transport_request> and updating apps "Configure Software Packages" (with 'Automatic Task Handling' on) and "Register Extensions for Transport." as per KBA 3032581 - Publish Custom Fields fail with transport request/tasks related issues. - SAP for Me 

The deletion in the Custom Fields app still fails,

With Custom Fields and Logic UI displays messages:

  • Please check custom field log (transaction SCFD_LOG for further messages).
  • Could not be published

In transaction SCFD_LOG, the following error is raised during deletion:

  • Return Code: 8. Object type: TABL. Object name: /IAM/D_I_ROOT. Nametab for table /IAM/D_I_ROOT cannot be generated.

Inspection of table /IAM/D_I_ROOT in SE11 shows the root cause:

  • Table change not possible. There is a DB trigger on the table.
    Table /IAM/D_I_ROOT (Statements could not be generated)

Environment

  • Custom Fields app
  • SAP S/4HANA Cloud Private Edition 2023
  • HEC_ABAP

Reproducing the Issue

  1. Log in to FLP .
  2. Search for the 'Custom Fields' Fiori app.
  3. Open the custom field.
  4. Click on the delete button available at the bottom.
  5. System gives an error: "The selected field(s) cannot be removed from the business context due to the following existing usage(s). Please remove the usages and try again."

Cause

Although BRF+/FDT references were removed from all relevant clients and created a task for the transport request <transport_request> and updated apps "Configure Software Packages" (with 'Automatic Task Handling' on) and "Register Extensions for Transport", the custom field deletion still fails because the generated database structure /IAM/D_I_ROOT cannot be regenerated.

In transaction SCFD_LOG, the following error is raised during deletion:

  • Return Code: 8. Object type: TABL. Object name: /IAM/D_I_ROOT. Nametab for table /IAM/D_I_ROOT cannot be generated

Inspection of table /IAM/D_I_ROOT in SE11 shows the root cause:

  • Table change not possible. There is a DB trigger on the table.
    Table /IAM/D_I_ROOT (Statements could not be generated)
  • As long as the DB trigger is present, the system cannot regenerate or adjust the table structure
  • This prevents the deletion and regeneration cycle required for removing the custom field

This behavior is aligned with SAP Note 2284776 - Table change not possible. There is a DB trigger on the table. - SAP for Me

Custom field deletion is a cross-client technical operation. Any DDIC structure regeneration (including append removal) must succeed. A DB trigger on the affected table blocks this step.

Resolution

For error "The selected field(s) cannot be removed from the business context due to the following existing usage(s). Please remove the usages and try again.":

  1. Remove the custom field from the specified FDT functions and activate them.
  2. Fix errors in the BRF+ function Z_CR_PR_TEMP_FUNCTION following the provided steps:

    1. Open t-code BRF+,  BRF+ workbench will be opened in browser

    2. Search for function Z_CR_PR_TEMP_FUNCTION , click check button and observe errors.

    3. Click Top Expression PR_MAIN_DECITION_TABLE => click Edit button for the decision table PR_MAIN_DECITION_TABLE

    4. Click button "Export To Excel" to backup the data as Excel

    5. Click button "Table Settings" -> In the popup, remove the Z elements.

    6. Press OK to close the popup and press save and activate the change.

    7. Press check button for function Z_CR_PR_TEMP_FUNCTION to see if the error still happen.

 

For error "Create a task for transport request <transport_request>.":

Create a task for the transport request <transport_request> and update apps "Configure Software Packages" (with 'Automatic Task Handling' on) and "Register Extensions for Transport." as per KBA 3032581 - Publish Custom Fields fail with transport request/tasks related issues. - SAP for Me 

 

For errors:

  • Please check custom field log (transaction SCFD_LOG for further messages).
  • Could not be published
  • Return Code: 8. Object type: TABL. Object name: /IAM/D_I_ROOT. Nametab for table /IAM/D_I_ROOT cannot be generated.

Remove the DB trigger on table /IAM/D_I_ROOT and retry the custom field deletion. Once the deletion is finished, the trigger can be added again if required. Further information regarding this issue can be found in note 2284776 - Table change not possible. There is a DB trigger on the table. - SAP for Me

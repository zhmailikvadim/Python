Symptom

Validate and Enrich functionality not working in Guided Buying (GB). When both validation error and enrich field are sent back in the response, the validation error is shown in GB, but the field is not enriched from the response.

Reproducing the Issue

  • Create a requisition in GB.
  • Submit the PR in GB.
  • Validate and Enrich call is sent from GB to external system.
  • External system sends both Validation error and a field for enrichment in response.
  • In GB, the validation error is shown but the field is not enriched.

Sample response from external system is as follows.

<Response>
    <Requisition_ProcessReqExtValidationStatusResponseImport_Item>
        <item>

            </EventDetails>
            <UniqueName>PRXXXX</UniqueName>
        </item>
    </Requisition_ProcessReqExtValidationStatusResponseImport_Item>
    <Requisition_ProcessReqExtEnrichResponseImport_Item>
        <item>
            <UniqueName>PRXXXX</UniqueName>
            <custom>
                <CustomString name="SampleEnrichFieldText">SampleText</CustomString>
            </custom>
        </item>
    </Requisition_ProcessReqExtEnrichResponseImport_Item>
    <ValidationError_ValidateErrorImport_Item>
        <item>
            <Date>2026-03-16T12:36:00Z</Date>
            <ErrorDetails>
                <item>
                    <ErrorCode>100</ErrorCode>
                    <ErrorMessage>V&E Sample error</ErrorMessage>
                    <ErrorType>1</ErrorType>
                </item>
            </ErrorDetails>
            <Id>PRXXXX</Id>
        </item>
    </ValidationError_ValidateErrorImport_Item>
</Response>

Resolution

By standard design, Guided Buying will treat Validate AND Enrich more like Validate OR Enrich because it doesn't allow for both to happen in the same response from the ERP. 

To have the system accept both validation error and enrichment together, parameter Application.GuidedBuy.EnableVandERevalidation needs to be enabled through Intelligent Configuration Manager (ICM).

See Also

If the requirement is to retrigger the approval flow after the Validate and Enrich response is received, the parameter Application.GuidedBuy.RerunApprovalAfterValidation needs to be enabled.

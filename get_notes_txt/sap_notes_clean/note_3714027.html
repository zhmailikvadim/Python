Symptom

You are migrating purchase orders (PO) using the Migration of PO documents to support new FM-MM integration (PTFM_MM_MIG_PO) report and you face one the following issues:

  • Items with no pre-commitment document informed in PO item are migrated and the pre-commitment document value is automatically generated by the migration process
  • You can't migrate PO with service items with multiple account assignment records as error message ‘Change indicators could not be uniquely determined in material mapping‘ (M_ 011) is raised.

Other Terms

Portugal localization, Public Sector Management, Funds Management, M_011 Error message

Reason and Prerequisites

You are implementing Public Sector for Portugal functionality.

You are upgrading your LOCPTFM add-on solution to SAP S/4HANA.

Solution

The PTFM_MM_MIG_PO report has been corrected to remove the issues described in this SAP correction note:

  • The pre-commitment document is no longer automatically generated,
  • A PO with service items with multiple account assignment records can now be migrated.

Import the Support Package specified for your release or implement the attached correction instructions.

As a general rule, SAP recommends that you implement the Support Package specified for your release. However, if you need to install a solution earlier, use the SAP Note Assistant to implement the correction instruction. You can find more information about the SAP Note Assistant in SAP Support Portal, under SAP Note Assistant.

Symptom

In ASE documentation, the default value of parameter "replication agent memory size" is documented as 4096, but with replication agent memory size = DEFAULT set in CFG file, the sp_configure "replication agent memory size" result shows the "Default" column value as 51200.

Environment

  • SAP ASE 16.0 SP03 PL10 and later
  • SAP ASE 16.0 SP04 PL02 and later
  • SAP ASE 16.1 GA and later

Cause

This is a documentation error. The default value of of parameter "replication agent memory size" has been changed to 51200 in above ASE versions. This is to make sure repagent can be started in HADR environment for most of the time.

Resolution

The documentation correction request has been raised. If the ASE is the source of a replication environment (HADR or non-HADR), please make sure "replication agent memory size" is set to default value (51200) at least or even higher in the versions above for the best performance of Rep Agent.

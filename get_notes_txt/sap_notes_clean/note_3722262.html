Symptom

  • Certain items such as Material Group, Ordering Unit, and Item Category added from the Item Master Data are not editable in the Production System for SAP Ariba Contracts.
  • The same template is used in both Production and Test, but the issue only occurs in the Production System.

Environment

Contract Line Items Document (CLID)

Reproducing the Issue

  1. Add an item from the Item Master Data to the Contract Line Item Document (CLID) in the production system.
  2. Attempt to edit fields such as Material Group, Ordering Unit, and Item Category.
  3. Observe that these fields are not editable in the Production System.
  4. Repeat the same steps in Test and the fields are editable.

Resolution

In order to correct this issue, follow the below steps:

  1. The groups, Integration Protected Fields, Procurement Agent, Procurement Manager, and Purchasing Agent are required to be able to edit the items from the Item Master Data.
  2. Verify the Groups assigned to the user in the Test System. The above mentioned groups will be found in the User's Profile in the Test system.
  3. Add the required groups, Integration Protected Fields, Procurement Agent, Procurement Manager, and Purchasing Agent to the user's profile in the Production System also.
  4. Test the editing functionality in the Production System after updating the User Groups.
  5. The specified fields from the Item Master Data should now be editable.
  6. If the issue is not caused by Group Assignment, reach out to SAP Support for further investigation.

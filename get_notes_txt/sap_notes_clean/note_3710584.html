Symptom

In advance search component in backoffice, when B2BUnit is used as the search condition, it is not searchable by its uid, instead it is only searchable by its name in the reference editor.

Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental. 

Environment

SAP Commerce Cloud

Cause

In OOTB default configuration, there is no separated explicit simple-search configuration for B2BUnit, its configuration inherits from its parent type OrgUnit, and only name attribute is defined.

commerceservicesbackoffice/resources/commerceservicesbackoffice-backoffice-config.xml


<context merge-by="type" parent="GenericItem" type="OrgUnit" component="simple-search">
    <simple-search:simple-search xmlns:simple-search="http://www.hybris.com/cockpitng/config/simplesearch">
        <field xmlns="http://www.hybris.com/cockpitng/config/simplesearch" name="name" />
    </simple-search:simple-search>
</context>

Resolution

In order to search B2BUnit by uid in reference editor, please add additional simple-search configuration for B2BUnit, e.g. below sample configuration.

<context merge-by="type" parent="OrgUnit" type="B2BUnit" component="simple-search">
    <simple-search:simple-search xmlns:simple-search="http://www.hybris.com/cockpitng/config/simplesearch">
        <simple-search:field name="uid"/>
    </simple-search:simple-search>
</context>

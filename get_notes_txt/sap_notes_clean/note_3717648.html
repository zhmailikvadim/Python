Symptom

An error occurs during the employee outbound run for an employee.

  • “Invalid list item is selected for Custom1: Custom 01 – CN Employee Group”
  • “Employee is not replicated”

Environment

SAP Business ByDesign 

Reproducing the Issue

  1. Go to Master Data Replication work center.
  2. Select Employee Outbound Run view.
  3. Choose the latest Application Log.
  4. In the results tab below error message will appear

“Invalid list item is selected for Custom1: Custom 01 – CN Employee Group”

“Employee is not replicated”

Cause

  • The replication error is caused by an invalid value maintained in the SAP Concur–side Custom01 field (Custom 01 – CN Employee Group) for the affected employee.
  • This field value is not mapped or does not match the expected list values, and the invalid entry is stored in the SAP Concur backend, resulting in replication failure.

Resolution

  1. Go to Business Configuration work center.
  2. In the overview
  3. Search for the activity Employee Replication into SAP Concur.
  4. Select Employee Replication into SAP Concur.
  5. Maintain the required field and value mapping for Custom01, as described in  Field and Value Mapping | SAP Help Portal .
  6. Ensure the Custom01 value included in the payload matches the valid list values maintained on the SAP Concur side.
  7. Verify whether multiple source systems are connected to the same SAP Concur tenant for employee replication, as this scenario can also cause similar replication issues.

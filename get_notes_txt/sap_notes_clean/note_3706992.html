Symptom

The system does not support posting goods issue (PGI) for medication kit returns from clinical sites through MIGO.

Other Terms

ISLS, Intelligent Clinical Supply Management, Deliver, Clinical Site Returns, 231 movement type, MIGO

Reason and Prerequisites

You use the Intelligent Clinical Supply Management.

Medication Kits that are in status ASOR (Assigned to Sales Order) are not considered AVAL (Available) at Clinical Sites.
Because of this, they do not appear in the selectable list during PGI when performing goods movement via MIGO.

This behavior is expected system logic:

  • ASOR means the MedKits are tied to a specific Sales Order and reserved for processing.
  • Only MedKits in status AVAL are considered available stock for site usage and therefore selectable during PGI.

Solution

Implement the attached correction instructions or apply the corresponding Support Package.

Prerequisites:

SAP Note 3701686 is a prerequisite Note, please implement both Notes together.

Before posting the 231 movement in MIGO, ensure the following prerequisites are met:

  1. A Return Sales Order containing Medication Kits has been created.
  2. In MIGO, the following fields must be completed:
    • Material
    • Quantity
    • Sales Order / Item
    • Virtual Plant / Storage Location
    • Batch
  3. The MedKit Status must be ASOR (Assigned to Sales Order) before posting the 231 movement.

Instructions:

To ensure the correct Medication Kits are posted during PGI:

  1. In MIGO, navigate to the Study Information tab.
  2. Manually maintain the Medication Kits, or Copy the MedKits from the Return Sales Order.
  3. Verify that the correct MedKits are assigned before completing the posting.

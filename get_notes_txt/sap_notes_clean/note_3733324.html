Symptom

Incorrect P_6* field values may be generated in the KSeF XML for billing scenarios.

Other Terms

Poland, eDocument, eDoc, eInvoicing, electronic invoice, electronic invoicing, Poland eInvoice, Document Compliance, Digital Compliance, DC, B2B, KSeF, Electronic Document, Document and Reporting Compliance, DRC, Podmiot3, FA(3) Schema Version, P_6, P_6A, P_6_OD, P_6_DO, periodical billing, non-periodical billing.

Reason and Prerequisites

  • For a complete prerequisite list, refer to SAP Note 3279206 (Electronic Document Processing for Poland - KSeF: Implementation Overview).

Solution

Date                  Version         Reason                                                                                        
31.03.2026 1 Released for Customers

The logic for determining the P_6* fields in the KSeF XML has been corrected. The system now handles all affected billing scenarios properly, including periodic, non‑periodic, (mixed items, and correction documents). The generated XML now reflects the expected values.

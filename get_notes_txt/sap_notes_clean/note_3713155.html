Symptom

  • The "Get feedback" function in the Latest Performance Agreement Form is not displaying goals for reviewers, despite goals being set up by employees.
  • Reviewers see a message stating "No goals yet."

Environment

  • SAP SuccessFactors Performance Management

Reproducing the Issue

  1. Enable the Latest Performance Agreement Form in the Preview instance.
  2. Test the "Get feedback" functionality
  3. Send a feedback request to the reviewer.
  4. Reviewer attempts to view goals in the form but sees the message "No goals yet."

Cause

The issue arises due to the enhanced permission framework in the Latest Performance experience. If users are not granted the appropriate Read or specific permissions for fields in the Plan template, they are unable to view goals on the form.

Resolution

We recommend configuring the Goal Plan XML and assigning the appropriate Read and Write permissions to the “F” role (Form Reviewer). The purpose of the “F” role is to grant form reviewers additional access that they may not normally receive through the Goal Management.

All TGM permission roles apply consistently across the system. For example, if a user has Write permission as EM (Employee), they will have Write permission both on the form and in TGM. The only special behavior for the “F” role is that a user is assigned to this role only while reviewing a form, which allows temporary elevated access during the review process.

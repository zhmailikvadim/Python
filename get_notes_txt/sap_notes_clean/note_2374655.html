Symptom

When deploying an application from Web-IDE to the Portal, the property "Relative Path for SAPUI5 Applications" value in the created iView is not generated.

Other Terms

WEB-IDE, deploy plugin

Reason and Prerequisites

Prerequisites:

Your support package (SP) and patch level for the relevant component(s) is lower than the SP/patch combination which fixes this issue for your portal version. The specific SPs and patches containing the fix are visible in the "Support Packages & Patches" section of this note.

Reason:
Program Error

Solution

Refer to Support Packages & Patches link to view the versions including the fix.

See the following link for information about providing feedback about this note:

http://scn.sap.com/community/netweaver-portal/blog/2013/11/24/feedback-on-netweaver-enterprise-portal-notes

WorkAround: edit the iView's properties and set the path to your html with the same timestamp value as set for SAPUI5 path property.
Note that if this workaround will have to be made after every re-deployment of the same application.

Symptom

  • The system encounters a runtime error "DBSQL_SQL_ERROR" during the archiving process for FI_DOCUMNT.
  • SQL error "SQL code: -10807" occurs while accessing table "BSID".
  • The error is associated with ABAP program IM_EX_FDM_INVOICE_ARC_CHECK===CP".
  • The archiving job terminates with a runtime error "DBSQL_SQL_ERROR" and the job is canceled.

Environment

  • FSCM Collections Management
  • S/4HANA 

Reproducing the Issue

  1. Execute transaction code SARA.
  2. Select object FI_DOCUMNT.
  3. Navigate to the job overview.
  4. Execute the job FI_DOCUMENT_WRI.
  5. Review the job log for runtime errors such as "DBSQL_SQL_ERROR".

Category               ABAP programming error
Runtime Errors         DBSQL_SQL_ERROR
Except.                CX_SY_OPEN_SQL_DB
ABAP Program           IM_EX_FDM_INVOICE_ARC_CHECK===CP

Cause

The ILM archiving process triggers the BAdI implementation FDM_INVOICE_ARC_CHECK (BAdI Definition: FI_DOCUMNT_CHECK) to verify whether a document is referenced in any open items.

Based on the ST22 info, this check is performed on the BSID view where it is taking more time, which is built on top of the BSEG table. The selection criteria involve the following fields: BUKRS, KUNNR, REBZG, REBZJ, and REBZZ.

Method: IF_EX_FI_DOCUMNT_CHECK~CHECK_FOR_ARCHIVING
*   analyse whether there are open items referring to this invoice
    SELECT SINGLE mandt FROM  bsid INTO l_mandt
                        WHERE bukrs EQ ls_item-bukrs
                        AND   kunnr EQ ls_item-kunnr
                        AND   rebzg EQ ls_invoice_key-belnr
                        AND   rebzj EQ ls_invoice_key-gjahr
                        AND   rebzz EQ ls_invoice_key-buzei.

Resolution

  1. Verify if the system has a high document volume in the BSEG table.
  2. Check the selection criteria used in the archiving process, specifically fields: BUKRS, KUNNR, REBZG, REBZJ, and REBZZ.
  3. If the system performance is impacted due to high document volume, create a secondary index on the BSEG table using these fields.
  4. Monitor the system performance after implementing the secondary index to ensure the issue is resolved.

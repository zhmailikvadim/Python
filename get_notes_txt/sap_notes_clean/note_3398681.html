Symptom

Planning to do a system copy and have the plan that at first you pre-install a new target system with SWPM tool and after it perform a database restore from the source system.

But you also would like to know whether you need to run SWPM with System Copy scenario instead.

 

Environment

SAP NetWeaver 7.1 and higher

OS and DB independent

AS ABAP

AS JAVA

Reproducing the Issue

Pre-install a new target system with SWPM tool and perform a database restore from the source system on this newly installed system.

Resolution

Perform a system copy, there is no need to pre-install a new system and do a backup restore after it.

SWPM tool has the function and you can perform a system copy (target system installation) with Database Independent (export/import) or with Database Specific (backup/restore) option.

 

Kindly check the official system copy guide about this, for example:

Guide name: System Copy for SAP Systems Based on the Application Server ABAP of SAP NetWeaver 7.3 EHP1 to 7.52 on UNIX 

Phase: Database-Specific System Copy

 

Link: https://help.sap.com/docs/SLTOOLSET/6ffd9a3438944dc39dfe288d758a2ed5/7fba665344db418d882539927430791c.html?version=CURRENT_VERSION&locale=en-US

 

Note

When performing a system copy using a database-specific method, it is not required to run the software provisioning manager in the source system to export it. You only have to run the software provisioning manager on the target host to install the target system.

[...]

Start the software provisioning manager as described in the installation guide and follow the instructions on the software provisioning manager screens until you are requested to perform the database backup/restore.

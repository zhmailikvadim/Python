Symptom

The customer ran a LoadRunner simulation and encountered the aforementioned LockTimeoutException.

Other Terms

PA7.3, AS5.2, AuthoritySuite, ProductAuthority

Reason and Prerequisites

Steps to Reproduce:

 Run a LoadRunner simulation for 136 users.

Actual Result: The simulation is generating the LockTimeoutException in the log file.

Expected Result: No LockTimeoutException generated in the log file.

Cause:

The Spring Web Flow is trying to acquire a lock that is being impeded by undetermined cause, most probably because a thread is locking the database.

Solution

Allow the time to wait to acquire the lock to be slightly higher to be sure.

Symptom

Message No. PG628 "Information type * not permitted" occurs when creating an infotype via t-code PA30.
"Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental."

Environment

  • Human Capital Management (HCM)
  • SAP ERP
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for HANA
  • SAP S/4HANA
  • SAP S/4HANA Cloud Private Edition
  • SAP SuccessFactors Employee Central Payroll

Reproducing the Issue

  1. Run t-code PA30.
  2. Choose a pernr.
  3. Create IT3426 for the employee.
  4. Observe the error message: "Information type 'infotype number' not permitted (Message No. PG628)".

Cause

V_T582L is not maintained correctly.

Resolution

V_T582L is used to control which infotype/subtype can be used as per country localization. Please check the configuration of V_T582L and adjust the configuration as per the business requirement.

Example:

  • Take IT3426 as an example. IT3426 can only be used for country Ireland as per following configuration, which means that IT3426 cannot be used for other countries. 
     
  • Please do not maintain entries like below. The first entry gives Switzerland (HCMLoc = 02) only the authorization for the IT0010; the second entry gives all countries/regions except Germany (HCMLoc = 01) the authorization for the IT0010. Therein lies the inconsistency and lead to Message no. PG622.
    ITypeInfotype TextTypeNameHCMLocNameAllN.AllwithWa 
    0010CapitalFor*allSubt02SwitzerlandX   
    0010CapitalFor*allSubt01Germany X  

    Note: HCM Localization '99 Other countries' is used for countries that do not have a specific code. For example, Germany is 01, the US is 10, and the UK is 08. MOLGA 99 ‘Other countries’ doesn't imply coverage for all countries.

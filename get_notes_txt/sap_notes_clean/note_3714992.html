Symptom

  • Value of fields VALFR and VALTO are missing in the extractor /IBP/PRODUCTIONSOURCEITEM_ATTR.
    • Both fields are marked as "hidden" in RSA5 or 
    • The IBP add-on Support Package is above 1.1 SP24, but these two fields are missing in RSA6

Environment

  • SAP ERP, supply chain integration add-on for SAP Integrated Business Planning
  • SAP S/4HANA, supply chain integration add-on for SAP Integrated Business Planning
  • SAP Integrated Business Planning (SCM-IBP)

Reproducing the Issue

  1. Open extractor /IBP/PRODUCTIONSOURCEITEM_ATTR with program /IBP/ETS_REPL_TEST in transaction SE38.
  2. Filter production source item entries by validity fields using the extractor /IBP/PRODUCTIONSOURCEITEM_ATTR.  
  3. 3. Confirm that validity dates exist in table /IBP/SOSBOM_EXT, but no entries are extracted for columns VALFR and VALTO.  

Cause

• Fields VALFR and VALTO were not valid for use before IBP add-on SP24. 
• The latest version of the DataSource may not have been activated after the IBP add-on support package upgrade. 

Resolution

  1. Verify the IBP add-on support package version in the affected system. Ensure it is SP24 or higher, as fields VALFR and VALTO are valid starting from SP24. Refer to 3252355 - IBP Add-On: Add validity fields to extractors PRODUCTIONSOURCEITEM, PRODUCTIONRESOURCE, SOURCELOCATION, and SOURCEPRODUCTION for more details.
  2. Activate the DataSource /IBP/PRODUCTIONSOURCEITEM_ATTR in RSA5 to ensure the latest version is active. 
  3. Verify that the fields VALFR and VALTO are visible in RSA6 after activation.
  4. Run the extractor /IBP/PRODUCTIONSOURCEITEM_ATTR using program /IBP/ETS_REPL_TEST in SE38 and confirm that the fields are displayed correctly.

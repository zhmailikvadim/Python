Symptom

"Failed processing idoc for interchange ********- group *****- doc 0001: Incoming cXML is referencing an invalid OrderRequest with reference ********* Thanking you, The Undefined string with key path: "Personality.sapBusinessNetworkBrandNameTeam""

Cause

By default, Legacy EDI uses the BAK09 element to match the order date in the system. However, when the DTM*004 segment is present, it takes precedence over the date provided in BAK09 for order date matching.

A comparison between successful and failed examples shows that, in failed cases, the DTM*004 segment contains a date that does not match the actual purchase order date. As a result, although the order ID is correct, the mismatched date prevents the application from matching the order confirmation to the corresponding purchase order.

All successful examples provided include the correct order date in the DTM*004 segment, while all failed examples contain an incorrect date in this field.

Resolution

To resolve this issue, the EDI implementation must be reviewed to ensure that the correct purchase order date is consistently sent in the DTM*004 segment.

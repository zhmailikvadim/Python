Symptom

404 Missing Page while trying to access the BI LaunchPad using SAML.

Environment

  • SAP BusinessObjects Business Intelligence 4.x

Reproducing the Issue

  1. Configure SAML
  2. Go to BI LaunchPad using SAML.
  3. Prompts the 404 Missing Page error.

Cause

On the SecurityContext.xml file, the entityBaseURLis is being set incorrectly, leading to the 404 Missing Page error.

Resolution

  1. Go to the entityBaseURL of your SecurityContext.xml file.
  2. Check if the points below are being followed correctly.
    1. The entityBaseURL must be the base URL to access BI or CMC
      1. https://www.example.com/BOE
    2. If the serverPort is a non-standard http/https port (not 80/443), then add the port to the value URL (example: https://www.example.com:8443/BOE)
  3. Since the SecurityContext.xml was changed, a new Metadata file must be generated to be uploaded to the IDP.
    1. Please follow the Section 7 to 8 from the KBA 2791348 - How To: Configure Front-End SAML Authentication on BI 4.3 / 2025 - SAP for Me for detailed information.
  4. After the above steps were done, restart Tomcat, and test the SAML Authentication.

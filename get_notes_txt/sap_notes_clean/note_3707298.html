Symptom

PP01 is a generic transaction used across the applications based on the personnel development framework, hence the authorization checks are generic.

  • Possible data inconsistencies while using transaction PP01.
  • LSO authorization checks not used via direct access to transaction PP01.

Environment

  • Human Capital Management (HCM)
  • SAP R/3
  • SAP R/3 Enterprise 4.7
  • SAP ERP Core Component
  • SAP ERP
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA

Reproducing the Issue

  1. Goto  transaction PP01.
  2. Select the Plan Version select i.e.  Current Plan.
  3. Select the Object Type i.e. L Course group.
  4. Select the Object ID and press F4 to open Restrict Value Range and press continue.
  5. When the Course Group list appears on the screen, select one of the Object IDs.
  6. For example, changing the Relationship of this object.
  7. Highlight the Relationship line , select Change icon on the screen. It will take you inside Relationship infotype screen.
  8. Any changes made to this infotype may cause inconsistencies.

Cause

This happens because transaction PP01 should not have been used to create/delete relationships.

Resolution

  1. Use only LSO specific transactions for the LSO application to avoid inconsistencies
  2. Avoid making changes in generic transactions like PP01, as they are not supported for LSO applications. 

Keywords

PP01 transaction, data inconsistencies, LSO_PSV2 transaction, training management & LSO application

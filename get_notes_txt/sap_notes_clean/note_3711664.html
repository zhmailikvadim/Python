Symptom

You execute an Asset Accounting report like the Asset History Sheet for a mid-year date with an asset as described in SAP Note 2868410.  An example of this asset might be:

1. An asset that has been retired in the middle of the year and uses a retirement period control method such as 'End of the year (Start of next fiscal year)'.

2. An asset created via a mid-year Legacy Data Transfer, for which no current-year posted depreciation was transferred.

The displayed current-year depreciation and/or the end book value is incorrect.

Other Terms

 POPER_SHIFTED, AR02N, AR20N

Solution

SAP GUI reporting does not consider the field POPER_SHIFTED from table FAAT_PLAN_VALUES when creating the final output table.

Consider switching to SAP Fiori reporting. Please note that only Fiori-native apps, such as the ones mentioned in SAP Note 3052618, will consider the POPER_SHIFTED field correctly.

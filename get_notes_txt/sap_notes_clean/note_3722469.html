You can find information about the minimum version and release requirements for integrated applications in the Integration Guide for SAP Digital Manufacturing on the SAP Help Portal. Manual Activities

Update UI5 Library

  • SAP UI5 version updates from 1.136.9 to 1.136.15 as part of the UI5 patch upgrade within the 1.136 LTS maintenance version.
  • You can find UI5 version change logs at the link below. Choosing the desired version shows a list of change logs for that version: https://ui5.sap.com/versionoverview.html.
  • The selected version 1.136.15 is the latest stable patch in the current LTS line during upgrade planning.

  • Applications using custom themes created on older SAPUI5 base themes need to ensure compatibility after the upgrade. You might need to rebuild themes if the original base theme is deprecated.

  • Applications with custom CSS overrides should perform regression testing because framework-level styling changes can affect rendering behavior.
  • Custom development extensions like POD plugins or customer-owned templates also need validation by the owning development team, as SAP cannot predict UI5 control usage inside custom extensions.

AI-Assisted Production Engineering in the Design Production Processes and the Monitor Production Processes Apps (Service Catalog ID FA0054 and FA0058)

With the official launch of AI-assisted production engineering in release 2605, if you’re a new user or beta user, you need the Base_AI_Admin role assigned to you. You also need the Production_Engineer role to use it.

Additionally, you need to turn on Base AI Features toggle in the Manage Business Settings app to enable the feature. Keep in mind that the feature takes effect 10 minutes after you turn on the toggle. If you don’t turn it on, the feature stays hidden.

Scripts you generated during the beta testing phase aren’t affected.

Edge – Upgrade PostgreSQL

PostgreSQL 14 goes out of maintenance in November 2026. SAP Digital Manufacturing for edge computing supports PostgreSQL 16 starting with release 2605. We recommend you upgrade to PostgreSQL 16 as soon as you can. Make sure you upgrade PostgreSQL first before upgrade SAP Digital Manufacturing for edge computing.

For embedded PostgreSQL, perform these migration steps before the database upgrades automatically during your SAP Digital Manufacturing for edge computing application upgrade to 2605:

1. Update values.yaml:

a. Set global.database.pg_upgrade.enabled to "true".

b. Set global.database.pg_upgrade.oldPgVersion to "14" and global.database.pg_upgrade.targetPgVersion to "16".

2. Upgrade your SAP Digital Manufacturing for edge computing application to 2605.

Note:

a. After you complete the embedded PostgreSQL upgrade, remove global.database.pg_upgrade.enabled and the other values from your values.yaml.

b. The PostgreSQL upgrade creates the service account pg-upgrade-job-service-account, the role pg-upgrade-job-role, and the role binding pg-upgrade-job-role-binding to facilitate the upgrade. You can delete these resources manually after the upgrade completes when you no longer need them.

Deprecations

Incompatible Changes

Reserved Words for Script Tasks in the Design Production Processes app (Service Catalog ID FA0054)

To avoid runtime errors or unintended behaviors, you can't save or deploy production processes that contain reserved words in script code content. For details on the reserved words, see SAP Note 3379404.

AIN Equipment Creation (Service Catalog ID FA0092, FA0093, and FA0095)

The AIN Equipment Creation scenario has been removed, so the following integration elements aren't available in SAP Digital Manufacturing:

  • Collaboration link AIN_COLLABORATION_SFC_DONE and collaboration directive AIN_CREATE_EQUIPMENT_POST
  • Integration message AIN Creation

Limits for Service Loop Calls Triggered by Event Business Rules from the Manage Automatic Triggers app (Service Catalog ID FA0057)

To avoid recursive loops, we have set limits on how many times event business rules can trigger production processes in a single day. For details on the limitation, see SAP Note 3379404.

Additional Enhancements

Inventory API (Service Catalog ID AP0010)

  • GET /api/v2/inventory/goodsIssues - Service: Retrieve Goods Issue V2 - The maximum page size allowed in the request payload for this endpoint is increased from 100 records to 500 records. - A new field bomComponentSequence has been introduced in the API response. This field is used to differentiate goods issue postings for line items with the same material and helps identify the corresponding BOM line item when the same material appears in multiple lines. Note: The bomConponentSequence information is available through this API only for material consumption records executed after the 2605 release.
  • GET V2/inventory/goodsReceipts - Service: Get Goods Receipts V2 - The maximum page size allowed in the request payload for this endpoint has been increased from 100 records to 500 records.

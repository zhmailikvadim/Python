Symptom

  • Errorlog contains error 14315 error, "Error while inserting text into syscomments for partition condition <value> in database <value> for table <value>, indid <value>."
  • Error is followed by a stack trace which includes the modules 'ptnc__fill_syscomments' and 'crt_main'

Environment

SAP Adaptive Server Enterprise 

Cause

There is not enough free space available in the database while creating a table with partitions or creating new partitions for a table.

The error is expected, but the stack trace is not.

The issue has been reported to engineering as CR 824878

Resolution

Alter the database to make more free space on the system segment and retry the CREATE TABLE statement.

CR 824878 is fixed in

  • ASE 16,0 SP03 PL11
  • ASE 16.0 SP04 PL02

With the fix in place, the error will still be raised when creating a partitioned table when the database is out of space, but the stack trace will not be output

Keywords

14315, ptnc__fill_syscomments, crt_main

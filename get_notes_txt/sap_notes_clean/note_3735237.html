Symptom

After I performed a successful PATCH request using the SAP Ariba SCIM API to update the purchasingUnits attribute for a user, the updated value is not reflected in the SAP Ariba UI.

Reproducing the Issue

  1. Create or identify an existing user in SAP Ariba with an assigned purchasingUnits value.
  2. Execute a PATCH request using the SCIM API endpoint to update the purchasingUnits attribute to a new value.
  3. Verify the PATCH response returns HTTP `200 OK` indicating success. 
  4. Navigate to Core Administration Users in the SAP Ariba UI.
  5. Observe that the purchasingUnits value displayed in the UI still shows the previous value and has not been updated.
  6. Confirm the updated value is visible by performing a GET request on the same user via the SCIM API.

Cause

The SAP Ariba UI does not immediately reflect changes made via the SCIM API to the purchasingUnits attribute due to a cache-related behavior on the UI side.

Resolution

Clear the browser cache and cookies and reload the SAP Ariba UI to see the updated attribute values. The data is correctly updated at the backend level and is accessible via the GET endpoint 

For steps on how to clear browser cache and cookies, refer to: How to clear Cache and Cookies of Browser

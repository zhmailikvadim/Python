Symptom

When you manually move partitions, the partitions end up on a single NUMA node of the target host.

Whether you run ALTER TABLE <TABLE_NAME> MOVE PARTITION <PARTITION_ID> TO LOCATION '<HOST:PORT>'; or perform the move using administration tools, the moved partitions are placed on the same NUMA node.

You can confirm it by checking the NUMA_NODE_INDEX column in the M_CS_COLUMNS monitoring view.

Other Terms

NUMA, Partition

Reason and Prerequisites

Issue number 356353

Reason:

Due to a programming error in SAP HANA, the metadata for the partition information does not update during the manual move of the partitions.

Resulting in the manual movement of partitions to end up in the same NUMA node.

Affected Releases:

  • SAP HANA 2:
    • Revisions of SPS00 - SPS06
    • Revisions <= 079.09 (SPS07)
    • Revisions <= 089.01 (SPS08)

Prerequisites:

  • The HANA system is a scale-out configuration.
  • Partitions are moved between different hosts.
  • The target host has 2 or more NUMA nodes.

Solution

Apply one of the following SAP HANA Revisions:

  • SAP HANA 2:
    • Revisions of SPS07: SAP is currently working on a permanent solution to this issue. This SAP Note will be updated as soon as a revision containing the solution is available
    • Revisions of SPS08: SAP is currently working on a permanent solution to this issue. This SAP Note will be updated as soon as a revision containing the solution is available

Workaround:

  • After alter/move is completed, unload and reload the table from memory.
      1. UNLOAD <SCHEMA>.<TABLE>
      2. LOAD <SCHEMA>.<TABLE> ALL
  • After the reload, check the NUMA distribution of partitions using M_CS_COLUMNS

Symptom

When configuring Joule in SAP BTP, which is connected to IAS Tenant A and SAP Integrated Solution system is connected to IAS Tenant B. User need clarification on whether Joule integration requires:
  • A single shared SAP Cloud Identity Services (IAS) tenant across systems, or
  • Whether Joule can function when multiple IAS tenants are used.

Environment

  • SAP BTP, Cloud Foundry runtime and environment
  • Joule

Resolution

  • Joule and all SAP products integrated with Joule must use the same SAP Cloud Identity Services tenant. Additionally, within all integrated products, users must be consistently represented by the same identity (Global User ID). For more details about the Global User ID, refer to Global User ID in Integration Scenarios.

  • Failure to configure or incorrect configuration of the Global User ID field in Joule (via the assertion attribute user_uuid in the corresponding IAS application) and integrated cloud applications can result in functional issues within Joule and its integrations.

  • Entitlements for Cloud Identity Services- Identity Authentication and Identity Provisioning. Joule uses the Identity Authentication for user login and Identity Provisioning service to handle provisioning of identities and their authorizations to various business applications (SAP SuccessFactors, Identity Authentication Service Tenant, SAP Build Work Zone). For more information, see Tenant Models and licensing information for Identity Authentication and Identity Provisioning.

  • Ensure that there is one IAS tenant per single independent entity and operate every single independent entity from its own dedicated SAP BTP subaccount.

Symptom

Conversion from ECC to S/4HANA 2023 using Software Update Manager (SUM) is halted in phase PREP_CONFIGURATION/ADDONPRECHECK with error:

ADDONPRECHECK.LOG:

2 ETQ374 Parameter "EV_RC" = "8"
2 ETQ373 Parameter "EV_MESSAGE" = "The Add-On check found at least one error"
2WETQ399 Add-on component ST-A/PI: ACP or upgrade package missing.

Environment

  • SAP NetWeaver AS ABAP
  • SAP ECC, any Enhancement Pack
  • SAP S/4HANA
  • Software Update Manager (SUM)

Reproducing the Issue

Repeat the error phase in SUM.

Cause

ST-A/PI is an independent component of SP stack, hence while planning a stack.xml in Maintenance Planner, ST-A/PI recent package is not included in stack.xml used during Upgrade.

Explained in KBA 2424507 - Missing ST-A/PI in stack.xml

Resolution

  1. Download the latest version of ST-A/PI compatible with Target stack. Versions compatibility can be checked from SAP Note 69455 - Servicetools for Applications ST-A/PI (ST14, RTCCTOOL, ST12).
  2. Download the respective Addon Installation Package and Addon Upgrade package from Software Downloads and upload it in system using SPAM/SAINT. This step will extract the SAR files and upload it in EPS/in.
  3. Repeat the phase from SUM.

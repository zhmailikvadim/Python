Symptom

When comparing results between transactions MD06 and RMMD06NEW, the output does not match.

 

Environment

  • SAP S/4HANA
  • SAP S/4HANA Cloud Private Edition

Cause

Standard system behavior.

Resolution

When evaluating material "1234" for a specific plant and MRP controller, the following logic applies:

  • MD06 retrieves existing MRP lists based on the assigned selection criteria. It reads data from table MDKP, where an MRP list may already exist for the material.
  • RMMD06NEW selects materials directly from the material master according to the MRP controller maintained in the master data.

Differences in the results may occur when the MRP controller stored in the MRP list (MDKP) does not match the MRP controller maintained in the material master, which is an unusual but possible data inconsistency.

This behavior is by design.

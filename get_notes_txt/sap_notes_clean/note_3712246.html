Symptom

The (FAIR-based) process "Execute Allocation" (/IBS/BSC_ALLOC_ALPROC) processes building savings contracts with posted disbursements even if the contract phase check "Check Disbursement Date" (PAYOUT_DATE) is active.

Reason and Prerequisites

The assignment date is determined incorrectly in the check, which means that the check does not start.
In addition, contracts with disbursements that have already been posted have the status "Building Savings Contract Ended" (100) and should also not be taken into account without the contract phase check "Check Disbursement Date" in the "Execute Allocation" process.

Solution

The "Selection Procedures for Granting" can be added for the "Execute Grant" process using the IMG activities "Assign Selection Classes to Actions" (/IBS/VBSC_ACTNFC) and "Assign Join Classes to Actions" (/IBS/VBSC_ACTNFJ). By assigning the classes

  • /IBS/CL_BSC_WC_INCL_ST_AFEEPYD as "Class for Where Enhancement" in IMG activity /IBS/VBSC_ACTNFC
  • /IBS/CL_BSC_JC_TFM_BBS as "Class for Join Enhancement" in IMG activity /IBS/VBSC_ACTNFJ

For the action ALPROC (application ALLOCA and relevant building savings price(s)), you can ensure that only contracts with the status "Building Savings Contract Active, Settlement Charge Paid" (20) are selected. This implicitly also leads to the exclusion of contracts with disbursements that have already been posted. After you implement this SAP Note, check the Customizing in the aforementioned IMG activities and add the settings if required.

The contract phase check "Check Disbursement Date" introduced with SAP Note 3661064 is no longer required and is removed. As a result, it is necessary to remove all usages of the check PAYOUT_DATE in the IMG activity "Assign Business Transaction to Contract Phase" (/IBS/CBSC_CKPHBT). After you implement this SAP Note, check the Customizing in the aforementioned IMG activity and clean up the settings if required.

Symptom

  • When running FEB_FILE_HANDLING with a Brazil Boleto CNAB 400 file, the process finishes but no records are saved in bank statement tables (FEBKO/FEBEP).
  • No error message is shown; the transaction indicates completion without creating entries.
  • Importing the same content via FF_5 works and creates entries in FEBKO/FEBEP.

Environment

  • SAP S/4HANA Cloud Private Edition
  • SAP S/4HANA

Reproducing the Issue

  1. Prepare a Brazil Boleto CNAB 400 return file.
  2. Run transaction FEB_FILE_HANDLING with the CNAB 400 file as input.
  3. Observe that the process ends without errors but no entries are written to FEBKO/FEBEP.
  4. (Comparison) Run transaction FF_5 with the same CNAB 400 content and observe entries are created in FEBKO/FEBEP.

Cause

FEB_FILE_HANDLING currently supports only the CNAB 240 format; CNAB 400 files are not supported, therefore no bank statement entries are created.

Resolution

  1. Use the supported CNAB 240 format with FEB_FILE_HANDLING. Confirm with the bank whether CNAB 240 files can be provided.
  2. If only CNAB 400 files are available, import using FF_5 as an alternative process.

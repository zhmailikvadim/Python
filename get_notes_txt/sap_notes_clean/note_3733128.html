Symptom

  • Managed system setup of data services system
  • AC Tool run completes only scripts 1, 2, and 4; scripts 3 and 5 are missing.
  • After following the document https://help.sap.com/docs/SUPPORT_CONTENT/sm/3362968151.html
    only few scripts can be seen

Environment

  • Diagnostics
  • Data services system
  • AC tool
  • Solution manager 7.2

Reproducing the Issue

  1. Run ACTool for the managed system setup of Data Services and observe only scripts 1, 2, and 4 executed; scripts 3 and 5 do not run.
  2. Attempt SLD registration (for example via ACTool or sld_ds.bat) and observe HTTP 401 (Unauthorized) in the registration output.

Cause

  • Scripts should be available in the  products.xml  file

Resolution

  1. Make sure you are using latest version of AC Tool
  2. Follow all the steps as shown in the following help doc  Managed system setup of Data Services 4.x system in the Solution Manager 7.1 and 7.2 | SAP Help Portal 
  3. Check for products.xml file

  • Here you should see something like this
    Product name="SAP DATA SERVICES" 
  • With in this you will find the steps that are enabled as shown below:
    <Step step_id="2." step_name="DS - Update DSConfig.txt for Data Service Cluster"  available="True" installation="SAP DATA SERVICES">
        <Property name="step_des" value="Update DSConfig.txt for Data Service Cluster"/>
        <Property name="step_doc" value="Reference Wiki"/>
        <Property name="step_doc_url" value="http://wiki.scn.sap.com/wiki/display/SMSETUP/Managed+system+setup+of+Data+Services+system+in+the+Solman+7.1"/>
        <Property name="class_name" value="com.sap.ac.transaction.steps.ds.UpdateDSCONFIG" />
    </Step>
  • available="true" is for the steps which are enabled. false is for the ones which are disabled.
    <Step step_id="3." step_name="DS - Enable Saposcol process monitoring"  available="True" installation="SAP DATA SERVICES">and
  • and
    <Step step_id="5." step_name="DS - Check DS SLD Registration"  available="True" installation="SAP DATA SERVICES">

Symptom

Is there any Impact on existing connections when generating a new client secret in the Learning OAuth token server ?

Environment

SAP SuccessFactors Learning

Reproducing the Issue

1] Go to Learning Administration side

2] Search for Configuration > OAuth Token Server

3] Click on "Generate New Client Secret Key"

Resolution

Generating a New Client secret will invalidate all previously Generated secrets.

Customer can Generate the key from the different admin account which is not used for the previous integrations.
So that it would not impact any previous integrations." - If the client secret generated at the Admin account level is being used somewhere and it's regenerated, it's going to be break the integration as well.

  • Connection between Learning Management System and BizX - This isn't impacted by the client secret regeneration or used by the client secret itself.
  • Connection between Learning Management System and IAS - This isn't impacted by the client secret regeneration or used by the client secret itself.
  • Connection related to Learning Management System and Story Reports - This isn't impacted by the client secret regeneration or used by the client secret itself.

The client secret is used with APIs and integrations that use the Learning APIs (POST, PUT, GET), but not the ones mentioned in the numbered list.

Symptom

Data Replication Proxy for Planned Working Times is created but subnode Planned Working times shows message:

  • "No data for Planned Working Times available or you do not have the necessary permission."

Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.

Environment

  • SAP SuccessFactors Employee Central Payroll

  • Point to Point Time Data Replication Process

Cause

This is the expected behavior of the system. 

Resolution

This is not caused by missing authorizations. The subnode Planned Working Time in the object Data Replication Proxy will never show any information. As a result, the Manage Data UI displays the message "No data for Planned Working Times available or you do not have the necessary permission."

The subnode is required in this object for technical reasons to allow the call to dynamically retrieve planned working time information during the query call of Data Replication Proxy.

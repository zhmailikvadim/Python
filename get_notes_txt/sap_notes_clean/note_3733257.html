Symptom

  • Joule is integrated with Microsoft 365 Copilot.
  • When sending a prompt in Copilot, it shows error: "SAP Joule does not seem to be set up for your tenant"

"Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental."

Environment

  • SAP Business Technology Platform for Cloud Foundry Environment
  • Joule AI copilot

Reproducing the Issue

  1. Integrate Joule with Microsoft 365 Copilot following document Integrating Joule with Microsoft 365 Copilot.
  2. Send a prompt and the error occurs.

Cause

Microsoft 365 Copilot is not added to BTP system and not included in Joule formation.

Resolution

Follow below steps as per document Integrating Joule with Microsoft 365 Copilot:

  1. Note down Microsoft M365 Tenant Id corresponding to productive Microsoft M365 Copilot/Teams deployment. This need to be provided during the Joule landscape setup. 

  2. In the BTP Cockpit, go to the system landscape and add a system of type M365 Copilot for SAP Joule. Enter the tenant ID noted down before.

  3. If you are setting up Joule for the first time you can activate the Microsoft 365 Copilot integration when running the Joule Booster.

    If you already have a running Joule setup, include the newly created system to the formation via the BTP cockpit.

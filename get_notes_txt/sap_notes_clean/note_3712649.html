Symptom

Password Reset email is not sent for SAML accounts with two identities: Site and SAML.

Therefore, users cannot sign in the with the Site identity.

Environment

  • SAP Customer Data Cloud
  • SAML

Reproducing the Issue

  1. Register a SAML account
  2. In CDC Admin console > Identity Access Profile, edit the new account e.g. change birth-year.
  3. Verify the account with two identities: SAML and Site.
  4. SAML identity has (allowLogin=true) and Site identity has (allowLogin=false).
  5. On CDC login screenset, click (Forgot Password?) link
  6. Enter the email address of the account to trigger password reset.
  7. Result: password reset request is successful but the password reset is not sent.

Cause

The issue is caused by the site policy settings for the "federation" parameter. When the "allowMultipleIdentities" setting is set to false, CDC does not trigger password reset email for SAML accounts. The default value is false.

Resolution

  1. Call the REST API endpoint `accounts.setPolicies` to update the site policy settings.
  2. Set the "federation" parameter to allow multiple identities by configuring `"allowMultipleIdentities": true`.

Example:

curl --request POST \
  --url https://accounts.us1.gigya.com/accounts.setPolicies \
  --header 'content-type: multipart/form-data' \
  --form apiKey=3_CkKYWkpM4xxxxxvNW64C1J \
  --form userKey=AGa \
  --form 'secret=mrrrH' \
  --form 'federation={
    "allowMultipleIdentities": true
}'

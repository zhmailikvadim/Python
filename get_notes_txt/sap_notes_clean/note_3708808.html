Symptom

 
Contact persons created with gender selection show incorrect synchronization where the legacy Gender field is set to 'E' instead of the selected gender value, while GenderEx field contains the correct value.
 

Reproducing the issue

Prerequisites:

SAP Business One FP 2508 or higher (Contact person functionality with GenderEx field available)

Scenario:

  1. Go to Business Partners → Business Partner Master Data
  2. Open an existing business partner or create a new one
  3. Navigate to the Contact Persons tab
  4. Fill in the following fields:
    • Contact ID Enter any value
    • Gender: Select for instance 'Male'
  5. Fill in any UDF (User Defined Field) or standard field on the contact tab (such as Title, Position, E-Mail, Phone, etc.)
  6. Click Add to save the contact person
  7. Check the OCPR table data for the created contact person record

Expected behavior:

Both Gender and GenderEx fields in the OCPR table should contain synchronized values that correspond to the gender selection made in the user interface. Gender should contain:

  • 'M' for Male or 'F' for Female when standard genders are selected
  • 'E' when gender is not specified
  • 'I' when gender is one of the genders defined by users in the Gender - Setup window

Actual behavior:

When any UDFs or fields on the contact tab are populated after selecting a Gender:
  • GenderEx field contains the correct value ('M', 'F', 'E', or user-defined gender code)
  • Gender field is incorrectly set to 'E' instead of the selected gender value

Additional Information:

When no UDFs or additional fields on the contact tab are populated after selecting a Gender, both fields synchronize correctly.

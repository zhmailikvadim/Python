Symptom

Within the app "Manage Production Orders"

  • the column progress of operation, the values displayed do not match to unit of measure defined within first operations.
  • after adding adding a value to standard filter the table filter does not work properly if custom filter like "Status: deselect ALL, Issue: No Filter, Delay: >=0 Hours" was used before.

Other Terms

Commons Library, Manage Production Orders App, F2336

Reason and Prerequisites

  • Different unit of measure in order header and first operation

Solution

Program correction

Workaround for filter issue:

Select all values in status filters and add the filter selection of the user and hit "Go". KBA KB0563806 - https://itsm.services.sap/kb?id=kb_article_view&sysparm_article=KB0563806

Symptom

  • In the new table experience, newly created tables based on planning models do not show the Month option under the Date dimension
  • Converted tables from the old table experience initially retain the Month row, but if Month is unchecked in the builder panel, it disappears and is no longer available to add back
  • In the model, the Date dimension’s default hierarchy includes Month
  • No error messages are displayed

Environment

  • SAP Analytics Cloud

Reproducing the Issue

  1. Open a story with a table based on a planning (import) model in the old table experience and go to Edit
  2. Select the table and open the Builder panel; verify that Month is available under Rows (Date hierarchy)
  3. Convert the story/table to the new table experience and open the same table’s Builder panel; Month is available
  4. Uncheck Month from Rows
  5. Observe that Month disappears and is no longer offered as a selectable option under the Date dimension
  6. Copying an existing table that still has Month will show the Month is available 
  7. Create a new table on the same model in the new table experience
  8. Open the Builder panel; under Date, Month is not available as an option

Cause

  • The cause is under investigation by Development 

Resolution

  • The solution of this issue is currently under investigation by Development

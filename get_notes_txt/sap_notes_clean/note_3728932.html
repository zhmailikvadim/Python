Symptom

BAPI_INSPOPER_GETDETAIL returns a non‑initial quantity for an inspection point although the corresponding database field is initial (INSPPOINTS‑QUANTITY vs. QAPP‑MENGE). The behavior reproduces when comparing the BAPI output with table QAPP for the same inspection point.

Other Terms

BAPI_INSPOPER_GETDETAIL, QAPP, INSPPOINTS, MENGE, QUANTITY, lcl_inspection_point, LIF_INSPECTION_POINT~GET_LIST

Reason and Prerequisites

The reason is a program error. A missing CLEAR of the inspection point structure at the start of the loop over LT_QAPP in method lcl_inspection_point=>get_instance()->get_list causes values to persist between iterations, which leads to inconsistent INSPPOINTS‑QUANTITY when QAPP‑MENGE is initial.

Solution

Implement correction insturction.

Symptom

  • Accounting integration between a Project Management project and a Project System project fails.
  • Transaction COCPCPR (Controlling Cockpit) shows error message "Object PSP-Element XXX could not be changed" (Message no. CNIF_PI018).

Environment

  • SAP Portfolio and Project Management (PPM)
  • SAP Portfolio and Project Management for SAP S/4HANA (PPM)
  • SAP Portfolio and Project Management in SAP S/4HANA (PPM)

Reproducing the Issue

  1. Accounting integration between Project Management and Project System is set up on your system and works.
  2. Your PPM project has status replicated and transferred to PS.
  3. You carry out some change in the PPM project and save the project.
  4. You notice that accounting integration for this project fails.
  5. You check transaction COCPCPR to identify the cause of the failed integration. Here you can see that error message "Object PSP-Element XXX could not be changed" is shown for your project. This error message however is generic and does not inform you of the root cause.

Cause

  • One possible reason for this system behaviour is that the duration of some project element (e.g. task) which should be replicated to Project System is too long and causes a program overflow during scheduling in Project System.
  • The underlying restriction and root cause is on Project System (PS) side, as the issue occurs within the processing of the BAPI to update the PS project. The limitation concerning the duration of PS projects reflects a fundamental restriction in ERP, which concerns not only PS, and which is documented e.g. in SAP note 49257. The restriction can easily be seen by e.g. manually setting a finish date past year 2048 for a project in transaction CJ20N: this will lead to an error message CJ012 "Finish date must be earlier than 19.01.2048".

Resolution

Ensure that the project elements in the PPM project (e.g. tasks, roles, phases) do not have durations which exceed the aforementioned limitation (i.e. the finish date of which is past year 2048). Afterwards schedule the project before saving it.

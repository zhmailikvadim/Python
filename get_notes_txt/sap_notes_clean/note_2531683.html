Symptom

  • Relay Server Outbound Enabler log shows:

    E. 2017-08-31 05:48:16.483-0300 <Backend-0001> OEE1045: System error [SYS0: no system error] while interrupting send with the back-end server for sidx=1026 socket=049bf630 sfp=abed40cf

Environment

  • RSOE version 16 and above
  • Relay Server version 16 and above

Cause

The primary cause of error  'OEE_BACKEND_IO_SYS_ERROR' is connectivity problems to the backend-end server that lead to failed socket operations.
Here are some examples:

  1. An async socket Send() operation has failed (to actually place the send bytes request).
  2. An async socket Receive() operation has failed (to actually place the receive bytes request).
  3. An async socket Receive() has completed but with error status.
  4. A pending async socket Receive() (after completing successfully at ‎‎#2) was interrupted.
  5. An async socket Send() has completed but with error status.
  6. A pending async socket Send() (after completing successfully at  #1) was interrupted.
  7. A pending async socket Close() came back with an error.

It is also possible that an OEE_BACKEND_IO_SYS_ERROR could trigger an OEE_SESSION_ACCESS_FAILED.

Resolution

  1. Identify the root cause by whatever tools are available.  Once the root cause is understood a specific resolution can be found.

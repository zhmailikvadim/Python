Symptom

  • A Freight Order is created in SAP Transportation Management (TM) containing items marked as Dangerous Goods.
  • During the tendering process, the Freight Order is sent to the BN4L system via the TransportOrderRequest_Out XML message (or via TransportationOrderBookingRequest_Out if a Freight Booking is used).
  • In the BN4L system, under the Freight Order details, the Dangerous Goods indicator is displayed as “Yes”. However, when clicking the hyperlink to view the Dangerous Goods information, a blank screen is displayed instead of the Dangerous Goods details.

Environment

  • SAP Transportation Management 9.1 - 9.6
  • SAP Transportation Management embedded in S/4HANA 1709 and subsequent releases

Reproducing the Issue

  1. In the TM system, create a Freight Order containing items marked as Dangerous Goods
  2. In the Freight Order UI, choose "Subcontracting -> Send to Carrier"
  3. The TransportOrderRequest_Out XML message is generated and sent from TM to the BN4L system
  4. In the BN4L system, open the Freight Order
  5. Under the Freight Order details, check the Dangerous Goods indicated as "Yes"
  6. Click the Dangerous Goods hyperlink
  7. A blank screen is displayed instead of the Dangerous Goods information

Cause

  • The TextCollection section, which is responsible for transferring the Dangerous Goods information from the TM system to the BN4L system, is missing or empty in the TransportOrderRequest_Out XML message.
  • If the TextCollection is not properly populated in the XML, the Dangerous Goods information cannot be displayed in the BN4L system.
  • Even when the Dangerous Goods details are missing, the Freight Order in BN4L will still show Dangerous Goods = Yes, because the XML contains the field DangerousGoodsIndicator = true, indicating that the shipment contains Dangerous Goods.
  • To correctly populate the TextCollection section and enable the display of Dangerous Goods information in BN4L, certain customizing steps must be maintained in the TM system.

Resolution

  • 1. Maintain EHS Environment Parameter:
    • First, verify that the EHS environment parameter DG_PRINT_STRUCTURE is created and maintained with the parameter value RDGPRINT. Customizing paths:
      • SAP TM Standalone: SPRO -> SAP Transportation Management -> SCM Basis -> EH&S Services -> Basic Services -> Specify Environment Parameters
      • SAP TM embedded in S/4HANA: SPRO -> Transportation Management -> Basic Functions -> Dangerous Goods -> Dangerous Goods Processing Based On PS&S -> PS&S Services -> Basic Services -> Specify Environment Parameters
    • For detailed configuration instructions, refer to the following KBA:
      • KBA 2736311 - TM Outbound XML TextCollection is missing Proper Shipping Name and other texts
  • 2. Maintain Languages for Dangerous Goods Regulations
    • In TM, the Dangerous Goods data used for B2B mapping is retrieved through print function modules, which require languages to be assigned to the regulations.
    • In addition to the EHS environment parameter, ensure that languages are maintained for the Dangerous Goods Regulations used in the Freight Order. Customizing paths:
      • SAP TM Standalone: SPRO -> SAP Transportation Management -> SCM Basis -> EH&S Services -> Dangerous Goods Management -> Dangerous Goods Checks and Dangerous Goods Documents -> Dangerous Goods Documents -> Specify Languages
      • SAP TM embedded in S/4HANA: SPRO -> Transportation Management -> Basic Functions -> Dangerous Goods -> Dangerous Goods Processing Based On PS&S -> PS&S Services -> Dangerous Goods Management -> Dangerous Goods Checks and Dangerous Goods Documents -> Dangerous Goods Documents -> Specify Languages
  • After maintaining both configurations, trigger the XML message again from the Freight Order and verify that the TextCollection section is present and populated with the Dangerous Goods information in the XML.
  • How to Proceed if the Issue Persists:
    • If the TextCollection section is still missing in the XML, create a case with SAP Transportation Management Support under component TM-CF-DG.
    • If the TextCollection is present in the XML, but the Dangerous Goods information is still not displayed in BN4L, create a case with SAP Business Network for Logistics Support under component SBN-LBN-FC-FOC.

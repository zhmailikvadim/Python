Symptom

For purchase order after creating a new version and completing the release process, the fields “Released By” and “Released On” remains blank in the Version tab. 

Environment

Materials Management (MM)
SAP ERP Central Component
SAP ERP
SAP enhancement package for SAP ERP
SAP enhancement package for SAP ERP, version for SAP HANA
SAP S/4HANA

Reproducing the Issue

1. Open PO in ME23N.
2. Check versions tab.
3. New version triggered with value change 0.00.
4. Check “Released By” and “Released On” fields remain blank.

Cause

Standard system Design.

Resolution

After creation of a new version and completing the release process for purchase order, again when you change the quantity or value, so that new version was triggered. Then for new version you mark completed indicator and click on check so new release strategy was triggered. Then again you change the quantity or value to last quantity or value, so new version was triggered with value change 0 and new release strategy which was determined was also changed to old was which was already released. So there is no new release strategy to approve and that's why the fields “Released By” and “Released On” remain blank in the Version tab. This is standard system behavior of SAP. There is no change log generated in Header changes for purchase order.

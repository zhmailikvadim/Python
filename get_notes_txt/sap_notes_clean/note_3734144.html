Symptom

  • After entering an installation or profile number in transaction EEDM02, only a limited portion of the data is visible.
  • Selecting the "Expand area" button increases the visible area and allows viewing more details.
  • When leaving EEDM02 and starting it again, the previously collapsed area is not retained; the user must select "Expand area" again.
  • In addition, it is not possible to set default values to be used for seach

Environment

  • SAP S/4HANA, release IS-UT 801 and higher
  • SAP for Utilities as Part of SAP ERP, release IS-UT 600 and higher

Reproducing the Issue

  1. Start transaction EEDM02.
  2. Enter an installation or profile number and press Enter.
  3. Observe that only a limited portion of the data is visible.
  4. Select the "Expand area" button to extend the working area and view more details.
  5. Exit EEDM02 and start it again.
  6. Observe that the collapsed state is not retained and the area must be collapsed again.

Cause

  • This transaction is designed to handle complex operations and is categorized as a multi-level transaction.
  • Initially, users must select the required data in a dedicated area.
  • After data selection, a tree structure is generated based on the user's input.
  • Users can double-click on specific items within the tree to view detailed information about the selected data.

Resolution

  1. Restart the transaction to display the data selection area again.
  2. Select the necessary data to ensure the tree structure is populated.
  3. Expand the tree structure to explore further details.
  4. Double- click on profile numbers within the tree to access their specific details.

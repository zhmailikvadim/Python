Symptom

When you execute the report ELStAM PHI: Process Notifications (RP_PAYDE_E2_PKV_PROCESS_NOTIFS), PHI data for persons with a registered work relationship is automatically imported to the infotype SI Additional Insurance (0079).

Solution

Implement the attached correction instructions or import the specified support package. After you import the HR Support Package or implement the correction instructions, you do not have to perform any further follow-on activities.

After you implement the corrections, the data is no longer automatically transferred to IT79. Instead, the system issues one of the following warnings:

"IT20 (<BEGDA>-<ENDDA>) maintained as HI/CI exempt with person group key 190" (HRPAYDE_E2_PKV 036)

"IT20 (<BEGDA>-<ENDDA>) maintained as mini jobber with person group key 109" (HRPAYDE_E2_PKV 053)

The notification is not included in the worklist for processing as a check case. The notification is processed without updating the infotype SI Supplementary Insurance (0079) and the status is set to Processed  accordingly. This means that the notification can only be displayed in the administrator list by choosing Display All Notifications. The system issues a corresponding warning message in the log of RP_PAYDE_E2_PKV_PROCESS_NOTIFS and an entry in the Notification Tool (NT). If the entry is not required in NT, it can be controlled using the general NT function.

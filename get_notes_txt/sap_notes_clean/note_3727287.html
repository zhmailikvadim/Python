Symptom

Logout audit logs timestamp is not being logged when a user receives a session timeout on the system.

Environment

  • SAP Sales Performance Management
  • Incentive Management

Reproducing the Issue

  1. Log into the system.
  2. Wait for the system to log out due to inactivity.
  3. Check the Audit Logs workspace in Incentive Management Admin.
  4. Observe that no audit logs are displayed for a "logout" event when the user is logged out due to session timeout.

Cause

This is a Known Issue.

Resolution

The fix for this issue is planned to be deployed with the update 2605. For update timelines, please review SAP Community - SAP SPM 2026 Release Schedule

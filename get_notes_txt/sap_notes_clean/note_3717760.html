Symptom

Even though the user is restricted to access a particular job profile through RBP Manage Job profile and skill visibility >> View Permission, the job profile link shows up in the people profile and when the user clicks it, it keeps on loading.

Environment

  • SAP SuccessFactors Job Profile Builder

  • SAP SuccessFactors Employee Profile

Reproducing the Issue

  1. Proxy a user with restricted job profile view access to specific job families.
  2. Navigate to People Profile > Personal Data > Employee Information > Employment Information page 
  3. Observe that the job profile link is visible but on clicking continues to load indefinitely without displaying data.

Cause

System design: The "Job Profile" link in the People Profile remains visible for users with "Employee Data >> Job Code >> View" permission and "MDF Foundation Objects >> Object: Position >> View Current" permission, even if the job profile view is restricted by role.family.

Resolution

This is the expected behaviour even when you restrict the Job profile view access through object level permission, since user has User Permissions >>Employee Data>>Job Code>>View permission and User Permissions >>MDF Foundation Objects >>Object: Position>>View Current permission , the Job profile link will always show up in the employee profile .It will always be a clickable link and since the job profile view is restricted by role.family in your case, job profile is not loading.

If this behavior is considered an important enhancement for business operations, share the idea in the SAP Community for consideration as a future enhancement. Refer to: KBA 2090228 - How to submit enhancement requests

alternatively customers can configure employee profile blocks with job code and job profile to provide access only to admin users, ensuring the employee profile block is not accessible to all users.

Utilize the option of ad hoc reporting to fetch details on job profiles as needed.

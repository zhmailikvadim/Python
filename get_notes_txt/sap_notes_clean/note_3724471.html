Symptom

After publishing a new consent locale e.g English with a new document URL in consent setup, production website still shows the old consent document link.

Environment

  • SAP Customer Data Cloud
  • Screen sets
  • Consent - Enterprise Preference Manager 

Reproducing the Issue

  1. Create a new consent with a published locale English
  2. Add a link of a consent document in the Document URL field.
  3. On CDC registration screen, add a checkbox
  4. In the label field, include a link to the consent document URL
  5. Example: <a href="{{schema.preferencesSchema.fields['privacy.Terms'].legalStatements[screenset.lang].documentUrl }}" target="_blank">View Consent</a>
  6. Load the registration screen to verify the correct document URL
  7. In the same consent setup, publish a new locale version with a new document URL
  8. Load the registration screen
  9. Result: it loads the old document URL

Cause

The localization file (English) of the screen set contains the wrong consent document URL in the customization column.

Resolution

  1. Download the localization file (English) in the screen set.
  2. Open the localization file .csv and find the exact checkbox field on the registration screen
  3. Under the customization column (F), verify the correct URL.
  4. If the link under column F is not correct, update it with the correct link
  5. Check the URL to verify the correct the incorrect consent document URL.
  6. Save the changes to the localization file.
  7. Upload the new localization .csv in the screen set
  8. Save the screen set

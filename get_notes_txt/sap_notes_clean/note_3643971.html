Symptom

  • Attempting to delete a time account type from a time type results in the error message: "You cannot delete the time account type because time accounts exist for it."
  • The deletion process is blocked due to existing time accounts associated with the time account type.

Environment

SAP SuccessFactors Employee Central Time Off

Reproducing the Issue

  1. Navigate to Manage Data/Time Off Structures.
  2. Select a Time Type.
  3. Attempt to delete the associated Time Account Type.
  4. Observe the error message: "You cannot delete the time account type because time accounts exist for it."

Cause

The system enforces a hardcoded validation that prevents the deletion of a time account type when there are existing time accounts associated with it.

Resolution

In order to delete/remove the Time Account Type, the associated Time Accounts would need to be deleted. This approach would not be recommended as you would be losing historical data, however if required can be done with the following steps:. 

  1. Navigate to Import and Export > Export > Time Account. 
  2. Select Yes for Include Inactive Records and export the data.
  3. Open the exported file and filter for the specific time account type.
  4. Delete all time accounts associated with the time account type using either: 
  5. Time Account import functionality, or
  6. Manage Data > Time Account in the UI. 

    Workaround:

    1. Create a new Time Type and associated the new Time Account Type. 
    2. Utilise the effective dated time profile feature > Insert New Record.
    3. Add the new time type and replace the old one.

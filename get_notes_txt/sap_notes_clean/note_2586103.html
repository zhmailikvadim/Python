Symptom

When you create or change an Ingresos adicionales (0090) infotype entry, the system is displaying a list of keys and subkeys that contains entries that are not valid anymore and missing some of the new keys.

Other Terms

&190; &IRPF (further information see note 1071282)

Reason and Prerequisites

The issue is caused by a program error when filtering the entries that are not valid anymore and due to missing customizing.

The following Support Package level is the minimum prerequisite:

Release Support Package
6.04 HR SP 65
6.00 HR SP 99

(For more information, click on "R/3 HR Support Package Schedule" in www.service.sap.com/ocs-schedules.)

Solution

Corrections to source code of 0090 infotype have been provided to solve this issue.

New entries are being delivered for the Claves y subclaves de percepción (V_T5E41) and Claves y subclaves de percepción - infotipo 0090 (V_5E41_B) views to address the keys that are not valid anymore and the ones where the IT90 (V_5E41_B-INF90) field was not flagged.

The attached files of .mhtml extension can be opened using an internet browser or Excel.

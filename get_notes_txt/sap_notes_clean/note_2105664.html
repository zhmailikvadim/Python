Symptom

  • Installed SMP 3.0 SP4
    Using Oracle as a custom Persistence Database.
  • Everything works as expected, but large amount of database connections are being made to this remote custom ( Oracle11 ) database.
  • These connections seem to do nothing but as they occur more than once every second the amount is huge.
  • See below for an example of the logfile.
  • Please note that the OS user SMPServiceUser is created manually before the installation of SMP using the Sybase instructions in the install docs ( or help pages ).
  • What can be done about this ?

Environment

  • SMP 3.0 SP4, SAP Mobile Platform
  • Oracle 11

Cause

  • This works as designed for the cluster sync / live check purposes.
  • SMP Engineering implemented clustering in SMP 3.0 SP03.
    The state of the cluster nodes is stored in the database.
     
  • They tested the performance of this implementation and did not notice any issues.
    Also, we are not aware of any resource leaks with this implementation.

Resolution

  • This has been resolved in the SMP 3.0 SP05 Release of SMP Server
    and you will not see this issue anymore.

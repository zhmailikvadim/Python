Symptom

This KBA introduces on how to remove unexpected infotype records from PAxxxx tables. It is important to note that infotypes or subtypes with a time constraint of '1' or 'A' are not covered within the scope of this KBA. "Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental."

Environment

  • Human Capital Management (HCM)
  • SAP ERP
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for HANA
  • SAP S/4HANA
  • SAP S/4HANA Cloud Private Edition
  • SAP SuccessFactors Employee Central Payroll

Resolution

RPUREOPN can be used to delete infotype records. 

Processing modes:

Report program RPUREOPN has three different processing modes:

  • Test:
    No changes are made to the database. 
  • Delete:
    Infotype records are deleted when program is run.
  • Flag:
    Records that are flagged are changed on the database. Field HISTO in PAnnnn table is set to ‘A', which means the record cannot be modified anymore.

Limitations:

  • Infotypes or subtypes with time constraint '1' or 'A' cannot be deleted using this report. For instance, infotypes like IT0000 and IT0001 cannot be removed.

  •  Only infotypes with a validity period ending on or before the 'Date of reorganization' can be deleted. It is not possible to selectively delete a record that sits in the middle of a sequence. 
     
    In the following scenario, it is not possible to delete only the record (01.02.2026 ~ 28.02.2026).
    1. 01.01.2025 ~ 31.01.2025
    2. 01.02.2026 ~ 28.02.2026
    3. 01.03.2026 ~ 31.01.2026

Steps for Deletion:

  1. Run t-code SE38 to execute report RPUREOPN.
  2. Test Run: Fille in required details for deletion. It is strongly recommended to perform a Test run before executing a live deletion. 
    Mode: Set it to 'TEST'

  3. Click the 'Execute' button.
  4. Check if the result is expected in the report log.
  5. Live run: Fill in all required fields. Please pay special attention to following fields:
    Mode: Set it to 'DELE'
    Password: Enter the username of the individual performing the deletion. 
  6. Click the 'Execute' button.
  7. Observe that infotype records are deleted successfully.

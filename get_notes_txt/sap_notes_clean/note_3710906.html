Symptom

  • After a client copy, Standard Area InfoSets appear with status “Modified” in SQ02 and Standard-area user groups are missing.
  • Attempting to generate/activate InfoSets fails with the below information:

Environment

SAP Query

Reproducing the Issue

  1. Run SQ02.
  2. Change Query Area to Standard Area.
  3. Observe that all InfoSets are in Modified status.
  4. Attempt to generate/activate an InfoSet - error occurs.

Cause

The table AQLDB (which stores Standard Area SAP Query objects: InfoSets, queries and user groups) was excluded during the client creation (or its entries were corrupted), so the Standard-area objects are absent or inconsistent.

Resolution

  1. Identify a source client/system where Standard Area objects are intact.
  2. Transport the Standard Area objects (Queries, InfoSets and User Groups) from that system into the affected client.

For detailed information about how to perform the transport, refer to the link  Transporting Standard Area Objects | SAP Help Portal

Symptom

  • Inquiry regarding whether the data provisioning (DP) agent is compliant with federal information processing standards (FIPS).
  • Request for clarification on whether compliance is for FIPS 140-2, FIPS 140-3, or other standards.

Other Terms

Keywords
fips compliance, dp agent, hana smart data integration, fips 140-2, fips 140-3, commoncryptolib, jdbc driver, bouncy castle fips, sapmachine 17, adapters, secure communication, external communication, federal information processing standards, hana client, operating system configuration
Environment
SAP HANA Smart Data Integration

Solution

SDI Data Provisioning Agent (DPAgent) is FIPS compliant with prerequisites below

  1. SAP HANA is configured to be FIPS-compliant. Refer to SAP HANA Platform Documentation, also SAP note 2180024
  2. FIPS mode is enabled on the operating system hosting the DP agent.
  3. DP agent is configured to be FIPS compliant.
    a) Register DP agent with SAP HANA using JDBC mode. Note that DP agent version 2.8.2.x uses SAP HANA JDBC driver version 2.27.6, which supports FIPS compliance ever since JDBC driver version 2.24. Refer to SAP HANA Client Documentation,FIPS 140 Compliance Documentation
    b) DPAgent version higher than 2.8.2 uses SAPMachine 17 (a flavor of OpenJDK), users need to configure the java.security file to use FIPS-validated security provider like Bouncy Castle FIPS (BCFIPS)
  4. Ensure DP agent's adapters use FIPS-compliant JDBC drivers to connect to source systems. Compatibility of JDBC drivers is published in the Product Availability Matrix (PAM). Refer to: PAM for SAP HANA SDI. FIPS compliant drivers are declared by whoever published the drivers and outside the scope for DP Agent

For the following sources, the FIPS compliance is detailed in references below

HANA JDBC driver:

The HDB_FIPS_MODE environment variable will activate the FIPS support for the selected crypto provider, users should ensure the latest version of the provider has been installed to get FIPS 140-3 support.

The JDBC driver uses the FIPS support built into the JVM that it runs in. SapMachine uses CCL for its crypto support which is FIPS 140-3 certified with version 8.6.

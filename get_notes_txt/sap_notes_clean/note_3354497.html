Symptom

You have replicated Business partner relevant locations using DRFOUT transaction. Any changes to the Business partner address is not copied to the Location.

To address this issue ,report /SAPAPO/UPDATE_LOC_ADDRNR can be used which maps the address numbers of such locations to the BP address numbers.

Other Terms

DRF Replication, Update BP Location address numbers.

Reason and Prerequisites

Program error.

Solution

Please implement the correction instructions in this note or upgrade to relevant Support pack.

When performing changes on a condition group it can happen that during the check for backdated changes a wrong change date is determined. As a result, the changes might wrongly be prevented, claiming that your changes reach beyond the date threshold until when backdated changes are allowed. Indeed, your changes should be allowed, but you are being presented ther error messages PRI_PPL_AL 056 (" ") and BCA_BACKDATED_CHANGE 050.

Reason and Prerequisites

Program error

Solution

Import the relevant Support Package for your release. You can use Transaction SNOTE to implement the following correction instructions in advance. There are no manual activities necessary.

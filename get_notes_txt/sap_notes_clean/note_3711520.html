Symptom

There is a need to set unique dummy emails in target system for users from SAP Application Server ABAP system.

Environment

  • Identity Provisioning

  • Identity Authentication

Resolution

  1. Go to IPS administration console->Source systems->Select source system for SAP Application Server ABAP->go to the Transformations tab->Switch to the JSON editor by choosing the code-bracket icon->Edit.
  2. If username from AS ABAP should be used to generate dummy email, add the following to the user mapping. Note, you should keep the default email mapping for users already with unique emails:

                {
                    "sourcePath": "$.ADDRESS.E_MAIL",
                    "targetPath": "$.emails[0].value",
                    "optional": true,
                    "correlationAttribute": true
                },
                {
                    "sourcePath": "$.USERNAME",
                    "targetPath": "$.emails[0].value",
                    "condition": "$.ADDRESS.E_MAIL EMPTY true",
                    "functions": [
                        {
                            "function": "concatString",
                            "suffix": "@sap-test.de"
                        }
                    ],
                    "correlationAttribute": true
                },
                {
                    "targetPath": "$.emails[0].primary",
                    "constant": true
                },
                {
                    "targetPath": "$.emails[0].type",
                    "constant": "work"
                },

NOTE:  @sap-test.de is the recommended test domain approved by SAP for customer dummy (testing) requests. Thus, please use only this one for the concatString function! If you use another non-existing domain, it might get blocked for security reasons.

Symptom

  •  In a workspace, certain parts of objects are not visible to the user

Environment

  • SAP Analytics Cloud
  • Workspace

Reproducing the Issue

  1. Log in to the SAP Analytics Cloud tenant with an Admin role.
  2. Create a folder in the public folder as an Admin user.
  3. Move or create a folder/story using one of the following methods:
  • Method 1: Navigate to "My Files" → "Public" → Select the public folder → Move to Workspace → Share the folder with a user with Full Control rights → The user requires private file rights to view the folder.
  • Method 2: Navigate to "Files" → "System" → Select the public folder → Move to Workspace → Share the folder with a user with Full Control rights → The user requires only public file rights to view the folder.
  • Method 3: In workspace → Create a folder directly → The user requires private file rights to view the folder.

Cause

  • The issue occurs due to the behavior of folder permissions in SAP Analytics Cloud, which depend on the location where the folder is created and how it is moved to the workspace.

Resolution

  • Moving folders from My Files to a workspace makes it still private content under the workspace, thus requiring private file privileges to access it for other users.
  • Moving folders from System view does not have this restriction as it will be accessible regardless of who moved it. So if your customer's intention is to allow access for everyone, then move it using the System view. 
  • Creating folders from Workspace is also requiring private file privileges to access it for other users

Symptom

When updating the settings for a notification type in the Fiori Launchpad (FLP) and choosing Save, an error occurs stating that the "Property DoNotDeliverEmail is missing". As a result, the settings are not saved successfully.

Other Terms

Property "DoNotDeliverEmail" is invalid. FLP settings update not successful.

Reason and Prerequisites

The property DoNotDeliverEmail is not supported in SAP systems running SAP_GWFND release lower than 754. In these releases, the OData service /IWNGW/NOTIFICATION_SRV does not expose the DoNotDeliverEmail property in its metadata.

The issue is caused by a release mismatch between the frontend and backend systems when the SAP_UI component is on a release starting 754, while the backend SAP_GWFND release is lower than 754.

While updating the notification settings in FLP, the frontend sends an update request containing the property DoNotDeliverEmail, which is only supported in higher releases. Since this property is not available in the lower backend release(<754), the update request fails.

Solution

 Kinldy implement this note.

Symptom

  • Fixed asset master data was migrated to the production system using the legacy data transfer feature in the Make Company Code Settings - Asset Accounting-Specific app.
  • The system does not allow changing the transfer date to another date, because legacy assets are already in the system.

Environment

SAP S/4HANA Public Cloud

Reproducing the Issue

  1. Migrate fixed asset master data to the production system using the legacy data transfer feature in the Make Company Code Settings - Asset Accounting-Specific app.
  2. Attempt to change the legacy transfer date in the Transfer Settings.
  3. Observe that the system does not allow the change due to legacy assets already existing in the system.
  4. When trying to use `Schedule Accounting Data Changes` app with Reset Transaction Data job template, an error occurs: You can execute the job in any system except in a production system.

Cause

Only use the job template in the quality management system (Q system), and not in the production system (P system).

Resolution

If having transferred legacy data with errors to the Q system, either manually or using the SAP S/4HANA migration cockpit, it is possible to completely delete this incorrect data.

To do so, use the Reset Transaction Data job template from the app Schedule Accounting Data Corrections.

Use this job template to delete master data and transaction data from Asset Accounting, along with other accounting data.

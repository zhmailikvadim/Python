Symptom

Prices are not updated in SAP Backend System like S/4HANA when replicating order from SAP Commerce via CPI using async price.

Environment

  • SAP Commerce Cloud
  • SAP CPI
  • S/4HANA

Reproducing the Issue

  1. Submit an order in the storefront
  2. sap-order-process will replicate the order from Commerce to CPI to S/4HANA
  3. Order is properly created but prices are updated.

Cause

Missing COND_TYPE in E1BPCOND segment of IDOC SALESORDER_CREATEFROMDAT204 sent to S/4HANA caused the prices not updated. See IDOC mapping in the iflow Replicate-Orders-From-SAP-Commerce-Cloud-To-SAP-S4HANA.

This is caused by missing conditionCode in sapCpiOutboundPriceComponents of the payload sent from Commerce like below.
"sapCpiOutboundPriceComponents": [
{
"conditionCounter": "1",
"orderId": "123456",
"absolute": "true",
"entryNumber": "-1",
"currencyIsoCode": "USD",
"value": "9.99",
"integrationKey": "-1|123456"
},
{
"conditionCounter": "25",
"orderId": "123456",
"absolute": "false",
"entryNumber": "0",
"value": "0.0",
"conditionCode": "MWST",
"integrationKey": "0|123456"
},
{
"conditionCounter": "2",
"orderId": "123456",
"absolute": "true",
"entryNumber": "-1",
"currencyIsoCode": "USD",
"value": "0.0",
"integrationKey": "-1|123456"
},
{
"conditionCounter": "3",
"priceQuantity": "1.0",
"unit": "PCE",
"orderId": "123456",
"absolute": "true",
"entryNumber": "0",
"currencyIsoCode": "USD",
"value": "28.0",
"integrationKey": "0|123456"
}
],
While using asynchronous price, during the order conversion, each price component will be created from the order entries and the conditionCode is set from corresponding SAPConfiguration - PRICING CONDITION TYPES FOR SALES ORDERS associated with the order.

In this case, empty PRICING CONDITION TYPES FOR SALES ORDERS will lead to missing conditionCode in each price component.


Resolution

Please follow Mapping Price and Cost Values to map price components to condition types in the SAP back end (SAP S/4HANA, SAP ERP, or SAP CRM). Mapping makes it possible for price components to be replicated with sales orders from SAP Commerce Cloud to the SAP back end via CPI or DataHub.

Symptom

When you execute the report SI: Get Data from SI Agencies  (RP_PAYDE_B2A_SV_COLLECTOR), the system issues the following error message:

"No valid certificate BNRAG <BNRAG> for confirming non-confirmed data" (HRPAYDEB2A 111).

Reason and Prerequisites

You start the report in parallel for different data providers. The retrieved data packages are written to the same database table by both runs before they are confirmed. After you confirm the pickup in each run, the system checks whether there are still messages in the database that have not yet been confirmed. In this case, the data packages that have not yet been confirmed are selected from the parallel report run and are tried to confirm. However, since this data belongs to a different data provider, the system issues an error message.

Solution

Implement the attached correction instructions or import the specified support package. After you import the HR Support Package or implement the correction instructions, you do not have to perform any further follow-on activities.

After you implement the corrections, only those data packages that have not yet been confirmed are selected in each report run that match the data providers of the selection screen. As a result, the system no longer issues an unjustified error message for parallel runs.

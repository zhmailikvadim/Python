Symptom

Selective (delta) deletion of invoice data is not supported — Delete removes the entire extracted invoice dataset .

Other Terms

 Spend Control Tower .

Reason and Prerequisites

Selective (delta) deletion of invoice data was not possible because this functionality was not supported by the design of the delete load process.

Solution

Implement the NOTE 3711638.

Additional steps for implementing NOTE 3711638:

  1. For SAP_BASIS Release 740 SP up to 20:
    • It is essential to implement Correction Instruction 1765378, which includes the UDO report required for the manual implementation of all non-supported Report Texts.
    • Execute the UDO Report 'NOTE_3711638_RT' through Transaction Code SE38.
    • Ensure to implement other applicable Correction Instructions.

  2. For SAP_BASIS Release 750 SP up to 14:
    • It is essential to implement Correction Instruction 1765380, which includes the UDO report required for the manual implementation of all non-supported Report Texts.
    • Execute the UDO Report 'NOTE_3711638_RT' through Transaction Code SE38.
    • Ensure to implement other applicable Correction Instructions.

Now you can restrict deletion using:

  • Document Number

  • Company Code

  • Fiscal Year

  • Invoice Posting Date Range

  • Accounting Document Type

  • Supplier

Using one or more of these filters ensures only targeted data is deleted.

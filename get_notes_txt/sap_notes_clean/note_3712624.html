Symptom

  • HTTP Status 404 – Not Found error occurs when attempting to access the URLs /BOE/BI and /BOE/CMC URLs.
  • Springsaml.log indicate parsing errors in the `securityContext.xml` file, including malformed XML.
    • ERROR ContextLoader:299 - Context initialization failed
      org.springframework.beans.factory.xml.XmlBeanDefinitionStoreException: Line 86 in XML document from ServletContext resource [/WEB-INF/securityContext.xml] is invalid; nested exception is org.xml.sax.SAXParseException; lineNumber: 86; columnNumber: 52; cvc-complex-type.2.4.a: Invalid content was found starting with element 'property'. One of '{"http://www.springframework.org/schema/beans":import, "http://www.springframework.org/schema/beans":alias, "http://www.springframework.org/schema/beans":bean, WC[##other:"http://www.springframework.org/schema/beans"], "http://www.springframework.org/schema/beans":beans}' is expected.
    • Caused by: org.xml.sax.SAXParseException; lineNumber: 86; columnNumber: 52; cvc-complex-type.2.4.a: Invalid content was found starting with element 'property'. One of '{"http://www.springframework.org/schema/beans":import, "http://www.springframework.org/schema/beans":alias, "http://www.springframework.org/schema/beans":bean, WC[##other:"http://www.springframework.org/schema/beans"], "http://www.springframework.org/schema/beans":beans}' is expected.

Environment

  • SAP BusinessObjects Business Intelligence platform 4.3/2025
  • SAML

Reproducing the Issue

  1. Configure SSO/SAML/IDP on server according to KBA  3559381 - Configuring Trusted Authentication in 4.3 SP04 Patch 1000+ & BI 2025 - SAP for Me 
  2. Attempt to access the URLs /BOE/BI or /BOE/CMC
  3. Get HTTP 404 error

Cause

Syntax errors in the securityContext.xml file, including improperly closed tags and incorrect commenting syntax.

Resolution

  1. Correct the syntax errors in the securityContext.xml file
  2. Ensure tags are properly closed. For example
  3. Restart Tomcat and monitor logs for any remaining errors.
  4. You can also ask for a default securitycontext.xml file for SAP Support to compare

Note: If the issue persists, test accessing the URLs with "https://hostname:8443/BOE/BI/logonNoSso.jsp" to bypass SSO and verify if the problem is related to SAML configuration or deployment.

Symptom

Customers schedule matching run jobs regularly in their system. They want the period in the Filter string to dynamically change during the ICMR matching run.

For example: 

Current period is 003, they want the filter string is FISCYEARPER <='2026003'. And in the next period, the filter string will change to FISCYEARPER <='2026004' automatically.

Environment

  • Group Reporting
  • SAP S/4HANA
  • SAP S/4HANA Cloud Private Edition

Reproducing the Issue

How to request

Resolution

To run matching for each period at your specified day of the period, you can schedule monthly jobs. In doing so, you need to exclude date-related filters, such as FISCYEARPER (Fiscal Year Period), RYEAR (Fiscal Year), and POPER (Posting Period), from the filter string. (Please also don't make Filter string as BLANK) Then the values of these fields will be automatically determined based on current date and fiscal year variant.

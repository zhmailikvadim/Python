Symptom

  • Oracle Database has Extended Varchar feature enabled 
  •  SAP Data Services is not loading more than 4000 varchar2 characters when using a case transform in the Job Design
  • When testing without the case transform, the issue does not occur.

Other Terms

varchar2, length, maximum, case transform, oracle, 4000

Reason and Prerequisites

 Environment: 

  • SAP Data Services 4.3 SP03 Patch 03 
  • Oracle Database with Extended Varchar feature as Source / Target

Reproducing the Issue: 

  1. Enable Oracle Extended Varchar in the Oracle Database 
  2. Insert more than 4000 characters in a varchar column 
  3. Try to execute a job with case transform 
  4. Validade if more than 4000 characters were loaded to the target table

Solution

This has been forwarded to development for further investigation.

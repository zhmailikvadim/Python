Symptom

  • After enabling the environment and completing the subscription, user unable to access the BTP environment.
  • The issue occurs when launching the integration suite application; other subscribed applications load as expected.
  • Access works from non-corporate networks or personal devices but fails from corporate-managed devices.
  • Corporate network tooling reports an “SSL handshake failure”
  • Adding the “/itspaces” path to the application url does not resolve the issue.



    *Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.*

Environment

  • Cloud Integration
  • SAP Integration Suite

Reproducing the Issue

  1. Enable the environment and complete the subscription for the integration suite application in the BTP cockpit.
  2. From a corporate-managed device on the corporate network, open the application via the standard cockpit launch link.
  3. Observe that the page does not load and access fails.
  4. Test from a non-corporate network or personal device and observe that access succeeds.

Cause

Corporate network SSL inspection blocking the SAP Integration Suite URL due to an SSL handshake failure 

Resolution

Allowlisting or excluding the BTP tenant URL from SSL inspection should prevent the SSL handshake failure and allow a direct HTTPS connection to SAP BTP.

Symptom

  • Customer-specific field information maintained at the Customer master level does not populate automatically in downstream transactional documents (Quotation and Sales Order) during creation.
  • The issue persists despite attempts to resolve it by creating additional extension scenarios.
  • No error messages are displayed in the UI or application logs.

Environment

  • SAP Business ByDesign
  • SAP Cloud Applications Studio
  • SAP Cloud for Customer

Reproducing the Issue

  1. Maintain custom extension fields in the Customer master data.
  2. Create a new Quotation using the same Customer.
  3. Observe that the custom extension fields on the Quotation remain empty.
  4. Repeat the same steps for Sales Order creation.

Cause

Two conflicting extension flows were defined: "Business Partner - General Information --> Sales Quote - General Information" and "Account - Sales Data to Sales Document - General Information." This causes the issue since Customer.Common has the data but Customer.SalesArrangement does not.

Resolution

  1. Provide the same value for Customer-specific fields or revisit and recreate these fields to remove the conflict.
  2. Test the solution to confirm proper functionality.

Symptom

  • Currency is not defaulted in EAM Work Order Purchase Requisition from Info record.
  • Info record currency is A, but the currency in the linked Purchase Requisition is B.

Environment

  • SAP ERP Plant Maintenance (PM)
  • SAP ERP Customer Service (CS)
  • SAP R/3
  • SAP R/3 Enterprise
  • SAP ERP Central Component
  • SAP ERP
  • SAP Enhancement package for SAP ERP
  • SAP Enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA, on-premise
  • SAP S/4HANA Cloud Private Edition

Reproducing the Issue

  1. Create a Work Order in transaction IW31.
  2. Enter necessary data.
  3. Enter product and quantity.
  4. Observe that while a lot of information is defaulted, the currency is not defaulted from the Info record.
  5. Info record currency is A, but the currency in the linked Purchase Requisition is B.

Cause

The cause would be that by MM Purchase Requisition determination an info record is found but not taken over to the Purchase requisition item.
Because no source determination is triggered from EAM side for MM. Due to this fact on EAM side a development has been done for supporting the source determination with the Business Function. So, if you would like to use this you can active the Business Function and the corresponding customizing.

Resolution

  1. If the business requirement is to default the currency from the Info record, activate the Business Function LOG_EAM_CI_19 by enabling the switch EAM_SFWS_CC4_DETERMINE_SOS.
  2. Configure the system to support source determination functionality with the Business Function.
  3. In the customizing settings, set the flag for automatic contract determination to take over the source Info record completely.
  4. Customizing path: Plant Maintenance and Customer Service -> Maintenance and Service Processing -> Maintenance and Service Orders -> Functions and settings for Order Types -> Define Change Documents, Collective Purchase Requisition, MRP relevance.

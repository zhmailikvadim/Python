Symptom

  • In the "Additional EANs" tab under Additional Data section in MM01/MM02/MM03, the alternative units of measure (UOMs) are automatically sorted, disregarding the sequence defined in "Unit of Measure" tab.
  • The expected behavior is for the sequence in the "Additional EAN" tab to mirror the sort order established in the "Unit of Measure" tab to ensure data consistency for the end-user.

Environment

  • SAP ERP Logistics (LO)
  • SAP R/3
  • SAP R/3 Enterprise
  • SAP ERP Central Component
  • SAP ERP
  • SAP Enhancement package for SAP ERP
  • SAP Enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA Cloud Private Edition

Reproducing the Issue

  1. Access MM03.
  2. Enter Material.
  3. Select Basic Data view.
  4. Enter Additional Data section.
  5. Access the "Unit of Measure" tab and define a specific sequence for Alternative Units of Measure (UOMs). This sequence is defined based on in which order the alternative units are saved.
  6. Proceed to the "Additional EAN" tab.
  7. Observe that the Alternative UOMs are automatically sorted in alphabetical order, and the sequence does not match the one defined in the "Unit of Measure" tab.

Cause

The discrepancy in the sorting of Alternative UOMs between the "Unit of Measure" tab and the "Additional EAN" tab is standard system behavior. 
The system automatically sorts entries in the "Additional EANs" tab, typically by UOM or by internal table key which is in an alphabetical order, and does not replicate the sequence defined in the "Unit of Measure" tab.

Resolution

Check out SAP NOTE 11 for raising a feature request to be considered in future product development. SAP provides its customers the option to influence future product development.

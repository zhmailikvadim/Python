Symptom

  • The Normal change status is reset to "In Development" when moved to "Successfully unit Tested."
  • Modifiable Transport Request is changed in Development system
  • Transport of Copies is already in Released Status
  • Consistency check TASK_TOC_EQ_TR is triggered

Environment

SAP Solution Manager 7.2

Reproducing the Issue

  1. Change the Normal change status to "Successfully Tested"
  2. Document status is reset to "In Development"

Cause

The consistency check TASK_TOC_EQ_TR examines the Transport Request Last Change date and time and compares it to the Transport of Copies Released At date and Time. If the Transport Request is a greater value than the TOC's released at value, then the consistency check equals false. The consistency check is detecting that there may be untested changes in the TR.

Managed Development System Main Transport Request:

Solution Manager SE16 > /TMWFLOW/TRORD_N > TOC details:

Resolution

A new Transport of Copies for the main Transport Request must be created and imported into the target system.

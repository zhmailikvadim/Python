Symptom

In Fiori application "Manage Manual Reservations" (F4839) there is an issue with validation of an entered combination of material and storage location.


When processing an already created Reservation in "Edit" mode and changing the storage location to a value, where the material is actually not maintained, the Reservation still saves successfully. This shall not be allowed.


Instead, a check shall happen, if the material can be created automatically in the respective storage location. If not, an error message shall be returned.

Other Terms

Manage Manual Reservations, F4839, M7503, CM_IMRS010, storage location data

Solution

Implement the correction instructions or import the support package.

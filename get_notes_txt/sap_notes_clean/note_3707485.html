Symptom

When the override button is clicked, DPQS timing field defaults to "C" (contract) irrespective of any timing maintained during creation of contract.

Environment

  • Sap Agricultural Contract Management
  • SAP S/4HANA2021

Reproducing the Issue

  1. Open an already created ACM contract using transaction WB22.
  2. Navigate to the contract item and select override button under the DPQS section.
  3. Observe that the timing changes from "D" discharge to "C" contract.

Cause

When the override button is used, the system always selects the DPQS that is specifically on the contract.

Resolution

When the Override option is selected, the system should not consider any other versions of the value schedule from the DPQS master data based on the relevant event (load or unload).  It should consider the relevant version of value schedule from which the contract was saved. Due to this the system defaults the timing to "C". 

Keywords

Sap Agricultural Contract Management, Override Button, Timing Field, ACM Contract, WB22 Transaction

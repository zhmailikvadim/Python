Symptom

  • An email channel is created in SAP Service Cloud Version 2, and email forwarding is configured to the tenant’s inbound (technical) email address.
  • When requesting the verification code (coming from Amazon SES) to activate the email channel, the code is not received in the expected mailbox. As a result, the email channel setup cannot be completed.

Environment

  • SAP Service Cloud Version 2 1.0
  • SAP Sales Cloud Version 2 1.0 

Reproducing the Issue

  1. Create an email channel and set up email forwarding to the tenant's inbound technical email.
  2. Request a verification code to activate the channel.
  3. Check the inbox (e.g., Outlook or another email client).
  4. Observe that no verification code email is received.

Cause

The incoming email is forwarded directly to the tenant’s technical inbound address, without retaining a local copy of the message in the originating mailbox. As a result, the verification code is not visible in the local email client.

Resolution

Recommended Setup Flow:

  1. First, create and configure the email inbox on your mail server (e.g., Microsoft Exchange).
  2. Create the email channel in Service Cloud.
  3. Go through the email verification process (e.g., by requesting the verification code).
  4. After successful verification, configure email forwarding to the tenant's technical inbound email address.

This ensures you can monitor and confirm email verification before messages are redirected.

Alternative Workaround:

If the forwarding is already active and the code email was sent:

  1. Open the Email Inbound Monitor in Service Cloud.
  2. Search for the forwarded verification email.
  3. Download the MIME file of the message.
  4. Open it using a text editor and locate the verification code.
  5. Use the code to complete the channel activation.
  6. Repeat for any additional verification emails as needed.

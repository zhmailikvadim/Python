Symptom

During disassembly order calculation, when multiple source cuts are present, the actual quantity is not distributed proportionally across the source cuts.
Instead, the full actual quantity is assigned to the first cut, resulting in incorrect calculation results.

Other Terms

IS-MFM, MEAT, Disassembly Order, /SPMEAT/CTOB, proportional allocation, source cuts, actual quantity distribution, SAP Meat and Fish Management by msg.

Reason and Prerequisites

The issue occurs when several source cuts exist in the calculation. The current logic assigns the entire actual quantity to the first cut rather than distributing it proportionally based on the target quantities of the individual source cuts.
This deviates from the expected behavior where the actual quantity should be apportioned across all relevant cuts.

Solution

History:
This SAP Note was created on 06 February 2026

Version 2. Note added for version 6.20.

Description of Correction:

The correction introduces proportional distribution of the actual quantity across all source cuts according to their respective target quantities. The adjusted algorithm:

- Determines the sum of target quantities for all source cuts.
- Computes a share factor per cut based on its target quantity over the total.
-Allocates the actual quantity proportionally using these factors to ensure correct and traceable results.

No functional changes are introduced beyond correcting the allocation behavior.

Functional Dependencies:
None.

Manual Changes:
No manual activities are required.

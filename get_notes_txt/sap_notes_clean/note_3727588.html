Symptom

  • Using Key User Adaptation (Adapting SAP Fiori UIs at Runtime - Key User Adaptation | SAP Help Portal) > Using App Variant (Using App Variants | SAP Help Portal).
  • Creating an App Variant and publishing it to the same catalog as the original application.
  • Removing the original application access (deactivate IAM App from the business role).
  • Getting errors:
    • "No authorization to access service group"
    • "Could not open app"

Environment

SAP S/4HANA Cloud Public Edition

Reproducing the Issue

  1. Create an App Variant from the standard application.
  2. Assign the App Variant to the same business catalog as the standard application using the Custom Catalog Extensions app (as per Publishing App Variants | SAP Help Portal) or assign the App Variant to a different business catalog.
  3. Deactivate the standard application (IAM App) from the business role (as per How to Activate or Deactivate IAM Apps (App Authorization Variants) | SAP Help Portal).
  4. Search for the App Variant using the Fiori Launchpad search.
  5. Attempt to open the App Variant.
  6. Observe error messages:
    1. "No authorization to access service group"
    2. "Could not open app"

Cause

Key user adaptations do not support scenarios where the app variant is assigned to a catalog, but the standard app access is removed from the user's business role.

Key user app variants need to be in the same custom catalog as the original app to retrieve the necessary authorizations.

Resolution

  1. Ensure that the app variant and the original standard app are included in the same business catalog.
  2. Assign the business catalog of the corresponding standard app to the user's business role.

     

Alternatively, explore the possibility to use Adaptation Projects. For more information, please refer to the following documentation:

  • Develop an SAPUI5 Adaptation Project | SAP Help Portal
  • Extending an SAP Fiori Application for SAP S/4HANA Cloud Public Edition and SAP BTP, ABAP Environment | SAP Help Portal

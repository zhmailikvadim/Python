Symptom

When using Joule with the Document Grounding feature enabled, users encounter the following responses:

  • Responses do not appear in the expected language specified in the prompt (for example, the user enters prompts in Language A but receives responses partially or fully in Language B).
  • Responses do not reflect the expected grounded content or intent from the referenced documents (for example, they may be missing, incomplete, or contain irrelevant details compared to the source documents).

Environment

  • SAP BTP, Cloud Foundry runtime and environment
  • Joule
  • Retrieval Augmented Generation for Enterprise (Document Grounding)

Resolution

  • As per the help document Joule Multi Language Support, Joule Multi-Language Support enables users to interact with Joule in their preferred language by providing localized responses, including messages and product-related information.

  • Joule automatically adopts the language configured in the underlying SAP application and is also capable of understanding user prompts in languages beyond those that are officially supported based on Joule Language Support.

  • Joule uses the language settings of the SAP application when a user request is sent. Upon receiving a request through an SAP application, Joule automatically adopts the language configured within that application to ensure a seamless experience.

    • If the language is not explicitly set in the SAP application, Joule sets the language based on:

      • Your browser settings on PC or Laptop devices

      • Your device settings on mobile devices.

    • If the SAP application, device, or browser language settings are not supported by Joule, Joule will automatically default to English.

  • Joule will respond in the language defined in the settings, regardless of the language in which the user asks the question.

    For example,

    Problem: If Joule is set to English based on browser settings and the user asks their question in Spanish. Joule will respond in English, since that is the language defined in the settings. However, there may be unexpected exceptions where the language entered by the user is not reflected in the response. 

    Answer: To resolve this above conflict, change the browser language settings to user prompt language (based on above example, browser language settings should be tuned to Spanish.

Symptom

When creating multiple invoices subject to accumulated IRRF withholding tax, the IRRF calculated after the INSS threshold is reached is incorrect. The calculation works correctly for the first documents, but becomes incorrect for subsequent ones.

 

Environment

  • SAP S/HANA
  • SAP ERP

Reproducing the Issue

  1. Please create invoices via FB60 or another relevant transaction.
  2. Then verify the IRRF calculation after the maximum INSS threshold is reached.
  3. You will notice that the IRRF calculation is incorrect for the subsequent documents, while it is correct for the first ones.

Cause

The tax type is configured to use the Min/Max amount at the type level instead of the tax code level.

Resolution

  • Change the configuration of the INSS tax type to use the Min/Max threshold at the tax code level.
  • Maintain the entries in “Define Min/Max Amounts for Withholding Tax Codes” instead of maintaining them at the tax type level.

Symptom

  • Dataflow fails with error message: "General error;258 insufficient privilege: Detailed info for this error can be found with guid 'XXXXXXXXXXXXXXXXXXXXXXXXXXXXXXX'".
  • Error occurs during execution of SQL statement:
    • DELETE FROM "SPACE"."LOCAL_TABLE"...
      OR 
    • INSERT FROM "SPACE"."LOCAL_TABLE"...
  • Subsequent failure indicate missing privileges for DELETE or INSERT operations.

Environment

  • SAP Datasphere

Reproducing the Issue

  1. Execute the Data Flow with a target table located in an Open SQL schema.
  2. Observe the failure with the error message indicating insufficient privileges for DELETE or INSERT operation.

Cause

  • The technical Space DB user lacks the necessary privileges (DELETE or INSERT) for the target table in Open SQL schema.

Resolution

Perform steps from Help Allow the Space to Access the Open SQL Schema:

  1. Using the Open SQL DB User, drant the required privileges to the Space Technical Databse User (same name as Space's technical name) for operations on the target table located in Open SQL Schema:

    • Execute the following statement to grant DELETE privilege:
    CALL "DWC_GLOBAL"."GRANT_PRIVILEGE_TO_SPACE" (OPERATION => 'GRANT', PRIVILEGE => 'DELETE', SCHEMA_NAME => '<Insert_here_OpenSQLSchema>', OBJECT_NAME => '<Insert_Here_Target_Table_in_OpenSQL_Schema>', SPACE_ID => '<Insert_Here_Space_Name>');

          OR/AND

    • Execute the following statement to grant INSERT privilege:
    CALL "DWC_GLOBAL"."GRANT_PRIVILEGE_TO_SPACE" (OPERATION => 'GRANT', PRIVILEGE => 'INSERT', SCHEMA_NAME => '<Insert_here_OpenSQLSchema>', OBJECT_NAME => '<Insert_Here_Target_Table_in_OpenSQL_Schema>', SPACE_ID => '<Insert_Here_Space_Name>');
  2. Re-run the Data Flow to confirm successful execution.

Symptom

  • Supplier was deactivated in SAP Ariba Supplier Lifecycle and Performance (SLP) by uploading a supplier file in SM admin, changing the active flag from true to false.
  • This action caused the supplier to become inactive in SLP, triggering a confirmation payload to S/4HANA, which resulted in the supplier being deleted in S/4HANA.

Environment

Supplier Lifecycle & Performance - XML Integration

Reproducing the Issue

  1. Deactivate a supplier by uploading a supplier file in SM admin, changing the active flag from true to false.
  2. Observe that the supplier becomes inactive in SLP.
  3. Note that a confirmation payload is triggered to S/4HANA, resulting in the supplier being deleted in S/4HANA.

Cause

Payloads are not generated for suppliers missing ACM_ID or address line in the import file. And this is system expected behavior

Resolution

  1. Ensure that the supplier file includes complete address information and ACM_ID for all suppliers before importing it into SM admin.
  2. Validate the supplier file to confirm that no entries have missing address lines or ACM_ID.
  3. Re-import the corrected supplier file into SM admin to trigger the payload for all suppliers.

Symptom

When any change is made to an Agreement and the document is saved, the system performs output authorization checks and displays the message “No authorization to print” on every save.This behavior occurs even when no output types are configured for the Agreement. In such cases, the system should not validate output generation authorizations at all.

This SAP Note resolves the issue by ensuring that authorization checks are performed only for configured output types. If no output types are maintained, the system skips the output authorization validation, thereby preventing the incorrect authorization message during Agreement save.

Other Terms

Output Type, Authorization Checks, /IRM/IPCRAP, Agreement

Reason and Prerequisites

This is caused by program error

SAP Note#3708838 is available for ECC Release

Symptom

'BTC User Error' as JD Status in transaction SJOBREPO although no specific user has been maintained in the job definition (field User Name shows '(DEFAULT)').

Other Terms

Technical Job Repository, User name = (DEFAULT) in job definition

Reason and Prerequisites

The job definition has been created and activated in SE80 without the dummy entry (DEFAULT) in field User Name. After saving, the dummy entry is displayed in the workbench object, but in table STJR_JOBD_ROOT, field BTCJOB_USER is empty. Therefore, the Job Repository runtime user check results in 'BTC User Error' and job is not being executed by Job Repository generator - although a valid stepuser has been maintained in SJOBREPO_STEPUSER.

Solution

Implement the correction instructions.

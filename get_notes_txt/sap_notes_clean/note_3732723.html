Symptom

When you update the Branch ID (BPLId) of a Chart of Accounts via DI API or Service Layer, it can be set to an invalid value of "0".

Reproducing the issue

Prerequisites:

Multiple Branches is enabled via Administration → Company Details → Basic Initialization → Enable Multiple Branches.

Scenario:

  1. Go to Financials → Chart of Accounts and create a Chart of Accounts as 'TESTACCT'.
  2. Log in to Service Layer.
  3. Use the following HTTP request to update the Chart of Accounts via Service Layer:
    PATCH /b1s/v1/ChartOfAccounts('TESTACCT')
    {
        "BPLId": 0
    }
  4. Go to the SAP Business One client, navigate to Financials → Chart of Accounts, and search for Chart of Accounts 'TESTACCT'.

Expected behavior:

The Branch ID (BPLId) should not be updated to 0; an error message should display.

Actual behavior:

The Branch ID (BPLId) can be updated to 0.

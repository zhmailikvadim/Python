Symptom

While replicating Employee Master Data or Employee Time Data using the batch job/scheduled job, the following error is raised in SLG1:

Error:
Code: AuthenticationFailed
Message: Authentication service failed to return token.
Details: Only support OAuth/OIDC authentication type.

However, while performing a Manual Master Data or Time Data replication, a different error could appear in SLG1:

Error:
Code: AuthenticationFailed
Message: Authentication service failed to return token.
Details: [AUTH0032]Invalid client certificate

NOTE: the error appears in SLG1 with the following JSON structure:

Login response: {"error":{"code":"&1","message":"&2","details":&3"}} 
Employee Central login failed. Aborted employee replication based on configuration query.

Environment

  • Employee Master Data Replication from Employee Central to Employee Central Payroll
  • Employee Time Data Replication from Employee Central to Employee Central Payroll
  • SAP SuccessFactors Employee Central Payroll

Reproducing the Issue

  1. Run the Scheduled job or Manually Replicate Employee Data (Master Data or Time Data) Replication from EC to ECP.
  2. Check the error in the Application Logs (SLG1).

Cause

The X509 Certificate used in the OAuth 2.0 SSL Handshake between the Employee Central Payroll System and the Employee Central Instance is not the correct one, this might be because the X509 Public Key mapping hasn't been correctly performed or not performed at all, the API user set up in the mapping is incorrect or it doesn't have the necessary permissions to access the Compound Employee API.

IMPORTANT: Employee Central Payroll does not currently support OpenID Connect (OIDC) to communicate with SuccessFactors Employee Central, instead, it uses OAuth 2.0 to communicate and authenticate users when logging into the SuccessFactors Employee Central's Compound Employee API, used in the Master Data Replication.

Resolution

To fix this issue, please, do the following:

  1. Export the X509 Certificate within the Own Certificate section from the STRUST folder '<client_id>_SD', by performing ONE of the two following options:
    1. Ideally, download the X509 Public Key using the Download Public Key field of the Point-to-Point configuration report that you access with the HRSFEC_PTP_CONFIG transaction, as described in the implementation guide: Set Up Connection Between an Employee Central Payroll System and Employee Central (step iii).
      OR
    2. Alternatively, download the X509 Certificate Public Key exactly as described in the following guide section: Exporting Client Certificates from STRUST.

      NOTE: If the relevant <client_number>_SD folder can't be found, create a case under the component LOD-EC-ECP-OPS and request the creation of the missing clients.

  2. Ensure the X509 Public Key Certificate Mapping configuration in the Security Center has been correctly performed with the exported X509 Certificate in the previous step, as described in the implementation guide: Importing Client Certificates to the Security Center.

  3. Ensure that the API User used in the previous step have the necessary permissions to log into the Compound Employee API, as described in the following implementation guide sections: Granting Permissions for Full Access to the CompoundEmployee API and Granting Permissions to API Users.

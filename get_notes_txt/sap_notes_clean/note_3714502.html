Symptom

  • When using transaction ES51 in the transaction launcher in SAP S/4HANA, data is not saved.
  • Performing the same steps in the backend transaction ES51 allows data to be saved successfully.

Environment

  • SAP S4CRM for the releases:
    • S/4HANA 1909 
    • S/4HANA 2020
    • S/4HANA 2021
    • S/4HANA 2022
    • S/4HANA 2023 and higher

Reproducing the Issue

  1. Log in to the WebUI with role S4C_UT_IC or a copy of it. 
  2. Search for technical object in free text search.
  3. Go To One-Click-Action: premise -> transactions -> maintain owner 
  4. For this use transaction ES51 via the transaction launcher.
  5. Enter any business partner and save the data.
  6. Nothing is saved and changed.
  7. When doing the same steps in backend transaction ES51 everything is saved successfully.

    Cause

    The parameter RACOMMIT is not set in the action profile.

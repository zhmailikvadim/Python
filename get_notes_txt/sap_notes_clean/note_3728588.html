Symptom

In an Onboarding checklist task, the items included in the checklist are initially displayed as configured.

However, after editing the checklist (for example selecting, adding, or modifying at least one item), the items may be displayed in a different sequence. The order shown in the user interface may not follow:

  • the configured order of checklist items
  • alphabetical order
  • the original creation order

As a result, checklist items may appear to be displayed in a random order.

Environment

SAP SuccessFactors Onboarding

Resolution

This behavior is consistent with the current system design.

In Onboarding, checklist items within onboarding tasks do not support a configurable or fixed sorting logic in the user interface. When a checklist task is edited, the application does not apply an ordering parameter when rendering the checklist entries.

Therefore, after modifications are made to the checklist, the system may display the items in a sequence that differs from the originally configured order.

The behavior is expected according to the current product design of the checklist component in Onboarding.

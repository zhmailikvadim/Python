Symptom

  • ABAP dump occurs with Runtime Error RAISE_EXCEPTION in ABAP program CL_UKM_DCD_CASE===============CP. 
  • Short text indicates Exception condition "ILLEGAL_CASE_TYPE" triggered. 

Environment

  • SAP enhancement package for SAP ERP, version for SAP HANA All versions
  • SAP S/4HANA
  • SAP S/4HANA FINANCE 
  • Financial Accounting (FI)

Cause

Case Type Name is missing in SPRO in the following IMG path
Financial Supply Chain Management
Credit Management
Credit Risk Monitoring
Documented Credit Decision
Define Case Types for Documented Credit Decision:
→
BOB Credit Decision for Document   B2B-EXIST   FDCD   Case Type Name is MISSING
BOB Credit Decision for Document   B2B-NEW     FDCD   Case Type Name is MISSING
BOB Credit Decision for Document   B2C-EXIST   FDCD   Case Type Name is MISSING
BOB Credit Decision for Document   B2C-NEW     FDCD   Case Type Name is MISSING
BOB Credit Decision for Document   MIG             FDCD   Case Type Name is MISSING 

Resolution

Maintain manually the missing Case Type Name for the specified entries in SPRO. The correct name for the Case Type is "Documented Credit Decision". Update the Case Type Name for the following entries:

  • BOB Credit Decision for Document B2B-EXIST FDCD
  • BOB Credit Decision for Document B2B-NEW FDCD
  • BOB Credit Decision for Document B2C-EXIST FDCD
  • BOB Credit Decision for Document B2C-NEW FDCD
  • BOB Credit Decision for Document MIG FDCD

Keywords

SAP Credit Management, FSCM Credit Management, dump, SPRO, DCD, ILLEGAL_CASE_TYPE, RAISE_EXCEPTION, CL_UKM_DCD_CASE===============CP

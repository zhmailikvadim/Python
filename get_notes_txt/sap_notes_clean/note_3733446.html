Symptom

In the DRC Cloud Edition "Resolve Errors" app, the following message is shown for the affected eDocument ID:

"Error occurred while the PUSH step, detailed information: Check the destination configurations for destination '<destination_name>'. Response returned HTTP status code: 403. Check the destination configurations maintained in your BTP subaccount. If the problem still persists, create an incident in the LOD-LH-DRC-FND application component."

Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.

Environment

  • SAP ERP
  • SAP S/4HANA
  • SAP S/4HANA Cloud Private Edition
  • SAP BTP
  • SAP Document and Reporting Compliance, Cloud Edition

Reproducing the Issue

  1. Launch transaction edoc_cockpit
  2. Submit an eDocument
  3. Check the logs in DRC Cloud Edition - Resolve Error application

Cause

  1. Missing authorizations
  2. Incorrect setup between Cloud connector & business systems

Resolution

  1. Verify whether the HTTP service GCO_INTERNAL is active in the relevant system. If inactive, activate the HTTP service GCO_INTERNAL by following below help portal documentation
    • SAP S/4HANA 2022 or Higher - Activating HTTP Service for SAP S/4HANA 2022 or Higher or for SAP S/4HANA Cloud Private Edition | SAP Help Portal
    • SAP S/4HANA 2021 or Lower - Creating HTTP Service | SAP Help Portal
    • SAP ERP - Activating HTTP Service | SAP Help Portal 
  2. Check the authorization objects for the technical user set in the SAP BTP destination, as follows:
    • SAP S/4HANA: Check user authorization described in the Setting Up Connectivity.
    • SAP ERP: Check user authorization described in the Setting Up Connectivity.
  3. Check the setup between the Cloud Connector and SAP S/4HANA or SAP ERP, as follows:
    • SAP S/4HANA: Review the steps of the Connecting SAP BTP Subaccount to Cloud Connector.
    • SAP ERP: Review the steps of the Connecting SAP BTP Subaccount to Cloud Connector.

Symptom

An issue occurs during cost allocation using transaction KEU3, where a posting with value type 11 remains unallocated on cost center.

Environment

  • Controlling (CO)
  • SAP R/3
  • SAP R/3 Enterprise 
  • SAP ERP Central Component
  • SAP ERP
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA Finance
  • SAP S/4HANA

Reproducing the Issue

  1. Execute transaction KEU3 for cost allocation.
  2. Observe that posting remains on the cost center with value type 11.

Cause

Value type 11 in SAP CO postings represents statistical actual postings, which are not eligible for allocation, settlement, or distribution in cycles like KEU3.

Resolution

  1. Exclude value type 11 via cycle segments in KEU3 (select only actual primary via value type filter if needed).
  2. Repost the statistical line as real via KB11N if business requires allocation.

See Also

Refer to: KBA KB0870455 - Certain sender costs not considered in Allocation execution

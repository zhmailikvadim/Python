Symptom

The Connector Cockpit provides monitoring capabilities for inbound and outbound processing, including an overview of system queues. It displays the number of queues in the following states: Ready, Active, and Not Ready. Currently, the Cockpit evaluates all registered system queues. In environments where the queue mechanism is heavily used, several hundred queues may be registered simultaneously. This can negatively impact Cockpit initialization time and overall performance.

Other Terms

FSM, E4C, Connector, Queue, Cockpit, IDoc

Reason and Prerequisites

We have improved the performance of the Connector Cockpit by limiting queue evaluation to Connector-relevant queues where appropriate, while preserving the current full-scope monitoring behavior as the default option. Therefore if the filtering is needed, please, enable 'Only /PACG/' flag using transaction /PACG/ECM_COCKPITCON. In case the field is not visible and you have component (S4)PACG200, make sure you have performed all manual post-installation steps of prerequisite SAP Note 3728308.

Solution

Please, implement (in case of (S4)PACG200 component) the prerequisite SAP Note 3728308 and perform manual pre-installation steps. Next, or when using component S4PACG400 implement this correction instruction.

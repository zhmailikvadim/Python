When importing Region of Birth for the 5 supported countries (France, Italy, Chile, Brazil, Venezuela), the field initially accepts free text instead of appearing as a picklist.

Environment

SAP SuccessFactors Employee Central 

Reproducing the Issue

  1. Prepare an import file including the Region of Birth field for one of the supported countries (France, Italy, Chile, Brazil, Venezuela).
  2. Enter the Region of Birth value as free text in the import file.
  3. Perform the import using Import Employee Data.
  4. After import, open the employee’s Biographical Information record.
  5. Observe that the Region of Birth value is stored as free text.

  

Cause

This is expected system behavior. The Region of Birth field defaults to free text during import or initial entry. The picklist is displayed only editing the Region of Birth field from profile.

Resolution

For import scenarios, free text entry for Region of Birth is currently expected behavior. There are a few enhancements planned for the future regarding this issue.

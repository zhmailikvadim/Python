Symptom

Users with active accounts cannot be found using Proxy Now search when their previous Employee Central (EC) employment is terminated.

This issue typically occurs for users who:

  • Have multiple contracts under the same account
  • Previously had an EC employment record that is now terminated
  • Currently exist as active non-EC (EP) users

Environment

SAP SuccessFactors HCM Suite

Reproducing the Issue

  1. A user has:
    • An user with active  EP account (non-EC)
    • A terminated EC employment record
  2. Navigate to Proxy Now
  3. Search for the affected user
  4.  The user does not appear in the search results

Cause

The issue occurs due to the design of the Proxy Now search, which is scoped around employment and operates through a two-step process:

  • Initially, the system retrieves data from Solr (legacy people search), where the user is identified as having an EC account based on historical records.
  • It then invokes the EC API for further filtering. Since the EC employment is terminated, the API returns no results.

This behavior is expected. The search logic prioritizes EC system records; therefore, if a user has any EC contract history, they are consistently treated as an EC user. As a result, EC-based filtering continues to be applied even after the user transitions to a non-EC status, leading to the user being excluded from the search results.

Resolution

To address this, users should use the Enhanced People Search functionality, as it does not rely on the EC filter API.

  • Enhanced People Search directly indexes and retrieves data from EC tables.
  • When a user transitions to a non-EC status, the system correctly sets the "isECSystemRecord"value to 0, ensuring the user is returned accurately in search results.

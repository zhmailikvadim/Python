Symptom

  • When using the Perform Group Checkout function in the FSM iOS application, some activities that were previously checked out do not appear in the list of activities available for Group Checkout.
  • Although these activities are correctly closed and should be eligible for selection, they remain hidden until the device performs a sync, after which the activity data is updated and the activities become visible.

Environment

SAP Field Service Management

Reproducing the Issue

Prerequisite:

  • Ensure that multiple Activities exist under the same Service Call

Reproduction Steps:

  1. Check out one Activity without generating a report (e.g., Finish Without Report)
  2. Without synchronizing the mobile app, open a second Activity and initiate Perform Group Checkout
  3. Attempt to include the previously checked‑out Activity in the Group Checkout list.
  4. Observe that the Activity does not appear
  5. Still without syncing, check out another Activity under the same Service Call
  6. Open the Group Checkout screen for this newly checked‑out Activity
  7. Perform one or more synchronizations of the mobile application
  8. Reopen the Group Checkout screen

Actual Behavior: 

One or more Activities do not appear in the Group Checkout list initially. The missing Activities only become visible after a sync, once the system updates the Activity data and attachments.

Expected Behavior:

All closed Activities belonging to the same Service Call should appear consistently in the Group Checkout list without requiring a sync or any additional system processing.

Cause

  • The FSM iOS mobile application filters Activities displayed in Perform Group Checkout based on several internal conditions.
    One of these conditions requires that the Activity must already have a report attachment stored locally on the device.
  • When a technician checks out an Activity without generating a report, the backend system (ERP/ECC or Cloud) creates or attaches the report only after the next synchronization.
    Until this sync occurs, the Activity does not meet the “has report attachment” condition and is therefore excluded from the Group Checkout list.
  • This results in Activities being temporarily hidden even though they are already in a Closed status and belong to the same Service Call.

Resolution

To ensure that recently checked‑out Activities appear in the Group Checkout screen, the mobile device must receive the updated Activity information, including the report attachment generated by the backend.

  1. Perform a synchronization on the mobile app after checking out the Activity.
  2. Wait for the backend‑generated report to be attached during the sync process.
  3. Reopen the Group Checkout list.

Once the Activity has a report attachment and the device has synchronized the updated data, the Activity will appear as expected.

Note:

The need for a sync is caused by the current iOS implementation, which only lists Activities that already have a local report attachment.

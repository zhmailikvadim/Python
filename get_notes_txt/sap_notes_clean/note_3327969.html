Symptom

Account login fails with error "Account temporarily locked out" when using an invalid email address that does not exist in Customer Data Cloud (CDC).

The error "Account temporarily locked out" appears when a Risk Based Authentication (RBA) rule to lockout account after a few failed login attempts enabled.

Environment

  • SAP Customer Data Cloud
  • Risk Based Authentication (RBA)

Reproducing the Issue

  • Create an RBA rule to lock account after 3 failed login attempts
  • On Gigya Login screen, enter an email address that does not exist in CDC (e.g. WhoAreYou@test.com)
  • Enter anything for the password
  • Submit button
  • Result: accounts.login fails with error "Account temporarily locked out"

Resolution

This is the expected behavior in the current design.

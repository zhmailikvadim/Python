Symptom

In transaction /UCOM/MDX_DATA_CONT, it was not possible to set newly created layouts as a default layout. As a result, the preferred layout always needed to be selected manually when the transaction was opened.

This issue is relevant for all market roles.

Other Terms

Report /UCOM/RP_MDX_DATA_CONTAINER, MDX data container, Default Layout, ALV grid, transaction /UCOM/MDX_DATA_CONT

Reason and Prerequisites

Missing functionality.

Solution

Implement the corrections of this SAP Note or import the relevant support package.

After implementing this SAP Note, the system provides a "Default" checkbox in transaction /UCOM/MDX_DATA_CONT, allowing you to set a default layout for the ALV grid. You need to create a default layout as needed.

You can verify the fix by running transaction /UCOM/MDX_DATA_CONT and confirming that the "Default" checkbox is available to set the default layout for the ALV grid.

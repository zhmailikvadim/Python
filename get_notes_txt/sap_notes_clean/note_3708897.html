Symptom

Error message CN697 "Enter a G/L account" occurs when saving production orders / process orders or displaying production order / process order details.

"Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental."

Environment

  • SAP ERP Production Planning (PP)
  • SAP R/3
  • SAP R/3 Enterprise
  • SAP ERP Central Component
  • SAP ERP
  • SAP Enhancement package for SAP ERP
  • SAP Enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA Cloud Private Edition

Reproducing the Issue

  1. Execute transaction code CO03.
  2. Open production order.
  3. Navigate to the component overview and double-click on the relevant component.
  4. Click "Back".
  5. Observe the error message CN697 "Enter a G/L account".

Cause

There might be a Non-stock item between the components of the production order. To verify, the value in field "Item category" need to be checked in the Component overview of the production order in for example transaction CO03. Non-stock materials require a G/L account to be assigned for purchasing purposes.

Resolution

    If a Non-stock item is used in the production order, then a G/L Account need to be entered to proceed further. There is the possibility to manually enter the G/L account. To do so in transaction CO01 / CO02 / COR1 / COR2 go to the component overview, select the relevant component, click "Component details" and then navigate to "Purch. Data" tab. There can the "G/L Account" field be maintained.

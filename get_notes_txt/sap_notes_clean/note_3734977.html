Symptom

When posting confirmation using BAPI - BAPI_ALM_CONF_CREATE error "Specify time within permitted value range" (message E 00 069) 

Environment

  • SAP ERP Plant Maintenance (PM)
  • SAP R/3
  • SAP R/3 Enterprise
  • SAP ERP Central Component
  • SAP ERP
  • SAP Enhancement package for SAP ERP
  • SAP Enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA, on-premise
  • SAP S/4HANA, Cloud Private Edition

Reproducing the Issue

  1. Start transaction SE37.
  2. Run function module BAPI_ALM_CONF_CREATE.
  3. Open the Test data directory.
  4. Execute

Cause

The provided confirmation timestamp falls into a non-existent daylight saving time jump (for example, 02:00:00–02:59:59 local time on the DST start date in CET), so the time is outside the permitted range.

Resolution

  1. Adjust the confirmation time fields (for example, TIMETICKETS-EXEC_START_TIME and corresponding end time EXEC_FIN_TIME) to a valid local time that exists on the specified date (on DST start dates, use 03:00:00 or later instead of any 02:xx:xx time).
  2. Ensure the date and time zone used for confirmation align with a valid local timestamp (avoid the DST gap).
  3. Re-run BAPI_ALM_CONF_CREATE with the corrected time values.

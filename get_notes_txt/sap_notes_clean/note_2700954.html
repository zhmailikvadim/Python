Symptom

Default values in infotype Social Insurance (0059) cannot be differenciated per Collective agreement.

Other Terms

NLSVD

Social Insurance (0059)

Reason and Prerequisites

The feature NLSVD - Default Values for infotype Social Insurance (0059) does not consider the Pay scale type (TRFAR) and Area (TRFGB) in the decission tree.

Solution

The feature NLSVD has been enhanced so that the default values for infotype infotype Social Insurance (0059) can be set dependant of the the Pay scale Type and Area values from infotype Basic Pay (0008).

To use this enhancement, change the feature structure of feature NLSVD from PME01 to PMENS and select the fields BUKRS, WERKS, BTRTL, PERSG, PERSK, PERNR, TRFAR and TRFGB. To do this, use feature maintenance Transaction PE03 (Select Attributes -> Change -> Struct.). Then activate the feature in all clients. As an alternative to this, you can delete the feature and implement your Decision Tree again.

Delivery

Import the Support Package specified for your release.

Check on the SAP Support Portal under SAP ERP Human Capital Management Support Package Schedules for the availability of the relevant HR SP and possible dependencies.

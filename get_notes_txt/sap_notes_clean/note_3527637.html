Symptom

In upgrade phase MAIN_SHDRUN/ACT_UPG, there is the following activation error:

"Customer Index" "<table_name>" "<index_name>" could not be activated

Environment

SAP NetWeaver release independent

Reproducing the Issue

Cause

In the log that contains detailed information about the error, it is possible to see the cause of it:

Indexes "<index_standard>" and "<index_Z*>" for table "<table_name>" have identical fields

As both indexes contain the same fields, one of the two is not necessary. The error is occurring during an upgrade, then it is most likely that the customer index was created before the upgrade and now there is a new index in the standard which contains the same fields as the custom one.

Resolution

In case one of the two indexes is a custom one, it is recommended to delete the custom index and just save the changes (do not perform any manual activation on shadow instance, the object will be activated when running ACT_UPG phase again, by the SUM tool).

If there are further queries about the scenario or there is no custom index involved, it is recommended to contact the application team responsible for the table.

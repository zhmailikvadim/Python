Symptom

When running silent data migration for CL_SDM_MIGR_PROJ_01 or CL_SDM_MIGR_AUFK_01, there is exception raised with type:CX_SY_OPEN_SQL_DB.

This issue is observed in migrate_data_finished method when there are large number of projects or Networks being updated in SDM execution.

Other Terms

CX_SY_OPEN_SQL_DB, CL_SDM_MIGR_AUFK_01, CL_SDM_MIGR_PROJ_01, MIGRATE_DATA_FINISHED, SDM_MON

Reason and Prerequisites

 Program Error

Solution

Please follow the below steps in sequence.

1. Create correction report:ZPS_SDM_MIGR_CORR_REPO and copy the source code provided in the SAP note attachments "ZPS_SDM_MIGR_CORR_REPO.txt". Execute the created report to update the SDM migration status to '02' in PROJ and AUFK table.

2. To reset the SDM migration class status,

    2.1 Execute the SDM_MON transaction to reset the status for the migration class CL_SDM_MIGR_PROJ_01 or CL_SDM_MIGR_AUFK_01 , for the cancelled migrations.
    2.2 Execute the report R_SDM_ONLINE_EXEC to rerun the SDM migration for the class CL_SDM_MIGR_PROJ_01 or CL_SDM_MIGR_AUFK_01 to update the SDM migration as finished

Symptom

  • User is unable to open a signed .pdf document downloaded from Ariba Contract Workspace.
  • The document was created by the system and signed using DocuSign - the Signed .pdf document then returns to Ariba via Signature Provider, but it cannot be opened due to corruption in the file.

Environment

Ariba Contracts
Signature Tasks
DocuSign integration

Reproducing the Issue

  1. Download the signed document from Ariba Contract Workspace.
  2. Attempt to open the downloaded document.

Cause

The corruption in the Signed .pdf document could have happened during its conversion on the DocuSign end, or when the already converted .pdf was in flight to Ariba.

Resolution

  1. Make sure the original file, before conversion, is not corrupted.
  2. Try to download the signed .pdf file from DocuSign.
  • If you are able to open the file, manually upload it to the desired folder in Ariba.

     3. If you do not desire to proceed with the above workaround and the original pre-signature document is not corrupted:

  •  Withdraw the Signature Task.
  •  Start a New Round.
  •  See if you can finish the Signature process and have the proper .pdf file afterwards, clear of any corruption.

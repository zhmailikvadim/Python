Symptom

For a cross-border employee France with a special agreement, that is, withholding tax liability in the canton VD, NE, JU, SO, BS, BL, BE and VS, erscheint die Warnung 'Vordef. QSt-Kategorie <01> passt nicht zum Tarif 'GF' (QSTPF=4)'
HRPAYCH_ELM50 022).

However, the master data is correct for the current period.

Environment

  • Personnel Administration Switzerland
  • Payroll Switzerland

Reproduce Issue

  • The master data is maintained for at least one period as follows:

    1. IT0006:
    Residence in France

2. IT0038:
* Canton with special agreement (for example, BS, BL, and so on)
* Tax liability = 4 (withholding tax cross-border employee France)
* Pay Scale = GF (Cross-Border Employees France WHT Exempt)

  • You run payroll for the employee using RPCALCC0*. Either the IN period or one of the FOR periods contains the master data from 1.
    (See payroll cluster RPCLSTRC, tables ST or WHTM)


  • The data extractor for the uniform wage notification procedure (ELM), 5.0, domain QST issues the warning message "Predef. WHT category <01> does not match pay scale 'GF' (QSTPF=4)'.

Cause

As of 1/1/2024, the pay scale = GF can no longer be maintained for cantons for which the cross-border employee France special agreement applies.

Solution

1. Correct the master data in the IN period (= selection period of RPLELMC0) for all periods (IN and FOR periods):

  IT0038:
* Canton with special agreement (for example, BS, BL, and so on)
* Tax liability = 4 (withholding tax cross-border employee France)
* Pay Scale = SFN (Special Arrangement Border Gr. France)

2. and run payroll again for the employee (RPCALCC0*) if this is possible (Admin. record of payroll period).

Symptom

You are using SAP S/4HANA Group Reporting and you intent to post manual journal entry.

When document to be posted is checked, there is a popup with error message: "Field XYZ is not activated as an additional field in the data model" at screen.

Some of additional fields is inactive in the data model even it was working before upgrade to OP 2025.

Other Terms

Additional MD, field inactive, PGJE

Reason and Prerequisites

There is a change in processing logic between versions OP 2023 and OP 2025 release.

As of OP2025, system strictly evaluates flag "Enable Inputs" for additional characteristics.

Solution

1. Go to SPRO transaction, path: Activate additional master data fields (flag "Enable Inputs") when required for operations (SAP Customizing Implementation Guide -> SAP S/4HANA for Group Reporting -> Master Data -> Define Consolidation Master Data Fields).

2. Clean up cache of gateway model: /n/IWFND/CACHE_CLEANUP

3. Clean up cache for gateway backend model: /n/IWBEP/CACHE_CLEANUP

2. Clean + Regenerate metadata after making changes to the model setup (step 1):

  • execute /n/IWFND/MAINT_SERVICE, find FINCS_JRNLENTR_POST service (ZFINCS_JRNLENTR_POST_SRV)
  • open "SAP Gateway Client", in top menu select Metadata ->Clean up Cache, select Metadata -> Rebuild Cache for all services

Repeat with FINCS_JRNLENTR_LIST and FINCS_IMPORT_JOURNAL_ENTRY.

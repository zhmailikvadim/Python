Symptom

ORA-16000: database open for read-only access
Created new DR system and while executing transaction codes the "sql error 16000 when accessing table SNAP" error message is raised

Environment

SAP systems with Oracle database
Oracle standby database

Cause

Database open for READ-ONLY access and NOT in READ-WRITE mode
Usually, the Standby Database is opened in 'Read-Only Mode', mounted state. 

Resolution

So this is the case if one tries to connect from SAP to the Standby database. This is expected, nothing wrong with that.
SAP Applications must connect to the Primary Database all the time. It is impossible to connect to Standby from SAP.

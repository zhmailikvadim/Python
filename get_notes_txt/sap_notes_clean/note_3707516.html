Symptom

  • When deploying an application to SAP BTP, the following error appears: Service broker error: Service broker xsuaa failed with: Instance secret is no longer allowed to be used.
  • When creating a Service Instance in the SAP BTP cockpit, the following error appears: Service broker error: Service broker xsuaa failed with: Instance secret is no longer allowed to be used.

Environment

SAP BTP, Cloud Foundry runtime and environment

Cause

Instance Secret Credential Type is deprecated starting January 19th 2026.

Resolution

  • Instead of using the instance-secret credential type, applications and services must now use the binding-secret credential type (see Migrate from Instance Secrets to Binding Secrets) or x509 (see Retrieving Access Tokens with Mutual Transport Layer Security (mTLS)).
    • To test your application with these credential types, update your security descriptor to set the credential type to ["binding-secret", "x509"].
    • For more information on the security descriptor, see Application Security Descriptor Configuration Syntax.

Symptom

  • The percentage account values are shown as decimal in Chart in SAP Analytics Cloud
  • This may happen when the measure of the Chart was assigned to some calculated measures or restricted measures.

Environment

SAP Analytics Cloud 2025.21

Reproducing the Issue

  1. Prepare a model has measure and Account dimension.
  2. Make sure one Account member scale setting is set as Percentage.
  3. Create a SAP Analytics Cloud story and add a Combination Column & Line chart.
  4. In Column Axis, assign it with some account members.
  5. In Line Axis, assign it with the percentage account member.
  6. In Measure, assign some calculated measure or restricted measure created within this story. 
  7. After Dimension is assigned, check the Line Axis, the value for the percentage account member is shown as decimal but not percentage (%).

Cause

This is because, in this chart context, the measure is using calculations. Refer to the following guide, Calculations have the higher priority than accounts. Therefore, the scale setting in the account member is not used.

Set Structure Priority and Create Custom Solve Order | SAP Help Portal

Resolution

Workaround: Create a Calculated Account member instead. For example, in the above workflow, in Line Axis, create a new Calculated Account as the following step:

  1. In Line Axis, click Add Account -> Add Calculation
  2. Set Type to be Calculation Account and make the formula just use the existing percentage account. Click OK to complete.
  3. For this newly created Calculated Account, there will be the Format option. Click Format to edit.
  4. Set the Scale to be Percentage and save the setting.
  5. In Chart, using this newly created Calculated Account and then remove the original percentage account.

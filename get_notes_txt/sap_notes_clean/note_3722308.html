Symptom

Error encountered when running the SAPUI5 Adaptation Project generator in SAP Business Application Studio:

@sap/fiori:adp generator failed - {"message":"Could not update method 'when' in 'projectType' question in generator '@sap/fiori:adp' - evaluateMethod() Cannot read properties of undefined (reading 'CLOUD_READY_AND_ON_PREM')","stack":"TypeError: evaluateMethod() Cannot read properties of undefined (reading 'CLOUD_READY_AND_ON_PREM')\n at default_1.when (/extbin/globals/pnpm/5/.pnpm/@sap+generator-fiori@1.20.3/node_modules/@sap/generator-fiori/generators/generator-adp/index.js:1:33642)\n at w.<anonymous> (/extbin/local/openvscode-server/extensions/yeoman-ui-1.20.2/dist/extension.js:2:3754248)\n at Generator.next (<anonymous>)\n at /extbin/local/openvscode-server/extensions/yeoman-ui-1.20.2/dist/extension.js:2:3748589\n at new Promise (<anonymous>)\n at r (/extbin/local/openvscode-server/extensions/yeoman-ui-1.20.2/dist/extension.js:2:3748334)\n at w.evaluateMethod (/extbin/local/openvscode-server/extensions/yeoman-ui-1.20.2/dist/extension.js:2:3753993)

This is a known issue and SAP has released a fix for it which requires customer to create a new dev space to retest the same. However, this fix has caused a different error (even with a new dev space):

Getting error at the step of 'System and Application Selection': Could not load applications: Request failed with status code 400

Environment

SAP Business Application Studio

Resolution

SAP is investigating the issue and will update this KBA when there's a solution.

For now please try the following workaround:

1. Create a new dev space 

2. Use the "Legacy" wizard as a workaround for now

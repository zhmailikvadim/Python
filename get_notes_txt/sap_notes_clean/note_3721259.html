Symptom

When executing transaction /IRM/IPBBASTC manually, the error message is displayed as expected.

However, when the same transaction is executed in the background, the error details are not shown in the job log. As a result, users are unable to identify the reason for the failure during background processing.
This note corrects the behavior by ensuring that the error details are properly captured and displayed in the background job log when executing the transaction in background mode.

Other Terms

/IRM/IPBBASTC, background execution issue, missing error details, job log issue, schedule calculation run, terminated agreements, custom logic message not displayed

Reason and Prerequisites

This is caused by Program error.

Solution

Please implement the correction instructions. These corrections are only valid if you have applied the following Support Packs.

Release V/4 – 2023 V2, 2023 V3 and 2023 V3 Support Package 1

1. Perform the manual pre-implementation steps that are described below
2. Use transaction SNOTE to implement the correction instructions specified below and activate the affected program object

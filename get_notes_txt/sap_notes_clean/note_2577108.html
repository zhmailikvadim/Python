Symptom

  • Memory leak in the SMP MBO server 3.0.13 and below versions.
  • The leak is caused by many instances of registering/unregistering devices.
  • The leak is occurring in the obmo.exe component of the MBO server
  • The leak may lead to the obmo process memory growing continuously over time up to a maximum of 4 GB.

Environment

  • SMP MBO server 2.3.x and 3.0.x on Windows OS

Reproducing the Issue

  1. Applications that do frequent register/unregister of devices will see the obmo.exe process grow in memory usage if not restarted periodically.

Cause

Device registration information is stored in an internal array but is not removed from the array when the device is unregistered.

Resolution

  1. The internal code in obmo.exe has been fixed and will be in SMP MBO server 3.0.15 currently scheduled for release in March 2018.

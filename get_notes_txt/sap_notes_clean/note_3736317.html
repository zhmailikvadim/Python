Symptom

Email notifications are not sent after an admin modifies the class agenda (for example, editing a time slot date/time, facility, or instructor).

Environment

SAP SuccessFactors Learning

Reproducing the Issue

  1. Ensure that the prerequisites mentioned on KBA 3650024 are met >
  2. Create a Class >
  3. Assign an instructor to all time slots >
  4. Enroll users >
  5. Edit a time slot (for example, change date/time, facility, or instructor) and save >
  6. See that no email notification is received by the instructor or enrolled users.

Cause

Engineering confirmed this is expected system behavior: the system does not automatically send new notifications when time slots are edited in the class agenda.

Resolution

After modifying time slots, use “Resend Registration Notification” to manually send the Registration Notifications to the enrolled users: Resending Class Registration Email Notifications

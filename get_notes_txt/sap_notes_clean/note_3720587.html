Symptom

Administrators configure custom Solr cache parameters or other Solr properties using placeholders in solrconfig.xml, for example:

 

size="${py.documentCache.size:49152}"

initialSize="${py.documentCache.initialSize:8192}"

 

Values are also defined in SAP Commerce Cloud (CCv2) Portal → Solr Service → Properties, but during runtime Solr ignores the configured values and instead always uses the fallback defaults defined after the colon in the placeholder (e.g. 49152, 8192).

After deployment or restart:

    • The expected custom property values are not applied
    • Solr does not pick up the values set in the Cloud Portal
    • Cache sizes or other configuration parameters remain unchanged
    • No errors appear in logs; Solr simply resolves the placeholders using the default value

Environment

  • SAP Commerce Cloud (CCv2)

Reproducing the Issue

  1. Add a placeholder property in solrconfig.xml, for example:
    size="${py.documentCache.size:49152}"

  2. Set the same property with different value in the Cloud Portal under
    Environment → Services → Solr → Properties, e.g.:
    py.documentCache.size=8192.

  3. Deploy the environment or restart Solr so it reloads configuration.

  4. Check Solr Admin UI or logs and observe that Solr still uses the fallback value (49152) instead of the value configured in the Cloud Portal.


    Result:
    Solr ignores the properties set in the Cloud Portal and always uses the default values from the placeholder.

Cause

  1. Solr only resolves placeholder properties in solrconfig.xml from JVM system properties, solr.xml, or configoverlay.json (set through the Solr Config API).
  2. The properties configured in the SAP Commerce Cloud Portal under Solr → Properties are not passed to the Solr JVM and therefore are not available for placeholder substitution.
  3. Because Solr cannot find the property value, it automatically uses the fallback value defined in the placeholder, such as:            
      ${py.documentCache.size:49152}
  4. As a result, Solr always applies the default value and ignores the values set in the Cloud Portal.

Resolution

To apply custom Solr property values in SAP Commerce Cloud, the configuration must be set using the Solr Config API, because Cloud Portal Solr properties are not passed to the Solr JVM.

Use the following API call to set the property:

POST /collections/config
{
  "set-user-property": {
    "py.documentCache.size": "8192"
  }
}


This writes the value into configoverlay.json, which Solr uses when resolving placeholders in solrconfig.xml.
The settings:

  • persist across deployments and restarts
  • support different values per environment (D1/S1/P1)

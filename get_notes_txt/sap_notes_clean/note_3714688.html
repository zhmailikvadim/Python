Symptom

After the implementation of SAP Note 3575511, when executing transaction code FAGLF101 (FAGL_CL_REGROUP), the posting fails due to runtime error CONVT_NO_NUMBER.

The termination occurred in ABAP program or include "FAGL_CL_REGROUP", in "BELEGE_RASTERN". The main program was "FAGL_CL_REGROUP"

Error analysis :  An attempt was made to interpret value "XXX" as a number. As this value contravenes the rules for displaying numbers correctly, this was not possible.

Environment

  • Financial Accounting
  • SAP ERP Central Component
  • SAP ERP
  • SAP S/4HANA Finance
  • SAP S/4HANA
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA

Reproducing the Issue

  1. Execute transaction code FAGLF101
  2. Insert values in mandatory fields (Company Code, Balance Sheet Key Date, Sort Method, Valuation Area and Doc.Type)   
  3. Select an option from Account Type (D/K/S)  and the Group definition

Cause

Missing code corrections in program FAGL_CL_REGROUP_BELEGE_RASTERN

Resolution

Implement latest version of SAP Note 3677144

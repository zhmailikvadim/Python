Symptom

In certain situations, processing within the FSM Connector is blocked.

The issue can be observed in transaction SM13, where a lock entry related to table /PACG/ECM_ASCII remains in status “initial”.

The corresponding update request does not complete and the lock entry is not released.

As a consequence:

  • Further processing of the affected queue is blocked
  • Subsequent messages remain unprocessed
  • Manual intervention (lock deletion) may be required

The issue occurs sporadically and cannot be deterministically reproduced.

Other Terms

FSM, Connector, SM13, blocked queue

Reason and Prerequisites

The update of table /PACG/ECM_ASCII is executed using the ABAP update task mechanism (CALL FUNCTION … IN UPDATE TASK).

In rare situations, the update task remains in an incomplete state, leaving a lock entry (SM13) in status “initial”.

The update of table /PACG/ECM_ASCII does not require asynchronous update processing and is not technically dependent on the update task mechanism.

The use of the update task in this context was not functionally required.

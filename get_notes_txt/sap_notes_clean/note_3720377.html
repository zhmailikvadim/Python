Symptom

The notification IDs ApmTrainingExpiration and ApmTrainingOverdue have a token named "User Name (Legacy)". Are there other tokens that can be used to fetch the users' names?

Environment

SAP SuccessFactors Learning

Resolution

The tokens which contain the word "(legacy)" are legacy tokens and are deprecated. These shouldn't be used in the templates and will be removed from the system in coming releases.

Please note that, for the ApmTrainingExpiration and ApmTrainingOverdue templates, the token "User Name (Legacy)" has been replaced by 4 user name tokens for each assignment type ("User Name for Assigned Items not in a Curriculum or Program", "Program User Name", "Curriculum User Name", and "Requirement Learner Name"). For example:

  • Token "Curriculum User Name", which should be used within the loop section for curriculum records and will display if user has any pending curricula records.

When looking at the default templates of ApmTrainingExpiration and ApmTrainingOverdue, there are 4 loops (one for each assignment):

  1. For free floating items (i.e., which are assigned directly and not part of any curricula, program, nor requirement);
  2. For items which are part of curricula;
  3. For Items which are part of program;
  4. For items which are part of requirements of curricula.

Even though it has 4 loops, only the ones for which the users has assignments that meet the criteria will be present in the notification. For example:

  • If the user is assigned with a program, then only the program loop will be executed and the email will be generated for it. All other loops will automatically be ignored by the system.

It's important to highlight that each loop has its own set of tags, which need to be used within the respective loops.

To suggest different system behavior, it would be possible to have enhancement requests created (as per KBA 2090228), where the Learning Product Management team can further review and assess the requested functionality to be considered in a future release.

IMPORTANT:

Although customers are allowed to modify and customize the standard notifications, if there are issues with templates that are modified, this is beyond scope for Product Support team (2706322) since it requires specific knowledge of hypertext markup language (HTML) the SAP Support team doesn't have the necessary insights to assist on custom notification troubleshooting.

Symptom

When clicking on Launchpad > transport within a specific launchpad in the LPD_CUST transaction, the below message error appears after adding Package and Namespace information:

SAP object DOCT XXXXXXXXXXXXX cannot be assigned to package &3

Message No. TO142

Diagnosis

Object DOCT XXXXXXXXXXXXX  is in the name range reserved for SAP. The object
cannot be assigned to package /SAPPCK/, since it belongs to a different name range or
namespace.

System Response

Editing is terminated.

Procedure

Assign the object to an SAP package whose name begins with A-S or U-X.
If you want to create the object as a test object, you can also assign it to a local package
(1st character = $) or a test package (1st character = T).

Observation: package "/SAPPCK/" is only used as an example in this documentation.

Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.

Environment

  • SAP S/4HANA 
  • ABAP Platform
  • SAP NetWeaver

Reproducing the Issue

1. Try to transport an specific launchpad via LPD_CUST:

2. Add Namespace and Package information:

3.Error message "Message No. TO142 (SAP object &1 &2 cannot be assigned to package &3)" is displayed:

Cause

For a specific launchpad (e.g HRPAO / XYZ_ABCD)  there might be entries with SAP object 0 in APB_LPD_OTR_KEYS  table which do not have the customer namespace (e.g,  /SAPPCK/) as prefix of the text ID:

MANDT ROLE       INSTANCE                    SAP_OBJECT            TEXT_ID

100   HRPAO       XYZ_ABCD                           0                      /SAPPCK/LEHI000004U100074
100   HRPAO       XYZ_ABCD                           0                      SAPPCK/LEHI000004U100078
100   HRPAO       XYZ_ABCD                        0                      LSD10000002000026
100   HRPAO       XYZ_ABCD                        0                      LSD10000002000027

Observation: HRPAO, XYZ_ABCD and /SAPPCK/ are only examples within this KBA.

Resolution

These textkeys are likely not (no longer) used in the launchpad by checking launchpad XML in transaction LPD_CUST. Deleting the those entries should solve the transport issue.

As the texts are not used, the respective entries can be deleted.

Symptom

1. Restart occurred in the SAP HANA high availability cluster following an OS upgrade.

2. The restart was triggered by the cluster and not the database itself.

3. The SAP HANA resource agent initiated a clean stop and subsequent restart of the system.

Environment

SAP HANA High Availability (System Replication, DR, etc.)

Reproducing the Issue

  1. Perform an RHEL OS upgrade on the secondary node of the SAP HANA cluster.
  2. Observe the restart of the primary node upon exiting maintenance mode.

Cause

The restart was triggered by the SAP HANA resource agent as part of the standard system procedure during the OS upgrade.

Resolution

  1. Restart the entire cluster simultaneously when replication is enabled to avoid unexpected restarts.
  2. Verify the system logs to confirm the sequence of events during the OS upgrade and SAP HANA restart.
  3. This seems as a normal system procedure which takes place when the operating system is upgraded.
  4. As the SAP Hana resource agent initiated the clean stop ("Stop OK" ), which led to sapstart sending SIGINT, which led to SIGTERM shutdown of all HANA processes, node reboot for the OS upgrade, and then Pacemaker automatically restarting HANA ("Start OK") with successful site resync.

Symptom

When executing the overview screen in transaction KP06 with the planner profile X_SAPALL, an error is encountered: "Excel couldn't be started," with error number K9356.

Environment

  SAP ERP
  SAP enhancement package for SAP ERP
  SAP enhancement package for SAP ERP, version for SAP HANA
  SAP S/4HANA
  SAP S/4HANA Finance
  Controlling CO

Reproducing the Issue

1. Start transaction KP06.
2. Set Planner Profile X_SAPALL.
3. Select the following data:
- Version: 
- Period: 
- Period: 
- Fiscal Year: 
- Cost Center: 
- Cost Element:
4. Execute the transaction.

Cause

The issue is related to the compatibility between the 64-bit SAP Business Client and the 32-bit MS Excel 2016. A 32-bit Microsoft Excel process cannot natively invoke 64-bit ActiveX/OLE components (specifically the Logon, Table, Function, and BAPI controls) that are bundled with the 64-bit version. 

Resolution

1.  Install the latest SAP GUI for Windows patch (e.g., version 8.00) from the SAP Support Portal.
2.  Ensure compatibility by installing the 64-bit SAP GUI patch for the 64-bit SAP Business Client.
3.  For future rollout, use the 64-bit version to avoid OLE communication issues with the 64-bit SAP installation.

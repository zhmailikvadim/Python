Symptom

  • In an SAP Cloud Integration scenario using the JMS Sender Adapter, messages are processed in parallel even though the “Number of Concurrent Processes” parameter is set to 1 in the adapter configuration.
  • The issue occurs when the “Access Type” parameter is maintained as “Non-Exclusive” and the tenant landscape consists of two or more worker nodes.

Environment

  • SAP Cloud Integration
  • BTP Integration Suite

Cause

The “Access Type” parameter in the JMS Sender Adapter is set to “Non-Exclusive.” With this setting, multiple worker nodes are allowed to read and process messages from the queue in parallel. This behavior is by design and is intended for scenarios where message sequencing is not required and parallel processing is desired to improve throughput.

Resolution

Maintain the “Access Type” parameter as “Exclusive” in the JMS Sender Adapter configuration. With this setting, only one worker node is granted access to the queue at a time, preventing parallel processing across multiple nodes. This configuration ensures sequential message consumption and is recommended for scenarios where message order must be preserved.

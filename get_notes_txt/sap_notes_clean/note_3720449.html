Symptom

Certain fields (payment terms (ZTERM), fixed value date(VALDT), additional value days(VALTG) etc.) are not captured in the change log during the creation of condition records using VK11. When these fields are modified later using t-code VK12, changes are correctly captured in the change log.

Environment

•Sales And Distribution (SD)
•Logistics Execution (LE)
•SAP R/3
•SAP R/3 Enterprise 4.7
•SAP ERP Central Component
•SAP ERP
•SAP Enhancement package for SAP ERP
•SAP Enhancement package for SAP ERP, version for SAP HANA
•SAP S/4HANA

Resolution

When creating a condition record (e.g. VK11), only some changes are captured in the change log. If it is needed to change the standard behavior, please refer to Note 367203 which introduces the change and consider a code modification like to clear payment terms field (CLEAR TKONP-ZTERM). Also refer to a similar change done when creating condition record using IDoc's in note 3136935. For dealing with modification, please also consider information from note 170183.

Keywords

Symptom

  • Existing implementations of BAdI RPM_OBJECT_SERVICES are not being processed after a system upgrade.
  • Custom logic for portfolio item numbering is not being processed as a result.

Environment

  • SAP S/4HANA
  • SAP S/4HANA Cloud Private Edition 

Reproducing the Issue

  1. Customer logic for Portfolio Item numbering is implemented in BAdI RPM_OBJECT_SERVICES method GET_NEXT_NUMBER_FOR_EXTID.
  2. Create a Portfolio Item
  3. Observe that the customer logic for item numbering is not processed.

Cause

  • BADI RPM_OBJECT_SERVICES was converted from a classical BADI to a Kernel BAdI (Enhancement Spot) with SAP S/4HANA release 2023 FPS03.
  • Existing BAdI implementations must also be migrated as a result.

Resolution

Implement the custom code in an implementation of the enhancement spot (i.e. kernel BAdI) RPM_OBJECT_SERVICES.

See Also

Refer to topic Migrating BAdIs in the SAP Help for instructions how to migrate/copy a BAdI implementation following the BAdI migration from classic BAdI to kernel BAdI.

Symptom

Approval task for Supplier Invoices is sent to the manager of the Project Responsible instead of the Project Responsible.

Environment

SAP Business ByDesign

Reproducing the Issue

  1. Go to Application and User Management work center.
  2. Go to Approval Process view.
  3. Select the active Supplier Invoice approval process.
  4. Choose view button.
  5. Verify the work distribution is maintained as "Approval by Cost Center Manager based on Project Task".
  6. Condition is maintained for the step is "Account Assignment Type[] contains Project Task".

  1. Go to the Supplier Invoicing work center.
  2. Go to the Invoices and Credit Memos view.
  3. Open Supplier Invoice ABC (ABC represents Supplier Invoice ID).
  4. Navigate to the Approvals tab.
  5. Approver is the manager of the Project Responsible.

Cause

The rule approval by Cost Center Manager based on Project Task checks and assigns the approval task to the first person found based on the following checks:

  1. Cost Center Manager by project task.
  2. Project task Manager.
  3. Project Manager.

The approval rule will work based on the Org Structure. For Example,

Project EFG (EFG represents Project ID) is created with the Responsible Unit as Org Unit MNO (MNO represents Org Unit ID) and Person Responsible as JKL (JKL represents Employee ID).

NOTE: Org Unit MNO and Person Responsible JKL belongs to the One Single Org Unit PQR (PQR represents Org unit ID) which has a manager.

Org Unit MNO does not have a manager. So, if a Supplier Invoice ABC is created with the Account Assignment Type as "Project EFG". Since no Cost Center manager is available for the Org Unit MNO. Expecting the Project Responsible to be the approver. But if no Cost Center Manager is available, system will check the higher-level Cost Center Manager.

In this Scenario, the Higher-level manager and Person Responsible's manager are the same person. According to the Structure of the Org Units, the approval is triggered for the higher-level Cost Center Manager.

Resolution

This is the expected system behavior. If there is no Org Unit was assigned to the Project Task, then the system will trigger the approval to Project Person Responsible.

According to the Orgs structure explained above, currently there is no standard approval rule to determine the Project Manager as the Approver. Turn to SAP BYD Influence Forum for new requirement proposals: https://influence.sap.com/sap/ino/#/campaign/896

Symptom

In the technical settings of RKO7CJ8G, there are No. of Senders, CO object for trace and No. Work Processes options.
What is the upper limit for setting the number of senders?

Environment

  • Financial Accounting (FI)
  • SAP R/3
  • SAP R/3 Enterprise 
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA Finance
  • SAP S/4HANA

Resolution

Regarding the number of senders: This number means after how many processed WBS Elements a commit will be done. It can help with performance only in some specific cases. The long text of this field it says that when settling to assets this field is not relevant anymore. It is relevant for receivers such as cost centers, orders and so on.

Note
If a sender is settled to a fixed asset or a material, the settlement result is always posted immediately, as the fixed asset or the material would otherwise be locked for the next sender. The number of senders to be posted together is only considered for other receivers such as cost centers, orders and so on.

Therefore user can keep this setting as default, 10. 

There is no upper limit defined for this setting, this needs to be determined via testing.
User should focus on determining the most optimal number of parallel processes for performance optimization.

Keywords

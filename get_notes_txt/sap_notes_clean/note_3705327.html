Symptom

  • During outbound processing, cumulative delivery and goods issue quantities are being incorrectly updated, resulting in doubled or tripled values depending on the number of deliveries and goods issue/reversal postings.
  • The issue affects table S073, causing incorrect cumulative quantities in sales scheduling agreements.

Environment

  • Sales And Distribution (SD)
  • Logistics Execution (LE)
  • SAP S/4HANA
  • SAP S/4HANA Cloud 
  • SAP ERP Central Component
  • SAP ERP
  • SAP Enhancement package for SAP ERP
  • SAP Enhancement package for SAP ERP, version for SAP HANA

Reproducing the Issue

  1. Go to Transaction VA32, and open the document.
  2. Double click on the item to go to item data.
  3. Go to Forecast delivery schedule and click on deliveries.
  4. Verify the delivery cumulative quantity.
  5. Click on sales document and then Deliver.
  6. Choose a quantity and save the document.
  7. Verify again the cumulative delivery quantity.
  8. Verify also in table S073 by transaction SE16.

Cause

  • The statistics table S073 is updated with function module MCV_STATISTICS_DELIVERY, thereby the parameter XLIPS and YLIPS must be filled correctly. XLIPS contains the updated delivery item data and YLIPS contain the old delivery item data from database.
  • The database table S073 is always updated, if a delivery with reference to the scheduling agreement is created or changed. 
  • If there is any modifications (normally in MV50AFZ1), in which XLIKP/XLIPS table(s) will be updated, but corresponding YLIKP/YLIPS entries have not been recreated. 

Resolution

  1. Use correction report SDLIPL01 to rectify the data and ensure proper updates to S073 using MCV_STATISTICS_DELIVERY with correctly filled XLIPS and YLIPS parameters.
  2. Investigate potential modifications in MV50AFZ1 that may cause mismatches between XLIKP/XLIPS and YLIKP/YLIPS tables.

Symptom

As of August 22, 2023, SAP Swissdec wage standard ELM 5.0 has been certified with the component PY-CH Swissdec ELM 5.0.

In accordance with the Swissdec guidelines for wage data processing (Version 5.0), retroactive amount changes must be processed for the monthly ELM statistics notification (domain BFS).

You have already implemented the related SAP Note 3585246 so that retroactive number changes (cumulation wage types CWF1 and CWF2 of the subapplication BFS1, view V_T596J) are reported in the notification month in accordance with the realization principle. However, enter or correct the hours worked for hourly wage earners (or lessons) retroactively for several periods, the values (field ANZHL) may be reported several times (cumulated) in the notification month:

  • Cumulation wage type CWF1: Number of hours worked <TotalHoursOfWork>
  • Cumulation wage type CWF2: Number of lessons worked <TotalLessonsOfWork>
  • Number of Hours Worked and Lessons <TotalHoursAndLessonsOfWork>

Other Terms

Payroll Switzerland, ELM ("Uniform Wage Notification Procedure"), ELM 5.0, domain: BFS/Statistics, class: CL_HRPAYCH_ELM50_DAQ_BFS (method: 'PROCESS_RETRO_CHANGE')

Reason and Prerequisites

Program correction

Solution

Import the HR Support Package relevant for your release. If you have to implement the solution in advance, use the SAP Note Assistant to implement the correction instructions relevant for your release. You can find more information about the SAP Note Assistant on SAP Support Portal at https://support.sap.com/notes.

After you implement this SAP Note, number changes entered retroactively (cumulation wage types CWF1 and CWF2 of the subapplication BFS1) are no longer cumulated several times. This also prevents negative hours/lessons from occurring.

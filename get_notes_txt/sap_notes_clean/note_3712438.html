Symptom

  • IDoc is not generated when performing a change in the article hierarchy, specifically a node description change.
  • Change pointers for message type WMATGRP are not created in BDCP2 table during the copying of changes from planned hierarchy to active hierarchy.
  • BD21 execution does not result in IDoc creation for node description changes.

Environment

  • Retail
  • SAP ERP
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA
  • SAP S/4HANA Cloud Private Edition

Reproducing the Issue

  1. Select the planned article hierarchy.
  2. Perform a description change in node using WMATGRP02.
  3. Copy the changes into the active article hierarchy using WMATGRP06N.
  4. Run RBDMIDOC.
  5. Observe that no BDCP2 entries for message type WMATGRP are created, and IDoc is not generated.

Cause

  • When node description changes are copied to the active hierarchy, the system generates change pointers with the message type CIC_ARTICLE_HIERARCHY instead of WMATGRP in the BDCP2 table.
  • These change pointers are not considered during BD21 execution, as BD21 processes change pointers only for the specified message type WMATGRP.
  • As outlined in the SAP Help Portal - Communication of Changes to the Article Hierarchy, only the following types of changes are communicated within an article hierarchy:
    • Shifting of existing article hierarchy elements (nodes or articles)
    • Setting a deletion indicator for article hierarchy elements whose assignment validity has expired
    • Reversal of a deletion flag by extending the validity (valid-to) date
    • Physical deletion of article hierarchy elements (nodes or articles)

        In summary, any change related to structure and article assignments in an active article hierarchy.

Resolution

Standard behavior.

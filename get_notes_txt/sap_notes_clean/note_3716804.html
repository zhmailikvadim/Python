Symptom

  • When updating a position via quick action Position to Job Info sync is not triggered
  • This causes other rules to potentially trigger unwanted validation errors

Environment

SAP SuccessFactors Employee Central

Reproducing the Issue

  1. Go to employee record.
  2. Select Quick Action > update position
  3. Select position.
  4. Click Save.
  5. Validation error occurs; rule checks old position instead of the new one.

Cause

  • Position A & Position B have the same external name but different external codes.
  • In certain scenarios, the UI does not register the position change hence not triggering the rule.

Resolution

Engineering are investigating the issue, KBA will be updated with permanent fix.

Workaround:

  1. Clear the position field in the Quick Action transfer before selecting a new position.
  2. Once the position field is empty or null, re-select the new position.
  3. Save the changes.
  4. The sync rule should trigger correctly, avoiding the validation error.

Symptom

The contract is generated in English instead of Spanish, even the logon language is Spanish.

Environment

  • Human Capital Management (HCM)
  • SAP R/3
  • SAP ERP Core Component
  • SAP ERP
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP SuccessFactors Employee Central Payroll - Spain

Reproducing the Issue

  1. Log in to the SAP system in ES language
  2. Select employee ID XXXX.
  3. Execute transaction PA30.
  4. Navigate to Infotype 0016.
  5. In the print tab, visualize the form.
  6. The contract is in English language.
  7. Execute transaction SFP.
  8. Choose the contract name type you get in English language (HR_ES_COINEM_PE_[TYPE]).
  9. In the Original Language field you are going to see EN, instead of ES.

Cause

When creating the contract HR_ES_COINEM_PE_[TYPE], using the T-CODE SFP, you are log into the system in English language.

Resolution

  1. Log on to the system in ES language.
  2. Execute transaction SFP.
  3. Delete the contract type with the language error (HR_ES_COINEM_PE_[TYPE]).
  4. Create the form from scratch following the manual steps of the note in which the contract was delivered.

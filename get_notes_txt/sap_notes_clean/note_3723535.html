Symptom

  1. You create a new wage type (for example, ZA14). In the Complete View T512W (V_512W_O) view, you set the following for this wage type:
    • Specification of EA Forms (06) evaluation class: Benefit of Leave Passage for Travel (2E)
    • Specification of Tax Exempt Allowance Processing (70) processing class: blank
  1. You run Malaysia payroll driver (RPCALCL0) report for an employee to calculate this wage type with a customized payroll schema. In the payroll result (RT) table:
    • Variable assignment number (V0ZNR) field is not blank.
    • HR Payroll: Amount (BETRG) field is 7800.
  1. You run the EA or EC Annual Tax Report (RPCTEAL0_01) report for this employee for tax year 2025, the value displayed on B3 Benefits in kind field of EA Form is not 7800.

After you've installed this SAP Note, the value becomes 7800 as expected.

This SAP Note also enhanced the Getting the PDF stream for EA (HRXSS_MY_EA_GET_PDF) function module and Getting the PDF stream for PCB2 (HRXSS_MY_PCB2_GET_PDF) function module. 2 new export parameters were added: PDF_TAB and REF_TAB.

Other Terms

EA or EC Annual Tax Report, RPCTEAL0_01, 2E, EA2E, V0ZNR, HRXSS_MY_EA_GET_PDF, HRXSS_MY_PCB2_GET_PDF, SALV, 14REP, T512W, RPCPDFL0

Reason and Prerequisites

This is a program error. You have installed SAP Note 3397634.

Please notice the EA Form (C.P.8A - Pin. 2016) removes the field B2(e) Benefit of leave passage for travel that exists in EA form (C.P.8A - Pin. 2010). Therefore, for leave passages, it is not recommended to set any wage type with specification Leave Passage for Travel (2E) of EA Forms (06) evaluation class after year 2016.

From 2016 onward, use one of the following options for leave passages:

  • Use a wage type with specification Leave passages (N) of Tax Exempt Allowance Processing (70) processing class to indicate such kind of incomes. If the annual total amount of leave passages is more than MYR 3000, the exceeding part will display on B3 Benefits in kind of EA Form.
  • Or set the specification of EA Forms (06) evaluation class as Benefit in Kind (B1) for those wage types that you want to display on B3 Benefits in kind of EA Form.

Solution

You can apply HR Support Package or implement the correction instructions in this SAP Note and carry out the manual activities attached.

Workaround: set the specification of EA Forms (06) evaluation class as Benefit in Kind (B1) for those wage types that you want to display on B3 Benefits in kind of EA Form.

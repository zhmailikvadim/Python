Symptom

Support component does not exist when trying to configure Automatic case Creation

Environment

SAP Solution Manager 7.2

Reproducing the Issue

  1. Execute transaction solman_setup
  2. Select Application Operations
  3. Go to System Monitoring
  4. Select 2 Configure Infrastructure
  5. Go to Step 2.3 Default Settings
  6. Select incidents Tab and Edit mode
  7. Configure Automatic incident Creation "Active"
  8. Select Support Component e.g: SV-SMG-MON-SYS
  9. Error Message "Support Compnent does not exist"




    or
  10. Search: Support Component
    "No components exist!"

Cause

Missing entries in tables DSWPCSNCOMPTXT and DSWP_CSNCOMP where SAP component data is stored

Resolution

  1. Check if job SM:GET CSN COMPONENTS is running in t-code SM37.
    It gets the application components from CSN, it is scheduled every week.
  2. If it is not, call transaction SE38
  3. Execute (F8) report DSWP_GET_CSN_COMPONENTS

See Also

KBA: 2104380 - Empty result when using F4 on Component field while creating incident / SAP Collaboration AB in Solution Manager

Keywords

SOLMAN, DSWPCSNCOMPTXT, DSWP_CSNCOMP, DSWP_GET_CSN_COMPONENTS, SM:GET CSN COMPONENTS

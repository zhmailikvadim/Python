Symptom

In the "Billing Data Analytics" app, when modifying the currency, the message below popups:

  • Could not convert from '<Currency1>' to '<Currency2>'. Please provide missing rates for all needed types, categories and versions in the rate table.

And the billing analytics graphs and details are not displayed.

Environment

SAP Subscription Billing

Reproducing the Issue

  1. Navigate to the Billing Data Analytics app.
  2. Change the display currency.
  3. Observe that the app reports a rate conversion error and analytics do not render.

Cause

  • Currency exchange rates were not configured for the involved currencies when switching analytics to a different target currency.
  • Exchange rates were not maintained for the earliest billing date present in existing billing data.

Resolution

  1. Configure currency exchange rates using one of the following options:
    • Use the Exchange Rates API to create and manage exchange rate data for the target currency. 
    • Replicate exchange rates via SAP Master Data Integration to synchronize exchange rate information. 
  2. Ensure that exchange rates exist for all relevant currency pairs and cover the earliest billing date contained in existing billing data.
  3. Reopen the Billing Data Analytics app and verify that analytics graphs and details are displayed without rate conversion errors.

Symptom

Employee’s Unemployment Insurance premium is deducted even though the employee has become "taisyogai" in IT0140.

Environment

  • Human Capital Management (HCM)
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for HANA
  • SAP S/4HANA
  • SAP S/4HANA Cloud Private Edition
  • SAP SuccessFactors Employee Central Payroll

Reproducing the Issue

  1.  Maintain Infotype 0140 (Unemployment Insurance) as below:
      

    Sub type              start date           end date         Indicator for deduction

    4        EE's ins.        21.04.2026        31.12.9999

    4        EE's ins.        01.07.2025        20.04.2026        X



  2. Run payroll for 05/2026 and observe that the employee's Unemployment Insurance premium is not calculated. 
  3.  Run payroll for 06/2026 and note that wage types /310, /311, and /312 are calculated.

Cause

A retroactive calculation has occurred, resulting in the calculation of retro wage types. The system checks the status of LI for each pay date. If the wage types were created for an insured period, even after the “taisyogai” indicator is set, the system calculates the premium accordingly. 

Resolution

  • Verify if there is a retro calculation for the insured payroll period.
  • Check if wage type /1E7 matches the retro difference for the relevant period.

Keywords

 Unemployment Insurance, IT0140, P0140, /310, /311, /312, Premium calculation.

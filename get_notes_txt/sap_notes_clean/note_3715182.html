Symptom

When sending a reversal print document, the system successfully sends the reversal print document even though the original print document received an APERAK/CONTRL message, which should prevent the reversal print document from being sent.

This issue is relevant for the market role: DSO.

Other Terms

GUB, APERAK, CONTRL, Reversal, Print Document, /US4G/CL_GU_BILLING_API, /US4G/IF_GU_BILLING_API_PROC~TRIG_INVOIC_PROC, CHECK_ORIG_GU_STATUS_EXISTS, /us4g/proc_billing200

Reason and Prerequisites

The error occurred because the system was not checking the process business status of the sent original invoice when sending the reversal invoice.

Solution

Implement the corrections of this SAP Note or import the relevant support package.
After implementing this SAP Note, the system raises error message /us4g/proc_billing200 when sending the reversal invoice if the original invoice received a CONTRL/APERAK message.
You need to re-send the reversal invoice.
You can verify the fix by checking that error message /us4g/proc_billing200 is raised when sending the reversal invoice if the original invoice received a CONTRL/APERAK message.

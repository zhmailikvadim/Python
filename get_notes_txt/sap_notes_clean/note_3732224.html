Symptom

There is a requirement to enter fields (e.g. account assignments) for the accounting document that do not exist on the standard input screen in the Fiori apps for creation payment requests.

Environment

payment program/payment request

Reproducing the Issue

Create a free form payment using Process Free Form Payments/App ID F8654 or My/Process Free Form Payments/App ID F2564

Cause

  • For native Fiori apps (for example, Process Free Form Payments/App ID F8654 or My/Process Free Form Payments/App ID F2564) user interaction via SAP GUI- dialog boxes are not supported; additional fields must be filled programmatically, otherwise required fields (such as Profit Center) remain empty and cause a mandatory-field error.

Resolution

  1. Before S/4HANA 2020 the only way is to implement the BAdI FIBL_OPAY_POST to fill fields programmatically; suppress any calls to SAP GUI dialog boxes as they are not supported by Fiori (refer to SAP Note 1656961)
  2. From S/4HANA 2020
    there is also the possibility to extend App Process Free Form Payments and/or My Free Form Payments using the documented app extensibility options App Extensibility: Free Form Payments

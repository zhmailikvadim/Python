Symptom

  • I have submitted an item and it cannot be approved because it went into Approval Paused status
  • Invoice is stuck in Approval Paused status
  • Miscellaneous invoice is stuck in Approval Paused status
  • Job Posting is stuck in Approval Paused status
  • Work Order is stuck in Approval Paused status
  • Time Sheet is stuck in Approval Paused status
  • Expense Sheet is stuck in Approval Paused status
  • SOW is stuck in Approval Paused status

Environment

Production and Non-production

Reproducing the Issue

  1. Submit an item such as a Time Sheet or a Job Posting in SAP Fieldglass
  2. At any approval level, the item gets stuck in Approval Paused status

Cause

  • The User Role associated to the current approver has no Approve/reject permission for the module of this item
  • The approver has no enough Signature Authority to approve this item
  • If this level must be approved by a specific User Role, there are no users associated to this role or the user do not have visibility on the item. The visibility issue could be related to the users not having the same Site, Location, Business Unit, Legal Entity or if user are MSP Users, not having the same MSP Company

Resolution

  • Assign the User Role Permission

    1. Sign into your SAP Fieldglass account
    2. Admin Configuration (Gear Icon) > User Role > Edit
    3. Locate the respective Module for the Item (e.g. Time Sheet, Job Posting, Work Order...)
    4. Check the Approve column check box > Update

  • Provide enough Signature Authority for an user

    1. Sign into your SAP Fieldglass account
    2. Admin Configuration (Gear Icon) > User > Edit
    3. Locate the Signature Authority for the respective currency
    4. Adjust the Amount > Update

  • Add the missing admin item association

    1. Sign into your SAP Fieldglass account
    2. Admin Configuration (Gear Icon) > User > Edit
    3. On the left side menu, go to the Admin Item that is missing > Add > Update (Repeat this action for each missing association)
      • Note: Once a buyer user is created, the MSP Company cannot be updated for security reasons. To have this field changed on the user, close the existing user and create a new user with the correct MSP Company

Once completing the necessarie above action, open the record in Approval Paused, go to the Approvals / Audit Trail tab and click Restart

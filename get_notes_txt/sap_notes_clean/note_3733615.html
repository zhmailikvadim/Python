Symptom

After upgrading the system the currency does not appear in transaction FF67.

Environment

SAP S/4HANA

Reproducing the Issue

  1. Upgrade the system to a newer release.
  2. Open transaction FF67 (Manual Bank Statement).
  3. Observe that the currency field is not populated.

Cause

The currency key in table T012K (WAERS) was not populated after the system upgrade, although it should be derived from FCLM_BAM_AMD-WAERS.

Resolution

  1. Review KBA 3294355 for the described issue and the provided correction report.
  2. Implement and run the correction report that attached to KBA 3294355 to perform a mass correction so that T012K-WAERS is filled from FCLM_BAM_AMD-WAERS.
  3. Verify that table T012K now contains the correct currency key (WAERS) for the relevant entries.
  4. Recheck transaction FF67 to confirm the currency is now displayed as expected.

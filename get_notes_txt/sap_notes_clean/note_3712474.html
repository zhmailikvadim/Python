Symptom

  • The user is seeking guidance on configuring an OIDC connection in SAP Customer Data Cloud (CDC) and referencing it in a Screen-Set.
  • The user is unfamiliar with the setup process and requests detailed instructions.
  • The user has reviewed SAP CDC documentation but could not find sufficient guidance.
  • The user has inquired about how to enter Redirect/Callback URIs from SAP CDC and obtain the URL.

Environment

  • RaaS (Screen-Sets, Site Policies, Schema).

Reproducing the Issue

  1. Navigate to the SAP Customer Data Cloud (CDC) admin console.
  2. Attempt to configure an OIDC connection under Identity → Connect → OpenID Connect (Relying Party).
  3. Attempt to reference the OIDC connection in a Screen-Set.
  4. Attempt to enter Redirect/Callback URIs from SAP CDC and obtain the URL.

Cause

Resolution

  1. Open the SAP Customer Data Cloud (CDC) admin console.
  2. Navigate to Identity → Connect → OpenID Connect (Relying Party).
  3. Create a new Relying Party entry with the following attributes:
  • Client ID: As issued by the external identity provider (IdP).
  • Client Secret: Secret from the external OIDC provider.
  • Redirect/Callback URLs: Where the IdP must send responses after authentication.
  • Scopes: Typically include "openid", "profile", "email", etc.
  1. Save and enable the Relying Party configuration.
  2. Obtain the Redirect/Callback URI from SAP CDC:
  • Open the OIDC connection and save it. The Redirect URI appears only after saving.
  • On the connection details page, copy the "Redirect URI" (read-only). Do not modify it; it is data-center specific.
  1. Enter the Redirect/Callback URI in the external IdP:
  • Register a new OIDC application/client.
  • Enter the Redirect/Callback URIs from SAP CDC.
  • Configure required scopes and client credentials.
  • Enable the Authorization Code flow (with PKCE if desired).
  1. Reference the OIDC connection in the Screen-Set:
  • Go to User Interface → UI Builder in the CDC console and open the relevant Screen-Set (e.g., login Screen-Set).
  • Add or enable a social or external login button that initiates the OIDC flow.
  • Specify the provider key created in the OIDC Relying Party configuration into the Screen-Set’s social providers widget or custom input.
  1. Verify and test the setup:
  • Navigate to the site’s login page.
  • Choose the new OIDC login option.
  • Confirm that the redirect goes to the external IdP, authentication succeeds, and CDC receives and processes the login token.

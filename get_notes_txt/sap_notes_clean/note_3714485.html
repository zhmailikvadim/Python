Symptom

  • This is a standard system limitation in the current SAP S/4HANA version.
  • The Bank Area value help is not supported correctly in the affected Fiori applications.
  • Although configuration is correct, the system does not fetch or display Bank Area values.
  • A relevant SAP correction note exists, but cannot be implemented due to version incompatibility.

Environment

SAP S/4HANA 2023

Reproducing the Issue

  • Log in to the SAP Fiori Launchpad.

  • Open the Fiori application “Manage In-House Bank Accounts”.

  • Choose Create to create a new In-House Bank Account.

  • In the creation screen, click on the “Bank Area” field to select a value.

  • Observe that the value help list appears empty, and no Bank Area values are displayed.

Cause

  • Although Bank Area is correctly configured in the system, users cannot select Bank Area in any Fiori application, for example, “Manage In-House Bank Accounts”.

  • The issue occurs because the authorization role does not contain explicit values for the Bank Area field (/PF1/BKARE) in authorization object /PF1/IHBBK.

  • If /PF1/BKARE is left unrestricted, the system cannot derive the Bank Area context, resulting in empty value help and no selectable values across all related Fiori apps.

Resolution

  • Although the authorization object /PF1/IHBBK is assigned and activity is maintained, the Bank Area field (/PF1/BKARE) is not restricted to any value.

  • For IHB / Fiori applications, the system requires explicit Bank Area values to be maintained in the role.

  • If /PF1/BKARE is left without restriction, the authorization check cannot derive the Bank Area context, and therefore no values are displayed in the app or F4 help.

  • Maintain the required Bank Area value(s) explicitly in field /PF1/BKARE, regenerate the role, and re-test the application.

Symptom

You would like to influence the postings in EWM so there is a check to confirm if the posting can happen on ERP side.

Environment

  • SAP Extended Warehouse Management (SCM-EWM);
  • SAP Extended Warehouse Management Interfaces (SCM-EWM-IF);
  • SAP S/4HANA.

Resolution

For error  M7174:

Consider customizing the error in table T160M to prevent it from being raised.

If the error check is required, explore the following options:

  • For Embedded EWM: Activate synchronous Goods Movements to raise the error during the EWM Posting Change, preventing the posting change on both sides.
  • For non-Embedded EWM: Implement BAdI /SCWM/EX_CORE_GM_STOCK, method CHECK_MOVEMENT, to reject posting changes for products with differing serial number profiles.

To check if GR/GI reversal is possible on ERP side:
You may use one of the below BAdIs to check if the posting in ERP is allowed or if the queue would get stuck in SMQ2:

  • BAdI /SCWM/EX_CORE_GM_STOCK method CHECK_MOVEMENT;
  • BAdI /SCWM/DLV_GM_PROCESS (or /SCWM/EX_DLV_I_GM) method CHECK_DOCUMENT_CANCEL;

Because of the above mentioned complexity SAP cannot provide any template code for the BAdI.

Alternatively, Synchronous Goods Movement (Sync GM) has been made available so a direct goods movement posting from EWM to ERP does not use qRFC communication and immediate post goods movement in both applications.
See KBA 3522773 - 'Synchronous goods movement ERP > EWM and EWM > ERP' for more information

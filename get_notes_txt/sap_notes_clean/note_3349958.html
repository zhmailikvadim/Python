Symptom

"Request is unparsable, syntactically incorrect, or violates schema." error on attempt to use api for Read Job

Environment

 SAP CLOUD PLATFORM

Reproducing the Issue

  1. Create a Connection from IAS as Source to SAC as Target.
  2. Enable for SAC the new scim api
    sac.api.version=2
    csrf.token.path=/api/v1/scim2/Users?count=1
  3. Refresh the Transformation
  4. Start a Read Job (Simulation Job ends with no ERROR)
  5. Read Job produces Error:

    error=org.apache.camel.http.base.HttpOperationFailedException: HTTP operation failed invoking https://....hcs.cloud.sap/api/v1/scim2/Groups with statusCode: 400, Response: {"schemas":["urn:ietf:params:scim:api:messages:2.0:Error"],"status":"400","scimType":"invalidValue","detail":"Request is unparsable, syntactically incorrect, or violates schema."},

Cause

There are special characters like dash or apostrophe used in Team or User Name 

Resolution

Replace the special characters with Underscore or other allowed characters.

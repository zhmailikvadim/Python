Symptom

When creating an accounting document through an IDoc, the VAT registration number (BSEG-STCEG) is absent in the accounting document. Even after implementing customer enhancements such as a user exit to pass BSEG-STCEG, the IDoc fails with the error message: “No batch input data found for dynpro SAPMF05A 0300. SAPLFCPD 0100 expected.” The posting process via FB01 attempts to fill BSEG-STCEG on the CPD screen (SAPLFCPD 0100), which is not accessed for standard customer postings. 

Environment

  • Financial Accounting (FI)
  • SAP ERP
  • SAP ERP Central Component
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA
  • SAP S/4HANA Finance
  • SAP Fiori for SAP S/4HANA
  • SAP S/4HANA Cloud Private Edition

Reproducing the Issue

  • Process an inbound FI posting IDoc that posts via FB01 (e.g., reprocess using BD87).
  • Implement logic (like a user exit) to populate BSEG-STCEG (VAT registration number) for a standard (non-one-time) customer.
  • Observe the absence of a VAT registration number in the accounting document, or encounter a posting failure with “No batch input data found for dynpro SAPMF05A 0300. SAPLFCPD 0100 expected."

Cause

Custom logic in the posting interface (for example, user exit) attempted to write BSEG-STCEG for a standard customer account. The VAT registration field (BSEG-STCEG) is display-only for normal customers and can only be provided for one-time (CPD) accounts. This led the system to try to write to the CPD screen (SAPLFCPD 0100) that is not called for standard customers, causing an inconsistent input and the batch input error.

Resolution

  • Move/Delete Custom Logic: Remove or deactivate any custom logic that tries to fill BSEG-STCEG for standard (non-one-time) customer accounts in the posting process.

  • Master Data Maintenance: If a VAT registration number is necessary for standard customers, ensure it is maintained within the customer or business partner master data. The system will display and, where appropriate, automatically transfer it according to standard procedures.

  • One-time Customer Processing: For one-time (CPD) customers, input the VAT registration number through the one-time account entry screen (SAPLFCPD), as this is the standard method. Passing STCEG is exclusively supported in this scenario.

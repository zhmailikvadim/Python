Symptom

In SAP Trading Partner Management (TPM) B2B scenarios, there is a requirement to identify which runtime headers are responsible for controlling and populating the functional group header fields GS02 (Application Sender Code) and GS03 (Application Receiver Code) in EDI messages.

 
 

Environment

  • SAP Integration Suite
  • Cloud Integration

Resolution

In SAP Trading Partner Management (TPM) B2B scenarios, the functional group header fields are populated using specific runtime headers:

  • GS02 (Application Sender Code) is derived from the runtime header SAP_EDI_REC_Sender_System_ID
  • GS03 (Application Receiver Code) is derived from the runtime header SAP_EDI_REC_Receiver_System_ID

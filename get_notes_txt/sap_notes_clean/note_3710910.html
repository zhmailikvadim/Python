Symptom

During SAP HANA tenant recovery using Backint-based backups, the recovery process reports the error message “Could not read backup log file … internal error”.
In SAP HANA Studio or SAP HANA Cockpit, the recovery may appear as terminated, although backend activity may still be observed. This behavior is typically observed during recovery of a production database into a sandbox or test environment and may occur in scale-out SAP HANA systems.

Environment

SAP HANA DB Version 2.0

Reproducing the Issue

  1. Restore an SAP HANA tenant database using Backint-based backups.
  2. During recovery, the error “Could not read backup log file … internal error” is reported.
  3. The SAP HANA Studio or SAP HANA Cockpit UI may indicate that the recovery is terminated, while recovery-related backend activity may still be observed. 

Cause

This error indicates that a required log backup file could not be read via the Backint interface during recovery.
In Backint-based backup environments, the underlying cause must also be verified on the third-party backup tool side.

Resolution

The SAP HANA Studio or SAP HANA Cockpit UI may not fully reflect the actual backend recovery progress in this situation.
For tenant database recovery progress, check the documented recovery progress information from the system database, for example via system view SYS_DATABASES.M_RECOVERY_PROGRESS. Recovery to the most recent state requires that the required log backups are available and readable. In Backint-based backup environments, also review backint.log and verify the third-party backup tool configuration and availability.

Keywords

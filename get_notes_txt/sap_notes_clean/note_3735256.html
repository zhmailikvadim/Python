Symptom

When users perform Mass change for cost center Fields, some fields are required to be in display mode for the approver.

Environment

  • SAP Master Data Governance
  • SAP S/4 HANA

Reproducing the Issue

  1. Go to Search cost center application.
  2. Select any cost center example and click on Mass Change. 
  3. Observe that fields are editable for the approver, instead of being display-only.

Cause

The change request status used for the approver step (for example, “Final check to be performed”) was configured with “Execution of changes” in Permitted Processing, allowing editing in that step.

Resolution

  1. Open configuration: MDGIMG -> Master Data Governance -> Classic Mode in SAP MDG -> Central Governance -> General Settings -> Process Modeling -> Change Requests -> Edit Statuses of Change Requests.
  2. Select the change request status used for the approver step (for example, “Final check to be performed”).
  3. In field “Permitted processing,” set the value to “No Processing.”
  4. Save the configuration.
  5. Retest with an approver: the approver can no longer add or delete objects or change existing objects in the change request; fields are display-only in mass processing.

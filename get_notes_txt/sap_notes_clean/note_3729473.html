Symptom

  • Separate sites exist, each linked to a single SAP account.
  • Introduction of "environments" for each site requires a 1:1 relationship between each site and an SAP account.
  • Implementation of multiple accounts for a single user is being considered using Google Workspace (GWS) email alias feature.
  • Feasibility of this approach is questioned, and alternative methods are requested.

Environment

Customer Data Cloud

Admin Console (Settings, Administration, Queries / Reports)

Reproducing the Issue

  1. Identify the need to create a 1:1 relationship between each site and an SAP account.
  2. Attempt to register multiple accounts for a single user using Google Workspace (GWS) email alias feature.

Cause

SAP Customer Data Cloud does not handle S-users; restrictions can be set at the administrator level.

Resolution

  1. Utilize the Scope feature in Permission Groups to assign a permission group to an admin user, where restrictions are set for a specific site.
  2. Access the Scope tab to view the list of sites enabled for the group.
  3. Choose to include or exclude the listed sites based on requirements.
  4. Apply changes by selecting the appropriate option in the Scope tab.

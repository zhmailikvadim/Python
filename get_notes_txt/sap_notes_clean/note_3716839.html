Symptom

You are working with field catalogs in the transportation cockpit. Each field catalog generates a UI structure to be used for the respective UIBB in the transportation cockpit, depending on the components defined as display (default or personalization) in the respective field catalog. Technically, this UI structure is based on the maximum UI structure, where all possible components/fields are available. This maximum UI structure is organized with substructures. A substructure contains components/fields for a semantic topic, such as TOR root or TOR stop data. The generation of the UI structure is generating substructures that are not needed, because no components/fields are defined as display (default or personalization).

Other Terms

FC, /SCMTMS/FIELD_CONFIG

Reason and Prerequisites

The problem is caused by a program error.

Remark: The substructures for TOR root, item, stage and stops for Loading, Unloading, departure and arrival will be generated for each field catalog valid in field catalogs for transportation order business object.

Solution

  1. Implement the attached correction instructions or import the specified Support Package
  2. Correct the generated UI structure of the affected field catalogs. To correct it, follow the steps mentioned below.

    For customer specific field catalogs:
  • Make a dummy change (set a field to 'Display (Default)' or 'Display (Personalization)') in an affected field catalog, and save the field catalog.
  • Afterwards, remove the dummy change and save the field catalog again. The save leads to a re-generation of the respective UI structure and correct it.
  • It is also possible to correct the generated UI structure described for SAP standard field catalogs below.

    For SAP standard field catalogs: 
  • Start the transportation Cockpit with a page layout where an affected SAP standard field catalog is used.
  • An abortion with the error message: "A newer version of data type '/FBI/STC*' was found than the one required," with runtime error TYPELOAD_NEW_VERSION in method CL_ABAP_TYPEDESCR->DESCRIBE_BY_NAME can occur in each affected SAP standard field catalog if the transportation cockpit is started for the first time after the specified correction instruction has been implemented. This abortion is caused by the fact that the saved generated UI structure will be checked and regenerated (if necessary) during the start of the TC, but the newly generated UI structures do not fit the (technical FBI structure) UI structures currently used. Afterward the issue will be automatically fixed and the transportation cockpit can be started again without issue for the respective field catalogs.

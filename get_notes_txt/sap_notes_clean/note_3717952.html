Symptom

What is the exact Sequential Process to Clean Up or Delete an SAP Ariba Invoicing Subaccount to Re-Run the Booster in BTP?

Resolution

Please follow the steps below strictly in the given sequence. Skipping or changing the order may cause the next Booster run to fail or unsubscription to Fail.

1. Delete the SAP Ariba Invoicing formation.
2. Delete the MDI formation.
3. Terminate the connected system from SAP Ariba Invoicing.
4. Unsubscribe the SAP Ariba Invoicing and MDI subscriptions in sequence using the Clean Up Resources Created from Failed Booster Run.
5. Delete the subaccount.

Symptom

You have maintained your own event in CRMV_EVENT. After system upgrade, you find this event is not called anymore.

For example, you have maintained a Z function module in CRMV_EVENT to update a Z table when saving a sales order. After upgrade, this table is not updated anymore.

Environment

  • SAP Customer Relationship Management 7.0 and higher
  • SAP enhancement package for SAP Customer Relationship Management
  • SAP enhancement package for SAP CRM, version for SAP HANA

Reproducing the Issue

  1. Create a sales order on WebUI.
  2. Set a breakpoint in the Z function module.
  3. Save the sales order.
  4. The Z function module is not called and the Z table is not inserted with the expected entries.

Cause

The event is stored in system table CRMC_EVENT_CALL with delivery class S ystem table, maint. only by SAP, change = modification. Find the details below.

  1. Go to SE11 and open transparent table CRMC_EVENT_CALL.
  2. Navigate to tab Delivery and Maintenance.
  3. Put cursor in field Delivery Class and press F1 on keyboard.
  4. Click on link "Reference Documentation for Delivery Class of Database Tables".
  5. Find the details for delivery class S.

In installations, updates, and language imports, all data in a transparent table with delivery class S is imported and existing data is overwritten. It means all the non-standard entries will be deleted.

Resolution

Please maintain the missing event in tcode CRMV_EVENT again.

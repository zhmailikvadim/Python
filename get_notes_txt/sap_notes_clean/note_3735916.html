Symptom

Error R1099 'Number not in interval' is raised during BP inbound webservice processing.

Environment

  • SAP S/4HANA
  • SAP S/4HANA Cloud Private Edition

Reproducing the Issue

  1. Run a BP replication from the source to the target system (for example, via DRFOUT).
  2. In the target system, review the inbound processing and observe that it fails with error R1099 "Number not in interval".

Cause

The external number from payload field <InternalID> is not within the defined external number range assigned to BP grouping, so the inbound check raises R1099.

Resolution

  • If the target system should assign a new number to the BP, change the BP grouping to use an internal number range.
  • If the inbound BP number must match the BP number from the source system, adjust the external number range in target system so that it includes that the number from payload field <InternalID>.

Keywords

R1099, interval, external number range, DRFOUT, grouping, number range, BUP_NUMBER_CHECK, set_objecttask_and_numbers

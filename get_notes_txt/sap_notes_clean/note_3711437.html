Symptom

  • User unable to access the home portal while proxying as a specific role.
  • Logs and settings are not accessible in the home portal during proxying.

Environment

  • Incentive Management

Reproducing the Issue

  1. Create a new role with specific business permissions.
  2. Attempt to proxy as the newly created role.
  3. Observe that the home portal does not display logs or settings.

Resolution

  1. Verify if the user has roles assigned.
  2. Add the user as an Internal User to enable role assignment.
  3. Assign the required roles to the user.
  4. If access to logs is required, assign the Administrator role to the user.
  5. Validate the solution by testing the access to the home portal while proxying as the specified role.

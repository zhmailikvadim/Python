Symptom

  • Rates on the job posting are sent to suppliers instead of the rates defined in the rate grid.
  • This occurs even when the supplier and site are correctly associated with the rate grid, and the expectation is for the job posting rates to be replaced during distribution.

Cause

In Rates 1.0, rate grid values only replace job posting rates when the system can apply the rate types without creating an invalid mix.

Rates 1.0 does not allow the following rate types to be mixed on a job posting or during rate replacement:

  • rate
  • pay rate based
  • bill rate based
  • pay rate with coefficient

If any Rate Category/UOM on the job posting uses one of these rate types and the corresponding rate group uses a different type, and the rate group does not include every Rate Category/UOM combination from the job posting, the system cannot apply a partial replacement.
Because these rate types cannot coexist in a single job posting in Rates 1.0, the system prevents mixing and defaults to pulling all rates from the job posting instead of the rate grid.

Factor‑type rates are not part of this restriction and do not affect this replacement logic.

This behavior is specific to Rates 1.0.

Resolution

To ensure that rate grid values replace job posting rates:

  1. Review all Rate Category and UOM combinations on your job posting template.
  2. Review the same combinations in the rate group used by your rate grid.
  3. Compare the sets to identify combinations missing from the rate group.
  4. For any missing combinations, choose one of the following:
    • Remove the missing combination from the job posting template, or
    • Add the missing combination to the rate group using a consistent rate type.
Important:
Changes to rate groups, rate types, or job posting templates do not retroactively update previously distributed job postings.
All corrections apply only to job postings created or distributed after the configuration changes are made.

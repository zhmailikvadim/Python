Symptom

You have implemented the patches from SAP note 3593356 and after applying this correction to your system while converting the User Mapping data to be encrypted with the new algorithm by clicking on the following buttons: Modify Configuration -> Start Conversion.

The conversion is stuck and there is no progress

Reproducing the Issue

Go to https://<host>:<port>/useradmin -> Configuration -> User Mapping and convert your User Mapping data , by clicking on the following buttons: Modify Configuration -> Start Conversion

Cause

In case there are no user mappings defined the user mapping conversion should not be triggered. 

Resolution

The conversion is relevant for systems where UME User Mapping is configured. If you don't use and have not configured UME User Mapping in your system, then you can don't need to start the conversion

To resolve this, set the following configuration parameter values:

ume.usermapping.encryption.algorithm=AES
ume.usermapping.unsecure=false

Then restart the system and re-check the algorithm NWA -> Certificates and Keys -> UMEKeystore

This should solve the issue caused by the empty User Mapping Data.

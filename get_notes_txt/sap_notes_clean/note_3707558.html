Symptom

After upgrading from SAP S/4HANA 2020 to SAP S/4HANA 2025, users encounter an issue when attempting to publish code modifications in the Custom Logic application.

The issue occurs while adjusting enhancements reported in SPAU_ENH, specifically affecting Cloud BAdI implementations that existed before the upgrade.

When selecting Publish in the Custom Logic app, the following error message is displayed:

  • "An internal error occurred. Report an incident for BC-SRV-APS-EXT-BL."

As part of the analysis and troubleshooting, the following steps were also performed:

  1. Use Fiori app "Register Extension for Transport", select the "Custom Logic item", and use "assign to transport request -> for full retransport" in order to capture all the needed artifacts in a new transport request.
  2. Add the affected user to the assigned Transport request

But the issue still persists.

And there are no authorization failures recorded in SU53, as referenced in SAP Note 3600195, indicating that the issue is not authorization-related. 

Environment

SAP S/4HANA On-Premise

Upgrade from SAP S/4HANA 2020 to SAP S/4HANA 2025

Custom Logic application 

Reproducing the Issue

  1. Open the Custom Logic application.
  2. Modify any of the existing enhancements.
  3. Click on Publish.
  4. Observe the error message: "An internal error occurred. Report an incident for BC-SRV-APS-EXT-BL."

Cause

During the upgrade, migration logic was introduced in the Custom Logic application, which changes the structure of key user classes.

Some existing draft custom logic implementations could not be migrated correctly due to coding and syntax inconsistencies after the upgrade.

As a result:

  • The affected custom logic implementations remain in an inconsistent state.
  • Publishing these implementations leads to a runtime internal error.
  • The error is not authorization-related and does not produce SU53 entries

In short, there is an inconsistency in the BADI implementation due to syntax errors in the custom logic implementations after the system upgrade.

Resolution

To fix the issue:

  1. Implement SAP Note 3706271 - Custom Logic - Implementation with syntax errors cannot be corrected.
  2. After implementing the note, modify the code of the affected custom logic implementations

Once these steps are completed, the custom logic implementations can be published and edited successfully.

Symptom

Error message: "DATE_EXCEEDS_RESTRICTION, errorMessage: Invalid date: last_modified_on field value 'yyyy-MM-dd' exceeds restriction of 3 months into the past. The earliest date you can use is 'yyyy-MM-dd'." in standard iFlow "Replicate Employee from SAP SuccessFactors Employee Central". 

Environment

CPI tenant, Cloud for Customer, SuccessFactors Employee Central 

Package: SAP Cloud for Customer Integration with SAP SuccessFactors Employee Central 

iFlow name: Replicate Employee from SAP SuccessFactors Employee Central 

Reproducing the Issue

  1. Check message monitor in CPI;
  2. You can see failed message for standard iFlow "Replicate Employee from SAP SuccessFactors Employee Central" with error: "DATE_EXCEEDS_RESTRICTION, errorMessage: Invalid date: last_modified_on field value 'yyyy-MM-dd' exceeds restriction of 3 months into the past. The earliest date you can use is 'yyyy-MM-dd'."

Cause

As per SuccessFactors Employee Central side, last_modified_on should no longer than 3 months in the past.

In this iFlow, 'last_modified_on' field is controlled via variable GV_LAST_MODIFIED_DATE.

The value of this variable can be checked under Monitor -> Manage Stores -> Variables in CPI.

Resolution

As per SuccessFactors Employee Central side, the only possible solution is to change the date used in the fields last_modified_on so that the date is not longer than 3 months in the past.

Per standard design from iFlow side, if the iFlow get executed successfully, the value would be rewrite as current data.

But in the case the last successful execution date already past over 3 month, and then not been successful executed after that. Until after 3 month, it will never get executed successful due to this error. 

In this case, you may need to re-trigger the initial load. This can be done via deleting the GV_LAST_MODIFIED_DATE entry from the screen attached before, which will automatically make the iFlow run in initial load mode in the next pulling cycle i.e. all employees (from Transmission_Start_Date) will be pulled from EC again and replicated to C4C.

For more details, please refer to this blog article: <https://community.sap.com/t5/crm-and-cx-blog-posts-by-sap/employee-central-integration-with-cloud-for-customer-via-hci-part-ii/ba-p/13278629>.

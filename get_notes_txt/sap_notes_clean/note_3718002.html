Symptom

After updating the index price in FDCS17, the floating leg price of Commodity Swap is not automatically updated in FTR_EDIT when the deal is opened in display mode. 

Environment

SAP S/4HANA

Reproducing the Issue

  1. Display price for floating leg of Commodity Swap (FTR_EDIT)

  2. Maintain Today’s Price (FDCS17)

 3. Check the price for floating leg of Commodity Swap in display mode (FTR_EDIT) - PRICE NOT UPDATE

 4. Now open the transaction in Change Mode and hit the Enter button (FTR_EDIT) - PRICE UPDATE

Cause

Updating the index price in FDCS17 does not automatically trigger:

  • Flow rebuild
  • Recalculation of floating leg
  • Market data refresh

The recalculation logic is triggered only during:

  • Change mode processing
  • ENTER event
  • SAVE event

This is due to standard system design:
Display mode does not execute recalculation logic or rebuild flows.

Resolution

This is expected standard behavior.

Symptom

During a tenant recovery or system refresh, the following SQL command fails when executed on SYSTEMDB:

ALTER DATABASE <tenant_name> ADD 'scriptserver';

with the error:

general error: nameserver failed to process the request;
No changes to database in recovery

The tenant remains in recovery state and required tenant services cannot be added or modified.

Environment

SAP HANA DB Version 2.0

Cause

The tenant database is already in recovery state.

This can occur when attempting to apply a documented resolution (for example, adding ScriptServer) after the recovery process has already been started.

Resolution

Tenant services must be configured before starting the tenant recovery.

For restore/refresh target scenarios, use one of the following approaches:

  • Preferred (clean approach):
    Drop and recreate the target tenant, enable the required services (for example ScriptServer), and then start the tenant recovery.
  • Alternative:
    Ensure the tenant is fully out of recovery state before attempting any service or topology changes.

After the tenant is in a clean state and the required services are configured, re-run the tenant recovery using the same backup.

SAP Help documentation for tenant lifecycle and service configuration:

  • CREATE DATABASE / DROP DATABASE (Tenant Management)
  • Enable ScriptServer for a tenant

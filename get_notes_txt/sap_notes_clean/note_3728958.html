Symptom

  • When loading a story, there is a warning message "Your story includes scripts. Re-saving it may enable the Lite Viewer and improve loading performance " 
  • Why does this warning message occurs ? 

Environment

  • SAP Analytics Cloud 2026.2.3  

Resolution

  • When do you receive messages and why?
    We give a notification the moment that we see a script in the story but the story doesn't have a scripting metadata.
  • Why can this issue be resolved only by resaving the story?
    1. Scripting metadata is added to story definition when we re-save the story.
    2. if metadata satisfies the list of supported items in LiteViewer, it might open the story in Liteviewer in future and it would improve performance.

This warning message is displayed to the user who has Edit rights.
To support "Lite Viewer", it requires that a resave of the story is done to pass the benefits of performance to the end users. This resave updates the definition of the story in way to get information on lite viewer compatibility.

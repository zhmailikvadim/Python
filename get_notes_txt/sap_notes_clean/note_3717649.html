Symptom

When trying to create an entry for leading Ledger 0L in customizing transaction GCVP "Maintain Plan versions", an error message No GG850 is raised:

You cannot create an entry for ledger 0L
Message no. GG850

Diagnosis
You want to create an entry for ledger 0L.  You cannot do so because either:

Environment

  • Financial Accounting
  • SAP ERP Central Component
  • SAP ERP
  • SAP S/4HANA Finance
  • SAP S/4HANA
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA

Reproducing the Issue

  1. Execute transaction code GCVP or SPRO >> Financial Accounting >> Special Purpose ledger >> Planning >> Plan settings >> Maintain Plan versions.
  2. New Entries
  3. Ledger 0L

Cause

Leading ledger 0L belongs to a different application than the one being currently maintained, or ledger 0L is a dependent ledger (for example, an average balance ledger). Error GG850 is raised correctly in this case as ledger 0L belongs to a different application.
It's not possible to add leading ledger 0L into customizing from GCVP (SPRO >> Financial Accounting >> Special Purpose ledger >> Planning >> Plan settings >> Maintain Plan versions).

Resolution

In order to  maintain the Plan Version for Leading ledger 0L, use transaction code GLPV instead.

Symptom

Applications running on the WebCUIF use certain OData Services to support the features

  • Process Flow
  • Core Data Service (CDS) - based value helps
  • Attachments
  • Output Control
  • Situations Handling

You are running an application in Service Management (e.g. Manage Service Orders) and you expect the above features to work properly. However, you might observe a blank assignment block or the feature is not available or the related function is without an effect.

Other Terms

SICF, /IWFND/MAINT_SERVICE, PFCG

Reason and Prerequisites

One or more of the following reasons might be the cause for the observed symptom

  • The relevant HTTP Service (SICF) of the feature is not activated
  • There is no Backend Service for the UI5 OData Service defined in the Gateway Client
  • The corresponding ICF Node for the UI5 OData Service is not activated in the Gateway Client
  • A System Alias is not defined for the UI5 OData Service in the Gateway Client
  • Your backend user is lacking authorizations which are necessary to run the feature properly

Solution

Activation of HTTP Services and Definition of Backend Services for OData Services

There are two options to activate the HTTP Services and define the Backend Services for the OData Services

First Option: Activate the HTTP Services and define and activate OData Services by Task List

Follow the steps that are described in SAP Help Portal in page https://help.sap.com/docs/SAP_S4HANA_ON-PREMISE/22bbe89ef68b4d0e98d05f0d56a7f6c8/92ed0a55f1983d6ae10000000a44176d.html

Second Option: Activate the HTTP Services and define and activate OData Services manually in transactions SICF and "/IWFND/MAINT_SERVICE"

Activate HTTP Services in transaction SICF:

  • Process Flow and Core Data Service (CDS) - based value helps: CRMS4_SERV_ORDER_CONF_MANAGE_SRV
  • Attachments: CV_ATTACHMENT_SRV
  • Output Control: CA_OC_OUTPUT_REQUEST_SRV
  • Situations Handling: C_SITNINSTANCETP_CDS

For each HTTP Service perform the following steps:

  • Start transaction "SICF" (Define Services)
  • Enter the name of HTTP Service in field "Service Name"
  • Hit F8
  • In the menu select "Service/Host-->Activate"

Define Backend Services for the OData Services

For each Backend Service perform the following steps: 

  • Start transaction "/IWFND/MAINT_SERVICE" (Activate and Maintain Services)
  • Click button "Add Service"
  • In field "System Alias" select the right value from the Input help, i.e. select the system where the OData Service will be called
  • Enter the name of the Backend Service in field "Technical Service Name"
  • Click button "Get Services"
  • In the result list, select the Backend Service and click button "Add Selected Services"
  • In the popup screen "Add Service" provide the necessary details and click button "Enter"
  • An information message is shown that the Service was created and its metadata was loaded successfully
  • Click button "Enter"
  • The Backend is removed from the Result List
  • Navigate back to the Service Catalog Overview Screen and select the just created Service
  • In section "ICF nodes" click button "ICF node-->Activate"
  • Select a package and hit enter
  • As a result the status shows a green traffic light indicating that the ICF node has been activated
  • In section "System Aliases" check that the settings are correct

Assign the relevant OData Services to one of the PFCG roles that are assigned to your backend user

  • Start transaction "PFCG" (Role Maintenance)
  • In field "Role" enter a PFCG role that is assigned to the users that are using applications using the above mentioned features
  • Click button "Change"
  • In tab "Menu" in section "Hierarchy" select node "Role Menu"
  • On button "Insert Node" click the Dropdown Menu and select "Authorization Default"
  • In the popup window in field "Authorization Default" select the value "SAP Gateway Business Suite Enablement - Service"
  • Enter the following TADIR Services into the table view:
    • Process Flow and Core Data Service (CDS) - based value helps: "CRMS4_SERV_ORDER_CONF_MANAGE_SRV   0001"
    • Attachments: "CV_ATTACHMENT_SRV                  0001"
    • Output Control: "CA_OC_OUTPUT_REQUEST_SRV           0001"
    • Situations Handling: "C_SITNINSTANCETP_CDS               0001"
  • Click button "Apply"
  • Navigate to tab "Authorizations"
  • Click on button "Change Authorization Data" or on button "Expert Mode for Profile Generation" and provide the authorization values for the newly added authorization objects
  • Click button "Generate" to generate the Authorization Profile

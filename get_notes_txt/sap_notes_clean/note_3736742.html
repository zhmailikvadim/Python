Symptom

After system upgrade, ASSERTION_FAILED dump occurs when calling method APPSERVER_FILE_WRITE(CL_RSAN_UT_APPSERV_FILE_WRITER), e.g via a custom solution, like a custom transaction or custom program.

Environment

  • SAP NetWeaver
  • SAP ERP

Cause

This occurs because of the change introduced via SAP Note 3628183: the mentioned method in the class was removed as it is not used anymore in the system. Implementing the correction alone or upgrading to a Support Package containing the solution will remove this method.
As the method is not used anymore in the system, however, this change has no impact on any SAP-standard solution and generally should not result in any observable difference(like a system error).

It is presumable that related error is observed still because it was used in a custom solution, like a custom program or custom transaction. For this, it is important to note that this method is not an External API; it is an internal code with no intent to be used or consumed for external purposes.

Resolution

Adjust the custom solution on a way where this method is not used. Building in this method into custom coding is not supported and is not intended for such purpose.

For alternatives, consult with consultant or local functional expert to find an alternative solution that can suffice the business requirement.

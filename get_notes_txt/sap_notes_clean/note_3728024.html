Symptom

In T-code MD04, you expect system to display Release order for a stock transfer order (Ord.DS) in supplying plant basing on Shipping notification (ShpgNt) in receiving plant.

Environment

  • SAP ERP Production Planning (PP) Release Independent
  • SAP ERP Release Independent
  • SAP ERP Central Component Release Independent
  • SAP Enhancement package for SAP ERP Release Independent
  • SAP S/4HANA, on-premise Release Independent
  • SAP Fiori for SAP ERP Release Independent
  • SAP Fiori for SAP S/4HANA Release Independent
  • SAP S/4HANA Cloud Private Edition Release Independent

Reproducing the Issue

  1. Access transaction ME21N and register a stock transport order with the delivery date set to 2026-04-01.
  2. Access transaction ME22N and navigate to the confirmation tab. Enter confirmation category AB and input the confirmation delivery date as 2026-04-06.
  3. Access transaction MD04 to verify the displayed dates for Ord.Ds and ShpgNt.
  • For the receiving plant, the ShpgNt is displayed as confirmation delivery date = 2026-04-06.
  • For the supplying plant, the Ord.Ds remains unchanged and is displayed as PO delivery date = 2026-04-01.

Cause

This is standard behavior.

Resolution

In t-code MD04, Ord.Ds date is decided by stock transfer order(STPO)'s delivery date instead of the confirmation delivery date of ShpgNt.
As per the hard-coding design, t-code MD04 reads Ord.Ds date from table EKET-EINDT, while ShipNt date is from EKES-EINDT.
Hence, it is not supported to display Ord.Ds date basing on ShipNt confirmation date.

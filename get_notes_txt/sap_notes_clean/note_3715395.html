Symptom

You run the Calculo nómina y vacaciones  ( CO ) (HCOCALV0) report for an employee in the following scenario:

  • The employee has a valid entry in the Embargo ( CO ) (IT0835) infotype for the payroll period.
  • The payroll is executed as an off-cycle payroll.

In the scenario described above, the system calculates the worked days proportion even for off-cycle reasons that should not be considered for proportionalization.

Example

  • An employee is processed in a liquidation special payroll.

  • The employee has a valid entry in the Embargo ( CO ) (IT0835) infotype for the payroll period.

  • An off-cycle payroll is executed with reason Pago por adelantado períodos: Originales (0095).

In the scenario described above, the system calculates the worked days proportion differently than expected.

Reason and Prerequisites

Reason

This issue happens because the system does not correctly restrict the proportional worked days calculation to the defined off-cycle scenarios.

Prerequisites

The following Support Package levels are the minimum prerequisites:

S4HCM 102 HR SP 00
S4HCM 101 HR SP 05
S4HCM 100 HR SP 18
SAP_HR 6.08 HR SP C9

For more information, access the SAP ERP Human Capital Management - Support Package Schedules.

When you install an SAP Note, some prerequisites might be required. The prerequisites are SAP Notes that have already been released and that change the same context of the current one. Then, even if these prerequisites are not necessarily related to the symptom of this SAP Note, you must implement them due to technical reasons.

Solution

From now on, the system applies the worked days proportional calculation only for the defined scenarios.

This SAP Note delivers a new feature, Decisión de off-cycles soportadas en liquidación especial (38OLI), which enables you to define the off-cycle reasons for which the worked days proportion must be applied.

Example

38OLI Decisión de off-cycles soportadas en liquidación especial

+-| OCRSN Reason for Off-cycle Payroll

+- | 0010

'X'

+- | 0095

'X'

+- | PRMA

'X'

+- | otherwise

' '

In this example, the worked days proportion is applied only for off-cycle reasons 0010, 0095, and PRMA. For all other off-cycle reasons, the proportional calculation is not executed.


If the "otherwise" branch is configured with value 'X', the proportional calculation is applied to all remaining off-cycle reasons.

Symptom

  • Need information on the strategies used for Retrofit and Conflict Management in Cloud ALM
  • Want to utilize Retrofit in Cloud ALM and setting up a N+1 landscape
  • Trying to understand how to manage the conflicts between the maintenance and implementation tracks where the same objects are modified
  • Is Cross-System Object Lock (CSOL) supported in SAP Cloud ALM?

Environment

  • SAP Cloud ALM
  • SAP S/4HANA PRIV. CL. ED.

Cause

The handling of conflicting objects in Cloud ALM's Retrofit feature is different than in Solution Manager.  Conflicts are checked, but it's not using the Solution Manager's CSOL function.

Resolution

At the release time of a Retrofit source system's transport(s), the objects in the TR are categorized against any TRs in the target system:

  • which are modifiable
  • any TR that was released in the last X timeframe, default timeframe being 6 months (where X is based on the setting found in se16 > /SDF/CMD_TR_CONF > Number of days in analysis period)

If changes are found for the same object, the object is categorized as manual.  If there is no change to the involved object, it will be categorized as Automatic and the object is added to a ToC (transport of copies) which is used in Retrofit to move the changes from the Retrofit source system to the Retrofit target system.

Any object categorized as Manual must be moved to the Retrofit target system manually, to avoid downgrade.

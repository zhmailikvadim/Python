Symptom

When loading SwiftRef directory it might happen that the decrypted/verified message payload did not yet reach the database when the SwiftRef-Load job is already trying to read it. Due to that the SwiftRef job is using encrypted data which then fails.

This note fixes this issue to make sure that the job waits for the unencrypted data to be committed. Once the data becomes available it proceeds with processing.

Solution

 Install the relevant support package or implement this SAP note directly.

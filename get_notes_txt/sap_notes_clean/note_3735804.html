Symptom

  • Using @OData.hierarchy.recursiveHierarchy on a projection view directly on a root table entity causes the service definition/binding to fail.
  • The same model works when an additional view layer is introduced between the table entity and the root consumption view.
  • Parser message: "TABLE ENTITY is only supported for basic parsing. Turn off the following bitmask switches: semantic_check_on, semantic_local_check, adt_pretty_printing, resolve_view_extends, new_cds_source."
  • Runtime/activation trace may show: "LCX_QLAST_PARSER_FAILED CL_SADL_CONS_VIEW_PARSER======CCIMP:105 Internal exception raised (... TABLE ENTITY is only supported for basic parsing ...)".

Environment

SAP BTP ABAP environment 2511

Reproducing the Issue

  1. Define a root table entity using DEFINE TABLE ENTITY as the BO implementation layer.
  2. Define a root view entity as a projection directly on that table entity and annotate it with @OData.hierarchy.recursiveHierarchy:[{ entity.name }].
  3. Expose the view via a service definition and create/activate a service binding (for example, OData V2).
  4. Start the service check or activate the binding and observe the technical error messages noted in the Symptom section.

Cause

The table entity on the BO implementation layer is not the source of the hierarchy used by @OData.hierarchy.recursiveHierarchy. The hierarchy source is not directly or indirectly the source of the annotated view, which leads to service compilation errors.

Resolution

  1. Ensure that the source of the hierarchy specified in @OData.hierarchy.recursiveHierarchy is directly or indirectly the source of the annotated view.
  2. If the BO implementation uses a table entity, make that table entity the source of the hierarchy view.
  3. When necessary, introduce a layered design (e.g. table entity -> hierarchy view -> root projection/consumption view) so the hierarchy source is in the view chain.
  4. Regenerate and activate the service definition and service binding.
  5. Test the service again to confirm successful activation and hierarchy behavior.

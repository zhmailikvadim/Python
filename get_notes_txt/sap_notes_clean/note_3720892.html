Symptom

Some distribution rule groups that were used to post settlement documents and that can be seen in the WBS element master data (t-codes CJ03/CJ20N) are missing in CJID.

Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.

Environment

  • SAP ERP
  • SAP ERP Central Component
  • SAP Enhancement package for SAP ERP
  • SAP Enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA
  • SAP S/4HANA Finance
  • SAP S/4HANA Cloud Private Edition

Cause

Only one distribution rule group can be displayed for each combination of controlling document and posting row. This means that the distribution rule group of the last settlement (for example, "settlement completely") is stored and displayed in transaction code CJID.

Let's take the following example: distribution rule groups 1 and 2 are available under WBS element AAA. However, only distribution rule group 2 is displayed in CJID. In COEPD table, the following data is stored:

BELNR     BUZEI   BUREG ABWST

111111    001     001   06    
111111    001     002   09

As seen above, the distribution rule group 1 (field BUREG) was used during line item settlement of posting row 001 of document 111111, but only for the partial settlement (ABWST = 06). This line item was settled completely (ABWST = 09) by distribution rule group 2, which is why this group can be found in transaction CJID.

Resolution

This is the standard system design and cannot be changed.

Environment

SAP S/4HANA

SAP BASIS 758 SP03 

Cause

Application program calls method SALV_FUNCTIONAL_SETTINGS->SORT_ON_HEADER_CLICK( ) which means SALV uses the new (2024) LVC adapter instead of the standard SLIS adapter. After a user command, the LVC adapter mistakenly calls the REUSE function a second time. When the user exits the display, control returns to the first function call, but the grid has been freed resulting in an abend.

The mistake arises because:

  1. LVC checks the LV_NEW_DATA_REQUESTED variable instead of PROCESSING_TYPE as in SLIS
  2. LV_NEW_DATA_REQUESTED variable is set according to changes of type APPLICATION_DATA
  3. Change Type APPLICATION_DATA is set by the SALV API method DISPLAY( )
  4. Application calls DISPLAY( ) to refresh the output, rather than REFRESH( ).

Resolution

Implement SAP Note - 3627824 - ALV: SALV Grid is initialized on refresh

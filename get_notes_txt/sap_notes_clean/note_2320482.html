Symptom

  • ASE returns error 10901, a shortage of 'number of aux scan descriptors', to the client:

    Msg 10901, Level 17, State 1:
    Server 'ASE', Line 1:
    This query requires 8 auxiliary scan descriptors, but currently there are only 1 auxiliary scan descriptors available. Either raise the value of the 'number of aux scan descriptors' configuration parameter or try your query later. 

    • Example of JConnect application error:

      com.sybase.jdbc4.jdbc.SybSQLException: This query requires 8 auxiliary scan descriptors, but currently there are only 1 auxiliary scan descriptors available.
      Either raise the value of the 'number of aux scan descriptors' configuration parameter or try your query later.

  • ASE reports error 603 in the errorlog:

    Error: 603, Severity: 17, State: 1
    There are not enough system session descriptors available to run this query. Raise the value of the configuration parameter 'number of aux scan descriptors' or retry your query later. You will need to increase the 'number of aux scan descriptors' to resolve this issue.

  • How to determine what query is using up the 'number of aux scan descriptors' ?

Environment

SAP Adaptive Server Enterprise (ASE). All versions.

Cause

  • These messages means that ASE has run out of auxiliary scan descriptors.
  • Refer to these ASE product manual pages for more information and troubleshooting steps

    • Troubleshooting: Error Messages / Access Method Errors / Error 603
    • Reference Manual: Configuration Parameters / number of aux scan descriptors
      • Monitoring and Estimating Scan Descriptor Usage

Resolution

  • There is no singular or straightforward way to calculate a serverwide value for this parameter.
  • Nor is there a query or diagnostic that can be run to show how many aux descriptors are used by a query or which query is using a large amount of them
  • The most important aspect is understanding which queries have
    • a larger number of tables in the 'from' clause and
    • have views built out of a larger number of tables and
    • whether these queries get executed concurrently and if so, by how many connections at a time

  • The general advice is to increase the parameter when this error is seen and to double the current value:

    sp_configure 'number of aux scan descriptors' , <new_value>

  • The overall number of aux descriptors in use can be monitored by running

    sp_monitorconfig 'number of aux scan descriptors'

Symptom

  • Installing HADR system is successful on the Primary.
  • Installing on the Companion fails with error: The DR Agent does not have write access to the file '/sybase/<SID>/DM/interfaces'         

ConfigureRemoteRSInstanceTask Name                     Modify Interfaces File                                                           
ConfigureRemoteRSInstanceTask State                    Error                                                                            
ConfigureRemoteRSInstanceShort Description             Modify Interfaces File                                                           
ConfigureRemoteRSInstanceLong Description              Checking file '/sybase/SID/<SID>/interfaces'                                        
ConfigureRemoteRSInstanceFailing Command               Checking file '/sybase/SID/<SID>/interfaces'                                        
ConfigureRemoteRSInstanceFailing Command Error MessageThe DR Agent does not have write access to the file '/sybase/<SID>/DM/interfaces'.

Environment

SAP Adaptive Server Enterprise (ASE) 16.0 SP04 PL04 with High Availability Disaster Recovery (HADR) for Business Suite

Cause

Wrong permission set on $SYBASE/DM/DM/interfaces file 

Resolution

  1. Set write access permission to $SYBASE/DM/interfaces file:

    cd $SYBASE/DM
    chmod 664 interfaces


  2. Re-run setuphadr <response_file>

Keywords

setuphadr, permission, HADR

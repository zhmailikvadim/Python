Symptom

  • External batch provided for the finished product is successfully integrated when goods receipt is posted manually via MIGO (serialization type=3 in table MCH1).
  • External batch provided for the finished product is not integrated when goods receipt is posted via MBGMCR IDOC (serialization type=0 in table MCH1).

Environment

SAP Advanced Track and Trace for Pharmaceuticals

Reproducing the Issue

  1. Post goods receipt manually via MIGO for a finished product with an external batch.
  2. Verify the serialization type in table MCH1 (expected value: 3).
  3. Post goods receipt using MBGMCR IDOC for a finished product with an external batch.
  4. Verify the serialization type in table MCH1 (expected value: 0).

Cause

Serialization fields are updated when the batch is created via function module VB_CREATE_BATCH. This function module is not utilized when batches are created via MBGMCR IDOC.

Resolution

  1. Use the correction report /STTPEC/BATCH_MCH1_CORR to identify discrepancies in the cross-plant batch management entries (MCH1).
  2. Update the additional fields with the latest information from materials and plants using the correction report.

Keywords

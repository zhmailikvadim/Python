Symptom

An event has been configured in Job & Automation Monitoring and it is expected that an alert/event action will be generated based on the Job name, but there is no alert even though the job instance has a red rating and the alert is activated

Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental

Environment

SAP Cloud ALM

Reproducing the Issue

  1. Log into Cloud ALM

  2. Click on Operations

  3. Click on Job & Automation Monitoring

  4. When clicking into the UI, the job status is detected as Red Status

Cause

A job group has been configured due to a job name that is unique each time it runs. The data provider found that the name of the job / automation is constantly changing. Therefore grouping the executions under one name is not possible. 

Resolution

Change the Job Group Name

  1. When clicking on the Monitoring menu, you can see there is a job group defined when there is an edit / pencil icon beside the job name, which by default starts with: JOBS_EXECUTING




  2. The Job Executions can be seen with issues detected, in this example based on excessive runtime - and no alert is generated




  3. It is possible to adjust the Job / Automation name by clicking on the Job Group and select the new name and click "Save"

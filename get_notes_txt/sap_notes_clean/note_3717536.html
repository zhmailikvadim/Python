Symptom

  • Error message: "No RFC authorization for user XYZ" when carrying out various steps in Portfolio Management after an upgrade or support package implementation.
  • Newly introduced authorization checks in RFC function modules in Portfolio Management area.

Environment

  • SAP S/4HANA
  • SAP S/4HANA Cloud Private Edition 

Reproducing the Issue

  1. Carry out some navigation or functional steps in Portfolio Management after a product upgrade or support package implementation.
  2. Encounter the error message: "No RFC authorization for user XYZ".

Cause

The issue arises due to newly introduced authorization checks in RFC function modules in Portfolio Management area, which have been implemented via support package in all SAP S/4HANA releases including and above S4CORE 102.

This is the list of affected function modules:

  • /RPM/ACL_DISPLAY
  • /RPM/ACL_MODIFY
  • /RPM/AUTHORIZATION_PRE_CHECK
  • /RPM/COMMENT_GETLIST
  • /RPM/COMMENTS_DISPLAY
  • /RPM/COMMENTS_MODIFY
  • /RPM/CONTEXT_GETLIST
  • /RPM/GET_ALL_CUSTOM_FIELDS
  • /RPM/GET_CACHE_CONFIGURATION
  • /RPM/GET_CUSTOMIZING_TEXTS
  • /RPM/GET_DERIVED_AUTHORIZATION
  • /RPM/GET_NAVIGATION_INFO
  • /RPM/GET_OBJ_CUSTOMIZING_TEXTS
  • /RPM/GET_USER_PORTFOLIO
  • /RPM/GET_USER_ROLE_DETAILS
  • /RPM/GET_USER_SETTINGS
  • /RPM/MSPC_DOWNLOAD
  • /RPM/MSPC_UPLOAD
  • /RPM/MSPS_DOWNLOAD
  • /RPM/MSPS_GET_COOKIE
  • /RPM/MSPS_UPLOAD
  • /RPM/SET_USER_PORTFOLIO
  • /RPM/SET_USER_SETTINGS
  • /RPM/SUBSTITUTE_DISPLAY
  • /RPM/SUBSTITUTE_MODIFY

Resolution

Users must be provided authorization for authorization object S_RFC and the affected function modules or corresponding function groups.

  • See above for the list of affected function modules.
  • The relevant function groups are /RPM/CONFIG and /RPM/INTEGRATIONMSP.

Keywords

RFC authorization checks, /RPM/CONFIG, /RPM/GET_VALUE_HELP, 3590114, authorization roles, function modules

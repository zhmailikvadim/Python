Symptom

  • When creating a new Monitoring Watch in 4.2 SP06+, a Boolean Expression containing a parenthesis (bracket) can not be used. If a bracket is used, and the Boolean expression is refreshed, the Formula is deleted / disappears, and the message 'Please Enter a Formula' appears.

    eg:
    The following boolean expression works:
    <HOST>.CentralManagementServer$'Auditing Thread Last Polling Cycle Duration _seconds_'>1800 || <HOST>.CentralManagementServer$'Average Query Response Time Since Startup _msec_'>750 && <HOST>.CentralManagementServer$'Average Commit Response Time Since Startup _msec_'>750 || <HOST>.CentralManagementServer$'Number of Sessions Established by All Users'>400 || <HOST>.CentralManagementServer$'Current Number of Auditing Events in the Queue'>500

    The following Boolean expression does not work:
    <HOST>.CentralManagementServer$'Auditing Thread Last Polling Cycle Duration _seconds_'>1800 || (<HOST>.CentralManagementServer$'Average Query Response Time Since Startup _msec_'>750 && <HOST>.CentralManagementServer$'Average Commit Response Time Since Startup _msec_'>750) || <HOST>.CentralManagementServer$'Number of Sessions Established by All Users'>400 || <HOST>.CentralManagementServer$'Current Number of Auditing Events in the Queue'>500
  • When viewing a monitoring watch in Central Management Console (CMC) -> Monitoring -> Watchlist -> <WATCH> -> <Metric> -> switch from 'Live' to 'History' -> Choose any date, the following error message appears:
    Start date must be greater than end date
    This error occurs no matter what date is selected.

  • When viewing a monitoring watch in Central Management Console (CMC) -> Monitoring -> Watchlist -> <WATCH> -> <Metric> -> switch from 'Live' to 'History' -> Choose a date earlier than the end date, the following error message appears:
    Date cannot be greater than today's date
    An earlier date was selected, but the date was switched from DD.MM.YYYY to MM.DD.YYYY after selecting, eg selecting 12th February 2019 (12.02.2019), results in the date becoming 02.12.2019 (2nd December 2019).

  • In Central Management Console (CMC) -> Monitoring -> Watchlist clicking on "+" (Create New Watch), doesn't navigate to Create watch wizard in Japanese and Chinese languages.

Reason and Prerequisites

Environment

  • SAP BusinessObjects Business Intelligence Platform 4.2 SP06+
  • Monitoring
  • Windows
  • Linux / Unix 

Steps to Reproduce

  1. Login to Central Management Console (CMC)
  2. Navigate to Monitoring -> Watchlist
  3. Select to CentralManagementServerWatch (Example)
  4. While viewing graph data of any metrics of a watch switching to "History" of graph data and trying to select "Start date" and "End date" doesn't take the values selected from Date picker appropriately.
  5. Click on Edit
  6. Click on Step2
  7. Click on refresh icon next “Boolean Expression”
  8. Enter a Boolean expression with parenthesis then refresh. It disappears.
  9. Trying to create a new watch from Monitoring -> Watchlist -> "+" (Create New Watch) doesn't navigate to Create watch wizard in Japanese and Chinese languages.

Solution

This issue is or will be fixed in the patches listed in the "Support Packages & Patches" section below.

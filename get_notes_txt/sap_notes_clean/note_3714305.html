Symptom

  • When calculating the charges for Freight Orders, the calculated percentage amount for percentage-based charge lines is incorrect when having 3-decimal configuration in TCURX:
    • For example, the rate amount is displayed as 1,100%, whereas it should be calculated as 1.1% of the Base Freight
    • The system calculates 0.11 (11%) instead of 0.011 (1.1%)
    • When the rate is changed to 1%, it is automatically converted to 1,000, and the calculation remains incorrect.

Environment

SAP S/4HANA 2020 and above releases.

Cause

  • Percentages are maintained with 3 decimal places, which causes SAP TM to internally scale the percentage value before applying it to the base freight.
  • Because of this internal scaling and aggregation logic, the calculation does not appear as a simple "1% of [e.g. Base Charge]" when viewed technically.
  • If the percentage were maintained with 2 decimal places instead of 3, the calculation would align more closely with the expected manual percentage logic.
  • Changing the percentage decimal configuration (TCURX) is a system-wide setting and should be done with caution, as it may impact other calculations across the system.
  • The observed behavior is therefore configuration-driven and not a system defect. 

Resolution

  • The use the SAP-provided BADI at the process_calculation_sheet level may be needed (method /SCMTMS/CL_TCC_ENGINE ~> PROCESS_CALCULATION_SHEETS_RF)
  • In this implementation, there is a need to loop over all charge lines, identify the percentage-based charge lines, and then divide the amount by 10 to adjust the scaling factor accordingly.
  • This approach will ensure that the percentage charge is calculated correctly, while still maintaining the 3-decimal configuration in TCURX.
  • Since this is related to a custom enhancement, we recommend reaching out to the in-house technical team to further help as we only solve the bugs in the SAP standard code.

Symptom

When I attempt to use the SOW Submit Upload connector to transition a Statement of Work (SOW) from Draft to a submitted status, the upload fails with the following error:

Error:

Record 1: Please select at least one SOW Worker Role/ Plant/ UOM combination.

Cause

The SOW Submit Upload connector is strictly designed to update the status of an SOW (e.g., to Pending Approval, Approval Paused, or Submitted). It is not designed to add or modify SOW data such as worker roles, units of measure, or plant codes.

This error is triggered when the uploaded file contains header names or data for SOW Worker Role, Unit Of Measure, or Plant Code, which the connector cannot process.

Resolution

To resolve this issue, you must use the appropriate connectors for adding data before submitting the SOW, or remove the unsupported columns from your submit file.

To add worker roles to an existing SOW via upload:

  • Use the SOW Worker Role Detail Upload connector instead of the submit connector. For detailed instructions, refer to the SOW Worker Role Detail Upload Connector Documentation.

To successfully use the SOW Submit Upload:

  1. Ensure your upload file only contains the fields defined in the technical specification for the SOW Submit Upload.

  2. Remove any columns related to SOW Worker Role, UOM, or Plant.
  3. Re-run the upload to update the SOW status.

Symptom

SAP Note 3490451 addresses the issue of duplicate reversals that may occur during allocation processes. The user is aware of the potential risks and still wish to enable asynchronous reversal to enhance performance.

This note introduces a configurable parameter that allows customers to switch to asynchronous reversal mode at their discretion. By adjusting the parameter ALLOC_ASYNC_REV, customers have the flexibility to choose between prioritizing performance and accepting potential risks, or maintaining a safer but potentially less optimal allocation process.

Other Terms

k_allocations_reverse, 3490451, allocations, KSU5, KSV5, faglga35, faglga15, commit work and wait,

Reason and Prerequisites

SAP Note 3490451 introduced synchronous mode in order to prevent duplicate reversals.

Solution

Implement the following correction instructions or relevant support package.

After the note is implemented, please check transaction FIN_FLAGS for the following: Application = CO, Sub. application = Allocation, Parameter = ALLOC_ASYNC_REV.

In case the above parameter is not available, add the Application, Sub-Application and Parameter manually. For transactions FIN_FLAGS_CUST1, FIN_FLAGS_CUST2 you can follow the same instructions as in SAP note 3361820.

For transaction FIN_FLAGS_CUST3, please add the following:

Application Subapplication Parameters Data element  SAP Note Number Chk Values Medium Field Label Long Field Label Description
CO Allocation ALLOC_ASYNC_REV FIN_FLAGS_ON_OFF 3727719 X Reversal switch Switch for async reversal Switch for asynchronous reversal.

By setting the parameter value to 'X', processing will revert to asynchronous reversal, mimicking the behavior prior to the introduction of SAP Note 3490451. If the parameter is left initial, processing will default to synchronous mode.

Symptom

BOP determines a worse alternative for ABC wrt. the delivery date in case the original requested delivery date is in the past and a substitute has an earlier rescheduled delivery date than the original request.

Other Terms

aATP; ABC; BOP; Delivery Date; Past

Reason and Prerequisites

The reason is that during BOP, the rescheduled delivery date of the original request is used as an upper bound for the ABC computation.

Then, ABC cannot prefer the earlier available quantity from the other substitute and hence, decides for the original request.

With this correction this is changed by removing the upper bound for the original request during BOP in case it is set to the maximum confirmation.

Solution

Apply the correction instructions for this note or upgrade to a SAP S/4HANA support package that contains this correction.

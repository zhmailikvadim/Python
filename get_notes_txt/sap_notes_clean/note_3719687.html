Symptom

  • Error message: "Employee 1234 has not been created."
  • Occurs during billing element creation in the Plan Customer Projects app.
  • Issue arises when an employee with a future employment start date is assigned as Engagement Partner in the project role.

Note: (1234 represents the employee ID)

Environment

SAP S/4HANA Cloud Public Edition

Reproducing the Issue

  1. Open the Plan Customer Projects app.
  2. Navigate to the Billing tab.
  3. Attempt to create billing elements while assigning an employee with a future employment start date as Engagement Partner in the project role.

Cause

  • The system behavior is standard in SAP S/4HANA Cloud Public Edition.
  • If a project role is staffed with an employee whose employment start date lies in the future, the system prevents the creation of billing elements due to stricter validations required for billing-relevant processes.

Resolution

  1. Confirm that the employee master data exists in the system and verify the employment start date.
  2. Ensure that the assigned employee is active and valid on the project key date to proceed with billing element creation.

Keywords

error message, billing element creation, future-dated employee, sap s/4hana cloud, project billing, engagement partner, employee start date

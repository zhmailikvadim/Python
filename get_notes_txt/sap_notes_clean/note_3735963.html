Symptom

  • You want to shrink the database log device and release the space back to OS.
  • There is available storage that can be used to create a temporary transaction log device.

Environment

  • SAP Adaptive Server Enterprise (ASE) 16.0 SP03 PL06 or higher for Business Suite
  • SAP Solution Manager (SOLMAN) 7.x
  • SAP NetWeaver (NW) - All versions 

Resolution

This describes a procedure to shrink an SAP ASE transaction log device by using a temporary transaction log device. 
The procedure creates a temporary log device, adds it to the database, recreates the target transaction log device with the desired size, and then removes the temporary log device.

Shrinking the log device file involves inherent risks. Accordingly, the necessity of this operation must be verified, and it must be performed by a DBA.

Prerequisites:
Before performing this operation, please obtain the results of sp_helpdevice and sp_helpdb <database_name> for the target database.

  1. Stop all applications connected to the ASE server

    for example:

    - Replication Server
    - HADR
    - NetWeaver
    - Solution Manager

    For details on how to stop each application, refer to the documentation of the respective application.

    Below is the procedure to stop the Job Scheduler in ASE:

    use sybmgmtdb
    go
    sp_sjobcontrol @name = NULL, @option = "stop_js"
    go

  2. Take full backups of the master database and the target database

    dump database master to '<full_path_file_name>'
    go
    dump database <database_name> to '<full_path_file_name>'
    go

  3. Create a temporary transaction log device and add it to the database
    Create a temporary transaction log device on an available device and add it to the database.

    use master
    go
    disk init name = '<temp_device_name>', physname = '<full path of the physical file>', size = '<device size>'
    go
    alter database <database_name> log on <temp_device_name> = '<size>'
    go

  4. Remove the transaction log device to be shrunk from the database

    dump tran <database_name> with truncate_only
    go
    alter database <database_name> log off <database_device>
    go

    If Msg 5053 occurs after executing the alter database ... log off command, execute the following to move the first page of the transaction log to another device:

    Error message:
    Msg 5053, Level 16, State 1:
    Server '<server_name>', Line 1:
    You cannot shrink the log in database '<database_name>' because there is at least one page to be removed (xxxxxxx) that is part of the log. Dump the transaction log and retry the ALTER DATABASE command.

    Retry the operation:

    dump tran <database_name> with truncate_only
    go
    alter database <database_name> log off <database_device>
    go

  5. Drop the removed transaction log device

    sp_dropdevice <database_device>, dropfile
    go

  6. Recreate the transaction log device with the desired size and add it to the database

    disk init name = '<device_name>', physname = '<full path of the physical file>',  size = '<device size>'
    go
    alter database <database_name> log on <database_device> = '<size>'
    go

  7. Remove the temporary transaction log device from the database

    dump tran <database_name> with truncate_only
    go
    alter database <database_name> log off <temp_device_name>
    go

    If Msg 5053 occurs, run the following again to move the first page of the transaction log to another device and retry:

    dump tran <database_name> with truncate_only
    go
    alter database <database_name> log off <temp_device_name>
    go

  8. Drop the temporary transaction log device

    sp_dropdevice <temp_device_name>, dropfile
    go

  9. Verify the transaction log device size

    Confirm that the log device has been recreated with the specified size.

    use <database_name>
    go
    sp_helpdb <database_name>
    go

  10. Take full backups after shrinking the transaction log device

    dump database master to '<full_path_file_name>'
    go
    dump database <database_name> to '<full_path_file_name>'
    go

  11. Restart applications connected to the ASE server
    Restart all applications that were stopped earlier. Refer to each application's documentation for the startup procedure.

    In ASE, start the Job Scheduler as follows:

    use sybmgmtdb
    go
    sp_configure "enable job scheduler", 1
    go

Symptom

This is a detailed step-by-step guide on how to perform backups and restores in the Cloud Portal environment.

Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.

Environment

SAP Commerce Cloud in the Public Cloud

Resolution

  • Backup Process
    • Understand Backup Types: 
      Standard Data Backup is a comprehensive process that allows you to create a copy of your database, your storage files, or both. These backups take more time to generate, but the resulting copies are permanent and never expire. When you restore from a standard backup, the backup copy itself is not deleted and remains available for future use.
      Quick Data Backup provides a faster method for creating a copy, but it's limited to the database only. You have to enable the Selective Data Backup feature to use Quick Data Backup.
    • Performing a Backup:
      1. Navigate to the Environments section in your cloud portal.
      2. Select the environment you wish to back up.
      3. Locate the Data Backups panel.
      4. Choose the appropriate backup option (Standard or Quick) based on your needs.
      5. Follow the prompts to complete the backup process.
  • Restore Process
    • Locate Backups:
      1. Go to the Environments section and select the environment from which you want to restore.
      2. Check the Data Backups panel to see if any backups are available.
      3. Click Manage to open the Data Backups overview.  
    • Initiate Restore:
      1. Choose the backup you wish to use for the restore process.
      2. Click Restore from Actions.
      3. Follow the prompts to complete the Restore process.

Symptom

AS ABAP Kernel software component from Codeline 793 shows a growing consumption of Mutex handles and memory when performing a huge amount of "HTTP over TLS" connections to a server which requests a client certificate via TLSv1.2 renegotiation.

The constant growth of memory and Mutex handle footprint slows down the operation performance of the affected AS ABAP Kernel component (such as icman) until the affected process runs out of system resources (out of handle or out of memory).

For example, on the Microsoft Windows Platform there seems to be an OS-level Mutex handle limit at about 17 million handles, at which point lots of Windows API calls start failing unexpectedly (e.g. establishing new network connections becomes impossible).

Reason and Prerequisites

There is a (refcounting/release) regression error in the new client-side SSL session cache management code of the low-level AS ABAP Kernel software module BC-SEC-SSL.

The (somewhat uncommon) server behavior prerequisites where this refcounting/release regression error kicks in:
   (1) server negotiates TLS protocol versions TLSv1.0/TLSv1.1/TLSv1.2
   (2) server issues TLS sessionIDs in ServerHello (i.e. the server asserts availaibilty session caching)
   (3) server must request a TLS client certificate via TLS renegotiation
   (4) the server exhibits a very low TLS session resumption rate

AS ABAP Kernel software components from codeline 793 may show a monotonically increasing resource footprint when performing lots of "HTTP over TLS" client/outgoing communication to a server with the behaviour described above.

In contrast to that, AS ABAP Kernel software components from codelines 754, 777 and 91x fail to perform client-side session caching for renegotiated TLS sessions, which avoids the refcounting/release regression error to take effect (i.e. no visible resource leak).

Solution

If you are affected by the problem described in this Note and are using an affected Kernel (see the metadata of this Note),
then please download and install new AS ABAP Kernel software with the minimum patch level indicated in the metadata of this Note.

See also SAP Note 2083594 on the installation of AS ABAP Kernel patches in general.

See also SAP Note 2907361 on Downward Compatible Kernels and the Release Roadmap for Kernel 77x, 78x, 793 and 916

A possible workaround for the resouce leak in Kernel 793 is to disable the client-side SSL session cache in the Kernel software module BC-SEC-SSL using the custom profile parameter value ssl/client_cache_size=0.

Symptom

  • Users encounter an HTTP 431 – Request Header Fields Too Large error when attempting to log in to SAP Build Work Zone.
  • The error occurs even after SAP Identity Provisioning Service (IPS) is configured and provisioning jobs are running successfully.
  • The issue is observed for users who belong to a large number of groups or role collections.

Environment

  • SAP Build Work Zone, Standard Edition
  • SAP Cloud Identity Service

Cause

The HTTP 431 error occurs when the SAML/OIDC token size exceeds the allowed header size limit, typically when a user belongs to more than 150 role collections.
Even with IPS enabled and jobs running successfully, the error may persist if the groups attribute is enabled in the IAS WorkZone application. This causes IAS to include all group memberships directly in the authentication token at login time — independent of IPS — keeping the token oversized and triggering the 431 error.

Resolution

Since IPS is already provisioning both users and groups from Source system for example: S/4 HANA directly into WorkZone's database, disable the groups attribute in the IAS WorkZone application. This attribute only controls what is sent in the SAML/OIDC token at login time. As WorkZone already receives group data via IPS, it no longer needs to rely on the token for group assignments.

Steps to disable Groups attribute:

  1. Log in to the IAS Admin Console
  2. Navigate to Applications → WorkZone application
  3. Go to Attributes configuration
  4. Locate the groups attribute and disable it
  5. Save the changes and retest login

Note: This change is limited to the WorkZone application entry in IAS. Other applications with their own separate IAS application entries will be completely unaffected.

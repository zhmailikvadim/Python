Symptom

You are saving changes in the customizing for breakdown categories (T-code CX1I4).

Either selected customizing request does not contain all necessary details for Breakdown categories (subassignments) and these details are registered under workbench request,

or you observe an error message TK 897 upon saving CX1I4 settings: "You cannot use request XY. Request cannot be used since it does not have the type as requested. Choose another request.".

Other Terms

Breakdown category, Content activation, TF103, TF103A, VC_TF103, CX1I4, TK 897

Reason and Prerequisites

Program processes the change under existing workbench request (order) automatically instead of selected customizing request by user.

Note:

  • In context of Best Practice content for SAP S/4HANA for Group Reporting (1SG) activation -> *workbench* and *customizing* request is expected.
  • In context of CX1I4 settings save -> *customizing* request only is expected.

Solution

Import the support package or implement the advance correction using the correction instructions.

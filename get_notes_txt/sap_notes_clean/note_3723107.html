Symptom

During SAP HANA instance startup, the landscapeHostConfiguration.py Python script reports inconsistent status transitions. The script reports "partial" status first, then changes to "unknown", and finally shows "starting" when executed repeatedly during the nameserver startup process.

In environments using Cluster Pacemaker Resource Managers, this can trigger unintended automated node fencing during database startup operations, particularly in SAP HANA scale-out (multi-node) environments.

Other Terms

landscapeHostConfiguration.py, nameserver startup, Pacemaker cluster environment, SAPHanaSR resource agents, network requets, unknown

Reason and Prerequisites

Issue number 348679

Reason:

Due to an enhancement in network availability detection during nameserver initialization, the network port is detected earlier in the startup sequence than previously. This creates a brief window where the port is accessible but the nameserver is not yet ready to process requests, causing the monitoring script to report "unknown" status instead of "partial". This status change may trigger Cluster Pacemaker Resource Managers to initiate node fencing operations.

Affected Releases:

  • SAP HANA 2:
    • Revisions: 079.03 - 079.08 (SPS07)
    • Revisions: 083.00 - 089.00 (SPS08)

Prerequisites:

  • SAP HANA is configured in a Pacemaker cluster environment using SAPHanaSR resource agents.
  • The landscapeHostConfiguration.py script is executed during nameserver startup (typically called by Pacemaker's SAPHanaController or SAPHana resource agent for monitoring). See SAP Note 3288971. 
  • The timing of the script execution coincides with the period after the network port has been opened but before the NameServer can handle network requests. 

Solution

Apply one of the following SAP HANA Revisions:

  • SAP HANA 2:
    • Revisions >= 079.09 (SPS07)
    • Revisions >= 089.01 (SPS08)
    • or higher

Symptom

This SAP Note contains five read OData services from the loss environment in reinsurance that are required as technical pre-implementation steps for further development.

These services refer to Customizing that is relevant for a loss. The main purpose is to provide the consumer of these services with the possibility to generate corresponding value helps.

Other Terms

  • Web Service
  • Remote API

Reason and Prerequisites

The retrieval of Customizing data that is relevant for a loss.

Solution

Troubleshooting description:
The following services exist in the system after you implement this SAP Note:

  • /MSG/API_CBC_INSURCLCAT_RD_O4: Treaty Class - Read

  • /MSG/API_CBC_OBJTYPE_READ_O4: Object Type - Read

  • /MSG/API_CBC_SGLRSKACCTMODE_O4: Accounting Mode - Read

  • /MSG/API_CBC_SGLRSKPRTCPTYP_O4: Read Participation Type

  • /MSG/API_CBC_SGLRSKSTS_READ_O4: Single Risk Status - Read

Note that the services are currently not released for productive use. For this reason, non-compatible changes to the services may take place in the future. The services will be released at a later point in time.

Manual activities:
Manual post-implementation steps are required.

If, when you implement this SAP Note, the system issues a message stating that a class cannot be created in the namespace /MSG/, you can ignore this message.

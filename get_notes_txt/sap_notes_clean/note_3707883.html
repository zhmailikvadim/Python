Symptom

When extracting resumes or attachments from Integration Center, Content Preview can't be viewed for some attachments. Instead this message appears:

Attachment type application/vnd.openxmlformats-officedocument.wordprocessingml is not viewable. A manual download is required.

Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental

    Environment

    SAP SuccessFactors Recruiting Management

    Reproducing the Issue

    1. Create an Integration Center job (Integration Type: Attachment)
    2. Select an starting entity (e.g. JobApplication)
    3. In Configure Fields tab, complete the association of content and content type
    4. Attempt to preview files in Content Preview
    5. This message appears for some attachments: "Attachment type application/vnd.openxmlformats-officedocument.wordprocessingml is not viewable. A manual download is required."

    Cause

    The message is not an error but expected behavior, as some file types aren't supported to be viewed in Content Preview.

    Resolution

    This message appears when the attachment file is in .docx format to inform user that file isn't supported for Content Preview:

    Attachment type application/vnd.openxmlformats-officedocument.wordprocessingml is not viewable. A manual download is required.

    While this message appears if attachment file is in .doc format:

    Attachment type application/ms-word is not viewable. A manual download is required.

    Note: PDF files and images are supported in Content Preview

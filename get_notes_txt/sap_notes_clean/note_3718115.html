Symptom

  • When attempting to perform confirmation of maintenance order using the Perform Maintenance Jobs app an error is received:
    "Runtime Error: BEHAVIOR_ILLEGAL_STATEMENT"
  • In ST22 there is the following Dump:

    Category               ABAP programming error
    Runtime Errors         BEHAVIOR_ILLEGAL_STATEMENT
    ABAP Program           SAPLEBNE
    Application Component  MM-PUR-REQ

     Short Text
         Statement "CALL_SCREEN" is not allowed with this status.

Environment

  • SAP ERP Plant Maintenance (PM)
  • SAP S/4HANA, on-premise
  • SAP S/4HANA, Cloud Private Edition

Reproducing the Issue

  1. Create a Maintenance Order using the Fiori app "Create Maintenance Order."
  2. Add Operation 0010 in the Operations tab.
  3. Navigate to the Components tab for Operation 0010.
  4. Add a Stock Material.
  5. Maintain Item Category = 'N' (Non-Stock).
  6. Save the Maintenance Order.
  7. Verify that a Purchase Requisition (PR) is generated automatically.
  8. Open the "Perform Maintenance Jobs" app (or use IW41 / IW42 for confirmation).
  9. Enter the Maintenance Order number.
  10. Attempt to confirm Operation 0010.
  11. The system keeps loading and eventually results in a backend short dump (ST22).

Cause

The 'CALL SCREEN' is not compatible with the Fiori user interface

Resolution

  • SAP Note 3441587 has been released to resolve the underlying issue which causes this error and dump.
  • Review an implement this SAP Note or update to the relevant Support Package for your release

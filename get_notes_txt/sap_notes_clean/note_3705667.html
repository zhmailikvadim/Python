Symptom

The following problems occur in connection with search strings in the electronic bank statement:

  1. Performance
    If you use complex search strings several times, the runtime of the electronic bank statement is unnecessarily high.
     
  2. FEBSTS
    When you test the search patterns in FEBSTS, the fields Check Number (CHECT), Payment Reference (KIDNO), IBAN of Business Partner (PIBAN), and BIC/SWIFT Code of Business Partner Bank (PASWI) are not evaluated. 
     
  3. Output Messages
    - If several search string target fields are filled differently for an item, the error message FB677 is not sufficient.
    - Message FB134 'Pattern search too complex for pattern &.' is not displayed.

Other Terms

OTPM BSPS_PATTERN_SEARCH PAMA_MATCH FB 134 677

Reason and Prerequisites

Unfavorable programming

Solution

When you implement the attached source code corrections, the following changes are made:

  1. Performance The reading of search strings that are used multiple times to influence different target fields is executed only once.
     
  2. FEBSTS The simulation of the search string now behaves in the same way as the search string with regard to the additional fields during import/postprocessing.
    In addition, FEBSTS issues the last message of the module BSPS_PATTERN_SEARCH.

  3. Output Messages
    - If several target fields are filled differently for an item, the system now issues error message FB677 for each target field.
    In addition, the long text of message FB677 is enhanced so that the user is informed of the names of the search patterns that trigger duplicate hits.
    - If a search pattern is not executed due to complexity, the system issues message FB134.

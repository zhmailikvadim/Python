Environment

Oracle 19c Database ,Oracle Grid Infrastructure ,opatchauto 

Reproducing the Issue

ROOTCRS_PREPATCH execution for while SBP applied on the database

Cause

This is because the value of below command is SUCCESS.

$GI_HOME/bin/cluutil -ckpt -oraclebase $ORACLE_BASE -chkckpt -name ROOTCRS_PREPATCH -status
SUCCESS


This can happen if the postpatch was not completed successfully in the previous run and was ignored.

Resolution

1) As grid user , execute :
$GI_HOME/bin/cluutil -ckpt -oraclebase $ORACLE_BASE -writeckpt -name ROOTCRS_PREPATCH -state START

2) Verify:
$GI_HOME/bin/cluutil -ckpt -oraclebase $ORACLE_BASE -chkckpt -name ROOTCRS_PREPATCH -status
START

3) Check for all nodes of the cluster and perform the above actions where required.

4) Re-execute opatchauto apply command or 'rootcrs.sh -prepatch' as required.

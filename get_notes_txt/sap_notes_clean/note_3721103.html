Symptom

This SAP Note provides your with corrections on the issue that wrong CPF PIE wage types, including wage type /31L, /31M and /31N, saved in CRT table. This issue occurrs for cases that multiple payroll run in the last period, for example you run a bonus payroll run after regular payroll, then you would find amount of CPF PIE wage types doubled in CRT table.

Other Terms

 CPF Paid-in-Error, Wage Type /31L, Wage Type /31M, Wage Type /31N, CRT Table

Reason and Prerequisites

This is a program error.

Solution

You could apply HR Support Pakcage or implement instructions in this SAP Note.

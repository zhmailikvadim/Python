  • The function DB_GET_TRIGGERS for table cluster RFBLG shows there are SLT DB triggers. 

Environment

  • SAP NetWeaver Release independent
  • SAP Landscape Transformation Replication Server

Cause

Return code 128 means a "duplicate key" error. Duplicate is caused by the records in the logging table created by SLT.

IUUC_SEQUENCE is the primary key of the logging table. It should be unique and constantly incremented.

The cause is likely one for the following:

  • A DB refresh was done, resulting in reset of logging table IUUC_SEQUENCE such that the sequence value was lower than existing records in the logging table.  As the triggers fire, the sequence object increments higher and if the sequence value conflicts with any pre-existing record in the logging table, duplicate key occurs.  
  • An action occurred in the DB that has reset sequence object, such as dropping and recreating it causing it to start from initial.

Resolution

See KBA 2751637 - Duplicate key error for logging table.

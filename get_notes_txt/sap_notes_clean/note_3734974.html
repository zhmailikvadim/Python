Symptom

If you use the IAIK library, for example to generate X.509 certificates, and register it as the first global (JVM-wide) JCE security provider, for example by calling iaik.security.provider.IAIK.addAsProvider(), TLS handshakes to systems using certificates chained to DigiCert Global Root G3 can fail.

When IAIK is registered as the first provider, its implementation can be used during TLS certificate validation. Validation may fail if the selected IAIK provider setup does not provide the ECC support required for the certificate chain presented by the peer.

Other Terms

IAIK, TLS handshake, DigiCert Global Root G3, BTP Neo

Reason and Prerequisites

SAP BTP, Neo Environment

To comply with the latest recommendation from the German BSI on key length for certificates and to improve security, SAP BTP, Neo environment adopts the higher-security-level Root Certificate Authority “DigiCert Global Root G3” PKI, see 3666687.

Not adding the DigiCert Global Root G3 to any of the trust stores before the certificate exchange in BTP, Neo Environment happens, may result in communication failures or handshake errors, or even outages. 

Solution

To allow ongoing validation and avoid the IAIK-related problems, you have the following options:

1. Remove Global Provider Registration                                                                                                                  

    - Remove the IAIK.addAsProvider() statement to avoid registering IAIK as a global (JVM-wide) security provider

2. Register IAIK as Last Provider                                                                                                                       

    - Replace IAIK.addAsProvider() with Security.addProvider(new IAIK()) to register IAIK as the last security provider in the chain

3. Add ECC Support                                                                                                                                      

    - Register the IAIK ECCProvider to provide ECC (Elliptic Curve Cryptography) algorithms that are not included in the default IAIK JCE security provider

    - Add the iaik:iaik_ecc_signed dependency to your project

    - Ensure ECCProvider is registered immediately after the default IAIK provider (at position #2 in the security provider list)

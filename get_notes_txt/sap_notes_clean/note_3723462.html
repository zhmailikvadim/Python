Symptom

During installation of SAP HANA XS Advanced (XSA) using hdblcm, the installation step 

add_host_roles

fails on the system that previously acted as primary site in a SAP HANA System Replication (SR) landscape.

The failure occurs while executing: hdbnsutil -addHostRoles --role=xs_worker
An error similar to the following is reported: Could not determine the host roles of the target system(s). Possible reason: System is offline. (50264)

In some scenarios the same installation procedure may succeed on other hosts in the former replication landscape, which can create the impression that the issue only affects the former primary system.

Environment

SAP HANA DB Version 2.0

Cause

This situation can occur if system replication was not fully decoupled before starting the XSA installation.

If SR unregister or topology cleanup was not completed successfully, the former primary system may still contain system replication topology metadata referencing remote sites.

When SR topology information is still present, host role determination during the hdblcm step add_host_roles may fail with error 50264, because the system is not operating in a fully standalone state.

Resolution

Ensure that system replication has been fully decoupled before starting the XSA installation.

Verify the SR state on the former primary system: hdbnsutil -sr_state

Confirm that the system no longer reports attached secondary or consumer sites.

If SR was previously configured in a multi-tier topology, ensure that all sites are properly unregistered according to the SAP HANA System Replication documentation.

In general, downstream sites should be unregistered before upstream sites to ensure that the topology is removed correctly.

Example:

  • unregister the DR site first
  • then unregister the secondary site

If the unregister procedure is executed from the primary site context, ensure that the remote systems are stopped before executing the unregister command.

Once all SR relationships have been removed and each system is operating as a standalone system, perform the XSA installation individually on each host or system.

After the installation has completed successfully, system replication can be configured again if required.

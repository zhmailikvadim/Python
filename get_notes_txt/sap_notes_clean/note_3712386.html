Symptom

While selecting values on the selection screen of the Run Statutory Report, some fields are greyed out and values cannot be entered.

Environment

SAP S/4HANA 2023

Reproducing the Issue

  1. Open the Run Statutory Reports Fiori app.
  2. Generate a report (for example, Bulgaria: Fixed Assets Report).
  3. Click on New Run. The selection screen appears, but some fields are greyed out and do not allow value entry. For example, mandatory fields such as Company Code, Book Depreciation Area, and Fiscal Year cannot be filled because they are disabled.


    "Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental." 

Cause

Expected behaviour for the parameters.

Resolution

  1. In the image referring to the Define Statutory Report for the Report Category ID eg: BG_FXDAST_REP, the report definition BG_FXDAST_REP_v0 is displayed.


    "Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental." 



  2. Under the Parameter Definition section, the Company Code field is set to Read Only = Yes, meaning it can be viewed but not edited.

    "Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental." 


  3. As a result, fields such as Book Depreciation Area and Fiscal Year are also set to Read Only, which prevents any value entry.
  4. If the Read Only setting is changed to No, these fields will become editable.
  5. The same logic applies to other Report Definitions as well. This is standard system behavior based on how the field properties are configured.

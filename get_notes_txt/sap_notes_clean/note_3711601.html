Symptom

EWM printer-pool functionality can be used in two different ways:

- Without RF: Printer-pool is used to distribute print jobs e.g. Pick list to several physical printers.

- With RF: Printer-pool is used to assign a printer which is linked to a resource e.g. forklift. This functionality was introduced with note 3396347.

In both cases, the determination of the printer (which should be used) happens automatically in the execution phase of the PPF (Post-Processing Framework) action for printing.

The printer-pool is ignored if the PPF action uses so-called 'Processing Parameters'. In EWM they are only used for re-printing where user manually enters the printer, he wants to use.

Other Terms

/SCWM/RFUI, /SCWM/RSRC_PRN, /SCWM/60000431, /SCWM/PRHU6, /SCWM/PRWO6

Reason and Prerequisites

PPF creates processing parameters but with initial values and EWM only checks if they exist but not if they are filled. Because of this, the printer-pool is ignored.

Solution

Apply the note to your system.

After that EWM checks the content of the processing parameters and use the printer-pool if they are initial.

Symptom

You have observed that the Fiori Launchpad Startup Page performance is slow. Further analysis has revealed that the OData $batch request lasts several seconds.

Other Terms

Fiori Startup Page, Page Builder, performance

Reason and Prerequisites

A bug in the software prevents the requests contained in $batch to be processed in parallel. Instead each request is waiting for the predecessor to release its locks. The effect is that the requests are processed sequentially thus counteracting the purpose of $batch.

Solution

This is part 2 of two notes that fix the issue. Please apply the correction measure or Support Package 08 of the UI2 Add-On.

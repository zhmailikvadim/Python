Symptom

  • Access to the Record GHG Data application in SAP Sustainability Control Tower (SCT) results in a connectivity refusal.
  • Error message: "<authentication URL> refused to connect".
  • Application functionality depends on having a parallel session of SAP Sustainability Footprint Management (SFM) open in the same browser.
  • Connectivity failure occurs during the authentication handshake between the SCT Launchpad and the calculation engine.

Environment

  • SAP Sustainability Control Tower.
  • SAP BTP (Business Technology Platform) Subaccount.
  • SAP Sustainability Footprint Management.

Reproducing the Issue

  1. Access the SAP Fiori Launchpad for SAP Sustainability Control Tower.
  2. Ensure no active sessions of SAP Sustainability Footprint Management are open in other tabs.
  3. Open the Record GHG Data application.
  4. Observe the authentication error: refused to connect.

Cause

The Record GHG Data application utilizes the SAP Sustainability Footprint Management (essential) engine. The behavior is caused by missing trust configurations in the SAP BTP subaccount for sustainability-specific authentication domains. A parallel SFM session provides a temporary workaround by establishing a valid security context, but does not resolve the underlying configuration gap.

Resolution

  1. Log in to the SAP BTP Cockpit and navigate to the specific subaccount used for SAP Sustainability Control Tower.
  2. In the left-hand panel, go to Security → Settings.
  3. Locate the Trusted Domains section.
  4. Add the following wildcard domain to the list: https://*.sustainability.cloud.sap.
  5. Save the changes.
  6. Clear the browser cache or use an Incognito window to test the Record GHG Data app again without opening SFM in parallel.

Please follow the detailed resolution steps provided in the official documentation:

  • SAP Help Portal: Enabling Corporate Carbon Footprint Calculation.

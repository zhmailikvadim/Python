Symptom

  • The "Smart Insights" functionality on a dashboard based on an Import Model to BW/4HANA is not working as expected.
  • Insights are not displaying correct values, such as incorrect variance calculations and percentage values.
  • This issue occurs for all stories.
  • The functionality was working correctly on dashboards developed over the past year.

Environment

SAC Analytics Cloud 2026.3.1 

Reproducing the Issue

  1. Log into SAC;
  2. Open the Story;
  3. Navigate to the left side of the dashboard and go to the chart;
  4. Click on 'More Actions' (three dots) and open 'Smart Insights';
  5. Observe that the first insight shows the top 10 values as 1% overall, however, the real value would be 22%;
    1. It also happens with another insights.

Cause

This issue is related to filters of type range, as observed in the linked story. When a filter of type “member selection” is applied instead (using the same date values), the issue no longer occurs.

    Resolution

    • The Smart Insight feature has been redesigned in the upcoming release on QRC2 with an updated architecture. With this new implementation, the issue is fully resolved.
    • Workaround: keep the existing Range filter and add an additional filter of type “Member Selection” on the same dimension, using the same date value(s). Applying the Member Selection filter on top of the existing Range filter prevents the issue from occurring.
      • This configuration can be used until the permanent fix is delivered in the upcoming release.

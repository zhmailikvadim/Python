Symptom

You confirm a warehouse task with standard exception code BIDF or BIDP to trigger the creation of an alternative picking WT. The WT is created correctly, but it is not assigned to the current WHO.

Reason and Prerequisites

Symptom

Symptom

Technical note for checkmarx findings fix.

Other Terms

F2097, Checkmarx,

Reason and Prerequisites

Technical note

Solution

This note contains technical corrections related to checkmarx.

Implement the attached Correction Instruction or the mentioned Support Package.

Afterwards reindex the application on your frontend (gateway) system as described in SAP Note 2227577 and clean the caches according to SAP Note 2017117.

You select a component in the Compliance Workbench. This component contains a basic material from a supplier. The system generates and displays text that combines the supplier name and the basic material name. The system displays this combined text instead of the basic material name at the following locations:

  • In the product structure
  • On tab Regulatory Information of the requirement Automotive IMDS Reg. Information
  • In the generated compliance documents

  

Other Terms

International Material Data System (IMDS)

Reason and Prerequisites

The symptom occurs due to a program error.

Prerequisites are available in the relevant correction instructions.

Solution

After implementing this SAP Note, the system exclusively displays the name of basic material without adding the name of the supplier.

Alternatively, import either of the corresponding Support Packages for the component extension of SAP EHS Management or the Feature Packages Stacks for SAP S/4HANA.

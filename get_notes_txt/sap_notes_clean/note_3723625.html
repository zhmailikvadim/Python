Symptom

The PUT Participant API failed to locate the record for Payee when attempting to update participant data.

The error message stated: "Could not find the participant with the given value of the key(s) (payeeSeq), (4503599623456789)."

Environment

SAP Sales Performance Management
- Incentive Management

Reproducing the Issue

  1. Attempt to retrieve participant data using GET /mtsvc/tcmp/rest/v2/participants({id})/versions with payeeSeq filter:
  • "payeeSeq": "4503599623456789", "payeeId": "50002651913".
  1. Execute PUT /mtsvc/tcmp/rest/v2/participants with the following payload:

```json

[

{

"payeeSeq": "4503599623456789",

"payeeId": "50002651913",

"effectiveStartDate": "2026-02-01T00:00:00.000+00:00",

"effectiveEndDate": "2200-01-01T00:00:00.000+00:00",

"genericAttribute2": "Support_Test"

}

]

```

  1. Receive the following error response:

```json

{

"participants": [

{

"_ERROR_": {

"message": "Could not find the participant with the given value of the key(s) (payeeSeq), (4503599623456789)."

}

}

]

}

```

Cause

The PUT command is designed to update participant data but does not support modifications to effective dates with Time stamps.

Resolution

  1. This has been reported to our dev team, for more information Refer to the See Also for detailed guidance on using the other API.
  2. A code correction has been created to address this issue, and the fix is scheduled to be included in the 2604 release:
  • Non-PRD release date: Saturday, April 11, 2026.
  • PRD release date: Saturday, April 18, 2026.

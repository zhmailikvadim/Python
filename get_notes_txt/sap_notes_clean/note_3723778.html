Symptom

Tasklist Scope collection for Remote/Shell BW/4HANA fails with error:

  • XXXX not found or not released for ODP in system XXX (Myself System) (see SAP Notes 2232584 and 1560241).

Environment

  • SAP NetWeaver
  • SAP BW/4HANA

Reproducing the Issue

Start the tasklist run for a Remote or Shell BW/4HANA conversion.

Cause

Tasklist run includes a customer datasource, created in the same BW system (Myself Datasource). These scenarios need to be replaced by ODP extraction. Since SAP BW/4HANA does not allow to create load scenarios with the SAPI-Context, it is recommended to transfer them to the ODP-CDS. However for convenience transfer to ODP-SAPI is supported as well. Before the BW Datasource (TLOGO RSDS) can be transferred, the TLOGO OSOA of the appropriate datasource has to be transported to the receiver system. 

Resolution

In order to be able to use still datasources which has been created in transaction RSO2 in the sender system, transport first the TLOGO OSOA to the receiver system. In order to collect the required objects, report RS_B4HANA_CREATE_OSOA_TRANSP can be used. 

  1. Execute report RS_B4HANA_CREATE_OSOA_TRANSP in the sender system and maintain the appropriate tasklist run.
  2. Check the created transport request if all necessary objects are available.
  3. Customer extraction tables/view has to be transported manually from the sender to the receiver system. 
  4. Transport the generated OSOA transport request to the receiver system. 
  5. If the datasource is active in the receiver system but still not available in table ROOSATTR, execute report RODPS_OS_EXPOSE in the appropriate receiver system. 
  6. Start the scope collection via tasklist run in the sender system.

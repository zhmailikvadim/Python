Symptom

Are SAP AS server its tools impacted by log4j vulnerabilities from CVE-2025-68161 ?

Environment

SAP Adaptive Server Enterprise (ASE) 16.0 & 16.1

Cause

  • SAP ASE and its tools are not impacted by CVE-2025-68161
  • CVE-2025-68161 is in relation to use the Socket Appender component within log4j libraries
  • The SocketAppender code is not used in the log4j implementation in ASE and therefore ASE is not exposed to this

  • CVE-2025-68161 vulnerabilities are fixed in log4j-2.25.3

Resolution

  • While SAP ASE is not exposed to CVE-2025-68161, upcoming ASE versions will include a newer version of log4j jar files where this vulnerability is fixed
  • ASE 16.0 SP04 PL09 is scheduled to shipped with log4j version with the fix for CVE-2025-68161
  • For information on the release schedule, refer to 
    KBA 2075653 - Targeted ASE 16.0 Release Schedule and CR list Information - SAP ASE

  • To avoid false positive alerts being raised against log4j files by security scanner, log4j can be removed but this may cause errors for certain features
  • Refer to this KBA 3129897 for more information when considering removing log4j files and to what functionality these files are linked

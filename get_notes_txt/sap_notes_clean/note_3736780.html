Symptom

When starting with SRFVC_RPG_ENT for Colombia to set periodicity of the reports CO_WHT_INCOME_DCL or CO_WHT_VAT_DCL, an error occurs: "Tax calendar CO_R is not implemented for selected from 01.01.2022 and to 31.12.9999 date Message No. SRF_CONFIG021."

"Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental."

Environment

  • SAP S/4HANA
  • DRC Colombia

Reproducing the Issue

  1. Open transaction SM34.
  2. Enter view cluster SRFVC_RPG_ENT for Colombia.
  3. Select reporting entity: CO_RPG_ENT.
  4. Choose report category (for example, CO_WHT_INCOME_CERT) and attempt to set periodicity with tax calendar CO_R.

Cause

Parameter specific to the reporting entity is missing: CO_WHT_NIT_NUMBER.

Resolution

  1. Open SM34 > SRFVC_RPG_ENT > Define Compliance Reporting Entities > Enter Parameters Specific to Reporting Entity
  2. Maintain the reporting-entity-specific parameter CO_WHT_NIT_NUMBER as per product documentation: Implementation guidance for Colombia withholding tax reports.
  3. Save the configuration and ensure the parameter is active for the intended validity period.
  4. Return to SM34 > SRFVC_RPG_ENT > Define Compliance Reporting Entities > Assign Report Categories to Reporting Entity > “Set Periodicity of Report Category” for the affected report categories (for example, CO_WHT_INCOME_DCL and CO_WHT_VAT_DCL).
  5. Maintain the periodicity, including Tax Calendar = CO_R and the required date range, then save.
  6. Re-test the change to confirm that message SRF_CONFIG021 no longer occurs.

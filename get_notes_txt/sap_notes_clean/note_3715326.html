Symptom

  • Classification data is not updating in S/4HANA 2025 system even though the GTS log states that it was successful.
  • The program /SAPSLL/COMCO_RETRANSFER runs successfully, but the classification number is not populated in the S/4HANA 2025 system.

Environment

  • Global Trade Services (GTS)
  • Global Trade Services / Logistics Services (SLL)
  • SAP Global Trade Services
  • SAP Global Trade Services, Edition for Hana
  • SAP S/4HANA 2025

Reproducing the Issue

  1. You have upgraded your S/4HANA system to the S/4HANA 2025 release.
  2. Maintain a classification number for a product in GTS.
  3. Execute the program /SAPSLL/COMCO_RETRANSFER.
  4. The GTS log displays successful message.
  5. In S/4HANA system check table /SAPSLL/MARITC or display classified products app and the classification did not get updated.

Cause

  • In S/4HANA 2025 a some new fields have been introduced in the definition of the numbering scheme. The issue occurs if configuration 'Receiving of Classification' is set to 'local' in the definition of numbering scheme in the S/4HANA 2025 system.

Resolution

  1. Adjust the configuration for 'Receiving of Classification' in the definition of numbering scheme to 'SAP GTS (RFC)' in the following configuration point:

               SPRO > IMG > Governance, Risk and Compliance > Classification > Commodity Codes > Define Numbering Scheme for Commodity Codes > Go the relevant numbering scheme details > Distribution of Classification > Receiving of Classification

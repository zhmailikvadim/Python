Symptom

Branch Code functionality is not available for Singapore in Contract Accounts Receivable and Payable or Convergent Invoicing for Government Individual Endpoint ID and Sub-BU.

When processing incoming and outgoing invoices for Singapore, customers need to support branch codes. The current system does not enable branch code processing for Singapore country-specific requirements, especifically for:

- Government Individual Endpoint ID

- Sub-Business Unit (Sub-BU) handling

- Direct Reporting Channel integration

Users cannot configure or use branch codes for Singapore transactions in FI-CA and Contract Accounts components, limiting compliance with Singapore regulatory requirements.

Other Terms

Singapore Branch Code, DRC, Invoice

Reason and Prerequisites

The branch code functionality was originally developed for Thailand as a country-specific legal requirement (delivered as from SAP S/4HANA 1709). The same technical framework should be enabled for Singapore and other countries.

Singapore has similar regulatory requirements for branch code reporting, particularly for:

- Government entities requiring Individual Endpoint identification

- Sub-Business Unit tracking for large organizations

- Direct Reporting Channel compliance

Solution

This branch code framework can be used in other countries where tracking the branch of a business partner is required for business or regulatory purposes. The solution is not limited to Singapore and Thailand only - it provides a flexible, country-extensible framework that can be configured for any country with similar requirements for branch identification and reporting.

The view for activation of branch code can be called via transaction SM30, with VTFKBCODE1.

To customize the branch code determination logic, create a copy of the sample event (transaction FQEVENTS) FKK_SAMPLE_0047. This is essential in scenarios where a branch code is not explicitly assigned to the contract account, as the system must fall back to deriving the default value from the associated business partner.

This SAP Note is relevant only for Contract Accounts Receivable and Payable (FI-CA) in SAP S/4HANA and SAP S/4HANA Cloud Private Edition.

Symptom

  • When reapplying a new contract to a nomination, the Schedule Date at nomination item level cannot be changed.
  • Attempting to change the Schedule Date causes the Nomination tab to freeze; no validation or error message is displayed and changes cannot be saved.
  • The Schedule Date field appears non-editable.

Environment

SAP S/4HANA

Reproducing the Issue

  1. Create a commodity contract and a nomination linked to it.
  2. Create an LDC for the nomination and generate the Application Document (AppDoc)
  3. Reverse the AppDoc using the Document Flow work center.
  4. Open the nomination  → Apply New Contract to assign a different contract  through “Mass Change” option on Nomination Tab 
  5. Manually update the pricing lot id.
  6. Attempt to change the Schedule Date for the nomination item.
  7. Observe that the Nomination tab freezes and the Schedule Date cannot be changed.

Cause

The LDC associated with the nomination was not reversed; this keeps the Schedule Date locked.

Resolution

Standard behavior is that when the LDC has been released and not reversed/obsoleted, the schedule date field is protected the behavior is as per ACM design and not a bug  as part of the standard product design, the 'schedule date' is not editable.

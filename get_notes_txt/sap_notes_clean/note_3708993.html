Symptom

This report can delimit all roles for BPs connected to personnel numbers who are so called No-Shows.
No shows are people that were hired but never showed up for work and therefore are set to “Withdrawn” (Employment Status = 0) via executing a “No-Show” action.

Reason and Prerequisites

For this report to work you must have activated the business function /SHCM/EE_BP_1.

Solution

The following input parameters are available for this report:

No Show Measure:
This lets you customize which measure you use to designate no shows. For a measure to be considered a No-Show measure it has to have the values "1" for the field "FUNCH" (Function character for action) and "0" for the field "STAT2" (Employment Status) in table "T529A". The measure "70" (Incorrect entry) is usually used to designate No Shows.

No Show Date:
Only personnel numbers who were hired at or after the given date are considered for this report.

Personnel Number:
Here you can specify a single personnel number to check.
If an input is made here only the given number will be handled by the report.
If the input field is empty, all personnel numbers with a measure given by "No Show Measure" and created at or after "No Show Date" are handled by this report.

Roles to Delimit:
Here you can specify which roles to delimit.
If an input is given only these roles will be delimited. All other roles will be ignored.
If no input is given all roles for the found BPs will be delimited.
If only the main personnel number in a concurrent employment legacy case is a no show the role "BUP003" cannot be delimited and will not be delimited even if it is entered here.

Past Days:
This field should be used if the report is planned to run in the background on a regular basis. When this report runs in the background it only looks at personnel numbers created on the current date. With this parameter you can specify a number of days to look in the past.

Lock BPs:
If this flag is set all the BPs connected to the No-Show personnel numbers will additionally be locked.

Test Run:
If this flag is set no changes will be made to the data. The report will just inform you of the changes that would be made if you run the report in a regular run.

If the program detects No-Shows, the output is delivered in a table format.
The column "PersNo." lists all the No-Show personnel numbers found.
The column "Bus. Partner" lists all the BPs connected to these personnel numbers.
The column "BP Role" lists all the roles a BP has.
The column "Status" shows a message describing what was done to the role.

If a user is not authorized to use the report, or the business function /SHCM/EE_BP_1 is not turned on the report will display a message when trying to start it.

If no No-Shows are found the message a message will inform the user that there are no No-Shows in the given time frame or with the given measure.

If the input in field "No Show Measure" is not a No-Show measure a message will inform the user about this.

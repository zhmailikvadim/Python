Symptom

When you read an application log from the database using the function module BAL_DB_LOAD, the short dump IMPORT_DECOMPRESS_FAILED or CONNE_IMPORT_WRONG_COMP_LENG occurs.

Reason and Prerequisites

Application log messages are stored in a compressed binary format in the database. This binary data is corrupted. Accordingly, it is no longer possible to unpack all messages, which triggers the short dump.

Solution

Implement the correction instructions contained in this SAP Note or import the relevant support package.

After you implement the corrections, this inconsistency no longer causes a short dump. Instead, all messages that can still be unpacked are loaded by the function module BAL_DB_LOAD.

The messages that cannot be unpacked are missing in the application log. Instead, the system issues error message BL256 'The log is incomplete'. is also written to the application log. As a result, the behavior is the same as before this correction for other inconsistencies in the compressed data.

Symptom

Customer needs to register their OpenText software installations to SAP Solution Manager or Focus Run system via SLD registration process. 

Environment

SAP Archiving and Document Access 16.2.10 or above (both SAP ERP and S/4)

SAP Extended Enterprise Content Management 16.2.10 or above (both SAP ERP and S/4)

Resolution

SAP supports the following OpenText software components using the SLD data supplier of Apache Tomcat. 

  • OpenText Directory Service (OTDS)
  • OpenText ECM WebService
  • OpenText WebViewer

Customers could perform the following steps to complete the registration: 

1. Install the Apache Tomcat Data Supplier according to SAP note 1508421

2. Download the corresponding OpenText software metadata files from this KBA

As the OpenText software components are defined in multiple SAP products, customer needs to determine which SAP product is the best candidate to register. There are 4 possible options:

    • SAP Archiving and Document Access by OpenText
    • SAP Archiving and Document Access by OpenText for SAP S/4HANA
    • SAP Extended Enterprise Content Management by OpenText
    • SAP Extended Enterprise Content Management by OpenText for SAP S/4HANA

If the OpenText system is used for both SAP Archiving and Document Access and SAP Extended Enterprise Content Management scenarios, choose SAP Extended Enterprise Content Management product. 

If the OpenText system is connected with both SAP ERP and SAP S/4HANA systems, choose SAP S/4HANA related product. 

3. Place the corresponding product release metadata files under the META-INF folder of the installed SLD Data Supplier application, e.g. D:\Program Files\Apache Software Foundation\Tomcat 9\webapps\sap.com~TomcatSLDDataSupplierWEB\META-INF

4. Trigger the SLD Data Supplier. 

5. Verify the SLD registration result in LMDB. 

Keywords

SLD registration, TOMCAT, OpenText Directory Service, OpenText Image WebViewer, OpenText Extended ECM Webservices

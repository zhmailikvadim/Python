Symptom

  • ALV grid selection on touch devices does not function as expected when calling a transaction via SAPGUI for HTML / WebGUI.
  • The screen does not navigate to the next screen on the mobile device.
  • Double-tap functionality does not work in touch mode.
  • The issue does not occur in the desktop version of the application.

Environment

SAP GUI for HTML / WebGUI when touch mode is activated.


Menu -> Settings -> Interaction Design -> Touch Mode: On 

Reproducing the Issue

  1. Login via a Touch Screen 'Android' or 'iOS' Device.
  2. Call the standard SM34" transaction ( this is not specific scenario to this transaction )
  3. View Cluster: VC_NOTIF
  4. Open the application in Display Mode. Click on the 'Display' BUTTON
  5. Try and double click on one of the "Notification Type" item's
  6. Observe that the screen does not proceed further into the "Permitted Periods" screen 

    ( In a DESKTOP Browser a double mouse click will drill into a the Notification Type in more detail. )

     

     

Cause

The touch mode on mobile devices does not support 'double-click' functionality, which is needed for navigation within the ALV grid structure.
 

Resolution

  1. Use a 'long press' on the field in touch mode to open the context menu.
  2. Select "Details" from the context menu to navigate to the desired screen.

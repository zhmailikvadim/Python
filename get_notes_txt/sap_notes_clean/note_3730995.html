Symptom

Prerequisite:

  • SADL Framework used.

Symptom:

  • Fiori app "Display Export Declarations" shows slow performance.

Environment

  • SAP S/4HANA
  • Fiori

Cause

  • Hint is already in place in the view CEXPDCLITM: @Consumption.dbHints: ['NO_SUBPLAN_SHARING', 'USE_HEX_PLAN', 'PRELIMIT_BEFORE_JOIN'] 
  • Execution does not use HEX engine despite the hint applied (hint is not pushed down into HANA DB during runtime).
  • 3709708 - Performance Issue with SADL Framework SELECT statements

Check (manually), to confirm the issue:

  1. From ST12 trace copy the statement with parameters and execute it manually (ST04 > Diagnostics > SQL Editor).
  2. Find the SQL Statement hash of the execution (ST04 > Performance > SQL Plan Cache).
  3. Check with the report (3327150 - SQL Statement Collection: "HANA_SQL_StatementHash_DataCollector" report for SAP HANA) and confirm the execution does not use HEX engine.

Reproduce the issue with simple SELECT statement (ST04 > Diagnostics > SQL Editor), e.g.:

Statement:
SELECT * FROM "SAPHANADB"."CEXPDCLITM"( P_DISPLAYCURRENCY => 'USD', P_DISPLAYUNIT => 'KG', P_EXCHANGERATETYPE => 'M', P_EXCHANGERATEDATE => '20250308') where ExportDeclaration = '2260000016'

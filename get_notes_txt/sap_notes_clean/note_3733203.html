Symptom

Wrong results when using Import from Excel for Inventory Counting when working with Batch and with UOM.

Reproducing the issue

Scenario:

  1. Create item "uom1" with UOM group where 1 Case = 43.02 Pack. Go to Inventory Data, choose "Pack" for UoM Code,  and choose "Case" for

    Inventory Counting UoM Code.

  2. Add Goods Receipt for this item with quantity 273.87 and allocate 3 batches the following way:

    C1 - 35.70

    C2 - 86.04

    C3 - 152.13.

  3. Create Inventory Counting for the item:
    • Set focus on "UoM Counted Qty"  and click Add Items > Import Items.
    • In the Import from Excel form pop-up, map the file columns to the object fields as follows:

A - Item Code
B - Item Description
C - Warehouse Code
D - Counted Quantity
E - UoM Code
F - Batch Number  

    • Import the following Excel file: 

uom1    uom1    01    0.83    Case    C1
uom1    uom1    01    2.19    Case    C2
uom1    uom1    01    3.23    Case    C3

4. UoM Counted Qty is 6.25 and press Ctrl + Tab to allocate the batches. The Batch Number Counting form pop-up appears.

Expected behavior:

You will see in the Rows from Documents that Quantity is 268.875, Total Needed is 0 (blank), and Total Selected is 268.875. 

Actual behavior:

You will see in the Rows from Documents that Quantity is 268.875, Total Needed is -0.001, and Total Selected is 268.876.

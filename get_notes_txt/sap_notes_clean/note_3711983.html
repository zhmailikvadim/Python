Symptom

A Consolidation process has errors at the validation step but failed at the activation step due to the same errors as validation step and hard stop error.  

Environment

  • SAP Master Data Governance-Consolidation and Mass Processing

Resolution

  • There can be two types of errors during step execution. First type is business errors when some data or system customizing settings are incorrect, e.g. missing company code or plant records, incorrect accounting or characteristic customizing settings, etc. These are recoverable error because data or system settings can be corrected and step rerun. This is a type of errors that application counts at Validation step when it checks threshold after validation. Second type is unexpected exceptions causing Process failure, e.g. software defects (bugs) or incorrect infrastructure configuration, etc. These errors are usually not recoverable and Process status is set to "Failed". Such errors should not happen and needs to be analysed and fixed. 

  • If the checks are performed and errors are raised at validation and the same errors are raised during activation step because app performs same checks as validation step before saving data into active area. If the Activation step is configured to activate data with errors into the change request.  In the provided scenario, an unexpected error (usually code or data related) occurs, causing the entire process to fail.

    If the goal of the test scenario to complete the Process and successfully activate only data that does not contain errors, user can use the "No Activation" target for the data having errors. 

    Activation step configuration customizing details can be found in the following documentation: Configure Activation for Business Partner 

Keywords

threshold, activation, validation, MDG, CMP, MDC

Symptom

  • Database error -117 for
    > dsql_db6_exec_immediate( SQLExecDirect ): [IBM][CLI
    > Driver][DB2/AIX64] SQL0117N The number of values assigned i
    > not the same as the number of specified or implied columns o
    > variables. LINE NUMBER=1. SQLSTATE=42802
  • While trying to add it as INDX table in LTRS, getting error "INDX Table unknown with STXL in <MTID> error with <The RFC connection>"
  • Error in LTRC if try to replicate the index table:
    The following error messages occurred on: 12/16/2019 14:15:50
    Failed to create triggers for table STXL with logging table /1CADMC/xxxxxxxx
    RFC destination: <The RFC connection>
    Database connection: R/3*

Environment

  • SAP Landscape Transformation Replication Server
  • DMIS 2010 SP10

Cause

It is not supported to replicate INDX-LIKE tables from a source on DMIS 2010 SP10

Resolution

SLT did offer a manual configuration for INDX like tables, but that also requires that the source is minimum DMIS 2011 SP09  (see note 2191214, where INDX replication was in pilot mode).

For the LTRS > INDX-like Table Settings to work the source and SLT needs to be at minimum DMIS 2011 SP13.

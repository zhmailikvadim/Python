Symptom

When generating the R-4099 event for EFD-REINF in Tax Declaration Framework (TDF), a duplicated R-4099 register is created.

This issue may be observed after attempting to change the end date (VIG_ATE) for a signatory in the organizational structure (transaction /TMF/ORGSRT01). The system may not update the /TMF/D_SIGNATAR table correctly, resulting in two signatories having an overlapping validity period.

Environment

Tax Declaration Framework (TDF) for Brazil

Cause

The issue is caused by an incorrect configuration in the Organizational Structure. When multiple signatories are defined for the same period, more than one signatory is incorrectly marked as the "EFD-Reinf Contact" in the Signatory Identification section of the CNPJ Root tab.

According to the standard system design, only one signatory should have this field checked at any given time.

Resolution

To resolve the issue, ensure that only one signatory is designated as the EFD-REINF contact within the organizational structure.

  1. Go to the Organizational Structure maintenance (transaction /TMF/ORGSRT01).
  2. In the CNPJ Root tab, navigate to the Signatory Identification section.
  3. Review the list of signatories. For the period in which the duplication occurs, ensure that only one signatory has the EFD-Reinf Contact checkbox selected.
  4. Deselect the EFD-Reinf Contact checkbox for any additional signatories that have an overlapping validity period.
  5. Save the changes.

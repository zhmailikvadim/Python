Symptom

  • An error FS229 occurs while making the  FI posting in accounting interface or pure FI transactions such as FB01, FBA2, FBA3, FBA6 , FBA7.
  • Error may also occur when creating a Down Payment or Down Payment request in FI.

Environment

  • Integration/Accounting Interface
  • SAP ERP
  • SAP enhancement package for SAP ERP
  • SAP S/4 Hana Cloud Private Edition
  • SAP S/4 Hana

Reproducing the Issue

  1. Attempt to make an account posting for Invoice in VF02 or any down payment FI transaction.
  2. Error message: "No VAT registration numbers are allowed for down payments/requests Message no. FS229."

Cause

The error occurs because corresponding tax code configured with EU Code "1" in the properties of FTXP. Tax codes with EU codes "1", "3", or "4" are output tax codes with a tax rate of 0% and are relevant for the EC sales list. Down payments and down payment requests must not be displayed in the EC sales list, which leads to the system raising the error message FS229 during account posting.

Resolution

  1. Verify the EU Code configuration for tax code in the properties of FTXP.
  2. If the tax code has EU code set to "1", "3", or "4", correct the EU Code to a valid value that is not relevant for the EC sales list.
  3. Alternatively, remove the EU Code from the tax code properties to prevent the error message FS229 from being raised during account posting for down payment requests/ down payments.
  4. Refer to SAP Note 2402710 for additional details on the EU Code field (EGRKZ).

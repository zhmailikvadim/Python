Symptom

  • After copying the system to a new test environment, several integration flows fail because object (Activity, Service, ect.) numbers (sequence code) already exists in the ERP data (e.g. in S4) .
  • This causes an overlap between the SAP Field Service Management activity sequence and the existing ERP activity identifiers, leading to conflicts that disrupt integration flows.

Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.

Environment

SAP Field Service Management

Cause

There is no possibility to change code sequence via UI.

Resolution

Update the Sequence via Data API.

  1. Create a Client in FSM admin

    1. Log into the Admin application of your account (https://{cluster}.fsm.cloud.sap/admin/)

    2. Click on Client on left sidebar

    3. Click on “Create” button

      Fig. 1: Clients screen in Admin

    4. Define:
      1. “client-id”
      2. select “Client-Authentication Method”,
      3. select the “User Policy Group”

        Fig. 2: Create a new Client in Admin


    5. Copy the “client-id” and “client-secret”

      Fig. 3: Client configuration in Admin 



  2. Encode Client Id and Client Secret
    1. Arrange “Client Id” and “Client Secret”, separated by a colon, into a single line of text like this:

      00017214-myTestClient:f4e23cf5-b2fe-4314-bc2b-fcbcc127220b

    2. Use any offline or online tool to encode this text into Base64 format. Make sure to not include any invisible newline characters at the end. 


      Fig. 4: Offline Editor (e.g. notepadd++) to decode/encode base64 contents 


    3. The result should look similar to this:

      Fig. 5: Authentication Basic header for the OAuth 2 API request



  3. Request OAuth 2.0 Token

    1. Copy the following command into a terminal application on your local machine and replace the {cluster} placeholder with the cluster of your account and the Base64 encoded value after Authentication Basic with your own value:

      CURL:
      curl --request POST \
        --url https://{{cluster}}.fsm.cloud.sap/api/oauth2/v2/token \
        --header 'authorization: Basic {{Encoded Client Id and Client Secret}}' \
       --header 'content-type: application/x-www-form-urlencoded' \
       --data grant_type=client_credentials



      Fig. 6: Headers information via 'Bruno' API Client 

      Method:     POST
      URL:     https://{cluster}.fsm.cloud.sap/api/oauth2/v2/token

      Headers:
         Authorization: Basic {{Encoded Client Id and Client Secret}}
         Content-Type: application/x-www-form-urlencoded

      Body:
      grant_type: client_credentials

    2. The response from the server should look similar to this:

      Fig. 7: Token response via 'Bruno' API Client 

    3. Copy the received token



  4. Retrieve the SEQUENCE
    1. FSM Admin -> Company -> Query API

    2. Perform the query:
      SELECT s.id,  s.currentValue, s.objectType FROM Sequence s;


      Fig. 8: Query API in Admin

    3. Copy the Sequence ID

    4. Open the HTTP client for API call

      Method:     GET
      URL:    https://{cluster}.fsm.cloud.sap/api/data/v4/Sequence/{Sequence-ID}?dtos=Sequence.{dtoVersion}

    5. Headers

      Fig. 9: URL request incl. ID and DTO version in the Headers information via 'Bruno' API Client 

      Authorization:     Bearer {{token}}
      X-Account-ID:    {{fsm-account-id}}
      X-Company-ID:    {{fsm-company-id}}
      X-Client-ID:    {{any}}
      X-Client-Version:    {{any}}

    6. Params

      Fig. 10: Parameters via 'Bruno' API client

      CURL:
      curl --request GET \
        --url 'https://de.fsm.cloud.sap/api/data/v4/Sequence/{Sequence-ID}?dtos=Sequence.{dtoVersion}' \
        --header 'authorization: Bearer {{token}}' \
        --header 'x-account-id: {{fsm-account-id}}' \
        --header 'x-client-id: {{any}}' \
        --header 'x-client-version: {{any}}' \
        --header 'x-company-id: {{fsm-company-id}}'

    7. perform the API request

      The result should looks as following:

      Fig. 11: Response via 'Bruno' API client

    8. The received data you can use for your next API request.


  5. Update the Sequence

    1. Method:     PUT / PATCH
      URL:           https://{cluster}.fsm.cloud.sap/api/data/v4/Sequence/{Sequence-ID}?dtos=Sequence.{dtoVersion}


    2. Headers

      Fig. 12: PATCH via 'Bruno' API client 


      Authorization:     Bearer {{token}}
      X-Account-ID:    {{fsm-account-id}}
      X-Company-ID:    {{fsm-company-id}}
      X-Client-ID:    {{any}}
      X-Client-Version:    {{any}}

    3. Params

      Fig. 13: Params at PATCH via 'Bruno' API client 

      forceUpdate:     true
      dtos:        {{dto}}.{{version}}


    4. Body – add fields with values you want to update 

      Fig. 14: Body at PATCH via 'Bruno' API client 

            {
              "currentValue": 300
            }

      CURL:

      curl --request PATCH \
        --url 'https://de.fsm.cloud.sap/api/data/v4/Sequence/{Sequence-ID}?dtos=Sequence.{dtoVersion}&forceUpdate=true' \
        --header 'authorization: Bearer {{token}}' \
        --header 'content-type: application/json' \
        --header 'x-account-id: {{fsm-account-id}}' \
        --header 'x-client-id: {{any}}' \
        --header 'x-client-version: {{any}}' \
        --header 'x-company-id: {{fsm-company-id}}' \
        --data '{
        "currentValue": 300
      }'



    5.  Perform the request


  6. Check the API response

    Expected response: currentValue": 300.


    Fig. 15: Successful response  via 'Bruno' API client

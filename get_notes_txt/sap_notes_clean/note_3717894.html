Symptom

Although your BAdI implementation for BAdI /SHCM/B_EP_BP_ROLE_SYNC returns in method IS_CUSTOMER_REQUIRED or IS_VENDOR_REQUIRED that for current processed personnel number no Customer/Vendor should be created the synchronization creates a Customer/Vendor or vice versa.

Other Terms

PTRV_VENDOR_SYNC, /SHCM/RECON, /SHCM/RECON_CUST, /SHCM/EE_BP_1,

Reason and Prerequisites

 The business function /SHCM/EE_BP_1 is active.

Solution

Implement the attached correction instructions or import the specified support package.
If you implement the fix via attached correction instructions for S4COREOP 107 - S4COREOP 109 you have to change the message text for message 053 /SHCM/EE_BP_S4_BL_RO as below described:
Execute transaction SE91 for message clas /SHCM/EE_BP_S4_BL_RO and change the text for message number 053 to 'Personnel Number (PERNR) &1 has Customer &2.' and save changes

The correction instruction for S4CORE 106 contains only ATC error corrections and no bug fixes.

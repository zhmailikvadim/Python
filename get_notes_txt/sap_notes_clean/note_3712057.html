Symptom

  • After upgrading from BO4.3 SP4 to SP5, the new buttons in the "Authorized URLs" section of the Web Intelligence application do not appear.
  • Install logs (setupengine.log) shows the temporary CMS started but deployfiles couldn't log in to it 

Environment

  • SAP BusinessObjects BI Platform 4.3 
  • SAP BusinessObjects Client Tools

Reproducing the Issue

  1. Open Central Management Console (CMC).
  2. Navigate to Applications.
  3. Select "Web Intelligence."
  4. Go to "Authorized URLs."

Cause

  • Deploy files failed during the upgrade process due to inability to log into the temporary CMS.
  • Side-by-side installation of SAP BusinessObjects BI Platform Server and its Clients on the same machine could have caused conflicts during the upgrade.

Resolution

  1. Uninstall the Client Tools from the server machine.
  2. Run a repair on the BI Server. ( 1803645 - Modify or repair a BI 4.x install, patch or update [How-To] - SAP for Me )
  3. Ensure the firewall/anti-virus is turned off during the repair process.

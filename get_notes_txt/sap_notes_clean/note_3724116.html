Symptom

  • A change was requested for one of the fields in info structure, after the change was made via MC22 and generation, the info structure stopped returning results when executed.
  • The field related to sales organization was removed, and the sales office field was inserted via MC22.

Environment

  • Sales And Distribution (SD)
  • Logistics Execution (LE)
  • SAP S/4HANA
  • SAP S/4HANA Cloud 
  • SAP ERP Central Component
  • SAP ERP
  • SAP Enhancement package for SAP ERP
  • SAP Enhancement package for SAP ERP, version for SAP HANA

Reproducing the Issue

  1. Remove the field related to sales organization.
  2. Insert the sales office field via MC22.
  3. Generate the structure.
  4. Execute the info structure.

Cause

If there is changes in the info structure, it needs to be generated and inserted the update.

Resolution

  1. Check the transport request which contains the change.
  2. Double-check that the request really contains the change (at least R3TR MCIN S*** on the object list)
  3. Look up the return code of the transport log of that request. If it have a RC <> 0 they might need to adapt the DB with the DB utility (SE14) for the new structure of the table.
  4. Repeat the import to get rid of the bad RC
  5. If that does not work for some reason (or the RC was already 0) , you can only set up the statistical data again External Knowledge Link for 0000064636.

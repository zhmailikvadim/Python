Symptom

A stored procedure appears to be running for an extended period, resulting in:

  • Performance degradation

  • Blocking situations

  • High CPU or I/O consumption

There is a need to determine which SQL statement is currently being executed.

Environment

SAP Adaptive Server Enterprise (ASE)

Cause

Standard monitoring commands in ASE only display session status and do not show the exact SQL statement currently being processed inside a stored procedure.

Resolution

 

1. Identify Active Sessions

sp_who

or

select spid, status, cpu, physical_io, cmd
from master..sysprocesses
where status != 'sleeping'


Identify the SPID of the active process.

2. Display the Currently Executing SQL Statement

dbcc traceon(3604)

go

dbcc sqltext(<spid>)

go 


This command displays the SQL statement currently being executed by the ASE engine.

It is particularly useful when a stored procedure is running and you need to determine which internal SQL statement is active at that moment.

Keywords

dbcc sqltext, running stored procedure,  SQL statement

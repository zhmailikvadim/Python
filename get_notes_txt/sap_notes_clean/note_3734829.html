Symptom

Due to the indexed perform (see Note 330389), there is only one restricted number interval for customer-specific functions and operations. This area is used up for some customers. You want to create the custom function OPXCALL_CUST or FUXCALL_CUST for your custom operations and functions in time evaluation.

Other Terms

PE04, time management, RPTIME00, FUNT, OPET, functions, operations, OPXCALL, FUXCALL

Reason and Prerequisites

This is a program error.

Solution

Implement the attached corrections instructions or install the related support package.

With regard to payroll, in Note 1324745, the number interval that was restricted by the indexed perform (see Note 330389) was extended for functions and operations with note 1619700 for time evaluation.  In transaction PE04, you specify the relevant subroutine FUXCALL for functions or OPXCALL for operations as "Name of user-defined form routine". Source code navigation jumps directly to your customer-specific form routine OPXCALL_CUST or FUXCALL_CUST in the customer include RPTMOZ00 (in the program SAPFP51T or SAPFP51T_V3S) if you are using the customer namespace for your functions. There are no country implementations of FUXCALL_NATIO for functions or OPXCALL_NATIO for operations available in time management.

Note that performance may suffer for the functions and operations created in this way since the indexed perform is now no longer used immediately for access. However, you should notice the effects only if the function is called very often in your schema. Use this new function therefore only if you find the number interval of the indexed perform inadequate.

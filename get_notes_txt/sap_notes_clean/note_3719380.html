Symptom

"Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental."

  • Error "Sorry, we've encountered an error during the account verification setup. Please try again later."  appear when trying to enable custom IdP in SAP Analytics Cloud (SAC)
  • Error "uploadIdPMetadata failed with 404 status"  can be found in HAR trace collected as per SAP KBA 2280022
  • This issue may also happen with SAP Business Data Cloud (BDC) and SAP Datasphere (DSP) due to same reason

Environment

  • SAP Analytics Cloud, Enterprise Edition
  • SAP Business Data Cloud
  • SAP Datasphere

Reproducing the Issue

  1. Configure Custom SAML IdP as per SAP Analytics Cloud guide: https://help.sap.com/doc/00f68c2e08b941f081002fd3691d86a7/release/en-US/3651184dad944aa2b361ad029a7a8cae.html
  2. Click Verify Account button.
    => Error "Sorry, we've encountered an error during the account verification setup. Please try again later." appears.

Cause

  • You should have started the bundling process though IDP Admin by following Create or Modify a Bundled SAP Cloud Identity Services Tenant so you cannot use the same IDP with legacy Self Service SAML

Resolution

  • If you wish to use legacy Self Service SAML, you must complete the bundling process on Identity Provider Administration or use a different IdP.

Symptom

A default filter is required so each logged-in user sees only tasks where they are the owner.

The default "My Tasks" returns tasks where the user is author/processor, organizer, or owner. There is no option to filter tasks by owner only in Activity Manager.

Also, restricting TaskService permissions to "Myself" does not change "My Tasks" behavior.

Environment

  • SAP Sales Cloud Version 2

Reproducing the Issue

  1. Log in as the affected user.
  2. Open Activity Manager.
  3. Navigate to Tasks.
  4. Apply the default filter "My Tasks".
  5. Observe that results include tasks where the user is author/processor, organizer, or owner, and that no owner-only filter is available.

Cause

By design, the standard "My Tasks" query includes tasks where the logged-in user is owner, processor/author, or is an involved party (for example, organizer). An owner-only default filter is not provided.

Resolution

  1. Go to Activity Manager.
  2. Open the Tasks view.
  3. Open Advanced Search.
  4. Set a condition for Owner equals current user.
  5. Save the query as a personal filter using Save > Save As, and provide a meaningful name.
  6. Use this saved filter whenever you need to see only tasks owned by the logged-in user.

Note: At the moment, this is the standard behavior across all activities. The standard "My Tasks" is intentionally broader to support additional use cases; personalize with saved filters as needed.

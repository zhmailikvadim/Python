Symptom

After upgrading to SAPJVM 8.1.108 or SAP Machine 21, connections to a peer that requires the older cipher suites listed below fail. 

TLS_RSA_WITH_AES_256_GCM_SHA384
TLS_RSA_WITH_AES_128_GCM_SHA256
TLS_RSA_WITH_AES_256_CBC_SHA256
TLS_RSA_WITH_AES_128_CBC_SHA256
TLS_RSA_WITH_AES_256_CBC_SHA
TLS_RSA_WITH_AES_128_CBC_SHA

Environment

SAP Cloud Connector 2.18 - 2.19

sapjvm 8.1.108

sap machine 21

Reproducing the Issue

Cause

TLS_RSA cipher suites are disabled by default in the updated JVM, so connections to systems that only support TLS_RSA_WITH_* fail due to cipher mismatch.

Resolution

  1. Locate the JVM installation used by the cloud connector.
  2. For Windows
    For SAPJVM Open the file: C:\Program Files\SAP JVM\sapjvm_8\jre\lib\security\java.security .
    For SAPMachine  Open the file: C:\Program Files\SapMachine\JDK\21\conf\security\java.security.
  3. Find the property jdk.tls.disabledAlgorithms.
  4. Remove the entry prefix TLS_RSA_ from the list to re-enable TLS_RSA cipher suites. Note: Re-enabling legacy cipher suites reduces security and should be done at the user's own risk.
  5. Save the file.
  6. Restart the cloud connector service.
  7. Re-test the connection to confirm the TLS_RSA_WITH_AES_128_GCM_SHA256 cipher is negotiated successfully.

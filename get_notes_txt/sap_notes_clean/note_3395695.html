Symptom

When login EDS portal with the correct username and password, you still see the error message "Incorrect Username Or Password".

Environment

You are working in an environment where Service Layer and Electronic Document Service are installed.

Reproducing the Issue

Prerequisites:

You are using Electronic Documents Service (EDS).

Scenario:

  1. Open EDS portal.
  2. Enter the correct username and password.
  3. Click the "Login" button.

Actual Result:

Logon to EDS fails and the error message "Incorrect Username Or Password" displays.

Check the SLClient-1.log from EDS log folder and you can see one of the following error messages:

"System.Net.Http.HttpRequestException: The SSL connection could not be established, see inner exception."

"System.Security.Authentication.AuthenticationException: The remote certificate was rejected by the provided RemoteCertificateValidationCallback."

"NotTimeValid: certificate has expired"

"The remote certificate has not been verified."

Expected Result:

Logon to EDS is successful.

Cause

The SLD certificate is not valid any more.

Resolution

To know more information about certificate, please check the B1 Admin guide, sections "Preparing Certificates for HTTPS Services" and "Security Certificate Verification During SSL Communication".

To obtain a functional self-signed cert, please check SAP Note 3537539 - SAP Business One requires a valid SSL certificate.

Use Reconfiguration option from Server Component Setup Wizard to update the SLD certificate.

Keywords

certificate, password, EDS, expired

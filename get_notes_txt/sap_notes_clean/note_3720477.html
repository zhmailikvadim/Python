Symptom

You are using EC-ERP replication using BIB framework and facing issue with the replication of non-effective dated entity to the infotypes.

This note handles two issues:

While replicating data from a non-effective dated custom portlet to an infotype, you want to determine the BEGDA of the infotype based on the last modified date in EC. But the C_LASTMODIF rule is not working as expected in the custom portlet .

While replicating data from a non-effective dated to an infotype, you want to determine the BEGDA of the infotype based on the payroll period start date. During the initial hire replication, infotype is getting replicated based on the payroll period start date and not based on hire date.

Other Terms

Non effective dated custom portlet

last modified date

C_LASTMODIF

C_PAYRDATE

payroll period start date with hire replication

Non effective dated entity

Reason and Prerequisites

Currently in the logic we are using lastModifiedOn field to determine the last modified date. But for the custom entity, the field will be lastModifiedDate. With this note we have added logic to consider this also.

Incase of payroll period start date based replication in infotypes during the initial hire replication, hire date should be considered instead of payroll period start date as BEGDA in infotypes.

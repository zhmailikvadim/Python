Symptom

  • In the Find Maintenance Notification (F2071) app, the "Change Responsibility" button shows a successful save message but does not assign the selected notifications to a person.
  • The person responsible does not appear in the notification despite the save confirmation.

Environment

  • SAP ERP Plant Maintenance (PM)
  • SAP S/4HANA, on-premise
  • SAP S/4HANA, Cloud Private Edition

Reproducing the Issue

  1. Open the Fiori App: Find Maintenance Notification (F2071).
  2. Select one or multiple notifications.
  3. Use the "Change Responsibility" button to assign the notifications to a person.
  4. Observe that the system shows a successful save message but does not perform the assignment.

Cause

The customizing settings for the Notification Partner Function, Approval, were incorrectly configured in the affected system.

Resolution

  1. Navigate to the following customizing path:
  2. SAP Customizing Implementation Guide -> Plant Maintenance and Customer Service -> Maintenance and Service Processing -> Maintenance and Service Notifications -> Overview of Notification Type -> Y1: Reactive Request type -> Partner Function, Approval.
  3. Set the "Person Responsible" under the "Partner Functions for Notification Processing" for the Person respons. value (VW).
  4. Move the updated customizing settings to the affected system.
  5. Retest the scenario to confirm the issue is resolved.

Symptom

  • Physical Options of tables are always generated despite of the Generation Options.
  • The Physical Options check box is not selected for Table & Column in the Database Generation Options window.
  • The Default Physical Options window does not contain any item.
  • This issue occurs with IBM DB2 for z/OS. It does not happen with DB2 MVS columns and indexes.
  • It works fine with other DBMS, e.g. SQL Anywhere, Oracle, Microsoft SQL Server, IBM DB2 for LUX, IBM DB2 UDB Common Server.

Environment

  • SAP PowerDesigner (PD) 16.6
  • IBM DB2 for z/OS

Reproducing the Issue

  1. Select File > New Model to open the New Model dialog.
  2. Select Physical Data Model in the left-hand pane.
  3. Select Physical Diagram in the right-hand pane.
  4. Select IBM DB2 Version 12 for z/OS in the list of DBMS's.
  5. Click OK.
  6. Click the Table tool from the toolbox to create one table. Default name is Table_1.
  7. Double-click it to edit the table.
  8. Click the Physical Options tab to add a few options, then click Apply.



  9. Click the Preview tab and click the Show Generation Options tool.
  10. Select the Table & Column entry in the left-hand pane, then deselect Physical Options under Create Table.



  11. Click OK.

     

Cause

The issue has been identified and logged under CR 811999.

Resolution

There is currently no target SAP PD 16.x Service Pack scheduled for this issue SAP CR 811999. It is being reviewed by our Product Group.

Symptom

Lean Service Entry Sheet (SES) created from external system using ServiceEntrySheetRequest_In web service fail returning error message "ActionCode &1 is not supported - 054 (MM_PUR_SES)".

Environment

  • Ariba Supplier Business Network (SBN)
  • Materials Management (MM)
  • SAP Fiori
  • SAP S/4HANA
  • SAP S/4HANA Cloud Private Edition

Reproducing the Issue

  1. Submit a Service Entry Sheet from external system to S/4HANA system
  2. In S/4HANA, monitor the inbound web service ServiceEntrySheetRequest_In in SRT_MONI
  3. Observe that the inbound message fails with the error "ActionCode &1 is not supported - 054 (MM_PUR_SES)"

Cause

The inbound ServiceEntrySheetRequest_In XML is either missing the ActionCode element required by S/4HANA or informing an invalid ActionCode element.

Resolution

As explained in the SAP Help, for Lean Services Entry Sheets, the parameter "ActionCode" needs to be filled with either value 01 for operation "Create" or value 03 for operation "Delete".

Service Entry Sheet - Create, Delete (Lean Services) | SAP Help Portal

If this inbound payload is coming from Supplier Business Network, this field is derived from the cXML tag "operation" sent by this external system.

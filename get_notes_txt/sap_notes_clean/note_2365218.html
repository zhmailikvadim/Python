Symptom

1, What is a quotation item?
2, What is pure quotation, sales order and mixed quotation order?
3, What are the restrictions of follow-up scenarios?

Environment

  • SAP Customer Relationship Management 7.0 and higher
  • SAP enhancement package for SAP Customer Relationship Management
  • SAP enhancement package for SAP CRM, version for SAP HANA

Resolution

1, What is a quotation item?

Navigate to [SPRO - Customer Relationship Management - Transactions - Basic Settings - Define Item Categories - Assignment of Business Transaction Categories - BUS2000115 Sales - Customizing Item].

"Relevance for Quotation" and "Subsequ. processing" shall be selected like below to make the item category a quotation item. Item category is considered as a sales order item category with other combinations.

Relevance for Quotation2 Initially a Quotation
Subsequ. processingX Quotation is copied to subsequent document

 

2, What is pure quotation, sales order and mixed quotation order?

It depends on the item categories assigned to the transaction type in [SPRO - Customer Relationship Management - Transactions - Basic Settings - Define Item Categories Determination].

Pure QuotationTransaction type is considered as a pure quotation when all the item categories assigned to it are quotation items.
Mixed Quotation OrderTransaction type is considered as a mixed quotation order when the item categories assigned are mixed (some are quotation items and others are sales order items). Mixed quotation order is considered as a sales order.
Sales OrderTransaction type is considered as a sales order when all the item categories assigned are sales order items.

The check is done in CL_CRM_UIU_BT_GET_PROCTYPES ~ IS_QUOTATION or CRM_QUOTATION_BEFORE_COPY_EC.

 

3, What are the restrictions of follow-up scenarios?

3.1 Quotation -> Quotation (Pure to Pure) and sales order.

System does not allow creating follow-up quotation or sales order from such completed quotation.

When all the items' status are set to completed in preceding quotation, the quotation header status is set to completed. When preceding quotation header status is set to completed, all the items are set to completed.
When creating follow-up quotation or sales order from such completed quotation, the system still raises transaction type selection popup and provides the follow-up quotation and sales order transaction types. It allows selecting the follow-up transaction type. When doing so, system check header status for I1005 completed, raises error CRM_QUOTATION018  Completed quotation cannot be referenced and abort follow-up. This check is done in CRM_QUOTATION_BEFORE_COPY_EC.
Solution: When the preceding quotation is completed, it means all the items in it has been fulfilled by follow-up sales orders. There is no need to create follow-up from this preceding quotation anymore. Otherwise, it will lead to a potential result that more products are sold to the customer than the agreed quotation. Thus, this is the designed system behavior valid for majority of the business cases. You may use BAdI CRM_COPY_BADI, in case you have any reason to manipulate this behavior.

In addition,

  • When the quotation header is set to release, it will be replicated to ECC. In this case, it is possible to create follow-up document but no item from preceding quotation can be referenced to.
  • When only some of the items are completed, the quotation header is not set to completed. In this case, it is possible to create follow-up document but the completed items cannot be referenced to anymore.

3.2 Quotation/any transaction type -> Mixed Quotation Order

System does not allow creating follow-up mixed quotation order regardless of the preceding quotation's status.

For any kind of the preceding transaction type. regardless of the preceding quotation's status on header /item, system always check if the follow-up transaction type is a mixed quotation order. Such transaction type will not be provided in the transaction type selection popup.
Solution: mixed quotation order shall not be used according to SAP Note 1299189 - Mixed documents Sales Order / Quotation in CRM UI. Please delete the quotation item categories or the sales item categories in item category determination for the mixed quotation order's transaction type. Please use pure quotation or sales order only.

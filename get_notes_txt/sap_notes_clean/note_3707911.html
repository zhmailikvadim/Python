Symptom

User settings and filters in F3951 Capacity Scheduling Board are lost when navigating between views or exiting the application.

Environment

  • SAP ERP Capacity Planning and Finite Sequencing (PP-CFS)
  • SAP R/3
  • SAP R/3 Enterprise
  • SAP ERP Central Component
  • SAP ERP
  • SAP Enhancement package for SAP ERP
  • SAP Enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA, on-premise
  • SAP S/4HANA Cloud Private Edition

Reproducing the Issue

1. Open the Fiori app Capacity Scheduling Board (F3951)
2. Select any evaluation horizon from the available options
3. Navigate to the settings configuration menu
4. Select the checkboxes for "show status bar," "order id," and "product id"
5. Press OK and Go to apply the selected settings
6. View the scheduling board to confirm all selected details are displayed
7. Choose any operation and select "manage operations" or exit the application
8. Return to the Capacity Scheduling Board app
9. Observe that information like order id and product id is lost through navigation and checkboxes must be set again in the manage operation view

Cause

 The issue arises due to the standard system design where certain fields ('Order ID', 'Product ID', and 'Work center group') are set to be not visible by default and cannot be saved as a variant.

Resolution

If you require these fields to be visible at all times and saved as a variant, this functionality would need to be implemented through a development request.
If you would like to be part of the SAP Customer Influence program and suggest this feature for the standard system, check the SAP Note 11 - Required function is not part of the standard system.  

Keywords

Symptom

  • OCC calls to CMS endpoints return 400 Bad Request.
  • Back end logs show: de.hybris.platform.cms2.exceptions.TypePermissionException: Unauthorized type access. Reason: User does not have permissions to read an instance of type MediaFormat.
  • The issue occurs when calling cms/pages (for example, CMS OCC endpoints); other endpoints like basesites work.
  • Reproduces for the anonymous session user.

Environment

  • Product: SAP Commerce Cloud, composable storefront
  • SAP Commerce Cloud Spartacus

Reproducing the Issue

  1. Remove read permission for anonymous user
  2. Observe a 400 Bad Request response for the cms/pages OCC call.
  3. Check the commerce logs and find TypePermissionException indicating lack of read permission for MediaFormat.

Cause

Missing read permission on type MediaFormat (and related CMS types) for the session principal (typically the anonymous user), leading to TypePermissionException during CMS OCC requests.

Resolution

  1. Review server logs to confirm TypePermissionException for MediaFormat during the failing OCC CMS call.
  2. Identify the session principal used for the request (often the anonymous user).
  3. Ensure the anonymous user is assigned to the appropriate group (for example, customergroup) per OCC user roles.
  4. Grant read permission on MediaFormat (and other required CMS types, as needed) to the relevant group via Backoffice permissions configuration or by importing a user rights impex.
  5. Retest the cms/pages OCC endpoint from the storefront and verify a successful response.

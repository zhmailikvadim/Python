Symptom

  • The unit is unable to display in the SAC Story table column header
  • The unit display as ## in the SAC Story when against the BW Query

Environment

  • SAP Analytics Cloud

Reproducing the Issue

  1. In BW query, create a measure defined by NDIV0().
  2. The NDIV0() will return 0 without Unit.
    Calculations on Key Figures with Mixed Currencies or Units | SAP Help Portal  
  3. Create a Model in SAC against this Query, note that the NDIV0() retrun 0 without unit.
  4. Create a Story base on this model.
  5. In the story table, select the Measure column, Styling->Display Unit/Currencies->Column.
    Note that the Unit will display as ## as there is unit for the 0 value.

Cause

  • This behavior is by design. 
  • Story Table cannot merge the Unit as the 0 value without unit from BW side.

Resolution

  • Create a Calculation Measure in SAC, set dedicate Unit then it works as expected.

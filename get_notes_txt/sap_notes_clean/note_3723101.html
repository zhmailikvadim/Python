Symptom

The duration of an existing activity without resource can not be changed in the liveCache.

Other Terms

liveCache LCAPPS Scheduler OM_ACT_SCHEDULE Dummy Activity Resource Mode Duration Start End Time

Reason and Prerequisites

The duration of a "dummy" activity that does not use a resource, i.e. does not have a mode, is derived from end time - start time when it is created in the liveCache. This duration can not be changed later on. When a new start and end time is specified in OM_ACT_SCHEDULE, the start time (in forward scheduling) or the end time (in backward scheduling) is adjusted as requested, but the duration remains the same. By default OM_ACT_SCHEDULE does not support the modification of the duration by adopting both times specified.

Solution

Implement this note to obtain an additional import parameter IV_CHANGE_DUMMY_DURATION for OM_ACT_SCHEDULE. When this parameter is set to true, OM_ACT_SCHEDULE derives a new duration from end time - start time for all activities without resource specified in import parameter IT_ACTIVITIES. For activities that use a resource the behavior remains unchanged.

Note that after deriving the new duration the liveCache still might move the activities without resource to a date other than specified, e.g. to fulfil relationships to other activities.

This new feature is supported as of liveCache build 101.25, see note 2074843. When using older liveCache versions the implementation of this note has no effect.

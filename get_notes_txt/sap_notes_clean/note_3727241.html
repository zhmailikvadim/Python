Symptom

When executing the SAML Assertion Validation policy (ValidateSAMLAssertion), the request fails during signature verification with a digital signature validation error. The following messages are observed in the error details/logs:

  • ValidateSAMLAssertion[validate-samel]: Digital Signature Validation Failed
  • error.cause.cause: Null KeyInfo object!
  • error.cause: cannot find validation key
  • type: ErrorPoint
  • state: PROXY_REQ_FLOW
  • error.class: com.apigee.kernel.exceptions.spi.UncheckedException

 

 

Environment

  • SAP Integration Suite
  • API Management

Reproducing the Issue

  1. Configure an API proxy (or shared flow) to use the ValidateSAMLAssertion policy
  2. Configure the policy to validate the SAML assertion signature using a specific TrustStore (certificate expected to be present there).
  3. The assertion signature includes <dsig:KeyName> within <dsig:KeyInfo>, which is used to identify the signing key.

Cause

The keyName element inside KeyInfo is not supported for SAML Assertion Validation in SAP API Management. 

Resolution

This behavior is documented in SAP Help under the SAML Assertion Policy notes, “Policy Processing” section.

Recommended resolution: please adjust the SAML assertion/signature so it does not rely on KeyInfo/KeyName for identifying the verification key, and instead uses a supported approach as described in the documentation.

Symptom

This document describes the steps to run a Cloud Integration iFlow in simulation mode

Environment

  • SAP Cloud Integration
  • SAP Integration Suite

Cause

Cloud Integration iFlow Simulation Mode is used for testing the iFlow in the Design tab (before deployment). It can be run when iFlow is in read mode as well as in edit mode. In Simulation mode execution calls never leave the tenant, so if you have any receiver channel in the iFlow they won't actually be called. The corresponding expected response from receiver has to be added during simulation to test it fully.

Resolution

  1. On the tenant's Design tab select the package and iFlow or artifact in it which is to be simulated.
  2. Click on the link from Start point to the first step of iFlow and select Start Simulation Point from menu as shown below. Note that the simulation can start at any point in the iFlow, not necessarily the first step.
  3. Click on the link between the last step and End Message and select Simulation Endpoint from the menu as shown below. The end of simulation can happen at any step after the start, does not need to be the last step of iFlow.
  4. On any receiver channels select "Add Simulation Response" in the menuas shown below and specify "expected" response by the receiver.

         

   5. On Simulation Start point add required Simulation Input such as headers, payload body by clicking the Start Simulation Point Button as shown below. It opens Add Simulation Input dialog box.

   6. Run the simulation by clicking run simulation button as shown below.

   7. After the Simulation ends click on envelop symbols to view payload at each point

Symptom

The retro calculation logic in payroll function HKCON is not as expected when meets below conditions:

  • Base wage type with PCL70 = 2 is used as MPF or ORSO contribution base(when retro, base difference will be brought forward);
  • There's an off-cycle correction run after main run in one payroll period(e.g April 2025).
  • In the next payroll period there's a retro to previous period (e.g Retro to April in May).


In this case , in payroll function HKCON of retro period of April main run, contribution amount are directly retrieved from ORT to IT.  This calculation logic is inconsistent with other retro calculation logic. When you have customizing rules after HKCON which changes the contribution amount, this logic may cause the incorrect result.

Other Terms

 HKCON; Retro

Reason and Prerequisites

This is a correction of legal function.

Solution

Apply the Support Package or implement the corresponding correction instructions.

Symptom

You're looking for guidelines regarding SAP HANA 2.0 SPS08 or higher with Linux transparent huge page (THP).

Environment

  • SAP HANA PLATFORM EDITION 2.0

Resolution

With SAP HANA >= 2.0 SPS 08 a Linux issue with transparent huge page (THP) defragmentation can significantly impact the performance of memory reclaims.

See SUSE knowledge article SAP HANA system hangs with THP on multi NUMA node systems and SAP Note 2131662 and disable THP defragmentation as a workaround if required:

echo "never" > /sys/kernel/mm/transparent_hugepage/defrag
echo 0 > /sys/kernel/mm/transparent_hugepage/khugepaged/defrag

While resource container shrink activities are documented in the database trace (SAP Note 2380176) with entries like "Information about shrink".

Self compactions (defragmentations) aren't displayed in the trace files. You can find details about reclaim activities in the system via SQL: "HANA_Memory_Reclaims" (SAP Note 1969700).

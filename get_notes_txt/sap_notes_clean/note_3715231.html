Symptom

Error message ACC_AA276  "CoCode xxxx: deactivation not possible as ledger yy is assigned in FI-AA"

arises when trying to deactivate any non-leading ledger in the configuration.

Environment

  • FIN Asset Accounting
  • SAP S/4HANA
  • SAP S/4HANA Finance
  • SAP S/4HANA Cloud Private Edition

Reproducing the Issue

  1. Attempt to deactivate a non-leading ledger in the configuration.
  2. Encounter the error message ACC_AA276  "CoCode xxxx: deactivation not possible as ledger yy is assigned in FI-AA"

Cause

If a ledger is a representative ledger in a ledger group and this ledger group is assigned to a depreciation area in a chart of depreciation, which is assigned to a company code, the ledger can not be deactivated for the company code.

Fixed Asset Accounting cannot deactivate a online posting depreciation area subsequently" - this is in accordance to the information in SAP Note 2722809 which introduced the error ACC_AA276 to stop such an attempt. 

Resolution

In SAP Standard it is not possible to deactivate an existing non-leading ledger and re-activate it again in future.


The SAP Note 1565231 provides a program as well to check the ledger groups a de-activated  ledger is used in and the risk of these ledger groups a de-activated ledger is used in and the risk of these ledger groups in the customizing for generating values. If such issue is reported, please check this customizing if it needs to be adopted (e.g. depreciation for a valuation area assigned to the de-activated ledger). See SAP Note 2722809 for more details.


In Asset Accounting, deleting those areas is a task for the local SAP consultant. Please pay attention to SAP Note 2034119 about a potential issue related to deactivation of a ledger.

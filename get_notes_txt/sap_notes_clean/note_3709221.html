Symptom

When updating/deleting a large number of line items from a service/purchase order the following error message appears: DP_SOFTCANCEL_RESET_AFTER_ERROR 

Updating/Deletion of individual or small batches of line items is possible, but using the "Select All" button to delete all line items results in the error.

In the work process trace file (dev_w*) :

M  ThSoftCancel: delete <terminal id> because of soft cancel with reason DP_SOFTCANCEL_RESET_AFTER_ERROR
M  ThPerformDeleteThisSession: switch off statistics
...
M  *** WARNING => ThResetInThJHandleVmStrategy: allow recursion [thxxjava.c   7408]
..
 M  ***LOG R47=> ThResFreeSession, delete () [thxxmode.c   1166] 

Environment

  • SAP Netweaver 
  • ABAP Platform

Virtual Machine Container (VMC) is active


Cause

Insufficient quotas for Virtual Machine Container (VMC) session PRIV modes and session size settings for Java sessions.

Resolution

  • Increase the quotas for VMC session PRIV modes by adjusting the following parameters:

    "rdisp/max_vm_priv_no" 
    "rdisp/max_vm_priv_time"

  • Adjust the session size for Java sessions by modifying the parameter:
    "vmcj/property/SESSION.MAX.SIZE"

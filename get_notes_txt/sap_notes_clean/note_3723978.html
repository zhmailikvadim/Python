Symptom

Some issues occur during the execution of Fiori APP F6868. For example,

  • Workflow user occupies ongoing memory in SM04
  • Lock entry remains in SM12 for the workflow user
  • RFC transaction for job execution user continues in SM58

Environment

  • Value Chain Monitoring Framework

Reproducing the Issue

  1. Open the Fiori application F6868 "Schedule Resolution of Value Chain".
  2. Create a job using the application with user.
  3. Execute the job.
  4. Observe the following:
  • Workflow user occupies ongoing memory in SM04.
  • Workflow user lock entry remains in SM12.
  • Transaction RFC for job execution user continues in SM58.
  • Sales orders are locked, and no goods issue confirmation occurs.

Cause

The Fiori APP F6868 is mainly developed for public cloud, because public cloud customers cannot schedule VCM_RESOLVE in their backend system since they do not have access to it. Technically, it offers the same functionality as scheduling VCM_RESOLVE with SM37 and a corresponding variant to select the relevant value chains.

Resolution

Please confirm if the same issue occurs during the background job for VCM_RESOLVE in SE37.

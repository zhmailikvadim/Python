Symptom

You are using the new tank planning functions in PP/DS in SAP S/4HANA. When using transaction (/SAPAPO/STORDEF) to maintain the so called Product Storage Definition (PSD) dumps of type CX_CSP_ACT_RESPONSE occur. These dumps are wrongly triggered by the application when some kind of data inconsistency is detected at runtime.

Other Terms

Tank planning, Advanced Planning, PP/DS, /SAPAPO/CL_PSD_UI_MANAGER, CONVERT_DB_ATTR_TO_UI_ATTR

Reason and Prerequisites

The occurring dumps are caused by inconsistent data constellations where the original root cause for how the can be created is still not exactly identified.

Solution

Implement the attached correction instructions or import the relevant support package for your release.

The changes provided with this SAP Note ensure that the user is still informed about the existence of inconsistencies. The user sees the recommendation to create a support ticket to report the situation to SAP for further analysis. However, after implementing the code change, the user gets this information via a popup. By confirming this dialog box, the user can continue and navigate to PSD maintenance and work with existing data that is displayed on UI screens. Inconsistent data is filtered out in advance and is not displayed.

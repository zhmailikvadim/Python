Symptom

You use S4HANA Group Reporting, release higher than S4CORE 103 (OP 1809).

Application "Manage Global Hierarchies" does not load Importing file and shows error message: "Subnode exceeds xx characters".

For example you are loading Consolidation Financial Statement Item hierarchy, and the error message informs you about exceeding 10 characters for a subnode.

It could also happen that the content of hierarchy is not getting loaded within the app.

Other Terms

F7995, F2918, UHDT_CATG, exceeded length of node IDs

Reason and Prerequisites

You upgraded from lower release than S4CORE 104 (OP 1909).

The lengths of Node IDs were changed to meet requirements.

On lower releases, it was possible to specify Node IDs using longer lengths of strings.

Solution

Starting OP1909, you need to ensure that Node IDs are respecting the maximum hierarchy ID length.

The easiest way to perform the changes is an import of respective MD - Excel import.

Please note, higher releases allow to use more hierarchy types, so not every entity from following list is part of OP 1909 e.g.

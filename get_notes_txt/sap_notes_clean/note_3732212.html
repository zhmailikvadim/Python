Symptom

During closeout claim creation, the return field should retain its value to ensure correct amount determination.However, the return field is getting cleared during the process, which leads to incorrect amount calculation.

This note will fix the issue so that the return field is retained and the amount is determined correctly during closeout claim creation.

Other Terms

 SHKZG, Return field, 2 customers,/IRM/GCR_CREATE, /IRM/IPBBA23, amount determination, incorrect calculation

Reason and Prerequisites

This is caused by program error.

SAP Note #3732222 is available for ECC Releases.

Solution

Please implement the correction instructions attached. These corrections are only valid if you have applied the Support Packs.

Release 6.4 Base Package or Support pack 1, 6.3 Base Package.

Use transaction SNOTE to implement the correction instructions specified below and activate the affected program object.

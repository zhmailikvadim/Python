Symptom

  • SFTP receiver channel configured with skipRemoteDirCheck attribute.
  • Despite configuration, PI/PO continues to perform remote directory checks.
  • Every file transfer triggers the error: "Error: Failed to stat remote directory".
  • File transfers fail systematically.

Environment

Secure File Transfer Protocol Adapter

Reproducing the Issue

  1. Configure the SFTP receiver channel with the skipRemoteDirCheck attribute.
  2. Attempt to transfer files via the channel.
  3. Observe that the error "Error: Failed to stat remote directory" is triggered during file transfer.

Cause

The SFTP adapter continues to perform the directory check because the property skipRemoteDirCheck is either not placed under the correct module, not correctly applied, or ignored due to an adapter engine cache.

Resolution

  1. Verify that the skipRemoteDirCheck attribute is correctly configured in the SFTP receiver channel.
  2. Ensure the attribute is placed under the correct module in the channel configuration.
  3. Clear the adapter engine cache to ensure the updated configuration is applied.
  4. Monitor the channel after implementing the changes to confirm successful file transfers.

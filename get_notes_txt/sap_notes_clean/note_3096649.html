Symptom

Objects are incorrectly resized in Portal using the Same size tool.
The selected symbols do not get the width of the widest selected symbol nor the height of the tallest selected symbol.
Instead, the format captured by Ctrl+Shift+S is applied.

Environment

SAP PDes 16.7.4

Reproducing the Issue

  • Create an Object A and B in a EA model.
  • Resize Object A.
  • Pick up the size of Object A with keys CTRL + Shift + S .
  • Apply size on Object B with CTRL + Alt + S .
  • Both have now same size.
  • Add Object C and D.
  • Change size on Object C.
  • Select Object C and D.
  • Use the tool in the toolbar to make Object D same size as Object C.
  • ObjectS C and D instead gets the same size as stored so they will get the same size as Object A and B.

Cause

The issue has been identified and logged under SAP CR 825505. Its description reads,"Align Tools > Same Size applies wrong width and height".

Resolution

There is currently no target SAP IQ Service Pack scheduled for SAP CR 825505. It is being reviewed by our Product Group. Please mark this KBA as a Favorite to be automatically notified when the CR status is changed.

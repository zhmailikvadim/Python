Symptom

Data mismatch when WebI is migrated from BOXI 3.1 SP5 to BI 4.1 SP3

Other Terms

Environment:

SAP BI 4.1 SP2 Patch1

Steps to Reproduce:

1. Import the WebI document from BIAR into BI4.1 SP3/SP4/SP2+Patches
2. Open the imported WID from WebI Rich Client or BILaunchpad
3. Goto second page and observe that there is a data mismatch after decimal points for columns.
4. The data changes again when doc is refreshed.
5. The issue is seen when we do Live-to-Live or BIAR-to-Live or directly open the XI3.1 WID in BI4.1 environment.
6. The change is seen even when we import the same document into XI3.1 SP6 as well.
7. Tested and replicated issue in BI4.1 SP2 Patch6, BI4.1 SP3 Patch1 and BI4.1 SP4 (Internal build).

Reason and Prerequisites

The issue has been identified and logged under Problem Report ID ADAPT01727146

Solution

By design : this is a problem of calculation precision. Aggregation are not garanteed to be done excatly in same order between version which can lead to a different rounding error.

Symptom

When creating a new job information record via the MSS Edit or History action on the Job Information card in the Latest People Profile, with new option Enable the latest Editing and History UI for Employee Central entities on full Profile active, the list of available events is not filtered correctly to only show events for which the user have event reasons with Edit/view permission.

Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.

Environment

  • SAP SuccessFactors HCM Suite
  • SAP SuccessFactors Employee Central

Reproducing the Issue

  1. With the Enable the latest Editing and History UI for Employee Central entities on full Profile option enabled in Configure Latest People Profile
  2. Go to the Employee Profile > Job Data
  3. Click on Edit (pencil icon) in Job Information card
  4. See that some Events appear in the dropdown list, even though the login user does not have permission to view or edit any of the Event Reasons associated with such Event.

Cause

Restriction in the 2H2025 code.

Resolution

The fix for this issue is planned to be deployed on 1H 2026 release.

For release timelines, please review SAP SuccessFactors Product Release & Road Map Information.

Workaround

Disable the 'Enable the latest Editing and History UI for Employee Central entities on full Profile' option from Configure Latest People Profile and only the Events which user has permission to view/edit the Event Reasons should appear.

Symptom

  • Messages consumed from Kafka are reprocessed after a failure occurs.
  • Successfully processed messages are replayed together with failed messages.
  • Kafka offsets appear not to be committed when an error occurs during processing.
  • Duplicate messages or duplicate business documents may be created in downstream systems.

Environment

  • Cloud Integration
  • SAP Integration Suite

Cause

SAP Cloud Integration Kafka adapter uses a fixed automatic offset commit strategy. Offsets are auto-committed every 5 seconds, independent of batch size or message count. When a FAILED message occurs during processing:

  • The Kafka client disconnects and reconnects to the broker.
  • No offset is committed before the reconnect.

As a result, all messages processed after the last successful auto-commit are replayed.

Resolution

This is standard behavior of the SAP Kafka adapter and does not indicate a product defect. The adapter does not support manual offset control or selective commits per message. Available options to address the scenario:

  • Implement a dead-letter-like pattern by catching errors and forwarding failed messages to a separate Kafka topic.
  • Re-evaluate whether a third-party Kafka adapter better fits the required offset handling behavior.
  • Consider using the Adapter Development Kit (ADK) to build a custom adapter with specific offset management capabilities.
  • Submit and vote for enhancement requests via the SAP Influence Program for future adapter features. See 3078373 - How to Submit Ideas for Cloud Integration or SAP Integration Suite.

See Also

This standard behavior is confirmed in the following SAP documentation:

  • Cloud Integration – What You Need to Know About the Kafka Adapter, which states that offsets are committed automatically every 5 seconds and may be replayed if a rebalance or reconnect occurs.
  • KBA 3638908 – Kafka sender adapter reprocessing successfully completed messages in Cloud Integration, which explains that offsets are not committed if the consumer is removed from the consumer group, resulting in message reprocessing.

Symptom

You previously relied on transaction codes OA85 and OAYL to define weighting information in the system—for example, when allocating yearly depreciation according to a 4-4-5 pattern.

However, in UPA, these transaction codes no longer function as they did before.

Other Terms

Universal Parallel Accounting, Fiscal Year Variant, 4-4-5

Reason and Prerequisites

In UPA, Asset Accounting is built on a new architecture with redesigned database structures. As a result, those transaction codes that depended on the old database tables are no longer compatible and therefore no longer function as they did before.

Solution

In UPA, it is recommended to maintain weighting information using the view cluster FINSVC_T009. Within the view “Period Weighting (Fixed Assets)”, you can enter the exact same weighting details that you previously maintained in OA85.

However, enabling or disabling weighting at the level of individual depreciation areas is not supported. Instead, this must be controlled at the ledger level.

For example, if you want to apply weighted depreciation only to ledger 2L (your local GAAP ledger), proceed as follows:

  1. Create a separate fiscal year variant (FYV), such as W4, using transaction OB29.
  2. In transaction FINSC_LEDGER, select ledger 2L, navigate to your company code, and assign FYV W4.
  3. In the view cluster FINSVC_T009, maintain the weighting information only for FYV W4.

Once these steps are completed, the system will distribute annual depreciation according to the specified weighting for ledger 2L.

Please note that changes in FINSC_LEDGER are critical configuration activities. You may only perform them when the company code has no existing postings.

Symptom

  • Users are unable to retrieve a saved layout/view in Output Items within the Manage Work Packs app.

  • After customizing the layout (adding/removing fields) and saving it using “Save As”, the saved view does not appear in the dropdown list.

  • The saved variant is not visible when reopening the view selection menu.

  • The issue persists even after repeating the save process.

Other Terms

F6065, Manage Work Pack , Output Management.

Reason and Prerequisites

In the Manage Work Packs application, when navigating through work packs and job packs to the Output Items section, users can modify the layout using the settings. Although the system allows saving the customized layout using the “Save As” functionality, the saved view is not listed afterward in the available views dropdown.

This indicates that the layout variant is either:

  • Not being persisted correctly in the backend, or

  • Not being retrieved/displayed properly in the UI layer after saving.

The issue affects usability, as users cannot reuse their customized layouts and must repeatedly configure fields manually.

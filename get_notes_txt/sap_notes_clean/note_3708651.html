Symptom

  • Full data backups of SAP HANA are executed daily or regularly although no backup schedules are configured in SAP HANA Cockpit or DB13.
  • All HANA Cockpit–created backup schedules have been deleted and verified as removed.
  • M_SCHEDULER_JOBS shows no backup jobs with status SCHEDULED, only historical execution entries with status SUCCESS.
  • Customers are unable to identify the trigger source and assume an internal SAP HANA scheduling issue.

Environment

SAP HANA DB Version 2.0

Cause

According to the documented SAP HANA backup scheduling model, recurring data backups are scheduled through SAP HANA cockpit or DBA Planning Calendar (DB13), or they are triggered externally, for example by third-party backup software via Backint or by customer-side automation executing backup commands.

If no active backup schedule exists in SAP HANA cockpit or DB13, this scenario does not indicate a documented SAP HANA internal recurring scheduler for the observed full data backups. In such cases, an external trigger source must be verified.

In such scenarios, typical external trigger sources may include:

  • Customer-side operating system scheduling
  • Third-party backup software using the Backint interface
  • Enterprise scheduling or automation tools
  • Custom scripts or automation triggering backups via SQL commands 

SAP HANA executes the received backup request, while the scheduling logic is managed outside of the SAP HANA database.

Resolution

To conclusively identify and stop the source of recurring full data backups, proceed as follows.

First, exclude SAP HANA internal scheduling.
Verify that no backup schedules exist in SAP HANA cockpit or DB13. Check M_SCHEDULER_JOBS and confirm that no backup-related jobs are currently scheduled. Entries with status SUCCESS represent completed historical executions and do not by themselves indicate an active future schedule. If required, remove any Cockpit-created backup schedules as described in SAP KBA 3495556.

Second, identify the actual trigger using the SAP HANA backup catalog.
From the next execution of a full data backup, analyze M_BACKUP_CATALOG and M_BACKUP_CATALOG_FILES. Evaluate at least START_TIME, USER_NAME, and APPLICATION_NAME from M_BACKUP_CATALOG, and SOURCE_TYPE_NAME from M_BACKUP_CATALOG_FILES. This information helps correlate the backup execution with the responsible tool or trigger source.

Third, verify external trigger sources.
Check whether a third-party backup tool using the Backint interface is installed or configured. Review customer-side operating-system scheduling, enterprise scheduling, or custom automation that may execute backup commands. Correlate the backup start time with these external trigger sources. If required, also review backup.log and, when Backint is used, backint.log.

Once the trigger source is identified, the recurring full data backups must be adjusted or disabled at that source. Based on this scenario, no SAP HANA defect is indicated.

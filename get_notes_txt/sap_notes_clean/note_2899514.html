Symptom

When reverse engineering a Teradata V16 database via odbc or jdbc, the datatype TIME(6) changes to AT.

Environment

  • SAP PowerDesigner (PD) 16.6 SP10
  • Teradata 16

Cause

The issue has been identified and logged under CR 821199. Its description reads, "Teradata reverse engineer TIME(6) datatype reverses as AT"

Resolution

There is currently no target SAP PD 16.x Service Pack scheduled for this issue SAP CR 821199. It is being assigned to our Product Group. Please mark this KBA as a Favorite to be automatically notified when the CR status is changed.

See Also

KBA 2171560 about how to be notified of new or updated SAP Notes or KBAs in the ONE Support Launchpad.

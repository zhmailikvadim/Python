Symptom

When connecting to the HTML5 web client using SAML authentication, certain strings—such as the "Reference period" variable in report bundles and the column header names in the printed controls grid—are not displayed in the application language selected in the HTML5 web client.

This issue occurs only when SAML authentication is used.

Other Terms

BFC, BOFC, wrong, printout, localization, translation

Reason and Prerequisites

Steps to reproduce

  1. Connect to the HTML5 web client using SAML authentication, and select an application language during logon.
  2. Open the Report Bundles tile.
  3. Open a report bundle that includes "Reference period" as a variable.

Result 1: The "Reference period" variable is not displayed in the selected application language.

  1. Open the Packages tile.
  2. Open a package.
  3. Select the CONTROLS tab.
  4. Print the package.

Result 2: The column header names in the printed controls grid are not displayed in the selected application language.

Solution

Update your SAP Financial Consolidation server installation to a support package, patch or release where the issue is fixed.

See the Support Packages & Patches section for details and available patches.

Symptom

You are using SAP Solution Manager 7.2 with Focused Build (ST-OST)
In Focused Build Test Suite Dashboard - Test Step Progress Monitor you get error message 'Exception raised without specific error'
This may happen after implementing collective note for Test Suite Dashboard.

Environment

SAP Solution Manager 7.2 with Focused Build

Reproducing the Issue

  1. Access the Focused Build Test Suite Dashboard (Fiori-based).
  2. Navigate to the Test Step Progress Monitor view within the Overview tab.
  3. Observe the exception error displayed on the dashboard.

Cause

Database inconsistency HDB
In SM21 log you may see error message like:
Database error 266 with PRE access to table /SALM/TMTPLNROOT
The error text is: inconsistent datatype: VARBINARY type is not comparable with BIGINT type.: line 1 col 1461 (at pos 1460).   

Resolution

Refer to SAP Note 3665968 for the Solution of your DB Level.

Afterwards regenerate the view - /SALM/TMTSEXLR


1. Go to transaction Code SE38/SA38 and execute report - RUTDDLSACT
2. Pass /SALM/TMTSEXLR as DDL Source or view name in the selection screen and execute the report
Above steps will regenerate the view and should fix the reported issue.

Then run SE38 report  /UI5/APP_INDEX_CALCULATE - select Full Calculation - hit enter 2x to skip the warnings and execute.

Clear all caches by:
- run tx /n/IWFND/CACHE_CLEANUP
- run tx /n/IWBEP/CACHE_CLEANUP

Check Test Step Progress Monitor again.

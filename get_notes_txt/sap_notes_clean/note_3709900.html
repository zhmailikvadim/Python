Symptom

In the processes "Triggering of Device Change on SUP" (DE_B_MDC_TRG_DEV_CHANGE_SUP) and "Triggering of Device Change on DSO View" (DE_B_MDC_TRG_DEV_CHANGE_DSO), there is no extension point to add a custom wait step between step 0200 and step 0300.

This issue is relevant for the market roles: SUP, DSO.

Other Terms

 IMS, TAF7, TAF1, MeLo, MaLo, UTILMD, device change

Reason and Prerequisites

 The error occurred because the standard TAF7 check expects both OBIS values to be present in the MeLo UTILMD message. When partners split the OBIS values between MeLo and MaLo, the check fails, causing the system to fall back to TAF1.

Solution

Implement the corrections of this SAP Note or import the relevant support package.

After implementing this SAP Note, the system adds extension steps to the process in step 0210, allowing you to enhance the process to handle split OBIS scenarios.

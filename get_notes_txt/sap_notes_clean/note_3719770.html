Symptom

When creating or editing an integration endpoint in the Configuration Manager, the dropdown menu to select an SSH Key is missing. Consequently, users can only manually paste a public key, but cannot link the mandatory private key, causing the endpoint to remain in an "Inactive" state despite appearing "Active" in list views.

Cause

This behavior occurs when the specific system-level variable (e.g., SAMS_SSHKEY_ENABLED_COMPANIES) is not enabled for the company code in that particular environment. Without this system-level enablement, the UI does not render the selection tool required to bind managed SSH keys to an integration asset.

Resolution

This issue cannot be resolved via self-service and requires backend enablement by SAP Fieldglass Support.

  1. Verify the UI: Confirm that the SSH Key dropdown is indeed missing from the Maintain DocuSign Endpoints or standard SFTP Endpoint configuration pages.
  2. Contact Support: Open a support case with SAP Fieldglass Integration Support.
  3. Required Information: Provide the following details in your case:
    • Company Code.
    • Environment (e.g., Test/COMP, Production).
    • Endpoint Name impacted.
  4. Backend Enablement: SAP Support will update the system-level environment variables to enable the SSH selection feature for your company.
  5. Re-configure: Once Support confirms enablement, return to the Configuration Manager, select the key from the now-visible dropdown, and test the connectivity.

Symptom

There is a business/project requirement that will result in a task being created on a service with the Task Engine disabled.

Environment

  • SAP Commerce
  • SAP Commerce Cloud

Cause

Further understanding of the SAP Commerce workflow for handling the affected task is required.

Resolution

It depends on the configuration specified for nodeId and nodeGroup. When a task is created on a node where the Task Engine is disabled, the entry is still visible to the services responsible for processing it (e.g., Background Processing and Backoffice).

However, if a nodeId is configured for the task, the entry might be lost if that node is unavailable during a particular timeframe, which can result in the task being stranded. Once the responsible node returns, the task will not be picked up afterward. 

Please refer to the AuxiliaryTablesBasedTaskProvider Strategy for Fetching Tasks and Advanced Concepts documentations for further informations on this topic.

Symptom

  • An issue occurs when attempting to send the "birthDate" field from Emarsys to SAP Customer Data Platform.
  • The webhook discards the event and returns the error message: "string 1980-05-03 does not validate against format 'date-time'."
  • The "birthDate" field in the webhook model is automatically created as "Date," and no transformation is allowed during mapping.
  • Testing with the field created as "String" results in the contact arriving without the date field populated.

Environment

SAP Customer Data Platform

Reproducing the Issue

  1. Update the "birthDate" field in Emarsys to "1980-05-03."
  2. Attempt to ingest the updated field into SAP Customer Data Platform via the webhook.
  3. Observe the webhook discarding the event and returning the error message: "string 1980-05-03 does not validate against format 'date-time'."

Cause

Resolution

  1. Define the "birthDate" field as a string in the webhook model.
  2. Add a string-to-date transformation for mapping the "birthDate" field.
  3. Verify the configuration and test the ingestion process to ensure the issue is resolved.

See Also

Keywords

SAP Customer Data Platform, string-to-date transformation, date-time validation error

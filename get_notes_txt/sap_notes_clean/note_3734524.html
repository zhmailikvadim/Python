Symptom

  • No policy found for audit area xxxx of ILM object (LRM_RULE_EXEC 311) occurs when running the archive write for ILM object.

Environment

  • SAP Business Technology Platform
  • SAP Basis release independent

Cause

An ILM policy for the audit area assigned to ILM object is not available in “Live” status in IRMPOL.

Resolution

  1. In transaction ILMARA, verify that ILM object is assigned to the intended audit area.
  2. If the assignment is not required, set the ILM object assignment to the audit area to “Inactive” and save, then test the archive write again.
  3. If the assignment is required, open transaction IRMPOL and check for an ILM policy for the same audit area and ILM object.
  4. If a policy exists but is not active, set it to “Live”. If no suitable policy exists, create a new policy with appropriate rules for ILM object and activate it (“Live”).
  5. Save the changes and run the archive write job for ILM object again to confirm the error no longer appears.

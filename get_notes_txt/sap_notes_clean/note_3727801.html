Symptom

  • Address section is displayed in the Add New Employee tool even though the mapping from recruiting has been removed
  • The Country Address Type field is prepopulated

Environment

SAP SuccessFactors Employee Central

Reproducing the Issue

  1. Log in to the instance
  2. Navigate to the Add New Employee tool
  3. Initiate the hiring process for a user
  4. Go until Address Information section
  5. Observe that the Address Type and Country fields are automatically filled out, and all address fields are displayed by default.

Cause

Resolution

It is expected behavior

The 'Address Type' field and 'Country' are automatically set in the hiring process within the Addresses section.

Symptom

In 2211.49, two new properties have been introduced to improve performance by optimizing connection pool usage and reducing database queries.

"Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental."

Environment

SAP Commerce Cloud

Resolution

Two new properties have been introduced to improve performance by optimizing connection pool usage and reducing database queries:

  • serialnumber.use.internal.datasource - isolates serial number generation database traffic using a dedicated connection pool. When set to true, number-series operations, such as reads, writes, and initialization, use a dedicated internal data source instead of the tenant master data source. This prevents contention and reduces the risk of number generation operations waiting for available connections when the master pool is exhausted.

    You can configure the dedicated pool size using the following properties:

    • numberseries.db.pool.maxActive - Maximum active connections, with the default value of 5

    • numberseries.db.pool.maxIdle - Maximum idle connections, with the default value of 5

  • servicelayerutils.cache.tenant.initialization - caches results from ServicelayerUtils.isSystemInitialized() calls instead of querying the Jalo connection on every invocation. When set to true, this reduces database load, lowers overhead, and improves latency when initialization checks occur frequently in your application's hot path. The cache has a time-to-live of approximately 6 seconds.

Please check following help document for more details regarding these two parameters.

Symptom

Changes in SICF intended to disable a specific authentication method does not take effect.

Environment

  • ABAP Web Services
  • SAP NetWeaver
  • SAP NetWeaver Application Server for SAP S/4HANA
  • ABAP PLATFORM - Application Server ABAP

Reproducing the Issue

  1. Enable an Authentication Method or Logon Procedure in SICF,
  2. Access the the ICF service,
  3. Observe that the request succeeds with a different Authentication Method.

Cause

SICF services inherit the logon procedure settings from their parent nodes by default, unless the "Ignore inherited setting" checkbox is enabled for the specific service.

Resolution

To maintain a specific Logon Procedure for an ICF service:

  1. Open transaction SICF,
  2. Locate the target service by entering its Service Name in the search,
  3. Open the service and go to the “Service Data” tab,
  4. In “Service Options”, enable the “Ignore inherited setting” checkbox to stop inheriting the parent node’s logon procedure,
  5. Switch to the “Logon Data” tab and configure the Logon Procedure as required for the service (for example, set to Alternative and remove Basic Authentication so that only the desired method remains),
  6. Save the changes.

Optional: From the Service menu, choose “Display Inheritance” to verify the service no longer inherits logon settings from parent nodes.

Symptom

In Open SQL, a select from an internal table or buffered table yields wrong results. 
Typically a filter condition on a CHAR or NUMC or similar column is involved.

Other Terms

SSE2, AVX, SQLEngine, jit-compiler , CHAR comparison

Reason and Prerequisites

The problem does not usually appear as an optimization targeting an old cpu vectorization capabilty is affected. Typically on newer hardware different vectorization instructions are applied. 
The SAP Kernel runs on a very old  x86_64 CPU or a virtualization where the effective CPU capabilities are hidden by the virtualization, so that the ABAP Kernel assumes only SSE2 optimization can be done. 
The problem may only appear on some queries and may be dependend on the OS, as it depends on register choice of the jit engine. 

The problem disappears if the dynamic profile parameter rsdb/tbi_jit is set to the value OFF.
Note: This should be only used for problem analysis and the default setting ON should be used for productive usage.
The problem disappears on virtualized hardware if the CPU vectorization is configured to use AVX or similar.

Solution

Install the kernel patch.
The kernel patch can be applied irrespective of the specific hardware or vectorization used in your system. As the flawed vectorization is not chosen, it has no effect then.

After installing the kernel patch, illegal values may be treated differently (e.g sorted differntly). 
In a future release the operations will fail. Thus it is of paramount importance that the source of these values is eliminated in the SAP Coding. 

Apply the indicates Kernel Patch or above.

These corrections were originally delivered with the kernel patch level specified in the "Support Package Patches" section of this SAP Note. All later kernel patch levels also contain these corrections. Note that only the latest kernel patch is available for download. It contains all previously delivered corrections. This means that you can always download and apply the latest kernel patch in order to implement the corrections.

If no kernel patch level is entered in the "Support Package Patches" section, the patch is not yet available. The entry appears with the delivery of the corrections. Check this SAP Note to find out whether new kernel patch levels have been entered.

These corrections are delivered with the following SAP kernel archives:

  • Hotfix - file dw.sar
  • SP Stack Kernel files SAPEXE.SAR and SAPEXEDB.SAR 

Follow our recommendations for selecting and applying the kernel patch:

  1. Import the latest SP Stack Kernel if it contains the corrections. For a list of current SP Stack Kernels, see SAP Note 2083594 ("SAP Kernel Versions and SAP Kernel Patch Levels").
  2. Otherwise, the corrections will be provided with the next SP Stack Kernel. Apply the hotfix only if you need the corrections quickly.
  3. The correction might not be available for kernel releases that are no longer maintained. Perform an upgrade to the relevant downward-compatible kernel (DCK); see SAP Note 2083594 (SAP kernel versions and SAP kernel patch levels).
  4. Check the regression note for the target patch level before applying the patch. For detailed information, see SAP Note 1802333 (Finding information about regressions in the SAP kernel).
  5. For instructions on how to download and apply kernel patches, see SAP Note 19466 (Downloading SAP kernel patches).

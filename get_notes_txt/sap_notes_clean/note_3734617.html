Symptom

Some applications have update task function modules for their app to update application tables. But purpose determination framework only supplies a function module, which can only be executed outside of the update task. Now when a commit is triggered today by the app, the update task will run (not immediately) but our Function module will immediately commit.

Other Terms

Purpose determination, Update task,

Reason and Prerequisites

New feature

Solution

Implement the note

Applications want all FM's are registered under update task FM. So, Created update task FM to support applications whose Save call is running in update task.

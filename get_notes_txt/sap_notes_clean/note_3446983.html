Symptom

Job AGSDVMAS_EX_<SID_CLIENT>_CLOUD cancels with the following error in the job's log:

User * has used the destination SAP-SUPPORT_PORTAL as S user S**********
Error while sending data to SAP: Invalid user S********** when calling SAP backend system
Job cancelled after system exception ERROR_MESSAGE

Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.

Environment

  • SAP Solution Manager 7.2
  • DVM Cloud Application

Reproducing the Issue

  1. In Solution Manager system, execute Tcode: SM37
  2. Search for job name:  AGSDVMAS_EX_<SID_CLIENT>_CLOUD
  3. In the User name field enter: *
  4. Click on "Execute"
  5. The job will be shown as Cancelled
  6. Access the job's log
  7. The following error message is present: Error while sending data to SAP: Invalid user S********** when calling SAP backend system 

Cause

Possibility of residual, temporary data has been copied after a new data collection. This residual data might occur while doing a system copy.

Resolution

Steps to Delete temporary / residual data which might be causing the job: AGSDVMAS_EX_<SID_CLIENT>_CLOUD to fail. Follow the steps below:

  1. In the managed system, execute Tcode: SE37
  2. in Function Module field enter: /SDF/DVM_S4_DEL_ANALYSIS_RES
  3. click on Test/Execute button:




  4. Leave the RFC target system blank and click on Execute:



  5. In the next steps, continue as directed.

This report is safe to use and will clear temporary/ residual data which might be causing the job AGSDVMAS_EX_<SID_CLIENT>_CLOUD to fail.

After the report is executed, please wait for the next run of the job AGSDVMAS_EX_<SID_CLIENT>_CLOUD and check if the issue persists.

After execution,  do not perform  any manual runs in the meantime.

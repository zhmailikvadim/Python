Symptom

  • After importing a model via transport, data locking settings reset to the default configuration.
  • A warning message appears during the transport/import process.
  • The default lock state switches to "locked".

Environment

  • Product: SAP Analytics Cloud

Reproducing the Issue

  1. Import a model into a target tenant using a transport package where the existing model has different data locking settings.
  2. Complete the model transport/import.
  3. Observe the warning message and that data locking resets, with the default lock state set to "locked".

Cause

Backward compatibility issue when importing packages originating from pre-QRC1 tenants into a QRC1 tenant, which causes data locking settings to reset to the default configuration.

Resolution

This as a known issue related to backward compatibility when importing packages originating from pre-QRC1 tenants into a QRC1 tenant. 

The issue has now been fixed and the correction will be delivered with the upcoming patch updates.

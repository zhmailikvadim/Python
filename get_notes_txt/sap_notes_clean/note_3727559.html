Environment

  • SAP Replication Server (SRS) 16.0
  • SAP Adaptive Server Enterprise (ASE) 16.0 with Always-on Option (HADR)
  • SAP Adaptive Server Enterprise (ASE) 16.0 with High Availability Disaster Recovery (HADR) for Business Suite
  • SAP Replication Server (SRS) 16.1
  • SAP Adaptive Server Enterprise (ASE) 16.1 with Always-on Option (HADR)
  • SAP Adaptive Server Enterprise (ASE) 16.1 with High Availability Disaster Recovery (HADR) for Business Suite

Cause

  • The RMA process was not running on the STANDBY machine, which caused the synchronization mode change command to fail.
  • This resulted in the inability to fully apply the command and led to the error encountered.

Resolution

  1. Start the RMA:

    •  On UNIX:

       cd $SYBASE/$SYBASE_ASE/bin/
       nohup ./rma &

    •   On Windows, start the RMA Windows service by either method (where <cluster_ID> is the ID of the cluster):

          - Start Sybase DR Agent - <cluster_ID> from the Services panel
        or
          - Issue this command:  net start SybaseDRAgent_<cluster_ID>

  2. Connect to the PRIMARY RMA

  3. Execute the command sap_update_replication synchronization_mode:

      sap_update_replication synchronization_mode, <ACTIVE_SITE_NAME>, async

Symptom

  • Error logs are not being captured in the case information system logs for cases created in the system.
  • Even when cases go into exception, the error messages are not recorded in the case information system logs.
  • Error entries are visible in general system logs but not associated with any case.

Environment

  • SAP Sales Performance Management
  • Advanced Workflow

Reproducing the Issue

  1. Log in to the workflow system.
  2. Create a workflow using "+" icon.
  3. Check case information system logs.

Cause

A utils script is executed directly instead of using a workflow script or invoking a specific method within a utils script. Executing the entire utils script causes the case_id context to be lost, so errors are not attached to the case logs.

Resolution

  1. Update implementation to use a workflow script that preserves case context.
  2. Replace direct execution of a utils script with the equivalent workflow script.
  3. If a utils script must be used, refactor it to expose and call a specific method rather than executing the entire script, ensuring the case context (case_id) is retained.

Workaround:

  1. Apply the temporary workaround to enable case-based searching of errors:
  2. Add the case_id to the error message in the script using logger.error so the case_id is part of the message.
  3. Search logs with: log_level = 'ERROR' and message like '%case_id=%'.
  4. Verify with the debug tool that logger.error executes in the expected code path and that errors appear either in case information system logs (after step 1) or are retrievable via the query (after step 2).

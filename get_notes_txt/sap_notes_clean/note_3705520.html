Symptom

  • Errors occur with the commentary service in Analysis for Office.
  • Error message: "Comments cannot be synchronized with BI platform. Reason: Encode failure. The database for comments did not start correctly."
  • Selecting option Use other support database and inserting same credentials of Audit database, comments works fine.

Environment

  • SAP BusinessObjects Business Intelligence Platform 4.x
  • SAP BusinessObjects Business Intelligence Platform 2025
  • Analysis for Office
  • SQL Server as Audit Database

Reproducing the Issue

  1. Open CMC> Applications> Commentary Service
  2. Select the option Use Audit database
  3. Open an Analysis for Office report from the BO portal.
  4. Observe the error message: "Comments cannot be synchronized with BI platform. Reason: Encode failure. The database for comments did not start correctly."
  5. Go to CMC> Applications> Commentary Service
  6. Select the option Use other Support database
  7. Put the credentials to same Audit database
  8. Restart APS with Commentary Service
  9. See the AO comments working

Cause

  • Selecting to use Audit DB will use an ODBC connection while selecting other supported databases, will use a JDBC. 
  • Wrong configuration of ODBC connection.

Resolution

  1. In the Central Management Console (CMC), go to Applications > Commentary Service > Properties.
  2. Select "use the Audit Database".
  3. Save and close.
  4. Configure the corresponding .sbo file located at <install_dir>\SAP BusinessObjects Enterprise XI 4.0\dataAccess\connectionServer\odbc to add the alias.
  5. For example, if using MS SQL Server, make a copy of sqlsrv.sbo file
  6. Open and edit the sqlsrv.sbo file.

            <Aliases>

                 <Alias>MS SQL Server 2005</Alias>
                 <Alias>MS SQL Server 2000</Alias>
                 <Alias>MS SQL Server 7.x</Alias>
                 <Alias>MS SQL Server 6.5</Alias>
             </Aliases>

 <DataBases>
      <DataBase Active="Yes" Name="MS SQL Server 2008">
            <Aliases>
                 <!-- You can add an alias here if you are using some connections
that are defined with an older database engine -->
                 <Alias>MS SQL Server 2005</Alias>

    7. Save the file
    8. Restart the APS with Commentary Service.
    9. Open Analysis for Office and see the comments working fine.

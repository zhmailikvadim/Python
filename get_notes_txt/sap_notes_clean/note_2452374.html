Symptom

During the the change of a material master with change number in MM12 but it leads to the shortdump in SAPLMALH because Data element "PSPNR" is not contained in "MARA".

Environment

  • SAP ERP Logistics (LO)
  • SAP R/3
  • SAP R/3 Enterprise
  • SAP ERP Central Component
  • SAP ERP
  • SAP Enhancement package for SAP ERP
  • SAP Enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA, on-premise
  • SAP S/4HANA, cloud edition

Reproducing the Issue

  1. GOTO Transaction MM12
  2. Enter a Material number
  3. Enter a Change number
  4. System dumps with a SYNTAX_ERROR 

         

         

Cause

  • Include LMALHARA has generated coding, which is created with the GENERATE program. 
  • Typically, this problem occurs when the customer appends customer-fields to the MARA table but the GENERATE program does not run after it.

Resolution

Please run this report GENERATE for table MARA, and then check the system behaviour again.

  1. GOTO SE38
  2. Enter program GENERATE
  3. Click display
  4. Execute
  5. Enter Table MARA

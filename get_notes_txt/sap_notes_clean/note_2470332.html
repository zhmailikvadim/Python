Symptom

You create vendors based on HR master data using reports RPRAPA00 / RFBIKR00 which create batch input sessions for transactions XK01 and XK02. The payee name in infotype 9 is filled:

After run of reports RPRAPA00 and RFBIKR00 and processing the batch input session, you notice that the payee name is not transferred as account holder name to the vendor master data:

 

"Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental."

 

Environment

SAP travel management - on premise: no S/4 HANA system

Cause

You can see in the previous 2 screen shots that the name of the employee equals the vendor name and that the name of the employee equals the payee name in infotype 9. Whenever the payee name equals the employee name and consequently also equals the vendor name, the account holder name on the vendor master data will stay empty. The reason is, that this field only needs to be filled, if the account holder name is not equal to the vendor name. Or looked from the HR master data side, if the payee name in infotype 9 is not equal to the employee name as displayed in infotype 6. With respect to the behavior of HR master data behavior please read the SAP note 560796.

Please read the F1 field help for the account holder field in the vendor master data. It will tell you this:

 

Resolution

If you search for a way to always fill the account holder name whether or not it equals the employee name, you can only reach this by using the BADI BADI_EXITS_RPRAPA00 with method SET_VALUES_FOR_BLFBK. Here you can fill the content of C_BLFBK-KOINH.

PLEASE HAVE IN MIND: When viewing the infotype 9 data in personal administration (for example in transaction PA20), you see the payee name, whether or not the name of the payee equals the employee name. In the example shown in the 1st screen shot, the displayed payee name however is only a transient field. This means it is filled dynamically at runtime of PA20 and it is NOT saved in database table PA0009. In this respect, also the personal administration follows the same logic: account holder name (payee name) will only be saved, if it is different to employee name. Please again refer to SAP note 560796. Furthermore this means: in your BAdi implementation you may not use the content of PA0009-EMFTX. It will be empty if the name of the payee equals the employee name. You may use the employee name, for example the content of PA0001-ENAME.

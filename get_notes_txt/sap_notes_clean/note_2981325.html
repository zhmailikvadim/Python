Symptom

The test of the SAP BusinessOjects Intercompany datastore fails if the user has the "Product Locale" property set with “Use browser locale” value in the Central Management Console's user preferences.

The following message is displayed ( if web browser is set with "English - United States" language) :

"Failed while calling the 'MetadataWebServices.asmx?wsdl' system to perform the ´getDimensionsInfos' action.

Couldn't find 3-letter language code for en-us"

No issue if the "Product Locale" property is set with  “English” value.

Other Terms

BOE, CMC, languages, browser regional settings, preferences

Reason and Prerequisites

Steps to reproduce:

Connect to the Central Management Console with a   SAP Financial Information Management user .

Click on Preferences.

Set the "Product Locale" property  of the current user with the “Use browser locale”  value.

In the web browser, set the language to "English - United States".

Connect to SAP Financial Information Management with the user.

Go to the list of datastores.

Create a datastore  with SAP BusinessOjects Intercompany type ( or open an existing one).

Test the datastore.

Result: Test fails with the following message:

"Failed while calling the 'MetadataWebServices.asmx?wsdl' system to perform the ´getDimensionsInfos' action.

Couldn't find 3-letter language code for en-us"

Solution

Update your SAP Financial Information Management installation to a Support Package, Patch or Release where the issue is fixed.

See the Support Packages & Patches section for details and available patches.

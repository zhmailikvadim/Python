Symptom

  • When loading or viewing data across different interfaces (UI, Excel Loader, REST API, or ODI/XDL), users may notice a one-day discrepancy in how effective end dates are displayed or processed.
  • The discrepancy in the effective end date between the UI and backend table is causing confusion. 

Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.

Environment

  • SAP Sales Performance Management
  • Incentive Management

Reproducing the Issue

To observe the discrepancy between inclusive and exclusive date logic:

    1. UI Observation: Load a record via ODI (e.g., a Participant or Position) with an effective end date of Jan 01, 2026. Check the loaded information in the UI it will show Dec 31, 2025.

    2. Database/ODI Verification: Query database directly. Notice the end date appears as Jan 01, 2026.

    3. Excel/API Verification: Pull the same record via REST API V2. Notice the date returns to Dec 31, 2025, matching the UI.

    4. EOT Test: Update the record in the UI to the "End of Time" date: Jan 01, 2200.

    5. Final Verification: Check the Database. The date will be Jan 01, 2200, showing that the standard "minus one" logic is bypassed for this specific hard-coded value.

Cause

The system utilizes two different logical approaches for date ranges depending on the interface used. This is due to the underlying architecture of each component:

  1. Exclusive Logic (Up to but not including): Primarily used by the Database and the ODI/XDL layers. The end date represents the first day the record is no longer active.
  2. Inclusive Logic (Up to and including): Primarily used by the UI and consumer-facing APIs. The end date represents the last day the record is active.

Resolution

Users must apply the correct date format based on the tool being utilized. 

  1. Standard Date Logic (Example: Calendar Year 2025)
    For a record intended to be active through the end of 2025: 
    InterfaceLogix TypeEnd Date Format
    User Interface (UI)Inclusive2025-12-31
    Excel Data Loader Inclusive2025-12-31
    REST API V2 Inclusive2025-12-31
    ODI / CDL Exclusive2026-01-01
    Database (Table) Exclusive2026-01-01

  2. The "End of Time" (EOT) Exception 
    The date 01/01/2200 is a special hard-coded constant. All interfaces—regardless of their underlying logic—treat this date as a static value. No offsets or "minus one" calculations are performed. 
    InterfaceLogix TypeEnd Date Format
    User Interface (UI)Permanent / Open-Ended 2200-01-01 
    Excel Data Loader Permanent / Open-Ended2200-01-01 
    REST API V2 Permanent / Open-Ended 2200-01-01 
    ODI / CDL Permanent / Open-Ended2200-01-01 
    Database (Table) Permanent / Open-Ended2200-01-01

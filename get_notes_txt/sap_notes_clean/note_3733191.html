Symptom

Why is the system not reverting to the original value upon changing the item category?
Why does the system not revert to the original value when the item category is changed from Limit to Service or Service to Limit?

Reproducing the Issue

  • Create  Requisition > Select the Item category as Limit ( based on limit functionality, the Overall limit field and Expected Amount will be displayed).
  • Enter the amount for both the overall limit and Expected amount for ex: $123 USD respectively. 
  • Change the item category to Service (D).
  • When it is selected as Service item category, based on this functionality Max Amount, Expected Amount, Service Start and End date fields will be visible.
  • Enter the values for the Max amount and expected amount as $1234 USD.
  • Revert the item category field value as Limit from Service > system still holds up the value $123 USD.
  • The values are not getting refreshed or Zero(0).

Cause

This is expected system behavior. When the Item Category Limit (B) is selected, the Overall Limit and Expected Amount fields are displayed based on its functionality.

When the Item Category Service (D) is selected, the Max Amount, Expected Amount, and Service Start and End Date fields are displayed according to its functionality.

Both item categories have different dependent fields and serve distinct functionalities.

Resolution

It is expected behavior that when changing the Item Category from Limit to Service or vice versa, the user must manually clear the values from the fields associated with the previous item category before submitting the requisition.

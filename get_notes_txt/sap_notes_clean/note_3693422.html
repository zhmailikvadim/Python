Symptom

No communication log generated in the "Process Payment Items for Invoices - China" app after completing payment-related processes.

Environment

SAP S/4HANA Cloud Public Edition

Reproducing the Issue

  1. Open the "Process Payment Items for Invoices - China" app.
  2. Process payment-related items, including invoice submission and payment.
  3. Observe that no communication log is generated after completing the payment process.

Cause

In the "Process Payment Items for Invoices - China" app, after completing the "Payment Proposal", "Submit" and "Pay" process, the status of the payment items should be "Payment Order Created". At this point, there has been no communication with the bank/middleware, so no communication logs will be generated.

Resolution

  1. Verify the configuration of SAP_COM_0554 and click "Check Connection" in the Communication Arrangement app.
  2. If issues are detected in the API configuration, contact the API service provider to ensure API availability.
  3. Once the API is confirmed to be functional, attempt to send the payment file to the bank/middleware in the "Process Payment Files - China" (App ID: EPIC_FILE ) and observe the communication logs.

Symptom

This SAP Note serves as the master note for the product SAP GRC UIDP Masking for SAP HANA. The product consists of three add-ons: SAPGRC 100, GRCUIDP 100, and UISAPGRC 100.

For details on which add-on should be installed in which system according to your system landscape requirements, refer to the Installation Guide or Upgrade Guide available on the product page for SAP GRC UIDP Masking for SAP HANA.

The previous version of product SAP GRC UIDP Masking for SAP HANA is UI Data Protection Masking for SAP S/4HANA 2011. The corresponding Master Note is 2980561.

Important Information - While installing this product always implement the latest support pack. Refer below PAM link for latest support pack.

Information on Enhancements/Modifications:

SAP GRC UIDP Masking for SAP HANA product is developed to meet specific customer requirements, which may require modifications and enhancements in the standard frameworks. As a result, upgrades can be challenging, and certain modifications or enhancements may be lost during the upgrade process. To assist customers, a Readiness Check report is delivered in the plugin component to identify any missing modifications or enhancements after the upgrade, allowing them to be reapplied as needed.

Solution

After completing the installation, implement the following manual SAP Notes in plugin system (where add-ons GRCUIDP 100 and UISAPGRC 100 is installed) to integrate masking with the required frameworks.

For masking in SAP GUI screens, below notes need to be applied in plugin backend system where add-on GRCUIDP 100 is installed

Pre-implementation steps:

  1. Please change the value of RZ11 parameter 'dynp/usrmasking' to OFF before applying below manual notes changes in your system.
  2. Delete entry from table maintenance generator /LOGS4H/TSECLOGV using transaction SM30

 Note Number

 Text

 Category

 2773421

 S/4HANA Masking: Manual changes for SAP GUI - ALV Item Tree & Query (Ad-Hoc & SQVI)

 Manual Change

 Manual Change

Post-implementation steps:

  1. After applying above manual notes successfully, kindly change the value of RZ11 parameter 'dynp/usrmasking' to ALL.
  2. Maintain entry X in table maintenance generator /LOGS4H/TSECLOGV using transaction SM30

Symptom

The error following occurs while adding a new business place, company code, profit center, section code, segment, or tax jurisdiction in Assign Org. Units to a Reporting Entity within SSCUI ID 101071 – Setting Up Your Statutory Reporting.

Error appears when saving: "Assign values to all mandatory org. units of the report categories."

Environment

SAP S/4HANA Cloud Public Edition

Reproducing the Issue

  1. Open SSCUI ID 101071 “Setting Up Your Statutory Reporting”.
  2. Select the reporting entity, open “Set Validity of Organizational Unit”, and navigate to “Assign Org. Units to a Reporting Entity”.
  3. Assign a new entity.
  4. Save, then the above error is displayed.

Cause

Mandatory organizational units are missing for the report entity.

Every reporting entity has its mandatory organizational units, you can check it in the help portal Supported Compliance Tasks by Country/Region | SAP Help Portal and navigate to the Setting Up the Report page to know which organizational units are mandatory for one report task.

For example, for reporting entity IN_RPG_ENT (India Withholding Tax Form 26Q report), both COMPANYCODE and SECTION_CODE are mandatory organizational units; if only COMPANYCODE has been maintained and SECTION_CODE is missing, system will report the error to remind user.

Resolution

Kindly refer to the following steps to resolve this issue after learned about the mandatory organizational units for the reporting entity:

  1. Open SSCUI ID 101071 “Setting Up Your Statutory Reporting”.
  2. Select the reporting entity and go to “Set Validity of Organizational Unit” and open “Assign Org. Units to a Reporting Entity”.
  3. Maintain both mandatory organizational units as needed.
  4. Save the changes and recheck that the error no longer appears.

Please note that you can assign multiple values to only an organizational unit of a single type. For example, if you assign company code and tax jurisdiction to your reporting entity, you can enter multiple values for either the company code or tax jurisdiction, but not for both.

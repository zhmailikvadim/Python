Symptom

  •  After system upgrade to SP34 the SFTP Communication Channels with file content conversion start failing.
  • The error message encountered during file content conversion is: "Exception caught by adapter framework: java.lang.ArrayIndexOutOfBoundsException: while trying to load from index 1 of an object array with length 1, loaded from local variable 'nameSplit'".

Environment

  • SAP NetWeaver 7.5

Cause

Known coding issue

Resolution

Please apply the patch fix under the current Support Package, it might no need to make any changes in your configuration with respect to the parameter in this note if you have already in place. 

  • 3633876 - Removal of separators in FCC SFTP Receiver Adapter

See Also

SAP Note 3561795 - Removal of separators in FCC File Receiver Adapter

Keywords

Process Orchestration 7.5, PI 7.5, PO 7.5, FCC, SFTP, Receiver Adapter

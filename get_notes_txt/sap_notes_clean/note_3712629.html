Symptom

  • An error occurs when attempting to re-upload all lines of the bill of materials (BoM) through Data Transfer Workbench (DTW) after removing all lines from the BoM using the "bill of materials management" window.
  • Error message: "9 Update Failed 00146 user cannot be edited [ITT1.Quantity][line: 18] user cannot be edited [ITT1.Quantity][line: 18]65171".
  • The issue persists despite verifying items, components, quantities in the template, and decimals in SAP general settings.

Environment

Data Transfer Workbench for:

  • SAP Business One 
  • SAP Business One, version for SAP HANA

Reproducing the Issue

Steps to reproduce:

1. Remove all content lines from a BoM product using the "Bill of Materials" window in SAP Business One Client.

2. Create a new Data Transfer Workbench (DTW) import template for BoM rows containing “Text” data types for the BoM product. 

3. Fill the Quantity and Price columns with 0s for the "Text" data type rows in the DTW template.

4. Attempt to update the BoM product with new rows by loading the new DTW template with the data through Data Transfer Workbench (DTW).

Actual Result:

DTW returns the error message: "9 Update Failed 00146 user cannot be edited [ITT1.Quantity][line: 18] user cannot be edited [ITT1.Quantity][line: 18]65171".

Expected Result:

No errors occurrs, and the DTW template is successfully loaded, allowing the BoM product to be updated with new rows.

Cause

The issue was caused by '0's in the Quantity and Price columns for the text data type lines in the DTW import template.

Resolution

1. Open the template used for uploading the BoM rows.

2. Locate the Quantity and Price columns for the text lines.

3. Remove any '0's present in these columns for the text lines.

4. Save the updated template.

5. Re-upload the BoM lines through DTW using the updated template.

6. Verify that the BoM updates successfully without errors.

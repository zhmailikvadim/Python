Symptom

  • When executing a structure report (e.g. transaction CN43N) in a background job, the WBS-Element ID column (or other) output does not display full details/length.
  • In dialog, the WBS-Element ID column output appears with full details.

Environment

  • SAP ERP Central Component
  • SAP ERP
  • SAP Enhancement package for SAP ERP
  • SAP S/4HANA
  • SAP S/4HANA Cloud Private Edition

Reproducing the Issue

  1. Execute the structure report in a background job.
  2. Observe that the WBS-Element ID column (or other) output does not display full details/length.
  3. Execute the structure report in dialog mode.
  4. Observe that the WBS-Element ID column (or other) output displays full details.

Cause

The issue is caused by the "With optimum column width" option being flagged in the structure reports layout settings. 

Resolution

  1. Unflag the "With optimum column width" option in the layout settings.
  2. Adjust the column width to a more reasonable size based on the length of data shown in the columns.
  3. Save the updated layout settings.
  4. Execute the structure report in a background job using the updated layout.
  5. Verify that the column output displays full details in the background job.

Alternatively, a change in the ALV processing code of SAP standard is necessary to activate the "With optimum column width" option for background processing as well.

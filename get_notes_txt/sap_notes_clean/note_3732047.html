Symptom

  • Opening a shared survey link displays the last respondent’s submitted results instead of a new blank survey instance.
  • A new participant is only offered to edit the existing submission rather than create an independent response.
  • Multiple recipients cannot create separate individual responses using the same survey link.
  • No error messages are displayed.

Environment

  • Product: SAP LeanIX solutions
  • SAP LeanIX - Application Portfolio Management

Reproducing the Issue

  1. Send a configured survey link to one user and have the user complete and submit the survey.
  2. Send the same survey link to another user.
  3. The second user opens the link.
  4. Observe that the survey shows the previous respondent’s results and allows editing instead of creating a new response.

Cause

By design, surveys store a single response per fact sheet; once one participant submits, the survey is considered complete for that fact sheet and subsequent openings show the last submitted response for potential editing.

Resolution

  1. Note that this is expected behavior: a single response is stored per fact sheet, and later openings of the same survey link display the latest submission for editing.
  2. If separate, independent responses are required from multiple participants, submit a feature request to product management: use the idea portal via the link in See also.

Symptom

In MDC processes using the template "MDC Template: Responsible to Authorized (Authorized View)" (DE_B_MDC_RECV_AUTH_TEMPLATE), such as "MDC: Receipt of 55218 from Responsible DSO to SUP" (DE_B_MDC_55218_DSO2SUP_RECV), the system incorrectly linked rejection cases when the MDC process was associated with SOS, SBS, or NIN processes in step 1350. Additionally, the MDC process searched for process documents only for the first time slice of ValidData.

This issue is relevant for all market roles.

Other Terms

SOS, SBS, NIN, DE_B_GPKE_SOS_SUP, DE_B_MPES_SOS_SUP, DE_B_GPKE_SBS_SUP, DE_B_GPKE_NIN_SUP, DE_B_MPES_NIN_SUP, /US4G/CL_CHK_MDC_PROC~CHECK_MDC_ASSOCIATE, ValidData

Reason and Prerequisites

The error occurred because the system linked rejection cases when the MDC process was associated with SOS, SBS, or NIN processes, and the system searched process documents only for the first time slice of ValidData.

Solution

Implement the corrections of this SAP Note or import the relevant support package.

After implementing this SAP Note, the MDC process correctly links the SOS, SBS, or NIN process passed from the cloud MDC process and excludes rejection cases. The system also searches for process documents across all time slices of ValidData. This fix is related to emergency patch EP 2000011622 on the cloud side. You can verify the fix by checking that the MDC process links the correct SOS, SBS, or NIN process passed from the cloud process without including rejection cases.

Symptom

  • The Header data is hidden in transaction UDM_DISPUTE in Fiori Launchpad
  • Issue occurs since kernel update.
  • There is no issue running UDM_DISPUTE in SAPGUI for Windows.

Environment

  • transaction UDM_DISPUTE in Fiori Launchpad

Reproducing the Issue

  1. Access the transaction UDM_DISPUTE in Fiori Launchpad
  2. Select "Look for Collection Case" and click "Search" on the right side.
  3. Double-click a column in the "Case ID" field.
  4. Observe that the Header Data is not displayed and is hidden behind the frame.
  5. Scroll down the bottom frame to snap it, and the data might reappears.

Cause

  • The issue arises due to the design of the application where the deepest container has fixed sizes that exceed the browser window size.
  • The focus is programmatically set to the first cell of the table, which is outside the visible range of the screen, causing the browser to perform a native scroll-into-view action.

Resolution

Implement SAP Note 3680679 - Header Data is not visible in the Case Management (SCASE) Web application.

Keywords

SCASE, kernel patch, accessibility, header data not visible, Header Data

Symptom

The parameter Application.SM.QuestionnaireVersionOnERPSync is set to True in SM Administration. When the end-user resubmits the questionnaire without modifying any answers, a new response version is created. The expectation is that no new version of the response should be created when the end-user resubmits the same response to the registration questionnaire.

Environment

Supplier Lifecycle & Performance > Supplier Registration Project

Reproducing the Issue

  1. Manage > SM Administration > Configuration Parameters > set parameter Application.SM.QuestionnaireVersionOnERPSync to True.
  2. Resubmit the questionnaire without modifying any answers.
  3. Observe in the UI that a new response version is created.

Cause

This behavior is expected as the parameter Application.SM.QuestionnaireVersionOnERPSync is not intended to prevent the creation of a new questionnaire response submitted by an end-user. The parameter is designed to prevent the system from automatically creating a new version of the questionnaire response when a sync is triggered by the ERP, but there is no new data to review and approve. The differentiation lies in the fact that the parameter blocks new system-generated responses triggered by an ERP sync, not user-generated responses. Below is a description of the parameter's function as per our documentation:

"When enabled, this parameter specifies that ERP synchronization only creates a new questionnaire version if the sync includes an update to at least one mapped question in the currently active version of the questionnaire. Otherwise, ERP synchronizations can create new questionnaire versions even when no data in the currently active version of the questionnaire is updated."

Resolution

This system behavior is expected.

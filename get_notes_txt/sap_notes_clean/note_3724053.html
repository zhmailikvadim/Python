Symptom

  • When trying to activate calculation view based on a table function, it failed with error:
    "Repository: Encountered an error in repository runtime extension; model inconsistency. Object of id yy#xxxxxxx not exist"
  • This usually happens after a system upgrade, migration or system copy.

Environment

  • SAP HANA Platform Edition 1.0
  • SAP HANA Platform Edition 2.0

Reproducing the Issue

  1. Perform an upgrade, migration, or system copy.
  2. Attempt to activate a calculation view based on a table function.
  3. Observe the error message: "Repository: Encountered an error in repository runtime extension; model inconsistency. Object of id yy#xxxxxxx not exist."

Cause

The issue was identified as a corrupt remote dependency problem. A missing object might be an object from cross-database access.

Resolution

To identify the corruption or whether the object is from a remote database, execute the following SQL commands:

CALL CHECK_CATALOG('CHECK_DEPENDENCY', '', '', '');
SELECT * FROM SYS.P_OBJECTDEPENDENCY_ WHERE DEPENDENTOID = (SELECT OBJECT_OID FROM OBJECTS WHERE OBJECT_NAME = '<PACKAGE NAME>::<OBJECT NAME>') ORDER BY BASEOID DESC;

To repair the remote dependencies, run the following SQL commands with the administrator user:

CALL CHECK_CATALOG('REBUILD_REMOTE_DEPENDENCY', NULL, NULL, NULL);
CALL CHECK_CATALOG('REPAIR_REMOTE_DEPENDENCY', NULL, NULL, NULL);`

Keywords

Calculation View, Table Function, Activation Error, Repository Runtime Extension, Model Inconsistency, Object ID Not Exist, Corrupt Remote Dependency, Cross-Database Access

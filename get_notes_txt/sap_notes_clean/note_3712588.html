Symptom

We would like to gain a detailed understanding of each column and element in the Process Model Metrics report.

Specifically, we would like clarification on the source and calculation of custom attributes that appear in this report, such as:

  • Number of Forks
  • Operators
  • Number of Subprocesses
  • Ratio of linked subprocesses
  • Number of linked files
  • Number of dictionary links
  • Number of process steps without responsibility definition
  • Number of responsibility handovers
  • Publishing state
  • Last saved by

We would appreciate more detailed information on how these fields are derived and their underlying data sources.

Resolution

Below is an explanation of the included columns, with a focus on their meaning in a BPMN (Business Process Model and Notation) context:

  • Type: Indicates the primary type of the diagram (e.g., "BPMN Process Diagram," "BPMN Collaboration Diagram").
  • Number of Elements: Counts all individual graphical objects that are part of the BPMN model. This includes tasks, events, gateways, pools, lanes, data objects, and groups.
  • Number of Edges: Refers to the connectors between elements in a BPMN diagram. This counts sequence flows (solid arrows for process flow), message flows (dashed arrows for communication), and association flows (dotted lines).
  • Number of Forks: Typically refers to a gateway that splits a single path of execution into multiple subsequent paths (e.g., a Parallel, Exclusive, or Inclusive Gateway with multiple outgoing sequence flows). This metric counts such splitting gateways.
  • Operators: Refers to the distinct roles, organizational units, or systems responsible for performing tasks in the process, often represented by Lanes or Pool participants. It counts unique performers or participant types.
  • Number of Subprocesses: Counts elements that encapsulate a more detailed process. This includes "collapsed" subprocesses (represented as a task with a plus sign) and "embedded" subprocesses (a self-contained process within a larger one, bordered by a rounded rectangle).
  • Number of linked Files: Counts external documents, attachments, or media files that have been associated or linked as attributes to various elements within the BPMN diagram.
  • Number of Dictionary Links: Refers to the central "dictionary" for reusable components like roles, systems, data objects, or KPIs. This metric counts how many elements in the BPMN diagram are linked to entries within this central dictionary, promoting consistency and reuse.
  • Number of Process Steps without Responsibility Definition: A "process step" in BPMN typically refers to a Task or Subprocess. This metric counts how many of these tasks or subprocesses in the diagram have not been assigned to a specific role, organizational unit, or system (e.g., by being placed in a Lane or having a performer attribute assigned).
  • Number of Responsibility Handovers: In a BPMN diagram, a responsibility handover occurs when the flow of execution moves from one participant (e.g., a role or department represented by a Lane) to another. This metric counts the number of sequence flows that cross lane boundaries.
  • Path: The hierarchical folder location of the diagram within your SAP Signavio workspace.
  • ID: The unique system-generated identifier for the specific diagram.
  • Revision: The version number of the diagram, indicating its update history.
  • Last saved by: The username or identifier of the user who last saved modifications to the diagram.
  • Date of last publication: The date when the diagram was last officially published, making it available for wider consumption or review.
  • Open in Explorer link: A direct hyperlink to open the diagram within the SAP Signavio Process Manager Explorer interface.
  • Open in SAP Signavio Process Collaboration Hub Preview: A direct hyperlink to view the diagram in the SAP Signavio Process Collaboration Hub, typically used for collaboration and stakeholder engagement.

Symptom

Adding the Form Section table to a Story report based on a Single Performance Template data source results in duplicate rows for each section.

Environment

  • SAP SuccessFactors Performance Management
  • SAP SuccessFactors People Analytics

Reproducing the Issue

  1. Log in to the system and navigate to Reporting > Story Reports.

  2. Create a new Story report.

  3. Select a data source based on a single Form Template.

  4. In the report, add the following tables:

    • The Form Template table (single template).

    • The Form Section table.

  5. Add the required standard fields from the Form Section table to the report.

  6. Add one or more custom elements (custom fields) from the selected Form Template to the report:

    • Note that when using a single Form Template as the data source, custom elements appear as columns.

  7. Run the report and review the output.

  8. In the Preview- duplicate columns can be seen for each custom element for each section.

Cause

This is expected behavior by design. In Story Reports, each custom element in a form section is treated as a separate record, resulting in one row per custom element and causing duplicate section rows.

Resolution

Use the Cross Tab option in the Story Report to restructure the data so custom elements display as columns and duplicate rows are consolidated.

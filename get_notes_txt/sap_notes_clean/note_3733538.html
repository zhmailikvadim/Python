Symptom

Component: Environment Management
Module: Data Collection

You run program R_EHENV_MDEF_CHECK_EXECUTE to collect missing and overdue amounts with the filter selection "All Data" or "All non hourly Data Periods". The report has relatively poor performance when the last execution date and next execution date of the data collection schedule contain incorrect date information.

Other Terms

CL_EHENV_MDEF_MISSING_CHECK, R_EHENV_MDEF_CHECK_EXECUTE

Reason and Prerequisites

A program error in the system causes the symptom.

You find the prerequisites in the relevant correction instructions.

Solution

After implementing this SAP Note, the report skips the incorrect data collection schedule and displays an error message in the logs with Object EHENV_EMIS (Environmental Management Integration System) and Subobject AUTO_MDEF_CHECK (Schedule job for missing and due amounts) in transaction SLG1.

Alternatively, you import the corresponding Support Package for the component extension of SAP Environment, Health, and Safety Management or the Feature Package Stack for the SAP S/4HANA system.

Symptom

When testing the connection from SAP Integration Suite to SAP Cloud Connector (SCC), the following error occurs:

  • Could not connect to cloud connector.

In the SCC trace files, the following exception is logged:

  • An exception was thrown by com.sap.core.connectivity.tunnel.client.ssl.TunnelClientSSLHandshakeValidator

Environment

  • Cloud Integration
  • SAP Integration Suite

Cause

SSL handshake failure between SCC and the SAP BTP Connectivity endpoint due to untrusted or missing certificates in the SCC JVM trust store, typically triggered by outdated SCC/JVM versions or recent root certificate changes.

Resolution

To resolve the issue, perform the following steps:

  1. Upgrade SAP Cloud Connector
    • Update to the latest supported SCC version.
  2. Update the Java Runtime (JVM)
    • Ensure the SCC is running on a supported and up-to-date SAP JVM or OpenJDK version.
  3. Update Trust Store
    • Import any missing root and intermediate CA certificates into the SCC trust store.
  4. Check Network Security Components
    • Ensure no SSL inspection, reverse proxy, or firewall is intercepting or re-signing traffic.
  5. Refresh Subaccount Configuration
    • Re-download the subaccount authentication data from SAP BTP.
    • Remove and re-add the subaccount in SCC.
  6. Verify System Time
    • Ensure correct time synchronization (NTP) on the SCC host.

Upgrading both Cloud Connector and the JVM to the latest supported versions resolves most SSL handshake related connectivity failures.

Symptom

  • You are performing an upgrade using the Software Update Manager (SUM) in the first system in your landscape
  • In the SUM phase ACT_TRANS or ACT_UPG, you are prompted to perform the SPDD adjustments
  • After performing the adjustments, you wish to assign a transport and mark it as an adjustment transport for the subsequent systems
Utilities --> Assign Transport
For which Software Logistics tool do you want to flag a modification adjustment transport?
Support Package Manager/Add-on Installation Tool (-->"OCS") or Upgrade/Enhancement Package Installer (-->"Upgrade")
--> You choose Upgrade
  • You notice that the SAP_ADJUST is not assigned under SAP_CTS_PROJECT for the transport request as is the case during a SPAM/SAINT update

Cause

This is normal behavior. The SUM upgrade uses a different mechanism for flagging the adjustment requests than the SPAM/SAINT. In case you choose upgrade in the pop-up above, then the transport request is added to the umodauto.lst but no SAP_ADJUST is assigned to it. In the case of SPAM/SAINT updates, the SAP_CTS_PROJECT SAP_ADJUST is assigned.

Resolution

1- There is no action required. Please verify if the transport number is added to the <path>/SUM/abap/bin/umodauto.lst file, then it will be automatically picked by the SUM upgrade in the subsequent system.

2- If the system is under ChaRM control, then the Transport Organizer will not let you release the SPDD request without a SAP Project entered.  ChaRM makes SAP_CTS_PROJECT and SAP_TMWFLOW mandatory attributes in every transport request. Therefore, in such cases, it is recommended to enter SAP_ADJUST as CTS project in your SPDD request.

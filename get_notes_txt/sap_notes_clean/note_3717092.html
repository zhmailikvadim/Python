Symptom

Please note that this SAP note is intended for customers who have upgraded their S/4HANA system using the S4FND TCI note.

Add form categories in the BTP SAP Dynamic Forms - Form Categories app and add corresponding mappings in the Form Definition Attribute Conditions in the MAIF configuration panel. If there are no entries in the Business Object Entity Condition table, the correct forms will be assigned to all sub-objects.

For example, if a form definition has category customAttribute1 = 0010 (for work order operation number), and a work order has 3 operations, you will notice that the correct form is selected, but it is assigned to all 3 operations. This note will use the category value as a secondary filter so that only operation 0010 will get the form as expected.

This enhancement will only work if you have enabled form definition caching by enabling "Use Template Cache" in the dynamic form agent settings in the MAIF configuration panel. When this setting is turned on, form definitions are referenced from a cache table on (this applies globally, including in SSAM requests). This cache table is updated via a background job: documentation

This note also includes fixes for the following issues:

  • If cached keys are used when requesting DynamicFormConfigFiles from SSAM, no data is returned and there are a series of 500 errors in the SLG1 logs due to missing RFC destination
  • Event framework requires at least one status restriction ("Require * status") for the event to run
  • When "Form Definition By Rule" Form Assignment is used, form categories matching logic is always called even if no category mapping configuration is active
  • If translated form agent description exists in /SMFND/C_DFA01T and missing in /SMFND/YC_DFA01T, the standard config is not getting picked up in form explorer
  • Form status is always shown in English in the form explorer
  • Forms created via task list assignment no longer works after implementing SAP note 3700567
  • UI issues in the configuration panel related to "Form Definition Attribute Conditions" including delete button visibility, and updating existing records

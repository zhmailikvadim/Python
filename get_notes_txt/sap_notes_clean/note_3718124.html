Symptom

Error message "Deletion not possible for record created by SAP" appears when attempting to delete billing types from configuration activity "Assign eDocument Type to Billing Document Type for GST Reporting".

Environment

S/4HANA Cloud Public Edition

Reproducing the Issue

  1. Navigate to the configuration activity "Assign eDocument Type to Billing Document Type for GST Reporting" in the CBC client.
  2. Attempt to delete billing types F8 which is created by SAP.
  3. Observe that billing type created by customer is deleted successfully, but billing type F8 which is created by SAP triggers the error message "Deletion not possible for record created by SAP".
  4. Attempt to delete billing type F8 individually, and observe the same error message.

Cause

  • Billing type F8 was created by SAP_SYSTEM, and entries created by SAP cannot be deleted due to system restrictions.
  • Billing type created by a user and can be deleted without restrictions.

Resolution

  1. Identify the billing types that need to be deleted in the configuration activity "Assign eDocument Type to Billing Document Type for GST Reporting".
  2. Separate the deletion process for billing types created by SAP and those created by users.
  3. Delete billing types individually which was created by the user.
  4. For deleting billing type F8 or any billing types created by SAP, get in touch with your SAP CEE/CSM or usual SAP contact, also raise an incident in component XX-S4C-SRV-CON. Refer community link for more details.

Note: This issue has been corrected in 2602 release, but the entry would still exist in the view for existing public cloud customers. Hence follow the above steps mentioned.

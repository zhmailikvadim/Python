Symptom

GST tax amount shall be calculated based on the Retail Sale Price (RSP) for notified goods, with the provision coming into effect from 1 February.
Tax amount = (Retail sale price X tax rate in % of applicable taxes) / (100+ sum of applicable tax rate) 
"Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental." 

Environment

  • SAP S/4HANA Private Cloud
  • SAP S/4HANA On-Premise
  • SAP ERP

Resolution

Configurations on Sales side: 

Step 1: Create an access sequence to identify Tobacco goods and retrieve the relevant condition records that are mentioned in further steps. 

Transaction: V/07 

Step 2: Create new condition types for RSP Price, GST types and a GST Total for sum of these taxes. Configure the above access sequence created. 

Transaction: V/06 

ZMRP : To maintain the Retail Sale price (RSP) 

ZOIG, ZOCG, ZOSG, ZOUG : For calculation of tax amounts 

  

 

ZMRP Condition: To maintain the Retail Sale price (RSP) 

 

 

ZOIG,ZOCG, ZOSG, ZOUG: For calculation of tax amounts 

 

 

ZRGS: Sum of total GST that is calculated (Note that it should be configured as Negative)  

Note: This condition ZRGS is required only if Base Value is inclusive of tax. It is not required if Base Value (PPR0) does not include taxes. 

 

 

Step 3: Identify the routine number that needs to be enhanced. Suggested routine is 345. 

Enhance the routine logic with sample code provided below 

CALL FUNCTION 'J_1BSA_COMPONENT_ACTIVE' 

EXPORTING 

bukrs = komk-bukrs 

component = '**' 

IMPORTING 

active_component = lv_comp 

EXCEPTIONS 

component_not_active = 1 

OTHERS = 2. 

IF lv_comp EQ 'IN'. 

DATA: lv_cond TYPE j_1icondname. 

DATA: lv_taxrate TYPE komv-kwert. 

CONSTANTS mc_10 TYPE kwert VALUE 10. 

CONSTANTS mc_100 TYPE kwert VALUE 10000. 

DATA: lv_mrp_po_kwert TYPE kwert. 

CALL FUNCTION 'J_1I4_GET_KWERT' 

IMPORTING 

MRP_kwert = lv_mrp_po_kwert. 

IF lv_mrp_po_kwert IS NOT INITIAL. 

xkomv-kawrt = lv_mrp_po_kwert. 

ENDIF. 

lv_taxrate = xkomv-kbetr / mc_10. 

SELECT SINGLE cond_name FROM j_1iexcdefn INTO lv_cond WHERE kalsm = komk-kalsm AND kschl = xkomv-kschl. 

CASE lv_cond. 

WHEN 'IGSTAR' OR 'IGSTSOFFAP'. 

xkwert = ( xkomv-kawrt * lv_taxrate ) / ( mc_100 + lv_taxrate ) . 

xkomv-kawrt = xkomv-kawrt - xkwert . 

WHEN 'CGSTAR' OR 'SGSTAR' OR 'UTGSTAR' OR 'CGSTSOFFAP' OR 'SGSTSOFFAP' OR 'UTGSTSOFFAP'. 

xkwert = ( xkomv-kawrt * lv_taxrate ) / ( mc_100 + ( lv_taxrate * 2 ) ). 

xkomv-kawrt = xkomv-kawrt - ( 2 * xkwert ) . 

ENDCASE. 

ENDIF. 

Step 4: Configure the Pricing procedure with above condition types 

Transaction: V/08 

ZMRP : Should be statistical 

ZOIG, ZOCG, ZOSG, ZOUG: Maintain the above routine that calculates the Tax amount in ‘Alt Calc Cndn Amnt’.  

ZRGS: Sum of all above 4 GST conditions. It doesn’t have Alt Calc Cndn. Amn.t condition formula assigned to it like for other conditions. 

 

 

 

 

Step 5: Maintain the condition records  

Transaction: VK11 

ZMRP: Maintain the retail sale price of the good  

 

 

ZOIG, ZOCG, ZOSG, ZOUG:: Maintain the Tax rate of the good  

 

 

 

ZRGS : Configure 100%  

 

Step 6: Maintain tax conditions mapping in J_1IEXCDEFN table 

IMG Path: Financial Accounting -> Financial Accounting Global Settings -> Tax on Sales/Purchases -> Basic Settings -> India -> Goods and Services Tax -> General Settings -> Classify Condition Types for Goods and Services Tax

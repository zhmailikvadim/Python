Symptom

For the electronic invoice or invoice printing, XML Version 5.0, this SAP Note contains further enhancements that have resulted from the introduction of the new outpatient flat rates and TARDOC.

Version 4: This version corrects the display of an outpatient flat rate with regard to the sequence number of an outpatient treatment. Up to now, the pay scale language session was incorrectly output. However, the outpatient treatment must be displayed correctly.

Version 6: Services that are entered in the Customizing table CH: Information Services XML Invoice and Print (TNWCHAMBP_XMLSRV) but are assigned to an outpatient treatment without an outpatient flat rate (that is, settlement according to individual TARDOC services) may not appear on the invoice as information services (with price 0).

Other Terms

MEDIDATA 4.0, XML standard 5.0, INXML, electronic invoice, EOXML, electronic invoice for the external order, invoice printing
TARDOC, outpatient flat rates, report or print drugs on invoice

Reason and Prerequisites

Prerequisites:

  • Country Version Switzerland (CH) 
  • Use of XML standard 5.0 (for INXML or EOXML or invoice printing)  
  • SAP Note 3723953 (pre-implementation steps)

Reason:

Adjustments to the new version 5.0 for the (electronic) invoice

Solution

Implement the source code corrections and perform the manual tasks.

This SAP Note contains the following points:

  • Invoice Printing

The printout of the "partners" (physicians, employers) is sorted as follows: Referrer - Employer - Service Provider (service_provider)

  • Electronic Invoice (Correction)

According to guidelines, the invoice:street element is 35 characters long. Previously, if the street name with the house number was longer, the system filled it with a length of up to 46 characters.

Note the following: Depending on the setting, you can enter the street name and the "house number" in two separate input fields for address data, or only in one. Both field contents may be combined for the element invoice:street. This results in the total length.

  • Notification of Additional Drugs Not Billed

Previously, the electronic invoice (and the printout) only included billed services (the invoice items). You can now use the CH: Information Services XML Invoice and Print activity point to inform the system that you are using the services entered there (for example: medication) in the invoice. These must, of course, have been entered beforehand for the case as a prerequisite for the additional specification on the invoice, but do not need to be billable.

You maintain the entries for these information services in Customizing under SAP Healthcare - Industry-Specific Components for Hospitals --> Communication --> Electronic Data Interchange --> Country Specifics --> Country Specifics Switzerland  --> INXML/EOXML -->  CH: Information Services XML Invoice and Print

For more information, see the documentation for this point.

  • Session number of "additionally generated items" (e.g. TMA, information services; item amount 0) (change)

The system no longer takes this session number from the table field NLEI-LFDTSI, but from NLEI-LFDAMB (for printing: column "Gr").

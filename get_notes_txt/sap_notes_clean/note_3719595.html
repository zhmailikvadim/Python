Symptom

  • SAP Analytics Cloud (SAC) Content models fail to load.
  • In the browser console, the following error appears:
    "42014", "BACKEND_ERROR", "Can't read the view : _SYS_BIC:t.UKHNW8.sap.fpa.services.userFriendlyPerfLog/MODEL_NAME\u000a" 

Environment

  • SAP Analytics Cloud 2026.2.3

Reproducing the Issue

  1. Log in to the SAC tenant.
  2. In the left navigation panel, go to Files.
  3. Navigate to:
    System → Common → SAC Content → SAC Content Overview – Models
    or
    System → Common → SAC Content → SAC Content Performance – Models.
  4. Open any SAC Content model, for example: RESOURCE.
  5. Observe:
    • The screen remains blank.
    • A backend error appears in Network/Console.

Cause

SAC Content models are tenant‑specific and not meant to be transported between tenants.
In this case, models were imported from another environment, causing corrupted SAC Content artifacts.

How to verify the issue:

  1. Navigate to
    System → Common → SAC Content → SAC Content Overview – Models
    or
    System → Common → SAC Content → SAC Content Performance – Models.
  2. Check the Changed By column:
    • If it is not empty, the model was altered or imported, which should not occur for SAC Content models.
  3. In Transport → Monitor, confirm that these models were included in an imported package.

Resolution

To restore the SAC Content models correctly:

  1. Delete all affected models and any dependent SAC Content stories (standard or custom).
    • Ensure this is done before the next full system update.
    • Refer to the SAC maintenance documentation for update schedules.
  2. Remove the standard SAC Content stories based on the affected models from Deleted Files.
    • Important: Standard content must be permanently removed from Deleted Files so the update can redeploy them.
  3. After the next full system update:
    • SAC Content models and standard stories will redeploy correctly.
  4. Once the standard content is restored, re‑import or restore your custom stories (if still exist in Deleted Files).

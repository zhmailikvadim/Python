Symptom

The AVC configuration screen is not loaded from harmonized SAPGUI transaction in the FIORI Launchpad.

Disclaimer: "Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental."

Environment

  • SAP S/4HANA Cloud Private Edition

Reproducing the Issue

  1. Open a sales order in harmonized SAPGUI transaction in FIORI Launchpad.
  2. Select an item and perform AVC configuration for this material.
  3. The configuration screen is not loaded.

Cause

Check the "Network" tab in the browsers developers tool (F12).

The metadata query is failing, the target URL looks as follows e.g.:

  • https://<host>/sap/opu/odata/SAP/LO_VCHCLF_INTEGRATION_LEGACY_SRV;o=S4SD/$metadata?....

The error message returned by the backend system for this GET request is e.g.:

  • “System Alias 'S4SD' not assigned to user '<your user>' and service 'ZLO_VCHCLF_INTEGRATION_LEGACY_SRV_0001'” (/IWFND/CM_COS/067)

The OData service setup (transaction /IWFND/MAINT_SERVICE) for the service LO_VCHCLF_INTEGRATION_LEGACY_SRV is set to Processing Mode = “Routing-based”.

But the Alias mentioned in the origin parameter in the URL is not listed under the “System Aliases”.

Resolution

  1. Open transaction /IWFND/MAINT_SERVICE.
  2. Filter for the relevant service.
  3. Add the missing system alias in the section "System Aliases":

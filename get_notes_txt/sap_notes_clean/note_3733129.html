Symptom

  • System Monitoring metrics display the message: "No data load into BW system for this metric since "
  • Metric history is not displayed for affected metrics.
  • Reapplying and activating monitoring templates does not change the behavior and may still show update required.
  • No data load into BW for this metric (no entry in table E2EREP_MAPPING) 

Environment

  • Technical Monitoring
  • Solution Manager 7.2

Reproducing the Issue

  1. Open System Monitoring.
  2. Select any affected managed object and open one of its metrics.
  3. Observe the message "No data load into BW system for this metric since " and missing metric history.

Cause

  • Configuration in the system monitoring For the metrics with the issue, in Template Maintenance -> In the Data Usage tab
  • Configuration in MAI tools - Only selected metrics to BW are enabled

Resolution

Activity - 1

Follow the steps as mentioned:

  1. Go to Transaction Mai_tools
  2. Select Activate Expert Mode
  3. Configuration tab --------> Select Restrict metrics written to BW
  4. Under Enable/disable data volume reduction of monitoring data written to BW
  5. Check/Enable "All metrics are written to BW"
  6. Re-execute the program 
  7. Afterwards, activate the monitoring for your system again:
  8. Solman_Setup -> Application Operations System Monitoring -> 5 Define Scope <Extended System ID> -> 6 Setup Monitoring -> Apply and Activate

For more information check with the KBA: 2900928

Activity - 2

  1. Follow the steps as mentioned:
  1. Go to SOLMAN_SETUP > Application Operations > System Monitoring
  2. In Step 4 Template Maintenance
  3. Select Template Name and in Expert Mode
  4. Metrics tab
  5. Select specific Metric name with issue
  6. In the Data Usage tab maintain correct BW Mapping Rule:
    Note: Only if Data Collector Name of this metric is CCMS MTE, its BW Mapping Rule should be "Standard BI-mapping for CCMS metrics (ONLY Data Provider: CCMS MTE)", otherwise, please use "Default BI-mapping rule".
    (Data Collector Name can be checked in Data Collection tab after you activated Expert mode)
  7. Afterwards, activate the monitoring for your systems again:
    SOLMAN_SETUP -> Application Operations -> System Monitoring -> 5 Define Scope <Extended System ID> -> 6 Setup Monitoring -> Apply and Activate

For more information refer to KBA - 3351651

Activity - 3

Check the infrastructure preparation step - 3 ( Set Up SAP BW ). Make sure all the steps ( 3.1 , 3.2 , 3.3 ) are green.

Activity - 4

Basic configuration Step - 2 ( Scheduled Jobs ). Check if all the jobs running or not.

Activity - 5

System Preparation Step - 5 ( Maintain Technical Users ). Make sure that all the required roles are assigned to the user SM_EFWK. Also check SU53 authorizations for the user.

Symptom

The display of PDFs is not working inside the GUI, only a gray/blank screen is displayed:


The issue happen with GUI 8.00 64 and "Internet Explorer" as browser control
This issue happen after installing Microsoft KB5074109

"Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental." 

Environment

  • SAP GUI for Windows
  • SAP Netweaver
  • ABAP Platforms

Reproducing the Issue

  1. Set Adobe Acrobat Reader as the default PDF viewer on any client machine.
  2. Log in to any SAP system while Internet Explorer browser control in SAP GUI
  3. Open a transaction where print preview or PDFs are used (e.g., QA32, QC20, QC21, QC22, sp01, SFP).
  4. Click on Print Preview using the LOCL output device.
  5. Observe that a blank grey screen appears, and the PDF document does not open.

Cause

The issue happen after installing KB5074109
for More information please check the following article in Adobe community

https://community.adobe.com/questions-12/adobe-acrobat-reader-shows-a-grey-screen-after-installing-kb5074109-when-utilising-the-webbrowser-component-in-windows-1546977

Resolution

We have received final status that the fix is released from Microsoft side.

 https://support.microsoft.com/en-us/topic/march-10-2026-kb5079473-os-builds-26200-8037-and-26100-8037-9c222a8e-cc02-40d4-a1f8-ad86be1bc8b6



--Workaround--
Install a GUI 8.00 32 bit
or 
Uninstall the KB5074109

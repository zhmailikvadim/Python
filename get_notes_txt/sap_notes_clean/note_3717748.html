Symptom

In Create Maintenance Request (F1511A), the fields Main Work Center (MainWorkCenter), Main Work Center Plant (MainWorkCenterPlant), and Maintenance Planning Plant (MaintenancePlanningPlant) are shown and validated as required even when the processing context is not Emergency Work or Unplanned Work.
Users see the mandatory asterisk and may encounter required-field validation in non-applicable contexts.
Note: The Unplanned Work processing context is only relevant from OP2025 SP1.

Other Terms

F1511A, Create Maintenance Request, 'MainWorkCenter', 'MainWorkCenterPlant, 'MaintenancePlanningPlant', 'Notification Processing Context', 'Emergency Work', webapp/view/CreateWorkRequest.view.xml

Reason and Prerequisites

The reason is a program error.
The mandatory evaluation for the listed fields was applied unconditionally, without checking the selected processing context, causing the required indicator and validation to appear outside Emergency/Unplanned Work.

Solution

Apply attached Correction Instruction.
The correction makes the mandatory state of these fields depend on the processing context so they are required only for Emergency Work and (from OP2025 SP1) Unplanned Work.
No manual post-implementation steps are required.

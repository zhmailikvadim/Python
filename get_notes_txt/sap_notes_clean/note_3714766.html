Symptom

Project Management milestone is replicated to Project System as a WBS element.

Environment

  • SAP Portfolio and Project Management
  • SAP Portfolio and Project Management for SAP S/4HANA
  • SAP S/4HANA
  • SAP S/4HANA Cloud Private Edition 

Reproducing the Issue

  1. Create a milestone in a Project Management project with accounting integration to Project System.
  2. Observe that the Project Management milestone is replicated as a WBS element in the Project System project (not as a Project System milestone).

Cause

  1. Accounting Integration to Project System only supports the creation of WBS elements as accounting elements. It does not support the creation of milestones.
  2. A Project Management milestone is technically a special kind of Project Management task. Automatic account assignment triggers the creation of a WBS element in the accounting Project System project when replicating the Project Management milestone task.

Resolution

After the implementation of SAP Note Nr. 3435161 or a Support Package which contains this note, the replication of such milestones to Project System can be prevented by assigning them a task type whose customizing includes flags:

  • NoWBS (technical name NO_FI_INT) to suppress WBS creation, and
  • optionally, MilStOnly (technical name MILESTONE_ONLY) if wishing the task type to be used for milestones only.

See Also

Refer to Customer Influence Improvement Request 304965 which introduced support of these flags in Task Type customizing to cover this scenario.

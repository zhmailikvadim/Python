Symptom

When you try to update an electronic credit memo, which was added through Data Interface (DI) API, it fails with error: [Electronic Transactions - Generation Type] [line: 0]: Field cannot be updated (ODBC -1029) [Message 131-183].

Reproducing the issue

Prerequisites:

You are using a company with India (IN) localization.

Scenario:

  1. Go to Administration → System Initialization → Document Settings, Electronic Documents tab, enable protocol E-Way Bill (other protocols NOT enabled) and change the Default eDoc Generation Type to 'Generate - Later'.
  2. Create a Customer type Business Partner:C100
  3. Create a new location: Haryana for GST transactions, fill GSTIN, set GST Type: 'Regular/TDS/ISD' under Administration → Setup → Inventory → Locations - Setup → Accounting Data tab → GST tab.
  4. Set Country/Region: India, State: Haryana under Locations - Setup → General tab.
  5. Fill PAN No. data under Locations - Setup → Accounting Data tab → General tab.
  6. Go to Administration → System Initialization → Document Numbering, create digital series 'eRIN' for document type 'A/R Credit Memo' and document subtype 'GST Tax Invoice'.
  7. Create a service type A/R Credit Memo with transaction type 'GST Tax Invoice' through DI API.

Sample code:

Dim oRIN As SAPbobsCOM.Documents

oRIN = oCompany.GetBusinessObject(SAPbobsCOM.BoObjectTypes.oCreditNotes)

oRIN.DocType = BoDocumentTypes.dDocument_Service

oRIN.DocumentSubType = BoDocumentSubType.bod_GSTTaxInvoice

oRIN.CardCode = "C100"

oRIN.Series = 85                   ---digital series 'eRIN'

oRIN.Lines.ItemDescription = "Added via DI API"

oRIN.Lines.Price = 100

oRIN.Lines.TaxCode = "IGST@18"

oRIN.Lines.AccountCode = "101000"

oRIN.Lines.LocationCode = "2"     ---code for GST relevant location

oRIN.Lines.SACEntry = -422

oRIN.Add()

        8. Open the credit memo in SAP Business One application and try to update the document, for example the Remarks field.

Expected behavior:

The document is updated successfully.

Actual behavior:

The document update fails with error: [Electronic Transactions - Generation Type][line: 0] , 'Field cannot be updated (ODBC -1029)'  [Message 131-183].

Solution

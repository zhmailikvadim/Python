Symptom

Note 1480677 introduced a less restrictive method how to allow users to copy jobs of other users. However, this method does not work if you want to reschedule a job of another user from transaction SM37.

Reason and Prerequisites

Missing functionality

Solution

After the implementation of this note (or the support packages mentioned herein), you can reschedule the job of another user if you have one of the following authorizations:

a) S_BTCH_ADM, BTCADMIN = Y

b) S_BTCH_JOB, JOBGROUP = <job owner name>, JOBACTION = 'MODI' (see note 1623250)

c) S_BTCH_ADM, BTCADMIN = C
AND
 S_BTCH_JOB, JOBGROUP  = <job owner name>, JOBACTION = PLAN

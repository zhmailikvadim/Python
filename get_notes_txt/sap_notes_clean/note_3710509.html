Symptom

  • Advice request schema validation failed.
  • Path: AdviceRequestHeader.FromDate, error message: must not be null.

Environment

  • SAP digital payments add-on
  • SAP S/4HANA
  • SAP S/4HANA Cloud Public Edition

Reproducing the Issue

  1. Schedule a Payment Advice Processing Job in SAP S/4HANA Cloud Public Edition.
  2. In the "Parameters" section under "Further Selections", set "Last x Days" to 0.
  3. Provide only the "Transaction To" date without specifying the "Transaction From" date.
  4. Execute the job and observe the error.

Cause

  • The issue arises due to incorrect configuration of the date range parameters in the "Further Selections" section.
  • Setting "Last x Days" to 0 does not influence transaction selection, requiring a manual date range to be specified.
  • The absence of the "Transaction From" date alongside the "Transaction To" date leads to the error.

Resolution

  1. Ensure a valid date range is provided in the "Further Selections" section while scheduling the Payment Advice Processing Job.
  2. Choose one of the following options:

     a) Set "Last x Days" to a value of 1 or greater to define a transaction selection range from the past x days until the current day.

     b) Manually specify both "Transaction From" and "Transaction To" dates to define the range.

     Note: Avoid mixing the two options to prevent errors.

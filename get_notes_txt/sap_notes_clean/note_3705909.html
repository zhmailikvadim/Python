Symptom

In the infotype 0038 ("Fiscal Data CH"), the following display problems occur in connection with the "Municipality" field (P0038-GEMND):

  1. Display in infotype: The text field for describing the municipality code does not display the official name of the political municipality. Instead, a list of the localities belonging to that municipality is displayed.

  2. After you import the BFS municipality list: Texts for entries that were imported using the new import program for the BFS municipality list published by ESTV (RPUPLZC2: Import of BFS municipality codes and postal codes, as of 2025) are not displayed on the screen.

  3. Input help (F4): When you call the F4 input help for the "Municipality" field, only the name of the locality associated with the postal code maintained in infotype 0006 is displayed for each entry instead of the municipality names that belong to the proposed municipality numbers.

Other Terms

IT0038, P0038, withholding tax, WHT, municipality, municipality code, localities, T5C1O, T5C1O_EX, F4 help, input help, T5C1O-GTEXT, CL_HRPAYCH_P0038_HELPER, GET_GEMND_T5C1O_EX

Reason and Prerequisites

For 1: The text determination for the municipality field previously accessed the table T5C1O_EX (Directory of Postal Codes and Localities). Since a political municipality can comprise several localities, this led to the display of locality names instead of the higher-level municipality name (which are in the table T5C1O).

For 2: Entries that were imported into the table T5C1O_EX by the new BFS municipality list are saved with PLZTYP = '00', which excluded them from the output.

For 3: The F4 help used the standard search mechanism, which was also based on the table T5C1O_EX and could therefore not determine the suitable municipality names.

Solution

Import the HR Support Package relevant for your release. If you implement the solution in advance, use the Note Assistant to implement the correction instructions relevant for your release. You can find more information about the Note Assistant on SAP Support Portal at https://support.sap.com/notes.

When you implement this SAP Note, the symptoms mentioned above are corrected:

  1. Municipality names are read and displayed directly from table T5C1O.

  2. The filtering of the entries in the table T5C1O_EX is enhanced so that valid entries imported with the BFS municipality list are also taken into account and are therefore displayed in the text field.

  3. The F4 help for the "Municipality" field now displays the suitable municipality names. (This change is currently delivered only for the new CE infotype framework.)

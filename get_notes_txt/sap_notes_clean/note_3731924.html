Symptom

In the log file of a document with 'Failed to Push' status in Monitor Document section of Peppol Exchange app, the error message 'Server returned HTTP response code: 500 for URL' or 'Server returned HTTP response code: 503 for URL' is being thrown. 

Environment

  • SAP ERP
  • SAP S/4HANA Private Cloud Edition
  • SAP Document Reporting and Compliance service

Resolution

This indicates that the SAP ERP has been accessed successfully; however, there appears to be an issue with the Provider web service configuration. To address this, follow these steps:

  1. Follow the instructions in Configuring Provider Service | SAP Help Portal.

  2. Verify the internal transport protocol between the Cloud Connector and SAP. It should be set to either "HTTPS" or "HTTP." Ensure it is consistent with the service configuration mentioned above. Also, check the Transport Level Security section to confirm it is set to "None (http)" or "HTTPS."

  3. Verify that the user in the destination has the necessary credentials, as specified in the "User" section of Setting Up Connectivity | SAP Help Portal.

  4. Ensure that the full path of the Provider service is correctly entered in the Destination's URL, particularly when the URL is not visible in the error message.

  5. Verify if the IP's of EU10/EU11 region from Help page Regions and API Endpoints Available for the Cloud Foundry Environment | SAP Help Portal are whitelisted in your firewall.

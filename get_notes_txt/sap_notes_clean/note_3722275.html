Symptom

Cost center determined through CO substitution is not updated during certain Controlling postings.

Examples include:

    • Cost center is expected to be derived via CO substitution and posted as a statistical cost center on internal order postings.

    • Substitution works correctly for several order-related processes such as:

      • Order confirmation

      • Component consumption

      • Goods receipt for production orders

This behavior can be observed during:

  • Template allocation executed with transaction CPTA or CPTD

  • Revaluation at actual activity prices executed with transaction CON2

Environment

  • Controlling (CO)
  • SAP R/3
  • SAP R/3 Enterprise 
  • SAP ERP Central Component
  • SAP ERP
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA Finance
  • SAP S/4HANA

Cause

In the current SAP system design, CO substitution cannot be used to add additional CO objects for the following processes:

  • Template allocation (transactions CPTA / CPTD)

  • Order revaluation at actual activity prices (transaction CON2)

These applications do not support adding additional account assignment objects through CO substitution.

Resolution

Adding additional CO objects via CO substitution for the above processes is not supported in the SAP standard system.

If this functionality is required, consider submitting an enhancement request through the SAP Customer Influence program ( https://influence.sap.com/ )

See Also

Refer to: SAP Note 0000000011 for information on submitting a development request.

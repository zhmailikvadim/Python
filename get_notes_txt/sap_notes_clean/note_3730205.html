Symptom

The OAuth Client in use is with the Technical User purpose. After successfully logging into Datasphere via the Command Line Interface (CLI), running the command "datasphere spaces list" returns the error: Failed to return a list of existing spaces.

Environment

SAP Datasphere

Reproducing the Issue

  1. Use an OAuth client configured with the Technical User purpose.
  2. Select the roles with the role templates "DW Administrator" and "DW Space Administrator" for this OAuth Client.
  3. Run the datasphere login command and authenticate successfully via CLI.
  4. Run the command "datasphere spaces list".
  5. Observe the error.

Cause

Except for space connections, other space-related commands cannot be executed by the OAuth Client with the Technical User purpose. More information is available here:
Create OAuth2.0 Clients to Authenticate Against SAP Datasphere

Resolution

  • If you would like to continue with a Technical User OAuth client, restrict usage to supported commands only:
    • marketplace (see Manage the Data Marketplace via the Command Line)
    • tasks (see Manage Tasks and Task Chains via the Command Line). Note: It's not supported to use a technical user via the CLI to start task chains containing SAP BW bridge process chains.
    • objects (see Managing Modeling Objects via the Command Line).
    • configuration certificates (see Manage TLS Server Certificates via the Command Line).
    • spaces connections (see Manage Connections via the Command Line).
  • If you need to run datasphere spaces list or other broader CLI commands, create and use an OAuth client with the Interactive Usage purpose:
    Create an OAuth2.0 Client with an Interactive Usage Purpose

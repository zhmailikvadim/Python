Symptom

After upgrading to PCo 15.5.5.6 or higher, OPC UA connections may fail if the ApplicationUri in the server certificate does not exactly match the ApplicationUri in the endpoint configuration. In previous PCo versions, connections could succeed even if the ApplicationUri in the certificate did not match exactly.

Other Terms

Plant Connectivity, PCo, OPC UA, ApplicationUri, certificate validation, EndpointDescription

Reason and Prerequisites

In PCo 15.5.5.6 and higher, strict ApplicationUri validation is enforced for all OPC UA connections:

  • The ApplicationUri in the server certificate must exactly match the ApplicationUri defined in the EndpointDescription.

  • This check is mandatory and cannot be disabled.

  • It ensures compliance with OPC UA security specifications.

Previous PCo versions did not enforce this strict validation, which may explain why connections worked without matching ApplicationUris.

Solution

To resolve the OPC UA connection issue in PCo 15.5.5.6 or higher:

  1. Create a new OPC UA server certificate with the correct ApplicationUri that exactly matches the EndpointDescription.

  2. Replace any existing certificate on the PCo server with this newly created certificate.

  3. Re-trust the new certificate on all OPC UA clients (e.g., PLCs).

After completing these steps, OPC UA connections will succeed, and the mandatory ApplicationUri validation will pass.

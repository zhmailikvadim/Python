Symptom

After upgrading to the JDK 21 Commerce Suite (2211-jdk21.5), executing custom unit tests with the Ant unittests command fails due to an out-of-memory error. This issue occurs specifically when running tests on an AWS EC2 instance but does not appear in local environments or with the previous JDK 17-compatible suite. Below are error messages that may appear in the console during test execution: 

[yunitlauncher] Exception in thread "main" java.lang.OutOfMemoryError: Java heap space
[yunitlauncher] java.lang.OutOfMemoryError: Java heap space
[yunitlauncher] *** java.lang.instrument ASSERTION FAILED ***: "!errorOutstanding" with message can't create name string at ./src/java.instrument/share/native/libinstrument/JPLISAgent.c line: 838
[yunitlauncher] Exception: java.lang.OutOfMemoryError thrown from the UncaughtExceptionHandler in thread "ShutdownHookThread"
[yunitlauncher] Forked test(s) failed with an exception

Environment

SAP Commerce Cloud

Cause

The upgrade to JDK 21.5 changed the default test runner to the JUnit 5-based yunitlauncher. This switch can expand test discovery and instrumentation but might increase memory consumption if the custom tests are not fully aligned with JUnit 5 requirements. Ant’s build process runs in a separate JVM with default memory settings (2GB), which may be insufficient under the new setup when executed on AWS. 

Resolution

To resolve the Out of Memory issue, please follow these steps:

1. Increase Ant JVM Memory: Modify the memory settings for the Ant build process. Update the setantenv.sh script to increase memory allocation:

if [ -z "${ANT_MEM_OPTS}" ]; then
  MEM_OPTS='-Xmx4G' # Adjust this value based on your needs
fi

After making this change, make sure to re-source the setantenv.sh script to apply the updated environment settings.

2. Review and Update Test Configurations: Ensure your custom tests are compliant with JUnit 5 standards to prevent duplicate execution and unnecessary memory usage. Tests not fully adjusted to JUnit 5 may consume more resources.

3. Fix Tests with Large Output: Identify and address test classes that generate large stack traces or excessive output, which can lead to increased memory consumption. Reducing log output can help mitigate this issue.

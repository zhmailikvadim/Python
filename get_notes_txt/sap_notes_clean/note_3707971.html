Symptom

System mapping is not reachable and fails with error "Peer certificate rejected by ChainVerifier".

Environment

  • SAP Cloud Connector
  • SAP Cloud ALM

Cause

The request is sent to UI port of SCC (by default it's 8443), but the UI certificate is not trusted by SCC if the "Determining Trust Through Allowlist" is switched on.

SCC -> Configuration -> On-Premises -> Backend Trust Store

Resolution

  1. Export the UI certificate of SCC from browser, below is an example of Chrome.






    Click on the Details tab of the certificate in the popup, export it.

  2. Import the certificate into SCC -> Configuration -> On-Premises -> Backend Trust Store -> Allowlist -> Add

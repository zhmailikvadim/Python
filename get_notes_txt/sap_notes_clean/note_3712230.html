Symptom

You want to maintain different business partner configurations for each company code with varying addressing schemes and electronic delivery channels.

Other Terms

eDocument, eDoc, eInvoicing, e-Invoicing, electronic invoice, electronic invoicing, e-document, e-Document, e-Docs, e-Doc, e-docs, e-doc, eInvoice, email, Document and Reporting Compliance, electronic document processing, Business Partner Configuration, EDOEUBUPA, EDOEUBUPAV

Reason and Prerequisites

Reason:

You manage business partners across multiple company codes, and you want to ensure accurate and efficient electronic document processing tailored to each business unit's specific requirements.


Prerequisites:

This SAP Note is applicable under the following conditions:

  • You exchange electronic invoices through the Peppol network.
  • You have implemented the prerequisite SAP Note 3724581.

Solution

Import the Support Package specified for your release or implement the attached correction instructions.

As a general rule, SAP recommends that you implement the support package specified for your product version.  If you require a solution earlier, use the SAP Note Assistant to implement the correction instructions. You can find more information about the SAP Note Assistant in SAP Support Portal, under SAP Note Assistant.

Note:

If a pop-up appears during implementation, select all objects, including objects with the message "Object already exists and the system overwrites it." Otherwise, the system does not implement this SAP Note correctly, and accessing the customizing view EDOEUBUPAV under Financial Accounting → Document and Reporting Compliance → Electronic Invoicing → Business Partner Configuration causes syntax errors.

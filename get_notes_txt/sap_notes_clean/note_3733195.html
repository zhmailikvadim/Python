Symptom

Error AA347: "You cannot post to asset in company code … fiscal year …" occurs when saving a new time-dependent interval in AS02 after changing the cost center or profit center.

Environment

  • SAP R/3
  • SAP ERP Central Component
  • SAP ERP
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA Finance
  • SAP S/4HANA

Reproducing the Issue

  1. Run transaction AS02 and open an asset;
  2. Go to the Time-dependent tab and choose More intervals > Add interval;
  3. The current system date falls within a fiscal year that has not yet been opened for the company code;
  4. Enter the From-date for the new interval and change the cost center or profit center;
  5. Choose Save;
  6. Observe message AA347 indicating posting to the asset is not possible in the (current-date) fiscal year.

Cause

Basis condition : FIN_AA_SEGMENT_REPORTING is activated in S/4HANA environment or FIN_GL_REORG_1 is activated in ECC environment.

One of the following flags is set to "ON":
- Segment Reporting (IMG: ORFA -> Integration with General Ledger -> Segment Reporting -> Activate Segment Reporting)
- Organizational Units (IMG: ORFA -> Master Data -> Specify Time-Independent Management of Organizational Units)

When either of these flags is activated, changing the cost center / profit center in the asset master triggers an internal transfer posting. 
Since the transfer posting date defaults to the current system date, if that date falls within a fiscal year that is not open for the company code, message AA347 will be raised.

Notes: 
The "Segment Reporting" flag takes higher priority, meaning this behavior applies even if "Specify Time-Independent Management of Organizational Units" (XZORG) is not active.
For more information, refer to KBA 3592093 - Profit center changes in asset master data triggers a transfer document without XZORG active.

Resolution

  1. Open the fiscal year corresponding to the current system date for the affected company code to enable the transfer posting without encountering error AA347.
  2. Alternatively, perform the master data change when the current fiscal year (as per the system date) is already open.
  3. Refer to KBA 2596921 for detailed guidance on year-end closing procedures.
  4. If a different posting date is required for the transfer document (for example, to align with the interval's start date) based on the customer's own business requirements, consider implementing a substitution to set the desired posting date.
    It is recommended to discuss this option with the local consulting team.

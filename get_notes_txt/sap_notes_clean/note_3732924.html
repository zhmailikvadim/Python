Symptom

The following error is displayed when saving data to a key figure:

Some of the changes that you made to <Number> cells have been rejected, as follows:

  • You can only change the key figures for existing attribute and value combinations. To add key figure values for new attribute and value combinations, create the respective planning objects first. (<Number> Change)

Environment

  • SAP Integrated Business Planning
  • SAP IBP, add-in for Microsoft Excel

Cause

One or more attributes in the planning view are not included in the base planning level of the edited key figure, so the system cannot map the entries to existing planning objects.

Resolution

  1. Log in to the SAP IBP Web UI and open the Planning Areas — Model Configuration application.
  2. Open the affected planning area.
  3. Select the Key Figures tab and locate the impacted key figure.
  4. Open the Base Planning Level for that key figure.
  5. In the Base Planning Level popup, open the Attributes tab.
  6. For each attribute used in the planning view, search and add the attribute to the Base Planning Level.
  7. Also add the attribute to all corresponding disaggregate planning levels (for example, if adding the attribute to WKPRODLOC, also add it to WKPRODLOCCUST).
  8. Reactivate the planning area and verify that the change can now be saved.

Alternative: Remove any attributes from the planning view that are not present on the Base Planning Level; the key figure will then become savable.

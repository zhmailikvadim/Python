Symptom

  • In the "VA01 – Create Sales Document" screen in the SAP Fiori interface, application toolbar for buttons were visible earlier are now missing.
  • Missing tabs include: Sales, Item Overview, Ordering Party, and Characteristics.
  • These buttons are visible in the SAP GUI when SAP Signature Theme is applied.
  • Similar issue happens to VA02, VA03...

Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.

Environment

  • Product: SAP S/4HANA foundation
  • SAP GUI for HTML
  • Fiori Launchpad
  • SAP GUI for Windows

Reproducing the Issue

  1. Check VA01 on SAP GUI for Windows with SAP Signature Theme applies, the application buttons are dislayed in toolbar:
  2. Open VA01 with Webgui, the buttons are missing:

Cause

  • Check VA01 via SAP GUI and apply Fiori theme:

    In SAP GUI Options -> "Visual Design" -> "Theme Settings", select "Quartz Theme" with "Activate SAP Fiori features" checked

  • Further checking the button Function Attributes, the missing fields are intentionally set as hidden for Fiori themes. 

Resolution

  • This is per design by the application that the buttons are hidden when apply Fiori themes.
    Compare the look with SAP GUI applying the same theme setting to confirm: 
    In SAP GUI Options -> "Visual Design" -> "Theme Settings", select "Quartz Theme" with "Activate SAP Fiori features" checked
  • As a workaround, use the context menu: right‑click inside the app to reveal the header options when needed.

*A kernel bug that previously caused buttons to display incorrectly in WebGUI was resolved following a kernel update, which is why these buttons are no longer visible for later release.

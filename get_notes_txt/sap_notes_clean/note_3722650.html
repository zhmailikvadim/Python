Symptom

The buttons to approve, reject, and perform other actions for time-off requests are not appearing in Joule, preventing users from processing requests.

Images/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.

Environment

SAP SuccessFactors Employee Central > Time Off

SAP SuccessFactors Platform > Joule

Reproducing the Issue

  1. Start a conversation with Joule by saying something like, "I'd like to approve time-off requests".
  2. Requests are displayed, but no buttons to approve or decline the requests appear.

Cause

  • The 'Prevent Quick Action Approval of Workflow' permission has been enabled for the approver.
  • When the 'Prevent Quick Action Approval of Workflow' permission has been enabled for the approver, they cannot approve the request directly in Joule. Instead, they have to use the View Details link to open the workflow details page to approve the request.

Resolution

Identify the permission role granting the affected user the 'Prevent Quick Action Approval of Workflow' permission via RBP Troubleshooting, and remove the permission.

Note 

  • Time off requests submitted while the permission was enabled will not display the approve/decline buttons, even if the permission is later removed for the user. This is because the system respects the permissions assigned to the user at the time the time off request was submitted. Please see Prevent Quick Approval for Workflow | SAP Help Portal for more information on this.
  • Time off requests submitted after the permission is removed will display the approve/decline buttons.

Symptom

You are using ABAP Data Integration with SAP SLT and your source system is an S/4 HANA system. You have defined ranges for initial load in LTRS. In case a table in the source system is partitioned it can cause that some or all rows will not be read from the table.

Other Terms

S/4HANA, ABAP Data Integration, DHADM, Access Plan Calculation, partition, ACD, ACP

Reason and Prerequisites

The ABAP-based source system is an S/4HANA 2025 FPS00-SP01 system.

The ABAP-based source system is an S/4HANA 2023 FPS00-SP05 system.

The ABAP-based source system is an S/4HANA 2022 FPS00-SP07 system.

The ABAP-based source system is an S/4HANA 2021 FPS00-SP09 system.

Solution

Implement the corrections in the ABAP-based source system.

Restart the data replication process in SLT for the affected table.

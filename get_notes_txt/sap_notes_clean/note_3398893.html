Symptom

You use SAP Process Orchestration(PO) and REST Receiver channel. You configure the channel to use Apache HTTP client and you would like to use different operations including PATCH.

Other Terms

Apache

Reason and Prerequisites

The current option is to use module parameter "overrideHttpMethod" with value "PATCH".

Your scenario requires dynamic REST Operation determination and overriding with single REST operation doesn't meet the requirements.

Solution

Apply the patch matching your Support Package version as listed in the Support Package & Patches section of this  SAP Note. Follow the instructions described in SAP Note 952402.

After that, it will be possible to use also PATCH without module parameter "overrideHttpMethod" configured. You can configure this using Mapping Source Values to Operation in adapter configuration.

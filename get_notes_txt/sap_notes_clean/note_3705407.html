Symptom

In JDK 17, empty townor street namefields appear blank in the back-office. After upgrading to Commerce Cloud JDK 21, these fields now display the text "null" when empty.

Other Terms

N/A

Reason and Prerequisites

Root Cause: The system has two beans for addressLabelProvider:
A platform-provided bean with the alias "addressLabelProvider".
A bean defined in customersupportbackofficewith the bean id "addressLabelProvider".
In JDK 17, Spring's bean resolution prioritized the alias, so the platform bean was used. In JDK 21, due to changes in Spring's bean searching algorithm, the bean with the matching ID now takes precedence. The backoffice-specific bean was not fully implemented, causing the "null" display issue.
Solution: Remove the duplicate bean definition from customersupportbackoffice-backoffice-spring.xmlto restore the correct behavior.

<bean id="addressLabelProvider" class="de.hybris.platform.customersupportbackoffice.labelproviders.AddressLabelProvider" />

Solution

Fix will be available in SAP Commerce Cloud Update Release 2211-jdk21.8

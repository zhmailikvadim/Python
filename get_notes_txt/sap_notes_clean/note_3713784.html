Symptom

When changing the depreciation method, such as  from ‘0000’ to ‘IND5’, an error occurs: Use a depreciation key without calculation to the day, such as 0000.

Environment

SAP S/4HANA Cloud Public Edition

Reproducing the Issue

1. Open the Manage Fixed Assets app.

2. Select the fixed asset that needs to be changed.

3. Click the "Edit" button.

4. Modify the depreciation method, and an error occurs: Use a depreciation key without calculation to the day, such as 0000.

Cause

The basic problem when switching between daily and non-daily depreciation is the calculation of the expired useful life. A non-daily depreciation (assumed 12 periods are the regular number of periods in the underlying fiscal year variant) asset calculated the expired useful life based on 12 periods, while the daily depreciation asset calculates the expired useful life based on 365 days.

If the asset exists only in open fiscal years, this is no problem. Changing the number of periods from 12 to 365 and vice versa can be updated easily in all asset tables.

However, if the asset has data in fiscal years that are closed, then this is not possible anymore. The asset that is capitalized in fiscal year 2026, then it can change daily to non-daily. If the asset is capitalized in 2023 it depends now. If 2023 is open, the change is possible because the expired useful life at start of 2026 can be changed and updated in the asset tables. 

If 2023 is closed, this is not possible anymore and the error makes sense. Because the useful life might have been calculated based on 365 days in 2023, and 2023 is not changeable anymore. Hence a change to non-daily for such an asset is not allowed anymore. 

Resolution

System behavior.

Keywords

depreciation key, calculation to the day

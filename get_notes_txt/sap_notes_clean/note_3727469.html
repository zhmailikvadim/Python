Symptom

  • A custom program, application or interface uses a function module or other existing Portfolio Management code to create or modify a Portfolio Item.
  • A program issue, error, dump or exception is encountered.

Environment

  • SAP Portfolio and Project Management
  • SAP Portfolio and Project Management for SAP S/4HANA
  • SAP S/4HANA
  • SAP S/4HANA Cloud Private Edition 

Reproducing the Issue

  1. In a custom program or in transaction SE37, use a Portfolio Management function module (e.g. /RPM/ITEM_MODIFY) or other existing Portfolio Management code to create or modify a Portfolio Item.
  2. Observe some kind of issue/malfunction (e.g. the item is created but no corresponding project is created) or encounter a dump or exception.

Cause

  • Function module /RPM/ITEM_MODIFY is not released for use by customers, it is also not used internally in the SAP Portfolio Management standard when changing an item in the PPM frontend.
  • The attempt to use this function module or some other unreleased function module or code for item creation or modification will lead to program malfunction, inconsistencies or other errors, due to the fact that this is not supported.

Resolution

1) In product versions below SAP S/4HANA 2025, Portfolio Management application does not provide any BAPIs/APIs which could be used for item creation or modification via a custom program or application. Item creation/change should be carried out in the Portfolio Management frontend or via program RPM_DX_ITEM.

2) PPM in SAP S/4HANA 2025 provides a new Portfolio Item change/update/delete API which can be used to create and update items. The following links provide information about this API:

  • APIs for SAP Portfolio and Project Management
  • SAP Business Accelerator Hub

See Also

SAP Note Nr. 109533 provides information concerning the risks and restrictions of using of unreleased function modules in customer programs. As explained in the note, issues arising in this context cannot be supported by SAP Support.

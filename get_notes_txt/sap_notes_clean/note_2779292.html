Symptom

  • ASE becomes unresponsive or may show many errors disrupting application activity
  • The ASE errorlog is reporting messages indicating I/O error:

    sddone: write error on virtual disk x block xxxxx:
    sddone: Input/output error
  • A variety of further messages may be seen immediately or some time after
  • These often include error 823:

    Error: 823, Severity: 24, State: 2
    I/O error detected during wait for BUF ... 

  • And may show other errors, for example 69x:

    Error: 694, Severity: 24, State: 10
    An attempt was made to read logical page 'xxx', virtpage 'xxxx' from virtual device '0' for database 'master' (1), object 'syslogins' (33), index 'syslogins' (0), partition 'syslogins_33' (33). The page was not read successfully. You may have a device problem or an operating system problem.

  • These errors may result in zombie / blocking spids, especially when error 6103 is reported:

    Error: 6103, Severity: 17, State: 1
    Unable to do cleanup for the killed process; received Msg 823.
    WARNING: Pss 0x0x00002abab6f906e8 found with open sdes. pspid 12605, psuid 68, pcurdb 71, system table entry 5, sdesp 0x0x00002ab7ff211170, objid 99.

Environment

SAP Adaptive Server Enterprise (ASE) All Versions

Cause

These I/O errors are usually generated in response to a hardware problem that affects device(s) where ASE data is stored. Often the issue is in disk / storage area. ASE cannot run correctly under these conditions.

Resolution

  • There is no fix that can be applied inside of ASE for hardware issues.
  • Sometimes a reboot of ASE and OS may correct a fault, but otherwise the hardware issue needs to be identified and fixed.
  • It may be needed to move database(s) to different storage or create new device(s).
  • Depending on the issue and which database(s) are affected, it may be necessary to load a clean backup of a database.

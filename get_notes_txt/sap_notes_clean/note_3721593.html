Symptom

  • You are running Inventory Optimization in an HPA planning area (based on I_SAPIBP2).
  • The inventory optimization (multi-stage) does not respect the input for minimum safety days of coverage (I_MINSAFETYDAYSOFCOVERAGE).
  • There might be some change in total safety stock (see key figure I_DELTAMINMAXSAFETYSTOCK), but the min requirement is not met.
  • The product/location in question is NOT a purely customer-facing location.

Environment

SAP Integrated Business Planning for Supply Chain (SCM-IBP) 2602 prior to HFC06, 2511 (all) and 2508 (all) 

Reproducing the Issue

  1. At a hybrid node, set a value for the minimum safety days of coverage in the I_MINSAFETYDAYSOFCOVERAGE attribute in the Location Product md and save.
  2. Run the multi-stage inventory optimization and calculate inventory components functions.
  3. Observe the output key figure "Safety Stock in Days of Coverage" (I_SAFETYSTOCKDAYSOFCOVERAGE) showing a value significantly lower than the input value.

Cause

  • There is a defect in the Multi-Stage Optimization function that defaults the value of I_DEMANDTYPEFORCOVERAGE to C for Customer-Demand Only when the attribute value is null or any value other than "T" or "C". 
  • Attribute I_DEMANDTYPEFORCOVERAGE determines which demand type is used when converting Min/Max Safety Stock Days of Coverage constraint into quantity.
  • The Calculate Inventory Components function defaults the value to T for Total Demand.
  • The Multi-Stage function calculates SS to cover only the IOFCST portion, while the Calculate Inventory Components calculates I_SAFETYSTOCKDAYSOFCOVERAGE using SS from Multi-Stage while calculating PROPAGATEDDEMANDMEAN (PDM) that is based on total demand.

Resolution

The fix is being worked on and is targeted for 2602 HFC06, which is scheduled for weekend of March 14/15, 2026 for all customer systems.

A workaround does exist:

  1. Set the attribute I_DEMANDTYPEFORCOVERAGE in the Location Product master data to "T" (total demand).
  2. Ensure that the attribute I_SAFETYSTOCKPOLICY does not have a value of "I."
  3. Run the inventory optimization (Multi-Stage and Calculate Inventory Components) again to verify that the output respects the minimum safety days of coverage input.

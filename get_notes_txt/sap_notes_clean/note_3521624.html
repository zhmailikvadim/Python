Symptom

Master Data is marked for deletion – And was already removed from Integration Model from External System:

In the attempt to delete master data from a planning area based on SAP7F (Flexible Model) using:

  • Manually Manage Master Data App.
  • Application Job Order-Based Planning: Delete Master Data

It failed to delete this master data due to dependencies.

Environment

  • SAP Supply Chain Management (SCM)
  • SAP Integrated Business Planning (SCM-IBP)

Cause

There are vetoes during master data deletion.

These vetoes exist and are important to avoid inconsistencies, master data shall be deleted in cascade.

Resolution

To delete a material, delete the following dependent objects first:

  • Transactional data
  • Production data structures
  • Transportation lanes

To delete a work center, delete the dependent production data structures first.

To delete a customer, delete the dependent sales orders first.

To delete a plant, first delete all the transactional and master data connected to this plant.

To delete a vendor, first delete transportation lanes and dependent transactional data such as purchase requisitions or purchase orders.

The deletion process is explained in the below help page:

SAP Help | Deletion of Master Data in the guide Main Steps

 

There can be dependent master data or transactional data that veto this master data to be deleted, below steps can help to check/remove these vetoes:

  • Use the App Supply Chain Dependencies to check Master Data.
  • Use Orders  App to check if there are existing transactional data references.
  • After marking master data for deletion, run a Supply Planning Run job in order to remove existing orders and pegging.
  • If there are fulfilled order (with no open quantities) that are available only under Orders App, run the Housekeeping job (E.g Order-Based Planning: Delete Fulfilled Orders) for this scenario.
  • In case of "Loc. prod. in Version BASELINE referenced by network data in BASELINE", if there are no existing orders, run the planning job with Gating factors and Supply Usage = Yes as internal gating orders or detailed pegging might veto the master data and the detailed pegging is only updated by this step/job.

Symptom

  • In transaction EG88, when creating a periodic replacement list, no records are written to table EGTUR.
  • The periodic replacement list remains empty.
  • Error message EG675 "No resources available for parellel processing" indicating no resources/free dialog processes are available for parallel processing and the run is terminated.

Environment

  • SAP for Utilities as Part of SAP ERP, release IS-UT 600 and higher.
  • SAP S/4HANA, release IS-UT 801 and higher.

Reproducing the Issue

  1. Open transaction EG88.
  2. Start creation of a periodic replacement list.
  3. Observe that message EG675 is shown and no entries are written to table EGTUR.

Cause

  • Parallel processing prerequisites are not met because the RFC group EGTUR might not defined in SM59.
  • There are insufficient free dialog work processes and/or the operation mode has too few dialog processes relative to batch processes, preventing parallel processing (EG675).

Resolution

  1. In transaction SM59, choose Extras -> RFC groups, then define and activate the RFC group EGTUR.
  2. Ensure that more dialog processes than batch processes are available during the EG88 run; coordinate with the system administration team to provide sufficient free dialog processes.
  3. If needed, define and activate a new operation mode to provide enough dialog processes for the periodic replacement run.
  4. Rerun EG88 to create the periodic replacement list and verify that entries are written to EGTUR.
  5. If the issue persists, refer to SAP Note 537391 for detailed guidance on parallel processing for periodic replacement.

See Also

  • SAP Note 537391 - Parallel processing during periodic replacement
  • Refer to: Product documentation path Utilities -> Device Management -> Inspection -> Periodic replacement list -> General information about the periodic replacement list.

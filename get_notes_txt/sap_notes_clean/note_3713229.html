Symptom

  • It was noted that upgrading to ASE 16.0 SP04 PL01 or higher "sp_version" is showing "OLEDB MDA" Scripts along with "ADO.NET" MDA Scripts. 
  • If it is a new installation on ASE 16.0 SP02, "sp_version" only shows "OLEDB MDA" Scripts.
  • However, if it's downgraded from ASE 16.0 SP04 PL01 or higher,  "ADO.NET MDA" Scripts remaines in "sp_version".
  • For example:

      • ASE downgraded  from ASE 16.0 SP04 PL01 to ASE 16.0 SP02 PL04

        sp_version
        go

        Script Version Status
        ------------------- ---------------------------------------------------------------------------------------------------------------------- --------
        ADO.NET MDA Scripts 16.0.04.07.1020/Fri Mar 21 UTC 07:54:25 2025 Complete
        ODBC MDA Scripts 16.0.02.04.1016/Fri May 13 UTC 03:01:05 2016 Complete
        OLEDB MDA Scripts 16.0.02.04.1016/Fri May 13 UTC 03:01:05 2016 Complete
        ...
        ...
        (8 rows affected)

      • Whereas, a New installation ASE 16.0 SP02 PL04, shows:

        sp_version
        go

        Script Version Status
        ------------------- ---------------------------------------------------------------------------------------------------------------------- --------
        ODBC MDA Scripts 16.0.02.04.1016/Fri May 13 UTC 03:01:05 2016 Complete
        OLEDB MDA Scripts 16.0.02.04.1016/Fri May 13 UTC 03:01:05 2016 Complete

Environment

  • SAP Adaptive Server Enterprise 16.0 
  • SDK for SAP Adaptive Server Enterprise 16.0 

Cause

  • Display issue as "ADO.NET MDA" script is not in ASE 16.0 SP02.
  • Similar issue as KBA 3118310

Resolution

Remove "ADO.NET MDA" Scripts information from system table manually with the method provided in KBA 3677272.

sp_configure 'allow updates',1
go
delete from master.dbo.sysattributes where class=23 and object_cinfo='ADO.NET MDA Scripts'
go
sp_configure 'allow updates',0
go

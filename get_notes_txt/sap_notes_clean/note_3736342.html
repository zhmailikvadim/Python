Symptom

Clarification on security parameter "Identifier First – Return Obfuscated Identifier Values" behavior and API impact in login flows

Environment

  • SAP Customer Data Cloud
  • Customer Identity

Reproducing the Issue

  1. Enable "Identifier First – Return obfuscated identifier values" in the console (Identity → Security → Authentication → Identifier First).
  2. Start an Identifier First login flow with a user with multiple loginIDs.
  3. Observe that the identifiers returned for display are obfuscated (for example, partially masked emails or phones), and not full values.

Cause

Resolution

  1. Enable the "Identifier First – Return obfuscated identifier values" setting to mitigate login identifier exposure when using Identifier First flows and when displaying multiple loginIDs to a user.
  2. Use Identifier First-compatible APIs (for example, accounts.auth.getMethods) and verify that only obfuscated identifiers are returned for display instead of full emails or phone numbers.
  3. Note the scope: this setting affects only APIs used during the Identifier First flow and does not apply the broader behavior of "Protect Against Login Identifier Harvesting" across all API integrations.
  4. Adopt this setting as a recommended practice when "Protect Against Login Identifier Harvesting" cannot be enabled due to functional constraints, to reduce identifier exposure while maintaining existing flows.

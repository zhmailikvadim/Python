Symptom

Inquiry about potential impacts of deactivating user accounts in Boomi who previously owned and managed processes (for example: contract ended with implementation partner).

Environment

  • SAP SuccessFactors HCM Suite
  • Boomi

Reproducing the Issue

  1. Deactivate a user account in Boomi who previously managed processes.
  2. Observe whether there is any impact on currently running or scheduled processes.

Cause

There is no mapping in Boomi between user accounts and processes.

Resolution

  1. Ensure administrative privileges are available to delete user accounts.
  2. Confirm the user account being deleted is not the account of the administrator performing the action.
  3. Proceed with the deletion of the user account.
  4. Note that this action has no impact on currently running or scheduled processes.

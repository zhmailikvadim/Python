Symptom

In transaction HDM_CUST while trying to add document types for activating Harmonized Document Management (HDM), warning is raised that makes users aware that:

"Dropdown list is not supported in view clusters. - Message No. SV538

Environment

SAP S/4HANA

Reproducing the Issue

  1. Transaction HDM_CUST
  2. Select Attachment Object Type (i.e. PURCHASECONTRACT)
  3. Select "Activate Document Frameworks" 
  4. Select Framework to be activated (i.e. DMS, Archive Link) 
  5. Select "Maintain Default and Restricted Document Types
  6. Error message pops up: "Dropdown list is not supported in view clusters" - Message no. SV538

 

Cause

The reason for this warning message in the standard design here is to make users aware that no additional document type can be added manually here using the dropdown function. 

At the node "Maintain Default and Restricted Document Types" the entries are read by the system automatically and there is no need to add document types manually. So it is granted that all document types listed here have the required settings in their customizing transactions as explained in the "Solution" part of this KBA.

For the DMS framework the system reads all document types that have the business object maintained as object link in transaction DC10.
For the GOS framework the system normally uses the document type GOS by default. 
For the ArchiveLink framework the system considers the connection data between object type and document type in transaction OAC3.

Resolution

To get another document type into the list shown in HDM_CUST transaction you can do the following:

  • For the DMS framework, go to transaction DC10 and define a link for the document type and object type under node "Define object links"
  • For the ArchiveLink framework, go to transaction OAC3 to maintain an entry for the object tpye, the document type and the content repository.
  • For the GOS framework the document type GOS is the default one

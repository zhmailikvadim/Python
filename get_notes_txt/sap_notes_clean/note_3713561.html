Symptom

Following issues in the program DMS_KPRO_CONVERT_BACKGROUND are addressed with this note:

1. When running this program in background mode the conversion ends up in an error. This is because, always the FTP-destination is set to SAPFTP. But in background mode the FTP-destination needs to be set as SAPFTPA.

2. Also the logic to delete file from presentation server is removed because, in background mode, SAP system has no connection to user's presentation server.

3. When the report is running in background mode, after the files are migrated, before checking in the files to content server, the FTP and HTTP destinations needs to be set to SAPFTPA and SAPHTTPA

Other Terms

 DMS_KPRO_CONVERT_BACKGROUND, FTP_DESTINATION, SAPFTP, SAPFTPA, FTP_CONNECT, LCV120F06

Reason and Prerequisites

Program error

Solution

Implement this SAP note

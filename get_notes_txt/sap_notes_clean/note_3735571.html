Symptom

If a RecipientBankRoutingID is passed to the class CL_FSTB_PAYM_TMP_VALIDATION during creation or update of a payment template, without passing a RecipientBankCountryCode the RecipientBankRoutingID is not checked. This has now been corrected.

Other Terms

Payment Templates, Validation, RecipientBankRoutingID, CL_FSTB_PAYM_TMP_VALIDATION

Solution

Import the relevant Support Package that is specified for your release. As a manual advance correction, you can implement the attached correction instructions using the transaction SNOTE. There are no manual activities.

Symptom

When the “Stop At” field in the Workflow Group Scheduling Day Plans is not configured, the Workflow Group will not be able to calculate the next execution time on the day when Daylight Saving Time starts.

This will be fixed in upcoming versions of MediationZone and Usage Engine Private Edition 

Environment

MediationZone 9.3.3.0 and above,

MediationZone 9.4.1.0 and above,

MediationZone 9.5.0.0 and above, PE 5.1.0 and above,

PE 5.2.0 and above 

Cause

Resolution

As a workaround, modify the Day Plan in the workflow group Scheduling to set values in the “Stop At” and “Repeat Every” fields.

For example:

Existing Workflow Group configuration

Day: Every Day
Start At: 02:00:00
Stop At: Empty
Repeat Every: Empty


After modification

Day: Every Day
Start At: 02:00:01 (add 1 second)
Stop At: 02:01:00 (set it 59 seconds after the Start At)
Repeat Every: 60 seconds

Note

If the Platform were restarted on the day when Daylight Saving Time starts, and the Workflow Group gets an error message like "No group with key xxxx was found.", remove the Day Plans in the Workflow Group Scheduling and save the Workflow Group Configuration.

After that, re-add the Day Plans with values in the “Stop At” and “Repeat Every” fields as described above.

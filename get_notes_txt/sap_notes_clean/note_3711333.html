Symptom

When using "Search Contract by Cointainer" within the business role /WATP/EWA_IC (Waste & Recycling) in interaction center, you can confirm the found search result.

If the selected business partner differs from the previously confirmed one, the current business partner is unconfirmed and the chosen one is confirmed.

This switch does not necessarily end a running interaction. Upcoming messages – like “The interaction record is not reverted to reset Business Partners” - reflect this behavior.

Especially, if an interaction record was explicitely saved before, this record is kept for further confirmed business partners when switched by using methods CL_CRM_EEWA_IC_CONTEXTHELPER=>CL_UNCONFIRM_BP.

Other Terms

/WATP/CL_CWC_QUERY_SRC_RE_IMPL, /WATP/CM_NOTE_3711334_SWITCH_BP, /WATP/CL_CRM_IC_CONTEXTHELPER

Solution

Implement the correction instructions. Please consider the manual tasks you will be reminded during implemention.

The note introduces a new method /WATP/CL_CRM_IC_CONTEXTHELPER=>END_INTERACTION that is now called before switching to a new business partner. So a new interaction (record) is opened for further processes.

As ending an interaction is a heavier intervention than only un/confirming business partners, you might notice, that - as an example - the page history is reset after starting a new interaction. It is the same behavior as if you explicitely press the "End"-button in the ic-toolbar to end a session.

The implementation uses parameter IC_MS_1TAB_LOG_LINK (SAP Customizing Implementation Guide -> Service -> UI Framework -> Technical Role Definition -> Define Parameters) to ensure navigation to the overview page, when a new business partner is confirmed. Its value is set to the logical link of the used overview page (here: /WATP/OVBP). Please consider that fact, when using your own Profile definition in your business role (Function Profile ID : PARAMETERS).

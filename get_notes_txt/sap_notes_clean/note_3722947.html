Symptom

In the process "Meter Point Takeover for Electricity (New MOS View)
 "(DE_C_WIM_MPT), step 0200 fails to create the IFTSTA-21010 business message and returns a DP error.
This issue is relevant for the market role: New MOS.

Other Terms

WIM, DE_C_WIM_MPT, DE_B_WIM_MPTKO_MOSN,IFTSTA-21010, completion date, /US4G/CL_EXCP_PROC_SOLVE_WIM~MPT_DL_OVER_DUE, metering point takeover, DP error

Reason and Prerequisites

The error occurred because the completion date field wasn't filled, preventing the system from creating the IFTSTA-21010 message.

Solution

Implement the corrections of this SAP Note or import the relevant support package.

After implementing this SAP Note, the system fills the completion date field correctly and sends the IFTSTA-21010 message successfully. You need to reset the process and trigger a new transfer document from the business system. You can verify the fix by retesting the process from the back end and checking that the IFTSTA-21010 message is created successfully.

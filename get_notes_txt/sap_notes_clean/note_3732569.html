Symptom

  • Error: "The range of the statement to be passed to the database is too wide" (Message No. SODQ666).
  • Usually the dtp fails while filling the dtis table.
  • No logs are created in odqmon (the request does not reach odqmon).
  • Related symptom: DBSQL_STMNT_TOO_LARGE may occur when displaying or executing a dtp.

Environment

  • SAP BW/4HANA
  • SAP NetWeaver 7.5

Reproducing the Issue

  1. Open transaction RSDTP.
  2. Select the affected DTP that extracts from ODP (extractors).
  3. Start a delta run (parallel extraction may be active).
  4. Observe that the run fails with SODQ666 “The range of the statement to be passed to the database is too wide”, and the request does not appear in ODQMON.

Cause

Resolution

  1. Run report RSDTIS_CLEANUP to correct potential DTIS inconsistencies. Refer to: RSDTIS_CLEANUP SAP Help and SAP Note 3037467 - DTIS overview and important notes.
  2. If the issue persists, implement: SAP Note 3598234 - DBSQL_STMNT_TOO_LARGE error when displaying or executing DTP.
  3. Run Z_SAP_BW_NOTE_ANALYZER and implement all missing corrections listed in: SAP Note 2603241 - Important notes for BW transformations with SAP HANA and ABAP.
  4. Re-activate the involved transformation using report RSDG_TRFN_ACTIVATE. Refer to: SAP Note 2965878 - RSDG_TRFN_ACTIVATE.

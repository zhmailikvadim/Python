Symptom

You are using the Fiori app 'Migrate Your Data' and the 'SAP S/4HANA Migration Object Modeler' to migrate data to SAP S/4HANA.

After an SAP S/4HANA upgrade, you choose the action 'Prepare' in the Migrate your Data app, or you generated a migration object in transaction LTMOM.
However, errors similar to the following occur:

  • DMC_RT_GEN 005/1LT/*
  • DMC_RT_MSG 000The statement "PS ENDFORM" is not expected. A correct similar statement is "ENDFORM"

The error message belongs to a created function module that contains syntax errors.

Other Terms

Migration Cockpit, staging table, staging approach, data migration, simulation, Migrate your Data App, LTMOM, Migration Object Modeler, Transport, DMC_RT_GEN 005,

Reason and Prerequisites

You are using:

SAP S/4HANA
 • Migrate Your Data - Migration Cockpit (migration approach 'Migrate Data Using Staging Tables')
 • Migration Object Modeler

SAP S/4HANA Migration Cockpit functionality is used on one of the following releases:

  • SAP S/4HANA 2022 FPS00 - SPS07
  • SAP S/4HANA 2023 FPS00 - SPS05
  • SAP S/4HANA 2025 FPS00 - FPS01

Solution

1. Implement the attached correction using transaction SNOTE.

2. Create a workbench transport request for your system landscape in the development system.

3. Execute report DMC_REPAIR_CONTENT_UPDATE_DATE without simulation mode (ensure that the 'Test' checkbox is not selected) and enter the newly created transport request.

4. Release the transport request and propagate it across your system landscape.

5. Update the migration object where the error occurred, either in the Migrate your Data app or in transaction LTMOM.

6. Repeat the relevant action for the migration object – either choose the action 'Prepare' in the Migrate your Data app, or generate the migration object in transaction LTMOM.

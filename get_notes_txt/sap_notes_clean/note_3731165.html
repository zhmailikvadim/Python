Symptom

  • Runtime error when executing Readiness Check:
    Runtime Errors         SYNTAX_ERROR
    ABAP Program           SAPLRS_B4HANA_CODESCAN
    The current ABAP program "RC_BW_COLLECT_ANALYSIS_DATA" had to be terminated
    The program "SAPLRS_B4HANA_CODESCAN" is not Unicode-compatible, according to its program attributes." 
  • Checking program SAPLRS_B4HANA_CODESCAN the following error occurs:
    The program "SAPLRS_B4HANA_CODESCAN" is not Unicode-compatible

Environment

SAP NetWeaver

Cause

Manual steps and/or the correction of SAP Note 2876024 has not been implemented correctly. 

Resolution

  • De-implement SAP Note 2876024.
  • In case the BW/4HANA Starter Add-on is not installed, have a look at SAP Note ##3103162 and execute the report as described and implement the Note. 
    The Starter Add-on is necessary for INPLACE SAP BW/4HANA conversion, see SAP Note 2383530.
  • Follow ALL manual steps described in the SAP Note.
    • Execute report ZNOTE_2876024 before implementing the correction.
    • "Activate anyway" when only an errors occurs concerning structure/table RSB4HANAXREFTLOG.
    • Ensure to select all the objects, also the yellow objects.
    • While implementing the note, follow the below steps if you get the message "Not all corrections applied; see change display" for the object REPS LRS_B4HANA_CODESCANTOP.
        • Stop the implementation of note.
        • In SE80, enter the function group as RS_B4HANA_CODESCAN and display.
          Edit the Include LRS_B4HANA_CODESCANTOP and clear the code of this Include. Activate the object in spite of the syntax errors. The syntax error will get corrected after implementation of note.
        • Continue implementation of note.
    • Execute the post manual steps.

Details are available in SAP Note 2876024.

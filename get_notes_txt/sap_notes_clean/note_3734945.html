Symptom

In the Data Workbench template, custom KUT fields created in the business object display their technical names instead of their maintained labels.

Environment

SAP Cloud For Customer

Reproducing the Issue

  1. Create custom KUT fields in the business object and maintain UI labels for these fields.
  2. Generate a Data Workbench template for the business Object.
  3. Observe that the custom KUT fields appear with their technical names instead of the maintained labels.

Cause

  • Data Workbench uses the UI label (sap:label) for KUT fields; if no UI text is maintained, the technical name is presented.
  • Multiple KUT fields having identical label text may lead to issues displaying labels.
  • Some KUT fields were maintained such that the UI label equals the technical name; thus the template shows the technical name by design.

Resolution

  1. Review the UI labels of the affected KUT fields in the business object and confirm what is set as the label (avoid setting the label equal to the technical name unless desired).
  2. If two or more KUT fields share the same label text, rename one or more labels so each field has a unique label.
  3. If any field lacks a UI label, maintain the desired label text.
  4. Save and activate the KUT changes, then reactivate the service.
  5. Regenerate the Data Workbench template for the business Object and verify that the fields now display the intended labels.

Symptom

  • Posting a billing document (release to accounting) via VF02/VFX3 leads to the dump
  • Dump TSV_TNEW_PAGE_ALLOC_FAILED in FUNCTION=CO_DOCUMENT_REVERSE_PRECHECK with DATA=LT_COEPL 
  • Termination occurs in program/include SAPLKAID, in REVERSAL_ORG_CODOC_SELECT (include LKAIDF1X, around line 48)
  • ST22 indicates SQL_CAUGHT_RABAX preceding the memory dump

Environment

•Controlling (CO)
•SAP ERP Central Component
•SAP ERP

Reproducing the Issue

  1. Start transaction VF02 (or VFX3).
  2. Select a billing document.
  3. Choose “Release to Accounting”.
  4. Observe that the system terminates with dump TSV_TNEW_PAGE_ALLOC_FAILED

Cause

 Memory overflow occurs when processing large billing documents during reversal operations. 

Resolution

Implement SAP Note 3707089 - The real reversal reacts non as expected, (composite) which includes fixes from SAP Notes 3691087, 3674435, and 3660858.

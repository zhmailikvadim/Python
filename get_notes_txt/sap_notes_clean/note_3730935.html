Symptom

Tracking events for Airfreight are failing with error: { "message": "Error while sending request to Tracking Service. Status Code: 400 BAD_REQUEST, Response: JSON parse error: Cannot deserialize value of type `com.sap.lbn.trackingservice.domain.input.generic.TransportationMode` from String \"04\": not one of the values accepted for Enum class: [01, 03]" } 

Cause

The standard tracking capabilities in Freight Collaboration only supports Road (03) and Ocean (01) transport modes. The system fails to deserialize the request because the "04" code for Air transport is not part of the standard FC tracking service's.

Resolution

You can only report tracking events for Road (03) and Ocean (01) transports when using the standard tracking service in Freight Collaboration. According to the standard code lists for the network, the code "04" is designated specifically for Air transport.

To track airfreight events, you must utilize the advanced tracking solution provided by the network Global Track and Trace.

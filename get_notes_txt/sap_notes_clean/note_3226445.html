Symptom

If a new change status is delivered for an insured risk that is already assigned to a prior cession group, the system checks whether the time of the respective delivered risk period overlaps with the existing prior cession group assignment. In this case, the risk period must either be completely assigned to the prior cession group or not assigned to it at all. Otherwise, the system issues an error message.

For prior cession groups for underwriting years, you want the system to perform a check only against risk assignments with the same underwriting date as the delivered change status. However, the system currently checks all assignments whose assignment period affects the change status, regardless of the underwriting date.

Reason and Prerequisites

For an insured risk with an assignment to an underwriting year prior cession group, change statuses are delivered that affect the assignment to the prior cession group.

Solution

History:

Version 1 from Tuesday, July 19, 2022: New

Implement the source code corrections.

Solution:

When checking the full assignment of a change status of an insured risk, the system takes into account only assignments with the underwriting date of the delivered change status for underwriting year prior cession groups.

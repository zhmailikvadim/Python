Symptom

Symptom

You have maintained a purchasing info record for the material and the purchasing group is also maintained in it. However, after MRP run, this purchasing group is not copied to the purchase requisition generated. Instead, the purchasing group in the material master data is copied.

Environment

SAP S/4HANA Cloud Public Edition

Cause

It is standard behavior that MRP will take the purchasing group in the material master data instead of the one in the purchasing info record.

Resolution

Use BADI PPH_MRP_PURREQ_CFL to implement custom logic to modify the purchasing group assignment during the MRP run.

The CDS view to retrieve the purchasing group data from the purchasing info record is I_PURGINFORECDORGPLNTDATAAPI01.

Keywords

MRP, purchase requisition, purchasing info record, purchasing group, material master, BADI, PPH_MRP_PURREQ_CFL, public cloud, CDS view, I_PurgInfoRecdOrgPlntDataApi01

Environment

  • Human Capital Management (HCM)
  • SAP ERP
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP SuccessFactors Employee Central Payroll
  • SAP S/4 HANA

Reproducing the Issue

  1. Run PUOC_XX transaction (XX means country grouping) and input Personnel number/id.
  2. Under 'Payroll' tab, select off-cycle reason with payroll type B.
  3. Click 'Start payroll' button and below error happens.

Cause

The payment date of the off-cycle run with payroll type B is not correct.

Example:

  • For pernr 1811, the last payment day in t-code PC_PAYRESULT is 28.09.2012.
  • The next payment date is 29.10.2012 based on table T549S.
  • If the current payment date of the off-cycle run is set to 28.09.2012 or 30.10.2012, then Message no. HRPAY99CALC716 will occur.  

Resolution

The off-cycle run with payroll type B configured in table T52OCR is only allowed when

  1. The current payment date is greater than the last payment date.
  2. The current payment date is less than or equal to the next payment date.

In above example, the payment date of the off-cycle run with payroll type B should be set from 29.09.2012 to 29.10.2012, such as 28.10.2012.

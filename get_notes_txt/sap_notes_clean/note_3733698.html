Symptom

  • Deployment with migrate data gets stuck after updating to patch version 2211-jdk21.6.
  • In the logs you observe as below:
    Connect timed out. Verify the connection properties. Make sure that an instance of SQL Server is running on the host and accepting TCP/IP connections at the port. Make sure that TCP connections to the port are not blocked by a firewall

    com.microsoft.sqlserver.jdbc.SQLServerException


Environment

Product: SAP Commerce Cloud

Reproducing the Issue

  1. Update the manifest to use commerceSuiteVersion 2211-jdk21.6.
  2. Start a deployment with migrate data.
  3. Observe that the deployment remains in the data migration phase for significantly longer than usual and appears stuck, with connectivity timeout messages in logs.

Cause

Patch 2211-jdk21.6 enables cleanup cron jobs by default; large historical data causes purge operations to run for a long time during migration, leading to apparent stalls and timeouts.

Resolution

  1. Temporarily disable cleanup cron jobs (for example, OAuth-related cleanup) before starting the migrate data deployment.
  2. Run the deployment with migrate data.
  3. After migration completes successfully, perform the cleanups manually to reduce historical log/media data.
  4. Configure dedicated maintenance cron jobs and schedules to routinely purge log entries and associated files, preventing excessive growth and future migration delays.

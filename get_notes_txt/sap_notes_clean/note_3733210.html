Symptom

  • When attempting to create or click on service keys for the HANA Cloud database in the Cloud Foundry environment, only the "{}" symbol is displayed instead of a populated service key.
  • The behavior occurs when trying to create a service key for the hana-cloud-connection service (instead of hdi-shared). For Cloud Foundry, a JSON payload is required during service key creation.

Environment

  • SAP HANA Cloud

Reproducing the Issue

  1. Log in as an admin user to the Cloud Foundry environment.
  2. Attempt to create or open service keys for the HANA Cloud database service instance.
  3. Observe that only "{}" is displayed and no service key content is generated.

Cause

Creating a service key for the hana-cloud-connection service plan (which links the database to the data lake) does not generate a service key by design;

Additionally, in Cloud Foundry a JSON payload must be provided when creating service keys.

Resolution

  1. Verify the selected service instance and plan. If the plan is hana-cloud-connection, note that no service key is generated by design for this plan.
  2. To obtain a service key, create it for the HANA database/HDI service using the hdi-shared plan.
  3. When creating the service key in Cloud Foundry, provide the required JSON parameters as described in the product documentation.
  4. Refer to: SAP HANA Cloud – Database service keys (documentation) for the required parameters and examples.

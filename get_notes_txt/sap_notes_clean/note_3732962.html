Symptom

When releasing a billing document to accounting in App Change Billing Document (VF02) the system raises error “Tax Account could not be determined for YCOA/&/&/&/&” Message no. FOT_TXA020

"Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental."

Environment

SAP S/4HANA Cloud Public Edition

Reproducing the Issue

  1. Open App Change Billing Document (VF02).
  2. Display a billing document number
  3. Click Release to Accounting
  4. Observe the error “Tax Account could not be determined for YCOA/&/&/&/&” Message no. FOT_TXA020

Cause

The tax account is not determined in the configuration activity Automatic Account Determination (Configuration Activity ID 100297).

Resolution

1. Open configuration activity Automatic Account Determination (Configuration Activity ID 100297).

2. For the relevant Chart of Accounts (for example, YCOA) and tax transaction key (for example, PIS), maintain the tax account assignments for the tax code and the Company Code Country/Region and Tax Country/Region combination shown in the error for example BR/BR.

3. Ensure the assigned G/L accounts exist in the chart of accounts, are valid for posting, and meet any required debit/credit configuration for the tax transaction.

4. Save the configuration and attempt Release to Accounting again from App Change Billing Document (VF02).

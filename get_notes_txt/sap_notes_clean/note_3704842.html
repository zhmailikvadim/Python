Symptom

  • The record.evaluate component in a dataflow does not route to the error branch when "return null;" is used; instead, it continues to the success branch.
  • The user seeks a method to send records to the error branch without causing the dataflow to fail.

Environment

  • Customer Data Cloud
  • Platform Extensibility (IdentitySync, Webhooks, Extensions)

Reproducing the Issue

  1. Create a dataflow in SAP Customer Data Cloud.
  2. Use the record.evaluate component within the dataflow.
  3. Implement the script with "return null;" to attempt routing to the error branch.
  4. Observe that the record continues to the success branch instead of the error branch.

Cause

  • The record.evaluate component is designed for data transformation and does not inherently support routing records to the error branch.
  • Returning null or undefined is considered a successful operation, and the record proceeds through the success branch.
  • Throwing an exception stops the entire dataflow, which is not suitable for routing individual records.

Resolution

  1. Use an error handler script to manage record routing after the record.evaluate component.
  2. Implement the following example code within the script:

function process(record, ctx, logger, next, error) {error.accept(record, record);
}

Symptom

  • Error message: "426 Upgrade Required" encountered during HTTP RFC connection test.
  • The issue occurs specifically when HTTP/1.0 is used instead of HTTP/1.1 for the connection test in transaction SM59.

"Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental." 

Environment

  • ABAP Web Services
  • SAP NetWeaver
  • SAP NetWeaver Application Server for SAP S/4HANA
  • ABAP PLATFORM - Application Server ABAP

Reproducing the Issue

  1. Configure an RFC destination of type 'G' or 'H' in transaction SM59.
  2. Set the HTTP version to HTTP/1.0 under the "Special Options" tab.
  3. Perform the connection test for the configured RFC destination.
  4. Observe the error message "426 Upgrade Required".

Cause

The issue occurs when the internal test report in transaction SM59, verifies connections for HTTP destinations using version 1.1. It may set the HTTP version incorrectly to 1.0 if no compression is applied in the HTTP request, influenced by the system version.

When any HTTP destination is tested in transaction SM59 using HTTP version 1.1 without compression, the HTTP version is set to 1.0 in the resulting request.

Resolution

The problem is not within the ABAP HTTP client. When the ABAP HTTP client is used directly in the application, this issue is not observed.

  1. Implement SAP Note 2547968 - "Test Connection for HTTP Destination uses wrong HTTP version" in the affected system.
  2. Restart the Internet Communication Manager (ICM) via transaction SMICM.
  3. Retest the RFC connection to confirm the Resolution.

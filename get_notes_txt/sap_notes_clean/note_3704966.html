Symptom

When the Order-specific routing is created for production order in a Change Reocrd, then the Order Change Status is "In Process“, then when the order-specific routing is released, the status changes to "Completed“. This is correct. However, in case the order-specific routing is created and then deleted, then the status shall be "No Change“ but currently it is shown as "Completed“. This needs to be changed as order-specific routing is not released and no change has been made.

Other Terms

Order Change Status, Order Change for Production Order, Change Record, Manage Change Records

Solution

 Implement the attached correction instruction.

When a production order is added to Change Record and there is no order-specific routing created for it - then the Order Change Status is "No Change“.

When a production order is added to Change Record and there was an order-specific routing created and not released - then the Order Change Status is "In Process“.

When a production order is added to Change Record and there was an order-specific routing created and released - then the Order Change Status is "Completed“.

However, in case the order-specific routing is created and then deleted, then the Order Change Status shall be "No Change".

Order Change Status is shown as  “No Change” if its related order specific routing does not exist in that context of change record and if the order specific routing is also deleted or not created yet. 

This SAP Note contains below enhancements:
 1. Change in the calculation of the Order Change Status field based on order specific routing status and not on Production Hold End Date Time. 
 2. Modelling changes for performance improvements in the Production Order section of change record.

Symptom

When sending emails, the following error is throwing in SOST transaction: 

Message No. XS850: Cannot process message; maximum size exceeded."either a document or the entire message has exceeded the maximum permissible size.

Maximum permissible size: 8388608 bytes.

Current size: 8530693 bytes

Additional information: <XXXXXXXXXXXXX is too big MAXLEN_BODYPART_EXT_SMTP = 8388608.

System Response: Processing is terminated, and the message is not sent.

Environment

Release Independent 

Reproducing the Issue

  1. Send an email from your SAP System
  2. Check the Status on SOST transaction 
  3. The error will be visible in red status

Cause

The parameters which controls the emails size are not with the correctly value for your business requirement. 

Resolution

To address the size restriction issue in the system, is necessary the following steps:

  •  Increase the parameter values listed in note 0000837300 - Size restriction when sending documents. Please ensure that you do not apply the correction provided in the note, as it is not compatible with your release. Instead, focus solely on adjusting the parameters outlined in the note.
  • Adjust the parameter `mpi/total_size_MB` to a value four times larger than the maximum size of the emails you intend to send.
  • Consult with your mail server administrator to verify if there are any restrictions on their end. While SAP does not impose limitations on email size and allows parameter adjustments to accommodate larger sizes, it is prudent to confirm that your mail server does not have constraints that could impact functionality.

Symptom

Your HANA database crashes with the following stack:

qo3::OptimizerImpl::estimateItab(...)         
qo3::OptimizerImpl::estimateTerm(...)         
qo3::OptimizerImpl::estimateTree(...)         
qo3::OptimizerImpl::estimate(...)             
qo3::OptimizerImpl::evaluate(...)             
qo3::Optimizer::evaluate(...)                 
qo3::Optimizer::evaluate(...)                 
TRexAPI::Search::OptimizerImpl::evaluate(...) 
TRexAPI::LocalSearchImpl::doSearch(...)       
TRexAPI::SearchAPI::search(...)

Other Terms

SAP HANA DB, Crash, Composite OOM

Reason and Prerequisites

Reason:

Due to programming error there was a crash in fallback because an internal object is in an invalid state.

Affected Releases:

  • SAP HANA 1.0 database:
    • Revisions of SPS07 - SPS11
    • Revisions <= 122.08 (SPS12)

  • SAP HANA 2.0 database:
    • Revisions <= 002.00 (SPS00)

Prerequisites:

  • There was no particular case for this crash because the issue is caused by internal transaction.

Solution

Apply one of the following SAP HANA database Revisions:

  • SAP HANA 1.0 database:
    • Revisions >= 122.09 (SPS12) or higher

  • SAP HANA 2.0 database:
    • Revisions >= 002.01 (SPS00) or higher

Workaround:

  • NONE

Symptom

  • The system does not assign internal order numbers sequentially as expected.
  • With automatic internal numbering and the buffer enabled, each created order skips 20 numbers.
  • Resetting the buffer did not change the behavior.
  • Behavior may differ between environments or users (one user sees skips, another does not).
  • No specific error messages or error codes were provided.

Environment

  • Product: SAP S/4HANA

Reproducing the Issue

  1. Ensure internal orders use automatic internal numbering and number range buffering is active for the relevant number range object (for internal orders: AUFTRAG).
  2. Create an internal order.
  3. Observe that the next available order number advances by 20 for each order created.

Cause

The standard number range buffering mechanism reserves blocks of numbers (typically 20) to improve performance and support parallel processing, resulting in apparent gaps when numbers are buffered per request; in this scenario each creation consumes the buffered block.

Resolution

In the standard system, number range buffering with an increment of typically 20 numbers is provided for order numbers. When a request number is requested, this number of numbers (partial interval) is transferred to the application server by the database server for buffering. The current status of the number range interval and the number assigned during the next request, for example, after a crash of the application server, then corresponds to the number range status on the database server.

If a number range object delivered in standard uses main memory buffering in the standard system as the object "AUFRAG' does, the buffering type should be changed only in the case of statutory requirements.

In particular, read Note 0000678501, bearing in mind that changing the buffering type - if not explicitly recommended by SAP - constitutes a modification. For other possible solutions, refer to the following notes: 0000179224, 0000599157 and 0000840901.

For the the following number range objects, gaps may cause users to have doubts since they are expecting a sequential numbering:
CO area:
- AUFTRAG (order numbers, product/process/maintenance orders, network numbers)
CAUTION: Note that when you deactivate buffering, the problems described in SAP Notes 0000020965 and 0000029030 can occur.

For a more detailed explanation, see SAP Note 0000062077 and KBA 0001843002.

Conclusion

Given the above information and explanation, you should consider using the buffer for object AUFTRAG.  

  • Buffering improves performance because more frequent database accesses are avoided and parallel posting transactions are enabled. It is not absolutely necessary to assign order numbers without gaps. We therefore recommend that you keep the buffering of the order numbers.
  • However, if you want the order numbers to be assigned without gaps, you can deactivate the main memory buffering. For more information about this, see SAP Note 62077. The object name to be used in transaction SNRO is "AUFTRAG".
  • Another option for assigning order numbers without gaps is to save an interval taken from the database server in a local file (NRIV_LOKAL buffering method - see SAP Note 0000179224). Opposed to the advantage of this method of not restricting the performance of number assignment as when deactivating buffering is the disadvantage of not being able to assign the order numbers in chronological order.
  • Avoid manipulation to the number range status to achieve a sequence of order numbers without gaps. Otherwise, you must calculate with update terminations (SAPSQL_ARRAY_INSERT_DUPREC) if the numbers to be assigned overlap.
  • You can create new number range groups for the object "AUFTRAG" using transaction KONK or using the menu path "Customizing -> Controlling -> Internal Orders -> Order Master Data -> Maintain Number Ranges for Orders".

The above items are foreseen in SAP consulting note 0000333356 - Internal order: Order numbers assigned incompletely

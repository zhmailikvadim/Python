Symptom

The storage bins are sorted at APP Sort Storage Bins (APP /SCWM/SBST) for activity picking already. But system did follow the sequence for source bin determination during picking.

Environment

SAP S/4HANA Cloud Public Edition

Cause

The storage bin sequence at APP Sort Storage Bins (APP ID /SCWM/SBST) doesn't influence for stock determination for picking.  The storage bin sequence at APP Sort Storage Bin is for warehouse order creation rule instead of source bin determination during picking.

Resolution

 Go to APP Manage Outbound Delivery Orders -> Create Pick Task Manually -> Stk Can be Removed. The stock list is sorted before picking task is created, system sort the list based on the GUID list by default. 

There is a customizing field "Stock Removal Rule" at customizing "Determine Storage Type Sequence for Stock Removal" and "Maintain Storage Types", there are four standard options which can sort the list under "Stock can be removed" at APP Manage Outbound Delivery Orders.

  • FEFO First Expired First Out
  • FIFO - First In First Out
  • LIFO -Last In First Out
  • SQFO - Small Quantities First Out

 

If the standard stock removal rule cannot fulfill the requirement, it is possible to create customized Stock Removal Rule at customizing "Specify Stock Removal Rules" to priories the stock list by Sort Field.

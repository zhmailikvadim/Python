1.What is Migration server?

 

The SAP Migration Server is a temporary, non-productive server, unmanaged (IaaS) used to perform RISE migration where the customer and partner don’t have direct access to RISE target servers in the RISE environment.

 

2.In which scenarios is a migration server required?

 

The Migration server is a Virtual Machine (VM) where partners have access to set up, install ASCS (ABAP SAP Central Services), PAS (Primary application server) or Additional application server (in case of DMOVE2S4 migration method) and connect to ECS Database (HANA/ASE) to perform migration/conversion. Some examples are SUM DMO based migration, SWPM export import method based migrations.

 

SAP RISE does not allow OS/root access to application or database servers in our cloud.

Therefore by default, SAP RISE provides an additional virtual Linux server (Migration server) - with full root access.

Migration Server will be added automatically to the landscape for RISE with SAP as applicable.

300GB with /sapmnt 50GB, /usr/sap/<SID> 150GB, /Migration: 100GB

 

If the project will use DMOVE2S4, the Customer needs to subscribe to temporary additional storage for /usr/sap/<SID>. E.g.: 300 GB

Up to 2TB block storage can be requested for 6 months duration for temporary usage on the migration server, under the RISE contract.

After successful migration, the Migration Server will be decommissioned.

 

3.In which scenarios is a migration server not required?

 

If migration partner/customer is using migration methods (for example the tenant copy method) which don’t require OS (operating system) level access to target (RISE side) and therefore don’t require any temporary Primary application server and ASCS (ABAP SAP Central Services) to be installed.

 

4.Can we get a Windows Migration server as by default SAP RISE provides a Linux migration server?

 

For Windows/MSSQL on-premise - Windows migration server should be additionally requested, as by default a Linux Migration server is provisioned.

 

5.Can we take a backup of the Migration server?

 

ECS (SAP Enterprise Cloud Services) does not provide backups of the Migration Server. Backup requests for the Migration Server cannot be submitted or processed through a Service Request (SR).

 

6.Is the Migration Server permanent or temporary?

 

Migration Server is a temporary IaaS (Infrastructure as a Service) server only provided until migration.

 

7.What is the validation period for the migration server?

 

12 months. ECS will trigger decommissioning once the system goes live.

 

8.What are the roles and responsibilities concerning the migration server?

 

Migration Server is an unmanaged server. The customer and certified migration partners are granted full authorization (root access for Linux and administrator access for Windows based migration servers) to install any required software.

No monitoring server is provided from the ECS (SAP Enterprise Cloud Services) side.

No operations such as managing OS folders, files, server patching, or installing software are performed by the service provider.

No backup solution is provided from the ECS (SAP Enterprise Cloud Services) side.

No SLA is provided for any applications running on the migration server from the ECS (SAP Enterprise Cloud Services) side.

 

9.How do I connect to the migration server?

 

Prerequisite: Inbound access through port 22 is not allowed by default. So, to access Migration Server from the customer source network, ensure port 22 has been whitelisted.

If not, please submit a Service Request with template “Add/Modify Hyperscalers NSG/FW Rules (Azure/AWS/GCP)”

You can connect to the migration servers using PuTTY for Linux-based servers and Remote Desktop Connection for Windows-based migration servers.

 

10.How do I reboot the Migration Server (IaaS)?

 

Since root access is provided for the migration servers, you may use the below commands:

For Linux-based migration systems, use the “reboot now” command.

For Windows-based migration systems, use the “Restart-Computer” PowerShell command.

 

11.How do I download Notes on the migration server?

The migration team or customer is responsible for configuring the RFC connection to enable Notes download through the customer’s proxy.

 

12.How do I open a remote connection (STFK) to applications running on the migration server?

 

Please follow the below steps:

Setup SAProuter for ABAP instance on migration server

Logon to “SAP for Me”

Find SID (LT1 by default) of the migration server in the “System” tab

Ensure Hostname and Port of the migration server have been added for R/3 connection

Ensure ECS CGS vh<cid><datacenterid>gwa has been added for additional SAProuter

Refer to note 812732 - R/3 support service connection

 

13.How do I switch to the primary application server?

 

 Customer/Migration partners can raise a service request “Switch Application back to Main Instance after Migration”.

 Please note: schema and tenant database passwords would be changed once the service request “Switch Application back to Main   Instance after Migration” is completed.

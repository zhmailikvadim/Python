Symptom

  • Deadlock in table MLRUNLIST shows ABAP termination in program SAPLFCML4H_UPD_MASTER during FCML4H_STATUS_POSTING (main program SAPMSSY4) includes LFCML4H_UPD_MASTERU07 at UPDATE MLRUNLIST.
  • Runtime error: DBSQL_SQL_INTERNAL_ROLLBACK with lock wait timeout on mlrunlist during fcml4h_status_posting.

Environment

  • Controlling (CO)
  • SAP S/4HANA 1709
  • SAP S/4HANA 1809 
  • SAP S/4HANA 1909
  • SAP S/4HANA 2020

Reproducing the Issue

Check dumps in transaction ST22 and locate DBSQL_SQL_INTERNAL_ROLLBACK in SAPLFCML4H_UPD_MASTER within FCML4H_STATUS_POSTING; check include LFCML4H_UPD_MASTERU07 at the UPDATE MLRUNLIST statement. Note that occurrences are random and may not be reproducible on demand.

Cause

Unnecessary update of table MLRUNLIST during status posting in material ledger/actual costing leading to lock wait timeouts/deadlocks.

Resolution

  1. Implement SAP Note 3712641 Unnecessary update of MLRUNLIST.
  2. Also, the following SAP Notes for performance improvements need to be implemented:

        SAP Note 3085944 – Performance improvement for insertion of records into table MLRUNLIST.

        SAP Note 2956839 – WIP calculation leads to DEADLOCK on table MLRUNLIST.

        SAP Note 3211549 – Performance improvement for internal buffers in material ledger.

For lower support packages SAP S/4 HANA 1610 or 1709, implement SAP Note 2610890 Deadlock when updating the table MLRUNLIST.

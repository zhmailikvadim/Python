Symptom

After inserting new record to position to change a field, the jobinfo record was created due to positionTojob sync rule, at the same time, the onSave rules under jobinfo was triggered, in the rule, the mandatory field was set as null, but no error occurred and jobinfo record was saved successfully. So why even there is no value for mandatory field, the record can still be saved without error.

But if inserting jobinfo record via import or UI, the mandatory field is set as null due to onsave rule, error will occur and saying that mandatory field can't be blank.

Environment

SAP SuccessFactors Employee Central

  • PositionToJob Sync

Reproducing the Issue

  1. manage position < create new record for position A, the effective date is MM/DD/YYYY
  2. After confirming the data can be synced to user a's jobinfo, check a's MM/DD/YYYY record, found no value to mandatory field "XX"
  3. Confirm in the rule trace, "XX" is set as null due to rule.

Resolution

 Expected behaviour. The sync is system-driven and strict validations here could block these automated syncs.

Keywords

INC19408078, positionTojob sync, position to job sync

Symptom

In Language Adaptation → Opportunity Service → Codelist → Sales Phase:

  • When updating multiple Sales Phase descriptions at once:
    • Only some entries are saved
    • Other entries are not persisted after clicking save
  • No error message is displayed

Environment

  • SAP Sales Cloud And SAP Service Cloud Version 2
  • SAP Sales Cloud Version 2

Reproducing the Issue

1. Navigate to Language Adaptation.  
2. Go to Opportunity Service.  
3. Open the Codelist tab.  
4. Select Sales Phase.  
5. Modify multiple Sales Phase descriptions (e.g., update multiple entries).  
6. Click Save.  
7. Observe that the data is not saved for all Sales Phase descriptions.  

Cause

  • A Sales Cycle and its associated Sales Phases are persisted as a single record
  • This means multiple Sales Phases under the same Sales Cycle are saved together as one entity.

Example:

SalesCycleA

  • SalesPhase_1
  • SalesPhase_2
  • SalesPhase_3

SalesCycleB

  • SalesPhase_5
  • SalesPhase_6
  • SalesPhase_7
  • If you update:
    • SalesPhase_1 and SalesPhase_2 (same SalesCycleA) →  Only one change is saved
    • Because both belong to the same record (SalesCycleA)
  • If you update:
    • SalesPhase_1 (SalesCycleA) and SalesPhase_5 (SalesCycleB) →  Both changes are saved
    • Because they belong to different records (different Sales Cycles)

Resolution

This is a known system limitation.

Workaround:

  • Update one Sales Phase at a time per Sales Cycle
    OR
  • Update Sales Phases across different Sales Cycles in a single operation

Enhancement:

  • This limitation has been acknowledged by development and is being considered as an enhancement for future releases

Symptom

  • Running bcp with option -Y and an incorrect password terminates with “Segmentation fault (core dumped)”.
  • Without -Y, the same connection attempt returns expected errors instead of crashing: “Login failed.” and “CTLIB Message: ct_connect(): protocol specific layer: external error: The attempt to connect to the server failed.”
  • With the correct password, bcp completes without issue even with -Y.

Environment

  • SAP Adaptive Server Enterprise (ASE) 16.0, 16.1
  • SAP Adaptive Server Enterprise SDK 16.0, 16.1

Reproducing the Issue

Run a bcp in operation with client-side conversion enabled (-Y) and supply an incorrect password, for example:

bcp tempdb..test in test.txt -Usa -P<Wrong Passwd> -S<ASEName> -c -Y

Observe that bcp terminates with “Segmentation fault (core dumped)”.

    Cause

    The issue has been addressed under SAP CR 831885. Its description reads, "The bcp utility may encounter a segmentation fault when an incorrect password is supplied with the -Y option."

    Resolution

    1. There is currently no target Fix Pack or Service Pack scheduled for this issue. It is being reviewed by our Product Group.
    2. SAP CR 831885 is scheduled to be addressed in a future SAP ASE Service Pack. Please mark this KBA as a Favorite to be automatically notified when the CR is fixed. 
    3. If -Y must be used, ensure correct credentials are provided to prevent triggering the defect.
    4. Validate character set requirements for your load. If client-side conversion is not mandatory, prefer server-side conversion (omit -Y) until a fix is available.

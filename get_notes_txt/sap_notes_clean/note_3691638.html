Symptom

Legal change according to ESMA Level II Validations: Trade ID validations

You use the New Correspondence Framework for Financial Transactions and import SWIFT MT300 messages for your FX transactions. You observe that even invalid Trade IDs are imported and stored in your FX transactions (after matching).

With note 2215269 a new BAdI was introduced (BADI_TLR_TRADE_ID_CHECK), see further details there.

This validation interface is now called from the automatic correspondence import functions as well. If you want the same behaviour (Error or Warning message) as for other processes you only need to check/adjust your customizable messages (see note 2215269).

  • Issuing an error message prevents the import of the Trade ID to your financial transactions, but does not influence the matching process itself.
  • Issuing a warning message imports even invalid Trade ID to your financial transactions and does not influence the matching process as well.

If you want to establish a different behaviour for correspondence processes than for other functions you may need to implement your own BAdI implementation class in order to implement a different message handling if parameter IV_CORRESPONDENCE_MODE is set ('X').

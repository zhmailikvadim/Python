Symptom

  • Provisioning job failing on source system
  • Error message: "Oauth request failed with status: 400 and body: {"error":"unauthorized_client","error_description":"AADSTS700016: Application with identifier < `xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx` > was not found in the directory < Azure directory name > . This can happen if the application has not been installed by the administrator of the tenant or consented to by any user in the tenant. You may have sent your authentication request to the wrong tenant. 

Environment

Identity Provisioning Service (IPS)

Reproducing the Issue

  1. Configure the EntraID source system with application identifier 'xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxxx'.
  2. Grant EntraID API permissions.
  3. Run the provisioning job on the CIS tenant.
  4. Observe the error message indicating failure due to the application identifier not being found in the directory.

Cause

The user attribute configured in the Microsoft Entra ID source system does not match the application ID in Azure.

Resolution

  1. Verify the application ID configured in the source system
  2. Correct the application ID to match the actual application ID in Azure
  3. Re-run the provisioning job simulation on the CIS tenant.
  4. Confirm that the job executes successfully without errors.

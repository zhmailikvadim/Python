Symptom

  • After migrating from SAP ECC to SAP S/4HANA, report J_1BBR20 does not populate Segment Z information in the bank return file.
  • In SAP S/4HANA, Segment Z is only filled if a preceding segment in the file is 'O' or 'N'. 
  • Outgoing files generated by classical programs (for example, RFFOBR_V) show the same behavior change for segment Z.
  • This behavior differs from SAP ECC, where Segment Z was populated regardless of the preceding segment type.

Environment

  • SAP S/4HANA
  • SAP S/4HANA Cloud Private Edition 

Reproducing the Issue

  1. In an SAP S/4HANA system (version 1909 or higher), execute report J_1BBR20.
  2. Use input data where segments other than 'O' or 'N' should have a corresponding Segment Z.
  3. Observe that the Segment Z information is not generated for these segments.

Cause

The functionality to process Vendor Operation segments in Brazilian bank statements was made obsolete in SAP S/4HANA version 1909 and higher. The logic for populating Segment Z was part of this obsolete functionality.

Resolution

This is the standard system behavior in SAP S/4HANA 1909 and all subsequent releases. The functionality for Vendor Operation segments, including the associated logic for Segment Z in report J_1BBR20 and classical programs like RFFOBR_V, is obsolete and no longer supported.

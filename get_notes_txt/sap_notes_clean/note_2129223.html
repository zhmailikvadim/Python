Other Terms

P4, rmi_p4

Reason and Prerequisites

If a dispatcher node restart occurs, it closes all opened connections at the current moment. After that, it sends notification to all elements in the same cluster in order to remove any P4 calls that wait for reply from these connections. Due to a program error, the server node which receives such notifications removes only P4 calls from the same server node.

We introduce a patch that clears all P4 calls waiting for reply from connections which are closed by the restarted dispatcher node.

Solution

Update your AS Java to a release where the issue is fixed. Apply the patch provided in the Support Packages & Patches section of this SAP Note.

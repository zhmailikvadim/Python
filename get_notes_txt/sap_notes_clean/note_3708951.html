Symptom

When you execute queries using the ABAP function module TREX_EXT_AGGREGATE, the following error may occur:

Bad request from application: H001.PRED does not exist in object structure

This error appears when the query filter in import parameters I_QUERY_ENTRIES or I_QUERY_ENTRIES2 of the ABAP interface TREX_EXT_AGGREGATE contains multiple OR conditions on hierarchy-related columns (such as H001.PRED).

Other Terms

TREX_DBS_AGGREGATE, trexviadbsl, trexviadbslwithparameter

Reason and Prerequisites

Prerequisite:
The query is a hierarchy query. The import parameter I_INDEX_ALIAS_OPERATION is defined for the ABAP interface TREX_EXT_AGGREGATE.

Cause:
A bug exists in the HANA ABAP client. According to SAP Note 2893791, multiple OR conditions in the query filter table are normally optimized during query execution. However, in complex scenarios involving hierarchy queries, hierarchy-related columns in the query filter table are not recognized because they are not part of the main query object.

Solution

To resolve this issue, apply the ABAP patch or source code correction provided in the relevant SAP note. This enables the optimiser to recognize hierarchy-related columns in the query filter table.

Workaround:
Until the fix is applied, you can temporarily disable the optimization feature as described in SAP Note 2925382.

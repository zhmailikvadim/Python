Symptom

After migration the Information Steward repository from one type of Database to a different one, importing objects into the new Database fails.

Environment

Information Steward 4.x

Cause

This appears to be expected behavior when migrating between different database types. The import tool does not automatically convert database connection configurations across different database systems. This behavior is currently under review.

Resolution

Workaround Steps:

  1. Create a temporary project in the target environment:

    • Open an empty/uninvolved IS project in the new environment (with SAP HANA database)
    • Manually add tables to this project
    • Create a rule and a rule task

  2. Export the temporary project:

    • Export the entire temporary project
    • This will generate a zip file with the correct SAP HANA connection XML format

  3. Extract the correct connection information:

    • Open the exported zip file from step 2
    • Locate and copy the SAP HANA connection information from the XML files

  4. Modify the original export file:

    • Take your original export file (e.g., simplified_Project_20251204_110946.zip from the DB2 environment)
    • Open this zip file
    • Replace the old connection information with the SAP HANA connection information copied in step 3
    • Important: Leave all other files unchanged

  5. Save and import:

    • Save the modified zip file (e.g., simplified_Project_20260107_110946_HANA.zip)
    • Import this edited zip file into the target environment
    • Import should now complete successfully

Symptom

The AMQP sender adapter consumed the same message multiple times from SAP Event Mesh and forwarded it to the receiver system repeatedly, resulting in duplicate data records in the target system.

Environment

  • SAP Cloud Integration
  • BTP Integration Suite

Cause

In SAP Cloud Integration, the Quality of Service (QoS) is configured as At-Least-Once. Under this delivery semantics, messages can be delivered and processed more than once.

For the AMQP adapter, duplicate message delivery may occur if a message is successfully processed but the acknowledgment (ACK) sent back to the broker is not received (for example, due to network or connectivity issues). In such cases, the broker cannot confirm successful processing and consequently redelivers the same message.

Resolution

  • Implement an Idempotent Process Call to identify duplicate messages and ensure that they are not processed more than once.
  • Refer to the documentation below for detailed instructions on defining an Idempotent Process Call:

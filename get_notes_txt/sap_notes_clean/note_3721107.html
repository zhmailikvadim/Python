Symptom

  • In the FIORI app F4572 (Pricing Elements - Flexible Analysis), the error message "Result set for query 2CCSDBDIPEFDAYQ is empty" appears when making a search with a small value for search field "Number of Evaluation Days".
  • Search result number varies when searching with different values in field "Number of Evaluation Days".
  • It is not clear what 'Number of Evaluation Days' standards for. This article describes the meaning for this field.

*Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.

Environment

  • Sales and Distribution (SD) 
  • SAP S/4HANA

Reproducing the Issue

  1. Open the FIORI app F4572 (Pricing Elements - Flexible Analysis).
  2. Pass filters including the Number of Evaluation Days in the filter area.
  3. Execute the query. 

Resolution

  • CDS View for Fiori App F4572 is C_BILLGITMPRCGELMFORDAYSQ, SQL View name is CSDBDIPEFDAYQ. 
  • Parameter 'Number of Evaluation Days' is the number of days counting back from the current date. Technically, its value is used to compare with field BillingDocumentDate in C_BILLGITMPRCGELMFORDAYSQ.
  • Source data for field BillingDocumentDate is VBRP-FKDAT_ANA.
  • For example, if 'Number of Evaluation Days' is set to 1, today is 26 Feb. 2026, the result is resricted that BillingDocumentDate is larger than 25 Feb. 2026, and less/equal to 26 Feb. 2026. Notice, it is larger than 25 Feb. 2026.  The day of 25 Feb. 2026 itself is not included.

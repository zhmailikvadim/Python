Reason and Prerequisites

Issue number 353054

Reason:

Due to a programming error in SAP HANA, indexserver does not handle the invalid syntax query, causing an indexserver crash.
Here is a example for the invalid syntax query:

MDX WITH....
SELECT
NON EMPTY
{ ( [dimension_A].[hierarchy_A].&[member_A],[dimension_B].[hierarchy_C].&[member_E] ),
( [dimension_A].[hierarchy_B].&[member_B],[dimension_B].[hierarchy_C].&[member_F] )},
{ [dimension_A].[hierarchy_A].&[member_C] },
{ [dimension_A].[hierarchy_A].&[member_D] }
ON COLUMNS,
... ON ROWS FROM .... WHERE(...)

The first set in above query is illegal, because the tuples inside the set have different dimensions or hierarchies.

Affected Releases:

  • SAP HANA 2:
    • Revisions of SPS00 - SPS06
    • Revisions <= 079.08 (SPS07)
    • Revisions <= 089.01 (SPS08)

Prerequisites:

  • You are executing a MDX query with a set consists of tuples that have different dimensions or hierarchies.

Solution

Apply one of the following SAP HANA Revisions to prevent indexserver crash, and get expected error thrown for the query:

  • SAP HANA 2:
    • Revisions of SPS07: SAP is currently working on a permanent solution to this issue. This SAP Note will be updated as soon as a revision containing the solution is available
    • Revisions of SPS08: SAP is currently working on a permanent solution to this issue. This SAP Note will be updated as soon as a revision containing the solution is available
    • or higher

  • or higher

Apply one of following workarounds:

  • Make sure that tuples in a set have same dimensions and hierarchies as below.
    If you need to query on different dimensions or hierarchies, please execute in a separate statement.

    MDX WITH....
    SELECT
    NON EMPTY
    { ( [dimension_A].[hierarchy_A].&[member_A],[dimension_B].[hierarchy_C].&[member_E] ),
    ( [dimension_A].[hierarchy_A].&[member_B],[dimension_B].[hierarchy_C].&[member_F] )},
    { [dimension_A].[hierarchy_A].&[member_C] },
    { [dimension_A].[hierarchy_A].&[member_D] }
    ON COLUMNS,
    ... ON ROWS FROM .... WHERE(...)

    or
  • Wrap the sets with CROSSJOIN( ).
    This workaround handles invalid queries by returning an expected syntax error indicating that the MDX set must be defined with the same dimensions and hierarchies.

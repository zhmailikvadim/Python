Symptom

Remote source configuration:

  • Transmit Data in Compact Mode = true
  • Enable Transaction Merge = false
  • Enable Statement-level Trigger = false
  • Triggers Record PK Only = false

Update data is not replicated to target table. There's no obvious error or warning.

Other Terms

Replication, compact mode, trigger-based, SDI

Environment:

SAP HANA SDI 2.0 SP08 Patch 2 and before

Steps to Reproduce:

  • Create remote source with the configurations as above
  • In source, create table like these:

CREATE TABLE AAATABLE (
      INT_C1 INTEGER NOT NULL PRIMARY KEY,
      SMALLINT_C2 SMALLINT,
      BIGINT_C3 BIGINT,
      DECIMAL_C4 DECIMAL(10,4),
      NUMERIC_C5 NUMERIC(10,4));
CREATE TABLE BBBTABLE (
      INT_C1 INTEGER NOT NULL PRIMARY KEY,
      SMALLINT_C2 SMALLINT,
      BIGINT_C3 BIGINT,
      DECIMAL_C4 DECIMAL(10,4),
      NUMERIC_C5 NUMERIC(10,4));

  • Start CDC replication
  • Suspend remote source
  • In source, set auto-commit=false and do DMLs like these:

insert into AAATABLE values (100, 100, 100000, 123.4567, null);
insert into BBBTABLE values (100, 100, 100000, 123.4567, null);
Commit;

UPDATE AAATABLE SET NUMERIC_C5 = 555.5555;
Commit;

  • Resume remote source
  • In the end, NUMERIC_C5 in target table of AAATABLE is null, but it’s 555.5555 in source.

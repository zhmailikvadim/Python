Symptom

After running MRP, it is observed Splitting quota did not happen as expected.

Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.

Environment

  • SAP ERP Production Planning (PP) Release Independent
  • SAP ERP Release Independent
  • SAP ERP Central Component Release Independent
  • SAP Enhancement package for SAP ERP Release Independent
  • SAP S/4HANA, on-premise Release Independent
  • SAP Fiori for SAP ERP Release Independent
  • SAP Fiori for SAP S/4HANA Release Independent
  • SAP S/4HANA Cloud Private Edition Release Independent

Reproducing the Issue

  1. T-code MEQ1: maintain splitting quota.
  2. Run MRP and check result.

Cause

Minimum qty split has been maintained for the splitting quota.
In case Minimum qty split is greater than the requirement qty, then splitting quota cannot occur during MRP run.

Resolution

Please remove the value of filed Minimum qty split if needed.

  • Minimum Quantity for Splitting Quota /EQUK-SCMNG

Specifies for the splitting quota the minimum quantity that a lot must have in order to be apportioned among several sources of supply in the planning run.

If the lot is smaller than or equal to the minimum quantity, the system will proceed on the basis of the usual quota arrangement logic.

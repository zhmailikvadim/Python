Symptom

  • For BOMs with a history requirement, a change number should be mandatory when editing in app F1813.
  • The system does not display the error message "Enter Change Number - BOM has history requirement" when editing BOMs.
  • The flag "Hist req" in T-code: OS27 is checked.
  • Note 3564217 is implemented.

Environment

  • SAP ERP Logistics (LO)
  • SAP S/4HANA, on-premise
  • SAP S/4HANA Cloud Private Edition

Reproducing the Issue

  1. Open the "Maintain BOM" app (F1813).
  2. Enter material and plant/usage navigate to the details.
  3. Click Edit button. message  "Enter Change Number - BOM has history requirement" not displayed.
  4. Add items or modify the BOM.
  5. Save the changes.
  6. Observe that the error message "Enter Change Number - BOM has history requirement" appears only upon saving.

Cause

Missing note correction.

Resolution

Please implement SAP Note 3666963 and 3668428 , then follow below steps:

  1. Execute the report `/UI5/APP_INDEX_CALCULATE`.
  2. Select the tab 'Single SAPUI5 Repository Only'.
  3. Enter the UI5 repository `bommaint_mans1` and execute.
  4. Execute the T-code `/n/cache_cleanup`.
  5. Run the report `/ui2/delete_cache_after_imp`.

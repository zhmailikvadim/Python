Symptom

You post a document with business partner items with different plus/minus signs that balance to zero, for example, from transaction FPE1. The account assignments of the expenses and revenues are identical and add up to zero. The system does not create any general ledger items and does not inherit any account assignments to the tax items. The document cannot be posted if the business area, profit center, or segment is set as a required entry field.

Other Terms

DFKKOPAA, tax split, TFK001B, TXAAR, GSBER, PRCTR, SEGMENT

Reason and Prerequisites

This is a program error. It occurs if the following prerequisites are met:

  • The function 'Splitting of Tax Items by Account Assignments' is active in the company code configuration (TFK001B-TXAAR).
  • The system determines the general ledger lines for revenues and expenses at event 1100 using the standard module ISU_EVENT_1100.
  • The account assignments of the general ledger lines for the revenues and expenses are stored identically in posting area R001.
  • The G/L lines for revenues and expenses aggregate to zero and the system does not create them.

Solution

After you implement this SAP Note, the system creates separate items for expenses and revenues in the case described above. The account assignments are distributed to the tax items.

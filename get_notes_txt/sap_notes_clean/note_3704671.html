Symptom

  • The <package> was created as part of a proof of concept (POC) and is no longer needed.
  • Then it was removed from the transport request, but it cannot be deleted, even though it is empty.
  • Error message ARS_DB027: "Package &1 must be transported. Add to transport request before release" encountered during Simulate Release of transport request.

Environment

  • SAP S/4HANA (ABAP Platform)
  • Transport Organizer (Workbench Request)
  • SAP S/4HANA 2023 (onPrem system)

Reproducing the Issue

  1. Open Transport Organizer.
  2. Select the transport request.
  3. Attempt to simulate the release of transport request via the Simulate Release option.
  4. Observe the error message ARS_DB027: "Package &1 must be transported. Add to transport request before release."
  5. Attempt to delete package contains that TR.
  6. The package cannot be deleted, even though there are no objects assigned to it and no references or usage from other objects in SE80 (Object Navigator)

Cause

The package contained objects that were deleted earlier.

However, the deletion of these objects has not yet been transported, and the package itself has also not yet been transported.

Because of this, the system prompts the error message reports that the package must still be transported during the Simulate Release process.

Resolution

Follow these steps to resolve the issue:

  1. Add the package to the transport request as:
    • R3TR DEVC <Package_Name>.
    • This step is necessary because the request contains objects that were part of this package, and the package has not been transported yet.
  2. Release the transport request.
  3. Delete the package and record the deletion on a new workbench request.
  4. Release the new transport request containing the deletion of the package.

Additional clarification:

  • The package appears empty because its objects were already deleted.
  • These deleted objects cannot be seen in ADT or SE80 because they no longer exist.
  • However, the deletion still needs to be transported.
  • Hence, following the above procedure resolves the issue.

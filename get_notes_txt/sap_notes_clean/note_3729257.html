Symptom

Smart Data Access (SDA) remote queries that access a remote calculation view through a virtual table may return unexpected results when the virtual table is created with a projection column list.

Other Terms

Virtual Table, Calculation View, Smart Data Access, SDA, Unexpected Results, Remote Query

Reason and Prerequisites

Issue number 356354

Reason:

Due to a programming error in SAP HANA, the result-correction process for remote queries against a remote calculation view is not applied with the virtual table created with a projection column list.

As documented in SAP Note 1764658, calculation views do not behave in a purely relational manner in all scenarios. When accessing a calculation view remotely, SAP HANA includes a correction step to ensure consistent results. However, this step is not triggered correctly for virtual tables defined with projection columns.

Affected Releases:

  • SAP HANA 2:
    • Revisions of SPS00 - SPS06
    • Revisions <= 079.08 (SPS07)
    • Revisions <= 089.01 (SPS08)
  • SAP HANA Cloud:
    • 2025 QRC2 - 2026 QRC1

Prerequisites:

  • SDA is configured, and its remote source points to SAP HANA system.
  • The target object referenced by the virtual table is a calculation view on the remote SAP HANA system.
  • The virtual table is created with an explicit projection column list:

CREATE VIRTUAL TABLE [<local_schema_name>.]<local_table_name>
(<column_lists>)
AT "<remote_source>"."<database_name>"."<remote_schema_name>"."<remote_calculation_veiw_name>";

Solution

Apply one of the following SAP HANA Revisions:

  • SAP HANA 2:
    • Revisions of SPS07: SAP is currently working on a permanent solution to this issue. This SAP Note will be updated as soon as a revision containing the solution is available
    • Revisions of SPS08: SAP is currently working on a permanent solution to this issue. This SAP Note will be updated as soon as a revision containing the solution is available
    • or higher
  • or higher
  • SAP HANA Cloud:
    • SAP is currently working on a permanent solution to this issue. This SAP Note will be updated as soon as a revision containing the solution is available

Workaround:

Recreate the virtual table without a projection column list so that all columns are included:

CREATE VIRTUAL TABLE [<local_schema_name>.]<local_table_name>
AT "<remote_source>"."<database_name>"."<remote_schema_name>"."<remote_calculation_veiw_name>";

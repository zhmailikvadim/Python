Symptom

  • External Time Data import failed with Import Time Sheet Alert "There is already a time sheet in this period for this user with the status."
  • Time Events are paired, but Employee Time Sheet Entry failed to generate with the error message.

Environment

  • SAP SuccessFactors Employee Central - Time Sheet
  • SAP SuccessFactors Time Tracking

Reproducing the Issue

  1. The clock-time based positive time sheet is in Approved status. 
  2. Create or import new Time Events for the Approved time sheet.
  3. The time events are paired and generate new External Time Data. 
  4. With the instant import feature enabled, an event-based import time sheet job processes the External Time Data, triggering the amendment of the Employee Time Sheet.
  5. Concurrently, a daily scheduled Import Time Sheet job processes the same External Time Data, leading to duplicate time sheets being generated.
  6. when creating time events again, the External Time Data failed with error "There is already a time sheet in this period for this user with the status.". 

Cause

The issue occurs due to concurrent processing of the same External Time Data (External Time Data) by both the event-based import job and the daily scheduled Import Time Sheet job, leading to duplicate time sheets.

Resolution

  1. Check for duplicate Pending Employee Time Sheets with the same "Replaces" field value.
  2. Delete one of the duplicate Pending time sheets.
  3. Trigger the import time sheet job again.
    OR
  • Consider disabling the instant import to ensure all External Time Data are processed by the daily Import Time Sheet job.

It's recommended to submit and approve the time sheet after the time sheet period completed to prevent the similar issue.

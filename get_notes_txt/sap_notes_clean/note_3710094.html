Symptom

Unable to delete subscription in ODQMON for ADSO. The following short dump may occur:

  • Category               ABAP programming error
    Runtime Errors         TIME_OUT
    ABAP: Program          CL_RSPM_LOG===================CP
    Application Component  BW4-DM

    "TIME_OUT"
    "CL_RSPM_LOG===================CP" bzw. CL_RSPM_LOG===================CM004
    "IF_RSPM_LOG~CREATE_ENTRY"
  • Category               ABAP programming error
    Runtime Errors         TIME_OUT
    ABAP: Program          CL_RSPM_PERSISTENCY_STANDARD==CP
    Application Component  BW4-DM

Environment

  • SAP Netweaver
  • SAP BW/4HANA

Reproducing the Issue

  1. Navigate to transaction ODQMON.
  2. Select the ADSO queue.
  3. Click button 'End subscription'.
  4. Encounter short dump TIME_OUT.

Cause

  • Excessive number of processes for the ADSO in the RSPMPROCESS table (PROCESS_TYPE as DMART)
  • Subscription deletion attempts to remove all relevant processes, leading to TIME_OUT due to the large volume of entries.

Resolution

  1. Run housekeeping report (RSPM_HOUSEKEEPING) for the issued ADSO.
  2. Set "Select Everything Older Than" to as 7 days.
  3. Execute the housekeeping report in simulation mode to identify processes.
  4. Confirm identified processes and execute the housekeeping report to remove them.
  5. Attempt to delete the subscription again after completing housekeeping.

Keywords

subscription deletion, CL_ODQ_MONITOR=>DROP_SUBSCRIPTION, IF_ODQ_SUBSCRIPTION_SERVICE~DROP_SUBSCRIPTION, ADJUST_SUBSCRIPTIONS, CL_RSODP_BW_CURSOR=>REMOVE_DATA_MART_POINTERS

Symptom

A UI issue occurs in SAP Service and Asset Manager when using multi-user mode during new user onboarding.

On the End User License Agreement (EULA) screen, the "Agree" button is not visible. However, the button remains functionally active, and users can proceed by tapping the area where the button is expected to appear.

This behavior has been observed on the SSAM iOS standard client version 2511.

Environment

SAP Service and Asset Manager

Reproducing the Issue

  1. Log in to SAP Service and Asset Manager with an initial user on the device.
  2. Add a new user on the device in multi-user mode.
  3. Proceed with the onboarding process until the EULA screen is displayed.
  4. Observe that the "Disagree" button is visible on the bottom-left corner of the screen.
  5. Note that the "Agree" button is not visible on the bottom-right corner.
  6. Tap the bottom-right area where the "Agree" button is expected to appear.

Cause

This issue is caused by a defect in the MDK client framework

Resolution

The issue has been fixed in SAP MDK Client 25.9.2.

Reference: 3714434 - SAP Mobile development kit client 25.9.2 - SAP for Me 

For SAP Service and Asset Manager:

  • SSAM 2511.1.0 (planned for end of Q1/2026) will include MDK 25.9.1.
  • SSAM 2511.2.0 (planned for Q2/2026) is planned to include MDK 25.9.2.

Customers experiencing this issue should upgrade to a SSAM client version that includes MDK 25.9.2 or higher once available.

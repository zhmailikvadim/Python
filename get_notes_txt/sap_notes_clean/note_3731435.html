Symptom

During MDX tracking of settlement MaLo data updates, the installation is not determined from the first time slice. As a result, the division category is not determined, which leads to unexpected MDX change entries being generated.

Other Terms

MaCo, /US4G/CL_MDX_CHGTRK_SETTLMALO, PREPARE_CHANGED_DATA

Reason and Prerequisites

Program error

Solution

Implement the corrections of this SAP Note or import the relevant support package.

After implementing this SAP Note, the system determines the installation and division category correctly. You can verify the fix by updating and saving settlement MaLo data, then checking that the correct MDX change entries are generated.

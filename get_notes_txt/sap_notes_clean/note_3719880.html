Symptom

A reason code is defaulted when creating dispute cases via FBL5n or Manage dispute cases apps (F0702 or F0702A).

"Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental."

Environment

  • Dispute Management
  • SAP ERP       
  • SAP enhancement package for SAP ERP
  • SAP S/4HANA      

Reproducing the Issue

  1. Call transaction FBL5N and enter the customer's data.

  2. Select an invoice and click "Create Dispute Case"





  3. Possible values for Reason code could be customized for the relevant Case type under:
    SPRO>
    Financial Supply Chain Management>
     Dispute Management>
      Dispute Case Processing>
        Case Types>
          Create Values for Attribute "Reason"




  4. Reason code is filled in defaulted.




Cause

If customizing is set in table V_TFDM_DEFCC it overwrittes the reason code.

Resolution

  1. Access the SAP system and navigate to the following path:

           SPRO >
               FSCM >
                 Dispute Management >
                    Process Integration with Accounts Receivable Accounting >
                        Define Default Values for Creation of Dispute Cases

         2. Locate the entry for the Company Code and check the details.
             
             
             

         3. Update the defaulted reason code if necessary for the dispute case type or leave it blank. Save the changes.

Symptom

Customers experience difficulty creating a correct Financial Statement Version (FSV) using "BS" and "IS" as placeholders. 

Environment

  • SAP S/4HANA Finance
  • SAP S/4HANA

Reproducing the Issue

  1. Attempt to import the FSV.
  2. Encounter confusion or errors during the import process.

Cause

Unfamiliarity with the correct process for importing and configuring the FSV template.

Resolution

  1. Download the latest FSV template from the attachment provided.
  2. Create a hierarchy of type "Financial Statement Version" with name "KRFS" in app "Manage Global Hierarchies", import the enclosed excel file when you edit the newly created FSV, then import the enclosed excel file for the created "KRFS" hierarchy.
  3. Ensure the Financial Statement Version nodes use "BS" and "IS" as placeholders at the beginning.
  4. Update the name in the DRC configuration, using "KRFS" as the parameter "G_FIN_STMNT_HIER_ID."
  5. If defining a different name in "Manage Global Hierarchies" for the South Korea Balance Sheet/Income Statement hierarchy, maintain the value of parameter "G_FIN_STMNT_HIER_ID" consistent with the name in the app.
  6. Modify the chart of accounts before importing the template, as the chart of accounts in the Excel file may differ from your system.
  7. For the "Functional Area," the Balance Sheet/Income Statement for South Korea follows the standard behavior of a Standard Financial Statement

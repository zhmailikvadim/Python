Symptom

When maintaining a condition record via transaction /SCWM/PRHU6, the record is saved without issue. After reloading the t-code or even just reloading the screen, some values like action definition are gone and the record is in status red.

Environment

  • SAP Extended Warehouse Management (SCM-EWM)
  • SAP Extended Warehouse Management Printing Handling Unit (SCM-EWM-PRN-HU)

Reproducing the Issue

  1. access t-code /SCWM/PRHU6
  2. add action definition to store action profile, and save
  3. refresh the record or access the t-code again, some data is lost

Cause

Generation of the condition technique DDIC objects are corrupted or not performed completely.

Resolution

Generate the object in the following path:

SPRO->SCM Extended Warehouse Management->Extended Warehouse Management->Cross-Process Settings->Handling Units->Basics->Print->Create Field Catalog

Press 'Change<>Display' button, and then press 'Generate' button.

Keywords

/SAPCND/CL_DB_ACCESS_SRV, PH_PPFACTDEF

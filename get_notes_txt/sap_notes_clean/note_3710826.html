Symptom

You execute program Employment Tax (IRPF) reports (RPC190E0) to generate form 345 and you find entries with duplicated amounts and incomplete pension plan data.

Environment

  • Human Capital Management (HCM)
  • SAP R/3
  • SAP ERP Core Component
  • SAP ERP
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA

Reproducing the Issue

  • The pension plan configuration has been done according to SAP Note 544495 - CONSULT.: Modelo 345 and the required BAdI 'Obtain additional data to create template 345' (HR_ES_MODELO_345) is implemented.
  • The employee has a pension plan recorded in infotype 'Retirement Pension Plan' (3213).
  • The employee has in their payroll results pension plan wage types (/PPB, /PPC, /PPE, etc.).
  • The employee also has in  their payroll results other wage types with the same amounts of the pension plan wage types and configured with field 'Field 190' (T5EI5-DIESP) with value 'Template 345' (P) in view 'Payroll results reading Form 190' (V_T5EI5).
  • You generate form 345 and the amounts are duplicated and the pension plan data is incomplete. 

Cause

  • When program RPC190E0 is initialized, the activation of the pension plan is checked:

     *&---------------------------------------------------------------------*
    *&      Form  e_initilization
    *&---------------------------------------------------------------------*
    FORM e_initilization.

      ini_before_output.

    * Año fiscal->año anterior
      tmp_year = sy-datum(4).
      SUBTRACT 1 FROM tmp_year.
      MOVE tmp_year TO year.
    * Posicionar para variantes y ejecuciones múltiples
      LOOP AT s_11xper.
      ENDLOOP.

    * pp >>
    *  IF CL_HRPAYES_SWITCH_CHECK=>IS_PENSION_PLAN_ACTIVE( ) EQ abap_true.
      IF CL_HRPAYES_SWITCH_CHECK=>IS_PENSION_PLAN_ACTIVE( ) EQ abap_true.
        PP_ON = 'X'.
      ENDIF.

    *<< pp

    ENDFORM.                               " e_initilization 

  • If active, the system fetches the pension plan contributions without the need of configuring additional wage types in view V_T5EI5.

Resolution

Remove or delimit those pension plan entries (DIESP 'P') from view V_T5EI5 for those wage types that represent the same pension plan wage types already fetched by the program.

In order to fill the pension plan data, BAdI HR_ES_MODELO_345 may require some adjustment.

Symptom

OData messages in SAP Integration Advisor are being considered for EDI integrations, and clarification is required on:
  • whether the OData message sets are still maintained/updated
  • the recommended approach going forward

The OData message sets in Integration Advisor were published as a preliminary set in 2019. OData has proven to be generally less suitable for complex EDI integrations, and adoption among customers is low compared to other message types

Environment

  • SAP Cloud Integration
  • SAP Integration Suite

Resolution

There are currently no plans to provide further updates to the OData message sets within Integration Advisor.

For EDI requirements, the most commonly used and more robust options are:
  • SAP SOAP messages
  • SAP IDoc messages
Evaluation of SOAP or IDoc for the implementation is strongly recommended.

If the project must remain on OData, the recommended path is to use standard SAP Cloud Integration capabilities to build the required integration, for example:
  • Integration Flows (iFlows)
  • Message Mappings

Keywords

OData, Odata V4, Integration Advisor, MIG creation, MIG

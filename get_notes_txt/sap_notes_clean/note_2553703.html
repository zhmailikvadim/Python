Symptom

Improvement Request

As part of the Customer Connection project for SAP GTS Preference Management (https://influence.sap.com/SAPGTSTradePreferenceManagement), this SAP Note provides the solution for the following improvement request:


D10247 "Preference determination log in Order-to-Cash documents"


Original description: "[...] the preference indicator in the OTC documents is determined based on several criteria. Today there is no monitoring opportunity on the issued invoices with or without preference status. It is very exhausting to collect the data out of the ERP and GTS system to prove the determined preference result. In some countries like South Korea there is a customs authority requirement to declare the issued invoices with originating status on a monthly basis. […]"

Request Reason

Within the SAP ERP-GTS integration, the preference situation in GTS is queried during processing of the ERP SD documents. The determined preference decision is defined in the preference indicator for each sales document item within the SD document.
In the current SAP standard, there is no option to monitor or analyze the setting of this preference status.

Other Terms

Customer Connection, CustConn, preference, preference management, preference indicator, preference determination, monitoring of preference decision

Reason and Prerequisites

Function does not exist in the SAP standard system.

Solution

Improvement

All the information relevant for this kind of monitoring is available in the SAP GTS system at the time the preference indicator is determined for the feeder system.
The BAdI /sapsll/pre_ind_det can be used to access and save this data persistently before it is sent to the feeder system.
This SAP Note provides an example of a possible usage of the BAdI /sapsll/pre_ind_det for billing documents. The example implementation captures relevant information from the determination of the preference indicator and safes it persistently. The additional sample report can be used to analyze this information at a later point in time.
Information: If the preference indicator is determined for new orders that have not been saved yet, the order number has not been determined when the BAdI is called. That is why the analysis in the example only involves billing items.

Benefit

The example implementation of the BAdI enables analysis of the determination of the preference indicator for billing items.

Delivery

To implement the solution, instantiate the BAdI /sapsll/pre_ind_det in a way similar to the example provided in the SAP Note. The attached example report in the SAP Note provides an option for formatting the data.

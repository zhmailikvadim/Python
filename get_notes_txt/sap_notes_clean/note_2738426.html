Symptom

You run the Payroll Driver Canada (PC00_M07_CALC) transaction for an employee who had a business number change and for a payroll period when the employee reaches or has already reached the maximum CPP/QPP contributions. However, the system calculates the C2/C2Q contributions incorrectly by considering the contributions from the previous business number.

Other Terms

C2, C2Q, PI, CPP2, QPP2, CRA, KATAX, RPCALCK0;

Reason and Prerequisites

Reason

When calculating the C2/C2Q contributions, the system does not reset the pensionable income year-to-date amount after a business number change.

This issue is a side effect caused by the solution delivered by SAP Note 3426360 - Issues with resetting contributions when business number changes.

Prerequisites

The following Support Package level is the minimum prerequisite:

Release Support Package
S4HCM 101 HR SP 00
S4HCM 100 HR SP 13
SAP_HR 6.08 HR SP C4
SAP_HR 6.04 HR SP J6
SAP_HR 6.00 HR SP N0

For more information, access the SAP ERP Human Capital Management - Support Package Schedules (SAP_HR) or SAP S/4HANA On Premise Human Capital Management (S4HCM).

When you install an SAP Note, some prerequisites might be required. The prerequisites are SAP Notes that have already been released and that change the same context of the current one. Then, even if these prerequisites are not necessarily related to the symptom of this SAP Note, you must implement them due to technical reasons.

Solution

From now on, the system no longer considers the pensionable income year-to-date amount after a business number change.

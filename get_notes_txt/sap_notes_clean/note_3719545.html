Symptom

When executing program /irm/ipclb_archive_write for a range of bucket numbers (for example, 200 buckets), if any one bucket contains tracking data, the program terminates completely instead of continuing with the remaining buckets.
In addition, the program does not display which specific bucket contains tracking data, making it difficult to identify the affected entry.
The expected behavior is that the program should skip the bucket that contains tracking data, display a clear message indicating the relevant bucket number, and continue processing and archiving the remaining bucket numbers.
This note corrects the behavior by ensuring that the program skips buckets with tracking data, logs the corresponding bucket number, and continues processing the remaining buckets without terminating.

Other Terms

/irm/ipclb_archive_write, bucket archiving issue, tracking data, program termination, skip bucket with tracking, archive remaining buckets

Reason and Prerequisites

This is caused by Program error.

Solution

Please implement the correction instructions attached. These corrections are only valid if you have applied the Support Packs.

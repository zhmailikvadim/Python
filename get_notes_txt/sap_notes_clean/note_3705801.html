Symptom

When creating and saving a new Access Sequence in Activity/SSCUI Define Inventory Management Access Sequences, no Transport Request (TR) is generated.

Environment

SAP S/4HANA Cloud Public Edition

Reproducing the Issue

  1. Open the SSCUI Define Inventory Management Access Sequences.
  2. Create a new Access Sequence.
  3. Save.
  4. Observe that no Transport Request is created.

Cause

Access Sequences are technically classified as Extension Items rather than standard Customizing Items. Due to this technical classification, they are not automatically captured in standard Customizing transport requests.

Extension Items must be transported using the Export Software Collection app (Key User Extensibility) instead of standard Transport Requests, which are used for other configuration objects.

Resolution

Access Sequences must be manually added to a Software Collection in order to be transported. Please follow the steps below:

  1. Open the Export Software Collection app.
  2. Create a new Software Collection or open an existing one.
  3. Choose Add Items.
  4. Filter the items by Type = Access Sequence of Condition Technique.
  5. Select the required Access Sequences to be transported.
  6. Export the Software Collection.

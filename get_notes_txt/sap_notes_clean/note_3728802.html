Symptom

In a task list with variants saved, you have the Logical System Name Mapping (P_LS) parameter of the SCM Refresh Post Processing for liveCache LOGSYS content (CL_STCET_SC_POSTPROC_SCM_LC_LS) task set with a higher value in the P_LS-LOW field compared to the P_LS-HIGH. When you update the task list and open one of the existing variants, the variant migration pop-up appears, after selecting and running the "Migrate variant to new task list structure" option, the system display the following error messages:

  • "For parameter P_LS the lower limit is greater than the upper limit" [Message class: STC_TASK, Message number: 081]
  • "Invalid parameters provided; check parameter" [Message class: STC_TASK, Message number: 075]

Other Terms

Task Manager, Task List, STC01,  STC02, STC_TASK

Reason and Prerequisites

The Task Manager Framework performs technical checks for ranges using option = 'BT' to validate the lower and the higher limit which fails for this task because the parameter is used as "From source, To target", where the first parameter (lower range), can be higher than the second parameter (higher range).

Solution

From now on, the Task Manager Framework does not perform the range validation for the P_LS parameter of the CL_STCET_SC_POSTPROC_SCM_LC_LS class.

Apply this note or the related support packages to disable the technical check for these tasks.

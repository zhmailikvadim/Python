Symptom

  • A report is run (for example KE24, KE30), and its output is exported to Excel.
  • The Period/Year field format is incorrect when exporting, showing 'YYYYXXX' instead of 'XXX.YYYY'. 
  • The UoM may also be displayed incorrectly in excel.

Environment

  • Controlling (CO)
  • SAP S/4HANA
  • Profitability Analysis 
  • SAP_BASIS 758    SP 00

Reproducing the Issue

  1. Execute the reporting transaction.
  2. Input the parameters.
  3. Execute the report.
  4. Export to Excel
  5. The issue is observed.

Cause

The conversion exit was not taken into account by the WRITE function when the data table does not have an associated DDIC structure.

Resolution

  1. Install the code correction note 3536643 - GUI ALV Export: Conversion exit not applied

 

Keywords

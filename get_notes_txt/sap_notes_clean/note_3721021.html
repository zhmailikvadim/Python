Symptom

  • MM02 transaction results in a short dump after SAP S4 HANA upgrade to 2023.
  • Field "GS_MARC_APS_EXT" is reported as unknown.
  • Runtime error: SYNTAX_ERROR in custom program.

Environment

Material Master

Reproducing the Issue

  1. Execute transaction MM02
  2. Perform actions related to the Material Master.
  3. Encounter a short dump with the error message: "Field 'GS_MARC_APS_EXT' is unknown."

Cause

  • The error occurs due to the field "GS_MARC_APS_EXT" being introduced in the standard include "LMGD1O22" during the SAP S4 HANA upgrade, but not explicitly declared in the custom program.

Resolution

  1. Navigate to transaction SE38 and perform a syntax check on the custom program.
  2. Double-click on the line entry "GS_MARC_APS_EXT" where the error is occurring.
  3. When the popup appears stating "Global field gs_marc_aps_ext does not exist. Do you want to create the object?", click "Yes".
  4. Create the data field "gs_marc_aps_ext" in the custom implementation of "LMGD1TOP".
  5. Compare the definition of "gs_marc_aps_ext" in the standard include "LMGD1TOP" with the custom code and ensure consistency.
  6. Define the data field as follows:

DATA:

gs_marc_aps_ext

gr_ddpen_alv_control TYPE REF TO cl_gui_alv_grid

gt_ddpen_dmnd_penalty_new TYPE ppo_t_dmnd_penalty

TABLES: marc_aps_ext

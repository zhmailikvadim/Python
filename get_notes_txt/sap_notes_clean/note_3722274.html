Symptom

  • Importing custom MDF records with inactive picklist values via Import and Export Data tool completed without any error or warning message.
  • Existing custom MDF records with inactive picklist values are displayed successfully in Manage Data.

Environment

SAP SuccessFactors HCM Core

Reproducing the Issue

  1. Create a custom MDF object and custom field with picklist data type.
  2. Create a new record (Record A) for the custom MDF object in Manage Data and select an active picklist value for this custom field.
  3. Navigate to Picklist Center and deactivate the selected picklist value.
  4. Import the existing custom MDF record (Record A) with the same inactive picklist value via Import and Export Data tool.
  5. The job log shows that record A is imported successfully in Manage Job Scheduler and Record A is displayed without error in Manage Data.

Cause

The MDF Import tool is designed to save inactive picklist values without error messages, but inactive data is not displayed by default in Manage Data unless the option to include inactive data is selected.

Resolution

  1. As confirmed with Engineering team, this should be expected behavior that custom record with inactive picklist value can be imported successfully via Import and Export tool.
  2. To view inactive data in Manage Data, select the option to include inactive data.

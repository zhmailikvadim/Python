Symptom

You try to reverse a Payment Distribution Item after end of term is reached. When executing transaction /FSPD/IPD_MR2 (Automatic Postprocessing for Payment Distribution), you notice an error 'Cannot correct Billing item'. This error is created by a billing correction triggered by the 'Call Billing' flag set at the transaction type customizing.

Other Terms

end of term split, 0900, LOAN-event, settlement exit, processed-flag, 'P', BCA_CN_EV_ACBAL

Reason and Prerequisites

Note 2266267 introduced this error. Billing correction wants to remove a previously billed item on a date for which no condition is due.

Solution

Install the latest support package or implement the attached correction instructions in advance.

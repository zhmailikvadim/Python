Symptom

You need to maintain or create connection entries in the hdbuserstore for a specific service user (e.g., SAPServiceSID).

You are logged in with an administrative account and need to modify the store of a different OS user without logging into that user’s desktop session.

Attempts to run the command result in "Access Denied" if not executed with proper administrative elevation.

Environment

  • SAP HANA
  • hdbuserstore 

Resolution

When using hdbuserstore.exe for a specific domain service account (like SAPService<SID>), the syntax requires careful placement of the -u flag and quotes to handle the backslash.

 

The Correct Syntax:

hdbuserstore.exe -u "DOMAIN\SAPServiceSID" SET DEFAULT "host:port" "DB_USER" "PASSWORD"

 

Key Components:

-u "DOMAIN\User": This specifies the Operating System user whose store you are modifying. You must run this command with Administrator privileges to access another user's profile.

SET DEFAULT: This creates the connection key named DEFAULT.

"host:port": The database server and SQL port (e.g., hdbserver:30015).

"DB_USER": The Database user (e.g., SYSTEM or SAPABAP1) that should be used to log in to HANA.

"PASSWORD": The password for that Database user.

 

Concrete Example:

If your domain is CORP, your SID is HDB, the host is hanadb01, and the port is 30015:

hdbuserstore.exe -u "CORP\SAPServiceHDB" SET DEFAULT "hanadb01:30015" "SAPABAP1" "MySecurePassword123"

 

Important Tips:

Always wrap the domain user and the host string in double quotes (" ") to prevent the command line from misinterpreting the backslash or colons.

Verification: After running the command, verify the entry was created successfully for that specific user:

PowerShell

hdbuserstore.exe -u "CORP\SAPServiceHDB" LIST

 

Permissions:

If you get an "Access Denied" error, ensure your command prompt is running as "Administrator".

  

Using SSO (Kerberos) instead?

If you want to use the domain account to log in without storing a password (Single Sign-On), you need the -zk flag:

 hdbuserstore.exe -u "DOMAIN\SAPServiceSID" -zk SET DEFAULT "host:port"

Symptom

  • In SAP XS Advanced, the syntax "PARTITION BY KEEPING EXISTING LAYOUT" in .hdbcds files preserves partition syntax added to tables.
  • Equivalent partition syntax is required for .hdbtable in SAP HANA Cloud for all generated tables.

Environment

  • SAP HANA Cloud
  • SAP Business Application Studio

Reproducing the Issue

Cause

Consulting

Resolution

  1. In .hdbtable (and .hdbdropcreatetable) artifacts, add the comment line --!HDI KEEP_EXISTING_LAYOUT to retain the existing partitions of a deployed table, even if the table definition specifies a different partitioning layout.
  2. Alternatively, configure the build plugin parameter keep_existing_layout to achieve the same behavior during HDI builds.
  3. Redeploy the table artifact and verify that the previously deployed partition layout is preserved.

* For on-premise HANA, there is no such feature mentioned in the guide Tables (.hdbtable and .hdbdropcreatetable) for HANA Plateform.

  • "Partitioning of the table should only be specified or changed in the design-time table resource, since changes applied to the run-time table object are lost each time the design-time table is redeployed."

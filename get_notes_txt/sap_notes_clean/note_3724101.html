Symptom

Data selection from table /PACG/ECM_CLTDH (Checklist Instance) is currently time-consuming. This is due to the absence of a database index on the fields OBJ_REF_TYPE (Reference Object Type) and OBJ_REF_KEY (Reference Object Identifier). Selection based on these two parameters is commonly used in the FSM connector.

Other Terms

FSM, E4C, Connector, Checklist, Instance, Table, Index

Reason and Prerequisites

In SAP systems, database indexes are used to improve the performance of data retrieval. An index allows the database to locate records more quickly without scanning the entire table. This is especially important for large tables or frequently executed queries. By creating an index on fields that are commonly used in selections, joins, or filters, the system can significantly reduce response times and improve overall application performance.

Solution

This SAP Note provides manual steps to create an appropriate database index for these fields. This implementation will optimize data retrieval from the table, making the selection process more efficient and improving overall system performance.

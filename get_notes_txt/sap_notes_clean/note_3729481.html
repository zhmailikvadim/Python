Symptom

When using RAP business object I_MAINTENANCEPLANTP_2 in EML (Entity Manipulation Language) statements, you notice performance degradation during system status change actions in a single LUW (Logical Unit of Work).

The issue appears after implementing SAP Note 3527055, which previously resolved a problem where BADIs were not triggered during processing via the RAP BO.

Other Terms

setmaintplanstatustoinactive, setmaintplanstatustoactive, SetMainPlanStsToMrkdForDeltn, ResetMaintPlanStsMrkdForDeltn, mpla, mpos, warpl, if_eam_maintenanceplan_rap, cl_eam_maintenanceplan_rap, set_plan_status, mplan_read, mplan_read_multiple, RAP_EAM_MAINTENANCEPLAN, mplan_persistency

Reason and Prerequisites

The reason is unnecessary repeated database access.

Solution

Implement the attached correction instructions or the corresponding support package.

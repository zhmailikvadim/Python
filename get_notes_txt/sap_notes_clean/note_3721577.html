Symptom

  • During the Automatic Payment Run, the payment list printout is generated in German instead of English.
  • Although the vendor master language is maintained as English, the output form is still being printed in German.

"Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental." 

 

Environment

SAP S/4HANA Cloud Public Edition

Reproducing the Issue

  1. Open Manage Business Partner app and check Supplier Master Language -> English.
  2. Execute payment run using Manage Automatic Payments.
  3. Print the payment document generated.
  4. Observe that the output is displayed in German instead of English.

Cause

  • In Output Parameter Determination, the form FIN_FO_PAYM_LIST_V2_DE is assigned.
  • This form template is maintained in the German language.
  • Therefore, the payment list printout is generated in German, regardless of the language maintained in the vendor master.

Resolution

  1. Update the configuration in Output Parameter Determination to use the English template FIN_FO_PAYM_LIST_V2_EN.
  2. Verify that after this change, the Payment List output is now printing correctly in English.

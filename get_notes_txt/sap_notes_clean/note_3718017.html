Symptom

 After clicking on the "Confirm" button on the Operation object card the "Add time confirmation" modal page appears and if you click "Cancel", the operation will be confirmed. After an "Unconfirm" action and clicking on "Confirm" one more time and clicking on the "Done" button on the "Add time confirmation" modal page, "Confirmation already exists" warning appears.

Reason and Prerequisites

Reason:

Due to the wrong value of "isFinalConfirmation" state variable two blank confirmations are created which causes an issue with changing the "isFinal" flag in the confirmation object.

Prerequisites:

  •  BAS or Visual Studio Code with SAM MDK project
  • `patch` command (available on BAS and macOS by default) via terminal or other patching utility
  • Read the guidelines on how to apply the patch and read the appendix section if you receive any error

Solution

Fixed by setting the correct value of "isFinalConfirmation" state variable.

To apply the patch, download the attached 3718017.patch file and save it in a known location, preferably adjacent to your SAM.mdkproject.

If you are on Windows, see SAP Note 3533879 for how to install a .patch file in Windows. If you are on a Mac, follow the steps below:

  1. Open a terminal in the root of your SAM metadata (i.e. SAM.mdkproject/metadata or SAM.mdkproject/metadata/base.definitions)
  2. Run the terminal command `patch --no-backup-if-mismatch -p4 < path/to/3718017.patch`. 
  3. Verify in the terminal window that the following files have been modified and listed as patched, as indicated by the console output:
  • Rules/Operations/MobileStatus/OperationMobileStatusLibrary.js

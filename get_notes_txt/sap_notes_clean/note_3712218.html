Symptom

The required Pre-Persist-Hook header is missing in OData requests during base unit​ replication from ERP to SAP Commerce Cloud.

Reason and Prerequisites

The OOTB XSLT mapping BatchUnitsData.xsl only includes the persist-hook header for sales units, but omits it for base units.

Solution


Add four parameters in iflow Replicate B2B Customer From SAP ERP To SAP Commerce Cloud. This change has been included in package SAP Commerce Cloud Integration with SAP ERP 1.31.1. After upgrading the package, select the integration iflow, select Edit. Enter the following values for each field, as needed:
  • b2bCustomerPostPersistHook: If needed, define the custom PostPersistHook for the B2BCustomer receiver.
  • b2bCustomerPrePersistHook: If needed, define the custom PrePersistHook for the B2BCustomer receiver.
  • b2bUnitPostPersistHook: If needed, define the custom PostPersistHook for the B2BUnits receiver.
  • b2bUnitPrePersistHook: If needed, define the custom PrePersistHook for the B2BUnits receiver.

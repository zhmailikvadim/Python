Symptom

Symptom

You use program /FRE/FU_CALC_FCBASIS (transaction /FRE/TS_AGG) to calculate Aggregated Store Sales (time series AGG_ST_SLS) for the distribution center.

You realize that on days where calculated values for Aggregated Store Sales are zero system replaces these values with Aggregated Store Forecast (time series AGG_STORE_FC) values.

Other Terms

/FRE/TS_AGG, /FRE/FU_CALC_FCBASIS, Aggregated Store Sales, AGG_ST_SLS, ASS, Aggregated Store Forecast, AGG_STORE_FC, ASF, INIT, /FRE/FU_CL_TS_AGGREG, FILL_LTS_GAP_END_OF_HORIZON

Reason and Prerequisites

Program error.

Solution

Implement automatic correction instruction from this note or corresponding Support Package.

After implementation of this note system replaces Aggregated Store Sales values with Aggregated Store Forecast values only in case that Aggregated Store Sales value is initial (INIT flag is set).
In standard system INIT flag for Aggregated Store Sales can be set only before left border or after right border of time series.

Other Terms

Screen SAPLNBE2/2000, RNDUNF-UNFZT, RNDUNF-UNFDT, NV2000, NV2001, accident data, 00069, 00 069

Reason and Prerequisites

Program error.

Solution

Implement the correction instructions.

After you implement the correction instructions, you can no longer enter the accident time without specifying the accident date. In this case, the system issues error message 00069 ("Specify time within permitted value range") and thus forces the entry of the accident date. It does not make sense to enter the accident time without an accident date.

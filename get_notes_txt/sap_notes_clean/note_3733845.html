Symptom

  • In the "Target Plan RoW" workflow, the "Assigned to" field is cleared after a status change, preventing users from approving their target sheets.
  • The issue does not occur in the older "General Target Plan" workflow, which works as intended.
  • The "Change Status" action removes the assignee even though no script or event is attached to the action.
  • The issue persists across multiple actions in the same workflow, not just "Change Status".
  • In some cases, after attempting the workaround, the case becomes assigned to the performer (current user) instead of keeping the previous assignee.

Environment

  • SAP Sales Performance Management
  • Advanced Workflow

Reproducing the Issue

  1. Go to Workflow.
  2. Create Action "Target Plan RoW".
  3. Fill in required information and submit (Evaluator: Simon Wildmann, Evaluated Person: Mauricio Costa, BT Head: Simon Wildmann, Position: SI1-COSTAMAU). Other values can be filled with test data.
  4. In Case Overview, verify that "Assign to" is prefilled.
  5. Open the case and use the "Change Status" action (for example, Draft to Definition).
  6. Observe that the "Assign to" field becomes empty.

Cause

  • A bug in the new UI Actions page causes incorrect assignment logic, leading to the assignee being cleared after a status change.
  • If an action is created and saved in the new UI, subsequently saving it on the legacy Actions page can set an inadequate assignee type (PERFORMER), causing the case to assign to the current user.

Resolution

  1. For each affected action, open it using the legacy Actions page (use the option to switch to the old page).
  2. Save the action without making any changes to refresh the assignment metadata.
  3. Retest the workflow step; if the assignee now persists, repeat steps 1–2 only for other actions where the issue occurs.
  4. If the action was originally created and saved in the new UI and resaving on the legacy page assigns the case to the current user (PERFORMER):
  5. 1 Create a new replacement action directly on the legacy Actions page and configure it as required; or
  6. 2 Contact support to request a backend correction for the misassigned action.
  7. After confirming correct behavior, propagate the workflow to production using Migration Manager.

The Engineering team is conducting a thorough investigation to propose a permanent fix for this behavior, which is currently targeted to be implemented in the 2604 release. Please find below the expected 2026 default release dates for all versions:

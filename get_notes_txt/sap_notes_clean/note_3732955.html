Symptom

Project owner would like to enforce an upper and lower boundary for price in their event to prevent erroneous supplier bids.

Cause

Extended Price term had Acceptable Values set to Limited range, but the Delegate to initiator options were not enabled at the template level (under Lot Terms and under Bidding Rules), preventing the project owner from editing the range.

Resolution

There are two conditions which must be set at the template level to allow a range to be set to enforce an upper and lower limit for the price term. 

  1. On the term, the Acceptable Values field must be set to Limited Range.
    • Setting the acceptable values to limited range will allow the range fields (To: and From:) to appear on the event.
  2. In the template level Definitions, the Delegate to Initiator options must be checked in the Lot Terms and Bidding Rules for the desired lot type. 
    • Checking these two boxes allows the project owner of an event created from the template to edit the Range fields to enforce an upper and lower limit on the term.

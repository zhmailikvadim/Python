Symptom

  • In VF44, the deletion/blocking ID for older contracts is found not set (status changed).
  • A periodic job subsequently posts revenue for those items.
  • There is a need to retrieve change logs for the deletion/blocking ID for audit purposes.

Reproducing the Issue

  1. Open transaction VF44 (Revenue Recognition — Periodic Processing).
  2. Select the relevant sales document and item.
  3. Use “Set Blocking ID” or “Delete Blocking ID”.
  4. When the blocking/deletion ID is removed, observe that subsequent revenue posting is allowed by periodic processing.

Cause

The deletion/blocking ID was removed in VF44, releasing revenue recognition documents for posting.

Resolution

There is no SAP Standard functionality in SD Revenue Recognition to log username nor date/time for Set Blocking ID and Remove Blocking ID from VF44 transaction.

If there is a need to know if a user removed blocking ID and posted blocked SD documents through VF44 the local consultant may explore the ways available through SGL1 and SM20 transactions.

If there is a need to implement audit log mechanism then local consultant may explore custom implementation in enhancements and/or BTEs ( SD RR Best Practices Part 2.PDF on page 5 and 6 ( PDF is attached in SAP note 0001172799 ).

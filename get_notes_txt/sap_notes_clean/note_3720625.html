Symptom

The XML files generated for an A/R invoice and its cancellation are identical, making it impossible to distinguish between the original invoice and its cancellation.

Environment

SAP Business One

Reproducing the Issue

Prerequisites:

  • You are working with IT localization
  • You have set up the database according to SAP Note n. 2666498 - IT_Electronic Invoicing Central Note

Scenario:

  1. Go to Sales → A/R Invoice and create an electronic A/R Invoice. The electronic document is created successfully. 
  2. Re-enter the same document and create a cancellation. Export the xml for this document. 
  3. Compare the xml for A/R Invoice and its cancellation.

Expected behavior:

The two XMLs are different. It is clear which one is a cancellation.

Actual behavior:

The two XMLs are exactly the same. It is not possible to know which one is the cancellation. 

Cause

Functionality description

Resolution

This is correct system behavior because as per SAP Note 3081917 - IT_Electronic Invoice Changes for A/P and Communication, cancelation cannot be sent to authorities. The correct way to cancel an invoice is creating a credit memo. The cancelation invoice is only if invoice was created by mistake or with error, and before it is send to customer. 

I discussed it with Marian and he said the content is fine.

Symptom

  • Using Manage Launchpad Pages app.
  • The application "Open SAP Build Content Catalog" (U0074) is no longer visible in custom page.
  • It shows "Reference is broken".

"Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental." 

Environment

SAP S/4HANA Cloud Public Edition 2602

Reproducing the Issue

  1. Access Manage Launchpad Pages application 
  2. Access a custom page with a reference to application "Open SAP Build Content Catalog", business catalog SAP_CORE_BC_EXT_BLD_PC.
  3. Observe that the application "Open SAP Build Content Catalog" (U0074) is not visible with error message "Reference is broken".

Cause

As per 3659244 - Identity and Access Management (IAM): Central Change Overview for SAP S/4HANA Cloud Public Edition 2602 - SAP for Me

Delta_S4CE_2602_2508.xlsx > "TilesTMsDeleted-Moved" tab:

X-SAP-UI2-PAGE:X-SAP-UI2-CATALOGPAGE:SAP_CORE_BC_EXT_BLD_PC:3WO90XU93YRTTI15Y9H4A1KQX    42010AEF3EE61EEEBFAE0B39AF84C966    BC-SRV-APS-EXT-BLD            ExtensibilityItem    showContentCatalog    Standard Application    SAP_CORE_BC_EXT_BLD_PC    Extensibility - SAP Build    U0074    Open Content Catalog    Open SAP Build Content Catalog        Deprecated                                

                

The deprecated application "Open SAP Build Content Catalog" was removed from the catalog SAP_CORE_BC_EXT_BLD_PC, and the reference in custom page points to a deleted Target Mapping (TM).

Resolution

  1. Access the Manage Launchpad Pages in customizing tenant
  2. Access the custom page.
  3. Click on Edit
  4. Search by the application name "Open SAP Build Content Catalog".
  5. Add the new application as in the reference screenshot.

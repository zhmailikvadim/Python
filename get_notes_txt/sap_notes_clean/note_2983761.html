Symptom

You would like to execute InA/MDS requests on the secondary server in the Active/Active (Read Enabled) setup.

Other Terms

Ina/MDS, Active/Active, Read Only Server, Secondary Server, EXECUTE_MDS

Reason and Prerequisites

Reason:

If your HANA systems has Active/Active (Read Enabled) configured, you can leverage the secondary server for query execution which will reduce the server load of the primary systems.

Prerequisites:

  • Active/Active (Read Enabled) is configured in the HANA systems.

Solution

Execution of InA/MDS requests on the secondary server is supported from the following versions:

  • SAP HANA 2.0 database:
    • Revisions of SPS06
    • or higher
  • or higher

First, make sure the configuration for the query execution on the secondary server is not disabled explicitly.

Server Section Key Value (default in bold) Comment
indexserver.ini mds
enable_execution_on_secondary
 false, true Enable query execution on the secondary server of Active/Active (Read Enabled)

To execute InA/MDS requests on the secondary server, you have to use SQLDBC, JDBC, or HAA as the InA end-point.

SQLDBC/JDBC

When using SQLDBC or JDBC, you can use hint based statement routing to route the InA/MDS requests.

For example, CALL EXECUTE_MDS_READ_ONLY('Analytics','','','','',?,?) WITH HINT(RESULT_LAG('hana_sr'))

Because the secondary server only allows read only statements, EXECUTE_MDS_READ_ONLY procedure must be used instead of the regular EXECUTE_MDS procedure for it to be routed correctly.

Please refer to the following HANA guide for more details on the routing: https://help.sap.com/viewer/4e9b18c116aa42fc84c7dbfd02111aba/2.0.05/en-US/8c2a7541593b422295ae39ea51ad0608.html

HAA

To take advantage of the Active/Active (read enabled) system using HAA, simply set the SECONDARY_ROUTING_PERCENT parameter in your manifest.yml/mtad.yml

---
applications:
- name: xsahaa
  env:
    SECONDARY_ROUTING_PERCENT: 50

SECONDARY_ROUTING_PERCENT controls what percent of queries should be routed to the secondary server. If you want queries to be balanced equally between servers you would set it to 50. You can also disable this behaviour by setting it to 0.

Limitations:

Currently, following requests are considered write request and therefore not allowed on the secondary system:

  • Requests with EPM query source as a datasource.
  • Requests with ResultSetPersistance feature.

Symptom

The customer is up to perform a restore of the Microsoft SQL Server 2019 Transaction log backups through SQL Server Management Studio tool (SSMS) and while trying to to restore multiple Transaction log backup files at the same time, SSMS returns following error message:

"An exception occurred while executing a Transact-SQL statement or batch. (Microsoft.SqlServer.ConnectionInfo)Additional information:

Too many backup devices specified for backup or restore; only 64 are allowed.
RESTORE HEADERONLY is terminating abnormally. (Microsoft SQL Server, Error: 3205)"  


"Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental."

Environment

SAP NetWeaver on Microsoft SQL Server.

Reproducing the Issue

Apply multiple transaction log backups (multiple .BAK files) at once in SQL Server Management Studio tool (SSMS), selecting all transaction log backups/devices available.

Cause

SSMS does not give any option to specify groups of Transaction log backups (or folder) to restore in SQL Server Management Studio. 

Resolution

If you want to restore multiple Transaction log backup files at the same time, you will need to open multiple connections and run them at the same time.
It is not possible to define simultaneous restores in a single T-SQL statement; they will be completed one after the other.
As an alternative, you could create a customized script to generate SQL statements for restoring database from backup and applying all Transaction logs backups performed from last full database backup.

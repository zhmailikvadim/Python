Symptom

You are facing the BDOC error >P080 'Validity of role MKK must not be restricted' while replicating a BP between two systems. This only occurs between a certain time e.g 5pm to 7pm.

The KBA 1902462 - Error "validity of role MKK must not be restricted" during delta replication from CRM to ERP has already been consulted.

Environment

  • SAP Customer Relationship Management (CRM)
  • SAP enhancement package for SAP CRM
  • SAP enhancement package for SAP CRM, version for SAP HANA

Reproducing the Issue

  1. You are replicating a BP from a CRM system to an ECC system
  2. The BP you are replicating has role MKK
  3. You have tried corrections in the note: 2342596 - Time zone issue for valid from time when adding a new role for a BP.

Cause

  • The >P080 error message is used in the class CL_IM_VKK_PARTNER_ROLES and method CHECK
  • In the method there is a check of the MKK's Valid_From date with the current users' system date

Resolution

  1.  Go to transaction SU01
  2.  Enter the name for your RFCUSER
  3.  Check the timezone for this user
  4.  If the timezone is blank, please match the timezone to both systems

Keywords

Role MKK, Validity MKK, >P080, >P 080, CL_IM_VKK_PARTNER_ROLES, RFCUSER

Symptom

  • In transaction FCHI, the check lot number has reached its maximum value (9999), so a new check lot cannot be created.
  • FCHI cannot create Lot number 10000

Environment

  • SAP ERP
  • SAP ERP Central Component
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA

Cause

The check lot in transaction FCHI has reached its maximum limit (9999), which prevents creating additional check lots.

Resolution

The potential solutions are:

  1. Archive old check information (recommended): Use transaction SARA with archiving object FI_SCHECK to archive eligible checks (only cashed or voided). This frees up numbers and allows reuse without issues. Execute write and delete jobs per your retention policy and compliance guidelines, then create the required lot in FCHI.
  2. Add a leading zero to extend the range: In FCHI, split the existing interval to isolate any unused portion and define a new interval with a leading zero in the check number range. This avoids overlap and enables continued use. Refer to the linked notes for detailed guidance and typical message handling.
  3. Manually delete obsolete check information (use with caution): Depending on configuration, remove old check records via:
  • FCHD (checks from payment runs),
  • FCHE (manually voided checks),
  • FCHF (manual checks).

     Always run in test/simulation first and carefully restrict selection criteria to avoid unintended deletions. After cleanup, create the new lot in FCHI.

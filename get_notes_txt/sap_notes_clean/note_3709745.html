Symptom

When using the SAP HA Script Connector Library (SAP note 1693245) under Linux,

  1. [With SAP HA Script Connecor Library patch level below patch level in "Support Packages & Patches" section of this note]
    sapstartsrv hangs in an endless loop.

    Following log lines appear in sapstartsrv.log:
    SAP HA Trace: start_system_cmd: timeout expires
    SAP HA Trace: start_system_cmd: timeout expires
    SAP HA Trace: start_system_cmd: timeout expires
    SAP HA Trace: start_system_cmd: timeout expires
    SAP HA Trace: start_system_cmd: timeout expires
    [...]

  2. [With SAP HA Script Connecor Library patch level equal or higher than patch level in "Support Packages & Patches" section of this note]
    HA Setup does not work correctly and HA Script Connector Libary does not react as expected.

    Following log lines appear in sapstartsrv.log:
    SAP HA Trace: start_system_cmd: waiting for pid 1234
    SAP HA Error: start_system_cmd: waitpid failed for pid 1234: 10 No child processes -1
    SAP HA Error: start_system_cmd: return code of script unknown due to failed waitpid, returning error, see SAP note 3709745.
    SAP HA Trace: list_sap_resource: Script returns rc=-1
    SAP HA Error: SAP_HA_FindSAPInstance: ERROR Connector /usr/bin/sap_cluster_connector returns -1 -> SAP_HA_NOT_CLUSTERED

Other Terms

 SAP HA Script Connector Library

Reason and Prerequisites

Under unknown conditions the Linux/UNIX standard parent/child process handling does not work as expected.
The 'waitpid' system call does not work and the parent process does not get the (usually important) return code from the child process.

The reason for this behavior is still unknown and under investigation.

For previous SAP HA Scipt Connector Library versions this causes an endless loop in the SAP HA Script Connector Library.

Solution

    1. Fix of SAP HA Script Connector Library

      Apply the patch attached in the "Support Packages & Patches" section of this note.
      It is not required to replace and restart the complete SAP Kernel. Replacing the SAP HA Script Connector Library (saphascriptco.so) is sufficient.
      Restarting SAP Start Services after replacement is required.

      This only fixes the wrong endless loop behavior of the SAP HA Script Connector Library.
      It does NOT fix the problem with the unability of a parent process to get the return code of the child process (i.e. ususally a script provided by HA vendor).

    2. Find the root cause

      Please create a Support Case in order to find a possible root cause on your system. We suspect 3rd Party software that changes Linux kernel behavior, but this is only an assumption.

    3. Workaround

      Following workaround is available that solves the issue, but does not help SAP or Linux Distributor or HA Vendor or 3rd Party to fix this issue.

      Set profile parameter:
        service/halib_vendor_trace_mode = 0

      This disables HA Vendor Trace (stdout/stderr of vendor script is logged in sapstartsrv.log).

      It disables the parallel work of parent process sapstartsrv (reading pipes connected to child process and logging) during run time of the called script.

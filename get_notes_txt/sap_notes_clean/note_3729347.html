Symptom

When the job approver selects "Approve the whole series" for the recurring DRTM master purge job, the approver status will no longer be checked after the first job runs successfully. Even if the approver's status later becomes inactive, the job will still execute.

Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.

Environment

SAP SuccessFactors HCM Suite

Reproducing the Issue

1.Go to Admin Center

2.Data Retention Management-->Create New Purge Request-->Select DRTM Master Data Purge-->Add approvers-->Schedule this job to recur

3.Login as Approver-->Purge Request Monitor-->Requests Pending Initial Approval-->Initial Approve-->Select Approve the whole series

4.Job runs successfully

5. When the approver's status changes to inactive, the job still runs successfully

Cause

This is an expected behavior. 

When the approver selects "Approve the whole series," only the first job will check the user's status and permissions. After the first job runs successfully, the system will not check the user's status for subsequent jobs. 

Resolution

If you don't want the scheduled recurring job to be automatically approved, the approver can select "Require final approval for each occurrence." This requires the approver to approve each recurring job, allowing for the check of the approver's status each time.

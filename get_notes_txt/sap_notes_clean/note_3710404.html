Symptom

The PDF visualization for Poland's National e-Invoicing System (KSeF) does not display all the required data fields. Additionally, the current layout contains several visual inconsistencies, missing legal annotations, and incorrect data handling, which can lead to incomplete or non-compliant invoice documents for end-users.

Other Terms

Reason and Prerequisites

This note introduces a new version of the PDF visualization to align with recent updates to the KSeF logical structure and to address reported bugs and layout improvements. The changes ensure greater compliance, improved readability, and a more user-friendly presentation of the invoice data.

This note provide information about the changes included to the PDF visualization for Polish e-invoices. The changes are detailed below and are grouped by functional area.

Note Version 1:
  1. New Fields and Data Points Added

The PDF layout has been enhanced to include the following new fields, ensuring all relevant data from the XML is visible:

  • DaneIdentyfikacyjne/IDWew: The IDWew field from the DaneIdentyfikacyjne section is now displayed in Podmiot3 section.
  • Fa/KursWalutyZ: The exchange rate used for tax calculation (KursWalutyZ) is now visible when an invoice is issued in a foreign currency.
  • Correction Invoice Data:
    • Fa/Podmiot1K: The Corrected Seller section is now fully supported and displayed in correction invoices.
    • Fa/Podmiot2K: The Corrected Buyer section is now fully supported and displayed in correction invoices.
    • Fa/Podmiot1K/GLN: The Global Location Number (GLN) for the Corrected Seller (Podmiot1K) is now included.
  • Party Identifiers:
    • NrID: Tax identifiers for Podmiot3 (Third Party) are now displayed.
    • Fa/Podmiot2: Local Government (JST) or Governmental (GV) unit identifiers for the Buyer (Podmiot2) are now correctly shown.
  • Invoice Line Item Details (FaWiersz):
    • Fa/FaWiersz/P_12_XII: The excise duty amount (P_12_XII) is now displayed in the line item table.
    • Fa/FaWiersz/P_12_Zal_15: The flag indicating product or service is listed in Attachment 15 of VAT Act (P_12_Zal_15) is now included.
    • Fa/FaWiersz: Item classification codes GTIN, PKWiU, CN, and GTU are now visible for each invoice line.
  • Payment and Settlement:
    • Fa/Rozliczenie/DoRozliczenia: The amount remaining to be settled (DoRozliczenia) is now displayed in the Settlement (Rozliczenie) section.
    • Fa/Platnosc: The Paid Amount (Zaplacono) section has been adjusted to correctly calculate and display values in all scenarios.
  • Tax Summary Table:
    • Fa/Podsumowanie: The field P_13_6_3 has been added to the tax summary.
  1. Legal Annotations and Mandatory Phrases

The following legally required annotations and indicators are now correctly displayed on the PDF when present in the XML data:

  • P_17: Self-Billing is now displayed when the invoice is issued under a self-billing agreement.
  • P_18: Reverse Charge: The "odwrotne obciążenie" annotation is now displayed when the P_18 flag is set.
  • Fa/Adnotacje/P_23: Split Payment: The "Mechanizm podzielonej płatności" annotation is now included when required.
  • Fa/FaWiersz/P_12_XII: The tax amount for OSS procedure (P_12_XII) is now displayed in the line item table.
  • Fa/Podmiot3: Third Party Roles: Descriptions for the role of Podmiot3 are now included for better context.
  1. Layout Enhancements and Visibility Rules

To improve clarity and reduce clutter, the layout has been optimized with dynamic visibility rules:

  • Elimination of Empty Tables:Tables that previously showed headers but had no content (e.g., extra columns for invoice items) will now be completely hidden if they are empty. This applies to several sections to ensure a cleaner look.
  1. Table Restructuring and Fixes

Several sections and tables have been adjusted for better alignment and correctness:

  • Fa/Transport: Transport Section: The layout of the transport details table has been revised for improved readability.
  • Fa/Platnosc: Payment Information Section: This section has been updated to correctly display all payment-related fields.
  • Fa/Platnosc: Functional Link for Cashless Payments: Where applicable, the payment section now includes a link to facilitate cashless payments.

Note Version 2:

  • Fa/PodmiotUpowazniony: Full support for displaying the details of the Authorized Entity (PodmiotUpowazniony) section when present in the XML.
  • Fa/Podmiot2/BrakID: The flag indicating that the Buyer has no identifier is now supported and displayed.
  • Fa/Podmiot3/IDNabywcy:   The Buyer's ID within the Podmiot3 section system is now visible.
  • Fa/Podmiot3/GLN:  The Global Location Number (GLN) for the Podmiot3 is now included.
  • Fa/Podmiot3/AdresKoresp: The Correspondence Address for the Podmiot3 is now displayed.
  • Fa/Podmiot3/NrKlienta: The Customer Number assigned by the Seller in the Podmiot3 is now shown
  • Fa/ZaliczkaCzesciowa: The section for Partial Advance Payments is shown.
  • Fa/Platnosc/PlatnoscInna: Other Payment Method information is displayed.
  • Fa/Platnosc/OpisPlatnosci: A detailed description of the other payment method is displayed.
  • Fa/Platnosc/RachunekBankowy/RachunekWlasnyBanku: The "Own Bank Account" flag for standard bank accounts is presented.
  • Fa/Platnosc/RachunekBankowyFaktora/RachunekWlasnyBanku: The "Own Bank Account" flag for the factor's bank account is presented.
  • Fa/Adnotacje/NoweSrodkiTransportu: The full section indicating intra-community supply of new means of transportation is now displayed, including vehicle details and reference to invoice items.

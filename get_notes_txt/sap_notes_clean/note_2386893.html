Symptom

While creating access request, role search show results from only one CUA system other CUA system roles ignored

Environment

Access Control

  • Access Request

Resolution

When in TCODE SPRO, the parameter "CUA Global System" is assigned with a particular connector and the same is enabled as "Active" than role search in access request screen only fetches from the same connector only. In case, customer wants access request to fetch roles from other systems than they need to disable the "Active" option from parameter "CUA Global System" by following the navigating in TCODE SPRO Governance, Risk and Compliance-->Access Control-->User Provisioning-->Maintain CUA Settings

Keywords

No roles,roles missing

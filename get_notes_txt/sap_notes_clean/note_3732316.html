Symptom

• Due to changes in the team, a user requests to reassign existing VPN tokens to another team member who does not have access.

• Instead of creating a new VPN token, the user prefers to reassign the existing one. 

Environment

  • SAP Sales Performance Management
  • Incentive Management
  • Incentive Management on Legacy Oracle
  • Incentive Management on Legacy HANA 

Reproducing the Issue

1. Identify the team member who has left the team.

2. Verify that the new user does not already have access to the VPN token.

3. Request reassignment of the existing VPN token instead of creating a new one. 

Cause

A team member has left the team, and the existing VPN token can be reassigned to another user instead of creating a new one.

Resolution

1. Raise a P3 Support ticket to SAP Support.

2. Attach the VPN excel sheet and complete the reassignment columns with the required details. 

3. Include the manager's approval for the reassignment of the VPN token.

4. Provide the full name and email address of the user whose VPN token should be reassigned and the user to whom it should be reassigned.

5. The VPN token will be reassigned to the new email address.

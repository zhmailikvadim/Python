Symptom

Freight documents cannot be filtered before integrating TM with event management. This capability may be needed in bulk processing scenarios, for example e.g.: creation of a settlement for multiple freight documents.

Other Terms

Event Management, EM-TM Integration, CALL_EM_TOR_DATA

Reason and Prerequisites

This is due to Program Error

Solution

Implement this SAP Note or upgrade to the relevant Support Package.

With this SAP Note, a new method: FILTER_RELEVANT_TOR has been introduced in the BAdI: /SCMTMS/SEND_TOR_DATA which can be used to remove those TOR objects that are not relevant for EM integration during the current process.

To use this method for filtering data, you need to additionally redefine the method: /SCMTMS/IF_COMMON_BADI~SET_BADI_WORK_MODE and add a new entry to the changing parameter CT_WORK_MODE with the following values:

(a) INTF_METHOD_NAME = /SCMTMS/IF_SEND_TOR_DATA=>MC_INTF_METHOD_FILTER_REL_TOR

(b) BADI_WORK_MODE = /SCMTMS/IF_COMMON_BADI~GC_MODE_MODIFY_STAND_RESULTS

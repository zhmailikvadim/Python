Symptom

While calling https://graph.microsoft.com/v1.0/me/sendMail API with HTTP receiver adapter the following errors can be found in CPI:

  • Error in CPI message: org.apache.camel.component.ahc.AhcOperationFailedException: HTTP operation failed invoking https://graph.microsoft.com/v1.0/me/sendMail with statusCode: 401
  • Error in Response Headers: {"error":{"code":"InvalidAuthenticationToken","message":"Access token validation failure. Invalid audience.","innerError":{"date":"2026-02-17T12:40:13","request-id":"9aef0feb-fbcd-4a0f-97af-5cd1040635cb","client-request-id":"9aef0feb-fbcd-4a0f-97af-5cd1040635cb"}}}

Environment

  • SAP Integration Suite
  • SAP Cloud Integration

Cause

  • The Scope in the CPI Security Material credential was set to "https://outlook.office.com/SMTP.Send offline_access", which requests a token for Outlook (outlook.office.com).
  • When this token was sent to Microsoft API, it was rejected due to a mismatch in the audience, resulting in the "Invalid audience" error.

Resolution

To resolve this issue, the Scope needs to be corrected in the Security Material.

  1. Update the Scope in the CPI Security Material credential to "openid profile offline_access https://graph.microsoft.com/Mail.Send".
  2. Save the updated credential.
  3. Re-fetch and re-consent the credential with the sending mailbox user.
  4. Verify that the CPI receives a token issued for Microsoft API with the correct audience "https://graph.microsoft.com".
  5. Test the /v1.0/me/sendMail call to confirm successful execution.

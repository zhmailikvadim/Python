Symptom

After enhancing the data model BP with custom Entities/ attributes, performing the adjust staging in IMG :'MDGIMG->General Settings->Data Modeling->Edit Data Model->BP->Adjust staging area of linked change requests' is resulting in error "system cannot schedule conversion program USMD_ADJUST_STAGING in client XX(model BP)".

Environment

  • SAP Master Data Governance-Central Governance

Reproducing the Issue

  1. Launch MDGIMG
  2. Navigate to IMG : MDGIMG -> Edit Data Model -> Adjust staging area of linked Change request
  3. Error is raised : "system cannot schedule conversion program USMD_ADJUST_STAGING in client XX(model BP)"
  4. Dumps are raised in system
    Category               Resource bottleneck
    Runtime Errors         TSV_TNEW_PAGE_ALLOC_FAILED 

Resolution

  • It appears that the DDIC user was missing certain authorizations in system, which resulted in the message USMD1407. After you adjusted these authorizations, the messages should be resolved.

  • Although the job was triggered successfully, it seems that the DDIC user is locking the BP data model due to ongoing activities in the system, possibly in client 000. As a result, the BP data model remains locked in the database table USMD_CR_BLOCK. Once the BP data model is unlocked, you should be able to rerun the adjust staging job and proceed with unlocking it.
  • For the batch jobs of USMD_ADJUST_STAGING, the usage of the DDIC user is hard coded in the routines which create the batch jobs. The USMD_ADJUST_STAGING job internally switches to DDIC user context for certain system-level operations, regardless of the job owner. 

Keywords

USMD1407, Adjust staging, USMD_ADJUST_STAGING, DDIC, Adjust staging area of linked Change request, MDGIMG, MDG

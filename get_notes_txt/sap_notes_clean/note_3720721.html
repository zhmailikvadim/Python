Symptom

In the SAP EarlyWatch Alert (EWA) for an SAP system running on SAP HANA, a red rating is reported due to the finding:

SAP HANA database: Number of Records exceeds critical limit

The finding is reported for a specific table (for example a custom Z* table) whose record count is stable and not increasing over time.

The customer attempts to suppress the finding by following SAP Note 3338242 and maintaining an entry in:

_SYS_STATISTICS.STATISTICS_EXCLUDE_TABLES

After maintaining the entry, the EarlyWatch Alert continues to report the finding and the rating remains unchanged.

Environment

SAP HANA DB Version 2.0

Cause

This behavior is by design.

The situation results from the use of different monitoring layers:

  • SAP HANA database alerting (database monitoring framework / SAP HANA Cockpit)
  • SAP EarlyWatch Alert (service-level evaluation content based on collected system data and evaluated in SAP for Me / SAP Solution Manager)
     

The table _SYS_STATISTICS.STATISTICS_EXCLUDE_TABLES is evaluated within the SAP HANA database monitoring framework and affects database alerts generated by SAP HANA internal monitoring (for example HANA Alert 17).

The EarlyWatch Alert rating is determined by SAP service content checks based on collected system information and does not evaluate the SAP HANA monitoring framework (used by SAP HANA Cockpit).

Therefore, maintaining entries in _SYS_STATISTICS.STATISTICS_EXCLUDE_TABLES does not change the EarlyWatch Alert evaluation.

Resolution

No database correction or SQL modification is required.

According to SAP Note 1909763, a high record count in a non-growing table alone does not necessarily represent a database problem.

For known and accepted findings, the finding can be handled using the Hide or Snooze functionality of the EarlyWatch Alert Workspace (see SAP KBA 2520319).

This applies only to the selected finding in the specific system and does not affect monitoring of other tables.

The table _SYS_STATISTICS.STATISTICS_EXCLUDE_TABLES belongs to SAP HANA database alert configuration and is not intended to control EarlyWatch Alert ratings.

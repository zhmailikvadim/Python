Question
  1. How can I correct the User '' not found error for my Import Legacy Contracts?

  2. How can I correct the Of ContractId instances with ='[BaseId]', one has DocumentId [BaseId], but another has (missing argument)ariba.collaborate.contracts.ContractWorkspace error for my Import Legacy Contracts?
Answer

To correct this issue, please check all comma-separated value (CSV) files of the upload for extra commas or characters below the necessary information.

The one referencing "user" is most likely coming from the Contracts.csv

The one referencing "Of ContractID" is coming from either the ContractDocuments.csv or ContractTeams.csv

  1. Open the CSV files in a text editor program (Microsoft® Notepad, Notepad++).
  2. Check below the final row of actual data for extra rows of only commas or characters and delete them.
  3. Save the files, re-zip, and try the upload again.

Example:

These commas can appear if you reuse a CSV file as a template that contained more data in a previous load.

Opening in a Text Editor program.

  • Right click the CSV file

  • Select Edit with Notepad++ (if installed) or click Open With > Notepad

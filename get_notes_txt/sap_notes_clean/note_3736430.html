Symptom

  • Routing performance forms via the Route Form tool or forwarding from form buttons fails.
  • Error displayed: route form failed: Wrapped Exception: Wrapped Exception: Exception caught: DAOException get sysconfig type: pm_only_save_to_feedback Key: pm_only_save_to_feedback.
  • While routing the form via route form tool or from user inbox, it shows successful message but the form doesn't route.
  • System logs may show: The How vs What matrix is misconfigured. Each Axis must contain either a Competency or Objective scale. Found: Performance scale.

Environment

  • Product: SAP SuccessFactors Performance & Goals

Reproducing the Issue

  1. Go to Admin Center > Route Form.
  2. Enter a performance form id.
  3. Attempt to route the form to any step or forward from the form buttons.
  4. Observe the error “route form failed: Wrapped Exception: Wrapped Exception: Exception caught: DAOException get sysconfig type: pm_only_save_to_feedback Key: pm_only_save_to_feedback.”

Cause

A misconfiguration in the How vs. What Matrix Grid Report: an axis is set to use a “Performance” scale. Each Axis (axis) must use either an Objective scale (“How”) or a Competency scale (“What”). The presence of a Performance scale leads to routing failures.

Resolution

  1. Go to Admin Center > Matrix Grid Reports: Performance-Potential or How vs. What.
  2. Review the configuration for both axes (Axis) of the How vs. What Matrix Grid Report.
  3. Change any axis currently set to “Performance” to the correct element (Trend): use Objective for the “How” dimension and Competency for the “What” dimension.
  4. Verify the process (source of ratings) for each axis (for example, Live Profile, Performance Forms, or Calibration Sessions) is correctly selected and consistent with the chosen element.
  5. Save the configuration.
  6. Re-test by routing a performance form again via the Route Form tool or from form buttons to confirm the error no longer occurs.

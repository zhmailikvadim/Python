Symptom

System becomes unavailable because a tablespace gets full, though DBACockpit (DB02) shows AUTOEXTEND ON for the same tablespace.

Environment

  • Oracle any version
  • Any OS platform
  • SAP Netweaver

Reproducing the Issue

Issue will show up during regular system usage.

Cause

The AUTOEXTEND feature is enabled at the datafile level not at the TABLESPACE, and SAP plays no role in that this is controlled by Oracle itself. DBACockpit (DB02) will may show AUTOEXTEND ON and still system runs out of space because there's no more possible growth for the available datafiles.

Resolution

IMPORTANT: Enabling AUTOEXTEND by itself does not guarantee no lack of space errors will happen at the database, since the feature only applies to the datafile level, and there's a maximum size for each file.

Make sure there's always datafiles with AUTOEXTEND enabled that have not reached the MAXSIZE yet, when that happens (ideally before that fact) new datafiles must be added using BR*Tools.

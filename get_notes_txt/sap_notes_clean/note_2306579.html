Symptom

Exchange of custom properties like Document metadata, Classification etc., from SAP Easy DMS to the Microsoft Applications (like Word, Excel) is not working for Office 2007 formats and above (like DOCX, XLSX).

Other Terms

Property Exchange, New file formats, EASYDMS_OI_R3_TO_FILE, RespectOfficeCustomizing.

Reason and Prerequisites

Pre-requisites:

1. The Registry entry 'PropExchangeActive' should be set to value '1' in the path: "HKEY_LOCAL_MACHINE\SOFTWARE\SAP\Easy DMS\General"

2. The parameter 'USE_SAPGUI' should be set to the value '2'. For more details on this, please refer to SAP Note: 1393924.

3. Implement SAP Note: 2306557.

Solution

This functionality will be supported for Microsoft Applications like Word, Excel and Power point from SAP Easy DMS 7.1 SP06 onwards. The exchange of properties will work from SAP Easy DMS to the Microsoft office applications.

In addition, a new registry entry  'RespectOfficeCustomizing' is introduced of type REG_DWORD in the HKLM Section ("HKEY_LOCAL_MACHINE\SOFTWARE\SAP\Easy DMS\General") for SAP Easy DMS to support Office customizing in DC30 transaction. The default value of this registry entry is '0' which means by default Office Integration customizing is not supported.

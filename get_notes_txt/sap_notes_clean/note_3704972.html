Symptom

SAP Build Process Automation job fails with the Document information extraction service. The error below is thrown:

Could not get status for templates: "unable to get local issuer certificate"

Environment

  • SAP Build Process Automation
  • Document Extraction service version 
  • SAP Intelligent Robotic Process Automation

Reproducing the Issue

Run an automation that uses the Document Information Extraction service activities.

Cause

The issue is related to the certificate update involving the G5 certificate on Cloud Foundry in 2025, reported in the following resources:  

  • SAP BTP Cloud Foundry: switching to higher securit... - SAP Community   
  • KBA 0003566727 - Root Certificate Replacement in the SAP BTP, Cloud Foundry Environment  

During automation runtime, the Digicert G5 certificate is searched locally on the machine/current user the agent is installed on. If the G5 Digicert certificate cannot be found the automation fails. 

Resolution

To resolve this, the G5 certificate needs to be imported into operating system.

  1. Download the G5 certificate from DigiCert at: DigiCertTLSRSA4096RootG5.crt.
  2. Verify whether the agent installation is configured per user or per machine:
    • On the Desktop Agent, Navigate to About > Agent Details > User Value.
    • If the user value matches the currently logged-in session user, install the certificate for the current user only, rather than the entire machine.
    • If the user value does not match the currently logged-in session user, install the certificate for the entire machine.
  3. Install the certificate into the operating system:
    1. For Mac OS: Refer to Import and export keychain items using Keychain Access on Mac - Apple Support.
    2. For Windows: Refer to Install imported certificates.

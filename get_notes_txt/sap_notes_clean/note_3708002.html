Symptom

You are encountering the following or a similar error message when querying a remote Microsoft SQL Server Database via Smart Data Access.

  • [Microsoft][ODBC Driver XX for SQL Server] [SQL Server] Reference to database and/or server name in '<MSSQL OBJECT>' is not supported in this version of SQL Server. For query "<SQL QUERY>"

Other Terms

Federation; SDA; MSSQL; Azure; Microsoft;

Reason and Prerequisites

Reason:

The error message, which commonly occurs in Azure SQL Databases, indicates an attempt to perform cross-database or cross-server queries. Unlike on-premises SQL Server, Azure SQL Database is a fully managed service that enforces isolation between databases. As a result, direct referencing using three-part (e.g., Database.Schema.Table) or four-part (e.g., Server.Database.Schema.Table) naming conventions is not supported.

Prerequisites:

  1. Smart Data Access connection to MSSQL (Azure) and
  2. Cross-database / cross-server queries are executed in MSSQL (Azure).

Solution

Please note that Microsoft Azure SQL Database is supported in HANA Cloud only through the Data Access Agent, and in HANA 2.0 only through Smart Data Integration, as outlined in SAP Note 2600176.

Any other scenario is not supported. For supported configurations, please contact Microsoft Support for further guidance.

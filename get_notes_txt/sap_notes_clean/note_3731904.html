Symptom

You are creating a new DIF assignment using the Manage Demand Influencing Factors app in SAP Customer Activity Repository applications bundle 5.0 FPS02, 5.0 FPS03, 5.0 FPS04, 5.0 SPS05, 5.0 SPS06, 5.0 SPS07, 5.0 SPS08, 6.0 SPS01 or 6.0 FPS02. You select a DIF with one of the following assignment levels:

  • 01 - Product, Location, Sales Organization, Distribution Channel, Order Channel
  • 02 - Location, Sales Organization, Distribution Channel, Order Channel

The multichannel fields Order Channel, Distribution Channel, and Sales Organization are not displayed and cannot be assigned.

Other Terms

CARAB 5.0 FPS02, CARAB 5.0 FPS03, CARAB 5.0 FPS04, CARAB 5.0 SPS05, CARAB 5.0 SPS06, CAR, CARAB, SAP FIORI FOR SAP CARAB, SAP CARAB 5.0 FPS02, SAP CARAB 5.0 FPS03, SAP CARAB 5.0 FPS04, SAP CARAB 5.0 SPS05, SAP CARAB 5.0 SPS06, SAP CARAB 5.0 SPS07, SAP CARAB 5.0 SPS08, SAP CARAB 6.0 SPS01, SAP CARAB 6.0 FPS02, Manage DIFs, Manage Demand Influencing Factors, user DIF, DIF assignment, demand influencing factor, DIF, Unified Demand Forecast, UDF, SAP CARAB, forecasting, modeling, DIF validity, working with DIF assignments, forecast app, app for demand forecasting, UDF app, app for demand forecasting, demand influencing factor, DIF, DIF library, forecast simulation, simulation

Reason and Prerequisites

Reason

The location block containing the multichannel fields is loaded asynchronously before the data model is set in newer SAPUI5 versions.

Prerequisites

You have set up the Manage Demand Influencing Factors app as described in the Common Installation Guide (for new installations) or the Common Upgrade Guide (for upgrade scenarios) for SAP Customer Activity Repository applications bundle 5.0 FPS02, 5.0 FPS03, 5.0 FPS04, 5.0 SPS05, 5.0 SPS06, 5.0 SPS07, 5.0 SPS08, 6.0 SPS01 or 6.0 FPS02. You can find both guides on SAP Help Portal at https://help.sap.com/docs/CARAB. Select your version at the top.

Solution

Implement the attached correction instructions in your front-end system.

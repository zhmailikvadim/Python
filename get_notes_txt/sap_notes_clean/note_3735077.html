Symptom

When opening a Fiori Application or making an OData V4 call against the Netweaver Gateway, an error similar to the following is presented, where <service_name> is the OData V4 service name and <current user> is the user making the OData call:

"No system alias found for service group '<service name>' and user '<current user>'"

Environment

Netweaver Gateway

Cause

A system alias was not defined for a routing-based OData V4 service, so no backend is available to service the request.

Resolution

  1. Open the transaction N/IWFND/V4_ADMIN in the NetWeaver Gateway system
  2. Select the service group referenced by the error message
  3. Assign the appropriate system alias for the OData V4 service by clicking "Add Assignment"

See Also

Refer to: 3604891 - How to assign a system alias to an OData V4 service?

Symptom

In transaction FAGLB03, the corresponding SU22 or SU24 entry for the check on the objects F_BKPF_KOA and F_BKPF_BLA is not defined correctly.

This must be checked because it is used in a possible call to FB03/FB02.

Other Terms

FAGL_ACCOUNT_BALANCE, FAGLB03, FAGLL03, SU22, SU24, F_BKPF_KOA, F_BKPF_BLA

Reason and Prerequisites

Incorrect customizing

Solution

Import the support package or implement the attached correction instruction.

The correction implementation updates the SU22 data of transaction FAGLB03 only.
In order to copy the changes to the SU24 data start the "Expert Mode for Step 2" (Ctrl+F1) in the initial screen of transaction SU25 and execute it for transaction FAGLB03 by retaining all default settings.

Due to the application concept in FI it suffices to change the maintenance status of F_BKPF_BLA and F_BKPF_KOA from "Authorization Check Inactive" (check indicator "No Check") to "No default" (check indicator "Check"). Authorization default values for both objects are not required.

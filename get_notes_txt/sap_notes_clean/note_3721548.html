Symptom

A service order is created in SAP ECC or SAP S/4HANA and successfully replicated to SAP Field Service Management Cloud.

On the FSM Planning Board:

  • The service call (service order) is visible.

  • However, no activity is displayed or assigned to the service call.

In standard scenarios, an activity is created and transferred together with the service order. In this case, the activity is missing.

Environment

  • SAP Field Service Management (FSM)
  • Proaxia Connector (PACG)
  • ECC, S/4 HANA

Reproducing the Issue

Expected Behavior

  • A service order is created in ECC/S/4HANA.

  • The service order and its activity are replicated to FSM Cloud.

  • The activity is visible on the FSM Planning Board and can be assigned to a technician.

Actual Behavior

  • The service order is successfully created and replicated to FSM Cloud.

  • The service call appears on the FSM Planning Board.

  • No activity exists for the service call.

Cause

This issue can occur due to one or more of the following reasons:

  1. Incorrect or incomplete customizing in ECC/S/4HANA

    • Relevant customizing tables are not properly maintained.

    • Required configuration entries for activity creation are missing.

  2. Service order operation does not meet customizing requirements

    • The operation master data does not fulfill the conditions required for activity creation.

    • Mandatory fields or control keys are missing or incorrectly maintained.

  3. Service order created before FSM integration was implemented

    • The service order was created before the FSM connector was activated.

    • The activity was never generated for the existing service order.

    • Therefore, no activity was available for replication.

Resolution

In the standard configuration of proaxia’s FSM Cloud Connector, an activity is automatically created once the service order is released.

However, activity creation can be restricted by configuration settings based on the following parameters:

  • Control Key

  • Planner Group

  • Work Center

  • Status

Follow the steps below to analyze and resolve the issue.

1. Check Whether the Activity Was Created

  1. Open the service order.

  2. Use the GOS (Generic Object Services) icon.

  3. Navigate to the Relationships section.

Result

  • If the activity exists → Proceed to section 2.

  • If no activity exists → Proceed to section 3.

2. Activity Exists but Is Not Visible in FSM

If the activity is visible in the GOS relationships:

  1. Open transaction WE09.

  2. Search for outbound IDoc type: /PACG/ECM_ACTIVITYOUT

  3. Use the activity number as selection criteria.

 

Check the IDoc Status

If the IDoc status is 41 and the activity is not visible on the FSM Planning Board:

  • Verify whether the connector setup program has been run.

  • Compare the connector setup version with the current entries in table: /PACG/ECS_SETUP

  • If inconsistencies are detected, run transaction: /PACG/ECM_SETUP

This ensures that the connector configuration matches the maintained setup values.

3. No Activity Was Created

If no activity exists in the GOS relationship, verify whether the operation data meets the activity generation criteria.

Compare the service order operation values with the configuration maintained in the following transactions:

Control Keys 

  • Defines control keys relevant for activity generation.

  • Transaction: /PACG/ECM_TKDEF

     
     
  • Documentation: /PACG/ECM_TKDEF - Control keys relevant for activity generation - FSM Cloud Connector - Confluence 

  • Defines planner group filters for service calls.

  • Transaction: /PACG/ECM_SC_PGF

     
  • Documentation:  [2.18] /PACG/ECM_SC_PGF - Planner group filter for service calls - FSM Cloud Connector - Confluence 

  • Defines work center inclusion/exclusion rules.

  • Transaction: /PACG/ECM_SCTD_WCE

  • Documentation:  /PACG/ECM_SCTD_WCE - Workcenter exclusion/inclusion for service calls - FSM Cloud Connector - Confluence 

  • Defines relevant service call statuses for activity creation.

  • Transaction: /PACG/ECM_SCSTD

  • Documentation:  /PACG/ECM_SCSTD - Status of Service Call - FSM Cloud Connector - Confluence 

 

Ensure that:

  • The control key is allowed for activity generation.

  • The planner group is not excluded.

  • The work center is not filtered out.

  • The service order has a status that permits activity creation.

    4. Manually Trigger Activity Creation

    If all parameters are correctly maintained and the activity still does not exist:

    1. Open the service order in change mode.

    2. Save the document again.

    Alternatively, use the transaction to generate the activity: /PACG/ECM_ACT_GEN

    • [2.18] /PACG/ECM_ACT_GEN - Mass Activity Generator - FSM Cloud Connector - Confluence

    5. Re-Trigger Activity Transfer to FSM

    If the activity exists but is still not transferred:

    1. Run transaction: /PACG/ECM_TRIG_SCALL

    2. Select the option: Send activity

    3. Run the report and review the application log.

    The log will indicate whether:

    • The activity is relevant for sending, or

    • It was filtered out due to configuration.

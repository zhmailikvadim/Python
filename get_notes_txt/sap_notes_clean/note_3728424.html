Symptom

  • Internal hires initiated from the onboarding process do not appear on the onboarding dashboard.
  • Timezone error observed for internal hires in the system.
  • Error message: "HRIS_ERROR_INVALID_TIMEZONE".

Environment

SuccessFactors Onboarding

Reproducing the Issue

  1. Proxy as a user.
  2. Navigate to the "Recruiting" module in SuccessFactors.
  3. Open the job requisition with the relevant ID.
  4. Initiate onboarding for an internal hire.
  5. Observe that the internal hire does not appear on the onboarding dashboard.

Cause

  • Dummy mapping associated with the timezone field within the Recruit-to-hire mapping in the system configuration.

Resolution

1. Remove the dummy mapping associated with the timezone field within the Recruit-to-hire mapping in the system configuration.

2. Update the field settings in "Manage Business Configuration" to make the timezone field non-mandatory.

3. Ensure that the user has the following permissions:

  • Miscellaneous Permissions → Payment Information → View Current, Create, Insert, and Correct permission.
  • Ensure that the permission "admin access to odata api" under Metadata is not granted to the user.
  • In "Configure Object Definition" → "Object Definition" → "Payment Information", set 'Read-only' visibility to 'mdfSystemExternalUserVisibility'.

4. Navigate to "BPE Monitor Processes" and retrieve the Process Definition ID for the affected candidate.

5. Go to "BPE Manage Process Tasks" and locate the same Process Definition ID retrieved from step 4.

6. Restart the task associated with the Process Definition ID.

7. Re-initiate the onboarding process for the candidate.

Symptom

Despite maintaining the customizing activity of Help page Defining Bank Accounts for Company Code | SAP Help Portal, the bank information is not being displayed in the XML of an invoice.

Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.

Environment

  • SAP ERP
  • SAP S/4HANA

Cause

It is required to have a Payment Method defined in the source document for the related bank details to be retrieved.

Resolution

For this example, a Credit Note Billing Document scenario will be utilized to better illustrate the behavior.

  1. In the Accounting Document Header, it is possible to see that the payment terms utilized during the source document creation is '0003':
  2. To check the configuration of the payment term, access transaction 'OBB8':
    •  
  3. This payment term does not have any payment method defined as default, which means that it will remain empty until manually inserted in the billing document header:
  4. The Payment Methods available can be configured or checked in transaction 'FBZP'. This will have code 'G':
  5. Make sure this value exists in Value Mapping 'PAYMENT_MEANS_CODE', as mentioned in Help page Configuring UNCL4461 Code List (PAYMENT_MEANS_CODE) | SAP Help Portal and that the code matches the Peppol requirements of page Payment means code (UNCL4461):
    •  
  6. Once this is performed, it can be selected in the dropdown table of the Billing Document Header:
  7. Now that the Payment Method is configured, the system will lookup the customizing performed in view EDOEUBANKACCV and display it in the XML:

IMPORTANT TO NOTE:

It is not possible to determine a Default Payment Method from the Customer/Vendor Master Data for scenarios other than automatic payment transactions. More details are explained in KBAs 1727750 - Default Payment Method from Vendor Master data - SAP for Me and 1814039 - Payment method not copied from customer master data to sales order - SAP for Me.

Symptom

  • Tags - P_18A was not getting filled properly for FI invoice
  • The tags of the correction invoices were not getting populated properly.

Other Terms

Poland, eDocument, eDoc, eInvoicing, electronic invoice, electronic invoicing, Poland eInvoice, Document Compliance, Digital Compliance, B2B, KSeF, Electronic Document, DRC (Document and Reporting Compliance), FA(3), FA(3) Schema Version, FI Invoivce, Correction eDocument

Reason and Prerequisites

  • Implement the SAP Note 3717287 Electronic Document Processing for Poland - KSeF: Incorrect Value Mapping Determination for PAYMENT_METHOD_TYPE

  • For a complete prerequisite list, refer to SAP Note 3279206 (Electronic Document Processing for Poland - KSeF: Implementation Overview).

Solution

Date Version Reason
28.02.2026 1 Released for Customer

The FILL_INV_ANNOTATIONS3 method now determines the correct annotation tags - P_18A accounting documents.

Tags of the correction invoices are now being displayed properly.

For more information about this legal change, check the following Help Portal links:

  • SAP S/4HANA: Enabling FA(3) Schema Version
  • What's New SAP S/4HANA: Cloud Service Enablement for Electronic Customer Invoices

Symptom

  • Data Services Management Console is missing the tabs Administrator, Auto Documentation, Data Validation, Impact and Lineage Analysis, Operational Dashboard and Data Quality Reports after the CMS Login.
  • The Data Services Management Console URL is still showing as <hostname>:<port>/DataServices/launch/logon instead of <hostname>:<port>/DataServices/launch/launch.action
  • The Tomcat stderr.log shows the following error:
    WARNING: Failed to scan [file:/D:/Program%20Files%20(x86)/SAP%20BusinessObjects/tomcat/lib/jaxws-rt.jar] from classloader hierarchy

Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental

Environment

  • SAP Data Services 2025 
  • Windows OS
  • Bundled Tomcat

Reproducing the Issue

  1. Go to <hostname>:<port>/DataServices/
  2. Enter credentials for the log on in the CMS System.
  3. Data Services Management Console is showing a blank page

Cause

  • The <Tomcat_InstallDir>\bin\ folder had a library called jaxws-rt.jar, these library was conflicting with the IPS / BI default Tomcat libraries created during the installation.

Resolution

  • Go to <Tomcat_InstallDir>\bin\ and remove the jaxws-rt.jar library.

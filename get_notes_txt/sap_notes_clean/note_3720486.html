Symptom

When you apply a HEX hint by specifying the source table, it only takes effect on the coordinator where a source table resides and is not propagated to a corresponding replica table.

Other Terms

Performance, Source Table, Replica Table, HEX, hint

Reason and Prerequisites

Issue number 353782

Reason:

Due to a programming error in SAP HANA, HEX hints are not replica-aware and therefore match only the exact source table name.

Affected Releases:

  • SAP HANA 2:
    • Revisions <= 089.01 (SPS08)

Prerequisites:

  • The HEX hint references only the source table and does not explicitly mention the replica table.

Solution

Apply one of the following SAP HANA Revisions:

  • SAP HANA 2:
    • Revisions of SPS08: SAP is currently working on a permanent solution to this issue. This SAP Note will be updated as soon as a revision containing the solution is available
    • or higher

  • or higher

    Workaround:

    • Explicitly specify both the source and replica tables in the hint, e.g., <HEX_HINT_NAME>("<SOURCE_TABLE>"."<COLUMN>", "<REPLICA_TABLE>"."<COLUMN>"):

    ALTER SYSTEM ALTER STATEMENT HINT (<HEX_HINT_NAME>("<SOURCE_TABLE>"."<COLUMN>", "<REPLICA_TABLE>"."<COLUMN>")) COMMENT 'SAP Note 3720486. Revert after upgrade to a revision containing the fix' FOR <statement_string> or <statement_hash>;

    or

    <statement_ string> WITH HINT(<HEX_HINT_NAME>("<SOURCE_TABLE>"."<COLUMN>", "<REPLICA_TABLE>"."<COLUMN>"));

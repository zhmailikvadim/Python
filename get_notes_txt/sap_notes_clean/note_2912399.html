Symptom

  • Universe filters used in Web Intelligence Document return duplicates in the results
  • Using Query Filters built directly on Dimensions work as expected
  • Issue is observed with BAPI Universe

Other Terms

webi,web intelligence,BAPI,filters,OR,universe,unv,duplicate

Reason and Prerequisites

Environment:

  • SAP BusinessObjects Business Intelligence Platform 4.2
  • Web Intelligence (Webi)
  • BAPI Universe

Steps To Reproduce:

  1. Create a SAP BW Query with 2  Customer-Exit Variables that are used to extract data of a particular dimension from last day
  2. Create an UNV universe based on the BW Query and create 2 predefined filters for the variables.
  3. Create a Web Intelligence Document based on a the universe and create a Query with the 2 filters from universe using “OR” condition  
  4. The data extracted is duplicated
  5. Create a similar Query in Web Intelligence and this time apply filter on objects using OR condition instead of using universe filters
  6. The data extracted is not duplicated

 

Solution

Currently issue is limitation in BAPI universes. This issue occures when combined query operation on dynamic filters.

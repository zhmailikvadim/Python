Other Terms

Table buffer, buffer synchronization, cstr_cstr_comp, strings

Reason and Prerequisites

This is caused by an error in SAP Kernel 7.45 and 7.49. This error always occurs if changes to a single-record-buffered table that contains string fields (ABAP Dictionary data type SSTRING) in the key need to be synchronized. From the information about the runtime error STRING_BAD_REF in transaction ST22, it is not possible to see which table is involved. However, the following SQL statement can be used to find all potentially affected tables (in the standard SAP system, there are only a very few single-record-buffered tables that contain string fields in the key):

select TABNAME from DD02L
where AS4LOCAL = 'A' and TABCLASS = 'TRANSP'
and exists (select TABNAME from DD09L where TABNAME = DD02L.TABNAME and AS4LOCAL = 'A' and PUFFERUNG = 'P')
and exists (select TABNAME from DD03L where TABNAME = DD02L.TABNAME and AS4LOCAL = 'A' and KEYFLAG = 'X' and DATATYPE = 'SSTR')

Solution

The error is rectified with the patch level specified in this SAP Note.

As a workaround, the affected single-record-buffered tales can be removed from the buffer. The buffering property can be changed accordingly in the technical settings in transaction SE11.

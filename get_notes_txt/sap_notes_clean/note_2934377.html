Symptom

You are using HR as distributed SAP component (ALE - Application Link Enabling) and one of the following symptoms is observed:

Symptom 1

You run the International Payment Medium - Check (with check management) (PC00_M10_FFOC) transaction / (RFFOUS_C) report or the Payment Medium USA - Transfers/Bank Direct Debits in ACH Format (RFFOUS_T) report, and the 3PR acknowledgment on HR side is not performed.

Symptom 2

You run the Reconciliation of Remittance Posting Items (PC00_M99_URMR) transaction / (RPURMR00) report with one the following checkboxes selected: Check already posted items, or Correct already posted items, and the following short dump is raised:

  • Category: ABAP Programming Error
  • Runtime Errors: RAISE_EXCEPTION
  • ABAP Program: SAPLHRCA_THIRD_PART

Symptom 3

You run Reconciliation of Remittance Posting Items (PC00_M99_URMR) transaction / (RPURMR00) report with one or both of the following checkboxes selected: Correct stopped status, or Correct already posted items. However, the items are not updated as expected.

Other Terms

F110, PC00_M99_URMR, RPURMR00, PC00_M10_FFOC, RFFOUS_C, RFFOUS_T, CL_HRPAYNA_EXTERNAL_SERVICES, CL_HRPAYNA_3PR_FI_ITEMS_FIND, FI_DOC_TPR_ITEMS_FIND_RFC, RP_REMITTANCE_ACKNOWLEDGEMENT, HRCA_GET_FI_DOC_ITEMS, HRCA_UPDATE_FI_DOC_ITEMS_DUEDT, HR3PRNA 215, HR3PRNA 216, HR3PRNA 217, HR3PRNA 218, BUS6006

Reason and Prerequisites

Reason for Symptom 1

To inform HR that the payments were done, FI payment reports call HR function module RP_REMITTANCE_ACKNOWLEDGEMENT, via RFC, to perform the 3PR payment acknowledge.

The HR function module RP_REMITTANCE_ACKNOWLEDGEMENT also calls back FI, to request documentation information. This request uses the RFC provided for method AcctngEmplyeePaybles.post (BUS6006).

As of SAP Note 2748294, when no RFC destination is maintained for BUS6006, the RP_REMITTANCE_ACKNOWLEDGEMENT started producing a short dump in the method GET_UNIQUE_RECEIVER of class CL_HRPAYNA_3PR_RFC_VALIDATIONS.

Reason for Symptom 2

The problem is caused by an incorrect handling of the exceptions raised by the function modules HRCA_GET_FI_DOC_ITEMS and HRCA_UPDATE_FI_DOC_ITEMS_DUEDT, when one of the following exceptions happens:

  • The HR payee has no due date.
  • No company code was given.
  • More than one receiving system has been specified for a BAPI.
  • No RFC destination could be determined for a method call in a logical system.
  • You are not authorized to display accounting docs in the remote system.

Reason for Symptom 3

The problem is caused by the incorrect handling of exceptions in the method FI_DOC_TPR_ITEMS_FIND_RFC of class CL_HRPAYNA_3PR_FI_ITEMS_FIND.

Solution

In order to allow 3PR acknowledge remittance process to reach FI, it is required to have method AcctngEmplyeePaybles.post (BUS6006) configured in Display ALE Customizing (transaction SALE). But to properly handle scenarios where the configuration is missing, the source code was corrected and new messages were delivered to enhance the error description in case of exceptions.

  2. Use Note Assistant (SNOTE) to implement the correction instructions.

Symptom

We would like to filter TIH attributes in Story reporting by Attribute Type, but the filter only works in one locale.

Environment

SAP SuccessFactors Performance Management

SAP SuccessFactors Talent Intelligence Hub

SAP SuccessFactors People Analytics 

Reproducing the Issue

  1. Create a Story report with TIH attributes data queried.
  2. Apply a Filter to the Story data source.
  3. As a filter use the Attribute type name in English locale, for example "Skill".
  4. Switch to another locale and run the same report or click "Preview query".
  5. Notice that the report doesn't return any data, however with the filter removed, the report works as expected.

Cause

With text filter by an Attribute type name it's necessary to add all the attribute type translations to the filer for it to be working in different locales. 

Resolution

To add all Attribute type translations to the filer:

  1. Change the system locale to another.
  2. Edit the Story and go to Edit Data source.
  3. Change the filter by adding the relevant Attribute type translation to the filter. 
  4. Save the Story in every locale used.

Workaround: if you don't want to use Attribute type name filer in your Story, please consider using Attribute type External Code instead, for example "SKILL" for skills, in this case the story will automatically filter and pull data for all locales.

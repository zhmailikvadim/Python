Symptom

  • In the Import Collection app, filtering by the "Imported At" column for a specific month does not return the correct transport items.
  • The UI shows values such as "Versions (3)" or "Versions (0)" instead of the expected transport results for the selected period.
  • Exporting after applying the "Imported At" filter generates an export of all transports in the system rather than only those within the selected period.

Environment

SAP S/4HANA Cloud Public Edition

Reproducing the Issue

  1. Open the Import Collection app (Manage and Import Software Collections).
  2. Set the "Imported At" filter to a specific month date range.
  3. Observe that the list shows "Versions (0)" or an unexpected versions count instead of the filtered transport results.
  4. Export the list to a spreadsheet and observe that all transports are exported, ignoring the applied "Imported At" filter.

Cause

Resolution

The bug appears when only applying one field. Searching a combination of fields should still work. 

Apply only one field e.g. "Import At" or "Last Changed On", a workaround could be that you also apply another field such as "foward status" and select ALL forward status. Then the result should be correct. 

It's a known issue and we will fix it with release 2608.

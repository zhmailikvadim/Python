Symptom

Updating the BP takes significantly longer than expected, resulting in noticeable delays during save or change operations. Sometimes the error message “E:/TXINTF/ERROR_CLASS:130” is raised when updating BP from US or CA.

Other Terms

 SETI, TXINTF

Reason and Prerequisites

For US and Canada BP, address validation is triggered for persons and contact persons even though the configuration in table /TXINTF/ADDR_ARE is maintained. The validation area is not being set properly, causing the system to apply address validation unexpectedly.

Solution

Additional validations were implemented in CHECK_ACTIVE method of class /TXINTF/CL_ADDRESS_VALIDATION to check the Program Name, Event Type, and Event Name in the ABAP Call Stack for event related to Contact Person data. When these conditions are met, the validation area is correctly set to the value ‘C’ (Contact Person address of the customer). Retrieval for BP type and BP account group are added for tagging the appropriate area to be cross checked on the customizing table (/TXINTF/ADDR_ARE) to switch on/off the address validation. New BP type related areas are added as below.

P - BP Person
O - BP Organization
G - Group

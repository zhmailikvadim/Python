Symptom

  1. Managed Gateway for Ariba apps are migrating from Neo to CF tenants in the US & EUR data centers SAP Integration Suite, Managed Gateway for Spend Management and SAP Business Network - SAP BTP NEO to CF Migration .
  2. Following changes are required to be made in the SAP Cloud Connector according to "What do Customers have to do?"  in the SAP Cloud Connector.

Environment

  • SAP Cloud Connector release independent.
  • Managed Gateway for Ariba.

Cause

Managed Gateway for Ariba apps moving from Neo to CF tenants for the US & EUR data centers.

Resolution

   1.Refer to: SAP Integration Suite, Managed Gateway for Spend Management and SAP Business Network - SAP BTP NEO to CF Migration  for the changes needed to be performed with respect to the SAP Cloud Connector.

  • Buyers
    • Register for new SAP Cloud Connector Subaccounts for the required sites/location ASAP.
      • Please see this How to Setup Cloud Connector demo!
    • If IP Allow Listing is needed, ensure new BTP Cloud Foundry addresses prior to the migration date are Allow Listed. The necessary IP details are published in the Regions and API Endpoints for SAP Cloud Foundry Environment resource.
  • Suppliers
    • If IP Allow Listing is needed, ensure new BTP Cloud Foundry addresses prior to the migration date are Allow Listed. The necessary IP details are published in the Regions and API Endpoints for SAP Cloud Foundry Environment resource.

IMPORTANT: All necessary changes must be completed in advance before the scheduled migration date. It is not required to wait until the migration date to accommodate these changes. Make these changes as soon as possible to avoid unnecessary disruption.

It is recommended to adopt new domain URLs.

   2. For further queries and issues pertaining to the above points mentioned, create a case to BNS-ARI-CI-FND-PTL component for further support.

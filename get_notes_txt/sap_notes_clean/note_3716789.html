Symptom

As of November 2026, the current versions of payment files for SEPA Credit Transfer and SEPA Direct Debit, as well as the payment file format DTAZV for foreign payments, will end in Germany and will no longer be accepted by banks. From November 2026 German banks will only accept the following format versions:

  • pain.001.001.09 for SEPA Credit Transfer initiation
  • pain.008.001.08 for SEPA Direct Debit initiation

All details regarding the validity of SEPA format versions in Germany can be found on the official website: Format LifeCycle - EBICS.

Cause

Legal change

Solution

SAP intends to provide a feature package or patch to solve the symptom that is described in this SAP Note. The section References below lists the related feature package or patch once they become available. Be aware that References can only be confirmed at feature package or patch release date. SAP delivers feature packages or patches only for selected releases at its own discretion, based on the business impact and the complexity of implementation.

SAP plans to provide the updated format versions pain.001.001.09 for SEPA Credit Transfer initiation and pain.008.001.08 for SEPA Direct Debit initiation, and to remove the outdated payment file format DTAZV.

Planned delivery of legal change solution: August 31, 2026.

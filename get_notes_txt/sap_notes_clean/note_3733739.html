Symptom

  • In the Fiori app "Manage Customer Materials", creating a new Customer Material Information Record (CMIR) using the copy option does not copy all details from the source record.
  • After entering a target material and customer material and choosing create, only some data appears in the new record, remaining details are not copied.

Environment

  • Sales and Distribution (SD)
  • SAP S/4HANA
  • SAP S/4HANA Cloud Private Edition
  • Fiori app: Manage Customer Materials

Reproducing the Issue

  1. Open the Fiori app "Manage Customer Materials".
  2. Select an existing customer material info record and click on Copy.
  3. Enter a different target material and customer material and click on Create.
  4. Observe that not all details from the source record are copied to the new record.

Cause

The Copy function only copies the key fields of the Customer-Material Info Record, such as Customer, Sales Organization, and Distribution Channel, allowing the creation of a new record for a different material. Material dependent information, such as classification data, is not copied.

Resolution

  1. Use the Copy option to prefill the key fields (customer, sales organization, distribution channel) for the new record.
  2. After creation, manually maintain material-dependent and other non-key data (for example, classification characteristics) in the new record, as these are not copied by design.
  3. Refer to: Help - Customer-Material Info Record for details on which data is copied and the intended behavior.

Symptom

  • Modifying the layout of the print cash journal program RFCASH20, does not take effect.
  • When adding a field to the layout, the added field does not appear in the output.
  • Fields displayed become disorganized, and some fields that were previously visible no longer appear.

Environment

  • Cash Journal
  • SAP S/4HANA
  • SAP S/4HANA Finance
  • SAP S/4HANA Cloud Private Edition

Reproducing the Issue

  1. Transaction code FBCJ.
  2. Select an specific display period.
  3. Enter the Company Code and Cash Journal.
  4. Click the "Print Cash" button.
  5. Change the Column Set and select the desired field.
  6. Click the "Copy" button: "Layout was applied" message is issued.
  7. The added field(s) does not appear in the output. Additionally, the fields displayed become disorganized, and some fields that were previously visible no longer appear.

Cause

System bug.

Resolution

  1. Apply the following SAP Notes:
  • 3537467 - ALV Grid - After changing layout data of a column are not updated.
  • 3576667 - ALV: Problems with filter rules in unified settings dialog.

    2. If the issue persists after applying the above notes, apply the following additional SAP Notes:

  • 3497952 - ALV Layout - Short dump when choosing additional fields in layout selection.
  • 3580468 - ALV: Wrong column processed in list display settings.
  • 3546552 - ALV Layout: Standard Layouts are always displayed in Layout Administration.

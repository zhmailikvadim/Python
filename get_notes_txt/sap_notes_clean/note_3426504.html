Environment

SAP HANA 2.0 with system replication

Reproducing the Issue

  1. Removing the Tenant DB global.ini [system_replication] section parameters
  2. Upgrade secondary site with hdblcm
  3. Secondary site fails to start

Cause

Tenant DB global.ini [system_replication] section parameters shouldn't be removed for the upgrade. 

Resolution

SAP HANA system replication secondary site should be upgraded following SAP document Update SAP HANA Systems Running in a System Replication Setup

Keywords

Secondary site upgrade, Fetching root keys failed, service is aborting, hdblcm, system_replication

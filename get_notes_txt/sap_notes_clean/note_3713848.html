Symptom

  • Serial number (SN) status in the SNR_LIST table is marked as "Reported Lost".
  • Status in the SNR_CRY table is updated as "5-Disposed".
  • Serial numbers from the old range definition are requested by the line system and need to be processed in the system.
  • How to change the Crypto table status in SAP ATTP (SNR_CRY) table.

Environment

SAP Advanced Track and Trace for Pharmaceuticals

Reproducing the Issue

  1. Perform the "Replace Range Definition" action for a KZ GTIN.
  2. Observe that the SN status is marked as "Reported Lost" in the SNR_LIST table.
  3. Check the SNR_CRY table and note that the status is updated to "5-Disposed".
  4. Attempt to process serial numbers from the old range definition in the system.

Cause

Replace Range Definition permanently invalidates the old serial number range, marking those serials as Reported Lost / Disposed which by design cannot be reused or reactivated.

Resolution

  1. Use the report `/STTP/SNR_CRY_CORRECT` with the option "Check and Correct Item Status" to correct the statuses in the `/STTP/SNR_CRY` table. Note: the report is designed for handling exceptional cases and does not assume active use in the production.
  2. Test the report in a pre-production environment before implementing it in the production environment.
  3. Refer to the document "Crypto Code Handling Report" for additional guidance on report usage -https://help.sap.com/docs/SAP_ADVANCED_TRACK_AND_TRACE_FOR_PHARMACEUTICALS/c9af81154f504f9aa2795b427855a106/9bbb940a3f5e485aadefd651b0398872.html?version=3.2&locale=en-US.

Symptom

When attempting to edit an existing dynamic user list, the system returns the following error message: "Following your updates of the rule condition, the user list will be identical to existing user lists. Please modify the condition or use the existing user list instead"

Environment

  • SAP SuccessFactors Work Zone
  • SAP Build Work Zone, advanced edition
  • SAP SuccessFactors HCM Suite

Reproducing the Issue

  1. Open Admin Console.
  2. Navigate to User Lists > Dynamic User Lists.
  3. Select an existing dynamic user list.
  4. Click Edit.
  5. Go to Organization.
  6. Add an organization to the rule.
  7. Click Done.
  8. Click Publish.
  9. Get the error message: "Following your updates of the rule condition, the user list will be identical to existing user lists - Please modify the condition or use the existing user list instead."

Cause

The beta feature "add_unique_membership_conditions__limitation_dwp_2505b" introduces a duplication check for membership conditions that can falsely detect duplicates, blocking save/publish of dynamic user list changes.

Resolution

This is a known issue confirmed by Engineering.

A fix is planned to be deployed on June 11th, 2026.

Until the fix is implemented this is the workaround:

  1. Turn off the beta feature named "add_unique_membership_conditions__limitation_dwp_2505b".
  2. This will bypass the membership condition duplication check, and allow the customer to save the new condition.

Symptom

The system does not save data when attempting to remove the tick mark for "Activate Inventory-Managed MPN" in SPRO

Environment

  • SAP ERP Logistics (LO)
  • SAP R/3
  • SAP R/3 Enterprise
  • SAP ERP Central Component
  • SAP ERP
  • SAP Enhancement package for SAP ERP
  • SAP Enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA, on-premise
  • SAP S/4HANA Cloud Private Edition

Reproducing the Issue

1. SPRO: Purchasing -> Material Master -> Inventory-Managed Manufacturer Part Number -> Specify Settings for Inventory-Managed MPN.
2. Attempt to remove the tick mark for "Activate Inventory-Managed MPN" 
3. Save the changes.
4. Observe that the system does not save the data or remove the tick mark.

Cause

The system doesn't allow to deactivate Inventory-Managed MPN once it is activated. Once this setting is activated material master data will be created based on this. So generally system doesn't allow to deactivate it as it can cause DB inconsistency and lead to further issues. 

Resolution

If activated by mistake and want to revert it. Please consider option of setting MPNCNV-CONEXT field as blank through debugging in SE16N. Check If the material master data is created using MPN and handle accordingly.

Keywords

MPNCNV-CONEXT, MPN, Inventory-Managed MPN

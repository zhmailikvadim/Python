Symptom

You change data of a contract element that doesn't support release process and a work item gets created.
For example, all changes to an account are relevant for release and only Correspondence recipient data (BCRM ) is changed and a work item gets created. This should not happen because BCRM data is never relevant for release.

Other Terms

Alias Names, BCAL,

Reason and Prerequisites

 Program error

Solution

Import the relevant Support Package for your release. You can use transaction SNOTE to implement the correction instructions relevant for your release in advance.

There are no manual activities.

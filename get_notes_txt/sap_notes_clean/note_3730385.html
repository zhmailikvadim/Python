Symptom

As described in SAP ONE Support Launchpad Notification of Legal Change Unified monthly employer report 2026 (JMHZ)

The functionality delivered in this SAP Note is currently under development. It is intended exclusively for customers registered as JMHZ pilot testers, or for the preview of features that may become available for productive use in the future.

These updates are provided:

  • REGZEC (DM - data mining, CS - compose submission, PF - process feedback, SO - special operations, INIT - import of IDs to IT3457, XP - XML for ePortal)

    • DM: The table V_T7CZPPD has been extended with the field "Continuous Operation". The logic for retrieving REGZEC data is now as follows:
      Continuous Operation (10407 cont) – AUS04 from IT0033: If no value is filled in IT0033, the value is taken from table V_T7CZPPD. If it is not set in this table either, the default value is set to 'N'.

    • XP: New report HCZCJHR0_XP (transaction PC00_M18_JHR_XP) for creation of XML file for ePortal. This report will create XML file REGZEC with action A03 for employments which were active in Q1/2026 and which and which are not active from 1.4.2026. Output XML file should be used for upload on ePortal

    • Pilot and preview locks are removed

  • MH (DM - data mining, CS - compose submission, WB - workbench)
    Corrections that may cause that corrective individual forms will be created for already submitted periods are marked with code abbreviation (DIF )

    • In section DME, attribute 10417 is filled with summary of attributes 10418, 10292, 10293, 10294, 10295, 10296 (this stays without change). But newly, these attributes 10418, 10292, 10293, 10294, 10295, 10296 should not be used to report details of those payments and therefore are set to zero. (DIF)

    • After applying SAP Note 3727732, warnings appeared in the log: "In table T5T54, the municipality ... is assigned to multiple regions." The warning has been removed.

    • In the ALV displaying individual forms, in the PRZ section, values in the attributes for work obstacles - elements 10471 and 10472 were not displayed.

    • The documentation for table T5T04_03 incorrectly stated that the “best match” algorithm is implemented for the Absence Valuation Rule column (KLBEW). It meant that the program first tries to find a record with a matching KLBEW, and if not found, only then searches for a record without a value in the KLBEW field. The HCZCJHM0_DM program did not and does not work this way. The program counts with all records that have:

      • The same grouping of sub-personnel areas for types of absence/presence (MOABW)
      • The same type of absence/presence (SUBTY)
      • The same or empty absence valuation rule (KLBEW)
      • Grouping of employees for absence evaluation (MODIF) - see next

    • When configuring table T5T04_03, the following parameters can be used:

      • Existing: JMH_POJ_ELDP/T5T04_03_MODIF_USED.
        Value X: When reading table T5T04_03, only records with the same absence modifier value (MODIF-MOD0A) will be selected. Other value or parameter not specified: the absence modifier value will be ignored.
      • New: JMH_POJ_ELDP/T5T04_03_MODIF_BEST_MATCH
        Optional parameter that can be used if T5T04_03_MODIF_USED is set to X. If, when reading table T5T04_03 using the absence modifier (MODIF-MOD0A), no suitable record is found, the table will be read again without using the absence modifier.

    • When the HCZCJHM0_DM program finds a subtype of infotype 6 for the place of work, it tries to determine the place of work from it. If the program does not find the municipality name using the reference to T5T54, it compiles it from fields ORT-01 and ORT-02. The program - even in the case of successful compilation of the municipality name - reported an error. Now replaced with a warning only if the municipality name cannot be compiled.

    • A new rule has been added that if attribute 10273 (number of hours worked in hazardous work) is higher than attribute 10268 (number of hours worked), attribute 10273 is reduced to the value of attribute 10268.

    • For attributes 10328, 10329, 10330, 10337, 10338, 10339, 10340, 10341, mathematical rounding of values to whole crowns has been added (DIF)

    • Logical checks 30, 281, 317, and 318 have been deactivated. Logical check 338 has been implemented.

    • In section DME, attribute 10417 should be filled with summary of attributes 10418, 10292, 10293, 10294, 10295, 10296 (this stays without change). But newly, these attributes 10418, 10292, 10293, 10294, 10295, 10296 should not be used to report details of those payments and therefore are set to zero.

    • Pilot and preview locks are removed

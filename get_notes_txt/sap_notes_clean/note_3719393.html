Symptom

In the SAP Web Dispatcher (WDP) administrator page, the back end system instance(s) status is GREEN.

Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.  

However when perform the WDP checkconfig via command line "sapwebdisp -checkconfig pf=<WDP_Profile>", the error code 403 is returned.

Environment

SAP Web Dispatcher

Cause

In the back end system ICM, the modification rule parameter icm/HTTP/mod_X is configured/activated, and in the modification rule file, we have configured some rules, e.g.
IF "%{HTTP_HOST}" !regimatch "<hostname>"
    RegForbiddenUrl /(.*) -

the above modification rule means if the request coming to ICM does not contain the 'host' header, the ICM returns error code 403 error code to the front end.

When performing a WDP checkconfig, the WDP sends a request from WDP to each back end system instance ICM, however the header 'host' is not included/contained in the request, so the ICM returns error code 403 to WDP, and we get the error code in the output of WDP checkconfig.

Resolution

In this case, the checkconfig just sends a ping request to the back end system, though the error code code 403 is retuned, this error could be ignored as it is not a real error, all the requests to back end system through WDP could be processed successfully with the configured modification rules.

Or we could comment out the corresponding modification rule parameter icm/HTTP/mod_X of back end system (or comment out the corresponding rule setting), then restart the back end system to activate the change.

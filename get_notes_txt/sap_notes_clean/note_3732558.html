Symptom

I enable SSO integration with IAS in the Authentication tab using ICM.
I want to confirm if all users must sign in using SSO or if some users can still use username and password.

Environment

SAP Ariba Procurement

Resolution

When you enable SSO with IAS, the authentication behavior depends on the user type. If you are an enterprise user, you sign in using single sign-on, and username and password authentication is no longer available once SSO is enabled.

However, if you are a third-party user, you sign in using username and password, as SSO authentication does not apply to this user type.

If you want to allow some users to continue using basic authentication while SSO is enabled, you must create them as third-party users.

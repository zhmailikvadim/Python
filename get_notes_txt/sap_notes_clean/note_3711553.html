Symptom

  • WBS Element description in T-code:CNS41/CN41N differs from T-code:CJ20N when creating a WBS by copying another WBS element.
  • WBS element description differs between table PRPS and CNLDST_SHORTTX.

Environment

  • SAP ERP Project System (PS)
  • SAP R/3
  • SAP R/3 Enterprise
  • SAP ERP Central Component
  • SAP ERP
  • SAP Enhancement package for SAP ERP
  • SAP Enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA, on-premise
  • SAP S/4HANA cloud private edition

Reproducing the Issue

  1. Open T-code:CJ20N and include a WBS Element (Right Click -> Include -> WBS Element).
  2. Assign a new WBS Element number and new description, then check and include.
  3. Save the changes.
  4. Check PS report in T-code:CN41N, observe that the description differs from what is seen in the Project Builder.

Cause

Multi-language functionality for standard WBS elements (object type PT) is not activated.
When the multi-language functionality is not activated, the system logic results in copying the text from the template WBS element again, which consequently overwrites the previously determined text. 

Resolution

Please activate multi-language functionality for standard WBS elements (Object type PT) in transaction code CNLDSTINIT.

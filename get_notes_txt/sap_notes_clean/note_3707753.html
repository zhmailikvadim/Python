Symptom

The credential type instance-secret has reached end-of-life.

The following scenarios are affected by this change:

  •  There is an explicitly configured credential type instance-secret in the application security descriptor.

  • The service instance was created before 12 November 2024 without the explicit configuration of credential types in the application security descriptor.

Environment

  • SAP Business Technology Platform, Cloud Foundry Environment

Reproducing the Issue

The following errors can be encountered when facing issues regarding this change:

  • when trying to create a binding with a credential type that is not contained in the xs-security.json the following error is visible : "Unsupported credential type" 

  • when trying to create a binding while explicitly specifying the instance secret in the request body the following error is visible: "Instance secret is no longer allowed to be used." 

  • when trying to update a service instance with the instance secret, but it was not specified in the xs-security.json before, the following error is visible: "Inconsistent xs-security.json: unsupported credential-type instance-secret" 

Cause

This change has the following impact:

  • Creating service instances with the credential type instance-secret in the security descriptor fails.

  • Updating service instances with the credential type instance-secret in the security descriptor succeeds if instance-secret was explicitly configured before and fails otherwise.

  • Creating bindings or service keys with the credential type instance-secret fail.

  • Existing instance secrets are removed or rejected if they haven't been used for more than 3 months.

  • For service instances created before 12 November 2024 without the explicit configuration of credential types in the security descriptor, the service instance update changes the default binding type from instance-secret to binding-secret.

  • New bindings or service keys for other BTP services can't contain instance secrets.

Resolution

Instead of using the instance-secret credential type, applications and services use the binding-secret credential type (see Migrate from Instance Secrets to Binding Secrets) or x509 (see Retrieving Access Tokens with Mutual Transport Layer Security (mTLS)).

To test an application with these credential types, update the security descriptor to set the credential type to ["binding-secret", "x509"].

For more information on the security descriptor, see Application Security Descriptor Configuration Syntax.

In case the affected application is not a custom one, but rather an SAP managed application, please open a SAP Support Case in the application/service owner component.

Symptom

  • Create a remote source with any adapter.
  • Testing the remote source connection fails.
    • Error message thrown in framework.trc:
      [ERROR] [] DPFramework | WorkerThread.processRequest [] - com.sap.hana.dp.adapter.sdk.AdapterException: Adapter type can not be null Context: Adapter type can not be null
  • Refreshing the adapter fails:
    • Error message thrown to the SQ console:
      Could not execute 'ALTER ADAPTER "<adapter_name>" REFRESH AT LOCATION AGENT "<agent_name>"
      Error: (dberror) [403]: internal error: Cannot get adapter capabilities: Adapter type can not be null
  • The adapter is listed for the agent in adapter_locations.
    • Can be verified with this command:
      select * from adapter_locations where adapter_name = '<adapter_name>';
  • The adapter is indicated as registered via the Configuration Tool.

Environment

SAP HANA Smart Data Integration

Cause

The exact cause is unknown. The error is more likely to happen when the target HANA system is migrated (for example, from an on premise landscape to the cloud). The target HANA recognizes the adapter as being registered for a particular agent location, but cannot use it when establishing remote source connections.

Resolution

  1. Delete the remote sources created using the affected adapters or move them to another agent location.
  2. Unregister the adapters from the agent using either the Configuration Tool or SQL commands.
    • SQL command:
      alter adapter "<adapter_name>" remove location agent "<agent_name>";
    • Note: it is preferable to use the Configuration Tool CLI for this action.
  3. Register the adapters again on the agent using either the Configuration Tool or SQL commands:
    • SQL command:
      alter adapter "<adapter_name>" add location agent "<agent_name>";
    • Note: it is preferable to use the Configuration Tool CLI for this action.
  4. Recreate the remote sources and test the connections.

Symptom

  • The error "Can't bill SD documents of type * using billing type *" occurs during the simulation step when attempting to create a billing document request directly using the "Manage Billing Document Requests" app.

Environment

SAP S/4HANA Cloud Public Edition

Reproducing the Issue

  1. Access the “Manage Billing Document Request” app.
  2. Click the "Create" and "Import from Microsoft Excel File" buttons.
  3. Upload data using Excel import in the app.
  4. Validate Item, Pricing Conditions, Partner data and proceed to the Simulation step.
  5. Observe errors during simulation preventing the document creation.

Cause

  • The mandatory fields "PRECEDINGDOCUMENTTYPE", "PRECEDINGDOCUMENTITEMCATEGORY" and "EXTBILLINGDOCREQUESTTYPE" are not correctly filled in the uploaded excel file.

Resolution

  1. To Check and correct the excel file referring the documentation below and ensure the mandatory fields "PRECEDINGDOCUMENTTYPE", "PRECEDINGDOCUMENTITEMCATEGORY" and "EXTBILLINGDOCREQUESTTYPE" are correctly filled in the excel file:
    • Creating External Billing Document Requests via Spreadsheet Import | SAP Help Portal
  2. After that, to upload the correct excel file and test again.

See Also

  • Creating External Billing Document Requests via Spreadsheet Import | SAP Help Portal

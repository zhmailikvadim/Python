Reason and Prerequisites

Issue number 331064

Reason:

As described in SAP Note 3649396, queries that contain a very large number of WHEN ... THEN clauses may not be executable because their execution is exceeding the configured stack size.
Due to a programming error in SAP HANA, the execution of such queries can lead to an indexserver crash instead of a graceful failure.

Affected Releases:

  • SAP HANA 2:
    • Revisions of SPS00 - SPS06
    • Revisions <= 079.06 (SPS07)

Prerequisites:

  • A query containing an excessive number of WHEN ... THEN clauses in its filter condition is executed, exceeding the configured stack size.
    • 'An excessive number' is relative; it depends on the required and configured stack size rather than solely on the number of WHEN … THEN clauses in the query.

Solution

The issue occurs due to the complexity of the SQL query. The query requires optimization, regardless of its source. If the query is generated by an application, you may need to consult the application support team to address the query complexity.

As of the following revisions, instead of causing an indexserver crash, the query execution will fail with the error "Check stack size for <number> bytes failed, only <number> bytes left":

  • SAP HANA 2:
    • Revisions >= 079.07 (SPS07)

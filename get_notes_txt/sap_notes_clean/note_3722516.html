Symptom

In ARM customizing, General Settings > Automatic Creation of Follow-Up Documents > Mapping Document Types and Return Reasons > Specify Follow-Up Document Types for Store Returns, you set up the document types (Intra-Company and Cross-Company) for creating a Stock Transport Order (STO) for Send Back to Last Plant. You configure these document types in the way that they are active for ARM.

You create a returns Stock Transport Order from one plant to another one. In the warehouse inspection, you decide to send the material back to the last plant (follow-up code 0022).
In the tracking overview, you see the corresponding Stock Transfer Order twice:
1) as Returns Stock Transport Order                                   -> this is wrong
2) as Stock Transport Order for Send Back to Previous Plant -> this is correct

Other Terms

duplicate entry, sebalap, rSTO, STO, ccrSTO, ccSTO, Send Back to Previous Plant

Reason and Prerequisites

You have configured the document types for Send Back to Last Plant in the way that they are active for ARM.

Solution

Your customizing is wrong.
The document types for Send Back to Last Plant both for Intra-Company and for Cross-Company must not be active for ARM.
The purchasing document for Send Back to Last Plant is not a returns STO but an STO.
Consequently, the document types for Send Back to Last Plant cannot be the same as for Intracompany Store Return and Cross-Company-Code Store Return.

Symptom

You want to restrict the user authorization to create a purchase requisition to only a specific storage location.

To do that you set the object M_BANF_LGO via PFCG but it is not working.

Environment

  • Materials Management (MM)
  • SAP ERP Central Component
  • SAP ERP
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA

Reproducing the Issue

As an example, a user works on storage location 123. So he can create purchase requisition only for storage location 123.

Cause

Although the authorization object M_BANF_LGO (storage location in purchase requisition) exists, but this is only checked in the IS-DFS (Industry-Spec. Comp. Defense Forces and Public Security). See Note 770961 about this.

The storage location authorization objects are not checked in the standard enjoy purchase requisition transactions. Check Note 751129 for the list of objects checked.

Resolution

In order to check the storage location, you need to use the Badi ME_PROCESS_REQ_CUST for a custom check.

Keywords

Symptom

The processing of the message type /PACG/ECM_ATTCHMIN ends with the short dump with the following runtime error ITAB_ILLEGAL_SORT_ORDER. The first processing of the idoc successfully stores the attachment in the object, e.g. an activity or a service order, however the status of the idoc is not changed and remains in 64. The next processing changes the status of that idoc to 51 and displays the error message "Attachment ARCH/PACG/ECM... already created".

Other Terms

FSM, E4C, Connector, Attachment, /PACG/ECM_ATTCHMIN, ITAB_ILLEGAL_SORT_ORDER, /PACG/ECM_CL_X_UTIL, /PACG/ECM_ASCI

Reason and Prerequisites

This is caused by incorrect handling of sorted table in method 'IDOC_FOR_AUFNR' (class /PACG/ECM_CL_X_UTIL).

Solution

Please, implement this correction instruction and change the status of the affected idocs to 68. This can be done from transaction /PACG/ECM_COCKPIT2 by using the option "Set to processed".

Symptom

When user create a Web Site or Web Application project run in IIS Express, if there's only a CrystalreportsViewer added to the ASPX web from, and no Crystal Reports file added to the project, this ASPX web page could not run successfully in IIS Express

Other Terms

Symboliclink, aspnet_client, CrystalReportsViewer, ASP.Net, VS2013, VS2012, IIS Express

Reason and Prerequisites

Reason:

Symboliclink "aspnet_client"  is not created

Environment:

SAP Crystal Reports, version for Microsoft Visual Studio

Visual Studio 2012, Visual Studio 2013, use IIS Express for new created Web Site project

Steps to Reproduce:

1. Create a Web Site or Web Application project

2. Add a new Web Form to project

3. Drag & Drop a CrystalReportsViewer from toolbox

4. Binding a Crystal Report file as a ReportSouce to this CrystalReportsViewer via Code

5. Run this ASPX web page

Solution

Install the relevant Support Package. See the Support Packages & Patches section for details and available patches.

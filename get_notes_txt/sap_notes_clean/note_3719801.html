Symptom

This article shows how to create a hierarchy dimension in Embedded Analytics.

"Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental." 

Environment

  • SAP Sales Performance Management
  • Incentive Management on Hyperscalers
  • Incentive Management on Legacy HANA 

Resolution

  1. Create the table:
    1. CREATE TABLE EXT.ORDENS (
          USER VARCHAR(50),
          CITY VARCHAR(100),
          COUNTY VARCHAR(100),
          COUNTRY VARCHAR(100),
          NUMBER_VALUE DECIMAL(10,2)
      );
    2. Insert Values on the table:
      1. INSERT INTO EXT.ORDENS (USER, CITY, COUNTY, COUNTRY, NUMBER_VALUE) VALUES
        ('mock_name ', 'mock_city', 'mock_state', 'mock_country', mock_value)
    3. Create the view:
      1. CREATE VIEW EXT.VW_ORDENS AS
        SELECT USER, CITY, COUNTY, COUNTRY, NUMBER_VALUE
        FROM ext.ORDENS;
    4. Create the csn file with defining the hierarquy:
      1. "EXT.VW_ORDENS": {
            "kind": "entity",
            "@Common.Label": "View Ordens",
            "@Analytics.query": true,
            "@Analytics.dataCategory": { "#": "CUBE" },
            "elements": {
              "USER": {
                "@Common.Label": "User",
                "type": "cds.String",
                "@Analytics.Dimension": true
              },
              "CITY": {
                "@Common.Label": "City",
                "type": "cds.String",
                "@Analytics.Dimension": true
              },
              "COUNTY": {
                "@Common.Label": "County",
                "type": "cds.String",
                "@Analytics.Dimension": true
              },
              "COUNTRY": {
                "@Common.Label": "Country",
                "type": "cds.String",
                "@Analytics.Dimension": true
              },
              "NUMBER_VALUE": {
                "@Common.Label": "Number Value",
                "type": "cds.Decimal",
                "scale": 2,
                "@Analytics.Measure": true,
                "@Aggregation.default": { "#": "SUM" }
              }
            },
            "@Hierarchy.leveled": [
              {
                "name": "GeoHierarchy",
                "label": "Geographic Hierarchy",
                "levels": [
                  {
                    "element": { "=": "COUNTRY" },
                    "order": {
                      "by": { "=": "COUNTRY" },
                      "direction": { "#": "ASC" }
                    }
                  },
                  {
                    "element": { "=": "COUNTY" },
                    "order": {
                      "by": { "=": "COUNTY" },
                      "direction": { "#": "ASC" }
                    }
                  },
                  {
                    "element": { "=": "CITY" },
                    "order": {
                      "by": { "=": "CITY" },
                      "direction": { "#": "ASC" }
                    }
                  }
                ]
              }
            ]
          }
    5. Upload the model on EA admin.
    6. Create the story.
    7. Select the newly created custom model.
    8. Add a chart or a table.
    9. Add a measure.
    10. Add the hierarchy:

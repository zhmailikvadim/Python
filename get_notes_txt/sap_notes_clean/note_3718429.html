Symptom

  • Users are automatically logged out from BI Launchpad or Central Management Console (CMC) after a short period of inactivity.
  • Session expires and displays “Your session has expired” or redirects users back to the login page unexpectedly.
  • Users report that they are unable to work continuously because the session times out too quickly.

Environment

  • SAP BusinessObjects Business Intelligence Platform 4.x 
  • SAP BusinessObjects Business Intelligence Platform 2025
  • Apache Tomcat
  • All supported OS.

Reproducing the Issue

  1. Log in to BI Launchpad (BILP) or Central Management Console (CMC) using any valid user account.
  2. Navigate to any page (e.g., Folders, Servers, Users, Applications).
  3. Remain inactive for the duration of the current session timeout limit.
  4. After the configured inactivity period, attempt to perform an action (for example: open a report, click a folder, refresh the page, or navigate to another section).
  5. Observe that the user is redirected to the login page or displays a “Session expired” message.
  6. The behavior is the same for multiple users and browsers.

Cause

  • The issue occurs because the default session timeout value configured in the BI Launchpad or CMC web application is set to 20 minutes. When users remain inactive beyond this configured duration, the application automatically ends the session.

Resolution

Follow the steps below to increase the session timeout value for BI Launchpad and Central Management Console (CMC):

  1. Stop the Tomcat service.
  2. Navigate to the following directory:
    <BOINSTALLDIR>\Tomcat\webapps\BOE\WEB-INF
  3. Open the web.xml file using a text editor such as Notepad.
  4. Locate the following tag in the file:
    <session-config>
        <session-timeout>20</session-timeout>
    </session-config>
  5. Increase the session-timeout value to the desired duration (in minutes).
    Example: Setting the timeout to 40 minutes.
    <session-config>
        <session-timeout>40</session-timeout>
    </session-config
  6. Save the file.
  7. Start the Tomcat service.

The updated session timeout value will now apply to BI Launchpad and CMC.

Symptom

When the standard data filter LIMIT_FOR_NUMBER is maintained for InspectionHistory, the SSAM app synchronization returns up to the specified number of records for each unique combination of MasterInspChar, MasterInspVersion, and Plant. As of SAP Note 3505548, it is now possible to additionally filter Inspection History results by the EquipId or FuncLocIdIntern property. Since those properties are not taken into account when applying the record limit, the client may display an empty result, although corresponding records exist in the backend.

To address this, two new standard data filters, LIMIT_WITH_EQUIP_ID (True/False) and LIMIT_WITH_FUNC_LOC (True/False), are introduced. When LIMIT_FOR_NUMBER is maintained the uniqueness of each record combination is determined by the properties mentioned above, EquipId (if LIMIT_WITH_EQUIP_ID is set to True) and FuncLocIdIntern (if LIMIT_WITH_FUNC_LOC is set to True).

Please note that the record limitation is applied on the backend. As a result, when only one of the new filters is enabled, for example, LIMIT_WITH_FUNC_LOC, client-side InspectionHistory requests with an additional filter on FuncLocIdIntern will return up to the number of records defined in LIMIT_FOR_NUMBER, whereas requests without a filter on FuncLocIdIntern or with a filter on EquipId may return more records. When both new filters are enabled, client-side InspectionHistory requests with an additional filter on EquipId or FuncLocIdIntern may return more records then defined in LIMIT_FOR_NUMBER.

Additionally, as part of this SAP Note, the sorting order of records prior to applying the LIMIT_FOR_NUMBER filter has been changed. Records are now sorted in descending order by the last change date or creation date, so that the most recently changed or created records are returned first. Previously, records were sorted in ascending order by the creation date, and the oldest records were returned.

Other Terms

S4MOBADDON-8074, InspectionHistory, /MERP/CL_PM_INSPECTION_HIST_OD, APPLY_HISTORY_LIMIT

Reason and Prerequisites

New Functionality (new filter) and Programming Error (sorting change)

Solution

Please implement the SAP Note and, if required, configure the new filters LIMIT_WITH_EQUIP_ID and/or LIMIT_WITH_FUNC_LOC in tr-code /n/SYCLO/CONFIGPANEL -> Section 'OData Mobile Data Object Configuration' -> oMDO Id SAM<XXXX>_INSPECTION_HISTORY -> Tab 'Data Filter' -> Operation READ -> Standard Filter, where <XXXX> is your application version, for example, 2505.

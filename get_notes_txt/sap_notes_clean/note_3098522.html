Symptom

Email notifications are configured for newly created defects or damaged test case work items.
In the manual test execution Web Dynpro, such email notifications are then sent as soon as the creation dialog of the new defect or damaged test case work item ends.

Environment

SAP Solution Manager 7.2, all Support Packages

Reproducing the Issue

  1. Using transaction ‘SMARTFORMS’ create a new smartform SMF informing potential defect processors about the creation of a new defect.
  2. Using the ABAP workbench create a processing class PCL inheriting from class CL_SF_PROCESSING_PPF with a processing method PMTD having the same signature as method EXEC_SMART_FORM of class CL_SF_PROCESSING_PPF. The method implementation needs to provide all the information needed by the smartform to be sent.
  3. In the SAP Reference IMG, run the activity ‘Define Action Profiles and Actions’ (SAP Solution Manager Implementation Guide -> Customer Relationship Management -> Basic Functions ->Actions -> Actions in Transactions -> Change Actions and Conditions -> Define Action Profiles and Actions). Select the action profile ‘SMIN_STD’ maintained in the configuration of the transaction type ‘SMIN’ and navigate to the action definitions. Create a new action record SMDT_MAIL_2_PROC_NEW of processing type ‘Smart Forms Mail’ and enter the following mail settings: Form name = ‘SMF’ (from point 1 above), Processing Class = ‘PCL’ (from point 2 above), Processing Method = ‘PMTD’ (from point 2 above) and Archive Mode = ‘Mail Only’. Save your changes.
  4. In the SAP Reference IMG, run the activity ‘Define Conditions’ (SAP Solution Manager Implementation Guide -> Customer Relationship Management -> Basic Functions ->Actions -> Actions in Transactions -> Change Actions and Conditions -> Define Conditions).
  5. Select the action profile ‘SMIN_STD’ maintained in the transaction type ‘SMIN’ of the defect by double clicking on it and choose and the email notification action SMDT_MAIL_2_PROC_NEW (see point 3 above).
  6. In the details of the action conditions choose the ‘Overview’ tab and select the following settings: Scheduling Condition = ‘Schedule Automatically’, Time of Processing = ‘Processing when Saving Document’.
  7. On the  ‘Schedule Condition’ tab, create a new condition with Name = ‘New Defect’ and body ‘ &CRM Service Request.User Status&  = E0001SMIN0001’ corresponding to the initial status ‘NEW’  ( user status ID ‘E0001’)  of status profile ‘SMIN0001’ of  transaction type ‘SMIN’ )Save your changes.
  8. In the manual test case execution Web Dynpro application create a new defect. Do not press the save button.

Cause

technical limitation

Resolution

This behavior is a known limitation that cannot be resolved.

Keywords

Manual test execution Web Dynpro application, defect, email notification, smartform, CRM post processing framework, action, condition.

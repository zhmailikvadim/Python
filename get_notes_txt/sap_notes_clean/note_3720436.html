Symptom

  • When you execute Argentina Journal Ledger report (AR_JOURNAL_LEDGER) for a specific month, the report considers documents posted on different period.
  • You have "Tax Reporting Date" (VATDATE) activated for the Company Code.

Environment

  • SAP S/4HANA
  • DRC Argentina
  • Statutory Reporting

Reproducing the Issue

  1. Open Fiori Launchpad
  2. Execute "Run Statutory Reports" app (F1515).
  3. Select Argentina Journal Ledger report
  4. Select a month for the Run
  5. Execute a "New Run"
  6. Check the result and there are documents from different posting period than the one selected.

Cause

"Tax Reporting Date" (VATDATE) field incorrectly set for the document. This field is defined with BAdI VATDATE_RULES or by the user when entering the document. More information about it can be found at SAP Note 1232484.

Resolution

  1. Ensure the "Tax Reporting Date" parameter in the documents is correctly set to the desired reporting period.
  2. Alternatively, leave the "Tax Reporting Date" field empty to allow the Posting Date to be considered.
  3. To adjust a document with an incorrect "Tax Reporting Date," use the "Manage Tax Items for Legal Reporting" app (App ID F3172) or FB02.
  4. For mass update of this field, use report GLO_VATDATE.

Symptom

You are looking for patches for your SAP Mobile Documents server.

Other Terms

SAP Mobile Docs, MCM, Bridge, sync

Solution

For more information about how to update your server, see Installing and Updating the SAP Mobile Documents Server.

The following SAP Notes describe the required patch levels for the the underlying AS Java component necessary to run SAP Mobile Documents:

  • Patch Level for AS Java: Note 1818538
  • Kernel Patch Level: Note 1815995

Update your installation to the patch levels mentioned in these SAP Notes.

Version 1.0 SP3

Patch 4

  • Introduced a fix, which solves issues during creation of a public link in an on-premise setup. The root cause of the issue is based in the KM connector and prevents from assigning an anonymous group to the target share, which is needed to allow public access.
  • Upgraded OpenCMIS version to overcome problems with the cookie handling.

Patch 3

  • Introduced a new config value for the connection configuration, which allows to set the maximum number of results for a search. To set the configuration, add the following parameter during the connection configuration in the SAP Mobile Documents Admin UI: com.sap.mcm.QueryMaxItems=<max num. of Results>
  • Fixed a bug in the JCo connection handling, which causes the connection to fail if a user performs multiple operations in parallel.

Patch 2
Fixed a bug when files in a SharePoint repository are marked for sync but deleted on the repository which lead to an error in the iOS client.

Patch 1
Provided a workaround to make SharePoint queries work.

Version 1.0 SP2

Patch 4
Fixed a bug that prevented file uploads in the sharing UI with Microsoft Internet Explorer. Fixed a bug in the admin UI that lead to JavaScript errors when saving settings.

Patch 3
Fixed a cross-site scripting (XSS) vulnerability in the Web browser. Improved SharePoint compatibility for users with read-only permissions. Fixed a bug where the Web client does not display the Create Folder and Upload Document buttons in a SharePoint repository.

Patch 2
Improved compatibility with SharePoint 2010 and SharePoint 2013.

Patch 1
Increased the file size limitation to a maximum of 50 GB. Improved IE9 support. Fixed a bug that lead to wrong statistics.

Patch 0
Released to customer.

If you update the MCMBRIDGE software component to SP2, the Software Update Manager will report a deployment exception. This is due to an internal component refactoring; if you choose Repeat step from point of failure and choose Continue, the update should complete successfully. Restart your application server after the patch is installed.

If you receive HTTP error 500 after this restart, then restart the SAP Mobile Documents application only (com.sap.mcm.server.nw) in SAP NetWeaver Administrator.

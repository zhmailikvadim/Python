Symptom

SUSS withholding tax accumulates base amounts for months in which the vendor is exempt. Accumulation should start from the first non-exempt month. Paid documents from exempt months are being incorrectly recorded instead of starting accumulation from the non-exempt month.

Environment

  • SAP ERP
  • SAP S/4HANA Cloud Private Edition
  • Withholding Taxes for Argentina

Reproducing the Issue

  1. Maintain a vendor with a SUSS exemption for Withholding Tax valid for a given period.
  2. Post invoices and run a payment for documents within the exempt period and for the first non‑exempt month.
  3. Observe that table WTAK (cumulative values for accounts payable) records "WT_BSxx" base amounts for exempt months and that withholding is carried over to the next month.

Cause

  • Argentina-specific SUSS customization is missing: the RG 26821 or RG 26822 withholding tax key is not maintained.
  • The SUSS withholding tax key is not assigned to the withholding tax type and withholding tax code.
  • An invalid CUIT is used for SUSS validation, preventing correct calculation.
  • Incorrect minimum amount customizing is active for the withholding tax type.
  • Open items created before the customizing changes retain incorrect WTAK cumulative base amounts.

Resolution

  • Create the Argentina SUSS withholding tax key (RG 26821 or RG 26822):

    1. Navigate to SPRO > Financial Accounting > Financial Accounting Global Settings > Withholding Tax > Extended Withholding Tax > Calculation > Argentina > SUSS Withholding Tax Keys.
    2. Maintain a key containing 26821 or 26822.

    Assign the created SUSS withholding tax key to Withholding Tax Type and Withholding Tax Code:

    • Navigate to SPRO > Financial Accounting > Financial Accounting Global Settings > Withholding Tax > Extended Withholding Tax > Calculation > Argentina > Assign Withholding Tax Keys.

    Validate and, if needed, correct the CUIT used for SUSS validation:

    • Maintain a valid CUIT via transaction J1ATAXP.

    Important to note:

    • Review the minimum amount customizing for Withholding Tax Type and remove any incorrect "minimum amount" entry so only the correct variant remains.
    • Retest by creating new invoices and processing payments; verify that no base is recorded during exempt periods and that accumulation starts in the first non-exempt month.
    • For open items created before the configuration change, reverse and reissue the documents so that WTAK cumulative bases are rebuilt under the corrected configuration.

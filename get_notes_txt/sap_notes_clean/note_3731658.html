Symptom

When modifying a financial statement version by moving nodes in the Manage Global Hierarchies app (F2918), there is an error that "Subnodes cannot be added to this node"

Environment

SAP S/4HANA Cloud Public Edition

Reproducing the Issue

  1. Open Manage Global Hierarchies app (F2918)
  2. Select a FSV to edit
  3. Select one node and try to assign it as a sub-node of an existing item in the FSV using the cut and paste option
  4. There is in error displayed "Subnodes cannot be added to this node"

Cause

It is not possible to create a subordinate node under an existing node, if the existing node has accounts assigned to it. This is due to the fact any given node can either have subordinate nodes assigned to it or accounts assigned to it. The combination is not possible.

Resolution

  • For any node in a Financial Statement Version, its children can have only one type.
  • This means that if a G/L Account is added under one node, sub-nodes/functional areas cannot then be added under that node.
  • If, however, the accounts assigned to the desired parent node were first removed, subordinate nodes could then be created.

See Also

Refer to SAP Note 3016475 - Unable to Create Sub-nodes for Certain Nodes in a Financial Statement Version

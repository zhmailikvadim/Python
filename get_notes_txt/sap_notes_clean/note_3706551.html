Symptom

A restore or recovery using Backint-based backups does not show the expected backup information on the target SAP HANA system.

The issue occurs after copying Backint configuration files (for example, parameter files for a third-party Backint provider such as AWS Backint Agent) from a source system to a target system.

Even though the same Backint provider and the same physical backup destination (for example, an object store such as Amazon S3) are used, the target SAP HANA system does not display the expected backups during recovery preparation.

This behavior may lead to the assumption that backup catalog or backup metadata from the source system is not directly accessible in the current recovery context.

Environment

SAP HANA DB Version 2.0

Cause

In SAP HANA, backup catalog handling in recovery and copy scenarios depends on the recovery method and tool context.
In Backint-based cross-system scenarios, copying Backint parameter files alone does not by itself guarantee that backups or catalog information from the source system can be directly used on the target system.
If the recovery is not performed with the correct supported procedure and source-system context, the target system may not display the expected backup information.
This does not by itself indicate a product defect.

Resolution

For cross-system restore or copy scenarios with Backint-based backups, use the SAP HANA recovery or copy procedure that is documented for third-party backup tools.
Depending on the scenario, SAP HANA supports recovery without using an existing backup catalog, in which case the required backup must be selected directly from the backup destination.
For Backint-based copy scenarios, ensure that the source-system context is specified as required by the SAP HANA recovery/copy procedure and that the Backint configuration is maintained correctly on the target system.
Refer to the SAP HANA documentation for supported recovery and database copy procedures using third-party backup tools.

Keywords

SAP HANA, Backint, backup catalog, cross-system restore, recovery without backup catalog

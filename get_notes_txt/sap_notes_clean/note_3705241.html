Symptom

We are observing the slowness while creating the custom "Ad-hoc Payment Request Form".
This slowness is while selecting data for chooser for Company Code, On-Behalf of, Cost Center, Internal Order, WBS, General Ledger.

Reproducing the Issue

  1. Create "Ad-hoc Payment Request Form"/any custom form. 
  2. Select the data in the fields Company Code, On-Behalf of, Cost Center, Internal Order, WBS, General Ledger

Cause

If respective datasets having thousands/lakhs of records, there should be slight delay by considering the given volume of data being processed which is an expected behavior in Forms. 

Resolution

To overcome this, we recommend you to please utilize the Dynamic Filtering option. This feature allows you to set up Dynamic Filtering for a field based on predefined relationships between it and another field while designing a form in the forms builder. Implementing this can help improve the efficiency of data retrieval and enhance your overall experience.  

Also, we recommend optimizing the file by removing any unnecessary entries. This will help improve the overall performance and efficiency of the system when processing such large data volumes.  

See Also

Please go through the section: Dynamic Filtering of Custom Form Fields Based on Predefined Relationships Between Fields completely in this help document: https://help.sap.com/docs/buying-invoicing/building-forms-in-forms-builder/dynamic-filtering-of-custom-form-fields-based-on-predefined-relationships-between-fields to understand and to configure to improve the performance of field data while filling the form.

Symptom

  • An error "Change number cannot be used to change material BOM" occurs when attempting to "create" or "change" an object BOM using the generated change number from the change record.

Environment

  • SAP S/4HANA, on-premise
  • SAP S/4HANA Cloud Private Edition

Reproducing the Issue

  1. Attempt to create or change an object BOM using the generated change number from the change record.
  2. Ensure the BOM object is included in the change record item with "Relevance" set to "create/change/reference."
  3. Observe the error occurrence.

Cause

Resolution

Refer to the following path for configuration:

SPRO: SAP Customizing Implementation Guide -> Logistics - General -> Product Lifecycle Management (PLM) -> Change Records -> Settings for Change Records -> Define Change Record Types

Regulated Industry Indicator Defines if the change record is used for a regulated industry. If this checkbox is not marked and a user uses a specific change number to make changes to a business object, the system will use the change number to find the related change record and check if the changed object is assigned to this change record. This check is done for objects of type bill of material (BOM), BOM item (BIT), material (MAT), document (DOC), production routing (PRG) equipment task list (ETL), functional location (FTL), maintenance task list (MTL), and product structure variant (PSV).

Please set Set the no Regulated Industry Indicator for the configured record type and retest to see if the issue persists.

  1. Set the "No Regulated Industry Indicator" for the configured record type.
  2. Retest to verify if the issue persists.

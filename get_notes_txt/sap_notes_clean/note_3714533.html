Symptom

After enabling the parameter Application.Base.AdvancedCurrencyPrecisionHandling and setting the rounding values to ROUNDCEILING for JPY, a discrepancy of 1 JPY has occurred between the accepted amount and the negative acceptance amount of all line items. As a result, the purchase order remains in the receiving status and cannot be processed further.

Reproducing the Issue

1. Enable the parameter Application.Base.AdvancedCurrencyPrecisionHandling.

2. Import the CurrencyInfo.csv file with the following data for JPY:

 UTF-8,,,,,,
 Currency,Precision,Rounding,PricePrecision,AmountPrecision,TaxRounding
 JPY,0,ROUNDCEILING,0,0,ROUNDHALFUP

3. You initially accepted all line items, then negatively accepted all line items for a purchase order.

Cause

It is behaving as expected considering the rounding you use ROUNDCEILING for JPY.

Resolution

  1. Sign into your realm
  2. Click Manager > Core Administration
  3. Click Site Manager > Data Import/Export
  4. Run Import Currency Information task and use this task to configure precision and rounding as ROUNDHALFUP for JPY. 

        A sample CurrencyInfo.csv may look like this:

     UTF-8,,,,,,
     Currency,Precision,Rounding,PricePrecision,AmountPrecision,TaxRounding
     JPY,0,ROUNDHALFUP,0,0,ROUNDHALFUP

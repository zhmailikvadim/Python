Symptom

In Vendor Invoice Management (VIM), using VIM license report or USMM license measurement, the license count might be wrong.

Solution

The issue is resolved in VIM 25.4 SPS1.

For older versions, "Opentext VIM license report 2026" transports D93K901852 (for VIM 7.6/20.4), D51K903865 (for VIM 23.4) and D65K900166 (for VIM 25.4) are available for download in OpenText Support KBA KB0861364:

https://support.opentext.com/csm?id=kb_article_view&sysparm_article=KB0861364

Three activities are required on all SAP systems that process OpenText Vendor Invoice Management (VIM) for SAP Solutions/ICC/IES data:

Step 1 - Import the transport

Step 2 - Report /OTX/PF09_P_PURGE has to be executed (confirm deletion)

Step 3 - Report /OTX/PF09_P_AGGR has to be executed

OpenText strongly recommends validating and testing the license count data in the test system.

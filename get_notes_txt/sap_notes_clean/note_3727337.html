Symptom

  • The Committed Spend amount does not change after modifying a Work Order.
  • The updated Work Order value differs from the Committed Spend amount.
  • While editing a Work Order, the Committed Spend remains unchanged.
  • The values displayed in the Current and New columns appear inconsistent.

Cause

When a Work Order revision is created, the updated financial values are not applied until the revision is fully approved.

During the revision process, the system displays different values for comparison purposes:

  • Current column: Shows the amount from the active Work Order.
  • New column: Shows the updated amount that will apply after approval.

The Committed Spend reflects only the currently approved Work Order value. It will not update until the Work Order revision completes the approval workflow.

Resolution

Verify the values displayed in the Work Order revision:

  • The Current column shows the existing approved amount.
  • The New column shows the revised amount that will apply after approval.

The Committed Spend will update automatically once the Work Order revision is fully approved.

No action is required unless the revision remains pending approval. If necessary, contact the assigned approver to complete the approval process.

Symptom

During upgrade, the tool reports “Open Repairs found” and the transport/task cannot be released. After checking, you find the following errors:

  • Object CPRO CL_SATC_CI_CHECK_ENH_MODULE is inactive
  • Object CPRO CL_SATC_CI_CHECK_MODULE is inactive

Environment

  • SAP NetWeaver - All BASIS releases.
  • ABAP Check Frameworks - ABAP Test Cockpit, Code Inspector

Reproducing the Issue

  1. Start an S/4HANA upgrade and proceed to the pre-processing phase.
  2. Attempt to release a task or transport request as part of the upgrade activities.
  3. The process is blocked with a list of inactive objects, including CPRO CL_SATC_CI_CHECK_ENH_MODULE and CPRO CL_SATC_CI_CHECK_MODULE, and the transport/task cannot be released.

Cause

Resolution

Refer to KBA 3308795 - Error message "Name <MIME objects> is not unique" or "Characteristics of object 'M_TEXT_P' not valid" happens when implementing SAP note 2758146 for S/4 HANA Readiness Check.

See Also

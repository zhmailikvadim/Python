Symptom

In the process "Receipt of Configuration Order After Supplier Assignment to Market Location or Tranche (MOS MaLo View)" (DE_B_ORDR_SUPASS_CONFIG_MOSMALO) at step 0330, or in the process "Preparation and Waiting for Sending of Meter Reading (Template Process)" (DE_B_MET_TMP_PREPARE_SEND_MR), the MSCONS message is not sent when the system attempts to resend the meter reading to a new supplier within 6 hours of the initial send to the DSO.

This issue is relevant for the market role: MOS.

Other Terms

MSCONS, meter reading resend, /US4G/RP_MR_RESEND, /US4G/CL_IM_BADI_DATEXCONN002, DET_REVERSAL_RESEND_MR, /US4G/FM_SEND_MR, IV_SKIP_MR_RESEND_WARNING, US4G_MR_RESEND, DE_B_MET_PREPARE_SEND_MR_MOSMALO

Reason and Prerequisites

The error occurred because the standard solution includes logic to prevent duplicate MSCONS messages in meter reading resend scenarios. When the system finds an existing MSCONS sending record, it checks whether the meter reading has been sent to all receivers. If this check returns true, the system blocks resending the MSCONS within 6 hours, even when the receiver is different.

Solution

Implement the corrections of this SAP Note or import the relevant support package.

After implementing this SAP Note, the system allows you to skip the 6-hour check when a specific receiver is designated for the meter reading resend scenario. The system now correctly sends the MSCONS message to the new supplier, even within 6 hours of the initial send. You can verify the fix by testing a meter reading resend scenario where the MSCONS needs to be sent to a new supplier within 6 hours of the initial send to the DSO, and confirming that the MSCONS message is successfully sent.

If you want to implement your own enhancements for the meter reading (MR) resend, refer to function module /US4G/FM_SEND_MR. Note the following:

1. Call function module ISU_COMEV_MREADING_OUT asynchronously to avoid APE commit conflicts.
2. Specify the actual MR receivers (new supplier) for memory ID US4G_MR_RECEIVERS.
3. Use the below statement to skip the 6 hours check. /us4g/cl_memory_data=>set_skip_mr_resend_warning( abap_true ).

After you complete these steps, the MSCONS with MR mode New is sent to the new supplier.

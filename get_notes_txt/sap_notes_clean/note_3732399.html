Symptom

  • Attempt to upload the TCI correction in SPAM via frontend (Load Packages > SAP archives from Front End) fails due to file size limitations.
  • Message displayed:

    "The file transfer with the GUI upload function can take a long time for files that are larger than 200 MB.
    The file 'KXXXXXXXXXX.SAR' is XXX.XXX.XXX bytes in size. You should therefore use FTP to transfer it to the application server and unpack
    it there, as described in the 'Load Support Package' section of the Online Help."


Environment

  • Support Package Manager (SPAM)
  • SAP System based on SAP NetWeaver
  • S/4 HANA

Reproducing the Issue

  1. Attempt to upload SAR file for the TCI correction via SPAM using Load Packages > SAP archives from Front End
  2. Observe the size-related warning for files >200 MB and that the upload does not proceed

Cause

The correction archive (SAR file) exceeds the effective limit for SPAM GUI upload (>200 MB).

Resolution

If the archives are larger than 200 MB, proceed as follows Loading Support Packages from the Application Server.

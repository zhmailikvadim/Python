Symptom

The response for the Import Requisitions created from Ariba does not contain the UniqueName or StatusString fields. Instead, a blank response is returned, similar to the example shown below:

<soap:Envelope xmlns:soap="http://schemas.xmlsoap.org/soap/envelope/" xmlns:xsd="http://www.w3.org/2001/XMLSchema" xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance">
    <soap:Header>
        <Headers xmlns="urn:Ariba:Buyer:@variant">
            <variant>@variant</variant>
            <partition>@partition</partition>
        </Headers>
    </soap:Header>
    <soap:Body>
        <RequisitionImportPullReply xmlns="urn:Ariba:Buyer:@variant" partition="@partition" variant="@variant"/>
    </soap:Body>
</soap:Envelope>

Cause

In the standard configuration, when internal budgets are enabled for requisitions and an imported requisition is created and moves to Submitted status, internal budget processing is triggered for that requisition. Since both the internal budget processing and the import requisition processing run in the same thread, the web service response does not include the UniqueName or StatusString.

Resolution

To resolve this issue, the system can be configured to process certain requests in the background instead of handling everything within the same thread. This helps maintain the correct system context and ensures that responses are returned properly, even when internal budget processing is involved.

This behavior is controlled by the Application.Budget.InvokeBudgetPDEInSeperateThread parameter. When this setting is enabled, any requisitions imported into SAP Ariba Buying solutions will be processed and submitted separately whenever Budget PDE calls are required.

In simpler terms, enabling this option allows the system to handle complex steps independently without interrupting the main process. This improves reliability and helps prevent situations where the system does not return a response as expected.

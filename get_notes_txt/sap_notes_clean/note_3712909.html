Symptom

1) A file for the Flexible Upload of Reported Financial Data contains the following information:
  • The PERIODICAL parameter is blank for the cumulative (YTD) input type.
  • The rows of the uploaded data contain multiple values in the “FiscalPeriod” column.
2) The file was successfully uploaded.
3) The information uploaded to the ACDOCU is incorrect and does not match the uploaded file.

Environment

  • SAP S/4HANA 1909
  • SAP S/4HANA 2020
  • SAP S/4HANA 2021
  • SAP S/4HANA 2022
  • SAP S/4HANA 2023
  • SAP S/4HANA 2025

Reproducing the Issue

  1. Prepare a file to import using the Flexible Upload of Reported Financial Data functionality, with an empty PERIODICAL parameter and operations spanning more than one period.
  2. Perform the import.
  3. Review the resulting records in the ACDOCU.

Cause

  1. The described behavior is a deliberate limitation of the Flexible Upload of Reported Financial Data functionality: "Year-to-Date (YTD)" mode is not supported when uploading multiple periods in a single file.
  2. As per SAP Documentation (see "Parameters: > PERIODICAL" section), "Enter x if you upload multiple periods. Cumulative data is not supported for this case for performance reasons."

Resolution

The files to upload using the Flexible Upload of Reported Financial Data  should be prepared considering the mentioned limitation:
  1. If the "PERIODICAL" parameter is left empty (YTD), it’s necessary to prepare several separate files, one for each period.
  2. Alternatively, to import multiple periods into a single file, set the "PERIODICAL" parameter to "X" (periodic input, PER). The file should contain periodic amounts, not YTD amounts.

Symptom

  • After upgrading ASE from version 16.0 to 16.1, significant performance degradation is observed when executing SQL statements containing a large number of 'INSERT <table> SELECT <scalars> UNION ALL SELECT <scalars>' when there are many union sides.
  • Trace Flag 491 is enabled in both 16.1 and 16.0 servers.
  • Compared to ASE 16.0, the same workload requires significantly more time to complete on ASE 16.1.

Environment

SAP Adaptive Server Enterprise (ASE) 16.1 SP00 PL01

Cause

  • For UNION ALL SELECT query with many union sides, the performance degrades dramatically when using ASE Optimizer from version 15.x and higher. This was introduced in an old CR 582989 titled "ASE 15 optimizer takes 18x longer to run a UNION ALL query vs Compatibility Mode", and Trace Flag 491 was introduced to fix the issue.
  • TF491 actually is calling a piece of code internally called 'fastpath', the fastpath code is encapsulated in the Compatibility Mode code. However in SAP ASE 16.1, by default the "compatibility mode" behavior has been removed and the Compatibility Mode code are blocked to be accessed, so the TF491 has no effect anymore.
  • Enable Trace Flag 11035 can restore the access to the Compatibility Mode code, so the 'fastpath' code can be accessed again for TF491.

  • The issue has been logged under SAP CR 831733. Its description reads, "Performance degradation occurred after upgrading to ASE 16.1 SP00 PL01".

Resolution

There is currently no target SAP ASE Service Pack scheduled for CR 831733. It has been assigned to our Product Group. Please mark this KBA as a Favorite to be automatically notified when the CR status is changed.

Please enable Trace Flag -T11035 and keep existing Trace Flag -T491 together to improve the performance.

Symptom

  • After enabling the NSE Advisor in SAP HANA Cockpit 2.17.7, the advisor immediately switches back to Disabled.
  • No recommendations are generated even when a long collection duration is configured
  • The behavior occurs in the NSE tile of Hana cockpit version 2.17.7.
  • Advisor runs do not return any output.
  • No specific error messages or error codes are shown.

Environment

  • SAP HANA Platform 2.0
  • Native Storage Extension (NSE) Advisor
  • SAP HANA Cockpit 2.17.7

Reproducing the Issue

  • Open SAP HANA Cockpit
  • Navigate to: Workload Management → Native Storage Extension Advisor
  • Enable the NSE Advisor and configure a long data collection duration (for example, 1440 minutes).
  • Advisor briefly shows state Enabled and automatically returns to Disabled.
  • Allow the advisor job to run to completion in the database and then check the tile; no recommendations are shown.

Cause

  • Known issue in SAP HANA Cockpit 2.17.7 affecting advisor initialization
  • If there is little or no relevant workload during the advisor window, the advisor may produce no recommendations, so the tile appears to show nothing.
  • Workload Capture is not running.
  • Insufficient privileges for the cockpit technical user.

Resolution

  • Avoid configuring very large collection durations for the NSE Advisor (for example, reduce from 1440 minutes).
  • Plan to upgrade to the SAP HANA Cockpit release that includes the fix (SP19) when it is available.
  • Ensure sufficient database workload occurs during the advisor window (for example, queries, inserts, updates, selects) so the advisor can gather statistics and produce recommendations.
  • Verify advisor execution and outcomes by querying system views, for example, M_CS_NSE_ADVISOR_STATISTICS, to confirm status and collected data.
  • Assign required system privileges to the cockpit user.

Symptom

  • The following error occurs in the "Response" field while testing connection to SFTP server via SSH in CPI:

    com.jcraft.jsch.JSchException: verify: false

  • Parameter "Enable Support for Deprecated Algorithms" is enabled.

Environment

  • SAP Integration Suite
  • Cloud Integration

Reproducing the Issue

  1. Login to CPI.
  2. Navigate to "Integrations and APIs".
  3. Select tile "Connectivity Tests".
  4. Choose SSH.
  5. Insert the access details of SFTP remote server.
  6. Check/enable parameter "Enable Support for Deprecated Algorithms".
  7. Press "Send".

Cause

BTP including CPI upgraded JSch from version 0.2.17 to 2.27.4. Starting with JSch version 0.2.19, during the Diffie-Hellman Group Exchange, the group exchange key length (SSH_MSG_KEX_DH_GEX_REQUEST) is required to be within the range of 2048 bits (minimum) to 8192 bits (maximum). Older JSch versions (≤ 0.2.17) contained a defect that allowed out-of-range Diffie-Hellman key sizes to be silently accepted, which is why the connection succeeded prior to the upgrade.

Resolution

Refer to standard RFC 8270 and ensure all SFTP servers comply with the selected algorithm security standards, specifically that the group exchange key length is no less than 2048 bits and no greater than 8192 bits, as this is a security enforcement requirement.

See Also

RFC 8270: Increase the Secure Shell Minimum Recommended Diffie-Hellman Modulus Size to 2048 Bits

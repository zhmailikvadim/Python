Symptom

A (partially) unplanned Freigt Unit is constantly locked. This predominantly happens in an intercompany context.

Other Terms

Transportation Management; Logistics Integration; Trigger; /SCMTMS/CL_TM_LE_DLV_COMPARE;

Reason and Prerequisites

Technical reason/context: TOR save strategy triggers asynchronous TM-LE update, which eventually causes the asychronous save strategy, again. The loop is endless because the necessity to update LE out of TM is detected again (program error).

Solution

 Implement the corrections of the attached correction instructions or apply the corresponding support package.

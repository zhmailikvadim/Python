Symptom

  • In production or process orders (e.g. created via CO01 / COR1), inconsistencies are observed in:

    • Basic start and end dates

    • Scheduled dates

    • Operation start and finish dates

  • Example symptoms include:

    • Operation sequence appears inconsistent (e.g. operation 0010 starts after operation 0020)

    • Dates change automatically after pressing Enter during order creation

    • Small changes in order quantity lead to different scheduling results

    • Start/finish dates do not follow the expected routing sequence

Environment

  • SAP ERP Capacity Planning (PP-CRP)
  • SAP R/3
  • SAP R/3 Enterprise
  • SAP ERP Central Component
  • SAP ERP
  • SAP Enhancement package for SAP ERP
  • SAP Enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA, on-premise
  • SAP S/4HANA Cloud Private Edition

Reproducing the Issue

  1. Access transaction CO01

  2. Maintain basic dates (e.g. end date)

  3. Press Enter

  4. Basic dates, scheduled date or operations dates cannot be explained

Cause

Cause 1: Inconsistency Between Capacity Intervals and Factory Calendar

  • Work center capacity intervals (trasnaction CR02) extend beyond the validity of the assigned factory calendar

  • This leads to incorrect or unexpected scheduling results

Example:

  • Work center Interval maintained until 2035

  • Factory calendar valid only until 2030

Scheduling processes inconsistent time definitions

Cause 2: “Relevant for Finite Scheduling” Indicator Not Set in Work Center

  • The indicator “Relevant for finite scheduling” is not activated in one or more work centers used in the order.

In this case:

  • The system may ignore capacity constraints for these operations

  • Scheduling results can become inconsistent when compared to other operations that are capacity-relevant

This may lead to:

  • Operations being scheduled in an unexpected sequence

  • Differences in start and finish dates between operations

Cause 3: Scheduling Type Not Aligned with Alignment Key

The scheduling result may be inconsistent if the scheduling type (e.g. backward scheduling) is not aligned with the alignment key configuration.

According to standard system behavior (see SAP Note 416571), the scheduling direction must be consistent with the alignment logic:

  • Backward scheduling → should be used together with “Alignment with latest dates”

  • Forward scheduling → should be used together with “Alignment with earliest dates”

If this alignment is not respected:

  • The system may calculate basic and scheduled dates inconsistently

  • Operation dates may appear out of sequence

  • Dates may shift unexpectedly after pressing Enter during order creation

Resolution

Cause 1: Inconsistency Between Capacity Intervals and Factory Calendar

  • Access transaction CR02

  • Navigate to:

    • Capacity tab → Capacity Header Data → Intervals and Shifts

  • Adjust the interval so that:

    • It does not exceed the validity of the factory calendar

Example:

  • If the factory calendar is valid until 2030, limit the work center interval to 31.12.2030

Mass Check for Inconsistencies

To identify all affected work centers:

  1. Go to transaction SE38

  2. Execute report RCCYCALCHECK

The report lists work centers with inconsistencies between:

  • Capacity intervals

  • Factory calendar validity

⚠️ Important:

  • Report RCCYCALCHECK is for analysis purposes only

  • It does not correct any inconsistencies automatically

All required corrections must be performed manually, for example, using transaction CR02 to adjust capacity intervals accordingly.

Cause 2: “Relevant for Finite Scheduling” Indicator Not Set in Work Center

  1. Go to transaction CR02

  2. Access the relevant work center

  3. Navigate to the Capacity tab

  4. Ensure that the indicator “Relevant for finite scheduling” is activated

After adjusting, re-create or re-schedule the order to validate the behavior

Cause 3: Scheduling Type Not Aligned with Alignment Key

  • Verify the scheduling type used in the order:

    • Forward or Backward scheduling

  • Check the configuration of Alignment key in the routing (transaction CA02) or directly in the order in "Sequences" tab.

  • Ensure that the configuration is consistent:

    • Backward scheduling → Alignment with latest dates

    • Forward scheduling → Alignment with earliest dates

  • If inconsistencies are found:

    • Adjust the customizing accordingly

    • Recreate or re-schedule the order to validate the behavior

For more information, refer to  SAP note 416571 - First/last operation not like order start/end

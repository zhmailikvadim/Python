Symptom

Customers use "non‑zero differences" filter in ICMR APP Reconciliation Balances. Division by zero error undefined error like below appeared.

“A database error occured. The error message in the database was: "division by zero undefined: search table error: [6859] AttributeEngine: divide by zero;exception 70006859: AttributeEngine: divide by zero;abs(decfloat(sqladd(fixed("COL$0$", 27, 2), fixed("COL$1$", 27, 2))) [here]/ decfloat("COL$0$")) * decfloat(100) > 0.00, (COL$0$ = 0.00[fixed12_26.2], COL$1$ = 0.00[fixed12_26.2]),docid 1,PS1::SAPHANADB:#_SYS_QO_COL_L:2566_7f2793954130:401800000857efd5 (t 4619567317915267324)".”"

Environment

  • Group Reporting
  • SAP S/4HANA
  • SAP S/4HANA Cloud Private Edition

Reproducing the Issue

  1. Open the "Reconciliation Balances" Fiori app.
  2. Select the relevant Reconciliation Case and Display Group, enter the Period/Year, and choose Go.
  3. In the Differences column, open the context menu and choose "Non‑Zero Differences".
  4. Observe that a database error "division by zero undefined…" is raised.

Cause

Missing SAP Notes.

Resolution

Implement below SAP Notes in your system and then test again.

SAP Note 3729733 - Division by zero undefined error in Reconciliation Balances app

SAP Note 3098362 - FIN_CS_ICARR Frontend Correction for SAP S/4HANA 2021 and Beyond

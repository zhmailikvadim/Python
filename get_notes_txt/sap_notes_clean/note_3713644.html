Symptom

Saving Process Steps in Solution Documentation is slow

Other Terms

Element Order, Sequence

Reason and Prerequisites

Sequence calculation algorithm is slow for many data.
If a customer creates only Process Steps with option ‘New Process Step Original’ in Solution Documentation, then all of them are created under one folder. This can lead to performance issues if there are many.

Solution

Please implement the current correction instruction with transaction SNOTE.
After that, the performance should be much better.
Use the report SMUD_DELETE_INCON_ORDER to delete the inconsistent order / sequence changes.
Please read the attached document 'Process Step Creation'.
Use option ‘Specify Executable’ when creating for a specific executable a Process Step Original the first time and for the next Process Step Original for the same executable with option ‘Search in Process Step Library’, this reduces the creation of process step originals.

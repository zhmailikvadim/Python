Symptom

When try to replicate documents from CFIN source system to target system, it triggers error message FINS_CFIN_AC_DOC273 in CFIN target system,

Message Text: Valuation area not determined, although it is filled in source system. 

Environment

  • Financial Accounting (FI)
  • SAP S/4HANA 
  • SAP S/4HANA Finance
  • SAP S/4HANA Cloud Private Edition

Reproducing the Issue

  1. Try to replicate documents from CFIN source system to CFIN target system, 
  2. Check with initial load result with transaction code FINS_CFIN_LOAD_GRP_M, or check the real time replication in transaction /AIF/ERR,
    It observes the error messages: 
    Valuation area not determined, although it is filled in source system. Message number: FINS_CFIN_AC_DOC273.

Cause

There is check logic introduced with SAP Note 2670981

"...

BWKEY is determined according to the following rules:

  • If the valuation area is not filled in the Source System, it is not filled in the Central Finance System either
  • If the valuation area is filled in the Source System, it is determined newly, depending on the Valuation area setting (database field TCURM-BWKRS_CUS):
    • If the value of the field TCURM-BWKRS_CUS is 1 (=plant), the valuation area is set to the name of the plant (database field T001W-BWKEY) in the Central Finance System.
    • If the value of the field TCURM-BWKRS_CUS is 3 (=company code), the valuation area is set to the company code (database field BUKRS) in the Central Finance System.

After the valuation area has been determined, the following Checks have to be done:

  • For inventory postings (KTOSL = 'BSX'), the valuation area must be filled
  • If the valuation area is filled in the Source System, the valuation area must be filled the Central Final System also.

..."

If the valuation area is available in the document of CFIN source system, it needs to make sure the related value for 'plant' or 'company code' is available in CFIN target system, the value of 'plant' or 'company code' will affect the valuation area determination in CFIN target system.

Resolution

It needs review the related mapping for Plant - WERKS, also the value maintained in CFIN target system.

It needs make sure the required value is available in CFIN target system.

    

It also needs pay attention about the mapping action for the mapping entity "PLANT_ID", 

In some scenarios, if the mapping action is configured with "3 Clear Data", during replication the value of PLANT_ID will be deleted, if the value of PLANT_ID is initial, then it will affect the valuation area determination logic.

For this specific scenario, it will need consider to change the mapping action accordingly.

  

Kindly note the mapping changes in production system needs be fully tested in non-production system first!

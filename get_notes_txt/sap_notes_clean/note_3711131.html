Symptom

You ran the SPM_AU_LOG archiving with the "Archive with Deletion" option.

Symptom #1: After the job completed, no records were deleted from the GRACAUDITLOG table, and a DBSQL_SQL_ERROR runtime error appears in ST22.

Database error text: "SQL message: [ASE Error SQL12213]Resource Governor lock count limit (#####) exceeded"

Symptom #2: The archiving process is canceled by a runtime error: TSV_TNEW_PAGE_ALLOC_FAILED.

Other Terms

GRC, Access Control, GRACAUDITLOG, archiving, SPM_AU_LOG, GRAC_SPM_AUDIT_LOG_ARCHIVE_W, dump, ST22,  DBSQL_SQL_ERROR,  CX_SY_OPEN_SQL_DB, TSV_TNEW_PAGE_ALLOC_FAILED, Resource Governor lock.

Reason and Prerequisites

Reason #1: Resource Governor lock count limit was exceeded due to the large volume of auditlog data selected for archiving.

Reason #2: No more memory available to add rows to an internal table.

Solution

Implement the attached Correction Instruction.

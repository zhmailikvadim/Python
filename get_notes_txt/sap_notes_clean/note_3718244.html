Symptom

When generating the I-9 Section 3 (Reverification) PDF form, the document appears with blank values instead of the expected employee data.

Environment

SAP SuccessFactors Onboarding

Cause

This issue is caused by a data inconsistency in the I-9 records.

The I-9 task needs to be retriggered due to incorrect or incomplete status updates in the system.

Resolution

To resolve the issue, update the status in both Manage Data > I-9 User Data and Manage Data > I-9 Audit Trail to SECTION2_SIGNED.

  • In I-9 Audit Trail, ensure you search for and update the record with the latest sequence number.

After correcting these objects, verify whether the I-9 Section 3 task is automatically created and assigned to the responsible team.

If the task is not created automatically, manually trigger the process by navigating to Manage Data > Create New > Trigger I-9 Flow > I-9 Reverification.

Once completed, the I-9 Section 3 PDF form should generate with the correct values.

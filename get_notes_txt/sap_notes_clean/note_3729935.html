Symptom

When updating a User-Defined Table (UDT) using the Data Interface (DI) API within a Global Transaction, the operation may fail with the following error if an unrelated document or master data is created in the SAP Business One client at the same time:

"Another user or another operation modified data; to continue, open the window again (ODBC -2039)"

If the DI API application uses the GetByXML operation and writes to log files, you notice that the UDT metadata may no longer be visible.

Reproducing the issue

Prerequisites:

  • A UDT of type No Object exists, with at least one User-Defined Field (UDF).
  • The DI API application uses a Global Transaction (StartTransaction / EndTransaction).
  • The DI API connection is established before any actions are performed in the SAP Business One client.

Scenario:

  1. Launch the DI API application and connect to the database.
  2. Perform an update on the UDF within the UDT.
  3. Log in to the SAP Business One client and create a new document or master data.
  4. Using the DI API application, attempt to update the same UDT again.

Expected behavior:

The UDF is updated successfully every time, regardless of other operations in the SAP Business One client.

Actual behavior:

Intermittently, the DI API update fails with the error:

"Another user or another operation modified data; to continue, open the window again (ODBC -2039)"

After this occurs, further updates on the UDT fail consistently.

The DI API log files may also contain an additional error:

"-3000: The logged-on user does not have permission to use this object"

Solution

Symptom

  • Deleted performance form cannot be restored using the "Restore Deleted Document" functionality
  • The system displays the message: "1 total forms. Successfully restored: 0. Failed to restore: 1."
  • In logs and when trying to retore form via Provisioning, error appears: "foreign key constraint violation: TrexUpdate failed on table XXX:FOLDER_MAP with error: Only found 0 of 1 rows from XXX:FOLDER_MAP(FOLDER_ID) in XXX:FOLDER(FOLDER_ID)"

Environment

SAP SuccessFactors Performance Management 2025

Reproducing the Issue

  1. Open "Restore Deleted Form" in the Admin Center
  2. Enter the Form ID and click "Search"
  3. Select the form and click "Restore"
  4. Observe the message stating: "1 total forms. Successfully restored: 0. Failed to restore: 1."

Cause

These logs indicate that the form cannot be found in the employee's folder. This occurs because the form's owner or subject user has either been purged or deactivated.

Resolution

Verify whether the form's owner or subject user has been purged or deactivated.

  • If the employee has been purged: The forms cannot be restored because the chain of user association has been removed. Purge the affected forms using the legacy "PM or SM Data" purge type.
  • If the employee has been deactivated: Reactivate all users involved in the form's route map. Afterwards, retry restoring the form using the "Restore Deleted Document" feature

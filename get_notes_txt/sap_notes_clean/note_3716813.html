Symptom

  • Planning level is not updated for Money Market(MM) transaction in cash management integration.
  • Liquidity report does not display data for transaction 1000000000010 under liquidity item e.g. ST_LQI.

Environment

SAP S/4HANA Cloud Public Edition

Reproducing the Issue

  1. Verify the planning level assignment for transaction 1000000000010 in the configuration activity "Assign Planning Levels".
  2. Observe that data for transaction 1000000000010 is not displayed under liquidity item ST_LQI.

Cause

  • The combination of product type e.g. 55B (Tranche), transaction type 200 (Borrowing), activity category e.g. Contract settlement (28), and product category e.g. 550 (Interest Rate Instrument) is not defined in the configuration "Assign Planning Levels" for MM planning level.

Resolution

  1. Navigate to the configuration activity "Basic Settings for Cash Management Integration".
  2. Define the planning level MM explicitly for the combination of product type 55B (Tranche), transaction type 200 (Borrowing), activity category Contract settlement (28), and product category 550 (Interest Rate Instrument) in the configuration "Assign Planning Levels".
  3. Execute the Flow Builder to update the cash flow data.

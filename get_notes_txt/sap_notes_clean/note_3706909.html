Symptom

When trying to run t-code WSP4 in background mode or schedule a background job with program RWSORT14, the job is canceled with the following error message: 
WM750 - Material XXXX locked by assortment functions being performed by user  XXXX.

Environment

  • SAP ERP
  • SAP S/4HANA
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • Retail

Reproducing the Issue

  1. Go to T-code WSP4 and input the necessary data.
  2. Click on 'Program' and select 'Execute in Background'.
  3. Check transaction SM37; the job is canceled. The error message WM750 appears in the error log.

Cause

The transaction WSP4 (program RWSORT14) has never been enabled to run in background mode.
The reason for this is that the process is designed to list only one article to one assortment in dialog mode.
When used in background mode, it not only causes locking issues but also triggers a selection screen as a follow-up process, which is not possible in background mode, leading to additional errors.

Resolution

Instead of using WSP4, use WSM3 to list articles via a background job.

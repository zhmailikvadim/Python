Symptom

You are running SAP LT Replication Server (SLT) and you are using an S/4HANA 1809 FPS02 source system. When you want to load a table with a compatibility view and you choose reading type 5 together with an entry in table IUUC_PRECALC_OBJ (to parallelize the access plan calculation), the access plan calculation fails. You find UNCAUGHT_EXCEPTION-runtime errors in the source system. The exception is CX_SQL_EXCEPTION and it is raised in class CL_SQL_STATEMENT. The error analysis section of the runtime error states something like "sql syntax error: incorrect syntax near "ANDBUKRS"..." (i.e. the "AND" in a WHERE-condition and the column name "BUKRS" are glued together).

Other Terms

LT Replication Server, SLT, replication, replicator, trigger, realtime, S/4HANA 1809

Reason and Prerequisites

S/4HANA 1809 FPS02 is used as a source system for SAP LT Replication Server or S/4HANA 1809 FPS02 is used as SAP LT Replication Server in the context of Central Finance.

Solution

Implement the required corrections on the S/4HANA source system (and the S/4HANA central system when using it in the context of Central Finance) in accordance with the correction instructions.
Replication Jobs have to be stopped during note implementation: no

Manual Steps: none

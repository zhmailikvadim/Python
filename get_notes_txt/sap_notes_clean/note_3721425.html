Symptom

The classic FI amount release workflow for parked documents is customized. For some approved documents the posting step in background fails. This happens for accounts with the "Supplement for automatic postings" indicator active in master data settings and corresponding postings with automatic line item creation (e.g. some tax related postings). Symptoms like "No batch input data found for dynpro SAPMF05A 0700. SAPLKACB 0002" (00 344) can occur in the workflow log.

Other Terms

FP132, FP 132, SAPMF05A, SAPLKACB 0002, SAPMF05A 0700, 00344, 00 344

Reason and Prerequisites

Program issue.

Solution

Implement the attached correction or corresponding support package. After implementation the posting step in the workflow won't consider this master data setting ("Supplement for automatic postings") of the GL account so that the posting can take place. As the approver has approved the document and the technical workflow user can't add anything to automatically created line items, it does not make sense to stop the posting otherwise.

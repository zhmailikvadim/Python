Symptom

  • Custom fields were added to the supplier registration form and mapped in the integration.
  • The outbound Request_In payload reaches SRT_MONI without errors.
  • Custom fields are not detectable in available BAdIs on the S/4HANA side.
  • The receiving system proposed changing the XML payload structure/namespace so the fields can be recognized.

Environment

  • Product: SAP Supplier Lifecycle and Performance
  • Supplier Lifecycle & Performance - XML Integration

Reproducing the Issue

  1. Add custom fields to the supplier registration form and complete the integration mapping.
  2. Submit a supplier registration so the outbound Request_In payload is sent to S/4HANA.
  3. Verify in SRT_MONI that the message posts without errors.
  4. Check available BAdIs on the S/4HANA side and observe that the custom fields are not visible/consumed.
  5. Consideration is raised to adjust the payload namespace/structure to surface the fields.

Cause

customise Outbound/Request_In payload sent from Ariba SLP.

Resolution

  1. The Outbound/Request_In payload from SLP is system-generated and cannot be modified or altered, preventing the proposed XML structure/namespace changes.
  2. Engage the ERP/ABAP team to implement an appropriate BAdI on the S/4HANA side that reads and processes the current GenericCustomField elements as delivered.
  3. Review and, if needed, adjust the ERP-side inbound mapping so the BAdI receives the expected fields from the existing payload structure.

Keywords

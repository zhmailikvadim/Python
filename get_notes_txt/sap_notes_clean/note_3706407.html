Symptom

  1. TR requests is handled by ChARM process.  
  2. SE09 shows them that they are successfully imported in Target system.
  3. the transport entries in UJT_TRANS_OBJECT are showing  "D" version. [Expect to be A version]

Environment

  • SAP Business Planning and Consolidation 10.1, version for SAP NetWeaver
  • SAP Business Planning and Consolidation 11.X, version for SAP BW/4HANA
  • SAP Business Planning and Consolidation 2021, version for SAP BW/4HANA

Reproducing the Issue

  1. Use T-cd: STMS to import a TR on the target system
  2. SE09, check the transport request status as imported
  3. SE16, UJT_TRANS_OBJECT shows transport entry as D version
  4. transported objects are not updated in BPC environment

Cause

logon data missing in BW RFC (which is defined as the BW transport destination: Check table RSADMINA, field TPBWMANDTRFC, or transaction code RSTPRFC)

Resolution

  1. SM59, find RFC entry, and fill the client, user and password, the status should be "Saved"
  2. for the affected requests:
    1.  import the requests via the repair tool: BW program RSDG_AFTER_IMPORT_FOR_CORR
    2.  the affected transport request entries in UJT_TRANS_OBJECTS should have OBJVERS set to 'A',

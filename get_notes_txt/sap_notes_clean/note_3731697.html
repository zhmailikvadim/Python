Symptom

After a downtime-optimized conversion (doC) to SAP S/4HANA with source database platform DB2 for Linux, Unix and Windows, entries may be missing in table NRIV.
All of the following conditions must be fulfilled for a system to be affected:

  1. The scenario running is a downtime-optimized conversion (doC) to SAP S/4HANA using SUM 2.0 SP25.
  2. The ECC source system is running on database platform DB2 for Linux, Unix and Windows.
  3. The system was used productively during the SUM run (not a sandbox), and number ranges were created or updated after SUM phase EU_CLONE_MIG_OPTDMO_INI_RUN.

Other Terms

doC, NRIV, DB2, number ranges

Reason and Prerequisites

Changes to table NRIV are not updated in the target SAP HANA database.
Root cause is a programming error. 

Solution

The error is fixed with SUM 2.0 SP26.

It will also be fixed in SUM 2.0 SP25. Once the patch is available in SUM 2.0 SP25, this note will be updated with the exact patch level that contains the fix.

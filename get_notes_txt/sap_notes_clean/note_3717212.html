Symptom

Reopened row in item type order can be deleted without any error.

Environment

SAP Business One

Reproducing the Issue

Prerequisites:

In Administration → System Initialization → Document Settings, option Reopen Doc. by Creating Returns/Goods Returns/Credit Memos Based on Doc. is selected

Scenario:

  1. Create an item type sales order with 2 rows.
  2. Copy the first row to an A/R invoice.
  3. Either cancel the A/R Invoice or copy the A/R Invoice to an A/P credit memo.
  4. Open the base Sales Order and right click the reopened first row.
  5. Choose Delete Row and click Update.

Expected behavior:

The reopened row cannot be deleted.

Actual behavior: 

The reopened row is able to be deleted successfully.

The behavior is same in Purchase Order.

Resolution

This is the correct system behavior.

Keywords

Sales Order, Purchase Order, Reopen, Delete Row, Target, Base, Item

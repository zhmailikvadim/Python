Symptom

All rows after a million rows are lost while generating a huge Web Intelligence report by a schedule in Microsoft Excel format

Environment

  • SAP BusinessObjects BI Platform 4.x
  • Data Provider: Oracle Database 10g/11g
  • Microsoft Office Excel 2010
  • Microsoft Windows Server 2008 R2

Reproducing the Issue

  1. Log on to BI Launch Pad
  2. Right-click a huge Web Intelligence report, containing more than a million rows, select "Schedule"
  3. Select "Formats" tab, set "Microsoft Excel" checkmark
  4. Click Schedule
  5. Wait until the Web Intelligence report is complete
  6. Open the Excel Spreadsheet resulted
  7. See there are no rows after a million rows

Cause

The root cause is under investigation

Resolution

For workaround: （For BI versions before BI4.2 SP03）

Configure on Adaptive Job Server to use .xls file format as follows:

  1. Log on to Central Management Console (CMC) as an Enterprise Administrator
  2. Go to Servers -> Web Intelligence Services -> Adaptive Job Server
  3. Right click on it and select "Destination"
  4. From the drop down box, Add File system destination
  5. Specify the directory location in "Directory" section
  6. Under "File name" option, Select the option "Use Specified File Name"add the Placeholder for "Title" and append it with .xls (this line will look like as %SI_NAME%.xls)
  7. Uncheck the option "Add file extension"
  8. Save and close

Schedule the Web Intelligence report as follows:

  1. Log on to BI Launch Pad
  2. Find the report which you want to schedule, right click it
  3. Click on "Schedule"
  4. Select the format as "Microsoft Excel" in Format section
  5. In "Destinations" option
  6. Select the "File system" destination
  7. Check the option "Use Default Settings"
  8. Click on schedule

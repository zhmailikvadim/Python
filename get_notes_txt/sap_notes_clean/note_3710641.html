Symptom

The system defaulted the FTE to 1 when Location, Job Classification is changed from Quick actions. 

Environment

SAP SuccessFactors Employee Central: Job Information

Reproducing the Issue

  1. Make sure that standard field "standard-hours" & "FTE" are enabled. 
  2. Navigate to user profile: abc. 
  3. Take Action > Select quick action: Change Location.
  4. Select new location. 
  5. Click save. 
  6. Observe that FTE is changed to 1. 

Resolution

  • This is expected behavior and is as per system design due to the system logic of automatic FTE calculation when FTE field is not visible on UI. guide: Calculating FTE | SAP Help Portal 
  • The automatic FTE calculation is triggered for quick action, "Change Location", "Change Job Classification".
  • If you wish to enhance this behavior. Kindly, create a influence request via influence tool (https://www.sap.com/germany/about/customer-involvement/influence-adopt.html) -> SAP Continous Influence -> SAP SuccessFactors Employee Central (https://influence.sap.com/sap/ino/#/campaign/85).
  • Please refer  https://launchpad.support.sap.com/#/notes/2090228

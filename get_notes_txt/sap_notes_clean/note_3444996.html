Symptom

You have executed actual costing run, and if you check price difference details for some activity types, you may find it's not according to your expectation.

Environment

SAP S/4HANA Cloud

Cause

You can check app Activity Consumption Analysis (CKM3A) for that specific activity type.

You can from Prices section to know the current status and the correct calculation.

"Standard Price" means preliminary price. It's the plan cost rate set in the period beginning.

"Periodic Unit Price" is the actual cost rate (normally calculated by app "Actual Cost Rate Calculation - Cost Centers").

The correct unit price difference should be "Periodic Unit Price" - "Standard Price".

There is a common issue, if you set "Do not process" in steps "Preparation" and "Settlement" in app "Edit Actual Costing Run", any changes will not be captured.

Resolution

Suggest you to select "Process Again" option in the steps and execute all the steps again.

Keywords

CKMLCP, price difference, activity type, KSII, wrong, incorrect, not as expected.

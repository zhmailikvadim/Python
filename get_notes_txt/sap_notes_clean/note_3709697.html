Symptom

You want to execute a valuation for a perpetual bond. This valuation includes an amortization. 
The amortization result is wrong if no notice date was entered in class master data and other component like Deferral Value, Issue Spread or Negotiation Spread is selected.

Other Terms

Negotiation Spread, Accrue_comp, LAC, SAC, Amortization, TPM1, Perpetual Bond, Issuer Call Date, Notice Date

Solution

Implement the correction instructions or import the corresponding support package.

Amortization for a perpetual bond is only possible if 'Amortization To' has one of the following options: 'Amortization to Issuer's Next Termination Date' or 'Amortization to Date of Termination (SE)/Fixed Per. End (LO)'. If this setting is used in the system, ensure that the setting 'Notice Periods of Issuer' has been defined for the security ID in the class master data.
For amortization procedure where other component is selected we have a error message as it is only possible to amorize correctly if derived business transaction for negotiation spread is calculated for purchase.

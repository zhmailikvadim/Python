Symptom

  • There is a delay in the Printout which the spool remains for a long time in the Print Server and return after a few seconds with confirmed status to SAP.
  • The Do not query host spooler for output status option on Output Device is not selected.

Environment

  • SAP NetWeaver, all releases
  • SAP S/4HANA on-premise, all releases

Resolution

  1. Please check the information about the option 'Do not query host spooler for output status' and how it works.
    Select for no query about print requests:
    The spool work process keeps track of print requests sent to the host spool system to be able to update the status of the requests in the SAP system. To do this, the host spooler must be queried periodically.
    If a host spool system is slow or there are problems querying network or remote printers, the query may take a long time. This may affect printing performance as the spool work process cannot process any other print requests while waiting for a response from the query. In cases like this, you can deactivate querying using this switch.
    If you check the option "Do Not Query Host Spooler for Output Status", the only impact is that SAP considers the spool request to be complete when the spool is successfully passed to the OS system.

  2. See KBA 2597749 - Spool status "Waiting in host spooler".

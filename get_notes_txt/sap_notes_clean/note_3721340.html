Symptom

  • Joule Booster fails with the error "To proceed with this Joule setup, you must have an SAP Build Work Zone system registered"
  • Is it possible to proceed with Joule integration without using SAP Build Work Zone?
  • Which SAP Build Work Zone subscription plan should be used for Joule integration?

"Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental."

Environment

  • SAP BTP, Cloud Foundry runtime and environment
  • Joule
  • SAP Build Work Zone, standard edition
  • SAP Build Work Zone, advanced edition
  • SAP SuccessFactors Work Zone

Resolution

  • Certain Integrated SAP Solution requires SAP Build Work Zone service should be subscribed to the subaccount and added into "Select Integrations" part of the Joule Formation.
  • Joule can only interact with one SAP Build Work Zone instance, therefore it is recommended to install Joule into an existing SAP Build Work Zone instance. If no SAP Build Work Zone instance exists user can start with "foundation" plan included in the SAP BTP Global account. Make sure that user choose the same plan for SAP Build Work Zone, standard edition for the application and service.
  • Joule uses the navigation service component of SAP Build Work Zone, standard edition to resolve intent based navigation targets and configure additional content providers. The content needs to be exposed by the SAP product (source system) and consumed by SAP Build Work Zone (target system).
  • All SAP BTP Global accounts already include entitlements for the foundation plan of SAP Build Work Zone, standard edition. For applications which leverage SAP Build Work Zone for Joule integration, see Integrating Your SAP Solution, in case user already have a subscription to SAP Build Work Zone, advanced edition or SAP SuccessFactors Work Zone, user can add one of these to the Joule formation by updating the current Joule formation, see Run the Joule Booster.

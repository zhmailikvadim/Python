Symptom

The generation of a checklist instance contains specific logic for the CreatedDateTime timestamp. Unfortunately, the current implementation does not include time zone information. As a result, the CET time zone is assumed when creating the checklist instance via Proaxia FSM Cloud Connector. This approach is incorrect when the actual time zone differs, leading to an incorrect date and time being displayed in FSM.

Other Terms

FSM, E4C, Connector, Checklist, Instance

Reason and Prerequisites

This is caused by missing time zone information when creating the 'CreatedDateTime' time stamp.

Solution

Please, implement this correction instruction.

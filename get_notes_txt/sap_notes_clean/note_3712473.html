Symptom

In a variant of the Clinical Process Builder (CPB), you change the address on the Next of Kin  subscreen. After this change, the system may issue error message AM 215 "Region &2 is not defined for country &1". It is not possible to save the changed data. The region is not available in the address of the dependent and therefore cannot be adjusted.

Other Terms

AM215, AM 215, region, country, address, next of kin, CPB, Clinical Process Builder, NV2000, SAPLNBUPA_PNOK_SCR, screen 0100, RNDADDR-REGION, RNDADDR-REGION_TXT, 00058, 00 058, TNDCV

Reason and Prerequisites

Reason: Check of regional structure using BAS (Business Address Services); the "Region" field is not visible for the next of kin.

Prerequisites:

  • The business function ISH_BP_OM "SAP Patient Management Rearchitecture BP/OM" is active in your system. This business function is available as of SAP enhancement package 4 for SAP ERP 6.0, Industry Extension Healthcare (IS-H 604).
  • You have assigned a region to the cities in the regional structure in the Implementation Guide.
  • The change of the address leads to a different region according to the maintained data in the regional structure.
  • You change the address manually, that is: You overwrite the data. 

If you select the location using the input help, the error does not occur. The system can automatically copy the region assigned to this city.

Solution

Import the specified Support Package.

If the business function ISH_BP_OM 'SAP Patient Management Rearchitecture BP/OM' is active in your system, the Region field is now available on the Next of Kin subscreen regardless of the country version. In this way, you can also adjust the region manually if you make a manual change.

If you now manually change the address in such a way that this leads to a different region without adjusting the region, the system issues error message 00 058 "Entry <country> <region> does not exist in T005S" (check your entry).

If you have generally not assigned any regions in the regional structure in your system and you do not want to use the Region field in the next of kin's address, you can hide the field using the screen modification function in SAP Healthcare - Industry-Specific Components for Hospitals. To do this, use the IMG activity Maintain Screen Modification (transaction ON04) to create an entry with the following values:

  • Institution: * or a specific institution
  • Program: SAPLNBUPA_PNOK_SCR
  • Screen: 0100
  • Set the two fields RNDADDR-REGION (Region) and RNDADDR-REGION_TXT (Text for Region) to inactive.

As long as the corresponding support package is not implemented in your system, you can use the input help to select the location again after the system has sent message AM 215. As a result, the system adopts the region and you can save the data.

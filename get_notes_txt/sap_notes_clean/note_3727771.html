Symptom

Container columns are dislocated in the configurator when the product is read-only
Even when actions are not available for the container, when it's read-only, the column "actions" is displayed.

Environment

SAP SALES CLOUD CPQ

Reproducing the Issue

  1. Have a container with no actions
  2. Edit the product with the container
  3. Add to the quote
  4. View the product

Cause

Bug in the application.

Resolution

  • Tentative fix version is CPQ-2605 release.
  • As a workaround, actions can be enabled for the container attribute and the container will appear normally in the configurator view.

Keywords

container, miss, aligned, attribute, quote, view, read-only, dislocated, layout, configurator, CPQ

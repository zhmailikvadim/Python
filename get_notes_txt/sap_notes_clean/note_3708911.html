Symptom

  • In EL09 transaction, a dump occurs with the error message: S000() length < 7 MESSAGE_TYPE_X SAPLEMSG.
  • The issue persists even after applying SAP Note 3501179.

Environment

  • SAP for Utilities as Part of SAP ERP, release IS-UT 600 and higher.
  • SAP S/4HANA, release IS-UT 801 and higher.

Reproducing the Issue

Run transaction EL09.

Cause

The issue is caused by insufficient enqueue table size, leading to lock table overflow during the execution of EL09 transaction.

Resolution

  1. Increased the value of the MAX_LOCKS parameter in enq/server/schema_0 to 1.000.000 
  2. Increase the enqueue table size using the parameter enque/table_size.
  • This parameter defines the size of the lock table in KBytes.
  • When running Standalone Enqueue Server, the value shown on RSPARAM or RZ11 should not be considered.
  • Instead, check the value in the dev_enqsrv trace file, written when the Enqueue Server starts:

EnqLockTableCreate: create lock table (size = ).

  • Modify the profile where the Enqueue Server is defined and set the parameter accordingly.
  • Restart the Enqueue Server to apply the changes.
  1. Refer to SAP Note 1565578 for detailed instructions on increasing the enqueue table size.
  2. Remove the shared lock wait time setting to prevent further locking issues:
  • Execute report SAP_RSADMIN_MAINTAIN in transaction SE38.
  • Enter TSN_SHARED_LOCK_WAIT in the object field.
  • Select the "DELETE" radio button and press F8.
  1. Test the scenario after applying the changes to ensure the issue is resolved.

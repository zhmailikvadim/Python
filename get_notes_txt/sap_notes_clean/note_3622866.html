Symptom

SAP NetWeaver ABAP DBACOCKPIT 'SQL Server Setup Check' monitor shows following exception message:

DB Collector job is not enabled for <sid>

"Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental."

Environment

SAP NetWeaver ABAP systems running on Microsoft SQL Server 20xy

Reproducing the Issue

Execute the 'SQL Server Setup check' monitor:

  1. Goto transaction DBACOCKPIT
  2. Doubleclick Configuration
  3. Doubleclick SQL Server Setup Check

see the right side pane Status Messages 

Cause

Job name SAP_<sid>_<SID>_MSSQL_COLLECTOR appears not to be scheduled SAP NetWeaver ABAP DBACOCKPIT 'Scheduled SQL Agent Jobs' monitor.

Execute the 'Scheduled SQL Agent Jobs' monitor monitor:

  1. Goto transaction DBACOCKPIT;
  2. Doubleclick 'Backup & Recovery'; 
  3. Doubleclick 'Scheduled SQL Agent Jobs'.
  4. Check if job name SAP_<sid>_<SID>_MSSQL_COLLECTOR exists.

Resolution

Proceed as follows:

1.

  • Open the SQL Server Management Studio (SSMS);
  • Navigate to 'SQL Server Agent';
  • Expand Jobs;
  • Create the new job   SAP_<sid>_<SID>_MSSQL_COLLECTOR.


2.

  • Login to <sid> ABAP system;
  • Execute transaction DBACOCKPIT;
  • Go to Configuration;
  • Go to SQL Script Maintenance;
  • Click Execute all;
  • Click Execute SQL script(s).

3.
In case such SAP_<sid>_<SID>_MSSQL_COLLECTOR job already exists in SSMS 'SQL Server Agent', it is recommend to delete the job, to recreate it, as explained above, and to perform a full stop/restart of the Microsoft SQL Server DB instance, of the SAP instance(s) and the OS.

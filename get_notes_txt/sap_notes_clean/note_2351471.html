Symptom

Read operations on table VBFA (SD document flow) with the document number of the predecessor document (VBELV) show a bad performance in S/4HANA.

Other Terms

S/4HANA, VBFA, Performance, document flow

Reason and Prerequisites

To improve the performance of selections on the SD document flow table a new secondary index on field VBELV is provided.
Compare note 2160391 for general considerations about table indexes on HANA.

Solution

Please download the note and execute the report NOTE_2351471. This will install the correction in the note.

After successful implementation of the note a new Secondary Index '1HD' on field VBELV will be available for table VBFA.

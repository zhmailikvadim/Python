Symptom

After reconfiguration, the following services don't start automatically and require manual intervention:

  • sapb1servertools-jobservice.service
  • sapb1servertools-license.service
  • sapb1servertools-servicelayercontroller.service
  • sapb1servertools-analytics.service

Reproducing the issue

Prerequisites：

  • You are using SAP Business One 10.0, version for SAP HANA
  • You have installed various server components

Scenario:

  1. Reconfigure your server components by running /usr/sap/SAPBusinessOne/setup.
  2. After the reconfiguration is complete, check the status of a service using the command:

    systemctl status <service_name>

Expected behavior:

The service is active.

Actual behavior:

The service is inactive.

Cause

Program Error

Solution

SAP intends to provide a feature package or patch to solve the symptom that is described in this SAP Note. The section References below lists the related feature package or patch once they become available. Be aware that References can only be confirmed at feature package or patch release date. SAP delivers feature packages or patches only for selected releases at its own discretion, based on the business impact and the complexity of implementation.

Workaround:

Start the service by running the following command:

systemctl start <service_name>

Other terms

Symptom

  • Some error messages related to the Date dimension would appear when saving a Seamless Planning model in SAC

Environment

 

  • SAP Analytics Cloud 2025.21.11
  • Datasphere 2026.2.2

    Reproducing the Issue

    1. Log onto the SAC
    2. Log onto the Datasphere
      => Assign the Datasphere user to space A
    3. Navigate to Modeler and start creating an empty model
    4. On the "Data Storage Location" panel, select the Data Storage Location as SAP Datasphere, select the Datasphere space as A and click on Create
    5. On the following newly created planning model, select the Date dimension and open the Date dimension table
      => Observe that the message "An error occurred while loading the members of the dimension." would appear
    6. Click on the Refresh button on this Date dimension table page
      => Observe that the error message "An error occurred while loading the members of the dimension." would still appear
    7. Go back to the Model Structure workspace, add any measure into the model and save the model
      => Observe that some error messages related to the date dimension would appear

    Cause

    The issue is caused by a defect in the product.

    Resolution

    • It is recommended initializing the space first using the modeler flow before performing an import transport of the seamless planning model.
    • This issue has been addressed in SAC 2026 QRC1 update with version 2026.2

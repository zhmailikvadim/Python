Symptom

  • When running a Web intelligence document based on a ".unx"universe converted from ".unv", the following error appears: 

"Query script generation failed. See your administrator. while trying to invoke the method com.businessobjects.mds.datafoundationsecurityaccess.datasecurityoptions.DataSecurityOptions.getRowRestrictionAggregationMode() of a null object loaded from field com.businessobjects.mds.security.DataFoundationSecurityAccessManager.dso of an object loaded from local variable 'this' "

  • The converted universe contains security restrictions (data security profile).

Environment

  • SAP BusinessObjects Business Intelligence platform 4.3
  • Web Intelligence.
  • Information design tool.

Reproducing the Issue

  1. Open Information Design Tool (IDT).
  2. Convert a ".unv" universe that has row level security into ".unx" format.
  3. Open WEBI and create a new report based on the ".unx" universe.
  4. Drag any object and run query.
    >>Below error occurs: "Query script generation failed. See your administrator. while trying to invoke the method com.businessobjects.mds.datafoundationsecurityaccess.datasecurityoptions.DataSecurityOptions.getRowRestrictionAggregationMode() of a null object loaded from field com.businessobjects.mds.security.DataFoundationSecurityAccessManager.dso of an object loaded from local variable 'this' ".

Cause

  • Converting universe into ".unx" format creates metadata objects such as Data security Options Eg: "BSO.dso" and "DSO.dso".
  • The "DSO.dso" objects are getting corrupted when converting the universe, causing both IDT and WEBI to return the below exception:  
    "null object loaded from field com.businessobjects.mds.security.DataFoundationSecurityAccessManager.dso".

Resolution

  1. Open Information Design Tool, click Window > Security Editor and logon as administrator.
  2. Under "Universes / Profiles", expand the ".unx" which was converted.
  3. Double-click on the Data security profile.
  4. Change any of the settings inside the Data security profile (eg: Multiple SQL statements for each measure)
  5. Click Okay then save the universe.
  6. Test to create webi with this ".unx" with the user again, the issue should be resolved.
  7. Logon Security Editor in IDT as administrator again and roll back changes made in Step4.
  8. Save the universe again.

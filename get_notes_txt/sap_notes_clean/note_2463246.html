Symptom

After releasing the forecast from Demand planning to Supply Network planning, you notice incorrect start and end dates for requirements.

Environment

  • SAP Supply Chain Management (SCM)
  • SAP Enhancement package for SAP Supply Chain Management
  • SAP Supply Chain Management Demand Planning (SCM-APO-FCS)

Reproducing the Issue

  1. Release the forecast from demand planning to SNP using transaction /SAPAPO/MC90. Note the horizon start date to end date.
  2. Check the released forecast in transaction /SAPAPO/RRP3.
  3. Double click on any forecast requirement in elements tab.
  4. Notice the Start and End date of the released forecast. Does this mean the forecast requirement span from start to end date? Will the incoming sales order consume the forecast as per this start and end date?

Cause

After you double click on any forecast order, double click on requirement, this basically shows that the forecast creation (release) start date and end date.

These dates correspond to the start(From) and end (to) date mentioned in the /SAPAPO/MC90 variant, where forecast was available in forecast keyfigure(check in Demand Planning planning book).

The actual availability of the forecast order can be seen in forecast tab of /SAPAPO/RRP3.

Resolution

Check the forecast detail tab for checking the span of forecast requirements.

Keywords

PIR, GT_IO, /SAPAPO/RTSOUTPUT_FCST, /SAPAPO/SAPLRRP_FRAMES, /SAPAPO/SDP94, DP, SNP, /SAPAPO/SNP94

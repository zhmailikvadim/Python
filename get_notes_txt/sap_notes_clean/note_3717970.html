Symptom

  • Continuous SYSTEM_NO_ROLL dumps observed in the BW system every minutes, caused by repeated failures of the job ODQ_TQ_JOB.
  • The job ODQ_TQ_JOB is unexpectedly triggering at a 5-minute frequency instead of the previously scheduled 30-minute interval.
  • Runtime error SYSTEM_NO_ROLL occurs with the message "CL_ABAP_GZIP==================CP" and "COMPRESS_BINARY".
  • Job logs indicate the termination occurred in the ABAP program "CL_ABAP_GZIP==================CP" during the execution of the method COMPRESS_BINARY.
  • The ODQ_TQ_JOB fails to compress entries in the RSPMLOG table, resulting in SYSTEM_NO_ROLL errors.

Environment

  • SAP NETWEAVER/BW 7.5 all SP levels

Reproducing the Issue

  1. Check the job log using transaction SM37 for the job name ODQ_TQ_JOB.
  2. Check runtime dumps using transaction ST22.

Cause

  • Regular RSPM housekeeping was not performed, and the daemon responsible for running the ODQ job was inactive in the system until recently.
  • Since 'RSPM Reorg' has not been executed , leading to accumulation of millions of entries in the RSPMLOG table, including entries dating back to entries were not compressed for example (2021), prevents the ODQ job from compressing these entries, causing SYSTEM_NO_ROLL errors.

Resolution

  1. Verify the number of entries in the RSPMLOG table for specific time ranges using the TIMESTAMP field and selecting CATEGORY not equal to 'RSPM_COMPRESSED'.
  2. Run the report `RSPM_COMPRESS_RSPMLOG` multiple times, starting with older dates were not compressed marked in 'RSPM_COMPRESSED' : 
  • Set the TO DATE parameter to older date not compressed for example (01.04.2021).
  • Increment it by one month for subsequent runs.

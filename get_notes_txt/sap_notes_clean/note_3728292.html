Symptom

During the onboarding process in SAP SuccessFactors, specifically at the “Review New Hire Data” step, HR users are required to select a Custom Manager for the new hire.

Within the Job Relationships section, the Custom Manager relationship type currently allows HR to search across the entire user population. This broad search increases the risk of selecting an incorrect individual.

 

Environment

SAP SuccessFactors Onboarding

Reproducing the Issue

  1. Recruiters attempt to select a custom manager for the new hire
  2. Access the list of users and notice that entire user population in the system
  3. This can lead to inconsistent or incorrect selection of custom managers

Cause

 The system’s current design does not support restricting the candidate list in Job Relationship search fields (including Custom Manager, Matrix Manager, etc.) to a predefined group of users.

Job Relationship picklists pull from the entire active user population, and no configuration option exists to:

  • Limit the search to a specific user group
  • Filter only employees tagged as Custom Managers
  • Restrict visibility based on permission groups or dynamic roles

 

Resolution

This requirement cannot be achieved with current configuration capabilities in SuccessFactors. Filtering the Custom Manager search field to display only users designated as Custom Managers is not supported.

There is no workaround within standard configuration to implement this restriction.

Recommended Action

If restricting the search field is critical for your business process, SAP recommends submitting an Enhancement Request through the Influence & Adopt portal:

  • Influence Portal

This allows SAP Product Management to review and potentially include this feature in a future release.

Symptom

To address the security issue CVE-2025-66516 with a low version of tika libraries (<3.2.1).

Environment

sap commerce cloud + all solr version

Resolution

In this case manual step will be required for customers, but only if customer already:

  • enabled langid module in solr.xml (language detection module is not enabled by default) - https://solr.apache.org/guide/solr/latest/indexing-guide/language-detection.html
  • enabled TikaLanguageIdentifierUpdateProcessorFactory in solrconfig.xml (by default solrconfig.xml does not contain this configuration) - https://solr.apache.org/guide/solr/latest/indexing-guide/language-detection.html#configuring-tika-language-detection

If above conditions are met, 1 line in solrconfig.xml will need to be changed

BeforeAfter
<processor class="org.apache.solr.update.processor.LangDetectLanguageIdentifierUpdateProcessorFactory">
  ...
</processor>
 
<processor class="de.hybris.platform.solr.update.processor.SAPTikaLanguageIdentifierUpdateProcessorFactory"> ... </processor>

If you do not have any such configuration in solrconfig.xml, you can ignore the changes.

Keywords

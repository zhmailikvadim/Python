Symptom

The system is incorrectly picking the clearing date as a date other than the posting date in the clearing document.

Environment

  • Financial Accounting (FI)
  • SAP ERP
  • SAP ERP Central Component
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA Finance
  • SAP S/4HANA

Reproducing the Issue

1. Access transaction FB1D
2. Posting a clearing document on 21.01.2026
3. Checking the clearing date in this clearing document, it is updated as 22.01.2026.

Cause

The system logic is designed to compare the posting dates among all cleared and clearing documents when multiple items are cleared. It will then update the latest posting date as the clearing date. In your case, the latest posting date is 22.01.2026, which has been accordingly updated in the field BSEG-AUGBL.

Resolution

This is correct system behavior.

See Also

Please note that the functionality of the Clearing Date field in transaction codes like FB1D, FB1S, and FB1K, the field is labeled as "Clearing date," but technically, its designation is BKPF-BUDAT. Thus, it functions as a posting date within the system. 

Keywords

Symptom

  • The system allows Stock Transfer Order creation even when stock is unavailable in the supplying plant against the issuing storage location.
  • You want to activate the ATP check for STO document types.  

Environment

  • SAP ERP Central Component
  • SAP ERP
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA

Reproducing the Issue

  1. Create a PO with quantity exceeding the available stock in the supplying plant.
  2. Perform an availability check for the PO.
  3. Observe that the system allows STO PO creation despite insufficient stock.
  4. Post outbound delivery for one PO.
  5. Perform availability check for another PO, and observe the updated stock quantity reflecting the posted delivery.

Cause

Missing configuration.

Resolution

Follow the configuration as below. 

  1. Enter transaction code OVZ9.
  2. Use availability check 02 + checking rule A.
  3. Change the flag 'With Stock Transport' from Exclude to X or A, depending on whether requirements from stock transport orders, stock transport scheduling agreements, and/or stock transfer requisitions should be included in the availability check.

If you want system to raise error while putting PO if stock is not available in the supplying plant, Configure message number 06 450 to be an error message using the following steps:

a. Navigate to transaction code OLME.

b. Go to Environment data.

c. Select Define Attributes of System Messages.

This message can be configured to be an error in accordance with the your business needs.

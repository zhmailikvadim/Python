Symptom

For situation where, while intiating a Data Transfer the scheduling entry is not visible in the Data Transfer facet of Subcription application. However the Data Transfer as such is in progress. In such cases it is difficult to track or if needed re-processing of Data Transfer is not possible.

This could happen if the scheduling entry in table DFS_IL_SCHD_HDR is not updated correctly (like missing time stamp or status).

The below mentioned utility report could be used to update the scheduling entry in such emergency situation.

Report: DFS_DIS_IL_SCHD_UPD_UTIL

Transaction Code: DFS_IL_SCHD

Other Terms

Defense Disconnected S/4, DFS_IL_SCHD_HDR, DFS_IL_SCHD, Data Transfer

Solution

Implement the Correction Instruction provided with the NOTE.

Symptom

  • You have received an email notification regarding Change in Load Balancer Certificate Renewal Process, for Cloud Integration and SAP Integration Suite.
  • You would like to understand the actions to be performed at your end.

Environment

  • Cloud Integration
  • SAP Integration Suite

Cause

Adjustment to Certificate Renewal Process: SAP has established an automated, standardized process for Load balancer certificate renewal as part of our continuous efforts to improve security and efficiency.
As a result, effective Oct 1, 2024, SAP will discontinue the distribution of certificate chains, including Server, Intermediate, and Root certificates. i.e., the upcoming certificate renewals will no longer be notified to you via the email notification service. SAP will ensure that the renewal happens automatically before the certificate expiry date.

For Neo, the same changes will be implied from Feb 2025.

Resolution

Below are the actions to be performed at your end to adapt to the new certificate renewal process for Cloud Integration.

  1. Adaptation to Changes in Certificate Chain Distribution: To ensure trust for the SAP load balancer server certificates, please make sure the "DigiCert Global Root G2" and "DigiCert TLS RSA4096 Root G5" are imported into sender system as Trust anchor. We will promptly notify you of any changes to Root CA certificates. If there is any change to the Root certificate, it will be one of the CAs listed in SAP Note 3472211 - BTP Trust Store: Collection of TLS Server Root CA certificates used by SAP BTP.

    Any Sender System communicating with your Cloud Integration tenant via HTTP(S) protocol must trust Root CA for establishing SSL trust. This action is essential for every sender system or HTTP-based client accessing Integration Flow endpoints on Worker Nodes. To make sure that the TLS handshake is successful, it is important to import the respective Root Certificate into your client trust configuration.

  2. Depend only on Root Certificate: Since the usage of server certificate for trust enablement is not required, please update your SSL communication setup to depend solely on the Root certificate. Using the Root certificate instead of the Intermediate / Server certificate will help maintain seamless communication with SAP services and ensure compatibility with our systems.
  3. Mail Subscription: For any newly onboarded Cloud Integration tenant administrator, subscribe to notifications as per:
    SAP KBA 2900069 - How S-users manage their own Cloud Notification subscriptions - SAP for Me

NOTE : For queries regarding MBC (Multi Bank Connectivity) , Please raise a ticket with component LOD-FSM-INT .

FAQ

  1. Who is the target for this notification?
    A: Cloud Integration/iSuite system administrators.

  2. Why is this change happening?
    A: To improve security and streamline operations, the certificate renewal process (Design-time & Run-time) will be automated by SAP. This will ensure seamless certificate updates and eliminate manual errors during certificate exchanges.

  3. Where to download Root CA?
    A: It's recommended to download the Root CA certificates from the Github: https://github.com/sap-software/btp-trust-store/tree/main/required/pem as mentioned in SAP Note 3472211.

  4. How to import to S4 system?
    A: Please refer to SAP KBA 3238733 - How to Import a Certificate Into a STRUST PSE's Certificate List In a NetWeaver AS ABAP

  5. What is the impact on Design time and Runtime?
    A:
    • Design Time : There is no action to be performed at customer end.
      SAP will update and maintain design time certificate for seamless operations.

    • RunTime : We do not expect any disruptions impacting runtime scenarios, if below requirements are fulfilled.
      Please ensure to import the respective Root Certificate into your client trust store configuration.

      Any Sender System communicating with your Cloud Integration tenant via HTTP(S) protocol must trust Root CA for establishing SSL trust. This action is essential for every sender system or HTTP-based client accessing Integration Flow endpoints on Worker Nodes

      Currently, Server certificates are authenticated by "DigiCert Global Root G2" or "DigiCert TLS RSA4096 Root G5".We will promptly notify you of any changes to Root CA certificates. If there is any change to the Root certificate, it will be one of the CAs listed in the above KBA.

  6. Understanding of Root CAs?
    A: The technical reason why only a Root CA is sufficient is that the Root CA is the anchor of trust in the SSL/TLS certificate chain. When a Root CA is trusted by a client system (such as a web browser or application), it implicitly trusts all certificates issued by that Root CA and any intermediate CAs beneath it. This trust hierarchy allows the client to verify the authenticity and integrity of the server's certificate, ensuring secure communication.

    Anchor of Trust: The Root CA is the top-level authority that is inherently trusted.
    Chain Verification: Trust in the Root CA enables verification of the entire certificate chain (including intermediate and server certificates).
    Implicit Trust: Trusting the Root CA means all certificates issued under it are also trusted.

  7. Is it required to explicitly add the server certificate into sender’s system trust store?
    A: It is advisable to refrain from adding server certificates to the client system's truststore. On SAP BTP level we only consider validating the Root certificate of the chain.
    By discontinuing trust in Intermediate/Server CAs and relying solely on Root CAs, reduce the complexity of your certificate management.

  8. What is the date of this change?
    A: Stay informed about any changes to Root CA certificates by subscribing to SAP notifications. Ensure that your trust store is promptly updated with the Root CA certificate(s) to avoid disruptions. This change will be effective starting from October 2024, although the exact date may vary depending on which data center / environment your tenant belongs to.

    For NEO environments the changes will be applied on Feb 2025.

  9. How can I get notified of changes related to Cloud Integration?
    A: SAP sends automated emails from time to time with updates about your Cloud service. System Administrators must subscribe to receive these emails. For more details on the steps involved, refer to the below resources:
    • SAP KBA 2900069 - How S-users manage their own Cloud Notification subscriptions - SAP for Me
    • SAP KBA 2765458 - Cloud Integration - Availability and Notifications

  10. Who should subscribe to the automated emails?
    A: This is a matter for each customer to decide. Certainly the Administrator(s) should be subscribed. End users/developers may also benefit from this. Business leaders should also be aware of major changes so they can ensure adequate planning and testing is put in place.

  11. Will there be a change in the Root CA Certificate? e.g., from DigiCert Global Root G2 / DigiCert TLS RSA4096 Root G5 to something else?
    A: Rest assured, we will promptly notify you of any changes to Root CA certificates. If there is any change to the Root certificate, it will be one of the CAs listed in the SAP Note 3472211. As customers, please make sure to subscribe to SAP Cloud notifications, following SAP KBA 2900069 - How S-users manage their own Cloud Notification subscriptions - SAP for Me

    The new Root CA Certificate in the future can be found in the list of trusted CAs in SAP Note 3472211 - BTP Trust Store: Collection of TLS Server Root CA certificates used by SAP BTP

  12. I'm seeing certificates like "CN=*.it-cpixxx-rt.cfapps.xxxx.hana.ondemand.com" or "CN=<Subaccount ID>.cpi.hana.ondemand.com" in STRUST, SSL Client PSE. Are they impacted?
    A: They should be removed from STRUST, SSL Client PSE. See SAP KBA 3193513 - Cloud Integration Certificate Is Expiring in STRUST

  13. Can I ask SAP to examine my system, to make sure a seamless transition?
    A: Please reach out to your CEE/CSP to arrange professional service. Such service request cannot be fulfilled by SAP Support channel. (i.e., LOD-HCI-PI-OPS component WON'T be able to check your system configuration.)

Symptom

  • After enabling concurrent employment, users cannot access the Variable Pay Individual View when switching to their old/inactive employment profile.
  • The Individual View opens but displays a blank screen/no columns or values for the inactive profile.
  • Compensation and bonus statements linked to the old profile are accessible, but the Variable Pay Individual View is not.
  • No error message is displayed.

Environment

  • SAP SuccessFactors Compensation
  • Variable Pay

Reproducing the Issue

  1. Enable Concurrent Employment and ensure a user has a previous employment set to inactive.
  2. As the employee, switch to the inactive employment profile via the concurrent employment switch.
  3. Navigate to an employee Variable Pay Individual View for a previous year.
  4. Select a Compensation Statement
  5. Observe that the Individual View displays a blank page (no columns/values).

Cause

The issue is caused by field-based permissions (FBP) configured in the Variable Pay template, which restrict access to the Individual View for inactive employment profiles.

Resolution

  1. Temporarily disable field-based permissions (FBP) in the affected Variable Pay template to validate that the Individual View loads for the inactive profile.
  2. Retest by switching to the inactive employment profile and opening the Variable Pay Individual View; confirm that data is now displayed.
  3. If FBP must remain enabled, review and adjust FBP so that the employee access group has read permission to all fields shown in the Individual View across inactive employments. If current configuration cannot meet the requirement.

Symptom

When SFTP sender adapter with chunk mode is used, the processed input files need to be archived to prevent the same files from being processed again as they remain in the input directory. The files will be archived within the archive directory on the SFTP server.

Other Terms

SFTP Sender, chunk mode, archive, SFTP server

Reason and Prerequisites


When the SFTP sender adapter with chunk mode is used, in the existing feature there is no option to archive the input files after processing. This causes due previously processed files are being reprocessed since they remain in the input directory.

Solution

Code changes have been made in the SFTP sender side processing to support archiving of files when chunk mode is used.

Archiving can be done by setting the parameter in the Additional Parameters table in Advanced Tab: archiveChunkModeOnSFTPServerPath to the archive directory path on the SFTP server. The default value of the parameter is null.

For example: 

Name: archiveChunkModeOnSFTPServerPath

Value: ~/sftp/archive

After setting this parameter to the correct archive directory path, the files will be successfully archived to the same path on the SFTP server after processing is completed.  

To get this correction available in your Support package you need to apply the patch matching to your support package version as per the instructions in the NetWeaver Support Package Stack guide which can be found on the SAP Service Marketplace as described in SAP Note 952402.

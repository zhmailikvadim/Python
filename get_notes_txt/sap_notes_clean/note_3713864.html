Symptom

In Medication, you use templates to create medication orders.
It is possible to determine the start and the duration of medication using the "Start Delay" and "Duration" fields.

However, these specifications always also refer to the time of the start of the order.
If an order starts at 08:00 and the delay has been set to 1 day and the duration to 3 days, this means that
that the order starts on the next day at 08:00 and the order then ends at 08:00 according to the duration 3 days later.

This is not always practical in everyday medical life.
It is better to have a duration and delay in whole calendar days, regardless of the time at which the order starts.

Other Terms

Medication Order, Medication Template, Duration in Calendar Days, Delay in Calendar Days

Reason and Prerequisites

Enhancement

Solution

This SAP Note provides the new feature "Treatment Days in Calendar Days".

You can use the new checkbox "Treatment in Calendar Days" to display the new fields "DurDayEnd" (duration at the end of the day) and
"WaiverDay" (Delayed Order Start at Start of Day).

You can then enter either a duration in calendar days and/or a delay of the start in calendar days in these two new columns independently of each other.

For more information about this new feature, see the online documentation or the system documentation for the new fields.

Note the following:
As soon as the new checkbox "Treatment in Calendar Days" is selected, the existing fields for "Delay" (plus their unit) and
"Duration (plus its unit) deactivated and its content deleted.

The same applies even if you delete the checkbox again:
In this case, the content of the new fields is deleted and the fields are deactivated.

Import the specified Support Packages to solve the problem.

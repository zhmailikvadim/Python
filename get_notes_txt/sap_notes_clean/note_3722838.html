Symptom

  • An error occurs during the creation of a Request for Quotation (RFQ) using the Manage RFQs Fiori application.
  • The error does not block the RFQ creation process but prevents output generation.
  • No output parameter is determined.

Environment

  • Materials Management (MM)
  • SAP ERP
  • Central Component SAP ERP
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA
  • SAP Fiori

Reproducing the Issue

  1. Clear the cache in the system as per SAP Note 2319491.
  2. Import XML files as per SAP Note 2248229 using a new transport request.
  3. Verify that REQUEST_FOR_QUOTATION is listed in the application object type in active status.
  4. Check and activate all parameters in the OPD transaction for RFQ.
  5. Test the RFQ creation in the Manage RFQs Fiori application (F2049).

Cause

The incorrect XML file (OPD_MM_CENTRAL_RFQ.xml) was imported, which is not relevant for the business scenario.

Resolution

  1. Download and import the correct XML file, OPD_EF_REQUEST_FOR_QUOTATION, relevant for the business scenario as per SAP Note 2248229.
  2. Clear the cache in the system as per SAP Note 2319491.
  3. Ensure that the application object type REQUEST_FOR_QUOTATION is listed in the Manage Application Object Type Activation node and is in active status.
  4. Verify and activate all necessary parameters in the OPD transaction for RFQ.
  5. Ensure that the OData Service CA_OC_OUTPUT_REQUEST_SRV is maintained and active in transaction /N/IWFND/MAINT_SERVICE, as well as the ICF service CA_OC_OUTPUT_REQUEST_SRV in transaction SICF.
  6. Ensure that the authorization objects mentioned in SAP Note 3113243 are assigned to the user, and verify that the user is not maintained as a support user.

Symptom

When creating 'Allowance'-type Benefits, they can be configured to be integrated with Pay Components; this will allow the employee to be given a particular Pay Component in their Compensation Information portlet when they claim the Allowance, but particular configurations need to be done for this, which are noted below.

Environment

SAP SuccessFactors Employee Central

Resolution

To integrate Allowance benefits with employees' Compensation Information / Pay Components, the event reasons 'PCAssignment' and 'PCAssignmentCompletion' need to be configured.
This is because, when inserting entries in the Compensation Information portlet, an “Event Reason” needs to be assigned, one for the starting claim record and one for the end claim record. The 'PCAssignment' and 'PCAssignmentCompletion' are the standard event reasons which need to be configured for this.

In order to create the Event Reasons, if they do not already exist in your environment, the following steps can be used:

  1. Go to "Manager Organization, Pay and Job Structures"
  2. Select "Create New" and select "Event Reason"
  3. First, you will need to create the "PCAssignment" event reason for the start record, and it will be associated to the 'Assignment' Event.
    1. Note that you need to have the 'Assignment' event configured in your 'event' picklist
    2. The 'Event Reason ID' should specifically be 'PCAssignment', and this should not be changed
    3. The 'Employee Status' value will depend on your configuration and requirement; if you select "Active", the system may change the previous Employee Status of the employee and insert a Job Information to reflect this change. If you leave it as "No Selection", no changes will occur compared to the employee's previous Employee Status value.
  4. You may then create the 'PCAssignmentCompletion' event reason, which will be for the end record, which is also associated to the 'Assignment' event.
    1. The 'Event Reason ID' should specifically be 'PCAssignmentCompletion', and this should not be changed

With the creation of these event reasons, when an employee makes a claim on an Allowance benefit which is integrated with Compensation Information / Pay Components, records will be created in their Compensation Information portlet; a 'start of allowance' record with the 'PCAssignment' event reason, and an 'end of allowance' record with the 'PCAssignmentCompletion' event reason.

NOTE: Since records are being created in Compensation Information because of this configuration, Compensation Information business rules will trigger if you have them configured - you may want to evaluate your business rules so that they specifically exclude / do not trigger when using the 'PCAssignment' and 'PCAssignmentCompletion' event reasons, if you do not want other data changes to occur from business rules as a result of the insertion of these records.

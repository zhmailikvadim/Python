Symptom

There is a slight difference in quantitative characteristic values in table PLMK after data migration or simply between different systems - like ECC and S/4.

Example fields and values:

  • SOLLWERT - Target Value for a Quantitative Characteristic
  • TOLERANZOB - Upper Specification Limit
  • TOLERANZUN - Lower Specification Limit
  • GRENZEOB1 - First Upper Specification Limit
  • GRENZEUN1 - First Lower Specification Limit
  • GRENZEOB2 - Second Upper Specification Limit
  • GRENZEUN2 - Second Lower Specification Limit
  • PLAUSIOBEN - Upper Plausibility Limit
  • PLAUSIUNTE - Lower Plausibility Limit
  • TOLERWEIOB - Change to Upper Specification Limit
  • TOLERWEIUN - Change to Lower Specification Limit

One system shows

1.5919999999999999E+02, while the other

1,5920000000000002E+02.

There is a marginal (0,0000000000000003) difference in this example.

Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental.

Environment

  • SAP ERP Central Component
  • SAP ERP
  • SAP Enhancement package for SAP ERP
  • SAP S/4HANA, on-premise
  • SAP S/4HANA Cloud Private Edition

Resolution

The two affected systems have different databases: one of them is on Oracle DB, while the other is - for example - on HANA DB. As per SAP Note

1118068 - Rounding effects for floating point numbers & Oracle DBs

there can be rounding effects for floating point numbers (FLTP data type) on Oracle databases.

Remark: During migration not an exact copy of the values in database tables (like PLMK) takes place, but a mapping taking into consideration any applicational and technical differences between the two system environments. This can include such acceptable rounding differences in FLTP values.

Symptom

According to the Font Awesome documentation, version 7.x introduces a breaking change

Environment

SAP Commerce Cloud, composable storefront

Resolution

  • By default, all Font Awesome icons now render with a fixed width instead of adjusting automatically
  • As a result, some icons may overlap adjacent HTML elements. So far, this behavior has been identified in two places: the Product Configuration alert icon and the legacy carousel circle icons
  • This issue can be resolved by manually adding the following in root.scss:
                    --fa-width: auto;
  • This issue will be resolved in an upcoming release of composable storefront, after which the workaround described above will no longer be necessary

  • More information about this solution can be found in the official Font Awesome documentation

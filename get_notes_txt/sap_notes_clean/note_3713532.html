Symptom

The field Product Type Group (EBAN-PRODUCTTYPE) is not populated when purchase requisitions are created by MRP Classic or MRP Live (where EBAN-ESTKZ = B).

"Image/data in this KBA is from SAP internal systems, sample data, or demo systems. Any resemblance to real data is purely coincidental."

Environment

  • SAP S/4HANA, on-premise Release Independent
  • SAP S/4HANA Cloud Private Edition Release Independent 

Reproducing the Issue

  1. Plan the material with MRP using transactions MD01N or MD01/MD02/MD03
  2. Observe that purchase requisitions are created for the material
  3. When displaying the purchase requisition created by MRP, the field Product Type Group in Material Data tab is not populated

Cause

System standard behavior.

Resolution

To ensure the Product Type Group field (EBAN‑PRODUCTTYPE) is populated for purchase requisitions created by MRP, implement the logic using the BAdI PPH_MRP_RUN_BADI, specifically the method PURREQ_BEFORE_UPDATE_ADJUST.

This enhancement allows you to adjust and enrich purchase requisition data during the MRP run.

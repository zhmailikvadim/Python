Symptom

Material code in Purchase Requisition (PR) line item is greyed out and cannot be changed, even after the linked Purchase Order (PO) line item is deleted.

Environment

  • Materials Management (MM)
  • SAP ERP Central Component
  • SAP ERP
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA
  • SAP S/4HANA Cloud Private Edition

Reproducing the Issue

1.    Create a PR using transaction ME51N.
2.    Create a PO with reference to that PR.
3.    Delete the PO line item.
4.    In ME52N, attempt to change the material code → field is greyed out.

Cause

Once the Purchase Requisition has been converted to a Purchase Order, the Order status EBAN-STATU will be updated to 'B' (PO created), even if the PO is deleted. This occurs because the deletion of a PO can be reversed, it's not a permanent deletion of the PO document.

When the Order status (EBAN-STATU) is not 'N' (Not edited), the system switches the screen layout of the PR material field to 'display only'. This is designed as a standard system behavior to maintain consistency between the PR and PO documents.

Resolution

• This is the intended system design to maintain data integrity between tables EKPO (PO items) and EBAN (PR items).
• To change the material code is not possible, delete the corresponding line item in the PR.
• Create a new PR line item with the desired material code instead of attempting to modify the existing one.

Symptom

After data archiving using SARA, although data was deleted but memory size was not reduced as expected

Environment

SAP HANA database.

Reproducing the Issue

Archiving or delete data, then check memory size.

Cause

  1. It is necessary to wait until delta merge to be finished.
  2. It is necessary to check database used memory size.

Resolution

As mentioned in KBA:
1999997 - FAQ: SAP HANA Memory

For efficiency reasons SAP HANA frees allocated memory in a "lazy" way and so the allocated memory can grow up to the available memory and the global allocation limit while the used memory remains at a much lower level.
From a memory analysis perspective we can usually focus on the used memory and assume that the allocated memory is released whenever required.

Please check HANA database used memory size instead of resident memory size. You can see this information from transaction DBACOCKPIT->Performance->Load history.
Then please specify the date and time to the time period when archiving was executed, and check "Database Used Memory" and "Database Resident Memory". If delta merge has been finished, we can confirm that "Database Used Memory" has been reduced but "Database Resident Memory" may be not changed. But this is working as design and we don't need to worry about it. Because operating system will reclaim resident memory in case it is necessary.

Symptom

TTP1100-2. INVOIC Out: Proforma determination

Default CIP2SAP implementation of proforma determination can deliver into SIDoc maximum first 15 proforma invoices: (3 proforma invoices pro RFF+AAB segment) x 5 maximum RFF segments.

Other Terms

CIP2SAP, SIDOC, INVOIC, TTP1100, TTP1100-2,

Reason and Prerequisites

TTP1100-2. INVOIC Out: Proforma determination

Reason: You need to implement own proforma determination in order to deliver different (from CIP2SAP standard) number of proforma invoices in RFF+AAB segments.

Solution

Implement correction instructions.

Solution: You need to implement the BAdI /SIE/SF_ED_BADI_INVOIC_SD_MM and in interface /SIE/SF_IF_BADI_INVOIC_SD_MM redefine method Y_FIND_PROFORMA_REF. Method have to return populated table PT_YYCOMRFF, where you have to add new entries with qualify AAB, as well as output parameter PV_PROFORMA_DEFINED that should be set into 'X' (means that proforma determination is redefined and CIP2SAP standard must not be executed).

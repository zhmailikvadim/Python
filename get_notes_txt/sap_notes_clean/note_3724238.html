Symptom

In My Timesheet (S/4HANA/Version 4) application:

  • The selected order of assignments within an assignment group was not properly retained in some cases.
  • Switching from the To Do List to the Timesheet tab, the table was shown in both display and edit mode.
  • When changing a time entry, the first column of the table was taking too much width.
  • The feature to autocopy the last end time to the start time of a new record in a day was not working.

Other Terms

F3074A, My Timesheet V4, Assignment Group, Assignment, To Do List, Timesheet, Table, Vertical, Strict layout, Copy Time, Feature, EnableCopyEndTimeToNextRow, HRFIO_B_TSH_FEATURE_ENABLER, ENABLE_COPY_ENDTIME_TO_NEXTROW, EnableDefaultClockTimes, Act As, On Behalf, EnableDefaultDraft.

Reason and Prerequisites

  • When the number of assignments exceeded 9, the sorting logic failed.
  • Table reload in edit mode was performed without destruction of the table in display mode.
  • When columns were less in the table, the Strict mode was not applied which resulted in a wider highlight column.
  • The feature: Enable Copy End Time To Next Row (View: V_HRFIO_FEAT_LST) along with BAdI method: ENABLE_COPY_ENDTIME_TO_NEXTROW (BAdI: HRFIO_B_TSH_FEATURE_ENABLER) can be now used for getting the functionality.

Solution

Please implement the automatic correction instruction attached to this SAP Note or alternatively you can implement the corresponding support pack.

Symptom

  • The Goods Movement Status in the outbound delivery (OBD) changes to "C" after running /SCWM/PRDO to set the quantity to 0 and create an outbound delivery.
  • Shipment and billing are created for OBDs with status "C" and quantity 0.

Environment

•Sales And Distribution (SD)
•Logistics Execution (LE)
•SAP R/3 •SAP R/3 Enterprise 4.7
•SAP ERP Central Component
•SAP ERP
•SAP Enhancement package for SAP ERP
•SAP Enhancement package for SAP ERP, version for SAP HANA
•SAP S/4HANA
•S/4HANA Cloud Private Edition

Reproducing the Issue

  1. Run transaction /SCWM/PRDO to change the delivery quantity to 0 and create an outbound delivery.
  2. Observe that the OBD Goods Movement Status changes to "C".
  3. Observe that shipment and billing documents are created for the OBD with 0 quantity.

Cause

The item category is configured in transaction VOV7 with billing relevance "A" (delivery-related billing document), which makes deliveries billable even when the delivery quantity is 0.

Resolution

  1. Open transaction VOV7 (Item Categories).
  2. Identify the item category used in the affected outbound deliveries.
  3. Change the Billing Relevance from "A" (Delivery-related billing document) to "H" (Delivery-related billing – no zero quantities).
  4. Save the customizing change and transport it as per your change management process.
  5. Test the process: create a delivery with 0 quantity and confirm that no billing document is created.

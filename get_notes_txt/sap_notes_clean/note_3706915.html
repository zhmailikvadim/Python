Symptom

HR Coordinators are unable to locate onboardees using the “New Hire” filter on the Onboarding Dashboard when the onboardee’s main onboarding processes are completed but compliance tasks remain pending. The onboardee can still be found using other filters such as Hiring Manager or Start Date (From).

Environment

SAP SuccessFactors Onboarding

Reproducing the Issue

  1. Initiate an onboarding process for a new hire.
  2. Complete the onboarding tasks
  3. Leave only Compliance tasks pending
  4. As an admin, navigate to Onboarding Dashboard.
  5. Select “New Hire” filter.
  6. Search for the onboardee.

Expected Result:
Onboardee is displayed under New Hire filter.

Actual Result:
Onboardee is not returned in search results under New Hire filter, but can be found using Hiring Manager or Start Date (From).

Cause

  • The “New Hire” filter is designed to display onboardees who are actively progressing through the main onboarding workflow.
  • Once the main onboarding process is completed and only compliance tasks remain, the onboardee is no longer considered an active “New Hire” in the context of this filter. Therefore, the system intentionally excludes them from the “New Hire” view.

Resolution

This is expected system behavior in Onboarding. HR Coordinators should use alternative filters such as:

  • Hiring Manager
  • Start Date (From)
  • Other applicable filters

If the customer wishes to have onboardees with pending compliance tasks appear under the “New Hire” filter, they can submit an Enhancement Request for future product consideration.

Symptom

  • ARM follow-up document in SDF scenario is not created
  • Subsequent document in ARM process is not created automatically due to error
  • The document to be created would be incomplete, details are missing and causing impletion

Environment

  • Sales and Distribution (SD)
  • Advanced Returns Management (ARM)
  • SAP S/4HANA
  • SAP S/HANA Cloud Private Edition

Reproducing the Issue

  1. Create a return order in transaction VA01.
  2. Maintain details on the Returns tab.
  3. Save the order, expecting SDF documents to be automatically created.
  4. Observe that no error log details are present. 

Cause

The 'Incompletion Messages' setting in transaction VOV8 for sales document types prevents incomplete documents from being saved during background processing. This setting halts the creation of ARM follow-up documents in the background due to incompletion.

Resolution

  1. Open transaction VOV8 and locate the sales document types created as follow-ups (e.g., SDF scenario). 
  2. Ensure that the 'Incompletion Messages' setting (TVAK-DIAFE) is not enabled.
  3. Verify that the follow-up document can now be created in an incomplete state, allowing users to fill in the required details later. 

Keywords

ARM, SDF, TVAK-DIAFE, incompletion check, incomplete, automatic follow-up, background, error log

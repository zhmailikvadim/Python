Symptom

The storage location list picker is not showing correct data on the meter create and update page

Other Terms

  • SAM
  • SSAM
  • Asset Manager
  • Meter
  • Storage Location
  • Location

Reason and Prerequisites

Reason:

 The storage location entity set is incorrect on the MeterCreateUpdate page, it should be 'Storage Locations'.

Prerequisites:

  •  BAS or Visual Studio Code with SAM MDK project
  • `patch` command (available on BAS and macOS by default) via terminal or other patching utility

Solution

We made some changes to the metadata on the MeterCreateUpdate page to resolve the issue.

Download the attached 3457112.patch file and save it in a known location, preferably adjacent to your SAM.mdkproject

  1. Open a terminal in the root of your SAM metadata (i.e. SAM.mdkproject/metadata or SAM.mdkproject/metadata/base.definitions)
  2. Run the command `patch --no-backup-if-mismatch -p4 < path/to/3457112.patch` in the terminal.
  3. For 2310, use 3457112-2310.patch
  4. Validate that changes have been applied to the following files:

For 2305 and 2310

  • patching file 'Pages/Meters/MeterCreateUpdate.page'

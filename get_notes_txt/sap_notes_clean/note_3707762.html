Symptom

  • MC$6 report reflects value 0 for Del.dt.vr1 – Del.dt.vr5 when checking on-time performance of suppliers.
  • After defining ranges in MCFC transaction, MC$6 reflects new values, but historical values remain unchanged.

Environment

  • Materials Management (MM)
  • SAP ERP Central Component
  • SAP ERP
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA

Reproducing the Issue

  1. Run standard SAP report MC$6 to check on-time performance for suppliers.
  2. Observe that Del.dt.vr1 – Del.dt.vr5 fields show value 0.
  3. Define ranges in MCFC transaction for Delivery Date Variance Profiles.
  4. Run MC$6 report again and observe that new values are reflected, but historical values remain unchanged.

Cause

  • Historical delivery performance values were stored as 0 due to the absence of variance rules before defining ranges in MCFC.
  • The system does not retroactively recalculate historical data; new variance logic applies only to data posted after the configuration change.

Resolution

  1. Configure Delivery Date Variance Grid:

a. Navigate to SPRO > Materials Management > Purchasing > Statistics > Define Grid for Delivery Date Variance.

b. Set interval levels for variance intervals (Del.dt.vr1 to Del.dt.vr5) based on business requirements. Example configuration:

  • From -999 to -20 days (Del.dt.vr1)
  • From -19 to -10 days (Del.dt.vr2)
  • From -9 to 10 days (Del.dt.vr3)
  • From 11 to 20 days (Del.dt.vr4)
  • From 21 to 999 days (Del.dt.vr5)
  1. Assign the configured grid to the relevant purchasing organizations.
  2. Ensure statistical delivery dates are maintained in purchase orders.
  3. Verify that goods receipt postings include proper delivery date references, as the system calculates variance between statistical delivery date and actual goods receipt date.

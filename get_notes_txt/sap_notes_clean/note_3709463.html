Symptom

Condition update functionality is not working for contracts, inquiries, scheduling agreements, quotations.

Environment

  • SAP R/3
  • SAP S/4HANA
  • S/4HANA Cloud Private Edition

Reproducing the Issue

  1. Set up a condition in V/06 for which the condition update flag (T685A-KOUPD) is set in V/06.
  2. In VK11 maintain a condition record.
    Under Goto > Additional data maintain any of the following fields:
    Max.condition value
    Max.number.of.order
    Max.cond.base value

  3. Create a contract, inquiry or quotation with condition record previously maintained.
  4. Verify S071.

Cause

Due to standard system design, the condition update functionality is not supported for contracts, inquiries, scheduling agreements, or quotations. This functionality is only available for sales orders, credit/debit memos, and billing documents. 

During pricing if a non supported document type is being used T685A-KOUPD (condition update flag) is initialized:

LV61AA12
...
   * Only set "condition flag" (Konditionsschalter) for orders, credit
   * memos, debit memos and invoices
           IF komk-vbtyp = 'U' OR ( komk-vbtyp NA vbtyp_anfo AND
              komk-vbtyp NA vbtyp_auft AND komk-vbtyp NA vbtyp_fakt ).
             t685a-koupd = space.
           ENDIF.
...

Resolution

In standard the condition update is only available for sales orders credit/debit memos, and billing documents.

See Also

Refer to SAP Note 0000615370 - Condition update: Functions and restrictions.

Symptom

This issue may occur in the following situations, but is not limited to these:

Prerequisite: The currency used on header level (EKKO-WAERS) is different from the company currency (T001-WAERS).

Possible situation:

  1. Below error shows during PO creation/change:
    "Translation result XXX is too big - Message No. SG106"
    However, the result should not be such large based on the conversion rate maintained on header level.
  2. During release strategy determination with multiple currencies included, the release strategy is not triggered as expected but based on the rate and net value, the release strategy should be determined. 

Environment

  • Materials Management (MM)
  • SAP ERP Central Component
  • SAP ERP
  • SAP enhancement package for SAP ERP
  • SAP enhancement package for SAP ERP, version for SAP HANA
  • SAP S/4HANA

Cause

The currency exchange rate on the header level is entered using a 1:1 ratio between the two currencies, but in OB08 the ratio between the two currencies is not 1:1. For example, in OB08, the field Ratio (from) or Ratio (to) is 100. 

Resolution

Make sure the ratio of the currency exchange rate on header level is consistent with the one maintained in OB08. 

If not consistent, either re-maintain the ratio in OBBS and OB08, or adjust the conversion rate used in PO based on the ratio in OB08.

Keywords

ratio, TFACT, FFACT

Symptom

When creating a Purchase Contract with reference to a Purchase Requisition (PR) in the Fiori Process Purchase Requisitions V2 app (F1048A), the GR-Based Invoice Verification flag is not set automatically.

Environment

  • SAP Fiori
  • SAP S/4HANA 2023 onwards
  • SAP S/4HANA Cloud Private Edition

Reproducing the Issue

  1. Open the Process Purchase Requisitions V2 app
  2. Enter a PR number
  3. Select the PR and click on the "Create Contract" button
  4. Enter all the necessary details in the contract
  5. Go to the GR/IR Control tab at item level
  6. The GR Based Invoice Verification flag is not set.

Cause

This is a technical limitation and the current system design.

Resolution

As a workaround, the Badi MM_PUR_S4_CTR_MODIFY_ITEM can be used to set the flag. 

The field INVOICEISGOODSRECEIPTBASED within the parameter PURCHASECONTRACTITEMCHANGE_NEW can be set to X to achieve the desired functionality.  

Please note: This is considered a custom modification and is not covered by SAP Standard Support.
